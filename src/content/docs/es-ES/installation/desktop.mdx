Esta guía cubre la instalación y el uso de HagiCode Desktop. Es el cliente de escritorio oficial y mantiene la ruta de configuración local predeterminada más los controles diarios en un solo lugar.

:::tip[Método recomendado]
El escritorio es la ruta local predeterminada para usuarios individuales y desarrolladores porque:
- el instalador mantiene el flujo predeterminado en una ruta guiada
- Las dependencias y servicios se gestionan en la aplicación.
- La interfaz de usuario facilita la inspección del estado y las operaciones.
- La gestión de versiones integrada facilita el cambio.
- los datos permanecen locales por defecto
:::

## ¿Qué es el escritorio HagiCode?

HagiCode Desktop es la aplicación de escritorio oficial de HagiCode, diseñada para sistemas Windows, macOS y Linux. Incluye las funciones principales de HagiCode y el instalador mantiene la ruta de configuración predeterminada en un flujo guiado.

### Principales ventajas

- **Instalación con un clic**: Complete todos los pasos de configuración automáticamente a través del asistente de instalación
- **Preparación automática del tiempo de ejecución**: el instalador predeterminado ya incluye el tiempo de ejecución que necesita, por lo que no es necesario instalar otro. `.NET` tiempo de ejecución primero
- **Gestión de servicios**: Gestión integrada del servidor HagiCode, inicia y detiene servicios fácilmente
- **Cambio de versión**: soporte para administración de múltiples versiones, cambie libremente entre diferentes versiones
- **Operación local**: Todos los datos se almacenan localmente, protegiendo su privacidad
- **Actualizaciones automáticas**: detecta y descarga automáticamente la última versión

### Escenarios aplicables

El escritorio es una buena opción cuando:
- Desarrolladores individuales que quieran utilizar HagiCode localmente
- Usuarios que necesitan utilizar HagiCode sin conexión
- Usuarios no técnicos que quieran simplificar el proceso de instalación.
- Desarrolladores que necesitan cambiar frecuentemente entre diferentes versiones

### Otros métodos de descarga

También puede obtener el instalador de HagiCode Desktop en los siguientes canales:

1. **Página de lanzamiento oficial**: Visita [Lanzamientos de HagiCode GitHub](https://github.com/HagiCode-org/releases) para descargar la última versión

### Elija la versión correcta

Seleccione el instalador correspondiente a su sistema operativo:
- **Windows**: descargue el `.exe` instalador
- **macOS**: descarga el `.dmg` instalador
- **Linux**: Descarga el `.AppImage` o `.tar.gz` paquete de instalación

:::nota[Requisitos del sistema]
- **Windows**: Windows 10 o versión superior (64 bits)
- **macOS**: macOS 10.15 o superior
- **Linux**: Distribuciones convencionales de Linux (Ubuntu 20.04+, Fedora 33+, etc.)
:::

:::tip[Requisitos previos predeterminados]
Para el último flujo de instalación local de HagiCode, normalmente no es necesario preparar un archivo por separado. `.NET` tiempo de ejecución. Desktop ya incluye el tiempo de ejecución necesario para el flujo predeterminado e inicializa la ruta SQLite local compatible de forma predeterminada.
:::

## Notas de instalación y lanzamiento

:::precaución[Aviso del Gatekeeper de macOS]
Si ya instaló la aplicación en la ruta predeterminada `/Applications/HagiCode.app` pero macOS muestra "la aplicación está dañada", "no se puede abrir" o una advertencia de Gatekeeper similar en el primer inicio, la causa habitual es la `com.apple.quarantine` atributo que macOS agrega a las aplicaciones descargadas de Internet, no una falla en el instalador de HagiCode Desktop.

Ejecute el siguiente comando en la Terminal. Requiere privilegios de administrador y elimina recursivamente el atributo de cuarentena de la aplicación en la ruta de instalación predeterminada:

```bash
sudo xattr -dr com.apple.quarantine /Applications/HagiCode.app
```

Una vez finalizado el comando, reinicie HagiCode Desktop.
:::

## Ejecutar escritorio

Después de la descarga, abra el instalador o el paquete de la aplicación y siga las instrucciones en pantalla. Desktop prepara el tiempo de ejecución en la ruta local actualmente admitida, administra el servidor HagiCode y muestra el progreso y el estado cuando se requieren descargas de red.

:::nota[Requisitos de red]
Asegúrese de que su conexión de red esté disponible. Es posible que el escritorio necesite obtener información de la versión o paquetes de componentes de Internet; Si falla la descarga, verifique la configuración de su red, proxy o firewall y vuelva a intentarlo.
:::

## Gestión de versiones

HagiCode Desktop admite la gestión de múltiples versiones, lo que le permite cambiar libremente entre diferentes versiones.

![Gestión de versiones](../../img/screenshots/installation/install-desktop/12-version-management/original.png)

### Cambiar versión

1. Abrir el escritorio de HagiCode
2. Vaya a la página "Administración de versiones"
3. Seleccione la versión que desea cambiar
4. Haga clic en el botón "Cambiar versión"
5. Espere a que se complete el cambio

:::nota[Notas sobre el cambio de versión]
- El cambio de versión reiniciará automáticamente el servidor HagiCode
- Asegúrese de guardar todo el trabajo antes de cambiar
- Las configuraciones del proyecto pueden diferir entre versiones.
:::

## Iniciar y detener el servicio

HagiCode Desktop proporciona una forma sencilla de administrar el servicio HagiCode Server.

### Iniciar servicio

En la página de inicio de la aplicación de escritorio, haga clic en el botón "Iniciar servicio" para iniciar HagiCode Server.

![Iniciar servicio](../../img/screenshots/installation/install-desktop/13-start-service-homepage/original.png)

### Detener servicio

Para detener HagiCode Server, haga clic en el botón "Detener servicio" en la página de inicio.

:::consejo[Estado del servicio]
La aplicación de escritorio muestra el estado de ejecución del servidor HagiCode en tiempo real, que incluye:
- Estado del servicio (en ejecución/detenido)
- Número de puerto de servicio
- Información de la versión actual
:::

## Solución de problemas

### Fallo de instalación

Si encuentra problemas durante la instalación:

1. **Verifique la conexión de red**: asegúrese de poder acceder a Internet
2. **Verificar espacio en disco**: asegúrese de tener suficiente espacio en disco
3. **Ejecutar como administrador**: en Windows, haga clic derecho en el instalador y seleccione "Ejecutar como administrador".
4. **Ver archivos de registro**: el instalador genera archivos de registro detallados que pueden ayudar a diagnosticar problemas.

### Fallo de instalación de dependencia

Si falla la instalación de la dependencia:

1. **Verifique la compatibilidad del sistema**: asegúrese de que su sistema operativo cumpla con los requisitos mínimos
2. **Verifique la configuración de red y firewall**: asegúrese de que el instalador pueda alcanzar las descargas necesarias y que su firewall no lo bloquee.
3. **Confirmar los permisos del instalador**: en Windows, intente ejecutar el instalador como administrador para que tenga permisos suficientes.
4. **Reinicie la aplicación y vuelva a intentarlo**: cierre y vuelva a abrir HagiCode Desktop, luego vuelva a intentar el paso de instalación
5. **Ver registros**: revise los registros de errores detallados en la aplicación de escritorio

### Fallo de inicio del servicio

Si el servidor HagiCode no se inicia:

1. **Verificar ocupación del puerto**: asegúrese de que el puerto 45000 no esté ocupado por otros programas
2. **Verifique la configuración del firewall**: asegúrese de que el firewall permita el acceso a la red del servidor HagiCode
3. **Reiniciar aplicación**: Intente reiniciar HagiCode Desktop
4. **Ver registros**: vea registros de errores detallados en la aplicación de escritorio
