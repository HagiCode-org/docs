## Design.md: una solución para que la IA realice diseños de UI frontend de manera consistente

> En la era del desarrollo frontend asistido por IA, ¿cómo mantener la coherencia de las UI generadas por la IA? Este artículo comparte nuestra experiencia práctica construyendo un sitio de galería de diseño basado en awesome-design-md, y cómo crear design.md estructurados para guiar a la IA en el diseño de UI normalizado.

## Antecedentes

Aquellos que han utilizado IA para escribir código frontend probablemente hayan tenido experiencias similares: para la misma página, si pides a la IA que la genere varias veces, cada vez tendrá un estilo diferente. A veces es con bordes redondeados, a veces con bordes rectos; a veces el espaciado es de 8px, otras veces se convierte en 16px; incluso el mismo botón puede verse diferente en conversaciones distintas.

Esto no es un fenómeno aislado. Con la popularización del desarrollo asistido por IA, **la falta de coherencia en las UI frontend generadas por IA** se ha convertido en un problema generalizado. Diferentes asistentes de IA, diferentes prompts, e incluso el mismo asistente en diferentes conversaciones, producen diseños de interfaz con estilos muy distintos. Esto genera enormes costos de mantenimiento para la iteración del producto.

La raíz del problema es en realidad muy simple: falta un **documento de referencia de diseño autorizado**. Los archivos de estilos CSS tradicionales solo pueden decir a los desarrolladores "cómo implementar", pero no pueden transmitir completamente "por qué se diseña así" ni "qué patrón de diseño usar en qué escenario". Para la IA, necesita aún más una descripción estructurada clara para entender las especificaciones de diseño.

Al mismo tiempo, la comunidad de código abierto ya tiene algunos recursos excelentes. El proyecto [VoltAgent/awesome-design-md](https://github.com/VoltAgent/awesome-design-md) recopila documentos de sistemas de diseño de muchas empresas conocidas, donde cada directorio contiene README.md, DESIGN.md y HTML de vista previa. Pero estos están dispersos en los repositorios upstream, lo que dificulta su consulta y comparación rápidas.

Entonces, ¿podríamos integrar estos recursos y crear una galería de diseño conveniente de consultar, mientras consolidamos un design.md estructurado para que lo use la IA?

La respuesta es afirmativa. A continuación, compartiremos nuestra solución.

## Análisis

Antes de comenzar a escribir código, analicemos los varios desafíos técnicos de este problema.

### Gestión de fuentes de contenido: ¿cómo unificar los recursos de diseño dispersos?

El repositorio upstream awesome-design-md contiene una gran cantidad de documentos de diseño, pero necesitamos una forma de incorporarlo a nuestro proyecto.

**Solución: usar git submodule**

```
awesome-design-md-site
└── vendor/awesome-design-md          # Recursos upstream (git submodule)
```

Esto tiene varios beneficios:
- Control de versiones: se puede bloquear una versión upstream específica
- Construcción offline: no es necesario solicitar APIs externas durante la construcción
- Revisión de contenido: se pueden ver cambios específicos en PRs

### Estandarización de datos: ¿cómo unificar diferentes estructuras de documentos?

Las estructuras de documentos de diseño de diferentes empresas pueden ser diferentes; a algunos les faltan archivos de vista previa, otros tienen nombres inconsistentes. Necesitamos realizar procesamiento de estandarización durante el tiempo de construcción.

### Arquitectura de sitio estático: ¿cómo proporcionar búsqueda dinámica manteniendo el despliegue estático?

Dado que es una galería de diseño, la función de búsqueda es indispensable. Pero Astro es un generador de sitios estáticos, ¿cómo implementar búsqueda en tiempo real?

**Solución: React island + sincronización de parámetros de consulta URL**

```typescript
// src/components/gallery/SearchToolbar.tsx

export function SearchToolbar() {
```

El beneficio de esto es que conserva la capacidad de despliegue de sitios estáticos (se puede desplegar en cualquier servicio de alojamiento estático), al tiempo que proporciona una experiencia de usuario de filtrado instantáneo.

### Documentación de diseño: ¿cómo hacer que la IA entienda y respete las especificaciones de diseño?

Este es el núcleo de toda la solución. Necesitamos crear un design.md estructurado que permita a la IA entender y aplicar nuestras especificaciones de diseño.

**Solución: tomar como referencia la estructura de ClickHouse DESIGN.md**

El [DESIGN.md de ClickHouse](https://github.com/ClickHouse/ClickHouse/blob/master/DESIGN.md) es una excelente referencia; incluye:

Nuestro enfoque es: **referenciar la estructura, reescribir el contenido**. Mantener la estructura de capítulos de ClickHouse DESIGN.md, pero reemplazar el contenido con los tokens de diseño y especificaciones de componentes que realmente usamos en nuestro proyecto.

## Solución

Basándonos en el análisis anterior, nuestra solución contiene cuatro módulos principales.

### 1. Línea de ingesta de contenido

Esta es la base de todo el sistema, responsable de extraer y estandarizar contenido de los recursos upstream.

```typescript
// src/lib/content/awesomeDesignCatalog.ts

export async function scanSourceEntries(): Promise<DesignEntry[]> {
  const designDir = 'vendor/awesome-design-md/design-md';
  const entries: DesignEntry[] = [];

  for (const dir of await fs.readdir(designDir)) {
    const entryPath = path.join(designDir, dir);
    if (await isValidDesignEntry(entryPath)) {
      const entry = await normalizeDesignEntry(entryPath);
      entries.push(entry);
    }
  }

  return entries;
}

async function isValidDesignEntry(dir: string): Promise<boolean> {
  const requiredFiles = ['README.md', 'DESIGN.md'];
  for (const file of requiredFiles) {
    if (!(await fileExists(path.join(dir, file)))) {
      return false;
    }
  }
  return true;
}
```

### 2. Interfaz de navegación de galería

La interfaz de galería incluye tres partes principales:

**Página de inicio**: muestra una cuadrícula de tarjetas con todas las entradas de diseño, donde cada tarjeta contiene:
- Título y resumen de la entrada de diseño
- Imagen de vista previa (si está disponible)
- Resaltado de búsqueda rápida

**Página de detalles**: muestra la información completa de una sola entrada de diseño:
- Documento README
- Documento DESIGN
- Vista previa (con soporte para cambio de tema claro/oscuro)
- Navegación entre entradas adyacentes

**Navegación**: soporte para volver a la galería, navegar entradas adyacentes

La galería de la página de inicio utiliza un diseño de tarjetas de alta densidad, distribuyendo entradas design.md de diferentes fuentes en un marco visual unificado, lo que facilita comparar rápidamente estilos de marca, patrones de botones y ritmos de composición.

### 3. Función de búsqueda

La función de búsqueda se basa en filtrado del lado del cliente, usando parámetros de consulta URL para mantener el estado:

```typescript
// src/components/gallery/SearchToolbar.tsx

function SearchToolbar({ entries }: { entries: DesignEntry[] }) {
  const [query, setQuery] = useState('');
  const [results, setResults] = useState(entries);

  useEffect(() => {
    const params = new URLSearchParams(window.location.search);
    const initialQuery = params.get('q') || '';
    setQuery(initialQuery);
    filterEntries(initialQuery);
  }, []);

  const filterEntries = (searchQuery: string) => {
    const filtered = entries.filter(entry =>
      entry.searchText.toLowerCase().includes(searchQuery.toLowerCase())
    );
    setResults(filtered);
  };

  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const value = e.target.value;
    setQuery(value);
    filterEntries(value);

    // Actualizar URL (sin activar recarga de página)
    const newUrl = value
      ? `${window.location.pathname}?q=${encodeURIComponent(value)}`
      : window.location.pathname;
    window.history.replaceState({}, '', newUrl);
  };

  return (
    <div className="search-toolbar">
      <input
        type="text"
        value={query}
        onChange={handleChange}
        placeholder="Buscar entradas de diseño..."
      />
      <span className="result-count">{results.length} resultados</span>
    </div>
  );
}
```

## Práctica

Después de entender la solución, ¿cómo se implementa específicamente?

### Pasos de implementación

**Paso 1: Inicializar submódulo**

```bash
# Añadir repositorio upstream como submódulo
git submodule add https://github.com/VoltAgent/awesome-design-md.git vendor/awesome-design-md

# Inicializar y actualizar submódulo
git submodule update --init --recursive
```

**Paso 2: Crear línea de contenido**

Implementar `awesomeDesignCatalog.ts`, incluyendo:
- Lógica de escaneo y verificación de archivos
- Renderizado Markdown (usando el renderizador incorporado de Astro)
- Extracción de datos de entrada

**Paso 3: Construir UI de galería**

Usar Astro + React Islands para crear:
- Diseño de galería de página de inicio (cuadrícula de tarjetas)
- Componente de tarjeta de diseño
- Barra de herramientas de búsqueda
- Diseño de página de detalles

**Paso 4: Escribir documento de diseño**

Basándose en la estructura de ClickHouse DESIGN.md, llenar los tokens de diseño reales del proyecto. Actualizar README.md, añadiendo un enlace hacia design.md.

### Consideraciones

**Seguridad**: El renderizado Markdown necesita filtrar HTML inseguro. El renderizador incorporado de Astro por defecto filtra etiquetas script, pero aún así hay que prestar atención a riesgos XSS.

**Rendimiento**: Grandes cantidades de iframe de vista previa pueden afectar la carga inicial. Se sugiere usar `loading="lazy"` para carga diferida del contenido de vista previa.

**Mantenibilidad**: design.md necesita mantenerse sincronizado con la implementación del código. Se sugiere añadir verificaciones en CI para asegurar que las variables CSS sean consistentes entre la documentación y el código.

**Accesibilidad**: Asegurar que el contraste de colores cumpla con el estándar WCAG AA (al menos 4.5:1).

### Guía de uso de IA

Después de crear design.md, ¿cómo hacer que la IA realmente lo use? Aquí hay algunos trucos prácticos:

**Truco 1: Referencia explícita en el prompt**

```
Por favor, consulta el archivo design.md en el directorio raíz del proyecto y usa las especificaciones de diseño definidas en él para implementar los siguientes componentes:
- Botón: usar tono primary, bordes redondeados 8px
- Tarjeta: usar nivel de sombra elevation-2
```

**Truco 2: Pedir a la IA que referencie variables CSS específicas**

```
Implementa una barra de navegación, requiriendo:
- Color de fondo usando --color-bg-primary
- Borde usando --color-border-subtle
- Texto usando --text-color-primary
```

**Truco 3: Incluir contenido de design.md en el prompt del sistema**

Si tu herramienta de IA soporta prompts del sistema personalizados, puedes añadir directamente el contenido principal de design.md.

### Estrategia de pruebas

**Pruebas de línea de contenido**:
- Escenarios de archivos faltantes (falta README.md o DESIGN.md)
- Escenarios de error de formato (falla de análisis Markdown)
- Escenarios de directorio vacío

**Pruebas de función de búsqueda**:
- Manejo de resultados vacíos
- Caracteres especiales (como chino, emoji)
- Verificación de sincronización URL

**Pruebas de componentes UI**:
- Cambio de tema claro/oscuro
- Diseño responsivo
- Estado de carga de vista previa

Se sugiere automatizar la actualización de submódulos y el despliegue de construcción, lo que puede activar automáticamente el proceso CI cuando se actualice el repositorio upstream.

## Resumen

El problema de falta de coherencia en las UI generadas por IA que encontró HagiCode durante el desarrollo es esencialmente la falta de un documento de referencia de diseño estructurado. A través de la construcción de un sitio de galería de diseño y la creación de design.md normalizados, resolvimos con éxito este problema.

El valor principal de esta solución radica en:
- **Recursos unificados**: integrar documentos de sistemas de diseño dispersos
- **Especificaciones estructuradas**: presentar especificaciones de diseño en una forma que la IA pueda entender
- **Mantenimiento continuo**: mantener contenido actualizado a través de git submodule

Si también estás usando IA para asistir el desarrollo frontend, te sugiero probar esta solución. Crear un design.md estructurado no solo mejora la coherencia del código generado por la IA, sino que también ayuda al equipo a mantener la unidad de las especificaciones de diseño.
