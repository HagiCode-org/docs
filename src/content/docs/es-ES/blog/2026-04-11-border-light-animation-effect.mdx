## Guía de implementación de efectos de animación de luz de borde

> ¿Cómo se hace con CSS puro ese elemento importante que los usuarios notan a primera vista? En realidad no es difícil, es solo dar un rodeo. Este artículo te guía desde cero para implementar animaciones de luz que rodean el borde, y también habla de los obstáculos que encontramos en el proyecto HagiCode.

## Antecedentes

Los desarrolladores frontend probablemente han tenido esta experiencia: el gerente de producto se acerca, con esa expresión de "este requisito es muy simple" en su rostro: "¿Puedes añadir un efecto especial a esta tarea en ejecución para que los usuarios la vean de inmediato?"

Dices que sí, claro, añade un borde que cambie de color. Pero el otro niega con la cabeza, con una mirada que dice "tú no entiendes": "No es lo suficientemente obvio, quiero ese efecto donde la luz corre alrededor del borde, como en las películas de ciencia ficción."

En este momento podrías preguntarte: ¿Cómo se implementa esto? ¿Usar Canvas? ¿SVG? ¿O CSS puede hacerlo? Después de todo, nadie quiere admitir que no sabe cómo hacerlo.

En realidad, las animaciones de luz que rodean el borde son muy comunes en las aplicaciones web modernas, utilizadas principalmente en estos escenarios:

- **Indicador de estado**: Marcar tareas en progreso o proyectos activos
- **Foco visual**: Resaltar áreas de contenido importantes
- **Mejora de marca**: Crear una experiencia visual con sensación tecnológica y moderna
- **Temas festivos**: Crear una atmósfera celebratoria para ocasiones especiales

Cuando trabajamos en [HagiCode](https://hagicode.com), nos encontramos con este requisito: los usuarios necesitaban ver de un vistazo qué sesiones estaban en ejecución y qué propuestas estaban siendo procesadas. Probamos varios esquemas, algunos caminos eran un poco más fáciles, otros un poco más tortuosos, y finalmente consolidamos un enfoque de implementación bastante maduro.

## Ideas de implementación central

Después de analizar el código de HagiCode, resumimos los siguientes patrones de implementación principales, cada uno con sus escenarios aplicables, o más bien, cada uno tiene su razón de ser.

### 1. Halo de rotación con gradiente cónico (el más utilizado)

Esta es la forma clásica de implementar la luz que rodea el borde. La idea central es usar `conic-gradient` de CSS para crear un gradiente cónico y luego hacerlo girar. Como una farola nocturna, girando y girando allí.

**Elementos clave:**
- Usar el pseudo-elemento `::before` para crear la capa del halo
- Usar `conic-gradient` para definir la distribución del color del gradiente
- Usar el pseudo-elemento `::after` para enmascarar el área central (opcional)
- Usar `@keyframes` para implementar la animación de rotación

### 2. Línea luminosa lateral

Este es adecuado para indicadores de estado de elementos de lista, simplemente crea una línea delgada brillante en un lado del elemento, sin que todo el borde se mueva. A veces, un poco de luz es suficiente, no necesitas iluminar el mundo entero.

**Elementos clave:**
- Elemento de línea fina posicionado absolutamente
- Usar `box-shadow` para crear el efecto de brillo
- Usar `scale` y `opacity` para implementar la animación de respiración

### 3. Fondo brillante con Box-Shadow

Si no necesitas el efecto envolvente y solo quieres un halo de fondo suave, basta con superponer múltiples capas de `box-shadow`. A veces, las cosas simples son mejores.

### 4. Soporte de accesibilidad

Esto se pasa por alto fácilmente, pero es especialmente importante. Todas las animaciones deben considerar la consulta de medios `prefers-reduced-motion`, proporcionando una alternativa estática para los usuarios que no les gustan las animaciones. Después de todo, no a todos les gusta que las cosas se muevan, respetar la elección de cada persona es lo correcto.

## Soluciones de implementación

### Solución 1: Borde rotativo con gradiente cónico (recomendado)

Esta es la implementación más completa del efecto de luz envolvente, y también es la solución más utilizada en HagiCode. Después de todo, si algo funciona bien, ¿por qué cambiarlo?

```css
/* Contenedor principal */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Capa de halo rotativo */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Capa de máscara (opcional, para crear efecto de borde hueco) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

El principio de esta solución es bastante simple: crear un pseudo-elemento más grande que el contenedor principal, dibujar un gradiente cónico encima, y luego hacerlo girar continuamente. El contenedor principal establece `overflow: hidden`, por lo que solo se puede ver la parte del borde con la luz girando. Es como ver una farola desde una ventana, solo podemos ver ese pequeño tramo cuando pasa.

### Solución 2: Borde de luz rotativo simplificado

Si no necesitas un efecto tan complejo, HagiCode tiene una implementación de clase de utilidad más ligera. A veces, simple es mejor.

```css
/* Clase de utilidad de borde de luz rotativo */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Soporte de accesibilidad */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Nota el `will-change: transform` aquí, esto le dice al navegador "este elemento va a cambiar constantemente", el navegador hará algunas optimizaciones por adelantado, y la animación será más fluida. Después de todo, prepararse con anticipación es mejor que buscar soluciones a último momento.

### Solución 3: Línea brillante lateral

Esto es particularmente adecuado para indicadores de estado de elementos de lista, la lista de sesiones de HagiCode usa esta solución. Una línea delgada, pero que puede destacar entre muchos elementos, ¿no es esta también una filosofía de vida?

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Aquí usamos `isolation: isolate` para crear un nuevo contexto de apilamiento, y luego usamos `z-index` para controlar el orden de visualización de cada capa. `pointer-events: none` también es clave, de lo contrario el pseudo-elemento bloqueará las operaciones de clic del usuario. Como algunas cosas, se ven bien, pero no deben estorbar.

### Solución 4: Componente React encapsulado

Si usas React en tu proyecto, puedes encapsular un componente para manejar esta lógica, especialmente la parte de accesibilidad. Después de todo, escribir el código una vez y usarlo muchas veces es lo que queremos.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Módulo CSS correspondiente:

```css
/* GlowBorder.module.css */

/* Versión animada */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Versión estática (accesibilidad) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

El hook `useReducedMotion` de `framer-motion` detecta automáticamente las preferencias del sistema del usuario. Si el usuario ha activado "reducir movimiento", devolverá true, y en ese caso se muestra la versión estática. Después de todo, respetar la elección del usuario es más importante que mostrarlo por la fuerza.

## Compartir experiencia práctica

A continuación, se presentan algunas experiencias que resumimos después de encontrar obstáculos al hacer HagiCode. En realidad, son solo algunos pensamientos, espero que puedan ayudarte a ti, que vendrás después.

### 1. Sistema de variables de tema

Usar variables CSS para implementar soporte multi-tema es especialmente conveniente. Después de todo, ¿quién quiere cambiar un montón de código cada vez que cambia de tema?

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Uso */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

De esta manera, al cambiar de tema, solo necesitas modificar la class de la etiqueta `html`, y todos los colores de animación se actualizarán automáticamente. Un código, dos estilos, ¿no es esto lo que perseguimos?

### 2. Optimización del rendimiento

**Usa `will-change` para solicitar al navegador que optimice:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Dile al navegador con anticipación, y te ayudará a hacer algunas optimizaciones. Como muchas cosas en la vida, prepararse con anticipación siempre es bueno.

**Evita usar box-shadow complejos en elementos grandes:**

```css
/* No recomendado: usar sombras difusas en elementos grandes */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Mejor: usar pseudo-elementos para limitar el área brillante */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Probamos en HagiCode, agregar sombras difusas directamente en tarjetas grandes hace que la velocidad de fotogramas de desplazamiento caiga a menos de 30fps, después de cambiar a pseudo-elementos se mantiene estable en 60fps. Los usuarios pueden sentir esta diferencia en la experiencia.

### 3. Accesibilidad

Esto realmente no se puede omitir, algunos usuarios sentirán que las animaciones son mareantes o ruidosas, respetar su elección es una cualidad básica al hacer productos. Después de todo, las cosas hermosas no deben imponerse a los demás.

**Consulta de medios CSS:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Proporcionar una alternativa estática */
    opacity: 1;
  }
}
```

**Detectar preferencias del usuario en React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Control de nivel de intensidad

El indicador de rendimiento de Token en HagiCode muestra diferentes colores de luz según el rendimiento en tiempo real, esto se implementa dinámicamente. Después de todo, diferentes estados deberían tener diferentes formas de expresión.

```tsx
const colors = [
  null,       // Nivel 0 - sin color
  '#3b82f6',  // Nivel 1 - Azul
  '#34d399',  // Nivel 2 - Esmeralda
  '#facc15',  // Nivel 3 - Amarillo
  '#fbbf24',  // Nivel 4 - Ámbar
  '#f97316',  // Nivel 5 - Naranja
  '#22d3ee',  // Nivel 6 - Cian
  '#d946ef',  // Nivel 7 - Fucsia
  '#f43f5e',  // Nivel 8 - Rosa
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Precauciones

Algunos detalles aún deben tenerse en cuenta, de lo contrario será demasiado tarde cuando te des cuenta después de caer en una trampa.

| Precaución | Descripción |
|---------|------|
| **Gestión de z-index** | La capa del halo debe tener un z-index apropiado para evitar afectar la interacción del contenido |
| **pointer-events** | Los pseudo-elementos del halo deben establecer `pointer-events: none` |
| **Desbordamiento de límites** | El contenedor principal necesita establecer `overflow: hidden` o ajustar el tamaño del pseudo-elemento |
| **Impacto en el rendimiento** | Las animaciones complejas pueden afectar el rendimiento en dispositivos móviles, es necesario probar |
| **Modo oscuro** | Asegúrate de que el color brillante sea claramente visible en fondos oscuros |
| **Cambio de tema** | Usa variables CSS para asegurar que los colores de animación se actualicen correctamente al cambiar de tema |

### 6. Técnicas de depuración

A veces los pseudo-elementos son difíciles de encontrar en las herramientas de desarrollo, puedes agregar temporalmente un borde para ver la posición.

```css
/* Mostrar temporalmente los límites del pseudo-elemento para depuración */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Después de ajustar la posición, recuerda comentar o eliminar esta línea, de lo contrario el entorno de producción será incómodo. Algunas cosas es mejor dejarlas en el entorno de desarrollo.

## Resumen

Las animaciones de luz que rodean el borde no son difíciles ni simples. El núcleo es `conic-gradient` más rotación, pero para lograr buen rendimiento, mantenibilidad y accesibilidad amigable, hay muchos detalles a tener en cuenta.

HagiCode encontró muchos obstáculos en esto y también resumió algunas mejores prácticas. Hacer proyectos es así, probar una y otra vez, mejorar una y otra vez. Si estás haciendo requisitos similares, espero que este artículo te ayude a evitar algunos desvíos.

Después de todo, algunas cosas solo se pueden entender realmente cuando las practicas personalmente.
