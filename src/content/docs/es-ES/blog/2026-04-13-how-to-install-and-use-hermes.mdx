Para instalar y comenzar a usar Hermes, el camino más corto realmente solo tiene tres pasos:

1. Ejecutar el comando oficial de instalación
2. Iniciar la CLI con `hermes` en la terminal
3. Si deseas continuar usándolo en Feishu, configura `hermes gateway setup`

Este artículo no pretende explicar todas las capacidades de Hermes de una vez, sino ayudarte a completar el ciclo de introducción más crítico: instalarlo, ejecutarlo, comenzar a usarlo, y luego conectarlo con el escenario de plataforma de mensajería más común.

## Qué es Hermes y para quién es

Hermes Agent es un agente de IA que se puede usar tanto en la terminal local como a través de gateways de plataformas de mensajería.

Para la mayoría de los desarrolladores, tiene dos puntos de entrada más comunes:

- **CLI**: Escribe `hermes` en la terminal para entrar directamente a la interfaz interactiva
- **Messaging Gateway**: Ejecuta `hermes gateway` y luego chatea con él desde plataformas como Feishu, Telegram, Discord, Slack, etc.

Si tu objetivo actual es empezar rápidamente, se recomienda no invertir el orden, sigue esta ruta:

- Primero instala Hermes
- Primero verifica si funciona desde la CLI
- Luego decide si quieres conectar una plataforma de mensajería

Esto hace que sea más fácil localizar problemas y también es más adecuado para usuarios que se están acercando a Hermes por primera vez.

## Qué saber antes de instalar Hermes

Según el README de Hermes, la ruta de instalación rápida oficial admite estos entornos:

- Linux
- macOS
- WSL2
- Android via Termux

### Notas para usuarios de Windows

Hermes actualmente no admite la ejecución directa en Windows nativo. Si usas Windows, se recomienda instalar primero WSL2 y luego ejecutar el comando de instalación dentro de WSL2.

Es mejor mencionar esto claramente al principio del artículo, porque muchos fallos de instalación en realidad no son problemas con el comando, sino entornos de ejecución que no cumplen con los requisitos.

## Cómo instalar Hermes

El comando de instalación rápida proporcionado por el README de Hermes es:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Este comando ejecutará el script de instalación oficial y manejará el proceso de inicialización específico de la plataforma.

### Refrescar el shell después de la instalación

Después de que finalice la instalación, recarga el entorno del shell primero. Lo más común es:

```bash
source ~/.bashrc
```

Si usas zsh, también puedes cambiarlo a:

```bash
source ~/.zshrc
```

### Cómo confirmar que Hermes está instalado

La forma más directa de verificar es ejecutar:

```bash
hermes
```

Si deseas confirmar aún más si la configuración y las dependencias son normales, también puedes ejecutar:

```bash
hermes doctor
```

`hermes doctor` es adecuado para usar en estos momentos:

- El comportamiento del comando no es normal después de la instalación
- Fallo en la configuración del modelo
- Fallo al iniciar el gateway
- No estás seguro de si las dependencias del entorno están completas

## Cómo usar Hermes por primera vez

### Empieza desde la CLI

Si solo quieres confirmar lo más rápido posible si Hermes funciona, el método más simple es:

```bash
hermes
```

Esto iniciará la CLI interactiva de Hermes. Para los usuarios que se acercan a Hermes por primera vez, este también es el punto de partida más recomendado, porque primero puedes verificar las cosas más críticas:

- Si el comando realmente funciona
- Si la configuración del modelo actual es normal
- Si la cadena de herramientas de la terminal funciona normalmente
- Si la experiencia interactiva es la forma que necesitas

### Estos comandos son suficientes para completar la primera ronda de configuración

Los varios comandos de alta frecuencia enumerados en el README de Hermes básicamente constituyen la ruta de primera interacción:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Si no sabes qué hace cada uno, puedes recordarlos así:

- `hermes model`: Seleccionar y cambiar modelos
- `hermes tools`: Ver y configurar las herramientas disponibles actuales
- `hermes config set`: Cambiar elementos de configuración específicos
- `hermes setup`: Ejecutar un asistente de inicialización completo
- `hermes update`: Actualizar Hermes
- `hermes doctor`: Realizar solución de problemas

El orden más práctico para los novatos suele ser:

1. Primero ejecuta `hermes model`
2. Si deseas configurar los elementos comunes de una vez, luego ejecuta `hermes setup`

## Las dos formas más comunes de usar Hermes

### 1. Usar Hermes como asistente de desarrollo diario en la terminal

El modo CLI es adecuado para estos escenarios:

- Hacer preguntas directamente mientras escribes código localmente
- Revisar proyectos, modificar archivos, ejecutar comandos
- Hacer depuración o review únicas
- Colaboración continua en el directorio de trabajo actual

Su mayor ventaja es que el camino es más corto: no necesitas conectar plataformas adicionales, no necesitas manejar primero la configuración del bot, y también es lo más adecuado para establecer los primeros hábitos de uso.

### 2. Usar Hermes a través de plataformas de mensajería

Si deseas chatear con Hermes en plataformas como Feishu, Telegram, Discord, etc., necesitas usar el messaging gateway.

Los comandos de entrada más comunes son:

```bash
hermes gateway setup
hermes gateway
```

Donde:

- `hermes gateway setup` se usa para hacer la configuración interactiva de la plataforma
- `hermes gateway` se usa para iniciar el proceso del gateway

Según la documentación oficial, el gateway es un proceso en segundo plano unificado que se usa para conectar tus plataformas configuradas, administrar sesiones y manejar tareas programadas y otras funciones.

## Cómo conectar Hermes a una plataforma de mensajería usando Feishu como ejemplo

Si tu trabajo diario es principalmente en Feishu, entonces Feishu/Lark será una forma natural de integrar Hermes.

### La forma mínima viable de integración

La entrada recomendada por la documentación oficial para Feishu/Lark es:

```bash
hermes gateway setup
```

Después de ejecutarlo, selecciona **Feishu / Lark** en el asistente.

Los dos modos de conexión dados por la documentación de Feishu son:

- `websocket`: Recomendado
- `webhook`: Opcional

Si Hermes se ejecuta en tu notebook, estación de trabajo o servidor privado, usar `websocket`优先 será más simple, porque no necesitas exponer adicionalmente una dirección de retorno pública.

### Si configuras manualmente, al menos necesitas conocer estas variables

Si no configuras a través del asistente, sino que escribes la configuración manualmente, las variables principales enumeradas en la documentación de Feishu incluyen:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Entre las cuales las más notables son dos:

- `FEISHU_ALLOWED_USERS`: Se recomienda configurar para evitar que cualquiera que pueda contactar al bot pueda usarlo directamente
- `FEISHU_HOME_CHANNEL`: Puedes especificar un chat por adelantado para recibir resultados de cron o notificaciones predeterminadas

### Por qué a veces no responde en los grupos de Feishu

Este punto es fácil de pasar por alto: en los grupos de Feishu, Hermes por defecto no responde a cada mensaje que ve.

La documentación oficial explica claramente:

- En chats privados, Hermes responderá a los mensajes
- En grupos, debe mencionar explícitamente `@` el bot para que procese el mensaje

Si deseas configurar una sesión de Feishu como home channel, también puedes usar en el chat:

```text
/set-home
```

O escribir por adelantado en la configuración:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Comandos de Hermes que los novatos deberían recordar primero

Ya sea que estés en la CLI o en una plataforma de mensajería, recordar los siguientes comandos ya es suficiente:

- `/new` o `/reset`: Iniciar nueva sesión
- `/model`: Ver o cambiar modelo
- `/retry`: Reintentar la ronda anterior
- `/undo`: Deshacer la interacción anterior
- `/compress`: Comprimir manualmente el contexto
- `/help`: Ver ayuda

Si usas principalmente en plataformas de mensajería, recuerda uno adicional:

- `/sethome` o `/set-home`: Establecer el chat actual como home channel

Estos comandos cubren las operaciones más comunes en la etapa de novato: reiniciar, ajustar, retroceder, ver y continuar usando.

## Preguntas frecuentes

### ¿Se puede instalar Hermes directamente en Windows?

No. La documentación oficial actual explica claramente que Windows nativo no es compatible, se recomienda usar WSL2.

### ¿Qué hacer si después de la instalación escribir `hermes` no responde?

Se recomienda seguir este orden para verificar:

1. Primero recarga el shell, por ejemplo `source ~/.bashrc`
2. Luego vuelve a ejecutar `hermes`
3. Si todavía es anormal, ejecuta `hermes doctor`

### ¿Por qué el bot no responde en los grupos de Feishu?

Primero verifica estos tres elementos:

- Si mencionaste `@` Hermes en el grupo
- Si `FEISHU_ALLOWED_USERS` limitó al usuario actual
- Si la política de grupo actual permite procesar mensajes de grupo

Según la documentación oficial de Feishu, en escenarios de grupo, la mención explícita @mention es una condición necesaria.

3. Luego usa `hermes model` y `hermes setup` para completar la configuración básica
