Una de las ventajas principales de HagiCode es el soporte para múltiples herramientas de programación con IA CLI diferentes a través de una capa de abstracción unificada. Los beneficios de este diseño, para decirlo claramente, son solo esto: las cosas nuevas pueden entrar, las viejas pueden quedarse, y el código no se desordena. Después de todo, todos esperamos que la vida sea así, ¿verdad?

La plataforma define los tipos de proveedores CLI compatibles a través de la enumeración `AIProviderType`:

```csharp
public enum AIProviderType

    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI (nuevo)

```

Se puede ver que Gemini CLI se ha unido a esta gran familia como el décimo miembro. Cada CLI tiene características únicas y escenarios aplicables, y los usuarios pueden elegir flexiblemente según sus necesidades. Después de todo, todos los caminos llevan a Roma, solo que algunos caminos son más fáciles de recorrer y otros son un poco más tortuosos.

### Arquitectura Provider

HagiCode.Libs.Providers proporciona una interfaz Provider unificada, haciendo que la integración de cada CLI sea estándar y concisa. Tomando Gemini CLI como ejemplo:

```csharp
public class GeminiProvider : ICliProvider<GeminiOptions>
{
    private static readonly string[] DefaultExecutableCandidates = ["gemini", "gemini-cli"];
    private const string ManagedBootstrapArgument = "--acp";

    public string Name => "gemini";
    public bool IsAvailable => _executableResolver.ResolveFirstAvailablePath(DefaultExecutableCandidates) is not null;
}
```

Los beneficios de este diseño son:
- La integración de un nuevo CLI solo requiere implementar una clase Provider
- Gestión unificada del ciclo de vida y agrupación de sesiones
- Resolución automatizada de alias y búsqueda de archivos ejecutables

Para decirlo claramente, este diseño en realidad simplifica las cosas complejas, haciendo la vida un poco más fácil.

### Provider Registry

Provider Registry maneja automáticamente el mapeo y registro de alias:

```csharp
if (provider is GeminiProvider)
{
    registry.Register(provider.Name, provider, ["gemini-cli"]);
    continue;
}
```

Esto significa que los usuarios pueden usar `gemini` o `gemini-cli` de dos maneras para llamar a Gemini CLI, y el sistema lo reconocerá automáticamente. Es como tener muchos amigos, algunos los llamas por su nombre completo, otros por su apodo, al final son la misma persona, puedes llamarlos como quieras.

## Soporte del modelo GLM-5.1

GLM-5.1 es el modelo insignia más reciente de Zhipu AI, y HagiCode ha completado su soporte completo.

### Secondary Professions Catalog

HagiCode gestiona todos los modelos compatibles a través del Secondary Professions Catalog. A continuación se muestra la configuración de la serie GLM:

| Model ID | Name | SupportsImage | Compatible CLI Families |
|----------|------|----------------|-------------------------|
| `glm-4.7` | GLM 4.7 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM 5 Turbo | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (Legacy) | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | **true** | claude, codebuddy, hermes, qoder, kiro |

Las características clave de GLM-5.1 se pueden resumir en:
- Identificación de versión independiente, sin lastre de legacy
- Primer modelo GLM que admite entrada de imágenes
- Capacidades de razonamiento más fuertes y comprensión de código
- Amplia compatibilidad multi-CLI

### GLM-5.1 vs GLM-5.0

Desde el nivel de código, las diferencias clave entre GLM-5.1 y GLM-5.0:

```csharp
// GLM-5.0 (Legacy) - Tiene lógica de retención especial
private const string Glm50CodebuddySecondaryProfessionId = "secondary-glm-5-codebuddy";
private const string Glm50CodebuddyModelValue = "glm-5.0";

// GLM-5.1 - Nuevo identificador de modelo independiente
private const string Glm51SecondaryProfessionId = "secondary-glm-5-1";
private const string Glm51ModelValue = "glm-5.1";
```

GLM-5.0 lleva la marca "Legacy", que es un identificador de versión antiguo retenido para compatibilidad向后. GLM-5.1 es una versión completamente nueva e independiente, sin ningún lastre histórico. Es como algunas personas, siempre viviendo en el pasado; mientras que otras, viajan ligeras y caminan más rápido.

## Función de carga de imágenes

El soporte de imágenes de HagiCode se implementa a través de la propiedad `SupportsImage` de SecondaryProfession:

```csharp
public class HeroSecondaryProfessionSettingDto
{
    public bool SupportsImage { get; set; }
}
```

En el Secondary Professions Catalog, la configuración de GLM-5.1 es la siguiente:

```json
{
  "id": "secondary-glm-5-1",
  "supportsImage": true
}
```

Esto significa que los usuarios pueden cargar directamente capturas de pantalla para que la IA las analice, por ejemplo:
- Capturas de pantalla de mensajes de error
- Problemas de interfaz de usuario
- Gráficos de visualización de datos
- Resultados de ejecución de código

No es necesario describir manualmente el problema, simplemente capture una pantalla, la conveniencia de esta función se entiende después de usarla. Después de todo, algunas cosas, ver una vez es mejor que describir mil palabras.

## Integración de Gemini CLI

Gemini CLI, como el décimo Agent CLI, se integra en HagiCode a través de la arquitectura Provider estándar.

### Opciones de configuración

Gemini CLI admite opciones de configuración ricas:

```csharp
public class GeminiOptions
{
    public string? ExecutablePath { get; set; }
    public string? WorkingDirectory { get; set; }
    public string? SessionId { get; set; }
    public string? Model { get; set; }
    public string? AuthenticationMethod { get; set; }
    public string? AuthenticationToken { get; set; }
    public Dictionary<string, string?> AuthenticationInfo { get; set; }
    public Dictionary<string, string?> EnvironmentVariables { get; set; }
    public string[] ExtraArguments { get; set; }
    public TimeSpan? StartupTimeout { get; set; }
    public CliPoolSettings? PoolSettings { get; set; }
}
```

Estas opciones cubren todos los aspectos, desde la configuración básica hasta características avanzadas, y los usuarios pueden configurarlas flexiblemente según sus necesidades. Después de todo, las necesidades de cada persona son diferentes, ser flexible es siempre bueno.

### Protocolo de comunicación ACP

Gemini CLI admite el protocolo de comunicación ACP (Agent Communication Protocol), que es el estándar de comunicación CLI unificado de HagiCode. A través de ACP, diferentes CLI pueden interactuar con la plataforma de manera consistente, simplificando enormemente el trabajo de integración. Para decirlo claramente, es unificar las cosas complejas para que todos puedan tenerlo un poco más fácil.

## Mejores prácticas de integración multi-CLI

HagiCode.Libs.Providers proporciona un mecanismo unificado de registro y uso:

```csharp
services.AddHagiCodeLibs();

var gemini = serviceProvider.GetRequiredService<ICliProvider<GeminiOptions>>();
var codebuddy = serviceProvider.GetRequiredService<ICliProvider<CodebuddyOptions>>();
var hermes = serviceProvider.GetRequiredService<ICliProvider<HermesOptions>>();
```

Este diseño de inyección de dependencias hace que el uso de cada CLI sea muy conciso, y también facilita las pruebas unitarias y simulaciones. Después de todo, escribir código más limpio también es una responsabilidad para uno mismo.

### Precauciones

En el uso real, hay varios lugares a tener en cuenta:

1. **Configuración de API Key**: Asegúrese de configurar correctamente `ANTHROPIC_AUTH_TOKEN`, de lo contrario no podrá llamar al modelo
2. **Disponibilidad del modelo**: GLM-5.1 necesita abrir permisos en el proveedor de modelos correspondiente
3. **Función de imágenes**: Solo los modelos que admiten `supportsImage: true` pueden usar la función de carga de imágenes
4. **Instalación de CLI**: Antes de usar Gemini CLI, asegúrese de que `gemini` o `gemini-cli` esté en la variable PATH del sistema

Estas son todas cosas pequeñas, pero si no se manejan bien las cosas pequeñas, también pueden convertirse en cosas grandes. Así que todavía hay que prestar atención.
