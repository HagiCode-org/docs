## Diseño de 12 experiencias de inicio extremas para HagiCode: del minimalismo al cyberpunk

> En el breve intervalo entre la descarga y el Hydration de la aplicación React 19, se presenta una ventana de oportunidad de oro para que los usuarios perciban la personalidad de la marca. Este artículo comparte nuestra experiencia en el proyecto HagiCode, basado en HTML/CSS/JS para construir un sistema completo de estilos de inicio.


## Contexto

HagiCode, como aplicación moderna basada en ASP.NET Core 10 y React 19 (Vite), adopta una arquitectura de despliegue separada entre frontend y backend. Los productos del frontend se empaquetan y colocan en el directorio `wwwroot/` del backend para ser alojados por ASP.NET Core.

Sin embargo, esta arquitectura trae un punto de dolor clásico en la experiencia del usuario: cuando los usuarios visitan la página web, el navegador necesita cargar primero el HTML, luego descargar el enorme JS Bundle, y finalmente React ejecuta el Hydration (hidratación). En este "periodo de vacío" de varios cientos de milisegundos a unos pocos segundos, los usuarios se enfrentan a una pantalla completamente en blanco, o a una página estática sin vida.

Para llenar este intervalo e inyectar la personalidad de la marca de HagiCode, necesitamos diseñar un sistema de estilos de inicio completamente basado en código en línea en `index.html`.

## Sobre HagiCode

El plan de diseño de pantalla de inicio compartido en este artículo proviene de nuestra experiencia práctica en el proyecto [HagiCode](https://github.com/HagiCode-org/site). Como asistente de código con IA, HagiCode no solo se preocupa por la eficiencia de la generación de código, sino que también valora la experiencia visual de los desarrolladores. Este sistema de inicio es precisamente el producto de nuestra búsqueda de rendimiento frontend extremo.

## Desafíos centrales y diseño arquitectónico

Antes de comenzar el diseño, debemos primero clarificar las restricciones técnicas. Dado que debemos implementarlo en línea en `index.html`, significa que no podemos cargar ningún archivo CSS o JS externo (además del Bundle de React en sí).

### Análisis de restricciones técnicas

1.  **Principio de dependencia cero**: todos los estilos deben escribirse dentro de etiquetas `<style>`, la lógica dentro de etiquetas `<script>`.
2.  **CSS defensivo**: para evitar que los estilos globales contaminen la pantalla de inicio después de que la aplicación React se monte, decidimos usar prefijos ID de alta prioridad (como `#boot-screen`) para envolver todos los estilos de inicio.
3.  **Rendimiento primero**: las animaciones deben usar CSS `transform` y `opacity` tanto como sea posible, evitando activar reflow para asegurar no bloquear el hilo principal.
4.  **Consistencia visual**: los colores y fuentes deben mantenerse consistentes con la configuración de Tailwind de HagiCode.

### Patrón arquitectónico: Shell & Injector

Adoptamos un **patrón de variante**. La lógica central se encapsula en una función de ejecución inmediata (IIFE), la lógica de renderizado específica se inyecta como elemento de configuración. De esta manera, podemos cambiar diferentes estilos a través de una configuración simple, sin necesidad de repetir la lógica de manipulación DOM.

A continuación se muestra el código arquitectónico central:

```html
<!-- En línea en index.html -->
<div id="boot-root"></div>

<script>
(function() {
  const BootSequence = {
    config: {
      theme: 'terminal', // se puede configurar como 'minimal', 'skeleton', 'code-rain', etc.
      color: '#3b82f6'   // color de marca
    },
    
    // Ciclo de vida central
    init() {
      this.render();
      this.listenForMount();
    },

    // Renderizar el estilo seleccionado actualmente
    render() {
      const root = document.getElementById('boot-root');
      if (this.variants[this.config.theme]) {
        root.innerHTML = this.variants[this.config.theme].render();
      }
    },

    // Escuchar el montaje exitoso de React, salir elegantemente
    listenForMount() {
      window.addEventListener('hagicode:ready', () => {
        const screen = document.getElementById('boot-root');
        // Primero desvanecer, luego eliminar DOM, evitar parpadeo
        screen.style.opacity = '0';
        screen.style.transition = 'opacity 0.3s ease';
        setTimeout(() => screen.remove(), 300);
      });
    },

    // La lógica de implementación de 12 estilos se concentra aquí
    variants: {
      // ... implementación específica ver abajo
    }
  };

  BootSequence.init();
})();
</script>
```

## Lista de diseño de 12 estilos de inicio

Dividimos estos 12 estilos en seis categorías principales para satisfacer diferentes escenarios y necesidades estéticas.

### A. Minimalismo

> "Menos es más". Para escenarios que buscan velocidad de carga extrema, proporcionamos la solución más ligera.

#### 1. Minimalist Dot (respiración minimalista)
Solo un simple punto en el centro de la pantalla, combinado con animación de respiración.
*   **Implementación**: CSS `@keyframes` controla scale y opacity.
*   **Aplicable**: cualquier ocasión que necesite mantener la página absolutamente limpia.

#### 2. Brand Reveal (revelación de marca)
A través de la animación SVG `stroke-dasharray`, simula el dibujo a mano de las líneas del Logo de HagiCode, luego aparece el texto gradualmente.
*   **Técnica**: usar animación de ruta SVG, muy alta calidad.

### B. Mimetismo de pantalla de carga

> "El arte de engañar a los ojos". Al simular el diseño de UI real, los usuarios sienten que la página ya se ha cargado a medias.

#### 3. Sidebar Chat Skeleton (pantalla de carga de barra lateral)
Esta puede ser la más práctica. Construimos manualmente con HTML un diseño exactamente igual que los componentes React `Sidebar` y `ChatInput`, cubriendo animaciones de rayas grises.
*   **Valor**: cuando React completa el hydrate, la pantalla de carga se convierte instantáneamente en componentes reales, los usuarios casi no sienten el cambio.

#### 4. Card Stack Skeleton (pila de tarjetas)
Simula el efecto de apilamiento cuando se cargan las tarjetas de propuestas, usando transformaciones 3D para hacer que las tarjetas floten ligeramente.

### C. Abstracto y artístico

> Mostrar los genes geek de HagiCode.

#### 5. Geometric Morph (transformación geométrica)
Renderiza una forma geométrica (cuadrado) en el centro de la pantalla, que se transforma suavemente en círculo, triángulo con el tiempo, y finalmente se convierte en Logo.
*   **Tecnología**: transición suave de CSS `border-radius`.

#### 6. Code Rain (lluvia de código)
Homenaje a "The Matrix". Usa la fuente JetBrains Mono, dejando caer un flujo de caracteres tenue en el fondo.
*   **Nota**: por rendimiento, el flujo de caracteres debe limitarse a un área más pequeña o reducir la frecuencia de actualización.

#### 7. Neon Pulse (pulso de neón)
Anillos brillantes estilo cyberpunk, usando la superposición múltiple de `box-shadow` para producir una fuerte sensación de brillo.

### D. Marca y temas

> Hacer que el sistema "viva".

#### 8. Seasonal Theme (tema festivo)
Este es un cargador dinámico. Determina el festival según la fecha actual (como Año Nuevo chino, Navidad), cargando la animación SVG correspondiente.
*   **Ejemplo**: en Año Nuevo chino, habrá linternas rojas balanceando suavemente en la parte inferior de la pantalla.

#### 9. Gradient Flow (flujo degradado)
El fondo usa el degradado fluido del color de marca de HagiCode, combinado con animaciones de `background-size` y `background-position`, creando una sensación de flujo tipo aurora.

### E. Sensación técnica

> Homenaje a los desarrolladores.

#### 10. Terminal Boot (inicio de terminal)
Simula la salida de consola. Líneas de código se desplazan rápidamente:
```text
> Initializing HagiCode Core...
> Loading models...
> Connecting to neural network...
```
Esto hará que cada desarrollador se sienta cómodo.

#### 11. Progress Bar (barra de progreso minimalista)
Una barra de progreso delgada en la parte superior de la pantalla, mostrando el porcentaje a la derecha. Aunque no podemos obtener el progreso real de descarga, podemos usar un temporizador para simular un proceso de carga "creíble" (80% rápido, 20% lento).

### F. Creatividad

#### 12. Pixel Assembly (ensamblaje de píxeles)
Esta es una idea muy interesante. Algunos bloques están dispersos en la pantalla, se convergen hacia el centro, ensamblando gradualmente el icono del Logo de HagiCode. Simbolizando el proceso de construcción del código.

## Resumen de mejores prácticas y lecciones aprendidas

En el desarrollo real de HagiCode, resumimos algunos detalles de práctica cruciales.

### 1. El CSS defensivo es obligatorio
No seas perezoso y omitas prefijos. Una vez, no limitamos los estilos de la pantalla de inicio con ID, lo que causó que los estilos globales `div` después del montura de React afectaran accidentalmente la pantalla de inicio, provocando el colapso del diseño.
**Experiencia**: todos los selectores CSS deben colgarse de `#boot-screen`, y usar `!important` para aumentar la prioridad (solo en el CSS de la pantalla de inicio).

### 2. Transición elegante
Después del montura exitoso de React, no `remove()` directamente el DOM de la pantalla de inicio.
**Práctica correcta**:
1.  React dispara `window.dispatchEvent(new Event('hagicode:ready'))`.
2.  La pantalla de inicio escucha el evento, primero establece `opacity: 0`.
3.  Espera 300ms (tiempo de transición CSS), asegúrate de que el usuario no pueda verlo, luego ejecuta `.remove()`.

### 3. Sincronización de variables de tema
El código de color de la pantalla de inicio está escrito en `index.html`. Si modificamos el color principal de Tailwind, debemos modificarlo aquí sincrónicamente.
**Plan de optimización**: en el script de construcción de Vite, escribe un plugin simple, lee `tailwind.config.js` e inyecta las variables de color en las variables de plantilla de `index.html`, logrando una sola fuente de datos.

### 4. Precarga de fuentes
La pantalla de inicio generalmente necesita usar fuentes de marca, pero si la carga de fuentes es lenta, aparecerá FOUT (Flash of Unstyled Text).
**Solución**: agrega `<link rel="preload" href="/fonts/JetBrainsMono.woff2" as="font" type="font/woff2" crossorigin>` en `<head>`. Este es un medio de bajo costo y alto retorno para mejorar la experiencia.

### 5. Monitoreo de rendimiento
Inyectamos `performance.mark('boot-start')` en la parte inferior de `index.html`, y marcamos `boot-end` cuando el montura de React tiene éxito.
**Significado**: a través de Application Insights recopilamos estos datos, podemos ver verdaderamente hasta qué punto la pantalla de inicio acorta el tiempo de espera percibido por el usuario (Perceived Loading Time). Los datos indican que una excelente pantalla de carga puede aumentar la tolerancia de los usuarios a la "red lenta" en más del 50%.
