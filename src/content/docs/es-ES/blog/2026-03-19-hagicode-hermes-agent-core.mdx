## Por qué HagiCode elige Hermes como núcleo de Agente integral

> Al construir una plataforma de codificación asistida por IA, elegir el núcleo de Agente adecuado determina directamente el techo de las capacidades del sistema. Después de todo, algunas cosas no se pueden forzar——elegir el marco equivocado, por más que lo intentes, nunca se sentirá bien. Este artículo comparte el pensamiento de HagiCode en la selección técnica y la práctica de integración del Hermes Agent.

## Antecedentes

Hacer codificación asistida por IA, lo más doloroso es elegir el marco de Agente subyacente. En realidad, hay bastantes opciones en el mercado, solo que——algunas con funciones demasiado simples, otras con implementaciones demasiado complejas, otras con escalabilidad insuficiente. Lo que queremos es una solución que pueda ejecutarse tanto en un VPS de 5 dólares como en un clúster GPU. El requisito no es demasiado alto, pero tampoco bajo, a bastante gente la asusta.

Pero la realidad es que muchos llamados "Agentes todo en uno" solo pueden ejecutarse en la nube o tienen requisitos de implementación local ridículamente altos. Después de dos semanas investigando varias soluciones, tomamos una decisión audaz: reconstruir todo el núcleo del Agente, adoptando Hermes como motor subyacente para el Agente integral.

Todo lo que trajo esta decisión, quizás estaba destinado.

## Sobre HagiCode

La solución compartida en este artículo proviene de la experiencia práctica en el proyecto [HagiCode](https://hagicode.com). HagiCode es una plataforma de codificación asistida por IA que, a través de extensiones VSCode, clientes de escritorio y servicios web, proporciona a los desarrolladores un asistente de codificación inteligente. Quizás hayas usado herramientas similares, solo que siempre sentías que les faltaba algo——eso también lo entendemos.

- GitHub: [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site)
- Sitio web oficial: [hagicode.com](https://hagicode.com)

## Por qué HagiCode necesita Hermes

Antes de presentar Hermes en detalle, hablemos primero de por qué HagiCode tiene tales necesidades. En este mundo, las cosas a menudo no salen como quieres, siempre hay que encontrar una razón adecuada.

Como asistente de código con IA, HagiCode necesita soportar simultáneamente múltiples escenarios de uso:
- Entorno de desarrollo local: los desarrolladores desean ejecutarlo en sus propias computadoras, los datos no salen de lo local——en estos tiempos, la seguridad de datos no es algo demasiado grande ni demasiado pequeño
- Entorno de colaboración en equipo: los equipos pequeños pueden compartir un Agente desplegado en un servidor——para ahorrar dinero, nadie lo tiene fácil
- Escalabilidad elástica en la nube: al manejar tareas complejas, puede escalar automáticamente a clústeres GPU——por si acaso

Estas necesidades de "querer todo y más", nos hicieron mirar hacia Hermes. No sé si esta elección es correcta, solo que en ese momento no había otra mejor opción.

## Qué es Hermes Agent

Hermes Agent es un Agente de IA autónomo creado por **Nous Research**. Puede que algunos no conozcan Nous Research——son el laboratorio que desarrolló modelos grandes de código abierto como Hermes, Nomos y Psyché. Dicen que también les ha costado bastante, han hecho tantas cosas buenas pero poca gente las conoce.

A diferencia de los asistentes de programación IDE tradicionales o simples envoltorios de chat API, Hermes tiene una característica: cuanto más tiempo se ejecuta, más fuerte se vuelve. No se trata de completar una tarea de una vez y terminar, sino de poder continuar aprendiendo y acumulando experiencia en ejecuciones prolongadas. Esto también se parece bastante a los humanos, ¿verdad?

### Características principales

Las varias características principales de Hermes, coinciden perfectamente con las necesidades de HagiCode. Dime, ¿no es una coincidencia?

Esto significa que HagiCode puede elegir el método de implementación más adecuado según el escenario del usuario. Usuarios locales ejecutan localmente, usuarios de equipo despliegan en servidor, tareas complejas van a GPU——todo con un mismo código. En este mundo, poder ahorrar una cosa es ahorrar una cosa.

**Pasarela de mensajes multiplataforma**
Hermes soporta nativamente plataformas como Telegram, Discord, Slack, WhatsApp, etc. Para HagiCode, esto significa que en el futuro se pueden soportar fácilmente estos canales de asistente IA. Después de todo, ¿quién no quiere tener más caminos?

**Sistema de herramientas rico**
40+ herramientas incorporadas, más capacidad de extensión MCP (Model Context Protocol). Para un asistente de código esto es demasiado importante——ejecutar comandos shell, operar el sistema de archivos, llamar a Git, todo esto necesita soporte de herramientas. Un Agente sin herramientas es como un pájaro sin alas——quiere volar pero no puede.

**Memoria entre sesiones**
Hermes tiene un sistema de memoria persistente, usa recuperación de texto completo FTS5 para recordar conversaciones históricas. Esto permite que el Agente recuerde el contexto anterior, no "pierde la memoria" cada vez. A veces también quiero perder la memoria un poco, no pensar en nada, pero simplemente no puedo.

## Cómo HagiCode integra Hermes

Después de hablar del "por qué", veamos el "cómo". Algunas cosas, cuando las entiendes, tienes que ponerte manos a la obra, solo pensar sin hacer no sirve.

### Abstracción de capa Provider

En la arquitectura de HagiCode, todos los AI Provider implementan la interfaz unificada `IAIProvider`:

```csharp
public sealed class HermesCliProvider : IAIProvider, IVersionedAIProvider
{
    public ProviderCapabilities Capabilities { get; } = new ProviderCapabilities
    {
        SupportsStreaming = true,   // Soporta salida en flujo
        SupportsTools = true,       // Soporta llamadas a herramientas
        SupportsSystemMessages = true, // Soporta mensajes del sistema
        SupportsArtifacts = false
    };
}
```

Esta capa de abstracción permite a HagiCode cambiar sin problemas entre diferentes AI Provider, ya sea OpenAI, Claude o Hermes, la forma de llamada de la capa superior es completamente consistente. En el fondo, es para ahorrar trabajo.

### Protocolo de comunicación ACP

Hermes usa **ACP (Agent Communication Protocol)** para comunicarse. Este es un protocolo diseñado específicamente para la comunicación de Agentes, los métodos principales incluyen:

| Método | Descripción |
|------|------|
| `initialize` | Inicializa la conexión, obtiene la versión del protocolo y las capacidades del cliente |
| `authenticate` | Maneja la autenticación, soporta múltiples métodos de autenticación |
| `session/new` | Crea una nueva sesión, establece el directorio de trabajo y servidores MCP |
| `session/prompt` | Envía un prompt y obtiene una respuesta |

HagiCode implementa la capa de transporte ACP a través de `StdioAcpTransport`, inicia un subproceso Hermes y se comunica a través de entrada/salida estándar. Esto suena complicado, pero en la práctica se lleva bien——lo principal es tener paciencia.

### Flujo de ejecución completo

A continuación se presenta la lógica principal de ejecución del Hermes Provider:

```csharp
private async IAsyncEnumerable<AIStreamingChunk> StreamCoreAsync(
    AIRequest request,
    string? embeddedCommandPrompt,
    [EnumeratorCancellation] CancellationToken cancellationToken)
{
    // 1. Crear capa de transporte, iniciar subproceso Hermes
    await using var transport = new StdioAcpTransport(
        platformConfiguration.GetExecutablePath(),
        platformConfiguration.GetArguments(),
        platformConfiguration.GetEnvironmentVariables(),
        platformConfiguration.GetStartupTimeout(),
        _loggerFactory.CreateLogger<StdioAcpTransport>());
    await transport.ConnectAsync(cancellationToken);

    // 2. Inicializar, obtener versión del protocolo y métodos de autenticación
    var initializeResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "initialize",
        BuildInitializeParameters(platformConfiguration), cancellationToken);

    // 3. Manejar autenticación
    var authMethods = ParseAuthMethods(initializeResult);
    if (!isAuthenticated)
    {
        var methodId = platformConfiguration.Authentication.ResolveMethodId(authMethods);
        await SendHermesRequestAsync(transport, nextRequestId++, "authenticate", ...);
    }

    // 4. Crear sesión
    var newSessionResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "session/new",
        BuildNewSessionParameters(platformConfiguration, workingDirectory, model), cancellationToken);
    var sessionId = ParseSessionId(newSessionResult);

    // 5. Ejecutar prompt y recopilar respuestas en flujo
    await foreach (var payload in transport.ReceiveMessagesAsync(cancellationToken))
    {
        // Manejar notificaciones session/update, convertir a bloques en flujo
        if (TryParseSessionNotification(root, out var notification))
        {
            if (_responseMapper.TryConvertToStreamingChunk(notification, out var chunk))
            {
                yield return chunk;
            }
        }
    }
}
```

## Consideraciones en la práctica

En el proceso de integración de Hermes, hay algunos agujeros que vale la pena conocer de antemano. En estos tiempos, ¿quién no ha caído en unos cuantos agujeros?

### Configuración de métodos de autenticación

Hermes soporta múltiples métodos de autenticación (API Key, Token, etc.), es necesario elegir según la situación real de implementación. Una configuración incorrecta causará fallas de conexión, pero el mensaje de error puede no ser lo suficientemente intuitivo. A veces el mensaje de error está muy lejos de la causa real, hay que investigar poco a poco.

### Configuración de servidores MCP

Al crear una sesión se puede configurar una lista de servidores MCP, permitiendo que Hermes llame herramientas externas. Pero hay que tener en cuenta:
- Las direcciones de los servidores MCP deben ser accesibles
- Los tiempos de espera deben establecerse de manera razonable
- Manejo de degradación cuando los servidores no están disponibles

En este mundo, es imposible prevenir todo.

### Gestión del directorio de trabajo

Cada sesión necesita especificar un directorio de trabajo, asegurando que Hermes pueda acceder correctamente a los archivos del proyecto. Para escenarios de múltiples proyectos, es necesario cambiar dinámicamente el directorio de trabajo. Suena simple, pero en la práctica hay que considerar bastantes situaciones.

### Procesamiento de agregación de respuestas

Las respuestas de Hermes pueden dispersarse en notificaciones `session/update` y el resultado final, es necesario fusionarlas correctamente, de lo contrario habrá pérdida de contenido. Esto también me ha costado bastante, poco a poco mejora.

### Estrategia de manejo de errores

Los errores en tiempo de ejecución deben devolverse claramente, en lugar de volver silenciosamente a otros Provider. Así los usuarios saben que es Hermes el que tiene problemas, no que misteriosamente se cambió a otro modelo. Al final, engañar tampoco es una forma de engañar.

## Resumen

HagiCode eligió Hermes como núcleo de Agente integral, no fue una decisión improvisada, sino una elección cuidadosa basada en necesidades reales y características técnicas. Si esta elección es correcta, ahora es demasiado temprano para decirlo, solo que por ahora funciona bastante bien.

La capacidad de implementación flexible que proporciona Hermes permite a HagiCode adaptarse a varios escenarios de uso; el poderoso sistema de herramientas y soporte MCP permite que el asistente de IA realmente haga trabajo real; mientras que el protocolo ACP y la capa de abstracción Provider hacen que todo el proceso de integración sea claro y controlable.
