## Cómo hacer que la programación con IA sea más divertida con el diseño de gamificación

> En realidad, las herramientas tradicionales de programación con IA son bastante potentes, solo les falta un poco de calidez. Al desarrollar HagiCode pensamos: si de todos modos hay que escribir código, ¿por qué no convertirlo en un juego?

## Antecedentes

Quienes hayan utilizado asistentes de programación con IA probablemente hayan tenido esta experiencia: al principio parece bastante nuevo e interesante, pero después de usarlo por un tiempo sientes que falta algo. La herramienta en sí es muy potente, puede hacer generación de código, autocompletado, corrección de errores, todo lo que necesitas, pero... no tiene mucho calor, y después de usarla por un tiempo se vuelve monótona y aburrida.

Y eso es todo, después de todo, ¿quién quiere estar frente a una herramienta fría todos los días?

Es como jugar un videojuego, si simplemente completas una lista de tareas sin crecimiento de personajes, sin desbloqueo de logros, sin trabajo en equipo, pronto perderás el interés. Las cosas o personas bellas no necesitan ser poseídas, solo necesitan ser admiradas por su belleza. Pero las herramientas de programación ni siquiera tienen esa belleza, lo que inevitablemente desanima a las personas.

Durante el desarrollo de [HagiCode](https://hagicode.com) encontramos este problema. Como plataforma de colaboración de múltiples asistentes de IA, HagiCode necesita mantener el entusiasmo de los usuarios a largo plazo. Pero la realidad es que, incluso la mejor herramienta, si carece de conexión emocional, es difícil que los usuarios la sigan usando.

Para resolver este problema, tomamos una decisión audaz: convertir la programación en un juego. No un simple tablero de puntajes, sino una experiencia de juego de rol real. Los cambios que esta decisión trajo pueden ser mayores de lo que imaginas.

Después de todo, las personas siempre necesitan un poco de ceremonialidad.

## Sobre HagiCode

La solución compartida en este artículo proviene de nuestra experiencia práctica en el proyecto [HagiCode](https://hagicode.com). HagiCode es una plataforma de colaboración de múltiples asistentes de IA que admite el trabajo colaborativo de varios asistentes como Claude Code, Codex, Copilot, OpenCode, entre otros. Si te interesa la colaboración de múltiples IAs y la programación gamificada, puedes visitar [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site) para obtener más información.

En realidad, no hay nada especial, solo convertimos la programación en una aventura.

## Por qué elegir la gamificación

La esencia de la gamificación no es "agregar un tablero de puntajes", sino establecer un sistema completo de incentivos que permita a los usuarios experimentar una sensación de crecimiento, logro y reconocimiento social mientras completan tareas.

El diseño de gamificación de HagiCode gira en torno a un concepto central: cada asistente de IA es un "héroe", y el usuario es el capitán de este equipo de héroes. Tú lideras a estos héroes para conquistar varias "mazmorras" (tareas de programación), y en este proceso, los héroes ganarán experiencia, subirán de nivel y desbloquearán habilidades, y tú y tu equipo también obtendrán varios logros.

No es un truco de marketing, sino un diseño cuidadoso basado en la psicología del comportamiento humano. Cuando las tareas tienen significado y retroalimentación de progreso, el compromiso y la perseverancia de las personas mejoran significativamente.

Como dice el antiguo proverbio, "este sentimiento puede convertirse en un recuerdo, pero en ese momento ya estaba confundido". Integramos esta experiencia emocional en la herramienta, haciendo que la programación no sea solo escribir código, sino un viaje memorable.

## Sistema de personajes Hero

Hero es el concepto central del sistema de gamificación de HagiCode. Cada Hero representa un asistente de IA, por ejemplo, Claude Code es un Hero, Codex también es un Hero.

### Los tres slots de Hero

Hero tiene tres slots de equipamiento, este diseño es bastante ingenioso:

1. **Slot CLI (profesión principal)**: determina las habilidades básicas de Hero, como si es Claude Code o Codex
2. **Slot Model (profesión secundaria)**: determina el modelo a usar, como Claude 4.5 o Claude 4.6
3. **Slot Style (estilo)**: determina el estilo de comportamiento de Hero, como "estratega de Fengluo" u "otros estilos"

La combinación de los tres slots puede crear configuraciones únicas de Hero. Al igual que equipar en un juego, necesitas elegir la combinación adecuada según las características de la tarea. Después de todo, lo que te conviene es lo mejor, esto es similar a elegir un camino en la vida, todos los caminos llevan a Roma, solo que algunos son más fáciles y otros un poco más sinuosos.

### Sistema de crecimiento de Hero

Cada Hero tiene su propia XP (puntos de experiencia) y nivel:

```typescript
type HeroProgressionSnapshot = {
  currentLevel: number;                    // nivel actual
  totalExperience: number;                 // puntos de experiencia totales
  currentLevelStartExperience: number;     // experiencia de inicio del nivel actual
  nextLevelExperience: number;             // experiencia necesaria para el siguiente nivel
  experienceProgressPercent: number;       // porcentaje de progreso
  remainingExperienceToNextLevel: number;  // experiencia restante para el siguiente nivel
  lastExperienceGain: number;              // última experiencia obtenida
  lastExperienceGainAtUtc?: string | null; // hora de obtención de experiencia
};
```

Los niveles se dividen en cuatro etapas, y el nombre de cada etapa es muy inmersivo:

```typescript
export const resolveHeroProgressionStage = (level?: number | null): HeroProgressionStage => {
  const normalizedLevel = Math.max(1, level ?? 1);
  if (normalizedLevel <= 100) return 'rookieSprint';      // sprint de novato
  if (normalizedLevel <= 300) return 'growthRun';         // carrera de crecimiento
  if (normalizedLevel <= 700) return 'veteranClimb';      // escalada de veterano
  return 'legendMarathon';                                // maratón de leyenda
};
```

Desde "novato" hasta "leyenda", este camino de crecimiento da a los usuarios una sensación clara de objetivos y logros. Al igual que el crecimiento en la vida, siempre hay que pasar del desconocimiento a la madurez, solo que aquí este proceso se materializa.

### Crear un Hero personalizado

Al crear un Hero, necesitas configurar los tres slots:

```typescript
const heroDraft: HeroDraft = {
  name: 'Athena',
  icon: 'hero-avatar:storm-03',
  description: 'Estratega con gran inteligencia',
  executorType: AIProviderType.CLAUDE_CODE_CLI,
  slots: {
    cli: {
      id: 'profession-claude-code',
      parameters: { /* parámetros relacionados con CLI */ }
    },
    model: {
      id: 'secondary-claude-4-sonnet',
      parameters: { /* parámetros relacionados con el modelo */ }
    },
    style: {
      id: 'fengluo-strategist',
      parameters: { /* parámetros relacionados con el estilo */ }
    }
  }
};
```

Cada Hero tiene un avatar, descripción y posicionamiento profesional únicos, lo que hace que los fríos asistentes de IA tengan personalidad y calidez. Después de todo, ¿quién quiere tratar con herramientas sin personalidad?

## Sistema de mazmorras Dungeon

"Mazmorra" es un concepto clásico en los juegos, representa un desafío que debe ser superado en equipo. En HagiCode, cada flujo de trabajo es una Dungeon.

### Organización de Dungeon

Dungeon organiza los flujos de trabajo en diferentes "mazmorras":

- **Mazmorra de generación de propuestas**: responsable de generar propuestas técnicas
- **Mazmorra de ejecución de propuestas**: responsable de ejecutar las tareas de la propuesta
- **Mazmorra de archivado de propuestas**: responsable de organizar y archivar las propuestas completadas

Cada mazmorra tiene su propio Hero Captain (capitán), el capitán se selecciona automáticamente como el primer Hero habilitado.

En realidad, esto es como la división del trabajo en la vida, cada persona tiene su propio rol, solo que aquí esta división se convierte en un mecánico de juego.

### Mecanismo de colaboración en equipo

Puedes configurar diferentes equipos de Hero para diferentes mazmorras:

```typescript
const dungeonRoster: HeroDungeonRoster = {
  scriptKey: 'proposal.generate',
  displayName: 'Generación de propuestas',
  members: [
    { heroId: 'hero-1', name: 'Athena', executorType: 'ClaudeCode' },
    { heroId: 'hero-2', name: 'Apollo', executorType: 'Codex' }
  ]
};
```

Por ejemplo, usa Athena (experta en estrategia) al generar propuestas, y Apollo (experto en implementación) al ejecutar código, para que cada héroe pueda desempeñar sus fortalezas. Es como formar una banda, cada persona tiene su propio instrumento, y juntos pueden producir una melodía hermosa.

### Control de flujo de Dungeon

Dungeon usa un scriptKey fijo para identificar diferentes flujos:

```typescript
// las claves de script corresponden a diferentes flujos de trabajo
const dungeonScripts = {
  'proposal.generate': 'Generación de propuestas',
  'proposal.execute': 'Ejecución de propuestas',
  'proposal.archive': 'Archivado de propuestas'
};
```

El flujo de estado de las tareas es: queued (en cola) → dispatching (despachando) → dispatched (despachado). Todo el proceso está automatizado y no requiere intervención manual. Este es también nuestro pequeño truco de pereza, después de todo, ¿quién quiere administrar estas cosas manualmente?

## Sistema de XP y niveles

XP (puntos de experiencia) es el mecanismo de retroalimentación más central del sistema de gamificación. Los usuarios obtienen XP completando tareas, el XP hace que los héroes suban de nivel, subir de nivel desbloquea nuevas habilidades, formando un ciclo positivo.

### Formas de obtener XP

En HagiCode, XP se puede obtener a través de las siguientes actividades:

- Ejecución de código completada
- Llamada de herramienta exitosa
- Generación de propuestas
- Operaciones de gestión de sesiones
- Operaciones de proyecto

Cada vez que completes una operación válida, el Hero correspondiente obtendrá XP. Al igual que el crecimiento en la vida, cada paso cuenta, solo que aquí cuantificamos este crecimiento.

### Visualización de progreso en tiempo real

El progreso de XP y niveles se visualiza en tiempo real:

```typescript
type HeroDungeonMember = {
  heroId: string;
  name: string;
  icon?: string | null;
  executorType: PCode_Models_AIProviderType;
  currentLevel?: number;                    // nivel actual
  totalExperience?: number;                 // puntos de experiencia totales
  experienceProgressPercent?: number;       // porcentaje de progreso
};
```

Los usuarios pueden ver el nivel y progreso de cada Hero en cualquier momento, esta retroalimentación inmediata es clave en el diseño de gamificación. Después de todo, las personas siempre necesitan algo de retroalimentación, ¿si no cómo saben que están progresando?

## Sistema de logros

Los logros son otro elemento importante en la gamificación, proporcionan objetivos a largo plazo y una sensación de logro milestone.

### Tipos de logros

HagiCode admite varios tipos de logros:

- **Generación de código**: generar X líneas de código, generar Y archivos
- **Gestión de sesiones**: completar Z conversaciones
- **Operaciones de proyecto**: trabajar en W proyectos

En realidad, estos logros son como milestones en la vida, solo que los convertimos en un mecánico de juego.

### Estados de logros

Los logros tienen tres estados:

```typescript
type AchievementStatus = 'unlocked' | 'in-progress' | 'locked';
```

Los tres estados tienen una distinción visual clara:

- **Desbloqueado**: gradiente dorado + efecto de halo
- **En progreso**: animación de pulso azul
- **Bloqueado**: gris, muestra condiciones de desbloqueo

Cada logro muestra claramente las condiciones de activación, para que el usuario sepa qué hacer a continuación. Cuando estás perdido, tener una guía siempre es bueno.

### Efecto de celebración al desbloquear

Cuando se desbloquea un logro, se activa una animación de celebración. Este refuerzo positivo hará que los usuarios sientan la satisfacción de "lo logré", motivándolos a continuar. Es como pequeñas recompensas en la vida, aunque no sean grandes, pueden hacer feliz a alguien por mucho tiempo.

## Battle Report Reporte de batalla diario

Battle Report es una característica única de HagiCode, genera un informe de estilo de batalla en pantalla completa al final de cada día.

### Contenido del reporte

Battle Report muestra la siguiente información:

```typescript
type HeroBattleReport = {
  reportDate: string;
  summary: {
    totalHeroCount: number;        // número total de héroes
    activeHeroCount: number;       // número de héroes activos
    totalBattleScore: number;      // puntuación de batalla total
    mvp: HeroBattleHero;           // héroe más valioso
  };
  heroes: HeroBattleHero[];        // datos detallados de todos los héroes
};
```

- Puntuación total del equipo
- Número de Heroes activos
- Número de llamadas a herramientas
- Tiempo total de trabajo
- MVP (héroe más valioso)
- Tarjeta detallada de cada Hero

### Resaltado de MVP

MVP es el Hero con mejor rendimiento del día, se resaltará en el reporte. Esto no es solo estadística de datos, sino un reconocimiento de honor. Después de todo, ¿quién no quiere ser reconocido?

### Tarjeta detallada de Hero

La tarjeta de cada Hero contiene:

- Progreso de nivel
- Cantidad de XP obtenida
- Número de ejecuciones
- Tiempo de uso

Estos datos permiten a los usuarios entender claramente el estado de trabajo del equipo. Después de todo, conocer los resultados de tu propio esfuerzo también es una satisfacción.

## Implementación técnica

El sistema de gamificación de HagiCode utiliza una pila tecnológica moderna y patrones de diseño. En realidad, no hay nada especial, solo elegimos algunas herramientas convenientes.

### Selección de pila tecnológica

```typescript
// frontend usa React + TypeScript
import React from 'react';

// animaciones usan Framer Motion
import { AnimatePresence, motion } from 'framer-motion';

// gestión de estado usa Redux Toolkit
import { useAppDispatch, useAppSelector } from '@/store';

// componentes UI usan shadcn/ui
import { Dialog, DialogContent } from '@/components/ui/dialog';
```

Framer Motion es responsable de todos los efectos de animación, shadcn/ui proporciona componentes UI básicos, Redux Toolkit gestiona el estado complejo de gamificación. Después de todo, quien quiere hacer bien su trabajo debe primero afilar sus herramientas.

### Sistema de diseño UI de gamificación

HagiCode adopta el estilo de diseño Glassmorphism + Tech Dark:

```css
/* gradiente principal */
background: linear-gradient(135deg, #22C55E 0%, #25c2a0 50%, #06b6d4 100%);

/* efecto de vidrio */
backdrop-filter: blur(12px);

/* efecto de brillo */
background: radial-gradient(circle at center, rgba(34, 197, 94, 0.15) 0%, transparent 70%);
```

Los gradientes verdes combinados con efectos de vidrio crean una sensación tecnológica y futurista. Después de todo, la belleza visual también es parte de la experiencia del usuario.

### Implementación de efectos de animación

Usa Framer Motion para implementar animaciones de entrada fluidas:

```typescript
<motion.div
  animate={{ opacity: 1, y: 0 }}
  initial={{ opacity: 0, y: 18 }}
  transition={{ duration: 0.35, ease: 'easeOut', delay: index * 0.08 }}
  className="card"
>
  {/* contenido de la tarjeta */}
</motion.div>
```

Cada tarjeta entra secuencialmente, con un retraso de 0.08 segundos, creando un efecto visual fluido. Después de todo, las animaciones fluidas mejoran la experiencia, esto es indiscutible.

### Persistencia de datos

Los datos de gamificación usan el sistema de almacenamiento Grain, asegurando la consistencia del estado. Incluso los datos granulares como la acumulación de XP de Hero se pueden persistir con precisión. Después de todo, nadie quiere perder la experiencia acumulada con tanto esfuerzo.

## Guía práctica

### Crear el primer Hero

Crear el primer Hero es bastante simple:

1. Entra a la página de gestión de Hero
2. Haz clic en el botón "Crear Hero"
3. Configura los tres slots (CLI, Model, Style)
4. Dale un nombre y descripción al Hero
5. Guarda, y tu primer Hero nacerá

Es como conocer a un nuevo amigo, necesitas darle un nombre, conocer sus características, y luego pueden aventurar juntos.

### Formar equipo de Dungeon

Formar un equipo también es simple:

1. Entra a la página de gestión de Dungeon
2. Selecciona la mazmorra a configurar (como "Generación de propuestas")
3. Selecciona miembros de tu lista de Hero
4. El sistema selecciona automáticamente el primer Hero habilitado como capitán
5. Guarda la configuración

En realidad, este es un proceso de formar equipo, como formar un equipo en la vida, cada persona tiene su propio rol.

### Ver reporte diario

Al final de cada día, puedes ver el Battle Report del día:

1. Haz clic en el botón "Reporte de batalla"
2. Muestra en pantalla completa los resultados del trabajo del día
3. Ve los detalles de MVP y cada Hero
4. Comparte con miembros del equipo (opcional)

Esta también es una forma de ceremonialidad, para que sepas cuánto effort pusiste hoy y cuán lejos estás de tu objetivo.

## Consideraciones y mejores prácticas

### Optimización de rendimiento

Usa React.memo para evitar re-renderizados innecesarios:

```typescript
const HeroCard = React.memo(({ hero }: { hero: HeroDungeonMember }) => {
  // implementación del componente
});
```

Después de todo, el rendimiento también es importante, ¿quién quiere usar una herramienta con lag?

### Animaciones reducibles

Detecta la configuración de preferencia de movimiento del usuario, proporciona una experiencia simplificada para usuarios sensibles:

```typescript
const prefersReducedMotion = useReducedMotion();
const duration = prefersReducedMotion ? 0 : 0.35;
```

Después de todo, no a todos les gustan las animaciones, respetar las preferencias del usuario también es parte del diseño.

### Compatibilidad hacia atrás

Mantén legacyIds para admitir la migración de versiones antiguas:

```typescript
type HeroDungeonMember = {
  heroId: string;
  legacyIds?: string[];  // soporta mapeo de ID de versión antigua
  // ...
};
```

Después de todo, nadie quiere perder datos debido a una actualización de versión.

### Soporte de internacionalización

Todo el texto usa claves de traducción i18n, facilitando el soporte multilingüe:

```typescript
const displayName = t(`dungeon.${scriptKey}`, { defaultValue: displayName });
```

Después de todo, el idioma no debería ser una barrera para el uso.

## Conclusión

La gamificación no es un simple tablero de puntajes, sino un sistema completo de incentivos. HagiCode convierte el trabajo de programación en un viaje heroico lleno de espíritu de aventura a través del sistema Hero, el sistema Dungeon, el sistema de XP/niveles, el sistema de logros y Battle Report.

El valor central de este sistema radica en:

- **Conexión emocional**: hace que los fríos asistentes de IA tengan personalidad
- **Retroalimentación positiva**: cada operación tiene retroalimentación inmediata
- **Objetivos a largo plazo**: niveles y logros proporcionan rutas de crecimiento
- **Identidad de equipo**: sensación de colaboración en equipo de Dungeon
- **Reconocimiento de honor**: Battle Report y exhibición MVP

El diseño de gamificación hace que la programación ya no sea aburrida, sino una aventura interesante. Los usuarios, al completar tareas de código, experimentan la diversión del crecimiento de personajes, la colaboración en equipo y el desbloqueo de logros, mejorando así la retención y actividad.

En realidad, la programación en sí es una forma de creación, solo hicimos este proceso de creación un poco más interesante.
