## 先完成 OpenCode 官方安装

### 常见安装方式

根据 `repos/opencode/README.zh.md` 当前内容，OpenCode CLI 常用安装方式包括：

```bash
# 官方安装脚本
curl -fsSL https://opencode.ai/install | bash
```

```bash
# npm
npm i -g opencode-ai@latest
```

```bash
# Homebrew
brew install anomalyco/tap/opencode
```

```powershell
# Windows
scoop install opencode
```

```powershell
# Windows
choco install opencode
```

如果你使用的是 Arch Linux、mise、nix 或其他方式，也可以继续按 OpenCode README 中列出的路径安装。

### 安装完成后的最小验证

安装后先执行：

```bash
opencode --version
```

**预期结果**：终端返回版本号，说明 `opencode` 已经进入当前环境的 PATH。

### 可选：桌面版

OpenCode README 还提供了桌面版安装方式：

```bash
brew install --cask opencode-desktop
```

如果你只是要把 OpenCode 接入 HagiCode，CLI 即可；桌面版不是 HagiCode 集成的必需前提。

## HagiCode 当前的 OpenCode 集成基线

根据本仓库当前实现，HagiCode 把 OpenCode 当作一个独立主职业，并在前端与后端都保留了固定的接入基线。

### 主职业默认参数

`repos/web/src/types/hero.ts` 当前给出的 OpenCode 主职业默认参数是：

```yaml
binary: opencode
runtimeModel: anthropic/claude-sonnet-4
managedRuntimeModel: glm-4.7
requestTimeoutSeconds: 3600
```

这意味着：

- CLI 可执行名默认是 `opencode`
- OpenCode 主运行模型默认会带上 `anthropic/claude-sonnet-4`
- HagiCode 当前也保留了一个 `glm-4.7` 的托管运行模型标记
- 长请求场景下，前端预设的超时时间比普通 provider 更长

### 后端 / 运行时配置参考

`HagiCode 多 AI Provider 架构` 文章里给出的 OpenCode 配置示例如下：

```yaml
AI:
  Providers:
    OpenCodeCli:
      Type: "OpenCodeCli"
      Enabled: true
      ExecutablePath: "opencode"
      Model: "anthropic/claude-sonnet-4"
      WorkingDirectory: null

OpenCode:
  Enabled: true
  BaseUrl: "http://localhost:38376"
  ExecutablePath: "opencode"
  StartupTimeoutSeconds: 30
  RequestTimeoutSeconds: 120
```

从这个基线可以看出，HagiCode 当前对 OpenCode 的依赖至少包括：

- `OpenCodeCli` provider 已启用
- 本机可直接找到 `opencode` 可执行文件
- OpenCode runtime 能在本机正常启动
- 默认 HTTP 入口可以被 HagiCode 访问

## 推荐的接入检查顺序

建议按下面的顺序排查：

1. 先完成 OpenCode 官方安装
2. 运行 `opencode --version`，确认命令可用
3. 单独启动一次 OpenCode，确认 CLI 本身没有初始化错误
4. 再回到 HagiCode，检查 Provider 状态、启动日志或健康检查输出
5. 发起一次最小请求，确认 `OpenCodeCli` 能返回内容

## 常见问题排查

| 问题现象 | 更可能的原因 | 建议处理方式 |
|---|---|---|
| `opencode: command not found` | CLI 安装完成，但 PATH 没生效 | 重新打开终端；检查安装目录是否已加入 PATH |
| HagiCode 启动 OpenCode 失败 | HagiCode 找不到 `ExecutablePath: "opencode"` | 先在同一台机器上确认 `opencode --version` 正常，再检查 HagiCode 启动环境 |
| OpenCode runtime 无法响应 | 本地 runtime 没成功启动，或 `BaseUrl` 不可达 | 先确认 OpenCode 本身可启动，再核对 HagiCode 中的 endpoint 与端口配置 |
| 请求总是超时 | 当前任务超出默认请求窗口，或模型响应太慢 | 先降低任务规模；必要时检查 `RequestTimeoutSeconds` 是否被覆盖 |
| 模型行为和预期不符 | `Model` / `runtimeModel` / `managedRuntimeModel` 配置不一致 | 对照当前 HagiCode 基线，统一主模型与托管模型设置 |
