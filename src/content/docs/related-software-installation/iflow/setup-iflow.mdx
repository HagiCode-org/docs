## iFlow 简介

这篇页面优先复述 iFlow 官方 `Quickstart` 的安装与首次登录流程，再补充 HagiCode 当前的 ACP WebSocket 接入要求。

也就是说：

1. 先按 iFlow 官方文档把 CLI 安装并登录成功
2. 再检查 HagiCode 所需的 `--experimental-acp --port {port}` 启动方式

## 适用范围与官方来源

- **适用对象**：希望在本地终端安装 iFlow CLI，并进一步接入 HagiCode 的用户
- **最后核对日期**：2026-03-13
- **官方快速开始**：[iFlow CLI Quickstart](https://platform.iflow.cn/cli/quickstart)
- **HagiCode 集成基线**：`ExecutablePath: "iflow"`、`Arguments: "--experimental-acp --port {port}"`、`AuthMethod: "iflow"`

## 先按官方快速开始完成安装

### 官方系统要求

根据 iFlow 官方快速开始页面，安装前至少需要：

- Node.js `22+`
- `4GB+` 内存
- 互联网连接
- 基本的终端操作经验

官方页面还特别建议 Windows 用户优先使用 **Windows Terminal**，以减少兼容性问题。

### macOS / Linux 官方安装方式

如果你想使用官方一键脚本，官方页面给出的命令是：

```bash
bash -c "$(curl -fsSL https://gitee.com/iflow-ai/iflow-cli/raw/main/install.sh)"
```

如果你的机器已经安装了 Node.js `22+`，官方页面也给出了 npm 方式：

```bash
npm install -g @iflow-ai/iflow-cli@latest
```

### Windows 官方安装方式

官方快速开始页给出的顺序是：

1. 先从 Node.js 官网安装最新版本 Node.js
2. 重启 CMD 或 PowerShell
3. 执行以下命令安装 iFlow CLI

```bash
npm install -g @iflow-ai/iflow-cli@latest
```

安装完成后，官方页面建议直接运行：

```bash
iflow --version
```

**预期结果**：终端返回版本号，说明 CLI 安装成功。

## 先按官方流程完成首次设置

### 1) 首次启动

官方快速开始页给出的入口命令就是：

```bash
iflow
```

### 2) 选择登录方式

官方页面说明，iFlow CLI 当前支持三种登录方式：

- **Login with iFlow（推荐）**：功能最完整，支持 WebSearch、WebFetch、多模态与更完整的工具调用优化
- **iFlow API Key**：适合服务器环境或无法打开浏览器的场景；官方页面注明 API Key 有效期为 `7` 天，需要定期更新
- **OpenAI Compatible API**：适合接入自有模型或兼容 OpenAI 协议的服务，但不具备 iFlow 平台提供的 WebSearch、WebFetch 和内置多模态能力

:::warning
iFlow 官方快速开始页面在 `2025-10-10` 的重要通知中明确说明：认证方式会统一切换，届时需要手动更新 API Key，并建议使用 `/auth` 通过 iFlow 登录，避免服务中断。
:::

### 3) 完成首次登录后再继续 HagiCode 集成

先确保你已经能在终端里正常进入 iFlow 会话，然后再继续下面的 HagiCode 专属配置。这样可以把“官方登录问题”和“HagiCode 接入问题”分开排查。

## HagiCode 额外需要的 ACP WebSocket 校验

上面的步骤已经覆盖了官方安装与首次登录；而 HagiCode 还会额外要求 iFlow 以 ACP WebSocket 模式启动。

### 1) 先做一次本地 ACP 启动检查

HagiCode 当前实现会以如下方式启动 iFlow：

```bash
iflow --experimental-acp --port 8765
```

**预期结果**：CLI 成功监听本地端口；验证完成后可使用 `Ctrl+C` 退出。

如果命令直接报错、端口无法绑定，或当前版本不识别 `--experimental-acp`，请先解决这些问题，再继续 HagiCode 集成。

### 2) 验证 HagiCode 已识别 iFlow

建议按下面顺序检查：

1. 先按官方 Quickstart 完成安装，并运行 `iflow --version`
2. 执行 `iflow`，确认你已经完成首次登录或 `/auth` 登录流程
3. 手动运行 `iflow --experimental-acp --port 8765`，确认 ACP 模式能启动
4. 启动 HagiCode，检查 Provider 状态、健康检查或启动日志
5. 确认没有出现端口占用、鉴权失败、找不到 `iflow` 或 ACP 建连失败等错误

## 常见问题排查

| 问题现象 | 更贴近官方文档的判断方式 | 建议处理步骤 |
|---|---|---|
| `iflow` 命令不存在 | 官方安装步骤尚未完成，或安装后的终端没有重启 | 重新打开终端；重新执行安装命令；再用 `iflow --version` 验证 |
| 首次登录卡住或认证失败 | 登录方式选择不对，或 `2025-10-10` 后仍沿用旧 API Key 习惯 | 优先使用官方推荐的 `Login with iFlow` 或 `/auth`；若使用 API Key，确认密钥仍在有效期内 |
| API Key 登录后能力不完整 | 官方页面区分了 iFlow 平台登录与 OpenAI Compatible API 模式 | 若需要 WebSearch、WebFetch、多模态等平台能力，改用 `Login with iFlow` 或 iFlow API Key |
| `--experimental-acp` 报错 | 这是 HagiCode 集成附加要求，不属于官方 Quickstart 的基础安装验证 | 先确认 CLI 安装和登录正常，再单独排查当前版本、端口占用和本地安全软件 |
| HagiCode 无法连接 iFlow | CLI 已安装，但 WebSocket、本地端口或鉴权没有通过 | 保留启动日志，重点检查端口绑定、登录状态和本机网络环境 |
