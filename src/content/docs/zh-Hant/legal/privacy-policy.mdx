生效日期：2026年4月15日

本隱私權政策解釋了 HagiCode 在提供資料時可能處理的資料類別 `hagicode.com` 及其所有子網域網站、軟體下載指南、支援管道和相關產品體驗，以及您可用的目的、保留方法和權利。

## 我們收集的資訊類別

根據您與 HagiCode 互動的方式，我們可能會處理：

- **存取和裝置資訊**，例如瀏覽器類型、作業系統、存取的頁面、引用連結、時間戳記、粗略區域和裝置識別碼；
- **使用行為資料**，例如頁面瀏覽量、頁面停留時間、點擊次數、捲動活動、會話記錄以及用於了解的聚合分析 `hagicode.com` 及其所有子網域網站以及相關產品頁面；
- **支援和溝通訊息**，例如您的姓名、電子郵件地址、問題描述、日誌摘錄或您透過電子郵件、問題、社區管道或表格自願提交的附件；和
- **下載和設定上下文**，例如您造訪的安裝指南、下載條目、預設文件或發行說明，以幫助我們了解文件和發行品質。

## 使用目的

我們處理該資訊的目的是：

- 提供和維護 `hagicode.com` 及其所有子網域網站、下載入口點和支援管道；
- 了解哪些頁面、設定步驟或產品說明需要改進；
- 解決網站錯誤、下載問題、相容性故障和安全風險；
- 回應您的問題、支持請求、回饋或投訴；和
- 必要時滿足適用的法律、監管、執法或平台安全要求。

## 站點和產品資料接觸點

目前的HagiCode `hagicode.com` property 及其所有子網域網站可能透過以下方式接觸相關資料：

- 網站存取日誌和基本請求元資料；
- Microsoft Clarity 或類似的頁面分析用於了解閱讀流程和互動熱點；
- 51LA 或類似的流量分析用於了解流量趨勢和引薦來源網址；和
- 您自願發送至的支持訊息 `support@hagicode.com` 或 GitHub 問題。

如果 HagiCode 桌面用戶端、後端服務或第三方 AI 整合在單獨的產品上下文中處理其他數據，則這些詳細資訊會記錄在相關產品材料、設定或第三方條款中。

## 儲存和保留

我們根據營運需求、法律義務和安全要求將資料保留合理的期限：

- 分析資料通常以匯總形式保存或按照分析提供者的預設保留規則保存；
- 在解決問題、審核通訊或履行法律職責所需時保留支援電子郵件、問題報告和故障排除記錄；和
- 出於安全、防止濫用、合規性保留或爭議解決的需要，某些日誌和支援記錄可能會保留更長時間。
- 某些資料可能透過 Microsoft Azure、阿里雲或相關基礎架構功能進行託管、處理、傳輸、備份或以其他方式處理，這意味著相關資料可能在不同國家或地區之間移動。

當這些目的不再需要資料時，我們會將其刪除、匿名化或允許其根據平台的預設保留規則過期。

## 第三方服務

HagiCode 可能依賴第三方基礎設施、分析提供者、程式碼託管平台、電子郵件服務或人工智慧服務供應商。這些各方僅根據提供服務所需的數據處理數據，並遵守各自的條款和隱私權聲明。

在目前或未來的產品和網站營運中，我們可能會透過 Microsoft Azure 和阿里雲等雲端基礎架構處理或以其他方式處理某些資料。這意味著資料可以跨不同國家或地區傳輸以進行儲存、備份、日誌分析、故障排除、內容交付或服務編排。

在合理可行的情況下，我們會以匿名、匯總或去識別化的形式使用相關數據進行營運分析、服務改進、穩定性工作和安全保護。但是，您瞭解並同意，我們無法完全預防、控製或保證雲端供應商、基礎設施路徑或跨區域傳輸始終保留在單一國家或地區內。

當您使用第三方整合時，您還應該查看相關提供者政策，以了解他們自己的處理和跨境傳輸實踐。

如果您無法接受這種基於雲端的處理、跨境傳輸或相關的資料匿名使用，請勿下載、安裝、存取或繼續使用HagiCode產品、服務或 `hagicode.com` 及其任何子網域網站。

## 使用者權利

在適用法律規定的範圍內，您可以要求：

- 了解我們是否持有與您相關的個人資訊；
- 修正不準確或不完整的資訊；
- 刪除不再需要的資訊或您有權刪除的資訊；
- 限製或反對某些處理活動；和
- 取得有關支持溝通或投訴的進一步解釋。

如果請求影響帳戶安全、法律義務、系統安全、濫用預防或第三方平台限制，我們可能需要先驗證身分或在法律允許的情況下保留有限的資料。

## 政策更新

如果 HagiCode 網站功能、分析工具、聯絡管道或合規要求發生變化，我們可能會更新本隱私權政策。更新版本將在此頁面上發布，並附有修訂後的生效日期。
