# 安裝 OpenSpec

OpenSpec 管理提案、規格和變更記錄。此页面致力于实现一个目标：清楚地解释安装路径，以便您可以将其附加到现有的工作流程中。

## 什么是 OpenSpec？

OpenSpec 是一個命令列工具，用於管理軟體工作中的提案和規格。核心工作流程包括：

- **提案管理**：创建、审查和跟踪功能提案
- **規範驗證**：確保程式碼變更符合預先定義的規範
- **變更追蹤**：記錄所有已審核的變更歷史記錄
- **團隊協作**：支援多人協作開發

### 在HagiCode專案中的應用

在 HagiCode 專案中，OpenSpec 用於：

1. **功能开发管理**：所有新功能都必须经过 OpenSpec 提案流程
2. **代码审查**：确保更改符合项目规范
3. **文檔同步**：保持文檔和程式碼實現的一致性
4. **版本控制集成**：與版本控制工作流程無縫集成

## 先決條件

在安裝 OpenSpec 之前，請確保您的系統符合以下要求：

### Node.js 和 npm

OpenSpec 透過 npm 分發，需要 Node.js 環境。 OpenSpec 需要 **Node.js 18.0 或更高版本**。

驗證 Node.js 安裝：

```bash
node --version
```

驗證 npm 安裝：

```bash
npm --version
```

若未安裝或版本過舊，請參考相關軟體安裝說明 [安裝指南](/zh-Hant/installation).

## 安裝 OpenSpec

:::提示[版本要求]
此專案需要使用**OpenSpec 1.0.0 或更高**版本範圍。支援所有 1.x 版本，以確保與專案工作流程的兼容性。
:::

OpenSpec 透過 npm 在全球範圍內安裝，使其可以從系統上的任何位置使用。

### 使用 npm 全域安裝

<Tabs>
  <TabItem value="win" label="Windows">
    在 PowerShell 或命令提示字元中執行：

```powershell
npm install -g @fission-ai/openspec@1
```

:::注意[Windows 權限問題]
如果在安裝過程中遇到權限錯誤，請以管理員身分執行 PowerShell 或命令提示字元。
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    在終端機中運作：

```bash
npm install -g @fission-ai/openspec@1
```

:::注意[macOS 權限問題]
如果你遇到 `EACCES` 權限錯誤，嘗試：
```bash
sudo npm install -g @fission-ai/openspec@1
```
或使用 [非揮發性](https://github.com/nvm-sh/nvm) 管理 Node.js 版本並避免權限問題。
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    在終端機中運作：

```bash
npm install -g @fission-ai/openspec@1
```

:::注意[Linux 權限問題]
如果你遇到 `EACCES` 權限錯誤，您可以：
1. 使用 `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. 配置 npm 以使用使用者目錄：
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### 驗證安裝

安裝完成後，驗證OpenSpec是否安裝正確且版本符合要求：

```bash
openspec --version
```

如果安裝成功，您將看到版本號輸出，例如：

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::注意[版本範圍]
請確保安裝的 OpenSpec 版本為 1.0.0 或更高版本。該專案已驗證與1.1.1版本的兼容性。建議使用最新的兼容版本以獲得新功能。
:::

## 常見問題和故障排除

### 權限錯誤

#### 問題： `EACCES` 權限被拒絕

安裝或執行 OpenSpec 時遇到權限錯誤。

**解決方案**：

<Tabs>
  <TabItem value="win" label="Windows">
    以管理員身分執行 PowerShell 或命令提示字元：
    1. 右鍵點選 PowerShell 或命令提示符
    2. 選擇“以管理員身份執行”
    3. 重新運行安裝命令
  </TabItem>
  <TabItem value="mac" label="macOS">
    安裝使用 `sudo`:

```bash
sudo npm install -g @fission-ai/openspec@1
```

或使用 [非揮發性](https://github.com/nvm-sh/nvm) 管理 Node.js 並避免全域安裝權限問題。
  </TabItem>
  <TabItem value="linux" label="Linux">
    解決方案1：使用sudo
```bash
sudo npm install -g @fission-ai/openspec@1
```

解決方案 2：配置 npm 以使用使用者目錄
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @fission-ai/openspec@1
```
  </TabItem>
</Tabs>

### 網路問題

#### 問題：npm 安裝失敗或非常慢

由於網路原因，無法從 npm 註冊表下載 OpenSpec。

**解決方案**：

1. **使用國內npm鏡像**：

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **恢復官方來源**（如果需要）：

```bash
npm config set registry https://registry.npmjs.org
```

3. **使用 cnpm** （可選）：

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

:::提示
推薦中國大陸用戶使用淘寶鏡像，可顯著提高下載速度。
:::

### 版本相容性問題

#### 問題：OpenSpec 指令運行異常或報告錯誤

安裝的 OpenSpec 版本與目前環境不相容。

**解決方案**：

1. **檢查 Node.js 版本**：

```bash
node --version
```

確保版本為 18.0 或更高版本。如果版本過低，請升級Node.js。

2. **檢查 OpenSpec 版本**：

```bash
openspec --version
```

3. **重新安裝特定版本**：

```bash
npm install -g @fission-ai/openspec@1
```

4. **清除 npm 快取**（如果問題仍然存在）：

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### 未找到指令

#### 問題：打字 `openspec` 命令顯示“未找到命令”

OpenSpec 已安裝，但係統找不到該指令。

**解決方案**：

<Tabs>
  <TabItem value="win" label="Windows">
    1. 確認npm全域安裝路徑：

```powershell
npm config get prefix
```

2. 新增路徑到系統PATH：
   - 開啟“系統屬性”→“進階”→“環境變數”
   - 尋找 `Path` 在「系統變數」中
   - 新增npm全域路徑（通常 `C:\Users\<username>\AppData\Roaming\npm`)
   - 重新啟動命令列視窗
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. 確認npm全域安裝路徑：

```bash
npm config get prefix
```

2. 將路徑加入 PATH（在 `~/.zshrc` 或 `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. 重新載入配置：

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. 確認npm全域安裝路徑：

```bash
npm config get prefix
```

2. 將路徑加入 PATH（在 `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. 重新載入配置：

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
