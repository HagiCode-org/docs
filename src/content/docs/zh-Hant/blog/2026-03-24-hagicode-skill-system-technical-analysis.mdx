## HagiCode Skill 系統技術解析：如何打造可擴展的 AI 技能管理平台

> 本文深入解析 HagiCode 項目中 Skill（技能）管理系統的架構設計與實現方案，涵蓋本地全局管理、市場搜索、智能推薦、授信提供者管理四大核心功能的技術實現細節。

## 背景

在 AI 代碼助手這個領域，如何擴展 AI 的能力邊界，其實一直是一個核心課題。Claude Code 本身的代碼輔助能力是挺強的，只是不同開發團隊、不同技術棧，往往需要針對特定場景的專業能力——比如處理 Docker 部署、數據庫優化、前端組件生成之類的。這時候，Skill（技能）系統就顯得尤為重要了。

HagiCode 項目在開發過程中也遇到了類似的挑戰：怎麼讓 Claude Code 能夠像人一樣「學會」新的專業技能，同時保持良好的用戶體驗和工程可維護性？畢竟這個問題，說難也難，說簡單也簡單。圍繞這個問題，我們設計並實現了一套完整的 Skill 管理系統。

本文將詳細解析這個系統的技術架構和核心實現，適合對 AI 擴展性、命令行工具集成感興趣的開發者閱讀。或許對你有用，或許沒用，但總歸是寫出來了。

## 關於 HagiCode

本文分享的方案來自我們在 [HagiCode](https://hagicode.com) 項目中的實踐經驗。HagiCode 是一個開源的 AI 代碼助手項目，旨在幫助開發團隊提升研發效率。項目的技術棧涵蓋 ASP.NET Core、Orleans 分佈式框架、TanStack Start + React 前端，以及本文要介紹的 Skill 管理子系統。

項目的 GitHub 地址是 [HagiCode-org/site](https://github.com/HagiCode-org/site)，如果你覺得本文介紹的技術方案有價值，歡迎給個 Star。畢竟 Star 多了，心情也會好一些。

## 系統架構概覽

Skill 系統採用前後端分離的架構設計，說起來也沒什麼特別的。

**前端部分** 使用 TanStack Start + React 構建用戶界面，通過 Redux Toolkit 管理狀態，四個主要功能分別對應四個 Tab 組件：本地技能、市場畫廊、智能推薦、授信提供者。這樣設計，其實也是為了用戶體驗罷了。

**後端部分** 基於 ASP.NET Core + ABP Framework，使用 Orleans Grain 實現分佈式狀態管理。在線 API 客戶端封裝了 `IOnlineApiClient` 接口，用於與遠程技能目錄服務通信。

整體架構的設計原則是「命令執行與業務邏輯分離」，通過適配器模式將 npm/npx 命令執行的細節屏蔽在獨立模塊中。畢竟誰願意看到一堆命令行散落在代碼各處呢？

## 核心功能一：本地全局管理

本地全局管理是最基礎的功能模塊，負責列出已安裝的技能並支持卸載操作。也沒什麼複雜的，就是把事情做好而已。

### 技術方案

實現位置在 `LocalSkillsTab.tsx` 和 `LocalSkillCommandAdapter.cs`。核心思路是封裝 `npx skills` 命令，解析其 JSON 輸出，轉換為內部數據結構。說起來簡單，做起來其實也簡單。

```csharp
public async Task<IReadOnlyList<LocalSkillInventoryResponseDto>> GetLocalSkillsAsync(
    CancellationToken cancellationToken = default)
{
    var result = await _commandAdapter.ListGlobalSkillsAsync(cancellationToken);
    return result.Skills.Select(skill => new LocalSkillInventoryResponseDto
    {
        Name = skill.Name,
        Version = skill.Version,
        Source = skill.Source,
        InstalledPath = skill.InstalledPath,
        Description = skill.Description
    }).ToList();
}
```

數據流非常清晰：前端發起請求 → SkillGalleryAppService 接收 → LocalSkillCommandAdapter 執行 npx 命令 → 解析 JSON 結果 → 返回 DTO 對象。一環扣一環，也沒什麼好說的。

卸載技能使用 `npx skills remove -g <skillName> -y` 命令，系統會自動處理依賴關係和清理工作。安裝元數據存儲在技能目錄的 `managed-install.json` 中，記錄了安裝時間、來源版本等信息，便於後續更新和審計。畢竟有些東西，記下來總是好的。

### 安裝流程詳解

技能安裝涉及多個步驟的協調，怎麼說呢，其實也不算太複雜：

```csharp
public async Task<SkillInstallResultDto> InstallAsync(
    SkillInstallRequestDto request,
    CancellationToken cancellationToken = default)
{
    // 1. 規範化安裝引用
    var normalized = _referenceNormalizer.Normalize(
        request.SkillId,
        request.Source,
        request.SkillSlug,
        request.Version);

    // 2. 檢查先決條件
    await _prerequisiteChecker.CheckAsync(cancellationToken);

    // 3. 獲取安裝鎖
    using var installLock = await _lockProvider.AcquireAsync(normalized.SkillId);

    // 4. 執行安裝命令
    var result = await _installCommandRunner.ExecuteAsync(
        new SkillInstallCommandExecutionRequest
        {
            Command = $"npx skills add {normalized.FullReference} -g -y",
            Timeout = TimeSpan.FromMinutes(4)
        },
        cancellationToken);

    // 5. 持久化安裝元數據
    await _metadataStore.WriteAsync(normalized.SkillPath, request);

    return new SkillInstallResultDto { Success = result.Success };
}
```

這裡用到了幾個關鍵的設計模式：**引用規範化器** 負責將各種輸入格式（如 `tanweai/pua`、`@opencode/docker-skill`）轉換為統一的內部表示；**安裝鎖機制** 確保同一技能同時只有一個安裝操作在進行；**流式輸出** 通過 Server-Sent Events 向前端實時推送安裝進度，用戶可以看到類似終端的實時日誌。

這些設計模式，說到底，也還是為了讓事情變得簡單罷了。

## 核心功能二：市場搜索

市場搜索讓用戶能夠發現和安裝來自社區的技能。畢竟一個人的能力是有限的，眾人的智慧才是無窮的。

### 技術方案

搜索功能依賴在線 API `https://api.hagicode.com/v1/skills/search`。為了提升響應速度，系統實現了緩存機制。緩存這東西，就像記憶一樣，有些東西記住了，下次就不用再費勁去想。

```csharp
private async Task<IReadOnlyList<SkillGallerySkillDto>> SearchCatalogAsync(
    string query,
    CancellationToken cancellationToken,
    IReadOnlySet<string>? allowedSources = null)
{
    var cacheKey = $"skill_search:{query}:{string.Join(",", allowedSources ?? Array.Empty<string>())}";

    if (_memoryCache.TryGetValue(cacheKey, out var cached))
        return (IReadOnlyList<SkillGallerySkillDto>)cached!;

    var response = await _onlineApiClient.SearchAsync(
        new SearchSkillsRequest
        {
            Query = query,
            Limit = _options.LimitPerQuery,
        },
        cancellationToken);

    var results = response.Skills
        .Where(skill => allowedSources is null || allowedSources.Contains(skill.Source))
        .Select(skill => new SkillGallerySkillDto { ... })
        .ToList();

    _memoryCache.Set(cacheKey, results, TimeSpan.FromMinutes(10));
    return results;
}
```

搜索結果支持按授信來源過濾，只顯示用戶信任的技能源。預置的種子查詢用於初始化目錄，比如「popular」、「recent」等，讓用戶在首次打開時就能看到推薦的熱門技能。畢竟第一印象還是重要的。

## 核心功能三：智能推薦

智能推薦是系統中最複雜的功能，它能根據用戶當前項目的情況，自動推薦最適合的技能。複雜歸複雜，但做出來還是值得的。

### 推薦流程

整個推薦流程分為五個階段：

```
1. 構建項目上下文
   ↓
2. AI 生成搜索查詢
   ↓
3. 並行搜索在線目錄
   ↓
4. AI 對候選進行排名
   ↓
5. 返回推薦列表
```

首先，系統分析項目的技術棧、編程語言、域名結構等特徵，構建一個「項目畫像」。這個畫像，就像一個人的簡歷一樣，記錄著所有的特徵。

然後，使用 AI Grain 生成針對性的搜索查詢。這裡的設計其實挺有意思——不是直接問 AI「推薦什麼技能」，而是讓它先思考「什麼樣的搜索詞能找到相關技能」。畢竟有時候，問問題的方法比答案本身更重要：

```csharp
var queryGeneration = await aiGrain.GenerateSkillRecommendationQueriesAsync(
    projectContext,      // 項目上下文
    locale,              // 用戶語言偏好
    maxQueries,           // 最大查詢數量
    effectiveSearchHero); // AI 模型選擇
```

接著，並行執行這些搜索查詢，獲取候選技能列表。並行處理，說到底也是為了節省時間罷了。

最後，使用另一個 AI Grain 對候選技能進行排名。這一步會綜合考慮技能與項目的相關性、授信狀態、用戶歷史偏好等因素：

```csharp
var ranking = await aiGrain.RankSkillRecommendationsAsync(
    projectContext,
    candidates,
    installedSkillNames,
    locale,
    maxRecommendations,
    effectiveRankingHero);

response.Items = MergeRecommendations(projectContext, candidates, ranking, maxRecommendations);
```

### 回退機制

AI 模型可能出現響應慢或暫時不可用的情況。畢竟再好的系統，也有掉鏈子的時候。為此，系統設計了確定性回退機制：當 AI 服務不可用時，使用基於規則啟發式算法生成推薦，比如根據 package.json 中的依賴推斷可能需要的技能。

這個回退機制，說穿了，也就是給系統留了一條後路罷了。

## 核心功能四：授信提供者管理

授信提供者管理允許用戶控制哪些技能源是可信的。畢竟信任這東西，還是要自己把握的。

### 匹配規則

授信提供者支持兩種匹配規則：**精確匹配**（exact）和**前綴匹配**（prefix）。

```csharp
public static TrustedSkillProviderResolutionSnapshot Resolve(
    TrustedSkillProviderSnapshot snapshot,
    string source)
{
    var normalizedSource = Normalize(source);

    foreach (var entry in snapshot.Entries.OrderBy(e => e.SortOrder))
    {
        if (!entry.IsEnabled) continue;

        foreach (var rule in entry.MatchRules)
        {
            bool isMatch = rule.MatchType switch
            {
                TrustedSkillProviderMatchRuleType.Exact
                    => string.Equals(normalizedSource, Normalize(rule.Value),
                        StringComparison.OrdinalIgnoreCase),
                TrustedSkillProviderMatchRuleType.Prefix
                    => normalizedSource.StartsWith(Normalize(rule.Value) + "/",
                        StringComparison.OrdinalIgnoreCase),
                _ => false
            };

            if (isMatch)
                return new TrustedSkillProviderResolutionSnapshot
                {
                    IsTrustedSource = true,
                    ProviderId = entry.ProviderId,
                    DisplayName = entry.DisplayName
                };
        }
    }

    return new TrustedSkillProviderResolutionSnapshot { IsTrustedSource = false };
}
```

預置的授信提供者包括 Vercel、Azure、anthropics、Microsoft、browser-use 等知名組織和項目。自定義提供者可以通過配置文件添加，指定提供者 ID、顯示名稱、徽章標籤、匹配規則等。畢竟世界那麼大，不可能只有幾家是可信的。

### 持久化實現

授信配置使用 Orleans Grain 持久化存儲：

```csharp
public class TrustedSkillProviderGrain : Grain<TrustedSkillProviderState>,
    ITrustedSkillProviderGrain
{
    public async Task UpdateConfigurationAsync(TrustedSkillProviderSnapshot snapshot)
    {
        State.Snapshot = snapshot;
        await WriteStateAsync();
    }

    public Task<TrustedSkillProviderSnapshot> GetConfigurationAsync()
    {
        return Task.FromResult(State.Snapshot);
    }
}
```

這種方式的好處是配置變更會自動同步到所有節點，無需手動刷新緩存。畢竟自動化，說到底也是為了讓人少操心罷了。

## 關鍵技術設計

### 命令執行適配器模式

Skill 系統需要執行各種 npx 命令，如果把這些邏輯散落在各處，代碼會變得難以維護。因此我們設計了適配器接口。設計模式這東西，說到底，也還是為了讓代碼更好維護而已：

```csharp
public interface ISkillInstallCommandRunner
{
    Task<SkillInstallCommandExecutionResult> ExecuteAsync(
        SkillInstallCommandExecutionRequest request,
        CancellationToken cancellationToken = default);
}
```

不同的命令有不同的執行器實現，全部實現同一個接口，便於測試和替換。

### SSE 流式輸出

安裝進度通過 Server-Sent Events 實時推送到前端：

```csharp
public async Task InstallWithProgressAsync(
    SkillInstallRequestDto request,
    IServerStreamWriter<SkillInstallProgressEventDto> stream,
    CancellationToken cancellationToken)
{
    var process = new Process
    {
        StartInfo = new ProcessStartInfo
        {
            FileName = "npx",
            Arguments = $"skills add {request.FullReference} -g -y",
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false
        }
    };

    process.OutputDataReceived += async (sender, e) =>
    {
        await stream.WriteAsync(new SkillInstallProgressEventDto
        {
            EventType = "output",
            Data = e.Data ?? string.Empty
        });
    };

    process.Start();
    process.BeginOutputReadLine();
    await process.WaitForExitAsync(cancellationToken);
}
```

用戶在前端可以看到類似終端的實時輸出，體驗非常直觀。畢竟實時反饋，讓人安心。

## 實踐指南

### 安裝社區技能

以安裝 `pua` 技能為例（這是一個流行的社區技能）：

1. 打開 Skills 抽屜，切換到「Skill Gallery」標籤
2. 輸入「pua」進行搜索
3. 點擊搜索結果查看技能詳情
4. 點擊「Install」按鈕安裝
5. 切換到「Local Skills」標籤確認安裝成功

安裝命令是 `npx skills add tanweai/pua -g -y`，系統會自動處理所有細節。其實也沒那麼多步驟，一步步來就是了。

### 添加自定義授信來源

如果你的團隊有自己的技能倉庫，可以添加為授信來源：

```yaml
providerId: "my-team"
displayName: "My Team Skills"
badgeLabel: "MyTeam"
isEnabled: true
sortOrder: 100
matchRules:
  - matchType: "prefix"
    value: "my-team/"
  - matchType: "exact"
    value: "my-team/special-skill"
```

這樣來自你團隊的所有技能都會顯示授信徽章，用戶可以更放心地安裝。畢竟有標記的東西，總是讓人安心一些。

### 技能開發基礎

創建自定義技能需要遵循以下結構：

```
my-skill/
├── SKILL.md          # 技能元數據（YAML front matter）
├── index.ts          # 技能入口
├── agents/           # 支持的代理配置
└── references/       # 參考資源
```

`SKILL.md` 的格式示例：

```yaml
---
name: my-skill
description: A brief description of what this skill does
---

# My Skill

Detailed documentation...
```

## 注意事項

1. **網絡要求**：技能搜索和安裝需要能訪問 `api.hagicode.com` 和 npm registry
2. **Node.js 版本**：建議使用 Node.js 18 或更高版本
3. **權限要求**：需要全局 npm 安裝權限
4. **並發控制**：同一技能同時只能有一個安裝或卸載操作在執行
5. **超時設置**：安裝操作默認超時時間為 4 分鐘，複雜場景可能需要調整

這些注意事項，說到底，也還是為了讓事情順利進行罷了。

## 總結

本文介紹了 HagiCode 項目中 Skill 管理系統的完整實現。這個系統通過前後端分離的架構、適配器模式、Orleans 分佈式狀態管理等技術手段，實現了：

- **本地全局管理**：通過封裝 npx skills 命令，提供統一的技能管理接口
