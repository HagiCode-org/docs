## HagiCode Skill 系統技術剖析：如何打造可擴充的 AI 技能管理平台

> 本文深入剖析 HagiCode 專案中 Skill（技能）管理系統的架構設計與實作方案，涵蓋本地全局管理、市場搜索、智能推薦、授信提供者管理四大核心功能的技術實作細節。

## 背景

在 AI 程式碼助手這個領域，如何擴充 AI 的能力邊界，其實一直是個核心課題。Claude Code 本身的程式碼輔助能力是挺強的，只是不同開發團隊、不同技術棧，往往需要針對特定場景的專業能力——比如處理 Docker 部署、資料庫優化、前端元件產生之類的。這時候，Skill（技能）系統就顯得尤爲重要了。

HagiCode 專案在開發程序中也遇到了類似的挑戰：怎麼讓 Claude Code 能夠像人一樣「學會」新的專業技能，同時保持良好的使用者體驗和工程可維護性？畢竟這個問題，說難也難，說簡單也簡單。圍繞這個問題，我們設計並實作了一套完整的 Skill 管理系統。

本文將詳細剖析這個系統的技術架構和核心實作，適合對 AI 擴充性、命令列工具集成感興趣的開發人員閱讀。或許對你有用，也或許沒用，但總歸是寫出來了。

## 關於 HagiCode

本文分享的方案來自我們在 [HagiCode](https://hagicode.com) 專案中的實踐經驗。HagiCode 是一個開放原始碼的 AI 程式碼助手專案，旨在說明開發團隊提升研發效率。專案的技術棧涵蓋 ASP.NET Core、Orleans 分佈式框架、TanStack Start + React 前端，以及本文要介紹的 Skill 管理子系統。

專案的 GitHub 地址是 [HagiCode-org/site](https://github.com/HagiCode-org/site)，如果你覺得本文介紹的技術方案有價值，歡迎給個 Star。畢竟Star多了，心情也會好一些。

## 系統架構概覽

Skill 系統採用前後端分離的架構設計，說起來也沒什麼特別的。

**前端部分** 使用 TanStack Start + React 建置使用者界面，透過 Redux Toolkit 管理狀態，四個主要功能分別對應四個 Tab 元件：本地技能、市場畫廊、智能推薦、授信提供者。這樣設計，其實也是爲了使用者體驗罷了。

**後端部分** 基於 ASP.NET Core + ABP Framework，使用 Orleans Grain 實作分佈式狀態管理。在線 API 用戶端封裝了 `IOnlineApiClient` 介面，用於與遠程技能目錄服務通信。

整體架構的設計原則是「命令執行與業務邏輯分離」，透過配接器模式將 npm/npx 命令執行的細節屏蔽在獨立模組中。畢竟誰願意看到一堆命令列散落在程式碼各處呢？

## 核心功能一：本地全局管理

本地全局管理是最基礎的功能模組，負責列出已安裝的技能並支援卸載操作。也沒什麼複雜的，就是把事情做好而已。

### 技術方案

實作位置在 `LocalSkillsTab.tsx` 和 `LocalSkillCommandAdapter.cs`。核心思路是封裝 `npx skills` 命令，剖析其 JSON 輸出，轉換爲內部資料結構。說起來簡單，做起來其實也簡單。

```csharp
public async Task<IReadOnlyList<LocalSkillInventoryResponseDto>> GetLocalSkillsAsync(
    CancellationToken cancellationToken = default)
{
    var result = await _commandAdapter.ListGlobalSkillsAsync(cancellationToken);
    return result.Skills.Select(skill => new LocalSkillInventoryResponseDto
    {
        Name = skill.Name,
        Version = skill.Version,
        Source = skill.Source,
        InstalledPath = skill.InstalledPath,
        Description = skill.Description
    }).ToList();
}
```

資料流非常清晰：前端發起請求 → SkillGalleryAppService 接收 → LocalSkillCommandAdapter 執行 npx 命令 → 剖析 JSON 結果 → 傳回 DTO 物件。一環扣一環，也沒什麼好說的。

卸載技能使用 `npx skills remove -g <skillName> -y` 命令，系統會自動處理相依關係和清理工作。安裝元資料存儲在技能目錄的 `managed-install.json` 中，記錄了安裝時間、來源版本等資訊，便於後續更新和審計。畢竟有些東西，記下來總是好的。

### 安裝流程詳解

技能安裝涉及多個步驟的協調，怎麼說呢，其實也不算太複雜：

```csharp
public async Task<SkillInstallResultDto> InstallAsync(
    SkillInstallRequestDto request,
    CancellationToken cancellationToken = default)
{
    // 1. 规范化安装引用
    var normalized = _referenceNormalizer.Normalize(
        request.SkillId,
        request.Source,
        request.SkillSlug,
        request.Version);

    // 2. 检查先决条件
    await _prerequisiteChecker.CheckAsync(cancellationToken);

    // 3. 获取安装锁
    using var installLock = await _lockProvider.AcquireAsync(normalized.SkillId);

    // 4. 执行安装命令
    var result = await _installCommandRunner.ExecuteAsync(
        new SkillInstallCommandExecutionRequest
        {
            Command = $"npx skills add {normalized.FullReference} -g -y",
            Timeout = TimeSpan.FromMinutes(4)
        },
        cancellationToken);

    // 5. 持久化安装元数据
    await _metadataStore.WriteAsync(normalized.SkillPath, request);

    return new SkillInstallResultDto { Success = result.Success };
}
```

這裏用到了幾個關鍵的設計模式：**參考規範化器** 負責將各種輸入格式（如 `tanweai/pua`、`@opencode/docker-skill`）轉換爲統一的內部表示；**安裝鎖定機制** 確保同一技能同時只有一個安裝操作在進行；**流式輸出** 透過 Server-Sent Events 向前端實時推送安裝進度，使用者可以看到類似終端機的實時日誌。

這些設計模式，說到底，也還是爲了讓事情變得簡單罷了。

## 核心功能二：市場搜索

市場搜索讓使用者能夠發現和安裝來自社區的技能。畢竟一個人的能力是有限的，衆人的智慧纔是無窮的。

### 技術方案

搜索功能相依在線 API `https://api.hagicode.com/v1/skills/search`。爲了提升回應速度，系統實作了快取機制。快取這東西，就像記憶一樣，有些東西記住了，下次就不用再費勁去想。

```csharp
private async Task<IReadOnlyList<SkillGallerySkillDto>> SearchCatalogAsync(
    string query,
    CancellationToken cancellationToken,
    IReadOnlySet<string>? allowedSources = null)
{
    var cacheKey = $"skill_search:{query}:{string.Join(",", allowedSources ?? Array.Empty<string>())}";

    if (_memoryCache.TryGetValue(cacheKey, out var cached))
        return (IReadOnlyList<SkillGallerySkillDto>)cached!;

    var response = await _onlineApiClient.SearchAsync(
        new SearchSkillsRequest
        {
            Query = query,
            Limit = _options.LimitPerQuery,
        },
        cancellationToken);

    var results = response.Skills
        .Where(skill => allowedSources is null || allowedSources.Contains(skill.Source))
        .Select(skill => new SkillGallerySkillDto { ... })
        .ToList();

    _memoryCache.Set(cacheKey, results, TimeSpan.FromMinutes(10));
    return results;
}
```

搜索結果支援按授信來源過濾，只顯示使用者信任的技能源。預置的種子查詢用於初始化目錄，比如「popular」、「recent」等，讓使用者在首次開啟時就能看到推薦的熱門技能。畢竟第一印象還是重要的。

## 核心功能三：智能推薦

智能推薦是系統中最複雜的功能，它能根據使用者當前專案的情況，自動推薦最適合的技能。複雜歸複雜，但做出來還是值得的。

### 推薦流程

整個推薦流程分爲五個階段：

```
1. 构建项目上下文
   ↓
2. AI 生成搜索查询
   ↓
3. 并行搜索在线目录
   ↓
4. AI 对候选进行排名
   ↓
5. 返回推荐列表
```

首先，系統分析專案的技術棧、程式設計語言、域名結構等特徵，建置一個「專案畫像」。這個畫像，就像一個人的簡歷一樣，記錄着所有的特徵。

然後，使用 AI Grain 產生針對性的搜索查詢。這裏的設計其實挺有意思——不是直接問 AI「推薦什麼技能」，而是讓它先思考「什麼樣的搜索詞能找到相關技能」。畢竟有時候，問問題的方法比答案本身更重要：

```csharp
var queryGeneration = await aiGrain.GenerateSkillRecommendationQueriesAsync(
    projectContext,      // 项目上下文
    locale,              // 用户语言偏好
    maxQueries,           // 最大查询数量
    effectiveSearchHero); // AI 模型选择
```

接着，平行執行這些搜索查詢，取得候選技能清單。平行處理，說到底也是爲了節省時間罷了。

最後，使用另一個 AI Grain 對候選技能進行排名。這一步會綜合考慮技能與專案的相關性、授信狀態、使用者歷史偏好設定等因素：

```csharp
var ranking = await aiGrain.RankSkillRecommendationsAsync(
    projectContext,
    candidates,
    installedSkillNames,
    locale,
    maxRecommendations,
    effectiveRankingHero);

response.Items = MergeRecommendations(projectContext, candidates, ranking, maxRecommendations);
```

### 回退機制

AI 模型可能出現回應慢或暫時不可用的情況。畢竟再好的系統，也有掉鏈子的時候。爲此，系統設計了確定性回退機制：當 AI 服務不可用時，使用基於規則啓發式演算法產生推薦，比如根據 package.json 中的相依推斷可能需要的技能。

這個回退機制，說穿了，也就是給系統留了一條後路罷了。

## 核心功能四：授信提供者管理

授信提供者管理允許使用者控制哪些技能源是可信的。畢竟信任這東西，還是要自己把握的。

### 匹配規則

授信提供者支援兩種匹配規則：**精確匹配**（exact）和**前綴匹配**（prefix）。

```csharp
public static TrustedSkillProviderResolutionSnapshot Resolve(
    TrustedSkillProviderSnapshot snapshot,
    string source)
{
    var normalizedSource = Normalize(source);

    foreach (var entry in snapshot.Entries.OrderBy(e => e.SortOrder))
    {
        if (!entry.IsEnabled) continue;

        foreach (var rule in entry.MatchRules)
        {
            bool isMatch = rule.MatchType switch
            {
                TrustedSkillProviderMatchRuleType.Exact
                    => string.Equals(normalizedSource, Normalize(rule.Value),
                        StringComparison.OrdinalIgnoreCase),
                TrustedSkillProviderMatchRuleType.Prefix
                    => normalizedSource.StartsWith(Normalize(rule.Value) + "/",
                        StringComparison.OrdinalIgnoreCase),
                _ => false
            };

            if (isMatch)
                return new TrustedSkillProviderResolutionSnapshot
                {
                    IsTrustedSource = true,
                    ProviderId = entry.ProviderId,
                    DisplayName = entry.DisplayName
                };
        }
    }

    return new TrustedSkillProviderResolutionSnapshot { IsTrustedSource = false };
}
```

預置的授信提供者包括 Vercel、Azure、anthropics、Microsoft、browser-use 等知名組織和專案。自定義提供者可以透過設定檔案新增，指定提供者 ID、顯示名稱、徽章標籤、匹配規則等。畢竟世界那麼大，不可能只有幾家是可信的。

### 永續性實作

授信設定使用 Orleans Grain 永續性存儲：

```csharp
public class TrustedSkillProviderGrain : Grain<TrustedSkillProviderState>,
    ITrustedSkillProviderGrain
{
    public async Task UpdateConfigurationAsync(TrustedSkillProviderSnapshot snapshot)
    {
        State.Snapshot = snapshot;
        await WriteStateAsync();
    }

    public Task<TrustedSkillProviderSnapshot> GetConfigurationAsync()
    {
        return Task.FromResult(State.Snapshot);
    }
}
```

這種方式的好處是設定變更會自動同步到所有節點，無需手動重新整理快取。畢竟自動化，說到底也是爲了讓人少操心罷了。

## 關鍵技術設計

### 命令執行配接器模式

Skill 系統需要執行各種 npx 命令，如果把這些邏輯散落在各處，程式碼會變得難以維護。因此我們設計了配接器介面。設計模式這東西，說到底，也還是爲了讓程式碼更好維護而已：

```csharp
public interface ISkillInstallCommandRunner
{
    Task<SkillInstallCommandExecutionResult> ExecuteAsync(
        SkillInstallCommandExecutionRequest request,
        CancellationToken cancellationToken = default);
}
```

不同的命令有不同的執行器實作，全部實作同一個介面，便於測試和取代。

### SSE 流式輸出

安裝進度透過 Server-Sent Events 實時推送到前端：

```csharp
public async Task InstallWithProgressAsync(
    SkillInstallRequestDto request,
    IServerStreamWriter<SkillInstallProgressEventDto> stream,
    CancellationToken cancellationToken)
{
    var process = new Process
    {
        StartInfo = new ProcessStartInfo
        {
            FileName = "npx",
            Arguments = $"skills add {request.FullReference} -g -y",
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false
        }
    };

    process.OutputDataReceived += async (sender, e) =>
    {
        await stream.WriteAsync(new SkillInstallProgressEventDto
        {
            EventType = "output",
            Data = e.Data ?? string.Empty
        });
    };

    process.Start();
    process.BeginOutputReadLine();
    await process.WaitForExitAsync(cancellationToken);
}
```

使用者在前端可以看到類似終端機的實時輸出，體驗非常直觀。畢竟實時反饋，讓人安心。

## 實踐指南

### 安裝社區技能

以安裝 `pua` 技能爲例（這是一個流行的社區技能）：

1. 開啟 Skills 抽屜，切換到「Skill Gallery」標籤
2. 輸入「pua」進行搜索
3. 點擊搜索結果查看技能詳情
4. 點擊「Install」按鈕安裝
5. 切換到「Local Skills」標籤確認安裝成功

安裝命令是 `npx skills add tanweai/pua -g -y`，系統會自動處理所有細節。其實也沒那麼多步驟，一步步來就是了。

### 新增自定義授信來源

如果你的團隊有自己的技能存放庫，可以新增爲授信來源：

```yaml
providerId: "my-team"
displayName: "My Team Skills"
badgeLabel: "MyTeam"
isEnabled: true
sortOrder: 100
matchRules:
  - matchType: "prefix"
    value: "my-team/"
  - matchType: "exact"
    value: "my-team/special-skill"
```

這樣來自你團隊的所有技能都會顯示授信徽章，使用者可以更放心地安裝。畢竟有標記的東西，總是讓人安心一些。

### 技能開發基礎

建立自定義技能需要遵循以下結構：

```
my-skill/
├── SKILL.md          # 技能元数据（YAML front matter）
├── index.ts          # 技能入口
├── agents/           # 支持的代理配置
└── references/       # 参考资源
```

`SKILL.md` 的格式示例：

```yaml
---
name: my-skill
description: A brief description of what this skill does
---

# My Skill

Detailed documentation...
```

## 注意事項

1. **網路要求**：技能搜索和安裝需要能存取 `api.hagicode.com` 和 npm registry
2. **Node.js 版本**：建議使用 Node.js 18 或更高版本
3. **權限要求**：需要全局 npm 安裝權限
4. **並行控制**：同一技能同時只能有一個安裝或卸載操作在執行
5. **超時設定**：安裝操作預設超時時間爲 4 分鐘，複雜場景可能需要調整

這些注意事項，說到底，也還是爲了讓事情順利進行罷了。
