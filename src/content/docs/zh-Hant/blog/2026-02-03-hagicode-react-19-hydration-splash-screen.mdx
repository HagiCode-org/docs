## 爲 HagiCode 設計 12 種極致的啓動體驗：從極簡到賽博朋克

> 在 React 19 應用下載和 Hydration 的短暫間隙，是留給使用者感知品牌個性的黃金視窗。本文分享了我們在 HagiCode 專案中，基於 HTML/CSS/JS 建置的一套完整的啓動風格系統。


## 背景

HagiCode 作爲一個基於 ASP.NET Core 10 和 React 19 (Vite) 的現代化應用，採用了前後端分離部署的架構。前端產物被打包放置於後端的 `wwwroot/` 目錄下由 ASP.NET Core 代管。

然而，這種架構帶來了一個經典的使用者體驗痛點：當使用者存取網頁時，瀏覽器需要先載入 HTML，再下載巨大的 JS Bundle，最後由 React 執行 Hydration（注水）。在這幾百毫秒到數秒的"真空期"裏，使用者面對的是一片空白，或者是一個毫無生氣的靜態頁面。

爲了填補這段間隙，並注入 HagiCode 的品牌個性，我們需要設計一套完全基於 `index.html` 內聯程式碼的啓動風格系統。

## 關於 HagiCode

本文分享的啓動頁設計方案來自我們在 [HagiCode](https://github.com/HagiCode-org/site) 專案中的實踐經驗。作爲一個 AI 程式碼助手，HagiCode 不僅關注程式碼產生的效率，也同樣重視開發人員的視覺體驗。這套啓動系統正是我們在追求極致前端效能程序中的產物。

## 核心挑戰與架構設計

在動手設計之前，我們必須先明確技術約束。既然要在 `index.html` 中內聯實作，意味着我們不能載入任何外部 CSS 或 JS 檔案（除了 React 本身的 Bundle）。

### 技術約束分析

1.  **零相依原則**：所有樣式必須寫在 `<style>` 標籤內，邏輯寫在 `<script>` 標籤內。
2.  **防禦式 CSS**：爲了防止 React 應用掛載後，全局樣式污染啓動頁，我們決定使用高優先級的 ID 前綴（如 `#boot-screen`）包裹所有啓動樣式。
3.  **效能優先**：動畫儘量使用 CSS `transform` 和 `opacity`，避免觸發重排，確保不封鎖主執行緒。
4.  **視覺一致性**：顏色、字體必須與 HagiCode 的 Tailwind 設定保持一致。

### 架構模式：Shell & Injector

我們採用了一種**變體模式**。核心邏輯封裝在一個立即執行函式（IIFE）中，具體的渲染邏輯作爲設定項注入。這樣我們就可以透過簡單的設定切換不同的風格，而不需要重複編寫 DOM 操作邏輯。

以下是核心的架構程式碼：

```html
<!-- 内联于 index.html -->
<div id="boot-root"></div>

<script>
(function() {
  const BootSequence = {
    config: {
      theme: 'terminal', // 可配置为 'minimal', 'skeleton', 'code-rain' 等
      color: '#3b82f6'   // 品牌色
    },
    
    // 核心生命周期
    init() {
      this.render();
      this.listenForMount();
    },

    // 渲染当前选定的风格
    render() {
      const root = document.getElementById('boot-root');
      if (this.variants[this.config.theme]) {
        root.innerHTML = this.variants[this.config.theme].render();
      }
    },

    // 监听 React 挂载成功，优雅退出
    listenForMount() {
      window.addEventListener('hagicode:ready', () => {
        const screen = document.getElementById('boot-root');
        // 先淡出，再移除 DOM，避免闪烁
        screen.style.opacity = '0';
        screen.style.transition = 'opacity 0.3s ease';
        setTimeout(() => screen.remove(), 300);
      });
    },

    // 12种风格的实现逻辑集中在这里
    variants: {
      // ...具体实现见下文
    }
  };

  BootSequence.init();
})();
</script>
```

## 12 種啓動風格設計清單

我們將這 12 種風格分爲了六大類，以滿足不同場景和審美需求。

### A. 極簡主義

> "少即是多"。對於追求極致載入速度的場景，我們提供了最輕量的方案。

#### 1. Minimalist Dot (極簡呼吸)
螢幕中心只有一個簡單的圓點，配合呼吸動畫。
*   **實作**：CSS `@keyframes` 控制scale和opacity。
*   **適用**：任何需要保持頁面絕對乾淨的場合。

#### 2. Brand Reveal (品牌揭示)
透過 SVG `stroke-dasharray` 動畫，模擬手繪般繪製出 HagiCode 的 Logo 線條，隨後淡入文字。
*   **技巧**：使用 SVG 路徑動畫，極具質感。

### B. 骨架屏擬態

> "欺騙眼睛的藝術"。透過模擬真實 UI 佈局，讓使用者感覺頁面已經載入了一半。

#### 3. Sidebar Chat Skeleton (側邊欄骨架屏)
這可能是最實用的一種。我們手動用 HTML 建置了與 React 元件 `Sidebar` 和 `ChatInput` 一模一樣的佈局，並覆寫灰色條紋動畫。
*   **價值**：當 React hydrate 完成時，骨架屏瞬間變成真實元件，使用者幾乎感覺不到切換。

#### 4. Card Stack Skeleton (卡片堆疊)
模擬提案卡片載入時的堆疊動效，使用 3D 變換讓卡片微微浮動。

### C. 抽象與藝術

> 展示 HagiCode 的極客基因。

#### 5. Geometric Morph (幾何變形)
在螢幕中心渲染一個幾何體（正方形），它會隨着時間平滑地變換爲圓形、三角形，最後變成 Logo。
*   **技術**：CSS `border-radius` 的平滑過渡。

#### 6. Code Rain (程式碼雨)
向《黑客帝國》致敬。使用 JetBrains Mono 字體，在背景中落下淡淡的字符流。
*   **注意**：爲了效能，字符流必須限制在較小的區域或降低重新整理頻率。

#### 7. Neon Pulse (霓虹脈衝)
賽博朋克風格的發光圓環，利用 `box-shadow` 的多重疊加產生強烈的發光感。

### D. 品牌與主題

> 讓系統"活"起來。

#### 8. Seasonal Theme (節日主題)
這是一個動態載入器。根據當前日期判斷節日（如春節、聖誕節），載入對應的 SVG 動畫。
*   **例子**：春節時，螢幕下方會有紅燈籠輕輕擺動。

#### 9. Gradient Flow (漸變流)
背景使用 HagiCode 品牌色的流體漸變，配合 `background-size` 和 `background-position` 的動畫，營造出極光般的流動感。

### E. 技術感

> 向開發人員致敬。

#### 10. Terminal Boot (終端機啓動)
模擬主控台輸出。一行行程式碼快速捲動：
```text
> Initializing HagiCode Core...
> Loading models...
> Connecting to neural network...
```
這會讓每一個開發人員都感到親切。

#### 11. Progress Bar (極簡進度條)
螢幕頂部一條細細的進度條，右側顯示百分比。雖然我們無法取得真實的下載進度，但可以用一個計時器模擬出一個"可信"的載入程序（前 80% 快速，後 20% 減速）。

### F. 創意

#### 12. Pixel Assembly (像素組裝)
這是一個很有趣的創意。螢幕上散落着一些方塊，它們匯聚到中心，逐漸拼湊出 HagiCode 的 Logo 圖標。象徵着程式碼的建置程序。

## 最佳實踐與踩坑總結

在 HagiCode 的實際開發中，我們總結了一些至關重要的實踐細節。

### 1. 防禦式 CSS 是必須的
千萬別偷懶不寫前綴。曾經有一次，我們沒有給啓動頁樣式加 ID 限制，導致 React 掛載後的全局 `div` 樣式意外影響了啓動頁，導致佈局崩壞。
**經驗**：所有 CSS 選擇器都掛在 `#boot-screen` 下，且使用 `!important` 提升優先級（僅在啓動頁 CSS 中）。

### 2. 優雅的過渡
React mount 成功後，不要直接 `remove()` 啓動頁 DOM。
**正確做法**：
1.  React 觸發 `window.dispatchEvent(new Event('hagicode:ready'))`。
2.  啓動頁監聽到事件，先設定 `opacity: 0`。
3.  等待 300ms (CSS transition 時間)，確保使用者看不見了，再執行 `.remove()`。

### 3. 主題變數同步
啓動頁的顏色程式碼是寫死在 `index.html` 裏的。如果我們修改了 Tailwind 的主色，必須同步修改這裏。
**優化方案**：在 Vite 建置腳本中，編寫一個簡單的外掛，讀取 `tailwind.config.js` 並將顏色變數注入到 `index.html` 的範本變數中，實作單一資料源。

### 4. 字體預載入
啓動頁通常需要使用品牌字體，但如果字體載入慢，會出現 FOUT (Flash of Unstyled Text)。
**解決方案**：在 `<head>` 中加入 `<link rel="preload" href="/fonts/JetBrainsMono.woff2" as="font" type="font/woff2" crossorigin>`。這是提升體驗的低成本高回報手段。

### 5. 效能監控
我們在 `index.html` 底部注入了 `performance.mark('boot-start')`，並在 React 掛載成功時標記 `boot-end`。
**意義**：透過 Application Insights 收集這些資料，我們可以真實看到啓動頁對使用者感知等待時間（Perceived Loading Time）的縮短程度。資料表明，優秀的骨架屏能讓使用者對"慢速網路"的容忍度提升 50% 以上。
