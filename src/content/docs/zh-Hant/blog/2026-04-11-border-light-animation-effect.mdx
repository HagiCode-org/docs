## 邊框燈光環繞動畫特效實現指南

> 那個讓用戶一眼就注意到的重要元素，到底是怎麼用純 CSS 做出來的？其實也不難，就是繞了個彎子罷了。這篇文章帶你從零開始實現邊框燈光環繞動畫，也順帶聊聊我們在 HagiCode 專案裡踩過的那些坑。

## 背景

做前端的同學應該都有過這樣的經歷：產品經理跑過來，臉上掛著那種"這需求很簡單"的表情——"這個正在運行的任務，能不能加個特效讓用戶一眼就能看到？"

你說行啊，加個邊框變色唄。結果對方搖搖頭，眼神裡透著一種"你不懂"的意味："不夠明顯，要那種燈光繞著邊框跑的效果，跟科幻電影裡一樣。"

這時候你可能就會犯嘀咕：這玩意兒怎麼實現？用 Canvas？用 SVG？還是說 CSS 能搞？畢竟誰也不想承認自己不會嘛。

其實啊，邊框燈光環繞動畫在現代 Web 應用中特別常見，主要用在這麼幾個場景：

- **狀態指示**：標記正在進行的任務或活躍的項目
- **視覺焦點**：突出顯示重要的內容區域
- **品牌增強**：營造科技感和現代感的視覺體驗
- **節日主題**：配合特殊節日創建慶祝氛圍

我們做 [HagiCode](https://hagicode.com) 的時候就遇到過這個需求——用戶需要一眼看出哪些會話正在運行、哪些提案正在處理中。試了好幾種方案，有些路好走一點，有些路稍微曲折一點罷了，最後沉澱出了一套還算成熟的實現思路。

## 關於 HagiCode

本文分享的方案來自我們在 [HagiCode](https://hagicode.com) 專案中的實踐經驗。HagiCode 是一個 AI 驅動的代碼助手專案，在界面中大量使用邊框燈光動畫來指示各種運行狀態。比如會話列表的運行狀態、提案流程圖的狀態過渡、吞吐量指示器的強度顯示等等。

其實這些效果說起來也不複雜，就是做的時候踩了不少坑。如果你想看看實際效果，可以訪問我們的 [GitHub 倉庫](https://github.com/HagiCode-org/site) 或者直接去 [官網](https://hagicode.com) 了解一下，畢竟能用的才是最好的嘛。

## 核心實現思路

經過對 HagiCode 代碼的分析，我們總結出了下面幾種核心的實現模式，每種都有它適用的場景，或者說，每種都有它存在的意義罷了。

### 1. Conic Gradient 旋轉光暈（最常用）

這是最經典的邊框燈光環繞實現方式，核心思路是用 CSS 的 `conic-gradient` 創建一個圓錐漸變，然後讓它轉起來。就像夜晚的路燈，一直在那裡轉啊轉的。

**關鍵要素：**
- 用 `::before` 偽元素創建光暈層
- 用 `conic-gradient` 定義漸變色分佈
- 用 `::after` 偽元素遮罩中心區域（可選）
- 用 `@keyframes` 實現旋轉動畫

### 2. 側邊發光線條

這個適用於列表項的狀態指示，在元素的一側創建發光的細線條就行，不用整個邊框都動。畢竟有時候，一點光就夠了，不需要照亮整個世界。

**關鍵要素：**
- 絕對定位的細線元素
- 用 `box-shadow` 創建發光效果
- 用 `scale` 和 `opacity` 實現呼吸動畫

### 3. Box-Shadow 發光背景

如果不需要環繞效果，只是想要個柔和的背景光暈，那用多層 `box-shadow` 叠加就夠了。有些東西，簡單點反而更好。

### 4. 無障礙訪問支持

這個容易被忽略，但特別重要。所有動畫都應該考慮 `prefers-reduced-motion` 媒體查詢，給不喜歡動畫的用戶提供一個靜態替代方案。畢竟不是所有人都喜歡動來動去的，尊重每個人的選擇才是對的。

## 實現方案

### 方案一：Conic Gradient 旋轉邊框（推薦）

這是最完整的環繞燈光效果實現，也是 HagiCode 裡用得最多的方案。畢竟，如果一樣東西好用，為什麼還要換呢？

```css
/* 父容器 */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* 旋轉的光暈層 */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* 遮罩層（可選，用於創建空心邊框效果） */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

這個方案的原理挺簡單的：創建一個比父容器大的偽元素，上面畫一個圓錐漸變，然後讓它不停旋轉。父容器設置 `overflow: hidden`，所以只能看到邊框那一部分的光在轉。就像我們在窗子裡看外面的路燈，只能看到它經過的那一小段罷了。

### 方案二：簡化版旋轉光邊框

如果你不需要那麼複雜的效果，HagiCode 裡有個更輕量的工具類實現。畢竟簡單點，有時候反而更好。

```css
/* 旋轉光邊框工具類 */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* 無障礙支持 */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

注意這裡的 `will-change: transform`，這是告訴瀏覽器"這個元素要一直變"，瀏覽器就會提前做些優化，動畫會更流暢。畢竟提前準備，總比臨時抱佛腳強嘛。

### 方案三：側邊發光線條

列表項的狀態指示用這個特別合適，HagiCode 的會話列表就是用的這個方案。一條細線，卻能在眾多項目中脫穎而出，這不也是一種生活哲學嗎？

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

這裡用了 `isolation: isolate` 創建一個新的層疊上下文，然後用 `z-index` 控制各層的顯示順序。`pointer-events: none` 也很關鍵，不然那個偽元素會擋住用戶的點擊操作。就像有些東西，好看是好看，但是不能礙事才行。

### 方案四：React 組件封裝

如果你專案裡用 React，可以封裝一個組件來處理這些邏輯，特別是無障礙訪問的部分。畢竟代碼寫一次，用很多次，這才是我們想要的嘛。

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

對應的 CSS 模組：

```css
/* GlowBorder.module.css */

/* 動畫版本 */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* 靜態版本（無障礙） */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

`framer-motion` 的 `useReducedMotion` hook 會自動檢測用戶的系統偏好，如果用戶開啟了"減弱動態效果"，就會返回 true，這時候就顯示靜態版本。畢竟，尊重用戶的選擇比強行展示更重要。

## 實踐經驗分享

下面這些是我們在做 HagiCode 時踩過坑、總結出來的經驗。其實也就是些碎碎念罷了，希望能幫到後來的你。

### 1. 主題變量系統

用 CSS 變量實現多主題支持特別方便。畢竟誰也不想每次切換主題都要改一堆代碼呢？

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* 使用 */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

這樣切換主題的時候只需要改一下 `html` 標籤的 class，所有動畫顏色都會自動更新。一套代碼，兩種風格，這不就是我們追求的嗎？

### 2. 性能優化

**使用 `will-change` 提示瀏覽器優化：**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

提前告訴瀏覽器，它就會幫你做些優化。就像生活中的很多事情，提前準備總是好的。

**避免在大面積元素上使用複雜的 box-shadow：**

```css
/* 不好 - 大面積元素上使用模糊陰影 */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* 更好 - 使用偽元素限制發光區域 */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

我們在 HagiCode 裡測試過，在大卡片上直接加模糊陰影會讓滾動幀率掉到 30fps 以下，改用偽元素後就穩穩 60fps 了。這種體驗上的差異，用戶是能感覺到的。

### 3. 無障礙訪問

這個真的不能省，有些用戶會覺得動畫很暈或者很吵，尊重他們的選擇是做產品的基本素養。畢竟美的事物不必強加於人嘛。

**CSS 媒體查詢：**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* 提供靜態替代方案 */
    opacity: 1;
  }
}
```

**React 中檢測用戶偏好：**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. 強度級別控制

HagiCode 裡的 Token 吞吐量指示器會根據實時吞吐量顯示不同顏色的燈光，這個是動態實現的。畢竟不同的狀態，應該有不一樣的表達方式。

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. 注意事項

有些細節還是要注意的，不然踩了坑才知道就晚了。

| 注意事項 | 說明 |
|---------|------|
| **z-index 管理** | 光暈層應設置合適的 z-index，避免影響內容交互 |
| **pointer-events** | 光暈偽元素應設置 `pointer-events: none` |
| **邊界溢出** | 父容器需要設置 `overflow: hidden` 或調整偽元素尺寸 |
| **性能影響** | 複雜動畫在移動設備上可能影響性能，需要測試 |
| **深色模式** | 確保發光顏色在深色背景下清晰可見 |
| **主題切換** | 使用 CSS 變量確保主題切換時動畫顏色正確更新 |

### 6. 調試技巧

偽元素在開發者工具裡有時候不太好找，可以臨時加個邊框來看看位置。

```css
/* 臨時顯示偽元素邊界用於調試 */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

調好位置之後記得把這行注釋掉或者刪掉，不然生產環境會很尷尬。有些東西，還是留在開發環境比較好。

## 總結

邊框燈光環繞動畫說難不難，說簡單也不簡單。核心就是 `conic-gradient` 加旋轉，但要做到性能好、可維護、無障礙友好，還是有不少細節要注意的。

HagiCode 在這個上面踩了不少坑，也總結出了一些最佳實踐。其實做專案就是這樣，一遍遍試錯，一遍遍改進。如果你在做類似的需求，希望這篇文章能幫你少走點彎路。

畢竟，有些東西，還是要親自實踐才知道深淺。
