## .NET 程式碼保護實戰：從混淆到虛擬機保護

> 本文將介紹如何在 .NET 專案中實施多層次程式碼保護策略，涵蓋從基礎混淆到專業虛擬機保護的全部方案。

## 背景

在 .NET 應用程式開發中，保護核心程式碼（如授權驗證、業務邏輯、敏感設定等）不被反編譯和逆向分析，怎麼說呢，這也是個繞不開的話題。隨著 .NET 生態系統的成熟，開發者有了多種程式碼保護手段，從內建的混淆屬性到專業的虛擬化保護工具，選擇倒挺多的。

作為一個複雜的多語言 monorepo 專案，HagiCode 包含了桌面應用程式、建構系統和授權管理功能。程式碼中不可避免地涉及授權驗證邏輯、敏感設定（如 API 金鑰、產品 ID）以及業務核心邏輯，這些東西還得好好保護一下，畢竟誰也不想自己的心血輕易被人看了去。

本文將分享我們在 HagiCode 專案中實際採用的程式碼保護方案，總結從踩坑到優化的完整過程，或許能給你一些啟發。

## 關於 HagiCode

[HagiCode](https://hagicode.com) 是一個開源的 AI 程式碼助手專案，致力於為開發者提供智慧化的程式設計輔助體驗。專案採用 monorepo 架構，同時維護著 VSCode 擴充功能、後端 AI 服務、跨平台桌面客戶端等多個元件。這種多語言、多平台的複雜度，使得程式碼保護成為必須面對的工程挑戰，也沒轍，誰讓專案這麼複雜呢。

本文分享的方案，正是我們在開發 HagiCode 過程中實際踩坑、實際優化出來的。如果你想了解我們是如何解決這些技術難題的，請繼續往下看，或許會有一些意外的收穫。

## 核心內容

### 1. 微軟內建混淆 Attribute

.NET Framework 提供了一個內建的 `[ObfuscationAttribute]`，這是最基礎也是最常用的程式碼混淆標記。該屬性位於 `System.Reflection` 命名空間下，可以在不引入第三方工具的情況下對程式碼進行基礎保護，倒也挺方便的。

**核心特性**：
- `Feature` 屬性：指定混淆特性，如 `"ultra"`（高度混淆）、`"all"`（全部混淆）
- `Exclude` 屬性：`true` 表示排除混淆，`false` 表示應用混淆
- 可套用於類別、方法、屬性等類型成員

在 HagiCode 專案中，可以看到實際使用範例：

```csharp
[Obfuscation(Feature = "ultra", Exclude = false)]
public async Task<LicenseValidationResult?> ValidateLicenseAsync(...)
```

這種方式的**優勢**還挺明顯的：
- 無需額外依賴，.NET Framework 內建自帶，省了不少事
- 可被第三方混淆工具識別和處理
- 不會顯著增加編譯後的組件大小

不過它也有局限性：僅是標記作用，實際混淆效果依賴工具實作，無法提供虛擬機保護級別的安全性，這也罷了，畢竟它本來就不是為此而生的。

### 2. VMP（Virtual Machine Protection）

VMP 是一個專業的程式碼保護工具，透過將程式碼編譯為虛擬機指令來提供高級別的保護。與簡單的名稱混淆不同，VMP 真正將程式碼邏輯轉換為無法被常規反編譯器還原的形式，這點倒是挺厲害的。

**保護級別分類**：

| 級別 | 虛擬化 | 變異 | 反除錯 | 字串加密 | 適用場景 |
|------|--------|------|--------|------------|----------|
| HIGH | full | high | 啟用 | 啟用 | 授權驗證、會話並發、敏感常量 |
| MEDIUM | partial | medium | 啟用 | 啟用 | 業務邏輯、領域模型 |
| LOW | none | low | 停用 | 停用 | 工具類、非關鍵程式碼 |

HagiCode 專案定義了一套宣告式屬性系統來標記需要保護的程式碼：

```csharp
// 高優先級保護
[VmProtect(VmProtectionPriority.High, Reason = "Contains license verification logic")]
public class KeygenClient { ... }

// 排除保護
[VmExclude(Reason = "Public API that must remain unchanged")]
public class PublicApi { ... }

// 繼承保護
[VmProtect(Priority.High, ProtectDerived = true)]
public class BaseLicenseValidator { ... }
```

### 3. 建構時保護策略

VMP 保護不僅在執行時生效，更需要在建構流程中自動化處理，畢竟手動來做也太麻煩了。HagiCode 的建構系統支援多種模式：

- **Windows 原生模式**：直接呼叫 VMProtect 工具
- **Linux Docker 容器模式**：在容器中執行 VMP（解決跨平台相容性問題）
- **Attribute 掃描**：自動發現程式碼中的保護標記
- **驗證機制**：確認保護已成功套用

這些功能組合起來，倒也挺省心的。

## 解決方案

### 1. 微軟內建混淆 Attribute 的使用

在程式碼中直接套用 `ObfuscationAttribute`：

```csharp
using System.Reflection;

[Obfuscation(Feature = "ultra", Exclude = false)]
public class LicenseService
{
    [Obfuscation(Feature = "ultra", Exclude = false)]
    public async Task<bool> ValidateLicenseAsync(string key)
    {
        // 授權驗證邏輯
    }

    [Obfuscation(Feature = "flow", Exclude = false)]
    private string DecryptToken(string encrypted)
    {
        // 解密邏輯
    }
}
```

有時需要讓測試組件存取內部成員，同時保持生產程式碼的安全性：

```csharp
// AssemblyInfo.cs
[assembly: InternalsVisibleTo("HagiCode.Application.Tests")]
[assembly: InternalsVisibleTo("DynamicProxyGenAssembly2")] // for Moq
```

這樣測試起來方便多了，畢竟程式碼還是要測試的。

### 2. VMP 保護的自訂 Attribute 定義

建立自訂保護屬性來控制 VMP 的行為：

```csharp
using System;

[AttributeUsage(AttributeTargets.Class | AttributeTargets.Method | AttributeTargets.Property)]
public class VmProtectAttribute : Attribute
{
    public VmProtectionPriority Priority { get; set; }
    public string? Reason { get; set; }
    public bool ProtectDerived { get; set; }
}

[AttributeUsage(AttributeTargets.Class | AttributeTargets.Method | AttributeTargets.Property)]
public class VmExcludeAttribute : Attribute
{
    public string? Reason { get; set; }
}

public enum VmProtectionPriority
{
    None = 0,
    Low = 1,
    Medium = 2,
    High = 3
}
```

自己定義的屬性用起來也更順手，畢竟了解自己的需求。

### 3. VMP 設定檔

```yaml
# vmp_config.yml
protection:
  priority_mode: "attribute"  # 基於 Attribute 的優先級
  default_level: "medium"

tools:
  - name: "vmprotect"
    path: "C:\\Program Files\\VMProtect Ultimate\\VMProtect.exe"

protection_levels:
  high:
    virtualization: "full"
    mutation: "high"
    anti_debug: true
    anti_dump: true
    encrypt_strings: true
    encrypt_resources: true

  medium:
    virtualization: "partial"
    mutation: "medium"
    anti_debug: true
    encrypt_strings: true

  low:
    virtualization: "none"
    mutation: "low"
    anti_debug: false
```

設定寫清楚一點，後面維護起來也方便。

## 實踐指南

### 1. 關鍵元件保護實踐

根據 HagiCode 的 code-protection 規範，以下元件必須使用 HIGH 優先級保護：

```csharp
// 生產環境常量 - 必須加密並受 VMP 保護
[VmProtect(VmProtectionPriority.High, Reason = "Production constants")]
public static class ProductionConstants
{
    // 加密字串存取器，由 VMP 保護
    [VmProtect(VmProtectionPriority.High)]
    public static string GetLicenseServerUrl(IOptions<LicenseOptions> options) => ...;
}

// 授權驗證邏輯
[VmProtect(VmProtectionPriority.High, Reason = "License verification logic")]
public class KeygenClient : IKeygenClient
{
    [Obfuscation(Feature = "ultra", Exclude = false)]
    public async Task<LicenseValidationResult?> ValidateLicenseAsync(...) { ... }
}

// 機器指紋服務
[VmProtect(VmProtectionPriority.High)]
public class MachineFingerprintService : IMachineFingerprintService { ... }
```

關鍵的程式碼還是要重點保護，畢竟核心邏輯洩露了就麻煩了。

### 2. 字串加密執行時解密

建構時加密字串，執行時解密：

```csharp
public static class StringDecryption
{
    [VmProtect(VmProtectionPriority.High, Reason = "CRITICAL SECURITY")]
    public static string DecryptString(byte[] encryptedData, byte[] key, byte[] iv)
    {
        using var aes = Aes.Create();
        aes.Key = key;
        aes.IV = iv;

        using var decryptor = aes.CreateDecryptor();
        using var ms = new MemoryStream(encryptedData);
        using var cs = new CryptoStream(ms, decryptor, CryptoStreamMode.Read);
        using var reader = new StreamReader(cs);

        return reader.ReadToEnd();
    }
}

// 生產常量存取器（懶載入 + 快取）
public static class ProductionConstants
{
    private static string? _cachedLicenseServerUrl;

    public static string GetLicenseServerUrl(IOptions<LicenseOptions> options)
    {
        if (_cachedLicenseServerUrl == null)
        {
            var encrypted = GetEncryptedLicenseServerUrl();
#if DEBUG
            _cachedLicenseServerUrl = options.Value.PrimaryServer.Url;
#else
            _cachedLicenseServerUrl = StringDecryption.DecryptString(
                encrypted,
                GetEncryptionKey(),
                GetEncryptionIV());
#endif
        }
        return _cachedLicenseServerUrl;
    }
}
```

字串加密這個環節也挺重要的，畢竟敏感資訊不能明文放著。

### 3. VMP 保護驗證

建構後必須驗證保護是否成功套用，不然怎麼知道保護生效了呢：

```csharp
// 驗證腳本範例
public bool VerifyProtection(string assemblyPath)
{
    // 1. 檢查 VMP 簽名
    var bytes = File.ReadAllBytes(assemblyPath);
    var vmpSignature = Encoding.ASCII.GetBytes("VMProtect");
    if (bytes.Any(b => vmpSignature.Contains(b)))
    {
        return true;
    }

    // 2. 檢查檔案大小變化（保護後通常會增大）
    var originalInfo = new FileInfo(assemblyPath.Replace(".dll", ".bak"));
    if (originalInfo.Exists)
    {
        var sizeRatio = (double)new FileInfo(assemblyPath).Length / originalInfo.Length;
        return sizeRatio > 1.1;
    }

    return false;
}
```

驗證一下總是好的，省得到時候出了問題還不知道。

### 4. 注意事項

這裡有幾個坑需要特別注意，畢竟我們都是踩過來的：

1. **不要混淆所有程式碼**：公共 API、介面定義、 DTO 類別通常不需要保護，過度混淆會影響效能和除錯，HagiCode 專案在這方面就吃過虧，這點得注意

2. **保護金鑰存取器**：加密金鑰的取得方法必須與加密資料享受同級或更高级別的保護，否則就是形同虛設，也沒什麼意義了

3. **測試與生產的平衡**：DEBUG 建構應跳過加密以便於開發除錯，RELEASE 建構啟用完整保護，記得用條件編譯 `#if DEBUG` 來區分，這樣開發起來也方便

4. **Docker 環境考慮**：在 Linux 環境下執行 VMP 需要使用容器化方案，確保保護工具的相容性，HagiCode 使用的是 Wine + VMP 的容器方案，跨平台的問題倒是这样解決的

5. **驗證不可少**：建構完成後必須驗證保護是否成功，否則可能導致敏感程式碼暴露，前面的驗證程式碼就是這個作用，還是檢查一下比較好

## 總結

透過這種多層保護策略，HagiCode 實現了從基礎混淆到虛擬機保護的全面程式碼安全體系：

- **第一層**：使用 `ObfuscationAttribute` 進行基礎標記，為第三方工具提供提示
- **第二層**：透過自訂 `VmProtectAttribute` 宣告保護意圖和優先級
- **第三層**：VMP 虛擬機保護將關鍵程式碼轉換為不可逆的虛擬機指令
- **第四層**：建構時自動掃描套用保護，驗證保護結果

這套方案既能抵禦普通反編譯工具，又能對抗高階逆向分析攻擊。如果你也在開發需要程式碼保護的 .NET 應用，希望能給你一些參考，哪怕是一點點啟發也夠了。
