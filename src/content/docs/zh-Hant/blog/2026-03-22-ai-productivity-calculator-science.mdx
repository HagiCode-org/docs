## 我好像會被 Agent 淘汰，我用數據算了一算

> 用數據量化 AI 替代風險：深入解析 HagiCode 團隊如何用 6 個核心公式，重新定義知識工作者的競爭力評估標準。

## 背景

在 AI 技術飛速發展的今天，每一個知識工作者都面臨一個緊迫的問題：**在 AI 時代，我是否會被淘汰？**

這個問題聽起來有點危言聳聽，但其實很多人心裡都在打鼓。前腳剛學會一個框架，後腳 AI 就說你這個崗位要被替代了；好不容易精通了一門語言，結果發現用 AI 的人產出是你的三倍——這種焦慮感，我相信屏幕前的你多少都能體會。

其實吧，這種焦慮也不是沒有道理。畢竟誰也不願意承認，自己奮鬥多年的技能，可能被一個 ChatGPT 就給超越了。只是焦慮歸焦慮，日子還得過不是？

傳統觀點往往從「AI 能做什麼」出發討論替代風險，但這種方法忽略了兩個關鍵維度：

1. **企業視角**：企業是否願意為一個員工配備 AI 工具，取決於 AI 成本相對於人力成本的性價比。不是說 AI 能取代這個崗位，企業就會立刻換人，還要算算經濟賬。畢竟資本家也不是慈善家，每一分錢都要花在刀刃上。

2. **效率視角**：AI 帶來的效率提升需要被量化，而不是簡單地認為「用了 AI 就更強」。你用 AI 效率提升了 2 倍，但他用 AI 提升了 5 倍，這裡面的差距可不小。就像學生時代，都在聽課，有的考 90 分，有的才及格——差距就是這麼拉開的。

所以關鍵問題是：**怎麼把這種模糊的焦慮，變成可以量化的指標？**

畢竟知道自己的位置在哪，總比在黑暗中摸索要好一些。這就是我們今天要聊的——HagiCode 團隊開發的 AI 人效計算器背後的設計邏輯。

於是我做了一個 https://cost.hagicode.com 的網站。

## 核心公式：6 個關鍵指標

### 1. 年度全用工成本

企業為員工付出的真實成本遠不止工資。這一點很多人跳槽的時候才發現——明明談的是 2 萬月薪，到手怎麼就 1 萬 4？公司那邊可不止出 2 萬，社保、公積金、培訓、招聘成本都要算進去。

根據 `calculate-ai-risk.ts` 中的實現：

```
年度全用工成本 = 年薪 × (1 + 城市係數) + 年薪 / 12
```

城市係數反映的是不同城市的人才招募和保留成本：

| 城市層級 | 代表城市 | 係數 |
|---------|---------|------|
| 一線 | 北京/上海/深圳/廣州 | 0.4 |
| 新一線 | 杭州/成都/蘇州/南京 | 0.3 |
| 二線 | 武漢/西安/天津/鄭州 | 0.2 |
| 其他 | 宜昌/洛陽等 | 0.1 |

一線城市係數是 0.4，意思是企業需要額外支付約 40% 的招募、培訓、社保等附加成本。在北京招一個人的綜合成本，確實比在二線城市高不少。

畢竟在大城市生存，生活成本也高，這算是另一種形式的「漂泊者稅」了吧。

### 2. 綜合 Token 單價

不同 AI 模型有 Input 和 Output 兩種價格，而且差異巨大。程式碼場景下輸入輸出比例大約是 3:1——你給 AI 一段程式碼讓它 review，輸出的分析文字通常比輸入的程式碼短很多。

綜合單價計算公式：

```
綜合單價 = (輸入輸出比例 × 輸入單價 + 輸出單價) / (輸入輸出比例 + 1)
```

拿 GPT-5 舉個例子：
- 輸入：$2.5/1M tokens
- 輸出：$15/1M tokens
- 綜合 = (3 × 2.5 + 15) / 4 = $5.625/1M tokens

對於 USD 定價的模型，還需要按匯率轉換。這個匯率 HagiCode 團隊設定為 7.25，會隨市場波動更新。

匯率這東西，就像股市一樣，誰也猜不準。只能跟著走，罷了。

### 3. AI 年成本

```
日均 AI 成本 = 日均 Token 需求 (M) × 綜合單價 (CNY/1M)
年 AI 成本 = 日均 AI 成本 × 264 個工作日
```

264 = 22 天/月 × 12 月，這是標準工作制下的年度工作日數量。為什麼不用 365 天？因為你要考慮週末、節假日、病假等因素。

畢竟咱們也不是機器人，該休息的時候還是要休息的。雖說 AI 可能不需要休息，但咱們還是要給自己留點喘息的空間。

### 4. 核心創新：等效人力

這是整個評估體系的核心，也是 HagiCode 團隊最有洞察力的地方。

```
可負擔工作流份數 = 年度全用工成本 / AI 年成本
可負擔比例 = min(可負擔工作流份數, 1)
等效人力 = 1 + (效率倍數 - 1) × 可負擔比例
```

等等，這個公式有點繞，讓我解釋一下：

傳統觀點會直接說「你的效率提升了 2 倍」，但這個公式考慮了一個關鍵約束：**企業的 AI 預算是否可持續？**

舉個例子：小明效率提升了 3 倍，但他的 AI 消耗成本每年要 30 萬；而公司給他的年薪才 20 萬。這種情況下，雖然小明個人效率很高，但他實際上是不可持續的——公司不可能為了讓他維持高效率而虧本。

可負擔比例就是這個意思：如果企業只能負擔 0.5 份 AI 工作流，那小明的等效人力 = 1 + (3-1) × 0.5 = 2 人，而不是 3 人。

**核心洞察**：不是你的效率倍數有多高，而是企業能否負擔得起你維持這個效率所需的 AI 投入。

其實這個道理也挺簡單的，只是很多人沒往這方面想罷了。畢竟咱們習慣了從自己的角度看問題，很少站在老闆的角度考慮一下——他們的錢也不是大風刮來的。

### 5. 成本效益比

```
AI 成本佔比 = AI 年成本 / 年度全用工成本
效率增幅 = 效率倍數 - 1
成本效益比 = 效率增幅 / AI 成本佔比
```

- 成本效益比 < 1：AI 投入不划算，效率提升抵不上成本
- 成本效益比 1-2：剛好划算
- 成本效益比 > 2：高收益，強烈推薦

這個指標對於企業管理者特別有用，可以快速評估某個崗位是否值得投入 AI 工具。

畢竟 ROI 才是王道，你說自己效率提升再多，成本爆炸也沒人買賬。

### 6. 風險等級

根據等效人力劃分風險：

| 等效人力 | 風險等級 | 結論 |
|---------|---------|------|
| >= 2.0 | 高危 | 同事一旦具備同等條件，對你威脅很高 |
| 1.5 - 2.0 | 警示 | 同事已開始形成明顯效率優勢 |
| < 1.5 | 安全 | 暫時還能保持差距 |

看到這個表格，你心裡大概也有個數了吧。只是別太焦慮，畢竟焦慮也解決不了問題——不如想想怎麼提升自己的效率倍數。

## 實際使用示例

假設你是一個北京的開發者，年薪 40 萬，使用 Claude Sonnet 4.6，日均 Token 消耗 50M，自評效率提升 3 倍。模擬輸入：

```typescript
const input = {
  annualIncomeCny: 400000,
  cityTier: "tier1",           // 北京
  modelId: "claude-sonnet-4-6",
  performanceMultiplier: 3.0,
  dailyTokenUsageM: 50,
}

// 計算過程
// 年度全用工成本 = 40萬 × (1 + 0.4) + 40萬/12 ≈ 60.33萬
// AI 年成本 ≈ 50 × 7.125 × 264 ≈ 9.4萬
// 可負擔工作流份數 ≈ 60.33 / 9.4 ≈ 6.4 份
// 等效人力 = 1 + (3 - 1) × 1 = 3 人
```

結論：你的同事如果具備相同條件，能相當於 3 個人的產能，你已經處於高危區。

如果你發現自己的 AI 用法「不划算」（成本效益比 < 1），可以考慮：

1. **降低 Token 消耗**：使用更高效的 prompt，減少無效請求
2. **選擇性價比模型**：如 DeepSeek-V3（人民幣計價，更便宜）
3. **提升效率倍數**：學習高級 Agent 使用技巧，真正把 AI 變成生產力

其實這些問題，歸根結底就是一個平衡的藝術罷了。用多了浪費錢，用少了沒效果——找到那個剛剛好的點，才是關鍵。
