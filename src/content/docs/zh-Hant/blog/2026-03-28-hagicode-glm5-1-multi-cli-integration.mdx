## Hagicode 聯合 GLM-5.1 多 CLI 整合指南

:::note[時間語境說明]
本文記錄的是發布當時的多 CLI 接入設計與模型對應。目前常駐支援範圍請以 [AI Agent CLI 安裝](/related-software-installation/ai-agent-cli/) 與 [安裝指南](/installation) 為準；文中的 `IFlowCli` 僅代表當時的歷史配置，不屬於目前活躍支援範圍。
:::

## 背景

在 Hagicode 專案中，使用者可以選擇多種 CLI 工具來驅動 AI 程式設計助理，包括 Claude Code CLI、GitHub Copilot、OpenCode CLI、Codebuddy CLI、Hermes CLI 等。這些 CLI 工具本身是通用的 AI 程式設計工具，但透過 Hagicode 的抽象層，可以靈活地接入不同的 AI 模型供應商。

智譜 AI（ZAI）提供了與 Anthropic Claude API 相容的介面，使得這些 CLI 工具可以直接使用國產 GLM 系列模型。其中 GLM-5.1 是智譜最新發布的大型語言模型，相比 GLM-5.0 有顯著的改進。

## Hagicode 的 CLI 抽象架構

Hagicode 透過 `AIProviderType` 枚舉定義了 11 種 CLI 供應商類型，覆蓋了主流的 AI 程式設計 CLI 工具：

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

每種 CLI 都有對應的模型參數配置，支援 `model` 和 `reasoning` 參數：

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],
        [AIProviderType.CodexCli] = ["model", "reasoning"],
        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],
        [AIProviderType.HermesCli] = ["model", "reasoning"],
        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini 不支援 reasoning 參數
        // ...
    };
```

## GLM 模型支援體系

Hagicode 的 Secondary Professions Catalog 中定義了完整的 GLM 系列模型支援：

| Model ID | Name | Default Reasoning | Compatible CLI Families |
|----------|------|-------------------|-------------------------|
| `glm-4.7` | GLM 4.7 | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM 5 Turbo | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (Legacy) | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | high | claude, codebuddy, hermes, qoder, kiro |

## 在不同 CLI 中使用 GLM-5.1

### 1. Claude Code CLI + GLM-5.1

Claude Code CLI 是 Hagicode 的核心 CLI 之一，透過 Hero 配置系統設定：

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

對應的 `HeroEquipmentCatalogItem` 配置：

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI 是最靈活的 CLI，支援 `provider/model` 格式指定任意模型：

**方式一：使用 ZAI provider 前綴**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**方式二：直接使用模型 ID**
```json
{
  "model": "glm-5.1"
}
```

**方式三：前端配置介面**

在 `HeroModelEquipmentForm.tsx` 中，OpenCode CLI 有特殊的佔位符提示：

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

使用者可以輸入：
```
zai/glm-5.1
glm-5.1
```

**OpenCode CLI 的模型解析邏輯**：

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // 無斜線：直接使用模型 ID
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // 有斜線：解析 provider/model 格式
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI 有特殊的 legacy 處理邏輯：

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**注意**：Codebuddy 對 GLM-5.0 有特殊保留，不走 legacy normalization：

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// CodebuddyCli 時，glm-5.0 不會被正規化為 glm-5-turbo
```

## GLM-5.1 的改進優勢

相比 GLM-5.0，GLM-5.1 有以下顯著改進：

### 1. 更好的推理能力

根據智譜官方發布資訊，GLM-5.1 的改進包括：

- **更強的程式碼理解能力**：對複雜程式碼結構的解析更準確
- **更長的上下文理解**：支援更長的對話上下文
- **工具呼叫增強**：MCP 工具呼叫的成功率提升
- **輸出穩定性**：減少隨機性和幻覺

### 2. 多 CLI 全面相容

GLM-5.1 覆蓋了 Hagicode 支援的所有主流 CLI：

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## 注意事項

### 1. API Key 配置

確保正確設定 `ANTHROPIC_AUTH_TOKEN` 環境變數，這是所有 CLI 連接模型的必要憑證。

### 2. 模型可用性

GLM-5.1 需要在對應的模型供應商處開通：
- 智譜 AI ZAI 平台預設支援
- 阿里雲 DashScope 可能需要單獨申請

### 3. OpenCode CLI 格式

使用 `provider/model` 格式時，確保 provider ID 正確：
- 智譜 AI：`zai` 或 `zhipuai`
- 阿里雲：`aliyun` 或 `dashscope`

### 4. Reasoning 參數

- 建議使用 `high` 級別以獲得最佳程式碼生成效果
- Gemini CLI 不支援 reasoning 參數，會自動忽略該配置

## 總結

Hagicode 透過統一的抽象層，實現了 GLM-5.1 與多種 CLI 的靈活整合。開發者可以根據自己的偏好和使用場景，選擇合適的 CLI 工具，並透過簡單的配置使用最新的 GLM-5.1 模型。

GLM-5.1 作為智譜最新的模型版本，相比 GLM-5.0 有明顯的改進：
- 獨立的版本標識，無 legacy 包袱
- 更強的推理能力和程式碼理解
- 廣泛的多 CLI 相容性
- 靈活的推理級別配置

透過正確配置環境變數和 Hero 裝備，玩家可以在不同的 CLI 環境中充分發揮 GLM-5.1 的強大能力。
