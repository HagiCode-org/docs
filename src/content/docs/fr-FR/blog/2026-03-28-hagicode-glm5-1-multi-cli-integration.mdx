## Guide d'intégration multi-CLI de Hagicode avec GLM-5.1

:::note[Note sur le contexte temporel]
Cet article documente la conception d'intégration multi-CLI et le mappage des modèles au moment de la publication. Pour la plage actuelle de support permanent, veuillez vous référer à [Installation de l'AI Agent CLI](/related-software-installation/ai-agent-cli/) et au [Guide d'installation](/installation) ; le `IFlowCli` mentionné dans cet article représente uniquement la configuration historique de l'époque et n'appartient pas à la plage de support active actuelle.
:::

## Contexte

Dans le projet Hagicode, les utilisateurs peuvent choisir parmi plusieurs outils CLI pour piloter l'assistant de programmation IA, notamment Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI, etc. Ces outils CLI sont eux-mêmes des outils de programmation IA génériques, mais grâce à la couche d'abstraction de Hagicode, ils peuvent s'intégrer de manière flexible à différents fournisseurs de modèles IA.

Zhipu AI (ZAI) fournit une interface compatible avec l'API Anthropic Claude, permettant à ces outils CLI d'utiliser directement les modèles de la série GLM domestique. Parmi eux, GLM-5.1 est le dernier grand modèle de langage publié par Zhipu, avec des améliorations significatives par rapport à GLM-5.0.

## Architecture d'abstraction CLI de Hagicode

Hagicode définit 11 types de fournisseurs CLI via l'énumération `AIProviderType`, couvrant les principaux outils CLI de programmation IA :

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Chaque CLI possède une configuration de paramètres de modèle correspondante, prenant en charge les paramètres `model` et `reasoning` :

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],
        [AIProviderType.CodexCli] = ["model", "reasoning"],
        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],
        [AIProviderType.HermesCli] = ["model", "reasoning"],
        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini ne prend pas en charge le paramètre reasoning
        // ...
    };
```

## Système de support des modèles GLM

Le Secondary Professions Catalog de Hagicode définit le support complet de la série de modèles GLM :

| Model ID | Name | Default Reasoning | Compatible CLI Families |
|----------|------|-------------------|-------------------------|
| `glm-4.7` | GLM 4.7 | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM 5 Turbo | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (Legacy) | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | high | claude, codebuddy, hermes, qoder, kiro |

## Utilisation de GLM-5.1 avec différents CLI

### 1. Claude Code CLI + GLM-5.1

Claude Code CLI est l'un des CLI principaux de Hagicode, configuré via le système de configuration Hero :

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Configuration `HeroEquipmentCatalogItem` correspondante :

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI est le CLI le plus flexible, prenant en charge le format `provider/model` pour spécifier n'importe quel modèle :

**Méthode 1 : Utiliser le préfixe du fournisseur ZAI**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Méthode 2 : Utiliser directement l'ID du modèle**
```json
{
  "model": "glm-5.1"
}
```

**Méthode 3 : Interface de configuration frontend**

Dans `HeroModelEquipmentForm.tsx`, OpenCode CLI a un indicateur d'espace réservé spécial :

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Les utilisateurs peuvent saisir :
```
zai/glm-5.1
glm-5.1
```

**Logique de résolution de modèle d'OpenCode CLI** :

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // Pas de barre oblique : utiliser directement l'ID du modèle
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Barre oblique présente : analyser le format provider/model
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI a une logique de traitement legacy spéciale :

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Attention** : Codebuddy a une conservation spéciale pour GLM-5.0, ne passant pas par la normalisation legacy :

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// Avec CodebuddyCli, glm-5.0 ne sera pas normalisé en glm-5-turbo
```

## Avantages des améliorations de GLM-5.1

Par rapport à GLM-5.0, GLM-5.1 présente les améliorations significatives suivantes :

### 1. Meilleures capacités de raisonnement

Selon les informations officielles de publication de Zhipu, les améliorations de GLM-5.1 comprennent :

- **Capacité de compréhension du code renforcée** : analyse plus précise des structures de code complexes
- **Compréhension contextuelle plus longue** : prise en charge de contextes de conversation plus longs
- **Appels d'outils améliorés** : augmentation du taux de succès des appels d'outils MCP
- **Stabilité de sortie** : réduction de l'aléatoire et des hallucinations

### 2. Compatibilité multi-CLI complète

GLM-5.1 couvre tous les CLI principaux pris en charge par Hagicode :

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Points à noter

### 1. Configuration de la clé API

Assurez-vous de définir correctement la variable d'environnement `ANTHROPIC_AUTH_TOKEN`, qui est la credential nécessaire pour que tous les CLI se connectent aux modèles.

### 2. Disponibilité des modèles

GLM-5.1 doit être activé auprès du fournisseur de modèles correspondant :
- La plateforme Zhipu AI ZAI prend en charge par défaut
- Alibaba Cloud DashScope peut nécessiter une demande séparée

### 3. Format OpenCode CLI

Lors de l'utilisation du format `provider/model`, assurez-vous que l'ID du fournisseur est correct :
- Zhipu AI : `zai` ou `zhipuai`
- Alibaba Cloud : `aliyun` ou `dashscope`

### 4. Paramètre Reasoning

- Il est recommandé d'utiliser le niveau `high` pour obtenir les meilleurs résultats de génération de code
- Gemini CLI ne prend pas en charge le paramètre reasoning et ignorera automatiquement cette configuration

## Résumé

Hagicode réalise, grâce à une couche d'abstraction unifiée, l'intégration flexible de GLM-5.1 avec plusieurs CLI. Les développeurs peuvent choisir l'outil CLI approprié en fonction de leurs préférences et de leurs scénarios d'utilisation, et utiliser le dernier modèle GLM-5.1 grâce à une configuration simple.

En tant que dernière version du modèle de Zhipu, GLM-5.1 présente des améliorations évidentes par rapport à GLM-5.0 :
- Identifiant de version indépendant, sans fardeau legacy
- Capacités de raisonnement et compréhension du code renforcées
- Large compatibilité multi-CLI
- Configuration flexible des niveaux de raisonnement

En configurant correctement les variables d'environnement et les équipements Hero, les joueurs peuvent pleinement exploiter la puissance de GLM-5.1 dans différents environnements CLI.
