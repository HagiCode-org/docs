## Comment implémenter la réessai automatique pour les CLIs Agent comme Claude Code et Codex

> L'expression réessai automatique ressemble à un petit interrupteur, mais une fois placée dans un environnement d'ingénierie réel, c'est tout autre chose. Bonjour à tous, je suis Yu Kun, créateur de HagiCode. Aujourd'hui, je ne veux pas faire de discours creux. Je veux parler de la façon dont le réessai automatique pour les CLIs Agent tels que Claude Code et Codex devrait être réellement implémenté, afin qu'il puisse à la fois récupérer des exceptions et éviter d'entraîner le système dans une exécution répétée sans fin.

## Contexte

Si vous travaillez également sur le codage IA dernièrement, vous avez probablement déjà rencontré ce genre de problème : la tâche n'échoue pas immédiatement, mais se bloque en cours d'exécution.

Dans une requête HTTP ordinaire, cela signifie généralement simplement renvoyer la requête, peut-être avec un backoff exponentiel. Mais les CLIs Agent sont différents. Des outils comme Claude Code et Codex s'exécutent généralement de manière en flux continu, poussant la sortie morceau par morceau. Pendant ce processus, ils peuvent aussi se lier à un thread, une session ou un token de reprise.

La première fois que beaucoup d'équipes construisent cette partie, elles écrivent instinctivement la version la plus naïve : en cas d'erreur, réessayer une fois. Cette idée est parfaite en théorie, mais en pratique, elle peut transformer une erreur temporaire en un désastre permanent.

## À propos de HagiCode

L'approche partagée dans cet article provient de notre expérience pratique dans le projet [HagiCode](https://hagicode.com). HagiCode est un assistant de codage IA qui intègre plusieurs CLIs Agent.

## Architecture du réessai

La conception du réessai automatique pour les CLIs Agent doit prendre en compte plusieurs dimensions : le contenu déjà émis est-il encore valide ? Le contexte actuel peut-il continuer à s'exécuter ? Cette erreur doit-elle être récupérée automatiquement ?

## Conclusion

Le réessai automatique pour les CLIs Agent est un problème d'ingénierie plus complexe qu'il n'y paraît. La clé est de comprendre la sémantique de l'erreur et de concevoir une stratégie de réessai appropriée.
