Si vous souhaitez installer Hermes et commencer à l'utiliser, le chemin le plus court se résume vraiment à trois étapes :

1. Exécuter la commande d'installation officielle
2. Lancer le CLI dans votre terminal avec `hermes`
3. Si vous souhaitez continuer à l'utiliser dans Feishu, configurez `hermes gateway setup`

Cet article ne cherche pas à expliquer toutes les capacités d'Hermes d'un coup. Il vous aide plutôt à boucler d'abord la boucle de démarrage la plus importante : l'installer, le lancer, commencer à l'utiliser, puis le connecter à l'un des scénarios de plateforme de messagerie les plus courants.

## Ce qu'est Hermes et à qui il s'adresse

Hermes Agent est un agent IA que vous pouvez utiliser soit depuis un terminal local, soit via une passerelle de plateforme de messagerie.

Pour la plupart des développeurs, il offre deux points d'entrée courants :

- **CLI** : tapez `hermes` dans votre terminal pour accéder directement à l'interface interactive.
- **Passerelle de messagerie** : exécutez `hermes gateway`, puis discutez avec Hermes depuis des plateformes telles que Feishu, Telegram, Discord et Slack.

Si votre objectif actuel est simplement de démarrer rapidement, ne modifiez pas cet ordre. Suivez plutôt ce cheminement :

- Installez Hermes d'abord
- Vérifiez qu'il fonctionne depuis le CLI d'abord
- Ensuite, décidez si vous souhaitez connecter une plateforme de messagerie

Cela facilite le diagnostic des problèmes et convient mieux aux personnes qui utilisent Hermes pour la première fois.

## Ce qu'il faut savoir avant d'installer Hermes

Selon le README d'Hermes, le chemin d'installation rapide officiel prend en charge ces environnements :

- Linux
- macOS
- WSL2
- Android via Termux

### Notes pour les utilisateurs Windows

Hermes ne prend actuellement pas en charge l'exécution directe sous Windows natif. Si vous utilisez Windows, l'approche recommandée est d'installer WSL2 en premier, puis d'exécuter la commande d'installation à l'intérieur de WSL2.

Il est préférable de le préciser dès le début, car beaucoup d'échecs d'installation ne sont pas causés par la commande elle-même, mais par l'utilisation d'un environnement d'exécution non pris en charge.

## Comment installer Hermes

La commande d'installation rapide fournie dans le README d'Hermes est :

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Cette commande exécute le script d'installation officiel et gère les étapes d'initialisation spécifiques à la plateforme.

### Recharger votre shell après l'installation

Une fois l'installation terminée, rechargez d'abord votre environnement de shell. La commande la plus courante est :

```bash
source ~/.bashrc
```

Si vous utilisez zsh, vous pouvez utiliser :

```bash
source ~/.zshrc
```

### Comment confirmer qu'Hermes est installé correctement

Le moyen le plus direct de vérifier est d'exécuter :

```bash
hermes
```

Si vous souhaitez une confirmation supplémentaire que votre configuration et vos dépendances fonctionnent, vous pouvez aussi exécuter :

```bash
hermes doctor
```

`hermes doctor` est particulièrement utile dans ces situations :

- La commande se comporte anormalement après l'installation
- La configuration du modèle échoue
- La passerelle échoue au démarrage
- Vous n'êtes pas sûr que vos dépendances d'environnement sont complètes

## Comment commencer à utiliser Hermes pour la première fois

### Commencer avec le CLI

Si vous voulez simplement confirmer le plus rapidement possible qu'Hermes fonctionne, la méthode la plus simple est :

```bash
hermes
```

Cela lance le CLI interactif d'Hermes. Pour les primo-utilisateurs d'Hermes, c'est aussi le point de départ le plus recommandé, car vous pouvez vérifier d'abord les choses les plus essentielles :

- Que la commande est effectivement disponible
- Que la configuration du modèle actuel fonctionne correctement
- Que la chaîne d'outils du terminal fonctionne correctement
- Que le style d'interaction correspond à vos besoins

### Ces commandes suffisent pour votre premier cycle de configuration

Le README d'Hermes liste plusieurs commandes à haute fréquence, qui forment ensemble un cheminement pratique de première utilisation :

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Si vous n'êtes pas sûr de ce que fait chacune d'elles, retenez-les ainsi :

- `hermes model` : choisir ou changer de modèle
- `hermes tools` : voir et configurer les outils actuellement disponibles
- `hermes config set` : modifier des éléments de configuration spécifiques
- `hermes setup` : exécuter l'assistant d'initialisation complet une fois
- `hermes update` : mettre à jour Hermes
- `hermes doctor` : résoudre les problèmes

Pour les débutants, l'ordre le plus pratique est généralement :

1. Exécutez d'abord `hermes model`
2. Si vous souhaitez configurer toutes les options courantes d'un coup, exécutez ensuite `hermes setup`

## Les deux façons les plus courantes d'utiliser Hermes

### 1. Utiliser Hermes dans le terminal comme assistant de développement quotidien

Le mode CLI convient bien à ces scénarios :

- Poser des questions directement pendant l'écriture de code en local
- Inspecter des projets, modifier des fichiers et exécuter des commandes
- Effectuer du débogage ponctuel ou des revues de code
- Collaborer en continu dans le répertoire de travail actuel

Son plus grand avantage est qu'il représente le chemin le plus court : pas d'intégration de plateforme supplémentaire, pas de configuration de bot à gérer au préalable, et c'est la meilleure façon de construire vos premières habitudes d'utilisation.

### 2. Utiliser Hermes via une plateforme de messagerie

Si vous souhaitez discuter avec Hermes sur des plateformes comme Feishu, Telegram ou Discord, vous devez utiliser la passerelle de messagerie.

Les commandes d'entrée les plus courantes sont :

```bash
hermes gateway setup
hermes gateway
```

Plus précisément :

- `hermes gateway setup` est utilisé pour la configuration interactive de la plateforme
- `hermes gateway` est utilisé pour lancer le processus de passerelle

Selon la documentation officielle, la passerelle est un processus d'arrière-plan unifié qui connecte vos plateformes configurées, gère les sessions et traite des fonctionnalités telles que les tâches planifiées.

## Exemple avec Feishu : comment connecter Hermes à une plateforme de messagerie

Si la majeure partie de votre travail quotidien se déroule dans Feishu, Feishu/Lark est une façon très naturelle d'utiliser Hermes.

### Le chemin d'intégration minimal viable

La documentation officielle recommande cette commande d'entrée pour Feishu/Lark :

```bash
hermes gateway setup
```

Après l'avoir exécutée, choisissez simplement **Feishu / Lark** dans l'assistant.

La documentation Feishu décrit deux modes de connexion :

- `websocket` : recommandé
- `webhook` : optionnel

Si Hermes s'exécute sur votre ordinateur portable, votre station de travail ou un serveur privé, l'utilisation de `websocket` en premier est généralement plus simple car vous n'avez pas besoin d'exposer une URL de callback publique.

### Si vous configurez manuellement, connaissez au moins ces variables

Si vous n'utilisez pas l'assistant et que vous écrivez la configuration manuellement, la documentation Feishu liste ces variables principales :

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Deux d'entre elles méritent une attention particulière :

- `FEISHU_ALLOWED_USERS` : recommandé, pour que tout le monde pouvant atteindre le bot ne puisse pas l'utiliser directement
- `FEISHU_HOME_CHANNEL` : vous permet de prédéfinir un canal d'accueil pour recevoir les résultats des tâches planifiées ou les notifications par défaut

### Pourquoi Hermes ne répond parfois pas dans les conversations de groupe Feishu

Ce détail est facile à oublier : dans les conversations de groupe Feishu, Hermes ne répond pas à chaque message par défaut.

La documentation officielle précise clairement :

- En messages directs, Hermes répond aux messages
- En conversations de groupe, vous devez explicitement `@` le bot pour qu'il traite le message

Si vous souhaitez définir une conversation Feishu comme canal d'accueil, vous pouvez aussi utiliser ceci dans le chat :

```text
/set-home
```

Ou le définir dans la configuration à l'avance :

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Les commandes Hermes que les débutants devraient retenir en priorité

Que vous utilisiez Hermes en CLI ou sur une plateforme de messagerie, retenir les commandes suivantes suffit déjà pour démarrer :

- `/new` ou `/reset` : démarrer une nouvelle session
- `/model` : voir ou changer de modèle
- `/retry` : réessayer le tour précédent
- `/undo` : annuler l'interaction précédente
- `/compress` : compresser manuellement le contexte
- `/help` : afficher l'aide

Si vous utilisez principalement Hermes sur une plateforme de messagerie, retenez-en une de plus :

- `/sethome` ou `/set-home` : définir le chat actuel comme canal d'accueil

Ces commandes couvrent les opérations les plus courantes au stade débutant : redémarrer, ajuster, revenir en arrière, vérifier et continuer.

## Questions fréquentes

### Peut-on installer Hermes directement sous Windows ?

Non. La documentation officielle actuelle précise clairement que Windows natif n'est pas pris en charge, et WSL2 est recommandé.

### Que dois-je faire si taper `hermes` ne fait rien après l'installation ?

Il vaut mieux dépanner dans cet ordre :

1. Rechargez d'abord votre shell, par exemple avec `source ~/.bashrc`
2. Exécutez `hermes` à nouveau
3. Si c'est toujours anormal, exécutez `hermes doctor`

### Pourquoi le bot ne répond-il pas dans un groupe Feishu ?

Vérifiez d'abord ces trois points :

- Que vous avez `@` mentionné Hermes dans le groupe
- Que `FEISHU_ALLOWED_USERS` ne restreint pas l'utilisateur actuel
- Que la politique de conversation de groupe actuelle autorise le traitement des messages de groupe

Selon la documentation officielle Feishu, l'utilisation explicite d'une mention @ est requise dans les scénarios de conversation de groupe.

3. Utilisez `hermes model` et `hermes setup` pour compléter la configuration de base
4. Si vous souhaitez continuer à l'utiliser dans Feishu, configurez `hermes gateway setup`
