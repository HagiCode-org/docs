Pour installer et commencer à utiliser Hermes, le chemin le plus court ne comprend en réalité que trois étapes :

1. Exécuter la commande d'installation officielle
2. Lancer `hermes` dans le terminal pour démarrer le CLI
3. Si vous souhaitez continuer à l'utiliser dans Feishu, configurez `hermes gateway setup`

Cet article ne vise pas à expliquer toutes les capacités d'Hermes en une seule fois, mais plutôt à vous aider à compléter la boucle d'initiation la plus critique : l'installer, le faire fonctionner, commencer à l'utiliser, puis le connecter à un scénario de plateforme de messagerie courant.

## Qu'est-ce qu'Hermes et à qui s'adresse-t-il

Hermes Agent est un agent AI qui peut être utilisé à la fois dans un terminal local et via une passerelle de plateforme de messagerie.

Pour la plupart des développeurs, il existe deux points d'entrée les plus courants :

- **CLI** : Tapez `hermes` dans le terminal pour accéder directement à l'interface interactive
- **Messaging Gateway** : Exécutez `hermes gateway`, puis dialoguez avec lui depuis des plateformes comme Feishu, Telegram, Discord, Slack, etc.

Si votre objectif actuel est simplement de prendre en main rapidement, il est conseillé de ne pas inverser l'ordre, mais de suivre ce chemin :

- Commencez par installer Hermes
- Vérifiez d'abord qu'il fonctionne via le CLI
- Décidez ensuite si vous souhaitez connecter une plateforme de messagerie

Cela facilite le diagnostic des problèmes et convient mieux aux utilisateurs qui découvrent Hermes pour la première fois.

## Ce qu'il faut savoir avant d'installer Hermes

Selon le README d'Hermes, le chemin d'installation rapide officielle prend en charge ces environnements :

- Linux
- macOS
- WSL2
- Android via Termux

### Remarques pour les utilisateurs Windows

Hermes ne prend pas actuellement en charge l'exécution native directe sur Windows. Si vous utilisez Windows, il est recommandé d'installer d'abord WSL2, puis d'exécuter la commande d'installation dans WSL2.

Il est préférable de le préciser dès le début de l'article, car beaucoup d'échecs d'installation ne sont pas dus à un problème de commande, mais à un environnement d'exécution qui ne répond pas aux exigences.

## Comment installer Hermes

La commande d'installation rapide fournie par le README d'Hermes est :

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Cette commande exécute le script d'installation officiel et gère les processus d'initialisation spécifiques à la plateforme.

### Actualiser le shell après l'installation

Une fois l'installation terminée, rechargez d'abord l'environnement du shell. Le plus courant est :

```bash
source ~/.bashrc
```

Si vous utilisez zsh, vous pouvez également utiliser :

```bash
source ~/.zshrc
```

### Comment vérifier qu'Hermes est correctement installé

La méthode de vérification la plus directe consiste à exécuter :

```bash
hermes
```

Si vous souhaitez vérifier davantage que la configuration et les dépendances sont normales, vous pouvez également exécuter :

```bash
hermes doctor
```

`hermes doctor` est utile dans ces situations :

- Comportement anormal après l'installation
- Échec de la configuration du modèle
- Échec du démarrage de la passerelle
- Incertitude sur la complétude des dépendances de l'environnement

## Comment utiliser Hermes pour la première fois

### Commencer par le CLI

Si vous voulez simplement confirmer rapidement qu'Hermes fonctionne, la méthode la plus simple est :

```bash
hermes
```

Cela lancera le CLI interactif d'Hermes. Pour les utilisateurs qui découvrent Hermes pour la première fois, c'est le point de départ recommandé, car vous pouvez d'abord vérifier les éléments les plus essentiels :

- La commande est-elle vraiment disponible
- La configuration du modèle actuel est-elle normale
- La chaîne d'outils du terminal fonctionne-t-elle normalement
- L'expérience interactive correspond-elle à ce dont vous avez besoin

### Ces commandes suffisent pour compléter la première configuration

Les commandes à haute fréquence répertoriées dans le README d'Hermes constituent essentiellement le chemin de première prise en main :

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Si vous ne savez pas ce qu'elles font, vous pouvez vous en souvenir comme suit :

- `hermes model` : choisir et changer de modèle
- `hermes tools` : voir et configurer les outils disponibles
- `hermes config set` : modifier des éléments de configuration spécifiques
- `hermes setup` : exécuter une fois l'assistant d'initialisation complet
- `hermes update` : mettre à jour Hermes
- `hermes doctor` : effectuer le dépannage

L'ordre le plus pratique pour les débutants est généralement :

1. Exécuter d'abord `hermes model`
2. Si vous souhaitez configurer complètement les éléments courants en une seule fois, exécutez ensuite `hermes setup`

## Les deux modes d'utilisation les plus courants d'Hermes

### 1. Utiliser Hermes comme assistant de développement quotidien dans le terminal

Le mode CLI convient à ces scénarios :

- Poser des questions directement lors de l'écriture de code local
- Consulter des projets, modifier des fichiers, exécuter des commandes
- Effectuer un dépannage ou une révision unique
- Collaborer en continu dans le répertoire de travail actuel

Son plus grand avantage est le chemin le plus court : pas besoin de connecter une plateforme supplémentaire, pas besoin de gérer d'abord la configuration du bot, et c'est le plus adapté pour établir les premières habitudes d'utilisation.

### 2. Utiliser Hermes via des plateformes de messagerie

Si vous souhaitez dialoguer avec Hermes sur des plateformes comme Feishu, Telegram, Discord, etc., vous devez utiliser la passerelle de messagerie.

Les commandes d'entrée les plus courantes sont :

```bash
hermes gateway setup
hermes gateway
```

Où :

- `hermes gateway setup` sert à faire la configuration interactive de la plateforme
- `hermes gateway` sert à lancer le processus de la passerelle

Selon la documentation officielle, la passerelle est un processus d'arrière-plan unifié, utilisé pour connecter les plateformes que vous avez configurées, gérer les sessions, et traiter les tâches planifiées, etc.

## Comment connecter Hermes à une plateforme de messagerie, avec Feishu comme exemple

Si votre travail quotidien se fait principalement dans Feishu, Feishu/Lark sera une méthode d'intégration très naturelle pour Hermes.

### La méthode d'intégration minimale viable

L'entrée recommandée par la documentation officielle pour Feishu/Lark est :

```bash
hermes gateway setup
```

Après l'exécution, sélectionnez **Feishu / Lark** dans l'assistant.

Les deux modes de connexion donnés par la documentation Feishu sont :

- `websocket` : recommandé
- `webhook` : optionnel

Si Hermes fonctionne sur votre ordinateur portable, votre station de travail ou un serveur privé, il est préférable d'utiliser prioritairement `websocket`, car cela évite d'avoir à exposer une adresse de rappel publique.

### Si vous configurez manuellement, vous devez au moins connaître ces variables

Si vous n'utilisez pas l'assistant pour la configuration, mais que vous écrivez la configuration manuellement, les variables principales répertoriées dans la documentation Feishu comprennent :

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Les deux éléments les plus notables sont :

- `FEISHU_ALLOWED_USERS` : recommandé, pour éviter que toute personne pouvant accéder au bot puisse l'utiliser directement
- `FEISHU_HOME_CHANNEL` : peut spécifier à l'avance un chat home pour recevoir les résultats de cron ou les notifications par défaut

### Pourquoi Hermes ne répond-il parfois pas dans les conversations de groupe Feishu

C'est un point facile à oublier : dans les conversations de groupe Feishu, Hermes ne répond pas par défaut à chaque message qu'il voit.

La documentation officielle précise clairement :

- En conversation privée, Hermes répondra aux messages
- Dans les conversations de groupe, il faut explicitement `@` le bot pour qu'il traite le message

Si vous souhaitez définir une conversation Feishu comme home channel, vous pouvez également utiliser dans le chat :

```text
/set-home
```

Ou l'écrire à l'avance dans la configuration :

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Les commandes Hermes les plus importantes à retenir pour les débutants

Que vous soyez dans le CLI ou sur une plateforme de messagerie, retenir ces commandes suffit pour commencer :

- `/new` ou `/reset` : commencer une nouvelle session
- `/model` : voir ou changer de modèle
- `/retry` : réessayer le tour précédent
- `/undo` : annuler l'interaction précédente
- `/compress` : compresser manuellement le contexte
- `/help` : voir l'aide

Si vous utilisez principalement sur une plateforme de messagerie, retenez en plus une :

- `/sethome` ou `/set-home` : définir le chat actuel comme home channel

Ces commandes couvrent les opérations les plus courantes au stade débutant : recommencer, ajuster, revenir, voir et continuer à utiliser.

## Questions fréquentes

### Peut-on installer Hermes directement sur Windows ?

Non. La documentation officielle précise actuellement que Windows natif n'est pas pris en charge, l'utilisation de WSL2 est recommandée.

### Que faire si après l'installation, taper `hermes` ne donne aucune réponse ?

Il est conseillé de suivre cet ordre pour le diagnostic :

1. Rechargez d'abord le shell, par exemple `source ~/.bashrc`
2. Exécutez à nouveau `hermes`
3. Si c'est toujours anormal, exécutez `hermes doctor`

### Pourquoi le bot ne répond-il pas dans les groupes Feishu ?

Vérifiez d'abord ces trois éléments :

- Avez-vous fait `@` Hermes dans le groupe
- `FEISHU_ALLOWED_USERS` a-t-il limité l'utilisateur actuel
- La stratégie de conversation de groupe actuelle permet-elle de traiter les messages de groupe

Selon la documentation officielle Feishu, dans les scénarios de conversation de groupe, la mention explicite @mention est une condition nécessaire.
