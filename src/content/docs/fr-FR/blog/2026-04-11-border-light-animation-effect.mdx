## Guide de mise en œuvre de l'effet d'animation de lumière de bordure

> Cet élément important qui capte immédiatement l'attention des utilisateurs, comment est-il réalisé avec du CSS pur ? Ce n'est pas vraiment difficile, c'est juste un peu détourné. Cet article vous guide de zéro pour réaliser une animation de lumière de bordure, et discute au passage des pièges que nous avons rencontrés dans le projet HagiCode.

## Contexte

Les développeurs frontend ont probablement tous vécu cette expérience : le chef de produit arrive, avec cette expression "ce besoin est simple" sur le visage — "cette tâche en cours d'exécution, pouvons-nous ajouter un effet pour que les utilisateurs puissent la voir d'un coup d'œil ?"

Vous dites OK, ajoutons un changement de couleur de bordure. Mais l'autre secoue la tête, avec un regard qui dit "tu ne comprends pas" : "pas assez visible, il faut cet effet où la lumière tourne autour de la bordure, comme dans les films de science-fiction."

À ce moment-là, vous pourriez vous demander : comment réaliser ça ? Avec Canvas ? Avec SVG ? Ou CSS peut le faire ? Après tout, personne ne veut admettre qu'il ne sait pas faire.

En fait, les animations de lumière de bordure sont particulièrement courantes dans les applications Web modernes, principalement utilisées dans ces scénarios :

- **Indication d'état** : marquer les tâches en cours ou les éléments actifs
- **Focalisation visuelle** : mettre en évidence les zones de contenu importantes
- **Renforcement de la marque** : créer une expérience visuelle high-tech et moderne
- **Thèmes festifs** : créer une ambiance festive pour les occasions spéciales

Lorsque nous développions [HagiCode](https://hagicode.com), nous avons rencontré ce besoin — les utilisateurs doivent pouvoir voir instantanément quelles sessions sont en cours d'exécution et quelles propositions sont en cours de traitement. Nous avons essayé plusieurs solutions, certaines plus faciles, d'autres un peu plus détournées, et nous avons finalement établi une approche de mise en œuvre assez mature.

## À propos de HagiCode

La solution partagée dans cet article provient de notre expérience pratique dans le projet [HagiCode](https://hagicode.com). HagiCode est un projet d'assistant de code piloté par l'IA, qui utilise largement les animations de lumière de bordure dans l'interface pour indiquer divers états d'exécution. Par exemple, l'état d'exécution de la liste des sessions, les transitions d'état du diagramme de flux des propositions, l'affichage de l'intensité de l'indicateur de débit, etc.

En fait, ces effets ne sont pas très complexes, mais nous avons trébuché pas mal de fois en les réalisant. Si vous voulez voir l'effet réel, vous pouvez visiter notre [dépôt GitHub](https://github.com/HagiCode-org/site) ou aller directement sur le [site officiel](https://hagicode.com) pour en savoir plus — après tout, ce qui fonctionne est le meilleur.

## Concepts de mise en œuvre fondamentaux

Après analyse du code de HagiCode, nous avons résumé les plusieurs modèles de mise en œuvre fondamentaux suivants, chacun ayant son scénario d'application, ou plutôt, chacun ayant sa raison d'être.

### 1. Halo rotatif Conic Gradient (le plus courant)

C'est la méthode la plus classique pour réaliser une lumière de bordure rotative. L'idée centrale est d'utiliser le `conic-gradient` de CSS pour créer un dégradé conique, puis le faire tourner. Comme un réverbère la nuit, qui tourne et tourne sans arrêt.

**Éléments clés :**
- Créer une couche de halo avec le pseudo-élément `::before`
- Définir la distribution des couleurs de dégradé avec `conic-gradient`
- Masquer la zone centrale avec le pseudo-élément `::after` (optionnel)
- Réaliser l'animation de rotation avec `@keyframes`

### 2. Ligne lumineuse latérale

Celui-ci convient pour l'indication d'état des éléments de liste, il suffit de créer une ligne lumineuse fine sur un côté de l'élément, sans que toute la bordure bouge. Parfois, un peu de lumière suffit, pas besoin d'illuminer le monde entier.

**Éléments clés :**
- Élément de ligne fine en position absolue
- Créer un effet de lueur avec `box-shadow`
- Réaliser une animation de respiration avec `scale` et `opacity`

### 3. Arrière-plan lumineux Box-Shadow

Si vous n'avez pas besoin d'un effet rotatif et voulez juste un halo d'arrière-plan doux, superposer plusieurs `box-shadow` suffit. Parfois, plus c'est simple, mieux c'est.

### 4. Support d'accessibilité

C'est facile à oublier, mais particulièrement important. Toutes les animations devraient prendre en compte la requête média `prefers-reduced-motion`, offrant une alternative statique aux utilisateurs qui n'aiment pas les animations. Après tout, tout le monde n'aime pas les choses qui bougent, respecter le choix de chacun est la bonne chose à faire.

## Solutions de mise en œuvre

### Solution 1 : Bordure rotative Conic Gradient (recommandée)

C'est la réalisation la plus complète de l'effet de lumière rotative, et aussi la solution la plus utilisée dans HagiCode. Après tout, si quelque chose fonctionne bien, pourquoi changer ?

```css
/* Conteneur parent */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Couche de halo rotatif */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Couche de masque (optionnel, pour créer un effet de bordure creuse) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Le principe de cette solution est assez simple : créer un pseudo-élément plus grand que le conteneur parent, dessiner un dégradé conique dessus, puis le faire tourner sans arrêt. Le conteneur parent définit `overflow: hidden`, donc on ne voit que la lumière de la partie bordure qui tourne. Comme quand on regarde un réverbère à travers une fenêtre, on ne voit que ce petit passage.

### Solution 2 : Bordure lumineuse rotative simplifiée

Si vous n'avez pas besoin d'un effet si complexe, HagiCode a une implémentation de classe utilitaire plus légère. Parfois, plus simple, c'est mieux.

```css
/* Classe utilitaire de bordure lumineuse rotative */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Support d'accessibilité */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Notez le `will-change: transform` ici, qui dit au navigateur "cet élément va toujours changer", le navigateur fera donc des optimisations à l'avance, l'animation sera plus fluide. Après tout, se préparer à l'avance vaut mieux que de s'y prendre au dernier moment.

### Solution 3 : Ligne lumineuse latérale

C'est particulièrement approprié pour l'indication d'état des éléments de liste, la liste des sessions de HagiCode utilise cette solution. Une ligne fine, mais capable de se démarquer parmi de nombreux éléments, n'est-ce pas aussi une philosophie de vie ?

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Ici on utilise `isolation: isolate` pour créer un nouveau contexte d'empilement, puis `z-index` pour contrôler l'ordre d'affichage des couches. `pointer-events: none` est aussi crucial, sinon le pseudo-élément bloquerait les clics de l'utilisateur. Comme certaines choses, c'est beau, mais ça ne doit pas gêner.

### Solution 4 : Encapsulation de composant React

Si votre projet utilise React, vous pouvez encapsuler un composant pour gérer cette logique, en particulier la partie accessibilité. Après tout, écrire le code une fois et l'utiliser plusieurs fois, c'est ce que nous voulons.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Le module CSS correspondant :

```css
/* GlowBorder.module.css */

/* Version animée */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Version statique (accessibilité) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Le hook `useReducedMotion` de `framer-motion` détecte automatiquement les préférences système de l'utilisateur. Si l'utilisateur a activé "réduire les effets dynamiques", il renverra true, et on affichera la version statique. Après tout, respecter le choix de l'utilisateur est plus important que d'imposer un affichage.

## Partage d'expériences pratiques

Voici les expériences que nous avons résumées en trébuchant lors du développement de HagiCode. Ce sont juste des petites réflexions, j'espère qu'elles pourront vous aider.

### 1. Système de variables de thème

Utiliser des variables CSS pour le support multi-thème est particulièrement pratique. Après tout, personne ne veut modifier un tas de code à chaque changement de thème.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Utilisation */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

Ainsi, lors du changement de thème, il suffit de modifier la classe de la balise `html`, toutes les couleurs d'animation se mettront à jour automatiquement. Un code, deux styles, n'est-ce pas ce que nous poursuivons ?

### 2. Optimisation des performances

**Utiliser `will-change` pour suggérer une optimisation au navigateur :**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Dites-le au navigateur à l'avance, il fera des optimisations pour vous. Comme beaucoup de choses dans la vie, se préparer à l'avance est toujours une bonne chose.

**Évitez d'utiliser des box-shadow complexes sur de grands éléments :**

```css
/* Pas bon - ombre floue sur un grand élément */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Mieux - utiliser un pseudo-élément pour limiter la zone lumineuse */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Nous avons testé dans HagiCode, ajouter directement des ombres floues sur de grandes cartes fait chuter le taux de rafraîchissement du défilement sous 30fps, après avoir utilisé des pseudo-éléments, on tient stable à 60fps. Les utilisateurs peuvent ressentir cette différence d'expérience.

### 3. Accessibilité

Ça ne doit vraiment pas être omis, certains utilisateurs trouveront les animations étourdissantes ou bruyantes, respecter leur choix est une compétence de base en conception de produits. Après tout, les belles choses ne doivent pas être imposées.

**Requête média CSS :**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Fournir une alternative statique */
    opacity: 1;
  }
}
```

**Détection des préférences utilisateur dans React :**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Contrôle du niveau d'intensité

L'indicateur de débit de Token dans HagiCode affiche différentes couleurs de lumière selon le débit en temps réel, c'est réalisé dynamiquement. Après tout, différents états devraient avoir différentes expressions.

```tsx
const colors = [
  null,       // Level 0 - pas de couleur
  '#3b82f6',  // Level 1 - Bleu
  '#34d399',  // Level 2 - Émeraude
  '#facc15',  // Level 3 - Jaune
  '#fbbf24',  // Level 4 - Ambre
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Précautions

Certains détails doivent être notés, sinon ce sera trop tard quand vous trébucherez.

| Précaution | Description |
|---------|------|
| **Gestion du z-index** | La couche de halo doit avoir un z-index approprié pour éviter d'affecter l'interaction du contenu |
| **pointer-events** | Les pseudo-éléments de halo doivent définir `pointer-events: none` |
| **Débordement de bordure** | Le conteneur parent doit définir `overflow: hidden` ou ajuster la taille du pseudo-élément |
| **Impact sur les performances** | Les animations complexes peuvent affecter les performances sur les appareils mobiles, il faut tester |
| **Mode sombre** | Assurez-vous que les couleurs lumineuses sont clairement visibles sur fond sombre |
| **Changement de thème** | Utilisez des variables CSS pour assurer la mise à jour correcte des couleurs d'animation lors du changement de thème |

### 6. Techniques de débogage

Les pseudo-éléments sont parfois difficiles à trouver dans les outils de développement, vous pouvez temporairement ajouter une bordure pour voir la position.

```css
/* Afficher temporairement les limites du pseudo-élément pour le débogage */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Après avoir ajusté la position, n'oubliez pas de commenter ou supprimer cette ligne, sinon ce sera gênant en production. Certaines choses devraient rester dans l'environnement de développement.
