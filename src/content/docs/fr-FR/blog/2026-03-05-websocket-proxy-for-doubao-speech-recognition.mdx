## Proxy WebSocket pour la reconnaissance vocale Doubao : résoudre les problèmes d'accèsinter-domaines

## Contexte

L'API de reconnaissance vocale de Doubao utilise le protocole WebSocket, ce qui pose un défi pour les applications web front-end en raison des restrictions de politique de même origine (CORS). Le navigateur bloque les connexions WebSocket vers des domaines tiers, rendant impossible l'utilisation directe de l'API depuis le front-end.

Pour résoudre ce problème, nous avons construit un proxy WebSocket dans le backend HagiCode qui relaie les flux audio entre le client et l'API Doubao.

## À propos de HagiCode

L'approche partagée dans cet article provient de notre expérience pratique dans le projet [HagiCode](https://hagicode.com).

## Architecture du proxy

Le proxy WebSocket fonctionne comme un intermédiaire transparent : le client se connecte au proxy local, qui se connecte ensuite à l'API Doubao et relaie les données dans les deux sens.

## Conclusion

Le proxy WebSocket est une solution élégante pour les problèmes d'accèsinter-domaines, permettant une intégration fluide des API WebSocket tierces dans les applications web.
