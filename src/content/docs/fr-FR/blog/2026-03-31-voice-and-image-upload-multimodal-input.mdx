## Mieux vaut parler que taper, mieux vaut capturer que parler — Pratique d'entrée multimodale pour l'assistant de code IA

> En fait, pour écrire du code, il y a une limite à la vitesse de frappe. Parfois, une phrase suffit, mais il faut taper pendant longtemps; parfois, une image peut expliquer clairement, mais il faut utiliser une pile de texte pour le décrire. Cet article discute des problèmes que nous avons rencontrés lors du développement de [HagiCode](https://hagicode.com) — reconnaissance vocale, téléchargement d'images, bref, nous voulons simplement rendre l'assistant de code IA un peu plus facile à utiliser, c'est tout.

## Contexte

Lors du développement de HagiCode, nous avons découvert un problème — ou plutôt, un problème qui est naturellement apparu lorsque les utilisateurs l'ont utilisé davantage : s'appuyer uniquement sur la frappe peut être parfois assez fatigant.

Réfléchissez : l'interaction entre les utilisateurs et l'Agent est un scénario central. Mais devoir s'asseoir devant le clavier à chaque fois pour taper frénétiquement, comment dire, l'efficacité n'est pas vraiment élevée :

1. **La frappe est trop lente** : Pour certains problèmes complexes, comme les erreurs, les problèmes d'interface, etc., cela prend une demi-minute à taper, mais peut-être dix secondes à l'oral. Cet écart de temps est assez gênant.

2. **Les images sont plus directes** : Parfois, l'interface signale une erreur, ou on veut comparer une maquette, ou montrer la structure du code... "Une image vaut mille mots", bien que ce dicton soit vieux, il reste vrai. Permettre à l'IA de "voir" directement le problème est beaucoup plus clair que de le décrire pendant longtemps.

3. **L'interaction devrait être plus naturelle** : Les assistants IA actuels devraient supporter le texte, la voix, les images, non ? Les utilisateurs devraient pouvoir utiliser ce qu'ils veulent, c'est ça qui est naturel, non ?

Donc, nous avons pensé qu'il serait mieux d'ajouter des fonctionnalités de reconnaissance vocale et de téléchargement d'images à HagiCode, pour rendre les opérations de l'Agent plus pratiques. Après tout, permettre aux utilisateurs de taper moins de mots est une bonne chose.

## À propos de HagiCode

Les solutions partagées dans cet article proviennent de notre pratique dans le projet [HagiCode](https://hagicode.com) — ou plutôt, de l'expérience acquise en trébuchant constamment.

HagiCode est un projet d'assistant de code IA open source, l'idée est simple : utiliser la technologie IA pour améliorer l'efficacité du développement. En développant, nous avons découvert que la demande des utilisateurs pour l'entrée multimodale est en fait assez forte — parfois dire une phrase est plus rapide que taper une pile de mots, parfois une capture d'écran est plus claire que de décrire pendant longtemps.

Ces demandes nous ont poussés à avancer, et finalement nous avons eu ces fonctionnalités de reconnaissance vocale et de téléchargement d'images. Les utilisateurs peuvent interagir avec l'IA de la manière la plus naturelle, c'est une bonne sensation.

## Analyse

### Défis techniques de la reconnaissance vocale

Lors du développement de la fonctionnalité de reconnaissance vocale, nous avons rencontré un problème assez épineux : **l'API WebSocket du navigateur ne supporte pas les en-têtes HTTP personnalisés**.

Et le service de reconnaissance vocale que nous avons choisi est l'API de reconnaissance vocale Doubao de ByteDance. Cette API exige偏偏 de transmettre les informations d'authentification via l'en-tête HTTP, comme `accessToken`, `secretKey`, etc. Voilà, le conflit technique est arrivé :

```javascript
// L'API WebSocket du navigateur ne supporte pas la méthode suivante
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // Non supporté
  }
});
```

Les solutions devant nous sont généralement au nombre de deux :

1. **Solution des paramètres de requête URL** : Mettre les informations d'authentification dans l'URL
   - L'avantage est que la mise en œuvre est simple
   - L'inconvénient est que les informations d'identification sont exposées sur le frontend, la sécurité est médiocre ; et certaines API exigent une vérification par en-tête

2. **Solution de proxy backend** : Implémenter un proxy WebSocket dans le backend
   - L'avantage est que les informations d'identification sont stockées en toute sécurité dans le backend ; entièrement compatible avec les exigences de l'API
   - L'inconvénient est que la mise en œuvre est légèrement plus complexe

Finalement, nous avons choisi la **solution de proxy backend**. Après tout, la sécurité est une ligne de fond qui ne peut être compromise — à ce sujet, personne ne peut tromper.

### Besoins fonctionnels du téléchargement d'images

Pour la fonctionnalité de téléchargement d'images, nos besoins sont en fait assez simples :

1. **Méthodes de téléchargement multiples** : Clic pour sélectionner le fichier, glisser-déposer, collage du presse-papiers, il faut les avoir, non ?
2. **Validation des fichiers** : Limitations de type (PNG, JPG, WebP, GIF), limitations de taille (5-10 Mo), ce sont des opérations de base
3. **Expérience utilisateur** : Progression du téléchargement, prévisualisation, messages d'erreur, il faut laisser les gens savoir ce qui se passe
4. **Sécurité** : Validation côté serveur, prévention du téléchargement de fichiers malveillants, c'est une affaire importante

## Solutions

### Reconnaissance vocale : Architecture de proxy WebSocket

Nous avons conçu une solution de reconnaissance vocale à trois couches, comment dire, nous avons trouvé une voie :

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**Implémentation des composants principaux** :

1. **Processeur AudioWorklet frontend** :

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Rééchantillonner à 16kHz (exigence de l'API Doubao)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Accumuler les échantillons en blocs de 500 ms
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Convertir en PCM 16-bit et envoyer
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **Gestionnaire WebSocket backend** (C#) :

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **Composant VoiceTextArea frontend** :


### Téléchargement d'images : Composant de téléchargement multi-méthodes

Nous avons créé un composant de téléchargement d'images complet en fonctionnalités, supportant les trois méthodes de téléchargement, comment dire, nous avons couvert les scénarios courants des utilisateurs.

**Fonctionnalités principales** :

1. **Trois méthodes de téléchargement** :

```tsx
// Téléchargement par clic
const handleClick = () => fileInputRef.current?.click();

// Téléchargement par glisser-déposer
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Collage du presse-papiers
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **Validation frontend** :

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **Traitement du téléchargement backend** (TypeScript) :

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Validation
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Sauvegarder le fichier
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## Guide de pratique

### Comment utiliser la reconnaissance vocale

1. **Configurer le service de reconnaissance vocale** :
   - Aller à la page des paramètres de reconnaissance vocale
   - Configurer le `AppId` et l'`AccessToken` de Doubao Voice
   - (Facultatif) Configurer des mots clés pour améliorer la précision de reconnaissance des termes techniques

2. **Utiliser dans la zone de saisie** :
   - Cliquer sur l'icône du microphone à gauche de la zone de saisie
   - Commencer à parler après avoir vu l'animation de forme d'onde
   - Cliquer à nouveau sur l'icône pour arrêter l'enregistrement
   - Le résultat de la reconnaissance sera automatiquement inséré à la position du curseur

3. **Exemple de configuration des mots clés** :

```text
TypeScript
React
useState
useEffect
```

### Comment utiliser le téléchargement d'images

1. **Méthodes de téléchargement** :
   - Cliquer sur le bouton de téléchargement pour sélectionner le fichier
   - Faire glisser directement l'image vers la zone de téléchargement
   - Utiliser `Ctrl+V` pour coller la capture d'écran du presse-papiers

2. **Formats supportés** : PNG, JPG, JPEG, WebP, GIF
3. **Limite de taille** : 5 Mo par défaut (configurable)

### Précautions

1. **Reconnaissance vocale** :
   - Nécessite l'autorisation du microphone
   - Il est recommandé de l'utiliser dans un environnement calme
   - La durée maximale d'enregistrement supportée est de 300 secondes (configurable)

2. **Téléchargement d'images** :
   - Supporte uniquement les formats d'image courants
   - Faites attention à la limite de taille du fichier
   - Les images téléchargées généreront automatiquement une URL de prévisualisation

3. **Considérations de sécurité** :
   - Les informations d'identification de reconnaissance vocale sont stockées dans le backend
   - Le téléchargement d'images a une validation côté serveur stricte
   - En production, il est recommandé d'utiliser HTTPS/WSS

## Résumé

Après avoir ajouté la reconnaissance vocale et le téléchargement d'images, l'expérience utilisateur de HagiCode s'est considérablement améliorée. Les utilisateurs peuvent maintenant interagir avec l'IA de manière plus naturelle — parler au lieu de taper, capturer des écrans au lieu de décrire. Cette sensation, comment dire, c'est comme avoir enfin trouvé une façon plus confortable de communiquer.

Lors du développement de cette fonctionnalité, nous avons rencontré le problème que le WebSocket du navigateur ne supporte pas les en-têtes personnalisés, et finalement nous l'avons résolu par la solution de proxy backend. Cette solution garantit non seulement la sécurité, mais pose également les bases pour l'intégration future d'autres services WebSocket nécessitant une authentification — c'est aussi un gain inattendu.

Le composant de téléchargement d'images est pareil, en utilisant plusieurs méthodes de téléchargement, permettant aux utilisateurs de choisir la plus pratique selon le scénario. Clic, glisser-déposer, ou collage direct, tous permettent de terminer le téléchargement rapidement. Tous les chemins mènent à Rome, c'est juste que certains chemins sont plus faciles à parcourir, d'autres un peu plus sinueux.

"Mieux vaut parler que taper, mieux vaut capturer que parler", cette phrase est assez appropriée ici. Si vous développez également des produits d'assistant IA similaires, j'espère que ces expériences pourront vous aider, même un tout petit peu.
