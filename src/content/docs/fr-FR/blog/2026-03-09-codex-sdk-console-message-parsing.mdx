## Mécanisme de flux d'événements du SDK Codex

### Format de sortie

Codex CLI produit des événements au format JSONL (JSON Lines), un objet JSON par ligne. Chaque événement contient au minimum deux champs : `type` et `data`.

```json
{"type":"start","data":{"session_id":"abc123","timestamp":"2026-03-09T10:00:00Z"}}
{"type":"progress","data":{"step":1,"total":5,"description":"Analyzing code"}}
{"type":"output","data":{"content":"console.log('hello')","role":"assistant"}}
{"type":"end","data":{"status":"completed","duration_ms":5000}}
```

### Types d'événements principaux

Le SDK Codex définit plusieurs types d'événements standard :

| Type | Description | Données clés |
|------|-------------|--------------|
| `start` | Début de l'exécution | `session_id`, `timestamp` |
| `progress` | Progression de l'exécution | `step`, `total`, `description` |
| `output` | Sortie du modèle | `content`, `role` |
| `tool_use` | Appel d'outil | `tool_name`, `tool_input` |
| `tool_result` | Résultat d'outil | `tool_name`, `tool_output` |
| `error` | Erreur d'exécution | `code`, `message` |
| `end` | Fin de l'exécution | `status`, `duration_ms` |
| `token_usage` | Utilisation des tokens | `prompt_tokens`, `completion_tokens` |

## Implémentation de l'analyseur

### Architecture de base

L'analyseur de messages doit être capable de :

1. Lire le flux JSONL ligne par ligne
2. Analyser chaque ligne en objet événement
3. Distribuer les événements aux gestionnaires correspondants selon le type
4. Gérer les erreurs de parsing de manière gracieuse

### Exemple d'implémentation en TypeScript

```typescript
import { EventEmitter } from 'events';

interface CodexEvent {
  type: string;
  data: Record<string, unknown>;
}

class CodexEventParser extends EventEmitter {
  parse(line: string): CodexEvent | null {
    try {
      const event = JSON.parse(line) as CodexEvent;
      this.emit(event.type, event.data);
      this.emit('*', event); // Wildcard for all events
      return event;
    } catch (error) {
      this.emit('parse_error', { line, error });
      return null;
    }
  }

  parseStream(stream: ReadableStream): void {
    const reader = stream.getReader();
    const decoder = new TextDecoder();
    let buffer = '';

    const processChunk = async ({ done, value }: ReadableStreamReadResult<Uint8Array>) => {
      if (done) {
        if (buffer.trim()) this.parse(buffer);
        return;
      }
      
      buffer += decoder.decode(value, { stream: true });
      const lines = buffer.split('\n');
      buffer = lines.pop() || '';
      
      for (const line of lines) {
        if (line.trim()) this.parse(line);
      }
      
      return reader.read().then(processChunk);
    };

    reader.read().then(processChunk);
  }
}
```

### Exemple d'implémentation en C#

```csharp
public class CodexEventParser
{
    public event Action<string, JsonElement>? OnEvent;
    public event Action<string, Exception>? OnParseError;

    public async Task ParseStreamAsync(Stream stream, CancellationToken ct = default)
    {
        using var reader = new StreamReader(stream);
        while (!reader.EndOfStream)
        {
            ct.ThrowIfCancellationRequested();
            var line = await reader.ReadLineAsync(ct);
            if (line == null) break;
            if (string.IsNullOrWhiteSpace(line)) continue;

            try
            {
                var json = JsonDocument.Parse(line);
                var type = json.RootElement.GetProperty("type").GetString() ?? "";
                var data = json.RootElement.GetProperty("data");
                OnEvent?.Invoke(type, data);
            }
            catch (Exception ex)
            {
                OnParseError?.Invoke(line, ex);
            }
        }
    }
}
```

## Bonnes pratiques

### 1. Gestion robuste des erreurs

Ne laissez jamais une seule ligne invalide arrêter tout le flux. Enregistrez l'erreur et continuez.

### 2. Support de l'annulation

Utilisez `CancellationToken` en C# ou `AbortController` en TypeScript pour permettre l'annulation des longues opérations.

### 3. Journalisation structurée

Enregistrez chaque événement avec un horodatage et un identifiant de session pour faciliter le débogage.

### 4. Bufferisation pour les événements de progression

Les événements de progression peuvent arriver très rapidement. Mettez-les en tampon et mettez à jour l'interface utilisateur à une fréquence raisonnable (par exemple, 10 fois par seconde).
