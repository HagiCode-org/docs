## Comment étudier des projets dans l'ère de l'IA : Système de stockage persistant multi-projets Vault

> Dans l'ère du développement assisté par l'IA, comment permettre à l'assistant IA de mieux comprendre nos ressources d'apprentissage ? Le projet HagiCode réalise une couche d'abstraction de stockage de connaissances unifiée et compréhensible par l'IA à travers le système Vault, améliorant considérablement l'efficacité d'apprentissage des projets d'étude.

## Contexte

Dans l'ère de l'IA, la manière dont les développeurs apprennent de nouvelles technologies et architectures est en train de changer profondément. "Étudier des projets" - c'est-à-dire étudier en profondeur le code, l'architecture et les modèles de conception de projets open source excellents - est devenu une méthode d'apprentissage efficace. Par rapport à la lecture traditionnelle de livres et au visionnage de vidéos, lire et exécuter directement des projets open source de haute qualité vous permet de comprendre plus rapidement les pratiques d'ingénierie du monde réel.

Cependant, cette méthode d'apprentissage fait également face à de nombreux défis.

Les ressources d'apprentissage sont trop dispersées. Vos notes peuvent être dans Obsidian, les dépôts de code sont dispersés dans différents dossiers, et l'historique de conversation de l'assistant IA est encore une île de données isolée. Lorsque vous voulez que l'IA vous aide à analyser un projet, vous devez copier manuellement des extraits de code et organiser le contexte, un processus assez fastidieux.

Ce qui est encore plus gênant, c'est la rupture de contexte. L'assistant IA ne peut pas accéder directement à vos ressources d'apprentissage locales, et chaque conversation nécessite de fournir à nouveau des informations de contexte. De plus, les dépôts de code étudiés sont mis à jour rapidement, la synchronisation manuelle est sujette à erreurs, et il est difficile de partager des connaissances entre plusieurs projets d'apprentissage.

Ces problèmes sont essentiellement causés par des "îlots de données". S'il existait une couche d'abstraction de stockage unifiée permettant à l'assistant IA de comprendre et d'accéder à toutes vos ressources d'apprentissage, le problème serait résolu.

## À propos de HagiCode

Le système Vault partagé dans cet article est précisément la solution que nous avons développée au cours du développement de [HagiCode](https://hagicode.com). HagiCode est un projet d'assistant de code IA. Dans notre développement quotidien, nous avons souvent besoin d'apprendre et de nous référer à divers projets open source. Pour permettre à l'assistant IA de mieux comprendre ces ressources d'apprentissage, nous avons conçu le système de stockage persistant multi-projets Vault.

Cette solution a été vérifiée en pratique dans HagiCode. Si vous êtes confronté à des problèmes similaires de gestion des connaissances, j'espère que cette expérience pourra vous inspirer. Après tout, certains obstacles une fois franchis, il faut bien laisser quelque chose aux successors.

## Philosophie de conception du système Vault

L'idée centrale du système Vault est très simple : créer une couche d'abstraction de stockage de connaissances unifiée et compréhensible par l'IA. Du point de vue de l'implémentation, le système possède plusieurs caractéristiques clés.

### Support multi-types

Le système prend en charge quatre types de vault, correspondant respectivement à différents scénarios d'utilisation :

```typescript
// folder : type de dossier général
export const DEFAULT_VAULT_TYPE = 'folder';

// coderef : type dédié à l'étude de projets de code
export const CODEREF_VAULT_TYPE = 'coderef';

// obsidian : intégration avec le logiciel de notes Obsidian
export const OBSIDIAN_VAULT_TYPE = 'obsidian';

// system-managed : vault géré automatiquement par le système
export const SYSTEM_MANAGED_VAULT_TYPE = 'system-managed';
```

Parmi eux, le type **coderef** est le plus couramment utilisé dans HagiCode. Il est spécialement conçu pour étudier des projets de code, fournissant une structure de répertoires standardisée et des descriptions de métadonnées lisibles par l'IA.

### Mécanisme de stockage persistant

Le registre des Vaults est stocké de manière persistante au format JSON, garantissant que la configuration reste disponible après le redémarrage de l'application :

```csharp
public class VaultRegistryStore : IVaultRegistryStore
{
    private readonly string _registryFilePath;

    public VaultRegistryStore(IConfiguration configuration, ILogger<VaultRegistryStore> logger)
    {
        var dataDir = configuration["DataDir"] ?? "./data";
        var absoluteDataDir = Path.IsPathRooted(dataDir)
            ? dataDir
            : Path.GetFullPath(Path.Combine(Directory.GetCurrentDirectory(), dataDir));

        _registryFilePath = Path.Combine(absoluteDataDir, "personal-data", "vaults", "registry.json");
    }
}
```

L'avantage de cette conception est sa simplicité et sa fiabilité. Le format JSON est lisible par l'homme, facilitant le débogage et la modification manuelle ; le stockage sur système de fichiers évite la complexité d'une base de données et réduit les dépendances du système. Après tout, parfois plus c'est simple, mieux c'est.

### Intégration du contexte IA

Le plus important, c'est que le système peut automatiquement injecter les informations du vault dans le contexte des propositions de l'IA :

```typescript
export function buildTargetVaultsText(
  vaults: VaultForText[],
  template: VaultPromptTemplate = DEFAULT_VAULT_PROMPT_TEMPLATE,
): string {
  const readOnlyVaults = vaults.filter((vault) => vault.accessType === 'read');
  const editableVaults = vaults.filter((vault) => vault.accessType === 'write');

  if (readOnlyVaults.length === 0 && editableVaults.length === 0) {
    return '';
  }

  const sections = [
    buildVaultSection(readOnlyVaults, template.reference),
    buildVaultSection(editableVaults, template.editable),
  ].filter(Boolean);

  return `\n\n### ${template.heading}\n\n${sections.join('\n')}`;
}
```

Cela réalise une fonctionnalité importante : l'assistant IA peut automatiquement comprendre les ressources d'apprentissage disponibles, sans que l'utilisateur n'ait à fournir manuellement le contexte. C'est une forme de compréhension mutuelle.

## Structure standardisée du CodeRef Vault

Pour les vaults de type coderef, HagiCode fournit une structure de répertoires standardisée :

```
my-coderef-vault/
├── index.yaml          # description des métadonnées du vault
├── AGENTS.md           # guide de fonctionnement de l'assistant IA
├── docs/               # stocke les notes d'apprentissage et la documentation
└── repos/              # gère les dépôts de code étudiés via les sous-modules Git
```

Lors de la création d'un vault, le système initialise automatiquement cette structure :

```csharp
private async Task EnsureCodeRefStructureAsync(
    string vaultName,
    string physicalPath,
    ICollection<VaultBootstrapDiagnosticDto> diagnostics,
    CancellationToken cancellationToken)
{
    Directory.CreateDirectory(physicalPath);

    var indexPath = Path.Combine(physicalPath, CodeRefIndexFileName);
    var docsPath = Path.Combine(physicalPath, CodeRefDocsDirectoryName);
    var reposPath = Path.Combine(physicalPath, CodeRefReposDirectoryName);

    // Créer la structure de répertoires standard
    if (!Directory.Exists(docsPath))
    {
        Directory.CreateDirectory(docsPath);
    }

    if (!Directory.Exists(reposPath))
    {
        Directory.CreateDirectory(reposPath);
    }

    // Créer le guide AGENTS.md
    await EnsureCodeRefAgentsDocumentAsync(physicalPath, cancellationToken);

    // Créer les métadonnées index.yaml
    await WriteCodeRefIndexDocumentAsync(indexPath, mergedDocument, cancellationToken);
}
```

La conception de cette structure est également réfléchie :

- Le répertoire **docs/** stocke vos notes d'apprentissage, vous pouvez enregistrer votre compréhension du code, l'analyse de l'architecture, l'expérience des obstacles rencontrés, etc. au format Markdown
- Le répertoire **repos/** gère les dépôts étudiés via des sous-modules Git, plutôt que de copier directement le code. Cela permet à la fois de maintenir la synchronisation du code et d'économiser de l'espace
- **index.yaml** contient les métadonnées du vault, permettant à l'assistant IA de comprendre rapidement l'objectif et le contenu de ce vault
- **AGENTS.md** est un guide spécialement écrit pour l'assistant IA, expliquant comment traiter le contenu de ce vault

Peut-être qu'en organisant ainsi, l'IA peut aussi plus facilement comprendre vos idées.

## Initialisation automatique gérée par le système

En plus de créer manuellement des vaults, HagiCode prend également en charge les vaults gérés automatiquement par le système :

```csharp
public async Task<IReadOnlyList<VaultRegistryEntry>> EnsureAllSystemManagedVaultsAsync(
    CancellationToken cancellationToken = default)
{
    var definitions = GetAllResolvedDefinitions();
    var entries = new List<VaultRegistryEntry>(definitions.Count);

    foreach (var definition in definitions)
    {
        entries.Add(await EnsureResolvedSystemManagedVaultAsync(definition, cancellationToken));
    }

    return entries;
}
```

Le système crée et gère automatiquement les vaults suivants :

- **hagiprojectdata** : stockage des données du projet, utilisé pour enregistrer la configuration et l'état du projet
- **personaldata** : stockage des données personnelles, utilisé pour enregistrer les préférences de l'utilisateur
- **hbsprompt** : bibliothèque de modèles de prompts, utilisée pour gérer les prompts IA courants

Ces vaults sont automatiquement initialisés au démarrage du système, sans que l'utilisateur n'ait à les configurer manuellement. Après tout, certaines choses sont mieux laissées au système, pourquoi les humains s'en soucieraient-ils ?

## Mécanisme de contrôle d'accès

Une conception importante est le contrôle d'accès. Le système divise les vaults en deux types d'accès :

```typescript
export interface VaultForText {
  id: string;
  name: string;
  type: string;
  physicalPath: string;
  accessType: 'read' | 'write';  // Clé : distingue lecture seule et modifiable
}
```

- **reference (lecture seule)** : l'IA ne l'utilise que pour l'analyse et la compréhension, ne peut pas modifier le contenu. Convient aux projets open source de référence, à la documentation, etc.
- **editable (modifiable)** : l'IA peut modifier le contenu selon les besoins de la tâche. Convient à vos notes, brouillons, etc.

Cette distinction est importante. Elle permet à l'IA de savoir quel contenu est "référence en lecture seule" et quel contenu est "peut être modifié", évitant ainsi les risques d'opération erronée. Après tout, personne ne veut voir son travail accidentellement effacé.

## Pratique : Créer et utiliser Vault

Après avoir vu les principes, voyons comment l'utiliser en pratique.

### Créer un CodeRef Vault

Voici un exemple complet d'appel frontend :

```typescript
const createCodeRefVault = async () => {
  const response = await VaultService.postApiVaults({
    requestBody: {
      name: "React Learning Vault",
      type: "coderef",
      physicalPath: "/Users/developer/vaults/react-learning",
      gitUrl: "https://github.com/facebook/react.git"
    }
  });

  // Le système automatiquement :
  // 1. Clone le dépôt React dans vault/repos/react
  // 2. Crée le répertoire docs/ pour les notes
  // 3. Génère les métadonnées index.yaml
  // 4. Crée le fichier de guide AGENTS.md

  return response;
};
```

Cet appel API effectue une série d'opérations : création de la structure des répertoires, initialisation des sous-modules Git, génération des fichiers de métadonnées, etc. Vous n'avez qu'à fournir les informations de base, le système s'occupe du reste. C'est plutôt rassurant.

### Utiliser Vault dans les propositions IA

Une fois le vault créé, vous pouvez le référencer dans les propositions de l'IA :

```typescript
const proposal = composeProposalChiefComplaint({

```

Le système injecte automatiquement les informations du vault dans le contexte de l'IA, permettant à l'IA de savoir quelles ressources d'apprentissage sont disponibles. L'IA comprend vos idées, c'est aussi une forme de compréhension mutuelle précieuse.

## Meilleures pratiques et précautions

Au cours de l'utilisation du système Vault, nous avons résumé certaines leçons d'expérience.

### Sécurité des chemins

Le système vérifie strictement les chemins pour prévenir les attaques de traversal de chemin :

```csharp
private static string ResolveFilePath(string vaultRoot, string relativePath)
{
    var rootPath = EnsureTrailingSeparator(Path.GetFullPath(vaultRoot));
    var combinedPath = Path.GetFullPath(Path.Combine(rootPath, relativePath));
    if (!combinedPath.StartsWith(rootPath, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode,
            "Vault file paths must stay inside the registered vault root.");
    }
    return combinedPath;
}
```

C'est important. Si vous personnalisez le chemin du vault, assurez-vous qu'il se trouve dans la plage autorisée, sinon le système refusera l'opération. La sécurité, on ne peut jamais trop insister là-dessus.

### Gestion des sous-modules Git

CodeRef vault recommande d'utiliser des sous-modules Git plutôt que de copier directement le code :

```csharp
private static string BuildCodeRefAgentsContent()
{
    return """
    # CodeRef Vault Guide

    Repositories under `repos/` should be maintained through Git submodules
    rather than copied directly into the vault root.

    Keep this structure stable so assistants and tools can understand the vault quickly.
    """ + Environment.NewLine;
}
```

Cela présente plusieurs avantages : maintenir le code synchronisé avec l'amont, économiser l'espace disque, faciliter la gestion de plusieurs versions de code. Après tout, qui voudrait télécharger encore et encore les mêmes choses ?

### Limites de prévisualisation des fichiers

Pour prévenir les problèmes de performance, le système limite la taille et le type des fichiers :

```csharp
private const int FileEnumerationLimit = 500;
private const int PreviewByteLimit = 256 * 1024;  // 256KB
```

Si votre vault contient un grand nombre de fichiers ou des fichiers très volumineux, cela peut affecter les performances de la fonction de prévisualisation. Dans ce cas, vous pouvez envisager un traitement par lots ou utiliser des outils de recherche spécialisés. Après tout, certaines choses trop volumineuses sont plus difficiles à traiter.

### Informations de diagnostic

La création d'un vault renvoie des informations de diagnostic pour aider au débogage :

```csharp
List<VaultBootstrapDiagnosticDto> bootstrapDiagnostics = [];

if (IsCodeRefVaultType(normalizedType))
{
    bootstrapDiagnostics = await EnsureCodeRefBootstrapAsync(
        normalizedName,
        normalizedPhysicalPath,
        normalizedGitUrl,
        cancellationToken);
}
```

Si la création échoue, vous pouvez consulter les informations de diagnostic pour comprendre la cause spécifique. En cas d'erreur, regardez les informations de diagnostic, c'est aussi une méthode de résolution de problèmes.

## Résumé

Le système Vault résout les points de douleur centraux de l'étude de projets dans l'ère de l'IA grâce à une couche d'abstraction de stockage unifiée :

- **Gestion centralisée des connaissances** : toutes les ressources d'apprentissage sont concentrées en un seul endroit, plus dispersées partout
- **Injection automatique du contexte IA** : l'assistant IA peut automatiquement comprendre les ressources d'apprentissage disponibles, sans fournir manuellement le contexte
- **Réutilisation des connaissances multi-projets** : plusieurs projets d'apprentissage peuvent partager et réutiliser des connaissances
- **Structure de répertoires standardisée** : fournit une structure de répertoires cohérente, réduisant les coûts d'apprentissage

Cette solution a été vérifiée en pratique dans le projet HagiCode. Si vous développez également des outils liés au développement assisté par l'IA, ou si vous êtes confronté à des problèmes similaires de gestion des connaissances, j'espère que cette expérience pourra vous servir de référence.

En fait, la valeur d'une solution technique ne réside pas dans sa complexité, mais dans sa capacité à résoudre des problèmes réels. L'idée centrale du système Vault est très simple - établir une couche de stockage de connaissances unifiée et compréhensible par l'IA. Mais c'est précisément cette abstraction simple qui a considérablement amélioré notre efficacité de développement.
