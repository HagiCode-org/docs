HagiCode prend en charge plusieurs chemins d'installation. Commencez par la méthode de déploiement qui correspond à votre scénario, puis ajoutez le logiciel de prise en charge uniquement lorsque vous en avez besoin.

La plage actuelle de prise en charge active de l'Agent CLI la plus pertinente pour la configuration est : Claude Code, Codex, GitHub Copilot, OpenCode, Hermes, QoderCLI, Kiro, Kimi, Gemini, DeepAgents et Codebuddy. Hermes, Gemini et DeepAgents utilisent actuellement des flux d'installation ou d'authentification manuels ; les autres CLI exposent des points d’entrée d’installation en un clic ou des flux d’installation guidée.

## Étapes suivantes recommandées

- Si vous êtes nouveau sur HagiCode, commencez par [Installation de bureau](/fr-FR/installation/desktop).
- Si vous effectuez un déploiement pour un environnement de serveur ou d'équipe, utilisez [Installation de Docker Composer](/fr-FR/installation/docker-compose).
- Si vous devez confirmer l'état de prise en charge, le mode de configuration ou les notes de compatibilité historiques pour une CLI, ouvrez [Installation de la CLI de l'agent IA](/fr-FR/related-software-installation/ai-agent-cli/).
