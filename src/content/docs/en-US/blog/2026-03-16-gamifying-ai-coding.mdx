## How to Make AI Coding More Fun Through Gamification Design

> Traditional AI coding tools are actually quite powerful, but they lack a bit of warmth. When building HagiCode, we thought: since we have to write code anyway, why not turn it into a game?

## Background

Friends who have used AI coding assistants should have this experience: it feels fresh at first, but after using it for a while, you feel something is missing. The tools themselves are powerful, capable of code generation, auto-completion, bug fixing—everything, but... they lack warmth, and after using them for a long time, they feel monotonous and boring.

That's fine, after all, who wants to face a cold tool every day.

It's like playing a game: if you just complete a task list without character growth, achievement unlocks, or team cooperation, you'll quickly lose interest. Beautiful things or people don't need to be possessed—just appreciate their beauty from afar. But programming tools don't even have this beauty, which inevitably leaves people disappointed.

We encountered this problem during the development of [HagiCode](https://hagicode.com). As a multi-AI assistant collaboration platform, HagiCode needs to keep users enthusiastic about long-term use. But the reality is that even the best tools are hard for users to stick with without emotional connection.

To solve this pain point, we made a bold decision: turn programming into a game. Not just a simple points leaderboard, but a true role-playing game experience. The changes brought by this decision might be bigger than you imagine.

After all, humans always need a sense of ritual.

## Why Gamification

The essence of gamification is not "adding a leaderboard," but building a complete incentive system that allows users to experience a sense of growth, achievement, and social recognition while completing tasks.

HagiCode's gamification design revolves around a core concept: each AI assistant is a "hero," and the user is the captain of this hero team. You lead these heroes to conquer various "dungeons" (programming tasks). In this process, heroes gain experience, level up and unlock abilities, and you and your team also earn various achievements.

This is not a gimmick, but a carefully designed system based on human behavioral psychology. When tasks are given meaning and progress feedback, human engagement and persistence significantly improve.

As the ancients said, "This love could have become a memory, but I was already lost in the moment." We incorporate this emotional experience into the tool, making programming not just about typing code, but a journey worth remembering.

## Hero Character System

Hero is the core concept of HagiCode's gamification system. Each Hero represents an AI assistant—for example, Claude Code is a Hero, and Codex is also a Hero.

### The Three Hero Slots

Heroes have three equipment slots—this design is actually quite clever:

1. **CLI Slot (Primary Profession)**: Determines the Hero's basic abilities, such as Claude Code or Codex
2. **Model Slot (Secondary Profession)**: Determines the model used, such as Claude 4.5 or Claude 4.6
3. **Style Slot (Style)**: Determines the Hero's behavioral style, such as "Windfall Strategist" or "other styles"

The combination of three slots can create unique Hero configurations. Like equipment loadouts in games, you need to choose the right combination based on task characteristics. After all, what suits you is best—this is similar to choosing paths in life. All roads lead to Rome, but some are easier to walk, while others are a bit more winding.

### Hero Progression System

Each Hero has their own XP (experience points) and level:

```typescript
type HeroProgressionSnapshot = {
  currentLevel: number;                    // Current level
  totalExperience: number;                 // Total experience
  currentLevelStartExperience: number;     // Experience at start of current level
  nextLevelExperience: number;             // Experience needed for next level
  experienceProgressPercent: number;       // Progress percentage
  remainingExperienceToNextLevel: number;  // Experience remaining to next level
  lastExperienceGain: number;              // Most recent experience gain
  lastExperienceGainAtUtc?: string | null; // Time of experience gain
};
```

Levels are divided into four stages, with each stage's naming quite immersive:

```typescript
export const resolveHeroProgressionStage = (level?: number | null): HeroProgressionStage => {
  const normalizedLevel = Math.max(1, level ?? 1);
  if (normalizedLevel <= 100) return 'rookieSprint';      // Rookie Sprint
  if (normalizedLevel <= 300) return 'growthRun';         // Growth Run
  if (normalizedLevel <= 700) return 'veteranClimb';      // Veteran Climb
  return 'legendMarathon';                                // Legend Marathon
};
```

From "Rookie" to "Legend," this growth path gives users a clear sense of goals and achievement. Like life's growth, you always go through the process from ignorance to maturity—it's just that this process is visualized here.

## Dungeon System

"Dungeon" is a classic concept in games, representing challenges that require a team to conquer. In HagiCode, each workflow is a Dungeon.

### Dungeon Organization

Dungeons organize workflows into different "dungeons":

- **Proposal Generation Dungeon**: Responsible for generating technical proposals
- **Proposal Execution Dungeon**: Responsible for executing tasks in proposals
- **Proposal Archive Dungeon**: Responsible for organizing and archiving completed proposals

Each dungeon has its own Captain (leader) Hero, which is automatically selected as the first enabled Hero.

Actually, this is like the division of labor in life—everyone has their role, it's just that here this division becomes a game mechanic.

### Team Collaboration Mechanism

You can configure different Hero teams for different dungeons:

```typescript
const dungeonRoster: HeroDungeonRoster = {
  scriptKey: 'proposal.generate',
  displayName: 'Proposal Generation',
  members: [
    { heroId: 'hero-1', name: 'Athena', executorType: 'ClaudeCode' },
    { heroId: 'hero-2', name: 'Apollo', executorType: 'Codex' }
  ]
};
```

For example, use Athena (good at strategy) when generating proposals, and Apollo (good at implementation) when executing code, so each hero can play to their strengths. It's like forming a band—everyone has their instrument, and together they create beautiful music.

### Dungeon Flow Control

Dungeons use a fixed scriptKey to identify different processes:

```typescript
// Script keys correspond to different workflows
const dungeonScripts = {
  'proposal.generate': 'Proposal Generation',
  'proposal.execute': 'Proposal Execution',
  'proposal.archive': 'Proposal Archive'
};
```

Task status flow is: queued → dispatching → dispatched. The entire process is automated without manual intervention. This is also our little lazy trick—after all, who wants to manually manage these things?

## XP and Level System

XP (experience points) is the core feedback mechanism in the gamification system. Users gain XP by completing tasks, XP makes heroes level up, leveling up unlocks new abilities, forming a positive loop.

### XP Acquisition Methods

In HagiCode, XP can be earned through the following activities:

- Code execution completion
- Successful tool calls
- Proposal generation
- Session management operations
- Project operations

Each time a valid operation is completed, the corresponding Hero gains XP. Like growth in life, every step counts—it's just that growth is quantified here.

### Real-time Progress Visualization

XP and level progress is visualized in real-time:

```typescript
type HeroDungeonMember = {
  heroId: string;
  name: string;
  icon?: string | null;
  executorType: PCode_Models_AIProviderType;
  currentLevel?: number;                    // Current level
  totalExperience?: number;                 // Total experience
  experienceProgressPercent?: number;       // Progress percentage
};
```

Users can see each Hero's level and progress at any time. This immediate feedback is key to gamification design. After all, people always need some feedback, otherwise how do they know they're improving?

## Achievement System

Achievements are another important element in gamification, providing long-term goals and milestone satisfaction.

### Achievement Types

HagiCode supports multiple achievement types:

- **Code Generation**: Generate X lines of code, generate Y files
- **Session Management**: Complete Z conversations
- **Project Operations**: Worked on W projects

Actually, these achievements are like milestones in life—we just turned them into a game mechanic.

### Achievement Status

Achievements have three states:

```typescript
type AchievementStatus = 'unlocked' | 'in-progress' | 'locked';
```

Three states have clear visual distinction:

- **Unlocked**: Gold gradient + glow effect
- **In Progress**: Blue pulse animation
- **Locked**: Gray, displaying unlock conditions

Each achievement clearly displays trigger conditions, letting users know what to do next. After all, when you're lost, having guidance is always good.

### Celebration Effect on Unlock

When an achievement is unlocked, a celebration animation is triggered. This positive reinforcement gives users a sense of "I did it" satisfaction, motivating them to continue. Like small rewards in life—though not big, they can make people happy for a long time.

## Battle Report Daily Combat Report

Battle Report is a unique feature of HagiCode that generates a full-screen battle-style report at the end of each day.

### Report Content

Battle Report displays the following information:

```typescript
type HeroBattleReport = {
  reportDate: string;
  summary: {
    totalHeroCount: number;        // Total hero count
    activeHeroCount: number;       // Active hero count
    totalBattleScore: number;      // Total battle score
    mvp: HeroBattleHero;           // Most valuable hero
  };
  heroes: HeroBattleHero[];        // Detailed data for all heroes
};
```

- Team total score
- Active Hero count
- Tool call count
- Total work time
- MVP (Most Valuable Hero)
- Detailed cards for each Hero

### MVP Highlight Display

The MVP is the Hero with the best performance that day and is highlighted in the report. This is not just data statistics, but a form of honor recognition. After all, who doesn't want to be recognized?

### Hero Detail Cards

Each Hero's card contains:

- Level progress
- XP gained
- Execution count
- Usage time

This data lets users clearly understand the team's work status. After all, understanding your effort results is also a form of satisfaction.

## Practice Guide

### Creating Your First Hero

Creating your first Hero is actually quite simple:

1. Go to the Hero management page
2. Click the "Create Hero" button
3. Configure three slots (CLI, Model, Style)
4. Give the Hero a name and description
5. Save, and your first Hero is born

Like making a new friend, you need to give them a name, understand their characteristics, and then you can start adventuring together.

### Building Dungeon Teams

Building a team is also simple:

1. Go to the Dungeon management page
2. Select the dungeon to configure (e.g., "Proposal Generation")
3. Select members from your Hero list
4. The system automatically selects the first enabled Hero as captain
5. Save the configuration

Actually, this is a team-building process—like forming a team in life, everyone has their role.

### Viewing Daily Reports

At the end of each day, you can view that day's Battle Report:

1. Click the "Battle Report" button
2. Full-screen display of that day's work results
3. View MVP and detailed data for each Hero
4. Share with team members (optional)

This is also a sense of ritual, letting you know how much effort you put in today and how far you are from your goal.

## Considerations and Best Practices

### Performance Optimization

Use React.memo to avoid unnecessary re-renders:

```typescript
const HeroCard = React.memo(({ hero }: { hero: HeroDungeonMember }) => {
  // Component implementation
});
```

After all, performance is important—who wants to use a laggy tool?

### Animation Fallback

Detect user's motion preference settings to provide a simplified experience for sensitive users:

```typescript
const prefersReducedMotion = useReducedMotion();
const duration = prefersReducedMotion ? 0 : 0.35;
```

After all, not everyone likes animations, and respecting user preferences is part of design.

### Backward Compatibility

Preserve legacyIds to support old version migration:

```typescript
type HeroDungeonMember = {
  heroId: string;
  legacyIds?: string[];  // Support old version ID mapping
  // ...

```

After all, no one wants to lose data due to version upgrades.

## Conclusion

Gamification is not just a simple points leaderboard, but a complete incentive system. Through the Hero system, Dungeon system, XP/level system, achievement system, and Battle Report, HagiCode transforms programming work into a heroic journey full of adventure.

The core value of this system lies in:

- **Emotional Connection**: Making cold AI assistants feel personalized
- **Positive Feedback**: Immediate feedback for every action
- **Long-term Goals**: Levels and achievements provide growth paths

Gamification design makes programming no longer boring, but a fun adventure. While completing code tasks, users experience the joy of character growth, team collaboration, and achievement unlocks, thereby increasing usage stickiness and activity.

Actually, programming itself is a form of creation—we just made this creation process a little more interesting.
