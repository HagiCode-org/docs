## Background

In Hagicode projects, users can choose from multiple CLI tools to drive AI programming assistants, including Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI, and more. These CLI tools are general-purpose AI programming tools, but through Hagicode's abstraction layer, they can flexibly integrate with different AI model providers.

Zhipu AI (ZAI) provides an interface compatible with the Anthropic Claude API, enabling these CLI tools to directly use domestic GLM series models. GLM-5.1 is the latest large language model released by Zhipu, showing significant improvements over GLM-5.0.

## Using GLM-5.1 with Different CLIs

### 1. Claude Code CLI + GLM-5.1

Claude Code CLI is one of Hagicode's core CLIs, configured through the Hero configuration system:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Corresponding `HeroEquipmentCatalogItem` configuration:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI is the most flexible CLI, supporting the `provider/model` format to specify any model:

**Method 1: Using ZAI provider prefix**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Method 2: Directly using model ID**
```json
{
  "model": "glm-5.1"
}
```

**Method 3: Frontend Configuration Interface**

In `HeroModelEquipmentForm.tsx`, OpenCode CLI has special placeholder hints:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Users can input:
```
zai/glm-5.1
glm-5.1
```

**OpenCode CLI Model Resolution Logic**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: directly use model ID
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Has slash: parse provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI has special legacy handling logic:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Note**: Codebuddy has special reservation for GLM-5.0, bypassing legacy normalization:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// When CodebuddyCli, glm-5.0 will not be normalized to glm-5-turbo
```

According to Zhipu's official release information, GLM-5.1 improvements include:

- **Stronger code understanding**: More accurate parsing of complex code structures
- **Longer context understanding**: Supports longer conversation contexts
- **Enhanced tool calling**: Improved success rate for MCP tool calls
- **Output stability**: Reduced randomness and hallucinations

## Notes

### 1. API Key Configuration

Ensure correct setup of the `ANTHROPIC_AUTH_TOKEN` environment variable, which is the necessary credential for all CLIs to connect to models.

### 2. Model Availability

GLM-5.1 needs to be enabled at the corresponding model provider:
- Zhipu AI ZAI platform supports by default
- Alibaba Cloud DashScope may require separate application

### 3. OpenCode CLI Format

When using `provider/model` format, ensure the provider ID is correct:
- Zhipu AI: `zai` or `zhipuai`
- Alibaba Cloud: `aliyun` or `dashscope`

### 4. Reasoning Parameter

- Recommended to use `high` level for best code generation results
- Gemini CLI does not support the reasoning parameter and will automatically ignore this configuration
