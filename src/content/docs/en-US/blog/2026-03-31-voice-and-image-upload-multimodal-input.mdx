## Talking Beats Typing, Screenshots Beat Talking—Multimodal Input Practices for AI Code Assistants

> Actually, when it comes to writing code, there's a limit to how fast typing can be. Sometimes something that can be said in a sentence requires hammering on the keyboard for a long time; sometimes a single image can make things clear, but requires using a bunch of words to describe. This article talks about the things we encountered while building [HagiCode](https://hagicode.com)—be it voice recognition or image upload, the goal was simply to make the AI code assistant a bit easier to use, that's all.

## Background

When building HagiCode, we discovered a problem—or rather, a problem that naturally emerged as users used it more: relying solely on typing can be quite tiring.

Think about it, users interacting with Agent is a core scenario. But having to sit at the keyboard typing away every time, well, the efficiency isn't exactly high:

1. **Typing is too slow**: For some complex issues, like errors or UI problems, typing them out takes half a minute, but saying them might take just ten seconds. This time difference is quite frustrating.

2. **Images are more direct**: Sometimes when the UI has an error, or you want to compare with a design mockup, or show code structure... the saying "a picture is worth a thousand words" may be old, but the logic holds true. Letting AI "see" the problem directly is much clearer than describing it for a long time.

3. **Interaction should be natural**: Modern AI assistants should support text, voice, images and other methods, right? Users should be able to use whatever they want—that's what natural means, isn't it?

So we thought, why not add voice recognition and image upload features to HagiCode to make Agent operations more convenient. After all, letting users type fewer words is a good thing.

## About HagiCode

The solutions shared in this article come from our practice in the [HagiCode](https://hagicode.com) project—or rather, experience gained through constantly stumbling and exploring.

HagiCode is an open-source AI code assistant project with a simple idea: use AI technology to improve development efficiency. As we built it, we discovered that users have quite strong demand for multimodal input—sometimes saying a sentence is faster than typing a bunch of words, sometimes a screenshot is clearer than describing for a long time.

These requirements pushed us forward, and eventually we ended up with features like voice recognition and image upload. Users can interact with AI in the most natural way, and it feels pretty good.

## Analysis

### Technical Challenges of Voice Recognition

When implementing voice recognition functionality, we encountered a tricky problem: **the browser's WebSocket API does not support custom HTTP headers**.

The voice recognition service we chose is ByteDance's Doubao Voice Recognition API. This API偏偏 requires passing authentication information through HTTP headers, things like `accessToken`, `secretKey`, etc. This created a technical conflict:

```javascript
// Browser WebSocket API does not support the following approach
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // Not supported
  }
});
```

The solutions before us were roughly two:

1. **URL query parameter approach**: Put authentication information in the URL
   - Advantage: simple to implement
   - Disadvantage: credentials exposed on frontend, poor security; and some APIs strictly require header validation

2. **Backend proxy approach**: Implement WebSocket proxy on the backend
   - Advantage: credentials stored securely on backend; fully compatible with API requirements
   - Disadvantage: slightly more complex to implement

In the end we chose the **backend proxy approach**. After all, security is a bottom line that cannot be compromised—no one should try to get around this.

### Image Upload Functional Requirements

For image upload functionality, our requirements are actually quite simple:

1. **Multiple upload methods**: Click to select file, drag-and-drop upload, clipboard paste—these should all be available, right?
2. **File validation**: Type restrictions (PNG, JPG, WebP, GIF), size restrictions (5-10MB)—these are basic operations
3. **User experience**: Upload progress, preview, error prompts—users should at least know what's happening
4. **Security**: Server-side validation, prevent malicious file uploads—this is a big deal

## Solutions

### Voice Recognition: WebSocket Proxy Architecture

We designed a three-layer architecture voice recognition solution, which I'd say we found a path:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**Core Component Implementation**:

1. **Frontend AudioWorklet Processor**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Resample to 16kHz (Doubao API requirement)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Accumulate samples to 500ms chunks
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Convert to 16-bit PCM and send
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **Backend WebSocket Handler** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **Frontend VoiceTextArea Component**:

```tsx
export const VoiceTextArea = forwardRef<HTMLTextAreaElement, VoiceTextAreaProps>(
  ({ value, onChange, onTextRecognized, maxDuration }, ref) => {
    const { isRecording, interimText, volume, duration, startRecording, stopRecording } =

    return (
      <div className="flex gap-2">

        </button>
        {/* Text input */}
        <textarea value={displayValue} onChange={handleChange} />
      </div>
    );
  }
);
```

### Image Upload: Multi-method Upload Component

We built a fully functional image upload component that supports all three upload methods, which I'd say covers the common scenarios users need.

**Core Features**:

1. **Three Upload Methods**:

```tsx
// Click to upload
const handleClick = () => fileInputRef.current?.click();

// Drag and drop upload
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Clipboard paste
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **Frontend Validation**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **Backend Upload Handler** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Validate
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Save file
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## Practice Guide

### How to Use Voice Recognition

1. **Configure Voice Recognition Service**:
   - Go to voice recognition settings page
   - Configure Doubao Voice's `AppId` and `AccessToken`
   - (Optional) Configure hotwords to improve professional term recognition accuracy

2. **Use in Input Box**:
   - Click the microphone icon on the left side of the input box
   - Start speaking when you see the waveform animation
   - Click the icon again to stop recording
   - Recognition results will be automatically inserted at the cursor position

### How to Use Image Upload

1. **Upload Methods**:
   - Click upload button to select file
   - Drag and drop image directly to upload area
   - Use `Ctrl+V` to paste screenshot from clipboard

2. **Supported Formats**: PNG, JPG, JPEG, WebP, GIF
3. **Size Limit**: Default 5MB (configurable)

### Notes

1. **Voice Recognition**:
   - Requires microphone permission
   - Recommended to use in a quiet environment
   - Maximum recording duration is 300 seconds (configurable)

2. **Image Upload**:
   - Only supports common image formats
   - Pay attention to file size limits
   - Uploaded images will automatically generate preview URLs

3. **Security Considerations**:
   - Voice recognition credentials stored on backend
   - Image upload has strict server-side validation
   - Production environment recommends using HTTPS/WSS

## Summary

After adding voice recognition and image upload, HagiCode's user experience has improved significantly. Users can now interact with AI in more natural ways—speaking instead of typing, screenshots instead of describing. This feeling, I'd say, is like finally finding a more comfortable way to communicate.

When building this feature, we encountered the problem that browser WebSocket doesn't support custom headers, and we finally solved it through the backend proxy approach. This solution not only ensures security, but also lays the foundation for integrating other authenticated WebSocket services in the future—consider it an unexpected gain.

The image upload component is similar, using multiple upload methods so users can choose the most convenient one based on the scenario. Clicking, dragging, or pasting directly can all complete uploads quickly. All roads lead to Rome, it's just that some roads are easier to walk, others are a bit more winding.

"Talking beats typing, screenshots beat talking"—this saying fits quite well here. If you're also building similar AI assistant products, I hope these experiences can help you, even if just a little.
