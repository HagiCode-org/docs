## Designing 12 Ultimate Splash Experiences for HagiCode: From Minimalist to Cyberpunk

> In the brief gap between React 19 application download and Hydration, it's a golden window for users to perceive brand personality. This article shares our complete set of splash screen style systems built on HTML/CSS/JS in the HagiCode project.


## Background

As a modern application based on ASP.NET Core 10 and React 19 (Vite), HagiCode adopts a separated frontend-backend deployment architecture. Frontend build artifacts are packaged and placed in the backend's `wwwroot/` directory, hosted by ASP.NET Core.

However, this architecture brings a classic user experience pain point: when users visit a webpage, the browser needs to first load the HTML, then download the massive JS Bundle, and finally React executes Hydration. During this hundreds of milliseconds to several seconds of "vacuum period," users face either a blank screen or a lifeless static page.

To fill this gap and inject HagiCode's brand personality, we need to design a complete splash screen style system based entirely on inline code in `index.html`.

## About HagiCode

The splash screen design solution shared in this article comes from our practical experience in the [HagiCode](https://github.com/HagiCode-org/site) project. As an AI code assistant, HagiCode not only focuses on code generation efficiency but also equally values developers' visual experience. This splash screen system is precisely the product of our pursuit of ultimate frontend performance.

## Core Challenges and Architecture Design

Before designing, we must first clarify technical constraints. Since we need to implement it inline in `index.html`, it means we cannot load any external CSS or JS files (except React's own Bundle).

### Technical Constraint Analysis

1.  **Zero Dependency Principle**: All styles must be written within `<style>` tags, and logic within `<script>` tags.
2.  **Defensive CSS**: To prevent global styles from polluting the splash screen after the React app mounts, we decided to use high-priority ID prefixes (like `#boot-screen`) to wrap all splash styles.
3.  **Performance First**: Animations should use CSS `transform` and `opacity` as much as possible to avoid triggering reflow and ensure the main thread is not blocked.
4.  **Visual Consistency**: Colors and fonts must remain consistent with HagiCode's Tailwind configuration.

### Architecture Pattern: Shell & Injector

We adopted a **Variant Pattern**. The core logic is encapsulated in an Immediately Invoked Function Expression (IIFE), with specific rendering logic injected as configuration items. This allows us to switch between different styles through simple configuration without repeatedly writing DOM manipulation logic.

Here is the core architecture code:

```html
<!-- Inline in index.html -->
<div id="boot-root"></div>

<script>
(function() {
  const BootSequence = {
    config: {
      theme: 'terminal', // Can be configured as 'minimal', 'skeleton', 'code-rain', etc.
      color: '#3b82f6'   // Brand color
    },
    
    // Core lifecycle
    init() {
      this.render();
      this.listenForMount();
    },

    // Render the currently selected style
    render() {
      const root = document.getElementById('boot-root');
      if (this.variants[this.config.theme]) {
        root.innerHTML = this.variants[this.config.theme].render();
      }
    },

    // Listen for React mount success, exit gracefully
    listenForMount() {
      window.addEventListener('hagicode:ready', () => {
        const screen = document.getElementById('boot-root');
        // Fade out first, then remove DOM to avoid flickering
        screen.style.opacity = '0';
        screen.style.transition = 'opacity 0.3s ease';
        setTimeout(() => screen.remove(), 300);
      });
    },

    // 12 style implementations concentrated here
    variants: {
      // ...Specific implementations见 below
    }
  };

  BootSequence.init();
})();
</script>
```

## 12 Splash Style Design List

We divided these 12 styles into six major categories to meet different scenarios and aesthetic needs.

### A. Minimalism

> "Less is more." For scenarios pursuing ultimate loading speed, we provide the lightest solution.

#### 1. Minimalist Dot (Minimalist Breathing)
Only a simple dot in the center of the screen, combined with a breathing animation.
*   **Implementation**: CSS `@keyframes` controls scale and opacity.
*   **Applicable**: Any occasion that requires keeping the page absolutely clean.

#### 2. Brand Reveal
Through SVG `stroke-dasharray` animation, simulate hand-drawing HagiCode's Logo lines, then fade in text.
*   **Technique**: Use SVG path animation for extremely high quality.

### B. Skeleton Screen Mimicry

> "The art of deception." By simulating real UI layout, users feel the page is half loaded.

#### 3. Sidebar Chat Skeleton
This might be the most practical one. We manually built a layout identical to the React components `Sidebar` and `ChatInput` using HTML, and covered it with gray stripe animations.
*   **Value**: When React hydration completes, the skeleton screen instantly transforms into real components, users hardly notice the switch.

#### 4. Card Stack Skeleton
Simulate the stacking animation when proposal cards load, using 3D transforms to make cards float slightly.

### C. Abstract and Artistic

> Showcase HagiCode's geek genes.

#### 5. Geometric Morph
Render a geometric shape (square) in the center of the screen, it smoothly transforms into a circle, triangle over time, and finally becomes the Logo.
*   **Technology**: Smooth transition of CSS `border-radius`.

#### 6. Code Rain
A tribute to "The Matrix." Use JetBrains Mono font, with faint character streams falling in the background.
*   **Note**: For performance, character streams must be limited to smaller areas or reduce refresh frequency.

#### 7. Neon Pulse
Cyberpunk style glowing ring, using multiple superpositions of `box-shadow` to create intense glow.

### D. Brand and Theme

> Make the system "alive."

#### 8. Seasonal Theme
This is a dynamic loader. Based on the current date to determine holidays (such as Spring Festival, Christmas), load corresponding SVG animations.
*   **Example**: During Spring Festival, red lanterns gently sway at the bottom of the screen.

#### 9. Gradient Flow
Background uses fluid gradient of HagiCode brand colors, combined with `background-size` and `background-position` animations to create an aurora-like flowing sensation.

### E. Technical Feel

> A tribute to developers.

#### 10. Terminal Boot
Simulate console output. Lines of code scroll quickly:
```text
> Initializing HagiCode Core...
> Loading models...
> Connecting to neural network...
```
This will make every developer feel familiar.

#### 11. Progress Bar (Minimalist Progress Bar)
A thin progress bar at the top of the screen, with percentage displayed on the right. Although we cannot get real download progress, we can use a timer to simulate a "credible" loading process (first 80% fast, last 20% slow down).

### F. Creative

#### 12. Pixel Assembly
This is an interesting creative idea. Squares scattered on the screen gather to the center, gradually piecing together the HagiCode Logo icon. Symbolizing the code construction process.

## Best Practices and Pitfall Summary

In the actual development of HagiCode, we summarized some crucial practical details.

### 1. Defensive CSS is Essential
Never be lazy and skip prefixes. Once, we didn't add ID restrictions to splash screen styles, causing global `div` styles after React mount to accidentally affect the splash screen, leading to layout collapse.
**Experience**: All CSS selectors are under `#boot-screen`, and use `!important` to increase priority (only in splash screen CSS).

### 2. Graceful Transition
After React mount succeeds, don't directly `remove()` the splash screen DOM.
**Correct Approach**:
1.  React triggers `window.dispatchEvent(new Event('hagicode:ready'))`.
2.  The splash screen listens to the event, first sets `opacity: 0`.
3.  Wait 300ms (CSS transition time), ensure users can't see it, then execute `.remove()`.

### 3. Theme Variable Synchronization
Splash screen color codes are hardcoded in `index.html`. If we modify Tailwind's primary color, we must synchronize the modification here.
**Optimization**: In the Vite build script, write a simple plugin to read `tailwind.config.js` and inject color variables into template variables in `index.html`, achieving a single source of truth.

### 4. Font Preloading
Splash screens usually need to use brand fonts, but if font loading is slow, FOUT (Flash of Unstyled Text) will occur.
**Solution**: Add `<link rel="preload" href="/fonts/JetBrainsMono.woff2" as="font" type="font/woff2" crossorigin>` in `<head>`. This is a low-cost high-return means to improve experience.

### 5. Performance Monitoring
We injected `performance.mark('boot-start')` at the bottom of `index.html`, and marked `boot-end` when React mount succeeds.
**Significance**: By collecting this data through Application Insights, we can truly see the degree to which the splash screen shortens users' perceived waiting time. Data shows that excellent skeleton screens can increase users' tolerance for "slow networks" by over 50%.
