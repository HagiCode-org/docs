To install and start using Hermes, the shortest path actually only takes three steps:

1. Run the official installation command
2. Start the CLI with `hermes` in the terminal
3. If you want to continue using it in Feishu, configure `hermes gateway setup`

This article doesn't intend to cover all of Hermes's capabilities at once. Instead, it helps you complete the most critical onboarding loop: install it, get it running, start using it, and then connect it to the most common messaging platform scenario.

## What is Hermes and Who Should Use It

Hermes Agent is an AI agent that can be used both in a local terminal and through messaging platform gateways.

For most developers, there are two most common entry points:

- **CLI**: Type `hermes` in the terminal to directly enter the interactive interface
- **Messaging Gateway**: Run `hermes gateway`, then talk to it from platforms like Feishu, Telegram, Discord, Slack, etc.

If your current goal is just to get started quickly, it's recommended not to reverse this order, but to follow this route:

- Install Hermes first
- Verify it works from CLI first
- Then decide whether to connect a messaging platform

This makes it easier to troubleshoot issues and is more suitable for first-time Hermes users.

## What to Know Before Installing Hermes

According to the Hermes README, the official quick installation path supports these environments:

- Linux
- macOS
- WSL2
- Android via Termux

### Notes for Windows Users

Hermes currently does not support running directly on native Windows. If you use Windows, it's recommended to install WSL2 first, then execute the installation command within WSL2.

This is best stated clearly at the beginning of the article, because many installation failures are not due to command issues, but because the runtime environment doesn't meet the requirements.

## How to Install Hermes

The quick installation command given by the Hermes README is:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

This command executes the official installation script and handles platform-specific initialization processes.

### Refresh Shell After Installation

After installation completes, first reload the shell environment. The most common approach is:

```bash
source ~/.bashrc
```

If you're using zsh, you can also use:

```bash
source ~/.zshrc
```

### How to Confirm Hermes is Installed

The most direct way to check is to run:

```bash
hermes
```

If you want to further confirm that configuration and dependencies are normal, you can also execute:

```bash
hermes doctor
```

`hermes doctor` is suitable for use in these situations:

- Command behavior is abnormal after installation
- Model configuration fails
- Gateway startup fails
- Unsure if environment dependencies are complete

## How to Use Hermes for the First Time

### Start with CLI First

If you just want to confirm as quickly as possible whether Hermes works, the simplest method is:

```bash
hermes
```

This starts Hermes's interactive CLI. For first-time Hermes users, this is also the most recommended starting point, as you can first verify the most core things:

- Whether the command is actually available
- Whether current model configuration is normal
- Whether terminal toolchain works properly
- Whether the interactive experience is what you need

### These Commands Are Enough for Your First Round of Configuration

The high-frequency commands listed in the Hermes README basically form the first round of getting started:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

If you don't know what each does, you can remember them this way:

- `hermes model`: Select model, switch model
- `hermes tools`: View and configure currently available tools
- `hermes config set`: Modify specific configuration items
- `hermes setup`: Run a complete initialization wizard
- `hermes update`: Update Hermes
- `hermes doctor`: Perform troubleshooting

The most practical order for beginners is usually:

1. First run `hermes model`
2. If you want to configure common items completely at once, then run `hermes setup`

## Two Most Common Ways to Use Hermes

### 1. Use Hermes as a Daily Development Assistant in the Terminal

CLI mode is suitable for these scenarios:

- Ask questions directly when writing code locally
- Check projects, modify files, run commands
- Do one-time debugging or review
- Continuous collaboration in the current working directory

Its biggest advantage is the shortest path: no need to connect an additional platform, no need to handle bot configuration first, and it's most suitable for establishing first-round usage habits.

### 2. Use Hermes Through Messaging Platforms

If you want to talk to Hermes on platforms like Feishu, Telegram, Discord, etc., you need to use the messaging gateway.

The most common entry commands are:

```bash
hermes gateway setup
hermes gateway
```

Among them:

- `hermes gateway setup` is used for interactive platform configuration
- `hermes gateway` is used to start the gateway process

According to the official documentation, gateway is a unified background process used to connect your configured platforms, manage sessions, and handle scheduled tasks and other functions.

## How to Connect Hermes to a Messaging Platform Using Feishu as an Example

If your daily work is mainly in Feishu, then Feishu/Lark will be a very natural Hermes integration method.

### Minimum Viable Integration Method

The official documentation's recommended entry point for Feishu/Lark is:

```bash
hermes gateway setup
```

After running, select **Feishu / Lark** in the wizard.

The two connection modes given by the Feishu documentation are:

- `websocket`: Recommended
- `webhook`: Optional

If Hermes is running on your laptop, workstation, or private server, using `websocket` will be simpler because you don't need to expose an additional public callback address.

### If You Configure Manually, At Least Know These Variables

If you're not configuring through the wizard but writing configuration manually, the core variables listed in the Feishu documentation include:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

The two most noteworthy items are:

- `FEISHU_ALLOWED_USERS`: Recommended configuration to avoid anyone who can access the bot being able to use it directly
- `FEISHU_HOME_CHANNEL`: Can pre-specify a home chat to receive cron results or default notifications

### Why Hermes Sometimes Doesn't Reply in Feishu Group Chats

This point is easily overlooked: In Feishu group chats, Hermes doesn't respond to every message it sees by default.

The official documentation clearly states:

- In private chats, Hermes will respond to messages
- In group chats, you must explicitly `@` the bot for it to process messages

If you want to set a Feishu conversation as the home channel, you can also use in chat:

```text
/set-home
```

Or write it in advance in the configuration:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Hermes Commands Most Worth Remembering for Beginners

Whether you're in CLI or on a messaging platform, remembering the following commands is enough to get started:

- `/new` or `/reset`: Start a new session
- `/model`: View or switch models
- `/retry`: Retry the previous round
- `/undo`: Undo the previous interaction
- `/compress`: Manually compress context
- `/help`: View help

If you mainly use it on messaging platforms, remember one more:

- `/sethome` or `/set-home`: Set the current chat as the home channel

These commands cover the most common operations in the beginner phase: restart, adjust, roll back, view, and continue using.

## Common Questions

### Can I Install Hermes Directly on Windows?

No. The current official documentation clearly states that native Windows is not supported, and WSL2 is recommended.

### What to Do If Entering `hermes` After Installation Has No Response?

It's recommended to troubleshoot in the following order:

1. First reload the shell, such as `source ~/.bashrc`
2. Then run `hermes` again
3. If it's still abnormal, execute `hermes doctor`

### Why Doesn't the Bot Reply in Feishu Groups?

First check these three items:

- Whether you `@` Hermes in the group
- Whether `FEISHU_ALLOWED_USERS` restricts the current user
- Whether the current group chat policy allows processing group messages

According to the official Feishu documentation, in group chat scenarios, explicit @mention is a necessary condition.
