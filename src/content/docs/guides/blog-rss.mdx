## 兼容性

`https://docs.hagicode.com/blog/rss.xml` 已移除。请改为订阅具体语言的 RSS 地址。英文订阅继续保留 `rss.en.xml`，同时提供 `rss.en-US.xml` 作为桌面端规范代码的等价入口。

## 语言规则

RSS 生成时会先为每篇博客文章归一化语言：

- 如果文章 frontmatter 显式设置了 `language`，会归一化为桌面端规范代码：`zh-CN`、`zh-Hant`、`en-US`、`ja-JP`、`ko-KR`、`de-DE`、`fr-FR`、`es-ES`、`pt-BR` 或 `ru-RU`。
- 如果没有显式语言字段，则根据内容路径判断：`src/content/docs/blog/` 视为 `zh-CN`，`src/content/docs/en-US/blog/` 视为 `en-US`，其他语言使用对应目录名。

每个 RSS 条目都会带有对应的 `<language>` 标记，便于订阅器或下游自动化流程识别来源语言。

## 路由约定

博客保留现有简体中文根路由和英文短路由，并为其他桌面端语言使用规范语言代码作为路由前缀。

| 语言代码 | 博客路由 | 内容目录 | 回退语言 |
|----------|----------|----------|----------|
| `zh-CN` | `/blog/<slug>/` | `src/content/docs/blog/` | `en-US` |
| `zh-Hant` | `/zh-Hant/blog/<slug>/` | `src/content/docs/zh-Hant/blog/` | `zh-CN`, `en-US` |
| `en-US` | `/en-US/blog/<slug>/` | `src/content/docs/en-US/blog/` | `en-US` |
| `ja-JP` | `/ja-JP/blog/<slug>/` | `src/content/docs/ja-JP/blog/` | `en-US` |
| `ko-KR` | `/ko-KR/blog/<slug>/` | `src/content/docs/ko-KR/blog/` | `en-US` |
| `de-DE` | `/de-DE/blog/<slug>/` | `src/content/docs/de-DE/blog/` | `en-US` |
| `fr-FR` | `/fr-FR/blog/<slug>/` | `src/content/docs/fr-FR/blog/` | `en-US` |
| `es-ES` | `/es-ES/blog/<slug>/` | `src/content/docs/es-ES/blog/` | `en-US` |
| `pt-BR` | `/pt-BR/blog/<slug>/` | `src/content/docs/pt-BR/blog/` | `en-US` |
| `ru-RU` | `/ru-RU/blog/<slug>/` | `src/content/docs/ru-RU/blog/` | `en-US` |

## 新增博客文章流程

新增博客文章时，为上表中的每个内容目录都添加同名 `.mdx` 文件。文件名中的日期和 slug 必须保持一致，例如 `2026-04-29-example.mdx` 在所有语言目录中都必须存在。

`language` frontmatter 可以省略，验证器会从路径推导语言；如果显式填写，必须与所在路由匹配。提交前从 `repos/docs` 运行 `npm run verify:blog-i18n-completeness`，确认所有语言目录、同名文件和语言元数据都通过检查。
