## 背景

Hagicode プロジェクトでは、Claude Code CLI、GitHub Copilot、OpenCode CLI、Codebuddy CLI、Hermes CLI など、複数の CLI ツールを選択して AI プログラミングアシスタントを驱动できます。これらの CLI ツール自体は汎用的な AI プログラミングツールですが、Hagicode の抽象化レイヤーを通じて、異なる AI モデルプロバイダーに柔軟に接続できます。

智譜AI（ZAI）は Anthropic Claude API と互換性のあるインターフェースを提供しており、これらの CLI ツールは国産 GLM シリーズモデルを直接使用できます。その中で GLM-5.1 は智譜AIが最新リリースした大規模言語モデルであり、GLM-5.0 に比べて大幅な改良がなされています。

## Hagicode の CLI 抽象化アーキテクチャ

Hagicode は `AIProviderType` 列挙型で 11 種類の CLI プロバイダータイプを定義し、主流の AI プログラミング CLI ツールをカバーしています：

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

各 CLI には対応するモデルパラメータ設定があり、`model` と `reasoning` パラメータをサポートしています：

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],
        [AIProviderType.CodexCli] = ["model", "reasoning"],
        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],
        [AIProviderType.HermesCli] = ["model", "reasoning"],
        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini は reasoning パラメータをサポートしない
        // ...
    };
```

## GLM モデルサポート体系

Hagicode の Secondary Professions Catalog では完全な GLM シリーズモデルサポートが定義されています：

| Model ID | Name | Default Reasoning | Compatible CLI Families |
|----------|------|-------------------|-------------------------|
| `glm-4.7` | GLM 4.7 | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM 5 Turbo | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (Legacy) | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | high | claude, codebuddy, hermes, qoder, kiro |

## GLM-5.1 vs GLM-5.0 の重要な違い

`AcpSessionModelBootstrapper.cs` の実装から、GLM-5.1 と GLM-5.0 の違いを明確に理解できます：

### GLM-5.1 の独立実装

GLM-5.1 は独立した新しいモデル識別子であり、レガシー処理ロジックは一切ありません：

## 異なる CLI での GLM-5.1 使用

### 1. Claude Code CLI + GLM-5.1

Claude Code CLI は Hagicode のコア CLI の一つであり、Hero 設定システムで設定します：

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

対応する `HeroEquipmentCatalogItem` 設定：

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI は最も柔軟な CLI であり、`provider/model` 形式で任意のモデルを指定できます：

**方法 1：ZAI provider プレフィックスを使用**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**方法 2：モデル ID を直接使用**
```json
{
  "model": "glm-5.1"
}
```

**方法 3：フロントエンド設定インターフェース**

`HeroModelEquipmentForm.tsx` で、OpenCode CLI には特別なプレースホルダープロンプトがあります：

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

ユーザーは以下を入力できます：
```
zai/glm-5.1
glm-5.1
```

**OpenCode CLI のモデル解析ロジック**：

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // スラッシュなし：モデル ID を直接使用
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // スラッシュあり：provider/model 形式を解析
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI には特別なレガシー処理ロジックがあります：

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**注意**：Codebuddy は GLM-5.0 を特別に保持しており、レガシー正規化は適用されません：

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// CodebuddyCli の場合、glm-5.0 は glm-5-turbo に正規化されない
```

## GLM-5.1 の改良优势

GLM-5.0 に比べて、GLM-5.1 には以下の顕著な改良があります：

### 1. より優れた推論能力

智譜AI 公式リリース情報によると、GLM-5.1 の改良には以下が含まれます：

- **より強力なコード理解能力**：複雑なコード構造の解析がより正確
- **より長いコンテキスト理解**：より長い対話コンテキストをサポート
- **ツール呼び出しの強化**：MCP ツール呼び出しの成功率が向上
- **出力の安定性**：ランダム性と幻覚を低減

### 2. 複数 CLI の全面的互換性

GLM-5.1 は Hagicode がサポートするすべての主流 CLI をカバーしています：

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## 注意事項

### 1. API Key 設定

すべての CLI がモデルに接続するために必要な認証情報として、`ANTHROPIC_AUTH_TOKEN` 環境変数が正しく設定されていることを確認してください。

### 2. モデル可用性

GLM-5.1 は対応するモデルプロバイダーで開通する必要があります：
- 智譜AI ZAI プラットフォームはデフォルトでサポート
- アリババクラウド DashScope は別途申請が必要な場合があります

### 3. OpenCode CLI 形式

`provider/model` 形式を使用する場合、provider ID が正しいことを確認してください：
- 智譜AI：`zai` または `zhipuai`
- アリババクラウド：`aliyun` または `dashscope`

### 4. Reasoning パラメータ

- 最適なコード生成効果を得るために `high` レベルの使用を推奨します
- Gemini CLI は reasoning パラメータをサポートしていないため、この設定は自動的に無視されます

## まとめ

Hagicode は統一された抽象化レイヤーを通じて、GLM-5.1 と複数の CLI の柔軟な統合を実現しています。開発者は自分の嗜好と使用シナリオに応じて適切な CLI ツールを選択し、簡単な設定で最新の GLM-5.1 モデルを使用できます。

智譜AI の最新モデルバージョンとして、GLM-5.1 は GLM-5.0 に比べて明らかな改良があります：
- 独立したバージョン識別子、レガシーな負担なし
- より強力な推論能力とコード理解
- 広範な複数 CLI 互換性
- 柔軟な推論レベル設定

正しく環境変数と Hero 装備を設定することで、プレイヤーは異なる CLI 環境で GLM-5.1 の強力な能力を最大限に発揮できます。

## HagiCode を続き体験する

GLM-5.1、複数 CLI オーケストレーション、HagiCode の設定体系を実際に動かしたい場合は、以下のエントリーポイントから始められます：

- GitHub でメインプロジェクトと最新の実装進捗を確認：[github.com/HagiCode-org/site](https://github.com/HagiCode-org/site)
- 公式サイトで製品ロードマップ、能力境界、インストールエントリーを確認：[hagicode.com](https://hagicode.com)
- まず Docker Compose ドキュメントに従ってワンクリックデプロイを完了し、その後モデルや CLI を切り替えて実際に検証：[docs.hagicode.com/installation/docker-compose](https://docs.hagicode.com/installation/docker-compose)
- ローカルデスクトップ体験を重視する場合は、Desktop エントリーから直接始めることも可能：[hagicode.com/desktop/](https://hagicode.com/desktop/)

Kimi、Claude Code、OpenCode などの CLI と GLM-5.1 を同じ抽象化レイヤーで比較するとき、モデル切り替え、パラメータマッピング、エンジニアリング境界に関する多くの問題が、一気にはっきりと理解できるようになります。
