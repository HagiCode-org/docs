## ゲーム化デザインでAIプログラミングをもっと楽しくする方法

> 実は、従来のAIプログラミングツールは機能が非常に強力ですが、どこか温かみに欠けています。HagiCodeを開発する際に、コードを書くなら、なぜそれをゲームにしないのかと考えました。

## 背景

AIプログラミングアシスタントを使ったことのある方は、このような経験があるはずです：最初は新鮮に感じますが、使い続けているうちに何かが足りないと感じるようになります。ツール自体の機能は非常に強力で、コード生成、自動補完、バグ修正など、あらゆることができますが、どこか温かみがなく、使い続けていると単調で退屈に感じることがあります。

それはそれで構いません。結局のところ、毎日冷たいツールと向き合いたい人は誰もいませんから。

これはゲームをするのと似ています。単にタスクリストをこなすだけで、キャラクターの成長も、達成感のアンロックも、チームワークもなければ、すぐに退屈に感じてしまいます。美しい事物や人は、必ずしも所有する必要はなく、ただ美しくあり、自分でその美しさを眺めていればいいのです。しかし、プログラミングツールにはそのような美しささえなく、难免让人心灰意冷。

[HagiCode](https://hagicode.com)を開発する過程で、私たちはこの問題に遭遇しました。HagiCodeは複数のAIアシスタント協業プラットフォームとして、ユーザーが長期的に使用する熱意を保つ必要があります。しかし、現実は、どんなに優れたツールでも、感情的なつながりが不足していれば、ユーザーも継続するのは難しいでしょう。

この問題を解決するために、私たちは大胆な決定を下しました：プログラミングをゲームにするのです。単純なポイントランキングではなく、真のRPGスタイルのゲーム化体験です。この決定がもたらす変化は、あなたが想像する以上に大きいかもしれません。

結局のところ、人は常に儀式感が必要なものです。

## HagiCodeについて

この記事で共有するソリューションは、[HagiCode](https://hagicode.com)プロジェクトでの実践経験から来ています。HagiCodeは複数のAIアシスタント協業プラットフォームで、Claude Code、Codex、Copilot、OpenCodeなど、さまざまなAIアシスタントの連携をサポートしています。複数AIの連携やゲーム化プログラミングに興味がある場合は、[github.com/HagiCode-org/site](https://github.com/HagiCode-org/site)にアクセスして詳細をご確認ください。

実は特別なことは何もありません。ただ、私たちがプログラミングを冒険に変えただけです。

## なぜゲーム化を選ぶのか

ゲーム化の本質は「ポイントランキングを追加する」ことではなく、ユーザーがタスクをこなす過程で成長感、達成感、社会的承認を体験できるような完全なインセンティブシステムを構築することです。

HagiCodeのゲーム化デザインは、1つのコアコンセプトを中心に展開されています：各AIアシスタントは「ヒーロー」であり、ユーザーはこのヒーローチームのキャプテンです。あなたはこれらのヒーローを率いてさまざまな「ダンジョン」（プログラミングタスク）を征服し、その過程でヒーローは経験を得てレベルアップして能力をアンロックし、あなたとあなたのチームもさまざまな実績を得ます。

これは単なるギミックではなく、人間の行動心理学に基づいた慎重な設計です。タスクに意味と進捗フィードバックが与えられると、人の集中力と継続力は大幅に向上します。

古人が言ったように、「この情は追憶となるべく、ただ当時は既に惘然たり」。私たちはこの感情的体験をツールに組み込み、プログラミングを単なるコード入力から、思い出に残る旅にしました。

## Hero キャラクターシステム

HeroはHagiCodeゲーム化システムのコアコンセプトです。各HeroはAIアシスタントを表します。たとえば、Claude CodeはHeroであり、CodexもHeroです。

### Heroの3つのスロット

Heroには3つの装備スロットがあります。この設計は実に巧妙です：

1. **CLIスロット（メインクラス）**：Heroの基本能力を決定します。たとえば、Claude CodeかCodexか
2. **Modelスロット（サブクラス）**：使用するモデルを決定します。たとえば、Claude 4.5かClaude 4.6か
3. **Styleスロット（スタイル）**：Heroの行動スタイルを決定します。たとえば、「風落戦略家」か「その他のスタイル」か

3つのスロットの組み合わせで独自のHero設定を作成できます。ゲームでの装備設定と同じで、タスクの特性に応じて適切な組み合わせを選択する必要があります。結局のところ、自分に適したものが最高です。これは人生での道選びと似ています。すべての道はローマに通じますが、ある道は歩きやすく、ある道は少し曲がりくねっているだけです。

### Heroの成長システム

各Heroには独自のXP（経験値）とレベルがあります：

```typescript
type HeroProgressionSnapshot = {
  currentLevel: number;                    // 現在のレベル
  totalExperience: number;                 // 総経験値
  currentLevelStartExperience: number;     // 現在のレベルの開始経験値
  nextLevelExperience: number;             // 次のレベルに必要な経験値
  experienceProgressPercent: number;       // 進捗パーセンテージ
  remainingExperienceToNextLevel: number;  // 次のレベルまでの残り経験値
  lastExperienceGain: number;              // 最後に獲得した経験値
  lastExperienceGainAtUtc?: string | null; // 経験値を獲得した時間
};
```

レベルは4つの段階に分かれており、各段階の名前は非常に没入感があります：

```typescript
export const resolveHeroProgressionStage = (level?: number | null): HeroProgressionStage => {
  const normalizedLevel = Math.max(1, level ?? 1);
  if (normalizedLevel <= 100) return 'rookieSprint';      // 新人スプリント
  if (normalizedLevel <= 300) return 'growthRun';         // 成長ラン
  if (normalizedLevel <= 700) return 'veteranClimb';      // ベテランクライム
  return 'legendMarathon';                                // レジェンドマラソン
};
```

「新人」から「レジェンド」まで、この成長パスによりユーザーは明確な目標感と達成感を得られます。人生の成長と同じで、懵懂から成熟への過程を経る必要がありますが、ここではその過程を具体化しただけです。

### カスタムHeroの作成

Heroを作成する際には3つのスロットを設定する必要があります：

```typescript
const heroDraft: HeroDraft = {
  name: 'Athena',
  icon: 'hero-avatar:storm-03',
  description: '知略に優れた戦略家',
  executorType: AIProviderType.CLAUDE_CODE_CLI,
  slots: {
    cli: {
      id: 'profession-claude-code',
      parameters: { /* CLI関連パラメータ */ }
    },
    model: {
      id: 'secondary-claude-4-sonnet',
      parameters: { /* モデル関連パラメータ */ }
    },
    style: {
      id: 'fengluo-strategist',
      parameters: { /* スタイル関連パラメータ */ }
    }
  }
};
```

各Heroには独自のアバター、説明、専門的な定位があり、これにより冷たいAIアシスタントが個性と温かみを持つようになります。結局のところ、誰も性格のないツールと付き合いたくないでしょう？

## Dungeon ダンジョンシステム

「ダンジョン」はゲームの古典的な概念で、チームで攻略する必要のあるチャレンジを表します。HagiCodeでは、各ワークフローが1つのDungeonです。

### Dungeonの組織方式

Dungeonはワークフローを異なる「ダンジョン」に組織します：

- **提案生成ダンジョン**：技術提案の生成を担当
- **提案実行ダンジョン**：提案内のタスクの実行を担当
- **提案アーカイブダンジョン**：完了した提案の整理とアーカイブを担当

各ダンジョンには独自のCaptain（キャプテン）Heroがいます。キャプテンは有効なHeroから自動的に最初のものが選ばれます。

これは実は人生での分業と同じで、每个人都有自己的役割ですが、ここではこの分働をゲームメカニクスに変えただけです。

### チーム連携メカニズム

異なるダンジョンに対して異なるHeroチームを設定できます：

```typescript
const dungeonRoster: HeroDungeonRoster = {
  scriptKey: 'proposal.generate',
  displayName: '提案生成',
  members: [
    { heroId: 'hero-1', name: 'Athena', executorType: 'ClaudeCode' },
    { heroId: 'hero-2', name: 'Apollo', executorType: 'Codex' }
  ]
};
```

たとえば、提案を生成する際はAthena（戦略が得意）を使用し、コードを実行する際はApollo（実装が得意）を使用するなど、各ヒーローが自分の専門分野を発揮できます。バンドを編成するのと同じで、每个人都有自分の楽器を持っており、合わせて初めて美しい旋律を奏でることができます。

### Dungeonフロー制御

Dungeonは固定されたscriptKeyを使用して異なるフローを識別します：

```typescript
// スクリプトキーは異なるワークフローに対応
const dungeonScripts = {
  'proposal.generate': '提案生成',
  'proposal.execute': '提案実行',
  'proposal.archive': '提案アーカイブ'
};
```

タスク状態の遷移は：queued（待機中）→ dispatching（配信中）→ dispatched（配信済み）。プロセス全体が自動化されており、手動での介入は不要です。这也是我们偷懒的小心思，毕竟谁愿意手动管这些事呢。

## XPとレベルシステム

XP（経験値）はゲーム化システムで最もコアなフィードバックメカニズムです。ユーザーはタスクを完了することでXPを得て、XPでヒーローがレベルアップし、レベルアップで新しい能力がアンロックされ、正の循環が形成されます。

### XPの取得方法

HagiCodeでは、以下のアクティビティからXPを得ることができます：

- コード実行完了
- ツール呼び出し成功
- 提案生成
- セッション管理操作
- プロジェクト操作

有効な操作を完了するたびに、対応するHeroがXPを得ます。人生での成長と同じで、一歩一歩がカウントされますが、ここではその成長を定量化しただけです。

### リアルタイム進捗の可視化

XPとレベルの進捗はリアルタイムで可視化されます：

```typescript
type HeroDungeonMember = {
  heroId: string;
  name: string;
  icon?: string | null;
  executorType: PCode_Models_AIProviderType;
  currentLevel?: number;                    // 現在のレベル
  totalExperience?: number;                 // 総経験値
  experienceProgressPercent?: number;       // 進捗パーセンテージ
};
```

ユーザーはいつでも各Heroのレベルと進捗を確認でき、この即時フィードバックがゲーム化デザインの鍵です。毕竟人总是需要点反馈，不然怎么知道自己进步了呢？

## 実績システム

実績はゲーム化のもう一つの重要な要素で、長期目標とマイルストーン式的な達成感を提供します。

### 実績タイプ

HagiCodeは複数の実績タイプをサポートしています：

- **コード生成タイプ**：X行のコードを生成、Y個のファイルを生成
- **セッション管理タイプ**：Z回の会話を完了
- **プロジェクト操作タイプ**：W個のプロジェクトで作業

其实这些成就就像人生中的里程碑，只是我们把它们变成了一种游戏机制而已。

### 実績ステータス

実績には3つのステータスがあります：

```typescript
type AchievementStatus = 'unlocked' | 'in-progress' | 'locked';
```

3つのステータスには明確な視覚的区別があります：

- **アンロック済み**：金色グラデーション + 光暈効果
- **進行中**：青色パルスアニメーション
- **未アンロック**：灰色、アンロック条件を表示

各実績はトリガー条件を明確に表示し、ユーザーが次に何をすべきかを知ることができます。毕竟迷茫的时候，有个指引总是好的。

### アンロック時の祝福エフェクト

実績がアンロックされると、祝福アニメーションがトリガーされます。この正の強化により、ユーザーは「やった！」という満足感を得て、前進し続ける動機づけになります。就像生活中小小的奖励，虽然不大，却能让人开心很久。

## Battle Report 毎日戦闘レポート

Battle ReportはHagiCodeの特徴的な機能で、毎日終了時にフルスクリーンで表示される戦闘スタイルのレポートを生成します。

### レポート内容

Battle Reportは以下の情報を表示します：

```typescript
type HeroBattleReport = {
  reportDate: string;
  summary: {
    totalHeroCount: number;        // 総ヒーロー数
    activeHeroCount: number;       // アクティブヒーロー数
    totalBattleScore: number;      // 総戦闘スコア
    mvp: HeroBattleHero;           // 最も価値のあるヒーロー
  };
  heroes: HeroBattleHero[];        // すべてのヒーローの詳細データ
};
```

- チームの総スコア
- アクティブHeroの数
- ツール呼び出し回数
- 総作業時間
- MVP（最も価値のあるヒーロー）
- 各Heroの詳細カード

### MVPのハイライト表示

MVPはその日で最もパフォーマンスの良いHeroで、レポート内でハイライト表示されます。これは単なるデータ統計ではなく、栄誉の認識でもあります。毕竟谁不希望自己被认可呢？

### Heroの詳細カード

各Heroのカードには以下が含まれます：

- レベル進捗
- XP獲得量
- 実行回数
- 使用時間

これらのデータにより、ユーザーはチームの作業状態を明確に把握できます。毕竟了解自己的努力成果，也是一种满足感。

## 技術実装

HagiCodeのゲーム化システムは、現代的な技術スタックとデザインパターンを採用しています。其实也没什么特别的，只是选了一些趁手的工具而已。

### 技術スタックの選択

```typescript
// フロントエンドは React + TypeScript を使用
import React from 'react';

// アニメーションは Framer Motion を使用
import { AnimatePresence, motion } from 'framer-motion';

// 状態管理は Redux Toolkit を使用
import { useAppDispatch, useAppSelector } from '@/store';

// UI コンポーネントは shadcn/ui を使用
import { Dialog, DialogContent } from '@/components/ui/dialog';
```

Framer Motionはすべてのアニメーション効果を担当し、shadcn/uiは基本的なUIコンポーネントを提供し、Redux Toolkitは複雑なゲーム化状態を管理します。毕竟工欲善其事，必先利其器。

### ゲーム化UIデザインシステム

HagiCodeはGlassmorphism + Tech Darkのデザインスタイルを採用しています：

```css
/* メイングラデーション色 */
background: linear-gradient(135deg, #22C55E 0%, #25c2a0 50%, #06b6d4 100%);

/* ガラスモーフィズム効果 */
backdrop-filter: blur(12px);

/* 光輝効果 */
background: radial-gradient(circle at center, rgba(34, 197, 94, 0.15) 0%, transparent 70%);
```

緑色系のグラデーションとガラスモーフィズム効果の組み合わせで、テクノロジー感と未来感を演出します。毕竟视觉上的美感，也是用户体验的一部分。

### アニメーション効果の実装

Framer Motionを使用して滑らかな入場アニメーションを実現します：

```typescript
<motion.div
  animate={{ opacity: 1, y: 0 }}
  initial={{ opacity: 0, y: 18 }}
  transition={{ duration: 0.35, ease: 'easeOut', delay: index * 0.08 }}
  className="card"
>
  {/* カード内容 */}
</motion.div>
```

各カードが順に入場し、0.08秒の遅延で、滑らかな視覚効果を作り出します。毕竟流畅的动画能让体验更好，这是毋庸置疑的。

### データ永続化

ゲーム化データはGrainストレージシステムを使用し、状態の一貫性を確保します。HeroのXP累積のような細粒度のデータでさえ、正確に永続化できます。毕竟谁也不想让辛苦积累的经验丢失。

## 実践ガイド

### 最初のHeroを作成する

最初のHeroを作成するのは実に簡単です：

1. Hero管理ページに入る
2. 「Heroを作成」ボタンをクリック
3. 3つのスロット（CLI、Model、Style）を設定
4. Heroに名前と説明を付ける
5. 保存すると、最初のHeroが誕生します

新しい友達と知り合うのと同じで、名前を付けて特徴を理解し、一緒に冒険に出かけましょう。

### Dungeonチームを編成する

チームを編成するのも簡単です：

1. Dungeon管理ページに入る
2. 設定するダンジョン（「提案生成」など）を選択
3. Heroリストからメンバーを選択
4. システムが自動的に最初の有効なHeroをキャプテンとして選択
5. 設定を保存

これは実はチーム編成のプロセスで、人生でチームを編成するのと同じで、每个人都有自己的役割です。

### 毎日のレポートを確認する

毎日終了後、当日のBattle Reportを確認できます：

1. 「戦闘レポート」ボタンをクリック
2. 当日の作業成果をフルスクリーンで表示
3. MVPと各Heroの詳細データを確認
4. チームメンバーと共有（オプション）

这也是一种仪式感，让自己知道今天努力了多少，离目标还有多远。

## 注意事項とベストプラクティス

### パフォーマンス最適化

React.memoを使用して不要な再レンダリングを回避します：

```typescript
const HeroCard = React.memo(({ hero }: { hero: HeroDungeonMember }) => {
  // コンポーネント実装
});
```

毕竟性能也很重要，谁愿意用卡顿的工具呢？

### アニメーションのダウングレード

ユーザーの動き設定を検出し、敏感なユーザーに簡素化されたエクスペリエンスを提供します：

```typescript
const prefersReducedMotion = useReducedMotion();
const duration = prefersReducedMotion ? 0 : 0.35;
```

毕竟不是所有人都喜欢动画，尊重用户的偏好也是设计的一部分。

### 後方互換性

legacyIdsを保持して旧バージョンからの移行をサポートします：

```typescript
type HeroDungeonMember = {
  heroId: string;
  legacyIds?: string[];  // 旧バージョンIDマッピングをサポート
  // ...
};
```

毕竟谁也不希望因为版本升级就丢失数据。

### 国際化サポート

すべてのテキストはi18n翻訳キーを使用し、多言語サポートを容易にします：

```typescript
const displayName = t(`dungeon.${scriptKey}`, { defaultValue: displayName });
```

毕竟语言不应该成为使用的障碍。

## まとめ

ゲーム化は単純なポイントランキングではなく、完全なインセンティブシステムです。HagiCodeはHeroシステム、Dungeonシステム、XP/レベルシステム、実績システム、Battle Reportを通じて、プログラミング作業を冒険精神に満ちたヒーローの旅に変えます。

このシステムのコア価値は以下の通りです：

- **感情的つながり**：冷たいAIアシスタントに個性を持たせる
- **正のフィードバック**：各操作に即時フィードバックがある
- **長期目標**：レベルと実績が成長パスを提供
- **チームアイデンティティ**：Dungeonチーム連携感
- **栄誉の認識**：Battle ReportとMVPの表示

ゲーム化デザインにより、プログラミングは退屈なものから面白い冒険になります。ユーザーはコードタスクを完了しながら、キャラクターの成長、チーム連携、実績アンロックの楽しみを体験し、使用の定着性とアクティビティを向上させます。
