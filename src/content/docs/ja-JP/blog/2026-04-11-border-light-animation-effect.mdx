## ボーダー光ループアニメーション効果実装ガイド

> ユーザーの目を引く重要な要素を、純粋なCSSで実装する方法とは？実はそれほど難しくありません。この記事では、ゼロからボーダー光ループアニメーションを実装する方法をご紹介します。また、HagiCodeプロジェクトで直面した課題についてもお話しします。

## 背景

フロントエンド開発者なら、誰もがこんな経験を持っているはずです。プロダクトマネージャーが「この要件は簡単です」というような表情で近づいてくる。「実行中のタスクに、ユーザーが一目で分かるようなエフェクトを追加できない？」

「もちろん、ボーダーの色を変えるだけでいいですね」と答えると、相手は首を横に振り、「あなたは分かっていない」というような眼差しを向けてきます。「目立ちません。光がボーダーを回るような効果が必要です。SF映画のように。」

この時、あなたはこう思うかもしれません。「これをどう実装するんだ？Canvas？SVG？それともCSSでできるのか？」結局のところ、自分ができないとは認めたくないものです。

実のところ、ボーダー光ループアニメーションは現代のWebアプリケーションで非常に一般的で、主に以下のようなシナリオで使用されています：

- **ステータスインジケーター**：実行中のタスクやアクティブなプロジェクトをマーク
- **ビジュアルフォーカス**：重要なコンテンツエリアを強調表示
- **ブランド強化**：技術感と現代的なビジュアル体験を演出
- **祝日テーマ**：特別な祝日に合わせた祝祭雰囲気の作成

私たちが[HagiCode](https://hagicode.com)を作る際、この要件に直面しました。ユーザーはどのセッションが実行中か、どの提案が処理中かを一目で区別できる必要があります。いくつかのアプローチを試しました。道が平坦なものもあれば、少し曲折するものもありましたが、最終的に比較的成熟した実装アプローチを確立しました。

## コア実装アプローチ

HagiCodeのコードを分析した結果、以下のいくつかのコア実装パターンをまとめました。それぞれに適用シナリオがある、あるいは、それぞれに存在意義があると言えるでしょう。

### 1. Conic Gradient回転光輪（最も一般的）

これは最も古典的なボーダー光ループ実装方式で、コアのアイデアはCSSの`conic-gradient`を使って円錐状グラデーションを作成し、それを回転させることです。夜の街灯のように、ずっと回り続けるのです。

**主要要素：**
- `::before`擬似要素で光輪層を作成
- `conic-gradient`でグラデーション色分布を定義
- `::after`擬似要素で中心領域をマスク（オプション）
- `@keyframes`で回転アニメーションを実装

### 2. 側面発光ライン

これはリストアイテムのステータスインジケーターに適しています。要素の片側に発光する細いラインを作成すれば、ボーダー全体を動かす必要はありません。時には、少しの光で十分で、全世界を照らす必要はないのです。

**主要要素：**
- 絶対配置された細いライン要素
- `box-shadow`で発光効果を作成
- `scale`と`opacity`で呼吸アニメーションを実装

### 3. Box-Shadow発光背景

ループ効果が不要で、柔らかな背景光輪だけが必要な場合、複数の`box-shadow`を重ねるだけで十分です。時には、シンプルな方が良いこともあります。

### 4. アクセシビリティサポート

これは見落とされがちですが、非常に重要です。すべてのアニメーションは`prefers-reduced-motion`メディアクエリを考慮し、アニメーションを好まないユーザーに静的な代替案を提供すべきです。結局のところ、すべての人が動きのあるものを好むわけではなく、それぞれの選択を尊重することが正しいのです。

## 実装ソリューション

### ソリューション1：Conic Gradient回転ボーダー（推奨）

これは最も完全なループ光効果実装で、HagiCodeで最も多く使用されているソリューションです。結局のところ、一つのものが良く使えるなら、なぜ変える必要があるのでしょうか？

```css
/* 親コンテナ */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* 回転する光輪層 */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* マスク層（オプション、中空ボーダー効果を作成するため） */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

このソリューションの原理はとてもシンプルです。親コンテナよりも大きな擬似要素を作成し、その上に円錐状グラデーションを描き、回転させ続けます。親コンテナに`overflow: hidden`を設定するので、ボーダーの部分の光だけが回転しているのが見えます。窓から外の街灯を見るようなもので、通過する一小節しか見えないのです。

### ソリューション2：簡易版回転光ボーダー

それほど複雑な効果が不要な場合、HagiCodeにはより軽量なユーティリティクラス実装があります。結局のところ、シンプルな方が良いこともあります。

```css
/* 回転光ボーダーユーティリティクラス */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* アクセシビリティサポート */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

ここの`will-change: transform`に注目してください。これはブラウザに「この要素は常に変化する」と伝えるもので、ブラウザは事前に最適化を行い、アニメーションがより滑らかになります。結局のところ、事前準備は临时抱佛脚（泥縄）より良いのです。

### ソリューション3：側面発光ライン

リストアイテムのステータスインジケーターにはこれが特に適しています。HagiCodeのセッションリストはこのソリューションを使用しています。一本の細いラインですが、多くのプロジェクトの中で際立つことができます。これもある種の人生哲学ではないでしょうか？

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

ここでは`isolation: isolate`を使って新しいスタッキングコンテキストを作成し、`z-index`で各層の表示順序を制御しています。`pointer-events: none`も重要で、そうしないとその擬似要素がユーザーのクリック操作を妨げてしまいます。あるものは見た目は良いですが、邪魔にならないことが重要です。

### ソリューション4：Reactコンポーネントカプセル化

プロジェクトでReactを使用している場合、これらのロジック、特にアクセシビリティの部分を処理するコンポーネントをカプセル化できます。結局のところ、コードは一度書いて何度も使う、これが私たちが望むものです。

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

対応するCSSモジュール：

```css
/* GlowBorder.module.css */

/* アニメーションバージョン */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* 静的バージョン（アクセシビリティ） */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

`framer-motion`の`useReducedMotion`フックは自動的にユーザーのシステム設定を検出し、ユーザーが「動作を減らす」を有効にしている場合、trueを返し、その際は静的バージョンを表示します。結局のところ、ユーザーの選択を尊重することは、無理に表示することよりも重要です。

## 実践経験の共有

以下は、HagiCodeを作る際に直面した課題と、そこから得た経験です。実のところ、ただの雑談のようなものですが、後続のあなたの役に立つことを願っています。

### 1. テーマ変数システム

CSS変数を使ってマルチテーマ対応を実装すると特に便利です。結局のところ、テーマを切り替えるたびにコードを大量に修正したくはないものです。

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* 使用 */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

こうすれば、テーマを切り替えるときに`html`タグのclassを変更するだけで、すべてのアニメーション色が自動的に更新されます。一つのコード、二つのスタイル、これこそが私たちが追求するものではありませんか？

### 2. パフォーマンス最適化

**`will-change`を使ってブラウザに最適化を促す：**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

事前にブラウザに伝えておけば、最適化を行ってくれます。人生の多くのことのように、事前準備は常に良いものです。

**大面積要素で複雑なbox-shadowを使用するのを避ける：**

```css
/* 悪い例 - 大面積要素でぼかし影を使用 */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* より良い - 擬似要素を使って発光領域を制限 */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

私たちはHagiCodeでテストしました。大きなカードに直接ぼかし影を追加すると、スクロールフレームレートが30fps以下に落ちましたが、擬似要素に変更すると安定して60fpsになりました。このような体験の違いは、ユーザーが感じることができます。

### 3. アクセシビリティ

これは本当に省略できません。一部のユーザーはアニメーションを眩しく感じたり、うるさく感じたりします。彼らの選択を尊重することは、製品を作る基本的な素養です。結局のところ、美しいものを無理強いすべきではありません。

**CSSメディアクエリ：**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* 静的な代替案を提供 */
    opacity: 1;
  }
}
```

**Reactでユーザー設定を検出：**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. 強度レベル制御

HagiCodeのTokenスループットインジケーターは、リアルタイムスループットに基づいて異なる色の光を表示します。これは動的に実装されています。結局のところ、異なる状態には異なる表現方法が必要です。

```tsx
const colors = [
  null,       // Level 0 - 色なし
  '#3b82f6',  // Level 1 - 青
  '#34d399',  // Level 2 - エメラルド
  '#facc15',  // Level 3 - 黄
  '#fbbf24',  // Level 4 - アンバー
  '#f97316',  // Level 5 - オレンジ
  '#22d3ee',  // Level 6 - シアン
  '#d946ef',  // Level 7 - フクシア
  '#f43f5e',  // Level 8 - ローズ
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. 注意事項

いくつかの詳細に注意する必要があります。さもないと、落とし穴に落ちてからでは遅すぎます。

| 注意事項 | 説明 |
|---------|------|
| **z-index管理** | 光輪層は適切なz-indexを設定し、コンテンツの相互作用に影響しないようにする |
| **pointer-events** | 光輪擬似要素には`pointer-events: none`を設定する |
| **境界オーバーフロー** | 親コンテナに`overflow: hidden`を設定するか、擬似要素のサイズを調整する |
| **パフォーマンス影響** | 複雑なアニメーションはモバイルデバイスでパフォーマンスに影響する可能性があるため、テストが必要 |
| **ダークモード** | ダーク背景で発光色がはっきりと見えることを確認する |
| **テーマ切り替え** | CSS変数を使用し、テーマ切り替え時にアニメーション色が正しく更新されるようにする |

### 6. デバッグのヒント

擬似要素は開発者ツールで見つけにくいことがあります。一時的に境界線を追加して位置を確認できます。

```css
/* デバッグ用に擬似要素の境界線を一時的に表示 */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

位置を調整した後、この行をコメントアウトまたは削除することを忘れないでください。そうしないと、本番環境で気まずいことになります。あるものは、開発環境に留めておくのが良いのです。
