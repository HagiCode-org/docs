## .NET コード保護実戦：難読化から仮想マシン保護へ

> 本記事では、.NET プロジェクトで多層的なコード保護戦略を実装する方法を紹介します。基本的な難読化から専門的な仮想マシン保護まで、全てのソリューションを網羅します。

## 背景

.NET アプリケーション開発において、コアコード（ライセンス検証、ビジネスロジック、機密設定など）を逆コンパイルやリバースエンジニアリングから保護することは、どういうわけか、避けて通れない話題です。.NET エコシステムの成熟に伴い、開発者は組み込みの難読化属性から専門的な仮想化保護ツールまで、さまざまなコード保護手段を持つようになり、選択肢はかなり豊富です。

複雑な多言語 monorepo プロジェクトとして、HagiCode はデスクトップアプリケーション、ビルドシステム、ライセンス管理機能を含んでいます。コードには避けられない形でライセンス検証ロジック、機密設定（API キー、製品 ID など）、およびビジネスコアロジックが含まれており、これらはしっかり保護する必要があります。毕竟誰も自分の心血を簡単に他人に見られたくないでしょう。

本記事では、HagiCode プロジェクトで実際に採用しているコード保護ソリューションを共有し、試行錯誤から最適化に至るまでの完全なプロセスをまとめます。何らかのヒントになれば幸いです。

## HagiCode について

[HagiCode](https://hagicode.com) はオープンソースの AI コードアシスタントプロジェクトであり、開発者にインテリジェントなプログラミング支援体験を提供することを目指しています。プロジェクトは monorepo アーキテクチャを採用し、VSCode 拡張、バックエンド AI サービス、クロスプラットフォームデスクトップクライアントなど、複数のコンポーネントを同時に保守しています。このような多言語・多プラットフォームの複雑さにより、コード保護は直面しなければならないエンジニアリング課題となっています。仕方ない、プロジェクトがこれほど複雑なので。

本記事で共有するソリューションは、まさに HagiCode の開発過程で実際に試行錯誤し、実際に最適化したものです。私たちがどのようにこれらの技術的課題を解決したかに興味がある場合は、読み進めてください。予期せぬ発見があるかもしれません。

## コアコンテンツ

### 1. マイクロソフト組み込み難読化 Attribute

.NET Framework は組み込みの `[ObfuscationAttribute]` を提供しており、これが最も基本的かつ最も一般的に使用されるコード難読化マークです。この属性は `System.Reflection` 名前空間にあり、サードパーティツールを導入することなくコードを基本保護できるため、かなり便利です。

**コア機能**：
- `Feature` プロパティ：難読化機能を指定。`"ultra"`（高度な難読化）、`"all"`（全ての難読化）など
- `Exclude` プロパティ：`true` は難読化を除外、`false` は難読化を適用
- クラス、メソッド、プロパティなどの型メンバーに適用可能

HagiCode プロジェクトでは、実際の使用例を見ることができます：

```csharp
[Obfuscation(Feature = "ultra", Exclude = false)]
public async Task<LicenseValidationResult?> ValidateLicenseAsync(...)
```

この方法の**メリット**はかなり明らかです：
- 追加の依存関係が不要、.NET Framework に組み込まれている、手間が省ける
- サードパーティの難読化ツールによって認識・処理可能
- コンパイル後のアセンブリサイズを大幅に増加させない

ただし、制限もあります。単なるマークであり、実際の難読化効果はツールの実装に依存します。仮想マシン保護レベルのセキュリティは提供できません。まあ、元々そのために作られたものではないので、仕方ないでしょう。

### 2. VMP（Virtual Machine Protection）

VMP は専門的なコード保護ツールであり、コードを仮想マシン命令にコンパイルすることで高度な保護を提供します。単純な名前の難読化とは異なり、VMP はコードロジックを通常の逆コンパイラーでは復元できない形式に実際に変換します。この点はかなり優れています。

**保護レベル分類**：

| レベル | 仮想化 | 変異 | アンチデバッグ | 文字列暗号化 | 適用シナリオ |
|------|--------|------|--------|------------|----------|
| HIGH | full | high | 有効 | 有効 | ライセンス検証、セッション並行性、機密定数 |
| MEDIUM | partial | medium | 有効 | 有効 | ビジネスロジック、ドメインモデル |
| LOW | none | low | 無効 | 無効 | ユーティリティクラス、非重要コード |

HagiCode プロジェクトでは、保護が必要なコードをマークするために宣言的属性システムを定義しています：

```csharp
// 高優先保護
[VmProtect(VmProtectionPriority.High, Reason = "Contains license verification logic")]
public class KeygenClient { ... }

// 保護除外
[VmExclude(Reason = "Public API that must remain unchanged")]
public class PublicApi { ... }

// 継承保護
[VmProtect(Priority.High, ProtectDerived = true)]
public class BaseLicenseValidator { ... }
```

### 3. ビルド時保護戦略

VMP 保護は実行時に有効であるだけでなく、ビルドプロセスで自動的に処理する必要があります。手動でやるにはあまりに面倒すぎます。HagiCode のビルドシステムは複数のモードをサポートしています：

- **Windows ネイティブモード**：VMProtect ツールを直接呼び出す
- **Linux Docker コンテナモード**：コンテナ内で VMP を実行する（クロスプラットフォーム互換性問題を解決）
- **Attribute スキャン**：コード内の保護マークを自動発見
- **検証メカニズム**：保護が正常に適用されたことを確認

これらの機能を組み合わせることで、かなり安心です。

## ソリューション

### 1. マイクロソフト組み込み難読化 Attribute の使用

コード内で `ObfuscationAttribute` を直接適用します：

```csharp
using System.Reflection;

[Obfuscation(Feature = "ultra", Exclude = false)]
public class LicenseService
{
    [Obfuscation(Feature = "ultra", Exclude = false)]
    public async Task<bool> ValidateLicenseAsync(string key)
    {
        // ライセンス検証ロジック
    }

    [Obfuscation(Feature = "flow", Exclude = false)]
    private string DecryptToken(string encrypted)
    {
        // 復号ロジック
    }
}
```

テストアセンブリが内部メンバーにアクセスできるようにしつつ、本番コードのセキュリティを維持する必要がある場合があります：

```csharp
// AssemblyInfo.cs
[assembly: InternalsVisibleTo("HagiCode.Application.Tests")]
[assembly: InternalsVisibleTo("DynamicProxyGenAssembly2")] // for Moq
```

これでテストがはるかに便利になります。毕竟 コードはテストする必要があります。

### 2. VMP 保護のカスタム Attribute 定義

VMP の動作を制御するためにカスタム保護属性を作成します：

```csharp
using System;

[AttributeUsage(AttributeTargets.Class | AttributeTargets.Method | AttributeTargets.Property)]
public class VmProtectAttribute : Attribute
{
    public VmProtectionPriority Priority { get; set; }
    public string? Reason { get; set; }
    public bool ProtectDerived { get; set; }
}

[AttributeUsage(AttributeTargets.Class | AttributeTargets.Method | AttributeTargets.Property)]
public class VmExcludeAttribute : Attribute
{
    public string? Reason { get; set; }
}

public enum VmProtectionPriority
{
    None = 0,
    Low = 1,
    Medium = 2,
    High = 3
}
```

自分で定義した属性の方が使いやすいです。毕竟 自分のニーズを理解しています。

### 3. VMP 設定ファイル

```yaml
# vmp_config.yml
protection:
  priority_mode: "attribute"  # Attribute に基づく優先度
  default_level: "medium"

  - name: "vmprotect"
    path: "C:\\Program Files\\VMProtect Ultimate\\VMProtect.exe"

protection_levels:
  high:
    virtualization: "full"
    mutation: "high"
    anti_debug: true
    anti_dump: true
    encrypt_strings: true
    encrypt_resources: true

  medium:
    virtualization: "partial"
    mutation: "medium"
    anti_debug: true
    encrypt_strings: true

  low:
    virtualization: "none"
    mutation: "low"
    anti_debug: false
```

設定は明確に書いておけば、後の保守も便利です。

## 実践ガイド

### 1. 重要コンポーネント保護の実践

HagiCode の code-protection 規格に従い、以下のコンポーネントは HIGH 優先度保護を使用する必要があります：

```csharp
// 本番環境定数 - 暗号化し VMP 保護が必要
[VmProtect(VmProtectionPriority.High, Reason = "Production constants")]
public static class ProductionConstants
{
    // 暗号化された文字列アクセサ、VMP によって保護される
    [VmProtect(VmProtectionPriority.High)]
    public static string GetLicenseServerUrl(IOptions<LicenseOptions> options) => ...;
}

// ライセンス検証ロジック
[VmProtect(VmProtectionPriority.High, Reason = "License verification logic")]
public class KeygenClient : IKeygenClient
{
    [Obfuscation(Feature = "ultra", Exclude = false)]
    public async Task<LicenseValidationResult?> ValidateLicenseAsync(...) { ... }
}

// マシンフィンガープリントサービス
[VmProtect(VmProtectionPriority.High)]
public class MachineFingerprintService : IMachineFingerprintService { ... }
```

重要なコードは重点的に保護する必要があります。毕竟 コアロジックが漏洩すると面倒です。

### 2. 文字列暗号化ランタイム復号

ビルド時に文字列を暗号化し、実行時に復号します：

```csharp
public static class StringDecryption
{
    [VmProtect(VmProtectionPriority.High, Reason = "CRITICAL SECURITY")]
    public static string DecryptString(byte[] encryptedData, byte[] key, byte[] iv)
    {
        using var aes = Aes.Create();
        aes.Key = key;
        aes.IV = iv;

        using var decryptor = aes.CreateDecryptor();
        using var ms = new MemoryStream(encryptedData);
        using var cs = new CryptoStream(ms, decryptor, CryptoStreamMode.Read);
        using var reader = new StreamReader(cs);

        return reader.ReadToEnd();
    }
}

// 本番定数アクセサ（遅延読み込み + キャッシュ）
public static class ProductionConstants
{
    private static string? _cachedLicenseServerUrl;

    public static string GetLicenseServerUrl(IOptions<LicenseOptions> options)
    {
        if (_cachedLicenseServerUrl == null)
        {
            var encrypted = GetEncryptedLicenseServerUrl();
#if DEBUG
            _cachedLicenseServerUrl = options.Value.PrimaryServer.Url;
#else
            _cachedLicenseServerUrl = StringDecryption.DecryptString(
                encrypted,
                GetEncryptionKey(),
                GetEncryptionIV());
#endif
        }
        return _cachedLicenseServerUrl;
    }
}
```

文字列暗号化もかなり重要なステップです。毕竟 機密情報を平文で置いておくわけにはいきません。

### 3. VMP 保護の検証

ビルド後に保護が正常に適用されたことを検証する必要があります。そうでないと、保護が有効かどうかわかりません：

```csharp
// 検証スクリプトの例
public bool VerifyProtection(string assemblyPath)
{
    // 1. VMP 署名をチェック
    var bytes = File.ReadAllBytes(assemblyPath);
    var vmpSignature = Encoding.ASCII.GetBytes("VMProtect");
    if (bytes.Any(b => vmpSignature.Contains(b)))
    {
        return true;
    }

    // 2. ファイルサイズの変化をチェック（保護後は通常増加）
    var originalInfo = new FileInfo(assemblyPath.Replace(".dll", ".bak"));
    if (originalInfo.Exists)
    {
        var sizeRatio = (double)new FileInfo(assemblyPath).Length / originalInfo.Length;
        return sizeRatio > 1.1;
    }

    return false;
}
```

検証しておくと良いです。問題が発生したときに気づけないから。

### 4. 注意事項

ここでは特に注意すべき落とし穴がいくつかあります。毕竟 私たちも全て踏んできたのです：

1. **全てのコードを難読化しないでください**：パブリック API、インターフェース定義、DTO クラスは通常保護不要です。過度な難読化はパフォーマンスとデバッグに影響します。HagiCode プロジェクトはこの点で苦い経験があります。注意が必要です。

2. **キーアクセサを保護してください**：暗号化キーの取得メソッドは、暗号化データと同等以上の保護レベルを受ける必要があります。そうでなければ形だけのものになり、意味がありません。

3. **テストと本番のバランス**：DEBUG ビルドは開発デバッグを容易にするために暗号化をスキップし、RELEASE ビルドは完全な保護を有効にします。条件コンパイル `#if DEBUG` を使用して区別することを覚えておいてください。こうすれば開発も便利です。

4. **Docker 環境の考慮**：Linux 環境で VMP を実行するにはコンテナ化ソリューションを使用する必要があります。保護ツールの互換性を確保してください。HagiCode は Wine + VMP のコンテナソリューションを使用しており、クロスプラットフォームの問題はこのように解決されています。

5. **検証は必須**：ビルド完了後に保護が成功したことを検証する必要があります。そうでないと機密コードが露出する可能性があります。前述の検証コードがこの役割を果たします。やはりチェックした方が良いです。

## まとめ

このような多層保護戦略により、HagiCode は基本的な難読化から仮想マシン保護までの包括的なコードセキュリティシステムを実現しています：

- **第 1 層**：`ObfuscationAttribute` を使用して基本マークを行い、サードパーティツールにヒントを提供
- **第 2 層**：カスタム `VmProtectAttribute` を通じて保護意図と優先度を宣言
- **第 3 層**：VMP 仮想マシン保護が重要なコードを不可逆な仮想マシン命令に変換
- **第 4 層**：ビルド時に自動スキャンで保護を適用し、保護結果を検証

このソリューションは、一般的な逆コンパイルツールだけでなく、高度なリバースエンジニアリング攻撃にも対抗できます。コード保護が必要な .NET アプリケーションを開発している場合、参考になれば幸いです。ほんの少しのヒントでも十分です。
