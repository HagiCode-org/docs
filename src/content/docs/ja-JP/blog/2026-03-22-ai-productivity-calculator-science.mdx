## Agent に淘汰される気がしたので、データで計算してみた

> データで AI 代替リスクを量化：HagiCode チームが 6 つのコア公式で知識ワーカーの競争力評価基準を再定義する方法を徹底解説。

## 背景

AI 技術が急速に発展している今日、すべての知識ワーカーは切迫した問題に直面しています：**AI 時代において、私は淘汰されるのか？**

この問題は少し大げさに聞こえるかもしれませんが、実は多くの人が心の中で不安を感じています。あるフレームワークを覚えたばかりなのに、AI が「この業務は代替される」と言う。ある言語をようやくマスターしたと思ったら、AI を使う人の生産性は自分の 3 倍であることに気づく——このような不安感は、画面の前にいるあなたも多少感じたことがあるでしょう。

実は、この不安も無理从はありません。結局のところ、誰も長年積み上げてきたスキルが ChatGPT 一つで超越される可能性を認めたくないからです。不安は不安として、日々の生活は続けなければなりません。

従来の見方はしばしば「AI が何にできるか」という観点から代替リスクを論じますが、この方法は 2 つの重要な次元を見落としています：

1. **企業視点**：企業が従業員に AI ツールを提供するかどうかは、AI コストの人件コストに対するコストパフォーマンスに依存します。AI がその業務を代替できるからといって、企業がすぐに人を入れ替えるわけではありません。結局、経済的な計算をする必要があります。資本家は慈善家ではないので、すべてのお金を最適な場所に使う必要があるからです。

2. **効率視点**：AI による効率向上は量化される必要があり、単に「AI を使えばより強い」と単純に考えるべきではありません。あなたが AI を使って効率が 2 倍になったとしても、彼は AI を使って 5 倍になった——この差は小さくありません。学生時代のように、同じ授業を受けていても、ある人は 90 点、ある人はギリギリ合格——こうやって差がつくのです。

したがって、重要な問題は：**この曖昧な不安を、量化可能な指標に変えるにはどうすればよいか？**

自分の位置を知っていることは、暗闇の中で模索するよりも良いでしょう。これが今日話し合う内容です——HagiCode チームが開発した AI 人材効率計算機の背後にある設計ロジック。

そこで、私は https://cost.hagicode.com というウェブサイトを作りました。

## コア公式：6 つの重要指標

### 1. 年度総雇用コスト

企業が従業員に支払う真のコストは給与だけではありません。この点は多くの人が転職時に初めて気づきます——明確に月給 2 万元で話し合ったのに、手取りはどうして 1 万 4 千元なのか？企業側は 2 万元以上を支払っています。社会保険、住宅積立金、研修、採用コストもすべて計算する必要があります。

`calculate-ai-risk.ts` の実装に基づくと：

```
年度総雇用コスト = 年給 × (1 + 都市係数) + 年給 / 12
```

都市係数は異なる都市の人才採用と確保のコストを反映します：

| 都市レベル | 代表都市 | 係数 |
|---------|---------|------|
| 一級 | 北京/上海/深圳/広州 | 0.4 |
| 新一級 | 杭州/成都/蘇州/南京 | 0.3 |
| 二級 | 武漢/西安/天津/鄭州 | 0.2 |
| その他 | 宜昌/洛陽など | 0.1 |

一級都市の係数は 0.4 で、企業が追加で約 40% の採用、研修、社会保険などの付加コストを支払う必要があることを意味します。北京で一人を採用する総合コストは、確かに二級都市よりかなり高いです。

結局、大都市で生存するには生活コストも高く、これは別の形の「漂流者税」と言えるでしょう。

### 2. 総合 Token 単価

異なる AI モデルには Input と Output の 2 種類の価格があり、その差は巨大です。コードシナリオでは入出力比は約 3:1 です——AI にコードを渡してレビューさせると、出力される分析文は通常、入力されたコードよりかなり短くなります。

総合単価計算式：

```
総合単価 = (入出力比 × 入力単価 + 出力単価) / (入出力比 + 1)
```

GPT-5 を例に挙げます：
- 入力：$2.5/1M tokens
- 出力：$15/1M tokens
- 総合 = (3 × 2.5 + 15) / 4 = $5.625/1M tokens

USD 価格設定のモデルについては、為替レートで換算する必要があります。この為替レートは HagiCode チームが 7.25 に設定しており、市場の変動に合わせて更新されます。

為替レートは株式市場と同じで、誰にも正確には予測できません。ただ従うしかありません。

### 3. AI 年間コスト

```
1 日平均 AI コスト = 1 日平均 Token 需要量 (M) × 総合単価 (CNY/1M)
年間 AI コスト = 1 日平均 AI コスト × 264 勤務日
```

264 = 22 日/月 × 12 月で、標準勤務制度下の年間勤務日数です。なぜ 365 日を使わないのか？週末、祝日、病欠などの要素を考慮する必要があるからです。

結局、私たちはロボットではないので、休むときは休む必要があります。AI は休憩を必要としないかもしれませんが、私たちは自分に少しの呼吸スペースを残す必要があります。

### 4. コアイノベーション：等価人材

これは評価システム全体のコアであり、HagiCode チームが最も洞察力を持つ点です。

```
負担可能ワークフロー数 = 年度総雇用コスト / AI 年間コスト
負担可能比率 = min(負担可能ワークフロー数, 1)
等価人材 = 1 + (効率倍率 - 1) × 負担可能比率
```

待って、この公式は少し複雑なので、説明させてください：

従来の見方は単に「あなたの効率は 2 倍向上した」と言いますが、この公式は一つの重要な制約を考慮しています：**企業の AI 予算は持続可能か？**

例を挙げます：小明の効率は 3 倍向上しましたが、彼の AI 消費コストは毎年 30 万元かかります。一方、会社が彼に支払う年給は 20 万元だけです。この場合、小明個人の効率は非常に高いですが、彼は実際には持続不可能です——会社が彼に高効率を維持させるために損失を出し続けることは不可能です。

負担可能比率はまさにこの意味です：もし企業が 0.5 份の AI ワークフローしか負担できないなら、小明の等価人材 = 1 + (3-1) × 0.5 = 2 人であって、3 人ではありません。

**コア洞察**：あなたの効率倍率がどれほど高いかではなく、企業がその効率を維持するために必要な AI 投入を負担できるかどうかです。

実はこの道理もかなり単純で、ただ多くの人はこの方面を考えていないだけです。結局、私たちは自分の視点から物事を見ることに慣れていて、めったにボスの視点から考えません——彼らのお金も風が吹いて降ってきたものではありません。

### 5. コストパフォーマンス比

```
AI コスト占有率 = AI 年間コスト / 年度総雇用コスト
効率向上幅 = 効率倍率 - 1
コストパフォーマンス比 = 効率向上幅 / AI コスト占有率
```

- コストパフォーマンス比 < 1：AI 投入は割に合わず、効率向上がコストに見合わない
- コストパフォーマンス比 1-2：ちょうど割に合う
- コストパフォーマンス比 > 2：高収益、強く推奨

この指標は企業管理者にとって特に有用で、ある業務に AI ツールを投入する価値があるかどうかを迅速に評価できます。

結局、ROI こそが王道です。自分の効率がどれほど向上したと言っても、コストが爆発すれば誰も納得しません。

### 6. リスクレベル

等価人材に基づいてリスクを分類します：

| 等価人材 | リスクレベル | 結論 |
|---------|---------|------|
| >= 2.0 | 高リスク | 同僚が同等の条件を持つと、あなたへの脅威が非常に高い |
| 1.5 - 2.0 | 警告 | 同僚がすでに明らかな効率優位性を形成し始めている |
| < 1.5 | 安全 | 一時的にはまだ差を維持できる |

この表を見て、心の中でだいたいの数がつかめたでしょう。ただ、あまり不安にならないでください。結局、不安も問題を解決できないので——むしろ自分の効率倍率をどう向上させるか考えてみましょう。

## ゲーム化デザイン：7 種類の特殊称号

評価結果をより面白くするために、計算機は 7 種類の特殊称号システムを導入しました。称号は `localStorage` で永続化され、ユーザーは自分の「実績」をアンロックして表示できます。

| 称号 ID | 名称 | 取得条件 |
|--------|------|---------|
| craftsman-spirit | 職人精神 | 1 日平均 Token = 0 |
| prompt-alchemist | プロンプト錬金術師 | 1 日 Token \<= 20M 且つ効率倍率 \>= 6 |
| all-in-operator | 全オペレーター | 1 日 Token \>= 150M 且つ効率倍率 \>= 3 |
| minimalist-runner | ミニマリストランナー | 1 日 Token \<= 5M 且つ効率倍率 \>= 2 |
| cost-tamer | コストテイマー | コストパフォーマンス比 \>= 2.5 且つ AI 占有率 \<= 15% |
| danger-oracle | 危険予言者 | 等価人材 \>= 2.5 または高リスク圏に入る |
| budget-coordinator | 予算コーディネーター | 負担可能ワークフロー数 \>= 8 |

各称号の背後には隠された意味があります：

| 称号 | 隠された意味 |
|-----|---------|
| 職人精神 | AI を使わなくてもうまくやれるが、独自の競争力が必要 |
| プロンプト錬金術師 | 少量の Token で高生産性を達成、ギーク型ユーザー |
| 全オペレーター | 高投入高生産性、高頻度シナリオに適している |
| ミニマリストランナー | 軽量級 AI 使用、軽度補助シナリオに適している |
| コストテイマー | ROI が極めて高く、企業が最も好む従業員タイプ |
| 危険予言者 | あなたはすでにまたはまもなく高リスク群 |
| 予算コーディネーター | あなたは複数の AI ワークフローを同時に運営できる |

実はゲーム化这个东西、说白了就是给枯燥的数据加点趣味性罢了。毕竟谁不喜欢收集成就呢？就像游戏里的徽章、虽然没啥实际用处、但看着心里就是舒服。

## 実際の使用例

あなたが北京の開発者で、年給 40 万元、Claude Sonnet 4.6 を使用し、1 日平均 Token 消費量 50M、自己評価効率向上 3 倍だと仮定します。シミュレーション入力：

```typescript
const input = {
  annualIncomeCny: 400000,
  cityTier: "tier1",           // 北京
  modelId: "claude-sonnet-4-6",
  performanceMultiplier: 3.0,
  dailyTokenUsageM: 50,
}

// 計算プロセス
// 年度総雇用コスト = 40万 × (1 + 0.4) + 40万/12 ≈ 60.33万
// AI 年間コスト ≈ 50 × 7.125 × 264 ≈ 9.4万
// 負担可能ワークフロー数 ≈ 60.33 / 9.4 ≈ 6.4 份
// 等価人材 = 1 + (3 - 1) × 1 = 3 人
```

結論：あなたの同僚が同じ条件を持つ場合、3 人分の生産性に相当し、あなたはすでに高リスク圏にいます。

もし自分の AI の使い方が「割に合わない」（コストパフォーマンス比 < 1）ことに気づいた場合は、以下を検討できます：

1. **Token 消費を削減する**：より効率的なプロンプトを使用し、無効なリクエストを減らす
2. **コストパフォーマンスの高いモデルを選択する**：DeepSeek-V3 など（人民元建てでより安い）
3. **効率倍率を向上させる**：高度な Agent 使用テクニックを学び、AI を本当に生産性に変える

実はこれらの問題は、結局のところバランスの芸術にすぎません。使いすぎるとお金を浪費し、使わなすぎると効果がない——そのちょうど良い点を見つけるのが鍵です。

## 技術アーキテクチャのハイライト

HagiCode チームがこの計算機を設計する際、参考に価値のあるいくつかのエンジニアリング決定がありました：

1. **純粋なフロントエンド計算**：すべての計算はブラウザで完了し、バックエンド API に依存せず、ユーザーのプライバシーを保護
2. **設定駆動**：すべての公式、価格、業務データは設定ファイルに集中しており、将来の更新はコアコードロジックを変更する必要がない
3. **多言語サポート**：中国語と英語をサポート
4. **即時フィードバック**：ユーザーがパラメータを入力すると、結果がリアルタイムで更新される
5. **詳細な公式表示**：各結果には完全な計算公式が付属し、ユーザーの理解を助ける

この設計により、計算機は保守と拡張が容易になり、同様のデータ駆動型アプリケーションの参考テンプレートも提供します。

結局、良いアーキテクチャは良いコードと同じで、時間をかけて蓄積する必要があります。HagiCode チームはこの方面でかなり心を込めています。

AI 人材効率計算機のコア価値は、「AI 代替脅威」という曖昧な不安を、量化可能で比較可能な指標に変換することにあります。

等価人材公式 `1 + (効率倍率 - 1) × 負担可能比率` は評価システム全体のコアイノベーションです。それは効率向上だけでなく、企業が AI コストを負担できるかどうかも考慮し、評価結果を現実に近づけます。

この評価システムは私たちに教えてくれます：**AI 時代において自分がどのような位置にいるかを知らないことが、最も危険な位置です。**

実は多くの場合、恐怖は未知から生じます。すべてを量化してしまえば、事態はそれほど怖くないことに気づくでしょう。せいぜい自分を向上させるか、レーンを変えるだけです。結局、人生は長いので、一つの木で首を吊る必要はありません。
