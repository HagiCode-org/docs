## タイプするより話す方がいい、話すよりスクリーンショットがいい——AIコードアシスタントのマルチモーダル入力の実践

> 実はコードを書くこと、タイプがいくら速くても限界がある。時には一言で済むことを、半日もキーボードを叩かなきゃいけない；時には一枚の画像で伝わることを、たくさんの文字で説明しなきゃいけない。本文では、私たちが[HagiCode](https://hagicode.com)を作る際に出くわした那些事儿について話す——音声認識だろうが、画像アップロードだろうが、とにかくAIコードアシスタントを少し使いやすくしたいだけ、それだけ。

## 背景

HagiCodeを作る際、私たちは一つの問題に気づいた——あるいは、ユーザーが使ううちに自然と現れてきた問題：タイプだけに頼るのは、時々結構疲れる。

考えてみてください。ユーザーとAgentのインタラクション、それは核心的なシナリオです。でも毎回キーボードの前でカチャカチャとタイプしなきゃいけない、どう言ったらいいか、効率があまり高くない：

1. **タイプが遅すぎる**：複雑な問題、エラーだの、画面上のことだの、タイプで伝えるのに半分かかるけど、口で言えば10秒で終わるかもしれない。この時間差、結構不快。

2. **画像の方が直接的**：時には画面でエラーが出たり、デザイン案と比較したかったり、コード構造を見せたかったり……「一枚の画像は千の言葉に価する」という言葉は古いけど、理屈は間違ってない。AIに直接「見てもらう」方が、半日説明するよりずっと明確。

3. **インタラクションはもっと自然にあるべき**：今のAIアシスタントは、テキスト、音声、画像これらの方式をサポートすべきだろ？ユーザーが何を使いたいで使えばいい、それこそが自然、そう思わない？

だから、私たちはHagiCodeに音声認識と画像アップロードの機能を追加しようと思った、Agentの操作をもっと便利にするために。結局、ユーザーが少しでもタイプする手間を減らせるなら、それはいいこと。

## HagiCodeについて

本文で共有するこれらの方案は、[HagiCode](https://hagicode.com)プロジェクトでの実践から来たもの——あるいは、絶えず穴にハマりながら探り出した経験。

HagiCodeはオープンソースのAIコードアシスタントプロジェクトで、アイデアはシンプル：AI技術で開発効率を向上させる。やっていくうちに気づいたのは、ユーザーのマルチモーダル入力への需要は実は結構強い——時には一言タイプするより速く、時には一枚のスクリーンショットの方が半日説明するより明確。

これらの需要が私たちを前に進め、最終的に音声認識と画像アップロードこれらの機能ができた。ユーザーは最も自然な方式でAIとインタラクションできる、この感じ、結構いい。

## 分析

### 音声認識の技術的課題

音声認識機能を作る際、私たちは一つの厄介な問題に遭遇した：**ブラウザのWebSocket APIはカスタムHTTPヘッダーをサポートしていない**。

そして私たちが選んだ音声認識サービスは、字节跳动の豆包音声認識API。このAPIはどういうわけかHTTPヘッダーで認証情報を渡すことを要求する、`accessToken`だの、`secretKey`だの。これでよろしい、技術的矛盾が来た：

```javascript
// ブラウザWebSocket APIは以下の方式をサポートしない
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // 非サポート
  }
});
```

私たちの前にある方案は、大体二つ：

1. **URLクエリパラメータ方案**：認証情報をURLに置く
   - メリットは、実装が簡単
   - デメリットは、フロントエンドに証拠書類が露出する、セキュリティが悪い；また、API強制ヘッダー検証もある

2. **バックエンドプロキシ方案**：バックエンドでWebSocketプロキシを実装
   - メリットは、証拠書類がバックエンドに安全に保存；API要求を完全に互換
   - デメリットは、実装が少し複雑

最終的に私たちは**バックエンドプロキシ方案**を選んだ。やっぱり、セキュリティというものは、妥協できないボトムライン——この点、誰もごまかせない。

### 画像アップロードの機能要件

画像アップロード機能、私たちの需要は実は結構シンプル：

1. **複数のアップロード方式**：クリックでファイル選択、ドラッグ＆ドロップ、クリップボード貼り付け、最低限これくらいは？
2. **ファイル検証**：タイプ制限（PNG、JPG、WebP、GIF）、サイズ制限（5-10MB）、これらは基本操作
3. **ユーザー体験**：アップロード進捗、プレビュー、エラー提示、最低限何が起きたか分かるように
4. **セキュリティ**：サーバー側検証、悪意あるファイルアップロード防止、これは大事

## 解決方案

### 音声認識：WebSocketプロキシアーキテクチャ

私たちは三層アーキテクチャの音声認識方案を設計した、どう言ったらいいか、一つの道を見つけた感じ：

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**核心コンポーネント実装**：

1. **フロントエンドAudioWorkletプロセッサー**：

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // 16kHzにリサンプリング（豆包API要求）
    const samples = this.resampleAudio(input, 48000, 16000);

    // サンプルを500msチャンクに蓄積
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // 16-bit PCMに変換して送信
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **バックエンドWebSocketハンドラー**（C#）：

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **フロントエンドVoiceTextAreaコンポーネント**：

```tsx
export const VoiceTextArea = forwardRef<HTMLTextAreaElement, VoiceTextAreaProps>(
  ({ value, onChange, onTextRecognized, maxDuration }, ref) => {
    const { isRecording, interimText, volume, duration, startRecording, stopRecording } =

```

### 画像アップロード：複数方式アップロードコンポーネント

私たちは機能が完備された画像アップロードコンポーネントを作った、三つのアップロード方式を全部サポート、どう言ったらいいか、ユーザーがよく使うシーンを大体カバーした。

**核心特性**：

1. **三つのアップロード方式**：

```tsx
// クリックアップロード
const handleClick = () => fileInputRef.current?.click();

// ドラッグ＆ドロップアップロード
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// クリップボード貼り付け
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **フロントエンド検証**：

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **バックエンドアップロード処理**（TypeScript）：

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // 検証
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // ファイル保存
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## 実践ガイド

### 音声認識の使い方

1. **音声認識サービスを設定**：
   - 音声認識設定ページに入る
   - 豆包音声の`AppId`と`AccessToken`を設定
   - （オプション）ホットワードを設定して専門用語認識精度を向上

2. **入力ボックスで使用**：
   - 入力ボックスの左側のマイクアイコンをクリック
   - 波形アニメーションが見えたら話し始める
   - もう一度アイコンをクリックして録音停止
   - 認識結果は自動的にカーソル位置に挿入される

3. **ホットワード設定例**：

```text
TypeScript
React
useState
useEffect
```

### 画像アップロードの使い方

1. **アップロード方式**：
   - アップロードボタンをクリックしてファイル選択
   - 画像をアップロードエリアに直接ドラッグ＆ドロップ
   - `Ctrl+V`でクリップボードのスクリーンショットを貼り付け

2. **サポート形式**：PNG、JPG、JPEG、WebP、GIF
3. **サイズ制限**：デフォルト5MB（設定可能）

### 注意事項

1. **音声認識**：
   - マイク権限が必要
   - 静かな環境で使用を推奨
   - 最大録音時間は300秒（設定可能）

2. **画像アップロード**：
   - 一般的な画像形式のみサポート
   - ファイルサイズ制限に注意
   - アップロード後の画像は自動的にプレビューURLが生成される

3. **セキュリティ考慮**：
   - 音声認証証拠書類はバックエンドに保存
   - 画像アップロードは厳格なサーバー側検証あり
   - 本番環境ではHTTPS/WSS使用を推奨

## まとめ

音声認識と画像アップロードを追加した後、HagiCodeのユーザー体験は確かにかなり向上した。ユーザーは今、もっと自然な方式でAIとインタラクションできる——話す代替タイプ、スクリーンショット代替説明。この感じ、どう言ったらいいか、ついにもっと快適なコミュニケーション方式を見つけたみたい。

この機能を作る際、私たちはブラウザWebSocketがカスタムヘッダーをサポートしない問題に遭遇し、最終的にバックエンドプロキシ方案で解決した。この方案はセキュリティを保証しただけでなく、後続の他の認証が必要なWebSocketサービス統合の基礎も築いた——これも意外な収穫かも。

画像アップロードコンポーネントも同様、複数のアップロード方式を使い、ユーザーがシーンに応じて最も便利なものを選べるようにした。クリックでも、ドラッグでも、あるいは直接貼り付けでも、素早くアップロードを完了できる。すべての道はローマに通じる、ただある道は歩きやすく、ある道は少し曲がりくねっているだけ。

「タイプするより話す方がいい、話すよりスクリーンショットがいい」という言葉、ここに置くと、結構適切。もし君も似たようなAIアシスタント製品を作っているなら、これらの経験が少しでも役に立てば幸い。
