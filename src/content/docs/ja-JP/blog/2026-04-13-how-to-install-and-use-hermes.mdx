Hermesをインストールして使い始めるには、実は最短でたった3ステップです：

1. 公式インストールコマンドを実行
2. ターミナルで `hermes` を入力してCLIを起動
3. もしFeishuで使い続けたいなら、`hermes gateway setup` を設定

この記事ではHermesの全機能を一度に解説するのではなく、最も重要な入門サイクルを先に完了させることに焦点を当てます：インストールして、動かして、使い始めて、その後に最も一般的なメッセージングプラットフォームシナリオを接続する。

## Hermesとは、誰に適しているか

Hermes Agentは、ローカルターミナルで使用できるだけでなく、メッセージングプラットフォームゲートウェイを通じて使用できるAI agentです。

多くの開発者にとって、最も一般的な2つのエントリーポイントがあります：

- **CLI**：ターミナルで `hermes` と入力して、直接インタラクティブインターフェースに入る
- **Messaging Gateway**：`hermes gateway` を実行し、Feishu、Telegram、Discord、Slackなどのプラットフォームから会話する

今の目標がクイックスタートだけであれば、順序を逆にしないことをお勧めします。まずこのルートで進めてください：

- 先にHermesをインストール
- 先にCLIで使用可能か確認
- その後にメッセージングプラットフォームに接続するか決める

こうすれば問題の特定が容易になり、Hermesに初めて触れるユーザーにも適しています。

## Hermesをインストールする前に知っておくべきこと

Hermes READMEによると、公式クイックインストールパスは以下の環境をサポートしています：

- Linux
- macOS
- WSL2
- Android via Termux

### Windowsユーザーへの注意事項

Hermesは現在、ネイティブWindowsでの直接実行をサポートしていません。Windowsを使用している場合、先にWSL2をインストールし、その中でインストールコマンドを実行することをお勧めします。

この点は記事の最初に明確にしておくのが最適です。多くのインストール失敗は実際にはコマンドの問題ではなく、実行環境が要件を満たしていないことが原因だからです。

## Hermesのインストール方法

Hermes READMEで提供されているクイックインストールコマンドは以下の通りです：

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

このコマンドは公式インストールスクリプトを実行し、プラットフォームに関連する初期化プロセスを処理します。

### インストール完了後のshellリフレッシュ

インストール終了後、先にshell環境を再読み込みしてください。最も一般的なのは：

```bash
source ~/.bashrc
```

zshを使用している場合は、以下に変更しても構いません：

```bash
source ~/.zshrc
```

### Hermesが正常にインストールされたかの確認方法

最も直接的な確認方法は以下を実行することです：

```bash
hermes
```

さらに設定と依存関係が正常か確認したい場合は、以下も実行できます：

```bash
hermes doctor
```

`hermes doctor` は以下のような場合に適しています：

- インストール後のコマンド動作が異常
- モデル設定失敗
- gateway起動失敗
- 環境依存関係が完全か不明

## 初めてHermesを使い始める方法

### 先にCLIから始める

Hermesが使えるかどうかをすぐに確認したいだけであれば、最も簡単な方法は：

```bash
hermes
```

これでHermesのインタラクティブCLIが起動します。Hermesに初めて触れるユーザーにとって、これが最も推奨されるスタート地点です。なぜなら、最も重要なことを先に検証できるからです：

- コマンドが本当に使えるか
- 現在のモデル設定が正常か
- ターミナルツールチェーンが正常に動作しているか
- インタラクティブ体験が自分に合った方式か

### 最初の設定を完了するのに十分なコマンド

Hermes READMEにリストされている高頻度コマンドは、基本的に最初の使用パスを構成します：

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

それらがそれぞれ何をするかわからない場合は、先にこう覚えておいてください：

- `hermes model`：モデルの選択、切り替え
- `hermes tools`：現在利用可能なツールの表示と設定
- `hermes config set`：特定の設定項目の変更
- `hermes setup`：完全な初期化ウィザードを一度実行
- `hermes update`：Hermesの更新
- `hermes doctor`：トラブルシューティング

初心者に最も実用的な順序は通常以下の通りです：

1. 先に `hermes model` を実行
2. もし一般的な項目を一度に設定したい場合は、`hermes setup` を実行

## Hermesの最も一般的な2つの使用方法

### 1. ターミナルでHermesを日常の開発アシスタントとして使用

CLIモードは以下のシナリオに適しています：

- ローカルでコードを書くときに直接質問する
- プロジェクトを確認し、ファイルを変更し、コマンドを実行する
- 一回限りのデバッグやレビューを行う
- 現在の作業ディレクトリで継続的に協業する

最大の利点はパスが最短であること：追加のプラットフォーム接続が不要で、ボット設定を先に処理する必要がなく、最初の使用習慣を確立するのにも最適です。

### 2. メッセージングプラットフォームを通じてHermesを使用

もしFeishu、Telegram、DiscordなどのプラットフォームでHermesと会話したい場合は、messaging gatewayを使用する必要があります。

最も一般的なエントリーポイントコマンドは：

```bash
hermes gateway setup
hermes gateway
```

そのうち：

- `hermes gateway setup` はインタラクティブなプラットフォーム設定に使用
- `hermes gateway` はゲートウェイプロセスの起動に使用

公式ドキュメントによると、gatewayは統一されたバックグラウンドプロセスであり、設定済みのプラットフォームに接続し、セッションを管理し、タイマータスクなどの機能を処理するために使用されます。

## Feishuを例に、Hermesをメッセージングプラットフォームに接続する方法

もし日々の作業が主にFeishuで行われているなら、Feishu/LarkはHermes接続の非常に自然な方法になるでしょう。

### 最小実用接続方法

公式ドキュメントでFeishu/Larkに推奨されているエントリーポイントは：

```bash
hermes gateway setup
```

実行後、ウィザードで **Feishu / Lark** を選択するだけです。

Feishuドキュメントで提示されている2つの接続モードは：

- `websocket`：推奨
- `webhook`：オプション

もしHermesがノートPC、ワークステーション、またはプライベートサーバーで動いているなら、`websocket`を優先的に使用する方が簡単です。追加で公開コールバックアドレスを暴露する必要がないからです。

### 手動設定する場合、少なくともこれらの変数を知っておく必要がある

もしウィザードではなく手動で設定を書く場合、Feishuドキュメントにリストされている核心的な変数には以下が含まれます：

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

その中で最も注目すべき2項目は：

- `FEISHU_ALLOWED_USERS`：設定を推奨。ボットに接触できる誰もが直接使用できるのを避けるため
- `FEISHU_HOME_CHANNEL`：事前にhome chatを指定でき、cron結果やデフォルト通知を受信するために使用

### Feishuグループチャットで時々返信しない理由

この点は見落とされがちです：Feishuグループチャットでは、Hermesはデフォルトで全てのメッセージを見ても反応するわけではありません。

公式ドキュメントは明確に説明しています：

- プライベートチャットでは、Hermesはメッセージに反応します
- グループチャットでは、明示的にbotを `@` しない限り、メッセージを処理しません

もし特定のFeishuセッションをhome channelに設定したい場合は、チャット内で以下を使用できます：

```text
/set-home
```

または事前に設定で以下を記述：

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## 初心者が先に覚えるべき価値のあるHermesコマンド

CLI内でも、メッセージングプラットフォーム内でも、以下のコマンドを先に覚えておけば十分です：

- `/new` または `/reset`：新しいセッションを開始
- `/model`：モデルの表示または切り替え
- `/retry`：前のラウンドを再試行
- `/undo`：前のインタラクションを取り消し
- `/compress`：コンテキストの手動圧縮
- `/help`：ヘルプを表示

主にメッセージングプラットフォームで使用する場合は、もう一つ追加で覚えてください：

- `/sethome` または `/set-home`：現在のチャットをhome channelに設定

これらのコマンドは初心者段階で最も一般的な操作をカバーしています：新規開始、調整、取り消し、表示、そして継続使用。

## よくある質問

### WindowsにHermesを直接インストールできますか？

いいえ。現在公式ドキュメントは明確に、ネイティブWindowsは非サポートで、WSL2の使用を推奨しています。

### インストール後に `hermes` と入力しても反応しない場合はどうすればよいですか？

以下の順序でトラブルシューティングすることをお勧めします：

1. 先にshellを再読み込み、例えば `source ~/.bashrc`
2. もう一度 `hermes` を実行
3. まだ異常なら、`hermes doctor` を実行

### Feishuグループでbotが返信しないのはなぜですか？

先にこの3項目を確認してください：

- グループでHermesを `@` したか
- `FEISHU_ALLOWED_USERS` が現在のユーザーを制限していないか
- 現在のグループチャットポリシーがグループメッセージの処理を許可しているか

公式Feishuドキュメントによると、グループチャットシナリオでは、明示的な@mentionは必要条件です。

## 次のステップの推奨

もしすぐにHermesを使い始めたいだけであれば、最も推奨される順序は：

1. 先にインストールコマンドを実行
2. 先に `hermes` でローカルCLIを開始
3. その後に `hermes model` と `hermes setup` で基本設定を補完
4. もしFeishuで使い続けたいなら、`hermes gateway setup` を設定
