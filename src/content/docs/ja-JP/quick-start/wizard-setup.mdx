デスクトップのインストールと最初の起動後、HagiCode は新しい初期化ウィザードを自動的に開きます。現在のバージョンでは、以前の長い複数ステップのガイドには従わなくなりました。代わりに、実際のワークスペースに入る前に、基礎設定を完了し、少なくとも 1 つのプロジェクトを作成し、コミュニティ エントリ ポイントを近くに置いておくという 3 つの主要な段階で構成されています。

:::note[このページの見方]
この書き換えでは、次のマネージド スクリーンショット ライブラリを使用します。 `repos/docs/src/content/docs/img/screenshots/quick-start/wizard-setup/` 真実の情報源として。現在の管理セットには、 `Foundation settings` このページではその部分を詳細に説明し、後の 2 つのステージを高いレベルに保ちます。
:::

## ウィザードの概要

現在の初期化ウィザードは、次の 3 つの主要な段階を経て進みます。

1. **基本設定**: インターフェイス言語を選択し、テーマを選択し、依存関係を確認し、デフォルトのエージェントが存在しない場合は作成します。
2. **プロジェクトの作成**: ウィザードの出力が実際の作業ディレクトリに接続されるように、少なくとも 1 つのプロジェクトを作成します。
3. **コミュニティ ガイド**: 通常のワークスペースに入る前に、GitHub とその他のフォローアップ エントリ ポイントを手の届くところに置いてください。

## ステージ 1: 基礎設定

左側のナビゲーションでは、最初のステージにすでに次のラベルが付いています。 `基础设置` そしてその目的を明確に要約します: インターフェイス言語を選択し、春のテーマを維持し、検証します。 `node`, `npm`, `git`、そして `OpenSpec`、デフォルトを作成します `Hagicoder` まだヒーローが存在しない場合。結論: この段階では、散在していた古い準備作業が 1 つの連続したセットアップ サーフェスに統合されます。

### 1.1 インターフェース言語とテーマを選択する

![初期化ウィザードの基本設定: 言語とテーマ](../../img/screenshots/quick-start/wizard-setup/guide1-language-theme/original.png)

最初のスクリーンショットは、基礎設定画面の上部を示しています。それはから始まります `选择界面语言`、ヘルパー テキストはスコープを明示的にします。これにより、ヒーローの言語スタイルではなく、**インターフェイスのコピーのみ**が変更されます。例では、 `简体中文` としてマークされています `当前`、一方 `English` 右側は引き続き利用可能です。

同じ画面がすぐに続きます。 `选择主题`。現在のビルドでは Spring Light と Spring Dark のバリアントが公開されており、サンプルでは適切なテーマ カードを次のようにマークしています。 `当前`。主な変更点は構造的なものです。言語とテーマは、古いガイドから個別にジャンプするのではなく、1 つの連続したセクションとして処理されるようになりました。

### 1.2 依存関係を確認し、インストール アクションを近くに置く

![初期化ウィザードの基本設定: 依存関係のチェック](../../img/screenshots/quick-start/wizard-setup/guide2-dependency-check/original.png)

2 番目のスクリーンショットは次のようにスクロールします。 `组件检测`。この領域は自動的にチェックされます `Node.js`, `npm`, `Git`、そして `OpenSpec`、各項目を現在の可用性でレンダリングします。右上隅には `重新检测` アクション、そして `OpenSpec` 行も保持します `安装 OpenSpec` エントリー。

ロジックは直接的です。ウィザードは、すべての前提条件を手動で覚えるのではなく、現在のマシンの準備ができているかどうかを通知します。これらのコア コンポーネントが利用可能になったら、デフォルトのエージェント作成を続行し、プロジェクトの作成に進むことができます。

### 1.3 最もよく使用される CLI とモデルを選択し、デフォルトのエージェントを作成します

![初期化ウィザードの基本設定: デフォルトのエージェントガイダンス](../../img/screenshots/quick-start/wizard-setup/guide3-default-agent-cli-model/original.png)

3 番目のスクリーンショットは、 `默认Agent引导`。エージェントがまだ存在しない場合、ウィザードは最もよく使用されるエージェントを尋ねます。 `AgentCLI` 最初にモデルを作成し、次にデフォルトのエージェントを自動作成し、それを後の構成グループに組み込みます。この例では、選択されたデフォルトは次のとおりです。 `Claude Code` そして `GLM 5.1`.

この画面では、デフォルトの ID も処理します。下部の領域ではアバターのプレビューが表示され、 `换一组` アクションを実行し、メインの CTA を右下に配置します。 `确认并创建 Hagicoder`。そのため、現在のウィザードは古いマルチスクリーン ロール テーブルを最初に開くことはなくなりました。まず、使用可能なデフォルト エグゼキューターが 1 つ存在することが保証されます。

## ステージ 2: プロジェクトの作成

左側のナビゲーションでは、第 2 ステージに次のラベルが付けられています `创建项目`。ページレベルの概要は、その役割をすでに明示しています。これは、カジュアルなオプションの停止ではありません。言語、テーマ、依存関係の状態、および基礎設定で生成されたデフォルトのエージェントを実際の作業ディレクトリに接続できるように、**少なくとも 1 つのプロジェクト**を作成する必要があります。

現在の管理対象スクリーンショット セットには、この段階専用のスクリーンショットがまだ含まれていないため、このページでは、現時点では概要レベルの説明を維持します。そこに到達したら、先に進む前に実際のリポジトリまたは作業フォルダーに接続します。

## ステージ 3: コミュニティ ガイド

左側のナビゲーションの 3 番目のステージは、 `社区引导`。ヘッダーの概要には、実際のワークスペースに入る前に、基礎設定を完了し、少なくとも 1 つのプロジェクトを作成し、**コミュニティ エントリ ポイントを近くに置いておく**という順序が明確に記載されています。

現在管理されているスクリーンショット セットにはこのステージ専用のスクリーンショットも欠如しているため、ドキュメントではここでのフローの意味のみが保持されています。これは、フォローアップ エントリ ポイントの最後のステージであり、コアの初期化決定を行う場所ではありません。
