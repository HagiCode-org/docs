## Por que a HagiCode escolheu o Hermes como núcleo de Agent integrado

> Ao construir uma plataforma de codificação assistida por IA, escolher o núcleo de Agent adequado determina diretamente o teto das capacidades do sistema. Afinal, algumas coisas não podem ser forçadas — escolha a estrutura errada e nada parece funcionar. Este artigo compartilha as reflexões da HagiCode na seleção tecnológica e a prática de integração do Hermes Agent.

## Contexto

Ao trabalhar em codificação assistida por IA, o mais doloroso é escolher a estrutura de Agent subjacente. Na verdade, há muitas opções disponíveis no mercado, só que algumas têm funcionalidades muito simples, outras são muito complexas de implantar, e outras ainda não têm escalabilidade suficiente. Precisávamos de uma solução que pudesse rodar tanto em um VPS de 5 dólares quanto em um cluster GPU — esse requisito não é tão alto, mas não são poucos os que desistem.

Mas a situação real é que muitos pretensos "Agentes onipotentes" ou só rodam na nuvem, ou exigem requisitos locais absurdamente altos. Depois de passar duas semanas pesquisando várias soluções, tomamos uma decisão ousada: reconstruir completamente o núcleo do Agent, adotando o Hermes como mecanismo subjacente para o Agent integrado.

Tudo que essa decisão trouxe, talvez fosse o destino.

## Sobre a HagiCode

A solução compartilhada neste artigo vem da experiência prática do projeto [HagiCode](https://hagicode.com). A HagiCode é uma plataforma de codificação assistida por IA que, através de extensões VSCode, clientes desktop e serviços web, fornece assistentes de codificação inteligente para desenvolvedores. Talvez você tenha usado ferramentas semelhantes, apenas sentiu que faltava algo — nós também entendemos.

- GitHub: [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site)

## Por que a HagiCode precisa do Hermes

Antes de detalhar o Hermes, vamos falar sobre por que a HagiCode tem essa necessidade. Neste mundo, as coisas nem sempre funcionam como queremos, sempre precisamos encontrar uma razão apropriada.

Como um assistente de código de IA, a HagiCode precisa suportar simultaneamente vários cenários de uso:
- Ambiente de desenvolvimento local: desenvolvedores desejam executar em seus próprios computadores, os dados não saem do local — nestes dias, a segurança dos dados não é algo nem tão grande nem tão pequeno
- Ambiente de colaboração em equipe: pequenas equipes podem compartilhar Agent implantado em servidores — economizar dinheiro, ninguém tem isso fácil
- Expansão elástica na nuvem: ao processar tarefas complexas, pode expandir automaticamente para clusters GPU — prevenir é melhor que remediar

Essa necessidade de "ter tudo de uma vez" nos direcionou ao Hermes. Não sei se essa escolha está certa, apenas na época não havia outra melhor.

## O que é o Hermes Agent

O Hermes Agent é um AI Agent autônomo criado pelo **Nous Research**. Algumas pessoas podem não estar familiarizadas com o Nous Research — eles são o laboratório que desenvolveu modelos grandes de código aberto como Hermes, Nomos e Psyché. Dizem, eles também não tiveram fácil, fizeram tantas coisas boas, mas poucas pessoas conhecem.

Ao contrário dos assistentes de programação IDE tradicionais ou wrappers simples de chat de API, o Hermes tem uma característica: quanto mais tempo executa, mais forte se torna. Ele não completa tarefas de uma vez e pronto, mas pode aprender continuamente e acumular experiência em execuções de longa duração. Isso também é bastante parecido com humanos, não é?

### Recursos principais

Os vários recursos principais do Hermes se alinham perfeitamente com as necessidades da HagiCode. Que coincidência, não?

Isso significa que a HagiCode pode escolher o método de implantação mais adequado de acordo com o cenário do usuário. Usuários individuais executam localmente, usuários de equipe implantam em servidores, tarefas complexas usam GPU — um conjunto de código resolve tudo. Neste mundo, poder economizar uma tarefa já é algo.

**Gateway de mensagens multiplataforma**
O Hermes suporta nativamente plataformas como Telegram, Discord, Slack, WhatsApp. Para a HagiCode, isso significa que no futuro pode facilmente suportar assistentes de IA nesses canais. Afinal, quem não quer ter mais caminhos?

**Sistema rico de ferramentas**
Mais de 40 ferramentas integradas, mais capacidade de extensão MCP (Model Context Protocol). Isso é muito importante para um assistente de código — executar comandos shell, operar sistema de arquivos, chamar Git, tudo isso precisa de suporte de ferramentas. Um Agent sem ferramentas é como um pássaro sem asas — quer voar mas não consegue.

**Memória entre sessões**
O Hermes tem um sistema de memória persistente, usando recuperação de texto completo FTS5 para recordar conversas históricas. Isso permite que o Agent se lembre do contexto anterior, não "esqueça" toda vez. Às vezes eu também quero esquecer um pouco, não pensar em nada, mas simplesmente não consigo.

## Como a HagiCode integra o Hermes

Depois de falar sobre "por que", vamos ver "como fazer". Algumas coisas, quando entendidas, precisam ser colocadas em prática, apenas pensar sem agir também não resolve.

### Abstração da camada Provider

Na arquitetura da HagiCode, todos os AI Providers implementam a interface unificada `IAIProvider`:

```csharp
public sealed class HermesCliProvider : IAIProvider, IVersionedAIProvider
{
    public ProviderCapabilities Capabilities { get; } = new ProviderCapabilities
    {
        SupportsStreaming = true,   // Suporta saída em fluxo
        SupportsTools = true,       // Suporta chamada de ferramentas
        SupportsSystemMessages = true, // Suporta mensagens do sistema
        SupportsArtifacts = false
    };
}
```

Essa camada de abstração permite que a HagiCode possa alternar perfeitamente entre diferentes AI Providers, seja OpenAI, Claude ou Hermes, a forma de chamada da camada superior é completamente consistente. Simplificando, é economizar trabalho.

### Protocolo de comunicação ACP

O Hermes usa **ACP (Agent Communication Protocol)** para comunicação. Este é um protocolo especificamente projetado para comunicação de Agent, os principais métodos incluem:

| Método | Descrição |
|------|------|
| `initialize` | Inicializa conexão, obtém versão do protocolo e capacidades do cliente |
| `authenticate` | Processa autenticação, suporta vários métodos de autenticação |
| `session/new` | Cria nova sessão, define diretório de trabalho e servidores MCP |
| `session/prompt` | Envia prompt e obtém resposta |

A Hagi implementa a camada de transporte ACP através de `StdioAcpTransport`, iniciando o subprocesso Hermes e comunicando através de entrada e saída padrão. Isso soa complexo, mas na prática é ok — o principal é ter paciência.

### Gerenciamento de configuração

Gerencia configurações através da classe `HermesPlatformConfiguration`:

```csharp
public sealed class HermesPlatformConfiguration : IAcpPlatformConfiguration
{
    public string ExecutablePath { get; set; } = "hermes";
    public string Arguments { get; set; } = "acp";
    public int StartupTimeoutMs { get; set; } = 5000;
    public string ClientName { get; set; } = "HagiCode";
    public HermesAuthenticationConfiguration Authentication { get; set; }
    public HermesSessionDefaultsConfiguration SessionDefaults { get; set; }
}
```

Configuração, olhando simples, realmente precisa de algum esforço para acertar.

### Arquitetura distribuída Orleans

A HagiCode usa Orleans para construir sistemas distribuídos, a integração Hermes é realizada através dos seguintes componentes:

- **HermesGrain**: implementação de Orleans Grain, processa execução de sessões
- **HermesPlatformConfiguration**: configuração específica da plataforma
- **HermesAcpSessionAdapter**: adaptador de sessão ACP
- **HermesConsole**: console de validação dedicado

Orleans, esse nome é bonito, a lenda de Aladim — embora este Orleans não seja aquele Orleans, mas um nome bonito sempre é um ponto a favor.

### Fluxo de execução completo

Abaixo está a lógica principal de execução do Hermes Provider:

```csharp
private async IAsyncEnumerable<AIStreamingChunk> StreamCoreAsync(
    AIRequest request,
    string? embeddedCommandPrompt,
    [EnumeratorCancellation] CancellationToken cancellationToken)
{
    // 1. Criar camada de transporte, iniciar subprocesso Hermes
    await using var transport = new StdioAcpTransport(
        platformConfiguration.GetExecutablePath(),
        platformConfiguration.GetArguments(),
        platformConfiguration.GetEnvironmentVariables(),
        platformConfiguration.GetStartupTimeout(),
        _loggerFactory.CreateLogger<StdioAcpTransport>());
    await transport.ConnectAsync(cancellationToken);

    // 2. Inicializar, obter versão do protocolo e métodos de autenticação
    var initializeResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "initialize",
        BuildInitializeParameters(platformConfiguration), cancellationToken);

    // 3. Processar autenticação
    var authMethods = ParseAuthMethods(initializeResult);
    if (!isAuthenticated)
    {
        var methodId = platformConfiguration.Authentication.ResolveMethodId(authMethods);
        await SendHermesRequestAsync(transport, nextRequestId++, "authenticate", ...);
    }

    // 4. Criar sessão
    var newSessionResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "session/new",
        BuildNewSessionParameters(platformConfiguration, workingDirectory, model), cancellationToken);
    var sessionId = ParseSessionId(newSessionResult);

    // 5. Executar prompt e coletar resposta em fluxo
    await foreach (var payload in transport.ReceiveMessagesAsync(cancellationToken))
    {
        // Processar notificações session/update, converter para blocos de fluxo
        if (TryParseSessionNotification(root, out var notification))
        {
            if (_responseMapper.TryConvertToStreamingChunk(notification, out var chunk))
            {
                yield return chunk;
            }
        }
    }
}
```

Código, quando você vê muito, é basicamente isso. O importante é a ideia, certo?

### Verificação de integridade

Para garantir a disponibilidade do serviço Hermes, a HagiCode implementou um mecanismo de verificação de integridade:

```csharp
public async Task<ProviderTestResult> PingAsync(CancellationToken cancellationToken = default)
{
    var response = await ExecuteAsync(
        new AIRequest
        {
            Prompt = "Reply with exactly PONG.",
            CessionId = null,
            AllowedTools = Array.Empty<string>(),
            WorkingDirectory = ResolveWorkingDirectory(null)
        },
        cancellationToken);

    var success = string.Equals(response.Content.Trim(), "PONG", StringComparison.OrdinalIgnoreCase);
    return new ProviderTestResult
    {
        ProviderName = Name,
        Success = success,
        ResponseTimeMs = stopwatch.ElapsedMilliseconds,
        ErrorMessage = success ? null : $"Unexpected Hermes ping response: '{response.Content}'."
    };
}
```

Isso é mais ou menos a tal "verificação de integridade". Na verdade, é o mesmo para pessoas, sempre de vez em quando precisamos verificar nós mesmos — apenas geralmente ninguém nos diz o que verificar.

## Considerações na prática

No processo de integração do Hermes, há algumas armadilhas que valem a pena conhecer antecipadamente. Nestes dias, quem não pisou em algumas armadilhas?

### Configuração do método de autenticação

O Hermes suporta vários métodos de autenticação (API Key, Token etc.), precisa escolher de acordo com a situação real de implantação. Configuração errada leva a falha de conexão, mas a mensagem de erro pode não ser intuitiva. Às vezes a mensagem de erro difere da causa real por uma grande distância, precisa investigar lentamente.

### Configuração do servidor MCP

Ao criar uma sessão, você pode configurar a lista de servidores MCP, permitindo que o Hermes chame ferramentas externas. Mas atenção:
- O endereço do servidor MCP deve estar acessível
- O tempo limite deve ser definido razoavelmente
- Processamento de degradação quando o servidor não está disponível

Neste mundo, impossível se prevenir contra tudo.

### Gerenciamento do diretório de trabalho

Cada sessão precisa especificar um diretório de trabalho, garantindo que o Hermes possa acessar corretamente os arquivos do projeto. Para cenários multiprojeto, precisa alternar dinamicamente o diretório de trabalho. Parece simples, na prática há muitos casos a considerar.

### Processamento de agregação de resposta

A resposta do Hermes pode estar dispersa em notificações `session/update` e resultado final, precisa processar fusão corretamente, caso contrário ocorrerá perda de conteúdo. Isso também já me causou prejuízo, com o tempo melhora.

### Estratégia de tratamento de erros

Erros em tempo de execução devem ser retornados explicitamente, em vez de recuar silenciosamente para outros Providers. Assim os usuários sabem que o problema é do Hermes, não mudaram misteriosamente para outro modelo. Afinal, enganar também não é desse jeito.

## Conclusão

A HagiCode escolheu o Hermes como núcleo de Agent integrado, não foi uma decisão impulsiva, mas uma escolha cuidadosa baseada em necessidades reais e características técnicas. Se essa escolha está certa, agora é cedo para dizer, apenas atualmente está funcionando bem.

A capacidade de implantação flexível fornecida pelo Hermes permite que a HagiCode possa se adaptar a vários cenários de uso; o poderoso sistema de ferramentas e suporte MCP permite que o assistente de IA realmente faça trabalho real; e o protocolo ACP e a camada de abstração Provider tornam todo o processo de integração claro e controlável.

Se você está escolhendo uma estrutura de Agent para seu projeto de IA, espero que este artigo possa fornecer alguma referência. Afinal, escolher a arquitetura subjacente certa tornará o desenvolvimento subsequente muito mais fácil......
