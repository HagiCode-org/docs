## Guia de Integração Hagicode com GLM-5.1 Multi-CLI

:::note[Nota sobre Contexto Temporal]

Para o escopo de suporte atual, consulte [Instalação do AI Agent CLI](/related-software-installation/ai-agent-cli/) e [Guia de Instalação](/installation); o `IFlowCli` mencionado neste artigo representa apenas a configuração histórica da época e não pertence ao escopo de suporte ativo atual.

## Contexto

No projeto Hagicode, os usuários podem escolher entre várias ferramentas CLI para acionar o assistente de programação AI, incluindo Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI, entre outros. Essas ferramentas CLI são ferramentas de programação AI genéricas, mas através da camada de abstração do Hagicode, podem se integrar flexivelmente a diferentes provedores de modelos AI.

A Zhipu AI (ZAI) fornece uma interface compatível com a Anthropic Claude API, permitindo que essas ferramentas CLI usem diretamente os modelos da série GLM domésticos. O GLM-5.1 é o modelo de linguagem mais recente lançado pela Zhipu, com melhorias significativas em relação ao GLM-5.0.

## Arquitetura de Abstração CLI do Hagicode

O Hagicode define 11 tipos de provedores CLI através do enum `AIProviderType`, cobrindo as principais ferramentas CLI de programação AI:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Cada CLI possui sua configuração correspondente de parâmetros de modelo, suportando os parâmetros `model` e `reasoning`:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
```

## Sistema de Suporte de Modelos GLM

O Secondary Professions Catalog do Hagicode define suporte completo para a série de modelos GLM:

| Model ID | Name | Default Reasoning | Compatible CLI Families |
|----------|------|-------------------|-------------------------|
| `glm-4.7` | GLM 4.7 | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM 5 Turbo | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (Legacy) | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | high | claude, codebuddy, hermes, qoder, kiro |

## Diferenças Chave entre GLM-5.1 e GLM-5.0

A partir da implementação em `AcpSessionModelBootstrapper.cs`, podemos ver claramente as diferenças entre GLM-5.1 e GLM-5.0:

### Implementação Independente do GLM-5.1

GLM-5.1 é um novo identificador de modelo independente, sem qualquer lógica de tratamento legacy:

## Usando GLM-5.1 em Diferentes CLIs

### 1. Claude Code CLI + GLM-5.1

Claude Code CLI é uma das CLIs principais do Hagicode, configurado através do sistema de configuração Hero:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Configuração correspondente do `HeroEquipmentCatalogItem`:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI é a CLI mais flexível, suportando o formato `provider/model` para especificar qualquer modelo:

**Método 1: Usar prefixo do provider ZAI**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Método 2: Usar diretamente o ID do modelo**
```json
{
  "model": "glm-5.1"
}
```

**Método 3: Interface de configuração frontend**

Em `HeroModelEquipmentForm.tsx`, OpenCode CLI possui um placeholder especial:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Os usuários podem inserir:
```
zai/glm-5.1
glm-5.1
```

**Lógica de resolução de modelo do OpenCode CLI**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // Sem barra: usa diretamente o ID do modelo
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Com barra: analisa o formato provider/model
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI possui uma lógica especial de tratamento legacy:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Nota**: Codebuddy tem uma retenção especial para GLM-5.0, não fazendo legacy normalization:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// Quando for CodebuddyCli, glm-5.0 não será normalizado para glm-5-turbo
```

## Vantagens das Melhorias do GLM-5.1

Comparado ao GLM-5.0, GLM-5.1 possui as seguintes melhorias significativas:

### 1. Melhor Capacidade de Raciocínio

Segundo as informações oficiais de lançamento da Zhipu, as melhorias do GLM-5.1 incluem:

- **Compreensão de código mais forte**: análise mais precisa de estruturas de código complexas
- **Compreensão de contexto mais longa**: suporte para contextos de conversa mais longos
- **Chamadas de ferramentas aprimoradas**: aumento da taxa de sucesso de chamadas de ferramentas MCP
- **Estabilidade de saída**: redução de aleatoriedade e alucinações

### 2. Compatibilidade Completa com Múltiplas CLIs

GLM-5.1 cobre todas as principais CLIs suportadas pelo Hagicode:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Considerações Importantes

### 1. Configuração de API Key

Certifique-se de configurar corretamente a variável de ambiente `ANTHROPIC_AUTH_TOKEN`, que é a credencial necessária para todas as CLIs conectarem aos modelos.

### 2. Disponibilidade do Modelo

GLM-5.1 precisa ser ativado no provedor de modelos correspondente:
- Plataforma Zhipu AI ZAI suporta por padrão
- Alibaba Cloud DashScope pode solicitar ativação separada

### 3. Formato OpenCode CLI

Ao usar o formato `provider/model`, certifique-se de que o ID do provider está correto:
- Zhipu AI: `zai` ou `zhipuai`
- Alibaba Cloud: `aliyun` ou `dashscope`

### 4. Parâmetro Reasoning

- Recomenda-se usar o nível `high` para obter os melhores resultados de geração de código
- Gemini CLI não suporta o parâmetro reasoning e ignorará automaticamente essa configuração

## Resumo

O Hagicode, através de uma camada de abstração unificada, implementa a integração flexível do GLM-5.1 com múltiplas CLIs. Os desenvolvedores podem escolher a ferramenta CLI apropriada de acordo com suas preferências e cenários de uso, e usar o modelo mais recente GLM-5.1 através de configuração simples.

Como a versão mais recente do modelo da Zhipu, o GLM-5.1 possui melhorias óbvias em relação ao GLM-5.0:
- Identificador de versão independente, sem bagagem legacy
- Capacidade de raciocínio mais forte e compreensão de código
- Ampla compatibilidade com múltiplas CLIs
- Configuração flexível de níveis de raciocínio

Através da configuração correta das variáveis de ambiente e equipamentos Hero, os jogadores podem aproveitar totalmente o poder do GLM-5.1 em diferentes ambientes CLI.
