## Guia de Implementação de Efeito de Animação de Luz Circundante na Borda

> Aquele elemento importante que chama a atenção do usuário à primeira vista, como é feito com CSS puro? Na verdade não é tão difícil, é só dar uma volta. Este artigo guia você desde o início para implementar a animação de luz circundante na borda, e também conversa sobre os buracos que pisamos no projeto HagiCode.

## Histórico

Colegas que trabalham com front-end provavelmente já tiveram essa experiência: o gerente de produto vem com aquela expressão de "este requisito é muito simples" — "esta tarefa em execução, pode adicionar um efeito especial para que o usuário possa ver de relance?"

Você diz claro, vamos adicionar uma mudança de cor na borda. Mas a outra pessoa balança a cabeça, com um olhar que diz "você não entende": "não é óbvio o suficiente, preciso daquele efeito onde a luz corre ao redor da borda, igual nos filmes de ficção científica."

Nesse momento você pode se perguntar: como implementar isso? Usar Canvas? Usar SVG? Ou será que o CSS consegue? Afinal, ninguém quer admitir que não sabe.

Na verdade, a animação de luz circundante na borda é muito comum em aplicações Web modernas, sendo usada principalmente nestes cenários:

- **Indicador de status**: Marca tarefas em andamento ou projetos ativos
- **Foco visual**: Destaca áreas de conteúdo importantes
- **Aprimoramento de marca**: Cria uma experiência visual com sensação tecnológica e moderna
- **Temas festivos**: Cria atmosfera de celebração para festas especiais

Quando construímos o [HagiCode](https://hagicode.com), encontramos essa necessidade — os usuários precisam ver de relance quais sessões estão em execução, quais propostas estão sendo processadas. Testamos várias soluções, alguns caminhos eram mais fáceis, outros um pouco mais tortuosos, e finalmente consolidamos uma abordagem de implementação bastante madura.

## Sobre o HagiCode

As soluções compartilhadas neste artigo vêm de nossa experiência prática no projeto [HagiCode](https://hagicode.com). HagiCode é um projeto de assistente de código alimentado por IA, que faz uso intensivo de animações de luz nas bordas na interface para indicar vários estados de execução. Por exemplo, status de execução da lista de sessões, transições de estado do fluxograma de propostas, exibição de intensidade do indicador de throughput, entre outros.

Na verdade, esses efeitos não são tão complexos, mas pisamos em vários buracos durante a implementação. Se você quiser ver o efeito real, pode visitar nosso [repositório GitHub](https://github.com/HagiCode-org/site) ou ir diretamente ao [site oficial](https://hagicode.com) para saber mais — afinal, o que funciona é o melhor.

## Ideias Centrais de Implementação

Após analisar o código do HagiCode, resumimos os seguintes padrões centrais de implementação, cada um com seus cenários aplicáveis, ou melhor, cada um com sua razão de existir.

### 1. Gradiente Cônico Rotativo de Halo (Mais Comum)

Esta é a forma mais clássica de implementar luz circundante na borda, a ideia central é usar o `conic-gradient` do CSS para criar um gradiente cônico e depois fazê-lo girar. Como um poste de luz à noite, sempre girando e girando.

**Elementos-chave:**
- Usar o pseudo-elemento `::before` para criar a camada de halo
- Usar `conic-gradient` para definir a distribuição de cores do gradiente
- Usar o pseudo-elemento `::after` para mascarar a área central (opcional)
- Usar `@keyframes` para implementar a animação de rotação

### 2. Linha Luminosa Lateral

Esta é adequada para indicadores de status de itens de lista, basta criar uma linha fina luminosa em um lado do elemento, sem precisar que toda a borda se mova. Às vezes, um pouco de luz é suficiente, não há necessidade de iluminar o mundo todo.

**Elementos-chave:**
- Elemento de linha fina com posicionamento absoluto
- Usar `box-shadow` para criar efeito luminoso
- Usar `scale` e `opacity` para implementar animação de respiração

### 3. Fundo Luminoso Box-Shadow

Se você não precisa do efeito circundante e quer apenas um halo de fundo suave, sobrepor várias camadas de `box-shadow` é suficiente. Algumas coisas, mais simples é melhor.

### 4. Suporte de Acessibilidade

Este é frequentemente esquecido, mas muito importante. Todas as animações devem considerar a media query `prefers-reduced-motion`, fornecendo uma alternativa estática para usuários que não gostam de animações. Afinal, nem todos gostam de coisas se movendo, respeitar a escolha de cada um é o correto.

## Soluções de Implementação

### Solução 1: Borda Rotativa com Gradiente Cônico (Recomendado)

Esta é a implementação mais completa do efeito de luz circundante, e também a mais usada no HagiCode. Afinal, se algo funciona bem, por que mudar?

```css
/* Container pai */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Camada de halo rotativa */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Camada de máscara (opcional, para criar efeito de borda vazada) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

O princípio desta solução é bastante simples: criar um pseudo-elemento maior que o container pai, desenhar um gradiente cônico sobre ele e depois fazê-lo girar continuamente. O container pai define `overflow: hidden`, então só é possível ver a parte da luz girando na borda. Como quando olhamos um poste de luz lá fora através da janela, só vemos aquele pequeno trecho quando ele passa.

### Solução 2: Borda de Luz Rotativa Simplificada

Se você não precisa de um efeito tão complexo, o HagiCode tem uma implementação de classe utilitária mais leve. Às vezes, mais simples é melhor.

```css
/* Classe utilitária de borda de luz rotativa */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Suporte de acessibilidade */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Note o `will-change: transform` aqui, isso diz ao navegador "este elemento vai mudar sempre", e o navegador fará algumas otimizações antecipadas, a animação ficará mais suave. Afinal, se preparar com antecedência é melhor do que improvisar.

### Solução 3: Linha Luminosa Lateral

Esta é especialmente adequada para indicadores de status de itens de lista, a lista de sessões do HagiCode usa esta solução. Uma linha fina, mas que se destaca entre muitos projetos, não é essa também uma filosofia de vida?

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Aqui usamos `isolation: isolate` para criar um novo contexto de empilhamento, e depois `z-index` para controlar a ordem de exibição das camadas. `pointer-events: none` também é crucial, caso contrário o pseudo-elemento bloqueará as operações de clique do usuário. Como algumas coisas, bonito é bonito, mas não pode atrapalhar.

### Solução 4: Encapsulamento de Componente React

Se o seu projeto usa React, você pode encapsular um componente para lidar com essa lógica, especialmente a parte de acessibilidade. Afinal, escrever código uma vez e usar muitas vezes é o que queremos.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Módulo CSS correspondente:

```css
/* GlowBorder.module.css */

/* Versão animada */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Versão estática (acessibilidade) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

O hook `useReducedMotion` do `framer-motion` detecta automaticamente as preferências do sistema do usuário, se o usuário ativou "reduzir movimento animado", retornará true, e então exibe a versão estática. Afinal, respeitar a escolha do usuário é mais importante do que forçar a exibição.

## Compartilhamento de Experiência Prática

Abaixo estão as experiências que resumimos ao pisar em buracos durante o desenvolvimento do HagiCode. Na verdade são só algumas observações, esperando ajudar você que virá depois.

### 1. Sistema de Variáveis de Tema

Usar variáveis CSS para implementar suporte a múltiplos temas é muito conveniente. Afinal, ninguém quer mudar um monte de código toda vez que muda o tema.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Uso */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

Ao trocar o tema, basta alterar a classe da tag `html`, e todas as cores de animação serão atualizadas automaticamente. Um código, dois estilos, não é isso que buscamos?

### 2. Otimização de Desempenho

**Use `will-change` para sugerir otimização do navegador:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Diga ao navegador com antecedência, e ele fará algumas otimizações para você. Como muitas coisas na vida, se preparar com antecedência é sempre bom.

**Evite usar box-shadow complexo em elementos grandes:**

```css
/* Ruim - usar sombra desfocada em elementos grandes */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Melhor - usar pseudo-elemento para limitar a área de brilho */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Testamos no HagiCode, adicionar sombra desfocada diretamente em cartões grandes faz a taxa de quadros de rolagem cair abaixo de 30fps, e ao usar pseudo-elementos fica estável em 60fps. Os usuários conseguem sentir essa diferença de experiência.

### 3. Acessibilidade

Isso realmente não pode ser ignorado, alguns usuários podem sentir que as animações causam tontura ou são barulhentas, respeitar a escolha deles é um requisito básico para fazer produtos. Afinal, coisas bonitas não precisam ser impostas a ninguém.

**Media query CSS:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Fornecer alternativa estática */
    opacity: 1;
  }
}
```

**Detectar preferências do usuário no React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Controle de Nível de Intensidade

O indicador de throughput de Tokens no HagiCode exibe luzes de diferentes cores com base no throughput em tempo real, isso é implementado dinamicamente. Afinal, estados diferentes devem ter formas de expressão diferentes.

```tsx
const colors = [
  null,       // Level 0 - sem cor
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Precauções

Alguns detalhes precisam de atenção, caso contrário só saberá que pisou no buraco quando for tarde demais.

| Precaução | Descrição |
|---------|------|
| **Gerenciamento de z-index** | A camada de halo deve ter um z-index adequado para evitar afetar a interação do conteúdo |
| **pointer-events** | Pseudo-elementos de halo devem ter `pointer-events: none` |
| **Transbordamento de borda** | O container pai precisa definir `overflow: hidden` ou ajustar o tamanho do pseudo-elemento |
| **Impacto no desempenho** | Animações complexas podem afetar o desempenho em dispositivos móveis, é necessário testar |
| **Modo escuro** | Certifique-se de que a cor do brilho seja claramente visível em fundo escuro |
| **Troca de tema** | Use variáveis CSS para garantir que as cores da animação sejam atualizadas corretamente ao trocar o tema |

### 6. Técnicas de Depuração

Pseudo-elementos às vezes são difíceis de encontrar nas ferramentas de desenvolvedor, você pode adicionar temporariamente uma borda para ver a posição.

```css
/* Exibir temporariamente limites do pseudo-elemento para depuração */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Depois de ajustar a posição, lembre-se de comentar ou excluir esta linha, caso contrário o ambiente de produção ficará embaraçoso. Algumas coisas são melhores deixar no ambiente de desenvolvimento.
