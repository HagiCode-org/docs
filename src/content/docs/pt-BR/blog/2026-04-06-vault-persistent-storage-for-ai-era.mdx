## Como estudar projetos na era da IA: sistema de armazenamento persistente entre projetos Vault

> Na era do desenvolvimento assistido por IA, como ajudar o assistente de IA a entender melhor nossos recursos de aprendizado? O projeto HagiCode implementa, através do sistema Vault, uma camada de abstração de armazenamento de conhecimento unificada e compreensível por IA, melhorando significativamente a eficiência do aprendizado de estudo de projetos.

## Contexto

Na era da IA, a forma como os desenvolvedores aprendem novas tecnologias e arquiteturas está passando por mudanças profundas. "Estudar projetos" — ou seja, pesquisar e aprender profundamente o código, arquitetura e padrões de design de projetos open source excelentes — tornou-se um método de aprendizado eficiente. Em comparação com os métodos tradicionais de ler livros e assistir vídeos, ler e executar diretamente projetos open source de alta qualidade permite que você entenda as práticas de engenharia do mundo real mais rapidamente.

No entanto, esse método de aprendizado também enfrenta vários desafios.

Os materiais de aprendizado estão muito dispersos. Suas anotações podem estar no Obsidian, repositórios de código espalhados por várias pastas, e o histórico de conversas do assistente de IA é outra ilha de dados independente. Quando você quer que a IA ajude a analisar um projeto, precisa copiar manualmente trechos de código e organizar o contexto, um processo bastante tedioso.

Na verdade, o problema mais sério é a descontinuidade do contexto. O assistente de IA não consegue acessar diretamente seus recursos de aprendizado locais, e cada conversa precisa fornecer novamente informações de contexto. Além disso, os repositórios de código estudados são atualizados com frequência, a sincronização manual é propensa a erros, e é difícil compartilhar conhecimento entre vários projetos de aprendizado.

Essencialmente, todos esses problemas são causados por "ilhas de dados". Se houvesse uma camada de abstração de armazenamento unificada que permitisse que o assistente de IA entendesse e acessasse todos os seus recursos de aprendizado, o problema estaria resolvido.

## Sobre HagiCode

O sistema Vault compartilhado neste artigo é exatamente a solução que praticamos durante o desenvolvimento do [HagiCode](https://hagicode.com). HagiCode é um projeto de assistente de código IA, e em nosso desenvolvimento diário, frequentemente precisamos aprender e referenciar vários projetos open source. Para permitir que o assistente de IA entenda melhor esses recursos de aprendizado, projetamos o sistema de armazenamento persistente entre projetos Vault.

Esta solução já foi verificada na prática no HagiCode. Se você também enfrenta problemas semelhantes de gerenciamento de conhecimento, espero que essas experiências possam lhe dar algumas inspirações. Afinal, alguns obstáculos foram superados, e é sempre bom deixar algo para quem vem depois.

## Filosofia de design do sistema Vault

A ideia central do sistema Vault é muito simples: criar uma camada de abstração de armazenamento de conhecimento unificada e compreensível por IA. Do ponto de vista de implementação, o sistema possui várias características principais.

### Suporte a múltiplos tipos

O sistema suporta quatro tipos de vault, correspondendo a diferentes cenários de uso:

```typescript
// folder: tipo de pasta geral
export const DEFAULT_VAULT_TYPE = 'folder';

// coderef: tipo usado especificamente para estudar projetos de código
export const CODEREF_VAULT_TYPE = 'coderef';

// obsidian: integração com o software de anotações Obsidian
export const OBSIDIAN_VAULT_TYPE = 'obsidian';

// system-managed: vault gerenciado automaticamente pelo sistema
export const SYSTEM_MANAGED_VAULT_TYPE = 'system-managed';
```

Entre eles, o tipo **coderef** é o mais usado no HagiCode. É projetado especificamente para estudar projetos de código, fornecendo uma estrutura de diretórios padronizada e descrições de metadados legíveis por IA.

### Mecanismo de armazenamento persistente

O registro de Vault é armazenado de forma persistente em formato JSON, garantindo que a configuração permaneça disponível após a reinicialização do aplicativo:

```csharp
public class VaultRegistryStore : IVaultRegistryStore
{
    private readonly string _registryFilePath;

    public VaultRegistryStore(IConfiguration configuration, ILogger<VaultRegistryStore> logger)
    {
        var dataDir = configuration["DataDir"] ?? "./data";
        var absoluteDataDir = Path.IsPathRooted(dataDir)
            ? dataDir
            : Path.GetFullPath(Path.Combine(Directory.GetCurrentDirectory(), dataDir));

        _registryFilePath = Path.Combine(absoluteDataDir, "personal-data", "vaults", "registry.json");
    }
}
```

A vantagem desse design é a simplicidade e confiabilidade. O formato JSON é legível por humanos, facilitando a depuração e modificação manual; o armazenamento no sistema de arquivos evita a complexidade de bancos de dados, reduzindo as dependências do sistema. Afinal, às vezes o simples é o melhor.

### Integração de contexto de IA

O mais importante é que o sistema consegue injetar automaticamente informações do vault no contexto das propostas de IA:

```typescript
export function buildTargetVaultsText(
  vaults: VaultForText[],
  template: VaultPromptTemplate = DEFAULT_VAULT_PROMPT_TEMPLATE,
): string {
  const readOnlyVaults = vaults.filter((vault) => vault.accessType === 'read');
  const editableVaults = vaults.filter((vault) => vault.accessType === 'write');

  if (readOnlyVaults.length === 0 && editableVaults.length === 0) {
    return '';
  }

  const sections = [
    buildVaultSection(readOnlyVaults, template.reference),
    buildVaultSection(editableVaults, template.editable),
  ].filter(Boolean);

  return `\n\n### ${template.heading}\n\n${sections.join('\n')}`;
}
```

Isso implementa uma função importante: o assistente de IA consegue entender automaticamente os recursos de aprendizado disponíveis, sem que o usuário precise fornecer contexto manualmente. Isso também é uma espécie de entendimento mútuo.

## Estrutura padronizada do CodeRef Vault

Para vaults do tipo coderef, o HagiCode fornece uma estrutura de diretórios padronizada:

```
my-coderef-vault/
├── index.yaml          # descrição de metadados do vault
├── AGENTS.md           # guia de operações do assistente de IA
├── docs/               # armazena anotações de aprendizado e documentos
└── repos/              # gerencia repositórios de código estudados através de submódulos Git
```

Ao criar um vault, o sistema inicializa automaticamente essa estrutura:

```csharp
private async Task EnsureCodeRefStructureAsync(
    string vaultName,
    string physicalPath,
    ICollection<VaultBootstrapDiagnosticDto> diagnostics,
    CancellationToken cancellationToken)
{
    Directory.CreateDirectory(physicalPath);

    var indexPath = Path.Combine(physicalPath, CodeRefIndexFileName);
    var docsPath = Path.Combine(physicalPath, CodeRefDocsDirectoryName);
    var reposPath = Path.Combine(physicalPath, CodeRefReposDirectoryName);

    // cria estrutura de diretórios padronizada
    if (!Directory.Exists(docsPath))
    {
        Directory.CreateDirectory(docsPath);
    }

    if (!Directory.Exists(reposPath))
    {
        Directory.CreateDirectory(reposPath);
    }

    // cria guia AGENTS.md
    await EnsureCodeRefAgentsDocumentAsync(physicalPath, cancellationToken);

    // cria metadados index.yaml
    await WriteCodeRefIndexDocumentAsync(indexPath, mergedDocument, cancellationToken);
}
```

O design dessa estrutura também é bem pensado:

- O diretório **docs/** armazena suas anotações de aprendizado, podendo registrar compreensão do código, análise de arquitetura, experiências de obstáculos superados, etc. em formato Markdown
- O diretório **repos/** gerencia repositórios estudados através de submódulos Git, em vez de copiar o código diretamente. Assim, mantém o código sincronizado e economiza espaço
- **index.yaml** contém metadados do vault, permitindo que o assistente de IA entenda rapidamente o propósito e conteúdo deste vault
- **AGENTS.md** é um guia escrito especificamente para o assistente de IA, explicando como processar o conteúdo deste vault

Talvez, organizando assim, a IA também possa entender mais facilmente suas ideias.

## Inicialização automática gerenciada pelo sistema

Além de criar vaults manualmente, o HagiCode também suporta vaults gerenciados automaticamente pelo sistema:

```csharp
public async Task<IReadOnlyList<VaultRegistryEntry>> EnsureAllSystemManagedVaultsAsync(
    CancellationToken cancellationToken = default)
{
    var definitions = GetAllResolvedDefinitions();
    var entries = new List<VaultRegistryEntry>(definitions.Count);

    foreach (var definition in definitions)
    {
        entries.Add(await EnsureResolvedSystemManagedVaultAsync(definition, cancellationToken));
    }

    return entries;
}
```

O sistema cria e gerencia automaticamente os seguintes vaults:

- **hagiprojectdata**: armazenamento de dados do projeto, usado para salvar configurações e estado do projeto
- **personaldata**: armazenamento de dados pessoais, usado para salvar configurações de preferências do usuário
- **hbsprompt**: biblioteca de modelos de prompts, usado para gerenciar prompts de IA comuns

Esses vaults são inicializados automaticamente na inicialização do sistema, sem necessidade de configuração manual pelo usuário. Afinal, algumas coisas devem ser deixadas para o sistema fazer, por que os humanos se preocuparem?

## Mecanismo de controle de acesso

Um design importante é o controle de acesso. O sistema divide vaults em dois tipos de acesso:

```typescript
export interface VaultForText {
  id: string;
  name: string;
  type: string;
  physicalPath: string;
  accessType: 'read' | 'write';  // chave: distingue entre somente leitura e editável
}
```

- **reference (somente leitura)**: a IA usa apenas para análise e compreensão, não pode modificar o conteúdo. Aplicável a projetos open source de referência, documentos, etc.
- **editable (editável)**: a IA pode modificar o conteúdo conforme necessário para a tarefa. Aplicável a suas anotações, rascunhos, etc.

Essa distinção é importante. Permite que a IA saiba quais conteúdos são "referência somente leitura" e quais são "podem ser modificados", evitando riscos de operação incorreta. Afinal, ninguém quer que seu trabalho seja perdido inadvertidamente.

## Prática: criar e usar Vault

Depois de entender os princípios, vamos ver como usar na prática.

### Criar CodeRef Vault

Aqui está um exemplo completo de chamada frontend:

```typescript
const createCodeRefVault = async () => {
  const response = await VaultService.postApiVaults({
    requestBody: {
      name: "React Learning Vault",
      type: "coderef",
      physicalPath: "/Users/developer/vaults/react-learning",
      gitUrl: "https://github.com/facebook/react.git"
    }
  });

  // o sistema fará automaticamente:
  // 1. clonar o repositório React para vault/repos/react
  // 2. criar diretório docs/ para anotações
  // 3. gerar metadados index.yaml
  // 4. criar arquivo de guia AGENTS.md

  return response;
};
```

Essa chamada de API completa uma série de operações: criar estrutura de diretórios, inicializar submódulos Git, gerar arquivos de metadados, etc. Você só precisa fornecer informações básicas, o resto é entregue ao sistema. Na verdade, isso também é bastante conveniente.

### Usar Vault em propostas de IA

Depois de criar o vault, você pode referenciá-lo em propostas de IA:

```typescript
const proposal = composeProposalChiefComplaint({

```

O sistema injeta automaticamente as informações do vault no contexto da IA, permitindo que a IA saiba quais recursos de aprendizado você tem disponíveis. A IA entende suas ideias, o que também é uma espécie de entendimento mútuo raro.

## Melhores práticas e precauções

Durante o uso do sistema Vault, resumimos algumas experiências e lições.

### Segurança de caminhos

O sistema valida rigorosamente os caminhos para prevenir ataques de travessia de caminho:

```csharp
private static string ResolveFilePath(string vaultRoot, string relativePath)
{
    var rootPath = EnsureTrailingSeparator(Path.GetFullPath(vaultRoot));
    var combinedPath = Path.GetFullPath(Path.Combine(rootPath, relativePath));
    if (!combinedPath.StartsWith(rootPath, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode,
            "Vault file paths must stay inside the registered vault root.");
    }
    return combinedPath;
}
```

Isso é muito importante. Se você estiver personalizando caminhos de vault, certifique-se de que os caminhos estão dentro do permitido, caso contrário o sistema recusará a operação. Segurança, não se pode enfatizar demais.

### Gerenciamento de submódulos Git

CodeRef vault recomenda usar submódulos Git em vez de copiar código diretamente:

```csharp
private static string BuildCodeRefAgentsContent()
{
    return """
    # CodeRef Vault Guide

    Repositories under `repos/` should be maintained through Git submodules
    rather than copied directly into the vault root.

    Keep this structure stable so assistants and tools can understand the vault quickly.
    """ + Environment.NewLine;
}
```

Isso tem vários benefícios: mantém o código sincronizado com o upstream, economiza espaço em disco, facilita o gerenciamento de múltiplas versões de código. Afinal, quem quer baixar repetidamente as mesmas coisas?

### Limitações de visualização de arquivos

Para prevenir problemas de performance, o sistema limita o tamanho e tipo de arquivos:

```csharp
private const int FileEnumerationLimit = 500;
private const int PreviewByteLimit = 256 * 1024;  // 256KB
```

Se seu vault contém muitos arquivos ou arquivos muito grandes, pode afetar a performance da função de visualização. Nesse caso, considere processamento em lotes ou usar ferramentas de busca especializadas. Afinal, algumas coisas muito grandes são difíceis de processar.

### Informações de diagnóstico

Ao criar um vault, informações de diagnóstico são retornadas para ajudar na depuração:

```csharp
List<VaultBootstrapDiagnosticDto> bootstrapDiagnostics = [];

if (IsCodeRefVaultType(normalizedType))
{
    bootstrapDiagnostics = await EnsureCodeRefBootstrapAsync(
        normalizedName,
        normalizedPhysicalPath,
        normalizedGitUrl,
        cancellationToken);
}
```

Se a criação falhar, você pode verificar as informações de diagnóstico para entender a causa específica. Se der erro, veja as informações de diagnóstico, essa também é uma forma de resolver problemas.

## Conclusão

O sistema Vault resolve as principais dores do estudo de projetos na era da IA através de uma camada de abstração de armazenamento unificada:

- **Gerenciamento centralizado de conhecimento**: todos os recursos de aprendizado centralizados em um só lugar, não mais espalhados
- **Injeção automática de contexto de IA**: o assistente de IA consegue entender automaticamente os recursos de aprendizado disponíveis, sem necessidade de fornecer contexto manualmente
- **Reutilização de conhecimento entre projetos**: múltiplos projetos de aprendizado podem compartilhar e reutilizar conhecimento
- **Estrutura de diretórios padronizada**: fornece estrutura de diretórios consistente, reduzindo custos de aprendizado
