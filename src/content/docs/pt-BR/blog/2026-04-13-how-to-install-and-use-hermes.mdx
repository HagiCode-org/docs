Para instalar e começar a usar o Hermes, o caminho mais curto tem apenas três passos:

1. Execute o comando oficial de instalação
2. Inicie o CLI no terminal com `hermes`
3. Se quiser continuar usando no Feishu, configure `hermes gateway setup`

Este artigo não pretende explicar todas as capacidades do Hermes de uma vez, mas ajudá-lo a completar o ciclo inicial mais crítico: instalar, executar, começar a usar e, em seguida, integrar com um cenário comum de plataforma de mensagens.

## O que é o Hermes e para quem é indicado

O Hermes Agent é um agente de IA que pode ser usado tanto no terminal local quanto através de um gateway de plataforma de mensagens.

Para a maioria dos desenvolvedores, ele tem dois pontos de entrada mais comuns:

- **CLI**: digite `hermes` no terminal para entrar diretamente na interface interativa
- **Messaging Gateway**: execute `hermes gateway` e converse com ele através de plataformas como Feishu, Telegram, Discord, Slack, etc.

Se o seu objetivo agora é apenas começar rapidamente, recomendo não inverter esta ordem, siga este caminho:

- Primeiro instale o Hermes
- Primeiro verifique se funciona através do CLI
- Depois decida se deseja integrar com uma plataforma de mensagens

Isso facilita a identificação de problemas e é mais adequado para usuários que estão entrando em contato com o Hermes pela primeira vez.

## O que saber antes de instalar o Hermes

De acordo com o Hermes README, o caminho oficial de instalação rápida suporta estes ambientes:

- Linux
- macOS
- WSL2
- Android via Termux

### Notas para usuários de Windows

O Hermes atualmente não suporta execução nativa direta no Windows. Se você usa Windows, recomenda-se primeiro instalar o WSL2 e depois executar o comando de instalação dentro do WSL2.

É melhor deixar isso claro logo no início do artigo, pois muitas falhas de instalação não são problemas de comando, mas sim ambientes de execução que não atendem aos requisitos.

## Como instalar o Hermes

O comando de instalação rápida fornecido pelo Hermes README é:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Este comando executa o script oficial de instalação, tratando os processos de inicialização específicos da plataforma.

### Atualize o shell após a instalação

Após a conclusão da instalação, recarregue o ambiente do shell. O mais comum é:

```bash
source ~/.bashrc
```

Se você usa zsh, também pode alterar para:

```bash
source ~/.zshrc
```

### Como verificar se o Hermes está instalado

A forma mais direta de verificação é executar:

```bash
hermes
```

Se quiser confirmar ainda mais se a configuração e as dependências estão normais, pode executar:

```bash
hermes doctor
```

`hermes doctor` é adequado para uso nestas situações:

- Comportamento anormal do comando após instalação
- Falha na configuração do modelo
- Falha ao iniciar o gateway
- Incerteza se as dependências do ambiente estão completas

## Como começar a usar o Hermes pela primeira vez

### Comece pelo CLI

Se você só quer confirmar o mais rápido possível se o Hermes funciona, o método mais simples é:

```bash
hermes
```

Isso iniciará o CLI interativo do Hermes. Para usuários que estão entrando em contato com o Hermes pela primeira vez, este também é o ponto de partida mais recomendado, pois você pode primeiro verificar as coisas mais centrais:

- Se o comando realmente está disponível
- Se a configuração atual do modelo está normal
- Se a cadeia de ferramentas do terminal está funcionando normalmente
- Se a experiência de interação é do jeito que você precisa

### Estes comandos são suficientes para completar a primeira rodada de configuração

Vários comandos de alta frequência listados no Hermes README basicamente formam o caminho inicial:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Se você não sabe o que cada um faz, pode lembrar assim:

- `hermes model`: selecionar e trocar modelos
- `hermes tools`: ver e configurar ferramentas disponíveis atualmente
- `hermes config set`: alterar itens de configuração específicos
- `hermes setup`: executar um assistente de inicialização completo
- `hermes update`: atualizar o Hermes
- `hermes doctor`: fazer solução de problemas

A ordem mais prática para iniciantes geralmente é:

1. Primeiro execute `hermes model`
2. Se quiser configurar itens comuns de uma vez, execute `hermes setup`

## As duas formas mais comuns de usar o Hermes

### 1. Usar o Hermes no terminal como assistente de desenvolvimento diário

O modo CLI é adequado para estes cenários:

- Fazer perguntas diretamente ao escrever código localmente
- Consultar projetos, modificar arquivos, executar comandos
- Fazer depuração ou review únicos
- Colaboração contínua no diretório de trabalho atual

Sua maior vantagem é o caminho mais curto: sem necessidade de integrar plataformas adicionais, sem configurar robôs primeiro, e é mais adequado para estabelecer os primeiros hábitos de uso.

### 2. Usar o Hermes através de plataformas de mensagens

Se quiser conversar com o Hermes em plataformas como Feishu, Telegram, Discord, etc., você precisa usar o messaging gateway.

Os comandos de entrada mais comuns são:

```bash
hermes gateway setup
hermes gateway
```

Onde:

- `hermes gateway setup` é usado para fazer configuração interativa da plataforma
- `hermes gateway` é usado para iniciar o processo do gateway

De acordo com a documentação oficial, o gateway é um processo unificado em segundo plano, usado para conectar as plataformas que você configurou, gerenciar sessões e lidar com tarefas agendadas e outras funções.

## Como integrar o Hermes em uma plataforma de mensagens, usando Feishu como exemplo

Se o seu trabalho diário é principalmente no Feishu, então Feishu/Lark será uma forma natural de integração com o Hermes.

### Forma mínima viável de integração

A entrada recomendada pela documentação oficial para Feishu/Lark é:

```bash
hermes gateway setup
```

Após executar, selecione **Feishu / Lark** no assistente.

A documentação do Feishu fornece dois modos de conexão:

- `websocket`: recomendado
- `webhook`: opcional

Se o Hermes estiver rodando no seu notebook, estação de trabalho ou servidor privado, priorizar o uso de `websocket` será mais simples, pois não precisa expor adicionalmente um endereço de callback público.

### Se você configurar manualmente, pelo menos precisa saber estas variáveis

Se você não estiver configurando através do assistente, mas escrevendo configuração manualmente, as variáveis centrais listadas na documentação do Feishu incluem:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

As mais notáveis são:

- `FEISHU_ALLOWED_USERS`: recomenda-se configurar para evitar que qualquer pessoa que tenha acesso ao bot possa usá-lo diretamente
- `FEISHU_HOME_CHANNEL`: pode especificar previamente um home chat para receber resultados de cron ou notificações padrão

### Por que às vezes o Hermes não responde em grupos do Feishu

Este ponto é facilmente ignorado: em grupos do Feishu, o Hermes por padrão não responde a todas as mensagens que vê.

A documentação oficial deixa claro:

- Em chat privado, o Hermes responderá às mensagens
- Em grupos, é necessário `@` o bot explicitamente para que ele processe a mensagem

Se quiser definir uma conversa do Feishu como home channel, também pode usar no chat:

```text
/set-home
```

Ou definir previamente na configuração:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Comandos do Hermes que iniciantes devem lembrar primeiro

Seja você no CLI ou em plataformas de mensagens, lembrar destes comandos já é suficiente:

- `/new` ou `/reset`: iniciar nova sessão
- `/model`: ver ou trocar modelo
- `/retry`: tentar novamente a última rodada
- `/undo`: desfazer a última interação
- `/compress`: comprimir contexto manualmente
- `/help`: ver ajuda

Se você usa principalmente em plataformas de mensagens, lembre-se adicionalmente de um:

- `/sethome` ou `/set-home`: definir o chat atual como home channel

Estes comandos cobrem as operações mais comuns na fase iniciante: recomeçar, ajustar, voltar, ver e continuar usando.

## Perguntas frequentes

### Posso instalar o Hermes diretamente no Windows?

Não. A documentação oficial atual deixa claro que o Windows nativo não é suportado, recomenda-se usar WSL2.

### O que fazer se digitar `hermes` após a instalação não tiver反应?

Recomenda-se verificar nesta ordem:

1. Primeiro recarregue o shell, por exemplo `source ~/.bashrc`
2. Execute `hermes` novamente
3. Se ainda estiver anormal, execute `hermes doctor`

### Por que o bot não responde em grupos do Feishu?

Primeiro verifique estes três itens:

- Você `@` o Hermes no grupo
- `FEISHU_ALLOWED_USERS` limitou o usuário atual
- Se a política do grupo atual permite processar mensagens de grupo

De acordo com a documentação oficial do Feishu, em cenários de grupo, @mention explícito é uma condição necessária.

4. Se quiser continuar usando no Feishu, configure `hermes gateway setup`
