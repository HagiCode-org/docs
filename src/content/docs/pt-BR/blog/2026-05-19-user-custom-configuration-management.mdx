## Melhores Práticas de Gerenciamento de Configuração Personalizada do Usuário

> Nada de mais, apenas compartilhando uma solução de gerenciamento de configuração. Da arquitetura à prática, espero que isso possa ajudá-lo a evitar alguns problemas.

## Contexto

Gerenciamento de configuração é algo que, ao mesmo tempo, é importante e trivial. É como aqueles pequenos hábitos na vida — alguns gostam de acordar cedo para tomar café, outros gostam de ficar acordado até tarde programando, essas preferências aparentemente insignificantes, na verdade, silenciosamente definem quem você é. Configurações personalizadas do usuário são semelhantes — alternância de temas, escolha de idioma, personalização de atalhos, quando esses recursos são bem feitos, os usuários sentem que o produto "os entende", e a retenção naturalmente aumenta.

No entanto, tornar este sistema completo não é tão simples quanto parece. Controle de versão, compatibilidade retroativa, validação de dados, controle de concorrência, controle de recursos DLC — esses problemas surgem um após o outro, como preocupações na adolescência, inevitáveis. Durante o desenvolvimento do HagiCode, também fomos atormentados por esses problemas, mas no final conseguimos entender alguns truques.

## Sobre o HagiCode

Esta solução, na verdade, é algo que descobrimos no HagiCode. O HagiCode é um assistente de código de IA com muitos recursos — idioma da interface, preferências de idioma de IA, tema, reconhecimento de voz, notificações, ações rápidas, há muita coisa que pode ser configurada. É precisamente porque os usuários querem tanto que tivemos que construir este sistema.

Endereço do projeto: [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site)

## Design de Arquitetura

### Arquitetura em Camadas

O sistema de gerenciamento de configuração do HagiCode, como dizer, é dividido em várias camadas, cada uma com sua própria função:

```
┌─────────────────────────────────────────┐
│         Frontend (React + Redux)        │
│  - Gerenciamento de estado de config    │
│  - Renderização de formulários UI       │
│  - Persistência de config por grupo     │
└─────────────────────────────────────────┘
                    ↓ HTTP/REST API
┌─────────────────────────────────────────┐
│   Application Service Layer             │
│  - FrontendConfigAppService             │
│  - Processamento de lógica de negócio   │
│  - Controle de permissões               │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│   Domain Layer (Config Store)           │
│  - FrontendConfigStore                  │
│  - Leitura/escrita de config            │
│  - Validação e normalização de dados    │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│   Infrastructure Layer                  │
│  - Armazenamento de arquivos YAML       │
│  - ISystemManagedVaultService           │
│  - Controle de concorrência (SemaphoreSlim) │
└─────────────────────────────────────────┘
```

Os benefícios dessa camada são realmente bastante óbvios:
- **Responsabilidades claras**: cada camada cuida de suas próprias coisas, sem interferir umas nas outras
- **Fácil de testar**: cada camada pode ser testada individualmente, alterações com mais confiança
- **Extensível flexível**: quer mudar o método de armazenamento ou modificar a API, outras camadas continuam funcionando

### Design de Interface Principal

A interface para armazenamento de configuração backend é mais ou menos assim:

```csharp
public interface IFrontendConfigStore
{
    // Obter configuração completa do usuário
    Task<FrontendConfigStoreResult> GetAsync(CancellationToken cancellationToken = default);
    
    // Atualizar configuração (suporta atualização parcial)
    Task<FrontendConfigStoreResult> UpdateAsync(
        UpdateFrontendConfigRequestDto input,
        CancellationToken cancellationToken = default);
    
    // Gerenciamento de estado de idioma de IA
    Task<FrontendConfigAiLanguageState> GetAiLanguageStateAsync(
        string userId,
        CancellationToken cancellationToken = default);
        
    Task<FrontendConfigAiLanguageState> SetAiLanguageAsync(
        string userId,
        string language,
        CancellationToken cancellationToken = default);
}
```

Os pontos-chave desta interface são realmente bastante compreensíveis:
- **Operações assíncronas**: tudo é assíncrono, afinal ninguém quer esperar
- **Token de cancelamento**: operações muito longas atingem timeout, não fique consumindo recursos
- **Atualização parcial**: atualize apenas o que precisa ser alterado, sem ter que passar por toda a configuração

Agrupamento de configuração, na verdade, é como categorizar as trivialidades da vida — trabalho é trabalho, lazer é lazer, relacionamentos são relacionamentos. Misturar tudo fica bagunçado, separar claramente torna as coisas mais fáceis:
- **Atualização sob demanda**: altere o que precisa ser alterado, sem afetar tudo mais

## Validação e Normalização de Dados

### Exemplo de Normalização de Idioma

Normalização de configuração, basicamente, é manter os dados consistentes. Tomando como exemplo a configuração de idioma:

```csharp
public static class FrontendConfigLanguageRules
{
    // Lista de idiomas de interface suportados
    public static readonly HashSet<string> ValidUiLanguages = new(StringComparer.OrdinalIgnoreCase)
    {
        "zh-CN", "zh-Hant", "en-US", "ja-JP", "ko-KR", 
        "de-DE", "fr-FR", "es-ES", "pt-BR", "ru-RU"
    };
    
    public static string NormalizeUiLanguage(string? value)
    {
        // Tratamento de valor vazio: retorna idioma padrão
        if (string.IsNullOrWhiteSpace(value)) return "en-US";
        
        var normalized = value.Trim();
        
        // Tratamento de alias: converte aliases comuns para códigos padrão
        normalized = normalized.ToLower() switch
        {
            "zh" or "chinese" or "cn" => "zh-CN",
            "en" or "english" => "en-US",
            "ja" or "japanese" => "ja-JP",
            "ko" or "korean" => "ko-KR",
            _ => normalized
        };
        
        // Tratamento de variantes de dialeto
        if (normalized.StartsWith("zh-Hans", StringComparison.OrdinalIgnoreCase))
            return "zh-CN";
        if (normalized.StartsWith("zh-TW", StringComparison.OrdinalIgnoreCase))
            return "zh-Hant";
            
        // Valida e retorna
        return ValidUiLanguages.Contains(normalized) ? normalized : "en-US";
    }
}
```

Normalização, na verdade, é como arrumar o quarto — quando as coisas ficam bagunçadas, você precisa organizar, caso contrário nem você mesmo consegue encontrar nada:
1. **Valor padrão para vazio**: dê um valor padrão razoável para valores vazios, não pode ficar vazio
2. **Mapeamento de alias**: aliases comuns, abreviações, convertidos uniformemente para formato padrão
3. **Unificação de dialetos**: variantes de dialeto mescladas em códigos padrão, afinal programar não é pesquisar dialetos
4. **Validação final**: garante que o valor retornado esteja sempre na lista válida, caso contrário todo o trabalho foi em vão

## Controle de Recursos DLC

O HagiCode suporta interruptores de recursos DLC, algumas opções de configuração avançadas só podem ser usadas depois de comprar o DLC. Isso é bastante comum em software comercial — recursos básicos são gratuitos, para usar coisas boas você precisa pagar, afinal os desenvolvedores também precisam comer.

### Verificação de Acesso DLC

```csharp
private async Task<PreparedFrontendConfigUpdate> PrepareUpdateAsync(
    UpdateFrontendConfigRequestDto input,
    FrontendConfigStoreResult current,
    CancellationToken cancellationToken)
{
    // Verifica permissões de acesso DLC
    var accessState = await _grainFactory
        .GetDlcAccessStateGrain(TurboEngineDlcId)
        .GetAccessStateAsync();
        
    var blockedFields = new List<string>();
    
    if (!accessState.IsActive)
    {
        // DLC não ativado, mantém configuração atual
        if (input.GeneralSettings?.BrandingLogo != null)
        {
            blockedFields.Add("brandingLogo");
            // Mantém valor antigo
            input.GeneralSettings.BrandingLogo = current.Snapshot.GeneralSettings.BrandingLogo;
        }
        
        if (input.GeneralSettings?.BrandingTitle != null)
        {
            blockedFields.Add("brandingTitle");
            input.GeneralSettings.BrandingTitle = current.Snapshot.GeneralSettings.BrandingTitle;
        }
    }
    
    return new PreparedFrontendConfigUpdate(
        input,
        new FrontendConfigUpdateDiagnosticsDto
        {
            Status = blockedFields.Count > 0 ? "partially-applied" : "success",
            BlockedFields = blockedFields,
        });
}
```

## Gerenciamento de Estado Frontend

### Redux Slice

O frontend usa Redux para gerenciar o estado de configuração, também é bastante convencional:

```typescript
export const frontendConfigSlice = createSlice({
  name: 'frontendConfig',
  initialState,
  reducers: {
    setConfigStatus(state, action: PayloadAction<FrontendConfigStatus>) {
      state.status = action.payload;
    },
    updateConfigGroups(state, action: PayloadAction<Partial<FrontendConfigSnapshot>>) {
      // Mescla atualizações de configuração
      Object.assign(state.snapshot, action.payload);
    },
  },
  extraReducers: (builder) => {
    builder
      .addCase(fetchConfig.pending, (state) => {
        state.status = 'loading';
      })
      .addCase(fetchConfig.fulfilled, (state, action) => {
        state.status = 'succeeded';
        state.snapshot = action.payload;
      })
      .addCase(fetchConfig.rejected, (state, action) => {
        state.status = 'failed';
        state.error = action.error.message;
      });
  },
});
```

    });

## Controle de Concorrência

Operações de atualização de configuração devem garantir thread-safety, caso contrário, se duas pessoas alterarem a configuração ao mesmo tempo, qual será salva no final? O HagiCode usa `SemaphoreSlim` para controle de concorrência, como dizer, também é um truque comum:

```csharp
private readonly SemaphoreSlim _semaphore = new(1, 1);

public async Task<FrontendConfigStoreResult> UpdateAsync(
    UpdateFrontendConfigRequestDto input,
    CancellationToken cancellationToken = default)
{
    await _semaphore.WaitAsync(cancellationToken);
    try
    {
        // Lê configuração atual
        var current = await GetAsync(cancellationToken);
        
        // Prepara atualização
        var prepared = await PrepareUpdateAsync(input, current, cancellationToken);
        
        // Escreve configuração
        await WriteConfigAsync(prepared.UpdatedConfig, cancellationToken);
        
        return new FrontendConfigStoreResult(prepared.UpdatedConfig, prepared.Diagnostics);
    }
    finally
    {
        _semaphore.Release();
    }
}
```

### Tratamento de Problemas Comuns

1. **Perda de configuração**: mecanismos de backup e restauração, sempre esteja preparado
2. **Conflitos de concorrência**: bloqueio otimista ou pessimista, como SemaphoreSlim
3. **Problemas de desempenho**: suporte a atualização parcial, não mova o objeto de configuração inteiro para cima e para baixo toda vez
4. **Segurança**: configurações sensíveis (como chaves de API) devem ser armazenadas criptografadas, caso contrário serão roubadas e causarão problemas
