## Design de 12 Experiências de Inicialização Extremas para o HagiCode: do Minimalismo ao Cyberpunk

> No breve intervalo entre o download e a Hydration de aplicações React 19, há uma janela de ouro para os usuários perceberem a personalidade da marca. Este artigo compartilha nossa experiência no projeto HagiCode, baseado em um sistema completo de estilos de inicialização construído com HTML/CSS/JS.


## Contexto

Como uma aplicação moderna baseada em ASP.NET Core 10 e React 19 (Vite), o HagiCode adota uma arquitetura de implantação separada entre frontend e backend. Os artefatos do frontend são empacotados e colocados no diretório `wwwroot/` do backend, hospedados pelo ASP.NET Core.

No entanto, essa arquitetura traz uma dor clássica na experiência do usuário: quando os usuários acessam a página, o navegador precisa primeiro carregar o HTML, depois baixar o enorme JS Bundle, e finalmente o React executa a Hydration. Nesses "períodos de vácuo" de algumas centenas de milissegundos a alguns segundos, os usuários enfrentam uma tela em branco ou uma página estática sem vida.

Para preencher esse intervalo e injetar a personalidade da marca HagiCode, precisamos projetar um sistema de estilos de inicialização completamente baseado em código inline no `index.html`.

## Sobre o HagiCode

O plano de design de tela de inicialização compartilhado neste artigo vem da nossa experiência prática no projeto [HagiCode](https://github.com/HagiCode-org/site). Como um assistente de código AI, o HagiCode não se preocupa apenas com a eficiência da geração de código, mas também valoriza a experiência visual dos desenvolvedores. Este sistema de inicialização é exatamente o produto de nossa busca por desempenho frontend extremo.

## Desafios Centrais e Design de Arquitetura

Antes de começar a projetar, devemos primeiro definir as restrições técnicas. Como precisamos implementar inline no `index.html`, isso significa que não podemos carregar nenhum arquivo CSS ou JS externo (exceto o próprio Bundle do React).

### Análise de Restrições Técnicas

1.  **Princípio de Dependência Zero**: Todos os estilos devem ser escritos dentro da tag `<style>`, e a lógica dentro da tag `<script>`.
2.  **CSS Defensivo**: Para evitar que estilos globais poluam a tela de inicialização após o aplicativo React ser montado, decidimos usar prefixos de ID de alta prioridade (como `#boot-screen`) para envolver todos os estilos de inicialização.
3.  **Performance em Primeiro Lugar**: Animações devem usar principalmente `transform` e `opacity` do CSS, evitando acionar refluxos e garantindo que não bloqueiem a thread principal.
4.  **Consistência Visual**: Cores e fontes devem permanecer consistentes com a configuração Tailwind do HagiCode.

### Padrão de Arquitetura: Shell & Injector

Adotamos um **padrão de variante**. A lógica central é encapsulada em uma função imediatamente invocada (IIFE), e a lógica de renderização específica é injetada como itens de configuração. Assim, podemos alternar entre diferentes estilos por meio de configuração simples, sem precisar repetir a lógica de manipulação do DOM.

Aqui está o código da arquitetura central:

```html
<!-- Inline em index.html -->
<div id="boot-root"></div>

<script>
(function() {
  const BootSequence = {
    config: {
      theme: 'terminal', // Pode ser configurado como 'minimal', 'skeleton', 'code-rain', etc.
      color: '#3b82f6'   // Cor da marca
    },
    
    // Ciclo de vida central
    init() {
      this.render();
      this.listenForMount();
    },

    // Renderiza o estilo selecionado atual
    render() {
      const root = document.getElementById('boot-root');
      if (this.variants[this.config.theme]) {
        root.innerHTML = this.variants[this.config.theme].render();
      }
    },

    // Escuta a montagem bem-sucedida do React, sai elegantemente
    listenForMount() {
      window.addEventListener('hagicode:ready', () => {
        const screen = document.getElementById('boot-root');
        // Primeiro fade out, depois remove o DOM, evitando flickering
        screen.style.opacity = '0';
        screen.style.transition = 'opacity 0.3s ease';
        setTimeout(() => screen.remove(), 300);
      });
    },

    // A lógica de implementação dos 12 estilos está concentrada aqui
    variants: {
      // ...implementação específica ver abaixo
    }
  };

  BootSequence.init();
})();
</script>
```

## Lista de 12 Estilos de Inicialização

Dividimos esses 12 estilos em seis categorias principais, para atender a diferentes cenários e necessidades estéticas.

### A. Minimalismo

> "Menos é mais". Para cenários que buscam velocidade de carregamento extrema, fornecemos a solução mais leve.

#### 1. Minimalist Dot (Ponto Respirante Minimalista)
Apenas um ponto simples no centro da tela, com animação de respiração.
*   **Implementação**: CSS `@keyframes` controlando scale e opacity.
*   **Aplicação**: Qualquer ocasião que necessite manter a página absolutamente limpa.

#### 2. Brand Reveal (Revelação da Marca)
Através da animação SVG `stroke-dasharray`, simula o desenho manual das linhas do Logo HagiCode, seguido pelo fade in do texto.
*   **Técnica**: Uso de animação de caminhos SVG, extremamente refinada.

### B. Mimicry de Skeleton Screen

> "A arte de enganar os olhos". Ao simular o layout real da UI, faz os usuários sentirem que a página já está carregada pela metade.

#### 3. Sidebar Chat Skeleton (Skeleton Screen da Barra Lateral)
Este pode ser o mais prático. Construímos manualmente em HTML um layout idêntico aos componentes React `Sidebar` e `ChatInput`, cobrindo com animação de listras cinzas.
*   **Valor**: Quando o React hidrata completamente, o skeleton screen se transforma instantaneamente em componentes reais, e os usuários quase não percebem a troca.

#### 4. Card Stack Skeleton (Pilha de Cartões)
Simula o efeito de empilhamento durante o carregamento de cartões de proposta, usando transformações 3D para fazer os cartões flutuarem levemente.

### C. Abstrato e Artístico

> Mostrar os genes geek do HagiCode.

#### 5. Geometric Morph (Morfismo Geométrico)
Renderiza uma forma geométrica (quadrado) no centro da tela, que se transforma suavemente em círculo, triângulo, e finalmente se torna o Logo.
*   **Técnica**: Transição suave de `border-radius` do CSS.

#### 6. Code Rain (Chuva de Código)
Uma homenagem a "Matrix". Usa a fonte JetBrains Mono, com fluxos de caracteres caindo suavemente no fundo.
*   **Atenção**: Para performance, o fluxo de caracteres deve ser limitado a uma área menor ou reduzir a frequência de atualização.

#### 7. Neon Pulse (Pulso Neon)
Anel luminoso estilo cyberpunk, usando sobreposição múltipla de `box-shadow` para criar um forte efeito de brilho.

### D. Marca e Tema

> Faça o sistema "ganhar vida".

#### 8. Seasonal Theme (Tema Sazonal)
Este é um carregador dinâmico. Determina o feriado com base na data atual (como Ano Novo Chinês, Natal), carregando a animação SVG correspondente.
*   **Exemplo**: No Ano Novo Chinês, lanternas vermelhas balançam suavemente na parte inferior da tela.

#### 9. Gradient Flow (Fluxo de Gradiente)
O fundo usa gradiente fluido nas cores da marca HagiCode, combinado com animação de `background-size` e `background-position`, criando uma sensação de fluxo auroral.

### E. Sensação Técnica

> Homenagem aos desenvolvedores.

#### 10. Terminal Boot (Inicialização de Terminal)
Simula a saída do console. Linhas de código rolando rapidamente:
```text
> Initializing HagiCode Core...
> Loading models...
> Connecting to neural network...
```
Isso fará cada desenvolvedor se sentir em casa.

#### 11. Progress Bar (Barra de Progresso Minimalista)
Uma barra de progresso fina no topo da tela, com porcentagem exibida à direita. Embora não possamos obter o progresso real do download, podemos usar um timer para simular um processo de carregamento "credível" (primeiros 80% rápidos, últimos 20% mais lentos).

### F. Criatividade

#### 12. Pixel Assembly (Montagem de Pixels)
Esta é uma ideia muito interessante. Quadrados espalhados na tela convergem para o centro, gradualmente formando o ícone do Logo HagiCode. Simboliza o processo de construção do código.

## Melhores Práticas e Lições Aprendidas

No desenvolvimento real do HagiCode, resumimos alguns detalhes de prática cruciais.

### 1. CSS Defensivo é Essencial
Não preguiçoso em não escrever prefixos. Uma vez, não limitamos os estilos da tela de inicialização com ID, o que fez com que os estilos globais `div` após a montagem do React afetassem inesperadamente a tela de inicialização, causando quebra de layout.
**Experiência**: Todos os seletores CSS devem estar sob `#boot-screen`, e usar `!important` para aumentar a prioridade (apenas no CSS da tela de inicialização).

### 2. Transição Elegante
Após o mount do React ter sucesso, não faça `remove()` diretamente no DOM da tela de inicialização.
**Abordagem Correta**:
1.  React dispara `window.dispatchEvent(new Event('hagicode:ready'))`.
2.  A tela de inicialização escuta o evento, primeiro define `opacity: 0`.
3.  Aguarda 300ms (tempo de transição CSS), garantindo que o usuário não veja mais, depois executa `.remove()`.

### 3. Sincronização de Variáveis de Tema
Os códigos de cor da tela de inicialização estão hardcoded no `index.html`. Se modificarmos a cor principal do Tailwind, devemos modificar aqui também.
**Solução de Otimização**: No script de build do Vite, escreva um plugin simples que lê `tailwind.config.js` e injeta as variáveis de cor nas variáveis de template do `index.html`, realizando fonte única de dados.

### 4. Pré-carregamento de Fontes
A tela de inicialização geralmente precisa usar a fonte da marca, mas se a fonte carregar lentamente, ocorrerá FOUT (Flash of Unstyled Text).
**Solução**: Adicione `<link rel="preload" href="/fonts/JetBrainsMono.woff2" as="font" type="font/woff2" crossorigin>` no `<head>`. Este é um meio de baixo custo e alto retorno para melhorar a experiência.

### 5. Monitoramento de Performance
Injetamos `performance.mark('boot-start')` na parte inferior do `index.html`, e marcamos `boot-end` quando o React é montado com sucesso.
**Significado**: Através do Application Insights coletando esses dados, podemos ver realisticamente o grau de redução do tempo de espera percebido pelo usuário (Perceived Loading Time) pela tela de inicialização. Os dados indicam que skeletons screens excelentes podem aumentar a tolerância dos usuários a "redes lentas" em mais de 50%.

## Conclusão

Uma boa tela de inicialização não é apenas "decoração durante a espera", é o sinal de handshake da primeira interação entre o produto e o usuário. No projeto HagiCode, este sistema de inicialização baseado no **Padrão Variants** nos permite alternar flexivelmente entre diferentes estilos em diferentes feriados e versões, aumentando enormemente a diversão e profissionalismo do produto.

A solução compartilhada neste artigo é baseada inteiramente em padrões Web nativos, sem introduzir nenhuma dependência pesada, o que é exatamente a manifestação da busca do HagiCode por "leve e poderoso".
