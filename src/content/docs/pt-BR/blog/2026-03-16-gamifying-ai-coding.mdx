## Como tornar a programação com IA mais divertida usando design de gamificação

> Na verdade, as ferramentas de programação com IA tradicionais são bastante poderosas, apenas carecem de um pouco de calor. Ao desenvolver o HagiCode, pensamos: já que vamos escrever código de qualquer forma, por que não transformá-lo em um jogo?

## Contexto

Quem já utilizou assistentes de programação com IA provavelmente teve essa experiência: a princípio parece novo e interessante, mas com o tempo você sente que falta algo. A ferramenta em si é muito poderosa, capaz de fazer geração de código, preenchimento automático e correção de bugs, mas... não tem muita personalidade, e com o tempo pode parecer monótono e sem graça.

Mas tudo bem, afinal, quem gostaria de ficar encarando uma ferramenta fria e impessoal todos os dias?

É como jogar um jogo: se for apenas completar uma lista de tarefas, sem desenvolvimento de personagem, sem desbloquear conquistas, sem trabalho em equipe, logo se torna sem graça. As coisas belas ou pessoas bonitas, não é necessário possuí-las, apenas apreciar sua beleza é suficiente. Mas ferramentas de programação nem sequer possuem essa beleza, o que inevitavelmente desanima as pessoas.

Durante o desenvolvimento do [HagiCode](https://hagicode.com), enfrentamos esse desafio. Como uma plataforma colaborativa de múltiplos assistentes de IA, o HagiCode precisa manter os usuários engajados a longo prazo. Mas a realidade é que, mesmo com uma excelente ferramenta, sem conexão emocional, é difícil para os usuários persistirem.

Para resolver essa dor, tomamos uma decisão audaciosa: transformar a programação em um jogo. Não uma simples tabela de pontuação, mas uma experiência de gamificação real estilo RPG. As mudanças resultantes dessa decisão podem ser maiores do que você imagina.

Afinal, humanos sempre precisam de um pouco de cerimônia.

## Por que escolher gamificação

A essência da gamificação não é "adicionar uma tabela de pontuação", mas estabelecer um sistema completo de incentivos que permita aos usuários experimentarem senso de crescimento, conquista e reconhecimento social ao realizar tarefas.

O design de gamificação do HagiCode gira em torno de um conceito central: cada assistente de IA é um "herói", e o usuário é o capitão dessa equipe de heróis. Você lidera esses heróis para conquistar várias "masmorras" (tarefas de programação), e neste processo, os heróis ganham experiência, sobem de nível e desbloqueiam habilidades, enquanto você e sua equipe ganham várias conquistas.

Isso não é uma estratégia de marketing, mas um design cuidadoso baseado na psicologia comportamental humana. Quando tarefas ganham significado e feedback de progresso, o comprometimento e a persistência das pessoas aumentam significativamente.

Como diziam os antigos, "este sentimento pode se tornar uma lembrança, mas naquele momento já estava confuso". Incorporamos essa experiência emocional na ferramenta, tornando a programação não apenas digitar código, mas uma jornada digna de lembrança.

## Sistema de Personagens Hero

Hero é o conceito central do sistema de gamificação do HagiCode. Cada Hero representa um assistente de IA, por exemplo, Claude Code é um Hero, e Codex também é um Hero.

### Os Três Slots do Hero

O Hero possui três slots de equipamento, este design é na verdade bastante engenhoso:

1. **Slot CLI (Profissão Principal)**: Determina as habilidades base do Hero, como se é Claude Code ou Codex
2. **Slot Model (Profissão Secundária)**: Determina o modelo usado, como Claude 4.5 ou Claude 4.6
3. **Slot Style (Estilo)**: Determina o estilo de ação do Hero, como "Estrategista de Fengluo" ou "outro estilo"

A combinação dos três slots pode criar configurações únicas de Hero. Assim como configurar equipamentos em um jogo, você precisa escolher a combinação apropriada com base nas características da tarefa. Afinal, o que se adequa a você é o melhor, isso é semelhante a escolher um caminho na vida, todos os caminhos levam a Roma, apenas alguns caminhos são mais fáceis, outros um pouco mais tortuosos.

### Sistema de Progressão do Hero

Cada Hero possui sua própria XP (pontos de experiência) e nível:

```typescript
type HeroProgressionSnapshot = {
  currentLevel: number;                    // Nível atual
  totalExperience: number;                 // Experiência total
  currentLevelStartExperience: number;     // Experiência inicial do nível atual
  nextLevelExperience: number;             // Experiência necessária para o próximo nível
  experienceProgressPercent: number;       // Percentual de progresso
  remainingExperienceToNextLevel: number;  // Quanta experiência falta para o próximo nível
  lastExperienceGain: number;              // Experiência ganha na última vez
  lastExperienceGainAtUtc?: string | null; // Momento em que ganhou experiência
};
```

Os níveis são divididos em quatro estágios, e a nomenclatura de cada estágio é muito imersiva:

```typescript
export const resolveHeroProgressionStage = (level?: number | null): HeroProgressionStage => {
  const normalizedLevel = Math.max(1, level ?? 1);
  if (normalizedLevel <= 100) return 'rookieSprint';      // Sprint de Iniciante
  if (normalizedLevel <= 300) return 'growthRun';         // Corrida de Crescimento
  if (normalizedLevel <= 700) return 'veteranClimb';      // Escalada de Veterano
  return 'legendMarathon';                                // Maratona Lendária
};
```

De "iniciante" a "lenda", este caminho de progresso oferece aos usuários um senso claro de objetivo e realização. Assim como o crescimento na vida, é preciso passar do ingênuo ao maduro, apenas aqui tornamos esse processo concreto.

### Criando Heróis Personalizados

Ao criar um Hero, você precisa configurar os três slots:

```typescript
const heroDraft: HeroDraft = {
  name: 'Atena',
  icon: 'hero-avatar:storm-03',
  description: 'Estrategista de sabedoria excepcional',
  executorType: AIProviderType.CLAUDE_CODE_CLI,
  slots: {
    cli: {
      id: 'profession-claude-code',
      parameters: { /* Parâmetros relacionados ao CLI */ }
    },
    model: {
      id: 'secondary-claude-4-sonnet',
      parameters: { /* Parâmetros relacionados ao modelo */ }
    },
    style: {
      id: 'fengluo-strategist',
      parameters: { /* Parâmetros relacionados ao estilo */ }
    }
  }
};
```

Cada Hero tem um avatar, descrição e posicionamento profissional únicos, o que dá aos assistentes de IA frios personalidade e calor. Afinal, quem gostaria de lidar com ferramentas sem personalidade?

## Sistema de Masmorras

"Masmorra" é um conceito clássico em jogos, representando desafios que exigem equipe para superar. No HagiCode, cada fluxo de trabalho é uma Dungeon.

### Organização das Dungeons

As Dungeons organizam os fluxos de trabalho em diferentes "masmorras":

- **Masmorra de Geração de Propostas**: Responsável por gerar propostas técnicas
- **Masmorra de Execução de Propostas**: Responsável por executar tarefas das propostas
- **Masmorra de Arquivamento de Propostas**: Responsável por organizar e arquivar propostas concluídas

Cada masmorra tem seu próprio Hero Capitão, o capitão é automaticamente selecionado como o primeiro Hero habilitado.

Na verdade, isso é como a divisão de tarefas na vida, cada pessoa tem seu próprio papel, apenas aqui tornamos essa divisão uma mecânica de jogo.

### Mecanismo de Trabalho em Equipe

Você pode configurar equipes de Hero diferentes para diferentes masmorras:

```typescript
const dungeonRoster: HeroDungeonRoster = {
  scriptKey: 'proposal.generate',
  displayName: 'Geração de Proposta',
  members: [
    { heroId: 'hero-1', name: 'Atena', executorType: 'ClaudeCode' },
    { heroId: 'hero-2', name: 'Apolo', executorType: 'Codex' }
  ]
};
```

Por exemplo, usar Atena (especialista em estratégia) ao gerar propostas, e Apolo (especialista em implementação) ao executar código, assim cada herói pode desempenhar suas especialidades. Como formar uma banda, cada pessoa tem seu próprio instrumento, juntos podem criar uma melodia harmoniosa.

### Controle de Fluxo da Dungeon

As Dungeons usam scriptKeys fixos para identificar diferentes fluxos:

```typescript
// Chaves de script correspondem a diferentes fluxos de trabalho
const dungeonScripts = {
  'proposal.generate': 'Geração de Proposta',
  'proposal.execute': 'Execução de Proposta',
  'proposal.archive': 'Arquivamento de Proposta'
};
```

O fluxo de status das tarefas é: queued (na fila) → dispatching (distribuindo) → dispatched (distribuído). Todo o processo é automatizado, sem necessidade de intervenção manual. Esta é também nossa pequena preguiça, afinal, quem queria gerenciar essas coisas manualmente?

## Sistema de XP e Níveis

XP (pontos de experiência) é o mecanismo de feedback mais central no sistema de gamificação. Os usuários ganham XP completando tarefas, XP faz os heróis subirem de nível, subir de nível desbloqueia novas habilidades, formando um ciclo positivo.

### Maneiras de Obter XP

No HagiCode, XP pode ser obtida através das seguintes atividades:

- Execução de código completada
- Chamadas de ferramenta bem-sucedidas
- Geração de propostas
- Operações de gerenciamento de sessões
- Operações de projeto

Ao completar uma operação válida, o Hero correspondente ganha XP. Como o crescimento na vida, cada passo conta, apenas aqui quantificamos esse crescimento.

### Visualização de Progresso em Tempo Real

O progresso de XP e níveis é visualizado em tempo real:

```typescript
type HeroDungeonMember = {
  heroId: string;
  name: string;
  icon?: string | null;
  executorType: PCode_Models_AIProviderType;
  currentLevel?: number;                    // Nível atual
  totalExperience?: number;                 // Experiência total
  experienceProgressPercent?: number;       // Percentual de progresso
};
```

Os usuários podem ver o nível e progresso de cada Hero a qualquer momento, este feedback imediato é fundamental para o design de gamificação. Afinal, as pessoas sempre precisam de feedback, senão como saber que progrediram?

## Sistema de Conquistas

Conquistas são outro elemento importante na gamificação, fornecendo metas de longo prazo e sensação de marco.

### Tipos de Conquistas

O HagiCode suporta vários tipos de conquistas:

- **Geração de código**: Gerar X linhas de código, gerar Y arquivos
- **Gerenciamento de sessões**: Completar Z conversas
- **Operações de projeto**: Trabalhou em W projetos

Na verdade, essas conquistas são como marcos na vida, apenas as transformamos em uma mecânica de jogo.

### Status de Conquistas

As conquistas têm três estados:

```typescript
type AchievementStatus = 'unlocked' | 'in-progress' | 'locked';
```

Três estados têm distinção visual clara:

- **Desbloqueado**: Gradiente dourado + efeito de brilho
- **Em andamento**: Animação de pulso azul
- **Bloqueado**: Cinza, exibe condições de desbloqueio

Cada conquista exibe claramente as condições de gatilho, permitindo que os usuários saibam o que fazer a seguir. Afinal, quando confuso, ter uma orientação é sempre bom.

### Efeitos de Celebração ao Desbloquear

Quando uma conquista é desbloqueada, é acionada uma animação de celebração. Este reforço positivo gera satisfação de "eu consegui", incentivando-os a continuar avançando. Como pequenas recompensas na vida, embora não grandes, podem deixar as pessoas felizes por muito tempo.

## Relatório de Batalha Diário

Battle Report é um recurso exclusivo do HagiCode, gerando um relatório de estilo batalha em tela cheia ao final de cada dia.

### Conteúdo do Relatório

Battle Report exibe as seguintes informações:

```typescript
type HeroBattleReport = {
  reportDate: string;
  summary: {
    totalHeroCount: number;        // Número total de heróis
    activeHeroCount: number;       // Número de heróis ativos
    totalBattleScore: number;      // Pontuação de batalha total
    mvp: HeroBattleHero;           // Herói mais valioso
  };
  heroes: HeroBattleHero[];        // Dados detalhados de todos os heróis
};
```

- Pontuação total da equipe
- Número de Heróis ativos
- Número de chamadas de ferramenta
- Tempo total de trabalho
- MVP (Herói Mais Valioso)
- Cartões detalhados de cada Hero

### Destaque do MVP

O MVP é o Hero com melhor desempenho do dia, destacado no relatório. Isto não é apenas estatística de dados, mas uma forma de reconhecimento de honra. Afinal, quem não deseja ser reconhecido?

### Cartões Detalhados do Hero

Cada cartão de Hero contém:

- Progresso do nível
- Quantidade de XP ganha
- Número de execuções
- Tempo de uso

Estes dados permitem que os usuários entendam claramente o estado de trabalho da equipe. Afinal, entender os resultados do próprio esforço também é uma forma de satisfação.

## Implementação Técnica

O sistema de gamificação do HagiCode adota uma stack tecnológica moderna e padrões de design. Na verdade, não há nada de especial, apenas escolhemos algumas ferramentas adequadas.

### Escolha da Stack Tecnológica

```typescript
// Frontend usando React + TypeScript
import React from 'react';

// Animações usando Framer Motion
import { AnimatePresence, motion } from 'framer-motion';

// Gerenciamento de estado usando Redux Toolkit
import { useAppDispatch, useAppSelector } from '@/store';

// Componentes de UI usando shadcn/ui
import { Dialog, DialogContent } from '@/components/ui/dialog';
```

Framer Motion é responsável por todos os efeitos de animação, shadcn/ui fornece componentes de UI básicos, Redux Toolkit gerencia estados complexos de gamificação. Afinal, quem deseja fazer bem seu trabalho, deve primeiro afiar suas ferramentas.

### Sistema de Design de UI Gamificado

O HagiCode adota o estilo de design Glassmorphism + Tech Dark:

```css
/* Cor de gradiente principal */
background: linear-gradient(135deg, #22C55E 0%, #25c2a0 50%, #06b6d4 100%);

/* Efeito de vidro */
backdrop-filter: blur(12px);

/* Efeito de brilho */
background: radial-gradient(circle at center, rgba(34, 197, 94, 0.15) 0%, transparent 70%);
```

Gradientes em tons de verdes combinados com efeitos de vidro criam sensação tecnológica e futurista. Afinal, a beleza visual também é parte da experiência do usuário.

### Implementação de Efeitos de Animação

Usando Framer Motion para implementar animações de entrada suaves:

```typescript
<motion.div
  animate={{ opacity: 1, y: 0 }}
  initial={{ opacity: 0, y: 18 }}
  transition={{ duration: 0.35, ease: 'easeOut', delay: index * 0.08 }}
  className="card"
>
  {/* Conteúdo do cartão */}
</motion.div>
```

Cada cartão entra sequencialmente, com atraso de 0.08 segundos, criando um efeito visual suave. Afinal, animações fluidas melhoram a experiência, isso é indiscutível.

### Persistência de Dados

Dados de gamificação são armazenados usando o sistema de armazenamento Grain, garantindo consistência de estado. Mesmo dados granulares como acúmulo de XP do Hero podem ser persistidos com precisão. Afinal, ninguém quer perder experiência acumulada arduamente.

## Guia Prático

### Criando o Primeiro Hero

Criar o primeiro Hero é na verdade bastante simples:

1. Acesse a página de gerenciamento de Hero
2. Clique no botão "Criar Hero"
3. Configure os três slots (CLI, Model, Style)
4. Dê um nome e descrição ao Hero
5. Salve, seu primeiro Hero está criado

Como conhecer um novo amigo, você precisa dar um nome a ele, entender suas características, e então podem aventurar juntos.

### Formando Equipes de Dungeon

Formar equipes também é simples:

1. Acesse a página de gerenciamento de Dungeon
2. Selecione a masmorra para configurar (como "Geração de Proposta")
3. Escolha membros da sua lista de Heróis
4. O sistema seleciona automaticamente o primeiro Hero habilitado como capitão
5. Salve a configuração

Na verdade, este é um processo de formação de equipe, como formar uma equipe na vida, cada pessoa tem seu próprio papel.

### Visualizando Relatórios Diários

Ao final de cada dia, você pode visualizar o Battle Report do dia:

1. Clique no botão "Relatório de Batalha"
2. Exibição em tela cheia dos resultados de trabalho do dia
3. Veja o MVP e dados detalhados de cada Hero
4. Compartilhe com membros da equipe (opcional)

Esta também é uma forma de cerimônia, permitindo que você saiba o quanto se esforçou hoje e quanto falta para o objetivo.

## Considerações e Melhores Práticas

### Otimização de Desempenho

Use React.memo para evitar re-renderizações desnecessárias:

```typescript
const HeroCard = React.memo(({ hero }: { hero: HeroDungeonMember }) => {
  // Implementação do componente
});
```

Afinal, desempenho também é importante, quem quer usar ferramentas travadas?

### Animações Degradáveis

Detecte configurações de preferência de movimento do usuário, fornecendo experiência simplificada para usuários sensíveis:

```typescript
const prefersReducedMotion = useReducedMotion();
const duration = prefersReducedMotion ? 0 : 0.35;
```

Afinal, nem todos gostam de animações, respeitar as preferências dos usuários também é parte do design.

### Compatibilidade Retroativa

Mantenha legacyIds para suportar migração de versões antigas:

```typescript
type HeroDungeonMember = {
  heroId: string;
  legacyIds?: string[];  // Suporta mapeamento de IDs de versões antigas
  // ...
};
```

Afinal, ninguém quer perder dados devido a upgrades de versão.

### Suporte de Internacionalização

Todo o texto usa chaves de tradução i18n, facilitando suporte multilíngue:

```typescript
const displayName = t(`dungeon.${scriptKey}`, { defaultValue: displayName });
```

Afinal, idioma não deve ser uma barreira ao uso.

## Conclusão

Gamificação não é uma simples tabela de pontuação, mas um sistema completo de incentivos. Através do sistema Hero, sistema Dungeon, sistema de XP/níveis, sistema de conquistas e Battle Report, o HagiCode transforma o trabalho de programação em uma jornada heroica cheia de espírito de aventura.

O valor central deste sistema reside em:

- **Conexão emocional**: Transforma assistentes de IA frios em personalidades
- **Feedback positivo**: Cada operação tem feedback imediato
- **Metas de longo prazo**: Níveis e conquistas fornecem caminho de crescimento
- **Identidade de equipe**: Sensação de colaboração em equipe Dungeon
- **Reconhecimento de honra**: Battle Report e exibição de MVP

O design de gamificação torna a programação não monótona, mas uma aventura interessante. Os usuários experimentam o prazer de crescimento de personagem, colaboração em equipe e desbloqueio de conquistas enquanto completam tarefas de código, aumentando assim a retenção e atividade.

Na verdade, a programação em si é uma forma de criação, apenas tornamos esse processo criativo um pouco mais interessante.
