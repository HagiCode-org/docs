## Digitar é pior do que falar, falar é pior do que tirar uma captura de tela — a prática de entrada multimodal do assistente de código AI

> Na verdade, quando se trata de escrever código, por mais rápido que você digite, há um limite. Às vezes, algo que pode ser dito em uma frase exige digitar no teclado por muito tempo; às vezes, uma imagem pode explicar claramente, mas requer muito texto para descrever. Este artigo discute as experiências que tivemos ao desenvolver o [HagiCode](https://hagicode.com) — seja reconhecimento de voz ou upload de imagem, o objetivo é tornar o assistente de código AI um pouco mais fácil de usar, é só isso.

## Contexto

Ao desenvolver o HagiCode, descobrimos um problema — ou melhor, um problema que naturalmente se tornou aparente conforme os usuários usavam mais: depender apenas de digitação pode ser bastante cansativo.

Pense bem, a interação entre usuários e Agent é um cenário central. Mas ter que sentar e digitar no teclado o tempo todo, bem, a eficiência não é muito alta:

1. **Digitar é muito lento**: Alguns problemas complexos, como erros, coisas na interface, etc., levariam meia minuto para digitar, mas na boca podem ser falados em dez segundos. Essa diferença de tempo é bastante desconfortável.

2. **Imagens são mais diretas**: Às vezes a interface apresenta um erro, ou você quer comparar com um design, ou quer mostrar a estrutura do código... "uma imagem vale mais que mil palavras", embora seja um ditado antigo, o princípio continua válido. Deixar a AI "ver" o problema diretamente é muito mais claro do que descrever por muito tempo.

3. **A interação deve ser natural**: Os assistentes de AI atuais devem suportar texto, voz, imagem e outros métodos, certo? Os usuários devem poder usar o que quiserem, isso é que é natural, não é?

Então, pensamos, por que não adicionar recursos de reconhecimento de voz e upload de imagem ao HagiCode, tornando a operação do Agent mais conveniente. Afinal, fazer os usuários digitarem menos caracteres já é bom.

## Sobre o HagiCode

As soluções compartilhadas neste artigo vêm da nossa prática no projeto [HagiCode](https://hagicode.com) — ou melhor, experiências exploradas através de tentativas e erros.

O HagiCode é um projeto de assistente de código AI de código aberto, a ideia é simples: usar tecnologia de AI para melhorar a eficiência de desenvolvimento. Conforme fomos desenvolvendo, descobrimos que a demanda por entrada multimodal é realmente forte — às vezes dizer uma frase é mais rápido do que digitar um monte de caracteres, às vezes uma captura de tela é mais clara do que descrever por muito tempo.

Essas demandas nos impulsionaram para frente, e finalmente surgiram recursos como reconhecimento de voz e upload de imagem. Os usuários podem interagir com a AI da forma mais natural, e essa sensação é muito boa.

## Análise

### Desafios técnicos do reconhecimento de voz

Ao desenvolver o recurso de reconhecimento de voz, encontramos um problema bastante difícil: **a API WebSocket do navegador não suporta cabeçalhos HTTP personalizados**.

E o serviço de reconhecimento de voz que escolhemos foi a API de reconhecimento de voz Doubao do ByteDance. Esta API justamente exige que as informações de autenticação sejam passadas através de cabeçalhos HTTP, como `accessToken`, `secretKey`, etc. Agora surgiu o conflito técnico:

```javascript
// A API WebSocket do navegador não suporta a seguinte forma
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // Não suportado
  }
});
```

As soluções diante de nós são basicamente duas:

1. **Solução de parâmetros de consulta de URL**: Colocar as informações de autenticação na URL
   - A vantagem é que é simples de implementar
   - A desvantagem é que as credenciais ficam expostas no frontend, segurança ruim; e algumas APIs exigem obrigatoriamente verificação por header

2. **Solução de proxy backend**: Implementar proxy WebSocket no backend
   - A vantagem é que as credenciais são armazenadas com segurança no backend; totalmente compatível com os requisitos da API
   - A desvantagem é que é um pouco mais complexo de implementar

Finalmente escolhemos a **solução de proxy backend**. Afinal, a segurança é uma linha de base que não pode ser comprometida — disso, ninguém pode enganar ninguém.

### Requisitos funcionais do upload de imagem

Quanto ao recurso de upload de imagem, nossas necessidades são realmente bastante simples:

1. **Várias formas de upload**: Clique para selecionar arquivo, arrastar para upload, colar da área de transferência, deve ter tudo, certo?
2. **Validação de arquivo**: Limitação de tipo (PNG, JPG, WebP, GIF), limitação de tamanho (5-10MB), isso é operação básica
3. **Experiência do usuário**: Progresso de upload, visualização, avisos de erro, as pessoas precisam saber o que está acontecendo
4. **Segurança**: Validação no servidor, prevenção de upload de arquivos maliciosos, isso é coisa séria

## Soluções

### Reconhecimento de voz: Arquitetura de proxy WebSocket

Desenvolvemos uma solução de reconhecimento de voz de três camadas, digamos, encontramos um caminho:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**Implementação dos componentes principais**:

1. **Processador AudioWorklet no frontend**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Reamostrar para 16kHz (requisito da API Doubao)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Acumular amostras em blocos de 500ms
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Converter para 16-bit PCM e enviar
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **Manipulador WebSocket no backend** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **Componente VoiceTextArea no frontend**:

```tsx
export const VoiceTextArea = forwardRef<HTMLTextAreaElement, VoiceTextAreaProps>(
  ({ value, onChange, onTextRecognized, maxDuration }, ref) => {
    const { isRecording, interimText, volume, duration, startRecording, stopRecording } =
      useVoiceRecording({ onTextRecognized, maxDuration });

    return (
      <div className="flex gap-2">
        {/* Botão de voz */}
        <button onClick={handleButtonClick}>
          {isRecording ? <VolumeWaveform volume={volume} /> : <Mic />}
        </button>
        {/* Campo de entrada de texto */}
        <textarea value={displayValue} onChange={handleChange} />
      </div>
    );
  }
);
```

### Upload de imagem: Componente de upload de múltiplas formas

Fizemos um componente de upload de imagem completo em recursos, suportando três formas de upload, digamos, cobrimos todos os cenários comuns de usuários.

**Características principais**:

1. **Três formas de upload**:

```tsx
// Upload por clique
const handleClick = () => fileInputRef.current?.click();

// Upload por arrastar
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Colar da área de transferência
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **Validação no frontend**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **Processamento de upload no backend** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Validar
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Salvar arquivo
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## Guia prático

### Como usar o reconhecimento de voz

1. **Configure o serviço de reconhecimento de voz**:
   - Vá para a página de configurações de reconhecimento de voz
   - Configure o `AppId` e `AccessToken` do Doubao Voice
   - (Opcional) Configure palavras-chave para melhorar a precisão de reconhecimento de termos técnicos

2. **Use na caixa de entrada**:
   - Clique no ícone de microfone à esquerda da caixa de entrada
   - Comece a falar quando vir a animação de forma de onda
   - Clique no ícone novamente para parar a gravação
   - O resultado do reconhecimento será inserido automaticamente na posição do cursor

3. **Exemplo de configuração de palavras-chave**:

```text
TypeScript
React
useState
useEffect
```

### Como usar o upload de imagem

1. **Formas de upload**:
   - Clique no botão de upload para selecionar o arquivo
   - Arraste a imagem diretamente para a área de upload
   - Use `Ctrl+V` para colar a captura de tela da área de transferência

2. **Formatos suportados**: PNG, JPG, JPEG, WebP, GIF
3. **Limite de tamanho**: 5MB por padrão (configurável)

### Considerações

1. **Reconhecimento de voz**:
   - Requer permissão de microfone
   - Recomenda-se usar em ambiente silencioso
   - Duração máxima de gravação suportada é de 300 segundos (configurável)

2. **Upload de imagem**:
   - Suporta apenas formatos de imagem comuns
   - Atenção ao limite de tamanho do arquivo
   - URLs de visualização serão geradas automaticamente após o upload

3. **Considerações de segurança**:
   - Credenciais de reconhecimento de voz são armazenadas no backend
   - Upload de imagem tem validação rigorosa no servidor
   - Em produção, recomenda-se usar HTTPS/WSS

## Resumo

Depois de adicionar reconhecimento de voz e upload de imagem, a experiência do usuário do HagiCode melhorou significativamente. Os usuários agora podem interagir com a AI de forma mais natural — falar em vez de digitar, captura de tela em vez de descrição. Essa sensação, digamos, é como finalmente encontrar uma forma mais confortável de comunicação.

Ao desenvolver este recurso, encontramos o problema de que o WebSocket do navegador não suporta cabeçalhos personalizados, e finalmente resolvemos através da solução de proxy backend. Esta solução não apenas garante a segurança, mas também estabelece a base para integração futura de outros serviços WebSocket que exigem autenticação —也算 um ganho inesperado.

O componente de upload de imagem também, usando várias formas de upload, permite que os usuários escolham a mais conveniente de acordo com o cenário. Clique, arraste ou cole diretamente, todos podem concluir o upload rapidamente. Todos os caminhos levam a Roma, apenas alguns caminhos são mais fáceis, outros um pouco mais sinuosos.

"Digitar é pior do que falar, falar é pior do que tirar uma captura de tela", essa frase é bastante apropriada aqui. Se você também está desenvolvendo produtos de assistente AI semelhantes, espero que essas experiências possam ajudá-lo, mesmo que seja só um pouco.
