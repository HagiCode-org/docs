В этом руководстве рассматривается установка и использование HagiCode Desktop. Это официальный клиент для настольных компьютеров, который хранит путь локальной установки по умолчанию и ежедневные элементы управления в одном месте.

:::tip[Рекомендуемый метод]
Рабочий стол — это локальный путь по умолчанию для отдельных пользователей и разработчиков, потому что:
- установщик сохраняет поток по умолчанию в одном управляемом пути
- зависимости и сервисы управляются в приложении
- пользовательский интерфейс упрощает проверку состояния и операций
- встроенное управление версиями упрощает переключение
- данные остаются локальными по умолчанию
:::

## Что такое HagiCode Desktop?

HagiCode Desktop — это официальное настольное приложение HagiCode, разработанное для систем Windows, macOS и Linux. Он объединяет основные функции HagiCode, а установщик сохраняет путь установки по умолчанию в одном управляемом процессе.

### Основные преимущества

- **Установка в один клик**: автоматически выполните все шаги настройки с помощью мастера установки.
- **Автоматическая подготовка среды выполнения**. Программа установки по умолчанию уже включает в себя необходимую среду выполнения, поэтому вам не нужно устанавливать отдельную `.NET` сначала время выполнения
- **Управление службами**: встроенное управление сервером HagiCode, простой запуск и остановка служб.
- **Переключение версий**: поддержка управления несколькими версиями, свободное переключение между различными версиями.
- **Локальное управление**: все данные хранятся локально, защищая вашу конфиденциальность.
- **Автоматические обновления**: автоматически обнаруживает и загружает последнюю версию.

### Применимые сценарии

Рабочий стол подойдет, если:
- Отдельные разработчики, желающие использовать HagiCode локально.
- Пользователи, которым необходимо использовать HagiCode в автономном режиме
- Нетехнические пользователи, желающие упростить процесс установки.
- Разработчики, которым необходимо часто переключаться между разными версиями

### Другие способы загрузки

### Выберите правильную версию

Выберите соответствующий установщик для вашей операционной системы:
- **Windows**: загрузите `.exe` установщик
- **macOS**: загрузите `.dmg` установщик
- **Linux**: загрузите `.AppImage` или `.tar.gz` установочный пакет

:::примечание[Системные требования]
- **Windows**: Windows 10 или более поздняя версия (64-разрядная версия).
- **macOS**: macOS 10.15 или более поздняя версия.
- **Linux**: основные дистрибутивы Linux (Ubuntu 20.04+, Fedora 33+ и т. д.).
:::

:::tip[Предварительные требования по умолчанию]
Для последней локальной установки HagiCode обычно не требуется готовить отдельный файл. `.NET` время выполнения. Desktop уже включает среду выполнения, необходимую для потока по умолчанию, и по умолчанию инициализирует поддерживаемый локальный путь SQLite.
:::

## Примечания по установке и запуску

:::предупреждение[уведомление macOS Gatekeeper]
Если вы уже установили приложение по пути по умолчанию `/Applications/HagiCode.app` но macOS показывает «приложение повреждено», «невозможно открыть» или подобное предупреждение Gatekeeper при первом запуске. Обычной причиной является `com.apple.quarantine` атрибут, который macOS добавляет к приложениям, загруженным из Интернета, а не сбой в самом установщике HagiCode Desktop.

Запустите следующую команду в Терминале. Для этого требуются права администратора, и он рекурсивно удаляет атрибут карантина из приложения в пути установки по умолчанию:

```bash
sudo xattr -dr com.apple.quarantine /Applications/HagiCode.app
```

После завершения команды перезапустите HagiCode Desktop.
:::

## Запустить рабочий стол

После загрузки откройте установщик или пакет приложения и следуйте инструкциям на экране. Desktop подготавливает среду выполнения по поддерживаемому в настоящее время локальному пути, управляет сервером HagiCode и показывает ход выполнения и состояние, когда требуются сетевые загрузки.

:::примечание[Требования к сети]
Убедитесь, что ваше сетевое соединение доступно. Desktop может потребоваться получить информацию о версии или пакеты компонентов из Интернета; Если загрузка не удалась, проверьте настройки сети, прокси-сервера или брандмауэра и повторите попытку.
:::

## Управление версиями

HagiCode Desktop поддерживает управление несколькими версиями, что позволяет вам свободно переключаться между различными версиями.

![Управление версиями](../../img/screenshots/installation/install-desktop/12-version-management/original.png)

### Переключить версию

1. Открыть рабочий стол HagiCode
2. Перейдите на страницу «Управление версиями».
3. Выберите версию, которую вы хотите переключить
4. Нажмите кнопку «Сменить версию».
5. Дождитесь завершения переключения

:::note[Примечания по переключению версий]
- При переключении версий сервер HagiCode автоматически перезапустится.
- Обязательно сохраните всю работу перед переключением
- Конфигурации проекта могут отличаться в разных версиях.
:::

## Запуск и остановка службы

HagiCode Desktop предоставляет простой способ управления службой сервера HagiCode.

### Запустить службу

На домашней странице настольного приложения нажмите кнопку «Запустить службу», чтобы запустить HagiCode Server.

![Запустить службу](../../img/screenshots/installation/install-desktop/13-start-service-homepage/original.png)

### Остановить службу

Чтобы остановить сервер HagiCode, нажмите кнопку «Остановить службу» на домашней странице.

:::tip[Состояние службы]
Приложение для настольного компьютера отображает статус работы сервера HagiCode в режиме реального времени, включая:
- Статус службы (работает/остановлено)
- Номер сервисного порта
- Информация о текущей версии
:::

## Устранение неполадок

### Ошибка установки

Если во время установки возникли проблемы:

1. **Проверьте сетевое подключение**: убедитесь, что у вас есть доступ к Интернету.
2. **Проверьте место на диске**: убедитесь, что на диске достаточно места.
3. **Запуск от имени администратора**. В Windows щелкните правой кнопкой мыши установщик и выберите «Запуск от имени администратора».
4. **Просмотр файлов журналов**. Программа установки создает подробные файлы журналов, которые помогают диагностировать проблемы.

### Ошибка установки зависимостей

Если установка зависимостей не удалась:

1. **Проверьте совместимость системы**: убедитесь, что ваша операционная система соответствует минимальным требованиям.
2. **Проверьте настройки сети и брандмауэра**. Убедитесь, что установщик может получить необходимые загрузки и не блокируется брандмауэром.
3. **Подтвердите разрешения установщика**. В Windows попробуйте запустить установщик от имени администратора, чтобы у него были достаточные разрешения.
4. **Перезапустите приложение и повторите попытку**: закройте и снова откройте HagiCode Desktop, затем повторите этап установки.
5. **Просмотр журналов**: просмотрите подробные журналы ошибок в приложении для ПК.

### Ошибка запуска службы

Если сервер HagiCode не запускается:

1. **Проверьте занятость порта**: убедитесь, что порт 45000 не занят другими программами.
2. **Проверьте настройки брандмауэра**: убедитесь, что брандмауэр разрешает доступ к сети сервера HagiCode.
3. **Перезапустите приложение**: попробуйте перезапустить HagiCode Desktop.
4. **Просмотр журналов**: просмотр подробных журналов ошибок в приложении для ПК.
