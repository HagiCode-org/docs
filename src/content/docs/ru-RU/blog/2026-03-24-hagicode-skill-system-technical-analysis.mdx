## Технический анализ системы HagiCode Skill: как создать масштабируемую платформу для управления AI-навыками

> В этой статье подробно рассматриваются архитектурный дизайн и реализация системы управления Skill (навыками) в проекте HagiCode, включая технические детали реализации четырёх основных функций: локального глобального управления, поиска на рынке, интеллектуальных рекомендаций и управления доверенными поставщиками.

## Предпосылки

В области AI-помощников по коду вопрос о том, как расширить границы возможностей AI, всегда был центральной темой. Собственные возможности Claude Code по assisting с кодом довольно сильны, но разные команды разработчиков и разные технологические стеки часто требуют профессиональных возможностей для конкретных сценариев — например, обработки Docker-развертывания, оптимизации баз данных, генерации фронтенд-компонентов и т.д. В таких случаях система Skill (навыков) становится особенно важной.

В процессе разработки проекта HagiCode мы столкнулись с аналогичными вызовами: как позволить Claude Code «изучать» новые профессиональные навыки, как человек, сохраняя при этом хороший пользовательский опыт и техническую поддерживаемость? В конце концов, эта проблема и сложная, и простая одновременно. Вокруг этой проблемы мы разработали и реализовали полную систему управления Skill.

В этой статье будет подробно описана техническая архитектура и основная реализация этой системы. Она подходит для разработчиков, интересующихся расширяемостью AI и интеграцией инструментов командной строки. Возможно, она вам пригодится, а возможно и нет, но в любом случае она написана.

Технологический стек проекта включает ASP.NET Core, распределённую среду Orleans, TanStack Start + React для фронтенда, а также подсистему управления Skill, о которой пойдёт речь в этой статье.

## Обзор системной архитектуры

Система Skill использует архитектуру с разделением фронтенда и бэкенда. В общем-то, ничего особенного.

**Фронтенд-часть** использует TanStack Start + React для создания пользовательского интерфейса, состояние управляется через Redux Toolkit, четыре основные функции соответствуют четырём компонентам Tab: локальные навыки, галерея рынка, интеллектуальные рекомендации, доверенные поставщики. Такая设计 на самом деле сделана ради пользовательского опыта.

**Бэкенд-часть** основана на ASP.NET Core + ABP Framework, использует Orleans Grain для реализации распределённого управления состоянием. Онлайн-API-клиент инкапсулирует интерфейс `IOnlineApiClient` для общения с удалённым сервисом каталога навыков.

Принцип дизайна общей архитектуры — «разделение выполнения команд и бизнес-логики». Через паттерн адаптера детали выполнения npm/npx-команд скрыты в независимом модуле. Ведь кто хочет видеть кучу командных строк, разбросанных по коду?

## Основная функция 1: Локальное глобальное управление

Локальное глобальное управление — это базовый функциональный модуль, отвечающий за список установленных навыков и поддерживающий операцию удаления. Ничего сложного, просто всё сделать хорошо.

### Техническое решение

Реализация находится в `LocalSkillsTab.tsx` и `LocalSkillCommandAdapter.cs`. Основная идея — инкапсулировать команду `npx skills`, разобрать её JSON-вывод и преобразовать во внутреннюю структуру данных. Говорить легко, на деле тоже просто.

```csharp
public async Task<IReadOnlyList<LocalSkillInventoryResponseDto>> GetLocalSkillsAsync(
    CancellationToken cancellationToken = default)
{
    var result = await _commandAdapter.ListGlobalSkillsAsync(cancellationToken);
    return result.Skills.Select(skill => new LocalSkillInventoryResponseDto
    {
        Name = skill.Name,
        Version = skill.Version,
        Source = skill.Source,
        InstalledPath = skill.InstalledPath,
        Description = skill.Description
    }).ToList();
}
```

Поток данных очень чёткий: фронтенд отправляет запрос → SkillGalleryAppService принимает → LocalSkillCommandAdapter выполняет npx команду → разбирает JSON-результат → возвращает DTO-объект. Одно звено за другим, нечего сказать.

Удаление навыка использует команду `npx skills remove -g <skillName> -y`, система автоматически обрабатывает зависимости и очистку. Метаданные установки хранятся в `managed-install.json` в каталоге навыка, записывая время установки, исходную версию и т.д., что удобно для последующих обновлений и аудита. Ведь некоторые вещи лучше записать.

### Подробное описание процесса установки

Установка навыка включает координацию нескольких шагов. Как сказать, не так уж и сложно:

```csharp
public async Task<SkillInstallResultDto> InstallAsync(
    SkillInstallRequestDto request,
    CancellationToken cancellationToken = default)
{
    // 1. Нормализация ссылки установки
    var normalized = _referenceNormalizer.Normalize(
        request.SkillId,
        request.Source,
        request.SkillSlug,
        request.Version);

    // 2. Проверка предварительных условий
    await _prerequisiteChecker.CheckAsync(cancellationToken);

    // 3. Получение блокировки установки
    using var installLock = await _lockProvider.AcquireAsync(normalized.SkillId);

    // 4. Выполнение команды установки
    var result = await _installCommandRunner.ExecuteAsync(
        new SkillInstallCommandExecutionRequest
        {
            Command = $"npx skills add {normalized.FullReference} -g -y",
            Timeout = TimeSpan.FromMinutes(4)
        },
        cancellationToken);

    // 5. Сохранение метаданных установки
    await _metadataStore.WriteAsync(normalized.SkillPath, request);

    return new SkillInstallResultDto { Success = result.Success };
}
```

Здесь использовано несколько ключевых паттернов дизайна: **нормализатор ссылок** отвечает за преобразование различных входных форматов (например, `tanweai/pua`, `@opencode/docker-skill`) в унифицированное внутреннее представление; **механизм блокировки установки** гарантирует, что для одного навыка одновременно выполняется только одна операция установки; **потоковый вывод** в реальном времени через Server-Sent Events отправляет прогресс установки на фронтенд, пользователь может видеть реальный лог как в терминале.

Эти паттерны дизайна в конечном итоге сделаны для того, чтобы всё стало проще.

## Основная функция 2: Поиск на рынке

Поиск на рынке позволяет пользователям находить и устанавливать навыки от сообщества. Ведь возможности одного человека ограничены, а мудрость многих бесконечна.

### Техническое решение

Функция поиска зависит от онлайн-API `https://api.hagicode.com/v1/skills/search`. Для повышения скорости ответа система реализует механизм кэширования. Кэш — это как память: что-то запомнил, в следующий раз не нужно напрягаться.

```csharp
private async Task<IReadOnlyList<SkillGallerySkillDto>> SearchCatalogAsync(
    string query,
    CancellationToken cancellationToken,
    IReadOnlySet<string>? allowedSources = null)
{
    var cacheKey = $"skill_search:{query}:{string.Join(",", allowedSources ?? Array.Empty<string>())}";

    if (_memoryCache.TryGetValue(cacheKey, out var cached))
        return (IReadOnlyList<SkillGallerySkillDto>)cached!;

    var response = await _onlineApiClient.SearchAsync(
        new SearchSkillsRequest
        {
            Query = query,
            Limit = _options.LimitPerQuery,
        },
        cancellationToken);

    var results = response.Skills
        .Where(skill => allowedSources is null || allowedSources.Contains(skill.Source))
        .Select(skill => new SkillGallerySkillDto { ... })
        .ToList();

    _memoryCache.Set(cacheKey, results, TimeSpan.FromMinutes(10));
    return results;
}
```

Результаты поиска поддерживают фильтрацию по доверенным источникам, показывая только источники навыков, которым доверяет пользователь. Предустановленные исходные запросы используются для инициализации каталога, например «popular», «recent» и т.д., чтобы пользователь при первом открытии мог увидеть рекомендуемые популярные навыки. Ведь первое впечатление всё-таки важно.

## Основная функция 3: Интеллектуальные рекомендации

Интеллектуальные рекомендации — это самая сложная функция в системе. Она может автоматически рекомендовать наиболее подходящие навыки на основе текущего проекта пользователя. Сложно, но реализовать стоит.

### Процесс рекомендации

Весь процесс рекомендации делится на пять этапов:

```
1. Построение контекста проекта
   ↓
2. AI генерирует поисковые запросы
   ↓
3. Параллельный поиск онлайн-каталога
   ↓
4. AI ранжирует кандидатов
   ↓
5. Возврат списка рекомендаций
```

Сначала система анализирует технический стек, язык программирования, структуру домена и другие характеристики проекта, создавая «портрет проекта». Этот портрет, как резюме человека, записывает все характеристики.

Затем используется AI Grain для генерации целевых поисковых запросов. Здесь дизайн довольно интересен — не спрашивать AI напрямую «какие навыки порекомендовать», а позволить ему сначала подумать «какие поисковые слова могут найти соответствующие навыки». Ведь иногда способ задать вопрос важнее самого ответа:

```csharp
var queryGeneration = await aiGrain.GenerateSkillRecommendationQueriesAsync(
    projectContext,      // контекст проекта
    locale,              // языковые предпочтения пользователя
    maxQueries,           // максимальное количество запросов
    effectiveSearchHero); // выбор AI-модели
```

Затем эти поисковые запросы выполняются параллельно, получается список навыков-кандидатов. Параллельная обработка в конечном итоге сделана для экономии времени.

Наконец, используется другой AI Grain для ранжирования навыков-кандидатов. На этом этапе комплексно учитываются релевантность навыка и проекта, статус доверия, исторические предпочтения пользователя и другие факторы:

```csharp
var ranking = await aiGrain.RankSkillRecommendationsAsync(
    projectContext,
    candidates,
    installedSkillNames,
    locale,
    maxRecommendations,
    effectiveRankingHero);

response.Items = MergeRecommendations(projectContext, candidates, ranking, maxRecommendations);
```

### Механизм отката

AI-модель может отвечать медленно или временно быть недоступной. Ведь даже у лучших систем бывают сбои. Для этого система спроектировала детерминированный механизм отката: когда AI-сервис недоступен, для генерации рекомендаций используется эвристический алгоритм на основе правил, например, вывод возможных необходимых навыков из зависимостей в package.json.

Этот механизм отката, по сути, просто оставляет системе путь к отступлению.

## Основная функция 4: Управление доверенными поставщиками

Управление доверенными поставщиками позволяет пользователям контролировать, какие источники навыков являются доверенными. Ведь доверие — это то, что нужно контролировать самому.

### Правила сопоставления

Доверенные поставщики поддерживают два правила сопоставления: **точное сопоставление** (exact) и **префиксное сопоставление** (prefix).

```csharp
public static TrustedSkillProviderResolutionSnapshot Resolve(
    TrustedSkillProviderSnapshot snapshot,
    string source)
{
    var normalizedSource = Normalize(source);

    foreach (var entry in snapshot.Entries.OrderBy(e => e.SortOrder))
    {
        if (!entry.IsEnabled) continue;

        foreach (var rule in entry.MatchRules)
        {
            bool isMatch = rule.MatchType switch
            {
                TrustedSkillProviderMatchRuleType.Exact
                    => string.Equals(normalizedSource, Normalize(rule.Value),
                        StringComparison.OrdinalIgnoreCase),
                TrustedSkillProviderMatchRuleType.Prefix
                    => normalizedSource.StartsWith(Normalize(rule.Value) + "/",
                        StringComparison.OrdinalIgnoreCase),
                _ => false
            };

            if (isMatch)
                return new TrustedSkillProviderResolutionSnapshot
                {
                    IsTrustedSource = true,
                    ProviderId = entry.ProviderId,
                    DisplayName = entry.DisplayName
                };
        }
    }

    return new TrustedSkillProviderResolutionSnapshot { IsTrustedSource = false };
}
```

Предустановленные доверенные поставщики включают Vercel, Azure, anthropics, Microsoft, browser-use и другие известные организации и проекты. Пользовательские поставщики могут быть добавлены через файл конфигурации, указывая ID поставщика, отображаемое имя, ярлык значка, правила сопоставления и т.д. Ведь мир большой, не может быть только несколько доверенных.

### Реализация персистентности

Конфигурация доверия использует персистентное хранилище Orleans Grain:

```csharp
public class TrustedSkillProviderGrain : Grain<TrustedSkillProviderState>,
    ITrustedSkillProviderGrain
{
    public async Task UpdateConfigurationAsync(TrustedSkillProviderSnapshot snapshot)
    {
        State.Snapshot = snapshot;
        await WriteStateAsync();
    }

    public Task<TrustedSkillProviderSnapshot> GetConfigurationAsync()
    {
        return Task.FromResult(State.Snapshot);
    }
}
```

Преимущество этого способа в том, что изменения конфигурации автоматически синхронизируются со всеми узлами, без необходимости вручную обновлять кэш. Ведь автоматизация в конечном итоге сделана для того, чтобы люди меньше беспокоились.

## Ключевые технические решения

### Паттерн адаптера выполнения команд

Системе Skill нужно выполнять различные npx команды. Если эту логику разбросать по разным местам, код станет трудно поддерживать. Поэтому мы разработали интерфейс адаптера. Паттерны дизайна в конечном итоге сделаны для лучшей поддерживаемости кода:

```csharp
public interface ISkillInstallCommandRunner
{
    Task<SkillInstallCommandExecutionResult> ExecuteAsync(
        SkillInstallCommandExecutionRequest request,
        CancellationToken cancellationToken = default);
}
```

Разные команды имеют разные реализации исполнителя, все реализуют один и тот же интерфейс, что удобно для тестирования и замены.

### Потоковый вывод SSE

Прогресс установки в реальном времени отправляется на фронтенд через Server-Sent Events:

```csharp
public async Task InstallWithProgressAsync(
    SkillInstallRequestDto request,
    IServerStreamWriter<SkillInstallProgressEventDto> stream,
    CancellationToken cancellationToken)
{
    var process = new Process
    {
        StartInfo = new ProcessStartInfo
        {
            FileName = "npx",
            Arguments = $"skills add {request.FullReference} -g -y",
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false
        }
    };

    process.OutputDataReceived += async (sender, e) =>
    {
        await stream.WriteAsync(new SkillInstallProgressEventDto
        {
            EventType = "output",
            Data = e.Data ?? string.Empty
        });
    };

    process.Start();
    process.BeginOutputReadLine();
    await process.WaitForExitAsync(cancellationToken);
}
```

Пользователь на фронтенде может видеть реальный вывод как в терминале, опыт очень интуитивный. Ведь реальная обратная связь успокаивает.

## Практическое руководство

### Установка community-навыков

На примере установки навыка `pua` (это популярный community-навык):

1. Откройте ящик Skills, переключитесь на вкладку «Skill Gallery»
2. Введите «pua» для поиска
3. Нажмите на результат поиска, чтобы увидеть подробности о навыке
4. Нажмите кнопку «Install» для установки
5. Переключитесь на вкладку «Local Skills», чтобы подтвердить успешную установку

Команда установки — `npx skills add tanweai/pua -g -y`, система автоматически обработает все детали. На самом деле не так много шагов, просто делайте по порядку.

### Добавление пользовательского доверенного источника

Если у вашей команды есть собственный репозиторий навыков, можно добавить его как доверенный источник:

```yaml
providerId: "my-team"
displayName: "My Team Skills"
badgeLabel: "MyTeam"
isEnabled: true
sortOrder: 100
matchRules:
  - matchType: "prefix"
    value: "my-team/"
  - matchType: "exact"
    value: "my-team/special-skill"
```

Так все навыки от вашей команды будут показывать значок доверия, пользователи могут устанавливать их с большей уверенностью. Ведь помеченные вещи всегда успокаивают.

### Основы разработки навыков

Для создания пользовательского навыка нужно соблюдать следующую структуру:

```
my-skill/
├── SKILL.md          # метаданные навыка (YAML front matter)
├── index.ts          # вход навыка
├── agents/           # конфигурации поддерживаемых агентов
└── references/       # справочные ресурсы
```

Пример формата `SKILL.md`:

```yaml
---
name: my-skill
description: A brief description of what this skill does
---

# My Skill

Detailed documentation...
```

## Меры предосторожности

1. **Требования к сети**: для поиска и установки навыков необходим доступ к `api.hagicode.com` и npm registry
2. **Версия Node.js**: рекомендуется использовать Node.js 18 или выше
3. **Требования к разрешениям**: необходимы разрешения на глобальную установку npm
4. **Конкурентный контроль**: для одного навыка одновременно может выполняться только одна операция установки или удаления
5. **Настройка таймаута**: время таймаута операции установки по умолчанию составляет 4 минуты, в сложных сценариях может потребоваться корректировка

Эти меры предосторожности в конечном итоге сделаны для того, чтобы всё шло гладко.

## Заключение

В этой статье представлена полная реализация системы управления Skill в проекте HagiCode. Эта система через архитектуру с разделением фронтенда и бэкенда, паттерн адаптера, распределённое управление состоянием Orleans и другие технические средства реализует:

- **Локальное глобальное управление**: через инкапсуляцию команды npx skills предоставляет унифицированный интерфейс управления навыками
- **Поиск на рынке**: использует онлайн-API и механизм кэширования для быстрого обнаружения community-навыков
- **Интеллектуальные рекомендации**: сочетая возможности AI, рекомендирует наиболее подходящие навыки на основе контекста проекта
- **Управление доверием**: гибкая система конфигурации позволяет пользователям контролировать границы доверия

这套设计思路不仅适用于 управления Skill, но и имеет справочное значение для любых сценариев, требующих интеграции инструментов командной строки, сочетания локального хранилища и онлайн-сервисов.
