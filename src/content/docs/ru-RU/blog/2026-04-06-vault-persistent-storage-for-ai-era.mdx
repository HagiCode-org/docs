## Как копировать проекты в эпоху ИИ: система постоянного хранения Vault для перекрёстных проектов

> В эпоху ИИ-сопровождаемой разработки, как позволить ИИ-ассистенту лучше понять наши учебные ресурсы? Проект HagiCode реализует единый, понятный для ИИ уровень абстракции хранения знаний через систему Vault, что значительно повышает эффективность обучения копированию проектов.

## Предпосылки

В эпоху ИИ способы, которыми разработчики изучают новые технологии и архитектуры, претерпевают глубокие изменения. "Копирование проектов" — то есть углублённое изучение и изучение кода, архитектуры и паттернов проектирования превосходных проектов с открытым исходным кодом — стало эффективным методом обучения. По сравнению с традиционным чтением книг и просмотром видео, прямое чтение и запуск высококачественных проектов с открытым исходным кодом позволяет вам быстрее понять инженерные практики реального мира.

Однако такой способ обучения также сталкивается со множеством проблем.

Учебные материалы слишком разбросаны. Ваши заметки могут быть в Obsidian, репозитории кода разбросаны по разным папкам, а история диалогов ИИ-ассистента — это отдельный силос данных. Когда вы хотите, чтобы ИИ помог вам проанализировать какой-то проект, вам нужно вручную копировать фрагменты кода, упорядочивать контекст, этот процесс довольно утомительный.

На самом деле ещё более проблематичным является разрыв контекста. ИИ-ассистент не может напрямую получить доступ к вашим локальным учебным ресурсам, каждый диалог требует повторного предоставления фоновой информации. Кроме того, копируемые репозитории кода обновляются очень быстро, ручная синхронизация подвержена ошибкам, и трудно обмениваться знаниями между несколькими учебными проектами.

Эти проблемы по сути вызваны "силосами данных". Если бы существовал единый уровень абстракции хранения, позволяющий ИИ-ассистенту понимать и получать доступ ко всем вашим учебным ресурсам, проблема была бы решена.

В нашей повседневной разработке нам часто нужно изучать и ссылаться на различные проекты с открытым исходным кодом. Чтобы позволить ИИ-ассистенту лучше понять эти учебные ресурсы, мы разработали систему постоянного хранения Vault для перекрёстных проектов.

Это решение прошло практическую проверку в HagiCode. Если вы также сталкиваетесь с подобными проблемами управления знаниями, я надеюсь, что этот опыт даст вам некоторое вдохновение. Ведь в некоторых ямах уже наступили, нужно оставить что-то для последующих.

## Концепция проектирования системы Vault

Основная идея системы Vault очень проста: создать единый, понятный для ИИ уровень абстракции хранения знаний. С точки зрения реализации, система обладает несколькими ключевыми характеристиками.

### Поддержка нескольких типов

Система поддерживает четыре типа vault, каждый из которых соответствует различным сценариям использования:

```typescript
// folder: универсальный тип папки
export const DEFAULT_VAULT_TYPE = 'folder';

// coderef: тип, специально используемый для копирования кодовых проектов
export const CODEREF_VAULT_TYPE = 'coderef';

// obsidian: интеграция с программой заметок Obsidian
export const OBSIDIAN_VAULT_TYPE = 'obsidian';

// system-managed: vault, управляемый автоматически системой
export const SYSTEM_MANAGED_VAULT_TYPE = 'system-managed';
```

Среди них тип **coderef** — наиболее часто используемый в HagiCode. Он специально разработан для копирования кодовых проектов и предоставляет стандартизированную структуру каталогов и понятные для ИИ метаданные.

### Механизм постоянного хранения

Реестр Vault постоянно хранится в формате JSON, гарантируя, что конфигурация остаётся доступной после перезапуска приложения:

```csharp
public class VaultRegistryStore : IVaultRegistryStore
{
    private readonly string _registryFilePath;

    public VaultRegistryStore(IConfiguration configuration, ILogger<VaultRegistryStore> logger)
    {
        var dataDir = configuration["DataDir"] ?? "./data";
        var absoluteDataDir = Path.IsPathRooted(dataDir)
            ? dataDir
            : Path.GetFullPath(Path.Combine(Directory.GetCurrentDirectory(), dataDir));

        _registryFilePath = Path.Combine(absoluteDataDir, "personal-data", "vaults", "registry.json");
    }
}
```

Преимущество этого дизайна — простота и надёжность. Формат JSON читаем для людей, удобен для отладки и ручного изменения; хранение в файловой системе избегает сложности базы данных, снижает зависимости системы. Ведь иногда простое — наилучшее.

### Интеграция ИИ-контекста

Самое важное — система может автоматически внедрять информацию о vault в контекст ИИ-предложения:

```typescript
export function buildTargetVaultsText(
  vaults: VaultForText[],
  template: VaultPromptTemplate = DEFAULT_VAULT_PROMPT_TEMPLATE,
): string {
  const readOnlyVaults = vaults.filter((vault) => vault.accessType === 'read');
  const editableVaults = vaults.filter((vault) => vault.accessType === 'write');

  if (readOnlyVaults.length === 0 && editableVaults.length === 0) {
    return '';
  }

  const sections = [
    buildVaultSection(readOnlyVaults, template.reference),
    buildVaultSection(editableVaults, template.editable),
  ].filter(Boolean);

  return `\n\n### ${template.heading}\n\n${sections.join('\n')}`;
}
```

Таким образом реализуется важная функция: ИИ-ассистент может автоматически понимать доступные учебные ресурсы без необходимости вручную предоставлять контекст пользователем. Это тоже своего рода молчаливое понимание.

## Стандартизированная структура CodeRef Vault

Для vault типа coderef HagiCode предоставляет стандартизированную структуру каталогов:

```
my-coderef-vault/
├── index.yaml          # описание метаданных vault
├── AGENTS.md           # руководство по работе для ИИ-ассистента
├── docs/               # хранение учебных заметок и документации
└── repos/              # управление копируемыми репозиториями кода через Git подмодули
```

При создании vault система автоматически инициализирует эту структуру:

```csharp
private async Task EnsureCodeRefStructureAsync(
    string vaultName,
    string physicalPath,
    ICollection<VaultBootstrapDiagnosticDto> diagnostics,
    CancellationToken cancellationToken)
{
    Directory.CreateDirectory(physicalPath);

    var indexPath = Path.Combine(physicalPath, CodeRefIndexFileName);
    var docsPath = Path.Combine(physicalPath, CodeRefDocsDirectoryName);
    var reposPath = Path.Combine(physicalPath, CodeRefReposDirectoryName);

    // создание стандартной структуры каталогов
    if (!Directory.Exists(docsPath))
    {
        Directory.CreateDirectory(docsPath);
    }

    if (!Directory.Exists(reposPath))
    {
        Directory.CreateDirectory(reposPath);
    }

    // создание руководства AGENTS.md
    await EnsureCodeRefAgentsDocumentAsync(physicalPath, cancellationToken);

    // создание метаданных index.yaml
    await WriteCodeRefIndexDocumentAsync(indexPath, mergedDocument, cancellationToken);
}
```

Дизайн этой структуры тоже имеет свои особенности:

- Каталог **docs/** хранит ваши учебные заметки, можно записывать понимание кода, анализ архитектуры, опыт с ямами и т. д. в формате Markdown
- Каталог **repos/** управляет копируемыми репозиториями через Git подмодули, а не напрямую копирует код. Так можно поддерживать синхронизацию кода и сэкономить место
- **index.yaml** содержит метаданные vault, позволяя ИИ-ассистенту быстро понять назначение и содержимое этого vault
- **AGENTS.md** — это руководство, специально написанное для ИИ-ассистента, объясняющее, как обрабатывать содержимое в этом vault

Возможно, если так организовать, ИИ также сможет легче понять ваши мысли.

## Автоматическая инициализация системного управления

Помимо ручного создания vault, HagiCode также поддерживает автоматически управляемые системой vault:

```csharp
public async Task<IReadOnlyList<VaultRegistryEntry>> EnsureAllSystemManagedVaultsAsync(
    CancellationToken cancellationToken = default)
{
    var definitions = GetAllResolvedDefinitions();
    var entries = new List<VaultRegistryEntry>(definitions.Count);

    foreach (var definition in definitions)
    {
        entries.Add(await EnsureResolvedSystemManagedVaultAsync(definition, cancellationToken));
    }

    return entries;
}
```

Система автоматически создаст и управляет следующими vault:

- **hagiprojectdata**: хранение данных проекта для сохранения конфигурации и состояния проекта
- **personaldata**: хранение личных данных для сохранения настроек пользователя
- **hbsprompt**: библиотека шаблонов подсказок для управления часто используемыми ИИ-подсказками

Эти vault автоматически инициализируются при запуске системы, пользователю не нужно вручную настраивать. Ведь有些东西 следует оставить системе,何必 беспокоиться людям.

## Механизм контроля доступа

Важный дизайн — это контроль доступа. Система делит vault на два типа доступа:

```typescript
export interface VaultForText {
  id: string;
  name: string;
  type: string;
  physicalPath: string;
  accessType: 'read' | 'write';  // ключ: различие только для чтения и для редактирования
}
```

- **reference (только чтение)**: ИИ используется только для анализа и понимания, не может изменять содержимое. Подходит для справочных проектов с открытым исходным кодом, документации и т. д.
- **editable (для редактирования)**: ИИ может изменять содержимое в соответствии с потребностями задачи. Подходит для ваших заметок, черновиков и т. д.

Это различие очень важно. Оно позволяет ИИ знать, какое содержимое "справочное только для чтения", а какое "можно изменить вручную", избегая риска неправильных операций. Ведь никто не хочет, чтобы его плоды труда были случайно стерты.

## Практика: создание и использование Vault

Разобравшись с принципами, давайте посмотрим, как использовать на практике.

### Создание CodeRef Vault

Ниже приведён полный пример фронтенд-вызова:

```typescript
const createCodeRefVault = async () => {
  const response = await VaultService.postApiVaults({
    requestBody: {
      name: "React Learning Vault",
      type: "coderef",
      physicalPath: "/Users/developer/vaults/react-learning",
      gitUrl: "https://github.com/facebook/react.git"
    }
  });

  // система автоматически:
  // 1. клонирует репозиторий React в vault/repos/react
  // 2. создаёт каталог docs/ для заметок
  // 3. генерирует метаданные index.yaml
  // 4. создаёт файл руководства AGENTS.md

  return response;
};
```

Этот вызов API выполнит一系列 операций: создание структуры каталогов, инициализация Git подмодулей, генерация файлов метаданных и т. д. Вам нужно только предоставить основную информацию, остальное система обработает. На самом деле это тоже довольно облегчает.

### Использование Vault в ИИ-предложениях

После создания vault вы можете ссылаться на него в ИИ-предложении:

Система автоматически внедрит информацию о vault в контекст ИИ, позволяя ИИ знать, какие у вас есть доступные учебные ресурсы. ИИ может понять ваши мысли — это тоже своего рода редкое молчаливое понимание.

## Лучшие практики и меры предосторожности

В процессе использования системы Vault мы подвели некоторые итоги уроков.

### Безопасность путей

Система будет строго проверять пути, предотвращая атаки обхода пути:

```csharp
private static string ResolveFilePath(string vaultRoot, string relativePath)
{
    var rootPath = EnsureTrailingSeparator(Path.GetFullPath(vaultRoot));
    var combinedPath = Path.GetFullPath(Path.Combine(rootPath, relativePath));
    if (!combinedPath.StartsWith(rootPath, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode,
            "Vault file paths must stay inside the registered vault root.");
    }
    return combinedPath;
}
```

Это очень важно. Если вы настраиваете пользовательский путь vault, убедитесь, что путь находится в разрешённом диапазоне, иначе система откажет в операции. Безопасность — это то, что нельзя переоценить.

### Управление Git подмодулями

CodeRef vault рекомендует использовать Git подмодули вместо прямого копирования кода:

```csharp
private static string BuildCodeRefAgentsContent()
{
    return """
    # CodeRef Vault Guide

    Repositories under `repos/` should be maintained through Git submodules
    rather than copied directly into the vault root.

    Keep this structure stable so assistants and tools can understand the vault quickly.
    """ + Environment.NewLine;
}
```

Это даёт несколько преимуществ: поддержание синхронизации кода с вышестоящим репозиторием, экономия дискового пространства, удобство управления несколькими версиями кода. Ведь кто хочет раз за разом повторно скачивать одно и то же.

### Ограничения предварительного просмотра файлов

Чтобы предотвратить проблемы с производительностью, система ограничивает размер и тип файлов:

```csharp
private const int FileEnumerationLimit = 500;
private const int PreviewByteLimit = 256 * 1024;  // 256KB
```

Если ваш vault содержит большое количество файлов или сверхбольшие файлы, это может повлиять на производительность функции предварительного просмотра. В этом случае можно рассмотреть пакетную обработку или использование специализированных поисковых инструментов. Ведь有些东西 слишком большие, наоборот трудно обрабатывать.

### Диагностическая информация

При создании vault возвращается диагностическая информация, помогающая в отладке:

```csharp
List<VaultBootstrapDiagnosticDto> bootstrapDiagnostics = [];

if (IsCodeRefVaultType(normalizedType))
{
    bootstrapDiagnostics = await EnsureCodeRefBootstrapAsync(
        normalizedName,
        normalizedPhysicalPath,
        normalizedGitUrl,
        cancellationToken);
}
```

Если создание не удалось, вы можете просмотреть диагностическую информацию, чтобы узнать конкретную причину. При ошибке посмотрите диагностическую информацию — это тоже один из способов решения проблем.

## Резюме

Система Vault через единый уровень абстракции хранения решает основные проблемы копирования проектов в эпоху ИИ:

- **Централизованное управление знаниями**: все учебные ресурсы централизованы в одном месте, больше не разбросаны по разным местам
- **Автоматическое внедрение ИИ-контекста**: ИИ-ассистент может автоматически понимать доступные учебные ресурсы, без необходимости вручную предоставлять контекст
- **Повторное использование знаний между проектами**: несколько учебных проектов могут обмениваться и повторно использовать знания
- **Стандартизированная структура каталогов**: обеспечивает согласованную структуру каталогов, снижая затраты на обучение

Это решение прошло практическую проверку в проекте HagiCode. Если вы также занимаетесь инструментами, связанными с ИИ-сопровождаемой разработкой, или сталкиваетесь с подобными проблемами управления знаниями, я надеюсь, что этот опыт даст вам некоторую справку.

На самом деле ценность технического решения не в том, насколько оно сложное, а в том, может ли оно решить практические проблемы. Основная идея системы Vault очень проста — просто создать единый, понятный для ИИ уровень хранения знаний. Но именно эта простая абстракция значительно повысила нашу эффективность разработки.
