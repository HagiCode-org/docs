## Создание 12 исключительных стартовых экранов для HagiCode: от минимализма до киберпанка

> Краткая пауза между загрузкой приложения React 19 и завершением Hydration — это золотое окно для восприятия брендовой индивидуальности пользователями. В этой статье мы делимся полной системой стартовых стилей, построенной для проекта HagiCode на HTML/CSS/JS.

## Предпосылки

Как современное приложение, построенное на ASP.NET Core 10 и React 19 (Vite), HagiCode использует архитектуру с раздельным развёртыванием фронтенда и бэкенда. Результат сборки фронтенда упаковывается в директорию `wwwroot/` бэкенда и обслуживается ASP.NET Core.

Однако эта архитектура создаёт классическую проблему UX: когда пользователь открывает страницу, браузер сначала загружает HTML, затем скачивает большой JS-бандл, и наконец React выполняет Hydration. В этот «вакуумный период», длящийся от нескольких сотен миллисекунд до нескольких секунд, пользователь видит либо белый экран, либо безжизненную статическую страницу.

Чтобы заполнить этот пробел и привнести брендовую индивидуальность HagiCode, нам нужно было разработать систему стартовых стилей, полностью реализованную на инлайн-коде внутри `index.html`.

## О HagiCode

Подход к дизайну заставки, описанный в этой статье, основан на нашем практическом опыте в проекте [HagiCode](https://github.com/HagiCode-org/site). Как AI-ассистент программирования, HagiCode заботится не только об эффективности генерации кода, но и о визуальном впечатлении разработчика. Эта система стартовых экранов — один из результатов нашего стремления к максимальной производительности фронтенда.

## Ключевые вызовы и архитектурный дизайн

Прежде чем приступить к дизайну, мы должны были уточнить технические ограничения. Поскольку всё должно быть реализовано инлайн в `index.html`, мы не могли загружать внешние CSS или JS-файлы (кроме бандла React).

### Анализ технических ограничений

1.  **Принцип нулевых зависимостей**: все стили должны находиться внутри тега `<style>`, а логика — внутри тега `<script>`.
2.  **Оборонительный CSS**: чтобы глобальные стили после монтирования React не загрязнили заставку, мы решили обернуть все стартовые стили префиксом ID с высоким приоритетом, таким как `#boot-screen`.
3.  **Производительность прежде всего**: анимации по возможности использовать CSS `transform` и `opacity`, избегая reflow и не блокируя основной поток.
4.  **Визуальная согласованность**: цвета и шрифты должны соответствовать конфигурации Tailwind HagiCode.

### Архитектурный паттерн: Shell & Injector

Мы采用了 **вариантный паттерн**. Основная логика инкапсулирована в IIFE (немедленно вызываемой функции), а конкретная логика рендеринга внедряется через конфигурацию. Это позволяет переключать стили через простую конфигурацию, не переписывая логику манипуляции DOM.

Основной архитектурный код:

```html
<!-- Инлайн в index.html -->
<div id="boot-root"></div>

<script>
(function() {
  const BootSequence = {
    config: {
      theme: 'terminal', // Настраивается: 'minimal', 'skeleton', 'code-rain' и т.д.
      color: '#3b82f6'   // Брендовый цвет
    },

    // Основной жизненный цикл
    init() {
      this.render();
      this.listenForMount();
    },

    // Рендер текущего выбранного стиля
    render() {
      const root = document.getElementById('boot-root');
      if (this.variants[this.config.theme]) {
        root.innerHTML = this.variants[this.config.theme].render();
      }
    },

    // Прослушивание успешного монтирования React и элегантный выход
    listenForMount() {
      window.addEventListener('hagicode:ready', () => {
        const screen = document.getElementById('boot-root');
        // Сначала затухание, затем удаление DOM для избежания мерцания
        screen.style.opacity = '0';
        screen.style.transition = 'opacity 0.3s ease';
        setTimeout(() => screen.remove(), 300);
      });
    },

    // Логика реализации всех 12 стилей находится здесь
    variants: {
      // ...подробности ниже
    }
  };

  BootSequence.init();
})();
</script>
```

## Каталог из 12 стартовых стилей

Мы сгруппировали 12 стилей в шесть основных категорий для удовлетворения различных сценариев и эстетических предпочтений.

### A. Минимализм

> «Меньше — значит больше». Для сценариев, требующих максимальной скорости загрузки, мы предоставляем самые лёгкие варианты.

#### 1. Минималистичная точка
Простая точка в центре экрана с анимацией «дыхания».
*   **Реализация**: CSS `@keyframes` управляет scale и opacity.
*   **Подходит для**: любого случая, когда страница должна оставаться абсолютно чистой.

#### 2. Раскрытие бренда
С помощью SVG-анимации `stroke-dasharray` имитируется ручное прорисовывание линий логотипа HagiCode, затем плавно появляется текст.
*   **Техника**: SVG path-анимация с высокой полировкой ощущений.

### B. Имитация скелетон-экрана

> «Искусство обмана глаз». Имитируя реальную разметку UI, создаётся ощущение, что страница уже наполовину загружена.

#### 3. Скелетон боковой панели и чата
Возможно, самый практичный вариант. Мы вручную создали HTML-разметку, точно повторяющую компоненты React `Sidebar` и `ChatInput`, и наложили анимацию серых полос.
*   **Ценность**: когда React завершает гидратацию, скелетон мгновенно становится реальным компонентом, и пользователь почти не замечает переключения.

#### 4. Скелетон стопки карточек
Имитирует эффект стопки карточек предложений при загрузке с использованием 3D-трансформаций для лёгкого парения.

### C. Абстракция и искусство

> Демонстрация «гик»-ДНК HagiCode.

#### 5. Геометрическая морф
В центре экрана рендерится геометрическая фигура (квадрат), которая плавно трансформируется в круг, треугольник и, наконец, в логотип.
*   **Технология**: плавные переходы CSS `border-radius`.

#### 6. Кодовый дождь
Дань уважения «Матрице». С использованием шрифта JetBrains Mono на фоне падают лёгкие потоки символов.
*   **Примечание**: для производительности потоки символов должны быть ограничены небольшой областью или использовать пониженную частоту обновления.

#### 7. Неоновый пульс
Киберпанк-стиль светящегося кольца с использованием множественных слоёв `box-shadow` для создания мощного неонового свечения.

### D. Бренд и темы

> «Оживление» системы.

#### 8. Сезонная тема
Динамический загрузчик. Определяет текущую дату для праздников (например, Lunar New Year или Рождество) и загружает соответствующую SVG-анимацию.
*   **Пример**: во время Lunar New Year в нижней части экрана мягко покачиваются красные фонарики.

#### 9. Градиентный поток
Фон использует жидкий градиент на основе брендовых цветов HagiCode. В сочетании с анимацией `background-size` и `background-position` создаётся ощущение движения, подобное северному сиянию.

### E. Техническая эстетика

> Дань уважения разработчикам.

#### 10. Терминальная загрузка
Имитация вывода консоли. Строки кода быстро прокручиваются:
```text
> Initializing HagiCode Core...
> Loading models...
> Connecting to neural network...
```
Это мгновенно вызывает чувство знакомства у каждого разработчика.

#### 11. Прогресс-бар
Тонкая полоса прогресса в верхней части экрана с процентами справа. Хотя мы не можем получить реальный прогресс загрузки, можно использовать таймер для симуляции «убедительного» процесса: быстро до 80%, затем постепенно замедляясь до 100%.

### F. Творчество

#### 12. Пиксельная сборка
Очень интересная идея. Мелкие квадраты разбросаны по экрану, затем сходятся к центру и постепенно собираются в иконку логотипа HagiCode. Символизирует процесс сборки кода.

## Лучшие практики и подводные камни

В реальной разработке HagiCode мы выделили несколько критически важных деталей реализации.

### 1. Оборонительный CSS обязателен
Никогда не ленитесь пропускать префикс. Однажды мы забыли ограничить стили заставки ID, и глобальные стили `div` после монтирования React неожиданно повлияли на заставку, нарушив разметку.
**Урок**: поместите все CSS-селекторы под `#boot-screen` и при необходимости используйте `!important`, но только внутри CSS заставки.

### 2. Элегантные переходы
После успешного монтирования React не вызывайте `remove()` DOM заставки напрямую.
**Правильный подход**:
1.  React инициирует `window.dispatchEvent(new Event('hagicode:ready'))`.
2.  Заставка прослушивает событие и устанавливает `opacity: 0`.
3.  Ждём 300ms (время CSS-перехода), убедившись, что экран невидим, и только затем вызываем `.remove()`.

### 3. Синхронизация переменных темы
Значения цветов заставки хардкодятся в `index.html`. Если мы изменим основной цвет Tailwind, нужно обновить и заставку.
**Оптимизация**: напишите простой плагин в скрипте сборки Vite, который читает `tailwind.config.js` и внедряет цветовые переменные в шаблонные переменные `index.html`, создав единый источник истины.

### 4. Предзагрузка шрифтов
Заставка часто использует брендовый шрифт, но при медленной загрузке шрифта может появиться FOUT (вспышка нестилизованного текста).
**Решение**: добавьте `<link rel="preload" href="/fonts/JetBrainsMono.woff2" as="font" type="font/woff2" crossorigin>` внутри `<head>`. Это низкозатратный, высокоэффективный способ улучшить впечатление.

### 5. Мониторинг производительности
Мы добавили `performance.mark('boot-start')` в нижнюю часть `index.html` и отметили `boot-end` при успешном монтировании React.
**Значение**: через Application Insights мы можем напрямую измерить, насколько заставка сокращает воспринимаемое время ожидания. Данные показывают, что отличный скелетон-экран может повысить терпимость пользователей к «медленной сети» более чем на 50%.
