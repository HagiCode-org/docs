## Дизайн 12 исключительных опытов загрузки для HagiCode: от минимализма до киберпанка

> Короткий промежуток между загрузкой и гидратацией в приложениях React 19 — это золотое окно для восприятия бренда пользователями. В этой статье мы делимся нашей полной системой стилей запуска, основанной на HTML/CSS/JS из проекта HagiCode.


## Фон

HagiCode — это современное приложение, основанное на ASP.NET Core 10 и React 19 (Vite), которое использует архитектуру с раздельным развертыванием frontend и backend. Фронтенд-ресурсы упаковываются и помещаются в каталог `wwwroot/` backend, где они размещаются на ASP.NET Core.

Однако такая архитектура создаёт классическую проблему пользовательского опыта: когда пользователь посещает веб-страницу, браузер должен сначала загрузить HTML, затем загрузить огромный JS Bundle, и, наконец, React выполняет гидратацию. В этот "вакуумный период" от нескольких сотен миллисекунд до нескольких секунд пользователи сталкиваются с пустым экраном или статической страницей без жизни.

Чтобы заполнить этот промежуток и внедрить индивидуальность бренда HagiCode, нам нужно разработать систему стилей запуска, полностью основанную на встроенном коде `index.html`.

## О HagiCode

Схема дизайна экрана загрузки, описанная в этой статье, основана на нашем практическом опыте в проекте [HagiCode](https://github.com/HagiCode-org/site). Как помощник по AI коду, HagiCode не только фокусируется на эффективности генерации кода, но также уделяет большое внимание визуальному опыту разработчиков. Эта система запуска является результатом нашего стремления к исключительной производительности frontend.

## Основные проблемы и архитектурный дизайн

Прежде чем начать проектирование, мы должны сначала определить технические ограничения. Поскольку реализация встроена в `index.html`, это означает, что мы не можем загружать какие-либо внешние CSS или JS файлы (кроме самого Bundle React).

### Анализ технических ограничений

1.  **Принцип нулевых зависимостей**: Все стили должны быть написаны в теге `<style>`, логика — в теге `<script>`.
2.  **Защищённый CSS**: Чтобы предотвратить загрязнение глобальных стилей экраном загрузки после монтирования приложения React, мы решили использовать префикс ID с высоким приоритетом (например, `#boot-screen`) для обёртывания всех стилей запуска.
3.  **Производительность прежде всего**: Анимации должны использовать CSS `transform` и `opacity`, избегая перекомпоновки, чтобы не блокировать основной поток.
4.  **Визуальная согласованность**: Цвета и шрифты должны соответствовать конфигурации Tailwind в HagiCode.

### Архитектурный паттерн: Shell & Injector

Мы используем **вариантный паттерн**. Основная логика инкапсулирована в немедленно выполняющейся функции (IIFE), а конкретная логика рендеринга внедряется как элемент конфигурации. Это позволяет нам переключаться между разными стилями с помощью простой конфигурации, не повторяя логику операций DOM.

Ниже приведён основной архитектурный код:

```html
<!-- Встроено в index.html -->
<div id="boot-root"></div>

<script>
(function() {
  const BootSequence = {
    config: {
      theme: 'terminal', // Может быть настроено как 'minimal', 'skeleton', 'code-rain' и т. д.
      color: '#3b82f6'   // Цвет бренда
    },
    
    // Основной жизненный цикл
    init() {
      this.render();
      this.listenForMount();
    },

    // Рендеринг текущего выбранного стиля
    render() {
      const root = document.getElementById('boot-root');
      if (this.variants[this.config.theme]) {
        root.innerHTML = this.variants[this.config.theme].render();
      }
    },

    // Слушаем успешное монтирование React, элегантный выход
    listenForMount() {
      window.addEventListener('hagicode:ready', () => {
        const screen = document.getElementById('boot-root');
        // Сначала плавно скрываем, затем удаляем DOM, чтобы избежать мигания
        screen.style.opacity = '0';
        screen.style.transition = 'opacity 0.3s ease';
        setTimeout(() => screen.remove(), 300);
      });
    },

    // 12 стилей реализации логики сосредоточены здесь
    variants: {
      // ...конкретная реализация см. ниже
    }
  };

  BootSequence.init();
})();
</script>
```

## Список из 12 стилей запуска

Мы разделили эти 12 стилей на шесть категорий, чтобы удовлетворить различные сценарии и эстетические потребности.

### A. Минимализм

> "Меньше — значит больше". Для сценариев, требующих предельной скорости загрузки, мы предоставляем наиболее легковесное решение.

#### 1. Minimalist Dot (Минималистичное дыхание)
В центре экрана только простая точка с анимацией дыхания.
*   **Реализация**: CSS `@keyframes` контролирует scale и opacity.
*   **Применение**: Любой случай, когда нужно сохранить абсолютную чистоту страницы.

#### 2. Brand Reveal (Раскрытие бренда)
Через анимацию SVG `stroke-dasharray` имитируется ручное рисование линий логотипа HagiCode, затем плавно появляется текст.
*   **Техника**: Использование анимации SVG-путей для создания текстуры.

### B. Скелетные экраны

> "Искусство обмана глаз". Имитируя реальную компоновку UI, мы даём пользователям ощущение, что страница уже наполовину загружена.

#### 3. Sidebar Chat Skeleton (Скелетный экран боковой панели)
Это, возможно, самый практичный вариант. Мы вручную создали на HTML точно такую же компоновку, как компоненты React `Sidebar` и `ChatInput`, и покрыли их анимацией серых полос.
*   **Ценность**: Когда React завершает гидратацию, скелетный экран мгновенно превращается в реальные компоненты, пользователи почти не чувствуют переключения.

#### 4. Card Stack Skeleton (Стопка карточек)
Имитация эффекта стопки карточек при загрузке предложений, использование 3D-преобразований для лёгкого парения карточек.

### C. Абстракция и искусство

> Демонстрация гики-генов HagiCode.

#### 5. Geometric Morph (Геометрическая трансформация)
В центре экрана рендерится геометрическая фигура (квадрат), которая со временем плавно трансформируется в круг, треугольник и, наконец, в логотип.
*   **Технология**: Плавный переход CSS `border-radius`.

#### 6. Code Rain (Кодовый дождь)
Трибьют "Матрице". Использование шрифта JetBrains Mono, на фоне падают слабые потоки символов.
*   **Примечание**: Для производительности потоки символов должны быть ограничены небольшой областью или снижена частота обновления.

#### 7. Neon Pulse (Неоновый пульс)
Светящийся круг в стиле киберпанк, использование множественного наложения `box-shadow` для создания сильного свечения.

### D. Бренд и темы

> Оживить систему.

#### 8. Seasonal Theme (Сезонная тема)
Это динамический загрузчик. В зависимости от текущей даты определяется праздник (например, китайский Новый год, Рождество), загружается соответствующая SVG-анимация.
*   **Пример**: Во время китайского Нового года внизу экрана будут слегка покачиваться красные фонари.

#### 9. Gradient Flow (Градиентный поток)
Фон использует жидкий градиент цвета бренда HagiCode, сочетая анимацию `background-size` и `background-position`, создавая ощущение потока, подобного северному сиянию.

### E. Технический стиль

> Трибьют разработчикам.

#### 10. Terminal Boot (Терминальный запуск)
Имитация вывода консоли. Строки кода быстро прокручиваются:
```text
> Initializing HagiCode Core...
> Loading models...
> Connecting to neural network...
```
Это заставит каждого разработчика почувствовать себя как дома.

#### 11. Progress Bar (Минималистичная полоса прогресса)
Тонкая полоса прогресса вверху экрана, справа отображается процент. Хотя мы не можем получить реальный прогресс загрузки, можно использовать таймер для имитации "правдоподобного" процесса загрузки (первые 80% быстро, последние 20% медленно).

### F. Креатив

#### 12. Pixel Assembly (Пиксельная сборка)
Это интересная идея. На экране разбросаны квадраты, которые собираются в центре, постепенно складываясь в иконку логотипа HagiCode. Символизирует процесс построения кода.

## Лучшие практики и-summary ошибок

В реальной разработке HagiCode мы обобщили некоторые важные детали практики.

### 1. Защищённый CSS обязателен
Не ленитесь не писать префиксы. Однажды мы не ограничили стили экрана запуска ID, что привело к тому, что глобальные стили `div` после монтирования React случайно повлияли на экран запуска, вызвав разрушение компоновки.
**Опыт**: Все селекторы CSS должны быть под `#boot-screen`, и использовать `!important` для повышения приоритета (только в CSS экрана запуска).

### 2. Элегантный переход
После успешного монтирования React не удаляйте DOM экрана запуска напрямую через `remove()`.
**Правильный подход**:
1.  React запускает `window.dispatchEvent(new Event('hagicode:ready'))`。
2.  Экран запуска слушает событие, сначала устанавливает `opacity: 0`。
3.  Ожидает 300ms (время CSS transition), чтобы убедиться, что пользователь не видит, затем выполняет `.remove()`。

### 3. Синхронизация переменных темы
Коды цветов экрана запуска жёстко прописаны в `index.html`. Если мы изменим основной цвет Tailwind, нужно синхронно изменить здесь.
**Оптимизированное решение**: В скрипте сборки Vite написать простой плагин, который читает `tailwind.config.js` и внедряет переменные цветов в шаблонные переменные `index.html`, реализуя единый источник данных.

### 4. Предзагрузка шрифтов
Экран запуска обычно использует брендовый шрифт, но если шрифт загружается медленно, возникает FOUT (Flash of Unstyled Text).
**Решение**: Добавить в `<head>` `<link rel="preload" href="/fonts/JetBrainsMono.woff2" as="font" type="font/woff2" crossorigin>`。Это низкозатратное и высокодоходное средство для улучшения опыта.

### 5. Мониторинг производительности
Мы внедрили `performance.mark('boot-start')` внизу `index.html` и отметили `boot-end` при успешном монтировании React.
**Значение**: Сбор этих данных через Application Insights позволяет реально увидеть, насколько экран запуска сокращает воспринимаемое пользователем время ожидания (Perceived Loading Time)。Данные показывают, что отличный скелетный экран может повысить терпимость пользователей к "медленной сети" более чем на 50%。
