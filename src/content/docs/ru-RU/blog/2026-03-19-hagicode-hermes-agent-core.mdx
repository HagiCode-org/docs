## Почему HagiCode выбрала Hermes в качестве основы комплексного агента

> При создании платформы AI-ассистента для программирования выбор подходящей основы агента напрямую определяет потолок возможностей системы. Ведь некоторые вещи насильно не сделаешь — выберешь не тот фреймворк, и никакие усилия не помогут. В этой статье мы делимся размышлениями HagiCode о выборе технологий и практикой интеграции агента Hermes.

## Предпосылки

Самое сложное в создании AI-ассистента для программирования — это выбор базового фреймворка агента. На рынке есть множество решений, но — у каких-то функционал слишком прост, у каких-то部署 слишком сложная, у каких-то недостаточно гибкость. Нам нужно решение, которое может работать и на VPS за 5 долларов, и подключаться к GPU-кластерам. Требования не то чтобы слишком высокие, но многих они отпугивают.

Но на практике многие так называемые «универсальные агенты» работают только в облаке или требуют необоснованно высоких требований для локального развертывания. После двух недель исследования различных вариантов мы приняли смелое решение: полностью переписать основу агента и использовать Hermes как底层ный двигатель для комплексного агента.

Всё, что последовало за этим решением, возможно, было предопределено судьбой.

## О HagiCode

Решение, описанное в этой статье, основано на опыте проекта [HagiCode](https://hagicode.com). HagiCode — это платформа AI-ассистента для программирования, которая предоставляет разработчикам интеллектуального помощника через расширение VSCode, десктопный клиент и веб-сервис. Возможно, вы использовали похожие инструменты, но всегда чувствовали, что чего-то не хватает — мы это понимаем.

- GitHub: [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site)
- Официальный сайт: [hagicode.com](https://hagicode.com)

## Почему HagiCode нужен Hermes

Прежде чем подробно рассказывать о Hermes, давайте объясним, почему у HagiCode возникла такая потребность. В этом мире не всегда получается так, как хочется, нужно найти подходящий повод.

Как AI-ассистент для программирования, HagiCode должен поддерживать несколько сценариев использования:
- Локальная среда разработки: разработчики хотят запускать на своём компьютере, данные не покидают локальную среду — в наше время вопрос безопасности данных не то чтобы огромный, но и не маленький
- Среда командной работы: небольшие команды могут совместно использовать агент, развёрнутый на сервере — экономия, всем не легко
- Облачное эластичное масштабирование: при обработке сложных задач автоматически масштабироваться до GPU-кластера — на всякий случай

Такие требования «и то, и другое» заставили нас обратить внимание на Hermes. Правильный ли это выбор, я не знаю, но в то время других лучших вариантов не было.

## Что такое Hermes Agent

Hermes Agent — это автономный AI-агент, созданный **Nous Research**. Возможно, кто-то не знаком с Nous Research — это лаборатория, разработавшая такие открытые модели большого языка, как Hermes, Nomos и Psyché. Кстати, им тоже непросто — сделали так много всего хорошего, но мало кто знает.

В отличие от традиционных помощников по программированию в IDE или простых оболочек для API-чатов, Hermes имеет одну особенность: чем дольше он работает, тем он сильнее. Он не просто завершает задачу за один раз, а может непрерывно обучаться и накапливать опыт при длительной работе. Это похоже на людей, не правда ли?

### Основные особенности

Несколько основных особенностей Hermes точно соответствуют потребностям HagiCode. Какое совпадение, правда?

Это означает, что HagiCode может выбирать наиболее подходящий способ развёртывания в зависимости от сценария пользователя. Локальный запуск для личных пользователей, развёртывание на сервере для командных пользователей, GPU для сложных задач — один код для всех вариантов. В этом мире, чем можно сэкономить, тем лучше.

**Мультиплатформенный шлюз сообщений**
Hermes изначально поддерживает платформы Telegram, Discord, Slack, WhatsApp и другие. Для HagiCode это означает, что в будущем можно будет легко поддерживать AI-ассистентов на этих каналах. Ведь кто не хочет иметь больше путей?

**Богатая система инструментов**
Более 40 встроенных инструментов плюс возможность расширения через MCP (Model Context Protocol). Это критически важно для ассистента по программированию — выполнение shell-команд, работа с файловой системой, вызов Git — всё это требует поддержки инструментов. Агент без инструментов — это как птица без крыльев — хочешь лететь, но не можешь.

**Память между сессиями**
Hermes имеет систему постоянной памяти с полнотекстовым поиском FTS5 для извлечения истории диалогов. Это позволяет агенту помнить предыдущий контекст и не «забывать» каждый раз. Иногда я тоже хочу забыть, ни о чём не думать, но просто не получается.

## Как HagiCode интегрирует Hermes

Рассказав «почему», теперь посмотрим «как». 有些 вещи нужно не просто понять, а сделать — одних мыслей недостаточно.

### Абстракция уровня провайдера

В архитектуре HagiCode все AI-провайдеры реализуют единый интерфейс `IAIProvider`:

```csharp
public sealed class HermesCliProvider : IAIProvider, IVersionedAIProvider
{
    public ProviderCapabilities Capabilities { get; } = new ProviderCapabilities
    {
        SupportsStreaming = true,   // Поддержка потокового вывода
        SupportsTools = true,       // Поддержка вызова инструментов
        SupportsSystemMessages = true, // Поддержка системных подсказок
        SupportsArtifacts = false
    };
}
```

Этот уровень абстракции позволяет HagiCode бесшовно переключаться между различными AI-провайдерами — будь то OpenAI, Claude или Hermes, способ вызова верхнего уровня полностью одинаков. Короче говоря, это экономит время.

### Протокол связи ACP

Hermes использует **ACP (Agent Communication Protocol)** для связи. Это протокол, специально разработанный для связи агентов, основные методы включают:

| Метод | Описание |
|------|------|
| `initialize` | Инициализация соединения, получение версии протокола и возможностей клиента |
| `authenticate` | Обработка аутентификации, поддержка различных методов аутентификации |
| `session/new` | Создание новой сессии, настройка рабочего каталога и MCP-серверов |
| `session/prompt` | Отправка подсказки и получение ответа |

HagiCode реализует уровень транспортировки ACP через `StdioAcpTransport`, запускает дочерний процесс Hermes и общается через стандартный ввод/вывод. Это звучит сложно, но на деле не так уж и сложно — главное, набраться терпения.

### Управление конфигурацией

```csharp
public sealed class HermesPlatformConfiguration : IAcpPlatformConfiguration
```

### Распределённая архитектура Orleans

HagiCode использует Orleans для построения распределённой системы, интеграция Hermes реализована через следующие компоненты:

- **HermesGrain**: реализация Orleans Grain, обработка выполнения сессий
- **HermesPlatformConfiguration**: специфичная для платформы конфигурация
- **HermesAcpSessionAdapter**: адаптер сессии ACP
- **HermesConsole**: специальная консоль для проверки

Название Orleans звучит приятно, легендарный Аладдин — хотя этот Орлеан не тот Орлеан, но хорошее название всегда добавляет очки.

### Полный процесс выполнения

Ниже приведена основная логика выполнения Hermes Provider:

```csharp
private async IAsyncEnumerable<AIStreamingChunk> StreamCoreAsync(
    AIRequest request,
    string? embeddedCommandPrompt,
    [EnumeratorCancellation] CancellationToken cancellationToken)
{
    // 1. Создание уровня транспортировки, запуск дочернего процесса Hermes
    await using var transport = new StdioAcpTransport(
        platformConfiguration.GetExecutablePath(),
        platformConfiguration.GetArguments(),
        platformConfiguration.GetEnvironmentVariables(),
        platformConfiguration.GetStartupTimeout(),
        _loggerFactory.CreateLogger<StdioAcpTransport>());
    await transport.ConnectAsync(cancellationToken);

    // 2. Инициализация, получение версии протокола и методов аутентификации
    var initializeResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "initialize",
        BuildInitializeParameters(platformConfiguration), cancellationToken);

    // 3. Обработка аутентификации
    var authMethods = ParseAuthMethods(initializeResult);
    if (!isAuthenticated)
    {
        var methodId = platformConfiguration.Authentication.ResolveMethodId(authMethods);
        await SendHermesRequestAsync(transport, nextRequestId++, "authenticate", ...);
    }

    // 4. Создание сессии
    var newSessionResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "session/new",
        BuildNewSessionParameters(platformConfiguration, workingDirectory, model), cancellationToken);
    var sessionId = ParseSessionId(newSessionResult);

    // 5. Выполнение подсказки и сбор потоковых ответов
    await foreach (var payload in transport.ReceiveMessagesAsync(cancellationToken))
    {
        // Обработка уведомлений session/update, преобразование в потоковые блоки
        if (TryParseSessionNotification(root, out var notification))
        {
            if (_responseMapper.TryConvertToStreamingChunk(notification, out var chunk))
            {
                yield return chunk;
            }
        }
    }
}
```

### Проверка работоспособности

Для обеспечения доступности сервиса Hermes HagiCode реализовала механизм проверки работоспособности:

```csharp
public async Task<ProviderTestResult> PingAsync(CancellationToken cancellationToken = default)
{
    var response = await ExecuteAsync(
        new AIRequest
        {
            Prompt = "Reply with exactly PONG.",
            CessionId = null,
            AllowedTools = Array.Empty<string>(),
            WorkingDirectory = ResolveWorkingDirectory(null)
        },
        cancellationToken);

    var success = string.Equals(response.Content.Trim(), "PONG", StringComparison.OrdinalIgnoreCase);
    return new ProviderTestResult
    {
        ProviderName = Name,
        Success = success,
        ResponseTimeMs = stopwatch.ElapsedMilliseconds,
        ErrorMessage = success ? null : $"Unexpected Hermes ping response: '{response.Content}'."
    };
}
```

Это так называемая «проверка работоспособности». На самом деле люди тоже так — иногда нужно проверить себя, просто обычно никто не говорит нам, что именно проверять.

## Важные моменты на практике

В процессе интеграции Hermes есть несколько подводных камней, о которых стоит знать заранее. В наше время кто не наступал на пару граблей?

### Конфигурация методов аутентификации

Hermes поддерживает несколько методов аутентификации (API Key, Token и т. д.), нужно выбирать в зависимости от реального развёртывания. Ошибочная конфигурация приведёт к неудачному подключению, но сообщение об ошибке может быть недостаточно понятным. Иногда сообщение об ошибке очень далеко от реальной причины, нужно медленно разбираться.

### Конфигурация MCP-серверов

При создании сессии можно настроить список MCP-серверов, чтобы Hermes мог вызывать внешние инструменты. Но нужно注意:
- Адрес MCP-сервера должен быть доступен
- Время ожидания должно быть разумно настроено
- Обработка деградации при недоступности сервера

В этом мире от всего не застрахуешься.

### Управление рабочим каталогом

Каждая сессия требует указания рабочего каталога, чтобы Hermes мог правильно обращаться к файлам проекта. Для сценариев с несколькими проектами нужно динамически переключать рабочий каталог. Звучит просто, но на деле нужно учитывать много ситуаций.

### Обработка агрегации ответов

Ответы Hermes могут быть разбросаны между уведомлениями `session/update` и окончательным результатом, нужно правильно объединять обработку, иначе произойдёт потеря содержания. Я тоже на этом потерпел, но постепенно стало лучше.

### Стратегия обработки ошибок

Ошибки во время выполнения должны явно возвращаться, а не молча откатываться к другим провайдерам. Так пользователь узнает, что проблема в Hermes, а не загадочным образом сменился на другую модель. Ведь обман — это не тот путь.

## Заключение

HagiCode выбрала Hermes в качестве основы комплексного агента не спонтанно, а как обдуманный выбор на основе реальных потребностей и технических особенностей. Правильный ли этот выбор, сейчас говорить рано, но пока работает неплохо.
