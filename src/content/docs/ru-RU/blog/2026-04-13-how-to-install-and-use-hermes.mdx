Если вы хотите установить и начать использовать Hermes, кратчайший путь состоит всего из трёх шагов:

1. Запустите официальную команду установки
2. Запустите CLI с помощью `hermes` в терминале
3. Если вы хотите продолжить использовать в Feishu, настройте `hermes gateway setup`

Эта статья не ставит целью охватить все возможности Hermes за один раз, а помогает вам пройти самый важный вводный цикл: установить, запустить, начать использовать, а затем подключить самый распространённый сценарий messaging-платформы.

## Что такое Hermes и для кого он предназначен

Hermes Agent — это AI agent, который можно использовать как в локальном терминале, так и через шлюз messaging-платформы.

Для большинства разработчиков есть два самых распространённых способа входа:

- **CLI**: введите `hermes` в терминале, чтобы перейти прямо в интерактивный интерфейс
- **Messaging Gateway**: запустите `hermes gateway`, а затем общайтесь с ним через Feishu, Telegram, Discord, Slack и другие платформы

Если ваша текущая цель — просто быстро начать, рекомендую не менять порядок действий, а сначала пройти этот путь:

- Сначала установите Hermes
- Сначала проверьте работоспособность через CLI
- Затем решайте, нужно ли подключать messaging-платформу

Так проще локализовать проблемы, и это больше подходит для пользователей, которые впервые сталкиваются с Hermes.

## Что нужно знать перед установкой Hermes

Согласно README Hermes, официальный быстрый путь установки поддерживает следующие среды:

- Linux
- macOS
- WSL2
- Android через Termux

### Примечание для пользователей Windows

Hermes в настоящее время не поддерживает непосредственный запуск в нативном Windows. Если вы используете Windows, рекомендуется сначала установить WSL2, а затем выполнить команду установки в WSL2.

Об этом лучше сказать в самом начале статьи, так как многие неудачные установки на самом деле не связаны с проблемами команд, а с несоответствием требований к среде выполнения.

## Как установить Hermes

Команда быстрой установки, указанная в README Hermes:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Эта команда выполнит официальный сценарий установки и обработает инициализацию, специфичную для платформы.

### Обновление shell после установки

После завершения установки сначала перезагрузите среду shell. Чаще всего используется:

```bash
source ~/.bashrc
```

Если вы используете zsh, можно также использовать:

```bash
source ~/.zshrc
```

### Как проверить, что Hermes установлен

Самый прямой способ проверки — запустить:

```bash
hermes
```

Если вы хотите дополнительно подтвердить, что конфигурация и зависимости в норме, можно также выполнить:

```bash
hermes doctor
```

`hermes doctor` подходит для использования в следующих случаях:

- После установки поведение команды ненормальное
- Сбой конфигурации модели
- Сбой запуска gateway
- Не уверены, полны ли зависимости среды

## Как начать использовать Hermes в первый раз

### Начните с CLI

Если вы просто хотите как можно скорее подтвердить, работает ли Hermes, самый простой способ:

```bash
hermes
```

Это запустит интерактивный CLI Hermes. Для пользователей, которые впервые сталкиваются с Hermes, это также самая рекомендуемая отправная точка, так как вы можете сначала проверить самые важные вещи:

- Работает ли команда действительно
- Нормальна ли текущая конфигурация модели
- Работает ли цепочка инструментов терминала нормально
- Подходит ли вам такой интерактивный опыт

### Эти команды достаточно для выполнения первого раунда конфигурации

Несколько часто используемых команд, перечисленных в README Hermes, в основном составляют путь первого знакомства:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Если вы не знаете, что они делают, можете сначала запомнить так:

- `hermes model`: выбор модели, переключение модели
- `hermes tools`: просмотр и настройка доступных инструментов
- `hermes config set`: изменение конкретных параметров конфигурации
- `hermes setup`: запуск полного мастера инициализации
- `hermes update`: обновление Hermes
- `hermes doctor`: устранение неполадок

Самый практичный порядок для новичков обычно:

1. Сначала запустите `hermes model`
2. Если вы хотите настроить все часто используемые элементы за один раз, затем запустите `hermes setup`

## Два самых распространённых способа использования Hermes

### 1. Использовать Hermes в терминале как повседневного помощника по разработке

Режим CLI подходит для следующих сценариев:

- Непосредственные вопросы при написании кода локально
- Просмотр проектов, изменение файлов, запуск команд
- Однократная отладка или review
- Непрерывное сотрудничество в текущем рабочем каталоге

Его самое большое преимущество — кратчайший путь: не нужно дополнительно подключать платформу, не нужно сначала обрабатывать конфигурацию бота, и это больше всего подходит для формирования привычки первого использования.

### 2. Использование Hermes через messaging-платформу

Если вы хотите общаться с Hermes на таких платформах, как Feishu, Telegram, Discord, вам нужно использовать messaging gateway.

Самые распространённые команды входа:

```bash
hermes gateway setup
hermes gateway
```

Где:

- `hermes gateway setup` используется для интерактивной конфигурации платформы
- `hermes gateway` используется для запуска процесса gateway

Согласно официальной документации, gateway — это единый фоновый процесс, используемый для подключения настроенных платформ, управления сеансами и обработки запланированных задач.

## Как подключить Hermes к messaging-платформе на примере Feishu

Если ваша повседневная работа в основном проходит в Feishu, то Feishu/Lark будет естественным способом подключения Hermes.

### Минимальный жизнеспособный способ подключения

Официальная документация рекомендует следующий вход для Feishu/Lark:

```bash
hermes gateway setup
```

После запуска выберите **Feishu / Lark** в мастере.

Feishu документация предоставляет два режима подключения:

- `websocket`: рекомендуется
- `webhook`: опционально

Если Hermes работает на вашем ноутбуке, рабочей станции или частном сервере, приоритетное использование `websocket` будет проще, так как не требуется дополнительное открытие публичного адреса обратного вызова.

### Если вы настраиваете вручную,至少 нужно знать эти переменные

Если вы настраиваете не через мастер, а вручную пишете конфигурацию, ключевые переменные, перечисленные в документации Feishu, включают:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Из них наиболее заслуживают внимания два:

- `FEISHU_ALLOWED_USERS`: рекомендуется настроить, чтобы любой, кто имеет доступ к боту, не мог напрямую его использовать
- `FEISHU_HOME_CHANNEL`: можно заранее указать home chat для получения результатов cron или уведомлений по умолчанию

### Почему в групповых чатах Feishu иногда нет ответа

Этот момент легко упустить: в групповых чатах Feishu Hermes по умолчанию не отвечает на каждое сообщение.

Официальная документация чётко указывает:

- В личных чатах Hermes отвечает на сообщения
- В групповых чатах необходимо явно `@` bot, чтобы он обрабатывал сообщения

Если вы хотите установить определённый чат Feishu как home channel, можно также использовать в чате:

```text
/set-home
```

Или заранее указать в конфигурации:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Команды Hermes, которые новички должны запомнить в первую очередь

Независимо от того, используете ли вы CLI или messaging-платформу, запоминание следующих команд уже будет достаточно:

- `/new` или `/reset`: начать новый сеанс
- `/model`: просмотр или переключение модели
- `/retry`: повторить предыдущий раунд
- `/undo`: отменить предыдущее взаимодействие
- `/compress`: вручную сжать контекст
- `/help`: просмотр справки

Если вы в основном используете messaging-платформу, дополнительно запомните ещё одну:

- `/sethome` или `/set-home`: установить текущий чат как home channel

Эти команды охватывают самые распространённые операции на этапе новичка: перезапуск, настройка, откат, просмотр и продолжение использования.

## Часто задаваемые вопросы

### Можно ли напрямую установить Hermes на Windows?

Нет. Текущая официальная документация чётко указывает, что нативный Windows не поддерживается, рекомендуется использовать WSL2.

### Что делать, если после установки ввод `hermes` не даёт реакции?

Рекомендуется проверить в следующем порядке:

1. Сначала перезагрузите shell, например `source ~/.bashrc`
2. Затем перезапустите `hermes`
3. Если всё ещё есть проблемы, выполните `hermes doctor`

### Почему bot не отвечает в группах Feishu?

Сначала проверьте эти три элемента:

- Вы `@` Hermes в группе
- Ограничивает ли `FEISHU_ALLOWED_USERS` текущего пользователя
- Разрешает ли текущая политика группового чата обработку групповых сообщений

Согласно официальной документации Feishu, в сценариях группового чата явное @mention является обязательным условием.
