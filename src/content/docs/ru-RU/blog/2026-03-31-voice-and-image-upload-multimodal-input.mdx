## Лучше говорить, чем печатать, лучше показать скриншотом, чем описывать — практика мультимодального ввода в AI-помощнике по коду

> На самом деле, в написании кода есть предел, как быстро бы вы ни печатали. Иногда одно предложение можно сказать за секунды, но придётся долго стучать по клавиатуре; иногда один скриншот объяснит всё, но нужно куча текста для описания. В этой статье поговорим о том, с чем мы столкнулись при создании [HagiCode](https://hagicode.com) — распознавание речи也好, загрузка изображений也罢, в общем, хотим сделать AI-помощника по коду более удобным, вот и всё.

## Предыстория

При работе над HagiCode мы обнаружили проблему — или, скорее сказать, проблема проявила себя, когда пользователи начали активно пользоваться сервисом: только печатать, иногда утомительно.

Подумайте сами: взаимодействие пользователя с Agent — это же ключевая ситуация. Но каждый раз нужно сидеть за клавиатурой и стучать, ну, как бы это сказать, эффективность действительно невысока:

1. **Печатать слишком медленно**: некоторые сложные проблемы, какие-то ошибки, что-то на интерфейсе — рассказывать об этом печатанием придётся полминуты, а голосом можно сказать за десять секунд. Эта разница во времени, довольно неприятно.

2. **Изображения более наглядны**: иногда на интерфейсе возникает ошибка, или нужно сравнить с дизайном, или показать структуру кода... фраза "один рисунок стоит тысячи слов" хоть и старая, но верная. Позволить AI напрямую "увидеть" проблему гораздо яснее, чем описывать долго.

3. **Взаимодействие должно быть естественным**: современные AI-помощники должны поддерживать текст, голос, изображения, верно? Пользователь должен использовать то, что хочет, это и называется естественным, не так ли?

Так вот мы подумали, а не добавить ли в HagiCode функции распознавания речи и загрузки изображений, чтобы сделать работу с Agent более удобной. Ведь позволить пользователям печатать меньше — это уже хорошо.

## О HagiCode

Решения, описанные в этой статье, основаны на нашем опыте в проекте [HagiCode](https://hagicode.com) — или, можно сказать, на опыте, добытом в постоянных пробах и ошибках.

HagiCode — это проект AI-помощника по коду с открытым исходным кодом, идея проста: использовать технологии AI для повышения эффективности разработки. В процессе работы мы обнаружили, что потребность в мультимодальном вводе довольно сильна — иногда сказать одно предложение быстрее, чем печатать кучу текста, иногда один скриншот яснее, чем долгие описания.

Эти потребности толкали нас вперёд, и в итоге появились функции распознавания речи и загрузки изображений. Пользователи могут взаимодействовать с AI самым естественным способом, это чувство, ну, хорошее.

## Анализ

### Технические проблемы распознавания речи

При создании функции распознавания речи мы столкнулись с довольно сложной проблемой: **WebSocket API браузера не поддерживает пользовательские HTTP заголовки**.

А выбранный нами сервис распознавания речи — это API распознавания речи Doubao от ByteDance. Этот API как раз требует передачи информации об аутентификации через HTTP заголовки, какие-то `accessToken`, `secretKey` и тому подобное. Вот отлично, техническое противоречие:

```javascript
// WebSocket API браузера не поддерживает следующий способ
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // не поддерживается
  }
});
```

Перед нами примерно два варианта:

1. **Параметры URL**: поместить информацию об аутентификации в URL
   - Преимущество: простота реализации
   - Недостаток: учётные данные exposed на фронтенде, плохая безопасность; кроме того, некоторые API принудительно требуют проверки заголовков

2. **Backend-прокси**: реализовать WebSocket-прокси на бэкенде
   - Преимущество: учётные данные безопасно хранятся на бэкенде; полностью соответствует требованиям API
   - Недостаток: реализация немного сложнее

В итоге мы выбрали **backend-прокси**. Ведь безопасность — это то, с чем нельзя компрометировать — в этом никто не сможет обмануть.

### Функциональные требования загрузки изображений

Что касается функции загрузки изображений, наши требования тоже довольно просты:

1. **Различные способы загрузки**: клик для выбора файла, перетаскивание, вставка из буфера обмена — это же база?
2. **Проверка файлов**: ограничение типа (PNG, JPG, WebP, GIF), ограничение размера (5-10 МБ) — это базовые операции
3. **Пользовательский опыт**: прогресс загрузки, предпросмотр, сообщения об ошибках — нужно же дать знать, что происходит
4. **Безопасность**: проверка на сервере, предотвращение загрузки вредоносных файлов — это же важно

## Решения

### Распознавание речи: архитектура WebSocket-прокси

Мы разработали трёхслойную архитектуру распознавания речи, ну,可以说, нашли путь:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**Реализация основных компонентов**:

1. **AudioWorklet-процессор на фронтенде**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Ресемплинг до 16kHz (требование API Doubao)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Накопление выборок в блоки по 500мс
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Преобразование в 16-bit PCM и отправка
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **WebSocket-обработчик на бэкенде** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **Компонент VoiceTextArea на фронтенде**:

```tsx
export const VoiceTextArea = forwardRef<HTMLTextAreaElement, VoiceTextAreaProps>(
  ({ value, onChange, onTextRecognized, maxDuration }, ref) => {
    const { isRecording, interimText, volume, duration, startRecording, stopRecording } =
      useVoiceRecording({ onTextRecognized, maxDuration });

    return (
      <div className="flex gap-2">
        {/* Кнопка голоса */}
        <button onClick={handleButtonClick}>
          {isRecording ? <VolumeWaveform volume={volume} /> : <Mic />}
        </button>
        {/* Поле ввода текста */}
        <textarea value={displayValue} onChange={handleChange} />
      </div>
    );
  }
);
```

### Загрузка изображений: компонент множественной загрузки

Мы создали полнофункциональный компонент загрузки изображений, поддерживающий все три способа, ну,可以说, охватили все часто используемые пользователями сценарии.

**Основные возможности**:

1. **Три способа загрузки**:

```tsx
// Клик для загрузки
const handleClick = () => fileInputRef.current?.click();

// Перетаскивание
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Вставка из буфера обмена
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **Проверка на фронтенде**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **Обработка загрузки на бэкенде** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Проверка
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Сохранение файла
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## Практическое руководство

### Как использовать распознавание речи

1. **Настройка сервиса распознавания речи**:
   - Перейдите на страницу настроек распознавания речи
   - Настройте `AppId` и `AccessToken` Doubao
   - (Опционально) настройте горячие слова для повышения точности распознавания профессиональной терминологии

2. **Использование в поле ввода**:
   - Нажмите на иконку микрофона слева от поля ввода
   - Начните говорить, когда увидите анимацию волны
   - Снова нажмите на иконку для остановки записи
   - Результаты распознавания автоматически вставятся в позицию курсора

3. **Пример настройки горячих слов**:

```text
TypeScript
React
useState
useEffect
```

### Как использовать загрузку изображений

1. **Способы загрузки**:
   - Нажмите кнопку загрузки для выбора файла
   - Перетащите изображение непосредственно в область загрузки
   - Используйте `Ctrl+V` для вставки скриншота из буфера обмена

2. **Поддерживаемые форматы**: PNG, JPG, JPEG, WebP, GIF
3. **Ограничение размера**: по умолчанию 5 МБ (настраивается)

### Меры предосторожности

1. **Распознавание речи**:
   - Требуется разрешение на использование микрофона
   - Рекомендуется использовать в тихой среде
   - Максимальная продолжительность записи — 300 секунд (настраивается)

2. **Загрузка изображений**:
   - Поддерживаются только распространённые форматы изображений
   - Обратите внимание на ограничение размера файла
   - После загрузки автоматически генерируется URL предпросмотра

3. **Соображения безопасности**:
   - Учетные данные распознавания речи хранятся на бэкенде
   - Загрузка изображений имеет строгую проверку на сервере
   - В производственной среде рекомендуется использовать HTTPS/WSS

## Заключение

После добавления распознавания речи и загрузки изображений пользовательский опыт HagiCode значительно улучшился. Теперь пользователи могут взаимодействовать с AI более естественным способом — говорить вместо печати, показывать скриншоты вместо описания. Это чувство, ну,可以说, как будто наконец нашли более удобный способ общения.

При создании этой функции мы столкнулись с проблемой отсутствия поддержки пользовательских заголовков в WebSocket браузера, и в итоге решили её через backend-прокси. Это решение не только гарантирует безопасность, но и создаёт основу для последующей интеграции других WebSocket-сервисов, требующих аутентификации — можно сказать, неожиданный бонус.

Компонент загрузки изображений тоже поддерживает различные способы загрузки, позволяя пользователям выбирать наиболее удобный в зависимости от ситуации. Клик也好, перетаскивание也罢, или прямая вставка — всё позволяет быстро завершить загрузку. "Все дороги ведут в Рим", только некоторые дороги легче, некоторые немного извилистее.

"Лучше говорить, чем печатать, лучше показать скриншотом, чем описывать", эта фраза здесь как раз кстати. Если вы тоже создаёте похожий AI-помощник, надеюсь, этот опыт будет вам полезен, пусть даже ненамного.
