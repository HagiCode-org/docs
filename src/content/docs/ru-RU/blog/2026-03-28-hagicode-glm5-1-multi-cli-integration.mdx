## Руководство по интеграции Hagicode с GLM-5.1 и множественными CLI

:::note[Примечание о временном контексте]
В этой статье фиксируется дизайн интеграции множественных CLI и сопоставление моделей на момент публикации. Текущий постоянно поддерживаемый диапазон указан в [Установка AI Agent CLI](/related-software-installation/ai-agent-cli/) и [Руководство по установке](/installation); `IFlowCli` в этой статье представляет только историческую конфигурацию того времени и не относится к текущему активному поддерживаемому диапазону.
:::

## Предпосылки

В проекте Hagicode пользователи могут выбрать различные инструменты CLI для управления помощником AI-программирования, включая Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI и другие. Эти инструменты CLI сами по себе являются универсальными инструментами AI-программирования, но через слой абстракции Hagicode могут гибко подключаться к различным поставщикам AI-моделей.

Zhipu AI (ZAI) предоставляет интерфейс, совместимый с Anthropic Claude API, что позволяет этим инструментам CLI напрямую использовать отечественные серии моделей GLM. Среди них GLM-5.1 — это последняя выпущенная Zhipu большая языковая модель, которая значительно улучшена по сравнению с GLM-5.0.

## CLI-абстрактная архитектура Hagicode

Hagicode через перечисление `AIProviderType` определяет 11 типов поставщиков CLI, покрывая основные инструменты AI-программирования CLI:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Каждый CLI имеет соответствующую конфигурацию параметров модели, поддерживающую параметры `model` и `reasoning`:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],
        [AIProviderType.CodexCli] = ["model", "reasoning"],
        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],
        [AIProviderType.HermesCli] = ["model", "reasoning"],
        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini не поддерживает параметр reasoning
        // ...
    };
```

## Система поддержки моделей GLM

В Secondary Professions Catalog Hagicode определена полная поддержка серии моделей GLM:

| Model ID | Name | Default Reasoning | Compatible CLI Families |
|----------|------|-------------------|-------------------------|
| `glm-4.7` | GLM 4.7 | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM 5 Turbo | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (Legacy) | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | high | claude, codebuddy, hermes, qoder, kiro |

## Ключевые различия между GLM-5.1 и GLM-5.0

GLM-5.1 — это независимый идентификатор новой модели без какой-либо логики обработки legacy:

## Использование GLM-5.1 в разных CLI

### 1. Claude Code CLI + GLM-5.1

Claude Code CLI — один из основных CLI Hagicode, настраиваемый через систему конфигурации Hero:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Соответствующая конфигурация `HeroEquipmentCatalogItem`:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI — наиболее гибкий CLI, поддерживающий формат `provider/model` для указания любой модели:

**Способ 1: Использование префикса ZAI provider**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Способ 2: Прямое использование ID модели**
```json
{
  "model": "glm-5.1"
}
```

**Способ 3: Интерфейс фронтенд-конфигурации**

В `HeroModelEquipmentForm.tsx` у OpenCode CLI есть специальное приглашение-заполнитель:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Пользователь может ввести:
```
zai/glm-5.1
glm-5.1
```

**Логика парсинга модели OpenCode CLI**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // Без косой черты: прямое использование ID модели
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // С косой чертой: разбор формата provider/model
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

У Codebuddy CLI есть специальная логика обработки legacy:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Примечание**: Codebuddy имеет специальное резервирование для GLM-5.0, не использует legacy normalization:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// Для CodebuddyCli, glm-5.0 не будет нормализован в glm-5-turbo
```

## Преимущества улучшений GLM-5.1

По сравнению с GLM-5.0, GLM-5.1 имеет следующие значительные улучшения:

### 1. Лучшие возможности рассуждения

Согласно официальной информации о выпуске от Zhipu, улучшения GLM-5.1 включают:

- **Более сильные возможности понимания кода**: более точный анализ сложных структур кода
- **Более длинное понимание контекста**: поддержка более длинного контекста диалога
- **Усиленные вызовы инструментов**: повышение успешности вызова инструментов MCP
- **Стабильность вывода**: уменьшение случайности и галлюцинаций

### 2. Полная совместимость с множественными CLI

GLM-5.1 покрывает все основные CLI, поддерживаемые Hagicode:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Примечания

### 1. Конфигурация API Key

Убедитесь в правильной установке переменной среды `ANTHROPIC_AUTH_TOKEN` — это необходимый учетный данные для всех CLI при подключении к моделям.

### 2. Доступность моделей

GLM-5.1 необходимо открыть у соответствующего поставщика моделей:
- Платформа Zhipu AI ZAI поддерживает по умолчанию
- Alibaba Cloud DashScope может потребовать отдельной заявки

### 3. Формат OpenCode CLI

При использовании формата `provider/model` убедитесь в правильности ID поставщика:
- Zhipu AI: `zai` или `zhipuai`
- Alibaba Cloud: `aliyun` или `dashscope`

### 4. Параметр Reasoning

- Рекомендуется использовать уровень `high` для получения наилучшего эффекта генерации кода
- Gemini CLI не поддерживает параметр reasoning и автоматически игнорирует эту конфигурацию

## Заключение

Hagicode через единый слой абстракции реализовал гибкую интеграцию GLM-5.1 с множественными CLI. Разработчики могут выбрать подходящий инструмент CLI в соответствии со своими предпочтениями и сценариями использования, а через простую конфигурацию использовать последнюю модель GLM-5.1.

GLM-5.1 как последняя версия модели от Zhipu имеет очевидные улучшения по сравнению с GLM-5.0:
- Независимый идентификатор версии, без нагрузки legacy
- Более сильные возможности рассуждения и понимания кода
- Широкая совместимость с множественными CLI
- Гибкая конфигурация уровней рассуждения

Через правильную конфигурацию переменных среды и снаряжения Hero игроки могут в полной мере использовать мощные возможности GLM-5.1 в различных средах CLI.
