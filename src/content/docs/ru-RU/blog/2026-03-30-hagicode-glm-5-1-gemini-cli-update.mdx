## Предыстория

Время летит быстро, развитие больших языковых моделей растёт, как весенний бамбук — быстро вверх. Когда-то мы ещё радовались «AI, который может писать код», а теперь уже эпоха многомодельной координации и мультиинструментальной интеграции. Это интересно? Возможно, ведь разработчикам нужны не просто инструменты, а способность адаптироваться к разным сценариям и гибко переключаться.

HagiCode как платформа AI-поддержки программирования недавно пережила два важных события: полное подключение модели GLM-5.1 от Zhipu AI и официальное включение Gemini CLI в качестве десятого поддерживаемого Agent CLI. Эти события не очень большие и не очень маленькие, но для совершенствования платформы это определённо хорошая вещь.

GLM-5.1 — это новейшая флагманская модель от Zhipu AI. По сравнению с GLM-5.0, её способность к рассуждению стала сильнее, понимание кода глубже, а вызов инструментов плавнее. Важнее всего то, что это первая модель GLM, поддерживающая ввод изображений — что это значит? Это значит, что пользователи могут напрямую делать скриншоты, чтобы AI видел проблему, больше не нужно трудиться над её описанием. Удобство этого функционала поймёшь, как только попробуешь.

В то же время, HagiCode через архитектуру HagiCode.Libs.Providers успешно интегрировала Gemini CLI. Это уже десятый Agent CLI, честно говоря, достижение этого этапа вызывает некоторое чувство удовлетворения.

Стоит упомянуть, что функция загрузки изображений в HagiCode позволяет пользователям напрямую общаться с AI через скриншоты. Даже при запуске версии GLM 4.7 платформа по-прежнему хорошо работает и уже помогла проекту завершить множество важных задач по сборке. Что касается GLM-5.1? Естественно, это ещё один шаг вперёд.

## Архитектура мульти-CLI

Одно из ключевых преимуществ HagiCode — поддержка множества различных AI-инструментов CLI через унифицированный слой абстракции. Преимущества этого дизайна, если честно, таковы: новые вещи могут входить, старые могут оставаться, код не путается. Ведь каждый надеется, что жизнь могла бы быть такой, верно?

### Перечисление AIProviderType

Платформа через перечисление `AIProviderType` определяет поддерживаемые типы поставщиков CLI:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI (новый)
}
```

Видно, что Gemini CLI присоединился к этой большой семье в качестве десятого члена. У каждого CLI есть уникальные особенности и применимые сценарии, пользователи могут гибко выбирать в соответствии со своими потребностями. Ведь все дороги ведут в Рим, просто некоторые дороги легче, некоторые немного извилистее.

### Архитектура Provider

HagiCode.Libs.Providers предоставляет унифицированный интерфейс Provider, делая интеграцию каждого CLI стандартизированной и лаконичной. На примере Gemini CLI:

```csharp
public class GeminiProvider : ICliProvider<GeminiOptions>
{
    private static readonly string[] DefaultExecutableCandidates = ["gemini", "gemini-cli"];
    private const string ManagedBootstrapArgument = "--acp";

    public string Name => "gemini";
    public bool IsAvailable => _executableResolver.ResolveFirstAvailablePath(DefaultExecutableCandidates) is not null;
}
```

Преимущества этого дизайна:
- Интеграция нового CLI требует реализации только одного класса Provider
- Унифицированное управление жизненным циклом и пул сессий
- Автоматическое разрешение псевдонимов и поиск исполняемых файлов

По сути, этот дизайн просто упрощает сложные вещи, делая жизнь немного проще.

### Provider Registry

Provider Registry автоматически обрабатывает отображение псевдонимов и регистрацию:

```csharp
if (provider is GeminiProvider)
{
    registry.Register(provider.Name, provider, ["gemini-cli"]);
    continue;
}
```

Это значит, что пользователи могут использовать `gemini` или `gemini-cli` для вызова Gemini CLI, система автоматически распознаёт. Это как когда у тебя много друзей, одних зовут по имени, других по прозвищу, во всех случаях это он, как хочешь, так и зовёшь.

## Поддержка модели GLM-5.1

GLM-5.1 — это новейшая флагманская модель от Zhipu AI, HagiCode завершила её полную поддержку.

### Secondary Professions Catalog

HagiCode через Secondary Professions Catalog управляет всеми поддерживаемыми моделями. Ниже представлена конфигурация серии GLM:

| Model ID | Name | SupportsImage | Compatible CLI Families |
|----------|------|----------------|-------------------------|
| `glm-4.7` | GLM 4.7 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM 5 Turbo | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (Legacy) | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | **true** | claude, codebuddy, hermes, qoder, kiro |

Ключевые особенности GLM-5.1 можно резюмировать так:
- Независимый идентификатор версии, без багажа legacy
- Первая модель GLM, поддерживающая ввод изображений
- Более сильные способности к рассуждению и пониманию кода
- Широкая мульти-CLI совместимость

### GLM-5.1 vs GLM-5.0

С точки зрения кода, ключевые различия между GLM-5.1 и GLM-5.0:

```csharp
// GLM-5.0 (Legacy) - имеет специальную логику сохранения
private const string Glm50CodebuddySecondaryProfessionId = "secondary-glm-5-codebuddy";
private const string Glm50CodebuddyModelValue = "glm-5.0";

// GLM-5.1 - независимый новый идентификатор модели
private const string Glm51SecondaryProfessionId = "secondary-glm-5-1";
private const string Glm51ModelValue = "glm-5.1";
```

GLM-5.0 имеет метку "Legacy", это старый идентификатор версии, сохранённый для обратной совместимости. А GLM-5.1 — это совершенно новая независимая версия без какого-либо исторического багажа. Это как некоторые люди всегда живут в прошлом, а другие с лёгким багажом идут быстрее.

### Конфигурация GLM-5.1

Пример конфигурации для использования GLM-5.1 в HagiCode:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

## Функция загрузки изображений

Поддержка изображений в HagiCode реализована через свойство `SupportsImage` в SecondaryProfession:

```csharp
public class HeroSecondaryProfessionSettingDto
{
    public bool SupportsImage { get; set; }
}
```

В Secondary Professions Catalog конфигурация GLM-5.1 выглядит так:

```json
{
  "id": "secondary-glm-5-1",
  "supportsImage": true
}
```

Это значит, что пользователи могут напрямую загружать скриншоты для анализа AI, например:
- Скриншоты сообщений об ошибках
- Проблемы с UI-интерфейсом
- Графики визуализации данных
- Результаты выполнения кода

Не нужно вручную описывать проблему, просто сделайте скриншот — удобство этой функции поймёшь, как только попробуешь. Ведь некоторые вещи лучше увидеть, чем долго описывать.

## Интеграция Gemini CLI

Gemini CLI как десятый Agent CLI интегрирован в HagiCode через стандартную архитектуру Provider.

### Опции конфигурации

Gemini CLI поддерживает богатые опции конфигурации:

```csharp
public class GeminiOptions
{
    public string? ExecutablePath { get; set; }
    public string? WorkingDirectory { get; set; }
    public string? SessionId { get; set; }
    public string? Model { get; set; }
    public string? AuthenticationMethod { get; set; }
    public string? AuthenticationToken { get; set; }
    public Dictionary<string, string?> AuthenticationInfo { get; set; }
    public Dictionary<string, string?> EnvironmentVariables { get; set; }
    public string[] ExtraArguments { get; set; }
    public TimeSpan? StartupTimeout { get; set; }
    public CliPoolSettings? PoolSettings { get; set; }
}
```

Эти опции охватывают все аспекты от базовой конфигурации до расширенных функций, пользователи могут гибко настраивать в соответствии со своими потребностями. Ведь у каждого свои потребности, гибкость всегда лучше.

### Протокол связи ACP

Gemini CLI поддерживает протокол связи ACP (Agent Communication Protocol), это унифицированный стандарт связи CLI в HagiCode. Через ACP разные CLI могут взаимодействовать с платформой единообразным способом, что значительно упрощает работу по интеграции. По сути, это просто унификация сложных вещей, чтобы всем было легче.

## Практика сборки HagiCode

Что касается практики, лучший пример — это собственный процесс сборки платформы HagiCode. В разработке HagiCode уже полностью используются возможности AI:

### Хороший запуск даже на GLM 4.7

Платформа HagiCode оптимизирована довольно хорошо, даже при использовании GLM 4.7 можно получить хороший опыт разработки. Платформа уже помогла завершить несколько важных проектов сборки, включая:
- Интеграцию множества CLI Provider
- Реализацию функции загрузки изображений
- Генерацию документации и публикацию контента

Это тоже неплохо, ведь не всем нужно использовать самые новые вещи. Подходящее — лучшее.

### GLM-5.1 усиливает эффект

После обновления до GLM-5.1 эти способности будут дополнительно усилены:
- Более сильная способность понимания кода, меньше来回沟通
- Более точный анализ зависимостей, сразу указывает правильное направление
- Более эффективная диагностика ошибок, быстрое定位 проблем
- Поддержка ввода изображений, ускорение описания проблем

Это как переход от велосипеда к автомобилю, места те же, просто скорость и комфорт другие.

## Лучшие практики интеграции мульти-CLI

HagiCode.Libs.Provides предоставляет унифицированный механизм регистрации и использования:

```csharp
services.AddHagiCodeLibs();

var gemini = serviceProvider.GetRequiredService<ICliProvider<GeminiOptions>>();
var codebuddy = serviceProvider.GetRequiredService<ICliProvider<CodebuddyOptions>>();
var hermes = serviceProvider.GetRequiredService<ICliProvider<HermesOptions>>();
```

Этот дизайн внедрения зависимостей делает использование каждого CLI очень лаконичным, а также удобно для модульного тестирования и имитации. Ведь писать код чище — это тоже ответственность перед собой.

### Меры предосторожности

В реальном использовании есть несколько моментов, на которые стоит обратить внимание:

1. **Конфигурация API Key**：Убедитесь в правильной настройке `ANTHROPIC_AUTH_TOKEN`, иначе невозможно вызвать модель
2. **Доступность модели**：GLM-5.1 нужно открыть разрешения у соответствующего поставщика моделей
3. **Функция изображений**：Только модели с поддержкой `supportsImage: true` могут использовать функцию загрузки изображений
4. **Установка CLI**：Перед использованием Gemini CLI убедитесь, что `gemini` или `gemini-cli` находятся в системном PATH

Это всё мелочи, но если мелочи не处理好, могут стать большими проблемами. Так что стоит обратить внимание.
