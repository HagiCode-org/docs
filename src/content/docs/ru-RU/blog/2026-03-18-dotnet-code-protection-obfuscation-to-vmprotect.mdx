## .NET Защита кода на практике: от обфускации до защиты виртуальной машины

> В этой статье будет рассказано, как реализовать многоуровневую стратегию защиты кода в проектах .NET, охватывая все решения от базовой обфускации до профессиональной защиты виртуальной машины.

## Предпосылки

В разработке приложений .NET защита основного кода (например, проверка лицензий, бизнес-логика, конфиденциальные конфигурации и т. д.) от декомпиляции и обратного анализа — это неизбежная тема. По мере созревания экосистемы .NET у разработчиков появилось множество средств защиты кода: от встроенных атрибутов обфускации до профессиональных инструментов виртуализации. Выбор действительно довольно велик.

Как сложный мультиязычный monorepo-проект, HagiCode включает в себя десктопные приложения, системы сборки и функции управления лицензиями. В коде неизбежно присутствует логика проверки лицензий, конфиденциальные конфигурации (например, ключи API, идентификаторы продуктов) и основная бизнес-логика. Всё это необходимо тщательно защищать — ведь никто не хочет, чтобы свои усилия легко увидели другие.

В этой статье мы поделимся решением по защите кода, которое мы фактически применили в проекте HagiCode, обобщим полный процесс от ошибок до оптимизации. Возможно, это даст вам некоторое вдохновение.

## Основное содержание

### 1. Встроенный атрибут обфускации Microsoft

.NET Framework предоставляет встроенный `[ObfuscationAttribute]` — это самый базовый и наиболее часто используемый маркер обфускации кода. Этот атрибут находится в пространстве имён `System.Reflection` и позволяет выполнять базовую защиту кода без использования сторонних инструментов.

**Основные характеристики**:
- Свойство `Feature`: определяет характеристики обфускации, например `"ultra"` (высокая обфускация), `"all"` (полная обфускация)
- Свойство `Exclude`: `true` означает исключение из обфускации, `false` означает применение обфускации
- Может применяться к классам, методам, свойствам и другим членам типов

В проекте HagiCode можно увидеть пример фактического использования:

```csharp
[Obfuscation(Feature = "ultra", Exclude = false)]
public async Task<LicenseValidationResult?> ValidateLicenseAsync(...)
```

**Преимущества** такого подхода довольно очевидны:
- Не требует дополнительных зависимостей, встроен в .NET Framework
- Может распознаваться и обрабатываться сторонними инструментами обфускации
- Не существенно увеличивает размер скомпилированной сборки

Однако у него есть ограничения: только функция маркировки, фактический эффект обфускации зависит от реализации инструмента, не может обеспечить безопасность на уровне защиты виртуальной машины. Но это вполне нормально,毕竟 он изначально для этого и не создавался.

### 2. VMP (Virtual Machine Protection)

VMP — это профессиональный инструмент защиты кода, который обеспечивает высокий уровень защиты путём компиляции кода в инструкции виртуальной машины. В отличие от простой обфускации имён, VMP действительно преобразует логику кода в форму, которая не может быть восстановлена обычными декомпиляторами. Это довольно впечатляет.

**Классификация уровней защиты**:

| Уровень | Виртуализация | Мутация | Антиотладка | Шифрование строк | Применимые сценарии |
|---------|---------------|---------|-------------|------------------|---------------------|
| HIGH | full | high | Включено | Включено | Проверка лицензий, параллелизм сеансов, конфиденциальные константы |
| MEDIUM | partial | medium | Включено | Включено | Бизнес-логика, доменные модели |
| LOW | none | low | Отключено | Отключено | Утилитные классы, некритический код |

В проекте HagiCode определена система декларативных атрибутов для маркировки кода, требующего защиты:

```csharp
// Защита с высоким приоритетом
[VmProtect(VmProtectionPriority.High, Reason = "Contains license verification logic")]
public class KeygenClient { ... }

// Исключение из защиты
[VmExclude(Reason = "Public API that must remain unchanged")]
public class PublicApi { ... }

// Защита наследования
[VmProtect(Priority.High, ProtectDerived = true)]
public class BaseLicenseValidator { ... }
```

### 3. Стратегия защиты при сборке

Защита VMP действует не только во время выполнения, но и должна быть автоматически обработана в процессе сборки — вручную делать это слишком хлопотно. Система сборки HagiCode поддерживает несколько режимов:

- **Родной режим Windows**: прямой вызов инструмента VMProtect
- **Режим контейнера Linux Docker**: запуск VMP в контейнере (решение проблем совместимости между платформами)
- **Сканирование атрибутов**: автоматическое обнаружение маркеров защиты в коде
- **Механизм проверки**: подтверждение успешного применения защиты

Эти функции в сочетании довольно удобны.

## Решение

### 1. Использование встроенного атрибута обфускации Microsoft

Прямое применение `ObfuscationAttribute` в коде:

```csharp
using System.Reflection;

[Obfuscation(Feature = "ultra", Exclude = false)]
public class LicenseService
{
    [Obfuscation(Feature = "ultra", Exclude = false)]
    public async Task<bool> ValidateLicenseAsync(string key)
    {
        // Логика проверки лицензии
    }

    [Obfuscation(Feature = "flow", Exclude = false)]
    private string DecryptToken(string encrypted)
    {
        // Логика расшифровки
    }
}
```

Иногда нужно разрешить тестовым сборкам доступ к внутренним членам, сохраняя безопасность производственного кода:

```csharp
// AssemblyInfo.cs
[assembly: InternalsVisibleTo("HagiCode.Application.Tests")]
[assembly: InternalsVisibleTo("DynamicProxyGenAssembly2")] // for Moq
```

Так тестировать гораздо удобнее — ведь код всё равно нужно тестировать.

### 2. Определение пользовательских атрибутов для защиты VMP

Создание пользовательских атрибутов защиты для управления поведением VMP:

```csharp
using System;

[AttributeUsage(AttributeTargets.Class | AttributeTargets.Method | AttributeTargets.Property)]
public class VmProtectAttribute : Attribute
{
    public VmProtectionPriority Priority { get; set; }
    public string? Reason { get; set; }
    public bool ProtectDerived { get; set; }
}

[AttributeUsage(AttributeTargets.Class | AttributeTargets.Method | AttributeTargets.Property)]
public class VmExcludeAttribute : Attribute
{
    public string? Reason { get; set; }
}

public enum VmProtectionPriority
{
    None = 0,
    Low = 1,
    Medium = 2,
    High = 3
}
```

Собственные атрибуты использовать удобнее —毕竟 вы лучше знаете свои потребности.

### 3. Файл конфигурации VMP

```yaml
# vmp_config.yml

  priority_mode: "attribute"  # Приоритет на основе атрибутов
  default_level: "medium"

  high:

    virtualization: "partial"

```

Четче написанная конфигурация облегчает дальнейшее обслуживание.

## Практическое руководство

### 1. Практика защиты ключевых компонентов

Согласно спецификации code-protection в HagiCode, следующие компоненты должны использовать защиту с приоритетом HIGH:

```csharp
// Константы производственной среды - должны быть зашифрованы и защищены VMP
[VmProtect(VmProtectionPriority.High, Reason = "Production constants")]
public static class ProductionConstants
{
    // Доступ к зашифрованным строкам, защищённый VMP
    [VmProtect(VmProtectionPriority.High)]
    public static string GetLicenseServerUrl(IOptions<LicenseOptions> options) => ...;
}

// Логика проверки лицензий
[VmProtect(VmProtectionPriority.High, Reason = "License verification logic")]
public class KeygenClient : IKeygenClient
{
    [Obfuscation(Feature = "ultra", Exclude = false)]
    public async Task<LicenseValidationResult?> ValidateLicenseAsync(...) { ... }
}

// Сервис машинного отпечатка
[VmProtect(VmProtectionPriority.High)]
public class MachineFingerprintService : IMachineFingerprintService { ... }
```

Ключевой код нуждается в усиленной защите — ведь утечка основной логики создаст проблемы.

### 2. Расшифровка строк во время выполнения

Шифрование строк при сборке, расшифровка во время выполнения:

```csharp
public static class StringDecryption
{
    [VmProtect(VmProtectionPriority.High, Reason = "CRITICAL SECURITY")]
    public static string DecryptString(byte[] encryptedData, byte[] key, byte[] iv)
    {
        using var aes = Aes.Create();
        aes.Key = key;
        aes.IV = iv;

        using var decryptor = aes.CreateDecryptor();
        using var ms = new MemoryStream(encryptedData);
        using var cs = new CryptoStream(ms, decryptor, CryptoStreamMode.Read);
        using var reader = new StreamReader(cs);

        return reader.ReadToEnd();
    }
}

// Доступ к константам производства (ленивая загрузка + кэширование)
public static class ProductionConstants
{
    private static string? _cachedLicenseServerUrl;

    public static string GetLicenseServerUrl(IOptions<LicenseOptions> options)
    {
        if (_cachedLicenseServerUrl == null)
        {
            var encrypted = GetEncryptedLicenseServerUrl();
#if DEBUG
            _cachedLicenseServerUrl = options.Value.PrimaryServer.Url;
#else
            _cachedLicenseServerUrl = StringDecryption.DecryptString(
                encrypted,
                GetEncryptionKey(),
                GetEncryptionIV());
#endif
        }
        return _cachedLicenseServerUrl;
    }
}
```

Шифрование строк тоже важный этап — конфиденциальную информацию нельзя хранить в открытом виде.

### 3. Проверка защиты VMP

После сборки необходимо проверить, успешно ли применена защита — иначе как узнать, действует ли защита:

```csharp
// Пример сценария проверки
public bool VerifyProtection(string assemblyPath)
{
    // 1. Проверка подписи VMP
    var bytes = File.ReadAllBytes(assemblyPath);
    var vmpSignature = Encoding.ASCII.GetBytes("VMProtect");
    if (bytes.Any(b => vmpSignature.Contains(b)))
    {
        return true;
    }

    // 2. Проверка изменения размера файла (после защиты обычно увеличивается)
    var originalInfo = new FileInfo(assemblyPath.Replace(".dll", ".bak"));
    if (originalInfo.Exists)
    {
        var sizeRatio = (double)new FileInfo(assemblyPath).Length / originalInfo.Length;
        return sizeRatio > 1.1;
    }

    return false;
}
```

Проверка всегда полезна — чтобы не узнать о проблеме слишком поздно.

### 4. Меры предосторожности

Здесь есть несколько подводных камней, на которые нужно обратить особое внимание — мы наступали на все эти грабли:

1. **Не混淆весь код**: общедоступные API, определения интерфейсов, классы DTO обычно не нуждаются в защите. Чрезмерная обфускация повлияет на производительность и отладку. Проект HagiCode в этом отношении吃亏 — обратите на это внимание

2. **Защита доступа к ключам**: методы получения ключей шифрования должны иметь тот же или более высокий уровень защиты, чем зашифрованные данные, иначе это бесполезно

3. **Баланс между тестированием и производством**: DEBUG-сборки должны пропускать шифрование для удобства отладки при разработке, RELEASE-сборки включают полную защиту. Помните использовать условную компиляцию `#if DEBUG` для различия — так разработка удобнее

4. **Учёт среды Docker**: для запуска VMP в среде Linux нужно использовать контейнерное решение для обеспечения совместимости инструментов защиты. HagiCode использует контейнерное решение Wine + VMP — так решаются проблемы кроссплатформенности

5. **Проверка обязательна**: после завершения сборки необходимо проверить, успешно ли применена защита, иначе может произойти утечка конфиденциального кода. Приведённый выше код проверки служит именно этой цели — лучше проверить

## Резюме

Благодаря такой многоуровневой стратегии защиты HagiCode реализовала комплексную систему безопасности кода от базовой обфускации до защиты виртуальной машины:

- **Первый уровень**: использование `ObfuscationAttribute` для базовой маркировки, предоставление подсказок сторонним инструментам
- **Второй уровень**: объявление намерений защиты и приоритетов через пользовательский `VmProtectAttribute`
- **Третий уровень**: защита виртуальной машины VMP преобразует ключевой код в необратимые инструкции виртуальной машины
- **Четвёртый уровень**: автоматическое сканирование и применение защиты при сборке, проверка результатов защиты

这套решение может противостоять не только обычным инструментам декомпиляции, но и атакам продвинутого обратного анализа. Если вы также разрабатываете приложения .NET, требующие защиты кода, надеемся, это даст вам некоторую справку — даже небольшое вдохновение уже достаточно.
