## Руководство по созданию эффекта анимации вращающегося света по границе

> Как с помощью чистого CSS создать тот важный элемент, который сразу привлекает внимание пользователей? На самом деле это не так уж сложно, просто нужно найти правильный подход. Эта статья проведёт вас через создание анимации вращающегося света по границе с нуля, а также расскажет о тех проблемах, с которыми мы столкнулись в проекте HagiCode.

## Предыстория

У фронтенд-разработчиков наверняка был такой опыт: менеджер продукта подходит с выражением лица, которое говорит «это требование очень простое» — «Можно ли добавить эффект к этой выполняющейся задаче, чтобы пользователь мог сразу её увидеть?»

Вы говорите «конечно», можно просто изменить цвет границы. Но собеседник качает головой, в его глазах читается «ты не понимаешь»: «Недостаточно заметно, нужен эффект, когда свет вращается по границе, как в научно-фантастических фильмах.»

В этот момент вы можете начать гадать: как это реализовать? Canvas? SVG? Или CSS сможет справиться? Ведь никто не хочет признаваться, что он этого не умеет.

На самом деле анимация вращающегося света по границе очень распространена в современных веб-приложениях, в основном используется в следующих сценариях:

- **Индикация состояния**: маркировка выполняющихся задач или активных проектов
- **Визуальный фокус**: выделение важных областей содержимого
- **Усиление бренда**: создание визуального ощущения технологий и современности
- **Праздничные темы**: создание праздничной атмосферы для особых случаев

При разработке [HagiCode](https://hagicode.com) мы столкнулись с этой задачей — пользователям нужно было с первого взгляда понимать, какие сеансы выполняются, а какие предложения обрабатываются. Мы попробовали несколько вариантов, некоторые пути были проще, другие немного сложнее, и в результате сформировали достаточно зрелый подход к реализации.

## Основные идеи реализации

После анализа кода HagiCode мы выделили несколько основных паттернов реализации, каждый из которых применим в определённых сценариях, или, можно сказать, каждый имеет своё предназначение.

### 1. Вращающееся свечение с помощью Conic Gradient (наиболее распространённый)

Это классический способ реализации вращающегося света по границе, основная идея — использовать `conic-gradient` в CSS для создания конического градиента, а затем заставить его вращаться. Как уличный фонарь ночью, который постоянно вращается.

**Ключевые элементы:**
- Использование псевдоэлемента `::before` для создания слоя свечения
- Использование `conic-gradient` для определения распределения градиента
- Использование псевдоэлемента `::after` для маскирования центральной области (опционально)
- Использование `@keyframes` для реализации анимации вращения

### 2. Светящаяся линия с боковой стороны

Этот вариант подходит для индикации состояния элементов списка, достаточно создать светящуюся тонкую линию с одной стороны элемента, не нужно анимировать всю границу. Ведь иногда достаточно немного света, не нужно освещать весь мир.

**Ключевые элементы:**
- Тонкий линейный элемент с абсолютным позиционированием
- Использование `box-shadow` для создания эффекта свечения
- Использование `scale` и `opacity` для реализации дыхательной анимации

### 3. Светящийся фон с Box-Shadow

Если не нужен эффект вращения, а просто нужно мягкое фоновое свечение, достаточно наложить несколько слоёв `box-shadow`. Иногда простое решение лучше.

### 4. Поддержка доступности

Этот аспект часто игнорируют, но он очень важен. Все анимации должны учитывать медиа-запрос `prefers-reduced-motion`, предоставляя пользователям, которые не любят анимацию, статичную альтернативу. Ведь не все любят постоянное движение, уважение к выбору каждого — это правильный подход.

## Варианты реализации

### Вариант 1: Вращающаяся граница с Conic Gradient (рекомендуется)

Это наиболее полная реализация эффекта вращающегося света по границе,也是最 используемое решение в HagiCode. Ведь если что-то работает хорошо, зачем менять?

```css
/* Родительский контейнер */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Вращающийся слой свечения */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Слой маски (опционально, для создания эффекта пустотелой границы) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Принцип этого решения довольно прост: создаётся псевдоэлемент больше родительского контейнера, на котором рисуется конический градиент, а затем он постоянно вращается. Родительский контейнер имеет `overflow: hidden`, поэтому видна только та часть света, которая проходит по границе. Это как смотреть на уличный фонарь из окна — можно увидеть только тот маленький отрезок, который проходит мимо.

### Вариант 2: Упрощённая вращающаяся светящаяся граница

Если вам не нужен такой сложный эффект, в HagiCode есть более лёгкая реализация с помощью утилитарного класса. Иногда простое решение лучше.

```css
/* Утилитарный класс вращающейся светящейся границы */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Поддержка доступности */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Обратите внимание на `will-change: transform` — это говорит браузеру «этот элемент будет постоянно меняться», и браузер заранее выполнит оптимизацию, анимация станет более плавной. Ведь заранее подготовиться всегда лучше, чем в последний момент.

### Вариант 3: Боковая светящаяся линия

Этот вариант особенно подходит для индикации состояния элементов списка, именно его использует список сеансов в HagiCode. Тонкая линия, которая может выделиться среди множества элементов — разве это не тоже жизненная философия?

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Здесь используется `isolation: isolate` для создания нового контекста наложения, а затем `z-index` для контроля порядка отображения слоёв. `pointer-events: none` также очень важно, иначе псевдоэлемент блокирует клики пользователя. Это как некоторые вещи — красивые, но не должны мешать.

### Вариант 4: Инкапсуляция в React-компонент

Если в вашем проекте используется React, можно инкапсулировать эти логики в компонент, особенно для части доступности. Ведь код пишется один раз, используется много раз — это именно то, что нам нужно.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Соответствующий CSS-модуль:

```css
/* GlowBorder.module.css */

/* Анимационная версия */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Статическая версия (для доступности) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Хук `useReducedMotion` из `framer-motion` автоматически определяет системные предпочтения пользователя, если пользователь включил «уменьшение анимации», он вернёт true, и тогда отображается статическая версия. Ведь уважение к выбору пользователя важнее принудительного показа.

## Обмен практическим опытом

Ниже приведён опыт, который мы накопили, преодолевая проблемы при разработке HagiCode. Это просто некоторые заметки, надеюсь, они помогут вам в будущем.

### 1. Система主题ных переменных

Использование CSS-переменных для поддержки множественных тем особенно удобно. Ведь никто не хочет каждый раз при переключении темы изменять кучу кода.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Использование */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

Таким образом при переключении темы нужно только изменить класс тега `html`, все цвета анимации автоматически обновятся. Один код, два стиля — это именно то, к чему мы стремимся.

### 2. Оптимизация производительности

**Используйте `will-change` для подсказки браузеру об оптимизации:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Заранее скажите браузеру, и он поможет вам выполнить оптимизацию. Это как во многих жизненных ситуациях — заранее подготовиться всегда хорошо.

**Избегайте использования сложного box-shadow на больших элементах:**

```css
/* Плохо - использование размытой тени на большом элементе */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Лучше - использование псевдоэлемента для ограничения области свечения */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Мы тестировали это в HagiCode — прямое добавление размытой тени на большие карточки приводило к падению частоты кадров при прокрутке ниже 30fps, а после перехода на псевдоэлементы стабильно держалось 60fps. Пользователи могут почувствовать эту разницу в опыте.

### 3. Доступность

Этот аспект действительно нельзя опускать, некоторые пользователи могут чувствовать головокружение или раздражение от анимации, уважение к их выбору — базовая этика создания продуктов. Ведь красивые вещи не должны навязываться.

**Медиа-запрос CSS:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Предоставление статичной альтернативы */
    opacity: 1;
  }
}
```

**Определение предпочтений пользователя в React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Контроль уровней интенсивности

Индикатор пропускной способности токенов в HagiCode отображает свет разных цветов в зависимости от реальной пропускной способности, это реализовано динамически. Ведь разные состояния должны иметь разные способы выражения.

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Меры предосторожности

Некоторые детали всё же нужно учитывать, иначе будет поздно, когда наступите на грабли.

| Мера предосторожности | Описание |
|---------------------|----------|
| **Управление z-index** | Слой свечения должен иметь подходящий z-index, чтобы не влиять на взаимодействие с содержимым |
| **pointer-events** | Псевдоэлементы свечения должны иметь `pointer-events: none` |
| **Переполнение границ** | Родительский контейнер должен иметь `overflow: hidden` или нужно настроить размер псевдоэлемента |
| **Влияние на производительность** | Сложные анимации могут влиять на производительность на мобильных устройствах, необходимо тестировать |
| **Тёмный режим** | Убедитесь, что цвета свечения чётко видны на тёмном фоне |
| **Переключение тем** | Используйте CSS-переменные, чтобы обеспечить правильное обновление цветов анимации при переключении тем |

### 6. Техники отладки

Псевдоэлементы иногда сложно найти в инструментах разработчика, можно временно добавить границу для просмотра позиции.

```css
/* Временное отображение границ псевдоэлемента для отладки */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

После настройки позиции не забудьте закомментировать или удалить эту строку, иначе в production будет неловко. Некоторые вещи лучше оставлять в среде разработки.
