## Как сделать AI-программирование более увлекательным с помощью геймификации

> На самом деле традиционные инструменты AI-программирования довольно мощные, им просто не хватает тепла. При создании HagiCode мы задумались: раз уж нужно писать код, почему бы не превратить это в игру?

## Предыстория

Те, кто пользовался AI-помощниками для программирования, вероятно, знакомы с этим опытом: поначалу кажется интересным, но со временем чувствуется, что чего-то не хватает. Сам по себе инструмент очень мощный: генерация кода, автодополнение, исправление ошибок — всё это умеет, просто... нет тепла, со временем становится скучно и монотонно.

Это ещё полбеды,毕竟 кто хочет каждый день смотреть на холодный инструмент?

Это как в видеоиграх: если просто выполнять список задач без развития персонажа, без разблокировки достижений, без командной работы, то быстро станет скучно. Красивые вещи или люди не обязательно должны принадлежать тебе, просто они красивые, и ты любуешься их красотой. Но у инструментов программирования даже этой красоты нет, что неизбежно вызывает разочарование.

В процессе разработки [HagiCode](https://hagicode.com) мы столкнулись с этой проблемой. HagiCode как платформа для сотрудничества нескольких AI-помощников должна поддерживать интерес пользователей на протяжении долгого времени. Но реальность такова, что даже лучший инструмент будет трудно использовать регулярно без эмоциональной связи.

Чтобы решить эту проблему, мы приняли смелое решение: превратить программирование в игру. Не просто с таблицей лидеров, а настоящий ролевой геймифицированный опыт. Изменения, которые принесло это решение, могут быть больше, чем вы можете себе представить.

В конце концов, людям всегда нужно немного ритуала.

## О HagiCode

Решения, описанные в этой статье, основаны на нашем опыте работы над проектом [HagiCode](https://hagicode.com). HagiCode — это платформа для сотрудничества нескольких AI-помощников, поддерживающая совместную работу Claude Code, Codex, Copilot, OpenCode и других AI-помощников. Если вам интересны сотрудничество нескольких AI и геймифицированное программирование, посетите [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site) для получения дополнительной информации.

На самом деле ничего особенного, просто мы превратили программирование в приключение.

## Почему геймификация

Суть геймификации — не "добавить таблицу лидеров", а создать целостную систему мотивации, позволяющую пользователям испытывать чувство роста, достижения и социального признания в процессе выполнения задач.

Геймификация HagiCode построена вокруг одной ключевой концепции: каждый AI-помощник — это "герой", а пользователь — капитан этой команды героев. Вы ведёте этих героев покорять различные "подземелья" (задачи программирования), и в этом процессе герои получают опыт, улучшают свои способности, а вы и ваша команда получаете различные достижения.

Это не маркетинговый трюк, а продуманный дизайн, основанный на психологии человеческого поведения. Когда задачи обретают смысл и обеспечивают обратную связь о прогрессе, вовлечённость и настойчивость людей значительно повышаются.

Как говорили древние, "эту любовь можно было бы сохранить как воспоминание, но в тот момент я уже был потерян". Мы интегрировали этот эмоциональный опыт в инструмент, делая программирование не просто написанием кода, а путешествием, которое стоит запомнить.

## Система персонажей Hero

Hero — это ключевая концепция системы геймификации HagiCode. Каждый Hero представляет собой AI-помощника: например, Claude Code — это Hero, Codex — тоже Hero.

### Три слота Hero

У Hero есть три слота экипировки, это довольно хитрый дизайн:

1. **Слот CLI (основная профессия)**: определяет базовые способности Hero, например, Claude Code или Codex
2. **Слот Model (вторичная профессия)**: определяет используемую модель, например, Claude 4.5 или Claude 4.6
3. **Слот Style (стиль)**: определяет стиль действий Hero, например, "стратег ветра падения" или "другой стиль"

Комбинация трёх слотов может создать уникальную конфигурацию Hero. Как и в игре с экипировкой, вам нужно выбрать подходящую комбинацию в зависимости от характеристик задачи. Ведь подходящее для вас — это лучшее, это как выбор пути в жизни: все дороги ведут в Рим, просто некоторые дороги проще, а другие немного извилистее.

### Система развития Hero

У каждого Hero есть свои XP (очки опыта) и уровень:

```typescript
type HeroProgressionSnapshot = {
  currentLevel: number;                    // текущий уровень
  totalExperience: number;                 // общий опыт
  currentLevelStartExperience: number;     // опыт начала текущего уровня
  nextLevelExperience: number;             // опыт для следующего уровня
  experienceProgressPercent: number;       // процент прогресса
  remainingExperienceToNextLevel: number;  // опыт до следующего уровня
  lastExperienceGain: number;              // последний полученный опыт
  lastExperienceGainAtUtc?: string | null; // время получения опыта
};
```

Уровни разделены на четыре этапа, названия каждого этапа очень атмосферны:

```typescript
export const resolveHeroProgressionStage = (level?: number | null): HeroProgressionStage => {
  const normalizedLevel = Math.max(1, level ?? 1);
  if (normalizedLevel <= 100) return 'rookieSprint';      // спринт новичка
  if (normalizedLevel <= 300) return 'growthRun';         // забег роста
  if (normalizedLevel <= 700) return 'veteranClimb';      // подъём ветерана
  return 'legendMarathon';                                // марафон легенды
};
```

От "новичка" до "легенды" этот путь развития даёт пользователям чёткое чувство цели и достижения. Как рост в жизни, нужно пройти процесс от невежества до зрелости, просто здесь этот процесс конкретизирован.

### Создание настраиваемого Hero

При создании Hero нужно настроить три слота:

```typescript
const heroDraft: HeroDraft = {
  name: 'Athena',
  icon: 'hero-avatar:storm-03',
  description: 'Мудрая стратегиня',
  executorType: AIProviderType.CLAUDE_CODE_CLI,
  slots: {
    cli: {
      id: 'profession-claude-code',
      parameters: { /* параметры CLI */ }
    },
    model: {

      parameters: { /* параметры модели */ }
    },
    style: {
      id: 'fengluo-strategist',
      parameters: { /* параметры стиля */ }
    }
  }
};
```

У каждого Hero есть уникальный аватар, описание и профессиональная направленность, что придаёт холодным AI-помощникам индивидуальность и тепло. Ведь кто хочет иметь дело с инструментом без характера?

## Система подземелий Dungeon

"Подземелье" — классическая концепция в играх, представляющая вызовы, которые требуют командной работы. В HagiCode каждый рабочий процесс — это подземелье.

### Организация подземелий

Dungeon организует рабочие процессы в различные "подземелья":

- **Подземелье генерации предложений**: отвечает за генерацию технических предложений
- **Подземелье выполнения предложений**: отвечает за выполнение задач из предложения
- **Подземелье архивирования предложений**: отвечает за организацию и архивирование выполненных предложений

У каждого подземелья есть свой капитан Hero, капитан автоматически выбирается как первый из включённых Hero.

На самом деле это как разделение труда в жизни: у каждого своя роль, просто здесь это разделение превращено в игровой механизм.

### Механизм командной работы

Вы можете настроить разные команды Hero для разных подземелий:

```typescript
const dungeonRoster: HeroDungeonRoster = {
  scriptKey: 'proposal.generate',
  displayName: 'Генерация предложений',
  members: [
    { heroId: 'hero-1', name: 'Athena', executorType: 'ClaudeCode' },
    { heroId: 'hero-2', name: 'Apollo', executorType: 'Codex' }
  ]
};
```

Например, при генерации предложения использовать Афину (знает стратегии), при выполнении кода — Аполлона (знает реализацию), так каждый герой может проявить свои сильные стороны. Как создание музыкальной группы: у каждого свой инструмент, вместе они создают прекрасную мелодию.

### Управление процессом Dungeon

Dungeon использует фиксированный scriptKey для идентификации различных процессов:

```typescript
// ключи скриптов соответствуют разным рабочим процессам
const dungeonScripts = {
  'proposal.generate': 'Генерация предложений',
  'proposal.execute': 'Выполнение предложений',
  'proposal.archive': 'Архивирование предложений'
};
```

Поток состояний задачи: queued (в очереди) → dispatching (распределение) → dispatched (распределено). Весь процесс автоматизирован, не требует ручного вмешательства. Это тоже наша маленькая ленивая хитрость: ведь кто хочет вручную управлять всем этим?

## Система XP и уровней

XP (очки опыта) — это ключевой механизм обратной связи в системе геймификации. Пользователи получают XP за выполнение задач, XP позволяет героям повышать уровень, повышение уровня разблокирует новые способности, формируя положительный цикл.

### Получение XP

В HagiCode XP можно получить за следующую деятельность:

- Завершение выполнения кода
- Успешный вызов инструмента
- Генерация предложения
- Операции управления сеансами
- Операции с проектами

Каждое завершённое действием приносит соответствующему Hero XP. Как рост в жизни: каждый шаг важен, просто здесь этот рост количественно измерен.

### Визуализация прогресса в реальном времени

Прогресс XP и уровней визуализируется в реальном времени:

```typescript
type HeroDungeonMember = {
  heroId: string;
  name: string;
  icon?: string | null;
  executorType: PCode_Models_AIProviderType;
  currentLevel?: number;                    // текущий уровень
  totalExperience?: number;                 // общий опыт
  experienceProgressPercent?: number;       // процент прогресса
};
```

Пользователи могут в любое время видеть уровень и прогресс каждого Hero, эта немедленная обратная связь — ключ к геймификации. Ведь людям всегда нужна обратная связь, иначе как узнать о прогрессе?

## Система достижений

Достижения — другой важный элемент геймификации, они предоставляют долгосрочные цели и чувство достижения на этапах.

### Типы достижений

HagiCode поддерживает различные типы достижений:

- **Генерация кода**: сгенерировать X строк кода, создать Y файлов
- **Управление сеансами**: завершить Z диалогов
- **Операции с проектами**: работать в W проектах

На самом деле эти достижения как вехи в жизни, просто мы превратили их в игровой механизм.

### Состояния достижений

У достижений есть три состояния:

```typescript
type AchievementStatus = 'unlocked' | 'in-progress' | 'locked';
```

Три состояния имеют чёткое визуальное различие:

- **Разблокировано**: золотой градиент + эффект свечения
- **В процессе**: синяя пульсирующая анимация
- **Заблокировано**: серый цвет, отображаются условия разблокировки

Каждое достижение чётко показывает условия активации, позволяя пользователям знать, что делать дальше. Когда вы потеряны, руководство всегда полезно.

### Эффект празднования при разблокировке

Когда достижение разблокировано, запускается праздничная анимация. Такое положительное подкрепление даёт пользователям чувство "я сделал это", побуждая их продолжать. Как маленькие награды в жизни: хоть и небольшие, но могут радовать долго.

## Ежедневный боевой отчёт Battle Report

Battle Report — это уникальная функция HagiCode, каждый день в конце генерирует полноэкранный отчёт в стиле боя.

### Содержимое отчёта

Battle Report отображает следующую информацию:

```typescript
type HeroBattleReport = {
  reportDate: string;
  summary: {
    totalHeroCount: number;        // общее количество героев
    activeHeroCount: number;       // количество активных героев
    totalBattleScore: number;      // общий боевой счёт
    mvp: HeroBattleHero;           // самый ценный герой
  };
  heroes: HeroBattleHero[];        // подробные данные всех героев
};
```

- Общий счёт команды
- Количество активных Hero
- Количество вызовов инструментов
- Общее время работы
- MVP (самый ценный герой)
- Подробные карточки каждого Hero

### Выделение MVP

MVP — это герой с лучшей производительностью за день, он будет выделен в отчёте. Это не просто статистика, а форма признания заслуг. Ведь кто не хочет быть признанным?

### Подробная карточка Hero

Карточка каждого Hero содержит:

- Прогресс уровня
- Полученный XP
- Количество выполнений
- Время использования

Эти данные позволяют пользователям чётко понимать состояние работы команды. Ведь понимание результатов своих усилий — тоже форма удовлетворения.

## Техническая реализация

Система геймификации HagiCode использует современный технологический стек и паттерны проектирования. На самом деле ничего особенного, просто выбрали удобные инструменты.

### Выбор технологического стека

```typescript
// фронтенд использует React + TypeScript
import React from 'react';

// анимация использует Framer Motion
import { AnimatePresence, motion } from 'framer-motion';

// управление состоянием использует Redux Toolkit
import { useAppDispatch, useAppSelector } from '@/store';

// UI-компоненты используют shadcn/ui
import { Dialog, DialogContent } from '@/components/ui/dialog';
```

Framer Motion отвечает за все анимационные эффекты, shadcn/ui предоставляет базовые UI-компоненты, Redux Toolkit управляет сложным состоянием геймификации. Ведь чтобы хорошо делать дело, нужны хорошие инструменты.

### Система дизайна геймифицированного UI

HagiCode использует стиль Glassmorphism + Tech Dark:

```css
/* основной градиент */
background: linear-gradient(135deg, #22C55E 0%, #25c2a0 50%, #06b6d4 100%);

/* эффект стекла */
backdrop-filter: blur(12px);

/* эффект сияния */
background: radial-gradient(circle at center, rgba(34, 197, 94, 0.15) 0%, transparent 70%);
```

Зелёный градиент в сочетании с эффектом стекла создаёт ощущение технологий и будущего. Ведь визуальная красота — тоже часть пользовательского опыта.

### Реализация анимационных эффектов

Использование Framer Motion для плавных анимаций входа:

```typescript
<motion.div
  animate={{ opacity: 1, y: 0 }}
  initial={{ opacity: 0, y: 18 }}
  transition={{ duration: 0.35, ease: 'easeOut', delay: index * 0.08 }}
  className="card"
>
  {/* содержимое карточки */}
</motion.div>
```

Каждая карточка входит последовательно с задержкой 0.08 секунды, создавая плавный визуальный эффект. Ведь плавная анимация улучшает опыт, это бесспорно.

### Сохранение данных

Данные геймификации используют систему хранения Grain, обеспечивая согласованность состояния. Даже такие детальные данные, как накопление XP Hero, точно сохраняются. Ведь никто не хочет потерять накопленный с трудом опыт.

## Практическое руководство

### Создание первого Hero

Создание первого Hero довольно простое:

1. Перейдите на страницу управления Hero
2. Нажмите кнопку "Создать Hero"
3. Настройте три слота (CLI, Model, Style)
4. Дайте Hero имя и описание
5. Сохраните, ваш первый Hero создан

Как знакомство с новым другом: нужно дать ему имя, узнать его особенности, а затем вы можете отправиться в приключение вместе.

### Создание команды Dungeon

Создание команды тоже простое:

1. Перейдите на страницу управления Dungeon
2. Выберите подземелье для настройки (например, "Генерация предложений")
3. Выберите участников из вашего списка Hero
4. Система автоматически выберет первого включённого Hero как капитана
5. Сохраните конфигурацию

На самом деле это процесс формирования команды, как создание команды в жизни: у каждого своя роль.

### Просмотр ежедневного отчёта

В конце каждого дня вы можете посмотреть Battle Report за день:

1. Нажмите кнопку "Боевой отчёт"
2. Полноэкранный отображает результаты работы за день
3. Посмотрите MVP и подробные данные каждого Hero
4. Поделитесь с членами команды (опционально)

Это тоже своего рода ритуал: позволяющий узнать, сколько вы приложили усилий сегодня и как далеко до цели.

## Предостережения и лучшие практики

### Оптимизация производительности

Используйте React.memo для предотвращения ненужных повторных рендеров:

```typescript
const HeroCard = React.memo(({ hero }: { hero: HeroDungeonMember }) => {
  // реализация компонента
});
```

Ведь производительность тоже важна, кто хочет использовать зависающий инструмент?

### Падение анимации

Проверьте настройки движения пользователей, предоставьте упрощённый опыт для чувствительных пользователей:

```typescript
const prefersReducedMotion = useReducedMotion();
const duration = prefersReducedMotion ? 0 : 0.35;
```

Ведь не всем нравится анимация, уважение к предпочтениям пользователей — тоже часть дизайна.

### Обратная совместимость

Сохраните legacyIds для поддержки миграции старых версий:

```typescript
type HeroDungeonMember = {
  heroId: string;
  legacyIds?: string[];  // поддержка отображения старых ID
  // ...
};
```

Ведь никто не хочет потерять данные из-за обновления версии.

### Поддержка интернационализации

Весь текст использует ключи перевода i18n для удобства многоязычной поддержки:

```typescript
const displayName = t(`dungeon.${scriptKey}`, { defaultValue: displayName });
```

Ведь язык не должен быть барьером для использования.

## Заключение

Геймификация — это не просто таблица лидеров, а целостная система мотивации. HagiCode через систему Hero, систему Dungeon, систему XP/уровней, систему достижений и Battle Report превращает программирование в героическое путешествие, полное духа приключений.

Ключевая ценность этой системы:

- **Эмоциональная связь**: придаёт холодным AI-помощникам индивидуальность
- **Положительная обратная связь**: каждая операция имеет немедленную обратную связь
- **Долгосрочные цели**: уровни и достижения предоставляют путь развития
- **Командная идентичность**: чувство командной работы в Dungeon
- **Признание заслуг**: отображение Battle Report и MVP

Геймифицированный дизайн делает программирование не скучным, а интересным приключением. Пользователи при выполнении задач кода испытывают удовольствие от развития персонажа, командной работы и разблокировки достижений, повышая удержание и активность.

На самом деле программирование — это уже творчество, просто мы сделали этот процесс творчества немного интереснее.
