## Positioning

MiniMax is one of the providers currently recommended by the docs preset set for Claude Code. A single default model across tiers, suitable for teams that value simple configuration and predictable onboarding.

## Good fit for

- You want the same mapped model across all Claude tiers
- You prefer low-friction setup and maintenance
- You want to get Claude Code running quickly before tuning model strategy

## Key things to notice

- The current preset metadata marks MiniMax as recommended.
- It uses `MiniMax-M2.5` as the default mapped model for all tiers.
- This is a good fit when operational simplicity matters more than tier-specific tuning.

## Default model mapping

| Claude tier | Default model |
|-------------|---------------|
| Sonnet | `MiniMax-M2.5` |
| Opus | `MiniMax-M2.5` |
| Haiku | `MiniMax-M2.5` |

:::tip
Pricing, promotions, and package details can change over time. If cost is your main concern, confirm the latest information directly on the provider page.
:::

## Suggested next steps

1. Confirm which subscription tier or API activation path you want to use.
2. Return to the Claude Code setup guide and place the base URL plus token into `settings.json` or environment variables.
3. Validate the connection locally after setup.
