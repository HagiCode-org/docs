## About iFlow

This page deliberately starts with the official iFlow quickstart and only then adds the extra ACP WebSocket checks required by HagiCode.

In practice that means:

1. install and sign in to iFlow CLI exactly as the official quickstart describes
2. then validate the `--experimental-acp --port {port}` flow that HagiCode currently uses

## Scope and Official Source

- **Target**: users who want to install iFlow CLI locally and then connect it to HagiCode
- **Last verified**: 2026-03-13
- **Official quickstart**: [iFlow CLI Quickstart](https://platform.iflow.cn/cli/quickstart)
- **HagiCode integration baseline**: `ExecutablePath: "iflow"`, `Arguments: "--experimental-acp --port {port}"`, `AuthMethod: "iflow"`

## Finish the official quickstart first

### Official system requirements

The official iFlow quickstart lists these prerequisites:

- Node.js `22+`
- `4GB+` memory
- internet connectivity
- basic terminal familiarity

The official page also strongly recommends **Windows Terminal** on Windows to reduce compatibility issues.

### Official install path for macOS / Linux

If you want the official one-click installer, the quickstart gives this command:

```bash
bash -c "$(curl -fsSL https://gitee.com/iflow-ai/iflow-cli/raw/main/install.sh)"
```

If Node.js `22+` is already installed, the official npm command is:

```bash
npm install -g @iflow-ai/iflow-cli@latest
```

### Official install path for Windows

The official quickstart says to:

1. install the latest Node.js from the official Node.js download page
2. restart CMD or PowerShell
3. run the CLI install command

```bash
npm install -g @iflow-ai/iflow-cli@latest
```

After installation, the official page tells you to verify with:

```bash
iflow --version
```

**Expected result**: the terminal prints a version string, confirming that the CLI is installed.

## Complete the official first-time setup

### 1) First launch

The official quickstart starts from:

```bash
iflow
```

### 2) Choose a login method

The official page says iFlow CLI currently supports three sign-in modes:

- **Login with iFlow (recommended)**: the fullest experience, including WebSearch, WebFetch, multimodal capability, and better tool-calling behavior
- **iFlow API Key**: suited for server environments or machines without browser access; the official page says the API key is valid for `7` days and must be refreshed regularly
- **OpenAI Compatible API**: suited for your own model service or another OpenAI-compatible endpoint, but without the iFlow platform features such as WebSearch, WebFetch, and built-in multimodal support

:::warning
The official quickstart includes an important notice dated `2025-10-10`: authentication is being unified, API keys may need manual refresh, and the recommended recovery path is to use `/auth` to sign in through iFlow.
:::

### 3) Finish sign-in before moving to HagiCode

Make sure you can enter a normal iFlow session in the terminal first. That keeps official login issues separate from HagiCode integration issues.

## Extra ACP WebSocket validation required by HagiCode

The steps above cover the official install and first login. HagiCode adds one more requirement: iFlow must also be able to start in ACP WebSocket mode.

### 1) Run a local ACP startup check

HagiCode currently launches iFlow like this:

```bash
iflow --experimental-acp --port 8765
```

**Expected result**: the CLI binds the local port successfully. After the check, stop it with `Ctrl+C`.

If the command errors immediately, cannot bind the port, or does not recognize `--experimental-acp`, fix that before continuing.

### 2) Confirm HagiCode can detect iFlow

Recommended validation order:

1. finish the official quickstart and run `iflow --version`
2. run `iflow` and confirm the first-login flow or `/auth` flow has completed
3. run `iflow --experimental-acp --port 8765` manually
4. start HagiCode and inspect provider status, health checks, or startup logs
5. confirm there are no missing `iflow`, auth, port, or ACP handshake errors

## Troubleshooting

| Symptom | Best interpretation based on the official docs | Suggested resolution |
|---|---|---|
| `iflow` command not found | The install step did not finish cleanly, or the new terminal session was never opened | Reopen the terminal, rerun the install command, and verify again with `iflow --version` |
| First login stalls or auth fails | The selected sign-in mode is wrong, or a pre-`2025-10-10` API-key habit is still being used | Prefer the official `Login with iFlow` flow or `/auth`; if you use API keys, confirm the key is still valid |
| Features are missing after API-based login | The official quickstart distinguishes iFlow platform login from OpenAI Compatible API mode | Switch to `Login with iFlow` or iFlow API Key if you need WebSearch, WebFetch, multimodal features, or iFlow-optimized tool calling |
| `--experimental-acp` fails | This is an extra HagiCode integration requirement, not part of the base quickstart verification | First verify install and sign-in, then isolate version issues, port conflicts, and local security software |
| HagiCode cannot connect to iFlow | The CLI is installed, but local WebSocket, port binding, or auth is still failing | Keep the startup logs and focus on port binding, sign-in state, and local networking conditions |
