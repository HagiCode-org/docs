## Brief Overview

The .NET SDK is a required dependency for running Hagicode. After installation, you can use Hagicode.

:::tip[Version Requirements]
Hagicode requires **.NET SDK >= 10.0** as a runtime dependency.
:::

## Installing .NET SDK

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Method One: Official Installer (Recommended)

1. Visit the [.NET Official Website](https://dotnet.microsoft.com/download)
2. Select **.NET 10** version
3. Download the **SDK** (not Runtime) Windows installer (`.exe` or `.msi` file)
4. Double-click to run the installer
5. Follow the installation wizard prompts to complete installation:
   - Accept the license agreement
   - Choose installation path (default is fine)
   - Click "Install" to start installation
6. After installation completes, click "Close"

### Method Two: Using winget

Open PowerShell or Command Prompt and run:

```powershell
winget install Microsoft.DotNet.SDK.10
```

:::tip
If winget cannot find .NET 10 SDK, you can update the winget source first:
```powershell
winget source update
```
:::

### Method Three: Using Chocolatey

If you already have [Chocolatey](https://chocolatey.org/) installed:

```powershell
choco install dotnet-10-sdk
```

  </TabItem>

### Method One: Official Installer (Recommended)

1. Visit the [.NET Official Website](https://dotnet.microsoft.com/download)
2. Select **.NET 10** version
3. Download the **SDK** macOS installer (`.pkg` file)
4. Double-click to run the installer
5. Follow the installation wizard prompts to complete installation:
   - Click "Continue"
   - Accept the license agreement
   - Choose installation disk
   - Click "Install" and enter administrator password
6. After installation completes, click "Close"

### Method Two: Using Homebrew

If you have [Homebrew](https://brew.sh/) installed, you can install .NET SDK via terminal:

```bash
# Add Microsoft tap
brew tap microsoft/dotnet

# Install .NET 10 SDK
brew install dotnet-sdk-10
```

:::tip
Homebrew will automatically add .NET SDK to your PATH. If the command is not available, restart the terminal or run:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

### Method Three: Using .NET Install Script

```bash
# Download and run the installation script
curl -sSL https://dot.net/v1/dotnet-install.sh | bash /dev/stdin --channel 10.0

# Add .NET to PATH (add to ~/.zshrc or ~/.bashrc)
export DOTNET_ROOT=$HOME/.dotnet
export PATH=$PATH:$DOTNET_ROOT:$DOTNET_ROOT/tools
```

### Using Microsoft Package Repository (Recommended)

This is the best way to get the latest .NET SDK version:

```bash
# Add Microsoft package repository
wget https://packages.microsoft.com/config/ubuntu/$(lsb_release -rs)/packages-microsoft-prod.deb -O packages-microsoft-prod.deb
sudo dpkg -i packages-microsoft-prod.deb
rm packages-microsoft-prod.deb

# Update package list
sudo apt-get update

# Install .NET 10 SDK
sudo apt-get install -y dotnet-sdk-10.0
```

### Using apt Package Manager

The default repository for Ubuntu 24.04+ may contain .NET 10 SDK:

```bash
sudo apt-get update
sudo apt-get install -y dotnet-sdk-10.0
```

:::note
If .NET 10 SDK is not in the default repository, use the Microsoft package repository method.
:::

### Using Snap

Ubuntu also supports installation via Snap:

```bash
sudo snap install dotnet-sdk --classic --channel=10
```

### Using Microsoft Package Repository (Recommended)

```bash
# Add Microsoft package repository
sudo rpm -Uvh https://packages.microsoft.com/config/centos/$(rpm -E %{rhel})/packages-microsoft-prod.rpm

# Update package list
sudo yum update

# Install .NET 10 SDK
sudo yum install -y dotnet-sdk-10.0
```

### Using yum Package Manager

```bash
sudo yum install -y dotnet-sdk-10.0
```

:::note
If .NET 10 SDK is not in the system repository, use the Microsoft package repository method.
:::

  </TabItem>
  <TabItem value="fedora" label="Fedora">

### Using dnf Package Manager

```bash
# Add Microsoft package repository
sudo rpm -Uvh https://packages.microsoft.com/config/fedora/$(rpm -E %fedora)/packages-microsoft-prod.rpm

# Update package list
sudo dnf update

# Install .NET 10 SDK
sudo dnf install -y dotnet-sdk-10.0
```

:::tip
Fedora's software repository usually contains newer versions of .NET and can be installed directly via dnf.
:::

  </TabItem>
  <TabItem value="arch" label="Arch Linux">

### Using pacman Package Manager

```bash
# Install .NET 10 SDK
sudo pacman -S dotnet-sdk-10.0
```

:::tip
Arch Linux uses rolling updates and usually contains the latest version of .NET SDK.
:::

## Verifying Installation

After installation, verify that .NET SDK is installed correctly:

```bash
# Check .NET SDK version
dotnet --version

# View detailed SDK information
dotnet --info
```

:::note
Make sure .NET SDK version >= 10.0.0 (recommended to use the latest .NET 10 version). If the version is too low, please reinstall using the methods above.
:::

## Troubleshooting

### Windows

If the `dotnet` command is not available after installation:

1. Restart Command Prompt or PowerShell
2. If still not available, manually add .NET to PATH:
   - Search for "Environment Variables" and open "Edit system environment variables"
   - Click "Environment Variables"
   - Find "Path" in "System variables" and click "Edit"
   - Add the .NET SDK installation path (default: `C:\Program Files\dotnet\`)
   - Click "OK" to save changes

### macOS

If the command is not available after installation:

1. Restart Terminal
2. If installed via Homebrew, run:
   ```bash
   eval "$(/opt/homebrew/bin/brew shellenv)"
   ```
3. Manually add to PATH (in `~/.zshrc` or `~/.bash_profile`):
   ```bash
   export DOTNET_ROOT=$HOME/.dotnet
   export PATH=$PATH:$DOTNET_ROOT:$DOTNET_ROOT/tools
   ```

### Linux

If you encounter permission errors when running `dotnet`:

```bash
# Ensure the installation path is executable
sudo chmod +x $HOME/.dotnet/dotnet

# If installed via system package manager, reinstall
sudo apt-get install --reinstall dotnet-sdk-10.0
```

## Appendix: Uninstalling .NET SDK

If you need to uninstall .NET SDK:

<Tabs>
  <TabItem value="win-uninstall" label="Windows">

1. Open "Control Panel" > "Programs and Features"
2. Find "Microsoft .NET SDK - 10.0.0"
3. Right-click and select "Uninstall"

Or use command line:

```powershell
# Uninstall via winget
winget uninstall Microsoft.DotNet.SDK.10

# Uninstall via Chocolatey
choco uninstall dotnet-10-sdk
```

  </TabItem>
  <TabItem value="mac-uninstall" label="macOS">

```bash
# If installed via Homebrew
brew uninstall dotnet-sdk-10

# If installed via installer
sudo rm -rf /usr/local/share/dotnet
sudo rm -rf /usr/local/bin/dotnet
```

  </TabItem>
  <TabItem value="linux-uninstall" label="Linux">

```bash
# Ubuntu/Debian
sudo apt-get remove dotnet-sdk-10.0
sudo apt-get autoremove

# CentOS/RHEL
sudo yum remove dotnet-sdk-10.0

# Fedora
sudo dnf remove dotnet-sdk-10.0

# If installed via install script
rm -rf ~/.dotnet
```
