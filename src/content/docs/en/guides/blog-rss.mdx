## Compatibility

`https://docs.hagicode.com/blog/rss.xml` is the existing compatibility feed and continues to include posts in every supported language. Existing feed readers, automation scripts, and README sync workflows do not need to change. The English feed keeps `rss.en.xml` and also exposes `rss.en-US.xml` for the desktop canonical code.

## Language Rule

RSS generation normalizes a language value for every blog post before filtering:

- If a post explicitly sets `language` in frontmatter, it is normalized to a desktop canonical code: `zh-CN`, `zh-Hant`, `en-US`, `ja-JP`, `ko-KR`, `de-DE`, `fr-FR`, `es-ES`, `pt-BR`, or `ru-RU`.
- If a post does not set explicit language metadata, the language is derived from the content path: `src/content/docs/blog/` is `zh-CN`, `src/content/docs/en/blog/` is `en-US`, and the other language trees use their directory names.

Each RSS item includes a matching `<language>` marker so feed readers and downstream automation can identify the post language.

## Route Conventions

The blog preserves the existing Simplified Chinese root route and the English short route. Other desktop languages use their canonical language code as the route prefix.

| Language code | Blog route | Content directory | Fallback languages |
|---------------|------------|-------------------|--------------------|
| `zh-CN` | `/blog/<slug>/` | `src/content/docs/blog/` | `en-US` |
| `zh-Hant` | `/zh-Hant/blog/<slug>/` | `src/content/docs/zh-Hant/blog/` | `zh-CN`, `en-US` |
| `en-US` | `/en/blog/<slug>/` | `src/content/docs/en/blog/` | `en-US` |
| `ja-JP` | `/ja-JP/blog/<slug>/` | `src/content/docs/ja-JP/blog/` | `en-US` |
| `ko-KR` | `/ko-KR/blog/<slug>/` | `src/content/docs/ko-KR/blog/` | `en-US` |
| `de-DE` | `/de-DE/blog/<slug>/` | `src/content/docs/de-DE/blog/` | `en-US` |
| `fr-FR` | `/fr-FR/blog/<slug>/` | `src/content/docs/fr-FR/blog/` | `en-US` |
| `es-ES` | `/es-ES/blog/<slug>/` | `src/content/docs/es-ES/blog/` | `en-US` |
| `pt-BR` | `/pt-BR/blog/<slug>/` | `src/content/docs/pt-BR/blog/` | `en-US` |
| `ru-RU` | `/ru-RU/blog/<slug>/` | `src/content/docs/ru-RU/blog/` | `en-US` |

## Add A Blog Post

When you add a blog post, add an `.mdx` file with the same filename to every content directory listed above. The date and slug must stay identical across languages, for example `2026-04-29-example.mdx`.

The `language` frontmatter field is optional because the validator can derive language from the route. If you declare it explicitly, it must match the file's route language. Before submitting the change, run `npm run verify:blog-i18n-completeness` from `repos/docs` to check language directories, counterpart files, and language metadata.
