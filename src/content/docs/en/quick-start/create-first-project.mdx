Welcome to Hagicode! After installation, let's create your first project. This guide walks you through the setup step by step, from preparing the code repository to creating the project in Hagicode, so you can move straight into the next workflow.

## Prerequisites

Before creating your first project, make sure you have:

- A code repository you want to manage with Hagicode
- Git installed and configured
- Basic understanding of command line operations

Don't worry, these are all simple!

## Step 1: Prepare Your Code Repository

Before adding a project to Hagicode, you need to prepare a code repository that you want to manage with Hagicode.

### Clone Your Code Repository

If the code you want to manage is already in a remote repository (GitHub, GitLab, etc.), clone it locally:

```bash
# Clone your code repository locally
git clone https://github.com/your-username/your-repo.git
cd your-repo
```

After cloning, note down the repository's local path (e.g., `C:\Users\YourName\Projects\your-repo` or `/home/yourname/projects/your-repo`), you will need this path in the next step.

:::tip
Hagicode is suitable for managing any code project you want to assist in development and optimization with AI, including:
- Feature projects currently under development
- Existing code that needs refactoring or optimization
- Projects for team collaboration
- Personal open source projects
:::

### If Your Code is Only Local

If the code you want to manage is still local and hasn't been pushed to a remote repository:

```bash
cd /path/to/your/project
git init
git add .
git commit -m "Initial commit"
```

It is recommended to create a remote repository on GitHub/GitLab and push:

```bash
git remote add origin https://github.com/your-username/your-repo.git
git push -u origin main
```

## Step 2: Add Project in Hagicode Interface

Now let's add the project to the Hagicode interface.

### Access Project Page

1. Navigate to `http://127.0.0.1:34567` in your browser
2. Click **Projects** in the navigation sidebar
3. Click the **Add Project** button

![Click new project button](../../img/screenshots/quick-start/create-project/step1-click-new-project-button/original.png)

### Configure Project Settings

Fill in project information:

![Fill in general project information](../../img/screenshots/quick-start/create-project/step2-fill-project-information/original.png)

:::note
The repository path must point to a valid Git repository on your local computer.
:::

:::note[Docker Compose Deployment Notes]
If you are using the **Docker Compose deployment method**, you need to pay attention when filling in the repository path:

- **Use container path**: Fill in the path inside the container, not the host's path
- **Path mapping relationship**: Based on the path mapping configuration in `docker-compose.yml`
  - Host path: `/path/to/your/repos` (configured by you in docker-compose.yml)
  - Container path: `/app/workdir` (fixed path)

**Example**:
If your `docker-compose.yml` is configured as:
```yaml
volumes:
  - /home/user/myproject:/app/workdir
```

Then when filling in the repository path, you should use the container path:
```
/app/workdir
```

**Important Tips**:
- Ensure your code repository is mapped to the `/app/workdir` path in the container
- If using software package deployment method, just fill in the actual path on the host
:::

### Create Project

After filling in the required information:

1. Click the **Create Project** button to add the project
2. Hagicode will verify the repository path
3. Your project will appear in the project list

![After creating a new project, you can see this project in the project list](../../img/screenshots/quick-start/create-project/step3-project-list-view/original.png)

### After Creation

Once the project appears in the list, it is ready to use in Hagicode. From there, you can continue directly into the session workflows and start exploring AI-assisted development.
