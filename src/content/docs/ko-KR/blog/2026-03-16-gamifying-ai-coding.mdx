## 게임화 디자인으로 AI 프로그래밍을 더 재미있게 만드는 방법

> 사실 기존의 AI 프로그래밍 도구는 기능이 꽤 강력하지만, 다소 따뜻함이 부족합니다. 우리가 HagiCode를 만들 때 생각한 것은, 어차피 코드를 작성해야 한다면 왜 게임으로 만들지 않을까 하는 것이었습니다.

## 배경

AI 프로그래밍 도우미를 사용해 본 분들은 이런 경험을 해보셨을 것입니다. 처음에는 꽤 신선하게 느껴지다가도, 사용하다 보면 뭔가 부족하다는 느낌을 받게 됩니다. 도구 자체의 기능은 매우 강력해서 코드 생성, 자동 완성, 버그 수정 등 모든 것을 할 수 있지만... 다소 온기가 없고, 오래 사용하면 단조롭고 지루하게 느껴집니다.

이건 그렇다 쳐도, 매일 차가운 도구를 마주하고 싶은 사람은 아무도 없습니다.

이건 마치 게임과 같습니다. 단순히 미션 목록을 완료하기만 하고, 캐릭터 성장도 없고, 성취감 해금도 없고, 팀워크도 없다면 금방 지루함을 느끼게 됩니다. 아름다운 것 또는 사람을 꼭 소유할 필요는 없습니다. 그저 아름다운 것은 아름다운 것이고, 자신이 그 아름다움을 잘 바라보면 됩니다. 하지만 프로그래밍 도구에는 이런 아름다움조차 없어서 낙담하게 됩니다.

우리가 [HagiCode](https://hagicode.com)를 개발하는 과정에서 이 문제에 직면했습니다. HagiCode는 다중 AI 도우미 협업 플랫폼으로, 사용자가 장기적으로 사용 열정을 유지하도록 해야 합니다. 하지만 현실은 아무리 좋은 도구라도 정서적 연결이 부족하면 사용자가 지속하기 어렵습니다.

이러한 문제를 해결하기 위해 우리는 대담한 결정을 내렸습니다. 프로그래밍을 게임으로 만드는 것입니다. 단순한 점수 순위표가 아니라 진정한 롤플레잉식 게임화 경험입니다. 이 결정이 가져온 변화는 상상보다 훨씬 클 수 있습니다.

毕竟，人嘛，总是需要点仪式感的。

## HagiCode에 대해

이 문서에서 공유하는 방안은 [HagiCode](https://hagicode.com) 프로젝트에서의 실천 경험에서 나온 것입니다. HagiCode는 Claude Code, Codex, Copilot, OpenCode 등 다양한 AI 도우미가 함께 작업할 수 있는 다중 AI 도우미 협업 플랫폼입니다. 다중 AI 협업, 게임화 프로그래밍에 관심이 있다면 [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site)를 방문하여 자세히 알아볼 수 있습니다.

사실 특별한 것은 없습니다. 우리는 그저 프로그래밍을 모험으로 만들었을 뿐입니다.

## 왜 게임화를 선택했는가

게임화의 본질은 "점수판 추가"가 아니라, 사용자가 작업을 수행하는 과정에서 성장감, 성취감, 사회적 인정을 경험할 수 있는 완전한 인센티브 체계를 구축하는 것입니다.

HagiCode의 게임화 디자인은 핵심 개념을 중심으로 전개됩니다. 모든 AI 도우미는 "영웅"이고, 사용자는 이 영웅 팀의 팀장입니다. 영웅들을 이끌고 다양한 "던전"(프로그래밍 작업)을 정복하면서, 영웅들은 경험을 얻고 레벨업으로 능력을 해제하며, 사용자와 팀은 다양한 성취를 얻게 됩니다.

이것은 어떤 기교가 아니라 인간 행동 심리학을 기반으로 한 신중한 디자인입니다. 작업에 의미와 진행 피드백이 부여될 때, 사람의 몰입도와 지속 정도가 현저히 향상됩니다.

옛사람들이 말한 것처럼 "이 정은 추억이 될 수 있으나, 당시에는 이미 막막했다"라고 합니다. 우리는 이러한 정서적 경험을 도구에 녹여내어, 프로그래밍이 단순히 코드를 입력하는 것이 아니라 추억할 만한 여정이 되도록 했습니다.

## Hero 캐릭터 시스템

Hero는 HagiCode 게임화 시스템의 핵심 개념입니다. 모든 Hero는 AI 도우미를 나타냅니다. 예를 들어 Claude Code는 Hero이고, Codex도 Hero입니다.

### Hero의 세 개 슬롯

Hero에는 세 개의 장비 슬롯이 있으며, 이 디자인은 꽤 영리합니다.

1. **CLI 슬롯(주요 직업)**: Hero의 기본 능력을 결정합니다. 예를 들어 Claude Code 또는 Codex
2. **Model 슬롯(보조 직업)**: 사용하는 모델을 결정합니다. 예를 들어 Claude 4.5 또는 Claude 4.6
3. **Style 슬롯(스타일)**: Hero의 행동 스타일을 결정합니다. 예를 들어 "풀락 전략가" 또는 "다른 스타일"

세 슬롯의 조합은 독특한 Hero 구성을 만들 수 있습니다. 게임에서 장비를 구성하듯, 작업 특징에 따라 적절한 조합을 선택해야 합니다. 자신에게 맞는 것이 가장 좋다는 것은 인생의 길을 선택하는 것과 비슷합니다. 모든 길은 로마로 통하지만, 어떤 길은 더 쉽고 어떤 길은 조금 더 굽이굽이할 뿐입니다.

### Hero의 성장 시스템

모든 Hero는 자신만의 XP(경험치)와 레벨을 가집니다.

```typescript
type HeroProgressionSnapshot = {
  currentLevel: number;                    // 현재 레벨
  totalExperience: number;                 // 총 경험치
  currentLevelStartExperience: number;     // 현재 레벨 시작 경험치
  nextLevelExperience: number;             // 다음 레벨所需 경험치
  experienceProgressPercent: number;       // 진행률 백분율
  remainingExperienceToNextLevel: number;  // 다음 레벨까지 남은 경험치
  lastExperienceGain: number;              // 마지막으로 얻은 경험치
  lastExperienceGainAtUtc?: string | null; // 경험치 획득 시간
};
```

레벨은 네 단계로 나뉘며, 각 단계의 명명은 몰입감을 줍니다.

```typescript
export const resolveHeroProgressionStage = (level?: number | null): HeroProgressionStage => {
  const normalizedLevel = Math.max(1, level ?? 1);
  if (normalizedLevel <= 100) return 'rookieSprint';      // 신인 스프린트
  if (normalizedLevel <= 300) return 'growthRun';         // 성장 런
  if (normalizedLevel <= 700) return 'veteranClimb';      // 베테랑 클라임
  return 'legendMarathon';                                // 레전드 마라톤
};
```

"신인"에서 "레전드"까지, 이 성장 경로는 사용자에게 명확한 목표감과 성취감을 줍니다. 인생의 성장과 마찬가지로, 무지에서 성숙으로 가는 과정을 겪어야 하며, 여기서는 이러한 과정을 구체화했을 뿐입니다.

### 사용자 정의 Hero 생성

Hero를 생성할 때 세 슬롯을 구성해야 합니다.

```typescript
const heroDraft: HeroDraft = {
  name: 'Athena',
  icon: 'hero-avatar:storm-03',
  description: '지략이 뛰어난 전략가',
  executorType: AIProviderType.CLAUDE_CODE_CLI,
  slots: {
    cli: {
      id: 'profession-claude-code',
      parameters: { /* CLI 관련 매개변수 */ }
    },
    model: {
      id: 'secondary-claude-4-sonnet',
      parameters: { /* 모델 관련 매개변수 */ }
    },
    style: {
      id: 'fengluo-strategist',
      parameters: { /* 스타일 관련 매개변수 */ }
    }
  }
};
```

모든 Hero는 독특한 아바타, 설명 및 전문 포지션을 가지고 있어, 차가운 AI 도우미가 개성과 온기를 가지게 됩니다. 성격이 없는 도구와 거래하고 싶은 사람은 아무도 없습니다.

## Dungeon 던전 시스템

"던전"은 게임의 고전적인 개념으로, 팀을 모아攻略해야 하는 도전을 나타냅니다. HagiCode에서 모든 워크플로는 하나의 Dungeon입니다.

### Dungeon의 조직 방식

Dungeon은 워크플로를 다른 "던전"으로 조직합니다.

- **제안 생성 던전**: 기술 제안 생성을 담당
- **제안 실행 던전**: 제안의 작업 실행을 담당
- **제안 보관 던전**: 완료된 제안의 정리 및 보관을 담당

모든 던전은 자신만의 Captain(팀장) Hero를 가지며, 팀장은 활성화된 Hero 중 첫 번째를 자동으로 선택합니다.

사실 이것은 삶의 분업과 같습니다. 모든 사람에게 자신만의 역할이 있지만, 여기서는 이러한 분업을 게임 메커니즘으로 만들었을 뿐입니다.

### 팀 협업 메커니즘

다른 던전에 대해 다른 Hero 팀을 구성할 수 있습니다.

```typescript
const dungeonRoster: HeroDungeonRoster = {
  scriptKey: 'proposal.generate',
  displayName: '제안 생성',
  members: [
    { heroId: 'hero-1', name: 'Athena', executorType: 'ClaudeCode' },
    { heroId: 'hero-2', name: 'Apollo', executorType: 'Codex' }
  ]
};
```

예를 들어 제안 생성에는 Athena(전략에 능함)를 사용하고, 코드 실행에는 Apollo(구현에 능함)를 사용하여, 모든 영웅이 자신의 장점을 발휘할 수 있습니다. 밴드를 결성하는 것과 같습니다. 모든 사람에게 자신만의 악기가 있고, 합쳐야 아름다운 선율을 연주할 수 있습니다.

### Dungeon 프로세스 제어

Dungeon은 고정된 scriptKey를 사용하여 다른 프로세스를 식별합니다.

```typescript
// 스크립트 키 값은 다른 워크플로에 해당
const dungeonScripts = {
  'proposal.generate': '제안 생성',
  'proposal.execute': '제안 실행',
  'proposal.archive': '제안 보관'
};
```

작업 상태 흐름은 queued(대기 중) → dispatching(전송 중) → dispatched(전송 완료)입니다. 전체 과정은 자동화되어 수동 개입이 필요하지 않습니다. 이것도 우리의 게으른 마음입니다. 누가 이런 일을 수동으로 관리하고 싶겠습니까?

## XP 및 레벨 시스템

XP(경험치)는 게임화 시스템의 핵심 피드백 메커니즘입니다. 사용자는 작업을 완료하여 XP를 얻고, XP는 영웅을 레벨업시키며, 레벨업은 새로운 능력을 해제하여 긍정적인 순환을 형성합니다.

### XP 획득 방법

HagiCode에서 XP는 다음 활동을 통해 얻을 수 있습니다.

- 코드 실행 완료
- 도구 호출 성공
- 제안 생성
- 세션 관리 작업
- 프로젝트 작업

유효한 작업을 완료할 때마다 해당 Hero가 XP를 얻습니다. 삶의 성장과 같습니다. 모든 단계가 중요하지만, 여기서는 이러한 성장을 정량화했을 뿐입니다.

### 실시간 진행률 시각화

XP 및 레벨의 진행률은 실시간으로 시각화됩니다.

```typescript
type HeroDungeonMember = {
  heroId: string;
  name: string;
  icon?: string | null;
  executorType: PCode_Models_AIProviderType;
  currentLevel?: number;                    // 현재 레벨
  totalExperience?: number;                 // 총 경험치
  experienceProgressPercent?: number;       // 진행률 백분율
};
```

사용자는 언제든지 모든 Hero의 레벨과 진행률을 볼 수 있으며, 이러한 즉각적인 피드백은 게임화 디자인의 핵심입니다. 사람은 항상 피드백이 필요합니다. 그렇지 않으면 자신이 발전했는지 어떻게 알겠습니까?

## 성취 시스템

성취는 게임화의 또 다른 중요한 요소로, 장기 목표와 이정표式的인 성취감을 제공합니다.

### 성취 유형

HagiCode는 다양한 성취 유형을 지원합니다.

- **코드 생성 유형**: X행 코드 생성, Y개 파일 생성
- **세션 관리 유형**: Z회 대화 완료
- **프로젝트 작업 유형**: W개 프로젝트에서 작업

사실 이러한 성취는 인생의 이정표와 같지만, 우리는 그것을 게임 메커니즘으로 만들었을 뿐입니다.

### 성취 상태

성취에는 세 가지 상태가 있습니다.

```typescript
type AchievementStatus = 'unlocked' | 'in-progress' | 'locked';
```

세 가지 상태에는 명확한 시각적 구분이 있습니다.

- **해제됨**: 금색 그라데이션 + 후광 효과
- **진행 중**: 파란색 펄스 애니메이션
- **잠김**: 회색, 해제 조건 표시

모든 성취는 트리거 조건을 명확하게 표시하여 사용자가 다음 단계에서 무엇을 해야 할지 알 수 있게 합니다. 막막할 때 가이드가 있는 것은 언제나 좋은 일입니다.

### 해제 시 축하 효과

성취가 해제되면 축하 애니메이션이 트리거됩니다. 이러한 긍정적 강화는 사용자에게 "나는 했다"는 만족감을 주어 계속前进하도록 동기를 부여합니다. 삶의 작은 보상과 같습니다. 비록 크지는 않지만 오랫동안 기분 좋게 만들 수 있습니다.

## Battle Report 일일 전투 보고서

Battle Report는 HagiCode의 독특한 기능으로, 매일 끝날 때 전체 화면으로 표시되는 전투 스타일 보고서를 생성합니다.

### 보고서 내용

Battle Report는 다음 정보를 표시합니다.

```typescript
type HeroBattleReport = {
  reportDate: string;
  summary: {
    totalHeroCount: number;        // 총 영웅 수
    activeHeroCount: number;       // 활성 영웅 수
    totalBattleScore: number;      // 총 전투 점수
    mvp: HeroBattleHero;           // 최우수 영웅
  };
  heroes: HeroBattleHero[];        // 모든 영웅의 상세 데이터
};
```

- 팀 총 점수
- 활성 Hero 수
- 도구 호출 횟수
- 총 작업 시간
- MVP(최우수 영웅)
- 모든 Hero의 상세 카드

### MVP 하이라이트 표시

MVP는 당일 가장 뛰어난 성과를 낸 Hero로, 보고서에서 하이라이트로 표시됩니다. 이것은 단순한 데이터 통계가 아니라 명예 인정입니다. 자신이 인정받기를 원하지 않는 사람은 누구일까요?

### Hero 상세 카드

모든 Hero의 카드에는 다음이 포함됩니다.

- 레벨 진행률
- XP 획득량
- 실행 횟수
- 사용 시간

이러한 데이터는 사용자가 팀의 작업 상태를 명확하게 알 수 있게 합니다. 자신의 노력 결과를 아는 것도 일종의 만족감입니다.

## 기술적 구현

HagiCode의 게임화 시스템은 현대적인 기술 스택과 디자인 패턴을 채택했습니다. 사실 특별한 것은 없습니다. 그저 적합한 도구를 선택했을 뿐입니다.

### 기술 스택 선택

```typescript
// 프런트엔드는 React + TypeScript 사용
import React from 'react';

// 애니메이션은 Framer Motion 사용
import { AnimatePresence, motion } from 'framer-motion';

// 상태 관리는 Redux Toolkit 사용
import { useAppDispatch, useAppSelector } from '@/store';

// UI 컴포넌트는 shadcn/ui 사용
import { Dialog, DialogContent } from '@/components/ui/dialog';
```

Framer Motion은 모든 애니메이션 효과를 담당하고, shadcn/ui는 기본 UI 컴포넌트를 제공하며, Redux Toolkit은 복잡한 게임화 상태를 관리합니다. 일을 잘하려면 도구를 먼저 sharpening해야 합니다.

### 게임화 UI 디자인 시스템

HagiCode는 Glassmorphism + Tech Dark 디자인 스타일을 채택했습니다.

```css
/* 주요 그라데이션 색상 */
background: linear-gradient(135deg, #22C55E 0%, #25c2a0 50%, #06b6d4 100%);

/* 글래스모피즘 효과 */
backdrop-filter: blur(12px);

/* 광채 효과 */
background: radial-gradient(circle at center, rgba(34, 197, 94, 0.15) 0%, transparent 70%);
```

녹색 계열 그라데이션과 글래스모피즘 효과가 결합되어 기술감과 미래감을 조성합니다. 시각적 미학도 사용자 경험의 일부입니다.

### 애니메이션 효과 구현

Framer Motion을 사용하여 유창한 진입 애니메이션을 구현합니다.

```typescript
<motion.div
  animate={{ opacity: 1, y: 0 }}
  initial={{ opacity: 0, y: 18 }}
  transition={{ duration: 0.35, ease: 'easeOut', delay: index * 0.08 }}
  className="card"
>
  {/* 카드 내용 */}
</motion.div>
```

모든 카드가 순차적으로 진입하며 0.08초 지연되어 유창한 시각적 효과를 만듭니다. 유창한 애니메이션은 경험을 더 좋게 만듭니다. 이것은 의심의 여지가 없습니다.

### 데이터 지속성

게임화 데이터는 Grain 저장 시스템을 사용하여 상태 일관성을 보장합니다. Hero의 XP 누적과 같은 세분화된 데이터도 정확하게 지속화할 수 있습니다. 고생해서 모은 경험이 사라지길 원하는 사람은 아무도 없습니다.

## 실천 가이드

### 첫 번째 Hero 생성

첫 번째 Hero를 생성하는 것은 꽤 간단합니다.

1. Hero 관리 페이지로 이동
2. "Hero 생성" 버튼 클릭
3. 세 슬롯 구성(CLI, Model, Style)
4. Hero 이름과 설명 입력
5. 저장, 첫 번째 Hero가 탄생합니다

새로운 친구를 사귀는 것과 같습니다. 이름을 지어주고 특징을 이해한 다음 함께 모험을 떠날 수 있습니다.

### Dungeon 팀 구성

팀을 구성하는 것도 간단합니다.

1. Dungeon 관리 페이지로 이동
2. 구성할 던전 선택(예: "제안 생성")
3. Hero 목록에서 구성원 선택
4. 시스템이 첫 번째 활성화된 Hero를 팀장으로 자동 선택
5. 구성 저장

사실 이것은 일종의 팀 결성 과정입니다. 삶에서 팀을 구성하는 것과 같습니다. 모든 사람에게 자신만의 역할이 있습니다.

### 일일 보고서 보기

매일 끝나면 당일의 Battle Report를 볼 수 있습니다.

1. "전투 보고서" 버튼 클릭
2. 전체 화면으로 당일의 작업 성과 표시
3. MVP와 모든 Hero의 상세 데이터 보기
4. 팀 구성원과 공유(선택 사항)

이것도 일종의 의식입니다. 오늘 얼마나 노력했고 목표에서 얼마나 멀리 있는지 알 수 있습니다.

## 주의 사항 및 모범 사례

### 성능 최적화

React.memo를 사용하여 불필요한 재렌더링을 피하세요.

```typescript
const HeroCard = React.memo(({ hero }: { hero: HeroDungeonMember }) => {
  // 컴포넌트 구현
});
```

성능도 중요합니다. 누가 버벅거리는 도구를 사용하고 싶겠습니까?

### 애니메이션 축소 가능

사용자의 움직임 기본 설정을 감지하여 민감한 사용자에게 간소화된 경험을 제공하세요.

```typescript
const prefersReducedMotion = useReducedMotion();
const duration = prefersReducedMotion ? 0 : 0.35;
```

모든 사람이 애니메이션을 좋아하는 것은 아닙니다. 사용자의 기본 설정을 존중하는 것도 디자인의 일부입니다.

### 후방 호환성

legacyIds를 유지하여 구버전 마이그레이션을 지원하세요.

```typescript
type HeroDungeonMember = {
  heroId: string;
  legacyIds?: string[];  // 구버전 ID 매핑 지원
  // ...
};
```

버전 업그레이드로 데이터가 손실되기를 원하는 사람은 아무도 없습니다.

### 국제화 지원

모든 텍스트는 i18n 번역 키를 사용하여 다국어 지원을 용이하게 하세요.

```typescript
const displayName = t(`dungeon.${scriptKey}`, { defaultValue: displayName });
```

언어는 사용의 장벽이 되어서는 안 됩니다.

## 요약

게임화는 단순한 점수 순위표가 아니라 완전한 인센티브 체계입니다. HagiCode는 Hero 시스템, Dungeon 시스템, XP/레벨 시스템, 성취 시스템 및 Battle Report를 통해 프로그래밍 작업을 모험 정신이 가득한 영웅의 여정으로 전환합니다.

이 시스템의 핵심 가치는 다음과 같습니다.

- **정서적 연결**: 차가운 AI 도우미에 개성을 부여
- **긍정적 피드백**: 모든 작업에 즉각적인 피드백 제공
- **장기 목표**: 레벨과 성취로 성장 경로 제공
- **팀 정체성**: Dungeon 팀 협업감
- **명예 인정**: Battle Report 및 MVP 전시

게임화 디자인은 프로그래밍을 지루하지 않고 재미있는 모험으로 만듭니다. 사용자는 코드 작업을 완료하는 동시에 캐릭터 성장, 팀 협업 및 성취 해제의 즐거움을 경험하여 사용粘性과 활성도를 높입니다.

사실 프로그래밍 자체는 일종의 창조입니다. 우리는 그저 이러한 창조 과정을 조금 더 재미있게 만들었을 뿐입니다.
