## Agent에게 대체될 것 같아서, 데이터로 계산해 보았다

> 데이터로 AI 대체 위험을 정량화: HagiCode 팀이 6개 핵심 공식으로 지식 근로자의 경쟁력 평가 기준을 어떻게 재정의했는지 심층 분석합니다.

## 배경

AI 기술이 급속히 발전하는 오늘날, 모든 지식 근로자는 시급한 문제에 직면해 있습니다. **AI 시대에 나는 도태될 것인가?**

이 질문은 약간 위협적으로 들리지만, 사실 많은 사람들이 마음속으로 불안해하고 있습니다. 방금 프레임워크 하나를 배웠는데 AI가 이 직무를 대체할 것이라고 합니다. 간신히 한 언어에 능통해졌는데, AI를 사용하는 사람의 산출량이 자신의 3배라는 것을 발견했습니다. 이러한 불안감은 화면 앞의 여러분도 어느 정도 체감하실 것이라고 믿습니다.

사실 이러한 불안에는 이유가 있습니다.毕竟 자신이 수년간 노력한 기술이 ChatGPT 하나에 의해 초월될 수 있다는 것을 인정하고 싶지 않은 사람은 없을 것입니다. 다만 불안은 불안일 뿐, 생활은 계속되어야 합니다.

전통적인 관점은 종종 "AI가 무엇을 할 수 있는가"에서 대체 위험을 논의하지만, 이 방법은 두 가지 핵심 차원을 무시합니다:

1. **기업 관점**: 기업이 직원에게 AI 도구를 배치할 의향은 AI 비용의 인건비용 대비 효율에 달려 있습니다. AI가 이 직무를 대체할 수 있다고 해서 기업이 즉시 사람을 바꾸는 것은 아닙니다. 경제적 장부를 계산해야 합니다.毕竟 자본가도 자선사단가는 아니며, 모든 돈은 효율적으로 쓰여야 합니다.

2. **효율 관점**: AI가 가져오는 효율 향상은 정량화되어야 하며, 단순히 "AI를 사용하면 더 강해진다"고 생각해서는 안 됩니다. AI로 효율이 2배 향상되었지만, 그는 5배 향상시켰다면 그 차이는 결코 작지 않습니다. 학생 시절 모두 수업을 들었는데, 어떤 학생은 90점, 어떤 학생은 겨우 합격 — 차이는 이렇게 벌어집니다.

그렇다면 핵심 질문은: **이 막연한 불안을 정량화할 수 있는 지표로 어떻게 바꿀 것인가?**

毕竟 자신의 위치를 아는 것이 어둠 속에서 헤매는 것보다 낫습니다. 오늘 이야기할 것은 바로 HagiCode 팀이 개발한 AI 인건효율 계산기背后的 설계 논리입니다.

그래서 https://cost.hagicode.com 이라는 웹사이트를 만들었습니다.

## 핵심 공식: 6개 핵심 지표

### 1. 연간 전체 인건비용

기업이 직원에게 지불하는 실제 비용은 급여에 불과합니다. 이 점은 이직할 때 많은 사람들이야 발견합니다. 분명 월급 2만으로 이야기했는데, 실수령은 왜 1만 4천입니까? 회사는 2만 이상을 지출합니다. 사회보험, 적립금, 교육, 채용 비용까지 모두 계산해야 합니다.

`calculate-ai-risk.ts`의 구현에 따르면:

```
연간 전체 인건비용 = 연봉 × (1 + 도시 계수) + 연봉 / 12
```

도시 계수는 다양한 도시의 인재 채용 및 유지 비용을 반영합니다:

| 도시 계층 | 대표 도시 | 계수 |
|---------|---------|------|
| 1선 | 베이징/상하이/선전/광저우 | 0.4 |
| 신1선 | 항저우/청두/쑤저우/난징 | 0.3 |
| 2선 | 우한/시안/텐진/정저우 | 0.2 |
| 기타 | 이창/뤄양 등 | 0.1 |

1선 도시 계수는 0.4로, 기업이 약 40%의 추가 채용, 교육, 사회보험 등 부가 비용을 지불해야 함을 의미합니다. 베이징에서 한 명을 채용하는 종합 비용은 2선 도시보다 확실히 높습니다.

毕竟 대도시에서 생존하려면 생활비도 높습니다. 이것은 일종의 "부유자 세금"입니다.

### 2. 종합 Token 단가

다양한 AI 모델에는 Input과 Output 두 가지 가격이 있으며, 차이가 큽니다. 코드 시나리오에서 입력 출력 비율은 약 3:1입니다. AI에게 코드 조각을 주어 review하게 하면, 출력 분석 텍스트는 일반적으로 입력 코드보다 훨씬 짧습니다.

종합 단가 계산 공식:

```
종합 단가 = (입력 출력 비율 × 입력 단가 + 출력 단가) / (입력 출력 비율 + 1)
```

GPT-5를 예로 들면:
- 입력: $2.5/1M tokens
- 출력: $15/1M tokens
- 종합 = (3 × 2.5 + 15) / 4 = $5.625/1M tokens

USD 가격의 모델은 환율로 변환해야 합니다. HagiCode 팀이 설정한 환율은 7.25이며, 시장 변동에 따라 업데이트됩니다.

환율이란 주식 시장과 같아서 아무도 정확히 예측할 수 없습니다. 따라가는 수밖에 없습니다.

### 3. AI 연간 비용

```
일평균 AI 비용 = 일평균 Token 수요 (M) × 종합 단가 (CNY/1M)
연간 AI 비용 = 일평균 AI 비용 × 264 영업일
```

264 = 22일/월 × 12월, 이것은 표준 근무 제도의 연간 영업일 수입니다. 왜 365일을 사용하지 않을까요? 주말, 공휴일, 병가 등의 요인을 고려해야 하기 때문입니다.

毕竟 우리는 로봇이 아닙니다. 쉴 때는 쉬어야 합니다. AI는 쉴 필요가 없을 수 있지만, 우리에게는 숨을 돌릴 여유가 필요합니다.

### 4. 핵심 혁신: 등가 인건

이것은 전체 평가 체계의 핵심이며, HagiCode 팀이 가장 통찰력 있는 부분입니다.

```
부담 가능 워크플로 부수 = 연간 전체 인건비용 / AI 연간 비용
부담 가능 비율 = min(부담 가능 워크플로 부수, 1)
등가 인건 = 1 + (효율 배수 - 1) × 부담 가능 비율
```

잠깐, 이 공식은 약간 복잡합니다. 설명해 드리겠습니다:

전통적인 관점은 "효율이 2배 향상되었다"고 직접 말하지만, 이 공식은 핵심 제약을 고려했습니다: **기업의 AI 예산이 지속 가능한가?**

예를 들어, 샤오밍은 효율이 3배 향상되었지만, 그의 AI 소비 비용은 매년 30만 위안입니다. 반면 회사가 그에게 지급하는 연봉은 20만 위안입니다. 이 경우 샤오밍의 개인 효율은 매우 높지만, 실제로는 지속 불가능합니다. 회사가 그가 높은 효율을 유지하기 위해 적자를 감수할 수는 없습니다.

부담 가능 비율이 바로 이 뜻입니다. 기업이 0.5부의 AI 워크플로만 부담할 수 있다면, 샤오밍의 등가 인건 = 1 + (3-1) × 0.5 = 2명이 되며, 3명이 아닙니다.

**핵심 통찰**: 효율 배수가 얼마나 높은 것이 아니라, 기업이 그 효율을 유지하는 데 필요한 AI 투자를 부담할 수 있는가입니다.

사실 이 이치도 꽤 간단합니다. 다만 많은 사람들이 이 방면으로 생각하지 못했을 뿐입니다.毕竟 우리는 자신의 관점에서 문제를 바라보는 데 익숙하고, 보스의 관점에서 고려하는 경우는 거의 없습니다. 그들의 돈도 바람에서 떨어진 것이 아닙니다.

### 5. 비용 효율 비율

```
AI 비용 비중 = AI 연간 비용 / 연간 전체 인건비용
효율 증폭 = 효율 배수 - 1
비용 효율 비율 = 효율 증폭 / AI 비용 비중
```

- 비용 효율 비율 < 1: AI 투자가 불리함, 효율 향상이 비용을 상쇄하지 못함
- 비용 효율 비율 1-2: 겨우 수지 맞음
- 비용 효율 비율 > 2: 고수익, 강력히 권장

이 지표는 기업 관리자에게 특히 유용하여, 특정 직무에 AI 도구 투자가 가치 있는지 빠르게 평가할 수 있습니다.

毕竟 ROI가 왕입니다. 효율이 아무리 향상되어도 비용이 폭발하면 아무도 사지 않습니다.

### 6. 위험 등급

등가 인건에 따라 위험을 분류합니다:

| 등가 인건 | 위험 등급 | 결론 |
|---------|---------|------|
| >= 2.0 | 고위험 | 동료가 동일한 조건을 갖추면 당신에게 큰 위협이 됩니다 |
| 1.5 - 2.0 | 경고 | 동료가 이미 명확한 효율 우위를 형성하기 시작했습니다 |
| < 1.5 | 안전 | 당분 격차를 유지할 수 있습니다 |

이 표를 보시면 대략 감이 잡히실 것입니다. 다만 너무 불안해하지 마세요.毕竟 불안도 문제를 해결하지 못합니다. 자신의 효율 배수를 어떻게 높일지 생각해 보세요.

## 게임화 설계: 7가지 특수 칭호

평가 결과에 더 많은 흥미를 부여하기 위해 계산기는 7가지 특수 칭호 시스템을 도입했습니다. 칭호는 `localStorage`로 영속화되며, 사용자는 자신의 "성취"를 잠금 해제하고 표시할 수 있습니다.

| 칭호 ID | 이름 | 획득 조건 |
|--------|------|---------|
| craftsman-spirit | 장인 정신 | 일평균 Token = 0 |
| prompt-alchemist | 프롬프트 연금술사 | 일 Token <= 20M이고 효율 배수 >= 6 |
| all-in-operator | 올인 오퍼레이터 | 일 Token >= 150M이고 효율 배수 >= 3 |
| minimalist-runner | 미니멀리스트 러너 | 일 Token <= 5M이고 효율 배수 >= 2 |
| cost-tamer | 비용 테이머 | 비용 효율 비율 >= 2.5이고 AI 비중 <= 15% |
| danger-oracle | 위험 예언자 | 등가 인건 >= 2.5 또는 고위험 구역 진입 |
| budget-coordinator | 예산 코디네이터 | 부담 가능 워크플로 부수 >= 8 |

각 칭호 뒤에는 숨겨진 의미가 있습니다:

| 칭호 | 숨겨진 의미 |
|-----|---------|
| 장인 정신 | AI 없이도 잘 살 수 있지만, 고유한 경쟁력이 필요합니다 |
| 프롬프트 연금술사 | 적은 Token으로 높은 산출을 달성하는 극단적 사용자 |
| 올인 오퍼레이터 | 고투입 고산출, 고빈도 시나리오에 적합 |
| 미니멀리스트 러너 | 경량급 AI 사용, 경미 보조 시나리오에 적합 |
| 비용 테이머 | ROI가 극히 높아 기업이 가장 좋아하는 직원 유형 |
| 위험 예언자 | 당신은 이미 또는 곧 고위험 그룹입니다 |
| 예산 코디네이터 | 당신은 동시에 여러 AI 워크플로를 운영할 수 있습니다 |

사실 게임화란 건, 건조한 데이터에 약간의 흥미를 더하는 것뿐입니다.毕竟 누가 성취를 모으는 것을 좋아하지 않겠습니까? 게임의 배지와 같아서 실제 쓸모는 없지만, 보기에는 마음이 편합니다.

## 실제 사용 예제

베이징의 개발자로서 연봉 40만, Claude Sonnet 4.6 사용, 일평균 Token 소비 50M, 자가 평가 효율 향상 3배라고 가정합니다. 시뮬레이션 입력:

```typescript
const input = {
  annualIncomeCny: 400000,
  cityTier: "tier1",           // 베이징
  modelId: "claude-sonnet-4-6",
  performanceMultiplier: 3.0,
  dailyTokenUsageM: 50,
}

// 계산 과정
// 연간 전체 인건비용 = 40만 × (1 + 0.4) + 40만/12 ≈ 60.33만
// AI 연간 비용 ≈ 50 × 7.125 × 264 ≈ 9.4만
// 부담 가능 워크플로 부수 ≈ 60.33 / 9.4 ≈ 6.4부
// 등가 인건 = 1 + (3 - 1) × 1 = 3명
```

결론: 동료가 동일한 조건을 갖추면 3명의 생산 능력에 해당하며, 당신은 이미 고위험 구역에 있습니다.

AI 사용법이 "불리하다"(비용 효율 비율 < 1)고 발견되면 다음을 고려할 수 있습니다:

1. **Token 소비 감소**: 더 효율적인 프롬프트 사용, 무효 요청 감소
2. **가성비 모델 선택**: 예를 들어 DeepSeek-V3(인민폭 가격, 더 저렴)
3. **효율 배수 향상**: 고급 Agent 사용 기술 학습, AI를 진정한 생산력으로 변환

사실 이 문제들은 근원적으로 균형의 예술입니다. 너무 많이 쓰면 돈이 낭비되고, 너무 적게 쓰면 효과가 없습니다. 그 적절한 점을 찾는 것이 핵심입니다.

## 기술 아키텍처 하이라이트

HagiCode 팀이 이 계산기를 설계할 때 몇 가지 참고할 만한 엔지니어링 결정이 있습니다:

1. **순수 프론트엔드 계산**: 모든 계산이 브라우저에서 완료되며, 백엔드 API에 의존하지 않아 사용자 개인정보를 보호합니다
2. **설정 구동**: 모든 공식, 가격, 직무 데이터를 설정 파일에 중앙 집중하여 향후 업데이트 시 핵심 코드 로직 수정이 필요 없습니다
3. **다국어 지원**: 중국어와 영어를 지원합니다
4. **즉각 피드백**: 사용자가 매개변수를 입력하면 결과가 실시간으로 업데이트됩니다
5. **상세 공식 표시**: 각 결과에는 완전한 계산 공식이 동반되어 사용자의 이해를 돕습니다

이 설계는 계산기를 유지보수 및 확장하기 쉽게 만들고, 유사한 데이터 기반 애플리케이션에 참고 템플릿을 제공합니다.

毕竟 좋은 아키텍처는 좋은 코드와 같이 시간이 필요한沉淀입니다. HagiCode 팀은 이 부분에서 꽤 마음을 썼습니다.
