## Agent에게 대체될 것 같아서 데이터로 계산해 보았다

> 데이터로 AI 대체 위험을 정량화: HagiCode 팀이 6개 핵심 공식으로 지식 근로자의 경쟁력 평가 기준을 어떻게 재정의했는지 심층 분석합니다.

## 배경

AI 기술이 급속히 발전하는 오늘날, 모든 지식 근로자는 시급한 문제에 직면해 있습니다: **AI 시대에, 저는 도태될까요?**

이 문제는 다소 과장되게 들릴 수 있지만, 실제로 많은 사람들이 마음속으로 불안해하고 있습니다. 어떤 프레임워크를 막 배웠는데 AI가 이 직무가 대체될 것이라고 하고, 어떤 언어를 겨우 익혔는데 AI를 사용하는 사람의 생산성이 자신의 3배라는 것을 알게 되는 경우—이러한 불안감을 화면 앞의 여러분도 어느 정도 느끼실 것이라고 생각합니다.

사실 이러한 불안감은 근거가 없지 않습니다. 아무도 자신이 오랫동안 노력해온 기술이 ChatGPT 하나에 의해 뛰어넘어질 수 있다는 사실을 인정하고 싶지 않을 테니까요. 하지만 불안해하면서도 생활은 계속되어야 하지 않을까요?

전통적인 관점은 종종 "AI가 무엇을 할 수 있는가"라는 관점에서 대체 위험을 논의하지만, 이 방법은 두 가지 핵심 차원을 무시합니다:

1. **기업 관점**: 기업이 직원에게 AI 도구를 제공할지 여부는 인건비에 대한 AI 비용의 가성비에 달려 있습니다. AI가 이 직무를 대체할 수 있다고 해서 기업이 즉시 사람을 교체하는 것은 아닙니다. 경제적인 계산이 필요합니다. 자본가도 자선가가 아니며, 모든 돈을 효율적으로 써야 합니다.

2. **효율성 관점**: AI가 가져오는 효율성 향상은 정량화되어야 하며, 단순히 "AI를 사용하면 더 강하다"고 생각해서는 안 됩니다. AI로 효율성이 2배 향상되었지만, 그는 AI로 5倍 향상되었습니다. 이 격차는 적지 않습니다. 학생 시절과 같습니다. 모두 수업을 듣지만, 어떤 학생은 90점, 어떤 학생은 겨우 합격—이런 식으로 격차가 벌어지는 것입니다.

따라서 핵심 문제는 **이러한 모호한 불안을 정량화할 수 있는 지표로 어떻게 변환할 것인가**입니다.

자신의 위치를 아는 것이 어둠 속에서摸索하는 것보다 낫습니다. 이것이 오늘 우리가 이야기할 것입니다—HagiCode 팀이 개발한 AI 인간 효율성 계산기 뒤의 설계 논리.

## 핵심 공식: 6개 핵심 지표

### 1. 연간 총 인건비

기업이 직원에게 지불하는 실제 비용은 급여 그 이상입니다. 이 점은 많은 사람들이 이직할 때才发现합니다—명백히 2만 월급을 협상했는데, 실제 수령은 1만 4천? 회사 쪽은 2만 이상을 지불합니다. 사회보험, 주택공적금, 교육, 채용 비용을 모두 포함해야 합니다.

`calculate-ai-risk.ts`의 구현에 따르면:

```
연간 총 인건비 = 연봉 × (1 + 도시 계수) + 연봉 / 12
```

도시 계수는不同 도시의 인재 모집 및 유지 비용을 反映합니다:

| 도시 등급 | 대표 도시 | 계수 |
|---------|---------|------|
| 1선 | 베이징/상하이/선전/광저우 | 0.4 |
| 新1선 | 항저우/청두/쑤저우/난징 | 0.3 |
| 2선 | 우한/시안/톈진/정저우 | 0.2 |
| 其他 | 이창/뤄양 등 | 0.1 |

1선 도시 계수는 0.4입니다. 이는 기업이 약 40%의 추가 모집, 교육, 사회보험 등 부대 비용을 지불해야 함을 의미합니다. 베이징에서 사람을 채용하는 종합 비용은 2선 도시보다 확실히 높습니다.

큰 도시에서 생존하려면 생활비도 높습니다. 이것은 다른 형태의 "유목민세"라고 할 수 있습니다.

### 2. 종합 Token 단가

다른 AI 모델은 Input과 Output 두 가지 가격이 있으며, 차이가 매우 큽니다. 코드 시나리오에서 입력 출력 비율은 약 3:1입니다—AI에 코드를 보내고 리뷰를 요청하면, 출력되는 분석 텍스트는 일반적으로 입력 코드보다 훨씬 짧습니다.

종합 단가 계산 공식:

```
종합 단가 = (입력 출력 비율 × 입력 단가 + 출력 단가) / (입력 출력 비율 + 1)
```

GPT-5를 예로 들어 보겠습니다:
- 입력: $2.5/1M tokens
- 출력: $15/1M tokens
- 종합 = (3 × 2.5 + 15) / 4 = $5.625/1M tokens

USD 가격의 모델의 경우 환율에 따라 변환해야 합니다. 이 환율은 HagiCode 팀이 7.25로 설정했으며, 시장 변동에 따라 업데이트됩니다.

환율은 주식과 같습니다. 아무도 정확히 예측할 수 없습니다. 따라갈 수밖에 없습니다.

### 3. AI 연간 비용

```
일일 AI 비용 = 일일 Token 수요 (M) × 종합 단가 (CNY/1M)
연간 AI 비용 = 일일 AI 비용 × 264个工作日
```

264 = 22일/월 × 12월, 이것은 표준 근무제下的 연간 근무일 수입니다. 왜 365일을 사용하지 않습니까? 주말, 공휴일, 병가 등 요소를 고려해야 하기 때문입니다.

우리는 로봇이 아니므로, 쉴 때는 쉬어야 합니다. AI는 쉴 필요가 없을지 몰라도, 우리는 여전히 자신에게 숨쉴 공간을 남겨두어야 합니다.

### 4. 핵심 혁신: 등가 인력

이것은 전체 평가 시스템의 핵심이며, HagiCode 팀이 가장 통찰력 있는 곳입니다.

```
부담 가능한 워크플로우 부수 = 연간 총 인건비 / AI 연간 비용
부담 가능 비율 = min(부담 가능한 워크플로우 부수, 1)
등가 인력 = 1 + (효율성 배수 - 1) × 부담 가능 비율
```

잠깐만, 이 공식은有点 복잡합니다. 설명해 드리겠습니다:

전통적인 관점은 "효율성이 2배 향상되었습니다"라고 직접 말하지만, 이 공식은 핵심 제약을 고려합니다: **기업의 AI 예산이 지속 가능한가?**

예를 들어: 샤오밍은 효율성이 3배 향상되었지만, 그의 AI 소비 비용은 매년 30만입니다. 회사가 그에게 지급하는 연봉은 20만에 불과합니다. 이 경우 샤오밍은 개인적으로 효율성이 매우 높지만, 실제로는 지속 불가능합니다—회사는 그가 높은 효율성을 유지하기 위해 손해를 보면서까지 지불하지 않을 것입니다.

부담 가능 비율이 바로 이 의미입니다: 기업이 0.5부의 AI 워크플로우만 부담할 수 있다면, 샤오밍의 등가 인력 = 1 + (3-1) × 0.5 = 2명, 3명이 아닙니다.

**핵심 통찰**: 효율성 배수가 얼마나 높느냐가 아니라, 기업이 이 효율성을 유지하는 데 필요한 AI 투자를 부담할 수 있느냐입니다.

사실 이 원리도 간단합니다. 많은 사람들이 이 방향으로 생각하지 않았을 뿐입니다. 우리는 익숙하게 자신의 관점에서 문제를 보고, 거의老板의 관점에서 고려하지 않습니다—그들의 돈도 바람에서 불어온 것이 아닙니다.

### 5. 비용 효율성 비율

```
AI 비용 비율 = AI 연간 비용 / 연간 총 인건비
효율성 증폭 = 효율성 배수 - 1
비용 효율성 비율 = 효율성 증폭 / AI 비용 비율
```

- 비용 효율성 비율 < 1: AI 투자가 불리하며, 효율성 향상이 비용을 상쇄하지 못함
- 비용 효율성 비율 1-2:刚好有利
- 비용 효율성 비율 > 2: 높은 수익, 강력 추천

이 지표는 기업 관리자에게 특히 유용하며, 특정 직무가 AI 도구 투자할 가치가 있는지 빠르게 평가할 수 있습니다.

ROI才是王道, 효율성이 얼마나 향상되었든 비용이 폭발하면 아무도 사지 않을 것입니다.

### 6. 위험 등급

등가 인력에 따라 위험을 나눕니다:

| 등가 인력 | 위험 등급 | 결론 |
|---------|---------|------|
| >= 2.0 | 고위험 | 동료가 동일한 조건을 갖추면, 당신에게 대한 위협이 매우 높음 |
| 1.5 - 2.0 | 경고 | 동료가 이미 명확한 효율성 우위를 형성하기 시작함 |
| < 1.5 | 안전 | 일시적으로 격차를 유지할 수 있음 |

이 표를 보면, 마음속으로大概有个数了吧.ただしあまり焦らないでください毕竟焦虑也解决不了问题不如想想怎么提升自己的效率倍数

## 게임화 설계: 7가지 특수 칭호

| 칭호 ID | 이름 | 획득 조건 |
|--------|------|---------|
| craftsman-spirit | 장인 정신 | 일일 Token = 0 |
| prompt-alchemist | 프롬프트 연금술사 | 일일 Token \<= 20M 이고 효율성 배수 \>= 6 |
| all-in-operator | 올인 오퍼레이터 | 일일 Token \>= 150M 이고 효율성 배수 \>= 3 |
| minimalist-runner | 미니멀리스트 러너 | 일일 Token \<= 5M 이고 효율성 배수 \>= 2 |
| cost-tamer | 비용 테이머 | 비용 효율성 비율 \>= 2.5 이고 AI 占比 \<= 15% |
| danger-oracle | 위험 예언가 | 등가 인력 \>= 2.5 또는 고위험 구역 진입 |
| budget-coordinator | 예산 코디네이터 | 부담 가능한 워크플로우 부수 \>= 8 |

각 칭호 뒤에는 숨겨진 의미가 있습니다:

| 칭호 | 숨겨진 의미 |
|-----|---------|
| 장인 정신 | AI를 사용하지 않아도 잘 살 수 있지만, 독특한 경쟁력이 필요함 |
| 프롬프트 연금술사 | 소량의 Token으로 높은 산출을 달성, 기광형 사용자 |
| 올인 오퍼레이터 | 높은 투입 높은 산출, 고빈도 시나리오에 적합 |
| 미니멀리스트 러너 | 가벼운 AI 사용, 경량 보조 시나리오에 적합 |
| 비용 테이머 | ROI가 매우 높음, 기업이 가장 좋아하는 직원 유형 |
| 위험 예언가 | 이미 또는 곧 고위험 그룹 |
| 예산 코디네이터 | 동시에 여러 AI 워크플로우를 운영할 수 있음 |

## 실제 사용 예시

베이징 개발자라고 가정하겠습니다. 연봉 40만, Claude Sonnet 4.6 사용, 일일 Token 소비 50M, 자체 평가 효율성 향상 3배. 시뮬레이션 입력:

```typescript
const input = {
  annualIncomeCny: 400000,
  cityTier: "tier1",           // 베이징
  modelId: "claude-sonnet-4-6",
  performanceMultiplier: 3.0,
  dailyTokenUsageM: 50,
}

// 계산 과정
// 연간 총 인건비 = 40만 × (1 + 0.4) + 40만/12 ≈ 60.33만
// AI 연간 비용 ≈ 50 × 7.125 × 264 ≈ 9.4만
// 부담 가능한 워크플로우 부수 ≈ 60.33 / 9.4 ≈ 6.4부
// 등가 인력 = 1 + (3 - 1) × 1 = 3명
```

결론: 동료가 동일한 조건을 갖춘다면, 3명의 생산성에 상응하며, 이미 고위험 구역에 있습니다.

AI 사용법이 "불리하다"(비용 효율성 비율 < 1)는 것을 발견하면 다음을 고려할 수 있습니다:

1. **Token 소비 감소**: 더 효율적인 프롬프트 사용, 무효 요청 감소
2. **가성비 모델 선택**: DeepSeek-V3(위안화 가격, 더 저렴함)等
3. **효율성 배수 향상**: 고급 Agent 사용 기술 학습, AI를 진정으로 생산성으로 변환

사실 이 문제들은归根结底균형의 예술입니다. 너무 많이 쓰면 돈이 낭비되고, 너무 적게 쓰면 효과가 없습니다—그刚刚好的 포인트를 찾는 것이 핵심입니다.

## 기술 아키텍처 하이라이트

HagiCode 팀은 이 계산기를 설계할 때 몇 가지 참고할 만한 엔지니어링 결정을 했습니다:

1. **순수 프론트엔드 계산**: 모든 계산이 브라우저에서 완료되며, 백엔드 API에 의존하지 않고 사용자 개인정보를 보호합니다
2. **구성 드라이브**: 모든 공식, 가격, 직무 데이터가 구성 파일에 집중되어 있으며, 미래 업데이트에 핵심 코드 논리 수정이 필요하지 않습니다
3. **다국어 지원**: 중국어와 영어 지원
4. **즉시 피드백**: 사용자가 매개변수를 입력한 후 결과가 실시간으로 업데이트됩니다
5. **상세한 공식 표시**: 각 결과에는 완전한 계산 공식이 포함되어 있어 사용자가 이해하는 데 도움이 됩니다

이러한 설계는 계산기를 유지 관리하고 확장하기 쉽게 만들며, 유사한 데이터 기반 응용 프로그램을 위한 참조 템플릿도 제공합니다.

좋은 아키텍처는 좋은 코드와一样시간의沉淀이 필요합니다. HagiCode 팀은 이方面에서 꽤用心합니다.
