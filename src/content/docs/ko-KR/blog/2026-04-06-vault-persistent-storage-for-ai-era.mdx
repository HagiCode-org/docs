## AI 시대의 프로젝트 복제 방법: Vault, 크로스 프로젝트 지속 저장 시스템

> AI 보조 개발 시대에 AI 어시스턴트가 학습 리소스를 더 잘 이해할 수 있도록 어떻게 돕습니까? HagiCode 프로젝트는 AI가 이해할 수 있는 통합 지식 저장 추상화 계층인 Vault 시스템을 구축하여, 프로젝트 복제를 통한 학습 효율을 크게 높였습니다.

## 배경

AI 시대에 개발자가 새로운 기술과 아키텍처를 학습하는 방식은 근본적으로 변화하고 있습니다. "프로젝트 복제" 즉, 우수한 오픈소스 프로젝트의 코드, 아키텍처, 설계 패턴을 심층적으로 연구하고 학습하는 것이 효율적인 학습 방법이 되었습니다. 책을 읽거나 영상을 시청하는 전통적인 방법과 비교하면, 고�질 오픈소스 프로젝트를 직접 읽고 실행하는 것이 실제 엔지니어링 실무를 훨씬 빠르게 이해할 수 있습니다.

하지만 이 학습 방법에도 꽤 많은 과제가 있습니다.

학습 자료가 너무 흩어져 있습니다. 메모는 Obsidian에 있을 수 있고, 코드 저장소는 여러 폴더에 흩어져 있으며, AI 어시스턴트의 대화 기록은 또 다른 고립된 데이터 섬이가 됩니다. AI에게 프로젝트를 분석해 달라고 하면 코드 스니펫을 수동으로 복사하고 컨텍스트를 정리해야 하는데, 이는 꽤 지루한 일입니다.

더 골치로운 것은 단절된 컨텍스트입니다. AI 어시스턴트는 사용자의 로컬 학습 리소스에 직접 접근할 수 없으므로, 모든 대화에서 배경 정보를 다시 제공해야 합니다. 거기에 복제한 코드 저장소는 빠르게 업데이트되고, 수동 동기화는 오류가 발생하기 쉽고, 지식을 여러 학습 프로젝트 간에 공유하기 어렵습니다.

근본적으로 이 모든 문제는 "데이터 섬"에서 옵니다. AI 어시스턴트가 이해하고 접근할 수 있는 통합 저장 추상화 계층이 있다면 문제가 깔끔하게 해결될 것입니다.

## Vault 시스템 설계 철학

Vault 시스템의 핵심 아이디어는 간단합니다. AI가 이해할 수 있는 통합 지식 저장 추상화 계층을 만드는 것입니다. 구현 관점에서 시스템에는 몇 가지 핵심 특성이 있습니다.

### 다중 유형 지원

시스템은 네 가지 Vault 유형을 지원하며, 각각 다른 사용 시나리오에 해당합니다:

```typescript
// folder: 범용 폴더 유형
export const DEFAULT_VAULT_TYPE = 'folder';

// coderef: 복제 코드 프로젝트 전용 유형
export const CODEREF_VAULT_TYPE = 'coderef';

// obsidian: Obsidian 메모장 소프트웨어 통합
export const OBSIDIAN_VAULT_TYPE = 'obsidian';

// system-managed: 시스템이 자동 관리하는 Vault
export const SYSTEM_MANAGED_VAULT_TYPE = 'system-managed';
```

그 중 **coderef** 유형이 HagiCode에서 가장 일반적으로 사용됩니다. 복제 코드 프로젝트 전용으로 설계되었으며, 표준화된 디렉토리 구조와 AI가 읽을 수 있는 메타데이터 설명을 제공합니다.

### 지속 저장 메커니즘

Vault 레지스트리는 JSON 형식으로 지속 저장되어, 애플리케이션 재시작 후에도 구성이 유지됩니다:

```csharp
public class VaultRegistryStore : IVaultRegistryStore
{
    private readonly string _registryFilePath;

    public VaultRegistryStore(IConfiguration configuration, ILogger<VaultRegistryStore> logger)
    {
        var dataDir = configuration["DataDir"] ?? "./data";
        var absoluteDataDir = Path.IsPathRooted(dataDir)
            ? dataDir
            : Path.GetFullPath(Path.Combine(Directory.GetCurrentDirectory(), dataDir));

        _registryFilePath = Path.Combine(absoluteDataDir, "personal-data", "vaults", "registry.json");
    }
}
```

이 설계의 장점은 간단하고 신�할 수 있다는 것입니다. JSON은 사람이 읽을 수 있어 디버깅과 수동 편집이 더 쉽고, 파일 시스템 저장은 데이터베이스의 복잡성을 피하고 시스템 의존성을 줄입니다. 때로는 가장 간단한 옵션이 진정으로 최고인 경우가 있습니다.

### AI 컨텍스트 통합

가장 중요한 것은 시스템이 Vault 정보를 AI 제안의 컨텍스트에 자동으로 주입할 수 있다는 것입니다:

```typescript
export function buildTargetVaultsText(
  vaults: VaultForText[],
  template: VaultPromptTemplate = DEFAULT_VAULT_PROMPT_TEMPLATE,
): string {
  const readOnlyVaults = vaults.filter((vault) => vault.accessType === 'read');
  const editableVaults = vaults.filter((vault) => vault.accessType === 'write');

  if (readOnlyVaults.length === 0 && editableVaults.length === 0) {
    return '';
  }

  const sections = [
    buildVaultSection(readOnlyVaults, template.reference),
    buildVaultSection(editableVaults, template.editable),
  ].filter(Boolean);

  return `\n\n### ${template.heading}\n\n${sections.join('\n')}`;
}
```

이것은 중요한 기능을 가능하게 합니다. AI 어시스턴트가 사용자가 수동으로 컨텍스트를 제공하지 않아도 사용 가능한 학습 리소스를 자동으로 이해할 수 있습니다. 일종 암묵적 이해라고 할 수 있습니다.

## CodeRef Vault의 표준화된 구조

coderef 유형의 Vault에 대해 HagiCode는 표준화된 디렉토리 구조를 제공합니다:

```
my-coderef-vault/
├── index.yaml          # Vault 메타데이터 설명
├── AGENTS.md           # AI 어시스턴트를 위한 운영 가이드
├── docs/               # 학습 메모 및 문서 저장
└── repos/              # Git 서브모듈을 통해 복제 코드 저장소 관리
```

Vault 생성 시 시스템이 이 구조를 자동으로 초기화합니다:

```csharp
private async Task EnsureCodeRefStructureAsync(
    string vaultName,
    string physicalPath,
    ICollection<VaultBootstrapDiagnosticDto> diagnostics,
    CancellationToken cancellationToken)
{
    Directory.CreateDirectory(physicalPath);

    var indexPath = Path.Combine(physicalPath, CodeRefIndexFileName);
    var docsPath = Path.Combine(physicalPath, CodeRefDocsDirectoryName);
    var reposPath = Path.Combine(physicalPath, CodeRefReposDirectoryName);

    // 표준 디렉토리 구조 생성
    if (!Directory.Exists(docsPath))
    {
        Directory.CreateDirectory(docsPath);
    }

    if (!Directory.Exists(reposPath))
    {
        Directory.CreateDirectory(reposPath);
    }

    // AGENTS.md 가이드 생성
    await EnsureCodeRefAgentsDocumentAsync(physicalPath, cancellationToken);

    // index.yaml 메타데이터 생성
    await WriteCodeRefIndexDocumentAsync(indexPath, mergedDocument, cancellationToken);
}
```

이 구조도 신중하게 설계되었습니다:

- **docs/**에 학습 메모를 저장합니다. 코드에 대한 이해, 아키텍처 분석, 교훈 등을 Markdown으로 기록할 수 있습니다
- **repos/**는 Git 서브모듈을 통해 복제 저장소를 관리하여, 코드를 직접 복사하는 대신 동기화를 유지하고 공간을 절약합니다
- **index.yaml**은 Vault 메타데이터를 포함하여 AI 어시스턴트가 Vault의 목적과 콘텐츠를 빠르게 이해할 수 있습니다
- **AGENTS.md**는 AI 어시스턴트를 위해 작성된 가이드로, Vault의 내용을 처리하는 방법을 설명합니다

이렇게 정리하면 AI가 여러분이 무엇을 생각하고 있는지 조금 더 쉽게 이해할 수 있을 것입니다.

### 시스템 관리 Vault의 자동 초기화

수동으로 생성된 Vault 외에도 HagiCode는 시스템 관리 Vault를 지원합니다:

```csharp
public async Task<IReadOnlyList<VaultRegistryEntry>> EnsureAllSystemManagedVaultsAsync(
    CancellationToken cancellationToken = default)
{
    var definitions = GetAllResolvedDefinitions();
    var entries = new List<VaultRegistryEntry>(definitions.Count);

    foreach (var definition in definitions)
    {
        entries.Add(await EnsureResolvedSystemManagedVaultAsync(definition, cancellationToken));
    }

    return entries;
}
```

시스템이 자동으로 다음 Vault를 생성하고 관리합니다:

- **hagiprojectdata**: 프로젝트 데이터 저장소로 프로젝트 구성 및 상태 저장에 사용됩니다
- **personaldata**: 개인 데이터 저장소로 사용자 환경설정 저장에 사용됩니다
- **hbsprompt**: 프롬프트 템플릿 라이브러리로 자주 사용하는 AI 프롬프트를 관리합니다

이러한 Vault는 시스템 시작 시 자동으로 초기화되므로 사용자가 수동으로 구성할 필요가 없습니다. 어떤 것은 사람이 걱정할 필요 없이 시스템이 처리하는 것이 더 낫습니다.

### 접근 제어 메커니즘

설계의 중요한 부분은 접근 제어입니다. 시스템은 Vault를 두 가지 접근 유형으로 나눕니다:

```typescript
export interface VaultForText {
  id: string;
  name: string;
  type: string;
  physicalPath: string;
  accessType: 'read' | 'write';  // 핵심: 읽기 전용과 편집 가능 구분
}
```

- **reference (읽기 전용)**: AI는 분석 및 이해에만 사용되며 콘텐츠를 수정할 수 없습니다. 참조용 오픈소스 프로젝트, 문서 등의 자료에 적합합니다
- **editable (편집 가능)**: AI가 작업에 필요에 따라 콘텐츠를 수정할 수 있습니다. 사용자의 메모, 초안 등의 자료에 적합합니다

이 구분이 중요합니다. AI에게 어떤 콘텐츠가 "읽기 전용 참조"이고 어떤 콘텐츠가 "안전하게 편집 가능한"인지 알려주어, 의도치 않은 변경의 위험을 줄입니다. 아무도 자신의 노력이 실수로 사라지는 것을 원하지 않습니다.

## 실무: Vault 생성 및 사용

### CodeRef Vault 생성

완전한 프론트엔드 호출 예시입니다:

```typescript
const createCodeRefVault = async () => {
  const response = await VaultService.postApiVaults({
    requestBody: {
      name: "React 학습 Vault",
      type: "coderef",
      physicalPath: "/Users/developer/vaults/react-learning",
      gitUrl: "https://github.com/facebook/react.git"
    }
  });

  // 시스템이 자동으로 다음을 수행합니다:
  // 1. React 저장소를 vault/repos/react에 클론

  // 3. index.yaml 메타데이터 생성
  // 4. AGENTS.md 가이드 파일 생성

  return response;
};
```

이 API 호출은 일련의 작업을 완료합니다. 디렉토리 구조 생성, Git 서브모듈 초기화, 메타데이터 파일 생성 등. 기본 정보만 제공하고 나머지 시스템이 처리하면 됩니다. 실제로 꽤 걱정이 없는 접근 방식입니다.

## 모범 사례 및 주의 사항

Vault 시스템을 사용하면서 몇 가지 교훈을 정리했습니다.

### 경로 안전

시스템은 경로 탐색 공격을 방지하기 위해 경로를 엄격하게 검증합니다:

```csharp
private static string ResolveFilePath(string vaultRoot, string relativePath)
{
    var rootPath = EnsureTrailingSeparator(Path.GetFullPath(vaultRoot));
    var combinedPath = Path.GetFullPath(Path.Combine(rootPath, relativePath));
    if (!combinedPath.StartsWith(rootPath, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode,
            "Vault 파일 경로는 등록된 Vault 루트 내에 있어야 합니다.");
    }
    return combinedPath;
}
```

이것은 중요합니다. 커스텀 Vault 경로를 사용자 정의하는 경우, 허용된 범위 내에 있는지 확인하십시오. 그렇지 않으면 시스템이 작업을 거부합니다. 보안을 아무리 강조해도 지나칠 수 없습니다.

### Git 서브모듈 관리

CodeRef Vault는 Git 서브모듈을 사용하여 직접 코드를 복사하는 대신 권장합니다:

```csharp
private static string BuildCodeRefAgentsContent()
{
    return """
    # CodeRef Vault 가이드

    `repos/` 아래의 저장소는 Git 서브모듈을 통해 관리되어야 합니다.
    직접 복사하는 대신 서브모듈을 사용하면 코드가 업스트림과 동기화되고 디스크 공간도 절약됩니다.

    이 구조를 안정적으로 유지하면 어시스턴트와 도구가 Vault를 빠르게 이해할 수 있습니다.
    """ + Environment.NewLine;
}
```

이것은 여러 장점을 제공합니다. 코드를 업스트림과 동기화하고, 디스크 공간을 절약하며, 코드의 여러 버전을 더 쉽게 관리합니다. 같은 것을 계속 다시 다운로드하고 싶은 사람은 아무도 없을 것입니다.

### 파일 미리보기 제한

성능 문제를 방지하기 위해 시스템은 파일 크기와 유형을 제한합니다:

```csharp
private const int FileEnumerationLimit = 500;
private const int PreviewByteLimit = 256 * 1024;  // 256KB
```

Vault에 대량의 파일이 있거나 매우 큰 파일이 있는 경우 미리보기 성능에 영향을 받을 수 있습니다. 이 경우 파일을 배치로 처리하거나 전용 검색 도구를 사용하는 것을 고려하십시오. 어떤 것이 너무 커지면 처리하기 더 어려워지고, 쉬워지기 때문입니다.

```csharp
}
```

생성에 실패하면 진단 정보를 검사하여 구체적인 원인을 파악할 수 있습니다. 문제가 발생하면 진단 정보를 확인하는 것이 가장 직접적인 해결 방법입니다.

## 요약

통합 저장 추상화 계층을 통해 Vault 시스템은 AI 시대의 프로젝트 복제의 몇 가지 핵심 문제를 해결합니다:

- **집중화된 지식 관리**: 모든 학습 리소스가 한 곳에 모이는 흩어진 대신 집중됩니다
- **자동 AI 컨텍스트 주입**: AI 어시스턴트가 사용 가능한 학습 리소스를 수동 컨텍스트 설정 없이 자동으로 이해할 수 있습니다
- **크로스 프로젝트 지식 재사용**: 지식이 여러 학습 프로젝트 간에 공유 및 재사용될 수 있습니다
- **표준화된 디렉토리 구조**: 일관된 디렉토리 레이아웃이 학습 곡선을 낮춥니다

이 해결책은 HagiCode 프로젝트에서 이미 검증되었습니다. AI 보조 개발 관련 도구를 구축 중이시거나, 비슷한 지식 관리 문제에 직면하셨다면 이 경험이 유용한 참고가 될 수 있습니다.

솔직히 기술 해결책의 가치는 얼마나 복잡한가가 아니라, 실제 문제를 해결하는가에 있습니다. Vault 시스템의 핵심 아이디어는 매우 간단합니다. AI가 이해할 수 있는 통합 지식 저장 계층을 구축하는 것입니다. 그러나 바로 그 간단한 추상화가 저희의 개발 효율을 꽤 높였습니다.

때로는 간단한 접근 방식이 진정으로 최고일 때가 있습니다. 복잡한 것들은 종종 더 많은 함정을 숨기고 있습니다...
