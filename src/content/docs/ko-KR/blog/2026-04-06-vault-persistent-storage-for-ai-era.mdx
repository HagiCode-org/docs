## AI 시대 프로젝트 모방 방법: Vault 교차 프로젝트 영구 저장 시스템

> AI 보조 개발 시대에 AI 어시스턴트가 학습 리소스를 더 잘 이해하도록 하는 방법은? HagiCode 프로젝트는 Vault 시스템을 통해 통합되고 AI가 이해할 수 있는 지식 저장 추상화 계층을 구현하여, 모방 프로젝트의 학습 효율을 크게 향상시켰습니다.

## 배경

AI 시대에 개발자가 새로운 기술과 아키텍처를 학습하는 방식이 깊은 변화를 겪고 있습니다. "모방 프로젝트" — 즉 훌륭한 오픈소스 프로젝트의 코드, 아키텍처, 디자인 패턴을 깊이 연구하고 학습하는 것 — 이미 효율적인 학습 방법이 되었습니다. 전통적인 독서, 비디오 시청에 비해 고품질 오픈소스 프로젝트를 직접 읽고 실행하면 실제 엔지니어링 실무를 더 빨리 이해할 수 있습니다.

하지만 이러한 학습 방식도 많은 과제에 직면해 있습니다.

학습 자료가 너무 분산되어 있습니다. 메모는 Obsidian에 있고, 코드 저장소는 여러 폴더에 흩어져 있으며, AI 어시스턴트의 대화 기록은 또 다른 독립된 데이터 사일로입니다. AI에게 프로젝트 분석을 요청할 때 코드 조각을 수동으로 복사하고 컨텍스트를 정리해야 하니, 이 과정이 상당히 번거롭습니다.

사실 더 번거로운 것은 컨텍스트 단절입니다. AI 어시스턴트는 로컬 학습 리소스에 직접 액세스할 수 없어 매번 대화할 때마다 배경 정보를 다시 제공해야 합니다. 또한 모방하는 코드 저장소는 빠르게 업데이트되어 수동 동기화는 오류가 발생하기 쉽고, 여러 학습 프로젝트 간에 지식을 공유하기도 어렵습니다.

이러한 문제들은 본질적으로 "데이터 사일로"로 인한 것입니다. 통합된 저장 추상화 계층이 있어 AI 어시스턴트가 모든 학습 리소스를 이해하고 액세스할 수 있다면 문제는 해결됩니다.

이 솔루션은 HagiCode에서 이미 실제로 검증되었습니다. 만약 당신도 유사한 지식 관리 문제에 직면해 있다면 이러한 경험이 영감을 줄 수 있기를 바랍니다. 결국 어떤 함정을 겪었다면, 후배들을 위해 무언가 남겨야 하지 않겠습니까.

## Vault 시스템 설계理念

Vault 시스템의 핵심 아이디어는 매우 간단합니다. 통합되고 AI가 이해할 수 있는 지식 저장 추상화 계층을 만드는 것입니다. 구현 관점에서 시스템은 몇 가지 핵심 특징을 가집니다.

### 다중 유형 지원

시스템은 네 가지 vault 유형을 지원하며 각각 다른 사용 시나리오에 해당합니다.

```typescript
// folder: 범용 폴더 유형
export const DEFAULT_VAULT_TYPE = 'folder';

// coderef: 모방 코드 프로젝트 전용 유형
export const CODEREF_VAULT_TYPE = 'coderef';

// obsidian: Obsidian 메모 소프트웨어 통합
export const OBSIDIAN_VAULT_TYPE = 'obsidian';

// system-managed: 시스템 자동 관리 vault
export const SYSTEM_MANAGED_VAULT_TYPE = 'system-managed';
```

그중 **coderef** 유형이 HagiCode에서 가장 많이 사용됩니다. 이는 모방 코드 프로젝트를 위해 특별히 설계되어 표준화된 디렉터리 구조와 AI가 읽을 수 있는 메타데이터 설명을 제공합니다.

### 영구 저장 메커니즘

Vault 레지스트리는 JSON 형식으로 영구 저장되어 애플리케이션 재시작 후에도 구성을 사용할 수 있습니다.

```csharp
public class VaultRegistryStore : IVaultRegistryStore
{
    private readonly string _registryFilePath;

    public VaultRegistryStore(IConfiguration configuration, ILogger<VaultRegistryStore> logger)
    {
        var dataDir = configuration["DataDir"] ?? "./data";
        var absoluteDataDir = Path.IsPathRooted(dataDir)
            ? dataDir
            : Path.GetFullPath(Path.Combine(Directory.GetCurrentDirectory(), dataDir));

        _registryFilePath = Path.Combine(absoluteDataDir, "personal-data", "vaults", "registry.json");
    }
}
```

이 설계의 장점은 간단하고 신뢰할 수 있다는 것입니다. JSON 형식은 사람이 읽을 수 있어 디버깅과 수동 수정에 편리합니다. 파일 시스템 저장은 데이터베이스의 복잡성을 피하고 시스템 의존성을 낮춥니다. 결국 간단한 것이 때로는 최고입니다.

### AI 컨텍스트 통합

가장 중요한 것은 시스템이 vault 정보를 AI 제안의 컨텍스트에 자동으로 주입할 수 있다는 것입니다.

```typescript
export function buildTargetVaultsText(
  vaults: VaultForText[],
  template: VaultPromptTemplate = DEFAULT_VAULT_PROMPT_TEMPLATE,
): string {
  const readOnlyVaults = vaults.filter((vault) => vault.accessType === 'read');
  const editableVaults = vaults.filter((vault) => vault.accessType === 'write');

  if (readOnlyVaults.length === 0 && editableVaults.length === 0) {
    return '';
  }

  const sections = [
    buildVaultSection(readOnlyVaults, template.reference),
    buildVaultSection(editableVaults, template.editable),
  ].filter(Boolean);

  return `\n\n### ${template.heading}\n\n${sections.join('\n')}`;
}
```

이렇게 하면 중요한 기능이 구현됩니다. AI 어시스턴트는 사용자가 수동으로 컨텍스트를 제공할 필요 없이 사용 가능한 학습 리소스를 자동으로 이해할 수 있습니다. 이것도 일종의 묵시적인 이해라고 할 수 있겠네요.

## CodeRef Vault의 표준화된 구조

coderef 유형의 vault에 대해 HagiCode는 표준화된 디렉터리 구조를 제공합니다.

```
my-coderef-vault/
├── index.yaml          # vault 메타데이터 설명
├── AGENTS.md           # AI 어시스턴트 운영 가이드
├── docs/               # 학습 메모 및 문서 보관
└── repos/              # Git 하위 모듈을 통한 모방 코드 저장소 관리
```

vault를 생성할 때 시스템은 이 구조를 자동으로 초기화합니다.

```csharp
private async Task EnsureCodeRefStructureAsync(
    string vaultName,
    string physicalPath,
    ICollection<VaultBootstrapDiagnosticDto> diagnostics,
    CancellationToken cancellationToken)
{
    Directory.CreateDirectory(physicalPath);

    var indexPath = Path.Combine(physicalPath, CodeRefIndexFileName);
    var docsPath = Path.Combine(physicalPath, CodeRefDocsDirectoryName);
    var reposPath = Path.Combine(physicalPath, CodeRefReposDirectoryName);

    // 표준 디렉터리 구조 생성
    if (!Directory.Exists(docsPath))
    {
        Directory.CreateDirectory(docsPath);
    }

    if (!Directory.Exists(reposPath))
    {
        Directory.CreateDirectory(reposPath);
    }

    // AGENTS.md 가이드 생성
    await EnsureCodeRefAgentsDocumentAsync(physicalPath, cancellationToken);

    // index.yaml 메타데이터 생성
    await WriteCodeRefIndexDocumentAsync(indexPath, mergedDocument, cancellationToken);
}
```

이 구조의 설계에도 고려사항이 있습니다.

- **docs/** 디렉터리에는 학습 메모를 보관하며 Markdown 형식으로 코드 이해도, 아키텍처 분석, 시행착오 경험 등을 기록할 수 있습니다
- **repos/** 디렉터리는 Git 하위 모듈을 통해 모방 저장소를 관리하며 코드를 직접 복사하지 않습니다. 이렇게 하면 코드 동기화를 유지하면서 공간을 절약할 수 있습니다
- **index.yaml**은 vault의 메타데이터를 포함하며 AI 어시스턴트가 이 vault의 용도와 내용을 빠르게 이해할 수 있도록 합니다
- **AGENTS.md**는 AI 어시스턴트를 위해 작성된 가이드로 이 vault의 내용을 처리하는 방법을 설명합니다

이렇게 정리하면 AI도 아이디어를 더 쉽게 이해할 수 있을지도 모르겠네요.

## 시스템 관리 자동 초기화

수동으로 vault를 생성하는 것 외에도 HagiCode는 시스템 자동 관리 vault를 지원합니다.

```csharp
public async Task<IReadOnlyList<VaultRegistryEntry>> EnsureAllSystemManagedVaultsAsync(
    CancellationToken cancellationToken = default)
{
    var definitions = GetAllResolvedDefinitions();
    var entries = new List<VaultRegistryEntry>(definitions.Count);

    foreach (var definition in definitions)
    {
        entries.Add(await EnsureResolvedSystemManagedVaultAsync(definition, cancellationToken));
    }

    return entries;
}
```

시스템은 자동으로 다음 vault를 생성하고 관리합니다.

- **hagiprojectdata**: 프로젝트 데이터 저장, 프로젝트 구성 및 상태 저장에 사용
- **personaldata**: 개인 데이터 저장, 사용자 기본 설정 저장에 사용
- **hbsprompt**: 프롬프트 템플릿 라이브러리, 일반적인 AI 프롬프트 관리에 사용

이러한 vault는 시스템 시작 시 자동으로 초기화되며 사용자가 수동으로 구성할 필요가 없습니다. 결국 어떤 일은 시스템에 맡기면 되지, 인간이 왜 신경 써야 합니까.

## 액세스 제어 메커니즘

중요한 설계 중 하나는 액세스 제어입니다. 시스템은 vault를 두 가지 액세스 유형으로 나눕니다.

```typescript
export interface VaultForText {
  id: string;
  name: string;
  type: string;
  physicalPath: string;
  accessType: 'read' | 'write';  // 핵심: 읽기 전용과 편집 가능 구분
}
```

- **reference(읽기 전용)**: AI는 분석과 이해에만 사용하며 내용을 수정할 수 없습니다. 참조용 오픈소스 프로젝트, 문서 등에 적용됩니다
- **editable(편집 가능)**: AI는 작업 필요에 따라 내용을 수정할 수 있습니다. 메모, 초안 등에 적용됩니다

이러한 구분은 중요합니다. AI가 어떤 내용이 "읽기 전용 참조"이고 무엇을 "수정할 수 있는지" 알게 하여 오작업 위험을 피합니다. 결국 누구도 자신의心血이 무의식中 수정되는 것을 원하지 않을 겁니다.

## 실전: Vault 생성 및 사용

원리를 살펴보았으니 실제로 사용하는 방법을 알아보겠습니다.

### CodeRef Vault 생성

다음은 완전한 프런트엔드 호출 예제입니다.

```typescript
const createCodeRefVault = async () => {
  const response = await VaultService.postApiVaults({
    requestBody: {
      name: "React Learning Vault",
      type: "coderef",
      physicalPath: "/Users/developer/vaults/react-learning",
      gitUrl: "https://github.com/facebook/react.git"
    }
  });

  // 시스템은 자동으로 다음을 수행합니다:
  // 1. React 저장소를 vault/repos/react에 복제

  // 3. index.yaml 메타데이터 생성
  // 4. AGENTS.md 가이드 파일 생성

```

이 API 호출은 일련의 작업을 완료합니다. 디렉터리 구조 생성, Git 하위 모듈 초기화, 메타데이터 파일 생성 등입니다. 기본 정보만 제공하면 나머지는 시스템이 처리합니다. 사실 이렇게도 꽤 마음 편하네요.

## 모범 사례와 주의사항

Vault 시스템을 사용하는 과정에서 우리는 몇 가지 경험과 교훈을 정리했습니다.

### 경로 보안

시스템은 경로 순회 공격을 방지하기 위해 경로를 엄격하게 검증합니다.

```csharp
private static string ResolveFilePath(string vaultRoot, string relativePath)
{
    var rootPath = EnsureTrailingSeparator(Path.GetFullPath(vaultRoot));
    var combinedPath = Path.GetFullPath(Path.Combine(rootPath, relativePath));
    if (!combinedPath.StartsWith(rootPath, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode,
            "Vault file paths must stay inside the registered vault root.");
    }
    return combinedPath;
}
```

이 점은 중요합니다. 사용자 지정 vault 경로를 지정할 때 경로가 허용된 범위 내에 있는지 확인해야 하며 그렇지 않으면 시스템이 작업을 거부합니다. 보안은 아무리 강조해도 지나치지 않습니다.

### Git 하위 모듈 관리

CodeRef vault는 코드를 직접 복사하는 대신 Git 하위 모듈 사용을 권장합니다.

```csharp
private static string BuildCodeRefAgentsContent()
{
    return """
    # CodeRef Vault Guide

    Repositories under `repos/` should be maintained through Git submodules
    rather than copied directly into the vault root.

    Keep this structure stable so assistants and tools can understand the vault quickly.
    """ + Environment.NewLine;
}
```

이렇게 하면 여러 가지 장점이 있습니다. 업스트림과 코드 동기화 유지, 디스크 공간 절약, 여러 버전의 코드를 쉽게 관리할 수 있습니다. 결국 누가 똑같은 것을 반복해서 다운로드하고 싶어 하겠습니까.

### 파일 미리보기 제한

성능 문제를 방지하기 위해 시스템은 파일 크기와 유형을 제한합니다.

```csharp
private const int FileEnumerationLimit = 500;
private const int PreviewByteLimit = 256 * 1024;  // 256KB
```

vault에 대용량 파일이나 초대형 파일이 포함된 경우 미리보기 기능의 성능에 영향을 미칠 수 있습니다. 이러한 경우 일괄 처리나 전용 검색 도구 사용을 고려할 수 있습니다. 결국 너무 큰 것은 처리하기 어렵습니다.

### 진단 정보

vault를 생성할 때 진단 정보를 반환하여 디버깅을 돕습니다.

```csharp
List<VaultBootstrapDiagnosticDto> bootstrapDiagnostics = [];

if (IsCodeRefVaultType(normalizedType))
{
    bootstrapDiagnostics = await EnsureCodeRefBootstrapAsync(
```

생성에 실패하면 진단 정보를 확인하여 구체적인 원인을 알 수 있습니다. 오류가 발생하면 진단 정보를 확인하는 것도 문제 해결 방법입니다.

## 요약

Vault 시스템은 통합된 저장 추상화 계층을 통해 AI 시대 모방 프로젝트의 핵심 문제를 해결합니다.

- **지식 중앙 관리**: 모든 학습 리소스를 한 곳에集中 관리하여 더 이상 흩어지지 않습니다
- **AI 컨텍스트 자동 주입**: AI 어시스턴트는 사용자가 수동으로 컨텍스트를 제공할 필요 없이 사용 가능한 학습 리소스를 자동으로 이해할 수 있습니다
- **교차 프로젝트 지식 재사용**: 여러 학습 프로젝트 간에 지식을 공유하고 재사용할 수 있습니다
- **표준화된 디렉터리 구조**: 일관된 디렉터리 구조를 제공하여 학습 비용을 낮춥니다

이 솔루션은 HagiCode 프로젝트에서 이미 실제로 검증되었습니다. AI 보조 개발 관련 도구를 개발 중이거나 유사한 지식 관리 문제에 직면한 경우 이러한 경험이 참고가 되기를 바랍니다.

사실 기술 솔루션의 가치는 얼마나 복잡한지가 아니라 실제 문제를 해결할 수 있는지에 있습니다. Vault 시스템의 핵심 아이디어는 매우 간단합니다. 통합되고 AI가 이해할 수 있는 지식 저장 계층을 만드는 것입니다. 하지만 바로 이 간단한 추상화가 우리의 개발 효율을 크게 향상시켰습니다.

때로는 간단한 것이 최고입니다. 결국 복잡한 것은 종종 더 많은 함정을 숨기고 있습니다......
