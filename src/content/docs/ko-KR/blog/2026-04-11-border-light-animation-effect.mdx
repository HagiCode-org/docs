## 테두리 조명 회전 애니메이션 효과 구현 가이드

> 사용자가 한눈에 알아차리는 중요한 요소를 순수 CSS로 어떻게 구현하는 걸까? 사실 어렵지 않아요, 약간 돌아가는 방법일 뿐이죠. 이 글에서 처음부터 테두리 조명 회전 애니메이션을 구현해 드리고, HagiCode 프로젝트에서 겪었던 시행착오도 함께 이야기해 볼게요.

## 배경

프론트엔드를 하는 분들은 다들 이런 경험이 있을 거예요: 제품 매니저가 다가와서 "이 요구사항은 간단해"라는 표정을 지으며 — "이 실행 중인 작업에, 사용자가 한눈에 볼 수 있는 효과를 추가할 수 있어?"

그러래요, 테두리 색상을 바꾸면 된다고 했더니, 상대방은 고개를 저으며 "넌 몰라"라는 눈빛을 보내요: "충분하지 않아. 조명이 테두리를 따라 도는 효과가 필요해, SF 영화처럼 말이야."

그때면 고민하게 될 거예요: 이걸 어떻게 구현하지? Canvas? SVG? 아니면 CSS로 가능? 그래도 자신이 못한다고 인정하고 싶은 사람은 없으니까요.

사실 테두리 조명 회전 애니메이션은 현대 웹 애플리케이션에서 특히 흔하게 사용되며, 주로 다음과 같은 상황에서 활용됩니다:

- **상태 표시**: 진행 중인 작업이나 활성 항목 표시
- **시각적 초점**: 중요한 콘텐츠 영역 강조
- **브랜드 강화**: 기술감과 현대적인 느낌의 시각적 경험 조성
- **휴일 테마**: 특별한 휴일과 함께 축하 분위기 조성

우리가 [HagiCode](https://hagicode.com)를 만들 때 이 요구사항을 만났어요 — 사용자가 어떤 세션이 실행 중이고, 어떤 제안이 처리 중인지 한눈에 알아야 했죠. 여러 가지 방법을 시도했어요, 어떤 길은 조금 더 쉽고, 어떤 길은 조금 더曲折했지만, 결국 상당히 성숙한 구현 아이디어를 정리하게 되었습니다.

## 핵심 구현 아이디어

HagiCode 코드를 분석한 결과, 우리는 몇 가지 핵심 구현 패턴을 요약했습니다. 각각은 적용 가능한 상황이 있거나, 말하자면 각각은 존재 의미가 있을 뿐이죠.

### 1. Conic Gradient 회전 광효과 (가장 일반적)

가장 고전적인 테두리 조명 회전 구현 방식입니다. 핵심 아이디어는 CSS의 `conic-gradient`를 사용하여 원뿔형 그라디언트를 만든 다음 회전시키는 것입니다. 밤에 가로등이 계속 돌아가는 것과 같죠.

**핵심 요소:**
- `::before` 가상 요소를 사용하여 광효과 레이어 만들기
- `conic-gradient`를 사용하여 그라디언트 색상 분포 정의
- `::after` 가상 요소를 사용하여 중심 영역 마스크 (선택 사항)
- `@keyframes`를 사용하여 회전 애니메이션 구현

### 2. 측면 발광 라인

리스트 항목의 상태 표시에 적합합니다. 요소의 한쪽에 발광하는 얇은 선을 만들면 됩니다. 전체 테두리를 움직일 필요가 없어요. 때로는 조금의 빛만으로도 충분하고, 전체 세상을 비출 필요는 없으니까요.

**핵심 요소:**
- 절대 위치 지정된 얇은 선 요소
- `box-shadow`를 사용하여 발광 효과 만들기
- `scale`과 `opacity`를 사용하여 호흡 애니메이션 구현

### 3. Box-Shadow 발광 배경

회전 효과가 필요하지 않고 부드러운 배경 광효과만 원한다면 여러 층의 `box-shadow`를 겹치면 충분합니다. 어떤 것들은 간단한 것이 더 낫습니다.

### 4. 접근성 지원

이건 쉽게 간과되지만 특히 중요합니다. 모든 애니메이션은 `prefers-reduced-motion` 미디어 쿼리를 고려해야 하며, 애니메이션을 좋아하지 않는 사용자에게 정적 대안을 제공해야 합니다. 모든 사람이 움직이는 것을 좋아하는 것은 아니니, 각자의 선택을 존중하는 것이 맞습니다.

## 구현 방안

### 방안 1: Conic Gradient 회전 테두리 (권장)

가장 완전한 회전 조명 효과 구현이며, HagiCode에서 가장 많이 사용하는 방안입니다. 한 가지가 잘 작동하면 왜 바꿔요?

```css
/* 부모 컨테이너 */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* 회전하는 광효과 레이어 */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* 마스크 레이어 (선택 사항, 빈 테두리 효과 만들기용) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

이 방안의 원리는 꽤 간단합니다: 부모 컨테이너보다 큰 가상 요소를 만들고, 그 위에 원뿔형 그라디언트를 그린 다음 계속 회전시킵니다. 부모 컨테이너는 `overflow: hidden`을 설정하므로 테두리 부분의 빛만 회전하는 것을 볼 수 있습니다. 마치 창문에서 밖의 가로등을 보는 것과 같아요, 지나가는 그 작은 구간만 볼 수 있는 뿐이죠.

### 방안 2: 간소화된 회전 광 테두리

그렇게 복잡한 효과가 필요하지 않다면, HagiCode에는 더 가벼운 유틸리티 클래스 구현이 있습니다. 간단한 것이 때로는 더 낫습니다.

```css
/* 회전 광 테두리 유틸리티 클래스 */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* 접근성 지원 */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

여기서 `will-change: transform`에 주의하세요. 이것은 브라우저에게 "이 요소는 계속 바뀔 거야"라고 알려주는 것이고, 브라우저는 미리 최적화를 하므로 애니메이션이 더 부드러워집니다. 미리 준비하는 것이 임시변통보다 낫습니다.

### 방안 3: 측면 발광 라인

리스트 항목의 상태 표시에 특히 적합합니다. HagiCode의 세션 목록이 이 방안을 사용합니다. 얇은 선 하나지만 많은 항목에서 돋보일 수 있으니, 이것도 일종의 인생 철학 아닐까요?

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

여기서는 `isolation: isolate`를 사용하여 새로운 쌓임 컨텍스트를 만들고, `z-index`로 각 레이어의 표시 순서를 제어합니다. `pointer-events: none`도 중요합니다. 그렇지 않으면 가상 요소가 사용자의 클릭 작업을 막을 거니까요. 어떤 것들은 보기 좋지만 방해가 되어서는 안 됩니다.

### 방안 4: React 컴포넌트 캡슐화

프로젝트에서 React를 사용한다면, 특히 접근성 부분을 처리하기 위해 컴포넌트로 캡슐화할 수 있습니다. 코드를 한 번 쓰고 여러 번 사용하는 것이 우리가 원하는 것이니까요.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

해당 CSS 모듈:

```css
/* GlowBorder.module.css */

/* 애니메이션 버전 */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* 정적 버전 (접근성) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

`framer-motion`의 `useReducedMotion` hook은 사용자의 시스템 기본 설정을 자동으로 감지합니다. 사용자가 "동작 감소"를 켜면 true를 반환하고, 이때 정적 버전을 표시합니다. 사용자의 선택을 존중하는 것이 강제로 표시하는 것보다 더 중요합니다.

## 실제 경험 공유

아래는 HagiCode를 만들 때 겪었던 함정과 요약한 경험입니다. 사실 잡담일 뿐이지만, 후에 오시는 분들께 도움이 되기를 바랍니다.

### 1. 테마 변수 시스템

CSS 변수를 사용하여 다중 테마 지원을 구현하는 것이 특히 편리합니다. 매번 테마를 바꿀 때마다 코드를 수정하고 싶은 사람은 없으니까요.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* 사용 */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

이렇게 하면 테마를 바꿀 때 `html` 태그의 클래스만 수정하면 모든 애니메이션 색상이 자동으로 업데이트됩니다. 한 세트의 코드, 두 가지 스타일. 이것이 우리가 추구하는 것이 아닐까요?

### 2. 성능 최적화

**`will-change`를 사용하여 브라우저 최적화 제안:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

미리 브라우저에 알려주면 최적화를 도와줍니다. 인생의 많은 것들처럼 미리 준비하는 것이 항상 좋습니다.

**큰 요소에 복잡한 box-shadow 사용 피하기:**

```css
/* 나쁨 - 큰 요소에 흐림 그림자 사용 */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* 더 좋음 - 가상 요소를 사용하여 발광 영역 제한 */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

우리는 HagiCode에서 테스트해봤는데, 큰 카드에 직접 흐림 그림자를 추가하면 스크롤 프레임 속도가 30fps 이하로 떨어지고, 가상 요소로 변경하면 60fps로 안정적입니다. 이런 경험의 차이는 사용자가 느낄 수 있습니다.

### 3. 접근성

이것은 정말 생략할 수 없습니다. 어떤 사용자들은 애니메이션이 현기증 나거나 시끄럽다고 느낄 수 있으므로, 그들의 선택을 존중하는 것이 제품을 만드는 기본 소양입니다. 아름다운 것은 강요할 필요가 없으니까요.

**CSS 미디어 쿼리:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* 정적 대안 제공 */
    opacity: 1;
  }
}
```

**React에서 사용자 기본 설정 감지:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. 강도 레벨 제어

HagiCode의 Token 처리량 표시기는 실시간 처리량에 따라 다른 색상의 조명을 표시합니다. 이것은 동적으로 구현됩니다. 다른 상태는 다른 표현 방식이 있어야 하니까요.

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. 주의사항

일부 세부 사항은 주의해야 합니다. 그렇지 않으면 함정에 빠진 후에야 알게 됩니다.

| 주의사항 | 설명 |
|---------|------|
| **z-index 관리** | 광효과 레이어는 적절한 z-index를 설정해야 하며, 콘텐츠 상호작용에 영향을 주지 않아야 합니다 |
| **pointer-events** | 광효과 가상 요소는 `pointer-events: none`을 설정해야 합니다 |
| **경계 오버플로우** | 부모 컨테이너는 `overflow: hidden`을 설정하거나 가상 요소 크기를 조정해야 합니다 |
| **성능 영향** | 복잡한 애니메이션은 모바일 장치에서 성능에 영향을 줄 수 있으므로 테스트가 필요합니다 |
| **다크 모드** | 발광 색상이 어두운 배경에서 명확하게 보이는지 확인하세요 |
| **테마 전환** | CSS 변수를 사용하여 테마 전환 시 애니메이션 색상이 올바르게 업데이트되도록 하세요 |

### 6. 디버깅 팁

가상 요소는 개발자 도구에서 찾기 어려울 때가 있으므로 임시로 테두리를 추가하여 위치를 볼 수 있습니다.

```css
/* 임시로 가상 요소 경계 표시, 디버깅용 */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

위치를 조정한 후에는 이 줄을 주석 처리하거나 삭제하는 것을 잊지 마세요. 그렇지 않으면 프로덕션 환경에서 곤란해질 거예요. 어떤 것들은 개발 환경에 두는 것이 좋습니다.

## 요약

테두리 조명 회전 애니메이션은 어렵지 않지만 간단하지도 않습니다. 핵심은 `conic-gradient`와 회전이지만, 성능이 좋고 유지 관리가 가능하며 접근성이 친화적이려면 주의해야 할 세부 사항이 많습니다.

HagiCode는 이 부분에서 많은 함정을 밟았고 일부 최적의 방법을 요약했습니다. 사실 프로젝트를 하는 것이 그런 것입니다. 반복해서 시행착오를 하고 반복해서 개선합니다. 비슷한 요구사항을 하신다면 이 글이 도움이 되어 detour를 줄이기를 바랍니다.

어떤 것들은 직접 실천해야 깊이를 알 수 있습니다.
