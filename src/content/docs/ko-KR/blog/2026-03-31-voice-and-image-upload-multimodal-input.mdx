## 타이핑보다 말하기, 말하기보다 스크린샷—AI 코드 어시스턴트의 멀티모달 입력 실천

> 사실 코드를 작성하는 일은, 타이핑이 아무리 빨라도 한계가 있습니다. 때로는 한 마디면 될 일을 반나절 동안 키보드를 두드려야 하고, 때로는 한 장의 이미지로 명확해질 수 있는 것을 여러 텍스트로 설명해야 합니다. 이 글에서는 [HagiCode](https://hagicode.com)를 만들면서 겪었던 일들에 대해 이야기합니다—음성 인식이든, 이미지 업로드든, 어쨌든 AI 코드 어시스턴트를 더 쓰기 편하게 만들고 싶었을 뿐입니다.

## 배경

HagiCode를 만들 때 우리는 하나의 문제를 발견했습니다—또는, 사용자들이 많이 사용하면서 자연스럽게 드러나는 문제라고 할까요: 타이핑만으로는 때로 꽤 피곤합니다.

생각해 보세요, 사용자와 Agent의 상호작용, 이건 핵심 시나리오입니다. 하지만 매번 키보드 앞에 앉아서 두드려야 한다니, 효율성이 그리 높지 않습니다:

1. **타이핑이 너무 느림**: 어떤 복잡한 문제들, 예를 들어 에러 메시지나 인터페이스 관련 일들은 타이핑으로 설명하려면 반 분 정도 걸리는데, 말로는 10초면 끝날 수 있습니다. 이 시간 차가 꽤 괴롭습니다.

2. **이미지가 더 직관적**: 때로는 인터페이스에서 오류가 났거나 디자인 초안과 비교하고 싶거나 코드 구조를 보여주고 싶을 때가 있습니다... "한 장의 이미지가 천 마디 말보다 낫다"는 말이 오래됐지만, 여전히 유효합니다. AI가 직접 "보도록" 하는 것이 반나절 설명하는 것보다 훨씬 명확합니다.

3. **상호작용은 자연스러워야 함**: 요즘 AI 어시스턴트는 텍스트, 음성, 이미지 같은 방식들을 지원해야 하지 않을까요? 사용자가 무엇을 사용하고 싶은지 선택하게 하는 것이 자연스러운 일이 아닐까요?

그래서 우리는 HagiCode에 음성 인식과 이미지 업로드 기능을 추가해서 Agent 작업을 더 편하게 만들어 보자고 생각했습니다. 어쨌든 사용자가 몇 자라도 덜 타이핑하게 하는 것도 좋은 일입니다.

## 분석

### 음성 인식의 기술적 도전

음성 인식 기능을 만들 때 우리는 꽤 까다로운 문제에 직면했습니다: **브라우저의 WebSocket API는 사용자 정의 HTTP header를 지원하지 않습니다**.

우리가 선택한 음성 인식 서비스는 바이트댄스의 더우바우(Doubao) 음성 인식 API입니다. 이 API는偏偏 HTTP header를 통해 인증 정보를 전달해야 합니다, 예를 들어 `accessToken`, `secretKey` 같은 것들입니다. 이제 좋습니다, 기술적 모순이 발생했습니다:

```javascript
// 브라우저 WebSocket API는 다음 방식을 지원하지 않습니다
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // 지원하지 않음
  }
});
```

우리 앞에 놓인 방법은 대략 두 가지가 있습니다:

1. **URL 쿼리 파라미터 방안**: 인증 정보를 URL에 넣는 것
   - 장점은 구현하기 간단합니다
   - 단점은 자격 증명이 프론트엔드에 노출되어 보안성이 좋지 않습니다; 또한 일부 API는 강제로 header 검증을 요구합니다

2. **백엔드 프록시 방안**: 백엔드에서 WebSocket 프록시를 구현하는 것
   - 장점은 자격 증명이 백엔드에 안전하게 저장됩니다; API 요구 사항을 완전히 준수합니다
   - 단점은 구현이 조금 더 복잡합니다

마지막으로 우리는 **백엔드 프록시 방안**을 선택했습니다. 어쨌든 보안은 타협할 수 없는 최소 요구사항입니다—이 점에서 아무도 속일 수 없습니다.

### 이미지 업로드의 기능 요구사항

이미지 업로드 기능에 대한 우리의 요구사항도 꽤 간단합니다:

1. **다양한 업로드 방식**: 클릭하여 파일 선택, 드래그 앤 드롭 업로드, 클립보드 붙여넣기, 이건 기본적으로 있어야 하지 않을까요?
2. **파일 검증**: 타입 제한(PNG, JPG, WebP, GIF), 크기 제한(5-10MB), 이건 기본 동작입니다
3. **사용자 경험**: 업로드 진행률, 미리보기, 오류 메시지, 무슨 일이 일어났는지 알려줘야 합니다
4. **보안**: 서버 측 검증, 악성 파일 업로드 방지, 이건 중요한 일입니다

## 해결 방안

### 음성 인식: WebSocket 프록시 아키텍처

우리는 3계층 아키텍처의 음성 인식 방안을 설계했습니다, 어떻게 말하면 좋을까요, 한 가지 길을 찾은 것 같습니다:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**핵심 구성요소 구현**:

1. **프론트엔드 AudioWorklet 프로세서**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // 16kHz로 리샘플링 (더우바우 API 요구사항)
    const samples = this.resampleAudio(input, 48000, 16000);

    // 샘플을 500ms 청크로 누적
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // 16-bit PCM으로 변환하여 전송
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **백엔드 WebSocket 핸들러** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{

    }
}
```

### 이미지 업로드: 다중 방식 업로드 구성요소

우리는 기능이 완비된 이미지 업로드 구성요소를 만들었습니다, 세 가지 업로드 방식을 모두 지원합니다, 어떻게 말하면 좋을까요, 사용자가 자주 사용하는 시나리오를 대부분 커버한 것 같습니다.

**핵심 특징**:

1. **세 가지 업로드 방식**:

```tsx
// 클릭 업로드
const handleClick = () => fileInputRef.current?.click();

// 드래그 앤 드롭 업로드
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// 클립보드 붙여넣기
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **프론트엔드 검증**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **백엔드 업로드 처리** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // 검증
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // 파일 저장
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

   - 인식 결과가 커서 위치에 자동으로 삽입됩니다

### 이미지 업로드 사용 방법

1. **업로드 방식**:
   - 업로드 버튼 클릭하여 파일 선택

3. **크기 제한**: 기본 5MB(구성 가능)

1. **음성 인식**:
   - 마이크 권한이 필요합니다

## 요약
