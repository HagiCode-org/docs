## HagiCode를 위한 12가지 최고의 부팅 경험 디자인: 미니멀에서 사이버펑크까지

> React 19 앱 다운로드와 Hydration 사이의 짧은 틈은 사용자가 브랜드 개성을 인지하는 골든 타임입니다. 이 글에서는 HagiCode 프로젝트에서 HTML/CSS/JS를 기반으로 구축한 완전한 부팅 스타일 시스템을 공유합니다.


## 배경

HagiCode는 ASP.NET Core 10과 React 19 (Vite)를 기반으로 하는 현대적인 앱으로, 프론트엔드와 백엔드 분리 배포 아키텍처를 채택했습니다. 프론트엔드 빌드产物는 백엔드의 `wwwroot/` 디렉터리에 배치되어 ASP.NET Core에 의해 호스팅됩니다.

그러나 이 아키텍처는 전형적인 사용자 경험 문제를 가져옵니다. 사용자가 웹페이지에 접속하면 브라우저는 먼저 HTML을 로드한 다음 거대한 JS 번들을 다운로드하고 마지막으로 React가 Hydration(수화)을 수행해야 합니다. 이 수백 밀리초에서 수 초에 이르는 "진공 상태" 동안 사용자는 빈 화면이나 생기 없는 정적 페이지를 마주하게 됩니다.

이 간격을 채우고 HagiCode의 브랜드 개성을 주입하기 위해 `index.html` 인라인 코드를 완전히 기반으로 하는 부팅 스타일 시스템을 설계해야 했습니다.

## HagiCode에 대해

이 글에서 공유하는 부팅 화면 설계 방안은 [HagiCode](https://github.com/HagiCode-org/site) 프로젝트에서의 실천 경험에서 나왔습니다. AI 코드 어시스턴트로서 HagiCode는 코드 생성 효율성뿐만 아니라 개발자의 시각 경험도 동등하게 중요하게 생각합니다. 이 부팅 시스템은 바로 최고의 프론트엔드 성능을 추구하는 과정에서 탄생한产物입니다.

## 핵심 도전과 아키텍처 설계

디자인에 착수하기 전에 먼저 기술적 제약을 명확히 해야 합니다. `index.html`에서 인라인으로 구현해야 하므로 React 자체 번들 외에는 외부 CSS나 JS 파일을 로드할 수 없다는 것을 의미합니다.

### 기술적 제약 분석

1.  **제로 의존성 원칙**: 모든 스타일은 `<style>` 태그 내에 작성하고 로직은 `<script>` 태그 내에 작성해야 합니다.
2.  **방어적 CSS**: React 앱 마운트 후 전역 스타일이 부팅 화면을 오염시키는 것을 방지하기 위해 높은 우선순위의 ID 접두사(예: `#boot-screen`)를 사용하여 모든 부팅 스타일을 감싸기로 결정했습니다.
3.  **성능 우선**: 애니메이션은 가능한 CSS `transform`과 `opacity`를 사용하여 리플로우를 트리거하지 않고 메인 스레드를 차단하지 않도록 합니다.
4.  **시각적 일관성**: 색상, 폰트는 HagiCode의 Tailwind 구성과 일치해야 합니다.

### 아키텍처 패턴: Shell & Injector

**변이 패턴**을 채택했습니다. 핵심 로직은 즉시 실행 함수(IIFE)에 캡슐화되고 구체적인 렌더링 로직은 구성 항목으로 주입됩니다. 이렇게 하면 DOM 조작 로직을 반복 작성하지 않고도 간단한 구성으로 다른 스타일을 전환할 수 있습니다.

다음은 핵심 아키텍처 코드입니다:

```html
<!-- index.html에 인라인 -->
<div id="boot-root"></div>

<script>
(function() {
  const BootSequence = {
    config: {
      theme: 'terminal', // 'minimal', 'skeleton', 'code-rain' 등으로 구성 가능
      color: '#3b82f6'   // 브랜드 색상
    },
    
    // 핵심 생명주기
    init() {
      this.render();
      this.listenForMount();
    },

    // 현재 선택된 스타일 렌더링
    render() {
      const root = document.getElementById('boot-root');
      if (this.variants[this.config.theme]) {
        root.innerHTML = this.variants[this.config.theme].render();
      }
    },

    // React 마운트 성공 모니터링, 우아한 종료
    listenForMount() {
      window.addEventListener('hagicode:ready', () => {
        const screen = document.getElementById('boot-root');
        // 먼저 페이드아웃한 다음 DOM을 제거하여 깜빡임 방지
        screen.style.opacity = '0';
        screen.style.transition = 'opacity 0.3s ease';
        setTimeout(() => screen.remove(), 300);
      });
    },

    // 12가지 스타일의 구현 로직이 여기에 집중
    variants: {
      // ...구체적인 구현은 아래 참조
    }
  };

  BootSequence.init();
})();
</script>
```

## 12가지 부팅 스타일 설계 목록

이 12가지 스타일을 6대 카테고리로 나누어 다양한 시나리오와 미적 요구를 충족시킵니다.

### A. 미니멀리즘

> "적을수록 좋다". 최고의 로딩 속도를 추구하는 시나리오를 위해 가장 가벼운 솔루션을 제공합니다.

#### 1. Minimalist Dot (미니멀 호흡)
화면 중앙에 간단한 점 하나만 있고 호흡 애니메이션과 함께합니다.
*   **구현**: CSS `@keyframes`로 scale과 opacity를 제어합니다.
*   **적용**: 페이지가 절대적으로 깨끗해야 하는 모든场合.

#### 2. Brand Reveal (브랜드 계시)
SVG `stroke-dasharray` 애니메이션을 통해 손그림처럼 HagiCode 로고 라인을 그린 다음 텍스트가 페이드인됩니다.
*   **기교**: SVG 경로 애니메이션을 사용하여 질감이 매우 뛰어납니다.

### B. 스켈레톤 스크린 모방

> "눈을 속이는 예술". 실제 UI 레이아웃을 시뮬레이션하여 사용자가 페이지가 절반 정도 로드된 것 같은 느낌을 줍니다.

#### 3. Sidebar Chat Skeleton (사이드바 스켈레톤 스크린)
가장 실용적인 스타일일 수 있습니다. React 컴포넌트 `Sidebar`와 `ChatInput`과 완전히 동일한 레이아웃을 HTML로 수동으로 구축하고 회색 스트라이프 애니메이션을 덮어씌웁니다.
*   **가치**: React hydrate가 완료되면 스켈레톤 스크린이 순간적으로 실제 컴포넌트로 변하여 사용자는 전환을 거의 느끼지 못합니다.

#### 4. Card Stack Skeleton (카드 스택)
제안 카드 로딩 시의 스택 애니메이션을 시뮬레이션하고 3D 변환을 사용하여 카드가 살짝 떠있는 효과를 냅니다.

### C. 추상과 예술

> HagiCode의 기어 유전자를 보여줍니다.

#### 5. Geometric Morph (기하학적 변형)
화면 중앙에 기하학적 도형(정사각형)을 렌더링하고 시간이 지남에 따라 원형, 삼각형으로 부드럽게 변한 다음 마지막으로 로고가 됩니다.
*   **기술**: CSS `border-radius`의 부드러운 전환.

#### 6. Code Rain (코드 비)
《해커》에 오마주합니다. JetBrains Mono 폰트를 사용하여 배경에 흐릿한 문자 흐름이 떨어집니다.
*   **주의**: 성능을 위해 문자 흐름은 작은 영역으로 제한하거나 리프레시 빈도를 낮춰야 합니다.

#### 7. Neon Pulse (네온 펄스)
사이버펑크 스타일의 발광 링으로 `box-shadow`의 다중 중첩을 활용하여 강렬한 발광감을 만듭니다.

### D. 브랜드와 테마

> 시스템을 "살아있게" 만듭니다.

#### 8. Seasonal Theme (명절 테마)
동적 로더입니다. 현재 날짜를 판단하여 명절(설날, 크리스마스 등)에 따라 해당 SVG 애니메이션을 로드합니다.
*   **예시**: 설날에는 화면 하단에 빨간 랜턴이 살랑거립니다.

#### 9. Gradient Flow (그라디언트 플로우)
배경은 HagiCode 브랜드 색상의 유체 그라디언트를 사용하고 `background-size`와 `background-position` 애니메이션과 결합하여 오로라 같은 유동감을 조성합니다.

### E. 기술적 느낌

> 개발자에게 헌사합니다.

#### 10. Terminal Boot (터미널 부팅)
콘솔 출력을 시뮬레이션합니다. 코드가 한 줄씩 빠르게 스크롤됩니다:
```text
> Initializing HagiCode Core...
> Loading models...
> Connecting to neural network...
```
이것은 모든 개발자에게 친숙함을 느끼게 합니다.

#### 11. Progress Bar (미니멀 프로그레스 바)
화면 상단에 가느다란 프로그레스 바 하나가 있고 오른쪽에 백분율이 표시됩니다. 실제 다운로드 진행률을 가져올 수는 없지만 타이머를 사용하여 "신뢰할 수 있는" 로딩 과정을 시뮬레이션할 수 있습니다(전체 80%는 빠르게, 후반 20%는 느리게).

### F. 창의성

#### 12. Pixel Assembly (픽셀 조립)
매우 흥미로운 창의 아이디어입니다. 화면에 흩어진 블록들이 중앙으로 모여 HagiCode 로고 아이콘을 점차적으로 조립합니다. 코드의 빌드 과정을 상징합니다.

## 모범 사례와 시행착오 요약

HagiCode의 실제 개발에서 중요한 실천 디테일을 요약했습니다.

### 1. 방어적 CSS는 필수입니다
접두사를 작성하지 않는 편법을 쓰지 마세요. 한 번은 부팅 화면 스타일에 ID 제한을 추가하지 않아 React 마운트 후 전역 `div` 스타일이 의외로 부팅 화면에 영향을 주어 레이아웃이 깨진 적이 있습니다.
**경험**: 모든 CSS 선택자는 `#boot-screen` 아래에 걸고 `!important`를 사용하여 우선순위를 높입니다(부팅 화면 CSS에서만).

### 2. 우아한 전환
React 마운트 성공 후 부팅 화면 DOM을 직접 `remove()`하지 마세요.
**올바른 방법**:
1.  React가 `window.dispatchEvent(new Event('hagicode:ready'))`를 트리거합니다.
2.  부팅 화면이 이벤트를 감지하면 먼저 `opacity: 0`을 설정합니다.
3.  300ms(CSS transition 시간) 동안 기다려 사용자가 볼 수 없게 된 것을 확인한 후 `.remove()`를 실행합니다.

### 3. 테마 변수 동기화
부팅 화면의 색상 코드는 `index.html`에 하드코딩되어 있습니다. Tailwind의 메인 색상을 수정하면 여기도 동기화해야 합니다.
**최적화 방안**: Vite 빌드 스크립트에서 간단한 플러그인을 작성하여 `tailwind.config.js`를 읽고 색상 변수를 `index.html`의 템플릿 변수에 주입하여 단일 데이터 소스를 실현합니다.

### 4. 폰트 프리로드
부팅 화면은 일반적으로 브랜드 폰트를 사용해야 하지만 폰트 로딩이 느리면 FOUT(스타일이 적용되지 않은 텍스트 번쩍임)가 발생합니다.
**해결 방안**: `<head>`에 `<link rel="preload" href="/fonts/JetBrainsMono.woff2" as="font" type="font/woff2" crossorigin>`을 추가합니다. 이것은 경험을 향상시키는 저비용 고효율 수단입니다.

### 5. 성능 모니터링
`index.html` 하단에 `performance.mark('boot-start')`를 주입하고 React 마운트 성공 시 `boot-end`를 표시합니다.
**의미**: Application Insights를 통해 이 데이터를 수집하면 부팅 화면이 사용자의 인식된 대기 시간(Perceived Loading Time)을 단축시키는 정도를 실제로 볼 수 있습니다. 데이터에 따르면 훌륭한 스켈레톤 스크린은 사용자의 "느린 네트워크"에 대한容忍度를 50% 이상 높일 수 있습니다.

## 요약

좋은 부팅 화면은 단순히 "대기 시간의 장식"이 아니라 제품과 사용자의 첫 번째 상호작용 핸드셰이크 신호입니다. HagiCode 프로젝트에서 **Variants 패턴**을 기반으로 하는 이 부팅 시스템을 통해 우리는 다른 명절과 다른 버전 간에 스타일을 유연하게 전환할 수 있어 제품의 재미와 전문성을 크게 향상시켰습니다.

이 글에서 공유한 방안은 완전히 원생 웹 표준을 기반으로 하며 무거운 의존성을 전혀 도입하지 않았습니다. 이것이 바로 HagiCode가 "가볍고 강력함"을 추구하는 구현입니다. 이 방안이 가치 있다고 생각하신다면 HagiCode 저장소에서 소스 코드 구현을 살펴보거나 창의적인 디자인을 기여해 주세요!
