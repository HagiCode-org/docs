## HagiCode를 위한 12가지 극한의 부트 경험 설계: 미니멀에서 사이버펑크까지

> React 19 애플리케이션이 다운로드되고 Hydration되는 짧은 간격은 사용자가 브랜드 개성을 인지할 수 있는 황금 창구입니다. 이 글에서는 HagiCode 프로젝트에서 HTML/CSS/JS 기반으로 구축한 완전한 부트 스타일 시스템을 공유합니다.


## 배경

HagiCode는 ASP.NET Core 10과 React 19 (Vite) 기반의 현대 애플리케이션으로, 프론트엔드와 백엔드 분리 배포 아키텍처를 채택합니다. 프론트엔드 산출물은 백엔드의 `wwwroot/` 디렉토리에 배치되어 ASP.NET Core에 의해 호스팅됩니다.

그러나 이 아키텍처는 전형적인 사용자 경험 문제를 가져왔습니다: 사용자가 웹페이지를 방문할 때 브라우저는 먼저 HTML을 로드한 다음 거대한 JS Bundle을 다운로드하고 마지막으로 React가 Hydration(수화)을 실행합니다. 이 수백 밀리초에서 수 초의 "진공기" 동안 사용자는 빈 화면 또는 생기 없는 정적 페이지를 마주하게 됩니다.

이 간격을 채우고 HagiCode의 브랜드 개성을 주입하기 위해 `index.html` 인라인 코드에 완전히 기반한 부트 스타일 시스템을 설계해야 했습니다.

## HagiCode 소개

이 글에서 공유하는 부트 스크린 설계 방안은 [HagiCode](https://github.com/HagiCode-org/site) 프로젝트에서의 실천 경험에서 비롯되었습니다. AI 코드 어시스턴트인 HagiCode는 코드 생성 효율뿐만 아니라 개발자의 시각적 경험도 동등하게 중시합니다. 이 부트 시스템은 극한의 프론트엔드 성능을 추구하는 과정에서 나온 산물입니다.

## 핵심 도전과 아키텍처 설계

설계를 시작하기 전에 기술적 제약을 명확히 해야 합니다. `index.html`에 인라인으로 구현해야 하므로 외부 CSS나 JS 파일을 로드할 수 없습니다(React 자체의 Bundle 제외).

### 기술 제약 분석

1.  **제로 의존성 원칙**: 모든 스타일은 `<style>` 태그 내에, 로직은 `<script>` 태그 내에 작성해야 합니다.
2.  **방어적 CSS**: React 애플리케이션이 마운트된 후 글로벌 스타일이 부트 스크린을 오염시키는 것을 방지하기 위해 높은 우선순위의 ID 접두사(예: `#boot-screen`)로 모든 부트 스타일을 래핑합니다.
3.  **성능 우선**: 애니메이션은 CSS `transform`과 `opacity`를 최대한 사용하여 리플로우를 트리거하지 않고 메인 스레드를 차단하지 않도록 합니다.
4.  **시각적 일관성**: 색상, 폰트는 HagiCode의 Tailwind 구성과 일치해야 합니다.

### 아키텍처 패턴: Shell & Injector

우리는 **변형 패턴**을 채택했습니다. 핵심 로직은 즉시 실행 함수(IIFE)에 캡슐화되고, 구체적인 렌더링 로직은 구성 항목으로 주입됩니다. 이렇게 하면 DOM 조작 로직을 반복 작성하지 않고 간단한 구성으로 다양한 스타일을 전환할 수 있습니다.

핵심 아키텍처 코드는 다음과 같습니다:

```html
<!-- index.html에 인라인 -->
<div id="boot-root"></div>

<script>
(function() {
  const BootSequence = {
    config: {
      theme: 'terminal', // 'minimal', 'skeleton', 'code-rain' 등으로 구성 가능
      color: '#3b82f6'   // 브랜드 색상
    },

    // 핵심 생명주기
    init() {
      this.render();
      this.listenForMount();
    },

    // 현재 선택된 스타일 렌더링
    render() {
      const root = document.getElementById('boot-root');
      if (this.variants[this.config.theme]) {
        root.innerHTML = this.variants[this.config.theme].render();
      }
    },

    // React 마운트 성공 감지, 우아한 종료
    listenForMount() {
      window.addEventListener('hagicode:ready', () => {
        const screen = document.getElementById('boot-root');
        // 먼저 페이드아웃, 다음 DOM 제거, 깜빡임 방지
        screen.style.opacity = '0';
        screen.style.transition = 'opacity 0.3s ease';
        setTimeout(() => screen.remove(), 300);
      });
    },

    // 12가지 스타일의 구현 로직이 여기에 집중됩니다
    variants: {
      // ...구체적인 구현은 아래를 참조하세요
    }
  };

  BootSequence.init();
})();
</script>
```

## 12가지 부트 스타일 설계 목록

12가지 스타일을 6대 범주로 분류하여 다양한 시나리오와 미적 요구를 충족합니다.

### A. 미니멀리즘

> "적으면 적을수록 낫다". 극한의 로딩 속도를 추구하는 시나리오를 위해 가장 가벼운 방안을 제공합니다.

#### 1. Minimalist Dot (미니멀 호흡)
화면 중앙에 간단한 점 하나만 있고, 호흡 애니메이션과 함께합니다.
*   **구현**: CSS `@keyframes`로 scale과 opacity를 제어합니다.
*   **적용**: 페이지를 절대적으로 깨끗하게 유지해야 하는 모든 경우.

#### 2. Brand Reveal (브랜드 리빌)
SVG `stroke-dasharray` 애니메이션으로 수작업처럼 HagiCode의 Logo 선을 그린 후 텍스트를 페이드인합니다.
*   **기술**: SVG 경로 애니메이션을 사용하여 질감이 뛰어납니다.

### B. 스켈레톤 스크린 모방

> "눈을 속이는 예술". 실제 UI 레이아웃을 모방하여 사용자가 페이지가 이미 절반 로드된 것처럼 느끼게 합니다.

#### 3. Sidebar Chat Skeleton (사이드바 스켈레톤 스크린)
가장 실용적인 방법일 수 있습니다. React 컴포넌트 `Sidebar` 및 `ChatInput`과 정확히 동일한 레이아웃을 HTML로 수동 구축하고 회색 줄무늬 애니메이션을 겹쳐씌웁니다.
*   **가치**: React 하이드레이션이 완료되면 스켈레톤 스크린이 순간적으로 실제 컴포넌트로 변환되어 사용자가 전환을 거의 느끼지 못합니다.

#### 4. Card Stack Skeleton (카드 스택)
제안 카드 로딩 시의 스택 애니메이션을 모방하고 3D 변환으로 카드를 살짝 떠 있게 합니다.

### C. 추상과 예술

> HagiCode의 극장 유전자를 전시합니다.

#### 5. Geometric Morph (기하학적 변형)
화면 중앙에 기하학적 형태(정사각형)를 렌더링하고, 시간에 따라 부드럽게 원, 삼각형으로 변환한 후 마지막으로 Logo가 됩니다.
*   **기술**: CSS `border-radius`의 부드러운 전환.

#### 6. Code Rain (코드 비)
"매트릭스"에 오마주. JetBrains Mono 폰트를 사용하여 배경에 옅은 문자 흐름을 떨어뜨립니다.
*   **주의**: 성능을 위해 문자 흐름을 작은 영역으로 제한하거나 새로고침 빈도를 낮춰야 합니다.

#### 7. Neon Pulse (네온 펄스)
사이버펑크 스타일의 발광 링으로 `box-shadow`의 다중 겹침으로 강렬한 발광감을 만들어냅니다.

### D. 브랜드와 테마

> 시스템을 "살아있게" 만듭니다.

#### 8. Seasonal Theme (명절 테마)
동적 로더입니다. 현재 날짜에 따라 명절(예: 춘절, 크리스마스)을 판단하고 해당 SVG 애니메이션을 로드합니다.
*   **예**: 춘절에는 화면 하단에 붉은 등불이 가볍게 흔들립니다.

#### 9. Gradient Flow (그래디언트 흐름)
배경에 HagiCode 브랜드 색상의 유체 그래디언트를 사용하고 `background-size` 및 `background-position` 애니메이션과 함께 오로라 같은 흐름 느낌을 만들어냅니다.

### E. 기술감

> 개발자에게 경의를 표합니다.

#### 10. Terminal Boot (터미널 부트)
콘솔 출력을 모방합니다. 코드가 한 줄씩 빠르게 스크롤됩니다:
```text
> Initializing HagiCode Core...
> Loading models...
> Connecting to neural network...
```
모든 개발자에게 친숙하게 느껴질 것입니다.

#### 11. Progress Bar (미니멀 프로그레스 바)
화면 상단에 가는 프로그레스 바가 있고 오른쪽에 백분율이 표시됩니다. 실제 다운로드 진행률을 얻을 수 없지만 타이머로 "신뢰할 수 있는" 로딩 과정을 시뮬레이션할 수 있습니다(처음 80%는 빠르게, 나머지 20%는 감속).

### F. 창의성

#### 12. Pixel Assembly (픽셀 조립)
매우 흥미로운 아이디어입니다. 화면에 사각형들이 흩어져 있고 중심으로 모여 HagiCode의 Logo 아이콘을 점진적으로 조립합니다. 코드의 구축 과정을 상징합니다.

## 모범 사례 및 시행착오 요약

HagiCode 실제 개발에서 몇 가지 중요한 실천 세부 사항을 요약했습니다.

### 1. 방어적 CSS는 필수입니다
접두사를 작성하지 않는 게으름을 피하지 마세요. 한 번은 부트 스크린 스타일에 ID 제한을 추가하지 않아 React 마운트 후 글로벌 `div` 스타일이 부트 스크린에 영향을 주어 레이아웃이 붕괴된 적이 있습니다.
**경험**: 모든 CSS 선택자를 `#boot-screen` 아래에 배치하고 `!important`로 우선순위를 높입니다(부트 스크린 CSS에서만).

### 2. 우아한 전환
React 마운트 성공 후 부트 스크린 DOM을 직접 `remove()`하지 마세요.
**올바른 방법**:
1.  React가 `window.dispatchEvent(new Event('hagicode:ready'))`를 트리거합니다.
2.  부트 스크린이 이벤트를 수신하고 먼저 `opacity: 0`으로 설정합니다.
3.  300ms(CSS transition 시간) 대기하여 사용자가 보이지 않는 것을 확인한 후 `.remove()`를 실행합니다.

### 3. 테마 변수 동기화
부트 스크린의 색상 코드는 `index.html`에 하드코딩되어 있습니다. Tailwind의 기본 색상을 수정하면 여기도 동기적으로 수정해야 합니다.
**최적화 방안**: Vite 빌드 스크립트에서 간단한 플러그인을 작성하여 `tailwind.config.js`를 읽고 색상 변수를 `index.html`의 템플릿 변수에 주입하여 단일 데이터 소스를 실현합니다.

### 4. 폰트 프리로드
부트 스크린은 일반적으로 브랜드 폰트를 사용해야 하지만 폰트 로딩이 느리면 FOUT (Flash of Unstyled Text)가 발생합니다.
**해결 방법**: `<head>`에 `<link rel="preload" href="/fonts/JetBrainsMono.woff2" as="font" type="font/woff2" crossorigin>`을 추가합니다. 이는 경험을 향상시키는 저비용 고수익 수단입니다.

### 5. 성능 모니터링
`index.html` 하단에 `performance.mark('boot-start')`를 주입하고 React 마운트 성공 시 `boot-end`를 표시했습니다.
**의의**: Application Insights로 이 데이터를 수집하여 부트 스크린이 사용자의 인지 대기 시간(Perceived Loading Time)을 얼마나 단축했는지 실제로 확인할 수 있습니다. 데이터에 따르면 우수한 스켈레톤 스크린은 사용자의 "느린 네트워크"에 대한容忍度를 50% 이상 향상시킵니다.

## 요약

좋은 부트 스크린은 단순히 "대기 시 장식"이 아니라 제품과 사용자의 첫 번째 상호작용 핸드셰이크 신호입니다. HagiCode 프로젝트에서 **Variants 패턴** 기반의 이 부트 시스템을 통해 다양한 명절, 다양한 버전 간에 유연하게 스타일을 전환하여 제품의 흥미와 전문성을 크게 강화했습니다.

이 글에서 공유한 방안은 완전히 네이티브 Web 표준 기반이며 어떠한 무거운 의존성도 도입하지 않았습니다. 이는 HagiCode가 추구하는 "가볍고 강력함"의 체현입니다. 이 방안이 가치 있다고 생각되시면 HagiCode 저장소에서 소스코드 구현을 확인하거나 창의적인 디자인을 기여해 주세요!
