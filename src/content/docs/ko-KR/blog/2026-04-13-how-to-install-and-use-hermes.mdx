Hermes를 설치하고 사용하기 시작하려면, 실제로는 단 세 단계면 충분합니다:

1. 공식 설치 명령 실행
2. 터미널에서 `hermes`로 CLI 시작
3. Feishu에서 계속 사용하고 싶다면 `hermes gateway setup` 구성

이 문서는 Hermes의 모든 기능을 한 번에 설명하려는 것이 아니라, 가장 중요한 시작 루프를 완료하는 데 도움을 주려는 목적입니다: 설치하고, 실행하고, 사용을 시작한 다음, 가장 일반적인 메시징 플랫폼 시나리오에 연결합니다.

## Hermes란 무엇이며, 누구에게 적합한가

Hermes Agent는 로컬 터미널에서 사용할 수도 있고, 메시징 플랫폼 게이트웨이를 통해 사용할 수도 있는 AI 에이전트입니다.

대부분의 개발자에게 가장 일반적인 두 가지 진입점이 있습니다:

- **CLI**: 터미널에서 `hermes`를 입력하여 직접 대화형 인터페이스 진입
- **Messaging Gateway**: `hermes gateway`를 실행한 다음 Feishu, Telegram, Discord, Slack 등의 플랫폼에서 대화

현재 목표가 빠르게 시작하는 것이라면, 순서를 반대로 하지 않고 다음 경로를 먼저 따르는 것이 좋습니다:

- Hermes를 먼저 설치
- CLI에서 먼저 사용 가능 여부 확인
- 메시징 플랫폼 연결 여부 결정

이렇게 하면 문제를 더 쉽게 찾을 수 있고, Hermes를 처음 접하는 사용자에게도 더 적합합니다.

## Hermes 설치 전 알아야 할 사항

Hermes README에 따르면, 공식 빠른 설치 경로는 다음 환경을 지원합니다:

- Linux
- macOS
- WSL2
- Android via Termux

### Windows 사용자 참고사항

Hermes는 현재 네이티브 Windows에서 직접 실행하는 것을 지원하지 않습니다. Windows를 사용 중이라면 WSL2를 먼저 설치한 다음, WSL2 내에서 설치 명령을 실행하는 것이 좋습니다.

이 부분은 문서 시작 부분에서 명확히 하는 것이 좋습니다. 많은 설치 실패는 명령어 문제가 아니라 실행 환경이 요구 사항에 맞지 않기 때문입니다.

## Hermes 설치 방법

Hermes README에서 제공하는 빠른 설치 명령은 다음과 같습니다:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

이 명령은 공식 설치 스크립트를 실행하여 플랫폼 관련 초기화 프로세스를 처리합니다.

### 설치 완료 후 shell 새로고침

설치가 끝나면 먼저 shell 환경을 다시 로드하세요. 가장 일반적인 방법은 다음과 같습니다:

```bash
source ~/.bashrc
```

zsh를 사용 중이라면 다음으로 변경할 수도 있습니다:

```bash
source ~/.zshrc
```

### Hermes 설치 확인 방법

가장 직접적인 확인 방법은 다음을 실행하는 것입니다:

```bash
hermes
```

구성 및 종속성이 정상인지 추가로 확인하려면 다음을 실행할 수도 있습니다:

```bash
hermes doctor
```

`hermes doctor`는 다음과 같은 상황에서 사용하기에 적합합니다:

- 설치 후 명령 동작이 정상적이지 않을 때
- 모델 구성 실패
- gateway 시작 실패
- 환경 종속성이 완전한지 확실하지 않을 때

## Hermes를 처음 사용하는 방법

### 먼저 CLI부터 시작

Hermes가 사용 가능한지만 빠르게 확인하고 싶다면 가장 간단한 방법은 다음과 같습니다:

```bash
hermes
```

이 명령은 Hermes의 대화형 CLI를 시작합니다. Hermes를 처음 접하는 사용자에게 가장 권장되는 시작점입니다. 가장 핵심적인 몇 가지를 먼저 확인할 수 있기 때문입니다:

- 명령이 실제로 작동하는지
- 현재 모델 구성이 정상인지
- 터미널 도구 체인이 정상 작동하는지
- 대화형 경험이 필요한 방식인지

### 첫 번째 구성을 완료하는 데 충분한 명령들

Hermes README에 나열된 몇 가지 고빈도 명령이 기본적으로 첫 번째 사용 경로를 구성합니다:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

각각이 무엇을 하는지 모른다면, 다음과 같이 기억하세요:

- `hermes model`: 모델 선택 및 전환
- `hermes tools`: 현재 사용 가능한 도구 보기 및 구성
- `hermes config set`: 특정 구성 항목 변경
- `hermes setup`: 전체 초기화 마법사 실행
- `hermes update`: Hermes 업데이트
- `hermes doctor`: 문제 해결 수행

초보자에게 가장 실용적인 순서는 보통 다음과 같습니다:

1. 먼저 `hermes model` 실행
2. 일반적으로 사용하는 항목을 한 번에 구성하려면 `hermes setup` 실행

## Hermes의 가장 일반적인 두 가지 사용 방법

### 1. 터미널에서 Hermes를 일상 개발 도우미로 사용

CLI 모드는 다음 시나리오에 적합합니다:

- 로컬에서 코드를 작성할 때 직접 질문
- 프로젝트 확인, 파일 수정, 명령 실행
- 일회성 디버깅 또는 검토
- 현재 작업 디렉토리에서 지속적으로 협업

가장 큰 장점은 경로가 가장 짧다는 것입니다: 추가 플랫폼 연결이 필요 없고, 봇 구성을 먼저 처리할 필요도 없으며, 첫 번째 사용 습관을 형성하는 데 가장 적합합니다.

### 2. 메시징 플랫폼을 통해 Hermes 사용

Feishu, Telegram, Discord 등의 플랫폼에서 Hermes와 대화하고 싶다면 메시징 게이트웨이를 사용해야 합니다.

가장 일반적인 진입 명령은 다음과 같습니다:

```bash
hermes gateway setup
hermes gateway
```

여기서:

- `hermes gateway setup`은 대화형 플랫폼 구성에 사용
- `hermes gateway`는 게이트웨이 프로세스 시작에 사용

공식 문서에 따르면, gateway는 구성된 플랫폼에 연결하고, 세션을 관리하며, 예약 작업等功能을 처리하는 통합 백그라운드 프로세스입니다.

## Feishu를 예로 들어, Hermes를 메시징 플랫폼에 연결하는 방법

일상 업무가 주로 Feishu에서 이루어진다면, Feishu/Lark는 매우 자연스러운 Hermes 연결 방식이 될 것입니다.

### 최소 실행 가능 연결 방식

Feishu/Lark에 대한 공식 문서의 권장 진입점은 다음과 같습니다:

```bash
hermes gateway setup
```

실행 후 마법사에서 **Feishu / Lark**를 선택합니다.

Feishu 문서에서 제공하는 두 가지 연결 모드는 다음과 같습니다:

- `websocket`: 권장
- `webhook`: 선택사항

Hermes가 노트북, 워크스테이션 또는 프라이빗 서버에서 실행 중인 경우, 공용 콜백 주소를 추가로 노출할 필요가 없으므로 `websocket`을 우선 사용하는 것이 더 간단합니다.

### 수동 구성 시 최소한 알아야 할 변수

마법사가 아닌 수동으로 구성하는 경우, Feishu 문서에 나열된 핵심 변수는 다음과 같습니다:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

그중 가장 주의해야 할 두 가지는 다음과 같습니다:

- `FEISHU_ALLOWED_USERS`: 구성을 권장합니다. bot에 접근할 수 있는 모든 사람이 직접 사용할 수 있는 것을 방지하기 위함입니다
- `FEISHU_HOME_CHANNEL`: cron 결과 또는 기본 알림을 받을 home chat을 미리 지정할 수 있습니다

### Feishu 그룹 채팅에서 때때로 응답하지 않는 이유

이 부분은 간과하기 쉽습니다: Feishu 그룹 채팅에서 Hermes는 기본적으로 모든 메시지를 보고 응답하지 않습니다.

공식 문서에서 명확히 설명하고 있습니다:

- 개인 채팅에서는 Hermes가 메시지에 응답
- 그룹 채팅에서는 명시적으로 bot을 `@` 멘션해야만 메시지를 처리

특정 Feishu 대화를 home channel로 설정하려면 채팅에서 다음을 사용할 수도 있습니다:

```text
/set-home
```

또는 구성에 미리 작성할 수도 있습니다:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## 초보자가 먼저 기억해야 할 Hermes 명령

CLI에서든 메시징 플랫폼에서든, 다음 명령들만 기억하면 충분합니다:

- `/new` 또는 `/reset`: 새 세션 시작
- `/model`: 모델 보기 또는 전환
- `/retry`: 이전 라운드 재시도
- `/undo`: 이전 상호작용 실행 취소
- `/compress`: 수동으로 컨텍스트 압축
- `/help`: 도움말 보기

주로 메시징 플랫폼에서 사용한다면 다음 하나를 추가로 기억하세요:

- `/sethome` 또는 `/set-home`: 현재 채팅을 home channel로 설정

이러한 명령은 초보자 단계에서 가장 일반적인 작업을 포함합니다: 다시 시작, 조정, 되돌리기, 보기, 계속 사용.

## 자주 묻는 질문

### Windows에서 Hermes를 직접 설치할 수 있나요?

아니요. 현재 공식 문서에서 네이티브 Windows는 지원하지 않으며, WSL2 사용을 권장합니다.

### 설치 후 `hermes` 입력 후 반응이 없으면 어떻게 하나요?

다음 순서로 확인해 보세요:

1. 먼저 shell을 다시 로드하세요(예: `source ~/.bashrc`)
2. 다시 `hermes` 실행
3. 여전히 비정상이면 `hermes doctor` 실행

### Feishu 그룹에서 bot이 응답하지 않는 이유는 무엇인가요?

다음 세 가지를 먼저 확인하세요:

- 그룹에서 Hermes를 `@` 멘션했나요
- `FEISHU_ALLOWED_USERS`가 현재 사용자를 제한하고 있나요
- 현재 그룹 채팅 정책이 그룹 메시지 처리를 허용하나요

공식 Feishu 문서에 따르면, 그룹 채팅 시나리오에서 명시적 @mention은 필수 조건입니다.

## 다음 단계 제안

빠르게 Hermes 사용을 시작하고 싶다면 가장 권장하는 순서는 다음과 같습니다:

1. 먼저 설치 명령 실행
2. 먼저 `hermes`를 사용하여 로컬 CLI에서 시작
3. `hermes model`과 `hermes setup`으로 기본 구성 완료
4. Feishu에서 계속 사용하고 싶다면 `hermes gateway setup` 구성

이 문서가 시리즈의 첫 번째 문서로서 가장 적합한 역할은 "모든 고급 기능을 한 번에 설명하는 것"이 아니라, 사용자를 먼저 안내하는 것입니다.

다음과 같은 주제로 계속 분리하는 것이 더 적합합니다:

- Hermes Feishu 연동 완전 가이드
- Hermes 일반 슬래시 명령 가이드
- Hermes 게이트웨이 구성 및 문제 해결 가이드

Hermes 콘텐츠를 계속 만들 계획이라면, 이 문서를 후속 문서의 시작점으로 삼고 내부 링크 체계를 점진적으로 보완할 수 있습니다.
