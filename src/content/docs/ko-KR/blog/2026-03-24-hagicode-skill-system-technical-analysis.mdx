## HagiCode Skill 시스템 기술 분석: 확장 가능한 AI 스킬 관리 플랫폼 구축 방법

> 本文은 HagiCode 프로젝트의 Skill(스킬) 관리 시스템의 아키텍처 설계와 구현 방안을 심층 분석합니다. 로컬 전역 관리, 마켓 검색, 지능형 추천, 신뢰 공급자 관리 네 가지 핵심 기능의 기술 구현 세부 사항을 다룹니다.

## 배경

AI 코드 어시스턴트 분야에서 AI의 능력 경계를 확장하는 것은 항상 핵심 과제였습니다. Claude Code 자체의 코드 보조 능력은 상당히 강력하지만, 다른 개발팀과 다른 기술 스택은 종종 특정 시나리오에 대한 전문 능력이 필요합니다. 예를 들어 Docker 배포 처리, 데이터베이스 최적화, 프론트엔드 컴포넌트 생성 등이 있습니다. 이때 Skill(스킬) 시스템이 특히 중요해집니다.

HagiCode 프로젝트도 개발 과정에서 비슷한 과제에 직면했습니다. Claude Code가 사람처럼 새로운 전문 기술을 '학습'하면서도 좋은 사용자 경험과 엔지니어링 유지보수성을 유지하게 하려면 어떻게 해야 할까요?毕竟这个问题，说难也难，说简单也简单。围绕这个问题，我们设计并实现了一套完整的 Skill 管理系统。

本文은 이 시스템의 기술 아키텍처와 핵심 구현을 상세히 분석하며, AI 확장성과 명령줄 도구 통합에 관심 있는 개발자에게 적합합니다. 당신에게 도움이 될 수도 있고 아닐 수도 있지만, 어쨌든 작성했습니다.

## 시스템 아키텍처 개요

Skill 시스템은 프론트엔드와 백엔드가 분리된 아키텍처 설계를 채택하고 있습니다.说起来也没什么特别的。

**프론트엔드 부분**은 TanStack Start + React로 사용자 인터페이스를 구축하고 Redux Toolkit로 상태를 관리하며, 네 개의 주요 기능은 각각 네 개의 Tab 컴포넌트에 해당합니다. 로컬 스킬, 마켓 갤러리, 지능형 추천, 신뢰 공급자입니다. 이렇게 설계한 것도 사실은 사용자 경험을 위해서입니다.

**백엔드 부분**은 ASP.NET Core + ABP Framework를 기반으로 하며 Orleans Grain으로 분산 상태 관리를 구현합니다. 온라인 API 클라이언트는 `IOnlineApiClient` 인터페이스를 캡슐화하여 원격 스킬 디렉터리 서비스와 통신합니다.

전체 아키텍처의 설계 원칙은 '명령 실행과 비즈니스 로직 분리'이며, 어댑터 패턴을 통해 npm/npx 명령 실행의 세부 사항을 독립 모듈로 차단합니다.毕竟谁愿意看到一堆命令行散落在代码各处呢？

## 핵심 기능 1: 로컬 전역 관리

로컬 전역 관리는 가장 기초적인 기능 모듈로, 설치된 스킬 목록을 표시하고 제거 작업을 지원합니다。也没什么复杂的，就是把事情做好而已。

### 기술 방안

구현 위치는 `LocalSkillsTab.tsx`와 `LocalSkillCommandAdapter.cs`입니다. 핵심 아이디어는 `npx skills` 명령을 캡슐화하고 해당 JSON 출력을 구문 분석하여 내부 데이터 구조로 변환하는 것입니다。说起来简单，做起来其实也简单。

```csharp
public async Task<IReadOnlyList<LocalSkillInventoryResponseDto>> GetLocalSkillsAsync(
    CancellationToken cancellationToken = default)
{
    var result = await _commandAdapter.ListGlobalSkillsAsync(cancellationToken);
    return result.Skills.Select(skill => new LocalSkillInventoryResponseDto
    {
        Name = skill.Name,
        Version = skill.Version,
        Source = skill.Source,
        InstalledPath = skill.InstalledPath,
        Description = skill.Description
    }).ToList();
}
```

데이터 흐름은 매우 명확합니다. 프론트엔드 요청 → SkillGalleryAppService 수신 → LocalSkillCommandAdapter가 npx 명령 실행 → JSON 결과 구문 분석 → DTO 객체 반환。一环扣一环，也没什么好说的。

스킬 제거는 `npx skills remove -g <skillName> -y` 명령을 사용하며 시스템은 자동으로 종속성과 정리 작업을 처리합니다. 설치 메타데이터는 스킬 디렉터리의 `managed-install.json`에 저장되며 설치 시간, 소스 버전 등의 정보를 기록하여 후속 업데이트와 감사에 편리합니다。毕竟有些东西，记下来总是好的。

### 설치 프로세스 상세 분석

스킬 설치는 여러 단계의 조정이 필요합니다。怎么说呢，其实也不算太复杂：

```csharp
public async Task<SkillInstallResultDto> InstallAsync(
    SkillInstallRequestDto request,
    CancellationToken cancellationToken = default)
{
    // 1. 설치 참조规范化
    var normalized = _referenceNormalizer.Normalize(
        request.SkillId,
        request.Source,
        request.SkillSlug,
        request.Version);

    // 2. 전제 조건 확인
    await _prerequisiteChecker.CheckAsync(cancellationToken);

    // 3. 설치 잠금获取
    using var installLock = await _lockProvider.AcquireAsync(normalized.SkillId);

    // 4. 설치 명령 실행
    var result = await _installCommandRunner.ExecuteAsync(
        new SkillInstallCommandExecutionRequest
        {
            Command = $"npx skills add {normalized.FullReference} -g -y",
            Timeout = TimeSpan.FromMinutes(4)
        },
        cancellationToken);

    // 5. 설치 메타데이터持久化
    await _metadataStore.WriteAsync(normalized.SkillPath, request);

    return new SkillInstallResultDto { Success = result.Success };
}
```

여기서는 몇 가지 핵심 설계 패턴이 사용됩니다. **참조规范化器**는 다양한 입력 형식(예: `tanweai/pua`, `@opencode/docker-skill`)을 통합된 내부 표현으로 변환을 담당합니다. **설치 잠금 메커니즘**은 동일한 스킬에 대해 동시에 하나의 설치 작업만 수행되도록 보장합니다. **스트리밍 출력**은 Server-Sent Events를 통해 프론트엔드에 설치 진행 상황을 실시간으로 푸시하여 사용자가 터미널과 유사한 실시간 로그를 볼 수 있습니다.

이러한 설계 패턴은说到底，也还是为了让事情变得简单罢了。

## 핵심 기능 2: 마켓 검색

마켓 검색을 통해 사용자는 커뮤니티의 스킬을 발견하고 설치할 수 있습니다。毕竟一个人的能力是有限的，众人的智慧才是无穷的。

### 기술 방안

검색 기능은 온라인 API `https://api.hagicode.com/v1/skills/search`에 의존합니다. 응답 속도를 높이기 위해 시스템은 캐싱 메커니즘을 구현했습니다。缓存这东西，就像记忆一样，有些东西记住了，下次就不用再费劲去想。

```csharp
private async Task<IReadOnlyList<SkillGallerySkillDto>> SearchCatalogAsync(
    string query,
    CancellationToken cancellationToken,
    IReadOnlySet<string>? allowedSources = null)
{
    var cacheKey = $"skill_search:{query}:{string.Join(",", allowedSources ?? Array.Empty<string>())}";

    if (_memoryCache.TryGetValue(cacheKey, out var cached))
        return (IReadOnlyList<SkillGallerySkillDto>)cached!;

    var response = await _onlineApiClient.SearchAsync(
        new SearchSkillsRequest
        {
            Query = query,
            Limit = _options.LimitPerQuery,
        },
        cancellationToken);

    var results = response.Skills
        .Where(skill => allowedSources is null || allowedSources.Contains(skill.Source))
        .Select(skill => new SkillGallerySkillDto { ... })
        .ToList();

    _memoryCache.Set(cacheKey, results, TimeSpan.FromMinutes(10));
    return results;
}
```

검색 결과는 신뢰 소스별 필터링을 지원하며 사용자가 신뢰하는 스킬 소스만 표시합니다. 미리 설정된 시드 쿼리는 디렉터리 초기화에 사용됩니다. 예를 들어 'popular', 'recent' 등이 있어 사용자가 처음 열었을 때 추천되는 인기 스킬을 볼 수 있습니다。毕竟第一印象还是重要的。

## 핵심 기능 4: 신뢰 공급자 관리

신뢰 공급자 관리를 통해 사용자는 어떤 스킬 소스를 신뢰할 수 있는지 제어할 수 있습니다。毕竟信任这东西，还是要自己把握的。

### 일치 규칙

신뢰 공급자는 두 가지 일치 규칙을 지원합니다. **정확히 일치**(exact)와 **접두사 일치**(prefix)입니다。

```csharp
public static TrustedSkillProviderResolutionSnapshot Resolve(
    TrustedSkillProviderSnapshot snapshot,
    string source)
{
    var normalizedSource = Normalize(source);

    foreach (var entry in snapshot.Entries.OrderBy(e => e.SortOrder))
    {
        if (!entry.IsEnabled) continue;

        foreach (var rule in entry.MatchRules)
        {
            bool isMatch = rule.MatchType switch
            {
                TrustedSkillProviderMatchRuleType.Exact
                    => string.Equals(normalizedSource, Normalize(rule.Value),
                        StringComparison.OrdinalIgnoreCase),
                TrustedSkillProviderMatchRuleType.Prefix
                    => normalizedSource.StartsWith(Normalize(rule.Value) + "/",
                        StringComparison.OrdinalIgnoreCase),
                _ => false
            };

            if (isMatch)
                return new TrustedSkillProviderResolutionSnapshot
                {
                    IsTrustedSource = true,
                    ProviderId = entry.ProviderId,
                    DisplayName = entry.DisplayName
                };
        }
    }

    return new TrustedSkillProviderResolutionSnapshot { IsTrustedSource = false };
}
```

미리 설정된 신뢰 공급자에는 Vercel, Azure, anthropics, Microsoft, browser-use 등 유명 조직과 프로젝트가 포함됩니다. 사용자 정의 공급자는 구성 파일을 통해 추가할 수 있으며 공급자 ID, 표시 이름, 배지 라벨, 일치 규칙 등을 지정할 수 있습니다。毕竟世界那么大，不可能只有几家是可信的。

### 지속성 구현

신뢰 구성은 Orleans Grain을 사용하여 지속성 저장합니다：

```csharp
public class TrustedSkillProviderGrain : Grain<TrustedSkillProviderState>,
    ITrustedSkillProviderGrain
{
    public async Task UpdateConfigurationAsync(TrustedSkillProviderSnapshot snapshot)
    {
        State.Snapshot = snapshot;
        await WriteStateAsync();
    }

    public Task<TrustedSkillProviderSnapshot> GetConfigurationAsync()
    {
        return Task.FromResult(State.Snapshot);
    }
}
```

이 방식의 장점은 구성 변경이 자동으로 모든 노드에 동기화되며 캐시를 수동으로 새로 고칠 필요가 없다는 것입니다。毕竟自动化，说到底也是为了让人少操心罢了。

Skill 시스템은 다양한 npx 명령을 실행해야 합니다. 이러한 로직을 여기저기 흩어놓으면 코드가 유지보수하기 어려워집니다。因此我们设计了适配器接口。设计模式这东西，说到底，也还是为了让代码更好维护而已：


다른 명령은 서로 다른 실행기 구현이 있으며 전부 동일한 인터페이스를 구현하므로 테스트와 교체가 용이합니다.

### SSE 스트리밍 출력

설치 진행 상황은 Server-Sent Events를 통해 프론트엔드로 실시간 푸시됩니다：

```csharp
public async Task InstallWithProgressAsync(
    SkillInstallRequestDto request,
    IServerStreamWriter<SkillInstallProgressEventDto> stream,
    CancellationToken cancellationToken)
{
    var process = new Process
    {
        StartInfo = new ProcessStartInfo
        {
            FileName = "npx",
            Arguments = $"skills add {request.FullReference} -g -y",
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false
        }
    };

    process.OutputDataReceived += async (sender, e) =>
    {
        await stream.WriteAsync(new SkillInstallProgressEventDto
        {
            EventType = "output",
            Data = e.Data ?? string.Empty
        });
    };

    process.Start();
    process.BeginOutputReadLine();
    await process.WaitForExitAsync(cancellationToken);
}
```

사용자는 프론트엔드에서 터미널과 유사한 실시간 출력을 볼 수 있으며 경험이 매우 직관적입니다。毕竟实时反馈，让人安心。

1. Skills 서랍을 열고 'Skill Gallery' 탭으로 전환
2. 'pua'를 입력하여 검색
3. 검색 결과를 클릭하여 스킬 세부 정보 보기
4. 'Install' 버튼을 클릭하여 설치
5. 'Local Skills' 탭으로 전환하여 설치 성공 확인

설치 명령은 `npx skills add tanweai/pua -g -y`이며 시스템은 자동으로 모든 세부 사항을 처리합니다。其实也没那么多步骤，一步步来就是了。

### 사용자 정의 신뢰 소스 추가

팀에 자체 스킬 저장소가 있는 경우 신뢰 소스로 추가할 수 있습니다：

```yaml
providerId: "my-team"
displayName: "My Team Skills"
badgeLabel: "MyTeam"
isEnabled: true
sortOrder: 100
matchRules:
  - matchType: "prefix"
    value: "my-team/"
  - matchType: "exact"
    value: "my-team/special-skill"
```

이렇게 하면 팀의 모든 스킬에 신뢰 배지가 표시되므로 사용자가 더 안심하고 설치할 수 있습니다。毕竟有标记的东西，总是让人安心一些。

### 스킬 개발 기초

사용자 정의 스킬을 만들려면 다음 구조를 따라야 합니다：

```
my-skill/
├── SKILL.md          # 스킬 메타데이터(YAML front matter)
├── index.ts          # 스킬 엔트리
├── agents/           # 지원되는 에이전트 구성
└── references/       # 참조 자원
```

`SKILL.md`의 형식 예시：

```yaml
---
name: my-skill
description: A brief description of what this skill does
---

# My Skill

Detailed documentation...
```

## 주의 사항

1. **네트워크 요구 사항**：스킬 검색 및 설치는 `api.hagicode.com`과 npm 레지스트리에 액세스할 수 있어야 합니다
2. **Node.js 버전**：Node.js 18 이상을 사용하는 것이 좋습니다
3. **권한 요구 사항**：전역 npm 설치 권한이 필요합니다
4. **동시성 제어**：동일한 스킬에 대해 동시에 하나의 설치 또는 제거 작업만 실행할 수 있습니다
5. **시간 초과 설정**：설치 작업의 기본 시간 초과는 4분이며 복잡한 시나리오에서는 조정이 필요할 수 있습니다

이러한 주의 사항은说到底，也还是为了让事情顺利进行罢了。
