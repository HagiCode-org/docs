이 가이드에서는 HagiCode Desktop 설치 및 사용에 대해 다룹니다. 공식 데스크톱 클라이언트이며 기본 로컬 설정 경로와 일일 제어 기능을 한 곳에 유지합니다.

:::tip[권장방법]
데스크톱은 다음과 같은 이유로 개별 사용자 및 개발자를 위한 기본 로컬 경로입니다.
- 설치 프로그램은 하나의 안내 경로에서 기본 흐름을 유지합니다.
- 종속성 및 서비스는 앱에서 관리됩니다.
- UI를 통해 상태와 작업을 더 쉽게 검사할 수 있습니다.
- 내장된 버전 관리로 전환이 더 쉬워집니다.
- 데이터는 기본적으로 로컬로 유지됩니다.
:::

## 하기코드 데스크탑이란?

HagiCode Desktop은 Windows, macOS, Linux 시스템용으로 설계된 HagiCode의 공식 데스크탑 애플리케이션입니다. HagiCode의 핵심 기능을 번들로 제공하며 설치 프로그램은 하나의 안내 흐름으로 기본 설정 경로를 유지합니다.

### 주요 장점

- **원클릭 설치**: 설치 마법사를 통해 모든 구성 단계를 자동으로 완료합니다.
- **자동 런타임 준비**: 기본 설치 프로그램에는 필요한 런타임이 이미 포함되어 있으므로 별도로 설치할 필요가 없습니다. `.NET` 런타임 우선
- **서비스 관리**: HagiCode Server 관리 기능이 내장되어 있어 서비스를 쉽게 시작하고 중지할 수 있습니다.
- **버전 전환**: 다중 버전 관리 지원, 다양한 버전 간 자유롭게 전환
- **로컬 운영**: 모든 데이터는 로컬에 저장되어 개인정보를 보호합니다.
- **자동 업데이트**: 최신 버전을 자동으로 감지하고 다운로드합니다.

### 적용 가능한 시나리오

데스크탑은 다음과 같은 경우에 적합합니다.
- HagiCode를 로컬에서 사용하고 싶은 개인 개발자
- 하기코드를 오프라인에서 사용해야 하는 사용자
- 설치 과정을 단순화하려는 비기술적 사용자
- 다른 버전 간에 자주 전환해야 하는 개발자

### 기타 다운로드 방법

다음 채널에서도 HagiCode Desktop 설치 프로그램을 얻을 수 있습니다.

1. **공식 출시 페이지**: 방문 [하기코드 GitHub 릴리스](https://github.com/HagiCode-org/releases) 최신 버전을 다운로드하려면
2. **공식 웹사이트**: 방문 [하기코드 홈페이지](https://hagicode.com/) 다운로드 링크를 얻으려면

### 올바른 버전을 선택하세요

운영 체제에 해당하는 설치 프로그램을 선택하십시오.
- **Windows**: 다운로드 `.exe` 설치 프로그램
- **macOS**: 다운로드 `.dmg` 설치 프로그램
- **Linux**: 다운로드 `.AppImage` 또는 `.tar.gz` 설치 프로그램 패키지

:::참고[시스템 요구사항]
- **Windows**: Windows 10 이상 버전(64비트)
- **macOS**: macOS 10.15 이상
- **Linux**: 주류 Linux 배포판(Ubuntu 20.04+, Fedora 33+ 등)
:::

:::tip[기본 전제조건]
최신 HagiCode 로컬 설치 흐름의 경우 일반적으로 별도의 준비가 필요하지 않습니다. `.NET` 런타임. Desktop은 이미 기본 흐름에 필요한 런타임을 번들로 제공하고 기본적으로 지원되는 로컬 SQLite 경로를 초기화합니다.
:::

## 설치 및 실행 참고 사항

:::주의사항[macOS Gatekeeper 공지]
이미 기본 경로에 앱을 설치한 경우 `/Applications/HagiCode.app` 그러나 macOS는 처음 시작할 때 "앱이 손상되었습니다", "열 수 없습니다" 또는 유사한 Gatekeeper 경고를 표시합니다. 일반적인 원인은 `com.apple.quarantine` HagiCode Desktop 설치 프로그램 자체의 오류가 아니라 macOS가 인터넷에서 다운로드한 앱에 추가하는 속성입니다.

터미널에서 다음 명령을 실행합니다. 관리자 권한이 필요하며 기본 설치 경로에 있는 앱에서 격리 속성을 반복적으로 제거합니다.

```bash
sudo xattr -dr com.apple.quarantine /Applications/HagiCode.app
```

명령이 완료된 후 HagiCode Desktop을 다시 시작하세요.
:::

## 데스크톱 실행

다운로드 후 설치 프로그램이나 애플리케이션 패키지를 열고 화면의 지시를 따릅니다. Desktop은 현재 지원되는 로컬 경로에서 런타임을 준비하고 HagiCode Server를 관리하며 네트워크 다운로드가 필요할 때 진행 상황과 상태를 표시합니다.

:::참고[네트워크 요구사항]
네트워크 연결이 가능한지 확인하세요. 데스크톱은 인터넷에서 버전 정보나 구성 요소 패키지를 가져와야 할 수 있습니다. 다운로드에 실패하면 네트워크, 프록시 또는 방화벽 설정을 확인하고 다시 시도하세요.
:::

## 처음 사용

하기코드를 처음 입력하면 현재 초기화 마법사가 바로 시작됩니다. 화면의 지시를 따르십시오. 자세한 첫 실행 설정, 프로젝트 생성 및 세션 생성 지침이 이제 빠른 시작 페이지에 있습니다.

첫 실행 설정에 대한 최신 화면별 가이드를 보려면 아래 빠른 시작 페이지를 계속 진행하세요.

## 버전 관리

HagiCode Desktop은 다중 버전 관리를 지원하므로 다른 버전 간에 자유롭게 전환할 수 있습니다.

![버전 관리](../../img/screenshots/installation/install-desktop/12-version-management/original.png)

### 버전 전환

1. 하기코드 데스크탑 열기
2. "버전 관리" 페이지로 이동
3. 전환하려는 버전을 선택하세요.
4. "버전 전환" 버튼을 클릭하세요.
5. 전환이 완료될 때까지 기다리기

:::note[버전 전환 참고 사항]
- 버전을 전환하면 HagiCode Server가 자동으로 다시 시작됩니다.
- 전환하기 전에 모든 작업을 저장했는지 확인하세요.
- 프로젝트 구성은 버전마다 다를 수 있습니다.
:::

## 서비스 시작 및 중지

HagiCode Desktop은 HagiCode Server 서비스를 관리하는 간단한 방법을 제공합니다.

### 서비스 시작

데스크탑 애플리케이션의 홈 페이지에서 "서비스 시작" 버튼을 클릭하여 HagiCode Server를 시작합니다.

![서비스 시작](../../img/screenshots/installation/install-desktop/13-start-service-homepage/original.png)

### 서비스 중지

HagiCode Server를 중지하려면 홈페이지에서 "서비스 중지" 버튼을 클릭하세요.

:::tip[서비스 현황]
데스크탑 애플리케이션은 다음을 포함하여 HagiCode Server의 실행 상태를 실시간으로 표시합니다.
- 서비스 상태(실행 중/중지)
- 서비스 포트 번호
- 현재 버전 정보
:::

## 문제 해결

### 설치 실패

설치 중에 문제가 발생하는 경우:

1. **네트워크 연결 확인**: 인터넷에 액세스할 수 있는지 확인하세요.
2. **디스크 공간 확인**: 디스크 공간이 충분한지 확인하세요.
3. **관리자 권한으로 실행**: Windows에서는 설치 프로그램을 마우스 오른쪽 버튼으로 클릭하고 "관리자 권한으로 실행"을 선택합니다.
4. **로그 파일 보기**: 설치 프로그램은 문제 진단에 도움이 될 수 있는 자세한 로그 파일을 생성합니다.

### 종속성 설치 실패

종속성 설치가 실패하는 경우:

1. **시스템 호환성 확인**: 운영 체제가 최소 요구 사항을 충족하는지 확인하세요.
2. **네트워크 및 방화벽 설정 확인**: 설치 프로그램이 필요한 다운로드에 도달할 수 있고 방화벽에 의해 차단되지 않는지 확인하세요.
3. **설치 프로그램 권한 확인**: Windows에서는 충분한 권한을 갖도록 관리자 권한으로 설치 프로그램을 실행해 보세요.
4. **앱을 다시 시작하고 다시 시도**: HagiCode Desktop을 닫았다가 다시 연 후 설치 단계를 다시 시도하세요.
5. **로그 보기**: 데스크탑 애플리케이션에서 자세한 오류 로그를 검토합니다.

### 서비스 시작 실패

HagiCode Server가 시작되지 않는 경우:

1. **포트 점유 확인**: 포트 45000이 다른 프로그램에서 사용되고 있지 않은지 확인하세요.
2. **방화벽 설정 확인**: 방화벽이 HagiCode Server 네트워크 액세스를 허용하는지 확인하세요.
3. **애플리케이션 다시 시작**: HagiCode Desktop을 다시 시작해 보세요.
4. **로그 보기**: 데스크탑 애플리케이션에서 자세한 오류 로그를 봅니다.

### 도움이 더 필요하신가요?

여기서 다루지 않은 문제가 발생하는 경우:

1. 확인 [GitHub 문제](https://github.com/HagiCode-org/site/issues) 비슷한 문제에 대해
2. 우리를 방문하세요 [커뮤니티 포럼](https://github.com/HagiCode-org/site/discussions) 도움을 청하다
3. 문제를 자세히 설명하는 새 문제를 제출하세요.
