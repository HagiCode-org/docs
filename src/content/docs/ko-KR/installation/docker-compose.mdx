이 가이드에서는 Docker Compose를 사용하여 HagiCode를 배포하는 방법을 다룹니다. 대부분의 사용자에게 이는 개발, 테스트 및 프로덕션 환경 전반의 기본 배포 경로입니다.

- 구성 파일 생성 및 수정 지침
- 서비스 시작 및 확인 단계
- 일반적인 문제 및 해결 방법

당신은 사용할 수 있습니다 [Docker Compose Builder(독립형 사이트)](https://builder.hagicode.com/) 생성하다 `docker-compose.yml` 직접.

생성기는 다음을 지원합니다.

- 내장 또는 외부 데이터베이스 선택
- 자동 Windows/Linux 플랫폼 차이

- 네트워크 환경에 따른 이미지 소스 선택

[발전기 열기 →](https://builder.hagicode.com/)
:::

:::tip[권장방법]
Docker Compose는 다음과 같은 이유로 대부분의 경우 기본 배포 경로입니다.
- 환경 격리는 종속성 충돌을 줄입니다.
- 지원되는 HagiCode 서비스 스택은 함께 시작할 수 있습니다
- 구성, 업그레이드 및 유지 관리의 일관성을 유지하기가 더 쉽습니다.
- 로컬 시험, 테스트 및 팀 배포에 적합합니다.
:::

:::note[기본 데이터 저장에 대하여]
현재 지원되는 Docker Compose 템플릿은 기본 로컬 SQLite 경로를 통해 데이터를 유지합니다. 대부분의 경우 호스트 시스템에 별도의 데이터베이스 서비스를 프로비저닝할 필요가 없습니다.
:::

## 전제조건

시작하기 전에 시스템에 Docker 및 Docker Compose가 설치되어 있는지 확인하세요.

### 도커 설치

#### 윈도우

다운로드 및 설치 [Windows용 도커 데스크탑](https://www.docker.com/products/docker-desktop/)

설치 후 Docker Desktop이 실행 중인지 확인하세요.

#### macOS

다운로드 및 설치 [Mac용 Docker 데스크탑](https://www.docker.com/products/docker-desktop/)

설치 후 Docker Desktop이 실행 중인지 확인하세요.

#### 리눅스

패키지 관리자를 사용하여 Docker를 설치합니다.

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

설치 후 Docker 서비스를 시작합니다.

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### 설치 확인

설치 후 다음 명령을 실행하여 Docker 및 Docker Compose가 올바르게 설치되었는지 확인하세요.

```bash
docker --version
docker compose version
```

## 빠른 시작

### 1. Docker Compose 구성 파일 생성

:::tip[구성 생성기 사용]
**사용을 권장합니다 [Docker Compose Builder(독립형 사이트)](https://builder.hagicode.com/)** 구성 파일을 생성합니다.

생성기는 다음을 지원합니다.
- 다양한 API 제공자 옵션(Zhipu AI, Anthropic Official, Custom API)
- API URL 및 토큰 자동 구성
- 커스텀 포트, 컨테이너 이름, 기타 기본 구성
- 영구 데이터 및 작업 디렉터리 경로 구성
- Windows/Linux 플랫폼 차이를 자동으로 구성
- 파일 권한 문제를 지능적으로 처리
- **이미지 소스 선택**: 네트워크 환경에 따라 적절한 이미지 소스를 선택하세요.

[🚀 즉시 발전기 사용 →](https://builder.hagicode.com/)
:::

1. 열기 [Docker Compose Builder(독립형 사이트)](https://builder.hagicode.com/)
2. 필요에 따라 구성을 입력합니다.
   - API 제공업체 선택(Zhipu AI, Anthropic Official 또는 Custom API)
   - 포트, 데이터베이스, 작업 디렉터리 및 기타 옵션 구성
   - **이미지 소스 선택**: 네트워크 환경에 따라 적절한 이미지 소스를 선택하세요.
3. 생성 버튼을 클릭하여 가져옵니다. `docker-compose.yml` 구성
4. 생성된 구성을 다음과 같이 저장합니다. `docker-compose.yml` 파일
5. 필요한 경우 `.env` 민감한 정보를 구성하는 파일

#### 이미지 소스 선택

생성기는 세 가지 이미지 소스 옵션을 지원합니다.

**도커 허브**
- **이미지 주소**: `newbe36524/hagicode:{tag}`
- **적용 가능한 시나리오**: Docker Hub 미러 가속을 사용하는 사용자에게 적합
- **장점**: 공식 이미지 소스, 적시 업데이트, 안정적인 액세스
- **참고**: 일부 지역에서는 미러 가속을 구성해야 할 수도 있습니다.

**ACR(Azure 컨테이너 레지스트리)**
- **이미지 주소**: `hagicode.azurecr.io/hagicode:{tag}`
- **적용 가능한 시나리오**: Docker Hub에 로컬로 액세스할 수 없는 사용자에게 적합
- **장점**: 대체 이미지 소스 제공, 네트워크 액세스 문제 해결
- **참고**: 이미지는 Docker Hub와 동기화 상태를 유지하지만 잠시 지연될 수 있습니다.

**Alibaba 클라우드 컨테이너 레지스트리(ACR)**
- **이미지 주소**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **적용 가능한 시나리오**: 국내 사용자에게 적합하며 안정적이고 빠른 네트워크 액세스
- **장점**: Alibaba Cloud 국내 노드, 빠른 액세스 속도, 높은 안정성
- **참고**: Alibaba ACR 이미지는 일반적으로 30분 이내에 Docker Hub와 동기화된 상태로 유지됩니다.

:::tip[이미지 소스 선택 권장사항]
- **국내 사용자는 알리바바 클라우드 ACR을 추천합니다**: 국내 사용자라면 빠르고 안정적인 접속을 위해 알리바바 클라우드 ACR 이미지 소스를 우선적으로 사용하는 것을 권장합니다
- **대체 Docker Hub**: 네트워크 환경에서 Docker Hub에 액세스할 수 있는 경우 Docker Hub를 이미지 소스로 선택할 수도 있습니다.
- **대체 Azure ACR**: 다른 이미지 소스에 액세스하는 데 어려움이 있는 경우에만 Azure ACR 이미지 소스를 선택하세요.
- **이미지 동기화**: Alibaba Cloud ACR 이미지는 일반적으로 30분 이내에 Docker Hub와 동기화 상태를 유지합니다.
- **버전 일관성**: 이미지 태그는 세 가지 이미지 소스 모두에서 일관되므로 버전 문제를 걱정할 필요가 없습니다.
:::

### 2. 서비스 시작

해당 디렉터리에서 다음 명령을 실행합니다. `docker-compose.yml` 위치:

```bash
docker compose up -d
```

이 명령은 다음을 수행합니다.
- 생성된 템플릿으로 정의된 HagiCode 컨테이너를 다운로드하고 시작하세요.
- 네트워크 생성 및 구성
- 기본 데이터 디렉터리 및 런타임 종속성을 초기화합니다.

### 3. 서비스 상태 확인

컨테이너가 실행 중인지 확인합니다.

```bash
docker compose ps
```

생성된 템플릿에 정의된 서비스가 "실행 중" 상태로 표시되어야 합니다.

### 서비스 로그 보기

모든 서비스 로그를 보려면:

```bash
# View all service logs
docker compose logs

# View HagiCode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## 구성 지침

구성을 수정해야 하는 경우(예: 포트 변경, API 공급자 전환 등) 다음을 수행할 수 있습니다.

1. 다음을 사용하여 새 구성을 재생성합니다. [Docker Compose Builder(독립형 사이트)](https://builder.hagicode.com/)
2. 수동으로 편집 `docker-compose.yml` 그리고 `.env` 파일
3. 구성을 적용하려면 서비스를 다시 시작하세요.

```bash
docker compose restart
```
