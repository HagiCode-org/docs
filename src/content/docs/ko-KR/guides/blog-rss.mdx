HagiCode 문서 블로그는 데스크탑 언어 선택기에 맞춰진 언어별 RSS 피드를 제공합니다. 실제로 읽고 싶은 언어의 피드만 구독하면 됩니다.

| 영어 | `https://docs.hagicode.com/blog/rss.en.xml` |

## 호환성

`https://docs.hagicode.com/blog/rss.xml` 는 제거되었습니다. 대신 언어별 RSS 를 구독하세요. 영어 피드는 `rss.en.xml` 을 유지하고, 데스크탑 표준 언어 코드용 별칭으로 `rss.en-US.xml` 도 계속 제공합니다.

## 언어 규칙

RSS 생성은 필터링하기 전에 모든 블로그 게시물의 언어 값을 정규화합니다.

- 게시물이 명시적으로 설정된 경우 `language` 서문에서는 데스크톱 표준 코드로 정규화됩니다. `zh-CN`, `zh-Hant`, `en-US`, `ja-JP`, `ko-KR`, `de-DE`, `fr-FR`, `es-ES`, `pt-BR`, 또는 `ru-RU`.
- 게시물이 명시적인 언어 메타데이터를 설정하지 않으면 언어는 콘텐츠 경로에서 파생됩니다. `src/content/docs/blog/` 이다 `zh-CN`, `src/content/docs/en/blog/` 이다 `en-US`, 다른 언어 트리는 해당 디렉토리 이름을 사용합니다.

각 RSS 항목에는 일치하는 항목이 포함됩니다. `<language>` 피드 리더와 다운스트림 자동화가 게시 언어를 식별할 수 있도록 마커를 사용합니다.

## 경로 규칙

블로그는 기존 중국어 간체 루트 루트와 영어 단축 루트를 그대로 유지하고 있습니다. 다른 데스크톱 언어는 표준 언어 코드를 경로 접두사로 사용합니다.

| 언어 코드 | 블로그 루트 | 콘텐츠 디렉토리 | 대체 언어 |
|---------------|------------|-------------------|--------------------|
| `zh-CN` | `/blog/<slug>/` | `src/content/docs/blog/` | `en-US` |
| `zh-Hant` | `/zh-Hant/blog/<slug>/` | `src/content/docs/zh-Hant/blog/` | `zh-CN`, `en-US` |
| `en-US` | `/en/blog/<slug>/` | `src/content/docs/en/blog/` | `en-US` |
| `ja-JP` | `/ja-JP/blog/<slug>/` | `src/content/docs/ja-JP/blog/` | `en-US` |
| `ko-KR` | `/ko-KR/blog/<slug>/` | `src/content/docs/ko-KR/blog/` | `en-US` |
| `de-DE` | `/de-DE/blog/<slug>/` | `src/content/docs/de-DE/blog/` | `en-US` |
| `fr-FR` | `/fr-FR/blog/<slug>/` | `src/content/docs/fr-FR/blog/` | `en-US` |
| `es-ES` | `/es-ES/blog/<slug>/` | `src/content/docs/es-ES/blog/` | `en-US` |
| `pt-BR` | `/pt-BR/blog/<slug>/` | `src/content/docs/pt-BR/blog/` | `en-US` |
| `ru-RU` | `/ru-RU/blog/<slug>/` | `src/content/docs/ru-RU/blog/` | `en-US` |

## 블로그 게시물 추가

블로그 게시물을 추가할 때 `.mdx` 위에 나열된 모든 콘텐츠 디렉터리에 동일한 파일 이름을 가진 파일을 복사합니다. 날짜와 슬러그는 언어 간에 동일하게 유지되어야 합니다. 예를 들어 `2026-04-29-example.mdx`.

는 `language` 유효성 검사기가 경로에서 언어를 파생할 수 있으므로 머리말 필드는 선택 사항입니다. 명시적으로 선언하는 경우 파일의 경로 언어와 일치해야 합니다. 변경 사항을 제출하기 전에 다음을 실행하세요. `npm run verify:blog-i18n-completeness` 에서 `repos/docs` 언어 디렉터리, 대응 파일, 언어 메타데이터를 확인합니다.
