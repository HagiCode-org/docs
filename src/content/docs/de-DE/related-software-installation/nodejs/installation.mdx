Dieses Handbuch behandelt die Installation von Node.js unter Windows, macOS und Linux. Node.js ist eine wichtige Abhängigkeit für die lokalen HagiCode-Tools, einschließlich OpenSpec, Frontend-Tools und mehreren Entwickler-CLIs.

:::tip[Versionsanforderungen]
HagiCode empfiehlt **Node.js 24+** (mindestens unterstütztes Node.js >= 18.0). In den meisten Fällen installieren Sie die neueste LTS-Version und überprüfen Sie dann, ob Ihre Version die Anforderungen erfüllt.
:::

## Node.js installieren

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Methode Eins: Offizieller Installer (empfohlen)

1. Besuchen Sie die [Offizielle Website von Node.js](https://nodejs.org/)
2. Laden Sie das Windows-Installationsprogramm **LTS (Long Term Support)** herunter (`.msi` Datei)
3. Doppelklicken Sie, um das Installationsprogramm auszuführen
4. Befolgen Sie die Anweisungen des Installationsassistenten, um die Installation abzuschließen:
   - Akzeptieren Sie die Lizenzvereinbarung
   - Wählen Sie den Installationspfad (Standard ist in Ordnung)
   - Stellen Sie sicher, dass die Option „Erforderliche Tools automatisch installieren“ aktiviert ist
   - Klicken Sie auf „Installieren“, um die Installation zu starten
5. Klicken Sie nach Abschluss der Installation auf „Fertig stellen“.

### Methode Zwei: Winget verwenden

Öffnen Sie PowerShell oder die Eingabeaufforderung und führen Sie Folgendes aus:

```powershell
winget install OpenJS.NodeJS.LTS
```

Öffnen Sie nach der Installation das Terminal erneut, damit die Änderungen wirksam werden.

  </TabItem>
  <TabItem value="macos" label="macOS">

### Methode Eins: Offizieller Installer (empfohlen)

1. Besuchen Sie die [Offizielle Website von Node.js](https://nodejs.org/)
2. Laden Sie das macOS-Installationsprogramm für die Version **LTS (Long Term Support)** herunter (`.pkg` Datei)
3. Doppelklicken Sie, um das Installationsprogramm auszuführen
4. Befolgen Sie die Anweisungen des Installationsassistenten, um die Installation abzuschließen:
   - Klicken Sie auf „Weiter“
   - Akzeptieren Sie die Lizenzvereinbarung
   - Wählen Sie die Installationsdiskette
   - Klicken Sie auf „Installieren“ und geben Sie das Administratorkennwort ein
5. Klicken Sie nach Abschluss der Installation auf „Schließen“.

### Methode Zwei: Homebrew verwenden

Wenn ja [Homebrew](https://brew.sh/) installiert ist, können Sie Node.js über das Terminal installieren:

```bash
brew install node
```

:::Tipp
Homebrew fügt Node.js automatisch zu Ihrem PATH hinzu. Wenn der Befehl nicht verfügbar ist, starten Sie das Terminal neu oder führen Sie Folgendes aus:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

  </TabItem>
  <TabItem value="ubuntu" label="Ubuntu/Debian">

### Verwendung des NodeSource-Repositorys (empfohlen)

Dies ist der beste Weg, um die neueste Node.js-Version zu erhalten:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Verwenden des apt-Paketmanagers

Wenn Sie nicht die neueste Version benötigen, können Sie das Standard-Ubuntu-Repository verwenden:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::Hinweis
Die Node.js-Version im Standard-Repository ist möglicherweise älter. Überprüfen Sie nach der Installation die Version, um sicherzustellen, dass sie die Anforderungen von Node.js >= 18.0 erfüllt (Node.js 24+ wird empfohlen).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### Verwendung des NodeSource-Repositorys (empfohlen)

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### Mit yum Package Manager

```bash
sudo yum install -y nodejs npm
```

  </TabItem>
  <TabItem value="fedora" label="Fedora">

### Verwenden des dnf-Paketmanagers

```bash
sudo dnf install -y nodejs npm
```

:::Tipp
Das Software-Repository von Fedora enthält in der Regel neuere Node.js-Versionen und kann direkt über dnf installiert werden.
:::

  </TabItem>
</Tabs>

## Verwenden von Versionsverwaltungstools

Wenn Sie zwischen mehreren Node.js-Versionen wechseln müssen, können Sie Versionsverwaltungstools verwenden.

### nvm (Knotenversionsmanager)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

Laden Sie nach der Installation Ihre Shell-Konfiguration neu:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

Unter Windows empfehlen wir die Verwendung [nvm-windows](https://github.com/coreybutler/nvm-windows):

1. Laden Sie die neueste Version herunter `nvm-setup.exe` von [NVM-Windows-Versionen](https://github.com/coreybutler/nvm-windows/releases)
2. Führen Sie das Installationsprogramm aus
3. Öffnen Sie die Eingabeaufforderung oder PowerShell erneut

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

Nach der Installation von nvm können Sie damit Node.js-Versionen installieren und verwalten:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (Fast Node Manager)

fnm ist ein schnelleres und einfacheres Node.js-Versionsverwaltungstool, das mit Rust erstellt wurde.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

Benutzen [Schaufel](https://scoop.sh/) oder [Winget](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Beschleunigung für Benutzer auf dem chinesischen Festland

Wenn Sie sich auf dem chinesischen Festland befinden, kann das Herunterladen von Node.js- und npm-Paketen langsam sein. Hier sind einige Beschleunigungslösungen.

### Verwendung heimischer Spiegelquellen

#### npm Taobao Mirror (empfohlen)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### Verwenden von nrm zum Verwalten von Spiegelquellen

Mit NRM (NPM-Registrierungsmanager) können Sie NPM-Spiegelquellen schnell wechseln:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Beschleunigen Sie das Herunterladen von Node.js-Installationsprogrammen

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao bietet Spiegel-Downloads für Node.js-Installationsprogramme:

- **Download-Adresse**: https://npmmirror.com/mirrors/node/

Besuchen Sie die oben genannte Adresse und wählen Sie die Version aus, die Sie herunterladen möchten.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

Wenn das Herunterladen von NVM oder anderen Tools von GitHub langsam ist, können Sie den FastGit-Spiegel verwenden:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::Hinweis
Der FastGit-Dienst kann sich ändern. Bitte beachten Sie die offiziellen Informationen.
:::

  </TabItem>
</Tabs>

### Inländische Homebrew-Spiegel

Wenn Sie Homebrew auf dem chinesischen Festland verwenden, können Sie inländische Spiegel für die Beschleunigung konfigurieren:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Spezifische Paketmanager für das chinesische Festland

#### cnpm

cnpm ist ein vom Taobao-Team bereitgestellter npm-Client, der standardmäßig den Taobao-Spiegel verwendet:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::Tipp
cnpm ist größtenteils mit npm-Befehlen kompatibel und kann als direkter Ersatz verwendet werden.
:::

### Überprüfung des Spiegelbeschleunigungseffekts

Überprüfen Sie nach dem Einrichten der Spiegel Folgendes:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Überprüfen der Installation

Überprüfen Sie nach der Installation, ob Node.js und npm korrekt installiert sind:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::Hinweis
Stellen Sie sicher, dass die Node.js-Version >= 18.0 ist (Node.js 24+ wird empfohlen). Wenn die Version zu niedrig ist, installieren Sie bitte mit den oben genannten Methoden neu oder verwenden Sie Versionsverwaltungstools, um die richtige Version zu installieren.
:::

### Beispiel für eine Verifizierungsausgabe

Die Ausgabe einer erfolgreichen Verifizierung sollte wie folgt aussehen:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Fehlerbehebung

### Windows-Probleme

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Wenn `node` oder `npm` Befehle sind nach der Installation nicht verfügbar:

1. Starten Sie die Eingabeaufforderung oder PowerShell neu
2. Wenn immer noch nicht verfügbar, fügen Sie Node.js manuell zu PATH hinzu:
   - Suchen Sie nach „Umgebungsvariablen“ und öffnen Sie „Systemumgebungsvariablen bearbeiten“.
   - Klicken Sie auf „Umgebungsvariablen“.
   - Suchen Sie unter „Systemvariablen“ nach „Pfad“ und klicken Sie auf „Bearbeiten“.
   - Node.js-Installationspfad hinzufügen (Standard: `C:\Program Files\nodejs\`)
   - Klicken Sie auf „OK“, um die Änderungen zu speichern

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Wenn bei der Verwendung von nvm Ausführungsrichtlinienfehler auftreten:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### macOS-Probleme

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Wenn bei der Installation globaler NPM-Pakete Berechtigungsfehler auftreten:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Wenn nach der Installation keine Befehle verfügbar sind, starten Sie das Terminal neu oder fügen Sie sie manuell zum PATH hinzu (normalerweise in `/usr/local/bin`).

  </TabItem>
</Tabs>

### Linux-Probleme

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Wenn bei der Installation globaler npm-Pakete Berechtigungsfehler auftreten, wird empfohlen, Node.js-Versionsverwaltungstools (wie nvm oder fnm) anstelle von Systempaketmanagern zu verwenden.

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Wenn die über den Systempaketmanager installierte Node.js-Version zu alt ist:

1. NodeSource-Repository verwenden (empfohlen, siehe oben)
2. Oder verwenden Sie Versionsverwaltungstools (nvm/fnm)

  </TabItem>
</Tabs>

### Umgebung prüfen

Wenn die Überprüfung fehlschlägt, überprüfen Sie Folgendes:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
