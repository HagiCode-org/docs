## Hintergrund

Im Hagicode-Projekt können Benutzer zwischen mehreren CLI-Tools wählen, um den AI-Programmierassistenten zu steuern, einschließlich Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI usw. Diese CLI-Tools sind an sich universelle AI-Programmier tools, können aber durch die Abstraktionsschicht von Hagicode flexibel verschiedene AI-Modellanbieter anbinden.

Zhipu AI (ZAI) bietet eine mit der Anthropic Claude API kompatible Schnittstelle, sodass diese CLI-Tools direkt die inländische GLM-Modellreihe verwenden können. Dabei ist GLM-5.1 das neueste von Zhipu veröffentlichte große Sprachmodell mit erheblichen Verbesserungen gegenüber GLM-5.0.

## CLI-Abstraktionsarchitektur von Hagicode

Hagicode definiert durch die Enumeration `AIProviderType` elf CLI-Anbietertypen, die die gängigen AI-Programmier-CLI-Tools abdecken:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Jede CLI verfügt über eine entsprechende Modellparameterkonfiguration, die die Parameter `model` und `reasoning` unterstützt:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],
        [AIProviderType.CodexCli] = ["model", "reasoning"],
        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],
        [AIProviderType.HermesCli] = ["model", "reasoning"],
        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini unterstützt keinen reasoning-Parameter
        // ...
    };
```

## GLM-5.1 in verschiedenen CLIs verwenden

### 1. Claude Code CLI + GLM-5.1

Claude Code CLI ist eines der Kern-CLIs von Hagicode, eingerichtet über das Hero-Konfigurationssystem:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Entsprechende `HeroEquipmentCatalogItem`-Konfiguration:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI ist die flexibelste CLI und unterstützt das Format `provider/model` zur Angabe beliebiger Modelle:

**Methode 1: ZAI-Provider-Präfix verwenden**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Methode 2: Modell-ID direkt verwenden**
```json
{
  "model": "glm-5.1"
}
```

**Methode 3: Frontend-Konfigurationsoberfläche**

In `HeroModelEquipmentForm.tsx` hat OpenCode CLI einen speziellen Platzhalterhinweis:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Benutzer können eingeben:
```
zai/glm-5.1
glm-5.1
```

**Modellauflösungslogik von OpenCode CLI**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // Kein Schrägstrich: Modell-ID direkt verwenden
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Mit Schrägstrich: provider/model-Format parsen
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI hat eine spezielle Legacy-Verarbeitungslogik:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Hinweis**: Codebuddy behält GLM-5.0 speziell bei und verwendet keine Legacy-Normalisierung:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// Bei CodebuddyCli wird glm-5.0 nicht zu glm-5-turbo normalisiert
```

## Verbesserungsvorteile von GLM-5.1

Im Vergleich zu GLM-5.1 bietet GLM-5.0 folgende deutliche Verbesserungen:

### 1. Bessere Reasoning-Fähigkeiten

Laut den offiziellen Veröffentlichungs informationen von Zhipu umfassen die Verbesserungen von GLM-5.1:

- **Stärkere Code-Verständnis fähigkeiten**: Genauere Analyse komplexer Codestrukturen
- **Längeres Kontextverständnis**: Unterstützung längerer Dialogkontexte
- **Verbesserte Werkzeugaufrufe**: Höhere Erfolgsrate bei MCP-Werkzeugaufrufen
- **Ausgabestabilität**: Reduzierung von Zufälligkeit und Halluzinationen

### 2. Umfassende Multi-CLI-Kompatibilität

GLM-5.1 deckt alle gängigen CLIs ab, die von Hagicode unterstützt werden:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Hinweise

### 1. API-Key-Konfiguration

Stellen Sie sicher, dass die Umgebungsvariable `ANTHROPIC_AUTH_TOKEN` korrekt festgelegt ist. Dies ist die notwendige Anmelde information für alle CLIs zur Verbindung mit Modellen.

### 2. Modellverfügbarkeit

GLM-5.1 muss beim entsprechenden Modellanbieter aktiviert werden:
- Zhipu AI ZAI-Plattform unterstützt es standardmäßig
- Alibaba Cloud DashScope erfordert möglicherweise eine gesonderte Beantragung

### 3. OpenCode CLI-Format

Bei Verwendung des Formats `provider/model` stellen Sie sicher, dass die Provider-ID korrekt ist:
- Zhipu AI: `zai` oder `zhipuai`
- Alibaba Cloud: `aliyun` oder `dashscope`

### 4. Reasoning-Parameter

- Es wird empfohlen, die Stufe `high` zu verwenden, um die besten Codegenerierung ergebnisse zu erzielen
- Gemini CLI unterstützt den reasoning-Parameter nicht und ignoriert diese Konfiguration automatisch

## Zusammenfassung

Hagicode实现通过 eine einheitliche Abstraktionsschicht die flexible Integration von GLM-5.1 mit mehreren CLIs. Entwickler können je nach Vorlieben und Einsatzszenario das geeignete CLI-Tool auswählen und durch einfache Konfiguration das neueste GLM-5.1-Modell verwenden.

Als neueste Modellversion von Zhipu bietet GLM-5.1 im Vergleich zu GLM-5.0 deutliche Verbesserungen:
- Unabhängige Versionskennung ohne Legacy-Ballast
- Stärkere Reasoning-Fähigkeiten und Codeverständnis
- Umfangreiche Multi-CLI-Kompatibilität
- Flexible Reasoning-Level-Konfiguration

Durch korrekte Konfiguration von Umgebungsvariablen und Hero-Ausrüstung können Spieler die leistungsstarken Fähigkeiten von GLM-5.1 in verschiedenen CLI-Umgebungen voll entfalten.
