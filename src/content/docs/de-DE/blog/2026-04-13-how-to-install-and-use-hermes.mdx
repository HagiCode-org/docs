Wenn Sie Hermes installieren und sofort verwenden moechten, ist der kuerzeste Weg wirklich nur drei Schritte:

1. Den offiziellen Installationsbefehl ausfuehren
2. Die CLI in Ihrem Terminal mit `hermes` starten
3. Wenn Sie es weiterhin in Feishu verwenden moechten, konfigurieren Sie `hermes gateway setup`

Dieser Artikel versucht nicht, alle Hermes-Faehigkeiten auf einmal zu erklaeren. Stattdessen hilft er Ihnen, die wichtigste Anfaenger-Schleife zuerst abzuschliessen: installieren, starten, verwenden und dann mit einem der haeufigsten Messaging-Plattform-Szenarien verbinden.

## Was Hermes ist und fuer wen es gedacht ist

Hermes Agent ist ein KI-Agent, den Sie entweder von einem lokalen Terminal oder ueber ein Messaging-Plattform-Gateway verwenden koennen.

Fuer die meisten Entwickler gibt es zwei haeufige Einstiegspunkte:

- **CLI**: Tippen Sie `hermes` in Ihr Terminal, um direkt in die interaktive Oberflaeche zu gelangen.
- **Messaging-Gateway**: Führen Sie `hermes gateway` aus und chatten Sie dann von Plattformen wie Feishu, Telegram, Discord und Slack.

Wenn Ihr Ziel jetzt einfach ist, schnell loszulegen, kehren Sie die Reihenfolge nicht um. Beginnen Sie stattdessen mit diesem Weg:

- Installieren Sie zuerst Hermes
- Verifizieren Sie zuerst, dass es ueber die CLI funktioniert
- Entscheiden Sie dann, ob Sie eine Messaging-Plattform verbinden moechten

Das macht die Problemdiagnose einfacher und ist besser fuer Personen geeignet, die Hermes zum ersten Mal verwenden.

## Was Sie vor der Installation von Hermes wissen sollten

Laut dem Hermes README unterstuetzt der offizielle Schnellinstallationspfad folgende Umgebungen:

- Linux
- macOS
- WSL2
- Android ueber Termux

### Hinweise fuer Windows-Benutzer

Hermes unterstuetzt derzeit nicht das direkte Ausfuehren auf nativem Windows. Wenn Sie Windows verwenden, wird empfohlen, zuerst WSL2 zu installieren und dann den Installationsbefehl innerhalb von WSL2 auszufuehren.

Es ist am besten, dies gleich zu Beginn klarzustellen, da viele Installationsfehler nicht durch den Befehl selbst verursacht werden, sondern durch die Verwendung einer nicht unterstuetzten Laufzeitumgebung.

## Wie man Hermes installiert

Der Schnellinstallationsbefehl im Hermes README lautet:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Dieser Befehl fuehrt das offizielle Installationsskript aus und behandelt plattformspezifische Initialisierungsschritte.

### Shell nach der Installation neu laden

Sobald die Installation abgeschlossen ist, laden Sie zuerst Ihre Shell-Umgebung neu. Der haeufigste Befehl ist:

```bash
source ~/.bashrc
```

Wenn Sie zsh verwenden, koennen Sie:

```bash
source ~/.zshrc
```

### Wie man bestaetigt, dass Hermes korrekt installiert ist

Der direkteste Weg zur Pruefung ist:

```bash
hermes
```

Wenn Sie zusaetzliche Bestaetigung moechten, dass Ihre Konfiguration und Abhaengigkeiten funktionieren, koennen Sie auch ausfuehren:

```bash
hermes doctor
```

`hermes doctor` ist besonders nuetzlich in diesen Situationen:

- Der Befehl verhaelt sich nach der Installation abnorm
- Die Modellkonfiguration schlaegt fehl
- Das Gateway startet nicht
- Sie sind nicht sicher, ob Ihre Umgebungsabhaengigkeiten vollstaendig sind

## Wie man Hermes zum ersten Mal verwendet

### Mit der CLI starten

Wenn Sie nur moeglichst schnell bestaetigen moechten, dass Hermes funktioniert, ist die einfachste Methode:

```bash
hermes
```

Dies startet die interaktive Hermes-CLI. Fuer Erstbenutzer von Hermes ist dies auch der empfohlene Einstiegspunkt, da Sie zuerst die wichtigsten Dinge verifizieren koennen:

- Ob der Befehl tatsaechlich verfuegbar ist
- Ob die aktuelle Modellkonfiguration ordnungsgemaess funktioniert
- Ob die Terminal-Toolchain korrekt funktioniert
- Ob der Interaktionsstil zu Ihren Beduerfnissen passt

### Diese Befehle reichen fuer die erste Einrichtung

Das Hermes README listet mehrere haeufige Befehle auf, die zusammen einen praktischen Erste-Nutzung-Pfad bilden:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Wenn Sie nicht sicher sind, was jeder einzelne tut, merken Sie sie sich so:

- `hermes model`: Modelle waehlen oder wechseln
- `hermes tools`: Aktuell verfuegbare Tools anzeigen und konfigurieren
- `hermes config set`: Spezifische Konfigurationselemente aendern
- `hermes setup`: Den vollstaendigen Initialisierungs-Assistenten einmal ausfuehren
- `hermes update`: Hermes aktualisieren
- `hermes doctor`: Probleme beheben

Fuer Anfaenger ist die praktischste Reihenfolge normalerweise:

1. Zuerst `hermes model` ausfuehren
2. Wenn Sie alle haeufigen Optionen auf einmal konfigurieren moechten, dann `hermes setup` ausfuehren

## Die zwei haeufigsten Arten, Hermes zu verwenden

### 1. Hermes im Terminal als taeglichen Entwicklungsassistenten verwenden

Der CLI-Modus eignet sich gut fuer diese Szenarien:

- Direkt beim lokalen Coden Fragen stellen
- Projekte inspizieren, Dateien bearbeiten und Befehle ausfuehren
- Einmaliges Debugging oder Review durchfuehren
- Kontinuierlich im aktuellen Arbeitsverzeichnis zusammenarbeiten

Sein groesster Vorteil ist, dass es der kuerzeste Weg ist: keine zusaetzliche Plattformintegration, keine Bot-Konfiguration im Voraus, und es ist der beste Weg, Ihre ersten Nutzungsgewohnheiten aufzubauen.

### 2. Hermes ueber eine Messaging-Plattform verwenden

Wenn Sie mit Hermes auf Plattformen wie Feishu, Telegram oder Discord chatten moechten, muessen Sie das Messaging-Gateway verwenden.

Die haeufigsten Einstiegsbefehle sind:

```bash
hermes gateway setup
hermes gateway
```

Konkret:

- `hermes gateway setup` wird fuer die interaktive Plattformkonfiguration verwendet
- `hermes gateway` wird zum Starten des Gateway-Prozesses verwendet

Laut der offiziellen Dokumentation ist das Gateway ein einheitlicher Hintergrundprozess, der Ihre konfigurierten Plattformen verbindet, Sitzungen verwaltet und Funktionen wie Cron-Jobs behandelt.

## Feishu als Beispiel: Hermes mit einer Messaging-Plattform verbinden

Wenn der Groessteil Ihrer taeglichen Arbeit in Feishu stattfindet, ist Feishu/Lark eine sehr natuerliche Art, Hermes zu verwenden.

### Der minimal funktionsfaehige Integrationspfad

Die offizielle Dokumentation empfiehlt diesen Einstiegsbefehl fuer Feishu/Lark:

```bash
hermes gateway setup
```

Nachdem Sie ihn ausgefuehrt haben, waehlen Sie einfach **Feishu / Lark** im Assistenten.

Die Feishu-Dokumentation beschreibt zwei Verbindungsmodi:

- `websocket`: empfohlen
- `webhook`: optional

Wenn Hermes auf Ihrem Laptop, Arbeitsplatz oder privaten Server laeuft, ist die Verwendung von `websocket` zuerst normalerweise einfacher, da Sie keine oeffentliche Callback-URL freigeben muessen.

### Wenn Sie es manuell konfigurieren, kennen Sie zumindest diese Variablen

Wenn Sie den Assistenten nicht verwenden und die Konfiguration manuell schreiben, listet die Feishu-Dokumentation diese Kernvariablen:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Zwei davon verdienen besondere Aufmerksamkeit:

- `FEISHU_ALLOWED_USERS`: empfohlen, damit nicht jeder, der den Bot erreichen kann, ihn direkt verwenden kann
- `FEISHU_HOME_CHANNEL`: ermoeglicht Ihnen, einen Heim-Chat vorzudefinieren, um Cron-Ergebnisse oder Standardbenachrichtigungen zu empfangen

### Warum Hermes manchmal nicht in Feishu-Gruppenchats antwortet

Dieses Detail wird leicht uebersehen: In Feishu-Gruppenchats antwortet Hermes standardmaessig nicht auf jede Nachricht.

Die offizielle Dokumentation stellt klar:

- In Direktnachrichten antwortet Hermes auf Nachrichten
- In Gruppenchats muessen Sie den Bot explizit `@` erwaehnen, bevor er die Nachricht verarbeitet

Wenn Sie eine Feishu-Konversation als Heim-Kanal festlegen moechten, koennen Sie auch dies im Chat verwenden:

```text
/set-home
```

Oder definieren Sie es im Voraus in der Konfiguration:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Die Hermes-Befehle, die Anfaenger zuerst merken sollten

Ob Sie Hermes in der CLI oder auf einer Messaging-Plattform verwenden, reicht das Merken der folgenden Befehle bereits zum Loslegen:

- `/new` oder `/reset`: eine neue Sitzung starten
- `/model`: Modell anzeigen oder wechseln
- `/retry`: den vorherigen Durchgang wiederholen
- `/undo`: die vorherige Interaktion rueckgaengig machen
- `/compress`: den Kontext manuell komprimieren
- `/help`: Hilfe anzeigen

Wenn Sie Hermes hauptsaechlich auf einer Messaging-Plattform verwenden, merken Sie sich noch einen:

- `/sethome` oder `/set-home`: den aktuellen Chat als Heim-Kanal festlegen

Diese Befehle decken die haeufigsten Anfaenger-Operationen ab: Neustart, Anpassung, Zurueckrollen, Pruefung und Fortsetzung.

## Haeufig gestellte Fragen

### Kann Windows Hermes direkt installieren?

Nein. Die aktuelle offizielle Dokumentation stellt klar, dass nativs Windows nicht unterstuetzt wird und WSL2 empfohlen wird.

### Was soll ich tun, wenn die Eingabe von `hermes` nach der Installation nichts bewirkt?

Am besten sollten Sie in dieser Reihenfolge Fehler beheben:

1. Zuerst die Shell neu laden, z.B. mit `source ~/.bashrc`
2. `hermes` erneut ausfuehren
3. Wenn es immer noch abnorm ist, `hermes doctor` ausfuehren

### Warum antwortet der Bot nicht in einer Feishu-Gruppe?

Pruefen Sie zuerst diese drei Dinge:

- Ob Sie Hermes in der Gruppe `@` erwaehnt haben
- Ob `FEISHU_ALLOWED_USERS` den aktuellen Benutzer einschraenkt
- Ob die aktuelle Gruppenchat-Richtlinie das Behandeln von Gruppennachrichten erlaubt

Laut der offiziellen Feishu-Dokumentation ist in Gruppenchat-Szenarien eine explizite @-Erwaehnung erforderlich.

## Empfohlene naechste Schritte

Wenn Sie einfach moeglichst schnell mit der Verwendung von Hermes beginnen moechten, ist dies die empfohlene Reihenfolge:

1. Zuerst den Installationsbefehl ausfuehren
2. Zuerst mit `hermes` in der lokalen CLI starten
3. Mit `hermes model` und `hermes setup` die Grundkonfiguration abschliessen
4. Wenn Sie es weiterhin in Feishu verwenden moechten, dann `hermes gateway setup` konfigurieren
