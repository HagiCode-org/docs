Wenn Sie Hermes installieren und verwenden möchten, ist der kürzeste Weg eigentlich nur drei Schritte:

1. Führen Sie den offiziellen Installationsbefehl aus
2. Starten Sie die CLI mit `hermes` im Terminal
3. Wenn Sie es in Feishu weiterverwenden möchten, konfigurieren Sie `hermes gateway setup`

Dieser Artikel wird nicht alle Fähigkeiten von Hermes auf einmal erklären, sondern Ihnen helfen, den wichtigsten geschlossenen Einsteigerkreislauf zu vollenden: installieren, ausführen, anfangen zu verwenden und dann das häufigste Nachrichtenplattform-Szenario anzuschließen.

## Was ist Hermes und für wen ist es geeignet

Hermes Agent ist ein KI-Agent, der sowohl im lokalen Terminal als auch über Nachrichtenplattform-Gateways verwendet werden kann.

Für die meisten Entwickler gibt es zwei häufigste Einstiegspunkte:

- **CLI**: Geben Sie `hermes` im Terminal ein, um direkt in die interaktive Oberfläche zu gelangen
- **Messaging Gateway**: Führen Sie `hermes gateway` aus und chatten Sie mit ihm über Plattformen wie Feishu, Telegram, Discord, Slack

Wenn Ihr aktuelles Ziel nur ein schneller Start ist, empfehle ich, diese Reihenfolge nicht umzukehren, sondern zuerst diesen Weg zu gehen:

- Installieren Sie zuerst Hermes
- Verifizieren Sie zuerst mit der CLI, ob es verfügbar ist
- Entscheiden Sie dann, ob Sie eine Nachrichtenplattform anschließen möchten

So ist es einfacher, Probleme zu lokalisieren, und es eignet sich besser für Benutzer, die Hermes zum ersten Mal verwenden.

## Was Sie wissen müssen, bevor Sie Hermes installieren

Laut Hermes README unterstützt der offizielle Schnellinstallationspfad diese Umgebungen:

- Linux
- macOS
- WSL2
- Android via Termux

### Hinweise für Windows-Benutzer

Hermes unterstützt derzeit keine native native Windows-Ausführung. Wenn Sie Windows verwenden, empfehle ich, zuerst WSL2 zu installieren und dann die Installationsbefehle in WSL2 auszuführen.

Dies ist am besten am Anfang des Artikels klarzustellen, da viele Installationsfehler eigentlich keine Befehlsprobleme sind, sondern die Laufzeitumgebung nicht den Anforderungen entspricht.

## Wie man Hermes installiert

Der Schnellinstallationsbefehl aus dem Hermes README ist:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Dieser Befehl führt das offizielle Installationsskript aus und behandelt plattformspezifische Initialisierungsprozesse.

### Shell nach der Installation aktualisieren

Nach Abschluss der Installation laden Sie zuerst die Shell-Umgebung neu. Am häufigsten ist:

```bash
source ~/.bashrc
```

Wenn Sie zsh verwenden, können Sie auch ändern zu:

```bash
source ~/.zshrc
```

## So bestätigen Sie, dass Hermes bereits installiert ist

Der direkteste Überprüfungsweg ist auszuführen:

```bash
hermes
```

Wenn Sie weiter bestätigen möchten, ob Konfiguration und Abhängigkeiten normal sind, können Sie auch ausführen:

```bash
hermes doctor
```

`hermes doctor` eignet sich für diese Zeiten:

- Nach der Installation verhält sich der Befehl nicht normal
- Modellkonfiguration schlägt fehl
- Gateway-Start schlägt fehl
- Sie sind unsicher, ob Umgebungsabhängigkeiten vollständig sind

## So verwenden Sie Hermes zum ersten Mal

### Zuerst mit der CLI beginnen

Wenn Sie nur schnell bestätigen möchten, ob Hermes verwendet werden kann, ist die einfachste Methode:

```bash
hermes
```

Dies startet die interaktive CLI von Hermes. Für Benutzer, die Hermes zum ersten Mal verwenden, ist dies auch der empfohlene Startpunkt, da Sie zuerst die wichtigsten Dinge verifizieren können:

- Ob der Befehl wirklich verfügbar ist
- Ob die aktuelle Modellkonfiguration normal ist
- Ob die Terminal-Toolkette normal funktioniert
- Ob die Interaktionserfahrung die Art ist, die Sie benötigen

### Diese Befehle reichen für Ihre erste Runde der Konfiguration

Die häufigen Befehle, die im Hermes README aufgeführt sind, bilden im Grunde den ersten Einsteigerpfad:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Wenn Sie nicht wissen, was sie jeweils tun, können Sie sich zuerst so merken:

- `hermes model`: Modell auswählen, Modell wechseln
- `hermes tools`: Aktuelle verfügbare Tools ansehen und konfigurieren
- `hermes config set`: Spezielle Konfigurationselemente ändern
- `hermes setup`: Einmal vollständigen Initialisierungsassistenten ausführen
- `hermes update`: Hermes aktualisieren
- `hermes doctor`: Fehlerbehebung durchführen

Die für Anfänger praktischste Reihenfolge ist normalerweise:

1. Führen Sie zuerst `hermes model` aus
2. Wenn Sie hoffen, häufige Elemente auf einmal zu konfigurieren, führen Sie dann `hermes setup` aus

## Die zwei häufigsten Verwendungsmethoden von Hermes

### 1. Verwenden Sie Hermes im Terminal als täglichen Entwicklungsassistenten

Der CLI-Modus eignet sich für diese Szenarien:

- Direktes Fragen beim lokalen Schreiben von Code
- Projekte anzeigen, Dateien ändern, Befehle ausführen
- Einmaliges Debugging oder Review durchführen
- Kontinuierliche Zusammenarbeit im aktuellen Arbeitsverzeichnis

Sein größter Vorteil ist der kürzeste Weg: keine zusätzliche Plattformanbindung, keine vorherige Bot-Konfiguration, und es eignet sich am besten, um die erste Runde Verwendungsgewohnheiten aufzubauen.

### 2. Verwenden Sie Hermes über Nachrichtenplattformen

Wenn Sie mit Hermes über Plattformen wie Feishu, Telegram, Discord chatten möchten, müssen Sie das Messaging-Gateway verwenden.

Die häufigsten Einstiegsbefehle sind:

```bash
hermes gateway setup
hermes gateway
```

Wobei:

- `hermes gateway setup` wird für interaktive Plattformkonfiguration verwendet
- `hermes gateway` wird zum Starten des Gateway-Prozesses verwendet

Laut der offiziellen Dokumentation ist das Gateway ein einheitlicher Hintergrundprozess, der Ihre konfigurierten Plattformen verbindet, Sitzungen verwaltet und geplante Aufgaben verarbeitet.

## Wie man Hermes an die Nachrichtenplattform anschließt, am Beispiel von Feishu

Wenn Ihre tägliche Arbeit hauptsächlich in Feishu stattfindet, ist Feishu/Lark eine sehr natürliche Hermes-Integrationsmethode.

### Minimal viable Integrationsmethode

Der empfohlene Einstieg für Feishu/Lark in der offiziellen Dokumentation ist:

```bash
hermes gateway setup
```

Führen Sie es aus und wählen Sie im Assistenten **Feishu / Lark**.

Die beiden Verbindungsmodi, die in der Feishu-Dokumentation angegeben sind, sind:

- `websocket`: Empfohlen
- `webhook`: Optional

Wenn Hermes auf Ihrem Laptop, Ihrer Arbeitsstation oder Ihrem privaten Server läuft, ist die priorisierte Verwendung von `websocket` einfacher, da Sie keine zusätzliche öffentliche Rückrufadresse freilegen müssen.

### Wenn Sie manuell konfigurieren, müssen Sie zumindest diese Variablen kennen

Wenn Sie nicht über den Assistenten konfigurieren, sondern manuell Konfiguration schreiben, umfassen die Kernvariablen, die in der Feishu-Dokumentation aufgeführt sind:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Am bemerkenswertesten sind zwei Punkte:

- `FEISHU_ALLOWED_USERS`: Empfohlen zu konfigurieren, um zu vermeiden, dass jeder, der den Bot erreichen kann, ihn direkt verwenden kann
- `FEISHU_HOME_CHANNEL`: Sie können im Voraus einen Home-Chat angeben, um Cron-Ergebnisse oder Standardbenachrichtigungen zu erhalten

### Warum antwortet Hermes in Feishu-Gruppenchats manchmal nicht

Dieser Punkt wird leicht übersehen: In Feishu-Gruppenchats reagiert Hermes standardmäßig nicht auf jede Nachricht.

Die offizielle Dokumentation erklärt klar:

- Im Privat Chat reagiert Hermes auf Nachrichten
- In Gruppenchats muss der Bot explizit `@` werden, damit er Nachrichten verarbeitet

Wenn Sie einen Feishu-Chat als Home-Channel festlegen möchten, können Sie auch im Chat verwenden:

```text
/set-home
```

Oder vorher in der Konfiguration schreiben:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Hermes-Befehle, die Anfänger zuerst im Gedächtnis behalten sollten

Unabhängig davon, ob Sie in der CLI oder auf einer Nachrichtenplattform sind, reicht es, zuerst diese Befehle zu merken:

- `/new` oder `/reset`: Neue Sitzung starten
- `/model`: Aktuelles Modell anzeigen oder wechseln
- `/retry`: Letzte Runde wiederholen
- `/undo`: Letzte Interaktion rückgängig machen
- `/compress`: Kontext manuell komprimieren
- `/help`: Hilfe anzeigen

Wenn Sie hauptsächlich auf Nachrichtenplattformen verwenden, merken Sie sich zusätzlich einen:

- `/sethome` oder `/set-home`: Aktuellen Chat als Home-Channel festlegen

Diese Befehle decken die häufigsten Operationen in der Anfängerphase ab: neu starten, anpassen, zurückgehen, anzeigen und weiterverwenden.

## Häufig gestellte Fragen

### Kann Hermes direkt unter Windows installiert werden?

Nein. Die aktuelle offizielle Dokumentation erklärt klar, dass native Windows nicht unterstützt wird, WSL2 wird empfohlen.

### Was tun, wenn nach der Installation `hermes` keine Reaktion zeigt?

Empfohlen wird, in dieser Reihenfolge zu prüfen:

1. Laden Sie zuerst die Shell neu, zum Beispiel `source ~/.bashrc`
2. Führen Sie dann `hermes` erneut aus
3. Wenn es immer noch anomal ist, führen Sie `hermes doctor` aus

### Warum antwortet der Bot in der Feishu-Gruppe nicht?

Überprüfen Sie zuerst diese drei Punkte:

- Haben Sie `@` Hermes in der Gruppe
- Ob `FEISHU_ALLOWED_USERS` den aktuellen Benutzer einschränkt
- Ob die aktuelle Gruppenchat-Richtlinie die Verarbeitung von Gruppennachrichten zulässt

Laut der offiziellen Feishu-Dokumentation ist eine explizite @Erwähnung in Gruppenszenarien eine notwendige Bedingung.
