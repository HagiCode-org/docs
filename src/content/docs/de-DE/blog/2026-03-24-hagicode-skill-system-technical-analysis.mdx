## Technische Analyse des HagiCode-Skill-Systems: Wie man eine skalierbare AI-Skill-Managementplattform erstellt

> Dieser Artikel bietet eine eingehende Analyse der Architektur und Implementierung des Skill-Verwaltungssystems im HagiCode-Projekt und deckt die technischen Implementierungsdetails der vier Kernfunktionen ab: lokale globale Verwaltung, Marktsuche, intelligente Empfehlungen und Verwaltung vertrauenswürdiger Anbieter.

## Hintergrund

Im Bereich der AI-Code-Assistenten ist die Erweiterung der Fähigkeiten von AI一直 ein zentrales Thema. Die Codeunterstützung von Claude Code selbst ist ziemlich stark, aber verschiedene Entwicklungsteams und verschiedene Technologiestapel benötigen oft professionelle Fähigkeiten für spezifische Szenarien – wie zum Beispiel die Verarbeitung von Docker-Bereitstellungen, Datenbankoptimierung, Frontend-Komponentengenerierung und ähnliches. In solchen Fällen wird das Skill-System besonders wichtig.

Das HagiCode-Projekt stieß während der Entwicklung auf ähnliche Herausforderungen: Wie man Claude Code ermöglichen kann, neue professionelle Fähigkeiten wie ein Mensch zu „lernen“, während gleichzeitig eine gute Benutzererfahrung und technische Wartbarkeit gewährleistet wird? Nachdem dieses Problem weder zu schwierig noch zu einfach ist, haben wir ein komplettes Skill-Verwaltungssystem entwickelt und implementiert.

Dieser Artikel wird die technische Architektur und die Kernimplementierung dieses Systems detailliert analysieren und eignet sich für Entwickler, die sich für AI-Erweiterbarkeit und die Integration von Befehlszeilentools interessieren. Vielleicht ist es für Sie nützlich, vielleicht auch nicht, aber es ist zumindest geschrieben.

## Überblick über die Systemarchitektur

Das Skill-System verwendet eine Architektur mit getrenntem Frontend und Backend, was nichts Besonderes ist.

**Frontend-Teil** verwendet TanStack Start + React zum Erstellen der Benutzeroberfläche, verwaltet den Zustand mit Redux Toolkit, und die vier Hauptfunktionen entsprechen jeweils vier Tab-Komponenten: lokale Skills, Marktgalerie, intelligente Empfehlungen, vertrauenswürdige Anbieter. Dieses Design dient letztendlich der Benutzererfahrung.

**Backend-Teil** basiert auf ASP.NET Core + ABP Framework, verwendet Orleans Grain für die Verwaltung des verteilten Zustands. Der Online-API-Client kapselt die `IOnlineApiClient`-Schnittstelle für die Kommunikation mit dem Remote-Skill-Katalogdienst.

Das Designprinzip der gesamten Architektur ist „Trennung von Befehlsausführung und Geschäftslogik“, wobei die Details der Ausführung von npm/npx-Befehlen durch das Adapter-Muster in unabhängigen Modulen verborgen werden. Schließlich möchte niemand eine Menge Befehlszeilen überall im Code verstreut sehen.

## Kernfunktion 1: Lokale globale Verwaltung

Die lokale globale Verwaltung ist das grundlegendste Funktionsmodul, verantwortlich für das Auflisten installierter Skills und die Unterstützung von Deinstallationsvorgängen. Es ist nicht komplex, es wird einfach gut gemacht.

### Technische Lösung

Die Implementierung befindet sich in `LocalSkillsTab.tsx` und `LocalSkillCommandAdapter.cs`. Der Kernansatz besteht darin, den `npx skills`-Befehl zu kapseln, seine JSON-Ausgabe zu analysieren und in interne Datenstrukturen umzuwandeln. Es klingt einfach, ist aber auch einfach zu implementieren.

```csharp
public async Task<IReadOnlyList<LocalSkillInventoryResponseDto>> GetLocalSkillsAsync(
    CancellationToken cancellationToken = default)
{
    var result = await _commandAdapter.ListGlobalSkillsAsync(cancellationToken);
    return result.Skills.Select(skill => new LocalSkillInventoryResponseDto
    {
        Name = skill.Name,
        Version = skill.Version,
        Source = skill.Source,
        InstalledPath = skill.InstalledPath,
        Description = skill.Description
    }).ToList();
}
```

Der Datenfluss ist sehr klar: Frontend initiiert Anfrage → SkillGalleryAppService empfängt → LocalSkillCommandAdapter führt npx-Befehl aus → JSON-Ergebnis analysieren → DTO-Objekt zurückgeben. Ein Glied nach dem anderen, nichts Besonderes.

Zum Deinstallieren von Skills wird der Befehl `npx skills remove -g <skillName> -y` verwendet, und das System verarbeitet automatisch Abhängigkeiten und Bereinigungsarbeiten. Installationsmetadaten werden in `managed-install.json` im Skill-Verzeichnis gespeichert und zeichnen Installationszeit, Quellversion und andere Informationen auf, was spätere Updates und Audits erleichtert. Schließlich ist es gut, einige Dinge festzuhalten.

### Detaillierte Erklärung des Installationsprozesses

Die Skill-Installation beinhaltet die Koordination mehrerer Schritte, wie man es ausdrückt, eigentlich nicht zu kompliziert:

```csharp
public async Task<SkillInstallResultDto> InstallAsync(
    SkillInstallRequestDto request,
    CancellationToken cancellationToken = default)
{
    // 1. Normalisierung der Installationsreferenz
    var normalized = _referenceNormalizer.Normalize(
        request.SkillId,
        request.Source,
        request.SkillSlug,
        request.Version);

    // 2. Überprüfung der Voraussetzungen
    await _prerequisiteChecker.CheckAsync(cancellationToken);

    // 3. Installationssperre erwerben
    using var installLock = await _lockProvider.AcquireAsync(normalized.SkillId);

    // 4. Installationsbefehl ausführen
    var result = await _installCommandRunner.ExecuteAsync(
        new SkillInstallCommandExecutionRequest
        {
            Command = $"npx skills add {normalized.FullReference} -g -y",
            Timeout = TimeSpan.FromMinutes(4)
        },
        cancellationToken);

    // 5. Persistenz der Installationsmetadaten
    await _metadataStore.WriteAsync(normalized.SkillPath, request);

    return new SkillInstallResultDto { Success = result.Success };
}
```

Hier werden mehrere wichtige Entwurfsmuster verwendet: Der **Referenznormalisierer** ist verantwortlich für die Umwandlung verschiedener Eingabeformate (wie `tanweai/pua`, `@opencode/docker-skill`) in eine einheitliche interne Darstellung; der **Installationssperren-Mechanismus** stellt sicher, dass für denselben Skill nur ein Installationsvorgang gleichzeitig durchgeführt wird; das **Streaming** sendet den Installationsfortschritt in Echtzeit über Server-Sent Events an das Frontend, sodass Benutzer Echtzeit-Protokolle ähnlich wie im Terminal sehen können.

Diese Entwurfsmuster dienen letztendlich dazu, die Dinge einfacher zu machen.

## Kernfunktion 2: Marktsuche

Die Marktsuche ermöglicht es Benutzern, Skills aus der Community zu entdecken und zu installieren. Schließlich sind die Fähigkeiten einer Person begrenzt, aber die Weisheit der Vielen ist unbegrenzt.

### Technische Lösung

Die Suchfunktion hängt von der Online-API `https://api.hagicode.com/v1/skills/search` ab. Um die Antwortgeschwindigkeit zu verbessern, implementiert das System einen Caching-Mechanismus. Caching ist wie das Gedächtnis – wenn manche Dinge gespeichert sind, muss man das nächste Mal nicht mehr so hart nachdenken.

```csharp
private async Task<IReadOnlyList<SkillGallerySkillDto>> SearchCatalogAsync(
    string query,
    CancellationToken cancellationToken,
    IReadOnlySet<string>? allowedSources = null)
{
    var cacheKey = $"skill_search:{query}:{string.Join(",", allowedSources ?? Array.Empty<string>())}";

    if (_memoryCache.TryGetValue(cacheKey, out var cached))
        return (IReadOnlyList<SkillGallerySkillDto>)cached!;

    var response = await _onlineApiClient.SearchAsync(
        new SearchSkillsRequest
        {
            Query = query,
            Limit = _options.LimitPerQuery,
        },
        cancellationToken);

    var results = response.Skills
        .Where(skill => allowedSources is null || allowedSources.Contains(skill.Source))
        .Select(skill => new SkillGallerySkillDto { ... })
        .ToList();

    _memoryCache.Set(cacheKey, results, TimeSpan.FromMinutes(10));
    return results;
}
```

Die Suchergebnisse unterstützen das Filtern nach vertrauenswürdigen Quellen, wobei nur Skill-Quellen angezeigt werden, denen der Benutzer vertraut. Vorab eingestellte Seed-Abfragen werden verwendet, um den Katalog zu initialisieren, wie zum Beispiel „popular“, „recent“ usw., damit Benutzer beim ersten Öffnen empfohlene beliebte Skills sehen können. Schließlich ist der erste Eindruck wichtig.

## Kernfunktion 3: Intelligente Empfehlungen

Die intelligente Empfehlung ist die komplexeste Funktion des Systems und kann basierend auf der aktuellen Projektsituation des Benutzers automatisch die am besten geeigneten Skills empfehlen. Obwohl komplex, ist es dennoch wertvoll, sie zu implementieren.

### Empfehlungsprozess

Der gesamte Empfehlungsprozess ist in fünf Phasen unterteilt:

```
1. Projektkontext erstellen
   ↓
2. AI generiert Suchabfragen
   ↓
3. Parallele Suche im Online-Katalog
   ↓
4. AI rangiert Kandidaten
   ↓
5. Empfehlungsliste zurückgeben
```

Zuerst analysiert das System die Merkmale des Projekts wie Technologiestapel, Programmiersprache, Domänenstruktur usw. und erstellt ein „Projektprofil“. Dieses Profil ist wie ein Lebenslauf einer Person und zeichnet alle Merkmale auf.

Dann wird ein AI Grain verwendet, um gezielte Suchabfragen zu generieren. Dieses Design ist ziemlich interessant – statt die AI direkt zu fragen „welche Skills werden empfohlen“, lässt man sie zuerst darüber nachdenken „welche Suchbegriffe können relevante Skills finden“. Manchmal ist die Art zu fragen wichtiger als die Antwort selbst:

```csharp
var queryGeneration = await aiGrain.GenerateSkillRecommendationQueriesAsync(
    projectContext,      // Projektkontext
    locale,              // Sprachpräferenz des Benutzers
    maxQueries,           // Maximale Anzahl von Abfragen
    effectiveSearchHero); // AI-Modellauswahl
```

Anschließend werden diese Suchabfragen parallel ausgeführt, um eine Liste der Kandidaten-Skills zu erhalten. Parallele Verarbeitung dient letztendlich dazu, Zeit zu sparen.

Schließlich wird ein anderer AI Grain verwendet, um die Kandidaten-Skills zu rangieren. Dieser Schritt berücksichtigt umfassend Faktoren wie die Relevanz des Skills zum Projekt, den Vertrauensstatus, die historischen Präferenzen des Benutzers usw.:

```csharp
var ranking = await aiGrain.RankSkillRecommendationsAsync(
    projectContext,
    candidates,
    installedSkillNames,
    locale,
    maxRecommendations,
    effectiveRankingHero);

response.Items = MergeRecommendations(projectContext, candidates, ranking, maxRecommendations);
}
```

### Fallback-Mechanismus

AI-Modelle können langsam antworten oder vorübergehend nicht verfügbar sein. Schließlich versagt auch das beste System manchmal. Zu diesem Zweck hat das System einen deterministischen Fallback-Mechanismus entworfen: Wenn der AI-Dienst nicht verfügbar ist, werden Empfehlungen mit einem regelbasierten heuristischen Algorithmus generiert, zum Beispiel das Ableiten möglicher benötigter Skills aus den Abhängigkeiten in package.json.

Dieser Fallback-Mechanismus ist letztendlich nur ein Rückweg für das System.

## Kernfunktion 4: Verwaltung vertrauenswürdiger Anbieter

Die Verwaltung vertrauenswürdiger Anbieter ermöglicht es Benutzern zu kontrollieren, welche Skill-Quellen vertrauenswürdig sind. Schließlich muss man Vertrauen selbst kontrollieren.

### Abgleichsregeln

Vertrauenswürdige Anbieter unterstützen zwei Abgleichsregeln: **Exakter Abgleich** (exact) und **Präfix-Abgleich** (prefix).

```csharp
public static TrustedSkillProviderResolutionSnapshot Resolve(
    TrustedSkillProviderSnapshot snapshot,
    string source)
{
    var normalizedSource = Normalize(source);

    foreach (var entry in snapshot.Entries.OrderBy(e => e.SortOrder))
    {
        if (!entry.IsEnabled) continue;

        foreach (var rule in entry.MatchRules)
        {
            bool isMatch = rule.MatchType switch
            {
                TrustedSkillProviderMatchRuleType.Exact
                    => string.Equals(normalizedSource, Normalize(rule.Value),
                        StringComparison.OrdinalIgnoreCase),
                TrustedSkillProviderMatchRuleType.Prefix
                    => normalizedSource.StartsWith(Normalize(rule.Value) + "/",
                        StringComparison.OrdinalIgnoreCase),
                _ => false
            };

            if (isMatch)
                return new TrustedSkillProviderResolutionSnapshot
                {
                    IsTrustedSource = true,
                    ProviderId = entry.ProviderId,
                    DisplayName = entry.DisplayName
                };
        }
    }

    return new TrustedSkillProviderResolutionSnapshot { IsTrustedSource = false };
}
```

Vorab eingestellte vertrauenswürdige Anbieter umfassen bekannte Organisationen und Projekte wie Vercel, Azure, anthropics, Microsoft, browser-use usw. Benutzerdefinierte Anbieter können über Konfigurationsdateien hinzugefügt werden, wobei Anbieter-ID, Anzeigename, Badge-Label, Abgleichsregeln usw. angegeben werden. Schließlich ist die Welt so groß, es kann nicht nur ein paar vertrauenswürdige geben.

### Persistenz-Implementierung

Die Vertrauenskonfiguration wird mit Orleans Grain persistent gespeichert:

```csharp
public class TrustedSkillProviderGrain : Grain<TrustedSkillProviderState>,
    ITrustedSkillProviderGrain
{
    public async Task UpdateConfigurationAsync(TrustedSkillProviderSnapshot snapshot)
    {
        State.Snapshot = snapshot;
        await WriteStateAsync();
    }

    public Task<TrustedSkillProviderSnapshot> GetConfigurationAsync()
    {
        return Task.FromResult(State.Snapshot);
    }
}
```

Der Vorteil dieses Ansatzes ist, dass Konfigurationsänderungen automatisch auf alle Knoten synchronisiert werden, ohne dass der Cache manuell aktualisiert werden muss. Automatisierung dient letztendlich dazu, sich weniger Sorgen zu machen.

## Wichtige technische Entwürfe

### Befehlsausführungs-Adapter-Muster

Das Skill-System muss verschiedene npx-Befehle ausführen. Wenn diese Logik überall verstreut ist, wird der Code schwer zu warten. Daher haben wir eine Adapter-Schnittstelle entworfen. Entwurfsmuster dienen letztendlich dazu, den Code besser wartbar zu machen:

```csharp
public interface ISkillInstallCommandRunner
{
    Task<SkillInstallCommandExecutionResult> ExecuteAsync(
        SkillInstallCommandExecutionRequest request,
        CancellationToken cancellationToken = default);
}
```

Verschiedene Befehle haben verschiedene Ausführungs-Implementierungen, die alle dieselbe Schnittstelle implementieren, was das Testen und Ersetzen erleichtert.

### SSE-Streaming-Ausgabe

Der Installationsfortschritt wird in Echtzeit über Server-Sent Events an das Frontend gesendet:

```csharp
public async Task InstallWithProgressAsync(
    SkillInstallRequestDto request,
    IServerStreamWriter<SkillInstallProgressEventDto> stream,
    CancellationToken cancellationToken)
{
    var process = new Process
    {
        StartInfo = new ProcessStartInfo
        {
            FileName = "npx",
            Arguments = $"skills add {request.FullReference} -g -y",
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false
        }
    };

    process.OutputDataReceived += async (sender, e) =>
    {
        await stream.WriteAsync(new SkillInstallProgressEventDto
        {
            EventType = "output",
            Data = e.Data ?? string.Empty
        });
    };

    process.Start();
    process.BeginOutputReadLine();
    await process.WaitForExitAsync(cancellationToken);
}
```

Benutzer können am Frontend eine Echtzeitausgabe ähnlich wie im Terminal sehen, was sehr intuitiv ist. Echtzeit-Feedback gibt Sicherheit.

## Praxis-Leitfaden

### Community-Skills installieren

Beispiel für die Installation des `pua`-Skills (ein beliebter Community-Skill):

1. Öffnen Sie die Skills-Schublade, wechseln Sie zum Tab „Skill Gallery“
2. Geben Sie „pua“ ein, um zu suchen
3. Klicken Sie auf das Suchergebnis, um die Skill-Details anzuzeigen
4. Klicken Sie auf die Schaltfläche „Install“, um zu installieren
5. Wechseln Sie zum Tab „Local Skills“, um die erfolgreiche Installation zu bestätigen

Der Installationsbefehl ist `npx skills add tanweai/pua -g -y`, und das System verarbeitet automatisch alle Details. Eigentlich gibt es nicht so viele Schritte, man geht einfach Schritt für Schritt vor.

### Benutzerdefinierte vertrauenswürdige Quellen hinzufügen

Wenn Ihr Team ein eigenes Skill-Repository hat, können Sie es als vertrauenswürdige Quelle hinzufügen:

```yaml
providerId: "my-team"
displayName: "My Team Skills"
badgeLabel: "MyTeam"
isEnabled: true
sortOrder: 100
matchRules:
  - matchType: "prefix"
    value: "my-team/"
  - matchType: "exact"
    value: "my-team/special-skill"
```

Auf diese Weise zeigen alle Skills aus Ihrem Team ein Vertrauens-Badge, und Benutzer können sie mit mehr Vertrauen installieren. Schließlich geben markierte Dinge Sicherheit.

### Skill-Entwicklungsgrundlagen

Das Erstellen benutzerdefinierter Skills erfordert die folgende Struktur:

```
my-skill/
├── SKILL.md          # Skill-Metadaten (YAML front matter)
├── index.ts          # Skill-Einstiegspunkt
├── agents/           # Unterstützte Agent-Konfigurationen
└── references/       # Referenz-Ressourcen
```

Beispiel für das Format von `SKILL.md`:

```yaml
---
name: my-skill
description: Eine kurze Beschreibung dessen, was dieser Skill tut
---

# My Skill

Detaillierte Dokumentation...
```

## Hinweise

1. **Netzwerkanforderungen**: Die Skillsuche und -installation erfordert Zugriff auf `api.hagicode.com` und das npm-Registry
2. **Node.js-Version**: Es wird empfohlen, Node.js 18 oder höher zu verwenden
3. **Berechtigungsanforderungen**: Globale npm-Installationsberechtigungen sind erforderlich
4. **Nebenläufigkeitssteuerung**: Für denselben Skill kann gleichzeitig nur ein Installations- oder Deinstallationsvorgang ausgeführt werden
5. **Timeout-Einstellung**: Die Standard-Timeout-Zeit für Installationsvorgänge beträgt 4 Minuten, komplexe Szenarien können eine Anpassung erfordern

Diese Hinweise dienen letztendlich dazu, dass die Dinge reibungslos verlaufen.

## Zusammenfassung

Dieser Artikel stellte die vollständige Implementierung des Skill-Verwaltungssystems im HagiCode-Projekt vor. Durch technische Mittel wie Architektur mit getrenntem Frontend und Backend, Adapter-Muster und Orleans-verteile Zustandsverwaltung implementiert dieses System:

- **Lokale globale Verwaltung**: Durch Kapselung von npx skills-Befehlen wird eine einheitliche Skill-Verwaltungsschnittstelle bereitgestellt
- **Marktsuche**: Nutzung von Online-API und Caching-Mechanismus für die schnelle Entdeckung von Community-Skills
- **Intelligente Empfehlungen**: Kombination mit AI-Fähigkeiten zur Empfehlung der am besten geeigneten Skills basierend auf Projektkontext
- **Vertrauenswürdigkeitsverwaltung**: Flexibles Konfigurationssystem, das Benutzern die Kontrolle über Vertrauensgrenzen ermöglicht

Dieser Designansatz ist nicht nur auf die Skill-Verwaltung anwendbar, sondern hat auch Referenzwert für jedes Szenario, das die Integration von Befehlszeilentools und die gleichzeitige Berücksichtigung lokaler Speicherung und Online-Dienste erfordert.
