## Technische Analyse des HagiCode Skill-Systems: Aufbau einer skalierbaren KI-Skill-Verwaltungsplattform

> Dieser Artikel bietet einen tiefen Einblick in die Architektur und Implementierung des Skill-Verwaltungssystems im HagiCode-Projekt und behandelt die technischen Details hinter vier Kernfähigkeiten: lokaler/globaler Verwaltung, Marktplatzsuche, intelligente Empfehlungen und vertrauenswürdiger Anbieterverwaltung.

## Hintergrund

Im Bereich der KI-Code-Assistenten ist die Frage, wie man die Grenzen der KI-Fähigkeiten erweitern kann, immer eine Kernfrage. Claude Code selbst ist bereits stark in der Code-Unterstützung, aber verschiedene Entwicklungsteams und verschiedene Technologie-Stacks benötigen oft spezialisierte Fähigkeiten für bestimmte Szenarien, wie z. B. die Handhabung von Docker-Deployment, Datenbankoptimierung oder Frontend-Komponentengenerierung. Genau hier wird ein Skill-System besonders wichtig.

Während der Entwicklung des HagiCode-Projekts sind wir auf eine ähnliche Herausforderung gestoßen: Wie lassen wir Claude Code "lernen", wie ein Mensch neue professionelle Fähigkeiten erwerben, während wir gleichzeitig eine solide Benutzererfahrung und gute technische Wartbarkeit beibehalten? Dieses Problem ist auf seine Weise sowohl schwer als auch einfach. Um diese Frage herum haben wir ein vollständiges Skill-Verwaltungssystem entworfen und implementiert.

Dieser Artikel führt im Detail durch die technische Architektur und die Kernimplementierung des Systems. Er richtet sich an Entwickler, die sich für KI-Erweiterbarkeit und Kommandozeilenwerkzeug-Integration interessieren. Es könnte für Sie nützlich sein oder auch nicht, aber zumindest ist es jetzt aufgeschrieben.

## Ueber HagiCode

Die in diesem Artikel geteilten Inhalte stammen aus der realen Entwicklung des [HagiCode](https://hagicode.com)-Projekts. HagiCode ist ein KI-Code-Assistent-Projekt, das ein Skill-System integriert, um die Fähigkeiten von Claude Code zu erweitern.

## Kernanforderungen

Bevor wir in die technische Umsetzung einsteigen, müssen wir die Kernanforderungen klären. Requirements festzulegen ist ein Schritt, den man nicht überspringen sollte, denn ohne klare Anforderungen neigt die Implementierung dazu, in alle möglichen Richtungen abzudriften.

### 1. Lokale und globale Skill-Verwaltung

Benutzer müssen Skills installieren, aktualisieren, deinstallieren und auflisten können. Skills können lokal im Projekt oder global auf dem System installiert werden. Die Prioritätslogik lautet: Lokale Skills haben Vorrang vor globalen Skills mit demselben Namen. Dies ist ein häufiges Muster in Werkzeugen wie npm und VS Code, also warum nicht den bewährten Praktiken folgen?

### 2. Marktplatzsuche

Benutzer müssen im Marktplatz nach verfügbaren Skills suchen können. Die Suchergebnisse sollten Sortierung und Filterung unterstützen, um die richtige Wahl zu erleichtern. Im Grunde ist es eine kleine Suchmaschine, aber für Skills.

### 3. Intelligente Empfehlungen

Das System sollte intelligent Skills empfehlen können, die für den aktuellen Kontext des Benutzers relevant sind. Zum Beispiel, wenn der Benutzer ein Projekt mit Docker-Konfiguration öffnet, Docker-bezogene Skills zu empfehlen. Dies ist eine Funktion, die das Gefühl von "KI-gestütztem" wirklich zum Tragen bringt.

### 4. Vertrauenswürdiger Anbieter

Benutzer müssen bestimmte Anbieter als vertrauenswürdig markieren können, was die Installation von Skills aus diesen Anbietern ohne Warnungen ermöglicht. Dies ist ein Sicherheitsmerkmal, das bei der Verwaltung von Drittanbieter-Erweiterungen unerlässlich ist.

## Technische Architektur

### Gesamtarchitektur

Das Skill-System verwendet eine mehrschichtige Architektur, die von der Benutzerschnittstelle bis zur Datenspeicherung reicht:

```
┌──────────────────────────────────────────────┐
│              CLI-Befehlsschicht               │
│    (install, search, list, recommend, trust) │
├──────────────────────────────────────────────┤
│              Service-Schicht                  │
│  (SkillService, SearchService, ...           │
│   mit IHttpClient und IConsole)              │
├──────────────────────────────────────────────┤
│              Infrastrukturschicht              │
│  (Dateisystem, HTTP, Konfiguration)          │
└──────────────────────────────────────────────┘
```

Die Architektur ist auf einer simplen Idee aufgebaut: Trennung der Verantwortlichkeiten. Jede Schicht erledigt ihre eigene Arbeit, ohne sich in die anderen einzumischen. Das ist ein Prinzip, das im Code wie im Leben gilt.

### Technologie-Stack

| Komponente | Technologie |
|------|------|
| Backend | .NET (Orleans Grain) |
| Frontend | React + TypeScript |
| Datenanbindung | TanStack Query |
| UI-Komponenten | shadcn/ui |
| CLI-Integration | HagiCode.Libs |

Die Wahl des Technologie-Stacks folgte einem einfachen Prinzip: die Werkzeuge verwenden, die wir am besten kennen, um die Entwicklungseffizienz zu maximieren. Es gibt keinen perfekten Stack; es gibt nur den Stack, der am besten zu Ihnen passt.

## Kernimplementierung

### 1. Orleans Grain-Implementierung

Das Skill-System wird über Orleans Grain implementiert, was Verteilung und Skalierung unterstützt. Die Kerngrain-Klasse sieht wie folgt aus:

```csharp
public class SkillGrain : Grain, ISkillGrain
{
    private readonly IOptions<SkillOptions> _options;
    private readonly IHttpClientFactory _httpClientFactory;
    private readonly IConsole _console;
    private readonly ILogger<SkillGrain> _logger;

    public override async Task OnActivateAsync(CancellationToken token)
    {
        // Initialisierung beim Start
    }

    public async Task<IReadOnlyList<SkillInfo>> GetInstalledSkillsAsync(
        SkillScope scope = SkillScope.All,
        CancellationToken cancellationToken = default)
    {
        var skills = new List<SkillInfo>();

        // Globale Skills laden
        if (scope is SkillScope.All or SkillScope.Global)
        {
            var globalDir = _options.Value.GlobalSkillDirectory;
            if (Directory.Exists(globalDir))
            {
                skills.AddRange(LoadSkillsFromDirectory(globalDir, SkillScope.Global));
            }
        }

        // Lokale Skills laden
        if (scope is SkillScope.All or SkillScope.Local)
        {
            var localDir = _options.Value.LocalSkillDirectory;
            if (Directory.Exists(localDir))
            {
                skills.AddRange(LoadSkillsFromDirectory(localDir, SkillScope.Local));
            }
        }

        return skills;
    }
}
```

### 2. Skill-Installationslogik

Die Installation eines Skills umfasst das Herunterladen aus der Quelle und das Extrahieren in das Zielverzeichnis:

```csharp
public async Task InstallSkillAsync(
    string skillName,
    SkillScope scope = SkillScope.Global,
    CancellationToken cancellationToken = default)
{
    // 1. Skill-Informationen vom Marktplatz abrufen
    var skillInfo = await GetSkillInfoAsync(skillName, cancellationToken);

    // 2. Zielverzeichnis bestimmen
    var targetDir = scope switch
    {
        SkillScope.Global => Path.Combine(
            _options.Value.GlobalSkillDirectory, skillInfo.Name),
        SkillScope.Local => Path.Combine(
            _options.Value.LocalSkillDirectory, skillInfo.Name),
        _ => throw new ArgumentOutOfRangeException(nameof(scope))
    };

    // 3. Herunterladen und Entpacken
    var downloadUrl = skillInfo.DownloadUrl;
    var response = await _httpClientFactory
        .CreateClient()
        .GetAsync(downloadUrl, cancellationToken);

    // 4. Dateien extrahieren
    using var stream = await response.Content.ReadAsStreamAsync(cancellationToken);
    ZipFile.ExtractToDirectory(stream, targetDir, overwrite: true);

    _console.WriteLine($"Skill '{skillName}' erfolgreich in {scope} installiert.");
}
```

Die Logik ist einfach, aber die Details zählen. Fehlerbehandlung, Fortschrittsanzeige und die Verwaltung von Konflikten mit bereits installierten Skills – all das braucht besondere Aufmerksamkeit.

### 3. Marktplatzsuche

Die Marktplatzsuche erfolgt über eine HTTP-API:

```csharp
public async Task<IReadOnlyList<MarketplaceSkillInfo>> SearchSkillsAsync(
    string query,
    int limit = 10,
    CancellationToken cancellationToken = default)
{
    var client = _httpClientFactory.CreateClient("marketplace");
    var response = await client.GetAsync(
        $"/api/skills/search?q={Uri.EscapeDataString(query)}&limit={limit}",
        cancellationToken);

    response.EnsureSuccessStatusCode();

    var content = await response.Content.ReadFromJsonAsync<SearchResponse>(
        cancellationToken);

    return content?.Skills ?? [];
}
```

Die Suchergebnisse werden im Frontend mit TanStack Query verwaltet:

```typescript
export function useSearchSkills(query: string) {
  return useQuery({
    queryKey: ['skill-search', query],
    queryFn: () => searchSkills(query),
    staleTime: 5 * 60 * 1000, // 5 Minuten Cache
    enabled: query.length >= 2,  // Suche erst ab 2 Zeichen
  });
}
```

### 4. Intelligente Empfehlungen

Die intelligente Empfehlung basiert auf der Analyse des aktuellen Projektkontexts:

```csharp
public async Task<IReadOnlyList<SkillInfo>> GetRecommendedSkillsAsync(
    ProjectContext context,
    CancellationToken cancellationToken = default)
{
    var recommendations = new List<(SkillInfo Skill, double Score)>();

    // Projektdateien scannen, um relevante Tags zu identifizieren
    var tags = AnalyzeProjectTags(context.ProjectDirectory);

    // Skills anhand der Tags abgleichen
    var allSkills = await GetAllAvailableSkillsAsync(cancellationToken);
    foreach (var skill in allSkills)
    {
        var score = CalculateRelevanceScore(skill.Tags, tags);
        if (score > 0.5)
        {
            recommendations.Add((skill, score));
        }
    }

    // Nach Relevanz sortieren
    return recommendations
        .OrderByDescending(r => r.Score)
        .Take(5)
        .Select(r => r.Skill)
        .ToList();
}
```

Die Tag-Analyse erkennt Merkmale wie Docker-Konfigurationsdateien, Datenbankschemas, Frontend-Frameworks usw. und ordnet sie den Skill-Tags zu. Es ist im Grunde ein einfaches Empfehlungssystem, aber es erfüllt seinen Zweck.

### 5. Vertrauenswürdiger Anbieter

Die vertrauenswürdige Anbieterverwaltung wird über eine Konfigurationsdatei implementiert:

```csharp
public class TrustedProviderConfiguration
{
    public List<string> TrustedProviders { get; set; } = [];
}

public bool IsProviderTrusted(string providerName)
{
    return _options.Value.TrustedProviders
        .Contains(providerName, StringComparer.OrdinalIgnoreCase);
}
```

Benutzer können vertrauenswürdige Anbieter über den CLI-Befehl verwalten:

```bash
# Anbieter als vertrauenswürdig markieren
hagicode skill trust --provider "hagicode-official"

# Vertrauenswürdigen Anbieter entfernen
hagicode skill untrust --provider "unknown-provider"
```

## Frontend-Implementierung

### TanStack Query für Datenverwaltung

Das Frontend verwendet TanStack Query für die Datenverwaltung, was eine konsistente Datenanbindung mit automatischem Caching bietet:

```typescript
// Alle installierten Skills abrufen
export function useInstalledSkills(scope?: SkillScope) {
  return useQuery({
    queryKey: ['installed-skills', scope],
    queryFn: () => getInstalledSkills(scope),
    staleTime: 2 * 60 * 1000,
  });
}

// Marktplatzsuche
export function useSearchSkills(query: string) {
  return useQuery({
    queryKey: ['skill-search', query],
    queryFn: () => searchSkills(query),
    staleTime: 5 * 60 * 1000,
    enabled: query.length >= 2,
  });
}

// Skill installieren (Mutation)
export function useInstallSkill() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: (params: InstallSkillParams) => installSkill(params),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['installed-skills'] });
    },
  });
}
```

TanStack Query macht die Datenverwaltung viel einfacher. Caching, Invalidierung und Ladezustände werden automatisch gehandhabt. In der modernen Frontend-Entwicklung ist das fast schon ein Muss.

## Praktische Hinweise

### Fehlerbehandlung

Skill-Installationen können aus verschiedenen Gründen fehlschlagen: Netzwerkprobleme, ungültige Archive, Berechtigungsprobleme. Die Fehlerbehandlung muss informativ sein und dem Benutzer klare Hinweise zur Problemlösung geben.

### Versionsverwaltung

Skills sollten Versionsnummern unterstützen, und das System sollte prüfen, ob eine neuere Version verfügbar ist. Das ist Standard für Paketverwaltungssysteme, aber es muss sorgfältig implementiert werden.

### Abhängigkeitsprüfung

Einige Skills hängen möglicherweise von anderen Skills ab. Das System sollte diese Abhängigkeiten erkennen und dem Benutzer mitteilen, welche zusätzlichen Skills installiert werden müssen. Das ist etwas, das man leicht übersieht, bis die Benutzer anfangen, Fehler zu melden.

### Sicherheit

Da Skills Drittanbietercode ausführen können, ist die Sicherheit ein wichtiges Thema. Die vertrauenswürdige Anbieterverwaltung ist nur eine Schicht des Schutzes. Idealerweise sollte das System auch eine Sandbox-Umgebung für die Skill-Ausführung bieten. Aber das ist ein Thema für einen zukünftigen Artikel.

## Zusammenfassung

Das HagiCode Skill-System ist eine vollständige Lösung für die Verwaltung von KI-Skills. Durch die Kombination von Orleans Grain auf dem Backend, TanStack Query auf dem Frontend und einer klaren Schichtenarchitektur bietet es eine skalierbare und wartbare Plattform für die Erweiterung der KI-Fähigkeiten.

Die vier Kernfähigkeiten – lokale/globale Verwaltung, Marktplatzsuche, intelligente Empfehlungen und vertrauenswürdiger Anbieterverwaltung – decken die wichtigsten Anwendungsszenarien ab. Das System ist nicht perfekt, aber es erfüllt seine Aufgabe.
