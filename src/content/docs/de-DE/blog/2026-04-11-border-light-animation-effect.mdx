## Leitfaden zur Implementierung von animierten Lichteffekten um den Rahmen

> Wie werden diese wichtigen Elemente, die Benutzer sofort bemerken, eigentlich mit reinem CSS erstellt? Eigentlich gar nicht so schwer – es ist nur ein kleiner Umweg. Dieser Artikel führt Sie Schritt für Schritt durch die Implementierung von animierten Rahmenlichteffekten und erzählt dabei auch von den Fallstricken, die wir im HagiCode-Projekt erlebt haben.

## Hintergrund

Frontend-Entwickler kennen das sicher: Der Produktmanager kommt vorbei, mit diesem Ausdruck im Gesicht, der sagt "diese Anforderung ist ganz einfach" – "Diese laufende Aufgabe, können wir da einen Effekt hinzufügen, damit die Benutzer ihn sofort sehen?"

Sie sagen ja klar, färbt den Rand eben ein. Aber der andere schüttelt den Kopf, mit einem Blick, der so viel sagt wie "du verstehst es nicht": "Nicht auffällig genug, es soll dieses Licht sein, das um den Rand herumläuft, wie in Sci-Fi-Filmen."

In diesem Moment fragen Sie sich vielleicht: Wie setzt man das um? Mit Canvas? Mit SVG? Oder kann CSS das? Schließlich will niemand zugeben, dass er es nicht kann.

Eigentlich sind animierte Rahmenlichteffekte in modernen Web-Anwendungen besonders häufig, hauptsächlich in diesen Szenarien:

- **Statusanzeige**: Markierung laufender Aufgaben oder aktiver Projekte
- **Visueller Fokus**: Hervorhebung wichtiger Inhaltsbereiche
- **Markenverstärkung**: Schaffung eines technologischen und modernen visuellen Erlebnisses
- **Feiertagsthemen**: Erstellung einer festlichen Atmosphäre für besondere Feiertage

Wir sind bei [HagiCode](https://hagicode.com) auf diese Anforderung gestoßen – Benutzer müssen sofort erkennen können, welche Sitzungen laufen und welche Vorschläge verarbeitet werden. Wir haben mehrere Lösungen ausprobiert, einige waren einfacher, andere etwas umständlicher, aber am Ende haben wir eine ziemlich ausgereifte Implementierungsidee entwickelt.

## Kernimplementierungskonzepte

Durch die Analyse des HagiCode-Codes haben wir die folgenden Kernimplementierungsmuster zusammengefasst, jedes mit seinem geeigneten Anwendungsszenario, oder sagen wir so, jedes mit seinem Existenzberechtigung.

### 1. Conic Gradient rotierender Lichtschein (am häufigsten verwendet)

Dies ist die klassischste Implementierung von animierten Rahmenlichteffekten. Der Kernansatz besteht darin, mit dem CSS `conic-gradient` einen konischen Farbverlauf zu erstellen und ihn dann rotieren zu lassen. Wie eine Straßenlaterne in der Nacht, die immer weiterdreht.

**Wesentliche Elemente:**
- Erstellung eines Lichtschein-Layers mit dem `::before`-Pseudoelement
- Definition der Farbverteilung mit `conic-gradient`
- Maskierung des mittleren Bereichs mit dem `::after`-Pseudoelement (optional)
- Implementierung der Rotationsanimation mit `@keyframes`

### 2. Seitliche leuchtende Linie

Dies eignet sich für die Statusanzeige von Listenelementen – erstellen Sie einfach eine leuchtende dünne Linie auf einer Seite des Elements, ohne dass der gesamte Rahmen bewegt wird. Denn manchmal reicht ein bisschen Licht aus, man muss nicht die ganze Welt beleuchten.

**Wesentliche Elemente:**
- Absolut positioniertes dünnes Linienelement
- Erstellung des Leuchteffekts mit `box-shadow`
- Implementierung der atmenden Animation mit `scale` und `opacity`

### 3. Box-Shadow leuchtender Hintergrund

Wenn Sie keinen Rundum-Effekt benötigen, sondern nur einen sanften Hintergrund-Lichtschein wollen, reicht die Überlagerung mehrerer `box-shadow`-Ebenen aus. Manche Dinge sind einfacher besser.

### 4. Barrierefreiheitsunterstützung

Dies wird oft übersehen, ist aber besonders wichtig. Alle Animationen sollten die Medienabfrage `prefers-reduced-motion` berücksichtigen und Benutzern, die keine Animationen mögen, eine statische Alternative bieten. Schließlich mag nicht jeder Bewegung, und die Wahl jedes Einzelnen zu respektieren, ist das Richtige.

## Implementierungslösungen

### Lösung 1: Conic Gradient rotierender Rahmen (empfohlen)

Dies ist die vollständigste Implementierung von Rundum-Lichteffekten und auch die im HagiCode am häufigsten verwendete Lösung. Denn wenn etwas gut funktioniert, warum sollte man es ändern?

```css
/* Übergeordneter Container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotierender Lichtschein-Layer */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Maskierungsebene (optional, zum Erstellen eines hohlen Rahmeneffekts) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Das Prinzip dieser Lösung ist ziemlich einfach: Erstellen Sie ein Pseudoelement, das größer als das übergeordnete Element ist, zeichnen Sie einen konischen Farbverlauf darauf und lassen Sie es ununterbrochen rotieren. Das übergeordnete Element hat `overflow: hidden` eingestellt, sodass man nur den Lichtteil am Rand rotieren sieht. Wie wenn wir aus dem Fenster auf eine Straßenlaterne schauen – man sieht nur den kleinen Abschnitt, an dem sie vorbeikommt.

### Lösung 2: Vereinfachte rotierende lichtrandige Umrandung

Wenn Sie keinen so komplexen Effekt benötigen, gibt es in HagiCode eine leichtere Utility-Class-Implementierung. Denn einfacher ist manchmal besser.

```css
/* Utility-Class für rotierende lichtrandige Umrandung */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Barrierefreiheitsunterstützung */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Beachten Sie hier das `will-change: transform` – dies teilt dem Browser mit: "Dieses Element wird sich ständig ändern", und der Browser wird im Voraus einige Optimierungen vornehmen, sodass die Animation flüssiger verläuft. Schließlich ist Vorbereitung besser als letzte Hast.

### Lösung 3: Seitliche leuchtende Linie

Besonders geeignet für die Statusanzeige von Listenelementen – die HagiCode-Sitzungsliste verwendet genau diese Lösung. Eine dünne Linie, die sich unter zahlreichen Projekten hervorhebt – ist das nicht auch eine Lebensphilosophie?

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Hier wird `isolation: isolate` verwendet, um einen neuen Stapelkontext zu erstellen, und dann `z-index` verwendet, um die Anzeigereihenfolge der Ebenen zu steuern. `pointer-events: none` ist ebenfalls entscheidend, sonst würde das Pseudoelement die Klickoperationen des Benutzers blockieren. Wie bei manchen Dingen – hübsch ist hübsch, aber sie dürfen nicht im Wege sein.

### Lösung 4: React-Komponentenkapselung

Wenn Sie in Ihrem Projekt React verwenden, können Sie eine Komponente kapseln, die diese Logik verarbeitet, insbesondere den Barrierefreiheitsteil. Denn Code einmal schreiben, mehrfach verwenden – das ist es, was wir wollen.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Das entsprechende CSS-Modul:

```css
/* GlowBorder.module.css */

/* Animationsversion */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Statische Version (Barrierefreiheit) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Der `useReducedMotion`-Hook von `framer-motion` erkennt automatisch die Systempräferenzen des Benutzers. Wenn der Benutzer "Bewegung reduzieren" aktiviert hat, gibt er true zurück, und in diesem Fall wird die statische Version angezeigt. Schließlich ist die Entscheidung des Benutzers zu respektieren wichtiger als eine erzwungene Präsentation.

## Praktische Erfahrungen

Das Folgende sind Erfahrungen, die wir bei der Arbeit an HagiCode gesammelt haben – Fallstricke, in die wir getappt sind, und daraus gewonnene Erkenntnisse. Eigentlich nur ein paar Gedanken, aber ich hoffe, sie können Ihnen helfen.

### 1. Themenvariablen-System

Die Verwendung von CSS-Variablen zur Unterstützung mehrerer Themen ist besonders praktisch. Schließlich will niemand bei jedem Themenwechsel einen Haufen Code ändern?

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Verwendung */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

So müssen Sie beim Wechseln des Themas nur die Klasse des `html`-Tags ändern, und alle Animationsfarben werden automatisch aktualisiert. Ein Code, zwei Stile – ist das nicht das, worauf wir hinauswollen?

### 2. Leistungsoptimierung

**Verwenden Sie `will-change`, um den Browser zur Optimierung anzuhalten:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Teilen Sie dem Browser im Voraus mit, und er wird einige Optimierungen für Sie vornehmen. Wie bei vielen Dingen im Leben – Vorbereitung ist immer gut.

**Vermeiden Sie komplexe box-shadow auf großen Elementen:**

```css
/* Nicht gut - Unschärfe-Schatten auf großen Elementen */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Besser - Verwenden Sie Pseudoelemente, um den Leuchtbereich zu begrenzen */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Wir haben in HagiCode getestet, dass das direkte Hinzufügen von Unschärfe-Schatten auf großen Karten die Bildrate beim Scrollen auf unter 30fps sinken lässt, nach der Umstellung auf Pseudoelemente wurden stabil 60fps erreicht. Dieser Unterschied im Erlebnis können Benutzer tatsächlich spüren.

### 3. Barrierefreiheit

Das darf wirklich nicht weggelassen werden – manche Benutzer finden Animationen schwindelerregend oder laut, und ihre Wahl zu respektieren, ist grundlegende Anstand bei der Produktentwicklung. Schließlich muss man schöne Dinge niemandem aufzwingen.

**CSS-Medienabfrage:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Statische Alternative bereitstellen */
    opacity: 1;
  }
}
```

**Erkennung der Benutzereinstellungen in React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Intitätsstufensteuerung

Der Token-Durchsatzindikator in HagiCode zeigt je nach Echtzeit-Durchsatz unterschiedlichfarbige Lichter – dies ist dynamisch implementiert. Verschiedene Zustände sollten schließlich auch unterschiedliche Ausdrucksweisen haben.

```tsx
const colors = [
  null,       // Level 0 - keine Farbe
  '#3b82f6',  // Level 1 - Blau
  '#34d399',  // Level 2 - Smaragdgrün
  '#facc15',  // Level 3 - Gelb
  '#fbbf24',  // Level 4 - Bernstein
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rosa
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Zu beachtende Punkte

Einige Details sollten beachtet werden, sonst ist es zu spät, wenn man in die Falle getappt ist.

| Zu beachtende Punkte | Beschreibung |
|---------------------|-------------|
| **z-index-Verwaltung** | Der Lichtschein-Layer sollte einen geeigneten z-index haben, um die Inhaltsinteraktion nicht zu beeinträchtigen |
| **pointer-events** | Für Lichtschein-Pseudoelemente sollte `pointer-events: none` eingestellt werden |
| **Randüberlauf** | Das übergeordnete Element muss `overflow: hidden` eingestellt haben oder die Pseudoelementgröße anpassen |
| **Leistungseinfluss** | Komplexe Animationen können auf mobilen Geräten die Leistung beeinträchtigen, Testen ist erforderlich |
| **Dunkelmodus** | Sicherstellen, dass Leuchtfarben auf dunklem Hintergrund klar sichtbar sind |
| **Themenwechsel** | Verwenden Sie CSS-Variablen, um sicherzustellen, dass Animationsfarben bei Themenwechsel korrekt aktualisiert werden |

### 6. Debugging-Tipps

Pseudoelemente sind in den Entwicklertools manchmal schwer zu finden – Sie können vorübergehend einen Rahmen hinzufügen, um die Position zu sehen.

```css
/* Vorübergehende Anzeige der Pseudoelementgrenzen zum Debuggen */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Nachdem Sie die Position eingestellt haben, denken Sie daran, diese Zeile auszukommentieren oder zu löschen, sonst ist es in der Produktionsumgebung peinlich. Manche Dinge bleiben besser in der Entwicklungsumgebung.

## Zusammenfassung

Animierte Rahmenlichteffekte sind weder schwer noch einfach – der Kern ist `conic-gradient` plus Rotation, aber um gute Leistung, Wartbarkeit und Barrierefreiheit zu erreichen, gibt es noch viele Details zu beachten.

HagiCode ist in diese hineingetappt und hat einige bewährte Praktiken zusammengetragen. Eigentlich ist das bei Projekten so – immer wieder ausprobieren, immer wieder verbessern. Wenn Sie an ähnlichen Anforderungen arbeiten, hoffe ich, dass dieser Artikel Ihnen hilft, ein paar Umwege zu vermeiden.

Schließlich muss man manche Dinge selbst ausprobieren, um die Tiefe zu kennen.
