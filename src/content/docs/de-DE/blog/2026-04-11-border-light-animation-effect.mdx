## Leitfaden zur Erstellung eines Rahmen-Licht-Sweep-Animationseffekts

> Wie baut man dieses wichtige Element, das Benutzer auf einen Blick bemerken, mit reinem CSS? Es ist eigentlich nicht so schwer. Der Trick besteht nur darin, einen leicht umwegigen Weg zu nehmen. In diesem Artikel fuehre ich Sie Schritt fuer Schritt durch den Aufbau einer Rahmen-Licht-Sweep-Animation von Grund auf und teile einige der Stolpersteine, auf die wir beim Bau von HagiCode gestossen sind.

## Hintergrund

Wenn Sie im Frontend arbeiten, haben Sie wahrscheinlich schon diese Erfahrung gemacht: ein Produktmanager kommt mit diesem „das ist definitiv einfach"-Gesichtsausdruck und sagt: „Koennen wir eine Art Spezialeffekt zu dieser laufenden Aufgabe hinzufuegen, damit Benutzer sie sofort erkennen koennen?"

Sie sagen natuerlich, wir koennen einfach die Rahmenfarbe aendern. Dann schuetteln sie den Kopf mit diesem Blick, der sagt: „Sie verstehen es nicht." Sie antworten: „Das ist nicht offensichtlich genug. Ich moechte die Art von Effekt, bei der Licht um den Rahmen laeuft, wie in einem Science-Fiction-Film."

An diesem Punkt fragen Sie sich vielleicht, wie man das baut. Canvas? SVG? Oder kann CSS das alleine? Schliesslich moechte niemand zugeben, dass er es nicht weiss.

In modernen Webanwendungen sind Rahmen-Licht-Sweep-Animationen eigentlich sehr haeufig. Sie werden hauptsaechlich in einigen Szenarien wie diesen verwendet:

- **Statusanzeigen**: Markierung von laufenden Aufgaben oder aktiven Elementen
- **Visueller Fokus**: Hervorheben wichtiger Inhaltsbereiche
- **Markenverstaerkung**: Schaffung eines schlanken, modernen, technikorientierten visuellen Stils
- **Saisonale Themen**: Aufbau einer festlichen Atmosphaere fuer besondere Anlaesse

Genau diese Anforderung haben wir beim Bau von [HagiCode](https://hagicode.com) getroffen. Benutzer mussten auf einen Blick sehen, welche Sitzungen liefen und welche Vorschlaege gerade verarbeitet wurden. Wir haben mehrere verschiedene Ansaetze versucht. Manche Wege waren glatter, andere etwas kurvenreicher. Am Ende entschieden wir uns fuer eine recht ausgereifte Implementierungsstrategie.

## Kergedesignideen

Nach der Analyse der HagiCode-Codebasis haben wir mehrere Kergedesignmuster zusammengefasst. Jedes passt zu einem anderen Szenario – oder anders gesagt: jedes existiert aus einem Grund.

### 1. Rotierendes Leuchten mit einem conic-Gradient (am haeufigsten)

Dies ist der klassische Weg zur Implementierung eines Rahmen-Licht-Sweep-Effekts. Die Kernidee besteht darin, CSS `conic-gradient` zu verwenden, um einen konischen Gradienten zu erzeugen, und ihn dann kontinuierlich zu rotieren. Wie eine Strassenlaterne, die sich in der Nacht dreht – sie dreht sich einfach immer weiter.

**Schluesselelemente:**
- Verwendung des `::before`-Pseudoelements zum Erstellen der Leuchtschicht
- Verwendung von `conic-gradient` zur Definition der Gradientenfarbverteilung
- Verwendung des `::after`-Pseudoelements zum Maskieren des Zentralbereichs (optional)
- Verwendung von `@keyframes` zur Implementierung der Rotationsanimation

### 2. Seitenleuchtlinie

Dies funktioniert gut fuer Statusanzeigen in Listenelementen. Sie erstellen eine duenne Leuchtlinie auf einer Seite des Elements, anstatt den gesamten Rahmen zu animieren. Manchmal reicht ein wenig Licht. Sie muessen nicht die ganze Welt beleuchten.

**Schluesselelemente:**
- Ein duennes, absolut positioniertes Linienelement
- Verwendung von `box-shadow` zur Erzeugung des Leuchteffekts
- Verwendung von `scale` und `opacity` fuer eine Atemanimation

### 3. Leuchtender Hintergrund mit box-shadow

Wenn Sie nicht den vollen Sweep-Effekt benoetigen und nur einen sanften Hintergrundschimmer moechten, reicht das Schichten mehrerer `box-shadow`-Werte aus. Manchmal ist die einfachere Option die bessere.

### 4. Barrierefreiheitsunterstuetzung

Dieser Teil wird leicht uebersehen, ist aber extrem wichtig. Jede Animation sollte die `prefers-reduced-motion`-Media-Query beruecksichtigen und eine statische Alternative fuer Benutzer bereitstellen, die keine Animation moechten. Nicht jeder geniesst staendige Bewegung, und dieser Praeferenz Rechnung zu tragen ist wichtig.

## Implementierungsoptionen

### Option 1: Rotierender conic-gradient-Rahmen (empfohlen)

Dies ist die vollstaendigste Implementierung des Sweep-Rahmenlichteffekts und auch die Option, die wir in HagiCode am haeufigsten verwenden. Schliesslich: wenn etwas gut funktioniert, warum es ersetzen?

```css
/* Uebergeordnetes Container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotierende Leuchtschicht */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Maskierschicht (optional, fuer einen hohlen Rahmeneffekt) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Das Prinzip hinter dieser Option ist recht einfach: ein Pseudoelement groesser als der Uebergeordnetecontainer erstellen, einen konischen Gradienten darauf zeichnen und ihn kontinuierlich rotieren lassen. Der Uebergeordnetecontainer verwendet `overflow: hidden`, sodass nur das Licht sichtbar bleibt, das am Rahmen vorbeilaeuft. Es ist ein bisschen wie eine Strassenlaterne durch ein Fenster zu beobachten. Man sieht immer nur das kleine Stueck, das vorbeifliegt.

### Option 2: Vereinfachter rotierender Lichtrahmen

Wenn Sie den vollen Effekt nicht benoetigen, enthaelt HagiCode auch eine leichtgewichtigere Utility-Class-Version. Manchmal ist der einfachere Ansatz wirklich besser.

```css
/* Rotierender Lichtrahmen Utility-Klasse */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Barrierefreiheitsunterstuetzung */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Beachten Sie das `will-change: transform` hier. Es sagt dem Browser: „Dieses Element wird sich staendig aendern," damit der Browser einige Optimierungen im Vorneherein vorbereiten und die Animation fluessiger halten kann. Vorbereitung im Voraus ist meistens besser als letztes Drangeln.

### Option 3: Seitenleuchtlinie

Dies ist besonders geeignet fuer Listenelement-Statusanzeigen und genau das, was die HagiCode-Sitzungsliste verwendet. Eine duenne Linie kann sich trotzdem unter vielen Elementen hervorheben. Das fuehlt sich ein bisschen an wie eine Lebensweisheit.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Hier wird `isolation: isolate` verwendet, um einen neuen Stapelkontext zu erstellen, dann wird die Anzeigereihenfolge jeder Schicht ueber `z-index` gesteuert. `pointer-events: none` ist ebenfalls unerlaesslich. Andernfalls wuerde das Pseudoelement Benutzerklicks blockieren. Manche Dinge koennen nett aussehen, sollten aber trotzdem nicht im Weg stehen.

### Option 4: In eine React-Komponente verpacken

Wenn Ihr Projekt React verwendet, koennen Sie diese Logik in eine Komponente verpacken, insbesondere die Barrierefreiheitsbehandlung. Einmal schreiben, mehrfach verwenden. Genau darum geht es.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Das passende CSS-Modul:

```css
/* GlowBorder.module.css */

/* Animierte Version */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Statische Version (Barrierefreiheit) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Der `useReducedMotion`-Hook aus `framer-motion` erkennt automatisch die Systempraeferenz des Benutzers. Wenn der Benutzer reduzierte Bewegung aktiviert hat, gibt er `true` zurueck und die Komponente zeigt die statische Version an. Der Praeferenz des Benutzers Rechnung zu tragen ist wichtiger als das Erzwingen eines auffaelligen Effekts.

## Praktische Erkenntnisse

Dies sind einige der Lektionen, die wir beim Bau von HagiCode gelernt haben. Man koennte sie auch Kampfnarben nennen. Hoffentlich helfen sie Ihnen, einige Umwege zu vermeiden.

### 1. Theme-Variablensystem

CSS-Variablen machen die Mehrfach-Theme-Unterstuetzung besonders bequem. Niemand moechte bei jedem Themawechsel einen Berg Code bearbeiten.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Verwendung */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

So erfordert das Wechseln des Themes nur das Aendern der Klasse auf dem `html`-Element, und jede Animationsfarbe aktualisiert sich automatisch. Eine Codebasis, zwei Stile. Genau das wollen wir.

### 2. Leistungsoptimierung

**`will-change` verwenden, um den Browser zur Optimierung anzuhalten:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Sagen Sie es dem Browser im Voraus, und er wird Ihnen helfen zu optimieren. Vieles im Leben funktioniert besser mit ein wenig Vorbereitung.

**Komplexe Box-Shadows auf grossen Elementen vermeiden:**

```css
/* Nicht ideal - verschwommener Schatten auf einem grossen Element */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Besser - Pseudoelement verwenden, um den Leuchtbereich einzuschraenken */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Wir haben das in HagiCode getestet. Das Hinzufuegen eines verschwommenen Schattens direkt zu einer grossen Karte senkte die Scroll-Frame-Rate unter 30fps. Der Wechsel zu einem Pseudoelement brachte die Dinge zurueck auf stabile 60fps. Benutzer koennen diesen Unterschied definitiv spueren.

### 3. Barrierefreiheit

Das sollten Sie wirklich nicht ueberspringen. Manche Benutzer empfinden Animationen als schwindelerregend oder ablenkend, und ihren Praeferenzen Rechnung zu tragen, ist Teil eines guten Produkts. Schoene Dinge muessen nicht jedem aufgezwungen werden.

**CSS-Media-Query:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Statische Alternative bereitstellen */
    opacity: 1;
  }
}
```

**Benutzerpraeferenz in React erkennen:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Inhalt
    </div>
  );
};
```

### 4. Intensitaetsstufensteuerung

Der Token-Durchsatzindikator in HagiCode zeigt verschiedene Leuchtfarben basierend auf dem Echtzeit-Durchsatz an, und dies wird dynamisch implementiert. Verschiedene Zustaende sollten unterschiedlich ausgedrueckt werden.

```tsx
const colors = [
  null,       // Stufe 0 - keine Farbe
  '#3b82f6',  // Stufe 1 - Blau
  '#34d399',  // Stufe 2 - Smaragdgruen
  '#facc15',  // Stufe 3 - Gelb
  '#fbbf24',  // Stufe 4 - Bernstein
  '#f97316',  // Stufe 5 - Orange
  '#22d3ee',  // Stufe 6 - Cyan
  '#d946ef',  // Stufe 7 - Fuchsia
  '#f43f5e',  // Stufe 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Dinge, auf die Sie achten sollten

Es gibt noch einige Details, die Beachtung verdienen, denn bis Sie diese Probleme auf die harte Tour entdecken, ist es bereits zu spaet.

| Dinge, auf die achten | Erklaerung |
|---------|------|
| **z-index-Verwaltung** | Die Leuchtschicht sollte einen geeigneten `z-index` verwenden, damit sie die Inhaltsinteraktion nicht stoert |
| **pointer-events** | Das Leucht-Pseudoelement sollte `pointer-events: none` setzen |
| **Randueberlauf** | Der Uebergeordnetecontainer benoetigt `overflow: hidden`, oder Sie muessen die Pseudoelementgroesse anpassen |
| **Leistungsauswirkung** | Komplexe Animationen koennen die Leistung auf Mobilgeraeten beeintraechtigen, also sorgfaeltig testen |
| **Dark Mode** | Stellen Sie sicher, dass die Leuchtfarbe auf dunklen Hintergruenden deutlich sichtbar bleibt |
| **Theme-Wechsel** | Verwenden Sie CSS-Variablen, damit sich Animationsfarben beim Themawechsel korrekt aktualisieren |

### 6. Debugging-Tipps

Pseudoelemente koennen in Entwicklertools schwer zu finden sein, also koennen Sie temporaer einen Rahmen hinzufuegen, um die Position zu pruefen.

```css
/* Temporaer Pseudoelement-Grenzen fuer Debugging anzeigen */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Nachdem Sie die Positionierung abgeschlossen haben, denken Sie daran, diese Zeile auszukommentieren oder zu entfernen. Andernfalls kann es in der Produktion schnell peinlich werden. Manche Dinge sind besser in der Entwicklung belassen.

## Zusammenfassung

Rahmen-Licht-Sweep-Animationen sind weder besonders schwer noch wirklich trivial. Im Kern ist die Formel `conic-gradient` plus Rotation, aber wenn Sie gute Leistung, Wartbarkeit und Barrierefreiheit moechten, gibt es noch einige Implementierungsdetails, die sorgfaeltig behandelt werden muessen.

HagiCode ist in viele dieser Stolpersteine getappt und hat schrittweise eine Reihe von Best Practices destilliert. So gehen Projekte: man experimentiert, macht Fehler und verbessert sich Schritt fuer Schritt. Wenn Sie etwas Aehnliches bauen, hoffe ich, dass dieser Artikel Ihnen hilft, einige unnötige Umwege zu vermeiden.

Manche Dinge werden erst klar, wenn man sie selbst baut.
