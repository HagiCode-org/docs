## Wie man Projekte im KI-Zeitalter reproduziert: Vault, ein projektuebergreifendes persistentes Speichersystem

> Im Zeitalter der KI-unterstuetzten Entwicklung: Wie koennen wir KI-Assistenten helfen, unsere Lernressourcen besser zu verstehen? Das HagiCode-Projekt baute das Vault-System als einheitliche Wissensspeicher-Abstraktionsschicht, die die KI verstehen kann, was die Effizienz des Lernens durch Projektreproduktion erheblich verbessert.

## Hintergrund

Im KI-Zeitalter aendert sich die Art und Weise, wie Entwickler neue Technologien und Architekturen lernen, grundlegend. „Projekte reproduzieren" - also das vertiefte Studium und Lernen aus dem Code, der Architektur und den Entwurfsmustern hervorragender Open-Source-Projekte - ist zu einer effizienten Lernmethode geworden.

Dennoch bringt diese Lernmethode einige Herausforderungen mit sich.

Lernmaterialien sind zu verstreut. Ihre Notizen liegen moeglicherweise in Obsidian, Coderepositories sind ueber verschiedene Ordner verstreut, und der Gespraechsverlauf Ihres KI-Assistenten wird zu einer weiteren isolierten Dateninsel.

Noch problematischer ist der gebrochene Kontext. KI-Assistenten koennen nicht direkt auf Ihre lokalen Lernressourcen zugreifen, sodass Sie in jedem Gespraech Hintergrundinformationen erneut bereitstellen muessen.

Im Kern stammen alle diese Probleme aus „Dateninseln". Wenn es eine einheitliche Speicher-Abstraktionsschicht gaebe, die KI-Assistenten verstehen und auf alle Ihre Lernressourcen zugreifen koennten, waere das Problem elegant geloest.

## Ueber HagiCode

Das in diesem Artikel geteilte Vault-System ist genau die Loesung, die wir beim Bau von [HagiCode](https://hagicode.com) entwickelt haben. HagiCode ist ein KI-Programmierassistent-Projekt, und in unserer taeglichen Entwicklungsarbeit muessen wir haeufig viele verschiedene Open-Source-Projekte studieren und referenzieren. Um KI-Assistenten dabei zu helfen, diese Lernressourcen besser zu verstehen, entwarfen wir Vault, ein projektuebergreifendes persistentes Speichersystem.

## Designphilosophie des Vault-Systems

Die Kernidee des Vault-Systems ist einfach: eine einheitliche Wissensspeicher-Abstraktionsschicht erstellen, die die KI verstehen kann. Aus der Implementierungsperspektive hat das System mehrere wesentliche Merkmale.

### Mehrfach-Typ-Unterstuetzung

Das System unterstuetzt vier Vault-Typen, die jeweils einem unterschiedlichen Nutzungsszenario entsprechen:

```typescript
// folder: allgemeiner Ordnertyp
export const DEFAULT_VAULT_TYPE = 'folder';

// coderef: ein Typ speziell fuer reproduzierte Code-Projekte
export const CODEREF_VAULT_TYPE = 'coderef';

// obsidian: Integration mit Obsidian-Notizsoftware
export const OBSIDIAN_VAULT_TYPE = 'obsidian';

// system-verwaltet: Vault automatisch vom System verwaltet
export const SYSTEM_MANAGED_VAULT_TYPE = 'system-managed';
```

Unter ihnen ist der **coderef**-Typ der am haeufigsten verwendete in HagiCode. Er ist speziell fuer reproduzierte Code-Projekte konzipiert und bietet eine standardisierte Verzeichnisstruktur und KI-lesbare Metadatenbeschreibungen.

### Persistenter Speichermechanismus

Die Vault-Registry wird persistent im JSON-Format gespeichert, um sicherzustellen, dass die Konfiguration nach dem Neustart der Anwendung verfuegbar bleibt:

```csharp
public class VaultRegistryStore : IVaultRegistryStore
{
    private readonly string _registryFilePath;

    public VaultRegistryStore(IConfiguration configuration, ILogger<VaultRegistryStore> logger)
    {
        var dataDir = configuration["DataDir"] ?? "./data";
        var absoluteDataDir = Path.IsPathRooted(dataDir)
            ? dataDir
            : Path.GetFullPath(Path.Combine(Directory.GetCurrentDirectory(), dataDir));

        _registryFilePath = Path.Combine(absoluteDataDir, "personal-data", "vaults", "registry.json");
    }
}
```

Der Vorteil dieses Designs ist, dass es einfach und zuverlaessig ist. JSON ist menschenlesbar, was das Debugging und manuelle Bearbeiten erleichtert; Dateisystemspeicherung vermeidet die Komplexitaet einer Datenbank und reduziert Systemabhaengigkeiten.

### KI-Kontextintegration

Wichtig ist, dass das System Vault-Informationen automatisch in den Kontext von KI-Vorschlaegen injizieren kann:

```typescript
export function buildTargetVaultsText(
  vaults: VaultForText[],
  template: VaultPromptTemplate = DEFAULT_VAULT_PROMPT_TEMPLATE,
): string {
  const readOnlyVaults = vaults.filter((vault) => vault.accessType === 'read');
  const editableVaults = vaults.filter((vault) => vault.accessType === 'write');

  if (readOnlyVaults.length === 0 && editableVaults.length === 0) {
    return '';
  }

  const sections = [
    buildVaultSection(readOnlyVaults, template.reference),
    buildVaultSection(editableVaults, template.editable),
  ].filter(Boolean);

  return `\n\n### ${template.heading}\n\n${sections.join('\n')}`;
}
```

Dies ermoeglicht eine wichtige Faehigkeit: KI-Assistenten koennen automatisch die verfuegbaren Lernressourcen verstehen, ohne dass Nutzer manuell Kontext bereitstellen muessen.

## Die standardisierte Struktur von CodeRef Vault

Fuer den coderef-Typ von Vault bietet HagiCode eine standardisierte Verzeichnisstruktur:

```
my-coderef-vault/
├── index.yaml          # Vault-Metadatenbeschreibung
├── AGENTS.md           # Betriebsanleitung fuer KI-Assistenten
├── docs/               # speichert Lernnotizen und Dokumente
└── repos/              # verwaltet reproduzierte Coderepositories durch Git-Submodule
```

Beim Erstellen eines Vault initialisiert das System automatisch diese Struktur:

```csharp
private async Task EnsureCodeRefStructureAsync(
    string vaultName,
    string physicalPath,
    ICollection<VaultBootstrapDiagnosticDto> diagnostics,
    CancellationToken cancellationToken)
{
    Directory.CreateDirectory(physicalPath);

    var indexPath = Path.Combine(physicalPath, CodeRefIndexFileName);
    var docsPath = Path.Combine(physicalPath, CodeRefDocsDirectoryName);
    var reposPath = Path.Combine(physicalPath, CodeRefReposDirectoryName);

    // Standardverzeichnisstruktur erstellen
    if (!Directory.Exists(docsPath))
    {
        Directory.CreateDirectory(docsPath);
    }

    if (!Directory.Exists(reposPath))
    {
        Directory.CreateDirectory(reposPath);
    }

    // AGENTS.md-Leitfaden erstellen
    await EnsureCodeRefAgentsDocumentAsync(physicalPath, cancellationToken);

    // index.yaml-Metadaten erstellen
    await WriteCodeRefIndexDocumentAsync(indexPath, mergedDocument, cancellationToken);
}
```

Diese Struktur ist sorgfaeltig gestaltet:

- **docs/** speichert Ihre Lernnotizen, wo Sie Ihr Verstaendnis des Codes, die Architekturanalyse, Erfahrungen usw. in Markdown festhalten koennen
- **repos/** verwaltet reproduzierte Repositories durch Git-Submodule statt Code direkt zu kopieren, was den Code synchron und Platz sparend haelt
- **index.yaml** enthaelt die Vault-Metadaten, sodass KI-Assistenten schnell Zweck und Inhalt des Vault verstehen koennen
- **AGENTS.md** ist ein Leitfaden, der speziell fuer KI-Assistenten geschrieben ist

## Automatische Initialisierung fuer systemverwaltete Vaults

Neben manuell erstellten Vaults unterstuetzt HagiCode auch systemverwaltete Vaults:

```csharp
public async Task<IReadOnlyList<VaultRegistryEntry>> EnsureAllSystemManagedVaultsAsync(
    CancellationToken cancellationToken = default)
{
    var definitions = GetAllResolvedDefinitions();
    var entries = new List<VaultRegistryEntry>(definitions.Count);

    foreach (var definition in definitions)
    {
        entries.Add(await EnsureResolvedSystemManagedVaultAsync(definition, cancellationToken));
    }

    return entries;
}
```

Das System erstellt und verwaltet automatisch folgende Vaults:

- **hagiprojectdata**: Projektdatenspeicher zum Speichern von Projektkonfiguration und -zustand
- **personaldata**: Persoenlicher Datenspeicher zum Speichern von Nutzervoreinstellungen
- **hbsprompt**: Eine Prompt-Vorlagenbibliothek zur Verwaltung haeufig verwendeter KI-Prompts

Diese Vaults werden beim Systemstart automatisch initialisiert, sodass Nutzer sie nicht manuell konfigurieren muessen.

## Zugriffssteuerungsmechanismus

Ein wichtiger Teil des Designs ist die Zugriffssteuerung. Das System teilt Vaults in zwei Zugriffstypen:

```typescript
export interface VaultForText {
  id: string;
  name: string;
  type: string;
  physicalPath: string;
  accessType: 'read' | 'write';  // Schluessel: Nur-Lese von Bearbeitbar unterscheiden
}
```

- **reference (nur lesen)**: KI wird nur zur Analyse und zum Verstaendnis verwendet und kann Inhalte nicht aendern. Geeignet fuer referenzierte Open-Source-Projekte, Dokumente und aehnliche Materialien
- **editable (bearbeitbar)**: KI kann Inhalte nach Bedarf fuer die Aufgabe aendern. Geeignet fuer Ihre Notizen, Entwuerfe und aehnliche Materialien

Diese Unterscheidung ist wichtig. Sie sagt der KI, welche Inhalte „nur-Lese-Referenz" und welche „sicher zu bearbeiten" sind, was das Risiko unbeabsichtigter Aenderungen reduziert.

## In der Praxis: Vault erstellen und verwenden

### Einen CodeRef Vault erstellen

Hier ist ein vollstaendiges Frontend-Aufrufbeispiel:

```typescript
const createCodeRefVault = async () => {
  const response = await VaultService.postApiVaults({
    requestBody: {
      name: "React Learning Vault",
      type: "coderef",
      physicalPath: "/Users/developer/vaults/react-learning",
      gitUrl: "https://github.com/facebook/react.git"
    }
  });

  // Das System wird automatisch:
  // 1. Das React-Repository in vault/repos/react klonen
  // 2. Das docs/-Verzeichnis fuer Notizen erstellen
  // 3. Die index.yaml-Metadaten generieren
  // 4. Die AGENTS.md-Leitfaddatei erstellen

  return response;
};
```

## Bewaehrte Praktiken und Hinweise

### Pfadsicherheit

Das System validiert Pfade streng, um Pfad-Traversal-Angriffe zu verhindern:

```csharp
private static string ResolveFilePath(string vaultRoot, string relativePath)
{
    var rootPath = EnsureTrailingSeparator(Path.GetFullPath(vaultRoot));
    var combinedPath = Path.GetFullPath(Path.Combine(rootPath, relativePath));
    if (!combinedPath.StartsWith(rootPath, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode,
            "Vault-Dateipfade muessen innerhalb des registrierten Vault-Stammverzeichnisses bleiben.");
    }
    return combinedPath;
}
```

### Git-Submodul-Verwaltung

CodeRef Vault empfiehlt Git-Submodule statt Code direkt zu kopieren:

```csharp
private static string BuildCodeRefAgentsContent()
{
    return """
    # CodeRef Vault Leitfaden

    Repositories unter `repos/` sollten durch Git-Submodule verwaltet werden
    anstatt direkt in das Vault-Stammverzeichnis kopiert zu werden.

    Halten Sie diese Struktur stabil, damit Assistenten und Werkzeuge den Vault schnell verstehen koennen.
    """ + Environment.NewLine;
}
```

## Zusammenfassung

Durch eine einheitliche Speicher-Abstraktionsschicht loest das Vault-System mehrere Kernschmerzpunkte der Projektreproduktion im KI-Zeitalter:

- **Zentralisiertes Wissensmanagement**: Alle Lernressourcen sind an einem Ort statt ueberall verstreut
- **Automatische KI-Kontextinjektion**: KI-Assistenten koennen automatisch die verfuegbaren Lernressourcen verstehen, ohne manuelle Kontexteinrichtung
- **Projektuebergreifende Wiederverwendung von Wissen**: Wissen kann ueber mehrere Lernprojekte geteilt und wiederverwendet werden
- **Standardisierte Verzeichnisstruktur**: Ein konsistentes Verzeichnislayout senkt die Lernkurve

Manchmal ist der einfache Ansatz tatsaechlich der beste.
