完成 Desktop 安装并首次启动后，Hagicode 会直接进入初始化向导。新版向导不再把“准备工具”“整理默认执行者”“创建项目”拆成互相割裂的入口，而是按当前产品顺序把首次使用需要确认的内容串成一条 8 步流程。

## 向导总览

当前初始化向导按以下顺序推进：

1. 选择界面语言与 AI 文案语言
2. 检查 Node.js、npm、OpenSpec 等系统组件
3. 选择界面颜色主题
4. 选择文案主题
5. 开启并整理 AgentCLI 能力
6. 配置 Agents，包括成员、岗位分配与 fallback / backup
7. 创建第一个项目，把向导结果接入真实工作目录
8. 查看社区入口，继续获取更新、教程与交流渠道

## 步骤 1：语言

![初始化向导语言设置界面](../img/screenshots/shared/guide1-language/original.png)

语言步骤先确定你希望如何与 Hagicode 交互。这里通常会同时确认两类设置：

- **界面语言**：决定 Desktop 和 Web 界面的显示语言。
- **AI 文案语言**：决定默认生成内容、说明文本或引导文案更偏向哪种语言。

如果你需要让团队成员使用不同语言，可以先从最熟悉的语言开始，后续再回到设置页调整。

## 步骤 2：系统组件检测

![初始化向导系统组件检测界面](../img/screenshots/shared/guide2-init-env/original.png)

第二步会检查当前设备是否已经具备向导后续步骤依赖的基础组件。截图中可以看到 Node.js、npm 与 OpenSpec 等关键项的状态。

这一页的作用不是让你手工逐项安装，而是先确认“当前环境是否已经达到可用状态”。当所有必需组件都准备完成后，向导才适合继续推进到 AgentCLI 与项目阶段。

## 步骤 3：颜色主题

![初始化向导颜色主题界面](../img/screenshots/shared/guide3-select-theme/original.png)

颜色主题决定的是产品界面的视觉风格，例如浅色或深色界面。这个选择会直接影响 Desktop 与相关工作界面的阅读体验。

如果你经常长时间查看终端、日志或代码窗口，建议优先选择自己日常最熟悉的配色，而不是只为了“看起来新鲜”而频繁切换。

## 步骤 4：文案主题

![初始化向导文案主题界面](../img/screenshots/shared/guide4-select-doc-theme/original.png)

文案主题会影响向导、角色描述和部分默认文案的叙事风格。截图中展示了不同风格的可选卡片，例如更偏基础说明的基线风格，以及更强调世界观感的 Fantasy Adventure 风格。

你可以把这一步理解为“产品默认叙事语气”的选择：界面功能不变，但引导文本、角色感受与整体包装会更贴合你喜欢的使用氛围。

## 步骤 5：AgentCLI Setup

![初始化向导 AgentCLI 设置界面](../img/screenshots/shared/guide5-enable-agent-cli/original.png)

AgentCLI Setup 用于决定哪些命令行执行能力会被纳入当前默认环境。常见动作包括：

- 开启或关闭某个 AgentCLI 提供者
- 查看它的状态、说明与适用场景
- 确认后续 Agents 能否调用这些能力

这一步决定了后面“角色能做什么”。如果你计划让不同 Agents 使用不同 CLI 能力，先在这里把基础能力准备好，后面的岗位分配会更清晰。

## 步骤 6：Agents

Agents 步骤负责把已经可用的 AgentCLI 能力编排成真正可执行的默认成员阵容。它在界面上拆成多个连续状态，但仍然属于同一个主步骤。

### 6.1 添加成员

![初始化向导 Agents 添加成员界面](../img/screenshots/shared/guide6-add-agents/original.png)

首先是添加成员。你会在表格中看到成员名称、所选 AgentCLI、模型与角色模板等配置项。这里的重点是先把“默认会有哪些执行者”确定下来。

如果你的工作流需要不同角色承担不同任务，例如规划、执行、归档或审查，这里就是建立第一版成员池的地方。

### 6.2 岗位分配

![初始化向导 Agents 岗位分配界面](../img/screenshots/shared/guide7-arrange-agents-for-job/original.png)

成员建立之后，需要把它们分配到具体岗位。截图中展示了生成、执行、归档等任务位的绑定方式。

岗位分配的作用，是把“有哪些成员”进一步转化为“谁负责哪一类工作”。这会直接影响之后会话、提案流程与自动化执行时的默认分工。

### 6.3 fallback / backup

![初始化向导 Agents 备用成员界面](../img/screenshots/shared/guide8-select-agents-as-backup/original.png)

最后是 fallback / backup 设置。它用于指定默认岗位不可用时，系统该切换到哪些备用成员。

这一层配置很关键，因为它能避免某个默认成员失效后整条流程被卡住。尤其当你启用了多个 AgentCLI 或多个模型时，提前准备备选方案会更稳。

## 步骤 7：Projects

![初始化向导项目创建界面](../img/screenshots/shared/guide9-create-a-new-project/original.png)

Projects 步骤把前面的语言、能力与角色设置落到一个真实项目上。你可以在这里：

- 新建项目入口
- 填写项目名称、路径和描述
- 让已有默认 Agents 与 CLI 配置直接服务于这个工作目录

这一页的重点，是把“向导中的准备动作”转化为“可以立即开始工作的项目上下文”。完成后，后续快速开始文档就能直接围绕真实仓库展开。

## 步骤 8：Community guide

![初始化向导社区入口界面](../img/screenshots/shared/guide10-star-or-join-group/original.png)

向导最后一步不会再修改本地配置，而是给出后续外部入口，例如 GitHub、Discord、Bilibili、QQ 群等社区与内容渠道。

如果你刚完成第一次安装，这一页适合做两件事：

- 收藏官方入口，方便后续查看发布与更新说明
- 加入社区或教程渠道，在需要时快速找到交流与反馈路径
