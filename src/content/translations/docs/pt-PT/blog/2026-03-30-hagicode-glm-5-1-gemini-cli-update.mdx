## Suporte total ao GLM-5.1 e integração Gemini CLI: o caminho da evolução de vários modelos do HagiCode

> Este artigo apresenta duas atualizações recentes importantes para a plataforma HagiCode: suporte total para o modelo Zhipu AI GLM-5.1 e a integração bem-sucedida do Gemini CLI como o décimo Agent CLI. Juntas, essas atualizações fortalecem ainda mais os recursos multimodelos e o ecossistema multi-CLI da plataforma.

:::nota[Aviso de Contexto de Tempo]
Este artigo registra o progresso da expansão multi-CLI conforme estava no momento do lançamento. Para o escopo atual sempre suportado, consulte [Instalação da CLI do Agente AI](/pt-PT/related-software-installation/ai-agent-cli/) e o [Guia de instalação](/pt-PT/installation); o `IFlowCli` mencionado aqui representa apenas uma configuração histórica da época e não faz parte do escopo atual de suporte ativo.
:::

## Plano de fundo

O tempo realmente passa rápido. O desenvolvimento de grandes modelos de linguagem tem aumentado como o bambu na primavera. Não muito tempo atrás, ainda torcíamos por “uma IA que possa escrever código”. Agora já estamos em uma era de colaboração multimodelo e integração multiferramenta. Isso é emocionante? Talvez. Afinal, o que os desenvolvedores precisam nunca foi apenas da ferramenta em si, mas da facilidade de se adaptar a diferentes cenários e alternar com flexibilidade quando necessário.

Como uma plataforma de codificação assistida por IA, o HagiCode acolheu recentemente dois desenvolvimentos importantes: primeiro, a integração total do modelo GLM-5.1 da Zhipu AI; segundo, a adição oficial do Gemini CLI como o décimo Agent CLI com suporte. Essas duas atualizações podem não parecer surpreendentes, mas são, sem dúvida, boas notícias para o contínuo amadurecimento da plataforma.

GLM-5.1 é o modelo carro-chefe mais recente da Zhipu AI. Comparado com o GLM-5.0, ele oferece raciocínio mais forte, compreensão mais profunda do código e chamada de ferramentas mais suave. Mais importante ainda, é o primeiro modelo GLM a suportar entrada de imagem. O que isso significa? Isso significa que os usuários podem permitir que a IA olhe diretamente para uma captura de tela, em vez de se esforçar para descrever o problema em palavras. Depois de usar essa conveniência, você compreenderá imediatamente seu valor.

Ao mesmo tempo, através do `HagiCode.Libs.Providers` arquitetura, o HagiCode integrou com sucesso o Gemini CLI na plataforma. Este é agora o décimo Agent CLI. Para ser honesto, chegar a este ponto traz uma modesta sensação de realização.

Também vale a pena mencionar que o recurso de upload de imagens do HagiCode permite que os usuários se comuniquem com a IA diretamente por meio de capturas de tela. Mesmo executando o GLM 4.7, a plataforma ainda funciona bem e já ajudou a concluir muitas tarefas importantes de construção. Quanto ao GLM-5.1, naturalmente, vai um passo além.

## Sobre HagiCode

A abordagem compartilhada neste artigo vem de nossa experiência prática no [HagiCode](https://hagicode.com) projeto. HagiCode é uma plataforma de codificação assistida por IA de código aberto projetada para fornecer aos desenvolvedores um assistente de programação de IA flexível e poderoso por meio de uma arquitetura multimodelo e multi-CLI. Repositório do projeto: [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site)

## Projeto de arquitetura multi-CLI

Um dos principais pontos fortes do HagiCode é o suporte a múltiplas ferramentas CLI de programação de IA por meio de uma camada de abstração unificada. A vantagem desse design é bastante simples: novas ferramentas podem entrar, ferramentas antigas podem permanecer e a base de código não se transforma em caos. Para ser justo, é assim que todos gostariam que a vida funcionasse.

### enumeração AIProviderType

A plataforma define os tipos de provedores CLI suportados por meio do `AIProviderType` enum:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI (new)
}
```

Como você pode ver, Gemini CLI se junta a esta família como o décimo membro. Cada CLI tem suas próprias características e cenários de uso distintos, para que os usuários possam escolher com flexibilidade com base em suas necessidades. Afinal, muitos caminhos levam a Roma; alguns são simplesmente mais fáceis do que outros.

### Arquitetura do provedor

`HagiCode.Libs.Providers` fornece uma interface de provedor unificada que torna cada integração CLI padronizada e concisa. Tomando o Gemini CLI como exemplo:

```csharp
public class GeminiProvider : ICliProvider<GeminiOptions>
{
    private static readonly string[] DefaultExecutableCandidates = ["gemini", "gemini-cli"];
    private const string ManagedBootstrapArgument = "--acp";

    public string Name => "gemini";
    public bool IsAvailable => _executableResolver.ResolveFirstAvailablePath(DefaultExecutableCandidates) is not null;
}
```

Os benefícios deste design são:
- A integração de uma nova CLI requer apenas a implementação de uma classe de provedor
- Gerenciamento unificado do ciclo de vida e pool de sessões
- Resolução automatizada de alias e descoberta de executáveis

Simplificando, esse design transforma coisas complicadas em coisas mais simples e torna a vida um pouco mais fácil.

### Registro de Provedor

O Registro do Provedor lida automaticamente com mapeamento e registro de alias:

```csharp
if (provider is GeminiProvider)
{
    registry.Register(provider.Name, provider, ["gemini-cli"]);
    continue;
}
```

Isso significa que os usuários podem invocar o Gemini CLI com qualquer um `gemini` ou `gemini-cli`, e o sistema o reconhecerá automaticamente. É como um amigo com nome formal e apelido - de qualquer forma, as pessoas sabem a quem você se refere.

## Suporte ao modelo GLM-5.1

GLM-5.1 é o modelo carro-chefe mais recente da Zhipu AI e o HagiCode completou o suporte completo para ele.

### Catálogo de Profissões Secundárias

HagiCode gerencia todos os modelos suportados através do Catálogo de Profissões Secundárias. Aqui está a configuração da série GLM:

| ID do modelo | Nome | Suporta imagem | Famílias CLI compatíveis |
|----------|------|----------------|-------------------------|
| `glm-4.7` | GLM 4.7 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM 5 Turbo | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (Legado) | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | **verdade** | claude, codebuddy, hermes, qoder, kiro |

As principais características do GLM-5.1 podem ser resumidas da seguinte forma:
- Um identificador de versão independente sem bagagem legada
- O primeiro modelo GLM com suporte para entrada de imagem
- Raciocínio mais forte e compreensão do código
- Ampla compatibilidade multi-CLI

### GLM-5.1 versus GLM-5.0

No nível do código, a principal diferença entre o GLM-5.1 e o GLM-5.0 é mostrada aqui:

```csharp
// GLM-5.0 (Legacy) - contains special retention logic
private const string Glm50CodebuddySecondaryProfessionId = "secondary-glm-5-codebuddy";
private const string Glm50CodebuddyModelValue = "glm-5.0";

// GLM-5.1 - standalone new model identifier
private const string Glm51SecondaryProfessionId = "secondary-glm-5-1";
private const string Glm51ModelValue = "glm-5.1";
```

GLM-5.0 carrega o rótulo "Legado" porque é um identificador de versão antiga retido para compatibilidade com versões anteriores. O GLM-5.1, por outro lado, é uma versão autônoma totalmente nova, sem carga histórica. Algumas coisas ficam no passado; outros viajam com mais leveza e se movem mais rápido.

### Configurar GLM-5.1

Aqui está um exemplo de configuração para usar GLM-5.1 no HagiCode:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

## Recurso de upload de imagem

O suporte de imagem do HagiCode é implementado através do `SupportsImage` propriedade em `SecondaryProfession`:

```csharp
public class HeroSecondaryProfessionSettingDto
{
    public bool SupportsImage { get; set; }
}
```

No Catálogo de Profissões Secundárias, a configuração do GLM-5.1 é assim:

```json
{
  "id": "secondary-glm-5-1",
  "supportsImage": true
}
```

Isso significa que os usuários podem fazer upload de capturas de tela diretamente para análise de IA, como:
- Capturas de tela de mensagens de erro
- Problemas em uma tela da IU
- Gráficos de visualização de dados
- Resultados de execução de código

Não há mais necessidade de descrever tudo manualmente - basta fazer upload da captura de tela. A conveniência desse recurso é óbvia depois de usá-lo. Às vezes, um olhar diz mais do que uma longa explicação.

## Integração Gemini CLI

Como o décimo Agente CLI, o Gemini CLI é integrado ao HagiCode por meio da arquitetura padrão do Provedor.

### Opções de configuração

Gemini CLI oferece suporte a um rico conjunto de opções de configuração:

```csharp
public class GeminiOptions
{
    public string? ExecutablePath { get; set; }
    public string? WorkingDirectory { get; set; }
    public string? SessionId { get; set; }
    public string? Model { get; set; }
    public string? AuthenticationMethod { get; set; }
    public string? AuthenticationToken { get; set; }
    public Dictionary<string, string?> AuthenticationInfo { get; set; }
    public Dictionary<string, string?> EnvironmentVariables { get; set; }
    public string[] ExtraArguments { get; set; }
    public TimeSpan? StartupTimeout { get; set; }
    public CliPoolSettings? PoolSettings { get; set; }
}
```

Essas opções cobrem tudo, desde configuração básica até recursos avançados, dando aos usuários a flexibilidade de configurar a CLI de acordo com suas próprias necessidades. O fluxo de trabalho de cada pessoa é diferente, por isso um pouco de flexibilidade é sempre bem-vindo.

### Protocolo de comunicação ACP

Gemini CLI suporta o ACP (Agent Communication Protocol), que é o padrão de comunicação CLI unificado do HagiCode. Através do ACP, diferentes CLIs podem interagir com a plataforma de forma consistente, simplificando bastante o trabalho de integração. Resumindo, padroniza as partes complicadas para que todos possam trabalhar com mais facilidade.

Para usar os modelos Zhipu AI, você precisa configurar as variáveis de ambiente correspondentes.

```bash
export ANTHROPIC_AUTH_TOKEN="***"
export ANTHROPIC_BASE_URL="https://open.bigmodel.cn/api/anthropic"
```

### Alibaba Cloud DashScope

```bash
export ANTHROPIC_AUTH_TOKEN="your-a...-key"

```

Uma vez configurado, o HagiCode pode chamar o modelo GLM-5.1 normalmente. Não é especialmente difícil nem especialmente fácil - você só precisa seguir a configuração conforme planejado.

## Prática de construção própria do HagiCode

Falando em prática do mundo real, o melhor exemplo é o fluxo de trabalho de construção da própria plataforma HagiCode. O processo de desenvolvimento do HagiCode já fez pleno uso dos recursos de IA.

### Funciona bem mesmo com GLM 4.7

O design da plataforma HagiCode é bem otimizado, por isso ainda pode fornecer uma boa experiência de desenvolvimento mesmo com o GLM 4.7. A plataforma já ajudou a concluir vários projetos de construção importantes, incluindo:
- Integração de vários provedores CLI
- Implementação do recurso de upload de imagens
- Geração de documentação e publicação de conteúdo

Na verdade, isso é uma coisa boa. Nem todo mundo precisa das novidades o tempo todo. O que melhor combina com você geralmente é o que mais importa.

### GLM-5.1 oferece mais com menos esforço

Após a atualização para o GLM-5.1, esses recursos se tornam ainda mais fortes:
- Maior compreensão do código, reduzindo a comunicação de ida e volta
- Análise de dependência mais precisa, apontando imediatamente na direção certa
- Diagnóstico de erros mais eficiente, localizando problemas mais rapidamente
- Suporte para entrada de imagens, acelerando descrições de problemas

É como mudar de bicicleta para carro. Você ainda pode chegar ao mesmo destino, mas a velocidade e o conforto não são os mesmos.

## Melhores práticas para integração multi-CLI

`HagiCode.Libs.Providers` fornece um mecanismo unificado para registro e uso:

```csharp
services.AddHagiCodeLibs();

var gemini = serviceProvider.GetRequiredService<ICliProvider<GeminiOptions>>();
var codebuddy = serviceProvider.GetRequiredService<ICliProvider<CodebuddyOptions>>();
var hermes = serviceProvider.GetRequiredService<ICliProvider<HermesOptions>>();
```

Esse design de injeção de dependência mantém o uso em diferentes CLIs muito conciso e também torna os testes de unidade e a simulação mais convenientes. Código limpo é uma forma de ser responsável consigo mesmo.

### Notas

Há algumas coisas a serem lembradas no uso real:

1. **Configuração da chave de API**: certifique-se `ANTHROPIC_AUTH_TOKEN` está definido corretamente ou o modelo não pode ser chamado
2. **Disponibilidade do modelo**: o GLM-5.1 precisa ser habilitado pelo fornecedor do modelo correspondente
3. **Recurso de imagem**: Somente modelos com `supportsImage: true` pode usar upload de imagem
4. **Instalação CLI**: Antes de usar o Gemini CLI, certifique-se `gemini` ou `gemini-cli` está no sistema PATH

Podem ser pequenos detalhes, mas pequenos detalhes mal tratados podem se transformar em grandes problemas, por isso vale a pena prestar atenção a eles.

## Conclusão

Com suporte total para GLM-5.1 e a integração bem-sucedida do Gemini CLI, o HagiCode fortalece ainda mais suas capacidades como uma plataforma de programação de IA multimodelo e multi-CLI. Essas atualizações não apenas oferecem aos usuários mais opções, mas também demonstram a arquitetura e escalabilidade voltadas para o futuro do HagiCode.

O suporte de imagem do GLM-5.1, combinado com o recurso de upload de captura de tela do HagiCode, torna possível permitir que a IA “entenda a partir da imagem” e reduz bastante o custo de descrição de problemas. E com suporte para dez CLIs, os usuários podem escolher com flexibilidade o assistente de programação de IA que melhor se adapta às suas preferências e cenários. Mais opções é quase sempre uma coisa boa.

Mais importante ainda, a própria prática de construção do HagiCode prova que a plataforma já pode funcionar bem e concluir tarefas complexas mesmo com o GLM 4.7, enquanto a atualização para o GLM-5.1 pode melhorar ainda mais a eficiência do desenvolvimento. A vida também costuma ser assim: você nem sempre precisa do melhor, apenas do que mais lhe convém. Claro, se o que combina com você pode se tornar ainda melhor, tanto melhor.

Se você estiver interessado em uma plataforma de programação de IA multimodelo e multi-CLI, experimente o HagiCode - código aberto, gratuito e ainda em evolução. Tentar não custa nada e pode acabar sendo exatamente o que você precisa.
