## Projetando 12 experiências de inicialização excepcionais para HagiCode: do minimalismo ao cyberpunk

> O breve intervalo entre o download de um aplicativo React 19 e a conclusão da Hidratação é uma oportunidade de ouro para os usuários sentirem a personalidade da sua marca. Neste artigo, compartilhamos um sistema completo de estilo de inicialização que construímos para o projeto HagiCode usando HTML/CSS/JS.

## Plano de fundo

Como um aplicativo moderno desenvolvido com ASP.NET Core 10 e React 19 (Vite), o HagiCode usa uma arquitetura de implantação separada frontend-backend. A saída da compilação do frontend é empacotada no backend `wwwroot/` diretório e hospedado pelo ASP.NET Core.

No entanto, essa arquitetura introduz um problema clássico de UX: quando os usuários visitam a página, o navegador primeiro carrega o HTML, depois baixa o grande pacote JS e, finalmente, permite que o React execute a hidratação. Durante esse “período de vácuo” que dura de algumas centenas de milissegundos a vários segundos, os usuários veem uma tela em branco ou uma página estática sem vida.

Para preencher essa lacuna e injetar a personalidade da marca HagiCode, precisávamos projetar um sistema estilo startup implementado inteiramente com código embutido dentro `index.html`.

## Sobre HagiCode

A abordagem de design da tela inicial compartilhada neste artigo vem de nossa experiência prática no [HagiCode](https://github.com/HagiCode-org/site) projeto. Como assistente de codificação de IA, o HagiCode se preocupa não apenas com a eficiência da geração de código, mas também com a experiência visual do desenvolvedor. Este sistema de inicialização é um dos resultados de nossa busca pelo melhor desempenho de front-end.

## Principais Desafios e Design de Arquitetura

Antes de começarmos a projetar, primeiro tivemos que esclarecer as restrições técnicas. Como tudo teve que ser implementado em linha no `index.html`, não foi possível carregar nenhum arquivo CSS ou JS externo além do próprio pacote do React.

### Análise de Restrições Técnicas

1.  **Princípio de dependência zero**: Todos os estilos devem viver dentro de um `<style>` tag, e toda lógica deve residir dentro de uma `<script>` etiqueta.
2.  **CSS defensivo**: para evitar que estilos globais poluam a tela inicial após a montagem do aplicativo React, decidimos agrupar todos os estilos de inicialização com um prefixo de ID de alta prioridade, como `#boot-screen`.
3.  **Desempenho em primeiro lugar**: as animações devem usar CSS `transform` e `opacity` sempre que possível para evitar refluxo e garantir que o thread principal permaneça desbloqueado.
4.  **Consistência visual**: cores e fontes devem permanecer alinhadas com a configuração Tailwind do HagiCode.

### Padrão de Arquitetura: Shell e Injetor

Adotamos um **padrão variante**. A lógica central é encapsulada dentro de uma expressão de função invocada imediatamente (IIFE), enquanto a lógica de renderização específica é injetada por meio da configuração. Isso nos permite alternar entre estilos diferentes por meio de configuração simples, em vez de reescrever a lógica de manipulação do DOM repetidamente.

Aqui está o código da arquitetura principal:

```html
<!-- Inline in index.html -->
<div id="boot-root"></div>

<script>
(function() {
  const BootSequence = {
    config: {
      theme: 'terminal', // Configurable as 'minimal', 'skeleton', 'code-rain', etc.
      color: '#3b82f6'   // Brand color
    },
    
    // Core lifecycle
    init() {
      this.render();
      this.listenForMount();
    },

    // Render the currently selected style
    render() {
      const root = document.getElementById('boot-root');
      if (this.variants[this.config.theme]) {
        root.innerHTML = this.variants[this.config.theme].render();
      }
    },

    // Listen for successful React mount and exit gracefully
    listenForMount() {
      window.addEventListener('hagicode:ready', () => {
        const screen = document.getElementById('boot-root');
        // Fade out first, then remove the DOM to avoid flicker
        screen.style.opacity = '0';
        screen.style.transition = 'opacity 0.3s ease';
        setTimeout(() => screen.remove(), 300);
      });
    },

    // The implementation logic for all 12 styles lives here
    variants: {
      // ...see details below
    }
  };

  BootSequence.init();
})();
</script>
```

## Uma lista de verificação de 12 designs de estilo de inicialização

Agrupamos esses 12 estilos em seis categorias principais para satisfazer diferentes cenários e preferências estéticas.

### A. Minimalismo

> "Menos é mais." Para cenários que buscam velocidade de carregamento máxima, oferecemos as opções mais leves possíveis.

#### 1. Ponto Minimalista
Um ponto simples fica no centro da tela, combinado com uma animação de respiração.
*   **Implementação**: CSS `@keyframes` controla a escala e a opacidade.
*   **Ideal para**: qualquer caso em que a página deva permanecer absolutamente limpa.

#### 2. Revelação da marca
Usando SVG `stroke-dasharray` animação, ela simula uma revelação desenhada à mão das linhas do logotipo HagiCode, seguida por um fade-in de texto.
*   **Técnica**: Animação de caminho SVG com um toque altamente refinado.

### B. Mimetismo de tela de esqueleto

> "A arte de enganar os olhos." Ao simular um layout de UI real, os usuários sentem que a página já está meio carregada.

#### 3. Esqueleto de bate-papo da barra lateral
Esta pode ser a opção mais prática. Construímos manualmente um layout em HTML que espelha o React `Sidebar` e `ChatInput` componentes exatamente e, em seguida, sobreponha-os com uma animação cintilante cinza.
*   **Valor**: quando a hidratação do React é concluída, o esqueleto se torna instantaneamente o componente real e os usuários mal conseguem perceber a mudança.

#### 4. Esqueleto da pilha de cartas
Ele simula o movimento empilhado dos cartões de proposta durante o carregamento, usando transformações 3D para fazer os cartões flutuarem sutilmente.

### C. Abstração e Arte

> Mostre o DNA geek do HagiCode.

#### 5. Morfismo Geométrico
Uma forma geométrica (um quadrado) é renderizada no centro da tela e, ao longo do tempo, transforma-se suavemente em um círculo, um triângulo e, finalmente, no logotipo.
*   **Tecnologia**: transições suaves com CSS `border-radius`.

#### 6. Código Chuva
Uma homenagem a *Matrix*. Usando a fonte JetBrains Mono, fluxos fracos de caracteres ficam em segundo plano.
*   **Observação**: Para desempenho, os fluxos de personagens devem permanecer em uma área menor ou usar uma taxa de atualização mais baixa.

#### 7. Pulso de néon
Um anel brilhante no estilo cyberpunk que usa vários `box-shadow` camadas para criar um poderoso brilho neon.

### D. Marca e Temas

> Faça o sistema parecer vivo.

#### 8. Tema sazonal
Este é um carregador dinâmico. Ele verifica a data atual de feriados como Ano Novo Lunar ou Natal e carrega a animação SVG correspondente.
*   **Exemplo**: durante o Ano Novo Lunar, lanternas vermelhas balançam suavemente na parte inferior da tela.

#### 9. Fluxo Gradiente
O fundo usa um gradiente fluido baseado nas cores da marca HagiCode. Combinado com animação `background-size` e `background-position`, cria uma sensação de movimento semelhante à da aurora.

### E. Sensação técnica

> Uma saudação aos desenvolvedores.

#### 10. Inicialização do terminal
Ele simula a saída do console. Linhas de código rolam rapidamente:
```text
> Initializing HagiCode Core...
> Loading models...
> Connecting to neural network...
```
Isso parece instantaneamente familiar para todos os desenvolvedores.

#### 11. Barra de progresso
Uma fina barra de progresso aparece na parte superior da tela, com uma porcentagem mostrada à direita. Embora não possamos acessar o progresso real do download, podemos usar um cronômetro para simular um processo de carregamento “crível”: rápido nos primeiros 80%, depois gradualmente mais lento nos últimos 20%.

### F. Criatividade

#### 12. Montagem de pixels
Esta é uma ideia muito interessante. Pequenos quadrados estão espalhados pela tela, depois convergem para o centro e gradualmente se reúnem no ícone do logotipo HagiCode. Simboliza o processo de construção de código.

## Melhores práticas e armadilhas

No trabalho real de desenvolvimento do HagiCode, resumimos vários detalhes de implementação extremamente importantes.

### 1. CSS defensivo é obrigatório
Nunca fique com preguiça e pule o prefixo. Certa vez, esquecemos de definir o escopo dos estilos da tela inicial com um ID e global `div` estilos após a montagem do React afetaram inesperadamente a tela inicial, quebrando o layout.
**Lição aprendida**: coloque todos os seletores de CSS em `#boot-screen`e usar `!important` para aumentar a prioridade quando necessário, mas apenas dentro do CSS da tela inicial.

### 2. Transições Graciosas
Depois que o React for montado com sucesso, não faça diretamente `remove()` o DOM da tela inicial.
**Abordagem correta**:
1.  Gatilhos de reação `window.dispatchEvent(new Event('hagicode:ready'))`.
2.  A tela inicial escuta o evento e os primeiros conjuntos `opacity: 0`.
3.  Aguarde 300 ms, que corresponde à duração da transição CSS, e chame `.remove()` somente depois que a tela estiver totalmente invisível.

### 3. Sincronização de variáveis de tema
Os valores de cores da tela inicial são codificados em `index.html`. Se alterarmos a cor primária do Tailwind, devemos atualizar a tela inicial também.
**Otimização**: Escreva um plugin simples no script de construção do Vite para ler `tailwind.config.js` e injetar variáveis de cor no `index.html` variáveis de modelo, criando uma única fonte de verdade.

### 4. Pré-carregamento de fonte
As telas iniciais geralmente precisam usar uma fonte de marca, mas se a fonte carregar lentamente, FOUT (Flash of Unstyled Text) pode aparecer.
**Solução**: Adicionar `<link rel="preload" href="/fonts/JetBrainsMono.woff2" as="font" type="font/woff2" crossorigin>` dentro `<head>`. Esta é uma forma de baixo custo e alto retorno de melhorar a experiência.

### 5. Monitoramento de desempenho
Nós injetamos `performance.mark('boot-start')` na parte inferior de `index.html`e marcado `boot-end` quando o React foi montado com sucesso.
**Por que é importante**: Ao coletar esses dados por meio do Application Insights, podemos medir diretamente o quanto a tela inicial reduz o tempo de espera percebido. Os dados mostram que uma excelente tela de esqueleto pode melhorar a tolerância dos usuários a uma “rede lenta” em mais de 50%.

## Resumo

Uma boa tela inicial é mais do que “decoração enquanto espera”. É o sinal de aperto de mão logo na primeira interação entre o produto e o usuário. No projeto HagiCode, esse sistema de inicialização baseado no **padrão Variants** nos permite alternar estilos de maneira flexível entre feriados e lançamentos, melhorando muito a sensação de diversão e profissionalismo do produto.

A solução compartilhada neste artigo é construída inteiramente em padrões nativos da web, sem introduzir quaisquer dependências pesadas, o que reflete a busca do HagiCode em ser “leve, mas poderoso”.
