## Guia para criar um efeito de animação de varredura de luz de borda

> Como você constrói aquele elemento importante que os usuários percebem rapidamente usando CSS puro? Na verdade não é tão difícil. O truque é apenas seguir um caminho ligeiramente indireto. Neste artigo, mostrarei como construir uma animação de varredura de luz de borda do zero e também compartilharei algumas das armadilhas que encontramos ao construir o HagiCode.

## Plano de fundo

Se você trabalha no frontend, provavelmente já teve essa experiência antes: um gerente de produto se aproxima com aquela expressão “isso é definitivamente simples” e diz: “Podemos adicionar algum tipo de efeito especial a esta tarefa em execução para que os usuários possam identificá-la imediatamente?”

Você diz que sim, podemos apenas mudar a cor da borda. Então eles balançam a cabeça com aquele olhar que diz: “Você não entendeu”. Eles respondem: "Isso não é suficientemente óbvio. Quero o tipo de efeito em que a luz contorna a fronteira, como num filme de ficção científica."

Nesse ponto, você pode começar a se perguntar como construí-lo. Tela? SVG? Ou o CSS pode lidar com isso sozinho? Afinal, ninguém quer admitir que não sabe como.

Em aplicativos da web modernos, animações de varredura de luz de borda são muito comuns. Eles são usados principalmente em alguns cenários como estes:

- **Indicadores de status**: marcação de tarefas em andamento ou itens ativos
- **Foco visual**: destacando áreas de conteúdo importantes
- **Aprimoramento da marca**: criação de um estilo visual elegante, moderno e tecnológico
- **Temas sazonais**: criando uma atmosfera comemorativa para ocasiões especiais

Encontramos exatamente esse requisito durante a construção [HagiCode](https://hagicode.com). Os usuários precisavam ver rapidamente quais sessões estavam em andamento e quais propostas estavam sendo processadas no momento. Tentamos várias abordagens diferentes. Alguns caminhos eram mais suaves, outros um pouco mais sinuosos. No final, estabelecemos uma estratégia de implementação bastante madura.

## Sobre HagiCode

A abordagem compartilhada neste artigo vem de nossa experiência prática no [HagiCode](https://hagicode.com) projeto. HagiCode é um projeto de assistente de codificação baseado em IA, e a interface faz uso extensivo de animações de luz de borda para indicar diferentes estados de tempo de execução. Os exemplos incluem o estado de execução dos itens na lista de sessões, transições de status no diagrama de fluxo da proposta e indicadores de intensidade para produtividade.

Esses efeitos não são especialmente complicados em princípio, mas definitivamente encontramos muitas armadilhas ao implementá-los. Se você quiser ver a coisa real, você pode visitar nosso [Repositório GitHub](https://github.com/HagiCode-org/site) ou vá para o [site oficial](https://hagicode.com). No final, o que mais importa é o que realmente funciona.

## Ideias Principais de Implementação

Depois de analisar a base de código HagiCode, resumimos vários padrões principais de implementação abaixo. Cada um se enquadra em um cenário diferente, ou seja, cada um existe por um motivo.

### 1. Brilho giratório com gradiente cônico (mais comum)

Esta é a maneira clássica de implementar um efeito de varredura de luz na borda. A ideia central é usar CSS `conic-gradient` para criar um gradiente cônico e gire-o continuamente. Como um poste de luz acendendo à noite, ele continua circulando.

**Elementos principais:**
- Use o `::before` pseudoelemento para criar a camada de brilho
- Usar `conic-gradient` para definir a distribuição de cores gradientes
- Use o `::after` pseudoelemento para mascarar a área central (opcional)
- Usar `@keyframes` para implementar a animação de rotação

### 2. Linha de brilho lateral

Isto funciona bem para indicadores de status em itens de lista. Você cria uma linha fina e brilhante em um lado do elemento em vez de animar toda a borda. Às vezes um pouco de luz é suficiente. Você não precisa iluminar o mundo inteiro.

**Elementos principais:**
- Um elemento de linha fino e absolutamente posicionado
- Usar `box-shadow` para criar o efeito de brilho
- Usar `scale` e `opacity` para uma animação de respiração

### 3. Fundo brilhante com sombra de caixa

Se você não precisa do efeito de varredura completa e deseja apenas um brilho suave no fundo, aplique várias camadas `box-shadow` valores são suficientes. Às vezes, a opção mais simples é a melhor.

### 4. Suporte de acessibilidade

Esta parte é fácil de ignorar, mas é extremamente importante. Cada animação deve levar em conta o `prefers-reduced-motion` consulta de mídia e fornece uma alternativa estática para usuários que não desejam animação. Nem todo mundo gosta de movimento constante e respeitar essa preferência é importante.

## Opções de implementação

### Opção 1: borda rotativa com gradiente cônico (recomendado)

Esta é a implementação mais completa do efeito de luz de borda arrebatadora e também é a opção que usamos com mais frequência no HagiCode. Afinal, se algo funciona bem, por que substituí-lo?

```css
/* Parent container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotating glow layer */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Mask layer (optional, for creating a hollow border effect) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

O princípio por trás desta opção é bastante simples: crie um pseudoelemento maior que o contêiner pai, desenhe um gradiente cônico nele e gire-o continuamente. O contêiner pai usa `overflow: hidden`, então apenas a luz que passa pela borda permanece visível. É um pouco como observar a luz de uma rua através de uma janela. Você só vê a pequena fatia que passa.

### Opção 2: Borda de luz rotativa simplificada

Se você não precisa do efeito completo, o HagiCode também inclui uma versão mais leve de classe utilitária. Às vezes, a abordagem mais simples é realmente melhor.

```css
/* Rotating light border utility class */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Accessibility support */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Observe o `will-change: transform` aqui. Ele informa ao navegador: “Este elemento continuará mudando”, para que o navegador possa preparar algumas otimizações com antecedência e manter a animação mais suave. Preparar-se com antecedência geralmente é melhor do que mexer no último minuto.

### Opção 3: linha de brilho lateral

Isso é especialmente adequado para indicadores de status de itens de lista e é exatamente o que a lista de sessões do HagiCode usa. Uma linha tênue ainda pode se destacar entre muitos itens. Isso parece uma lição de vida à sua maneira.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Isso usa `isolation: isolate` para criar um novo contexto de empilhamento, então depende de `z-index` para controlar a ordem de exibição de cada camada. `pointer-events: none` também é essencial. Caso contrário, o pseudoelemento bloquearia os cliques do usuário. Algumas coisas podem parecer bonitas, mas ainda assim não devem atrapalhar.

### Opção 4: envolva-o em um componente React

Se o seu projeto usa React, você pode agrupar essa lógica em um componente, especialmente o tratamento de acessibilidade. Escreva uma vez, use muitas vezes. Esse é o ponto principal.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

O módulo CSS correspondente:

```css
/* GlowBorder.module.css */

/* Animated version */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Static version (accessibility) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

O `useReducedMotion` gancho de `framer-motion` detecta automaticamente a preferência de sistema do usuário. Se o usuário habilitou o movimento reduzido, ele retorna `true`e o componente mostra a versão estática. Respeitar a preferência do usuário é mais importante do que forçar um efeito chamativo.

## Lições práticas aprendidas

Estas são algumas das lições que aprendemos ao construir o HagiCode. Você também poderia chamá-las de cicatrizes de batalha. Esperamos que eles ajudem você a evitar alguns desvios.

### 1. Sistema de variáveis temáticas

Variáveis CSS tornam o suporte multitema especialmente conveniente. Ninguém quer editar uma pilha de código toda vez que o tema muda.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Usage */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

Dessa forma, mudar de tema requer apenas mudar de turma no `html` elemento e cada cor de animação é atualizada automaticamente. Uma base de código, dois estilos. É exatamente isso que queremos.

### 2. Otimização de desempenho

**Usar `will-change` para sugerir que o navegador otimize:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Informe o navegador com antecedência e isso o ajudará a otimizar. Muitas coisas na vida funcionam melhor com um pouco de preparação.

**Evite usar sombras de caixa complexas em elementos grandes:**

```css
/* Not ideal - using a blurred shadow on a large element */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Better - use a pseudo-element to limit the glowing area */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Testamos isso no HagiCode. Adicionar uma sombra desfocada diretamente a um cartão grande reduziu as taxas de quadros de rolagem para menos de 30 fps. Mudar para um pseudo-elemento trouxe as coisas de volta a 60fps constantes. Os usuários podem sentir absolutamente essa diferença.

### 3. Acessibilidade

Você realmente não deveria pular isso. Alguns usuários acham a animação estonteante ou perturbadora, e respeitar suas preferências faz parte da construção de um bom produto. Coisas bonitas não precisam ser impostas a todos.

**Consulta de mídia CSS:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Provide a static fallback */
    opacity: 1;
  }
}
```

**Detecte a preferência do usuário no React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Controle de nível de intensidade

O indicador de rendimento do token no HagiCode mostra diferentes cores de brilho com base no rendimento em tempo real, e isso é implementado dinamicamente. Estados diferentes devem ser expressos de forma diferente.

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Coisas a observar

Ainda há alguns detalhes aos quais vale a pena prestar atenção, porque quando você descobrir esses problemas da maneira mais difícil, já será tarde demais.

| Coisas a observar | Explicação |
|---------|------|
| **gerenciamento de índice z** | A camada de brilho deve usar uma camada apropriada `z-index` então não interfere na interação do conteúdo |
| **eventos de ponteiro** | O pseudoelemento de brilho deve ser definido `pointer-events: none` |
| **Estouro de limite** | O contêiner pai precisa `overflow: hidden`, ou você precisa ajustar o tamanho do pseudoelemento |
| **Impacto no desempenho** | Animações complexas podem prejudicar o desempenho em dispositivos móveis, então teste com cuidado |
| **Modo escuro** | Certifique-se de que a cor do brilho permaneça claramente visível em fundos escuros |
| **Mudança de tema** | Use variáveis CSS para que as cores da animação sejam atualizadas corretamente quando o tema mudar |

### 6. Dicas de depuração

Os pseudoelementos podem ser um pouco difíceis de localizar nas ferramentas do desenvolvedor, então você pode adicionar temporariamente uma borda para verificar a posição.

```css
/* Temporarily show pseudo-element boundaries for debugging */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Depois de terminar de posicioná-lo, lembre-se de comentar ou remover essa linha. Caso contrário, a produção pode ficar complicada rapidamente. É melhor deixar algumas coisas em desenvolvimento.

## Resumo

As animações de varredura de luz de borda não são especialmente difíceis nem verdadeiramente triviais. No fundo, a fórmula é `conic-gradient` além de rotação, mas se você deseja bom desempenho, capacidade de manutenção e acessibilidade, ainda há muitos detalhes de implementação que devem ser tratados com cuidado.

O HagiCode encontrou muitas dessas armadilhas e gradualmente destilou um conjunto de práticas recomendadas. Os projetos são assim: você experimenta, comete erros e melhora um passo de cada vez. Se você estiver construindo algo semelhante, espero que este artigo o ajude a evitar alguns desvios desnecessários.

Algumas coisas só ficam claras quando você mesmo as constrói.
