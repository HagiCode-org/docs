## Por que a HagiCode escolheu Hermes como seu agente central integrado

> Ao construir uma plataforma de codificação assistida por IA, a escolha do núcleo do Agente certo determina diretamente o limite superior das capacidades do sistema. Algumas coisas simplesmente não podem ser forçadas; escolha a estrutura errada e nenhum esforço fará com que pareça certo. Este artigo compartilha o pensamento por trás da seleção técnica do HagiCode e nossa experiência prática na integração do Agente Hermes.

## Plano de fundo

Ao construir um produto de codificação assistido por IA, uma das partes mais difíceis é escolher a estrutura do agente subjacente. Na verdade, existem algumas opções no mercado, mas algumas são muito limitadas em funcionalidade, algumas são excessivamente complexas de implantar e outras simplesmente não são escalonáveis ​​o suficiente. O que precisávamos era de uma solução que pudesse rodar em um VPS de US$ 5 e ao mesmo tempo ser capaz de se conectar a um cluster de GPU. Esse requisito pode não parecer extremo, mas é suficiente para assustar muitas equipes.

Na prática, muitos dos chamados “agentes multifuncionais” ou funcionam apenas na nuvem ou exigem custos de implantação local absurdamente altos. Depois de passar duas semanas pesquisando diferentes abordagens, tomamos uma decisão ousada: reconstruir todo o núcleo do Agente em torno do Hermes como o mecanismo subjacente para nosso Agente integrado.

Tudo o que se seguiu pode ter sido simplesmente destino.

## Sobre HagiCode

A abordagem compartilhada neste artigo vem da experiência do mundo real no [HagiCode](https://hagicode.com) projeto. HagiCode é uma plataforma de codificação assistida por IA que fornece aos desenvolvedores um assistente de codificação inteligente por meio de uma extensão VSCode, um cliente de desktop e serviços da web. Você pode ter usado ferramentas semelhantes antes e sentiu que faltava aquele toque final; entendemos bem esse sentimento.

## Por que o HagiCode precisa do Hermes

Antes de mergulhar no Hermes em si, é útil explicar por que o HagiCode precisava de algo parecido em primeiro lugar. As coisas raramente funcionam exatamente como você deseja, então você precisa de um motivo prático para se comprometer com uma direção técnica.

Como assistente de codificação de IA, o HagiCode precisa oferecer suporte a vários cenários de uso ao mesmo tempo:
- Ambientes de desenvolvimento local: os desenvolvedores desejam executá-lo em suas próprias máquinas para que os dados nunca saiam do ambiente local. Hoje em dia, a segurança dos dados nunca é uma preocupação trivial.
- Ambientes de colaboração em equipe: equipes pequenas devem ser capazes de compartilhar uma implantação de Agente em execução em um servidor. Economizar dinheiro é importante e todos têm limites.
- Expansão elástica da nuvem: ao lidar com tarefas complexas, o sistema deve escalar automaticamente para um cluster de GPU. É sempre melhor estar preparado.

Essa exigência de “queremos tudo de uma vez” foi o que nos levou à Hermes. Se foi a escolha perfeita, não posso afirmar com certeza, mas na época não vimos opção melhor.

## O que é agente Hermes

O Agente Hermes é um Agente de IA autônomo criado pela **Nous Research**. Alguns leitores podem não estar familiarizados com a Nous Research; eles são o laboratório por trás de grandes modelos de código aberto, como Hermes, Nomos e Psyché. Eles construíram muitas coisas excelentes, mesmo que ainda sejam mais subestimadas do que merecem.

Ao contrário dos assistentes de codificação IDE tradicionais ou dos simples wrappers de chat de API, o Hermes tem uma característica definidora: quanto mais tempo ele roda, mais capaz ele se torna. Ele não foi projetado para concluir uma tarefa uma vez e parar; ele continua aprendendo e acumulando experiência em operações de longa duração. Nesse sentido, parece um pouco com uma pessoa.

### Recursos principais

Vários dos principais recursos do Hermes estão intimamente alinhados com as necessidades do HagiCode.

Isso significa que o HagiCode pode escolher o modelo de implantação mais adequado com base no cenário de cada usuário: indivíduos executam-no localmente, equipes implantam-no em servidores e tarefas complexas usam recursos de GPU. Uma base de código cuida de tudo isso. Num mundo tão ocupado, poupar uma camada de complexidade já é uma vitória.

**Gateway de mensagens multiplataforma**
Hermes suporta nativamente Telegram, Discord, Slack, WhatsApp e muito mais. Para o HagiCode, isso significa que poderemos oferecer suporte aos assistentes de IA nesses canais com muito mais facilidade no futuro. Mais caminhos a seguir são sempre bem-vindos.

**Sistema de ferramentas rico**
Hermes vem com mais de 40 ferramentas integradas e suporta extensões MCP (Model Context Protocol). Isso é essencial para um assistente de codificação: a execução de comandos shell, o trabalho com o sistema de arquivos e a chamada do Git dependem do suporte da ferramenta. Um Agente sem ferramentas é como um pássaro sem asas.

**Memória entre sessões**
Hermes inclui um sistema de memória persistente e usa pesquisa de texto completo FTS5 para recuperar conversas históricas. Isso permite que o Agente se lembre do contexto anterior em vez de "perder a memória" todas as vezes. Às vezes as pessoas gostariam de esquecer as coisas tão facilmente, mas a realidade geralmente é menos generosa.

## Como o HagiCode integra o Hermes

Agora que o “porquê” está claro, vejamos o “como”. Quando algo faz sentido na teoria, o próximo passo é construí-lo.

### Abstração da camada do provedor

Na arquitetura do HagiCode, todos os provedores de IA implementam um sistema unificado `IAIProvider` interface:

```csharp
public sealed class HermesCliProvider : IAIProvider, IVersionedAIProvider
{
    public ProviderCapabilities Capabilities { get; } = new ProviderCapabilities
    {
        SupportsStreaming = true,   // Supports streaming output
        SupportsTools = true,       // Supports tool invocation
        SupportsSystemMessages = true, // Supports system prompts
        SupportsArtifacts = false
    };
}
```

Esta camada de abstração permite que o HagiCode alterne perfeitamente entre diferentes provedores de IA. Quer o back-end seja OpenAI, Claude ou Hermes, o padrão de chamada da camada superior permanece exatamente o mesmo. Em termos simples, mantém as coisas simples.

### Protocolo de comunicação ACP

Hermes se comunica através do **ACP (Protocolo de Comunicação de Agente)**. Este protocolo foi projetado especificamente para comunicação de Agentes e seus principais métodos incluem:

| Método | Descrição |
|------|------|
| `initialize` | Inicialize a conexão e obtenha a versão do protocolo e os recursos do cliente |
| `authenticate` | Lidar com autenticação e oferecer suporte a vários métodos de autenticação |
| `session/new` | Crie uma nova sessão e configure o diretório de trabalho e os servidores MCP |
| `session/prompt` | Envie um prompt e receba uma resposta |

HagiCode implementa a camada de transporte ACP através `StdioAcpTransport`, iniciando um subprocesso Hermes e comunicando-se com ele por meio de entrada e saída padrão. Pode parecer complicado, mas na prática é administrável desde que você tenha paciência suficiente.

### Gerenciamento de configuração

A configuração é gerenciada através do `HermesPlatformConfiguration` aula:

```csharp
public sealed class HermesPlatformConfiguration : IAcpPlatformConfiguration
{
    public string ExecutablePath { get; set; } = "hermes";
    public string Arguments { get; set; } = "acp";
    public int StartupTimeoutMs { get; set; } = 5000;
    public string ClientName { get; set; } = "HagiCode";
    public HermesAuthenticationConfiguration Authentication { get; set; }
    public HermesSessionDefaultsConfiguration SessionDefaults { get; set; }
}
```

Configurar Hermes em `appsettings.json`:

```json
{
  "Providers": {
    "HermesCli": {
      "ExecutablePath": "hermes",
      "Arguments": "acp",
      "StartupTimeoutMs": 10000,
      "ClientName": "HagiCode",
      "Authentication": {
        "PreferredMethodId": "api-key",
        "MethodInfo": {
          "api-key": "your-api-key-here"
        }
      },
      "SessionDefaults": {
        "Model": "claude-sonnet-4-20250514",
        "ModeId": "default"
      }
    }
  }
}
```

A configuração muitas vezes parece simples no papel, mas acertar todos os detalhes ainda exige muito esforço.

### Arquitetura Distribuída Orleans

HagiCode usa Orleans para construir seu sistema distribuído, e a integração Hermes é implementada através dos seguintes componentes:

- **HermesGrain**: uma implementação do Orleans Grain que lida com a execução da sessão
- **HermesPlatformConfiguration**: configuração específica da plataforma
- **HermesAcpSessionAdapter**: adaptador de sessão ACP
- **HermesConsole**: um console de validação dedicado

O nome Orleans tem um certo charme. Mesmo que esta Orleans não tenha nada a ver com a cidade lendária, um bom nome nunca é demais.

### Fluxo de execução ponta a ponta

A seguir está a lógica de execução principal do Provedor Hermes:

```csharp
private async IAsyncEnumerable<AIStreamingChunk> StreamCoreAsync(
    AIRequest request,
    string? embeddedCommandPrompt,
    [EnumeratorCancellation] CancellationToken cancellationToken)
{
    // 1. Create transport layer and launch Hermes subprocess
    await using var transport = new StdioAcpTransport(
        platformConfiguration.GetExecutablePath(),
        platformConfiguration.GetArguments(),
        platformConfiguration.GetEnvironmentVariables(),
        platformConfiguration.GetStartupTimeout(),
        _loggerFactory.CreateLogger<StdioAcpTransport>());
    await transport.ConnectAsync(cancellationToken);

    // 2. Initialize and obtain protocol version and authentication methods
    var initializeResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "initialize",
        BuildInitializeParameters(platformConfiguration), cancellationToken);

    // 3. Handle authentication
    var authMethods = ParseAuthMethods(initializeResult);
    if (!isAuthenticated)
    {
        var methodId = platformConfiguration.Authentication.ResolveMethodId(authMethods);
        await SendHermesRequestAsync(transport, nextRequestId++, "authenticate", ...);
    }

    // 4. Create session
    var newSessionResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "session/new",
        BuildNewSessionParameters(platformConfiguration, workingDirectory, model), cancellationToken);
    var sessionId = ParseSessionId(newSessionResult);

    // 5. Execute prompt and collect streaming responses
    await foreach (var payload in transport.ReceiveMessagesAsync(cancellationToken))
    {
        // Handle session/update notifications and convert them into streaming chunks
        if (TryParseSessionNotification(root, out var notification))
        {
            if (_responseMapper.TryConvertToStreamingChunk(notification, out var chunk))
            {
                yield return chunk;
            }
        }
    }
}
```

Com o código, os detalhes eventualmente se tornam familiares. O que mais importa é a abordagem geral.

### Verificações de saúde

Para garantir que o Hermes permaneça disponível, o HagiCode implementa um mecanismo de verificação de integridade:

```csharp
public async Task<ProviderTestResult> PingAsync(CancellationToken cancellationToken = default)
{
    var response = await ExecuteAsync(
        new AIRequest
        {
            Prompt = "Reply with exactly PONG.",
            CessionId = null,
            AllowedTools = Array.Empty<string>(),
            WorkingDirectory = ResolveWorkingDirectory(null)
        },
        cancellationToken);

    var success = string.Equals(response.Content.Trim(), "PONG", StringComparison.OrdinalIgnoreCase);
    return new ProviderTestResult
    {
        ProviderName = Name,
        Success = success,
        ResponseTimeMs = stopwatch.ElapsedMilliseconds,
        ErrorMessage = success ? null : $"Unexpected Hermes ping response: '{response.Content}'."
    };
}
```

É mais ou menos assim que se parece um “exame de saúde” aqui. De certa forma, as pessoas não são tão diferentes: ajuda fazer check-in de vez em quando, mesmo que ninguém nos diga exatamente o que procurar.

## Considerações Práticas

Existem algumas armadilhas que vale a pena entender antes de integrar o Hermes. Todo mundo cai em algumas armadilhas, mais cedo ou mais tarde.

### Configuração do método de autenticação

Hermes oferece suporte a vários métodos de autenticação, incluindo chaves de API e tokens, portanto, você precisa escolher com base no cenário real de implantação. A configuração incorreta pode causar falhas de conexão e as mensagens de erro resultantes nem sempre são intuitivas. Às vezes, o erro relatado está longe da causa raiz real, o que significa que uma depuração lenta e cuidadosa é inevitável.

### Configuração do servidor MCP

Ao criar uma sessão, você pode configurar uma lista de servidores MCP para que o Hermes possa chamar ferramentas externas. Mas tenha em mente os seguintes pontos:
- Os endereços do servidor MCP devem estar acessíveis
- Os tempos limite devem ser configurados razoavelmente
- O sistema precisa de tratamento de degradação quando um servidor está indisponível

Na prática, o pensamento defensivo é mais importante do que as pessoas esperam.

### Gerenciamento de diretório de trabalho

Cada sessão deve especificar um diretório de trabalho para que o Hermes possa acessar os arquivos do projeto corretamente. Em cenários de vários projetos, o diretório de trabalho precisa ser alternado dinamicamente. Parece simples, mas há mais casos extremos do que você imagina.

### Agregação de Resposta

As respostas de Hermes podem ser divididas em `session/update` notificações e o resultado final, portanto devem ser mesclados corretamente. Caso contrário, o conteúdo poderá ser perdido.

### Estratégia de tratamento de erros

Os erros de tempo de execução devem ser retornados explicitamente, em vez de retornarem silenciosamente para outro provedor. Dessa forma, os usuários sabem que o problema veio da Hermes, em vez de se perguntarem por que o sistema trocou repentinamente de modelo nos bastidores.

## Conclusão

A decisão da HagiCode de usar Hermes como núcleo de agente integrado não foi um impulso casual. Foi uma escolha criteriosa baseada nos requisitos práticos e nas características técnicas da estrutura. Ainda é muito cedo para dizer se esta é a resposta perfeita a longo prazo, mas até agora tem-nos servido bem.

Hermes dá ao HagiCode flexibilidade para se adaptar a uma ampla gama de cenários. Seu poderoso sistema de ferramentas e suporte MCP permitem que o assistente de IA faça um trabalho real, enquanto o protocolo ACP e a camada de abstração do provedor mantêm o processo de integração claro e controlável.

Se você estiver escolhendo uma estrutura de agente para seu próprio projeto de IA, espero que este artigo ofereça uma referência útil. Escolher a arquitetura subjacente correta pode tornar tudo o que se segue muito mais fácil.
