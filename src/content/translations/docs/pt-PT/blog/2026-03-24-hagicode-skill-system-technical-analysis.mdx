## Análise técnica do sistema de habilidades HagiCode: construindo uma plataforma escalonável de gerenciamento de habilidades de IA

> Este artigo analisa em profundidade a arquitetura e implementação do sistema de gerenciamento de habilidades no projeto HagiCode, cobrindo os detalhes técnicos por trás de quatro capacidades principais: gerenciamento local global, pesquisa de mercado, recomendações inteligentes e gerenciamento de fornecedores confiáveis.

## Plano de fundo

No campo dos assistentes de codificação de IA, como estender os limites das capacidades de IA sempre foi uma questão central. O próprio Claude Code já é forte na assistência de código, mas diferentes equipes de desenvolvimento e diferentes pilhas de tecnologia geralmente precisam de recursos especializados para cenários específicos, como lidar com implantações de Docker, otimização de banco de dados ou geração de componentes de front-end. É exatamente aí que um sistema de Skills se torna especialmente importante.

Durante o desenvolvimento do projeto HagiCode, nos deparamos com um desafio semelhante: como deixar Claude Code “aprender” novas habilidades profissionais como uma pessoa faria, mantendo ao mesmo tempo uma sólida experiência de usuário e boa manutenção de engenharia? Este problema é difícil e simples à sua maneira. Em torno dessa questão, projetamos e implementamos um sistema completo de gestão de competências.

Este artigo aborda detalhadamente a arquitetura técnica e a implementação principal do sistema. Destina-se a desenvolvedores interessados ​​em extensibilidade de IA e integração de ferramentas de linha de comando. Pode ser útil para você ou não, mas pelo menos está escrito agora.

## Sobre HagiCode

A abordagem compartilhada neste artigo vem de nossa experiência prática no [HagiCode](https://hagicode.com) projeto. HagiCode é um projeto de assistente de codificação de IA de código aberto projetado para ajudar as equipes de desenvolvimento a melhorar a eficiência da engenharia. A pilha do projeto inclui ASP.NET Core, a estrutura distribuída Orleans, um frontend TanStack Start + React e o subsistema de gerenciamento de habilidades apresentado neste artigo.

O repositório GitHub é [HagiCode-org/site](https://github.com/HagiCode-org/site). Se você achar valiosa a abordagem técnica deste artigo, sinta-se à vontade para atribuir uma estrela. Afinal, mais estrelas tendem a melhorar o humor.

## Visão geral da arquitetura do sistema

O sistema Skill usa uma arquitetura separada frontend-backend. Não há nada especialmente misterioso nisso.

**Frontend** usa TanStack Start + React para construir a interface do usuário, com estado de gerenciamento do Redux Toolkit. Os quatro recursos principais são mapeados diretamente para quatro componentes da guia: habilidades locais, galeria de habilidades, recomendações inteligentes e provedores confiáveis. No final das contas, o objetivo principal do design é melhorar a experiência do usuário.

**Backend** é baseado em ASP.NET Core + ABP Framework, usando Orleans Grain para gerenciamento de estado distribuído. O cliente da API on-line envolve o `IOnlineApiClient` interface para se comunicar com o serviço de catálogo de habilidades remotas.

O princípio geral da arquitetura é separar a execução de comandos da lógica de negócios. Através do padrão adaptador, os detalhes de implementação da execução do comando npm/npx ficam ocultos dentro de módulos independentes. Afinal, ninguém quer chamadas de linha de comando espalhadas por toda a base de código.

## Capacidade Principal 1: Gestão Global Local

A gestão global local é o módulo mais básico. Ele é responsável por listar as habilidades instaladas e dar suporte às operações de desinstalação. Não há nada muito complicado aqui; trata-se principalmente de fazer bem o básico.

### Abordagem Técnica

A implementação vive em `LocalSkillsTab.tsx` e `LocalSkillCommandAdapter.cs`. A ideia central é envolver o `npx skills` comando, analisar sua saída JSON e convertê-la em estruturas de dados internas. Parece simples e, na prática, na maior parte é.

```csharp
public async Task<IReadOnlyList<LocalSkillInventoryResponseDto>> GetLocalSkillsAsync(
    CancellationToken cancellationToken = default)
{
    var result = await _commandAdapter.ListGlobalSkillsAsync(cancellationToken);
    return result.Skills.Select(skill => new LocalSkillInventoryResponseDto
    {
        Name = skill.Name,
        Version = skill.Version,
        Source = skill.Source,
        InstalledPath = skill.InstalledPath,
        Description = skill.Description
    }).ToList();
}
```

O fluxo de dados é muito claro: o frontend envia uma solicitação -> `SkillGalleryAppService` recebe -> `LocalSkillCommandAdapter` executa o comando npx -> o resultado JSON é analisado -> um DTO é retornado. Cada etapa segue naturalmente a anterior.

A desinstalação de habilidade usa o `npx skills remove -g <skillName> -y` comando, e o sistema lida automaticamente com dependências e limpeza. Os metadados de instalação são armazenados em `managed-install.json` dentro do diretório de habilidades, registrando informações como tempo de instalação e versão de origem para atualizações e auditorias posteriores. Algumas coisas simplesmente valem a pena registrar.

### Fluxo de instalação em detalhes

A instalação de habilidades requer várias etapas coordenadas. Na verdade, não é especialmente complicado:

```csharp
public async Task<SkillInstallResultDto> InstallAsync(
    SkillInstallRequestDto request,
    CancellationToken cancellationToken = default)
{
    // 1. Normalize the installation reference
    var normalized = _referenceNormalizer.Normalize(
        request.SkillId,
        request.Source,
        request.SkillSlug,
        request.Version);

    // 2. Check prerequisites
    await _prerequisiteChecker.CheckAsync(cancellationToken);

    // 3. Acquire installation lock
    using var installLock = await _lockProvider.AcquireAsync(normalized.SkillId);

    // 4. Execute installation command
    var result = await _installCommandRunner.ExecuteAsync(
        new SkillInstallCommandExecutionRequest
        {
            Command = $"npx skills add {normalized.FullReference} -g -y",
            Timeout = TimeSpan.FromMinutes(4)
        },
        cancellationToken);

    // 5. Persist installation metadata
    await _metadataStore.WriteAsync(normalized.SkillPath, request);

    return new SkillInstallResultDto { Success = result.Success };
}
```

Vários padrões de design importantes são usados aqui: o **normalizador de referência** converte diferentes formatos de entrada, como `tanweai/pua` e `@opencode/docker-skill`, numa representação interna unificada; o **mecanismo de bloqueio de instalação** garante que apenas uma operação de instalação possa ser executada para a mesma habilidade por vez; e a **saída de streaming** envia o progresso da instalação para o front-end em tempo real por meio de eventos enviados pelo servidor, para que os usuários possam observar logs semelhantes a terminais à medida que eles acontecem.

No final, todos esses padrões existem com um propósito: manter o sistema mais simples de usar e manter.

## Capacidade principal 2: pesquisa de mercado

A pesquisa no Marketplace permite que os usuários descubram e instalem habilidades da comunidade. A capacidade de uma pessoa é sempre limitada; o conhecimento coletivo vai muito além.

### Abordagem Técnica

O recurso de pesquisa depende da API online `https://api.hagicode.com/v1/skills/search`. Para melhorar a velocidade de resposta, o sistema implementa cache. Cache é um pouco como memória: se você mantiver coisas úteis por perto, não precisará pensar tanto na próxima vez.

```csharp
private async Task<IReadOnlyList<SkillGallerySkillDto>> SearchCatalogAsync(
    string query,
    CancellationToken cancellationToken,
    IReadOnlySet<string>? allowedSources = null)
{
    var cacheKey = $"skill_search:{query}:{string.Join(",", allowedSources ?? Array.Empty<string>())}";

    if (_memoryCache.TryGetValue(cacheKey, out var cached))
        return (IReadOnlyList<SkillGallerySkillDto>)cached!;

    var response = await _onlineApiClient.SearchAsync(
        new SearchSkillsRequest
        {
            Query = query,
            Limit = _options.LimitPerQuery,
        },
        cancellationToken);

    var results = response.Skills
        .Where(skill => allowedSources is null || allowedSources.Contains(skill.Source))
        .Select(skill => new SkillGallerySkillDto { ... })
        .ToList();

    _memoryCache.Set(cacheKey, results, TimeSpan.FromMinutes(10));
    return results;
}
```

Os resultados da pesquisa oferecem suporte à filtragem por fontes confiáveis, para que os usuários vejam apenas fontes de habilidades em que confiam. Consultas iniciais, como `popular` e `recent` são usados para inicializar o catálogo, permitindo que os usuários vejam as habilidades populares recomendadas na primeira vez que o abrirem. As primeiras impressões ainda importam.

## Capacidade Principal 3: Recomendações Inteligentes

As recomendações inteligentes são a parte mais complexa do sistema. Eles podem recomendar automaticamente as habilidades mais adequadas com base no contexto atual do projeto. Por mais complexo que seja, ainda vale a pena construir.

### Fluxo de recomendação

O fluxo completo de recomendação é dividido em cinco etapas:

```
1. Build project context
   ↓
2. AI generates search queries
   ↓
3. Search the online catalog in parallel
   ↓
4. AI ranks the candidates
   ↓
5. Return the recommendation list
```

Primeiro, o sistema analisa características como a pilha de tecnologia do projeto, linguagens de programação e estrutura de domínio para construir um “perfil do projeto”. Esse perfil é um pouco como um currículo, registrando as principais características do projeto.

Em seguida, um AI Grain é usado para gerar consultas de pesquisa direcionadas. Na verdade, esse design é bastante interessante: em vez de perguntar diretamente à IA: “Quais habilidades devo recomendar?”, primeiro pedimos que ela pense em “Quais termos de pesquisa têm probabilidade de encontrar habilidades relevantes?” Às vezes, a maneira como você faz a pergunta é mais importante do que a resposta em si:

```csharp
var queryGeneration = await aiGrain.GenerateSkillRecommendationQueriesAsync(
    projectContext,      // Project context
    locale,              // User language preference
    maxQueries,           // Maximum number of queries
    effectiveSearchHero); // AI model selection
```

Em seguida, essas consultas de pesquisa são executadas em paralelo para reunir uma lista de habilidades candidatas. O processamento paralelo é, no final das contas, apenas uma forma de economizar tempo.

Finalmente, outro AI Grain classifica as habilidades candidatas. Esta etapa considera fatores como relevância da habilidade para o projeto, status de confiança e preferências históricas do usuário:

```csharp
var ranking = await aiGrain.RankSkillRecommendationsAsync(
    projectContext,
    candidates,
    installedSkillNames,
    locale,
    maxRecommendations,
    effectiveRankingHero);

response.Items = MergeRecommendations(projectContext, candidates, ranking, maxRecommendations);
```

### Mecanismo de reserva

Os modelos de IA podem responder lentamente ou ficar temporariamente indisponíveis. Até os melhores sistemas tropeçam às vezes. Por esse motivo, o sistema inclui um mecanismo determinístico de fallback: quando o serviço de IA não está disponível, ele usa um algoritmo heurístico baseado em regras para gerar recomendações, como inferir prováveis habilidades necessárias a partir de dependências em `package.json`.

Simplificando, esse mecanismo alternativo é simplesmente um plano de backup para o sistema.

## Capacidade Principal 4: Gerenciamento de Provedor Confiável

O gerenciamento de provedores confiáveis permite que os usuários controlem quais fontes de habilidades são consideradas confiáveis. A confiança ainda é algo que os usuários devem ser capazes de definir por si próprios.

### Regras de correspondência

Provedores confiáveis oferecem suporte a duas regras de correspondência: **correspondência exata** (`exact`) e **correspondência de prefixo** (`prefix`).

```csharp
public static TrustedSkillProviderResolutionSnapshot Resolve(
    TrustedSkillProviderSnapshot snapshot,
    string source)
{
    var normalizedSource = Normalize(source);

    foreach (var entry in snapshot.Entries.OrderBy(e => e.SortOrder))
    {
        if (!entry.IsEnabled) continue;

        foreach (var rule in entry.MatchRules)
        {
            bool isMatch = rule.MatchType switch
            {
                TrustedSkillProviderMatchRuleType.Exact
                    => string.Equals(normalizedSource, Normalize(rule.Value),
                        StringComparison.OrdinalIgnoreCase),
                TrustedSkillProviderMatchRuleType.Prefix
                    => normalizedSource.StartsWith(Normalize(rule.Value) + "/",
                        StringComparison.OrdinalIgnoreCase),
                _ => false
            };

            if (isMatch)
                return new TrustedSkillProviderResolutionSnapshot
                {
                    IsTrustedSource = true,
                    ProviderId = entry.ProviderId,
                    DisplayName = entry.DisplayName
                };
        }
    }

    return new TrustedSkillProviderResolutionSnapshot { IsTrustedSource = false };
}
```

Provedores confiáveis ​​integrados incluem organizações e projetos conhecidos, como Vercel, Azure, anthropics, Microsoft e uso de navegador. Provedores personalizados podem ser adicionados por meio de arquivos de configuração especificando um ID de provedor, nome de exibição, rótulo de crachá, regras de correspondência e muito mais. O mundo é grande o suficiente para que confiar apenas em alguns integrados nunca seria suficiente.

### Implementação de Persistência

A configuração confiável é mantida usando um Orleans Grain:

```csharp
public class TrustedSkillProviderGrain : Grain<TrustedSkillProviderState>,
    ITrustedSkillProviderGrain
{
    public async Task UpdateConfigurationAsync(TrustedSkillProviderSnapshot snapshot)
    {
        State.Snapshot = snapshot;
        await WriteStateAsync();
    }

    public Task<TrustedSkillProviderSnapshot> GetConfigurationAsync()
    {
        return Task.FromResult(State.Snapshot);
    }
}
```

A vantagem desta abordagem é que as alterações de configuração são sincronizadas automaticamente em todos os nós, sem qualquer necessidade de atualizar os caches manualmente. Em última análise, a automação consiste em permitir que as pessoas se preocupem menos.

## Projeto Técnico Chave

### Padrão de Adaptador de Execução de Comando

O sistema Skill precisa executar vários comandos npx. Se essa lógica estivesse espalhada por todos os lugares, o código rapidamente se tornaria difícil de manter. É por isso que projetamos uma interface de adaptador. No final das contas, os padrões de design existem para tornar o código mais fácil de manter:

```csharp
public interface ISkillInstallCommandRunner
{
    Task<SkillInstallCommandExecutionResult> ExecuteAsync(
        SkillInstallCommandExecutionRequest request,
        CancellationToken cancellationToken = default);
}
```

Comandos diferentes têm implementações de executores diferentes, mas todos implementam a mesma interface, simplificando o teste e a substituição.

### Saída de streaming SSE

O progresso da instalação é enviado ao frontend em tempo real por meio de eventos enviados pelo servidor:

```csharp
public async Task InstallWithProgressAsync(
    SkillInstallRequestDto request,
    IServerStreamWriter<SkillInstallProgressEventDto> stream,
    CancellationToken cancellationToken)
{
    var process = new Process
    {
        StartInfo = new ProcessStartInfo
        {
            FileName = "npx",
            Arguments = $"skills add {request.FullReference} -g -y",
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false
        }
    };

    process.OutputDataReceived += async (sender, e) =>
    {
        await stream.WriteAsync(new SkillInstallProgressEventDto
        {
            EventType = "output",
            Data = e.Data ?? string.Empty
        });
    };

    process.Start();
    process.BeginOutputReadLine();
    await process.WaitForExitAsync(cancellationToken);
}
```

No frontend, os usuários podem ver a saída semelhante a um terminal em tempo real, o que torna a experiência muito intuitiva. O feedback em tempo real ajuda as pessoas a se sentirem à vontade.

## Guia Prático

### Instalando uma habilidade comunitária

Faça a instalação do `pua` habilidade como exemplo (é uma habilidade popular na comunidade):

1. Abra a gaveta Habilidades e mude para o `Skill Gallery` guia
2. Entrar `pua` na caixa de pesquisa
3. Clique no resultado da pesquisa para ver os detalhes da habilidade
4. Clique no `Install` botão
5. Mude para o `Local Skills` guia para confirmar que a instalação foi bem-sucedida

O comando de instalação é `npx skills add tanweai/pua -g -y`, e o sistema trata todos os detalhes automaticamente. Na verdade, não há muitas etapas depois que você as executa uma por uma.

### Adicionando uma fonte confiável personalizada

Se sua equipe tiver seu próprio repositório de habilidades, você poderá adicioná-lo como uma fonte confiável:

```yaml
providerId: "my-team"
displayName: "My Team Skills"
badgeLabel: "MyTeam"
isEnabled: true
sortOrder: 100
matchRules:
  - matchType: "prefix"
    value: "my-team/"
  - matchType: "exact"
    value: "my-team/special-skill"
```

Dessa forma, todas as habilidades da sua equipe exibirão um selo de confiança, deixando os usuários mais confortáveis em instalá-las. Etiquetas e sinais ajudam as pessoas a se sentirem mais confiantes.

### Noções básicas de desenvolvimento de habilidades

A criação de uma habilidade personalizada requer a seguinte estrutura:

```
my-skill/
├── SKILL.md          # Skill metadata (YAML front matter)
├── index.ts          # Skill entry point
├── agents/           # Supported agent configuration
└── references/       # Reference resources
```

Um exemplo `SKILL.md` formato:

```yaml
---
name: my-skill
description: A brief description of what this skill does
---

# My Skill

Detailed documentation...
```

## Notas

1. **Requisitos de rede**: a pesquisa e instalação de habilidades exigem acesso a `api.hagicode.com` e o registro npm
2. **Versão Node.js**: Node.js 18 ou posterior é recomendado
3. **Requisitos de permissão**: são necessárias permissões de instalação globais do npm
4. **Controle de simultaneidade**: apenas uma operação de instalação ou desinstalação pode ser executada para a mesma habilidade por vez
5. **Configurações de tempo limite**: o tempo limite padrão para instalação é de 4 minutos, mas cenários complexos podem exigir ajustes

Essas notas existem, em última análise, para ajudar as coisas a correrem bem.

## Conclusão

Este artigo apresentou a implementação completa do sistema de gerenciamento de habilidades no projeto HagiCode. Por meio de uma arquitetura separada de front-end e back-end, do padrão de adaptador, do gerenciamento de estado distribuído baseado em Orleans e de técnicas relacionadas, o sistema oferece:

- **Gerenciamento global local**: uma interface unificada de gerenciamento de habilidades criada por empacotamento `npx skills` comandos
- **Pesquisa no Marketplace**: descoberta rápida de habilidades da comunidade por meio da API on-line e de mecanismos de cache
- **Recomendações inteligentes**: recomendações de habilidades baseadas em IA com base no contexto do projeto
- **Gerenciamento de confiança**: um sistema de configuração flexível que permite aos usuários controlar os limites de confiança

Esta abordagem de design não se aplica apenas ao gerenciamento de habilidades. Também é útil como referência para qualquer cenário que precise integrar ferramentas de linha de comando enquanto equilibra armazenamento local e serviços online.
