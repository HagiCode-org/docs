Se você deseja instalar o Hermes e começar a usá-lo, o caminho mais curto são apenas três etapas:

1. Execute o comando de instalação oficial
2. Inicie a CLI em seu terminal com `hermes`
3. Se você quiser continuar usando no Feishu, configure `hermes gateway setup`

Este artigo não tenta explicar todas as capacidades do Hermes de uma só vez. Em vez disso, ele ajuda você a concluir primeiro o ciclo mais importante para iniciantes: instalá-lo, colocá-lo em execução, começar a usá-lo e, em seguida, conectá-lo a um dos cenários mais comuns de plataforma de mensagens.

## O que é Hermes e para quem serve

Hermes Agent é um agente de IA que você pode usar em um terminal local ou por meio de um gateway de plataforma de mensagens.

Para a maioria dos desenvolvedores, possui dois pontos de entrada comuns:

- **CLI**: Tipo `hermes` em seu terminal para entrar diretamente na interface interativa.
- **Gateway de mensagens**: Executar `hermes gateway`e converse com ele em plataformas como Feishu, Telegram, Discord e Slack.

Se o seu objetivo agora é simplesmente começar rapidamente, não inverta a ordem. Comece com este caminho:

- Instale o Hermes primeiro
- Verifique se funciona primeiro na CLI
- Em seguida, decida se deseja conectar uma plataforma de mensagens

Isso torna os problemas mais fáceis de diagnosticar e é mais adequado para pessoas que usam Hermes pela primeira vez.

## O que saber antes de instalar o Hermes

De acordo com o README do Hermes, o caminho oficial de instalação rápida suporta estes ambientes:

- Linux
- macOS
- WSL2
- Android via Termux

### Notas para usuários do Windows

Atualmente, o Hermes não oferece suporte à execução direta no Windows nativo. Se você estiver usando o Windows, a abordagem recomendada é instalar o WSL2 primeiro e depois executar o comando de instalação dentro do WSL2.

É melhor deixar isso claro no início, porque muitas falhas de instalação não são causadas pelo comando em si, mas pelo uso de um ambiente de tempo de execução não suportado.

## Como instalar o Hermes

O comando de instalação rápida fornecido no README do Hermes é:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Este comando executa o script de instalação oficial e trata das etapas de inicialização específicas da plataforma.

### Recarregue seu shell após a instalação

Assim que a instalação terminar, recarregue primeiro o ambiente shell. O comando mais comum é:

```bash
source ~/.bashrc
```

Se você usa zsh, você pode usar:

```bash
source ~/.zshrc
```

### Como confirmar se o Hermes está instalado corretamente

A maneira mais direta de verificar é executar:

```bash
hermes
```

Se quiser confirmação adicional de que sua configuração e dependências estão funcionando, você também pode executar:

```bash
hermes doctor
```

`hermes doctor` é especialmente útil nestas situações:

- O comando se comporta de maneira anormal após a instalação
- A configuração do modelo falha
- O gateway não inicia
- Você não tem certeza se as dependências do seu ambiente estão completas

## Como começar a usar Hermes pela primeira vez

### Comece com a CLI

Se você deseja apenas confirmar o mais rápido possível se o Hermes funciona, o método mais simples é:

```bash
hermes
```

Isso inicia o Hermes CLI interativo. Para usuários iniciantes do Hermes, também é o ponto de partida mais recomendado, pois você pode verificar primeiro o mais essencial:

- Se o comando está realmente disponível
- Se a configuração do modelo atual funciona corretamente
- Se a cadeia de ferramentas do terminal está funcionando corretamente
- Se o estilo de interação corresponde ao que você precisa

### Esses comandos são suficientes para sua primeira rodada de configuração

O README do Hermes lista vários comandos de alta frequência e, juntos, eles formam um caminho prático para o primeiro uso:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Se você não tem certeza do que cada um faz, lembre-se deles assim:

- `hermes model`: escolha ou troque de modelo
- `hermes tools`: visualize e configure as ferramentas atualmente disponíveis
- `hermes config set`: altera itens de configuração específicos
- `hermes setup`: execute o assistente de inicialização completo uma vez
- `hermes update`: atualizar Hermes
- `hermes doctor`: solucionar problemas

Para iniciantes, a ordem mais prática costuma ser:

1. Corre `hermes model` primeiro
2. Se você deseja configurar todas as opções comuns de uma vez, execute `hermes setup`

## As duas maneiras mais comuns de usar Hermes

### 1. Use Hermes no terminal como assistente de desenvolvimento diário

O modo CLI é uma boa opção para estes cenários:

- Faça perguntas diretamente enquanto escreve código localmente
- Inspecione projetos, edite arquivos e execute comandos
- Faça depuração única ou trabalho de revisão
- Colabore continuamente no diretório de trabalho atual

Sua maior vantagem é que é o caminho mais curto: sem integração extra de plataforma, sem configuração de bot para lidar antecipadamente e é a melhor maneira de construir seu primeiro conjunto de hábitos de uso.

### 2. Use o Hermes por meio de uma plataforma de mensagens

Se quiser bater um papo com Hermes em plataformas como Feishu, Telegram ou Discord, você precisa usar o gateway de mensagens.

Os comandos de entrada mais comuns são:

```bash
hermes gateway setup
hermes gateway
```

Especificamente:

- `hermes gateway setup` é usado para configuração de plataforma interativa
- `hermes gateway` é usado para iniciar o processo de gateway

De acordo com a documentação oficial, o gateway é um processo unificado em segundo plano que conecta suas plataformas configuradas, gerencia sessões e lida com recursos como cron jobs.

## Usando Feishu como exemplo: como conectar Hermes a uma plataforma de mensagens

Se a maior parte do seu trabalho diário acontece em Feishu, então Feishu/Lark é uma maneira muito natural de usar Hermes.

### O caminho de integração mínimo viável

A documentação oficial recomenda este comando de entrada para Feishu/Lark:

```bash
hermes gateway setup
```

Depois de executá-lo, basta escolher **Feishu / Lark** no assistente.

A documentação do Feishu descreve dois modos de conexão:

- `websocket`: recomendado
- `webhook`: opcional

Se o Hermes for executado em seu laptop, estação de trabalho ou servidor privado, usando `websocket` first geralmente é mais simples porque você não precisa expor uma URL de retorno de chamada pública.

### Se você configurar manualmente, pelo menos conheça essas variáveis

Se você não estiver usando o assistente e estiver escrevendo a configuração manualmente, a documentação do Feishu lista estas variáveis principais:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Dois deles merecem atenção especial:

- `FEISHU_ALLOWED_USERS`: recomendado, para que nem todos que conseguem acessar o bot possam usá-lo diretamente
- `FEISHU_HOME_CHANNEL`: permite predefinir um bate-papo inicial para receber resultados do cron ou notificações padrão

### Por que Hermes às vezes não responde nos bate-papos em grupo do Feishu

Esse detalhe é fácil de perder: nos chats em grupo do Feishu, Hermes não responde a todas as mensagens por padrão.

A documentação oficial afirma claramente:

- Nas mensagens diretas, Hermes responde às mensagens
- Em bate-papos em grupo, você deve explicitamente `@` o bot antes de processar a mensagem

Se quiser definir uma conversa Feishu como canal inicial, você também pode usar isto no chat:

```text
/set-home
```

Ou defina-o na configuração com antecedência:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Os comandos Hermes que os iniciantes devem lembrar primeiro

Quer você use o Hermes na CLI ou em uma plataforma de mensagens, lembrar os seguintes comandos já é suficiente para começar:

- `/new` ou `/reset`: iniciar uma nova sessão
- `/model`: visualizar ou mudar o modelo
- `/retry`: tente novamente o turno anterior
- `/undo`: desfaz a interação anterior
- `/compress`: compactar manualmente o contexto
- `/help`: ver ajuda

Se você usa Hermes principalmente em uma plataforma de mensagens, lembre-se de mais uma:

- `/sethome` ou `/set-home`: define o chat atual como canal inicial

Esses comandos cobrem as operações mais comuns do estágio inicial: reiniciar, ajustar, reverter, verificar e continuar.

## Perguntas frequentes

### O Windows pode instalar o Hermes diretamente?

A documentação oficial atual afirma claramente que o Windows nativo não é compatível e o WSL2 é recomendado.

### O que devo fazer se digitar `hermes` não faz nada após a instalação?

É melhor solucionar o problema nesta ordem:

1. Recarregue seu shell primeiro, por exemplo com `source ~/.bashrc`
2. Corre `hermes` de novo
3. Se ainda estiver anormal, execute `hermes doctor`

### Por que o bot não responde em um grupo Feishu?

Verifique estas três coisas primeiro:

- Se você `@` mencionou Hermes no grupo
- Se `FEISHU_ALLOWED_USERS` restringe o usuário atual
- Se a política atual de bate-papo em grupo permite lidar com mensagens de grupo

De acordo com a documentação oficial do Feishu, é necessário usar explicitamente uma @menção em cenários de bate-papo em grupo.
