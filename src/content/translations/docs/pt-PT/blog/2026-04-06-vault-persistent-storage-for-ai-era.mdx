## Como reproduzir projetos na era da IA: Vault, um sistema de armazenamento persistente entre projetos

> Na era do desenvolvimento assistido por IA, como podemos ajudar os assistentes de IA a compreender melhor os nossos recursos de aprendizagem? O projeto HagiCode construiu o sistema Vault como uma camada unificada de abstração de armazenamento de conhecimento que a IA pode compreender, melhorando significativamente a eficiência do aprendizado por meio da reprodução do projeto.

## Plano de fundo

Na era da IA, a forma como os desenvolvedores aprendem novas tecnologias e arquiteturas está mudando profundamente. "Reproduzir projetos" - isto é, estudar profundamente e aprender com o código, a arquitetura e os padrões de design de excelentes projetos de código aberto - tornou-se uma forma eficiente de aprender. Em comparação com métodos tradicionais, como ler livros ou assistir a vídeos, ler e executar diretamente projetos de código aberto de alta qualidade ajuda a compreender as práticas de engenharia do mundo real com muito mais rapidez.

Ainda assim, esse método de aprendizagem apresenta alguns desafios.

Os materiais de aprendizagem estão muito dispersos. Suas anotações podem estar no Obsidian, os repositórios de código podem estar espalhados em diferentes pastas e o histórico de conversas do seu assistente de IA se torna mais uma ilha de dados isolada. Quando você deseja que a IA ajude a analisar um projeto, você precisa copiar manualmente trechos de código e organizar o contexto, o que é bastante tedioso.

O que é ainda mais problemático é o contexto quebrado. Os assistentes de IA não podem acessar diretamente seus recursos de aprendizagem locais, então você precisa fornecer informações básicas novamente em cada conversa. Além disso, os repositórios de código reproduzido são atualizados rapidamente, a sincronização manual é propensa a erros e é difícil compartilhar o conhecimento entre vários projetos de aprendizagem.

Na raiz, todos esses problemas vêm de “ilhas de dados”. Se houvesse uma camada de abstração de armazenamento unificada que permitisse aos assistentes de IA entender e acessar todos os seus recursos de aprendizagem, o problema seria resolvido perfeitamente.

## Sobre HagiCode

O sistema Vault compartilhado neste artigo é exatamente a solução que desenvolvemos durante a construção [HagiCode](https://hagicode.com). HagiCode é um projeto assistente de codificação de IA e, em nosso trabalho diário de desenvolvimento, muitas vezes precisamos estudar e consultar muitos projetos de código aberto diferentes. Para ajudar os assistentes de IA a entender melhor esses recursos de aprendizagem, projetamos o Vault, um sistema de armazenamento persistente entre projetos.

Esta solução já foi validada no HagiCode em uso real. Se você estiver enfrentando desafios semelhantes de gestão do conhecimento, espero que essas experiências possam oferecer alguma inspiração. Afinal, depois de cair em alguns buracos, você deve deixar algo para trás para a próxima pessoa.

## Filosofia de design do sistema Vault

A ideia central do sistema Vault é simples: criar uma camada de abstração de armazenamento de conhecimento unificada que a IA possa entender. Do ponto de vista da implementação, o sistema possui várias características principais.

### Suporte multitipo

O sistema suporta quatro tipos de cofre, cada um correspondendo a um cenário de utilização diferente:

```typescript
// folder: general-purpose folder type
export const DEFAULT_VAULT_TYPE = 'folder';

// coderef: a type specifically for reproduced code projects
export const CODEREF_VAULT_TYPE = 'coderef';

// obsidian: integrated with Obsidian note-taking software
export const OBSIDIAN_VAULT_TYPE = 'obsidian';

// system-managed: vault automatically managed by the system
export const SYSTEM_MANAGED_VAULT_TYPE = 'system-managed';
```

Dentre eles, o tipo **coderef** é o mais utilizado no HagiCode. Ele foi projetado especificamente para projetos de código reproduzido, fornecendo uma estrutura de diretório padronizada e descrições de metadados legíveis por IA.

### Mecanismo de armazenamento persistente

O registro do Vault é armazenado persistentemente no formato JSON, garantindo que a configuração permaneça disponível após a reinicialização do aplicativo:

```csharp
public class VaultRegistryStore : IVaultRegistryStore
{
    private readonly string _registryFilePath;

    public VaultRegistryStore(IConfiguration configuration, ILogger<VaultRegistryStore> logger)
    {
        var dataDir = configuration["DataDir"] ?? "./data";
        var absoluteDataDir = Path.IsPathRooted(dataDir)
            ? dataDir
            : Path.GetFullPath(Path.Combine(Directory.GetCurrentDirectory(), dataDir));

        _registryFilePath = Path.Combine(absoluteDataDir, "personal-data", "vaults", "registry.json");
    }
}
```

A vantagem deste design é que ele é simples e confiável. JSON é legível por humanos, o que facilita a depuração e a edição manual; o armazenamento do sistema de arquivos evita a complexidade de um banco de dados e reduz as dependências do sistema. Afinal, às vezes a opção mais simples é realmente a melhor.

### Integração de contexto de IA

Mais importante ainda, o sistema pode injetar automaticamente informações do cofre no contexto das propostas de IA:

```typescript
export function buildTargetVaultsText(
  vaults: VaultForText[],
  template: VaultPromptTemplate = DEFAULT_VAULT_PROMPT_TEMPLATE,
): string {
  const readOnlyVaults = vaults.filter((vault) => vault.accessType === 'read');
  const editableVaults = vaults.filter((vault) => vault.accessType === 'write');

  if (readOnlyVaults.length === 0 && editableVaults.length === 0) {
    return '';
  }

  const sections = [
    buildVaultSection(readOnlyVaults, template.reference),
    buildVaultSection(editableVaults, template.editable),
  ].filter(Boolean);

  return `\n\n### ${template.heading}\n\n${sections.join('\n')}`;
}
```

Isso permite um recurso importante: os assistentes de IA podem compreender automaticamente os recursos de aprendizagem disponíveis sem que os usuários forneçam contexto manualmente. Você poderia dizer que isso conta como uma espécie de entendimento tácito.

## A estrutura padronizada do CodeRef Vault

Para o tipo de cofre coderef, o HagiCode fornece uma estrutura de diretório padronizada:

```
my-coderef-vault/
├── index.yaml          # vault metadata description
├── AGENTS.md           # operating guide for AI assistants
├── docs/               # stores study notes and documents
└── repos/              # manages reproduced code repositories through Git submodules
```

Ao criar um vault, o sistema inicializa automaticamente esta estrutura:

```csharp
private async Task EnsureCodeRefStructureAsync(
    string vaultName,
    string physicalPath,
    ICollection<VaultBootstrapDiagnosticDto> diagnostics,
    CancellationToken cancellationToken)
{
    Directory.CreateDirectory(physicalPath);

    var indexPath = Path.Combine(physicalPath, CodeRefIndexFileName);
    var docsPath = Path.Combine(physicalPath, CodeRefDocsDirectoryName);
    var reposPath = Path.Combine(physicalPath, CodeRefReposDirectoryName);

    // Create the standard directory structure
    if (!Directory.Exists(docsPath))
    {
        Directory.CreateDirectory(docsPath);
    }

    if (!Directory.Exists(reposPath))
    {
        Directory.CreateDirectory(reposPath);
    }

    // Create the AGENTS.md guide
    await EnsureCodeRefAgentsDocumentAsync(physicalPath, cancellationToken);

    // Create the index.yaml metadata
    await WriteCodeRefIndexDocumentAsync(indexPath, mergedDocument, cancellationToken);
}
```

Esta estrutura também foi cuidadosamente projetada:

- **docs/** armazena suas notas de estudo, onde você pode registrar sua compreensão do código, análise da arquitetura, lições aprendidas e assim por diante no Markdown
- **repos/** gerencia repositórios reproduzidos por meio de submódulos Git em vez de copiar o código diretamente, o que mantém o código sincronizado e economiza espaço
- **index.yaml** contém os metadados do cofre para que os assistentes de IA possam entender rapidamente a finalidade e o conteúdo do cofre
- **AGENTS.md** é um guia escrito especificamente para assistentes de IA, explicando como lidar com o conteúdo do cofre

Organizada dessa forma, talvez a IA possa entender o que você tem em mente com um pouco mais de facilidade.

## Inicialização automática para vaults gerenciados pelo sistema

Além dos cofres criados manualmente, o HagiCode também suporta cofres gerenciados pelo sistema:

```csharp
public async Task<IReadOnlyList<VaultRegistryEntry>> EnsureAllSystemManagedVaultsAsync(
    CancellationToken cancellationToken = default)
{
    var definitions = GetAllResolvedDefinitions();
    var entries = new List<VaultRegistryEntry>(definitions.Count);

    foreach (var definition in definitions)
    {
        entries.Add(await EnsureResolvedSystemManagedVaultAsync(definition, cancellationToken));
    }

    return entries;
}
```

O sistema cria e gerencia automaticamente os seguintes cofres:

- **hagiprojectdata**: armazenamento de dados do projeto usado para salvar a configuração e o estado do projeto
- **dados pessoais**: armazenamento de dados pessoais usado para salvar as preferências do usuário
- **hbsprompt**: uma biblioteca de modelos de prompt usada para gerenciar prompts de IA comumente usados

Esses vaults são inicializados automaticamente quando o sistema é iniciado, portanto os usuários não precisam configurá-los manualmente. É simplesmente melhor deixar algumas coisas para o sistema, em vez de os humanos se preocuparem com elas.

## Mecanismo de controle de acesso

Uma parte importante do projeto é o controle de acesso. O sistema divide os cofres em dois tipos de acesso:

```typescript
export interface VaultForText {
  id: string;
  name: string;
  type: string;
  physicalPath: string;
  accessType: 'read' | 'write';  // Key: distinguish read-only from editable
}
```

- **referência (somente leitura)**: a IA é usada apenas para análise e compreensão e não pode modificar o conteúdo. Adequado para projetos de código aberto referenciados, documentos e materiais semelhantes
- **editável (editável)**: a IA pode modificar o conteúdo conforme necessário para a tarefa. Adequado para suas anotações, rascunhos e materiais semelhantes

Essa distinção é importante. Ele informa à IA qual conteúdo é “referência somente leitura” e qual conteúdo é “seguro para edição”, reduzindo o risco de alterações acidentais. Afinal, ninguém quer que seu trabalho árduo desapareça devido a uma edição não intencional.

## Na prática: criando e usando o Vault

Agora que cobrimos as ideias, vamos ver como isso funciona na prática.

### Crie um cofre CodeRef

Aqui está um exemplo completo de chamada de front-end:

```typescript
const createCodeRefVault = async () => {
  const response = await VaultService.postApiVaults({
    requestBody: {
      name: "React Learning Vault",
      type: "coderef",
      physicalPath: "/Users/developer/vaults/react-learning",
      gitUrl: "https://github.com/facebook/react.git"
    }
  });

  // The system will automatically:
  // 1. Clone the React repository into vault/repos/react
  // 2. Create the docs/ directory for notes
  // 3. Generate the index.yaml metadata
  // 4. Create the AGENTS.md guide file

  return response;
};
```

Esta chamada de API conclui uma série de ações: criação da estrutura de diretórios, inicialização de submódulos Git, geração de arquivos de metadados e muito mais. Você só precisa fornecer as informações básicas e deixar o sistema cuidar do resto. Honestamente, é uma abordagem bastante livre de preocupações.

### Use o Vault em uma proposta de IA

Depois de criar o cofre, você poderá referenciá-lo em uma proposta de IA:

```typescript
const proposal = composeProposalChiefComplaint({
  chiefComplaint: "Help me analyze React's concurrent rendering mechanism",
  repositories: [
    { id: "react", gitUrl: "https://github.com/facebook/react.git" }

  vaults: [
    {
      id: "react-learning",
      name: "React Learning Vault",
      type: "coderef",

      accessType: "read"  // AI can only read, not modify
    }
  ],
  quickRequestText: "Focus on the Fiber architecture and scheduler implementation"
});
```

O sistema injeta automaticamente informações do cofre no contexto da IA, informando à IA quais recursos de aprendizagem estão disponíveis. Quando a IA consegue entender o que você tem em mente, é difícil conseguir esse tipo de compreensão tácita.

## Melhores práticas e itens a serem observados

Ao usar o sistema Vault, resumimos algumas lições aprendidas.

### Segurança do caminho

O sistema valida estritamente os caminhos para evitar ataques de passagem de caminho:

```csharp
private static string ResolveFilePath(string vaultRoot, string relativePath)
{
    var rootPath = EnsureTrailingSeparator(Path.GetFullPath(vaultRoot));
    var combinedPath = Path.GetFullPath(Path.Combine(rootPath, relativePath));
    if (!combinedPath.StartsWith(rootPath, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode,
            "Vault file paths must stay inside the registered vault root.");
    }
    return combinedPath;
}
```

Isto é importante. Se você personalizar um caminho do vault, certifique-se de que ele permaneça dentro do intervalo permitido, caso contrário o sistema rejeitará a operação. Você realmente não pode enfatizar demais a segurança.

### Gerenciamento de submódulos Git

CodeRef Vault recomenda submódulos Git em vez de copiar código diretamente:

```csharp
private static string BuildCodeRefAgentsContent()
{
    return """
    # CodeRef Vault Guide

    Repositories under `repos/` should be maintained through Git submodules
    rather than copied directly into the vault root.

    Keep this structure stable so assistants and tools can understand the vault quickly.
    """ + Environment.NewLine;
}
```

Isso traz diversas vantagens: manter o código sincronizado com o upstream, economizar espaço em disco e facilitar o gerenciamento de múltiplas versões do código. Afinal, quem quer baixar a mesma coisa repetidas vezes?

### Limites de visualização de arquivos

Para evitar problemas de desempenho, o sistema limita o tamanho e o tipo de arquivo:

```csharp
private const int FileEnumerationLimit = 500;
private const int PreviewByteLimit = 256 * 1024;  // 256KB
```

Se o seu vault contiver um grande número de arquivos ou arquivos muito grandes, o desempenho da visualização poderá ser afetado. Nesse caso, você pode considerar o processamento de arquivos em lotes ou o uso de ferramentas de pesquisa especializadas. Às vezes, quando algo fica muito grande, fica mais difícil de manusear, e não mais fácil.

### Informações de diagnóstico

Ao criar um cofre, o sistema retorna informações de diagnóstico para ajudar na depuração:

```csharp
List<VaultBootstrapDiagnosticDto> bootstrapDiagnostics = [];

if (IsCodeRefVaultType(normalizedType))
{
    bootstrapDiagnostics = await EnsureCodeRefBootstrapAsync(
        normalizedName,
        normalizedPhysicalPath,
        normalizedGitUrl,
        cancellationToken);
}
```

Se a criação falhar, você poderá inspecionar as informações de diagnóstico para entender a causa específica. Quando algo dá errado, verificar o diagnóstico costuma ser o caminho mais direto a seguir.

## Resumo

Por meio de uma camada de abstração de armazenamento unificada, o sistema Vault resolve vários pontos problemáticos da reprodução de projetos na era da IA:

- **Gerenciamento de conhecimento centralizado**: todos os recursos de aprendizagem são reunidos em um só lugar, em vez de espalhados por todos os lugares
- **Injeção automática de contexto de IA**: os assistentes de IA podem compreender automaticamente os recursos de aprendizagem disponíveis sem configuração manual de contexto
- **Reutilização do conhecimento entre projetos**: o conhecimento pode ser compartilhado e reutilizado em vários projetos de aprendizagem
- **Estrutura de diretórios padronizada**: um layout de diretório consistente reduz a curva de aprendizado

Esta solução já foi validada no projeto HagiCode. Se você também está construindo ferramentas relacionadas ao desenvolvimento assistido por IA ou enfrentando problemas semelhantes de gestão do conhecimento, espero que essas experiências possam servir como uma referência útil.

Na verdade, o valor de uma solução técnica não reside na sua complexidade, mas sim na capacidade de resolver problemas reais. A ideia central do sistema Vault é muito simples: construir uma camada unificada de armazenamento de conhecimento que a IA possa entender. No entanto, foi precisamente esta simples abstração que melhorou bastante a nossa eficiência de desenvolvimento.

Às vezes, a abordagem simples é realmente a melhor. Afinal, coisas complicadas muitas vezes escondem ainda mais armadilhas...
