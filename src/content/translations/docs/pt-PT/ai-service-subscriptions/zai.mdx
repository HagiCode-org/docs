## Posicionamento

ZAI é um dos provedores atualmente recomendados pela predefinição de documentos definida para Claude Code. GLM Coding planeja mapeamento de camadas para desenvolvedores que desejam Claude Code como driver diário.

## Bom ajuste para

- Você quer um plano de assinatura focado em codificação
- Você precisa de padrões mapeados de Sonnet/Opus/Haiku
- Você espera que um provedor funcione com Claude Code e ferramentas de codificação semelhantes

## Coisas importantes a serem observadas

- Os metadados predefinidos atuais marcam o ZAI como recomendado.
- Ele expõe um endpoint compatível com Antrópico que pode ser usado diretamente por Claude Code.
- A predefinição envia mapeamentos padrão para `glm-4.7`, `glm-5`e `glm-4.5-air`.

## Instantâneo de integração

| Artigo | Valor |
|------|-------|
| Entrada de assinatura | [Abrir página de assinatura](https://www.bigmodel.cn/claude-code?ic=14BY54APZA) |
| Documentos/console oficiais | [Abrir página oficial](https://open.bigmodel.cn/dev/api) |
| URL base compatível com Antrópico | `https://open.bigmodel.cn/api/anthropic` |
| Var ambiente de autenticação recomendado | `ANTHROPIC_AUTH_TOKEN` |

## Mapeamento de modelo padrão

| Nível Cláudio | Modelo padrão |
|-------------|---------------|
| Soneto | `glm-4.7` |
| obra | `glm-5` |
| Haiku | `glm-4.5-air` |

:::dica
Preços, promoções e detalhes do pacote podem mudar com o tempo. Se o custo for sua principal preocupação, confirme as informações mais recentes diretamente na página do fornecedor.
:::

## Próximas etapas sugeridas

1. Confirme qual nível de assinatura ou caminho de ativação de API você deseja usar.
2. Abra a entrada do Claude Code no hub CLI unificado do AI Agent e continue com os documentos oficiais de instalação e referência da Anthropic.
3. Valide a conexão localmente após a configuração.
