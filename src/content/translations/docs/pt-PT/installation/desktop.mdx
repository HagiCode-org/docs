Este guia aborda a instalação e o uso do Hagicode Desktop. É o cliente de desktop oficial e mantém o caminho de configuração local padrão e os controles diários em um só lugar.

:::dica[Método recomendado]
Desktop é o caminho local padrão para usuários e desenvolvedores individuais porque:
- o instalador mantém o fluxo padrão em um caminho guiado
- dependências e serviços são gerenciados no aplicativo
- a IU torna o estado e as operações mais fáceis de inspecionar
- o gerenciamento de versão integrado facilita a troca
- os dados permanecem locais por padrão
:::

## O que é Hagicode Desktop?

Hagicode Desktop é o aplicativo de desktop oficial do Hagicode, projetado para sistemas Windows, macOS e Linux. Ele reúne os principais recursos do Hagicode e o instalador mantém o caminho de configuração padrão em um fluxo guiado.

### Principais vantagens

- **Instalação com um clique**: Conclua todas as etapas de configuração automaticamente por meio do assistente de instalação
- **Preparação automática do tempo de execução**: o instalador padrão já inclui o tempo de execução necessário, portanto, você não precisa instalar um programa separado. `.NET` tempo de execução primeiro
- **Gerenciamento de serviços**: Gerenciamento integrado do servidor Hagicode, inicia e interrompe serviços facilmente
- **Troca de versão**: suporte para gerenciamento de várias versões, alterne livremente entre diferentes versões
- **Operação Local**: Todos os dados são armazenados localmente, protegendo sua privacidade
- **Atualizações automáticas**: detecta e baixa automaticamente a versão mais recente

### Cenários Aplicáveis

O desktop é uma boa opção quando:
- Desenvolvedores individuais que desejam usar o Hagicode localmente
- Usuários que precisam usar o Hagicode offline
- Usuários não técnicos que desejam simplificar o processo de instalação
- Desenvolvedores que precisam alternar frequentemente entre diferentes versões

### Escolha a versão correta

Selecione o instalador correspondente ao seu sistema operacional:
- **Windows**: Baixe o `.exe` instalador
- **macOS**: Baixe o `.dmg` instalador
- **Linux**: Baixe o `.AppImage` ou `.tar.gz` pacote de instalação

:::nota[Requisitos do Sistema]
- **Windows**: Windows 10 ou versão superior (64 bits)
- **macOS**: macOS 10.15 ou superior
- **Linux**: Distribuições Linux convencionais (Ubuntu 20.04+, Fedora 33+, etc.)
:::

:::dica[Pré-requisitos padrão]
Para o fluxo de instalação local mais recente do HagiCode, normalmente não é necessário preparar um arquivo separado `.NET` tempo de execução. O Desktop já agrupa o tempo de execução necessário para o fluxo padrão e inicializa o caminho SQLite local compatível por padrão.
:::

## Notas de instalação e lançamento

:::cuidado[Aviso do macOS Gatekeeper]
Se você já instalou o aplicativo no caminho padrão `/Applications/Hagicode.app` mas o macOS mostra "o aplicativo está danificado", "não pode ser aberto" ou um aviso semelhante do Gatekeeper na primeira inicialização, a causa comum é o `com.apple.quarantine` atributo que o macOS adiciona aos aplicativos baixados da Internet, e não uma falha no próprio instalador do Hagicode Desktop.

Execute o seguinte comando no Terminal. Requer privilégios de administrador e remove recursivamente o atributo de quarentena do aplicativo no caminho de instalação padrão:

```bash
sudo xattr -dr com.apple.quarantine /Applications/Hagicode.app
```

Após a conclusão do comando, reinicie o Hagicode Desktop.
:::

## Execute a área de trabalho

Após o download, abra o instalador ou pacote do aplicativo e siga as instruções na tela. O Desktop prepara o tempo de execução no caminho local atualmente suportado, gerencia o Hagicode Server e mostra o progresso e o status quando são necessários downloads de rede.

:::nota[Requisitos de rede]
Certifique-se de que sua conexão de rede esteja disponível. O Desktop pode precisar buscar informações de versão ou pacotes de componentes na Internet; se o download falhar, verifique as configurações de rede, proxy ou firewall e tente novamente.
:::

## Gerenciamento de versão

Hagicode Desktop oferece suporte ao gerenciamento de várias versões, permitindo alternar livremente entre diferentes versões.

![Gerenciamento de versão](../../img/screenshots/installation/install-desktop/12-version-management/original.png)

### Mudar versão

1. Abra o Hagicode Desktop
2. Vá para a página "Gerenciamento de versões"
3. Selecione a versão que deseja mudar
4. Clique no botão "Alterar versão"
5. Aguarde a conclusão da mudança

:::nota[Notas de troca de versão]
- A troca de versões reiniciará automaticamente o Hagicode Server
- Certifique-se de salvar todo o trabalho antes de mudar
- As configurações do projeto podem diferir entre as versões
:::

## Iniciar e parar o serviço

Hagicode Desktop fornece uma maneira simples de gerenciar o serviço Hagicode Server.

### Iniciar serviço

Na página inicial do aplicativo Desktop, clique no botão "Iniciar serviço" para iniciar o Hagicode Server.

![Iniciar serviço](../../img/screenshots/installation/install-desktop/13-start-service-homepage/original.png)

### Parar serviço

Para parar o servidor Hagicode, clique no botão "Parar serviço" na página inicial.

:::dica[Status do serviço]
O aplicativo Desktop exibe o status de execução do Hagicode Server em tempo real, incluindo:
- Status do serviço (em execução/parado)
- Número da porta de serviço
- Informações da versão atual
:::

## Solução de problemas

### Falha na instalação

Se você encontrar problemas durante a instalação:

1. **Verifique a conexão de rede**: verifique se você consegue acessar a Internet
2. **Verifique o espaço em disco**: certifique-se de ter espaço em disco suficiente
3. **Executar como Administrador**: No Windows, clique com o botão direito no instalador e selecione "Executar como administrador"
4. **Visualizar arquivos de log**: o instalador gera arquivos de log detalhados que podem ajudar a diagnosticar problemas

### Falha na instalação de dependência

Se a instalação da dependência falhar:

1. **Verifique a compatibilidade do sistema**: certifique-se de que seu sistema operacional atenda aos requisitos mínimos
2. **Verifique as configurações de rede e firewall**: certifique-se de que o instalador possa alcançar os downloads necessários e não esteja bloqueado pelo seu firewall
3. **Confirme as permissões do instalador**: No Windows, tente executar o instalador como administrador para que ele tenha permissões suficientes
4. **Reinicie o aplicativo e tente novamente**: feche e reabra o Hagicode Desktop e tente novamente a etapa de instalação
5. **Visualizar registros**: revise registros de erros detalhados no aplicativo Desktop

### Falha na inicialização do serviço

Se o servidor Hagicode falhar ao iniciar:

1. **Verifique a ocupação da porta**: certifique-se de que a porta 45000 não esteja ocupada por outros programas
2. **Verifique as configurações do firewall**: certifique-se de que o firewall permite acesso à rede do servidor Hagicode
3. **Reiniciar aplicativo**: tente reiniciar o Hagicode Desktop
4. **Ver logs**: veja logs de erros detalhados no aplicativo Desktop
