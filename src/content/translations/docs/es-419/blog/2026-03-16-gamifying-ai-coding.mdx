## Cómo el diseño de gamificación hace que la codificación con IA sea más divertida

> Las herramientas tradicionales de codificación de IA son bastante poderosas; simplemente les falta un poco de calidez. Cuando estábamos construyendo HagiCode, pensamos: si vamos a escribir código de todos modos, ¿por qué no convertirlo en un juego?

## Antecedentes

Cualquiera que haya utilizado un asistente de codificación con IA probablemente haya tenido esta experiencia: al principio se siente fresco y emocionante, pero después de un tiempo comienza a sentir que falta algo. La herramienta en sí es poderosa, capaz de generar código, autocompletar y corregir errores, pero... no se siente muy cálida y, con el tiempo, puede volverse monótona y aburrida.

Eso por sí solo es suficiente para hacerte preguntarte quién quiere mirar fijamente una herramienta fría e impersonal todos los días.

Es un poco como jugar un juego. Si todo lo que haces es terminar una lista de tareas, sin crecimiento del personaje, sin desbloqueo de logros y sin coordinación del equipo, rápidamente deja de ser divertido. No es necesario poseer las cosas y las personas hermosas para apreciarlas; su belleza es suficiente por sí sola. Las herramientas de programación ni siquiera ofrecen ese tipo de belleza, por lo que es fácil desanimarse.

Nos topamos exactamente con este problema mientras desarrollábamos [HagiCode](https://hagicode.com). Como plataforma de colaboración de asistentes de inteligencia artificial múltiple, HagiCode necesita mantener a los usuarios interesados ​​a largo plazo. Pero en realidad, incluso una gran herramienta es difícil de seguir si carece de conexión emocional.

Para resolver este problema, tomamos una decisión audaz: convertir la programación en un juego. No del tipo superficial con una simple tabla de clasificación de puntos, sino una verdadera experiencia gamificada de juego de rol. El impacto de esa decisión puede ser incluso mayor de lo que imagina.

Después de todo, la gente necesita un poco de ritual en sus vidas.

## Acerca de HagiCode

Las ideas compartidas en este artículo provienen de nuestra experiencia práctica en el [HagiCode](https://hagicode.com) proyecto. HagiCode es una plataforma de colaboración de múltiples asistentes de IA que admite que Claude Code, Codex, Copilot, OpenCode y otros asistentes de IA trabajen juntos. Si está interesado en la colaboración de múltiples IA o la programación gamificada, visite [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site) para aprender más.

No hay nada especialmente misterioso en ello. Simplemente convertimos la programación en una aventura.

## Por qué elegir la gamificación

La esencia de la gamificación no es sólo "añadir una tabla de clasificación". Se trata de construir un sistema de incentivos completo para que los usuarios puedan sentir crecimiento, logros y reconocimiento social mientras realizan tareas.

El diseño de gamificación de HagiCode gira en torno a una idea central: cada asistente de IA es un "héroe" y el usuario es el capitán de este equipo de héroes. Diriges a estos héroes a conquistar varias "mazmorras" (tareas de programación). En el camino, los héroes ganan experiencia, suben de nivel, desbloquean habilidades y tu equipo también obtiene logros.

Esto no es un truco. Es un diseño basado en la psicología del comportamiento humano. Cuando a las tareas se les da significado y se les retroalimenta el progreso, el compromiso y la perseverancia de las personas aumentan significativamente.

Como dice el viejo refrán: "Este sentimiento puede convertirse en un recuerdo, aunque en su momento nos haya dejado desconcertados". Incorporamos esa experiencia emocional a la herramienta, por lo que programar ya no es solo escribir código, sino un viaje que vale la pena recordar.

## Sistema de personajes héroe

Hero es el concepto central del sistema de gamificación de HagiCode. Cada héroe representa un asistente de IA. Por ejemplo, Claude Code es un héroe y Codex también es un héroe.

### Las tragamonedas de los tres héroes

Un héroe tiene tres espacios para equipos y el diseño es sorprendentemente elegante:

1. **Ranura CLI (clase principal)**: determina la habilidad base del héroe, por ejemplo, si es Claude Code o Codex.
2. **Ranura de modelo (clase secundaria)**: determina qué modelo se utiliza, como Claude 4.5 o Claude 4.6
3. **Ranura de estilo (estilo)**: determina el estilo de comportamiento del héroe, como "Fengluo Strategist" u otro estilo.

La combinación de estas tres ranuras crea configuraciones de héroe únicas. Al igual que el equipo integrado en los juegos, usted elige la configuración adecuada según la tarea. Al fin y al cabo, lo que más te conviene es lo que más importa. La vida es similar: muchos caminos conducen a Roma, pero algunos son más sencillos que otros.

### El sistema de crecimiento del héroe

Cada héroe tiene su propia XP y nivel:

```typescript
type HeroProgressionSnapshot = {
  currentLevel: number;                    // Current level
  totalExperience: number;                 // Total experience
  currentLevelStartExperience: number;     // Experience at the start of the current level
  nextLevelExperience: number;             // Experience required for the next level
  experienceProgressPercent: number;       // Progress percentage
  remainingExperienceToNextLevel: number;  // Experience still needed for the next level
  lastExperienceGain: number;              // Most recent experience gained
  lastExperienceGainAtUtc?: string | null; // Time when experience was gained
};
```

Los niveles se dividen en cuatro etapas y cada etapa tiene un nombre inmersivo:

```typescript
export const resolveHeroProgressionStage = (level?: number | null): HeroProgressionStage => {
  const normalizedLevel = Math.max(1, level ?? 1);
  if (normalizedLevel <= 100) return 'rookieSprint';      // Rookie sprint
  if (normalizedLevel <= 300) return 'growthRun';         // Growth run
  if (normalizedLevel <= 700) return 'veteranClimb';      // Veteran climb
  return 'legendMarathon';                                // Legend marathon
};
```

De "novato" a "leyenda", este camino de crecimiento brinda a los usuarios un claro sentido de dirección y logro. Refleja el crecimiento personal en la vida, desde la confusión hasta la madurez, sólo que aquí se hace más tangible.

### Creando un héroe personalizado

Para crear un héroe, necesitas configurar tres espacios:

```typescript
const heroDraft: HeroDraft = {
  name: 'Athena',
  icon: 'hero-avatar:storm-03',
  description: 'A brilliant strategist',
  executorType: AIProviderType.CLAUDE_CODE_CLI,
  slots: {
    cli: {
      id: 'profession-claude-code',
      parameters: { /* CLI-related parameters */ }
    },
    model: {
      id: 'secondary-claude-4-sonnet',
      parameters: { /* Model-related parameters */ }
    },
    style: {
      id: 'fengluo-strategist',
      parameters: { /* Style-related parameters */ }
    }
  }

```

Cada héroe tiene un avatar, una descripción y una identidad profesional únicos, lo que le da más personalidad y calidez a lo que de otro modo sería un frío asistente de IA. Al fin y al cabo, ¿quién quiere trabajar con una herramienta que no tiene carácter?

## Sistema de mazmorras

Una "Mazmorra" es un concepto de juego clásico que representa un desafío que requiere un equipo para superarlo. En HagiCode, cada flujo de trabajo es una mazmorra.

### Cómo se organizan las mazmorras

Dungeon organiza los flujos de trabajo en diferentes "Mazmorras":

- **Mazmorra de generación de propuestas**: Responsable de generar propuestas técnicas
- **Mazmorra de ejecución de propuestas**: Responsable de ejecutar tareas en propuestas
- **Mazmorra de archivo de propuestas**: Responsable de organizar y archivar las propuestas completadas.

Cada mazmorra tiene su propio Capitán Héroe, y el capitán se elige automáticamente como el primer Héroe habilitado.

En realidad, esto es solo división del trabajo, como en la vida cotidiana, excepto que se ha convertido en una mecánica de juego.

### Mecanismo de colaboración en equipo

Puedes configurar diferentes escuadrones de héroes para diferentes mazmorras:

```typescript
const dungeonRoster: HeroDungeonRoster = {
  scriptKey: 'proposal.generate',
  displayName: 'Proposal Generation',
  members: [
    { heroId: 'hero-1', name: 'Athena', executorType: 'ClaudeCode' },
    { heroId: 'hero-2', name: 'Apollo', executorType: 'Codex' }
  ]
};
```

Por ejemplo, puede utilizar Athena para generar propuestas porque es bueno en estrategia y Apollo para implementar código porque es bueno en ejecución. De esa manera, cada héroe puede aprovechar sus puntos fuertes. Es como formar una banda: cada persona tiene un instrumento y juntos crean algo hermoso.

### Control de flujo de mazmorra

Usos de mazmorra fijos. `scriptKey` valores para identificar diferentes flujos de trabajo:

```typescript
// Script keys map to different workflows
const dungeonScripts = {
  'proposal.generate': 'Proposal Generation',
  'proposal.execute': 'Proposal Execution',
  'proposal.archive': 'Proposal Archive'
};
```

El flujo de estado de la tarea es: `queued` (esperando) -> `dispatching` (siendo asignado) -> `dispatched` (asignado). Todo el proceso está automatizado y no requiere intervención manual. Eso también forma parte de nuestro lado vago, porque ¿quién quiere gestionar estas cosas a mano?

## XP y sistema de niveles

XP es el mecanismo central de retroalimentación en el sistema de gamificación. Los usuarios obtienen XP al completar tareas, XP sube de nivel a los héroes y al subir de nivel se desbloquean nuevas habilidades, formando un ciclo de retroalimentación positiva.

### Formas de ganar XP

En HagiCode, XP se puede obtener a través de las siguientes actividades:

- Completar la ejecución del código
- Herramientas de llamada exitosas
- Generando propuestas
- Operaciones de gestión de sesiones
- Operaciones del proyecto

Cada vez que se completa una acción válida, el héroe correspondiente gana XP. Al igual que el crecimiento en la vida, cada paso cuenta, sólo que aquí se cuantifica ese crecimiento.

### Visualización del progreso en tiempo real

La XP y el progreso de nivel se visualizan en tiempo real:

```typescript
type HeroDungeonMember = {
  heroId: string;
  name: string;
  icon?: string | null;
  executorType: PCode_Models_AIProviderType;
  currentLevel?: number;                    // Current level
  totalExperience?: number;                 // Total experience
  experienceProgressPercent?: number;       // Progress percentage
};
```

Los usuarios siempre pueden ver el nivel y el progreso de cada héroe, y esa retroalimentación inmediata es la clave para el diseño de gamificación. La gente necesita retroalimentación; de lo contrario, ¿cómo sabrían que están mejorando?

## Sistema de logros

Los logros son otro elemento importante en la gamificación. Proporcionan objetivos a largo plazo y satisfacción basada en hitos.

### Tipos de logros

HagiCode admite múltiples tipos de logros:

- **Logros de generación de código**: Generar X líneas de código, generar archivos Y
- **Logros de la gestión de sesiones**: completar conversaciones Z
- **Logros de la operación del proyecto**: Trabajo en todos los proyectos W

Estos logros son realmente como hitos en la vida, excepto que los hemos convertido en una mecánica de juego.

### Estados de logro

Los logros tienen tres estados:

```typescript
type AchievementStatus = 'unlocked' | 'in-progress' | 'locked';
```

Los tres estados tienen claras distinciones visuales:

- **Desbloqueado**: Degradado dorado con efecto halo
- **En progreso**: Animación de pulso azul
- **Bloqueado**: Gris, se muestran las condiciones de desbloqueo

Cada logro muestra claramente su condición de activación, para que los usuarios sepan qué hacer a continuación. Cuando la gente se siente perdida, un poco de orientación siempre ayuda.

### Efecto de celebración al desbloquear

Cuando se desbloquea un logro, se activa una animación de celebración. Ese tipo de refuerzo positivo brinda a los usuarios la sensación satisfactoria de "lo logré" y los motiva a seguir adelante. Las pequeñas recompensas en la vida funcionan de la misma manera: pueden ser pequeñas, pero la felicidad puede durar mucho tiempo.

## Informe de batalla Informe de combate diario

Battle Report es una de las características distintivas de HagiCode. Al final de cada día, genera un informe estilo batalla en pantalla completa.

### Informar contenido

El Informe de batalla muestra la siguiente información:

```typescript
type HeroBattleReport = {
  reportDate: string;
  summary: {
    totalHeroCount: number;        // Total number of Heroes
    activeHeroCount: number;       // Number of active Heroes
    totalBattleScore: number;      // Total battle score
    mvp: HeroBattleHero;           // Most valuable Hero
  };
  heroes: HeroBattleHero[];        // Detailed data for all Heroes
};
```

- Puntuación total del equipo
- Número de héroes activos
- Número de llamadas a herramientas
- tiempo total de trabajo
- MVP (héroe más valioso)
- Tarjeta detallada para cada héroe.

### Pantalla destacada de MVP

El MVP es el héroe con mejor desempeño del día y se destaca en el informe. No se trata sólo de datos estadísticos, sino de una forma de honor y reconocimiento. Después de todo, ¿quién no quiere ser reconocido?

### Cartas de héroe detalladas

Cada carta de Héroe incluye:

- Progreso de nivel
- XP ganado
- Número de ejecuciones
- tiempo de uso

Estas métricas ayudan a los usuarios a comprender claramente cómo se está desempeñando el equipo. Ver los resultados de tu propio esfuerzo es satisfactorio en sí mismo.

## Implementación técnica

El sistema de gamificación de HagiCode utiliza una pila de tecnología y patrones de diseño modernos. No tiene nada de mágico; simplemente elegimos herramientas que se adaptan al trabajo.

### Opciones de pila de tecnología

```typescript
// React + TypeScript for the frontend
import React from 'react';

// Framer Motion for animations
import { AnimatePresence, motion } from 'framer-motion';

// Redux Toolkit for state management
import { useAppDispatch, useAppSelector } from '@/store';

// shadcn/ui for UI components
import { Dialog, DialogContent } from '@/components/ui/dialog';
```

Framer Motion maneja todos los efectos de animación, shadcn/ui proporciona los componentes fundamentales de la interfaz de usuario y Redux Toolkit administra el complejo estado de gamificación. Las buenas herramientas hacen un buen trabajo.

### Sistema de diseño de interfaz de usuario gamificado

HagiCode utiliza un estilo de diseño Glassmorphism + Tech Dark:

```css
/* Primary gradient */
background: linear-gradient(135deg, #22C55E 0%, #25c2a0 50%, #06b6d4 100%);

/* Glass effect */
backdrop-filter: blur(12px);

/* Glow effect */
background: radial-gradient(circle at center, rgba(34, 197, 94, 0.15) 0%, transparent 70%);
```

El degradado verde combinado con el morfismo del vidrio crea una atmósfera técnica y futurista. La belleza visual también forma parte de la experiencia del usuario.

### Efectos de animación

Framer Motion se utiliza para crear animaciones de entrada fluidas:

```typescript
<motion.div
  animate={{ opacity: 1, y: 0 }}
  initial={{ opacity: 0, y: 18 }}
  transition={{ duration: 0.35, ease: 'easeOut', delay: index * 0.08 }}
  className="card"
>
  {/* Card content */}
</motion.div>
```

Cada tarjeta ingresa una tras otra con un retraso de 0,08 segundos, creando un efecto visual fluido. La animación fluida mejora la experiencia. Es difícil discutir esa parte.

### Persistencia de datos

Los datos de gamificación se almacenan utilizando el sistema de almacenamiento Grain para garantizar la coherencia del estado. Incluso los datos más detallados, como Hero XP acumulado, se pueden conservar con precisión. Nadie quiere perder la experiencia que tanto trabajó para ganar.

## Guía práctica

### Crea tu primer héroe

Crear tu primer héroe es bastante simple:

1. Ir a la página de gestión de héroes
2. Haga clic en el botón "Crear héroe"
3. Configure las tres ranuras (CLI, Modelo, Estilo)
4. Dale al héroe un nombre y una descripción.
5. Guárdalo y nacerá tu primer héroe.

Es como conocer a un nuevo amigo: le das un nombre, aprendes qué lo hace especial y luego emprendes una aventura juntos.

### Construye un equipo de mazmorra

Formar un equipo también es sencillo:

1. Ir a la página de gestión de mazmorras.
2. Elige la mazmorra que deseas configurar, como por ejemplo "Generación de propuestas"
3. Selecciona miembros de tu lista de héroes
4. El sistema selecciona automáticamente al primer héroe habilitado como Capitán.
5. Guardar la configuración

Este es simplemente el proceso de formar un equipo, muy parecido a construir un equipo en la vida real donde cada uno tiene su propio rol.

### Ver el informe diario

Al final de cada día, puedes ver el Informe de batalla del día:

1. Haga clic en el botón "Informe de batalla"
2. Vea los resultados del trabajo del día en una visualización de pantalla completa
3. Consulta el MVP y los datos detallados de cada héroe.
4. Compártelo con los miembros del equipo si quieres.

Esto también es una especie de ritual, una forma de ver cuánto esfuerzo has puesto hoy y qué tan lejos estás aún de tu objetivo.

## Notas y mejores prácticas

### Optimización del rendimiento

uso `React.memo` Para evitar re-renderizaciones innecesarias:

```typescript
const HeroCard = React.memo(({ hero }: { hero: HeroDungeonMember }) => {
  // Component implementation
});
```

El rendimiento también importa. Nadie quiere utilizar una herramienta lenta.

### El movimiento puede degradarse con gracia

Detecta la configuración de preferencias de movimiento del usuario y brinda una experiencia simplificada para los usuarios sensibles al movimiento:

```typescript
const prefersReducedMotion = useReducedMotion();
const duration = prefersReducedMotion ? 0 : 0.35;
```

No a todo el mundo le gusta la animación y respetar las preferencias del usuario forma parte de un buen diseño.

### Compatibilidad con versiones anteriores

mantener `legacyIds` para admitir la migración desde versiones anteriores:

```typescript
type HeroDungeonMember = {
  heroId: string;
  legacyIds?: string[];  // Supports legacy ID mapping
  // ...
};
```

Nadie quiere perder datos sólo por una actualización de versión.

### Apoyo a la Internacionalización

Utilice las claves de traducción i18n para todo el texto para facilitar la compatibilidad con varios idiomas:

```typescript
const displayName = t(`dungeon.${scriptKey}`, { defaultValue: displayName });
```

El idioma nunca debe ser una barrera para el uso del producto.

## Resumen

La gamificación no es sólo una simple tabla de clasificación de puntos, sino un completo sistema de incentivos. A través del sistema de héroe, el sistema de mazmorra, el sistema de niveles y XP, el sistema de logros y el informe de batalla, HagiCode transforma el trabajo de programación en un viaje heroico lleno de aventuras.

El valor central de este sistema radica en:

- **Conexión emocional**: Dar personalidad a los fríos asistentes de IA
- **Retroalimentación positiva**: Cada acción produce retroalimentación inmediata
- **Objetivos a largo plazo**: los niveles y logros proporcionan un camino de crecimiento
- **Identidad del equipo**: un sentido de colaboración dentro de los equipos de Dungeon
- **Honor y reconocimiento**: Informe de batalla y presentaciones de MVP

El diseño de gamificación hace que la programación ya no sea aburrida, sino una aventura interesante. Mientras completan tareas de codificación, los usuarios también experimentan la diversión del crecimiento del personaje, la colaboración en equipo y el desbloqueo de logros, lo que mejora la retención y la actividad.

En esencia, la programación ya es un acto de creación. Simplemente hicimos que el proceso creativo fuera un poco más divertido.
