Si desea instalar Hermes y comenzar a usarlo, el camino más corto es en realidad solo tres pasos:

1. Ejecute el comando de instalación oficial.
2. Inicie la CLI en su terminal con `hermes`
3. Si desea seguir usándolo en Feishu, configure `hermes gateway setup`

Este artículo no intenta explicar todas las capacidades de Hermes de una sola vez. En cambio, primero le ayuda a completar el ciclo más importante para principiantes: instalarlo, ponerlo en funcionamiento, comenzar a usarlo y luego conectarlo a uno de los escenarios de plataformas de mensajería más comunes.

## Qué es Hermes y para quién es

Hermes Agent es un agente de inteligencia artificial que puede utilizar desde una terminal local o mediante una puerta de enlace de una plataforma de mensajería.

Para la mayoría de los desarrolladores, tiene dos puntos de entrada comunes:

- **CLI**: Tipo `hermes` en su terminal para ingresar directamente a la interfaz interactiva.
- **Puerta de enlace de mensajería**: Ejecutar `hermes gateway`, luego chatea con él desde plataformas como Feishu, Telegram, Discord y Slack.

Si su objetivo ahora es simplemente comenzar rápidamente, no invierta el orden. Comience con esta ruta en su lugar:

- Instale Hermes primero
- Primero verifique que funcione desde la CLI
- Luego decide si quieres conectar una plataforma de mensajería.

Esto facilita el diagnóstico de los problemas y es más adecuado para personas que utilizan Hermes por primera vez.

## Qué saber antes de instalar Hermes

Según Hermes README, la ruta oficial de instalación rápida admite estos entornos:

- linux
- macos
- WSL2
- Android a través de Termux

### Notas para usuarios de Windows

Actualmente, Hermes no admite la ejecución directa en Windows nativo. Si está utilizando Windows, el método recomendado es instalar WSL2 primero y luego ejecutar el comando de instalación dentro de WSL2.

Es mejor dejar esto claro al principio, porque muchos fallos de instalación no son causados por el comando en sí, sino por el uso de un entorno de ejecución no compatible.

## Cómo instalar Hermes

El comando de instalación rápida proporcionado en el archivo README de Hermes es:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Este comando ejecuta el script de instalación oficial y maneja los pasos de inicialización específicos de la plataforma.

### Vuelva a cargar su shell después de la instalación

Una vez que finalice la instalación, primero vuelva a cargar su entorno de shell. El comando más común es:

```bash
source ~/.bashrc
```

Si usas zsh, puedes usar:

```bash
source ~/.zshrc
```

### Cómo confirmar que Hermes está instalado correctamente

La forma más directa de comprobarlo es ejecutar:

```bash
hermes
```

Si desea confirmación adicional de que su configuración y dependencias están funcionando, también puede ejecutar:

```bash
hermes doctor
```

`hermes doctor` Es especialmente útil en estas situaciones:

- El comando se comporta de manera anormal después de la instalación.
- La configuración del modelo falla
- La puerta de enlace no se inicia
- No está seguro de si las dependencias de su entorno están completas

## Cómo empezar a utilizar Hermes por primera vez

### Comience con la CLI

Si sólo desea confirmar lo más rápido posible que Hermes funciona, el método más sencillo es:

```bash
hermes
```

Esto inicia la CLI interactiva de Hermes. Para los usuarios primerizos de Hermes, también es el punto de partida más recomendado, porque primero pueden verificar las cosas más esenciales:

- Si el comando está realmente disponible
- Si la configuración del modelo actual funciona correctamente
- Si la cadena de herramientas del terminal funciona correctamente
- Si el estilo de interacción coincide con lo que necesitas

### Estos comandos son suficientes para tu primera ronda de configuración.

El archivo README de Hermes enumera varios comandos de alta frecuencia y juntos forman una ruta práctica de primer uso:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Si no estás seguro de qué hace cada uno, recuérdalos así:

- `hermes model`: elegir o cambiar de modelo
- `hermes tools`: ver y configurar las herramientas disponibles actualmente
- `hermes config set`: cambiar elementos de configuración específicos
- `hermes setup`: ejecuta el asistente de inicialización completo una vez
- `hermes update`: actualizar Hermes
- `hermes doctor`: solucionar problemas

Para principiantes, el orden más práctico suele ser:

1. correr `hermes model` primero
2. Si desea configurar todas las opciones comunes a la vez, ejecute `hermes setup`

## Las dos formas más comunes de utilizar Hermes

### 1. Utilice Hermes en la terminal como asistente de desarrollo diario.

El modo CLI es una buena opción para estos escenarios:

- Haga preguntas directamente mientras escribe código localmente
- Inspeccionar proyectos, editar archivos y ejecutar comandos
- Realice una depuración o un trabajo de revisión únicos
- Colaborar continuamente en el directorio de trabajo actual.

Su mayor ventaja es que es el camino más corto: no hay integración de plataforma adicional, ni configuración de bot que manejar por adelantado, y es la mejor manera de desarrollar su primer conjunto de hábitos de uso.

### 2. Utilice Hermes a través de una plataforma de mensajería.

Si desea chatear con Hermes en plataformas como Feishu, Telegram o Discord, debe utilizar la puerta de enlace de mensajería.

Los comandos de entrada más comunes son:

```bash
hermes gateway setup
hermes gateway
```

Específicamente:

- `hermes gateway setup` se utiliza para la configuración de la plataforma interactiva
- `hermes gateway` se utiliza para iniciar el proceso de puerta de enlace

Según la documentación oficial, la puerta de enlace es un proceso en segundo plano unificado que conecta sus plataformas configuradas, administra sesiones y maneja funciones como trabajos cron.

## Usando a Feishu como ejemplo: cómo conectar Hermes a una plataforma de mensajería

Si la mayor parte de tu trabajo diario se realiza en Feishu, entonces Feishu/Lark es una forma muy natural de utilizar Hermes.

### El camino mínimo viable de integración

La documentación oficial recomienda este comando de entrada para Feishu/Lark:

```bash
hermes gateway setup
```

Después de ejecutarlo, simplemente elija **Feishu / Lark** en el asistente.

La documentación de Feishu describe dos modos de conexión:

- `websocket`: recomendado
- `webhook`: opcional

Si Hermes se ejecuta en su computadora portátil, estación de trabajo o servidor privado, usando `websocket` El primero suele ser más sencillo porque no es necesario exponer una URL de devolución de llamada pública.

### Si lo configuras manualmente, al menos conoce estas variables.

Si no está utilizando el asistente y está escribiendo la configuración manualmente, la documentación de Feishu enumera estas variables principales:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Dos de ellos merecen especial atención:

- `FEISHU_ALLOWED_USERS`: recomendado, por lo que no todos los que pueden acceder al bot pueden usarlo directamente
- `FEISHU_HOME_CHANNEL`: le permite predefinir un chat de inicio para recibir resultados cron o notificaciones predeterminadas

### Por qué Hermes a veces no responde en los chats grupales de Feishu

Este detalle es fácil de pasar por alto: en los chats grupales de Feishu, Hermes no responde a todos los mensajes de forma predeterminada.

La documentación oficial dice claramente:

- En mensajes directos, Hermes responde a los mensajes.
- En los chats grupales, debes explícitamente `@` el bot antes de que procese el mensaje

Si deseas configurar una conversación de Feishu como canal de inicio, también puedes usar esto en el chat:

```text
/set-home
```

O defínelo en la configuración con anticipación:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Los comandos de Hermes que los principiantes deben recordar primero

Ya sea que utilice Hermes en la CLI o en una plataforma de mensajería, recordar los siguientes comandos es suficiente para comenzar:

- `/new` o `/reset`: iniciar una nueva sesión
- `/model`: ver o cambiar el modelo
- `/retry`: reintentar el turno anterior
- `/undo`: deshacer la interacción anterior
- `/compress`: comprime manualmente el contexto
- `/help`: ver ayuda

Si utilizas Hermes principalmente en una plataforma de mensajería, recuerda una más:

- `/sethome` o `/set-home`: establece el chat actual como canal de inicio

Estos comandos cubren las operaciones más comunes para principiantes: reiniciar, ajustar, retroceder, verificar y continuar.

## Preguntas frecuentes

### ¿Puede Windows instalar Hermes directamente?

No. La documentación oficial actual establece claramente que Windows nativo no es compatible y se recomienda WSL2.

### ¿Qué debo hacer si escribo? `hermes` ¿No hace nada después de la instalación?

Lo mejor es solucionar problemas en este orden:

1. Vuelva a cargar su shell primero, por ejemplo con `source ~/.bashrc`
2. correr `hermes` otra vez
3. Si todavía es anormal, ejecute `hermes doctor`

### ¿Por qué el bot no responde en un grupo Feishu?

Comprueba estas tres cosas primero:

- Ya sea que usted `@` mencionó a Hermes en el grupo
- Ya sea `FEISHU_ALLOWED_USERS` restringe el usuario actual
- Si la política actual de chat grupal permite manejar mensajes grupales

Según la documentación oficial de Feishu, se requiere el uso explícito de una @mención en escenarios de chat grupal.
