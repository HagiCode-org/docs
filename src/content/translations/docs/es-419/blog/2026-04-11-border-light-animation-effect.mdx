## Guía para crear un efecto de animación de barrido de luz de borde

> ¿Cómo se construye ese elemento importante que los usuarios notan de un vistazo usando CSS puro? En realidad no es tan difícil. El truco consiste simplemente en tomar un camino ligeramente indirecto. En este artículo, le explicaré cómo crear una animación de barrido de luz de borde desde cero y también compartiré algunos de los errores con los que nos topamos al crear HagiCode.

## Antecedentes

Si trabaja en el frontend, probablemente haya tenido esta experiencia antes: un gerente de producto se acerca con esa expresión "esto es definitivamente simple" y dice: "¿Podemos agregar algún tipo de efecto especial a esta tarea en ejecución para que los usuarios puedan detectarla de inmediato?"

Dices que claro, podemos cambiar el color del borde. Luego sacuden la cabeza con esa mirada que dice: "No lo entiendes". Ellos responden: "Eso no es lo suficientemente obvio. Quiero el tipo de efecto en el que la luz recorre el borde, como en una película de ciencia ficción".

En ese momento quizás empieces a preguntarte cómo construirlo. ¿Lienzo? ¿SVG? ¿O CSS puede manejarlo por sí solo? Después de todo, nadie quiere admitir que no sabe cómo hacerlo.

En las aplicaciones web modernas, las animaciones de barrido de luz de borde son muy comunes. Se utilizan principalmente en algunos escenarios como estos:

- **Indicadores de estado**: marcar tareas en progreso o elementos activos
- **Enfoque visual**: resaltar áreas de contenido importantes
- **Mejora de la marca**: creación de un estilo visual elegante, moderno y de vanguardia tecnológica
- **Temas de temporada**: creación de un ambiente de celebración para ocasiones especiales

Nos topamos exactamente con este requisito mientras construíamos [HagiCode](https://hagicode.com). Los usuarios necesitaban ver de un vistazo qué sesiones se estaban ejecutando y qué propuestas se estaban procesando actualmente. Probamos varios enfoques diferentes. Algunos caminos eran más suaves, otros un poco más sinuosos. Al final, nos decidimos por una estrategia de implementación bastante madura.

## Ideas básicas de implementación

Después de analizar el código base de HagiCode, resumimos varios patrones de implementación principales a continuación. Cada uno se ajusta a un escenario diferente, o en otras palabras, cada uno existe por una razón.

### 1. Resplandor giratorio con gradiente cónico (el más común)

Esta es la forma clásica de implementar un efecto de barrido de luz en los bordes. La idea central es utilizar CSS. `conic-gradient` para crear un degradado cónico, luego gírelo continuamente. Como una farola que gira en la noche, sigue dando vueltas.

**Elementos clave:**
- Utilice el `::before` pseudoelemento para crear la capa de brillo
- uso `conic-gradient` para definir la distribución del color del degradado
- Utilice el `::after` pseudoelemento para enmascarar el área central (opcional)
- uso `@keyframes` para implementar la animación de rotación

### 2. Línea de brillo lateral

Esto funciona bien para indicadores de estado en elementos de lista. Creas una delgada línea brillante en un lado del elemento en lugar de animar todo el borde. A veces un poco de luz es suficiente. No necesitas iluminar el mundo entero.

**Elementos clave:**
- Un elemento de línea delgado y absolutamente posicionado.
- uso `box-shadow` para crear el efecto de brillo
- uso `scale` y `opacity` para una animación de respiración

### 3. Fondo brillante con sombra de cuadro

Si no necesita el efecto de barrido completo y sólo desea un brillo de fondo suave, coloque varias capas `box-shadow` Los valores son suficientes. A veces la opción más sencilla es la mejor.

### 4. Soporte de accesibilidad

Esta parte es fácil de pasar por alto, pero es extremadamente importante. Cada animación debe tener en cuenta la `prefers-reduced-motion` consulta de medios y proporciona una alternativa estática para los usuarios que no desean animación. No todo el mundo disfruta del movimiento constante y respetar esa preferencia es importante.

## Opciones de implementación

### Opción 1: borde cónico-gradiente giratorio (recomendado)

Esta es la implementación más completa del efecto de luz de borde de barrido y también es la opción que usamos con más frecuencia en HagiCode. Después de todo, si algo funciona bien, ¿por qué reemplazarlo?

```css
/* Parent container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotating glow layer */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Mask layer (optional, for creating a hollow border effect) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

El principio detrás de esta opción es bastante simple: cree un pseudoelemento más grande que el contenedor principal, dibuje un degradado cónico sobre él y gírelo continuamente. El contenedor principal utiliza `overflow: hidden`, por lo que sólo permanece visible la luz que pasa alrededor del borde. Es un poco como mirar una farola a través de una ventana. Sólo se ve el pequeño trozo que pasa.

### Opción 2: borde de luz giratorio simplificado

Si no necesita el efecto completo, HagiCode también incluye una versión de utilidad más ligera. A veces, el enfoque más simple es realmente mejor.

```css
/* Rotating light border utility class */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Accessibility support */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Note el `will-change: transform` aquí. Le dice al navegador: "Este elemento seguirá cambiando", para que el navegador pueda preparar algunas optimizaciones con anticipación y mantener la animación más fluida. Prepararse con anticipación suele ser mejor que apresurarse en el último minuto.

### Opción 3: Línea de brillo lateral

Esto es especialmente adecuado para indicadores de estado de elementos de lista, y es exactamente lo que utiliza la lista de sesión de HagiCode. Una delgada línea aún puede destacarse entre muchos elementos. Eso se siente como una lección de vida a su manera.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Esto usa `isolation: isolate` para crear un nuevo contexto de apilamiento, luego se basa en `z-index` para controlar el orden de visualización de cada capa. `pointer-events: none` también es esencial. De lo contrario, el pseudoelemento bloquearía los clics del usuario. Algunas cosas pueden verse bien, pero aun así no deberían estorbar.

### Opción 4: envolverlo en un componente de React

Si su proyecto usa React, puede incluir esta lógica en un componente, especialmente el manejo de accesibilidad. Escríbelo una vez, úsalo muchas veces. Ese es el punto.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

El módulo CSS correspondiente:

```css
/* GlowBorder.module.css */

/* Animated version */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Static version (accessibility) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

el `useReducedMotion` gancho de `framer-motion` detecta automáticamente la preferencia del sistema del usuario. Si el usuario ha habilitado el movimiento reducido, regresa `true`y el componente muestra la versión estática en su lugar. Respetar las preferencias del usuario es más importante que forzar un efecto llamativo.

## Lecciones prácticas aprendidas

Estas son algunas de las lecciones que aprendimos mientras construíamos HagiCode. También podrías llamarlas cicatrices de batalla. Ojalá te ayuden a evitar algunos desvíos.

### 1. Sistema de variables temáticas

Las variables CSS hacen que la compatibilidad con múltiples temas sea especialmente conveniente. Nadie quiere editar un montón de código cada vez que cambia el tema.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Usage */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

De esa manera, cambiar de tema sólo requiere cambiar la clase en el `html` elemento, y cada color de animación se actualiza automáticamente. Una base de código, dos estilos. Eso es exactamente lo que queremos.

### 2. Optimización del rendimiento

**Usar `will-change` para sugerir al navegador que optimice:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Dígaselo al navegador con anticipación y le ayudará a optimizar. Muchas cosas en la vida funcionan mejor con un poco de preparación.

**Evite el uso de sombras de cuadros complejas en elementos grandes:**

```css
/* Not ideal - using a blurred shadow on a large element */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Better - use a pseudo-element to limit the glowing area */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Probamos esto en HagiCode. Agregar una sombra borrosa directamente a una tarjeta grande redujo la velocidad de cuadros de desplazamiento por debajo de 30 fps. Cambiar a un pseudoelemento devolvió las cosas a 60 fps constantes. Los usuarios pueden sentir absolutamente esa diferencia.

### 3. Accesibilidad

Realmente no deberías saltarte esto. Algunos usuarios encuentran que la animación les da vértigo o les distrae, y respetar sus preferencias es parte de la creación de un buen producto. Las cosas bellas no necesitan imponerse a todos.

**Consulta de medios CSS:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Provide a static fallback */
    opacity: 1;
  }
}
```

**Detectar preferencia del usuario en React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Control del nivel de intensidad

El indicador de rendimiento del token en HagiCode muestra diferentes colores de brillo según el rendimiento en tiempo real, y esto se implementa dinámicamente. Los diferentes estados deberían expresarse de manera diferente.

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Cosas a tener en cuenta

Todavía hay algunos detalles a los que vale la pena prestar atención, porque cuando descubres estos problemas de la manera más difícil, ya es demasiado tarde.

| Cosas a tener en cuenta | Explicación |
|---------|------|
| **gestión del índice z** | La capa luminosa debe utilizar un apropiado `z-index` para que no interfiera con la interacción del contenido |
| **eventos de puntero** | El pseudoelemento luminoso debería establecerse `pointer-events: none` |
| **Desbordamiento de límites** | El contenedor principal necesita `overflow: hidden`, o necesita ajustar el tamaño del pseudoelemento |
| **Impacto en el rendimiento** | Las animaciones complejas pueden afectar el rendimiento en dispositivos móviles, así que pruébelas con cuidado |
| **Modo oscuro** | Asegúrese de que el color brillante permanezca claramente visible en fondos oscuros. |
| **Cambio de tema** | Utilice variables CSS para que los colores de la animación se actualicen correctamente cuando cambie el tema |

### 6. Consejos de depuración

Los pseudoelementos pueden ser un poco difíciles de localizar en las herramientas de desarrollo, por lo que puedes agregar temporalmente un borde para verificar la posición.

```css
/* Temporarily show pseudo-element boundaries for debugging */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Una vez que termines de posicionarlo, recuerda comentar o eliminar esa línea. De lo contrario, la producción puede volverse complicada con bastante rapidez. Algunas cosas es mejor dejarlas en desarrollo.

## Resumen

Las animaciones de barrido de luz de borde no son especialmente difíciles ni verdaderamente triviales. En esencia, la fórmula es `conic-gradient` además de rotación, pero si desea un buen rendimiento, mantenibilidad y accesibilidad, todavía hay muchos detalles de implementación que manejar con cuidado.

HagiCode superó muchos de estos obstáculos y gradualmente destiló un conjunto de mejores prácticas. Así es como funcionan los proyectos: experimentas, cometes errores y mejoras paso a paso. Si está construyendo algo similar, espero que este artículo le ayude a evitar algunos desvíos innecesarios.

Algunas cosas sólo quedan claras una vez que las construyes tú mismo.
