## Escribir es más lento que hablar y hablar es más lento que una captura de pantalla: prácticas de entrada multimodal para asistentes de codificación con IA

> Escribir código tiene un límite de velocidad sin importar qué tan rápido escribas. A veces, algo que podrías decir en una frase tarda una eternidad en escribirse; A veces una captura de pantalla lo explica todo, pero aun así hay que describirlo con un montón de texto. Este artículo habla de lo que nos encontramos mientras construíamos. [HagiCode](https://hagicode.com), desde reconocimiento de voz hasta carga de imágenes. Al final, solo queríamos hacer que un asistente de codificación con IA fuera un poco más fácil de usar.

## Antecedentes

Mientras construíamos HagiCode, notamos un problema, o más bien, un problema que surgió naturalmente una vez que la gente comenzó a usarlo intensamente: depender únicamente de escribir puede ser agotador.

Piénselo: la interacción entre los usuarios y el Agente es un escenario central. Pero si cada intercambio requiere escribir sin parar en el teclado, la eficiencia no es muy buena:

1. **Escribir es demasiado lento**: para problemas complicados, como mensajes de error o problemas de interfaz de usuario, escribir todo puede llevar medio minuto, mientras que decirlo en voz alta puede tardar diez segundos. Esa brecha es real.

2. **Las imágenes son más directas**: a veces, la interfaz de usuario arroja un error, a veces desea comparar un borrador de diseño y, a veces, necesita mostrar una estructura de código. "Una imagen vale más que mil palabras" puede ser un viejo dicho, pero aún se mantiene. Dejar que la IA "vea" directamente el problema es mucho más claro que describirlo durante siglos.

3. **La interacción debe parecer natural**: los asistentes de IA modernos deben admitir texto, voz e imágenes. Los usuarios deberían poder elegir el método de entrada que les parezca más natural.

Por eso decidimos agregar soporte de reconocimiento de voz y carga de imágenes a HagiCode para que las interacciones de los agentes sean más convenientes. Si los usuarios pueden escribir un poco menos, eso ya es una victoria.

## Acerca de HagiCode

Las soluciones compartidas en este artículo provienen de nuestro trabajo práctico en el [HagiCode](https://hagicode.com) proyecto - o, más exactamente, de las lecciones aprendidas al tropezar con bastantes obstáculos.

HagiCode es un proyecto de asistente de codificación de IA de código abierto con un objetivo simple: utilizar la IA para mejorar la eficiencia del desarrollo. A medida que continuábamos construyendo, quedó claro que los usuarios deseaban encarecidamente información multimodal. A veces, decir una frase es más rápido que escribir un párrafo largo y, a veces, una captura de pantalla es mucho más clara que una explicación larga.

Esas necesidades nos impulsaron hacia adelante, y así es como finalmente tomaron forma funciones como el reconocimiento de voz y la carga de imágenes. Los usuarios ahora pueden interactuar con la IA de la manera más natural disponible para ellos, y eso se siente bien.

## Análisis

### Desafíos técnicos en el reconocimiento de voz

Al crear el reconocimiento de voz, nos encontramos con un problema complicado: **la API WebSocket del navegador no admite encabezados HTTP personalizados**.

El servicio de reconocimiento de voz que elegimos fue la API de reconocimiento de voz Doubao de ByteDance. Desafortunadamente, esta API requiere información de autenticación como `accessToken` y `secretKey` para pasar a través de encabezados HTTP. Eso creó un conflicto técnico inmediato:

```javascript
// The browser WebSocket API does not support this approach
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // Not supported
  }
});
```

Básicamente teníamos dos opciones:

1. **Enfoque de parámetro de consulta de URL**: coloque la información de autenticación en la URL
   - La ventaja es que es sencillo de implementar.
   - La desventaja es que las credenciales están expuestas a la interfaz, que es insegura; Algunas API también requieren autenticación basada en encabezados

2. **Enfoque de proxy backend**: implementar un proxy WebSocket en el backend
   - La ventaja es que las credenciales permanecen almacenadas de forma segura en el backend y la solución es totalmente compatible con los requisitos de API.
   - La desventaja es que la implementación es un poco más compleja.

Al final, elegimos el **enfoque de proxy backend**. La seguridad no es algo que se pueda comprometer.

### Requisitos funcionales para carga de imágenes

Nuestros requisitos para la carga de imágenes eran bastante sencillos:

1. **Múltiples métodos de carga**: haga clic para seleccionar un archivo, arrástrelo, suéltelo y péguelo desde el portapapeles.
2. **Validación de archivos**: las restricciones de tipo (PNG, JPG, WebP, GIF) y los límites de tamaño (5-10 MB) son requisitos básicos
3. **Experiencia de usuario**: cargue el progreso, vistas previas y mensajes de error para que los usuarios siempre sepan lo que está sucediendo
4. **Seguridad**: la validación del lado del servidor y la protección contra cargas de archivos maliciosos son esenciales

## Solución

### Reconocimiento de voz: arquitectura de proxy WebSocket

Diseñamos una arquitectura de tres capas para el reconocimiento de voz y encontramos un camino que funcionó:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**Implementaciones de componentes principales**:

1. **Procesador AudioWorklet frontal**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Resample to 16 kHz (required by the Doubao API)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Accumulate samples into 500 ms chunks
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Convert to 16-bit PCM and send
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **Controlador WebSocket backend** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **Frontal `VoiceTextArea` componente**:

```tsx
export const VoiceTextArea = forwardRef<HTMLTextAreaElement, VoiceTextAreaProps>(
  ({ value, onChange, onTextRecognized, maxDuration }, ref) => {
    const { isRecording, interimText, volume, duration, startRecording, stopRecording } =
      useVoiceRecording({ onTextRecognized, maxDuration });

    return (
      <div className="flex gap-2">
        {/* Voice button */}
        <button onClick={handleButtonClick}>
          {isRecording ? <VolumeWaveform volume={volume} /> : <Mic />}
        </button>
        {/* Text input area */}
        <textarea value={displayValue} onChange={handleChange} />
      </div>
    );
  }
);
```

### Carga de imágenes: componente de carga multimétodo

Creamos un componente de carga de imágenes con todas las funciones que admite los tres métodos de carga, cubriendo los escenarios más comunes con los que se encuentran los usuarios.

**Características principales**:

1. **Tres métodos de carga**:

```tsx
// Click to upload
const handleClick = () => fileInputRef.current?.click();

// Drag-and-drop upload
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Clipboard paste
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **Validación de interfaz**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **Controlador de carga de backend** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Validation
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Save file
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## Guía práctica

### Cómo utilizar el reconocimiento de voz

1. **Configurar el servicio de reconocimiento de voz**:
   - Abra la página de configuración de reconocimiento de voz
   - Configurar el discurso de Doubao `AppId` y `AccessToken`
   - Opcionalmente, configure palabras activas para mejorar la precisión del reconocimiento de términos específicos del dominio.

2. **Úselo en el cuadro de entrada**:
   - Haga clic en el icono del micrófono en el lado izquierdo del cuadro de entrada
   - Comience a hablar después de que aparezca la animación de forma de onda
   - Haga clic en el icono nuevamente para detener la grabación.
   - El texto reconocido se inserta automáticamente en la posición del cursor.

3. **Ejemplo de configuración de palabra activa**:

```text
TypeScript
React
useState
useEffect
```

### Cómo utilizar la carga de imágenes

1. **Métodos de carga**:
   - Haga clic en el botón cargar para elegir un archivo
   - Arrastre una imagen directamente al área de carga
   - uso `Ctrl+V` para pegar una captura de pantalla desde el portapapeles

2. **Formatos admitidos**: PNG, JPG, JPEG, WebP, GIF
3. **Límite de tamaño**: 5 MB por defecto (configurable)

### Notas

1. **Reconocimiento de voz**:
   - Se requiere permiso para el micrófono
   - Úselo en un ambiente tranquilo cuando sea posible.
   - La duración máxima de grabación admitida es de 300 segundos de forma predeterminada (configurable)

2. **Cargas de imágenes**:
   - Solo se admiten formatos de imagen comunes
   - Preste atención a los límites de tamaño de archivos
   - Las imágenes cargadas reciben automáticamente una URL de vista previa

3. **Consideraciones de seguridad**:
   - Las credenciales de reconocimiento de voz se almacenan en el backend.
   - Las cargas de imágenes pasan por una estricta validación del lado del servidor
   - Se recomienda HTTPS/WSS en entornos de producción

## Conclusión

Después de agregar reconocimiento de voz y carga de imágenes, la experiencia del usuario de HagiCode mejoró notablemente. Los usuarios ahora pueden interactuar con la IA de una manera más natural: hablando en lugar de escribir y compartiendo capturas de pantalla en lugar de describir todo manualmente. Se siente como si finalmente hubieran encontrado una forma más cómoda de comunicarse.

Mientras construíamos esta función, nos encontramos con el problema de que las API WebSocket del navegador no admiten encabezados personalizados. Al final, lo resolvimos con un enfoque de proxy backend. Esa solución no sólo preservó la seguridad, sino que también sentó las bases para la integración posterior de otros servicios WebSocket autenticados.

El componente de carga de imágenes también se beneficia al admitir múltiples métodos de carga, lo que permite a los usuarios elegir el que sea más conveniente en cada momento. Al hacer clic, arrastrar o pegar todo el trabajo, cada ruta hace el trabajo rápidamente.

"Escribir es más lento que hablar y hablar es más lento que una captura de pantalla" encaja bastante bien con el tema aquí. Si está creando un producto de asistente de IA similar, espero que estas experiencias le ayuden, aunque sea un poco.
