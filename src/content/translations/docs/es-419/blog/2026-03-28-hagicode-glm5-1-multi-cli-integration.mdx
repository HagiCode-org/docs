## Guía de integración de Hagicode y GLM-5.1 Multi-CLI

:::nota[Aviso de contexto temporal]
Este artículo documenta el diseño de integración multi-CLI y el mapeo del modelo tal como existían en el momento de la publicación. Para conocer el alcance actual siempre admitido, consulte [Instalación de la CLI del Agente AI](/es-419/related-software-installation/ai-agent-cli/) y el [Guía de instalación](/es-419/installation); el `IFlowCli` mencionado aquí solo representa una configuración histórica en ese momento y no es parte del alcance de soporte activo actual.
:::

## Antecedentes

En el proyecto Hagicode, los usuarios pueden elegir entre múltiples herramientas CLI para impulsar asistentes de programación de IA, incluidos Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI y más. Estas herramientas CLI son herramientas de programación de IA de uso general por sí solas, pero a través de la capa de abstracción de Hagicode, pueden conectarse de manera flexible a diferentes proveedores de modelos de IA.

Zhipu AI (ZAI) proporciona una interfaz compatible con la API Anthropic Claude, lo que permite que estas herramientas CLI utilicen directamente los modelos nacionales de la serie GLM. Entre ellos, GLM-5.1 es el último modelo de lenguaje grande de Zhipu, con mejoras significativas con respecto a GLM-5.0.

## Arquitectura de abstracción CLI de Hagicode

Hagicode define 11 tipos de proveedores CLI a través del `AIProviderType` enumeración, que cubre las principales herramientas CLI de programación de IA:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Cada CLI tiene la configuración de parámetros del modelo correspondiente y admite la `model` y `reasoning` parámetros:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>

        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],

        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],

        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini does not support the reasoning parameter
        // ...
    };
```

## Usando GLM-5.1 en diferentes CLI

### 1. Código Claude CLI + GLM-5.1

Claude Code CLI es una de las CLI principales de Hagicode y se configura a través del sistema de configuración Hero:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

correspondiente `HeroEquipmentCatalogItem` configuración:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. CLI de código abierto + GLM-5.1

OpenCode CLI es la CLI más flexible y admite la especificación de cualquier modelo en el `provider/model` formato:

**Método 1: utilice el prefijo de proveedor ZAI**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Método 2: utilizar el ID del modelo directamente**
```json
{
  "model": "glm-5.1"
}
```

**Método 3: UI de configuración de front-end**

en `HeroModelEquipmentForm.tsx`, OpenCode CLI tiene una sugerencia de marcador de posición dedicada:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Los usuarios pueden ingresar:
```
zai/glm-5.1
glm-5.1
```

**Lógica de análisis del modelo CLI OpenCode**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. CLI de Codebuddy + GLM-5.1

Codebuddy CLI tiene una lógica de manejo heredada dedicada:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Nota**: Codebuddy conserva un manejo especial para GLM-5.0 y no utiliza la normalización heredada:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## Ventajas de mejora de GLM-5.1

En comparación con GLM-5.0, GLM-5.1 aporta las siguientes mejoras significativas:

### 1. Mejor capacidad de razonamiento

Según la información de lanzamiento oficial de Zhipu, las mejoras en GLM-5.1 incluyen:

- **Comprensión de código más sólida**: análisis más preciso de estructuras de código complejas
- **Comprensión de contexto más amplia**: admite un contexto conversacional más extenso
- **Llamada de herramientas mejorada**: mayor tasa de éxito para las llamadas de herramientas MCP
- **Estabilidad de salida**: Reduce la aleatoriedad y las alucinaciones

### 2. Compatibilidad integral con múltiples CLI

GLM-5.1 cubre todas las CLI principales compatibles con Hagicode:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Notas

### 1. Configuración de la clave API

Asegúrate de que `ANTHROPIC_AUTH_TOKEN` La variable de entorno está configurada correctamente. Es la credencial requerida para que cada CLI se conecte al modelo.

### 2. Disponibilidad del modelo

GLM-5.1 debe ser habilitado por el proveedor del modelo correspondiente:
- La plataforma Zhipu AI ZAI lo admite de forma predeterminada
- Alibaba Cloud DashScope puede requerir una aplicación separada

### 3. Formato CLI OpenCode

Al usar el `provider/model` formato, asegúrese de que el ID del proveedor sea correcto:
- Zhipu IA: `zai` o `zhipuai`
- Nube de Alibaba: `aliyun` o `dashscope`

### 4. Parámetro de razonamiento

- `high` se recomienda para obtener los mejores resultados de generación de código
- Gemini CLI no admite el parámetro de razonamiento e ignorará esta configuración automáticamente

## Conclusión

A través de una capa de abstracción unificada, Hagicode permite una integración flexible entre GLM-5.1 y múltiples CLI. Los desarrolladores pueden elegir la herramienta CLI que mejor se adapte a sus preferencias y escenarios de uso y luego utilizar el último modelo GLM-5.1 mediante una configuración sencilla.

Como última versión del modelo de Zhipu, el GLM-5.1 ofrece claras mejoras con respecto al GLM-5.0:
- Un identificador de versión independiente sin carga heredada
- Mayor razonamiento y comprensión del código.
- Amplia compatibilidad con múltiples CLI
- Configuración de nivel de razonamiento flexible

Con las variables de entorno correctas y el equipo Hero configurado, los usuarios pueden desbloquear completamente el poder de GLM-5.1 en diferentes entornos CLI.
