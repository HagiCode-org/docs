## Cómo reproducir proyectos en la era de la IA: Vault, un sistema de almacenamiento persistente entre proyectos

> En la era del desarrollo asistido por IA, ¿cómo podemos ayudar a los asistentes de IA a comprender mejor nuestros recursos de aprendizaje? El proyecto HagiCode creó el sistema Vault como una capa de abstracción de almacenamiento de conocimiento unificada que la IA puede comprender, lo que mejora en gran medida la eficiencia del aprendizaje a través de la reproducción del proyecto.

## Antecedentes

En la era de la IA, la forma en que los desarrolladores aprenden nuevas tecnologías y arquitecturas está cambiando profundamente. "Reproducir proyectos", es decir, estudiar y aprender profundamente del código, la arquitectura y los patrones de diseño de excelentes proyectos de código abierto, se ha convertido en una forma eficaz de aprender. En comparación con los métodos tradicionales, como leer libros o mirar vídeos, leer y ejecutar directamente proyectos de código abierto de alta calidad le ayuda a comprender las prácticas de ingeniería del mundo real mucho más rápido.

Aún así, este método de aprendizaje presenta bastantes desafíos.

Los materiales de aprendizaje están demasiado dispersos. Tus notas pueden vivir en Obsidian, los repositorios de códigos pueden estar dispersos en diferentes carpetas y el historial de conversaciones de tu asistente de IA se convierte en otra isla de datos aislada. Cuando desea que la IA le ayude a analizar un proyecto, debe copiar manualmente fragmentos de código y organizar el contexto, lo cual es bastante tedioso.

Lo que es aún más problemático es el contexto roto. Los asistentes de IA no pueden acceder directamente a sus recursos de aprendizaje locales, por lo que debe volver a proporcionar información básica en cada conversación. Además de eso, los repositorios de código reproducido se actualizan rápidamente, la sincronización manual es propensa a errores y el conocimiento es difícil de compartir entre múltiples proyectos de aprendizaje.

En el fondo, todos estos problemas provienen de "islas de datos". Si hubiera una capa de abstracción de almacenamiento unificada que permitiera a los asistentes de IA comprender y acceder a todos sus recursos de aprendizaje, el problema se resolvería claramente.

## Acerca de HagiCode

El sistema Vault compartido en este artículo es exactamente la solución que desarrollamos mientras construíamos [HagiCode](https://hagicode.com). HagiCode es un proyecto de asistente de codificación de IA y, en nuestro trabajo de desarrollo diario, a menudo necesitamos estudiar y consultar muchos proyectos diferentes de código abierto. Para ayudar a los asistentes de IA a comprender mejor estos recursos de aprendizaje, diseñamos Vault, un sistema de almacenamiento persistente entre proyectos.

Esta solución ya ha sido validada en HagiCode en uso real. Si se enfrenta a desafíos similares en la gestión del conocimiento, espero que estas experiencias puedan ofrecerle algo de inspiración. Después de todo, una vez que hayas caído en algunos pozos, debes dejar algo para la siguiente persona.

## Filosofía de diseño del sistema de bóveda

La idea central del sistema Vault es simple: crear una capa de abstracción de almacenamiento de conocimiento unificada que la IA pueda entender. Desde una perspectiva de implementación, el sistema tiene varias características clave.

### Soporte multitipo

El sistema admite cuatro tipos de bóvedas, cada una correspondiente a un escenario de uso diferente:

```typescript
// folder: general-purpose folder type
export const DEFAULT_VAULT_TYPE = 'folder';

// coderef: a type specifically for reproduced code projects
export const CODEREF_VAULT_TYPE = 'coderef';

// obsidian: integrated with Obsidian note-taking software
export const OBSIDIAN_VAULT_TYPE = 'obsidian';

// system-managed: vault automatically managed by the system
export const SYSTEM_MANAGED_VAULT_TYPE = 'system-managed';
```

Entre ellos, el tipo **coderef** es el más utilizado en HagiCode. Está diseñado específicamente para proyectos de código reproducido y proporciona una estructura de directorio estandarizada y descripciones de metadatos legibles por IA.

### Mecanismo de almacenamiento persistente

El registro de Vault se almacena de forma persistente en formato JSON, lo que garantiza que la configuración permanezca disponible después de que se reinicie la aplicación:

```csharp
public class VaultRegistryStore : IVaultRegistryStore
{
    private readonly string _registryFilePath;

    public VaultRegistryStore(IConfiguration configuration, ILogger<VaultRegistryStore> logger)
    {
        var dataDir = configuration["DataDir"] ?? "./data";
        var absoluteDataDir = Path.IsPathRooted(dataDir)
            ? dataDir
            : Path.GetFullPath(Path.Combine(Directory.GetCurrentDirectory(), dataDir));

        _registryFilePath = Path.Combine(absoluteDataDir, "personal-data", "vaults", "registry.json");
    }
}
```

La ventaja de este diseño es que es sencillo y fiable. JSON es legible por humanos, lo que facilita la depuración y la edición manual; El almacenamiento del sistema de archivos evita la complejidad de una base de datos y reduce las dependencias del sistema. Después de todo, a veces la opción más sencilla es realmente la mejor.

### Integración del contexto de IA

Lo más importante es que el sistema puede inyectar automáticamente información de la bóveda en el contexto de las propuestas de IA:

```typescript
export function buildTargetVaultsText(
  vaults: VaultForText[],
  template: VaultPromptTemplate = DEFAULT_VAULT_PROMPT_TEMPLATE,
): string {
  const readOnlyVaults = vaults.filter((vault) => vault.accessType === 'read');
  const editableVaults = vaults.filter((vault) => vault.accessType === 'write');

  if (readOnlyVaults.length === 0 && editableVaults.length === 0) {
    return '';
  }

  const sections = [
    buildVaultSection(readOnlyVaults, template.reference),
    buildVaultSection(editableVaults, template.editable),
  ].filter(Boolean);

  return `\n\n### ${template.heading}\n\n${sections.join('\n')}`;
}
```

Esto habilita una capacidad importante: los asistentes de IA pueden comprender automáticamente los recursos de aprendizaje disponibles sin que los usuarios proporcionen contexto manualmente. Se podría decir que eso cuenta como una especie de entendimiento tácito.

## La estructura estandarizada de CodeRef Vault

Para el tipo de bóveda coderef, HagiCode proporciona una estructura de directorio estandarizada:

```
my-coderef-vault/
├── index.yaml          # vault metadata description
├── AGENTS.md           # operating guide for AI assistants
├── docs/               # stores study notes and documents
└── repos/              # manages reproduced code repositories through Git submodules
```

Al crear una bóveda, el sistema inicializa automáticamente esta estructura:

```csharp
private async Task EnsureCodeRefStructureAsync(
    string vaultName,
    string physicalPath,
    ICollection<VaultBootstrapDiagnosticDto> diagnostics,
    CancellationToken cancellationToken)
{
    Directory.CreateDirectory(physicalPath);

    var indexPath = Path.Combine(physicalPath, CodeRefIndexFileName);
    var docsPath = Path.Combine(physicalPath, CodeRefDocsDirectoryName);
    var reposPath = Path.Combine(physicalPath, CodeRefReposDirectoryName);

    // Create the standard directory structure
    if (!Directory.Exists(docsPath))
    {
        Directory.CreateDirectory(docsPath);
    }

    if (!Directory.Exists(reposPath))
    {
        Directory.CreateDirectory(reposPath);
    }

    // Create the AGENTS.md guide
    await EnsureCodeRefAgentsDocumentAsync(physicalPath, cancellationToken);

    // Create the index.yaml metadata
    await WriteCodeRefIndexDocumentAsync(indexPath, mergedDocument, cancellationToken);
}
```

Esta estructura también está cuidadosamente diseñada:

- **docs/** almacena tus notas de estudio, donde puedes registrar tu comprensión del código, análisis de arquitectura, lecciones aprendidas, etc. en Markdown
- **repos/** administra los repositorios reproducidos a través de submódulos de Git en lugar de copiar el código directamente, lo que mantiene el código sincronizado y ahorra espacio.
- **index.yaml** contiene los metadatos de la bóveda para que los asistentes de IA puedan comprender rápidamente el propósito y el contenido de la bóveda.
- **AGENTS.md** es una guía escrita específicamente para asistentes de IA, que explica cómo manejar el contenido de la bóveda.

Organizada de esta manera, tal vez la IA pueda entender lo que tienes en mente un poco más fácilmente.

## Inicialización automática para bóvedas administradas por el sistema

Además de las bóvedas creadas manualmente, HagiCode también admite bóvedas administradas por el sistema:

```csharp
public async Task<IReadOnlyList<VaultRegistryEntry>> EnsureAllSystemManagedVaultsAsync(
    CancellationToken cancellationToken = default)
{
    var definitions = GetAllResolvedDefinitions();
    var entries = new List<VaultRegistryEntry>(definitions.Count);

    foreach (var definition in definitions)
    {
        entries.Add(await EnsureResolvedSystemManagedVaultAsync(definition, cancellationToken));
    }

    return entries;
}
```

El sistema crea y administra automáticamente las siguientes bóvedas:

- **hagiprojectdata**: almacenamiento de datos del proyecto utilizado para guardar la configuración y el estado del proyecto.
- **datos personales**: almacenamiento de datos personales utilizado para guardar las preferencias del usuario
- **hbsprompt**: una biblioteca de plantillas de mensajes que se utiliza para administrar mensajes de IA de uso común

Estas bóvedas se inicializan automáticamente cuando se inicia el sistema, por lo que los usuarios no necesitan configurarlas manualmente. Algunas cosas simplemente es mejor dejarlas en manos del sistema en lugar de que los humanos se preocupen por ellas.

## Mecanismo de control de acceso

Una parte importante del diseño es el control de acceso. El sistema divide las bóvedas en dos tipos de acceso:

```typescript
export interface VaultForText {
  id: string;
  name: string;
  type: string;
  physicalPath: string;
  accessType: 'read' | 'write';  // Key: distinguish read-only from editable
}
```

- **referencia (solo lectura)**: la IA solo se utiliza para análisis y comprensión y no puede modificar el contenido. Adecuado para proyectos, documentos y materiales similares de código abierto referenciados
- **editable (editable)**: la IA puede modificar el contenido según sea necesario para la tarea. Adecuado para sus notas, borradores y materiales similares

Esta distinción importa. Le dice a la IA qué contenido es "referencia de solo lectura" y qué contenido es "seguro para editar", lo que reduce el riesgo de cambios accidentales. Después de todo, nadie quiere que su arduo trabajo desaparezca debido a una edición involuntaria.

## En la práctica: creación y uso de Vault

Ahora que hemos cubierto las ideas, veamos cómo funciona en la práctica.

### Crear una bóveda CodeRef

Aquí hay un ejemplo completo de llamada frontend:

```typescript
const createCodeRefVault = async () => {
  const response = await VaultService.postApiVaults({
    requestBody: {
      name: "React Learning Vault",
      type: "coderef",
      physicalPath: "/Users/developer/vaults/react-learning",
      gitUrl: "https://github.com/facebook/react.git"
    }
  });

  // The system will automatically:
  // 1. Clone the React repository into vault/repos/react
  // 2. Create the docs/ directory for notes
  // 3. Generate the index.yaml metadata
  // 4. Create the AGENTS.md guide file

  return response;
};
```

Esta llamada API completa una serie de acciones: crear la estructura del directorio, inicializar submódulos de Git, generar archivos de metadatos y más. Sólo necesitas proporcionar la información básica y dejar que el sistema se encargue del resto. Honestamente, es un enfoque bastante libre de preocupaciones.

### Utilice Vault en una propuesta de IA

Después de crear la bóveda, puedes hacer referencia a ella en una propuesta de IA:

```typescript
const proposal = composeProposalChiefComplaint({

```

El sistema inyecta automáticamente información de la bóveda en el contexto de la IA, permitiéndole saber qué recursos de aprendizaje están disponibles. Cuando la IA puede comprender lo que usted tiene en mente, ese tipo de comprensión tácita es difícil de lograr.

## Mejores prácticas y aspectos a tener en cuenta

Mientras utilizamos el sistema Vault, hemos resumido algunas lecciones aprendidas.

### Seguridad en el camino

El sistema valida estrictamente las rutas para evitar ataques de cruce de rutas:

```csharp
private static string ResolveFilePath(string vaultRoot, string relativePath)
{
    var rootPath = EnsureTrailingSeparator(Path.GetFullPath(vaultRoot));
    var combinedPath = Path.GetFullPath(Path.Combine(rootPath, relativePath));
    if (!combinedPath.StartsWith(rootPath, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode,
            "Vault file paths must stay inside the registered vault root.");
    }
    return combinedPath;
}
```

Esto es importante. Si personaliza una ruta de bóveda, asegúrese de que permanezca dentro del rango permitido; de lo contrario, el sistema rechazará la operación. Realmente no se puede dejar de enfatizar la seguridad.

### Gestión de submódulos de Git

CodeRef Vault recomienda submódulos de Git en lugar de copiar el código directamente:

```csharp
private static string BuildCodeRefAgentsContent()
{
    return """
    # CodeRef Vault Guide

    Repositories under `repos/` should be maintained through Git submodules
    rather than copied directly into the vault root.

    Keep this structure stable so assistants and tools can understand the vault quickly.
    """ + Environment.NewLine;
}
```

Esto trae varias ventajas: mantener el código sincronizado con el flujo ascendente, ahorrar espacio en disco y facilitar la administración de múltiples versiones del código. Después de todo, ¿quién quiere descargar lo mismo una y otra vez?

### Límites de vista previa de archivos

Para evitar problemas de rendimiento, el sistema limita el tamaño y el tipo de archivo:

```csharp
private const int FileEnumerationLimit = 500;
private const int PreviewByteLimit = 256 * 1024;  // 256KB
```

Si su bóveda contiene una gran cantidad de archivos o archivos muy grandes, el rendimiento de la vista previa puede verse afectado. En ese caso, puede considerar procesar archivos en lotes o utilizar herramientas de búsqueda especializadas. A veces, cuando algo se vuelve demasiado grande, se vuelve más difícil de manejar, no más fácil.

### Información de diagnóstico

Al crear una bóveda, el sistema devuelve información de diagnóstico para ayudar con la depuración:

```csharp
List<VaultBootstrapDiagnosticDto> bootstrapDiagnostics = [];

if (IsCodeRefVaultType(normalizedType))
{
    bootstrapDiagnostics = await EnsureCodeRefBootstrapAsync(
        normalizedName,
        normalizedPhysicalPath,
        normalizedGitUrl,
        cancellationToken);
}
```

Si la creación falla, puede inspeccionar la información de diagnóstico para comprender la causa específica. Cuando algo sale mal, comprobar los diagnósticos suele ser la forma más directa de avanzar.

## Resumen

A través de una capa de abstracción de almacenamiento unificada, el sistema Vault resuelve varios puntos débiles de la reproducción de proyectos en la era de la IA:

- **Gestión centralizada del conocimiento**: todos los recursos de aprendizaje están reunidos en un solo lugar en lugar de estar dispersos por todas partes.
- **Inyección automática de contexto de IA**: los asistentes de IA pueden comprender automáticamente los recursos de aprendizaje disponibles sin configuración manual del contexto
- **Reutilización del conocimiento entre proyectos**: el conocimiento se puede compartir y reutilizar en múltiples proyectos de aprendizaje.
- **Estructura de directorio estandarizada**: un diseño de directorio consistente reduce la curva de aprendizaje

Esta solución ya ha sido validada en el proyecto HagiCode. Si también está creando herramientas relacionadas con el desarrollo asistido por IA o enfrenta problemas similares de gestión del conocimiento, espero que estas experiencias puedan servirle como referencia útil.

En verdad, el valor de una solución técnica no reside en lo complicada que sea, sino en si resuelve problemas reales. La idea central del sistema Vault es muy simple: construir una capa de almacenamiento de conocimiento unificada que la IA pueda comprender. Sin embargo, es precisamente esta simple abstracción la que mejoró bastante nuestra eficiencia de desarrollo.

A veces, el enfoque simple es realmente el mejor. Después de todo, las cosas complicadas a menudo esconden aún más trampas...
