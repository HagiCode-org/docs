## Análisis técnico del sistema de habilidades HagiCode: creación de una plataforma escalable de gestión de habilidades de IA

> Este artículo analiza en profundidad la arquitectura y la implementación del sistema de gestión de habilidades en el proyecto HagiCode, cubriendo los detalles técnicos detrás de cuatro capacidades principales: gestión global local, búsqueda de mercado, recomendaciones inteligentes y gestión de proveedores confiables.

## Antecedentes

En el campo de los asistentes de codificación de IA, cómo ampliar los límites de las capacidades de la IA siempre ha sido una cuestión central. Claude Code ya es fuerte en asistencia de código, pero diferentes equipos de desarrollo y diferentes pilas de tecnología a menudo necesitan capacidades especializadas para escenarios específicos, como el manejo de implementaciones de Docker, optimización de bases de datos o generación de componentes frontend. Ahí es exactamente donde el sistema de Habilidades adquiere especial importancia.

Durante el desarrollo del proyecto HagiCode, nos topamos con un desafío similar: ¿cómo permitimos que Claude Code "aprenda" nuevas habilidades profesionales como lo haría una persona, manteniendo al mismo tiempo una experiencia de usuario sólida y una buena capacidad de mantenimiento de ingeniería? Este problema es a la vez difícil y simple a su manera. En torno a esa pregunta, diseñamos e implementamos un completo sistema de gestión de habilidades.

Este artículo analiza en detalle la arquitectura técnica y la implementación principal del sistema. Está destinado a desarrolladores interesados ​​en la extensibilidad de la IA y la integración de herramientas de línea de comandos. Puede que te resulte útil o no, pero al menos ya está escrito.

## Acerca de HagiCode

El enfoque compartido en este artículo proviene de nuestra experiencia práctica en el [HagiCode](https://hagicode.com) proyecto. HagiCode es un proyecto de asistente de codificación de IA de código abierto diseñado para ayudar a los equipos de desarrollo a mejorar la eficiencia de la ingeniería. La pila del proyecto incluye ASP.NET Core, el marco distribuido Orleans, una interfaz TanStack Start + React y el subsistema de gestión de habilidades presentado en este artículo.

El repositorio de GitHub es [HagiCode-org/sitio](https://github.com/HagiCode-org/site). Si encuentra valioso el enfoque técnico de este artículo, no dude en darle una estrella. Después de todo, más estrellas tienden a mejorar el estado de ánimo.

## Descripción general de la arquitectura del sistema

El sistema Skill utiliza una arquitectura separada frontend-backend. No hay nada especialmente misterioso en eso.

**Frontend** utiliza TanStack Start + React para crear la interfaz de usuario, con Redux Toolkit administrando el estado. Las cuatro capacidades principales se asignan directamente a cuatro componentes de la pestaña: Habilidades locales, Galería de habilidades, Recomendaciones inteligentes y Proveedores confiables. Al final, el diseño se trata principalmente de mejorar la experiencia del usuario.

**Backend** se basa en ASP.NET Core + ABP Framework y utiliza Orleans Grain para la gestión del estado distribuido. El cliente API en línea envuelve el `IOnlineApiClient` interfaz para comunicarse con el servicio de catálogo de habilidades remoto.

El principio arquitectónico general es separar la ejecución de comandos de la lógica empresarial. A través del patrón del adaptador, los detalles de implementación de la ejecución del comando npm/npx están ocultos dentro de módulos independientes. Después de todo, nadie quiere realmente que las llamadas a la línea de comandos estén esparcidas por todo el código base.

## Capacidad básica 1: Gestión global local

La gestión global local es el módulo más básico. Es responsable de enumerar las habilidades instaladas y respaldar las operaciones de desinstalación. No hay nada demasiado complicado aquí; se trata principalmente de hacer bien lo básico.

### Enfoque técnico

La implementación vive en `LocalSkillsTab.tsx` y `LocalSkillCommandAdapter.cs`. La idea central es envolver el `npx skills` comando, analiza su salida JSON y la convierte en estructuras de datos internas. Suena simple y en la práctica lo es.

```csharp
public async Task<IReadOnlyList<LocalSkillInventoryResponseDto>> GetLocalSkillsAsync(
    CancellationToken cancellationToken = default)
{
    var result = await _commandAdapter.ListGlobalSkillsAsync(cancellationToken);
    return result.Skills.Select(skill => new LocalSkillInventoryResponseDto
    {
        Name = skill.Name,
        Version = skill.Version,
        Source = skill.Source,
        InstalledPath = skill.InstalledPath,
        Description = skill.Description
    }).ToList();
}
```

El flujo de datos es muy claro: el frontend envía una solicitud -> `SkillGalleryAppService` lo recibe -> `LocalSkillCommandAdapter` ejecuta el comando npx -> se analiza el resultado JSON -> se devuelve un DTO. Cada paso sigue naturalmente al anterior.

La desinstalación de habilidades utiliza el `npx skills remove -g <skillName> -y` comando, y el sistema maneja automáticamente las dependencias y la limpieza. Los metadatos de instalación se almacenan en `managed-install.json` dentro del directorio de habilidades, registrando información como el tiempo de instalación y la versión fuente para actualizaciones y auditorías posteriores. Algunas cosas simplemente valen la pena registrarlas.

### Flujo de instalación en detalle

La instalación de habilidades requiere varios pasos coordinados. La verdad es que no es especialmente complicado:

```csharp
public async Task<SkillInstallResultDto> InstallAsync(
    SkillInstallRequestDto request,
    CancellationToken cancellationToken = default)
{
    // 1. Normalize the installation reference
    var normalized = _referenceNormalizer.Normalize(
        request.SkillId,
        request.Source,
        request.SkillSlug,
        request.Version);

    // 2. Check prerequisites
    await _prerequisiteChecker.CheckAsync(cancellationToken);

    // 3. Acquire installation lock
    using var installLock = await _lockProvider.AcquireAsync(normalized.SkillId);

    // 4. Execute installation command
    var result = await _installCommandRunner.ExecuteAsync(
        new SkillInstallCommandExecutionRequest
        {
            Command = $"npx skills add {normalized.FullReference} -g -y",
            Timeout = TimeSpan.FromMinutes(4)
        },
        cancellationToken);

    // 5. Persist installation metadata
    await _metadataStore.WriteAsync(normalized.SkillPath, request);

    return new SkillInstallResultDto { Success = result.Success };
}
```

Aquí se utilizan varios patrones de diseño clave: el **normalizador de referencia** convierte diferentes formatos de entrada, como `tanweai/pua` y `@opencode/docker-skill`, en una representación interna unificada; el **mecanismo de bloqueo de instalación** garantiza que solo se pueda ejecutar una operación de instalación para la misma habilidad a la vez; y **salida de transmisión** envía el progreso de la instalación al frontend en tiempo real a través de eventos enviados por el servidor, para que los usuarios puedan ver registros similares a los de una terminal a medida que ocurren.

Al final, todos estos patrones existen con un propósito: hacer que el sistema sea más sencillo de usar y mantener.

## Capacidad principal 2: búsqueda en el mercado

La búsqueda en Marketplace permite a los usuarios descubrir e instalar habilidades de la comunidad. La capacidad de una persona siempre es limitada; El conocimiento colectivo va mucho más allá.

### Enfoque técnico

La función de búsqueda se basa en la API en línea. `https://api.hagicode.com/v1/skills/search`. Para mejorar la velocidad de respuesta, el sistema implementa el almacenamiento en caché. La caché es un poco como la memoria: si guardas cosas útiles a mano, no tendrás que pensar tanto la próxima vez.

```csharp
private async Task<IReadOnlyList<SkillGallerySkillDto>> SearchCatalogAsync(
    string query,
    CancellationToken cancellationToken,
    IReadOnlySet<string>? allowedSources = null)
{
    var cacheKey = $"skill_search:{query}:{string.Join(",", allowedSources ?? Array.Empty<string>())}";

    if (_memoryCache.TryGetValue(cacheKey, out var cached))
        return (IReadOnlyList<SkillGallerySkillDto>)cached!;

    var response = await _onlineApiClient.SearchAsync(
        new SearchSkillsRequest
        {
            Query = query,
            Limit = _options.LimitPerQuery,
        },
        cancellationToken);

    var results = response.Skills
        .Where(skill => allowedSources is null || allowedSources.Contains(skill.Source))
        .Select(skill => new SkillGallerySkillDto { ... })
        .ToList();

    _memoryCache.Set(cacheKey, results, TimeSpan.FromMinutes(10));
    return results;
}
```

Los resultados de búsqueda admiten el filtrado por fuentes confiables, por lo que los usuarios solo ven las fuentes de habilidades en las que confían. Consultas de semillas como `popular` y `recent` se utilizan para inicializar el catálogo, lo que permite a los usuarios ver las habilidades populares recomendadas la primera vez que lo abren. Las primeras impresiones siguen siendo importantes.

## Capacidad central 3: recomendaciones inteligentes

Las recomendaciones inteligentes son la parte más compleja del sistema. Pueden recomendar automáticamente las habilidades más adecuadas en función del contexto actual del proyecto. Por complejo que sea, todavía vale la pena construirlo.

### Flujo de recomendación

El flujo completo de recomendaciones se divide en cinco etapas:

```
1. Build project context
   ↓
2. AI generates search queries
   ↓
3. Search the online catalog in parallel
   ↓
4. AI ranks the candidates
   ↓
5. Return the recommendation list
```

Primero, el sistema analiza características como la pila de tecnología del proyecto, los lenguajes de programación y la estructura de dominio para crear un "perfil del proyecto". Ese perfil es un poco como un currículum, que registra los rasgos clave del proyecto.

Luego se utiliza un AI Grain para generar consultas de búsqueda específicas. Este diseño es realmente bastante interesante: en lugar de preguntarle directamente a la IA "¿Qué habilidades debería recomendar?", primero le pedimos que piense en "¿Qué términos de búsqueda es probable que encuentren habilidades relevantes?". A veces, la forma en que haces la pregunta importa más que la respuesta misma:

```csharp
var queryGeneration = await aiGrain.GenerateSkillRecommendationQueriesAsync(
    projectContext,      // Project context
    locale,              // User language preference
    maxQueries,           // Maximum number of queries
    effectiveSearchHero); // AI model selection
```

A continuación, esas consultas de búsqueda se ejecutan en paralelo para recopilar una lista de habilidades candidatas. El procesamiento paralelo es, al fin y al cabo, sólo una forma de ahorrar tiempo.

Finalmente, otro AI Grain clasifica las habilidades de los candidatos. Este paso considera factores como la relevancia de las habilidades para el proyecto, el estado de confianza y las preferencias históricas del usuario:

```csharp
var ranking = await aiGrain.RankSkillRecommendationsAsync(
    projectContext,
    candidates,
    installedSkillNames,
    locale,
    maxRecommendations,
    effectiveRankingHero);

response.Items = MergeRecommendations(projectContext, candidates, ranking, maxRecommendations);
```

### Mecanismo de respaldo

Los modelos de IA pueden responder lentamente o dejar de estar disponibles temporalmente. Incluso los mejores sistemas a veces tropiezan. Por esa razón, el sistema incluye un mecanismo de respaldo determinista: cuando el servicio de IA no está disponible, utiliza un algoritmo heurístico basado en reglas para generar recomendaciones, como inferir las habilidades probablemente necesarias a partir de las dependencias en `package.json`.

Dicho claramente, este mecanismo de respaldo es simplemente un plan de respaldo para el sistema.

## Capacidad principal 4: Gestión de proveedores confiables

La gestión de proveedores confiables permite a los usuarios controlar qué fuentes de habilidades se consideran confiables. La confianza sigue siendo algo que los usuarios deberían poder definir por sí mismos.

### Reglas coincidentes

Los proveedores de confianza admiten dos reglas de coincidencia: **coincidencia exacta** (`exact`) y **coincidencia de prefijo** (`prefix`).

```csharp
public static TrustedSkillProviderResolutionSnapshot Resolve(
    TrustedSkillProviderSnapshot snapshot,
    string source)
{
    var normalizedSource = Normalize(source);

    foreach (var entry in snapshot.Entries.OrderBy(e => e.SortOrder))
    {
        if (!entry.IsEnabled) continue;

        foreach (var rule in entry.MatchRules)
        {
            bool isMatch = rule.MatchType switch
            {
                TrustedSkillProviderMatchRuleType.Exact
                    => string.Equals(normalizedSource, Normalize(rule.Value),
                        StringComparison.OrdinalIgnoreCase),
                TrustedSkillProviderMatchRuleType.Prefix
                    => normalizedSource.StartsWith(Normalize(rule.Value) + "/",
                        StringComparison.OrdinalIgnoreCase),
                _ => false
            };

            if (isMatch)
                return new TrustedSkillProviderResolutionSnapshot
                {
                    IsTrustedSource = true,
                    ProviderId = entry.ProviderId,
                    DisplayName = entry.DisplayName
                };
        }
    }

    return new TrustedSkillProviderResolutionSnapshot { IsTrustedSource = false };
}
```

Los proveedores de confianza integrados incluyen organizaciones y proyectos conocidos como Vercel, Azure, anthropics, Microsoft y el uso de navegadores. Se pueden agregar proveedores personalizados a través de archivos de configuración especificando un ID de proveedor, un nombre para mostrar, una etiqueta de insignia, reglas de coincidencia y más. El mundo es lo suficientemente grande como para que confiar únicamente en unos pocos integrados nunca sea suficiente.

### Implementación de persistencia

La configuración confiable persiste usando un Orleans Grain:

```csharp
public class TrustedSkillProviderGrain : Grain<TrustedSkillProviderState>,
    ITrustedSkillProviderGrain
{
    public async Task UpdateConfigurationAsync(TrustedSkillProviderSnapshot snapshot)
    {
        State.Snapshot = snapshot;
        await WriteStateAsync();
    }

    public Task<TrustedSkillProviderSnapshot> GetConfigurationAsync()
    {
        return Task.FromResult(State.Snapshot);
    }
}
```

La ventaja de este enfoque es que los cambios de configuración se sincronizan automáticamente en todos los nodos, sin necesidad de actualizar las cachés manualmente. La automatización consiste, en última instancia, en dejar que la gente se preocupe menos.

## Diseño técnico clave

### Patrón de adaptador de ejecución de comandos

El sistema Skill necesita ejecutar varios comandos npx. Si esa lógica estuviera esparcida por todas partes, el código rápidamente se volvería difícil de mantener. Por eso diseñamos una interfaz de adaptador. Al final, los patrones de diseño existen para hacer que el código sea más fácil de mantener:

```csharp
public interface ISkillInstallCommandRunner
{
    Task<SkillInstallCommandExecutionResult> ExecuteAsync(
        SkillInstallCommandExecutionRequest request,
        CancellationToken cancellationToken = default);
}
```

Diferentes comandos tienen diferentes implementaciones de ejecutor, pero todos implementan la misma interfaz, lo que hace que las pruebas y el reemplazo sean sencillos.

### Salida de transmisión SSE

El progreso de la instalación se envía al frontend en tiempo real a través de eventos enviados por el servidor:

```csharp
public async Task InstallWithProgressAsync(
    SkillInstallRequestDto request,
    IServerStreamWriter<SkillInstallProgressEventDto> stream,
    CancellationToken cancellationToken)
{
    var process = new Process
    {
        StartInfo = new ProcessStartInfo
        {
            FileName = "npx",
            Arguments = $"skills add {request.FullReference} -g -y",
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false
        }
    };

    process.OutputDataReceived += async (sender, e) =>
    {
        await stream.WriteAsync(new SkillInstallProgressEventDto
        {
            EventType = "output",
            Data = e.Data ?? string.Empty
        });
    };

    process.Start();
    process.BeginOutputReadLine();
    await process.WaitForExitAsync(cancellationToken);
}
```

En la interfaz, los usuarios pueden ver resultados similares a los de una terminal en tiempo real, lo que hace que la experiencia sea muy intuitiva. La retroalimentación en tiempo real ayuda a las personas a sentirse cómodas.

## Guía práctica

### Instalación de una habilidad comunitaria

Tome la instalación del `pua` habilidad como ejemplo (es una habilidad popular de la comunidad):

1. Abra el cajón de Habilidades y cambie al `Skill Gallery` pestaña
2. Entrar `pua` en el cuadro de búsqueda
3. Haga clic en el resultado de la búsqueda para ver los detalles de la habilidad.
4. Haga clic en `Install` botón
5. Cambiar a la `Local Skills` pestaña para confirmar que la instalación se realizó correctamente

El comando de instalación es `npx skills add tanweai/pua -g -y`, y el sistema maneja todos los detalles automáticamente. En realidad, no hay tantos pasos una vez que los sigues uno por uno.

### Agregar una fuente confiable personalizada

Si tu equipo tiene su propio repositorio de habilidades, puedes agregarlo como fuente confiable:

```yaml
providerId: "my-team"
displayName: "My Team Skills"
badgeLabel: "MyTeam"
isEnabled: true
sortOrder: 100
matchRules:
  - matchType: "prefix"
    value: "my-team/"
  - matchType: "exact"
    value: "my-team/special-skill"
```

De esta manera, todas las habilidades de su equipo mostrarán una insignia de confianza, lo que hará que los usuarios se sientan más cómodos al instalarlas. Las etiquetas y señales ayudan a las personas a sentirse más seguras.

### Conceptos básicos del desarrollo de habilidades

La creación de una habilidad personalizada requiere la siguiente estructura:

```
my-skill/
├── SKILL.md          # Skill metadata (YAML front matter)
├── index.ts          # Skill entry point
├── agents/           # Supported agent configuration
└── references/       # Reference resources
```

Un ejemplo `SKILL.md` formato:

```yaml
---
name: my-skill
description: A brief description of what this skill does
---

# My Skill

Detailed documentation...
```

## Notas

1. **Requisitos de red**: la búsqueda e instalación de habilidades requieren acceso a `api.hagicode.com` y el registro npm
2. **Versión de Node.js**: se recomienda Node.js 18 o posterior
3. **Requisitos de permiso**: se requieren permisos globales de instalación de npm
4. **Control de concurrencia**: solo se puede ejecutar una operación de instalación o desinstalación para la misma habilidad a la vez
5. **Configuración del tiempo de espera**: el tiempo de espera predeterminado para la instalación es de 4 minutos, pero los escenarios complejos pueden requerir ajustes

Estas notas existen, en última instancia, para ayudar a que todo salga bien.

## Conclusión

Este artículo presentó la implementación completa del sistema de gestión de habilidades en el proyecto HagiCode. A través de una arquitectura separada frontend-backend, el patrón de adaptador, administración de estado distribuida basada en Orleans y técnicas relacionadas, el sistema ofrece:

- **Gestión global local**: una interfaz unificada de gestión de habilidades creada mediante envoltura `npx skills` comandos
- **Búsqueda en el mercado**: descubrimiento rápido de habilidades de la comunidad a través de la API en línea y mecanismos de almacenamiento en caché
- **Recomendaciones inteligentes**: recomendaciones de habilidades basadas en IA basadas en el contexto del proyecto
- **Gestión de confianza**: un sistema de configuración flexible que permite a los usuarios controlar los límites de confianza

Este enfoque de diseño no sólo es aplicable a la gestión de habilidades. También es útil como referencia para cualquier escenario que necesite integrar herramientas de línea de comandos y al mismo tiempo equilibrar el almacenamiento local y los servicios en línea.
