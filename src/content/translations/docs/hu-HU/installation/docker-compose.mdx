## Előfeltételek

Mielőtt elkezdené, győződjön meg arról, hogy a rendszeren telepítve van a Docker és a Docker Compose.

### Telepítse a Dockert

#### Windows

Töltse le és telepítse [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)

A telepítés után győződjön meg arról, hogy a Docker Desktop fut.

#### macOS

Töltse le és telepítse [Docker Desktop for Mac](https://www.docker.com/products/docker-desktop/)

A telepítés után győződjön meg arról, hogy a Docker Desktop fut.

#### Linux

Telepítse a Dockert a csomagkezelő segítségével:

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

A telepítés után indítsa el a Docker szolgáltatást:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Ellenőrizze a telepítést

A telepítés után futtassa a következő parancsokat a Docker és a Docker Compose megfelelő telepítésének ellenőrzéséhez:

```bash
docker --version
docker compose version
```

## Gyors kezdés

### 1. Hozzon létre Docker Compose konfigurációs fájlt

:::tipp[A konfigurációgenerátor használata]

- Intelligensen kezelheti a fájlengedélyekkel kapcsolatos problémákat

[🚀 Azonnal használja a generátort →](https://builder.hagicode.com/)
:::

1. Nyissa meg [Docker Compose Builder (önálló webhely)](https://builder.hagicode.com/)
2. Töltse ki a konfigurációt igényei szerint:
   - API-szolgáltató kiválasztása (Zhipu AI, Anthropic Official vagy Custom API)
   - Konfigurálja a portokat, az adatbázist, a munkakönyvtárat és egyéb beállításokat
   - **Képforrás kiválasztása**: Válassza ki a megfelelő képforrást a hálózati környezete alapján
3. Kattintson a generálás gombra a letöltéshez `docker-compose.yml` konfigurációt
4. Mentse el a létrehozott konfigurációt a `docker-compose.yml` fájlt
5. Ha szükséges, hozzon létre a `.env` fájl az érzékeny információk konfigurálásához

#### Képforrás kiválasztása

A generátor három képforrás opciót támogat:

**Docker Hub**
- **Kép címe**: `newbe36524/hagicode:{tag}`
- **Alkalmazandó forgatókönyvek**: Docker Hub tükörgyorsítással rendelkező felhasználók számára megfelelő
- **Előnyök**: Hivatalos képforrás, időszerű frissítések, stabil hozzáférés
- **Megjegyzések**: Előfordulhat, hogy egyes régiókban konfigurálni kell a tükörgyorsítást

**Azure Container Registry (ACR)**
- **Kép címe**: `hagicode.azurecr.io/hagicode:{tag}`
- **Alkalmazandó forgatókönyvek**: Olyan felhasználók számára megfelelő, akik nem tudják helyileg elérni a Docker Hubot
- **Előnyök**: Alternatív képforrást biztosít, megoldja a hálózati hozzáférési problémákat
- **Megjegyzések**: A képek szinkronban maradnak a Docker Hubbal, de előfordulhatnak rövid késések

**Alibaba Cloud Container Registry (ACR)**
- **Kép címe**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **Alkalmazandó forgatókönyvek**: Alkalmas hazai felhasználók számára, stabil és gyors hálózati hozzáférés
- **Előnyök**: Alibaba Cloud belföldi csomópontok, gyors hozzáférési sebesség, nagy stabilitás
- **Megjegyzések**: Az Alibaba ACR-képek szinkronban maradnak a Docker Hubbal, általában 30 percen belül

:::tipp[A képforrás kiválasztására vonatkozó ajánlások]
- **A belföldi felhasználók az Alibaba Cloud ACR-t ajánlják**: Ha Ön belföldi felhasználó, javasoljuk, hogy előnyben részesítse az Alibaba Cloud ACR képforrás használatát a gyors és stabil hozzáférés érdekében
- **Alternatív Docker Hub**: Ha a hálózati környezet hozzáfér a Docker Hubhoz, akkor képforrásként is választhatja a Docker Hubot.
- **Alternatív Azure ACR**: Csak akkor válassza az Azure ACR képforrást, ha más képforrások hozzáférési nehézségei vannak
- **Képszinkronizálás**: Az Alibaba Cloud ACR képei szinkronban maradnak a Docker Hubbal, általában 30 percen belül
- **Verziókonzisztencia**: A képcímkék konzisztensek mindhárom képforrásban, nem kell aggódnia a verzióval kapcsolatos problémák miatt
:::

### 2. Indítsa el a Szolgáltatásokat

Hajtsa végre a következő parancsot abban a könyvtárban, ahol a `docker-compose.yml` található:

```bash
docker compose up -d
```

Ez a parancs:
- Töltse le és indítsa el a generált sablon által meghatározott Hagicode-tárolókat
- Hálózat létrehozása és konfigurálása
- Inicializálja az alapértelmezett adatkönyvtárat és a futásidejű függőségeket

### 3. Ellenőrizze a szolgáltatás állapotát

Ellenőrizze, hogy futnak-e a tárolók:

```bash
docker compose ps
```

A generált sablonban definiált szolgáltatásokat "futó" állapotban kell látnia.

### Szolgáltatásnaplók megtekintése

Az összes szolgáltatási napló megtekintéséhez:

```bash
# View all service logs
docker compose logs

# View Hagicode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## Konfigurációs útmutató

Ha módosítania kell a konfigurációt (például portot kell változtatnia, API-szolgáltatót kell váltania stb.), a következőket teheti:

1. Új konfiguráció létrehozása a használatával [Docker Compose Builder (önálló webhely)](https://builder.hagicode.com/)
2. Kézi szerkesztés `docker-compose.yml` és `.env` fájlokat
3. Indítsa újra a szolgáltatásokat a konfiguráció érvénybe lépéséhez:

```bash
docker compose restart
```
