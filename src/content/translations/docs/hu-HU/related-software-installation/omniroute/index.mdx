Ez az oldal azoknak az első HagiCode-felhasználóknak szól, akik a legrövidebb OmniRoute beállítási utat szeretnék: indítsa el a helyi szolgáltatást, ellenőrizze, hogy `http://localhost:20128/v1` működik, és ellenőrizze, hogy az irányítópult és az API is elérhető-e.

:::tipp[Ajánlott elérési út]
Ez az első verzió először a legstabilabb útvonalat tartja meg: `npm install -g omniroute`, `omniroute`, Irányítópult at `http://localhost:20128`és az OpenAI-kompatibilis alap URL-címet a címen `http://localhost:20128/v1`. A Docker, a VPS és a fejlettebb telepítési módok az opcionális részben maradnak.
:::

## Előfeltételek

- A Node.js már telepítve van. Teljes [A Node.js telepítése](/en/related-software-installation/nodejs/installation) először, ha szükséges.
- Helyi kikötő `20128` elérhető és még nincs foglalt.
- Legalább egy upstream modellszolgáltatói fiókja van, amelyet az OmniRoute-on keresztül kíván irányítani.
- A HagiCode, egy CLI vagy egy SDK integrálása előtt tartsa külön ezt a két címet:
  - Irányítópult: `http://localhost:20128`
  - OpenAI-kompatibilis alap URL: `http://localhost:20128/v1`

## Telepítse és futtassa helyben

Kezdje a javasolt globális npm elérési úttal:

```bash
npm install -g omniroute
omniroute
```

A szolgáltatás elindítása után érvényesítse a következő sorrendben:

1. Nyissa meg `http://localhost:20128` böngészőben, és ellenőrizze, hogy az irányítópult betöltődik.
2. Menj a `Endpoints` oldalt az irányítópulton, és hozzon létre vagy másoljon API-kulcsot.
3. Jegyezze fel az alap URL-t, amelyet később át fog adni az eszközöknek: `http://localhost:20128/v1`.

:::note[Miért érdemes az irányítópulttal kezdeni]
Az első beállításnál a Dashboard a leggyorsabb módja annak, hogy megbizonyosodjon arról, hogy az OmniRoute fut, a szolgáltatók csatlakoznak, és létezik API-kulcs. Ha egy későbbi ügyfélhívás sikertelen, az irányítópult első ellenőrzése általában gyorsabb, mint az SDK-kód azonnali hibakeresése.
:::

## Érvényesítse a futó szolgáltatást

### 1. érvényesítés: ellenőrizze, hogy a modellkatalógus olvasható-e

Helyezze be a kulcsot a `Endpoints` oldalt egy környezeti változóba:

```bash
export OMNIROUTE_API_KEY="replace-with-the-key-you-copied-from-the-Endpoints-page"
```

Akkor kérj `/v1/models`:

```bash
curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}"
```

Ha visszakap egy modelllistát, az OpenAI-kompatibilis végpont és hitelesítési folyamat működik.

### 2. érvényesítés: minimális csevegési kérés küldése

```bash
curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "replace-with-a-model-id-from-v1-models",
    "messages": [
      { "role": "user", "content": "Reply with ok." }
    ]
  }'
```

Itt nem a gyors minőség a cél. Ez annak bizonyítása, hogy a választott modell már sikeresen átirányítható az OmniRoute-on.
