Ez az útmutató a Node.js telepítését ismerteti Windows, macOS és Linux rendszereken. A Node.js a HagiCode helyi eszközeinek kulcsfontosságú függősége, beleértve az OpenSpec-et, a frontend szerszámokat és számos fejlesztői CLI-t.

:::tipp[Verziókövetelmények]
A HagiCode a **Node.js 24+**-ot ajánlja (minimálisan támogatott Node.js >= 18.0). A legtöbb esetben telepítse a legújabb LTS-verziót, majd ellenőrizze, hogy verziója megfelel-e a követelményeknek.
:::

## A Node.js telepítése

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Első módszer: Hivatalos telepítő (ajánlott)

1. Látogassa meg a [Node.js hivatalos webhely](https://nodejs.org/)
2. Töltse le a **LTS (Long Term Support)** verziójú Windows telepítőt (`.msi` fájl)
3. Kattintson duplán a telepítő futtatásához
4. A telepítés befejezéséhez kövesse a telepítővarázsló utasításait:
   - Fogadja el a licencszerződést
   - Válassza ki a telepítési útvonalat (az alapértelmezés megfelelő)
   - Győződjön meg arról, hogy "A szükséges eszközök automatikus telepítése" be van jelölve
   - Kattintson a "Telepítés" gombra a telepítés megkezdéséhez
5. A telepítés befejezése után kattintson a "Befejezés" gombra

### Második módszer: A szárny használata

Nyissa meg a PowerShellt vagy a Parancssort, és futtassa:

```powershell
winget install OpenJS.NodeJS.LTS
```

A telepítés után nyissa meg újra a terminált, hogy a változtatások életbe lépjenek.

  </TabItem>
  <TabItem value="macos" label="macOS">

### Első módszer: Hivatalos telepítő (ajánlott)

1. Látogassa meg a [Node.js hivatalos webhely](https://nodejs.org/)
2. Töltse le a **LTS (Long Term Support)** verziójú macOS telepítőt (`.pkg` fájl)
3. Kattintson duplán a telepítő futtatásához
4. A telepítés befejezéséhez kövesse a telepítővarázsló utasításait:
   - Kattintson a "Tovább" gombra
   - Fogadja el a licencszerződést
   - Válassza ki a telepítőlemezt
   - Kattintson a "Telepítés" gombra, és adja meg a rendszergazdai jelszót
5. A telepítés befejezése után kattintson a "Bezárás" gombra

### Második módszer: A Homebrew használata

Ha van [Homebrew](https://brew.sh/) telepítve, a Node.js terminálon keresztül telepíthető:

```bash
brew install node
```

:::tipp
A Homebrew automatikusan hozzáadja a Node.js fájlt a PATH-hoz. Ha a parancs nem érhető el, indítsa újra a terminált, vagy futtassa:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

  </TabItem>
  <TabItem value="ubuntu" label="Ubuntu/Debian">

### NodeSource Repository használata (ajánlott)

Ez a legjobb módja a Node.js legújabb verziójának beszerzésének:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Apt Package Manager használatával

Ha nincs szüksége a legújabb verzióra, használhatja az alapértelmezett Ubuntu tárolót:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

::: Megjegyzés
Az alapértelmezett lerakatban található Node.js verzió régebbi lehet. A telepítés után ellenőrizze, hogy a verzió megfelel-e a Node.js >= 18.0 követelményeinek (a Node.js 24+ ajánlott).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### NodeSource Repository használata (ajánlott)

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### A yum csomagkezelő használata

```bash
sudo yum install -y nodejs npm
```

  </TabItem>
  <TabItem value="fedora" label="Fedora">

### A dnf csomagkezelő használata

```bash
sudo dnf install -y nodejs npm
```

:::tipp
A Fedora szoftvertára általában újabb Node.js verziókat tartalmaz, és közvetlenül a dnf-en keresztül telepíthető.
:::

  </TabItem>

## Verziókezelő eszközök használata

Ha több Node.js-verzió között kell váltania, használhat verziókezelő eszközöket.

### nvm (Node Version Manager)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

A telepítés után töltse be újra a shell konfigurációját:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

Windows rendszeren a használatát javasoljuk [nvm-windows](https://github.com/coreybutler/nvm-windows):

1. Töltse le a legújabbat `nvm-setup.exe` től [nvm-windows kiadások](https://github.com/coreybutler/nvm-windows/releases)
2. Futtassa a telepítőt
3. Nyissa meg újra a Parancssort vagy a PowerShellt

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

Az nvm telepítése után használhatja a Node.js verzióinak telepítésére és kezelésére:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (Fast Node Manager)

Az fnm egy gyorsabb és egyszerűbb Node.js verziókezelő eszköz, amely Rusttal készült.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

Használja [gombóc](https://scoop.sh/) vagy [szárnyas](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Gyorsítás a szárazföldi kínai felhasználók számára

Ha Kínában tartózkodik, a Node.js és az npm csomagok letöltése lassú lehet. Íme néhány gyorsítási megoldás.

### Hazai tükörforrások használata

#### npm Taobao Mirror (ajánlott)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### Az nrm használata a tükörforrások kezelésére

Az nrm (npm registry manager) segít gyorsan váltani az npm tükrözési források között:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### A Node.js telepítő letöltéseinek felgyorsítása

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

A Taobao tükörletöltéseket biztosít a Node.js telepítői számára:

- **Letöltési cím**: https://npmmirror.com/mirrors/node/

Látogassa meg a fenti címet, és válassza ki a letölteni kívánt verziót.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

Ha az nvm vagy más eszközök letöltése a GitHubról lassú, használhatja a FastGit tükröt:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

::: Megjegyzés
A FastGit szolgáltatás változhat, kérjük tekintse meg a hivatalos információkat.
:::

  </TabItem>
</Tabs>

### Háztartási Homebrew tükrök

Ha a Homebrew-t Kínában használja, beállíthatja a hazai tükröket a gyorsításhoz:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Kínai specifikus csomagkezelők

#### cnpm

A cnpm a Taobao csapata által biztosított npm kliens, amely alapértelmezés szerint a Taobao tükröt használja:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::tipp
A cnpm többnyire kompatibilis az npm parancsokkal, és közvetlenül helyettesíthető.
:::

### Tükörgyorsulási hatás ellenőrzése

A tükrök beállítása után ellenőrizze a következőket:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Telepítés ellenőrzése

A telepítés után ellenőrizze, hogy a Node.js és az npm megfelelően van-e telepítve:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

::: Megjegyzés
Győződjön meg arról, hogy a Node.js verziója >= 18.0 (a Node.js 24+ ajánlott). Ha a verzió túl alacsony, kérjük, telepítse újra a fenti módszerekkel, vagy használjon verziókezelő eszközöket a megfelelő verzió telepítéséhez.
:::

### Minta ellenőrzési kimenet

A sikeres ellenőrző kimenetnek így kell kinéznie:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Hibaelhárítás

### Windows problémák

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Ha `node` vagy `npm` a parancsok nem érhetők el a telepítés után:

1. Indítsa újra a Parancssort vagy a PowerShellt
2. Ha még mindig nem érhető el, manuálisan adja hozzá a Node.js fájlt a PATH-hoz:
   - Keresse meg a "Környezeti változók" kifejezést, és nyissa meg a "Rendszerkörnyezeti változók szerkesztése" lehetőséget.
   - Kattintson a "Környezeti változók" elemre
   - Keresse meg az „Elérési út” részt a „Rendszerváltozók” részben, és kattintson a „Szerkesztés” gombra.
   - Node.js telepítési útvonal hozzáadása (alapértelmezett: `C:\Program Files\nodejs\`)
   - Kattintson az "OK" gombra a módosítások mentéséhez

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Ha végrehajtási házirend-hibákat észlel az nvm használata során:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### macOS problémák

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Ha engedélyezési hibákat észlel a globális npm-csomagok telepítésekor:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Ha a telepítés után nem állnak rendelkezésre parancsok, indítsa újra a terminált, vagy manuálisan adja hozzá a PATH-hoz (általában in `/usr/local/bin`).

  </TabItem>
</Tabs>

### Linux problémák

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Ha a globális npm-csomagok telepítésekor engedélyhibákat észlel, a rendszercsomag-kezelők helyett a Node.js verziókezelő eszközök (például nvm vagy fnm) használata javasolt.

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Ha a rendszercsomagkezelőn keresztül telepített Node.js verzió túl régi:

1. NodeSource tároló használata (ajánlott, lásd fent)
2. Vagy használjon verziókezelő eszközöket (nvm/fnm)

  </TabItem>
</Tabs>

### Környezet ellenőrzése

Ha az ellenőrzés sikertelen, ellenőrizze a következőket:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
