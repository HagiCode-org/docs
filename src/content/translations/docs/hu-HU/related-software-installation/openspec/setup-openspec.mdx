# Az OpenSpec telepítése

Az OpenSpec az ajánlatokat, a specifikációkat és a változásrekordokat kezeli. Ez az oldal egy célt szolgál: világosan magyarázza el a telepítési útvonalat, hogy csatolhassa azt meglévő munkafolyamatához.

## Mi az az OpenSpec?

Az OpenSpec egy parancssori eszköz az ajánlatok és specifikációk kezelésére a szoftvermunkában. Az alapvető munkafolyamat a következőket tartalmazza:

- **Ajánlatkezelés**: Funkciójavaslatok létrehozása, áttekintése és nyomon követése
- **A specifikáció érvényesítése**: Győződjön meg arról, hogy a kódmódosítások megfelelnek az előre meghatározott specifikációknak
- **Változáskövetés**: Rögzíti az összes felülvizsgált változástörténetet
- **Csapat együttműködés**: Többszemélyes együttműködési fejlesztés támogatása

### Alkalmazás a HagiCode projektekben

A HagiCode projektekben az OpenSpec a következőkre használatos:

1. **Funkciófejlesztés kezelése**: Minden új funkciónak át kell mennie az OpenSpec javaslattételi folyamatán
2. **Kód felülvizsgálata**: Győződjön meg arról, hogy a változtatások megfelelnek a projekt specifikációinak
3. **Dokumentáció szinkronizálása**: Fenntartja a következetességet a dokumentáció és a kód implementációja között
4. **Verziófelügyeleti integráció**: Zökkenőmentes integráció a verziófelügyeleti munkafolyamatokkal

## Előfeltételek

Az OpenSpec telepítése előtt győződjön meg arról, hogy rendszere megfelel a következő követelményeknek:

### Node.js és npm

Az OpenSpec npm-en keresztül terjeszthető, és Node.js környezetre van szükség. Az OpenSpec-hez **Node.js 18.0 vagy újabb** szükséges.

A Node.js telepítésének ellenőrzése:

```bash
node --version
```

Az npm telepítés ellenőrzése:

```bash
npm --version
```

Ha nincs telepítve, vagy a verzió túl régi, kérjük, olvassa el a kapcsolódó szoftvertelepítési utasításokat a [Telepítési útmutató](/en/installation).

## Az OpenSpec telepítése

:::tipp[Verziókövetelmények]
Ehhez a projekthez az **OpenSpec 1.0.0 vagy újabb** verziótartomány használatára van szükség. A projekt munkafolyamatával való kompatibilitás biztosítása érdekében minden 1.x verzió támogatott.
:::

Az OpenSpec az npm-en keresztül globálisan telepíthető, így a rendszer bárhonnan elérhető.

### Globális telepítés az npm használatával

<Tabs>
  <TabItem value="win" label="Windows">
    Futtassa a PowerShellben vagy a parancssorban:

```powershell
npm install -g @fission-ai/openspec@1
```

:::note[Windows engedélyekkel kapcsolatos problémák]
Ha a telepítés során engedélyhibákat észlel, futtassa a PowerShellt vagy a Command Promptot rendszergazdaként.
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    Futtatás a terminálban:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[macOS engedélyekkel kapcsolatos problémák]
Ha találkozik `EACCES` engedélyezési hibák, próbálkozzon:
```bash
sudo npm install -g @fission-ai/openspec@1
```
Vagy használja [nvm](https://github.com/nvm-sh/nvm) a Node.js verziók kezeléséhez és az engedélyekkel kapcsolatos problémák elkerüléséhez.
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    Futtatás a terminálban:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[Linux engedélyekkel kapcsolatos problémák]
Ha találkozik `EACCES` engedélyezési hibák esetén:
1. Használja `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. Az npm konfigurálása felhasználói könyvtár használatához:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### Ellenőrizze a telepítést

A telepítés után ellenőrizze, hogy az OpenSpec megfelelően van-e telepítve, és a verzió megfelel-e a követelményeknek:

```bash
openspec --version
```

Sikeres telepítés esetén megjelenik egy verziószám kimenet, például:

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::note[Verziótartomány]
Győződjön meg arról, hogy a telepített OpenSpec verzió 1.0.0 vagy újabb. A projekt igazolta a kompatibilitást az 1.1.1-es verzióval. Az új funkciók eléréséhez javasolt a legújabb kompatibilis verzió használata.
:::

## Gyakori problémák és hibaelhárítás

### Engedélyezési hibák

#### Probléma: `EACCES` Engedély megtagadva

Engedélyezési hibák az OpenSpec telepítésekor vagy futtatásakor.

**Megoldások**:

<Tabs>
  <TabItem value="win" label="Windows">
    Futtassa a PowerShellt vagy a Parancssort rendszergazdaként:
    1. Kattintson a jobb gombbal a PowerShellre vagy a Parancssorra
    2. Válassza a "Futtatás rendszergazdaként" lehetőséget
    3. Futtassa újra a telepítési parancsot
  </TabItem>
  <TabItem value="mac" label="macOS">
    Telepítés segítségével `sudo`:

```bash
sudo npm install -g @fission-ai/openspec@1
```

Vagy használja [nvm](https://github.com/nvm-sh/nvm) a Node.js kezelésére és a globális telepítési engedélyekkel kapcsolatos problémák elkerülésére.
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. megoldás: Használja a sudo-t
```bash
sudo npm install -g @fission-ai/openspec@1
```

2. megoldás: Konfigurálja az npm-et a felhasználói könyvtár használatára
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @fission-ai/openspec@1
```
  </TabItem>
</Tabs>

### Hálózati problémák

#### Probléma: npm A telepítés sikertelen vagy nagyon lassú

Hálózati okok miatt nem lehet letölteni az OpenSpec-et az npm rendszerleíró adatbázisból.

**Megoldások**:

1. **A belföldi npm tükör használata**:

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **A hivatalos forrás visszaállítása** (ha szükséges):

```bash
npm config set registry https://registry.npmjs.org
```

3. **Használja a cnpm-et** (opcionális):

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

:::tipp
A szárazföldi kínai felhasználók számára a Taobao tükör használatát javasoljuk, amely jelentősen javíthatja a letöltési sebességet.
:::

### Verziókompatibilitási problémák

#### Probléma: Az OpenSpec-parancsok rendellenesen futnak, vagy hibákat jelentenek

A telepített OpenSpec verzió nem kompatibilis a jelenlegi környezettel.

**Megoldások**:

1. **Ellenőrizze a Node.js verzióját**:

```bash
node --version
```

Győződjön meg arról, hogy a verzió 18.0 vagy újabb. Ha a verzió túl alacsony, frissítse a Node.js-t.

2. **Ellenőrizze az OpenSpec verzióját**:

```bash
openspec --version
```

3. **Az adott verzió újratelepítése**:

```bash
npm install -g @fission-ai/openspec@1
```

4. **Npm gyorsítótár törlése** (ha a probléma továbbra is fennáll):

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### A parancs nem található

#### Probléma: gépelés `openspec` A parancs azt mutatja, hogy "A parancs nem található"

Az OpenSpec telepítve van, de a rendszer nem találja a parancsot.

**Megoldások**:

<Tabs>
  <TabItem value="win" label="Windows">
    1. Erősítse meg az npm globális telepítési útvonalat:

```powershell
npm config get prefix
```

2. Útvonal hozzáadása a PATH rendszerhez:
   - Nyissa meg a "Rendszer tulajdonságai" → "Speciális" → "Környezeti változók" lehetőséget.
   - Keresse meg `Path` a "Rendszerváltozókban"
   - Adja hozzá az npm globális elérési utat (általában `C:\Users\<username>\AppData\Roaming\npm`)
   - Indítsa újra a parancssori ablakot
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. Erősítse meg az npm globális telepítési útvonalat:

```bash
npm config get prefix
```

2. Útvonal hozzáadása a PATH-hoz (in `~/.zshrc` vagy `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Konfiguráció újratöltése:

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. Erősítse meg az npm globális telepítési útvonalat:

```bash
npm config get prefix
```

2. Útvonal hozzáadása a PATH-hoz (in `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Konfiguráció újratöltése:

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
