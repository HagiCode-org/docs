## Háttér

A Hagicode projektben a felhasználók többféle CLI-eszköz közül választhatnak az AI programozási asszisztensek meghajtásához, beleértve a Claude Code CLI-t, a GitHub Copilot-ot, az OpenCode CLI-t, a Codebuddy CLI-t, a Hermes CLI-t és még sok mást. Ezek a CLI-eszközök önmagukban általános célú mesterségesintelligencia-programozási eszközök, de a Hagicode absztrakciós rétegén keresztül rugalmasan kapcsolódhatnak különböző AI-modell-szolgáltatókhoz.

A Zhipu AI (ZAI) az Anthropic Claude API-val kompatibilis interfészt biztosít, lehetővé téve ezeknek a CLI-eszközöknek a hazai GLM sorozatú modellek közvetlen használatát. Közülük a GLM-5.1 a Zhipu legújabb nagy nyelvi modellje, jelentős fejlesztésekkel a GLM-5.0-hoz képest.

## A Hagicode CLI absztrakciós architektúrája

A Hagicode 11 CLI-szolgáltatótípust határoz meg a `AIProviderType` enum, amely a főbb AI programozási CLI eszközöket fedi le:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Minden CLI rendelkezik a megfelelő modellparaméter-konfigurációval, és támogatja a `model` és `reasoning` paraméterek:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],
        [AIProviderType.CodexCli] = ["model", "reasoning"],
        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],

        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],

        // ...
    };
```

## Using GLM-5.1 in different CLIs

### 1. Claude Code CLI + GLM-5.1

A Claude Code CLI a Hagicode egyik alapvető CLI-je, és a Hero konfigurációs rendszeren keresztül konfigurálható:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Megfelelő `HeroEquipmentCatalogItem` konfiguráció:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

Az OpenCode CLI a legrugalmasabb CLI, és támogatja bármely modell megadását a `provider/model` formátum:

**1. módszer: Használja a ZAI szolgáltató előtagját**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Method 2: Use the model ID directly**
```json
{
  "model": "glm-5.1"
}
```

**3. módszer: Frontend konfigurációs felhasználói felület**

In `HeroModelEquipmentForm.tsx`, Az OpenCode CLI-nek van egy dedikált helyőrző tippje:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

A felhasználók beírhatják:
```
zai/glm-5.1
glm-5.1
```

**OpenCode CLI modell elemzési logika**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

A Codebuddy CLI dedikált örökölt kezelési logikával rendelkezik:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Megjegyzés**: A Codebuddy megtartja a GLM-5.0 speciális kezelését, és nem használ örökölt normalizálást:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## A GLM-5.1 fejlesztési előnyei

A GLM-5.0-hoz képest a GLM-5.1 a következő jelentős fejlesztéseket hozza:

### 1. Jobb érvelési képesség

A Zhipu hivatalos kiadási információi szerint a GLM-5.1 fejlesztései a következők:

- **Erősebb kódértés**: Az összetett kódstruktúrák pontosabb elemzése
- **Hosszabb kontextusértés**: Támogatja a hosszabb társalgási kontextust
- **Továbbfejlesztett eszközhívás**: Magasabb sikerességi arány az MCP-eszközhívásoknál
- **Kimeneti stabilitás**: Csökkenti a véletlenszerűséget és a hallucinációkat

### 2. Átfogó több CLI-kompatibilitás

A GLM-5.1 a Hagicode által támogatott összes mainstream CLI-t lefedi:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Megjegyzések

### 1. API kulcs konfiguráció

Győződjön meg arról, hogy a `ANTHROPIC_AUTH_TOKEN` környezeti változó megfelelően van beállítva. Ez a szükséges hitelesítő adat minden CLI számára a modellhez való csatlakozáshoz.

### 2. A modell elérhetősége

A GLM-5.1-et a megfelelő modellszolgáltatónak engedélyeznie kell:
- The Zhipu AI ZAI platform supports it by default
- Az Alibaba Cloud DashScope külön alkalmazást igényelhet

### 3. OpenCode CLI formátum

Amikor a `provider/model` format, make sure the provider ID is correct:
- Zhipu AI: `zai` vagy `zhipuai`
- Alibaba felhő: `aliyun` vagy `dashscope`

### 4. Érvelési paraméter

- `high` ajánlott a legjobb kódgenerálási eredmények érdekében
- A Gemini CLI nem támogatja az érvelési paramétert, és automatikusan figyelmen kívül hagyja ezt a konfigurációt

## Következtetés

Az egységes absztrakciós rétegen keresztül a Hagicode rugalmas integrációt tesz lehetővé a GLM-5.1 és több CLI között. A fejlesztők kiválaszthatják a preferenciáiknak és használati forgatókönyveiknek leginkább megfelelő CLI-eszközt, majd az egyszerű konfigurációval a legújabb GLM-5.1 modellt használhatják.

A Zhipu legújabb modellváltozataként a GLM-5.1 egyértelmű fejlesztéseket kínál a GLM-5.0-hoz képest:
- Független verzióazonosító, örökölt teher nélkül
- Erősebb érvelés és kódértés
- Széleskörű multi-CLI kompatibilitás
- Rugalmas érvelési szintű konfiguráció

A megfelelő környezeti változókkal és a konfigurált Hero berendezésekkel a felhasználók teljes mértékben kiaknázhatják a GLM-5.1 erejét a különböző CLI környezetekben.
