3. **Az interakciónak természetesnek kell lennie**: A modern AI-asszisztenseknek támogatniuk kell a szöveget, a hangot és a képeket. A felhasználóknak meg kell tudniuk választani a legtermészetesebb beviteli módot.

## Elemzés

### Technikai kihívások a beszédfelismerésben

A beszédfelismerés létrehozásakor egy trükkös problémába ütköztünk: **a böngésző WebSocket API nem támogatja az egyéni HTTP-fejléceket**.

A beszédfelismerő szolgáltatást a ByteDance Doubao Speech Recognition API-ja választottuk. Sajnos ehhez az API-hoz hitelesítési információkra van szükség, mint pl `accessToken` és `secretKey` HTTP-fejléceken keresztül kell átadni. Ez azonnali technikai konfliktust okozott:

```javascript
// The browser WebSocket API does not support this approach
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // Not supported
  }
});
```

Alapvetően két lehetőségünk volt:

1. **URL-lekérdezési paraméter megközelítés**: helyezze el a hitelesítési információkat az URL-ben
   - Előnye, hogy egyszerűen kivitelezhető
   - Hátránya, hogy a hitelesítő adatok ki vannak téve a frontendnek, ami nem biztonságos; egyes API-k fejléc alapú hitelesítést is igényelnek

2. **Háttérproxy megközelítés**: WebSocket proxy megvalósítása a háttérben
   - Előnye, hogy a hitelesítő adatok biztonságosan tárolva maradnak a háttérben, és a megoldás teljes mértékben kompatibilis az API követelményeivel
   - Hátránya, hogy a megvalósítás egy kicsit bonyolultabb

Végül a **backend proxy megközelítést** választottuk. A biztonság nem olyan dolog, amiben kompromisszumot kötsz.

### A képfeltöltés funkcionális követelményei

A képfeltöltési követelményeink valójában meglehetősen egyszerűek voltak:

1. **Több feltöltési mód**: kattintson a fájl kiválasztásához, húzza át és illessze be a vágólapról
2. **Fájlellenőrzés**: a típuskorlátozás (PNG, JPG, WebP, GIF) és a méretkorlátozás (5-10 MB) alapvető követelmény
3. **Felhasználói élmény**: Feltöltés folyamata, előnézetek és hibaüzenetek, hogy a felhasználók mindig tudják, mi történik
4. **Biztonság**: elengedhetetlen a szerveroldali ellenőrzés és a rosszindulatú fájlfeltöltések elleni védelem

## Megoldás

### Beszédfelismerés: WebSocket Proxy Architecture

Háromrétegű architektúrát terveztünk a beszédfelismeréshez, és találtunk egy működő utat:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**Alapkomponensek megvalósításai**:

1. **Frontend AudioWorklet processzor**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Resample to 16 kHz (required by the Doubao API)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Accumulate samples into 500 ms chunks
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Convert to 16-bit PCM and send
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **Háttér WebSocket kezelő** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **Elülső `VoiceTextArea` komponens**:

```tsx
export const VoiceTextArea = forwardRef<HTMLTextAreaElement, VoiceTextAreaProps>(
  ({ value, onChange, onTextRecognized, maxDuration }, ref) => {
    const { isRecording, interimText, volume, duration, startRecording, stopRecording } =
      useVoiceRecording({ onTextRecognized, maxDuration });

    return (
      <div className="flex gap-2">
        {/* Voice button */}
        <button onClick={handleButtonClick}>
          {isRecording ? <VolumeWaveform volume={volume} /> : <Mic />}
        </button>
        {/* Text input area */}
        <textarea value={displayValue} onChange={handleChange} />
      </div>
    );
  }
);
```

### Képfeltöltések: több módszeres feltöltési komponens

Készítettünk egy teljes értékű képfeltöltési összetevőt, amely mindhárom feltöltési módot támogatja, és lefedi a felhasználók leggyakoribb forgatókönyveit.

**Alapvető jellemzők**:

1. **Három feltöltési mód**:

```tsx
// Click to upload
const handleClick = () => fileInputRef.current?.click();

// Drag-and-drop upload
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Clipboard paste
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **Elülső ellenőrzés**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **Háttérfeltöltés-kezelő** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Validation
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Save file
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```
