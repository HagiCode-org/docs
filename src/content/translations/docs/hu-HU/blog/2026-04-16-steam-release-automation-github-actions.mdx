## A Steam-kiadások automatizálása GitHub-műveletekkel

> Ez a cikk megosztja azt a teljes megoldást, amelyet a HagiCode Desktop projektben az automatizált Steam kiadásokhoz vezettünk be, és lefedi a GitHub Release-től a Steam platformig terjedő automatizálási folyamatot, beleértve a kulcsfontosságú technikai részleteket, például a Steam Guard hitelesítést és a többplatformos Depot-feltöltéseket.

## Háttér

A Steam kiadási munkafolyamata valójában teljesen eltér a hagyományos alkalmazásterjesztéstől. A Steam saját, teljes frissítési rendszerrel rendelkezik. A fejlesztőknek a SteamCMD-t kell használniuk az összeállítási műtermékek Steam CDN-hálózatára való feltöltéséhez, ahelyett, hogy egyszerűen egy letöltési linket helyeznének el, mint más platformokon.

A HagiCode Desktop projekt Steam kiadásra készül, amely néhány új kihívást jelentett a kiadási munkafolyamatban:

1. A meglévő build-termékeket Steam-kompatibilis formátumba kellett konvertálnunk.
2. Fel kellett töltenünk őket a Steambe a SteamCMD-n keresztül.
3. A Steam Guard hitelesítést is kezelnünk kellett.
4. Támogatnunk kellett a többplatformos Depot-feltöltést Linux, Windows és macOS rendszerekhez.
5. Teljesen automatizált áramlást akartunk a GitHub Release-ről a Steamre.

A projekt már bevezette a "hordozható verzió módot", amely lehetővé teszi az alkalmazás számára, hogy észlelje a fix szolgáltatásba csomagolt hasznos terheket `extra` könyvtárat. Célunk az volt, hogy ezt a hordozható verzió módot zökkenőmentesen integráljuk a Steam disztribúcióval.

## A HagiCode-ról

A cikkben megosztott megoldás a gyakorlati tapasztalatainkból származik [HagiCode](https://hagicode.com) projekt. A HagiCode egy mesterséges intelligencia kódolási asszisztens, amely támogatja az asztali számítógépek használatát. Mivel aktívan dolgozunk a Steamen való elindításon, megbízható, automatizált kiadási munkafolyamatot kellett létrehoznunk.

## Építészeti tervezés

A Steam teljes kiadási folyamatának lényege egy GitHub Actions munkafolyamat, amely három fő szakaszra osztja a folyamatot:

```
┌─────────────────────────────────────────────────────────────┐
│ GitHub Actions Workflow (Steam Release)                    │
├─────────────────────────────────────────────────────────────┤
│ 1. Preparation Stage:                                      │
│    - Check out portable-version code                       │
│    - Download build artifacts from GitHub Release          │
│    - Extract and prepare the Steam content directory       │
│                                                             │
│ 2. SteamCMD Setup:                                         │
│    - Install or reuse SteamCMD                             │
│    - Authenticate with Steam Guard                         │
│                                                             │
│ 3. Release Stage:                                          │
│    - Generate Depot VDF configuration files                │
│    - Generate App Build VDF configuration files            │
│    - Invoke SteamCMD to upload to Steam                    │
└─────────────────────────────────────────────────────────────┘
```

Ez a kialakítás számos előnnyel jár:
- Újra felhasználja a meglévő GitHub Release melléktermékeket, és elkerüli ugyanazon kimenetek újraépítését.
- Saját üzemeltetésű futókat használ a biztonságos elszigeteléshez.
- Támogatja az előnézeti mód és a formális kiadási ágak közötti váltást.
- Teljes hibakezelést és naplózást foglal magában, ami megkönnyíti a hibák diagnosztizálását.

## Munkafolyamat végrehajtása

### Trigger paraméterek tervezése

Munkafolyamatunk a következő kulcsparamétereket támogatja:

```yaml
inputs:
  release:           # Portable Version release tag
    description: 'Version tag to release (for example v1.0.0)'
    required: true
  steam_preview:     # Whether to generate a preview build
    description: 'Whether to use preview mode'
    required: false
    default: 'false'
  steam_branch:      # Steam branch to set live
    description: 'Target Steam branch'
    required: false
    default: 'preview'
  steam_description: # Build description override
    description: 'Build description'
    required: false
```

### Saját üzemeltetésű Runner konfiguráció

Biztonsági okokból saját üzemeltető futót használunk a `steam` címke:

```yaml
runs-on:
  - self-hosted
  - Linux
  - X64
  - steam
```

Ez biztosítja, hogy a Steam kiadások egy dedikált futtatón fussanak, és biztonságosan elkülönítve tartják az érzékeny hitelesítő adatokat.

### Egyidejűség ellenőrzése

Annak elkerülése érdekében, hogy ugyanazon verzió kiadásai zavarják egymást, konfiguráltuk a párhuzamosság vezérlését:

```yaml
concurrency:
  group: portable-version-steam-${{ github.event.inputs.release }}
  cancel-in-progress: false
```

Vedd észre `cancel-in-progress: false` itt van beállítva, mert a Steam kiadások eltarthatnak egy ideig, és nem akarjuk, hogy egy újonnan elindított futtatás megszakítsa a már feltöltöttet.

## Core Script implementáció

### Kiadási bemenetek előkészítése

A `prepare-steam-release-input.mjs` A script felelős a kiadáshoz szükséges bemenetek előkészítéséért:

```javascript
// Download the GitHub Release build manifest and artifact inventory
const buildManifest = await downloadBuildManifest(releaseTag);
const artifactInventory = await downloadArtifactInventory(releaseTag);

// Download compressed archives for each platform
for (const platform of ['linux-x64', 'win-x64', 'osx-universal']) {
  const artifactUrl = getArtifactUrl(artifactInventory, platform);
  await downloadArtifact(artifactUrl, platform);
}

// Extract into the Steam content directory structure
await extractToSteamContent(sources, contentRoot);
```

### Steam Guard hitelesítés

A Steam megköveteli, hogy a fiókokat Steam Guarddal védjék, ezért egy megosztott titkos alapú kódgeneráló algoritmust implementáltunk:

```javascript
function generateSteamGuardCode(sharedSecret, timestamp = Date.now()) {
  const secret = decodeSharedSecret(sharedSecret);
  const time = Math.floor(timestamp / 1000 / 30);

  const timeBuffer = Buffer.alloc(8);
  timeBuffer.writeBigUInt64BE(BigInt(time));

  // Use HMAC-SHA1 to generate a time-based one-time code
  const hash = crypto.createHmac('sha1', secret)
    .update(timeBuffer)
    .digest();

  // Convert it into a 5-character Steam Guard code
  const code = steamGuardCode(hash);
  return code;
}
```

Ez a megvalósítás a Steam Guard TOTP (Time-based One-Time Password) mechanizmusán alapul, amely 30 másodpercenként új ellenőrző kódot generál.

### VDF konfiguráció generálása

A VDF (Valve Data Format) a Steam által használt konfigurációs formátum. Kétféle VDF fájlt kell létrehoznunk:

A **Depot VDF** a tartalom konfigurálására szolgál minden platformon:

```javascript
function buildDepotVdf(depotId, contentRoot) {
  return [
    '"DepotBuildConfig"',
    '{',
    `  "DepotID" "${escapeVdf(depotId)}"`,
    `  "ContentRoot" "${escapeVdf(contentRoot)}"`,
    '  "FileMapping"',
    '  {',
    '    "LocalPath" "*"',
    '    "DepotPath" "."',
    '    "recursive" "1"',
    '  }',
    '}'
  ].join('\n');
}
```

**App Build VDF** a teljes alkalmazás build konfigurálására szolgál:

```javascript
function buildAppBuildVdf(appId, depotBuilds, description, setLive) {
  const vdf = [
    '"appbuild"',
    '{',
    `  "appid" "${appId}"`,
    `  "desc" "${escapeVdf(description)}"`,
    `  "contentroot" "${escapeVdf(contentRoot)}"`,
    '  "buildoutput" "build_output"',
    '  "depots"',
    '  {'
  ];

  for (const [depotId, depotVdfPath] of Object.entries(depotBuilds)) {
    vdf.push(`    "${depotId}" "${depotVdfPath}"`);
  }

  if (setLive) {
    vdf.push(`  }`);
    vdf.push(`  "setlive" "${setLive}"`);
  }

  vdf.push('}');
  return vdf.join('\n');
}
```

### SteamCMD meghívás

Végül a feltöltés a SteamCMD meghívásával történik:

```javascript
await runCommand(steamcmdPath, [
  '+login', steamUsername, steamPassword, steamGuardCode,
  '+run_app_build', appBuildPath,
  '+quit'
]);
```

Ez az utolsó ugrás az egész munkafolyamatban. Ha ez sikerült, a kiadás megtörtént.

## Többplatformos raktárkezelés

A Steam a Depot rendszert használja a tartalom kezelésére különböző platformokon. Három fő depót támogatunk:

| Platform | Depó azonosító | Építészeti támogatás |
|-----|-----------|---------|
| Linux | `linux-x64` | x64_64 |
| Windows | `win-x64` | x64_64 |
| macOS | `osx-universal` | univerzális, x64_64, kar64 |

Minden Depot saját tartalomkönyvtárral és VDF konfigurációs fájllal rendelkezik. Ez biztosítja, hogy a különböző platformokon a felhasználók csak a ténylegesen szükséges tartalmat töltsék le.

## Munkafolyamat kiadása

### 1. lépés: Készítse elő a GitHub-kiadást

Először hozzon létre egy GitHub-kiadást a `portable-version` adattár, amely tartalmazza:
- Tömörített archívumok minden platformhoz
- Build manifest (`{tag}.build-manifest.json`)
- Műtermékkészlet (`{tag}.artifact-inventory.json`)

### 2. lépés: Indítsa el a Steam kiadás munkafolyamatát

Indítsa el a munkafolyamatot manuálisan a GitHub Actions segítségével, és töltse ki a szükséges paramétereket:
- `release`: a közzétenni kívánt verziócímke, pl `v1.0.0`
- `steam_branch`: a cél ág, mint pl `preview` vagy `public`
- `steam_preview`: használja-e az előnézeti módot

### 3. lépés: Hajtsa végre a Release Flow automatikus végrehajtását

A munkafolyamat automatikusan a következő lépéseket hajtja végre:

1. Töltse le és bontsa ki a GitHub Release műtermékeket.
2. Telepítse vagy frissítse a SteamCMD-t.
3. Steam VDF konfigurációs fájlok létrehozása.
4. Hitelesítés a Steam Guard segítségével.
5. Tölts fel tartalmat a Steam CDN-re.
6. Állítsa élesre a megadott ágat.

A sorozat befejezése után a teljes kiadási útvonal lefedett.

## Konfigurációs útmutató

### Szükséges Secrets konfiguráció

Konfigurálja a következő titkokat a GitHub tárhely beállításaiban:

| Titkos név | Leírás |
|-----------|------|
| `STEAM_USERNAME` | Steam fiók felhasználónév |
| `STEAM_PASSWORD` | Steam fiók jelszava |
| `STEAM_SHARED_SECRET` | Steam Guard megosztott titok (opcionális) |
| `STEAM_GUARD_CODE` | Steam Guard kód (opcionális) |
| `STEAM_APP_ID` | Steam alkalmazásazonosító |
| `STEAM_DEPOT_ID_LINUX` | Linux Depot azonosító |
| `STEAM_DEPOT_ID_WINDOWS` | Windows Depot azonosító |
| `STEAM_DEPOT_ID_MACOS` | macOS Depot azonosító |

Nincs semmi különösebben szokatlan ezekben a beállításokban. Egyszerűen az összes elvárt értéknek a helyén kell lennie.

### Környezeti változó konfigurációja

| Változó neve | Leírás | Alapértelmezett érték |
|---------|------|--------|
| `PORTABLE_VERSION_STEAMCMD_ROOT` | SteamCMD telepítési könyvtár | `~/.local/share/portable-version/steamcmd` |

## Legjobb gyakorlatok

### Steam Guard Authentication Management

Az első futtatáskor kézzel kell megadnia a Steam Guard kódot. Ezt követően ajánlatos beállítani a megosztott titkot, hogy a kód automatikusan generálható legyen. Ezzel elkerülhető a kézi beavatkozás minden kiadásnál.

A SteamCMD elmenti a bejelentkezési tokent, és későbbi futtatások során újra felhasználhatja. Továbbra is figyelnie kell a token lejáratát, mert a token lejárta után újra kell hitelesíteni.

### Tartalomkönyvtár szerkezete

Győződjön meg arról, hogy a Steam tartalomkönyvtár szerkezete megfelelő:

```
steam-content/
├── linux-x64/     # Linux platform content
├── win-x64/       # Windows platform content
└── osx-universal/ # macOS universal binary content
```

Minden könyvtárnak tartalmaznia kell a megfelelő platformhoz tartozó teljes alkalmazásfájlt.

### Előnézeti mód használata

Az előnézeti mód egyetlen ágat sem állít élesben, így alkalmas tesztelésre és érvényesítésre:

```yaml
if [ "$STEAM_PREVIEW_INPUT" = 'true' ]; then
  cmd+=(--preview)
fi
```

Ez lehetővé teszi, hogy először a Steambe töltsön fel ellenőrzést, majd miután minden leellenőrizte, váltson a formális ágra.

### Hibakezelés és naplózás

A szkriptek teljes hibakezelést és naplózást tartalmaznak:
- Ellenőrizze, hogy létezik-e a GitHub-kiadás.
- Ellenőrizze a szükséges metaadatfájlokat.
- Győződjön meg arról, hogy a platform tartalma jelen van.
- Hozzon létre egy GitHub Actions összefoglaló jelentést.

Ez az információ rendkívül értékes mind a hibakeresés, mind az auditálás szempontjából.

### Műtermékkezelés

A munkafolyamat kétféle műterméket hoz létre:
- `portable-steam-release-preparation-{tag}`: kiadás előkészítés metaadatai
- `portable-steam-build-metadata-{tag}`: Steam build metaadatok

Ezek a műtermékek későbbi ellenőrzéshez és hibakereséshez használhatók. A 30 napos megőrzési időszak gyakorlati alapértelmezés.

## Valós használat

A HagiCode projektben ez az automatizált kiadási munkafolyamat már több verzióban is sikeresen futott. A GitHub kiadástól a Steam platformig vezető teljes út teljesen automatizált, és nem igényel kézi beavatkozást.

Ez jelentősen javította a kibocsátási hatékonyságot és a megbízhatóságot. Korábban egy verzió manuális közzététele több mint 30 percig tartott. Most az egész folyamat néhány perc alatt befejeződik.

Ennél is fontosabb, hogy az automatizált munkafolyamat csökkenti az emberi hibák esélyét. Minden kiadás ugyanazt a szabványosított folyamatot követi, ami kiszámíthatóbbá teszi az eredményeket.
