## A HagiCode Skill System technikai elemzése: Skálázható mesterséges intelligencia készségkezelő platform építése

> Ez a cikk alapos pillantást vet a HagiCode projekt Skill Management rendszerének felépítésére és megvalósítására, amely négy alapvető képesség mögött meghúzódó technikai részleteket fedi le: helyi globális menedzsment, piactéri keresés, intelligens ajánlások és megbízható szolgáltatói menedzsment.

## Háttér

Az AI-kódolási asszisztensek területén mindig is alapvető kérdés volt, hogyan lehet kiterjeszteni az AI-képességek határait. Maga a Claude Code már erős a kódtámogatás terén, de a különböző fejlesztőcsapatoknak és a különböző technológiai csoportoknak gyakran speciális képességekre van szükségük bizonyos forgatókönyvekhez, például a Docker-telepítések kezeléséhez, az adatbázis-optimalizáláshoz vagy a frontend összetevők generálásához. Pontosan itt válik különösen fontossá a Skill rendszer.

A HagiCode projekt fejlesztése során mi is hasonló kihívásba ütköztünk: hogyan hagyhatjuk, hogy Claude Code új szakmai ismereteket „tanuljon el”, mint egy ember, miközben továbbra is megőrizzük a szilárd felhasználói élményt és a jó mérnöki karbantarthatóságot? Ez a probléma a maga módján nehéz és egyszerű is. E kérdés körül egy komplett Skill Management rendszert terveztünk és vezettünk be.

Ez a cikk részletesen bemutatja a rendszer műszaki felépítését és alapvető megvalósítását. Az AI bővíthetősége és a parancssori eszközök integrációja iránt érdeklődő fejlesztők számára készült. Lehet, hogy hasznos lesz számodra, vagy nem, de legalább most le van írva.

## A HagiCode-ról

Az ebben a cikkben megosztott megközelítés a gyakorlati tapasztalatainkból származik [HagiCode](https://hagicode.com) projekt. A HagiCode egy nyílt forráskódú AI kódolási asszisztens projekt, amelynek célja, hogy segítse a fejlesztőcsapatokat a mérnöki hatékonyság javításában. A projekt verem tartalmazza az ASP.NET Core-t, az Orleans elosztott keretrendszert, a TanStack Start + React frontendet és a jelen cikkben bemutatott Skill management alrendszert.

A GitHub adattár az [HagiCode-org/site](https://github.com/HagiCode-org/site). Ha értékesnek találja a cikkben szereplő technikai megközelítést, bátran adjon neki egy csillagot. A több csillag végül is javítja a hangulatot.

## Rendszerarchitektúra áttekintése

A Skill rendszer frontend-backend elválasztott architektúrát használ. Nincs ebben semmi különösebben titokzatos.

A **Frontend** a TanStack Start + React funkciót használja a felhasználói felület felépítéséhez, Redux Toolkit felügyeleti állapottal. A négy fő képesség közvetlenül a lap négy összetevőjéhez kapcsolódik: Helyi készségek, Képességtár, Intelligens ajánlások és Megbízható szolgáltatók. A tervezés végső soron leginkább a felhasználói élmény javításáról szól.

A **Backend** az ASP.NET Core + ABP Framework-re épül, és az Orleans Graint használja az elosztott állapotkezeléshez. Az online API kliens becsomagolja a `IOnlineApiClient` interfész a távoli készségkatalógus szolgáltatással való kommunikációhoz.

Az általános felépítési elv az, hogy el kell különíteni a parancsvégrehajtást az üzleti logikától. Az illesztőmintán keresztül az npm/npx parancs végrehajtásának részletei a független modulok belsejében vannak elrejtve. Végül is senki sem akarja, hogy a parancssori hívások szétszórva legyenek a kódbázison.

## 1. alapvető képesség: Helyi globális felügyelet

A helyi globális menedzsment a legalapvetőbb modul. Felelős a telepített készségek felsorolásáért és az eltávolítási műveletek támogatásáért. Nincs itt semmi túlzottan bonyolult; leginkább arról szól, hogy jól csináld az alapokat.

### Technikai megközelítés

A megvalósítás él `LocalSkillsTab.tsx` és `LocalSkillCommandAdapter.cs`. Az alapötlet az, hogy becsomagoljuk a `npx skills` parancsot, elemezze a JSON-kimenetét, és alakítsa át belső adatstruktúrákká. Egyszerűen hangzik, és a gyakorlatban többnyire az is.

```csharp
public async Task<IReadOnlyList<LocalSkillInventoryResponseDto>> GetLocalSkillsAsync(
    CancellationToken cancellationToken = default)
{
    var result = await _commandAdapter.ListGlobalSkillsAsync(cancellationToken);
    return result.Skills.Select(skill => new LocalSkillInventoryResponseDto
    {
        Name = skill.Name,
        Version = skill.Version,
        Source = skill.Source,
        InstalledPath = skill.InstalledPath,
        Description = skill.Description
    }).ToList();
}
```

Az adatáramlás nagyon egyértelmű: a frontend kérést küld -> `SkillGalleryAppService` megkapja -> `LocalSkillCommandAdapter` végrehajtja az npx parancsot -> a JSON eredményt elemzi -> egy DTO-t ad vissza. Minden lépés természetesen következik az előzőből.

A Skill eltávolítás a `npx skills remove -g <skillName> -y` parancsot, és a rendszer automatikusan kezeli a függőségeket és a tisztítást. A telepítési metaadatok a következő helyen vannak tárolva `managed-install.json` a készségkönyvtáron belül, rögzíti az információkat, például a telepítési időt és a forrásverziót a későbbi frissítésekhez és auditáláshoz. Néhány dolgot egyszerűen érdemes rögzíteni.

### A telepítés folyamata részletesen

A szakszerű telepítés több összehangolt lépést igényel. Valójában ez nem különösebben bonyolult:

```csharp
public async Task<SkillInstallResultDto> InstallAsync(
    SkillInstallRequestDto request,
    CancellationToken cancellationToken = default)
{
    // 1. Normalize the installation reference
    var normalized = _referenceNormalizer.Normalize(
        request.SkillId,
        request.Source,
        request.SkillSlug,
        request.Version);

    // 2. Check prerequisites
    await _prerequisiteChecker.CheckAsync(cancellationToken);

    // 3. Acquire installation lock
    using var installLock = await _lockProvider.AcquireAsync(normalized.SkillId);

    // 4. Execute installation command
    var result = await _installCommandRunner.ExecuteAsync(
        new SkillInstallCommandExecutionRequest
        {
            Command = $"npx skills add {normalized.FullReference} -g -y",
            Timeout = TimeSpan.FromMinutes(4)
        },
        cancellationToken);

    // 5. Persist installation metadata
    await _metadataStore.WriteAsync(normalized.SkillPath, request);

    return new SkillInstallResultDto { Success = result.Success };
}
```

Számos kulcsfontosságú tervezési mintát használnak itt: a **referencianormalizáló** különböző bemeneti formátumokat konvertál, mint pl. `tanweai/pua` és `@opencode/docker-skill`, egységes belső reprezentációvá; a **telepítési zárszerkezet** biztosítja, hogy egyszerre csak egy telepítési művelet futhasson ugyanarra a képességre; és a **streaming output** a telepítés előrehaladását valós időben a kezelőfelületre tolja a Server-Sent Events révén, így a felhasználók folyamatosan figyelhetik a terminálszerű naplókat.

Végül is ezek a minták egyetlen célt szolgálnak: a rendszer egyszerűbbé tételét és karbantartását.

## 2. alapvető képesség: Marketplace Search

A Marketplace keresés segítségével a felhasználók felfedezhetik és telepíthetik a közösség készségeit. Egy személy képessége mindig korlátozott; a kollektív tudás sokkal tovább megy.

### Technikai megközelítés

A keresési funkció az online API-n alapul `https://api.hagicode.com/v1/skills/search`. A válaszsebesség javítása érdekében a rendszer gyorsítótárazást valósít meg. A gyorsítótár egy kicsit olyan, mint a memória: ha hasznos dolgokat tartasz a közelben, akkor legközelebb nem kell annyira gondolkodnod.

```csharp
private async Task<IReadOnlyList<SkillGallerySkillDto>> SearchCatalogAsync(
    string query,
    CancellationToken cancellationToken,
    IReadOnlySet<string>? allowedSources = null)
{
    var cacheKey = $"skill_search:{query}:{string.Join(",", allowedSources ?? Array.Empty<string>())}";

    if (_memoryCache.TryGetValue(cacheKey, out var cached))
        return (IReadOnlyList<SkillGallerySkillDto>)cached!;

    var response = await _onlineApiClient.SearchAsync(
        new SearchSkillsRequest
        {
            Query = query,
            Limit = _options.LimitPerQuery,
        },
        cancellationToken);

    var results = response.Skills
        .Where(skill => allowedSources is null || allowedSources.Contains(skill.Source))
        .Select(skill => new SkillGallerySkillDto { ... })
        .ToList();

    _memoryCache.Set(cacheKey, results, TimeSpan.FromMinutes(10));
    return results;
}
```

A keresési eredmények támogatják a megbízható források szerinti szűrést, így a felhasználók csak azokat a tudásforrásokat látják, amelyekben megbíznak. Seed lekérdezések, mint pl `popular` és `recent` A katalógus inicializálására szolgálnak, lehetővé téve a felhasználók számára, hogy az ajánlott népszerű készségeket lássák az első megnyitáskor. Az első benyomás még mindig számít.

## 3. alapvető képesség: Intelligens ajánlások

Az intelligens ajánlások a rendszer legösszetettebb részét képezik. Automatikusan ajánlhatják a legmegfelelőbb készségeket az aktuális projektkörnyezet alapján. Bármilyen összetett, mégis érdemes építeni.

### Ajánlási folyamat

A teljes ajánlási folyamat öt szakaszra oszlik:

```
1. Build project context
   ↓
2. AI generates search queries
   ↓
3. Search the online catalog in parallel
   ↓
4. AI ranks the candidates
   ↓
5. Return the recommendation list
```

Először is, a rendszer elemzi az olyan jellemzőket, mint a projekt technológiai halmaza, programozási nyelvek és tartományszerkezet, hogy létrehozzon egy „projektprofilt”. Ez a profil egy kicsit olyan, mint egy önéletrajz, rögzíti a projekt legfontosabb jellemzőit.

Ezután egy AI Grain-t használnak a célzott keresési lekérdezések generálására. Ez a konstrukció valójában meglehetősen érdekes: ahelyett, hogy közvetlenül megkérdeznénk az AI-tól: "Milyen készségeket ajánljak?", először megkérjük, hogy gondolja át: "Milyen keresési kifejezések találhatnak megfelelő készségeket?" Néha a kérdés feltevés módja többet jelent, mint maga a válasz:

```csharp
var queryGeneration = await aiGrain.GenerateSkillRecommendationQueriesAsync(
    projectContext,      // Project context
    locale,              // User language preference
    maxQueries,           // Maximum number of queries
    effectiveSearchHero); // AI model selection
```

Ezután a keresési lekérdezések párhuzamosan végrehajtásra kerülnek, hogy összegyűjtsék a jelölt készségek listáját. A párhuzamos feldolgozás a nap végén csak egy módja annak, hogy időt takarítson meg.

Végül egy másik AI Grain rangsorolja a jelölt képességeit. Ez a lépés olyan tényezőket vesz figyelembe, mint a projekthez kapcsolódó készségek relevanciája, a bizalom állapota és a felhasználói előzménybeállítások:

```csharp
var ranking = await aiGrain.RankSkillRecommendationsAsync(
```

### Tartalék mechanizmus

Az AI-modellek lassan reagálhatnak, vagy átmenetileg elérhetetlenné válhatnak. Még a legjobb rendszerek is megbotlik néha. Emiatt a rendszer tartalmaz egy determinisztikus tartalék mechanizmust: ha a mesterséges intelligencia szolgáltatás nem elérhető, szabályalapú heurisztikus algoritmust használ ajánlások generálására, például a valószínűsíthetően szükséges készségekre a függőségekből következtetve. `package.json`.

Egyszerűen fogalmazva, ez a tartalék mechanizmus egyszerűen a rendszer tartalék terve.

## 4. alapvető képesség: Megbízható szolgáltatók kezelése

A megbízható szolgáltatók kezelése lehetővé teszi a felhasználók számára, hogy szabályozzák, mely készségforrások számítanak megbízhatónak. A bizalom továbbra is olyan dolog, amelyet a felhasználóknak maguknak kell meghatározniuk.

### Egyezési szabályok

A megbízható szolgáltatók két egyezési szabályt támogatnak: **pontos egyezés** (`exact`) és **előtag egyezés** (`prefix`).

```csharp
public static TrustedSkillProviderResolutionSnapshot Resolve(
    TrustedSkillProviderSnapshot snapshot,
    string source)
{
    var normalizedSource = Normalize(source);

    foreach (var entry in snapshot.Entries.OrderBy(e => e.SortOrder))
    {
        if (!entry.IsEnabled) continue;

        foreach (var rule in entry.MatchRules)
        {
            bool isMatch = rule.MatchType switch
            {
                TrustedSkillProviderMatchRuleType.Exact
                    => string.Equals(normalizedSource, Normalize(rule.Value),
                        StringComparison.OrdinalIgnoreCase),
                TrustedSkillProviderMatchRuleType.Prefix
                    => normalizedSource.StartsWith(Normalize(rule.Value) + "/",
                        StringComparison.OrdinalIgnoreCase),
                _ => false
            };

            if (isMatch)
                return new TrustedSkillProviderResolutionSnapshot
                {
                    IsTrustedSource = true,
                    ProviderId = entry.ProviderId,
                    DisplayName = entry.DisplayName
                };
        }
    }

    return new TrustedSkillProviderResolutionSnapshot { IsTrustedSource = false };
}
```

A beépített megbízható szolgáltatók közé tartoznak az olyan jól ismert szervezetek és projektek, mint a Vercel, az Azure, az anthropics, a Microsoft és a böngészőhasználat. Egyéni szolgáltatók hozzáadhatók konfigurációs fájlokon keresztül a szolgáltatói azonosító, a megjelenített név, a jelvénycímke, az illesztési szabályok és egyebek megadásával. A világ elég nagy ahhoz, hogy csak néhány beépített készülékben bízzon soha.

### Perzisztencia megvalósítás

A megbízható konfiguráció megmarad egy Orleans Grain használatával:

```csharp
public class TrustedSkillProviderGrain : Grain<TrustedSkillProviderState>,
    ITrustedSkillProviderGrain
{
    public async Task UpdateConfigurationAsync(TrustedSkillProviderSnapshot snapshot)
    {
        State.Snapshot = snapshot;
        await WriteStateAsync();
    }

    public Task<TrustedSkillProviderSnapshot> GetConfigurationAsync()
    {
        return Task.FromResult(State.Snapshot);
    }
}
```

Ennek a megközelítésnek az az előnye, hogy a konfigurációs változások automatikusan szinkronizálódnak az összes csomóponton, anélkül, hogy manuálisan kellene frissíteni a gyorsítótárakat. Az automatizálás végső soron arról szól, hogy az emberek kevésbé aggódjanak.

## Kulcsfontosságú műszaki tervezés

### Parancsvégrehajtási adapter minta

A Skill rendszernek különféle npx parancsokat kell végrehajtania. Ha ez a logika mindenhol szétszórva lenne, a kódot gyorsan megnehezítené a karbantartása. Ezért terveztünk egy adapter interfészt. A tervezési minták végül azért léteznek, hogy megkönnyítsék a kód karbantartását:

```csharp
public interface ISkillInstallCommandRunner
{
    Task<SkillInstallCommandExecutionResult> ExecuteAsync(
        SkillInstallCommandExecutionRequest request,
        CancellationToken cancellationToken = default);
}
```

A különböző parancsok különböző végrehajtó implementációkkal rendelkeznek, de mindegyik ugyanazt a felületet valósítja meg, így a tesztelés és a csere egyszerű.

### SSE streaming kimenet

A telepítés folyamata valós időben kerül a kezelőfelületre a szerver által küldött eseményeken keresztül:

```csharp
public async Task InstallWithProgressAsync(
    SkillInstallRequestDto request,
    IServerStreamWriter<SkillInstallProgressEventDto> stream,
    CancellationToken cancellationToken)
{
    var process = new Process
    {
        StartInfo = new ProcessStartInfo
        {
            FileName = "npx",
            Arguments = $"skills add {request.FullReference} -g -y",
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false
        }
    };

    process.OutputDataReceived += async (sender, e) =>
    {
        await stream.WriteAsync(new SkillInstallProgressEventDto
        {
            EventType = "output",
            Data = e.Data ?? string.Empty
        });
    };

    process.Start();
    process.BeginOutputReadLine();
    await process.WaitForExitAsync(cancellationToken);
}
```

Az előtérben a felhasználók valós időben láthatják a terminálszerű kimenetet, ami nagyon intuitívvá teszi az élményt. A valós idejű visszajelzés segít az embereknek, hogy jól érezzék magukat.

## Gyakorlati útmutató

### Közösségi készség telepítése

Telepítse a `pua` készség példaként (ez egy népszerű közösségi készség):

1. Nyissa ki a Készségek fiókot, és váltson a `Skill Gallery` lapon
2. Enter `pua` a keresőmezőben
3. Kattintson a keresési eredményre a készség részleteinek megtekintéséhez
4. Kattintson a `Install` gombot
5. Váltson a `Local Skills` lapon erősítse meg a telepítés sikerességét

A telepítési parancs az `npx skills add tanweai/pua -g -y`, és a rendszer minden részletet automatikusan kezel. Valójában nincs annyi lépés, ha egyesével megteszi őket.

### Egyéni megbízható forrás hozzáadása

Ha csapatának saját készségtára van, hozzáadhatja megbízható forrásként:

```yaml
providerId: "my-team"
displayName: "My Team Skills"
badgeLabel: "MyTeam"
isEnabled: true
sortOrder: 100
matchRules:
  - matchType: "prefix"
    value: "my-team/"
  - matchType: "exact"
    value: "my-team/special-skill"
```

Így a csapat minden készsége megbízható jelvényt fog megjeleníteni, így a felhasználók kényelmesebben telepíthetik őket. A címkék és jelek segítenek az embereknek magabiztosabbnak érezni magukat.

### Képességfejlesztési alapismeretek

Egyéni készség létrehozásához a következő struktúra szükséges:

```
my-skill/
├── SKILL.md          # Skill metadata (YAML front matter)
├── index.ts          # Skill entry point
├── agents/           # Supported agent configuration
└── references/       # Reference resources
```

Egy példa `SKILL.md` formátum:

```yaml

name: my-skill

---

Detailed documentation...
```
