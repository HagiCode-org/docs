## Alapvető megvalósítás

### A két hotword mód megértése

A Doubao beszédfelismerő API kétféleképpen konfigurálhatja a gyorsszavakat, és mindegyiknek megvannak a saját ideális használati esetei és előnyei.

**Egyéni hotword mód** lehetővé teszi, hogy a hotword szövegét közvetlenül a `corpus.context` mezőben. Ez a megközelítés különösen alkalmas olyan helyzetekben, ahol gyorsan be kell állítani néhány gyorsszót, például ideiglenesen fel kell ismerni egy termék vagy egy személy nevét. A HagiCode megvalósításában a felhasználó által beírt többsoros hotword szöveget karakterláncok listájába elemezzük, majd a `context_data` a Doubao API által megkövetelt tömb. Ez a megközelítés nagyon közvetlen: egyszerűen megmondja a rendszernek, hogy mely szavakra kell figyelnie, és pontosan ezt teszi.

**Platform hotword table mode** a `corpus.boosting_table_id` mező, amely egy előre konfigurált hotword-táblázatra hivatkozik a Doubao öntanuló platformon. Ez a megközelítés olyan forgatókönyvekhez alkalmas, ahol nagyszámú hotword-ot kell kezelni. Hotword táblákat hozhatunk létre és karbantarthatunk a Doubao öntanuló platformon, majd azonosító alapján hivatkozhatunk rájuk. Egy olyan projektnél, mint a HagiCode, ahol a speciális kifejezéseket folyamatosan frissíteni és karbantartani kell, ez a mód sokkal jobb kezelhetőséget kínál. Ha a gyorsszavak száma növekszik, sokkal jobb, ha központi helyen kezelheti őket, mintha minden alkalommal manuálisan írná be őket.

Érdekes módon ez a két mód együtt is használható. A Doubao API támogatja mind az egyéni hotwordok, mind a platform hotword-tábla azonosítójának beépítését ugyanabban a kérelemben, a kombinációs stratégiát a `combine_mode` paramétert. Ez a rugalmasság lehetővé teszi a HagiCode számára, hogy az összetett szakmai terminológia felismerési igények széles skáláját kezelje. Néha több megközelítés kombinálása jobb eredményeket hoz.

### Frontend típusdefiníciók és ellenőrzés

A HagiCode frontend implementációjában a hotword konfigurációs típusok és az érvényesítési logika teljes készletét definiáltuk. Az első rész a típusdefiníció:

```typescript
export interface HotwordConfig {
  contextText: string;           // Multi-line hotword text
  boostingTableId: string;      // Doubao platform hotword table ID
  combineMode: boolean;          // Whether to use both together
}
```

Ez az egyszerű felület tartalmazza a hotword funkció összes konfigurációs elemét. Köztük `contextText` ez az a rész, amellyel a felhasználók a legközvetlenebbül lépnek kapcsolatba: lehetővé tesszük a felhasználók számára, hogy soronként egy gyorsszó-kifejezést írjanak be, ami nagyon intuitív. Sokkal könnyebb megkérni a felhasználókat, hogy soronként egy kifejezést adjanak meg, mint megértetni velük a bonyolult konfigurációs formátumot.

Ezután következik az érvényesítési funkció. A Doubao API korlátozásai alapján szigorú érvényesítési szabályokat határoztunk meg: legfeljebb 100 sornyi hotword-szöveg, soronként legfeljebb 50 karakter, összesen legfeljebb 5000 karakter; `boosting_table_id` legfeljebb 200 karakterből állhat, és csak betűket, számokat, aláhúzásjeleket és kötőjeleket tartalmazhat. Ezek a korlátok nem önkényesek; közvetlenül a hivatalos Doubao dokumentációból származnak. Az API-korlátok API-korlátok, és ezeket be kell tartanunk.

```typescript
export function validateContextText(contextText: string): HotwordValidationResult {
  if (!contextText || contextText.trim().length === 0) {
    return { isValid: true, errors: [] };
  }

  const lines = contextText.split('\n').filter(line => line.trim().length > 0);
  const errors: string[] = [];

  if (lines.length > 100) {
    errors.push(`Hotword line count cannot exceed 100 lines; current count is ${lines.length}`);
  }

  const totalChars = contextText.length;
  if (totalChars > 5000) {
    errors.push(`Total hotword character count cannot exceed 5000; current count is ${totalChars}`);
  }

  for (let i = 0; i < lines.length; i++) {
    if (lines[i].length > 50) {
      errors.push(`Hotword on line ${i + 1} exceeds the 50-character limit`);
    }
  }

  return { isValid: errors.length === 0, errors };
}

export function validateBoostingTableId(boostingTableId: string): HotwordValidationResult {
  if (!boostingTableId || boostingTableId.trim().length === 0) {
    return { isValid: true, errors: [] };
  }

  const errors: string[] = [];

  if (boostingTableId.length > 200) {
    errors.push(`boosting_table_id cannot exceed 200 characters; current count is ${boostingTableId.length}`);
  }

  if (!/^[a-zA-Z0-9_-]+$/.test(boostingTableId)) {
    errors.push('boosting_table_id can contain only letters, numbers, underscores, and hyphens');
  }

  return { isValid: errors.length === 0, errors };
}
```

Ezek az érvényesítési funkciók azonnal futnak, amikor a felhasználó beállítja a gyorsszavakat, így biztosítva, hogy a problémákat a lehető legkorábban észleljék. A felhasználói élmény szempontjából ez a fajta azonnali visszajelzés nagyon fontos. Mindig jobb, ha a felhasználók gépelés közben tudják meg, mi a hiba, és nem az elküldés után.

### Frontend konfigurációs állandóság

A HagiCode frontend implementációjában a böngésző használatát választottuk `localStorage` hotword konfiguráció tárolására. E tervezési döntés hátterében több szempont állt. Először is, a hotword-konfiguráció erősen személyre szabott, és a különböző felhasználóknak eltérő tartományspecifikus igényei lehetnek. Másodszor, ez a megközelítés leegyszerűsíti a háttérrendszer megvalósítását, mivel nem igényel további adatbázistáblákat vagy API-végpontokat. Végül, miután a felhasználók egyszer konfigurálták a böngészőben, a beállítások automatikusan betölthetők a későbbi használat során, ami nagyon kényelmes. Egyszerűen fogalmazva, ez a legegyszerűbb megközelítés.

```typescript
const HOTWORD_STORAGE_KEYS = {
  contextText: 'hotword-context-text',
  boostingTableId: 'hotword-boosting-table-id',
  combineMode: 'hotword-combine-mode',
} as const;

export const DEFAULT_HOTWORD_CONFIG: HotwordConfig = {
  contextText: '',
  boostingTableId: '',
  combineMode: false,
};

// Load hotword configuration
export function loadHotwordConfig(): HotwordConfig {
  const contextText = localStorage.getItem(HOTWORD_STORAGE_KEYS.contextText) || '';
  const boostingTableId = localStorage.getItem(HOTWORD_STORAGE_KEYS.boostingTableId) || '';
  const combineMode = localStorage.getItem(HOTWORD_STORAGE_KEYS.combineMode) === 'true';

  return { contextText, boostingTableId, combineMode };
}

// Save hotword configuration
export function saveHotwordConfig(config: HotwordConfig): void {
  localStorage.setItem(HOTWORD_STORAGE_KEYS.contextText, config.contextText);
  localStorage.setItem(HOTWORD_STORAGE_KEYS.boostingTableId, config.boostingTableId);
  localStorage.setItem(HOTWORD_STORAGE_KEYS.combineMode, String(config.combineMode));
}
```

A kód logikája egyértelmű és világos. -től olvasunk `localStorage` konfiguráció betöltésekor, és írjon ide `localStorage` mentéskor. Alapértelmezett konfigurációt is biztosítunk, hogy a rendszer akkor is megfelelően működjön, ha még nincs konfiguráció. Végtére is kell egy ésszerű alapértelmezés.

Ha nem használja a funkciót, ne hagyja az útból.

### Raktározási logika

A hasznos teherfelépítés a teljes hotword funkció magja. Miután megvan a hotword konfiguráció, meg kell formáznunk a Doubao API által megkövetelt JSON-struktúrára. Ez a folyamat még azelőtt megtörténik, hogy az SDK elküldi a kérést:

```csharp
private void AddCorpusToRequest(Dictionary<string, object> request)
{
    var corpus = new Dictionary<string, object>();

    // Add custom hotwords
    if (Config.HotwordContexts != null && Config.HotwordContexts.Count > 0)
    {
        corpus["context"] = new Dictionary<string, object>
        {
            ["context_type"] = "dialog_ctx",
            ["context_data"] = Config.HotwordContexts
                .Select(text => new Dictionary<string, object> { ["text"] = text })
                .ToList()
        };
    }

    // Add platform hotword table ID
    if (!string.IsNullOrEmpty(Config.BoostingTableId))
    {
        corpus["boosting_table_id"] = Config.BoostingTableId;
    }

    // Add corpus to the request only when it is not empty
    if (corpus.Count > 0)
    {
        request["corpus"] = corpus;
    }
}
```

Ez a kód megmutatja, hogyan kell dinamikusan felépíteni a `corpus` mező konfiguráció alapján. A lényeg az, hogy hozzáadjuk a `corpus` mező csak akkor, ha a hotword konfigurációja valóban létezik. Ez a kialakítás biztosítja a visszamenőleges kompatibilitást: amikor nincs beállítva hotword, a kérés szerkezete pontosan ugyanaz marad, mint korábban. A visszamenőleges kompatibilitás számít; egy funkció hozzáadása nem zavarhatja meg a meglévő logikát.

### WebSocket paraméter átadása

Az előtér és a háttér között a hotword paraméterek WebSocket vezérlőüzeneteken keresztül kerülnek továbbításra. A HagiCode úgy lett megtervezve, hogy amikor a frontend elkezdi a rögzítést, betölti a hotword konfigurációt innen `localStorage` és WebSocket üzeneten keresztül elküldi a háttérnek.

```typescript
const controlMessage = {
  type: 'control',
  payload: {
    command: 'StartRecognition',
    contextText: '高血压\n糖尿病\n冠心病',
    boosting_table_id: 'medical_table',
    combineMode: false
  }
};
```

Itt egy részletet érdemes megjegyezni: a frontend többsoros szöveget ad át újsor karakterekkel elválasztva, és a backendnek elemeznie kell azt. A háttér WebSocket kezelő elemzi ezeket a paramétereket, és átadja őket az SDK-nak:

```csharp
private async Task HandleControlMessageAsync(
    string connectionId,
    DoubaoSession session,
    ControlMessage message)
{
    if (message.Payload is SessionControlRequest controlRequest)
    {
        // Parse hotword parameters
        string? contextText = controlRequest.ContextText;
        string? boostingTableId = controlRequest.BoostingTableId;
        bool? combineMode = controlRequest.CombineMode;

        // Parse multi-line text into a hotword list
        if (!string.IsNullOrEmpty(contextText))
        {
            var hotwords = contextText
                .Split('\n', StringSplitOptions.RemoveEmptyEntries)
                .Select(s => s.Trim())
                .Where(s => s.Length > 0)
                .ToList();

            session.HotwordContexts = hotwords;
        }

        session.BoostingTableId = boostingTableId;
    }
}
```

Ezzel a kialakítással a hotword-konfiguráció átadása a frontendről a háttérrendszerre egyértelművé és hatékonysá válik. Nincs benne semmi különösebben titokzatos; az adatokat egyszerűen rétegről rétegre továbbítják.

## Gyakorlati útmutató

### Egyéni gyorsszavak konfigurálása

Valós használatban az egyéni hotwords konfigurálása nagyon egyszerű. Nyissa meg a HagiCode beszédfelismerési beállítások oldalát, és keresse meg a „Hotword Configuration” részt. Az „Egyéni gyorsszó szövege” beviteli mezőbe soronként egy gyorsszó-kifejezést adjon meg.

Például, ha orvosi vonatkozású alkalmazást fejleszt, a következőképpen konfigurálhatja:

```
高血压
糖尿病
冠心病
心绞痛
心肌梗死
心力衰竭
```

A konfiguráció mentése után ezek a gyorsszavak automatikusan átadódnak a Doubao API-nak minden alkalommal, amikor a beszédfelismerés elindul. Teszteink során a gyorsszavak konfigurálása után a kapcsolódó szakmai kifejezések felismerési pontossága észrevehetően javult. A javulás valódi, és egyértelműen jobb, mint korábban.

### Platform Hotword Table konfigurálása

Ha nagyszámú hotword-ot kell kezelnie, vagy ha a hotwords gyakran frissítésre szorul, akkor a platform hotword táblázat mód jobban illeszkedik. Először hozzon létre egy hotword táblázatot a Doubao öntanuló platformon, és szerezze be a generált `boosting_table_id`, majd adja meg ezt az azonosítót a HagiCode beállítási oldalán.

A Doubao öntanuló platform olyan lehetőségeket biztosít, mint például a tömeges importálás és a hotwords kategorizált kezelése, ami nagyon praktikus olyan csapatok számára, akiknek nagy mennyiségű speciális terminológiát kell kezelniük. A hotwords platformon történő kezelésével központilag karbantarthatja őket, és következetesen telepítheti a frissítéseket. Amint a gyorsszavak listája nagy lesz, sokkal praktikusabb, ha egyetlen helyen kezelheti azt, mint minden alkalommal a kézi bevitelt.

### Kombinációs mód használata

Egyes összetett forgatókönyvekben előfordulhat, hogy egyszerre kell használni az egyéni hotwordokat és a platform gyorsszótáblázatát. Ebben az esetben egyszerűen konfigurálja mindkettőt a HagiCode-ban, és engedélyezze a „Kombinációs mód” kapcsolót.

Kombinációs módban a Doubao API mindkét hotword-forrást egyszerre veszi figyelembe, így a felismerési pontosság általában nagyobb, mintha bármelyik forrást önmagában használná. Érdemes azonban megjegyezni, hogy a kombinációs mód növeli a kérések bonyolultságát, ezért a gyakorlati tesztelés után a legjobb eldönteni, hogy engedélyezzük-e. A komplexitás csak akkor éri meg, ha a valós eredmények ezt igazolják.

## Ne feledje

A hotword funkció megvalósítása és használata során több pontra is különös figyelmet kell fordítani.

Az első a karakterkorlát. A Doubao API szigorú korlátozásokat tartalmaz a hotwordokra vonatkozóan, beleértve a sorok számát, a soronkénti karaktereket és a teljes karakterszámot. Bármely korlát túllépése esetén az API hibát ad vissza. A HagiCode frontend implementációjában ezeket a megszorításokat a felhasználói bevitel során ellenőrizzük az érvényesítési függvényekkel, ami megakadályozza, hogy érvénytelen konfigurációk kerüljenek a háttérbe. A problémák korai felismerése mindig jobb, mint megvárni, amíg az API meghibásodik.

A második a formátum `boosting_table_id`. Ez a mező csak betűket, számokat, aláhúzásjeleket és kötőjeleket engedélyez, és nem tartalmazhat szóközt vagy más speciális karaktert. Amikor hotword táblázatot hoz létre a Doubao öntanuló platformon, ügyeljen az elnevezési szabályok betartására. Ez a fajta szigorú formátumellenőrzés általános az API-k esetében.

A harmadik a visszamenőleges kompatibilitás. A hotword paraméterek teljesen opcionálisak. Ha nincs beállítva hotword, a rendszer pontosan úgy viselkedik, mint korábban. Ez a kialakítás biztosítja, hogy a meglévő felhasználókat semmilyen módon ne érintse, valamint megkönnyíti a fokozatos migrációt és a frissítéseket. Egy szolgáltatás hozzáadása nem zavarhatja meg a korábbi logikát.

Végül van hibakezelés. Ha a hotword konfiguráció érvénytelen, a Doubao API a megfelelő hibaüzeneteket adja vissza. A HagiCode megvalósítása részletes naplókat rögzít, hogy segítse a fejlesztőket a problémák hibaelhárításában. Ugyanakkor a kezelőfelület érvényesítési hibákat jelenít meg a felhasználói felületen, hogy segítse a felhasználókat a konfiguráció javításában. A jó hibakezelés természetesen jobb felhasználói élményhez vezet.
