## Teljes GLM-5.1 támogatás és Gemini CLI integráció: HagiCode útja a többmodell evolúcióhoz

> Ez a cikk a HagiCode platform két jelentős frissítését mutatja be: a Zhipu AI GLM-5.1 modell teljes körű támogatását és a Gemini CLI sikeres integrációját a tizedik Agent CLI-ként. Ezek a frissítések együtt tovább erősítik a platform többmodelles képességeit és több CLI ökoszisztémáját.

## Háttér

Az idő tényleg gyorsan telik. A nagy nyelvi modellek fejlődése tavasszal bambuszként emelkedik. Nem sokkal ezelőtt még szurkoltunk egy olyan AI-ért, amely képes kódot írni. Most már a többmodelles együttműködés és a többeszközös integráció korszakát éljük. Ez izgalmas? Talán. Végtére is, a fejlesztőknek soha nem csak magára az eszközre van szüksége, hanem a különböző forgatókönyvekhez való könnyű alkalmazkodásra és a rugalmas váltásra, amikor szükséges.

Mint mesterséges intelligencia által támogatott kódolási platform, a HagiCode a közelmúltban két fontos fejlesztést üdvözölt: először is a Zhipu AI GLM-5.1 modelljének teljes integrációját; Másodszor, a Gemini CLI hivatalos hozzáadása a tizedik támogatott CLI-ügynökként. Lehet, hogy ez a két frissítés nem hangzik megrendítőnek, de kétségtelenül jó hír a platform folyamatos fejlődéséhez.

A GLM-5.1 a Zhipu AI legújabb zászlóshajója. A GLM-5.0-hoz képest erősebb érvelést, mélyebb kódértést és gördülékenyebb eszközhívást kínál. Ennél is fontosabb, hogy ez az első GLM-modell, amely támogatja a képbevitelt. Ez mit jelent? Ez azt jelenti, hogy a felhasználók hagyhatják, hogy a mesterséges intelligencia közvetlenül egy képernyőképet nézzen ahelyett, hogy szavakkal leírná a problémát. Ha már kihasználta ezt a kényelmet, azonnal megérti az értékét.

Ugyanakkor a `HagiCode.Libs.Providers` architektúra, a HagiCode sikeresen integrálta a Gemini CLI-t a platformba. Ez immár a tizedik ügynök CLI. Őszintén szólva, idáig eljutni szerény sikerélményt hordoz magában.

Érdemes megemlíteni azt is, hogy a HagiCode képfeltöltési funkciója lehetővé teszi a felhasználók számára, hogy képernyőképeken keresztül közvetlenül kommunikáljanak az AI-val. Még a GLM 4.7 futtatásakor is jól működik a platform, és már számos fontos összeállítási feladat elvégzésében segített. Ami a GLM-5.1-et illeti, természetesen egy lépéssel tovább megy.

## Több CLI architektúra tervezés

A HagiCode egyik fő erőssége, hogy támogatja a több mesterséges intelligencia programozási CLI eszközt egy egységes absztrakciós rétegen keresztül. Ennek a kialakításnak az előnye valójában meglehetősen egyszerű: új eszközök jöhetnek, a régi eszközök maradhatnak, és a kódbázis nem válik káoszba. Az igazat megvallva, mindenki így szeretné, ha az élet működne.

### AIProviderType enum

A platform a támogatott CLI-szolgáltatótípusokat a `AIProviderType` enum:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI (new)
}
```

Amint látja, a Gemini CLI tizedik tagjaként csatlakozik ehhez a családhoz. Minden CLI-nek megvannak a saját jellegzetességei és használati forgatókönyvei, így a felhasználók rugalmasan választhatnak igényeik alapján. Hiszen sok út vezet Rómába; egyesek egyszerűen könnyebbek, mint mások.

### Szolgáltató architektúrája

`HagiCode.Libs.Providers` egységes szolgáltatói felületet biztosít, amely minden CLI-integrációt szabványossá és tömörsé tesz. A Gemini CLI példájaként:

```csharp
public class GeminiProvider : ICliProvider<GeminiOptions>
{
    private static readonly string[] DefaultExecutableCandidates = ["gemini", "gemini-cli"];
    private const string ManagedBootstrapArgument = "--acp";

    public string Name => "gemini";
    public bool IsAvailable => _executableResolver.ResolveFirstAvailablePath(DefaultExecutableCandidates) is not null;
}
```

Ennek a kialakításnak az előnyei a következők:
- Egy új CLI integrálásához csak egy Provider osztály megvalósítása szükséges
- Egységes életciklus-kezelés és munkamenet-összevonás
- Automatikus aliasfeloldás és végrehajtható felderítés

Egyszerűen fogalmazva, ez a kialakítás a bonyolult dolgokat egyszerűbbé varázsolja, és egy kicsit megkönnyíti az életet.

### Szolgáltatói nyilvántartás

A Szolgáltató Registry automatikusan kezeli az alias leképezést és regisztrációt:

```csharp
if (provider is GeminiProvider)
{
    registry.Register(provider.Name, provider, ["gemini-cli"]);
    continue;
}
```

Ez azt jelenti, hogy a felhasználók bármelyikkel meghívhatják a Gemini CLI-t `gemini` vagy `gemini-cli`, és a rendszer automatikusan felismeri. Olyan ez, mint egy barát, akinek hivatalos név és becenév is van – mindenesetre az emberek tudják, kire gondolsz.

## GLM-5.1 modell támogatás

A GLM-5.1 a Zhipu AI legújabb zászlóshajója, és a HagiCode teljes körű támogatást nyújtott hozzá.

### Másodlagos szakmák katalógusa

A HagiCode az összes támogatott modellt a másodlagos szakmakatalóguson keresztül kezeli. Íme a GLM sorozat konfigurációja:

| Modellazonosító | Név | SupportsImage | Kompatibilis CLI családok |
|----------|------|----------------|-------------------------|
| `glm-4.7` | GLM 4.7 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM 5 Turbo | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (örökölt) | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | **igaz** | claude, codebuddy, hermes, qoder, kiro |

A GLM-5.1 főbb jellemzői a következőkben foglalhatók össze:
- Önálló verzióazonosító, örökölt poggyász nélkül
- Az első GLM modell, amely támogatja a képbevitelt
- Erősebb érvelés és kódértés
- Széleskörű multi-CLI kompatibilitás

### GLM-5.1 vs GLM-5.0

Kódszinten a GLM-5.1 és a GLM-5.0 közötti fő különbség itt látható:

```csharp
// GLM-5.0 (Legacy) - contains special retention logic
private const string Glm50CodebuddySecondaryProfessionId = "secondary-glm-5-codebuddy";
private const string Glm50CodebuddyModelValue = "glm-5.0";

// GLM-5.1 - standalone new model identifier
private const string Glm51SecondaryProfessionId = "secondary-glm-5-1";
private const string Glm51ModelValue = "glm-5.1";
```

A GLM-5.0 a "Legacy" címkét viseli, mivel ez egy régi verzióazonosító, amelyet a visszamenőleges kompatibilitás érdekében megőriztek. Ezzel szemben a GLM-5.1 egy vadonatúj, önálló változat, történelmi terhek nélkül. Néhány dolog a múltban marad; mások könnyebben és gyorsabban haladnak.

### Konfigurálja a GLM-5.1-et

Íme egy konfigurációs példa a GLM-5.1 használatára a HagiCode-ban:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

## Gemini CLI integráció

A tizedik ügynök CLI-ként a Gemini CLI a szabványos Provider architektúrán keresztül integrálódik a HagiCode-ba.

### Konfigurációs lehetőségek

A Gemini CLI a konfigurációs lehetőségek gazdag készletét támogatja:

```csharp
public class GeminiOptions
{
    public string? ExecutablePath { get; set; }
    public string? WorkingDirectory { get; set; }
    public string? SessionId { get; set; }
    public string? Model { get; set; }
    public string? AuthenticationMethod { get; set; }
    public string? AuthenticationToken { get; set; }
    public Dictionary<string, string?> AuthenticationInfo { get; set; }
    public Dictionary<string, string?> EnvironmentVariables { get; set; }
    public string[] ExtraArguments { get; set; }
    public TimeSpan? StartupTimeout { get; set; }
    public CliPoolSettings? PoolSettings { get; set; }
}
```

Ezek az opciók az alapvető beállításoktól a haladó funkciókig mindent lefednek, így a felhasználók rugalmasan konfigurálhatják a CLI-t saját igényeik szerint. Mindenkinek más a munkafolyamata, ezért egy kis rugalmasság mindig üdvözlendő.

### ACP kommunikációs protokoll

A Gemini CLI támogatja az ACP-t (Agent Communication Protocol), amely a HagiCode egységes CLI kommunikációs szabványa. Az ACP-n keresztül a különböző CLI-k konzisztens módon kommunikálhatnak a platformmal, ami nagyban leegyszerűsíti az integrációs munkát. Röviden: szabványosítja a bonyolult részeket, így mindenki könnyebben tud dolgozni.

## A több CLI-s integráció legjobb gyakorlatai

`HagiCode.Libs.Providers` egységes mechanizmust biztosít a regisztrációhoz és a használathoz:

```csharp
services.AddHagiCodeLibs();

var gemini = serviceProvider.GetRequiredService<ICliProvider<GeminiOptions>>();
var codebuddy = serviceProvider.GetRequiredService<ICliProvider<CodebuddyOptions>>();
var hermes = serviceProvider.GetRequiredService<ICliProvider<HermesOptions>>();
```

Ez a függőségi befecskendezési kialakítás nagyon tömören tartja a különböző CLI-k használatát, és kényelmesebbé teszi az egységtesztelést és a gúnyolódást. A tiszta kód egy módja annak, hogy felelősséget vállalj önmagadért.

### Megjegyzések

A tényleges használat során néhány dolgot szem előtt kell tartani:

1. **API-kulcs-konfiguráció**: Győződjön meg róla `ANTHROPIC_AUTH_TOKEN` helyesen van beállítva, vagy a modell nem hívható meg
2. **Modell elérhetősége**: A GLM-5.1-et a megfelelő modellszolgáltatónak engedélyeznie kell
3. **Képjellemző**: Csak a következővel rendelkező modellek: `supportsImage: true` használhatja a képfeltöltést
4. **CLI telepítés**: A Gemini CLI használata előtt győződjön meg róla `gemini` vagy `gemini-cli` a PATH rendszerben van

Lehet, hogy apró részletekről van szó, de a rosszul kezelt apró részletek nagy problémákat okozhatnak, ezért érdemes odafigyelni rájuk.
