## Háttér

Ha a frontend-en dolgozik, valószínűleg korábban is volt már ilyen tapasztalata: egy termékmenedzser odamegy az "ez határozottan egyszerű" kifejezéssel, és azt mondja: "Adhatunk valamilyen speciális effektust ehhez a futó feladathoz, hogy a felhasználók azonnal észrevegyék?"

Ezen a ponton elkezdhet gondolkodni, hogyan kell megépíteni. Vászon? SVG? Vagy a CSS képes egyedül kezelni? Végtére is, senki sem akarja beismerni, hogy nem tudja, hogyan.

A modern webalkalmazásokban a szegélyfény-söpörő animációk valójában nagyon gyakoriak. Főleg néhány forgatókönyvben használatosak, például:

- **Szezonális témák**: Ünnepi hangulat kialakítása különleges alkalmakra

Pontosan ebbe a követelménybe ütköztünk az építés során [HagiCode](https://hagicode.com). A felhasználóknak egy pillantással látniuk kellett, hogy mely munkamenetek futnak, és mely javaslatok vannak éppen feldolgozás alatt. Többféle megközelítést is kipróbáltunk. Néhány utak simábbak voltak, néhányan kanyargósabbak voltak. Végül egy meglehetősen kiforrott megvalósítási stratégia mellett döntöttünk.

## Alapvető megvalósítási ötletek

A HagiCode kódbázis elemzése után az alábbiakban összefoglaltunk néhány alapvető megvalósítási mintát. Mindegyik más-más forgatókönyvhöz illik, vagy más szóval, mindegyik okkal létezik.

### 1. Forgó izzás kúpos gradienssel (leggyakoribb)

Ez a klasszikus módja a szegélyfény-seprő hatás megvalósításának. Az alapötlet a CSS használata `conic-gradient` kúpos gradiens létrehozásához, majd folyamatosan forgassa el. Mint az éjszakai utcai lámpa, csak kering.

**Főbb elemek:**
- Használja a `::before` pszeudoelemet a ragyogó réteg létrehozásához
- Használja `conic-gradient` a színátmenet színeloszlásának meghatározásához
- Használja a `::after` pszeudoelem a középső terület elfedésére (opcionális)
- Használja `@keyframes` a forgatási animáció megvalósításához

### 2. Oldalsó fényvonal

Ez jól működik a listaelemek állapotjelzőinél. A teljes szegély animálása helyett vékony, izzó vonalat hoz létre az elem egyik oldalán. Néha elég egy kis fény is. Nem kell az egész világot megvilágítani.

**Főbb elemek:**
- Vékony, abszolút pozicionált vonalelem
- Használja `box-shadow` ragyogó hatás létrehozásához
- Használja `scale` és `opacity` egy lélegző animációhoz

### 3. Ragyogó háttér doboz-árnyék

Ha nincs szüksége a teljes söprés effektusra, és csak egy lágy háttérfényre vágyik, több rétegben `box-shadow` értékek elegendőek. Néha az egyszerűbb lehetőség a jobb.

### 4. Kisegítő lehetőségek támogatása

Ezt a részt könnyű figyelmen kívül hagyni, de rendkívül fontos. Minden animációnak figyelembe kell vennie a `prefers-reduced-motion` médialekérdezést, és statikus alternatívát kínál azoknak a felhasználóknak, akik nem akarnak animációt. Nem mindenki élvezi az állandó mozgást, és ennek a preferenciának a tiszteletben tartása számít.

## Megvalósítási lehetőségek

### 1. lehetőség: Kúpos gradiens szegély elforgatása (ajánlott)

Ez a legteljesebb megvalósítása az elsöprő szegélyfény effektusnak, és a HagiCode-ban is ezt az opciót használjuk leggyakrabban. Végül is, ha valami jól működik, miért cserélik ki?

```css
/* Parent container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotating glow layer */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Mask layer (optional, for creating a hollow border effect) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Ennek az opciónak az alapelve meglehetősen egyszerű: hozzon létre egy pszeudoelemet, amely nagyobb, mint a szülőtároló, rajzoljon rá egy kúpos gradienst, és folyamatosan forgassa. A szülőtároló használja `overflow: hidden`, így csak a határon áthaladó fény marad látható. Kicsit olyan, mintha az utcai lámpát néznénk az ablakon keresztül. Mindig csak azt a kis szeletet látod, ami elhalad.

### 2. lehetőség: Egyszerűsített forgó fényszegély

Ha nincs szüksége a teljes effektusra, a HagiCode egy könnyebb segédprogram-osztályú verziót is tartalmaz. Néha valóban jobb az egyszerűbb megközelítés.

```css
/* Rotating light border utility class */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Accessibility support */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Vedd észre a `will-change: transform` itt. Azt mondja a böngészőnek: "Ez az elem folyamatosan változik", így a böngésző előre elkészíthet néhány optimalizálást, és simábbá teheti az animációt. Előre felkészülni általában jobb, mint az utolsó pillanatban tülekedni.

### 3. lehetőség: oldalsó izzóvonal

Ez különösen alkalmas listaelemek állapotjelzőire, és pontosan ezt használja a HagiCode munkamenet listája. Egy vékony vonal még mindig kiemelkedik a sok elem közül. Ez a maga módján egy életlecke.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Ez használ `isolation: isolate` új halmozási kontextus létrehozásához, majd erre támaszkodik `z-index` az egyes rétegek megjelenítési sorrendjének szabályozásához. `pointer-events: none` szintén elengedhetetlen. Ellenkező esetben a pszeudoelem blokkolná a felhasználói kattintásokat. Néhány dolog jól néz ki, de mégsem akadályozhatja őket.

### 4. lehetőség: csomagolja be egy React komponensbe

Ha a projektje Reactet használ, ezt a logikát egy összetevőbe csomagolhatja, különösen a kisegítő lehetőségek kezelésében. Írd le egyszer, használd sokszor. Ez az egész lényeg.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

A megfelelő CSS modul:

```css
/* GlowBorder.module.css */

/* Animated version */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Static version (accessibility) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

A `useReducedMotion` horog felől `framer-motion` automatikusan felismeri a felhasználó rendszerpreferenciáját. Ha a felhasználó engedélyezte a csökkentett mozgást, az visszatér `true`, és az összetevő a statikus verziót mutatja helyette. A felhasználó preferenciáinak tiszteletben tartása többet számít, mint a feltűnő hatás kikényszerítése.

## Gyakorlati leckék

Íme néhány tanulság, amit a HagiCode felépítése során tanultunk. Nevezhetjük őket harci hegeknek is. Remélhetőleg segítenek elkerülni néhány kitérőt.

### 1. Témaváltozó rendszer

A CSS-változók különösen kényelmessé teszik a több témájú támogatást. Senki sem akar egy halom kódot szerkeszteni minden alkalommal, amikor a téma megváltozik.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Usage */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

Ily módon a témaváltáshoz csak az osztály módosítása szükséges `html` elemet, és minden animáció színe automatikusan frissül. Egy kódbázis, két stílus. Pontosan ezt akarjuk.

### 2. Teljesítmény optimalizálás

** Használd `will-change` tipp a böngészőnek az optimalizálásra:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Előre szóljon a böngészőnek, és segít az optimalizálásban. Az életben sok minden jobban működik egy kis felkészüléssel.

**Kerülje az összetett keretes árnyékok használatát nagy elemeken:**

```css
/* Not ideal - using a blurred shadow on a large element */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Better - use a pseudo-element to limit the glowing area */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Ezt a HagiCode-ban teszteltük. Ha egy elmosódott árnyékot közvetlenül egy nagy kártyához adunk, a görgetési sebesség 30 képkocka/mp alá csökken. A pszeudoelemre váltás visszahozta a dolgokat egy állandó 60 képkocka/mp-re. A felhasználók teljesen érzik ezt a különbséget.

### 3. Hozzáférhetőség

Ezt tényleg nem szabad kihagyni. Egyes felhasználók szédítőnek vagy elvonónak találják az animációt, és preferenciáik tiszteletben tartása része a jó termék elkészítésének. A szép dolgokat nem kell mindenkire ráerőltetni.

**CSS médialekérdezés:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Provide a static fallback */
    opacity: 1;
  }
}
```

**A felhasználói preferenciák észlelése a Reactban:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Intenzitásszint szabályozás

A HagiCode Token áteresztőképesség-jelzője a valós idejű átviteli sebesség alapján különböző világító színeket mutat, és ez dinamikusan valósul meg. A különböző állapotokat másként kell kifejezni.

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Amikre figyelni kell

Van még néhány részlet, amire érdemes odafigyelni, mert mire ezeket a problémákat kemény úton felfedezi, már késő.

| Amikre figyelni kell | Magyarázat |
|---------|------|
| **z-index kezelése** | A fényes rétegnek megfelelő anyagot kell használnia `z-index` így nem zavarja a tartalmi interakciót |
| **pointer-események** | A világító pszeudoelemnek be kell állnia `pointer-events: none` |
| **Határtúlcsordulás** | A szülőtárolónak szüksége van `overflow: hidden`, vagy módosítania kell a pszeudoelem méretét |
| **Teljesítményhatás** | Az összetett animációk ronthatják a teljesítményt mobileszközökön, ezért alaposan tesztelje |
| **Sötét mód** | Győződjön meg arról, hogy a ragyogás színe jól látható marad sötét háttéren |
| **Témaváltás** | Használjon CSS-változókat, hogy az animációs színek megfelelően frissüljenek a téma megváltozásakor |

### 6. Hibakeresési tippek

A pszeudoelemeket kissé nehéz megtalálni a fejlesztői eszközökben, ezért ideiglenesen hozzáadhat egy szegélyt a pozíció ellenőrzéséhez.

```css
/* Temporarily show pseudo-element boundaries for debugging */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Miután befejezte a pozicionálást, ne felejtse el kommentálni vagy eltávolítani a sort. Ellenkező esetben a gyártás elég gyorsan kínossá válhat. Néhány dolgot jobb a fejlesztésben hagyni.
