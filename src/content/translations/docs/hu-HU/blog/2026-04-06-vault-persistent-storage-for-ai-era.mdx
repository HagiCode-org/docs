## Projektek reprodukálása az AI-korszakban: Vault, a projektek közötti állandó tárolórendszer

> Az AI által támogatott fejlesztés korszakában hogyan segíthetünk az AI-asszisztenseknek jobban megérteni tanulási erőforrásainkat? A HagiCode projekt a Vault rendszert olyan egységes tudástároló absztrakciós rétegként építette fel, amelyet az AI megért, és jelentősen javítja a tanulás hatékonyságát a projektreprodukció révén.

## Háttér

A mesterséges intelligencia korszakában a fejlesztők új technológiák és architektúrák elsajátításának módja alaposan megváltozik. A „projektek reprodukálása” – vagyis a kiváló nyílt forráskódú projektek kódjának, architektúrájának és tervezési mintáinak mélyreható tanulmányozása és tanulása – a tanulás hatékony módja lett. A hagyományos módszerekkel, például könyvolvasással vagy videók nézésével összehasonlítva a kiváló minőségű nyílt forráskódú projektek közvetlen olvasása és futtatása sokkal gyorsabban segít megérteni a valós világ mérnöki gyakorlatait.

Ennek a tanulási módszernek azonban számos kihívása van.

A tananyagok túlságosan szétszórtak. Jegyzetei az Obszidiánban élhetnek, a kódtárak szétszórva lehetnek különböző mappákban, és az AI-asszisztens beszélgetési előzményei egy újabb elszigetelt adatszigetté válhatnak. Ha azt szeretné, hogy a mesterséges intelligencia segítsen elemezni egy projektet, manuálisan kell másolnia a kódrészleteket, és rendszereznie kell a környezetet, ami meglehetősen fárasztó.

Ami még zavaróbb, az a hibás kontextus. Az AI-asszisztensek nem férhetnek hozzá közvetlenül a helyi tanulási forrásokhoz, ezért minden beszélgetés során újra háttérinformációkat kell megadnia. Ráadásul a reprodukált kódtárak gyorsan frissülnek, a kézi szinkronizálás hibás, és a tudást nehéz megosztani több tanulási projekt között.

Alapvetően ezek a problémák az „adatszigetekből” származnak. Ha létezne egy egységes tárolási absztrakciós réteg, amely lehetővé tenné az AI-asszisztensek számára, hogy megértsék és hozzáférjenek az összes tanulási erőforráshoz, a probléma szépen megoldódna.

## Vault rendszer tervezési filozófia

A Vault rendszer alapötlete egyszerű: hozzon létre egy egységes tudástároló absztrakciós réteget, amelyet az AI megért. A megvalósítás szempontjából a rendszernek számos kulcsfontosságú jellemzője van.

### Több típusú támogatás

A rendszer négy tárolótípust támogat, amelyek mindegyike más-más használati forgatókönyvnek felel meg:

```typescript
// folder: general-purpose folder type
export const DEFAULT_VAULT_TYPE = 'folder';

// coderef: a type specifically for reproduced code projects
export const CODEREF_VAULT_TYPE = 'coderef';

// obsidian: integrated with Obsidian note-taking software
export const OBSIDIAN_VAULT_TYPE = 'obsidian';

// system-managed: vault automatically managed by the system
export const SYSTEM_MANAGED_VAULT_TYPE = 'system-managed';
```

Közülük a **coderef** típus a leggyakrabban használt a HagiCode-ban. Kifejezetten reprodukált kódprojektekhez készült, szabványos címtárstruktúrát és mesterséges intelligencia által olvasható metaadatleírásokat biztosítva.

### Állandó tárolási mechanizmus

A Vault beállításjegyzék állandóan JSON formátumban van tárolva, biztosítva, hogy a konfiguráció elérhető maradjon az alkalmazás újraindítása után is:

```csharp
public class VaultRegistryStore : IVaultRegistryStore
{
    private readonly string _registryFilePath;

    public VaultRegistryStore(IConfiguration configuration, ILogger<VaultRegistryStore> logger)
    {
        var dataDir = configuration["DataDir"] ?? "./data";
        var absoluteDataDir = Path.IsPathRooted(dataDir)
            ? dataDir
            : Path.GetFullPath(Path.Combine(Directory.GetCurrentDirectory(), dataDir));

        _registryFilePath = Path.Combine(absoluteDataDir, "personal-data", "vaults", "registry.json");
    }
}
```

Ennek a kialakításnak az az előnye, hogy egyszerű és megbízható. A JSON ember által olvasható, ami megkönnyíti a hibakeresést és a kézi szerkesztést; A fájlrendszer tárolása elkerüli az adatbázisok bonyolultságát és csökkenti a rendszerfüggőségeket. Végül is néha a legegyszerűbb lehetőség a legjobb.

### AI kontextus integráció

A legfontosabb, hogy a rendszer automatikusan beillesztheti a tárolóinformációkat az AI-javaslatok kontextusába:

```typescript
export function buildTargetVaultsText(
  vaults: VaultForText[],
  template: VaultPromptTemplate = DEFAULT_VAULT_PROMPT_TEMPLATE,
): string {
  const readOnlyVaults = vaults.filter((vault) => vault.accessType === 'read');
  const editableVaults = vaults.filter((vault) => vault.accessType === 'write');

  if (readOnlyVaults.length === 0 && editableVaults.length === 0) {
    return '';
  }

  const sections = [
    buildVaultSection(readOnlyVaults, template.reference),
    buildVaultSection(editableVaults, template.editable),
  ].filter(Boolean);

  return `\n\n### ${template.heading}\n\n${sections.join('\n')}`;
}
```

Ez egy fontos képességet tesz lehetővé: az AI-asszisztensek automatikusan megérthetik a rendelkezésre álló tanulási erőforrásokat anélkül, hogy a felhasználók manuálisan megadnák a kontextust. Mondhatni, ez egyfajta hallgatólagos megértésnek számít.

## A CodeRef Vault szabványos szerkezete

A coderef típusú tárolóhoz a HagiCode szabványos könyvtárstruktúrát biztosít:

```
my-coderef-vault/
├── index.yaml          # vault metadata description
├── AGENTS.md           # operating guide for AI assistants
├── docs/               # stores study notes and documents
└── repos/              # manages reproduced code repositories through Git submodules
```

A tároló létrehozásakor a rendszer automatikusan inicializálja ezt a struktúrát:

```csharp
private async Task EnsureCodeRefStructureAsync(
    string vaultName,
    string physicalPath,
    ICollection<VaultBootstrapDiagnosticDto> diagnostics,
    CancellationToken cancellationToken)
{
    Directory.CreateDirectory(physicalPath);

    var indexPath = Path.Combine(physicalPath, CodeRefIndexFileName);
    var docsPath = Path.Combine(physicalPath, CodeRefDocsDirectoryName);
    var reposPath = Path.Combine(physicalPath, CodeRefReposDirectoryName);

    // Create the standard directory structure
    if (!Directory.Exists(docsPath))
    {
        Directory.CreateDirectory(docsPath);
    }

    if (!Directory.Exists(reposPath))
    {
        Directory.CreateDirectory(reposPath);
    }

    // Create the AGENTS.md guide
    await EnsureCodeRefAgentsDocumentAsync(physicalPath, cancellationToken);

    // Create the index.yaml metadata
    await WriteCodeRefIndexDocumentAsync(indexPath, mergedDocument, cancellationToken);
}
```

Ez a szerkezet is gondosan megtervezett:

- A **docs/** tárolja tanulmányi jegyzeteit, ahol rögzítheti a kód megértését, az architektúra elemzését, a tanulságokat és így tovább a Markdown alkalmazásban.
- A **repos/** a reprodukált adattárakat a Git almodulokon keresztül kezeli a kód közvetlen másolása helyett, ami szinkronban tartja a kódot és helyet takarít meg
- Az **index.yaml** tartalmazza a trezor metaadatait, így az AI-asszisztensek gyorsan megérthetik a trezor célját és tartalmát
- Az **AGENTS.md** egy kifejezetten az AI-asszisztensek számára készült útmutató, amely elmagyarázza, hogyan kell kezelni a trezor tartalmát

Így megszervezve a mesterséges intelligencia talán egy kicsit könnyebben megérti, mire gondol.

## Automatikus inicializálás a rendszer által felügyelt tárolókhoz

A manuálisan létrehozott tárolókon kívül a HagiCode támogatja a rendszer által felügyelt tárolókat is:

```csharp
public async Task<IReadOnlyList<VaultRegistryEntry>> EnsureAllSystemManagedVaultsAsync(
    CancellationToken cancellationToken = default)
{
    var definitions = GetAllResolvedDefinitions();
    var entries = new List<VaultRegistryEntry>(definitions.Count);

    foreach (var definition in definitions)
    {
        entries.Add(await EnsureResolvedSystemManagedVaultAsync(definition, cancellationToken));
    }

    return entries;
}
```

A rendszer automatikusan létrehozza és kezeli a következő tárolókat:

- **hagiprojectdata**: projekt adattárolása a projekt konfigurációjának és állapotának mentésére
- **személyes adatok**: személyes adatok tárolása a felhasználói beállítások mentésére
- **hbsprompt**: a gyakran használt mesterséges intelligencia-kérdések kezelésére szolgáló prompt sablonkönyvtár

Ezeket a tárolókat a rendszer automatikusan inicializálja a rendszer indításakor, így a felhasználóknak nem kell őket manuálisan konfigurálniuk. Néhány dolgot egyszerűen jobb a rendszerre bízni, ahelyett, hogy az emberek aggódnának értük.

## Beléptető mechanizmus

A tervezés fontos része a beléptetés. A rendszer a trezorokat két hozzáférési típusra osztja:

```typescript
export interface VaultForText {
  id: string;
  name: string;
  type: string;
  physicalPath: string;
  accessType: 'read' | 'write';  // Key: distinguish read-only from editable
}
```

- **hivatkozás (csak olvasható)**: Az AI-t csak elemzésre és megértésre használják, és nem módosíthatja a tartalmat. Alkalmas hivatkozott nyílt forráskódú projektekhez, dokumentumokhoz és hasonló anyagokhoz
- **szerkeszthető (szerkeszthető)**: Az AI a feladathoz szükség szerint módosíthatja a tartalmat. Alkalmas jegyzeteihez, piszkozataihoz és hasonló anyagokhoz

Ez a megkülönböztetés számít. Megmondja az AI-nak, hogy melyik tartalom "csak olvasható", és melyik tartalom "biztonságosan szerkeszthető", csökkentve a véletlen változtatások kockázatát. Végtére is, senki sem akarja, hogy a kemény munkája eltűnjön egy nem szándékos szerkesztés miatt.

## A gyakorlatban: Vault létrehozása és használata

Most, hogy áttekintettük az ötleteket, nézzük meg, hogyan működik a gyakorlatban.

### Hozzon létre egy CodeRef Vaultot

Íme egy teljes frontend hívási példa:

```typescript
const createCodeRefVault = async () => {
  const response = await VaultService.postApiVaults({
    requestBody: {
      name: "React Learning Vault",
      type: "coderef",
      physicalPath: "/Users/developer/vaults/react-learning",
      gitUrl: "https://github.com/facebook/react.git"
    }
  });

  // The system will automatically:
  // 1. Clone the React repository into vault/repos/react
  // 2. Create the docs/ directory for notes
  // 3. Generate the index.yaml metadata
  // 4. Create the AGENTS.md guide file

  return response;
};
```

Ez az API-hívás egy sor műveletet hajt végre: a címtárstruktúra létrehozása, a Git-almodulok inicializálása, a metaadatfájlok generálása és még sok más. Csak az alapvető információkat kell megadnia, a többit pedig hagyni kell a rendszernek. Őszintén szólva ez egy meglehetősen aggodalommentes megközelítés.

## Bevált gyakorlatok és dolgok, amelyekre figyelni kell

A Vault rendszer használata során összefoglaltunk néhány tanulságot.

### Útbiztonság

A rendszer szigorúan érvényesíti az útvonalakat, hogy megakadályozza az útvonal-bejárási támadásokat:

```csharp
private static string ResolveFilePath(string vaultRoot, string relativePath)
{
    var rootPath = EnsureTrailingSeparator(Path.GetFullPath(vaultRoot));
    var combinedPath = Path.GetFullPath(Path.Combine(rootPath, relativePath));
    if (!combinedPath.StartsWith(rootPath, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode,
            "Vault file paths must stay inside the registered vault root.");
    }
    return combinedPath;
}
```

Ez fontos. Ha testreszab egy tároló elérési utat, győződjön meg arról, hogy az a megengedett tartományon belül marad, ellenkező esetben a rendszer elutasítja a műveletet. Tényleg nem lehet eléggé hangsúlyozni a biztonságot.

### Git almodul kezelése

A CodeRef Vault a Git almodulokat ajánlja a kód közvetlen másolása helyett:

```csharp
private static string BuildCodeRefAgentsContent()
{
    return """
    # CodeRef Vault Guide

    Repositories under `repos/` should be maintained through Git submodules
    rather than copied directly into the vault root.

    Keep this structure stable so assistants and tools can understand the vault quickly.
    """ + Environment.NewLine;
}
```

Ez számos előnnyel jár: a kód szinkronban tartása az upstream-el, lemezterületet takarít meg, és megkönnyíti a kód több verziójának kezelését. Hiszen ki akarja újra és újra letölteni ugyanazt?

### Fájl-előnézeti korlátok

A teljesítményproblémák elkerülése érdekében a rendszer korlátozza a fájl méretét és típusát:

```csharp
private const int FileEnumerationLimit = 500;
private const int PreviewByteLimit = 256 * 1024;  // 256KB
```

Ha a tároló nagyszámú vagy nagyon nagy fájlt tartalmaz, az befolyásolhatja az előnézeti teljesítményt. Ebben az esetben fontolóra veheti a fájlok kötegelt feldolgozását vagy speciális keresőeszközök használatát. Néha, ha valami túl nagy, akkor nehezebb lesz kezelni, nem pedig könnyebb.

### Diagnosztikai információk

A tároló létrehozásakor a rendszer diagnosztikai információkat ad vissza a hibakereséshez:

```csharp
List<VaultBootstrapDiagnosticDto> bootstrapDiagnostics = [];

if (IsCodeRefVaultType(normalizedType))
```

Ha a létrehozás sikertelen, megvizsgálhatja a diagnosztikai információkat, hogy megértse a konkrét okot. Ha valami elromlik, gyakran a diagnosztika ellenőrzése a legközvetlenebb út.
