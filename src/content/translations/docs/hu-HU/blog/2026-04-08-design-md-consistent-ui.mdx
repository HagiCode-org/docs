## Design.md: Megoldás a konzisztens, mesterséges intelligencia által vezérelt felhasználói felület kialakításához

> A mesterséges intelligencia által támogatott frontend-fejlesztés korszakában hogyan lehet konzisztenssé tenni a mesterséges intelligencia által generált felhasználói felületeket? Ez a cikk az awesome-design-md-n alapuló tervezési galéria-webhely felépítésével kapcsolatos gyakorlati tapasztalatainkat, valamint strukturált `design.md` hogy a mesterséges intelligencia a szabványosított felhasználói felület felé terelje.

## Háttér

Aki használt mesterséges intelligenciát frontend kód írására, valószínűleg hasonló élményben volt része: kérje meg az AI-t, hogy többször hozza létre ugyanazt az oldalt, és minden eredmény más stílusban jelenik meg. Néha a sarkok lekerekítettek, néha élesek. A térköz néha 8 képpont, máskor pedig 16 képpont. Még ugyanaz a gomb is eltérően nézhet ki a különböző beszélgetésekben.

Ez nem elszigetelt probléma. Ahogy a mesterséges intelligencia által támogatott fejlesztés egyre gyakoribbá válik, a **konzisztencia hiánya a mesterséges intelligencia által generált frontend felhasználói felületen** széles körben elterjedt problémává vált. Különböző mesterséges intelligencia asszisztensek, különböző promptok és még ugyanaz az asszisztens is a különböző beszélgetések során drámaian eltérő interfész-terveket hozhat létre. Ez hatalmas karbantartási költséget jelent a termék iterációja során.

A kiváltó ok valójában egyszerű: nincs **hiteles tervezési referenciadokumentum**. A hagyományos CSS-stíluslapok megmondhatják a fejlesztőknek, hogy "hogyan kell megvalósítani" valamit, de nem tudják teljes mértékben közölni, hogy "miért van így tervezve" vagy "melyik tervezési mintát melyik forgatókönyvben kell használni". A mesterséges intelligencia esetében a világos és strukturált leírás még fontosabb a tervezési konvenciók megértéséhez.

Ugyanakkor a nyílt forráskódú közösség már most is kínál néhány kiváló forrást. A [VoltAgent/awesome-design-md](https://github.com/VoltAgent/awesome-design-md) projekt számos jól ismert cég tervezési rendszerdokumentációját gyűjti össze. Minden könyvtár tartalmaz egy `README.md`, a `DESIGN.md`, és tekintse meg a HTML előnézetét. Mindez azonban szétszórva található az upstream adattárban, ami megnehezíti a böngészést és a gyors összehasonlítást.

Tehát összevonhatjuk-e ezeket az erőforrásokat egy könnyen áttekinthető tervezési galériában, miközben egy strukturált `design.md` AI használni?

A válasz igen. Ezután hadd járjam végig a megközelítésünket.

### 1. Content Ingestion Pipeline

Ez az egész rendszer alapja, amely felelős a tartalom kinyeréséért és normalizálásáért az upstream erőforrásokból.

```typescript
// src/lib/content/awesomeDesignCatalog.ts

export async function scanSourceEntries(): Promise<DesignEntry[]> {
  const designDir = 'vendor/awesome-design-md/design-md';
  const entries: DesignEntry[] = [];

  for (const dir of await fs.readdir(designDir)) {
    const entryPath = path.join(designDir, dir);
    if (await isValidDesignEntry(entryPath)) {
      const entry = await normalizeDesignEntry(entryPath);
      entries.push(entry);
    }
  }

  return entries;
}

async function isValidDesignEntry(dir: string): Promise<boolean> {
  const requiredFiles = ['README.md', 'DESIGN.md'];
  for (const file of requiredFiles) {
    if (!(await fileExists(path.join(dir, file)))) {
      return false;
    }
  }
  return true;
}
```

### 3. Keresési funkciók

A keresési funkció az ügyféloldali szűrésen alapul, az állapotot az URL-lekérdezési paramétereken keresztül megőrzi:

```typescript
// src/components/gallery/SearchToolbar.tsx

function SearchToolbar({ entries }: { entries: DesignEntry[] }) {
  const [query, setQuery] = useState('');
  const [results, setResults] = useState(entries);

  useEffect(() => {
    const params = new URLSearchParams(window.location.search);
    const initialQuery = params.get('q') || '';
    setQuery(initialQuery);
    filterEntries(initialQuery);
  }, []);

  const filterEntries = (searchQuery: string) => {
    const filtered = entries.filter(entry =>
      entry.searchText.toLowerCase().includes(searchQuery.toLowerCase())
    );
    setResults(filtered);
  };

  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const value = e.target.value;
    setQuery(value);
    filterEntries(value);

    // Update the URL without triggering a page refresh
    const newUrl = value
      ? `${window.location.pathname}?q=${encodeURIComponent(value)}`
      : window.location.pathname;
    window.history.replaceState({}, '', newUrl);
  };

  return (
    <div className="search-toolbar">
      <input
        type="text"
        value={query}
        onChange={handleChange}
        placeholder="Search design entries..."
      />
      <span className="result-count">{results.length} results</span>
    </div>
  );
}
```

**3. lépés: A galéria felhasználói felület létrehozása**

Az Astro + React Islands segítségével hozza létre:
- Kezdőlap galéria elrendezése (kártyarács)
- Tervezési kártya alkatrészek
- Keresés eszköztár

**4. lépés: Írja meg a tervdokumentumot**

A ClickHouse felépítése alapján `DESIGN.md`, töltse ki a tényleges tervezési tokeneket a saját projektjéből. Frissítés `README.md` és adj hozzá egy linket `design.md`.

### Megjegyzések

**Biztonság**: A leértékeléshez szükséges a nem biztonságos HTML szűrése. Az Astro beépített renderelő szűrői `script` címkék alapértelmezés szerint, de továbbra is figyelnie kell az XSS kockázatokat.

**Teljesítmény**: Az iframe előnézetek nagy száma befolyásolhatja az első festés teljesítményét. Használata javasolt `loading="lazy"` az előnézeti tartalom lusta betöltéséhez.

**Karbantarthatóság**: `design.md` szinkronban kell maradnia a kód implementációjával. Javasoljuk a CI-ellenőrzések hozzáadását annak biztosítására, hogy a CSS-változók konzisztensek maradjanak a dokumentáció és a kód között.

**Hozzáférhetőség**: Győződjön meg arról, hogy a színkontraszt megfelel a WCAG AA szabványnak (legalább 4,5:1).

### AI használati útmutató

Létrehozás után `design.md`, hogyan lehet rávenni az AI-t a tényleges használatára? Íme néhány gyakorlati tipp:

**1. tipp: kifejezetten hivatkozzon rá a promptban**

```
Please refer to the design.md file in the project root and use the design conventions defined there to implement the following components:
- Buttons: use the primary color with an 8px border radius
- Cards: use the elevation-2 shadow level
```

**2. tipp: A mesterséges intelligencia megkövetelése, hogy konkrét CSS-változókra hivatkozzon**

```
Implement a navigation bar with the following requirements:
- Use --color-bg-primary for the background color
- Use --color-border-subtle for borders
- Use --text-color-primary for text
```

** 3. tipp: Tartalmazza `design.md` tartalom a rendszerpromptban**

Ha a mesterséges intelligencia eszköze támogatja az egyéni rendszerkéréseket, hozzáadhatja az alapvető tartalmat `design.md` közvetlenül hozzá.

### Tesztelési stratégia

**Tartalomfolyamat tesztelése**:
- Hiányzó fájlok forgatókönyvei (hiányzó `README.md` vagy `DESIGN.md`)
- Formázási hiba forgatókönyvei (Markdown elemzési hiba)
- Üres könyvtár forgatókönyvek

**Keresési funkciók tesztelése**:
- Üres eredménykezelés
- Speciális karakterek, például kínai és hangulatjelek
- URL-szinkronizálás ellenőrzése

**UI-komponens tesztelése**:
- Világos/sötét téma váltás
- Reszponzív elrendezés
- A betöltési állapotok előnézete
