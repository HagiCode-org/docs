## 12 kivételes indítási élmény tervezése a HagiCode számára: a minimalizmustól a cyberpunkig

> A React 19 alkalmazás letöltése és a Hidratálás befejezése között eltelt rövid idő remek lehetőség a felhasználók számára, hogy megtapasztalják a márka személyiségét. Ebben a cikkben egy teljes indítási stílusú rendszert osztunk meg, amelyet a HagiCode projekthez építettünk HTML/CSS/JS használatával.

## Háttér

Az ASP.NET Core 10 és React 19 (Vite) segítségével felépített modern alkalmazásként a HagiCode frontend-backend elkülönített telepítési architektúrát használ. A frontend build kimenete a háttérbe van csomagolva `wwwroot/` könyvtárat, és az ASP.NET Core üzemelteti.

Ez az architektúra azonban bevezet egy klasszikus UX-fájdalompontot: amikor a felhasználók felkeresik az oldalt, a böngésző először betölti a HTML-kódot, majd letölti a nagy JS-csomagot, végül pedig hagyja, hogy a React elvégezze a hidratálást. A néhány száz milliszekundumtól néhány másodpercig tartó „vákuumidőszak” alatt a felhasználók vagy üres képernyőt vagy élettelen statikus oldalt látnak.

Ennek a hiánynak a kitöltéséhez és a HagiCode márkaszemélyiségének beillesztéséhez meg kellett terveznünk egy indítási stílusú rendszert, amely teljes egészében beépített kóddal volt megvalósítva. `index.html`.

## A HagiCode-ról

Az ebben a cikkben megosztott indítóképernyő-tervezési megközelítés a gyakorlati tapasztalatainkból származik [HagiCode](https://github.com/HagiCode-org/site) projekt. AI kódolási asszisztensként a HagiCode nem csak a kódgenerálás hatékonyságával törődik, hanem a fejlesztő vizuális élményével is. Ez az indítási rendszer a végső frontend teljesítményre való törekvésünk egyik eredménye.

## Az alapvető kihívások és az építészeti tervezés

Mielőtt elkezdtük a tervezést, először tisztáznunk kellett a műszaki korlátokat. Mivel mindent inline kellett megvalósítani `index.html`, nem tudtunk betölteni semmilyen külső CSS- vagy JS-fájlt, kivéve a React saját csomagját.

### Technikai korlátok elemzése

1.  **Zéró függőségi elv**: Minden stílusnak a belül kell élnie `<style>` címke, és minden logikának az a belsejében kell élnie `<script>` címke.
2.  **Defenzív CSS**: Annak elkerülése érdekében, hogy a globális stílusok szennyezzék a nyitóképernyőt a React alkalmazás csatlakoztatása után, úgy döntöttünk, hogy az összes indítási stílust magas prioritású azonosító előtaggal csomagoljuk, mint pl. `#boot-screen`.
3.  **A teljesítmény mindenekelőtt**: Az animációknak CSS-t kell használniuk `transform` és `opacity` ahol csak lehetséges, elkerülje a visszafolyást, és biztosítsa, hogy a főszál ne legyen blokkolva.
4.  **Vizuális konzisztencia**: A színeknek és a betűtípusoknak összhangban kell lenniük a HagiCode Tailwind konfigurációjával.

### Építészeti minta: Shell & Injector

Elfogadtunk egy **változatmintát**. Az alapvető logika egy azonnal meghívott függvénykifejezésbe (IIFE) van beépítve, míg a konkrét megjelenítési logika konfiguráción keresztül kerül beillesztésre. Ez lehetővé teszi, hogy egyszerű konfigurációval válthassunk a különböző stílusok között, ahelyett, hogy ismételten átírnánk a DOM-manipulációs logikát.

Íme az alapvető architektúra kódja:

```html
<!-- Inline in index.html -->
<div id="boot-root"></div>

<script>
(function() {
  const BootSequence = {
    config: {
      theme: 'terminal', // Configurable as 'minimal', 'skeleton', 'code-rain', etc.
      color: '#3b82f6'   // Brand color
    },
    
    // Core lifecycle
    init() {
      this.render();
      this.listenForMount();
    },

    // Render the currently selected style
    render() {
      const root = document.getElementById('boot-root');
      if (this.variants[this.config.theme]) {
        root.innerHTML = this.variants[this.config.theme].render();
      }
    },

    // Listen for successful React mount and exit gracefully
    listenForMount() {
      window.addEventListener('hagicode:ready', () => {
        const screen = document.getElementById('boot-root');
        // Fade out first, then remove the DOM to avoid flicker
        screen.style.opacity = '0';
        screen.style.transition = 'opacity 0.3s ease';
        setTimeout(() => screen.remove(), 300);
      });
    },

    // The implementation logic for all 12 styles lives here
    variants: {
      // ...see details below
    }
  };

  BootSequence.init();
})();
</script>
```

## 12 indítási stílust tartalmazó ellenőrzőlista

Ezt a 12 stílust hat fő kategóriába soroltuk, hogy megfeleljünk a különböző forgatókönyveknek és esztétikai preferenciáknak.

### A. Minimalizmus

> "A kevesebb több." A maximális betöltési sebességre törekvő forgatókönyvekhez a lehető legkönnyebb opciókat kínáljuk.

#### 1. Minimalista pont
Egy egyszerű pont a képernyő közepén helyezkedik el, lélegző animációval párosítva.
*   **Megvalósítás**: CSS `@keyframes` szabályozza a léptéket és az átlátszóságot.
*   **A legjobb**: Minden olyan eset, amikor az oldalnak teljesen tisztának kell maradnia.

#### 2. Márka felfedése
SVG használata `stroke-dasharray` animáció, a HagiCode logósorok kézzel rajzolt feltárását szimulálja, majd a szöveg elhalványulását.
*   **Technika**: SVG útvonal-animáció rendkívül csiszolt érzéssel.

### B. Csontváz képernyőmimikri

> – A szem megtévesztésének művészete. Valódi felhasználói felület-elrendezés szimulálásával a felhasználók úgy érzik, hogy az oldal már félig be van töltve.

#### 3. Sidebar Chat Skeleton
Ez lehet a legpraktikusabb lehetőség. Kézzel készítettünk egy elrendezést HTML-ben, amely tükrözi a Reactot `Sidebar` és `ChatInput` komponenseket pontosan, majd egy szürke csillogó animációval fedje be.
*   **Érték**: Amikor a React hidratálás befejeződik, a csontváz azonnal az igazi komponenssé válik, és a felhasználók alig érzékelik a váltást.

#### 4. Kártyaverem csontváz
Simulálja az ajánlatkártyák egymásra halmozott mozgását betöltés közben, és 3D transzformációkat használ, hogy a kártyákat finoman lebegtesse.

### C. Absztrakció és művészet

> Mutasd meg a HagiCode geek DNS-ét.

#### 5. Geometriai Morph
Egy geometriai alakzat (négyzet) a képernyő közepén jelenik meg, majd az idő múlásával simán körré, háromszöggé és végül logóvá alakul.
*   **Technológia**: Sima átmenetek CSS-sel `border-radius`.

#### 6. Code Rain
Tisztelet *A Mátrix* előtt. A JetBrains Mono betűtípus használatával halvány karakterfolyamok esnek a háttérben.
*   **Megjegyzés**: A teljesítmény érdekében a karakterfolyamoknak kisebb területen belül kell maradniuk, vagy alacsonyabb frissítési gyakoriságot kell használniuk.

#### 7. Neon Pulse
Egy cyberpunk stílusú izzó gyűrű, amely több elemet használ `box-shadow` rétegekkel, hogy erőteljes neonfényt hozzon létre.

### D. Márkaépítés és témák

> Érezze, hogy a rendszer él.

#### 8. Szezonális téma
Ez egy dinamikus rakodó. Ellenőrzi az aktuális dátumot az ünnepek, például holdújév vagy karácsony esetén, és betölti a megfelelő SVG-animációt.
*   **Példa**: Holdújév idején a vörös lámpások finoman himbálóznak a képernyő alján.

#### 9. Gradiens Flow
A háttér a HagiCode márkaszíneken alapuló folyékony gradienst használ. Animációval kombinálva `background-size` és `background-position`, aurora-szerű mozgásérzetet kelt.

### E. Technikai érzés

> Üdvözlet a fejlesztőknek.

#### 10. Terminál rendszerindítás
A konzol kimenetét szimulálja. A kódsorok gyorsan gördülnek:
```text
> Initializing HagiCode Core...
> Loading models...
> Connecting to neural network...
```
Ez azonnal ismerősnek tűnik minden fejlesztő számára.

#### 11. Folyamat sáv
A képernyő tetején egy vékony folyamatjelző sáv jelenik meg, a jobb oldalon pedig egy százalék látható. Bár nem tudjuk elérni a valós letöltési folyamatot, egy időzítő segítségével szimulálhatunk egy "hihető" betöltési folyamatot: az első 80%-ban gyorsan, majd az utolsó 20%-ban fokozatosan lassabban.

### F. Kreativitás

#### 12. Pixel Assembly
Ez egy nagyon érdekes ötlet. A kis négyzetek szétszórva jelennek meg a képernyőn, majd a közepe felé konvergálnak, és fokozatosan összeállnak a HagiCode logó ikonjává. Az építési szabályzat folyamatát szimbolizálja.

## Bevált gyakorlatok és buktatók

A HagiCode valós fejlesztési munkája során több kritikusan fontos megvalósítási részletet foglaltunk össze.

### 1. A védekező CSS kötelező
Soha ne lustálkodjon, és hagyja ki az előtagot. Egyszer elfelejtettük az indítóképernyő-stílusokat azonosítóval és globálisan kezelni `div` a React beillesztése utáni stílusok váratlanul befolyásolták a nyitóképernyőt, megtörve az elrendezést.
**Leckék**: Tegyen minden CSS-választót alá `#boot-screen`, és használja `!important` hogy szükség esetén emelje fel a prioritást, de csak a CSS indítóképernyőn belül.

### 2. Kecses átmenetek
A React sikeres csatlakoztatása után ne közvetlenül `remove()` a DOM indítóképernyőt.
**Helyes megközelítés**:
1.  Reakció triggerek `window.dispatchEvent(new Event('hagicode:ready'))`.
2.  A nyitóképernyő figyeli az eseményt és az első sorozatokat `opacity: 0`.
3.  Várjon 300 ms-t, ami megegyezik a CSS átmenet időtartamával, és hívjon `.remove()` csak akkor, ha a képernyő teljesen láthatatlan.

### 3. Téma változó szinkronizálása
A kezdőképernyő színértékei be vannak kódolva `index.html`. Ha megváltoztatjuk a Tailwind elsődleges színét, frissítenünk kell a nyitóképernyőt is.
**Optimalizálás**: Írjon egy egyszerű beépülő modult a Vite build szkriptbe az olvasáshoz `tailwind.config.js` és szúrjon be színváltozókat a `index.html` sablonváltozókat, egyetlen igazságforrást hozva létre.

### 4. Betűtípus előtöltése
A kezdőképernyőknek gyakran márka betűtípust kell használniuk, de ha a betűtípus lassan töltődik be, megjelenhet a FOUT (Flash of Unstyled Text).
**Megoldás**: Add `<link rel="preload" href="/fonts/JetBrainsMono.woff2" as="font" type="font/woff2" crossorigin>` belül `<head>`. Ez egy alacsony költségű, magas megtérülésű módszer az élmény javítására.

### 5. Teljesítményfigyelés
Beadtuk `performance.mark('boot-start')` alján `index.html`, és megjelölve `boot-end` amikor a React sikeresen fel lett szerelve.
**Miért számít**: Ha ezeket az adatokat az Application Insights szolgáltatáson keresztül gyűjtjük, közvetlenül mérhetjük, hogy a nyitóképernyő mennyivel csökkenti a várakozási időt. Az adatok azt mutatják, hogy egy kiváló képernyőváz több mint 50%-kal javíthatja a felhasználók toleranciáját a "lassú hálózattal" szemben.
