## Cerințe preliminare

Înainte de a începe, asigurați-vă că sistemul dumneavoastră are instalate Docker și Docker Compose.

### Instalați Docker

#### Windows

Descărcați și instalați [Docker Desktop pentru Windows](https://www.docker.com/products/docker-desktop/)

După instalare, asigurați-vă că Docker Desktop rulează.

#### macOS

Descărcați și instalați [Docker Desktop pentru Mac](https://www.docker.com/products/docker-desktop/)

După instalare, asigurați-vă că Docker Desktop rulează.

#### Linux

Instalați Docker folosind managerul de pachete:

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

După instalare, porniți serviciul Docker:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Verificați instalarea

După instalare, rulați următoarele comenzi pentru a verifica că Docker și Docker Compose sunt instalate corect:

```bash
docker --version
docker compose version
```

## Pornire rapidă

### 1. Generați fișierul de configurare Docker Compose

:::sfat[Utilizați generatorul de configurare]
**Recomand folosirea [Docker Compose Builder (site autonom)](https://builder.hagicode.com/)** pentru a genera fișierul de configurare.

Generatorul suportă:
- Opțiuni multiple de furnizor de API (Zhipu AI, Anthropic Official, API personalizat)
- Configurarea automată a adresei URL și a simbolului API
- Porturi personalizate, nume de containere și alte configurații de bază
- Configurați datele persistente și căile directorului de lucru
- Configurați automat diferențele de platformă Windows/Linux
- Gestionați în mod inteligent problemele de permisiuni ale fișierelor
- **Selectarea sursei imaginii**: Pe baza mediului de rețea, selectați sursa de imagine adecvată

[🚀 Folosiți imediat generatorul →](https://builder.hagicode.com/)
:::

1. Deschide [Docker Compose Builder (site autonom)](https://builder.hagicode.com/)
2. Completați configurația în funcție de nevoile dvs.:
   - Selectați furnizorul de API (Zhipu AI, Anthropic Official sau Custom API)
   - Configurați porturile, baza de date, directorul de lucru și alte opțiuni
   - **Selectați sursa imaginii**: alegeți sursa de imagine adecvată în funcție de mediul dvs. de rețea
3. Faceți clic pe butonul de generare pentru a obține `docker-compose.yml` configurație
4. Salvați configurația generată ca a `docker-compose.yml` fişier
5. Dacă este necesar, creați un `.env` fișier pentru a configura informațiile sensibile

#### Selectarea sursei imaginii

Generatorul acceptă trei opțiuni de sursă de imagine:

**Docker Hub**
- **Adresa imaginii**: `newbe36524/hagicode:{tag}`
- **Scenarii aplicabile**: potrivit pentru utilizatorii cu accelerare în oglindă Docker Hub
- **Avantaje**: sursa oficială a imaginii, actualizări în timp util, acces stabil
- **Note**: Accelerarea în oglindă poate fi necesar să fie configurată în unele regiuni

**Azure Container Registry (ACR)**
- **Adresa imaginii**: `hagicode.azurecr.io/hagicode:{tag}`
- **Scenarii aplicabile**: potrivit pentru utilizatorii care nu pot accesa Docker Hub local
- **Avantaje**: Oferă sursă alternativă de imagine, rezolvă problemele de acces la rețea
- **Note**: Imaginile rămân sincronizate cu Docker Hub, dar pot avea scurte întârzieri

**Alibaba Cloud Container Registry (ACR)**
- **Adresa imaginii**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **Scenarii aplicabile**: potrivit pentru utilizatorii casnici, acces stabil și rapid la rețea
- **Avantaje**: Noduri interne Alibaba Cloud, viteză de acces rapidă, stabilitate ridicată
- **Note**: imaginile Alibaba ACR rămân sincronizate cu Docker Hub, de obicei în 30 de minute

:::sfat[Recomandări pentru selectarea sursei imaginii]
- **Utilizatorii domestici recomandă Alibaba Cloud ACR**: dacă sunteți un utilizator casnic, se recomandă să acordați prioritate utilizării sursei de imagine Alibaba Cloud ACR pentru un acces rapid și stabil
- **Docker Hub alternativ**: dacă mediul dvs. de rețea poate accesa Docker Hub, puteți alege și Docker Hub ca sursă de imagine
- **Azure ACR alternativă**: alegeți sursa de imagine Azure ACR numai dacă alte surse de imagine au dificultăți de acces
- **Sincronizare imagini**: imaginile Alibaba Cloud ACR rămân sincronizate cu Docker Hub, de obicei în 30 de minute
- **Consecvența versiunii**: etichetele de imagine sunt consecvente în toate cele trei surse de imagine, nu trebuie să vă faceți griji cu privire la problemele cu versiunea
:::

### 2. Porniți Servicii

Executați următoarea comandă în directorul în care dvs `docker-compose.yml` se afla:

```bash
docker compose up -d
```

Această comandă va:
- Descărcați și porniți containerele Hagicode definite de șablonul generat
- Creați și configurați rețeaua
- Inițializați directorul de date implicit și dependențele de rulare

### 3. Verificați starea serviciului

Verificați dacă containerele funcționează:

```bash
docker compose ps
```

Ar trebui să vedeți serviciile definite în șablonul dvs. generat în starea „în rulare”.

### Vedeți jurnalele de service

Pentru a vizualiza toate jurnalele de service:

```bash
# View all service logs
docker compose logs

# View Hagicode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## Instrucțiuni de configurare

Dacă trebuie să modificați configurația (cum ar fi schimbarea porturilor, schimbarea furnizorului API etc.), puteți:

1. Regenerați o nouă configurație folosind [Docker Compose Builder (site autonom)](https://builder.hagicode.com/)
2. Editați manual `docker-compose.yml` şi `.env` fișiere
3. Reporniți serviciile pentru ca configurația să aibă efect:

```bash
docker compose restart
```
