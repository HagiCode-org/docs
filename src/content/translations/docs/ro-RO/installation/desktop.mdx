Acest ghid acoperă instalarea și utilizarea Hagicode Desktop. Este clientul desktop oficial și păstrează calea implicită de configurare locală plus controalele zilnice într-un singur loc.

:::sfat[Metoda recomandată]
Desktop este calea locală implicită pentru utilizatorii și dezvoltatorii individuali, deoarece:
- instalatorul păstrează fluxul implicit într-o singură cale ghidată
- dependențele și serviciile sunt gestionate în aplicație
- UI face starea și operațiunile mai ușor de inspectat
- gestionarea încorporată a versiunilor facilitează schimbarea
- datele rămân locale în mod implicit
:::

## Ce este Hagicode Desktop?

Hagicode Desktop este aplicația desktop oficială a Hagicode, concepută pentru sistemele Windows, macOS și Linux. Acesta reunește caracteristicile de bază ale Hagicode, iar programul de instalare păstrează calea de configurare implicită într-un singur flux ghidat.

### Principalele avantaje

- **Instalare cu un singur clic**: finalizați automat toți pașii de configurare prin asistentul de instalare
- **Pregătirea automată a timpului de rulare**: programul de instalare implicit include deja timpul de rulare de care are nevoie, așa că nu trebuie să instalați un alt `.NET` mai întâi timpul de rulare
- **Service Management**: Management Hagicode Server încorporat, pornirea și oprirea cu ușurință a serviciilor
- **Schimbarea versiunii**: Suport pentru gestionarea mai multor versiuni, comutați liber între diferite versiuni
- **Funcționare locală**: Toate datele sunt stocate local, protejându-vă confidențialitatea
- **Actualizări automate**: detectează și descarcă automat cea mai recentă versiune

### Scenarii aplicabile

Desktop-ul se potrivește bine atunci când:
- Dezvoltatori individuali care doresc să folosească Hagicode la nivel local
- Utilizatori care trebuie să folosească Hagicode offline
- Utilizatori non-tehnici care doresc să simplifice procesul de instalare
- Dezvoltatorii care trebuie să comute frecvent între versiuni diferite

### Alte metode de descărcare

De asemenea, puteți obține programul de instalare Hagicode Desktop de pe următoarele canale:

1. **Pagina de lansare oficială**: Vizitați [Lansări Hagicode GitHub](https://github.com/Hagicode-org/releases) pentru a descărca cea mai recentă versiune
2. **Site-ul oficial**: Vizitați [Site-ul Hagicode](https://hagicode.com/) pentru a obține link-uri de descărcare

### Alegeți versiunea corectă

Selectați programul de instalare corespunzător pentru sistemul dvs. de operare:
- **Windows**: Descărcați `.exe` instalator
- **macOS**: descărcați `.dmg` instalator
- **Linux**: Descărcați `.AppImage` sau `.tar.gz` pachet de instalare

:::notă[Cerințe de sistem]
- **Windows**: Windows 10 sau versiunea superioară (64 de biți)
- **macOS**: macOS 10.15 sau o versiune ulterioară
- **Linux**: distribuții Linux principale (Ubuntu 20.04+, Fedora 33+ etc.)
:::

:::sfat[Precondiții implicite]
Pentru cel mai recent flux de instalare locală HagiCode, de obicei nu trebuie să pregătiți un separat `.NET` timpul de rulare. Desktop deja grupează timpul de execuție necesar pentru fluxul implicit și inițializează calea SQLite locală acceptată în mod implicit.
:::

## Note de instalare și lansare

:::atenție[MacOS Gatekeeper Notice]
Dacă ați instalat deja aplicația pe calea implicită `/Applications/Hagicode.app` dar macOS arată „aplicația este deteriorată”, „nu poate fi deschisă” sau un avertisment Gatekeeper similar la prima lansare, cauza obișnuită este `com.apple.quarantine` atribut pe care macOS îl adaugă la aplicațiile descărcate de pe internet, nu un eșec în programul de instalare Hagicode Desktop în sine.

Rulați următoarea comandă în Terminal. Necesită privilegii de administrator și elimină recursiv atributul de carantină din aplicație în calea implicită de instalare:

```bash
sudo xattr -dr com.apple.quarantine /Applications/Hagicode.app
```

După ce comanda se termină, relansați Hagicode Desktop.
:::

## Rulați Desktop

După descărcare, deschideți programul de instalare sau pachetul de aplicație și urmați instrucțiunile de pe ecran. Desktop pregătește timpul de execuție în calea locală acceptată în prezent, gestionează Hagicode Server și arată progresul și starea atunci când sunt necesare descărcări de rețea.

:::notă[Cerințe de rețea]
Asigurați-vă că este disponibilă conexiunea la rețea. Este posibil ca desktopul să fie nevoie să preia informații despre versiune sau pachete de componente de pe internet; dacă o descărcare eșuează, verificați setările de rețea, proxy sau firewall și încercați din nou.
:::

## Managementul versiunilor

Hagicode Desktop acceptă gestionarea mai multor versiuni, permițându-vă să comutați liber între versiuni diferite.

![Managementul versiunilor](../../img/screenshots/installation/install-desktop/12-version-management/original.png)

### Schimbați versiunea

1. Deschide Hagicode Desktop
2. Accesați pagina „Gestionare versiuni”.
3. Selectați versiunea pe care doriți să o schimbați
4. Faceți clic pe butonul „Schimbați versiunea”.
5. Așteptați ca comutarea să se finalizeze

:::notă[Note privind schimbarea versiunii]
- Schimbarea versiunilor va reporni automat Hagicode Server
- Asigurați-vă că salvați toată munca înainte de a comuta
- Configurațiile proiectului pot diferi între versiuni
:::

## Porniți și opriți serviciul

Hagicode Desktop oferă o modalitate simplă de a gestiona serviciul Hagicode Server.

### Porniți serviciul

Pe pagina de start a aplicației Desktop, faceți clic pe butonul „Start Service” pentru a porni Hagicode Server.

![Porniți serviciul](../../img/screenshots/installation/install-desktop/13-start-service-homepage/original.png)

### Opriți serviciul

Pentru a opri Hagicode Server, faceți clic pe butonul „Opriți serviciul” de pe pagina principală.

:::sfat[Starea serviciului]
Aplicația Desktop afișează starea de rulare a serverului Hagicode în timp real, inclusiv:
- Starea serviciului (în funcționare/oprit)
- Numărul portului de serviciu
- Informații despre versiunea curentă
:::

## Depanare

### Eșec de instalare

Dacă întâmpinați probleme în timpul instalării:

1. **Verificați conexiunea la rețea**: asigurați-vă că puteți accesa internetul
2. **Verificați spațiu pe disc**: asigurați-vă că aveți suficient spațiu pe disc
3. **Run ca administrator**: pe Windows, faceți clic dreapta pe programul de instalare și selectați „Run as administrator”
4. **Vizualizare fișiere jurnal**: programul de instalare generează fișiere jurnal detaliate care pot ajuta la diagnosticarea problemelor

### Eșec de instalare a dependenței

Dacă instalarea dependenței eșuează:

1. **Verificați compatibilitatea sistemului**: asigurați-vă că sistemul dvs. de operare îndeplinește cerințele minime
2. **Verificați setările de rețea și firewall**: asigurați-vă că instalatorul poate ajunge la descărcări necesare și că nu este blocat de firewall
3. **Confirmați permisiunile de instalare**: pe Windows, încercați să rulați programul de instalare ca administrator, astfel încât să aibă suficiente permisiuni
4. **Reporniți aplicația și reîncercați**: închideți și redeschideți Hagicode Desktop, apoi reîncercați pasul de instalare
5. **Vizualizați jurnalele**: examinați jurnalele de erori detaliate în aplicația Desktop

### Eșec de pornire a serviciului

Dacă Hagicode Server nu reușește să pornească:

1. **Verificați ocuparea portului**: asigurați-vă că portul 45000 nu este ocupat de alte programe
2. **Verificați setările paravanului de protecție**: asigurați-vă că paravanul de protecție permite accesul la rețea Hagicode Server
3. **Reporniți aplicația**: Încercați să reporniți Hagicode Desktop
4. **Vizualizați jurnalele**: vizualizați jurnalele de erori detaliate în aplicația Desktop
