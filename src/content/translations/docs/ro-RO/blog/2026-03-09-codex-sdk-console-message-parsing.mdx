## Ghid Complet pentru Analizarea Mesajelor de Consolă din SDK-ul Codex

> Acest articol explică mecanismul fluxului de evenimente din SDK-ul Codex, analizarea tipurilor de mesaje și bunele practici în proiecte reale, ajutând dezvoltatorii să stăpânească rapid abilitățile de bază din spatele serviciilor de execuție AI.

## Context

La construirea unui serviciu de execuție AI bazat pe SDK-ul Codex, ne confruntăm inevitabil cu o întrebare practică: cum ar trebui să gestionăm mesajele de eveniment transmise în flux returnate de Codex? Aceste mesaje conțin informații importante precum starea execuției, conținutul ieșirii și detaliile erorilor, deci merită o gestionare atentă.

Ca parte a proiectului HagiCode, aveam nevoie de un executor fiabil pentru scenariile de asistent de codare AI. Acesta este exact motivul pentru care am decis să studiem în profunzime mecanismul fluxului de evenimente al SDK-ului Codex. După toate, doar înțelegând cum funcționează mesajele de la nivelul de bază putem construi o platformă de execuție AI cu adevărat de nivel enterprise.

SDK-ul Codex este un SDK de asistență la programare lansat de OpenAI. Returnează rezultatele execuției printr-un flux de evenimente. Spre deosebire de modelul tradițional cerere-răspuns, Codex folosește evenimente în flux astfel încât să putem:

- Obține progresul execuției în timp real
- Gestionăm erorile prompt
- Obținem statistici detaliate de utilizare a token-urilor
- Suportăm sarcini complexe de lungă durată

Înțelegerea acestor tipuri de evenimente și analizarea lor corectă este esențială pentru implementarea unui executor AI complet capabil. În cele din urmă, nimeni nu vrea să lucreze cu o cutie neagră.

## Mecanismul Fluxului de Evenimente

### Concepte de Bază

SDK-ul Codex folosește metoda `thread.runStreamed()` pentru a returna un iterator de evenimente asincron:

```typescript
import { Codex } from '@openai/codex-sdk';

const client = new Codex({
  apiKey: process.env.CODEX_API_KEY,
  baseUrl: process.env.CODEX_BASE_URL,
});

const thread = client.startThread({
  workingDirectory: '/path/to/project',
  skipGitRepoCheck: false,
});

const { events } = await thread.runStreamed('your prompt here', {
  outputSchema: {
    type: 'object',
    properties: {
      output: { type: 'string' },
      status: { type: 'string', enum: ['ok', 'action_required'] },
    },
    required: ['output', 'status'],
  },
});

for await (const event of events) {
  // Handle each event
}
```

### Explicație Detaliată a Tipurilor de Evenimente

| Tip de Eveniment | Descriere | Date Cheie |
|---------|------|----------|
| `thread.started` | Firul de execuție a pornit cu succes | `thread_id` |
| `item.updated` | Conținutul mesajului a fost actualizat | `item.text` |
| `item.completed` | Mesajul a fost finalizat | `item.text` |
| `turn.completed` | Execuția s-a finalizat | `usage` (utilizare token-uri) |
| `turn.failed` | Execuția a eșuat | `error.message` |
| `error` | Eveniment de eroare | `message` |

În proiectele reale, componenta executor a HagiCode este construită pe aceste tipuri de evenimente. Trebuie să gestionăm cu atenție fiecare tip de eveniment pentru a asigura o experiență de utilizare fluidă. Sistemele bune sunt construite luând detaliile în serios.

## Implementarea Analizării Mesajelor

### Extragerea Conținutului Mesajelor

Conținutul mesajelor este extras printr-un handler de evenimente:

```typescript
private handleThreadEvent(event: ThreadEvent, onMessage: (content: string) => void): void {
  // Only handle message update and completion events
  if (event.type !== 'item.updated' && event.type !== 'item.completed') {
    return;
  }

  // Only handle agent message content
  if (event.item.type !== 'agent_message') {
    return;
  }

  // Extract text content
  onMessage(event.item.text);
}
```

Puncte cheie:
- Gestionează doar evenimentele `item.updated` și `item.completed`
- Gestionează doar conținutul de tip `agent_message`
- Conținutul mesajului se află în câmpul `event.item.text`

### Analizarea Ieșirii Structurate

Codex suportă ieșire structurată JSON. Poți specifica formatul de returnare prin parametrul `outputSchema`:

```typescript
const DEFAULT_OUTPUT_SCHEMA = {
  type: 'object',
  properties: {
    output: { type: 'string' },
    status: { type: 'string', enum: ['ok', 'action_required'] },
  },
  required: ['output', 'status'],
  additionalProperties: false,
} as const;
```

Funcția de analizare încearcă să analizeze JSON, iar dacă aceasta eșuează, revine la textul brut.

```typescript
function toStructuredOutput(raw: string): StructuredOutput {
  try {
    const parsed = JSON.parse(raw) as Partial<StructuredOutput>;
    if (typeof parsed.output === 'string') {
      return {
        output: parsed.output,
        status: parsed.status === 'action_required' ? 'action_required' : 'ok',
      };
    }
  } catch {
    // JSON parsing failed, fall back to the raw text
  }

  return {
    output: raw,
    status: 'ok',
  };
}
```

### Flux Complet de Gestionare a Evenimentelor

```typescript
private async runWithStreaming(
  thread: Thread,
  input: CodexStageExecutionInput
): Promise<{ output: string; usage: Usage | null }> {
  const abortController = new AbortController();
  const timeoutHandle = setTimeout(() => {
    abortController.abort();
  }, Math.max(1000, input.timeoutMs));

  let latestMessage = '';
  let usage: Usage | null = null;
  let emittedLength = 0;

  try {
    const { events } = await thread.runStreamed(input.prompt, {
      outputSchema: DEFAULT_OUTPUT_SCHEMA,
      signal: abortController.signal,
    });

    for await (const event of events) {
      // Handle message content
      this.handleThreadEvent(event, (nextContent) => {
        const delta = nextContent.slice(emittedLength);
        if (delta.length > 0) {
          emittedLength = nextContent.length;
          input.callbacks?.onChunk?.(delta);  // Streaming callback
        }
        latestMessage = nextContent;
      });

      // Process different data based on the event type
      if (event.type === 'thread.started') {
        this.threadId = event.thread_id;
      } else if (event.type === 'turn.completed') {
        usage = event.usage;
      } else if (event.type === 'turn.failed') {
        throw new CodexExecutorError('gateway_unavailable', event.error.message, true);
      } else if (event.type === 'error') {
        throw new CodexExecutorError('gateway_unavailable', event.message, true);
      }
    }
  } catch (error) {
    if (abortController.signal.aborted) {
      throw new CodexExecutorError(
        'upstream_timeout',
        `Codex stage timed out after ${input.timeoutMs}ms`,
        true
      );
    }
    throw error;
  } finally {
    clearTimeout(timeoutHandle);
  }

  const structured = toStructuredOutput(latestMessage);
  return { output: structured.output, usage };
}
```

## Strategia de Gestionare a Erorilor

### Maparea Codurilor de Eroare

Mapăm pattern-urile specifice de eroare la coduri de eroare concrete pentru ca straturile superioare să le poată gestiona mai ușor:

```typescript
function mapError(error: unknown): CodexExecutorError {
  if (error instanceof CodexExecutorError) {
    return error;
  }

  const message = error instanceof Error ? error.message : String(error);
  const normalized = message.toLowerCase();

  // Authentication errors - not retryable
  if (normalized.includes('401') ||
      normalized.includes('403') ||
      normalized.includes('api key') ||
      normalized.includes('auth')) {
    return new CodexExecutorError('auth_invalid', message, false);
  }

  // Rate limit errors - retryable
  if (normalized.includes('429') || normalized.includes('rate limit')) {
    return new CodexExecutorError('rate_limited', message, true);
  }

  // Timeout errors - retryable
  if (normalized.includes('timeout') || normalized.includes('aborted')) {
    return new CodexExecutorError('upstream_timeout', message, true);
  }

  // Default error
  return new CodexExecutorError('gateway_unavailable', message, true);
}
```

### Definiții ale Tipurilor de Eroare

```typescript
export type CodexErrorCode =
  | 'auth_invalid'      // Authentication failure
  | 'upstream_timeout'  // Upstream timeout
  | 'rate_limited'      // Rate limited
  | 'gateway_unavailable'; // Gateway unavailable

export class CodexExecutorError extends Error {
  readonly code: CodexErrorCode;
  readonly retryable: boolean;

  constructor(code: CodexErrorCode, message: string, retryable: boolean) {
    super(message);
    this.name = 'CodexExecutorError';
    this.code = code;
    this.retryable = retryable;
  }
}
```

## Directorul de Lucru și Configurația Mediului

### Validarea Directorului de Lucru

SDK-ul Codex necesită ca directorul de lucru să fie un depozit Git valid.

```typescript
export function validateWorkingDirectory(
  workingDirectory: string,
  skipGitRepoCheck: boolean
): void {
  const resolvedWorkingDirectory = path.resolve(workingDirectory);

  if (!existsSync(resolvedWorkingDirectory)) {
    throw new CodexExecutorError(
      'gateway_unavailable',
      'Working directory does not exist.',
      false
    );
  }

  if (!statSync(resolvedWorkingDirectory).isDirectory()) {
    throw new CodexExecutorError(
      'gateway_unavailable',
      'Working directory is not a directory.',
      false
    );
  }

  if (skipGitRepoCheck) {
    return;
  }

  const gitDir = path.join(resolvedWorkingDirectory, '.git');
  if (!existsSync(gitDir)) {
    throw new CodexExecutorError(
      'gateway_unavailable',
      'Working directory is not a git repository.',
      false
    );
  }
}
```

### Încărcarea Variabilelor de Mediu

SDK-ul Codex trebuie să încarce variabile de mediu din shell-ul de login pentru ca AI Agent să poată accesa comenzile sistemului:

```typescript
function parseEnvironmentOutput(output: Buffer): Record<string, string> {
  const parsed: Record<string, string> = {};

  for (const entry of output.toString('utf8').split('\0')) {
    if (!entry) continue;

    const separatorIndex = entry.indexOf('=');
    if (separatorIndex <= 0) continue;

    const key = entry.slice(0, separatorIndex);
    const value = entry.slice(separatorIndex + 1);
    if (key.length > 0) {
      parsed[key] = value;
    }
  }

  return parsed;
}

function tryLoadEnvironmentFromShell(shellPath: string): Record<string, string> | null {
  const result = spawnSync(shellPath, ['-ilc', 'env -0'], {
    env: process.env,
    stdio: ['ignore', 'pipe', 'pipe'],
    timeout: 5000,
  });

  if (result.error || result.status !== 0) {
    return null;
  }

  return parseEnvironmentOutput(result.stdout);
}

export function createExecutorEnvironment(
  envOverrides: Record<string, string> = {}

  // Load environment variables from the login shell
  const consoleEnv = loadConsoleEnvironmentFromShell();

  return {
    ...process.env,
```

## Exemplu Complet de Utilizare

### Utilizare de Bază

În proiectul HagiCode, folosim următoarea abordare pentru a inițializa clientul Codex și a executa sarcini:

```typescript
import { Codex } from '@openai/codex-sdk';

async function executeWithCodex(prompt: string, workingDir: string) {
  const client = new Codex({
    apiKey: process.env.CODEX_API_KEY,
    env: { PATH: process.env.PATH },
  });

  const thread = client.startThread({
    workingDirectory: workingDir,
  });

  const { events } = await thread.runStreamed(prompt);

  let result = '';
  for await (const event of events) {
    if (event.type === 'item.updated' && event.item.type === 'agent_message') {
      result = event.item.text;
    }
    if (event.type === 'turn.completed') {
      console.log('Token usage:', event.usage);
    }
  }

  // Try to parse JSON output
  try {
    const parsed = JSON.parse(result);
    return parsed.output;
  } catch {
    return result;
  }
}
```

### Implementare Completă cu Reîncercări

```typescript
export class CodexSdkExecutor {
  private readonly config: CodexRuntimeConfig;
  private readonly client: Codex;
  private threadId: string | null = null;

  async executeStage(input: CodexStageExecutionInput): Promise<CodexStageExecutionResult> {
    const maxAttempts = Math.max(1, this.config.retryCount + 1);
    let attempt = 0;
    let lastError: CodexExecutorError | null = null;

    while (attempt < maxAttempts) {
      attempt += 1;

      try {
        const thread = this.getThread(input.workingDirectory);
        const { output, usage } = await this.runWithStreaming(thread, input);

        return {
          output,
          usage,
          threadId: this.threadId!,
          attempts: attempt,
          latencyMs: Date.now() - startedAt,
        };
      } catch (error) {
        const mappedError = mapError(error);
        lastError = mappedError;

        // Non-retryable error or max retry attempts reached
        if (!mappedError.retryable || attempt >= maxAttempts) {
          throw mappedError;
        }

        // Wait before retrying
        await new Promise(resolve => setTimeout(resolve, 1000 * attempt));
      }
    }

    throw lastError!;
  }
}
```

## Bune Practici

### 1. Cerințe pentru Directorul de Lucru

- Asigură-te că directorul de lucru este un depozit Git valid
- Folosește variabila de mediu `PROJECT_ROOT` pentru a-l specifica explicit
- În timpul depanării, poți seta `CODEX_SKIP_GIT_REPO_CHECK=true` pentru a sări verificarea

### 2. Configurația Variabilelor de Mediu

- Transmite doar variabilele de mediu necesare printr-un mecanism de listă de permisiuni
- Folosește shell-ul de login pentru a încărca mediul complet
- Evită transmiterea informațiilor sensibile

### 3. Strategia de Timeout și Reîncercare

- Setează timeout-uri rezonabile bazate pe complexitatea sarcinii
- Implementează backoff exponențial pentru erorile reîncercabile
- Înregistrează numărul de reîncercări și motivele

### 4. Gestionarea Erorilor

- Distinge între erorile reîncercabile și cele nereîncercabile
- Oferă mesaje de eroare clare și sugestii
- Folosește coduri de eroare unificate pentru ca straturile superioare să le poată gestiona consecvent

### 5. Ieșirea în Flux

- Implementează callback-uri de ieșire incrementală pentru a îmbunătăți experiența utilizatorului
- Gestionează corect actualizările incrementale ale mesajelor
- Înregistrează utilizarea token-urilor pentru analiza costurilor
