## Suport complet pentru GLM-5.1 și integrarea Gemini CLI: Drumul HagiCode către evoluția multi-model

> Acest articol prezintă două actualizări majore recente ale platformei HagiCode: suportul complet pentru modelul GLM-5.1 de la Zhipu AI și integrarea cu succes a Gemini CLI ca al zecelea Agent CLI. Împreună, aceste actualizări consolida și mai mult capabilitățile multi-model ale platformei și ecosistemul multi-CLI.

## Context

Timpul chiar trece repede. Dezvoltarea modelelor de limbaj de mari dimensiuni a crescut ca bambusul în primăvară. Nu cu mult timp în urmă, ne bucuram încă de „un AI care poate scrie cod". Acum suntem deja într-o eră a colaborării multi-model și a integrării multi-instrument. Este oare interesant? Poate. La urma urmei, ceea ce au nevoie dezvoltatorii nu a fost niciodată doar instrumentul în sine, ci ușurința de a se adapta la scenarii diferite și de a comuta flexibil atunci când este nevoie.

Ca platformă de codare asistată de AI, HagiCode a primit recent două dezvoltări importante: în primul rând, integrarea completă a modelului GLM-5.1 de la Zhipu AI; în al doilea rând, adăugarea oficială a Gemini CLI ca al zecelea Agent CLI susținut. Aceste două actualizări pot nu părea revoluționare, dar sunt fără îndoială vești bune pentru maturizarea continuă a platformei.

GLM-5.1 este cel mai recent model flagship de la Zhipu AI. Comparativ cu GLM-5.0, oferă raționament mai puternic, înțelegere mai profundă a codului și apeluri de instrumente mai fluide. Mai important, este primul model GLM care suportă intrarea de imagine. Ce înseamnă asta? Înseamnă că utilizatorii pot lăsa AI-ul să se uite direct la o captură de ecran în loc să se străduiască să descrie problema în cuvinte. Odată ce ai folosit această comoditate, înțelegi imediat valoarea ei.

În același timp, prin arhitectura `HagiCode.Libs.Providers`, HagiCode a integrat cu succes Gemini CLI în platformă. Acesta este acum al zecelea Agent CLI. Sincer, ajungerea în acest punct aduce un modest sentiment de realizare.

De asemenea, merită menționat că funcția de încărcare a imaginilor din HagiCode permite utilizatorilor să comunice cu AI-ul direct prin capturi de ecran. Chiar și rulând GLM 4.7, platforma funcționează bine și a ajutat deja la finalizarea multor sarcini importante de construcție. Iar pentru GLM-5.1, în mod natural, merge cu un pas mai departe.

## Despre HagiCode

Abordarea împărtășită în acest articol provine din experiența noastră practică în proiectul [HagiCode](https://hagicode.com). HagiCode este o platformă de codare asistată de AI cu sursă deschisă, concepută pentru a oferi dezvoltatorilor un asistent de programare AI flexibil și puternic printr-o arhitectură multi-model, multi-CLI. Depozitul proiectului: [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site)

## Designul arhitecturii multi-CLI

Unul dintre punctele forte de bază ale HagiCode este suportul pentru mai multe instrumente CLI de programare AI printr-un strat de abstractizare unificat. Avantajul acestui design este de fapt destul de simplu: instrumente noi pot fi adăugate, instrumentele vechi pot rămâne, iar baza de cod nu se transformă în haos. Sincer, așa și-ar dori toți să funcționeze lucrurile.

### Enumerația AIProviderType

Platforma definește tipurile de furnizori CLI susținuți prin enumerația `AIProviderType`:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI (nou)
}
```

După cum puteți vedea, Gemini CLI se alătură acestei familii ca al zecelea membru. Fiecare CLI are propriile caracteristici distincte și scenarii de utilizare, astfel încât utilizatorii pot alege flexibil în funcție de nevoile lor. La urma urmei, multe drumuri duc la Roma; unele sunt pur și simplu mai ușoare decât altele.

### Arhitectura Provider

`HagiCode.Libs.Providers` oferă o interfață Provider unificată care face fiecare integrare CLI standardizată și concisă. Luând Gemini CLI ca exemplu:

```csharp
public class GeminiProvider : ICliProvider<GeminiOptions>
{
    private static readonly string[] DefaultExecutableCandidates = ["gemini", "gemini-cli"];
    private const string ManagedBootstrapArgument = "--acp";

    public string Name => "gemini";
    public bool IsAvailable => _executableResolver.ResolveFirstAvailablePath(DefaultExecutableCandidates) is not null;
}
```

Beneficiile acestui design sunt:
- Integrarea unui CLI nou necesită doar implementarea unei singure clase Provider
- Management unificat al ciclului de viață și pool de sesiuni
- Rezolvare automată a aliasurilor și descoperire a executabilelor

Pe scurt, acest design transformă lucrurile complicate în altele mai simple și face viața un pic mai ușoară.

### Registrul Provider

Registrul Provider gestionează automat maparea aliasurilor și înregistrarea:

```csharp
if (provider is GeminiProvider)
{
    registry.Register(provider.Name, provider, ["gemini-cli"]);
    continue;
}
```

Aceasta înseamnă că utilizatorii pot invoca Gemini CLI fie cu `gemini`, fie cu `gemini-cli`, iar sistemul îl va recunoaște automat. Este ca un prieten care are atât un nume formal, cât și un poreclă — în orice caz, oamenii știu despre cine este vorba.

## Suportul pentru modelul GLM-5.1

GLM-5.1 este cel mai recent model flagship de la Zhipu AI, iar HagiCode a finalizat suportul complet pentru acesta.

### Catalogul de Profesii Secundare

HagiCode gestionează toate modelele susținute prin Catalogul de Profesii Secundare. Iată configurația pentru seria GLM:

| ID Model | Nume | SuportsImage | Familii CLI Compatibile |
|----------|------|----------------|-------------------------|
| `glm-4.7` | GLM 4.7 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM 5 Turbo | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (Legacy) | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | **true** | claude, codebuddy, hermes, qoder, kiro |

Caracteristicile cheie ale GLM-5.1 pot fi rezumate după cum urmează:
- Un identificator de versiune independent, fără bagaj istoric
- Primul model GLM care suportă intrarea de imagine
- Raționament și înțelegere a codului mai puternice
- Compatibilitate multi-CLI extinsă

### GLM-5.1 vs GLM-5.0

La nivel de cod, diferența cheie dintre GLM-5.1 și GLM-5.0 este prezentată aici:

```csharp
// GLM-5.0 (Legacy) - conține logică specială de retenție
private const string Glm50CodebuddySecondaryProfessionId = "secondary-glm-5-codebuddy";
private const string Glm50CodebuddyModelValue = "glm-5.0";

// GLM-5.1 - identificator de model nou, independent
private const string Glm51SecondaryProfessionId = "secondary-glm-5-1";
private const string Glm51ModelValue = "glm-5.1";
```

GLM-5.0 poartă eticheta „Legacy" deoarece este un identificator de versiune veche păstrat pentru compatibilitate inversă. GLM-5.1, în contrast, este o versiune independentă complet nouă, fără povară istorică. Unele lucruri rămân în trecut; altele călătoresc mai ușor și se mișcă mai repede.

### Configurarea GLM-5.1

Iată un exemplu de configurare pentru utilizarea GLM-5.1 în HagiCode:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

## Funcția de încărcare a imaginilor

Suportul pentru imagini din HagiCode este implementat prin proprietatea `SupportsImage` pe `SecondaryProfession`:

```csharp
public class HeroSecondaryProfessionSettingDto
{
    public bool SupportsImage { get; set; }
}
```

În Catalogul de Profesii Secundare, configurația GLM-5.1 arată astfel:

```json
{
  "id": "secondary-glm-5-1",
  "supportsImage": true
}
```

Aceasta înseamnă că utilizatorii pot încărca capturi de ecran direct pentru analiza AI, cum ar fi:
- Capturi de ecran ale mesajelor de eroare
- Probleme într-un ecran UI
- Grafice de vizualizare a datelor
- Rezultate ale execuției codului

Nu mai este nevoie să descrieți totul manual — doar încărcați captura de ecran. Comoditatea acestei funcții este evidentă odată ce ați folosit-o. Uneori, o privire spune mai mult decât o lungă explicație.

## Integrarea Gemini CLI

Ca al zecelea Agent CLI, Gemini CLI este integrat în HagiCode prin arhitectura Provider standard.

### Opțiuni de configurare

Gemini CLI suportă un set bogat de opțiuni de configurare:

```csharp
public class GeminiOptions
{
    public string? ExecutablePath { get; set; }
    public string? WorkingDirectory { get; set; }
    public string? SessionId { get; set; }
    public string? Model { get; set; }
    public string? AuthenticationMethod { get; set; }
    public string? AuthenticationToken { get; set; }
    public Dictionary<string, string?> AuthenticationInfo { get; set; }
    public Dictionary<string, string?> EnvironmentVariables { get; set; }
    public string[] ExtraArguments { get; set; }
    public TimeSpan? StartupTimeout { get; set; }
    public CliPoolSettings? PoolSettings { get; set; }
}
```

Aceste opțiuni acoperă totul, de la configurarea de bază până la funcții avansate, oferind utilizatorilor flexibilitatea de a configura CLI-ul în funcție de propriile nevoi. Fluxul de lucru al fiecăruia este diferit, așa că puțină flexibilitate este întotdeauna binevenită.

### Protocolul de comunicare ACP

Gemini CLI suportă ACP (Agent Communication Protocol), care este standardul de comunicare CLI unificat al HagiCode. Prin ACP, diferite CLI-uri pot interacționa cu platforma într-un mod consistent, simplificând semnificativ munca de integrare. Pe scurt, standardizează părțile complicate pentru ca toți să poată lucra mai ușor.

## Cele mai bune practici pentru integrarea multi-CLI

`HagiCode.Libs.Providers` oferă un mecanism unificat pentru înregistrare și utilizare:

```csharp
services.AddHagiCodeLibs();

var gemini = serviceProvider.GetRequiredService<ICliProvider<GeminiOptions>>();
var codebuddy = serviceProvider.GetRequiredService<ICliProvider<CodebuddyOptions>>();
var hermes = serviceProvider.GetRequiredService<ICliProvider<HermesOptions>>();
```

Acest design prin injecție de dependențe menține utilizarea diferitelor CLI-uri foarte concisă și face testarea unitară și mock-ul mai convenabile. Codul curat este o modalitate de a fi responsabil față de tine însuți.

### Note

Există câteva lucruri de reținut în utilizarea reală:

1. **Configurarea cheii API**: Asigurați-vă că `ANTHROPIC_AUTH_TOKEN` este setat corect, altfel modelul nu poate fi apelat
2. **Disponibilitatea modelului**: GLM-5.1 trebuie să fie activat de furnizorul de model corespunzător
3. **Funcția de imagine**: Doar modelele cu `supportsImage: true` pot folosi încărcarea imaginilor
4. **Instalarea CLI**: Înainte de a utiliza Gemini CLI, asigurați-vă că `gemini` sau `gemini-cli` este în PATH-ul sistemului

Acestea pot fi detalii mici, dar detaliile mici gestionate prost se pot transforma în probleme mari, deci merită atenție.

Cu suportul complet pentru GLM-5.1 și integrarea cu succes a Gemini CLI, HagiCode își consolidează și mai mult capabilitățile ca platformă de programare AI multi-model, multi-CLI.
