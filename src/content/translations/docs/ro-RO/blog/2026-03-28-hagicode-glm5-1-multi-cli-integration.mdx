## Ghid de integrare multi-CLI Hagicode și GLM-5.1

:::note[Notă de context temporal]
Acest articol documentează designul de integrare multi-CLI și maparea modelelor așa cum existau la momentul publicării. Pentru domeniul curent de suport activ, consultați [Instalare AI Agent CLI](/en-US/related-software-installation/ai-agent-cli/) și [Ghidul de instalare](/en-US/installation); `IFlowCli` menționat aici reprezintă doar o configurație istorică de la acel moment și nu face parte din domeniul de suport activ curent.

## Context

În proiectul Hagicode, utilizatorii pot alege din multiple instrumente CLI pentru a conduce asistenți de programare AI, inclusiv Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI și altele. Aceste instrumente CLI sunt instrumente de programare AI cu scop general pe cont propriu, dar prin stratul de abstracție Hagicode, se pot conecta flexibil la diferiți provideri de modele AI.

Zhipu AI (ZAI) oferă o interfață compatibilă cu API-ul Anthropic Claude, permițând acestor instrumente CLI să folosească direct modelele din seria GLM autohtonă. Printre acestea, GLM-5.1 este cea mai recentă lansare de model de limbaj mare de la Zhipu, cu îmbunătățiri semnificative față de GLM-5.0.

## Arhitectura de abstracție CLI Hagicode

Hagicode definește 11 tipuri de provider CLI prin enum-ul `AIProviderType`, acoperind instrumentele CLI de programare AI mainstream:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Fiecare CLI are configurația corespunzătoare de parametri de model și suportă parametrii `model` și `reasoning`:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],
        [AIProviderType.CodexCli] = ["model", "reasoning"],
        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],

        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini does not support the reasoning parameter
        // ...
    };
```

## Sistemul de suport pentru modelele GLM

Catalogul de Profesii Secundare Hagicode definește suport complet pentru seria de modele GLM:

| ID Model | Nume | Reasoning implicit | Familii CLI compatibile |
|----------|------|-------------------|-------------------------|
| `glm-4.7` | GLM 4.7 | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM 5 Turbo | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (Legacy) | high | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | high | claude, codebuddy, hermes, qoder, kiro |

## Utilizarea GLM-5.1 în diferite CLI-uri

### 1. Claude Code CLI + GLM-5.1

Claude Code CLI este unul dintre CLI-urile de bază Hagicode și este configurat prin sistemul de configurație Hero:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Configurație `HeroEquipmentCatalogItem` corespunzătoare:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI este cel mai flexibil CLI și suportă specificarea oricărui model în formatul `provider/model`:

**Metoda 1: Folosiți prefixul providerului ZAI**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Metoda 2: Folosiți ID-ul modelului direct**
```json
{
  "model": "glm-5.1"
}
```

**Metoda 3: Interfață UI de configurație frontend**

În `HeroModelEquipmentForm.tsx`, OpenCode CLI are un indiciu placeholder dedicat:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Utilizatorii pot introduce:
```
zai/glm-5.1
glm-5.1
```

**Logica de parsare a modelului OpenCode CLI**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI are logică dedicată de tratare legacy:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Notă**: Codebuddy reține tratarea specială pentru GLM-5.0 și nu folosește normalizarea legacy:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## Avantajele de îmbunătățire ale GLM-5.1

Comparativ cu GLM-5.0, GLM-5.1 aduce următoarele îmbunătățiri semnificative:

### 1. Capacitate de raționament mai bună

Conform informațiilor oficiale de lansare de la Zhipu, îmbunătățirile în GLM-5.1 includ:

- **Înțelegere de cod mai puternică**: Analiză mai precisă a structurilor complexe de cod
- **Înțelegere context mai lungă**: Suportă context de conversație mai lung
- **Apelare de instrumente îmbunătățită**: Rată de succes mai mare pentru apeluri de instrumente MCP
- **Stabilitate a ieșirii**: Reduce aleatorietatea și halucinațiile

### 2. Compatibilitate multi-CLI comprehensivă

GLM-5.1 acoperă toate CLI-urile mainstream suportate de Hagicode:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Note

### 1. Configurarea cheii API

Asigurați-vă că variabila de mediu `ANTHROPIC_AUTH_TOKEN` este setată corect. Este credential-ul necesar pentru fiecare CLI pentru a se conecta la model.

### 2. Disponibilitatea modelului

GLM-5.1 trebuie activat de către providerul de model corespunzător:
- Platforma Zhipu AI ZAI îl suportă implicit
- Alibaba Cloud DashScope poate necesita o cerere separată

### 3. Formatul OpenCode CLI

Când folosiți formatul `provider/model`, asigurați-vă că ID-ul providerului este corect:
- Zhipu AI: `zai` sau `zhipuai`
- Alibaba Cloud: `aliyun` sau `dashscope`

### 4. Parametrul reasoning

- `high` este recomandat pentru cele mai bune rezultate de generare de cod
- Gemini CLI nu suportă parametrul reasoning și va ignora automat această configurație

## Concluzie

Prin intermediul unui strat de abstracție unificat, Hagicode permite integrarea flexibilă între GLM-5.1 și multiple CLI-uri. Dezvoltatorii pot alege instrumentul CLI care li se potrivește cel mai bine preferințelor și scenariilor de utilizare, apoi folosesc cel mai recent model GLM-5.1 prin configurație simplă.

Ca cea mai recentă versiune de model de la Zhipu, GLM-5.1 oferă îmbunătățiri clare față de GLM-5.0:
- Un identificator de versiune independent fără sarcină legacy
- Raționament și înțelegere de cod mai puternice
- Compatibilitate largă multi-CLI
- Configurare flexibilă a nivelului de reasoning

Cu variabilele de mediu corecte și echipamentul Hero configurat, utilizatorii pot debloca complet puterea GLM-5.1 în diferite medii CLI.
