## De ce a ales HagiCode Hermes ca nucleu de Agent integrat

> La construirea unei platforme de codare asistată de AI, alegerea nucleului de Agent potrivit determină direct limita superioară a capabilităților sistemului. Unele lucruri pur și simplu nu pot fi forțate; alegeți framework-ul greșit, și niciun efort nu îl va face să se simtă potrivit. Acest articol prezintă raționamentul din spatele selecției tehnice HagiCode și experiența noastră practică de integrare a Hermes Agent.

## Context

La construirea unui produs de codare asistată de AI, una dintre cele mai dificile părți este alegerea framework-ului Agent de bază. Există destul de multe opțiuni pe piață, dar unele sunt prea limitate funcțional, altele sunt excesiv de complexe de implementat, iar altele pur și simplu nu se scalează suficient. Ce aveam nevoie era o soluție care putea rula pe un VPS de 5$ și, în același timp, să se poată conecta la un cluster GPU. Această cerință poate nu sună extremă, dar este suficientă pentru a speria multe echipe.

În practică, multe așa-numite „Agenți all-in-one" fie rulează doar în cloud, fie necesită costuri de implementare locală absurd de mari. După două săptămâni de cercetare a diferitelor abordări, am luat o decizie curajoasă: reconstruirea întregului nucleu Agent în jurul Hermes ca motor de bază pentru Agentul nostru integrat.

Tot ce a urmat poate a fost pur și simplu destin.

## Despre HagiCode

Abordarea prezentată în acest articol provine din experiența reală în proiectul [HagiCode](https://hagicode.com). HagiCode este o platformă de codare asistată de AI care oferă dezvoltatorilor un asistent de codare inteligent prin intermediul unei extensii VSCode, unui client desktop și serviciilor web. S-ar putea să fi folosit instrumente similare înainte și să fi simțit că le lipsește acel ultim detaliu; înțelegem bine acel sentiment.

## De ce are nevoie HagiCode de Hermes

Înainte de a aprofida Hermes în sine, ajută să explicăm de ce HagiCode avea nevoie de așa ceva în primul rând. Lucrurile rareori funcționează exact așa cum doriți, așa că aveți nevoie de un motiv practic pentru a vă angaja într-o direcție tehnică.

Ca asistent de codare AI, HagiCode trebuie să suporte simultan mai multe scenarii de utilizare:
- Medii de dezvoltare locale: dezvoltatorii vor să îl ruleze pe propriile mașini astfel încât datele nu părăsesc niciodată mediul local. În zilele noastre, securitatea datelor nu este niciodată o preocupare trivială.
- Medii de colaborare în echipă: echipe mici ar trebui să poată partaja o implementare Agent care rulează pe un server. Economisirea banilor contează, și toată lumea are limite.
- Extindere elastică în cloud: la gestionarea sarcinilor complexe, sistemul ar trebui să se scaleze automat către un cluster GPU. Este întotdeauna mai bine să fii pregătit.

Această cerință de „a dori totul simultan" este ceea ce ne-a condus către Hermes. Dacă a fost alegerea perfectă, nu pot spune cu siguranță, dar la momentul respectiv nu am văzut o opțiune mai bună.

## Ce este Hermes Agent

Hermes Agent este un Agent AI autonom creat de **Nous Research**. Unii cititori s-ar putea să nu fie familiarizați cu Nous Research; ei sunt laboratorul din spatele modelelor mari open-source precum Hermes, Nomos și Psyché. Au construit multe lucruri excelente, chiar dacă sunt încă mai subapreciați decât merită.

Spre deosebire de asistenții tradiționali de codare IDE sau simplii wrapper-i de chat API, Hermes are o trăsătură definitorie: cu cât rulează mai mult, cu atât devine mai capabil. Nu este conceput să completeze o sarcină o singură dată și să se oprească; continuă să învețe și să acumuleze experiență pe parcursul operării de lungă durată. În acest sens, se simte oarecum ca o persoană.

### Caracteristici de bază

Mai multe capabilități de bază ale Hermes se aliniază foarte îndeaproape cu nevoile HagiCode.

Acest lucru înseamnă că HagiCode poate alege cel mai potrivit model de implementare bazat pe scenariul fiecărui utilizator: persoanele fizice îl rulează local, echipele îl implementează pe servere, iar sarcinile complexe folosesc resurse GPU. O singură bază de cod gestionează totul. Într-o lume atât de aglomerată, salvarea unui strat de complexitate este deja o victorie.

**Gateway de mesagerie multi-platformă**
Hermes suportă nativ Telegram, Discord, Slack, WhatsApp și multe altele. Pentru HagiCode, acest lucru înseamnă că putem suporta asistenți AI pe acele canale mult mai ușor în viitor. Mai multe căi înainte sunt întotdeauna binevenite.

**Sistem de instrumente bogat**
Hermes vine cu 40+ instrumente integrate și suportă extensii MCP (Model Context Protocol). Acest lucru este esențial pentru un asistent de codare: executarea comenzilor shell, lucrul cu sistemul de fișiere și apelarea Git depind toate de suportul instrumentelor. Un Agent fără instrumente este ca o pasăre fără aripi.

**Memorie cross-sesiune**
Hermes include un sistem de memorie persistentă și folosește căutare full-text FTS5 pentru a reamini conversațiile istorice. Aceasta permite Agentului să își amintească contextul anterior în loc să „își piardă memoria" de fiecare dată. Uneori oamenii și-ar dori să poată uita lucruri atât de ușor, dar realitatea este de obicei mai puțin generoasă.

## Cum integrează HagiCode Hermes

Acum că „de ce-ul" este clar, să ne uităm la „cum-ul". Odată ce ceva are sens în teorie, următorul pas este să îl construiești.

### Abstracția stratului Provider

În arhitectura HagiCode, toți Providerii AI implementează o interfață `IAIProvider` unificată:

```csharp
public sealed class HermesCliProvider : IAIProvider, IVersionedAIProvider
{
    public ProviderCapabilities Capabilities { get; } = new ProviderCapabilities
    {
        SupportsStreaming = true,   // Supports streaming output
        SupportsTools = true,       // Supports tool invocation
        SupportsSystemMessages = true, // Supports system prompts
        SupportsArtifacts = false
    };
}
```

Acest strat de abstracție permite HagiCode să comute fără probleme între diferiți Provideri AI. Indiferent dacă backend-ul este OpenAI, Claude sau Hermes, modelul de apelare de la nivel superior rămâne exact același. Pe scurt, menține lucrurile simple.

### Protocolul de comunicare ACP

Hermes comunică prin **ACP (Agent Communication Protocol)**. Acest protocol este conceput special pentru comunicarea între Agenți, iar metodele sale principale includ:

| Metodă | Descriere |
|------|------|
| `initialize` | Inițializează conexiunea și obține versiunea protocolului și capabilitățile clientului |
| `authenticate` | Gestionează autentificarea și suportă metode multiple de autentificare |
| `session/new` | Creează o sesiune nouă și configurează directorul de lucru și serverele MCP |
| `session/prompt` | Trimite un prompt și primește un răspuns |

HagiCode implementează stratul de transport ACP prin `StdioAcpTransport`, lansând un subproces Hermes și comunicând cu el prin intrarea și ieșirea standard. Poate suna complicat, dar în practică este gestionabil atâta timp cât aveți suficientă răbdare.

### Gestionarea configurației

Configurația este gestionată prin clasa `HermesPlatformConfiguration`:

```csharp
public sealed class HermesPlatformConfiguration : IAcpPlatformConfiguration
{
    public string ExecutablePath { get; set; } = "hermes";
    public string Arguments { get; set; } = "acp";
    public int StartupTimeoutMs { get; set; } = 5000;
    public string ClientName { get; set; } = "HagiCode";
    public HermesAuthenticationConfiguration Authentication { get; set; }
    public HermesSessionDefaultsConfiguration SessionDefaults { get; set; }
}
```

Configurați Hermes în `appsettings.json`:

```json
{
  "Providers": {
    "HermesCli": {
      "ExecutablePath": "hermes",
      "Arguments": "acp",
      "StartupTimeoutMs": 10000,
      "ClientName": "HagiCode",
      "Authentication": {
        "PreferredMethodId": "api-key",
        "MethodInfo": {
          "api-key": "your-api-key-here"
        }
      },
      "SessionDefaults": {
        "Model": "claude-sonnet-4-20250514",
        "ModeId": "default"
      }
    }
  }
}
```

Configurația pare adesea simplă pe hârtie, dar obținerea fiecărui detaliu corect necesită încă un efort real.

### Arhitectura distribuită Orleans

HagiCode folosește Orleans pentru a construi sistemul său distribuit, iar integrarea Hermes este implementată prin următoarele componente:

- **HermesGrain**: O implementare Orleans Grain care gestionează execuția sesiunilor
- **HermesPlatformConfiguration**: Configurație specifică platformei
- **HermesAcpSessionAdapter**: Adaptor de sesiune ACP
- **HermesConsole**: O consolă de validare dedicată

Numele Orleans are un anumit farmec. Chiar dacă acest Orleans nu are nicio legătură cu orașul legendar, un nume bun nu strică niciodată.

### Fluxul de execuție end-to-end

Următorul este logica de execuție de bază a Providerului Hermes:

```csharp
private async IAsyncEnumerable<AIStreamingChunk> StreamCoreAsync(
    AIRequest request,
    string? embeddedCommandPrompt,
    [EnumeratorCancellation] CancellationToken cancellationToken)
{
    // 1. Create transport layer and launch Hermes subprocess
    await using var transport = new StdioAcpTransport(
        platformConfiguration.GetExecutablePath(),
        platformConfiguration.GetArguments(),
        platformConfiguration.GetEnvironmentVariables(),
        platformConfiguration.GetStartupTimeout(),
        _loggerFactory.CreateLogger<StdioAcpTransport>());
    await transport.ConnectAsync(cancellationToken);

    // 2. Initialize and obtain protocol version and authentication methods
    var initializeResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "initialize",
        BuildInitializeParameters(platformConfiguration), cancellationToken);

    // 3. Handle authentication
    var authMethods = ParseAuthMethods(initializeResult);
    if (!isAuthenticated)
    {
        var methodId = platformConfiguration.Authentication.ResolveMethodId(authMethods);
        await SendHermesRequestAsync(transport, nextRequestId++, "authenticate", ...);
    }

    // 4. Create session
    var newSessionResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "session/new",
        BuildNewSessionParameters(platformConfiguration, workingDirectory, model), cancellationToken);
    var sessionId = ParseSessionId(newSessionResult);

    // 5. Execute prompt and collect streaming responses
    await foreach (var payload in transport.ReceiveMessagesAsync(cancellationToken))
    {
        // Handle session/update notifications and convert them into streaming chunks
        if (TryParseSessionNotification(root, out var notification))
        {
            if (_responseMapper.TryConvertToStreamingChunk(notification, out var chunk))
            {
                yield return chunk;
            }
        }
    }
}
```

Cu codul, detaliile devin în cele din urmă familiare. Ceea ce contează cel mai mult este abordarea generală.

### Verificări de sănătate

Pentru a ne asigura că Hermes rămâne disponibil, HagiCode implementează un mecanism de verificare a sănătății:

```csharp
public async Task<ProviderTestResult> PingAsync(CancellationToken cancellationToken = default)
{
    var response = await ExecuteAsync(
        new AIRequest
        {
            Prompt = "Reply with exactly PONG.",
            CessionId = null,
            AllowedTools = Array.Empty<string>(),
            WorkingDirectory = ResolveWorkingDirectory(null)
        },
        cancellationToken);

    var success = string.Equals(response.Content.Trim(), "PONG", StringComparison.OrdinalIgnoreCase);
    return new ProviderTestResult
    {
        ProviderName = Name,
        Success = success,
        ResponseTimeMs = stopwatch.ElapsedMilliseconds,
        ErrorMessage = success ? null : $"Unexpected Hermes ping response: '{response.Content}'."
    };
}
```

Acesta este, aproximativ, cum arată o „verificare de sănătate" aici. Într-un anumit sens, oamenii nu sunt foarte diferiți: ajută să te verifici din când în când, chiar dacă nimeni nu îți spune exact ce să cauți.

## Considerații practice

Există câteva capcane care merită înțelese înainte de a integra Hermes. Toată lumea cade în câteva capcane mai devreme sau mai târziu.

### Configurarea metodei de autentificare

Hermes suportă mai multe metode de autentificare, inclusiv chei API și token-uri, deci trebuie să alegeți în funcție de scenariul real de implementare. Configurația greșită poate cauza eșecuri de conectare, iar mesajele de eroare rezultate nu sunt întotdeauna intuitive. Uneori eroarea raportată este departe de cauza reală, ceea ce înseamnă că depanarea lentă și atentă este inevitabilă.

### Configurarea serverului MCP

La crearea unei sesiuni, puteți configura o listă de servere MCP astfel încât Hermes să poată apela instrumente externe. Dar aveți în vedere următoarele puncte:
- Adresele serverelor MCP trebuie să fie accesibile
- Timeout-urile trebuie configurate rezonabil
- Sistemul necesită gestionare a degradării când un server este indisponibil

În practică, gândirea defensivă contează mai mult decât se așteaptă oamenii.

### Gestionarea directorului de lucru

Fiecare sesiune trebuie să specifice un director de lucru astfel încât Hermes să poată accesa fișierele proiectului corect. În scenariile cu multiple proiecte, directorul de lucru trebuie să comute dinamic. Sună simplu, dar există mai multe cazuri limită decât v-ați putea gândi.

### Agregarea răspunsurilor

Răspunsurile Hermes pot fi împărțite între notificările `session/update` și rezultatul final, deci trebuie să fie îmbinate corect. Altfel, conținutul poate fi pierdut.

### Strategia de gestionare a erorilor

Erorile la execuție ar trebui returnate explicit în loc să se facă fallback silențios la un alt Provider. În acest fel, utilizatorii știu că problema a venit de la Hermes în loc să se întrebe de ce sistemul a comutat brusc modele în fundal.

## Concluzie

Decizia HagiCode de a folosi Hermes ca nucleu de Agent integrat nu a fost un impuls casual. A fost o alegere atentă bazată pe cerințele practice și caracteristicile tehnice ale framework-ului. Dacă se va dovedi a fi răspunsul perfect pe termen lung, este încă prea devreme pentru a spune, dar până acum ne-a servit bine.

Hermes oferă HagiCode flexibilitatea de a se adapta la o gamă largă de scenarii. Sistemul său puternic de instrumente și suportul MCP permit asistentului AI să facă muncă reală, în timp ce protocolul ACP și stratul de abstracție Provider mențin procesul de integrare clar și controlabil.

Dacă alegeți un framework Agent pentru propriul proiect AI, sper că acest articol oferă o referință utilă. Alegerea arhitecturii de bază potrivite poate face tot ceea ce urmează mult mai ușor.
