## Design Arhitectural

### Prezentare Generală a Arhitecturii Stratificate

Integrarea dintre C# și CodeBuddy folosește o arhitectură stratificată clară. Acest design face responsabilitățile explicite și face mentenanța pe termen lung mult mai ușoară:

```
┌─────────────────────────────────────────────┐
│           Stratul Contractului Furnizorului │
│   Enum AIProviderType + metode de extensie  │
├─────────────────────────────────────────────┤
│           Stratul Fabricii Furnizorilor     │
│   Fabrică de injecție de dependențe AIProviderFactory │
├─────────────────────────────────────────────┤
│           Stratul Implementării Furnizorului│
│   Implementare concretă CodebuddyCliProvider│
├─────────────────────────────────────────────┤
│           Stratul Infrastructurii ACP       │
│  ACPSessionManager / StdioAcpTransport      │
│  AcpRpcClient / AcpAgentClient              │
└─────────────────────────────────────────────┘
```

Care sunt beneficiile acestei stratificări? Simplu spus, fiecare strat stă departe de celelalte. Dacă mai târziu vrem să schimbăm mecanismul de comunicare, de exemplu de la `stdio` la WebSocket, trebuie să modificăm doar stratul de jos, iar logica de business de deasupra rămâne neatinsă. Nimeni nu vrea ca o schimbare de comunicare să se propage prin toată baza de cod.

### Defalcarea Componentelor de Bază

**Stratul contractului furnizorului** este fundația întregii arhitecturi. Definim enumerarea `AIProviderType`, unde `CodebuddyCli = 3` este folosită ca valoare de enumerare și implementăm maparea bidirecțională între șiruri și enumerări prin metode de extensie. Asta permite ca șirurile din fișierele de configurare să fie convertite convenabil în enumerări, iar enumerările să fie convertite înapoi în șiruri pentru ieșirea de depanare.

**Stratul fabricii furnizorilor** este responsabil pentru crearea instanței de Furnizor corespunzătoare pe baza configurației. Folosește injecția de dependențe .NET împreună cu `ActivatorUtilities.CreateInstance` pentru crearea dinamică. Avantajul modelului fabricii este că la adăugarea unui Furnizor nou trebuie doar să adaugi logica de creare în loc să modifici codul existent.

**Stratul implementării furnizorului** este unde are loc munca reală. `CodebuddyCliProvider` implementează interfața `IAIProvider` și oferă două moduri de invocare: `ExecuteAsync` pentru apeluri non-streaming și `StreamAsync` pentru apeluri în flux.

**Stratul infrastructurii ACP** oferă fundația de comunicare dedesubt. Acest strat gestionează toate detaliile de protocol, inclusiv gestionarea proceselor, serializarea mesajelor și analizarea răspunsurilor. Este fundația care menține totul deasupra stabil.

## Mecanismul de Comunicare

### Modul de Transport Stdio

CodeBuddy folosește **Stdio (intrare/ieșire standard)** pentru a comunica cu procesele externe. Comanda de pornire este simplă:

```bash
codebuddy --acp
```

După aceea, mesajele JSON-RPC sunt schimbate prin intrarea și ieșirea standard. Această abordare are mai multe avantaje:

1. **Pornire rapidă**: comunicarea prin procese locale evită latența de rețea
2. **Configurație simplă**: trebuie doar să specifici calea executabilului
3. **Izolare de mediu**: fiecare sesiune rulează într-un proces independent, deci nu se afectează reciproc

Injectarea variabilelor de mediu este suportată în timpul comunicării. Exemple comune includ:

- `CODEBUDDY_API_KEY`: autentificare prin cheie API
- `CODEBUDDY_INTERNET_ENVIRONMENT`: configurația mediului de rețea

Ca în comunicarea între oameni, ajută să alegi mai întâi un canal convenabil.

### Protocolul de Mesaje

ACP se bazează pe JSON-RPC 2.0. Formatul mesajului arată aproximativ astfel:

```json
// Request message
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "agent/prompt",
  "params": {
    "prompt": "Help me write a sorting algorithm",
    "sessionId": "session-123"
  }
}

// Response message
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": "Here is the AI response..."
  }
}
```

În implementarea reală, încapsulăm toate aceste detalii de protocol astfel încât stratul de business superior trebuie să se preocupe doar de prompt și răspuns.

## Implementarea de Bază

### 1. Restaurarea Contractului Furnizorului

Mai întâi, restaurează tipul CodeBuddy în fișierul de enumerare:

```csharp
// PCode.Models/AIProviderType.cs
public enum AIProviderType
{
    ClaudeCodeCli = 0,
    CodexCli = 1,
    GitHubCopilot = 2,
    CodebuddyCli = 3,  // Restore this enum value
    OpenCodeCli = 4,
    IFlowCli = 5,
}
```

Apoi adaugă maparea de șiruri în metodele de extensie astfel încât fișierul de configurare să poată specifica Furnizorul prin șir:

```csharp
// AIProviderTypeExtensions.cs
private static readonly Dictionary<string, AIProviderType> _typeMap = new(
    StringComparer.OrdinalIgnoreCase)
{
    ["CodebuddyCli"] = AIProviderType.CodebuddyCli,
    ["Codebuddy"] = AIProviderType.CodebuddyCli,
    ["codebuddy"] = AIProviderType.CodebuddyCli,
    // ... Mappings for other providers
};
```

### 2. Integrarea Fabricii Furnizorilor

Adaugă o ramură de creare CodeBuddy în clasa fabricii:

```csharp
// AIProviderFactory.cs
private IAIProvider? CreateProvider(AIProviderType providerType, ProviderConfiguration config)
{
    return providerType switch
    {
        AIProviderType.CodebuddyCli =>
            ActivatorUtilities.CreateInstance<CodebuddyCliProvider>(
                _serviceProvider,
                Options.Create(config)),
        // ... Other providers
        _ => throw new NotSupportedException($"Provider {providerType} not supported")
    };
}
```

Aceasta folosește injecția de dependențe prin `ActivatorUtilities`, care gestionează automat injecția parametrilor constructorului și este foarte convenabilă.

### 3. Implementarea Completă a Furnizorului

Mai jos este implementarea de bază a `CodebuddyCliProvider`, acoperind ambele moduri de invocare, în flux și non-flux:

```csharp
public class CodebuddyCliProvider : IAIProvider
{
    private readonly ILogger<CodebuddyCliProvider> _logger;
    private readonly IACPSessionManager _sessionManager;
    private readonly ProviderConfiguration _config;

    public string Name => "CodebuddyCli";
    public bool SupportsStreaming => true;
    public ProviderCapabilities Capabilities { get; }

    public CodebuddyCliProvider(
        ILogger<CodebuddyCliProvider> logger,
        IACPSessionManager sessionManager,
        IOptions<ProviderConfiguration> config)
    {
        _logger = logger;
        _sessionManager = sessionManager;
        _config = config.Value;

        // Define the capabilities of the current Provider
        Capabilities = new ProviderCapabilities
        {
            SupportsStreaming = true,
            SupportsTools = true,
            SupportsSystemMessages = true,
            SupportsArtifacts = false,
            MaxTokens = 8192
        };
    }

    // Non-streaming call: return all results together after completion
    public async Task<AIResponse> ExecuteAsync(
        AIRequest request,
        CancellationToken cancellationToken = default)
    {
        // Create an independent session for the request
        var session = await _sessionManager.CreateSessionAsync(
            "CodebuddyCli",
            request.WorkingDirectory,
            cancellationToken,
            request.SessionId);

        try
        {
            var fullPrompt = BuildPrompt(request);
            await session.SendPromptAsync(fullPrompt, cancellationToken);

            var responseBuilder = new StringBuilder();
            var toolCalls = new List<AIToolCall>();

            // Collect all response chunks
            await foreach (var chunk in StreamFromSession(session, cancellationToken))
            {
                if (!string.IsNullOrEmpty(chunk.Content))
                {
                    responseBuilder.Append(chunk.Content);
                }
                // Handle tool calls...
            }

            return new AIResponse
            {
                Content = AIResultContentSanitizer.SanitizeResultContent(
                    responseBuilder.ToString()),
                ToolCalls = toolCalls,
                Provider = Name,
                Model = string.Empty
            };
        }
        finally
        {
            // Release session resources
            await session.DisposeAsync();
        }
    }

    // Streaming call: return response chunks in real time
    public async IAsyncEnumerable<AIStreamingChunk> StreamAsync(
        AIRequest request,
        [EnumeratorCancellation] CancellationToken cancellationToken = default)
    {
        var session = await _sessionManager.CreateSessionAsync(
            "CodebuddyCli",
            request.WorkingDirectory,
            cancellationToken);

        try
        {
            var fullPrompt = BuildPrompt(request);
            await session.SendPromptAsync(fullPrompt, cancellationToken);

            await foreach (var chunk in StreamFromSession(session, cancellationToken))
            {
                yield return chunk;
            }
        }
        finally
        {
            await session.DisposeAsync();
        }
    }

    private async IAsyncEnumerable<AIStreamingChunk> StreamFromSession(
        IACPSession session,
        [EnumeratorCancellation] CancellationToken cancellationToken)
    {
        // Iterate through all updates in the session
        await foreach (var notification in session.ReceiveUpdatesAsync(cancellationToken))
        {
            switch (notification.Update)
            {
                case AgentMessageChunkSessionUpdate agentMessage:
                    // Handle text content chunks
                    if (agentMessage.Content is AcpImp.TextContentBlock textContent)
                    {
                        yield return new AIStreamingChunk
                        {
                            Content = textContent.Text,
                            Type = StreamingChunkType.ContentDelta,
                            IsComplete = false
                        };
                    }
                    break;

                case ToolCallSessionUpdate toolCall:
                    // Handle tool calls
                    yield return new AIStreamingChunk
                    {
                        Content = string.Empty,
                        Type = StreamingChunkType.ToolCallDelta,
                        ToolCallDelta = new AIToolCallDelta
                        {
                            Id = toolCall.ToolCallId,
                            Name = toolCall.Kind.ToString(),
                            Arguments = toolCall.RawInput?.ToString()
                        }
                    };
                    break;

                case AcpImp.PromptCompletedSessionUpdate:
                    // Response complete
                    yield break;
            }
        }
    }

    // Build the full prompt
    private string BuildPrompt(AIRequest request, string? embeddedCommandPrompt = null)
    {
        var sb = new StringBuilder();

        // Embedded command prompt, if present
        if (!string.IsNullOrEmpty(embeddedCommandPrompt))
        {
            sb.AppendLine(embeddedCommandPrompt);
            sb.AppendLine();
        }

        // System message
        if (!string.IsNullOrEmpty(request.SystemMessage))
        {
            sb.AppendLine(request.SystemMessage);
            sb.AppendLine();
        }

        // User prompt
        sb.Append(request.Prompt);
        return sb.ToString();
    }
}
```

Există mai multe puncte cheie în acest cod:

1. **Gestionarea sesiunilor**: fiecare cerere creează o sesiune independentă și eliberează resursele după ce cererea se finalizează. Aceasta este o lecție învățată prin încercări și erori. Dacă reutilizarea sesiunilor nu este gestionată bine, poluarea stării apare ușor.

2. **Procesarea în flux**: `IAsyncEnumerable` permite returnarea răspunsului în timp ce este încă generat, în loc să aștepte finalizarea întregului conținut. Acest lucru este deosebit de important pentru scenariile cu text lung și îmbunătățește semnificativ experiența utilizatorului.

3. **Apeluri de instrumente**: CodeBuddy suportă apelarea de instrumente (Function Calling), gestionate prin `ToolCallSessionUpdate`. Această capacitate este critică pentru sarcinile complexe de editare a codului.

4. **Filtrarea conținutului**: `AIResultContentSanitizer` este folosit pentru a filtra conținutul blocurilor Think și a menține ieșirea curată.

### 4. Configurarea Injecției de Dependențe

Adaugă serviciile corelate la înregistrarea modulului:

```csharp
// PCodeClaudeHelperModule.cs
public void ConfigureModule(IServiceCollection context)
{
    // Register Provider
    context.Services.AddTransient<CodebuddyCliProvider>();

    // Register ACP infrastructure
    context.Services.AddSingleton<IACPSessionManager, ACPSessionManager>();
    context.Services.AddSingleton<IAcpPlatformConfigurationResolver, AcpPlatformConfigurationResolver>();
    context.Services.AddSingleton<IAIRequestToAcpMapper, AIRequestToAcpMapper>();
    context.Services.AddSingleton<IAcpToAIResponseMapper, AcpToAIResponseMapper>();
}
```

## Exemplu de Configurație

### Fișier de Configurație

Adaugă configurația corelată CodeBuddy în `appsettings.json`:

```yaml
AI:
  # Default Provider to use
  DefaultProvider: "CodebuddyCli"

  # Provider configuration
  Providers:
    CodebuddyCli:
      Type: "CodebuddyCli"
      WorkingDirectory: "C:/projects/my-app"
      ExecutablePath: "C:/tools/codebuddy.cmd"

  # Platform-specific configuration
  PlatformConfigurations:
    CodebuddyCli:
      ExecutablePath: "C:/tools/codebuddy.cmd"
      Arguments: "--acp"
      StartupTimeoutMs: 5000
      EnvironmentVariables:
        CODEBUDDY_API_KEY: "${CODEBUDDY_API_KEY}"
        CODEBUDDY_INTERNET_ENVIRONMENT: "production"
```

### Modelul de Configurație

Definiția modelului de configurație corespunzător:

```csharp
public class CodebuddyPlatformConfiguration : IAcpPlatformConfiguration
{
    public string ProviderName => "CodebuddyCli";
    public AcpTransportType TransportType => AcpTransportType.Stdio;

    public string ExecutablePath { get; set; } = "codebuddy";
    public string Arguments { get; set; } = "--acp";
    public int StartupTimeoutMs { get; set; } = 5000;

    public Dictionary<string, string?>? EnvironmentVariables { get; set; }
}
```

## Lecții Practice Învățate

### Jurnalul Capcanelor

Ne-am confruntat cu mai multe capcane tipice în timpul implementării și le împărtășim aici pentru a ajuta pe alții să evite aceleași ocoluri:

1. **Problema scurgerii de sesiuni**: la început, sesiunile nu erau eliberate corect, ceea ce a epuizat resursele proceselor. Soluția a fost să folosim `try-finally` pentru a asigura eliberarea resurselor pentru fiecare cerere.

2. **Transmiterea variabilelor de mediu**: Windows și Linux folosesc sintaxe diferite pentru variabilele de mediu, deci ulterior am standardizat pe `Dictionary<string, string?>` pentru a gestiona acest lucru.

3. **Configurația timeout-ului**: pornirea CLI durează timp, deci am setat un timeout de pornire de 5 secunde pentru a evita eșecurile rapide ale cererilor.

4. **Probleme de codificare**: pe Windows, codificarea implicită poate cauza text chinezesc distorsionat, deci codificarea UTF-8 este specificată explicit la pornirea procesului.

### Optimizarea Performanței

1. **Pool de sesiuni**: pentru cereri scurte frecvente, consideră implementarea unui pool de sesiuni pentru a reutiliza procesele
2. **Cache de conexiuni**: clasa fabricii suportă deja stocarea în cache a instanțelor Furnizorilor
3. **Asincron pe primul loc**: folosește programare asincronă pe tot parcursul pentru a evita blocarea firelor de execuție

Performanța merită întotdeauna optimizată. Cu cât utilizatorii așteaptă mai mult, cu atât experiența devine mai proastă.

## Concluzie

Acest articol a prezentat o soluție completă pentru integrarea CodeBuddy CLI într-un backend C#, acoperind întregul proces de la designul arhitectural până la implementarea concretă. Printr-o arhitectură stratificată, separăm detaliile de protocol de logica de business, făcând codul mai clar și mai ușor de menținut.

Concluzii cheie:

- Folosește o arhitectură stratificată cu strat de contract Furnizor, strat de fabrică, strat de implementare și strat de infrastructură
- Folosește JSON-RPC peste Stdio pentru comunicare inter-proces
- Implementează configurație și extensibilitate flexibilă prin injecție de dependențe
- Oferă ambele moduri de invocare, în flux și non-flux

Această abordare nu este potrivită doar pentru CodeBuddy; adăugarea de Furnizori AI noi poate urma același model. Dacă construiești și tu o integrare similară multi-Furnizor AI, sper că acest articol îți oferă o referință utilă.
