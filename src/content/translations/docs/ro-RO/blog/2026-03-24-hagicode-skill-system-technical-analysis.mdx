## Analiză tehnică a sistemului de abilități HagiCode: Construirea unei platforme scalabile de gestionare a abilităților AI

> Acest articol analizează în profunzime arhitectura și implementarea sistemului de gestionare Skill din proiectul HagiCode, acoperind detaliile tehnice din spatele a patru capabilități de bază: gestionarea globală locală, căutare în marketplace, recomandări inteligente și gestionarea furnizorilor de încredere.

## Context

În domeniul asistenților de codare AI, modul de extindere a limitelor capabilităților AI a fost întotdeauna o întrebare de bază. Claude Code în sine este deja puternic la asistența de cod, dar diferite echipe de dezvoltare și diferite stack-uri tehnologice au adesea nevoie de capabilități specializate pentru scenarii specifice, cum ar fi gestionarea implementărilor Docker, optimizarea bazelor de date sau generarea de componente frontend. Exact acolo un sistem de abilități devine deosebit de important.

În timpul dezvoltării proiectului HagiCode, am întâmpinat o provocare similară: cum lăsăm Claude Code să „învețe" abilități profesionale noi ca o persoană, menținând în același timp o experiență de utilizare solidă și o bună mentenabilitate de inginerie? Această problemă este și grea și simplă în felul ei. În jurul acelei întrebări, am proiectat și implementat un sistem complet de gestionare Skill.

Acest articol parcurge arhitectura tehnică și implementarea de bază a sistemului în detaliu. Este destinat dezvoltatorilor interesați de extensibilitatea AI și integrarea instrumentelor din linia de comandă. V-ar putea fi util, sau nu, dar cel puțin este scris acum.

## Despre HagiCode

Abordarea prezentată în acest articol provine din experiența noastră practică în proiectul [HagiCode](https://hagicode.com). HagiCode este un proiect open-source de asistent de codare AI conceput pentru a ajuta echipele de dezvoltare să îmbunătățească eficiența de inginerie. Stack-ul proiectului include ASP.NET Core, framework-ul distribuit Orleans, un frontend TanStack Start + React și subsistemul de gestionare Skill introdus în acest articol.

Depozitul GitHub este [HagiCode-org/site](https://github.com/HagiCode-org/site). Dacă găsiți abordarea tehnică din acest articol valoroasă, nu ezitați să îi dați un Star. Mai multe Star-uri tind să îmbunătățească dispoziția, până la urmă.

## Prezentare generală a arhitecturii sistemului

Sistemul Skill folosește o arhitectură separată frontend-backend. Nu este nimic deosebit de misterios în asta.

**Frontend-ul** folosește TanStack Start + React pentru a construi interfața cu utilizatorul, cu Redux Toolkit gestionând starea. Cele patru capabilități principale se mapează direct la patru componente Tab: Abilități Locale, Galerie de Abilități, Recomandări Inteligente și Furnizori de Încredere. La urma urmei, designul este în mare parte despre îmbunătățirea experienței utilizatorului.

**Backend-ul** este bazat pe ASP.NET Core + ABP Framework, folosind Orleans Grain pentru gestionarea stării distribuite. Clientul API online împachetează interfața `IOnlineApiClient` pentru a comunica cu serviciul de catalog de abilități remote.

Principiul arhitectural general este separarea execuției comenzilor de logica de business. Prin modelul adaptorului, detaliile de implementare ale execuției comenzilor npm/npx sunt ascunse în interiorul modulelor independente. La urma urmei, nimeni nu vrea cu adevărat apeluri de linie de comandă împrăștiate peste tot în cod.

## Capabilitatea de bază 1: Gestionarea globală locală

Gestionarea globală locală este cel mai de bază modul. Este responsabilă pentru listarea abilităților instalate și suportarea operațiunilor de dezinstalare. Nu este nimic excesiv de complicat aici; este în mare parte despre a face bine elementele de bază.

### Abordarea tehnică

Implementarea se află în `LocalSkillsTab.tsx` și `LocalSkillCommandAdapter.cs`. Ideea de bază este de a împacheta comanda `npx skills`, de a parsa ieșirea sa JSON și de a o converti în structuri de date interne. Sună simplu, și în practică este în mare parte așa.

```csharp
public async Task<IReadOnlyList<LocalSkillInventoryResponseDto>> GetLocalSkillsAsync(
    CancellationToken cancellationToken = default)
{
    var result = await _commandAdapter.ListGlobalSkillsAsync(cancellationToken);
    return result.Skills.Select(skill => new LocalSkillInventoryResponseDto
    {
        Name = skill.Name,
        Version = skill.Version,
        Source = skill.Source,
        InstalledPath = skill.InstalledPath,
        Description = skill.Description
    }).ToList();
}
```

Fluxul de date este foarte clar: frontend-ul trimite o cerere -> `SkillGalleryAppService` o primește -> `LocalSkillCommandAdapter` execută comanda npx -> rezultatul JSON este parsat -> un DTO este returnat. Fiecare pas decurge în mod natural din cel anterior.

Dezinstalarea abilităților folosește comanda `npx skills remove -g <skillName> -y`, iar sistemul gestionează automat dependențele și curățarea. Metadatele de instalare sunt stocate în `managed-install.json` în interiorul directorului abilității, înregistrând informații precum timpul instalării și versiunea sursei pentru actualizări și audit ulterioare. Unele lucruri pur și simplu merită înregistrate.

### Fluxul de instalare în detaliu

Instalarea abilităților necesită mai mulți pași coordonați. În realitate, nu este deosebit de complicat:

```csharp
public async Task<SkillInstallResultDto> InstallAsync(
    SkillInstallRequestDto request,
    CancellationToken cancellationToken = default)
{
    // 1. Normalize the installation reference
    var normalized = _referenceNormalizer.Normalize(
        request.SkillId,
        request.Source,
        request.SkillSlug,
        request.Version);

    // 2. Check prerequisites
    await _prerequisiteChecker.CheckAsync(cancellationToken);

    // 3. Acquire installation lock
    using var installLock = await _lockProvider.AcquireAsync(normalized.SkillId);

    // 4. Execute installation command
    var result = await _installCommandRunner.ExecuteAsync(
        new SkillInstallCommandExecutionRequest
        {
            Command = $"npx skills add {normalized.FullReference} -g -y",
            Timeout = TimeSpan.FromMinutes(4)
        },
        cancellationToken);

    // 5. Persist installation metadata
    await _metadataStore.WriteAsync(normalized.SkillPath, request);

    return new SkillInstallResultDto { Success = result.Success };
}
```

Aici sunt folosite mai multe modele de design cheie: **normalizatorul de referințe** convertește formate de intrare diferite, cum ar fi `tanweai/pua` și `@opencode/docker-skill`, într-o reprezentare internă unificată; **mecanismul de blocare a instalării** asigură că doar o operațiune de instalare poate rula pentru aceeași abilitate la un moment dat; și **ieșirea streaming** împinge progresul instalării către frontend în timp real prin Server-Sent Events, astfel încât utilizatorii pot vedea log-uri de tip terminal în timp ce se întâmplă.

La urma urmei, toate aceste modele sunt acolo pentru un singur scop: să mențină sistemul mai simplu de utilizat și întreținut.

## Capabilitatea de bază 2: Căutare în Marketplace

Căutarea în marketplace permite utilizatorilor să descopere și să instaleze abilități din comunitate. Capacitatea unei singure persoane este întotdeauna limitată; cunoștințele colective merg mult mai departe.

### Abordarea tehnică

Funcționalitatea de căutare se bazează pe API-ul online `https://api.hagicode.com/v1/skills/search`. Pentru a îmbunătăți viteza de răspuns, sistemul implementează cache. Cache-ul este oarecum ca memoria: dacă păstrați lucruri utile pe aproape, nu trebuie să vă gândiți atât de tare data viitoare.

```csharp
private async Task<IReadOnlyList<SkillGallerySkillDto>> SearchCatalogAsync(
    string query,
    CancellationToken cancellationToken,
    IReadOnlySet<string>? allowedSources = null)
{
    var cacheKey = $"skill_search:{query}:{string.Join(",", allowedSources ?? Array.Empty<string>())}";

    if (_memoryCache.TryGetValue(cacheKey, out var cached))
        return (IReadOnlyList<SkillGallerySkillDto>)cached!;

    var response = await _onlineApiClient.SearchAsync(
        new SearchSkillsRequest
        {
            Query = query,
            Limit = _options.LimitPerQuery,
        },
        cancellationToken);

    var results = response.Skills
        .Where(skill => allowedSources is null || allowedSources.Contains(skill.Source))
        .Select(skill => new SkillGallerySkillDto { ... })
        .ToList();

    _memoryCache.Set(cacheKey, results, TimeSpan.FromMinutes(10));
    return results;
}
```

Rezultatele căutării suportă filtrare după surse de încredere, astfel încât utilizatorii văd doar sursele de abilități în care au încredere. Interogările seed precum `popular` și `recent` sunt folosite pentru a inițializa catalogul, permițând utilizatorilor să vadă abilități populare recomandate prima dată când îl deschid. Primele impresii contează încă.

## Capabilitatea de bază 3: Recomandări inteligente

Recomandările inteligente sunt cea mai complexă parte a sistemului. Ele pot recomanda automat cele mai potrivite abilități bazat pe contextul proiectului curent. Complexă cum este, merită încă construită.

### Fluxul de recomandare

Fluxul complet de recomandare este împărțit în cinci etape:

```
1. Build project context
   ↓
2. AI generates search queries
   ↓
3. Search the online catalog in parallel
   ↓
4. AI ranks the candidates
   ↓
5. Return the recommendation list
```

Mai întâi, sistemul analizează caracteristici precum stack-ul tehnologic al proiectului, limbajele de programare și structura de domeniu pentru a construi un „profil de proiect". Acel profil este oarecum ca un CV, înregistrând trăsăturile cheie ale proiectului.

Apoi un AI Grain este folosit pentru a genera interogări de căutare direcționate. Acest design este de fatche destul de interesant: în loc să întrebăm direct AI-ul, „Ce abilități ar trebui să recomand?", îl întrebăm mai întâi să se gândească la „Ce termeni de căutare ar putea găsi abilități relevante?" Uneori modul în care puneți întrebarea contează mai mult decât răspunsul în sine:

```csharp
var queryGeneration = await aiGrain.GenerateSkillRecommendationQueriesAsync(
    projectContext,      // Project context
    locale,              // User language preference
    maxQueries,           // Maximum number of queries
    effectiveSearchHero); // AI model selection
```

Apoi, acele interogări de căutare sunt executate în paralel pentru a aduna o listă de abilități candidate. Procesarea paralelă este, la urma urmei, doar un mod de a economisi timp.

În final, un alt AI Grain clasează abilitățile candidate. Acest pas consideră factori precum relevanța abilității față de proiect, statusul de încredere și preferințele istorice ale utilizatorului:

```csharp
var ranking = await aiGrain.RankSkillRecommendationsAsync(
    projectContext,
    candidates,
    installedSkillNames,
    locale,
    maxRecommendations,
    effectiveRankingHero);

response.Items = MergeRecommendations(projectContext, candidates, ranking, maxRecommendations);
```

### Mecanismul de fallback

Modelele AI pot răspunde lent sau deveni temporar indisponibile. Chiar și cele mai bune sisteme se împiedică uneori. Din acest motiv, sistemul include un mecanism de fallback determinist: când serviciul AI este indisponibil, folosește un algoritm euristic bazat pe reguli pentru a genera recomandări, cum ar fi inferarea abilităților probabil necesare din dependențele din `package.json`.

Pe scurt, acest mecanism de fallback este pur și simplu un plan de rezervă pentru sistem.

## Capabilitatea de bază 4: Gestionarea furnizorilor de încredere

Gestionarea furnizorilor de încredere permite utilizatorilor să controleze ce surse de abilități sunt considerate de încredere. Încrederea este încă ceva ce utilizatorii ar trebui să poată defini ei înșiși.

### Reguli de potrivire

Furnizorii de încredere suportă două reguli de potrivire: **potrivire exactă** (`exact`) și **potrivire prefix** (`prefix`).

```csharp
public static TrustedSkillProviderResolutionSnapshot Resolve(
    TrustedSkillProviderSnapshot snapshot,
    string source)
{
    var normalizedSource = Normalize(source);

    foreach (var entry in snapshot.Entries.OrderBy(e => e.SortOrder))
    {
        if (!entry.IsEnabled) continue;

        foreach (var rule in entry.MatchRules)
        {
            bool isMatch = rule.MatchType switch
            {
                TrustedSkillProviderMatchRuleType.Exact
                    => string.Equals(normalizedSource, Normalize(rule.Value),
                        StringComparison.OrdinalIgnoreCase),
                TrustedSkillProviderMatchRuleType.Prefix
                    => normalizedSource.StartsWith(Normalize(rule.Value) + "/",
                        StringComparison.OrdinalIgnoreCase),
                _ => false
            };

            if (isMatch)
                return new TrustedSkillProviderResolutionSnapshot
                {
                    IsTrustedSource = true,
                    ProviderId = entry.ProviderId,
                    DisplayName = entry.DisplayName
                };
        }
    }

    return new TrustedSkillProviderResolutionSnapshot { IsTrustedSource = false };
}
```

Furnizorii de încredere încorporați includ organizații și proiecte cunoscute precum Vercel, Azure, anthropics, Microsoft și browser-use. Furnizori personalizați pot fi adăugați prin fișiere de configurare specificând un ID de furnizor, nume de afișare, etichetă de insignă, reguli de potrivire și multe altele. Lumea este suficient de mare încât doar câțiva furnizori încorporați nu ar fi niciodată suficienți.

### Implementarea persistenței

Configurația de încredere este persistată folosind un Orleans Grain:

```csharp
public class TrustedSkillProviderGrain : Grain<TrustedSkillProviderState>,
    ITrustedSkillProviderGrain
{
    public async Task UpdateConfigurationAsync(TrustedSkillProviderSnapshot snapshot)
    {
        State.Snapshot = snapshot;
        await WriteStateAsync();
    }

    public Task<TrustedSkillProviderSnapshot> GetConfigurationAsync()
    {
        return Task.FromResult(State.Snapshot);
    }
}
```

Beneficiul acestei abordări este că modificările de configurație sunt sincronizate automat pe toate nodurile, fără necesitatea de a reîmprospăta manual cache-urile. Automatizarea este, în cele din urmă, despre a lăsa oamenii să se îngrijoreze mai puțin.

## Design tehnic cheie

### Modelul adaptor pentru execuția comenzilor

Sistemul Skill trebuie să execute diverse comenzi npx. Dacă acea logică ar fi împrăștiată peste tot, codul ar deveni rapid dificil de întreținut. De aceea am proiectat o interfață de adaptor. Modelele de design, la urma urmei, există pentru a face codul mai ușor de întreținut:

```csharp
public interface ISkillInstallCommandRunner
{
    Task<SkillInstallCommandExecutionResult> ExecuteAsync(
        SkillInstallCommandExecutionRequest request,
        CancellationToken cancellationToken = default);
}
```

Comenzi diferite au implementări de executor diferite, dar toate implementează aceeași interfață, făcând testarea și înlocuirea simplă.

### Ieșire streaming SSE

Progresul instalării este împins către frontend în timp real prin Server-Sent Events:

```csharp
public async Task InstallWithProgressAsync(
    SkillInstallRequestDto request,
    IServerStreamWriter<SkillInstallProgressEventDto> stream,
    CancellationToken cancellationToken)
{
    var process = new Process
    {
        StartInfo = new ProcessStartInfo
        {
            FileName = "npx",
            Arguments = $"skills add {request.FullReference} -g -y",
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false
        }
    };

    process.OutputDataReceived += async (sender, e) =>
    {
        await stream.WriteAsync(new SkillInstallProgressEventDto
        {
            EventType = "output",
            Data = e.Data ?? string.Empty
        });
    };

    process.Start();
    process.BeginOutputReadLine();
    await process.WaitForExitAsync(cancellationToken);
}
```

Pe frontend, utilizatorii pot vedea ieșire de tip terminal în timp real, ceea ce face experiența foarte intuitivă. Feedback-ul în timp real ajută oamenii să se simtă în largul lor.

## Ghid practic

### Instalarea unei abilități din comunitate

Luați ca exemplu instalarea abilității `pua` (este o abilitate populară din comunitate):

1. Deschideți drawer-ul Skills și comutați la tab-ul `Skill Gallery`
2. Introduceți `pua` în caseta de căutare
3. Click pe rezultatul căutării pentru a vedea detaliile abilității
4. Click pe butonul `Install`
5. Comutați la tab-ul `Local Skills` pentru a confirma că instalarea a reușit

Comanda de instalare este `npx skills add tanweai/pua -g -y`, iar sistemul gestionează automat toate detaliile. Nu sunt chiar atât de mulți pași odată ce îi luați unul câte unul.

### Adăugarea unei surse de încredere personalizate

Dacă echipa dumneavoastră are propriul depozit de abilități, îl puteți adăuga ca sursă de încredere:

```yaml
providerId: "my-team"
displayName: "My Team Skills"
badgeLabel: "MyTeam"
isEnabled: true
sortOrder: 100
matchRules:
  - matchType: "prefix"
    value: "my-team/"
  - matchType: "exact"
    value: "my-team/special-skill"
```

În acest fel, toate abilitățile de la echipa dumneavoastră vor afișa o insignă de încredere, făcând utilizatorii mai confortabili să le instaleze. Etichetele și semnalele ajută într-adevăr oamenii să se simtă mai încrezători.

### Bazele dezvoltării de abilități

Crearea unei abilități personalizate necesită următoarea structură:

```
my-skill/
├── SKILL.md          # Skill metadata (YAML front matter)
├── index.ts          # Skill entry point
├── agents/           # Supported agent configuration
└── references/       # Reference resources
```

Un exemplu de format `SKILL.md`:

```yaml
---
name: my-skill
description: A brief description of what this skill does
---

# My Skill

Detailed documentation...
```

## Note

1. **Cerințe de rețea**: căutarea și instalarea abilităților necesită acces la `api.hagicode.com` și registrul npm
2. **Versiunea Node.js**: Node.js 18 sau ulterior este recomandat
3. **Cerințe de permisiuni**: sunt necesare permisiuni de instalare globală npm
4. **Controlul concurenței**: doar o operațiune de instalare sau dezinstalare poate rula pentru aceeași abilitate la un moment dat
5. **Setări de timeout**: timeout-ul implicit pentru instalare este de 4 minute, dar scenariile complexe pot necesita ajustare

Aceste note există, în cele din urmă, pentru a ajuta lucrurile să meargă fără probleme.

## Concluzie

Acest articol a introdus implementarea completă a sistemului de gestionare Skill din proiectul HagiCode. Printr-o arhitectură separată frontend-backend, modelul adaptorului, gestionarea stării distribuite bazată pe Orleans și tehnicile asociate, sistemul livrează:

- **Gestionare globală locală**: o interfață unificată de gestionare a abilităților construită prin împachetarea comenzilor `npx skills`
- **Căutare în marketplace**: descoperire rapidă a abilităților din comunitate prin API-ul online și mecanismele de cache
- **Recomandări inteligente**: recomandări de abilități bazate pe AI conform contextului proiectului
- **Gestionare încredere**: un sistem flexibil de configurație care permite utilizatorilor să controleze limitele de încredere

Această abordare de design nu este aplicabilă doar gestionării Skill. Este de asemenea utilă ca referință pentru orice scenariu care necesită integrarea instrumentelor din linia de comandă echilibrând în același timp stocarea locală și serviciile online.
