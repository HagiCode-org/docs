## Proiectarea a 12 experiențe de pornire excepționale pentru HagiCode: De la minimalism la cyberpunk

> Breșa scurtă dintre descărcarea unei aplicații React 19 și completarea hidratării este o oportunitate de aur pentru ca utilizatorii să simtă personalitatea mărcii dvs. În acest articol, partajăm un sistem complet de stiluri de pornire pe care l-am construit pentru proiectul HagiCode folosind HTML/CSS/JS.

## Context

Ca aplicație modernă construită cu ASP.NET Core 10 și React 19 (Vite), HagiCode folosește o arhitectură de implementare cu separare frontend-backend. Ieșirea construcției frontend este ambalată în directorul backend `wwwroot/` și găzduită de ASP.NET Core.

Totuși, această arhitectură introduce un punct dureros clasic de UX: când utilizatorii vizitează pagina, browser-ul încarcă mai întâi HTML-ul, apoi descarcă bundle-ul JS mare, și în final lasă React să efectueze hidratarea. În timpul acestei „perioade de vid" care durează de la câteva sute de milisecunde la câteva secunde, utilizatorii văd fie un ecran alb, fie o pagină statică fără viață.

Pentru a umple acel gol și a injecta personalitatea mărcii HagiCode, trebuia să proiectăm un sistem de stiluri de pornire implementat în întregime cu cod inline în interiorul `index.html`.

## Despre HagiCode

Abordarea de design a ecranului de pornire partajată în acest articol provine din experiența noastră practică în proiectul [HagiCode](https://github.com/HagiCode-org/site). Ca asistent de programare AI, HagiCode nu se preocupă doar de eficiența generării codului, ci și de experiența vizuală a dezvoltatorului. Acest sistem de pornire este unul dintre rezultatele urmăririi performanței frontend ultime.

## Provocări de bază și design arhitectural

Înainte de a începe proiectarea, trebuia mai întâi să clarificăm constrângerile tehnice. Deoarece totul trebuia implementat inline în `index.html`, nu puteam încărca niciun fișier CSS sau JS extern în afara bundle-ului propriu al React.

### Analiza constrângerilor tehnice

1.  **Principiul zero-dependență**: Toate stilurile trebuie să trăiască în interiorul unui tag `<style>`, și toată logica trebuie să trăiască în interiorul unui tag `<script>`.
2.  **CSS defensiv**: Pentru a preveni stilurile globale de a polua ecranul de pornire după ce aplicația React se montează, am decis să împachetăm toate stilurile de pornire cu un prefix ID de prioritate ridicată precum `#boot-screen`.
3.  **Performanță pe primul loc**: Animațiile ar trebui să folosească `transform` și `opacity` CSS oriunde este posibil pentru a evita reflow-ul și a asigura că thread-ul principal rămâne deblocat.
4.  **Consecvență vizuală**: Culorile și fonturile trebuie să rămână aliniate cu configurația Tailwind a HagiCode.

### Model arhitectural: Shell & Injector

Am adoptat un **model de variante**. Logica de bază este încapsulată în interiorul unei expresii de funcție invocate imediat (IIFE), în timp ce logica specifică de randare este injectată prin configurație. Acest lucru ne permite să comutăm între diferite stiluri prin simplă configurație în loc de a rescrie logica de manipulare DOM repetat.

Iată codul arhitectural de bază:

```html
<!-- Inline in index.html -->
<div id="boot-root"></div>

<script>
(function() {
  const BootSequence = {
    config: {
      theme: 'terminal', // Configurable as 'minimal', 'skeleton', 'code-rain', etc.
      color: '#3b82f6'   // Brand color
    },
    
    // Core lifecycle
    init() {
      this.render();
      this.listenForMount();
    },

    // Render the currently selected style
    render() {
      const root = document.getElementById('boot-root');
      if (this.variants[this.config.theme]) {
        root.innerHTML = this.variants[this.config.theme].render();
      }
    },

    // Listen for successful React mount and exit gracefully
    listenForMount() {
      window.addEventListener('hagicode:ready', () => {
        const screen = document.getElementById('boot-root');
        // Fade out first, then remove the DOM to avoid flicker
        screen.style.opacity = '0';
        screen.style.transition = 'opacity 0.3s ease';
        setTimeout(() => screen.remove(), 300);
      });
    },

    // The implementation logic for all 12 styles lives here
    variants: {
      // ...see details below
    }
  };

  BootSequence.init();
})();
</script>
```

## O listă de verificare a 12 design-uri de stiluri de pornire

Am grupat aceste 12 stiluri în șase categorii majore pentru a satisface scenarii și preferințe estetice diferite.

### A. Minimalism

> „Mai puțin este mai mult." Pentru scenariile care urmăresc viteza de încărcare ultimă, oferim cele mai ușoare opțiuni posibile.

#### 1. Punct minimalist
Un punct simplu stă în centrul ecranului, asociat cu o animație de respirație.
*   **Implementare**: `@keyframes` CSS controlează scara și opacitatea.
*   **Cel mai potrivit pentru**: Orice caz în care pagina trebuie să rămână absolut curată.

#### 2. Dezvăluire de brand
Folosind animația SVG `stroke-dasharray`, simulează o dezvăluire desenată manual a liniilor logo-ului HagiCode, urmată de o estompare a textului.
*   **Tehnică**: Animație de path SVG cu un aspect foarte rafinat.

### B. Imitație ecran schelet

> „Arta de a înșela ochiul." Simulând un layout UI real, utilizatorii simt că pagina este deja pe jumătate încărcată.

#### 3. Schelet chat sidebar
Aceasta poate fi cea mai practică opțiune. Am construit manual un layout în HTML care oglindește exact componentele React `Sidebar` și `ChatInput`, apoi am suprapus o animație shimmer gri.
*   **Valoare**: Când hidratarea React se completează, scheletul devine instantaneu componenta reală, iar utilizatorii abia pot percepe comutarea.

#### 4. Schelet carduri stivuite
Simulează mișcarea de stivuire a cardurilor de propunere în timpul încărcării, folosind transformări 3D pentru a face cardurile să plutească subtil.

### C. Abstracție și artă

> Arătați ADN-ul geek al HagiCode.

#### 5. Morfism geometric
O formă geometrică (un pătrat) este randată în centrul ecranului, apoi se transformă lin în timp într-un cerc, un triunghi și în final logo-ul.
*   **Tehnologie**: Tranziții fluide cu `border-radius` CSS.

#### 6. Ploaie de cod
Un omagiu adus *The Matrix*. Folosind fontul JetBrains Mono, fluxuri slabe de caractere cad în fundal.
*   **Notă**: Pentru performanță, fluxurile de caractere trebuie să rămână într-o zonă mai mică sau să folosească o rată de reîmprospătare mai mică.

#### 7. Puls neon
Un inel strălucitor în stil cyberpunk care folosește multiple straturi `box-shadow` pentru a crea o strălucire neon puternică.

### D. Branding și teme

> Faceți sistemul să se simtă viu.

#### 8. Temă sezonieră
Acesta este un încărcător dinamic. Verifică data curentă pentru sărbători precum Anul Nou Lunar sau Crăciunul și încarcă animația SVG corespunzătoare.
*   **Exemplu**: În timpul Anului Nou Lunar, felinare roșii se leagănă ușor în partea de jos a ecranului.

#### 9. Flux gradient
Fundalul folosește un gradient fluid bazat pe culorile de brand HagiCode. Combinat cu `background-size` și `background-position` animate, creează o senzație de mișcare asemănătoare aurorei.

### E. Aspect tehnic

> Un salut către dezvoltatori.

#### 10. Pornire terminal
Simulează ieșirea de consolă. Linii de cod defilează rapid:
```text
> Initializing HagiCode Core...
> Loading models...
> Connecting to neural network...
```
Acest lucru se simte familiar instantaneu pentru fiecare dezvoltator.

#### 11. Bară de progres
O bară de progres subțire apare în partea de sus a ecranului, cu un procent afișat în dreapta. Deși nu putem accesa progresul real de descărcare, putem folosi un cronometru pentru a simula un proces de încărcare „credibil": rapid pentru primele 80%, apoi treptat mai lent pentru ultimele 20%.

### F. Creativitate

#### 12. Asamblare pixeli
Aceasta este o idee foarte interesantă. Pătrate mici sunt împrăștiate pe ecran, apoi converg spre centru și se asamblează treptat în iconița logo-ului HagiCode. Simbolizează procesul de construire a codului.

## Bune practici și capcane

În lucrarea reală de dezvoltare HagiCode, am rezumat câteva detalii de implementare critic de importante.

### 1. CSS defensiv este obligatoriu
Nu fiți leneși și săriți peste prefix. Odată, am uitat să domeniem stilurile ecranului de pornire cu un ID, iar stilurile globale `div` după montarea React au afectat neașteptat ecranul de pornire, stricând layout-ul.
**Lecția învățată**: Puneți fiecare selector CSS sub `#boot-screen`, și folosiți `!important` pentru a crește prioritatea când este necesar, dar doar în interiorul CSS-ului ecranului de pornire.

### 2. Tranziții grațioase
După ce React se montează cu succes, nu faceți direct `remove()` pe DOM-ul ecranului de pornire.
**Abordarea corectă**:
1.  React declanșează `window.dispatchEvent(new Event('hagicode:ready'))`.
2.  Ecranul de pornire ascultă evenimentul și setează mai întâi `opacity: 0`.
3.  Așteptați 300ms, ceea ce corespunde duratei tranziției CSS, și apelați `.remove()` doar după ce ecranul este complet invizibil.

### 3. Sincronizarea variabilelor de temă
Valorile de culoare ale ecranului de pornire sunt hardcodate în `index.html`. Dacă schimbăm culoarea primară Tailwind, trebuie să actualizăm și ecranul de pornire.
**Optimizare**: Scrieți un plugin simplu în scriptul de construcție Vite pentru a citi `tailwind.config.js` și a injecta variabile de culoare în variabilele șablon `index.html`, creând o singură sursă de adevăr.

### 4. Preîncărcarea fonturilor
Ecranele de pornire au adesea nevoie să folosească un font de brand, dar dacă fontul se încarcă încet, poate apărea FOUT (Flash of Unstyled Text).
**Soluție**: Adăugați `<link rel="preload" href="/fonts/JetBrainsMono.woff2" as="font" type="font/woff2" crossorigin>` în interiorul `<head>`. Aceasta este o modalitate cu cost redus și randament ridicat de a îmbunătăți experiența.

### 5. Monitorizarea performanței
Am injectat `performance.mark('boot-start')` în partea de jos a `index.html`, și am marcat `boot-end` când React s-a montat cu succes.
**De ce contează**: Colectând aceste date prin Application Insights, putem măsura direct cât de mult scurtează ecranul de pornire timpul perceput de așteptare. Datele arată că un ecran schelet excelent poate îmbunătăți toleranța utilizatorilor pentru o „rețea lentă" cu peste 50%.

## Rezumat

Un ecran de pornire bun este mai mult decât „decorațiune în timp ce aștepți". Este semnalul de handshake în prima interacțiune între produs și utilizator. În proiectul HagiCode, acest sistem de pornire bazat pe **modelul de variante** ne permite să comutăm flexibil stilurile în sărbători și lansări, îmbunătățind considerabil simțul de distracție și profesionalism al produsului.
