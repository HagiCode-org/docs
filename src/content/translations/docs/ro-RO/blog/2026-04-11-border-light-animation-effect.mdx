## Ghid pentru crearea unui efect de animație cu lumină de bordură

> Cum construiești acel element important pe care utilizatorii îl observă dintr-o privire folosind doar CSS? Nu este chiar atât de greu. Trucul este doar să iei o cale puțin ocolită. În acest articol, vă voi ghida pas cu pas prin construirea unei animații cu lumină de bordură de la zero și vă voi împărtăși câteva dintre capcanele pe care le-am întâlnit în timpul construirii HagiCode.

## Context

Dacă vă ocupați de frontend, probabil că ați avut această experiență înainte: un manager de produs vine cu expresia „asta e simplu" și spune: „Putem adăuga un fel de efect special la acest task în execuție, astfel încât utilizatorii să-l poată identifica imediat?"

Spuneți sigur, putem schimba doar culoarea bordurii. Apoi ei clatină din cap cu acea privire care spune „Nu înțelegi." Răspund: „Nu e suficient de evident. Vreau genul de efect în care lumina aleargă de-a lungul bordurii, ca într-un film SF."

În acel moment s-ar putea să vă întrebați cum să îl construiți. Canvas? SVG? Sau se poate descurca CSS singur? La urma urmei, nimeni nu vrea să recunoască că nu știe.

În aplicațiile web moderne, animațiile cu lumină de bordură sunt de fapt foarte comune. Ele sunt folosite în principal în câteva scenarii precum:

- **Indicatori de stare**: marcarea taskurilor în execuție sau a elementelor active
- **Focus vizual**: evidențierea zonelor de conținut importante
- **Consolidarea brandului**: crearea unui stil vizual elegant, modern și orientat spre tehnologie
- **Teme sezoniere**: construirea unei atmosfere de sărbătoare pentru ocazii speciale

Ne-am confruntat exact cu această cerință în timpul construirii [HagiCode](https://hagicode.com). Utilizatorii trebuiau să poată vedea dintr-o privire ce sesiuni erau în execuție și ce propuneri erau procesate momentan. Am încercat mai multe abordări diferite. Unele căi au fost mai line, altele puțin mai sinuoase. În cele din urmă, ne-am stabilit pe o strategie de implementare destul de matură.

## Idei de implementare de bază

După analiza codebase-ului HagiCode, am rezumat mai jos câteva modele de implementare de bază. Fiecare se potrivește unui scenariu diferit, sau cu alte cuvinte, fiecare există dintr-un motiv.

### 1. Strălucire rotativă cu gradient conic (cel mai comun)

Acesta este modul clasic de a implementa un efect de animație cu lumină de bordură. Ideea de bază este să folosești CSS `conic-gradient` pentru a crea un gradient conic, apoi să-l rotești continuu. Ca un far care se rotește noaptea, pur și simplu continuă să se învârtă.

**Elemente cheie:**
- Folosiți pseudo-elementul `::before` pentru a crea stratul de strălucire
- Folosiți `conic-gradient` pentru a defini distribuția culorilor gradientului
- Folosiți pseudo-elementul `::after` pentru a masca zona centrală (opțional)
- Folosiți `@keyframes` pentru a implementa animația de rotație

### 2. Linie de strălucire laterală

Aceasta funcționează bine pentru indicatorii de stare din elementele de listă. Creați o linie subțire strălucitoare pe o parte a elementului în loc să animați întreaga bordură. Uneori puțină lumină este suficientă. Nu trebuie să iluminați toată lumea.

**Elemente cheie:**
- Un element de linie subțire poziționat absolut
- Folosiți `box-shadow` pentru a crea efectul de strălucire
- Folosiți `scale` și `opacity` pentru o animație de respirație

### 3. Fundal strălucitor cu box-shadow

Dacă nu aveți nevoie de efectul complet de mișcare și doriți doar o strălucire moale a fundalului, este suficient să suprapuneți mai multe valori `box-shadow`. Uneori opțiunea mai simplă este cea mai bună.

### 4. Suport pentru accesibilitate

Această parte este ușor de trecut cu vederea, dar este extrem de importantă. Fiecare animație ar trebui să ia în considerare media query-ul `prefers-reduced-motion` și să ofere o alternativă statică pentru utilizatorii care nu doresc animație. Nu toată lumea se bucură de mișcare constantă, iar respectarea acestei preferințe contează.

## Opțiuni de implementare

### Opțiunea 1: Bordură rotativă cu conic-gradient (recomandată)

Aceasta este implementarea cea mai completă a efectului de lumină de bordură în mișcare și este, de asemenea, opțiunea pe care o folosim cel mai des în HagiCode. La urma urmei, dacă ceva funcționează bine, de ce l-ați înlocui?

```css
/* Container părinte */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Strat de strălucire rotativă */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Strat de mascare (opțional, pentru crearea unui efect de bordură goală) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Principiul din spatele acestei opțiuni este destul de simplu: creați un pseudo-element mai mare decât containerul părinte, desenați un gradient conic pe el și rotiți-l continuu. Containerul părinte folosește `overflow: hidden`, astfel încât doar lumina care trece de-a lungul bordurii rămâne vizibilă. Este puțin ca și cum ați privi un far prin fereastră. Vedeți doar fâșia mică care trece pe lângă.

### Opțiunea 2: Bordură de lumină rotativă simplificată

Dacă nu aveți nevoie de efectul complet, HagiCode include și o versiune mai ușoară cu clase utilitare. Uneori abordarea mai simplă este într-adevăr mai bună.

```css
/* Clasă utilitară pentru bordură de lumină rotativă */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Suport pentru accesibilitate */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Observați `will-change: transform` aici. Acesta îi spune browserului: „Acest element va continua să se schimbe", astfel încât browserul poate pregăti câteva optimizări din timp și poate menține animația mai fluidă. Pregătirea din timp este de obicei mai bună decât agitatia din ultimul moment.

### Opțiunea 3: Linie de strălucire laterală

Aceasta este deosebit de potrivită pentru indicatorii de stare ai elementelor de listă și este exact ceea ce folosește lista de sesiuni HagiCode. O singură linie subțire poate ieși în evidență printre multe elemente. Asta seamănă cu o lecție de viață în sine.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Aceasta folosește `isolation: isolate` pentru a crea un context de stivuire nou, apoi se bazează pe `z-index` pentru a controla ordinea de afișare a fiecărui strat. `pointer-events: none` este, de asemenea, esențial. Altfel, pseudo-elementul ar bloca clicurile utilizatorilor. Unele lucruri pot arăta frumos, dar tot nu ar trebui să stea în cale.

### Opțiunea 4: Împachetați într-un component React

Dacă proiectul dvs. folosește React, puteți împacheta această logică într-un component, în special gestionarea accesibilității. Scrieți o dată, folosiți de multe ori. Acesta este întregul scop.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Modulul CSS corespunzător:

```css
/* GlowBorder.module.css */

/* Versiunea animată */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Versiunea statică (accesibilitate) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Hook-ul `useReducedMotion` din `framer-motion` detectează automat preferința de sistem a utilizatorului. Dacă utilizatorul a activat mișcarea redusă, acesta returnează `true`, iar componenta afișează versiunea statică. Respectarea preferinței utilizatorului contează mai mult decât impunerea unui efect spectaculos.

## Lecții practice învățate

Acestea sunt câteva dintre lecțiile pe care le-am învățat în timpul construirii HagiCode. Le-ați putea numi și cicatrici de luptă. Sperăm să vă ajute să evitați câteva ocoluri.

### 1. Sistem de variabile de temă

Variabilele CSS fac suportul pentru teme multiple deosebit de convenabil. Nimeni nu vrea să editeze o grămadă de cod de fiecare dată când se schimbă tema.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Utilizare */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

În acest fel, schimbarea temei necesită doar modificarea clasei pe elementul `html`, iar fiecare culoare de animație se actualizează automat. Un codebase, două stiluri. Acesta este exact ceea ce dorim.

### 2. Optimizarea performanței

**Folosiți `will-change` pentru a sugera browserului să optimizeze:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Spuneți browserului din timp și vă va ajuta să optimizați. Multe lucruri în viață merg mai bine cu puțină pregătire.

**Evitați utilizarea box-shadow-uri complexe pe elemente mari:**

```css
/* Nu este ideal - utilizarea unei umbre estompate pe un element mare */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Mai bine - folosiți un pseudo-element pentru a limita zona strălucitoare */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Am testat acest lucru în HagiCode. Adăugarea unei umbre estompate direct pe un card mare a scăzut rata de cadre la scroll sub 30fps. Trecerea la un pseudo-element a adus lucrurile înapoi la 60fps stabil. Utilizatorii pot simți cu siguranță acea diferență.

### 3. Accesibilitate

Cu adevărat nu ar trebui să săriți peste acest aspect. Unii utilizatori consideră animația amețitoare sau distractivă, iar respectarea preferințelor lor face parte din construirea unui produs bun. Lucrurile frumoase nu trebuie impuse tuturor.

**Media query CSS:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Oferă o alternativă statică */
    opacity: 1;
  }
}
```

**Detectați preferința utilizatorului în React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Conținut
    </div>
  );
};
```

### 4. Controlul nivelului de intensitate

Indicatorul de throughput Token din HagiCode afișează culori de strălucire diferite în funcție de throughput-ul în timp real, iar acest lucru este implementat dinamic. Stări diferite ar trebui exprimate diferit.

```tsx
const colors = [
  null,       // Nivelul 0 - fără culoare
  '#3b82f6',  // Nivelul 1 - Albastru
  '#34d399',  // Nivelul 2 - Smarald
  '#facc15',  // Nivelul 3 - Galben
  '#fbbf24',  // Nivelul 4 - Chihlimbar
  '#f97316',  // Nivelul 5 - Portocaliu
  '#22d3ee',  // Nivelul 6 - Cyan
  '#d946ef',  // Nivelul 7 - Fuchsia
  '#f43f5e',  // Nivelul 8 - Roz
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Lucruri de care să țineți cont

Există încă câteva detalii demne de atenție, deoarece până când descoperiți aceste probleme pe calea grea, este deja prea târziu.

| Lucruri de care să țineți cont | Explicație |
|---------|------|
| **Gestionarea z-index** | Stratul de strălucire ar trebui să folosească un `z-index` adecvat pentru a nu interfera cu interacțiunea conținutului |
| **pointer-events** | Pseudo-elementul de strălucire ar trebui să aibă `pointer-events: none` |
| **Overflow la limite** | Containerul părinte necesită `overflow: hidden` sau trebuie să ajustați dimensiunea pseudo-elementului |
| **Impact asupra performanței** | Animațiile complexe pot afecta performanța pe dispozitive mobile, deci testați cu atenție |
| **Mod întunecat** | Asigurați-vă că culoarea de strălucire rămâne clar vizibilă pe fundaluri întunecate |
| **Comutarea temei** | Folosiți variabile CSS pentru ca culorile animației să se actualizeze corect la schimbarea temei |

### 6. Sfaturi de depanare

Pseudo-elementele pot fi puțin greu de localizat în instrumentele de dezvoltare, deci puteți adăuga temporar o bordură pentru a verifica poziția.

```css
/* Afișați temporar limitele pseudo-elementului pentru depanare */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

După ce terminați poziționarea, nu uitați să comentați sau să eliminați acea linie. Altfel, producția poate deveni destul de stânjenitoare. Unele lucruri sunt mai bine lăsate în dezvoltare.
