Dacă doriți să instalați Hermes și să începeți să îl folosiți, calea cea mai scurtă este de fapt doar trei pași:

1. Rulați comanda oficială de instalare
2. Porniți CLI-ul în terminal cu `hermes`
3. Dacă doriți să continuați să îl folosiți în Feishu, atunci configurați `hermes gateway setup`

Acest articol nu încearcă să explice fiecare capabilitate Hermes dintr-o dată. În schimb, vă ajută să completați mai întâi bucla de început cea mai importantă: instalați-l, faceți-l să funcționeze, începeți să îl folosiți și apoi conectați-l la unul dintre cele mai comune scenarii de platforme de mesagerie.

## Ce este Hermes și pentru cine este

Hermes Agent este un agent AI pe care îl puteți folosi fie dintr-un terminal local, fie printr-un gateway de platformă de mesagerie.

Pentru majoritatea dezvoltatorilor, are două puncte de intrare comune:

- **CLI**: Tastați `hermes` în terminal pentru a intra direct în interfața interactivă.
- **Gateway de mesagerie**: Rulați `hermes gateway`, apoi discutați cu el de pe platforme precum Feishu, Telegram, Discord și Slack.

Dacă obiectivul dvs. acum este pur și simplu să începeți rapid, nu inversați ordinea. Începeți cu această cale:

- Instalați Hermes mai întâi
- Verificați că funcționează din CLI mai întâi
- Apoi decideți dacă doriți să conectați o platformă de mesagerie

Acest lucru face problemele mai ușor de diagnosticat și este mai potrivit pentru persoanele care folosesc Hermes pentru prima dată.

## Ce trebuie să știți înainte de a instala Hermes

Conform README-ului Hermes, calea de instalare rapidă oficială suportă aceste medii:

- Linux
- macOS
- WSL2
- Android prin Termux

### Note pentru utilizatorii Windows

Hermes nu suportă în prezent rularea direct pe Windows nativ. Dacă folosiți Windows, abordarea recomandată este să instalați mai întâi WSL2 și apoi să rulați comanda de instalare în interiorul WSL2.

Este recomandat să clarificați acest lucru de la început, deoarece multe eșecuri de instalare nu sunt cauzate de comanda în sine, ci de utilizarea unui mediu de execuție nesuportat.

## Cum se instalează Hermes

Comanda de instalare rapidă furnizată în README-ul Hermes este:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Această comandă rulează scriptul oficial de instalare și gestionează pașii de inițializare specifici platformei.

### Reîncărcați shell-ul după instalare

Odată ce instalarea se termină, reîncărcați mai întâi mediul shell. Cea mai comună comandă este:

```bash
source ~/.bashrc
```

Dacă folosiți zsh, puteți folosi:

```bash
source ~/.zshrc
```

### Cum să confirmați că Hermes este instalat corect

Cea mai directă modalitate de a verifica este să rulați:

```bash
hermes
```

Dacă doriți o confirmare suplimentară că propria configurație și dependențe funcționează, puteți rula și:

```bash
hermes doctor
```

`hermes doctor` este deosebit de util în aceste situații:

- Comanda se comportă anormal după instalare
- Configurarea modelului eșuează
- Gateway-ul nu pornește
- Nu sunteți sigur dacă dependențele de mediu sunt complete

## Cum să începeți să folosiți Hermes pentru prima dată

### Începeți cu CLI-ul

Dacă doriți doar să confirmați cât mai repede posibil că Hermes funcționează, cea mai simplă metodă este:

```bash
hermes
```

Acest lucru lansează CLI-ul interactiv Hermes. Pentru utilizatorii care folosesc Hermes pentru prima dată, este, de asemenea, cel mai recomandat punct de plecare, deoarece puteți verifica mai întâi cele mai esențiale lucruri:

- Dacă comanda este de fapt disponibilă
- Dacă configurația curentă a modelului funcționează corect
- Dacă toolchain-ul terminalului funcționează corect
- Dacă stilul de interacțiune corespunde cu ceea ce aveți nevoie

### Aceste comenzi sunt suficiente pentru prima rundă de configurare

README-ul Hermes listează mai multe comenzi de înaltă frecvență, iar împreună ele formează o cale practică de prima utilizare:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Dacă nu sunteți sigur ce face fiecare, rețineți-le astfel:

- `hermes model`: alegeți sau schimbați modele
- `hermes tools`: vizualizați și configurați instrumentele disponibile curent
- `hermes config set`: modificați elemente specifice de configurare
- `hermes setup`: rulați wizard-ul complet de inițializare o dată
- `hermes update`: actualizați Hermes
- `hermes doctor`: depanați probleme

Pentru începători, ordinea cea mai practică este de obicei:

1. Rulați `hermes model` mai întâi
2. Dacă doriți să configurați toate opțiunile comune deodată, atunci rulați `hermes setup`

## Cele două moduri cele mai comune de a folosi Hermes

### 1. Folosiți Hermes în terminal ca asistent de dezvoltare zilnic

Modul CLI se potrivește bine acestor scenarii:

- Puneți întrebări direct în timp ce scrieți cod local
- Inspectați proiecte, editați fișiere și rulați comenzi
- Efectuați depanare punctuală sau lucrări de revizuire
- Colaborați continuu în directorul de lucru curent

Cel mai mare avantaj al său este că este calea cea mai scurtă: fără integrare suplimentară de platformă, fără configurare de bot de gestionat din start și este cel mai bun mod de a vă construi primul set de obiceiuri de utilizare.

### 2. Folosiți Hermes printr-o platformă de mesagerie

Dacă doriți să discutați cu Hermes pe platforme precum Feishu, Telegram sau Discord, trebuie să folosiți gateway-ul de mesagerie.

Cele mai comune comenzi de intrare sunt:

```bash
hermes gateway setup
hermes gateway
```

Mai exact:

- `hermes gateway setup` este folosit pentru configurarea interactivă a platformei
- `hermes gateway` este folosit pentru a porni procesul gateway

Conform documentației oficiale, gateway-ul este un proces de fundal unificat care conectează platformele configurate, gestionează sesiunile și gestionează funcții precum sarcinile cron.

## Folosind Feishu ca exemplu: cum să conectați Hermes la o platformă de mesagerie

Dacă majoritatea muncii dvs. zilnice are loc în Feishu, atunci Feishu/Lark este o modalitate foarte naturală de a folosi Hermes.

### Calea minimă viabilă de integrare

Documentația oficială recomandă această comandă de intrare pentru Feishu/Lark:

```bash
hermes gateway setup
```

După ce rulați, pur și simplu alegeți **Feishu / Lark** în wizard.

Documentația Feishu descrie două moduri de conexiune:

- `websocket`: recomandat
- `webhook`: opțional

Dacă Hermes rulează pe laptop-ul, stația dvs. de lucru sau serverul privat, folosirea `websocket` mai întâi este de obicei mai simplă deoarece nu aveți nevoie să expuneți un URL de callback public.

### Dacă configurați manual, cunoașteți cel puțin aceste variabile

Dacă nu folosiți wizard-ul și scrieți configurația manual, documentația Feishu listează aceste variabile de bază:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Două dintre ele merită atenție specială:

- `FEISHU_ALLOWED_USERS`: recomandat, astfel încât nu toți cei care pot accesa botul îl pot folosi direct
- `FEISHU_HOME_CHANNEL`: vă permite să predefiniți un chat de acasă pentru a primi rezultate cron sau notificări implicite

### De ce Hermes uneori nu răspunde în chat-urile de grup Feishu

Acest detaliu este ușor de ratat: în chat-urile de grup Feishu, Hermes nu răspunde la fiecare mesaj în mod implicit.

Documentația oficială afirmă clar:

- În mesaje directe, Hermes răspunde la mesaje
- În chat-uri de grup, trebuie să `@` menționați explicit botul înainte ca acesta să proceseze mesajul

Dacă doriți să setați o conversație Feishu ca canal de acasă, puteți folosi și asta în chat:

```text
/set-home
```

Sau definiți-o în configurație din timp:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Comenzile Hermes pe care începătorii ar trebui să le rețină mai întâi

Indiferent dacă folosiți Hermes în CLI sau pe o platformă de mesagerie, reținerea următoarelor comenzi este deja suficientă pentru a începe:

- `/new` sau `/reset`: începeți o sesiune nouă
- `/model`: vizualizați sau schimbați modelul
- `/retry`: reîncercați tura anterioară
- `/undo`: anulați interacțiunea anterioară
- `/compress`: comprimați manual contextul
- `/help`: vizualizați ajutorul

Dacă folosiți în principal Hermes pe o platformă de mesagerie, mai rețineți una:

- `/sethome` sau `/set-home`: setați chat-ul curent ca canal de acasă

Aceste comenzi acoperă cele mai comune operațiuni din stadiul de început: repornire, ajustare, revenire, verificare și continuare.

## Întrebări frecvente

### Poate Windows instala Hermes direct?

Nu. Documentația oficială curentă afirmă clar că Windows nativ nu este suportat și recomandă WSL2.

### Ce trebuie să fac dacă tastarea `hermes` nu face nimic după instalare?

Este recomandat să depanați în această ordine:

1. Reîncărcați shell-ul mai întâi, de exemplu cu `source ~/.bashrc`
2. Rulați `hermes` din nou
3. Dacă este încă anormal, rulați `hermes doctor`

### De ce bot-ul nu răspunde într-un grup Feishu?

Verificați aceste trei lucruri mai întâi:

- Dacă ați `@` menționat Hermes în grup
- Dacă `FEISHU_ALLOWED_USERS` restricționează utilizatorul curent
- Dacă politica de chat de grup curentă permite gestionarea mesajelor de grup

Conform documentației oficiale Feishu, utilizarea explicită a unei mențiuni @ este necesară în scenariile de chat de grup.

## Pașii următori recomandați

Dacă doriți pur și simplu să începeți să folosiți Hermes cât mai repede posibil, aceasta este ordinea cea mai recomandată:

1. Rulați comanda de instalare mai întâi
2. Începeți cu `hermes` în CLI-ul local mai întâi
3. Folosiți `hermes model` și `hermes setup` pentru a completa configurarea de bază
4. Dacă doriți să continuați să îl folosiți în Feishu, atunci configurați `hermes gateway setup`
