# Instalați OpenSpec

OpenSpec gestionează propunerile, specificațiile și înregistrările modificărilor. Această pagină urmărește un singur obiectiv: explicați clar calea de instalare, astfel încât să o puteți atașa fluxului de lucru existent.

## Ce este OpenSpec?

OpenSpec este un instrument de linie de comandă pentru gestionarea propunerilor și specificațiilor în munca software. Fluxul de lucru principal include:

- **Gestionarea propunerilor**: creați, examinați și urmăriți propunerile de funcții
- **Validarea specificațiilor**: Asigurați-vă că modificările codului sunt conforme cu specificațiile predefinite
- **Urmărirea modificărilor**: înregistrați tot istoricul modificărilor examinate
- **Colaborare în echipă**: sprijină dezvoltarea colaborativă cu mai multe persoane

### Aplicație în proiecte HagiCode

În proiectele HagiCode, OpenSpec este utilizat pentru:

1. **Gestionarea dezvoltării funcțiilor**: Toate funcțiile noi trebuie să treacă prin procesul de propunere OpenSpec
2. **Evaluarea codului**: Asigurați-vă că modificările respectă specificațiile proiectului
3. **Sincronizarea documentației**: mențineți coerența între documentație și implementarea codului
4. **Integrarea controlului versiunilor**: integrare perfectă cu fluxurile de lucru pentru controlul versiunilor

## Cerințe preliminare

Înainte de a instala OpenSpec, asigurați-vă că sistemul dvs. îndeplinește următoarele cerințe:

### Node.js și npm

OpenSpec este distribuit prin npm și necesită un mediu Node.js. OpenSpec necesită **Node.js 18.0 sau o versiune ulterioară**.

Verificați instalarea Node.js:

```bash
node --version
```

Verificați instalarea npm:

```bash
npm --version
```

Dacă nu este instalată sau versiunea este prea veche, vă rugăm să consultați instrucțiunile de instalare a software-ului aferente din [Ghid de instalare](/en/installation).

## Instalarea OpenSpec

:::sfat[Cerințe versiuni]
Acest proiect necesită utilizarea gamei de versiuni **OpenSpec 1.0.0 sau mai mare**. Toate versiunile 1.x sunt acceptate pentru a asigura compatibilitatea cu fluxul de lucru al proiectului.
:::

OpenSpec este instalat la nivel global prin npm, făcându-l disponibil de oriunde pe sistemul dumneavoastră.

### Instalați global folosind npm

<Tabs>
  <TabItem value="win" label="Windows">
    Rulați în PowerShell sau în linia de comandă:

```powershell
npm install -g @fission-ai/openspec@1
```

:::notă[Probleme de permisiuni Windows]
Dacă întâmpinați erori de permisiune în timpul instalării, rulați PowerShell sau Command Prompt ca administrator.
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    Rulați în terminal:

```bash
npm install -g @fission-ai/openspec@1
```

:::notă[Probleme de permisiuni macOS]
Daca intalnesti `EACCES` erori de permisiune, încercați:
```bash
sudo npm install -g @fission-ai/openspec@1
```
Sau folosiți [nvm](https://github.com/nvm-sh/nvm) pentru a gestiona versiunile Node.js și pentru a evita problemele de permisiuni.
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    Rulați în terminal:

```bash
npm install -g @fission-ai/openspec@1
```

:::notă[Probleme de permisiuni Linux]
Daca intalnesti `EACCES` erori de permisiune, puteți:
1. Utilizare `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. Configurați npm pentru a utiliza directorul de utilizatori:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### Verificați instalarea

După instalare, verificați dacă OpenSpec este instalat corect și că versiunea îndeplinește cerințele:

```bash
openspec --version
```

Dacă este instalat cu succes, veți vedea un număr de versiune, de exemplu:

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::notă[Interval de versiuni]
Asigurați-vă că versiunea OpenSpec instalată este 1.0.0 sau mai mare. Proiectul a verificat compatibilitatea cu versiunea 1.1.1. Este recomandat să utilizați cea mai recentă versiune compatibilă pentru a obține funcții noi.
:::

## Probleme comune și depanare

### Erori de permisiuni

#### Emisiune: `EACCES` Acces refuzat

Întâmpinați erori de permisiune la instalarea sau rularea OpenSpec.

**Soluții**:

<Tabs>
  <TabItem value="win" label="Windows">
    Rulați PowerShell sau Command Prompt ca administrator:
    1. Faceți clic dreapta pe PowerShell sau pe Command Prompt
    2. Selectați „Rulați ca administrator”
    3. Reluați comanda de instalare
  </TabItem>
  <TabItem value="mac" label="macOS">
    Instalați folosind `sudo`:

```bash
sudo npm install -g @fission-ai/openspec@1
```

Sau folosiți [nvm](https://github.com/nvm-sh/nvm) pentru a gestiona Node.js și a evita problemele globale de permisiuni de instalare.
  </TabItem>
  <TabItem value="linux" label="Linux">
    Soluția 1: Folosiți sudo
```bash
sudo npm install -g @fission-ai/openspec@1
```

Soluția 2: Configurați npm pentru a utiliza directorul de utilizatori
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @fission-ai/openspec@1
```
  </TabItem>
</Tabs>

### Probleme de rețea

#### Problemă: instalarea npm eșuează sau este foarte lentă

Nu se poate descărca OpenSpec din registry npm din motive de rețea.

**Soluții**:

1. **Utilizați oglindă npm domestică**:

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **Restabiliți sursa oficială** (dacă este necesar):

```bash
npm config set registry https://registry.npmjs.org
```

3. **Utilizați cnpm** (opțional):

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

:::sfat
Utilizatorilor din China continentală li se recomandă să folosească oglinda Taobao, care poate îmbunătăți semnificativ viteza de descărcare.
:::

### Probleme de compatibilitate cu versiunile

#### Problemă: Comenzile OpenSpec rulează anormal sau raportează erori

Versiunea OpenSpec instalată este incompatibilă cu mediul actual.

**Soluții**:

1. **Verificați versiunea Node.js**:

```bash
node --version
```

Asigurați-vă că versiunea este 18.0 sau mai mare. Dacă versiunea este prea scăzută, actualizați Node.js.

2. **Verificați versiunea OpenSpec**:

```bash
openspec --version
```

3. **Reinstalați versiunea specifică**:

```bash
npm install -g @fission-ai/openspec@1
```

4. **Ștergeți memoria cache npm** (dacă problema persistă):

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### Comanda nu a fost găsită

#### Problemă: tastarea `openspec` Comanda arată „Comanda nu a fost găsită”

OpenSpec este instalat, dar sistemul nu poate găsi comanda.

**Soluții**:

<Tabs>
  <TabItem value="win" label="Windows">
    1. Confirmați calea globală de instalare npm:

```powershell
npm config get prefix
```

2. Adăugați calea către sistem PATH:
   - Deschideți „Proprietăți sistem” → „Avansat” → „Variabile de mediu”
   - Găsi `Path` în „Variabile de sistem”
   - Adăugați calea globală npm (de obicei `C:\Users\<username>\AppData\Roaming\npm`)
   - Reporniți fereastra de linie de comandă
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. Confirmați calea globală de instalare npm:

```bash
npm config get prefix
```

2. Adăugați calea către PATH (în `~/.zshrc` sau `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Reîncărcați configurația:

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. Confirmați calea globală de instalare npm:

```bash
npm config get prefix
```

2. Adăugați calea către PATH (în `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Reîncărcați configurația:

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
