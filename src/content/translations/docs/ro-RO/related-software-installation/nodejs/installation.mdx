Acest ghid acoperă instalarea Node.js pe Windows, macOS și Linux. Node.js este o dependență cheie pentru instrumentele locale HagiCode, inclusiv OpenSpec, instrumentele de front-end și câteva CLI pentru dezvoltatori.

:::sfat[Cerințe versiuni]
HagiCode recomandă **Node.js 24+** (Node.js minim acceptat &gt;= 18.0). În cele mai multe cazuri, instalați cea mai recentă versiune LTS și apoi verificați dacă versiunea dvs. îndeplinește cerințele.
:::

## Instalarea Node.js

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Metoda unu: Instalator oficial (recomandat)

1. Vizitați [Site-ul oficial Node.js](https://nodejs.org/)
2. Descărcați **LTS (Long Term Support)** versiunea de instalare Windows (`.msi` fişier)
3. Faceți dublu clic pentru a rula programul de instalare
4. Urmați instrucțiunile asistentului de instalare pentru a finaliza instalarea:
   - Acceptați acordul de licență
   - Alegeți calea de instalare (implicit este bine)
   - Asigurați-vă că este bifat „Instalați automat instrumentele necesare”.
   - Faceți clic pe „Instalare” pentru a începe instalarea
5. După finalizarea instalării, faceți clic pe „Finish”

### Metoda a doua: Folosind winget

Deschideți PowerShell sau Command Prompt și rulați:

```powershell
winget install OpenJS.NodeJS.LTS
```

După instalare, redeschideți terminalul pentru ca modificările să intre în vigoare.

  </TabItem>
  <TabItem value="macos" label="macOS">

### Metoda unu: Instalator oficial (recomandat)

1. Vizitați [Site-ul oficial Node.js](https://nodejs.org/)
2. Descărcați programul de instalare macOS versiunea **LTS (Asistență pe termen lung)** (`.pkg` fişier)
3. Faceți dublu clic pentru a rula programul de instalare
4. Urmați instrucțiunile asistentului de instalare pentru a finaliza instalarea:
   - Faceți clic pe „Continuați”
   - Acceptați acordul de licență
   - Alegeți discul de instalare
   - Faceți clic pe „Instalare” și introduceți parola de administrator
5. După finalizarea instalării, faceți clic pe „Închidere”

### Metoda a doua: Folosirea Homebrew

Dacă ai [Homebrew](https://brew.sh/) instalat, puteți instala Node.js prin terminal:

```bash
brew install node
```

:::sfat
Homebrew va adăuga automat Node.js la PATH. Dacă comanda nu este disponibilă, reporniți terminalul sau rulați:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

  </TabItem>
  <TabItem value="ubuntu" label="Ubuntu/Debian">

### Utilizarea depozitului NodeSource (recomandat)

Acesta este cel mai bun mod de a obține cea mai recentă versiune Node.js:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Folosind apt Package Manager

Dacă nu aveți nevoie de cea mai recentă versiune, puteți utiliza depozitul implicit Ubuntu:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::nota
Versiunea Node.js din depozitul implicit poate fi mai veche. După instalare, verificați versiunea pentru a vă asigura că îndeplinește cerințele Node.js &gt;= 18.0 (se recomandă Node.js 24+).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### Utilizarea depozitului NodeSource (recomandat)

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### Folosind yum Package Manager

```bash
sudo yum install -y nodejs npm
```

  </TabItem>
  <TabItem value="fedora" label="Fedora">

### Folosind dnf Package Manager

```bash
sudo dnf install -y nodejs npm
```

:::sfat
Depozitul software al Fedora conține de obicei versiuni mai noi Node.js și poate fi instalat direct prin dnf.
:::

  </TabItem>
</Tabs>

## Utilizarea instrumentelor de gestionare a versiunilor

Dacă trebuie să comutați între mai multe versiuni Node.js, puteți utiliza instrumente de gestionare a versiunilor.

### nvm (Manager de versiune a nodului)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

După instalare, reîncărcați configurația shell-ului:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

Pe Windows, vă recomandăm să utilizați [nvm-windows](https://github.com/coreybutler/nvm-windows):

1. Descărcați cel mai recent `nvm-setup.exe` din [versiuni nvm-windows](https://github.com/coreybutler/nvm-windows/releases)
2. Rulați programul de instalare
3. Redeschideți promptul de comandă sau PowerShell

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

După instalarea nvm, îl puteți folosi pentru a instala și gestiona versiunile Node.js:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (Manager rapid de noduri)

fnm este un instrument de gestionare a versiunilor Node.js mai rapid și mai simplu, construit cu Rust.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

Utilizare [chiuretă](https://scoop.sh/) sau [aripă](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Accelerație pentru utilizatorii din China continentală

Dacă vă aflați în China continentală, descărcarea pachetelor Node.js și npm poate fi lentă. Iată câteva soluții de accelerare.

### Utilizarea surselor oglinzi domestice

#### npm Taobao Mirror (recomandat)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### Utilizarea nrm pentru a gestiona sursele oglindă

nrm (managerul de registru npm) vă ajută să comutați rapid sursele de oglindă npm:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Accelerarea descărcărilor programului de instalare Node.js

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao oferă descărcări în oglindă pentru instalatorii Node.js:

- **Adresa de descărcare**: https://npmmirror.com/mirrors/node/

Vizitați adresa de mai sus și selectați versiunea pe care trebuie să o descărcați.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

Dacă descărcarea nvm sau a altor instrumente din GitHub este lentă, puteți utiliza oglinda FastGit:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::nota
Serviciul FastGit se poate modifica, vă rugăm să consultați informațiile oficiale.
:::

  </TabItem>
</Tabs>

### Oglinzi domestice Homebrew

Dacă utilizați Homebrew în China continentală, puteți configura oglinzile interne pentru accelerare:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Manageri de pachete specifici Chinei continentale

#### cnpm

cnpm este un client npm furnizat de echipa Taobao care utilizează implicit oglinda Taobao:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::sfat
cnpm este în mare parte compatibil cu comenzile npm și poate fi folosit ca înlocuitor direct.
:::

### Verificarea efectului de accelerare a oglinzii

După configurarea oglinzilor, verificați prin:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Verificarea instalării

După instalare, verificați dacă Node.js și npm sunt instalate corect:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::nota
Asigurați-vă că versiunea Node.js &gt;= 18.0 (se recomandă Node.js 24+). Dacă versiunea este prea scăzută, reinstalați folosind metodele de mai sus sau utilizați instrumente de gestionare a versiunilor pentru a instala versiunea corectă.
:::

### Ieșire de verificare a probei

Ieșirea de verificare cu succes ar trebui să arate astfel:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Depanare

### Probleme Windows

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Dacă `node` sau `npm` comenzile nu sunt disponibile după instalare:

1. Reporniți promptul de comandă sau PowerShell
2. Dacă tot nu este disponibil, adăugați manual Node.js la PATH:
   - Căutați „Variabile de mediu” și deschideți „Editați variabilele de mediu ale sistemului”
   - Faceți clic pe „Variabile de mediu”
   - Găsiți „Cale” în „Variabile de sistem” și faceți clic pe „Editare”
   - Adăugați calea de instalare Node.js (implicit: `C:\Program Files\nodejs\`)
   - Faceți clic pe „OK” pentru a salva modificările

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Dacă întâmpinați erori de politică de execuție când utilizați nvm:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### Probleme macOS

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Dacă întâmpinați erori de permisiune la instalarea pachetelor globale npm:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Dacă comenzile nu sunt disponibile după instalare, reporniți terminalul sau adăugați manual la PATH (de obicei în `/usr/local/bin`).

  </TabItem>
</Tabs>

### Probleme Linux

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Dacă întâmpinați erori de permisiune la instalarea pachetelor globale npm, este recomandat să utilizați instrumentele de gestionare a versiunilor Node.js (cum ar fi nvm sau fnm) în loc de managerii de pachete de sistem.

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Dacă versiunea Node.js instalată prin managerul de pachete de sistem este prea veche:

1. Utilizați depozitul NodeSource (recomandat, vezi mai sus)
2. Sau utilizați instrumente de gestionare a versiunilor (nvm/fnm)

  </TabItem>
</Tabs>

### Verificarea Mediului

Dacă verificarea eșuează, verificați următoarele:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
