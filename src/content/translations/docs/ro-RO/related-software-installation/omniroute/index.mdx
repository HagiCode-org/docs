Această pagină este pentru utilizatorii HagiCode pentru prima dată care doresc cea mai scurtă cale de configurare OmniRoute: porniți serviciul local, verificați că `http://localhost:20128/v1` funcționează și confirmați că atât Tabloul de bord, cât și API-ul sunt disponibile.

:::sfat[Cale recomandată]
Această primă versiune păstrează mai întâi calea cea mai stabilă: `npm install -g omniroute`, `omniroute`, Tabloul de bord la `http://localhost:20128`, și adresa URL de bază compatibilă cu OpenAI la `http://localhost:20128/v1`. Docker, VPS și modurile de implementare mai avansate rămân în secțiunea opțională.
:::

## Cerințe preliminare

- Node.js este deja instalat. Complet [Instalarea Node.js](/en/related-software-installation/nodejs/installation) mai întâi dacă este nevoie.
- Port local `20128` este disponibil și nu este deja ocupat.
- Aveți cel puțin un cont de furnizor de model în amonte pe care intenționați să îl direcționați prin OmniRoute.
- Înainte de a integra HagiCode, un CLI sau un SDK, păstrați aceste două adrese separate:
  - Bord: `http://localhost:20128`
  - Adresa URL de bază compatibilă cu OpenAI: `http://localhost:20128/v1`

## Instalați și rulați local

Începeți cu calea globală recomandată npm:

```bash
npm install -g omniroute
omniroute
```

După pornirea serviciului, validați-l în această ordine:

1. Deschide `http://localhost:20128` într-un browser și confirmați că tabloul de bord se încarcă.
2. Du-te la `Endpoints` din Tabloul de bord și creați sau copiați o cheie API.
3. Rețineți adresa URL de bază pe care o veți transmite instrumentelor mai târziu: `http://localhost:20128/v1`.

:::notă[De ce să începem cu tabloul de bord]
Pentru prima configurare, tabloul de bord este cea mai rapidă modalitate de a confirma că OmniRoute rulează, furnizorii dvs. sunt conectați și că există o cheie API. Dacă un apel ulterioară a clientului eșuează, verificarea mai întâi a tabloului de bord este de obicei mai rapidă decât depanarea imediată a codului SDK.
:::

## Validați serviciul care rulează

### Validarea 1: confirmați că catalogul modelului este lizibil

Pune cheia din `Endpoints` pagină într-o variabilă de mediu:

```bash
export OMNIROUTE_API_KEY="replace-with-the-key-you-copied-from-the-Endpoints-page"
```

Apoi solicitați `/v1/models`:

```bash
curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}"
```

Dacă primiți înapoi o listă de modele, punctul final și fluxul de autentificare compatibil cu OpenAI funcționează.

### Validarea 2: trimiteți o cerere minimă de chat

```bash
curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "replace-with-a-model-id-from-v1-models",
    "messages": [
      { "role": "user", "content": "Reply with ok." }
    ]
  }'
```

Scopul aici nu este calitatea promptă. Este pentru a demonstra că modelul ales de dvs. poate fi deja rutat prin OmniRoute cu succes.
