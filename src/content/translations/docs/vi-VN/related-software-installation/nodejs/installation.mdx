Hướng dẫn này bao gồm cài đặt Node.js trên Windows, macOS và Linux. Node.js là phần phụ thuộc chính của công cụ cục bộ HagiCode, bao gồm OpenSpec, công cụ giao diện người dùng và một số CLI của nhà phát triển.

:::tip[Yêu cầu phiên bản]
HagiCode khuyến nghị **Node.js 24+** (Node.js được hỗ trợ tối thiểu >= 18.0). Trong hầu hết các trường hợp, hãy cài đặt phiên bản LTS mới nhất rồi xác minh rằng phiên bản của bạn đáp ứng yêu cầu.
:::

## Cài đặt Node.js

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Phương pháp thứ nhất: Trình cài đặt chính thức (Được khuyến nghị)

1. Ghé thăm [Trang web chính thức của Node.js](https://nodejs.org/)
2. Tải xuống phiên bản **LTS (Hỗ trợ dài hạn)** Trình cài đặt Windows (`.msi` tập tin)
3. Bấm đúp để chạy trình cài đặt
4. Làm theo lời nhắc của trình hướng dẫn cài đặt để hoàn tất cài đặt:
   - Chấp nhận thỏa thuận cấp phép
   - Chọn đường dẫn cài đặt (mặc định là được)
   - Đảm bảo đã chọn "Tự động cài đặt các công cụ cần thiết"
   - Nhấp vào "Cài đặt" để bắt đầu cài đặt
5. Sau khi cài đặt hoàn tất, nhấp vào "Hoàn tất"

### Phương pháp thứ hai: Sử dụng winget

Mở PowerShell hoặc Dấu nhắc lệnh và chạy:

```powershell
winget install OpenJS.NodeJS.LTS
```

Sau khi cài đặt, mở lại terminal để các thay đổi có hiệu lực.

  </TabItem>
  <TabItem value="macos" label="macOS">

### Phương pháp thứ nhất: Trình cài đặt chính thức (Được khuyến nghị)

1. Ghé thăm [Trang web chính thức của Node.js](https://nodejs.org/)
2. Tải xuống trình cài đặt macOS phiên bản **LTS (Hỗ trợ dài hạn)** (`.pkg` tập tin)
3. Bấm đúp để chạy trình cài đặt
4. Làm theo lời nhắc của trình hướng dẫn cài đặt để hoàn tất cài đặt:
   - Nhấp vào "Tiếp tục"
   - Chấp nhận thỏa thuận cấp phép
   - Chọn đĩa cài đặt
   - Nhấp vào "Cài đặt" và nhập mật khẩu quản trị viên
5. Sau khi cài đặt hoàn tất, nhấp vào "Đóng"

### Phương pháp thứ hai: Sử dụng Homebrew

Nếu bạn có [Homebrew](https://brew.sh/) đã cài đặt, bạn có thể cài đặt Node.js qua terminal:

```bash
brew install node
```

:::mẹo
Homebrew sẽ tự động thêm Node.js vào PATH của bạn. Nếu lệnh không có sẵn, hãy khởi động lại thiết bị đầu cuối hoặc chạy:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

  </TabItem>
  <TabItem value="ubuntu" label="Ubuntu/Debian">

### Sử dụng kho lưu trữ NodeSource (Được khuyến nghị)

Đây là cách tốt nhất để có phiên bản Node.js mới nhất:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Sử dụng Trình quản lý gói apt

Nếu không cần phiên bản mới nhất, bạn có thể sử dụng kho lưu trữ Ubuntu mặc định:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::lưu ý
Phiên bản Node.js trong kho lưu trữ mặc định có thể cũ hơn. Sau khi cài đặt, hãy kiểm tra phiên bản để đảm bảo đáp ứng các yêu cầu của Node.js >= 18.0 (khuyên dùng Node.js 24+).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### Sử dụng kho lưu trữ NodeSource (Được khuyến nghị)

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### Sử dụng Trình quản lý gói yum

```bash
sudo yum install -y nodejs npm
```

  </TabItem>
  <TabItem value="fedora" label="Fedora">

### Sử dụng Trình quản lý gói dnf

```bash
sudo dnf install -y nodejs npm
```

:::mẹo
Kho phần mềm của Fedora thường chứa các phiên bản Node.js mới hơn và có thể cài đặt trực tiếp qua dnf.
:::

  </TabItem>
</Tabs>

## Sử dụng công cụ quản lý phiên bản

Nếu cần chuyển đổi giữa nhiều phiên bản Node.js, bạn có thể sử dụng các công cụ quản lý phiên bản.

### nvm (Trình quản lý phiên bản nút)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

Sau khi cài đặt, tải lại cấu hình shell của bạn:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### cửa sổ

Trên Windows, chúng tôi khuyên bạn nên sử dụng [nvm-windows](https://github.com/coreybutler/nvm-windows):

1. Tải về mới nhất `nvm-setup.exe` từ [bản phát hành nvm-window](https://github.com/coreybutler/nvm-windows/releases)
2. Chạy trình cài đặt
3. Mở lại dấu nhắc lệnh hoặc PowerShell

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

Sau khi cài đặt nvm, bạn có thể sử dụng nó để cài đặt và quản lý các phiên bản Node.js:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (Fast Node Manager)

fnm là công cụ quản lý phiên bản Node.js nhanh hơn và đơn giản hơn được xây dựng bằng Rust.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### cửa sổ

sử dụng [muỗng](https://scoop.sh/) hoặc [cánh nhỏ](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Tăng tốc cho người dùng Trung Quốc đại lục

Nếu bạn ở Trung Quốc đại lục, việc tải xuống các gói Node.js và npm có thể bị chậm. Dưới đây là một số giải pháp tăng tốc.

### Sử dụng nguồn gương trong nước

#### npm Gương Taobao (Khuyến nghị)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### Sử dụng nrm để quản lý nguồn nhân bản

nrm (trình quản lý đăng ký npm) giúp bạn nhanh chóng chuyển đổi nguồn nhân bản npm:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Tăng tốc tải xuống trình cài đặt Node.js

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao cung cấp bản tải xuống nhân bản cho trình cài đặt Node.js:

- **Địa chỉ tải xuống**: https://npmmirror.com/mirrors/node/

Truy cập địa chỉ trên và chọn phiên bản bạn cần tải về.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

Nếu quá trình tải xuống nvm hoặc các công cụ khác từ GitHub chậm, bạn có thể sử dụng FastGit mirror:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::lưu ý
Dịch vụ FastGit có thể thay đổi, vui lòng tham khảo thông tin chính thức.
:::

  </TabItem>
</Tabs>

### Gương Homebrew trong nước

Nếu bạn sử dụng Homebrew ở Trung Quốc đại lục, bạn có thể định cấu hình máy nhân bản trong nước để tăng tốc:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Người quản lý gói hàng cụ thể ở Trung Quốc đại lục

#### cnpm

cnpm là một ứng dụng khách npm do nhóm Taobao cung cấp, sử dụng máy nhân bản taobao theo mặc định:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::mẹo
cnpm hầu hết tương thích với các lệnh npm và có thể được sử dụng để thay thế trực tiếp.
:::

### Xác minh hiệu ứng tăng tốc gương

Sau khi thiết lập gương, hãy xác minh bằng cách:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Xác minh cài đặt

Sau khi cài đặt, hãy xác minh rằng Node.js và npm đã được cài đặt chính xác:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::lưu ý
Đảm bảo phiên bản Node.js >= 18.0 (Khuyến nghị Node.js 24+). Nếu phiên bản quá thấp, vui lòng cài đặt lại bằng các phương pháp trên hoặc sử dụng các công cụ quản lý phiên bản để cài đặt đúng phiên bản.
:::

### Đầu ra xác minh mẫu

Đầu ra xác minh thành công sẽ giống như sau:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Khắc phục sự cố

### Sự cố về Windows

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Nếu `node` hoặc `npm` lệnh không có sẵn sau khi cài đặt:

1. Khởi động lại dấu nhắc lệnh hoặc PowerShell
2. Nếu vẫn không có sẵn, hãy thêm Node.js vào PATH theo cách thủ công:
   - Tìm kiếm "Biến môi trường" và mở "Chỉnh sửa biến môi trường hệ thống"
   - Nhấp vào "Biến môi trường"
   - Tìm "Đường dẫn" trong "Biến hệ thống" và nhấp vào "Chỉnh sửa"
   - Thêm đường dẫn cài đặt Node.js (mặc định: `C:\Program Files\nodejs\`)
   - Nhấp vào "OK" để lưu thay đổi

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Nếu bạn gặp lỗi chính sách thực thi khi sử dụng nvm:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### Sự cố về macOS

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Nếu bạn gặp lỗi về quyền khi cài đặt gói npm toàn cầu:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Nếu không có lệnh sau khi cài đặt, hãy khởi động lại thiết bị đầu cuối hoặc thêm thủ công vào PATH (thường là trong `/usr/local/bin`).

  </TabItem>
</Tabs>

### Các vấn đề về Linux

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Nếu bạn gặp phải lỗi về quyền khi cài đặt các gói npm toàn cầu, bạn nên sử dụng các công cụ quản lý phiên bản Node.js (như nvm hoặc fnm) thay vì trình quản lý gói hệ thống.

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Nếu phiên bản Node.js được cài đặt qua trình quản lý gói hệ thống quá cũ:

1. Sử dụng kho lưu trữ NodeSource (được khuyến nghị, xem ở trên)
2. Hoặc sử dụng các công cụ quản lý phiên bản (nvm/fnm)

  </TabItem>
</Tabs>

### Kiểm tra môi trường

Nếu xác minh không thành công, hãy kiểm tra những điều sau:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
