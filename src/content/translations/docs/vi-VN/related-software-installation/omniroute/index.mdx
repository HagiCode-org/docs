Trang này dành cho người dùng HagiCode lần đầu muốn có đường dẫn thiết lập OmniRoute ngắn nhất: khởi động dịch vụ cục bộ, xác minh rằng `http://localhost:20128/v1` hoạt động và xác nhận rằng cả Trang tổng quan và API đều khả dụng.

:::tip[Đường dẫn được đề xuất]
Phiên bản đầu tiên này giữ đường dẫn ổn định nhất trước tiên: `npm install -g omniroute`, `omniroute`, Bảng điều khiển tại `http://localhost:20128`và URL cơ sở tương thích với OpenAI tại `http://localhost:20128/v1`. Docker, VPS và các chế độ triển khai nâng cao hơn vẫn nằm trong phần tùy chọn.
:::

## Điều kiện tiên quyết

- Node.js đã được cài đặt. Hoàn thành [Cài đặt Node.js](/en/related-software-installation/nodejs/installation) đầu tiên nếu cần thiết.
- Cảng địa phương `20128` có sẵn và chưa được sử dụng.
- Bạn có ít nhất một tài khoản nhà cung cấp mô hình ngược tuyến mà bạn định định tuyến thông qua OmniRoute.
- Trước khi tích hợp HagiCode, CLI hoặc SDK, hãy tách riêng hai địa chỉ này:
  - Bảng điều khiển: `http://localhost:20128`
  - URL cơ sở tương thích với OpenAI: `http://localhost:20128/v1`

## Cài đặt và chạy cục bộ

Bắt đầu với đường dẫn npm toàn cầu được đề xuất:

```bash
npm install -g omniroute
omniroute
```

Sau khi dịch vụ bắt đầu, hãy xác thực nó theo thứ tự sau:

1. Mở `http://localhost:20128` trong trình duyệt và xác nhận rằng Trang tổng quan đã tải.
2. đi đến `Endpoints` trong Bảng điều khiển và tạo hoặc sao chép khóa API.
3. Lưu ý URL cơ sở mà bạn sẽ sử dụng sau này: `http://localhost:20128/v1`.

:::note[Tại sao nên bắt đầu với Trang tổng quan]
Đối với lần thiết lập đầu tiên, Bảng điều khiển là cách nhanh nhất để xác nhận rằng OmniRoute đang chạy, nhà cung cấp của bạn đã được kết nối và tồn tại khóa API. Nếu cuộc gọi sau này của khách hàng không thành công thì việc kiểm tra Trang tổng quan trước thường nhanh hơn việc gỡ lỗi mã SDK ngay lập tức.
:::

## Xác thực dịch vụ đang chạy

### Xác thực 1: xác nhận danh mục mô hình có thể đọc được

Đặt chìa khóa từ `Endpoints` trang thành một biến môi trường:

```bash
export OMNIROUTE_API_KEY="replace-with-the-key-you-copied-from-the-Endpoints-page"
```

Sau đó yêu cầu `/v1/models`:

```bash
curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}"
```

Nếu bạn lấy lại danh sách mô hình thì điểm cuối và quy trình xác thực tương thích với OpenAI đang hoạt động.

### Xác thực 2: gửi yêu cầu trò chuyện tối thiểu

```bash
curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "replace-with-a-model-id-from-v1-models",
    "messages": [
      { "role": "user", "content": "Reply with ok." }
    ]
  }'
```

Mục tiêu ở đây không phải là chất lượng nhanh chóng. Điều này nhằm chứng minh rằng mô hình bạn chọn có thể được định tuyến thành công thông qua OmniRoute.
