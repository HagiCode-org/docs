# Cài đặt OpenSpec

OpenSpec quản lý các đề xuất, thông số kỹ thuật và hồ sơ thay đổi. Trang này hướng tới một mục tiêu: giải thích đường dẫn cài đặt rõ ràng để bạn có thể đính kèm nó vào quy trình làm việc hiện tại của mình.

## OpenSpec là gì?

OpenSpec là một công cụ dòng lệnh để quản lý các đề xuất và thông số kỹ thuật trong công việc phần mềm. Quy trình làm việc cốt lõi bao gồm:

- **Quản lý đề xuất**: Tạo, đánh giá và theo dõi các đề xuất tính năng
- **Xác thực thông số kỹ thuật**: Đảm bảo các thay đổi mã tuân thủ các thông số kỹ thuật được xác định trước
- **Theo dõi thay đổi**: Ghi lại tất cả lịch sử thay đổi đã xem xét
- **Hợp tác nhóm**: Hỗ trợ phát triển hợp tác nhiều người

### Ứng dụng trong dự án HagiCode

Trong các dự án HagiCode, OpenSpec được sử dụng để:

1. **Quản lý phát triển tính năng**: Tất cả các tính năng mới phải trải qua quy trình đề xuất OpenSpec
2. **Đánh giá mã**: Đảm bảo các thay đổi tuân thủ thông số kỹ thuật của dự án
3. **Đồng bộ hóa tài liệu**: Duy trì tính nhất quán giữa tài liệu và triển khai mã
4. **Tích hợp kiểm soát phiên bản**: Tích hợp liền mạch với quy trình kiểm soát phiên bản

## Điều kiện tiên quyết

Trước khi cài đặt OpenSpec, hãy đảm bảo hệ thống của bạn đáp ứng các yêu cầu sau:

### Node.js và npm

OpenSpec được phân phối qua npm và yêu cầu môi trường Node.js. OpenSpec yêu cầu **Node.js 18.0 trở lên**.

Xác minh cài đặt Node.js:

```bash
node --version
```

Xác minh cài đặt npm:

```bash
npm --version
```

Nếu chưa được cài đặt hoặc phiên bản quá cũ, vui lòng tham khảo hướng dẫn cài đặt phần mềm liên quan trong phần [Hướng dẫn cài đặt](/en/installation).

## Cài đặt OpenSpec

:::tip[Yêu cầu phiên bản]
Dự án này yêu cầu sử dụng phạm vi phiên bản **OpenSpec 1.0.0 trở lên**. Tất cả các phiên bản 1.x đều được hỗ trợ để đảm bảo khả năng tương thích với quy trình làm việc của dự án.
:::

OpenSpec được cài đặt trên toàn cầu thông qua npm, giúp nó có sẵn ở mọi nơi trên hệ thống của bạn.

### Cài đặt toàn cầu bằng npm

<Tabs>
  <TabItem value="win" label="Windows">
    Chạy trong PowerShell hoặc Dấu nhắc lệnh:

```powershell
npm install -g @fission-ai/openspec@1
```

:::note[Vấn đề về quyền của Windows]
Nếu bạn gặp lỗi về quyền trong khi cài đặt, hãy chạy PowerShell hoặc Dấu nhắc lệnh với tư cách quản trị viên.
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    Chạy trong Terminal:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[Vấn đề về quyền của macOS]
Nếu bạn gặp phải `EACCES` lỗi cấp phép, hãy thử:
```bash
sudo npm install -g @fission-ai/openspec@1
```
Hoặc sử dụng [nvm](https://github.com/nvm-sh/nvm) để quản lý các phiên bản Node.js và tránh các vấn đề về quyền.
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    Chạy trong Terminal:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[Vấn đề về quyền của Linux]
Nếu bạn gặp phải `EACCES` lỗi về quyền, bạn có thể:
1. sử dụng `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. Định cấu hình npm để sử dụng thư mục người dùng:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### Xác minh cài đặt

Sau khi cài đặt, hãy xác minh rằng OpenSpec đã được cài đặt chính xác và phiên bản đáp ứng yêu cầu:

```bash
openspec --version
```

Nếu cài đặt thành công, bạn sẽ thấy đầu ra số phiên bản, ví dụ:

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::note[Phạm vi phiên bản]
Hãy đảm bảo phiên bản OpenSpec đã cài đặt là 1.0.0 trở lên. Dự án đã được xác minh khả năng tương thích với phiên bản 1.1.1. Nên sử dụng phiên bản tương thích mới nhất để có được các tính năng mới.
:::

## Các vấn đề thường gặp và khắc phục sự cố

### Lỗi cấp phép

#### Vấn đề: `EACCES` Quyền bị từ chối

Gặp lỗi cấp phép khi cài đặt hoặc chạy OpenSpec.

**Giải pháp**:

<Tabs>
  <TabItem value="win" label="Windows">
    Chạy PowerShell hoặc Dấu nhắc lệnh với tư cách quản trị viên:
    1. Bấm chuột phải vào PowerShell hoặc Dấu nhắc lệnh
    2. Chọn "Chạy với tư cách quản trị viên"
    3. Chạy lại lệnh cài đặt
  </TabItem>
  <TabItem value="mac" label="macOS">
    Cài đặt bằng cách sử dụng `sudo`:

```bash
sudo npm install -g @fission-ai/openspec@1
```

Hoặc sử dụng [nvm](https://github.com/nvm-sh/nvm) để quản lý Node.js và tránh các vấn đề về quyền cài đặt chung.
  </TabItem>
  <TabItem value="linux" label="Linux">
    Giải pháp 1: Sử dụng sudo
```bash
sudo npm install -g @fission-ai/openspec@1
```

Giải pháp 2: Cấu hình npm để sử dụng thư mục người dùng
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @fission-ai/openspec@1
```
  </TabItem>
</Tabs>

### Sự cố mạng

#### Sự cố: Cài đặt npm không thành công hoặc rất chậm

Không thể tải xuống OpenSpec từ sổ đăng ký npm vì lý do mạng.

**Giải pháp**:

1. **Sử dụng Gương npm trong nước**:

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **Khôi phục nguồn chính thức** (nếu cần):

```bash
npm config set registry https://registry.npmjs.org
```

3. **Sử dụng cnpm** (tùy chọn):

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

:::mẹo
Người dùng ở Trung Quốc đại lục được khuyến khích sử dụng nhân bản Taobao, điều này có thể cải thiện đáng kể tốc độ tải xuống.
:::

### Vấn đề tương thích phiên bản

#### Sự cố: Lệnh OpenSpec chạy bất thường hoặc báo lỗi

Phiên bản OpenSpec đã cài đặt không tương thích với môi trường hiện tại.

**Giải pháp**:

1. **Kiểm tra phiên bản Node.js**:

```bash
node --version
```

Đảm bảo phiên bản là 18.0 trở lên. Nếu phiên bản quá thấp, vui lòng nâng cấp Node.js.

2. **Kiểm tra phiên bản OpenSpec**:

```bash
openspec --version
```

3. **Cài đặt lại phiên bản cụ thể**:

```bash
npm install -g @fission-ai/openspec@1
```

4. **Xóa bộ nhớ cache npm** (nếu sự cố vẫn tiếp diễn):

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### Lệnh không tìm thấy

#### Vấn đề: Đánh máy `openspec` Lệnh hiển thị "Lệnh không tìm thấy"

OpenSpec đã được cài đặt nhưng hệ thống không thể tìm thấy lệnh.

**Giải pháp**:

<Tabs>
  <TabItem value="win" label="Windows">
    1. Xác nhận đường dẫn cài đặt toàn cầu npm:

```powershell
npm config get prefix
```

2. Thêm đường dẫn vào hệ thống PATH:
   - Mở "Thuộc tính hệ thống" → "Nâng cao" → "Biến môi trường"
   - Tìm `Path` trong "Biến hệ thống"
   - Thêm đường dẫn toàn cầu npm (thường `C:\Users\<username>\AppData\Roaming\npm`)
   - Khởi động lại cửa sổ dòng lệnh
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. Xác nhận đường dẫn cài đặt toàn cầu npm:

```bash
npm config get prefix
```

2. Thêm đường dẫn đến PATH (trong `~/.zshrc` hoặc `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Tải lại cấu hình:

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. Xác nhận đường dẫn cài đặt toàn cầu npm:

```bash
npm config get prefix
```

2. Thêm đường dẫn đến PATH (trong `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Tải lại cấu hình:

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
