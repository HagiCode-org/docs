## Điều kiện tiên quyết

Trước khi bắt đầu, hãy đảm bảo hệ thống của bạn đã cài đặt Docker và Docker Compose.

### Cài đặt Docker

#### cửa sổ

Tải xuống và cài đặt [Máy tính để bàn Docker cho Windows](https://www.docker.com/products/docker-desktop/)

Sau khi cài đặt, đảm bảo Docker Desktop đang chạy.

#### macOS

Tải xuống và cài đặt [Máy tính để bàn Docker cho Mac](https://www.docker.com/products/docker-desktop/)

Sau khi cài đặt, đảm bảo Docker Desktop đang chạy.

#### Linux

Cài đặt Docker bằng trình quản lý gói của bạn:

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

Sau khi cài đặt, khởi động dịch vụ Docker:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Xác minh cài đặt

Sau khi cài đặt, hãy chạy các lệnh sau để xác minh Docker và Docker Compose đã được cài đặt chính xác:

```bash
docker --version
docker compose version
```

## Bắt đầu nhanh

### 1. Tạo tệp cấu hình Docker Compose

:::tip[Sử dụng Trình tạo cấu hình]
**Khuyến khích sử dụng [Docker Compose Builder (trang web độc lập)](https://builder.hagicode.com/)** để tạo tập tin cấu hình của bạn.

Máy phát điện hỗ trợ:
- Nhiều tùy chọn nhà cung cấp API (Zhipu AI, Anthropic Official, API tùy chỉnh)
- Tự động cấu hình URL API và mã thông báo
- Cổng tùy chỉnh, tên vùng chứa và cấu hình cơ bản khác
- Định cấu hình dữ liệu liên tục và đường dẫn thư mục làm việc
- Tự động cấu hình sự khác biệt của nền tảng Windows/Linux
- Xử lý thông minh các vấn đề về quyền truy cập tệp
- **Lựa chọn nguồn hình ảnh**: Dựa trên môi trường mạng của bạn, hãy chọn nguồn hình ảnh thích hợp

[🚀 Sử dụng máy phát điện ngay →](https://builder.hagicode.com/)
:::

1. Mở [Docker Compose Builder (trang web độc lập)](https://builder.hagicode.com/)
2. Điền cấu hình theo nhu cầu của bạn:
   - Chọn nhà cung cấp API (Zhipu AI, Anthropic Official hoặc API tùy chỉnh)
   - Định cấu hình cổng, cơ sở dữ liệu, thư mục làm việc và các tùy chọn khác
   - **Chọn nguồn hình ảnh**: Chọn nguồn hình ảnh phù hợp dựa trên môi trường mạng của bạn
3. Nhấp vào nút tạo để nhận `docker-compose.yml` cấu hình
4. Lưu cấu hình đã tạo dưới dạng `docker-compose.yml` tập tin
5. Nếu cần, hãy tạo một `.env` tập tin để cấu hình thông tin nhạy cảm

#### Lựa chọn nguồn hình ảnh

Trình tạo hỗ trợ ba tùy chọn nguồn hình ảnh:

**Trung tâm Docker**
- **Địa chỉ hình ảnh**: `newbe36524/hagicode:{tag}`
- **Các tình huống áp dụng**: Thích hợp cho người dùng có khả năng tăng tốc nhân bản Docker Hub
- **Ưu điểm**: Nguồn ảnh chính thức, cập nhật kịp thời, truy cập ổn định
- **Ghi chú**: Có thể cần phải định cấu hình khả năng tăng tốc gương ở một số vùng

**Đăng ký vùng chứa Azure (ACR)**
- **Địa chỉ hình ảnh**: `hagicode.azurecr.io/hagicode:{tag}`
- **Các tình huống áp dụng**: Thích hợp cho người dùng không thể truy cập Docker Hub cục bộ
- **Ưu điểm**: Cung cấp nguồn hình ảnh thay thế, giải quyết các vấn đề truy cập mạng
- **Ghi chú**: Hình ảnh luôn được đồng bộ hóa với Docker Hub nhưng có thể có độ trễ ngắn

**Đăng ký vùng chứa đám mây của Alibaba (ACR)**
- **Địa chỉ hình ảnh**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **Các kịch bản áp dụng**: Phù hợp với người dùng gia đình, truy cập mạng ổn định và nhanh chóng
- **Ưu điểm**: Nút nội địa của Alibaba Cloud, tốc độ truy cập nhanh, độ ổn định cao
- **Ghi chú**: Hình ảnh ACR của Alibaba luôn được đồng bộ hóa với Docker Hub, thường trong vòng 30 phút

:::tip[Đề xuất lựa chọn nguồn hình ảnh]
- **Người dùng trong nước khuyên dùng Alibaba Cloud ACR**: Nếu bạn là người dùng trong nước, nên ưu tiên sử dụng nguồn hình ảnh Alibaba Cloud ACR để truy cập nhanh và ổn định
- **Trung tâm Docker thay thế**: Nếu môi trường mạng của bạn có thể truy cập Docker Hub, bạn cũng có thể chọn Docker Hub làm nguồn hình ảnh
- **Azure ACR thay thế**: Chỉ chọn nguồn hình ảnh Azure ACR nếu các nguồn hình ảnh khác gặp khó khăn khi truy cập
- **Đồng bộ hóa hình ảnh**: Hình ảnh Alibaba Cloud ACR được đồng bộ hóa với Docker Hub, thường trong vòng 30 phút
- **Tính nhất quán của phiên bản**: Thẻ hình ảnh nhất quán trên cả ba nguồn hình ảnh, không cần phải lo lắng về vấn đề phiên bản
:::

:::note[Nhận mã thông báo API]

**Zhipu AI (Được khuyến nghị)**: [Nhận mã thông báo API →](https://www.bigmodel.cn/glm-coding?ic=14BY54APZA)

### 2. Bắt đầu dịch vụ

Thực hiện lệnh sau trong thư mục chứa `docker-compose.yml` nằm ở:

```bash
docker compose up -d
```

Lệnh này sẽ:
- Tải xuống và khởi động các vùng chứa Hagicode được xác định bởi mẫu đã tạo
- Tạo và cấu hình mạng
- Khởi tạo thư mục dữ liệu mặc định và các phụ thuộc thời gian chạy

### 3. Xác minh trạng thái dịch vụ

Kiểm tra xem container có đang chạy không:

```bash
docker compose ps
```

Bạn sẽ thấy các dịch vụ được xác định trong mẫu đã tạo của mình ở trạng thái "đang chạy".

### Xem nhật ký dịch vụ

Để xem tất cả nhật ký dịch vụ:

```bash
# View all service logs
docker compose logs

# View Hagicode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## Hướng dẫn cấu hình

Nếu bạn cần sửa đổi cấu hình (chẳng hạn như thay đổi cổng, chuyển đổi nhà cung cấp API, v.v.), bạn có thể:

1. Tạo lại cấu hình mới bằng cách sử dụng [Docker Compose Builder (trang web độc lập)](https://builder.hagicode.com/)
2. Chỉnh sửa thủ công `docker-compose.yml` và `.env` tập tin
3. Khởi động lại dịch vụ để cấu hình có hiệu lực:

```bash
docker compose restart
```

## Ứng dụng truy cập

### Giao diện web

Đường dẫn Docker Compose được hỗ trợ sử dụng các tệp dữ liệu SQLite với khối lượng cố định theo mặc định. Nếu bạn cần kiểm tra các điểm gắn kết hoặc lập kế hoạch sao lưu, hãy xem lại ánh xạ ổ đĩa và thư mục dữ liệu trong tệp được tạo. `docker-compose.yml`.
