## Tự động đồng bộ hóa hình ảnh từ Docker Hub sang Azure ACR

> Bài viết này giải thích cách sử dụng GitHub Actions và công cụ đồng bộ hóa hình ảnh để tự động đồng bộ hóa hình ảnh từ Docker Hub sang Azure Container Register, giải quyết tình trạng truy cập chậm vào Docker Hub ở Trung Quốc và một số khu vực Azure, đồng thời cải thiện tính khả dụng của hình ảnh và hiệu quả triển khai trong môi trường Azure.

## Bối cảnh / Giới thiệu

Dự án HagiCode sử dụng hình ảnh Docker làm thành phần thời gian chạy cốt lõi, với các hình ảnh chính được lưu trữ trên Docker Hub. Khi dự án phát triển và nhu cầu triển khai Azure tăng lên, chúng tôi gặp phải những điểm khó khăn sau:

- Tốc độ kéo hình ảnh chậm, với quyền truy cập Docker Hub bị hạn chế ở Trung Quốc và một số khu vực Azure

- Azure Container Register cung cấp hiệu suất và tích hợp mạng tốt hơn khi chạy trong Azure

Để giải quyết những vấn đề này, chúng tôi cần một cơ chế đồng bộ hóa hình ảnh tự động để thường xuyên đồng bộ hóa hình ảnh từ Docker Hub sang Azure ACR, đảm bảo rằng người dùng có thể có được tốc độ kéo hình ảnh nhanh hơn và tính khả dụng cao hơn trong môi trường Azure.

## Triển khai quy trình làm việc hành động GitHub

Dưới đây là cấu hình quy trình làm việc thực tế đang được sử dụng (`.github/workflows/sync-docker-acr.yml`):

```yaml
name: Sync Docker Image to Azure ACR

on:
  schedule:
    - cron: "0 0 * * *" # Every day at 00:00 UTC
  workflow_dispatch: # Manual trigger
  push:
    branches: [publish]

permissions:
  contents: read

jobs:
  sync:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Download image-syncer
        run: |
          # Download the image-syncer binary
          wget https://github.com/AliyunContainerService/image-syncer/releases/download/v1.5.5/image-syncer-v1.5.5-linux-amd64.tar.gz
          tar -zxvf image-syncer-v1.5.5-linux-amd64.tar.gz
          chmod +x image-syncer

      - name: Create auth config
        run: |
          # Generate the authentication config file (YAML format)
          cat > auth.yaml <<EOF
          hagicode.azurecr.io:
            username: "${{ secrets.AZURE_ACR_USERNAME }}"
            password: "${{ secrets.AZURE_ACR_PASSWORD }}"
          EOF

      - name: Create images config
        run: |
          # Generate the image sync config file (YAML format)
          cat > images.yaml <<EOF
          docker.io/newbe36524/hagicode: hagicode.azurecr.io/hagicode
          EOF

      - name: Run image-syncer
        run: |
          # Run synchronization (using the newer --auth and --images parameters)
          ./image-syncer --auth=./auth.yaml --images=./images.yaml --proc=10 --retries=3

      - name: Upload logs
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: sync-logs
          path: image-syncer-*.log
          retention-days: 7
```

## Chi tiết cấu hình

### 1. Điều kiện kích hoạt
- Trình kích hoạt theo lịch trình: `cron: "0 0 * * *"` chạy hàng ngày lúc 00:00 UTC
- Kích hoạt thủ công: `workflow_dispatch` cho phép người dùng chạy thủ công từ giao diện người dùng GitHub
- Đẩy cò: `push: branches: [publish]` kích hoạt khi đẩy tới nhánh phát hành để đồng bộ hóa nhanh chóng

### 2. Cấu hình xác thực (`auth.yaml`)
```yaml
hagicode.azurecr.io:
  username: "${{ secrets.AZURE_ACR_USERNAME }}"
  password: "${{ secrets.AZURE_ACR_PASSWORD }}"
```

### 3. Cấu hình đồng bộ hóa hình ảnh
```yaml
docker.io/newbe36524/hagicode: hagicode.azurecr.io/hagicode
```
Cấu hình này có nghĩa là tất cả các thẻ từ `docker.io/newbe36524/hagicode` sẽ được đồng bộ hóa để `hagicode.azurecr.io/hagicode`.

### 4. thông số đồng bộ hóa hình ảnh
- `--auth=./auth.yaml`: đường dẫn tới file cấu hình xác thực
- `--images=./images.yaml`: đường dẫn tới file cấu hình đồng bộ ảnh
- `--proc=10`: sử dụng `10` công nhân kiêm nhiệm
- `--retries=3`: thử lại thất bại `3` lần

## Ghi chú và thực tiễn tốt nhất

### 1. Khuyến nghị bảo mật
- Luân chuyển mật khẩu Azure ACR thường xuyên (được khuyến nghị 90 ngày một lần)
- Sử dụng tài khoản dịch vụ ACR chuyên dụng với quyền chỉ đẩy
- Giám sát nhật ký truy cập ACR để phát hiện kịp thời các truy cập bất thường
- Không xuất thông tin đăng nhập vào nhật ký
- Không cam kết thông tin đăng nhập vào kho lưu trữ mã

### 2. Tối ưu hóa hiệu suất
- Điều chỉnh `--proc` tham số: điều chỉnh đồng thời dựa trên băng thông mạng (được khuyến nghị `5-20`)
- Giám sát thời lượng đồng bộ hóa: nếu quá trình đồng bộ hóa mất quá nhiều thời gian, hãy xem xét giảm mức độ đồng thời
- Dọn dẹp nhật ký thường xuyên: đặt mức hợp lý `retention-days` giá trị (`7` hiện tại)

### 3. Khắc phục sự cố

#### Vấn đề 1: Lỗi xác thực
```
Error: failed to authenticate to hagicode.azurecr.io
```
Giải pháp:
1. Kiểm tra xem Bí mật GitHub có được định cấu hình chính xác không
2. Xác minh xem mật khẩu Azure ACR đã hết hạn chưa
3. Xác nhận rằng quyền của tài khoản dịch vụ ACR là chính xác

#### Vấn đề 2: Hết thời gian chờ mạng
```
Error: timeout waiting for response
```
Giải pháp:
1. Kiểm tra kết nối mạng
2. Giảm số lượng công nhân làm việc đồng thời (các `--proc` tham số)
3. Đợi mạng phục hồi và chạy lại quy trình làm việc

#### Vấn đề 3: Đồng bộ hóa hình ảnh chưa hoàn chỉnh
```
Warning: some tags failed to sync
```
Giải pháp:
1. Kiểm tra nhật ký đồng bộ hóa để xác định các thẻ bị lỗi
2. Chạy lại quy trình làm việc theo cách thủ công để đồng bộ hóa lại
3. Xác minh rằng hình ảnh nguồn trên Docker Hub vẫn ổn

## Câu hỏi và giải pháp thường gặp

### Câu hỏi 1: Làm cách nào để đồng bộ hóa các thẻ cụ thể thay vì tất cả các thẻ?

Sửa đổi `images.yaml` tập tin cấu hình:
```yaml
# Sync only the latest and v1.0 tags
docker.io/newbe36524/hagicode:latest: hagicode.azurecr.io/hagicode:latest
docker.io/newbe36524/hagicode:v1.0: hagicode.azurecr.io/hagicode:v1.0
```

### Câu hỏi 2: Làm cách nào để đồng bộ hóa nhiều kho lưu trữ hình ảnh?

Thêm nhiều dòng vào `images.yaml`:
```yaml
docker.io/newbe36524/hagicode: hagicode.azurecr.io/hagicode
docker.io/newbe36524/another-image: hagicode.azurecr.io/another-image
```

### Câu hỏi 3: Làm cách nào để thử lại sau khi đồng bộ hóa không thành công?
- Tự động thử lại: trình đồng bộ hóa hình ảnh bao gồm cơ chế thử lại tích hợp (`3` thử lại theo mặc định)
- Thử lại thủ công: nhấp vào `Re-run all jobs` trên trang Hành động GitHub

### Q4: Làm cách nào tôi có thể xem tiến trình đồng bộ hóa chi tiết?
- Xem nhật ký thời gian thực trên trang Hành động
- Tải xuống `sync-logs` tạo tác để kiểm tra các tệp nhật ký đầy đủ
- Các tệp nhật ký bao gồm trạng thái đồng bộ hóa và tốc độ truyền cho mỗi thẻ

### Câu hỏi 5: Quá trình đồng bộ hóa mất bao lâu?
- Đồng bộ hóa hoàn toàn ban đầu: thường `10-30` phút tùy thuộc vào kích thước hình ảnh
- Đồng bộ hóa gia tăng: thường `2-5` phút nếu thay đổi hình ảnh nhỏ
- Thời gian phụ thuộc vào băng thông mạng, kích thước hình ảnh và cài đặt đồng thời

## Tiện ích mở rộng được đề xuất

### 1. Thêm thông báo đồng bộ hóa
Thêm bước thông báo vào quy trình làm việc:
```yaml
- name: Notify on success
  if: success()
  run: |
    echo "Docker images synced successfully to Azure ACR"
```

### 2. Thực hiện lọc thẻ hình ảnh
Thêm logic lọc thẻ vào quy trình làm việc:
```yaml
- name: Filter tags
  run: |
    # Sync only tags prefixed with v
    echo "docker.io/newbe36524/hagicode:v* : hagicode.azurecr.io/hagicode:v*" > images.yaml
```

### 3. Thêm báo cáo thống kê đồng bộ hóa
```yaml
- name: Generate report
  if: always()
  run: |
    echo "## Sync Report" >> $GITHUB_STEP_SUMMARY
    echo "- Total tags: $(grep -c 'synced' image-syncer-*.log)" >> $GITHUB_STEP_SUMMARY
    echo "- Sync time: ${{ steps.sync.outputs.duration }}" >> $GITHUB_STEP_SUMMARY
```
