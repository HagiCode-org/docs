## Thiết kế kiến trúc

### Tổng quan về kiến trúc lớp

Việc tích hợp giữa C# và CodeBuddy sử dụng kiến trúc phân lớp rõ ràng. Thiết kế này làm cho trách nhiệm trở nên rõ ràng và giúp việc bảo trì dài hạn trở nên dễ dàng hơn nhiều:

```
┌─────────────────────────────────────────────┐
│           Provider Contract Layer           │
│   AIProviderType enum + extension methods   │
├─────────────────────────────────────────────┤
│           Provider Factory Layer            │
│   AIProviderFactory dependency injection factory │
├─────────────────────────────────────────────┤
│           Provider Implementation Layer     │
│   CodebuddyCliProvider concrete implementation │
├─────────────────────────────────────────────┤
│           ACP Infrastructure Layer          │
│  ACPSessionManager / StdioAcpTransport      │
│  AcpRpcClient / AcpAgentClient              │
└─────────────────────────────────────────────┘
```

Lợi ích của việc phân lớp này là gì? Nói một cách đơn giản, mỗi lớp nằm cách xa các lớp khác. Nếu sau này chúng ta muốn thay đổi cơ chế giao tiếp, ví dụ từ `stdio` sang WebSocket, chúng ta chỉ cần sửa đổi lớp dưới cùng và logic nghiệp vụ phía trên nó vẫn không bị ảnh hưởng. Không ai muốn một sự thay đổi về giao tiếp ảnh hưởng đến toàn bộ cơ sở mã.

### Phân tích thành phần cốt lõi

**Lớp hợp đồng nhà cung cấp** là nền tảng của toàn bộ kiến trúc. Chúng tôi xác định `AIProviderType` enum, ở đâu `CodebuddyCli = 3` được sử dụng làm giá trị enum và triển khai ánh xạ hai chiều giữa chuỗi và enum thông qua các phương thức mở rộng. Điều đó cho phép các chuỗi trong tệp cấu hình được chuyển đổi một cách thuận tiện thành enum và các enum được chuyển đổi trở lại thành chuỗi để gỡ lỗi đầu ra.

**Lớp nhà máy của Nhà cung cấp** chịu trách nhiệm tạo phiên bản Nhà cung cấp tương ứng dựa trên cấu hình. Nó sử dụng tính năng tiêm phụ thuộc .NET cùng với `ActivatorUtilities.CreateInstance` để sáng tạo năng động. Ưu điểm của Factory Pattern là khi thêm Nhà cung cấp mới, bạn chỉ cần thêm logic tạo thay vì sửa đổi mã hiện có.

**Lớp triển khai của Nhà cung cấp** là nơi công việc thực tế diễn ra. `CodebuddyCliProvider` thực hiện `IAIProvider` giao diện và cung cấp hai chế độ gọi: `ExecuteAsync` cho các cuộc gọi không trực tuyến và `StreamAsync` để truyền phát cuộc gọi.

**Lớp cơ sở hạ tầng ACP** cung cấp nền tảng giao tiếp bên dưới. Lớp này xử lý tất cả các chi tiết giao thức, bao gồm quản lý quy trình, tuần tự hóa thông báo và phân tích phản hồi. Nó là nền tảng giữ cho mọi thứ ở trên nó ổn định.

## Cơ chế truyền thông

### Chế độ vận chuyển Stdio

CodeBuddy sử dụng **Stdio (đầu vào/đầu ra tiêu chuẩn)** để giao tiếp với các quy trình bên ngoài. Lệnh khởi động rất đơn giản:

```bash
codebuddy --acp
```

Sau đó, các thông báo JSON-RPC được trao đổi thông qua đầu vào và đầu ra tiêu chuẩn. Cách tiếp cận này có một số ưu điểm:

1. **Khởi động nhanh**: giao tiếp quy trình cục bộ tránh độ trễ mạng
2. **Cấu hình đơn giản**: bạn chỉ cần chỉ định đường dẫn thực thi
3. **Cách ly môi trường**: mỗi phiên chạy theo một tiến trình độc lập nên không ảnh hưởng lẫn nhau

Việc tiêm biến môi trường được hỗ trợ trong quá trình giao tiếp. Các ví dụ phổ biến bao gồm:

- `CODEBUDDY_API_KEY`: Xác thực khóa API
- `CODEBUDDY_INTERNET_ENVIRONMENT`: cấu hình môi trường mạng

Đối với giao tiếp giữa mọi người, việc chọn kênh thuận tiện trước tiên sẽ giúp ích.

### Giao thức tin nhắn

ACP dựa trên JSON-RPC 2.0. Định dạng tin nhắn trông gần như thế này:

```json
// Request message
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "agent/prompt",
  "params": {
    "prompt": "Help me write a sorting algorithm",
    "sessionId": "session-123"
  }
}

// Response message
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": "Here is the AI response..."
  }
}
```

Trong quá trình triển khai thực tế, chúng tôi gói gọn tất cả các chi tiết giao thức này để lớp nghiệp vụ phía trên chỉ cần quan tâm đến lời nhắc và phản hồi.

### 2. Tích hợp Nhà máy cung cấp

Thêm nhánh tạo CodeBuddy trong lớp nhà máy:

```csharp
// AIProviderFactory.cs
private IAIProvider? CreateProvider(AIProviderType providerType, ProviderConfiguration config)
{
    return providerType switch
    {
        AIProviderType.CodebuddyCli =>
            ActivatorUtilities.CreateInstance<CodebuddyCliProvider>(
                _serviceProvider,
                Options.Create(config)),
        // ... Other providers
        _ => throw new NotSupportedException($"Provider {providerType} not supported")
    };
}
```

Điều này sử dụng tính năng tiêm phụ thuộc thông qua `ActivatorUtilities`, nó tự động xử lý việc tiêm tham số hàm tạo và rất thuận tiện.

### 3. Hoàn thành việc triển khai nhà cung cấp

Dưới đây là cách triển khai cốt lõi của `CodebuddyCliProvider`, bao gồm cả chế độ gọi phát trực tuyến và không phát trực tuyến:

```csharp
public class CodebuddyCliProvider : IAIProvider
{
    private readonly ILogger<CodebuddyCliProvider> _logger;
    private readonly IACPSessionManager _sessionManager;
    private readonly ProviderConfiguration _config;

    public string Name => "CodebuddyCli";
    public bool SupportsStreaming => true;
    public ProviderCapabilities Capabilities { get; }

    public CodebuddyCliProvider(
        ILogger<CodebuddyCliProvider> logger,
        IACPSessionManager sessionManager,
        IOptions<ProviderConfiguration> config)
    {
        _logger = logger;
        _sessionManager = sessionManager;
        _config = config.Value;

        // Define the capabilities of the current Provider
        Capabilities = new ProviderCapabilities
        {
            SupportsStreaming = true,
            SupportsTools = true,
            SupportsSystemMessages = true,
            SupportsArtifacts = false,
            MaxTokens = 8192
        };
    }

    // Non-streaming call: return all results together after completion
    public async Task<AIResponse> ExecuteAsync(
        AIRequest request,
        CancellationToken cancellationToken = default)
    {
        // Create an independent session for the request
        var session = await _sessionManager.CreateSessionAsync(
            "CodebuddyCli",
            request.WorkingDirectory,
            cancellationToken,
            request.SessionId);

        try
        {
            var fullPrompt = BuildPrompt(request);
            await session.SendPromptAsync(fullPrompt, cancellationToken);

            var responseBuilder = new StringBuilder();
            var toolCalls = new List<AIToolCall>();

            // Collect all response chunks
            await foreach (var chunk in StreamFromSession(session, cancellationToken))
            {
                if (!string.IsNullOrEmpty(chunk.Content))
                {
                    responseBuilder.Append(chunk.Content);
                }
                // Handle tool calls...
            }

            return new AIResponse
            {
                Content = AIResultContentSanitizer.SanitizeResultContent(
                    responseBuilder.ToString()),
                ToolCalls = toolCalls,
                Provider = Name,
                Model = string.Empty
            };
        }
        finally
        {
            // Release session resources
            await session.DisposeAsync();
        }
    }

    // Streaming call: return response chunks in real time
    public async IAsyncEnumerable<AIStreamingChunk> StreamAsync(
        AIRequest request,
        [EnumeratorCancellation] CancellationToken cancellationToken = default)
    {
        var session = await _sessionManager.CreateSessionAsync(
            "CodebuddyCli",
            request.WorkingDirectory,
            cancellationToken);

        try
        {
            var fullPrompt = BuildPrompt(request);
            await session.SendPromptAsync(fullPrompt, cancellationToken);

            await foreach (var chunk in StreamFromSession(session, cancellationToken))
            {
                yield return chunk;
            }
        }
        finally
        {
            await session.DisposeAsync();
        }
    }

    private async IAsyncEnumerable<AIStreamingChunk> StreamFromSession(
        IACPSession session,
        [EnumeratorCancellation] CancellationToken cancellationToken)
    {
        // Iterate through all updates in the session
        await foreach (var notification in session.ReceiveUpdatesAsync(cancellationToken))
        {
            switch (notification.Update)
            {
                case AgentMessageChunkSessionUpdate agentMessage:
                    // Handle text content chunks
                    if (agentMessage.Content is AcpImp.TextContentBlock textContent)
                    {
                        yield return new AIStreamingChunk
                        {
                            Content = textContent.Text,
                            Type = StreamingChunkType.ContentDelta,
                            IsComplete = false
                        };
                    }
                    break;

                case ToolCallSessionUpdate toolCall:
                    // Handle tool calls
                    yield return new AIStreamingChunk
                    {
                        Content = string.Empty,
                        Type = StreamingChunkType.ToolCallDelta,
                        ToolCallDelta = new AIToolCallDelta
                        {
                            Id = toolCall.ToolCallId,
                            Name = toolCall.Kind.ToString(),
                            Arguments = toolCall.RawInput?.ToString()
                        }
                    };
                    break;

                case AcpImp.PromptCompletedSessionUpdate:
                    // Response complete
                    yield break;
            }
        }
    }

    // Build the full prompt
    private string BuildPrompt(AIRequest request, string? embeddedCommandPrompt = null)
    {
        var sb = new StringBuilder();

        // Embedded command prompt, if present
        if (!string.IsNullOrEmpty(embeddedCommandPrompt))
        {
            sb.AppendLine(embeddedCommandPrompt);
            sb.AppendLine();
        }

        // System message
        if (!string.IsNullOrEmpty(request.SystemMessage))
        {
            sb.AppendLine(request.SystemMessage);
            sb.AppendLine();
        }

        // User prompt
        sb.Append(request.Prompt);
        return sb.ToString();
    }
}
```

Có một số điểm chính trong mã này:

1. **Quản lý phiên**: mỗi yêu cầu tạo một phiên độc lập và giải phóng tài nguyên sau khi yêu cầu hoàn tất. Đây là một bài học được rút ra qua quá trình thử và sai. Nếu việc tái sử dụng phiên không được xử lý tốt, tình trạng ô nhiễm sẽ dễ dàng xuất hiện.

2. **Xử lý phát trực tuyến**: `IAsyncEnumerable` cho phép phản hồi được trả về trong khi nó vẫn đang được tạo, thay vì đợi tất cả nội dung kết thúc. Điều này đặc biệt quan trọng đối với các kịch bản văn bản dài và cải thiện đáng kể trải nghiệm người dùng.

3. **Gọi công cụ**: CodeBuddy hỗ trợ gọi công cụ (gọi hàm), được xử lý thông qua `ToolCallSessionUpdate`. Khả năng này rất quan trọng đối với các tác vụ chỉnh sửa mã phức tạp.

4. **Lọc nội dung**: `AIResultContentSanitizer` được sử dụng để lọc nội dung khối Think và giữ cho đầu ra sạch sẽ.

## Ví dụ về cấu hình

### Tệp cấu hình

Thêm cấu hình liên quan đến CodeBuddy vào `appsettings.json`:

```yaml
AI:
  # Default Provider to use
  DefaultProvider: "CodebuddyCli"

  # Provider configuration
  Providers:
    CodebuddyCli:
      Type: "CodebuddyCli"
      WorkingDirectory: "C:/projects/my-app"
      ExecutablePath: "C:/tools/codebuddy.cmd"

  # Platform-specific configuration
  PlatformConfigurations:
    CodebuddyCli:
      ExecutablePath: "C:/tools/codebuddy.cmd"
      Arguments: "--acp"
      StartupTimeoutMs: 5000
      EnvironmentVariables:
        CODEBUDDY_API_KEY: "${CODEBUDDY_API_KEY}"
        CODEBUDDY_INTERNET_ENVIRONMENT: "production"
```

### Mô hình cấu hình

Định nghĩa mô hình cấu hình tương ứng:

```csharp
public class CodebuddyPlatformConfiguration : IAcpPlatformConfiguration
{
    public string ProviderName => "CodebuddyCli";
    public AcpTransportType TransportType => AcpTransportType.Stdio;

    public string ExecutablePath { get; set; } = "codebuddy";
    public string Arguments { get; set; } = "--acp";
    public int StartupTimeoutMs { get; set; } = 5000;

    public Dictionary<string, string?>? EnvironmentVariables { get; set; }
}
```

## Bài học thực tế rút ra

### Nhật ký cạm bẫy

Chúng tôi đã gặp phải một số cạm bẫy điển hình trong quá trình triển khai và việc chia sẻ chúng ở đây có thể giúp những người khác tránh được những sai lầm tương tự:

1. **Sự cố rò rỉ phiên**: lúc đầu, các phiên không được phát hành chính xác, làm cạn kiệt tài nguyên quy trình. Giải pháp là sử dụng `try-finally` để đảm bảo tài nguyên được giải phóng cho mọi yêu cầu.

2. **Truyền biến môi trường**: Windows và Linux sử dụng cú pháp biến môi trường khác nhau, vì vậy sau này chúng tôi đã chuẩn hóa `Dictionary<string, string?>` để xử lý việc này.

3. **Cấu hình thời gian chờ**: Quá trình khởi động CLI cần có thời gian, vì vậy chúng tôi đặt thời gian chờ khởi động là 5 giây để tránh lỗi yêu cầu nhanh.

4. **Vấn đề về mã hóa**: trên Windows, mã hóa mặc định có thể khiến văn bản tiếng Trung bị cắt xén, do đó, mã hóa UTF-8 được chỉ định rõ ràng khi bắt đầu quá trình.

### Tối ưu hóa hiệu suất

1. **Nhóm phiên**: đối với các yêu cầu ngắn thường xuyên, hãy xem xét triển khai nhóm phiên để sử dụng lại các quy trình
2. **Bộ nhớ đệm kết nối**: lớp xuất xưởng đã hỗ trợ các phiên bản Nhà cung cấp bộ nhớ đệm
3. **Async trước**: sử dụng lập trình không đồng bộ xuyên suốt để tránh chặn luồng

Hiệu suất luôn đáng được tối ưu hóa. Người dùng càng chờ đợi lâu thì trải nghiệm càng trở nên tồi tệ.

## Kết luận

Bài viết này giới thiệu một giải pháp hoàn chỉnh để tích hợp CodeBuddy CLI vào chương trình phụ trợ C#, bao gồm toàn bộ quá trình từ thiết kế kiến trúc đến triển khai cụ thể. Thông qua kiến ​​trúc phân lớp, chúng tôi tách các chi tiết giao thức khỏi logic nghiệp vụ, giúp mã rõ ràng hơn và dễ bảo trì hơn.

Những điểm chính:

- Sử dụng kiến trúc phân lớp với lớp hợp đồng Nhà cung cấp, lớp nhà máy, lớp triển khai và lớp cơ sở hạ tầng
- Sử dụng JSON-RPC qua Stdio để liên lạc giữa các quá trình
- Triển khai cấu hình và khả năng mở rộng linh hoạt thông qua việc chèn phần phụ thuộc
- Cung cấp cả chế độ gọi phát trực tuyến và không phát trực tuyến

Cách tiếp cận này không chỉ phù hợp với CodeBuddy; việc thêm Nhà cung cấp AI mới có thể theo cùng một khuôn mẫu. Nếu bạn cũng đang xây dựng một tích hợp đa Nhà cung cấp AI tương tự, tôi hy vọng bài viết này cung cấp cho bạn một tài liệu tham khảo hữu ích.
