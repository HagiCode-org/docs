## Hướng dẫn đầy đủ về phân tích thông báo trên bảng điều khiển Codex SDK

> Bài viết này giải thích cơ chế truyền phát sự kiện Codex SDK, phân tích cú pháp loại thông báo và các phương pháp hay nhất trong các dự án thực, giúp nhà phát triển nhanh chóng nắm vững các kỹ năng cốt lõi đằng sau các dịch vụ thực thi AI.

## Nền

Khi xây dựng dịch vụ thực thi AI dựa trên Codex SDK, chúng tôi chắc chắn sẽ gặp phải một câu hỏi thực tế: chúng tôi nên xử lý các thông báo sự kiện phát trực tiếp do Codex trả về như thế nào? Những thông báo này chứa thông tin quan trọng như trạng thái thực thi, nội dung đầu ra và chi tiết lỗi, vì vậy chúng cần được xử lý cẩn thận.

Là một phần của dự án HagiCode, chúng tôi cần một người thực thi đáng tin cậy cho các kịch bản hỗ trợ mã hóa AI. Đó chính xác là lý do tại sao chúng tôi quyết định nghiên cứu sâu cơ chế truyền phát sự kiện Codex SDK. Rốt cuộc, chỉ bằng cách hiểu cách hoạt động của các thông điệp cơ bản, chúng ta mới có thể xây dựng một nền tảng thực thi AI cấp doanh nghiệp thực sự.

Codex SDK là SDK hỗ trợ lập trình do OpenAI phát hành. Nó trả về kết quả thực thi thông qua Luồng sự kiện. Không giống như mô hình phản hồi yêu cầu truyền thống, Codex sử dụng các sự kiện được truyền trực tuyến để chúng tôi có thể:

- Nhận tiến độ thực hiện trong thời gian thực
- Xử lý kịp thời các sai sót
- Nhận số liệu thống kê sử dụng mã thông báo chi tiết
- Hỗ trợ các tác vụ phức tạp chạy dài

Hiểu các loại sự kiện này và phân tích cú pháp chúng một cách chính xác là điều cần thiết để triển khai một trình thực thi AI có đầy đủ năng lực. Cuối cùng, không ai muốn làm việc với hộp đen.

## Cơ chế truyền phát sự kiện

### Khái niệm cơ bản

Codex SDK sử dụng `thread.runStreamed()` phương thức trả về một trình vòng lặp sự kiện không đồng bộ:

```typescript
import { Codex } from '@openai/codex-sdk';

const client = new Codex({
  apiKey: process.env.CODEX_API_KEY,
  baseUrl: process.env.CODEX_BASE_URL,
});

const thread = client.startThread({
  workingDirectory: '/path/to/project',
  skipGitRepoCheck: false,
});

const { events } = await thread.runStreamed('your prompt here', {
  outputSchema: {
    type: 'object',
    properties: {
      output: { type: 'string' },
      status: { type: 'string', enum: ['ok', 'action_required'] },
    },
    required: ['output', 'status'],
  },
});

for await (const event of events) {
  // Handle each event
}
```

### Giải thích chi tiết về các loại sự kiện

| Loại sự kiện | Mô tả | Dữ liệu chính |
|---------|------|----------|
| `thread.started` | Chủ đề đã bắt đầu thành công | `thread_id` |
| `item.updated` | Đã cập nhật nội dung tin nhắn | `item.text` |
| `item.completed` | Tin nhắn đã hoàn thành | `item.text` |
| `turn.completed` | Thực thi hoàn tất | `usage` (sử dụng mã thông báo) |
| `turn.failed` | Thực thi không thành công | `error.message` |
| `error` | Sự kiện lỗi | `message` |

Trong các dự án thực tế, thành phần thực thi của HagiCode được xây dựng dựa trên các loại sự kiện này. Chúng tôi cần xử lý từng loại sự kiện một cách cẩn thận để đảm bảo trải nghiệm người dùng suôn sẻ. Hệ thống tốt được xây dựng bằng cách xem xét chi tiết một cách nghiêm túc.

## Triển khai phân tích tin nhắn

### Trích xuất nội dung tin nhắn

Nội dung tin nhắn được trích xuất thông qua trình xử lý sự kiện:

```typescript
private handleThreadEvent(event: ThreadEvent, onMessage: (content: string) => void): void {
  // Only handle message update and completion events
  if (event.type !== 'item.updated' && event.type !== 'item.completed') {
    return;
  }

  // Only handle agent message content
  if (event.item.type !== 'agent_message') {
    return;
  }

  // Extract text content
  onMessage(event.item.text);
}
```

Những điểm chính:
- Chỉ xử lý `item.updated` và `item.completed` sự kiện
- Chỉ xử lý nội dung thuộc loại `agent_message`
- Nội dung tin nhắn nằm trong `event.item.text` lĩnh vực

### Phân tích đầu ra có cấu trúc

Codex hỗ trợ đầu ra có cấu trúc JSON. Bạn có thể chỉ định định dạng trả về thông qua `outputSchema` tham số:

```typescript
const DEFAULT_OUTPUT_SCHEMA = {
  type: 'object',
  properties: {
    output: { type: 'string' },
    status: { type: 'string', enum: ['ok', 'action_required'] },
  },
  required: ['output', 'status'],
  additionalProperties: false,
} as const;
```

Hàm phân tích cú pháp sẽ cố gắng phân tích cú pháp JSON và nếu thất bại, nó sẽ quay trở lại văn bản thô.

```typescript
function toStructuredOutput(raw: string): StructuredOutput {
  try {
    const parsed = JSON.parse(raw) as Partial<StructuredOutput>;
    if (typeof parsed.output === 'string') {
      return {
        output: parsed.output,
        status: parsed.status === 'action_required' ? 'action_required' : 'ok',
      };
    }
  } catch {
    // JSON parsing failed, fall back to the raw text
  }

  return {
    output: raw,
    status: 'ok',
  };
}
```

### Hoàn thành luồng xử lý sự kiện

```typescript
private async runWithStreaming(
  thread: Thread,
  input: CodexStageExecutionInput
): Promise<{ output: string; usage: Usage | null }> {
  const abortController = new AbortController();
  const timeoutHandle = setTimeout(() => {
    abortController.abort();
  }, Math.max(1000, input.timeoutMs));

  let latestMessage = '';
  let usage: Usage | null = null;
  let emittedLength = 0;

  try {
    const { events } = await thread.runStreamed(input.prompt, {
      outputSchema: DEFAULT_OUTPUT_SCHEMA,
      signal: abortController.signal,
    });

    for await (const event of events) {
      // Handle message content
      this.handleThreadEvent(event, (nextContent) => {
        const delta = nextContent.slice(emittedLength);
        if (delta.length > 0) {
          emittedLength = nextContent.length;
          input.callbacks?.onChunk?.(delta);  // Streaming callback
        }
        latestMessage = nextContent;
      });

      // Process different data based on the event type
      if (event.type === 'thread.started') {
        this.threadId = event.thread_id;
      } else if (event.type === 'turn.completed') {
        usage = event.usage;
      } else if (event.type === 'turn.failed') {
        throw new CodexExecutorError('gateway_unavailable', event.error.message, true);
      } else if (event.type === 'error') {
        throw new CodexExecutorError('gateway_unavailable', event.message, true);
      }
    }
  } catch (error) {
    if (abortController.signal.aborted) {
      throw new CodexExecutorError(
        'upstream_timeout',
        `Codex stage timed out after ${input.timeoutMs}ms`,
        true
      );
    }
    throw error;
  } finally {
    clearTimeout(timeoutHandle);
  }

  const structured = toStructuredOutput(latestMessage);
  return { output: structured.output, usage };
}
```

## Chiến lược xử lý lỗi

### Ánh xạ mã lỗi

Ánh xạ các mẫu lỗi cụ thể tới mã lỗi cụ thể để các lớp trên có thể xử lý chúng dễ dàng hơn:

```typescript
function mapError(error: unknown): CodexExecutorError {
  if (error instanceof CodexExecutorError) {
    return error;
  }

  const message = error instanceof Error ? error.message : String(error);
  const normalized = message.toLowerCase();

  // Authentication errors - not retryable
  if (normalized.includes('401') ||
      normalized.includes('403') ||
      normalized.includes('api key') ||
      normalized.includes('auth')) {
    return new CodexExecutorError('auth_invalid', message, false);
  }

  // Rate limit errors - retryable
  if (normalized.includes('429') || normalized.includes('rate limit')) {
    return new CodexExecutorError('rate_limited', message, true);
  }

  // Timeout errors - retryable
  if (normalized.includes('timeout') || normalized.includes('aborted')) {
    return new CodexExecutorError('upstream_timeout', message, true);
  }

  // Default error
  return new CodexExecutorError('gateway_unavailable', message, true);
}
```

### Định nghĩa loại lỗi

```typescript
export type CodexErrorCode =
  | 'auth_invalid'      // Authentication failure
  | 'upstream_timeout'  // Upstream timeout
  | 'rate_limited'      // Rate limited
  | 'gateway_unavailable'; // Gateway unavailable

export class CodexExecutorError extends Error {
  readonly code: CodexErrorCode;
  readonly retryable: boolean;

  constructor(code: CodexErrorCode, message: string, retryable: boolean) {
    super(message);
    this.name = 'CodexExecutorError';
    this.code = code;
    this.retryable = retryable;
  }
}
```

## Cấu hình thư mục và môi trường làm việc

### Xác thực thư mục làm việc

SDK Codex yêu cầu thư mục làm việc phải là kho lưu trữ Git hợp lệ.

```typescript
export function validateWorkingDirectory(
  workingDirectory: string,
  skipGitRepoCheck: boolean
): void {
  const resolvedWorkingDirectory = path.resolve(workingDirectory);

  if (!existsSync(resolvedWorkingDirectory)) {
    throw new CodexExecutorError(
      'gateway_unavailable',
      'Working directory does not exist.',
      false
    );
  }

  if (!statSync(resolvedWorkingDirectory).isDirectory()) {
    throw new CodexExecutorError(
      'gateway_unavailable',
      'Working directory is not a directory.',
      false
    );
  }

  if (skipGitRepoCheck) {
    return;
  }

  const gitDir = path.join(resolvedWorkingDirectory, '.git');
  if (!existsSync(gitDir)) {
    throw new CodexExecutorError(
      'gateway_unavailable',
      'Working directory is not a git repository.',
      false
    );
  }
}
```

### Đang tải các biến môi trường

SDK Codex cần tải các biến môi trường từ shell đăng nhập để Tác nhân AI có thể truy cập các lệnh hệ thống:

```typescript
function parseEnvironmentOutput(output: Buffer): Record<string, string> {
  const parsed: Record<string, string> = {};

  for (const entry of output.toString('utf8').split('\0')) {
    if (!entry) continue;

    const separatorIndex = entry.indexOf('=');
    if (separatorIndex <= 0) continue;

    const key = entry.slice(0, separatorIndex);
    const value = entry.slice(separatorIndex + 1);
    if (key.length > 0) {
      parsed[key] = value;
    }
  }

  return parsed;
}

function tryLoadEnvironmentFromShell(shellPath: string): Record<string, string> | null {
  const result = spawnSync(shellPath, ['-ilc', 'env -0'], {
    env: process.env,
    stdio: ['ignore', 'pipe', 'pipe'],
    timeout: 5000,
  });

  if (result.error || result.status !== 0) {
    return null;
  }

  return parseEnvironmentOutput(result.stdout);
}

export function createExecutorEnvironment(
  envOverrides: Record<string, string> = {}
): Record<string, string> {
  // Load environment variables from the login shell
  const consoleEnv = loadConsoleEnvironmentFromShell();

  return {
    ...process.env,
    ...consoleEnv,
    ...envOverrides,
  };
}
```

## Ví dụ sử dụng hoàn chỉnh

### Cách sử dụng cơ bản

Trong dự án HagiCode, chúng tôi sử dụng phương pháp sau để khởi tạo ứng dụng khách Codex và thực thi các tác vụ:

```typescript
import { Codex } from '@openai/codex-sdk';

async function executeWithCodex(prompt: string, workingDir: string) {
  const client = new Codex({
    apiKey: process.env.CODEX_API_KEY,
    env: { PATH: process.env.PATH },
  });

  const thread = client.startThread({
    workingDirectory: workingDir,
  });

  const { events } = await thread.runStreamed(prompt);

  let result = '';
  for await (const event of events) {
    if (event.type === 'item.updated' && event.item.type === 'agent_message') {
      result = event.item.text;
    }
    if (event.type === 'turn.completed') {
      console.log('Token usage:', event.usage);
    }
  }

  // Try to parse JSON output
  try {
    const parsed = JSON.parse(result);
    return parsed.output;
  } catch {
    return result;
  }
}
```

### Hoàn thành triển khai với số lần thử lại

```typescript
export class CodexSdkExecutor {
  private readonly config: CodexRuntimeConfig;
  private readonly client: Codex;
  private threadId: string | null = null;

  async executeStage(input: CodexStageExecutionInput): Promise<CodexStageExecutionResult> {
    const maxAttempts = Math.max(1, this.config.retryCount + 1);
    let attempt = 0;
    let lastError: CodexExecutorError | null = null;

    while (attempt < maxAttempts) {
      attempt += 1;

      try {
        const thread = this.getThread(input.workingDirectory);
        const { output, usage } = await this.runWithStreaming(thread, input);

        return {
          output,
          usage,
          threadId: this.threadId!,
          attempts: attempt,
          latencyMs: Date.now() - startedAt,
        };
      } catch (error) {
        const mappedError = mapError(error);
        lastError = mappedError;

        // Non-retryable error or max retry attempts reached
        if (!mappedError.retryable || attempt >= maxAttempts) {
          throw mappedError;
        }

        // Wait before retrying
        await new Promise(resolve => setTimeout(resolve, 1000 * attempt));
      }
    }

    throw lastError!;
  }
}
```

## Thực tiễn tốt nhất

### 1. Yêu cầu về Thư mục Làm việc

- Đảm bảo thư mục làm việc là kho lưu trữ Git hợp lệ
- Sử dụng `PROJECT_ROOT` biến môi trường để xác định nó một cách rõ ràng
- Trong quá trình gỡ lỗi, bạn có thể đặt `CODEX_SKIP_GIT_REPO_CHECK=true` bỏ qua việc kiểm tra

### 2. Cấu hình biến môi trường

- Chỉ truyền các biến môi trường cần thiết thông qua cơ chế danh sách trắng
- Sử dụng shell đăng nhập để tải toàn bộ môi trường
- Tránh truyền thông tin nhạy cảm

### 3. Chiến lược hết thời gian và thử lại

- Đặt thời gian chờ hợp lý dựa trên độ phức tạp của nhiệm vụ
- Triển khai thời gian chờ theo cấp số nhân cho các lỗi có thể thử lại
- Ghi lại số lần thử lại và lý do

### 4. Xử lý lỗi

- Phân biệt giữa lỗi có thể thử lại và lỗi không thể thử lại
- Cung cấp thông báo lỗi và đề xuất rõ ràng
- Sử dụng mã lỗi thống nhất để các lớp trên có thể xử lý chúng một cách nhất quán

### 5. Đầu ra phát trực tuyến

- Triển khai các lệnh gọi lại đầu ra gia tăng để cải thiện trải nghiệm người dùng
- Xử lý chính xác các cập nhật tin nhắn gia tăng
- Ghi lại việc sử dụng mã thông báo để phân tích chi phí

Trong môi trường sản xuất thực tế của dự án HagiCode, chúng tôi đã xác minh tính hiệu quả của các biện pháp thực hành tốt nhất ở trên. Cách tiếp cận này đã giúp chúng tôi xây dựng một dịch vụ thực thi AI ổn định và đáng tin cậy. Cuối cùng, việc xác nhận thực tế quan trọng hơn lý thuyết đơn thuần.
