## Từ lệnh gọi CLI đến tích hợp SDK: Các phương pháp thực hành tốt nhất cho GitHub Copilot trong các dự án .NET

> Lộ trình nâng cấp từ lệnh gọi dòng lệnh đến tích hợp SDK chính thức là một hành trình khá dài. Hôm nay, tôi muốn chia sẻ những cạm bẫy mà chúng tôi đã gặp phải và những điều chúng tôi đã học được khi xây dựng HagiCode.

## Nền

Sau khi SDK GitHub Copilot được phát hành chính thức vào năm 2025, chúng tôi bắt đầu tích hợp nó vào lớp khả năng AI của mình. Trước đó, dự án chủ yếu sử dụng GitHub Copilot bằng cách gọi trực tiếp công cụ dòng lệnh Copilot CLI, nhưng cách tiếp cận đó có một số vấn đề rõ ràng:

- **Quản lý quy trình phức tạp**: Chúng tôi phải quản lý thủ công vòng đời của quy trình CLI, thời gian chờ khởi động và dọn dẹp quy trình. Quá trình có thể sụp đổ mà không có cảnh báo.
- **Xử lý sự kiện chưa hoàn chỉnh**: Lệnh gọi CLI thô khiến việc nắm bắt các sự kiện chi tiết từ lý luận mô hình và thực thi công cụ trở nên khó khăn. Nó giống như chỉ nhìn thấy kết quả mà không có quá trình suy nghĩ.
- **Quản lý phiên khó khăn**: Không có cơ chế hiệu quả để sử dụng lại và khôi phục phiên nên mọi tương tác đều phải bắt đầu lại.
- **Vấn đề về khả năng tương thích**: Các đối số CLI thay đổi thường xuyên, nghĩa là chúng tôi liên tục phải duy trì logic tương thích cho các tham số đó.

Những vấn đề này ngày càng trở nên rõ ràng trong quá trình phát triển hàng ngày, đặc biệt là khi chúng ta cần theo dõi lý luận mô hình (`thinking`) và trạng thái thực thi công cụ trong thời gian thực. Vào thời điểm đó, rõ ràng là chúng tôi cần một cách tiếp cận tích hợp ở cấp độ thấp hơn và hoàn thiện hơn.

Dự án sử dụng kiến trúc phân lớp để giải quyết các hạn chế của lệnh gọi CLI:

```
┌─────────────────────────────────────────────────────────┐
│  hagicode-core (Orleans Grains + AI Provider Layer)    │
│  - CopilotAIProvider: Converts AIRequest to CopilotOptions │
│  - GitHubCopilotGrain: Orleans distributed execution interface │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  HagiCode.Libs (Shared Provider Layer)                 │
│  - CopilotProvider: CLI Provider interface implementation │
│  - ICopilotSdkGateway: SDK invocation abstraction      │
│  - GitHubCopilotSdkGateway: SDK session management and event dispatch │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  GitHub Copilot SDK (Official .NET SDK)                │
│  - CopilotClient: SDK client                           │
│  - CopilotSession: Session management                  │
│  - SessionEvent: Event stream                          │
└─────────────────────────────────────────────────────────┘
```

Thiết kế lớp này mang lại một số lợi thế kỹ thuật thực tế:

1. **Tách biệt các mối quan tâm**: Logic kinh doanh cốt lõi được tách rời khỏi chi tiết triển khai SDK.
2. **Khả năng kiểm tra**: `ICopilotSdkGateway` giao diện làm cho việc kiểm tra đơn vị trở nên đơn giản.
3. **Khả năng tái sử dụng**: `HagiCode.Libs` có thể được tham chiếu bởi nhiều dự án.
4. **Khả năng bảo trì**: Việc nâng cấp SDK chỉ yêu cầu thay đổi ở lớp Cổng, trong khi các lớp trên vẫn không bị ảnh hưởng.

## Triển khai cốt lõi

### Luồng xác thực

Xác thực là bước đầu tiên và quan trọng nhất của quá trình tích hợp SDK. Nếu xác thực thất bại, không có gì khác quan trọng. Chúng tôi đã thiết kế cấu hình xác thực linh hoạt hỗ trợ nhiều nguồn xác thực:

```csharp
// CopilotProvider.cs - Authentication source configuration
public class CopilotOptions
{
    public bool UseLoggedInUser { get; set; } = true;
    public string? GitHubToken { get; set; }
    public string? CliUrl { get; set; }
}

// Convert to SDK request
return new CopilotSdkRequest(
    GitHubToken: options.AuthSource == CopilotAuthSource.GitHubToken
        ? options.GitHubToken
        : null,
    UseLoggedInUser: options.AuthSource != CopilotAuthSource.GitHubToken
);
```

Lợi ích của thiết kế này khá rõ ràng:

- Nó hỗ trợ chế độ người dùng đăng nhập mà không yêu cầu mã thông báo, rất phù hợp với các tình huống trên máy tính để bàn.
- Nó hỗ trợ chế độ GitHub Token, phù hợp cho việc triển khai phía máy chủ và quản lý tập trung.
- Nó hỗ trợ ghi đè URL Copilot CLI, giúp cấu hình proxy doanh nghiệp.

Trong thực tế, mô hình xác thực linh hoạt này đã đơn giản hóa rất nhiều việc cấu hình trong các tình huống triển khai khác nhau. Máy khách để bàn có thể sử dụng trạng thái đăng nhập Copilot của riêng mỗi người dùng, trong khi phía máy chủ có thể quản lý xác thực tập trung thông qua mã thông báo.

### Xử lý luồng sự kiện

Một trong những khả năng mạnh mẽ nhất của SDK là ghi lại luồng sự kiện hoàn chỉnh. Chúng tôi đã triển khai hệ thống điều phối sự kiện có thể xử lý tất cả các loại sự kiện SDK trong thời gian thực:

```csharp
// GitHubCopilotSdkGateway.cs - Core logic for event dispatch
internal static SessionEventDispatchResult DispatchSessionEvent(
    SessionEvent evt, bool sawDelta)
{
    switch (evt)
    {
        case AssistantReasoningEvent reasoningEvent:
            // Capture the model reasoning process
            events.Add(new CopilotSdkStreamEvent(
                CopilotSdkStreamEventType.ReasoningDelta,
                Content: reasoningEvent.Data.Content));
            break;

        case ToolExecutionStartEvent toolStartEvent:
            // Capture the start of a tool invocation
            events.Add(new CopilotSdkStreamEvent(
                CopilotSdkStreamEventType.ToolExecutionStart,
                ToolName: toolStartEvent.Data.ToolName,
                ToolCallId: toolStartEvent.Data.ToolCallId));
            break;

        case ToolExecutionCompleteEvent toolCompleteEvent:
            // Capture tool completion and its result
            events.Add(new CopilotSdkStreamEvent(
                CopilotSdkStreamEventType.ToolExecutionEnd,
                Content: ExtractToolExecutionContent(toolCompleteEvent)));
            break;

        default:
            // Preserve unhandled events as RawEvent
            events.Add(new CopilotSdkStreamEvent(
                CopilotSdkStreamEventType.RawEvent,
                RawEventType: evt.GetType().Name));
            break;
    }
}
```

Giá trị của việc thực hiện này là rất đáng kể:

- **Nắm bắt hoàn chỉnh quá trình suy luận mô hình** (`thinking`): người dùng có thể xem AI suy luận như thế nào chứ không chỉ là câu trả lời cuối cùng.
- **Theo dõi thời gian thực về trạng thái thực thi công cụ**: chúng tôi biết công cụ nào đang chạy, thời điểm hoàn thành và kết quả trả về.
- **Không mất sự kiện**: bằng cách quay lại `RawEvent`, chúng tôi đảm bảo mọi sự kiện đều được ghi lại.

Trong HagiCode, những sự kiện chi tiết này giúp người dùng hiểu cách AI hoạt động nội bộ, đặc biệt là khi gỡ lỗi các tác vụ phức tạp.

Trong quá trình nâng cấp, cơ chế tương thích này đã giúp chúng tôi chuyển đổi suôn sẻ. Các tệp cấu hình hiện tại vẫn có thể được sử dụng và chỉ cần điều chỉnh dần dần dựa trên kết quả chẩn đoán.

### Tổng hợp thời gian chạy

Việc tạo các phiên SDK Copilot tương đối tốn kém và việc tạo và hủy các phiên quá thường xuyên sẽ ảnh hưởng đến hiệu suất. Để giải quyết vấn đề đó, chúng tôi đã triển khai hệ thống quản lý nhóm phiên:

```csharp
// CopilotProvider.cs - Session pool management
await using var lease = await _poolCoordinator.AcquireCopilotRuntimeAsync(
    request,
    async ct => await _gateway.CreateRuntimeAsync(sdkRequest, ct),
    cancellationToken);

if (lease.IsWarmLease)
{
    // Reuse an existing session
    yield return CreateSessionReusedMessage();
}

await foreach (var eventData in lease.Entry.Resource.SendPromptAsync(...))
{
    yield return MapEvent(eventData);
}
```

Lợi ích của việc gộp phiên bao gồm:

- **Tái sử dụng phiên**: các yêu cầu giống nhau `sessionId` có thể sử dụng lại các phiên hiện có và giảm chi phí khởi động.
- **Hỗ trợ khôi phục phiên**: sau khi mạng bị gián đoạn, trạng thái phiên trước đó có thể được khôi phục.
- **Quản lý tổng hợp tự động**: các phiên hết hạn sẽ tự động được dọn sạch để tránh rò rỉ tài nguyên.

Trong HagiCode, tính năng tổng hợp phiên được cải thiện đáng kể khả năng phản hồi, đặc biệt là đối với các cuộc hội thoại liên tục.

- **Cách ly nhiều người thuê**: Các phiên copilot dành cho những người dùng khác nhau vẫn được cách ly với nhau.
- **Trạng thái phiên liên tục**: trạng thái phiên có thể được khôi phục ngay cả sau khi máy chủ khởi động lại.

Đối với các tình huống cần xử lý số lượng lớn yêu cầu đồng thời, Orleans cung cấp khả năng mở rộng mạnh mẽ.

### Ghi chú sử dụng

Khi sử dụng trong thế giới thực, chúng tôi đã tóm tắt một số điểm đáng chú ý:

**Cấu hình thời gian chờ khởi động**: Lần khởi động đầu tiên của Copilot CLI có thể mất một thời gian tương đối dài, vì vậy chúng tôi khuyên bạn nên cài đặt `StartupTimeout` đến ít nhất 30 giây. Nếu đây là lần đăng nhập đầu tiên thì có thể mất nhiều thời gian hơn.

**Quản lý quyền**: Trong môi trường sản xuất, tránh sử dụng `AllowAllTools: true`. Sử dụng `AllowedTools` danh sách cho phép để kiểm soát những công cụ nào có sẵn và sử dụng `DeniedTools` danh sách từ chối để chặn các hoạt động nguy hiểm. Điều này ngăn cản AI thực hiện các lệnh rủi ro một cách hiệu quả.

**Quản lý phiên**: Các yêu cầu tương tự `sessionId` tự động sử dụng lại các phiên. Trạng thái phiên được duy trì thông qua `ProviderSessionId`. Việc hủy bỏ được truyền bá qua `CancellationTokenSource`.

**Đầu ra chẩn đoán**: Đối số CLI không tương thích sẽ tạo ra các loại thông báo `diagnostic`. Các sự kiện SDK thô được giữ nguyên dưới dạng `event.raw`. Thông báo lỗi bao gồm các danh mục như thời gian chờ khởi động và tính không tương thích đối số để giúp khắc phục sự cố dễ dàng hơn.

### Thực tiễn tốt nhất

Dựa trên kinh nghiệm thực tế của chúng tôi, đây là một số phương pháp hay nhất:

**1. Sử dụng danh sách công cụ được phép**

```csharp
var request = new AIRequest
{
    Prompt = "Analyze this file",
    AllowedTools = new[] { "Read", "Grep", "Bash(git:*)" }
};
```

Việc chỉ định rõ ràng các công cụ được phép thông qua danh sách cho phép giúp ngăn chặn các hành động AI không mong muốn. Điều này đặc biệt quan trọng đối với các công cụ có quyền ghi, chẳng hạn như `Edit`, cần được xử lý cẩn thận hơn.

**2. Đặt thời gian chờ hợp lý**

```csharp
options.Timeout = 3600;  // 1 hour
options.StartupTimeout = 60;  // 1 minute
```

Đặt giá trị thời gian chờ thích hợp dựa trên độ phức tạp của nhiệm vụ. Nếu giá trị quá ngắn, nhiệm vụ có thể bị gián đoạn. Nếu quá dài, tài nguyên có thể bị lãng phí khi chờ đợi những yêu cầu không được phản hồi.

**3. Cho phép sử dụng lại phiên**

```csharp
options.SessionId = "my-session-123";
```

Sử dụng tương tự `sessionId` đối với các tác vụ liên quan cho phép bạn sử dụng lại ngữ cảnh phiên trước đó và cải thiện tốc độ phản hồi.

**4. Xử lý phản hồi phát trực tuyến**

```csharp
await foreach (var chunk in provider.StreamAsync(request))
{
    switch (chunk.Type)
    {
        case StreamingChunkType.ThinkingDelta:
            // Handle the reasoning process

        case StreamingChunkType.ToolCallDelta:

            // Handle text output

```

Phản hồi trực tuyến cho phép bạn hiển thị tiến trình xử lý AI trong thời gian thực, giúp cải thiện trải nghiệm người dùng. Điều này đặc biệt có giá trị đối với các tác vụ chạy dài.

**5. Xử lý lỗi và thử lại**

```csharp

{
```

Xử lý lỗi thích hợp và logic thử lại sẽ cải thiện độ ổn định tổng thể của hệ thống.

- **Trải nghiệm người dùng tốt hơn**: phản hồi sự kiện theo thời gian thực giúp người dùng hiểu rõ hơn về những gì AI đang làm.

Việc nâng cấp này không chỉ là sự thay thế việc triển khai kỹ thuật. Nó cũng là sự tối ưu hóa kiến ​​trúc của toàn bộ lớp khả năng AI. Thông qua thiết kế lớp và giao diện trừu tượng, chúng tôi đã đạt được khả năng bảo trì và mở rộng tốt hơn.

Nếu bạn đang cân nhắc việc tích hợp GitHub Copilot vào dự án .NET của riêng mình, tôi hy vọng những bài học thực tế này sẽ giúp bạn tránh được những đường vòng không cần thiết. SDK chính thức thực sự ổn định và đầy đủ hơn so với lệnh gọi CLI và rất đáng để bạn dành thời gian tìm hiểu và áp dụng nó.
