## Phân tích kỹ thuật của Hệ thống kỹ năng HagiCode: Xây dựng nền tảng quản lý kỹ năng AI có thể mở rộng

> Bài viết này xem xét sâu về kiến trúc và cách triển khai hệ thống quản lý Kỹ năng trong dự án HagiCode, bao gồm các chi tiết kỹ thuật đằng sau bốn khả năng cốt lõi: quản lý toàn cầu cục bộ, tìm kiếm thị trường, đề xuất thông minh và quản lý nhà cung cấp đáng tin cậy.

## Nền

Trong lĩnh vực trợ lý mã hóa AI, làm thế nào để mở rộng ranh giới khả năng của AI luôn là câu hỏi cốt lõi. Bản thân Claude Code vốn đã mạnh về hỗ trợ mã, nhưng các nhóm phát triển khác nhau và các nhóm công nghệ khác nhau thường cần các khả năng chuyên biệt cho các tình huống cụ thể, chẳng hạn như xử lý việc triển khai Docker, tối ưu hóa cơ sở dữ liệu hoặc tạo thành phần giao diện người dùng. Đó chính xác là lúc hệ thống Kỹ năng trở nên đặc biệt quan trọng.

Trong quá trình phát triển dự án HagiCode, chúng tôi gặp phải một thách thức tương tự: làm cách nào để Claude Code "học" các kỹ năng chuyên môn mới như một người, trong khi vẫn duy trì trải nghiệm người dùng vững chắc và khả năng bảo trì kỹ thuật tốt? Vấn đề này vừa khó vừa đơn giản theo cách riêng của nó. Xoay quanh câu hỏi đó, chúng tôi đã thiết kế và triển khai một hệ thống quản lý Kỹ năng hoàn chỉnh.

Bài viết này trình bày chi tiết về kiến trúc kỹ thuật và cách triển khai cốt lõi của hệ thống. Nó dành cho các nhà phát triển quan tâm đến khả năng mở rộng AI và tích hợp công cụ dòng lệnh. Nó có thể hữu ích cho bạn, hoặc có thể không, nhưng ít nhất bây giờ nó đã được viết ra.

## Giới thiệu về HagiCode

Cách tiếp cận được chia sẻ trong bài viết này xuất phát từ kinh nghiệm thực tế của chúng tôi trong [Mã Hagi](https://hagicode.com) dự án. HagiCode là một dự án trợ lý mã hóa AI mã nguồn mở được thiết kế để giúp các nhóm phát triển nâng cao hiệu quả kỹ thuật. Ngăn xếp của dự án bao gồm ASP.NET Core, khung phân phối Orleans, giao diện TanStack Start + React và hệ thống con quản lý Kỹ năng được giới thiệu trong bài viết này.

Kho lưu trữ GitHub là [HagiCode-org/trang web](https://github.com/HagiCode-org/site). Nếu bạn thấy phương pháp tiếp cận kỹ thuật trong bài viết này có giá trị, vui lòng đánh dấu Sao cho nó. Rốt cuộc, nhiều ngôi sao hơn có xu hướng cải thiện tâm trạng.

## Tổng quan về kiến trúc hệ thống

Hệ thống Skill sử dụng kiến trúc tách biệt giữa frontend-backend. Không có gì đặc biệt bí ẩn về điều đó.

**Frontend** sử dụng TanStack Start + React để xây dựng giao diện người dùng, với trạng thái quản lý Redux Toolkit. Bốn khả năng chính ánh xạ trực tiếp tới bốn thành phần Tab: Kỹ năng địa phương, Thư viện kỹ năng, Đề xuất thông minh và Nhà cung cấp đáng tin cậy. Cuối cùng, thiết kế chủ yếu là làm cho trải nghiệm người dùng tốt hơn.

**Phần cuối** dựa trên ASP.NET Core + ABP Framework, sử dụng Orleans Grain để quản lý trạng thái phân tán. Ứng dụng khách API trực tuyến bao gồm `IOnlineApiClient` giao diện để giao tiếp với dịch vụ danh mục kỹ năng từ xa.

Nguyên tắc kiến trúc tổng thể là tách biệt việc thực thi lệnh khỏi logic nghiệp vụ. Thông qua mẫu bộ điều hợp, chi tiết triển khai thực thi lệnh npm/npx được ẩn bên trong các mô-đun độc lập. Suy cho cùng, không ai thực sự muốn các lệnh gọi dòng lệnh rải rác khắp cơ sở mã.

## Năng lực cốt lõi 1: Quản lý toàn cầu địa phương

Quản lý toàn cầu cục bộ là mô-đun cơ bản nhất. Nó chịu trách nhiệm liệt kê các kỹ năng đã cài đặt và hỗ trợ các thao tác gỡ cài đặt. Không có gì quá phức tạp ở đây; chủ yếu là làm tốt những điều cơ bản.

### Phương pháp kỹ thuật

Việc thực hiện tồn tại trong `LocalSkillsTab.tsx` và `LocalSkillCommandAdapter.cs`. Ý tưởng cốt lõi là bọc `npx skills` lệnh, phân tích đầu ra JSON của nó và chuyển đổi nó thành cấu trúc dữ liệu nội bộ. Nghe có vẻ đơn giản, và trong thực tế thì hầu như là như vậy.

```csharp
public async Task<IReadOnlyList<LocalSkillInventoryResponseDto>> GetLocalSkillsAsync(
    CancellationToken cancellationToken = default)
{
    var result = await _commandAdapter.ListGlobalSkillsAsync(cancellationToken);
    return result.Skills.Select(skill => new LocalSkillInventoryResponseDto
    {
        Name = skill.Name,
        Version = skill.Version,
        Source = skill.Source,
        InstalledPath = skill.InstalledPath,
        Description = skill.Description
    }).ToList();
}
```

Luồng dữ liệu rất rõ ràng: giao diện người dùng gửi yêu cầu -> `SkillGalleryAppService` nhận được nó -> `LocalSkillCommandAdapter` thực thi lệnh npx -> kết quả JSON được phân tích cú pháp -> trả về DTO. Mỗi bước tiếp theo một cách tự nhiên từ bước trước.

Gỡ cài đặt kỹ năng sử dụng `npx skills remove -g <skillName> -y` lệnh và hệ thống sẽ tự động xử lý các phần phụ thuộc và dọn dẹp. Siêu dữ liệu cài đặt được lưu trữ trong `managed-install.json` bên trong thư mục kỹ năng, ghi lại thông tin như thời gian cài đặt và phiên bản nguồn để cập nhật và kiểm tra sau này. Một số điều đơn giản là có giá trị ghi lại.

### Quy trình cài đặt chi tiết

Cài đặt kỹ năng đòi hỏi một số bước phối hợp. Trên thực tế, nó không đặc biệt phức tạp:

```csharp
public async Task<SkillInstallResultDto> InstallAsync(
    SkillInstallRequestDto request,
    CancellationToken cancellationToken = default)
{
    // 1. Normalize the installation reference
    var normalized = _referenceNormalizer.Normalize(
        request.SkillId,
        request.Source,
        request.SkillSlug,
        request.Version);

    // 2. Check prerequisites
    await _prerequisiteChecker.CheckAsync(cancellationToken);

    // 3. Acquire installation lock
    using var installLock = await _lockProvider.AcquireAsync(normalized.SkillId);

    // 4. Execute installation command
    var result = await _installCommandRunner.ExecuteAsync(
        new SkillInstallCommandExecutionRequest
        {
            Command = $"npx skills add {normalized.FullReference} -g -y",
            Timeout = TimeSpan.FromMinutes(4)
        },
        cancellationToken);

    // 5. Persist installation metadata
    await _metadataStore.WriteAsync(normalized.SkillPath, request);

    return new SkillInstallResultDto { Success = result.Success };
}
```

Một số mẫu thiết kế chính được sử dụng ở đây: **bộ chuẩn hóa tham chiếu** chuyển đổi các định dạng đầu vào khác nhau, chẳng hạn như `tanweai/pua` và `@opencode/docker-skill`, thành một đại diện nội bộ thống nhất; **cơ chế khóa cài đặt** đảm bảo mỗi lần chỉ có một thao tác cài đặt có thể chạy cho cùng một kỹ năng; và **đầu ra phát trực tuyến** đẩy tiến trình cài đặt lên giao diện người dùng theo thời gian thực thông qua Sự kiện do máy chủ gửi, do đó người dùng có thể xem nhật ký giống như thiết bị đầu cuối khi chúng diễn ra.

Cuối cùng, tất cả các mẫu này đều có một mục đích: giữ cho hệ thống dễ sử dụng và bảo trì hơn.

## Khả năng cốt lõi 2: Tìm kiếm trên thị trường

Tìm kiếm trên thị trường cho phép người dùng khám phá và cài đặt các kỹ năng từ cộng đồng. Khả năng của một người luôn có hạn; kiến thức tập thể tiến xa hơn nhiều.

### Phương pháp kỹ thuật

Tính năng tìm kiếm dựa trên API trực tuyến `https://api.hagicode.com/v1/skills/search`. Để cải thiện tốc độ phản hồi, hệ thống triển khai bộ nhớ đệm. Bộ nhớ đệm hơi giống bộ nhớ: nếu bạn giữ những thứ hữu ích xung quanh, bạn không cần phải suy nghĩ nhiều vào lần sau.

```csharp
private async Task<IReadOnlyList<SkillGallerySkillDto>> SearchCatalogAsync(
    string query,
    CancellationToken cancellationToken,
    IReadOnlySet<string>? allowedSources = null)
{
    var cacheKey = $"skill_search:{query}:{string.Join(",", allowedSources ?? Array.Empty<string>())}";

    if (_memoryCache.TryGetValue(cacheKey, out var cached))
        return (IReadOnlyList<SkillGallerySkillDto>)cached!;

    var response = await _onlineApiClient.SearchAsync(
        new SearchSkillsRequest
        {
            Query = query,
            Limit = _options.LimitPerQuery,
        },
        cancellationToken);

    var results = response.Skills
        .Where(skill => allowedSources is null || allowedSources.Contains(skill.Source))
        .Select(skill => new SkillGallerySkillDto { ... })
        .ToList();

    _memoryCache.Set(cacheKey, results, TimeSpan.FromMinutes(10));
    return results;
}
```

Kết quả tìm kiếm hỗ trợ lọc theo nguồn đáng tin cậy nên người dùng chỉ nhìn thấy những nguồn kỹ năng mà họ tin tưởng. Các truy vấn hạt giống như `popular` và `recent` được sử dụng để khởi tạo danh mục, cho phép người dùng xem các kỹ năng phổ biến được đề xuất ngay lần đầu tiên họ mở nó. Ấn tượng đầu tiên vẫn quan trọng.

## Khả năng cốt lõi 3: Khuyến nghị thông minh

Khuyến nghị thông minh là phần phức tạp nhất của hệ thống. Họ có thể tự động đề xuất các kỹ năng phù hợp nhất dựa trên bối cảnh dự án hiện tại. Dù phức tạp nhưng nó vẫn đáng để xây dựng.

### Luồng khuyến nghị

Luồng đề xuất đầy đủ được chia thành năm giai đoạn:

```
1. Build project context
   ↓
2. AI generates search queries
   ↓
3. Search the online catalog in parallel
   ↓
4. AI ranks the candidates
   ↓
5. Return the recommendation list
```

Đầu tiên, hệ thống phân tích các đặc điểm như nhóm công nghệ, ngôn ngữ lập trình và cấu trúc miền của dự án để xây dựng "hồ sơ dự án". Hồ sơ đó hơi giống một sơ yếu lý lịch, ghi lại những đặc điểm chính của dự án.

Sau đó, AI Grain được sử dụng để tạo các truy vấn tìm kiếm được nhắm mục tiêu. Thiết kế này thực sự khá thú vị: thay vì trực tiếp hỏi AI: "Tôi nên giới thiệu những kỹ năng nào?", trước tiên chúng tôi yêu cầu nó suy nghĩ về "Những cụm từ tìm kiếm nào có khả năng tìm thấy các kỹ năng liên quan?" Đôi khi cách bạn đặt câu hỏi quan trọng hơn chính câu trả lời:

```csharp
var queryGeneration = await aiGrain.GenerateSkillRecommendationQueriesAsync(
    projectContext,      // Project context
    locale,              // User language preference
    maxQueries,           // Maximum number of queries
    effectiveSearchHero); // AI model selection
```

Tiếp theo, các truy vấn tìm kiếm đó được thực hiện song song để thu thập danh sách kỹ năng của ứng viên. Cuối cùng, xử lý song song chỉ là một cách để tiết kiệm thời gian.

Cuối cùng, một AI Grain khác xếp hạng các kỹ năng của ứng viên. Bước này xem xét các yếu tố như mức độ liên quan của kỹ năng với dự án, trạng thái tin cậy và sở thích lịch sử của người dùng:

```csharp
var ranking = await aiGrain.RankSkillRecommendationsAsync(
    projectContext,
    candidates,
    installedSkillNames,
    locale,
    maxRecommendations,
    effectiveRankingHero);

response.Items = MergeRecommendations(projectContext, candidates, ranking, maxRecommendations);
```

### Cơ chế dự phòng

Các mô hình AI có thể phản hồi chậm hoặc tạm thời không khả dụng. Ngay cả những hệ thống tốt nhất đôi khi cũng vấp ngã. Vì lý do đó, hệ thống bao gồm một cơ chế dự phòng xác định: khi dịch vụ AI không khả dụng, nó sẽ sử dụng thuật toán phỏng đoán dựa trên quy tắc để tạo ra các đề xuất, chẳng hạn như suy ra các kỹ năng có thể được yêu cầu từ các phần phụ thuộc trong `package.json`.

Nói một cách dễ hiểu, cơ chế dự phòng này chỉ đơn giản là một phương án dự phòng cho hệ thống.

## Năng lực cốt lõi 4: Quản lý nhà cung cấp đáng tin cậy

Quản lý nhà cung cấp đáng tin cậy cho phép người dùng kiểm soát nguồn kỹ năng nào được coi là đáng tin cậy. Niềm tin vẫn là thứ mà người dùng có thể tự xác định.

### Quy tắc phù hợp

Các nhà cung cấp đáng tin cậy hỗ trợ hai quy tắc khớp: **khớp chính xác** (`exact`) và **khớp tiền tố** (`prefix`).

```csharp
public static TrustedSkillProviderResolutionSnapshot Resolve(
    TrustedSkillProviderSnapshot snapshot,
    string source)
{
    var normalizedSource = Normalize(source);

    foreach (var entry in snapshot.Entries.OrderBy(e => e.SortOrder))
    {
        if (!entry.IsEnabled) continue;

        foreach (var rule in entry.MatchRules)
        {
            bool isMatch = rule.MatchType switch
            {
                TrustedSkillProviderMatchRuleType.Exact
                    => string.Equals(normalizedSource, Normalize(rule.Value),
                        StringComparison.OrdinalIgnoreCase),
                TrustedSkillProviderMatchRuleType.Prefix
                    => normalizedSource.StartsWith(Normalize(rule.Value) + "/",
                        StringComparison.OrdinalIgnoreCase),
                _ => false
            };

            if (isMatch)
                return new TrustedSkillProviderResolutionSnapshot
                {
                    IsTrustedSource = true,
                    ProviderId = entry.ProviderId,
                    DisplayName = entry.DisplayName
                };
        }
    }

    return new TrustedSkillProviderResolutionSnapshot { IsTrustedSource = false };
}
```

Các nhà cung cấp đáng tin cậy được tích hợp sẵn bao gồm các tổ chức và dự án nổi tiếng như Vercel, Azure, anthropics, Microsoft và sử dụng trình duyệt. Bạn có thể thêm nhà cung cấp tùy chỉnh thông qua tệp cấu hình bằng cách chỉ định ID nhà cung cấp, tên hiển thị, nhãn huy hiệu, quy tắc so khớp, v.v. Thế giới đủ rộng lớn nên chỉ tin tưởng vào một vài phần mềm tích hợp sẽ không bao giờ là đủ.

### Thực hiện kiên trì

Cấu hình đáng tin cậy được duy trì bằng cách sử dụng Orleans Grain:

```csharp
public class TrustedSkillProviderGrain : Grain<TrustedSkillProviderState>,
    ITrustedSkillProviderGrain
{
    public async Task UpdateConfigurationAsync(TrustedSkillProviderSnapshot snapshot)
    {
        State.Snapshot = snapshot;
        await WriteStateAsync();
    }

    public Task<TrustedSkillProviderSnapshot> GetConfigurationAsync()
    {
        return Task.FromResult(State.Snapshot);
    }
}
```

Lợi ích của phương pháp này là các thay đổi cấu hình được tự động đồng bộ hóa trên tất cả các nút mà không cần phải làm mới bộ đệm theo cách thủ công. Tự động hóa cuối cùng là giúp mọi người bớt lo lắng hơn.

## Thiết kế kỹ thuật chính

### Mẫu bộ chuyển đổi thực thi lệnh

Hệ thống Skill cần thực thi các lệnh npx khác nhau. Nếu logic đó nằm rải rác khắp nơi, mã sẽ nhanh chóng trở nên khó bảo trì. Đó là lý do tại sao chúng tôi thiết kế giao diện bộ chuyển đổi. Cuối cùng, các mẫu thiết kế tồn tại để làm cho mã dễ bảo trì hơn:

```csharp
public interface ISkillInstallCommandRunner
{
    Task<SkillInstallCommandExecutionResult> ExecuteAsync(
        SkillInstallCommandExecutionRequest request,
        CancellationToken cancellationToken = default);
}
```

Các lệnh khác nhau có cách triển khai bộ thực thi khác nhau, nhưng tất cả chúng đều triển khai cùng một giao diện, khiến việc kiểm tra và thay thế trở nên đơn giản.

### Đầu ra phát trực tuyến SSE

Tiến trình cài đặt được đẩy lên giao diện người dùng theo thời gian thực thông qua Sự kiện do máy chủ gửi:

```csharp
public async Task InstallWithProgressAsync(
    SkillInstallRequestDto request,
    IServerStreamWriter<SkillInstallProgressEventDto> stream,
    CancellationToken cancellationToken)
{
    var process = new Process
    {
        StartInfo = new ProcessStartInfo
        {
            FileName = "npx",
            Arguments = $"skills add {request.FullReference} -g -y",
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false
        }
    };

    process.OutputDataReceived += async (sender, e) =>
    {
        await stream.WriteAsync(new SkillInstallProgressEventDto
        {
            EventType = "output",
            Data = e.Data ?? string.Empty
        });
    };

    process.Start();
    process.BeginOutputReadLine();
    await process.WaitForExitAsync(cancellationToken);
}
```

Ở giao diện người dùng, người dùng có thể thấy đầu ra giống như thiết bị đầu cuối trong thời gian thực, điều này mang lại trải nghiệm rất trực quan. Phản hồi theo thời gian thực giúp mọi người cảm thấy thoải mái.

## Hướng dẫn thực hành

### Cài đặt kỹ năng cộng đồng

Hãy cài đặt `pua` kỹ năng làm ví dụ (đó là một kỹ năng cộng đồng phổ biến):

1. Mở ngăn Kỹ năng và chuyển sang ngăn `Skill Gallery` tab
2. Nhập `pua` trong hộp tìm kiếm
3. Bấm vào kết quả tìm kiếm để xem chi tiết kỹ năng
4. Bấm vào `Install` nút
5. Chuyển sang `Local Skills` tab để xác nhận cài đặt thành công

Lệnh cài đặt là `npx skills add tanweai/pua -g -y`và hệ thống sẽ tự động xử lý tất cả các chi tiết. Thực sự không có nhiều bước khi bạn thực hiện từng bước một.

### Thêm nguồn đáng tin cậy tùy chỉnh

Nếu nhóm của bạn có kho lưu trữ kỹ năng riêng, bạn có thể thêm kho lưu trữ đó làm nguồn đáng tin cậy:

```yaml
providerId: "my-team"
displayName: "My Team Skills"
badgeLabel: "MyTeam"
isEnabled: true
sortOrder: 100
matchRules:
  - matchType: "prefix"
    value: "my-team/"
  - matchType: "exact"
    value: "my-team/special-skill"
```

Bằng cách này, tất cả các kỹ năng trong nhóm của bạn sẽ hiển thị huy hiệu đáng tin cậy, giúp người dùng thoải mái hơn khi cài đặt chúng. Nhãn và tín hiệu giúp mọi người cảm thấy tự tin hơn.

### Khái niệm cơ bản về phát triển kỹ năng

Tạo một kỹ năng tùy chỉnh yêu cầu cấu trúc sau:

```
my-skill/
├── SKILL.md          # Skill metadata (YAML front matter)
├── index.ts          # Skill entry point
├── agents/           # Supported agent configuration
└── references/       # Reference resources
```

Một ví dụ `SKILL.md` định dạng:

```yaml
---
name: my-skill
description: A brief description of what this skill does
---

# My Skill

Detailed documentation...
```

## Ghi chú

1. **Yêu cầu về mạng**: tìm kiếm và cài đặt kỹ năng yêu cầu quyền truy cập vào `api.hagicode.com` và đăng ký npm
2. **Phiên bản Node.js**: Nên sử dụng Node.js 18 trở lên
3. **Yêu cầu về quyền**: cần có quyền cài đặt npm toàn cầu
4. **Kiểm soát đồng thời**: mỗi lần chỉ có một thao tác cài đặt hoặc gỡ cài đặt có thể chạy cho cùng một kỹ năng
5. **Cài đặt thời gian chờ**: thời gian chờ cài đặt mặc định là 4 phút, nhưng các trường hợp phức tạp có thể cần điều chỉnh

Cuối cùng, những ghi chú này tồn tại để giúp mọi việc diễn ra suôn sẻ.
