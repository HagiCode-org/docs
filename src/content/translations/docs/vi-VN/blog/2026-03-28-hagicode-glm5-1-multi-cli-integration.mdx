## Hướng dẫn tích hợp Hagicode và GLM-5.1 Multi-CLI

:::note[Thông báo bối cảnh thời gian]
Bài viết này ghi lại thiết kế tích hợp multi-CLI và ánh xạ mô hình như chúng đã tồn tại tại thời điểm xuất bản. Để biết phạm vi luôn được hỗ trợ hiện tại, hãy tham khảo [Cài đặt AI Agent CLI](/vi-VN/related-software-installation/ai-agent-cli/) và [Hướng dẫn cài đặt](/vi-VN/installation); cái `IFlowCli` được đề cập ở đây chỉ thể hiện cấu hình lịch sử tại thời điểm đó và không nằm trong phạm vi hỗ trợ hiện hoạt.
:::

## Nền

Trong dự án Hagicode, người dùng có thể chọn từ nhiều công cụ CLI để điều khiển trợ lý lập trình AI, bao gồm Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI, v.v. Bản thân các công cụ CLI này là các công cụ lập trình AI có mục đích chung nhưng thông qua lớp trừu tượng của Hagicode, chúng có thể kết nối linh hoạt với các nhà cung cấp mô hình AI khác nhau.

Zhipu AI (ZAI) cung cấp giao diện tương thích với API Anthropic Claude, cho phép các công cụ CLI này sử dụng trực tiếp các mô hình dòng GLM trong nước. Trong số đó, GLM-5.1 là phiên bản mô hình ngôn ngữ lớn mới nhất của Zhipu, với những cải tiến đáng kể so với GLM-5.0.

## Kiến trúc trừu tượng CLI của Hagicode

Hagicode xác định 11 loại nhà cung cấp CLI thông qua `AIProviderType` enum, bao gồm các công cụ CLI lập trình AI chính thống:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Mỗi CLI có cấu hình tham số mô hình tương ứng và hỗ trợ `model` và `reasoning` thông số:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],
        [AIProviderType.CodexCli] = ["model", "reasoning"],
        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],
        [AIProviderType.HermesCli] = ["model", "reasoning"],
        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini does not support the reasoning parameter
        // ...
    };
```

## Hệ thống hỗ trợ mô hình GLM

Danh mục ngành nghề phụ của Hagicode xác định sự hỗ trợ hoàn chỉnh cho dòng mô hình GLM:

| Mã mẫu | Tên | Lý do mặc định | Các dòng CLI tương thích |
|----------|------|-------------------|-------------------------|
| `glm-4.7` | GLM 4.7 | cao | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | cao | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM 5 Turbo | cao | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (Cũ) | cao | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | cao | claude, codebuddy, hermes, qoder, kiro |

## Sử dụng GLM-5.1 trong các CLI khác nhau

### 1. Mã Claude CLI + GLM-5.1

Claude Code CLI là một trong những CLI cốt lõi của Hagicode và được cấu hình thông qua hệ thống cấu hình Hero:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

tương ứng `HeroEquipmentCatalogItem` cấu hình:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. Mã mở CLI + GLM-5.1

OpenCode CLI là CLI linh hoạt nhất và hỗ trợ chỉ định bất kỳ mô hình nào trong `provider/model` định dạng:

**Phương pháp 1: Sử dụng tiền tố nhà cung cấp ZAI**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Phương pháp 2: Sử dụng trực tiếp ID mẫu máy**
```json
{
  "model": "glm-5.1"
}
```

**Phương pháp 3: Giao diện người dùng cấu hình giao diện người dùng**

trong `HeroModelEquipmentForm.tsx`, OpenCode CLI có gợi ý giữ chỗ dành riêng:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Người dùng có thể nhập:
```
zai/glm-5.1
glm-5.1
```

**Logic phân tích mô hình OpenCode CLI**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI có logic xử lý kế thừa chuyên dụng:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Lưu ý**: Codebuddy giữ lại cách xử lý đặc biệt cho GLM-5.0 và không sử dụng chuẩn hóa cũ:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## Ưu điểm cải tiến của GLM-5.1

So với GLM-5.0, GLM-5.1 mang lại những cải tiến đáng kể sau:

### 1. Khả năng suy luận tốt hơn

Theo thông tin phát hành chính thức của Zhipu, những cải tiến trong GLM-5.1 bao gồm:

- **Hiểu mã mạnh mẽ hơn**: Phân tích chính xác hơn các cấu trúc mã phức tạp
- **Hiểu ngữ cảnh dài hơn**: Hỗ trợ ngữ cảnh hội thoại dài hơn
- **Gọi công cụ nâng cao**: Tỷ lệ thành công cao hơn cho các lệnh gọi công cụ MCP
- **Độ ổn định đầu ra**: Giảm tính ngẫu nhiên và ảo giác

### 2. Khả năng tương thích đa CLI toàn diện

GLM-5.1 bao gồm tất cả các CLI chính thống được Hagicode hỗ trợ:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Ghi chú

### 1. Cấu hình khóa API

Hãy chắc chắn rằng `ANTHROPIC_AUTH_TOKEN` biến môi trường được đặt chính xác. Đây là thông tin xác thực bắt buộc để mọi CLI kết nối với mô hình.

### 2. Tính sẵn có của mẫu mã

GLM-5.1 cần được kích hoạt bởi nhà cung cấp mô hình tương ứng:
- Nền tảng Zhipu AI ZAI hỗ trợ nó theo mặc định
- Alibaba Cloud DashScope có thể yêu cầu một ứng dụng riêng

### 3. Định dạng OpenCode CLI

Khi sử dụng `provider/model` định dạng, hãy đảm bảo ID nhà cung cấp là chính xác:
- Zhipu AI: `zai` hoặc `zhipuai`
- Đám mây của Alibaba: `aliyun` hoặc `dashscope`

### 4. Tham số lý luận

- `high` được khuyến nghị để có kết quả tạo mã tốt nhất
- Gemini CLI không hỗ trợ tham số lý luận và sẽ tự động bỏ qua cấu hình này

## Kết luận

Thông qua lớp trừu tượng thống nhất, Hagicode cho phép tích hợp linh hoạt giữa GLM-5.1 và nhiều CLI. Các nhà phát triển có thể chọn công cụ CLI phù hợp nhất với sở thích và tình huống sử dụng của mình, sau đó sử dụng mẫu GLM-5.1 mới nhất thông qua cấu hình đơn giản.

Là phiên bản mẫu mới nhất của Zhipu, GLM-5.1 mang đến những cải tiến rõ ràng so với GLM-5.0:
- Mã định danh phiên bản độc lập không có gánh nặng kế thừa
- Lý luận và hiểu mã mạnh mẽ hơn
- Khả năng tương thích đa CLI rộng
- Cấu hình mức lý luận linh hoạt

Với các biến môi trường chính xác và thiết bị Hero được định cấu hình, người dùng hoàn toàn có thể khai thác sức mạnh của GLM-5.1 trên các môi trường CLI khác nhau.
