## Gõ chậm hơn nói và nói chậm hơn ảnh chụp màn hình - Thực hành nhập liệu đa phương thức dành cho trợ lý mã hóa AI

> Viết mã có giới hạn tốc độ cho dù bạn gõ nhanh đến đâu. Đôi khi điều gì đó bạn có thể nói trong một câu sẽ mất rất nhiều thời gian để viết ra; đôi khi một ảnh chụp màn hình giải thích mọi thứ nhưng bạn vẫn phải mô tả nó bằng một đống văn bản. Bài viết này nói về những gì chúng tôi gặp phải khi xây dựng [Mã Hagi](https://hagicode.com), từ nhận dạng giọng nói đến tải hình ảnh lên. Cuối cùng, chúng tôi chỉ muốn làm cho trợ lý mã hóa AI dễ sử dụng hơn một chút.

## Nền

Trong khi xây dựng HagiCode, chúng tôi nhận thấy một vấn đề - hay nói đúng hơn là một vấn đề tự nhiên xuất hiện khi mọi người bắt đầu sử dụng nó nhiều: chỉ dựa vào việc gõ phím có thể mệt mỏi.

Hãy suy nghĩ về điều này: sự tương tác giữa người dùng và Tác nhân là một kịch bản cốt lõi. Nhưng nếu mỗi lần trao đổi đều yêu cầu gõ bàn phím không ngừng nghỉ thì hiệu quả không cao:

1. **Nhập quá chậm**: Đối với các vấn đề phức tạp, như thông báo lỗi hoặc sự cố giao diện người dùng, việc nhập mọi thứ có thể mất nửa phút, trong khi nói to có thể mất 10 giây. Khoảng cách đó là có thật.

2. **Hình ảnh trực tiếp hơn**: Đôi khi giao diện người dùng báo lỗi, đôi khi bạn muốn so sánh bản phác thảo thiết kế và đôi khi bạn cần hiển thị cấu trúc mã. “Một bức tranh đáng giá ngàn lời nói” có thể là một câu nói cổ nhưng nó vẫn đúng. Để AI trực tiếp “nhìn thấy” vấn đề rõ ràng hơn nhiều so với việc mô tả nó từ lâu.

3. **Tương tác phải tự nhiên**: Trợ lý AI hiện đại phải hỗ trợ văn bản, giọng nói và hình ảnh. Người dùng có thể chọn bất kỳ phương thức nhập nào cảm thấy tự nhiên nhất.

Vì vậy, chúng tôi quyết định bổ sung tính năng hỗ trợ nhận dạng giọng nói và tải hình ảnh lên HagiCode để giúp các tương tác của Đại lý thuận tiện hơn. Nếu người dùng có thể gõ ít hơn một chút thì đó đã là một chiến thắng.

## Phân tích

### Những thách thức kỹ thuật trong nhận dạng giọng nói

Khi xây dựng tính năng nhận dạng giọng nói, chúng tôi đã gặp phải một vấn đề phức tạp: **API WebSocket của trình duyệt không hỗ trợ tiêu đề HTTP tùy chỉnh**.

Dịch vụ nhận dạng giọng nói mà chúng tôi chọn là API nhận dạng giọng nói Doubao của ByteDance. Thật không may, API này yêu cầu thông tin xác thực như `accessToken` và `secretKey` được chuyển qua các tiêu đề HTTP. Điều đó tạo ra xung đột kỹ thuật ngay lập tức:

```javascript
// The browser WebSocket API does not support this approach
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // Not supported
  }
});
```

Về cơ bản chúng tôi có hai lựa chọn:

1. **Phương pháp tham số truy vấn URL**: đưa thông tin xác thực vào URL
   - Ưu điểm là thực hiện đơn giản
   - Nhược điểm là thông tin xác thực được hiển thị ở giao diện người dùng, điều này không an toàn; một số API cũng yêu cầu xác thực dựa trên tiêu đề

2. **Phương pháp proxy phụ trợ**: triển khai proxy WebSocket trên phần phụ trợ
   - Ưu điểm là thông tin xác thực vẫn được lưu trữ an toàn trên phần phụ trợ và giải pháp hoàn toàn tương thích với các yêu cầu API
   - Nhược điểm là việc thực hiện phức tạp hơn một chút

Cuối cùng, chúng tôi đã chọn **phương pháp proxy phụ trợ**. Bảo mật không phải là thứ bạn thỏa hiệp.

### Yêu cầu chức năng để tải lên hình ảnh

Yêu cầu của chúng tôi đối với việc tải lên hình ảnh thực sự khá đơn giản:

1. **Nhiều phương thức tải lên**: nhấp để chọn tệp, kéo và thả và dán từ bảng ghi tạm
2. **Xác thực tệp**: hạn chế về loại (PNG, JPG, WebP, GIF) và giới hạn kích thước (5-10 MB) là các yêu cầu cơ bản
3. **Trải nghiệm người dùng**: tiến trình tải lên, bản xem trước và thông báo lỗi để người dùng luôn biết điều gì đang xảy ra
4. **Bảo mật**: xác thực phía máy chủ và bảo vệ khỏi việc tải lên tệp độc hại là điều cần thiết

## Giải pháp

### Nhận dạng giọng nói: Kiến trúc proxy WebSocket

Chúng tôi đã thiết kế kiến trúc ba lớp để nhận dạng giọng nói và tìm ra cách hoạt động:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**Triển khai thành phần cốt lõi**:

1. **Bộ xử lý giao diện người dùng AudioWorklet**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Resample to 16 kHz (required by the Doubao API)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Accumulate samples into 500 ms chunks
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Convert to 16-bit PCM and send
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **Trình xử lý WebSocket phụ trợ** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **Giao diện người dùng `VoiceTextArea` thành phần**:

```tsx
export const VoiceTextArea = forwardRef<HTMLTextAreaElement, VoiceTextAreaProps>(
  ({ value, onChange, onTextRecognized, maxDuration }, ref) => {
    const { isRecording, interimText, volume, duration, startRecording, stopRecording } =
      useVoiceRecording({ onTextRecognized, maxDuration });

    return (
      <div className="flex gap-2">
        {/* Voice button */}
        <button onClick={handleButtonClick}>
          {isRecording ? <VolumeWaveform volume={volume} /> : <Mic />}
        </button>
        {/* Text input area */}
        <textarea value={displayValue} onChange={handleChange} />
      </div>
    );
  }
);
```

### Tải lên hình ảnh: Thành phần tải lên đa phương thức

Chúng tôi đã xây dựng thành phần tải lên hình ảnh đầy đủ tính năng với sự hỗ trợ cho cả ba phương pháp tải lên, bao gồm các tình huống phổ biến nhất mà người dùng gặp phải.

**Các tính năng cốt lõi**:

1. **Ba phương thức tải lên**:

```tsx
// Click to upload
const handleClick = () => fileInputRef.current?.click();

// Drag-and-drop upload
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Clipboard paste
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **Xác thực giao diện người dùng**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **Trình xử lý tải lên phụ trợ** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Validation
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Save file
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## Hướng dẫn thực hành

### Cách sử dụng tính năng nhận dạng giọng nói

1. **Định cấu hình dịch vụ nhận dạng giọng nói**:
   - Mở trang cài đặt nhận dạng giọng nói
   - Định cấu hình Doubao Speech `AppId` và `AccessToken`
   - Tùy chọn định cấu hình từ nóng để cải thiện độ chính xác nhận dạng cho các thuật ngữ dành riêng cho tên miền

2. **Sử dụng nó trong hộp nhập liệu**:
   - Nhấp vào biểu tượng micrô ở bên trái hộp nhập liệu
   - Bắt đầu nói sau khi hoạt ảnh dạng sóng xuất hiện
   - Bấm vào biểu tượng một lần nữa để dừng ghi
   - Văn bản được nhận dạng sẽ tự động được chèn vào vị trí con trỏ

3. **Ví dụ về cấu hình từ nóng**:

```text
TypeScript
React
useState
useEffect
```

### Cách sử dụng tải lên hình ảnh

1. **Phương pháp tải lên**:
   - Bấm vào nút tải lên để chọn tập tin
   - Kéo hình ảnh trực tiếp vào khu vực tải lên
   - sử dụng `Ctrl+V` để dán ảnh chụp màn hình từ bảng nhớ tạm

2. **Các định dạng được hỗ trợ**: PNG, JPG, JPEG, WebP, GIF
3. **Giới hạn kích thước**: 5 MB theo mặc định (có thể định cấu hình)

### Ghi chú

1. **Nhận dạng giọng nói**:
   - Cần có sự cho phép của micrô
   - Sử dụng trong môi trường yên tĩnh khi có thể
   - Thời lượng ghi được hỗ trợ tối đa theo mặc định là 300 giây (có thể định cấu hình)

2. **Tải lên hình ảnh**:
   - Chỉ hỗ trợ các định dạng hình ảnh phổ biến
   - Chú ý đến giới hạn kích thước tập tin
   - Hình ảnh được tải lên tự động nhận được URL xem trước

3. **Các cân nhắc về bảo mật**:
   - Thông tin nhận dạng giọng nói được lưu trữ ở phần phụ trợ
   - Việc tải lên hình ảnh phải trải qua quá trình xác thực nghiêm ngặt phía máy chủ
   - HTTPS/WSS được khuyến nghị trong môi trường sản xuất
