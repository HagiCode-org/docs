Para los usuarios de Windows, Windows Store es actualmente el punto de entrada público del cliente de escritorio. Esta página cubre solo esa ruta Store. Para la guía general de descarga, gestión de versiones y solución de problemas de Desktop, continúa con [Desktop Installation](/es-ES/installation/desktop).

La entrada principal de Steam ya fue retirada, así que Windows Store es ahora el canal principal para Windows. Para más contexto, consulta [Why HagiCode is not available on Steam](/es-ES/faq/steam-distribution-status/).

:::tip[Este camino encaja bien si]
- usas Windows 10 o Windows 11 y quieres instalar directamente desde la tienda del sistema
- prefieres recibir el paquete de la aplicación desde la Store en lugar de manejar instaladores manualmente
- solo necesitas un entorno local personal y no planeas un despliegue con Docker Compose
:::

## Antes de empezar

- el dispositivo debe poder abrir la [página de Windows Store de HagiCode for Windows](https://apps.microsoft.com/detail/9N3PM0N3SVDW)
- debe haber red disponible en el primer arranque porque la aplicación todavía puede preparar componentes adicionales
- algunos dispositivos, regiones o políticas de organización pueden pedir que inicies sesión antes con una cuenta Microsoft
- si el acceso a la Store está bloqueado en tu máquina, cambia a [Desktop Installation](/es-ES/installation/desktop)

## Instalar desde Windows Store

1. Abre [HagiCode for Windows](https://apps.microsoft.com/detail/9N3PM0N3SVDW).
2. Haz clic en `Obtener`, `Instalar` o la acción equivalente que muestre la Store.
3. Espera a que Windows Store termine la descarga y la instalación.
4. Después inicia HagiCode desde el menú Inicio o directamente desde la página de la Store.

## Qué ocurre en el primer arranque

Después de la instalación, HagiCode todavía continúa con el flujo estándar de inicialización de Desktop.

- el primer arranque puede parecer más lento porque aún debe prepararse el entorno local de ejecución
- el canal Windows Store / MSIX bloquea intencionadamente algunos ajustes de gestión de dependencias y rutas de runtime
- la aplicación no se actualiza sola de forma independiente; la entrega del paquete depende del canal de instalación, y en la versión Store queda ligada al canal Windows Store

Si quieres seguir con el onboarding, revisa [Wizard Setup](/es-ES/quick-start/wizard-setup).

## Cuándo usar otro canal

- no puedes acceder a Windows Store o tu organización bloquea la instalación desde la Store
- necesitas descargar manualmente archivos `.exe` o `.msix` y controlar por tu cuenta la fuente del instalador
- vas a desplegar en un servidor, NAS o entorno de equipo, en cuyo caso [Docker Compose Deployment](/es-ES/installation/docker-compose) es mejor opción

## Solución de problemas

### La página de Windows Store no se abre

Primero comprueba la red, la configuración de región y el estado de inicio de sesión de Microsoft Store. Si el enlace del navegador se abre pero el cliente Store sigue sin estar disponible, lo más habitual es una política local del sistema. En ese caso, usa [Desktop Installation](/es-ES/installation/desktop).

### Algunas opciones aparecen bloqueadas

Las instalaciones Store / MSIX bloquean intencionadamente algunas opciones de gestión de dependencias y rutas de datos de runtime. No es un fallo de instalación, sino una restricción normal de este canal de distribución.

### El primer arranque sigue descargando o preparando contenido

El paquete de Store solo entrega el paquete de la aplicación de escritorio. En el primer arranque, HagiCode todavía puede comprobar metadatos de versión, preparar componentes locales o continuar con la inicialización por defecto. Mantén la red disponible y espera a que finalice el paso actual.
