Pokud chcete nainstalovat Hermes a začít jej používat, nejkratší cesta je ve skutečnosti jen tři kroky:

1. Spusťte oficiální instalační příkaz
2. Spusťte CLI ve svém terminálu pomocí `hermes`
3. Pokud jej chcete nadále používat ve Feishu, nakonfigurujte jej `hermes gateway setup`

Tento článek se nesnaží vysvětlit všechny schopnosti Hermes najednou. Místo toho vám pomůže dokončit nejdůležitější začátečnickou smyčku jako první: nainstalovat ji, spustit, začít ji používat a poté ji připojit k jednomu z nejběžnějších scénářů platformy pro zasílání zpráv.

## Co je Hermes a pro koho je určen

Hermes Agent je agent AI, kterého můžete používat buď z místního terminálu, nebo prostřednictvím brány platformy pro zasílání zpráv.

Pro většinu vývojářů má dva společné vstupní body:

- **CLI**: Typ `hermes` ve vašem terminálu a vstoupíte přímo do interaktivního rozhraní.
- **Messaging Gateway**: Spustit `hermes gateway`a poté s ním chatujte z platforem jako Feishu, Telegram, Discord a Slack.

Pokud je vaším cílem právě teď rychle začít, neměňte pořadí. Začněte raději touto cestou:

- Nejprve nainstalujte Hermes
- Nejprve ověřte, zda funguje z CLI
- Poté se rozhodněte, zda chcete připojit platformu pro zasílání zpráv

To usnadňuje diagnostiku problémů a je vhodnější pro osoby, které používají Hermes poprvé.

## Co byste měli vědět před instalací Hermes

Podle Hermes README oficiální cesta rychlé instalace podporuje tato prostředí:

- Linux
- macOS
- WSL2
- Android přes Termux

### Poznámky pro uživatele Windows

Hermes aktuálně nepodporuje spouštění přímo na nativních Windows. Pokud používáte Windows, doporučený přístup je nejprve nainstalovat WSL2 a poté spustit instalační příkaz uvnitř WSL2.

Nejlepší je si to ujasnit hned na začátku, protože mnoho selhání instalace není způsobeno samotným příkazem, ale použitím nepodporovaného běhového prostředí.

## Jak nainstalovat Hermes

Příkaz pro rychlou instalaci uvedený v Hermes README je:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Tento příkaz spouští oficiální instalační skript a zpracovává inicializační kroky specifické pro platformu.

### Po instalaci znovu načtěte svůj shell

Po dokončení instalace nejprve znovu načtěte prostředí prostředí. Nejběžnější příkaz je:

```bash
source ~/.bashrc
```

Pokud používáte zsh, můžete použít:

```bash
source ~/.zshrc
```

### Jak ověřit, zda je Hermes správně nainstalován

Nejpřímější způsob kontroly je spustit:

```bash
hermes
```

Pokud chcete další potvrzení, že vaše konfigurace a závislosti fungují, můžete také spustit:

```bash
hermes doctor
```

`hermes doctor` je zvláště užitečné v těchto situacích:

- Příkaz se po instalaci chová abnormálně
- Konfigurace modelu se nezdařila
- Spuštění brány se nezdařilo
- Nejste si jisti, zda jsou vaše závislosti prostředí úplné

## Jak začít používat Hermes poprvé

### Začněte s CLI

Pokud chcete co nejrychleji potvrdit, že Hermes funguje, nejjednodušší metoda je:

```bash
hermes
```

Tím se spustí interaktivní Hermes CLI. Pro začínající uživatele Hermes je to také nejdoporučovanější výchozí bod, protože si můžete nejprve ověřit to nejpodstatnější:

- Zda je příkaz skutečně dostupný
- Zda aktuální konfigurace modelu funguje správně
- Zda terminálový toolchain funguje správně
- Zda styl interakce odpovídá tomu, co potřebujete

### Tyto příkazy jsou dostatečné pro vaše první kolo nastavení

Hermes README uvádí několik vysokofrekvenčních příkazů a společně tvoří praktickou cestu pro první použití:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Pokud si nejste jisti, co každý z nich dělá, zapamatujte si je takto:

- `hermes model`: výběr nebo změna modelů
- `hermes tools`: zobrazení a konfigurace aktuálně dostupných nástrojů
- `hermes config set`: změna konkrétních položek konfigurace
- `hermes setup`: jednou spusťte úplného průvodce inicializací
- `hermes update`: aktualizace Hermes
- `hermes doctor`: řešení problémů

Pro začátečníky je obvykle nejpraktičtější pořadí:

1. Spustit `hermes model` první
2. Pokud chcete nakonfigurovat všechny běžné možnosti najednou, spusťte `hermes setup`

## Dva nejběžnější způsoby použití Hermes

### 1. Použijte Hermes v terminálu jako denního vývojového asistenta

Režim CLI je vhodný pro tyto scénáře:

- Ptejte se přímo při místním psaní kódu
- Prohlížejte projekty, upravujte soubory a spouštějte příkazy
- Proveďte jednorázové ladění nebo revizi
- Spolupracujte průběžně v aktuálním pracovním adresáři

Jeho největší výhodou je, že je to nejkratší cesta: žádná další integrace platformy, žádná konfigurace botů, které by bylo třeba předem zvládnout, a je to nejlepší způsob, jak si vytvořit první sadu uživatelských návyků.

### 2. Použijte Hermes prostřednictvím platformy pro zasílání zpráv

Pokud chcete chatovat s Hermesem na platformách, jako je Feishu, Telegram nebo Discord, musíte použít bránu pro zasílání zpráv.

Nejběžnější vstupní příkazy jsou:

```bash
hermes gateway setup
hermes gateway
```

Konkrétně:

- `hermes gateway setup` se používá pro interaktivní konfiguraci platformy
- `hermes gateway` se používá ke spuštění procesu brány

Podle oficiální dokumentace je brána jednotný proces na pozadí, který propojuje vaše nakonfigurované platformy, spravuje relace a zpracovává funkce, jako jsou úlohy cron.

## Použití Feishu jako příklad: jak připojit Hermes k platformě pro zasílání zpráv

Pokud většina vaší každodenní práce probíhá ve Feishu, pak je Feishu/Lark velmi přirozeným způsobem, jak používat Hermes.

### Minimální životaschopná integrační cesta

Oficiální dokumentace doporučuje tento vstupní příkaz pro Feishu/Lark:

```bash
hermes gateway setup
```

Po jeho spuštění jednoduše vyberte v průvodci **Feishu / Lark**.

Dokumentace Feishu popisuje dva režimy připojení:

- `websocket`: doporučeno
- `webhook`: volitelné

Pokud Hermes běží na vašem notebooku, pracovní stanici nebo soukromém serveru, použijte `websocket` první je obvykle jednodušší, protože nemusíte zveřejňovat veřejnou adresu URL zpětného volání.

### Pokud to nakonfigurujete ručně, alespoň tyto proměnné znáte

Pokud nepoužíváte průvodce a zapisujete konfiguraci ručně, dokumentace Feishu uvádí tyto základní proměnné:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Dvě z nich si zaslouží zvláštní pozornost:

- `FEISHU_ALLOWED_USERS`: doporučeno, takže ne každý, kdo má přístup k robotovi, ho může použít přímo
- `FEISHU_HOME_CHANNEL`: umožňuje předdefinovat domácí chat pro příjem výsledků cronu nebo výchozích oznámení

### Proč Hermes někdy neodpovídá ve skupinových chatech Feishu

Tento detail lze snadno přehlédnout: ve skupinových chatech Feishu Hermes ve výchozím nastavení neodpovídá na každou zprávu.

Oficiální dokumentace jasně říká:

- V přímých zprávách Hermes odpovídá na zprávy
- Ve skupinových chatech musíte výslovně `@` robot před tím, než zprávu zpracuje

Pokud chcete nastavit konverzaci Feishu jako domovský kanál, můžete to také použít v chatu:

```text
/set-home
```

Nebo jej definujte v konfiguraci předem:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Hermesovy příkazy by si začátečníci měli pamatovat jako první

Ať už používáte Hermes v CLI nebo na platformě pro zasílání zpráv, zapamatování si následujících příkazů již stačí, abyste mohli začít:

- `/new` nebo `/reset`: zahájení nové relace
- `/model`: zobrazení nebo přepnutí modelu
- `/retry`: opakování předchozího odbočení
- `/undo`: vrátit zpět předchozí interakci
- `/compress`: ručně komprimovat kontext
- `/help`: zobrazit nápovědu

Pokud používáte Hermes hlavně na platformě pro zasílání zpráv, zapamatujte si ještě jedno:

- `/sethome` nebo `/set-home`: nastavení aktuálního chatu jako domovského kanálu

Tyto příkazy pokrývají nejběžnější operace v začátečnické fázi: restartování, nastavení, vrácení zpět, kontrola a pokračování.

## Často kladené otázky

### Může Windows nainstalovat Hermes přímo?

Ne. Aktuální oficiální dokumentace jasně uvádí, že nativní Windows nejsou podporovány a doporučuje se WSL2.

### Co mám dělat, když píšu `hermes` po instalaci nic nedělá?

Nejlepší je řešit problémy v tomto pořadí:

1. Nejprve znovu nabijte svůj shell, například pomocí `source ~/.bashrc`
2. Spustit `hermes` znovu
3. Pokud je to stále abnormální, běžte `hermes doctor`

### Proč robot neodpovídá ve skupině Feishu?

Nejprve zkontrolujte tyto tři věci:

- Ať už vy `@` zmínil Hermes ve skupině
- zda `FEISHU_ALLOWED_USERS` omezuje aktuálního uživatele
- Zda aktuální zásady skupinového chatu umožňují zpracování skupinových zpráv

Podle oficiální dokumentace Feishu je ve scénářích skupinového chatu vyžadováno explicitní použití @zmínky.
