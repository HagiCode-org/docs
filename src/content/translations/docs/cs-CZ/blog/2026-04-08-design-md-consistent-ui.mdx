## Design.md: Řešení pro konzistentní návrh uživatelského rozhraní řízeného umělou inteligencí

> Jak můžete v éře vývoje frontendu podporovaného umělou inteligencí udržet konzistentní uživatelské rozhraní generované umělou inteligencí? Tento článek sdílí naše praktické zkušenosti s vytvářením webu designové galerie založené na awesome-design-md, spolu s tím, jak vytvořit strukturovaný `design.md` vést AI ke standardizovanému designu uživatelského rozhraní.

## Pozadí

Každý, kdo používal AI k psaní frontendového kódu, měl pravděpodobně podobnou zkušenost: požádejte AI, aby vygenerovala stejnou stránku několikrát, a každý výsledek vyjde v jiném stylu. Někdy jsou rohy zaoblené, někdy ostré. Někdy je mezera 8 pixelů a jindy je to 16 pixelů. I to samé tlačítko může v různých konverzacích vypadat jinak.

Nejde o izolovaný problém. S tím, jak se vývoj podporovaný umělou inteligencí stává stále běžnějším, se **nedostatek konzistence v rozhraní frontendu generovaného umělou inteligencí** stal rozšířeným problémem. Různí asistenti AI, různé výzvy a dokonce stejný asistent v různých konverzacích mohou vytvářet dramaticky odlišné návrhy rozhraní. To vytváří obrovské náklady na údržbu během iterace produktu.

Hlavní příčina je ve skutečnosti jednoduchá: neexistuje žádný **směrodatný referenční dokument o designu**. Tradiční šablony stylů CSS mohou vývojářům říci, „jak implementovat“, ale nemohou plně sdělit „proč je to navrženo tímto způsobem“ nebo „který návrhový vzor by měl být použit v jakém scénáři“. Pro AI je jasný a strukturovaný popis ještě důležitější pro pochopení konvencí designu.

Zároveň komunita open-source již nabízí několik vynikajících zdrojů. The [VoltAgent/awesome-design-md](https://github.com/VoltAgent/awesome-design-md) projekt shromažďuje dokumentaci projektového systému od mnoha známých společností. Každý adresář obsahuje a `README.md`, a `DESIGN.md`a náhled HTML. To vše je však rozptýleno po upstream úložišti, takže je obtížné rychle procházet a porovnávat.

Můžeme tedy tyto zdroje konsolidovat do snadno procházet galerii návrhů a zároveň destilovat strukturovaný `design.md` pro použití AI?

Odpověď je ano. Dále mi dovolte projít si náš přístup.

## Analýza

Než začnete psát kód, pojďme si rozebrat technické problémy, které stojí za tímto problémem.

### Správa zdrojového obsahu: Jak sjednotíte rozptýlené zdroje návrhu?

Upstream úložiště awesome-design-md obsahuje velké množství návrhových dokumentů, ale potřebovali jsme způsob, jak je přenést do našeho vlastního projektu.

**Řešení: použijte submodul git**

```
awesome-design-md-site
└── vendor/awesome-design-md          # Upstream resources (git submodule)
```

To nám přináší několik výhod:
- Kontrola verzí: můžeme připnout konkrétní upstream verzi
- Offline sestavení: během sestavení není třeba vyžadovat externí rozhraní API
- Kontrola obsahu: konkrétní změny jsou viditelné v PR

### Normalizace dat: Jak standardizujete různé struktury dokumentů?

Různé společnosti strukturují své designové dokumenty odlišně. Některým chybí soubory náhledu a některé používají nekonzistentní názvy. Během procesu sestavování je musíme normalizovat.

**Řešení: skenujte a generujte normalizované záznamy v době sestavení**

Základní modul je `awesomeDesignCatalog.ts`, zodpovědný za:
1. Skenování `vendor/awesome-design-md/design-md/*` adresář
2. Ověření, zda každá položka obsahuje požadované soubory (`README.md`, `DESIGN.md`a alespoň jeden soubor náhledu)
3. Extrahování a vykreslování obsahu Markdown do HTML
4. Generování normalizovaných vstupních dat

### Statická architektura webu: Jak poskytujete dynamické vyhledávání a přitom zůstáváte plně statický?

Protože se jedná o designovou galerii, hledání je nutností. Ale Astro je generátor statických stránek, takže jak implementujete vyhledávání v reálném čase?

**Řešení: Ostrov React + synchronizace parametrů dotazu URL**

```typescript
// src/components/gallery/SearchToolbar.tsx

export function SearchToolbar() {
  const [query, setQuery] = useState('');

  // Sync with the URL
  useEffect(() => {
    const params = new URLSearchParams(window.location.search);
    setQuery(params.get('q') || '');
  }, []);

  // Filter in real time
  const filtered = entries.filter(entry =>
    entry.searchText.includes(query)
  );

  return <input value={query} onChange={e => {
    setQuery(e.target.value);
    updateURL(e.target.value);
  }} />;
}
```

Výhodou tohoto přístupu je, že zachovává nasaditelnost statického webu nedotčenou, což znamená, že jej lze nasadit na libovolnou statickou hostingovou službu, přičemž stále poskytuje okamžité filtrování.

### Návrhová dokumentace: Jak pomáháte umělé inteligenci porozumět a dodržovat designové standardy?

To je jádro celého řešení. Musíme vytvořit strukturovaný `design.md` které umělá inteligence dokáže pochopit a aplikovat.

**Řešení: vypůjčte si strukturu ClickHouse `DESIGN.md`**

[ClickHouse's `DESIGN.md`](https://github.com/ClickHouse/ClickHouse/blob/master/DESIGN.md) je vynikající reference. Zahrnuje:
- Vizuální téma a atmosféra
- Paleta barev a role
- Pravidla typografie
- Styly komponent
- Principy rozvržení
- Hloubka a nadmořská výška
- Co dělat a co nedělat
- Responzivní chování
- Průvodce promptem agenta

Náš přístup je: **znovu použijte strukturu, přepište obsah**. Zachováváme strukturu sekcí ClickHouse `DESIGN.md`, ale nahraďte obsah skutečnými tokeny návrhu a konvencemi komponent používanými v našem vlastním projektu.

## Řešení

Na základě výše uvedené analýzy se naše řešení skládá ze čtyř základních modulů.

### 1. Potrubí pro příjem obsahu

To je základ celého systému, který je zodpovědný za extrahování a normalizaci obsahu z upstreamových zdrojů.

```typescript
// src/lib/content/awesomeDesignCatalog.ts

export async function scanSourceEntries(): Promise<DesignEntry[]> {
  const designDir = 'vendor/awesome-design-md/design-md';
  const entries: DesignEntry[] = [];

  for (const dir of await fs.readdir(designDir)) {
    const entryPath = path.join(designDir, dir);
    if (await isValidDesignEntry(entryPath)) {
      const entry = await normalizeDesignEntry(entryPath);
      entries.push(entry);
    }
  }

  return entries;
}

async function isValidDesignEntry(dir: string): Promise<boolean> {
  const requiredFiles = ['README.md', 'DESIGN.md'];
  for (const file of requiredFiles) {
    if (!(await fileExists(path.join(dir, file)))) {
      return false;
    }
  }
  return true;
}
```

### 2. Rozhraní pro procházení galerie

Rozhraní galerie obsahuje tři hlavní části:

**Domovská stránka**: zobrazuje mřížku karet všech návrhů a každá karta obsahuje:
- Název a shrnutí návrhu návrhu
- Náhled obrázku, je-li k dispozici
- Zvýraznění rychlého vyhledávání

**Stránka s podrobnostmi**: shromažďuje úplné informace pro jednu položku návrhu:
- `README` dokument
- `DESIGN` dokument
- Náhled s přepínáním světlého/tmavého motivu
- Navigace k sousedním záznamům

**Navigace**: podporuje návrat do galerie a procházení sousedních položek

Galerie domovské stránky používá rozložení karet s vysokou hustotou, zploštění `design.md` vstupy z různých zdrojů do jednotného vizuálního rámce, takže týmy mohou rychle porovnávat styly značek, vzory tlačítek a typografický rytmus.

### 3. Funkce vyhledávání

Funkce vyhledávání je založena na filtrování na straně klienta, přičemž stav je zachován prostřednictvím parametrů dotazu adresy URL:

```typescript
// src/components/gallery/SearchToolbar.tsx

function SearchToolbar({ entries }: { entries: DesignEntry[] }) {
  const [query, setQuery] = useState('');
  const [results, setResults] = useState(entries);

  useEffect(() => {
    const params = new URLSearchParams(window.location.search);
    const initialQuery = params.get('q') || '';
    setQuery(initialQuery);
    filterEntries(initialQuery);
  }, []);

  const filterEntries = (searchQuery: string) => {
    const filtered = entries.filter(entry =>
      entry.searchText.toLowerCase().includes(searchQuery.toLowerCase())
    );
    setResults(filtered);
  };

  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const value = e.target.value;
    setQuery(value);
    filterEntries(value);

    // Update the URL without triggering a page refresh
    const newUrl = value
      ? `${window.location.pathname}?q=${encodeURIComponent(value)}`
      : window.location.pathname;
    window.history.replaceState({}, '', newUrl);
  };

  return (
    <div className="search-toolbar">
      <input
        type="text"
        value={query}
        onChange={handleChange}
        placeholder="Search design entries..."
      />
      <span className="result-count">{results.length} results</span>
    </div>
  );
}
```

## Praxe

Nyní, když jsme probrali řešení, jak ho vlastně implementujete?

### Kroky implementace

**Krok 1: Inicializujte submodul**

```bash
# Add the upstream repository as a submodule
git submodule add https://github.com/VoltAgent/awesome-design-md.git vendor/awesome-design-md

# Initialize and update the submodule
git submodule update --init --recursive
```

**Krok 2: Vytvořte kanál obsahu**

Implementovat `awesomeDesignCatalog.ts`, včetně:
- Logika skenování a ověřování souborů
- Markdown rendering pomocí vestavěného rendereru Astro
- Extrakce vstupních dat

**Krok 3: Vytvořte uživatelské rozhraní galerie**

Použijte Astro + React Islands k vytvoření:
- Rozvržení galerie domovské stránky (mřížka karty)
- Designové komponenty karty
- Panel nástrojů vyhledávání
- Podrobné rozložení stránky

**Krok 4: Napište návrhový dokument**

Na základě struktury ClickHouse `DESIGN.md`, vyplňte skutečné designové tokeny z vašeho vlastního projektu. Aktualizovat `README.md` a přidat odkaz na `design.md`.

### Poznámky

**Zabezpečení**: Vykreslování Markdown vyžaduje filtrování nebezpečného HTML. Vestavěné filtry rendereru Astro `script` tagy ve výchozím nastavení, ale stále musíte sledovat rizika XSS.

**Výkon**: Velký počet náhledů prvků iframe může ovlivnit výkon prvního malování. Doporučuje se používat `loading="lazy"` k línému načítání obsahu náhledu.

**Údržba**: `design.md` musí zůstat v souladu s implementací kódu. Doporučuje se přidat kontroly CI, aby bylo zajištěno, že proměnné CSS zůstanou konzistentní mezi dokumentací a kódem.

**Přístupnost**: Ujistěte se, že barevný kontrast odpovídá standardu WCAG AA (alespoň 4,5:1).

### Průvodce používáním AI

Po vytvoření `design.md`, jak přimět AI, aby ji skutečně používala? Zde je několik praktických tipů:

**Tip 1: Ve výzvě na to explicitně uveďte odkaz**

```
Please refer to the design.md file in the project root and use the design conventions defined there to implement the following components:
- Buttons: use the primary color with an 8px border radius
- Cards: use the elevation-2 shadow level
```

**Tip 2: Vyžadujte, aby umělá inteligence odkazovala na konkrétní proměnné CSS**

```
Implement a navigation bar with the following requirements:
- Use --color-bg-primary for the background color
- Use --color-border-subtle for borders
- Use --text-color-primary for text
```

**Tip 3: Zahrnout `design.md` obsah v systémové výzvě**

Pokud váš nástroj AI podporuje vlastní systémové výzvy, můžete přidat základní obsah `design.md` přímo k němu.

### Strategie testování

**Testování obsahu kanálu**:
- Scénáře chybějících souborů (chybějící `README.md` nebo `DESIGN.md`)
- Scénáře chyb formátování (selhání analýzy značky)
- Scénáře s prázdným adresářem

**Testování funkcí vyhledávání**:
- Zpracování prázdných výsledků
- Speciální znaky, jako je čínština a emotikony
- Ověření synchronizace URL

**Testování komponent uživatelského rozhraní**:
- Přepínání světlého/tmavého motivu
- Responzivní rozložení
- Náhled stavů načítání

### Pracovní postup nasazení

```bash

# 2. Rebuild the site

```

Doporučuje se automatizovat aktualizace submodulů plus sestavení a nasazení, aby se CI mohla spouštět automaticky, kdykoli je aktualizováno upstream úložiště.

## Závěr

Nekonzistence v UI generovaných umělou inteligencí, se kterou se HagiCode během vývoje setkala, byla ve své podstatě způsobena nedostatkem strukturovaného referenčního dokumentu pro návrh. Vybudováním stránky designové galerie a vytvořením standardizovaného `design.md`, jsme tento problém úspěšně vyřešili.

Hlavní hodnota tohoto řešení spočívá v:
- **Sjednocené zdroje**: konsolidace rozptýlené dokumentace konstrukčního systému
- **Strukturované standardy**: vyjádření konvencí designu ve formátu, kterému AI rozumí
- **Nepřetržitá údržba**: udržování obsahu v aktuálním stavu `git submodule`

Pokud používáte AI také pro vývoj frontendu, tento přístup stojí za vyzkoušení. Vytvoření strukturovaného `design.md` nejen zlepšuje konzistenci kódu generovaného umělou inteligencí, ale také pomáhá vašemu týmu interně udržovat jednotné standardy návrhu.
