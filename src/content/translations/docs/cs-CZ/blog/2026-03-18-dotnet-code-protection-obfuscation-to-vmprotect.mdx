## Ochrana kódu .NET v praxi: Od zmatku k ochraně virtuálních strojů

> Tento článek vysvětluje, jak implementovat vícevrstvou strategii ochrany kódu v projektech .NET, pokrývající celou cestu od základního zmatku k profesionální ochraně virtuálních strojů.

## Pozadí

Ve vývoji aplikací .NET je ochrana základního kódu, jako je ověřování licencí, obchodní logika a citlivá konfigurace před dekompilací a reverzním inženýrstvím, upřímně řečeno tématem, kterému se nemůžete vyhnout. Jak ekosystém .NET vyspěl, vývojáři získali přístup k řadě možností ochrany, od vestavěných atributů zmatku až po profesionální nástroje ochrany založené na virtualizaci.

Jako komplexní vícejazyčný monorepo projekt zahrnuje HagiCode desktopové aplikace, sestavovací systémy a možnosti správy licencí. Kód nevyhnutelně obsahuje logiku ověření licence, citlivou konfiguraci, jako jsou klíče API a ID produktů, a kritickou obchodní logiku. Tyto části potřebují seriózní ochranu, protože nikdo nechce, aby jejich tvrdá práce byla tak snadno odhalena.

Tento článek sdílí přístup k ochraně kódu, který jsme skutečně přijali v projektu HagiCode, a shrnuje celou cestu od počátečních úskalí k pozdější optimalizaci. Snad vám to dá nějaké užitečné nápady.

## Základní obsah

### 1. Vestavěný atribut zmatku společnosti Microsoft

.NET Framework poskytuje vestavěný `[ObfuscationAttribute]`, což je nejzákladnější a nejběžněji používaná značka obfuskace kódu. Tento atribut žije v `System.Reflection` jmenný prostor a umožňuje vám aplikovat základní ochranu na kód bez zavádění nástrojů třetích stran.

**Základní funkce**:
- `Feature` vlastnost: Určuje funkci zmatku, jako je např `"ultra"` (vysoká mlha) popř `"all"` (plné zmatnění)
- `Exclude` vlastnost: `true` znamená vyloučit z mlžení a `false` prostředky použít zmatek
- Lze použít na třídy, metody, vlastnosti a další členy typu

V projektu HagiCode jej můžete vidět takto:

```csharp
[Obfuscation(Feature = "ultra", Exclude = false)]
public async Task<LicenseValidationResult?> ValidateLicenseAsync(...)
```

**Výhody** tohoto přístupu jsou celkem zřejmé:
- Žádné další závislosti; je zabudován do .NET Framework ihned po vybalení
- Mohou být rozpoznány a zpracovány obfuskačními nástroji třetích stran
- Výrazně nezvětšuje velikost sestavené sestavy

To znamená, že má také omezení. Je to pouze značka a skutečný výsledek zmatku závisí na implementaci nástroje. Nemůže poskytovat zabezpečení na úrovni ochrany virtuálního počítače.

### 2. VMP (Virtual Machine Protection)

VMP je profesionální nástroj na ochranu kódu, který poskytuje ochranu na vysoké úrovni kompilací kódu do instrukcí virtuálního stroje. Na rozdíl od jednoduchého obfuskace názvů VMP ve skutečnosti transformuje logiku kódu do podoby, kterou konvenční dekompilátory nemohou rekonstruovat.

**Klasifikace úrovně ochrany**:

| úroveň | Virtualizace | Mutace | Anti-debugování | Šifrování řetězců | Případy použití |
|------|--------|------|--------|------------|----------|
| VYSOKÝ | plný | vysoká | povoleno | povoleno | Ověření licence, souběžnost relací, citlivé konstanty |
| STŘEDNÍ | částečné | střední | povoleno | povoleno | Obchodní logika, doménové modely |
| NÍZKÁ | žádný | nízká | disabled | disabled | Užitné třídy, nekritický kód |

Projekt HagiCode definuje deklarativní atributový systém pro označení kódu, který potřebuje ochranu:

```csharp
// High-priority protection
[VmProtect(VmProtectionPriority.High, Reason = "Contains license verification logic")]
public class KeygenClient { ... }

// Exclude from protection
[VmExclude(Reason = "Public API that must remain unchanged")]
public class PublicApi { ... }

// Inherited protection
[VmProtect(Priority.High, ProtectDerived = true)]
public class BaseLicenseValidator { ... }
```

### 3. Strategie ochrany po dobu výstavby

Ochrana VMP není důležitá pouze za běhu. Také je potřeba to zautomatizovat jako součást procesu sestavení, protože dělat to ručně by bylo příliš únavné. Sestavovací systém HagiCode podporuje několik režimů:

- **Nativní režim Windows**: Přímo vyvolejte nástroj VMProtect
- **Režim kontejneru Linux Docker**: Spusťte VMP v kontejneru a vyřešte problémy s kompatibilitou mezi platformami
- **Skenování atributů**: Automaticky zjistěte ochranné značky v kódu
- **Mechanismus ověření**: Potvrďte, že ochrana byla úspěšně aplikována

Dohromady tyto schopnosti usnadňují správu procesu.

## Řešení

### 1. Použití vestavěného atributu zmatku společnosti Microsoft

Použít `ObfuscationAttribute` přímo v kódu:

```csharp
using System.Reflection;

[Obfuscation(Feature = "ultra", Exclude = false)]
public class LicenseService
{
    [Obfuscation(Feature = "ultra", Exclude = false)]
    public async Task<bool> ValidateLicenseAsync(string key)
    {
        // License validation logic
    }

    [Obfuscation(Feature = "flow", Exclude = false)]
    private string DecryptToken(string encrypted)
    {
        // Decryption logic
    }
}
```

Někdy je potřeba umožnit testovacím sestavám přístup k interním členům a přitom zachovat zabezpečený produkční kód:

```csharp
// AssemblyInfo.cs
[assembly: InternalsVisibleTo("HagiCode.Application.Tests")]
[assembly: InternalsVisibleTo("DynamicProxyGenAssembly2")] // for Moq
```

Díky tomu je testování mnohem pohodlnější, protože kód je stále potřeba pořádně otestovat.

### 2. Definice vlastních atributů pro ochranu VMP

Vytvořte vlastní atributy ochrany pro řízení chování VMP:

```csharp
using System;

[AttributeUsage(AttributeTargets.Class | AttributeTargets.Method | AttributeTargets.Property)]
public class VmProtectAttribute : Attribute
{
    public VmProtectionPriority Priority { get; set; }
    public string? Reason { get; set; }
    public bool ProtectDerived { get; set; }
}

[AttributeUsage(AttributeTargets.Class | AttributeTargets.Method | AttributeTargets.Property)]
public class VmExcludeAttribute : Attribute
{
    public string? Reason { get; set; }
}

public enum VmProtectionPriority
{
    None = 0,
    Low = 1,
    Medium = 2,
    High = 3
}
```

S vlastními atributy se často pracuje snadněji, protože přímo odrážejí vaše vlastní požadavky na ochranu.

### 3. Konfigurační soubor VMP

```yaml
# vmp_config.yml
protection:
  priority_mode: "attribute"  # Attribute-based priority
  default_level: "medium"

  high:
    virtualization: "full"
    mutation: "high"
    anti_debug: true
    anti_dump: true
    encrypt_strings: true
    encrypt_resources: true

  medium:
    virtualization: "partial"
    mutation: "medium"
    anti_debug: true
    encrypt_strings: true

  low:
    virtualization: "none"
    mutation: "low"
    anti_debug: false
```

Přehlednější konfigurace usnadňuje pozdější údržbu systému.

## Praktický průvodce

### 1. Postupy ochrany kritických součástí

Podle specifikace ochrany kódu HagiCode musí následující součásti používat ochranu s VYSOKOU prioritou:

```csharp
// Production constants - must be encrypted and protected by VMP
[VmProtect(VmProtectionPriority.High, Reason = "Production constants")]
public static class ProductionConstants
{
    // Encrypted string accessor, protected by VMP
    [VmProtect(VmProtectionPriority.High)]
    public static string GetLicenseServerUrl(IOptions<LicenseOptions> options) => ...;
}

// License validation logic
[VmProtect(VmProtectionPriority.High, Reason = "License verification logic")]
public class KeygenClient : IKeygenClient
{
    [Obfuscation(Feature = "ultra", Exclude = false)]
    public async Task<LicenseValidationResult?> ValidateLicenseAsync(...) { ... }
}

// Machine fingerprint service
[VmProtect(VmProtectionPriority.High)]
public class MachineFingerprintService : IMachineFingerprintService { ... }
```

Kritický kód si zaslouží silnější ochranu, protože odhalení základní logiky by způsobilo skutečné problémy.

### 2. String Encryption a Runtime Decryption

Šifrujte řetězce v době sestavení a dešifrujte je za běhu:

```csharp
public static class StringDecryption
{
    [VmProtect(VmProtectionPriority.High, Reason = "CRITICAL SECURITY")]
    public static string DecryptString(byte[] encryptedData, byte[] key, byte[] iv)
    {
        using var aes = Aes.Create();
        aes.Key = key;
        aes.IV = iv;

        using var decryptor = aes.CreateDecryptor();
        using var ms = new MemoryStream(encryptedData);
        using var cs = new CryptoStream(ms, decryptor, CryptoStreamMode.Read);
        using var reader = new StreamReader(cs);

        return reader.ReadToEnd();
    }
}

// Production constant accessor (lazy loading + caching)
public static class ProductionConstants
{
    private static string? _cachedLicenseServerUrl;

    public static string GetLicenseServerUrl(IOptions<LicenseOptions> options)
    {
        if (_cachedLicenseServerUrl == null)
        {
            var encrypted = GetEncryptedLicenseServerUrl();
#if DEBUG
            _cachedLicenseServerUrl = options.Value.PrimaryServer.Url;
#else
            _cachedLicenseServerUrl = StringDecryption.DecryptString(
                encrypted,
                GetEncryptionKey(),
                GetEncryptionIV());
#endif
        }
        return _cachedLicenseServerUrl;
    }
}
```

Tento krok je důležitý, protože citlivé informace by nikdy neměly být ponechány v prostém textu.

### 3. Ověření ochrany VMP

Po sestavení musíte ověřit, zda byla ochrana úspěšně aplikována; jinak si nemůžete být jisti, že to skutečně funguje:

```csharp
// Example verification script
public bool VerifyProtection(string assemblyPath)
{
    // 1. Check the VMP signature
    var bytes = File.ReadAllBytes(assemblyPath);
    var vmpSignature = Encoding.ASCII.GetBytes("VMProtect");
    if (bytes.Any(b => vmpSignature.Contains(b)))
    {
        return true;
    }

    // 2. Check for file size changes (the protected file is usually larger)
    var originalInfo = new FileInfo(assemblyPath.Replace(".dll", ".bak"));
    if (originalInfo.Exists)
    {
        var sizeRatio = (double)new FileInfo(assemblyPath).Length / originalInfo.Length;
        return sizeRatio > 1.1;
    }

    return false;
}
```

Ověření se vždy vyplatí, protože jinak mohou problémy proklouznout bez povšimnutí.

### 4. Poznámky a upozornění

Je zde několik úskalí, která si zaslouží zvláštní pozornost:

1. **Nezamlžujte veškerý kód**: Veřejná rozhraní API, definice rozhraní a třídy DTO obvykle nepotřebují ochranu. Nadměrné mlžení může poškodit výkon a ladění. Projekt HagiCode se to naučil tvrdě.

2. **Chránit přístupové prvky klíče**: Metody, které získávají šifrovací klíče, musí mít stejnou nebo vyšší úroveň ochrany než samotná šifrovaná data; jinak celé nastavení ztratí svou hodnotu.

3. **Testování rovnováhy a produkce**: Sestavení DEBUG by měla přeskočit šifrování, aby se usnadnil vývoj a ladění, zatímco sestavení RELEASE by měla umožnit plnou ochranu. Nezapomeňte je oddělit podmíněnou kompilací jako např `#if DEBUG`.

4. **Zvažte prostředí Docker**: Spuštění VMP na Linuxu vyžaduje kontejnerový přístup, aby byla zajištěna kompatibilita nástrojů. HagiCode používá řešení kontejneru Wine + VMP k vyřešení problému napříč platformami.

5. **Ověření je povinné**: Po dokončení sestavení musíte ověřit, že byla ochrana úspěšně aplikována. V opačném případě může být citlivý kód stále odhalen a výše uvedený ověřovací kód existuje přesně pro tento účel.

## Závěr

S touto strategií vícevrstvé ochrany HagiCode vybudoval komplexní systém zabezpečení kódu, který pokrývá základní zmatení až po ochranu virtuálních strojů:

- **Vrstva 1**: Použijte `ObfuscationAttribute` pro základní značení a poskytovat rady nástrojům třetích stran
- **Vrstva 2**: Použijte vlastní `VmProtectAttribute` prohlášení vyjadřující záměr ochrany a prioritu
- **Vrstva 3**: Použijte ochranu virtuálního stroje VMP k transformaci kritického kódu na nevratné instrukce virtuálního stroje
- **Vrstva 4**: Automaticky skenujte a aplikujte ochranu během sestavení a poté ověřte výsledek

Tento přístup dokáže odolat běžným dekompilačním nástrojům a zároveň lépe obstojí proti pokročilým útokům reverzního inženýrství. Pokud vytváříte aplikaci .NET, která potřebuje ochranu kódu, doufám, že vám to poskytne alespoň užitečný referenční bod.
