## Navrhování 12 výjimečných startovacích zážitků pro HagiCode: Od minimalismu po kyberpunk

> Krátká mezera mezi stažením aplikace React 19 a dokončením Hydration je pro uživatele jedinečnou příležitostí, jak pocítit osobnost vaší značky. V tomto článku sdílíme kompletní systém stylu spouštění, který jsme vytvořili pro projekt HagiCode pomocí HTML/CSS/JS.

## Pozadí

Jako moderní aplikace postavená na ASP.NET Core 10 a React 19 (Vite) používá HagiCode architekturu nasazení oddělenou frontend-backend. Výstup sestavení frontendu je zabalen do backendu `wwwroot/` a hostitelem ASP.NET Core.

Tato architektura však přináší klasický UX bolestivý bod: když uživatelé navštíví stránku, prohlížeč nejprve načte HTML, pak stáhne velký balíček JS a nakonec nechá React provést hydrataci. Během tohoto „období vakua“, které trvá od několika set milisekund do několika sekund, uživatelé vidí buď prázdnou obrazovku, nebo nezáživnou statickou stránku.

Abychom zaplnili tuto mezeru a vtiskli osobnost značky HagiCode, potřebovali jsme navrhnout systém spouštěcího stylu implementovaný výhradně s vloženým kódem uvnitř. `index.html`.

## O HagiCode

Přístup k návrhu úvodní obrazovky sdílený v tomto článku pochází z našich praktických zkušeností v [HagiCode](https://github.com/HagiCode-org/site) projekt. Jako asistent kódování AI se HagiCode stará nejen o efektivitu generování kódu, ale také o vizuální zážitek vývojáře. Tento spouštěcí systém je jedním z výsledků naší snahy o maximální výkon frontendu.

## Základní výzvy a architektonický design

Než jsme začali navrhovat, museli jsme si nejprve ujasnit technická omezení. Protože vše muselo být implementováno inline `index.html`, nemohli jsme načíst žádné externí soubory CSS nebo JS kromě vlastního balíčku Reactu.

### Analýza technických omezení

1.  **Princip nulové závislosti**: Všechny styly musí žít uvnitř a `<style>` a veškerá logika musí žít uvnitř a `<script>` tag.
2.  **Obranný CSS**: Abychom zabránili tomu, aby globální styly znečišťovaly úvodní obrazovku po připojení aplikace React, rozhodli jsme se všechny spouštěcí styly zabalit do předpony ID s vysokou prioritou, jako je `#boot-screen`.
3.  **Výkon především**: Animace by měly používat CSS `transform` a `opacity` kdekoli je to možné, aby se zabránilo přeformátování a zajistilo se, že hlavní vlákno zůstane odblokováno.
4.  **Vizuální konzistence**: Barvy a písma musí zůstat v souladu s konfigurací Tailwind HagiCode.

### Vzor architektury: Shell & Injector

Přijali jsme **vzor varianty**. Základní logika je zapouzdřena uvnitř okamžitě vyvolaného funkčního výrazu (IIFE), zatímco specifická logika vykreslování je vložena prostřednictvím konfigurace. To nám umožňuje přepínat mezi různými styly pomocí jednoduché konfigurace namísto opakovaného přepisování logiky manipulace DOM.

Zde je základní kód architektury:

```html
<!-- Inline in index.html -->
<div id="boot-root"></div>

<script>
(function() {
  const BootSequence = {
    config: {
      theme: 'terminal', // Configurable as 'minimal', 'skeleton', 'code-rain', etc.
      color: '#3b82f6'   // Brand color
    },
    
    // Core lifecycle
    init() {
      this.render();
      this.listenForMount();
    },

    // Render the currently selected style
    render() {
      const root = document.getElementById('boot-root');
      if (this.variants[this.config.theme]) {
        root.innerHTML = this.variants[this.config.theme].render();
      }
    },

    // Listen for successful React mount and exit gracefully
    listenForMount() {
      window.addEventListener('hagicode:ready', () => {
        const screen = document.getElementById('boot-root');
        // Fade out first, then remove the DOM to avoid flicker
        screen.style.opacity = '0';
        screen.style.transition = 'opacity 0.3s ease';
        setTimeout(() => screen.remove(), 300);
      });
    },

    // The implementation logic for all 12 styles lives here
    variants: {
      // ...see details below
    }
  };

  BootSequence.init();
})();
</script>
```

## Kontrolní seznam 12 návrhů stylu spouštění

Těchto 12 stylů jsme seskupili do šesti hlavních kategorií, abychom uspokojili různé scénáře a estetické preference.

### A. Minimalismus

> "Méně je více." Pro scénáře, které sledují maximální rychlost načítání, nabízíme nejlehčí možné možnosti.

#### 1. Minimalistická tečka
Uprostřed obrazovky je jednoduchá tečka spárovaná s animací dýchání.
*   **Implementace**: CSS `@keyframes` řídí měřítko a neprůhlednost.
*   **Nejlepší pro**: Jakýkoli případ, kdy stránka musí zůstat absolutně čistá.

#### 2. Odhalení značky
Pomocí SVG `stroke-dasharray` animace, simuluje ručně nakreslené odkrytí řádků loga HagiCode, po kterém následuje prolínání textu.
*   **Technika**: Animace cesty SVG s vysoce leštěným dojmem.

### B. Skeleton Screen Mimikry

> "Umění klamat oko." Díky simulaci skutečného rozvržení uživatelského rozhraní mají uživatelé pocit, že stránka je již z poloviny načtena.

#### 3. Kostra chatu na bočním panelu
To může být nejpraktičtější varianta. Ručně jsme vytvořili rozložení v HTML, které zrcadlí React `Sidebar` a `ChatInput` komponenty přesně, pak je překryla animací šedého třpytu.
*   **Hodnota**: Po dokončení hydratace React se kostra okamžitě stane skutečnou součástí a uživatelé sotva vnímají spínač.

#### 4. Kostra zásobníku karet
Simuluje naskládaný pohyb karet návrhů při načítání pomocí 3D transformací, aby karty jemně plavaly.

### C. Abstrakce a umění

> Ukažte geek DNA HagiCode.

#### 5. Geometrická morfologie
Geometrický tvar (čtverec) se vykreslí ve středu obrazovky, pak se plynule přemění v průběhu času na kruh, trojúhelník a nakonec logo.
*   **Technologie**: Hladké přechody pomocí CSS `border-radius`.

#### 6. Code Rain
Pocta *The Matrix*. Při použití písma JetBrains Mono padají na pozadí slabé proudy znaků.
*   **Poznámka**: Kvůli výkonu musí proudy postav zůstat v menší oblasti nebo používat nižší obnovovací frekvenci.

#### 7. Neonový puls
Svítící prsten ve stylu kyberpunku, který používá vícenásobné `box-shadow` vrstvy pro vytvoření silné neonové záře.

### D. Branding a témata

> Nechte systém cítit se živý.

#### 8. Sezónní téma
Jedná se o dynamický nakladač. Zkontroluje aktuální datum svátků, jako je lunární Nový rok nebo Vánoce, a načte odpovídající animaci SVG.
*   **Příklad**: Během lunárního Nového roku se červené lucerny jemně houpou ve spodní části obrazovky.

#### 9. Gradientní tok
Pozadí používá tekutý přechod založený na barvách značky HagiCode. V kombinaci s animovaným `background-size` a `background-position`, vytváří pocit pohybu podobný polární záři.

### E. Technický pocit

> Pozdrav vývojářům.

#### 10. Zavádění terminálu
Simuluje výstup konzole. Řádky kódu se rychle posouvají:
```text
> Initializing HagiCode Core...
> Loading models...
> Connecting to neural network...
```
To okamžitě připadá každému vývojáři známé.

#### 11. Ukazatel pokroku
V horní části obrazovky se zobrazí tenký ukazatel průběhu s procenty zobrazenými vpravo. I když nemáme přístup ke skutečnému průběhu stahování, můžeme použít časovač k simulaci „uvěřitelného“ procesu načítání: rychlý pro prvních 80 %, pak postupně pomalejší pro posledních 20 %.

### F. Kreativita

#### 12. Sestavení pixelu
To je velmi zajímavý nápad. Malé čtverečky jsou rozesety po obrazovce, pak se sbíhají směrem ke středu a postupně se skládají do ikony loga HagiCode. Symbolizuje proces stavebního zákona.

## Osvědčené postupy a úskalí

Ve skutečné vývojové práci HagiCode jsme shrnuli několik kriticky důležitých implementačních detailů.

### 1. Defenzivní CSS je povinné
Nikdy nelenijte a přeskočte předponu. Jednou jsme zapomněli nastavit rozsah stylů úvodní obrazovky pomocí ID a globální `div` styly po připojení Reactu neočekávaně ovlivnily úvodní obrazovku a narušily rozložení.
**Poučení**: Umístěte všechny selektory CSS pod `#boot-screen`a používat `!important` v případě potřeby zvýšit prioritu, ale pouze uvnitř CSS úvodní obrazovky.

### 2. Půvabné přechody
Po úspěšném připojení Reactu ne přímo `remove()` úvodní obrazovka DOM.
**Správný přístup**:
1.  Reagujte spouštěče `window.dispatchEvent(new Event('hagicode:ready'))`.
2.  Úvodní obrazovka naslouchá události a prvním sadám `opacity: 0`.
3.  Počkejte 300 ms, což odpovídá délce přechodu CSS, a zavolejte `.remove()` až poté, co je obrazovka zcela neviditelná.

### 3. Synchronizace proměnných motivu
Hodnoty barev úvodní obrazovky jsou pevně zakódovány `index.html`. Pokud změníme primární barvu Tailwindu, musíme aktualizovat i úvodní obrazovku.
**Optimalizace**: Napište jednoduchý plugin do skriptu sestavení Vite ke čtení `tailwind.config.js` a vložit barevné proměnné do `index.html` šablonové proměnné, čímž se vytvoří jediný zdroj pravdy.

### 4. Předběžné načítání písem
Úvodní obrazovky často potřebují používat značkové písmo, ale pokud se písmo načítá pomalu, může se objevit FOUT (Flash of Unstyled Text).
**Řešení**: Přidat `<link rel="preload" href="/fonts/JetBrainsMono.woff2" as="font" type="font/woff2" crossorigin>` uvnitř `<head>`. Jedná se o nízkonákladový způsob s vysokou návratností, jak zlepšit zážitek.

### 5. Sledování výkonu
Píchli jsme injekci `performance.mark('boot-start')` na dně `index.html`a označené `boot-end` když se React úspěšně připojil.
**Proč na tom záleží**: Shromažďováním těchto dat prostřednictvím Application Insights můžeme přímo měřit, jak moc úvodní obrazovka zkracuje vnímanou čekací dobu. Data ukazují, že vynikající základní obrazovka může zlepšit toleranci uživatelů vůči „pomalé síti“ o více než 50 %.
