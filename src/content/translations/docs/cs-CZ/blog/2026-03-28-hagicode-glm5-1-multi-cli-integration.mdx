## Průvodce integrací Hagicode a GLM-5.1 Multi-CLI

:::note[Oznámení o časovém kontextu]
Tento článek dokumentuje návrh integrace multi-CLI a mapování modelu tak, jak existovaly v době publikace. Aktuální vždy podporovaný rozsah viz [Instalace AI Agent CLI](/cs-CZ/related-software-installation/ai-agent-cli/) a [Průvodce instalací](/cs-CZ/installation); a `IFlowCli` zde zmíněný představuje pouze historickou konfiguraci v té době a není součástí aktuálního rozsahu aktivní podpory.
:::

## Pozadí

V projektu Hagicode si uživatelé mohou vybrat z několika nástrojů CLI pro řízení programovacích asistentů AI, včetně Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI a dalších. Tyto nástroje CLI jsou samy o sobě univerzálními nástroji pro programování umělé inteligence, ale prostřednictvím abstraktní vrstvy Hagicode se mohou flexibilně připojit k různým poskytovatelům modelů umělé inteligence.

Zhipu AI (ZAI) poskytuje rozhraní kompatibilní s Anthropic Claude API, které umožňuje těmto nástrojům CLI přímo používat domácí modely řady GLM. Mezi nimi je GLM-5.1 nejnovější vydání velkého jazykového modelu Zhipu s významnými vylepšeními oproti GLM-5.0.

## Abstraktní architektura CLI Hagicode

Hagicode definuje 11 typů poskytovatelů CLI prostřednictvím `AIProviderType` enum, pokrývající běžné nástroje CLI programování AI:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Každé CLI má odpovídající konfiguraci parametrů modelu a podporuje `model` a `reasoning` parametry:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>

        [AIProviderType.GeminiCli] = ["model"],  // Gemini does not support the reasoning parameter
        // ...
    };
```

## Použití GLM-5.1 v různých CLI

### 1. Claude Code CLI + GLM-5.1

Claude Code CLI je jedním ze základních CLI Hagicode a konfiguruje se pomocí konfiguračního systému Hero:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Odpovídající `HeroEquipmentCatalogItem` konfigurace:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI je nejflexibilnější CLI a podporuje specifikaci jakéhokoli modelu v `provider/model` formát:

**Metoda 1: Použijte předponu poskytovatele ZAI**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Metoda 2: Použijte přímo ID modelu**
```json
{
  "model": "glm-5.1"
}
```

**Metoda 3: Uživatelské rozhraní pro konfiguraci rozhraní**

In `HeroModelEquipmentForm.tsx`, OpenCode CLI má vyhrazený zástupný symbol:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Uživatelé mohou zadat:
```
zai/glm-5.1
glm-5.1
```

**Logika analýzy modelu OpenCode CLI**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI má vyhrazenou logiku zpracování starších verzí:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Poznámka**: Codebuddy si zachovává speciální zpracování pro GLM-5.0 a nepoužívá starší normalizaci:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## Výhody vylepšení GLM-5.1

Ve srovnání s GLM-5.0 přináší GLM-5.1 následující významná vylepšení:

### 1. Lepší schopnost uvažování

Podle oficiálních informací o vydání Zhipu zahrnují vylepšení v GLM-5.1:

- **Silnější porozumění kódu**: Přesnější analýza složitých struktur kódu
- **Delší porozumění kontextu**: Podporuje delší kontext konverzace
- **Vylepšené volání nástrojů**: Vyšší úspěšnost volání nástrojů MCP
- **Stabilita výstupu**: Snižuje náhodnost a halucinace

### 2. Komplexní multi-CLI kompatibilita

GLM-5.1 pokrývá všechny běžné CLI podporované Hagicode:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Poznámky

### 1. Konfigurace klíče API

Ujistěte se, že `ANTHROPIC_AUTH_TOKEN` proměnná prostředí je nastavena správně. Je to požadované pověření pro připojení každého CLI k modelu.

### 2. Dostupnost modelu

GLM-5.1 musí povolit příslušný poskytovatel modelu:
- Platforma Zhipu AI ZAI to ve výchozím nastavení podporuje
- Alibaba Cloud DashScope může vyžadovat samostatnou aplikaci

### 3. Formát OpenCode CLI

Při použití `provider/model` formátu, ujistěte se, že ID poskytovatele je správné:
- Zhipu AI: `zai` nebo `zhipuai`
- Alibaba Cloud: `aliyun` nebo `dashscope`

### 4. Parametr uvažování

- `high` je doporučeno pro nejlepší výsledky generování kódu
- Gemini CLI nepodporuje parametr uvažování a bude tuto konfiguraci automaticky ignorovat

## Závěr

Prostřednictvím jednotné abstraktní vrstvy umožňuje Hagicode flexibilní integraci mezi GLM-5.1 a více CLI. Vývojáři si mohou vybrat nástroj CLI, který nejlépe vyhovuje jejich preferencím a scénářům použití, a poté pomocí jednoduché konfigurace použít nejnovější model GLM-5.1.

Jako nejnovější verze modelu Zhipu nabízí GLM-5.1 jasná vylepšení oproti GLM-5.0:
- Nezávislý identifikátor verze bez starší zátěže
- Silnější uvažování a porozumění kódu
- Široká kompatibilita s multi-CLI
- Flexibilní konfigurace úrovně uvažování

Se správnými proměnnými prostředí a nakonfigurovaným vybavením Hero mohou uživatelé plně odemknout výkon GLM-5.1 v různých prostředích CLI.
