## Psaní je pomalejší než mluvení a mluvení je pomalejší než snímek obrazovky – Multimodální postupy zadávání pro asistenty kódování AI

> Psaní kódu má omezenou rychlost bez ohledu na to, jak rychle píšete. Někdy něco, co můžete říct jednou větou, trvá věčnost, než se napíše; někdy jeden screenshot vše vysvětlí, přesto to musíte popsat hromadou textu. Tento článek hovoří o tom, na co jsme narazili při stavbě [HagiCode](https://hagicode.com)od rozpoznávání řeči po nahrávání obrázků. Nakonec jsme chtěli jen trochu usnadnit používání asistenta kódování AI.

## Pozadí

Při vytváření HagiCode jsme si všimli problému – nebo spíše problému, který se přirozeně vynořil, jakmile jej lidé začali intenzivně používat: spoléhat se pouze na psaní může být únavné.

Přemýšlejte o tom: interakce mezi uživateli a agentem je základní scénář. Ale pokud každá výměna vyžaduje nepřetržité psaní na klávesnici, efektivita není velká:

1. **Psaní je příliš pomalé**: U komplikovaných problémů, jako jsou chybové zprávy nebo problémy s uživatelským rozhraním, může psaní všeho trvat půl minuty, zatímco vyslovení nahlas může trvat deset sekund. Ta mezera je skutečná.

2. **Obrázky jsou přímější**: Někdy uživatelské rozhraní vyvolá chybu, někdy chcete porovnat návrh návrhu a někdy potřebujete ukázat strukturu kódu. „Obrázek vydá za tisíc slov“ je možná staré přísloví, ale stále platí. Nechat AI přímo „vidět“ problém je mnohem jasnější, než ho popisovat celé věky.

3. **Interakce by měla být přirozená**: Moderní asistenti umělé inteligence by měli podporovat text, hlas a obrázky. Uživatelé by měli mít možnost vybrat si metodu zadávání, která jim připadá nejpřirozenější.

Rozhodli jsme se tedy do HagiCode přidat podporu rozpoznávání řeči a nahrávání obrázků, aby byly interakce s agenty pohodlnější. Pokud uživatelé mohou psát o něco méně, je to již výhra.

## Analýza

### Technické výzvy v rozpoznávání řeči

Při vytváření rozpoznávání řeči jsme narazili na záludný problém: **rozhraní WebSocket API v prohlížeči nepodporuje vlastní hlavičky HTTP**.

Služba rozpoznávání řeči, kterou jsme si vybrali, byla Doubao Speech Recognition API od ByteDance. Bohužel toto API vyžaduje autentizační údaje jako např `accessToken` a `secretKey` procházet přes HTTP hlavičky. To způsobilo okamžitý technický konflikt:

```javascript
// The browser WebSocket API does not support this approach
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // Not supported
  }
});
```

Měli jsme v podstatě dvě možnosti:

1. **Přístup parametrů dotazu adresy URL**: vložte ověřovací informace do adresy URL
   - Výhodou je jednoduchá implementace
   - Nevýhodou je, že přihlašovací údaje jsou vystaveny frontendu, který není bezpečný; některá rozhraní API také vyžadují autentizaci založenou na záhlaví

2. **Backend proxy přístup**: implementujte WebSocket proxy na backend
   - Výhodou je, že přihlašovací údaje zůstávají bezpečně uloženy na backendu a řešení je plně kompatibilní s požadavky API
   - Nevýhodou je, že implementace je trochu složitější

Nakonec jsme zvolili **backend proxy přístup**. Bezpečnost není něco, v čem byste dělali kompromisy.

### Funkční požadavky pro nahrávání obrázků

Naše požadavky na nahrávání obrázků byly ve skutečnosti docela jednoduché:

1. **Více způsobů nahrávání**: kliknutím vyberte soubor, přetáhněte jej a vložte ze schránky
2. **Ověření souboru**: omezení typu (PNG, JPG, WebP, GIF) a omezení velikosti (5–10 MB) jsou základní požadavky
3. **Uživatelská zkušenost**: průběh nahrávání, náhledy a chybové zprávy, aby uživatelé vždy věděli, co se děje
4. **Zabezpečení**: Ověření na straně serveru a ochrana proti nahrávání škodlivých souborů jsou zásadní

## Řešení

### Rozpoznávání řeči: WebSocket Proxy Architecture

Navrhli jsme třívrstvou architekturu pro rozpoznávání řeči a našli cestu, která fungovala:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**Implementace základních komponent**:

1. **Frontend AudioWorklet procesor**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Resample to 16 kHz (required by the Doubao API)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Accumulate samples into 500 ms chunks
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Convert to 16-bit PCM and send
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **Backend WebSocket handler** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **Frontovat `VoiceTextArea` komponent**:

```tsx
export const VoiceTextArea = forwardRef<HTMLTextAreaElement, VoiceTextAreaProps>(
  ({ value, onChange, onTextRecognized, maxDuration }, ref) => {
    const { isRecording, interimText, volume, duration, startRecording, stopRecording } =
      useVoiceRecording({ onTextRecognized, maxDuration });

    return (
      <div className="flex gap-2">
        {/* Voice button */}
        <button onClick={handleButtonClick}>
          {isRecording ? <VolumeWaveform volume={volume} /> : <Mic />}
        </button>
        {/* Text input area */}
        <textarea value={displayValue} onChange={handleChange} />
      </div>
    );
  }
);
```

### Nahrávání obrázků: Komponenta nahrávání více metodami

Vytvořili jsme plnohodnotnou komponentu pro nahrávání obrázků s podporou všech tří metod nahrávání, která pokrývá nejběžnější scénáře, se kterými se uživatelé setkávají.

**Základní funkce**:

1. **Tři způsoby nahrávání**:

```tsx
// Click to upload
const handleClick = () => fileInputRef.current?.click();

// Drag-and-drop upload
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Clipboard paste
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **Ověření rozhraní**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **Backend upload handler** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Validation
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Save file
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## Praktický průvodce

### Jak používat rozpoznávání řeči

1. **Nakonfigurujte službu rozpoznávání řeči**:
   - Otevřete stránku nastavení rozpoznávání řeči
   - Nakonfigurujte řeč Doubao `AppId` a `AccessToken`
   - Volitelně nakonfigurujte klíčová slova pro zlepšení přesnosti rozpoznávání výrazů specifických pro doménu

2. **Použijte ve vstupním poli**:
   - Klikněte na ikonu mikrofonu na levé straně vstupního pole
   - Po zobrazení animace průběhu začněte mluvit
   - Dalším kliknutím na ikonu nahrávání zastavíte
   - Rozpoznaný text se automaticky vloží na pozici kurzoru

3. **Příklad konfigurace klíčových slov**:

```text
TypeScript
React
useState
useEffect
```

### Jak používat nahrávání obrázků

1. **Metody nahrávání**:
   - Kliknutím na tlačítko nahrát vyberte soubor
   - Přetáhněte obrázek přímo do oblasti pro nahrávání
   - Použijte `Ctrl+V` pro vložení snímku obrazovky ze schránky

2. **Podporované formáty**: PNG, JPG, JPEG, WebP, GIF
3. **Limit velikosti**: 5 MB ve výchozím nastavení (lze konfigurovat)

### Poznámky

1. **Rozpoznávání řeči**:
   - Je vyžadováno povolení k mikrofonu
   - Používejte pokud možno v tichém prostředí
   - Maximální podporovaná délka záznamu je ve výchozím nastavení 300 sekund (lze konfigurovat)

2. **Nahrání obrázků**:
   - Podporovány jsou pouze běžné obrazové formáty
   - Věnujte pozornost limitům velikosti souboru
   - Nahrané obrázky automaticky obdrží adresu URL náhledu

3. **Bezpečnostní aspekty**:
   - Pověření pro rozpoznávání řeči jsou uložena na backendu
   - Nahrávání obrázků prochází přísným ověřováním na straně serveru
   - HTTPS/WSS se doporučuje v produkčním prostředí
