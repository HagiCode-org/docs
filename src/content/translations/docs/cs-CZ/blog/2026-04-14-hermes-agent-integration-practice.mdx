## Hermes Agent Integration Practice: Od protokolu k produkci

> Sdílení kompletních zkušeností HagiCode s integrací Hermes Agent, včetně základních lekcí týkajících se adaptace protokolu ACP, správy fondu relací a synchronizace front-end/back-end smluv.

## Pozadí

Při budování HagiCode, kódovací platformy podporované umělou inteligencí, náš tým potřeboval integrovat Agent framework, který by mohl běžet lokálně a také škálovat do cloudu. Po průzkumu byl Hermes Agent z Nous Research vybrán jako základní motor pro naše univerzální schopnosti Agenta.

Po pravdě řečeno, výběr technologie není ani zvlášť těžký, ani zvlášť snadný. Na trhu je spousta silných Agent frameworků, ale Hermes vyčníval, protože jeho protokol ACP a systém nástrojů obzvláště dobře vyhovovaly náročným požadavkům HagiCode: místní vývoj, týmová spolupráce a rozšíření cloudu. Zavedení Hermese do skutečného výrobního systému však znamenalo vyřešení dlouhého seznamu technických problémů. Tato část byla všechno, jen ne triviální.

Zásobník HagiCode používá Orleans k sestavení distribuovaného systému, zatímco frontend je vytvořen pomocí React + TypeScript. Integrace Hermes znamenala zachování konzistence architektury a zároveň učinila Hermes prvotřídním exekutorem vedle ClaudeCode a OpenCode. Zní to jednoduše, ale realizace vždy vypráví skutečný příběh.

Tento článek sdílí naše praktické zkušenosti s integrací Hermes Agent do HagiCode a doufáme, že nabídne užitečný referenční materiál pro týmy, které čelí podobným potřebám. Jakmile totiž jednou spadnete do jámy, není důvod nechat někoho jiného spadnout do stejné jámy.

## Architektura Design

### Vrstvený designový přístup

Integrace Hermes HagiCode využívá jasnou vrstvenou architekturu, přičemž každá vrstva se zaměřuje na své vlastní odpovědnosti:

**Zadní jádrová vrstva**
- `HermesCliProvider`: implementuje `IAIProvider` rozhraní jako jednotný vstupní bod poskytovatele AI
- `HermesPlatformConfiguration`: spravuje cestu ke spustitelnému souboru Hermes, argumenty, ověřování a související nastavení
- `ICliProvider<HermesOptions>`: nízkoúrovňová abstrakce CLI poskytovaná HagiCode.Libs pro zpracování životních cyklů dílčích procesů

**Přepravní vrstva**
- `StdioAcpTransport`: komunikuje s podprocesem Hermes ACP prostřednictvím standardního vstupu a výstupu
- Metody protokolu ACP: `initialize`, `authenticate`, `session/new`, `session/prompt`

**Běhová vrstva**
- `HermesGrain`: Implementace Orleans Grain, která zpracovává spouštění distribuovaných relací
- `CliAcpSessionPool`: fond relací, který opakovaně používá podprocesy ACP, aby se zabránilo časté režii spouštění

**Přední vrstva**
- `ExecutorAvatar`: Vizuální identita a ikona Hermes
- `executorTypeAdapter`: Logika mapování typu poskytovatele
- SignalR zasílání zpráv v reálném čase: udržuje konzistenci identity Hermes v celém toku zpráv

Tento vrstvený design umožňuje, aby se každá vrstva vyvíjela nezávisle. Pokud například chceme v budoucnu přidat nový transportní mechanismus, jako je WebSocket, stačí upravit transportní vrstvu. Není potřeba převracet celý systém jen proto, že se změní jeden transport.

### Jednotná abstrakce rozhraní

Všichni poskytovatelé AI implementují `IAIProvider` rozhraní, které je jednou ze základních možností návrhu v architektuře HagiCode:

```csharp
public interface IAIProvider
{
    string Name { get; }
    ProviderCapabilities Capabilities { get; }

    IAsyncEnumerable<AIStreamingChunk> StreamAsync(
        AIRequest request,
        CancellationToken cancellationToken = default);

    Task<AIResponse> ExecuteAsync(
        AIRequest request,
        CancellationToken cancellationToken = default);
}
```

`HermesCliProvider` implementuje toto rozhraní a stojí na stejné úrovni `ClaudeCodeProvider`, `OpenCodeProvider`a další. Mezi výhody tohoto designu patří:

1. **Nahraditelnost**: Změna poskytovatelů neovlivňuje obchodní logiku vyšší vrstvy
2. **Testovatelnost**: Poskytovatelé mohou být snadno zesměšňováni pro testování jednotek
3. **Rozšiřitelnost**: přidání nového poskytovatele vyžaduje pouze implementaci rozhraní

Rozhraní jsou nakonec hodně podobná pravidlům. Jakmile jsou pravidla nastavena, mohou všichni harmonicky koexistovat, hrát podle svých předností a vyhýbat se vzájemnému šlapání. Je v tom jistá elegance.

## Implementace jádra

### Implementace vrstvy poskytovatele

`HermesCliProvider` je jádrem celé integrace. Koordinuje různé komponenty potřebné k dokončení jediného vyvolání AI:

```csharp
public sealed class HermesCliProvider : IAIProvider, IVersionedAIProvider
{
    private readonly ICliProvider<LibsHermesOptions> _provider;
    private readonly ConcurrentDictionary<string, string> _sessionBindings;

    public ProviderCapabilities Capabilities { get; } = new()
    {
        SupportsStreaming = true,
        SupportsTools = true,
        SupportsSystemMessages = true,
        SupportsArtifacts = false
    };

    public async IAsyncEnumerable<AIStreamingChunk> StreamAsync(
        AIRequest request,
        [EnumeratorCancellation] CancellationToken cancellationToken = default)
    {
        // 1. Resolve the session binding key
        var bindingKey = ResolveBindingKey(request.CessionId);

        // 2. Get or create a Hermes session through the session pool
        var options = new HermesOptions
        {
            ExecutablePath = _platformConfiguration.ExecutablePath,
            Arguments = _platformConfiguration.Arguments,
            SessionId = _sessionBindings.TryGetValue(bindingKey, out var sessionId) ? sessionId : null,
            WorkingDirectory = request.WorkingDirectory,
            Model = request.Model
        };

        // 3. Execute and collect the streaming response
        await foreach (var message in _provider.ExecuteAsync(options, request.Prompt, cancellationToken))
        {
            // 4. Map ACP messages to AIStreamingChunk
            if (_responseMapper.TryConvertToStreamingChunk(message, out var chunk))
            {
                yield return chunk;
            }
        }
    }
}
```

Zde je zvláště důležitých několik konstrukčních bodů:

1. **Vazba relace**: používá `CessionId` svázat více požadavků na stejný podproces Hermes a zachovat kontinuitu kontextu napříč konverzacemi s více otáčkami
2. **Mapování odezvy**: převádí formát zprávy Hermes ACP na jednotný `AIStreamingChunk` formát
3. **Podpora streamování**: použití `IAsyncEnumerable` pro podporu skutečných reakcí streamování

Session binding je trochu jako mezilidské vztahy. Jakmile je spojení navázáno, budoucí komunikace má kontext, takže nemusíte pokaždé začínat od nuly. Samozřejmě, že tento vztah musí být stále udržován.

### Adaptace protokolu ACP

Hermes používá ACP (Agent Communication Protocol), který se liší od tradičního HTTP API. ACP je protokol založený na standardním vstupu a výstupu a má několik vlastností:

1. **Startup marker**: po spuštění procesu Hermes se zobrazí `//ready` značka
2. **Dynamická autentizace**: metody autentizace nejsou pevně dané a musí být sjednány prostřednictvím protokolu
3. **Opětovné použití relace**: vytvořené relace jsou znovu použity `SessionId`
4. **Fragmentované odpovědi**: úplná odpověď může být rozdělena do více `session/update` oznámení

HagiCode tyto vlastnosti zvládá `StdioAcpTransport`:

```csharp
public class StdioAcpTransport
{
    public async Task InitializeAsync(CancellationToken cancellationToken)
    {
        // Wait for the //ready marker
        var readyLine = await _outputReader.ReadLineAsync(cancellationToken);
        if (readyLine != "//ready")
        {
            throw new InvalidOperationException("Hermes did not send ready signal");
        }

        // Send the initialize request
        await SendRequestAsync(new
        {
            jsonrpc = "2.0",
            id = 1,
            method = "initialize",
            @params = new
            {
                protocolVersion = "2024-11-05",
                capabilities = new { },
                clientInfo = new { name = "HagiCode", version = "1.0.0" }
            }
        }, cancellationToken);
    }
}
```

Protokoly jsou trochu jako vzájemné porozumění mezi lidmi. Jakmile toto porozumění existuje, komunikace plyne mnohem plynuleji. Jeho vybudování chce jen čas.

### Správa fondu relací

Časté spouštění podprocesů Hermes je drahé, proto jsme implementovali mechanismus fondu relací:

```csharp
services.AddSingleton(static _ =>
{
    var registry = new CliProviderPoolConfigurationRegistry();
    registry.Register("hermes", new CliPoolSettings
    {
        MaxActiveSessions = 50,
        IdleTimeout = TimeSpan.FromMinutes(10)
    });
    return registry;
});
```

Klíčové parametry fondu relací:
- `MaxActiveSessions`: řídí limit souběžnosti, aby se zabránilo vyčerpání zdrojů
- `IdleTimeout`: časový limit nečinnosti, který vyrovnává spouštěcí náklady a využití paměti

V praxi jsme zjistili, že:
1. Pokud je časový limit nečinnosti příliš krátký, relace se často restartují; pokud je příliš dlouhý, paměť zůstává obsazená
2. Limit souběžnosti musí být vyladěn podle skutečného zatížení, protože jeho příliš vysoké nastavení může způsobit zpomalení systému
3. Využití fondu relací vyžaduje monitorování, aby bylo možné parametry upravit včas

Je to podobné jako s mnoha životními volbami: příliš agresivní vytváří problémy, zatímco příliš konzervativní promarňuje příležitosti. Cílem je jednoduše najít tu správnou rovnováhu.

## Front-end integrace

### Mapování typů

Frontend musí správně identifikovat poskytovatele Hermes a zobrazovat odpovídající vizuální prvky:

```typescript
// executorTypeAdapter.ts
export const resolveExecutorVisualTypeFromProviderType = (
  providerType: PCode_Models_AIProviderType | null | undefined
): ExecutorVisualType => {
  switch (providerType) {
    case PCode_Models_AIProviderType.HERMES_CLI:
      return 'Hermes';
    default:
      return 'Unknown';
  }
};
```

### Vizuální prezentace

Hermes má svou vlastní ikonu a barevnou identitu:

```typescript
// ExecutorAvatar.tsx
const renderExecutorGlyph = (executorType: ExecutorVisualType, iconSize: number) => {
```

Krásné věci si přece zaslouží krásnou prezentaci. Zajištění toho, aby byla krása skutečně viditelná, stále závisí na předním řemeslném zpracování.

### Synchronizace smlouvy

Přední a zadní část udržují svou smlouvu v souladu s generováním OpenAPI. Zadní konec definuje `AIProviderType` výčet:

```csharp
public enum AIProviderType
{
    Unknown,
    ClaudeCode,
    OpenCode,
    HermesCli  // Newly added
}
```

Frontend generuje odpovídající typ TypeScript prostřednictvím OpenAPI, což zajišťuje, že hodnoty výčtu zůstanou konzistentní. To je klíč k tomu, abyste se vyhnuli zobrazování frontendu `Unknown`.

Smlouva je hodně jako slib. Po odsouhlasení se to musí ctít, jinak se dostanete do nepříjemných situací jako `Unknown`.

## Správa konfigurace

Konfigurace Hermes je spravována prostřednictvím `appsettings.json`:

```json
{
  "Providers": {
    "HermesCli": {
      "ExecutablePath": "hermes",
      "Arguments": "acp",
      "StartupTimeoutMs": 10000,
      "ClientName": "HagiCode",
      "Authentication": {
        "PreferredMethodId": "api-key",
        "MethodInfo": {
          "api-key": "your-api-key-here"
        }
      },
      "SessionDefaults": {
        "Model": "claude-sonnet-4-20250514",
        "ModeId": "default"
      }
    }
  }
}
```

Tento design založený na konfiguraci přináší flexibilitu:
- spustitelné cesty lze přepsat, což je vhodné pro vývoj a testování
- spouštěcí argumenty lze upravit tak, aby odpovídaly různým verzím Hermes
- autentizační informace lze nakonfigurovat tak, aby podporovaly více metod autentizace

Konfigurace je v životě trochu jako otázky s výběrem z více možností. Pokud je k dispozici dostatek možností, obvykle se najde ta, která se hodí. To znamená, že příliš mnoho možností může způsobit únavu z rozhodování.

## Praktické zkušenosti

### Zdravotní prohlídky

Vybudování spolehlivého poskytovatele vyžaduje komplexní zdravotní kontroly:

```csharp
public async Task<ProviderTestResult> PingAsync(CancellationToken cancellationToken = default)
{
    var response = await ExecuteAsync(new AIRequest
    {
        Prompt = "Reply with exactly PONG.",
        CessionId = null,
        AllowedTools = Array.Empty<string>(),
        WorkingDirectory = ResolveWorkingDirectory(null)
    }, cancellationToken);

    var success = string.Equals(response.Content.Trim(), "PONG", StringComparison.OrdinalIgnoreCase);
    return new ProviderTestResult
    {
        ProviderName = Name,
        Success = success,
        ResponseTimeMs = stopwatch.ElapsedMilliseconds,
        ErrorMessage = success ? null : $"Unexpected Hermes ping response: '{response.Content}'."
    };
}
```

Body, které je třeba sledovat ve zdravotních kontrolách:
1. Používejte jednoduché testovací případy a vyhněte se příliš složitým scénářům
2. Nastavte rozumné hodnoty časového limitu
3. Zaznamenejte dobu odezvy pro podporu analýzy výkonu

Stejně jako lidé potřebují fyzické kontroly, systémy potřebují také zdravotní kontroly. Čím dříve jsou problémy zjištěny, tím snazší je je opravit.

### Validační nástroj

HagiCode poskytuje vyhrazenou konzoli pro ověření integrace Hermes:

```bash
# Basic validation
HagiCode.Libs.Hermes.Console --test-provider

# Full suite (including repository analysis)
HagiCode.Libs.Hermes.Console --test-provider-full --repo .

# Custom executable
HagiCode.Libs.Hermes.Console --test-provider-full --executable /path/to/hermes
```

Tento nástroj je velmi užitečný při vývoji, protože nám umožňuje rychle ověřit, zda je integrace správná. Koneckonců, nikdo nechce čekat, až se problém objeví, než si vzpomene na testování.

### Řešení běžných problémů

**Selhání ověření**
- Zkontrolujte zda `Authentication.PreferredMethodId` odpovídá autentizační metodě, kterou Hermes skutečně podporuje
- Potvrďte, že je formát informací o ověření správný, například klíč API nebo token nosiče

**Časový limit relace**
- Zvyšte `StartupTimeoutMs` hodnotu
- Zkontrolujte dostupnost serveru MCP
- Zkontrolujte využití systémových prostředků

**Neúplná odpověď**
- Zajistěte `session/update` oznámení a konečný výsledek jsou agregovány správně
- Zkontrolujte logiku zrušení při zpracování streamování
- Ověřte, zda je zpracování chyb dokončeno

**Zobrazení předního panelu Neznámé**
- Potvrďte, že generování OpenAPI již obsahuje `HermesCli` hodnotu enum
- Zkontrolujte, zda je mapování typu správné
- Vymažte mezipaměť prohlížeče a obnovte typy

Problémy budou vždy existovat. Když se objeví, důležité je nepropadat panice. Krok za krokem sledujte příčinu a většinu z nich lze nakonec vyřešit.

### Návrhy na optimalizaci výkonu

1. **Použijte fond relací**: opětovné použití podprocesů ACP ke snížení režie při spuštění
2. **Vhodně nastavte časové limity**: vyvažte využití paměti a spouštěcí náklady
3. **Opětovné použití ID relace**: použijte stejné `CessionId` pro dávkové úkoly
4. **Konfigurujte MCP na vyžádání**: vyhněte se zbytečnému vyvolávání nástrojů

Výkon je hodně podobný efektivitě v každodenním životě. Když to uděláte správně, dosáhnete více s méně; když to uděláte špatně, úsilí se násobí, zatímco výsledky se zmenšují. Najít ten „správný“ bod vyžaduje zkušenosti i štěstí.

## Závěr

Integrace Hermes Agent do produkčního systému vyžaduje zvážení problémů napříč různými dimenzemi:

1. **Architektura**: navrhněte jednotné rozhraní poskytovatele a implementujte architekturu vyměnitelných komponent
2. **Protokol**: správně zpracovávat chování specifické pro AKT, jako jsou startovací značky a dynamické ověřování
3. **Výkon**: znovu používejte prostředky prostřednictvím fondu relací a vyvažujte spouštěcí náklady s využitím paměti
4. **Rozhraní**: zajistěte synchronizaci smlouvy a poskytněte konzistentní vizuální zážitek

Zkušenosti HagiCode ukazují, že s dobrým vrstveným návrhem a implementací řízenou konfigurací lze komplexní systém Agent hladce integrovat do existující architektury.

Tyto principy zní jednoduše, když jsou popsány slovy, ale skutečná implementace vždy přináší mnoho různých druhů problémů. To je v pořádku. Pokud se problém vyřeší, stane se z něj zkušenost. Pokud ne, stane se z toho lekce. Ať tak či onak, stále má hodnotu.

Krásné věci nebo lidi není třeba vlastnit; dokud zůstanou krásné, stačí tu krásu tiše oceňovat. Technologie je hodně stejná. Pokud to pomůže zlepšit systém, pak na konkrétním rámci nebo protokolu záleží mnohem méně, než si lidé někdy myslí.
