## Proč si HagiCode vybral Hermes jako své integrované jádro agenta

> Při budování kódovací platformy s podporou AI výběr správného jádra agenta přímo určuje horní hranici schopností systému. Některé věci prostě nelze vynutit; vyberte si špatný rámec a žádné vynaložené úsilí neučiní dojem, že je to správné. Tento článek sdílí myšlenky technického výběru HagiCode a naše praktické zkušenosti s integrací Hermes Agent.

## Pozadí

Při vytváření kódovacího produktu s podporou AI je jednou z nejtěžších částí výběr základního rámce agenta. Na trhu je ve skutečnosti poměrně dost možností, ale některé jsou příliš omezené na funkčnost, některé jsou příliš složité na nasazení a jiné se jednoduše dostatečně neškálují. To, co jsme potřebovali, bylo řešení, které by mohlo běžet na VPS za 5 $ a zároveň by se dalo připojit ke clusteru GPU. Tento požadavek nemusí znít extrémně, ale stačí k tomu, aby vyděsil spoustu týmů.

V praxi mnoho takzvaných „agentů all-in-one“ běží buď pouze v cloudu, nebo vyžadují absurdně vysoké náklady na místní nasazení. Poté, co jsme strávili dva týdny zkoumáním různých přístupů, učinili jsme odvážné rozhodnutí: přestavět celé jádro agenta kolem Hermes jako základní engine pro našeho integrovaného agenta.

Vše, co následovalo, mohl být prostě osud.

## Proč HagiCode potřebuje Hermes

Než se ponoříme do samotného Hermesu, pomůže nám vysvětlit, proč HagiCode něco takového vůbec potřeboval. Málokdy věci fungují přesně tak, jak chcete, takže potřebujete praktický důvod, abyste se zavázali k technickému směru.

Jako asistent kódování AI musí HagiCode podporovat několik scénářů použití současně:
- Místní vývojová prostředí: vývojáři to chtějí provozovat na svých vlastních počítačích, aby data nikdy neopouštěla místní prostředí. V dnešní době není bezpečnost dat nikdy triviální záležitostí.
- Prostředí týmové spolupráce: malé týmy by měly mít možnost sdílet nasazení agenta běžícího na serveru. Úspora peněz je důležitá a každý má své limity.
- Elastické rozšíření cloudu: při zpracování složitých úkolů by se systém měl automaticky škálovat na cluster GPU. Vždy je lepší být připraven.

Tento požadavek „chceme všechno najednou“ nás přivedl k Hermesovi. Zda to byla perfektní volba, nemohu s jistotou říci, ale v té době jsme neviděli lepší možnost.

## Co je Hermes Agent

Hermes Agent je autonomní AI Agent vytvořený **Nous Research**. Někteří čtenáři nemusí být obeznámeni s Nous Research; jsou laboratoří za velkými modely s otevřeným zdrojovým kódem, jako jsou Hermes, Nomos a Psyché. Vybudovali mnoho vynikajících věcí, i když jsou stále více nedoceněni, než si zaslouží.

Na rozdíl od tradičních IDE kódovacích asistentů nebo jednoduchých API chat wrapperů má Hermes definující rys: čím déle běží, tím je schopnější. Není navržen tak, aby dokončil úkol jednou a zastavil se; během dlouhodobého provozu se neustále učí a shromažďuje zkušenosti. V tomto smyslu mi to připadá trochu jako člověk.

### Základní vlastnosti

Některé z klíčových schopností Hermes se náhodou velmi těsně shodují s potřebami HagiCode.

To znamená, že HagiCode si může vybrat nejvhodnější model nasazení na základě scénáře každého uživatele: jednotlivci jej spouštějí lokálně, týmy jej nasazují na servery a komplexní úlohy využívají zdroje GPU. Jedna kódová základna to všechno zvládne. V takto uspěchaném světě je již úspora jedné vrstvy složitosti výhrou.

**Multiplatformová brána pro zasílání zpráv**
Hermes nativně podporuje Telegram, Discord, Slack, WhatsApp a další. Pro HagiCode to znamená, že můžeme v budoucnu mnohem snadněji podporovat asistenty AI na těchto kanálech. Další cesty vpřed jsou vždy vítány.

**Bohatý systém nástrojů**
Hermes přichází s více než 40 vestavěnými nástroji a podporuje rozšíření MCP (Model Context Protocol). To je pro asistenta kódování nezbytné: provádění příkazů shellu, práce se souborovým systémem a volání Gitu závisí na podpoře nástroje. Agent bez nástrojů je jako pták bez křídel.

**Paměť mezi relacemi**
Hermes obsahuje systém trvalé paměti a používá FTS5 fulltextové vyhledávání k vyvolání historických konverzací. To umožňuje agentovi pamatovat si předchozí kontext místo toho, aby pokaždé „ztrácel paměť“. Někdy si lidé přejí, aby mohli věci tak snadno zapomenout, ale realita je obvykle méně štědrá.

## Jak HagiCode integruje Hermes

Nyní, když je „proč“ jasné, podívejme se na „jak“. Jakmile něco dává teoreticky smysl, dalším krokem je postavit to.

### Abstrakce vrstvy poskytovatele

V architektuře HagiCode implementují všichni poskytovatelé AI jednotný `IAIProvider` rozhraní:

```csharp
public sealed class HermesCliProvider : IAIProvider, IVersionedAIProvider
{
    public ProviderCapabilities Capabilities { get; } = new ProviderCapabilities
    {
        SupportsStreaming = true,   // Supports streaming output
        SupportsTools = true,       // Supports tool invocation
        SupportsSystemMessages = true, // Supports system prompts
        SupportsArtifacts = false
    };
}
```

Tato abstraktní vrstva umožňuje HagiCode plynule přepínat mezi různými poskytovateli AI. Ať už je backend OpenAI, Claude nebo Hermes, vzor volání horní vrstvy zůstává naprosto stejný. Jednoduše řečeno, udržuje věci jednoduché.

### Komunikační protokol ACP

Hermes komunikuje prostřednictvím **ACP (Agent Communication Protocol)**. Tento protokol je navržen speciálně pro komunikaci agentů a jeho hlavní metody zahrnují:

| Metoda | Popis |
|------|------|
| `initialize` | Inicializujte připojení a získejte verzi protokolu a možnosti klienta |
| `authenticate` | Zvládněte autentizaci a podporujte více metod ověřování |
| `session/new` | Vytvořte novou relaci a nakonfigurujte pracovní adresář a servery MCP |
| `session/prompt` | Odešlete výzvu a obdržíte odpověď |

HagiCode implementuje transportní vrstvu ACP prostřednictvím `StdioAcpTransport`, spouští podproces Hermes a komunikuje s ním přes standardní vstup a výstup. Možná to zní složitě, ale v praxi se to dá zvládnout, pokud máte dostatek trpělivosti.

### Správa konfigurace

Konfigurace se spravuje přes `HermesPlatformConfiguration` třída:

```csharp
public sealed class HermesPlatformConfiguration : IAcpPlatformConfiguration
{
    public string ExecutablePath { get; set; } = "hermes";
    public string Arguments { get; set; } = "acp";
    public int StartupTimeoutMs { get; set; } = 5000;
    public string ClientName { get; set; } = "HagiCode";
    public HermesAuthenticationConfiguration Authentication { get; set; }
    public HermesSessionDefaultsConfiguration SessionDefaults { get; set; }
}
```

Nakonfigurujte Hermes v `appsettings.json`:

```json
{
  "Providers": {
    "HermesCli": {
      "ExecutablePath": "hermes",
      "Arguments": "acp",
      "StartupTimeoutMs": 10000,
      "ClientName": "HagiCode",
      "Authentication": {
        "PreferredMethodId": "api-key",
        "MethodInfo": {
          "api-key": "your-api-key-here"
        }
      },
      "SessionDefaults": {
        "Model": "claude-sonnet-4-20250514",
        "ModeId": "default"
      }
    }
  }
}
```

Konfigurace často vypadá na papíře jednoduše, ale správné provedení každého detailu stále vyžaduje skutečné úsilí.

### Distribuovaná architektura Orleans

HagiCode používá Orleans k vybudování svého distribuovaného systému a integrace Hermes je implementována prostřednictvím následujících komponent:

- **HermesGrain**: Implementace Orleans Grain, která zpracovává provádění relace
- **HermesPlatformConfiguration**: Konfigurace specifická pro platformu
- **HermesAcpSessionAdapter**: Adaptér relace ACP
- **HermesConsole**: Vyhrazená ověřovací konzole

Jméno Orleans má v sobě určité kouzlo. I když tento Orleans nemá nic společného s legendárním městem, dobré jméno nikdy neuškodí.

### End-to-End tok provádění

Níže je uvedena základní logika provádění Hermes Provider:

```csharp
private async IAsyncEnumerable<AIStreamingChunk> StreamCoreAsync(
    AIRequest request,
    string? embeddedCommandPrompt,
    [EnumeratorCancellation] CancellationToken cancellationToken)
{
    // 1. Create transport layer and launch Hermes subprocess
    await using var transport = new StdioAcpTransport(
        platformConfiguration.GetExecutablePath(),
        platformConfiguration.GetArguments(),
        platformConfiguration.GetEnvironmentVariables(),
        platformConfiguration.GetStartupTimeout(),
        _loggerFactory.CreateLogger<StdioAcpTransport>());
    await transport.ConnectAsync(cancellationToken);

    // 2. Initialize and obtain protocol version and authentication methods
    var initializeResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "initialize",
        BuildInitializeParameters(platformConfiguration), cancellationToken);

    // 3. Handle authentication
    var authMethods = ParseAuthMethods(initializeResult);
    if (!isAuthenticated)
    {
        var methodId = platformConfiguration.Authentication.ResolveMethodId(authMethods);
        await SendHermesRequestAsync(transport, nextRequestId++, "authenticate", ...);
    }

    // 4. Create session
    var newSessionResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "session/new",
        BuildNewSessionParameters(platformConfiguration, workingDirectory, model), cancellationToken);
    var sessionId = ParseSessionId(newSessionResult);

    // 5. Execute prompt and collect streaming responses
    await foreach (var payload in transport.ReceiveMessagesAsync(cancellationToken))
    {
        // Handle session/update notifications and convert them into streaming chunks
        if (TryParseSessionNotification(root, out var notification))
        {
            if (_responseMapper.TryConvertToStreamingChunk(notification, out var chunk))
            {
                yield return chunk;
            }
        }
    }
}
```

S kódem se podrobnosti nakonec seznámí. Nejdůležitější je celkový přístup.

### Zdravotní kontroly

Aby bylo zajištěno, že Hermes zůstane dostupný, HagiCode implementuje mechanismus kontroly stavu:

```csharp
public async Task<ProviderTestResult> PingAsync(CancellationToken cancellationToken = default)
{
    var response = await ExecuteAsync(
        new AIRequest
        {
            Prompt = "Reply with exactly PONG.",
            CessionId = null,
            AllowedTools = Array.Empty<string>(),
            WorkingDirectory = ResolveWorkingDirectory(null)
        },
        cancellationToken);

    var success = string.Equals(response.Content.Trim(), "PONG", StringComparison.OrdinalIgnoreCase);
    return new ProviderTestResult
    {
        ProviderName = Name,
        Success = success,
        ResponseTimeMs = stopwatch.ElapsedMilliseconds,
        ErrorMessage = success ? null : $"Unexpected Hermes ping response: '{response.Content}'."
    };
}
```

Zhruba tak zde vypadá „zdravotní prohlídka“. V některých ohledech se lidé tolik neliší: čas od času pomáhá, když se ohlásíme, i když nám nikdo přesně neřekne, co máme hledat.

## Praktické úvahy

Před integrací Hermes stojí za pochopení několik úskalí. Každý dříve nebo později vstoupí do několika pastí.

### Konfigurace metody autentizace

Hermes podporuje více metod ověřování, včetně klíčů API a tokenů, takže si musíte vybrat na základě skutečného scénáře nasazení. Nesprávná konfigurace může způsobit selhání připojení a výsledné chybové zprávy nejsou vždy intuitivní. Někdy je hlášená chyba daleko od skutečné hlavní příčiny, což znamená, že pomalé a pečlivé ladění je nevyhnutelné.

### Konfigurace MCP serveru

Při vytváření relace můžete nakonfigurovat seznam serverů MCP, aby Hermes mohl volat externí nástroje. Mějte však na paměti následující body:
- Adresy serveru MCP musí být dosažitelné
- Časové limity musí být nakonfigurovány rozumně
- Systém potřebuje zpracování degradace, když je server nedostupný

V praxi je defenzivní myšlení důležitější, než lidé očekávají.

### Správa pracovního adresáře

Každá relace musí specifikovat pracovní adresář, aby mohl Hermes správně přistupovat k souborům projektu. Ve scénářích s více projekty se pracovní adresář musí dynamicky přepínat. Zní to přímočaře, ale okrajových případů je více, než byste si mysleli.

### Agregace odpovědí

Hermesovy odpovědi mohou být rozděleny `session/update` oznámení a konečný výsledek, takže je třeba je správně sloučit. V opačném případě může dojít ke ztrátě obsahu.

### Strategie zpracování chyb

Chyby za běhu by měly být vráceny explicitně namísto tichého návratu k jinému poskytovateli. Tímto způsobem uživatelé vědí, že problém pochází od společnosti Hermes, než aby se divili, proč systém najednou v zákulisí změnil modely.
