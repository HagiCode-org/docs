## Plná podpora GLM-5.1 a integrace Gemini CLI: Cesta HagiCode k evoluci více modelů

> Tento článek představuje dvě velké nedávné aktualizace platformy HagiCode: plnou podporu pro model Zhipu AI GLM-5.1 a úspěšnou integraci Gemini CLI jako desátého Agent CLI. Tyto aktualizace společně dále posilují možnosti platformy pro více modelů a ekosystém s více rozhraními CLI.

:::note[Oznámení o časovém kontextu]
Tento článek zaznamenává postup rozšiřování multi-CLI tak, jak byl v době vydání. Aktuální vždy podporovaný rozsah viz [Instalace AI Agent CLI](/cs-CZ/related-software-installation/ai-agent-cli/) a [Průvodce instalací](/cs-CZ/installation); a `IFlowCli` zde zmíněný představuje pouze historickou konfiguraci v té době a není součástí aktuálního rozsahu aktivní podpory.
:::

## Pozadí

Čas jde opravdu rychle. Vývoj velkých jazykových modelů stoupá jako bambus na jaře. Není to tak dávno, co jsme ještě fandili „AI, která umí psát kód“. Nyní jsme již v éře multimodelové spolupráce a integrace více nástrojů. Je to vzrušující? Možná. Koneckonců, vývojáři nikdy nepotřebovali pouze samotný nástroj, ale snadné přizpůsobení různým scénářům a flexibilní přepínání v případě potřeby.

Jako kódovací platforma podporovaná umělou inteligencí HagiCode nedávno přivítal dva důležité pokroky: za prvé, plnou integraci modelu GLM-5.1 Zhipu AI; za druhé, oficiální přidání Gemini CLI jako desátého podporovaného Agent CLI. Tyto dvě aktualizace nemusí znít otřesně, ale jsou nepochybně dobrou zprávou pro další zrání platformy.

GLM-5.1 je nejnovější vlajkový model Zhipu AI. Ve srovnání s GLM-5.0 nabízí silnější uvažování, hlubší porozumění kódu a hladší volání nástrojů. Ještě důležitější je, že je to první model GLM, který podporuje zadávání obrázků. co to znamená? To znamená, že uživatelé mohou nechat AI podívat se přímo na snímek obrazovky, místo aby se snažili popsat problém slovy. Jakmile toto pohodlí využijete, okamžitě pochopíte jeho hodnotu.

Zároveň prostřednictvím `HagiCode.Libs.Providers` Architektura HagiCode úspěšně integrovala Gemini CLI do platformy. Toto je nyní desátý agent CLI. Abych byl upřímný, dostat se do tohoto bodu přináší skromný pocit úspěchu.

Za zmínku také stojí, že funkce nahrávání obrázků HagiCode umožňuje uživatelům komunikovat s AI přímo prostřednictvím snímků obrazovky. I když běží GLM 4.7, platforma stále funguje dobře a již pomohla dokončit mnoho důležitých stavebních úkolů. Pokud jde o GLM-5.1, jde přirozeně ještě o krok dále.

## O HagiCode

Přístup sdílený v tomto článku vychází z našich praktických zkušeností v [HagiCode](https://hagicode.com) projekt. HagiCode je open-source platforma pro kódování podporovaná umělou inteligencí navržená tak, aby vývojářům poskytla flexibilního a výkonného programovacího asistenta AI prostřednictvím architektury s více modely a rozhraním CLI. Projektové úložiště: [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site)

## Návrh architektury multi-CLI

Jednou ze silných stránek HagiCode je jeho podpora pro více nástrojů CLI programování AI prostřednictvím jednotné abstrakční vrstvy. Výhoda tohoto návrhu je vlastně docela jednoduchá: mohou přijít nové nástroje, staré nástroje mohou zůstat a kódová základna se nepromění v chaos. Abychom byli spravedliví, tak by si každý přál, aby život fungoval.

### Výčet AIProviderType

Platforma definuje podporované typy poskytovatelů CLI prostřednictvím `AIProviderType` výčet:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI (new)
}
```

Jak vidíte, Gemini CLI se k této rodině připojuje jako desátý člen. Každé CLI má své vlastní odlišné charakteristiky a scénáře použití, takže uživatelé si mohou flexibilně vybrat podle svých potřeb. Vždyť do Říma vede mnoho cest; některé jsou jednoduše jednodušší než jiné.

### Architektura poskytovatele

`HagiCode.Libs.Providers` poskytuje jednotné rozhraní poskytovatele, díky kterému je každá integrace CLI standardizovaná a stručná. Vezměme si Gemini CLI jako příklad:

```csharp
public class GeminiProvider : ICliProvider<GeminiOptions>
{
    private static readonly string[] DefaultExecutableCandidates = ["gemini", "gemini-cli"];
    private const string ManagedBootstrapArgument = "--acp";

    public string Name => "gemini";
    public bool IsAvailable => _executableResolver.ResolveFirstAvailablePath(DefaultExecutableCandidates) is not null;
}
```

Výhody tohoto designu jsou:
- Integrace nového CLI vyžaduje implementaci pouze jedné třídy Provider
- Jednotná správa životního cyklu a sdružování relací
- Automatizované rozlišení aliasů a zjišťování spustitelných souborů

Jednoduše řečeno, tento design mění složité věci na jednodušší a usnadňuje život.

### Registr poskytovatelů

Registr poskytovatelů automaticky zpracovává mapování aliasů a registraci:

```csharp
if (provider is GeminiProvider)
{
    registry.Register(provider.Name, provider, ["gemini-cli"]);
    continue;
}
```

To znamená, že uživatelé mohou vyvolat Gemini CLI s oběma `gemini` nebo `gemini-cli`a systém jej automaticky rozpozná. Je to jako přítel s formálním jménem i přezdívkou - v každém případě lidé vědí, koho máte na mysli.

## Podpora modelu GLM-5.1

GLM-5.1 je nejnovější vlajkový model Zhipu AI a HagiCode pro něj dokončil plnou podporu.

### Katalog středních profesí

HagiCode spravuje všechny podporované modely prostřednictvím katalogu sekundárních profesí. Zde je konfigurace pro řadu GLM:

| ID modelu | Jméno | Podporuje obrázek | Kompatibilní rodiny CLI |
|----------|------|----------------|-------------------------|
| `glm-4.7` | GLM 4.7 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM 5 Turbo | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (starší) | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | **pravda** | claude, codebuddy, hermes, qoder, kiro |

Klíčové vlastnosti GLM-5.1 lze shrnout následovně:
- Samostatný identifikátor verze bez starších zavazadel
- První model GLM, který podporuje vkládání obrázků
- Silnější uvažování a porozumění kódu
- Široká kompatibilita s multi-CLI

### GLM-5.1 vs. GLM-5.0

Na úrovni kódu je klíčový rozdíl mezi GLM-5.1 a GLM-5.0 zobrazen zde:

```csharp
// GLM-5.0 (Legacy) - contains special retention logic
private const string Glm50CodebuddySecondaryProfessionId = "secondary-glm-5-codebuddy";
private const string Glm50CodebuddyModelValue = "glm-5.0";

// GLM-5.1 - standalone new model identifier
private const string Glm51SecondaryProfessionId = "secondary-glm-5-1";
private const string Glm51ModelValue = "glm-5.1";
```

GLM-5.0 nese označení „Legacy“, protože se jedná o starý identifikátor verze, který je zachován pro zpětnou kompatibilitu. GLM-5.1 je naproti tomu zbrusu nová samostatná verze bez historické zátěže. Některé věci zůstávají v minulosti; jiní cestují lehčí a pohybují se rychleji.

### Nakonfigurujte GLM-5.1

Zde je příklad konfigurace pro použití GLM-5.1 v HagiCode:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

## Funkce nahrávání obrázků

Podpora obrázků HagiCode je implementována prostřednictvím `SupportsImage` majetek na `SecondaryProfession`:

```csharp
public class HeroSecondaryProfessionSettingDto
{
    public bool SupportsImage { get; set; }
}
```

V katalogu sekundárních profesí vypadá konfigurace GLM-5.1 takto:

```json
{
  "id": "secondary-glm-5-1",
  "supportsImage": true
}
```

To znamená, že uživatelé mohou nahrávat snímky obrazovky přímo pro analýzu AI, například:
- Screenshoty chybových zpráv
- Problémy na obrazovce uživatelského rozhraní
- Grafy vizualizace dat
- Výsledky spuštění kódu

Již není potřeba vše popisovat ručně – stačí nahrát screenshot. Pohodlí této funkce je zřejmé, jakmile ji použijete. Někdy jeden pohled řekne víc než dlouhé vysvětlování.

## Integrace Gemini CLI

Jako desátý agent CLI je Gemini CLI integrován do HagiCode prostřednictvím standardní architektury poskytovatele.

### Možnosti konfigurace

Gemini CLI podporuje bohatou sadu konfiguračních možností:

```csharp
public class GeminiOptions
{
    public string? ExecutablePath { get; set; }
    public string? WorkingDirectory { get; set; }
    public string? SessionId { get; set; }
    public string? Model { get; set; }
    public string? AuthenticationMethod { get; set; }
    public string? AuthenticationToken { get; set; }
    public Dictionary<string, string?> AuthenticationInfo { get; set; }
    public Dictionary<string, string?> EnvironmentVariables { get; set; }
    public string[] ExtraArguments { get; set; }
    public TimeSpan? StartupTimeout { get; set; }
    public CliPoolSettings? PoolSettings { get; set; }
}
```

Tyto možnosti pokrývají vše od základního nastavení až po pokročilé funkce a poskytují uživatelům flexibilitu konfigurovat CLI podle jejich vlastních potřeb. Každý má jiný pracovní postup, takže trocha flexibility je vždy vítána.

### Komunikační protokol ACP

Gemini CLI podporuje ACP (Agent Communication Protocol), což je jednotný komunikační standard CLI HagiCode. Prostřednictvím ACP mohou různá CLI komunikovat s platformou konzistentním způsobem, což značně zjednodušuje integrační práci. Stručně řečeno, standardizuje komplikované části, takže každý může pracovat snadněji.

## Vlastní sestavovací praxe HagiCode

Když už mluvíme o praxi v reálném světě, nejlepším příkladem je vlastní pracovní postup sestavení platformy HagiCode. Vývojový proces HagiCode již plně využil schopností AI.

### Funguje dobře i s GLM 4.7

Design platformy HagiCode je dobře optimalizovaný, takže může stále poskytovat dobrý vývojový zážitek i s GLM 4.7. Platforma již pomohla dokončit několik důležitých stavebních projektů, včetně:
- Integrace více poskytovatelů CLI
- Implementace funkce nahrávání obrázků
- Generování dokumentace a publikování obsahu

To je vlastně dobrá věc. Ne každý potřebuje neustále to nejnovější. To, co vám nejvíce vyhovuje, je často nejdůležitější.

### GLM-5.1 přináší více s menší námahou

Po upgradu na GLM-5.1 jsou tyto schopnosti ještě silnější:
- Lepší porozumění kódu, omezení komunikace tam a zpět
- Přesnější analýza závislostí, která okamžitě ukazuje správným směrem
- Efektivnější diagnostika chyb, rychlejší lokalizace problémů
- Podpora pro obrazový vstup, zrychlení popisů problémů

Je to jako přejít z kola na auto. Stále můžete dosáhnout stejného cíle, ale rychlost a pohodlí nejsou stejné.

## Osvědčené postupy pro integraci multi-CLI

`HagiCode.Libs.Providers` poskytuje jednotný mechanismus pro registraci a použití:

```csharp
services.AddHagiCodeLibs();

var gemini = serviceProvider.GetRequiredService<ICliProvider<GeminiOptions>>();
var codebuddy = serviceProvider.GetRequiredService<ICliProvider<CodebuddyOptions>>();
var hermes = serviceProvider.GetRequiredService<ICliProvider<HermesOptions>>();
```

Tento návrh vkládání závislostí udržuje použití v různých CLI velmi stručné a také usnadňuje testování jednotek a zesměšňování. Čistý kód je způsob, jak být k sobě zodpovědný.

### Poznámky

Při skutečném používání je třeba mít na paměti několik věcí:

1. **Konfigurace klíče API**: Ujistěte se `ANTHROPIC_AUTH_TOKEN` je správně nastaven nebo model nelze vyvolat
2. **Dostupnost modelu**: GLM-5.1 musí být povoleno poskytovatelem odpovídajícího modelu
3. **Funkce obrázku**: Pouze modely s `supportsImage: true` můžete použít nahrávání obrázků
4. **Instalace CLI**: Před použitím Gemini CLI se ujistěte `gemini` nebo `gemini-cli` je v systému PATH

Mohou to být malé detaily, ale malé detaily, se kterými se špatně manipuluje, se mohou změnit ve velké problémy, takže stojí za to jim věnovat pozornost.

## Závěr

S plnou podporou GLM-5.1 a úspěšnou integrací Gemini CLI HagiCode dále posiluje své schopnosti jako multimodelová a multi-CLI programovací platforma AI. Tyto aktualizace dávají uživatelům nejen více možností, ale také demonstrují architekturu a škálovatelnost HagiCode zaměřenou na budoucnost.

Podpora obrázků GLM-5.1 v kombinaci s funkcí nahrávání snímků obrazovky HagiCode umožňuje nechat AI „pochopit z obrázku“ a výrazně snižuje náklady na popis problémů. A díky podpoře deseti CLI si uživatelé mohou flexibilně vybrat programovacího asistenta AI, který nejlépe vyhovuje jejich preferencím a scénářům. Větší výběr je téměř vždy dobrá věc.

A co je nejdůležitější, vlastní praxe sestavování HagiCode dokazuje, že platforma již může dobře fungovat a dokončit složité úkoly i s GLM 4.7, zatímco upgrade na GLM-5.1 může dále zlepšit efektivitu vývoje. Život je často také takový: nepotřebujete vždy to nejlepší, jen to, co vám vyhovuje. Samozřejmě, pokud to, co vám vyhovuje, může být ještě lepší, pak tím lépe.
