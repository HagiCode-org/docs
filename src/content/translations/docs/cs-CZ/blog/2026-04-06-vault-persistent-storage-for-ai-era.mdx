## Jak reprodukovat projekty v éře AI: Vault, systém trvalého úložiště mezi projekty

> Jak můžeme v éře vývoje za pomoci umělé inteligence pomoci asistentům umělé inteligence lépe porozumět našim učebním zdrojům? Projekt HagiCode vytvořil systém Vault jako sjednocenou vrstvu abstrakce pro ukládání znalostí, které umělá inteligence dokáže porozumět, což výrazně zlepšuje efektivitu učení prostřednictvím reprodukce projektu.

## Pozadí

V éře umělé inteligence se způsob, jakým se vývojáři učí nové technologie a architektury, zásadně mění. „Reprodukce projektů“ – tedy hluboké studium a učení se z kódu, architektury a návrhových vzorů vynikajících open source projektů – se stalo efektivním způsobem učení. Ve srovnání s tradičními metodami, jako je čtení knih nebo sledování videí, vám přímé čtení a provozování vysoce kvalitních projektů s otevřeným zdrojovým kódem pomůže mnohem rychleji porozumět skutečným inženýrským postupům.

Přesto tato metoda učení přináší několik problémů.

Učební materiály jsou příliš rozházené. Vaše poznámky mohou žít v Obsidian, úložiště kódu mohou být rozptýlena v různých složkách a historie konverzace vašeho asistenta AI se stane dalším izolovaným datovým ostrovem. Když chcete, aby umělá inteligence pomohla analyzovat projekt, musíte ručně zkopírovat úryvky kódu a uspořádat kontext, což je poměrně únavné.

Co je ještě nepříjemnější, je narušený kontext. Asistenti umělé inteligence nemají přímý přístup k vašim místním výukovým zdrojům, takže v každé konverzaci musíte znovu poskytovat informace o pozadí. Kromě toho se repozitáře reprodukovaného kódu rychle aktualizují, ruční synchronizace je náchylná k chybám a znalosti je obtížné sdílet v rámci více vzdělávacích projektů.

V kořenu všechny tyto problémy pocházejí z „datových ostrovů“. Pokud by existovala sjednocená vrstva abstrakce úložiště, která by asistentům AI umožňovala porozumět všem vašim výukovým zdrojům a přistupovat k nim, problém by byl úhledně vyřešen.

## Filozofie návrhu trezorového systému

Základní myšlenka systému Vault je jednoduchá: vytvořit jednotnou vrstvu abstrakce úložiště znalostí, které umělá inteligence rozumí. Z hlediska implementace má systém několik klíčových vlastností.

### Podpora více typů

Systém podporuje čtyři typy trezorů, z nichž každý odpovídá jinému scénáři použití:

```typescript
// folder: general-purpose folder type
export const DEFAULT_VAULT_TYPE = 'folder';

// coderef: a type specifically for reproduced code projects
export const CODEREF_VAULT_TYPE = 'coderef';

// obsidian: integrated with Obsidian note-taking software
export const OBSIDIAN_VAULT_TYPE = 'obsidian';

// system-managed: vault automatically managed by the system
export const SYSTEM_MANAGED_VAULT_TYPE = 'system-managed';
```

Mezi nimi se v HagiCode nejčastěji používá typ **coderef**. Je speciálně navržen pro projekty s reprodukovaným kódem, poskytuje standardizovanou strukturu adresářů a popisy metadat čitelné pomocí AI.

### Mechanismus trvalého ukládání

Registr úložiště je trvale uložen ve formátu JSON, což zajišťuje, že konfigurace zůstane dostupná i po restartu aplikace:

```csharp
public class VaultRegistryStore : IVaultRegistryStore
{
    private readonly string _registryFilePath;

    public VaultRegistryStore(IConfiguration configuration, ILogger<VaultRegistryStore> logger)
    {
        var dataDir = configuration["DataDir"] ?? "./data";
        var absoluteDataDir = Path.IsPathRooted(dataDir)
            ? dataDir
            : Path.GetFullPath(Path.Combine(Directory.GetCurrentDirectory(), dataDir));

        _registryFilePath = Path.Combine(absoluteDataDir, "personal-data", "vaults", "registry.json");
    }
}
```

Výhodou tohoto designu je, že je jednoduchý a spolehlivý. JSON je čitelný pro člověka, což usnadňuje ladění a ruční úpravy; úložiště souborového systému se vyhýbá složitosti databáze a snižuje systémové závislosti. Koneckonců, někdy je ta nejjednodušší možnost opravdu ta nejlepší.

### Integrace kontextu AI

A co je nejdůležitější, systém dokáže automaticky vložit informace z trezoru do kontextu návrhů AI:

```typescript
export function buildTargetVaultsText(
  vaults: VaultForText[],
  template: VaultPromptTemplate = DEFAULT_VAULT_PROMPT_TEMPLATE,
): string {
  const readOnlyVaults = vaults.filter((vault) => vault.accessType === 'read');
  const editableVaults = vaults.filter((vault) => vault.accessType === 'write');

  if (readOnlyVaults.length === 0 && editableVaults.length === 0) {
    return '';
  }

  const sections = [
    buildVaultSection(readOnlyVaults, template.reference),
    buildVaultSection(editableVaults, template.editable),
  ].filter(Boolean);

  return `\n\n### ${template.heading}\n\n${sections.join('\n')}`;
}
```

To umožňuje důležitou schopnost: Asistenti umělé inteligence mohou automaticky porozumět dostupným výukovým zdrojům, aniž by uživatelé ručně poskytovali kontext. Dalo by se říci, že se to počítá jako druh tichého porozumění.

## Standardizovaná struktura CodeRef Vault

Pro typ úschovny coderef poskytuje HagiCode standardizovanou strukturu adresářů:

```
my-coderef-vault/
├── index.yaml          # vault metadata description
├── AGENTS.md           # operating guide for AI assistants
├── docs/               # stores study notes and documents
└── repos/              # manages reproduced code repositories through Git submodules
```

Při vytváření úschovny systém automaticky inicializuje tuto strukturu:

```csharp
private async Task EnsureCodeRefStructureAsync(
    string vaultName,
    string physicalPath,
    ICollection<VaultBootstrapDiagnosticDto> diagnostics,
    CancellationToken cancellationToken)
{
    Directory.CreateDirectory(physicalPath);

    var indexPath = Path.Combine(physicalPath, CodeRefIndexFileName);
    var docsPath = Path.Combine(physicalPath, CodeRefDocsDirectoryName);
    var reposPath = Path.Combine(physicalPath, CodeRefReposDirectoryName);

    // Create the standard directory structure
    if (!Directory.Exists(docsPath))
    {
        Directory.CreateDirectory(docsPath);
    }

    if (!Directory.Exists(reposPath))
    {
        Directory.CreateDirectory(reposPath);
    }

    // Create the AGENTS.md guide
    await EnsureCodeRefAgentsDocumentAsync(physicalPath, cancellationToken);

    // Create the index.yaml metadata
    await WriteCodeRefIndexDocumentAsync(indexPath, mergedDocument, cancellationToken);
}
```

Tato struktura je také pečlivě navržena:

- **docs/** ukládá vaše studijní poznámky, kde si můžete zaznamenat své porozumění kódu, analýzu architektury, získané lekce a tak dále v Markdown
- **repos/** spravuje reprodukovaná úložiště prostřednictvím submodulů Git namísto přímého kopírování kódu, což udržuje kód v synchronizaci a šetří místo
- **index.yaml** obsahuje metadata úschovny, takže asistenti umělé inteligence mohou rychle pochopit účel a obsah úschovny
- **AGENTS.md** je příručka napsaná speciálně pro asistenty AI, která vysvětluje, jak zacházet s obsahem trezoru

Díky tomuto uspořádání možná umělá inteligence pochopí, co máte na mysli, o něco snadněji.

## Automatická inicializace pro úložiště spravované systémem

Kromě ručně vytvořených vaultů podporuje HagiCode také systémem spravované vaulty:

```csharp
public async Task<IReadOnlyList<VaultRegistryEntry>> EnsureAllSystemManagedVaultsAsync(
    CancellationToken cancellationToken = default)
{
    var definitions = GetAllResolvedDefinitions();
    var entries = new List<VaultRegistryEntry>(definitions.Count);

    foreach (var definition in definitions)
    {
        entries.Add(await EnsureResolvedSystemManagedVaultAsync(definition, cancellationToken));
    }

    return entries;
}
```

Systém automaticky vytváří a spravuje následující vaulty:

- **hagiprojectdata**: úložiště dat projektu používané k uložení konfigurace a stavu projektu
- **personaldata**: úložiště osobních údajů sloužící k uložení uživatelských preferencí
- **hbsprompt**: knihovna šablon výzev, která se používá ke správě běžně používaných výzev AI

Tyto vaulty se inicializují automaticky při spuštění systému, takže je uživatelé nemusí konfigurovat ručně. Některé věci je prostě lepší nechat na systému, než aby se o ně starali lidé.

## Mechanismus kontroly přístupu

Důležitou součástí návrhu je kontrola přístupu. Systém rozděluje trezory na dva typy přístupu:

```typescript
export interface VaultForText {
  id: string;
  name: string;
  type: string;
  physicalPath: string;
  accessType: 'read' | 'write';  // Key: distinguish read-only from editable
}
```

- **reference (pouze pro čtení)**: AI se používá pouze pro analýzu a porozumění a nemůže upravovat obsah. Vhodné pro odkazované open source projekty, dokumenty a podobné materiály
- **upravitelné (upravitelné)**: AI může upravovat obsah podle potřeby pro daný úkol. Vhodné pro vaše poznámky, koncepty a podobné materiály

Na tomto rozlišení záleží. Říká AI, který obsah je „referenční pouze pro čtení“ a který obsah je „bezpečné pro úpravy“, čímž se snižuje riziko náhodných změn. Nikdo přece nechce, aby jejich dřina zmizela kvůli nechtěné úpravě.

## V praxi: vytváření a používání Vaultu

Když už jsme probrali nápady, pojďme se podívat, jak to funguje v praxi.

### Vytvořte trezor CodeRef

Zde je úplný příklad frontendového volání:

```typescript
const createCodeRefVault = async () => {
  const response = await VaultService.postApiVaults({
    requestBody: {
      name: "React Learning Vault",
      type: "coderef",
      physicalPath: "/Users/developer/vaults/react-learning",
      gitUrl: "https://github.com/facebook/react.git"
    }
  });

  // The system will automatically:
  // 1. Clone the React repository into vault/repos/react
  // 2. Create the docs/ directory for notes
  // 3. Generate the index.yaml metadata
  // 4. Create the AGENTS.md guide file

  return response;
};
```

Toto volání API dokončí řadu akcí: vytvoření adresářové struktury, inicializace submodulů Git, generování souborů metadat a další. Stačí zadat základní informace a zbytek nechat na systému. Je to upřímně docela bezstarostný přístup.

### Použijte Vault v návrhu AI

Po vytvoření trezoru na něj můžete odkazovat v návrhu AI:

```typescript
const proposal = composeProposalChiefComplaint({

```

Systém automaticky vkládá informace vaultu do kontextu umělé inteligence a dává tak umělé inteligenci vědět, které výukové zdroje jsou k dispozici. Když umělá inteligence dokáže pochopit, co máte na mysli, je těžké dosáhnout takového tichého porozumění.

## Doporučené postupy a věci, na které si dát pozor

Při používání systému Vault jsme shrnuli několik získaných poznatků.

### Bezpečnost cesty

Systém přísně ověřuje cesty, aby zabránil útokům typu Traversal:

```csharp
private static string ResolveFilePath(string vaultRoot, string relativePath)
{
    var rootPath = EnsureTrailingSeparator(Path.GetFullPath(vaultRoot));
    var combinedPath = Path.GetFullPath(Path.Combine(rootPath, relativePath));
    if (!combinedPath.StartsWith(rootPath, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode,
            "Vault file paths must stay inside the registered vault root.");
    }
    return combinedPath;
}
```

To je důležité. Pokud přizpůsobíte cestu k úschovně, ujistěte se, že zůstává v povoleném rozsahu, jinak systém operaci odmítne. Opravdu nemůžete přehnaně zdůrazňovat bezpečnost.

### Správa submodulů Git

CodeRef Vault doporučuje submoduly Git namísto přímého kopírování kódu:

```csharp
private static string BuildCodeRefAgentsContent()
{
    return """
    # CodeRef Vault Guide

    Repositories under `repos/` should be maintained through Git submodules
    rather than copied directly into the vault root.

    Keep this structure stable so assistants and tools can understand the vault quickly.
    """ + Environment.NewLine;
}
```

To přináší několik výhod: zachování synchronizace kódu s upstreamem, úspora místa na disku a snadnější správa více verzí kódu. Koneckonců, kdo chce stahovat to samé znovu a znovu?

### Limity náhledu souborů

Aby se předešlo problémům s výkonem, systém omezuje velikost a typ souboru:

```csharp
private const int FileEnumerationLimit = 500;
private const int PreviewByteLimit = 256 * 1024;  // 256KB
```

Pokud vaše úložiště obsahuje velký počet souborů nebo velmi velké soubory, může to ovlivnit výkon náhledu. V takovém případě můžete zvážit zpracování souborů v dávkách nebo použití specializovaných vyhledávacích nástrojů. Někdy, když je něco příliš velké, stává se to hůře ovladatelné, ne snadnější.

### Diagnostické informace

Při vytváření úschovny systém vrací diagnostické informace, které vám pomohou s laděním:

```csharp
List<VaultBootstrapDiagnosticDto> bootstrapDiagnostics = [];

if (IsCodeRefVaultType(normalizedType))
{
    bootstrapDiagnostics = await EnsureCodeRefBootstrapAsync(
        normalizedName,
        normalizedPhysicalPath,
        normalizedGitUrl,
        cancellationToken);
}
```

Pokud se vytvoření nezdaří, můžete zkontrolovat diagnostické informace, abyste pochopili konkrétní příčinu. Když se něco pokazí, kontrola diagnostiky je často tou nejpřímější cestou vpřed.

## Shrnutí

Prostřednictvím jednotné vrstvy abstrakce úložiště řeší systém Vault několik klíčových bolestivých bodů reprodukce projektů v éře AI:

- **Centralizovaná správa znalostí**: všechny výukové zdroje jsou shromážděny na jednom místě a nejsou roztroušeny všude
- **Automatické vkládání kontextu AI**: Asistenti AI mohou automaticky porozumět dostupným výukovým zdrojům bez ručního nastavování kontextu
- **Opětovné využití znalostí napříč projekty**: znalosti lze sdílet a znovu použít v rámci více vzdělávacích projektů
- **Standardizovaná struktura adresářů**: konzistentní rozložení adresářů zkracuje křivku učení
