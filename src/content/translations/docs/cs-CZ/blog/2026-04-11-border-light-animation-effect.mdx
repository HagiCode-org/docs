## Průvodce vytvořením efektu animace ohraničení světlem

> Jak pomocí čistého CSS vytvoříte důležitý prvek, kterého si uživatelé na první pohled všimnou? Ve skutečnosti to není tak těžké. Trik je v tom, že se vydáte mírně okružní cestou. V tomto článku vás provedu tím, jak vytvořit hraniční světelnou animaci od nuly, a také se podělím o několik úskalí, na která jsme narazili při vytváření HagiCode.

## Pozadí

Pokud pracujete na frontendu, pravděpodobně jste tuto zkušenost již někdy zažili: produktový manažer k vám přijde s výrazem „tohle je určitě jednoduché“ a řekne: „Můžeme k této běžící úloze přidat nějaký speciální efekt, aby si to uživatelé mohli okamžitě všimnout?“

V tu chvíli můžete začít přemýšlet, jak to postavit. Plátno? SVG? Nebo to CSS zvládne samo? Koneckonců, nikdo nechce přiznat, že neví jak.

- **Sezónní témata**: Budování slavnostní atmosféry pro zvláštní příležitosti

## Základní nápady na implementaci

Po analýze kódové základny HagiCode jsme níže shrnuli několik základních vzorů implementace. Každý z nich vyhovuje jinému scénáři, nebo jinými slovy, každý existuje z nějakého důvodu.

### 1. Rotující záře s kuželovým gradientem (nejběžnější)

Toto je klasický způsob implementace efektu zametání okraje. Základní myšlenkou je použití CSS `conic-gradient` vytvořte kuželový přechod a poté jej plynule otáčejte. Jako pouliční lampa, která se v noci otáčí, stále krouží.

**Klíčové prvky:**
- Použijte `::before` pseudo-prvek k vytvoření vrstvy záře
- Použijte `conic-gradient` k definování rozložení barev přechodu
- Použijte `::after` pseudoprvek pro maskování středové oblasti (volitelné)
- Použijte `@keyframes` implementovat animaci otáčení

### 2. Boční žhavicí čára

To funguje dobře pro indikátory stavu v položkách seznamu. Namísto animace celého okraje vytvoříte tenkou zářící čáru na jedné straně prvku. Někdy stačí trochu světla. Nemusíte osvětlovat celý svět.

**Klíčové prvky:**
- Tenký absolutně umístěný liniový prvek
- Použijte `box-shadow` pro vytvoření efektu záře
- Použijte `scale` a `opacity` pro animaci dýchání

### 3. Zářící pozadí s box-shadow

Pokud nepotřebujete efekt úplného tažení a chcete pouze měkkou záři pozadí, vrstvení více `box-shadow` hodnoty stačí. Někdy je jednodušší varianta tím lepší.

### 4. Podpora usnadnění

Tuto část lze snadno přehlédnout, ale je nesmírně důležitá. Každá animace by měla odpovídat za `prefers-reduced-motion` dotaz na média a poskytují statickou alternativu pro uživatele, kteří animaci nechtějí. Ne každý má rád neustálý pohyb a na respektování této preference záleží.

## Možnosti implementace

### Možnost 1: Otočení okraje s kuželovým přechodem (doporučeno)

Toto je nejúplnější implementace světelného efektu sweeping border a je to také možnost, kterou v HagiCode používáme nejčastěji. Koneckonců, když něco funguje dobře, proč to nahrazovat?

```css
/* Parent container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotating glow layer */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Mask layer (optional, for creating a hollow border effect) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Princip této možnosti je poměrně jednoduchý: vytvořte pseudoprvek větší než nadřazený kontejner, nakreslete na něj kuželový přechod a plynule jej otáčejte. Používá nadřazený kontejner `overflow: hidden`, takže zůstane viditelné pouze světlo procházející kolem hranice. Je to trochu jako sledovat pouliční lampu oknem. Vždy vidíte jen ten malý kousek, který prochází kolem.

### Možnost 2: Zjednodušený otočný světelný rámeček

Pokud nepotřebujete plný efekt, HagiCode obsahuje také lehčí verzi užitkové třídy. Někdy je jednodušší přístup opravdu lepší.

```css
/* Rotating light border utility class */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Accessibility support */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Všimněte si `will-change: transform` tady. Řekne prohlížeči: „Tento prvek se bude neustále měnit“, takže prohlížeč může předem připravit některé optimalizace a udržet animaci plynulejší. Připravit se předem je obvykle lepší než škrábat na poslední chvíli.

### Možnost 3: Boční žhavicí čára

To je zvláště vhodné pro indikátory stavu položek seznamu a přesně to používá seznam relací HagiCode. Mezi mnoha položkami stále může vyniknout jedna tenká linie. To mi svým způsobem připadá jako životní lekce.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Toto používá `isolation: isolate` vytvořit nový kontext skládání, pak se spoléhá na `z-index` pro ovládání pořadí zobrazení každé vrstvy. `pointer-events: none` je také zásadní. Jinak by pseudoprvek blokoval uživatelská kliknutí. Některé věci mohou vypadat hezky, ale přesto by neměly překážet.

### Možnost 4: Zabalte to do komponenty React

Pokud váš projekt používá React, můžete tuto logiku zabalit do komponenty, zejména do manipulace s usnadněním. Napište to jednou, použijte to mnohokrát. To je celá podstata.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Odpovídající modul CSS:

```css
/* GlowBorder.module.css */

/* Animated version */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Static version (accessibility) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

The `useReducedMotion` háček od `framer-motion` automaticky detekuje systémové preference uživatele. Pokud uživatel povolil omezený pohyb, vrátí se `true`a komponenta místo toho zobrazuje statickou verzi. Respektování preferencí uživatele je důležitější než vynucení okázalého efektu.

## Získané praktické lekce

Toto jsou některé z lekcí, které jsme se naučili při vytváření HagiCode. Dalo by se jim také říkat bojové jizvy. Doufejme, že vám pomohou vyhnout se objížďkám.

### 1. Téma proměnný systém

Díky proměnným CSS je podpora více témat obzvláště pohodlná. Nikdo nechce upravovat hromadu kódu pokaždé, když se téma změní.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Usage */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

Tímto způsobem přepínání motivů vyžaduje pouze změnu třídy na `html` a každá barva animace se automaticky aktualizuje. Jedna kódová základna, dva styly. To je přesně to, co chceme.

### 2. Optimalizace výkonu

**Použití `will-change` nápověda k optimalizaci prohlížeče:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Řekněte prohlížeči předem a pomůže vám s optimalizací. Spousta věcí v životě funguje lépe s trochou přípravy.

**Vyhněte se použití složitých stínů rámečku na velkých prvcích:**

```css
/* Not ideal - using a blurred shadow on a large element */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Better - use a pseudo-element to limit the glowing area */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Testovali jsme to v HagiCode. Přidání rozmazaného stínu přímo na velkou kartu snížilo rychlost rolování pod 30 snímků za sekundu. Přechod na pseudoprvek přinesl věci zpět na stabilních 60 snímků za sekundu. Uživatelé tento rozdíl naprosto pocítí.

### 3. Dostupnost

Tohle byste opravdu neměli přeskočit. Někteří uživatelé považují animaci za závratnou nebo rušivou a respektování jejich preferencí je součástí budování dobrého produktu. Krásné věci není třeba vnucovat každému.

**Dotaz na média CSS:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Provide a static fallback */
    opacity: 1;
  }
}
```

**Zjistit uživatelské preference v React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Řízení úrovně intenzity

Indikátor propustnosti tokenu v HagiCode ukazuje různé barvy záře na základě propustnosti v reálném čase, a to je implementováno dynamicky. Různé stavy by měly být vyjádřeny odlišně.

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Na co si dát pozor

Stále existuje několik detailů, které stojí za pozornost, protože ve chvíli, kdy tyto problémy odhalíte tvrdě, je již příliš pozdě.

| Na co si dát pozor | Vysvětlení |
|---------|------|
| **správa z-indexu** | Zářící vrstva by měla používat vhodnou `z-index` takže nezasahuje do interakce s obsahem |
| **ukazatel-události** | Pseudoprvek záře by se měl nastavit `pointer-events: none` |
| **Přetečení hranice** | Rodičovský kontejner potřebuje `overflow: hidden`nebo potřebujete upravit velikost pseudoprvku |
| **Dopad na výkon** | Složité animace mohou snížit výkon na mobilních zařízeních, proto testujte pečlivě |
| **Tmavý režim** | Ujistěte se, že barva záře zůstává jasně viditelná na tmavém pozadí |
| **Přepínání motivu** | Použijte proměnné CSS, aby se barvy animace správně aktualizovaly při změně motivu |

### 6. Tipy pro ladění

Pseudoprvky může být trochu těžké najít ve vývojářských nástrojích, takže můžete dočasně přidat ohraničení pro kontrolu pozice.

```css
/* Temporarily show pseudo-element boundaries for debugging */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Po dokončení umístění nezapomeňte tento řádek okomentovat nebo odstranit. Jinak může být výroba velmi rychle nepohodlná. Některé věci je lepší nechat ve vývoji.
