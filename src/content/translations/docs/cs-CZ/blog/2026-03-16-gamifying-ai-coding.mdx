## Jak Gamification Design dělá kódování AI zábavnější

> Tradiční nástroje pro kódování AI jsou ve skutečnosti docela výkonné; jen jim chybí trochu tepla. Když jsme budovali HagiCode, říkali jsme si: když už budeme psát kód, proč to neudělat do hry?

## Pozadí

Každý, kdo použil asistenta kódování AI, má pravděpodobně tuto zkušenost: zpočátku to působí svěžím a vzrušujícím dojmem, ale po chvíli začne mít pocit, že mu něco chybí. Samotný nástroj je výkonný, schopný generování kódu, automatického doplňování a opravy chyb, ale... nepůsobí moc hřejivě a časem se může stát monotónním a nudným.

To samo o sobě stačí k tomu, abyste se divili, kdo chce každý den zírat na chladný, neosobní nástroj.

Je to trochu jako hrát hru. Pokud vše, co uděláte, je dokončení seznamu úkolů bez růstu postavy, odemykání úspěchů a koordinace týmu, rychle vás to přestane bavit. Krásné věci a lidé nepotřebují být vlastněni, abychom je ocenili; jejich krása stačí sama o sobě. Programovací nástroje ani nenabízejí takovou krásu, takže je snadné ztratit srdce.

Přesně na tento problém jsme narazili při vývoji [HagiCode](https://hagicode.com). Jako platforma pro spolupráci s více asistenty AI potřebuje HagiCode dlouhodobě udržet uživatele v zapojení. Ale ve skutečnosti je těžké se držet i skvělého nástroje, pokud postrádá jakékoli emocionální spojení.

Abychom tento problém vyřešili, učinili jsme odvážné rozhodnutí: proměnit programování ve hru. Ne povrchní typ s jednoduchým bodovým žebříčkem, ale skutečným gamifikovaným zážitkem při hraní rolí. Dopad tohoto rozhodnutí může být ještě větší, než si představujete.

Koneckonců, lidé potřebují ve svém životě trochu rituálu.

## O HagiCode

Myšlenky sdílené v tomto článku pocházejí z našich praktických zkušeností na [HagiCode](https://hagicode.com) projekt. HagiCode je platforma pro spolupráci s více asistenty AI, která podporuje spolupráci Claude Code, Codex, Copilot, OpenCode a další asistenty AI. Pokud máte zájem o multi-AI spolupráci nebo gamified programování, navštivte [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site) dozvědět se více.

Není na tom nic zvlášť tajemného. Jednoduše jsme proměnili programování v dobrodružství.

## Proč zvolit Gamification

Podstatou gamifikace není jen „přidání žebříčku“. Jde o vybudování kompletního systému pobídek, aby uživatelé mohli při plnění úkolů cítit růst, úspěch a společenské uznání.

Návrh gamifikace HagiCode se točí kolem jedné základní myšlenky: každý asistent umělé inteligence je „hrdina“ a uživatel je kapitánem tohoto týmu hrdinů. Tyto hrdiny vedete k dobývání různých "Dungeonů" (programovacích úkolů). Po cestě získávají hrdinové zkušenosti, zvyšují úroveň, odemykají schopnosti a váš tým také získává úspěchy.

To není trik. Je to návrh založený na psychologii lidského chování. Když je úkolům dán smysl a zpětná vazba o pokroku, zapojení a vytrvalost lidí se výrazně zvýší.

Jak říká staré přísloví: "Tento pocit se může stát vzpomínkou, i když nás v tu chvíli nechal zmatený." Tento emocionální zážitek přinášíme do nástroje, takže programování již není jen psaní kódu, ale cesta, kterou stojí za to si zapamatovat.

## Systém postav hrdiny

Hero je základním konceptem gamifikačního systému HagiCode. Každý hrdina představuje jednoho asistenta AI. Například Claude Code je hrdina a Codex je také hrdina.

### The Three Hero Slots

Hero má tři sloty pro vybavení a design je překvapivě elegantní:

1. **Slot CLI (hlavní třída)**: Určuje základní schopnosti hrdiny, například zda jde o Claude Code nebo Codex
2. **Model slot (sekundární třída)**: Určuje, který model se použije, například Claude 4.5 nebo Claude 4.6
3. **Slot stylu (styl)**: Určuje styl chování hrdiny, jako je „Fengluo Strategist“ nebo jiný styl

Kombinace těchto tří slotů vytváří jedinečné konfigurace Hero. Podobně jako ve hrách se staví vybavení, i zde si vybíráte správné nastavení podle úkolu. Koneckonců, to, co vám nejvíce vyhovuje, je nejdůležitější. Život je podobný: do Říma vede mnoho cest, ale některé jsou hladší než jiné.

### Hero Growth System

Každý hrdina má své vlastní zkušenosti a úroveň:

```typescript
type HeroProgressionSnapshot = {
  currentLevel: number;                    // Current level
  totalExperience: number;                 // Total experience
  currentLevelStartExperience: number;     // Experience at the start of the current level
  nextLevelExperience: number;             // Experience required for the next level
  experienceProgressPercent: number;       // Progress percentage
  remainingExperienceToNextLevel: number;  // Experience still needed for the next level
  lastExperienceGain: number;              // Most recent experience gained
  lastExperienceGainAtUtc?: string | null; // Time when experience was gained
};
```

Úrovně jsou rozděleny do čtyř stupňů a každý stupeň má pohlcující název:

```typescript
export const resolveHeroProgressionStage = (level?: number | null): HeroProgressionStage => {
  const normalizedLevel = Math.max(1, level ?? 1);
  if (normalizedLevel <= 100) return 'rookieSprint';      // Rookie sprint
  if (normalizedLevel <= 300) return 'growthRun';         // Growth run
  if (normalizedLevel <= 700) return 'veteranClimb';      // Veteran climb
  return 'legendMarathon';                                // Legend marathon
};
```

Od „nováčka“ po „legendu“ tato cesta růstu dává uživatelům jasný smysl pro směr a úspěch. Odráží osobní růst v životě, od zmatku po zralost, jen je zde hmatatelnější.

### Vytvoření vlastního hrdiny

Chcete-li vytvořit hrdinu, musíte nakonfigurovat tři sloty:

```typescript
const heroDraft: HeroDraft = {
  name: 'Athena',
  icon: 'hero-avatar:storm-03',
  description: 'A brilliant strategist',
  executorType: AIProviderType.CLAUDE_CODE_CLI,
  slots: {
    cli: {
      id: 'profession-claude-code',
      parameters: { /* CLI-related parameters */ }
    },
    model: {
      id: 'secondary-claude-4-sonnet',
      parameters: { /* Model-related parameters */ }
    },
    style: {
      id: 'fengluo-strategist',
      parameters: { /* Style-related parameters */ }
    }
  }
};
```

Každý hrdina má jedinečný avatar, popis a profesionální identitu, což dává tomu, co by jinak bylo chladným asistentem umělé inteligence, více osobnosti a vřelosti. Koneckonců, kdo chce pracovat s nástrojem, který nemá charakter?

## Dungeon systém

"Dungeon" je klasický herní koncept představující výzvu, kterou vyžaduje tým, aby se vypořádal. V HagiCode je každý pracovní postup Dungeon.

### Jak jsou organizovány dungeony

Dungeon organizuje pracovní postupy do různých "Dungeonů":

- **Proposal generation dungeon**: Zodpovědný za generování technických návrhů
- **Žalář provádění návrhu**: Zodpovědný za provádění úkolů v návrzích
- **Proposal archive dungeon**: Zodpovědný za organizaci a archivaci hotových návrhů

Každý dungeon má svého vlastního hrdinu kapitána a kapitán je automaticky vybrán jako první povolený hrdina.

Toto je opravdu jen dělba práce, jako v běžném životě, až na to, že se změnila v herní mechaniku.

### Mechanismus týmové spolupráce

Pro různé dungeony můžete nakonfigurovat různé skupiny hrdinů:

```typescript
const dungeonRoster: HeroDungeonRoster = {
  scriptKey: 'proposal.generate',
  displayName: 'Proposal Generation',
  members: [
    { heroId: 'hero-1', name: 'Athena', executorType: 'ClaudeCode' },
    { heroId: 'hero-2', name: 'Apollo', executorType: 'Codex' }
  ]
};
```

Athenu můžete například použít pro generování návrhů, protože je dobrá ve strategii, a Apollo pro implementaci kódu, protože je dobrá při provádění. Každý hrdina tak může hrát podle svých předností. Je to jako založit kapelu: každý má nástroj a společně vytvoří něco krásného.

### Dungeon Flow Control

Dungeon používá pevné `scriptKey` hodnoty k identifikaci různých pracovních postupů:

```typescript
// Script keys map to different workflows
const dungeonScripts = {
  'proposal.generate': 'Proposal Generation',
  'proposal.execute': 'Proposal Execution',
  'proposal.archive': 'Proposal Archive'
};
```

Tok stavu úkolu je: `queued` (čekání) -> `dispatching` (přiděluje se) -> `dispatched` (přiděleno). Celý proces je automatizovaný a nevyžaduje žádné ruční zásahy. To je také část naší líné stránky, protože kdo chce tyto věci spravovat ručně?

## XP a systém úrovní

XP je základním mechanismem zpětné vazby v systému gamifikace. Uživatelé získávají XP plněním úkolů, povyšují XP hrdiny a postupování na vyšší úroveň odemyká nové schopnosti a vytváří pozitivní zpětnou vazbu.

### Způsoby, jak získat XP

V HagiCode lze získat XP prostřednictvím následujících činností:

- Dokončení provádění kódu
- Úspěšné volání nástrojů
- Generování návrhů
- Operace řízení relace
- Projektové operace

Pokaždé, když je dokončena platná akce, příslušný hrdina získá XP. Stejně jako růst v životě se počítá každý krok, jen zde je tento růst kvantifikován.

### Vizualizace pokroku v reálném čase

XP a pokrok na úrovni jsou vizualizovány v reálném čase:

```typescript
type HeroDungeonMember = {
  heroId: string;
  name: string;
  icon?: string | null;
  executorType: PCode_Models_AIProviderType;
  currentLevel?: number;                    // Current level
  totalExperience?: number;                 // Total experience
  experienceProgressPercent?: number;       // Progress percentage
};
```

Uživatelé mohou vždy vidět úroveň a pokrok každého hrdiny a tato okamžitá zpětná vazba je klíčem k návrhu gamifikace. Lidé potřebují zpětnou vazbu, jak by jinak věděli, že se zlepšují?

## Systém úspěchu

Dalším důležitým prvkem v gamifikaci jsou úspěchy. Poskytují dlouhodobé cíle a spokojenost řízenou milníkem.

### Typy úspěchů

HagiCode podporuje několik typů úspěchů:

- **Úspěchy při generování kódu**: Vygenerujte X řádků kódu, vygenerujte soubory Y
- **Úspěchy správy relace**: Dokončete Z konverzace
- **Úspěchy v provozu projektu**: Práce napříč W projekty

Tyto úspěchy jsou skutečně jako milníky v životě, až na to, že jsme je proměnili v herní mechaniku.

### Úspěchové státy

Úspěchy mají tři stavy:

```typescript
type AchievementStatus = 'unlocked' | 'in-progress' | 'locked';
```

Tyto tři stavy mají jasné vizuální rozdíly:

- **Odemčeno**: Zlatý přechod s halo efektem
- **Probíhá**: Animace modrého pulzu
- **Zamčeno**: Šedá, se zobrazenými podmínkami odemknutí

Každý úspěch jasně zobrazuje svou spouštěcí podmínku, takže uživatelé vědí, co dělat dál. Když se lidé cítí ztraceni, vždy pomůže malé vedení.

### Vliv oslav na odemknutí

Když je úspěch odemčen, spustí se animace oslav. Tento druh pozitivního posílení dává uživatelům uspokojivý pocit „udělal jsem to“ a motivuje je, aby pokračovali. Malé odměny v životě fungují stejně: mohou být malé, ale štěstí může trvat dlouho.

## Zpráva o bitvě Denní zpráva o boji

Battle Report je jednou z charakteristických funkcí HagiCode. Na konci každého dne vygeneruje celoobrazovkovou zprávu ve stylu bitvy.

### Obsah zprávy

Zpráva o bitvě zobrazuje následující informace:

```typescript
type HeroBattleReport = {
  reportDate: string;
  summary: {
    totalHeroCount: number;        // Total number of Heroes
    activeHeroCount: number;       // Number of active Heroes
    totalBattleScore: number;      // Total battle score
    mvp: HeroBattleHero;           // Most valuable Hero
  };
  heroes: HeroBattleHero[];        // Detailed data for all Heroes
};
```

- Celkové skóre týmu
- Počet aktivních hrdinů
- Počet volání nástroje
- Celková pracovní doba
- MVP (nejcennější hrdina)
- Podrobná karta pro každého hrdinu

### Zobrazení zvýraznění MVP

MVP je nejvýkonnějším hrdinou dne a je ve zprávě zvýrazněn. Nejde jen o datovou statistiku, ale o formu cti a uznání. Koneckonců, kdo by nechtěl být uznáván?

### Podrobné karty hrdinů

Každá karta hrdiny obsahuje:

- Pokrok na úrovni
- XP získané
- Počet exekucí
- Doba použití

Tyto metriky pomáhají uživatelům jasně pochopit, jak si tým vede. Vidět výsledky vlastního úsilí je samo o sobě uspokojující.

## Technická realizace

Gamifikace systému HagiCode využívá zásobník moderních technologií a designové vzory. Není na tom nic zvlášť magického; právě jsme si vybrali nástroje, které se hodí k práci.

### Technologie Stack Choices

```typescript
// React + TypeScript for the frontend
import React from 'react';

// Framer Motion for animations
import { AnimatePresence, motion } from 'framer-motion';

// Redux Toolkit for state management
import { useAppDispatch, useAppSelector } from '@/store';

// shadcn/ui for UI components
import { Dialog, DialogContent } from '@/components/ui/dialog';
```

Framer Motion zvládá všechny animační efekty, shadcn/ui poskytuje základní komponenty uživatelského rozhraní a Redux Toolkit spravuje komplexní stav gamifikace. Dobré nástroje dělají dobrou práci.

### Gamified UI Design System

HagiCode používá styl designu Glassmorphism + Tech Dark:

```css
/* Primary gradient */
background: linear-gradient(135deg, #22C55E 0%, #25c2a0 50%, #06b6d4 100%);

/* Glass effect */
backdrop-filter: blur(12px);

/* Glow effect */
background: radial-gradient(circle at center, rgba(34, 197, 94, 0.15) 0%, transparent 70%);
```

Zelený gradient v kombinaci se skleněným morfismem vytváří technickou, futuristickou atmosféru. Vizuální krása je také součástí uživatelské zkušenosti.

### Animační efekty

Framer Motion se používá k vytváření plynulých vstupních animací:

```typescript
<motion.div
  animate={{ opacity: 1, y: 0 }}
  initial={{ opacity: 0, y: 18 }}
  transition={{ duration: 0.35, ease: 'easeOut', delay: index * 0.08 }}
  className="card"
>
  {/* Card content */}
</motion.div>
```

Každá karta vstupuje jedna po druhé se zpožděním 0,08 sekundy a vytváří plynulý vizuální efekt. Plynulá animace zlepšuje zážitek. S touto částí je těžké polemizovat.

### Stálost dat

Data gamifikace jsou ukládána pomocí systému úložiště Grain, aby byla zajištěna konzistence stavu. Dokonce i jemnozrnná data, jako jsou nashromážděné Hero XP, mohou být zachovány přesně. Nikdo nechce přijít o zkušenosti, které si tvrdě vydělal.

## Praktický průvodce

### Vytvořte si svého prvního hrdinu

Vytvoření prvního hrdiny je vlastně docela jednoduché:

1. Přejděte na stránku správy hrdinů
2. Klikněte na tlačítko „Vytvořit hrdinu“.
3. Nakonfigurujte tři sloty (CLI, Model, Style)
4. Dejte hrdinovi jméno a popis
5. Uložte to a váš první hrdina je na světě

Je to jako setkání s novým přítelem: dáte mu jméno, dozvíte se, čím je výjimečný, a pak se společně vydáte za dobrodružstvím.

### Sestavte tým Dungeon

Sestavení týmu je také jednoduché:

1. Přejděte na stránku správy Dungeonu
2. Vyberte dungeon, který chcete nakonfigurovat, například „Generování návrhu“
3. Vyberte členy ze seznamu hrdinů
4. Systém automaticky vybere prvního povoleného hrdinu jako kapitána
5. Uložte konfiguraci

Toto je jednoduše proces formování týmu, podobně jako budování týmu v reálném životě, kde má každý svou vlastní roli.

### Zobrazit denní zprávu

Na konci každého dne si můžete prohlédnout denní bitevní zprávu:

1. Klikněte na tlačítko "Hlášení o bitvě".
2. Prohlédněte si výsledky práce za celý den na celé obrazovce
3. Zkontrolujte MVP a podrobné údaje pro každého hrdinu
4. Pokud chcete, sdílejte to se členy týmu

Je to také druh rituálu, způsob, jak zjistit, kolik úsilí jste dnes vynaložili a jak daleko jste ještě od svého cíle.

## Poznámky a doporučené postupy

### Optimalizace výkonu

Použijte `React.memo` abyste se vyhnuli zbytečnému překreslování:

```typescript
const HeroCard = React.memo(({ hero }: { hero: HeroDungeonMember }) => {
  // Component implementation
});
```

Důležitý je také výkon. Nikdo nechce používat laggy nástroj.

### Pohyb může ladně degradovat

Zjistěte nastavení předvoleb pohybu uživatele a poskytněte uživatelům citlivým na pohyb zjednodušené prostředí:

```typescript
const prefersReducedMotion = useReducedMotion();
const duration = prefersReducedMotion ? 0 : 0.35;
```

Ne každý má rád animace a respektování uživatelských preferencí je součástí dobrého designu.

### Zpětná kompatibilita

Nechat `legacyIds` pro podporu migrace ze starších verzí:

```typescript
type HeroDungeonMember = {
  heroId: string;
  legacyIds?: string[];  // Supports legacy ID mapping
  // ...
};
```

Nikdo nechce přijít o data jen kvůli upgradu verze.

### Podpora internacionalizace

Používejte překladové klíče i18n pro veškerý text, aby byla podpora více jazyků snadná:

```typescript
const displayName = t(`dungeon.${scriptKey}`, { defaultValue: displayName });
```

Jazyk by nikdy neměl být překážkou při používání produktu.

## Shrnutí

Gamifikace není jen jednoduchý bodový žebříček, ale kompletní motivační systém. Prostřednictvím systému Hero, Dungeon systému, XP a systému úrovní, systému úspěchů a Battle Reportu přeměňuje HagiCode programátorskou práci na hrdinskou cestu plnou dobrodružství.

Hlavní hodnota tohoto systému spočívá v:

- **Emocionální spojení**: Dodává chladným asistentům AI osobnost
- **Pozitivní zpětná vazba**: Každá akce vytváří okamžitou zpětnou vazbu
- **Dlouhodobé cíle**: Úrovně a úspěchy poskytují cestu růstu
- **Týmová identita**: Pocit spolupráce v týmech Dungeon
- **Čest a uznání**: Battle Report a ukázky MVP

Díky designu gamifikace už není programování nudné, ale zajímavé dobrodružství. Při plnění úkolů kódování uživatelé také zažijí zábavu z růstu postavy, týmové spolupráce a odemykání úspěchů, což zlepšuje udržení a aktivitu.

Ve svém jádru je programování již aktem stvoření. Jen jsme udělali kreativní proces trochu zábavnějším.
