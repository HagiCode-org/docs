Docker Compose je ve většině případů výchozí cesta nasazení, protože:

:::

## Předpoklady

Před spuštěním se ujistěte, že váš systém má nainstalované Docker a Docker Compose.

### Nainstalujte Docker

#### Windows

Stáhněte a nainstalujte [Docker Desktop pro Windows](https://www.docker.com/products/docker-desktop/)

Po instalaci se ujistěte, že Docker Desktop běží.

#### macOS

Stáhněte a nainstalujte [Docker Desktop pro Mac](https://www.docker.com/products/docker-desktop/)

Po instalaci se ujistěte, že Docker Desktop běží.

#### Linux

Nainstalujte Docker pomocí správce balíčků:

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

Po instalaci spusťte službu Docker:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Ověřte instalaci

Po instalaci spusťte následující příkazy a ověřte, zda jsou Docker a Docker Compose správně nainstalovány:

```bash
docker --version
docker compose version
```

## Rychlý start

### 1. Vygenerujte konfigurační soubor Docker Compose

:::tip[Použít generátor konfigurace]
**Doporučuji používat [Docker Compose Builder (samostatný web)](https://builder.hagicode.com/)** pro vygenerování konfiguračního souboru.

Generátor podporuje:
- Více možností poskytovatele API (Zhipu AI, Anthropic Official, Custom API)
- Automatická konfigurace API URL a tokenu
- Vlastní porty, názvy kontejnerů a další základní konfigurace
- Nakonfigurujte trvalá data a cesty pracovních adresářů
- Automaticky konfigurujte rozdíly mezi platformami Windows/Linux
- Inteligentně řešte problémy s oprávněními k souborům
- **Výběr zdroje obrazu**: Na základě vašeho síťového prostředí vyberte vhodný zdroj obrazu

[🚀 Okamžitě použijte generátor →](https://builder.hagicode.com/)
:::

1. Otevřít [Docker Compose Builder (samostatný web)](https://builder.hagicode.com/)
2. Vyplňte konfiguraci podle svých potřeb:
   - Vyberte poskytovatele rozhraní API (Zhipu AI, Anthropic Official nebo Custom API)
   - Nakonfigurujte porty, databázi, pracovní adresář a další možnosti
   - **Vybrat zdroj obrazu**: Vyberte vhodný zdroj obrazu podle vašeho síťového prostředí
3. Kliknutím na tlačítko generovat získáte `docker-compose.yml` konfigurace
4. Uložte vygenerovanou konfiguraci jako a `docker-compose.yml` soubor
5. V případě potřeby vytvořte a `.env` soubor pro konfiguraci citlivých informací

#### Výběr zdroje obrázku

Generátor podporuje tři možnosti zdroje obrazu:

**Docker Hub**
- **Adresa obrázku**: `newbe36524/hagicode:{tag}`
- **Použitelné scénáře**: Vhodné pro uživatele s akcelerací zrcadlení Docker Hub
- **Výhody**: Oficiální zdroj obrázků, včasné aktualizace, stabilní přístup
- **Poznámky**: V některých oblastech může být nutné nakonfigurovat zrychlení zrcadlení

**Azure Container Registry (ACR)**
- **Adresa obrázku**: `hagicode.azurecr.io/hagicode:{tag}`
- **Použitelné scénáře**: Vhodné pro uživatele, kteří nemají místní přístup k Docker Hub
- **Výhody**: Poskytuje alternativní zdroj obrazu, řeší problémy s přístupem k síti
- **Poznámky**: Obrázky zůstávají synchronizovány s Docker Hub, ale mohou mít krátké zpoždění

**Alibaba Cloud Container Registry (ACR)**
- **Adresa obrázku**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **Použitelné scénáře**: Vhodné pro domácí uživatele, stabilní a rychlý přístup k síti
- **Výhody**: Domácí uzly Alibaba Cloud, vysoká rychlost přístupu, vysoká stabilita
- **Poznámky**: Obrázky Alibaba ACR zůstávají synchronizovány s Docker Hub, obvykle do 30 minut

:::tip[Doporučení pro výběr zdroje obrázků]
- **Domácí uživatelé doporučují Alibaba Cloud ACR**: Pokud jste domácím uživatelem, doporučuje se upřednostnit použití zdroje obrazu Alibaba Cloud ACR pro rychlý a stabilní přístup
- **Alternativní Docker Hub**: Pokud má vaše síťové prostředí přístup k Docker Hub, můžete také zvolit Docker Hub jako zdroj obrázků
- **Alternativní Azure ACR**: Zdroj bitové kopie Azure ACR vyberte pouze v případě, že jiné zdroje bitové kopie mají potíže s přístupem
- **Synchronizace obrázků**: Obrázky Alibaba Cloud ACR zůstávají synchronizovány s Docker Hub, obvykle do 30 minut
- **Konzistence verzí**: Značky obrázků jsou konzistentní ve všech třech zdrojích obrázků, není třeba se obávat problémů s verzí
:::

:::note[Získat token API]
Chcete-li používat Hagicode, musíte nakonfigurovat Claude API Token:

**Zhipu AI (doporučeno)**: [Získat token API →](https://www.bigmodel.cn/glm-coding?ic=14BY54APZA)
- Stabilní domácí přístup, rychlejší odezva
- Vysoká cena-výkon, vhodné pro každodenní použití

:::

### 2. Spusťte Služby

Spusťte následující příkaz v adresáři, kde je váš `docker-compose.yml` se nachází:

```bash
docker compose up -d
```

Tento příkaz bude:
- Stáhněte a spusťte kontejnery Hagicode definované vygenerovanou šablonou
- Vytvořte a nakonfigurujte síť
- Inicializujte výchozí datový adresář a závislosti běhového prostředí

### 3. Ověřte stav služby

Zkontrolujte, zda běží kontejnery:

```bash
docker compose ps
```

Služby definované ve vygenerované šabloně byste měli vidět ve stavu „běžící“.

### Zobrazit protokoly služeb

Chcete-li zobrazit všechny protokoly služeb:

```bash
# View all service logs
docker compose logs

# View Hagicode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## Pokyny pro konfiguraci

Pokud potřebujete upravit konfiguraci (například změnit porty, přepnout poskytovatele API atd.), můžete:

1. Obnovte novou konfiguraci pomocí [Docker Compose Builder (samostatný web)](https://builder.hagicode.com/)
2. Ručně upravit `docker-compose.yml` a `.env` soubory
3. Restartujte služby, aby se konfigurace projevila:

```bash
docker compose restart
```

## Přístupová aplikace

### webové rozhraní

Po úspěšném spuštění služby přejděte prostřednictvím prohlížeče:

```
http://localhost:45000
```

### Stálost dat

Podporovaná cesta Docker Compose ve výchozím nastavení používá datové soubory SQLite s trvalými svazky. Pokud potřebujete zkontrolovat přípojné body nebo naplánovat zálohy, prohlédněte si mapování svazků a datových adresářů ve vygenerovaných `docker-compose.yml`.
