## Přehled

### Co je hlasový vstup Doubao

Doubao Voice Input je integrovaná funkce HagiCode pro rozpoznávání řeči, podporovaná ByteDance Doubao Open Platform. Mluvíte a systém převádí řeč na text. HagiCode také využívá aktuální kontext projektu, aby pomohl spolehlivěji rozpoznat slovní zásobu domény a technické výrazy.

### Klíčové případy použití

V HagiCode funguje hlasový vstup nejlépe pro:

- **Rychlé zadávání návrhů**: Vytvářejte a odesílejte návrhy pouhým mluvením, což výrazně zvyšuje efektivitu bez psaní
- **Poskytování komentářů**: Přidávejte komentáře a zpětnou vazbu během kontroly kódu nebo kontroly dokumentů pomocí hlasového vstupu
- **Odpovídání na zprávy**: Rychle odpovídejte na zprávy v konverzacích a diskuzích bez psaní
- **Vytváření dlouhého obsahu**: Rychle generujte projektovou dokumentaci, technické specifikace, poznámky k jednání a další dlouhý obsah

### Praktické výhody

- **Hluboká integrace s Context Awareness**: HagiCode se hluboce integruje s rozpoznáváním řeči Doubao, automaticky využívá aktuální kontext projektu (jako je struktura kódu, zásobník technologií, doménová terminologie) k zajištění přesného rozpoznávání slovní zásoby domény, což výrazně zlepšuje přesnost rozpoznávání technických termínů
- **Zkušební hodiny zdarma**: Platforma Doubao poskytuje novým uživatelům 20 hodin volného času na rozpoznávání, aby mohli zažít hlasový vstup s nulovou bariérou
- **Extrémně rychlé**: Rozpoznávání v reálném čase, text, když mluvíte, bez čekání
- **Vysoká přesnost**: Využívá pokročilých modelů rozpoznávání řeči Doubao v kombinaci s kontextem projektu pro přesné a spolehlivé výsledky
- **Bezproblémová integrace**: Přímo integrováno do vstupního pole zpráv HagiCode, není třeba přepínat aplikace
- **Snadné použití**: Začněte jednoduše kliknutím na mikrofon, intuitivní a přímočaré

## Rychlý start

### Předpoklady

Před použitím hlasového vstupu Doubao musíte:

1. **Získejte účet Doubao Open Platform**
   - Návštěva [Konzola pro rozpoznávání řeči Doubao](https://console.volcengine.com/speech/service/10038)
   - Zaregistrujte se nebo se přihlaste ke svému účtu

2. **Vytvořte aplikaci a získejte přihlašovací údaje**
   - Vytvořte na platformě aplikaci pro rozpoznávání řeči
   - Získejte své `APP ID` a `Access Token`

3. **Zajistěte připojení k síti**
   - Služba rozpoznávání řeči vyžaduje připojení k síti
   - Ujistěte se, že vaše zařízení má přístup ke službě Doubao API

### Základní tok použití

Zde jsou základní kroky pro použití hlasového vstupu Doubao:

1. Získejte `APP ID` a `Access Token`
2. Nakonfigurujte rozpoznávání hlasu v produktu
3. Otestujte klíč API
4. Najděte pole pro hlasový vstup
5. Klepněte na tlačítko mikrofonu
6. Udělte oprávnění k mikrofonu
7. Spusťte nahrávání
8. Zobrazte výsledky rozpoznávání v reálném čase
9. Kliknutím zastavíte nahrávání
10. Potvrďte vložení textu

## Technické požadavky

### Prostředí HTTP

Hlasový vstup Doubao vyžaduje spuštění v prostředí HTTP/HTTPS.

| Životní prostředí | Stav podpory | Popis |
|-------------|----------------|-------------|
| **místní hostitel** | ✓ | Místní vývojové prostředí (http://localhost nebo http://127.0.0.1) |
| **Vzdálený server HTTPS** | ✓ | Webové aplikace nasazené do veřejných sítí s HTTPS |
| **Vzdálený server HTTP** | ✗ | Webové aplikace nasazené do veřejných sítí s HTTP |
| **file:// protokol** | ✗ | Přímé otevírání souborů HTML není podporováno |

:::pozor
Hlasový vstup Doubao má následující omezení:
1. **Pro externí nasazení je vyžadováno HTTPS**: Kvůli omezením zabezpečení prohlížeče vyžaduje přístup k mikrofonu HTTPS při přístupu z veřejných sítí
2. **Místní adresy bez omezení**: Při použití jsou podporovány HTTP i HTTPS `localhost` nebo `127.0.0.1`
3. **protokol file:// není podporován**: The `file://` prostředí protokolu nemůže používat připojení mikrofonu a WebSocket
:::

### Verze pro stolní počítače

Desktopová verze HagiCode má vestavěné lokální HTTP prostředí a plně podporuje hlasový vstup. Rozpoznávání řeči můžete používat přímo bez další konfigurace.

### Režim hostitele

Režim místního hostitele podporuje hlasový vstup:
- Při použití jsou podporovány HTTP i HTTPS `localhost` nebo `127.0.0.1`
- HTTPS je vyžadováno při nasazení do veřejných sítí

## Kroky konfigurace

### Získejte ID APP

1. Návštěva [Konzola pro rozpoznávání řeči Doubao](https://console.volcengine.com/speech/service/10038)
2. Přihlaste se nebo si zaregistrujte účet
3. Přejděte do konzoly a vytvořte novou aplikaci pro rozpoznávání řeči
4. Na stránce podrobností aplikace vyhledejte a zkopírujte soubor `APP ID`

### Získejte přístupový token

1. V konzole Doubao Open Platform
2. Přejděte do aplikace pro rozpoznávání řeči
3. Najděte oblast správy klíčů API
4. Vygenerujte nebo zkopírujte `Access Token`

:::pozn
Prosím, ponechte si svůj `Access Token` bezpečné a nesdílejte je s ostatními. Pokud ji potřebujete vyměnit, můžete ji regenerovat na otevřené platformě Doubao.
:::

### Nakonfigurujte v produktu

1. Otevřete aplikaci HagiCode
2. Přejděte na **Nastavení** → **Nastavení rozpoznávání hlasu**
3. Do konfiguračního formuláře vyplňte následující údaje:
   - **Poskytovatel**: Vyberte `doubao` (Doubao)
   - **APP ID**: Vložte ID aplikace, které jste získali z platformy Doubao
   - **Přístupový token**: Vložte přístupový token, který jste získali z platformy Doubao
4. (Volitelné) Podle potřeby upravte další konfigurační parametry
5. Kliknutím na tlačítko **Test API Key** ověřte konfiguraci
6. Po úspěšném ověření se konfigurace automaticky uloží do místního úložiště prohlížeče

### Testovací klíč API

Po konfiguraci se doporučuje otestovat, zda je klíč API platný:

1. Klikněte na tlačítko **Test API Key**
2. Systém zavolá testovací rozhraní, aby ověřil vaši konfiguraci
3. Pokud je konfigurace správná, zobrazí se zpráva o úspěchu
4. Pokud je konfigurace nesprávná, zobrazí se chybová zpráva. Zkontrolujte prosím:
   - Zda jsou ID APP a přístupový token správné
   - Zda je připojení k síti normální
   - Zda vypršela platnost přístupového tokenu

:::tip
Pokud se test nezdaří, nejprve zkontrolujte, zda je síťové připojení normální, a poté potvrďte, že klíč API z Doubao Open Platform je platný a nevypršela jeho platnost.
:::

## Využití

### Najděte umístění hlasového vstupu

V HagiCode podporují textová pole s ikonou mikrofonu hlasový vstup. Tyto komponenty se obvykle nazývají `VoiceTextArea`.

Mezi běžná umístění hlasového vstupu patří:
- Pole pro zadávání zpráv
- Oblasti úpravy textu
- Různá pole formuláře, která vyžadují zadávání textu

### Spusťte rozpoznávání hlasu

1. Najděte vstupní pole s ikonou mikrofonu
2. Klepněte na tlačítko mikrofonu
3. Prohlížeč požádá o povolení mikrofonu
4. Kliknutím na **Povolit** povolíte přístup k mikrofonu

:::pozn
Pokud jste mikrofon dříve autorizovali, prohlížeč nemusí znovu žádat o povolení. Chcete-li zkontrolovat nebo změnit nastavení oprávnění, můžete je zobrazit v ikoně zámku vlevo od adresního řádku prohlížeče.
:::

### Popis stavu nahrávání

Po úspěšné autorizaci se automaticky spustí rozpoznávání hlasu a uvidíte:

- ** Animace křivky**: Dynamická křivka zvukové vlny se zobrazí uvnitř tlačítka mikrofonu, což znamená, že probíhá nahrávání
- **Zobrazení trvání**: Aktuální doba nahrávání se zobrazí pod tlačítkem
- **Rozpoznávání v reálném čase**: Rozpoznaný text se dočasně zobrazí na pozici kurzoru

### Zobrazení výsledků rozpoznávání v reálném čase

Během nahrávání modul rozpoznávání řeči převede vaši řeč na text v reálném čase:

- Rozpoznaný text se ve vstupním poli dočasně zobrazí šedě
- Výsledky rozpoznávání se budou průběžně aktualizovat, když mluvíte
- Podporuje rozpoznávání mandarínské čínštiny s vysokou přesností

:::tip
Pro lepší výsledky rozpoznávání se doporučuje:
- Používejte v tichém prostředí
- Mluvte jasně a mírným tempem
- Vyhněte se rušení šumem na pozadí
:::

### Zastavit rozpoznávání

Chcete-li zastavit rozpoznávání hlasu, můžete:

1. **Klikněte na tlačítko mikrofonu**: Dalším kliknutím na tlačítko nahrávání zastavíte
2. **Klikněte na vstupní pole**: Kliknutím na další oblasti vstupního pole také zastavíte nahrávání

Po zastavení bude konečný rozpoznaný text formálně vložen do vstupního pole a můžete pokračovat v úpravách nebo odesílání.
