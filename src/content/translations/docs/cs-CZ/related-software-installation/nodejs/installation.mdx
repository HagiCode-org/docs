Tato příručka popisuje instalaci Node.js na Windows, macOS a Linux. Node.js je klíčová závislost pro místní nástroje HagiCode, včetně OpenSpec, frontendových nástrojů a několika vývojářských CLI.

:::tip[Požadavky na verzi]
HagiCode doporučuje **Node.js 24+** (minimálně podporovaný Node.js &gt;= 18,0). Ve většině případů nainstalujte nejnovější verzi LTS a poté ověřte, zda vaše verze splňuje požadavky.
:::

## Instalace Node.js

<Tabs>
  <TabItem value="windows" label="Windows" default>

### První metoda: Oficiální instalační program (doporučeno)

1. Navštivte [Oficiální webové stránky Node.js](https://nodejs.org/)
2. Stáhněte si **LTS (Long Term Support)** verzi instalačního programu Windows (`.msi` soubor)
3. Dvojitým kliknutím spustíte instalační program
4. Dokončete instalaci podle pokynů průvodce instalací:
   - Přijměte licenční smlouvu
   - Vyberte instalační cestu (výchozí nastavení je v pořádku)
   - Ujistěte se, že je zaškrtnuto "Automaticky nainstalovat potřebné nástroje".
   - Klepnutím na „Instalovat“ zahájíte instalaci
5. Po dokončení instalace klikněte na „Dokončit“

### Metoda 2: Použití wingetu

Otevřete PowerShell nebo příkazový řádek a spusťte:

```powershell
winget install OpenJS.NodeJS.LTS
```

Po instalaci znovu otevřete terminál, aby se změny projevily.

  </TabItem>
  <TabItem value="macos" label="macOS">

### První metoda: Oficiální instalační program (doporučeno)

1. Navštivte [Oficiální webové stránky Node.js](https://nodejs.org/)
2. Stáhněte si instalační program **LTS (Long Term Support)** verze macOS (`.pkg` soubor)
3. Dvojitým kliknutím spustíte instalační program
4. Dokončete instalaci podle pokynů průvodce instalací:
   - Klikněte na "Pokračovat"
   - Přijměte licenční smlouvu
   - Vyberte instalační disk
   - Klikněte na "Instalovat" a zadejte heslo správce
5. Po dokončení instalace klikněte na "Zavřít"

### Metoda 2: Použití Homebrew

Pokud máte [Homebrew](https://brew.sh/) nainstalován, můžete nainstalovat Node.js přes terminál:

```bash
brew install node
```

:::tip
Homebrew automaticky přidá Node.js do vaší PATH. Pokud příkaz není k dispozici, restartujte terminál nebo spusťte:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

  </TabItem>
  <TabItem value="ubuntu" label="Ubuntu/Debian">

### Použití úložiště NodeSource (doporučeno)

Toto je nejlepší způsob, jak získat nejnovější verzi Node.js:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Pomocí Správce balíčků apt

Pokud nepotřebujete nejnovější verzi, můžete použít výchozí úložiště Ubuntu:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::pozn
Verze Node.js ve výchozím úložišti může být starší. Po instalaci zkontrolujte verzi a ujistěte se, že splňuje požadavky Node.js &gt;= 18.0 (doporučuje se Node.js 24+).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### Použití úložiště NodeSource (doporučeno)

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### Pomocí správce balíčků yum

```bash
sudo yum install -y nodejs npm
```

  </TabItem>
  <TabItem value="fedora" label="Fedora">

### Použití Správce balíčků dnf

```bash
sudo dnf install -y nodejs npm
```

:::tip
Softwarové úložiště Fedory obvykle obsahuje novější verze Node.js a lze jej nainstalovat přímo přes dnf.
:::

  </TabItem>
</Tabs>

## Použití nástrojů pro správu verzí

Pokud potřebujete přepínat mezi více verzemi Node.js, můžete použít nástroje pro správu verzí.

### nvm (Node Version Manager)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

Po instalaci znovu načtěte konfiguraci shellu:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

V systému Windows doporučujeme použít [nvm-windows](https://github.com/coreybutler/nvm-windows):

1. Stáhněte si nejnovější `nvm-setup.exe` od [vydání nvm-windows](https://github.com/coreybutler/nvm-windows/releases)
2. Spusťte instalační program
3. Znovu otevřete příkazový řádek nebo PowerShell

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

Po instalaci nvm jej můžete použít k instalaci a správě verzí Node.js:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (Fast Node Manager)

fnm je rychlejší a jednodušší nástroj pro správu verzí Node.js vytvořený s Rustem.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

Použijte [naběračka](https://scoop.sh/) nebo [winget](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Zrychlení pro uživatele z pevninské Číny

Pokud se nacházíte v pevninské Číně, stahování balíčků Node.js a npm může být pomalé. Zde jsou některá řešení zrychlení.

### Použití domácích zrcadlových zdrojů

#### npm Taobao Mirror (doporučeno)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### Použití nrm ke správě zrcadlových zdrojů

nrm (správce registru npm) vám pomůže rychle přepínat zdroje zrcadlení npm:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Zrychlení stahování instalačního programu Node.js

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao poskytuje zrcadlové stahování pro instalační programy Node.js:

- **Adresa ke stažení**: https://npmmirror.com/mirrors/node/

Navštivte výše uvedenou adresu a vyberte verzi, kterou potřebujete stáhnout.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

Pokud je stahování nvm nebo jiných nástrojů z GitHubu pomalé, můžete použít zrcadlo FastGit:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::pozn
Služba FastGit se může změnit, podívejte se prosím na oficiální informace.
:::

  </TabItem>
</Tabs>

### Domácí Homebrew Zrcadla

Pokud používáte Homebrew v pevninské Číně, můžete nakonfigurovat domácí zrcadla pro zrychlení:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Správci balíčků specifických pro pevninskou Čínu

#### cnpm

cnpm je klient npm poskytovaný týmem Taobao, který standardně používá zrcadlo Taobao:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::tip
cnpm je většinou kompatibilní s příkazy npm a může být použit jako přímá náhrada.
:::

### Ověření efektu zrychlení zrcadla

Po nastavení zrcadel ověřte:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Ověření instalace

Po instalaci ověřte, že Node.js a npm jsou správně nainstalovány:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::pozn
Ujistěte se, že verze Node.js &gt;= 18.0 (doporučuje se Node.js 24+). Pokud je verze příliš nízká, přeinstalujte ji pomocí výše uvedených metod nebo použijte nástroje pro správu verzí k instalaci správné verze.
:::

### Ukázkový výstup ověření

Úspěšný výstup ověření by měl vypadat takto:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Odstraňování problémů

### Problémy se systémem Windows

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Pokud `node` nebo `npm` příkazy nejsou po instalaci dostupné:

1. Restartujte příkazový řádek nebo PowerShell
2. Pokud stále není k dispozici, ručně přidejte Node.js do PATH:
   - Vyhledejte "Proměnné prostředí" a otevřete "Upravit systémové proměnné prostředí"
   - Klikněte na "Proměnné prostředí"
   - Najděte "Cesta" v "Systémové proměnné" a klikněte na "Upravit"
   - Přidat instalační cestu Node.js (výchozí: `C:\Program Files\nodejs\`)
   - Klikněte na "OK" pro uložení změn

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Pokud při používání nvm narazíte na chyby zásad provádění:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### Problémy se systémem macOS

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Pokud při instalaci globálních balíčků npm narazíte na chyby oprávnění:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Pokud příkazy po instalaci nejsou k dispozici, restartujte terminál nebo ručně přidejte do PATH (obvykle v `/usr/local/bin`).

  </TabItem>
</Tabs>

### Problémy se systémem Linux

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Pokud při instalaci globálních balíčků npm narazíte na chyby oprávnění, doporučujeme místo správců systémových balíčků použít nástroje pro správu verzí Node.js (jako nvm nebo fnm).

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Pokud je verze Node.js nainstalovaná prostřednictvím správce systémových balíčků příliš stará:

1. Použít úložiště NodeSource (doporučeno, viz výše)
2. Nebo použijte nástroje pro správu verzí (nvm/fnm)

  </TabItem>
</Tabs>

### Kontrola prostředí

Pokud se ověření nezdaří, zkontrolujte následující:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
