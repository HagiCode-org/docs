# Nainstalujte OpenSpec

OpenSpec spravuje návrhy, specifikace a záznamy změn. Tato stránka se drží jednoho cíle: srozumitelně vysvětlete cestu instalace, abyste ji mohli připojit ke svému stávajícímu pracovnímu postupu.

## Co je OpenSpec?

OpenSpec je nástroj příkazového řádku pro správu návrhů a specifikací v práci se softwarem. Základní pracovní postup zahrnuje:

- **Správa nabídek**: Vytvářejte, kontrolujte a sledujte návrhy funkcí
- **Ověření specifikace**: Zajistěte, aby změny kódu odpovídaly předem definovaným specifikacím
- **Sledování změn**: Zaznamenává veškerou zkontrolovanou historii změn
- **Týmová spolupráce**: Podpora rozvoje spolupráce více lidí

### Aplikace v projektech HagiCode

V projektech HagiCode se OpenSpec používá pro:

1. **Správa vývoje funkcí**: Všechny nové funkce musí projít procesem návrhu OpenSpec
2. **Kontrola kódu**: Zajistěte, aby změny byly v souladu se specifikacemi projektu
3. **Synchronizace dokumentace**: Udržujte konzistenci mezi dokumentací a implementací kódu
4. **Integrace správy verzí**: Bezproblémová integrace s pracovními postupy správy verzí

## Předpoklady

Před instalací OpenSpec se ujistěte, že váš systém splňuje následující požadavky:

### Node.js a npm

OpenSpec je distribuován prostřednictvím npm a vyžaduje prostředí Node.js. OpenSpec vyžaduje **Node.js 18.0 nebo vyšší**.

Ověřte instalaci Node.js:

```bash
node --version
```

Ověřte instalaci npm:

```bash
npm --version
```

Pokud není nainstalován nebo je verze příliš stará, přečtěte si pokyny k instalaci souvisejícího softwaru v dokumentu [Průvodce instalací](/en/installation).

## Instalace OpenSpec

:::tip[Požadavky na verzi]
Tento projekt vyžaduje použití **OpenSpec 1.0.0 nebo vyšší** rozsah verzí. Všechny verze 1.x jsou podporovány, aby byla zajištěna kompatibilita s pracovním postupem projektu.
:::

OpenSpec se instaluje globálně prostřednictvím npm, takže je dostupný odkudkoli ve vašem systému.

### Instalovat globálně pomocí npm

<Tabs>
  <TabItem value="win" label="Windows">
    Spusťte v PowerShell nebo příkazovém řádku:

```powershell
npm install -g @fission-ai/openspec@1
```

:::note[Problémy s oprávněním systému Windows]
Pokud během instalace narazíte na chyby oprávnění, spusťte PowerShell nebo příkazový řádek jako správce.
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    Spustit v terminálu:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[Problémy s oprávněním macOS]
Pokud se setkáte `EACCES` chyby oprávnění, zkuste:
```bash
sudo npm install -g @fission-ai/openspec@1
```
Nebo použít [nvm](https://github.com/nvm-sh/nvm) spravovat verze Node.js a vyhnout se problémům s oprávněními.
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    Spustit v terminálu:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[Problémy s oprávněním k Linuxu]
Pokud se setkáte `EACCES` chyby oprávnění, můžete:
1. Použijte `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. Nakonfigurujte npm pro použití uživatelského adresáře:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### Ověřte instalaci

Po instalaci ověřte, zda je OpenSpec správně nainstalován a verze splňuje požadavky:

```bash
openspec --version
```

Pokud je instalace úspěšná, uvidíte výstup s číslem verze, například:

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::note[Rozsah verzí]
Ujistěte se, že nainstalovaná verze OpenSpec je 1.0.0 nebo vyšší. Projekt má ověřenou kompatibilitu s verzí 1.1.1. Pro získání nových funkcí se doporučuje používat nejnovější kompatibilní verzi.
:::

## Běžné problémy a odstraňování problémů

### Chyby oprávnění

#### Vydání: `EACCES` Povolení odepřeno

Setkání s chybami oprávnění při instalaci nebo spuštění OpenSpec.

**Řešení**:

<Tabs>
  <TabItem value="win" label="Windows">
    Spusťte PowerShell nebo příkazový řádek jako správce:
    1. Klepněte pravým tlačítkem myši na PowerShell nebo Příkazový řádek
    2. Vyberte "Spustit jako správce"
    3. Znovu spusťte instalační příkaz
  </TabItem>
  <TabItem value="mac" label="macOS">
    Nainstalujte pomocí `sudo`:

```bash
sudo npm install -g @fission-ai/openspec@1
```

Nebo použít [nvm](https://github.com/nvm-sh/nvm) spravovat Node.js a vyhnout se globálním problémům s oprávněním k instalaci.
  </TabItem>
  <TabItem value="linux" label="Linux">
    Řešení 1: Použijte sudo
```bash
sudo npm install -g @fission-ai/openspec@1
```

Řešení 2: Nakonfigurujte npm pro použití uživatelského adresáře
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @fission-ai/openspec@1
```
  </TabItem>
</Tabs>

### Problémy se sítí

#### Problém: Instalace npm se nezdaří nebo je velmi pomalá

Nelze stáhnout OpenSpec z registru npm ze síťových důvodů.

**Řešení**:

1. **Použít domácí npm Mirror**:

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **Obnovte oficiální zdroj** (v případě potřeby):

```bash
npm config set registry https://registry.npmjs.org
```

3. **Použít cnpm** (volitelné):

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

:::tip
Uživatelům v pevninské Číně se doporučuje používat zrcadlo Taobao, které může výrazně zvýšit rychlost stahování.
:::

### Problémy s kompatibilitou verzí

#### Problém: Příkazy OpenSpec běží abnormálně nebo hlásí chyby

Nainstalovaná verze OpenSpec není kompatibilní s aktuálním prostředím.

**Řešení**:

1. **Zkontrolujte verzi Node.js**:

```bash
node --version
```

Ujistěte se, že je verze 18.0 nebo vyšší. Pokud je verze příliš nízká, upgradujte Node.js.

2. **Zkontrolujte verzi OpenSpec**:

```bash
openspec --version
```

3. **Přeinstalujte konkrétní verzi**:

```bash
npm install -g @fission-ai/openspec@1
```

4. **Vymažte mezipaměť npm** (pokud problém přetrvává):

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### Příkaz nenalezen

#### Problém: Psaní `openspec` Příkaz zobrazuje „Příkaz nenalezen“

OpenSpec je nainstalován, ale systém nemůže najít příkaz.

**Řešení**:

<Tabs>
  <TabItem value="win" label="Windows">
    1. Potvrďte globální instalační cestu npm:

```powershell
npm config get prefix
```

2. Přidat cestu k systému PATH:
   - Otevřete „Vlastnosti systému“ → „Upřesnit“ → „Proměnné prostředí“
   - Najít `Path` v "Systémové proměnné"
   - Přidejte globální cestu npm (obvykle `C:\Users\<username>\AppData\Roaming\npm`)
   - Restartujte okno příkazového řádku
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. Potvrďte globální instalační cestu npm:

```bash
npm config get prefix
```

2. Přidejte cestu do PATH (in `~/.zshrc` nebo `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Znovu načíst konfiguraci:

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. Potvrďte globální instalační cestu npm:

```bash
npm config get prefix
```

2. Přidejte cestu do PATH (in `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Znovu načíst konfiguraci:

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
