Tato stránka je pro uživatele HagiCode poprvé, kteří chtějí nejkratší cestu k nastavení OmniRoute: spusťte místní službu, ověřte, že `http://localhost:20128/v1` funguje a potvrďte, že jsou k dispozici řídicí panel i rozhraní API.

:::tip[Doporučená cesta]
Tato první verze nejprve zachovává nejstabilnější cestu: `npm install -g omniroute`, `omniroute`, Dashboard at `http://localhost:20128`a základní URL kompatibilní s OpenAI na adrese `http://localhost:20128/v1`. Docker, VPS a pokročilejší režimy nasazení zůstávají ve volitelné části.
:::

## Předpoklady

- Node.js je již nainstalován. Dokončeno [Instalace Node.js](/en/related-software-installation/nodejs/installation) v případě potřeby nejprve.
- Místní přístav `20128` je k dispozici a ještě není obsazeno.
- Máte alespoň jeden účet poskytovatele upstream modelu, který plánujete směrovat přes OmniRoute.
- Před integrací HagiCode, CLI nebo SDK ponechte tyto dvě adresy oddělené:
  - řídicí panel: `http://localhost:20128`
  - Základní URL kompatibilní s OpenAI: `http://localhost:20128/v1`

## Nainstalujte a spusťte lokálně

Začněte doporučenou globální cestou npm:

```bash
npm install -g omniroute
omniroute
```

Po spuštění služby ji ověřte v tomto pořadí:

1. Otevřít `http://localhost:20128` v prohlížeči a potvrďte, že se řídicí panel načte.
2. Přejít na `Endpoints` stránku na řídicím panelu a vytvořte nebo zkopírujte klíč API.
3. Poznamenejte si základní adresu URL, kterou nástrojům předáte později: `http://localhost:20128/v1`.

:::note[Proč začít s řídicím panelem]
Při prvním nastavení je Dashboard nejrychlejším způsobem, jak potvrdit, že OmniRoute běží, vaši poskytovatelé jsou připojeni a existuje klíč API. Pokud selže pozdější klientské volání, kontrola řídicího panelu je obvykle rychlejší než okamžité ladění kódu SDK.
:::

## Ověřte běžící službu

### Ověření 1: potvrďte, že je katalog modelů čitelný

Vložte klíč od `Endpoints` stránku do proměnné prostředí:

```bash
export OMNIROUTE_API_KEY="replace-with-the-key-you-copied-from-the-Endpoints-page"
```

Poté požádejte `/v1/models`:

```bash
curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}"
```

Pokud získáte zpět seznam modelů, koncový bod kompatibilní s OpenAI a postup ověřování fungují.

### Ověření 2: pošlete minimální požadavek na chat

```bash
curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "replace-with-a-model-id-from-v1-models",
    "messages": [
      { "role": "user", "content": "Reply with ok." }
    ]
  }'
```

Cílem zde není okamžitá kvalita. Je to dokázat, že vámi vybraný model již může být úspěšně směrován přes OmniRoute.
