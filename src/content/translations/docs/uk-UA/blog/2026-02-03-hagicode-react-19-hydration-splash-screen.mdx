## Розробка 12 виняткових запусків для HagiCode: від мінімалізму до кіберпанку

> Короткий проміжок між завантаженням програми React 19 і завершенням Hydration — це прекрасна можливість для користувачів відчути індивідуальність вашого бренду. У цій статті ми ділимося повною системою стилю запуску, яку ми створили для проекту HagiCode за допомогою HTML/CSS/JS.

## Фон

Будучи сучасним додатком, створеним за допомогою ASP.NET Core 10 і React 19 (Vite), HagiCode використовує архітектуру розгортання, розділену на інтерфейс і сервер. Вихідні дані збірки зовнішнього інтерфейсу упаковані у серверну частину `wwwroot/` і розміщено в ASP.NET Core.

Однак ця архітектура представляє класичну проблему UX: коли користувачі відвідують сторінку, браузер спочатку завантажує HTML, потім завантажує великий пакет JS і, нарешті, дозволяє React виконати Hydration. Протягом цього «періоду вакууму», який триває від кількох сотень мілісекунд до кількох секунд, користувачі бачать або порожній екран, або мляву статичну сторінку.

Щоб заповнити цю прогалину та впровадити індивідуальність бренду HagiCode, нам потрібно було розробити систему в стилі запуску, реалізовану повністю з вбудованим кодом `index.html`.

## Про HagiCode

Підхід до дизайну екрану-заставки, про який ми розповідаємо в цій статті, базується на нашому практичному досвіді в [HagiCode](https://github.com/HagiCode-org/site) проект. Як помічник кодування ШІ, HagiCode дбає не лише про ефективність генерації коду, але й про візуальний досвід розробника. Ця система запуску є одним із результатів нашого прагнення до максимальної продуктивності інтерфейсу.

## Основні виклики та архітектурний дизайн

Перш ніж ми почали проектувати, нам спочатку потрібно було уточнити технічні обмеження. Оскільки все мало бути реалізовано вбудовано `index.html`, ми не змогли завантажити будь-які зовнішні файли CSS або JS, крім власного пакету React.

### Аналіз технічних обмежень

1.  **Принцип нульової залежності**: усі стилі мають існувати всередині a `<style>` і вся логіка має бути всередині a `<script>` тег.
2.  **Захисний CSS**: щоб глобальні стилі не забруднювали екран-заставку після монтування програми React, ми вирішили обернути всі стилі запуску префіксом ID з високим пріоритетом, наприклад `#boot-screen`.
3.  **Продуктивність насамперед**: анімація має використовувати CSS `transform` і `opacity` усюди, де це можливо, щоб уникнути оплавлення та переконатися, що основний потік залишається незаблокованим.
4.  **Візуальна узгодженість**: кольори та шрифти мають відповідати конфігурації Tailwind HagiCode.

### Шаблон архітектури: оболонка та інжектор

Ми прийняли **варіантний шаблон**. Основна логіка інкапсульована всередині виразу функції, що негайно викликається (IIFE), тоді як конкретна логіка візуалізації вводиться через конфігурацію. Це дозволяє нам перемикатися між різними стилями за допомогою простої конфігурації замість повторного переписування логіки маніпулювання DOM.

Ось код основної архітектури:

```html
<!-- Inline in index.html -->
<div id="boot-root"></div>

<script>
(function() {
  const BootSequence = {
    config: {
      theme: 'terminal', // Configurable as 'minimal', 'skeleton', 'code-rain', etc.
      color: '#3b82f6'   // Brand color
    },
    
    // Core lifecycle
    init() {
      this.render();
      this.listenForMount();
    },

    // Render the currently selected style
    render() {
      const root = document.getElementById('boot-root');
      if (this.variants[this.config.theme]) {
        root.innerHTML = this.variants[this.config.theme].render();
      }
    },

    // Listen for successful React mount and exit gracefully
    listenForMount() {
      window.addEventListener('hagicode:ready', () => {
        const screen = document.getElementById('boot-root');
        // Fade out first, then remove the DOM to avoid flicker
        screen.style.opacity = '0';
        screen.style.transition = 'opacity 0.3s ease';
        setTimeout(() => screen.remove(), 300);
      });
    },

    // The implementation logic for all 12 styles lives here
    variants: {
      // ...see details below
    }
  };

  BootSequence.init();
})();
</script>
```

## Контрольний список із 12 дизайнів у стилі стартапу

Ми згрупували ці 12 стилів у шість основних категорій, щоб задовольнити різні сценарії та естетичні уподобання.

### А. Мінімалізм

> «Менше — це краще». Для сценаріїв, які прагнуть максимальної швидкості завантаження, ми надаємо найпростіші варіанти.

#### 1. Мінімалістична точка
Проста крапка розташована в центрі екрана в поєднанні з анімацією дихання.
*   **Реалізація**: CSS `@keyframes` контролює масштаб і непрозорість.
*   **Найкраще для**: будь-який випадок, коли сторінка має залишатися абсолютно чистою.

#### 2. Розкриття бренду
Використання SVG `stroke-dasharray` анімація, вона імітує намальоване від руки ліній логотипу HagiCode, після чого з’являється текст.
*   **Техніка**: анімація шляху у форматі SVG із відшліфованим відчуттям.

### B. Мімікрія скелетного екрана

> «Мистецтво обманювати око». Імітуючи макет реального інтерфейсу користувача, користувачі відчувають, що сторінка вже наполовину завантажена.

#### 3. Скелет чату бічної панелі
Це може бути найбільш практичним варіантом. Ми вручну створили макет у HTML, який відображає React `Sidebar` і `ChatInput` компонентів точно, а потім наклав на нього сіру мерехтливу анімацію.
*   **Цінність**: коли гідратація React завершується, скелет миттєво стає справжнім компонентом, і користувачі ледве помічають перемикання.

#### 4. Скелет стопки карток
Він імітує рух складених карток пропозицій під час завантаження, використовуючи 3D-перетворення, щоб зробити картки непомітно плаваючими.

### C. Абстракція та мистецтво

> Продемонструйте геніальну ДНК HagiCode.

#### 5. Геометрична морфа
Геометрична фігура (квадрат) відображається в центрі екрана, потім плавно трансформується з часом у коло, трикутник і, нарешті, логотип.
*   **Технологія**: плавні переходи за допомогою CSS `border-radius`.

#### 6. Код Дощ
Вшанування *Матриці*. Використовуючи шрифт JetBrains Mono, слабкі потоки символів падають на фоні.
*   **Примітка**: для продуктивності потоки символів повинні залишатися в межах меншої області або використовувати нижчу частоту оновлення.

#### 7. Неоновий пульс
Кільце, що світиться в стилі кіберпанк, у якому використовується кілька `box-shadow` шари для створення потужного неонового світіння.

### D. Брендінг і теми

> Зробіть так, щоб система відчувала себе живою.

#### 8. Сезонна тема
Це динамічний завантажувач. Він перевіряє поточну дату таких свят, як Новий рік або Різдво за місячним календарем, і завантажує відповідну анімацію SVG.
*   **Приклад**: під час Нового року за місячним календарем червоні ліхтарики м’яко погойдуються внизу екрана.

#### 9. Градієнтний потік
Для фону використовується плавний градієнт на основі кольорів бренду HagiCode. У поєднанні з анімованим `background-size` і `background-position`, це створює відчуття руху, схоже на полярне сяйво.

### E. Технічне відчуття

> Привіт розробникам.

#### 10. Завантаження терміналу
Він імітує вихід на консоль. Рядки коду швидко прокручуються:
```text
> Initializing HagiCode Core...
> Loading models...
> Connecting to neural network...
```
Це миттєво стає знайомим кожному розробнику.

#### 11. Індикатор прогресу
У верхній частині екрана з’являється тонка панель перебігу, праворуч – відсоток. Хоча ми не можемо отримати доступ до реального прогресу завантаження, ми можемо використати таймер, щоб імітувати «правдоподібний» процес завантаження: швидке протягом перших 80%, потім поступово повільніше протягом останніх 20%.

### Ф. Творчість

#### 12. Піксельна збірка
Це дуже цікава ідея. Маленькі квадратики розкидані по екрану, потім сходяться до центру і поступово збираються в значок логотипу HagiCode. Він символізує процес створення коду.

## Найкращі практики та підводні камені

У реальній роботі над розробкою HagiCode ми узагальнили кілька критично важливих деталей впровадження.

### 1. Захисний CSS є обов’язковим
Ніколи не лінуйтеся і пропускайте префікс. Одного разу ми забули визначити область дії стилів заставки за допомогою ідентифікатора та глобального `div` стилі після монтування React несподівано вплинули на екран-заставку, порушивши макет.
**Здобутий урок**: розмістіть кожен селектор CSS під `#boot-screen`, і використовуйте `!important` щоб підвищити пріоритет, коли це необхідно, але лише всередині CSS екрану-заставки.

### 2. Витончені переходи
Після успішного монтування React не монтуйте безпосередньо `remove()` заставка DOM.
**Правильний підхід**:
1.  Тригери React `window.dispatchEvent(new Event('hagicode:ready'))`.
2.  Екран-заставка прослуховує подію та перші набори `opacity: 0`.
3.  Зачекайте 300 мс, що відповідає тривалості переходу CSS, і зателефонуйте `.remove()` лише після того, як екран стане повністю невидимим.

### 3. Синхронізація змінних тем
Значення кольорів початкового екрана жорстко закодовано `index.html`. Якщо ми змінимо основний колір Tailwind, ми також повинні оновити екран-заставку.
**Оптимізація**: напишіть простий плагін у сценарії збірки Vite для читання `tailwind.config.js` і ввести змінні кольору в `index.html` шаблонні змінні, створюючи єдине джерело правди.

### 4. Попереднє завантаження шрифту
На заставках часто потрібно використовувати фірмовий шрифт, але якщо шрифт завантажується повільно, може з’явитися FOUT (спалах нестилізованого тексту).
**Рішення**: Додати `<link rel="preload" href="/fonts/JetBrainsMono.woff2" as="font" type="font/woff2" crossorigin>` всередині `<head>`. Це недорогий і високорентабельний спосіб покращити досвід.

### 5. Моніторинг продуктивності
Ми зробили ін'єкцію `performance.mark('boot-start')` на дні `index.html`, і позначено `boot-end` коли React успішно встановлено.
**Чому це важливо**: збираючи ці дані за допомогою Application Insights, ми можемо безпосередньо виміряти, наскільки екран-заставка скорочує очікуваний час очікування. Дані показують, що чудовий скелетний екран може покращити толерантність користувачів до «повільної мережі» більш ніж на 50%.
