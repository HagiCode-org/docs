## Чому HagiCode обрала Hermes як інтегроване ядро агента

> Під час створення платформи кодування за допомогою штучного інтелекту вибір правильного ядра агента безпосередньо визначає верхню межу можливостей системи. Деякі речі просто неможливо змусити; виберіть неправильну структуру, і жодні зусилля не змусять її відчути себе правильною. Ця стаття ділиться ідеєю технічного вибору HagiCode і нашим практичним досвідом інтеграції Hermes Agent.

## Фон

Під час створення продукту кодування за допомогою штучного інтелекту однією з найважчих частин є вибір основної структури агента. На ринку насправді є досить багато варіантів, але деякі з них надто обмежені у функціональності, деякі надто складні для розгортання, а інші просто недостатньо добре масштабуються. Нам було потрібно рішення, яке могло б працювати на VPS за 5 дол. США, а також мати можливість підключатися до кластера GPU. Ця вимога може здатися не екстремальною, але її достатньо, щоб відлякати багато команд.

На практиці багато так званих «універсальних агентів» або працюють лише в хмарі, або вимагають абсурдно високих витрат на локальне розгортання. Витративши два тижні на дослідження різних підходів, ми прийняли сміливе рішення: перебудувати все ядро ​​Agent навколо Hermes як основного механізму для нашого інтегрованого Agent.

Усе, що було далі, могло бути просто долею.

## Про HagiCode

Підхід, поширений у цій статті, базується на реальному досвіді в [HagiCode](https://hagicode.com) проект. HagiCode — це платформа кодування з підтримкою штучного інтелекту, яка надає розробникам інтелектуального помічника з кодування через розширення VSCode, настільний клієнт і веб-сервіси. Можливо, ви раніше користувалися подібними інструментами і відчували, що їм бракує останнього штриха; ми добре розуміємо це почуття.

- GitHub: [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site)
- Офіційний сайт: [hagicode.com](https://hagicode.com)

## Чому HagiCode потрібен Hermes

Перш ніж зануритися в сам Hermes, це допоможе пояснити, чому HagiCode взагалі знадобилося щось подібне. Речі рідко працюють саме так, як вам хочеться, тому вам потрібна практична причина, щоб вибрати технічний напрямок.

Як помічник кодування ШІ, HagiCode має підтримувати кілька сценаріїв використання одночасно:
- Локальні середовища розробки: розробники хочуть запускати його на власних машинах, щоб дані ніколи не залишали локальне середовище. У наші дні безпека даних ніколи не є тривіальною проблемою.
- Середовище командної співпраці: невеликі команди повинні мати можливість спільно використовувати розгортання агента, що працює на сервері. Економія грошей має значення, і в кожного є межі.
- Розширення еластичної хмари: при обробці складних завдань система повинна автоматично масштабуватися до кластера GPU. Завжди краще бути напоготові.

Ця вимога «ми хочемо все й одразу» привела нас до Hermes. Чи був це ідеальний вибір, я не можу сказати напевно, але на той момент ми не бачили кращого варіанту.

## Що таке Hermes Agent

Hermes Agent — це автономний агент ШІ, створений **Nous Research**. Деякі читачі можуть бути не знайомі з Nous Research; вони є лабораторією, що стоїть за великими моделями з відкритим кодом, такими як Hermes, Nomos і Psyché. Вони створили багато чудових речей, навіть якщо їх все ще недооцінюють більше, ніж вони заслуговують.

На відміну від традиційних помічників кодування IDE або простих програм-обгорток для чатів API, Hermes має визначальну рису: що довше він працює, то продуктивнішим стає. Він не призначений для того, щоб виконати завдання один раз і зупинитися; він продовжує навчатися та накопичувати досвід протягом тривалої роботи. У цьому сенсі це трохи схоже на людину.

### Основні характеристики

Кілька основних можливостей Hermes дуже тісно відповідають потребам HagiCode.

Це означає, що HagiCode може вибрати найбільш прийнятну модель розгортання на основі сценарію кожного користувача: окремі особи запускають її локально, команди розгортають її на серверах, а складні завдання використовують ресурси GPU. Одна кодова база обробляє все це. У такому насиченому світі збереження одного рівня складності – це вже перемога.

**Мультиплатформний шлюз обміну повідомленнями**
Hermes підтримує Telegram, Discord, Slack, WhatsApp тощо. Для HagiCode це означає, що в майбутньому ми зможемо набагато легше підтримувати помічників ШІ на цих каналах. Більше шляхів вперед завжди вітаються.

**Багата система інструментів**
Hermes поставляється з понад 40 вбудованими інструментами та підтримує розширення MCP (Model Context Protocol). Це важливо для помічника кодування: виконання команд оболонки, робота з файловою системою та виклик Git – усе залежить від підтримки інструменту. Агент без інструментів, як птах без крил.

**Кроссесійна пам'ять**
Hermes містить постійну систему пам’яті та використовує повнотекстовий пошук FTS5, щоб згадати історичні розмови. Це дозволяє агенту запам’ятовувати попередній контекст замість того, щоб щоразу «втрачати пам’ять». Іноді люди бажають так легко забути речі, але реальність зазвичай менш щедра.

## Як HagiCode інтегрує Hermes

Тепер, коли «чому» зрозуміло, давайте розглянемо «як». Коли щось має сенс у теорії, наступним кроком є ​​його створення.

### Абстракція рівня провайдера

В архітектурі HagiCode усі постачальники штучного інтелекту реалізують уніфікований код `IAIProvider` інтерфейс:

```csharp
public sealed class HermesCliProvider : IAIProvider, IVersionedAIProvider
{
    public ProviderCapabilities Capabilities { get; } = new ProviderCapabilities
    {
        SupportsStreaming = true,   // Supports streaming output
        SupportsTools = true,       // Supports tool invocation
        SupportsSystemMessages = true, // Supports system prompts
        SupportsArtifacts = false
    };
}
```

Цей рівень абстракції дозволяє HagiCode плавно перемикатися між різними постачальниками ШІ. Незалежно від того, чи є бекенд OpenAI, Claude або Hermes, шаблон виклику верхнього рівня залишається незмінним. Простіше кажучи, це робить речі простими.

### Протокол зв'язку ACP

Hermes спілкується через **ACP (Agent Communication Protocol)**. Цей протокол розроблено спеціально для зв’язку з агентами, і його основні методи включають:

| метод | опис |
|------|------|
| `initialize` | Ініціалізуйте підключення та отримайте версію протоколу та можливості клієнта |
| `authenticate` | Керуйте автентифікацією та підтримуйте кілька методів автентифікації |
| `session/new` | Створіть новий сеанс і налаштуйте робочий каталог і сервери MCP |
| `session/prompt` | Надішліть підказку та отримайте відповідь |

HagiCode реалізує транспортний рівень ACP `StdioAcpTransport`, запускаючи підпроцес Hermes і спілкуючись із ним через стандартний вхід і вихід. Це може здатися складним, але на практиці це можна зробити, якщо у вас достатньо терпіння.

### Управління конфігурацією

Конфігурація управляється через `HermesPlatformConfiguration` клас:

```csharp
public sealed class HermesPlatformConfiguration : IAcpPlatformConfiguration
{
    public string ExecutablePath { get; set; } = "hermes";
    public string Arguments { get; set; } = "acp";
    public int StartupTimeoutMs { get; set; } = 5000;
    public string ClientName { get; set; } = "HagiCode";
    public HermesAuthenticationConfiguration Authentication { get; set; }
    public HermesSessionDefaultsConfiguration SessionDefaults { get; set; }
}
```

Налаштуйте Hermes в `appsettings.json`:

```json
{
  "Providers": {
    "HermesCli": {
      "ExecutablePath": "hermes",
      "Arguments": "acp",
      "StartupTimeoutMs": 10000,
      "ClientName": "HagiCode",
      "Authentication": {
        "PreferredMethodId": "api-key",
        "MethodInfo": {
          "api-key": "your-api-key-here"
        }
      },
      "SessionDefaults": {
        "Model": "claude-sonnet-4-20250514",
        "ModeId": "default"
      }
    }
  }
}
```

Конфігурація часто виглядає простою на папері, але для правильного визначення кожної деталі потрібні серйозні зусилля.

### Орлеанська розподілена архітектура

HagiCode використовує Orleans для створення своєї розподіленої системи, а інтеграція Hermes реалізована за допомогою таких компонентів:

- **HermesGrain**: реалізація Orleans Grain, яка обробляє виконання сеансу
- **HermesPlatformConfiguration**: конфігурація для конкретної платформи
- **HermesAcpSessionAdapter**: адаптер сеансу ACP
- **HermesConsole**: спеціальна консоль перевірки

У назві Орлеан є певний шарм. Навіть якщо цей Орлеан не має нічого спільного з легендарним містом, добре ім’я ніколи не завадить.

### Наскрізний потік виконання

Нижче наведена основна логіка виконання Hermes Provider:

```csharp
private async IAsyncEnumerable<AIStreamingChunk> StreamCoreAsync(
    AIRequest request,
    string? embeddedCommandPrompt,
    [EnumeratorCancellation] CancellationToken cancellationToken)
{
    // 1. Create transport layer and launch Hermes subprocess
    await using var transport = new StdioAcpTransport(
        platformConfiguration.GetExecutablePath(),
        platformConfiguration.GetArguments(),
        platformConfiguration.GetEnvironmentVariables(),
        platformConfiguration.GetStartupTimeout(),
        _loggerFactory.CreateLogger<StdioAcpTransport>());
    await transport.ConnectAsync(cancellationToken);

    // 2. Initialize and obtain protocol version and authentication methods
    var initializeResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "initialize",
        BuildInitializeParameters(platformConfiguration), cancellationToken);

    // 3. Handle authentication
    var authMethods = ParseAuthMethods(initializeResult);
    if (!isAuthenticated)
    {
        var methodId = platformConfiguration.Authentication.ResolveMethodId(authMethods);
        await SendHermesRequestAsync(transport, nextRequestId++, "authenticate", ...);
    }

    // 4. Create session
    var newSessionResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "session/new",
        BuildNewSessionParameters(platformConfiguration, workingDirectory, model), cancellationToken);
    var sessionId = ParseSessionId(newSessionResult);

    // 5. Execute prompt and collect streaming responses
    await foreach (var payload in transport.ReceiveMessagesAsync(cancellationToken))
    {
        // Handle session/update notifications and convert them into streaming chunks
        if (TryParseSessionNotification(root, out var notification))
        {
            if (_responseMapper.TryConvertToStreamingChunk(notification, out var chunk))
            {
                yield return chunk;
            }
        }
    }
}
```

За допомогою коду деталі з часом стають знайомими. Найбільше значення має загальний підхід.

### Перевірки стану здоров'я

Щоб забезпечити доступність Hermes, HagiCode реалізує механізм перевірки працездатності:

```csharp
public async Task<ProviderTestResult> PingAsync(CancellationToken cancellationToken = default)
{
    var response = await ExecuteAsync(
        new AIRequest
        {
            Prompt = "Reply with exactly PONG.",
            CessionId = null,
            AllowedTools = Array.Empty<string>(),
            WorkingDirectory = ResolveWorkingDirectory(null)
        },
        cancellationToken);

    var success = string.Equals(response.Content.Trim(), "PONG", StringComparison.OrdinalIgnoreCase);
    return new ProviderTestResult
    {
        ProviderName = Name,
        Success = success,
        ResponseTimeMs = stopwatch.ElapsedMilliseconds,
        ErrorMessage = success ? null : $"Unexpected Hermes ping response: '{response.Content}'."
    };
}
```

Приблизно так тут виглядає «перевірка стану здоров’я». У певному сенсі люди не такі вже й різні: це допомагає час від часу перевірятися, навіть якщо ніхто не говорить нам, на що саме звернути увагу.

## Практичні міркування

Є кілька підводних каменів, які варто зрозуміти, перш ніж інтегрувати Hermes. Кожен рано чи пізно потрапляє в кілька пасток.

### Конфігурація методу автентифікації

Hermes підтримує кілька методів автентифікації, включаючи ключі API та токени, тому вам потрібно вибирати на основі фактичного сценарію розгортання. Неправильна конфігурація може спричинити збій підключення, а повідомлення про помилки не завжди інтуїтивно зрозумілі. Іноді повідомлення про помилку є далеким від справжньої першопричини, що означає, що повільного та ретельного налагодження не уникнути.

### Конфігурація сервера MCP

Під час створення сеансу ви можете налаштувати список серверів MCP, щоб Hermes міг викликати зовнішні інструменти. Але майте на увазі наступне:
- Адреси серверів MCP мають бути доступними
- Тайм-аути повинні бути налаштовані розумно
- Система потребує обробки погіршення, коли сервер недоступний

На практиці оборонне мислення має більше значення, ніж люди очікують.

### Керування робочим каталогом

Кожен сеанс має вказувати робочий каталог, щоб Hermes міг правильно отримати доступ до файлів проекту. У сценаріях із кількома проектами робочий каталог має динамічно перемикатися. Це звучить просто, але крайніх випадків більше, ніж ви думаєте.

### Агрегація відповідей

Відповіді Hermes можуть бути розділеними `session/update` сповіщення та кінцевий результат, тому їх потрібно правильно об’єднати. Інакше вміст може бути втрачено.

### Стратегія обробки помилок

Помилки виконання слід повертати явно замість мовчки повертатися до іншого постачальника. Таким чином, користувачі знають, що проблема виникла через Hermes, а не цікавляться, чому система раптом змінила моделі за кадром.

## Висновок

Рішення HagiCode використовувати Hermes як інтегроване ядро агента не було випадковим імпульсом. Це був ретельний вибір, виходячи з практичних вимог і технічних характеристик каркасу. Чи виявиться це ідеальною довгостроковою відповіддю, ще занадто рано говорити, але поки що він служить нам добре.

Hermes надає HagiCode гнучкість для адаптації до широкого діапазону сценаріїв. Його потужна система інструментів і підтримка MCP дозволяють помічнику штучного інтелекту виконувати реальну роботу, а протокол ACP і рівень абстракції постачальника забезпечують чіткість і контрольованість процесу інтеграції.

Якщо ви обираєте фреймворк агента для власного проекту штучного інтелекту, сподіваюся, ця стаття стане корисною довідкою. Вибір правильної базової архітектури може значно спростити все подальше.
