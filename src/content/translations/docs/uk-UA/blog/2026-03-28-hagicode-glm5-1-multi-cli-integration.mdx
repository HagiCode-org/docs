## Посібник з інтеграції Hagicode та GLM-5.1 Multi-CLI

:::note[Повідомлення про контекст часу]
Ця стаття документує дизайн інтеграції з кількома командами командного рядка та відображення моделі, як вони існували на момент публікації. Для поточної постійно підтримуваної області див [Інсталяція агента AI Agent CLI](/uk-UA/related-software-installation/ai-agent-cli/) і [Керівництво по установці](/uk-UA/installation); в `IFlowCli` згадана тут лише історична конфігурація на той час і не є частиною поточної активної підтримки.
:::

## Фон

У проекті Hagicode користувачі можуть вибирати з кількох інструментів CLI для керування помічниками програмування штучного інтелекту, включаючи CLAude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI тощо. Ці інструменти CLI самі по собі є інструментами програмування ШІ загального призначення, але через рівень абстракції Hagicode вони можуть гнучко підключатися до різних постачальників моделей ШІ.

Zhipu AI (ZAI) надає інтерфейс, сумісний з Anthropic Claude API, що дозволяє цим інструментам CLI безпосередньо використовувати вітчизняні моделі серії GLM. Серед них GLM-5.1 є останнім випуском великої мовної моделі Zhipu зі значними покращеннями порівняно з GLM-5.0.

## Архітектура абстракції CLI Hagicode

Hagicode визначає 11 типів провайдерів CLI через `AIProviderType` enum, що охоплює основні інструменти CLI програмування ШІ:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Кожен CLI має відповідну конфігурацію параметрів моделі та підтримує `model` і `reasoning` параметри:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],
        [AIProviderType.CodexCli] = ["model", "reasoning"],
        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],
        [AIProviderType.HermesCli] = ["model", "reasoning"],
        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini does not support the reasoning parameter
        // ...
    };
```

## Використання GLM-5.1 у різних CLI

### 1. Claude Code CLI + GLM-5.1

Інтерфейс командного рядка Claude Code є одним із основних інтерфейсів командного рядка Hagicode і налаштовується через систему конфігурації Hero:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Відповідний `HeroEquipmentCatalogItem` конфігурація:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI є найбільш гнучким CLI і підтримує вказівку будь-якої моделі в `provider/model` формат:

**Спосіб 1. Використовуйте префікс провайдера ZAI**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Спосіб 2: безпосередньо використовуйте ідентифікатор моделі**
```json
{
  "model": "glm-5.1"
}
```

**Спосіб 3: Інтерфейс конфігурації зовнішнього інтерфейсу**

в `HeroModelEquipmentForm.tsx`, OpenCode CLI має спеціальну підказку-заповнювач:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Користувачі можуть ввести:
```
zai/glm-5.1
glm-5.1
```

**Логіка розбору моделі OpenCode CLI**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI має спеціальну застарілу логіку обробки:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Примітка**: Codebuddy зберігає спеціальну обробку для GLM-5.0 і не використовує успадковану нормалізацію:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## Переваги вдосконалення GLM-5.1

Порівняно з GLM-5.0, GLM-5.1 забезпечує такі значні покращення:

### 1. Краща здатність міркувати

Згідно з офіційною інформацією Zhipu, покращення GLM-5.1 включають:

- **Краще розуміння коду**: точніший аналіз складних структур коду
- **Довше розуміння контексту**: підтримує довший розмовний контекст
- **Покращений виклик інструментів**: вищий рівень успіху для викликів інструментів MCP
- **Стабільність виводу**: Зменшує випадковість і галюцинації

### 2. Повна сумісність з кількома командами командного рядка

GLM-5.1 охоплює всі основні CLI, які підтримуються Hagicode:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Примітки

### 1. Конфігурація ключа API

Переконайтеся, що `ANTHROPIC_AUTH_TOKEN` змінну середовища встановлено правильно. Це необхідні облікові дані для кожного CLI для підключення до моделі.

### 2. Доступність моделі

GLM-5.1 має бути включений відповідним постачальником моделі:
- Платформа Zhipu AI ZAI підтримує його за замовчуванням
- Alibaba Cloud DashScope може потребувати окремої програми

### 3. Формат OpenCode CLI

При використанні `provider/model` переконайтеся, що ідентифікатор постачальника правильний:
- Жипу А.І.: `zai` або `zhipuai`
- Хмара Alibaba: `aliyun` або `dashscope`

### 4. Параметр міркування

- `high` рекомендовано для найкращих результатів генерації коду
- Gemini CLI не підтримує параметр аргументації та автоматично ігноруватиме цю конфігурацію

## Висновок

Завдяки уніфікованому рівню абстракції Hagicode забезпечує гнучку інтеграцію між GLM-5.1 і кількома CLI. Розробники можуть вибрати інструмент CLI, який найкраще відповідає їхнім уподобанням і сценаріям використання, а потім використовувати останню модель GLM-5.1 за допомогою простого налаштування.

Як остання версія моделі Zhipu, GLM-5.1 пропонує явні покращення порівняно з GLM-5.0:
- Незалежний ідентифікатор версії без успадкованого тягаря
- Покращене міркування та розуміння коду
- Широка сумісність з кількома командами командного рядка
- Гнучка конфігурація рівня аргументації

За допомогою правильних змінних середовища та налаштованого обладнання Hero користувачі можуть повністю розблокувати потужність GLM-5.1 у різних середовищах CLI.
