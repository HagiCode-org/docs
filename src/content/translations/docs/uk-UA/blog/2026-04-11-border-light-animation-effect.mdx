## Посібник зі створення анімаційного ефекту розгортання світла на межі

> Як створити цей важливий елемент, який користувачі помічають з першого погляду, використовуючи чистий CSS? Насправді це не так важко. Хитрість полягає в тому, що ви берете трохи обхідний шлях. У цій статті я розповім вам про те, як створити анімацію межі світла з нуля, а також поділюся кількома підводними каменями, з якими ми зіткнулися під час створення HagiCode.

## Фон

Якщо ви працюєте над зовнішнім інтерфейсом, у вас, напевно, був такий досвід раніше: менеджер із продуктів підходить із виразом «це точно просто» і каже: «Чи можемо ми додати якийсь спеціальний ефект до цього запущеного завдання, щоб користувачі могли це негайно помітити?»

Ви скажете, що ми можемо просто змінити колір рамки. Потім вони хитають головою з таким поглядом, який говорить: «Ти цього не розумієш». Вони відповідають: «Це недостатньо очевидно. Мені потрібен такий ефект, коли світло біжить навколо кордону, як у науково-фантастичному фільмі».

У цей момент ви можете почати задаватися питанням, як його побудувати. полотно? SVG? Або CSS може впоратися з цим самостійно? Зрештою, ніхто не хоче визнавати, що не вміє.

У сучасних веб-додатках анімація світлового згортання рамок насправді дуже поширена. Вони в основному використовуються в таких сценаріях:

- **Індикатори стану**: позначення завдань, що виконуються, або активних елементів
- **Візуальний фокус**: виділення важливих областей вмісту
- **Покращення бренду**: створення елегантного, сучасного, технологічного візуального стилю
- **Сезонні теми**: створення святкової атмосфери для особливих випадків

Ми спробували кілька різних підходів. Деякі стежки були плавнішими, деякі – трохи звивистішими. Зрештою, ми зупинилися на досить зрілій стратегії впровадження.

## Основні ідеї реалізації

Проаналізувавши кодову базу HagiCode, ми узагальнили кілька основних шаблонів реалізації нижче. Кожен з них відповідає різному сценарію, іншими словами, кожен існує з певної причини.

### 1. Обертове світіння з конічним градієнтом (найчастіше)

Це класичний спосіб реалізації ефекту розгортання світла межі. Основна ідея полягає у використанні CSS `conic-gradient` щоб створити конічний градієнт, потім безперервно обертайте його. Як вуличний ліхтар, що обертається вночі, він просто продовжує кружляти.

**Ключові елементи:**
- Використовуйте `::before` псевдоелемент для створення шару світіння
- використання `conic-gradient` щоб визначити градієнтний розподіл кольорів
- Використовуйте `::after` псевдоелемент для маскування центральної області (опціонально)
- використання `@keyframes` реалізувати анімацію обертання

### 2. Бічна лінія світіння

Це добре працює для індикаторів стану в елементах списку. Ви створюєте тонку сяючу лінію з одного боку елемента замість того, щоб анімувати всю рамку. Іноді достатньо трохи світла. Вам не потрібно освітлювати весь світ.

**Ключові елементи:**
- Тонкий абсолютно позиціонований лінійний елемент
- використання `box-shadow` для створення ефекту світіння
- використання `scale` і `opacity` для дихальної анімації

### 3. Сяючий фон із коробчатою тінню

Якщо вам не потрібен повний ефект розгортки, а просто потрібно м’яке сяйво фону, нанесіть кілька шарів `box-shadow` значень достатньо. Іноді простіший варіант є кращим.

### 4. Підтримка доступності

Цю частину легко не помітити, але вона надзвичайно важлива. Кожна анімація повинна враховувати `prefers-reduced-motion` медіа-запит і надати статичну альтернативу для користувачів, яким не потрібна анімація. Не всім подобається постійний рух, і поважати це бажання має значення.

## Варіанти реалізації

### Варіант 1: поворотна конічна градієнтна межа (рекомендовано)

Це найповніша реалізація світлового ефекту розгорнутої межі, і це також параметр, який ми найчастіше використовуємо в HagiCode. Зрештою, якщо щось працює добре, навіщо це замінювати?

```css
/* Parent container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotating glow layer */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Mask layer (optional, for creating a hollow border effect) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Принцип цієї опції досить простий: створіть псевдоелемент, більший за батьківський контейнер, намалюйте на ньому конічний градієнт і безперервно обертайте його. Батьківський контейнер використовує `overflow: hidden`, тому видимим залишається лише світло, що проходить навколо кордону. Це трохи схоже на те, щоб дивитися на вуличний ліхтар у вікно. Ви завжди бачите лише маленький шматочок, який проходить повз.

### Варіант 2: спрощена світлова рамка, що обертається

Якщо вам не потрібен повний ефект, HagiCode також містить полегшену версію утилітного класу. Іноді простіший підхід дійсно кращий.

```css
/* Rotating light border utility class */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Accessibility support */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Зверніть увагу на `will-change: transform` тут. Він повідомляє веб-переглядачу: «Цей елемент продовжуватиме змінюватися», тож веб-переглядач може заздалегідь підготувати деякі оптимізації та зробити анімацію плавнішою. Готуватися заздалегідь, як правило, краще, ніж намагатися в останню хвилину.

### Варіант 3: Бічна лінія світіння

Це особливо підходить для індикаторів статусу елементів списку, і це саме те, що використовує список сеансів HagiCode. Одна тонка лінія все ж може виділятися серед багатьох предметів. Це по-своєму виглядає як життєвий урок.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Це використовує `isolation: isolate` щоб створити новий контекст стекування, потім покладається на `z-index` щоб контролювати порядок відображення кожного шару. `pointer-events: none` також є істотним. Інакше псевдоелемент блокував би клацання користувача. Деякі речі можуть виглядати красиво, але вони все одно не повинні заважати.

### Варіант 4: оберніть його в компонент React

Якщо ваш проект використовує React, ви можете включити цю логіку в компонент, особливо обробку доступності. Напишіть один раз, використовуйте багато разів. В цьому й вся суть.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Відповідний модуль CSS:

```css
/* GlowBorder.module.css */

/* Animated version */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Static version (accessibility) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

The `useReducedMotion` гачок від `framer-motion` автоматично визначає системні налаштування користувача. Якщо користувач увімкнув зменшений рух, воно повертається `true`, і замість цього компонент показує статичну версію. Повага до вподобань користувача важливіша, ніж нав’язування яскравого ефекту.

## Отримані практичні уроки

Ось деякі з уроків, які ми засвоїли під час створення HagiCode. Ви також можете назвати їх бойовими шрамами. Сподіваюся, вони допоможуть вам уникнути деяких обхідних шляхів.

### 1. Тематична змінна система

Змінні CSS роблять підтримку кількох тем особливо зручною. Ніхто не хоче редагувати купу коду щоразу, коли змінюється тема.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Usage */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

Таким чином, перемикання тем вимагає лише зміни класу на `html` і кожен колір анімації оновлюється автоматично. Одна кодова база, два стилі. Це саме те, чого ми хочемо.

### 2. Оптимізація продуктивності

**Використання `will-change` підказати браузеру оптимізувати:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Скажіть браузеру заздалегідь, і він допоможе вам оптимізувати. Багато речей у житті працюють краще, якщо трохи підготуватися.

**Уникайте використання складних прямокутних тіней на великих елементах:**

```css
/* Not ideal - using a blurred shadow on a large element */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Better - use a pseudo-element to limit the glowing area */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Ми перевірили це в HagiCode. Додавання розмитої тіні безпосередньо на велику картку знизило частоту кадрів прокручування нижче 30 кадрів/с. Перехід на псевдоелемент повернув речі до стабільних 60 кадрів в секунду. Користувачі можуть абсолютно відчути цю різницю.

### 3. Доступність

Ви дійсно не повинні пропускати це. Деякі користувачі вважають анімацію запаморочливою або відволікаючою, і повага до їхніх уподобань є частиною створення хорошого продукту. Красиві речі не потрібно нав'язувати всім.

**CSS медіа-запит:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Provide a static fallback */
    opacity: 1;
  }
}
```

**Визначити налаштування користувача в React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Контроль рівня інтенсивності

Індикатор пропускної здатності Token у HagiCode показує різні кольори світіння на основі пропускної здатності в реальному часі, і це реалізується динамічно. Різні стани повинні виражатися по-різному.

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. На що слід звернути увагу

Є ще кілька деталей, на які варто звернути увагу, тому що до того часу, коли ви виявите ці проблеми важким шляхом, буде вже надто пізно.

| Речі, на які варто звернути увагу | Пояснення |
|---------|------|
| **керування z-індексом** | Шар світіння слід використовувати відповідний `z-index` тому це не заважає взаємодії вмісту |
| **показник-події** | Псевдоелемент світіння має встановитися `pointer-events: none` |
| **Переповнення кордону** | Необхідний батьківський контейнер `overflow: hidden`, або вам потрібно налаштувати розмір псевдоелемента |
| **Вплив на продуктивність** | Складні анімації можуть негативно вплинути на продуктивність мобільних пристроїв, тому ретельно тестуйте |
| **Темний режим** | Переконайтеся, що колір світіння залишається чітко видимим на темному тлі |
| **Зміна теми** | Використовуйте змінні CSS, щоб кольори анімації правильно оновлювалися під час зміни теми |

### 6. Поради щодо налагодження

Псевдоелементи може бути трохи складно знайти в інструментах розробника, тому ви можете тимчасово додати рамку, щоб перевірити положення.

```css
/* Temporarily show pseudo-element boundaries for debugging */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Після того, як ви завершите позиціонування, не забудьте прокоментувати або видалити цей рядок. Інакше виробництво може стати незручним досить швидко. Деякі речі краще залишити на розробку.

## Резюме

Анімація світлової розгортки меж не є ані особливо важкою, ані справді тривіальною. В основі формули `conic-gradient` плюс ротація, але якщо ви бажаєте хорошої продуктивності, зручності обслуговування та доступності, є ще багато деталей реалізації, з якими слід звертати увагу.

HagiCode зіштовхнувся з багатьма з цих пасток і поступово розробив набір найкращих практик. Саме так йдуть проекти: ви експериментуєте, робите помилки та вдосконалюєтеся крок за кроком. Якщо ви будуєте щось подібне, сподіваюся, ця стаття допоможе вам уникнути кількох непотрібних обхідних шляхів.

Деякі речі стають зрозумілими лише тоді, коли ви самі їх створите.
