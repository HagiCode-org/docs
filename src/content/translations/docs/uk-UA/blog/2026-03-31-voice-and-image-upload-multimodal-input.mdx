## Введення тексту повільніше, ніж розмова, а розмова повільніше, ніж знімок екрана

> Написання коду має обмеження швидкості незалежно від того, наскільки швидко ви друкуєте. Іноді щось, що ви можете сказати одним реченням, займає вічність, щоб надрукувати; іноді один скріншот пояснює все, але все одно потрібно описати це купою тексту. У цій статті розповідається про те, з чим ми зіткнулися під час будівництва [HagiCode](https://hagicode.com), від розпізнавання мовлення до завантаження зображень. Зрештою, ми просто хотіли зробити помічника кодування штучного інтелекту трохи простішим у використанні.

## Фон

Створюючи HagiCode, ми помітили проблему — точніше, проблему, яка природно виникла, коли люди почали активно його використовувати: покладатися лише на введення тексту може бути втомливим.

Подумайте про це: взаємодія між користувачами та Агентом є основним сценарієм. Але якщо кожен обмін вимагає безперервного набору тексту з клавіатури, ефективність не велика:

1. **Введення надто повільне**: для складних проблем, як-от повідомлення про помилки чи проблеми з інтерфейсом користувача, введення всього може тривати півхвилини, тоді як промовляння вголос може тривати десять секунд. Цей розрив реальний.

2. **Зображення більш прямі**: іноді інтерфейс користувача видає помилку, іноді потрібно порівняти чернетку дизайну, а іноді потрібно показати структуру коду. «Зображення варте тисячі слів», можливо, це стара приказка, але вона все ще актуальна. Дозволити штучному інтелекту безпосередньо «побачити» проблему набагато зрозуміліше, ніж описувати її століттями.

3. **Взаємодія має бути природною**: сучасні помічники ШІ мають підтримувати текст, голос і зображення. Користувачі повинні мати можливість вибирати будь-який спосіб введення, який буде найбільш природним.

Тому ми вирішили додати підтримку розпізнавання мовлення та завантаження зображень у HagiCode, щоб зробити взаємодію з агентами зручнішою. Якщо користувачі можуть друкувати трохи менше, це вже перемога.

## Аналіз

### Технічні проблеми в розпізнаванні мовлення

Під час створення розпізнавання мовлення ми зіткнулися зі складною проблемою: **API WebSocket браузера не підтримує користувацькі заголовки HTTP**.

Службою розпізнавання мовлення, яку ми обрали, був ByteDance Doubao Speech Recognition API. На жаль, для цього API потрібна інформація для автентифікації, наприклад `accessToken` і `secretKey` передаватися через заголовки HTTP. Це призвело до негайного технічного конфлікту:

```javascript
// The browser WebSocket API does not support this approach
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // Not supported
  }
});
```

В основному у нас було два варіанти:

1. **Підхід до параметрів запиту URL-адреси**: розмістіть інформацію про автентифікацію в URL-адресі
   - Перевага полягає в тому, що його просто реалізувати
   - Недоліком є те, що облікові дані доступні для інтерфейсу, що є небезпечним; деякі API також вимагають автентифікації на основі заголовка

2. **Бекенд-проксі-підхід**: запровадьте проксі-сервер WebSocket на сервері
   - Перевагою є те, що облікові дані надійно зберігаються на сервері, а рішення повністю сумісне з вимогами API
   - Недоліком є те, що впровадження трохи складніше

Зрештою, ми обрали **підхід серверного проксі**. Безпека – це не те, чим ви йдете на компроміс.

### Функціональні вимоги до завантаження зображень

Наші вимоги до завантаження зображень насправді були досить простими:

1. **Кілька методів завантаження**: натисніть, щоб вибрати файл, перетягніть і вставте з буфера обміну
2. **Перевірка файлів**: обмеження типу (PNG, JPG, WebP, GIF) і розміру (5-10 МБ) є основними вимогами
3. **Взаємодія з користувачем**: хід завантаження, попередній перегляд і повідомлення про помилки, щоб користувачі завжди знали, що відбувається
4. **Безпека**: перевірка на сервері та захист від зловмисного завантаження файлів є важливими

## Рішення

### Розпізнавання мовлення: архітектура проксі-сервера WebSocket

Ми розробили трирівневу архітектуру для розпізнавання мовлення та знайшли шлях, який спрацював:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**Реалізація основних компонентів**:

1. **Процесор Frontend AudioWorklet**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Resample to 16 kHz (required by the Doubao API)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Accumulate samples into 500 ms chunks
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Convert to 16-bit PCM and send
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **Бекендовий обробник WebSocket** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **Фронтенд `VoiceTextArea` компонент**:

```tsx
export const VoiceTextArea = forwardRef<HTMLTextAreaElement, VoiceTextAreaProps>(
  ({ value, onChange, onTextRecognized, maxDuration }, ref) => {
    const { isRecording, interimText, volume, duration, startRecording, stopRecording } =

  }

```

### Завантаження зображень: компонент завантаження кількох методів

Ми створили повнофункціональний компонент завантаження зображень із підтримкою всіх трьох методів завантаження, охоплюючи найпоширеніші сценарії, з якими стикаються користувачі.

**Основні функції**:

1. **Три методи завантаження**:

```tsx
// Click to upload
const handleClick = () => fileInputRef.current?.click();

// Drag-and-drop upload
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Clipboard paste
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **Перевірка інтерфейсу**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **Обробник серверного завантаження** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Validation
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Save file
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## Практичний посібник

### Як використовувати розпізнавання мовлення

1. **Налаштуйте службу розпізнавання мовлення**:
   - Відкрийте сторінку налаштувань розпізнавання мовлення
   - Налаштуйте мову Doubao `AppId` і `AccessToken`
   - Додатково налаштуйте гарячі слова, щоб підвищити точність розпізнавання доменних термінів

2. **Використовуйте його у полі введення**:
   - Натисніть піктограму мікрофона ліворуч від поля введення
   - Почніть говорити після появи анімації хвилі
   - Клацніть піктограму ще раз, щоб зупинити запис
   - Розпізнаний текст автоматично вставляється в позицію курсора

3. **Приклад налаштування гарячого слова**:

```text
TypeScript
React
useState
useEffect
```

### Як використовувати завантаження зображень

1. **Методи завантаження**:
   - Натисніть кнопку завантаження, щоб вибрати файл
   - Перетягніть зображення безпосередньо в область завантаження
   - використання `Ctrl+V` щоб вставити знімок екрана з буфера обміну

2. **Підтримувані формати**: PNG, JPG, JPEG, WebP, GIF
3. **Обмеження розміру**: 5 МБ за замовчуванням (налаштовується)

### Примітки

1. **Розпізнавання мовлення**:
   - Потрібен дозвіл на використання мікрофона
   - По можливості використовуйте в тихому середовищі
   - Максимальна підтримувана тривалість запису становить 300 секунд за замовчуванням (налаштовується)

2. **Завантаження зображень**:
   - Підтримуються лише типові формати зображень
   - Зверніть увагу на обмеження розміру файлу
   - Завантажені зображення автоматично отримують URL попереднього перегляду

3. **Заходи безпеки**:
   - Облікові дані розпізнавання мовлення зберігаються на сервері
   - Завантаження зображень проходить сувору перевірку на сервері
   - HTTPS/WSS рекомендується використовувати у робочих середовищах
