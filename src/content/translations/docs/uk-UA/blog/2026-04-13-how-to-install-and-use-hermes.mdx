Якщо ви хочете встановити Hermes і почати ним користуватися, найкоротший шлях складається з трьох кроків:

1. Виконайте офіційну команду встановлення
2. Запустіть CLI у вашому терміналі за допомогою `hermes`
3. Якщо ви хочете продовжувати використовувати його у Feishu, тоді налаштуйте `hermes gateway setup`

Ця стаття не намагається пояснити всі можливості Hermes відразу. Натомість це допоможе вам спочатку завершити найважливіший цикл для початківців: встановити його, запустити, почати використовувати, а потім підключити до одного з найпоширеніших сценаріїв платформи обміну повідомленнями.

## Що таке Гермес і для кого він

Hermes Agent — це агент штучного інтелекту, яким можна користуватися з локального терміналу або через шлюз платформи обміну повідомленнями.

Для більшості розробників він має дві загальні точки входу:

- **CLI**: тип `hermes` у вашому терміналі, щоб увійти безпосередньо в інтерактивний інтерфейс.
- **Шлюз повідомлень**: запустити `hermes gateway`, потім спілкуйтеся з ним на таких платформах, як Feishu, Telegram, Discord і Slack.

Якщо ваша мета зараз — просто швидко почати, не змінюйте порядок. Натомість почніть із цього шляху:

- Спочатку встановіть Hermes
- Спочатку переконайтеся, що він працює з CLI
- Потім вирішіть, чи хочете ви підключити платформу обміну повідомленнями

Це полегшує діагностику проблем і більше підходить для людей, які використовують Hermes вперше.

## Що потрібно знати перед встановленням Hermes

Відповідно до Hermes README, офіційний шлях швидкого встановлення підтримує такі середовища:

- Linux
- macOS
- WSL2
- Android через Termux

### Примітки для користувачів Windows

Наразі Hermes не підтримує запуск безпосередньо у рідній Windows. Якщо ви використовуєте Windows, рекомендований підхід полягає в тому, щоб спочатку встановити WSL2, а потім запустити команду встановлення в WSL2.

Найкраще пояснити це на початку, оскільки багато помилок інсталяції спричинені не самою командою, а використанням непідтримуваного середовища виконання.

## Як встановити Hermes

Команда швидкого встановлення, наведена в Hermes README, така:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Ця команда запускає офіційний сценарій інсталяції та обробляє специфічні для платформи кроки ініціалізації.

### Перезавантажте оболонку після встановлення

Після завершення встановлення спочатку перезавантажте середовище оболонки. Найпоширеніша команда:

```bash
source ~/.bashrc
```

Якщо ви використовуєте zsh, ви можете використовувати:

```bash
source ~/.zshrc
```

### Як переконатися, що Hermes встановлено правильно

Найпряміший спосіб перевірити це запустити:

```bash
hermes
```

Якщо вам потрібно додаткове підтвердження того, що ваша конфігурація та залежності працюють, ви також можете запустити:

```bash
hermes doctor
```

`hermes doctor` особливо корисний у таких ситуаціях:

- Після встановлення команда поводиться ненормально
- Помилка налаштування моделі
- Не вдається запустити шлюз
- Ви не впевнені, що ваші залежності середовища повні

## Як почати використовувати Hermes вперше

### Почніть з CLI

Якщо ви просто хочете якомога швидше підтвердити, що Hermes працює, найпростіший спосіб:

```bash
hermes
```

Це запускає інтерактивний Hermes CLI. Для тих, хто вперше користується Hermes, це також найбільш рекомендована відправна точка, оскільки ви можете спочатку перевірити найважливіші речі:

- Чи дійсно команда доступна
- Чи правильно працює поточна конфігурація моделі
- Чи правильно працює ланцюжок інструментів терміналу
- Чи відповідає стиль взаємодії тому, що вам потрібно

### Цих команд достатньо для першого етапу налаштування

Hermes README містить кілька високочастотних команд, і разом вони утворюють практичний шлях першого використання:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Якщо ви не впевнені, що кожен з них робить, запам’ятайте їх так:

- `hermes model`: вибрати або переключити моделі
- `hermes tools`: перегляд і налаштування доступних інструментів
- `hermes config set`: змінити певні елементи конфігурації
- `hermes setup`: один раз запустіть повний майстер ініціалізації
- `hermes update`: оновлення Hermes
- `hermes doctor`: усунути проблеми

Для початківців найбільш практичний порядок зазвичай такий:

1. бігти `hermes model` перший
2. Якщо ви хочете налаштувати всі загальні параметри одночасно, запустіть `hermes setup`

## Два найпоширеніших способи використання Hermes

### 1. Використовуйте Hermes у терміналі як щоденного помічника з розробки

Режим CLI добре підходить для таких сценаріїв:

- Ставте запитання безпосередньо під час написання коду локально
- Перевіряйте проекти, редагуйте файли та запускайте команди
- Виконайте одноразову роботу з налагодження або перегляду
- Постійно співпрацюйте в поточному робочому каталозі

Його найбільша перевага полягає в тому, що це найкоротший шлях: жодної додаткової інтеграції з платформою, жодної конфігурації ботів, які потрібно обробляти наперед, і це найкращий спосіб вибудувати свій перший набір звичок використання.

### 2. Використовуйте Hermes через платформу обміну повідомленнями

Якщо ви хочете спілкуватися з Hermes на таких платформах, як Feishu, Telegram або Discord, вам потрібно використовувати шлюз обміну повідомленнями.

Найпоширеніші команди введення:

```bash
hermes gateway setup
hermes gateway
```

зокрема:

- `hermes gateway setup` використовується для інтерактивної конфігурації платформи
- `hermes gateway` використовується для запуску процесу шлюзу

Згідно з офіційною документацією, шлюз — це уніфікований фоновий процес, який з’єднує ваші налаштовані платформи, керує сеансами та обробляє такі функції, як завдання cron.

## На прикладі Feishu: як підключити Hermes до платформи обміну повідомленнями

Якщо більша частина вашої щоденної роботи відбувається у Feishu, то Feishu/Lark — це дуже природний спосіб використання Hermes.

### Мінімальний життєздатний шлях інтеграції

Офіційна документація рекомендує цю команду введення для Feishu/Lark:

```bash
hermes gateway setup
```

Після запуску просто виберіть **Feishu / Lark** у майстрі.

Документація Feishu описує два режими підключення:

- `websocket`: рекомендовано
- `webhook`: необов'язковий

Якщо Hermes працює на вашому ноутбуці, робочій станції чи приватному сервері, використовуючи `websocket` перший зазвичай простіше, тому що вам не потрібно надавати загальнодоступну URL-адресу зворотного виклику.

### Якщо ви налаштовуєте це вручну, принаймні знайте ці змінні

Якщо ви не використовуєте майстер і пишете конфігурацію вручну, документація Feishu містить перелік цих основних змінних:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Особливої уваги заслуговують два з них:

- `FEISHU_ALLOWED_USERS`: рекомендовано, тому не всі, хто має доступ до бота, можуть використовувати його безпосередньо
- `FEISHU_HOME_CHANNEL`: дозволяє заздалегідь визначити домашній чат для отримання результатів cron або сповіщень за замовчуванням

### Чому Hermes іноді не відповідає в групових чатах Feishu

Цю деталь легко не помітити: у групових чатах Feishu Hermes за замовчуванням не відповідає на кожне повідомлення.

В офіційній документації чітко зазначено:

- У прямих повідомленнях Гермес відповідає на повідомлення
- У групових чатах ви повинні явно `@` бот, перш ніж він обробить повідомлення

Якщо ви хочете встановити розмову Feishu як домашній канал, ви також можете використовувати це в чаті:

```text
/set-home
```

Або визначте його в конфігурації заздалегідь:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Накази Гермеса новачкам слід запам'ятати в першу чергу

Незалежно від того, чи використовуєте ви Hermes у CLI чи на платформі обміну повідомленнями, для початку достатньо запам’ятати наступні команди:

- `/new` або `/reset`: почати новий сеанс
- `/model`: переглянути або переключити модель
- `/retry`: повторити попередній хід
- `/undo`: скасувати попередню взаємодію
- `/compress`: вручну стиснути контекст
- `/help`: переглянути довідку

Якщо ви переважно використовуєте Hermes на платформі обміну повідомленнями, запам’ятайте ще одне:

- `/sethome` або `/set-home`: встановити поточний чат як домашній канал

Ці команди охоплюють найпоширеніші операції початкового етапу: перезапуск, налаштування, відкат, перевірка та продовження.

## Часті запитання

### Чи може Windows безпосередньо встановити Hermes?

Ні. У поточній офіційній документації чітко зазначено, що рідна Windows не підтримується, а WSL2 рекомендовано.

### Що я маю робити, якщо друкую `hermes` нічого не робить після установки?

Найкраще усувати несправності в такому порядку:

1. Спочатку перезавантажте свою оболонку, наприклад за допомогою `source ~/.bashrc`
2. бігти `hermes` знову
3. Якщо це все ще ненормально, біжіть `hermes doctor`

### Чому бот не відповідає в групі Feishu?

Спочатку перевірте ці три речі:

- Чи ви `@` згадував Гермеса в групі
- чи `FEISHU_ALLOWED_USERS` обмежує поточного користувача
- Чи дозволяє поточна політика групового чату обробку групових повідомлень

Згідно з офіційною документацією Feishu, у сценаріях групового чату потрібне явне використання @mention.

## Рекомендовані наступні дії

Якщо ви просто хочете почати використовувати Hermes якомога швидше, це найбільш рекомендований порядок:

1. Спочатку запустіть команду встановлення
2. Почніть з `hermes` спочатку в локальному CLI
3. використання `hermes model` і `hermes setup` щоб завершити базову конфігурацію
4. Якщо ви хочете продовжувати використовувати його у Feishu, тоді налаштуйте `hermes gateway setup`

Якщо ця стаття є першою частиною серії, її найкраща роль полягає не в тому, щоб пояснити всі додаткові функції відразу, а в тому, щоб спочатку залучити користувачів.

Наступні теми краще розділити на наступні статті:

- Повний посібник з інтеграції Hermes Feishu
- Посібник із типових команд Hermes зі слешами
- Посібник із налаштування шлюзу Hermes та усунення несправностей

Якщо ви плануєте продовжувати створювати контент Hermes, ця стаття може послужити відправною точкою для наступних публікацій, поки ви поступово створюватимете внутрішню структуру посилань.
