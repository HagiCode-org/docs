## Як гейміфікаційний дизайн робить кодування ШІ веселішим

> Традиційні інструменти кодування ШІ насправді досить потужні; їм просто не вистачає тепла. Коли ми створювали HagiCode, ми думали: якщо ми все одно збираємося писати код, чому б не перетворити це на гру?

## Фон

Кожен, хто користувався помічником кодування штучного інтелекту, напевно, мав такий досвід: спочатку це здається свіжим і захоплюючим, але через деякий час починає здаватися, що чогось не вистачає. Сам інструмент потужний, здатний генерувати код, автозаповнювати та виправляти помилки, але... він не дуже теплий, і з часом може стати одноманітним і нудним.

Одного цього достатньо, щоб змусити вас задуматися, хто хоче щодня дивитися на холодний, безособовий інструмент.

Це трохи схоже на гру. Якщо все, що ви робите, це завершуєте список завдань, без зростання персонажа, без розблокування досягнень і без координації команди, це швидко перестає бути веселим. Красиві речі та люди не потребують володіння ними, щоб їх цінували; їх краси достатньо самої по собі. Інструменти програмування навіть не пропонують такої краси, тому легко впасти духом.

Саме з цією проблемою ми зіткнулися під час розробки [HagiCode](https://hagicode.com). Як платформа для спільної роботи з декількома помічниками штучного інтелекту, HagiCode має залучати користувачів протягом тривалого часу. Але насправді навіть чудовий інструмент важко використовувати, якщо він не має жодного емоційного зв’язку.

Щоб вирішити цю проблему, ми прийняли сміливе рішення: перетворити програмування на гру. Не поверхневий вид із простою таблицею лідерів, а справжня рольова ігра. Вплив цього рішення може бути навіть більшим, ніж ви собі уявляєте.

Зрештою, люди потребують трохи ритуалу в своєму житті.

## Про HagiCode

Ідеї, наведені в цій статті, випливають із нашого практичного досвіду [HagiCode](https://hagicode.com) проект. HagiCode — це платформа для спільної роботи з кількома помічниками ШІ, яка підтримує спільну роботу Claude Code, Codex, Copilot, OpenCode та інших помічників ШІ. Якщо вас цікавить співпраця з декількома ШІ або гейміфіковане програмування, відвідайте [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site) щоб дізнатися більше.

Нічого особливо загадкового в цьому немає. Ми просто перетворили програмування на пригоду.

## Чому варто вибрати Gamification

Суть гейміфікації полягає не просто в «додаванні таблиці лідерів». Йдеться про створення повної системи заохочень, щоб користувачі могли відчувати зростання, досягнення та соціальне визнання під час виконання завдань.

Дизайн гейміфікації HagiCode обертається навколо однієї ключової ідеї: кожен помічник ШІ є «Героєм», а користувач — капітаном цієї команди Героїв. Ви ведете цих Героїв підкорювати різноманітні «Підземелля» (завдання з програмування). По дорозі Герої отримують досвід, підвищують рівень, відкривають здібності, а ваша команда також отримує досягнення.

Це не трюк. Це дизайн, заснований на поведінковій психології людини. Коли завданням надається сенс і відбувається зворотний зв’язок, залученість і наполегливість людей значно зростає.

Як говорить стара приказка: «Це відчуття може стати спогадом, хоча в той час воно збентежило нас». Ми вносимо цей емоційний досвід в інструмент, тому програмування – це вже не просто введення коду, а подорож, яку варто запам’ятати.

## Система героїв

Герой — це основна концепція системи гейміфікації HagiCode. Кожен герой представляє одного помічника ШІ. Наприклад, Клод Код — Герой, і Кодекс — теж Герой.

### Слот три героя

Герой має три слоти для обладнання, а дизайн напрочуд елегантний:

1. **Слот CLI (основний клас)**: визначає базову здатність Героя, наприклад, Клод Код чи Кодекс
2. **Слот моделі (другий клас)**: визначає, яка модель використовується, наприклад Claude 4.5 або Claude 4.6
3. **Слот стилю (стиль)**: Визначає стиль поведінки Героя, як-от «Fengluo Strategist» або інший стиль

Поєднання цих трьох слотів створює унікальні конфігурації Героїв. Подібно до збірки обладнання в іграх, ви вибираєте правильне налаштування на основі завдання. Зрештою, найважливішим є те, що вам підходить найкраще. Життя схоже: багато доріг ведуть до Риму, але деякі більш гладкі, ніж інші.

### Система зростання героїв

Кожен герой має свій досвід і рівень:

```typescript
type HeroProgressionSnapshot = {
  currentLevel: number;                    // Current level
  totalExperience: number;                 // Total experience
  currentLevelStartExperience: number;     // Experience at the start of the current level
  nextLevelExperience: number;             // Experience required for the next level
  experienceProgressPercent: number;       // Progress percentage
  remainingExperienceToNextLevel: number;  // Experience still needed for the next level
  lastExperienceGain: number;              // Most recent experience gained
  lastExperienceGainAtUtc?: string | null; // Time when experience was gained
};
```

Рівні поділяються на чотири етапи, і кожен етап має захоплюючу назву:

```typescript
export const resolveHeroProgressionStage = (level?: number | null): HeroProgressionStage => {
  const normalizedLevel = Math.max(1, level ?? 1);
  if (normalizedLevel <= 100) return 'rookieSprint';      // Rookie sprint
  if (normalizedLevel <= 300) return 'growthRun';         // Growth run
  if (normalizedLevel <= 700) return 'veteranClimb';      // Veteran climb
  return 'legendMarathon';                                // Legend marathon
};
```

Від «новачка» до «легенди» цей шлях зростання дає користувачам чітке уявлення про напрямок і досягнення. Це віддзеркалення особистісного зростання в житті, від розгубленості до зрілості, лише тут стає більш відчутним.

Щоб створити Героя, потрібно налаштувати три слоти:

Кожен герой має унікальний аватар, опис і професійну ідентичність, що надає тому, що інакше було б холодним ШІ-помічником, більше індивідуальності та тепла. Зрештою, хто хоче працювати з інструментом, який не має характеру?

## Система підземель

«Підземелля» — це класична ігрова концепція, що представляє виклик, який вимагає команди, щоб пройти. У HagiCode кожен робочий процес є Dungeon.

### Як організовані підземелля

Dungeon організовує робочі процеси в різні «Підземелля»:

- **Підземелля створення пропозицій**: відповідає за створення технічних пропозицій
- **Підземелля виконання пропозицій**: відповідає за виконання завдань у пропозиціях
- **Підземелля архіву пропозицій**: Відповідає за організацію та архівування готових пропозицій

Кожне підземелля має власного капітана-героя, і капітан автоматично вибирається першим активним героєм.

Насправді це просто розподіл праці, як і в повсякденному житті, за винятком перетворення в ігрову механіку.

### Механізм командної співпраці

Ви можете налаштувати різні загони Героїв для різних підземель:

```typescript
const dungeonRoster: HeroDungeonRoster = {
  scriptKey: 'proposal.generate',
  displayName: 'Proposal Generation',
  members: [
    { heroId: 'hero-1', name: 'Athena', executorType: 'ClaudeCode' },
    { heroId: 'hero-2', name: 'Apollo', executorType: 'Codex' }
  ]
};
```

Наприклад, ви можете використовувати Athena для генерації пропозицій, оскільки вона хороша в стратегії, і Apollo для впровадження коду, оскільки вона хороша у виконанні. Таким чином кожен герой може використовувати свої сильні сторони. Це як створення гурту: кожна людина має інструмент, і разом вони створюють щось прекрасне.

### Керування потоком підземелля

Виправлено використання Dungeon `scriptKey` значення для визначення різних робочих процесів:

```typescript
// Script keys map to different workflows
const dungeonScripts = {
  'proposal.generate': 'Proposal Generation',
  'proposal.execute': 'Proposal Execution',
  'proposal.archive': 'Proposal Archive'
};
```

Потік стану завдання: `queued` (очікування) -> `dispatching` (призначається) -> `dispatched` (призначений). Весь процес автоматизований і не вимагає ручного втручання. Це теж частина нашої лінивої сторони, тому що хто хоче керувати цими речами вручну?

## XP і система рівня

XP є основним механізмом зворотного зв’язку в системі гейміфікації. Користувачі отримують XP, виконуючи завдання, XP підвищує рівень Героїв, а підвищення рівня відкриває нові здібності, формуючи позитивний зворотний зв’язок.

### Способи отримання XP

У HagiCode XP можна отримати за такі дії:

- Завершення виконання коду
- Успішний виклик інструментів
- Формування пропозицій
- Операції керування сеансами
- Проектні операції

Кожного разу, коли виконується дійсна дія, відповідний герой отримує XP. Так само, як зростання в житті, кожен крок має значення, тільки тут це зростання кількісно.

### Візуалізація прогресу в реальному часі

XP та прогрес рівня візуалізуються в реальному часі:

```typescript
type HeroDungeonMember = {
  heroId: string;
  name: string;
  icon?: string | null;
  executorType: PCode_Models_AIProviderType;
  currentLevel?: number;                    // Current level
  totalExperience?: number;                 // Total experience
  experienceProgressPercent?: number;       // Progress percentage
};
```

Користувачі завжди можуть бачити рівень і прогрес кожного Героя, і цей миттєвий відгук є ключем до дизайну гейміфікації. Людям потрібен зворотний зв’язок, інакше як би вони дізналися, що покращуються?

## Система досягнень

Досягнення є ще одним важливим елементом гейміфікації. Вони забезпечують довгострокові цілі та задоволення, кероване віхою.

### Типи досягнень

HagiCode підтримує декілька типів досягнень:

- **Досягнення генерації коду**: генеруйте X рядків коду, генеруйте Y файлів
- **Досягнення в управлінні сеансом**: завершіть бесіди Z
- **Досягнення роботи над проектом**: робота над проектами W

Ці досягнення справді схожі на віхи в житті, за винятком того, що ми перетворили їх на ігрову механіку.

### Досягнення держав

Досягнення мають три стани:

```typescript
type AchievementStatus = 'unlocked' | 'in-progress' | 'locked';
```

Три стани мають чіткі візуальні відмінності:

- **Розблоковано**: золотий градієнт із ефектом ореолу
- **Виконується**: синя анімація пульсу
- **Заблоковано**: сірий, із показаними умовами розблокування

Кожне досягнення чітко відображає умову запуску, щоб користувачі знали, що робити далі. Коли люди почуваються розгубленими, невелика підказка завжди допомагає.

### Ефект святкування при розблокуванні

Коли досягнення розблоковано, запускається анімація святкування. Таке позитивне підкріплення дає користувачам відчуття задоволення «Я це зробив» і мотивує їх продовжувати. Маленькі нагороди в житті діють так само: вони можуть бути невеликими, але щастя може тривати довго.

### Детальні карти героїв

Кожна карта Героя містить:

Ці показники допомагають користувачам чітко зрозуміти, як працює команда. Бачити результати власних зусиль саме по собі приносить задоволення.

## Технічна реалізація

Система гейміфікації HagiCode використовує стек сучасних технологій і шаблони проектування. У цьому немає нічого особливо чарівного; ми просто вибрали інструменти, які відповідають роботі.

### Вибір стеку технологій

```typescript
// React + TypeScript for the frontend
import React from 'react';

// Framer Motion for animations
import { AnimatePresence, motion } from 'framer-motion';

// Redux Toolkit for state management
import { useAppDispatch, useAppSelector } from '@/store';
```

Framer Motion обробляє всі анімаційні ефекти, shadcn/ui забезпечує базові компоненти інтерфейсу користувача, а Redux Toolkit керує складним станом гейміфікації. Хороші інструменти роблять хорошу роботу.

### Gamified UI Design System

HagiCode використовує стиль дизайну Glassmorphism + Tech Dark:

```css
/* Primary gradient */
background: linear-gradient(135deg, #22C55E 0%, #25c2a0 50%, #06b6d4 100%);

/* Glass effect */
backdrop-filter: blur(12px);

/* Glow effect */
```

Зелений градієнт у поєднанні зі скломорфізмом створює технічну футуристичну атмосферу. Візуальна краса також є частиною взаємодії з користувачем.

### Анімаційні ефекти

Framer Motion використовується для створення плавної анімації входу:

```typescript
<motion.div
  animate={{ opacity: 1, y: 0 }}
  initial={{ opacity: 0, y: 18 }}
  transition={{ duration: 0.35, ease: 'easeOut', delay: index * 0.08 }}
  className="card"
>
  {/* Card content */}

```

Кожна картка входить одна за одною із затримкою 0,08 секунди, створюючи плавний візуальний ефект. Плавна анімація покращує враження. З цією частиною важко сперечатися.

### Постійність даних

Дані гейміфікації зберігаються за допомогою системи зберігання Grain для забезпечення узгодженості стану. Навіть такі дрібні дані, як накопичений досвід Hero XP, можна точно зберігати. Ніхто не хоче втрачати досвід, над яким наполегливо працював.

## Практичний посібник

### Створіть свого першого героя

Створити свого першого героя насправді досить просто:

1. Перейдіть на сторінку керування героями
2. Натисніть кнопку «Створити героя».
3. Налаштуйте три слоти (CLI, модель, стиль)
4. Дайте Герою ім'я та опис
5. Збережіть його, і народиться ваш перший Герой

Це як знайомство з новим другом: ви даєте йому ім’я, дізнаєтесь, що робить його особливим, а потім разом вирушаєте назустріч пригодам.

### Створіть команду Dungeon

Створити команду також просто:

1. Перейдіть на сторінку управління підземеллями
2. Виберіть підземелля, яке ви хочете налаштувати, наприклад «Генерація пропозицій»
3. Виберіть учасників зі свого списку героїв
4. Система автоматично вибирає першого ввімкненого героя як капітана
5. Збережіть конфігурацію

Це просто процес формування команди, схожий на створення команди в реальному житті, де кожен має свою власну роль.

### Переглянути щоденний звіт

Наприкінці кожного дня ви можете переглянути звіт про битву за день:

1. Натисніть кнопку «Звіт про бій».
2. Перегляньте результати роботи за день у повноекранному режимі
3. Перевірте MVP і докладні дані для кожного героя
4. Поділіться ним з членами команди, якщо хочете

Це також свого роду ритуал, спосіб побачити, скільки ви сьогодні доклали зусиль і як далеко ви ще до своєї мети.

## Примітки та найкращі практики

### Оптимізація продуктивності

використання `React.memo` щоб уникнути непотрібних повторних візуалізацій:

```typescript
const HeroCard = React.memo(({ hero }: { hero: HeroDungeonMember }) => {
  // Component implementation
});
```

Продуктивність також має значення. Ніхто не хоче використовувати інструмент із затримкою.

### Рух може витончено погіршуватися

Визначайте налаштування рухів користувача та забезпечуйте спрощену роботу для користувачів, чутливих до рухів:

```typescript
const prefersReducedMotion = useReducedMotion();
const duration = prefersReducedMotion ? 0 : 0.35;
```

Не всім подобається анімація, і повага до вподобань користувача є частиною гарного дизайну.

### Зворотна сумісність

Тримайте `legacyIds` для підтримки міграції зі старіших версій:

```typescript
type HeroDungeonMember = {
  heroId: string;
  legacyIds?: string[];  // Supports legacy ID mapping
  // ...
};
```

Ніхто не хоче втрачати дані лише через оновлення версії.

### Підтримка інтернаціоналізації

Використовуйте ключі перекладу i18n для всього тексту, щоб полегшити багатомовну підтримку:

```typescript
const displayName = t(`dungeon.${scriptKey}`, { defaultValue: displayName });
```

Мова ніколи не повинна бути перешкодою для використання продукту.

## Резюме

Гейміфікація — це не просто таблиця лідерів, а повна система заохочень. За допомогою системи Hero, системи Dungeon, системи XP і рівнів, системи досягнень і Battle Report HagiCode перетворює роботу з програмування на героїчну подорож, повну пригод.

Основна цінність цієї системи полягає в:

- **Емоційний зв’язок**: надає холодну індивідуальність помічникам ШІ
- **Позитивний відгук**: кожна дія викликає негайний відгук
- **Довгострокові цілі**: рівні та досягнення забезпечують шлях зростання
- **Ідентичність команди**: відчуття співпраці в командах Dungeon
- **Честь і визнання**: демонстрації звіту про битву та MVP

Дизайн гейміфікації робить програмування не нудним, а цікавою пригодою. Виконуючи завдання кодування, користувачі також відчувають задоволення від розвитку персонажа, командної співпраці та розблокування досягнень, що покращує утримання та активність.
