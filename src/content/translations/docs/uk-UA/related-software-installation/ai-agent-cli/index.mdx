Ця сторінка є уніфікованим локальним записом інсталяції агента AI Agent CLI `repos/docs`. Наразі HagiCode підтримує 11 активних CLI агентів, які слідують за серверною частиною `PrimaryProfessions` порядок: Claude Code, Codex, GitHub Copilot, OpenCode, Hermes, QoderCLI, Kiro, Kimi, Gemini, DeepAgents і Codebuddy.

:::tip[Як користуватися цією сторінкою]
Спершу перевірте таблицю щодо поточного стану підтримки та режиму налаштування, а потім перейдіть до документів щодо інсталяції, автентифікації, ACP, дозволів і усунення несправностей, які підтримує постачальник. Обсяг підтримки та режим налаштування — це окремі питання.
:::

## Державний глосарій

- **Наразі підтримується** означає, що CLI належить до поточного активного діапазону підтримки HagiCode.
- **Встановлення одним клацанням миші** означає, що HagiCode забезпечує керовану точку входу для встановлення; деякі CLI все ще вимагають входу постачальника або етапу авторизації після встановлення.
- **Встановлення/автентифікація вручну** означає, що CLI зараз підтримується, але ви повинні завершити локальне встановлення або процес автентифікації, перш ніж HagiCode зможе виявити та використати його.
- **Лише сумісність зі старими версіями** означає вилучене значення, наприклад `IFlowCli` все ще існує для сумісності історичних даних, але він не є частиною поточного активного діапазону підтримки.

## Рекомендовані наступні дії

- Продовжити з [Встановіть Node.js](/en/related-software-installation/nodejs/installation) і [Встановіть OpenSpec](/en/related-software-installation/openspec/setup-openspec) якщо вам ще потрібні передумови.
- Якщо вам потрібен один локальний шлюз для кількох інструментів, сумісних з OpenAI, продовжуйте [Встановлення та використання OmniRoute](/en/related-software-installation/omniroute/).
- Повернутися до [Керівництво по установці](/en/installation) для ширшого шляху налаштування.
- Якщо ви потрапили сюди з історичної публікації в блозі, сприймайте цю сторінку та огляд інсталяції як поточне вічне джерело правди.
