# Встановіть OpenSpec

OpenSpec керує пропозиціями, специфікаціями та записами про зміни. Ця сторінка має одну мету: чітко пояснити шлях інсталяції, щоб ви могли приєднати його до наявного робочого процесу.

## Що таке OpenSpec?

OpenSpec — це інструмент командного рядка для керування пропозиціями та специфікаціями в роботі програмного забезпечення. Основний робочий процес включає:

- **Керування пропозиціями**: створюйте, переглядайте та відстежуйте пропозиції функцій
- **Перевірка специфікації**: переконайтеся, що зміни коду відповідають попередньо визначеним специфікаціям
- **Відстеження змін**: записуйте історію всіх переглянутих змін
- **Командна співпраця**: підтримка спільної розробки за участю кількох осіб

### Застосування в проектах HagiCode

У проектах HagiCode OpenSpec використовується для:

1. **Керування розробкою функцій**: усі нові функції мають пройти процес пропозиції OpenSpec
2. **Перевірка коду**: переконайтеся, що зміни відповідають специфікаціям проекту
3. **Синхронізація документації**: підтримуйте узгодженість між документацією та впровадженням коду
4. **Інтеграція контролю версій**: бездоганна інтеграція з робочими процесами контролю версій

## передумови

Перш ніж інсталювати OpenSpec, переконайтеся, що ваша система відповідає таким вимогам:

### Node.js і npm

OpenSpec поширюється через npm і потребує середовища Node.js. OpenSpec вимагає **Node.js 18.0 або вище**.

Перевірте встановлення Node.js:

```bash
node --version
```

Перевірте встановлення npm:

```bash
npm --version
```

Якщо не встановлено або версія застаріла, зверніться до відповідних інструкцій зі встановлення програмного забезпечення в [Керівництво по установці](/en/installation).

## Встановлення OpenSpec

:::tip[Вимоги до версії]
Цей проект вимагає використання **OpenSpec 1.0.0 або новішої версії**. Усі версії 1.x підтримуються для забезпечення сумісності з робочим процесом проекту.
:::

OpenSpec встановлюється глобально через npm, що робить його доступним з будь-якої точки вашої системи.

### Встановити глобально за допомогою npm

<Tabs>
  <TabItem value="win" label="Windows">
    Запустіть у PowerShell або командному рядку:

```powershell
npm install -g @fission-ai/openspec@1
```

:::note[Проблеми з дозволами Windows]
Якщо під час інсталяції виникнуть помилки дозволів, запустіть PowerShell або командний рядок від імені адміністратора.
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    Запустити в терміналі:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[Проблеми з дозволом macOS]
Якщо ви зіткнетеся `EACCES` помилки дозволу, спробуйте:
```bash
sudo npm install -g @fission-ai/openspec@1
```
Або використовувати [nvm](https://github.com/nvm-sh/nvm) щоб керувати версіями Node.js і уникнути проблем з дозволами.
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    Запустити в терміналі:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[Проблеми з дозволом Linux]
Якщо ви зіткнетеся `EACCES` помилки дозволу, ви можете:
1. використання `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. Налаштуйте npm для використання каталогу користувача:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### Перевірте встановлення

Після встановлення переконайтеся, що OpenSpec встановлено правильно та версія відповідає вимогам:

```bash
openspec --version
```

У разі успішного встановлення ви побачите вихідний номер версії, наприклад:

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::note[Діапазон версій]
Переконайтеся, що встановлена версія OpenSpec 1.0.0 або новіша. Проект має перевірену сумісність з версією 1.1.1. Рекомендується використовувати останню сумісну версію, щоб отримати нові функції.
:::

**Рішення**:

### Проблеми з мережею

#### Проблема: встановлення npm не вдається або працює дуже повільно

Не вдалося завантажити OpenSpec із реєстру npm через мережеві причини.

**Рішення**:

1. **Використовуйте локальне дзеркало npm**:

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **Відновити офіційне джерело** (за потреби):

```bash
npm config set registry https://registry.npmjs.org
```

3. **Використовуйте cnpm** (необов’язково):

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

:::підказка
Користувачам у материковому Китаї рекомендується використовувати дзеркало Taobao, яке може значно підвищити швидкість завантаження.
:::

### Проблеми сумісності версій

#### Проблема: команди OpenSpec виконуються ненормально або повідомляють про помилки

Установлена версія OpenSpec несумісна з поточним середовищем.

**Рішення**:

1. **Перевірте версію Node.js**:

```bash
node --version
```

Переконайтеся, що версія 18.0 або вище. Якщо версія занизька, оновіть Node.js.

2. **Перевірити версію OpenSpec**:

```bash
openspec --version
```

3. **Перевстановіть конкретну версію**:

```bash
npm install -g @fission-ai/openspec@1
```

4. **Очистіть кеш npm** (якщо проблема не зникає):

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### Команда не знайдена

#### Проблема: набір `openspec` Команда показує "Команду не знайдено"

OpenSpec встановлено, але система не може знайти команду.

**Рішення**:

<Tabs>
  <TabItem value="win" label="Windows">
    1. Підтвердьте глобальний шлях встановлення npm:

```powershell
npm config get prefix
```

2. Додати шлях до системи PATH:
   - Відкрийте «Властивості системи» → «Додатково» → «Змінні середовища»
   - знайти `Path` в "Системні змінні"
   - Додайте глобальний шлях npm (зазвичай `C:\Users\<username>\AppData\Roaming\npm`)
   - Перезапустити вікно командного рядка
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. Підтвердьте глобальний шлях встановлення npm:

```bash
npm config get prefix
```

2. Додайте шлях до PATH (in `~/.zshrc` або `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Перезавантажити конфігурацію:

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. Підтвердьте глобальний шлях встановлення npm:

```bash
npm config get prefix
```

2. Додайте шлях до PATH (in `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Перезавантажити конфігурацію:

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
