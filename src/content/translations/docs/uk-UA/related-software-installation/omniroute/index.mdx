Ця сторінка призначена для нових користувачів HagiCode, яким потрібен найкоротший шлях налаштування OmniRoute: запустіть локальну службу, переконайтеся, що `http://localhost:20128/v1` працює, і переконайтеся, що інформаційна панель і API доступні.

:::tip[рекомендований шлях]
Ця перша версія зберігає найстабільніший шлях спочатку: `npm install -g omniroute`, `omniroute`, Приладова панель на `http://localhost:20128`, а також OpenAI-сумісну базову URL-адресу за адресою `http://localhost:20128/v1`. Docker, VPS і більш розширені режими розгортання залишаються в додатковому розділі.
:::

## передумови

- Node.js уже встановлено. Повний [Встановлення Node.js](/en/related-software-installation/nodejs/installation) спочатку, якщо потрібно.
- Місцевий порт `20128` вільний і ще не зайнятий.
- У вас є принаймні один обліковий запис постачальника моделі висхідного потоку, який ви плануєте маршрутизувати через OmniRoute.
- Перш ніж інтегрувати HagiCode, CLI або SDK, розділіть ці дві адреси:
  - Приладова панель: `http://localhost:20128`
  - Базова URL-адреса, сумісна з OpenAI: `http://localhost:20128/v1`

## Встановіть і запустіть локально

Почніть із рекомендованого глобального шляху npm:

```bash
npm install -g omniroute
omniroute
```

Після запуску служби перевірте її в такому порядку:

1. відкритий `http://localhost:20128` у веб-переглядачі та переконайтеся, що інформаційна панель завантажується.
2. Перейти до `Endpoints` на інформаційній панелі та створіть або скопіюйте ключ API.
3. Зверніть увагу на базову URL-адресу, яку ви передасте інструментам пізніше: `http://localhost:20128/v1`.

:::note[Чому починати з інформаційної панелі]
Для першого налаштування інформаційна панель є найшвидшим способом підтвердити, що OmniRoute запущено, ваші постачальники підключені та існує ключ API. Якщо пізніший виклик клієнта зазнає невдачі, спочатку перевірити інформаційну панель зазвичай швидше, ніж негайно налагодити код SDK.
:::

## Перевірте запущену службу

### Перевірка 1: переконайтеся, що каталог моделей читається

Поставте ключ від в `Endpoints` сторінку в змінну середовища:

```bash
export OMNIROUTE_API_KEY="replace-with-the-key-you-copied-from-the-Endpoints-page"
```

Тоді запросіть `/v1/models`:

```bash
curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}"
```

Якщо ви повертаєте список моделей, сумісна з OpenAI кінцева точка та потік авторизації працюють.

### Перевірка 2: надішліть мінімальний запит на чат

```bash
curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "replace-with-a-model-id-from-v1-models",
    "messages": [
      { "role": "user", "content": "Reply with ok." }
    ]
  }'
```

Метою тут є не швидка якість. Це доказ того, що обрану вами модель уже можна успішно маршрутизувати через OmniRoute.
