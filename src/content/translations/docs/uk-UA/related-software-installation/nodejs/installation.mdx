Цей посібник описує встановлення Node.js у Windows, macOS і Linux. Node.js є ключовою залежністю для локальних інструментів HagiCode, включаючи OpenSpec, інтерфейсні інструменти та кілька команд команди розробників.

:::tip[Вимоги до версії]
HagiCode рекомендує **Node.js 24+** (мінімально підтримуваний Node.js &gt;= 18.0). У більшості випадків інсталюйте останню версію LTS, а потім переконайтеся, що ваша версія відповідає вимогам.
:::

## Встановлення Node.js

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Спосіб перший: офіційний інсталятор (рекомендовано)

1. Відвідайте [Офіційний сайт Node.js](https://nodejs.org/)
2. Завантажте інсталятор Windows версії **LTS (Long Term Support)** (`.msi` файл)
3. Двічі клацніть, щоб запустити інсталятор
4. Дотримуйтесь підказок майстра встановлення, щоб завершити встановлення:
   - Прийміть ліцензійну угоду
   - Виберіть шлях встановлення (за замовчуванням добре)
   - Переконайтеся, що позначено «Автоматично встановлювати необхідні інструменти».
   - Натисніть «Встановити», щоб почати встановлення
5. Після завершення встановлення натисніть «Готово»

### Другий спосіб: використання крила

Відкрийте PowerShell або командний рядок і запустіть:

```powershell
winget install OpenJS.NodeJS.LTS
```

Після встановлення повторно відкрийте термінал, щоб зміни набули чинності.

  </TabItem>
  <TabItem value="macos" label="macOS">

### Спосіб перший: офіційний інсталятор (рекомендовано)

1. Відвідайте [Офіційний сайт Node.js](https://nodejs.org/)
2. Завантажте інсталятор версії **LTS (Long Term Support)** для macOS (`.pkg` файл)
3. Двічі клацніть, щоб запустити інсталятор
4. Дотримуйтесь підказок майстра встановлення, щоб завершити встановлення:
   - Натисніть «Продовжити»
   - Прийміть ліцензійну угоду
   - Виберіть інсталяційний диск
   - Натисніть «Встановити» і введіть пароль адміністратора
5. Після завершення встановлення натисніть «Закрити»

### Спосіб другий: використання Homebrew

Якщо у вас є [Домашнє пиво](https://brew.sh/) встановлено, ви можете встановити Node.js через термінал:

```bash
brew install node
```

:::підказка
Homebrew автоматично додасть Node.js до вашого ШЛЯХУ. Якщо команда недоступна, перезапустіть термінал або запустіть:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

### Використання репозиторію NodeSource (рекомендовано)

Це найкращий спосіб отримати останню версію Node.js:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Використання apt Package Manager

Якщо вам не потрібна остання версія, ви можете скористатися репозиторієм Ubuntu за замовчуванням:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::примітка
Версія Node.js у сховищі за замовчуванням може бути старішою. Після встановлення перевірте версію, щоб переконатися, що вона відповідає вимогам Node.js &gt;= 18.0 (рекомендовано Node.js 24+).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### Використання репозиторію NodeSource (рекомендовано)

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### Використання менеджера пакетів yum

```bash
sudo yum install -y nodejs npm
```

  </TabItem>
  <TabItem value="fedora" label="Fedora">

### Використання менеджера пакетів dnf

```bash
sudo dnf install -y nodejs npm
```

:::підказка
Репозиторій програмного забезпечення Fedora зазвичай містить новіші версії Node.js і може бути встановлений безпосередньо через dnf.
:::

## Використання інструментів керування версіями

Якщо вам потрібно перемикатися між кількома версіями Node.js, ви можете скористатися інструментами керування версіями.

### nvm (Менеджер версій вузла)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

Після встановлення перезавантажте конфігурацію оболонки:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### вікна

У Windows ми рекомендуємо використовувати [nvm-вікна](https://github.com/coreybutler/nvm-windows):

1. Завантажте останню версію `nvm-setup.exe` від [випуски nvm-windows](https://github.com/coreybutler/nvm-windows/releases)
2. Запустіть інсталятор
3. Знову відкрийте командний рядок або PowerShell

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

Після встановлення nvm ви можете використовувати його для встановлення та керування версіями Node.js:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (швидкий менеджер вузлів)

fnm — це швидший і простіший інструмент керування версіями Node.js, створений за допомогою Rust.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### вікна

використання [совок](https://scoop.sh/) або [крило](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Прискорення для користувачів материкового Китаю

Якщо ви перебуваєте в материковому Китаї, завантаження пакетів Node.js і npm може бути повільним. Ось кілька рішень для прискорення.

### Використання вітчизняних дзеркальних джерел

#### Дзеркало npm Taobao (рекомендовано)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### Використання nrm для керування джерелами дзеркал

nrm (менеджер реєстру npm) допомагає швидко перемикати джерела дзеркал npm:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Прискорення завантажень інсталятора Node.js

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao забезпечує дзеркальні завантаження для інсталяторів Node.js:

- **Адреса для завантаження**: https://npmmirror.com/mirrors/node/

Перейдіть за вказаною вище адресою та виберіть версію, яку потрібно завантажити.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

Якщо завантаження nvm або інших інструментів із GitHub відбувається повільно, ви можете скористатися дзеркалом FastGit:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::примітка
Служба FastGit може змінюватися, будь ласка, зверніться до офіційної інформації.
:::

  </TabItem>
</Tabs>

### Домашні домашні дзеркала

Якщо ви використовуєте Homebrew у материковому Китаї, ви можете налаштувати внутрішні дзеркала для прискорення:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Менеджери пакунків материкового Китаю

#### cnpm

cnpm — це клієнт npm, наданий командою Taobao, який за замовчуванням використовує дзеркало Taobao:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::підказка
cnpm здебільшого сумісний із командами npm і може використовуватися як пряма заміна.
:::

### Перевірка ефекту дзеркального прискорення

Після налаштування дзеркал перевірте:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Перевірка встановлення

Після встановлення переконайтеся, що Node.js і npm встановлено правильно:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::примітка
Переконайтеся, що версія Node.js &gt;= 18.0 (рекомендується Node.js 24+). Якщо версія надто низька, будь ласка, перевстановіть за допомогою наведених вище методів або скористайтеся інструментами керування версіями, щоб установити правильну версію.
:::

### Зразок результату перевірки

Результати успішної перевірки мають виглядати так:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Усунення несправностей

### Проблеми з Windows

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Якщо `node` або `npm` команди недоступні після встановлення:

1. Перезапустіть командний рядок або PowerShell
2. Якщо все ще недоступний, вручну додайте Node.js до PATH:
   - Знайдіть «Змінні середовища» та відкрийте «Редагувати системні змінні середовища»
   - Натисніть «Змінні середовища»
   - Знайдіть «Шлях» у «Системних змінних» і натисніть «Редагувати».
   - Додати шлях встановлення Node.js (за замовчуванням: `C:\Program Files\nodejs\`)
   - Натисніть «ОК», щоб зберегти зміни

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Якщо ви зіткнулися з помилками політики виконання під час використання nvm:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### Проблеми macOS

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Якщо ви зіткнулися з помилками дозволу під час встановлення глобальних пакетів npm:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Якщо команди недоступні після встановлення, перезапустіть термінал або вручну додайте до PATH (зазвичай у `/usr/local/bin`).

  </TabItem>
</Tabs>

### Проблеми з Linux

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Якщо під час інсталяції глобальних пакетів npm виникають помилки дозволів, рекомендується використовувати інструменти керування версіями Node.js (наприклад, nvm або fnm) замість менеджерів системних пакетів.

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Якщо версія Node.js, встановлена через системний менеджер пакунків, надто стара:

1. Використовувати репозиторій NodeSource (рекомендовано, див. вище)
2. Або скористайтеся інструментами керування версіями (nvm/fnm)

  </TabItem>
</Tabs>

### Перевірка середовища

Якщо підтвердження не вдається, перевірте наступне:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
