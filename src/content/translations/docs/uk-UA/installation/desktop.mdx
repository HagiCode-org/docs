Цей посібник описує встановлення та використання Hagicode Desktop. Це офіційний клієнт для настільного комп’ютера, який зберігає локальний шлях налаштування за умовчанням, а також щоденні елементи керування в одному місці.

:::tip[рекомендований метод]
Робочий стіл є локальним шляхом за умовчанням для окремих користувачів і розробників, оскільки:
- інсталятор зберігає потік за замовчуванням в одному керованому шляху
- залежностями та послугами можна керувати в додатку
- інтерфейс користувача полегшує перевірку стану та операцій
- вбудоване керування версіями полегшує перемикання
- дані залишаються локальними за замовчуванням
:::

## Що таке Hagicode Desktop?

Hagicode Desktop — це офіційна настільна програма Hagicode, розроблена для систем Windows, macOS і Linux. Він об’єднує основні функції Hagicode, а інсталятор зберігає шлях налаштування за замовчуванням в одному керованому потоці.

### Основні переваги

- **Встановлення одним клацанням миші**: виконайте всі кроки налаштування автоматично за допомогою майстра встановлення
- **Автоматична підготовка середовища виконання**: інсталятор за замовчуванням уже містить необхідне середовище виконання, тому вам не потрібно встановлювати окремий `.NET` спочатку час виконання
- **Керування службами**: вбудоване керування сервером Hagicode, легкий запуск і зупинка служб
- **Перемикання версій**: підтримка керування кількома версіями, вільне перемикання між різними версіями
- **Локальна робота**: усі дані зберігаються локально, захищаючи вашу конфіденційність
- **Автоматичні оновлення**: автоматично визначає та завантажує останню версію

### Застосовні сценарії

Робочий стіл добре підходить, коли:
- Індивідуальні розробники, які хочуть використовувати Hagicode локально
- Користувачі, яким потрібно використовувати Hagicode офлайн
- Нетехнічні користувачі, які хочуть спростити процес встановлення
- Розробники, яким потрібно часто перемикатися між різними версіями

### Інші методи завантаження

Ви також можете отримати програму встановлення Hagicode Desktop із таких каналів:

1. **Офіційна сторінка випуску**: відвідайте [Випуски Hagicode GitHub](https://github.com/Hagicode-org/releases) щоб завантажити останню версію
2. **Офіційний веб-сайт**: відвідайте [Веб-сайт Hagicode](https://hagicode.com/) щоб отримати посилання для завантаження

### Виберіть правильну версію

Виберіть відповідний інсталятор для вашої операційної системи:
- **Windows**: завантажте `.exe` інсталятор
- **macOS**: завантажте `.dmg` інсталятор
- **Linux**: завантажте `.AppImage` або `.tar.gz` пакет інсталятора

:::note[Системні вимоги]
- **Windows**: Windows 10 або новіша версія (64-розрядна)
- **macOS**: macOS 10.15 або вище
- **Linux**: основні дистрибутиви Linux (Ubuntu 20.04+, Fedora 33+ тощо)
:::

:::tip[За замовчуванням передумови]
Для останнього процесу локального встановлення HagiCode зазвичай не потрібно готувати окремий `.NET` час виконання. Desktop уже об’єднує середовище виконання, необхідне для потоку за замовчуванням, і за замовчуванням ініціалізує підтримуваний локальний шлях SQLite.
:::

## Примітки щодо встановлення та запуску

:::обережно[повідомлення macOS Gatekeeper]
Якщо ви вже встановили програму за замовчуванням `/Applications/Hagicode.app` але macOS показує «програма пошкоджена», «неможливо відкрити» або подібне попередження Gatekeeper під час першого запуску, звичайною причиною є `com.apple.quarantine` атрибут, який macOS додає до програм, завантажених з Інтернету, а не збій у самому інсталяторі Hagicode Desktop.

Виконайте наступну команду в терміналі. Він вимагає прав адміністратора та рекурсивно видаляє атрибут карантину з додатка за стандартним шляхом встановлення:

```bash
sudo xattr -dr com.apple.quarantine /Applications/Hagicode.app
```

Після завершення виконання команди перезапустіть Hagicode Desktop.
:::

## Запустіть робочий стіл

Після завантаження відкрийте інсталятор або пакет програми та дотримуйтесь інструкцій на екрані. Desktop готує середовище виконання в поточному підтримуваному локальному шляху, керує Hagicode Server і показує прогрес і статус, коли потрібні мережеві завантаження.

:::note[Вимоги до мережі]
Переконайтеся, що підключення до мережі доступне. Desktop може знадобитися отримати інформацію про версію або пакети компонентів з Інтернету; якщо завантаження не вдається, перевірте налаштування мережі, проксі-сервера чи брандмауера та повторіть спробу.
:::

## Керування версіями

Hagicode Desktop підтримує керування кількома версіями, що дозволяє вільно перемикатися між різними версіями.

![Керування версіями](../../img/screenshots/installation/install-desktop/12-version-management/original.png)

### Змінити версію

1. Відкрийте Hagicode Desktop
2. Перейдіть на сторінку «Керування версіями».
3. Виберіть версію, яку потрібно змінити
4. Натисніть кнопку «Змінити версію».
5. Дочекайтеся завершення перемикання

:::note[Примітки щодо перемикання версій]
- Перемикання версій призведе до автоматичного перезапуску сервера Hagicode
- Переконайтеся, що ви зберегли всю роботу перед перемиканням
- Конфігурації проекту можуть відрізнятися залежно від версії
:::

## Запуск і зупинка служби

Hagicode Desktop забезпечує простий спосіб керування службою Hagicode Server.

### Запустити службу

На домашній сторінці програми Desktop натисніть кнопку «Запустити службу», щоб запустити Hagicode Server.

![Запустити службу](../../img/screenshots/installation/install-desktop/13-start-service-homepage/original.png)

### Зупинити службу

Щоб зупинити сервер Hagicode, натисніть кнопку «Зупинити службу» на головній сторінці.

:::tip[Статус служби]
Програма Desktop відображає стан роботи Hagicode Server у режимі реального часу, зокрема:
- Статус служби (працює/зупинено)
- Номер порту служби
- Інформація про поточну версію
:::

## Усунення несправностей

### Помилка встановлення

Якщо у вас виникли проблеми під час встановлення:

1. **Перевірте підключення до мережі**: переконайтеся, що ви можете отримати доступ до Інтернету
2. **Перевірте місце на диску**: переконайтеся, що у вас достатньо місця на диску
3. **Запуск від імені адміністратора**: у Windows клацніть інсталятор правою кнопкою миші та виберіть «Запуск від імені адміністратора».
4. **Перегляд файлів журналу**: інсталятор створює докладні файли журналу, які можуть допомогти діагностувати проблеми

### Помилка встановлення залежності

Якщо встановлення залежності не вдається:

1. **Перевірте сумісність системи**: переконайтеся, що ваша операційна система відповідає мінімальним вимогам
2. **Перевірте параметри мережі та брандмауера**: переконайтеся, що інсталятор може отримати необхідні завантаження та не заблоковано брандмауером
3. **Підтвердьте дозволи інсталятора**: у Windows спробуйте запустити інсталятор від імені адміністратора, щоб він мав достатні дозволи
4. **Перезапустіть програму та повторіть спробу**: закрийте та знову відкрийте Hagicode Desktop, а потім повторіть крок встановлення
5. **Переглянути журнали**: переглянути докладні журнали помилок у програмі для робочого столу

### Помилка запуску служби

Якщо Hagicode Server не запускається:

1. **Перевірте зайнятість порту**: переконайтеся, що порт 45000 не зайнятий іншими програмами
2. **Перевірте налаштування брандмауера**: переконайтеся, що брандмауер дозволяє доступ до мережі Hagicode Server
3. **Перезапустіть програму**: спробуйте перезапустити Hagicode Desktop
4. **Перегляд журналів**: перегляд докладних журналів помилок у програмі для робочого столу
