## передумови

Перш ніж почати, переконайтеся, що у вашій системі встановлено Docker і Docker Compose.

### Встановіть Docker

#### вікна

Завантажте та встановіть [Робочий стіл Docker для Windows](https://www.docker.com/products/docker-desktop/)

Після встановлення переконайтеся, що Docker Desktop запущено.

#### macOS

Завантажте та встановіть [Робочий стіл Docker для Mac](https://www.docker.com/products/docker-desktop/)

Після встановлення переконайтеся, що Docker Desktop запущено.

#### Linux

Встановіть Docker за допомогою менеджера пакетів:

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

Після встановлення запустіть службу Docker:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Перевірте встановлення

Після інсталяції виконайте такі команди, щоб переконатися, що Docker і Docker Compose встановлено правильно:

```bash
docker --version
docker compose version
```

#### Вибір джерела зображення

Генератор підтримує три варіанти джерел зображення:

**Docker Hub**
- **Адреса зображення**: `newbe36524/hagicode:{tag}`
- **Застосовні сценарії**: підходить для користувачів із дзеркальним прискоренням Docker Hub
- **Переваги**: Офіційне джерело зображень, своєчасні оновлення, стабільний доступ
- **Примітки**: у деяких регіонах може знадобитися налаштувати прискорення дзеркала

**Реєстр контейнерів Azure (ACR)**
- **Адреса зображення**: `hagicode.azurecr.io/hagicode:{tag}`
- **Застосовні сценарії**: підходить для користувачів, які не мають локального доступу до Docker Hub
- **Переваги**: забезпечує альтернативне джерело зображення, вирішує проблеми доступу до мережі
- **Примітки**: зображення залишаються синхронізованими з Docker Hub, але можуть мати короткі затримки

**Alibaba Cloud Container Registry (ACR)**
- **Адреса зображення**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **Застосовні сценарії**: підходить для домашніх користувачів, стабільний і швидкий доступ до мережі
- **Переваги**: внутрішні вузли Alibaba Cloud, висока швидкість доступу, висока стабільність
- **Примітки**: зображення Alibaba ACR синхронізуються з Docker Hub зазвичай протягом 30 хвилин

:::tip[Рекомендації щодо вибору джерела зображення]
- **Внутрішні користувачі рекомендують Alibaba Cloud ACR**: якщо ви домашній користувач, радимо надати пріоритет використанню джерела зображень Alibaba Cloud ACR для швидкого та стабільного доступу
- **Альтернативний Docker Hub**: якщо ваше мережеве середовище має доступ до Docker Hub, ви також можете вибрати Docker Hub як джерело зображення
- **Альтернативний Azure ACR**: вибирайте лише джерело зображення Azure ACR, якщо інші джерела зображень мають труднощі з доступом
- **Синхронізація зображень**: зображення Alibaba Cloud ACR синхронізуються з Docker Hub, зазвичай протягом 30 хвилин
- **Узгодженість версій**: теги зображень є узгодженими для всіх трьох джерел зображень, не потрібно турбуватися про проблеми з версіями
:::

### 2. Запустіть Служби

Виконайте наступну команду в каталозі, де знаходиться ваш `docker-compose.yml` знаходиться:

```bash
docker compose up -d
```

Ця команда:
- Завантажте та запустіть контейнери Hagicode, визначені згенерованим шаблоном
- Створення та налаштування мережі
- Ініціалізувати каталог даних за замовчуванням і залежності середовища виконання

### 3. Перевірте статус служби

Перевірте, чи запущені контейнери:

```bash
docker compose ps
```

Ви повинні побачити служби, визначені у створеному шаблоні, у статусі «запущено».

### Перегляд журналів обслуговування

Щоб переглянути всі журнали обслуговування:

```bash
# View all service logs
docker compose logs

# View Hagicode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## Інструкції з налаштування

Якщо вам потрібно змінити конфігурацію (наприклад, змінити порти, змінити постачальника API тощо), ви можете:

1. Відновіть нову конфігурацію за допомогою [Docker Compose Builder (окремий сайт)](https://builder.hagicode.com/)
2. Редагувати вручну `docker-compose.yml` і `.env` файли
3. Перезапустіть служби, щоб конфігурація набула чинності:

```bash
docker compose restart
```
