## Чому посібник LLM

У робочих процесах генерації коду продуктивність моделі значно відрізняється за якістю, затримкою та вартістю. Ця категорія забезпечує послідовну структуру оцінювання, щоб допомогти розробникам швидше вибрати моделі.

## Рекомендований шлях для читання

1. Почніть з [Порівняння та оцінка моделей у HagiCode](./model-comparison-evaluation/).
2. Вибирайте кандидатів на основі вашого пріоритету (насамперед якість, насамперед вартість або швидкість).
3. Перевірте нотатки про час тестування, перш ніж застосовувати висновки для виробничого використання.

## Принципи обслуговування

- Оновлюйте висновки, коли можливості моделі істотно змінюються.
- Повторно перевіряйте контекст тестового часу принаймні раз на місяць.
- Тримайте китайські та англійські сторінки вирівняними в структурі розділу.
