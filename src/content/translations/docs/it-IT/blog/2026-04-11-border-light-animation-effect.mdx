## Guida alla creazione di un effetto di animazione di scansione della luce del bordo

> Come si costruisce quell'elemento importante che gli utenti notano a colpo d'occhio utilizzando CSS puro? In realtà non è così difficile. Il trucco sta nel prendere un percorso leggermente tortuoso. In questo articolo ti spiegherò come creare da zero un'animazione di illuminazione dei bordi e condividerò anche alcune delle insidie ​​​​in cui ci siamo imbattuti durante la creazione di HagiCode.

## Sfondo

Se lavori sul frontend, probabilmente hai già avuto questa esperienza: un product manager si avvicina con quell'espressione "questo è decisamente semplice" e dice: "Possiamo aggiungere qualche tipo di effetto speciale a questa attività in esecuzione in modo che gli utenti possano individuarla immediatamente?"

Dici certo, possiamo semplicemente cambiare il colore del bordo. Poi scuotono la testa con quello sguardo che dice: "Non capisci". Rispondono: "Non è abbastanza ovvio. Voglio il tipo di effetto in cui la luce corre attorno al confine, come in un film di fantascienza".

A quel punto potresti iniziare a chiederti come costruirlo. Tela? SVG? Oppure i CSS possono gestirlo da soli? Dopotutto, nessuno vuole ammettere di non sapere come farlo.

Nelle moderne applicazioni web, le animazioni di luce di bordo sono in realtà molto comuni. Vengono utilizzati principalmente in alcuni scenari come questi:

- **Indicatori di stato**: contrassegnare le attività in corso o gli elementi attivi
- **Focus visivo**: evidenzia aree di contenuto importanti
- **Miglioramento del marchio**: creazione di uno stile visivo elegante, moderno e tecnologico
- **Temi stagionali**: creare un'atmosfera celebrativa per le occasioni speciali

Ci siamo imbattuti esattamente in questo requisito durante la costruzione [HagiCode](https://hagicode.com). Gli utenti avevano bisogno di vedere a colpo d'occhio quali sessioni erano in corso e quali proposte erano attualmente in fase di elaborazione. Abbiamo provato diversi approcci. Alcuni sentieri erano più lisci, altri un po’ più tortuosi. Alla fine, abbiamo optato per una strategia di implementazione abbastanza matura.

## Idee di implementazione fondamentali

Dopo aver analizzato la base di codice HagiCode, abbiamo riassunto di seguito diversi modelli di implementazione principali. Ognuno si adatta a uno scenario diverso o, in altre parole, ognuno esiste per una ragione.

### 1. Bagliore rotante con gradiente conico (il più comune)

Questo è il modo classico per implementare un effetto di spazzata di luce sui bordi. L'idea centrale è usare i CSS `conic-gradient` per creare un gradiente conico, quindi ruotarlo continuamente. Come un lampione che si accende nella notte, continua a girare.

**Elementi chiave:**
- Usa il `::before` pseudo-elemento per creare il livello bagliore
- Utilizzare `conic-gradient` per definire la distribuzione del colore del gradiente
- Usa il `::after` pseudo-elemento per mascherare l'area centrale (opzionale)
- Utilizzare `@keyframes` per implementare l'animazione di rotazione

### 2. Linea luminosa laterale

Funziona bene per gli indicatori di stato negli elementi dell'elenco. Crei una sottile linea luminosa su un lato dell'elemento invece di animare l'intero bordo. A volte basta un po' di luce. Non è necessario illuminare il mondo intero.

**Elementi chiave:**
- Un sottile elemento di linea posizionato in modo assoluto
- Utilizzare `box-shadow` per creare l'effetto bagliore
- Utilizzare `scale` e `opacity` per un'animazione respiratoria

### 3. Sfondo luminoso con scatola-ombra

Se non hai bisogno dell'effetto di scansione completo e desideri solo un bagliore di sfondo morbido, stratifica più `box-shadow` i valori sono sufficienti. A volte l’opzione più semplice è quella migliore.

### 4. Supporto per l'accessibilità

Questa parte è facile da trascurare, ma è estremamente importante. Ogni animazione dovrebbe tenere conto di `prefers-reduced-motion` media query e fornire un'alternativa statica per gli utenti che non desiderano l'animazione. Non a tutti piace il movimento costante e rispettare questa preferenza è importante.

## Opzioni di implementazione

### Opzione 1: rotazione del bordo con gradiente conico (consigliata)

Questa è l'implementazione più completa dell'effetto di luce del bordo ampio ed è anche l'opzione che utilizziamo più spesso in HagiCode. Dopotutto, se qualcosa funziona bene, perché sostituirlo?

```css
/* Parent container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotating glow layer */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Mask layer (optional, for creating a hollow border effect) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Il principio alla base di questa opzione è abbastanza semplice: crea uno pseudo-elemento più grande del contenitore principale, disegna su di esso un gradiente conico e ruotalo continuamente. Il contenitore principale utilizza `overflow: hidden`, quindi rimane visibile solo la luce che passa attorno al bordo. È un po’ come guardare un lampione attraverso una finestra. Si vede sempre e solo la piccola fetta che passa.

### Opzione 2: bordo luminoso rotante semplificato

Se non hai bisogno dell'effetto completo, HagiCode include anche una versione più leggera di classe utility. A volte l’approccio più semplice è davvero migliore.

```css
/* Rotating light border utility class */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Accessibility support */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Notare il `will-change: transform` qui. Dice al browser: "Questo elemento continuerà a cambiare", in modo che il browser possa preparare alcune ottimizzazioni in anticipo e mantenere l'animazione più fluida. Prepararsi in anticipo è solitamente meglio che affrettarsi all’ultimo minuto.

### Opzione 3: linea luminosa laterale

Ciò è particolarmente adatto per gli indicatori di stato degli elementi dell'elenco ed è esattamente ciò che utilizza l'elenco delle sessioni HagiCode. Una linea sottile può ancora risaltare tra molti elementi. Sembra una lezione di vita a modo suo.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Questo usa `isolation: isolate` per creare un nuovo contesto di impilamento, quindi si basa su `z-index` per controllare l'ordine di visualizzazione di ciascun livello. `pointer-events: none` è anche essenziale. Altrimenti lo pseudo-elemento bloccherebbe i clic dell'utente. Alcune cose possono sembrare belle, ma non dovrebbero comunque intralciarti.

### Opzione 4: avvolgilo in un componente React

Se il tuo progetto utilizza React, puoi racchiudere questa logica in un componente, in particolare la gestione dell'accessibilità. Scrivilo una volta, usalo tante volte. Questo è il punto.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Il modulo CSS corrispondente:

```css
/* GlowBorder.module.css */

/* Animated version */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Static version (accessibility) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Il `useReducedMotion` gancio da `framer-motion` rileva automaticamente le preferenze di sistema dell'utente. Se l'utente ha abilitato il movimento ridotto, ritorna `true`e il componente mostra invece la versione statica. Rispettare le preferenze dell'utente è più importante che forzare un effetto appariscente.

## Lezioni pratiche apprese

Queste sono alcune delle lezioni che abbiamo imparato durante la creazione di HagiCode. Potresti anche chiamarle cicatrici di battaglia. Speriamo che ti aiutino a evitare alcune deviazioni.

### 1. Sistema di variabili tematiche

Le variabili CSS rendono il supporto multitema particolarmente conveniente. Nessuno vuole modificare una pila di codice ogni volta che cambia il tema.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Usage */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

In questo modo, cambiare tema richiede solo la modifica della classe sul file `html` elemento e ogni colore di animazione si aggiorna automaticamente. Una base di codice, due stili. Questo è esattamente ciò che vogliamo.

### 2. Ottimizzazione delle prestazioni

**Utilizzare `will-change` per suggerire al browser di ottimizzare:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Informalo in anticipo sul browser e ti aiuterà a ottimizzare. Molte cose nella vita funzionano meglio con un po’ di preparazione.

**Evita di utilizzare box-ombre complesse su elementi di grandi dimensioni:**

```css
/* Not ideal - using a blurred shadow on a large element */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Better - use a pseudo-element to limit the glowing area */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Lo abbiamo testato in HagiCode. L'aggiunta di un'ombra sfocata direttamente a una scheda di grandi dimensioni ha ridotto il frame rate di scorrimento al di sotto dei 30 fps. Il passaggio a uno pseudo-elemento ha riportato le cose a 60 fps stabili. Gli utenti possono assolutamente sentire questa differenza.

### 3. Accessibilità

Non dovresti davvero saltarlo. Alcuni utenti trovano l'animazione vertiginosa o fonte di distrazione e rispettare le loro preferenze fa parte della costruzione di un buon prodotto. Le cose belle non devono essere imposte a tutti.

**Query multimediale CSS:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Provide a static fallback */
    opacity: 1;
  }
}
```

**Rileva le preferenze dell'utente in React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Controllo del livello di intensità

L'indicatore di throughput del token in HagiCode mostra diversi colori luminosi in base al throughput in tempo reale e questo viene implementato dinamicamente. Stati diversi dovrebbero essere espressi in modo diverso.

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Cose a cui prestare attenzione

Ci sono ancora alcuni dettagli a cui vale la pena prestare attenzione, perché quando scopri questi problemi nel modo più duro, è già troppo tardi.

| Cose a cui prestare attenzione | Spiegazione |
|---------|------|
| **Gestione z-index** | Lo strato luminoso dovrebbe utilizzare un appropriato `z-index` quindi non interferisce con l'interazione dei contenuti |
| **eventi-puntatore** | Lo pseudo-elemento bagliore dovrebbe essere impostato `pointer-events: none` |
| **Eccesione del confine** | Il contenitore principale ha bisogno `overflow: hidden`oppure è necessario regolare il dimensionamento degli pseudo-elementi |
| **Impatto sulle prestazioni** | Le animazioni complesse possono compromettere le prestazioni sui dispositivi mobili, quindi provale attentamente |
| **Modalità oscura** | Assicurati che il colore luminoso rimanga chiaramente visibile su sfondi scuri |
| **Cambio tema** | Utilizza le variabili CSS in modo che i colori dell'animazione si aggiornino correttamente quando il tema cambia |

### 6. Suggerimenti per il debug

Gli pseudo-elementi possono essere un po' difficili da individuare negli strumenti per sviluppatori, quindi puoi aggiungere temporaneamente un bordo per verificarne la posizione.

```css
/* Temporarily show pseudo-element boundaries for debugging */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Dopo aver finito di posizionarlo, ricordati di commentare o rimuovere quella riga. Altrimenti la produzione può diventare complicata abbastanza rapidamente. Alcune cose è meglio lasciarle in fase di sviluppo.
