Se vuoi installare Hermes e iniziare a usarlo, il percorso più breve è in realtà di soli tre passaggi:

1. Esegui il comando di installazione ufficiale
2. Avvia la CLI nel tuo terminale con `hermes`
3. Se vuoi continuare a usarlo in Feishu, configura `hermes gateway setup`

Questo articolo non tenta di spiegare tutte le funzionalità di Hermes tutte in una volta. Invece, ti aiuta a completare prima il ciclo per principianti più importante: installalo, mettilo in esecuzione, inizia a usarlo e quindi collegalo a uno degli scenari di piattaforma di messaggistica più comuni.

## Cos'è Hermes e a chi è rivolto

Hermes Agent è un agente AI che puoi utilizzare da un terminale locale o tramite un gateway della piattaforma di messaggistica.

Per la maggior parte degli sviluppatori, ha due punti di ingresso comuni:

- **CLI**: digitare `hermes` nel tuo terminale per accedere direttamente all'interfaccia interattiva.
- **Gateway di messaggistica**: esegui `hermes gateway`, quindi chatta con esso da piattaforme come Feishu, Telegram, Discord e Slack.

Se il tuo obiettivo in questo momento è semplicemente iniziare rapidamente, non invertire l'ordine. Inizia invece con questo percorso:

- Installa prima Hermes
- Verificare prima che funzioni dalla CLI
- Quindi decidi se desideri connettere una piattaforma di messaggistica

Ciò rende i problemi più facili da diagnosticare ed è più adatto alle persone che usano Hermes per la prima volta.

## Cosa sapere prima di installare Hermes

Secondo il README di Hermes, il percorso ufficiale di installazione rapida supporta questi ambienti:

- Linux
- macOS
- WSL2
- Android tramite Termux

### Note per gli utenti Windows

Hermes attualmente non supporta l'esecuzione diretta su Windows nativo. Se utilizzi Windows, l'approccio consigliato consiste nell'installare prima WSL2 e quindi eseguire il comando di installazione all'interno di WSL2.

È meglio chiarirlo fin dall'inizio, poiché molti errori di installazione non sono causati dal comando stesso, ma dall'utilizzo di un ambiente runtime non supportato.

## Come installare Hermes

Il comando di installazione rapida fornito nel README di Hermes è:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Questo comando esegue lo script di installazione ufficiale e gestisce i passaggi di inizializzazione specifici della piattaforma.

### Ricarica la shell dopo l'installazione

Una volta terminata l'installazione, ricaricare prima l'ambiente shell. Il comando più comune è:

```bash
source ~/.bashrc
```

Se usi zsh, puoi usare:

```bash
source ~/.zshrc
```

### Come verificare che Hermes sia installato correttamente

Il modo più diretto per verificare è eseguire:

```bash
hermes
```

Se desideri ulteriore conferma che la configurazione e le dipendenze funzionino, puoi anche eseguire:

```bash
hermes doctor
```

`hermes doctor` è particolarmente utile in queste situazioni:

- Il comando si comporta in modo anomalo dopo l'installazione
- La configurazione del modello non riesce
- Il gateway non si avvia
- Non sei sicuro che le dipendenze del tuo ambiente siano complete

## Come iniziare a utilizzare Hermes per la prima volta

### Inizia con la CLI

Se vuoi solo verificare il più rapidamente possibile che Hermes funzioni, il metodo più semplice è:

```bash
hermes
```

Questo avvia la CLI interattiva di Hermes. Per gli utenti Hermes alle prime armi, è anche il punto di partenza più consigliato, perché puoi verificare prima le cose più essenziali:

- Se il comando è effettivamente disponibile
- Se la configurazione del modello corrente funziona correttamente
- Se la toolchain del terminale funziona correttamente
- Se lo stile di interazione corrisponde a ciò di cui hai bisogno

### Questi comandi sono sufficienti per il primo ciclo di configurazione

Il README di Hermes elenca diversi comandi ad alta frequenza e insieme formano un percorso pratico di primo utilizzo:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Se non sei sicuro di cosa fanno ciascuno, ricordali in questo modo:

- `hermes model`: scegli o cambia modello
- `hermes tools`: visualizza e configura gli strumenti attualmente disponibili
- `hermes config set`: modifica elementi di configurazione specifici
- `hermes setup`: esegue la procedura guidata di inizializzazione completa una volta
- `hermes update`: aggiorna Hermes
- `hermes doctor`: risolvere i problemi

Per i principianti, l’ordine più pratico è solitamente:

1. Corri `hermes model` prima
2. Se vuoi configurare tutte le opzioni comuni contemporaneamente, esegui `hermes setup`

## I due modi più comuni per utilizzare Hermes

### 1. Usa Hermes nel terminale come assistente quotidiano allo sviluppo

La modalità CLI è adatta a questi scenari:

- Poni domande direttamente mentre scrivi il codice localmente
- Ispeziona progetti, modifica file ed esegui comandi
- Esegui il debugging o il lavoro di revisione una tantum
- Collabora continuamente nella directory di lavoro corrente

Il suo più grande vantaggio è che è il percorso più breve: nessuna integrazione aggiuntiva della piattaforma, nessuna configurazione di bot da gestire in anticipo ed è il modo migliore per creare la tua prima serie di abitudini di utilizzo.

### 2. Usa Hermes attraverso una piattaforma di messaggistica

Se vuoi chattare con Hermes su piattaforme come Feishu, Telegram o Discord, devi utilizzare il gateway di messaggistica.

I comandi di immissione più comuni sono:

```bash
hermes gateway setup
hermes gateway
```

Nello specifico:

- `hermes gateway setup` viene utilizzato per la configurazione della piattaforma interattiva
- `hermes gateway` viene utilizzato per avviare il processo del gateway

Secondo la documentazione ufficiale, il gateway è un processo in background unificato che collega le piattaforme configurate, gestisce le sessioni e gestisce funzionalità come i lavori cron.

## Usando Feishu come esempio: come connettere Hermes a una piattaforma di messaggistica

Se la maggior parte del tuo lavoro quotidiano si svolge a Feishu, allora Feishu/Lark è un modo molto naturale di usare Hermes.

### Il percorso minimo di integrazione praticabile

La documentazione ufficiale raccomanda questo comando di ingresso per Feishu/Lark:

```bash
hermes gateway setup
```

Dopo averlo eseguito, scegli semplicemente **Feishu / Lark** nella procedura guidata.

La documentazione Feishu descrive due modalità di connessione:

- `websocket`: consigliato
- `webhook`: facoltativo

Se Hermes viene eseguito sul tuo laptop, workstation o server privato, utilizzando `websocket` il primo è in genere più semplice perché non è necessario esporre un URL di richiamata pubblico.

### Se lo configuri manualmente, conosci almeno queste variabili

Se non stai utilizzando la procedura guidata e stai scrivendo la configurazione manualmente, la documentazione di Feishu elenca queste variabili principali:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Due di loro meritano un'attenzione particolare:

- `FEISHU_ALLOWED_USERS`: consigliato, quindi non tutti coloro che riescono a raggiungere il bot possono usarlo direttamente
- `FEISHU_HOME_CHANNEL`: ti consente di predefinire una chat domestica per ricevere risultati cron o notifiche predefinite

### Perché Hermes a volte non risponde nelle chat di gruppo di Feishu

È facile non notare questo dettaglio: nelle chat di gruppo Feishu, Hermes non risponde a tutti i messaggi per impostazione predefinita.

La documentazione ufficiale afferma chiaramente:

- Nei messaggi diretti, Hermes risponde ai messaggi
- Nelle chat di gruppo, devi esplicitamente `@` il bot prima che elabori il messaggio

Se vuoi impostare una conversazione Feishu come canale home, puoi anche usarla nella chat:

```text
/set-home
```

Oppure definirlo in anticipo nella configurazione:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## I comandi Hermes che i principianti dovrebbero ricordare per primi

Sia che utilizzi Hermes nella CLI o su una piattaforma di messaggistica, ricordare i seguenti comandi è già sufficiente per iniziare:

- `/new` o `/reset`: inizia una nuova sessione
- `/model`: visualizza o cambia il modello
- `/retry`: riprova il turno precedente
- `/undo`: annulla l'interazione precedente
- `/compress`: comprime manualmente il contesto
- `/help`: visualizza la guida

Se utilizzi Hermes principalmente su una piattaforma di messaggistica, ricordane un'altra:

- `/sethome` o `/set-home`: imposta la chat corrente come canale home

Questi comandi coprono le operazioni più comuni della fase principiante: riavvio, regolazione, rollback, controllo e continuazione.

## Domande frequenti

### Windows può installare direttamente Hermes?

No. L'attuale documentazione ufficiale afferma chiaramente che Windows nativo non è supportato e che si consiglia WSL2.

### Cosa devo fare se digito `hermes` non fa nulla dopo l'installazione?

È meglio risolvere i problemi in questo ordine:

1. Ricarica prima la shell, ad esempio con `source ~/.bashrc`
2. Corri `hermes` di nuovo
3. Se è ancora anormale, scappa `hermes doctor`

### Perché il bot non risponde in un gruppo Feishu?

Controlla prima queste tre cose:

- Sia tu `@` ha menzionato Hermes nel gruppo
- Sia `FEISHU_ALLOWED_USERS` limita l'utente corrente
- Se l'attuale politica della chat di gruppo consente la gestione dei messaggi di gruppo

Secondo la documentazione ufficiale di Feishu, negli scenari di chat di gruppo è richiesto l'utilizzo esplicito di una @menzione.
