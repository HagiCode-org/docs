## Pratica di integrazione degli agenti Hermes: dal protocollo alla produzione

> Condivisione dell'esperienza HagiCode completa relativa all'integrazione di Hermes Agent, comprese le lezioni fondamentali sull'adattamento del protocollo ACP, sulla gestione del pool di sessioni e sulla sincronizzazione dei contratti front-end/back-end.

## Sfondo

Durante la creazione di HagiCode, una piattaforma di codifica assistita dall'intelligenza artificiale, il nostro team aveva bisogno di integrare un framework di agenti che potesse essere eseguito localmente e scalabile anche nel cloud. Dopo la ricerca, Hermes Agent di Nous Research è stato scelto come motore sottostante per le nostre funzionalità di agente generico.

In verità, la selezione della tecnologia non è né particolarmente difficile né particolarmente facile. Sul mercato sono disponibili numerosi framework di agenti robusti, ma Hermes si è distinto perché il suo protocollo ACP e il suo sistema di strumenti si adattano particolarmente bene ai requisiti esigenti di HagiCode: sviluppo locale, collaborazione di gruppo ed espansione del cloud. Tuttavia, inserire Hermes in un sistema di produzione reale significava risolvere un lungo elenco di problemi ingegneristici. Questa parte era tutt'altro che banale.

Lo stack di HagiCode utilizza Orleans per costruire un sistema distribuito, mentre il front-end è costruito con React + TypeScript. Integrare Hermes significava preservare la coerenza architettonica e allo stesso tempo rendere Hermes un esecutore di prima classe insieme a ClaudeCode e OpenCode. Sembra abbastanza semplice, ma l’implementazione racconta sempre la vera storia.

Questo articolo condivide la nostra esperienza pratica nell'integrazione di Hermes Agent in HagiCode e speriamo che offra materiale di riferimento utile per i team che affrontano esigenze simili. Dopotutto, una volta che sei caduto in una fossa, non c'è motivo di lasciare che qualcun altro cada nella stessa.

## Progettazione dell'architettura

### Approccio progettuale a strati

L'integrazione Hermes di HagiCode utilizza un'architettura chiara a più livelli, in cui ogni livello è concentrato sulle proprie responsabilità:

**Strato centrale back-end**
- `HermesCliProvider`: implementa il `IAIProvider` interfaccia come punto di ingresso unificato del fornitore di intelligenza artificiale
- `HermesPlatformConfiguration`: gestisce il percorso dell'eseguibile Hermes, gli argomenti, l'autenticazione e le relative impostazioni
- `ICliProvider<HermesOptions>`: l'astrazione CLI di basso livello fornita da HagiCode.Libs per la gestione dei cicli di vita dei sottoprocessi

**Livello di trasporto**
- `StdioAcpTransport`: comunica con il sottoprocesso Hermes ACP tramite input e output standard
- Metodi del protocollo ACP: `initialize`, `authenticate`, `session/new`, `session/prompt`

**Livello runtime**
- `HermesGrain`: Implementazione di Orleans Grain che gestisce l'esecuzione di sessioni distribuite
- `CliAcpSessionPool`: pool di sessioni che riutilizza i sottoprocessi ACP per evitare frequenti sovraccarichi di avvio

**Livello front-end**
- `ExecutorAvatar`: Identità visiva e icona di Hermes
- `executorTypeAdapter`: logica di mappatura del tipo di provider
- Messaggistica in tempo reale SignalR: mantiene la coerenza dell'identità Hermes in tutto il flusso di messaggi

Questo design a strati consente a ogni strato di evolversi in modo indipendente. Ad esempio, se in futuro vogliamo aggiungere un nuovo meccanismo di trasporto, come WebSocket, dobbiamo solo modificare il livello di trasporto. Non è necessario ribaltare l'intero sistema solo perché cambia un trasporto.

### Astrazione dell'interfaccia unificata

Tutti i fornitori di intelligenza artificiale implementano il `IAIProvider` interfaccia, che è una delle scelte progettuali fondamentali nell'architettura di HagiCode:

```csharp
public interface IAIProvider
{
    string Name { get; }
    ProviderCapabilities Capabilities { get; }

    IAsyncEnumerable<AIStreamingChunk> StreamAsync(
        AIRequest request,
        CancellationToken cancellationToken = default);

    Task<AIResponse> ExecuteAsync(
        AIRequest request,
        CancellationToken cancellationToken = default);
}
```

`HermesCliProvider` implementa questa interfaccia e si trova su un piano di parità con `ClaudeCodeProvider`, `OpenCodeProvider`e altri. I vantaggi di questo design includono:

1. **Sostituibilità**: il cambio di fornitore non influisce sulla logica aziendale di livello superiore
2. **Testabilità**: i fornitori possono essere facilmente derisi per i test unitari
3. **Estensibilità**: l'aggiunta di un nuovo Provider richiede solo l'implementazione dell'interfaccia

Alla fine, le interfacce sono molto simili alle regole. Una volta stabilite le regole, tutti possono coesistere armoniosamente, sfruttare i propri punti di forza ed evitare di calpestarsi a vicenda. C'è una certa eleganza in questo.

## Implementazione principale

### Implementazione del livello provider

`HermesCliProvider` è il fulcro dell'intera integrazione. Coordina i vari componenti necessari per completare una singola invocazione dell'AI:

```csharp
public sealed class HermesCliProvider : IAIProvider, IVersionedAIProvider
{
    private readonly ICliProvider<LibsHermesOptions> _provider;
    private readonly ConcurrentDictionary<string, string> _sessionBindings;

    public ProviderCapabilities Capabilities { get; } = new()
    {
        SupportsStreaming = true,
        SupportsTools = true,
        SupportsSystemMessages = true,
        SupportsArtifacts = false
    };

    public async IAsyncEnumerable<AIStreamingChunk> StreamAsync(
        AIRequest request,
        [EnumeratorCancellation] CancellationToken cancellationToken = default)
    {
        // 1. Resolve the session binding key
        var bindingKey = ResolveBindingKey(request.CessionId);

        // 2. Get or create a Hermes session through the session pool
        var options = new HermesOptions
        {
            ExecutablePath = _platformConfiguration.ExecutablePath,
            Arguments = _platformConfiguration.Arguments,
            SessionId = _sessionBindings.TryGetValue(bindingKey, out var sessionId) ? sessionId : null,
            WorkingDirectory = request.WorkingDirectory,
            Model = request.Model
        };

        // 3. Execute and collect the streaming response
        await foreach (var message in _provider.ExecuteAsync(options, request.Prompt, cancellationToken))
        {
            // 4. Map ACP messages to AIStreamingChunk
            if (_responseMapper.TryConvertToStreamingChunk(message, out var chunk))
            {
                yield return chunk;
            }
        }
    }
}
```

Diversi punti di progettazione sono particolarmente importanti qui:

1. **Associazione di sessione**: utilizza `CessionId` per associare più richieste allo stesso sottoprocesso Hermes, preservando la continuità del contesto nelle conversazioni a più turni
2. **Mappatura della risposta**: converte il formato del messaggio Hermes ACP nel formato unificato `AIStreamingChunk` formato
3. **Supporto streaming**: utilizza `IAsyncEnumerable` per supportare risposte di streaming reali

Il legame tra sessioni è un po’ come le relazioni umane. Una volta stabilita la connessione, la comunicazione futura avrà un contesto, quindi non è necessario ricominciare da zero ogni volta. Naturalmente, quel rapporto deve ancora essere mantenuto.

### Adattamento del protocollo ACP

Hermes utilizza ACP (Agent Communication Protocol), che differisce da una tradizionale API HTTP. ACP è un protocollo basato su input e output standard e presenta diverse caratteristiche:

1. **Marcatore di avvio**: dopo l'avvio del processo Hermes, restituisce il file `//ready` marcatore
2. **Autenticazione dinamica**: i metodi di autenticazione non sono fissi e devono essere negoziati tramite il protocollo
3. **Riutilizzo della sessione**: le sessioni stabilite vengono riutilizzate `SessionId`
4. **Risposte frammentate**: una risposta completa può essere suddivisa in più risposte `session/update` notifiche

HagiCode gestisce queste caratteristiche `StdioAcpTransport`:

```csharp
public class StdioAcpTransport
{
    public async Task InitializeAsync(CancellationToken cancellationToken)
    {
        // Wait for the //ready marker
        var readyLine = await _outputReader.ReadLineAsync(cancellationToken);
        if (readyLine != "//ready")
        {
            throw new InvalidOperationException("Hermes did not send ready signal");
        }

        // Send the initialize request
        await SendRequestAsync(new
        {
            jsonrpc = "2.0",
            id = 1,
            method = "initialize",
            @params = new
            {
                protocolVersion = "2024-11-05",
                capabilities = new { },
                clientInfo = new { name = "HagiCode", version = "1.0.0" }
            }
        }, cancellationToken);
    }
}
```

I protocolli sono un po’ come la comprensione reciproca tra le persone. Una volta raggiunta questa comprensione, la comunicazione scorre molto più agevolmente. Costruirlo richiede solo tempo.

### Gestione del pool di sessioni

Avviare frequentemente i sottoprocessi Hermes è costoso, quindi abbiamo implementato un meccanismo di pool di sessioni:

```csharp
services.AddSingleton(static _ =>
{
    var registry = new CliProviderPoolConfigurationRegistry();
    registry.Register("hermes", new CliPoolSettings
    {
        MaxActiveSessions = 50,
        IdleTimeout = TimeSpan.FromMinutes(10)
    });
    return registry;
});
```

Parametri chiave del pool di sessioni:
- `MaxActiveSessions`: controlla il limite di concorrenza per evitare l'esaurimento delle risorse
- `IdleTimeout`: timeout di inattività che bilancia i costi di avvio con l'utilizzo della memoria

In pratica abbiamo riscontrato che:
1. Se il timeout di inattività è troppo breve, le sessioni si riavviano frequentemente; se è troppo lungo la memoria rimane occupata
2. Il limite di concorrenza deve essere regolato in base al carico effettivo, poiché impostarlo su un valore troppo elevato può rallentare il sistema
3. L'utilizzo del pool di sessioni deve essere monitorato in modo che i parametri possano essere modificati nel tempo

Questo è molto simile a molte scelte della vita: essere troppo aggressivi crea problemi, mentre essere troppo conservatori fa perdere opportunità. L’obiettivo è semplicemente trovare il giusto equilibrio.

### Sincronizzazione del contratto

Il front-end e il back-end mantengono allineati i loro contratti attraverso la generazione di OpenAPI. Il back-end definisce il file `AIProviderType` enumerazione:

```csharp
public enum AIProviderType
{
    Unknown,
    ClaudeCode,
    OpenCode,
    HermesCli  // Newly added
}
```

Il front-end genera il tipo TypeScript corrispondente tramite OpenAPI, garantendo che i valori enum rimangano coerenti. Questa è la chiave per evitare la visualizzazione del front-end `Unknown`.

Un contratto è molto simile a una promessa. Una volta concordato, deve essere onorato, altrimenti ti ritrovi in situazioni imbarazzanti come `Unknown`.

## Gestione della configurazione

La configurazione di Hermes è gestita tramite `appsettings.json`:

```json
{
  "Providers": {
    "HermesCli": {
      "ExecutablePath": "hermes",
      "Arguments": "acp",
      "StartupTimeoutMs": 10000,
      "ClientName": "HagiCode",
      "Authentication": {
        "PreferredMethodId": "api-key",
        "MethodInfo": {
          "api-key": "your-api-key-here"
        }
      },
      "SessionDefaults": {
        "Model": "claude-sonnet-4-20250514",
        "ModeId": "default"
      }
    }
  }
}
```

Questo design basato sulla configurazione offre flessibilità:
- i percorsi eseguibili possono essere sovrascritti, il che è utile per lo sviluppo e il test
- gli argomenti di avvio possono essere personalizzati per adattarsi alle diverse versioni di Hermes
- le informazioni di autenticazione possono essere configurate per supportare più metodi di autenticazione

La configurazione è un po’ come le domande a scelta multipla nella vita. Se sono disponibili abbastanza opzioni, di solito ce n'è una adatta. Detto questo, troppe opzioni possono creare affaticamento decisionale.

## Esperienza pratica

### Controlli sanitari

Costruire un fornitore affidabile richiede controlli sanitari completi:

```csharp
public async Task<ProviderTestResult> PingAsync(CancellationToken cancellationToken = default)
{
    var response = await ExecuteAsync(new AIRequest
    {
        Prompt = "Reply with exactly PONG.",
        CessionId = null,
        AllowedTools = Array.Empty<string>(),
        WorkingDirectory = ResolveWorkingDirectory(null)
    }, cancellationToken);

    var success = string.Equals(response.Content.Trim(), "PONG", StringComparison.OrdinalIgnoreCase);
    return new ProviderTestResult
    {
        ProviderName = Name,
        Success = success,
        ResponseTimeMs = stopwatch.ElapsedMilliseconds,
        ErrorMessage = success ? null : $"Unexpected Hermes ping response: '{response.Content}'."
    };
}
```

Punti da tenere d’occhio nei controlli sanitari:
1. Utilizzare casi di test semplici ed evitare scenari eccessivamente complessi
2. Impostare valori di timeout ragionevoli
3. Registrare i tempi di risposta per supportare l'analisi delle prestazioni

Proprio come le persone hanno bisogno di controlli fisici, anche i sistemi hanno bisogno di controlli sanitari. Prima vengono individuati i problemi, più facile sarà risolverli.

### Gestire problemi comuni

**Autenticazione fallita**
- Controlla se `Authentication.PreferredMethodId` corrisponde al metodo di autenticazione effettivamente supportato da Hermes
- Conferma che il formato delle informazioni di autenticazione sia corretto, ad esempio chiave API o token di connessione

**Timeout della sessione**
- Aumenta il `StartupTimeoutMs` valore
- Controlla la raggiungibilità del server MCP
- Esaminare l'utilizzo delle risorse di sistema

**Risposta incompleta**
- Garantire `session/update` le notifiche e il risultato finale vengono aggregati correttamente
- Controlla la logica di cancellazione nella gestione dello streaming
- Verificare che la gestione degli errori sia completa

**Display frontali sconosciuti**
- Conferma che la generazione OpenAPI include già il file `HermesCli` valore enum
- Controlla se la mappatura del tipo è corretta
- Svuota la cache del browser e rigenera i tipi

I problemi esisteranno sempre. Quando compaiono, l’importante è non farsi prendere dal panico. Traccia la causa passo dopo passo e, alla fine, la maggior parte di esse può essere risolta.

### Suggerimenti per l'ottimizzazione delle prestazioni

1. **Utilizza il pool di sessioni**: riutilizza i sottoprocessi ACP per ridurre il sovraccarico di avvio
2. **Imposta i timeout in modo appropriato**: bilancia l'utilizzo della memoria rispetto ai costi di avvio
3. **Riutilizza ID sessione**: usa lo stesso `CessionId` per attività batch
4. **Configura MCP su richiesta**: evita invocazioni di strumenti non necessarie

Le prestazioni sono molto simili all’efficienza nella vita quotidiana. Quando lo fai bene, ottieni di più con meno; quando si sbaglia, lo sforzo si moltiplica mentre i risultati si riducono. Trovare il punto “giusto” richiede esperienza e fortuna.

## Conclusione

L'integrazione di Hermes Agent in un sistema di produzione richiede la considerazione di problemi su più dimensioni:

1. **Architettura**: progettare un'interfaccia provider unificata e implementare un'architettura di componenti sostituibili
2. **Protocollo**: gestisce correttamente il comportamento specifico dell'ACP come i marcatori di avvio e l'autenticazione dinamica
3. **Prestazioni**: riutilizza le risorse attraverso il pool di sessioni e bilancia i costi di avvio con l'utilizzo della memoria
4. **Front-end**: garantisce la sincronizzazione dei contratti e fornisce un'esperienza visiva coerente

L'esperienza di HagiCode dimostra che con una buona progettazione a più livelli e un'implementazione basata sulla configurazione, un sistema di agenti complesso può essere integrato perfettamente in un'architettura esistente.

Questi principi sembrano semplici se descritti a parole, ma l’implementazione effettiva introduce sempre molti tipi diversi di problemi. Va bene. Se un problema viene risolto, diventa esperienza. Se così non fosse, diventa una lezione. In ogni caso, ha ancora valore.

Le cose o le persone belle non hanno bisogno di essere possedute; finché rimangono belli, è sufficiente apprezzare in silenzio quella bellezza. La tecnologia è più o meno la stessa. Se aiuta a migliorare il sistema, allora il quadro o il protocollo specifico conta molto meno di quanto a volte si pensi.
