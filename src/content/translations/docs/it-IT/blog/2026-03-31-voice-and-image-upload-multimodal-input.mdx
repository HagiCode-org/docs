## Digitare è più lento che parlare e parlare è più lento di uno screenshot - Pratiche di input multimodali per assistenti di codifica AI

> La scrittura del codice ha un limite di velocità, non importa quanto velocemente digiti. A volte ci vuole un'eternità per scrivere qualcosa che potresti dire in una frase; a volte uno screenshot spiega tutto, ma devi comunque descriverlo con una pila di testo. Questo articolo parla di ciò in cui ci siamo imbattuti durante la costruzione [HagiCode](https://hagicode.com), dal riconoscimento vocale al caricamento di immagini. Alla fine, volevamo solo rendere un assistente di codifica AI un po’ più facile da usare.

## Sfondo

Durante la creazione di HagiCode, abbiamo notato un problema - o meglio, un problema che è emerso naturalmente quando le persone hanno iniziato a usarlo pesantemente: fare affidamento solo sulla digitazione può essere stancante.

Pensaci: l'interazione tra gli utenti e l'agente è uno scenario fondamentale. Ma se ogni scambio richiede una digitazione continua sulla tastiera, l’efficienza non è eccezionale:

1. **La digitazione è troppo lenta**: per problemi complicati, come messaggi di errore o problemi dell'interfaccia utente, digitare tutto può richiedere mezzo minuto, mentre pronunciarlo ad alta voce potrebbe richiedere dieci secondi. Questo divario è reale.

2. **Le immagini sono più dirette**: a volte l'interfaccia utente genera un errore, a volte desideri confrontare una bozza di progettazione e talvolta devi mostrare una struttura di codice. "Un'immagine vale più di mille parole" potrebbe essere un vecchio detto, ma è ancora valido. Lasciare che l’IA “veda” direttamente il problema è molto più chiaro che descriverlo per secoli.

3. **L'interazione dovrebbe sembrare naturale**: i moderni assistenti IA dovrebbero supportare testo, voce e immagini. Gli utenti dovrebbero essere in grado di scegliere il metodo di input che ritengono più naturale.

Quindi abbiamo deciso di aggiungere il riconoscimento vocale e il supporto per il caricamento di immagini a HagiCode per rendere più convenienti le interazioni con gli agenti. Se gli utenti riescono a digitare un po’ meno, è già una vittoria.

## A proposito di HagiCode

Le soluzioni condivise in questo articolo provengono dal nostro lavoro pratico in [HagiCode](https://hagicode.com) progetto - o, più precisamente, dalle lezioni apprese inciampando in numerose insidie.

HagiCode è un progetto di assistente alla codifica AI open source con un obiettivo semplice: utilizzare l'intelligenza artificiale per migliorare l'efficienza dello sviluppo. Man mano che continuavamo a costruire, è diventato chiaro che gli utenti desideravano fortemente un input multimodale. A volte pronunciare una frase è più veloce che digitare un lungo paragrafo e talvolta uno screenshot è molto più chiaro di una lunga spiegazione.

Queste esigenze ci hanno spinto avanti ed è così che alla fine hanno preso forma funzionalità come il riconoscimento vocale e il caricamento di immagini. Gli utenti possono ora interagire con l’intelligenza artificiale nel modo più naturale a loro disposizione, e questo è una bella sensazione.

## Analisi

### Sfide tecniche nel riconoscimento vocale

Durante la creazione del riconoscimento vocale, ci siamo imbattuti in un problema complicato: **l'API WebSocket del browser non supporta le intestazioni HTTP personalizzate**.

Il servizio di riconoscimento vocale che abbiamo scelto è stato l'API di riconoscimento vocale Doubao di ByteDance. Sfortunatamente, questa API richiede informazioni di autenticazione come `accessToken` e `secretKey` da passare attraverso le intestazioni HTTP. Ciò ha creato un conflitto tecnico immediato:

```javascript
// The browser WebSocket API does not support this approach
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // Not supported
  }
});
```

Fondamentalmente avevamo due opzioni:

1. **Approccio con parametro di query URL**: inserisci le informazioni di autenticazione nell'URL
   - Il vantaggio è che è semplice da implementare
   - Lo svantaggio è che le credenziali sono esposte al frontend, il che non è sicuro; alcune API richiedono anche l'autenticazione basata sull'intestazione

2. **Approccio proxy backend**: implementa un proxy WebSocket sul backend
   - Il vantaggio è che le credenziali rimangono archiviate in modo sicuro sul backend e la soluzione è completamente compatibile con i requisiti API
   - Lo svantaggio è che l’implementazione è un po’ più complessa

Alla fine, abbiamo scelto l'**approccio proxy backend**. La sicurezza non è qualcosa su cui scendere a compromessi.

### Requisiti funzionali per i caricamenti di immagini

I nostri requisiti per il caricamento delle immagini erano in realtà piuttosto semplici:

1. **Metodi di caricamento multipli**: fai clic per selezionare un file, trascina e rilascia e incolla dagli appunti
2. **Convalida file**: restrizioni sul tipo (PNG, JPG, WebP, GIF) e limiti sulle dimensioni (5-10 MB) sono requisiti di base
3. **Esperienza utente**: carica progressi, anteprime e messaggi di errore in modo che gli utenti sappiano sempre cosa sta succedendo
4. **Sicurezza**: la convalida lato server e la protezione contro i caricamenti di file dannosi sono essenziali

## Soluzione

### Riconoscimento vocale: architettura proxy WebSocket

Abbiamo progettato un'architettura a tre livelli per il riconoscimento vocale e abbiamo trovato un percorso che ha funzionato:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**Implementazioni dei componenti principali**:

1. **Processore AudioWorklet frontend**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Resample to 16 kHz (required by the Doubao API)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Accumulate samples into 500 ms chunks
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Convert to 16-bit PCM and send
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **Gestore WebSocket backend** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **Frontend `VoiceTextArea` componente**:


### Caricamenti di immagini: componente di caricamento multimetodo

Abbiamo creato un componente di caricamento delle immagini completo con supporto per tutti e tre i metodi di caricamento, coprendo gli scenari più comuni in cui si imbattono gli utenti.

**Caratteristiche principali**:

1. **Tre metodi di caricamento**:

```tsx
// Click to upload
const handleClick = () => fileInputRef.current?.click();

// Drag-and-drop upload
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Clipboard paste
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **Convalida frontend**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **Gestore di caricamento backend** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Validation
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Save file
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## Guida pratica

### Come utilizzare il riconoscimento vocale

1. **Configura il servizio di riconoscimento vocale**:
   - Apri la pagina delle impostazioni del riconoscimento vocale
   - Configura il discorso Doubao `AppId` e `AccessToken`
   - Facoltativamente, configura le hotword per migliorare la precisione del riconoscimento dei termini specifici del dominio

2. **Utilizzalo nella casella di input**:
   - Fare clic sull'icona del microfono sul lato sinistro della casella di input
   - Inizia a parlare dopo che viene visualizzata l'animazione della forma d'onda
   - Fare nuovamente clic sull'icona per interrompere la registrazione
   - Il testo riconosciuto viene inserito automaticamente nella posizione del cursore

### Come utilizzare i caricamenti di immagini

1. **Metodi di caricamento**:
   - Fare clic sul pulsante di caricamento per scegliere un file
   - Trascina un'immagine direttamente nell'area di caricamento
   - Utilizzare `Ctrl+V` per incollare uno screenshot dagli appunti

2. **Formati supportati**: PNG, JPG, JPEG, WebP, GIF
3. **Limite dimensione**: 5 MB per impostazione predefinita (configurabile)

### Note

1. **Riconoscimento vocale**:
   - È richiesta l'autorizzazione del microfono
   - Utilizzare in un ambiente tranquillo quando possibile
   - La durata massima di registrazione supportata è di 300 secondi per impostazione predefinita (configurabile)

2. **Caricamenti di immagini**:
   - Sono supportati solo i formati immagine più comuni
   - Presta attenzione ai limiti di dimensione del file
   - Le immagini caricate ricevono automaticamente un URL di anteprima

3. **Considerazioni sulla sicurezza**:
   - Le credenziali di riconoscimento vocale vengono archiviate nel backend
   - I caricamenti delle immagini passano attraverso una rigorosa convalida lato server
   - HTTPS/WSS è consigliato negli ambienti di produzione
