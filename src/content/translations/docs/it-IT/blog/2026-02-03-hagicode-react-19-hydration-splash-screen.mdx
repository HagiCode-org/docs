## Progettare 12 esperienze di startup eccezionali per HagiCode: dal minimalismo al cyberpunk

> Il breve intervallo tra il download di un'app React 19 e il completamento di Hydration è un'occasione d'oro per gli utenti di percepire la personalità del tuo marchio. In questo articolo condividiamo un sistema di stile di avvio completo che abbiamo creato per il progetto HagiCode utilizzando HTML/CSS/JS.

## Sfondo

Essendo un'applicazione moderna creata con ASP.NET Core 10 e React 19 (Vite), HagiCode usa un'architettura di distribuzione separata frontend-backend. L'output della build del frontend è inserito nel backend `wwwroot/` directory e ospitato da ASP.NET Core.

Tuttavia, questa architettura introduce un classico punto dolente UX: quando gli utenti visitano la pagina, il browser carica prima l'HTML, quindi scarica il pacchetto JS di grandi dimensioni e infine consente a React di eseguire l'idratazione. Durante questo "periodo di vuoto" che dura da poche centinaia di millisecondi a diversi secondi, gli utenti vedono uno schermo vuoto o una pagina statica senza vita.

Per colmare questa lacuna e conferire personalità al marchio HagiCode, dovevamo progettare un sistema in stile startup implementato interamente con codice in linea all'interno `index.html`.

## A proposito di HagiCode

L'approccio alla progettazione dello splash screen condiviso in questo articolo deriva dalla nostra esperienza pratica nel [HagiCode](https://github.com/HagiCode-org/site) progetto. In qualità di assistente alla codifica AI, HagiCode si preoccupa non solo dell'efficienza della generazione del codice, ma anche dell'esperienza visiva dello sviluppatore. Questo sistema di avvio è uno dei risultati della nostra ricerca delle massime prestazioni del frontend.

## Sfide fondamentali e progettazione dell'architettura

Prima di iniziare la progettazione, abbiamo dovuto chiarire i vincoli tecnici. Poiché tutto doveva essere implementato in linea in `index.html`, non è stato possibile caricare file CSS o JS esterni diversi dal bundle di React.

### Analisi dei vincoli tecnici

1.  **Principio di dipendenza zero**: tutti gli stili devono vivere all'interno di a `<style>` tag e tutta la logica deve vivere all'interno di un tag `<script>` etichetta.
2.  **CSS difensivo**: per evitare che gli stili globali inquinino la schermata iniziale dopo il montaggio dell'app React, abbiamo deciso di racchiudere tutti gli stili di avvio con un prefisso ID ad alta priorità come `#boot-screen`.
3.  **Prima le prestazioni**: le animazioni devono utilizzare CSS `transform` e `opacity` ove possibile, per evitare il riflusso e garantire che il thread principale rimanga sbloccato.
4.  **Coerenza visiva**: i colori e i caratteri devono rimanere allineati con la configurazione Tailwind di HagiCode.

### Modello architettonico: guscio e iniettore

Abbiamo adottato un **modello di variante**. La logica principale è incapsulata all'interno di un'espressione di funzione immediatamente richiamata (IIFE), mentre la logica di rendering specifica viene iniettata tramite la configurazione. Ciò ci consente di passare da uno stile all'altro attraverso una semplice configurazione invece di riscrivere ripetutamente la logica di manipolazione del DOM.

Ecco il codice dell'architettura principale:

```html
<!-- Inline in index.html -->
<div id="boot-root"></div>

<script>
(function() {
  const BootSequence = {
    config: {
      theme: 'terminal', // Configurable as 'minimal', 'skeleton', 'code-rain', etc.
      color: '#3b82f6'   // Brand color
    },
    
    // Core lifecycle
    init() {
      this.render();
      this.listenForMount();
    },

    // Render the currently selected style
    render() {
      const root = document.getElementById('boot-root');
      if (this.variants[this.config.theme]) {
        root.innerHTML = this.variants[this.config.theme].render();
      }
    },

    // Listen for successful React mount and exit gracefully
    listenForMount() {
      window.addEventListener('hagicode:ready', () => {
        const screen = document.getElementById('boot-root');
        // Fade out first, then remove the DOM to avoid flicker
        screen.style.opacity = '0';
        screen.style.transition = 'opacity 0.3s ease';
        setTimeout(() => screen.remove(), 300);
      });
    },

    // The implementation logic for all 12 styles lives here
    variants: {
      // ...see details below
    }
  };

  BootSequence.init();
})();
</script>
```

## Una lista di controllo di 12 progetti di stili di avvio

Abbiamo raggruppato questi 12 stili in sei categorie principali per soddisfare diversi scenari e preferenze estetiche.

### R. Minimalismo

> "Meno è di più." Per gli scenari che perseguono la massima velocità di caricamento, forniamo le opzioni più leggere possibili.

#### 1. Punto minimalista
Un semplice punto si trova al centro dello schermo, abbinato a un'animazione che respira.
*   **Implementazione**: CSS `@keyframes` controlla la scala e l'opacità.
*   **Ideale per**: Ogni caso in cui la pagina deve rimanere assolutamente pulita.

#### 2. Rivelazione del marchio
Utilizzando SVG `stroke-dasharray` animazione, simula una rivelazione disegnata a mano delle linee del logo HagiCode, seguita da una dissolvenza in apertura del testo.
*   **Tecnica**: animazione del percorso SVG con un aspetto estremamente raffinato.

### B. Mimetismo dello schermo scheletrico

> "L'arte di ingannare l'occhio." Simulando un layout dell'interfaccia utente reale, gli utenti hanno la sensazione che la pagina sia già caricata a metà.

#### 3. Scheletro della chat nella barra laterale
Questa potrebbe essere l'opzione più pratica. Abbiamo creato manualmente un layout in HTML che rispecchia React `Sidebar` e `ChatInput` componenti esattamente, quindi sovrapposti con un'animazione grigia brillante.
*   **Valore**: una volta completata l'idratazione di React, lo scheletro diventa immediatamente il componente reale e gli utenti riescono a malapena a percepire il passaggio.

#### 4. Scheletro della pila di carte
Simula il movimento impilato delle carte proposta durante il caricamento, utilizzando trasformazioni 3D per far fluttuare leggermente le carte.

### C. Astrazione e arte

> Metti in mostra il DNA geek di HagiCode.

#### 5. Morfologia geometrica
Una forma geometrica (un quadrato) viene renderizzata al centro dello schermo, poi si trasforma gradualmente nel tempo in un cerchio, un triangolo e infine nel logo.
*   **Tecnologia**: transizioni fluide con CSS `border-radius`.

#### 6. Pioggia in codice
Un omaggio a *The Matrix*. Utilizzando il carattere JetBrains Mono, deboli flussi di caratteri cadono sullo sfondo.
*   **Nota**: per le prestazioni, i flussi di personaggi devono rimanere in un'area più piccola o utilizzare una frequenza di aggiornamento inferiore.

#### 7. Impulso al neon
Un anello luminoso in stile cyberpunk che ne utilizza molteplici `box-shadow` strati per creare un potente bagliore al neon.

### D. Marchio e temi

> Fai sentire il sistema vivo.

#### 8. Tema stagionale
Questo è un caricatore dinamico. Controlla la data corrente delle festività come il Capodanno lunare o il Natale e carica l'animazione SVG corrispondente.
*   **Esempio**: durante il Capodanno lunare, le lanterne rosse oscillano dolcemente nella parte inferiore dello schermo.

#### 9. Flusso gradiente
Lo sfondo utilizza una sfumatura fluida basata sui colori del marchio HagiCode. Combinato con animato `background-size` e `background-position`, crea un senso di movimento simile all'aurora.

### E. Sensazione tecnica

> Un saluto agli sviluppatori.

#### 10. Avvio del terminale
Simula l'output della console. Le righe di codice scorrono rapidamente:
```text
> Initializing HagiCode Core...
> Loading models...
> Connecting to neural network...
```
Questo sembra immediatamente familiare a ogni sviluppatore.

#### 11. Barra di avanzamento
Nella parte superiore dello schermo viene visualizzata una sottile barra di avanzamento, con una percentuale mostrata sulla destra. Anche se non possiamo accedere al reale avanzamento del download, possiamo utilizzare un timer per simulare un processo di caricamento "credibile": veloce per il primo 80%, poi gradualmente più lento per l'ultimo 20%.

### F. Creatività

#### 12. Assemblaggio dei pixel
Questa è un'idea molto interessante. Piccoli quadrati sono sparsi sullo schermo, quindi convergono verso il centro e si assemblano gradualmente nell'icona del logo HagiCode. Simboleggia il processo di costruzione del codice.

## Migliori pratiche e insidie

Nel vero lavoro di sviluppo di HagiCode, abbiamo riassunto diversi dettagli di implementazione di fondamentale importanza.

### 1. Il CSS difensivo è obbligatorio
Non essere mai pigro e non saltare il prefisso. Una volta, ci siamo dimenticati di definire l'ambito degli stili della schermata iniziale con un ID e globale `div` Gli stili dopo il montaggio di React hanno influenzato inaspettatamente la schermata iniziale, interrompendo il layout.
**Lezione appresa**: inserisci tutti i selettori CSS in `#boot-screen`e utilizzare `!important` per aumentare la priorità quando necessario, ma solo all'interno del CSS della schermata iniziale.

### 2. Transizioni aggraziate
Dopo che React è stato montato correttamente, non farlo direttamente `remove()` il DOM della schermata iniziale.
**Approccio corretto**:
1.  Reagire innesca `window.dispatchEvent(new Event('hagicode:ready'))`.
2.  La schermata iniziale ascolta l'evento e i primi set `opacity: 0`.
3.  Attendi 300 ms, che corrisponde alla durata della transizione CSS, e chiama `.remove()` solo dopo che lo schermo sarà completamente invisibile.

### 3. Sincronizzazione delle variabili del tema
I valori dei colori della schermata iniziale sono codificati `index.html`. Se cambiamo il colore primario di Tailwind, dobbiamo aggiornare anche la schermata iniziale.
**Ottimizzazione**: scrivi un semplice plugin nello script di build Vite da leggere `tailwind.config.js` e iniettare variabili di colore nel file `index.html` variabili modello, creando un'unica fonte di verità.

### 4. Precaricamento dei caratteri
Le schermate iniziali spesso devono utilizzare un carattere di marca, ma se il carattere viene caricato lentamente, può apparire FOUT (Flash of Unstyled Text).
**Soluzione**: aggiungi `<link rel="preload" href="/fonts/JetBrainsMono.woff2" as="font" type="font/woff2" crossorigin>` dentro `<head>`. Questo è un modo a basso costo e ad alto rendimento per migliorare l’esperienza.

### 5. Monitoraggio delle prestazioni
Abbiamo iniettato `performance.mark('boot-start')` in fondo a `index.html`, e segnato `boot-end` quando React è stato montato correttamente.
**Perché è importante**: raccogliendo questi dati tramite Application Insights, possiamo misurare direttamente quanto la schermata iniziale riduce il tempo di attesa percepito. I dati mostrano che un eccellente schermo scheletro può migliorare la tolleranza degli utenti verso una "rete lenta" di oltre il 50%.
