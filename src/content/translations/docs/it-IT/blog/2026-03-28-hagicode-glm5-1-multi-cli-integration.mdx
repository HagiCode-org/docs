## Guida all'integrazione multi-CLI di Hagicode e GLM-5.1

:::nota[Avviso sul contesto temporale]
Questo articolo documenta la progettazione dell'integrazione multi-CLI e la mappatura dei modelli così come esistevano al momento della pubblicazione. Per l'ambito corrente sempre supportato, fare riferimento a [Installazione della CLI dell'agente AI](/it-IT/related-software-installation/ai-agent-cli/) e il [Guida all'installazione](/it-IT/installation); il `IFlowCli` qui menzionato rappresenta solo una configurazione storica in quel momento e non fa parte dell'attuale ambito di supporto attivo.
:::

## Sfondo

Nel progetto Hagicode, gli utenti possono scegliere tra più strumenti CLI per gestire gli assistenti di programmazione AI, tra cui Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI e altri. Questi strumenti CLI sono di per sé strumenti di programmazione AI generici, ma attraverso il livello di astrazione di Hagicode possono connettersi in modo flessibile a diversi fornitori di modelli AI.

Zhipu AI (ZAI) fornisce un'interfaccia compatibile con l'API Anthropic Claude, consentendo a questi strumenti CLI di utilizzare direttamente i modelli domestici della serie GLM. Tra questi, GLM-5.1 è l'ultima versione del modello linguistico di grandi dimensioni di Zhipu, con miglioramenti significativi rispetto a GLM-5.0.

## Architettura di astrazione CLI di Hagicode

Hagicode definisce 11 tipi di provider CLI attraverso il file `AIProviderType` enum, che copre gli strumenti CLI di programmazione AI tradizionali:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Ogni CLI ha la configurazione dei parametri del modello corrispondente e supporta `model` e `reasoning` parametri:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],
        [AIProviderType.CodexCli] = ["model", "reasoning"],
        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],

        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini does not support the reasoning parameter
        // ...
    };
```

## Utilizzo di GLM-5.1 in diverse CLI

### 1. Codice Claude CLI + GLM-5.1

Claude Code CLI è una delle CLI principali di Hagicode ed è configurata tramite il sistema di configurazione Hero:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Corrispondente `HeroEquipmentCatalogItem` configurazione:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. CLI OpenCode + GLM-5.1

OpenCode CLI è la CLI più flessibile e supporta la specifica di qualsiasi modello nel file `provider/model` formato:

**Metodo 1: utilizzare il prefisso del provider ZAI**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Metodo 2: utilizzare direttamente l'ID modello**
```json
{
  "model": "glm-5.1"
}
```

**Metodo 3: interfaccia utente di configurazione frontend**

Dentro `HeroModelEquipmentForm.tsx`, la CLI di OpenCode dispone di un suggerimento segnaposto dedicato:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Gli utenti possono inserire:
```
zai/glm-5.1
glm-5.1
```

**Logica di analisi del modello CLI OpenCode**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. CLI di Codebuddy + GLM-5.1

La CLI di Codebuddy dispone di una logica di gestione legacy dedicata:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Nota**: Codebuddy mantiene una gestione speciale per GLM-5.0 e non utilizza la normalizzazione legacy:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## Vantaggi di miglioramento di GLM-5.1

Rispetto a GLM-5.0, GLM-5.1 apporta i seguenti miglioramenti significativi:

### 1. Migliore capacità di ragionamento

Secondo le informazioni sulla versione ufficiale di Zhipu, i miglioramenti in GLM-5.1 includono:

- **Migliore comprensione del codice**: analisi più accurata di strutture di codice complesse
- **Comprensione del contesto più lunga**: supporta un contesto di conversazione più lungo
- **Chiamate degli strumenti migliorate**: percentuale di successo più elevata per le chiamate degli strumenti MCP
- **Stabilità dell'output**: Riduce la casualità e le allucinazioni

### 2. Compatibilità multi-CLI completa

GLM-5.1 copre tutte le CLI tradizionali supportate da Hagicode:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Note

### 1. Configurazione della chiave API

Assicurati che `ANTHROPIC_AUTH_TOKEN` la variabile di ambiente è impostata correttamente. È la credenziale richiesta per ogni CLI per connettersi al modello.

### 2. Disponibilità del modello

GLM-5.1 deve essere abilitato dal fornitore del modello corrispondente:
- La piattaforma Zhipu AI ZAI lo supporta per impostazione predefinita
- Alibaba Cloud DashScope potrebbe richiedere un'applicazione separata

### 3. Formato CLI OpenCode

Quando si utilizza il `provider/model` formato, assicurati che l'ID del provider sia corretto:
- Zhipu AI: `zai` o `zhipuai`
- AlibabaNuvola: `aliyun` o `dashscope`

### 4. Parametro del ragionamento

- `high` è consigliato per ottenere i migliori risultati di generazione del codice
- Gemini CLI non supporta il parametro di ragionamento e ignorerà automaticamente questa configurazione

## Conclusione

Attraverso un livello di astrazione unificato, Hagicode consente un'integrazione flessibile tra GLM-5.1 e più CLI. Gli sviluppatori possono scegliere lo strumento CLI che meglio si adatta alle loro preferenze e ai loro scenari di utilizzo, quindi utilizzare l'ultimo modello GLM-5.1 attraverso una semplice configurazione.

Essendo l'ultima versione del modello Zhipu, GLM-5.1 offre chiari miglioramenti rispetto a GLM-5.0:
- Un identificatore di versione indipendente senza oneri legacy
- Ragionamento e comprensione del codice più forti
- Ampia compatibilità multi-CLI
- Configurazione flessibile del livello di ragionamento

Con le variabili di ambiente corrette e l'attrezzatura Hero configurata, gli utenti possono sfruttare appieno la potenza di GLM-5.1 in diversi ambienti CLI.
