Questa guida illustra l'installazione e l'utilizzo di Hagicode Desktop. È il client desktop ufficiale e mantiene il percorso di installazione locale predefinito oltre ai controlli giornalieri in un unico posto.

:::suggerimento[Metodo consigliato]
Desktop è il percorso locale predefinito per singoli utenti e sviluppatori perché:
- l'installatore mantiene il flusso predefinito in un unico percorso guidato
- dipendenze e servizi sono gestiti nell'app
- l'interfaccia utente semplifica l'ispezione dello stato e delle operazioni
- la gestione della versione integrata semplifica il passaggio
- i dati rimangono locali per impostazione predefinita
:::

## Cos'è Hagicode Desktop?

Hagicode Desktop è l'applicazione desktop ufficiale di Hagicode, progettata per sistemi Windows, macOS e Linux. Raggruppa le funzionalità principali di Hagicode e il programma di installazione mantiene il percorso di installazione predefinito in un flusso guidato.

### Principali vantaggi

- **Installazione con un clic**: completa automaticamente tutti i passaggi di configurazione tramite la procedura guidata di installazione
- **Preparazione automatica del runtime**: il programma di installazione predefinito include già il runtime necessario, quindi non è necessario installarne uno separato `.NET` prima l'esecuzione
- **Gestione dei servizi**: gestione integrata del server Hagicode, avvia e arresta facilmente i servizi
- **Cambio di versione**: supporto per la gestione multiversione, passaggio libero tra diverse versioni
- **Operazione locale**: tutti i dati vengono archiviati localmente, proteggendo la tua privacy
- **Aggiornamenti automatici**: rileva e scarica automaticamente la versione più recente

### Scenari applicabili

Il desktop è una buona soluzione quando:
- Singoli sviluppatori che desiderano utilizzare Hagicode localmente
- Utenti che necessitano di utilizzare Hagicode offline
- Utenti non tecnici che desiderano semplificare il processo di installazione
- Sviluppatori che hanno bisogno di passare frequentemente da una versione all'altra

### Altri metodi di download

Puoi anche ottenere il programma di installazione di Hagicode Desktop dai seguenti canali:

1. **Pagina di rilascio ufficiale**: visita [Rilasci di Hagicode GitHub](https://github.com/Hagicode-org/releases) per scaricare la versione più recente
2. **Sito ufficiale**: visita [Sito web Hagicode](https://hagicode.com/) per ottenere i collegamenti per il download

### Scegli la versione corretta

Seleziona il programma di installazione corrispondente al tuo sistema operativo:
- **Windows**: scarica il file `.exe` installatore
- **macOS**: scarica il file `.dmg` installatore
- **Linux**: scarica il file `.AppImage` o `.tar.gz` pacchetto di installazione

:::nota[Requisiti di sistema]
- **Windows**: Windows 10 o versione successiva (64 bit)
- **macOS**: macOS 10.15 o versione successiva
- **Linux**: distribuzioni Linux tradizionali (Ubuntu 20.04+, Fedora 33+, ecc.)
:::

:::tip[Prerequisiti predefiniti]
Per l'ultimo flusso di installazione locale di HagiCode, in genere non è necessario prepararne uno separato `.NET` tempo di esecuzione. Desktop raggruppa già il runtime richiesto per il flusso predefinito e inizializza il percorso SQLite locale supportato per impostazione predefinita.
:::

## Note di installazione e avvio

:::attenzione[Avviso Gatekeeper macOS]
Se hai già installato l'app nel percorso predefinito `/Applications/Hagicode.app` ma macOS mostra "l'app è danneggiata", "impossibile aprire" o un avviso simile del Gatekeeper al primo avvio, la causa più comune è `com.apple.quarantine` attributo che macOS aggiunge alle app scaricate da Internet, non un errore nel programma di installazione di Hagicode Desktop stesso.

Esegui il seguente comando nel Terminale. Richiede privilegi di amministratore e rimuove in modo ricorsivo l'attributo di quarantena dall'app nel percorso di installazione predefinito:

```bash
sudo xattr -dr com.apple.quarantine /Applications/Hagicode.app
```

Al termine del comando, riavvia Hagicode Desktop.
:::

## Esegui Desktop

Dopo il download, apri il programma di installazione o il pacchetto dell'applicazione e segui le istruzioni visualizzate sullo schermo. Desktop prepara il runtime nel percorso locale attualmente supportato, gestisce Hagicode Server e mostra l'avanzamento e lo stato quando sono richiesti download di rete.

:::nota[Requisiti di rete]
Assicurati che la connessione di rete sia disponibile. Potrebbe essere necessario che il desktop recuperi informazioni sulla versione o pacchetti di componenti da Internet; se il download non riesce, controlla le impostazioni di rete, proxy o firewall e riprova.
:::

## Gestione delle versioni

Hagicode Desktop supporta la gestione multiversione, consentendoti di passare liberamente da una versione all'altra.

![Gestione delle versioni](../../img/screenshots/installation/install-desktop/12-version-management/original.png)

### Cambia versione

1. Apri Hagicode Desktop
2. Vai alla pagina "Gestione versione".
3. Seleziona la versione che desideri cambiare
4. Fare clic sul pulsante "Cambia versione".
5. Attendi il completamento del passaggio

:::note[Note sul cambio di versione]
- Il cambio di versione riavvierà automaticamente Hagicode Server
- Assicurati di salvare tutto il lavoro prima di cambiare
- Le configurazioni del progetto possono differire tra le versioni
:::

## Avvia e arresta il servizio

Hagicode Desktop fornisce un modo semplice per gestire il servizio Hagicode Server.

### Avvia servizio

Nella home page dell'applicazione desktop, fare clic sul pulsante "Avvia servizio" per avviare Hagicode Server.

![Avvia servizio](../../img/screenshots/installation/install-desktop/13-start-service-homepage/original.png)

### Interrompere il servizio

Per arrestare Hagicode Server, fare clic sul pulsante "Interrompi servizio" nella home page.

:::suggerimento[Stato del servizio]
L'applicazione desktop visualizza lo stato di funzionamento di Hagicode Server in tempo reale, tra cui:
- Stato del servizio (in esecuzione/interrotto)
- Numero della porta del servizio
- Informazioni sulla versione corrente
:::

## Risoluzione dei problemi

### Errore di installazione

Se riscontri problemi durante l'installazione:

1. **Verifica connessione di rete**: assicurati di poter accedere a Internet
2. **Controlla spazio su disco**: assicurati di avere spazio su disco sufficiente
3. **Esegui come amministratore**: su Windows, fai clic con il pulsante destro del mouse sul programma di installazione e seleziona "Esegui come amministratore"
4. **Visualizza file di registro**: il programma di installazione genera file di registro dettagliati che possono aiutare a diagnosticare i problemi

### Errore di installazione della dipendenza

Se l'installazione delle dipendenze fallisce:

1. **Verifica compatibilità del sistema**: assicurati che il tuo sistema operativo soddisfi i requisiti minimi
2. **Controlla le impostazioni di rete e firewall**: assicurati che il programma di installazione possa raggiungere i download richiesti e non sia bloccato dal firewall
3. **Conferma le autorizzazioni del programma di installazione**: su Windows, prova a eseguire il programma di installazione come amministratore in modo che disponga di autorizzazioni sufficienti
4. **Riavvia l'app e riprova**: chiudi e riapri Hagicode Desktop, quindi riprova il passaggio di installazione
5. **Visualizza registri**: esamina i registri degli errori dettagliati nell'applicazione desktop

### Errore di avvio del servizio

Se Hagicode Server non si avvia:

1. **Controlla l'occupazione della porta**: assicurati che la porta 45000 non sia occupata da altri programmi
2. **Controlla le impostazioni del firewall**: assicurati che il firewall consenta l'accesso alla rete Hagicode Server
3. **Riavvia applicazione**: prova a riavviare Hagicode Desktop
4. **Visualizza registri**: visualizza i registri degli errori dettagliati nell'applicazione desktop
