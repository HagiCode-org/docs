## Prerequisiti

Prima di iniziare, assicurati che sul tuo sistema siano installati Docker e Docker Compose.

### Installa Docker

#### Finestre

Scarica e installa [Docker Desktop per Windows](https://www.docker.com/products/docker-desktop/)

Dopo l'installazione, assicurati che Docker Desktop sia in esecuzione.

#### macOS

Scarica e installa [Docker Desktop per Mac](https://www.docker.com/products/docker-desktop/)

Dopo l'installazione, assicurati che Docker Desktop sia in esecuzione.

#### Linux

Installa Docker utilizzando il tuo gestore pacchetti:

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

Dopo l'installazione, avvia il servizio Docker:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Verificare l'installazione

Dopo l'installazione, esegui i seguenti comandi per verificare che Docker e Docker Compose siano installati correttamente:

```bash
docker --version
docker compose version
```

## Avvio rapido

### 1. Genera il file di configurazione di composizione Docker

:::suggerimento[Usa generatore di configurazione]
**Si consiglia di utilizzare [Docker Compose Builder (sito autonomo)](https://builder.hagicode.com/)** per generare il file di configurazione.

Il generatore supporta:
- Molteplici opzioni del provider API (Zhipu AI, Anthropic Official, API personalizzata)
- Configurazione automatica dell'URL API e del token
- Porte personalizzate, nomi di contenitori e altre configurazioni di base
- Configurare i dati persistenti e i percorsi delle directory di lavoro
- Configura automaticamente le differenze della piattaforma Windows/Linux
- Gestisci in modo intelligente i problemi di autorizzazione dei file
- **Selezione della sorgente dell'immagine**: in base al tuo ambiente di rete, seleziona la sorgente dell'immagine appropriata

[🚀 Utilizza subito il generatore →](https://builder.hagicode.com/)
:::

1. Aperto [Docker Compose Builder (sito autonomo)](https://builder.hagicode.com/)
2. Compila la configurazione in base alle tue esigenze:
   - Seleziona il fornitore API (Zhipu AI, Anthropic Official o API personalizzata)
   - Configura porte, database, directory di lavoro e altre opzioni
   - **Seleziona origine immagine**: scegli la fonte immagine appropriata in base al tuo ambiente di rete
3. Fare clic sul pulsante Genera per ottenere il file `docker-compose.yml` configurazione
4. Salva la configurazione generata come file `docker-compose.yml` file
5. Se necessario, crea un file `.env` file per configurare informazioni sensibili

#### Selezione della sorgente dell'immagine

Il generatore supporta tre opzioni di origine immagine:

**Hub Docker**
- **Indirizzo immagine**: `newbe36524/hagicode:{tag}`
- **Scenari applicabili**: adatto agli utenti con accelerazione del mirroring di Docker Hub
- **Vantaggi**: fonte immagine ufficiale, aggiornamenti tempestivi, accesso stabile
- **Note**: in alcune regioni potrebbe essere necessario configurare l'accelerazione dello specchio

**Registro dei contenitori di Azure (ACR)**
- **Indirizzo immagine**: `hagicode.azurecr.io/hagicode:{tag}`
- **Scenari applicabili**: adatto agli utenti che non possono accedere a Docker Hub localmente
- **Vantaggi**: Fornisce una fonte di immagini alternativa, risolve i problemi di accesso alla rete
- **Note**: le immagini rimangono sincronizzate con Docker Hub, ma potrebbero subire brevi ritardi

**Registro dei contenitori cloud Alibaba (ACR)**
- **Indirizzo immagine**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **Scenari applicabili**: adatto per utenti domestici, accesso alla rete stabile e veloce
- **Vantaggi**: Nodi domestici Alibaba Cloud, alta velocità di accesso, elevata stabilità
- **Note**: le immagini Alibaba ACR rimangono sincronizzate con Docker Hub, in genere entro 30 minuti

:::tip[Consigli per la selezione della fonte dell'immagine]
- **Gli utenti domestici consigliano Alibaba Cloud ACR**: Se sei un utente domestico, si consiglia di dare priorità all'utilizzo della sorgente immagine Alibaba Cloud ACR per un accesso veloce e stabile
- **Docker Hub alternativo**: se il tuo ambiente di rete può accedere a Docker Hub, puoi anche scegliere Docker Hub come origine dell'immagine
- **Azure ACR alternativo**: scegli l'origine dell'immagine Azure ACR solo se altre origini dell'immagine hanno difficoltà di accesso
- **Sincronizzazione delle immagini**: le immagini ACR di Alibaba Cloud rimangono sincronizzate con Docker Hub, in genere entro 30 minuti
- **Coerenza della versione**: i tag immagine sono coerenti in tutte e tre le fonti di immagine, non è necessario preoccuparsi dei problemi di versione
:::

### 2. Avvia Servizi

Esegui il comando seguente nella directory in cui si trova il tuo `docker-compose.yml` si trova:

```bash
docker compose up -d
```

Questo comando:
- Scarica e avvia i contenitori Hagicode definiti dal modello generato
- Creare e configurare la rete
- Inizializza la directory dei dati predefinita e le dipendenze di runtime

### 3. Verificare lo stato del servizio

Controlla se i contenitori sono in esecuzione:

```bash
docker compose ps
```

Dovresti vedere i servizi definiti nel modello generato nello stato "in esecuzione".

### Visualizza i registri di servizio

Per visualizzare tutti i registri di servizio:

```bash
# View all service logs
docker compose logs

# View Hagicode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## Istruzioni per la configurazione

Se è necessario modificare la configurazione (ad esempio cambiando porte, cambiando provider API, ecc.), è possibile:

1. Rigenera la nuova configurazione utilizzando [Docker Compose Builder (sito autonomo)](https://builder.hagicode.com/)
2. Modifica manualmente `docker-compose.yml` e `.env` file
3. Riavviare i servizi per rendere effettiva la configurazione:

```bash
docker compose restart
```

## Accedi all'applicazione

### Interfaccia web

Dopo aver avviato con successo il servizio, accedi tramite il tuo browser:

```
http://localhost:45000
```

### Persistenza dei dati

Per impostazione predefinita, il percorso Docker Compose supportato utilizza file di dati SQLite con volumi persistenti. Se è necessario controllare i punti di montaggio o pianificare i backup, rivedere le mappature del volume e delle directory dei dati nel file generato `docker-compose.yml`.
