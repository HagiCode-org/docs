Questa pagina è per gli utenti HagiCode alle prime armi che desiderano il percorso di configurazione OmniRoute più breve: avviare il servizio locale, verificarlo `http://localhost:20128/v1` funziona e conferma che sia la dashboard che l'API sono disponibili.

:::tip[Percorso consigliato]
Questa prima versione mantiene per primo il percorso più stabile: `npm install -g omniroute`, `omniroute`, Cruscotto a `http://localhost:20128`e l'URL di base compatibile con OpenAI all'indirizzo `http://localhost:20128/v1`. Docker, VPS e modalità di distribuzione più avanzate rimangono nella sezione opzionale.
:::

## Prerequisiti

- Node.js è già installato. Completo [Installazione di Node.js](/en/related-software-installation/nodejs/installation) prima, se necessario.
- Porto locale `20128` è disponibile e non già occupato.
- Disponi di almeno un account del fornitore del modello upstream che intendi instradare tramite OmniRoute.
- Prima di integrare HagiCode, una CLI o un SDK, mantieni questi due indirizzi separati:
  - Cruscotto: `http://localhost:20128`
  - URL di base compatibile con OpenAI: `http://localhost:20128/v1`

## Installa ed esegui localmente

Inizia con il percorso npm globale consigliato:

```bash
npm install -g omniroute
omniroute
```

Dopo l'avvio del servizio, convalidarlo in questo ordine:

1. Aperto `http://localhost:20128` in un browser e verificare che la dashboard venga caricata.
2. Vai a `Endpoints` nella dashboard e creare o copiare una chiave API.
3. Prendi nota dell'URL di base che consegnerai agli strumenti in seguito: `http://localhost:20128/v1`.

:::nota[Perché iniziare con la Dashboard]
Per la prima configurazione, il dashboard è il modo più rapido per verificare che OmniRoute sia in esecuzione, che i tuoi provider siano connessi e che esista una chiave API. Se una successiva chiamata del client non riesce, controllare prima il Dashboard è in genere più veloce che eseguire immediatamente il debug del codice SDK.
:::

## Convalidare il servizio in esecuzione

### Convalida 1: conferma che il catalogo dei modelli è leggibile

Metti la chiave dal `Endpoints` pagina in una variabile d'ambiente:

```bash
export OMNIROUTE_API_KEY="replace-with-the-key-you-copied-from-the-Endpoints-page"
```

Quindi richiedi `/v1/models`:

```bash
curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}"
```

Se ricevi un elenco di modelli, l'endpoint compatibile con OpenAI e il flusso di autenticazione funzionano.

### Convalida 2: invia una richiesta di chat minima

```bash
curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "replace-with-a-model-id-from-v1-models",
    "messages": [
      { "role": "user", "content": "Reply with ok." }
    ]
  }'
```

L'obiettivo qui non è la qualità immediata. Serve per dimostrare che il modello scelto può già essere instradato correttamente tramite OmniRoute.
