# Installa OpenSpec

OpenSpec gestisce proposte, specifiche e record di modifiche. Questa pagina ha un unico obiettivo: spiegare chiaramente il percorso di installazione in modo da poterlo allegare al flusso di lavoro esistente.

## Cos'è OpenSpec?

OpenSpec è uno strumento da riga di comando per la gestione di proposte e specifiche nel lavoro del software. Il flusso di lavoro principale include:

- **Gestione delle proposte**: crea, esamina e monitora le proposte di funzionalità
- **Convalida delle specifiche**: garantisce che le modifiche al codice siano conformi alle specifiche predefinite
- **Tracciamento delle modifiche**: registra tutta la cronologia delle modifiche riviste
- **Collaborazione in team**: supporta lo sviluppo collaborativo tra più persone

### Applicazione nei progetti HagiCode

Nei progetti HagiCode, OpenSpec viene utilizzato per:

1. **Gestione dello sviluppo delle funzionalità**: tutte le nuove funzionalità devono passare attraverso il processo di proposta OpenSpec
2. **Revisione del codice**: assicurati che le modifiche siano conformi alle specifiche del progetto
3. **Sincronizzazione della documentazione**: mantieni la coerenza tra la documentazione e l'implementazione del codice
4. **Integrazione del controllo della versione**: integrazione perfetta con i flussi di lavoro del controllo della versione

## Prerequisiti

Prima di installare OpenSpec, assicurati che il tuo sistema soddisfi i seguenti requisiti:

### Node.js e npm

OpenSpec è distribuito tramite npm e richiede un ambiente Node.js. OpenSpec richiede **Node.js 18.0 o versione successiva**.

Verifica l'installazione di Node.js:

```bash
node --version
```

Verifica l'installazione di npm:

```bash
npm --version
```

Se non è installata o la versione è troppo vecchia, fare riferimento alle istruzioni di installazione del software correlato nella [Guida all'installazione](/en/installation).

## Installazione di OpenSpec

:::tip[Requisiti della versione]
Questo progetto richiede l'utilizzo dell'intervallo di versioni **OpenSpec 1.0.0 o successiva**. Sono supportate tutte le versioni 1.x per garantire la compatibilità con il flusso di lavoro del progetto.
:::

OpenSpec viene installato globalmente tramite npm, rendendolo disponibile da qualsiasi punto del sistema.

### Installa globalmente utilizzando npm

<Tabs>
  <TabItem value="win" label="Windows">
    Esegui in PowerShell o nel prompt dei comandi:

```powershell
npm install -g @fission-ai/openspec@1
```

:::nota[Problemi di autorizzazione di Windows]
Se riscontri errori di autorizzazione durante l'installazione, esegui PowerShell o il prompt dei comandi come amministratore.
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    Esegui nel terminale:

```bash
npm install -g @fission-ai/openspec@1
```

:::nota[Problemi di autorizzazione macOS]
Se incontri `EACCES` errori di autorizzazione, prova:
```bash
sudo npm install -g @fission-ai/openspec@1
```
Oppure utilizzare [nvm](https://github.com/nvm-sh/nvm) per gestire le versioni di Node.js ed evitare problemi di autorizzazione.
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    Esegui nel terminale:

```bash
npm install -g @fission-ai/openspec@1
```

:::nota[Problemi di autorizzazione Linux]
Se incontri `EACCES` errori di autorizzazione, puoi:
1. Utilizzare `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. Configura npm per utilizzare la directory utente:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### Verificare l'installazione

Dopo l'installazione, verificare che OpenSpec sia installato correttamente e che la versione soddisfi i requisiti:

```bash
openspec --version
```

Se installato correttamente, vedrai l'output del numero di versione, ad esempio:

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::nota[Intervallo di versioni]
Assicurati che la versione OpenSpec installata sia 1.0.0 o successiva. Il progetto ha verificato la compatibilità con la versione 1.1.1. Si consiglia di utilizzare l'ultima versione compatibile per ottenere nuove funzionalità.
:::

## Problemi comuni e risoluzione dei problemi

### Errori di autorizzazione

#### Problema: `EACCES` Autorizzazione negata

Si sono verificati errori di autorizzazione durante l'installazione o l'esecuzione di OpenSpec.

**Soluzioni**:

<Tabs>
  <TabItem value="win" label="Windows">
    Esegui PowerShell o il prompt dei comandi come amministratore:
    1. Fare clic con il pulsante destro del mouse su PowerShell o Prompt dei comandi
    2. Seleziona "Esegui come amministratore"
    3. Eseguire nuovamente il comando di installazione
  </TabItem>
  <TabItem value="mac" label="macOS">
    Installa utilizzando `sudo`:

```bash
sudo npm install -g @fission-ai/openspec@1
```

Oppure utilizzare [nvm](https://github.com/nvm-sh/nvm) per gestire Node.js ed evitare problemi di autorizzazione di installazione globale.
  </TabItem>
  <TabItem value="linux" label="Linux">
    Soluzione 1: utilizzare sudo
```bash
sudo npm install -g @fission-ai/openspec@1
```

Soluzione 2: configura npm per utilizzare la directory utente
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @fission-ai/openspec@1
```
  </TabItem>
</Tabs>

### Problemi di rete

#### Problema: l'installazione di npm non riesce o è molto lenta

Impossibile scaricare OpenSpec dal registro npm per motivi di rete.

**Soluzioni**:

1. **Utilizza mirror npm nazionale**:

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **Ripristina la fonte ufficiale** (se necessario):

```bash
npm config set registry https://registry.npmjs.org
```

3. **Utilizza cnpm** (facoltativo):

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

:::suggerimento
Si consiglia agli utenti della Cina continentale di utilizzare il mirror Taobao, che può migliorare significativamente la velocità di download.
:::

### Problemi di compatibilità della versione

#### Problema: i comandi OpenSpec vengono eseguiti in modo anomalo o segnalano errori

La versione OpenSpec installata non è compatibile con l'ambiente corrente.

**Soluzioni**:

1. **Controlla la versione di Node.js**:

```bash
node --version
```

Assicurati che la versione sia 18.0 o successiva. Se la versione è troppo bassa, aggiorna Node.js.

2. **Verifica la versione OpenSpec**:

```bash
openspec --version
```

3. **Reinstalla la versione specifica**:

```bash
npm install -g @fission-ai/openspec@1
```

4. **Cancella cache npm** (se il problema persiste):

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### Comando non trovato

#### Problema: digitazione `openspec` Il comando mostra "Comando non trovato"

OpenSpec è installato ma il sistema non riesce a trovare il comando.

**Soluzioni**:

<Tabs>
  <TabItem value="win" label="Windows">
    1. Conferma il percorso di installazione globale di npm:

```powershell
npm config get prefix
```

2. Aggiungi percorso al PATH del sistema:
   - Apri "Proprietà del sistema" → "Avanzate" → "Variabili d'ambiente"
   - Trova `Path` in "Variabili di sistema"
   - Aggiungi il percorso globale npm (solitamente `C:\Users\<username>\AppData\Roaming\npm`)
   - Riavvia la finestra della riga di comando
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. Conferma il percorso di installazione globale di npm:

```bash
npm config get prefix
```

2. Aggiungi il percorso a PATH (in `~/.zshrc` o `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Ricarica configurazione:

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. Conferma il percorso di installazione globale di npm:

```bash
npm config get prefix
```

2. Aggiungi il percorso a PATH (in `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Ricarica configurazione:

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
