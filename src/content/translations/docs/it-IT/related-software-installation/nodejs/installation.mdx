Questa guida copre l'installazione di Node.js su Windows, macOS e Linux. Node.js è una dipendenza chiave per gli strumenti locali HagiCode, inclusi OpenSpec, strumenti frontend e diverse CLI per sviluppatori.

:::tip[Requisiti della versione]
HagiCode consiglia **Node.js 24+** (Node.js minimo supportato &gt;= 18.0). Nella maggior parte dei casi, installa la versione LTS più recente e quindi verifica che la tua versione soddisfi i requisiti.
:::

## Installazione di Node.js

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Metodo uno: programma di installazione ufficiale (consigliato)

1. Visita il [Sito ufficiale di Node.js](https://nodejs.org/)
2. Scarica il programma di installazione di Windows della versione **LTS (supporto a lungo termine)** (`.msi` file)
3. Fare doppio clic per eseguire il programma di installazione
4. Seguire le istruzioni della procedura guidata di installazione per completare l'installazione:
   - Accettare il contratto di licenza
   - Scegli il percorso di installazione (l'impostazione predefinita va bene)
   - Assicurati che "Installa automaticamente gli strumenti necessari" sia selezionato
   - Fare clic su "Installa" per avviare l'installazione
5. Al termine dell'installazione, fare clic su "Fine"

### Metodo due: utilizzo di winget

Apri PowerShell o il prompt dei comandi ed esegui:

```powershell
winget install OpenJS.NodeJS.LTS
```

Dopo l'installazione, riaprire il terminale affinché le modifiche abbiano effetto.

  </TabItem>
  <TabItem value="macos" label="macOS">

### Metodo uno: programma di installazione ufficiale (consigliato)

1. Visita il [Sito ufficiale di Node.js](https://nodejs.org/)
2. Scarica il programma di installazione macOS della versione **LTS (supporto a lungo termine)** (`.pkg` file)
3. Fare doppio clic per eseguire il programma di installazione
4. Seguire le istruzioni della procedura guidata di installazione per completare l'installazione:
   - Fare clic su "Continua"
   - Accettare il contratto di licenza
   - Scegli il disco di installazione
   - Fare clic su "Installa" e inserire la password dell'amministratore
5. Al termine dell'installazione, fare clic su "Chiudi"

### Metodo due: utilizzare Homebrew

Se hai [Birra fatta in casa](https://brew.sh/) installato, puoi installare Node.js tramite terminale:

```bash
brew install node
```

:::suggerimento
Homebrew aggiungerà automaticamente Node.js al tuo PERCORSO. Se il comando non è disponibile, riavvia il terminale o esegui:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

  </TabItem>
  <TabItem value="ubuntu" label="Ubuntu/Debian">

### Utilizzo del repository NodeSource (consigliato)

Questo è il modo migliore per ottenere l'ultima versione di Node.js:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Utilizzando apt Gestore pacchetti

Se non hai bisogno della versione più recente, puoi utilizzare il repository Ubuntu predefinito:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::nota
La versione di Node.js nel repository predefinito potrebbe essere precedente. Dopo l'installazione, controlla la versione per assicurarti che soddisfi i requisiti di Node.js &gt;= 18.0 (si consiglia Node.js 24+).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### Utilizzo del repository NodeSource (consigliato)

```bash
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -

```

### Utilizzando il Gestore pacchetti yum

```bash
sudo yum install -y nodejs npm
```

## Utilizzo degli strumenti di gestione delle versioni

Se devi passare da una versione all'altra di Node.js, puoi utilizzare gli strumenti di gestione delle versioni.

### nvm (Gestore versione nodo)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

Dopo l'installazione, ricarica la configurazione della shell:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Finestre

Su Windows, si consiglia di utilizzare [nvm-windows](https://github.com/coreybutler/nvm-windows):

1. Scarica l'ultima versione `nvm-setup.exe` da [versioni di nvm-windows](https://github.com/coreybutler/nvm-windows/releases)
2. Esegui il programma di installazione
3. Riapri il prompt dei comandi o PowerShell

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

Dopo aver installato nvm, puoi utilizzarlo per installare e gestire le versioni di Node.js:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (Gestore nodo veloce)

fnm è uno strumento di gestione delle versioni Node.js più veloce e semplice creato con Rust.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Finestre

Utilizzare [paletta](https://scoop.sh/) o [alare](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Accelerazione per gli utenti della Cina continentale

Se ti trovi nella Cina continentale, il download dei pacchetti Node.js e npm potrebbe essere lento. Ecco alcune soluzioni di accelerazione.

### Utilizzo di fonti speculari domestiche

#### npm Specchio Taobao (consigliato)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### Utilizzo di nrm per gestire le origini mirror

nrm (gestore del registro npm) ti aiuta a cambiare rapidamente le origini del mirror npm:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Accelerazione dei download del programma di installazione di Node.js

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao fornisce download mirror per gli installatori di Node.js:

- **Indirizzo di download**: https://npmmirror.com/mirrors/node/

Visita l'indirizzo sopra e seleziona la versione che desideri scaricare.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

Se il download di NVM o altri strumenti da GitHub è lento, puoi utilizzare il mirror FastGit:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::nota
Il servizio FastGit potrebbe cambiare, fare riferimento alle informazioni ufficiali.
:::

  </TabItem>
</Tabs>

### Specchi fatti in casa

Se usi Homebrew nella Cina continentale, puoi configurare i mirror nazionali per l'accelerazione:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Gestori di pacchetti specifici per la Cina continentale

#### cnpm

cnpm è un client npm fornito dal team Taobao che utilizza il mirror Taobao per impostazione predefinita:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::suggerimento
cnpm è per lo più compatibile con i comandi npm e può essere utilizzato come sostituto diretto.
:::

### Verifica dell'effetto di accelerazione dello specchio

Dopo aver impostato i mirror, verificare tramite:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Verifica dell'installazione

Dopo l'installazione, verifica che Node.js e npm siano installati correttamente:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::nota
Assicurati che la versione di Node.js &gt;= 18.0 (si consiglia Node.js 24+). Se la versione è troppo bassa, reinstallala utilizzando i metodi sopra indicati o utilizza gli strumenti di gestione della versione per installare la versione corretta.
:::

### Esempio di output di verifica

L'output della verifica riuscita dovrebbe essere simile a:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Risoluzione dei problemi

### Problemi di Windows

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Se `node` o `npm` i comandi non sono disponibili dopo l'installazione:

1. Riavvia il prompt dei comandi o PowerShell
2. Se ancora non disponibile, aggiungi manualmente Node.js a PATH:
   - Cerca "Variabili d'ambiente" e apri "Modifica variabili d'ambiente di sistema"
   - Fare clic su "Variabili d'ambiente"
   - Trova "Percorso" in "Variabili di sistema" e fai clic su "Modifica"
   - Aggiungi il percorso di installazione di Node.js (predefinito: `C:\Program Files\nodejs\`)
   - Fare clic su "OK" per salvare le modifiche

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Se riscontri errori di policy di esecuzione quando usi nvm:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### Problemi con macOS

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Se riscontri errori di autorizzazione durante l'installazione dei pacchetti npm globali:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Se i comandi non sono disponibili dopo l'installazione, riavviare il terminale o aggiungerli manualmente a PATH (solitamente in `/usr/local/bin`).

  </TabItem>
</Tabs>

### Problemi di Linux

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Se riscontri errori di autorizzazione durante l'installazione dei pacchetti npm globali, ti consigliamo di utilizzare gli strumenti di gestione delle versioni di Node.js (come nvm o fnm) anziché i gestori di pacchetti di sistema.

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Se la versione di Node.js installata tramite il gestore pacchetti di sistema è troppo vecchia:

1. Utilizza il repository NodeSource (consigliato, vedi sopra)
2. Oppure utilizza gli strumenti di gestione delle versioni (nvm/fnm)

  </TabItem>
</Tabs>

### Controllo dell'ambiente

Se la verifica fallisce, controlla quanto segue:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
