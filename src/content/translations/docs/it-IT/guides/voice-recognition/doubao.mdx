## Panoramica

### Cos'è l'input vocale di Doubao

Doubao Voice Input è la funzionalità di riconoscimento vocale integrata di HagiCode, supportata dalla piattaforma aperta ByteDance Doubao. Tu parli e il sistema trasforma il discorso in testo. HagiCode utilizza anche il contesto attuale del progetto per aiutare a riconoscere il vocabolario del dominio e i termini tecnici in modo più affidabile.

### Casi d'uso chiave

In HagiCode, l'input vocale funziona meglio per:

- **Inserimento rapido di proposte**: crea e invia proposte semplicemente parlando, migliorando significativamente l'efficienza senza digitare
- **Fornitura di commenti**: aggiungi commenti e feedback durante le revisioni del codice o dei documenti con input vocale
- **Risposta ai messaggi**: rispondi rapidamente ai messaggi nelle conversazioni e nelle discussioni senza digitare
- **Creazione di contenuti di lunga durata**: genera rapidamente documentazione di progetto, specifiche tecniche, appunti di riunioni e altri contenuti di lunga durata

### Vantaggi pratici

- **Profonda integrazione con consapevolezza del contesto**: HagiCode si integra profondamente con il riconoscimento vocale Doubao, sfruttando automaticamente il contesto attuale del progetto (come struttura del codice, stack tecnologico, terminologia del dominio) per fornire un riconoscimento preciso del vocabolario del dominio, migliorando significativamente l'accuratezza del riconoscimento dei termini tecnici
- **Ore di prova gratuita**: la piattaforma Doubao offre 20 ore di riconoscimento gratuito per consentire ai nuovi utenti di sperimentare l'input vocale senza barriere
- **Estremamente veloce**: riconoscimento in tempo reale, vedi il testo mentre parli senza aspettare
- **Alta precisione**: basato sui modelli avanzati di riconoscimento vocale di Doubao combinati con il contesto del progetto per risultati precisi e affidabili
- **Integrazione perfetta**: Direttamente integrato nella casella di immissione dei messaggi di HagiCode, senza bisogno di cambiare applicazione
- **Facile da usare**: basta fare clic sul microfono per iniziare, intuitivo e diretto

## Avvio rapido

### Prerequisiti

Prima di utilizzare Doubao Voice Input, devi:

1. **Ottieni un account Doubao Open Platform**
   - Visita [Console di riconoscimento vocale Doubao](https://console.volcengine.com/speech/service/10038)
   - Registrati o accedi al tuo account

2. **Crea un'applicazione e ottieni le credenziali**
   - Creare un'applicazione di riconoscimento vocale sulla piattaforma
   - Ottieni il tuo `APP ID` e `Access Token`

3. **Assicurati la connessione di rete**
   - Il servizio di riconoscimento vocale richiede una connessione di rete
   - Assicurati che il tuo dispositivo possa accedere al servizio API Doubao

### Flusso di utilizzo di base

Ecco i passaggi di base per utilizzare Doubao Voice Input:

1. Ottieni `APP ID` e `Access Token`
2. Configura il riconoscimento vocale nel prodotto
3. Testare la chiave API
4. Trova la casella di input vocale
5. Fare clic sul pulsante del microfono
6. Concedere l'autorizzazione al microfono
7. Inizia a registrare
8. Visualizza i risultati del riconoscimento in tempo reale
9. Fare clic per interrompere la registrazione
10. Conferma che il testo è inserito

## Requisiti tecnici

### Ambiente HTTP

Doubao Voice Input richiede l'esecuzione in un ambiente HTTP/HTTPS.

| Ambiente | Stato del supporto | Descrizione |
|-------------|----------------|-------------|
| **host locale** | ✓ | Ambiente di sviluppo locale (http://localhost o http://127.0.0.1) |
| **Server remoto HTTPS** | ✓ | Applicazioni Web distribuite su reti pubbliche con HTTPS |
| **Server remoto HTTP** | ✗ | Applicazioni Web distribuite su reti pubbliche con HTTP |
| **protocollo file://** | ✗ | L'apertura diretta di file HTML non è supportata |

:::attenzione
Doubao Voice Input presenta le seguenti limitazioni:
1. **HTTPS richiesto per la distribuzione esterna**: a causa delle restrizioni di sicurezza del browser, l'accesso al microfono richiede HTTPS quando si accede da reti pubbliche
2. **Indirizzi locali senza restrizioni**: durante l'utilizzo sono supportati sia HTTP che HTTPS `localhost` o `127.0.0.1`
3. **protocollo file:// non supportato**: il file `file://` l'ambiente del protocollo non può utilizzare connessioni microfono e WebSocket
:::

### Versione desktop

La versione desktop di HagiCode ha un ambiente HTTP locale integrato e supporta completamente l'input vocale. È possibile utilizzare il riconoscimento vocale direttamente senza ulteriori configurazioni.

### Modalità ospite

La modalità host locale supporta l'input vocale:
- Durante l'utilizzo sono supportati sia HTTP che HTTPS `localhost` o `127.0.0.1`
- HTTPS è richiesto quando distribuito su reti pubbliche

## Passaggi di configurazione

### Ottieni l'ID dell'APP

1. Visita [Console di riconoscimento vocale Doubao](https://console.volcengine.com/speech/service/10038)
2. Accedi o registra un account
3. Vai alla console e crea una nuova applicazione di riconoscimento vocale
4. Nella pagina dei dettagli dell'applicazione, trova e copia il file `APP ID`

### Ottieni token di accesso

1. Nella console Doubao Open Platform
2. Vai alla tua applicazione di riconoscimento vocale
3. Trova l'area di gestione delle chiavi API
4. Genera o copia il file `Access Token`

:::nota
Per favore, tieni il tuo `Access Token` sicuro e non condividerlo con altri. Se devi sostituirlo, puoi rigenerarlo sulla Doubao Open Platform.
:::

### Configura in Prodotto

1. Apri l'applicazione HagiCode
2. Vai a **Impostazioni** → **Impostazioni riconoscimento vocale**
3. Compila le seguenti informazioni nel modulo di configurazione:
   - **Fornitore**: Seleziona `doubao` (Doubao)
   - **ID APP**: incolla l'ID APP ottenuto dalla piattaforma Doubao
   - **Token di accesso**: incolla il token di accesso che hai ottenuto dalla piattaforma Doubao
4. (Facoltativo) Regola gli altri parametri di configurazione secondo necessità
5. Fare clic sul pulsante **Test chiave API** per verificare la configurazione
6. Dopo aver effettuato con successo la verifica, la configurazione viene automaticamente salvata nella memoria locale del browser

### Testare la chiave API

Dopo la configurazione, si consiglia di verificare se la chiave API è valida:

1. Fai clic sul pulsante **Test chiave API**
2. Il sistema chiamerà l'interfaccia di test per verificare la configurazione
3. Se la configurazione è corretta, verrà visualizzato un messaggio di successo
4. Se la configurazione non è corretta verrà visualizzato un messaggio di errore. Si prega di verificare:
   - Se l'ID APP e il token di accesso sono corretti
   - Se la connessione di rete è normale
   - Se il token di accesso è scaduto

:::suggerimento
Se il test fallisce, controlla prima se la connessione di rete è normale, quindi conferma che la chiave API della piattaforma aperta Doubao sia valida e non scaduta.
:::

## Utilizzo

### Trova la posizione dell'input vocale

In HagiCode, le caselle di immissione testo con l'icona di un microfono supportano l'input vocale. Questi componenti vengono generalmente chiamati `VoiceTextArea`.

Le posizioni comuni dell'input vocale includono:
- Caselle di immissione dei messaggi
- Aree di modifica del testo
- Vari campi del modulo che richiedono l'immissione di testo

### Avvia il riconoscimento vocale

1. Trova la casella di input con l'icona di un microfono
2. Fare clic sul pulsante del microfono
3. Il browser richiederà l'autorizzazione del microfono
4. Fai clic su **Consenti** per autorizzare l'accesso al microfono

:::nota
Se hai precedentemente autorizzato il microfono, il browser potrebbe non richiedere nuovamente l'autorizzazione. Per verificare o modificare le impostazioni dei permessi, puoi visualizzarle nell'icona del lucchetto a sinistra della barra degli indirizzi del browser.
:::

### Descrizione dello stato della registrazione

Dopo che l'autorizzazione ha avuto esito positivo, il riconoscimento vocale si avvierà automaticamente e potrai vedere:

- **Animazione forma d'onda**: la forma d'onda dinamica dell'onda sonora verrà visualizzata all'interno del pulsante del microfono, indicando che la registrazione è in corso
- **Visualizzazione durata**: la durata della registrazione corrente verrà visualizzata sotto il pulsante
- **Riconoscimento in tempo reale**: il testo riconosciuto verrà temporaneamente visualizzato nella posizione del cursore

### Visualizzazione dei risultati del riconoscimento in tempo reale

Durante la registrazione, il motore di riconoscimento vocale convertirà il tuo discorso in testo in tempo reale:

- Il testo riconosciuto verrà temporaneamente visualizzato in grigio nella casella di input
- I risultati del riconoscimento verranno aggiornati continuamente mentre parli
- Supporta il riconoscimento del cinese mandarino con elevata precisione

:::suggerimento
Per ottenere risultati di riconoscimento migliori, si consiglia di:
- Utilizzare in un ambiente tranquillo
- Parla chiaramente e ad un ritmo moderato
- Evitare interferenze dovute al rumore di fondo
:::

### Interrompere il riconoscimento

Per interrompere il riconoscimento vocale, puoi:

1. **Fai clic sul pulsante del microfono**: fai nuovamente clic sul pulsante per interrompere la registrazione
2. **Fare clic sulla casella di input**: anche facendo clic su altre aree della casella di input si interromperà la registrazione

Dopo l'interruzione, il testo finale riconosciuto verrà inserito formalmente nella casella di input e sarà possibile continuare a modificarlo o inviarlo.
