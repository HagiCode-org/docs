หลังจากการติดตั้งเดสก์ท็อปและการเปิดตัวครั้งแรก Hagicode จะเปิดตัวช่วยสร้างการเริ่มต้นใหม่โดยอัตโนมัติ เวอร์ชันปัจจุบันไม่เป็นไปตามคำแนะนำแบบหลายขั้นตอนแบบยาวแบบเก่าอีกต่อไป แต่จะจัดเป็น 3 ขั้นตอนหลัก คือ เสร็จสิ้นการตั้งฐานราก สร้างอย่างน้อย 1 โครงการ และเก็บจุดเข้าชุมชนไว้ใกล้ ๆ ก่อนเข้าสู่พื้นที่ทำงานจริง

:::note[วิธีการอ่านหน้านี้]
การเขียนใหม่นี้ใช้ไลบรารีภาพหน้าจอที่ได้รับการจัดการภายใต้ `repos/docs/src/content/docs/img/screenshots/quick-start/wizard-setup/` เป็นแหล่งแห่งความจริง ชุดที่ได้รับการจัดการปัจจุบันมีเพียง 3 ภาพหน้าจอสำหรับ `Foundation settings` ดังนั้นหน้านี้จึงอธิบายส่วนนั้นโดยละเอียดและเก็บ 2 สเตจหลังไว้ในระดับสูง
:::

## ภาพรวมของวิซาร์ด

วิซาร์ดการเริ่มต้นในปัจจุบันจะเคลื่อนผ่าน 3 ขั้นตอนหลักเหล่านี้:

1. **การตั้งค่าพื้นฐาน**: เลือกภาษาอินเทอร์เฟซ เลือกธีม ตรวจสอบการขึ้นต่อกัน และสร้างเอเจนต์เริ่มต้นเมื่อไม่มีอยู่
2. **สร้างโปรเจ็กต์**: สร้างอย่างน้อยหนึ่งโปรเจ็กต์เพื่อให้เอาต์พุตของวิซาร์ดเชื่อมต่อกับไดเร็กทอรีการทำงานจริง
3. **คู่มือชุมชน**: รักษา GitHub และจุดเข้าติดตามผลอื่นๆ ให้อยู่ใกล้แค่เอื้อมก่อนเข้าสู่พื้นที่ทำงานปกติ

## ขั้นตอนที่ 1: การตั้งค่ารากฐาน

การนำทางด้านซ้ายติดป้ายกำกับระยะแรกเป็นแล้ว `基础设置` และสรุปวัตถุประสงค์อย่างชัดเจน: เลือกภาษาอินเทอร์เฟซ เก็บธีมสปริง ยืนยัน `node`, `npm`, `git`และ `OpenSpec`และสร้างค่าเริ่มต้น `Hagicoder` ถ้ายังไม่มีฮีโร่อยู่ สรุป: ขั้นตอนนี้จะผสานงานเตรียมการแบบเก่าที่กระจัดกระจายเข้าเป็นพื้นผิวการตั้งค่าต่อเนื่องเดียว

### 1.1 เลือกภาษาและธีมของอินเทอร์เฟซ

![การตั้งค่าพื้นฐานตัวช่วยสร้างการเริ่มต้น: ภาษาและธีม](../../img/screenshots/quick-start/wizard-setup/guide1-language-theme/original.png)

ภาพหน้าจอแรกแสดงด้านบนของหน้าจอการตั้งค่ารากฐาน มันเริ่มต้นด้วย `选择界面语言`และข้อความตัวช่วยทำให้ขอบเขตชัดเจน: สิ่งนี้จะเปลี่ยน **สำเนาอินเทอร์เฟซเท่านั้น** ไม่ใช่รูปแบบภาษาของฮีโร่ ในตัวอย่าง `简体中文` ถูกทำเครื่องหมายเป็น `当前`ในขณะที่ `English` ยังคงมีอยู่ทางด้านขวา

หน้าจอเดิมยังคงดำเนินต่อไปในทันที `选择主题`. โครงสร้างปัจจุบันเผยให้เห็นแสงสปริงและความมืดของฤดูใบไม้ผลิ และตัวอย่างจะทำเครื่องหมายการ์ดธีมที่ถูกต้องเป็น `当前`. การเปลี่ยนแปลงที่สำคัญคือโครงสร้าง: ขณะนี้ภาษาและธีมได้รับการจัดการเป็นส่วนต่อเนื่องกัน แทนที่จะแยกการข้ามจากคำแนะนำแบบเก่า

### 1.2 ตรวจสอบการขึ้นต่อกันและเก็บการดำเนินการติดตั้งไว้ใกล้เคียง

![การตั้งค่าพื้นฐานตัวช่วยสร้างการเริ่มต้น: การตรวจสอบการพึ่งพา](../../img/screenshots/quick-start/wizard-setup/guide2-dependency-check/original.png)

ภาพหน้าจอที่สองเลื่อนเข้าไป `组件检测`. พื้นที่นี้จะตรวจสอบโดยอัตโนมัติ `Node.js`, `npm`, `Git`และ `OpenSpec`และแสดงผลแต่ละรายการด้วยความพร้อมใช้งานในปัจจุบัน มุมบนขวาเก็บ a `重新检测` การกระทำ และ `OpenSpec` แถวยังเก็บ `安装 OpenSpec` รายการ

ตรรกะนั้นตรงไปตรงมา: ตัวช่วยสร้างจะบอกคุณว่าเครื่องปัจจุบันพร้อมหรือไม่ แทนที่จะบังคับให้คุณจำข้อกำหนดเบื้องต้นทั้งหมดด้วยตนเอง เมื่อส่วนประกอบหลักเหล่านั้นพร้อมใช้งาน คุณสามารถดำเนินการสร้างตัวแทนตามค่าเริ่มต้นต่อไปได้ จากนั้นจึงไปยังการสร้างโครงการต่อไป

### 1.3 เลือก CLI และโมเดลที่ใช้มากที่สุด จากนั้นสร้าง Agent เริ่มต้น

![การตั้งค่าพื้นฐานตัวช่วยสร้างการเริ่มต้น: คำแนะนำตัวแทนเริ่มต้น](../../img/screenshots/quick-start/wizard-setup/guide3-default-agent-cli-model/original.png)

ภาพหน้าจอที่สามแสดงให้เห็น `默认Agent引导`. เมื่อยังไม่มีตัวแทน ตัวช่วยสร้างจะถามถึงตัวแทนที่ใช้มากที่สุด `AgentCLI` และจำลองก่อน จากนั้นจะสร้างตัวแทนเริ่มต้นโดยอัตโนมัติและนำไปไว้ในกลุ่มการกำหนดค่าในภายหลัง ในตัวอย่าง ค่าเริ่มต้นที่เลือกคือ `Claude Code` และ `GLM 5.1`.

หน้าจอนี้ยังจัดการข้อมูลระบุตัวตนเริ่มต้นด้วย พื้นที่ด้านล่างแสดงตัวอย่างอวาตาร์ นำเสนอ `换一组` และวาง CTA หลักไว้ที่มุมขวาล่างเป็น `确认并创建 Hagicoder`. ดังนั้นวิซาร์ดปัจจุบันจะไม่เปิดตารางบทบาทแบบหลายหน้าจอแบบเก่าอีกต่อไป โดยจะรับประกันก่อนว่ามีตัวดำเนินการเริ่มต้นที่ใช้งานได้หนึ่งตัว

## ขั้นตอนที่ 2: สร้างโครงการ

การนำทางด้านซ้ายจะติดป้ายกำกับระยะที่สองเป็น `创建项目`. สรุประดับหน้าทำให้บทบาทของตนชัดเจนแล้ว: นี่ไม่ใช่การหยุดทางเลือกทั่วไป คุณได้รับการคาดหวังให้สร้าง **อย่างน้อยหนึ่งโปรเจ็กต์** เพื่อให้ภาษา ธีม สถานะการพึ่งพา และเอเจนต์เริ่มต้นที่สร้างขึ้นในการตั้งค่าพื้นฐานสามารถแนบไปกับไดเร็กทอรีการทำงานจริงได้

ชุดภาพหน้าจอที่ได้รับการจัดการในปัจจุบันยังไม่มีภาพหน้าจอเฉพาะสำหรับขั้นตอนนี้ ดังนั้นหน้านี้จึงเก็บคำอธิบายไว้ในระดับสูงไว้ในขณะนี้: เมื่อคุณไปถึงแล้ว ให้เชื่อมต่อพื้นที่เก็บข้อมูลจริงหรือโฟลเดอร์ทำงานก่อนที่จะดำเนินการต่อ

## ขั้นตอนที่ 3: คู่มือชุมชน

ขั้นตอนที่สามในการนำทางด้านซ้ายคือ `社区引导`. สรุปส่วนหัวระบุลำดับอย่างชัดเจน: เสร็จสิ้นการตั้งค่ารากฐาน สร้างอย่างน้อยหนึ่งโครงการ และ **เก็บจุดเข้าใช้งานของชุมชนไว้ใกล้เคียง** ก่อนเข้าสู่พื้นที่ทำงานจริง

เนื่องจากชุดภาพหน้าจอที่ได้รับการจัดการในปัจจุบันยังขาดภาพหน้าจอเฉพาะสำหรับขั้นตอนนี้ เอกสารจึงเก็บเฉพาะความหมายโฟลว์ไว้ที่นี่: เป็นขั้นตอนปิดสำหรับจุดเข้าติดตามผล ไม่ใช่ที่สำหรับการตัดสินใจในการเริ่มต้นแกนหลัก
