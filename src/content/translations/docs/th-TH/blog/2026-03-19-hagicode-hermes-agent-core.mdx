## เหตุใด HagiCode จึงเลือก Hermes เป็นแกนหลักแบบบูรณาการ

> เมื่อสร้างแพลตฟอร์มการเข้ารหัสที่ได้รับความช่วยเหลือจาก AI การเลือกแกนหลักของตัวแทนที่เหมาะสมจะเป็นตัวกำหนดขีดจำกัดสูงสุดของขีดความสามารถของระบบโดยตรง บางสิ่งก็ไม่สามารถบังคับได้ เลือกกรอบการทำงานที่ไม่ถูกต้อง และไม่มีความพยายามใดที่จะทำให้รู้สึกว่าถูกต้อง บทความนี้แบ่งปันความคิดเบื้องหลังการคัดเลือกทางเทคนิคของ HagiCode และประสบการณ์ตรงของเราในการบูรณาการ Hermes Agent

## พื้นหลัง

เมื่อสร้างผลิตภัณฑ์การเขียนโค้ดที่ได้รับความช่วยเหลือจาก AI หนึ่งในส่วนที่ยากที่สุดคือการเลือกเฟรมเวิร์กตัวแทนพื้นฐาน จริงๆ แล้วมีตัวเลือกค่อนข้างน้อยในตลาด แต่บางตัวมีฟังก์ชันการทำงานที่จำกัดเกินไป บางตัวมีความซับซ้อนมากเกินไปในการปรับใช้ และบางตัวก็ปรับขนาดได้ไม่ดีพอ สิ่งที่เราต้องการคือโซลูชันที่สามารถทำงานบน VPS มูลค่า $5 ในขณะที่ยังสามารถเชื่อมต่อกับคลัสเตอร์ GPU ได้ด้วย ข้อกำหนดนั้นอาจฟังดูไม่สุดโต่ง แต่ก็เพียงพอที่จะทำให้หลายทีมหวาดกลัว

ในทางปฏิบัติ สิ่งที่เรียกว่า "เอเจนต์ออลอินวัน" จำนวนมากอาจทำงานบนคลาวด์เท่านั้นหรือต้องใช้ต้นทุนการปรับใช้ภายในที่สูงจนเกินไป หลังจากใช้เวลาสองสัปดาห์ในการค้นคว้าแนวทางต่างๆ เราก็ตัดสินใจอย่างกล้าหาญ: สร้างแกนหลักเอเจนต์ใหม่โดยมี Hermes เป็นกลไกพื้นฐานสำหรับเอเจนต์ที่ผสานรวมของเรา

ทุกสิ่งที่ตามมาอาจเป็นเพียงโชคชะตา

## ทำไม HagiCode ถึงต้องการ Hermes

ก่อนที่จะดำดิ่งลงไปใน Hermes จะช่วยอธิบายได้ว่าเหตุใด HagiCode จึงต้องการบางสิ่งที่คล้ายกันตั้งแต่แรก สิ่งต่างๆ ไม่ค่อยทำงานตามที่คุณต้องการ ดังนั้นคุณจึงต้องมีเหตุผลเชิงปฏิบัติเพื่อมุ่งมั่นในทิศทางทางเทคนิค

ในฐานะผู้ช่วยเขียนโค้ด AI HagiCode จำเป็นต้องรองรับสถานการณ์การใช้งานหลายอย่างในเวลาเดียวกัน:
- สภาพแวดล้อมการพัฒนาในท้องถิ่น: นักพัฒนาต้องการรันบนเครื่องของตนเอง เพื่อให้ข้อมูลไม่ออกจากสภาพแวดล้อมท้องถิ่น ทุกวันนี้ ความปลอดภัยของข้อมูลไม่ใช่เรื่องสำคัญอีกต่อไป
- สภาพแวดล้อมการทำงานร่วมกันเป็นทีม: ทีมขนาดเล็กควรสามารถแชร์การปรับใช้ Agent ที่ทำงานบนเซิร์ฟเวอร์ได้ การออมเงินเป็นเรื่องสำคัญ และทุกคนก็มีขีดจำกัด
- การขยายระบบคลาวด์แบบยืดหยุ่น: เมื่อจัดการงานที่ซับซ้อน ระบบควรขยายขนาดเป็นคลัสเตอร์ GPU โดยอัตโนมัติ เป็นการดีกว่าเสมอที่จะเตรียมพร้อม

ข้อกำหนด "เราต้องการทุกสิ่งในคราวเดียว" นี้เป็นสิ่งที่นำเราไปสู่ Hermes ไม่ว่าจะเป็นตัวเลือกที่สมบูรณ์แบบหรือไม่ ฉันไม่สามารถพูดได้อย่างแน่นอน แต่ในเวลานั้นเราไม่เห็นตัวเลือกที่ดีกว่า

## ตัวแทน Hermes คืออะไร

Hermes Agent คือ AI Agent อัตโนมัติที่สร้างโดย **Nous Research** ผู้อ่านบางคนอาจไม่คุ้นเคยกับ Nous Research; เป็นห้องแล็บที่อยู่เบื้องหลังโมเดลโอเพ่นซอร์สขนาดใหญ่ เช่น Hermes, Nomos และ Psyché พวกเขาได้สร้างสิ่งที่ยอดเยี่ยมมากมาย แม้ว่าพวกเขาจะยังด้อยค่าเกินกว่าที่พวกเขาสมควรได้รับก็ตาม

แตกต่างจากผู้ช่วยเขียนโค้ด IDE แบบดั้งเดิมหรือโปรแกรมแชท API ทั่วไป Hermes มีคุณสมบัติที่กำหนด: ยิ่งทำงานนานเท่าไรก็ยิ่งมีความสามารถมากขึ้นเท่านั้น ไม่ได้ออกแบบมาเพื่อทำงานให้เสร็จสิ้นเพียงครั้งเดียวและหยุดลง มันคอยเรียนรู้และสั่งสมประสบการณ์จากการดำเนินงานที่ยาวนาน ในแง่นั้นมันให้ความรู้สึกเหมือนคนนิดหน่อย

### คุณสมบัติหลัก

ความสามารถหลักหลายประการของ Hermes นั้นสอดคล้องกับความต้องการของ HagiCode อย่างใกล้ชิด

ซึ่งหมายความว่า HagiCode สามารถเลือกรูปแบบการใช้งานที่เหมาะสมที่สุดตามสถานการณ์ของผู้ใช้แต่ละราย: แต่ละบุคคลเรียกใช้งานภายในเครื่อง ทีมปรับใช้บนเซิร์ฟเวอร์ และงานที่ซับซ้อนใช้ทรัพยากร GPU codebase เดียวจัดการได้ทั้งหมด ในโลกที่ยุ่งวุ่นวายนี้ การลดความซับซ้อนลงหนึ่งชั้นถือเป็นชัยชนะแล้ว

**เกตเวย์การส่งข้อความหลายแพลตฟอร์ม**
Hermes รองรับ Telegram, Discord, Slack, WhatsApp และอีกมากมายโดยกำเนิด สำหรับ HagiCode หมายความว่าเราสามารถสนับสนุนผู้ช่วย AI บนช่องทางเหล่านั้นได้ง่ายขึ้นมากในอนาคต ยินดีต้อนรับเส้นทางข้างหน้าเพิ่มเติมเสมอ

**ระบบเครื่องมือครบครัน**
Hermes มาพร้อมกับเครื่องมือในตัวมากกว่า 40 รายการ และรองรับส่วนขยาย MCP (Model Context Protocol) นี่เป็นสิ่งสำคัญสำหรับผู้ช่วยเขียนโค้ด: การรันคำสั่งเชลล์ การทำงานกับระบบไฟล์ และการเรียก Git ล้วนขึ้นอยู่กับการสนับสนุนของเครื่องมือ เจ้าหน้าที่ที่ไม่มีเครื่องมือก็เหมือนกับนกที่ไม่มีปีก

**หน่วยความจำข้ามเซสชัน**
Hermes มีระบบหน่วยความจำถาวรและใช้การค้นหาข้อความแบบเต็ม FTS5 เพื่อระลึกถึงการสนทนาในอดีต ซึ่งช่วยให้ Agent จดจำบริบทก่อนหน้าแทนที่จะ "สูญเสียความทรงจำ" ทุกครั้ง บางครั้งผู้คนก็หวังว่าพวกเขาจะลืมสิ่งต่าง ๆ ได้อย่างง่ายดาย แต่ความเป็นจริงมักจะไม่ค่อยมีน้ำใจ

## HagiCode ผสานรวม Hermes อย่างไร

เมื่อ "ทำไม" ชัดเจนแล้ว เรามาดู "อย่างไร" กันดีกว่า เมื่อบางสิ่งสมเหตุสมผลในทางทฤษฎีแล้ว ขั้นตอนต่อไปคือการสร้างมันขึ้นมา

### ผู้ให้บริการเลเยอร์นามธรรม

ในสถาปัตยกรรมของ HagiCode ผู้ให้บริการ AI ทุกรายจะใช้ระบบแบบครบวงจร `IAIProvider` อินเตอร์เฟซ:

```csharp
public sealed class HermesCliProvider : IAIProvider, IVersionedAIProvider
{
    public ProviderCapabilities Capabilities { get; } = new ProviderCapabilities
    {
        SupportsStreaming = true,   // Supports streaming output
        SupportsTools = true,       // Supports tool invocation
        SupportsSystemMessages = true, // Supports system prompts
        SupportsArtifacts = false
    };
}
```

เลเยอร์นามธรรมนี้ช่วยให้ HagiCode สลับระหว่างผู้ให้บริการ AI ต่างๆ ได้อย่างราบรื่น ไม่ว่าแบ็กเอนด์จะเป็น OpenAI, Claude หรือ Hermes รูปแบบการเรียกชั้นบนจะยังคงเหมือนเดิมทุกประการ พูดง่ายๆ ก็คือ มันทำให้ทุกอย่างเรียบง่าย

### โปรโตคอลการสื่อสาร ACP

Hermes สื่อสารผ่าน **ACP (Agent Communication Protocol)** โปรโตคอลนี้ได้รับการออกแบบมาโดยเฉพาะสำหรับการสื่อสารของตัวแทน และวิธีการหลักได้แก่:

| วิธีการ | คำอธิบาย |
|------|------|
| `initialize` | เริ่มต้นการเชื่อมต่อและรับเวอร์ชันโปรโตคอลและความสามารถของไคลเอ็นต์ |
| `authenticate` | จัดการการรับรองความถูกต้องและรองรับวิธีการรับรองความถูกต้องหลายวิธี |
| `session/new` | สร้างเซสชันใหม่และกำหนดค่าไดเร็กทอรีการทำงานและเซิร์ฟเวอร์ MCP |
| `session/prompt` | ส่งข้อความแจ้งและรับการตอบกลับ |

HagiCode ใช้งานเลเยอร์การขนส่ง ACP ผ่าน `StdioAcpTransport`เรียกใช้กระบวนการย่อย Hermes และสื่อสารกับกระบวนการดังกล่าวผ่านอินพุตและเอาต์พุตมาตรฐาน อาจฟังดูซับซ้อน แต่ในทางปฏิบัติสามารถจัดการได้ตราบใดที่คุณมีความอดทนเพียงพอ

### การจัดการการกำหนดค่า

การกำหนดค่าได้รับการจัดการผ่าน `HermesPlatformConfiguration` ชั้นเรียน:

```csharp
public sealed class HermesPlatformConfiguration : IAcpPlatformConfiguration
{
    public string ExecutablePath { get; set; } = "hermes";
    public string Arguments { get; set; } = "acp";
    public int StartupTimeoutMs { get; set; } = 5000;
    public string ClientName { get; set; } = "HagiCode";
    public HermesAuthenticationConfiguration Authentication { get; set; }
    public HermesSessionDefaultsConfiguration SessionDefaults { get; set; }
}
```

กำหนดค่า Hermes ใน `appsettings.json`:

```json
{
  "Providers": {
    "HermesCli": {
      "ExecutablePath": "hermes",
      "Arguments": "acp",
      "StartupTimeoutMs": 10000,
      "ClientName": "HagiCode",
      "Authentication": {
        "PreferredMethodId": "api-key",
        "MethodInfo": {
          "api-key": "your-api-key-here"
        }
      },
      "SessionDefaults": {
        "Model": "claude-sonnet-4-20250514",
        "ModeId": "default"
      }
    }
  }
}
```

การกำหนดค่ามักจะดูเรียบง่ายบนกระดาษ แต่การทำให้ทุกรายละเอียดถูกต้องยังคงต้องใช้ความพยายามอย่างแท้จริง

### สถาปัตยกรรมกระจายของออร์ลีนส์

HagiCode ใช้ Orleans เพื่อสร้างระบบแบบกระจาย และการนำ Hermes มาใช้งานผ่านส่วนประกอบต่อไปนี้:

- **HermesGrain**: การใช้งาน Orleans Grain ที่จัดการการประมวลผลเซสชัน
- **HermesPlatformConfiguration**: การกำหนดค่าเฉพาะแพลตฟอร์ม
- **HermesAcpSessionAdapter**: อะแดปเตอร์เซสชัน ACP
- **HermesConsole**: คอนโซลการตรวจสอบเฉพาะ

ชื่อออร์ลีนส์มีเสน่ห์บางอย่าง แม้ว่าออร์ลีนส์แห่งนี้จะไม่เกี่ยวข้องกับเมืองในตำนาน แต่ชื่อที่ดีก็ไม่เคยทำให้เจ็บปวด

### ขั้นตอนการดำเนินการแบบ end-to-end

ต่อไปนี้เป็นตรรกะการดำเนินการหลักของ Hermes Provider:

```csharp
private async IAsyncEnumerable<AIStreamingChunk> StreamCoreAsync(
    AIRequest request,
    string? embeddedCommandPrompt,
    [EnumeratorCancellation] CancellationToken cancellationToken)
{
    // 1. Create transport layer and launch Hermes subprocess
    await using var transport = new StdioAcpTransport(
        platformConfiguration.GetExecutablePath(),
        platformConfiguration.GetArguments(),
        platformConfiguration.GetEnvironmentVariables(),
        platformConfiguration.GetStartupTimeout(),
        _loggerFactory.CreateLogger<StdioAcpTransport>());
    await transport.ConnectAsync(cancellationToken);

    // 2. Initialize and obtain protocol version and authentication methods
    var initializeResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "initialize",
        BuildInitializeParameters(platformConfiguration), cancellationToken);

    // 3. Handle authentication
    var authMethods = ParseAuthMethods(initializeResult);
    if (!isAuthenticated)
    {
        var methodId = platformConfiguration.Authentication.ResolveMethodId(authMethods);
        await SendHermesRequestAsync(transport, nextRequestId++, "authenticate", ...);
    }

    // 4. Create session
    var newSessionResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "session/new",
        BuildNewSessionParameters(platformConfiguration, workingDirectory, model), cancellationToken);
    var sessionId = ParseSessionId(newSessionResult);

    // 5. Execute prompt and collect streaming responses
    await foreach (var payload in transport.ReceiveMessagesAsync(cancellationToken))
    {
        // Handle session/update notifications and convert them into streaming chunks
        if (TryParseSessionNotification(root, out var notification))
        {
            if (_responseMapper.TryConvertToStreamingChunk(notification, out var chunk))
            {
                yield return chunk;
            }
        }
    }
}
```

ด้วยโค้ด รายละเอียดต่างๆ จะกลายเป็นความคุ้นเคยในที่สุด สิ่งที่สำคัญที่สุดคือแนวทางโดยรวม

### การตรวจสุขภาพ

เพื่อให้แน่ใจว่า Hermes ยังคงมีอยู่ HagiCode จึงใช้กลไกการตรวจสุขภาพ:

```csharp
public async Task<ProviderTestResult> PingAsync(CancellationToken cancellationToken = default)
{
    var response = await ExecuteAsync(
        new AIRequest
        {
            Prompt = "Reply with exactly PONG.",
            CessionId = null,
            AllowedTools = Array.Empty<string>(),
            WorkingDirectory = ResolveWorkingDirectory(null)
        },
        cancellationToken);

    var success = string.Equals(response.Content.Trim(), "PONG", StringComparison.OrdinalIgnoreCase);
    return new ProviderTestResult
    {
        ProviderName = Name,
        Success = success,
        ResponseTimeMs = stopwatch.ElapsedMilliseconds,
        ErrorMessage = success ? null : $"Unexpected Hermes ping response: '{response.Content}'."
    };
}
```

"การตรวจสุขภาพ" หน้าตาประมาณนี้ ในบางแง่ ผู้คนก็ไม่ได้แตกต่างกันมากนัก การเช็คอินเป็นครั้งคราวจะช่วยได้มาก แม้ว่าจะไม่มีใครบอกเราอย่างชัดเจนว่าจะมองหาอะไรก็ตาม

## ข้อควรพิจารณาในทางปฏิบัติ

มีข้อผิดพลาดบางประการที่ควรทำความเข้าใจก่อนที่จะรวม Hermes ทุกคนก้าวเข้าสู่กับดักไม่ช้าก็เร็ว

### การกำหนดค่าวิธีการรับรองความถูกต้อง

Hermes รองรับวิธีการรับรองความถูกต้องหลายวิธี รวมถึงคีย์ API และโทเค็น ดังนั้นคุณจึงต้องเลือกตามสถานการณ์การใช้งานจริง การกำหนดค่าที่ไม่ถูกต้องอาจทำให้เกิดความล้มเหลวในการเชื่อมต่อ และข้อความแสดงข้อผิดพลาดที่เกิดขึ้นอาจไม่ได้เป็นไปตามธรรมชาติเสมอไป บางครั้งข้อผิดพลาดที่รายงานอยู่ห่างไกลจากสาเหตุที่แท้จริง ซึ่งหมายความว่าการดีบักที่ช้าและระมัดระวังเป็นสิ่งที่หลีกเลี่ยงไม่ได้

### การกำหนดค่าเซิร์ฟเวอร์ MCP

เมื่อสร้างเซสชัน คุณสามารถกำหนดค่ารายการเซิร์ฟเวอร์ MCP เพื่อให้ Hermes สามารถเรียกใช้เครื่องมือภายนอกได้ แต่โปรดคำนึงถึงประเด็นต่อไปนี้:
- ต้องเข้าถึงที่อยู่เซิร์ฟเวอร์ MCP ได้
- การหมดเวลาจะต้องได้รับการกำหนดค่าอย่างสมเหตุสมผล
- ระบบต้องการการจัดการการลดประสิทธิภาพเมื่อเซิร์ฟเวอร์ไม่พร้อมใช้งาน

ในทางปฏิบัติ การคิดเชิงป้องกันมีความสำคัญมากกว่าที่ผู้คนคาดหวัง

### การจัดการสารบบการทำงาน

แต่ละเซสชันจะต้องระบุไดเร็กทอรีการทำงานเพื่อให้ Hermes สามารถเข้าถึงไฟล์โปรเจ็กต์ได้อย่างถูกต้อง ในสถานการณ์หลายโครงการ ไดเร็กทอรีการทำงานจำเป็นต้องสลับแบบไดนามิก ฟังดูตรงไปตรงมา แต่มีกรณีขอบมากกว่าที่คุณคิด

### การรวมการตอบสนอง

คำตอบของ Hermes อาจถูกแบ่งออกเป็นส่วนๆ `session/update` การแจ้งเตือนและผลลัพธ์สุดท้ายจึงต้องรวมเข้าด้วยกันอย่างถูกต้อง มิฉะนั้นเนื้อหาอาจสูญหายได้

### กลยุทธ์การจัดการข้อผิดพลาด

ข้อผิดพลาดรันไทม์ควรส่งคืนอย่างชัดเจนแทนที่จะถอยกลับไปยังผู้ให้บริการรายอื่นโดยไม่แจ้งให้ทราบ ด้วยวิธีนี้ ผู้ใช้จะทราบว่าปัญหามาจาก Hermes แทนที่จะสงสัยว่าเหตุใดระบบจึงเปลี่ยนรุ่นเบื้องหลังกะทันหัน

## บทสรุป

การตัดสินใจของ HagiCode ที่จะใช้ Hermes เป็น Agent core ที่บูรณาการเข้าด้วยกันนั้นไม่ใช่แรงกระตุ้นทั่วไป เป็นทางเลือกที่ระมัดระวังโดยพิจารณาจากข้อกำหนดในทางปฏิบัติและคุณลักษณะทางเทคนิคของกรอบงาน ไม่ว่ามันจะพิสูจน์ได้ว่าเป็นคำตอบที่สมบูรณ์แบบในระยะยาวหรือไม่นั้นยังเร็วเกินไปที่จะพูด แต่จนถึงตอนนี้มันก็ให้บริการเราได้ดี

Hermes ช่วยให้ HagiCode มีความยืดหยุ่นในการปรับให้เข้ากับสถานการณ์ที่หลากหลาย ระบบเครื่องมืออันทรงพลังและการรองรับ MCP ช่วยให้ผู้ช่วย AI ทำงานจริงได้ ในขณะที่โปรโตคอล ACP และเลเยอร์นามธรรมของผู้ให้บริการทำให้กระบวนการบูรณาการมีความชัดเจนและควบคุมได้

หากคุณกำลังเลือกเฟรมเวิร์กตัวแทนสำหรับโปรเจ็กต์ AI ของคุณเอง ฉันหวังว่าบทความนี้จะเป็นข้อมูลอ้างอิงที่เป็นประโยชน์ การเลือกสถาปัตยกรรมพื้นฐานที่เหมาะสมจะทำให้ทุกสิ่งที่ตามมาง่ายขึ้นมาก
