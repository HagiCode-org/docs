## วิธีสร้างโปรเจ็กต์ในยุค AI: Vault ซึ่งเป็นระบบจัดเก็บข้อมูลถาวรข้ามโปรเจ็กต์

> ในยุคของการพัฒนาโดยใช้ AI เราจะช่วยให้ผู้ช่วย AI เข้าใจแหล่งการเรียนรู้ของเราดีขึ้นได้อย่างไร โปรเจ็กต์ HagiCode สร้างระบบ Vault ให้เป็นเลเยอร์นามธรรมของการจัดเก็บความรู้แบบครบวงจรที่ AI สามารถเข้าใจได้ ซึ่งช่วยปรับปรุงประสิทธิภาพของการเรียนรู้ผ่านการทำซ้ำโปรเจ็กต์ได้อย่างมาก

## พื้นหลัง

ในยุค AI วิธีที่นักพัฒนาเรียนรู้เทคโนโลยีและสถาปัตยกรรมใหม่ๆ กำลังเปลี่ยนแปลงไปอย่างมาก "การทำซ้ำโครงการ" ซึ่งก็คือการศึกษาและการเรียนรู้อย่างลึกซึ้งจากโค้ด สถาปัตยกรรม และรูปแบบการออกแบบของโครงการโอเพ่นซอร์สที่ยอดเยี่ยม ได้กลายเป็นวิธีการเรียนรู้ที่มีประสิทธิภาพ เมื่อเปรียบเทียบกับวิธีการดั้งเดิม เช่น การอ่านหนังสือหรือดูวิดีโอ การอ่านและดำเนินโครงการโอเพ่นซอร์สคุณภาพสูงโดยตรงจะช่วยให้คุณเข้าใจแนวทางปฏิบัติด้านวิศวกรรมในโลกแห่งความเป็นจริงได้เร็วยิ่งขึ้นมาก

อย่างไรก็ตาม วิธีการเรียนรู้นี้มาพร้อมกับความท้าทายค่อนข้างมาก

สื่อการเรียนรู้กระจัดกระจายเกินไป บันทึกของคุณอาจอยู่ใน Obsidian ที่เก็บโค้ดอาจกระจัดกระจายไปตามโฟลเดอร์ต่างๆ และประวัติการสนทนาของผู้ช่วย AI ของคุณก็กลายเป็นเกาะข้อมูลที่แยกออกมาอีกแห่งหนึ่ง เมื่อคุณต้องการให้ AI ช่วยวิเคราะห์โปรเจ็กต์ คุณจะต้องคัดลอกโค้ดและจัดระเบียบบริบทด้วยตนเอง ซึ่งค่อนข้างน่าเบื่อ

สิ่งที่ลำบากยิ่งกว่านั้นคือบริบทที่เสียหาย ผู้ช่วย AI ไม่สามารถเข้าถึงแหล่งข้อมูลการเรียนรู้ในท้องถิ่นของคุณได้โดยตรง ดังนั้นคุณต้องให้ข้อมูลพื้นฐานอีกครั้งในทุกการสนทนา ยิ่งไปกว่านั้น ที่เก็บโค้ดที่ทำซ้ำจะอัปเดตอย่างรวดเร็ว การซิงค์ด้วยตนเองมีแนวโน้มที่จะเกิดข้อผิดพลาด และความรู้นั้นยากที่จะแบ่งปันในโปรเจ็กต์การเรียนรู้หลายรายการ

โดยต้นตอของปัญหาเหล่านี้ทั้งหมดมาจาก "หมู่เกาะข้อมูล" หากมีชั้นนามธรรมการจัดเก็บข้อมูลแบบรวมที่อนุญาตให้ผู้ช่วย AI เข้าใจและเข้าถึงทรัพยากรการเรียนรู้ทั้งหมดของคุณ ปัญหาก็จะได้รับการแก้ไขอย่างเรียบร้อย

## เกี่ยวกับ ฮากิโค้ด

ระบบห้องนิรภัยที่แชร์ในบทความนี้เป็นโซลูชันที่เราพัฒนาในขณะที่สร้าง [ฮากิโค้ด](https://hagicode.com). HagiCode เป็นโปรเจ็กต์ผู้ช่วยเขียนโค้ด AI และในงานพัฒนาประจำวันของเรา เรามักจะต้องศึกษาและอ้างอิงถึงโปรเจ็กต์โอเพ่นซอร์สต่างๆ มากมาย เพื่อช่วยให้ผู้ช่วย AI เข้าใจแหล่งข้อมูลการเรียนรู้เหล่านี้ได้ดีขึ้น เราได้ออกแบบ Vault ซึ่งเป็นระบบจัดเก็บข้อมูลถาวรข้ามโปรเจ็กต์

โซลูชันนี้ได้รับการตรวจสอบแล้วใน HagiCode ในการใช้งานจริง หากคุณกำลังเผชิญกับความท้าทายในการจัดการความรู้ที่คล้ายกัน ฉันหวังว่าประสบการณ์เหล่านี้จะเป็นแรงบันดาลใจได้บ้าง ท้ายที่สุดแล้ว เมื่อคุณตกลงไปในหลุมเล็กๆ น้อยๆ ด้วยตัวเองแล้ว คุณควรทิ้งบางอย่างไว้ให้กับคนต่อไป

## ปรัชญาการออกแบบระบบห้องนิรภัย

แนวคิดหลักของระบบ Vault นั้นเรียบง่าย: สร้างเลเยอร์นามธรรมการจัดเก็บข้อมูลความรู้แบบครบวงจรที่ AI สามารถเข้าใจได้ จากมุมมองของการนำไปปฏิบัติ ระบบมีลักษณะสำคัญหลายประการ

### การสนับสนุนหลายประเภท

ระบบรองรับ Vault สี่ประเภท แต่ละประเภทสอดคล้องกับสถานการณ์การใช้งานที่แตกต่างกัน:

```typescript
// folder: general-purpose folder type
export const DEFAULT_VAULT_TYPE = 'folder';

// coderef: a type specifically for reproduced code projects
export const CODEREF_VAULT_TYPE = 'coderef';

// obsidian: integrated with Obsidian note-taking software
export const OBSIDIAN_VAULT_TYPE = 'obsidian';

// system-managed: vault automatically managed by the system
export const SYSTEM_MANAGED_VAULT_TYPE = 'system-managed';
```

ในบรรดาประเภทเหล่านั้น ประเภท **coderef** เป็นประเภทที่ใช้กันมากที่สุดใน HagiCode ได้รับการออกแบบมาเป็นพิเศษสำหรับโปรเจ็กต์โค้ดที่ทำซ้ำ โดยมีโครงสร้างไดเร็กทอรีที่เป็นมาตรฐานและคำอธิบายเมตาดาต้าที่ AI อ่านได้

### กลไกการจัดเก็บแบบถาวร

รีจิสทรีของห้องนิรภัยจะถูกจัดเก็บอย่างต่อเนื่องในรูปแบบ JSON เพื่อให้มั่นใจว่าการกำหนดค่าจะยังคงใช้งานได้หลังจากที่แอปพลิเคชันรีสตาร์ท:

```csharp
public class VaultRegistryStore : IVaultRegistryStore
{
    private readonly string _registryFilePath;

    public VaultRegistryStore(IConfiguration configuration, ILogger<VaultRegistryStore> logger)
    {
        var dataDir = configuration["DataDir"] ?? "./data";
        var absoluteDataDir = Path.IsPathRooted(dataDir)
            ? dataDir
            : Path.GetFullPath(Path.Combine(Directory.GetCurrentDirectory(), dataDir));

        _registryFilePath = Path.Combine(absoluteDataDir, "personal-data", "vaults", "registry.json");
    }
}
```

ข้อดีของการออกแบบนี้คือเรียบง่ายและเชื่อถือได้ JSON นั้นมนุษย์สามารถอ่านได้ ซึ่งทำให้การดีบักและการแก้ไขด้วยตนเองง่ายขึ้น ที่เก็บข้อมูลระบบไฟล์หลีกเลี่ยงความซับซ้อนของฐานข้อมูลและลดการพึ่งพาระบบ ท้ายที่สุดแล้ว บางครั้งตัวเลือกที่ง่ายที่สุดก็คือตัวเลือกที่ดีที่สุดจริงๆ

### การรวมบริบท AI

สิ่งสำคัญที่สุดคือ ระบบสามารถแทรกข้อมูลห้องนิรภัยลงในบริบทของข้อเสนอ AI ได้โดยอัตโนมัติ:

```typescript
export function buildTargetVaultsText(
  vaults: VaultForText[],
  template: VaultPromptTemplate = DEFAULT_VAULT_PROMPT_TEMPLATE,
): string {
  const readOnlyVaults = vaults.filter((vault) => vault.accessType === 'read');
  const editableVaults = vaults.filter((vault) => vault.accessType === 'write');

  if (readOnlyVaults.length === 0 && editableVaults.length === 0) {
    return '';
  }

  const sections = [
    buildVaultSection(readOnlyVaults, template.reference),
    buildVaultSection(editableVaults, template.editable),
  ].filter(Boolean);

  return `\n\n### ${template.heading}\n\n${sections.join('\n')}`;
}
```

สิ่งนี้ทำให้เกิดความสามารถที่สำคัญ: ผู้ช่วย AI สามารถเข้าใจทรัพยากรการเรียนรู้ที่มีอยู่ได้โดยอัตโนมัติ โดยที่ผู้ใช้ไม่ต้องให้บริบทด้วยตนเอง คุณสามารถพูดได้ว่านั่นถือเป็นความเข้าใจโดยปริยาย

## โครงสร้างมาตรฐานของ CodeRef Vault

สำหรับประเภท coderef ของ vault นั้น HagiCode จัดเตรียมโครงสร้างไดเร็กทอรีที่เป็นมาตรฐาน:

```
my-coderef-vault/
├── index.yaml          # vault metadata description
├── AGENTS.md           # operating guide for AI assistants
├── docs/               # stores study notes and documents
└── repos/              # manages reproduced code repositories through Git submodules
```

เมื่อสร้างห้องนิรภัย ระบบจะเริ่มต้นโครงสร้างนี้โดยอัตโนมัติ:

```csharp
private async Task EnsureCodeRefStructureAsync(
    string vaultName,
    string physicalPath,
    ICollection<VaultBootstrapDiagnosticDto> diagnostics,
    CancellationToken cancellationToken)
{
    Directory.CreateDirectory(physicalPath);

    var indexPath = Path.Combine(physicalPath, CodeRefIndexFileName);
    var docsPath = Path.Combine(physicalPath, CodeRefDocsDirectoryName);
    var reposPath = Path.Combine(physicalPath, CodeRefReposDirectoryName);

    // Create the standard directory structure
    if (!Directory.Exists(docsPath))
    {
        Directory.CreateDirectory(docsPath);
    }

    if (!Directory.Exists(reposPath))
    {
        Directory.CreateDirectory(reposPath);
    }

    // Create the AGENTS.md guide
    await EnsureCodeRefAgentsDocumentAsync(physicalPath, cancellationToken);

    // Create the index.yaml metadata
    await WriteCodeRefIndexDocumentAsync(indexPath, mergedDocument, cancellationToken);
}
```

โครงสร้างนี้ได้รับการออกแบบอย่างระมัดระวังเช่นกัน:

- **เอกสาร/** จัดเก็บบันทึกการเรียนรู้ของคุณ ซึ่งคุณสามารถบันทึกความเข้าใจเกี่ยวกับโค้ด การวิเคราะห์สถาปัตยกรรม บทเรียนที่ได้รับ และอื่นๆ ใน Markdown
- **repos/** จัดการพื้นที่เก็บข้อมูลที่ทำซ้ำผ่านโมดูลย่อย Git แทนที่จะคัดลอกโค้ดโดยตรง ซึ่งจะทำให้โค้ดซิงค์และประหยัดพื้นที่
- **index.yaml** มีข้อมูลเมตาของ Vault เพื่อให้ผู้ช่วย AI สามารถเข้าใจวัตถุประสงค์และเนื้อหาของ Vault ได้อย่างรวดเร็ว
- **AGENTS.md** เป็นคู่มือที่เขียนขึ้นโดยเฉพาะสำหรับผู้ช่วย AI โดยอธิบายวิธีจัดการเนื้อหาในห้องนิรภัย

เมื่อจัดแบบนี้ บางที AI ก็สามารถเข้าใจสิ่งที่คุณอยู่ในใจได้ง่ายขึ้นอีกหน่อย

## การเริ่มต้นอัตโนมัติสำหรับห้องนิรภัยที่ระบบจัดการ

นอกเหนือจากห้องนิรภัยที่สร้างขึ้นด้วยตนเองแล้ว HagiCode ยังรองรับห้องนิรภัยที่จัดการโดยระบบด้วย:

```csharp
public async Task<IReadOnlyList<VaultRegistryEntry>> EnsureAllSystemManagedVaultsAsync(
    CancellationToken cancellationToken = default)
{
    var definitions = GetAllResolvedDefinitions();
    var entries = new List<VaultRegistryEntry>(definitions.Count);

    foreach (var definition in definitions)
    {
        entries.Add(await EnsureResolvedSystemManagedVaultAsync(definition, cancellationToken));
    }

    return entries;
}
```

ระบบจะสร้างและจัดการห้องนิรภัยต่อไปนี้โดยอัตโนมัติ:

- **hagiprojectdata**: พื้นที่จัดเก็บข้อมูลโปรเจ็กต์ที่ใช้ในการบันทึกการกำหนดค่าและสถานะของโปรเจ็กต์
- **ข้อมูลส่วนบุคคล**: การจัดเก็บข้อมูลส่วนบุคคลใช้เพื่อบันทึกการตั้งค่าของผู้ใช้
- **hbsprompt**: ไลบรารีเทมเพลตพรอมต์ที่ใช้เพื่อจัดการพรอมต์ AI ที่ใช้กันทั่วไป

ห้องนิรภัยเหล่านี้จะเตรียมใช้งานโดยอัตโนมัติเมื่อระบบเริ่มทำงาน ดังนั้นผู้ใช้จึงไม่จำเป็นต้องกำหนดค่าด้วยตนเอง บางสิ่งก็ปล่อยให้อยู่ในระบบดีกว่าปล่อยให้มนุษย์กังวลกับมัน

## กลไกการควบคุมการเข้าถึง

ส่วนสำคัญของการออกแบบคือการควบคุมการเข้าถึง ระบบแบ่งห้องนิรภัยออกเป็นสองประเภทการเข้าถึง:

```typescript
export interface VaultForText {
  id: string;
  name: string;
  type: string;
  physicalPath: string;
  accessType: 'read' | 'write';  // Key: distinguish read-only from editable
}
```

- **ข้อมูลอ้างอิง (อ่านอย่างเดียว)**: AI ใช้เพื่อการวิเคราะห์และทำความเข้าใจเท่านั้น และไม่สามารถแก้ไขเนื้อหาได้ เหมาะสำหรับโครงการโอเพ่นซอร์ส เอกสาร และสื่อที่คล้ายกัน
- **แก้ไขได้ (editable)**: AI สามารถแก้ไขเนื้อหาได้ตามต้องการสำหรับงาน เหมาะสำหรับจดบันทึก ร่างจดหมาย และวัสดุที่คล้ายกัน

ความแตกต่างนี้มีความสำคัญ โดยจะบอก AI ว่าเนื้อหาใดเป็น "การอ้างอิงแบบอ่านอย่างเดียว" และเนื้อหาใด "ปลอดภัยต่อการแก้ไข" ซึ่งจะช่วยลดความเสี่ยงของการเปลี่ยนแปลงโดยไม่ตั้งใจ ท้ายที่สุดแล้ว ไม่มีใครอยากให้การทำงานหนักของพวกเขาหายไปเนื่องจากการตัดต่อโดยไม่ได้ตั้งใจ

## ในทางปฏิบัติ: การสร้างและใช้งานห้องนิรภัย

ตอนนี้เราได้ครอบคลุมแนวคิดต่างๆ แล้ว เรามาดูกันว่าแนวคิดนี้ทำงานอย่างไรในทางปฏิบัติ

### สร้างห้องนิรภัย CodeRef

นี่คือตัวอย่างการโทรส่วนหน้าที่สมบูรณ์:

```typescript
const createCodeRefVault = async () => {
  const response = await VaultService.postApiVaults({
    requestBody: {
      name: "React Learning Vault",
      type: "coderef",
      physicalPath: "/Users/developer/vaults/react-learning",
      gitUrl: "https://github.com/facebook/react.git"
    }
  });

  // The system will automatically:
  // 1. Clone the React repository into vault/repos/react
  // 2. Create the docs/ directory for notes
  // 3. Generate the index.yaml metadata
  // 4. Create the AGENTS.md guide file

  return response;
};
```

การเรียก API นี้จะเสร็จสิ้นชุดของการดำเนินการ: การสร้างโครงสร้างไดเร็กทอรี, การเริ่มต้นโมดูลย่อย Git, การสร้างไฟล์ข้อมูลเมตา และอื่นๆ คุณเพียงแค่ต้องให้ข้อมูลพื้นฐานและปล่อยให้ระบบจัดการส่วนที่เหลือ เป็นแนวทางที่ไร้กังวลโดยสุจริต

### ใช้ห้องนิรภัยในข้อเสนอ AI

หลังจากสร้าง Vault คุณสามารถอ้างอิงได้ในข้อเสนอ AI:

```typescript
const proposal = composeProposalChiefComplaint({
  chiefComplaint: "Help me analyze React's concurrent rendering mechanism",
  repositories: [
    { id: "react", gitUrl: "https://github.com/facebook/react.git" }
  ],
  vaults: [
    {
      id: "react-learning",
      name: "React Learning Vault",
      type: "coderef",
      physicalPath: "/vaults/react-learning",
      accessType: "read"  // AI can only read, not modify
    }
  ],
  quickRequestText: "Focus on the Fiber architecture and scheduler implementation"
});
```

ระบบจะแทรกข้อมูลห้องนิรภัยลงในบริบทของ AI โดยอัตโนมัติ เพื่อให้ AI ทราบว่ามีทรัพยากรการเรียนรู้ใดบ้าง เมื่อ AI สามารถเข้าใจสิ่งที่คุณมีในใจได้ ความเข้าใจโดยปริยายแบบนั้นก็เกิดขึ้นได้ยาก

## แนวทางปฏิบัติที่ดีที่สุดและสิ่งที่ควรระวัง

ในขณะที่ใช้ระบบห้องนิรภัย เราได้สรุปบทเรียนบางส่วนที่ได้เรียนรู้

### ความปลอดภัยของเส้นทาง

ระบบจะตรวจสอบเส้นทางอย่างเคร่งครัดเพื่อป้องกันการโจมตีการข้ามเส้นทาง:

```csharp
private static string ResolveFilePath(string vaultRoot, string relativePath)
{
    var rootPath = EnsureTrailingSeparator(Path.GetFullPath(vaultRoot));
    var combinedPath = Path.GetFullPath(Path.Combine(rootPath, relativePath));
    if (!combinedPath.StartsWith(rootPath, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode,
            "Vault file paths must stay inside the registered vault root.");
    }
    return combinedPath;
}
```

นี่เป็นสิ่งสำคัญ หากคุณปรับแต่งเส้นทาง vault ตรวจสอบให้แน่ใจว่าอยู่ภายในช่วงที่อนุญาต มิฉะนั้นระบบจะปฏิเสธการดำเนินการ คุณไม่สามารถเน้นความปลอดภัยมากเกินไปได้

### การจัดการโมดูลย่อย Git

CodeRef Vault แนะนำโมดูลย่อย Git แทนที่จะคัดลอกโค้ดโดยตรง:

```csharp
private static string BuildCodeRefAgentsContent()
{
    return """
    # CodeRef Vault Guide

    Repositories under `repos/` should be maintained through Git submodules
    rather than copied directly into the vault root.

    Keep this structure stable so assistants and tools can understand the vault quickly.
    """ + Environment.NewLine;
}
```

สิ่งนี้นำมาซึ่งข้อดีหลายประการ: ทำให้โค้ดซิงโครไนซ์กับอัพสตรีม ประหยัดพื้นที่ดิสก์ และทำให้ง่ายต่อการจัดการโค้ดหลายเวอร์ชัน ท้ายที่สุดแล้วใครอยากดาวน์โหลดสิ่งเดียวกันซ้ำแล้วซ้ำเล่า?

### ขีดจำกัดการแสดงตัวอย่างไฟล์

เพื่อป้องกันปัญหาด้านประสิทธิภาพ ระบบจะจำกัดขนาดและประเภทของไฟล์:

```csharp
private const int FileEnumerationLimit = 500;
private const int PreviewByteLimit = 256 * 1024;  // 256KB
```

หากห้องนิรภัยของคุณมีไฟล์จำนวนมากหรือไฟล์ขนาดใหญ่มาก ประสิทธิภาพการแสดงตัวอย่างอาจได้รับผลกระทบ ในกรณีดังกล่าว คุณสามารถพิจารณาประมวลผลไฟล์เป็นชุดหรือใช้เครื่องมือค้นหาเฉพาะทางได้ บางครั้งเมื่อบางสิ่งมีขนาดใหญ่เกินไป การจัดการก็จะยากขึ้น ไม่ใช่ง่ายยิ่งขึ้น

### ข้อมูลการวินิจฉัย

เมื่อสร้างห้องนิรภัย ระบบจะส่งคืนข้อมูลการวินิจฉัยเพื่อช่วยในการดีบัก:

```csharp
List<VaultBootstrapDiagnosticDto> bootstrapDiagnostics = [];

if (IsCodeRefVaultType(normalizedType))
{
    bootstrapDiagnostics = await EnsureCodeRefBootstrapAsync(
        normalizedName,
        normalizedPhysicalPath,
        normalizedGitUrl,
        cancellationToken);
}
```

หากการสร้างล้มเหลว คุณสามารถตรวจสอบข้อมูลการวินิจฉัยเพื่อทำความเข้าใจสาเหตุเฉพาะได้ เมื่อมีสิ่งผิดปกติเกิดขึ้น การตรวจสอบการวินิจฉัยมักเป็นแนวทางที่ตรงประเด็นที่สุด

## สรุป

ระบบ Vault แก้ปัญหาหลักหลายประการในการสร้างโปรเจ็กต์ใหม่ในยุค AI ผ่านเลเยอร์นามธรรมการจัดเก็บข้อมูลแบบรวมศูนย์:

- **การจัดการความรู้แบบรวมศูนย์**: แหล่งเรียนรู้ทั้งหมดถูกรวบรวมไว้ในที่เดียว แทนที่จะกระจัดกระจายไปทุกที่
- **การแทรกบริบท AI อัตโนมัติ**: ผู้ช่วย AI สามารถเข้าใจทรัพยากรการเรียนรู้ที่มีอยู่ได้โดยอัตโนมัติ โดยไม่ต้องตั้งค่าบริบทด้วยตนเอง
- **การนำความรู้ข้ามโครงการกลับมาใช้ใหม่**: สามารถแบ่งปันและนำความรู้กลับมาใช้ซ้ำในโครงการการเรียนรู้ต่างๆ ได้
- **โครงสร้างไดเร็กทอรีที่ได้มาตรฐาน**: เลย์เอาต์ไดเร็กทอรีที่สอดคล้องกันจะช่วยลดขั้นตอนการเรียนรู้

โซลูชันนี้ได้รับการตรวจสอบแล้วในโครงการ HagiCode หากคุณกำลังสร้างเครื่องมือที่เกี่ยวข้องกับการพัฒนาโดยใช้ AI หรือประสบปัญหาการจัดการความรู้ที่คล้ายกัน ฉันหวังว่าประสบการณ์เหล่านี้จะสามารถใช้เป็นข้อมูลอ้างอิงที่เป็นประโยชน์ได้

ความจริงแล้ว คุณค่าของโซลูชันทางเทคนิคไม่ได้อยู่ที่ความซับซ้อนของโซลูชัน แต่อยู่ที่ว่าจะสามารถแก้ไขปัญหาได้จริงหรือไม่ แนวคิดหลักของระบบ Vault นั้นง่ายมาก: สร้างชั้นการจัดเก็บความรู้แบบครบวงจรที่ AI สามารถเข้าใจได้ แต่มันเป็นนามธรรมง่ายๆ นี้เองที่ช่วยปรับปรุงประสิทธิภาพการพัฒนาของเราไม่น้อย

บางครั้งวิธีการง่ายๆ ก็เป็นแนวทางที่ดีที่สุดจริงๆ ท้ายที่สุดแล้ว สิ่งที่ซับซ้อนมักจะซ่อนข้อผิดพลาดเอาไว้มากกว่านั้น...
