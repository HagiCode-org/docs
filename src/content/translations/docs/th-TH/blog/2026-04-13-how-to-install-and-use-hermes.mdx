หากคุณต้องการติดตั้ง Hermes และเริ่มใช้งาน เส้นทางที่สั้นที่สุดมีเพียงสามขั้นตอนเท่านั้น:

1. รันคำสั่งการติดตั้งอย่างเป็นทางการ
2. เริ่ม CLI ในเทอร์มินัลของคุณด้วย `hermes`
3. หากคุณต้องการใช้ Feishu ต่อไป ให้กำหนดค่า `hermes gateway setup`

บทความนี้ไม่ได้พยายามอธิบายความสามารถของ Hermes ทุกประการในคราวเดียว แต่จะช่วยให้คุณดำเนินการวนรอบเริ่มต้นที่สำคัญที่สุดได้ก่อน: ติดตั้ง เรียกใช้งาน เริ่มใช้งาน จากนั้นเชื่อมต่อกับหนึ่งในสถานการณ์แพลตฟอร์มการส่งข้อความที่พบบ่อยที่สุด

## Hermes คืออะไรและมีไว้เพื่อใคร

Hermes Agent คือตัวแทน AI ที่คุณสามารถใช้จากเทอร์มินัลในพื้นที่หรือผ่านเกตเวย์แพลตฟอร์มการส่งข้อความ

สำหรับนักพัฒนาส่วนใหญ่ มีจุดเข้าใช้งานร่วมกันสองจุด:

- **CLI**: ประเภท `hermes` ในเทอร์มินัลของคุณเพื่อเข้าสู่อินเทอร์เฟซแบบโต้ตอบได้โดยตรง
- **เกตเวย์การส่งข้อความ**: เรียกใช้ `hermes gateway`จากนั้นแชทด้วยจากแพลตฟอร์มต่างๆ เช่น Feishu, Telegram, Discord และ Slack

หากเป้าหมายของคุณตอนนี้คือการเริ่มต้นอย่างรวดเร็ว อย่ากลับคำสั่ง เริ่มต้นด้วยเส้นทางนี้แทน:

- ติดตั้ง Hermes ก่อน
- ตรวจสอบว่ามันใช้งานได้จาก CLI ก่อน
- จากนั้นตัดสินใจว่าคุณต้องการเชื่อมต่อแพลตฟอร์มการรับส่งข้อความหรือไม่

ทำให้วินิจฉัยปัญหาได้ง่ายขึ้นและเหมาะกับผู้ที่ใช้ Hermes เป็นครั้งแรกมากกว่า

## สิ่งที่ควรรู้ก่อนติดตั้ง Hermes

ตาม Hermes README เส้นทางการติดตั้งด่วนอย่างเป็นทางการรองรับสภาพแวดล้อมเหล่านี้:

- ลินุกซ์
- ระบบปฏิบัติการ macOS
- WSL2
- ระบบปฏิบัติการ Android ผ่าน Termux

### หมายเหตุสำหรับผู้ใช้ Windows

ปัจจุบัน Hermes ไม่รองรับการทำงานโดยตรงบน Windows ดั้งเดิม หากคุณใช้ Windows แนวทางที่แนะนำคือติดตั้ง WSL2 ก่อน จากนั้นจึงรันคำสั่งการติดตั้งภายใน WSL2

วิธีที่ดีที่สุดคือทำให้ชัดเจนตั้งแต่เริ่มต้น เนื่องจากความล้มเหลวในการติดตั้งจำนวนมากไม่ได้เกิดจากคำสั่งเอง แต่เกิดจากการใช้สภาพแวดล้อมรันไทม์ที่ไม่สนับสนุน

## วิธีการติดตั้ง Hermes.msc

คำสั่งการติดตั้งด่วนที่ให้ไว้ใน Hermes README คือ:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

คำสั่งนี้รันสคริปต์การติดตั้งอย่างเป็นทางการและจัดการขั้นตอนการเริ่มต้นเฉพาะแพลตฟอร์ม

### โหลดเชลล์ของคุณใหม่หลังการติดตั้ง

เมื่อการติดตั้งเสร็จสิ้น ให้รีโหลดสภาพแวดล้อมเชลล์ของคุณก่อน คำสั่งที่พบบ่อยที่สุดคือ:

```bash
source ~/.bashrc
```

หากคุณใช้ zsh คุณสามารถใช้:

```bash
source ~/.zshrc
```

### วิธียืนยันการติดตั้ง Hermes อย่างถูกต้อง

วิธีตรวจสอบที่ตรงที่สุดคือการรัน:

```bash
hermes
```

หากคุณต้องการการยืนยันเพิ่มเติมว่าการกำหนดค่าและการขึ้นต่อกันของคุณใช้งานได้ คุณยังสามารถเรียกใช้:

```bash
hermes doctor
```

`hermes doctor` มีประโยชน์อย่างยิ่งในสถานการณ์เหล่านี้:

- คำสั่งทำงานผิดปกติหลังการติดตั้ง
- การกำหนดค่าโมเดลล้มเหลว
- เกตเวย์ไม่สามารถเริ่มต้นได้
- คุณไม่แน่ใจว่าการพึ่งพาสภาพแวดล้อมของคุณเสร็จสมบูรณ์หรือไม่

## วิธีเริ่มใช้ Hermes เป็นครั้งแรก

### เริ่มต้นด้วย CLI

หากคุณต้องการยืนยันโดยเร็วที่สุดว่า Hermes ใช้งานได้ วิธีที่ง่ายที่สุดคือ:

```bash
hermes
```

นี่เป็นการเปิดตัว Hermes CLI แบบโต้ตอบ สำหรับผู้ใช้ Hermes เป็นครั้งแรก ถือเป็นจุดเริ่มต้นที่แนะนำมากที่สุด เนื่องจากคุณสามารถตรวจสอบสิ่งสำคัญที่สุดก่อนได้:

- ไม่ว่าคำสั่งจะใช้งานได้จริงหรือไม่
- การกำหนดค่าโมเดลปัจจุบันทำงานอย่างถูกต้องหรือไม่
- ไม่ว่าห่วงโซ่เครื่องมือเทอร์มินัลทำงานอย่างถูกต้องหรือไม่
- รูปแบบการโต้ตอบตรงกับสิ่งที่คุณต้องการหรือไม่

### คำสั่งเหล่านี้เพียงพอสำหรับการตั้งค่ารอบแรก

Hermes README แสดงรายการคำสั่งความถี่สูงหลายคำสั่ง และเมื่อรวมกันเป็นเส้นทางการใช้งานครั้งแรกที่ใช้งานได้จริง:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

หากคุณไม่แน่ใจว่าแต่ละคนทำอะไร ให้จำไว้ดังนี้:

- `hermes model`: เลือกหรือสลับรุ่น
- `hermes tools`: ดูและกำหนดค่าเครื่องมือที่มีอยู่ในปัจจุบัน
- `hermes config set`: เปลี่ยนรายการการกำหนดค่าเฉพาะ
- `hermes setup`: รันวิซาร์ดการเริ่มต้นแบบเต็มหนึ่งครั้ง
- `hermes update`: อัพเดทเฮอร์มีส
- `hermes doctor`: แก้ไขปัญหา

สำหรับผู้เริ่มต้น ลำดับที่ใช้งานได้จริงที่สุดมักจะเป็น:

1. วิ่ง `hermes model` ก่อน
2. หากคุณต้องการกำหนดค่าตัวเลือกทั่วไปทั้งหมดในคราวเดียว ให้เรียกใช้ `hermes setup`

## สองวิธีที่พบบ่อยที่สุดในการใช้ Hermes

### 1. ใช้ Hermes ใน Terminal เป็นตัวช่วยในการพัฒนารายวัน

โหมด CLI เหมาะอย่างยิ่งสำหรับสถานการณ์เหล่านี้:

- ถามคำถามโดยตรงขณะเขียนโค้ดในเครื่อง
- ตรวจสอบโปรเจ็กต์ แก้ไขไฟล์ และรันคำสั่ง
- ทำการดีบักหรือตรวจทานแบบครั้งเดียว
- ทำงานร่วมกันอย่างต่อเนื่องในไดเร็กทอรีการทำงานปัจจุบัน

ข้อได้เปรียบที่ใหญ่ที่สุดคือเป็นเส้นทางที่สั้นที่สุด: ไม่มีการผสานรวมแพลตฟอร์มเพิ่มเติม ไม่มีการกำหนดค่าบอทให้จัดการล่วงหน้า และเป็นวิธีที่ดีที่สุดในการสร้างพฤติกรรมการใช้งานชุดแรกของคุณ

### 2. ใช้ Hermes ผ่านแพลตฟอร์มการส่งข้อความ

หากคุณต้องการแชทกับ Hermes บนแพลตฟอร์ม เช่น Feishu, Telegram หรือ Discord คุณต้องใช้เกตเวย์การรับส่งข้อความ

คำสั่งรายการที่พบบ่อยที่สุดคือ:

```bash
hermes gateway setup
hermes gateway
```

โดยเฉพาะ:

- `hermes gateway setup` ใช้สำหรับการกำหนดค่าแพลตฟอร์มแบบโต้ตอบ
- `hermes gateway` ใช้เพื่อเริ่มกระบวนการเกตเวย์

ตามเอกสารอย่างเป็นทางการ เกตเวย์เป็นกระบวนการพื้นหลังแบบรวมที่เชื่อมต่อแพลตฟอร์มที่กำหนดค่าของคุณ จัดการเซสชัน และจัดการคุณสมบัติ เช่น งาน cron

## ใช้ Feishu เป็นตัวอย่าง: วิธีเชื่อมต่อ Hermes กับแพลตฟอร์มการส่งข้อความ

หากงานประจำวันของคุณส่วนใหญ่เกิดขึ้นใน Feishu Feishu/Lark ก็เป็นวิธีที่เป็นธรรมชาติมากในการใช้ Hermes

### เส้นทางการรวมที่เป็นไปได้ขั้นต่ำ

เอกสารอย่างเป็นทางการแนะนำคำสั่งรายการนี้สำหรับ Feishu/Lark:

```bash
hermes gateway setup
```

หลังจากที่คุณเรียกใช้แล้ว เพียงเลือก **Feishu / Lark** ในตัวช่วยสร้าง

เอกสาร Feishu อธิบายโหมดการเชื่อมต่อสองโหมด:

- `websocket`: แนะนำ
- `webhook`: ไม่จำเป็น

หาก Hermes ทำงานบนแล็ปท็อป เวิร์กสเตชัน หรือเซิร์ฟเวอร์ส่วนตัวของคุณ ให้ใช้งาน `websocket` อย่างแรกมักจะง่ายกว่าเพราะคุณไม่จำเป็นต้องเปิดเผย URL การติดต่อกลับแบบสาธารณะ

### หากคุณกำหนดค่าด้วยตนเอง อย่างน้อยก็ควรทราบตัวแปรเหล่านี้

หากคุณไม่ได้ใช้วิซาร์ดและกำลังเขียนการกำหนดค่าด้วยตนเอง เอกสาร Feishu จะแสดงรายการตัวแปรหลักเหล่านี้:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

สองคนสมควรได้รับความสนใจเป็นพิเศษ:

- `FEISHU_ALLOWED_USERS`: แนะนำ ดังนั้นไม่ใช่ทุกคนที่เข้าถึงบอทได้จะสามารถใช้งานได้โดยตรง
- `FEISHU_HOME_CHANNEL`: ช่วยให้คุณสามารถกำหนดโฮมแชทล่วงหน้าเพื่อรับผลลัพธ์ cron หรือการแจ้งเตือนเริ่มต้น

### ทำไมบางครั้ง Hermes จึงไม่ตอบกลับในการแชทกลุ่ม Feishu

รายละเอียดนี้พลาดได้ง่าย: ในการแชทกลุ่ม Feishu นั้น Hermes จะไม่ตอบกลับทุกข้อความตามค่าเริ่มต้น

เอกสารอย่างเป็นทางการระบุไว้อย่างชัดเจน:

- ในข้อความตรง Hermes ตอบกลับข้อความ
- ในการแชทกลุ่มคุณต้องชัดเจน `@` บอทก่อนที่จะประมวลผลข้อความ

หากคุณต้องการตั้งค่าการสนทนา Feishu เป็นช่องทางหลัก คุณสามารถใช้สิ่งนี้ในการแชทได้:

```text
/set-home
```

หรือกำหนดไว้ในการกำหนดค่าล่วงหน้า:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## คำสั่ง Hermes ที่ผู้เริ่มต้นควรจำไว้ก่อน

ไม่ว่าคุณจะใช้ Hermes ใน CLI หรือบนแพลตฟอร์มการส่งข้อความ การจดจำคำสั่งต่อไปนี้ก็เพียงพอที่จะเริ่มต้นแล้ว:

- `/new` หรือ `/reset`: เริ่มเซสชั่นใหม่
- `/model`: ดูหรือสลับรุ่น
- `/retry`: ลองเทิร์นก่อนหน้าอีกครั้ง
- `/undo`: เลิกทำการโต้ตอบก่อนหน้า
- `/compress`: บีบอัดบริบทด้วยตนเอง
- `/help`: ดูวิธีใช้

หากคุณใช้ Hermes บนแพลตฟอร์มการส่งข้อความเป็นหลัก โปรดจำไว้อีกอย่างหนึ่ง:

- `/sethome` หรือ `/set-home`: ตั้งค่าแชทปัจจุบันเป็นช่องทางหลัก

คำสั่งเหล่านี้ครอบคลุมการดำเนินการขั้นเริ่มต้นที่พบบ่อยที่สุด: การรีสตาร์ท การปรับ การย้อนกลับ การตรวจสอบ และการดำเนินการต่อ

## คำถามที่พบบ่อย

### Windows สามารถติดตั้ง Hermes ได้โดยตรงหรือไม่

ไม่ เอกสารอย่างเป็นทางการในปัจจุบันระบุไว้อย่างชัดเจนว่าไม่รองรับ Windows แบบเนทีฟ และแนะนำให้ใช้ WSL2

### ฉันควรทำอย่างไรถ้าพิมพ์ `hermes` ไม่ทำอะไรเลยหลังการติดตั้ง?

วิธีที่ดีที่สุดคือแก้ไขปัญหาตามลำดับนี้:

1. รีโหลดเชลล์ของคุณก่อน เช่น ด้วย `source ~/.bashrc`
2. วิ่ง `hermes` อีกครั้ง
3. หากยังผิดปกติอยู่ให้วิ่ง `hermes doctor`

### เหตุใดบอทจึงไม่ตอบกลับในกลุ่ม Feishu

ตรวจสอบสามสิ่งนี้ก่อน:

- ไม่ว่าคุณจะ `@` กล่าวถึงเฮอร์มีสในกลุ่ม
- ไม่ว่าจะเป็น `FEISHU_ALLOWED_USERS` จำกัดผู้ใช้ปัจจุบัน
- นโยบายการแชทกลุ่มปัจจุบันอนุญาตให้จัดการข้อความกลุ่มหรือไม่

ตามเอกสารอย่างเป็นทางการของ Feishu จำเป็นต้องใช้ @mention อย่างชัดเจนในสถานการณ์การแชทเป็นกลุ่ม

## แนะนำขั้นตอนต่อไป

หากคุณต้องการเริ่มใช้ Hermes โดยเร็วที่สุด นี่คือคำสั่งซื้อที่แนะนำมากที่สุด:

1. รันคำสั่งการติดตั้งก่อน
2. เริ่มต้นด้วย `hermes` ใน CLI ท้องถิ่นก่อน
3. ใช้ `hermes model` และ `hermes setup` เพื่อเสร็จสิ้นการกำหนดค่าพื้นฐาน
4. หากคุณต้องการใช้ Feishu ต่อไป ให้กำหนดค่า `hermes gateway setup`

หากบทความนี้เป็นส่วนแรกของซีรีส์ บทบาทที่ดีที่สุดของบทความนี้คือไม่ต้องอธิบายคุณลักษณะขั้นสูงทั้งหมดทั้งหมดในคราวเดียว แต่เพื่อให้ผู้ใช้เข้ามามีส่วนร่วมก่อน

หัวข้อต่อไปนี้จะแบ่งออกเป็นบทความติดตามผลได้ดีกว่า:

- คู่มือบูรณาการ Hermes Feishu ฉบับสมบูรณ์
- คำแนะนำเกี่ยวกับคำสั่งสแลชทั่วไปของ Hermes
- คำแนะนำเกี่ยวกับการกำหนดค่าเกตเวย์ Hermes และการแก้ไขปัญหา

หากคุณวางแผนที่จะสร้างเนื้อหา Hermes ต่อไป บทความนี้สามารถใช้เป็นจุดเริ่มต้นสำหรับการโพสต์ครั้งต่อๆ ไป ในขณะที่คุณค่อยๆ สร้างโครงสร้างลิงก์ภายใน
