## การแก้ไขโค้ดอย่างรวดเร็วในเบราว์เซอร์: บูรณาการเว็บ VSCode ในทางปฏิบัติ

> หลังจากที่ AI วิเคราะห์โค้ดเสร็จแล้ว คุณจะเปิดโปรแกรมแก้ไขในเบราว์เซอร์ทันทีและเริ่มทำการเปลี่ยนแปลงได้อย่างไร บทความนี้แบ่งปันประสบการณ์เชิงปฏิบัติของเราในการบูรณาการโค้ดเซิร์ฟเวอร์ในโครงการ HagiCode เพื่อสร้างสะพานเชื่อมที่ราบรื่นระหว่างผู้ช่วย AI และประสบการณ์การแก้ไขโค้ด

## พื้นหลัง

ในยุคของการเขียนโปรแกรมแบบใช้ AI นักพัฒนามักจำเป็นต้องตรวจสอบและแก้ไขโค้ดอย่างรวดเร็ว ขั้นตอนการทำงานแบบดั้งเดิมนั้นเรียบง่าย: เปิดโปรเจ็กต์ใน IDE บนเดสก์ท็อป ค้นหาไฟล์ แก้ไข และบันทึก แต่ในบางสถานการณ์ กระแสนั้นมักจะรู้สึกผิดเล็กน้อยเสมอ

สถานการณ์ที่หนึ่ง: การพัฒนาระยะไกล เมื่อใช้ผู้ช่วย AI เช่น HagiCode แบ็กเอนด์อาจทำงานบนเซิร์ฟเวอร์ระยะไกลหรือภายในคอนเทนเนอร์ และเครื่องภายในไม่สามารถเข้าถึงไฟล์โปรเจ็กต์ได้โดยตรง ทุกครั้งที่คุณต้องการตรวจสอบหรือแก้ไขโค้ด คุณต้องเชื่อมต่อผ่าน SSH หรือวิธีอื่น และประสบการณ์จะรู้สึกกระจัดกระจาย มันเหมือนกับอยากเจอใครสักคนผ่านกระจกหนาๆ มองเห็นได้ แต่เอื้อมไม่ถึง

สถานการณ์ที่สอง: การแสดงตัวอย่างอย่างรวดเร็ว หลังจากที่ผู้ช่วย AI วิเคราะห์โค้ดแล้ว ผู้ใช้อาจเพียงต้องการเรียกดูไฟล์อย่างรวดเร็วหรือทำการเปลี่ยนแปลงเล็กน้อยเท่านั้น การเปิดตัว IDE บนเดสก์ท็อปเต็มรูปแบบให้ความรู้สึกหนักหน่วง ในขณะที่ตัวแก้ไขในเบราว์เซอร์แบบน้ำหนักเบาเหมาะกับความต้องการ "รูปลักษณ์แบบรวดเร็ว" มากกว่า ท้ายที่สุดแล้ว ใครอยากจะระดม toolchain ทั้งหมดเพื่อดูคร่าวๆ บ้าง?

สถานการณ์ที่สาม: การทำงานร่วมกันข้ามอุปกรณ์ เมื่อทำงานกับอุปกรณ์ต่างๆ ตัวแก้ไขที่ใช้เบราว์เซอร์จะมอบจุดเข้าใช้งานแบบรวมโดยไม่ต้องกำหนดค่าทุกเครื่องด้วยสภาพแวดล้อมการพัฒนา เพียงอย่างเดียวช่วยประหยัดปัญหาได้มาก ชีวิตนั้นสั้น เหตุใดการตั้งค่าเดิมซ้ำจึงทำงานซ้ำแล้วซ้ำเล่า?

เพื่อแก้ไขปัญหาเหล่านี้ เราได้รวม VSCode Web เข้ากับโปรเจ็กต์ HagiCode ช่วยให้ผู้ช่วย AI และประสบการณ์การแก้ไขโค้ดเชื่อมต่อกันได้อย่างราบรื่น หลังจากที่ AI วิเคราะห์โค้ดแล้ว ผู้ใช้สามารถเปิดโปรแกรมแก้ไขและทำการเปลี่ยนแปลงในเซสชันเบราว์เซอร์เดียวกันได้ทันที โดยไม่ต้องสลับแอปพลิเคชัน เป็นประสบการณ์แบบที่เมื่อคุณต้องการมันก็อยู่ที่นั่นทันที

## เกี่ยวกับ ฮากิโค้ด

วิธีแก้ปัญหาที่แบ่งปันในบทความนี้มาจากประสบการณ์จริงของเราใน [ฮากิโค้ด](https://hagicode.com) โครงการ HagiCode คือผู้ช่วยเขียนโค้ดที่ขับเคลื่อนด้วย AI ซึ่งออกแบบมาเพื่อปรับปรุงประสิทธิภาพการพัฒนาผ่านการโต้ตอบด้วยภาษาธรรมชาติ ในระหว่างการพัฒนา เราพบว่าผู้ใช้มักจะต้องสลับอย่างรวดเร็วระหว่างการวิเคราะห์ AI และการแก้ไขโค้ด ซึ่งผลักดันให้เราสำรวจวิธีการรวมตัวแก้ไขเข้ากับเบราว์เซอร์โดยตรง

ที่เก็บโครงการ: [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site)

## ตัวเลือกทางเทคนิค: ทำไมต้องใช้โค้ดเซิร์ฟเวอร์

ในบรรดาโซลูชัน VSCode Web ที่มีอยู่มากมาย เราเลือกโค้ดเซิร์ฟเวอร์ มีเหตุผลที่ชัดเจนบางประการที่อยู่เบื้องหลังการตัดสินใจครั้งนั้น

คุณสมบัติครบถ้วน code-server คือ VSCode เวอร์ชันเว็บ และรองรับฟีเจอร์เดสก์ท็อปส่วนใหญ่ รวมถึงระบบส่วนขยาย คำแนะนำอัจฉริยะ การดีบัก และอื่นๆ อีกมากมาย นั่นหมายความว่าผู้ใช้สามารถรับประสบการณ์การแก้ไขในเบราว์เซอร์ที่ใกล้เคียงกับเวอร์ชันเดสก์ท็อปมาก ท้ายที่สุดแล้วใครล่ะที่ต้องการประนีประนอมกับฟังก์ชันการทำงาน?

การใช้งานที่ยืดหยุ่น โค้ดเซิร์ฟเวอร์สามารถทำงานเป็นบริการอิสระและยังรองรับการปรับใช้บน Docker ซึ่งเข้ากันได้ดีกับสถาปัตยกรรมของ HagiCode แบ็คเอนด์ของเราเขียนด้วยภาษา C# ส่วนหน้าใช้ React และทั้งสองสื่อสารกับบริการโค้ดเซิร์ฟเวอร์ผ่าน REST API มันเหมือนกับการสร้างบล็อก: ทุกชิ้นมีที่ของมัน

การรับรองความถูกต้องที่ปลอดภัย รหัสเซิร์ฟเวอร์มีในตัว `connection-token` กลไกในการป้องกันการเข้าถึงโดยไม่ได้รับอนุญาต แต่ละเซสชันมีโทเค็นที่ไม่ซ้ำกันเพื่อให้เฉพาะผู้ใช้ที่ได้รับอนุญาตเท่านั้นที่สามารถเปิดโปรแกรมแก้ไขได้ การรักษาความปลอดภัยเป็นหนึ่งในสิ่งที่คุณจะประทับใจเมื่อคุณมีมันแล้วเท่านั้น

## การออกแบบสถาปัตยกรรม

การรวม VSCode Web ของ HagiCode ใช้สถาปัตยกรรมที่แยกส่วนหน้า/ส่วนหลัง

### ชั้นบริการส่วนหน้า

ส่วนหน้าตัดการโต้ตอบกับส่วนหลังผ่าน `vscodeServerService.ts`:

```typescript
// Open project
export async function openProjectInCodeServer(
  id: string,
  currentInterfaceLanguage?: string,
): Promise<VsCodeServerLaunchResponseDto>

// Open vault
export async function openVaultInCodeServer(
  id: string,
  path?: string,
  currentInterfaceLanguage?: string,
): Promise<VsCodeServerLaunchResponseDto>
```

ความแตกต่างระหว่างสองวิธีนี้ตรงไปตรงมา: `openProjectInCodeServer` เปิดทั้งโครงการในขณะที่ `openVaultInCodeServer` เปิดเส้นทางเฉพาะภายในห้องนิรภัย สำหรับโปรเจ็กต์ที่มีพื้นที่เก็บข้อมูลหลาย MonoSpecs ระบบจะสร้างไฟล์พื้นที่ทำงานโดยอัตโนมัติ ความรับผิดชอบที่ชัดเจนมักจะเพียงพอเมื่อแต่ละส่วนทำงานได้ดี

### ชั้นบริการแบ็กเอนด์

แบ็กเอนด์ `VaultAppService.cs` ใช้ตรรกะหลัก:

```csharp
public async Task<VsCodeServerLaunchResponseDto> OpenInCodeServerAsync(
    string id,
    string? relativePath = null,
    string? currentInterfaceLanguage = null,
    CancellationToken cancellationToken = default)
{
    // 1. Get settings and check whether the feature is enabled
    var settings = await _vsCodeServerSettingsService.GetResolvedSettingsAsync(cancellationToken);
    if (!settings.Enabled) {
        throw new BusinessException(VsCodeServerErrorCodes.Disabled, "VSCode Server is disabled.");
    }

    // 2. Get vault and resolve the launch directory
    var vault = await RequireVaultAsync(id, cancellationToken);
    var launchDirectory = ResolveLaunchDirectory(vault, relativePath);

    // 3. Ensure code-server is running and get runtime info
    var runtime = await _vsCodeServerManager.EnsureStartedAsync(settings, cancellationToken);

    // 4. Resolve language settings
    var language = _vsCodeServerSettingsService.ResolveLaunchLanguage(
        settings.Language,
        currentInterfaceLanguage);

    // 5. Build launch URL
    return new VsCodeServerLaunchResponseDto {
        LaunchUrl = AppendQueryString(runtime.BaseUrl, new Dictionary<string, string?> {
            ["folder"] = launchDirectory,
            ["tkn"] = runtime.ConnectionToken,
            ["vscode-lang"] = language,
        }),
        ConnectionToken = runtime.ConnectionToken,
        OpenMode = "folder",
        Runtime = VsCodeServerSettingsService.MapRuntime(
            await _vsCodeServerManager.GetRuntimeSnapshotAsync(cancellationToken)),
    };
}
```

วิธีการนี้มีความรับผิดชอบที่ชัดเจนมาก: ตรวจสอบการตั้งค่า แก้ไขเส้นทาง เริ่มบริการ และสร้าง URL ในหมู่พวกเขา `ResolveLaunchDirectory` วิธีการดำเนินการตรวจสอบความปลอดภัยของเส้นทางเพื่อป้องกันการโจมตีการข้ามเส้นทาง โค้ดอาจรู้สึกเหมือนบทกวีเมื่อทุกบรรทัดมีจุดประสงค์

### การจัดการรันไทม์อัตโนมัติ

แบ็กเอนด์จัดการกระบวนการโค้ดเซิร์ฟเวอร์ผ่าน `VsCodeServerManager`:

- ตรวจสอบสถานะกระบวนการ
- เริ่มบริการที่หยุดโดยอัตโนมัติ
- ส่งคืนสแนปช็อตรันไทม์ เช่น พอร์ต รหัสกระบวนการ และเวลาเริ่มต้น

การออกแบบนี้ช่วยให้ระบบจัดการวงจรการใช้งานเซิร์ฟเวอร์โค้ดได้โดยอัตโนมัติ ดังนั้นผู้ใช้จึงไม่จำเป็นต้องจัดการกระบวนการบริการด้วยตนเอง ชีวิตมีความซับซ้อนเพียงพอแล้ว สิ่งใดก็ตามที่สามารถทำให้เป็นอัตโนมัติได้ควรจะเป็น

### การซิงโครไนซ์ภาษา

HagiCode รองรับอินเทอร์เฟซหลายภาษา และโค้ดเซิร์ฟเวอร์จำเป็นต้องปฏิบัติตามการตั้งค่านั้น ระบบรองรับโหมดภาษาสามโหมด:

- `follow`: ทำตามภาษาอินเทอร์เฟซปัจจุบัน
- `zh-CN`: แก้ไขเป็นภาษาจีน
- `en-US`: แก้ไขเป็นภาษาอังกฤษ

การตั้งค่าจะถูกส่งไปยังเซิร์ฟเวอร์รหัสผ่านทาง `vscode-lang` พารามิเตอร์ URL เพื่อให้ภาษาของตัวแก้ไขมีความสอดคล้องกับอินเทอร์เฟซ HagiCode ภาษาจะรู้สึกดีที่สุดเมื่อรวมเป็นหนึ่งเดียว

### พื้นที่ทำงานหลายพื้นที่เก็บข้อมูล MonoSpecs

สำหรับโปรเจ็กต์ MonoSpecs ซึ่งมีที่เก็บข้อมูลย่อยหลายรายการภายใน monorepo เดียว ระบบจะสร้าง `.code-workspace` ไฟล์:

```csharp
private async Task<string> CreateWorkspaceFileAsync(Project project, Guid projectId)
{
    var folders = await ResolveWorkspaceFoldersAsync(project.Path);
    var workspaceDocument = new {
        folders = folders.Select(path => new { path }).ToArray(),
    };
    // Generate workspace file...
}
```

ทำให้สามารถแก้ไขที่เก็บข้อมูลย่อยหลายรายการในอินสแตนซ์โค้ดเซิร์ฟเวอร์เดียวกันได้ ซึ่งเป็นประโยชน์อย่างยิ่งสำหรับโปรเจ็กต์ Monorepo ขนาดใหญ่ ที่เก็บข้อมูลหลายรายการในหน้าต่างเดียวอาจรู้สึกเหมือนมีเรื่องราวมากมายรวมอยู่ในหนังสือเล่มเดียวกัน

## บูรณาการส่วนหน้า

ส่วนหน้า HagiCode ใช้ React + TypeScript และการรวมโค้ดเซิร์ฟเวอร์นั้นไม่ได้ซับซ้อนเป็นพิเศษ

### ปุ่มดำเนินการด่วน

เพิ่มปุ่มเซิร์ฟเวอร์โค้ดลงในการ์ดโครงการ:

```tsx
// QuickActionsZone.tsx
<Button
  size="sm"
  variant="default"
  onClick={() => onAction({ type: 'open-code-server' })}
>
  <Globe className="h-3 w-3 mr-1" />
  <span className="text-xs">{t('project.openCodeServer')}</span>
</Button>
```

ปุ่มนี้จะทริกเกอร์การดำเนินการแบบเปิดและเรียก API แบ็กเอนด์เพื่อรับ URL เปิดใช้งาน ปุ่มเดียว หนึ่งการกระทำ ตรงไปตรงมาและเรียบง่าย

### การจัดการกับ Open Action

```tsx
const handleAction = async (action: ProjectAction) => {
  if (action.type === 'open-code-server') {
    const response = await openProjectInCodeServer(project.id, i18n.language);
    window.open(response.launchUrl, '_blank', 'noopener,noreferrer');
  }
};
```

ใช้ `window.open` เพื่อเปิดเซิร์ฟเวอร์รหัสในแท็บใหม่ ที่ `noopener,noreferrer` พารามิเตอร์ให้ความปลอดภัยเป็นพิเศษ ในเรื่องความปลอดภัย ไม่มีคำว่าระมัดระวังจนเกินไป

### รายการแก้ไขห้องนิรภัย

เพิ่มปุ่มแก้ไขที่คล้ายกันในรายการห้องนิรภัย:

```tsx
const handleEditVault = async (vault: VaultItemDto) => {
  const response = await openVaultInCodeServer(vault.id);
  window.open(response.launchUrl, '_blank', 'noopener,noreferrer');
};
```

โปรเจ็กต์และห้องนิรภัยใช้กลไกแบบเปิดเดียวกัน ซึ่งทำให้การโต้ตอบมีความสอดคล้องกัน ความสม่ำเสมอมีความสำคัญพอๆ กับคุณลักษณะของตัวเอง

## ลอจิกการสร้าง URL

รูปแบบ URL สำหรับโค้ดเซิร์ฟเวอร์มีรายละเอียดเล็กๆ น้อยๆ ที่น่าสังเกต

**โหมดโฟลเดอร์**:
```
http://{host}:{port}/?folder={path}&tkn={token}&vscode-lang={lang}
```

**โหมดพื้นที่ทำงาน**:
```
http://{host}:{port}/?workspace={workspacePath}&tkn={token}&vscode-lang={lang}
```

ที่นี่ `tkn` คือโทเค็นการเชื่อมต่อ มันถูกสร้างขึ้นโดยอัตโนมัติทุกครั้งที่โค้ดเซิร์ฟเวอร์เริ่มทำงาน เพื่อให้มั่นใจถึงการเข้าถึงที่ปลอดภัย ที่ `vscode-lang` พารามิเตอร์ควบคุมภาษา UI ของตัวแก้ไข พารามิเตอร์แต่ละตัวเหล่านี้มีบทบาทในการเล่น

## สถานการณ์การใช้งาน

### สถานการณ์ที่หนึ่ง: การตรวจสอบโค้ดที่ได้รับความช่วยเหลือจาก AI

ผู้ใช้พูดคุยกับ HagiCode จากนั้น AI จะวิเคราะห์โค้ดโปรเจ็กต์และค้นหาปัญหาที่อาจเกิดขึ้น จากนั้นผู้ใช้คลิกปุ่ม "เปิดในเซิร์ฟเวอร์โค้ด" เพื่อเปิดตัวแก้ไขโดยตรงในเบราว์เซอร์ ตรวจสอบไฟล์ที่ได้รับผลกระทบ แก้ไข และกลับไปที่ HagiCode เพื่อดำเนินการสนทนาต่อ โฟลว์ทั้งหมดเกิดขึ้นในเบราว์เซอร์โดยไม่ต้องสลับแอปพลิเคชัน รู้สึกนุ่มนวลในแบบที่น้ำไหลรู้สึกนุ่มนวล

### สถานการณ์ที่สอง: การแก้ไขสื่อการเรียนรู้ในห้องนิรภัย

ผู้ใช้สร้างห้องนิรภัยสำหรับศึกษาโครงการโอเพ่นซอร์ส และต้องการเพิ่มบันทึกการศึกษาภายใต้ `docs/` ไดเรกทอรี ด้วยโค้ดเซิร์ฟเวอร์ พวกเขาสามารถแก้ไขไฟล์ Markdown ได้โดยตรงในเบราว์เซอร์ บันทึก และปล่อยให้ HagiCode อ่านบันทึกย่อที่อัปเดตทันที สิ่งนี้มีประโยชน์อย่างยิ่งสำหรับการสร้างฐานความรู้ส่วนบุคคล ความรู้จะมีค่ามากขึ้นเท่านั้นเมื่อคุณสะสมมันมากขึ้น

### สถานการณ์ที่สาม: การพัฒนา Multi-Repository ของ MonoSpecs

โปรเจ็กต์ MonoSpecs มีที่เก็บย่อยหลายรายการ และโค้ดเซิร์ฟเวอร์จะสร้างพื้นที่ทำงานหลายโฟลเดอร์โดยอัตโนมัติ ในเบราว์เซอร์ ผู้ใช้สามารถแก้ไขโค้ดในที่เก็บข้อมูลหลายแห่งพร้อมกัน จากนั้นส่งมอบการเปลี่ยนแปลงกลับไปยังที่เก็บ Git ของตน เวิร์กโฟลว์นี้เหมาะอย่างยิ่งสำหรับการเปลี่ยนแปลงที่ต้องขยายพื้นที่เก็บข้อมูลหลายแห่ง การแก้ไขที่เก็บข้อมูลหลายรายการพร้อมกันต้องใช้เทคนิคเล็กน้อย เช่นเดียวกับการจัดการหลายงานในเวลาเดียวกัน

## ข้อควรพิจารณาด้านความปลอดภัย

เมื่อใช้การรวมโค้ดเซิร์ฟเวอร์ การรักษาความปลอดภัยสมควรได้รับความสนใจเป็นพิเศษ หากการรักษาความปลอดภัยผิดพลาด คุณจะสังเกตเห็นว่าสายเกินไปเสมอ

### โทเค็นการเชื่อมต่อ

ที่ `connection-token` ถูกสร้างขึ้นแบบสุ่มและไม่ควรถูกเปิดเผย เหมาะที่สุดที่จะใช้ภายใต้ HTTPS เพื่อป้องกันไม่ให้โทเค็นถูกดักจับโดยบุคคลที่อยู่ตรงกลาง ข้อมูลที่ละเอียดอ่อนควรค่าแก่การปกป้องอย่างเหมาะสม

### ความปลอดภัยของเส้นทางไฟล์

แบ็กเอนด์ใช้การตรวจสอบการข้ามเส้นทาง:

```csharp
private static string ResolveLaunchDirectory(VaultRegistryEntry vault, string? relativePath)
{
    var vaultRoot = EnsureTrailingSeparator(Path.GetFullPath(vault.PhysicalPath));
    var combinedPath = Path.GetFullPath(Path.Combine(vaultRoot, relativePath ?? "."));
    if (!combinedPath.StartsWith(vaultRoot, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode, "Relative path traversal detected.");
    }
    return combinedPath;
}
```

รหัสนี้ช่วยให้แน่ใจว่าผู้ใช้ไม่สามารถใช้งานได้ `../` หรือรูปแบบที่คล้ายกันในการเข้าถึงไฟล์นอกไดเร็กทอรี Vault การตรวจสอบขอบเขตทำได้ดีกว่าการข้ามเสมอ

### การควบคุมการอนุญาต

กระบวนการโค้ดเซิร์ฟเวอร์ควรทำงานโดยมีสิทธิ์ผู้ใช้ที่เหมาะสม เพื่อไม่ให้สามารถเข้าถึงไฟล์ระบบที่ละเอียดอ่อนได้ เป็นการดีที่สุดที่จะรันบริการโค้ดเซิร์ฟเวอร์ภายใต้ผู้ใช้เฉพาะ การควบคุมสิทธิ์เป็นหนึ่งในพื้นฐานที่คุณควรยึดถือไว้เสมอ

## การเพิ่มประสิทธิภาพการทำงาน

เซิร์ฟเวอร์โค้ดใช้ทรัพยากรเซิร์ฟเวอร์ ดังนั้นนี่คือคำแนะนำในการเพิ่มประสิทธิภาพบางส่วน:

- ตรวจสอบการใช้งาน CPU และหน่วยความจำ และปรับขีดจำกัดทรัพยากรเมื่อจำเป็น
- โครงการขนาดใหญ่อาจต้องใช้เวลาหมดเวลานานขึ้น
- ใช้การล้างข้อมูลการหมดเวลาเซสชันอัตโนมัติเพื่อปล่อยทรัพยากร
- พิจารณาแคชเพื่อลดการคำนวณซ้ำ

HagiCode มี API การตรวจสอบสถานะรันไทม์ และส่วนหน้าสามารถเรียกใช้ได้ `getVsCodeServerSettings()` เพื่อดึงข้อมูลสถานะปัจจุบัน:

```typescript
const { settings, runtime } = await getVsCodeServerSettings();
// runtime.status: 'disabled' | 'stopped' | 'starting' | 'running' | 'unhealthy'
// runtime.baseUrl: "http://localhost:8080"
// runtime.processId: 12345
```

การออกแบบนี้ช่วยให้ผู้ใช้เข้าใจสถานะความสมบูรณ์ของโค้ดเซิร์ฟเวอร์ได้อย่างชัดเจน และค้นหาปัญหาได้อย่างรวดเร็วเมื่อมีสิ่งผิดปกติเกิดขึ้น เมื่อสถานะปรากฏ ผู้คนจะรู้สึกควบคุมได้มากขึ้น

## รายละเอียดประสบการณ์ผู้ใช้

ในระหว่างการใช้งาน เราค้นพบรายละเอียดบางอย่างที่ส่งผลต่อประสบการณ์ผู้ใช้อย่างเห็นได้ชัดและสมควรได้รับความสนใจเป็นพิเศษ

การเปิดเซิร์ฟเวอร์รหัสเป็นครั้งแรกอาจต้องรอการเริ่มต้นระบบ และความล่าช้านั้นอาจมีตั้งแต่ไม่กี่วินาทีถึงครึ่งนาที เป็นความคิดที่ดีที่จะแสดงสถานะการโหลดในส่วนหน้าเพื่อให้ผู้ใช้ทราบว่าระบบยังคงทำงานอยู่ การรอจะง่ายขึ้นเมื่อมีคำติชม

เบราว์เซอร์อาจบล็อกป๊อปอัป ดังนั้นผู้ใช้ควรได้รับแจ้งให้อนุญาตด้วยตนเอง ในการเปิดตัวครั้งแรก HagiCode จะแสดงคำแนะนำที่อธิบายวิธีการให้สิทธิ์เบราว์เซอร์ที่จำเป็น ประสบการณ์ผู้ใช้มักจะอยู่ในรายละเอียดเล็กๆ น้อยๆ เหล่านี้

นอกจากนี้ ยังเป็นความคิดที่ดีที่จะแสดงสถานะรันไทม์ เช่น การเริ่มต้น ทำงาน หรือข้อผิดพลาด เพื่อว่าเมื่อเกิดปัญหาขึ้น ผู้ใช้สามารถบอกได้อย่างรวดเร็วว่าปัญหาอยู่ที่ฝั่งเซิร์ฟเวอร์หรือในการดำเนินการของตนเอง การรู้ว่าปัญหาอยู่ที่จุดใดอย่างน้อยก็ช่วยให้คุณมีจุดเริ่มต้นได้

## ตัวอย่างการกำหนดค่า

การกำหนดค่าสำหรับโค้ดเซิร์ฟเวอร์นั้นไม่ซับซ้อน:

```json
{
  "vscodeServer": {
    "enabled": true,
    "host": "0.0.0.0",
    "port": 8080,
    "language": "follow"
  }
}
```

`enabled` ควบคุมว่าคุณสมบัตินั้นเปิดอยู่หรือไม่ `host` และ `port` กำหนดที่อยู่การฟังและ `language` ตั้งค่าโหมดภาษา การตั้งค่าเหล่านี้สามารถแก้ไขได้ผ่าน UI และมีผลทันที สิ่งง่ายๆ มักจะเป็นสิ่งที่ใช้ง่ายที่สุด

## บทสรุป

การผสานรวม VSCode Web ของ HagiCode มอบโซลูชันที่หรูหรา โดยช่วยให้ผู้ช่วย AI และประสบการณ์การแก้ไขโค้ดเชื่อมต่อกันได้อย่างราบรื่น ด้วยการรวมโค้ดเซิร์ฟเวอร์เข้ากับเบราว์เซอร์ ผู้ใช้สามารถดำเนินการกับผลการวิเคราะห์ AI ได้อย่างรวดเร็ว และดำเนินการขั้นตอนทั้งหมดตั้งแต่การวิเคราะห์ไปจนถึงการแก้ไขในเซสชันเบราว์เซอร์เดียวกัน

โซลูชันนี้นำมาซึ่งข้อดีที่สำคัญหลายประการ: ประสบการณ์ที่เป็นหนึ่งเดียว เนื่องจากโครงการและ Vaults ใช้กลไกแบบเปิดเดียวกัน การสนับสนุนหลายพื้นที่เก็บข้อมูลเนื่องจากโครงการ MonoSpecs จะสร้างพื้นที่ทำงานโดยอัตโนมัติ และการรักษาความปลอดภัยที่ควบคุมได้ ต้องขอบคุณการตรวจสอบสถานะรันไทม์และการตรวจสอบความปลอดภัยของเส้นทาง

แนวทางที่แบ่งปันในบทความนี้เป็นสิ่งที่ HagiCode กลั่นกรองมาจากงานการพัฒนาจริง หากคุณพบว่าโซลูชันนี้มีคุณค่า นั่นแสดงว่าการปฏิบัติงานด้านวิศวกรรมของเรากำลังทำสิ่งที่ถูกต้อง และ HagiCode เองก็อาจคุ้มค่าที่จะพิจารณาอย่างใกล้ชิด เครื่องมือดีๆ สมควรให้ผู้คนเห็นมากขึ้น
