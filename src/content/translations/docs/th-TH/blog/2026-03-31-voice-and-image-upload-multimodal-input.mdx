## การพิมพ์ช้ากว่าการพูด และการพูดคุยช้ากว่าภาพหน้าจอ - แนวทางปฏิบัติอินพุตหลายรูปแบบสำหรับผู้ช่วยการเข้ารหัส AI

> การเขียนโค้ดมีการจำกัดความเร็วไม่ว่าคุณจะพิมพ์เร็วแค่ไหนก็ตาม บางครั้งบางสิ่งที่คุณอาจพูดในประโยคเดียวอาจใช้เวลานานในการพิมพ์ บางครั้งภาพหน้าจอเดียวจะอธิบายทุกอย่าง แต่คุณยังคงต้องอธิบายด้วยข้อความจำนวนมาก บทความนี้พูดถึงสิ่งที่เราพบขณะสร้าง [ฮากิโค้ด](https://hagicode.com)จากการรู้จำคำพูดไปจนถึงการอัพโหลดรูปภาพ ท้ายที่สุดแล้ว เราแค่อยากทำให้ผู้ช่วยเขียนโค้ด AI ใช้งานง่ายขึ้นอีกหน่อย

## พื้นหลัง

ในขณะที่สร้าง HagiCode เราสังเกตเห็นปัญหา - หรือค่อนข้างจะเป็นปัญหาที่เกิดขึ้นตามธรรมชาติเมื่อผู้คนเริ่มใช้งานมันอย่างหนัก: การพึ่งพาการพิมพ์เพียงอย่างเดียวอาจทำให้เหนื่อย

ลองคิดดู: การโต้ตอบระหว่างผู้ใช้และตัวแทนถือเป็นสถานการณ์หลัก แต่หากการแลกเปลี่ยนทุกครั้งต้องพิมพ์แป้นพิมพ์ไม่หยุด ประสิทธิภาพก็ไม่ได้ดีนัก:

1. **การพิมพ์ช้าเกินไป**: สำหรับปัญหาที่ซับซ้อน เช่น ข้อความแสดงข้อผิดพลาดหรือปัญหา UI การพิมพ์ทุกอย่างอาจใช้เวลาครึ่งนาที ในขณะที่การพูดออกเสียงอาจใช้เวลาสิบวินาที ช่องว่างนั้นมีจริง

2. **รูปภาพมีความตรงกว่า**: บางครั้ง UI ก็แสดงข้อผิดพลาด บางครั้งคุณต้องการเปรียบเทียบแบบร่างการออกแบบ และบางครั้งคุณจำเป็นต้องแสดงโครงสร้างโค้ด "ภาพแทนคำพูดได้นับพันคำ" อาจเป็นคำพูดเก่าๆ แต่มันก็ยังคงอยู่ การให้ AI "เห็น" ปัญหาโดยตรงนั้นชัดเจนกว่าการอธิบายเป็นเวลานานๆ

3. **การโต้ตอบควรให้ความรู้สึกเป็นธรรมชาติ**: ผู้ช่วย AI ยุคใหม่ควรรองรับข้อความ เสียง และรูปภาพ ผู้ใช้ควรสามารถเลือกวิธีการป้อนข้อมูลใดก็ได้ที่ให้ความรู้สึกเป็นธรรมชาติที่สุด

ดังนั้นเราจึงตัดสินใจเพิ่มการรองรับการรู้จำเสียงและการอัพโหลดรูปภาพให้กับ HagiCode เพื่อให้การโต้ตอบของตัวแทนสะดวกยิ่งขึ้น หากผู้ใช้สามารถพิมพ์น้อยลงเล็กน้อย นั่นก็ถือว่าชนะแล้ว

HagiCode เป็นโปรเจ็กต์ผู้ช่วยเขียนโค้ด AI แบบโอเพ่นซอร์สโดยมีเป้าหมายง่ายๆ: ใช้ AI เพื่อปรับปรุงประสิทธิภาพการพัฒนา ขณะที่เราสร้างอย่างต่อเนื่อง ก็เห็นได้ชัดว่าผู้ใช้ต้องการอินพุตแบบหลายรูปแบบอย่างมาก บางครั้งการพูดประโยคเดียวก็เร็วกว่าการพิมพ์ย่อหน้ายาว และบางครั้งภาพหน้าจอก็ชัดเจนกว่าคำอธิบายที่ยาวมาก

ความต้องการเหล่านั้นผลักดันเราไปข้างหน้า และนั่นทำให้ฟีเจอร์ต่างๆ เช่น การรู้จำคำพูดและการอัพโหลดรูปภาพเป็นรูปเป็นร่างขึ้นมาในที่สุด ขณะนี้ผู้ใช้สามารถโต้ตอบกับ AI ได้อย่างเป็นธรรมชาติที่สุด และรู้สึกดี

## การวิเคราะห์

### ความท้าทายทางเทคนิคในการรู้จำเสียง

เมื่อสร้างการรู้จำเสียง เราประสบปัญหายุ่งยาก: **เบราว์เซอร์ WebSocket API ไม่รองรับส่วนหัว HTTP ที่กำหนดเอง**

บริการรู้จำคำพูดที่เราเลือกคือ Doubao Speech Recognition API ของ ByteDance ขออภัย API นี้ต้องการข้อมูลการตรวจสอบสิทธิ์ เช่น `accessToken` และ `secretKey` ที่จะส่งผ่านส่วนหัว HTTP ซึ่งทำให้เกิดความขัดแย้งทางเทคนิคในทันที:

```javascript
// The browser WebSocket API does not support this approach
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // Not supported
  }
});
```

โดยพื้นฐานแล้วเรามีสองทางเลือก:

1. **วิธีการใช้พารามิเตอร์การสืบค้น URL**: ใส่ข้อมูลการตรวจสอบสิทธิ์ใน URL
   - ข้อดีคือง่ายต่อการปฏิบัติ
   - ข้อเสียคือข้อมูลประจำตัวถูกเปิดเผยต่อส่วนหน้าซึ่งไม่ปลอดภัย API บางตัวยังต้องมีการตรวจสอบสิทธิ์ตามส่วนหัวด้วย

2. **แนวทางพร็อกซีแบ็กเอนด์**: ใช้พร็อกซี WebSocket บนแบ็กเอนด์
   - ข้อดีคือข้อมูลประจำตัวยังคงเก็บไว้อย่างปลอดภัยบนแบ็กเอนด์ และโซลูชันเข้ากันได้กับข้อกำหนด API อย่างสมบูรณ์
   - ข้อเสียคือการใช้งานนั้นซับซ้อนกว่าเล็กน้อย

ท้ายที่สุด เราเลือก **แนวทางพร็อกซีแบ็กเอนด์** การรักษาความปลอดภัยไม่ใช่สิ่งที่คุณประนีประนอม

### ข้อกำหนดด้านการทำงานสำหรับการอัพโหลดรูปภาพ

ข้อกำหนดของเราสำหรับการอัปโหลดรูปภาพนั้นค่อนข้างตรงไปตรงมา:

1. **วิธีการอัปโหลดหลายวิธี**: คลิกเพื่อเลือกไฟล์ ลากและวาง และวางจากคลิปบอร์ด
2. **การตรวจสอบไฟล์**: ข้อจำกัดประเภท (PNG, JPG, WebP, GIF) และขีดจำกัดขนาด (5-10 MB) เป็นข้อกำหนดพื้นฐาน
3. **ประสบการณ์ผู้ใช้**: อัปโหลดความคืบหน้า การแสดงตัวอย่าง และข้อความแสดงข้อผิดพลาด เพื่อให้ผู้ใช้ทราบอยู่เสมอว่าเกิดอะไรขึ้น
4. **ความปลอดภัย**: การตรวจสอบฝั่งเซิร์ฟเวอร์และป้องกันการอัพโหลดไฟล์ที่เป็นอันตรายถือเป็นสิ่งสำคัญ

## โซลูชั่น

### การรู้จำเสียง: สถาปัตยกรรมพร็อกซี WebSocket

เราได้ออกแบบสถาปัตยกรรมสามชั้นสำหรับการรู้จำเสียงและพบเส้นทางที่ใช้ได้:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**การใช้งานองค์ประกอบหลัก**:

1. **โปรเซสเซอร์ Frontend AudioWorklet**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Resample to 16 kHz (required by the Doubao API)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Accumulate samples into 500 ms chunks
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Convert to 16-bit PCM and send
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **ตัวจัดการ WebSocket แบ็กเอนด์** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **ส่วนหน้า `VoiceTextArea` ส่วนประกอบ**:

```tsx
export const VoiceTextArea = forwardRef<HTMLTextAreaElement, VoiceTextAreaProps>(
  ({ value, onChange, onTextRecognized, maxDuration }, ref) => {

);
```

### การอัพโหลดรูปภาพ: คอมโพเนนต์การอัพโหลดแบบหลายวิธี

เราสร้างองค์ประกอบการอัปโหลดรูปภาพที่มีคุณสมบัติครบถ้วนโดยรองรับวิธีการอัปโหลดทั้งสามวิธี ซึ่งครอบคลุมสถานการณ์ทั่วไปที่ผู้ใช้พบเจอ

**คุณสมบัติหลัก**:

1. **วิธีการอัพโหลดสามวิธี**:

```tsx
// Click to upload
const handleClick = () => fileInputRef.current?.click();

// Drag-and-drop upload
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Clipboard paste
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **การตรวจสอบส่วนหน้า**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **ตัวจัดการการอัปโหลดแบ็กเอนด์** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Validation
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Save file
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## คู่มือการปฏิบัติ

### วิธีใช้การรู้จำเสียง

1. **กำหนดค่าบริการรู้จำเสียงพูด**:
   - เปิดหน้าการตั้งค่าการรู้จำคำพูด
   - กำหนดค่าคำพูด Doubao `AppId` และ `AccessToken`
   - คุณสามารถเลือกกำหนดค่าคำที่นิยมเพื่อปรับปรุงความแม่นยำในการจดจำสำหรับข้อกำหนดเฉพาะโดเมน

2. **ใช้ในช่องป้อนข้อมูล**:
   - คลิกไอคอนไมโครโฟนทางด้านซ้ายของช่องป้อนข้อมูล
   - เริ่มพูดหลังจากภาพเคลื่อนไหวรูปคลื่นปรากฏขึ้น
   - คลิกไอคอนอีกครั้งเพื่อหยุดการบันทึก
   - ข้อความที่รู้จักจะถูกแทรกที่ตำแหน่งเคอร์เซอร์โดยอัตโนมัติ

3. **ตัวอย่างการกำหนดค่าคำที่นิยม**:

```text
TypeScript
React
useState
useEffect
```

### วิธีใช้การอัพโหลดรูปภาพ

1. **วิธีการอัพโหลด**:
   - คลิกปุ่มอัปโหลดเพื่อเลือกไฟล์
   - ลากรูปภาพลงในพื้นที่อัพโหลดโดยตรง
   - ใช้ `Ctrl+V` เพื่อวางภาพหน้าจอจากคลิปบอร์ด

2. **รูปแบบที่รองรับ**: PNG, JPG, JPEG, WebP, GIF
3. **ขีดจำกัดขนาด**: 5 MB โดยค่าเริ่มต้น (กำหนดค่าได้)

### หมายเหตุ

1. **การรู้จำคำพูด**:
   - ต้องได้รับอนุญาตจากไมโครโฟน
   - ใช้ในสภาพแวดล้อมที่เงียบสงบเมื่อเป็นไปได้
   - ระยะเวลาการบันทึกสูงสุดที่รองรับคือ 300 วินาทีโดยค่าเริ่มต้น (กำหนดค่าได้)

2. **อัพโหลดรูปภาพ**:
   - รองรับเฉพาะรูปแบบรูปภาพทั่วไปเท่านั้น
   - ใส่ใจกับการจำกัดขนาดไฟล์
   - ภาพที่อัปโหลดจะได้รับ URL ตัวอย่างโดยอัตโนมัติ

3. **ข้อควรพิจารณาด้านความปลอดภัย**:
   - ข้อมูลรับรองการรู้จำคำพูดจะถูกจัดเก็บไว้ในแบ็กเอนด์
   - การอัปโหลดรูปภาพต้องผ่านการตรวจสอบฝั่งเซิร์ฟเวอร์อย่างเข้มงวด
   - แนะนำให้ใช้ HTTPS/WSS ในสภาพแวดล้อมการใช้งานจริง

## บทสรุป

หลังจากเพิ่มการรู้จำคำพูดและการอัพโหลดรูปภาพ ประสบการณ์ผู้ใช้ HagiCode ก็ดีขึ้นอย่างเห็นได้ชัด ขณะนี้ผู้ใช้สามารถโต้ตอบกับ AI ได้อย่างเป็นธรรมชาติมากขึ้น โดยพูดแทนการพิมพ์ และแชร์ภาพหน้าจอแทนการอธิบายทุกอย่างด้วยตนเอง รู้สึกเหมือนได้ค้นพบวิธีสื่อสารที่สะดวกสบายมากขึ้นในที่สุด

ในขณะที่สร้างฟีเจอร์นี้ เราประสบปัญหาที่เบราว์เซอร์ WebSocket API ไม่รองรับส่วนหัวที่กำหนดเอง ท้ายที่สุด เราก็แก้ไขมันด้วยวิธีพร็อกซีแบ็กเอนด์ โซลูชันดังกล่าวไม่เพียงแต่รักษาความปลอดภัยเท่านั้น แต่ยังวางรากฐานสำหรับการผสานรวมบริการ WebSocket ที่ผ่านการรับรองความถูกต้องอื่นๆ ในภายหลัง

องค์ประกอบการอัปโหลดรูปภาพยังได้รับประโยชน์จากการรองรับวิธีการอัปโหลดหลายวิธี ทำให้ผู้ใช้สามารถเลือกวิธีที่สะดวกที่สุดในขณะนั้นได้ การคลิก ลาก หรือวางงานทั้งหมด และแต่ละเส้นทางจะทำให้งานเสร็จเร็ว

"การพิมพ์ช้ากว่าการพูด และการพูดคุยช้ากว่าภาพหน้าจอ" เหมาะกับธีมนี้ค่อนข้างดี หากคุณกำลังสร้างผลิตภัณฑ์ผู้ช่วย AI ที่คล้ายกัน ฉันหวังว่าประสบการณ์เหล่านี้จะช่วยคุณได้ แม้ว่าจะเพียงเล็กน้อยก็ตาม
