## การวิเคราะห์ทางเทคนิคของระบบทักษะ HagiCode: การสร้างแพลตฟอร์มการจัดการทักษะ AI ที่ปรับขนาดได้

> บทความนี้จะเจาะลึกเกี่ยวกับสถาปัตยกรรมและการใช้งานระบบการจัดการทักษะในโครงการ HagiCode ซึ่งครอบคลุมรายละเอียดทางเทคนิคที่อยู่เบื้องหลังความสามารถหลัก 4 ประการ ได้แก่ การจัดการระดับโลกในท้องถิ่น การค้นหาในตลาด คำแนะนำอัจฉริยะ และการจัดการผู้ให้บริการที่เชื่อถือได้

## พื้นหลัง

ในด้านผู้ช่วยเขียนโค้ด AI วิธีขยายขอบเขตความสามารถของ AI ถือเป็นคำถามหลักมาโดยตลอด Claude Code เองก็แข็งแกร่งอยู่แล้วในด้านความช่วยเหลือด้านโค้ด แต่ทีมพัฒนาที่แตกต่างกันและกลุ่มเทคโนโลยีที่แตกต่างกัน มักจะต้องการความสามารถพิเศษสำหรับสถานการณ์เฉพาะ เช่น การจัดการการปรับใช้ Docker การเพิ่มประสิทธิภาพฐานข้อมูล หรือการสร้างส่วนประกอบส่วนหน้า นั่นคือจุดที่ระบบทักษะมีความสำคัญอย่างยิ่ง

ในระหว่างการพัฒนาโครงการ HagiCode เราพบกับความท้าทายที่คล้ายกัน: เราจะปล่อยให้ Claude Code "เรียนรู้" ทักษะทางวิชาชีพใหม่ ๆ เหมือนบุคคลได้อย่างไร ในขณะที่ยังคงรักษาประสบการณ์ผู้ใช้ที่แข็งแกร่งและการบำรุงรักษาทางวิศวกรรมที่ดีได้อย่างไร ปัญหานี้ทั้งยากและง่ายในแบบของตัวเอง สำหรับคำถามนั้น เราได้ออกแบบและใช้งานระบบการจัดการทักษะที่สมบูรณ์

บทความนี้จะอธิบายเกี่ยวกับสถาปัตยกรรมทางเทคนิคและการใช้งานหลักของระบบโดยละเอียด มีไว้สำหรับนักพัฒนาที่สนใจในความสามารถในการขยาย AI และการรวมเครื่องมือบรรทัดคำสั่ง มันอาจจะมีประโยชน์สำหรับคุณหรืออาจจะไม่ แต่อย่างน้อยตอนนี้มันก็ถูกจดบันทึกไว้แล้ว

## เกี่ยวกับ ฮากิโค้ด

แนวทางที่แบ่งปันในบทความนี้มาจากประสบการณ์จริงของเราใน [ฮากิโค้ด](https://hagicode.com) โครงการ HagiCode เป็นโครงการผู้ช่วยเขียนโค้ด AI แบบโอเพ่นซอร์สที่ออกแบบมาเพื่อช่วยทีมพัฒนาปรับปรุงประสิทธิภาพทางวิศวกรรม สแต็กของโปรเจ็กต์ประกอบด้วย ASP.NET Core, เฟรมเวิร์กแบบกระจายของ Orleans, ส่วนหน้า TanStack Start + React และระบบย่อยการจัดการทักษะที่แนะนำในบทความนี้

พื้นที่เก็บข้อมูล GitHub คือ [HagiCode-org/site](https://github.com/HagiCode-org/site). หากคุณพบว่าแนวทางทางเทคนิคในบทความนี้มีคุณค่า คุณสามารถให้คะแนนดาวได้ ดาวจำนวนมากมักจะทำให้อารมณ์ดีขึ้น

## ภาพรวมสถาปัตยกรรมระบบ

ระบบทักษะใช้สถาปัตยกรรมที่แยกส่วนหน้า-ส่วนหลัง ไม่มีอะไรลึกลับเป็นพิเศษเกี่ยวกับเรื่องนั้น

**ส่วนหน้า** ใช้ TanStack Start + React เพื่อสร้างอินเทอร์เฟซผู้ใช้ โดยมีสถานะการจัดการ Redux Toolkit ความสามารถหลักสี่ประการแมปโดยตรงกับองค์ประกอบสี่แท็บ: ทักษะในท้องถิ่น แกลเลอรีทักษะ คำแนะนำที่ชาญฉลาด และผู้ให้บริการที่เชื่อถือได้ ท้ายที่สุดแล้ว การออกแบบส่วนใหญ่เกี่ยวกับการทำให้ประสบการณ์ผู้ใช้ดีขึ้น

**แบ็กเอนด์** ขึ้นอยู่กับ ASP.NET Core + ABP Framework โดยใช้ Orleans Grain สำหรับการจัดการสถานะแบบกระจาย ไคลเอนต์ API ออนไลน์ล้อมรอบไฟล์ `IOnlineApiClient` อินเทอร์เฟซเพื่อสื่อสารกับบริการแค็ตตาล็อกทักษะระยะไกล

หลักการทางสถาปัตยกรรมโดยรวมคือการแยกการดำเนินการคำสั่งออกจากตรรกะทางธุรกิจ ด้วยรูปแบบอะแด็ปเตอร์ รายละเอียดการใช้งานของการดำเนินการคำสั่ง npm/npx จะถูกซ่อนอยู่ภายในโมดูลอิสระ ท้ายที่สุดแล้ว ไม่มีใครอยากให้การเรียกบรรทัดคำสั่งกระจัดกระจายไปทั่วโค้ดเบส

## ความสามารถหลัก 1: การจัดการระดับโลกในระดับท้องถิ่น

การจัดการส่วนกลางในระดับท้องถิ่นเป็นโมดูลพื้นฐานที่สุด มีหน้าที่รับผิดชอบในการแสดงรายการทักษะที่ติดตั้งและสนับสนุนการดำเนินการถอนการติดตั้ง ไม่มีอะไรซับซ้อนเกินไปที่นี่ ส่วนใหญ่จะเกี่ยวกับการทำพื้นฐานให้ดี

### แนวทางทางเทคนิค

การนำไปปฏิบัติอาศัยอยู่ใน `LocalSkillsTab.tsx` และ `LocalSkillCommandAdapter.cs`. แนวคิดหลักคือการห่อ `npx skills` แยกวิเคราะห์เอาต์พุต JSON และแปลงเป็นโครงสร้างข้อมูลภายใน ฟังดูง่ายและในทางปฏิบัติก็เป็นเช่นนั้นเป็นส่วนใหญ่

```csharp
public async Task<IReadOnlyList<LocalSkillInventoryResponseDto>> GetLocalSkillsAsync(
    CancellationToken cancellationToken = default)
{
    var result = await _commandAdapter.ListGlobalSkillsAsync(cancellationToken);
    return result.Skills.Select(skill => new LocalSkillInventoryResponseDto
    {
        Name = skill.Name,
        Version = skill.Version,
        Source = skill.Source,
        InstalledPath = skill.InstalledPath,
        Description = skill.Description
    }).ToList();
}
```

กระแสข้อมูลมีความชัดเจนมาก: ส่วนหน้าส่งคำขอ -> `SkillGalleryAppService` รับมัน -> `LocalSkillCommandAdapter` ดำเนินการคำสั่ง npx -> ผลลัพธ์ JSON จะถูกแยกวิเคราะห์ -> DTO จะถูกส่งกลับ แต่ละขั้นตอนเป็นไปตามธรรมชาติจากขั้นตอนก่อนหน้า

การถอนการติดตั้งทักษะใช้ `npx skills remove -g <skillName> -y` และระบบจะจัดการการขึ้นต่อกันและการล้างข้อมูลโดยอัตโนมัติ ข้อมูลเมตาการติดตั้งจะถูกเก็บไว้ใน `managed-install.json` ภายในไดเร็กทอรีทักษะ ข้อมูลการบันทึก เช่น เวลาติดตั้งและเวอร์ชันต้นทางสำหรับการอัปเดตและการตรวจสอบในภายหลัง บางสิ่งก็คุ้มค่าที่จะบันทึก

### ขั้นตอนการติดตั้งโดยละเอียด

การติดตั้งทักษะต้องมีขั้นตอนการประสานงานหลายขั้นตอน ความจริงแล้ว มันไม่ซับซ้อนเป็นพิเศษ:

```csharp
public async Task<SkillInstallResultDto> InstallAsync(
    SkillInstallRequestDto request,
    CancellationToken cancellationToken = default)
{
    // 1. Normalize the installation reference
    var normalized = _referenceNormalizer.Normalize(
        request.SkillId,
        request.Source,
        request.SkillSlug,
        request.Version);

    // 2. Check prerequisites
    await _prerequisiteChecker.CheckAsync(cancellationToken);

    // 3. Acquire installation lock
    using var installLock = await _lockProvider.AcquireAsync(normalized.SkillId);

    // 4. Execute installation command
    var result = await _installCommandRunner.ExecuteAsync(
        new SkillInstallCommandExecutionRequest
        {
            Command = $"npx skills add {normalized.FullReference} -g -y",
            Timeout = TimeSpan.FromMinutes(4)
        },
        cancellationToken);

    // 5. Persist installation metadata
    await _metadataStore.WriteAsync(normalized.SkillPath, request);

    return new SkillInstallResultDto { Success = result.Success };
}
```

ที่นี่ใช้รูปแบบการออกแบบที่สำคัญหลายรูปแบบ: **ตัวปรับมาตรฐานอ้างอิง** จะแปลงรูปแบบอินพุตต่างๆ เช่น `tanweai/pua` และ `@opencode/docker-skill`สู่การเป็นตัวแทนภายในแบบครบวงจร **กลไกการล็อคการติดตั้ง** ช่วยให้มั่นใจว่าการดำเนินการติดตั้งเพียงครั้งเดียวสามารถทำงานได้สำหรับทักษะเดียวกันในแต่ละครั้ง และ **เอาต์พุตสตรีมมิ่ง** ผลักดันความคืบหน้าในการติดตั้งไปยังฟรอนต์เอนด์แบบเรียลไทม์ผ่านเหตุการณ์ที่เซิร์ฟเวอร์ส่ง ดังนั้นผู้ใช้สามารถดูบันทึกที่เหมือนกับเทอร์มินัลได้ทันที

ท้ายที่สุดแล้ว รูปแบบทั้งหมดเหล่านี้มีไว้เพื่อจุดประสงค์เดียว นั่นคือเพื่อให้ระบบใช้งานและบำรุงรักษาได้ง่ายขึ้น

## ความสามารถหลัก 2: การค้นหาตลาด

การค้นหา Marketplace ช่วยให้ผู้ใช้ค้นพบและติดตั้งทักษะจากชุมชน ความสามารถของคนคนหนึ่งนั้นถูกจำกัดอยู่เสมอ ความรู้โดยรวมไปไกลกว่านั้นมาก

### แนวทางทางเทคนิค

คุณลักษณะการค้นหาอาศัย API ออนไลน์ `https://api.hagicode.com/v1/skills/search`. เพื่อปรับปรุงความเร็วในการตอบสนอง ระบบจะใช้แคช แคชก็เหมือนกับหน่วยความจำ หากคุณเก็บสิ่งที่มีประโยชน์ไว้ คุณไม่จำเป็นต้องคิดหนักในครั้งต่อไป

```csharp
private async Task<IReadOnlyList<SkillGallerySkillDto>> SearchCatalogAsync(
    string query,
    CancellationToken cancellationToken,
    IReadOnlySet<string>? allowedSources = null)
{
    var cacheKey = $"skill_search:{query}:{string.Join(",", allowedSources ?? Array.Empty<string>())}";

    if (_memoryCache.TryGetValue(cacheKey, out var cached))
        return (IReadOnlyList<SkillGallerySkillDto>)cached!;

    var response = await _onlineApiClient.SearchAsync(
        new SearchSkillsRequest
        {
            Query = query,
            Limit = _options.LimitPerQuery,
        },
        cancellationToken);

    var results = response.Skills
        .Where(skill => allowedSources is null || allowedSources.Contains(skill.Source))
        .Select(skill => new SkillGallerySkillDto { ... })
        .ToList();

    _memoryCache.Set(cacheKey, results, TimeSpan.FromMinutes(10));
    return results;
}
```

ผลการค้นหารองรับการกรองตามแหล่งที่เชื่อถือได้ ดังนั้นผู้ใช้จึงเห็นเฉพาะแหล่งทักษะที่พวกเขาเชื่อถือเท่านั้น การค้นหาเมล็ดพันธุ์เช่น `popular` และ `recent` ใช้เพื่อเริ่มต้นแค็ตตาล็อกทำให้ผู้ใช้สามารถดูทักษะยอดนิยมที่แนะนำได้ในครั้งแรกที่เปิด การแสดงครั้งแรกยังคงมีความสำคัญ

## ความสามารถหลัก 3: คำแนะนำอันชาญฉลาด

คำแนะนำอัจฉริยะเป็นส่วนที่ซับซ้อนที่สุดของระบบ พวกเขาสามารถแนะนำทักษะที่เหมาะสมที่สุดโดยอัตโนมัติตามบริบทของโครงการปัจจุบัน ถึงแม้จะซับซ้อน แต่ก็ยังคุ้มค่าที่จะสร้าง

### ขั้นตอนการแนะนำ

ขั้นตอนการแนะนำทั้งหมดแบ่งออกเป็นห้าขั้นตอน:

```
1. Build project context
   ↓
2. AI generates search queries
   ↓
3. Search the online catalog in parallel
   ↓
4. AI ranks the candidates
   ↓
5. Return the recommendation list
```

ขั้นแรก ระบบจะวิเคราะห์คุณลักษณะต่างๆ เช่น กลุ่มเทคโนโลยีของโครงการ ภาษาการเขียนโปรแกรม และโครงสร้างโดเมนเพื่อสร้าง "โปรไฟล์โครงการ" โปรไฟล์นั้นเหมือนกับเรซูเม่เล็กน้อย โดยบันทึกลักษณะสำคัญของโปรเจ็กต์

จากนั้น AI Grain จะถูกนำมาใช้เพื่อสร้างคำค้นหาที่ตรงเป้าหมาย การออกแบบนี้ค่อนข้างน่าสนใจจริงๆ แทนที่จะถาม AI โดยตรงว่า "ฉันควรแนะนำทักษะใดบ้าง" เราขอให้ AI คิดก่อนว่า "คำค้นหาใดที่มีแนวโน้มที่จะพบทักษะที่เกี่ยวข้อง" บางครั้งวิธีที่คุณถามคำถามนั้นสำคัญกว่าคำตอบ:

```csharp
var queryGeneration = await aiGrain.GenerateSkillRecommendationQueriesAsync(
    projectContext,      // Project context
    locale,              // User language preference
    maxQueries,           // Maximum number of queries
    effectiveSearchHero); // AI model selection
```

จากนั้น คำค้นหาเหล่านั้นจะดำเนินการควบคู่กันไปเพื่อรวบรวมรายการทักษะของผู้สมัคร ท้ายที่สุดแล้ว การประมวลผลแบบขนานเป็นเพียงวิธีหนึ่งในการประหยัดเวลา

ในที่สุด AI Grain อีกอันจะจัดอันดับทักษะของผู้สมัคร ขั้นตอนนี้จะพิจารณาปัจจัยต่างๆ เช่น ความเกี่ยวข้องของทักษะกับโครงการ สถานะความน่าเชื่อถือ และการตั้งค่าประวัติของผู้ใช้:

```csharp
var ranking = await aiGrain.RankSkillRecommendationsAsync(
    projectContext,
    candidates,
    installedSkillNames,
    locale,
    maxRecommendations,
    effectiveRankingHero);

response.Items = MergeRecommendations(projectContext, candidates, ranking, maxRecommendations);
```

### กลไกทางเลือก

โมเดล AI สามารถตอบสนองได้ช้าหรือใช้งานไม่ได้ชั่วคราว แม้แต่ระบบที่ดีที่สุดก็ยังสะดุดในบางครั้ง ด้วยเหตุผลดังกล่าว ระบบจึงมีกลไกทางเลือกตามที่กำหนด: เมื่อบริการ AI ไม่พร้อมใช้งาน ระบบจะใช้อัลกอริธึมการเรียนรู้ตามกฎเพื่อสร้างคำแนะนำ เช่น การอนุมานทักษะที่จำเป็นที่อาจเกิดขึ้นจากการพึ่งพาใน `package.json`.

พูดตรงๆ กลไกทางเลือกนี้เป็นเพียงแผนสำรองสำหรับระบบ

## ความสามารถหลัก 4: การจัดการผู้ให้บริการที่เชื่อถือได้

การจัดการผู้ให้บริการที่เชื่อถือได้ช่วยให้ผู้ใช้สามารถควบคุมแหล่งที่มาของทักษะที่ถือว่าเชื่อถือได้ ความน่าเชื่อถือยังคงเป็นสิ่งที่ผู้ใช้ควรกำหนดได้ด้วยตนเอง

### กฎการจับคู่

ผู้ให้บริการที่เชื่อถือได้รองรับกฎการจับคู่สองกฎ: **ตรงทั้งหมด** (`exact`) และ **คำนำหน้าตรงกัน** (`prefix`).

```csharp
public static TrustedSkillProviderResolutionSnapshot Resolve(
    TrustedSkillProviderSnapshot snapshot,
    string source)
{
    var normalizedSource = Normalize(source);

    foreach (var entry in snapshot.Entries.OrderBy(e => e.SortOrder))
    {
        if (!entry.IsEnabled) continue;

        foreach (var rule in entry.MatchRules)
        {
            bool isMatch = rule.MatchType switch
            {
                TrustedSkillProviderMatchRuleType.Exact
                    => string.Equals(normalizedSource, Normalize(rule.Value),
                        StringComparison.OrdinalIgnoreCase),
                TrustedSkillProviderMatchRuleType.Prefix
                    => normalizedSource.StartsWith(Normalize(rule.Value) + "/",
                        StringComparison.OrdinalIgnoreCase),
                _ => false
            };

            if (isMatch)
                return new TrustedSkillProviderResolutionSnapshot
                {
                    IsTrustedSource = true,
                    ProviderId = entry.ProviderId,
                    DisplayName = entry.DisplayName
                };
        }
    }

    return new TrustedSkillProviderResolutionSnapshot { IsTrustedSource = false };
}
```

ผู้ให้บริการที่เชื่อถือได้ในตัว ได้แก่ องค์กรและโครงการที่มีชื่อเสียง เช่น Vercel, Azure, anthropics, Microsoft และการใช้เบราว์เซอร์ คุณสามารถเพิ่มผู้ให้บริการที่กำหนดเองผ่านไฟล์การกำหนดค่าได้โดยการระบุ ID ผู้ให้บริการ ชื่อที่แสดง ป้ายกำกับป้าย กฎการจับคู่ และอื่นๆ โลกนี้กว้างใหญ่พอที่จะไว้วางใจในตัวเพียงไม่กี่ตัวเท่านั้นที่จะไม่เพียงพอ

### การบังคับใช้ความคงอยู่

การกำหนดค่าที่เชื่อถือได้ยังคงอยู่โดยใช้ Orleans Grain:

```csharp
public class TrustedSkillProviderGrain : Grain<TrustedSkillProviderState>,
    ITrustedSkillProviderGrain
{
    public async Task UpdateConfigurationAsync(TrustedSkillProviderSnapshot snapshot)
    {
        State.Snapshot = snapshot;
        await WriteStateAsync();
    }

    public Task<TrustedSkillProviderSnapshot> GetConfigurationAsync()
    {
        return Task.FromResult(State.Snapshot);
    }
}
```

ข้อดีของแนวทางนี้คือการเปลี่ยนแปลงการกำหนดค่าจะซิงโครไนซ์กับโหนดทั้งหมดโดยอัตโนมัติ โดยไม่จำเป็นต้องรีเฟรชแคชด้วยตนเอง ในที่สุดระบบอัตโนมัติคือการทำให้ผู้คนกังวลน้อยลง

## การออกแบบทางเทคนิคที่สำคัญ

### รูปแบบอแด็ปเตอร์การดำเนินการคำสั่ง

ระบบทักษะจำเป็นต้องดำเนินการคำสั่ง npx ต่างๆ หากตรรกะนั้นกระจัดกระจายไปทุกที่ โค้ดก็จะรักษาได้ยากอย่างรวดเร็ว นั่นคือเหตุผลที่เราออกแบบอินเทอร์เฟซของอะแดปเตอร์ ท้ายที่สุดแล้วรูปแบบการออกแบบมีอยู่เพื่อทำให้โค้ดง่ายต่อการบำรุงรักษา:

```csharp
public interface ISkillInstallCommandRunner
{
    Task<SkillInstallCommandExecutionResult> ExecuteAsync(
        SkillInstallCommandExecutionRequest request,
        CancellationToken cancellationToken = default);
}
```

คำสั่งที่แตกต่างกันมีการใช้งานตัวดำเนินการที่แตกต่างกัน แต่คำสั่งทั้งหมดใช้อินเทอร์เฟซเดียวกัน ทำให้การทดสอบและการเปลี่ยนตรงไปตรงมา

### เอาต์พุตสตรีมมิ่ง SSE

ความคืบหน้าในการติดตั้งจะถูกส่งไปยังส่วนหน้าแบบเรียลไทม์ผ่านเหตุการณ์ที่เซิร์ฟเวอร์ส่ง:

```csharp
public async Task InstallWithProgressAsync(
    SkillInstallRequestDto request,
    IServerStreamWriter<SkillInstallProgressEventDto> stream,
    CancellationToken cancellationToken)
{
    var process = new Process
    {
        StartInfo = new ProcessStartInfo
        {
            FileName = "npx",
            Arguments = $"skills add {request.FullReference} -g -y",
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false
        }
    };

    process.OutputDataReceived += async (sender, e) =>
    {
        await stream.WriteAsync(new SkillInstallProgressEventDto
        {
            EventType = "output",
            Data = e.Data ?? string.Empty
        });
    };

    process.Start();
    process.BeginOutputReadLine();
    await process.WaitForExitAsync(cancellationToken);
}
```

ที่ส่วนหน้า ผู้ใช้สามารถเห็นเอาต์พุตที่คล้ายกับเทอร์มินัลแบบเรียลไทม์ ซึ่งทำให้ประสบการณ์นี้ใช้งานง่ายมาก ข้อเสนอแนะแบบเรียลไทม์ช่วยให้ผู้คนรู้สึกสบายใจ

## คู่มือการปฏิบัติ

### การติดตั้งทักษะชุมชน

ทำการติดตั้ง `pua` ทักษะเป็นตัวอย่าง (เป็นทักษะชุมชนยอดนิยม):

1. เปิดลิ้นชักทักษะแล้วสลับไปที่ `Skill Gallery` แท็บ
2. เข้า `pua` ในช่องค้นหา
3. คลิกผลการค้นหาเพื่อดูรายละเอียดทักษะ
4. คลิกที่ `Install` ปุ่ม
5. เปลี่ยนไปที่ `Local Skills` แท็บเพื่อยืนยันว่าการติดตั้งสำเร็จ

คำสั่งการติดตั้งคือ `npx skills add tanweai/pua -g -y`และระบบจะจัดการรายละเอียดทั้งหมดโดยอัตโนมัติ จริงๆ แล้วมีขั้นตอนไม่มากนักเมื่อคุณทำทีละขั้นตอน

### การเพิ่มแหล่งที่เชื่อถือได้ที่กำหนดเอง

หากทีมของคุณมีคลังทักษะของตนเอง คุณสามารถเพิ่มเป็นแหล่งที่เชื่อถือได้:

```yaml
providerId: "my-team"
displayName: "My Team Skills"
badgeLabel: "MyTeam"
isEnabled: true
sortOrder: 100
matchRules:
  - matchType: "prefix"
    value: "my-team/"
  - matchType: "exact"
    value: "my-team/special-skill"
```

ด้วยวิธีนี้ ทักษะทั้งหมดจากทีมของคุณจะแสดงป้ายสถานะที่เชื่อถือได้ ทำให้ผู้ใช้ติดตั้งได้สะดวกยิ่งขึ้น ป้ายกำกับและสัญญาณช่วยให้ผู้คนรู้สึกมั่นใจมากขึ้น

### พื้นฐานการพัฒนาทักษะ

การสร้างทักษะแบบกำหนดเองต้องมีโครงสร้างต่อไปนี้:

```
my-skill/
├── SKILL.md          # Skill metadata (YAML front matter)
├── index.ts          # Skill entry point
├── agents/           # Supported agent configuration
└── references/       # Reference resources
```

ตัวอย่าง `SKILL.md` รูปแบบ:

```yaml
---
name: my-skill
description: A brief description of what this skill does
---

# My Skill

Detailed documentation...
```

## หมายเหตุ

1. **ข้อกำหนดเครือข่าย**: การค้นหาทักษะและการติดตั้งจำเป็นต้องเข้าถึง `api.hagicode.com` และรีจิสทรี npm
2. **เวอร์ชัน Node.js**: แนะนำให้ใช้ Node.js 18 ขึ้นไป
3. **ข้อกำหนดการอนุญาต**: ต้องมีสิทธิ์ในการติดตั้ง npm ทั่วโลก
4. **การควบคุมการทำงานพร้อมกัน**: การดำเนินการติดตั้งหรือถอนการติดตั้งเพียงครั้งเดียวเท่านั้นที่สามารถทำงานได้สำหรับทักษะเดียวกันในแต่ละครั้ง
5. **การตั้งค่าการหมดเวลา**: การหมดเวลาเริ่มต้นสำหรับการติดตั้งคือ 4 นาที แต่สถานการณ์ที่ซับซ้อนอาจต้องมีการปรับเปลี่ยน

บันทึกเหล่านี้มีอยู่เพื่อช่วยให้ทุกอย่างดำเนินไปได้อย่างราบรื่น
