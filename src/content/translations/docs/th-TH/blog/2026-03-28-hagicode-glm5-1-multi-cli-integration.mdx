## คู่มือการรวม Hagicode และ GLM-5.1 Multi-CLI

:::note[ประกาศบริบทเวลา]
บทความนี้บันทึกการออกแบบการรวมหลาย CLI และการแม็ปแบบจำลองตามที่มีอยู่ ณ เวลาที่เผยแพร่ สำหรับขอบเขตที่รองรับเสมอในปัจจุบัน โปรดดูที่ [การติดตั้ง AI Agent CLI](/th-TH/related-software-installation/ai-agent-cli/) และ [คู่มือการติดตั้ง](/th-TH/installation); ที่ `IFlowCli` ที่กล่าวถึงในที่นี้แสดงถึงการกำหนดค่าในอดีตในขณะนั้นเท่านั้น และไม่ได้เป็นส่วนหนึ่งของขอบเขตการสนับสนุนที่ใช้งานอยู่ในปัจจุบัน
:::

## พื้นหลัง

ในโปรเจ็กต์ Hagicode ผู้ใช้สามารถเลือกจากเครื่องมือ CLI ที่หลากหลายเพื่อขับเคลื่อนผู้ช่วยการเขียนโปรแกรม AI รวมถึง Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI และอื่นๆ อีกมากมาย เครื่องมือ CLI เหล่านี้เป็นเครื่องมือการเขียนโปรแกรม AI สำหรับการใช้งานทั่วไป แต่ด้วยเลเยอร์นามธรรมของ Hagicode จึงสามารถเชื่อมต่อกับผู้ให้บริการโมเดล AI ต่างๆ ได้อย่างยืดหยุ่น

Zhipu AI (ZAI) มอบอินเทอร์เฟซที่เข้ากันได้กับ Anthropic Claude API ทำให้เครื่องมือ CLI เหล่านี้สามารถใช้โมเดลซีรีส์ GLM ในประเทศได้โดยตรง หนึ่งในนั้นคือ GLM-5.1 ถือเป็นการเปิดตัวโมเดลภาษาขนาดใหญ่ล่าสุดของ Zhipu โดยมีการปรับปรุงที่เหนือกว่า GLM-5.0 อย่างเห็นได้ชัด

## สถาปัตยกรรมนามธรรม CLI ของ Hagicode

Hagicode กำหนดประเภทผู้ให้บริการ CLI 11 ประเภทผ่าน `AIProviderType` enum ครอบคลุมเครื่องมือ CLI การเขียนโปรแกรม AI กระแสหลัก:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

CLI แต่ละตัวมีการกำหนดค่าพารามิเตอร์โมเดลที่สอดคล้องกันและรองรับ `model` และ `reasoning` พารามิเตอร์:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],
        [AIProviderType.CodexCli] = ["model", "reasoning"],
        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],
        [AIProviderType.HermesCli] = ["model", "reasoning"],
        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini does not support the reasoning parameter
        // ...
    };
```

## ระบบรองรับโมเดล GLM

แค็ตตาล็อกวิชาชีพรองของ Hagicode ให้คำจำกัดความการสนับสนุนที่สมบูรณ์สำหรับซีรีส์โมเดล GLM:

| รหัสรุ่น | ชื่อ | การใช้เหตุผลเริ่มต้น | ตระกูล CLI ที่เข้ากันได้ |
|----------|------|-------------------|-------------------------|
| `glm-4.7` | จีแอลเอ็ม 4.7 | สูง | โคล้ด, โค้ดบัดดี้, เฮอร์มีส, โกเดอร์, คิโระ |
| `glm-5` | จีแอลเอ็ม 5 | สูง | โคล้ด, โค้ดบัดดี้, เฮอร์มีส, โกเดอร์, คิโระ |
| `glm-5-turbo` | จีแอลเอ็ม 5 เทอร์โบ | สูง | โคล้ด, โค้ดบัดดี้, เฮอร์มีส, โกเดอร์, คิโระ |
| `glm-5.0` | GLM 5.0 (รุ่นเก่า) | สูง | โคล้ด, โค้ดบัดดี้, เฮอร์มีส, โกเดอร์, คิโระ |
| `glm-5.1` | จีแอลเอ็ม 5.1 | สูง | โคล้ด, โค้ดบัดดี้, เฮอร์มีส, โกเดอร์, คิโระ |

## ความแตกต่างที่สำคัญระหว่าง GLM-5.1 และ GLM-5.0

จากการนำไปปฏิบัติใน `AcpSessionModelBootstrapper.cs`เราจะเห็นความแตกต่างระหว่าง GLM-5.1 และ GLM-5.0 ได้อย่างชัดเจน:

### การใช้งาน GLM-5.1 แบบสแตนด์อโลน

GLM-5.1 เป็นตัวระบุโมเดลใหม่แบบสแตนด์อโลนโดยไม่มีตรรกะการจัดการแบบเดิม:

```csharp
private const string Glm51ModelValue = "glm-5.1";
```

คำจำกัดความในแค็ตตาล็อกวิชาชีพมัธยมศึกษา:

```json
{
  "id": "secondary-glm-5-1",
  "name": "GLM 5.1",
  "family": "anthropic",
  "summary": "hero.professionCopy.secondary.glm51.summary",
  "sourceLabel": "hero.professionCopy.sources.aiSharedAnthropicModel",
  "sortOrder": 64,
  "supportsImage": true,
  "compatiblePrimaryFamilies": [
    "claude",
    "codebuddy",
    "hermes",
    "qoder",
    "kiro"
  ],
  "defaultParameters": {
    "model": "glm-5.1",
    "reasoning": "high"
  }
}
```

## การใช้ GLM-5.1 ใน CLI ที่แตกต่างกัน

### 1. รหัสคล็อด CLI + GLM-5.1

Claude Code CLI เป็นหนึ่งใน CLI หลักของ Hagicode และได้รับการกำหนดค่าผ่านระบบการกำหนดค่า Hero:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

ที่สอดคล้องกัน `HeroEquipmentCatalogItem` การกำหนดค่า:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. โอเพ่นโค้ด CLI + GLM-5.1

OpenCode CLI เป็น CLI ที่ยืดหยุ่นที่สุด และรองรับการระบุรุ่นใดๆ ใน `provider/model` รูปแบบ:

**วิธีที่ 1: ใช้คำนำหน้าผู้ให้บริการ ZAI**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**วิธีที่ 2: ใช้รหัสรุ่นโดยตรง**
```json
{
  "model": "glm-5.1"
}
```

**วิธีที่ 3: UI การกำหนดค่าส่วนหน้า**

ใน `HeroModelEquipmentForm.tsx`, OpenCode CLI มีคำใบ้เฉพาะ:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

ผู้ใช้สามารถป้อน:
```
zai/glm-5.1
glm-5.1
```

**ตรรกะการแยกวิเคราะห์โมเดล OpenCode CLI**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. รหัสบัดดี้ CLI + GLM-5.1

Codebuddy CLI มีตรรกะการจัดการแบบเดิมโดยเฉพาะ:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**หมายเหตุ**: Codebuddy ยังคงการจัดการพิเศษสำหรับ GLM-5.0 และไม่ใช้การทำให้เป็นมาตรฐานแบบเดิม:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## ข้อดีการปรับปรุงของ GLM-5.1

เมื่อเปรียบเทียบกับ GLM-5.0 แล้ว GLM-5.1 มีการปรับปรุงที่สำคัญดังต่อไปนี้:

### 1. ความสามารถในการให้เหตุผลที่ดีขึ้น

ตามข้อมูลการเปิดตัวอย่างเป็นทางการของ Zhipu การปรับปรุง GLM-5.1 รวมถึง:

- **ความเข้าใจโค้ดที่แข็งแกร่งยิ่งขึ้น**: การวิเคราะห์โครงสร้างโค้ดที่ซับซ้อนได้แม่นยำยิ่งขึ้น
- **การทำความเข้าใจบริบทที่ยาวขึ้น**: รองรับบริบทการสนทนาที่ยาวขึ้น
- **การเรียกเครื่องมือที่ได้รับการปรับปรุง**: อัตราความสำเร็จที่สูงขึ้นสำหรับการเรียกเครื่องมือ MCP
- **ความเสถียรของเอาต์พุต**: ลดการสุ่มและภาพหลอน

### 2. ความเข้ากันได้หลาย CLI ที่ครอบคลุม

GLM-5.1 ครอบคลุม CLI หลักทั้งหมดที่รองรับโดย Hagicode:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## หมายเหตุ

### 1. การกำหนดค่าคีย์ API

ตรวจสอบให้แน่ใจว่า `ANTHROPIC_AUTH_TOKEN` ตัวแปรสภาพแวดล้อมได้รับการตั้งค่าอย่างถูกต้อง เป็นข้อมูลประจำตัวที่จำเป็นสำหรับ CLI ทุกตัวในการเชื่อมต่อกับโมเดล

### 2. ความพร้อมของรุ่น

GLM-5.1 จำเป็นต้องเปิดใช้งานโดยผู้ให้บริการรุ่นที่เกี่ยวข้อง:
- แพลตฟอร์ม Zhipu AI ZAI รองรับตามค่าเริ่มต้น
- Alibaba Cloud DashScope อาจต้องใช้แอปพลิเคชันแยกต่างหาก

### 3. รูปแบบ OpenCode CLI

เมื่อใช้ `provider/model` ตรวจสอบให้แน่ใจว่า ID ผู้ให้บริการถูกต้อง:
- ซิปู เอไอ: `zai` หรือ `zhipuai`
- อาลีบาบาคลาวด์: `aliyun` หรือ `dashscope`

### 4. พารามิเตอร์การใช้เหตุผล

- `high` แนะนำให้ใช้เพื่อผลลัพธ์การสร้างโค้ดที่ดีที่สุด
- Gemini CLI ไม่รองรับพารามิเตอร์การให้เหตุผล และจะเพิกเฉยต่อการกำหนดค่านี้โดยอัตโนมัติ

## บทสรุป

Hagicode ช่วยให้สามารถผสานรวม GLM-5.1 และ CLI หลายรายการได้อย่างยืดหยุ่นผ่านเลเยอร์นามธรรมที่เป็นหนึ่งเดียว นักพัฒนาสามารถเลือกเครื่องมือ CLI ที่เหมาะกับความต้องการและสถานการณ์การใช้งานของตนมากที่สุด จากนั้นจึงใช้โมเดล GLM-5.1 ล่าสุดผ่านการกำหนดค่าที่เรียบง่าย

เนื่องจากเป็นเวอร์ชันล่าสุดของ Zhipu GLM-5.1 จึงมีการปรับปรุงที่เหนือกว่า GLM-5.0 อย่างชัดเจน:
- ตัวระบุเวอร์ชันอิสระที่ไม่มีภาระแบบเดิม
- การใช้เหตุผลและความเข้าใจโค้ดที่แข็งแกร่งยิ่งขึ้น
- ความเข้ากันได้หลาย CLI ในวงกว้าง
- การกำหนดค่าระดับการให้เหตุผลที่ยืดหยุ่น

ด้วยตัวแปรสภาพแวดล้อมที่ถูกต้องและการกำหนดค่าอุปกรณ์ Hero ผู้ใช้สามารถปลดล็อกพลังของ GLM-5.1 ได้อย่างเต็มที่ในสภาพแวดล้อม CLI ที่แตกต่างกัน
