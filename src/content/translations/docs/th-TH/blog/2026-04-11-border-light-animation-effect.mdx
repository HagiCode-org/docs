## คำแนะนำในการสร้างเอฟเฟกต์ภาพเคลื่อนไหวแบบ Border Light Sweep

> คุณจะสร้างองค์ประกอบสำคัญที่ผู้ใช้สังเกตเห็นได้อย่างรวดเร็วโดยใช้ CSS บริสุทธิ์ได้อย่างไร จริงๆแล้วมันไม่ยากขนาดนั้น เคล็ดลับคือใช้เส้นทางวงเวียนเล็กน้อย ในบทความนี้ ผมจะอธิบายวิธีการสร้างแอนิเมชั่นกวาดแสงขอบตั้งแต่เริ่มต้น และแบ่งปันข้อผิดพลาดบางประการที่เราพบขณะสร้าง HagiCode

## พื้นหลัง

หากคุณทำงานที่ส่วนหน้า คุณอาจเคยมีประสบการณ์นี้มาก่อน: ผู้จัดการผลิตภัณฑ์เดินผ่านไปพร้อมกับนิพจน์ "นี่เป็นเรื่องง่ายอย่างแน่นอน" และพูดว่า "เราสามารถเพิ่มเอฟเฟกต์พิเศษบางประเภทให้กับงานที่กำลังทำงานอยู่นี้เพื่อให้ผู้ใช้สามารถมองเห็นได้ทันทีหรือไม่"

คุณบอกว่าแน่นอน เราสามารถเปลี่ยนสีเส้นขอบได้ จากนั้นพวกเขาก็ส่ายหัวด้วยท่าทางที่บอกว่า "คุณไม่เข้าใจ" พวกเขาตอบว่า "นั่นไม่ชัดเจนเพียงพอ ฉันต้องการเอฟเฟกต์ที่มีแสงวิ่งไปรอบๆ ชายแดน เหมือนในภาพยนตร์ไซไฟ"

เมื่อถึงจุดนั้นคุณอาจเริ่มสงสัยว่าจะสร้างมันขึ้นมาได้อย่างไร ผ้าใบ? เอสวีจี? หรือ CSS สามารถจัดการได้ด้วยตัวเอง? ท้ายที่สุดแล้วไม่มีใครอยากยอมรับว่าพวกเขาไม่รู้วิธี

ในเว็บแอปพลิเคชันสมัยใหม่ จริงๆ แล้วภาพเคลื่อนไหวแบบกวาดแสงขอบเป็นเรื่องปกติมาก ส่วนใหญ่จะใช้ในสถานการณ์บางอย่างเช่นนี้:

- **ตัวบ่งชี้สถานะ**: การทำเครื่องหมายงานที่กำลังดำเนินการหรือรายการที่ใช้งานอยู่
- **การเน้นภาพ**: เน้นส่วนเนื้อหาที่สำคัญ
- **การเพิ่มประสิทธิภาพแบรนด์**: การสร้างสไตล์ภาพที่โฉบเฉี่ยว ทันสมัย และล้ำหน้าเทคโนโลยี
- **ธีมตามฤดูกาล**: สร้างบรรยากาศการเฉลิมฉลองในโอกาสพิเศษ

เราพบข้อกำหนดนี้ทุกประการในขณะที่สร้าง [ฮากิโค้ด](https://hagicode.com). ผู้ใช้จำเป็นต้องดูอย่างรวดเร็วว่าเซสชันใดกำลังทำงานอยู่และข้อเสนอใดกำลังดำเนินการอยู่ เราลองใช้วิธีการต่างๆ หลายวิธี บางเส้นทางก็เรียบกว่า บางเส้นทางคดเคี้ยวกว่าเล็กน้อย ในท้ายที่สุด เราก็ตัดสินใจเลือกกลยุทธ์การดำเนินงานที่ค่อนข้างสมบูรณ์

## แนวคิดหลักในการนำไปปฏิบัติ

หลังจากวิเคราะห์ฐานโค้ด HagiCode แล้ว เราได้สรุปรูปแบบการใช้งานหลักหลายรูปแบบด้านล่าง แต่ละสถานการณ์เหมาะกับสถานการณ์ที่แตกต่างกัน หรืออีกนัยหนึ่ง แต่ละสถานการณ์มีเหตุผล

### 1. การเรืองแสงแบบหมุนที่มีการไล่ระดับสีแบบกรวย (พบมากที่สุด)

นี่เป็นวิธีคลาสสิกในการใช้เอฟเฟกต์การกวาดแสงเส้นขอบ แนวคิดหลักคือการใช้ CSS `conic-gradient` เพื่อสร้างการไล่ระดับสีแบบกรวย จากนั้นหมุนอย่างต่อเนื่อง เหมือนไฟถนนที่หมุนในตอนกลางคืน มันก็วนเวียนอยู่ตลอดเวลา

**องค์ประกอบสำคัญ:**
- ใช้ `::before` องค์ประกอบหลอกเพื่อสร้างเลเยอร์เรืองแสง
- ใช้ `conic-gradient` เพื่อกำหนดการกระจายสีแบบไล่ระดับสี
- ใช้ `::after` องค์ประกอบหลอกเพื่อปกปิดบริเวณกึ่งกลาง (ไม่บังคับ)
- ใช้ `@keyframes` เพื่อใช้ภาพเคลื่อนไหวการหมุน

### 2.เส้นเรืองแสงด้านข้าง

ซึ่งใช้ได้ดีกับตัวบ่งชี้สถานะในรายการ คุณสร้างเส้นเรืองแสงบางๆ ที่ด้านหนึ่งขององค์ประกอบ แทนที่จะทำให้เส้นขอบทั้งหมดเคลื่อนไหว บางครั้งแสงสว่างเพียงเล็กน้อยก็เพียงพอแล้ว คุณไม่จำเป็นต้องส่องสว่างไปทั่วโลก

**องค์ประกอบสำคัญ:**
- องค์ประกอบเส้นบางที่มีตำแหน่งที่แน่นอน
- ใช้ `box-shadow` เพื่อสร้างเอฟเฟกต์เรืองแสง
- ใช้ `scale` และ `opacity` สำหรับแอนิเมชั่นการหายใจ

### 3. พื้นหลังเรืองแสงพร้อมกล่องเงา

หากคุณไม่ต้องการเอฟเฟ็กต์การกวาดเต็มรูปแบบและเพียงต้องการให้พื้นหลังดูนุ่มนวล ให้ทาหลายชั้น `box-shadow` คุณค่าก็เพียงพอแล้ว บางครั้งตัวเลือกที่ง่ายกว่าก็คือตัวเลือกที่ดีกว่า

### 4. การสนับสนุนการเข้าถึง

ส่วนนี้มองข้ามได้ง่ายแต่สำคัญอย่างยิ่ง แอนิเมชั่นทุกรายการควรคำนึงถึง `prefers-reduced-motion` แบบสอบถามสื่อและเป็นทางเลือกแบบคงที่สำหรับผู้ใช้ที่ไม่ต้องการภาพเคลื่อนไหว ไม่ใช่ทุกคนที่ชอบการเคลื่อนไหวอย่างต่อเนื่อง และการคำนึงถึงความชอบนั้นเป็นสิ่งสำคัญ

## ตัวเลือกการดำเนินงาน

### ตัวเลือกที่ 1: การหมุนเส้นขอบไล่ระดับทรงกรวย (แนะนำ)

นี่คือการใช้งานเอฟเฟกต์แสงขอบแบบกวาดที่สมบูรณ์แบบที่สุด และยังเป็นตัวเลือกที่เราใช้บ่อยที่สุดใน HagiCode ท้ายที่สุดแล้ว หากบางสิ่งทำงานได้ดี เหตุใดจึงต้องเปลี่ยนมัน?

```css
/* Parent container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotating glow layer */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Mask layer (optional, for creating a hollow border effect) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

หลักการเบื้องหลังตัวเลือกนี้ค่อนข้างง่าย: สร้างองค์ประกอบเทียมที่มีขนาดใหญ่กว่าคอนเทนเนอร์หลัก วาดรูปไล่ระดับสีบนองค์ประกอบนั้น และหมุนอย่างต่อเนื่อง คอนเทนเนอร์หลักใช้ `overflow: hidden`จึงมองเห็นได้เฉพาะแสงที่ลอดผ่านขอบเท่านั้น มันเหมือนกับการมองไฟถนนผ่านหน้าต่างเล็กน้อย คุณจะเห็นเพียงชิ้นเล็ก ๆ ที่ผ่านไป

### ตัวเลือกที่ 2: ขอบไฟหมุนแบบง่าย

หากคุณไม่ต้องการเอฟเฟกต์เต็มรูปแบบ HagiCode ยังมีเวอร์ชันระดับยูทิลิตี้ที่เบากว่าด้วย บางครั้งวิธีการที่ง่ายกว่านั้นก็ดีกว่าจริงๆ

```css
/* Rotating light border utility class */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Accessibility support */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

สังเกต `will-change: transform` ที่นี่ โดยจะบอกเบราว์เซอร์ว่า "องค์ประกอบนี้จะมีการเปลี่ยนแปลงอยู่เรื่อยๆ" เพื่อให้เบราว์เซอร์สามารถเตรียมการเพิ่มประสิทธิภาพบางอย่างไว้ล่วงหน้าและทำให้ภาพเคลื่อนไหวราบรื่นขึ้น การเตรียมตัวล่วงหน้ามักจะดีกว่าการแย่งชิงกันในนาทีสุดท้าย

### ตัวเลือกที่ 3: เส้นเรืองแสงด้านข้าง

เหมาะอย่างยิ่งสำหรับตัวบ่งชี้สถานะรายการ และเป็นสิ่งที่รายการเซสชัน HagiCode ใช้อย่างแน่นอน เส้นบางๆ เส้นเดียวยังคงโดดเด่นท่ามกลางสินค้าหลายๆ ชิ้น นั่นให้ความรู้สึกเหมือนเป็นบทเรียนชีวิตในแบบของตัวเอง

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

นี้ใช้ `isolation: isolate` เพื่อสร้างบริบทการซ้อนใหม่จากนั้นจึงอาศัย `z-index` เพื่อควบคุมลำดับการแสดงของแต่ละเลเยอร์ `pointer-events: none` ก็เป็นสิ่งจำเป็นเช่นกัน มิฉะนั้นองค์ประกอบหลอกจะบล็อกการคลิกของผู้ใช้ บางสิ่งอาจดูดีแต่ก็ไม่ควรขวางทาง

### ตัวเลือกที่ 4: ห่อไว้ในองค์ประกอบ React

หากโปรเจ็กต์ของคุณใช้ React คุณสามารถรวมตรรกะนี้ไว้ในส่วนประกอบได้ โดยเฉพาะการจัดการการเข้าถึง เขียนครั้งเดียวใช้หลายครั้ง นั่นคือประเด็นทั้งหมด

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

โมดูล CSS ที่ตรงกัน:

```css
/* GlowBorder.module.css */

/* Animated version */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Static version (accessibility) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

ที่ `useReducedMotion` ขอจาก `framer-motion` ตรวจจับการตั้งค่าระบบของผู้ใช้โดยอัตโนมัติ หากผู้ใช้เปิดใช้งานการลดการเคลื่อนไหว ก็จะส่งคืน `true`และส่วนประกอบจะแสดงเวอร์ชันคงที่แทน การเคารพความชอบของผู้ใช้มีความสำคัญมากกว่าการบังคับให้เกิดเอฟเฟกต์ที่ฉูดฉาด

## บทเรียนภาคปฏิบัติที่ได้รับ

นี่คือบทเรียนบางส่วนที่เราได้เรียนรู้ขณะสร้าง HagiCode คุณสามารถเรียกพวกมันว่ารอยแผลเป็นจากการต่อสู้ก็ได้ หวังว่าพวกเขาจะช่วยคุณหลีกเลี่ยงการออกนอกเส้นทาง

### 1. ระบบตัวแปรธีม

ตัวแปร CSS ทำให้การรองรับหลายธีมสะดวกเป็นพิเศษ ไม่มีใครอยากแก้ไขกองโค้ดทุกครั้งที่ธีมเปลี่ยน

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Usage */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

ด้วยวิธีนี้ การเปลี่ยนธีมจะต้องเปลี่ยนคลาสบนเท่านั้น `html` องค์ประกอบและสีของภาพเคลื่อนไหวทั้งหมดจะอัปเดตโดยอัตโนมัติ หนึ่งโค้ดเบส สองสไตล์ นั่นคือสิ่งที่เราต้องการ

### 2. การเพิ่มประสิทธิภาพการทำงาน

**การใช้งาน `will-change` เพื่อบอกใบ้ให้เบราว์เซอร์เพิ่มประสิทธิภาพ:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

แจ้งเบราว์เซอร์ล่วงหน้าและมันจะช่วยคุณปรับให้เหมาะสม หลายๆ อย่างในชีวิตทำงานได้ดีขึ้นด้วยการเตรียมตัวเพียงเล็กน้อย

**หลีกเลี่ยงการใช้เงากล่องที่ซับซ้อนกับองค์ประกอบขนาดใหญ่:**

```css
/* Not ideal - using a blurred shadow on a large element */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Better - use a pseudo-element to limit the glowing area */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

เราทดสอบสิ่งนี้ใน HagiCode การเพิ่มเงาที่พร่ามัวลงในการ์ดขนาดใหญ่โดยตรงทำให้อัตราเฟรมการเลื่อนต่ำกว่า 30fps การเปลี่ยนมาใช้องค์ประกอบหลอกทำให้สิ่งต่างๆ กลับมาที่ 60fps ที่คงที่ ผู้ใช้สามารถรู้สึกถึงความแตกต่างนั้นได้อย่างแน่นอน

### 3. การเข้าถึง

คุณไม่ควรข้ามสิ่งนี้จริงๆ ผู้ใช้บางคนพบว่าแอนิเมชันทำให้เวียนหัวหรือเสียสมาธิ และการเคารพในความชอบของพวกเขาเป็นส่วนหนึ่งของการสร้างผลิตภัณฑ์ที่ดี สิ่งสวยงามไม่จำเป็นต้องบังคับทุกคน

**แบบสอบถามสื่อ CSS:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Provide a static fallback */
    opacity: 1;
  }
}
```

**ตรวจจับการตั้งค่าผู้ใช้ใน React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. การควบคุมระดับความเข้ม

ตัวบ่งชี้ปริมาณงานโทเค็นใน HagiCode จะแสดงสีเรืองแสงที่แตกต่างกันตามปริมาณงานแบบเรียลไทม์ และสิ่งนี้จะถูกนำไปใช้แบบไดนามิก รัฐที่ต่างกันควรแสดงออกแตกต่างกัน

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. สิ่งที่ต้องระวัง

ยังมีรายละเอียดเล็กๆ น้อยๆ ที่ควรค่าแก่การใส่ใจ เพราะเมื่อคุณค้นพบปัญหาเหล่านี้อย่างยากลำบาก มันก็สายเกินไปแล้ว

| สิ่งที่ต้องระวัง | คำอธิบาย |
|---------|------|
| **การจัดการดัชนี z** | ชั้นเรืองแสงควรใช้อย่างเหมาะสม `z-index` จึงไม่รบกวนการโต้ตอบของเนื้อหา |
| **เหตุการณ์ตัวชี้** | ควรตั้งค่าองค์ประกอบหลอกเรืองแสง `pointer-events: none` |
| **ขอบเขตล้น** | ความต้องการคอนเทนเนอร์หลัก `overflow: hidden`หรือคุณต้องปรับขนาดองค์ประกอบหลอก |
| **ผลกระทบด้านประสิทธิภาพ** | แอนิเมชั่นที่ซับซ้อนอาจส่งผลเสียต่อประสิทธิภาพการทำงานบนอุปกรณ์เคลื่อนที่ ดังนั้นควรทดสอบอย่างรอบคอบ |
| **โหมดมืด** | ตรวจสอบให้แน่ใจว่าสีเรืองแสงยังคงมองเห็นได้ชัดเจนบนพื้นหลังสีเข้ม |
| **การสลับธีม** | ใช้ตัวแปร CSS เพื่อให้สีของภาพเคลื่อนไหวอัปเดตอย่างถูกต้องเมื่อธีมเปลี่ยน |

### 6. เคล็ดลับการดีบัก

องค์ประกอบหลอกอาจค้นหาได้ยากเล็กน้อยในเครื่องมือสำหรับนักพัฒนาซอฟต์แวร์ ดังนั้นคุณจึงเพิ่มเส้นขอบชั่วคราวเพื่อตรวจสอบตำแหน่งได้

```css
/* Temporarily show pseudo-element boundaries for debugging */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

หลังจากที่คุณวางตำแหน่งเสร็จแล้ว อย่าลืมใส่ความคิดเห็นหรือลบบรรทัดนั้นออก มิฉะนั้นการผลิตอาจเกิดความอึดอัดใจได้อย่างรวดเร็ว บางสิ่งทิ้งไว้ในการพัฒนาจะดีกว่า

## สรุป

แอนิเมชั่นการกวาดแสงขอบนั้นไม่ได้ยากเป็นพิเศษหรือไม่สำคัญเลย โดยแก่นแท้ของสูตรก็คือ `conic-gradient` บวกกับการหมุนเวียน แต่ถ้าคุณต้องการประสิทธิภาพ การบำรุงรักษา และการเข้าถึงที่ดี ก็ยังมีรายละเอียดการใช้งานอีกมากมายที่ต้องจัดการอย่างระมัดระวัง

HagiCode พบกับข้อผิดพลาดเหล่านี้มากมาย และค่อยๆ กลั่นกรองชุดแนวทางปฏิบัติที่ดีที่สุด นั่นคือวิธีที่โครงการดำเนินไป: คุณทดลอง ทำผิดพลาด และปรับปรุงทีละขั้นตอน หากคุณกำลังสร้างสิ่งที่คล้ายกัน ฉันหวังว่าบทความนี้จะช่วยคุณหลีกเลี่ยงการออกนอกเส้นทางโดยไม่จำเป็น

บางสิ่งจะชัดเจนเมื่อคุณสร้างมันขึ้นมาเองเท่านั้น
