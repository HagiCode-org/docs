## รองรับ GLM-5.1 เต็มรูปแบบและบูรณาการ Gemini CLI: เส้นทางวิวัฒนาการหลายรุ่นของ HagiCode

> บทความนี้จะแนะนำการอัปเดตหลักสองรายการล่าสุดสำหรับแพลตฟอร์ม HagiCode: การสนับสนุนอย่างเต็มที่สำหรับโมเดล Zhipu AI GLM-5.1 และความสำเร็จในการบูรณาการ Gemini CLI ในฐานะ Agent CLI ที่สิบ การอัปเดตเหล่านี้ร่วมกันจะช่วยเสริมความแข็งแกร่งให้กับความสามารถหลายรุ่นของแพลตฟอร์มและระบบนิเวศ multi-CLI

:::note[ประกาศบริบทเวลา]
บทความนี้บันทึกความคืบหน้าของการขยาย multi-CLI ณ เวลาที่เผยแพร่ สำหรับขอบเขตที่รองรับเสมอในปัจจุบัน โปรดดูที่ [การติดตั้ง AI Agent CLI](/th-TH/related-software-installation/ai-agent-cli/) และ [คู่มือการติดตั้ง](/th-TH/installation); ที่ `IFlowCli` ที่กล่าวถึงในที่นี้แสดงถึงการกำหนดค่าในอดีตในขณะนั้นเท่านั้น และไม่ได้เป็นส่วนหนึ่งของขอบเขตการสนับสนุนที่ใช้งานอยู่ในปัจจุบัน
:::

## พื้นหลัง

เวลาผ่านไปเร็วจริงๆ การพัฒนาแบบจำลองภาษาขนาดใหญ่เพิ่มขึ้นเหมือนต้นไผ่ในฤดูใบไม้ผลิ ไม่นานมานี้เรายังคงเชียร์ "AI ที่สามารถเขียนโค้ดได้" ตอนนี้เราอยู่ในยุคของการทำงานร่วมกันหลายรูปแบบและการบูรณาการเครื่องมือที่หลากหลายแล้ว น่าตื่นเต้นไหม? บางที. ท้ายที่สุดแล้ว สิ่งที่นักพัฒนาต้องการไม่ได้เป็นเพียงเครื่องมือเท่านั้น แต่ยังรวมถึงความง่ายในการปรับให้เข้ากับสถานการณ์ต่างๆ และการเปลี่ยนอย่างยืดหยุ่นเมื่อจำเป็น

ในฐานะแพลตฟอร์มการเขียนโค้ดที่ได้รับความช่วยเหลือจาก AI เมื่อเร็วๆ นี้ HagiCode ยินดีกับการพัฒนาที่สำคัญ 2 ประการ ได้แก่ ประการแรก การบูรณาการโมเดล GLM-5.1 ของ Zhipu AI อย่างเต็มรูปแบบ; ประการที่สอง การเพิ่มอย่างเป็นทางการของ Gemini CLI ในฐานะ Agent CLI ที่รองรับอันดับที่สิบ การอัปเดตทั้งสองนี้อาจไม่ฟังดูสะเทือนใจ แต่เป็นข่าวดีอย่างไม่ต้องสงสัยสำหรับการเติบโตอย่างต่อเนื่องของแพลตฟอร์ม

GLM-5.1 คือรุ่นเรือธงล่าสุดของ Zipu AI เมื่อเปรียบเทียบกับ GLM-5.0 มันให้เหตุผลที่ดีกว่า ความเข้าใจโค้ดที่ลึกกว่า และการเรียกเครื่องมือที่ราบรื่นกว่า ที่สำคัญกว่านั้นคือเป็น GLM รุ่นแรกที่รองรับอินพุตรูปภาพ นั่นหมายความว่าอย่างไร? หมายความว่าผู้ใช้สามารถปล่อยให้ AI ดูภาพหน้าจอได้โดยตรง แทนที่จะพยายามอธิบายปัญหาด้วยคำพูด เมื่อคุณได้ใช้ความสะดวกสบายนั้น คุณจะเข้าใจคุณค่าของมันทันที

ขณะเดียวกันก็ผ่านทาง `HagiCode.Libs.Providers` สถาปัตยกรรม HagiCode ประสบความสำเร็จในการรวม Gemini CLI เข้ากับแพลตฟอร์ม ตอนนี้เป็น Agent CLI ตัวที่สิบแล้ว พูดตามตรง การมาถึงจุดนี้ทำให้รู้สึกถึงความสำเร็จเล็กน้อย

เป็นที่น่าสังเกตว่าฟีเจอร์การอัพโหลดรูปภาพของ HagiCode ช่วยให้ผู้ใช้สื่อสารกับ AI ได้โดยตรงผ่านภาพหน้าจอ แม้ว่าจะใช้ GLM 4.7 ก็ตาม แพลตฟอร์มดังกล่าวก็ยังทำงานได้ดีและได้ช่วยทำงานด้านการสร้างที่สำคัญหลายอย่างให้เสร็จสมบูรณ์แล้ว สำหรับ GLM-5.1 แน่นอนว่ามันจะก้าวไปอีกขั้นหนึ่ง

## การออกแบบสถาปัตยกรรม Multi-CLI

จุดแข็งหลักประการหนึ่งของ HagiCode คือการรองรับเครื่องมือ CLI ในการเขียนโปรแกรม AI หลายตัวผ่านเลเยอร์นามธรรมที่รวมเป็นหนึ่งเดียว ข้อดีของการออกแบบนี้จริงๆ แล้วค่อนข้างง่าย: มีเครื่องมือใหม่เข้ามา เครื่องมือเก่ายังคงอยู่ และโค้ดเบสไม่กลายเป็นความสับสนวุ่นวาย พูดตามตรง นั่นคือวิธีที่ทุกคนอยากให้ชีวิตดำเนินไป

### แจงนับ AIProviderType

แพลตฟอร์มนี้กำหนดประเภทผู้ให้บริการ CLI ที่รองรับผ่านทาง `AIProviderType` แจงนับ:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI (new)
}
```

อย่างที่คุณเห็น Gemini CLI เข้าร่วมครอบครัวนี้ในฐานะสมาชิกคนที่สิบ CLI แต่ละรายการมีลักษณะเฉพาะและสถานการณ์การใช้งานที่แตกต่างกัน ดังนั้นผู้ใช้สามารถเลือกได้อย่างยืดหยุ่นตามความต้องการ ท้ายที่สุดแล้ว ถนนหลายสายมุ่งสู่กรุงโรม บางอย่างก็ง่ายกว่าอย่างอื่น

### สถาปัตยกรรมของผู้ให้บริการ

`HagiCode.Libs.Providers` มอบอินเทอร์เฟซผู้ให้บริการแบบรวมที่ทำให้การรวม CLI แต่ละรายการมีมาตรฐานและกระชับ ยกตัวอย่าง Gemini CLI:

```csharp
public class GeminiProvider : ICliProvider<GeminiOptions>
{
    private static readonly string[] DefaultExecutableCandidates = ["gemini", "gemini-cli"];
    private const string ManagedBootstrapArgument = "--acp";

    public string Name => "gemini";
    public bool IsAvailable => _executableResolver.ResolveFirstAvailablePath(DefaultExecutableCandidates) is not null;
}
```

ข้อดีของการออกแบบนี้คือ:
- การรวม CLI ใหม่ต้องใช้คลาสผู้ให้บริการเพียงคลาสเดียวเท่านั้น
- การจัดการวงจรการใช้งานแบบรวมและการรวมเซสชัน
- การแก้ไขนามแฝงอัตโนมัติและการค้นหาปฏิบัติการ

พูดง่ายๆ ก็คือ การออกแบบนี้เปลี่ยนสิ่งที่ซับซ้อนให้กลายเป็นสิ่งที่เรียบง่ายขึ้น และทำให้ชีวิตง่ายขึ้นอีกเล็กน้อย

### ทะเบียนผู้ให้บริการ

Provider Registry จะจัดการการแมปและการลงทะเบียนนามแฝงโดยอัตโนมัติ:

```csharp
if (provider is GeminiProvider)
{
    registry.Register(provider.Name, provider, ["gemini-cli"]);
    continue;
}
```

ซึ่งหมายความว่าผู้ใช้สามารถเรียกใช้ Gemini CLI ได้ด้วย `gemini` หรือ `gemini-cli`และระบบจะรับรู้โดยอัตโนมัติ มันเหมือนกับเพื่อนที่มีทั้งชื่อที่เป็นทางการและชื่อเล่น ไม่ว่าจะด้วยวิธีใดก็ตาม ผู้คนก็รู้ว่าคุณหมายถึงใคร

## รองรับรุ่น GLM-5.1

GLM-5.1 เป็นรุ่นเรือธงล่าสุดของ Zhipu AI และ HagiCode ได้ให้การสนับสนุนอย่างเต็มรูปแบบแล้ว

### แคตตาล็อกวิชาชีพมัธยมศึกษา

HagiCode จัดการโมเดลที่รองรับทั้งหมดผ่าน Secondary Professions Catalog นี่คือการกำหนดค่าสำหรับซีรีส์ GLM:

| รหัสรุ่น | ชื่อ | รองรับรูปภาพ | ตระกูล CLI ที่เข้ากันได้ |
|----------|------|----------------|-------------------------|
| `glm-4.7` | จีแอลเอ็ม 4.7 | - | โคล้ด, โค้ดบัดดี้, เฮอร์มีส, โกเดอร์, คิโระ |
| `glm-5` | จีแอลเอ็ม 5 | - | โคล้ด, โค้ดบัดดี้, เฮอร์มีส, โกเดอร์, คิโระ |
| `glm-5-turbo` | จีแอลเอ็ม 5 เทอร์โบ | - | โคล้ด, โค้ดบัดดี้, เฮอร์มีส, โกเดอร์, คิโระ |
| `glm-5.0` | GLM 5.0 (รุ่นเก่า) | - | โคล้ด, โค้ดบัดดี้, เฮอร์มีส, โกเดอร์, คิโระ |
| `glm-5.1` | จีแอลเอ็ม 5.1 | **จริง** | โคล้ด, โค้ดบัดดี้, เฮอร์มีส, โกเดอร์, คิโระ |

ลักษณะสำคัญของ GLM-5.1 สามารถสรุปได้ดังนี้:
- ตัวระบุเวอร์ชันแบบสแตนด์อโลนที่ไม่มีสัมภาระแบบเดิม
- GLM รุ่นแรกที่รองรับอินพุตรูปภาพ
- การใช้เหตุผลและความเข้าใจโค้ดที่แข็งแกร่งยิ่งขึ้น
- ความเข้ากันได้หลาย CLI ในวงกว้าง

### GLM-5.1 กับ GLM-5.0

ในระดับโค้ด ข้อแตกต่างที่สำคัญระหว่าง GLM-5.1 และ GLM-5.0 แสดงไว้ที่นี่:

```csharp
// GLM-5.0 (Legacy) - contains special retention logic
private const string Glm50CodebuddySecondaryProfessionId = "secondary-glm-5-codebuddy";
private const string Glm50CodebuddyModelValue = "glm-5.0";

// GLM-5.1 - standalone new model identifier
private const string Glm51SecondaryProfessionId = "secondary-glm-5-1";
private const string Glm51ModelValue = "glm-5.1";
```

GLM-5.0 มีป้ายกำกับ "Legacy" เนื่องจากเป็นตัวระบุเวอร์ชันเก่าที่เก็บรักษาไว้สำหรับความเข้ากันได้แบบย้อนหลัง ในทางตรงกันข้าม GLM-5.1 นั้นเป็นเวอร์ชันสแตนด์อโลนใหม่ล่าสุดที่ไม่มีภาระในอดีต บางสิ่งยังคงอยู่ในอดีต บ้างก็เดินทางได้เบากว่าและเคลื่อนที่ได้เร็วกว่า

### กำหนดค่า GLM-5.1

นี่คือตัวอย่างการกำหนดค่าสำหรับการใช้ GLM-5.1 ใน HagiCode:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

## คุณสมบัติการอัพโหลดรูปภาพ

การสนับสนุนรูปภาพของ HagiCode ดำเนินการผ่านทาง `SupportsImage` คุณสมบัติบน `SecondaryProfession`:

```csharp
public class HeroSecondaryProfessionSettingDto
{
    public bool SupportsImage { get; set; }
}
```

ในแค็ตตาล็อกวิชาชีพรอง การกำหนดค่า GLM-5.1 มีลักษณะดังนี้:

```json
{
  "id": "secondary-glm-5-1",
  "supportsImage": true
}
```

ซึ่งหมายความว่าผู้ใช้สามารถอัปโหลดภาพหน้าจอเพื่อการวิเคราะห์ AI ได้โดยตรง เช่น:
- ภาพหน้าจอของข้อความแสดงข้อผิดพลาด
- ปัญหาในหน้าจอ UI
- แผนภูมิการแสดงข้อมูล
- ผลการรันโค้ด

ไม่จำเป็นต้องอธิบายทุกอย่างด้วยตนเองอีกต่อไป เพียงอัปโหลดภาพหน้าจอ ความสะดวกของคุณสมบัตินี้จะเห็นได้ชัดเมื่อคุณใช้งานแล้ว บางครั้งการมองครั้งเดียวก็บอกอะไรได้มากกว่าคำอธิบายที่ยาว

## บูรณาการราศีเมถุน CLI

ในฐานะ Agent CLI ตัวที่สิบ Gemini CLI ถูกรวมเข้ากับ HagiCode ผ่านสถาปัตยกรรมผู้ให้บริการมาตรฐาน

### ตัวเลือกการกำหนดค่า

Gemini CLI รองรับชุดตัวเลือกการกำหนดค่าที่หลากหลาย:

```csharp
public class GeminiOptions
{
    public string? ExecutablePath { get; set; }
    public string? WorkingDirectory { get; set; }
    public string? SessionId { get; set; }
    public string? Model { get; set; }
    public string? AuthenticationMethod { get; set; }
    public string? AuthenticationToken { get; set; }
    public Dictionary<string, string?> AuthenticationInfo { get; set; }
    public Dictionary<string, string?> EnvironmentVariables { get; set; }
    public string[] ExtraArguments { get; set; }
    public TimeSpan? StartupTimeout { get; set; }
    public CliPoolSettings? PoolSettings { get; set; }
}
```

ตัวเลือกเหล่านี้ครอบคลุมทุกอย่างตั้งแต่การตั้งค่าพื้นฐานไปจนถึงคุณลักษณะขั้นสูง ทำให้ผู้ใช้สามารถกำหนดค่า CLI ตามความต้องการของตนเองได้อย่างยืดหยุ่น ขั้นตอนการทำงานของทุกคนแตกต่างกัน ดังนั้นจึงยินดีต้อนรับความยืดหยุ่นเล็กน้อยเสมอ

### โปรโตคอลการสื่อสาร ACP

Gemini CLI รองรับ ACP (Agent Communication Protocol) ซึ่งเป็นมาตรฐานการสื่อสาร CLI แบบครบวงจรของ HagiCode ผ่าน ACP CLI ที่แตกต่างกันสามารถโต้ตอบกับแพลตฟอร์มในลักษณะที่สอดคล้องกัน ทำให้งานบูรณาการง่ายขึ้นมาก กล่าวโดยสรุปคือ มันสร้างมาตรฐานให้กับชิ้นส่วนที่ซับซ้อนเพื่อให้ทุกคนสามารถทำงานได้ง่ายขึ้น

## การกำหนดค่าสภาพแวดล้อม

หากต้องการใช้โมเดล Zhipu AI คุณต้องกำหนดค่าตัวแปรสภาพแวดล้อมที่เกี่ยวข้อง

### แพลตฟอร์ม Zipu AI ZAI

```bash
export ANTHROPIC_AUTH_TOKEN="***"
export ANTHROPIC_BASE_URL="https://open.bigmodel.cn/api/anthropic"
```

### อาลีบาบา คลาวด์ แดชสโคป

```bash
export ANTHROPIC_AUTH_TOKEN="your-a...-key"
export ANTHROPIC_BASE_URL="https://coding.dashscope.aliyuncs.com/apps/anthropic"
```

เมื่อกำหนดค่าแล้ว HagiCode จะสามารถเรียกใช้รุ่น GLM-5.1 ได้ตามปกติ มันไม่ได้ยากหรือง่ายเป็นพิเศษ คุณเพียงแค่ต้องปฏิบัติตามการตั้งค่าตามที่ตั้งใจไว้

## แบบฝึกหัดการสร้างของ HagiCode เอง

เมื่อพูดถึงการปฏิบัติในโลกแห่งความเป็นจริง ตัวอย่างที่ดีที่สุดคือเวิร์กโฟลว์การสร้างของแพลตฟอร์ม HagiCode เอง กระบวนการพัฒนาของ HagiCode ได้ใช้ความสามารถของ AI อย่างเต็มที่แล้ว

### ทำงานได้ดีแม้กับ GLM 4.7

การออกแบบแพลตฟอร์มของ HagiCode ได้รับการปรับให้เหมาะสมเป็นอย่างดี ดังนั้นจึงยังสามารถมอบประสบการณ์การพัฒนาที่ดีได้แม้จะใช้ GLM 4.7 ก็ตาม แพลตฟอร์มดังกล่าวได้ช่วยดำเนินโครงการก่อสร้างที่สำคัญหลายโครงการให้เสร็จสิ้นแล้ว ซึ่งรวมถึง:
- การรวมผู้ให้บริการ CLI หลายราย
- การใช้งานคุณสมบัติการอัพโหลดรูปภาพ
- การสร้างเอกสารและการเผยแพร่เนื้อหา

นั่นเป็นสิ่งที่ดีจริงๆ ไม่ใช่ทุกคนที่ต้องการสิ่งใหม่ล่าสุดตลอดเวลา สิ่งที่เหมาะกับคุณที่สุดมักจะเป็นสิ่งที่สำคัญที่สุด

### GLM-5.1 ให้มากขึ้นโดยออกแรงน้อยลง

หลังจากอัปเกรดเป็น GLM-5.1 ความสามารถเหล่านี้จะแข็งแกร่งยิ่งขึ้น:
- ความเข้าใจโค้ดที่แข็งแกร่งยิ่งขึ้น ลดการสื่อสารไปมา
- การวิเคราะห์การพึ่งพาที่แม่นยำยิ่งขึ้น ชี้ไปในทิศทางที่ถูกต้องทันที
- การวินิจฉัยข้อผิดพลาดมีประสิทธิภาพมากขึ้น ค้นหาปัญหาได้รวดเร็วยิ่งขึ้น
- รองรับการป้อนรูปภาพ เร่งคำอธิบายปัญหา

มันเหมือนกับการเปลี่ยนจากจักรยานเป็นรถยนต์ คุณยังสามารถไปถึงจุดหมายเดิมได้ แต่ความเร็ว และความสะดวกสบายไม่เท่ากัน

## แนวทางปฏิบัติที่ดีที่สุดสำหรับการบูรณาการหลาย CLI

`HagiCode.Libs.Providers` จัดให้มีกลไกแบบครบวงจรสำหรับการลงทะเบียนและการใช้งาน:

```csharp
services.AddHagiCodeLibs();

var gemini = serviceProvider.GetRequiredService<ICliProvider<GeminiOptions>>();
var codebuddy = serviceProvider.GetRequiredService<ICliProvider<CodebuddyOptions>>();
var hermes = serviceProvider.GetRequiredService<ICliProvider<HermesOptions>>();
```

การออกแบบการขึ้นต่อกันนี้ทำให้การใช้งานใน CLI ที่แตกต่างกันมีความกระชับมาก และยังทำให้การทดสอบและการเยาะเย้ยหน่วยสะดวกยิ่งขึ้น Clean Code เป็นวิธีหนึ่งในการรับผิดชอบต่อตัวคุณเอง

### หมายเหตุ

มีบางสิ่งที่ต้องคำนึงถึงในการใช้งานจริง:

1. **การกำหนดค่าคีย์ API**: ตรวจสอบให้แน่ใจ `ANTHROPIC_AUTH_TOKEN` ตั้งค่าถูกต้องแล้ว หรือไม่สามารถเรียกโมเดลได้
2. **รุ่นที่มีจำหน่าย**: GLM-5.1 จะต้องเปิดใช้งานโดยผู้ให้บริการรุ่นที่เกี่ยวข้อง
3. **คุณสมบัติรูปภาพ**: เฉพาะรุ่นที่มี `supportsImage: true` สามารถใช้อัพโหลดรูปภาพได้
4. **การติดตั้ง CLI**: ก่อนใช้ Gemini CLI โปรดตรวจสอบให้แน่ใจก่อน `gemini` หรือ `gemini-cli` อยู่ในระบบ PATH

สิ่งเหล่านี้อาจเป็นรายละเอียดเล็กๆ น้อยๆ แต่รายละเอียดเล็กๆ น้อยๆ ที่ได้รับการจัดการไม่ดีอาจกลายเป็นปัญหาใหญ่ได้ ดังนั้นจึงควรค่าแก่การใส่ใจ

## บทสรุป

ด้วยการรองรับ GLM-5.1 อย่างเต็มรูปแบบและการบูรณาการ Gemini CLI ที่ประสบความสำเร็จ ทำให้ HagiCode เสริมความแข็งแกร่งให้กับขีดความสามารถของตนในฐานะแพลตฟอร์มการเขียนโปรแกรม AI หลายรุ่นและหลาย CLI การอัปเดตเหล่านี้ไม่เพียงแต่ทำให้ผู้ใช้มีทางเลือกมากขึ้นเท่านั้น แต่ยังแสดงให้เห็นถึงสถาปัตยกรรมและความสามารถในการปรับขนาดที่คาดการณ์ล่วงหน้าของ HagiCode อีกด้วย

การรองรับรูปภาพของ GLM-5.1 รวมกับฟีเจอร์การอัปโหลดภาพหน้าจอของ HagiCode ช่วยให้ AI "เข้าใจจากรูปภาพ" ได้ และลดต้นทุนในการอธิบายปัญหาได้อย่างมาก และด้วยการรองรับ CLI สิบรายการ ผู้ใช้จึงสามารถเลือกผู้ช่วยเขียนโปรแกรม AI ที่เหมาะกับความต้องการและสถานการณ์ของตนได้อย่างยืดหยุ่น ทางเลือกที่มากขึ้นมักจะเป็นสิ่งที่ดีเสมอไป

สิ่งสำคัญที่สุดคือ แนวทางปฏิบัติด้านการสร้างของ HagiCode เองได้พิสูจน์ให้เห็นว่าแพลตฟอร์มดังกล่าวสามารถทำงานได้ดีอยู่แล้วและทำงานที่ซับซ้อนให้สำเร็จได้ แม้ว่าจะใช้ GLM 4.7 ก็ตาม ในขณะที่การอัพเกรดเป็น GLM-5.1 ก็สามารถปรับปรุงประสิทธิภาพการพัฒนาให้ดียิ่งขึ้นไปอีกได้ ชีวิตก็มักจะเป็นเช่นนั้น คุณไม่จำเป็นต้องได้รับสิ่งที่ดีที่สุดเสมอไป แต่ต้องการเฉพาะสิ่งที่เหมาะกับคุณเท่านั้น แน่นอนว่าหากสิ่งที่เหมาะสมกับคุณสามารถพัฒนาให้ดียิ่งขึ้นได้ ก็ยิ่งดีขึ้นมากเท่านั้น

หากคุณสนใจแพลตฟอร์มการเขียนโปรแกรม AI หลายรุ่น หลาย CLI ลองใช้ HagiCode ซึ่งเป็นโอเพ่นซอร์ส ฟรี และยังมีการพัฒนาอยู่ การลองใช้ไม่มีค่าใช้จ่ายใดๆ เลย และอาจกลายเป็นสิ่งที่คุณต้องการได้อย่างแน่นอน
