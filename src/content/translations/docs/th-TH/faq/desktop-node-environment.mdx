HagiCode Desktop มาพร้อมกับ Node toolchain ที่จัดการโดยเดสก์ท็อป จะไม่เข้าควบคุมการติดตั้งโหนดส่วนกลางบนคอมพิวเตอร์ของคุณ จุดประสงค์คือเพื่อให้การเริ่มต้นเดสก์ท็อปและโฟลว์ที่เกี่ยวข้องกับรันไทม์มีพื้นฐานรันไทม์ที่เสถียรและตรวจสอบได้

อ่านคำถามที่พบบ่อยนี้หากคุณเห็น `portable-fixed/toolchain` ในการวินิจฉัยจำเป็นต้องรู้ว่าอันไหน `node` เดสก์ท็อปใช้งานอยู่ หรือสงสัยว่าเหตุใดเวอร์ชันโหนดระบบของคุณจึงไม่กำหนดพฤติกรรมรันไทม์ของเดสก์ท็อปโดยตรง

## เหตุใด Desktop จึงรวม Node

เดสก์ท็อปจำเป็นต้องเรียกใช้โฟลว์การสนับสนุนเดียวกันทั่วทั้งระบบปฏิบัติการ ช่องทางการติดตั้ง และสภาพแวดล้อมของผู้ใช้ หากขึ้นอยู่กับการติดตั้งโหนดที่มีอยู่ของผู้ใช้เท่านั้น ฝ่ายสนับสนุนจะต้องจัดการกับเวอร์ชันที่ล้าสมัย, npm ที่ขาดหายไป, ลำดับเส้นทางที่ไม่ถูกต้อง, แพ็คเกจส่วนกลางที่เสียหาย และนโยบายการลงทะเบียนเฉพาะองค์กรทั้งหมดในคราวเดียว

เดสก์ท็อปอธิบายสิ่งนี้อย่างเฉพาะเจาะจงมากขึ้นว่าเป็นรันไทม์ Node/npm ที่แยกออกมาสำหรับ CLI ที่มีการจัดการ แทนที่จะนำสภาพแวดล้อมทั้งระบบกลับมาใช้ใหม่ การออกแบบนี้มีข้อดีหลักดังนี้:

- **เวอร์ชันที่สอดคล้องกัน**: เดสก์ท็อปใช้สัญญารันไทม์ของโหนด 22 ที่ปักหมุดไว้ แทนที่จะคาดเดาเวอร์ชันหลักในเครื่องของผู้ใช้
- **โฟลว์การเริ่มต้นที่มีการควบคุม**: คำสั่งที่จัดการโดยเดสก์ท็อปสามารถเลือกแบบแพ็กเกจได้ `node` และ `npm` รายการ
- **ขอบเขตการกำหนดค่าที่ชัดเจน**: การกำหนดค่าที่เกี่ยวข้องกับ CLI จะถูกแยกออกจากการกำหนดค่าส่วนกลางของโฮสต์ ดังนั้นการอัปเดต การสลับ และการกู้คืนจึงมีโอกาสน้อยที่จะถูกรบกวนจากความแตกต่างของสภาพแวดล้อมของระบบ
- **ข้อขัดแย้งที่ลดลง**: พาธแบบรวมใช้กับสภาพแวดล้อมคำสั่งที่สร้างโดยเดสก์ท็อป และไม่ได้แทนที่โหนดส่วนกลางของผู้ใช้ในเทอร์มินัล
- **พฤติกรรมที่ไม่รบกวนโฮสต์**: การแยกนี้จะช่วยหลีกเลี่ยงการเขียนสถานะ toolchain ลงในสภาพแวดล้อม Node/npm ที่มีอยู่ของผู้ใช้ และการถอนการติดตั้งเดสก์ท็อปจะไม่รบกวน Node, npm หรือการกำหนดค่าที่มีอยู่แล้วบนเครื่อง

## วิธีที่เดสก์ท็อปเลือกเส้นทางของโหนด

เมื่อคุณเรียกใช้ Desktop ตามปกติ Desktop จะเลือกใช้ Node toolchain ที่ติดตั้งมาพร้อมกับแอป แทนที่จะอ่านการติดตั้ง Node จากระบบของคุณก่อน ซึ่งจะทำให้เวอร์ชันของโหนดในเทอร์มินัลของคุณแยกจากเวอร์ชันของโหนดที่เดสก์ท็อปคาดหวังในขณะที่ทำงาน

เมื่อเปิดใช้งาน toolchain แบบรวม สภาพแวดล้อมคำสั่งที่สร้างโดย Desktop จะใส่เส้นทาง toolchain แบบรวมไว้ที่ด้านหน้า `PATH`. ด้วยเหตุนี้ แม้ว่าระบบของคุณจะติดตั้งโหนดเวอร์ชันอื่นไว้ แต่โฟลว์ที่จัดการโดยเดสก์ท็อปจะแก้ไขเป็นโหนดที่รวมกลุ่มของเดสก์ท็อปและ npm ก่อน

สิ่งนี้มีผลกับสภาพแวดล้อมรันไทม์ที่สร้างโดยเดสก์ท็อปเท่านั้น ถ้าคุณวิ่ง `node`, `npm`หรือคำสั่งอื่นๆ โดยตรงในเทอร์มินัลระบบของคุณ คำสั่งเหล่านั้นยังคงใช้สภาพแวดล้อมระบบปกติและการกำหนดค่าเชลล์ของคุณต่อไป

ในสภาพแวดล้อมการวินิจฉัยหรือกระบวนการย่อยที่เริ่มต้นโดยเดสก์ท็อป คุณอาจเห็นตัวแปรดังนี้:

```bash
HAGICODE_PORTABLE_TOOLCHAIN_ROOT=/path/to/portable-fixed/toolchain
```

ตัวแปรนี้หมายความว่าสภาพแวดล้อมรันไทม์ที่จัดการโดยเดสก์ท็อปรู้จักรูทของ Toolchain ที่รวมมา การเห็นเส้นทางนี้ไม่ได้หมายความว่าเดสก์ท็อปเปลี่ยนโหนดระบบของคุณ หมายความว่าเดสก์ท็อปกำลังใช้สภาพแวดล้อมโหนดที่มาพร้อมกับแอปภายในขอบเขตรันไทม์ของตัวเองเท่านั้น

## โดยที่ Desktop ติดตั้งแพ็คเกจ npm ที่ได้รับการจัดการ

Desktop เก็บไฟล์รันไทม์และไฟล์แพ็กเกจ npm ที่ไม่แน่นอนไว้ในตำแหน่งที่ต่างกัน เส้นทาง toolchain ที่รวมไว้ เช่น `portable-fixed/toolchain/node` เป็นแหล่งที่มาที่ไม่เปลี่ยนรูปแบบสำหรับเดสก์ท็อปที่จัดการ `node` และ `npm` คำสั่ง แพ็คเกจสากลที่ได้รับการจัดการจะถูกติดตั้งภายใต้ไดเร็กทอรีข้อมูลผู้ใช้ Electron แทน:

```text
userData/node22/npmGlobal
```

หากในภายหลัง Desktop ย้ายไปที่ Node 24 จะใช้แผนผังแพ็คเกจแยกต่างหาก เช่น `userData/node24/npmGlobal`. แพ็คเกจที่ติดตั้งสำหรับโหนด 22 จะยังคงอยู่ในดิสก์ แต่จะไม่ได้รับการปฏิบัติเหมือนติดตั้งสำหรับโหนด 24 จนกว่าจะมีการติดตั้งใหม่ลงในคำนำหน้าโหนดหลักใหม่

สถานะการพึ่งพาเดสก์ท็อปจะตรวจสอบเฉพาะการใช้งานเท่านั้น `userData/node<major>/npmGlobal` คำนำหน้า ไฟล์แพ็คเกจที่อยู่นอกคำนำหน้านั้นจะถูกละเว้นโดยการจัดการการขึ้นต่อกัน
