**โควต้าทดลองใช้**: Doubao Open Platform ให้ผู้ใช้ใหม่จดจำคำพูดได้ฟรี **20 ชั่วโมง** ซึ่งเพียงพอที่จะตรวจสอบว่ากระแสการป้อนข้อมูลด้วยเสียงนี้เหมาะกับงานของคุณหรือไม่
:::

## ภาพรวม

### Doubao Voice Input คืออะไร

การป้อนข้อมูลด้วยเสียง Doubao เป็นคุณลักษณะการรู้จำเสียงแบบบูรณาการของ HagiCode ซึ่งสนับสนุนโดย ByteDance Doubao Open Platform คุณพูด และระบบจะเปลี่ยนคำพูดเป็นข้อความ HagiCode ยังใช้บริบทของโครงการปัจจุบันเพื่อช่วยให้จดจำคำศัพท์ของโดเมนและคำศัพท์ทางเทคนิคได้อย่างน่าเชื่อถือมากขึ้น

### กรณีการใช้งานที่สำคัญ

ใน HagiCode การป้อนข้อมูลด้วยเสียงจะทำงานได้ดีที่สุดสำหรับ:

- **ป้อนข้อเสนออย่างรวดเร็ว**: สร้างและส่งข้อเสนอด้วยการพูดง่ายๆ ซึ่งช่วยเพิ่มประสิทธิภาพอย่างมากโดยไม่ต้องพิมพ์
- **การให้ความคิดเห็น**: เพิ่มความคิดเห็นและข้อเสนอแนะในระหว่างการตรวจสอบโค้ดหรือการตรวจสอบเอกสารด้วยการป้อนข้อมูลด้วยเสียง
- **การตอบกลับข้อความ**: ตอบกลับข้อความในการสนทนาและการสนทนาอย่างรวดเร็วโดยไม่ต้องพิมพ์
- **การสร้างเนื้อหาแบบยาว**: สร้างเอกสารประกอบโครงการ ข้อกำหนดทางเทคนิค บันทึกการประชุม และเนื้อหาแบบยาวอื่นๆ ได้อย่างรวดเร็ว

### ข้อดีในทางปฏิบัติ

- **การบูรณาการเชิงลึกพร้อมการรับรู้บริบท**: HagiCode บูรณาการอย่างลึกซึ้งกับการรู้จำเสียงพูด Doubao โดยใช้ประโยชน์จากบริบทของโครงการปัจจุบันโดยอัตโนมัติ (เช่น โครงสร้างรหัส กลุ่มเทคโนโลยี คำศัพท์เฉพาะทางโดเมน) เพื่อให้การจดจำคำศัพท์ของโดเมนที่แม่นยำ ปรับปรุงความแม่นยำในการรู้จำคำศัพท์ทางเทคนิคอย่างมาก
- **ชั่วโมงทดลองใช้ฟรี**: แพลตฟอร์ม Doubao ให้เวลาจดจำฟรี 20 ชั่วโมงสำหรับผู้ใช้ใหม่ เพื่อสัมผัสประสบการณ์การป้อนข้อมูลด้วยเสียงโดยไม่มีสิ่งกีดขวาง
- **รวดเร็วมาก**: การจดจำแบบเรียลไทม์ ดูข้อความในขณะที่คุณพูดโดยไม่ต้องรอ
- **ความแม่นยำสูง**: ขับเคลื่อนโดยโมเดลการรู้จำเสียงขั้นสูงของ Doubao รวมกับบริบทของโครงการเพื่อผลลัพธ์ที่แม่นยำและเชื่อถือได้
- **การบูรณาการอย่างราบรื่น**: บูรณาการโดยตรงในกล่องป้อนข้อความของ HagiCode โดยไม่จำเป็นต้องสลับแอปพลิเคชัน
- **ใช้งานง่าย**: เพียงคลิกไมโครโฟนเพื่อเริ่ม ใช้งานง่ายและตรงไปตรงมา

## เริ่มต้นอย่างรวดเร็ว

### ข้อกำหนดเบื้องต้น

ก่อนที่จะใช้การป้อนข้อมูลด้วยเสียง Doubao คุณต้อง:

1. **รับบัญชีแพลตฟอร์มเปิด Doubao**
   - เยี่ยมชม [คอนโซลการรู้จำเสียง Doubao](https://console.volcengine.com/speech/service/10038)
   - ลงทะเบียนหรือเข้าสู่บัญชีของคุณ

2. **สร้างแอปพลิเคชันและรับข้อมูลประจำตัว**
   - สร้างแอปพลิเคชันการรู้จำเสียงบนแพลตฟอร์ม
   - รับของคุณ `APP ID` และ `Access Token`

3. **ตรวจสอบการเชื่อมต่อเครือข่าย**
   - บริการรู้จำเสียงต้องมีการเชื่อมต่อเครือข่าย
   - ตรวจสอบให้แน่ใจว่าอุปกรณ์ของคุณสามารถเข้าถึงบริการ Doubao API ได้

### ขั้นตอนการใช้งานขั้นพื้นฐาน

ต่อไปนี้เป็นขั้นตอนพื้นฐานในการใช้ Doubao Voice Input:

1. รับ `APP ID` และ `Access Token`
2. กำหนดค่าการจดจำเสียงในผลิตภัณฑ์
3. ทดสอบคีย์ API
4. ค้นหาช่องป้อนข้อมูลด้วยเสียง
5. คลิกปุ่มไมโครโฟน
6. ให้สิทธิ์ไมโครโฟน
7. เริ่มการบันทึก
8. ดูผลการจดจำแบบเรียลไทม์
9. คลิกเพื่อหยุดการบันทึก
10. ยืนยันว่าได้แทรกข้อความแล้ว

## ข้อกำหนดทางเทคนิค

### สภาพแวดล้อม HTTP

การป้อนข้อมูลด้วยเสียง Doubao จำเป็นต้องทำงานในสภาพแวดล้อม HTTP/HTTPS

| สิ่งแวดล้อม | สถานะการสนับสนุน | คำอธิบาย |
|-------------|----------------|-------------|
| **โลคัลโฮสต์** | ✓ | สภาพแวดล้อมการพัฒนาท้องถิ่น (http://localhost หรือ http://127.0.0.1) |
| **เซิร์ฟเวอร์ระยะไกล HTTPS** | ✓ | เว็บแอปพลิเคชันปรับใช้กับเครือข่ายสาธารณะด้วย HTTPS |
| **เซิร์ฟเวอร์ระยะไกล HTTP** | ✗ | เว็บแอปพลิเคชันปรับใช้กับเครือข่ายสาธารณะด้วย HTTP |
| **ไฟล์::// โปรโตคอล** | ✗ | ไม่รองรับการเปิดไฟล์ HTML โดยตรง |

:::ข้อควรระวัง
Doubao Voice Input มีข้อจำกัดดังต่อไปนี้:
1. **HTTPS จำเป็นสำหรับการปรับใช้ภายนอก**: เนื่องจากข้อจำกัดด้านความปลอดภัยของเบราว์เซอร์ การเข้าถึงไมโครโฟนจึงต้องใช้ HTTPS เมื่อเข้าถึงจากเครือข่ายสาธารณะ
2. **ไม่จำกัดที่อยู่ในท้องถิ่น**: รองรับทั้ง HTTP และ HTTPS เมื่อใช้งาน `localhost` หรือ `127.0.0.1`
3. **file:// ไม่รองรับโปรโตคอล**: ไฟล์ `file://` สภาพแวดล้อมโปรโตคอลไม่สามารถใช้การเชื่อมต่อไมโครโฟนและ WebSocket
:::

### เวอร์ชันเดสก์ท็อป

เวอร์ชันเดสก์ท็อปของ HagiCode มีสภาพแวดล้อม HTTP ในตัวและรองรับการป้อนข้อมูลด้วยเสียงอย่างสมบูรณ์ คุณสามารถใช้การรู้จำเสียงได้โดยตรงโดยไม่ต้องกำหนดค่าเพิ่มเติม

### โหมดโฮสต์

โหมดโฮสต์ท้องถิ่นรองรับการป้อนข้อมูลด้วยเสียง:
- รองรับทั้ง HTTP และ HTTPS เมื่อใช้งาน `localhost` หรือ `127.0.0.1`
- จำเป็นต้องใช้ HTTPS เมื่อปรับใช้กับเครือข่ายสาธารณะ

## ขั้นตอนการกำหนดค่า

### รับรหัสแอป

1. เยี่ยมชม [คอนโซลการรู้จำเสียง Doubao](https://console.volcengine.com/speech/service/10038)
2. เข้าสู่ระบบหรือลงทะเบียนบัญชี
3. ไปที่คอนโซลและสร้างแอปพลิเคชันการรู้จำเสียงใหม่
4. ในหน้ารายละเอียดแอปพลิเคชัน ให้ค้นหาและคัดลอก `APP ID`

### รับโทเค็นการเข้าถึง

1. ในคอนโซล Doubao Open Platform
2. ไปที่แอปพลิเคชันการรู้จำเสียงของคุณ
3. ค้นหาพื้นที่การจัดการคีย์ API
4. สร้างหรือคัดลอก `Access Token`

:::หมายเหตุ
กรุณาเก็บของคุณ `Access Token` ปลอดภัยและอย่าแชร์กับผู้อื่น หากคุณต้องการเปลี่ยน คุณสามารถสร้างใหม่ได้บน Doubao Open Platform
:::

### กำหนดค่าในผลิตภัณฑ์

1. เปิดแอปพลิเคชัน HagiCode
2. ไปที่ **การตั้งค่า** → **การตั้งค่าการจดจำเสียง**
3. กรอกข้อมูลต่อไปนี้ในแบบฟอร์มการกำหนดค่า:
   - **ผู้ให้บริการ**: เลือก `doubao` (โด้เปา)
   - **APP ID**: วาง APP ID ที่คุณได้รับจากแพลตฟอร์ม Doubao
   - **โทเค็นการเข้าถึง**: วางโทเค็นการเข้าถึงที่คุณได้รับจากแพลตฟอร์ม Doubao
4. (ไม่บังคับ) ปรับพารามิเตอร์การกำหนดค่าอื่นๆ ตามความจำเป็น
5. คลิกปุ่ม **ทดสอบคีย์ API** เพื่อตรวจสอบการกำหนดค่า
6. หลังจากการตรวจสอบสำเร็จ การกำหนดค่าจะถูกบันทึกลงในที่จัดเก็บในตัวเครื่องของเบราว์เซอร์โดยอัตโนมัติ

### ทดสอบคีย์ API

หลังการกำหนดค่า ขอแนะนำให้ทดสอบว่าคีย์ API ถูกต้องหรือไม่:

1. คลิกปุ่ม **ทดสอบคีย์ API**
2. ระบบจะเรียกอินเทอร์เฟซทดสอบเพื่อตรวจสอบการกำหนดค่าของคุณ
3. หากการกำหนดค่าถูกต้อง ข้อความแสดงความสำเร็จจะปรากฏขึ้น
4. หากการกำหนดค่าไม่ถูกต้อง ข้อความแสดงข้อผิดพลาดจะปรากฏขึ้น กรุณาตรวจสอบ:
   - APP ID และโทเค็นการเข้าถึงถูกต้องหรือไม่
   - ไม่ว่าการเชื่อมต่อเครือข่ายจะเป็นปกติหรือไม่
   - โทเค็นการเข้าถึงหมดอายุแล้วหรือไม่

:::คำแนะนำ
หากการทดสอบล้มเหลว ขั้นแรกให้ตรวจสอบว่าการเชื่อมต่อเครือข่ายเป็นปกติหรือไม่ จากนั้นยืนยันว่าคีย์ API จากแพลตฟอร์มเปิด Doubao นั้นถูกต้องและยังไม่หมดอายุ
:::

## การใช้งาน

### ค้นหาตำแหน่งป้อนข้อมูลด้วยเสียง

ใน HagiCode กล่องป้อนข้อความที่มีไอคอนไมโครโฟนรองรับการป้อนด้วยเสียง ส่วนประกอบเหล่านี้มักเรียกว่า `VoiceTextArea`.

ตำแหน่งป้อนข้อมูลด้วยเสียงทั่วไป ได้แก่:
- กล่องใส่ข้อความ
- พื้นที่แก้ไขข้อความ
- ช่องแบบฟอร์มต่างๆ ที่ต้องป้อนข้อความ

### เริ่มการจดจำเสียง

1. ค้นหาช่องป้อนข้อมูลที่มีไอคอนไมโครโฟน
2. คลิกปุ่มไมโครโฟน
3. เบราว์เซอร์จะขออนุญาตไมโครโฟน
4. คลิก **อนุญาต** เพื่ออนุญาตการเข้าถึงไมโครโฟน

:::หมายเหตุ
หากคุณได้อนุญาตไมโครโฟนไว้ก่อนหน้านี้ เบราว์เซอร์อาจไม่ขออนุญาตอีก หากต้องการตรวจสอบหรือเปลี่ยนแปลงการตั้งค่าการอนุญาต คุณสามารถดูได้ในไอคอนแม่กุญแจทางด้านซ้ายของแถบที่อยู่ของเบราว์เซอร์
:::

### คำอธิบายสถานะการบันทึก

หลังจากการอนุญาตสำเร็จ การจดจำเสียงจะเริ่มต้นโดยอัตโนมัติ และคุณจะเห็น:

- **ภาพเคลื่อนไหวของรูปคลื่น**: รูปคลื่นของคลื่นเสียงไดนามิกจะแสดงภายในปุ่มไมโครโฟน ซึ่งบ่งชี้ว่าอยู่ระหว่างการบันทึก
- **การแสดงระยะเวลา**: ระยะเวลาการบันทึกปัจจุบันจะแสดงใต้ปุ่ม
- **การจดจำแบบเรียลไทม์**: ข้อความที่จดจำจะแสดงชั่วคราวที่ตำแหน่งเคอร์เซอร์

### การแสดงผลลัพธ์การรับรู้แบบเรียลไทม์

ในระหว่างการบันทึก ระบบรู้จำเสียงจะแปลงคำพูดของคุณเป็นข้อความแบบเรียลไทม์:

- ข้อความที่รู้จักจะแสดงเป็นสีเทาชั่วคราวในช่องป้อนข้อมูล
- ผลลัพธ์การจดจำจะอัปเดตอย่างต่อเนื่องเมื่อคุณพูด
- รองรับการจดจำภาษาจีนกลางด้วยความแม่นยำสูง

:::คำแนะนำ
เพื่อผลลัพธ์การจดจำที่ดีขึ้น ขอแนะนำให้:
- ใช้ในสภาพแวดล้อมที่เงียบสงบ
- พูดอย่างชัดเจนและปานกลาง
- หลีกเลี่ยงการรบกวนเสียงรบกวนพื้นหลัง
:::

### หยุดการรับรู้

หากต้องการหยุดการจดจำเสียง คุณสามารถ:

1. **คลิกปุ่มไมโครโฟน**: คลิกปุ่มอีกครั้งเพื่อหยุดการบันทึก
2. **คลิกช่องป้อนข้อมูล**: การคลิกพื้นที่อื่นๆ ของช่องป้อนข้อมูลจะหยุดการบันทึกด้วย

หลังจากหยุดแล้ว ข้อความที่รู้จักในขั้นตอนสุดท้ายจะถูกแทรกอย่างเป็นทางการลงในช่องป้อนข้อมูล และคุณสามารถแก้ไขหรือส่งต่อได้
