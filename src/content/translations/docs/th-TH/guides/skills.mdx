คู่มือนี้ครอบคลุมถึง `Skills` แผงใน HagiCode

เปิดทักษะเมื่อคุณต้องการ:

- ดูว่าทักษะใดบ้างที่ติดตั้งไว้ในเครื่องนี้แล้ว
- รับทักษะที่แนะนำตามโปรเจ็กต์และพื้นที่เก็บข้อมูลปัจจุบันของคุณ
- ค้นหาคลังทักษะที่กว้างขึ้น
- ยืนยันแหล่งที่มา สถานะความน่าเชื่อถือ และคำสั่งการติดตั้งจริงก่อนการติดตั้ง
- จัดการแหล่งที่มาที่มีป้ายกำกับว่าเชื่อถือได้

## ทักษะมีไว้เพื่ออะไร

Skills คือแผงการเรียกดูและการติดตั้งทักษะในตัวของ HagiCode โดยให้ที่เดียวแก่คุณในการทบทวนทักษะที่มีในท้องถิ่น ค้นหาคำแนะนำ ค้นหาแกลเลอรี และตรวจสอบรายละเอียดการติดตั้งก่อนที่คุณจะติดตั้งสิ่งใดๆ

ใช้เมื่อ:

- คุณยังใหม่กับ HagiCode และต้องการดูว่ามีอะไรบ้างที่มีอยู่แล้ว
- คุณรู้ว่าคุณกำลังทำอะไรอยู่และต้องการทักษะการจับคู่ผู้ช่วยที่ดีขึ้น
- คุณต้องการติดตั้งทักษะใหม่แต่ต้องตรวจสอบแหล่งที่มาก่อน
- คุณต้องการหลีกเลี่ยงการติดตั้งสิ่งที่มีอยู่แล้วในเครื่องใหม่

:::tip[ลำดับการอ่านที่แนะนำ]
ในการผ่านครั้งแรกเริ่มต้นด้วย `Local Skills`แล้วย้ายไปที่ `Recommendation` และ `Gallery`จากนั้นตรวจสอบแผงรายละเอียดก่อนการติดตั้งเท่านั้น
:::

## 1. เปิดทักษะ

ในแถบนำทางด้านบนของพื้นที่ทำงาน HagiCode ให้คลิก `Skills` ปุ่มเพื่อเปิดแผง

![การนำทางด้านบน HagiCode พร้อมรายการทักษะ](../../img/screenshots/skills/entry-open-skills-from-top-navigation/original.png)

ลูกศรสีแดงในภาพหน้าจอจะเน้นที่จุดเริ่มต้น

หลังจากเปิดแผงขึ้นมา คุณจะเห็นแท็บหลักสี่แท็บ:

- `Local Skills`
- `Recommendation`
- `Gallery`
- `Trusted Providers`

:::note[เมื่อใดจึงจะเปิดทักษะ]
คุณไม่จำเป็นต้องออกจากโปรเจ็กต์ เซสชัน หรือโฟลว์การเริ่มต้นใช้งานปัจจุบัน ทักษะจะเปิดจากแถบด้านบนภายในพื้นที่ทำงานเดียวกัน
:::

## 2. ตรวจสอบทักษะที่ติดตั้งในเครื่อง

`Local Skills` ตอบคำถามทั่วไปสองข้อ: **มีการติดตั้งอะไรไว้บ้างแล้ว?** และ **ทักษะใดบ้างที่สามารถอัปเดตได้ในขณะนี้**

![แท็บทักษะท้องถิ่นพร้อมรายการทักษะที่ติดตั้งและการดำเนินการเป็นชุด](../../img/screenshots/shared/local-skills-view/original.png)

พื้นที่ที่สำคัญที่สุดในหน้านี้คือ:

- **ช่องค้นหา**: กรองทักษะในท้องถิ่นตามชื่อ ID เส้นทาง หรือตัวแทน
- **`Refresh inventory`**: สแกนสิ่งที่ติดตั้งอยู่ในเครื่องนี้อีกครั้ง
- **`Update all`**: เรียกใช้การอัปเดตเป็นชุดสำหรับทักษะที่รองรับการอัปเดตที่ได้รับการจัดการ
- **`Batch update summary`**: ตรวจสอบผลลัพธ์ชุดล่าสุดและบันทึกสถานะ

แต่ละแถวมักจะแสดง:

- ชื่อทักษะ
- ตัวแทนที่เข้ากันได้
- เส้นทางการติดตั้งในเครื่อง
- การดำเนินการที่มีอยู่

หากแถวเตือนว่าข้อมูลเมตาการติดตั้งที่จัดการแบ็กเอนด์หายไป ทักษะนั้นจะไม่สามารถใช้การอัปเดตในคลิกเดียวได้ และควรได้รับการตรวจสอบด้วยตนเอง

:::ข้อควรระวัง[การอัปเดตเป็นกลุ่มยังต้องมีการตรวจสอบโดยเจ้าหน้าที่]
`Update all` หมายความว่าขั้นตอนการอัปเดตจะดำเนินการเป็นกลุ่มเท่านั้น ไม่ได้หมายความว่าทักษะที่ระบุไว้ได้รับการตรวจสอบความปลอดภัยสำหรับกรณีการใช้งานของคุณแล้ว
:::

## 3. รับทักษะที่แนะนำ

`Recommendation` สร้างรายชื่อผู้สมัครจากโปรเจ็กต์และที่เก็บข้อมูลปัจจุบันของคุณ ซึ่งจะมีประโยชน์เมื่อคุณรู้ว่าคุณกำลังทำงานประเภทใด แต่คุณยังไม่ได้ตัดสินใจว่าจะติดตั้งทักษะใด

![แท็บคำแนะนำพร้อมผู้สมัครทักษะส่วนบุคคล](../../img/screenshots/skills/recommendation-personalized-skill-candidates/original.png)

ลำดับการอ่านเชิงปฏิบัติคือ:

1. ตรวจสอบตัวเลือกฮีโร่ที่ด้านบนเพื่อทำความเข้าใจว่าบริบทใดที่ขับเคลื่อนคำแนะนำ
2. คลิก `Generate recommendations`
3. ตรวจสอบชิปข้อความค้นหาเพื่อดูว่าเหตุใดจึงเลือกผู้สมัครเหล่านี้
4. เปรียบเทียบการ์ดตัวเลือกก่อนเปิดแผงรายละเอียด

ป้ายกำกับทั่วไปบนการ์ดคำแนะนำหมายถึง:

- `Recommended`: ระบบแนะนำอยู่ในขณะนี้
- `Installed`: ติดตั้งเรียบร้อยแล้ว
- `Installed locally`: มีอยู่แล้วในสภาพแวดล้อมท้องถิ่น และโดยปกติไม่จำเป็นต้องติดตั้งใหม่

:::หมายเหตุ[อะไร `installed locally` หมายถึง]
ป้ายกำกับนี้หมายความว่า "คุณมีแล้ว" ไม่ใช่ข้อความแจ้งให้คุณติดตั้งทันที
:::

## 4. ค้นหาและเปรียบเทียบในแกลเลอรี

หากคุณรู้สิ่งที่คุณกำลังมองหาอยู่แล้ว หรือต้องการเปรียบเทียบทักษะที่คล้ายกันจากแหล่งต่างๆ ให้ใช้ `Gallery`.

![แท็บแกลเลอรีพร้อมผลการค้นหา ตัวกรองแหล่งที่มา และจำนวนการติดตั้ง](../../img/screenshots/shared/skills-gallary/original.png)

`Gallery` เป็นสถานที่ที่ดีที่สุดสำหรับ:

- ค้นหาด้วยคำสำคัญ
- กรองไปยังแหล่งที่เชื่อถือได้ก่อน
- เปรียบเทียบทักษะที่คล้ายกันจากแหล่งต่างๆ
- ตรวจสอบจำนวนการติดตั้งและสถานะการติดตั้งในเครื่อง

เริ่มต้นด้วยสามฟิลด์:

- **`Source`** : ใครเป็นผู้ให้ทักษะ
- **`Installs`**: จำนวนการติดตั้งซึ่งสามารถทำหน้าที่เป็นสัญญาณสนับสนุนได้
- **`Installed locally`**: ไม่ว่าทักษะจะมีอยู่ในเครื่องของคุณหรือไม่

:::tip[ตรวจสอบแหล่งที่มาก่อนจำนวนการติดตั้ง]
ความนิยมสามารถช่วยได้ แต่ไม่ควรแทนที่การตรวจสอบแหล่งที่มา เริ่มต้นด้วย `Source`จากนั้นใช้ `Installs` เป็นบริบทเพิ่มเติม
:::

## 5. ตรวจสอบรายละเอียดทักษะก่อนการติดตั้ง

เมื่อคุณเปิดทักษะ HagiCode จะแสดงแผงรายละเอียด นี่เป็นหน้าจอที่สำคัญที่สุดในการตรวจสอบก่อนการติดตั้ง

![แผงรายละเอียดทักษะพร้อมสถานะความน่าเชื่อถือและคำสั่งที่ได้รับการจัดการ](../../img/screenshots/skills/skill-details-trust-status-and-managed-command/original.png)

ให้ความสนใจเป็นพิเศษกับ:

- **ชื่อ แหล่งที่มา และจำนวนการติดตั้ง**: ตรวจสอบให้แน่ใจว่าคุณได้เปิดรายการที่ต้องการแล้ว
- **`Trust status`**: เหตุใดแหล่งข้อมูลนี้จึงถือว่าเชื่อถือได้
- **`Managed command`**: คำสั่งจริง HagiCode พร้อมที่จะรันแล้ว
- **`Status`**: พื้นที่ทำงานปัจจุบันสามารถติดตั้งทักษะนี้ได้หรือไม่
- **`Overview`**: ตัวระบุที่รองรับ เช่น รหัสทักษะ, กระสุน, แหล่งที่มา และจำนวนการติดตั้ง

`Managed command` สำคัญเพราะเป็นคำสั่งติดตั้งจริง ไม่ใช่ข้อความตกแต่ง

:::ข้อควรระวัง[`trusted` ไม่ได้แปลว่า "ข้ามรีวิว"]
แม้ว่าแหล่งที่มาจะถูกทำเครื่องหมายไว้ก็ตาม `trusted`คุณยังควรอ่านทั้งสองอย่าง `Trust status` และ `Managed command` ก่อนทำการติดตั้ง
:::

## 6. จัดการผู้ให้บริการที่เชื่อถือได้

`Trusted Providers` ควบคุมว่าแหล่งใดจะได้รับ `trusted` ฉลาก เปิดเมื่อคุณต้องการทราบว่าเหตุใดแหล่งข้อมูลจึงเชื่อถือได้ หรือเมื่อคุณต้องการปรับกฎการเชื่อถือของคุณ

![แท็บผู้ให้บริการที่เชื่อถือได้พร้อมกฎของผู้ให้บริการและสถานะความน่าเชื่อถือ](../../img/screenshots/skills/trusted-providers-rule-list/original.png)

ในหน้านี้:

- **`Rule format`** อธิบายวิธีการเขียนกฎเกณฑ์
- **`Add provider`** ให้คุณเพิ่มชุดกฎของผู้ให้บริการ
- **`Reset defaults`** คืนค่าที่ตั้งไว้ล่วงหน้าเริ่มต้น
- **`Rules`**, **`Status`** และ **`Source examples`** คอลัมน์แสดงกฎการจับคู่ สถานะปัจจุบัน และตัวอย่างการจับคู่

รูปแบบกฎทั่วไปได้แก่:

- `prefix:owner/` สำหรับความไว้วางใจตามคำนำหน้า
- `exact:owner/repo` เพื่อแหล่งเดียวที่แน่นอน

:::note[ผู้ให้บริการที่เชื่อถือได้ไม่ใช่หน้าข้อมูลประจำตัว]
หน้านี้เป็นเพียงการตัดสินใจว่าแหล่งที่มาใดมีป้ายกำกับว่าเชื่อถือได้เท่านั้น มันไม่ได้เก็บโทเค็น รหัสผ่าน หรือความลับอื่นๆ
:::

## ตรวจสอบ 4 สิ่งนี้ก่อนการติดตั้ง

ก่อนที่คุณจะติดตั้งทักษะ ให้ทบทวนสัญญาณเหล่านี้อย่างน้อยสี่ประการ:

| ตรวจสอบ | จะดูที่ไหน | มันบอกอะไรคุณ |
| --- | --- | --- |
| ไม่ว่าแหล่งที่มาจะเชื่อถือได้หรือไม่ | `Trusted Providers`บวก `Trust status` ในแผงรายละเอียด | เหตุใดแหล่งข้อมูลนี้จึงถูกระบุว่าเชื่อถือได้ |
| ไม่ว่าจะติดตั้งแล้วก็ตาม | ป้ายสถานะใน `Recommendation` และ `Gallery` | ถ้ามันบอกว่า `installed locally`โดยปกติคุณไม่จำเป็นต้องติดตั้งใหม่ |
| ผลการอัพเดตเป็นชุด | `Batch update summary` ใน `Local Skills` | การอัปเดตเป็นกลุ่มครั้งล่าสุดสำเร็จหรือไม่ และต้องมีการตรวจสอบโดยเจ้าหน้าที่หรือไม่ |
| คำสั่งติดตั้งจริง | `Managed command` ในแผงรายละเอียด | สิ่งที่ HagiCode กำลังทำงานอยู่จริงๆ |

## เส้นทางผู้ใช้ทั่วไป

ขั้นตอนทั่วไปในครั้งแรกคือ:

1. เปิด `Local Skills` และตรวจสอบสิ่งที่มีอยู่แล้ว
2. ใช้ `Recommendation` เพื่อค้นหาผู้สมัครที่เหมาะกับงานปัจจุบันของคุณ
3. ใช้ `Gallery` เพื่อค้นหาและเปรียบเทียบแหล่งที่มา
4. เปิดแผงรายละเอียดเพื่อตรวจสอบสถานะความน่าเชื่อถือและคำสั่งติดตั้ง
5. หากจำเป็น ให้ตรวจสอบหรือปรับเปลี่ยนกฎการเชื่อถือใน `Trusted Providers`
