คู่มือนี้ครอบคลุมถึงการปรับใช้ Hagicode ด้วย Docker Compose สำหรับผู้ใช้ส่วนใหญ่ นี่เป็นเส้นทางการปรับใช้เริ่มต้นสำหรับสภาพแวดล้อมการพัฒนา การทดสอบ และการใช้งานจริง

## วิดีโอสอน

หากคุณต้องการดูโฟลว์ทั้งหมดก่อน ให้เริ่มด้วยวิดีโอบทช่วยสอน:

[ชมวิดีโอสอนการใช้งาน Bilibili](https://www.bilibili.com/video/BV19967B6EHr/)

**เนื้อหาวิดีโอประกอบด้วย**:
- การสาธิตกระบวนการปรับใช้ Docker Compos อย่างสมบูรณ์
- คำแนะนำในการสร้างและแก้ไขไฟล์การกำหนดค่า
- ขั้นตอนการเริ่มต้นบริการและการตรวจสอบ
- ปัญหาและแนวทางแก้ไขทั่วไป

ระยะเวลาวิดีโอ: ประมาณ 15 นาที

:::note[ตัวสร้างนักเทียบท่าเขียน]
ขณะนี้ Docker Compose Builder พร้อมใช้งานเป็นเครื่องมือแบบสแตนด์อโลนที่ [https://builder.hagicode.com/](https://builder.hagicode.com/). ใช้ไซต์นั้นเป็นแหล่งที่มาหลักในการสร้างไฟล์การกำหนดค่าระดับเดียวกัน
:::

:::tip[ใช้เครื่องมือสร้างการกำหนดค่า]
คุณสามารถใช้ [Docker Compose Builder (ไซต์สแตนด์อโลน)](https://builder.hagicode.com/) เพื่อสร้าง `docker-compose.yml` โดยตรง

เครื่องกำเนิดไฟฟ้ารองรับ:
- พอร์ตแบบกำหนดเอง ชื่อคอนเทนเนอร์ และการตั้งค่าพื้นฐานอื่นๆ
- การเลือกฐานข้อมูลภายในหรือภายนอก
- ความแตกต่างของแพลตฟอร์ม Windows / Linux อัตโนมัติ
- การจัดการสิทธิ์ไฟล์ทั่วไป
- การเลือกแหล่งภาพขึ้นอยู่กับสภาพแวดล้อมเครือข่ายของคุณ

[เปิดตัวสร้าง →](https://builder.hagicode.com/)
:::

:::tip[วิธีการแนะนำ]
Docker Compose เป็นเส้นทางการปรับใช้เริ่มต้นในกรณีส่วนใหญ่ เนื่องจาก:
- การแยกสภาพแวดล้อมช่วยลดความขัดแย้งในการพึ่งพา
- สแต็กบริการ Hagicode ที่รองรับสามารถเริ่มต้นพร้อมกันได้
- การกำหนดค่า การอัพเกรด และการบำรุงรักษานั้นง่ายต่อการรักษาความสม่ำเสมอ
- มันทำงานได้ดีสำหรับการทดลองใช้ การทดสอบ และการปรับใช้ทีมในพื้นที่
:::

:::note[เกี่ยวกับการจัดเก็บข้อมูลเริ่มต้น]
เทมเพลต Docker Compose ที่รองรับในปัจจุบันจะคงข้อมูลผ่านเส้นทาง SQLite ในเครื่องเริ่มต้น ในกรณีส่วนใหญ่ คุณไม่จำเป็นต้องจัดเตรียมบริการฐานข้อมูลแยกต่างหากบนเครื่องโฮสต์
:::

## ข้อกำหนดเบื้องต้น

ก่อนเริ่มต้น ตรวจสอบให้แน่ใจว่าระบบของคุณติดตั้ง Docker และ Docker Compos แล้ว

### ติดตั้งนักเทียบท่า

#### หน้าต่าง

ดาวน์โหลดและติดตั้ง [นักเทียบท่าเดสก์ท็อปสำหรับ Windows](https://www.docker.com/products/docker-desktop/)

หลังการติดตั้ง ตรวจสอบให้แน่ใจว่า Docker Desktop ทำงานอยู่

#### ระบบปฏิบัติการ macOS

ดาวน์โหลดและติดตั้ง [นักเทียบท่าเดสก์ท็อปสำหรับ Mac](https://www.docker.com/products/docker-desktop/)

หลังการติดตั้ง ตรวจสอบให้แน่ใจว่า Docker Desktop ทำงานอยู่

#### ลินุกซ์

ติดตั้ง Docker โดยใช้ตัวจัดการแพ็คเกจของคุณ:

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

หลังการติดตั้ง ให้เริ่มบริการ Docker:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### ตรวจสอบการติดตั้ง

หลังการติดตั้ง ให้รันคำสั่งต่อไปนี้เพื่อตรวจสอบว่าติดตั้ง Docker และ Docker Compose อย่างถูกต้อง:

```bash
docker --version
docker compose version
```

## เริ่มต้นอย่างรวดเร็ว

### 1. สร้างไฟล์การกำหนดค่า Docker Compose

:::tip[ใช้เครื่องมือสร้างการกำหนดค่า]
**แนะนำให้ใช้ [Docker Compose Builder (ไซต์สแตนด์อโลน)](https://builder.hagicode.com/)** เพื่อสร้างไฟล์กำหนดค่าของคุณ

เครื่องกำเนิดไฟฟ้ารองรับ:
- ตัวเลือกผู้ให้บริการ API หลายตัว (Zhipu AI, Anthropic Official, API ที่กำหนดเอง)
- การกำหนดค่าอัตโนมัติของ URL API และโทเค็น
- พอร์ตที่กำหนดเอง ชื่อคอนเทนเนอร์ และการกำหนดค่าพื้นฐานอื่นๆ
- กำหนดค่าข้อมูลถาวรและเส้นทางไดเร็กทอรีการทำงาน
- กำหนดค่าความแตกต่างของแพลตฟอร์ม Windows/Linux โดยอัตโนมัติ
- จัดการปัญหาการอนุญาตไฟล์อย่างชาญฉลาด
- **การเลือกแหล่งที่มาของภาพ**: ขึ้นอยู่กับสภาพแวดล้อมเครือข่ายของคุณ ให้เลือกแหล่งที่มาของภาพที่เหมาะสม

[🚀 ใช้เครื่องกำเนิดไฟฟ้าทันที →](https://builder.hagicode.com/)
:::

1. เปิด [Docker Compose Builder (ไซต์สแตนด์อโลน)](https://builder.hagicode.com/)
2. กรอกการกำหนดค่าตามความต้องการของคุณ:
   - เลือกผู้ให้บริการ API (Zhipu AI, Anthropic Official หรือ API ที่กำหนดเอง)
   - กำหนดค่าพอร์ต ฐานข้อมูล ไดเร็กทอรีการทำงาน และตัวเลือกอื่นๆ
   - **เลือกแหล่งที่มาของภาพ**: เลือกแหล่งที่มาของภาพที่เหมาะสมตามสภาพแวดล้อมเครือข่ายของคุณ
3. คลิกปุ่มสร้างเพื่อรับ `docker-compose.yml` การกำหนดค่า
4. บันทึกการกำหนดค่าที่สร้างขึ้นเป็น `docker-compose.yml` ไฟล์
5. หากจำเป็น ให้สร้าง `.env` ไฟล์เพื่อกำหนดค่าข้อมูลที่ละเอียดอ่อน

#### การเลือกแหล่งที่มาของภาพ

ตัวสร้างรองรับตัวเลือกแหล่งรูปภาพสามตัวเลือก:

**นักเทียบท่าฮับ**
- **ที่อยู่รูปภาพ**: `newbe36524/hagicode:{tag}`
- **สถานการณ์ที่เกี่ยวข้อง**: เหมาะสำหรับผู้ใช้ที่มีการเร่งความเร็วมิเรอร์ Docker Hub
- **ข้อดี**: แหล่งที่มาของรูปภาพอย่างเป็นทางการ การอัปเดตทันเวลา การเข้าถึงที่เสถียร
- **หมายเหตุ**: อาจจำเป็นต้องกำหนดค่าการเร่งความเร็วกระจกในบางภูมิภาค

**Azure คอนเทนเนอร์รีจิสตรี (ACR)**
- **ที่อยู่รูปภาพ**: `hagicode.azurecr.io/hagicode:{tag}`
- **สถานการณ์ที่เกี่ยวข้อง**: เหมาะสำหรับผู้ใช้ที่ไม่สามารถเข้าถึง Docker Hub ภายในเครื่องได้
- **ข้อดี**: ให้แหล่งรูปภาพสำรอง แก้ไขปัญหาการเข้าถึงเครือข่าย
- **หมายเหตุ**: รูปภาพจะยังคงซิงโครไนซ์กับ Docker Hub แต่อาจมีความล่าช้าเล็กน้อย

**Alibaba Cloud Container Registry (ACR)**
- **ที่อยู่รูปภาพ**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **สถานการณ์ที่ใช้งานได้**: เหมาะสำหรับผู้ใช้ในประเทศ การเข้าถึงเครือข่ายที่เสถียรและรวดเร็ว
- **ข้อดี**: โหนดภายในประเทศของ Alibaba Cloud ความเร็วในการเข้าถึงที่รวดเร็ว ความเสถียรสูง
- **หมายเหตุ**: รูปภาพของ Alibaba ACR จะยังคงซิงโครไนซ์กับ Docker Hub โดยปกติจะใช้เวลาภายใน 30 นาที

:::tip[ข้อแนะนำในการเลือกแหล่งรูปภาพ]
- **ผู้ใช้ในประเทศแนะนำ Alibaba Cloud ACR**: หากคุณเป็นผู้ใช้ในประเทศ ขอแนะนำให้จัดลำดับความสำคัญโดยใช้แหล่งรูปภาพของ Alibaba Cloud ACR เพื่อการเข้าถึงที่รวดเร็วและเสถียร
- **Alternative Docker Hub**: หากสภาพแวดล้อมเครือข่ายของคุณสามารถเข้าถึง Docker Hub ได้ คุณสามารถเลือก Docker Hub เป็นแหล่งรูปภาพได้
- **Azure ACR ทางเลือก**: เลือกเฉพาะแหล่งรูปภาพ Azure ACR หากแหล่งรูปภาพอื่นมีปัญหาในการเข้าถึง
- **การซิงโครไนซ์รูปภาพ**: อิมเมจ ACR ของ Alibaba Cloud จะยังคงซิงโครไนซ์กับ Docker Hub โดยปกติจะใช้เวลาภายใน 30 นาที
- **ความสอดคล้องของเวอร์ชัน**: แท็กรูปภาพมีความสอดคล้องกันในแหล่งที่มาของรูปภาพทั้งสามแหล่ง ไม่จำเป็นต้องกังวลเกี่ยวกับปัญหาเวอร์ชัน
:::

:::note[รับโทเค็น API]
คุณต้องกำหนดค่า Claude API Token เพื่อใช้ Hagicode:

**Zhipu AI (แนะนำ)**: [รับโทเค็น API →](https://www.bigmodel.cn/glm-coding?ic=14BY54APZA)

- คุ้มค่าคุ้มราคาสูง เหมาะสำหรับการใช้งานในชีวิตประจำวัน

### 2. เริ่มบริการ

ดำเนินการคำสั่งต่อไปนี้ในไดเร็กทอรีที่คุณ `docker-compose.yml` ตั้งอยู่:

```bash
docker compose up -d
```

คำสั่งนี้จะ:
- ดาวน์โหลดและเริ่มต้นคอนเทนเนอร์ Hagicode ที่กำหนดโดยเทมเพลตที่สร้างขึ้น
- สร้างและกำหนดค่าเครือข่าย
- เตรียมใช้งานไดเร็กทอรีข้อมูลเริ่มต้นและการพึ่งพารันไทม์

### 3. ตรวจสอบสถานะการบริการ

ตรวจสอบว่าคอนเทนเนอร์กำลังทำงานอยู่หรือไม่:

```bash
docker compose ps
```

คุณควรเห็นบริการที่กำหนดไว้ในเทมเพลตที่คุณสร้างขึ้นในสถานะ "กำลังทำงาน"

### ดูบันทึกการบริการ

หากต้องการดูบันทึกการบริการทั้งหมด:

```bash
# View all service logs
docker compose logs

# View Hagicode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## คำแนะนำในการกำหนดค่า

หากคุณต้องการแก้ไขการกำหนดค่า (เช่น การเปลี่ยนพอร์ต การสลับผู้ให้บริการ API ฯลฯ) คุณสามารถ:

1. สร้างการกำหนดค่าใหม่โดยใช้ [Docker Compose Builder (ไซต์สแตนด์อโลน)](https://builder.hagicode.com/)
2. แก้ไขด้วยตนเอง `docker-compose.yml` และ `.env` ไฟล์
3. เริ่มบริการใหม่เพื่อให้การกำหนดค่ามีผล:

```bash
docker compose restart
```

## เข้าถึงแอปพลิเคชัน

### เว็บอินเตอร์เฟส

หลังจากเริ่มต้นบริการสำเร็จ ให้เข้าถึงผ่านเบราว์เซอร์ของคุณ:

```
http://localhost:45000
```

### ความคงอยู่ของข้อมูล

เส้นทาง Docker Compose ที่รองรับจะใช้ไฟล์ข้อมูล SQLite ที่มีวอลุ่มถาวรตามค่าเริ่มต้น หากคุณต้องการตรวจสอบจุดเชื่อมต่อหรือวางแผนการสำรองข้อมูล ให้ตรวจสอบการแมปไดรฟ์ข้อมูลและไดเร็กทอรีข้อมูลในตัวที่สร้างขึ้น `docker-compose.yml`.
