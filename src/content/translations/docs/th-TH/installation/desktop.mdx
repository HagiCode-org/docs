คู่มือนี้ครอบคลุมถึงการติดตั้งและใช้งาน Hagicode Desktop เป็นไคลเอนต์เดสก์ท็อปอย่างเป็นทางการและเก็บเส้นทางการตั้งค่าเริ่มต้นในเครื่องพร้อมการควบคุมรายวันไว้ในที่เดียว

:::tip[วิธีการแนะนำ]
เดสก์ท็อปเป็นเส้นทางภายในเริ่มต้นสำหรับผู้ใช้และนักพัฒนาแต่ละราย เนื่องจาก:
- โปรแกรมติดตั้งจะเก็บโฟลว์เริ่มต้นไว้ในพาธที่แนะนำเดียว
- การอ้างอิงและบริการได้รับการจัดการในแอป
- UI ช่วยให้ตรวจสอบสถานะและการดำเนินงานได้ง่ายขึ้น
- การจัดการเวอร์ชันในตัวทำให้การสลับง่ายขึ้น
- ข้อมูลจะยังคงอยู่ในเครื่องโดยค่าเริ่มต้น
:::

## Hagicode Desktop คืออะไร?

Hagicode Desktop เป็นแอปพลิเคชั่นเดสก์ท็อปอย่างเป็นทางการของ Hagicode ออกแบบมาสำหรับระบบ Windows, macOS และ Linux มันรวมคุณสมบัติหลักของ Hagicode ไว้ด้วยกัน และตัวติดตั้งจะเก็บเส้นทางการตั้งค่าเริ่มต้นไว้ในโฟลว์ที่แนะนำเดียว

### ข้อได้เปรียบหลัก

- **การติดตั้งในคลิกเดียว**: ทำตามขั้นตอนการกำหนดค่าทั้งหมดให้เสร็จสิ้นโดยอัตโนมัติผ่านวิซาร์ดการติดตั้ง
- **การเตรียมรันไทม์อัตโนมัติ**: ตัวติดตั้งเริ่มต้นได้รวมรันไทม์ที่ต้องการไว้แล้ว ดังนั้นคุณไม่จำเป็นต้องติดตั้งโปรแกรมอื่นแยกต่างหาก `.NET` รันไทม์ก่อน
- **การจัดการบริการ**: การจัดการเซิร์ฟเวอร์ Hagicode ในตัว เริ่มต้นและหยุดบริการได้อย่างง่ายดาย
- **การสลับเวอร์ชัน**: รองรับการจัดการหลายเวอร์ชัน สลับระหว่างเวอร์ชันต่างๆ ได้อย่างอิสระ
- **การดำเนินงานในท้องถิ่น**: ข้อมูลทั้งหมดจะถูกจัดเก็บไว้ในเครื่อง เพื่อปกป้องความเป็นส่วนตัวของคุณ
- **การอัปเดตอัตโนมัติ**: ตรวจจับและดาวน์โหลดเวอร์ชันล่าสุดโดยอัตโนมัติ

### สถานการณ์ที่เกี่ยวข้อง

เดสก์ท็อปเหมาะสมอย่างยิ่งเมื่อ:
- นักพัฒนารายบุคคลที่ต้องการใช้ Hagicode ภายในเครื่อง
- ผู้ใช้ที่จำเป็นต้องใช้ Hagicode ออฟไลน์
- ผู้ใช้ที่ไม่ใช่ด้านเทคนิคที่ต้องการลดความซับซ้อนของกระบวนการติดตั้ง
- นักพัฒนาซอฟต์แวร์ที่ต้องการสลับระหว่างเวอร์ชันต่างๆ บ่อยครั้ง

### วิธีการดาวน์โหลดอื่นๆ

คุณยังสามารถรับตัวติดตั้ง Hagicode Desktop ได้จากช่องทางต่อไปนี้:

1. **หน้าเผยแพร่อย่างเป็นทางการ**: เยี่ยมชม [เปิดตัว Hagicode GitHub](https://github.com/Hagicode-org/releases) เพื่อดาวน์โหลดเวอร์ชันล่าสุด
2. **เว็บไซต์อย่างเป็นทางการ**: เยี่ยมชม [เว็บไซต์ฮากิโค้ด](https://hagicode.com/) เพื่อรับลิงค์ดาวน์โหลด

### เลือกเวอร์ชันที่ถูกต้อง

เลือกตัวติดตั้งที่เกี่ยวข้องสำหรับระบบปฏิบัติการของคุณ:
- **Windows**: ดาวน์โหลด `.exe` ผู้ติดตั้ง
- **macOS**: ดาวน์โหลด `.dmg` ผู้ติดตั้ง
- **Linux**: ดาวน์โหลด `.AppImage` หรือ `.tar.gz` แพ็คเกจการติดตั้ง

:::note[ความต้องการของระบบ]
- **Windows**: Windows 10 หรือเวอร์ชันที่สูงกว่า (64 บิต)
- **macOS**: macOS 10.15 หรือสูงกว่า
- **Linux**: การเผยแพร่ Linux กระแสหลัก (Ubuntu 20.04+, Fedora 33+ ฯลฯ)
:::

:::tip[ข้อกำหนดเบื้องต้นเริ่มต้น]
สำหรับขั้นตอนการติดตั้งภายในเครื่อง HagiCode ล่าสุด คุณไม่จำเป็นต้องเตรียมขั้นตอนแยกต่างหาก `.NET` รันไทม์ เดสก์ท็อปได้รวมรันไทม์ที่จำเป็นสำหรับโฟลว์เริ่มต้นและเตรียมใช้งานเส้นทาง SQLite ภายในเครื่องที่รองรับตามค่าเริ่มต้น
:::

## หมายเหตุการติดตั้งและการเปิดตัว

:::ข้อควรระวัง[ประกาศผู้ดูแล macOS]
หากคุณติดตั้งแอปเป็นเส้นทางเริ่มต้นแล้ว `/Applications/Hagicode.app` แต่ macOS ขึ้นข้อความว่า "แอปเสียหาย", "ไม่สามารถเปิดได้" หรือคำเตือน Gatekeeper ที่คล้ายกันเมื่อเปิดตัวครั้งแรก สาเหตุปกติคือ `com.apple.quarantine` คุณลักษณะที่ macOS เพิ่มให้กับแอพที่ดาวน์โหลดจากอินเทอร์เน็ต ไม่ใช่ความล้มเหลวในตัวติดตั้ง Hagicode Desktop

เรียกใช้คำสั่งต่อไปนี้ใน Terminal ต้องใช้สิทธิ์ของผู้ดูแลระบบและลบแอตทริบิวต์กักกันออกจากแอปซ้ำในเส้นทางการติดตั้งเริ่มต้น:

```bash
sudo xattr -dr com.apple.quarantine /Applications/Hagicode.app
```

หลังจากคำสั่งเสร็จสิ้น ให้เปิด Hagicode Desktop อีกครั้ง
:::

## เรียกใช้เดสก์ท็อป

หลังจากดาวน์โหลด ให้เปิดโปรแกรมติดตั้งหรือแพ็คเกจแอปพลิเคชัน แล้วปฏิบัติตามคำแนะนำบนหน้าจอ Desktop เตรียมรันไทม์ในพาธภายในเครื่องที่รองรับในปัจจุบัน จัดการเซิร์ฟเวอร์ Hagicode และแสดงความคืบหน้าและสถานะเมื่อจำเป็นต้องดาวน์โหลดผ่านเครือข่าย

:::note[ข้อกำหนดด้านเครือข่าย]
ตรวจสอบให้แน่ใจว่าการเชื่อมต่อเครือข่ายของคุณพร้อมใช้งาน เดสก์ท็อปอาจจำเป็นต้องดึงข้อมูลเวอร์ชันหรือแพ็คเกจส่วนประกอบจากอินเทอร์เน็ต หากการดาวน์โหลดล้มเหลว ให้ตรวจสอบการตั้งค่าเครือข่าย พร็อกซี หรือไฟร์วอลล์แล้วลองอีกครั้ง
:::

## การจัดการเวอร์ชัน

Hagicode Desktop รองรับการจัดการหลายเวอร์ชัน ช่วยให้คุณสามารถสลับระหว่างเวอร์ชันต่างๆ ได้อย่างอิสระ

![การจัดการเวอร์ชัน](../../img/screenshots/installation/install-desktop/12-version-management/original.png)

### สลับเวอร์ชัน

1. เปิดเดสก์ท็อป Hagicode
2. ไปที่หน้า "การจัดการเวอร์ชัน"
3. เลือกเวอร์ชันที่คุณต้องการเปลี่ยน
4. คลิกปุ่ม "เปลี่ยนเวอร์ชัน"
5. รอให้สวิตช์เสร็จสิ้น

:::note[หมายเหตุการสลับเวอร์ชัน]
- การสลับเวอร์ชันจะรีสตาร์ทเซิร์ฟเวอร์ Hagicode โดยอัตโนมัติ
- ตรวจสอบให้แน่ใจว่าคุณบันทึกงานทั้งหมดก่อนที่จะเปลี่ยน
- การกำหนดค่าโปรเจ็กต์อาจแตกต่างกันระหว่างเวอร์ชัน
:::

## เริ่มและหยุดบริการ

Hagicode Desktop มอบวิธีง่ายๆ ในการจัดการบริการเซิร์ฟเวอร์ Hagicode

### เริ่มบริการ

บนโฮมเพจของแอปพลิเคชันเดสก์ท็อป ให้คลิกปุ่ม "เริ่มบริการ" เพื่อเริ่มเซิร์ฟเวอร์ Hagicode

![เริ่มบริการ](../../img/screenshots/installation/install-desktop/13-start-service-homepage/original.png)

### หยุดบริการ

หากต้องการหยุดเซิร์ฟเวอร์ Hagicode ให้คลิกปุ่ม "หยุดบริการ" บนหน้าแรก

:::tip[สถานะการบริการ]
แอปพลิเคชันเดสก์ท็อปจะแสดงสถานะการทำงานของเซิร์ฟเวอร์ Hagicode แบบเรียลไทม์ ได้แก่:
- สถานะการบริการ (ทำงาน/หยุด)
- หมายเลขพอร์ตบริการ
- ข้อมูลเวอร์ชันปัจจุบัน
:::

## การแก้ไขปัญหา

### การติดตั้งล้มเหลว

หากคุณประสบปัญหาระหว่างการติดตั้ง:

1. **ตรวจสอบการเชื่อมต่อเครือข่าย**: ตรวจสอบให้แน่ใจว่าคุณสามารถเข้าถึงอินเทอร์เน็ตได้
2. **ตรวจสอบเนื้อที่ดิสก์**: ตรวจสอบให้แน่ใจว่าคุณมีพื้นที่ดิสก์เพียงพอ
3. **เรียกใช้ในฐานะผู้ดูแลระบบ**: บน Windows ให้คลิกขวาที่ตัวติดตั้งและเลือก "เรียกใช้ในฐานะผู้ดูแลระบบ"
4. **ดูไฟล์บันทึก**: โปรแกรมติดตั้งจะสร้างไฟล์บันทึกโดยละเอียดที่สามารถช่วยวินิจฉัยปัญหาได้

### ความล้มเหลวในการติดตั้งการพึ่งพา

หากการติดตั้งการพึ่งพาล้มเหลว:

1. **ตรวจสอบความเข้ากันได้ของระบบ**: ตรวจสอบให้แน่ใจว่าระบบปฏิบัติการของคุณตรงตามข้อกำหนดขั้นต่ำ
2. **ตรวจสอบการตั้งค่าเครือข่ายและไฟร์วอลล์**: ตรวจสอบให้แน่ใจว่าตัวติดตั้งสามารถเข้าถึงการดาวน์โหลดที่จำเป็น และไม่ถูกบล็อกโดยไฟร์วอลล์ของคุณ
3. **ยืนยันสิทธิ์ของตัวติดตั้ง**: บน Windows ให้ลองเรียกใช้ตัวติดตั้งในฐานะผู้ดูแลระบบเพื่อให้มีสิทธิ์เพียงพอ
4. **รีสตาร์ทแอปแล้วลองอีกครั้ง**: ปิดและเปิด Hagicode Desktop อีกครั้ง จากนั้นลองขั้นตอนการติดตั้งอีกครั้ง
5. **ดูบันทึก**: ตรวจสอบบันทึกข้อผิดพลาดโดยละเอียดในแอปพลิเคชันเดสก์ท็อป

### ความล้มเหลวในการเริ่มต้นบริการ

หากเซิร์ฟเวอร์ Hagicode ไม่สามารถเริ่มต้นได้:

1. **ตรวจสอบการใช้พอร์ต**: ตรวจสอบให้แน่ใจว่าพอร์ต 45000 ไม่ได้ถูกครอบครองโดยโปรแกรมอื่น
2. **ตรวจสอบการตั้งค่าไฟร์วอลล์**: ตรวจสอบให้แน่ใจว่าไฟร์วอลล์อนุญาตให้เข้าถึงเครือข่ายเซิร์ฟเวอร์ Hagicode
3. **รีสตาร์ทแอปพลิเคชัน**: ลองรีสตาร์ท Hagicode Desktop
4. **ดูบันทึก**: ดูบันทึกข้อผิดพลาดโดยละเอียดในแอปพลิเคชันเดสก์ท็อป
