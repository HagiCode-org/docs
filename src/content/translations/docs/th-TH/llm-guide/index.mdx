## ทำไมต้องเป็นคู่มือ LLM

ในเวิร์กโฟลว์การสร้างโค้ด ประสิทธิภาพของโมเดลจะแตกต่างกันอย่างมากทั้งด้านคุณภาพ เวลาแฝง และต้นทุน หมวดหมู่นี้มีโครงสร้างการประเมินที่สอดคล้องกันเพื่อช่วยให้นักพัฒนาเลือกโมเดลได้เร็วขึ้น

## เส้นทางการอ่านที่แนะนำ

1. เริ่มต้นด้วย [การเปรียบเทียบและประเมินแบบจำลองใน HagiCode](./model-comparison-evaluation/).
2. เลือกผู้สมัครตามลำดับความสำคัญของคุณ (คุณภาพมาก่อน ต้นทุนมาก่อน หรือความเร็วมาก่อน)
3. ตรวจสอบบันทึกเวลาทดสอบก่อนที่จะนำข้อสรุปไปใช้กับการใช้งานจริง

## หลักการบำรุงรักษา

- รีเฟรชข้อสรุปเมื่อความสามารถของโมเดลเปลี่ยนแปลงไปอย่างมาก
- ตรวจสอบบริบทเวลาทดสอบอีกครั้งอย่างน้อยเดือนละครั้ง
- จัดแนวหน้าภาษาจีนและอังกฤษในโครงสร้างส่วน
