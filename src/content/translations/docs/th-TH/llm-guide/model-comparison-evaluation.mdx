## ขอบเขตและวิธีการ

- **เป้าหมาย**: ให้คำแนะนำในการเลือกโมเดลตามประสบการณ์การบูรณาการจริงใน HagiCode
- **ประเภทงาน**: การใช้งานคอมโพเนนต์ส่วนหน้า, การปรับโครงสร้าง API ของแบ็กเอนด์, การทดสอบให้เสร็จสิ้น และการสร้างเอกสาร
- **แกนประเมิน**: ประสิทธิภาพในการส่งมอบ (สามารถทำงานให้สำเร็จได้อย่างน่าเชื่อถือหรือไม่) และความคุ้มค่า (ต้นทุน + ความพร้อมในประเทศ)

## การจัดอันดับประสิทธิภาพการจัดส่ง (คำแนะนำของผู้เขียน)

จัดอันดับตามคุณภาพความสำเร็จของงานและคุณภาพการปฏิบัติงานด้านวิศวกรรม:

1. GPT 5.4
2. GPT 5.3 โคเด็กซ์
3. จีแอลเอ็ม 5
4. จีแอลเอ็ม 4.7
5. มินิแม็กซ์ M2.5
6. Qwen (3.5 / โค้ดถัดไป)

### ข้อค้นพบที่สำคัญ

- ยกเว้น Qwen โมเดลที่ได้รับการทดสอบอื่นๆ ทั้งหมดสามารถบรรลุผลตามเป้าหมายของเราได้ในระดับหนึ่ง
- โดยทั่วไปแล้ว GLM 4.7+ (GLM 4.7 และ GLM 5) จะตอบสนองความต้องการของเราได้อย่างราบรื่น
- GPT 5.3 Codex และ GPT 5.4 ไม่เพียงแต่ตอบสนองความต้องการเท่านั้น แต่ยังสร้างแนวปฏิบัติทางวิศวกรรมและคุณภาพการใช้งานที่ดีขึ้นอีกด้วย
- Minimax M2.5 มีจุดอ่อนที่เกิดซ้ำ: ข้อผิดพลาดในการปิดโค้ด (เช่น การปิดวงเล็บ/บล็อกที่ไม่สมบูรณ์) ดังนั้นจึงจำเป็นต้องมีการตรวจสอบเพิ่มเติม

## การจัดอันดับความคุ้มทุน (ต้นทุน + ความพร้อมใช้งานในประเทศ)

จัดอันดับตามต้นทุนทางเศรษฐกิจและความพร้อมใช้งานจริงในประเทศจีน:

1. มินิแม็กซ์ M2.5
2. จีแอลเอ็ม 4.7
3. จีแอลเอ็ม 5
4. Qwen 3.5 / รหัสถัดไป
5. GPT 5.3 โคเด็กซ์
6. GPT 5.4

> หมายเหตุ: การจัดอันดับนี้มีเจตนาที่จะแตกต่างจากการจัดอันดับประสิทธิภาพในการจัดส่ง

## คำแนะนำการเลือก

- **คุณภาพต้องมาก่อน**: เลือก GPT 5.4 / GPT 5.3 Codex
- **กลยุทธ์ที่สมดุล**: เลือก GLM 5 / GLM 4.7
- **ต้นทุนมาก่อน**: เลือก Minimax M2.5 (พร้อมการตรวจสอบการปิดรหัสที่เข้มงวดยิ่งขึ้น)
- **การกำหนดเส้นทางที่ใช้งานได้จริง**: ใช้โมเดลระดับพรีเมียมสำหรับงานสำคัญ และใช้โมเดลที่คุ้มค่าสำหรับงานประจำ

## คำสั่งโมเดลที่ยังไม่ทดลอง

สำหรับรุ่นที่ไม่อยู่ในรายการ ที่นี่ ขณะนี้เรา **ไม่มีข้อมูลการทดสอบ** และ **ไม่มีประสบการณ์จริง** ดังนั้นเราจึงไม่มีการประเมิน

หากผู้สนับสนุนให้สิทธิ์เข้าถึงโมเดลเพิ่มเติม เราจะดำเนินการประเมินตามประสบการณ์ในขั้นตอนการทำงานจริงของเราและอัปเดตหน้านี้
