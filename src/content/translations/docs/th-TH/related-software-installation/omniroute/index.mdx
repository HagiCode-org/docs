หน้านี้มีไว้สำหรับผู้ใช้ HagiCode ครั้งแรกที่ต้องการเส้นทางการตั้งค่า OmniRoute ที่สั้นที่สุด: เริ่มบริการในพื้นที่ ตรวจสอบว่า `http://localhost:20128/v1` ใช้งานได้และยืนยันว่าทั้งแดชบอร์ดและ API พร้อมใช้งาน

:::tip[เส้นทางแนะนำ]
เวอร์ชันแรกนี้จะเก็บเส้นทางที่เสถียรที่สุดไว้ก่อน: `npm install -g omniroute`, `omniroute`, แดชบอร์ดที่ `http://localhost:20128`และ URL พื้นฐานที่เข้ากันได้กับ OpenAI ที่ `http://localhost:20128/v1`. โหมดการใช้งาน Docker, VPS และขั้นสูงอื่นๆ จะยังคงอยู่ในส่วนเสริม
:::

## ข้อกำหนดเบื้องต้น

- ติดตั้ง Node.js แล้ว เสร็จสมบูรณ์ [การติดตั้ง Node.js](/en/related-software-installation/nodejs/installation) ก่อนหากจำเป็น
- ท่าเรือท้องถิ่น `20128` มีอยู่และยังไม่ได้ครอบครอง
- คุณมีบัญชีผู้ให้บริการโมเดลอัปสตรีมอย่างน้อยหนึ่งบัญชีที่คุณวางแผนจะกำหนดเส้นทางผ่าน OmniRoute
- ก่อนที่จะรวม HagiCode, CLI หรือ SDK ให้แยกที่อยู่ทั้งสองนี้ออกจากกัน:
  - แดชบอร์ด: `http://localhost:20128`
  - URL ฐานที่เข้ากันได้กับ OpenAI: `http://localhost:20128/v1`

## ติดตั้งและรันภายในเครื่อง

เริ่มต้นด้วยเส้นทาง npm ทั่วโลกที่แนะนำ:

```bash
npm install -g omniroute
omniroute
```

หลังจากที่บริการเริ่มต้น ให้ตรวจสอบความถูกต้องตามลำดับนี้:

1. เปิด `http://localhost:20128` ในเบราว์เซอร์และยืนยันว่าแดชบอร์ดโหลดแล้ว
2. ไปที่ `Endpoints` ในแดชบอร์ดและสร้างหรือคัดลอกคีย์ API
3. สังเกต URL หลักที่คุณจะมอบให้กับเครื่องมือในภายหลัง: `http://localhost:20128/v1`.

:::note[ทำไมต้องเริ่มต้นด้วยแดชบอร์ด]
สำหรับการตั้งค่าครั้งแรก แดชบอร์ดคือวิธีที่เร็วที่สุดในการยืนยันว่า OmniRoute กำลังทำงานอยู่ ผู้ให้บริการของคุณเชื่อมต่ออยู่ และมีคีย์ API อยู่ หากการเรียกไคลเอ็นต์ในภายหลังล้มเหลว การตรวจสอบแดชบอร์ดก่อนมักจะเร็วกว่าการดีบักโค้ด SDK ทันที
:::

## ตรวจสอบบริการที่ทำงานอยู่

### การตรวจสอบความถูกต้อง 1: ยืนยันว่าแค็ตตาล็อกโมเดลสามารถอ่านได้

ใส่กุญแจจาก `Endpoints` หน้าลงในตัวแปรสภาพแวดล้อม:

```bash
export OMNIROUTE_API_KEY="replace-with-the-key-you-copied-from-the-Endpoints-page"
```

แล้วขอ `/v1/models`:

```bash
curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}"
```

หากคุณได้รับรายชื่อโมเดลกลับมา แสดงว่าจุดสิ้นสุดที่เข้ากันได้กับ OpenAI และโฟลว์การตรวจสอบสิทธิ์กำลังทำงานอยู่

### การตรวจสอบความถูกต้อง 2: ส่งคำขอแชทขั้นต่ำ

```bash
curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "replace-with-a-model-id-from-v1-models",
    "messages": [
      { "role": "user", "content": "Reply with ok." }
    ]
  }'
```

เป้าหมายที่นี่ไม่ใช่คุณภาพที่รวดเร็ว เป็นการพิสูจน์ว่าโมเดลที่คุณเลือกสามารถกำหนดเส้นทางผ่าน OmniRoute ได้สำเร็จ
