คู่มือนี้ครอบคลุมถึงการติดตั้ง Node.js บน Windows, macOS และ Linux Node.js เป็นการพึ่งพาที่สำคัญสำหรับเครื่องมือภายใน HagiCode รวมถึง OpenSpec เครื่องมือส่วนหน้า และ CLI ของนักพัฒนาหลายราย

:::tip[ข้อกำหนดเวอร์ชัน]
HagiCode แนะนำ **Node.js 24+** (ขั้นต่ำที่รองรับ Node.js >= 18.0) ในกรณีส่วนใหญ่ ให้ติดตั้ง LTS เวอร์ชันล่าสุด จากนั้นตรวจสอบว่าเวอร์ชันของคุณตรงตามข้อกำหนดหรือไม่
:::

## การติดตั้ง Node.js

<Tabs>
  <TabItem value="windows" label="Windows" default>

### วิธีที่หนึ่ง: ตัวติดตั้งอย่างเป็นทางการ (แนะนำ)

1. เยี่ยมชม [เว็บไซต์อย่างเป็นทางการของ Node.js](https://nodejs.org/)
2. ดาวน์โหลดตัวติดตั้ง Windows เวอร์ชัน **LTS (Long Term Support)** (`.msi` ไฟล์)
3. ดับเบิลคลิกเพื่อเรียกใช้โปรแกรมติดตั้ง
4. ปฏิบัติตามคำแนะนำในการติดตั้งเพื่อทำการติดตั้งให้เสร็จสิ้น:
   - ยอมรับข้อตกลงใบอนุญาต
   - เลือกเส้นทางการติดตั้ง (ค่าเริ่มต้นคือใช้ได้)
   - ตรวจสอบให้แน่ใจว่าได้เลือก "ติดตั้งเครื่องมือที่จำเป็นโดยอัตโนมัติ"
   - คลิก "ติดตั้ง" เพื่อเริ่มการติดตั้ง
5. หลังจากการติดตั้งเสร็จสิ้น คลิก "เสร็จสิ้น"

### วิธีที่สอง: การใช้ winget

เปิด PowerShell หรือ Command Prompt แล้วเรียกใช้:

```powershell
winget install OpenJS.NodeJS.LTS
```

หลังการติดตั้ง ให้เปิดเทอร์มินัลอีกครั้งเพื่อให้การเปลี่ยนแปลงมีผล

  </TabItem>
  <TabItem value="macos" label="macOS">

### วิธีที่หนึ่ง: ตัวติดตั้งอย่างเป็นทางการ (แนะนำ)

1. เยี่ยมชม [เว็บไซต์อย่างเป็นทางการของ Node.js](https://nodejs.org/)
2. ดาวน์โหลดตัวติดตั้ง macOS เวอร์ชัน **LTS (Long Term Support)** (`.pkg` ไฟล์)
3. ดับเบิลคลิกเพื่อเรียกใช้โปรแกรมติดตั้ง
4. ปฏิบัติตามคำแนะนำในการติดตั้งเพื่อทำการติดตั้งให้เสร็จสิ้น:
   - คลิก "ดำเนินการต่อ"
   - ยอมรับข้อตกลงใบอนุญาต
   - เลือกดิสก์การติดตั้ง
   - คลิก "ติดตั้ง" และป้อนรหัสผ่านผู้ดูแลระบบ
5. หลังจากการติดตั้งเสร็จสิ้น คลิก "ปิด"

### วิธีที่สอง: การใช้ Homebrew

ถ้าคุณมี [โฮมบรูว์](https://brew.sh/) ติดตั้งแล้ว คุณสามารถติดตั้ง Node.js ผ่านเทอร์มินัลได้:

```bash
brew install node
```

:::คำแนะนำ
Homebrew จะเพิ่ม Node.js ให้กับ PATH ของคุณโดยอัตโนมัติ หากไม่มีคำสั่ง ให้รีสตาร์ทเทอร์มินัลหรือรัน:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

### การใช้พื้นที่เก็บข้อมูล NodeSource (แนะนำ)

นี่เป็นวิธีที่ดีที่สุดในการรับ Node.js เวอร์ชันล่าสุด:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### การใช้ apt Package Manager

หากคุณไม่ต้องการเวอร์ชันล่าสุด คุณสามารถใช้พื้นที่เก็บข้อมูลเริ่มต้นของ Ubuntu ได้:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::หมายเหตุ
เวอร์ชัน Node.js ในพื้นที่เก็บข้อมูลเริ่มต้นอาจเก่ากว่า หลังการติดตั้ง ให้ตรวจสอบเวอร์ชันเพื่อให้แน่ใจว่าเป็นไปตามข้อกำหนด Node.js >= 18.0 (แนะนำให้ใช้ Node.js 24+)
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### การใช้พื้นที่เก็บข้อมูล NodeSource (แนะนำ)

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### การใช้ yum Package Manager

```bash
sudo yum install -y nodejs npm
```

  </TabItem>
  <TabItem value="fedora" label="Fedora">

### การใช้ตัวจัดการแพ็คเกจ dnf

```bash
sudo dnf install -y nodejs npm
```

:::คำแนะนำ
ที่เก็บซอฟต์แวร์ของ Fedora มักจะมี Node.js เวอร์ชันใหม่กว่า และสามารถติดตั้งได้โดยตรงผ่าน dnf
:::

## การใช้เครื่องมือการจัดการเวอร์ชัน

หากคุณต้องการสลับระหว่าง Node.js หลายเวอร์ชัน คุณสามารถใช้เครื่องมือการจัดการเวอร์ชันได้

### nvm (ตัวจัดการเวอร์ชันโหนด)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### ลินุกซ์/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

หลังการติดตั้ง ให้รีโหลดการกำหนดค่าเชลล์ของคุณ:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### หน้าต่าง

บน Windows เราแนะนำให้ใช้ [nvm-windows](https://github.com/coreybutler/nvm-windows):

1. ดาวน์โหลดล่าสุด `nvm-setup.exe` จาก [การเผยแพร่ nvm-windows](https://github.com/coreybutler/nvm-windows/releases)
2. เรียกใช้โปรแกรมติดตั้ง
3. เปิดพร้อมท์คำสั่งหรือ PowerShell อีกครั้ง

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

หลังจากติดตั้ง nvm คุณสามารถใช้มันเพื่อติดตั้งและจัดการเวอร์ชัน Node.js ได้:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (ตัวจัดการโหนดด่วน)

fnm เป็นเครื่องมือจัดการเวอร์ชัน Node.js ที่เร็วและง่ายกว่าซึ่งสร้างด้วย Rust

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### ลินุกซ์/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### หน้าต่าง

ใช้ [ตัก](https://scoop.sh/) หรือ [ปีก](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## การเร่งความเร็วสำหรับผู้ใช้จีนแผ่นดินใหญ่

หากคุณอยู่ในจีนแผ่นดินใหญ่ การดาวน์โหลดแพ็คเกจ Node.js และ npm อาจช้า ต่อไปนี้เป็นวิธีแก้ปัญหาการเร่งความเร็วบางส่วน

### การใช้แหล่งกระจกเงาภายในประเทศ

#### npm กระจก Taobao (แนะนำ)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### การใช้ nrm เพื่อจัดการแหล่งที่มาของมิเรอร์

nrm (ตัวจัดการรีจิสทรี npm) ช่วยให้คุณสลับแหล่งมิเรอร์ npm ได้อย่างรวดเร็ว:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### การเร่งการดาวน์โหลดตัวติดตั้ง Node.js

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao ให้การดาวน์โหลดมิเรอร์สำหรับตัวติดตั้ง Node.js:

- **ที่อยู่ดาวน์โหลด**: https://npmmirror.com/mirrors/node/

ไปที่ที่อยู่ด้านบนและเลือกเวอร์ชันที่คุณต้องการดาวน์โหลด

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

หากการดาวน์โหลด nvm หรือเครื่องมืออื่นๆ จาก GitHub ช้า คุณสามารถใช้มิเรอร์ FastGit ได้:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::หมายเหตุ
บริการ FastGit อาจมีการเปลี่ยนแปลง โปรดดูข้อมูลอย่างเป็นทางการ
:::

  </TabItem>
</Tabs>

### กระจก Homebrew ในประเทศ

หากคุณใช้ Homebrew ในจีนแผ่นดินใหญ่ คุณสามารถกำหนดค่ามิเรอร์ในประเทศสำหรับการเร่งความเร็วได้:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### ผู้จัดการแพ็คเกจเฉพาะของจีนแผ่นดินใหญ่

#### ซีเอ็นพีเอ็ม

cnpm เป็นไคลเอนต์ npm ที่จัดทำโดยทีมงาน Taobao ที่ใช้มิเรอร์ Taobao เป็นค่าเริ่มต้น:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::คำแนะนำ
cnpm ส่วนใหญ่เข้ากันได้กับคำสั่ง npm และสามารถใช้แทนโดยตรงได้
:::

### การตรวจสอบผลการเร่งความเร็วของกระจก

หลังจากตั้งค่ากระจกแล้ว ให้ตรวจสอบโดย:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## การตรวจสอบการติดตั้ง

หลังการติดตั้ง ให้ตรวจสอบว่าติดตั้ง Node.js และ npm อย่างถูกต้อง:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::หมายเหตุ
ตรวจสอบให้แน่ใจว่าเวอร์ชัน Node.js >= 18.0 (แนะนำ Node.js 24+) หากเวอร์ชันต่ำเกินไป โปรดติดตั้งใหม่โดยใช้วิธีการข้างต้น หรือใช้เครื่องมือการจัดการเวอร์ชันเพื่อติดตั้งเวอร์ชันที่ถูกต้อง
:::

### ผลลัพธ์การตรวจสอบตัวอย่าง

ผลลัพธ์การยืนยันที่สำเร็จควรมีลักษณะดังนี้:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## การแก้ไขปัญหา

### ปัญหาเกี่ยวกับวินโดวส์

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

ถ้า `node` หรือ `npm` คำสั่งไม่พร้อมใช้งานหลังการติดตั้ง:

1. รีสตาร์ทพร้อมรับคำสั่งหรือ PowerShell
2. หากยังไม่พร้อมใช้งาน ให้เพิ่ม Node.js ไปยัง PATH ด้วยตนเอง:
   - ค้นหา "ตัวแปรสภาพแวดล้อม" และเปิด "แก้ไขตัวแปรสภาพแวดล้อมของระบบ"
   - คลิก "ตัวแปรสภาพแวดล้อม"
   - ค้นหา "เส้นทาง" ใน "ตัวแปรระบบ" และคลิก "แก้ไข"
   - เพิ่มเส้นทางการติดตั้ง Node.js (ค่าเริ่มต้น: `C:\Program Files\nodejs\`)
   - คลิก "ตกลง" เพื่อบันทึกการเปลี่ยนแปลง

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

หากคุณพบข้อผิดพลาดนโยบายการดำเนินการเมื่อใช้ nvm:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### ปัญหาเกี่ยวกับ macOS

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

หากคุณพบข้อผิดพลาดในการอนุญาตเมื่อติดตั้งแพ็คเกจ npm ทั่วโลก:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

หากไม่มีคำสั่งหลังการติดตั้ง ให้รีสตาร์ทเทอร์มินัลหรือเพิ่มลงใน PATH ด้วยตนเอง (โดยปกติจะอยู่ใน `/usr/local/bin`).

  </TabItem>
</Tabs>

### ปัญหาเกี่ยวกับลินุกซ์

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

หากคุณพบข้อผิดพลาดในการอนุญาตเมื่อติดตั้งแพ็คเกจ npm ส่วนกลาง ขอแนะนำให้ใช้เครื่องมือการจัดการเวอร์ชัน Node.js (เช่น nvm หรือ fnm) แทนตัวจัดการแพ็คเกจระบบ

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

หากเวอร์ชัน Node.js ที่ติดตั้งผ่านตัวจัดการแพ็คเกจระบบเก่าเกินไป:

1. ใช้ที่เก็บ NodeSource (แนะนำ ดูด้านบน)
2. หรือใช้เครื่องมือการจัดการเวอร์ชัน (nvm/fnm)

  </TabItem>
</Tabs>
