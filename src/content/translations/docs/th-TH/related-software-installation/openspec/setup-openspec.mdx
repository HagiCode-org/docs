# ติดตั้ง OpenSpec

OpenSpec จัดการข้อเสนอ ข้อมูลจำเพาะ และบันทึกการเปลี่ยนแปลง หน้านี้มีเป้าหมายเดียวคือ อธิบายเส้นทางการติดตั้งอย่างชัดเจน เพื่อให้คุณสามารถแนบไปกับขั้นตอนการทำงานที่มีอยู่ได้

## OpenSpec คืออะไร?

OpenSpec เป็นเครื่องมือบรรทัดคำสั่งสำหรับจัดการข้อเสนอและข้อกำหนดในงานซอฟต์แวร์ ขั้นตอนการทำงานหลักประกอบด้วย:

- **การจัดการข้อเสนอ**: สร้าง ตรวจสอบ และติดตามข้อเสนอฟีเจอร์
- **การตรวจสอบข้อกำหนด**: ตรวจสอบให้แน่ใจว่าการเปลี่ยนแปลงรหัสเป็นไปตามข้อกำหนดที่กำหนดไว้ล่วงหน้า
- **การติดตามการเปลี่ยนแปลง**: บันทึกประวัติการเปลี่ยนแปลงที่ตรวจสอบแล้วทั้งหมด
- **การทำงานร่วมกันเป็นทีม**: สนับสนุนการพัฒนาการทำงานร่วมกันแบบหลายคน

### การประยุกต์ใช้ในโครงการ HagiCode

ในโครงการ HagiCode นั้น OpenSpec ใช้สำหรับ:

1. **การจัดการการพัฒนาคุณสมบัติ**: คุณสมบัติใหม่ทั้งหมดจะต้องผ่านกระบวนการเสนอ OpenSpec
2. **การตรวจสอบโค้ด**: ตรวจสอบให้แน่ใจว่าการเปลี่ยนแปลงเป็นไปตามข้อกำหนดของโครงการ
3. **การซิงโครไนซ์เอกสาร**: รักษาความสอดคล้องระหว่างเอกสารประกอบและการนำโค้ดไปใช้
4. **การบูรณาการการควบคุมเวอร์ชัน**: การบูรณาการอย่างราบรื่นกับเวิร์กโฟลว์การควบคุมเวอร์ชัน

## ข้อกำหนดเบื้องต้น

ก่อนติดตั้ง OpenSpec ตรวจสอบให้แน่ใจว่าระบบของคุณตรงตามข้อกำหนดต่อไปนี้:

### Node.js และ npm

OpenSpec ได้รับการเผยแพร่ผ่าน NPM และต้องการสภาพแวดล้อม Node.js OpenSpec ต้องใช้ **Node.js 18.0 หรือสูงกว่า**

ตรวจสอบการติดตั้ง Node.js:

```bash
node --version
```

ตรวจสอบการติดตั้ง npm:

```bash
npm --version
```

หากไม่ได้ติดตั้งหรือเวอร์ชันเก่าเกินไป โปรดดูคำแนะนำในการติดตั้งซอฟต์แวร์ที่เกี่ยวข้องใน [คู่มือการติดตั้ง](/en/installation).

## การติดตั้ง OpenSpec

:::tip[ข้อกำหนดเวอร์ชัน]
โปรเจ็กต์นี้ต้องใช้ช่วงเวอร์ชัน **OpenSpec 1.0.0 หรือสูงกว่า** รองรับเวอร์ชัน 1.x ทั้งหมดเพื่อให้มั่นใจว่าเข้ากันได้กับเวิร์กโฟลว์ของโครงการ
:::

OpenSpec ได้รับการติดตั้งทั่วโลกผ่านทาง npm ทำให้สามารถใช้งานได้จากทุกที่ในระบบของคุณ

### ติดตั้งทั่วโลกโดยใช้ npm

<Tabs>
  <TabItem value="win" label="Windows">
    ทำงานใน PowerShell หรือพร้อมรับคำสั่ง:

```powershell
npm install -g @fission-ai/openspec@1
```

:::note[ปัญหาการอนุญาตของ Windows]
หากคุณพบข้อผิดพลาดในการอนุญาตระหว่างการติดตั้ง ให้เรียกใช้ PowerShell หรือ Command Prompt ในฐานะผู้ดูแลระบบ
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    ทำงานในเทอร์มินัล:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[ปัญหาการอนุญาตของ macOS]
หากพบเจอ `EACCES` ข้อผิดพลาดในการอนุญาต ลอง:
```bash
sudo npm install -g @fission-ai/openspec@1
```
หรือใช้ [นาโนเมตร](https://github.com/nvm-sh/nvm) เพื่อจัดการเวอร์ชัน Node.js และหลีกเลี่ยงปัญหาการอนุญาต
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    ทำงานในเทอร์มินัล:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[ปัญหาการอนุญาต Linux]
หากพบเจอ `EACCES` ข้อผิดพลาดในการอนุญาต คุณสามารถ:
1. ใช้ `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. กำหนดค่า npm เพื่อใช้ไดเร็กทอรีผู้ใช้:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### ตรวจสอบการติดตั้ง

หลังการติดตั้ง ให้ตรวจสอบว่าติดตั้ง OpenSpec อย่างถูกต้องและเวอร์ชันตรงตามข้อกำหนด:

```bash
openspec --version
```

หากติดตั้งสำเร็จ คุณจะเห็นผลลัพธ์หมายเลขเวอร์ชัน เช่น:

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::note[ช่วงเวอร์ชัน]
โปรดตรวจสอบให้แน่ใจว่าเวอร์ชัน OpenSpec ที่ติดตั้งเป็น 1.0.0 หรือสูงกว่า โครงการได้ตรวจสอบความเข้ากันได้กับเวอร์ชัน 1.1.1 แล้ว ขอแนะนำให้ใช้เวอร์ชันล่าสุดที่เข้ากันได้เพื่อรับคุณสมบัติใหม่ๆ
:::

## ปัญหาทั่วไปและการแก้ไขปัญหา

### ข้อผิดพลาดในการอนุญาต

#### ปัญหา: `EACCES` การอนุญาตถูกปฏิเสธ

พบข้อผิดพลาดในการอนุญาตเมื่อติดตั้งหรือใช้งาน OpenSpec

**แนวทางแก้ไข**:

<Tabs>
  <TabItem value="win" label="Windows">
    เรียกใช้ PowerShell หรือ Command Prompt ในฐานะผู้ดูแลระบบ:
    1. คลิกขวาที่ PowerShell หรือ Command Prompt
    2. เลือก "เรียกใช้ในฐานะผู้ดูแลระบบ"
    3. รันคำสั่งการติดตั้งอีกครั้ง
  </TabItem>
  <TabItem value="mac" label="macOS">
    ติดตั้งโดยใช้ `sudo`:

```bash
sudo npm install -g @fission-ai/openspec@1
```

หรือใช้ [นาโนเมตร](https://github.com/nvm-sh/nvm) เพื่อจัดการ Node.js และหลีกเลี่ยงปัญหาสิทธิ์ในการติดตั้งทั่วโลก
  </TabItem>
  <TabItem value="linux" label="Linux">
    โซลูชันที่ 1: ใช้ sudo
```bash
sudo npm install -g @fission-ai/openspec@1
```

โซลูชันที่ 2: กำหนดค่า npm เพื่อใช้ไดเร็กทอรีผู้ใช้
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @fission-ai/openspec@1
```
  </TabItem>
</Tabs>

### ปัญหาเครือข่าย

#### ปัญหา: การติดตั้ง npm ล้มเหลวหรือช้ามาก

ไม่สามารถดาวน์โหลด OpenSpec จากรีจิสทรี npm ได้เนื่องจากเหตุผลด้านเครือข่าย

**แนวทางแก้ไข**:

1. **ใช้กระจก npm ในประเทศ**:

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **กู้คืนแหล่งที่มาอย่างเป็นทางการ** (หากจำเป็น):

```bash
npm config set registry https://registry.npmjs.org
```

3. **ใช้ cnpm** (ไม่บังคับ):

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

:::คำแนะนำ
ผู้ใช้ในจีนแผ่นดินใหญ่แนะนำให้ใช้มิเรอร์ Taobao ซึ่งสามารถปรับปรุงความเร็วในการดาวน์โหลดได้อย่างมาก
:::

### ปัญหาความเข้ากันได้ของเวอร์ชัน

#### ปัญหา: คำสั่ง OpenSpec ทำงานผิดปกติหรือรายงานข้อผิดพลาด

เวอร์ชัน OpenSpec ที่ติดตั้งเข้ากันไม่ได้กับสภาพแวดล้อมปัจจุบัน

**แนวทางแก้ไข**:

1. **ตรวจสอบเวอร์ชัน Node.js**:

```bash
node --version
```

ตรวจสอบให้แน่ใจว่าเวอร์ชันเป็น 18.0 หรือสูงกว่า หากเวอร์ชันต่ำเกินไป โปรดอัปเกรด Node.js

2. **ตรวจสอบเวอร์ชัน OpenSpec**:

```bash
openspec --version
```

3. **ติดตั้งเวอร์ชันเฉพาะอีกครั้ง**:

```bash
npm install -g @fission-ai/openspec@1
```

4. **ล้างแคช npm** (หากยังเกิดปัญหา):

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### ไม่พบคำสั่ง

#### ปัญหา: การพิมพ์ `openspec` คำสั่งแสดง "ไม่พบคำสั่ง"

ติดตั้ง OpenSpec แล้ว แต่ระบบไม่พบคำสั่ง

**แนวทางแก้ไข**:

<Tabs>
  <TabItem value="win" label="Windows">
    1. ยืนยันเส้นทางการติดตั้งทั่วโลก npm:

```powershell
npm config get prefix
```

2. เพิ่มเส้นทางไปยังระบบ PATH:
   - เปิด "คุณสมบัติของระบบ" → "ขั้นสูง" → "ตัวแปรสภาพแวดล้อม"
   - ค้นหา `Path` ใน "ตัวแปรระบบ"
   - เพิ่มเส้นทางทั่วโลก npm (โดยปกติ `C:\Users\<username>\AppData\Roaming\npm`)
   - รีสตาร์ทหน้าต่างบรรทัดคำสั่ง
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. ยืนยันเส้นทางการติดตั้งทั่วโลก npm:

```bash
npm config get prefix
```

2. เพิ่มเส้นทางไปยัง PATH (ใน `~/.zshrc` หรือ `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. โหลดการกำหนดค่าใหม่:

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. ยืนยันเส้นทางการติดตั้งทั่วโลก npm:

```bash
npm config get prefix
```

2. เพิ่มเส้นทางไปยัง PATH (ใน `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. โหลดการกำหนดค่าใหม่:

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
