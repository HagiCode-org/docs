หน้านี้เป็นรายการการติดตั้ง AI Agent CLI ภายในแบบรวมศูนย์ `repos/docs`. ปัจจุบัน HagiCode รองรับ Agent CLI ที่ใช้งานอยู่ 11 รายการ ถัดจากแบ็กเอนด์ `PrimaryProfessions` ลำดับ: Claude Code, Codex, GitHub Copilot, OpenCode, Hermes, QoderCLI, Kiro, Kimi, Gemini, DeepAgents และ Codebuddy

:::tip[วิธีใช้งานหน้านี้]
ตรวจสอบตารางสำหรับสถานะการสนับสนุนปัจจุบันและโหมดการตั้งค่าก่อน จากนั้นดำเนินการติดตั้ง การตรวจสอบสิทธิ์ ACP การอนุญาต และเอกสารการแก้ไขปัญหาที่ผู้ขายดูแล ขอบเขตการสนับสนุนและโหมดการตั้งค่าเป็นข้อกังวลที่แยกจากกัน
:::

## อภิธานศัพท์ของรัฐ

- **รองรับในปัจจุบัน** หมายความว่า CLI อยู่ในช่วงรองรับ HagiCode ที่ใช้งานอยู่ในปัจจุบัน
- **ติดตั้งด้วยคลิกเดียว** หมายความว่า HagiCode จัดเตรียมจุดเริ่มต้นการติดตั้งที่แนะนำ CLI บางตัวยังต้องการการเข้าสู่ระบบของผู้ขายหรือขั้นตอนการตรวจสอบสิทธิ์หลังการติดตั้ง
- **การติดตั้ง / การตรวจสอบสิทธิ์ด้วยตนเอง** หมายความว่า CLI ได้รับการสนับสนุนแล้ว แต่คุณต้องดำเนินการติดตั้งในเครื่องหรือขั้นตอนการตรวจสอบสิทธิ์ให้เสร็จสิ้นก่อน HagiCode จึงสามารถตรวจพบและใช้งานได้
- **ความเข้ากันได้แบบเดิมเท่านั้น** หมายถึงค่าที่เลิกใช้แล้ว เช่น `IFlowCli` ยังคงมีอยู่สำหรับความเข้ากันได้ของข้อมูลในอดีต แต่ไม่ได้เป็นส่วนหนึ่งของช่วงการสนับสนุนที่ใช้งานอยู่ในปัจจุบัน

## แนะนำขั้นตอนต่อไป

- ต่อด้วย [ติดตั้ง Node.js](/en/related-software-installation/nodejs/installation) และ [ติดตั้ง OpenSpec](/en/related-software-installation/openspec/setup-openspec) หากคุณยังคงต้องการข้อกำหนดเบื้องต้น
- หากคุณต้องการเกตเวย์ภายในเครื่องเดียวสำหรับเครื่องมือที่เข้ากันได้กับ OpenAI หลายตัว ให้ดำเนินการต่อ [การติดตั้งและการใช้งาน OmniRoute](/en/related-software-installation/omniroute/).
- กลับไปสู่ [คู่มือการติดตั้ง](/en/installation) เพื่อเส้นทางการตั้งค่าที่กว้างขึ้น
- หากคุณมาที่นี่จากโพสต์บนบล็อกในอดีต ให้ถือว่าหน้านี้และภาพรวมการติดตั้งเป็นแหล่งที่มาของความจริงที่เขียวขจีในปัจจุบัน
