## Översikt

Monospecs är HagiCodes strategi för hantering av flera förvar. Den spårar och hanterar underlager genom `monospecs.yaml`, och det fungerar för alla projekt som behöver en plats för att koordinera flera oberoende förråd.

### Kärnkoncept

**Konfigurationsfil**

`monospecs.yaml` är kärnan i hela hanteringslösningen och definierar:
- Lista och placering av delförvar
- Git URL för varje arkiv
- Visa namn och ikon
- Integreringsbeteende med OpenSpec

**Skillnader från vanliga arkiv**

| Funktion | Vanligt förråd | Monospecs Managed Project |
|----------|-------------------|---------------------------|
| **OpenSpec Plats** | Rotkataloger `openspec/` | Huvudförvarets rotkataloger `openspec/` |
| **Ändra omfattning** | Begränsad till nuvarande förråd | Kan involvera flera underlager |
| **Renhet för underförvar** | Specifikationer blandade med kod | Specifikationer etablerade i huvudförvaret, underförråd är renare |
| **Arkivera automatisk commit** | Behöver manuella spec commits | `commit_when_archive: true` överför automatiskt specifikationer till huvudförvaret |

### Varför använda monospecs?

- **Unified Management**: Hantera alla underlager i en konfigurationsfil
- **Automationsstöd**: Tillhandahåller metadata för arkivet för skript och verktyg
- **AI-vänlighet**: Strukturerad konfiguration hjälper AI-agenten att förstå repositoryrelationer
- **OpenSpec-integration**: Sömlös integration med arbetsflödet för förändringshantering
- **Renare underförråd**: Specifikationerna hålls i huvudrepo, underförråd innehåller endast faktisk kod

### Tillämpliga scenarier

Använd Monospecs först i scenarier som dessa:

**1. Multi-Repository Collaborative Development**
När ett projekt är uppdelat i flera oberoende arkiv:
- Frontend-applikation, backend-tjänst och skrivbordsapp utvecklas separat
- Behöver koordinera versioner och beroenden mellan arkiv
- Dokumentation och hjälpprojekt skilda från huvudkoden

**2. Hjälpprojektledning**
Använd Monospecs för att hantera:
- Dokumentationsprojekt
- Officiell webbplats
- Bygg verktyg och skript
- Varje förråd har sin egen versionskontroll

**3. Cross-Repository Feature Development**
En funktion som kräver ändringar över flera arkiv:
- Behöver spåra ändringar i vilka förråd
- Överför automatiskt specifikationer för att rätta målförrådet
- Koordinera versionsuppdateringar och beroenden

## Snabbstart

### Initialisering

Följ dessa steg för att slutföra monospecs-initieringen:

#### 1. Skapa ett tomt git-förråd

Skapa en tom `git` mapp i ditt huvudlager (notera: detta är `git`, inte `repos/` - detta är monospecs-konventionen):

```bash
mkdir git
cd git
git init
cd ..
```

#### 2. Konfigurera monospecs-fil

Skapa `monospecs.yaml` konfigurationsfil i ditt huvudförvars rotkatalog:

```yaml
version: "1.0"
commit_when_archive: true

repositories:
  - path: "repos/frontend"
    url: "https://github.com/your-org/frontend.git"
    displayName: "Frontend App"
    icon: "🌐"

  - path: "repos/backend"
    url: "https://github.com/your-org/backend.git"
    displayName: "Backend Service"
    icon: "⚙️"
```

**Konfigurationsobjekt**:
| Konfigurationsobjekt | Typ | Obligatoriskt | Beskrivning |
|----------------|------|----------|-------------|
| `version` | sträng | Ja | Konfigurationsfilens version |
| `commit_when_archive` | booleskt | Nej | Om specifikationer ska överföras automatiskt vid arkivering (beslut inte sub-repokod) |
| `repositories` | array | Ja | Förvarskonfigurationslista |
| `repositories[].path` | sträng | Ja | Förvarssökväg (relativt projektrot) |
| `repositories[].url` | sträng | Ja | Git repository URL |
| `repositories[].displayName` | sträng | Nej | Förvarets visningsnamn |
| `repositories[].icon` | sträng | Nej | Förvarsikon (emoji) |

### Lägg till underförråd till huvudförrådet

Lägg till alla underlager till `repos/` katalog. Den här katalogen måste **uteslutas från huvudförvarets Git-versionskontroll** för att undvika spårning av innehåll i underförrådet:

```plaintext
# Exclude all sub-repositories
repos/
```

#### 3. Lägg till Parent Repository till HagiCode

Om du använder skrivbordsapplikationen HagiCode, öppna appen och lägg till ditt huvudarkiv som ett projekt. Klicka på knappen "Lägg till projekt" eller "+" på startskärmen, välj din huvudmapp för förvar, så kommer du att se alla underförråd och deras statusinformation.

#### 4. Klona underlager

Använd `monospecs.yaml` konfiguration för att klona alla konfigurerade underlager till `repos/` katalog:

Detta kommer att klona alla underlager baserat på URL:erna som anges i konfigurationsfilen.

## Konfigurationsfilens struktur

### Konfiguration av rotnivå

```yaml
version: "1.0"
commit_when_archive: true

repositories:
  # ... repository configurations
```

### Konfiguration av förvarsnivå

Varje underförråd kan ha:
- OpenSpec ändra mapp (`openspec/`)
- Oberoende `.git/` katalog
- Valfritt `AGENTS.md` för AI Agent-konfiguration

## Förvarshanteringsverksamhet

### Lägg till nytt arkiv

När du lägger till ett nytt underlager till `monospecs.yaml` `repositories` array:

```yaml
repositories:
  - path: "repos/new-service"
    url: "https://github.com/HagiCode-org/new-service.git"
    displayName: "New Service"
    icon: "🆕"
```

### Uppdatera förrådskonfiguration

När arkivets webbadresser eller metadata ändras:
1. Redigera `monospecs.yaml` för att uppdatera motsvarande poster
2. Kontrollera att YAML-syntaxen är korrekt
3. Om du synkroniserar ändringar, uppdatera konfigurationen för det lokala arkivet manuellt

## Stöd för konfiguration av AI Agent

### AGENTER.md

Den `AGENTS.md` filen ger förvarsspecifik vägledning för AI-agenten:

**Teknisk stackinformation**
- Rama in och bygga verktyg
- Kodkonventioner och namnpraxis
- Versionshantering och distributionskonfiguration

**Projektspecifikt beteende**
- Konfigurationsutvidgningar och speciella krav
- OpenSpec integrationsmönster
- Samordningsbehov mellan förvar

### AI-fördelar

Använder `AGENTS.md` gör det möjligt för AI-agenten att:
- Förstå förvarsrelationer och beroenden
- Generera kod och refactor över flera repositories
- Upprätthåll konsekvens i versionskontroll och konfigurationer

## OpenSpec-integration

### Hur Monospecs fungerar med OpenSpec

Oavsett om du hanterar ett monospecs-hanterat projekt eller ett vanligt arkiv, kan du använda HagiCodes OpenSpec-arbetsflöde:

**Vanligt förråd:**
- OpenSpec i root `openspec/` katalog
- Ändringar påverkar endast nuvarande arkiv

**Monospecs Managed Project:**
- OpenSpec läser `monospecs.yaml` konfiguration
- Ändringar spåras av underlager
- Specifikationer bestäms automatiskt för att korrigera mål

### Arkivbeteende

När `commit_when_archive: true`:
- Specifikationer kopplas automatiskt till huvudförvaret
- Underlagers kod är inte ansluten (förblir separat)
- Förenklar specshantering - inga manuella spec-commits behövs för underlager

### Manuella underförvarsförpliktelser

Underlager upprätthåller oberoende versionskontroll:
- Begå faktiska kodändringar till sina egna arkiv
- Specifikationer bestäms separat (från huvudarkivet vid arkivering)

## Bästa metoder

### Namnkonventioner för förvar

- Använd kebab-versaler (små bokstäver och bindestreck) för förvarsvägar
- Använd enkla, beskrivande visningsnamn på kinesiska
- Använd relevanta emojis som ikoner

### Riktlinjer för visningsnamn

- Håll det kortfattat och beskrivande
- Använd konsekvent namngivning över liknande arkiv
- Välj ikoner som representerar förvarets syfte

### Konfigurationsunderhåll

**Synkronisera konfiguration med faktisk status**
- Kontrollera regelbundet `monospecs.yaml` återspeglar den faktiska förvarsstrukturen
- Uppdatera eller ta bort poster när förråd läggs till eller tas bort
- Testa ändringar för att säkerställa att konfigurationen är giltig

**Versionskontroll för konfiguration**
- Spåra ändringar i konfigurationsfilen med git
- Dokumentera orsaker till större konfigurationsändringar
- Taggkonfigurationsfilversioner för enkel återställning

## Felsökning

### Konfigurationsfilen hittades inte

Om `monospecs.yaml` filen hittades inte i projektroten:
1. Kontrollera om filen finns på rätt plats
2. Kontrollera att du är i rätt arbetskatalog
3. Kontrollera om det finns stavfel i konfigurationsfilens namn

### YAML-syntaxfel

Vanliga YAML-syntaxproblem:
- Felaktig indrag (använd mellanslag, inte tabb)
- Saknade eller extra citattecken runt strängar
- Ogiltiga datatyper (strängar behöver citattecken, siffror behöver inte)
- Obligatoriska fält saknas (som `path` eller `url`)

### Förvar inte upptäckt

Om ett nyligen tillagt arkiv inte visas:
1. Kontrollera `path` är korrekt i `monospecs.yaml`
2. Verifiera att förvaret klonades
3. Kontrollera om förvaret finns på den förväntade platsen

### Klonfel

Om förvarskloning misslyckas:
1. Kontrollera nätverksanslutningen
2. Verifiera att Git-arkivets URL är korrekt
3. Kontrollera om det finns autentiseringsproblem
4. Granska Docker-containerstatus
5. Kontrollera tillgängligheten på diskutrymme
