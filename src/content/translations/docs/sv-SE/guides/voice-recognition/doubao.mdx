## Översikt

### Vad är Doubao Voice Input

Doubao Voice Input är HagiCodes integrerade taligenkänningsfunktion, med stöd av ByteDance Doubao Open Platform. Du talar och systemet förvandlar tal till text. HagiCode använder också aktuella projektkontexter för att hjälpa till att känna igen domänvokabulär och tekniska termer mer tillförlitligt.

### Viktiga användningsfall

I HagiCode fungerar röstinmatning bäst för:

- **Snabbinmatning av förslag**: Skapa och skicka in förslag genom att bara tala, vilket avsevärt förbättrar effektiviteten utan att skriva
- **Ge kommentarer**: Lägg till kommentarer och feedback under kodgranskning eller dokumentgranskning med röstinmatning
- **Svara på meddelanden**: Svara snabbt på meddelanden i konversationer och diskussioner utan att skriva
- **Skapa innehåll i långa former**: Generera snabbt projektdokumentation, tekniska specifikationer, mötesanteckningar och annat långformat innehåll

### Praktiska fördelar

- **Djup integration med kontextmedvetenhet**: HagiCode integreras djupt med Doubao taligenkänning, och utnyttjar automatiskt aktuellt projektkontext (som kodstruktur, teknologistack, domänterminologi) för att ge exakt domänvokabulärigenkänning, vilket avsevärt förbättrar teknisk termigenkänningsnoggrannhet
- **Gratis provperiod**: Doubao-plattformen ger 20 timmars gratis igenkänningstid för nya användare att uppleva röstinmatning utan hinder
- **Extremt snabbt**: Realtidsigenkänning, se text medan du talar utan att vänta
- **Hög noggrannhet**: Drivs av Doubaos avancerade taligenkänningsmodeller kombinerat med projektkontext för exakta och pålitliga resultat
- **Sömlös integration**: Direkt integrerad i HagiCodes meddelandeinmatningsruta, inget behov av att byta applikation
- **Lätt att använda**: Klicka bara på mikrofonen för att starta, intuitivt och enkelt

## Snabbstart

### Förutsättningar

Innan du använder Doubao Voice Input måste du:

1. **Få ett Doubao Open Platform-konto**
   - Besök [Doubao Speech Recognition Console](https://console.volcengine.com/speech/service/10038)
   - Registrera dig eller logga in på ditt konto

2. **Skapa en applikation och få inloggningsuppgifter**
   - Skapa en taligenkänningsapplikation på plattformen
   - Skaffa din `APP ID` och `Access Token`

3. **Se till nätverksanslutning**
   - Taligenkänningstjänst kräver nätverksanslutning
   - Se till att din enhet kan komma åt Doubao API-tjänsten

### Grundläggande användningsflöde

Här är de grundläggande stegen för att använda Doubao Voice Input:

1. Få `APP ID` och `Access Token`
2. Konfigurera röstigenkänning i produkten
3. Testa API-nyckeln
4. Hitta röstinmatningsrutan
5. Klicka på mikrofonknappen
6. Ge mikrofontillstånd
7. Börja spela in
8. Se igenkänningsresultat i realtid
9. Klicka för att stoppa inspelningen
10. Bekräfta att texten har infogats

## Tekniska krav

### HTTP-miljö

Doubao Voice Input kräver körning i en HTTP/HTTPS-miljö.

| Miljö | Supportstatus | Beskrivning |
|-------------|----------------|-------------|
| **lokal värd** | ✓ | Lokal utvecklingsmiljö (http://localhost eller http://127.0.0.1) |
| **HTTPS Remote Server** | ✓ | Webbapplikationer distribuerade till offentliga nätverk med HTTPS |
| **HTTP-fjärrserver** | ✗ | Webbapplikationer distribuerade till offentliga nätverk med HTTP |
| **fil://protokoll** | ✗ | Direktöppning av HTML-filer stöds inte |

::: försiktighet
Doubao Voice Input har följande begränsningar:
1. **HTTPS krävs för extern distribution**: På grund av webbläsarsäkerhetsbegränsningar kräver mikrofonåtkomst HTTPS vid åtkomst från offentliga nätverk
2. **Lokala adresser obegränsade**: Både HTTP och HTTPS stöds vid användning `localhost` eller `127.0.0.1`
3. **file://-protokollet stöds inte**: The `file://` protokollmiljön kan inte använda mikrofon- och WebSocket-anslutningar
:::

### Desktopversion

HagiCodes Desktop-version har en inbyggd lokal HTTP-miljö och har fullt stöd för röstinmatning. Du kan använda taligenkänning direkt utan ytterligare konfiguration.

### Värdläge

Lokalt värdläge stöder röstinmatning:
- Både HTTP och HTTPS stöds vid användning `localhost` eller `127.0.0.1`
- HTTPS krävs när det distribueras till offentliga nätverk

## Konfigurationssteg

### Skaffa APP-ID

1. Besök [Doubao Speech Recognition Console](https://console.volcengine.com/speech/service/10038)
2. Logga in eller registrera ett konto
3. Gå till konsolen och skapa en ny taligenkänningsapplikation
4. Hitta och kopiera på sidan med applikationsdetaljer `APP ID`

### Få åtkomsttoken

1. I Doubao Open Platform-konsolen
2. Gå till din taligenkänningsapplikation
3. Hitta API-nyckelhanteringsområdet
4. Generera eller kopiera `Access Token`

:::obs
Vänligen behåll din `Access Token` säkert och dela det inte med andra. Om du behöver byta ut den kan du återskapa den på Doubao Open Platform.
:::

### Konfigurera i produkt

1. Öppna HagiCode-applikationen
2. Gå till **Inställningar** → **Inställningar för röstigenkänning**
3. Fyll i följande information i konfigurationsformuläret:
   - **Leverantör**: Välj `doubao` (Doubao)
   - **APP-ID**: Klistra in APP-ID:t du fick från Doubao-plattformen
   - **Access Token**: Klistra in Access Token du fick från Doubao-plattformen
4. (Valfritt) Justera andra konfigurationsparametrar efter behov
5. Klicka på knappen **Testa API-nyckel** för att verifiera konfigurationen
6. Efter lyckad verifiering sparas konfigurationen automatiskt i webbläsarens lokala lagringsutrymme

### Testa API-nyckel

Efter konfigurationen rekommenderar vi att du testar om API-nyckeln är giltig:

1. Klicka på knappen **Testa API-nyckel**
2. Systemet anropar testgränssnittet för att verifiera din konfiguration
3. Om konfigurationen är korrekt, kommer ett framgångsmeddelande att visas
4. Om konfigurationen är felaktig kommer ett felmeddelande att visas. Kontrollera:
   - Om APP-ID och åtkomsttoken är korrekta
   - Om nätverksanslutningen är normal
   - Om åtkomsttoken har löpt ut

:::tips
Om testet misslyckas, kontrollera först om nätverksanslutningen är normal och bekräfta sedan att API-nyckeln från Doubao Open Platform är giltig och inte har gått ut.
:::

## Användning

### Hitta plats för röstinmatning

I HagiCode stöder textinmatningsrutor med en mikrofonikon röstinmatning. Dessa komponenter kallas vanligtvis `VoiceTextArea`.

Vanliga röstinmatningsplatser inkluderar:
- Inmatningsrutor för meddelanden
- Textredigeringsområden
- Olika formulärfält som kräver textinmatning

### Starta röstigenkänning

1. Hitta inmatningsrutan med en mikrofonikon
2. Klicka på mikrofonknappen
3. Webbläsaren kommer att begära mikrofontillstånd
4. Klicka på **Tillåt** för att auktorisera mikrofonåtkomst

:::obs
Om du tidigare har auktoriserat mikrofonen kan det hända att webbläsaren inte begär tillstånd igen. För att kontrollera eller ändra behörighetsinställningar kan du se dem i låsikonen till vänster om webbläsarens adressfält.
:::

### Inspelningsstatus Beskrivning

När auktoriseringen har lyckats startar röstigenkänningen automatiskt och du kan se:

- **Vågformsanimering**: Dynamisk ljudvågsvågform visas inuti mikrofonknappen, vilket indikerar att inspelning pågår
- **Visning av varaktighet**: Aktuell inspelningslängd visas under knappen
- **Realtidsigenkänning**: Identifierad text visas tillfälligt vid markörens position

### Visning av resultat för igenkänning i realtid

Under inspelning konverterar taligenkänningsmotorn ditt tal till text i realtid:

- Identifierad text kommer tillfälligt att visas i grått i inmatningsrutan
- Igenkänningsresultat uppdateras kontinuerligt medan du talar
- Stöder Mandarin kinesisk igenkänning med hög noggrannhet

:::tips
För bättre igenkänningsresultat, rekommenderas att:
- Använd i en lugn miljö
- Tala tydligt och i måttlig takt
- Undvik störningar av bakgrundsljud
:::

### Stoppa igenkänning

För att stoppa röstigenkänning kan du:

1. **Klicka på mikrofonknappen**: Klicka på knappen igen för att stoppa inspelningen
2. **Klicka på inmatningsrutan**: Klicka på andra områden i inmatningsrutan kommer också att stoppa inspelningen

Efter stopp kommer den slutgiltiga igenkända texten att infogas formellt i inmatningsrutan och du kan fortsätta redigera eller skicka.
