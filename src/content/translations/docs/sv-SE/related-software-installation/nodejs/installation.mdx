Den här guiden täcker installationen av Node.js på Windows, macOS och Linux. Node.js är ett nyckelberoende för HagiCode lokala verktyg, inklusive OpenSpec, frontend-verktyg och flera CLI:er för utvecklare.

:::tip[Versionskrav]
HagiCode rekommenderar **Node.js 24+** (minst stödd Node.js &gt;= 18.0). I de flesta fall installerar du den senaste LTS-versionen och verifierar sedan att din version uppfyller kravet.
:::

## Installerar Node.js

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Metod ett: officiellt installationsprogram (rekommenderas)

1. Besök [Node.js officiella webbplats](https://nodejs.org/)
2. Ladda ner **LTS (Long Term Support)** version Windows installationsprogram (`.msi` fil)
3. Dubbelklicka för att köra installationsprogrammet
4. Följ instruktionerna i installationsguiden för att slutföra installationen:
   - Acceptera licensavtalet
   - Välj installationsväg (standard är bra)
   - Se till att "Installera de nödvändiga verktygen automatiskt" är markerad
   - Klicka på "Installera" för att starta installationen
5. När installationen är klar, klicka på "Slutför"

### Metod två: Använd vinget

Öppna PowerShell eller kommandotolken och kör:

```powershell
winget install OpenJS.NodeJS.LTS
```

Efter installationen öppnar du terminalen igen för att ändringarna ska träda i kraft.

  </TabItem>
  <TabItem value="macos" label="macOS">

### Metod ett: officiellt installationsprogram (rekommenderas)

1. Besök [Node.js officiella webbplats](https://nodejs.org/)
2. Ladda ner **LTS (Long Term Support)** version macOS installationsprogram (`.pkg` fil)
3. Dubbelklicka för att köra installationsprogrammet
4. Följ instruktionerna i installationsguiden för att slutföra installationen:
   - Klicka på "Fortsätt"
   - Acceptera licensavtalet
   - Välj installationsskiva
   - Klicka på "Installera" och ange administratörslösenord
5. När installationen är klar, klicka på "Stäng"

### Metod två: Använda Homebrew

Om du har [Hembryggt](https://brew.sh/) installerat, kan du installera Node.js via terminal:

```bash
brew install node
```

:::tips
Homebrew lägger automatiskt till Node.js till din PATH. Om kommandot inte är tillgängligt, starta om terminalen eller kör:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

  </TabItem>
  <TabItem value="ubuntu" label="Ubuntu/Debian">

### Använda NodeSource Repository (rekommenderas)

Det här är det bästa sättet att få den senaste versionen av Node.js:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Använder apt Package Manager

Om du inte behöver den senaste versionen kan du använda standardförvaret för Ubuntu:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::obs
Node.js-versionen i standardförvaret kan vara äldre. Efter installationen, kontrollera versionen för att säkerställa att den uppfyller Node.js &gt;= 18.0-kraven (Node.js 24+ rekommenderas).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### Använda NodeSource Repository (rekommenderas)

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### Använder yum Package Manager

```bash
sudo yum install -y nodejs npm
```

  </TabItem>
  <TabItem value="fedora" label="Fedora">

### Använder dnf Package Manager

```bash
sudo dnf install -y nodejs npm
```

:::tips
Fedoras programvaruförråd innehåller vanligtvis nyare Node.js-versioner och kan installeras direkt via dnf.
:::

  </TabItem>
</Tabs>

## Använda versionshanteringsverktyg

Om du behöver växla mellan flera Node.js-versioner kan du använda versionshanteringsverktyg.

### nvm (Node Version Manager)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

Efter installationen laddar du om skalkonfigurationen:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

På Windows rekommenderar vi att du använder [nvm-fönster](https://github.com/coreybutler/nvm-windows):

1. Ladda ner det senaste `nvm-setup.exe` från [nvm-windows släpper](https://github.com/coreybutler/nvm-windows/releases)
2. Kör installationsprogrammet
3. Öppna kommandotolken eller PowerShell igen

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

Efter installation av nvm kan du använda den för att installera och hantera Node.js-versioner:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (Fast Node Manager)

fnm är ett snabbare och enklare versionshanteringsverktyg för Node.js byggt med Rust.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

Använd [skopa](https://scoop.sh/) eller [vinget](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Acceleration för användare av fastlandet i Kina

Om du är i Kina kan nedladdningen av Node.js- och npm-paket gå långsamt. Här är några accelerationslösningar.

### Använda inhemska spegelkällor

#### npm Taobao Mirror (rekommenderas)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### Använda nrm för att hantera spegelkällor

nrm (npm registerhanterare) hjälper dig att snabbt byta npm-spegelkällor:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Accelererar nedladdningar av Node.js Installer

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao tillhandahåller spegelnedladdningar för Node.js-installatörer:

- **Nedladdningsadress**: https://npmmirror.com/mirrors/node/

Besök ovanstående adress och välj den version du behöver ladda ner.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

Om nedladdningen av nvm eller andra verktyg från GitHub går långsamt kan du använda FastGit-spegeln:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::obs
FastGit-tjänsten kan ändras, se officiell information.
:::

  </TabItem>
</Tabs>

### Inhemska homebrew speglar

Om du använder Homebrew i Kina kan du konfigurera inhemska speglar för acceleration:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Ansvariga för specifika paket för fastlandet Kina

#### cnpm

cnpm är en npm-klient som tillhandahålls av Taobao-teamet som använder Taobao-spegeln som standard:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::tips
cnpm är mestadels kompatibel med npm-kommandon och kan användas som en direkt ersättning.
:::

### Verifierar spegelaccelerationseffekt

Efter att ha ställt in speglar, verifiera genom att:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Verifierar installationen

Efter installationen kontrollerar du att Node.js och npm är korrekt installerade:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::obs
Se till att Node.js version &gt;= 18.0 (Node.js 24+ rekommenderas). Om versionen är för låg, installera om med ovanstående metoder eller använd versionshanteringsverktyg för att installera rätt version.
:::

### Exempel på verifiering

Lyckad verifiering ska se ut så här:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Felsökning

### Windows-problem

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Om `node` eller `npm` kommandon är inte tillgängliga efter installationen:

1. Starta om kommandotolken eller PowerShell
2. Om det fortfarande inte är tillgängligt, lägg till Node.js manuellt till PATH:
   - Sök efter "Environment Variables" och öppna "Redigera systemmiljövariabler"
   - Klicka på "Miljövariabler"
   - Hitta "Sökväg" i "Systemvariabler" och klicka på "Redigera"
   - Lägg till installationssökväg för Node.js (standard: `C:\Program Files\nodejs\`)
   - Klicka på "OK" för att spara ändringarna

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Om du stöter på exekveringspolicyfel när du använder nvm:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### macOS-problem

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Om du stöter på behörighetsfel när du installerar globala npm-paket:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Om kommandon inte är tillgängliga efter installationen, starta om terminalen eller lägg manuellt till PATH (vanligtvis i `/usr/local/bin`).

  </TabItem>
</Tabs>

### Linux-problem

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Om du stöter på behörighetsfel när du installerar globala npm-paket, rekommenderas att du använder Node.js versionshanteringsverktyg (som nvm eller fnm) istället för systempakethanterare.

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Om Node.js-versionen installerad via systempakethanteraren är för gammal:

1. Använd NodeSource repository (rekommenderas, se ovan)
2. Eller använd versionshanteringsverktyg (nvm/fnm)

  </TabItem>
</Tabs>

### Kontrollera miljö

Om verifieringen misslyckas, kontrollera följande:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
