# Installera OpenSpec

OpenSpec hanterar förslag, specifikationer och ändringsposter. Den här sidan håller sig till ett mål: förklara installationsvägen tydligt så att du kan koppla den till ditt befintliga arbetsflöde.

## Vad är OpenSpec?

OpenSpec är ett kommandoradsverktyg för att hantera förslag och specifikationer i mjukvaruarbete. Det centrala arbetsflödet inkluderar:

- **Proposal Management**: Skapa, granska och spåra funktionsförslag
- **Specifikationsvalidering**: Se till att kodändringar överensstämmer med fördefinierade specifikationer
- **Ändringsspårning**: Registrera all granskad ändringshistorik
- **Team Collaboration**: Stöd samarbetsutveckling med flera personer

### Applikation i HagiCode Projects

I HagiCode-projekt används OpenSpec för:

1. **Funktionsutvecklingshantering**: Alla nya funktioner måste gå igenom OpenSpec-förslagsprocessen
2. **Kodgranskning**: Se till att ändringar överensstämmer med projektspecifikationerna
3. **Dokumentationssynkronisering**: Upprätthåll överensstämmelse mellan dokumentation och kodimplementering
4. **Integration med versionskontroll**: Sömlös integration med arbetsflöden för versionskontroll

## Förutsättningar

Innan du installerar OpenSpec, se till att ditt system uppfyller följande krav:

### Node.js och npm

OpenSpec distribueras via npm och kräver en Node.js-miljö. OpenSpec kräver **Node.js 18.0 eller högre**.

Verifiera installationen av Node.js:

```bash
node --version
```

Verifiera npm-installation:

```bash
npm --version
```

Om den inte är installerad eller om versionen är för gammal, vänligen se de relaterade installationsanvisningarna för programvaran i [Installationsguide](/en/installation).

## Installerar OpenSpec

:::tip[Versionskrav]
Detta projekt kräver att du använder **OpenSpec 1.0.0 eller högre** versionsintervall. Alla 1.x-versioner stöds för att säkerställa kompatibilitet med projektets arbetsflöde.
:::

OpenSpec installeras globalt via npm, vilket gör det tillgängligt var som helst på ditt system.

### Installera globalt med npm

<Tabs>
  <TabItem value="win" label="Windows">
    Kör i PowerShell eller Kommandotolken:

```powershell
npm install -g @fission-ai/openspec@1
```

:::note[Windows-behörighetsproblem]
Om du stöter på behörighetsfel under installationen, kör PowerShell eller Kommandotolken som administratör.
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    Kör i terminal:

```bash
npm install -g @fission-ai/openspec@1
```

:::not [problem med macOS-behörighet]
Om du stöter på `EACCES` behörighetsfel, försök:
```bash
sudo npm install -g @fission-ai/openspec@1
```
Eller använd [nvm](https://github.com/nvm-sh/nvm) för att hantera Node.js-versioner och undvika behörighetsproblem.
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    Kör i terminal:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[Linux-tillståndsproblem]
Om du stöter på `EACCES` behörighetsfel kan du:
1. Använd `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. Konfigurera npm för att använda användarkatalogen:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### Verifiera installationen

Efter installationen kontrollerar du att OpenSpec är korrekt installerat och att versionen uppfyller kraven:

```bash
openspec --version
```

Om installationen lyckades kommer du att se ett versionsnummer, till exempel:

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::note[Versionsintervall]
Se till att den installerade OpenSpec-versionen är 1.0.0 eller högre. Projektet har verifierat kompatibilitet med version 1.1.1. Det rekommenderas att använda den senaste kompatibla versionen för att få nya funktioner.
:::

## Vanliga problem och felsökning

### Behörighetsfel

#### Problem: `EACCES` Tillstånd nekad

Stöter på behörighetsfel vid installation eller körning av OpenSpec.

**Lösningar**:

<Tabs>
  <TabItem value="win" label="Windows">
    Kör PowerShell eller Kommandotolken som administratör:
    1. Högerklicka på PowerShell eller Kommandotolken
    2. Välj "Kör som administratör"
    3. Kör installationskommandot igen
  </TabItem>
  <TabItem value="mac" label="macOS">
    Installera med hjälp av `sudo`:

```bash
sudo npm install -g @fission-ai/openspec@1
```

Eller använd [nvm](https://github.com/nvm-sh/nvm) för att hantera Node.js och undvika globala installationsbehörighetsproblem.
  </TabItem>
  <TabItem value="linux" label="Linux">
    Lösning 1: Använd sudo
```bash
sudo npm install -g @fission-ai/openspec@1
```

Lösning 2: Konfigurera npm för att använda användarkatalogen
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @fission-ai/openspec@1
```
  </TabItem>
</Tabs>

### Nätverksproblem

#### Problem: npm Installationen misslyckas eller är mycket långsam

Det gick inte att ladda ner OpenSpec från npm-registret på grund av nätverksskäl.

**Lösningar**:

1. **Använd inhemsk npm-spegel**:

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **Återställ officiell källa** (om det behövs):

```bash
npm config set registry https://registry.npmjs.org
```

3. **Använd cnpm** (valfritt):

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

:::tips
Användare på det kinesiska fastlandet rekommenderas att använda Taobao-spegeln, som avsevärt kan förbättra nedladdningshastigheten.
:::

### Versionskompatibilitetsproblem

#### Problem: OpenSpec-kommandon körs onormalt eller rapportera fel

Den installerade OpenSpec-versionen är inkompatibel med den aktuella miljön.

**Lösningar**:

1. **Kontrollera Node.js-version**:

```bash
node --version
```

Se till att versionen är 18.0 eller högre. Om versionen är för låg, uppgradera Node.js.

2. **Kontrollera OpenSpec-version**:

```bash
openspec --version
```

3. **Installera om specifik version**:

```bash
npm install -g @fission-ai/openspec@1
```

4. **Rensa npm-cache** (om problemet kvarstår):

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### Kommandot hittades inte

#### Problem: Att skriva `openspec` Kommandot visar "Kommandot hittades inte"

OpenSpec är installerat men systemet kan inte hitta kommandot.

**Lösningar**:

<Tabs>
  <TabItem value="win" label="Windows">
    1. Bekräfta npm global installationssökväg:

```powershell
npm config get prefix
```

2. Lägg till sökväg till systemet PATH:
   - Öppna "Systemegenskaper" → "Avancerat" → "Miljövariabler"
   - Hitta `Path` i "Systemvariabler"
   - Lägg till npm global sökväg (vanligtvis `C:\Users\<username>\AppData\Roaming\npm`)
   - Starta om kommandoradsfönstret
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. Bekräfta npm global installationssökväg:

```bash
npm config get prefix
```

2. Lägg till sökväg till PATH (in `~/.zshrc` eller `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Ladda om konfiguration:

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. Bekräfta npm global installationssökväg:

```bash
npm config get prefix
```

2. Lägg till sökväg till PATH (in `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Ladda om konfiguration:

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
