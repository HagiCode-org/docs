Den här sidan är för förstagångsanvändare av HagiCode som vill ha den kortaste installationsvägen för OmniRoute: starta den lokala tjänsten, verifiera att `http://localhost:20128/v1` fungerar och bekräfta att både Dashboard och API är tillgängliga.

:::tip[Rekommenderad sökväg]
Denna första version håller den mest stabila vägen först: `npm install -g omniroute`, `omniroute`, Instrumentpanel kl `http://localhost:20128`, och den OpenAI-kompatibla baswebbadressen på `http://localhost:20128/v1`. Docker, VPS och mer avancerade distributionslägen stannar i den valfria sektionen.
:::

## Förutsättningar

- Node.js är redan installerat. Komplett [Installerar Node.js](/en/related-software-installation/nodejs/installation) först om det behövs.
- Lokal hamn `20128` är tillgänglig och inte redan upptagen.
- Du har minst ett uppströmsmodellleverantörskonto som du planerar att dirigera genom OmniRoute.
- Innan du integrerar HagiCode, en CLI eller en SDK, håll dessa två adresser åtskilda:
  - Instrumentpanel: `http://localhost:20128`
  - OpenAI-kompatibel basadress: `http://localhost:20128/v1`

## Installera och kör lokalt

Börja med den rekommenderade globala npm-vägen:

```bash
npm install -g omniroute
omniroute
```

När tjänsten startar, validera den i denna ordning:

1. Öppna `http://localhost:20128` i en webbläsare och bekräfta att Dashboard läses in.
2. Gå till `Endpoints` sida i instrumentpanelen och skapa eller kopiera en API-nyckel.
3. Notera den grundläggande webbadressen som du kommer att lämna till verktyg senare: `http://localhost:20128/v1`.

:::note[Varför börja med instrumentpanelen]
För den första installationen är Dashboard det snabbaste sättet att bekräfta att OmniRoute körs, att dina leverantörer är anslutna och att det finns en API-nyckel. Om ett senare klientanrop misslyckas går det vanligtvis snabbare att kontrollera Dashboard först än att felsöka SDK-kod omedelbart.
:::

## Validera den pågående tjänsten

### Validering 1: bekräfta att modellkatalogen är läsbar

Sätt nyckeln från `Endpoints` sida till en miljövariabel:

```bash
export OMNIROUTE_API_KEY="replace-with-the-key-you-copied-from-the-Endpoints-page"
```

Begär sedan `/v1/models`:

```bash
curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}"
```

Om du får tillbaka en modelllista fungerar den OpenAI-kompatibla slutpunkten och autentiseringsflödet.

### Validering 2: skicka en minimal chattförfrågan

```bash
curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "replace-with-a-model-id-from-v1-models",
    "messages": [
      { "role": "user", "content": "Reply with ok." }
    ]
  }'
```

Målet här är inte snabb kvalitet. Det är för att bevisa att din valda modell redan kan dirigeras genom OmniRoute framgångsrikt.
