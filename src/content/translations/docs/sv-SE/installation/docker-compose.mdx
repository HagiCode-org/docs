## Förutsättningar

Innan du börjar, se till att ditt system har Docker och Docker Compose installerat.

### Installera Docker

#### Windows

Ladda ner och installera [Docker Desktop för Windows](https://www.docker.com/products/docker-desktop/)

Efter installationen, se till att Docker Desktop körs.

#### macOS

Ladda ner och installera [Docker Desktop för Mac](https://www.docker.com/products/docker-desktop/)

Efter installationen, se till att Docker Desktop körs.

#### Linux

Installera Docker med din pakethanterare:

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

Efter installationen, starta Docker-tjänsten:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Verifiera installationen

Efter installationen kör du följande kommandon för att kontrollera att Docker och Docker Compose är korrekt installerade:

```bash
docker --version
docker compose version
```

## Snabbstart

### 1. Skapa Docker Compose-konfigurationsfil

:::tip[Använd Configuration Generator]
**Rekommenderar att använda [Docker Compose Builder (fristående webbplats)](https://builder.hagicode.com/)** för att generera din konfigurationsfil.

Generatorn stöder:
- Flera API-leverantörsalternativ (Zhipu AI, Anthropic Official, Custom API)
- Automatisk konfiguration av API URL och Token
- Anpassade portar, containernamn och annan grundläggande konfiguration
- Konfigurera beständiga data och sökvägar till arbetskataloger
- Konfigurera automatiskt skillnader i Windows/Linux-plattformar
- Hantera problem med filbehörighet intelligent
- **Val av bildkälla**: Välj lämplig bildkälla baserat på din nätverksmiljö

[🚀 Använd generatorn omedelbart →](https://builder.hagicode.com/)
:::

1. Öppna [Docker Compose Builder (fristående webbplats)](https://builder.hagicode.com/)
2. Fyll i konfigurationen efter dina behov:
   - Välj API-leverantör (Zhipu AI, Anthropic Official eller Custom API)
   - Konfigurera portar, databas, arbetskatalog och andra alternativ
   - **Välj bildkälla**: Välj lämplig bildkälla baserat på din nätverksmiljö
3. Klicka på generera-knappen för att få `docker-compose.yml` konfiguration
4. Spara den genererade konfigurationen som en `docker-compose.yml` fil
5. Om det behövs, skapa en `.env` fil för att konfigurera känslig information

#### Val av bildkälla

Generatorn stöder tre bildkällor:

**Docker Hub**
- **Bildadress**: `newbe36524/hagicode:{tag}`
- **Tillämpliga scenarier**: Lämplig för användare med Docker Hub-spegelacceleration
- **Fördelar**: Officiell bildkälla, snabba uppdateringar, stabil åtkomst
- **Anmärkningar**: Spegelacceleration kan behöva konfigureras i vissa regioner

**Azure Container Registry (ACR)**
- **Bildadress**: `hagicode.azurecr.io/hagicode:{tag}`
- **Tillämpliga scenarier**: Lämplig för användare som inte kan komma åt Docker Hub lokalt
- **Fördelar**: Ger alternativ bildkälla, löser problem med nätverksåtkomst
- **Anmärkningar**: Bilder förblir synkroniserade med Docker Hub, men kan ha korta förseningar

**Alibaba Cloud Container Registry (ACR)**
- **Bildadress**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **Tillämpliga scenarier**: Lämplig för inhemska användare, stabil och snabb nätverksåtkomst
- **Fördelar**: Alibaba Cloud inhemska noder, snabb åtkomsthastighet, hög stabilitet
- **Anmärkningar**: Alibaba ACR-bilder förblir synkroniserade med Docker Hub, vanligtvis inom 30 minuter

:::tips[Rekommendationer för val av bildkälla]
- **Inhemska användare rekommenderar Alibaba Cloud ACR**: Om du är en inhemsk användare, rekommenderas det att du prioriterar att använda Alibaba Cloud ACR-bildkälla för snabb och stabil åtkomst
- **Alternativ Docker Hub**: Om din nätverksmiljö kan komma åt Docker Hub kan du också välja Docker Hub som bildkälla
- **Alternativ Azure ACR**: Välj endast Azure ACR-bildkälla om andra bildkällor har åtkomstsvårigheter
- **Bildsynkronisering**: Alibaba Cloud ACR-bilder förblir synkroniserade med Docker Hub, vanligtvis inom 30 minuter
- **Versionskonsistens**: Bildtaggar är konsekventa i alla tre bildkällorna, du behöver inte oroa dig för versionsproblem
:::

### 2. Starta tjänster

Utför följande kommando i katalogen där du `docker-compose.yml` ligger:

```bash
docker compose up -d
```

Detta kommando kommer att:
- Ladda ner och starta Hagicode-behållarna som definieras av den genererade mallen
- Skapa och konfigurera nätverket
- Initiera standarddatakatalogen och körtidsberoenden

### 3. Verifiera tjänstens status

Kontrollera om behållare körs:

```bash
docker compose ps
```

Du bör se tjänsterna definierade i din genererade mall i statusen "kör".

### Visa serviceloggar

Så här visar du alla serviceloggar:

```bash
# View all service logs
docker compose logs

# View Hagicode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## Konfigurationsinstruktioner

Om du behöver ändra konfigurationen (som att byta portar, byta API-leverantör, etc.), kan du:

1. Återskapa ny konfiguration med [Docker Compose Builder (fristående webbplats)](https://builder.hagicode.com/)
2. Redigera manuellt `docker-compose.yml` och `.env` filer
3. Starta om tjänster för att få konfigurationen att träda i kraft:

```bash
docker compose restart
```

Efter framgångsrik start av tjänsten, åtkomst via din webbläsare:
