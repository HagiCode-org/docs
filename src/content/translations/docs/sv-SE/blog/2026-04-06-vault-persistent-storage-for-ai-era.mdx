## Hur man återskapar projekt i AI-eran: Vault, ett Cross-Project Persistent Storage System

> Hur kan vi hjälpa AI-assistenter att bättre förstå våra inlärningsresurser i en tid med AI-stödd utveckling? HagiCode-projektet byggde Vault-systemet som ett enhetligt abstraktionslager för kunskapslagring som AI kan förstå, vilket avsevärt förbättrar effektiviteten av lärande genom projektreproduktion.

## Bakgrund

Under AI-eran förändras sättet som utvecklare lär sig nya tekniker och arkitekturer djupt. "Reproducera projekt" - det vill säga att djupt studera och lära av koden, arkitekturen och designmönstren för utmärkta projekt med öppen källkod - har blivit ett effektivt sätt att lära sig. Jämfört med traditionella metoder som att läsa böcker eller titta på videor, direkt läsning och körning av högkvalitativa projekt med öppen källkod hjälper dig att förstå verkliga tekniker mycket snabbare.

Ändå kommer denna inlärningsmetod med en hel del utmaningar.

Läromedel är för spritt. Dina anteckningar kan finnas i Obsidian, kodförråd kan vara utspridda över olika mappar och din AI-assistents konversationshistorik blir ännu en isolerad dataö. När du vill att AI ska hjälpa till att analysera ett projekt måste du manuellt kopiera kodavsnitt och organisera sammanhanget, vilket är ganska tråkigt.

Vad som är ännu mer besvärligt är det brutna sammanhanget. AI-assistenter kan inte direkt komma åt dina lokala lärresurser, så du måste ge bakgrundsinformation igen i varje konversation. Utöver det uppdateras reproducerade kodlager snabbt, manuell synkronisering är felbenägen och kunskap är svår att dela mellan flera lärandeprojekt.

I grunden kommer alla dessa problem från "dataöar". Om det fanns ett enhetligt lager för lagringsabstraktion som gjorde det möjligt för AI-assistenter att förstå och komma åt alla dina lärresurser, skulle problemet lösas på ett snyggt sätt.

## Vault system designfilosofi

Kärnidén med Vault-systemet är enkel: skapa ett enhetligt abstraktionslager för kunskapslagring som AI kan förstå. Ur ett implementeringsperspektiv har systemet flera nyckelegenskaper.

### Flertypsstöd

Systemet stöder fyra valvtyper, som var och en motsvarar ett annat användningsscenario:

```typescript
// folder: general-purpose folder type
export const DEFAULT_VAULT_TYPE = 'folder';

// coderef: a type specifically for reproduced code projects
export const CODEREF_VAULT_TYPE = 'coderef';

// obsidian: integrated with Obsidian note-taking software
export const OBSIDIAN_VAULT_TYPE = 'obsidian';

// system-managed: vault automatically managed by the system
export const SYSTEM_MANAGED_VAULT_TYPE = 'system-managed';
```

Bland dem är **coderef**-typen den vanligaste i HagiCode. Den är speciellt designad för reproducerade kodprojekt, och tillhandahåller en standardiserad katalogstruktur och AI-läsbara metadatabeskrivningar.

### Ihållande lagringsmekanism

Vault-registret lagras konstant i JSON-format, vilket säkerställer att konfigurationen förblir tillgänglig efter att programmet startar om:

```csharp
public class VaultRegistryStore : IVaultRegistryStore
{
    private readonly string _registryFilePath;

    public VaultRegistryStore(IConfiguration configuration, ILogger<VaultRegistryStore> logger)
    {
        var dataDir = configuration["DataDir"] ?? "./data";
        var absoluteDataDir = Path.IsPathRooted(dataDir)
            ? dataDir
            : Path.GetFullPath(Path.Combine(Directory.GetCurrentDirectory(), dataDir));

        _registryFilePath = Path.Combine(absoluteDataDir, "personal-data", "vaults", "registry.json");
    }
}
```

Fördelen med denna design är att den är enkel och pålitlig. JSON är läsbar för människor, vilket gör felsökning och manuell redigering enklare; filsystemlagring undviker komplexiteten hos en databas och minskar systemberoendena. När allt kommer omkring, ibland är det enklaste alternativet verkligen det bästa.

### AI-kontextintegration

Det viktigaste är att systemet automatiskt kan injicera valvinformation i samband med AI-förslag:

```typescript
export function buildTargetVaultsText(
  vaults: VaultForText[],
  template: VaultPromptTemplate = DEFAULT_VAULT_PROMPT_TEMPLATE,
): string {
  const readOnlyVaults = vaults.filter((vault) => vault.accessType === 'read');
  const editableVaults = vaults.filter((vault) => vault.accessType === 'write');

  if (readOnlyVaults.length === 0 && editableVaults.length === 0) {
    return '';
  }

  const sections = [
    buildVaultSection(readOnlyVaults, template.reference),
    buildVaultSection(editableVaults, template.editable),
  ].filter(Boolean);

  return `\n\n### ${template.heading}\n\n${sections.join('\n')}`;
}
```

Detta möjliggör en viktig funktion: AI-assistenter kan automatiskt förstå de tillgängliga lärresurserna utan att användarna manuellt tillhandahåller sammanhang. Man kan säga att det räknas som ett slags tyst förståelse.

## Den standardiserade strukturen för CodeRef Vault

För valv av coderef-typ tillhandahåller HagiCode en standardiserad katalogstruktur:

```
my-coderef-vault/
├── index.yaml          # vault metadata description
├── AGENTS.md           # operating guide for AI assistants
├── docs/               # stores study notes and documents
└── repos/              # manages reproduced code repositories through Git submodules
```

När du skapar ett valv initierar systemet automatiskt denna struktur:

```csharp
private async Task EnsureCodeRefStructureAsync(
    string vaultName,
    string physicalPath,
    ICollection<VaultBootstrapDiagnosticDto> diagnostics,
    CancellationToken cancellationToken)
{
    Directory.CreateDirectory(physicalPath);

    var indexPath = Path.Combine(physicalPath, CodeRefIndexFileName);
    var docsPath = Path.Combine(physicalPath, CodeRefDocsDirectoryName);
    var reposPath = Path.Combine(physicalPath, CodeRefReposDirectoryName);

    // Create the standard directory structure
    if (!Directory.Exists(docsPath))
    {
        Directory.CreateDirectory(docsPath);
    }

    if (!Directory.Exists(reposPath))
    {
        Directory.CreateDirectory(reposPath);
    }

    // Create the AGENTS.md guide
    await EnsureCodeRefAgentsDocumentAsync(physicalPath, cancellationToken);

    // Create the index.yaml metadata
    await WriteCodeRefIndexDocumentAsync(indexPath, mergedDocument, cancellationToken);
}
```

Denna struktur är också noggrant utformad:

- **docs/** lagrar dina studieanteckningar, där du kan registrera din förståelse av koden, arkitekturanalys, lärdomar och så vidare i Markdown
- **repos/** hanterar reproducerade arkiv genom Git-undermoduler istället för att kopiera kod direkt, vilket håller koden synkroniserad och sparar utrymme
- **index.yaml** innehåller valvets metadata så att AI-assistenter snabbt kan förstå syftet med och innehållet i valvet
- **AGENTS.md** är en guide skriven speciellt för AI-assistenter, som förklarar hur man hanterar innehållet i valvet

Organiserad på det här sättet kanske AI kan förstå vad du har i åtanke lite lättare.

## Automatisk initiering för systemhanterade valv

Förutom manuellt skapade valv stöder HagiCode även systemhanterade valv:

```csharp
public async Task<IReadOnlyList<VaultRegistryEntry>> EnsureAllSystemManagedVaultsAsync(
    CancellationToken cancellationToken = default)
{
    var definitions = GetAllResolvedDefinitions();
    var entries = new List<VaultRegistryEntry>(definitions.Count);

    foreach (var definition in definitions)
    {
        entries.Add(await EnsureResolvedSystemManagedVaultAsync(definition, cancellationToken));
    }

    return entries;
}
```

Systemet skapar och hanterar automatiskt följande valv:

- **hagiprojectdata**: projektdatalagring som används för att spara projektkonfiguration och tillstånd
- **personuppgifter**: personlig datalagring som används för att spara användarinställningar
- **hbsprompt**: ett promptmallbibliotek som används för att hantera vanliga AI-prompter

Dessa valv initieras automatiskt när systemet startar, så användare behöver inte konfigurera dem manuellt. Vissa saker är helt enkelt bättre att lämna till systemet istället för att människor oroar sig för dem.

## Tillträdeskontrollmekanism

En viktig del av designen är passerkontroll. Systemet delar upp valven i två åtkomsttyper:

```typescript
export interface VaultForText {
  id: string;
  name: string;
  type: string;
  physicalPath: string;
  accessType: 'read' | 'write';  // Key: distinguish read-only from editable
}
```

- **referens (skrivskyddad)**: AI används endast för analys och förståelse och kan inte ändra innehåll. Lämplig för refererade open source-projekt, dokument och liknande material
- **redigerbar (redigerbar)**: AI kan ändra innehåll efter behov för uppgiften. Lämplig för dina anteckningar, utkast och liknande material

Denna distinktion spelar roll. Den talar om för AI vilket innehåll som är "skrivskyddad referens" och vilket innehåll som är "säkert att redigera", vilket minskar risken för oavsiktliga ändringar. När allt kommer omkring vill ingen att deras hårda arbete försvinner på grund av en oavsiktlig redigering.

## I praktiken: skapa och använda Vault

Nu när vi har täckt idéerna, låt oss titta på hur det fungerar i praktiken.

### Skapa ett CodeRef-valv

Här är ett komplett frontend-samtal exempel:

```typescript
const createCodeRefVault = async () => {
  const response = await VaultService.postApiVaults({
    requestBody: {
      name: "React Learning Vault",
      type: "coderef",
      physicalPath: "/Users/developer/vaults/react-learning",
      gitUrl: "https://github.com/facebook/react.git"
    }
  });

  // The system will automatically:
  // 1. Clone the React repository into vault/repos/react
  // 2. Create the docs/ directory for notes
  // 3. Generate the index.yaml metadata
  // 4. Create the AGENTS.md guide file

  return response;
};
```

Detta API-anrop slutför en serie åtgärder: skapa katalogstrukturen, initiera Git-undermoduler, generera metadatafiler och mer. Du behöver bara tillhandahålla den grundläggande informationen och låta systemet sköta resten. Det är ärligt talat ett ganska bekymmersfritt tillvägagångssätt.

## Bästa metoder och saker att titta på

När vi använder Vault-systemet har vi sammanfattat några lärdomar.

### Vägsäkerhet

Systemet validerar strikt sökvägar för att förhindra vägövergångsattacker:

```csharp
private static string ResolveFilePath(string vaultRoot, string relativePath)
{
    var rootPath = EnsureTrailingSeparator(Path.GetFullPath(vaultRoot));
    var combinedPath = Path.GetFullPath(Path.Combine(rootPath, relativePath));
    if (!combinedPath.StartsWith(rootPath, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode,
            "Vault file paths must stay inside the registered vault root.");
    }
    return combinedPath;
}
```

Detta är viktigt. Om du anpassar en valvväg, se till att den håller sig inom det tillåtna intervallet, annars kommer systemet att avvisa operationen. Du kan verkligen inte överbetona säkerhet.

### Git-undermodulhantering

CodeRef Vault rekommenderar Git-undermoduler istället för att direkt kopiera kod:

```csharp
private static string BuildCodeRefAgentsContent()
{
    return """
    # CodeRef Vault Guide

    Repositories under `repos/` should be maintained through Git submodules
    rather than copied directly into the vault root.

    Keep this structure stable so assistants and tools can understand the vault quickly.
    """ + Environment.NewLine;
}
```

Detta ger flera fördelar: att hålla koden synkroniserad med uppströms, spara diskutrymme och göra det lättare att hantera flera versioner av koden. När allt kommer omkring, vem vill ladda ner samma sak om och om igen?

### Gränser för filförhandsgranskning

För att förhindra prestandaproblem begränsar systemet filstorlek och filtyp:

```csharp
private const int FileEnumerationLimit = 500;
private const int PreviewByteLimit = 256 * 1024;  // 256KB
```

Om ditt valv innehåller ett stort antal filer eller mycket stora filer kan prestandan för förhandsgranskningen påverkas. I så fall kan du överväga att bearbeta filer i omgångar eller använda specialiserade sökverktyg. Ibland när något blir för stort blir det svårare att hantera, inte lättare.

### Diagnostisk information

När du skapar ett valv returnerar systemet diagnostisk information för att hjälpa till med felsökning:

```csharp
List<VaultBootstrapDiagnosticDto> bootstrapDiagnostics = [];

if (IsCodeRefVaultType(normalizedType))
{
    bootstrapDiagnostics = await EnsureCodeRefBootstrapAsync(
        normalizedName,
        normalizedPhysicalPath,
        normalizedGitUrl,
        cancellationToken);
}
```

Om skapandet misslyckas kan du inspektera diagnostikinformationen för att förstå den specifika orsaken. När något går fel är att kontrollera diagnostiken ofta den mest direkta vägen framåt.

## Sammanfattning

Genom ett enhetligt lager för lagringsabstraktion löser Vault-systemet flera viktiga smärtpunkter för att reproducera projekt i AI-eran:

- **Centraliserad kunskapshantering**: alla lärresurser är samlade på ett ställe istället för utspridda överallt
- **Automatisk AI-kontextinjektion**: AI-assistenter kan automatiskt förstå de tillgängliga lärresurserna utan manuell kontextinställning
- **Återanvändning av kunskap i flera projekt**: kunskap kan delas och återanvändas i flera lärandeprojekt
- **Standardiserad katalogstruktur**: en konsekvent kataloglayout sänker inlärningskurvan
