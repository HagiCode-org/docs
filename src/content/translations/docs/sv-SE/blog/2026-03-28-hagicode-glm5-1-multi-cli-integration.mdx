## Hagicode och GLM-5.1 Multi-CLI Integrationsguide

:::note[Time Context Notice]

## Bakgrund

I Hagicode-projektet kan användare välja mellan flera CLI-verktyg för att driva AI-programmeringsassistenter, inklusive Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI och mer. Dessa CLI-verktyg är generella AI-programmeringsverktyg för sig själva, men genom Hagicodes abstraktionslager kan de flexibelt ansluta till olika AI-modellleverantörer.

Zhipu AI (ZAI) tillhandahåller ett gränssnitt som är kompatibelt med Anthropic Claude API, vilket gör att dessa CLI-verktyg direkt kan använda inhemska GLM-seriemodeller. Bland dem är GLM-5.1 Zhipus senaste version av stora språkmodeller, med betydande förbättringar jämfört med GLM-5.0.

## Hagicodes CLI-abstraktionsarkitektur

Hagicode definierar 11 CLI-leverantörstyper genom `AIProviderType` enum, som täcker vanliga AI-programmerings-CLI-verktyg:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Varje CLI har motsvarande modellparameterkonfiguration och stöder `model` och `reasoning` parametrar:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],

        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],

        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini does not support the reasoning parameter
        // ...
    };
```

## Använder GLM-5.1 i olika CLI:er

### 1. Claude Code CLI + GLM-5.1

Claude Code CLI är en av Hagicodes kärn-CLI och är konfigurerad genom Hero-konfigurationssystemet:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Motsvarande `HeroEquipmentCatalogItem` konfiguration:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI är den mest flexibla CLI och stöder specificering av alla modeller i `provider/model` format:

**Metod 1: Använd ZAI-leverantörsprefixet**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Metod 2: Använd modell-ID direkt**
```json
{
  "model": "glm-5.1"
}
```

**Metod 3: Gränssnitt för konfiguration av gränssnitt**

In `HeroModelEquipmentForm.tsx`, OpenCode CLI har en dedikerad platshållartips:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Användare kan ange:
```
zai/glm-5.1
glm-5.1
```

**OpenCode CLI-modellanalyslogik**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI har dedikerad äldre hanteringslogik:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Obs**: Codebuddy behåller speciell hantering för GLM-5.0 och använder inte äldre normalisering:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## Förbättringsfördelar med GLM-5.1

Jämfört med GLM-5.0 ger GLM-5.1 följande betydande förbättringar:

### 1. Bättre resonemangsförmåga

Enligt Zhipus officiella releaseinformation inkluderar förbättringar i GLM-5.1:

- **Stärkare kodförståelse**: Mer exakt analys av komplexa kodstrukturer
- **Längre sammanhangsförståelse**: Stöder längre konversationskontext
- **Förbättrad verktygsanrop**: Högre framgångsfrekvens för MCP-verktygsanrop
- **Utgångsstabilitet**: Minskar slumpmässighet och hallucinationer

### 2. Omfattande multi-CLI-kompatibilitet

GLM-5.1 täcker alla vanliga CLI:er som stöds av Hagicode:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Anteckningar

### 1. API-nyckelkonfiguration

Se till att `ANTHROPIC_AUTH_TOKEN` miljövariabeln är korrekt inställd. Det är den nödvändiga referensen för varje CLI att ansluta till modellen.

### 2. Modelltillgänglighet

GLM-5.1 måste aktiveras av motsvarande modellleverantör:
- Zhipu AI ZAI-plattformen stöder det som standard
- Alibaba Cloud DashScope kan kräva en separat applikation

### 3. OpenCode CLI-format

När du använder `provider/model` format, se till att leverantörs-ID:t är korrekt:
- Zhipu AI: `zai` eller `zhipuai`
- Alibaba moln: `aliyun` eller `dashscope`

### 4. Resoneringsparameter

- `high` rekommenderas för bästa resultat för kodgenerering
- Gemini CLI stöder inte resonemangsparametern och kommer att ignorera denna konfiguration automatiskt

## Slutsats

Genom ett enhetligt abstraktionslager möjliggör Hagicode flexibel integration mellan GLM-5.1 och flera CLI:er. Utvecklare kan välja det CLI-verktyg som bäst passar deras preferenser och användningsscenarier, och sedan använda den senaste GLM-5.1-modellen genom enkel konfiguration.

Som Zhipus senaste modellversion erbjuder GLM-5.1 tydliga förbättringar jämfört med GLM-5.0:
- En oberoende versionsidentifierare utan äldre börda
- Starkare resonemang och kodförståelse
- Bred multi-CLI-kompatibilitet
- Flexibel resonemangsnivåkonfiguration

Med rätt miljövariabler och Hero-utrustning konfigurerad kan användare helt låsa upp kraften i GLM-5.1 över olika CLI-miljöer.
