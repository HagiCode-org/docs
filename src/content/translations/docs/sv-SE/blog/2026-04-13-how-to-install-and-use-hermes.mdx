Om du vill installera Hermes och börja använda den är den kortaste vägen egentligen bara tre steg:

1. Kör det officiella installationskommandot
2. Starta CLI i din terminal med `hermes`
3. Om du vill fortsätta använda den i Feishu, konfigurera sedan `hermes gateway setup`

Den här artikeln försöker inte förklara alla Hermes-funktioner på en gång. Istället hjälper det dig att slutföra den viktigaste nybörjarslingan först: installera den, få den igång, börja använda den och sedan ansluta den till ett av de vanligaste scenarierna för meddelandeplattformar.

## Vad Hermes är och vem det är till för

Hermes Agent är en AI-agent som du kan använda antingen från en lokal terminal eller via en gateway för meddelandeplattformar.

För de flesta utvecklare har den två vanliga ingångspunkter:

- **CLI**: Typ `hermes` i din terminal för att komma in i det interaktiva gränssnittet direkt.
- **Meddelandegateway**: Kör `hermes gateway`, chatta sedan med den från plattformar som Feishu, Telegram, Discord och Slack.

Om ditt mål just nu helt enkelt är att komma igång snabbt, vänd inte ordningen. Börja med denna väg istället:

- Installera Hermes först
- Kontrollera först att det fungerar från CLI
- Bestäm sedan om du vill ansluta en meddelandeplattform

Detta gör problem lättare att diagnostisera och är mer lämpligt för personer som använder Hermes för första gången.

## Vad du ska veta innan du installerar Hermes

Enligt Hermes README stöder den officiella snabbinstallationsvägen dessa miljöer:

- Linux
- macOS
- WSL2
- Android via Termux

### Anteckningar för Windows-användare

Hermes stöder för närvarande inte körning direkt på inbyggt Windows. Om du använder Windows är det rekommenderade tillvägagångssättet att installera WSL2 först och sedan köra installationskommandot inuti WSL2.

Det är bäst att göra detta tydligt i början, eftersom många installationsfel inte orsakas av kommandot i sig, utan av att använda en runtime-miljö som inte stöds.

## Hur man installerar Hermes

Snabbinstallationskommandot i Hermes README är:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Detta kommando kör det officiella installationsskriptet och hanterar plattformsspecifika initieringssteg.

### Ladda om ditt skal efter installationen

När installationen är klar laddar du om skalmiljön först. Det vanligaste kommandot är:

```bash
source ~/.bashrc
```

Om du använder zsh kan du använda:

```bash
source ~/.zshrc
```

### Så här bekräftar du att Hermes är korrekt installerat

Det mest direkta sättet att kontrollera är att köra:

```bash
hermes
```

Om du vill ha ytterligare bekräftelse på att din konfiguration och dina beroenden fungerar kan du också köra:

```bash
hermes doctor
```

`hermes doctor` är särskilt användbar i dessa situationer:

- Kommandot beter sig onormalt efter installationen
- Modellkonfigurationen misslyckas
- Gatewayen startar inte
- Du är inte säker på om dina miljöberoenden är fullständiga

## Hur man börjar använda Hermes för första gången

### Börja med CLI

Om du bara vill bekräfta så snabbt som möjligt att Hermes fungerar är den enklaste metoden:

```bash
hermes
```

Detta lanserar den interaktiva Hermes CLI. För förstagångsanvändare av Hermes är det också den mest rekommenderade utgångspunkten, eftersom du kan verifiera de viktigaste sakerna först:

- Om kommandot faktiskt är tillgängligt
- Om den nuvarande modellkonfigurationen fungerar korrekt
- Om terminalverktygskedjan fungerar korrekt
- Om interaktionsstilen matchar det du behöver

### Dessa kommandon räcker för din första installationsomgång

Hermes README listar flera högfrekventa kommandon, och tillsammans bildar de en praktisk första-användningsväg:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Om du inte är säker på vad var och en gör, kom ihåg dem så här:

- `hermes model`: välj eller byt modell
- `hermes tools`: visa och konfigurera för närvarande tillgängliga verktyg
- `hermes config set`: ändra specifika konfigurationsobjekt
- `hermes setup`: kör hela initieringsguiden en gång
- `hermes update`: uppdatera Hermes
- `hermes doctor`: felsöka problem

För nybörjare är den mest praktiska ordningen vanligtvis:

1. Kör `hermes model` först
2. Om du vill konfigurera alla vanliga alternativ på en gång, kör sedan `hermes setup`

## De två vanligaste sätten att använda Hermes

### 1. Använd Hermes i terminalen som daglig utvecklingsassistent

CLI-läge passar bra för dessa scenarier:

- Ställ frågor direkt samtidigt som du skriver kod lokalt
- Inspektera projekt, redigera filer och kör kommandon
- Utför engångsfelsökning eller granskningsarbete
- Samarbeta kontinuerligt i den aktuella arbetskatalogen

Dess största fördel är att det är den kortaste vägen: ingen extra plattformsintegration, ingen botkonfiguration att hantera i förväg, och det är det bästa sättet att bygga upp dina första användningsvanor.

### 2. Använd Hermes via en meddelandeplattform

Om du vill chatta med Hermes på plattformar som Feishu, Telegram eller Discord måste du använda meddelandegatewayen.

De vanligaste inmatningskommandona är:

```bash
hermes gateway setup
hermes gateway
```

Specifikt:

- `hermes gateway setup` används för interaktiv plattformskonfiguration
- `hermes gateway` används för att starta gatewayprocessen

Enligt den officiella dokumentationen är gatewayen en enhetlig bakgrundsprocess som kopplar ihop dina konfigurerade plattformar, hanterar sessioner och hanterar funktioner som cron-jobb.

## Med Feishu som exempel: hur man ansluter Hermes till en meddelandeplattform

Om det mesta av ditt dagliga arbete sker i Feishu, så är Feishu/Lark ett mycket naturligt sätt att använda Hermes.

### Den minsta möjliga integrationsvägen

Den officiella dokumentationen rekommenderar detta ingångskommando för Feishu/Lark:

```bash
hermes gateway setup
```

När du har kört det, välj helt enkelt **Feishu / Lark** i guiden.

Feishu-dokumentationen beskriver två anslutningslägen:

- `websocket`: rekommenderas
- `webhook`: valfritt

Om Hermes körs på din bärbara dator, arbetsstation eller privata server, med hjälp av `websocket` första är vanligtvis enklare eftersom du inte behöver exponera en offentlig återuppringnings-URL.

### Om du konfigurerar det manuellt, känna åtminstone till dessa variabler

Om du inte använder guiden och skriver konfigurationen manuellt, listar Feishu-dokumentationen dessa kärnvariabler:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Två av dem förtjänar särskild uppmärksamhet:

- `FEISHU_ALLOWED_USERS`: rekommenderas, så inte alla som kan nå boten kan använda den direkt
- `FEISHU_HOME_CHANNEL`: låter dig fördefiniera en hemchatt för att ta emot cron-resultat eller standardaviseringar

### Varför Hermes ibland inte svarar i Feishu gruppchattar

Den här detaljen är lätt att missa: i Feishu-gruppchattar svarar Hermes inte på alla meddelanden som standard.

Den officiella dokumentationen säger tydligt:

- I direktmeddelanden svarar Hermes på meddelanden
- I gruppchattar måste du uttryckligen `@` boten innan den kommer att bearbeta meddelandet

Om du vill ställa in en Feishu-konversation som hemmakanal kan du också använda detta i chatten:

```text
/set-home
```

Eller definiera det i konfigurationen i förväg:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Hermes-kommandona nybörjare bör komma ihåg först

Oavsett om du använder Hermes i CLI eller på en meddelandeplattform är det redan tillräckligt att komma ihåg följande kommandon för att komma igång:

- `/new` eller `/reset`: starta en ny session
- `/model`: visa eller byta modell
- `/retry`: Försök med föregående sväng igen
- `/undo`: ångra föregående interaktion
- `/compress`: komprimera sammanhanget manuellt
- `/help`: visa hjälp

Om du huvudsakligen använder Hermes på en meddelandeplattform, kom ihåg en till:

- `/sethome` eller `/set-home`: ställ in den aktuella chatten som hemkanal

Dessa kommandon täcker de vanligaste operationerna i nybörjarstadiet: starta om, justera, rulla tillbaka, kontrollera och fortsätta.

## Vanliga frågor

### Kan Windows installera Hermes direkt?

Nej. Den aktuella officiella dokumentationen anger tydligt att inbyggt Windows inte stöds, och WSL2 rekommenderas.

### Vad ska jag göra om jag skriver `hermes` gör ingenting efter installationen?

Det är bäst att felsöka i denna ordning:

1. Ladda om ditt skal först, till exempel med `source ~/.bashrc`
2. Kör `hermes` igen
3. Om det fortfarande är onormalt, kör `hermes doctor`

### Varför svarar inte boten i en Feishu-grupp?

Kontrollera dessa tre saker först:

- Oavsett om du `@` nämnde Hermes i gruppen
- Huruvida `FEISHU_ALLOWED_USERS` begränsar den aktuella användaren
- Om den nuvarande policyn för gruppchatt tillåter hantering av gruppmeddelanden

Enligt den officiella Feishu-dokumentationen krävs explicit användning av ett @omnämnande i gruppchattscenarier.

3. Använd `hermes model` och `hermes setup` för att slutföra den grundläggande konfigurationen
