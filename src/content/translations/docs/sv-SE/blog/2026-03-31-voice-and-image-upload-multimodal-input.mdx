## Att skriva är långsammare än att prata, och att prata är långsammare än en skärmdump - Multimodala inmatningsmetoder för AI-kodningsassistenter

> Att skriva kod har en hastighetsgräns oavsett hur snabbt du skriver. Ibland tar något man kan säga i en mening en evighet att skriva ut; ibland förklarar en skärmdump allt, men du måste fortfarande beskriva det med en hög text. Den här artikeln berättar om vad vi stötte på när vi byggde [HagiCode](https://hagicode.com), från taligenkänning till bilduppladdningar. Till slut ville vi bara göra en AI-kodningsassistent lite lättare att använda.

## Bakgrund

När vi byggde HagiCode märkte vi ett problem - eller snarare, ett problem som naturligt dök upp när folk började använda den flitigt: att lita på att bara skriva kan vara tröttsamt.

Tänk på det: interaktion mellan användare och agenten är ett kärnscenario. Men om varje utbyte kräver nonstop-skrivning på tangentbordet, är effektiviteten inte stor:

1. **Det går för långsamt att skriva**: För komplicerade problem, som felmeddelanden eller gränssnittsproblem, kan det ta en halv minut att skriva allt, medan det kan ta tio sekunder att säga det högt. Det gapet är verkligt.

2. **Bilderna är mer direkta**: Ibland ger användargränssnittet ett fel, ibland vill du jämföra ett designutkast och ibland behöver du visa en kodstruktur. "En bild säger mer än tusen ord" kan vara ett gammalt talesätt, men det håller fortfarande. Att låta AI direkt "se" problemet är mycket tydligare än att beskriva det i evigheter.

3. **Interaktion ska kännas naturligt**: Moderna AI-assistenter ska stödja text, röst och bilder. Användare ska kunna välja vilken inmatningsmetod som känns mest naturlig.

Så vi bestämde oss för att lägga till stöd för taligenkänning och bilduppladdning till HagiCode för att göra agentinteraktioner mer bekväma. Om användare kan skriva lite mindre är det redan en vinst.

## Om HagiCode

Lösningarna som delas i den här artikeln kommer från vårt praktiska arbete i [HagiCode](https://hagicode.com) projekt – eller, rättare sagt, från lärdomar samtidigt som man snubblade genom en hel del fallgropar.

HagiCode är ett AI-kodningsassistentprojekt med öppen källkod med ett enkelt mål: använd AI för att förbättra utvecklingseffektiviteten. När vi fortsatte att bygga blev det tydligt att användarna starkt ville ha multimodal input. Ibland är det snabbare att säga en mening än att skriva ett långt stycke, och ibland är en skärmdump mycket tydligare än en lång förklaring.

Dessa behov drev oss framåt, och det var så funktioner som taligenkänning och bilduppladdningar tog form. Användare kan nu interagera med AI på det mest naturliga sätt som är tillgängligt för dem, och det känns bra.

## Analys

### Tekniska utmaningar i taligenkänning

När vi byggde taligenkänning stötte vi på ett knepigt problem: **webbläsarens WebSocket API stöder inte anpassade HTTP-rubriker**.

Taligenkänningstjänsten vi valde var ByteDances Doubao Speech Recognition API. Tyvärr kräver detta API autentiseringsinformation som t.ex `accessToken` och `secretKey` ska skickas via HTTP-rubriker. Det skapade en omedelbar teknisk konflikt:

```javascript
// The browser WebSocket API does not support this approach
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // Not supported
  }
});
```

Vi hade i princip två alternativ:

1. **URL-frågeparametermetoden**: lägg in autentiseringsinformationen i URL:en
   - Fördelen är att den är enkel att implementera
   - Nackdelen är att referenser exponeras för frontend, vilket är osäkert; vissa API:er kräver också rubrikbaserad autentisering

2. **Backend proxy approach**: implementera en WebSocket proxy på backend
   - Fördelen är att referenserna förblir säkert lagrade på backend och lösningen är helt kompatibel med API-krav
   - Nackdelen är att implementeringen är lite mer komplex

Till slut valde vi **backend proxy-metoden**. Säkerhet är inget man kompromissar med.

### Funktionskrav för bilduppladdningar

Våra krav för bilduppladdning var faktiskt ganska enkla:

1. **Flera uppladdningsmetoder**: klicka för att välja en fil, dra och släpp och klistra in från urklippet
2. **Filvalidering**: typbegränsningar (PNG, JPG, WebP, GIF) och storleksbegränsningar (5-10 MB) är grundläggande krav
3. **Användarupplevelse**: ladda upp framsteg, förhandsgranskningar och felmeddelanden så att användarna alltid vet vad som händer
4. **Säkerhet**: validering på serversidan och skydd mot skadliga filuppladdningar är avgörande

## Lösning

### Taligenkänning: WebSocket Proxy Architecture

Vi designade en trelagersarkitektur för taligenkänning och hittade en väg som fungerade:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**Kärnkomponentimplementationer**:

1. **Frontend AudioWorklet-processor**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Resample to 16 kHz (required by the Doubao API)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Accumulate samples into 500 ms chunks
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Convert to 16-bit PCM and send
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **Backend WebSocket-hanterare** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **Frontend `VoiceTextArea` komponent**:

```tsx
export const VoiceTextArea = forwardRef<HTMLTextAreaElement, VoiceTextAreaProps>(
  ({ value, onChange, onTextRecognized, maxDuration }, ref) => {
    const { isRecording, interimText, volume, duration, startRecording, stopRecording } =
      useVoiceRecording({ onTextRecognized, maxDuration });

    return (
      <div className="flex gap-2">
        {/* Voice button */}
        <button onClick={handleButtonClick}>
          {isRecording ? <VolumeWaveform volume={volume} /> : <Mic />}
        </button>
        {/* Text input area */}
        <textarea value={displayValue} onChange={handleChange} />
      </div>
    );
  }
);
```

### Bilduppladdningar: Multi-Method Upload Component

Vi byggde en fullfjädrad bilduppladdningskomponent med stöd för alla tre uppladdningsmetoderna, som täcker de vanligaste scenarierna som användare stöter på.

**Kärnfunktioner**:

1. **Tre uppladdningsmetoder**:

```tsx
// Click to upload
const handleClick = () => fileInputRef.current?.click();

// Drag-and-drop upload
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Clipboard paste
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **Frontendvalidering**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **Hanterare för uppladdning av backend** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Validation
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Save file
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## Praktisk guide

### Hur man använder taligenkänning

1. **Konfigurera taligenkänningstjänsten**:
   - Öppna inställningssidan för taligenkänning
   - Konfigurera Doubao-talet `AppId` och `AccessToken`
   - Alternativt kan du konfigurera kommandoord för att förbättra igenkänningsnoggrannheten för domänspecifika termer

2. **Använd den i inmatningsrutan**:
   - Klicka på mikrofonikonen till vänster om inmatningsrutan
   - Börja tala efter att vågformsanimeringen visas
   - Klicka på ikonen igen för att stoppa inspelningen
   - Den igenkända texten infogas automatiskt vid markörens position

3. **Exempel på kommandoordskonfiguration**:

```text
TypeScript
React
useState
useEffect
```

### Hur man använder bilduppladdningar

1. **Uppladdningsmetoder**:
   - Klicka på uppladdningsknappen för att välja en fil
   - Dra en bild direkt till uppladdningsområdet
   - Använd `Ctrl+V` för att klistra in en skärmdump från klippbordet

2. **Format som stöds**: PNG, JPG, JPEG, WebP, GIF
3. **Storleksgräns**: 5 MB som standard (konfigurerbar)

### Anteckningar

1. **Taligenkänning**:
   - Mikrofontillstånd krävs
   - Använd i en lugn miljö när det är möjligt
   - Den maximala inspelningslängden som stöds är 300 sekunder som standard (konfigurerbar)

2. **Bilduppladdning**:
   - Endast vanliga bildformat stöds
   - Var uppmärksam på filstorleksgränserna
   - Uppladdade bilder får automatiskt en förhandsgransknings-URL

3. **Säkerhetsaspekter**:
   - Taligenkänningsuppgifter lagras i backend
   - Bilduppladdningar går igenom strikt validering på serversidan
   - HTTPS/WSS rekommenderas i produktionsmiljöer

## Slutsats

Efter att ha lagt till taligenkänning och bilduppladdningar förbättrades HagiCodes användarupplevelse märkbart. Användare kan nu interagera med AI på ett mer naturligt sätt - tala istället för att skriva och dela skärmdumpar istället för att beskriva allt manuellt. Det känns som att äntligen hitta ett bekvämare sätt att kommunicera på.

När vi byggde den här funktionen stötte vi på problemet att webbläsarens WebSocket API:er inte stöder anpassade rubriker. Till slut löste vi det med en backend-proxy-metod. Den lösningen bevarade inte bara säkerheten, utan lade också grunden för att senare integrera andra autentiserade WebSocket-tjänster.

Bilduppladdningskomponenten drar också nytta av att stödja flera uppladdningsmetoder, så att användarna kan välja det som är mest praktiskt just nu. Genom att klicka, dra eller klistra in allt arbete, och varje bana gör jobbet snabbt.

"Att skriva är långsammare än att prata, och att prata är långsammare än en skärmdump" passar temat här ganska bra. Om du bygger en liknande AI-assistentprodukt hoppas jag att dessa erfarenheter hjälper, även om det bara är lite.
