## Omfang og metode

- **Mål**: Giv modelvalgsvejledning baseret på ægte integrationserfaring i HagiCode.
- **Opgavetyper**: Frontend-komponentimplementering, backend API refactoring, testafslutning og dokumentationsgenerering.
- **Evalueringsakser**: leveringseffektivitet (kan det pålideligt afslutte rigtige opgaver) og omkostningseffektivitet (omkostninger + indenlandsk tilgængelighed).

## Notater om testtidspunkt og scenarier

- **Seneste testdato**: 2026-03-08
- **Testperiode**: 2026-03-01 til 2026-03-08
- **Eksempelgrundlag**: Subjektiv evaluering fra rigtige HagiCode-ingeniørarbejdsgange, ikke leverandørbenchmark-tal.
- **Anvendelse**: Konklusionerne er baseret på dette projekts nuværende arbejdsgang og begrænsninger.

## Rangering af leveringseffektivitet (forfatteranbefaling)

2. GPT 5.3 Codex

5. Minimax M2.5
6. Qwen (3,5 / Kode næste)

### Nøglefund

- Med undtagelse af Qwen kan alle andre testede modeller opnå vores målresultater til en vis grad.
- GLM 4.7+ (GLM 4.7 og GLM 5) opfylder generelt vores krav problemfrit.
- GPT 5.3 Codex og GPT 5.4 fuldender ikke kun kravene, men producerer også bedre ingeniørpraksis og implementeringskvalitet.
- Minimax M2.5 har en tilbagevendende svaghed: kodelukningsfejl (f.eks. ufuldstændig parentes/blok lukning), så ekstra gennemgang er nødvendig.

1. Minimax M2.5
2. GLM 4.7

4. Qwen 3.5 / Code Next
5. GPT 5.3 Codex
6. GPT 5.4

> Bemærk: denne rangering er bevidst forskellig fra rangering af leveringseffektivitet.

## Udvælgelsesvejledning

- **Kvalitet først**: vælg GPT 5.4 / GPT 5.3 Codex.
- **Balanceret strategi**: vælg GLM 5 / GLM 4.7.
- **Cost-first**: vælg Minimax M2.5 (med strengere kontrol med kodelukning).
- **Praktisk routing**: Brug premiummodeller til kritiske opgaver og omkostningseffektive modeller til rutineopgaver.

## Utestet modelerklæring

For modeller, der ikke er angivet her, har vi i øjeblikket **ingen testdata** og **ingen praktisk erfaring**, så vi giver ikke evalueringer.

Hvis sponsorer giver adgang til yderligere modeller, vil vi køre erfaringsbaserede evalueringer i vores rigtige arbejdsgang og opdatere denne side.
