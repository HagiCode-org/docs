# Installer OpenSpec

OpenSpec administrerer forslag, specifikationer og ændringsposter. Denne side holder sig til ét mål: Forklar installationsstien tydeligt, så du kan knytte den til din eksisterende arbejdsgang.

## Hvad er OpenSpec?

OpenSpec er et kommandolinjeværktøj til styring af forslag og specifikationer i softwarearbejde. Kernearbejdsgangen omfatter:

- **Proposal Management**: Opret, gennemgå og spor funktionsforslag
- **Specifikationsvalidering**: Sørg for, at kodeændringer er i overensstemmelse med foruddefinerede specifikationer
- **Ændringssporing**: Registrer al gennemgået ændringshistorik
- **Teamsamarbejde**: Støt samarbejdsudvikling med flere personer

### Anvendelse i HagiCode Projects

I HagiCode-projekter bruges OpenSpec til:

1. **Funktionsudviklingsstyring**: Alle nye funktioner skal gennemgå OpenSpec-forslagsprocessen
2. **Kodegennemgang**: Sørg for, at ændringer overholder projektspecifikationerne
3. **Dokumentationssynkronisering**: Oprethold overensstemmelse mellem dokumentation og kodeimplementering
4. **Version Control Integration**: Problemfri integration med versionskontrol workflows

## Forudsætninger

Før du installerer OpenSpec, skal du sikre dig, at dit system opfylder følgende krav:

### Node.js og npm

OpenSpec distribueres via npm og kræver et Node.js-miljø. OpenSpec kræver **Node.js 18.0 eller nyere**.

Bekræft Node.js-installationen:

```bash
node --version
```

Bekræft npm installation:

```bash
npm --version
```

Hvis den ikke er installeret, eller hvis versionen er for gammel, henvises der til den tilhørende softwareinstallationsvejledning i [Installationsvejledning](/en/installation).

## Installation af OpenSpec

:::tip[Versionskrav]
Dette projekt kræver brug af **OpenSpec 1.0.0 eller højere** versionsområde. Alle 1.x-versioner understøttes for at sikre kompatibilitet med projektets arbejdsgang.
:::

OpenSpec installeres globalt via npm, hvilket gør det tilgængeligt hvor som helst på dit system.

### Installer globalt ved hjælp af npm

<Tabs>
  <TabItem value="win" label="Windows">
    Kør i PowerShell eller kommandoprompt:

```powershell
npm install -g @fission-ai/openspec@1
```

:::note[Windows-tilladelsesproblemer]
Hvis du støder på tilladelsesfejl under installationen, skal du køre PowerShell eller kommandoprompt som administrator.
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    Kør i terminal:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[macOS-tilladelsesproblemer]
Hvis du støder på `EACCES` tilladelsesfejl, prøv:
```bash
sudo npm install -g @fission-ai/openspec@1
```
Eller brug [nvm](https://github.com/nvm-sh/nvm) at administrere Node.js-versioner og undgå tilladelsesproblemer.
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    Kør i terminal:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[Linux-tilladelsesproblemer]
Hvis du støder på `EACCES` tilladelsesfejl, kan du:
1. Brug `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. Konfigurer npm til at bruge brugermappe:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### Bekræft installationen

Efter installationen skal du kontrollere, at OpenSpec er installeret korrekt, og at versionen opfylder kravene:

```bash
openspec --version
```

Hvis installationen er vellykket, vil du se et versionsnummer, for eksempel:

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::note[Versionsområde]
Sørg for, at den installerede OpenSpec-version er 1.0.0 eller nyere. Projektet har verificeret kompatibilitet med version 1.1.1. Det anbefales at bruge den seneste kompatible version for at få nye funktioner.
:::

## Almindelige problemer og fejlfinding

### Tilladelsesfejl

#### Problem: `EACCES` Tilladelse nægtet

Støder på tilladelsesfejl ved installation eller kørsel af OpenSpec.

**Løsninger**:

<Tabs>
  <TabItem value="win" label="Windows">
    Kør PowerShell eller kommandoprompt som administrator:
    1. Højreklik på PowerShell eller kommandoprompt
    2. Vælg "Kør som administrator"
    3. Kør installationskommandoen igen
  </TabItem>
  <TabItem value="mac" label="macOS">
    Installer vha `sudo`:

```bash
sudo npm install -g @fission-ai/openspec@1
```

Eller brug [nvm](https://github.com/nvm-sh/nvm) at administrere Node.js og undgå problemer med globale installationstilladelser.
  </TabItem>
  <TabItem value="linux" label="Linux">
    Løsning 1: Brug sudo
```bash
sudo npm install -g @fission-ai/openspec@1
```

Løsning 2: Konfigurer npm til at bruge brugerbibliotek
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @fission-ai/openspec@1
```
  </TabItem>
</Tabs>

### Netværksproblemer

#### Problem: npm Installation mislykkes eller er meget langsom

Kan ikke downloade OpenSpec fra npm registreringsdatabasen på grund af netværksårsager.

**Løsninger**:

1. **Brug indenlandsk npm-spejl**:

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **Gendan officiel kilde** (hvis nødvendigt):

```bash
npm config set registry https://registry.npmjs.org
```

3. **Brug cnpm** (valgfrit):

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

::: tip
Brugere på det kinesiske fastland anbefales at bruge Taobao-spejlet, som kan forbedre downloadhastigheden markant.
:::

### Problemer med versionskompatibilitet

#### Problem: OpenSpec-kommandoer kører unormalt eller rapporter fejl

Den installerede OpenSpec-version er inkompatibel med det aktuelle miljø.

**Løsninger**:

1. **Tjek Node.js-version**:

```bash
node --version
```

Sørg for, at versionen er 18.0 eller nyere. Hvis versionen er for lav, opgrader venligst Node.js.

2. **Tjek OpenSpec-version**:

```bash
openspec --version
```

3. **Geninstaller specifik version**:

```bash
npm install -g @fission-ai/openspec@1
```

4. **Ryd npm-cache** (hvis problemet fortsætter):

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### Kommando ikke fundet

#### Problem: Indtastning `openspec` Kommando viser "Kommando ikke fundet"

OpenSpec er installeret, men systemet kan ikke finde kommandoen.

**Løsninger**:

<Tabs>
  <TabItem value="win" label="Windows">
    1. Bekræft npm global installationssti:

```powershell
npm config get prefix
```

2. Tilføj sti til system PATH:
   - Åbn "Systemegenskaber" → "Avanceret" → "Miljøvariabler"
   - Find `Path` i "Systemvariabler"
   - Tilføj npm global sti (normalt `C:\Users\<username>\AppData\Roaming\npm`)
   - Genstart kommandolinjevinduet
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. Bekræft npm global installationssti:

```bash
npm config get prefix
```

2. Tilføj sti til PATH (in `~/.zshrc` eller `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Genindlæs konfiguration:

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. Bekræft npm global installationssti:

```bash
npm config get prefix
```

2. Tilføj sti til PATH (in `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Genindlæs konfiguration:

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
