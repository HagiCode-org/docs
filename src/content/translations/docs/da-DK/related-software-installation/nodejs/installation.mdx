Denne vejledning dækker installation af Node.js på Windows, macOS og Linux. Node.js er en nøgleafhængighed for HagiCode lokalt værktøj, herunder OpenSpec, frontend-værktøj og adskillige udvikler-CLI'er.

:::tip[Versionskrav]
HagiCode anbefaler **Node.js 24+** (minimum understøttet Node.js &gt;= 18.0). I de fleste tilfælde skal du installere den seneste LTS-version og derefter kontrollere, at din version opfylder kravet.
:::

## Installerer Node.js

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Metode 1: Officielt installationsprogram (anbefalet)

1. Besøg [Node.js officielle hjemmeside](https://nodejs.org/)
2. Download **LTS (Long Term Support)** version af Windows-installationsprogrammet (`.msi` fil)
3. Dobbeltklik for at køre installationsprogrammet
4. Følg instruktionerne i installationsguiden for at fuldføre installationen:
   - Accepter licensaftalen
   - Vælg installationssti (standard er i orden)
   - Sørg for, at "Installer automatisk de nødvendige værktøjer" er markeret
   - Klik på "Installer" for at starte installationen
5. Når installationen er fuldført, skal du klikke på "Udfør"

### Metode to: Brug af winget

Åbn PowerShell eller kommandoprompt og kør:

```powershell
winget install OpenJS.NodeJS.LTS
```

Efter installationen skal du genåbne terminalen for at ændringerne træder i kraft.

  </TabItem>
  <TabItem value="macos" label="macOS">

### Metode 1: Officielt installationsprogram (anbefalet)

1. Besøg [Node.js officielle hjemmeside](https://nodejs.org/)
2. Download **LTS (Long Term Support)** version macOS installationsprogrammet (`.pkg` fil)
3. Dobbeltklik for at køre installationsprogrammet
4. Følg instruktionerne i installationsguiden for at fuldføre installationen:
   - Klik på "Fortsæt"
   - Accepter licensaftalen
   - Vælg installationsdisk
   - Klik på "Installer" og indtast administratoradgangskode
5. Når installationen er fuldført, skal du klikke på "Luk"

### Metode to: Brug af Homebrew

Hvis du har [Hjemmebryg](https://brew.sh/) installeret, kan du installere Node.js via terminal:

```bash
brew install node
```

::: tip
Homebrew tilføjer automatisk Node.js til din PATH. Hvis kommandoen ikke er tilgængelig, genstart terminalen eller kør:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

  </TabItem>
  <TabItem value="ubuntu" label="Ubuntu/Debian">

### Brug af NodeSource Repository (anbefalet)

Dette er den bedste måde at få den seneste Node.js-version på:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Brug af apt Package Manager

Hvis du ikke har brug for den nyeste version, kan du bruge standard Ubuntu-lageret:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::bemærk
Node.js-versionen i standardlageret kan være ældre. Efter installationen skal du kontrollere versionen for at sikre, at den opfylder kravene til Node.js &gt;= 18.0 (Node.js 24+ anbefales).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### Brug af NodeSource Repository (anbefalet)

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### Brug af yum Package Manager

```bash
sudo yum install -y nodejs npm
```

  </TabItem>
  <TabItem value="fedora" label="Fedora">

### Brug af dnf Package Manager

```bash
sudo dnf install -y nodejs npm
```

::: tip
Fedoras softwarelager indeholder normalt nyere Node.js-versioner og kan installeres direkte via dnf.
:::

  </TabItem>

## Brug af versionsstyringsværktøjer

Hvis du skal skifte mellem flere Node.js-versioner, kan du bruge versionsstyringsværktøjer.

### nvm (Node Version Manager)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

Efter installationen skal du genindlæse din shell-konfiguration:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

På Windows anbefaler vi at bruge [nvm-vinduer](https://github.com/coreybutler/nvm-windows):

1. Download den seneste `nvm-setup.exe` fra [nvm-windows udgivelser](https://github.com/coreybutler/nvm-windows/releases)
2. Kør installationsprogrammet
3. Genåbn kommandoprompt eller PowerShell

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

Efter installation af nvm, kan du bruge den til at installere og administrere Node.js-versioner:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (Fast Node Manager)

fnm er et hurtigere og enklere Node.js-versionsstyringsværktøj bygget med Rust.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

Brug [scoop](https://scoop.sh/) eller [vinge](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Acceleration for brugere af det kinesiske fastland

Hvis du er på det kinesiske fastland, kan det være langsomt at downloade Node.js- og npm-pakker. Her er nogle accelerationsløsninger.

### Brug af indenlandske spejlkilder

#### npm Taobao-spejl (anbefalet)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### Brug af nrm til at administrere spejlkilder

nrm (npm registry manager) hjælper dig med hurtigt at skifte npm-spejlkilder:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Accelerering af Node.js Installer-downloads

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao leverer spejldownloads til Node.js-installatører:

- **Download-adresse**: https://npmmirror.com/mirrors/node/

Besøg ovenstående adresse og vælg den version, du skal downloade.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

Hvis download af nvm eller andre værktøjer fra GitHub er langsom, kan du bruge FastGit-spejlet:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::bemærk
FastGit-tjenesten kan ændre sig, se venligst officiel information.
:::

  </TabItem>
</Tabs>

### Hjemmebrugsspejle

Hvis du bruger Homebrew i Kina, kan du konfigurere hjemlige spejle til acceleration:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Administratorer af specifikke pakker for det kinesiske fastland

#### cnpm

cnpm er en npm-klient leveret af Taobao-teamet, der bruger Taobao-spejlet som standard:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

::: tip
cnpm er for det meste kompatibel med npm-kommandoer og kan bruges som en direkte erstatning.
:::

### Bekræftelse af spejlaccelerationseffekt

Efter opsætning af spejle skal du bekræfte ved at:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Bekræftelse af installation

Efter installationen skal du kontrollere, at Node.js og npm er installeret korrekt:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::bemærk
Sørg for, at Node.js version &gt;= 18.0 (Node.js 24+ anbefales). Hvis versionen er for lav, skal du geninstallere ved hjælp af ovenstående metoder eller bruge versionsstyringsværktøjer til at installere den korrekte version.
:::

### Prøvebekræftelsesoutput

Vellykket bekræftelsesoutput skal se sådan ud:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Fejlfinding

### Windows-problemer

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Hvis `node` eller `npm` kommandoer er ikke tilgængelige efter installation:

1. Genstart kommandoprompt eller PowerShell
2. Hvis det stadig ikke er tilgængeligt, skal du manuelt tilføje Node.js til PATH:
   - Søg efter "Environment Variables" og åbn "Rediger systemmiljøvariabler"
   - Klik på "Miljøvariabler"
   - Find "Sti" i "Systemvariabler" og klik på "Rediger"
   - Tilføj Node.js installationssti (standard: `C:\Program Files\nodejs\`)
   - Klik på "OK" for at gemme ændringer

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Hvis du støder på kørselspolitikfejl, når du bruger nvm:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### macOS problemer

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Hvis du støder på tilladelsesfejl, når du installerer globale npm-pakker:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Hvis kommandoer ikke er tilgængelige efter installationen, skal du genstarte terminalen eller manuelt tilføje til PATH (normalt i `/usr/local/bin`).

  </TabItem>
</Tabs>

### Linux problemer

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Hvis du støder på tilladelsesfejl, når du installerer globale npm-pakker, anbefales det at bruge Node.js-versionsstyringsværktøjer (som nvm eller fnm) i stedet for systempakkeadministratorer.

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Hvis Node.js-versionen, der er installeret via systempakkehåndtering, er for gammel:

1. Brug NodeSource repository (anbefalet, se ovenfor)
2. Eller brug versionsstyringsværktøjer (nvm/fnm)

  </TabItem>
</Tabs>

### Kontrol af miljø

Hvis verifikationen mislykkes, skal du kontrollere følgende:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
