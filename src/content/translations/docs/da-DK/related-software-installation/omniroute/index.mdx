Denne side er for førstegangsbrugere af HagiCode, der ønsker den korteste OmniRoute-opsætningssti: start den lokale tjeneste, bekræft at `http://localhost:20128/v1` fungerer, og bekræft, at både Dashboard og API er tilgængelige.

:::tip[Anbefalet sti]
Denne første version holder den mest stabile vej først: `npm install -g omniroute`, `omniroute`, Dashboard kl `http://localhost:20128`, og den OpenAI-kompatible base-URL på `http://localhost:20128/v1`. Docker, VPS og mere avancerede implementeringstilstande forbliver i den valgfri sektion.
:::

## Forudsætninger

- Node.js er allerede installeret. Komplet [Installerer Node.js](/en/related-software-installation/nodejs/installation) først, hvis det er nødvendigt.
- Lokal havn `20128` er ledig og ikke allerede optaget.
- Du har mindst én opstrømsmodeludbyderkonto, som du planlægger at rute gennem OmniRoute.
- Før du integrerer HagiCode, en CLI eller et SDK, skal du holde disse to adresser adskilt:
  - Dashboard: `http://localhost:20128`
  - OpenAI-kompatibel base-URL: `http://localhost:20128/v1`

## Installer og kør lokalt

Start med den anbefalede globale npm-sti:

```bash
npm install -g omniroute
omniroute
```

Når tjenesten er startet, skal du validere den i denne rækkefølge:

1. Åbn `http://localhost:20128` i en browser og bekræft, at Dashboardet indlæses.
2. Gå til `Endpoints` side i Dashboardet og opret eller kopier en API-nøgle.
3. Bemærk den basis-URL, du vil give værktøjerne senere: `http://localhost:20128/v1`.

:::note[Hvorfor starte med Dashboard]
Til den første opsætning er Dashboard den hurtigste måde at bekræfte, at OmniRoute kører, at dine udbydere er tilsluttet, og at der findes en API-nøgle. Hvis et senere klientkald mislykkes, er det normalt hurtigere at tjekke dashboardet først end at fejlsøge SDK-kode med det samme.
:::

## Valider den kørende tjeneste

### Validering 1: Bekræft, at modelkataloget er læsbart

Læg nøglen fra `Endpoints` side ind i en miljøvariabel:

```bash
export OMNIROUTE_API_KEY="replace-with-the-key-you-copied-from-the-Endpoints-page"
```

Anmod derefter `/v1/models`:

```bash
curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}"
```

Hvis du får en modelliste tilbage, fungerer det OpenAI-kompatible slutpunkt og godkendelsesflow.

### Bekræftelse 2: Send en minimal chatanmodning

```bash
curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "replace-with-a-model-id-from-v1-models",
    "messages": [
      { "role": "user", "content": "Reply with ok." }
    ]
  }'
```

Målet her er ikke hurtig kvalitet. Det er for at bevise, at din valgte model allerede kan rutes gennem OmniRoute med succes.
