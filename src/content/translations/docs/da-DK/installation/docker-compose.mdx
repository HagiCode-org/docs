## Forudsætninger

Før du starter, skal du sikre dig, at dit system har Docker og Docker Compose installeret.

### Installer Docker

#### Windows

Download og installer [Docker Desktop til Windows](https://www.docker.com/products/docker-desktop/)

Efter installationen skal du sikre dig, at Docker Desktop kører.

#### macOS

Download og installer [Docker Desktop til Mac](https://www.docker.com/products/docker-desktop/)

Efter installationen skal du sikre dig, at Docker Desktop kører.

#### Linux

Installer Docker ved hjælp af din pakkehåndtering:

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

Efter installationen skal du starte Docker-tjenesten:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Bekræft installationen

Efter installationen skal du køre følgende kommandoer for at kontrollere, at Docker og Docker Compose er installeret korrekt:

```bash
docker --version
docker compose version
```

## Hurtig start

### 1. Generer Docker Compose-konfigurationsfil

:::tip[Brug Configuration Generator]
** Anbefaler at bruge [Docker Compose Builder (stand-alone site)](https://builder.hagicode.com/)** for at generere din konfigurationsfil.

Generatoren understøtter:
- Muligheder for flere API-udbydere (Zhipu AI, Anthropic Official, Custom API)
- Automatisk konfiguration af API URL og Token
- Brugerdefinerede porte, containernavne og anden grundlæggende konfiguration
- Konfigurer vedvarende data og arbejdsmappestier
- Konfigurer automatisk Windows/Linux platform forskelle
- Håndter problemstillinger med filtilladelser intelligent
- **Valg af billedkilde**: Vælg den relevante billedkilde baseret på dit netværksmiljø

[🚀 Brug generatoren med det samme →](https://builder.hagicode.com/)
:::

1. Åbn [Docker Compose Builder (stand-alone site)](https://builder.hagicode.com/)
2. Udfyld konfigurationen efter dine behov:
   - Vælg API-udbyder (Zhipu AI, Anthropic Official eller Custom API)
   - Konfigurer porte, database, arbejdsmappe og andre muligheder
   - **Vælg billedkilde**: Vælg den passende billedkilde baseret på dit netværksmiljø
3. Klik på knappen Generer for at få `docker-compose.yml` konfiguration
4. Gem den genererede konfiguration som en `docker-compose.yml` fil
5. Opret om nødvendigt en `.env` fil for at konfigurere følsomme oplysninger

#### Valg af billedkilde

Generatoren understøtter tre billedkildemuligheder:

**Docker Hub**
- **Billedadresse**: `newbe36524/hagicode:{tag}`
- **Gældende scenarier**: Velegnet til brugere med Docker Hub-spejlacceleration
- **Fordele**: Officiel billedkilde, rettidige opdateringer, stabil adgang
- **Bemærkninger**: Spejlacceleration skal muligvis konfigureres i nogle områder

**Azure Container Registry (ACR)**
- **Billedadresse**: `hagicode.azurecr.io/hagicode:{tag}`
- **Gældende scenarier**: Velegnet til brugere, der ikke kan få adgang til Docker Hub lokalt
- **Fordele**: Giver alternativ billedkilde, løser problemer med netværksadgang
- **Bemærkninger**: Billeder forbliver synkroniserede med Docker Hub, men kan have korte forsinkelser

**Alibaba Cloud Container Registry (ACR)**
- **Billedadresse**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **Gældende scenarier**: Velegnet til private brugere, stabil og hurtig netværksadgang
- **Fordele**: Alibaba Cloud indenlandske noder, hurtig adgangshastighed, høj stabilitet
- **Bemærkninger**: Alibaba ACR-billeder forbliver synkroniserede med Docker Hub, normalt inden for 30 minutter

:::tip[Anbefalinger til valg af billedkilde]
- **Indenlandske brugere anbefaler Alibaba Cloud ACR**: Hvis du er en indenlandsk bruger, anbefales det at prioritere at bruge Alibaba Cloud ACR billedkilde for hurtig og stabil adgang
- **Alternativ Docker Hub**: Hvis dit netværksmiljø kan få adgang til Docker Hub, kan du også vælge Docker Hub som billedkilde
- **Alternativ Azure ACR**: Vælg kun Azure ACR-billedkilde, hvis andre billedkilder har adgangsbesvær
- **Billedsynkronisering**: Alibaba Cloud ACR-billeder forbliver synkroniserede med Docker Hub, normalt inden for 30 minutter
- **Versionskonsistens**: Billedtags er konsistente på tværs af alle tre billedkilder, ingen grund til at bekymre dig om versionsproblemer
:::

### 2. Start Services

Udfør følgende kommando i den mappe, hvor din `docker-compose.yml` er placeret:

```bash
docker compose up -d
```

Denne kommando vil:
- Download og start Hagicode-beholderne, der er defineret af den genererede skabelon
- Opret og konfigurer netværket
- Initialiser standarddatabiblioteket og runtime-afhængigheder

### 3. Bekræft tjenestestatus

Tjek om containere kører:

```bash
docker compose ps
```

Du bør se de tjenester, der er defineret i din genererede skabelon i status "kører".

### Se servicelogfiler

Sådan får du vist alle servicelogfiler:

```bash
# View all service logs
docker compose logs

# View Hagicode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## Konfigurationsvejledning

Hvis du har brug for at ændre konfigurationen (såsom at ændre porte, skifte API-udbyder osv.), kan du:

1. Gendan ny konfiguration vha [Docker Compose Builder (stand-alone site)](https://builder.hagicode.com/)
2. Rediger manuelt `docker-compose.yml` og `.env` filer
3. Genstart tjenester for at få konfigurationen til at træde i kraft:

```bash
docker compose restart
```
