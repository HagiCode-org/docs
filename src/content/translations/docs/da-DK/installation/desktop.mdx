Denne vejledning dækker installation og brug af Hagicode Desktop. Det er den officielle desktop-klient og holder den lokale standardopsætningssti plus daglige kontroller på ét sted.

:::tip[Anbefalet metode]
Desktop er den lokale standardsti for individuelle brugere og udviklere, fordi:
- installationsprogrammet beholder standardflowet i én guidet sti
- afhængigheder og tjenester administreres i appen
- brugergrænsefladen gør tilstand og drift nemmere at inspicere
- indbygget versionsstyring gør skift lettere
- data forbliver lokalt som standard
:::

## Hvad er Hagicode Desktop?

Hagicode Desktop er Hagicodes officielle desktop-applikation, designet til Windows, macOS og Linux-systemer. Det samler Hagicodes kernefunktioner, og installationsprogrammet beholder standardopsætningsstien i ét guidet flow.

### Vigtigste fordele

- **Et-klik installation**: Fuldfør alle konfigurationstrin automatisk gennem installationsguiden
- **Automatisk Runtime Preparation**: Standardinstallationsprogrammet inkluderer allerede den runtime, det har brug for, så du behøver ikke at installere en separat `.NET` runtime først
- **Service Management**: Indbygget Hagicode Server-administration, let start og stop tjenester
- **Versionsskift**: Understøttelse af multiversionsstyring, skift frit mellem forskellige versioner
- **Lokal drift**: Alle data gemmes lokalt, hvilket beskytter dit privatliv
- **Automatiske opdateringer**: Registrerer og downloader automatisk den seneste version

### Gældende scenarier

Desktop passer godt, når:
- Individuelle udviklere, der ønsker at bruge Hagicode lokalt
- Brugere, der skal bruge Hagicode offline
- Ikke-tekniske brugere, der ønsker at forenkle installationsprocessen
- Udviklere, der ofte skal skifte mellem forskellige versioner

### Andre downloadmetoder

Du kan også få Hagicode Desktop-installationsprogrammet fra følgende kanaler:

1. **Officiel udgivelsesside**: Besøg [Hagicode GitHub udgivelser](https://github.com/Hagicode-org/releases) for at downloade den seneste version
2. **Officiel hjemmeside**: Besøg [Hagicode hjemmeside](https://hagicode.com/) for at hente links til download

### Vælg den rigtige version

Vælg det tilsvarende installationsprogram til dit operativsystem:
- **Windows**: Download `.exe` installatør
- **macOS**: Download `.dmg` installatør
- **Linux**: Download `.AppImage` eller `.tar.gz` installationspakke

:::note[Systemkrav]
- **Windows**: Windows 10 eller nyere version (64-bit)
- **macOS**: macOS 10.15 eller nyere
- **Linux**: Almindelige Linux-distributioner (Ubuntu 20.04+, Fedora 33+ osv.)
:::

:::tip[Standard forudsætninger]
For det seneste HagiCode lokale installationsflow behøver du normalt ikke at udarbejde en separat `.NET` køretid. Desktop bundter allerede den kørselstid, der kræves til standardflowet, og initialiserer den understøttede lokale SQLite-sti som standard.
:::

## Installations- og startnoter

:::forsigtig[macOS Gatekeeper-meddelelse]
Hvis du allerede har installeret appen til standardstien `/Applications/Hagicode.app` men macOS viser "appen er beskadiget", "kan ikke åbnes" eller en lignende Gatekeeper-advarsel ved første lancering, den sædvanlige årsag er `com.apple.quarantine` egenskab, som macOS tilføjer til apps, der er downloadet fra internettet, ikke en fejl i selve Hagicode Desktop-installationsprogrammet.

Kør følgende kommando i Terminal. Det kræver administratorrettigheder og fjerner rekursivt karantæneattributten fra appen i standardinstallationsstien:

```bash
sudo xattr -dr com.apple.quarantine /Applications/Hagicode.app
```

Når kommandoen er færdig, skal du genstarte Hagicode Desktop.
:::

## Kør Desktop

Efter download skal du åbne installationsprogrammet eller applikationspakken og følge instruktionerne på skærmen. Desktop forbereder runtime i den aktuelt understøttede lokale sti, administrerer Hagicode Server og viser fremskridt og status, når netværksdownloads er påkrævet.

:::note[Netværkskrav]
Sørg for, at din netværksforbindelse er tilgængelig. Desktop skal muligvis hente versionsoplysninger eller komponentpakker fra internettet; Hvis en download mislykkes, skal du kontrollere dine netværks-, proxy- eller firewallindstillinger og prøve igen.
:::

## Versionsstyring

Hagicode Desktop understøtter multi-version management, så du frit kan skifte mellem forskellige versioner.

![Versionsstyring](../../img/screenshots/installation/install-desktop/12-version-management/original.png)

### Skift version

1. Åbn Hagicode Desktop
2. Gå til siden "Versionsstyring".
3. Vælg den version, du vil skifte
4. Klik på knappen "Skift version".
5. Vent på, at skiftet er fuldført

:::note[Bemærkninger til versionsskift]
- Skift af version vil automatisk genstarte Hagicode Server
- Sørg for at gemme alt arbejde, før du skifter
- Projektkonfigurationer kan variere mellem versioner
:::

## Start og stop service

Hagicode Desktop giver en enkel måde at administrere Hagicode Server-tjenesten på.

### Start service

På skrivebordsapplikationens startside skal du klikke på knappen "Start Service" for at starte Hagicode Server.

![Start service](../../img/screenshots/installation/install-desktop/13-start-service-homepage/original.png)

### Stop Service

For at stoppe Hagicode Server skal du klikke på knappen "Stop service" på startsiden.

:::tip[Servicestatus]
Desktop-applikationen viser Hagicode Servers kørestatus i realtid, herunder:
- Servicestatus (kører/stoppet)
- Serviceportnummer
- Aktuelle versionsoplysninger
:::

## Fejlfinding

### Installationsfejl

Hvis du støder på problemer under installationen:

1. **Tjek netværksforbindelse**: Sørg for, at du kan få adgang til internettet
2. **Tjek diskplads**: Sørg for, at du har tilstrækkelig diskplads
3. **Kør som administrator**: I Windows skal du højreklikke på installationsprogrammet og vælge "Kør som administrator"
4. **Se logfiler**: Installationsprogrammet genererer detaljerede logfiler, der kan hjælpe med at diagnosticere problemer

### Afhængighedsinstallationsfejl

Hvis afhængighedsinstallationen mislykkes:

1. **Tjek systemkompatibilitet**: Sørg for, at dit operativsystem opfylder minimumskravene
2. **Tjek netværks- og firewallindstillinger**: Sørg for, at installationsprogrammet kan nå nødvendige downloads og ikke er blokeret af din firewall
3. **Bekræft installationstilladelser**: På Windows kan du prøve at køre installationsprogrammet som administrator, så det har tilstrækkelige tilladelser
4. **Genstart appen og prøv igen**: Luk og genåbn Hagicode Desktop, og prøv derefter installationstrinnet igen
5. **Se logfiler**: Gennemgå detaljerede fejllogfiler i skrivebordsapplikationen

### Servicestartfejl

Hvis Hagicode Server ikke starter:

1. **Tjek portbelægning**: Sørg for, at port 45000 ikke er optaget af andre programmer
2. **Tjek Firewall-indstillinger**: Sørg for, at firewall tillader Hagicode Server-netværksadgang
3. **Genstart applikation**: Prøv at genstarte Hagicode Desktop
4. **Se logfiler**: Se detaljerede fejllogfiler i skrivebordsapplikationen

### Har du brug for mere hjælp?

Hvis du støder på problemer, der ikke er dækket her:

1. Tjek [GitHub-problemer](https://github.com/HagiCode-org/site/issues) for lignende problemer
2. Besøg vores [Fællesskabsforum](https://github.com/HagiCode-org/site/discussions) om hjælp
3. Indsend et nyt problem med en detaljeret beskrivelse af dit problem
