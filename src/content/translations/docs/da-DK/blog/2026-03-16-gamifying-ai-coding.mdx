## Hvordan Gamification Design gør AI-kodning sjovere

> Traditionelle AI-kodningsværktøjer er faktisk ret kraftfulde; de mangler bare lidt varme. Da vi byggede HagiCode, tænkte vi: hvis vi alligevel skal skrive kode, hvorfor så ikke gøre det til et spil?

## Baggrund

Enhver, der har brugt en AI-kodningsassistent, har sikkert haft denne oplevelse: Først føles det frisk og spændende, men efter et stykke tid begynder det at føles, som om der mangler noget. Værktøjet i sig selv er kraftfuldt, i stand til kodegenerering, autofuldførelse og fejlrettelser, men... det føles ikke særlig varmt, og med tiden kan det blive ensformigt og kedeligt.

Det alene er nok til at få dig til at spekulere på, hvem der ønsker at stirre på et koldt, upersonligt værktøj hver dag.

Det er lidt ligesom at spille et spil. Hvis alt, du gør, er at afslutte en opgaveliste, uden karaktervækst, ingen præstationslåsninger og ingen teamkoordinering, holder det hurtigt op med at være sjovt. Smukke ting og mennesker behøver ikke at være besat for at blive værdsat; deres skønhed er nok alene. Programmeringsværktøjer tilbyder ikke engang den slags skønhed, så det er nemt at miste modet.

Vi stødte på netop dette problem under udviklingen [HagiCode](https://hagicode.com). Som en multi-AI-assistent samarbejdsplatform skal HagiCode holde brugerne engageret på lang sigt. Men i virkeligheden er selv et godt værktøj svært at holde fast i, hvis det mangler nogen følelsesmæssig forbindelse.

For at løse dette smertepunkt tog vi en dristig beslutning: Gør programmering til et spil. Ikke den overfladiske slags med et simpelt pointleaderboard, men en ægte rollespilsoplevelse. Virkningen af ​​denne beslutning kan være endnu større, end du forestiller dig.

Folk har jo brug for lidt ritual i deres liv.

## Om HagiCode

Idéerne i denne artikel kommer fra vores praktiske erfaring med [HagiCode](https://hagicode.com) projekt. HagiCode er en multi-AI-assistent-samarbejdsplatform, der understøtter Claude Code, Codex, Copilot, OpenCode og andre AI-assistenter, der arbejder sammen. Hvis du er interesseret i multi-AI-samarbejde eller gamified programmering, så besøg [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site) at lære mere.

Der er ikke noget særligt mystisk ved det. Vi gjorde simpelthen programmering til et eventyr.

## Hvorfor vælge Gamification

Essensen af gamification er ikke bare at "tilføje en rangliste." Det handler om at opbygge et komplet incitamentssystem, så brugerne kan mærke vækst, præstationer og social anerkendelse, mens de udfører opgaver.

HagiCodes gamification-design kredser om én kerneidé: hver AI-assistent er en "Hero", og brugeren er kaptajnen for dette Hero-team. Du leder disse helte til at erobre forskellige "Dungeons" (programmeringsopgaver). Undervejs får Heroes erfaring, stiger i niveau, låser op for evner, og dit hold tjener også præstationer.

Dette er ikke en gimmick. Det er et design baseret på menneskelig adfærdspsykologi. Når opgaver gives mening og fremskridtsfeedback, øges folks engagement og vedholdenhed markant.

Som det gamle ordsprog siger: "Denne følelse kan blive et minde, selvom den på det tidspunkt efterlod os forvirrede." Vi bringer den følelsesmæssige oplevelse ind i værktøjet, så programmering er ikke længere bare at skrive kode, men en rejse, der er værd at huske.

## Hero Character System

Hero er kernekonceptet i HagiCodes gamification-system. Hver helt repræsenterer en AI-assistent. For eksempel er Claude Code en helt, og Codex er også en helt.

### The Three Hero Slots

A Hero har tre udstyrspladser, og designet er overraskende elegant:

1. **CLI slot (hovedklasse)**: Bestemmer heltens basisevne, såsom om det er Claude Code eller Codex
2. **Model slot (sekundær klasse)**: Bestemmer, hvilken model der bruges, såsom Claude 4.5 eller Claude 4.6
3. **Stilplads (stil)**: Bestemmer heltens adfærdsstil, såsom "Fengluo Strategist" eller en anden stilart

Kombinationen af disse tre slots skaber unikke Hero-konfigurationer. Ligesom udstyrsbygning i spil, vælger du den rigtige opsætning baseret på opgaven. Når alt kommer til alt, er det, der passer dig bedst, det, der betyder mest. Livet ligner hinanden: mange veje fører til Rom, men nogle er mere glatte end andre.

### Heltenes vækstsystem

Hver helt har sit eget XP og niveau:

```typescript
type HeroProgressionSnapshot = {
  currentLevel: number;                    // Current level
  totalExperience: number;                 // Total experience
  currentLevelStartExperience: number;     // Experience at the start of the current level
  nextLevelExperience: number;             // Experience required for the next level
  experienceProgressPercent: number;       // Progress percentage
  remainingExperienceToNextLevel: number;  // Experience still needed for the next level
  lastExperienceGain: number;              // Most recent experience gained
  lastExperienceGainAtUtc?: string | null; // Time when experience was gained
};
```

Niveauer er opdelt i fire trin, og hvert trin har et fordybende navn:

```typescript
export const resolveHeroProgressionStage = (level?: number | null): HeroProgressionStage => {
  const normalizedLevel = Math.max(1, level ?? 1);
  if (normalizedLevel <= 100) return 'rookieSprint';      // Rookie sprint
  if (normalizedLevel <= 300) return 'growthRun';         // Growth run
  if (normalizedLevel <= 700) return 'veteranClimb';      // Veteran climb
  return 'legendMarathon';                                // Legend marathon
};
```

Fra "rookie" til "legende" giver denne vækststi brugerne en klar følelse af retning og præstation. Det afspejler personlig vækst i livet, fra forvirring til modenhed, kun gjort mere håndgribeligt her.

### Oprettelse af en brugerdefineret helt

For at skabe en helt skal du konfigurere tre slots:

```typescript
const heroDraft: HeroDraft = {
  name: 'Athena',
  icon: 'hero-avatar:storm-03',
  description: 'A brilliant strategist',
  executorType: AIProviderType.CLAUDE_CODE_CLI,
  slots: {
    cli: {
      id: 'profession-claude-code',
      parameters: { /* CLI-related parameters */ }
    },
    model: {
      id: 'secondary-claude-4-sonnet',
      parameters: { /* Model-related parameters */ }
    },
    style: {
      id: 'fengluo-strategist',
      parameters: { /* Style-related parameters */ }
    }
  }
};
```

Hver helt har en unik avatar, beskrivelse og professionel identitet, som giver, hvad der ellers ville være en kold AI-assistent, mere personlighed og varme. Når alt kommer til alt, hvem vil gerne arbejde med et værktøj, der ikke har nogen karakter?

## Dungeon System

Et "Dungeon" er et klassisk spilkoncept, der repræsenterer en udfordring, der kræver, at et hold rydder. I HagiCode er hver arbejdsgang et Dungeon.

### Hvordan fangehuller er organiseret

Dungeon organiserer arbejdsgange i forskellige "Dungeons":

- **Proposal generation dungeon**: Ansvarlig for at generere tekniske forslag
- **Proposal execution dungeon**: Ansvarlig for udførelse af opgaver i forslag
- **Proposal archive dungeon**: Ansvarlig for organisering og arkivering af færdige forslag

Hvert fangehul har sin egen kaptajnhelt, og kaptajnen bliver automatisk valgt som den først aktiverede helt.

Dette er egentlig bare arbejdsdeling, som i hverdagen, undtagen forvandlet til en spilmekaniker.

### Team samarbejdsmekanisme

Du kan konfigurere forskellige Hero squads til forskellige fangehuller:

```typescript
const dungeonRoster: HeroDungeonRoster = {
  scriptKey: 'proposal.generate',
  displayName: 'Proposal Generation',
  members: [
    { heroId: 'hero-1', name: 'Athena', executorType: 'ClaudeCode' },
    { heroId: 'hero-2', name: 'Apollo', executorType: 'Codex' }
  ]
};
```

For eksempel kan du bruge Athena til at generere forslag, fordi den er god til strategi, og Apollo til at implementere kode, fordi den er god til at eksekvere. På den måde kan enhver helt spille efter sine styrker. Det er som at danne et band: hver person har et instrument, og sammen skaber de noget smukt.

### Dungeon Flow Control

Dungeon bruger fast `scriptKey` værdier til at identificere forskellige arbejdsgange:

```typescript
// Script keys map to different workflows
const dungeonScripts = {
  'proposal.generate': 'Proposal Generation',
  'proposal.execute': 'Proposal Execution',
  'proposal.archive': 'Proposal Archive'
};
```

Opgavetilstandsflowet er: `queued` (venter) -> `dispatching` (ved at blive tildelt) -> `dispatched` (tildelt). Hele processen er automatiseret og kræver ingen manuel indgriben. Det er også en del af vores dovne side, for hvem har lyst til at styre det her i hånden?

## XP og Level System

XP er kernefeedback-mekanismen i gamification-systemet. Brugere opnår XP ved at udføre opgaver, XP stiger helte i niveau, og niveauer op låser nye evner op, hvilket danner en positiv feedback-loop.

### Måder at få XP

I HagiCode kan XP optjenes gennem følgende aktiviteter:

- Fuldfører kodeudførelse
- Opkald til værktøjer
- Generering af forslag
- Sessionsstyringsoperationer
- Projektdrift

Hver gang en gyldig handling er fuldført, vinder den tilsvarende helt XP. Ligesom vækst i livet tæller hvert skridt, kun her er væksten kvantificeret.

### Realtidsvisualisering af fremskridt

XP og niveaufremskridt visualiseres i realtid:

```typescript
type HeroDungeonMember = {
  heroId: string;
  name: string;
  icon?: string | null;
  executorType: PCode_Models_AIProviderType;
  currentLevel?: number;                    // Current level
  totalExperience?: number;                 // Total experience
  experienceProgressPercent?: number;       // Progress percentage
};
```

Brugere kan altid se hver enkelt helts niveau og fremskridt, og at øjeblikkelig feedback er nøglen til gamification-design. Folk har brug for feedback, hvordan ville de ellers vide, at de forbedrer sig?

## Præstationssystem

Præstationer er et andet vigtigt element i gamification. De giver langsigtede mål og milepælsdrevet tilfredshed.

### Præstationstyper

HagiCode understøtter flere typer præstationer:

- **Kodegenereringspræstationer**: Generer X linjer kode, generer Y-filer
- **Sessionsstyringspræstationer**: Gennemfør Z-samtaler
- **Projektdriftresultater**: Arbejde på tværs af W-projekter

Disse præstationer er virkelig som milepæle i livet, bortset fra at vi har forvandlet dem til en spilmekaniker.

### Præstationsstater

Præstationer har tre tilstande:

```typescript
type AchievementStatus = 'unlocked' | 'in-progress' | 'locked';
```

De tre stater har klare visuelle skel:

- **Ulåst**: Guldgradient med en halo-effekt
- **I gang**: Blå puls animation
- **Låst**: Grå, med oplåsningsbetingelser vist

Hver præstation viser tydeligt dens triggertilstand, så brugerne ved, hvad de skal gøre. Når folk føler sig fortabt, hjælper lidt vejledning altid.

### Fejringseffekt på oplåsning

Når en præstation låses op, udløses en fejringsanimation. Den form for positiv forstærkning giver brugerne den tilfredsstillende følelse af "jeg gjorde det" og motiverer dem til at fortsætte. Små belønninger i livet fungerer på samme måde: de kan være små, men lykken kan vare længe.

## Kamprapport Daglig kamprapport

Battle Report er en af HagiCodes signaturfunktioner. Ved slutningen af ​​hver dag genererer den en fuldskærmsrapport i kampstil.

### Rapportér indhold

Kamprapport viser følgende oplysninger:

```typescript
type HeroBattleReport = {
  reportDate: string;
  summary: {
    totalHeroCount: number;        // Total number of Heroes
    activeHeroCount: number;       // Number of active Heroes
    totalBattleScore: number;      // Total battle score
    mvp: HeroBattleHero;           // Most valuable Hero
  };
  heroes: HeroBattleHero[];        // Detailed data for all Heroes
};
```

- Samlet holdscore
- Antal aktive helte
- Antal værktøjsopkald
- Samlet arbejdstid
- MVP (Most Valuable Hero)
- Detaljeret kort for hver helt

### MVP Highlight Display

MVP'en er dagens bedst præsterende helt og er fremhævet i rapporten. Dette er ikke kun datastatistik, men en form for ære og anerkendelse. Når alt kommer til alt, hvem ønsker ikke at blive anerkendt?

### Detaljerede heltekort

Hvert heltekort inkluderer:

- Niveau fremskridt
- XP vundet
- Antal henrettelser
- Brugstid

Disse metrics hjælper brugerne med tydeligt at forstå, hvordan teamet klarer sig. At se resultaterne af din egen indsats er i sig selv tilfredsstillende.

## Teknisk implementering

HagiCodes gamification-system bruger en moderne teknologistak og designmønstre. Der er ikke noget særligt magisk over det; vi har bare valgt værktøjer, der passer til jobbet.

### Valg af teknologistak

```typescript
// React + TypeScript for the frontend
import React from 'react';

// Framer Motion for animations
import { AnimatePresence, motion } from 'framer-motion';

// Redux Toolkit for state management
import { useAppDispatch, useAppSelector } from '@/store';

// shadcn/ui for UI components
import { Dialog, DialogContent } from '@/components/ui/dialog';
```

Framer Motion håndterer alle animationseffekter, shadcn/ui leverer de grundlæggende UI-komponenter, og Redux Toolkit styrer den komplekse gamification-tilstand. Gode ​​værktøjer giver godt arbejde.

### Gamified UI Design System

HagiCode bruger en Glassmorphism + Tech Dark designstil:

```css
/* Primary gradient */
background: linear-gradient(135deg, #22C55E 0%, #25c2a0 50%, #06b6d4 100%);

/* Glass effect */
backdrop-filter: blur(12px);

/* Glow effect */
background: radial-gradient(circle at center, rgba(34, 197, 94, 0.15) 0%, transparent 70%);
```

Den grønne gradient kombineret med glasmorfisme skaber en teknisk, futuristisk atmosfære. Visuel skønhed er også en del af brugeroplevelsen.

### Animationseffekter

Framer Motion bruges til at skabe glatte indgangsanimationer:

```typescript
<motion.div
  animate={{ opacity: 1, y: 0 }}
  initial={{ opacity: 0, y: 18 }}
  transition={{ duration: 0.35, ease: 'easeOut', delay: index * 0.08 }}
  className="card"
>
  {/* Card content */}
</motion.div>
```

Hvert kort kommer ind efter hinanden med en forsinkelse på 0,08 sekunder, hvilket skaber en flydende visuel effekt. Glat animation forbedrer oplevelsen. Den del er svær at argumentere med.

### Datapersistens

Gamification-data lagres ved hjælp af Grain-lagringssystemet for at sikre tilstandskonsistens. Selv finkornede data som akkumuleret Hero XP kan bevares nøjagtigt. Ingen ønsker at miste den erfaring, de har arbejdet hårdt for at få.

## Praktisk vejledning

### Skab din første helt

At skabe din første helt er faktisk ret simpelt:

1. Gå til Hero-administrationssiden
2. Klik på knappen "Opret helt".
3. Konfigurer de tre slots (CLI, Model, Style)
4. Giv helten et navn og en beskrivelse
5. Gem det, og din første helt er født

Det er som at møde en ny ven: I giver dem et navn, lærer, hvad der gør dem specielle, og tager derefter ud på et eventyr sammen.

### Byg et Dungeon Team

At bygge et team er også enkelt:

1. Gå til Dungeon-administrationssiden
2. Vælg det fangehul, du vil konfigurere, såsom "Proposal Generation"
3. Vælg medlemmer fra din helteliste
4. Systemet vælger automatisk den første aktiverede helt som kaptajn
5. Gem konfigurationen

Dette er simpelthen processen med at danne et team, ligesom at bygge et team i det virkelige liv, hvor alle har deres egen rolle.

### Se den daglige rapport

I slutningen af hver dag kan du se dagens kamprapport:

1. Klik på knappen "Battle Report".
2. Se dagens arbejdsresultater i fuldskærmsvisning
3. Tjek MVP'en og de detaljerede data for hver helt
4. Del det med teammedlemmer, hvis du vil

Dette er også en slags ritual, en måde at se, hvor meget du gør i dag, og hvor langt du stadig er fra dit mål.

## Noter og bedste praksis

### Optimering af ydeevne

Brug `React.memo` for at undgå unødvendige gengivelser:

```typescript
const HeroCard = React.memo(({ hero }: { hero: HeroDungeonMember }) => {
  // Component implementation
});
```

Ydeevne betyder også noget. Ingen ønsker at bruge et laggy værktøj.

### Bevægelse kan nedbrydes på en yndefuld måde

Registrer brugerens bevægelsespræferenceindstillinger og giv en forenklet oplevelse for bevægelsesfølsomme brugere:

```typescript
const prefersReducedMotion = useReducedMotion();
const duration = prefersReducedMotion ? 0 : 0.35;
```

Ikke alle kan lide animation, og det er en del af godt design at respektere brugernes præferencer.

### Bagudkompatibilitet

Behold `legacyIds` for at understøtte migrering fra ældre versioner:

```typescript
type HeroDungeonMember = {
  heroId: string;
  legacyIds?: string[];  // Supports legacy ID mapping
  // ...
};
```

Ingen ønsker at miste data bare på grund af en versionsopgradering.

### Internationaliseringsstøtte

Brug i18n-oversættelsesnøgler til al tekst for at gøre understøttelse af flere sprog let:

```typescript
const displayName = t(`dungeon.${scriptKey}`, { defaultValue: displayName });
```

Sproget bør aldrig være en hindring for at bruge produktet.

## Resumé

Gamification er ikke bare et simpelt point leaderboard, men et komplet incitamentsystem. Gennem Hero-systemet, Dungeon-systemet, XP- og niveausystemet, præstationssystemet og Battle Report forvandler HagiCode programmeringsarbejde til en heroisk rejse fuld af eventyr.

Kerneværdien af dette system ligger i:

- **Følelsesmæssig forbindelse**: Giver kolde AI-assistenter personlighed
- **Positiv feedback**: Hver handling producerer øjeblikkelig feedback
- **Langsigtede mål**: Niveauer og præstationer giver en vækststi
- **Teamidentitet**: En følelse af samarbejde inden for Dungeon-teams
- **Ære og anerkendelse**: Kamprapport og MVP-showcases

Gamification design gør programmering ikke længere kedelig, men et interessant eventyr. Mens de udfører kodningsopgaver, oplever brugerne også det sjove ved karaktervækst, teamsamarbejde og opnåelse af præstationer, hvilket forbedrer fastholdelse og aktivitet.

I sin kerne er programmering allerede en skabelseshandling. Vi har lige gjort den kreative proces lidt sjovere.
