## Guide til oprettelse af en border Light Sweep-animationseffekt

> Hvordan opbygger du det vigtige element, som brugere bemærker med et øjeblik ved hjælp af ren CSS? Det er faktisk ikke så svært. Tricket er bare at tage en lidt rundkørsel. I denne artikel vil jeg guide dig gennem, hvordan du bygger en border light sweep-animation fra bunden, og jeg vil også dele et par af de faldgruber, vi stødte på, mens vi byggede HagiCode.

## Baggrund

Hvis du arbejder på frontend, har du sikkert haft denne oplevelse før: en produktchef går over med det "det her er helt sikkert simpelt" udtryk og siger: "Kan vi tilføje en slags speciel effekt til denne kørende opgave, så brugerne kan få øje på det med det samme?"

Du siger sikkert, vi kan bare ændre kantfarven. Så ryster de på hovedet med det blik, der siger: "Du fatter det ikke." De svarer: "Det er ikke indlysende nok. Jeg vil have den slags effekt, hvor lys løber rundt om grænsen, som i en sci-fi-film."

På det tidspunkt begynder du måske at spekulere på, hvordan man bygger det. Lærred? SVG? Eller kan CSS klare det på egen hånd? Når alt kommer til alt, vil ingen indrømme, at de ikke ved hvordan.

I moderne webapplikationer er border light sweep-animationer faktisk meget almindelige. De bruges hovedsageligt i nogle få scenarier som disse:

- **Statusindikatorer**: Markering af igangværende opgaver eller aktive elementer
- **Visuelt fokus**: Fremhævelse af vigtige indholdsområder
- **Mærkeforbedring**: Skaber en slank, moderne, teknologisk fremadrettet visuel stil
- **Sæsonbestemte temaer**: Opbygning af en festlig atmosfære til særlige lejligheder

Vi stødte på præcis dette krav, mens vi byggede [HagiCode](https://hagicode.com). Brugerne havde brug for at se med et øjeblik, hvilke sessioner der kørte, og hvilke forslag der i øjeblikket blev behandlet. Vi prøvede flere forskellige tilgange. Nogle stier var mere jævne, nogle var lidt mere snoede. I sidste ende blev vi enige om en ret moden implementeringsstrategi.

## Kerneimplementeringsidéer

Efter at have analyseret HagiCode-kodebasen opsummerede vi flere kerneimplementeringsmønstre nedenfor. Hver af dem passer til et andet scenarie, eller med andre ord, hver enkelt eksisterer af en grund.

### 1. Roterende glød med en konisk gradient (mest almindelig)

Dette er den klassiske måde at implementere en border light sweep-effekt på. Kerneideen er at bruge CSS `conic-gradient` for at skabe en konisk gradient, og drej den derefter kontinuerligt. Som en gadelygte, der tænder om natten, bliver den bare ved med at cirkle.

**Nøgleelementer:**
- Brug `::before` pseudo-element for at skabe glødslaget
- Brug `conic-gradient` for at definere gradientfarvefordelingen
- Brug `::after` pseudo-element til at maskere midterområdet (valgfrit)
- Brug `@keyframes` at implementere rotationsanimationen

### 2. Sideglødelinje

Dette fungerer godt for statusindikatorer i listeelementer. Du opretter en tynd glødende linje på den ene side af elementet i stedet for at animere hele kanten. Nogle gange er lidt lys nok. Du behøver ikke at oplyse hele verden.

**Nøgleelementer:**
- Et tyndt, absolut positioneret linjeelement
- Brug `box-shadow` for at skabe glødeffekten
- Brug `scale` og `opacity` til en vejrtrækningsanimation

### 3. Glødende baggrund med box-skygge

Hvis du ikke har brug for den fulde sweep-effekt og bare vil have en blød baggrundsglød, lag flere lag `box-shadow` værdier er nok. Nogle gange er den nemmeste løsning den bedre.

### 4. Tilgængelighedsstøtte

Denne del er let at overse, men den er ekstremt vigtig. Hver animation skal tage højde for `prefers-reduced-motion` medieforespørgsel og give et statisk alternativ til brugere, der ikke ønsker animation. Ikke alle nyder konstant bevægelse, og det er vigtigt at respektere den præference.

## Implementeringsmuligheder

### Mulighed 1: Roterende konisk-gradientkant (anbefales)

Dette er den mest komplette implementering af den fejende kantlyseffekt, og det er også den mulighed, vi oftest bruger i HagiCode. Når alt kommer til alt, hvis noget fungerer godt, hvorfor så udskifte det?

```css
/* Parent container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotating glow layer */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Mask layer (optional, for creating a hollow border effect) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Princippet bag denne mulighed er ret simpelt: Opret et pseudo-element, der er større end den overordnede container, tegn en konisk gradient på det, og drej det kontinuerligt. Den overordnede container bruger `overflow: hidden`, så kun lyset, der passerer rundt om grænsen, forbliver synligt. Det er lidt som at se en gadelygte gennem et vindue. Man ser kun den lille skive, der går forbi.

### Mulighed 2: Forenklet roterende lyskant

Hvis du ikke har brug for den fulde effekt, inkluderer HagiCode også en lettere version i værktøjsklassen. Nogle gange er den enklere tilgang virkelig bedre.

```css
/* Rotating light border utility class */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Accessibility support */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Læg mærke til `will-change: transform` her. Det fortæller browseren, "Dette element vil blive ved med at ændre sig," så browseren kan forberede nogle optimeringer på forhånd og holde animationen mere jævn. At forberede sig på forhånd er normalt bedre end at scramble i sidste øjeblik.

### Mulighed 3: Sideglødelinje

Dette er især velegnet til statusindikatorer for listeelementer, og det er præcis, hvad HagiCode-sessionslisten bruger. En tynd streg kan stadig skille sig ud blandt mange varer. Det føles som en livslektion på sin egen måde.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Dette bruger `isolation: isolate` at skabe en ny stablingskontekst, så stoler på `z-index` for at kontrollere visningsrækkefølgen for hvert lag. `pointer-events: none` er også væsentlig. Ellers ville pseudo-elementet blokere brugerklik. Nogle ting kan se pæne ud, men de skal alligevel ikke komme i vejen.

### Mulighed 4: Pak den ind i en React-komponent

Hvis dit projekt bruger React, kan du pakke denne logik ind i en komponent, især tilgængelighedshåndteringen. Skriv det én gang, brug det mange gange. Det er hele pointen.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Det matchende CSS-modul:

```css
/* GlowBorder.module.css */

/* Animated version */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Static version (accessibility) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Den `useReducedMotion` krog fra `framer-motion` registrerer automatisk brugerens systempræference. Hvis brugeren har aktiveret reduceret bevægelse, vender den tilbage `true`, og komponenten viser i stedet den statiske version. At respektere brugerens præferencer betyder mere end at fremtvinge en prangende effekt.

## Praktiske erfaringer

Dette er nogle af de erfaringer, vi lærte, mens vi byggede HagiCode. Man kan også kalde dem kampar. Forhåbentlig hjælper de dig med at undgå nogle omveje.

### 1. Tema variabelt system

CSS-variabler gør understøttelse af flere temaer særlig bekvem. Ingen ønsker at redigere en bunke kode, hver gang temaet ændres.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Usage */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

På den måde kræver skift af temaer kun at ændre klassen på `html` element, og hver animationsfarve opdateres automatisk. En kodebase, to stilarter. Det er præcis, hvad vi ønsker.

### 2. Performance optimering

**Brug `will-change` for at antyde, at browseren skal optimere:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Fortæl browseren på forhånd, så hjælper det dig med at optimere. Mange ting i livet fungerer bedre med lidt forberedelse.

**Undgå at bruge komplekse boksskygger på store elementer:**

```css
/* Not ideal - using a blurred shadow on a large element */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Better - use a pseudo-element to limit the glowing area */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Vi testede dette i HagiCode. Tilføjelse af en sløret skygge direkte til et stort kort faldt scrolling frame rates under 30fps. Skift til et pseudo-element bragte tingene tilbage til en stabil 60fps. Brugerne kan absolut mærke den forskel.

### 3. Tilgængelighed

Du bør virkelig ikke springe dette over. Nogle brugere finder animation svimlende eller distraherende, og respekt for deres præferencer er en del af opbygningen af ​​et godt produkt. Smukke ting skal ikke påtvinges alle.

**CSS medieforespørgsel:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Provide a static fallback */
    opacity: 1;
  }
}
```

**Opdag brugerpræference i React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Styring af intensitetsniveau

Token-gennemløbsindikatoren i HagiCode viser forskellige glødfarver baseret på realtidsgennemstrømning, og dette implementeres dynamisk. Forskellige tilstande bør udtrykkes forskelligt.

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Ting at passe på

Der er stadig et par detaljer, der er værd at være opmærksomme på, for når du opdager disse problemer på den hårde måde, er det allerede for sent.

| Ting at passe på | Forklaring |
|---------|------|
| **z-indeksstyring** | Glødelaget skal bruge en passende `z-index` så det forstyrrer ikke indholdsinteraktionen |
| **pointer-begivenheder** | Glød-pseudo-elementet skal indstilles `pointer-events: none` |
| **Grænseoverløb** | Forældrebeholderen har brug for `overflow: hidden`, eller du skal justere pseudoelementstørrelsen |
| **Ydeevnepåvirkning** | Komplekse animationer kan skade ydeevnen på mobile enheder, så test omhyggeligt |
| **Mørk tilstand** | Sørg for, at glødfarven forbliver tydeligt synlig på mørke baggrunde |
| **Temaskifte** | Brug CSS-variabler, så animationsfarver opdateres korrekt, når temaet ændres |

### 6. Debugging tips

Pseudo-elementer kan være lidt svære at finde i udviklerværktøjer, så du midlertidigt kan tilføje en kant for at kontrollere positionen.

```css
/* Temporarily show pseudo-element boundaries for debugging */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Når du er færdig med at placere den, skal du huske at kommentere eller fjerne den linje. Ellers kan produktionen blive akavet ret hurtigt. Nogle ting er bedre tilbage i udviklingen.

## Resumé
