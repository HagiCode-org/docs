Hvis du vil installere Hermes og begynde at bruge det, er den korteste vej i virkeligheden kun tre trin:

1. Kør den officielle installationskommando
2. Start CLI i din terminal med `hermes`
3. Hvis du vil fortsætte med at bruge det i Feishu, så konfigurer `hermes gateway setup`

Denne artikel forsøger ikke at forklare alle Hermes-kapaciteter på én gang. I stedet hjælper det dig med at fuldføre den vigtigste begynderløkke først: Installer den, få den til at køre, begynd at bruge den, og tilslut den derefter til et af de mest almindelige messaging-platformsscenarier.

## Hvad Hermes er, og hvem det er til

Hermes Agent er en AI-agent, som du kan bruge enten fra en lokal terminal eller gennem en gateway til meddelelsesplatformen.

For de fleste udviklere har den to fælles indgangspunkter:

- **CLI**: Type `hermes` i din terminal for at gå direkte ind i den interaktive grænseflade.
- **Beskedgateway**: Kør `hermes gateway`, og chat derefter med det fra platforme som Feishu, Telegram, Discord og Slack.

Hvis dit mål lige nu blot er at komme hurtigt i gang, skal du ikke vende rækkefølgen om. Start med denne sti i stedet:

- Installer Hermes først
- Bekræft først, at det virker fra CLI'en
- Beslut derefter, om du vil tilslutte en meddelelsesplatform

Dette gør problemer lettere at diagnosticere og er mere velegnet til folk, der bruger Hermes for første gang.

## Hvad du skal vide, før du installerer Hermes

Ifølge Hermes README understøtter den officielle hurtiginstallationssti disse miljøer:

- Linux
- macOS
- WSL2
- Android via Termux

### Noter til Windows-brugere

Hermes understøtter i øjeblikket ikke at køre direkte på native Windows. Hvis du bruger Windows, er den anbefalede fremgangsmåde at installere WSL2 først og derefter køre installationskommandoen inde i WSL2.

Det er bedst at gøre dette klart i begyndelsen, fordi mange installationsfejl ikke er forårsaget af selve kommandoen, men ved at bruge et ikke-understøttet runtime-miljø.

## Sådan installeres Hermes

Den hurtige installationskommando i Hermes README er:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Denne kommando kører det officielle installationsscript og håndterer platformsspecifikke initialiseringstrin.

### Genindlæs din skal efter installationen

Når installationen er færdig, skal du først genindlæse dit shell-miljø. Den mest almindelige kommando er:

```bash
source ~/.bashrc
```

Hvis du bruger zsh, kan du bruge:

```bash
source ~/.zshrc
```

### Sådan bekræfter du, at Hermes er installeret korrekt

Den mest direkte måde at kontrollere er at køre:

```bash
hermes
```

Hvis du ønsker yderligere bekræftelse på, at din konfiguration og afhængigheder fungerer, kan du også køre:

```bash
hermes doctor
```

`hermes doctor` er især nyttig i disse situationer:

- Kommandoen opfører sig unormalt efter installationen
- Modelkonfiguration mislykkes
- Gatewayen starter ikke
- Du er ikke sikker på, om dine miljøafhængigheder er fuldstændige

## Sådan begynder du at bruge Hermes for første gang

### Start med CLI

Hvis du blot vil bekræfte så hurtigt som muligt, at Hermes virker, er den enkleste metode:

```bash
hermes
```

Dette lancerer den interaktive Hermes CLI. For førstegangsbrugere af Hermes er det også det mest anbefalede udgangspunkt, fordi du kan verificere de vigtigste ting først:

- Om kommandoen faktisk er tilgængelig
- Om den aktuelle modelkonfiguration fungerer korrekt
- Om terminalværktøjskæden fungerer korrekt
- Om interaktionsstilen matcher det, du har brug for

### Disse kommandoer er nok til din første opsætningsrunde

Hermes README lister adskillige højfrekvente kommandoer, og tilsammen danner de en praktisk førstegangsvej:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Hvis du ikke er sikker på, hvad hver enkelt gør, så husk dem sådan her:

- `hermes model`: vælg eller skift model
- `hermes tools`: se og konfigurer aktuelt tilgængelige værktøjer
- `hermes config set`: ændre specifikke konfigurationselementer
- `hermes setup`: Kør den fulde initialiseringsguide én gang
- `hermes update`: Opdater Hermes
- `hermes doctor`: fejlfinding af problemer

For begyndere er den mest praktiske rækkefølge normalt:

1. Løb `hermes model` først
2. Hvis du vil konfigurere alle almindelige muligheder på én gang, så kør `hermes setup`

## De to mest almindelige måder at bruge Hermes på

### 1. Brug Hermes i terminalen som daglig udviklingsassistent

CLI-tilstand passer godt til disse scenarier:

- Stil spørgsmål direkte, mens du skriver kode lokalt
- Undersøg projekter, rediger filer og kør kommandoer
- Foretag engangsfejlretning eller gennemgang
- Samarbejd løbende i den aktuelle arbejdsmappe

Dens største fordel er, at det er den korteste vej: ingen ekstra platformsintegration, ingen bot-konfiguration at håndtere på forhånd, og det er den bedste måde at opbygge dit første sæt brugsvaner.

### 2. Brug Hermes gennem en meddelelsesplatform

Hvis du vil chatte med Hermes på platforme som Feishu, Telegram eller Discord, skal du bruge beskedgatewayen.

De mest almindelige indtastningskommandoer er:

```bash
hermes gateway setup
hermes gateway
```

Specifikt:

- `hermes gateway setup` bruges til interaktiv platformkonfiguration
- `hermes gateway` bruges til at starte gateway-processen

Ifølge den officielle dokumentation er gatewayen en samlet baggrundsproces, der forbinder dine konfigurerede platforme, administrerer sessioner og håndterer funktioner såsom cron-job.

## Brug af Feishu som et eksempel: hvordan man forbinder Hermes til en meddelelsesplatform

Hvis det meste af dit daglige arbejde foregår i Feishu, så er Feishu/Lark en meget naturlig måde at bruge Hermes på.

### Den mindst mulige integrationsvej

Den officielle dokumentation anbefaler denne indtastningskommando til Feishu/Lark:

```bash
hermes gateway setup
```

Når du har kørt det, skal du blot vælge **Feishu / Lark** i guiden.

Feishu-dokumentationen beskriver to forbindelsestilstande:

- `websocket`: anbefales
- `webhook`: valgfrit

Hvis Hermes kører på din bærbare computer, arbejdsstation eller private server, vha `websocket` først er normalt enklere, fordi du ikke behøver at afsløre en offentlig tilbagekalds-URL.

### Hvis du konfigurerer det manuelt, skal du i det mindste kende disse variabler

Hvis du ikke bruger guiden og skriver konfigurationen manuelt, viser Feishu-dokumentationen disse kernevariabler:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

To af dem fortjener særlig opmærksomhed:

- `FEISHU_ALLOWED_USERS`: anbefales, så ikke alle, der kan nå botten, kan bruge den direkte
- `FEISHU_HOME_CHANNEL`: lader dig foruddefinere en hjemmechat for at modtage cron-resultater eller standardmeddelelser

### Hvorfor Hermes nogle gange ikke svarer i Feishu gruppechat

Denne detalje er let at gå glip af: I Feishu gruppechat svarer Hermes ikke på hver besked som standard.

Den officielle dokumentation siger tydeligt:

- I direkte beskeder svarer Hermes på beskeder
- I gruppechat skal du eksplicit `@` botten, før den behandler beskeden

Hvis du vil indstille en Feishu-samtale som hjemmekanal, kan du også bruge dette i chatten:

```text
/set-home
```

Eller definer det i konfigurationen på forhånd:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Hermes-kommandoerne, begyndere skal huske først

Uanset om du bruger Hermes i CLI eller på en meddelelsesplatform, er det allerede nok at huske følgende kommandoer til at komme i gang:

- `/new` eller `/reset`: start en ny session
- `/model`: se eller skift model
- `/retry`: Prøv den forrige tur igen
- `/undo`: fortryd den tidligere interaktion
- `/compress`: komprimer konteksten manuelt
- `/help`: se hjælp

Hvis du primært bruger Hermes på en meddelelsesplatform, så husk en mere:

- `/sethome` eller `/set-home`: Indstil den aktuelle chat som hjemmekanal

Disse kommandoer dækker de mest almindelige handlinger på begynderstadiet: genstart, justering, rulle tilbage, kontrol og fortsættelse.

## Ofte stillede spørgsmål

### Kan Windows installere Hermes direkte?

Nej. Den aktuelle officielle dokumentation siger tydeligt, at native Windows ikke understøttes, og WSL2 anbefales.

### Hvad skal jeg gøre, hvis jeg skriver `hermes` gør ingenting efter installationen?

Det er bedst at fejlfinde i denne rækkefølge:

1. Genindlæs først din shell, for eksempel med `source ~/.bashrc`
2. Løb `hermes` igen
3. Hvis det stadig er unormalt, så løb `hermes doctor`

### Hvorfor svarer botten ikke i en Feishu-gruppe?

Tjek først disse tre ting:

- Uanset om du `@` nævnte Hermes i gruppen
- Om `FEISHU_ALLOWED_USERS` begrænser den aktuelle bruger
- Om den aktuelle gruppechatpolitik tillader håndtering af gruppebeskeder

Ifølge den officielle Feishu-dokumentation er eksplicit brug af en @omtale påkrævet i gruppechat-scenarier.

3. Brug `hermes model` og `hermes setup` for at fuldføre den grundlæggende konfiguration
