## Hagicode og GLM-5.1 Multi-CLI Integrationsvejledning

:::note[Bemærkning om tidskontekst]
Denne artikel dokumenterer multi-CLI-integrationsdesignet og modelkortlægningen, som de eksisterede på udgivelsestidspunktet. For det nuværende altid understøttede omfang henvises til [AI Agent CLI installation](/da-DK/related-software-installation/ai-agent-cli/) og den [Installationsvejledning](/da-DK/installation); den `IFlowCli` nævnt her repræsenterer kun en historisk konfiguration på det tidspunkt og er ikke en del af det nuværende aktive supportomfang.
:::

## Baggrund

I Hagicode-projektet kan brugere vælge mellem flere CLI-værktøjer til at drive AI-programmeringsassistenter, herunder Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI og mere. Disse CLI-værktøjer er AI-programmeringsværktøjer til generelle formål i sig selv, men gennem Hagicodes abstraktionslag kan de fleksibelt oprette forbindelse til forskellige AI-modeludbydere.

Zhipu AI (ZAI) leverer en grænseflade, der er kompatibel med Anthropic Claude API, hvilket gør det muligt for disse CLI-værktøjer at bruge indenlandske GLM-seriemodeller direkte. Blandt dem er GLM-5.1 Zhipu's seneste store sprogmodeludgivelse med betydelige forbedringer i forhold til GLM-5.0.

## Hagicodes CLI abstraktionsarkitektur

Hagicode definerer 11 CLI-udbydertyper gennem `AIProviderType` enum, der dækker almindelige AI-programmerings-CLI-værktøjer:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Hver CLI har tilsvarende modelparameterkonfiguration og understøtter `model` og `reasoning` parametre:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],
        [AIProviderType.CodexCli] = ["model", "reasoning"],
        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],
        [AIProviderType.HermesCli] = ["model", "reasoning"],
        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini does not support the reasoning parameter
        // ...
    };
```

## Brug af GLM-5.1 i forskellige CLI'er

### 1. Claude Kode CLI + GLM-5.1

Claude Code CLI er en af Hagicodes kerne-CLI'er og er konfigureret gennem Hero-konfigurationssystemet:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Tilsvarende `HeroEquipmentCatalogItem` konfiguration:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI er den mest fleksible CLI og understøtter specificering af enhver model i `provider/model` format:

**Metode 1: Brug ZAI-udbyderpræfikset**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Metode 2: Brug model-id'et direkte**
```json
{
  "model": "glm-5.1"
}
```

**Metode 3: Frontend-konfigurations-UI**

I `HeroModelEquipmentForm.tsx`, OpenCode CLI har et dedikeret pladsholdertip:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Brugere kan indtaste:
```
zai/glm-5.1
glm-5.1
```

**OpenCode CLI model parsing logik**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI har dedikeret ældre håndteringslogik:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Bemærk**: Codebuddy bevarer speciel håndtering til GLM-5.0 og bruger ikke ældre normalisering:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## Forbedringsfordele ved GLM-5.1

Sammenlignet med GLM-5.0 bringer GLM-5.1 følgende væsentlige forbedringer:

### 1. Bedre ræsonnement kapacitet

Ifølge Zhipu's officielle udgivelsesinformation omfatter forbedringer i GLM-5.1:

- **Større kodeforståelse**: Mere præcis analyse af komplekse kodestrukturer
- **Længere kontekstforståelse**: Understøtter længere samtalekontekst
- **Forbedret værktøjsopkald**: Højere succesrate for MCP-værktøjsopkald
- **Outputstabilitet**: Reducerer tilfældigheder og hallucinationer

### 2. Omfattende multi-CLI-kompatibilitet

GLM-5.1 dækker alle almindelige CLI'er understøttet af Hagicode:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Noter

### 1. API-nøglekonfiguration

Sørg for at `ANTHROPIC_AUTH_TOKEN` miljøvariablen er indstillet korrekt. Det er den nødvendige legitimation for hver CLI at oprette forbindelse til modellen.

### 2. Model tilgængelighed

GLM-5.1 skal aktiveres af den tilsvarende modeludbyder:
- Zhipu AI ZAI-platformen understøtter det som standard
- Alibaba Cloud DashScope kræver muligvis en separat applikation

### 3. OpenCode CLI-format

Ved brug af `provider/model` format, skal du sørge for, at udbyder-id'et er korrekt:
- Zhipu AI: `zai` eller `zhipuai`
- Alibaba Cloud: `aliyun` eller `dashscope`

### 4. Begrundelsesparameter

- `high` anbefales for de bedste kodegenereringsresultater
- Gemini CLI understøtter ikke begrundelsesparameteren og ignorerer denne konfiguration automatisk

## Konklusion

Gennem et samlet abstraktionslag muliggør Hagicode fleksibel integration mellem GLM-5.1 og flere CLI'er. Udviklere kan vælge det CLI-værktøj, der passer bedst til deres præferencer og brugsscenarier, og derefter bruge den nyeste GLM-5.1-model gennem simpel konfiguration.

Som Zhipu's seneste modelversion tilbyder GLM-5.1 klare forbedringer i forhold til GLM-5.0:
- En uafhængig versionsidentifikator uden en ældre byrde
- Stærkere ræsonnement og kodeforståelse
- Bred multi-CLI-kompatibilitet
- Fleksibel konfiguration af ræsonnementniveau

Med de korrekte miljøvariabler og Hero-udstyr konfigureret kan brugere fuldt ud låse op for kraften i GLM-5.1 på tværs af forskellige CLI-miljøer.
