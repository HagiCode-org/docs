## Sådan reproduceres projekter i AI-æraen: Vault, et Cross-Project Persistent Storage System

> Hvordan kan vi i en æra med AI-assisteret udvikling hjælpe AI-assistenter til bedre at forstå vores læringsressourcer? HagiCode-projektet byggede Vault-systemet som et samlet videnlagerabstraktionslag, som AI kan forstå, hvilket i høj grad forbedrer effektiviteten af ​​læring gennem projektreproduktion.

## Baggrund

I AI-æraen ændres den måde, udviklere lærer nye teknologier og arkitekturer på, dybtgående. "Reproducere projekter" - det vil sige dybt at studere og lære af koden, arkitekturen og designmønstrene for fremragende open source-projekter - er blevet en effektiv måde at lære på. Sammenlignet med traditionelle metoder som at læse bøger eller se videoer, hjælper direkte læsning og afvikling af højkvalitets open source-projekter dig med at forstå den virkelige verden ingeniørpraksis meget hurtigere.

Alligevel kommer denne læringsmetode med en del udfordringer.

Læremidler er for spredte. Dine noter kan leve i Obsidian, kodelagre kan være spredt ud over forskellige mapper, og din AI-assistents samtalehistorik bliver endnu en isoleret data-ø. Når du vil have AI til at hjælpe med at analysere et projekt, skal du manuelt kopiere kodestykker og organisere kontekst, hvilket er ret kedeligt.

Hvad der er endnu mere besværligt er den brudte kontekst. AI-assistenter kan ikke få direkte adgang til dine lokale læringsressourcer, så du skal give baggrundsinformation igen i hver samtale. Oven i købet opdateres reproducerede kodelagre hurtigt, manuel synkronisering er tilbøjelig til at fejle, og viden er svær at dele på tværs af flere læringsprojekter.

Grundlæggende kommer alle disse problemer fra "dataøer". Hvis der var et samlet lagerabstraktionslag, der gjorde det muligt for AI-assistenter at forstå og få adgang til alle dine læringsressourcer, ville problemet være løst pænt.

## Om HagiCode

Vault-systemet, der er delt i denne artikel, er præcis den løsning, vi udviklede, mens vi byggede [HagiCode](https://hagicode.com). HagiCode er et AI-kodningsassistentprojekt, og i vores daglige udviklingsarbejde skal vi ofte studere og referere til mange forskellige open source-projekter. For at hjælpe AI-assistenter med bedre at forstå disse læringsressourcer har vi designet Vault, et vedvarende lagringssystem på tværs af projekter.

Denne løsning er allerede blevet valideret i HagiCode i reel brug. Hvis du står over for lignende videnshåndteringsudfordringer, håber jeg, at disse erfaringer kan give inspiration. Når du selv er faldet i et par huller, bør du jo efterlade noget til den næste person.

## Vault system design filosofi

Kerneideen i Vault-systemet er enkel: Skab et samlet videnlagerabstraktionslag, som AI kan forstå. Fra et implementeringsperspektiv har systemet flere nøglekarakteristika.

### Multi-type support

Systemet understøtter fire bokstyper, der hver svarer til et forskelligt brugsscenarie:

```typescript
// folder: general-purpose folder type
export const DEFAULT_VAULT_TYPE = 'folder';

// coderef: a type specifically for reproduced code projects
export const CODEREF_VAULT_TYPE = 'coderef';

// obsidian: integrated with Obsidian note-taking software
export const OBSIDIAN_VAULT_TYPE = 'obsidian';

// system-managed: vault automatically managed by the system
export const SYSTEM_MANAGED_VAULT_TYPE = 'system-managed';
```

Blandt dem er **coderef**-typen den mest almindeligt anvendte i HagiCode. Det er specielt designet til reproducerede kodeprojekter, der giver en standardiseret mappestruktur og AI-læsbare metadatabeskrivelser.

### Vedvarende opbevaringsmekanisme

Vault-registreringsdatabasen gemmes vedvarende i JSON-format, hvilket sikrer, at konfigurationen forbliver tilgængelig, efter at applikationen genstartes:

```csharp
public class VaultRegistryStore : IVaultRegistryStore
{
    private readonly string _registryFilePath;

    public VaultRegistryStore(IConfiguration configuration, ILogger<VaultRegistryStore> logger)
    {
        var dataDir = configuration["DataDir"] ?? "./data";
        var absoluteDataDir = Path.IsPathRooted(dataDir)
            ? dataDir
            : Path.GetFullPath(Path.Combine(Directory.GetCurrentDirectory(), dataDir));

        _registryFilePath = Path.Combine(absoluteDataDir, "personal-data", "vaults", "registry.json");
    }
}
```

Fordelen ved dette design er, at det er enkelt og pålideligt. JSON kan læses af mennesker, hvilket gør fejlfinding og manuel redigering lettere; filsystemlagring undgår kompleksiteten af ​​en database og reducerer systemafhængigheder. Når alt kommer til alt, nogle gange er den enkleste mulighed virkelig den bedste.

### AI-kontekstintegration

Vigtigst er det, at systemet automatisk kan indsprøjte vault-information i sammenhæng med AI-forslag:

```typescript
export function buildTargetVaultsText(
  vaults: VaultForText[],
  template: VaultPromptTemplate = DEFAULT_VAULT_PROMPT_TEMPLATE,
): string {
  const readOnlyVaults = vaults.filter((vault) => vault.accessType === 'read');
  const editableVaults = vaults.filter((vault) => vault.accessType === 'write');

  if (readOnlyVaults.length === 0 && editableVaults.length === 0) {
    return '';
  }

  const sections = [
    buildVaultSection(readOnlyVaults, template.reference),
    buildVaultSection(editableVaults, template.editable),
  ].filter(Boolean);

  return `\n\n### ${template.heading}\n\n${sections.join('\n')}`;
}
```

Dette muliggør en vigtig egenskab: AI-assistenter kan automatisk forstå de tilgængelige læringsressourcer uden at brugerne manuelt giver kontekst. Man kan sige, at det tæller som en slags stiltiende forståelse.

## Den standardiserede struktur af CodeRef Vault

For coderef-typen af hvælving giver HagiCode en standardiseret mappestruktur:

```
my-coderef-vault/
├── index.yaml          # vault metadata description
├── AGENTS.md           # operating guide for AI assistants
├── docs/               # stores study notes and documents
└── repos/              # manages reproduced code repositories through Git submodules
```

Når du opretter en boks, initialiserer systemet automatisk denne struktur:

```csharp
private async Task EnsureCodeRefStructureAsync(
    string vaultName,
    string physicalPath,
    ICollection<VaultBootstrapDiagnosticDto> diagnostics,
    CancellationToken cancellationToken)
{
    Directory.CreateDirectory(physicalPath);

    var indexPath = Path.Combine(physicalPath, CodeRefIndexFileName);
    var docsPath = Path.Combine(physicalPath, CodeRefDocsDirectoryName);
    var reposPath = Path.Combine(physicalPath, CodeRefReposDirectoryName);

    // Create the standard directory structure
    if (!Directory.Exists(docsPath))
    {
        Directory.CreateDirectory(docsPath);
    }

    if (!Directory.Exists(reposPath))
    {
        Directory.CreateDirectory(reposPath);
    }

    // Create the AGENTS.md guide
    await EnsureCodeRefAgentsDocumentAsync(physicalPath, cancellationToken);

    // Create the index.yaml metadata
    await WriteCodeRefIndexDocumentAsync(indexPath, mergedDocument, cancellationToken);
}
```

Denne struktur er også omhyggeligt designet:

- **docs/** gemmer dine undersøgelsesnotater, hvor du kan registrere din forståelse af koden, arkitekturanalyse, erfaringer og så videre i Markdown
- **repos/** administrerer reproducerede repositories gennem Git-undermoduler i stedet for at kopiere kode direkte, hvilket holder koden synkroniseret og sparer plads
- **index.yaml** indeholder vault-metadataene, så AI-assistenter hurtigt kan forstå formålet med og indholdet af vaulten
- **AGENTS.md** er en vejledning skrevet specielt til AI-assistenter, der forklarer, hvordan man håndterer indholdet af hvælvingen

Organiseret på denne måde kan AI måske lidt lettere forstå, hvad du har i tankerne.

## Automatisk initialisering for systemadministrerede hvælvinger

Udover manuelt oprettede hvælvinger, understøtter HagiCode også systemstyrede hvælvinger:

```csharp
public async Task<IReadOnlyList<VaultRegistryEntry>> EnsureAllSystemManagedVaultsAsync(
    CancellationToken cancellationToken = default)
{
    var definitions = GetAllResolvedDefinitions();
    var entries = new List<VaultRegistryEntry>(definitions.Count);

    foreach (var definition in definitions)
    {
        entries.Add(await EnsureResolvedSystemManagedVaultAsync(definition, cancellationToken));
    }

    return entries;
}
```

Systemet opretter og administrerer automatisk følgende bokse:

- **hagiprojectdata**: projektdatalagring bruges til at gemme projektkonfiguration og tilstand
- **personlige data**: personlig datalagring, der bruges til at gemme brugerpræferencer
- **hbsprompt**: et promptskabelonbibliotek, der bruges til at administrere almindeligt anvendte AI-prompts

Disse hvælvinger initialiseres automatisk, når systemet starter, så brugerne behøver ikke at konfigurere dem manuelt. Nogle ting er simpelthen bedre overladt til systemet i stedet for at mennesker bekymrer sig om dem.

## Adgangskontrolmekanisme

En vigtig del af designet er adgangskontrol. Systemet opdeler hvælvinger i to adgangstyper:

```typescript
export interface VaultForText {
  id: string;
  name: string;
  type: string;
  physicalPath: string;
  accessType: 'read' | 'write';  // Key: distinguish read-only from editable
}
```

- **reference (skrivebeskyttet)**: AI bruges kun til analyse og forståelse og kan ikke ændre indhold. Velegnet til refererede open source-projekter, dokumenter og lignende materialer
- **redigerbar (redigerbar)**: AI kan ændre indhold efter behov for opgaven. Velegnet til dine noter, kladder og lignende materialer

Denne skelnen er vigtig. Det fortæller AI, hvilket indhold der er "skrivebeskyttet reference", og hvilket indhold der er "sikkert at redigere", hvilket reducerer risikoen for utilsigtede ændringer. Når alt kommer til alt, er der ingen, der ønsker, at deres hårde arbejde forsvinder på grund af en utilsigtet redigering.

## I praksis: oprettelse og brug af Vault

Nu hvor vi har dækket ideerne, lad os se på, hvordan det fungerer i praksis.

### Opret en CodeRef Vault

Her er et komplet frontend-opkaldseksempel:

```typescript
const createCodeRefVault = async () => {
  const response = await VaultService.postApiVaults({
    requestBody: {
      name: "React Learning Vault",
      type: "coderef",
      physicalPath: "/Users/developer/vaults/react-learning",
      gitUrl: "https://github.com/facebook/react.git"
    }
  });

  // The system will automatically:
  // 1. Clone the React repository into vault/repos/react
  // 2. Create the docs/ directory for notes
  // 3. Generate the index.yaml metadata
  // 4. Create the AGENTS.md guide file

  return response;
};
```

Dette API-kald fuldfører en række handlinger: oprettelse af mappestrukturen, initialisering af Git-undermoduler, generering af metadatafiler og mere. Du behøver kun at give de grundlæggende oplysninger og lade systemet klare resten. Det er ærligt talt en ret bekymringsfri tilgang.

## Bedste praksis og ting at holde øje med

Mens vi brugte Vault-systemet, har vi opsummeret nogle få erfaringer.

### Stisikkerhed

Systemet validerer strengt stier for at forhindre stigennemløbsangreb:

```csharp
private static string ResolveFilePath(string vaultRoot, string relativePath)
{
    var rootPath = EnsureTrailingSeparator(Path.GetFullPath(vaultRoot));
    var combinedPath = Path.GetFullPath(Path.Combine(rootPath, relativePath));
    if (!combinedPath.StartsWith(rootPath, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode,
            "Vault file paths must stay inside the registered vault root.");
    }
    return combinedPath;
}
```

Dette er vigtigt. Hvis du tilpasser en vault-sti, skal du sørge for, at den forbliver inden for det tilladte område, ellers vil systemet afvise handlingen. Du kan virkelig ikke overbetone sikkerhed.

### Git-undermodulstyring

CodeRef Vault anbefaler Git-undermoduler i stedet for direkte at kopiere kode:

```csharp
private static string BuildCodeRefAgentsContent()
{
    return """
    # CodeRef Vault Guide

    Repositories under `repos/` should be maintained through Git submodules
    rather than copied directly into the vault root.

    Keep this structure stable so assistants and tools can understand the vault quickly.
    """ + Environment.NewLine;
}
```

Dette medfører flere fordele: at holde kode synkroniseret med upstream, spare diskplads og gøre det nemmere at administrere flere versioner af koden. Når alt kommer til alt, hvem vil downloade det samme igen og igen?

### Grænser for forhåndsvisning af filer

For at forhindre ydeevneproblemer begrænser systemet filstørrelse og type:

```csharp
private const int FileEnumerationLimit = 500;
private const int PreviewByteLimit = 256 * 1024;  // 256KB
```

Hvis din boks indeholder et stort antal filer eller meget store filer, kan preview-ydeevnen blive påvirket. I så fald kan du overveje at behandle filer i batches eller bruge specialiserede søgeværktøjer. Nogle gange, når noget bliver for stort, bliver det sværere at håndtere, ikke nemmere.

### Diagnostiske oplysninger

Når du opretter en boks, returnerer systemet diagnostiske oplysninger for at hjælpe med fejlretning:

```csharp
List<VaultBootstrapDiagnosticDto> bootstrapDiagnostics = [];

if (IsCodeRefVaultType(normalizedType))
{
    bootstrapDiagnostics = await EnsureCodeRefBootstrapAsync(
        normalizedName,
        normalizedPhysicalPath,
        normalizedGitUrl,
        cancellationToken);
}
```

Hvis oprettelsen mislykkes, kan du inspicere de diagnostiske oplysninger for at forstå den specifikke årsag. Når noget går galt, er kontrol af diagnostikken ofte den mest direkte vej frem.

## Resumé

Gennem et samlet lagerabstraktionslag løser Vault-systemet adskillige kernesmerter ved reproduktion af projekter i AI-æraen:

- **Centraliseret videnstyring**: alle læringsressourcer er samlet ét sted i stedet for spredt overalt
- **Automatisk AI-kontekstinjektion**: AI-assistenter kan automatisk forstå de tilgængelige læringsressourcer uden manuel kontekstopsætning
- **Genbrug af viden på tværs af projekter**: viden kan deles og genbruges på tværs af flere læringsprojekter
- **Standardiseret biblioteksstruktur**: Et ensartet bibliotekslayout sænker indlæringskurven
