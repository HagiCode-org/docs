Ta strona jest przeznaczona dla początkujących użytkowników HagiCode, którzy chcą najkrótszej ścieżki konfiguracji OmniRoute: uruchom usługę lokalną, sprawdź, czy `http://localhost:20128/v1` działa i potwierdź, że zarówno panel kontrolny, jak i interfejs API są dostępne.

:::tip[Zalecana ścieżka]
Ta pierwsza wersja utrzymuje najpierw najbardziej stabilną ścieżkę: `npm install -g omniroute`, `omniroute`, Panel pod adresem `http://localhost:20128`oraz podstawowy adres URL zgodny z OpenAI pod adresem `http://localhost:20128/v1`. Docker, VPS i bardziej zaawansowane tryby wdrażania pozostają w sekcji opcjonalnej.
:::

## Warunki wstępne

- Node.js jest już zainstalowany. Kompletny [Instalowanie Node.js](/en/related-software-installation/nodejs/installation) pierwszy, jeśli zajdzie taka potrzeba.
- Port lokalny `20128` jest dostępny i nie jest jeszcze zajęty.
- Masz co najmniej jedno konto dostawcy modelu nadrzędnego, które planujesz kierować za pośrednictwem usługi OmniRoute.
- Przed integracją HagiCode, CLI lub SDK, zachowaj te dwa adresy oddzielnie:
  - Panel: `http://localhost:20128`
  - Podstawowy adres URL zgodny z OpenAI: `http://localhost:20128/v1`

## Zainstaluj i uruchom lokalnie

Zacznij od zalecanej globalnej ścieżki npm:

```bash
npm install -g omniroute
omniroute
```

Po uruchomieniu usługi zweryfikuj ją w następującej kolejności:

1. Otwórz `http://localhost:20128` w przeglądarce i potwierdź załadowanie Panelu.
2. Idź do `Endpoints` stronę w panelu kontrolnym i utwórz lub skopiuj klucz API.
3. Zanotuj podstawowy adres URL, który później przekażesz narzędziom: `http://localhost:20128/v1`.

:::note[Dlaczego warto zacząć od pulpitu nawigacyjnego]
W przypadku pierwszej konfiguracji panel kontrolny to najszybszy sposób sprawdzenia, czy OmniRoute działa, dostawcy są połączeni i czy istnieje klucz API. Jeśli późniejsze wywołanie klienta nie powiedzie się, sprawdzenie najpierw pulpitu nawigacyjnego jest zwykle szybsze niż natychmiastowe debugowanie kodu SDK.
:::

## Sprawdź działającą usługę

### Walidacja 1: upewnij się, że katalog modeli jest czytelny

Włóż klucz z `Endpoints` stronę do zmiennej środowiskowej:

```bash
export OMNIROUTE_API_KEY="replace-with-the-key-you-copied-from-the-Endpoints-page"
```

Następnie poproś `/v1/models`:

```bash
curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}"
```

Jeśli otrzymasz z powrotem listę modeli, oznacza to, że punkt końcowy i proces uwierzytelniania zgodny z OpenAI działają.

### Walidacja 2: wyślij minimalną prośbę o czat

```bash
curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "replace-with-a-model-id-from-v1-models",
    "messages": [
      { "role": "user", "content": "Reply with ok." }
    ]
  }'
```

Celem nie jest tu natychmiastowa jakość. Ma to udowodnić, że wybrany przez Ciebie model może już zostać pomyślnie przekierowany przez OmniRoute.
