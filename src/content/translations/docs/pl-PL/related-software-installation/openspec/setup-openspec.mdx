# Zainstaluj OpenSpec

OpenSpec zarządza propozycjami, specyfikacjami i zapisami zmian. Ta strona ma jeden cel: jasno wyjaśnić ścieżkę instalacji, aby można było ją dołączyć do istniejącego przepływu pracy.

## Co to jest OpenSpec?

OpenSpec to narzędzie wiersza poleceń służące do zarządzania propozycjami i specyfikacjami w pracy z oprogramowaniem. Podstawowy przepływ pracy obejmuje:

- **Zarządzanie propozycjami**: Twórz, przeglądaj i śledź propozycje funkcji
- **Weryfikacja specyfikacji**: Upewnij się, że zmiany w kodzie są zgodne z wcześniej zdefiniowanymi specyfikacjami
- **Śledzenie zmian**: Rejestruj całą historię sprawdzonych zmian
- **Współpraca zespołowa**: Wspieraj rozwój współpracy wielu osób

### Zastosowanie w projektach HagiCode

W projektach HagiCode OpenSpec jest używany do:

1. **Zarządzanie rozwojem funkcji**: Wszystkie nowe funkcje muszą przejść proces propozycji OpenSpec
2. **Przegląd kodu**: Upewnij się, że zmiany są zgodne ze specyfikacjami projektu
3. **Synchronizacja dokumentacji**: Zachowaj spójność pomiędzy dokumentacją a implementacją kodu
4. **Integracja kontroli wersji**: Bezproblemowa integracja z przepływami pracy kontroli wersji

## Warunki wstępne

Przed instalacją OpenSpec upewnij się, że Twój system spełnia następujące wymagania:

### Node.js i npm

OpenSpec jest dystrybuowany poprzez npm i wymaga środowiska Node.js. OpenSpec wymaga **Node.js 18.0 lub nowszego**.

Sprawdź instalację Node.js:

```bash
node --version
```

Sprawdź instalację npm:

```bash
npm --version
```

Jeśli nie jest zainstalowany lub jego wersja jest zbyt stara, zapoznaj się z instrukcjami instalacji odpowiedniego oprogramowania w dokumencie [Przewodnik instalacji](/en/installation).

## Instalowanie OpenSpeca

:::tip[Wymagania dotyczące wersji]
Ten projekt wymaga użycia zakresu wersji **OpenSpec 1.0.0 lub nowszego**. Obsługiwane są wszystkie wersje 1.x, aby zapewnić zgodność z przebiegiem projektu.
:::

OpenSpec jest instalowany globalnie poprzez npm, dzięki czemu jest dostępny z dowolnego miejsca w systemie.

### Zainstaluj globalnie za pomocą npm

<Tabs>
  <TabItem value="win" label="Windows">
    Uruchom w PowerShell lub wierszu poleceń:

```powershell
npm install -g @fission-ai/openspec@1
```

:::note[Problemy z uprawnieniami systemu Windows]
Jeśli podczas instalacji napotkasz błędy uprawnień, uruchom PowerShell lub Wiersz Poleceń jako administrator.
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    Uruchom w terminalu:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[Problemy z uprawnieniami macOS]
Jeśli napotkasz `EACCES` błędy uprawnień, spróbuj:
```bash
sudo npm install -g @fission-ai/openspec@1
```
Lub użyj [nvm](https://github.com/nvm-sh/nvm) aby zarządzać wersjami Node.js i uniknąć problemów z uprawnieniami.
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    Uruchom w terminalu:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[Problemy z uprawnieniami do Linuksa]
Jeśli napotkasz `EACCES` błędy uprawnień, możesz:
1. Użyj `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. Skonfiguruj npm, aby korzystał z katalogu użytkownika:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### Sprawdź instalację

Po instalacji sprawdź, czy OpenSpec jest zainstalowany poprawnie i czy wersja spełnia wymagania:

```bash
openspec --version
```

Jeśli instalacja przebiegła pomyślnie, zobaczysz numer wersji, na przykład:

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::note[Zakres wersji]
Upewnij się, że zainstalowana wersja OpenSpec to 1.0.0 lub nowsza. Projekt zweryfikował kompatybilność z wersją 1.1.1. Aby uzyskać nowe funkcje, zaleca się użycie najnowszej kompatybilnej wersji.
:::

## Typowe problemy i rozwiązywanie problemów

### Błędy uprawnień

#### Problem: `EACCES` Odmowa pozwolenia

Napotkanie błędów uprawnień podczas instalowania lub uruchamiania OpenSpec.

**Rozwiązania**:

<Tabs>
  <TabItem value="win" label="Windows">
    Uruchom PowerShell lub Wiersz Poleceń jako administrator:
    1. Kliknij prawym przyciskiem myszy PowerShell lub Wiersz poleceń
    2. Wybierz „Uruchom jako administrator”
    3. Uruchom ponownie polecenie instalacji
  </TabItem>
  <TabItem value="mac" label="macOS">
    Zainstaluj za pomocą `sudo`:

```bash
sudo npm install -g @fission-ai/openspec@1
```

Lub użyj [nvm](https://github.com/nvm-sh/nvm) aby zarządzać Node.js i uniknąć problemów z globalnymi uprawnieniami do instalacji.
  </TabItem>
  <TabItem value="linux" label="Linux">
    Rozwiązanie 1: Użyj sudo
```bash
sudo npm install -g @fission-ai/openspec@1
```

Rozwiązanie 2: Skonfiguruj npm do korzystania z katalogu użytkownika
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @fission-ai/openspec@1
```
  </TabItem>
</Tabs>

### Problemy z siecią

#### Problem: Instalacja npm kończy się niepowodzeniem lub jest bardzo powolna

Nie można pobrać OpenSpec z rejestru npm z powodów sieciowych.

**Rozwiązania**:

1. **Użyj domowego lustra npm**:

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **Przywróć oficjalne źródło** (w razie potrzeby):

```bash
npm config set registry https://registry.npmjs.org
```

3. **Użyj cnpm** (opcjonalnie):

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

:::wskazówka
Użytkownikom w Chinach kontynentalnych zaleca się korzystanie z lustra Taobao, które może znacznie poprawić prędkość pobierania.
:::

### Problemy ze zgodnością wersji

#### Problem: Polecenia OpenSpec działają nieprawidłowo lub zgłaszają błędy

Zainstalowana wersja OpenSpec jest niekompatybilna z bieżącym środowiskiem.

**Rozwiązania**:

1. **Sprawdź wersję Node.js**:

```bash
node --version
```

Upewnij się, że wersja to 18.0 lub nowsza. Jeśli wersja jest zbyt niska, zaktualizuj Node.js.

2. **Sprawdź wersję OpenSpec**:

```bash
openspec --version
```

3. **Zainstaluj ponownie określoną wersję**:

```bash
npm install -g @fission-ai/openspec@1
```

4. **Wyczyść pamięć podręczną npm** (jeśli problem będzie się powtarzał):

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### Nie znaleziono polecenia

#### Problem: Pisanie `openspec` Polecenie pokazuje „Nie znaleziono polecenia”

OpenSpec jest zainstalowany, ale system nie może znaleźć polecenia.

**Rozwiązania**:

<Tabs>
  <TabItem value="win" label="Windows">
    1. Potwierdź globalną ścieżkę instalacji npm:

```powershell
npm config get prefix
```

2. Dodaj ścieżkę do systemowej PATH:
   - Otwórz „Właściwości systemu” → „Zaawansowane” → „Zmienne środowiskowe”
   - Znajdź `Path` w „Zmienne systemowe”
   - Dodaj ścieżkę globalną npm (zwykle `C:\Users\<username>\AppData\Roaming\npm`)
   - Uruchom ponownie okno wiersza poleceń
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. Potwierdź globalną ścieżkę instalacji npm:

```bash
npm config get prefix
```

2. Dodaj ścieżkę do PATH (w `~/.zshrc` lub `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Załaduj ponownie konfigurację:

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. Potwierdź globalną ścieżkę instalacji npm:

```bash
npm config get prefix
```

2. Dodaj ścieżkę do PATH (w `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Załaduj ponownie konfigurację:

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
