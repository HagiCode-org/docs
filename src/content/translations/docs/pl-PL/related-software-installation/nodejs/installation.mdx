Ten przewodnik opisuje instalację Node.js w systemach Windows, macOS i Linux. Node.js jest kluczową zależnością dla lokalnych narzędzi HagiCode, w tym OpenSpec, narzędzi frontendowych i kilku interfejsów CLI dla programistów.

:::tip[Wymagania dotyczące wersji]
HagiCode zaleca **Node.js 24+** (minimalna obsługiwana wersja Node.js &gt;= 18.0). W większości przypadków zainstaluj najnowszą wersję LTS, a następnie sprawdź, czy Twoja wersja spełnia wymagania.
:::

## Instalowanie Node.js

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Metoda pierwsza: oficjalny instalator (zalecany)

1. Odwiedź [Oficjalna witryna Node.js](https://nodejs.org/)
2. Pobierz wersję **LTS (Long Term Support)** instalatora Windows (`.msi` plik)
3. Kliknij dwukrotnie, aby uruchomić instalator
4. Postępuj zgodnie z instrukcjami kreatora instalacji, aby dokończyć instalację:
   - Zaakceptuj umowę licencyjną
   - Wybierz ścieżkę instalacji (domyślna jest w porządku)
   - Upewnij się, że opcja „Automatycznie zainstaluj niezbędne narzędzia” jest zaznaczona
   - Kliknij „Zainstaluj”, aby rozpocząć instalację
5. Po zakończeniu instalacji kliknij „Zakończ”

### Metoda druga: użycie wingeta

Otwórz PowerShell lub Wiersz Poleceń i uruchom:

```powershell
winget install OpenJS.NodeJS.LTS
```

Po instalacji otwórz ponownie terminal, aby zmiany zaczęły obowiązywać.

  </TabItem>
  <TabItem value="macos" label="macOS">

### Metoda pierwsza: oficjalny instalator (zalecany)

1. Odwiedź [Oficjalna witryna Node.js](https://nodejs.org/)
2. Pobierz instalator systemu macOS w wersji **LTS (Long Term Support)** (`.pkg` plik)
3. Kliknij dwukrotnie, aby uruchomić instalator
4. Postępuj zgodnie z instrukcjami kreatora instalacji, aby dokończyć instalację:
   - Kliknij „Kontynuuj”
   - Zaakceptuj umowę licencyjną
   - Wybierz dysk instalacyjny
   - Kliknij „Zainstaluj” i wprowadź hasło administratora
5. Po zakończeniu instalacji kliknij „Zamknij”

### Metoda druga: użycie Homebrew

Jeśli masz [Domowe piwo](https://brew.sh/) zainstalowany, możesz zainstalować Node.js poprzez terminal:

```bash
brew install node
```

:::wskazówka
Homebrew automatycznie doda Node.js do Twojej PATH. Jeśli polecenie nie jest dostępne, zrestartuj terminal lub uruchom:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

  </TabItem>
  <TabItem value="ubuntu" label="Ubuntu/Debian">

### Korzystanie z repozytorium NodeSource (zalecane)

To najlepszy sposób na uzyskanie najnowszej wersji Node.js:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Korzystanie z Menedżera pakietów apt

Jeśli nie potrzebujesz najnowszej wersji, możesz skorzystać z domyślnego repozytorium Ubuntu:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::uwaga
Wersja Node.js w domyślnym repozytorium może być starsza. Po instalacji sprawdź wersję, aby upewnić się, że spełnia wymagania Node.js &gt;= 18.0 (zaleca się Node.js 24+).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### Korzystanie z repozytorium NodeSource (zalecane)

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### Używanie Menedżera pakietów yum

```bash
sudo yum install -y nodejs npm
```

  </TabItem>
  <TabItem value="fedora" label="Fedora">

### Korzystanie z Menedżera pakietów dnf

```bash
sudo dnf install -y nodejs npm
```

:::wskazówka
Repozytorium oprogramowania Fedory zwykle zawiera nowsze wersje Node.js i można je zainstalować bezpośrednio przez dnf.
:::

  </TabItem>
</Tabs>

## Korzystanie z narzędzi do zarządzania wersjami

Jeśli chcesz przełączać się między wieloma wersjami Node.js, możesz skorzystać z narzędzi do zarządzania wersjami.

### nvm (Menedżer wersji węzła)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

Po instalacji załaduj ponownie konfigurację powłoki:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Okna

W systemie Windows zalecamy użycie [nvm-windows](https://github.com/coreybutler/nvm-windows):

1. Pobierz najnowsze `nvm-setup.exe` od [wydania nvm-windows](https://github.com/coreybutler/nvm-windows/releases)
2. Uruchom instalator
3. Otwórz ponownie wiersz poleceń lub PowerShell

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

Po zainstalowaniu nvm możesz go używać do instalowania wersji Node.js i zarządzania nimi:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (szybki menedżer węzłów)

fnm to szybsze i prostsze narzędzie do zarządzania wersjami Node.js zbudowane w Rust.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Okna

Użyj [miarka](https://scoop.sh/) lub [skrzydło](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Przyspieszenie dla użytkowników z Chin kontynentalnych

Jeśli jesteś w Chinach kontynentalnych, pobieranie pakietów Node.js i npm może być powolne. Oto kilka rozwiązań przyspieszających.

### Korzystanie z krajowych źródeł lustrzanych

#### npm Taobao Mirror (zalecane)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### Używanie nrm do zarządzania źródłami lustrzanymi

nrm (menedżer rejestru npm) pomaga szybko przełączać źródła lustrzane npm:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Przyspieszenie pobierania instalatora Node.js

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao zapewnia pobieranie lustrzane dla instalatorów Node.js:

- **Adres pobierania**: https://npmmirror.com/mirrors/node/

Odwiedź powyższy adres i wybierz wersję, którą chcesz pobrać.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

Jeśli pobieranie nvm lub innych narzędzi z GitHuba jest powolne, możesz użyć kopii lustrzanej FastGit:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::uwaga
Usługa FastGit może ulec zmianie, prosimy zapoznać się z oficjalnymi informacjami.
:::

  </TabItem>
</Tabs>

### Domowe lusterka Homebrew

Jeśli używasz Homebrew w Chinach kontynentalnych, możesz skonfigurować krajowe serwery lustrzane w celu przyspieszenia:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Menedżerowie pakietów specyficznych dla Chin kontynentalnych

#### cnpm

cnpm to klient npm dostarczony przez zespół Taobao, który domyślnie korzysta z lustra Taobao:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::wskazówka
cnpm jest w większości kompatybilny z poleceniami npm i może być używany jako bezpośredni zamiennik.
:::

### Sprawdzanie efektu przyspieszenia lustrzanego

Po skonfigurowaniu kopii lustrzanych zweryfikuj poprzez:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Weryfikacja instalacji

Po instalacji sprawdź, czy Node.js i npm są poprawnie zainstalowane:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::uwaga
Upewnij się, że wersja Node.js &gt;= 18.0 (zalecana jest wersja Node.js 24+). Jeśli wersja jest zbyt niska, zainstaluj ponownie, korzystając z powyższych metod lub użyj narzędzi do zarządzania wersjami, aby zainstalować poprawną wersję.
:::

### Przykładowe dane wyjściowe weryfikacji

Wynik pomyślnej weryfikacji powinien wyglądać następująco:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Rozwiązywanie problemów

### Problemy z Windowsem

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Jeśli `node` lub `npm` polecenia nie są dostępne po instalacji:

1. Uruchom ponownie wiersz poleceń lub PowerShell
2. Jeśli nadal nie jest dostępny, ręcznie dodaj Node.js do PATH:
   - Wyszukaj „Zmienne środowiskowe” i otwórz „Edytuj systemowe zmienne środowiskowe”
   - Kliknij „Zmienne środowiskowe”
   - Znajdź „Ścieżkę” w „Zmiennych systemowych” i kliknij „Edytuj”
   - Dodaj ścieżkę instalacji Node.js (domyślnie: `C:\Program Files\nodejs\`)
   - Kliknij „OK”, aby zapisać zmiany

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Jeśli podczas korzystania z nvm napotkasz błędy zasad wykonywania:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### Problemy z macOSem

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Jeśli podczas instalowania globalnych pakietów npm wystąpią błędy uprawnień:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Jeśli polecenia nie są dostępne po instalacji, uruchom ponownie terminal lub ręcznie dodaj do PATH (zwykle w `/usr/local/bin`).

  </TabItem>
</Tabs>

### Problemy z Linuksem

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Jeśli podczas instalowania globalnych pakietów npm wystąpią błędy uprawnień, zaleca się użycie narzędzi do zarządzania wersjami Node.js (takich jak nvm lub fnm) zamiast systemowych menedżerów pakietów.

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Jeśli wersja Node.js zainstalowana za pomocą menedżera pakietów systemowych jest za stara:

1. Użyj repozytorium NodeSource (zalecane, patrz wyżej)
2. Lub użyj narzędzi do zarządzania wersjami (nvm/fnm)

  </TabItem>
</Tabs>

### Sprawdzanie środowiska

Jeśli weryfikacja nie powiedzie się, sprawdź następujące elementy:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
