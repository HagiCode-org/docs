Na tej stronie znajduje się ujednolicony wpis instalacji lokalnego interfejsu wiersza polecenia AI Agent `repos/docs`. HagiCode obsługuje obecnie 11 aktywnych interfejsów CLI agentów, zgodnie z backendem `PrimaryProfessions` kolejność: Claude Code, Codex, GitHub Copilot, OpenCode, Hermes, QoderCLI, Kiro, Kimi, Gemini, DeepAgents i Codebuddy.

:::tip[Jak korzystać z tej strony]
Najpierw sprawdź w tabeli bieżący stan wsparcia i tryb konfiguracji, a następnie kontynuuj korzystanie z udostępnianej przez dostawcę dokumentacji dotyczącej instalacji, uwierzytelniania, ACP, uprawnień i rozwiązywania problemów. Zakres wsparcia i tryb konfiguracji to osobne kwestie.
:::

## Słowniczek państwowy

- **Obecnie obsługiwane** oznacza, że CLI należy do aktualnie aktywnego zakresu wsparcia HagiCode.
- **Instalacja jednym kliknięciem** oznacza, że HagiCode zapewnia punkt wejścia do instalacji z przewodnikiem; niektóre interfejsy CLI nadal wymagają logowania dostawcy lub etapu autoryzacji po instalacji.
- **Ręczna instalacja/uwierzytelnianie** oznacza, że CLI jest teraz obsługiwane, ale musisz zakończyć lokalną instalację lub proces autoryzacji, zanim HagiCode będzie mógł go wykryć i użyć.
- **Tylko zgodność ze starszymi wersjami** oznacza wycofaną wartość, np `IFlowCli` nadal istnieje w celu zapewnienia zgodności danych historycznych, ale nie jest częścią bieżącego aktywnego zakresu wsparcia.

## Zalecane kolejne kroki

- Kontynuuj z [Zainstaluj Node.js](/en/related-software-installation/nodejs/installation) i [Zainstaluj OpenSpec](/en/related-software-installation/openspec/setup-openspec) jeśli nadal potrzebujesz wymagań wstępnych.
- Jeśli chcesz mieć jedną bramę lokalną dla wielu narzędzi zgodnych z OpenAI, kontynuuj [Instalacja i użytkowanie OmniRoute](/en/related-software-installation/omniroute/).
- Wróć do [Przewodnik instalacji](/en/installation) dla szerszej ścieżki konfiguracji.
- Jeśli trafiłeś tutaj z historycznego wpisu na blogu, traktuj tę stronę i przegląd instalacji jako aktualne, wiecznie aktualne źródło prawdy.
