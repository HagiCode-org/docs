## Przewodnik po tworzeniu efektu animacji rozciągnięcia światła obramowania

> Jak zbudować ten ważny element, który użytkownicy zauważą na pierwszy rzut oka, używając czystego CSS? Właściwie to nie jest takie trudne. Sztuka polega na tym, aby wybrać nieco okrężną ścieżkę. W tym artykule przeprowadzę Cię przez proces tworzenia od podstaw animacji przesuwania światła obramowania, a także podzielę się kilkoma pułapkami, na które natknęliśmy się podczas tworzenia HagiCode.

## Tło

Jeśli pracujesz nad frontendem, prawdopodobnie spotkałeś się już wcześniej z taką sytuacją: podchodzi menedżer produktu z wyrażeniem „to zdecydowanie proste” i pyta: „Czy możemy dodać jakiś efekt specjalny do tego uruchomionego zadania, aby użytkownicy mogli go natychmiast zauważyć?”

Mówisz pewnie, możemy po prostu zmienić kolor obramowania. Potem kręcą głową z wyrazem twarzy mówiącym: „Nie rozumiesz”. Odpowiadają: „To nie jest wystarczająco oczywiste. Chcę uzyskać efekt, w którym światło biegnie wzdłuż krawędzi, jak w filmie science-fiction”.

W tym momencie możesz zacząć się zastanawiać, jak to zbudować. Płótno? SVG? A może CSS poradzi sobie z tym sam? Przecież nikt nie chce się przyznać, że nie wie jak.

W nowoczesnych aplikacjach internetowych animacje przesuwania światła obramowania są bardzo powszechne. Są one używane głównie w kilku takich scenariuszach:

- **Wskaźniki stanu**: Oznaczanie zadań w toku lub aktywnych elementów
- **Koncentracja wizualna**: podkreślanie ważnych obszarów treści
- **Wzmocnienie marki**: Tworzenie eleganckiego, nowoczesnego i zaawansowanego technologicznie stylu wizualnego
- **Motywy sezonowe**: Tworzenie uroczystej atmosfery na specjalne okazje

Podczas budowy natrafiliśmy na dokładnie ten wymóg [Kod Hagi](https://hagicode.com). Użytkownicy musieli szybko sprawdzić, które sesje są w toku i które wnioski są aktualnie przetwarzane. Próbowaliśmy kilku różnych podejść. Niektóre ścieżki były gładsze, inne nieco bardziej kręte. Ostatecznie zdecydowaliśmy się na dość dojrzałą strategię wdrożenia.

## Podstawowe pomysły na wdrożenie

Po przeanalizowaniu bazy kodu HagiCode podsumowaliśmy poniżej kilka podstawowych wzorców implementacji. Każdy z nich pasuje do innego scenariusza, czyli innymi słowy, każdy istnieje z jakiegoś powodu.

### 1. Obrotowa poświata ze stożkowym gradientem (najczęściej)

Jest to klasyczny sposób wdrożenia efektu rozmycia światła obramowania. Podstawową ideą jest użycie CSS `conic-gradient` aby utworzyć gradient stożkowy, a następnie obracaj go w sposób ciągły. Podobnie jak latarnia uliczna włączająca się w nocy, po prostu krąży.

**Kluczowe elementy:**
- Skorzystaj z `::before` pseudoelement tworzący warstwę blasku
- Użyj `conic-gradient` aby zdefiniować rozkład kolorów gradientu
- Skorzystaj z `::after` pseudoelement maskujący obszar środkowy (opcjonalnie)
- Użyj `@keyframes` aby zaimplementować animację obrotu

### 2. Boczna linia blasku

Działa to dobrze w przypadku wskaźników stanu w elementach listy. Zamiast animować całą ramkę, tworzysz cienką świecącą linię po jednej stronie elementu. Czasami wystarczy trochę światła. Nie musisz oświetlać całego świata.

**Kluczowe elementy:**
- Cienki, absolutnie pozycjonowany element liniowy
- Użyj `box-shadow` aby uzyskać efekt blasku
- Użyj `scale` i `opacity` do animacji oddychania

### 3. Świecące tło z cieniem pudełkowym

Jeśli nie potrzebujesz efektu pełnego przeciągnięcia i chcesz po prostu miękkiej poświaty tła, nałóż wiele warstw `box-shadow` wartości wystarczą. Czasami prostsza opcja jest lepsza.

### 4. Wsparcie dostępności

Tę część łatwo przeoczyć, ale jest ona niezwykle ważna. Każda animacja powinna uwzględniać `prefers-reduced-motion` media query i zapewniają statyczną alternatywę dla użytkowników, którzy nie chcą animacji. Nie każdy lubi ciągły ruch i respektowanie tych preferencji ma znaczenie.

## Opcje wdrożenia

### Opcja 1: Obrotowa granica stożkowo-gradientowa (zalecana)

Jest to najbardziej kompletna implementacja efektu szerokiego światła obramowania i jest to również opcja, z której najczęściej korzystamy w HagiCode. W końcu, jeśli coś działa dobrze, po co to zmieniać?

```css
/* Parent container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotating glow layer */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Mask layer (optional, for creating a hollow border effect) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Zasada tej opcji jest dość prosta: utwórz pseudoelement większy niż kontener nadrzędny, narysuj na nim gradient stożkowy i obracaj go w sposób ciągły. Używa kontenera nadrzędnego `overflow: hidden`, więc widoczne pozostaje tylko światło przechodzące przez granicę. To trochę jak oglądanie latarni przez okno. Widzisz tylko mały kawałek, który przechodzi obok.

### Opcja 2: Uproszczona obrotowa ramka świetlna

Jeśli nie potrzebujesz pełnego efektu, HagiCode zawiera również lżejszą wersję klasy użytkowej. Czasami prostsze podejście naprawdę jest lepsze.

```css
/* Rotating light border utility class */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Accessibility support */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Zwróć uwagę na `will-change: transform` tutaj. Informuje przeglądarkę: „Ten element będzie się zmieniać”, dzięki czemu przeglądarka może z wyprzedzeniem przygotować pewne optymalizacje i zapewnić płynność animacji. Przygotowanie z wyprzedzeniem jest zwykle lepsze niż mieszanie się w ostatniej chwili.

### Opcja 3: Boczna linia świecąca

Jest to szczególnie przydatne w przypadku wskaźników stanu pozycji listy i dokładnie tego używa lista sesji HagiCode. Jedna cienka linia może nadal wyróżniać się spośród wielu przedmiotów. To na swój sposób lekcja życia.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

To wykorzystuje `isolation: isolate` aby utworzyć nowy kontekst układania, a następnie opiera się na `z-index` aby kontrolować kolejność wyświetlania każdej warstwy. `pointer-events: none` jest również niezbędne. W przeciwnym razie pseudoelement blokowałby kliknięcia użytkownika. Niektóre rzeczy mogą wyglądać ładnie, ale nadal nie powinny przeszkadzać.

### Opcja 4: Zawiń go w komponencie React

Jeśli Twój projekt korzysta z React, możesz zawinąć tę logikę w komponent, zwłaszcza obsługę dostępności. Napisz raz, użyj wiele razy. O to właśnie chodzi.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Pasujący moduł CSS:

```css
/* GlowBorder.module.css */

/* Animated version */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Static version (accessibility) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

The `useReducedMotion` hak od `framer-motion` automatycznie wykrywa preferencje systemowe użytkownika. Jeśli użytkownik włączył ograniczenie ruchu, nastąpi powrót `true`, a zamiast tego komponent wyświetla wersję statyczną. Szacunek do preferencji użytkownika ma większe znaczenie niż wymuszanie krzykliwego efektu.

## Wyciągnięte wnioski praktyczne

Oto niektóre z lekcji, których nauczyliśmy się podczas tworzenia HagiCode. Można je również nazwać bliznami bojowymi. Mamy nadzieję, że pomogą Ci uniknąć niektórych objazdów.

### 1. System zmiennych motywów

Zmienne CSS sprawiają, że obsługa wielu motywów jest szczególnie wygodna. Nikt nie chce edytować stosu kodu za każdym razem, gdy zmienia się motyw.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Usage */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

W ten sposób przełączanie motywów wymaga jedynie zmiany klasy w pliku `html` element, a każdy kolor animacji jest aktualizowany automatycznie. Jedna baza kodu, dwa style. Właśnie tego chcemy.

### 2. Optymalizacja wydajności

**Użyj `will-change` aby zasugerować przeglądarce optymalizację:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Powiadom przeglądarkę z wyprzedzeniem, a pomoże ona w optymalizacji. Wiele rzeczy w życiu działa lepiej przy odrobinie przygotowania.

**Unikaj stosowania skomplikowanych cieni prostokątnych na dużych elementach:**

```css
/* Not ideal - using a blurred shadow on a large element */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Better - use a pseudo-element to limit the glowing area */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Przetestowaliśmy to w HagiCode. Dodanie rozmytego cienia bezpośrednio do dużej karty obniżyło liczbę klatek przewijania poniżej 30 klatek na sekundę. Przejście na pseudoelement przywróciło stałe 60 klatek na sekundę. Użytkownicy mogą absolutnie poczuć tę różnicę.

### 3. Dostępność

Naprawdę nie powinieneś tego pomijać. Niektórym użytkownikom animacja przyprawia o zawrót głowy lub rozprasza, a respektowanie ich preferencji jest częścią budowania dobrego produktu. Nie każdemu trzeba narzucać piękne rzeczy.

**Zapytanie o media CSS:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Provide a static fallback */
    opacity: 1;
  }
}
```

**Wykryj preferencje użytkownika w React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Kontrola poziomu intensywności

Wskaźnik przepustowości tokena w HagiCode pokazuje różne kolory blasku w oparciu o przepustowość w czasie rzeczywistym i jest to realizowane dynamicznie. Różne stany należy wyrażać w różny sposób.

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Rzeczy, na które należy zwrócić uwagę

Jest jeszcze kilka szczegółów, na które warto zwrócić uwagę, bo zanim odkryjesz te problemy na własnej skórze, będzie już za późno.

| Rzeczy, na które należy zwrócić uwagę | Wyjaśnienie |
|---------|------|
| **Zarządzanie indeksem Z** | Warstwa blasku powinna używać odpowiedniego `z-index` więc nie zakłóca interakcji z treścią |
| **zdarzenia wskaźnikowe** | Powinien się ustawić pseudoelement blasku `pointer-events: none` |
| **Przekroczenie granicy** | Potrzebny jest kontener nadrzędny `overflow: hidden`lub musisz dostosować rozmiar pseudoelementów |
| **Wpływ na wydajność** | Złożone animacje mogą obniżyć wydajność na urządzeniach mobilnych, więc przetestuj je dokładnie |
| **Tryb ciemny** | Upewnij się, że kolor blasku jest wyraźnie widoczny na ciemnym tle |
| **Przełączanie motywu** | Użyj zmiennych CSS, aby kolory animacji aktualizowały się poprawnie po zmianie motywu |

### 6. Wskazówki dotyczące debugowania

Znalezienie pseudoelementów w narzędziach programistycznych może być trochę trudne, dlatego możesz tymczasowo dodać obramowanie, aby sprawdzić położenie.

```css
/* Temporarily show pseudo-element boundaries for debugging */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Po zakończeniu pozycjonowania pamiętaj o skomentowaniu lub usunięciu tej linii. W przeciwnym razie produkcja może dość szybko stać się niezręczna. Niektóre rzeczy lepiej pozostawić w fazie rozwoju.
