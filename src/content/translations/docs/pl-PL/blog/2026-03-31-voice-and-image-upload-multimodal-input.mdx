## Pisanie jest wolniejsze niż mówienie, a mówienie jest wolniejsze niż zrzut ekranu — praktyki wprowadzania multimodalnego dla asystentów kodowania AI

> Pisanie kodu ma ograniczenie prędkości, niezależnie od tego, jak szybko piszesz. Czasami przepisanie czegoś, co można powiedzieć w jednym zdaniu, zajmuje całą wieczność; czasami jeden zrzut ekranu wyjaśnia wszystko, a mimo to trzeba to opisać stosem tekstu. W tym artykule opisano, na co natknęliśmy się podczas budowy [Kod Hagi](https://hagicode.com), od rozpoznawania mowy po przesyłanie obrazów. Ostatecznie chcieliśmy po prostu uczynić asystenta kodowania AI nieco łatwiejszym w obsłudze.

## Tło

Podczas budowania HagiCode zauważyliśmy problem - a raczej problem, który naturalnie pojawił się, gdy ludzie zaczęli intensywnie go używać: poleganie na samym pisaniu może być męczące.

Pomyśl o tym: głównym scenariuszem jest interakcja między użytkownikami a Agentem. Ale jeśli każda wymiana wymaga ciągłego pisania na klawiaturze, wydajność nie jest świetna:

1. **Wpisywanie jest zbyt wolne**: W przypadku skomplikowanych problemów, takich jak komunikaty o błędach lub problemy z interfejsem użytkownika, wpisanie wszystkiego może zająć pół minuty, a wypowiedzenie tego na głos może zająć dziesięć sekund. Ta przepaść jest prawdziwa.

2. **Obrazy są bardziej bezpośrednie**: Czasami interfejs użytkownika zgłasza błąd, czasami chcesz porównać wersję roboczą projektu, a czasami musisz pokazać strukturę kodu. „Obraz jest wart tysiąca słów” to może stare powiedzenie, ale wciąż aktualne. Pozwolenie AI bezpośrednio „zobaczyć” problem jest znacznie jaśniejsze niż opisywanie go przez wieki.

3. **Interakcja powinna być naturalna**: Nowocześni asystenci AI powinni obsługiwać tekst, głos i obrazy. Użytkownicy powinni mieć możliwość wyboru metody wprowadzania tekstu, która wydaje im się najbardziej naturalna.

Dlatego zdecydowaliśmy się dodać obsługę rozpoznawania mowy i przesyłania obrazów do HagiCode, aby interakcje z agentami były wygodniejsze. Jeśli użytkownicy mogą pisać trochę mniej, to już jest wygrana.

## Analiza

### Wyzwania techniczne w rozpoznawaniu mowy

Podczas tworzenia funkcji rozpoznawania mowy napotkaliśmy trudny problem: **interfejs API WebSocket przeglądarki nie obsługuje niestandardowych nagłówków HTTP**.

Usługą rozpoznawania mowy, którą wybraliśmy, był interfejs API rozpoznawania mowy Doubao firmy ByteDance. Niestety ten interfejs API wymaga informacji uwierzytelniających, takich jak `accessToken` i `secretKey` przesyłane przez nagłówki HTTP. To spowodowało natychmiastowy konflikt techniczny:

```javascript
// The browser WebSocket API does not support this approach
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // Not supported
  }
});
```

W zasadzie mieliśmy dwie możliwości:

1. **Podejście oparte na parametrach zapytania URL**: umieść informacje uwierzytelniające w adresie URL
   - Zaletą jest to, że jest prosty w wykonaniu
   - Wadą jest to, że dane uwierzytelniające są udostępniane interfejsowi użytkownika, co jest niepewne; niektóre interfejsy API wymagają również uwierzytelniania opartego na nagłówkach

2. **Podejście oparte na serwerze proxy zaplecza**: zaimplementuj serwer proxy WebSocket na zapleczu
   - Zaletą jest to, że dane uwierzytelniające pozostają bezpiecznie przechowywane na zapleczu, a rozwiązanie jest w pełni kompatybilne z wymaganiami API
   - Wadą jest to, że implementacja jest nieco bardziej złożona

Ostatecznie wybraliśmy **podejście proxy backendu**. Bezpieczeństwo nie jest czymś, w czym można pójść na kompromis.

### Wymagania funkcjonalne dotyczące przesyłania obrazów

Nasze wymagania dotyczące przesyłania obrazów były w rzeczywistości dość proste:

1. **Wiele metod przesyłania**: kliknij, aby wybrać plik, przeciągnij i upuść, a następnie wklej ze schowka
2. **Weryfikacja pliku**: ograniczenia typu (PNG, JPG, WebP, GIF) i ograniczenia rozmiaru (5-10 MB) to wymagania podstawowe
3. **Doświadczenie użytkownika**: przesyłaj postęp, podglądy i komunikaty o błędach, aby użytkownicy zawsze wiedzieli, co się dzieje
4. **Bezpieczeństwo**: weryfikacja po stronie serwera i ochrona przed przesyłaniem złośliwych plików są niezbędne

## Rozwiązanie

### Rozpoznawanie mowy: architektura proxy WebSocket

Zaprojektowaliśmy trójwarstwową architekturę do rozpoznawania mowy i znaleźliśmy ścieżkę, która zadziałała:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**Implementacje podstawowych komponentów**:

1. **Procesor Frontend AudioWorklet**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Resample to 16 kHz (required by the Doubao API)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Accumulate samples into 500 ms chunks
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Convert to 16-bit PCM and send
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **Zaplecza obsługa protokołu WebSocket** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. ** Frontend `VoiceTextArea` składnik**:

```tsx
export const VoiceTextArea = forwardRef<HTMLTextAreaElement, VoiceTextAreaProps>(
  ({ value, onChange, onTextRecognized, maxDuration }, ref) => {
    const { isRecording, interimText, volume, duration, startRecording, stopRecording } =

    return (

  }
);
```

### Przesyłanie obrazów: Komponent przesyłania obejmujący wiele metod

Stworzyliśmy w pełni funkcjonalny komponent do przesyłania obrazów obsługujący wszystkie trzy metody przesyłania, obejmujący najczęstsze scenariusze, z którymi spotykają się użytkownicy.

**Podstawowe funkcje**:

1. **Trzy metody przesyłania**:

```tsx
// Click to upload
const handleClick = () => fileInputRef.current?.click();

// Drag-and-drop upload
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Clipboard paste
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **Weryfikacja frontendu**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **Proces obsługi przesyłania danych** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Validation
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Save file
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## Praktyczny przewodnik

### Jak korzystać z rozpoznawania mowy

1. **Skonfiguruj usługę rozpoznawania mowy**:
   - Otwórz stronę ustawień rozpoznawania mowy
   - Skonfiguruj mowę Doubao `AppId` i `AccessToken`
   - Opcjonalnie skonfiguruj słowa-klucze, aby poprawić dokładność rozpoznawania terminów specyficznych dla domeny

2. **Użyj go w polu wprowadzania**:
   - Kliknij ikonę mikrofonu po lewej stronie pola wprowadzania
   - Zacznij mówić po wyświetleniu animacji przebiegu
   - Kliknij ikonę ponownie, aby zatrzymać nagrywanie
   - Rozpoznany tekst jest automatycznie wstawiany w pozycji kursora

3. **Przykład konfiguracji słowa kluczowego**:

```text
TypeScript
React
useState
useEffect
```

### Jak korzystać z przesyłania obrazów

1. **Metody przesyłania**:
   - Kliknij przycisk przesyłania, aby wybrać plik
   - Przeciągnij obraz bezpośrednio do obszaru przesyłania
   - Użyj `Ctrl+V` aby wkleić zrzut ekranu ze schowka

2. **Obsługiwane formaty**: PNG, JPG, JPEG, WebP, GIF
3. **Limit rozmiaru**: domyślnie 5 MB (konfigurowalne)

### Notatki

1. **Rozpoznawanie mowy**:
   - Wymagane jest pozwolenie na korzystanie z mikrofonu
   - Jeśli to możliwe, używaj w cichym otoczeniu
   - Maksymalny obsługiwany czas nagrywania wynosi domyślnie 300 sekund (konfigurowalny)

2. **Przesyłanie obrazów**:
   - Obsługiwane są tylko popularne formaty obrazów
   - Zwróć uwagę na ograniczenia rozmiaru pliku
   - Przesłane obrazy automatycznie otrzymują adres URL podglądu

3. **względy bezpieczeństwa**:
   - Poświadczenia rozpoznawania mowy są przechowywane na zapleczu
   - Przesyłane obrazy przechodzą ścisłą weryfikację po stronie serwera
   - W środowiskach produkcyjnych zalecany jest protokół HTTPS/WSS

## Wniosek

Po dodaniu rozpoznawania mowy i przesyłania obrazów, doświadczenie użytkownika HagiCode uległo zauważalnej poprawie. Użytkownicy mogą teraz wchodzić w interakcję ze sztuczną inteligencją w bardziej naturalny sposób – mówić zamiast pisać i udostępniać zrzuty ekranu zamiast opisywać wszystko ręcznie. Mam wrażenie, że w końcu znalazłem wygodniejszy sposób komunikacji.

Podczas tworzenia tej funkcji napotkaliśmy problem polegający na tym, że interfejsy API protokołu WebSocket przeglądarki nie obsługują niestandardowych nagłówków. Ostatecznie rozwiązaliśmy ten problem, stosując podejście proxy zaplecza. To rozwiązanie nie tylko zapewniło bezpieczeństwo, ale także położyło podwaliny pod późniejszą integrację innych uwierzytelnionych usług WebSocket.

Komponent przesyłania obrazów również korzysta z obsługi wielu metod przesyłania, dzięki czemu użytkownicy mogą wybierać to, co jest w danym momencie najwygodniejsze. Kliknięcie, przeciągnięcie lub wklejenie całej pracy i każdej ścieżki powoduje szybkie wykonanie zadania.
