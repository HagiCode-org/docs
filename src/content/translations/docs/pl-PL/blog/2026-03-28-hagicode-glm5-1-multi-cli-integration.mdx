## Tło

W projekcie Hagicode użytkownicy mogą wybierać spośród wielu narzędzi CLI do obsługi asystentów programowania AI, w tym Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI i innych. Te narzędzia CLI same w sobie są narzędziami programowania AI ogólnego przeznaczenia, ale dzięki warstwie abstrakcji Hagicode mogą elastycznie łączyć się z różnymi dostawcami modeli AI.

Zhipu AI (ZAI) zapewnia interfejs kompatybilny z Anthropic Claude API, umożliwiając tym narzędziom CLI bezpośrednie korzystanie z krajowych modeli serii GLM. Wśród nich GLM-5.1 to najnowsza wersja wielkojęzycznego modelu firmy Zhipu, ze znacznymi ulepszeniami w stosunku do GLM-5.0.

## Architektura abstrakcji CLI Hagicode

Hagicode definiuje 11 typów dostawców CLI poprzez `AIProviderType` enum, obejmujący główne narzędzia CLI do programowania AI:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Każdy interfejs CLI ma odpowiednią konfigurację parametrów modelu i obsługuje `model` i `reasoning` parametry:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
```

## Używanie GLM-5.1 w różnych CLI

### 1. Kod Claude CLI + GLM-5.1

Claude Code CLI jest jednym z podstawowych interfejsów CLI Hagicode i jest konfigurowany poprzez system konfiguracji Hero:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Odpowiednie `HeroEquipmentCatalogItem` konfiguracja:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI jest najbardziej elastycznym interfejsem CLI i obsługuje określanie dowolnego modelu w pliku `provider/model` format:

**Metoda 1: Użyj prefiksu dostawcy ZAI**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Metoda 2: Użyj bezpośrednio identyfikatora modelu**
```json
{
  "model": "glm-5.1"
}
```

**Metoda 3: Interfejs konfiguracyjny frontendu**

w `HeroModelEquipmentForm.tsx`, OpenCode CLI ma dedykowaną wskazówkę zastępczą:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Użytkownicy mogą wprowadzić:
```
zai/glm-5.1
glm-5.1
```

**Logika analizowania modelu OpenCode CLI**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI ma dedykowaną logikę obsługi starszej wersji:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Uwaga**: Codebuddy zachowuje specjalną obsługę dla GLM-5.0 i nie używa starszej normalizacji:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## Zalety ulepszeń GLM-5.1

W porównaniu z GLM-5.0, GLM-5.1 wprowadza następujące znaczące ulepszenia:

### 1. Lepsza zdolność rozumowania

Według oficjalnych informacji o wydaniu Zhipu, ulepszenia w GLM-5.1 obejmują:

- **Lepsze zrozumienie kodu**: Dokładniejsza analiza złożonych struktur kodu
- **Dłuższe zrozumienie kontekstu**: Obsługuje dłuższy kontekst konwersacji
- **Ulepszone wywoływanie narzędzi**: Wyższy wskaźnik powodzenia wywołań narzędzi MCP
- **Stabilność wyjściowa**: Zmniejsza losowość i halucynacje

### 2. Kompleksowa kompatybilność z wieloma interfejsami CLI

GLM-5.1 obejmuje wszystkie główne interfejsy CLI obsługiwane przez Hagicode:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Notatki

### 1. Konfiguracja klucza API

Upewnij się, że `ANTHROPIC_AUTH_TOKEN` zmienna środowiskowa jest ustawiona poprawnie. Jest to wymagane poświadczenie dla każdego interfejsu CLI, aby połączyć się z modelem.

### 2. Dostępność modelu

GLM-5.1 musi być włączony przez odpowiedniego dostawcę modelu:
- Domyślnie obsługuje ją platforma Zhipu AI ZAI
- Alibaba Cloud DashScope może wymagać osobnej aplikacji

### 3. Format CLI OpenCode

Podczas korzystania z `provider/model` formacie, upewnij się, że identyfikator dostawcy jest poprawny:
- Zhipu AI: `zai` lub `zhipuai`
- Chmura Alibaba: `aliyun` lub `dashscope`

### 4. Parametr rozumowania

- `high` jest zalecane w celu uzyskania najlepszych wyników generowania kodu
- Gemini CLI nie obsługuje parametru wnioskowania i automatycznie zignoruje tę konfigurację

## Wniosek

Dzięki ujednoliconej warstwie abstrakcji Hagicode umożliwia elastyczną integrację pomiędzy GLM-5.1 i wieloma CLI. Programiści mogą wybrać narzędzie CLI, które najlepiej odpowiada ich preferencjom i scenariuszom użytkowania, a następnie skorzystać z najnowszego modelu GLM-5.1 poprzez prostą konfigurację.

Jako najnowsza wersja modelu Zhipu, GLM-5.1 oferuje wyraźne ulepszenia w stosunku do GLM-5.0:
- Niezależny identyfikator wersji bez obciążeń związanych ze starszymi wersjami
- Lepsze rozumowanie i zrozumienie kodu
- Szeroka kompatybilność z wieloma interfejsami CLI
- Elastyczna konfiguracja poziomu rozumowania

Po skonfigurowaniu odpowiednich zmiennych środowiskowych i skonfigurowaniu sprzętu Hero użytkownicy mogą w pełni odblokować moc GLM-5.1 w różnych środowiskach CLI.
