## Jak projektowanie grywalizacji sprawia, że kodowanie AI staje się przyjemniejsze

> Tradycyjne narzędzia do kodowania AI są w rzeczywistości dość potężne; po prostu brakuje im odrobiny ciepła. Kiedy budowaliśmy HagiCode, pomyśleliśmy: jeśli i tak mamy pisać kod, dlaczego nie przekształcić go w grę?

## Tło

Każdy, kto korzystał z asystenta kodowania AI, prawdopodobnie miał takie doświadczenie: na początku wydaje się świeże i ekscytujące, ale po pewnym czasie zaczyna mieć wrażenie, że czegoś brakuje. Samo narzędzie jest potężne, potrafi generować kod, autouzupełniać i naprawiać błędy, ale… nie sprawia wrażenia zbyt ciepłego, a z czasem może stać się monotonne i nudne.

Już samo to wystarczy, by zacząć się zastanawiać, kto chciałby codziennie wpatrywać się w zimne, bezosobowe narzędzie.

To trochę jak granie w grę. Jeśli jedyne, co musisz zrobić, to ukończyć listę zadań, bez rozwoju postaci, odblokowania osiągnięć i koordynacji zespołu, szybko przestaje to być zabawą. Pięknych rzeczy i ludzi nie trzeba posiadać, żeby je docenić; ich piękno wystarczy samo w sobie. Narzędzia programistyczne nie oferują nawet takiego piękna, więc łatwo stracić zapał.

Podczas opracowywania napotkaliśmy dokładnie ten problem [Kod Hagi](https://hagicode.com). Jako platforma współpracy z wieloma asystentami AI, HagiCode musi utrzymywać zaangażowanie użytkowników w dłuższej perspektywie. Ale w rzeczywistości nawet świetne narzędzie jest trudne do utrzymania, jeśli brakuje mu żadnego związku emocjonalnego.

Aby rozwiązać ten problem, podjęliśmy odważną decyzję: zamienić programowanie w grę. Nie jest to powierzchowna gra z prostą tabelą wyników, ale prawdziwe doświadczenie polegające na odgrywaniu ról. Skutki tej decyzji mogą być jeszcze większe, niż sobie wyobrażasz.

W końcu ludzie potrzebują odrobiny rytuału w swoim życiu.

## O HagiCodzie

Pomysły przedstawione w tym artykule pochodzą z naszych praktycznych doświadczeń związanych z [Kod Hagi](https://hagicode.com) projekt. HagiCode to platforma współpracy z wieloma asystentami AI, która obsługuje współpracę Claude Code, Codex, Copilot, OpenCode i innych asystentów AI. Jeśli interesuje Cię współpraca z wieloma sztucznymi inteligencjami lub programowanie z wykorzystaniem gier, odwiedź stronę [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site) aby dowiedzieć się więcej.

Nie ma w tym nic szczególnie tajemniczego. Po prostu zamieniliśmy programowanie w przygodę.

## Dlaczego warto wybrać grywalizację

Istotą grywalizacji nie jest tylko „dodanie tabeli liderów”. Chodzi o zbudowanie kompletnego systemu motywacyjnego, aby użytkownicy podczas wykonywania zadań mogli odczuć rozwój, osiągnięcia i uznanie społeczne.

Projekt grywalizacji HagiCode opiera się na jednej podstawowej idei: każdy asystent AI jest „Bohaterem”, a użytkownik jest kapitanem tego zespołu Bohaterów. Prowadzisz tych Bohaterów do podboju różnych „Lochów” (zadania programistyczne). Po drodze Bohaterowie zdobywają doświadczenie, awansują na wyższy poziom, odblokowują umiejętności, a Twój zespół również zdobywa osiągnięcia.

To nie jest chwyt marketingowy. Jest to projekt oparty na psychologii behawioralnej człowieka. Kiedy zadaniom nadano znaczenie i uzyskano informację zwrotną o postępie, zaangażowanie i wytrwałość ludzi znacznie wzrastają.

Jak mówi stare powiedzenie: „To uczucie może stać się wspomnieniem, chociaż w tamtym momencie wprawiło nas w oszołomienie”. Wnosimy te emocjonalne doświadczenia do narzędzia, więc programowanie nie jest już tylko wpisywaniem kodu, ale podróżą wartą zapamiętania.

## System postaci bohatera

Bohater to podstawowa koncepcja systemu grywalizacji HagiCode. Każdy Bohater reprezentuje jednego asystenta AI. Na przykład Claude Code jest Bohaterem i Codex także jest Bohaterem.

### Trzy miejsca na bohaterów

Bohater ma trzy miejsca na wyposażenie, a jego konstrukcja jest zaskakująco elegancka:

1. **Slot CLI (klasa główna)**: Określa podstawową zdolność Bohatera, np. czy jest to Kodeks Claude’a, czy Kodeks
2. **Gniazdo modelu (klasa drugorzędna)**: Określa używany model, np. Claude 4.5 lub Claude 4.6
3. **Slot na styl (styl)**: Określa styl zachowania Bohatera, np. „Strategista Fengluo” lub inny styl

Połączenie tych trzech miejsc tworzy unikalne konfiguracje Bohaterów. Podobnie jak w przypadku budowania sprzętu w grach, właściwą konfigurację wybierasz w zależności od zadania. W końcu to, co najbardziej Ci odpowiada, jest najważniejsze. Życie jest podobne: wiele dróg prowadzi do Rzymu, ale niektóre są gładsze od innych.

### System wzrostu bohatera

Każdy bohater ma swoje własne XP i poziom:

```typescript
type HeroProgressionSnapshot = {
  currentLevel: number;                    // Current level
  totalExperience: number;                 // Total experience
  currentLevelStartExperience: number;     // Experience at the start of the current level
  nextLevelExperience: number;             // Experience required for the next level
  experienceProgressPercent: number;       // Progress percentage
  remainingExperienceToNextLevel: number;  // Experience still needed for the next level
  lastExperienceGain: number;              // Most recent experience gained
  lastExperienceGainAtUtc?: string | null; // Time when experience was gained
};
```

Poziomy podzielone są na cztery etapy, a każdy etap ma wciągającą nazwę:

```typescript
export const resolveHeroProgressionStage = (level?: number | null): HeroProgressionStage => {
  const normalizedLevel = Math.max(1, level ?? 1);
  if (normalizedLevel <= 100) return 'rookieSprint';      // Rookie sprint
  if (normalizedLevel <= 300) return 'growthRun';         // Growth run
  if (normalizedLevel <= 700) return 'veteranClimb';      // Veteran climb
  return 'legendMarathon';                                // Legend marathon
};
```

Od „nowicjusza” do „legendy” – ta ścieżka rozwoju daje użytkownikom jasne poczucie kierunku i osiągnięć. Odzwierciedla osobisty rozwój w życiu, od zamętu do dojrzałości, tutaj tylko bardziej namacalny.

### Tworzenie niestandardowego bohatera

Aby stworzyć Bohatera, musisz skonfigurować trzy miejsca:

```typescript
const heroDraft: HeroDraft = {
  name: 'Athena',
  icon: 'hero-avatar:storm-03',
  description: 'A brilliant strategist',
  executorType: AIProviderType.CLAUDE_CODE_CLI,
  slots: {
    cli: {
      id: 'profession-claude-code',
      parameters: { /* CLI-related parameters */ }
    },
    model: {
      id: 'secondary-claude-4-sonnet',
      parameters: { /* Model-related parameters */ }
    },
    style: {
      id: 'fengluo-strategist',
      parameters: { /* Style-related parameters */ }
    }
  }
};
```

Każdy bohater ma unikalny awatar, opis i tożsamość zawodową, co nadaje temu, co w innym przypadku byłoby zimnym asystentem AI, większej osobowości i ciepła. W końcu kto chce pracować narzędziem, które nie ma charakteru?

## System Lochów

„Lochy” to klasyczna koncepcja gry stanowiąca wyzwanie, które wymaga pokonania przez zespół. W HagiCode każdy przepływ pracy jest Lochem.

### Jak zorganizowane są lochy

Dungeon organizuje przepływy pracy w różne „Lochy”:

- **Loch generowania propozycji**: Odpowiedzialny za generowanie propozycji technicznych
- **Loch realizacji propozycji**: Odpowiedzialny za realizację zadań w propozycjach
- **Loch archiwum wniosków**: Odpowiedzialny za organizację i archiwizację ukończonych wniosków

Każdy loch ma swojego własnego Kapitana Bohatera, który jest automatycznie wybierany na pierwszego włączonego Bohatera.

To tak naprawdę podział pracy, taki jak w życiu codziennym, tyle że przekształcony w mechanikę gry.

### Mechanizm współpracy zespołowej

Możesz skonfigurować różne drużyny bohaterów dla różnych lochów:

```typescript
const dungeonRoster: HeroDungeonRoster = {
  scriptKey: 'proposal.generate',
  displayName: 'Proposal Generation',
  members: [
    { heroId: 'hero-1', name: 'Athena', executorType: 'ClaudeCode' },
    { heroId: 'hero-2', name: 'Apollo', executorType: 'Codex' }
  ]
};
```

Na przykład możesz użyć Ateny do generowania propozycji, ponieważ jest dobra w strategii, a Apollo do implementowania kodu, ponieważ jest dobry w wykonywaniu. W ten sposób każdy Bohater będzie mógł wykorzystać swoje mocne strony. To jak z założeniem zespołu: każda osoba ma instrument i razem tworzą coś pięknego.

### Kontrola przepływu w lochach

Dungeon używa naprawiono `scriptKey` wartości identyfikujące różne przepływy pracy:

```typescript
// Script keys map to different workflows
const dungeonScripts = {
  'proposal.generate': 'Proposal Generation',
  'proposal.execute': 'Proposal Execution',
  'proposal.archive': 'Proposal Archive'
};
```

Przepływ stanu zadania to: `queued` (czekam) -> `dispatching` (przypisany) -> `dispatched` (przypisany). Cały proces jest zautomatyzowany i nie wymaga ręcznej interwencji. To także część naszej leniwej strony, bo kto chce zarządzać tymi rzeczami ręcznie?

## System XP i poziomów

XP to podstawowy mechanizm sprzężenia zwrotnego w systemie grywalizacji. Użytkownicy zdobywają PD, wykonując zadania, podnoszą poziom PD Bohaterów, a zdobywanie kolejnych poziomów odblokowuje nowe umiejętności, tworząc pętlę pozytywnego sprzężenia zwrotnego.

### Sposoby zdobywania XP

W HagiCode XP można zdobyć poprzez następujące czynności:

- Kończenie wykonywania kodu
- Pomyślnie wywołano narzędzia
- Generowanie propozycji
- Operacje zarządzania sesją
- Operacje projektowe

Za każdym razem, gdy wykonana zostanie odpowiednia akcja, odpowiedni Bohater zdobywa PD. Podobnie jak w życiu, liczy się każdy krok, tylko tutaj wzrost jest ilościowy.

### Wizualizacja postępu w czasie rzeczywistym

Postęp XP i poziomu jest wizualizowany w czasie rzeczywistym:

```typescript
type HeroDungeonMember = {
  heroId: string;
  name: string;
  icon?: string | null;
  executorType: PCode_Models_AIProviderType;
  currentLevel?: number;                    // Current level
  totalExperience?: number;                 // Total experience
  experienceProgressPercent?: number;       // Progress percentage
};
```

Użytkownicy zawsze mogą zobaczyć poziom i postęp każdego Bohatera, a natychmiastowa informacja zwrotna jest kluczem do projektowania grywalizacji. Ludzie potrzebują informacji zwrotnej, w przeciwnym razie skąd wiedzieliby, że się poprawiają?

## System osiągnięć

Osiągnięcia to kolejny ważny element grywalizacji. Zapewniają długoterminowe cele i satysfakcję wynikającą z kamieni milowych.

### Typy osiągnięć

HagiCode obsługuje wiele typów osiągnięć:

- **Osiągnięcia w zakresie generowania kodu**: Wygeneruj X linii kodu, wygeneruj Y plików
- **Osiągnięcia w zarządzaniu sesją**: Kompletne rozmowy Z
- **Osiągnięcia w zakresie projektów**: Praca w projektach W

Te osiągnięcia są naprawdę jak kamienie milowe w życiu, tyle że zamieniliśmy je w mechanikę gry.

### Stany osiągnięć

Osiągnięcia mają trzy stany:

```typescript
type AchievementStatus = 'unlocked' | 'in-progress' | 'locked';
```

Te trzy stany mają wyraźne różnice wizualne:

- **Odblokowane**: Złoty gradient z efektem aureoli
- **W toku**: Animacja niebieskiego pulsu
- **Zablokowany**: Szary, z pokazanymi warunkami odblokowania

Każde osiągnięcie wyraźnie wyświetla warunek aktywacji, dzięki czemu użytkownicy wiedzą, co robić dalej. Kiedy ludzie czują się zagubieni, odrobina wskazówek zawsze się przyda.

### Efekt świętowania po odblokowaniu

Po odblokowaniu osiągnięcia uruchamiana jest animacja świętowania. Tego rodzaju pozytywne wzmocnienie daje użytkownikom satysfakcjonujące poczucie „udało mi się” i motywuje ich do dalszego działania. Małe nagrody w życiu działają w ten sam sposób: mogą być małe, ale szczęście może trwać długo.

## Raport bitewny Codzienny raport bojowy

Raport bitewny jest jedną z charakterystycznych funkcji HagiCode. Na koniec każdego dnia generuje pełnoekranowy raport przypominający bitwę.

### Zgłoś treść

Raport bitwy wyświetla następujące informacje:

```typescript
type HeroBattleReport = {
  reportDate: string;
  summary: {
    totalHeroCount: number;        // Total number of Heroes
    activeHeroCount: number;       // Number of active Heroes
    totalBattleScore: number;      // Total battle score
    mvp: HeroBattleHero;           // Most valuable Hero
  };
  heroes: HeroBattleHero[];        // Detailed data for all Heroes
};
```

- Łączny wynik drużyny
- Liczba aktywnych Bohaterów
- Liczba wywołań narzędzi
- Całkowity czas pracy
- MVP (najcenniejszy bohater)
- Szczegółowa karta dla każdego Bohatera

### Wyświetlanie wyróżnień MVP

MVP jest Bohaterem dnia, który osiągnął najlepsze wyniki i jest on wyróżniony w raporcie. To nie tylko statystyki danych, ale forma honoru i uznania. W końcu kto nie chciałby być rozpoznawany?

### Szczegółowe karty bohaterów

Każda karta Bohatera zawiera:

- Postęp na poziomie
- Zdobyte XP
- Liczba egzekucji
- Czas użytkowania

Te wskaźniki pomagają użytkownikom jasno zrozumieć, jak radzi sobie zespół. Samo zobaczenie efektów własnego wysiłku jest satysfakcjonujące.

## Realizacja techniczna

System grywalizacji HagiCode wykorzystuje nowoczesny stos technologii i wzorce projektowe. Nie ma w tym nic szczególnie magicznego; po prostu wybraliśmy narzędzia, które pasują do zadania.

### Wybór stosu technologii

```typescript
// React + TypeScript for the frontend
import React from 'react';

// Framer Motion for animations
import { AnimatePresence, motion } from 'framer-motion';

// Redux Toolkit for state management
import { useAppDispatch, useAppSelector } from '@/store';

// shadcn/ui for UI components
import { Dialog, DialogContent } from '@/components/ui/dialog';
```

Framer Motion obsługuje wszystkie efekty animacji, shadcn/ui zapewnia podstawowe komponenty interfejsu użytkownika, a Redux Toolkit zarządza złożonym stanem grywalizacji. Dobre narzędzia zapewniają dobrą pracę.

### Zintegrowany system projektowania interfejsu użytkownika

HagiCode wykorzystuje styl projektowania Glassmorphism + Tech Dark:

```css
/* Primary gradient */
background: linear-gradient(135deg, #22C55E 0%, #25c2a0 50%, #06b6d4 100%);

/* Glass effect */
backdrop-filter: blur(12px);

/* Glow effect */
background: radial-gradient(circle at center, rgba(34, 197, 94, 0.15) 0%, transparent 70%);
```

Zielony gradient w połączeniu ze glassmorfizmem tworzy techniczną, futurystyczną atmosferę. Piękno wizualne jest również częścią doświadczenia użytkownika.

### Efekty animacji

Framer Motion służy do tworzenia płynnych animacji wejścia:

```typescript
<motion.div
  animate={{ opacity: 1, y: 0 }}
  initial={{ opacity: 0, y: 18 }}
  transition={{ duration: 0.35, ease: 'easeOut', delay: index * 0.08 }}
  className="card"
>
  {/* Card content */}
</motion.div>
```

Każda karta wchodzi jedna po drugiej z opóźnieniem 0,08 sekundy, tworząc płynny efekt wizualny. Płynna animacja poprawia wrażenia. Z tym fragmentem trudno polemizować.

### Trwałość danych

Dane grywalizacji są przechowywane przy użyciu systemu przechowywania Grain, aby zapewnić spójność stanu. Nawet drobnoziarniste dane, takie jak zgromadzone PD Bohatera, mogą zostać dokładnie utrwalone. Nikt nie chce stracić doświadczenia, na które ciężko pracował.

## Praktyczny przewodnik

### Stwórz swojego pierwszego bohatera

Tworzenie pierwszego Bohatera jest w rzeczywistości całkiem proste:

1. Przejdź do strony zarządzania bohaterem
2. Kliknij przycisk „Utwórz bohatera”.
3. Skonfiguruj trzy gniazda (CLI, Model, Style)
4. Nadaj Bohaterowi imię i opis
5. Zapisz go, a narodzi się twój pierwszy Bohater

To jak spotkanie nowego przyjaciela: nadajesz mu imię, dowiadujesz się, co czyni go wyjątkowym, a następnie wyruszacie razem na przygodę.

### Zbuduj zespół w lochach

Budowanie zespołu jest również proste:

1. Przejdź do strony zarządzania lochami
2. Wybierz loch, który chcesz skonfigurować, np. „Generowanie propozycji”
3. Wybierz członków ze swojej listy Bohaterów
4. System automatycznie wybiera pierwszego włączonego Bohatera na Kapitana
5. Zapisz konfigurację

Jest to po prostu proces tworzenia zespołu, podobny do budowania zespołu w prawdziwym życiu, gdzie każdy ma swoją rolę.

### Zobacz raport dzienny

Na koniec każdego dnia możesz wyświetlić raport z bitwy:

1. Kliknij przycisk „Raport z bitwy”.
2. Przeglądaj wyniki dziennej pracy na wyświetlaczu pełnoekranowym
3. Sprawdź MVP i szczegółowe dane każdego Bohatera
4. Jeśli chcesz, udostępnij go członkom zespołu

To także swego rodzaju rytuał, sposób na sprawdzenie, ile wysiłku włożyłeś dzisiaj i jak daleko jesteś jeszcze od celu.

## Uwagi i najlepsze praktyki

### Optymalizacja wydajności

Użyj `React.memo` aby uniknąć niepotrzebnych ponownych renderowań:

```typescript
const HeroCard = React.memo(({ hero }: { hero: HeroDungeonMember }) => {
  // Component implementation
});
```

Wydajność też ma znaczenie. Nikt nie chce używać opóźnionego narzędzia.

### Ruch może ulegać stopniowej degradacji

Wykryj ustawienia preferencji ruchu użytkownika i zapewnij uproszczoną obsługę użytkownikom wrażliwym na ruch:

```typescript
const prefersReducedMotion = useReducedMotion();
const duration = prefersReducedMotion ? 0 : 0.35;
```

Nie każdy lubi animację, a respektowanie preferencji użytkownika jest częścią dobrego projektu.

### Kompatybilność wsteczna

Zachowaj `legacyIds` w celu obsługi migracji ze starszych wersji:

```typescript
type HeroDungeonMember = {
  heroId: string;
  legacyIds?: string[];  // Supports legacy ID mapping
  // ...
};
```

Nikt nie chce stracić danych tylko z powodu aktualizacji wersji.

### Wsparcie internacjonalizacji

Użyj kluczy tłumaczących i18n dla całego tekstu, aby ułatwić obsługę wielu języków:

```typescript
const displayName = t(`dungeon.${scriptKey}`, { defaultValue: displayName });
```

Język nigdy nie powinien być przeszkodą w korzystaniu z produktu.

## Podsumowanie

Grywalizacja to nie tylko prosta tabela liderów punktowych, ale kompletny system motywacyjny. Dzięki systemowi Hero, systemowi Dungeon, systemowi XP i poziomów, systemowi osiągnięć i raportowi bitewnemu, HagiCode przekształca pracę programistyczną w heroiczną podróż pełną przygód.

Podstawową wartością tego systemu jest:

- **Połączenie emocjonalne**: Nadanie zimnej osobowości asystentom AI
- **Pozytywny feedback**: Każde działanie daje natychmiastową informację zwrotną
- **Cele długoterminowe**: Poziomy i osiągnięcia zapewniają ścieżkę rozwoju
- **Tożsamość zespołu**: Poczucie współpracy w zespołach Dungeon
- **Honor i uznanie**: Raport bitewny i pokazy MVP

Projektowanie grywalne sprawia, że programowanie nie jest już nudne, ale staje się ciekawą przygodą. Wykonując zadania związane z kodowaniem, użytkownicy mogą także czerpać przyjemność z rozwoju postaci, współpracy zespołowej i odblokowywania osiągnięć, co poprawia retencję i aktywność.

W swej istocie programowanie jest już aktem tworzenia. Po prostu sprawiliśmy, że proces twórczy stał się trochę zabawniejszy.
