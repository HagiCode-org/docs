Jeśli chcesz zainstalować Hermesa i zacząć go używać, najkrótsza ścieżka to tak naprawdę tylko trzy kroki:

1. Uruchom oficjalne polecenie instalacji
2. Uruchom interfejs CLI w terminalu za pomocą `hermes`
3. Jeśli chcesz nadal używać go w Feishu, skonfiguruj `hermes gateway setup`

Artykuł ten nie jest próbą wyjaśnienia wszystkich możliwości Hermesa na raz. Zamiast tego pomaga najpierw ukończyć najważniejszą pętlę dla początkujących: zainstalować, uruchomić, zacząć używać, a następnie podłączyć do jednej z najpopularniejszych platform do przesyłania wiadomości.

## Czym jest Hermes i dla kogo jest przeznaczony

Hermes Agent to agent AI, którego można używać z lokalnego terminala lub za pośrednictwem bramy platformy komunikacyjnej.

Dla większości programistów ma dwa typowe punkty wejścia:

- **CLI**: Wpisz `hermes` w terminalu, aby bezpośrednio wejść do interaktywnego interfejsu.
- **Bramka wiadomości**: Uruchom `hermes gateway`, a następnie rozmawiaj z nim na platformach takich jak Feishu, Telegram, Discord i Slack.

Jeśli Twoim celem w tej chwili jest po prostu szybkie rozpoczęcie pracy, nie odwracaj kolejności. Zamiast tego zacznij od tej ścieżki:

- Najpierw zainstaluj Hermesa
- Najpierw sprawdź, czy działa, korzystając z interfejsu CLI
- Następnie zdecyduj, czy chcesz podłączyć platformę do przesyłania wiadomości

Ułatwia to diagnozowanie problemów i jest bardziej odpowiednie dla osób korzystających z Hermesa po raz pierwszy.

## Co warto wiedzieć przed instalacją Hermesa

Zgodnie z plikiem README Hermes oficjalna ścieżka szybkiej instalacji obsługuje następujące środowiska:

- Linuksa
- macOS
- WSL2
- Android przez Termux

### Uwagi dla użytkowników systemu Windows

Hermes nie obsługuje obecnie uruchamiania bezpośrednio w natywnym systemie Windows. Jeśli używasz systemu Windows, zalecanym podejściem jest najpierw zainstalowanie WSL2, a następnie uruchomienie polecenia instalacji w WSL2.

Najlepiej wyjaśnić to na początku, ponieważ wiele niepowodzeń instalacji nie jest spowodowanych przez samo polecenie, ale przez użycie nieobsługiwanego środowiska wykonawczego.

## Jak zainstalować Hermesa

Polecenie szybkiej instalacji zawarte w pliku README Hermes to:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

To polecenie uruchamia oficjalny skrypt instalacyjny i obsługuje etapy inicjalizacji specyficzne dla platformy.

### Załaduj ponownie powłokę po instalacji

Po zakończeniu instalacji najpierw załaduj ponownie środowisko Shell. Najczęstszym poleceniem jest:

```bash
source ~/.bashrc
```

Jeśli używasz Zsh, możesz użyć:

```bash
source ~/.zshrc
```

### Jak sprawdzić, czy Hermes jest poprawnie zainstalowany

Najbardziej bezpośrednim sposobem sprawdzenia jest uruchomienie:

```bash
hermes
```

Jeśli chcesz dodatkowego potwierdzenia, że konfiguracja i zależności działają, możesz także uruchomić:

```bash
hermes doctor
```

`hermes doctor` jest szczególnie przydatny w następujących sytuacjach:

- Polecenie zachowuje się nieprawidłowo po instalacji
- Konfiguracja modelu nie powiodła się
- Brama nie uruchamia się
- Nie masz pewności, czy zależności środowiskowe są kompletne

## Jak zacząć korzystać z Hermesa po raz pierwszy

### Zacznij od interfejsu CLI

Jeśli chcesz po prostu jak najszybciej potwierdzić, że Hermes działa, najprostszą metodą jest:

```bash
hermes
```

Spowoduje to uruchomienie interaktywnego interfejsu wiersza polecenia Hermes. Dla początkujących użytkowników Hermesa jest to również najbardziej zalecany punkt wyjścia, ponieważ można najpierw zweryfikować najważniejsze rzeczy:

- Czy polecenie jest rzeczywiście dostępne
- Czy bieżąca konfiguracja modelu działa poprawnie
- Czy łańcuch narzędzi terminala działa poprawnie
- Czy styl interakcji odpowiada Twoim potrzebom

### Te polecenia wystarczą do pierwszej rundy konfiguracji

Plik README Hermes zawiera kilka poleceń o wysokiej częstotliwości, które razem tworzą praktyczną ścieżkę pierwszego użycia:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Jeśli nie jesteś pewien, co każdy z nich robi, zapamiętaj je w ten sposób:

- `hermes model`: wybierz lub zmień modele
- `hermes tools`: przeglądanie i konfiguracja aktualnie dostępnych narzędzi
- `hermes config set`: zmiana określonych elementów konfiguracji
- `hermes setup`: uruchom kreatora pełnej inicjalizacji raz
- `hermes update`: aktualizacja Hermesa
- `hermes doctor`: rozwiązywanie problemów

Dla początkujących najbardziej praktyczna kolejność to zazwyczaj:

1. Biegnij `hermes model` pierwszy
2. Jeśli chcesz skonfigurować wszystkie typowe opcje na raz, uruchom `hermes setup`

## Dwa najczęstsze sposoby wykorzystania Hermesa

### 1. Użyj Hermesa w terminalu jako codziennego asystenta programisty

Tryb CLI dobrze sprawdza się w następujących scenariuszach:

- Zadawaj pytania bezpośrednio, pisząc kod lokalnie
- Sprawdzaj projekty, edytuj pliki i uruchamiaj polecenia
- Wykonaj jednorazowe debugowanie lub przegląd
- Współpracuj stale w bieżącym katalogu roboczym

Jego największą zaletą jest to, że jest to najkrótsza ścieżka: brak dodatkowej integracji z platformą, brak konfiguracji bota do obsługi z góry i jest to najlepszy sposób na zbudowanie pierwszego zestawu nawyków użytkowania.

### 2. Korzystaj z Hermes za pośrednictwem platformy komunikacyjnej

Jeśli chcesz rozmawiać z Hermesem na platformach takich jak Feishu, Telegram lub Discord, musisz skorzystać z bramki komunikacyjnej.

Najczęstsze polecenia wejściowe to:

```bash
hermes gateway setup
hermes gateway
```

Konkretnie:

- `hermes gateway setup` służy do interaktywnej konfiguracji platformy
- `hermes gateway` służy do rozpoczęcia procesu bramy

Zgodnie z oficjalną dokumentacją brama to ujednolicony proces w tle, który łączy skonfigurowane platformy, zarządza sesjami i obsługuje funkcje, takie jak zadania cron.

## Na przykładzie Feishu: jak podłączyć Hermesa do platformy komunikacyjnej

Jeśli większość Twojej codziennej pracy odbywa się w Feishu, Feishu/Lark jest bardzo naturalnym sposobem wykorzystania Hermesa.

### Minimalna realna ścieżka integracji

Oficjalna dokumentacja zaleca to polecenie wejścia dla Feishu/Lark:

```bash
hermes gateway setup
```

Po uruchomieniu wybierz po prostu **Feishu / Lark** w kreatorze.

Dokumentacja Feishu opisuje dwa tryby połączenia:

- `websocket`: zalecane
- `webhook`: opcjonalne

Jeśli Hermes działa na Twoim laptopie, stacji roboczej lub prywatnym serwerze, użyj `websocket` pierwszy jest zwykle prostszy, ponieważ nie trzeba ujawniać publicznego adresu URL wywołania zwrotnego.

### Jeśli skonfigurujesz to ręcznie, poznaj przynajmniej te zmienne

Jeśli nie korzystasz z kreatora i piszesz konfigurację ręcznie, dokumentacja Feishu zawiera listę następujących podstawowych zmiennych:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Dwa z nich zasługują na szczególną uwagę:

- `FEISHU_ALLOWED_USERS`: zalecane, więc nie każdy, kto może połączyć się z botem, może z niego bezpośrednio skorzystać
- `FEISHU_HOME_CHANNEL`: umożliwia wstępne zdefiniowanie czatu domowego, aby otrzymywać wyniki cron lub domyślne powiadomienia

### Dlaczego Hermes czasami nie odpowiada na czatach grupowych Feishu

Ten szczegół łatwo przeoczyć: na czatach grupowych Feishu Hermes domyślnie nie odpowiada na każdą wiadomość.

Oficjalna dokumentacja wyraźnie stwierdza:

- W wiadomościach bezpośrednich Hermes odpowiada na wiadomości
- Na czatach grupowych musisz to wyraźnie wyrazić `@` bota, zanim przetworzy wiadomość

Jeśli chcesz ustawić rozmowę Feishu jako kanał domowy, możesz również użyć tego na czacie:

```text
/set-home
```

Lub zdefiniuj to wcześniej w konfiguracji:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Początkujący powinni najpierw zapamiętać polecenia Hermesa

Niezależnie od tego, czy używasz Hermesa w interfejsie CLI, czy na platformie komunikacyjnej, na początek wystarczy zapamiętanie następujących poleceń:

- `/new` lub `/reset`: rozpocznij nową sesję
- `/model`: zobacz lub zmień model
- `/retry`: spróbuj ponownie wykonać poprzednią turę
- `/undo`: cofa poprzednią interakcję
- `/compress`: ręcznie skompresuj kontekst
- `/help`: wyświetl pomoc

Jeśli używasz Hermesa głównie na platformie komunikacyjnej, pamiętaj o jeszcze jednym:

- `/sethome` lub `/set-home`: ustaw bieżący czat jako kanał domowy

Polecenia te obejmują najczęstsze operacje na etapie początkującym: ponowne uruchamianie, dostosowywanie, wycofywanie, sprawdzanie i kontynuowanie.

## Często zadawane pytania

### Czy system Windows może zainstalować Hermes bezpośrednio?

Nie. Obecna oficjalna dokumentacja wyraźnie stwierdza, że natywny system Windows nie jest obsługiwany i zaleca się WSL2.

### Co powinienem zrobić, jeśli piszę `hermes` nic nie robi po instalacji?

Najlepiej rozwiązywać problemy w następującej kolejności:

1. Najpierw przeładuj powłokę, na przykład za pomocą `source ~/.bashrc`
2. Biegnij `hermes` ponownie
3. Jeśli nadal jest to nienormalne, uciekaj `hermes doctor`

### Dlaczego bot nie odpowiada w grupie Feishu?

Sprawdź najpierw te trzy rzeczy:

- Czy ty `@` wspomniał o Hermesie w grupie
- Czy `FEISHU_ALLOWED_USERS` ogranicza bieżącego użytkownika
- Czy obecne zasady czatu grupowego umożliwiają obsługę wiadomości grupowych

Zgodnie z oficjalną dokumentacją Feishu, w scenariuszach czatu grupowego wymagane jest wyraźne użycie @wzmianki.

## Zalecane kolejne kroki

Jeśli chcesz po prostu jak najszybciej zacząć korzystać z Hermesa, jest to najbardziej zalecana kolejność:

1. Najpierw uruchom polecenie instalacji

3. Użyj `hermes model` i `hermes setup` aby dokończyć podstawową konfigurację
4. Jeśli chcesz nadal używać go w Feishu, skonfiguruj `hermes gateway setup`
