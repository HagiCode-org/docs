## Pełna obsługa GLM-5.1 i integracja Gemini CLI: Ścieżka ewolucji wielu modeli HagiCode

> W tym artykule przedstawiono dwie główne aktualizacje platformy HagiCode: pełne wsparcie dla modelu Zhipu AI GLM-5.1 oraz udaną integrację Gemini CLI jako dziesiątego Agent CLI. Łącznie te aktualizacje jeszcze bardziej wzmacniają możliwości platformy w zakresie wielu modeli i ekosystemu wielu interfejsów CLI.

:::note[Uwaga dotycząca kontekstu czasowego]
W tym artykule opisano postęp rozbudowy wielu interfejsów CLI w momencie wydania. Aby zapoznać się z bieżącym, zawsze obsługiwanym zakresem, zobacz [Instalacja interfejsu CLI agenta AI](/pl-PL/related-software-installation/ai-agent-cli/) i [Przewodnik instalacji](/pl-PL/installation); the `IFlowCli` wspomniane tutaj stanowi jedynie konfigurację historyczną w tamtym czasie i nie jest częścią bieżącego aktywnego zakresu wsparcia.
:::

## Tło

Czas naprawdę płynie szybko. Rozwój dużych modeli językowych rośnie jak bambus na wiosnę. Jeszcze niedawno kibicowaliśmy „sztucznej inteligencji, która potrafi pisać kod”. Teraz jesteśmy już w epoce współpracy wielu modeli i integracji wielu narzędzi. Czy to ekscytujące? Być może. W końcu programiści nigdy nie potrzebowali samego narzędzia, ale łatwości dostosowywania się do różnych scenariuszy i elastycznego przełączania w razie potrzeby.

Jako platforma kodowania wspomagana sztuczną inteligencją, HagiCode niedawno z zadowoleniem przyjęła dwa ważne wydarzenia: po pierwsze, pełną integrację modelu GLM-5.1 firmy Zhipu AI; po drugie, oficjalne dodanie Gemini CLI jako dziesiątego obsługiwanego agenta CLI. Te dwie aktualizacje może nie brzmią wstrząsająco, ale są niewątpliwie dobrą wiadomością dla dalszego dojrzewania platformy.

GLM-5.1 to najnowszy flagowy model Zhipu AI. W porównaniu z GLM-5.0 oferuje lepsze rozumowanie, głębsze zrozumienie kodu i płynniejsze wywoływanie narzędzi. Co ważniejsze, jest to pierwszy model GLM obsługujący wprowadzanie obrazu. Co to oznacza? Oznacza to, że użytkownicy mogą pozwolić sztucznej inteligencji spojrzeć bezpośrednio na zrzut ekranu, zamiast męczyć się z opisywaniem problemu słowami. Kiedy już skorzystasz z tej wygody, od razu zrozumiesz jej wartość.

Jednocześnie poprzez `HagiCode.Libs.Providers` architektury, HagiCode z powodzeniem zintegrował Gemini CLI z platformą. To już dziesiąty CLI agenta. Szczerze mówiąc, dotarcie do tego punktu daje skromne poczucie spełnienia.

Warto również wspomnieć, że funkcja przesyłania obrazów HagiCode umożliwia użytkownikom komunikację z sztuczną inteligencją bezpośrednio za pomocą zrzutów ekranu. Nawet po uruchomieniu GLM 4.7 platforma nadal działa dobrze i pomogła już w wykonaniu wielu ważnych zadań kompilacji. W przypadku GLM-5.1 idzie to oczywiście o krok dalej.

## Projekt architektury Multi-CLI

Jedną z głównych zalet HagiCode jest obsługa wielu narzędzi CLI do programowania AI poprzez ujednoliconą warstwę abstrakcji. Zaleta tego projektu jest w zasadzie dość prosta: mogą pojawić się nowe narzędzia, stare mogą pozostać, a baza kodu nie zamienia się w chaos. Szczerze mówiąc, tak każdy chciałby, aby życie wyglądało.

### Wyliczenie AIProviderType

Platforma definiuje obsługiwane typy dostawców CLI poprzez `AIProviderType` wyliczenie:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI (new)
}
```

Jak widać, Gemini CLI dołącza do tej rodziny jako dziesiąty członek. Każdy interfejs CLI ma swoją własną, odrębną charakterystykę i scenariusze użycia, dzięki czemu użytkownicy mogą elastycznie wybierać w zależności od swoich potrzeb. Przecież wiele dróg prowadzi do Rzymu; niektóre są po prostu łatwiejsze niż inne.

### Architektura dostawcy

`HagiCode.Libs.Providers` zapewnia ujednolicony interfejs dostawcy, dzięki któremu każda integracja CLI jest ustandaryzowana i zwięzła. Biorąc za przykład Gemini CLI:

```csharp
public class GeminiProvider : ICliProvider<GeminiOptions>
{
    private static readonly string[] DefaultExecutableCandidates = ["gemini", "gemini-cli"];
    private const string ManagedBootstrapArgument = "--acp";

    public string Name => "gemini";
    public bool IsAvailable => _executableResolver.ResolveFirstAvailablePath(DefaultExecutableCandidates) is not null;
}
```

Zalety tego projektu to:
- Integracja nowego interfejsu CLI wymaga zaimplementowania tylko jednej klasy dostawcy
- Ujednolicone zarządzanie cyklem życia i łączenie sesji
- Zautomatyzowane rozpoznawanie aliasów i wykrywanie plików wykonywalnych

Mówiąc wprost, ten projekt zamienia skomplikowane rzeczy w prostsze i sprawia, że życie jest nieco łatwiejsze.

### Rejestr dostawców

Rejestr dostawców automatycznie obsługuje mapowanie i rejestrację aliasów:

```csharp
if (provider is GeminiProvider)
{
    registry.Register(provider.Name, provider, ["gemini-cli"]);
    continue;
}
```

Oznacza to, że użytkownicy mogą wywoływać interfejs Gemini CLI za pomocą dowolnego z nich `gemini` lub `gemini-cli`, a system rozpozna go automatycznie. To jak przyjaciel, który ma zarówno formalne imię, jak i przezwisko – tak czy inaczej, ludzie wiedzą, o kim mówisz.

## Obsługa modelu GLM-5.1

GLM-5.1 to najnowszy flagowy model Zhipu AI, dla którego HagiCode zakończył pełne wsparcie.

### Katalog zawodów średnich

HagiCode zarządza wszystkimi obsługiwanymi modelami poprzez Katalog Zawodów Dodatkowych. Oto konfiguracja dla serii GLM:

| Identyfikator modelu | Imię | Obsługuje obraz | Kompatybilne rodziny CLI |
|----------|------|----------------|-------------------------|
| `glm-4.7` | GLM 4.7 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM5 Turbo | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (starsza wersja) | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | **prawda** | claude, codebuddy, hermes, qoder, kiro |

Kluczowe cechy GLM-5.1 można podsumować w następujący sposób:
- Samodzielny identyfikator wersji bez bagażu starszego typu
- Pierwszy model GLM obsługujący wprowadzanie obrazu
- Lepsze rozumowanie i zrozumienie kodu
- Szeroka kompatybilność z wieloma interfejsami CLI

### GLM-5.1 kontra GLM-5.0

Na poziomie kodu pokazano tutaj kluczową różnicę między GLM-5.1 i GLM-5.0:

```csharp
// GLM-5.0 (Legacy) - contains special retention logic
private const string Glm50CodebuddySecondaryProfessionId = "secondary-glm-5-codebuddy";
private const string Glm50CodebuddyModelValue = "glm-5.0";

// GLM-5.1 - standalone new model identifier
private const string Glm51SecondaryProfessionId = "secondary-glm-5-1";
private const string Glm51ModelValue = "glm-5.1";
```

GLM-5.0 nosi etykietę „Starsza wersja”, ponieważ jest to identyfikator starej wersji zachowany ze względu na kompatybilność wsteczną. Natomiast GLM-5.1 to zupełnie nowa, samodzielna wersja, pozbawiona obciążeń historycznych. Niektóre rzeczy pozostają w przeszłości; inni podróżują lżej i poruszają się szybciej.

### Skonfiguruj GLM-5.1

Oto przykład konfiguracji użycia GLM-5.1 w HagiCode:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

## Funkcja przesyłania obrazów

Obsługa obrazów HagiCode jest realizowana poprzez `SupportsImage` nieruchomość na `SecondaryProfession`:

```csharp
public class HeroSecondaryProfessionSettingDto
{
    public bool SupportsImage { get; set; }
}
```

W Katalogu zawodów dodatkowych konfiguracja GLM-5.1 wygląda następująco:

```json
{
  "id": "secondary-glm-5-1",
  "supportsImage": true
}
```

Oznacza to, że użytkownicy mogą przesyłać zrzuty ekranu bezpośrednio do analizy AI, takie jak:
- Zrzuty ekranu komunikatów o błędach
- Problemy na ekranie interfejsu użytkownika
- Wykresy wizualizacji danych
- Wyniki wykonania kodu

Nie ma już potrzeby opisywania wszystkiego ręcznie – wystarczy przesłać zrzut ekranu. Wygoda tej funkcji staje się oczywista, gdy z niej skorzystasz. Czasami jedno spojrzenie mówi więcej niż długie wyjaśnienie.

## Integracja z Gemini CLI

Jako dziesiąty Agent CLI, Gemini CLI jest zintegrowany z HagiCode poprzez standardową architekturę dostawcy.

### Opcje konfiguracji

Gemini CLI obsługuje bogaty zestaw opcji konfiguracyjnych:

```csharp
public class GeminiOptions
{
    public string? ExecutablePath { get; set; }
    public string? WorkingDirectory { get; set; }
    public string? SessionId { get; set; }
    public string? Model { get; set; }
    public string? AuthenticationMethod { get; set; }
    public string? AuthenticationToken { get; set; }
    public Dictionary<string, string?> AuthenticationInfo { get; set; }
    public Dictionary<string, string?> EnvironmentVariables { get; set; }
    public string[] ExtraArguments { get; set; }
    public TimeSpan? StartupTimeout { get; set; }
    public CliPoolSettings? PoolSettings { get; set; }
}
```

Opcje te obejmują wszystko, od podstawowej konfiguracji po zaawansowane funkcje, dając użytkownikom elastyczność konfiguracji interfejsu CLI zgodnie z własnymi potrzebami. Przepływ pracy każdego jest inny, dlatego odrobina elastyczności jest zawsze mile widziana.

### Protokół komunikacyjny ACP

Gemini CLI obsługuje ACP (Agent Communication Protocol), który jest ujednoliconym standardem komunikacji CLI firmy HagiCode. Dzięki ACP różne interfejsy CLI mogą współdziałać z platformą w spójny sposób, co znacznie upraszcza prace integracyjne. Krótko mówiąc, standaryzuje skomplikowane części, dzięki czemu każdy może pracować łatwiej.

## Najlepsze praktyki dotyczące integracji wielu interfejsów CLI

`HagiCode.Libs.Providers` zapewnia ujednolicony mechanizm rejestracji i użytkowania:

```csharp
services.AddHagiCodeLibs();

var gemini = serviceProvider.GetRequiredService<ICliProvider<GeminiOptions>>();
var codebuddy = serviceProvider.GetRequiredService<ICliProvider<CodebuddyOptions>>();
var hermes = serviceProvider.GetRequiredService<ICliProvider<HermesOptions>>();
```

Ten projekt wstrzykiwania zależności sprawia, że użycie w różnych interfejsach CLI jest bardzo zwięzłe, a także sprawia, że testowanie jednostkowe i kpienie są wygodniejsze. Czysty kod to sposób na bycie odpowiedzialnym za siebie.

### Notatki

Podczas rzeczywistego użytkowania należy pamiętać o kilku kwestiach:

1. **Konfiguracja klucza API**: Upewnij się `ANTHROPIC_AUTH_TOKEN` is set correctly, or the model cannot be called
2. **Dostępność modelu**: GLM-5.1 wymaga włączenia przez odpowiedniego dostawcę modelu
3. **Funkcja obrazu**: Tylko modele z `supportsImage: true` można użyć przesyłania obrazu
4. **Instalacja CLI**: Przed użyciem Gemini CLI upewnij się, że `gemini` lub `gemini-cli` znajduje się w systemowej PATH

Mogą to być drobne szczegóły, jednak niewłaściwie zagospodarowane drobne szczegóły mogą przerodzić się w duże problemy, dlatego warto zwrócić na nie uwagę.
