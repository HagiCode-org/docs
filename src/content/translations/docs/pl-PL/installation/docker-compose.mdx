## Warunki wstępne

Przed rozpoczęciem upewnij się, że w systemie są zainstalowane Docker i Docker Compose.

### Zainstaluj Dockera

#### Okna

Pobierz i zainstaluj [Pulpit Dockera dla Windows](https://www.docker.com/products/docker-desktop/)

Po instalacji upewnij się, że Docker Desktop jest uruchomiony.

#### macOS

Pobierz i zainstaluj [Pulpit Docker dla komputerów Mac](https://www.docker.com/products/docker-desktop/)

Po instalacji upewnij się, że Docker Desktop jest uruchomiony.

#### Linuksa

Zainstaluj Dockera za pomocą menedżera pakietów:

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

Po instalacji uruchom usługę Docker:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Sprawdź instalację

Po instalacji uruchom następujące polecenia, aby sprawdzić, czy Docker i Docker Compose są poprawnie zainstalowane:

```bash
docker --version
docker compose version
```

## Szybki start

### 1. Wygeneruj plik konfiguracyjny Docker Compose

:::tip[Użyj generatora konfiguracji]
**Polecam używać [Docker Compose Builder (samodzielna witryna)](https://builder.hagicode.com/)** aby wygenerować plik konfiguracyjny.

Generator obsługuje:
- Wiele opcji dostawców API (Zhipu AI, Anthropic Officer, Custom API)
- Automatyczna konfiguracja adresu URL API i tokena
- Niestandardowe porty, nazwy kontenerów i inna podstawowa konfiguracja
- Skonfiguruj ścieżki danych trwałych i katalogów roboczych
- Automatycznie konfiguruj różnice między platformami Windows/Linux
- Inteligentnie rozwiązuj problemy z uprawnieniami do plików
- **Wybór źródła obrazu**: W zależności od środowiska sieciowego wybierz odpowiednie źródło obrazu

[🚀 Natychmiast użyj generatora →](https://builder.hagicode.com/)
:::

1. Otwórz [Docker Compose Builder (samodzielna witryna)](https://builder.hagicode.com/)
2. Wypełnij konfigurację zgodnie ze swoimi potrzebami:
   - Wybierz dostawcę API (Zhipu AI, Anthropic Officer lub Custom API)
   - Skonfiguruj porty, bazę danych, katalog roboczy i inne opcje
   - **Wybierz źródło obrazu**: Wybierz odpowiednie źródło obrazu w oparciu o środowisko sieciowe
3. Kliknij przycisk generowania, aby uzyskać `docker-compose.yml` konfiguracja
4. Zapisz wygenerowaną konfigurację jako plik `docker-compose.yml` plik
5. W razie potrzeby utwórz plik `.env` plik, aby skonfigurować poufne informacje

#### Wybór źródła obrazu

Generator obsługuje trzy opcje źródła obrazu:

**Hub Docker**
- **Adres obrazu**: `newbe36524/hagicode:{tag}`
- **Obowiązujące scenariusze**: Odpowiednie dla użytkowników korzystających z akceleracji lustrzanej Docker Hub
- **Zalety**: Oficjalne źródło obrazu, aktualne aktualizacje, stabilny dostęp
- **Uwagi**: W niektórych regionach może być konieczne skonfigurowanie przyspieszenia lustrzanego

**Rejestr kontenerów platformy Azure (ACR)**
- **Adres obrazu**: `hagicode.azurecr.io/hagicode:{tag}`
- **Obowiązujące scenariusze**: Odpowiednie dla użytkowników, którzy nie mogą uzyskać lokalnego dostępu do Docker Hub
- **Zalety**: Zapewnia alternatywne źródło obrazu, rozwiązuje problemy z dostępem do sieci
- **Uwagi**: Obrazy pozostają zsynchronizowane z Docker Hub, ale mogą występować krótkie opóźnienia

**Rejestr kontenerów w chmurze Alibaba (ACR)**
- **Adres obrazu**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **Obowiązujące scenariusze**: Odpowiednie dla użytkowników domowych, stabilny i szybki dostęp do sieci
- **Zalety**: węzły krajowe Alibaba Cloud, duża prędkość dostępu, wysoka stabilność
- **Uwagi**: Obrazy Alibaba ACR są synchronizowane z Docker Hub, zwykle w ciągu 30 minut

:::tip[Zalecenia dotyczące wyboru źródła obrazu]
- **Użytkownicy krajowi polecają Alibaba Cloud ACR**: Jeśli jesteś użytkownikiem domowym, zaleca się priorytetowe wykorzystanie źródła obrazu Alibaba Cloud ACR w celu zapewnienia szybkiego i stabilnego dostępu
- **Alternatywny Docker Hub**: Jeśli Twoje środowisko sieciowe ma dostęp do Docker Hub, możesz także wybrać Docker Hub jako źródło obrazu
- **Alternatywna usługa Azure ACR**: wybierz źródło obrazu usługi Azure ACR tylko wtedy, gdy inne źródła obrazów mają trudności z dostępem
- **Synchronizacja obrazu**: Obrazy Alibaba Cloud ACR pozostają zsynchronizowane z Docker Hub, zwykle w ciągu 30 minut
- ** Spójność wersji**: Tagi obrazów są spójne we wszystkich trzech źródłach obrazów, nie musisz się martwić problemami z wersją
:::

:::note[Uzyskaj token API]
Aby korzystać z Hagicode, musisz skonfigurować token API Claude:

**Zhipu AI (zalecane)**: [Zdobądź token API →](https://www.bigmodel.cn/glm-coding?ic=14BY54APZA)
- Stabilny dostęp krajowy, szybsza reakcja
- Wysoka wydajność kosztowa, odpowiednia do codziennego użytku

**Urzędnik antropiczny**: [Zdobądź token API →](https://console.anthropic.com/)
- Bezpośrednie korzystanie z usług Anthropic

:::

### 2. Uruchom usługi

Wykonaj następujące polecenie w katalogu, w którym znajduje się plik `docker-compose.yml` znajduje się:

```bash
docker compose up -d
```

To polecenie:
- Pobierz i uruchom kontenery Hagicode zdefiniowane w wygenerowanym szablonie
- Utwórz i skonfiguruj sieć
- Zainicjuj domyślny katalog danych i zależności środowiska wykonawczego

### 3. Sprawdź stan usługi

Sprawdź, czy kontenery działają:

```bash
docker compose ps
```

Powinieneś zobaczyć usługi zdefiniowane w wygenerowanym szablonie jako „uruchomione”.

### Wyświetl dzienniki usług

Aby wyświetlić wszystkie dzienniki usług:

```bash
# View all service logs
docker compose logs

# View Hagicode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## Instrukcje konfiguracji

Jeśli chcesz zmodyfikować konfigurację (np. zmienić porty, zmienić dostawcę API itp.), możesz:

1. Wygeneruj nową konfigurację za pomocą [Docker Compose Builder (samodzielna witryna)](https://builder.hagicode.com/)
2. Edytuj ręcznie `docker-compose.yml` i `.env` pliki
3. Uruchom ponownie usługi, aby konfiguracja zaczęła obowiązywać:

```bash
docker compose restart
```

## Dostęp do aplikacji

### Interfejs sieciowy

Po pomyślnym uruchomieniu usługi uzyskaj dostęp poprzez przeglądarkę:

```
http://localhost:45000
```

### Trwałość danych

Obsługiwana ścieżka Docker Compose domyślnie używa plików danych SQLite ze stałymi woluminami. Jeśli chcesz sprawdzić punkty podłączenia lub zaplanować kopie zapasowe, przejrzyj mapowania woluminów i katalogów danych w wygenerowanych plikach `docker-compose.yml`.
