Ten przewodnik opisuje instalację i użytkowanie Hagicode Desktop. Jest to oficjalny klient stacjonarny, który przechowuje domyślną ścieżkę konfiguracji lokalnej oraz codzienną kontrolę w jednym miejscu.

:::tip[Zalecana metoda]
Pulpit jest domyślną ścieżką lokalną dla indywidualnych użytkowników i programistów, ponieważ:
- instalator utrzymuje domyślny przepływ w jednej ścieżce
- zależnościami i usługami zarządza się w aplikacji
- interfejs użytkownika ułatwia kontrolę stanu i operacji
- wbudowane zarządzanie wersjami ułatwia przełączanie
- dane domyślnie pozostają lokalne
:::

## Co to jest Hagicode Desktop?

Hagicode Desktop to oficjalna aplikacja komputerowa Hagicode przeznaczona dla systemów Windows, macOS i Linux. Łączy w sobie podstawowe funkcje Hagicode, a instalator zachowuje domyślną ścieżkę konfiguracji w jednym procesie.

### Główne zalety

- **Instalacja jednym kliknięciem**: Wykonaj wszystkie kroki konfiguracji automatycznie za pomocą kreatora instalacji
- **Automatyczne przygotowanie środowiska wykonawczego**: Domyślny instalator zawiera już potrzebne środowisko wykonawcze, więc nie ma potrzeby instalowania osobnego `.NET` najpierw czas wykonania
- **Zarządzanie usługami**: Wbudowane zarządzanie serwerem Hagicode, łatwe uruchamianie i zatrzymywanie usług
- **Przełączanie wersji**: Obsługa zarządzania wieloma wersjami, swobodne przełączanie między różnymi wersjami
- **Działanie lokalne**: Wszystkie dane są przechowywane lokalnie, co chroni Twoją prywatność
- **Aktualizacje automatyczne**: automatycznie wykrywa i pobiera najnowszą wersję

### Obowiązujące scenariusze

Komputer stacjonarny jest dobrym rozwiązaniem, gdy:
- Indywidualni programiści, którzy chcą używać Hagicode lokalnie
- Użytkownicy, którzy muszą używać Hagicode w trybie offline
- Użytkownicy nietechniczni, którzy chcą uprościć proces instalacji
- Programiści, którzy muszą często przełączać się między różnymi wersjami

### Inne metody pobierania

Instalator Hagicode Desktop możesz także pobrać z następujących kanałów:

1. **Strona oficjalnej wersji**: Odwiedź [Wydania Hagicode GitHub](https://github.com/Hagicode-org/releases) aby pobrać najnowszą wersję
2. **Oficjalna strona internetowa**: Odwiedź [Witryna Hagicode](https://hagicode.com/) aby uzyskać linki do pobrania

### Wybierz poprawną wersję

Wybierz odpowiedni instalator dla swojego systemu operacyjnego:
- **Windows**: Pobierz plik `.exe` instalator
- **macOS**: Pobierz plik `.dmg` instalator
- **Linux**: Pobierz plik `.AppImage` lub `.tar.gz` pakiet instalacyjny

:::note[Wymagania systemowe]
- **Windows**: Windows 10 lub nowsza wersja (64-bitowa)
- **macOS**: macOS 10.15 lub nowszy
- **Linux**: główne dystrybucje Linuksa (Ubuntu 20.04+, Fedora 33+ itp.)
:::

:::tip[Domyślne wymagania wstępne]
W przypadku najnowszej lokalnej instalacji HagiCode zwykle nie trzeba przygotowywać osobnej instalacji `.NET` czas wykonania. Program Desktop już pakuje środowisko wykonawcze wymagane dla domyślnego przepływu i domyślnie inicjuje obsługiwaną lokalną ścieżkę SQLite.
:::

## Uwagi dotyczące instalacji i uruchamiania

:::caution[Powiadomienie o strażniku systemu macOS]
Jeśli już zainstalowałeś aplikację w domyślnej ścieżce `/Applications/Hagicode.app` ale macOS wyświetla komunikat „aplikacja jest uszkodzona”, „nie można otworzyć” lub podobne ostrzeżenie Gatekeepera przy pierwszym uruchomieniu, typową przyczyną jest `com.apple.quarantine` atrybut, który macOS dodaje do aplikacji pobranych z Internetu, a nie błąd samego instalatora Hagicode Desktop.

Uruchom następujące polecenie w terminalu. Wymaga uprawnień administratora i rekurencyjnie usuwa atrybut kwarantanny z aplikacji w domyślnej ścieżce instalacji:

```bash
sudo xattr -dr com.apple.quarantine /Applications/Hagicode.app
```

Po zakończeniu polecenia uruchom ponownie Hagicode Desktop.
:::

## Uruchom pulpit

Po pobraniu otwórz instalator lub pakiet aplikacji i postępuj zgodnie z instrukcjami wyświetlanymi na ekranie. Desktop przygotowuje środowisko uruchomieniowe w aktualnie obsługiwanej ścieżce lokalnej, zarządza serwerem Hagicode oraz wyświetla postęp i status, gdy wymagane jest pobieranie plików sieciowych.

:::note[Wymagania sieciowe]
Upewnij się, że połączenie sieciowe jest dostępne. Desktop może wymagać pobrania informacji o wersji lub pakietów komponentów z Internetu; jeśli pobieranie nie powiedzie się, sprawdź ustawienia sieci, serwera proxy lub zapory sieciowej i spróbuj ponownie.
:::

## Zarządzanie wersjami

Hagicode Desktop obsługuje zarządzanie wieloma wersjami, umożliwiając swobodne przełączanie pomiędzy różnymi wersjami.

![Zarządzanie wersjami](../../img/screenshots/installation/install-desktop/12-version-management/original.png)

### Zmień wersję

1. Otwórz pulpit Hagicode
2. Przejdź do strony „Zarządzanie wersjami”.
3. Wybierz wersję, którą chcesz przełączyć
4. Kliknij przycisk „Przełącz wersję”.
5. Poczekaj na zakończenie przełączania

:::note[Uwagi dotyczące zmiany wersji]
- Zmiana wersji spowoduje automatyczne ponowne uruchomienie serwera Hagicode
- Przed zmianą upewnij się, że zapisałeś całą pracę
- Konfiguracje projektu mogą się różnić w zależności od wersji
:::

## Uruchom i zatrzymaj usługę

Hagicode Desktop zapewnia prosty sposób zarządzania usługą Hagicode Server.

### Uruchom usługę

Na stronie głównej aplikacji Desktop kliknij przycisk „Uruchom usługę”, aby uruchomić serwer Hagicode.

![Uruchom usługę](../../img/screenshots/installation/install-desktop/13-start-service-homepage/original.png)

### Zatrzymaj usługę

Aby zatrzymać serwer Hagicode, kliknij przycisk „Zatrzymaj usługę” na stronie głównej.

:::tip[Stan usługi]
Aplikacja Desktop wyświetla w czasie rzeczywistym status działania serwera Hagicode, w tym:
- Stan usługi (działa/zatrzymana)
- Numer portu serwisowego
- Informacje o aktualnej wersji
:::

## Rozwiązywanie problemów

### Błąd instalacji

Jeśli podczas instalacji wystąpią problemy:

1. **Sprawdź połączenie sieciowe**: Upewnij się, że masz dostęp do Internetu
2. **Sprawdź miejsce na dysku**: Upewnij się, że masz wystarczającą ilość miejsca na dysku
3. **Uruchom jako administrator**: W systemie Windows kliknij instalator prawym przyciskiem myszy i wybierz „Uruchom jako administrator”
4. **Wyświetl pliki dziennika**: Instalator generuje szczegółowe pliki dziennika, które mogą pomóc w diagnozowaniu problemów

### Niepowodzenie instalacji zależności

Jeśli instalacja zależności nie powiedzie się:

1. **Sprawdź zgodność systemu**: Upewnij się, że Twój system operacyjny spełnia minimalne wymagania
2. **Sprawdź ustawienia sieci i zapory sieciowej**: Upewnij się, że instalator może pobrać wymagane pliki i nie jest blokowany przez zaporę sieciową
3. **Potwierdź uprawnienia instalatora**: W systemie Windows spróbuj uruchomić instalator jako administrator, aby miał wystarczające uprawnienia
4. **Uruchom ponownie aplikację i spróbuj ponownie**: Zamknij i otwórz ponownie Hagicode Desktop, a następnie spróbuj ponownie wykonać krok instalacji
5. **Wyświetl dzienniki**: przeglądaj szczegółowe dzienniki błędów w aplikacji komputerowej

### Błąd uruchomienia usługi

Jeśli serwer Hagicode nie uruchomi się:

1. **Sprawdź zajętość portu**: Upewnij się, że port 45000 nie jest zajęty przez inne programy
2. **Sprawdź ustawienia zapory sieciowej**: Upewnij się, że zapora umożliwia dostęp do sieci serwera Hagicode
3. **Uruchom ponownie aplikację**: Spróbuj ponownie uruchomić Hagicode Desktop
4. **Wyświetl dzienniki**: Wyświetl szczegółowe dzienniki błędów w aplikacji komputerowej

### Potrzebujesz dodatkowej pomocy?

Jeśli napotkasz problemy nie omówione tutaj:

1. Sprawdź [Problemy z GitHubem](https://github.com/HagiCode-org/site/issues) dla podobnych problemów
2. Odwiedź nasze [Forum społeczności](https://github.com/HagiCode-org/site/discussions) o pomoc
3. Prześlij nowy problem, opisując szczegółowo swój problem
