HagiCode obsługuje wiele ścieżek instalacji. Zacznij od metody wdrażania, która pasuje do Twojego scenariusza, a następnie dodawaj oprogramowanie pomocnicze tylko wtedy, gdy jest potrzebne.

Obecny aktywny zakres obsługi interfejsu CLI agenta najbardziej odpowiedni dla konfiguracji to: Claude Code, Codex, GitHub Copilot, OpenCode, Hermes, QoderCLI, Kiro, Kimi, Gemini, DeepAgents i Codebuddy. Hermes, Gemini i DeepAgents korzystają obecnie z ręcznych procesów instalacji lub uwierzytelniania; pozostałe interfejsy CLI udostępniają punkty wejścia do instalacji jednym kliknięciem lub przebieg instalacji z przewodnikiem.

## Zalecane kolejne kroki

- Jeśli planujesz kupić i zainstalować HagiCode przez Steam, zacznij od [Instalacja parowa](/pl-PL/installation/steam).
- Jeśli jesteś nowy w HagiCode, zacznij od [Instalacja na pulpicie](/pl-PL/installation/desktop).
- Jeśli wdrażasz w środowisku serwerowym lub zespołowym, użyj [Instalacja Docker Compose](/pl-PL/installation/docker-compose).
- Jeśli chcesz potwierdzić stan wsparcia, tryb konfiguracji lub historyczne uwagi dotyczące zgodności dla interfejsu CLI, otwórz [Instalacja interfejsu CLI agenta AI](/pl-PL/related-software-installation/ai-agent-cli/).
