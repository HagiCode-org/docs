Тази страница е за потребители на HagiCode за първи път, които искат най-краткия път за настройка на OmniRoute: стартирайте локалната услуга, проверете дали `http://localhost:20128/v1` работи и потвърдете, че таблото за управление и API са налични.

:::tip[Препоръчан път]
Тази първа версия запазва първо най-стабилния път: `npm install -g omniroute`, `omniroute`, Табло при `http://localhost:20128`и OpenAI-съвместимия основен URL адрес на `http://localhost:20128/v1`. Docker, VPS и по-разширените режими на внедряване остават в раздела по избор.
:::

## Предпоставки

- Node.js вече е инсталиран. Завършено [Инсталиране на Node.js](/en/related-software-installation/nodejs/installation) първо, ако е необходимо.
- Местно пристанище `20128` е свободен и все още не е зает.
- Имате поне един акаунт на доставчик на модел нагоре по веригата, който планирате да маршрутизирате през OmniRoute.
- Преди да интегрирате HagiCode, CLI или SDK, пазете тези два адреса отделно:
  - Табло за управление: `http://localhost:20128`
  - Основен URL адрес, съвместим с OpenAI: `http://localhost:20128/v1`

## Инсталирайте и стартирайте локално

Започнете с препоръчания глобален npm път:

```bash
npm install -g omniroute
omniroute
```

След стартиране на услугата я потвърдете в следния ред:

1. Отворете `http://localhost:20128` в браузър и потвърдете, че таблото за управление се зарежда.
2. Отидете на `Endpoints` в таблото за управление и създайте или копирайте API ключ.
3. Обърнете внимание на основния URL адрес, който ще предадете на инструментите по-късно: `http://localhost:20128/v1`.

:::бележка[Защо да започнете с таблото за управление]
За първата настройка таблото за управление е най-бързият начин да потвърдите, че OmniRoute работи, вашите доставчици са свързани и съществува API ключ. Ако по-късно извикване на клиент е неуспешно, първо проверяването на таблото за управление обикновено е по-бързо от незабавното отстраняване на грешки в SDK кода.
:::

## Валидирайте работещата услуга

### Проверка 1: потвърдете, че каталогът на моделите е четим

Поставете ключа от `Endpoints` страница в променлива на средата:

```bash
export OMNIROUTE_API_KEY="replace-with-the-key-you-copied-from-the-Endpoints-page"
```

След това поискайте `/v1/models`:

```bash
curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}"
```

Ако получите обратно списък с модели, съвместимата с OpenAI крайна точка и потокът за удостоверяване работят.

### Проверка 2: изпратете минимална заявка за чат

```bash
curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "replace-with-a-model-id-from-v1-models",
    "messages": [
      { "role": "user", "content": "Reply with ok." }
    ]
  }'
```

Целта тук не е бързото качество. Това е да докаже, че избраният от вас модел вече може да бъде маршрутизиран успешно през OmniRoute.
