Това ръководство обхваща инсталирането на Node.js в Windows, macOS и Linux. Node.js е ключова зависимост за локалните инструменти на HagiCode, включително OpenSpec, инструменти за интерфейс и няколко CLI за разработчици.

:::tip[Изисквания за версията]
HagiCode препоръчва **Node.js 24+** (минимално поддържан Node.js >= 18.0). В повечето случаи инсталирайте най-новата LTS версия и след това проверете дали вашата версия отговаря на изискването.
:::

## Инсталиране на Node.js

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Първи метод: Официален инсталатор (препоръчително)

1. Посетете [Официален уебсайт на Node.js](https://nodejs.org/)
2. Изтеглете **LTS (дългосрочна поддръжка)** версията на инсталатора на Windows (`.msi` файл)
3. Щракнете двукратно, за да стартирате инсталатора
4. Следвайте подканите на съветника за инсталиране, за да завършите инсталацията:
   - Приемете лицензионното споразумение
   - Изберете пътя за инсталиране (по подразбиране е добре)
   - Уверете се, че е отметнато „Автоматично инсталиране на необходимите инструменти“.
   - Щракнете върху „Инсталиране“, за да започнете инсталацията
5. След като инсталацията приключи, щракнете върху „Край“

### Метод втори: Използване на крило

Отворете PowerShell или командния ред и изпълнете:

```powershell
winget install OpenJS.NodeJS.LTS
```

След инсталирането отворете отново терминала, за да влязат в сила промените.

### Първи метод: Официален инсталатор (препоръчително)

1. Посетете [Официален уебсайт на Node.js](https://nodejs.org/)
2. Изтеглете инсталатора на **LTS (дългосрочна поддръжка)** версия за macOS (`.pkg` файл)
3. Щракнете двукратно, за да стартирате инсталатора
4. Следвайте подканите на съветника за инсталиране, за да завършите инсталацията:
   - Щракнете върху „Продължи“
   - Приемете лицензионното споразумение
   - Изберете инсталационен диск
   - Щракнете върху „Инсталиране“ и въведете администраторска парола
5. След като инсталацията приключи, щракнете върху „Затвори“

### Метод втори: Използване на Homebrew

Ако имате [Homebrew](https://brew.sh/) инсталиран, можете да инсталирате Node.js чрез терминал:

```bash
brew install node
```

::: съвет
Homebrew автоматично ще добави Node.js към вашия PATH. Ако командата не е налична, рестартирайте терминала или изпълнете:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

### Използване на NodeSource Repository (препоръчително)

Това е най-добрият начин да получите най-новата версия на Node.js:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Използване на apt Package Manager

Ако не се нуждаете от най-новата версия, можете да използвате хранилището по подразбиране на Ubuntu:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::бележка
Версията Node.js в хранилището по подразбиране може да е по-стара. След инсталирането проверете версията, за да се уверите, че отговаря на изискванията на Node.js >= 18.0 (препоръчва се Node.js 24+).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### Използване на yum Package Manager

```bash
sudo yum install -y nodejs npm
```

## Използване на инструменти за управление на версии

Ако трябва да превключвате между няколко версии на Node.js, можете да използвате инструменти за управление на версиите.

### nvm (Мениджър на версии на възли)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

След инсталирането презаредете конфигурацията на обвивката си:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

В Windows препоръчваме да използвате [nvm-прозорци](https://github.com/coreybutler/nvm-windows):

1. Изтеглете най-новото `nvm-setup.exe` от [версии на nvm-windows](https://github.com/coreybutler/nvm-windows/releases)
2. Стартирайте инсталатора
3. Отворете отново командния ред или PowerShell

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

След като инсталирате nvm, можете да го използвате за инсталиране и управление на версии на Node.js:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (Бърз мениджър на възли)

fnm е по-бърз и опростен инструмент за управление на версии на Node.js, създаден с Rust.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

Използвайте [лъжичка](https://scoop.sh/) или [крило](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Ускорение за потребители от континентален Китай

Ако сте в континентален Китай, изтеглянето на Node.js и npm пакети може да е бавно. Ето някои решения за ускоряване.

### Използване на вътрешни огледални източници

#### npm Taobao Mirror (препоръчително)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### Използване на nrm за управление на огледални източници

nrm (мениджър на регистъра на npm) ви помага бързо да превключвате npm огледални източници:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Ускоряване на изтеглянията на инсталатора на Node.js

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao предоставя огледални изтегляния за инсталатори на Node.js:

- **Адрес за изтегляне**: https://npmmirror.com/mirrors/node/

Посетете горния адрес и изберете версията, която трябва да изтеглите.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

Ако изтеглянето на nvm или други инструменти от GitHub е бавно, можете да използвате огледалото FastGit:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::бележка
Услугата FastGit може да се промени, моля, вижте официалната информация.
:::

  </TabItem>
</Tabs>

### Домашни домашни огледала

Ако използвате Homebrew в континентален Китай, можете да конфигурирате домашни огледала за ускорение:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Мениджъри на пакети, специфични за континентален Китай

#### cnpm

cnpm е npm клиент, предоставен от екипа на Taobao, който използва Taobao огледалото по подразбиране:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

::: съвет
cnpm е най-вече съвместим с npm команди и може да се използва като директен заместител.
:::

### Проверка на огледалния ефект на ускорение

След като настроите огледалата, проверете чрез:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Проверка на инсталацията

След инсталирането проверете дали Node.js и npm са инсталирани правилно:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::бележка
Уверете се, че Node.js версия >= 18.0 (препоръчва се Node.js 24+). Ако версията е твърде ниска, моля, преинсталирайте чрез горните методи или използвайте инструменти за управление на версията, за да инсталирате правилната версия.
:::

### Примерен резултат за проверка

Резултатът за успешна проверка трябва да изглежда така:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Отстраняване на неизправности

### Проблеми с Windows

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Ако `node` или `npm` командите не са налични след инсталиране:

1. Рестартирайте командния ред или PowerShell
2. Ако все още не е наличен, ръчно добавете Node.js към PATH:
   - Потърсете „Променливи на средата“ и отворете „Редактиране на системни променливи на средата“
   - Кликнете върху „Променливи на средата“
   - Намерете „Път“ в „Системни променливи“ и щракнете върху „Редактиране“
   - Добавяне на път за инсталиране на Node.js (по подразбиране: `C:\Program Files\nodejs\`)
   - Щракнете върху „OK“, за да запазите промените

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Ако срещнете грешки в правилата за изпълнение, когато използвате nvm:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### Проблеми с macOS

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Ако срещнете грешки в разрешенията, когато инсталирате глобални npm пакети:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Ако командите не са налични след инсталирането, рестартирайте терминала или ръчно добавете към PATH (обикновено в `/usr/local/bin`).

  </TabItem>
</Tabs>

### Проблеми с Linux

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Ако срещнете грешки в разрешенията, когато инсталирате глобални npm пакети, се препоръчва да използвате инструменти за управление на версии на Node.js (като nvm или fnm) вместо системни мениджъри на пакети.

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Ако версията на Node.js, инсталирана чрез системния мениджър на пакети, е твърде стара:

1. Използвайте хранилище на NodeSource (препоръчително, вижте по-горе)
2. Или използвайте инструменти за управление на версии (nvm/fnm)

  </TabItem>
</Tabs>

### Проверка на околната среда

Ако проверката е неуспешна, проверете следното:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
