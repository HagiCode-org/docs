# Инсталирайте OpenSpec

OpenSpec управлява предложения, спецификации и записи за промени. Тази страница има една цел: обяснете ясно пътя на инсталиране, за да можете да го прикачите към съществуващия си работен процес.

## Какво е OpenSpec?

OpenSpec е инструмент от команден ред за управление на предложения и спецификации в софтуерната работа. Основният работен процес включва:

- **Управление на предложенията**: Създавайте, преглеждайте и проследявайте предложения за функции
- **Проверка на спецификацията**: Уверете се, че промените в кода отговарят на предварително зададени спецификации
- **Проследяване на промените**: Запишете цялата прегледана история на промените
- **Екипно сътрудничество**: Подкрепете разработката за сътрудничество с много хора

### Приложение в проекти на HagiCode

В проектите на HagiCode OpenSpec се използва за:

1. **Управление на разработката на функции**: Всички нови функции трябва да преминат през процеса на предлагане на OpenSpec
2. **Преглед на кода**: Уверете се, че промените отговарят на спецификациите на проекта
3. **Синхронизиране на документация**: Поддържайте съгласуваност между документацията и изпълнението на кода
4. **Интегриране на контрол на версиите**: Безпроблемна интеграция с работни потоци за контрол на версиите

## Предпоставки

Преди да инсталирате OpenSpec, уверете се, че вашата система отговаря на следните изисквания:

### Node.js и npm

OpenSpec се разпространява чрез npm и изисква Node.js среда. OpenSpec изисква **Node.js 18.0 или по-нова версия**.

Проверете инсталацията на Node.js:

```bash
node --version
```

Проверете инсталацията на npm:

```bash
npm --version
```

Ако не е инсталиран или версията е твърде стара, вижте съответните инструкции за инсталиране на софтуер в [Ръководство за инсталиране](/en/installation).

## Инсталиране на OpenSpec

:::tip[Изисквания за версията]
Този проект изисква използването на **OpenSpec 1.0.0 или по-висока** гама от версии. Всички версии 1.x се поддържат, за да се осигури съвместимост с работния процес на проекта.
:::

OpenSpec се инсталира глобално чрез npm, което го прави достъпен от всяка точка на вашата система.

### Инсталирайте глобално с помощта на npm

<Tabs>
  <TabItem value="win" label="Windows">
    Стартирайте в PowerShell или командния ред:

```powershell
npm install -g @fission-ai/openspec@1
```

:::бележка[Проблеми с разрешенията на Windows]
Ако по време на инсталацията срещнете грешки в разрешенията, стартирайте PowerShell или командния ред като администратор.
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    Стартирайте в терминала:

```bash
npm install -g @fission-ai/openspec@1
```

:::бележка[Проблеми с разрешения за macOS]
Ако срещнете `EACCES` грешки в разрешенията, опитайте:
```bash
sudo npm install -g @fission-ai/openspec@1
```
Или използвайте [nvm](https://github.com/nvm-sh/nvm) за управление на версиите на Node.js и избягване на проблеми с разрешенията.
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    Стартирайте в терминала:

```bash
npm install -g @fission-ai/openspec@1
```

:::бележка[Проблеми с разрешения за Linux]
Ако срещнете `EACCES` грешки в разрешенията, можете да:
1. Използвайте `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. Конфигурирайте npm да използва потребителска директория:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### Проверете инсталацията

След инсталирането проверете дали OpenSpec е инсталиран правилно и версията отговаря на изискванията:

```bash
openspec --version
```

Ако се инсталира успешно, ще видите изходен номер на версията, например:

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::бележка[диапазон от версии]
Моля, уверете се, че инсталираната версия на OpenSpec е 1.0.0 или по-нова. Проектът има проверена съвместимост с версия 1.1.1. Препоръчително е да използвате най-новата съвместима версия, за да получите нови функции.
:::

## Често срещани проблеми и отстраняване на неизправности

### Грешки в разрешенията

#### Проблем: `EACCES` Разрешението е отказано

Срещане на грешки при разрешаване при инсталиране или изпълнение на OpenSpec.

**Решения**:

<Tabs>
  <TabItem value="win" label="Windows">
    Стартирайте PowerShell или командния ред като администратор:
    1. Щракнете с десния бутон върху PowerShell или командния ред
    2. Изберете „Изпълни като администратор“
    3. Изпълнете отново инсталационната команда
  </TabItem>
  <TabItem value="mac" label="macOS">
    Инсталирайте с помощта на `sudo`:

```bash
sudo npm install -g @fission-ai/openspec@1
```

Или използвайте [nvm](https://github.com/nvm-sh/nvm) за управление на Node.js и избягване на проблеми с глобалните разрешения за инсталиране.
  </TabItem>
  <TabItem value="linux" label="Linux">
    Решение 1: Използвайте sudo
```bash
sudo npm install -g @fission-ai/openspec@1
```

Решение 2: Конфигурирайте npm да използва потребителска директория
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @fission-ai/openspec@1
```
  </TabItem>
</Tabs>

### Проблеми с мрежата

#### Проблем: Инсталирането на npm е неуспешно или е много бавно

Не може да се изтегли OpenSpec от регистъра на npm поради мрежови причини.

**Решения**:

1. **Използване на домашно npm огледало**:

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **Възстановете официалния източник** (ако е необходимо):

```bash
npm config set registry https://registry.npmjs.org
```

3. **Използвайте cnpm** (по избор):

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

::: съвет
На потребителите в континентален Китай се препоръчва да използват огледалото Taobao, което може значително да подобри скоростта на изтегляне.
:::

### Проблеми със съвместимостта на версията

#### Проблем: Командите на OpenSpec се изпълняват необичайно или съобщават за грешки

Инсталираната версия на OpenSpec е несъвместима с текущата среда.

**Решения**:

1. **Проверете версията на Node.js**:

```bash
node --version
```

Уверете се, че версията е 18.0 или по-нова. Ако версията е твърде ниска, моля, надстройте Node.js.

2. **Проверете версията на OpenSpec**:

```bash
openspec --version
```

3. **Преинсталирайте конкретна версия**:

```bash
npm install -g @fission-ai/openspec@1
```

4. **Изчистване на npm кеша** (ако проблемът продължава):

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### Командата не е намерена

#### Проблем: Въвеждане `openspec` Командата показва „Командата не е намерена“

OpenSpec е инсталиран, но системата не може да намери командата.

**Решения**:

<Tabs>
  <TabItem value="win" label="Windows">
    1. Потвърдете пътя за глобална инсталация на npm:

```powershell
npm config get prefix
```

2. Добавете път към системата PATH:
   - Отворете „Свойства на системата“ → „Разширени“ → „Променливи на средата“
   - Намерете `Path` в "Системни променливи"
   - Добавете npm глобален път (обикновено `C:\Users\<username>\AppData\Roaming\npm`)
   - Рестартирайте прозореца на командния ред
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. Потвърдете пътя за глобална инсталация на npm:

```bash
npm config get prefix
```

2. Добавете път към PATH (in `~/.zshrc` или `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Презареди конфигурация:

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. Потвърдете пътя за глобална инсталация на npm:

```bash
npm config get prefix
```

2. Добавете път към PATH (in `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Презареди конфигурация:

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
