## Проектиране на 12 изключителни стартиращи изживявания за HagiCode: от минимализъм до киберпънк

> Кратката пауза между изтеглянето на приложение React 19 и завършването на Hydration е златна възможност за потребителите да почувстват индивидуалността на вашата марка. В тази статия споделяме пълна система за стил на стартиране, която изградихме за проекта HagiCode, използвайки HTML/CSS/JS.

## Фон

Като модерно приложение, създадено с ASP.NET Core 10 и React 19 (Vite), HagiCode използва архитектура за разгръщане, разделена от интерфейса и бекенда. Изходът за компилация на интерфейса е пакетиран в бекенда `wwwroot/` директория и се хоства от ASP.NET Core.

Въпреки това, тази архитектура въвежда класическа UX болезнена точка: когато потребителите посещават страницата, браузърът първо зарежда HTML, след това изтегля големия JS пакет и накрая позволява на React да извърши Hydration. По време на този "вакуумен период", който продължава от няколкостотин милисекунди до няколко секунди, потребителите виждат или празен екран, или безжизнена статична страница.

За да запълним тази празнина и да инжектираме индивидуалността на марката на HagiCode, трябваше да проектираме система със стил на стартиране, внедрена изцяло с вграден код вътре `index.html`.

## Относно HagiCode

Подходът за дизайн на началния екран, споделен в тази статия, идва от нашия практически опит в [HagiCode](https://github.com/HagiCode-org/site) проект. Като помощник за AI кодиране, HagiCode се грижи не само за ефективността на генериране на код, но и за визуалното изживяване на разработчика. Тази стартираща система е един от резултатите от стремежа ни към максимална производителност на интерфейса.

## Основни предизвикателства и архитектурен дизайн

Преди да започнем да проектираме, първо трябваше да изясним техническите ограничения. Тъй като всичко трябваше да бъде внедрено inline in `index.html`, не можахме да заредим външни CSS или JS файлове, различни от собствения пакет на React.

### Анализ на техническите ограничения

1.  **Принцип на нулева зависимост**: Всички стилове трябва да живеят в a `<style>` и цялата логика трябва да живее вътре в a `<script>` етикет.
2.  **Отбранителен CSS**: За да предотвратим замърсяването на началния екран от глобални стилове след монтирането на приложението React, решихме да обвием всички стилове при стартиране с префикс на ID с висок приоритет, като например `#boot-screen`.
3.  **Ефективността на първо място**: Анимациите трябва да използват CSS `transform` и `opacity` навсякъде, където е възможно, за да се избегне преформатиране и да се гарантира, че основната нишка остава неблокирана.
4.  **Визуална последователност**: Цветовете и шрифтовете трябва да останат в съответствие с конфигурацията на Tailwind на HagiCode.

### Архитектурен модел: корпус и инжектор

Приехме **вариантен модел**. Основната логика е капсулирана в незабавно извикан функционален израз (IIFE), докато специфичната логика на изобразяване се инжектира чрез конфигурация. Това ни позволява да превключваме между различни стилове чрез проста конфигурация, вместо да пренаписваме многократно логиката за манипулиране на DOM.

Ето кода на основната архитектура:

```html
<!-- Inline in index.html -->
<div id="boot-root"></div>

<script>
(function() {
  const BootSequence = {
    config: {
      theme: 'terminal', // Configurable as 'minimal', 'skeleton', 'code-rain', etc.
      color: '#3b82f6'   // Brand color
    },
    
    // Core lifecycle
    init() {
      this.render();
      this.listenForMount();
    },

    // Render the currently selected style
    render() {
      const root = document.getElementById('boot-root');
      if (this.variants[this.config.theme]) {
        root.innerHTML = this.variants[this.config.theme].render();
      }
    },

    // Listen for successful React mount and exit gracefully
    listenForMount() {
      window.addEventListener('hagicode:ready', () => {
        const screen = document.getElementById('boot-root');
        // Fade out first, then remove the DOM to avoid flicker
        screen.style.opacity = '0';
        screen.style.transition = 'opacity 0.3s ease';
        setTimeout(() => screen.remove(), 300);
      });
    },

    // The implementation logic for all 12 styles lives here
    variants: {
      // ...see details below
    }
  };

  BootSequence.init();
})();
</script>
```

## Контролен списък с 12 дизайна в стил на стартиране

Групирахме тези 12 стила в шест основни категории, за да задоволим различни сценарии и естетически предпочитания.

### А. Минимализъм

> „По-малкото е повече“. За сценарии, които преследват максимална скорост на зареждане, ние предоставяме възможно най-леките опции.

#### 1. Минималистична точка
Проста точка седи в центъра на екрана, съчетана с дишаща анимация.
*   **Внедряване**: CSS `@keyframes` контролира мащаба и непрозрачността.
*   **Най-добро за**: Всеки случай, когато страницата трябва да остане абсолютно чиста.

#### 2. Разкриване на марката
Използване на SVG `stroke-dasharray` анимация, той симулира ръчно нарисувано разкриване на линиите на логото на HagiCode, последвано от избледняване на текст.
*   **Техника**: SVG анимация на пътя със силно изпипано усещане.

### Б. Мимикрия на скелетен екран

> „Изкуството да мамиш окото“. Чрез симулиране на реално оформление на потребителския интерфейс потребителите се чувстват сякаш страницата вече е наполовина заредена.

#### 3. Скелет за чат в страничната лента
Това може да е най-практичният вариант. Ръчно изградихме оформление в HTML, което отразява React `Sidebar` и `ChatInput` компоненти точно, след което го насложи със сива блестяща анимация.
*   **Стойност**: Когато хидратацията на React завърши, скелетът незабавно се превръща в истинския компонент и потребителите едва могат да усетят превключването.

#### 4. Скелет на купчина карти
Той симулира движението на подредени карти с предложения по време на зареждане, като използва 3D трансформации, за да накара картите да плават фино.

### В. Абстракция и чл

> Похвалете се с маниакското ДНК на HagiCode.

#### 5. Геометрична трансформация
Геометрична форма (квадрат) се изобразява в центъра на екрана, след което плавно се трансформира с течение на времето в кръг, триъгълник и накрая логото.
*   **Технология**: Плавни преходи с CSS `border-radius`.

#### 6. Код Дъжд
Почит към *Матрицата*. Използвайки шрифта JetBrains Mono, бледи потоци от знаци падат на заден план.
*   **Забележка**: За производителност потоците от символи трябва да останат в по-малка област или да използват по-ниска честота на опресняване.

#### 7. Неонов импулс
Светещ пръстен в стил киберпънк, който използва множество `box-shadow` слоеве за създаване на мощен неонов блясък.

### D. Брандиране и теми

> Накарайте системата да се почувства жива.

#### 8. Сезонна тема
Това е динамичен товарач. Той проверява текущата дата за празници като Лунна Нова година или Коледа и зарежда съответната SVG анимация.
*   **Пример**: По време на лунната Нова година червените фенери леко се поклащат в долната част на екрана.

#### 9. Градиентно течение
Фонът използва течен градиент, базиран на цветовете на марката HagiCode. В комбинация с анимирани `background-size` и `background-position`, създава усещане за движение, подобно на сияние.

## Най-добри практики и клопки

В реалната развойна работа на HagiCode ние обобщихме няколко критично важни подробности за внедряването.

### 1. Защитният CSS е задължителен
Никога не ставайте мързеливи и пропускайте префикса. Веднъж забравихме да обхванем стиловете на началния екран с ID и глобално `div` стиловете след монтирането на React неочаквано повлияха на началния екран, нарушавайки оформлението.
**Научен урок**: Поставете всеки CSS селектор под `#boot-screen`, и използвайте `!important` за повишаване на приоритета, когато е необходимо, но само в CSS началния екран.

### 2. Грациозни преходи
След като React се монтира успешно, не го правете директно `remove()` началния екран DOM.
**Правилен подход**:
1.  React задейства `window.dispatchEvent(new Event('hagicode:ready'))`.
2.  Началният екран слуша за събитието и първите настройки `opacity: 0`.
3.  Изчакайте 300 ms, което съответства на продължителността на CSS прехода, и се обадете `.remove()` само след като екранът е напълно невидим.

### 3. Синхронизиране на променливи на тема
Стойностите на цвета на началния екран са твърдо кодирани `index.html`. Ако променим основния цвят на Tailwind, трябва да актуализираме и началния екран.
**Оптимизация**: Напишете прост плъгин в скрипта за изграждане на Vite за четене `tailwind.config.js` и инжектирайте цветни променливи в `index.html` шаблонни променливи, създавайки един източник на истина.

### 4. Предварително зареждане на шрифта
Началните екрани често трябва да използват марков шрифт, но ако шрифтът се зарежда бавно, може да се появи FOUT (мигане на нестилизиран текст).
**Решение**: Добавяне `<link rel="preload" href="/fonts/JetBrainsMono.woff2" as="font" type="font/woff2" crossorigin>` вътре `<head>`. Това е евтин начин с висока възвръщаемост за подобряване на изживяването.

### 5. Наблюдение на производителността
Инжектирахме `performance.mark('boot-start')` в долната част на `index.html`, и маркиран `boot-end` когато React се монтира успешно.
**Защо има значение**: Събирайки тези данни чрез Application Insights, можем директно да измерим доколко началният екран съкращава възприеманото време за изчакване. Данните показват, че един отличен скелетен екран може да подобри толерантността на потребителите към „бавна мрежа“ с повече от 50%.
