## Ръководство за създаване на ефект на анимация на границата със светлинно движение

> Как изграждате този важен елемент, който потребителите забелязват с един поглед, като използвате чист CSS? Всъщност не е толкова трудно. Номерът е просто да поемете по леко заобиколен път. В тази статия ще ви преведа как да изградите анимация за преместване на светлината от нулата и също така ще споделя някои от клопките, на които се натъкнахме, докато създавахме HagiCode.

## Фон

Ако работите върху интерфейса, вероятно сте имали това преживяване и преди: продуктов мениджър се приближава с израза „това определено е просто“ и казва: „Можем ли да добавим някакъв специален ефект към тази изпълнявана задача, така че потребителите да могат да я забележат веднага?“

Казвате разбира се, можем просто да променим цвета на рамката. След това те поклащат глава с онзи поглед, който казва: „Не разбираш“. Те отговарят: „Това не е достатъчно очевидно. Искам ефекта, при който светлината тече около границата, като в научнофантастичен филм.“

В този момент може да започнете да се чудите как да го изградите. платно? SVG? Или може CSS да се справи сам? В крайна сметка никой не иска да признае, че не знае как.

В съвременните уеб приложения анимациите за завъртане на светлината на рамката всъщност са много често срещани. Те се използват главно в няколко сценария като тези:

- **Индикатори за състояние**: Маркиране на текущи задачи или активни елементи
- **Визуален фокус**: Открояване на важни области на съдържанието
- **Подобрение на марката**: Създаване на елегантен, модерен, технологичен визуален стил
- **Сезонни теми**: Изграждане на празнична атмосфера за специални поводи

Сблъскахме се точно с това изискване при изграждането [HagiCode](https://hagicode.com). Потребителите трябваше да видят с един поглед кои сесии се изпълняват и кои предложения се обработват в момента. Опитахме няколко различни подхода. Някои пътеки бяха по-плавни, други бяха малко по-криволичещи. В крайна сметка се спряхме на доста зряла стратегия за внедряване.

## Основни идеи за внедряване

След като анализирахме кодовата база на HagiCode, ние обобщихме няколко основни модела за внедряване по-долу. Всеки от тях отговаря на различен сценарий, или с други думи, всеки съществува с причина.

### 1. Въртящо се сияние с коничен градиент (най-често срещано)

Това е класическият начин за прилагане на ефекта на гранична светлина. Основната идея е да се използва CSS `conic-gradient` за да създадете коничен градиент, след което го завъртете непрекъснато. Като улична лампа, която се върти в нощта, тя просто продължава да кръжи.

**Ключови елементи:**
- Използвайте `::before` псевдо-елемент за създаване на светещ слой
- Използвайте `conic-gradient` за определяне на градиентното цветово разпределение
- Използвайте `::after` псевдоелемент за маскиране на централната област (по избор)
- Използвайте `@keyframes` за прилагане на анимацията за въртене

### 2. Странична светеща линия

Това работи добре за индикатори за състояние в елементи от списък. Създавате тънка светеща линия от едната страна на елемента, вместо да анимирате цялата граница. Понякога малко светлина е достатъчна. Не е нужно да осветявате целия свят.

**Ключови елементи:**
- Тънък абсолютно позициониран линеен елемент
- Използвайте `box-shadow` за да създадете ефект на блясък
- Използвайте `scale` и `opacity` за дихателна анимация

### 3. Светещ фон със сянка

Ако не се нуждаете от пълния ефект на измитане и просто искате меко фоново сияние, наслояване на няколко слоя `box-shadow` стойности е достатъчно. Понякога по-простият вариант е по-добрият.

### 4. Поддръжка за достъпност

Тази част е лесна за пренебрегване, но е изключително важна. Всяка анимация трябва да отчита `prefers-reduced-motion` медийни заявки и предоставят статична алтернатива за потребители, които не искат анимация. Не всеки се радва на постоянно движение и зачитането на това предпочитание има значение.

## Опции за внедряване

### Опция 1: Въртяща се граница с коничен градиент (препоръчително)

Това е най-пълната реализация на светлинния ефект на широката граница и също така е опцията, която използваме най-често в HagiCode. В крайна сметка, ако нещо работи добре, защо да го заменим?

```css
/* Parent container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotating glow layer */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Mask layer (optional, for creating a hollow border effect) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Принципът зад тази опция е доста прост: създайте псевдоелемент, по-голям от родителския контейнер, начертайте коничен градиент върху него и го завъртете непрекъснато. Контейнерът родител използва `overflow: hidden`, така че само светлината, преминаваща около границата, остава видима. Това е малко като да гледате улична лампа през прозорец. Виждаш само малкото парче, което минава.

### Вариант 2: Опростена въртяща се светлинна рамка

Ако не се нуждаете от пълния ефект, HagiCode включва и по-лека версия от полезен клас. Понякога по-простият подход наистина е по-добър.

```css
/* Rotating light border utility class */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Accessibility support */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Забележете `will-change: transform` тук Той казва на браузъра „Този ​​елемент ще продължи да се променя“, така че браузърът да може да подготви някои оптимизации преди време и да поддържа анимацията по-плавна. Подготовката предварително обикновено е по-добра от бъркането в последния момент.

### Вариант 3: Странична светеща линия

Това е особено подходящо за индикатори за състояние на списъчен елемент и е точно това, което използва списъкът със сесии на HagiCode. Една тънка линия все още може да се открои сред много елементи. Това изглежда като житейски урок по свой начин.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Това използва `isolation: isolate` за създаване на нов контекст на подреждане, след което разчита на `z-index` за да контролирате реда на показване на всеки слой. `pointer-events: none` също е от съществено значение. В противен случай псевдоелементът ще блокира кликванията на потребителите. Някои неща могат да изглеждат добре, но все пак не трябва да пречат.

### Вариант 4: Увийте го в React компонент

Ако вашият проект използва React, можете да обвиете тази логика в компонент, особено обработката на достъпността. Напишете го веднъж, използвайте го много пъти. Това е целият смисъл.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Съвпадащият CSS модул:

```css
/* GlowBorder.module.css */

/* Animated version */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Static version (accessibility) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

The `useReducedMotion` кука от `framer-motion` автоматично открива системните предпочитания на потребителя. Ако потребителят е активирал намалено движение, то се връща `true`, а вместо това компонентът показва статичната версия. Зачитането на предпочитанията на потребителя е по-важно от налагането на ефектен ефект.

## Научени практически уроци

Това са някои от уроците, които научихме, докато изграждахме HagiCode. Бихте могли да ги наречете и белези от битка. Надяваме се, че ви помагат да избегнете някои отклонения.

### 1. Тематична променлива система

CSS променливите правят поддръжката на много теми особено удобна. Никой не иска да редактира купчина код всеки път, когато темата се промени.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Usage */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

По този начин превключването на теми изисква само промяна на класа на `html` елемент и всеки цвят на анимация се актуализира автоматично. Една кодова база, два стила. Точно това искаме.

### 2. Оптимизиране на производителността

** Използвайте `will-change` за подсказване на браузъра за оптимизиране:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Кажете на браузъра предварително и той ще ви помогне да оптимизирате. Много неща в живота работят по-добре с малко подготовка.

**Избягвайте използването на сложни кутии-сенки върху големи елементи:**

```css
/* Not ideal - using a blurred shadow on a large element */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Better - use a pseudo-element to limit the glowing area */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Тествахме това в HagiCode. Добавянето на замъглена сянка директно към голяма карта намали скоростта на кадрите при превъртане под 30 кадъра в секунда. Преминаването към псевдо-елемент върна нещата до стабилни 60 кадъра в секунда. Потребителите могат абсолютно да усетят тази разлика.

### 3. Достъпност

Наистина не трябва да пропускате това. Някои потребители намират анимацията за главозамайваща или разсейваща и зачитането на техните предпочитания е част от изграждането на добър продукт. Не е необходимо красивите неща да се налагат на всички.

**CSS медийна заявка:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Provide a static fallback */
    opacity: 1;
  }
}
```

**Откриване на потребителски предпочитания в React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Контрол на нивото на интензивност

Индикаторът за пропускателна способност на Token в HagiCode показва различни цветове на светене въз основа на пропускателната способност в реално време и това се прилага динамично. Различните състояния трябва да се изразяват по различен начин.

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Неща, за които да внимавате

Все още има няколко подробности, на които си струва да обърнете внимание, защото докато откриете тези проблеми по трудния начин, вече е твърде късно.

| Неща, за които трябва да внимавате | Обяснение |
|---------|------|
| **z-индекс управление** | Светещият слой трябва да използва подходящ `z-index` така че не пречи на взаимодействието със съдържанието |
| **показател-събития** | Светещият псевдоелемент трябва да се зададе `pointer-events: none` |
| **Преливане на границата** | Нуждите на родителския контейнер `overflow: hidden`, или трябва да коригирате размера на псевдоелемента |
| **Влияние върху производителността** | Сложните анимации могат да навредят на производителността на мобилни устройства, така че тествайте внимателно |
| **Тъмен режим** | Уверете се, че светещият цвят остава ясно видим на тъмен фон |
| **Превключване на теми** | Използвайте CSS променливи, така че цветовете на анимацията да се актуализират правилно, когато темата се промени |

### 6. Съвети за отстраняване на грешки

Псевдоелементите могат да бъдат малко трудни за намиране в инструментите за разработчици, така че можете временно да добавите рамка, за да проверите позицията.

```css
/* Temporarily show pseudo-element boundaries for debugging */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

След като приключите с позиционирането му, не забравяйте да коментирате или премахнете този ред. В противен случай производството може да стане неудобно доста бързо. Някои неща е по-добре да се оставят в разработка.
