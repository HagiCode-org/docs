## Технически анализ на системата за умения HagiCode: Изграждане на мащабируема платформа за управление на умения с изкуствен интелект

> Тази статия разглежда задълбочено архитектурата и внедряването на системата за управление на уменията в проекта HagiCode, като обхваща техническите детайли зад четири основни възможности: локално глобално управление, търсене на пазара, интелигентни препоръки и управление на доверен доставчик.

## Фон

В областта на асистентите за кодиране на AI, как да се разширят границите на възможностите на AI винаги е бил основен въпрос. Самият Claude Code вече е силен в подпомагането на кода, но различни екипи за разработка и различни технологични стекове често се нуждаят от специализирани възможности за специфични сценарии, като работа с внедряване на Docker, оптимизиране на база данни или генериране на компоненти от интерфейса. Точно тук системата за умения става особено важна.

По време на разработването на проекта HagiCode се натъкнахме на подобно предизвикателство: как да позволим на Клод Код да „научи“ нови професионални умения, както би направил човек, като същевременно поддържаме солидно потребителско изживяване и добра инженерна поддръжка? Този проблем е едновременно труден и прост по свой начин. Около този въпрос проектирахме и внедрихме цялостна система за управление на уменията.

Тази статия разглежда подробно техническата архитектура и основната реализация на системата. Предназначен е за разработчици, които се интересуват от разширяемостта на AI и интегрирането на инструменти на командния ред. Може да ви е полезно, а може и да не е, но поне е записано сега.

## Относно HagiCode

Подходът, споделен в тази статия, идва от нашия практически опит в [HagiCode](https://hagicode.com) проект. HagiCode е проект за асистент за кодиране с изкуствен интелект с отворен код, предназначен да помогне на екипите за разработка да подобрят инженерната ефективност. Стекът на проекта включва ASP.NET Core, разпределената рамка Orleans, интерфейс TanStack Start + React и подсистемата за управление на уменията, представена в тази статия.

Хранилището на GitHub е [HagiCode-org/сайт](https://github.com/HagiCode-org/site). Ако намирате техническия подход в тази статия за ценен, не се колебайте да му дадете звезда. Все пак повече звезди подобряват настроението.

## Общ преглед на системната архитектура

Системата Skill използва архитектура, разделена от интерфейс и бекенд. В това няма нищо особено мистериозно.

**Frontend** използва TanStack Start + React за изграждане на потребителския интерфейс, със състояние на управление на Redux Toolkit. Четирите основни възможности се свързват директно с четири компонента на раздела: Местни умения, Галерия с умения, Интелигентни препоръки и Доверени доставчици. В крайна сметка, дизайнът е най-вече за подобряване на потребителското изживяване.

**Backend** е базиран на ASP.NET Core + ABP Framework, използвайки Orleans Grain за разпределено управление на състоянието. Онлайн API клиентът обгръща `IOnlineApiClient` интерфейс за комуникация с услугата каталог на отдалечени умения.

Общият архитектурен принцип е да се отдели изпълнението на команди от бизнес логиката. Чрез модела на адаптера подробностите за изпълнението на изпълнението на командата npm/npx са скрити в независими модули. В края на краищата, никой наистина не иска извикванията от командния ред да са разпръснати из цялата кодова база.

## Основна способност 1: Локално глобално управление

Местното глобално управление е най-основният модул. Той отговаря за изброяването на инсталираните умения и поддържането на операции за деинсталиране. Тук няма нищо прекалено сложно; става въпрос най-вече за това да правиш добре основите.

### Технически подход

Изпълнението живее `LocalSkillsTab.tsx` и `LocalSkillCommandAdapter.cs`. Основната идея е да увиете `npx skills` команда, анализира своя JSON изход и го преобразува във вътрешни структури от данни. Звучи просто и на практика най-вече е така.

```csharp
public async Task<IReadOnlyList<LocalSkillInventoryResponseDto>> GetLocalSkillsAsync(
    CancellationToken cancellationToken = default)
{
    var result = await _commandAdapter.ListGlobalSkillsAsync(cancellationToken);
    return result.Skills.Select(skill => new LocalSkillInventoryResponseDto
    {
        Name = skill.Name,
        Version = skill.Version,
        Source = skill.Source,
        InstalledPath = skill.InstalledPath,
        Description = skill.Description
    }).ToList();
}
```

Потокът от данни е много ясен: интерфейсът изпраща заявка -> `SkillGalleryAppService` получава го -> `LocalSkillCommandAdapter` изпълнява командата npx -> резултатът от JSON се анализира -> връща се DTO. Всяка стъпка следва естествено от предишната.

Деинсталирането на умения използва `npx skills remove -g <skillName> -y` и системата автоматично обработва зависимостите и почистването. Инсталационните метаданни се съхраняват в `managed-install.json` вътре в директорията с умения, записвайки информация като време на инсталиране и изходна версия за по-късни актуализации и одит. Някои неща просто си струва да бъдат записани.

### Процесът на инсталиране в детайли

Инсталирането на умения изисква няколко координирани стъпки. Всъщност не е особено сложно:

```csharp
public async Task<SkillInstallResultDto> InstallAsync(
    SkillInstallRequestDto request,
    CancellationToken cancellationToken = default)
{
    // 1. Normalize the installation reference
    var normalized = _referenceNormalizer.Normalize(
        request.SkillId,
        request.Source,
        request.SkillSlug,
        request.Version);

    // 2. Check prerequisites
    await _prerequisiteChecker.CheckAsync(cancellationToken);

    // 3. Acquire installation lock
    using var installLock = await _lockProvider.AcquireAsync(normalized.SkillId);

    // 4. Execute installation command
    var result = await _installCommandRunner.ExecuteAsync(
        new SkillInstallCommandExecutionRequest
        {
            Command = $"npx skills add {normalized.FullReference} -g -y",
            Timeout = TimeSpan.FromMinutes(4)
        },
        cancellationToken);

    // 5. Persist installation metadata
    await _metadataStore.WriteAsync(normalized.SkillPath, request);

    return new SkillInstallResultDto { Success = result.Success };
}
```

Тук се използват няколко ключови шаблона за проектиране: **референтният нормализатор** преобразува различни входни формати, като напр. `tanweai/pua` и `@opencode/docker-skill`, в единно вътрешно представителство; **инсталационният заключващ механизъм** гарантира, че само една инсталационна операция може да се изпълнява за едно и също умение в даден момент; и **поточно извеждане** избутва напредъка на инсталацията към интерфейса в реално време чрез изпратени от сървъра събития, така че потребителите да могат да гледат логове, подобни на терминал, докато се случват.

В крайна сметка всички тези модели са там с една цел: да поддържат системата по-лесна за използване и поддръжка.

## Основна възможност 2: Търсене на пазара

Търсенето в Marketplace позволява на потребителите да откриват и инсталират умения от общността. Способността на един човек винаги е ограничена; колективното знание отива много по-далеч.

### Технически подход

Функцията за търсене разчита на онлайн API `https://api.hagicode.com/v1/skills/search`. За да подобри скоростта на реакция, системата прилага кеширане. Кешът е малко като паметта: ако държите полезни неща наоколо, не е нужно да мислите толкова много следващия път.

```csharp
private async Task<IReadOnlyList<SkillGallerySkillDto>> SearchCatalogAsync(
    string query,
    CancellationToken cancellationToken,
    IReadOnlySet<string>? allowedSources = null)
{
    var cacheKey = $"skill_search:{query}:{string.Join(",", allowedSources ?? Array.Empty<string>())}";

    if (_memoryCache.TryGetValue(cacheKey, out var cached))
        return (IReadOnlyList<SkillGallerySkillDto>)cached!;

    var response = await _onlineApiClient.SearchAsync(
        new SearchSkillsRequest
        {
            Query = query,
            Limit = _options.LimitPerQuery,
        },
        cancellationToken);

    var results = response.Skills
        .Where(skill => allowedSources is null || allowedSources.Contains(skill.Source))
        .Select(skill => new SkillGallerySkillDto { ... })
        .ToList();

    _memoryCache.Set(cacheKey, results, TimeSpan.FromMinutes(10));
    return results;
}
```

Резултатите от търсенето поддържат филтриране по надеждни източници, така че потребителите виждат само източници на умения, на които имат доверие. Начални заявки като `popular` и `recent` се използват за инициализиране на каталога, което позволява на потребителите да видят препоръчаните популярни умения при първото им отваряне. Първите впечатления все още имат значение.

## Основна възможност 3: Интелигентни препоръки

Интелигентните препоръки са най-сложната част от системата. Те могат автоматично да препоръчат най-подходящите умения въз основа на текущия контекст на проекта. Колкото и сложен да е, все пак си струва да се построи.

### Поток на препоръките

Пълният поток от препоръки е разделен на пет етапа:

```
1. Build project context
   ↓
2. AI generates search queries
   ↓
3. Search the online catalog in parallel
   ↓
4. AI ranks the candidates
   ↓
5. Return the recommendation list
```

Първо, системата анализира характеристики като технологичния стек на проекта, езиците за програмиране и структурата на домейна, за да изгради „профил на проекта“. Този профил е малко като автобиография, записваща ключовите характеристики на проекта.

След това се използва AI Grain за генериране на целеви заявки за търсене. Този дизайн всъщност е доста интересен: вместо директно да питаме AI „Какви умения трябва да препоръчам?“, ние първо го молим да помисли за „Какви думи за търсене е вероятно да намерят подходящи умения?“ Понякога начинът, по който задавате въпроса, е по-важен от самия отговор:

```csharp
var queryGeneration = await aiGrain.GenerateSkillRecommendationQueriesAsync(
    projectContext,      // Project context
    locale,              // User language preference
    maxQueries,           // Maximum number of queries
    effectiveSearchHero); // AI model selection
```

След това тези заявки за търсене се изпълняват паралелно, за да се събере списък с умения на кандидати. В края на краищата паралелната обработка е просто начин да спестите време.

И накрая, друг AI Grain класира уменията на кандидата. Тази стъпка взема предвид фактори като уместност на уменията за проекта, статус на доверие и исторически предпочитания на потребителите:

```csharp
var ranking = await aiGrain.RankSkillRecommendationsAsync(
    projectContext,
    candidates,
    installedSkillNames,
    locale,
    maxRecommendations,
    effectiveRankingHero);

response.Items = MergeRecommendations(projectContext, candidates, ranking, maxRecommendations);
```

### Резервен механизъм

AI моделите могат да реагират бавно или временно да станат недостъпни. Дори и най-добрите системи понякога се спъват. Поради тази причина системата включва детерминистичен резервен механизъм: когато AI услугата е недостъпна, тя използва базиран на правила евристичен алгоритъм за генериране на препоръки, като извеждане на вероятно необходими умения от зависимости в `package.json`.

Казано ясно, този резервен механизъм е просто резервен план за системата.

## Основна възможност 4: Управление на доверен доставчик

Управлението на доверен доставчик позволява на потребителите да контролират кои източници на умения се считат за надеждни. Доверието все още е нещо, което потребителите трябва да могат да определят сами.

### Правила за съответствие

Доверените доставчици поддържат две правила за съвпадение: **точно съвпадение** (`exact`) и **съвпадение на префикса** (`prefix`).

```csharp
public static TrustedSkillProviderResolutionSnapshot Resolve(
    TrustedSkillProviderSnapshot snapshot,
    string source)
{
    var normalizedSource = Normalize(source);

    foreach (var entry in snapshot.Entries.OrderBy(e => e.SortOrder))
    {
        if (!entry.IsEnabled) continue;

        foreach (var rule in entry.MatchRules)
        {
            bool isMatch = rule.MatchType switch
            {
                TrustedSkillProviderMatchRuleType.Exact
                    => string.Equals(normalizedSource, Normalize(rule.Value),
                        StringComparison.OrdinalIgnoreCase),
                TrustedSkillProviderMatchRuleType.Prefix
                    => normalizedSource.StartsWith(Normalize(rule.Value) + "/",
                        StringComparison.OrdinalIgnoreCase),
                _ => false
            };

            if (isMatch)
                return new TrustedSkillProviderResolutionSnapshot
                {
                    IsTrustedSource = true,
                    ProviderId = entry.ProviderId,
                    DisplayName = entry.DisplayName
                };
        }
    }

    return new TrustedSkillProviderResolutionSnapshot { IsTrustedSource = false };
}
```

Вградените доверени доставчици включват добре познати организации и проекти като Vercel, Azure, anthropics, Microsoft и използване на браузър. Персонализираните доставчици могат да се добавят чрез конфигурационни файлове чрез указване на ID на доставчик, показвано име, етикет на значка, правила за съвпадение и др. Светът е достатъчно голям, че само доверието на няколко вградени компонента никога няма да е достатъчно.

### Изпълнение на постоянство

Доверената конфигурация се поддържа с помощта на Orleans Grain:

```csharp
public class TrustedSkillProviderGrain : Grain<TrustedSkillProviderState>,
    ITrustedSkillProviderGrain
{
    public async Task UpdateConfigurationAsync(TrustedSkillProviderSnapshot snapshot)
    {
        State.Snapshot = snapshot;
        await WriteStateAsync();
    }

    public Task<TrustedSkillProviderSnapshot> GetConfigurationAsync()
    {
        return Task.FromResult(State.Snapshot);
    }
}
```

Предимството на този подход е, че промените в конфигурацията се синхронизират автоматично във всички възли, без да е необходимо ръчно опресняване на кешовете. Автоматизацията в крайна сметка е да позволи на хората да се тревожат по-малко.

## Ключов технически дизайн

### Шаблон на адаптер за изпълнение на команда

Системата Skill трябва да изпълнява различни npx команди. Ако тази логика беше разпръсната навсякъде, кодът бързо щеше да стане труден за поддръжка. Ето защо създадохме адаптерен интерфейс. В крайна сметка шаблоните за проектиране съществуват, за да направят кода по-лесен за поддръжка:

```csharp
public interface ISkillInstallCommandRunner
{
    Task<SkillInstallCommandExecutionResult> ExecuteAsync(
        SkillInstallCommandExecutionRequest request,
        CancellationToken cancellationToken = default);
}
```

Различните команди имат различни изпълнения на изпълнители, но всички имплементират един и същ интерфейс, което прави тестването и подмяната лесни.

### SSE стрийминг изход

Напредъкът на инсталирането се изпраща към интерфейса в реално време чрез изпратени от сървъра събития:

```csharp
public async Task InstallWithProgressAsync(
    SkillInstallRequestDto request,
    IServerStreamWriter<SkillInstallProgressEventDto> stream,
    CancellationToken cancellationToken)
{
    var process = new Process
    {
        StartInfo = new ProcessStartInfo
        {
            FileName = "npx",
            Arguments = $"skills add {request.FullReference} -g -y",
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false
        }
    };

    process.OutputDataReceived += async (sender, e) =>
    {
        await stream.WriteAsync(new SkillInstallProgressEventDto
        {
            EventType = "output",
            Data = e.Data ?? string.Empty
        });
    };

    process.Start();
    process.BeginOutputReadLine();
    await process.WaitForExitAsync(cancellationToken);
}
```

Във фронтенда потребителите могат да видят изход, подобен на терминал, в реално време, което прави изживяването много интуитивно. Обратната връзка в реално време помага на хората да се чувстват спокойни.

## Практическо ръководство

### Инсталиране на общностно умение

Вземете инсталирането на `pua` умение като пример (това е популярно умение на общността):

1. Отворете секцията с умения и превключете на `Skill Gallery` раздел
2. Въведете `pua` в полето за търсене
3. Щракнете върху резултата от търсенето, за да видите подробностите за уменията
4. Щракнете върху `Install` бутон
5. Преминете към `Local Skills` раздела, за да потвърдите, че инсталацията е успешна

Командата за инсталиране е `npx skills add tanweai/pua -g -y`, а системата обработва всички подробности автоматично. Всъщност няма толкова много стъпки, след като ги направите една по една.

### Добавяне на персонализиран доверен източник

Ако вашият екип има собствено хранилище на умения, можете да го добавите като доверен източник:

```yaml
providerId: "my-team"
displayName: "My Team Skills"
badgeLabel: "MyTeam"
isEnabled: true
sortOrder: 100
matchRules:
  - matchType: "prefix"
    value: "my-team/"
  - matchType: "exact"
    value: "my-team/special-skill"
```

По този начин всички умения от вашия екип ще показват значка за надежден, което прави потребителите по-удобни при инсталирането им. Етикетите и сигналите помагат на хората да се чувстват по-уверени.

### Основи за развитие на умения

Създаването на персонализирано умение изисква следната структура:

```
my-skill/
├── SKILL.md          # Skill metadata (YAML front matter)
├── index.ts          # Skill entry point
├── agents/           # Supported agent configuration
└── references/       # Reference resources
```

Един пример `SKILL.md` формат:

```yaml
---
name: my-skill
description: A brief description of what this skill does
---

# My Skill

Detailed documentation...
```

## Бележки

1. **Мрежови изисквания**: търсенето и инсталирането на умения изискват достъп до `api.hagicode.com` и npm регистъра
2. **Node.js версия**: Препоръчва се Node.js 18 или по-нова версия
3. **Изисквания за разрешение**: необходими са глобални разрешения за инсталиране на npm
4. **Контрол на паралелността**: само една операция за инсталиране или деинсталиране може да се изпълнява за едно и също умение в даден момент
5. **Настройки за изчакване**: времето за изчакване по подразбиране за инсталиране е 4 минути, но сложните сценарии може да изискват корекция

Тези бележки съществуват, в крайна сметка, за да помогнат нещата да вървят гладко.
