Ако искате да инсталирате Hermes и да започнете да го използвате, най-краткият път всъщност е само три стъпки:

1. Изпълнете официалната инсталационна команда
2. Стартирайте CLI във вашия терминал с `hermes`
3. Ако искате да продължите да го използвате във Feishu, тогава конфигурирайте `hermes gateway setup`

Тази статия не се опитва да обясни всички възможности на Hermes наведнъж. Вместо това, той ви помага първо да завършите най-важния цикъл за начинаещи: инсталирайте го, стартирайте го, започнете да го използвате и след това го свържете към един от най-често срещаните сценарии за платформа за съобщения.

## Какво е Hermes и за кого е

Hermes Agent е AI агент, който можете да използвате или от локален терминал, или през шлюз на платформа за съобщения.

За повечето разработчици той има две общи входни точки:

- **CLI**: Тип `hermes` във вашия терминал, за да влезете директно в интерактивния интерфейс.
- **Шлюз за съобщения**: Стартирайте `hermes gateway`, след това разговаряйте с него от платформи като Feishu, Telegram, Discord и Slack.

Ако целта ви в момента е просто да започнете бързо, не обръщайте реда. Вместо това започнете с този път:

- Първо инсталирайте Hermes
- Първо се уверете, че работи от CLI
- След това решете дали искате да свържете платформа за съобщения

Това прави проблемите по-лесни за диагностициране и е по-подходящо за хора, които използват Hermes за първи път.

## Какво трябва да знаете, преди да инсталирате Hermes

Според Hermes README, официалният път за бързо инсталиране поддържа тези среди:

- Linux
- macOS
- WSL2
- Android чрез Termux

### Бележки за потребители на Windows

Понастоящем Hermes не поддържа директно изпълнение на Windows. Ако използвате Windows, препоръчителният подход е първо да инсталирате WSL2 и след това да изпълните инсталационната команда вътре в WSL2.

Най-добре е това да стане ясно в началото, защото много неуспешни инсталации не са причинени от самата команда, а от използването на неподдържана среда за изпълнение.

## Как да инсталирате Hermes

Командата за бързо инсталиране, предоставена в Hermes README, е:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Тази команда изпълнява официалния инсталационен скрипт и обработва специфични за платформата стъпки за инициализация.

### Презаредете вашата обвивка след инсталиране

След като инсталацията приключи, първо презаредете обвивката си. Най-често срещаната команда е:

```bash
source ~/.bashrc
```

Ако използвате zsh, можете да използвате:

```bash
source ~/.zshrc
```

### Как да потвърдите, че Hermes е инсталиран правилно

Най-директният начин за проверка е да стартирате:

```bash
hermes
```

Ако искате допълнително потвърждение, че вашата конфигурация и зависимости работят, можете също да стартирате:

```bash
hermes doctor
```

`hermes doctor` е особено полезно в следните ситуации:

- Командата се държи необичайно след инсталиране
- Model configuration fails
- The gateway fails to start
- Не сте сигурни дали зависимостите ви от средата са завършени

## How to start using Hermes for the first time

### Започнете с CLI

Ако просто искате да потвърдите възможно най-бързо, че Hermes работи, най-простият метод е:

```bash
hermes
```

This launches the interactive Hermes CLI. За първи потребители на Hermes това е и най-препоръчителната отправна точка, защото първо можете да проверите най-важните неща:

- Whether the command is actually available
- Дали текущата конфигурация на модела работи правилно
- Дали инструменталната верига на терминала работи правилно
- Дали стилът на взаимодействие отговаря на това, от което се нуждаете

### Тези команди са достатъчни за вашия първи кръг от настройка

Hermes README изброява няколко високочестотни команди и заедно те образуват практичен път за първа употреба:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Ако не сте сигурни какво прави всеки от тях, запомнете ги така:

- `hermes model`: choose or switch models
- `hermes tools`: view and configure currently available tools
- `hermes config set`: променете конкретни конфигурационни елементи
- `hermes setup`: стартирайте пълния съветник за инициализация веднъж
- `hermes update`: актуализирайте Hermes
- `hermes doctor`: отстраняване на проблеми

За начинаещи най-практичната поръчка обикновено е:

1. Бягай `hermes model` първи
2. Ако искате да конфигурирате всички общи опции наведнъж, стартирайте `hermes setup`

## Двата най-често срещани начина за използване на Hermes

### 1. Използвайте Hermes в терминала като ежедневен асистент за разработка

CLI mode is a good fit for these scenarios:

- Задавайте въпроси директно, докато пишете код локално
- Проверявайте проекти, редактирайте файлове и изпълнявайте команди
- Извършете еднократна работа по отстраняване на грешки или преглед
- Работете непрекъснато в текущата работна директория

Най-голямото му предимство е, че това е най-краткият път: без допълнителна интеграция на платформа, без конфигурация на бот, която да управлявате отпред, и това е най-добрият начин да изградите своя първи набор от навици за използване.

### 2. Use Hermes through a messaging platform

Ако искате да разговаряте с Hermes на платформи като Feishu, Telegram или Discord, трябва да използвате шлюза за съобщения.

The most common entry commands are:

```bash
hermes gateway setup
hermes gateway
```

По-конкретно:

- `hermes gateway setup` is used for interactive platform configuration
- `hermes gateway` се използва за стартиране на процеса на шлюза

Според официалната документация шлюзът е унифициран фонов процес, който свързва вашите конфигурирани платформи, управлява сесии и обработва функции като cron задания.

## Като използваме Feishu като пример: как да свържете Hermes с платформа за съобщения

Ако по-голямата част от ежедневната ви работа се случва във Feishu, тогава Feishu/Lark е много естествен начин да използвате Hermes.

### Минималният жизнеспособен път на интеграция

Официалната документация препоръчва тази команда за въвеждане за Feishu/Lark:

```bash
hermes gateway setup
```

След като го стартирате, просто изберете **Feishu / Lark** в съветника.

Документацията на Feishu описва два режима на свързване:

- `websocket`: препоръчително
- `webhook`: по избор

Ако Hermes работи на вашия лаптоп, работна станция или частен сървър, използвайки `websocket` първото обикновено е по-просто, защото не е необходимо да излагате публичен URL адрес за обратно извикване.

### Ако го конфигурирате ръчно, поне знайте тези променливи

Ако не използвате съветника и пишете конфигурацията ръчно, документацията на Feishu изброява тези основни променливи:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Две от тях заслужават специално внимание:

- `FEISHU_ALLOWED_USERS`: препоръчително, така че не всеки, който може да достигне до бота, може да го използва директно
- `FEISHU_HOME_CHANNEL`: позволява ви да дефинирате предварително домашен чат, за да получавате резултати от cron или известия по подразбиране

### Защо Hermes понякога не отговаря в груповите чатове на Feishu

Тази подробност е лесно да се пропусне: в груповите чатове на Feishu Hermes не отговаря на всяко съобщение по подразбиране.

Официалната документация ясно посочва:

- При директни съобщения Hermes отговаря на съобщения
- В груповите чатове трябва изрично `@` ботът, преди да обработи съобщението

Ако искате да зададете разговор на Feishu като начален канал, можете също да използвате това в чата:

```text
/set-home
```

Или го дефинирайте в конфигурацията предварително:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Командите на Хермес, които начинаещите трябва първо да запомнят

Независимо дали използвате Hermes в CLI или на платформа за съобщения, запомнянето на следните команди вече е достатъчно, за да започнете:

- `/new` или `/reset`: започнете нова сесия
- `/model`: прегледайте или превключете модела
- `/retry`: повторете предишния ход
- `/undo`: отменете предишното взаимодействие
- `/compress`: ръчно компресиране на контекста
- `/help`: вижте помощ

Ако използвате основно Hermes на платформа за съобщения, запомнете още едно:

- `/sethome` или `/set-home`: задайте текущия чат като начален канал

Тези команди обхващат най-често срещаните операции в начален етап: рестартиране, коригиране, връщане назад, проверка и продължаване.

## Често задавани въпроси

### Може ли Windows да инсталира Hermes директно?

Не. Текущата официална документация ясно посочва, че собственият Windows не се поддържа и се препоръчва WSL2.

### Какво трябва да направя, ако пиша `hermes` не прави нищо след инсталацията?

Най-добре е да отстраните неизправностите в този ред:

1. Първо презаредете вашата обвивка, например с `source ~/.bashrc`
2. Бягай `hermes` отново
3. Ако все още е ненормално, бягайте `hermes doctor`

### Защо ботът не отговаря в група Feishu?

Първо проверете тези три неща:

- Независимо дали вие `@` спомена Хермес в групата
- Дали `FEISHU_ALLOWED_USERS` ограничава текущия потребител
- Дали текущата политика за групов чат позволява обработка на групови съобщения

Според официалната документация на Feishu, изричното използване на @mention се изисква в сценарии за групов чат.

## Препоръчителни следващи стъпки

Ако просто искате да започнете да използвате Hermes възможно най-бързо, това е най-препоръчителният ред:

1. Първо изпълнете инсталационната команда
2. Започнете с `hermes` първо в локалния CLI
3. Използвайте `hermes model` и `hermes setup` за да завършите основната конфигурация
4. Ако искате да продължите да го използвате във Feishu, тогава конфигурирайте `hermes gateway setup`

Ако тази статия е първата част от поредица, нейната най-добра роля не е да обяснява всяка разширена функция наведнъж, а първо да привлече потребителите към вратата.
