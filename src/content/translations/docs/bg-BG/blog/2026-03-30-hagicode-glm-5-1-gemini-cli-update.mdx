## Фон

Времето наистина тече бързо. Развитието на големи езикови модели се издига като бамбук през пролетта. Не много отдавна ние все още приветствахме „ИИ, който може да пише код“. Вече сме в ера на сътрудничество с множество модели и интеграция с множество инструменти. Вълнуващо ли е? може би В крайна сметка това, от което разработчиците се нуждаят, никога не е бил само самият инструмент, а лекотата на адаптиране към различни сценарии и гъвкаво превключване, когато е необходимо.

Като платформа за кодиране с AI, HagiCode наскоро приветства две важни разработки: първо, пълната интеграция на GLM-5.1 модела на Zhipu AI; второ, официалното добавяне на Gemini CLI като десетия поддържан Agent CLI. Тези две актуализации може да не звучат разтърсващо, но безспорно са добра новина за продължаващото съзряване на платформата.

GLM-5.1 е най-новият водещ модел на Zhipu AI. В сравнение с GLM-5.0, той предлага по-силно разсъждение, по-задълбочено разбиране на кода и по-плавно извикване на инструменти. По-важното е, че това е първият GLM модел, който поддържа въвеждане на изображения. какво значи това Това означава, че потребителите могат да оставят AI да гледа директно екранна снимка, вместо да се мъчат да опишат проблема с думи. След като сте използвали това удобство, веднага разбирате неговата стойност.

В същото време, чрез `HagiCode.Libs.Providers` архитектура, HagiCode успешно интегрира Gemini CLI в платформата. Това вече е десетият агент CLI. Честно казано, достигането до този момент наистина носи скромно чувство за постижение.

Също така си струва да се спомене, че функцията за качване на изображения на HagiCode позволява на потребителите да комуникират с AI директно чрез екранни снимки. Дори когато работи GLM 4.7, платформата все още работи добре и вече е помогнала за изпълнението на много важни задачи за изграждане. Що се отнася до GLM-5.1, естествено, той отива една крачка напред.

## Мулти-CLI архитектурен дизайн

Една от основните силни страни на HagiCode е неговата поддръжка за множество CLI инструменти за програмиране на AI чрез унифициран слой на абстракция. Предимството на този дизайн всъщност е доста просто: нови инструменти могат да влязат, стари инструменти могат да останат и кодовата база не се превръща в хаос. Честно казано, така всеки би искал да работи животът.

### AIProviderType enum

Платформата дефинира поддържаните типове CLI доставчици чрез `AIProviderType` enum:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI (new)
}
```

Както можете да видите, Gemini CLI се присъединява към това семейство като десети член. Всеки CLI има свои собствени различни характеристики и сценарии на използване, така че потребителите могат да избират гъвкаво въз основа на техните нужди. Все пак много пътища водят към Рим; някои са просто по-лесни от други.

### Архитектура на доставчика

`HagiCode.Libs.Providers` предоставя унифициран интерфейс на доставчика, който прави всяка CLI интеграция стандартизирана и кратка. Вземайки Gemini CLI като пример:

```csharp
public class GeminiProvider : ICliProvider<GeminiOptions>
{
    private static readonly string[] DefaultExecutableCandidates = ["gemini", "gemini-cli"];
    private const string ManagedBootstrapArgument = "--acp";

    public string Name => "gemini";
    public bool IsAvailable => _executableResolver.ResolveFirstAvailablePath(DefaultExecutableCandidates) is not null;
}
```

Предимствата на този дизайн са:
- Интегрирането на нов CLI изисква само прилагане на един клас доставчик
- Унифицирано управление на жизнения цикъл и обединяване на сесии
- Автоматизирано разрешаване на псевдоними и откриване на изпълними файлове

Казано ясно, този дизайн превръща сложните неща в по-прости и прави живота малко по-лесен.

### Регистър на доставчика

Регистърът на доставчика автоматично обработва съпоставяне на псевдоними и регистрация:

```csharp
if (provider is GeminiProvider)
{
    registry.Register(provider.Name, provider, ["gemini-cli"]);
    continue;
}
```

Това означава, че потребителите могат да извикват Gemini CLI с двете `gemini` или `gemini-cli`, и системата ще го разпознае автоматично. Това е като приятел с официално име и псевдоним - така или иначе, хората знаят кого имате предвид.

## Поддръжка на модел GLM-5.1

GLM-5.1 е най-новият водещ модел на Zhipu AI и HagiCode завърши пълната поддръжка за него.

### Каталог на средните професии

HagiCode управлява всички поддържани модели чрез Каталога за вторични професии. Ето конфигурацията за серията GLM:

| ID на модела | Име | Поддържа изображение | Съвместими CLI семейства |
|----------|------|----------------|-------------------------|
| `glm-4.7` | GLM 4.7 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM 5 Turbo | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (Legacy) | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | **вярно** | claude, codebuddy, hermes, qoder, kiro |

Ключовите характеристики на GLM-5.1 могат да бъдат обобщени, както следва:
- Идентификатор на самостоятелна версия без наследен багаж
- Първият GLM модел, който поддържа въвеждане на изображение
- По-силно разсъждение и разбиране на кода
- Широка мулти-CLI съвместимост

### GLM-5.1 срещу GLM-5.0

На ниво код ключовата разлика между GLM-5.1 и GLM-5.0 е показана тук:

```csharp
// GLM-5.0 (Legacy) - contains special retention logic
private const string Glm50CodebuddySecondaryProfessionId = "secondary-glm-5-codebuddy";
private const string Glm50CodebuddyModelValue = "glm-5.0";

// GLM-5.1 - standalone new model identifier
private const string Glm51SecondaryProfessionId = "secondary-glm-5-1";
private const string Glm51ModelValue = "glm-5.1";
```

GLM-5.0 носи етикета „Legacy“, защото е идентификатор на стара версия, запазен за обратна съвместимост. GLM-5.1, напротив, е съвсем нова самостоятелна версия без историческо бреме. Някои неща остават в миналото; други пътуват по-леко и се движат по-бързо.

### Конфигурирайте GLM-5.1

Ето пример за конфигурация за използване на GLM-5.1 в HagiCode:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

## Функция за качване на изображения

Поддръжката на изображения на HagiCode се реализира чрез `SupportsImage` имот на `SecondaryProfession`:

```csharp
public class HeroSecondaryProfessionSettingDto
{
    public bool SupportsImage { get; set; }
}
```

В каталога на вторичните професии конфигурацията на GLM-5.1 изглежда така:

```json
{
  "id": "secondary-glm-5-1",
  "supportsImage": true
}
```

Това означава, че потребителите могат да качват екранни снимки директно за AI анализ, като например:
- Екранни снимки на съобщения за грешки
- Проблеми в екрана на потребителския интерфейс
- Диаграми за визуализация на данни
- Резултати от изпълнението на кода

Вече няма нужда да описвате всичко ръчно - просто качете екранната снимка. Удобството на тази функция е очевидно, след като сте я използвали. Понякога един поглед казва повече от дълго обяснение.

## Gemini CLI интеграция

Като десети Agent CLI, Gemini CLI е интегриран в HagiCode чрез стандартната архитектура на доставчика.

### Опции за конфигурация

Gemini CLI поддържа богат набор от опции за конфигурация:

```csharp
public class GeminiOptions
{
    public string? ExecutablePath { get; set; }
    public string? WorkingDirectory { get; set; }
    public string? SessionId { get; set; }
    public string? Model { get; set; }
    public string? AuthenticationMethod { get; set; }
    public string? AuthenticationToken { get; set; }
    public Dictionary<string, string?> AuthenticationInfo { get; set; }
    public Dictionary<string, string?> EnvironmentVariables { get; set; }
    public string[] ExtraArguments { get; set; }
    public TimeSpan? StartupTimeout { get; set; }
    public CliPoolSettings? PoolSettings { get; set; }
}
```

Тези опции покриват всичко от основната настройка до разширените функции, давайки на потребителите гъвкавостта да конфигурират CLI според собствените си нужди. Работният процес на всеки е различен, така че малко гъвкавост винаги е добре дошла.

### ACP комуникационен протокол

Gemini CLI поддържа ACP (Agent Communication Protocol), който е унифицираният CLI комуникационен стандарт на HagiCode. Чрез ACP различните CLI могат да взаимодействат с платформата по последователен начин, което значително опростява работата по интеграцията. Накратко, той стандартизира сложните части, така че всеки да може да работи по-лесно.

## Най-добри практики за мулти-CLI интеграция

`HagiCode.Libs.Providers` осигурява единен механизъм за регистрация и използване:

```csharp
services.AddHagiCodeLibs();

var gemini = serviceProvider.GetRequiredService<ICliProvider<GeminiOptions>>();
var codebuddy = serviceProvider.GetRequiredService<ICliProvider<CodebuddyOptions>>();
var hermes = serviceProvider.GetRequiredService<ICliProvider<HermesOptions>>();
```

Този дизайн на инжектиране на зависимости поддържа използването в различни CLI много стегнато и също така прави тестването на единици и подигравката по-удобни. Чистият код е начин да бъдете отговорни към себе си.

### Бележки

Има няколко неща, които трябва да имате предвид при реална употреба:

1. **Конфигурация на API ключ**: Уверете се `ANTHROPIC_AUTH_TOKEN` е настроен правилно или моделът не може да бъде извикан
2. **Наличност на модела**: GLM-5.1 трябва да бъде активиран от съответния доставчик на модел
3. **Функция на изображението**: Само модели с `supportsImage: true` може да използва качване на изображения
4. **CLI инсталация**: Преди да използвате Gemini CLI, уверете се `gemini` или `gemini-cli` е в системата PATH

Това може да са дребни детайли, но дребните детайли, обработени лошо, могат да се превърнат в големи проблеми, така че си струва да им се обърне внимание.
