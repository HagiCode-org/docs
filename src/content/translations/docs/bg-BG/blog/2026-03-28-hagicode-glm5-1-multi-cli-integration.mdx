## Фон

В проекта Hagicode потребителите могат да избират от множество CLI инструменти за управление на асистенти за програмиране на AI, включително Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI и др. Тези CLI инструменти сами по себе си са инструменти за програмиране на AI с общо предназначение, но чрез абстракционния слой на Hagicode те могат гъвкаво да се свързват с различни доставчици на AI модели.

Zhipu AI (ZAI) предоставя интерфейс, съвместим с Anthropic Claude API, което позволява на тези CLI инструменти да използват директно модели от серия GLM. Сред тях GLM-5.1 е най-новата версия на голям езиков модел на Zhipu, със значителни подобрения спрямо GLM-5.0.

## CLI абстракционна архитектура на Hagicode

Hagicode дефинира 11 типа CLI доставчик чрез `AIProviderType` enum, обхващащ основните CLI инструменти за програмиране на AI:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Всеки CLI има съответна конфигурация на параметрите на модела и поддържа `model` и `reasoning` параметри:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],
        [AIProviderType.CodexCli] = ["model", "reasoning"],
        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],

        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini does not support the reasoning parameter
        // ...
    };
```

## Използване на GLM-5.1 в различни CLI

### 1. Claude Code CLI + GLM-5.1

CLI Claude Code е един от основните CLI на Hagicode и се конфигурира чрез конфигурационната система Hero:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Кореспондентски `HeroEquipmentCatalogItem` конфигурация:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI е най-гъвкавият CLI и поддържа указване на всеки модел в `provider/model` формат:

**Метод 1: Използвайте префикса на доставчик на ZAI**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Метод 2: Използвайте директно ID на модела**
```json
{
  "model": "glm-5.1"
}
```

**Метод 3: Потребителски интерфейс за конфигурация на предния край**

в `HeroModelEquipmentForm.tsx`, OpenCode CLI има специален намек за контейнер:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Потребителите могат да въвеждат:
```
zai/glm-5.1
glm-5.1
```

**Логика за синтактично анализиране на CLI модел на OpenCode**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI има специална наследена логика за обработка:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Забележка**: Codebuddy запазва специална обработка за GLM-5.0 и не използва наследена нормализация:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## Предимства за подобряване на GLM-5.1

В сравнение с GLM-5.0, GLM-5.1 носи следните значителни подобрения:

### 1. По-добра способност за разсъждение

Според официалната информация за изданието на Zhipu, подобренията в GLM-5.1 включват:

- **По-добро разбиране на кода**: По-точен анализ на сложни кодови структури
- **Разбиране на по-дълъг контекст**: Поддържа по-дълъг контекст на разговор
- **Подобрено извикване на инструмент**: По-висок процент на успех за извиквания на MCP инструмент
- **Стабилност на изхода**: Намалява произволността и халюцинациите

### 2. Изчерпателна мулти-CLI съвместимост

GLM-5.1 покрива всички масови CLI, поддържани от Hagicode:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Бележки

### 1. API ключ конфигурация

Уверете се, че `ANTHROPIC_AUTH_TOKEN` променливата на средата е зададена правилно. Това е необходимото удостоверение за всеки CLI за свързване с модела.

### 2. Наличност на модела

GLM-5.1 трябва да бъде активиран от съответния доставчик на модел:
- Платформата Zhipu AI ZAI го поддържа по подразбиране
- Alibaba Cloud DashScope може да изисква отделно приложение

### 3. OpenCode CLI формат

При използване на `provider/model` формат, уверете се, че идентификаторът на доставчика е правилен:
- Zhipu AI: `zai` или `zhipuai`
- Alibaba Cloud: `aliyun` или `dashscope`

### 4. Параметър за разсъждение

- `high` се препоръчва за най-добри резултати при генериране на код
- Gemini CLI не поддържа параметъра за мотивиране и автоматично ще игнорира тази конфигурация

## Заключение

Чрез обединен абстракционен слой, Hagicode позволява гъвкава интеграция между GLM-5.1 и множество CLI. Разработчиците могат да изберат CLI инструмента, който най-добре отговаря на техните предпочитания и сценарии на използване, след което да използват най-новия модел GLM-5.1 чрез проста конфигурация.

Като последната версия на модела на Zhipu, GLM-5.1 предлага ясни подобрения спрямо GLM-5.0:
- Независим идентификатор на версия без наследствена тежест
- По-силно разсъждение и разбиране на кода
- Широка мулти-CLI съвместимост
- Гъвкава конфигурация на ниво на разсъждение

С правилните променливи на средата и конфигурирано оборудване Hero, потребителите могат напълно да отключат силата на GLM-5.1 в различни CLI среди.
