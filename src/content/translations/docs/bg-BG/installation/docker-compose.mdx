## Предпоставки

Преди да започнете, се уверете, че вашата система има инсталирани Docker и Docker Compose.

### Инсталирайте Docker

#### Windows

Изтеглете и инсталирайте [Docker Desktop за Windows](https://www.docker.com/products/docker-desktop/)

След инсталирането се уверете, че Docker Desktop работи.

#### macOS

Изтеглете и инсталирайте [Docker Desktop за Mac](https://www.docker.com/products/docker-desktop/)

След инсталирането се уверете, че Docker Desktop работи.

#### Linux

Инсталирайте Docker с вашия мениджър на пакети:

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

След инсталирането стартирайте услугата Docker:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Проверете инсталацията

След инсталирането изпълнете следните команди, за да проверите дали Docker и Docker Compose са инсталирани правилно:

```bash
docker --version
docker compose version
```

1. Отворете [Docker Compose Builder (самостоятелен сайт)](https://builder.hagicode.com/)
2. Попълнете конфигурацията според вашите нужди:
   - Изберете доставчик на API (Zhipu AI, Anthropic Official или Custom API)

#### Избор на източник на изображение

Генераторът поддържа три опции за източник на изображение:

**Docker Hub**
- **Адрес на изображението**: `newbe36524/hagicode:{tag}`
- **Приложими сценарии**: Подходящо за потребители с огледално ускорение на Docker Hub
- **Предимства**: Официален източник на изображения, навременни актуализации, стабилен достъп
- **Бележки**: Може да се наложи ускоряването на огледалото да бъде конфигурирано в някои региони

**Регистър на контейнери на Azure (ACR)**
- **Адрес на изображението**: `hagicode.azurecr.io/hagicode:{tag}`
- **Приложими сценарии**: Подходящо за потребители, които нямат локален достъп до Docker Hub
- **Предимства**: Осигурява алтернативен източник на изображения, разрешава проблеми с достъпа до мрежата
- **Бележки**: Изображенията остават синхронизирани с Docker Hub, но може да има кратко забавяне

**Alibaba Cloud Container Registry (ACR)**
- **Адрес на изображението**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **Приложими сценарии**: Подходящ за домашни потребители, стабилен и бърз достъп до мрежата
- **Предимства**: вътрешни възли на Alibaba Cloud, бърза скорост на достъп, висока стабилност
- **Бележки**: ACR изображенията на Alibaba остават синхронизирани с Docker Hub, обикновено в рамките на 30 минути

:::tip[Препоръки за избор на източник на изображение]
- **Местни потребители препоръчват Alibaba Cloud ACR**: Ако сте местен потребител, препоръчително е да дадете приоритет на използването на източник на изображения на Alibaba Cloud ACR за бърз и стабилен достъп
- **Алтернативен Docker Hub**: Ако вашата мрежова среда има достъп до Docker Hub, можете също да изберете Docker Hub като източник на изображение
- **Алтернативен Azure ACR**: Изберете Azure ACR източник на изображения само ако други източници на изображения имат затруднения с достъпа
- **Синхронизиране на изображения**: Изображенията на Alibaba Cloud ACR остават синхронизирани с Docker Hub, обикновено в рамките на 30 минути
- **Постоянство на версиите**: Етикетите на изображения са последователни във всичките три източника на изображения, няма нужда да се притеснявате за проблеми с версиите
:::

### 2. Стартирайте Услуги

Изпълнете следната команда в директорията, където сте `docker-compose.yml` се намира:

```bash
docker compose up -d
```

Тази команда ще:
- Изтеглете и стартирайте контейнерите на Hagicode, дефинирани от генерирания шаблон
- Създайте и конфигурирайте мрежата
- Инициализирайте директорията с данни по подразбиране и зависимостите по време на изпълнение

### 3. Проверете статуса на услугата

Проверете дали контейнерите работят:

```bash
docker compose ps
```

Трябва да видите услугите, дефинирани във вашия генериран шаблон, в състояние "работещи".

### Преглед на регистрационните файлове на услугата

За да видите всички регистрационни файлове на услугата:

```bash
# View all service logs
docker compose logs

# View Hagicode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## Инструкции за конфигуриране

Ако трябва да промените конфигурацията (като промяна на портове, превключване на доставчик на API и т.н.), можете:

1. Регенерирайте нова конфигурация с помощта на [Docker Compose Builder (самостоятелен сайт)](https://builder.hagicode.com/)
2. Редактирайте ръчно `docker-compose.yml` и `.env` файлове
3. Рестартирайте услугите, за да влезе в сила конфигурацията:

```bash
docker compose restart
```

## Приложение за достъп

### Уеб интерфейс

След успешно стартиране на услугата, достъп през вашия браузър:

```
http://localhost:45000
```

### Устойчивост на данните

Поддържаният път на Docker Compose използва SQLite файлове с данни с постоянни томове по подразбиране. Ако трябва да проверите точките на монтиране или да планирате резервни копия, прегледайте съпоставянията на тома и директорията с данни в генерирания `docker-compose.yml`.
