Если вы хотите установить Hermes и начать его использовать, самый короткий путь — это всего три шага:

1. Выполните официальную команду установки
2. Запустите CLI в терминале с помощью `hermes`
3. Если вы хотите продолжить использовать его в Feishu, настройте `hermes gateway setup`

Эта статья не пытается объяснить все возможности Hermes за один раз. Вместо этого она помогает вам сначала завершить самый важный начальный цикл: установите, запустите, начните использовать, а затем подключите к одному из самых распространённых сценариев использования мессенджеров.

## Что такое Hermes и для кого он предназначен

Hermes Agent — это ИИ-агент, которым можно пользоваться как из локального терминала, так и через шлюз мессенджера.

Для большинства разработчиков есть два распространённых способа входа:

- **CLI**: Введите `hermes` в терминале, чтобы сразу перейти в интерактивный интерфейс.
- **Шлюз мессенджера**: Запустите `hermes gateway`, а затем общайтесь с ним из таких платформ, как Feishu, Telegram, Discord и Slack.

Если ваша текущая цель — просто быстро начать, не меняйте порядок. Начните с этого пути:

- Сначала установите Hermes
- Сначала убедитесь, что он работает через CLI
- Затем решите, хотите ли вы подключить платформу мессенджера

Это упрощает диагностику проблем и лучше подходит для тех, кто использует Hermes впервые.

## Что нужно знать перед установкой Hermes

Согласно README Hermes, официальный путь быстрой установки поддерживает следующие среды:

- Linux
- macOS
- WSL2
- Android через Termux

### Примечание для пользователей Windows

Hermes в настоящее время не поддерживает работу напрямую на нативной Windows. Если вы используете Windows, рекомендуется сначала установить WSL2, а затем выполнить команду установки внутри WSL2.

Лучше прояснить это в самом начале, потому что многие сбои при установке вызваны не самой командой, а использованием неподдерживаемой среды выполнения.

## Как установить Hermes

Команда быстрой установки из README Hermes:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Эта команда запускает официальный сценарий установки и обрабатывает специфичные для платформы шаги инициализации.

### Перезагрузите оболочку после установки

После завершения установки сначала перезагрузите среду оболочки. Самая распространённая команда:

```bash
source ~/.bashrc
```

Если вы используете zsh, можно использовать:

```bash
source ~/.zshrc
```

### Как подтвердить, что Hermes установлен корректно

Самый прямой способ проверить — запустить:

```bash
hermes
```

Если вы хотите дополнительного подтверждения, что конфигурация и зависимости работают, можно также запустить:

```bash
hermes doctor
```

`hermes doctor` особенно полезен в следующих ситуациях:

- Команда работает ненормально после установки
- Сбой конфигурации модели
- Шлюз не запускается
- Вы не уверены, полны ли зависимости вашей среды

## Как впервые начать использовать Hermes

### Начните с CLI

Если вы просто хотите максимально быстро убедиться, что Hermes работает, самый простой способ:

```bash
hermes
```

Это запускает интерактивный CLI Hermes. Для впервые использующих Hermes это также самая рекомендуемая отправная точка, потому что вы можете сначала проверить самое необходимое:

- Доступна ли команда
- Правильно ли работает текущая конфигурация модели
- Корректно ли работает цепочка инструментов терминала
- Подходит ли стиль взаимодействия под ваши потребности

### Этих команд достаточно для первого раунда настройки

В README Hermes перечислены несколько часто используемых команд, которые вместе образуют практичный путь первого использования:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Если вы не уверены, что делает каждая из них, запомните их так:

- `hermes model`: выбрать или переключить модель
- `hermes tools`: просмотр и настройка доступных инструментов
- `hermes config set`: изменить конкретные параметры конфигурации
- `hermes setup`: запустить полный мастер инициализации
- `hermes update`: обновить Hermes
- `hermes doctor`: устранение неполадок

Для новичков наиболее практичный порядок обычно такой:

1. Сначала запустите `hermes model`
2. Если вы хотите настроить все основные параметры за один раз, запустите `hermes setup`

## Два наиболее распространённых способа использования Hermes

### 1. Использовать Hermes в терминале как повседневного помощника по разработке

Режим CLI хорошо подходит для следующих сценариев:

- Задавать вопросы прямо во время локальной написания кода
- Осматривать проекты, редактировать файлы и выполнять команды
- Выполнять разовые задачи отладки или ревью
- Постоянно сотрудничать в текущей рабочей директории

Его главное преимущество — это самый короткий путь: не нужна дополнительная интеграция с платформой, не нужна предварительная настройка бота, и это лучший способ сформировать первые привычки использования.

### 2. Использовать Hermes через платформу мессенджера

Если вы хотите общаться с Hermes на таких платформах, как Feishu, Telegram или Discord, вам нужно использовать шлюз мессенджера.

Наиболее распространённые команды входа:

```bash
hermes gateway setup
hermes gateway
```

А именно:

- `hermes gateway setup` используется для интерактивной настройки платформы
- `hermes gateway` используется для запуска процесса шлюза

Согласно официальной документации, шлюз — это единый фоновый процесс, который подключает настроенные вами платформы, управляет сессиями и обрабатывает такие функции, как запланированные задачи.

## На примере Feishu: как подключить Hermes к платформе мессенджера

Если большая часть вашей повседневной работы происходит в Feishu, то Feishu/Lark — очень естественный способ использования Hermes.

### Минимальный путь интеграции

Официальная документация рекомендует следующую команду входа для Feishu/Lark:

```bash
hermes gateway setup
```

После выполнения просто выберите **Feishu / Lark** в мастере.

Документация Feishu описывает два режима подключения:

- `websocket`: рекомендуется
- `webhook`: необязательно

Если Hermes работает на вашем ноутбуке, рабочей станции или частном сервере, обычно проще сначала использовать `websocket`, потому что вам не нужно открывать публичный URL обратного вызова.

### Если вы настраиваете вручную, хотя бы узнайте эти переменные

Если вы не используете мастер и пишете конфигурацию вручную, документация Feishu перечисляет следующие основные переменные:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Две из них заслуживают особого внимания:

- `FEISHU_ALLOWED_USERS`: рекомендуется, чтобы не каждый, кто может достичь бота, мог использовать его напрямую
- `FEISHU_HOME_CHANNEL`: позволяет заранее определить домашний чат для получения результатов запланированных задач или уведомлений по умолчанию

### Почему Hermes иногда не отвечает в групповых чатах Feishu

Эта деталь легко упустить: в групповых чатах Feishu Hermes по умолчанию не отвечает на каждое сообщение.

Официальная документация чётко указывает:

- В личных сообщениях Hermes отвечает на сообщения
- В групповых чатах нужно явно упомянуть бота через `@`, прежде чем он обработает сообщение

Если вы хотите установить беседу Feishu как домашний канал, можно также использовать это в чате:

```text
/set-home
```

Или заранее определить в конфигурации:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Команды Hermes, которые новичкам стоит запомнить в первую очередь

Независимо от того, используете ли вы Hermes в CLI или на платформе мессенджера, запоминания следующих команд уже достаточно для начала:

- `/new` или `/reset`: начать новую сессию
- `/model`: просмотр или переключение модели
- `/retry`: повторить предыдущий ход
- `/undo`: отменить предыдущее взаимодействие
- `/compress`: вручную сжать контекст
- `/help`: просмотр справки

Если вы в основном используете Hermes на платформе мессенджера, запомните ещё одну:

- `/sethome` или `/set-home`: установить текущий чат как домашний канал

Эти команды охватывают наиболее распространённые операции на начальном этапе: перезапуск, настройку, откат, проверку и продолжение.

## Часто задаваемые вопросы

### Можно ли установить Hermes напрямую на Windows?

Нет. Текущая официальная документация чётко указывает, что нативная Windows не поддерживается, и рекомендуется WSL2.

### Что делать, если после установки ввод `hermes` ничего не даёт?

Лучше всего устранять неполадки в следующем порядке:

1. Сначала перезагрузите оболочку, например с помощью `source ~/.bashrc`
2. Снова запустите `hermes`
3. Если по-прежнему аномально, запустите `hermes doctor`

### Почему бот не отвечает в группе Feishu?

Сначала проверьте три вещи:

- Упомянули ли вы Hermes через `@` в группе
- Ограничивает ли `FEISHU_ALLOWED_USERS` текущего пользователя
- Разрешает ли текущая политика группового чата обработку групповых сообщений

Согласно официальной документации Feishu, в сценариях группового чата требуется явное упоминание через @.
