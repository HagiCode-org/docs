Для пользователей Windows Microsoft Store сейчас является публичной точкой входа для десктопного клиента. Эта страница описывает только путь через Store. За общим руководством по загрузке Desktop, управлению версиями и устранению неполадок перейдите к [Desktop Installation](/ru-RU/installation/desktop).

Основная запись приложения в Steam уже выведена из эксплуатации, поэтому главным Windows-каналом сейчас является Microsoft Store. Контекст можно посмотреть в [Why HagiCode is not available on Steam](/ru-RU/faq/steam-distribution-status/).

:::tip[Когда этот путь подходит]
- вы используете Windows 10 или Windows 11 и хотите установить приложение напрямую из системного магазина
- вы предпочитаете получать пакет приложения через Store, а не работать с установочными файлами вручную
- вам нужен только локальный персональный сценарий, без развертывания через Docker Compose
:::

## Что проверить заранее

- устройство должно открывать [страницу Microsoft Store для HagiCode for Windows](https://apps.microsoft.com/detail/9N3PM0N3SVDW)
- при первом запуске должен быть доступ к сети, потому что приложение может продолжить подготовку дополнительных компонентов
- на некоторых устройствах, в некоторых регионах или по политике организации может потребоваться вход в Microsoft account
- если доступ к Store на вашей машине заблокирован, используйте [Desktop Installation](/ru-RU/installation/desktop)

## Установка из Microsoft Store

1. Откройте [HagiCode for Windows](https://apps.microsoft.com/detail/9N3PM0N3SVDW).
2. Нажмите `Получить`, `Установить` или аналогичное действие, которое показывает Store.
3. Дождитесь завершения загрузки и установки в Microsoft Store.
4. После этого запустите HagiCode из меню «Пуск» или прямо со страницы Store.

## Что происходит при первом запуске

После установки HagiCode все равно продолжает стандартный поток инициализации Desktop.

- первый запуск может выглядеть медленнее обычного, потому что локальная среда выполнения еще подготавливается
- канал Microsoft Store / MSIX намеренно блокирует часть настроек управления зависимостями и путями runtime
- приложение не обновляется полностью самостоятельно; доставка пакета зависит от канала установки, а для Store-сборки она привязана к каналу Microsoft Store

Если вы хотите продолжить онбординг, откройте [Wizard Setup](/ru-RU/quick-start/wizard-setup).

## Когда лучше выбрать другой путь

- вы не можете использовать Microsoft Store, или в вашей организации запрещена установка через Store
- вам нужно вручную скачивать файлы `.exe` или `.msix` и самостоятельно контролировать источник установщика
- вы разворачиваете систему на сервере, NAS или в командной среде; тогда лучше подходит [Docker Compose Deployment](/ru-RU/installation/docker-compose)

## Устранение проблем

### Страница Microsoft Store не открывается

Сначала проверьте сеть, региональные настройки и состояние входа в Microsoft Store. Если ссылка в браузере открывается, а клиент Store все равно недоступен, частой причиной является локальная системная политика. В таком случае используйте [Desktop Installation](/ru-RU/installation/desktop).

### Почему некоторые настройки заблокированы

Установки Store / MSIX намеренно блокируют часть параметров управления зависимостями и путями данных runtime. Это не ошибка установки, а обычное ограничение данного канала распространения.

### Почему при первом запуске еще идут загрузки или подготовка

Пакет Store доставляет только пакет десктопного приложения. При первом запуске HagiCode все еще может проверять метаданные версии, готовить локальные компоненты или продолжать стандартную инициализацию. Оставьте сеть доступной и дождитесь завершения текущего шага.
