Сеансы разговоров — это самая легкая точка входа в HagiCode. Начните с диалога, разберитесь в репозитории и позже решите, необходимо ли редактирование. Текущий пользовательский интерфейс по умолчанию открывает новые диалоги в режиме только для чтения, поэтому вы можете проверить базу кода, прежде чем просить ИИ изменить файлы.

:::note[Общие скриншоты]
Текущий пакет скриншотов показывает китайский интерфейс. Руководство на английском языке намеренно повторно использует одни и те же управляемые снимки экрана и объясняет каждый элемент управления на английском языке рядом с изображением.
:::

## Предварительные условия

Прежде чем начать сеанс беседы, убедитесь, что у вас есть:

- завершено [Установка на рабочем столе](/ru-RU/installation/desktop)
- завершено [Мастер настройки](/ru-RU/quick-start/wizard-setup) и создал хотя бы один проект
- вошел в рабочую область HagiCode для этого проекта

## Для чего нужен сеанс беседы?

Сеанс беседы лучше всего работает, когда вы хотите сначала понять, а потом редактировать. Типичное использование включает в себя:

- быстрый анализ незнакомого репозитория
- переключение пресетов героев для тона, провайдера или контекста
- оставайтесь в режиме только для чтения, пока не убедитесь, что редактирование стоит делать
- прикрепление изображений, быстрых подсказок или контекста истории перед отправкой следующего сообщения

Если ваша задача уже перешла от «понять проблему» к «планировать, анализировать и архивировать структурированные изменения», продолжайте с [Создать сеанс предложений](/ru-RU/quick-start/proposal-session). На этой странице описаны только сеансы разговоров.

## Обзор рабочего процесса

Основной путь кратко описан ниже: создайте разговор из списка сеансов, при необходимости выберите предустановку героя, задайте первый вопрос в режиме только для чтения и переключитесь в режим редактирования только после того, как вы подтвердите, что необходимы прямые изменения файла.

| Этап | Действие пользователя | Системный результат |
| --- | --- | --- |
| Войти в сеанс | Создайте разговор из верхних элементов управления сеансом. | Сеанс открывается с записью героя по умолчанию. |
| Дополнительный выбор героя | Откройте панель «Пресеты» и выберите героя. | Сеанс использует эту роль, поставщика или контекст. |
| Вопрос только для чтения | Спросите что-нибудь вроде `分析这个项目` | ИИ читает репозиторий и возвращает анализ |
| Определитесь с исполнением | Оставайтесь в режиме только для чтения или включите режим редактирования. | Безопасный анализ продолжается или начинается редактируемое выполнение. |
| Добавьте больше контекста | Используйте изображения, быстрые подсказки или инструменты истории. | Следующее сообщение содержит более богатый контекст. |

Первый пример, доступный только для чтения, представляет собой лишь короткую последовательность взаимодействия. Правило простое: ИИ может проверять и объяснять, но не редактирует файлы, пока вы не переключите режимы.

1. **Введите запрос**: пользователь вводит `分析这个项目`
2. **Отправить в режиме только для чтения**: пользовательский интерфейс отправляет запрос с безопасными разрешениями по умолчанию.
3. **Чтение контекста репозитория**: ИИ проверяет папки, файлы конфигурации и ключевые файлы.
4. **Анализ доходности**: ИИ обобщает структуру, модули и риски.
5. **Выберите следующий шаг**: пользователь либо продолжает общение, либо включает режим редактирования.

## Шаг 1. Создайте беседу из списка сеансов.

Текущая запись обычного разговора находится в верхних элементах управления сеансом. Вы можете сразу начать с героя по умолчанию или открыть раскрывающийся список аватаров справа перед входом в сеанс.

![Текущая запись разговора в верхних элементах управления сеансом с состоянием героя по умолчанию.](../../img/screenshots/quick-start/create-conversation-session/01-create-with-default-hero/original.png)

> Рисунок: верхние элементы управления сеансом — это текущая точка входа в диалог; раскрывающийся список аватара открывает дополнительный предустановленный путь героя.

Начните с двух деталей:

- **Запись находится в верхних элементах управления** вместо старой `Add Chat` поток кнопок.
- **Выбор героя не является обязательным**. Если вы просто хотите начать анализировать репозиторий, продолжайте напрямую.

## Шаг 2. При желании выберите пресет героя.

Если вам нужен другой тон, поставщик модели или контекст предустановки, откройте панель «Наборы настроек» и выберите героя, прежде чем продолжить.

![Расширенная панель пресетов с несколькими пресетами героев, доступными для сеансов разговора.](../../img/screenshots/quick-start/create-conversation-session/02-create-with-selected-hero/original.png)

> Рисунок: на панели «Пресеты» перечислены многократно используемые пресеты героев; выбор одного из них не является обязательным и не меняет основной ход разговора.

Выбор героя в первую очередь полезен, когда:

- вы уже знаете, какой поставщик или стиль вам нужен
- вы сохраняете разные роли сеанса для разных задач
- вы хотите сравнить точки зрения нескольких героев на один и тот же проект

## Шаг 3. Проанализируйте проект в режиме только для чтения по умолчанию.

После открытия сеанса режим только для чтения является безопасным режимом по умолчанию. На скриншоте первая подсказка: `分析这个项目`, а ИИ в ответ читает репозиторий и подводит итоги проекта.

![Пример диалога только для чтения с использованием приглашения 分析这个项目 для проверки репозитория](../../img/screenshots/quick-start/create-conversation-session/03-chat-view-readonly/original.png)

> Рисунок: сеанс разговора начинается в режиме только для чтения, который идеально подходит для анализа репозитория, проверки структуры и сбора контекста.

В режиме только для чтения ИИ обычно может:

- читать файлы и структуру каталогов
- кратко изложить цель проекта, ключевые модули и зависимости
- объяснить, за что отвечает функция, файл или модуль

В режиме только для чтения ИИ не делает напрямую:

- изменить файлы репозитория
- выполнять изменения, ориентированные на редактирование, прежде чем переключать режимы

Если ваша текущая цель — понять кодовую базу, прояснить проблему или подготовиться к дальнейшей работе, режима только для чтения обычно достаточно.

## Шаг 4. Включите режим редактирования и изучите панель инструментов.

Когда вы будете готовы к прямым изменениям, переключите сеанс в режим редактирования. В текущем пользовательском интерфейсе переключатель режима и вспомогательные инструменты размещаются в строке панели инструментов над кнопкой отправки.

![Переключение режима редактирования с вставкой изображения, быстрой подсказкой и инструментами истории в текущем композиторе](../../img/screenshots/quick-start/create-conversation-session/04-edit-mode-controls/original.png)

> Рисунок: строка панели инструментов сначала включает переключатель режима, затем вставку изображения, быструю подсказку и точки входа в историю.

Каждый элемент управления имеет определенную цель:

- **Переключение режима**: переключает сеанс из режима «только для чтения» по умолчанию в режим редактирования, чтобы ИИ мог вносить реальные изменения.
- **Кнопка изображения**: прикрепляет контекст изображения перед отправкой следующего сообщения.
- **Кнопка быстрой подсказки**: вставляет в разговор многоразовые ярлыки подсказок.
- **Кнопка «История»**: открывает контекст предыдущего сеанса или соответствующую историю.

Более безопасный шаблон прост: сначала подтвердите направление в режиме только для чтения, а затем включите режим редактирования, когда вы действительно хотите выполнить.

## Шаг 5. Прикрепите изображение перед отправкой следующего сообщения.

Сеансы диалога поддерживают вложение изображений непосредственно внутри текущего композитора. Изображение добавляется в качестве дополнительного контекста над областью ввода, и тот же композитор по-прежнему сохраняет доступность переключателя режима и вспомогательных инструментов.

![Компоновщик диалогов с предварительным просмотром прикрепленного изображения и той же панелью инструментов в режиме редактирования, которая все еще видна.](../../img/screenshots/quick-start/create-conversation-session/05-insert-image/original.png)

> Рисунок: предварительный просмотр изображения сначала появляется внутри области композитора, и перед отправкой вы все равно можете использовать переключатели и вспомогательные инструменты.

Это полезно, если вы хотите:

- попросите ИИ объяснить скриншот интерфейса
- прикрепите скриншот ошибки или макет дизайна и добавьте последующие инструкции
- обогатите следующую подсказку визуальным контекстом перед переходом к выполнению

## Typical use cases and progression

Рекомендуемый порядок действий на пяти скриншотах следующий:

1. **Быстро войдите в сеанс** с помощью верхних элементов управления.
2. **Выбирайте героя только при необходимости** на панели «Наборы настроек».
3. **Начните с анализа, доступного только для чтения**, используя такие подсказки, как `分析这个项目`.
4. **Переходите в режим редактирования намеренно**, когда возникает необходимость прямого изменения файла.
5. **Добавьте больше контекста** с помощью изображений, быстрых подсказок и истории перед следующим сообщением.

Если задача теперь требует планирования, проверки и архивной дисциплины, перейдите к следующему руководству по сеансу предложений.
