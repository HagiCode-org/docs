## 보더 라이트 스윕 애니메이션 효과 만들기 가이드

> 순수 CSS로 사용자가 한눈에 알아볼 수 있는 중요한 요소를 어떻게 만들까요? 사실 그렇게 어렵지 않습니다. 요령은 약간 우회하는 경로를 사용하는 것뿐입니다. 이 글에서는 보더 라이트 스윕 애니메이션을 처음부터 만드는 방법을 단계별로 안내하고, HagiCode를 구축하면서 겪은 몇 가지 함정도 공유합니다.

## 배경

프론트엔드를 하신다면 아마 이런 경험이 있으실 것입니다. 기획자가 "이건 확실히 간단하겠지"라는 표정으로 다가와서 "이 실행 중인 작업에 어떤 특수 효과를 넣을 수 있을까요? 사용자가 즉시 알아볼 수 있게요."

네, 할 수 있습니다. 테두리 색상을 바꾸면 됩니다. 그러면 그들은 "이해가 안 되시는 것 같네요"라는 표정으로 고개를 저으며 말합니다. "그건 너무 눈에 띄지 않아요. 테두리를 따라 빛이 도는 효과, SF 영화에서처럼 말이에요."

그 순간 어떻게 만들지 고민하기 시작할 것입니다. Canvas? SVG? 아니면 CSS가 혼자서 처리할 수 있을까요? 아무도 모른다는 것을 인정하고 싶지 않으니까요.

현대 웹 애플리케이션에서 보더 라이트 스윕 애니메이션은 실제로 매우 일반적입니다. 주로 다음과 같은 시나리오에서 사용됩니다.

- **상태 표시기**: 진행 중인 작업이나 활성 항목 표시
- **시각적 포커스**: 중요한 콘텐츠 영역 강조
- **브랜드 향상**: 세련되고 현대적인 기술 중심의 시각 스타일 구현
- **시즌 테마**: 특별한 행사를 위한 축하 분위기 조성

우리는 [HagiCode](https://hagicode.com)를 구축하면서 정확히 이러한 요구 사항에 직면했습니다. 사용자가 어떤 세션이 실행 중이고 어떤 제안이 현재 처리 중인지 한눈에 볼 수 있어야 했습니다. 여러 가지 접근 방식을 시도했습니다. 일부 경로는 더 순탄했고, 일부는 약간 더曲折했습니다. 결국 비교적 성숙한 구현 전략에 정착했습니다.

## 핵심 구현 아이디어

HagiCode 코드베이스를 분석하여 몇 가지 핵심 구현 패턴을 요약했습니다. 각각이 다른 시나리오에 맞습니다. 다시 말해, 각각은 존재할 이유가 있습니다.

### 1. 원뿔형 그라디언트로 회전하는 글로우 (가장 일반적)

이것은 보더 라이트 스윕 효과를 구현하는 고전적인 방법입니다. 핵심 아이디어는 CSS `conic-gradient`를 사용하여 원뿔형 그라디언트를 만들고, 이를 연속적으로 회전시키는 것입니다. 밤에 도는 가로등처럼 계속 빙글빙글 도는 것입니다.

**핵심 요소:**
- `::before` 가상 요소를 사용하여 글로우 레이어 생성
- `conic-gradient`를 사용하여 그라디언트 색상 분포 정의
- `::after` 가상 요소를 사용하여 중앙 영역 마스킹 (선택 사항)
- `@keyframes`를 사용하여 회전 애니메이션 구현

### 2. 사이드 글로우 라인

이것은 목록 항목의 상태 표시기에 잘 맞습니다. 전체 테두리에 애니메이션을 적용하는 대신 요소의 한쪽에 얇은 글로우 라인을 만듭니다. 때로는 작은 빛 하나만으로 충분합니다. 세상을 모두 밝힐 필요는 없으니까요.

**핵심 요소:**
- 얇은 절대 위치 라인 요소
- `box-shadow`를 사용하여 글로우 효과 생성
- `scale`과 `opacity`로 호흡 애니메이션 구현

### 3. box-shadow로 글로우 배경

전체 스윕 효과가 필요하지 않고 부드러운 배경 글로우만 원한다면, 여러 `box-shadow` 값을 겹겹이 쌓는 것으로 충분합니다. 때로는 더 간단한 옵션이 더 나은 옵션입니다.

### 4. 접근성 지원

이 부분은 간과하기 쉽지만 극히 중요합니다. 모든 애니메이션은 `prefers-reduced-motion` 미디어 쿼리를 고려해야 하며, 애니메이션을 원하지 않는 사용자를 위한 정적 대안을 제공해야 합니다. 모든 사람이 끊임없는 움직임을 즐기는 것은 아니며, 그 선호를 존중하는 것이 중요합니다.

## 구현 옵션

### 옵션 1: 회전하는 원뿔형 그라디언트 보더 (권장)

이것은 가장 완전한 스윕 보더 라이트 효과 구현이며, HagiCode에서도 가장 자주 사용하는 옵션입니다. 잘 작동한다면 왜 바꾸겠습니까?

```css
/* 부모 컨테이너 */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* 회전하는 글로우 레이어 */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* 마스크 레이어 (선택 사항, 중공 보더 효과 생성용) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

이 옵션의 원리는 꽤 간단합니다. 부모 컨테이너보다 큰 가상 요소를 만들고, 그 위에 원뿔형 그라디언트를 그린 다음 연속적으로 회전시킵니다. 부모 컨테이너는 `overflow: hidden`을 사용하므로 보더를 따라 지나가는 빛만 보입니다. 창문을 통해 가로등을 보는 것과 비슷합니다. 지나가는 작은 조각만 볼 수 있습니다.

### 옵션 2: 간소화된 회전 라이트 보더

전체 효과가 필요하지 않다면, HagiCode에는 더 가벼운 유틸리티 클래스 버전도 있습니다. 때로는 더 간단한 접근 방식이 진정으로 더 좋습니다.

```css
/* 회전 라이트 보더 유틸리티 클래스 */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* 접근성 지원 */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

여기서 `will-change: transform`에 주목하세요. 이것은 브라우저에게 "이 요소는 계속 변경될 것입니다"라고 알려주어, 브라우저가 미리 일부 최적화를 준비하고 애니메이션을 더 부드럽게 유지할 수 있게 합니다. 미리 준비하는 것이 마지막 순간에 허둥대는 것보다 항상 낫습니다.

### 옵션 3: 사이드 글로우 라인

이것은 목록 항목 상태 표시기에 특히 적합하며, HagiCode 세션 목록이 정확히 이 방식을 사용합니다. 얇은 선 하나로도 많은 항목 사이에서 눈에 띌 수 있습니다. 그것은 삶의 교훈 같기도 합니다.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

이것은 `isolation: isolate`를 사용하여 새로운 스태킹 컨텍스트를 만든 다음, `z-index`를 사용하여 각 레이어의 표시 순서를 제어합니다. `pointer-events: none`도 필수적입니다. 그렇지 않으면 가상 요소가 사용자 클릭을 차단합니다. 보기 좋은 것도 있지만, 방해가 되어서는 안 됩니다.

### 옵션 4: React 컴포넌트로 래핑

프로젝트에서 React를 사용한다면, 특히 접근성 처리를 이 로직으로 래핑할 수 있습니다. 한 번 작성하고 여러 번 사용하세요. 그것이 핵심입니다.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

일치하는 CSS 모듈:

```css
/* GlowBorder.module.css */

/* 애니메이션 버전 */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* 정적 버전 (접근성) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

`framer-motion`의 `useReducedMotion` 훅은 사용자의 시스템 설정을 자동으로 감지합니다. 사용자가 모션 감소를 활성화한 경우 `true`를 반환하며, 컴포넌트는 정적 버전을 대신 표시합니다. 사용자의 선호를 존중하는 것이 화려한 효과를 강제하는 것보다 중요합니다.

## 실전에서 배운 교훈

이것은 HagiCode를 구축하면서 배운 몇 가지 교훈입니다. 전투의 흉터라고도 할 수 있습니다. 이것이 여러분이 몇 가지 우회로를 피하는 데 도움이 되기를 바랍니다.

### 1. 테마 변수 시스템

CSS 변수는 다중 테마 지원을 특히 편리하게 만듭니다. 테마가 바뀔 때마다 코드 더미를 편집하고 싶은 사람은 없으니까요.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* 사용법 */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

그러면 테마 전환은 `html` 요소의 클래스를 변경하는 것만으로 충분하며, 모든 애니메이션 색상이 자동으로 업데이트됩니다. 하나의 코드베이스, 두 가지 스타일. 바로 우리가 원하는 것입니다.

### 2. 성능 최적화

**`will-change`를 사용하여 브라우저에 최적화를 힌트:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

브라우저에 미리 알려주면 최적화를 도와줍니다. 삶의 많은 것들이 약간의 준비가 있으면 더 잘 작동합니다.

**큰 요소에 복잡한 box-shadow 사용 피하기:**

```css
/* 이상적이지 않음 - 큰 요소에 흐림 그림자 사용 */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* 더 나음 - 가상 요소를 사용하여 글로우 영역 제한 */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

HagiCode에서 이것을 테스트했습니다. 큰 카드에 흐림 그림자를 직접 추가하면 스크롤 프레임 속도가 30fps 이하로 떨어졌습니다. 가상 요소로 전환하자 다시 안정적인 60fps로 돌아왔습니다. 사용자는 확실히 그 차이를 느낄 수 있습니다.

### 3. 접근성

이것을 건너뛰지 마세요. 일부 사용자는 애니메이션이 어지럽거나 산만하다고 느끼며, 그 선호를 존중하는 것은 좋은 제품을 만드는 일부입니다. 아름다운 것은 모든 사람에게 강요될 필요가 없습니다.

**CSS 미디어 쿼리:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* 정적 대체 제공 */
    opacity: 1;
  }
}
```

**React에서 사용자 설정 감지:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. 강도 수준 제어

HagiCode의 Token 처리량 표시기는 실시간 처리량에 따라 다른 글로우 색상을 표시하며, 이것은 동적으로 구현됩니다. 다른 상태는 다르게 표현되어야 합니다.

```tsx
const colors = [
  null,       // 레벨 0 - 색상 없음
  '#3b82f6',  // 레벨 1 - 파란색
  '#34d399',  // 레벨 2 - 에메랄드
  '#facc15',  // 레벨 3 - 노란색
  '#fbbf24',  // 레벨 4 - 앰버
  '#f97316',  // 레벨 5 - 주황색
  '#22d3ee',  // 레벨 6 - 시안
  '#d946ef',  // 레벨 7 - 퓨샤
  '#f43f5e',  // 레벨 8 - 로즈
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. 주의할 점

여전히 주의를 기울일 가치가 있는 몇 가지 세부 사항이 있습니다. 왜냐하면 이러한 문제들을 어려운 방법으로 발견하면 이미 너무 늦기 때문입니다.

| 주의할 점 | 설명 |
|---------|------|
| **z-index 관리** | 글로우 레이어는 콘텐츠 상호 작용에 방해되지 않도록 적절한 `z-index`를 사용해야 합니다 |
| **pointer-events** | 글로우 가상 요소는 `pointer-events: none`을 설정해야 합니다 |
| **경계 오버플로우** | 부모 컨테이너에는 `overflow: hidden`이 필요하거나, 가상 요소 크기를 조정해야 합니다 |
| **성능 영향** | 복잡한 애니메이션은 모바일 기기에서 성능에 영향을 줄 수 있으므로 신중하게 테스트하세요 |
| **다크 모드** | 글로우 색상이 어두운 배경에서도 명확하게 보이는지 확인하세요 |
| **테마 전환** | CSS 변수를 사용하여 테마가 변경될 때 애니메이션 색상이 올바르게 업데이트되도록 하세요 |

### 6. 디버깅 팁

가상 요소는 개발자 도구에서 찾기가 약간 어려울 수 있으므로, 위치를 확인하기 위해 임시로 테두리를 추가할 수 있습니다.

```css
/* 디버깅을 위해 가상 요소 경계를 임시로 표시 */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

위치를 조정한 후에는 해당 줄을 주석 처리하거나 제거하는 것을 잊지 마세요. 그렇지 않으면 프로덕션이 빠르게 어색해질 수 있습니다. 일부 것들은 개발에만 남겨두는 것이 좋습니다.
