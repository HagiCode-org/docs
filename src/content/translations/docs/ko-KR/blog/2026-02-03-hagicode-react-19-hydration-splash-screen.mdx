## HagiCode를 위한 12가지 극한의 시작 경험 디자인: 미니멀에서 사이버펑크까지

> React 19 애플리케이션 다운로드와 Hydration의 짧은 간격은 사용자가 브랜드 개성을 느낄 수 있는 황금 창입니다. 이 글에서는 HagiCode 프로젝트에서 HTML/CSS/JS 기반으로 구축한 완전한 시작 스타일 시스템을 공유합니다.


## 배경

HagiCode는 ASP.NET Core 10과 React 19 (Vite) 기반의 현대 애플리케이션으로, 프론트엔드-백엔드 분리 배포 아키텍처를 채택합니다. 프론트엔드 산출물은 패키징되어 백엔드의 `wwwroot/` 디렉토리에 배치되며 ASP.NET Core에 의해 호스팅됩니다.

그러나 이 아키텍처는 전형적인 사용자 경험 문제를 가져왔습니다: 사용자가 웹페이지를 방문할 때 브라우저는 먼저 HTML을 로드하고, 거대한 JS Bundle을 다운로드하며, 마지막으로 React가 Hydration(수화)을 실행합니다. 이 수백 밀리초에서 수 초의 "진공기" 동안 사용자는 빈 화면이나 아무 생기 없는 정적 페이지를 마주하게 됩니다.

이 간격을 채우고 HagiCode의 브랜드 개성을 주입하기 위해 `index.html` 인라인 코드에 기반한 완전한 시작 스타일 시스템을 설계해야 합니다.

## HagiCode에 대하여

이 글에서 공유하는 시작 페이지 디자인 솔루션은 [HagiCode](https://github.com/HagiCode-org/site) 프로젝트의 실천 경험에서 왔습니다. AI 코드 어시스턴트인 HagiCode는 코드 생성 효율뿐만 아니라 개발자의 시각적 경험도 중요하게 생각합니다. 이 시작 시스템은 극한의 프론트엔드 성능을 추구하는 과정의 산물입니다.

## 핵심 도전과 아키텍처 설계

설계에 앞서 기술적 제약을 명확히 해야 합니다. `index.html`에 인라인으로 구현해야 하므로 외부 CSS나 JS 파일(React 자체의 Bundle 제외)을 로드할 수 없습니다.

### 기술 제약 분석

1.  **제로 종속성 원칙**: 모든 스타일은 `<style>` 태그 내에, 로직은 `<script>` 태그 내에 작성해야 합니다.
2.  **방어적 CSS**: React 애플리케이션이 마운트된 후 글로벌 스타일이 시작 페이지를 오염시키는 것을 방지하기 위해 높은 우선순위의 ID 접두사(예: `#boot-screen`)로 모든 시작 스타일을 감싸기로 했습니다.
3.  **성능 우선**: 애니메이션은 가능한 CSS `transform`과 `opacity`를 사용하여 리플로우를 트리거하지 않고 메인 스레드를 차단하지 않도록 합니다.
4.  **시각적 일관성**: 색상, 폰트는 HagiCode의 Tailwind 구성과 일치해야 합니다.

### 아키텍처 패턴: Shell & Injector

**변이 패턴**을 채택했습니다. 핵심 로직은 즉시 실행 함수(IIFE)에 캡슐화되고, 구체적인 렌더링 로직은 구성 항목으로 주입됩니다. 이렇게 하면 DOM 조작 로직을 반복 작성하지 않고 간단한 구성으로 다른 스타일을 전환할 수 있습니다.

다음은 핵심 아키텍처 코드입니다:

```html
<!-- 内联于 index.html -->
<div id="boot-root"></div>

<script>
(function() {
  const BootSequence = {
    config: {
      theme: 'terminal', // 可配置为 'minimal', 'skeleton', 'code-rain' 等
      color: '#3b82f6'   // 品牌色
    },
    
    // 核心生命周期
    init() {
      this.render();
      this.listenForMount();
    },

    // 渲染当前选定的风格
    render() {
      const root = document.getElementById('boot-root');
      if (this.variants[this.config.theme]) {
        root.innerHTML = this.variants[this.config.theme].render();
      }
    },

    // 监听 React 挂载成功，优雅退出
    listenForMount() {
      window.addEventListener('hagicode:ready', () => {
        const screen = document.getElementById('boot-root');
        // 先淡出，再移除 DOM，避免闪烁
        screen.style.opacity = '0';
        screen.style.transition = 'opacity 0.3s ease';
        setTimeout(() => screen.remove(), 300);
      });
    },

    // 12种风格的实现逻辑集中在这里
    variants: {
      // ...具体实现见下文
    }
  };

  BootSequence.init();
})();
</script>
```

## 12가지 시작 스타일 디자인 목록

12가지 스타일을 6개 카테고리로 분류하여 다양한 시나리오와 미적 요구를 충족합니다.

### A. 미니멀리즘

> "적을수록 많다." 극한의 로딩 속도를 추구하는 시나리오에 가장 가벼운 솔루션을 제공합니다.

#### 1. Minimalist Dot (미니멀 호흡)
화면 중앙에 단순한 점 하나만 있고 호흡 애니메이션과 함께합니다.
*   **구현**: CSS `@keyframes`로 scale과 opacity를 제어합니다.
*   **적용**: 페이지를 절대적으로 깨끗하게 유지해야 하는 모든 상황.

#### 2. Brand Reveal (브랜드 리빌)
SVG `stroke-dasharray` 애니메이션을 통해 수작업처럼 HagiCode 로고 라인을 그린 다음 텍스트를 페이드인합니다.
*   **기법**: SVG 경로 애니메이션 사용, 매우 질감이 뛰어납니다.

### B. 스켈레톤 스크린 모방

> "눈을 속이는 예술." 실제 UI 레이아웃을 시뮬레이션하여 사용자가 페이지가 절반쯤 로드된 것처럼 느끼게 합니다.

#### 3. Sidebar Chat Skeleton (사이드바 스켈레톤 스크린)
가장 실용적인 것일 수 있습니다. React 컴포넌트 `Sidebar` 및 `ChatInput`과 완전히 동일한 레이아웃을 HTML로 수동 구축하고 회색 줄무늬 애니메이션을 덧씌웁니다.
*   **가치**: React hydrate가 완료되면 스켈레톤 스크린이 순간적으로 실제 컴포넌트로 변하여 사용자는 전환을 거의 느끼지 못합니다.

#### 4. Card Stack Skeleton (카드 스택)
제안 카드 로딩 시의 스택 애니메이션을 시뮬레이션하고 3D 변환으로 카드를 살짝 떠 있게 합니다.

### C. 추상과 예술

> HagiCode의 지크 유전자를 보여줍니다.

#### 5. Geometric Morph (기하학적 변형)
화면 중앙에 기하학적 도형(정사각형)을 렌더링하고 시간에 따라 부드럽게 원형, 삼각형으로 변환한 다음 마지막에 로고가 됩니다.
*   **기술**: CSS `border-radius`의 부드러운 전환.

#### 6. Code Rain (코드 비)
"매트릭스"에 오마주. JetBrains Mono 폰트를 사용하여 배경에 옅은 문자 흐름을 떨어뜨립니다.
*   **주의**: 성능을 위해 문자 흐름은 작은 영역으로 제한하거나 새로 고침 빈도를 낮춰야 합니다.

#### 7. Neon Pulse (네온 펄스)
사이버펑크 스타일의 발광 링으로 `box-shadow`의 다중 겹침으로 강렬한 발광감을 만듭니다.

### D. 브랜드와 테마

> 시스템을 "살아 있게" 만듭니다.

#### 8. Seasonal Theme (명절 테마)
동적 로더입니다. 현재 날짜로 명절(예: 춘절, 크리스마스)을 판단하여 해당 SVG 애니메이션을 로드합니다.
*   **예**: 춘절에는 화면 하단에 붉은 등불이 가볍게 흔들립니다.

#### 9. Gradient Flow (그라디언트 흐름)
배경에 HagiCode 브랜드 컬러의 유동 그라디언트를 사용하고 `background-size`와 `background-position` 애니메이션과 함께 오로라 같은 흐름감을 연출합니다.

### E. 기술 감성

> 개발자에게 경의를 표합니다.

#### 10. Terminal Boot (터미널 부트)
콘솔 출력을 시뮬레이션합니다. 코드가 한 줄씩 빠르게 스크롤됩니다:
```text
> Initializing HagiCode Core...
> Loading models...
> Connecting to neural network...
```
모든 개발자가 친숙하게 느낄 것입니다.

#### 11. Progress Bar (미니멀 진행 바)
화면 상단에 가는 진행 바가 있고 오른쪽에 백분율이 표시됩니다. 실제 다운로드 진행 상황을 가져올 수는 없지만 타이머로 "신뢰할 수 있는" 로딩 프로세스를 시뮬레이션할 수 있습니다(처음 80%는 빠르고, 나머지 20%는 감속).

### F. 크리에이티브

#### 12. Pixel Assembly (픽셀 조립)
매우 흥미로운 아이디어입니다. 화면에 흩어진 블록들이 중앙으로 모여 HagiCode 로고 아이콘을 점차 조립합니다. 코드의 빌드 과정을 상징합니다.

## 모범 사례와 문제 해결 요약

HagiCode의 실제 개발에서 매우 중요한 실천 세부 사항을 요약했습니다.

### 1. 방어적 CSS는 필수
접두사를 생략하지 마세요. 한 번 시작 페이지 스타일에 ID 제한을 추가하지 않아 React 마운트 후 글로벌 `div` 스타일이 시작 페이지에 영향을 주어 레이아웃이 무너진 적이 있습니다.
**경험**: 모든 CSS 선택자를 `#boot-screen` 아래에 걸고 `!important`로 우선순위를 높입니다(시작 페이지 CSS에서만).

### 2. 우아한 전환
React 마운트 성공 후 시작 페이지 DOM을 직접 `remove()`하지 마세요.
**올바른 방법**:
1.  React가 `window.dispatchEvent(new Event('hagicode:ready'))`를 트리거합니다.
2.  시작 페이지가 이벤트를 감지하고 먼저 `opacity: 0`으로 설정합니다.
3.  300ms(CSS transition 시간)를 기다려 사용자가 보이지 않는 것을 확인한 후 `.remove()`를 실행합니다.

### 3. 테마 변수 동기화
시작 페이지의 색상 코드는 `index.html`에 하드코딩됩니다. Tailwind의 기본 색상을 변경하면 여기도 동기화해야 합니다.
**최적화 솔루션**: Vite 빌드 스크립트에서 간단한 플러그인을 작성하여 `tailwind.config.js`를 읽고 색상 변수를 `index.html`의 템플릿 변수에 주입하여 단일 데이터 소스를 구현합니다.

### 4. 폰트 프리로드
시작 페이지는 일반적으로 브랜드 폰트를 사용하지만 폰트 로딩이 느리면 FOUT(Flash of Unstyled Text)가 발생합니다.
**해결 방법**: `<head>`에 `<link rel="preload" href="/fonts/JetBrainsMono.woff2" as="font" type="font/woff2" crossorigin>`을 추가합니다. 경험 향상의 저비용 고효율 수단입니다.

### 5. 성능 모니터링
`index.html` 하단에 `performance.mark('boot-start')`를 주입하고 React 마운트 성공 시 `boot-end`를 마킹했습니다.
**의미**: Application Insights를 통해 이 데이터를 수집하면 시작 페이지가 사용자의 인지 대기 시간(Perceived Loading Time)을 얼마나 단축시키는지 실제로 볼 수 있습니다. 데이터에 따르면 훌륭한 스켈레톤 스크린은 사용자의 "느린 네트워크"에 대한容忍度를 50% 이상 향상시킵니다.

## 요약

좋은 시작 페이지는 단순한 "대기 중 장식"이 아니라 제품과 사용자의 첫 번째 상호작용의 악수 신호입니다. HagiCode 프로젝트에서 **Variants 패턴** 기반의 이 시작 시스템은 다양한 명절과 버전 간에 스타일을 유연하게 전환할 수 있게 하여 제품의 재미와 전문성을 크게 높였습니다.

이 글에서 공유한 솔루션은 완전히 네이티브 웹 표준 기반이며 무거운 종속성을 도입하지 않았습니다. 이것이 HagiCode가 추구하는 "가볍고 강력함"의 구현입니다. 이 솔루션이 가치 있다고 생각되시면 HagiCode 저장소에서 소스코드 구현을 살펴보거나 크리에이티브 디자인을 기여해 주세요!
