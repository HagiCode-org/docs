## HagiCode 다중 AI 제공자 전환 및 상호운용성 구현 계획

현대 개발자 도구 생태계에서 개발자들은 업무를 지원하기 위해 다양한 AI 코딩 어시스턴트를 사용해야 하는 경우가 많습니다. Anthropic의 Claude Code CLI와 OpenAI의 Codex CLI는 각각 고유한 장점을 가지고 있습니다. Claude는 뛰어난 코드 이해력과 긴 컨텍스트 처리 능력으로 알려져 있으며, Codex는 코드 생성과 도구 사용에 탁월합니다.

이 글에서는 HagiCode 프로젝트가 여러 AI 제공자 간의 원활한 전환과 상호운용성을 어떻게 달성했는지, 핵심 아키텍처 설계, 주요 구현 세부 사항 및 실제 고려 사항을 중심으로 심층적으로 살펴봅니다.

## 배경

### 문제 영역

HagiCode 프로젝트가 직면한 핵심 과제는 동일한 플랫폼에서 여러 AI CLI를 지원하는 것으로, 사용자가 다음을 수행할 수 있도록 하는 것입니다.

1. 필요에 따라 AI 제공자 간에 유연하게 전환
2. 제공자 전환 중 세션 연속성 유지
3. 다양한 CLI 간의 API 차이를 공통 추상화로 통합
4. 향후 새로운 AI 제공자 추가를 위한 확장 포인트 예약

### 기술적 과제

1. **인터페이스 차이 통합**: Claude Code CLI는 명령어 호출을 통해 실행되며, Codex CLI는 JSON 이벤트 스트림을 사용합니다.
2. **스트리밍 응답 처리**: 두 제공자 모두 스트리밍 응답을 지원하지만 데이터 형식이 다릅니다.
3. **도구 호출 의미론**: Claude와 Codex는 도구 호출을 표현하고 수명 주기를 관리하는 방식이 다릅니다.
4. **세션 수명 주기**: 시스템은 각 제공자에 대해 세션 생성, 복원 및 종료를 올바르게 관리해야 합니다.

## 분석

### 아키텍처 설계 접근 방식

HagiCode는 **Provider 패턴**과 **Factory 패턴**을 결합하여 AI 서비스 호출을 추상화합니다. 이 설계의 핵심 아이디어는 다음과 같습니다.

1. **통일된 인터페이스 추상화**: 모든 AI 제공자의 공통 추상화로 `IAIProvider` 인터페이스 정의
2. **팩토리 기반 인스턴스 생성**: `AIProviderFactory`를 사용하여 유형에 따라 해당 제공자 인스턴스를 동적으로 생성
3. **지능적 선택 로직**: `AIProviderSelector`를 사용하여 시나리오와 구성에 따라 가장 적합한 제공자를 자동으로 선택
4. **세션 상태 관리**: 세션과 CLI 스레드 간의 바인딩 관계를 데이터베이스에 지속

### 핵심 구성 요소

| 구성 요소 | 책임 | 언어 |
|------|--------|------|
| `IAIProvider` | 통일된 제공자 인터페이스 | C# |
| `AIProviderFactory` | 제공자 인스턴스 생성 및 관리 | C# |
| `AIProviderSelector` | 지능적 제공자 선택 | C# |
| `ClaudeCodeCliProvider` | Claude Code CLI 구현 | C# |
| `CodexCliProvider` | Codex CLI 구현 | C# |
| `AgentCliManager` | 데스크톱 측 CLI 관리 | TypeScript |

## 해결 방안

### 1. 핵심 인터페이스 설계

**`IAIProvider` 인터페이스**는 통일된 제공자 추상화를 정의합니다:

    /// </summary>

    /// Execute a single AI request
    /// </summary>

### 2. 팩토리 패턴 구현

```csharp
public class AIProviderFactory : IAIProviderFactory
{
    private readonly IServiceProvider _serviceProvider;
    private readonly ILogger<AIProviderFactory> _logger;

    public AIProviderFactory(IServiceProvider serviceProvider, ILogger<AIProviderFactory> logger)
    {
        _serviceProvider = serviceProvider;
        _logger = logger;
    }

    public async Task<IAIProvider?> GetProviderAsync(AIProviderType providerType)
    {
        return providerType switch
        {
            AIProviderType.ClaudeCodeCli => _serviceProvider.GetRequiredService<ClaudeCodeCliProvider>(),
            AIProviderType.CodexCli => _serviceProvider.GetRequiredService<CodexCliProvider>(),
            _ => throw new NotSupportedException($"Provider type {providerType} is not supported")
        };
    }
}
```

### 3. 지능적 제공자 선택기

```csharp
public class AIProviderSelector : IAIProviderSelector
{
    private readonly IAIProviderFactory _providerFactory;
    private readonly AIProviderConfiguration _configuration;
    private readonly IMemoryCache _cache;

    public async Task<IAIProvider?> SelectProviderAsync(
        BusinessScenario scenario,
        CancellationToken cancellationToken = default)
    {
        // 1. 시나리오 매핑 확인
        if (_configuration.ScenarioProviderMapping.TryGetValue(scenario.ToString(), out var mappedProvider))
        {
            var provider = await _providerFactory.GetProviderAsync(mappedProvider);
            if (provider != null) return provider;
        }

        // 2. 기본 제공자 사용
        var defaultProvider = await _providerFactory.GetProviderAsync(_configuration.DefaultProvider);
        if (defaultProvider != null) return defaultProvider;

        // 3. 폴백 체인 시도
        foreach (var fallbackType in _configuration.FallbackChain)
        {
            var fallbackProvider = await _providerFactory.GetProviderAsync(fallbackType);
            if (fallbackProvider != null) return fallbackProvider;
        }

        return null;
    }

    public async Task<bool> IsProviderAvailableAsync(
        AIProviderType providerType,
        CancellationToken cancellationToken = default)
    {
        var cacheKey = $"provider_available_{providerType}";

        // Ping 호출을 줄이기 위해 캐시 사용
        if (_configuration.EnableCache &&
            _cache.TryGetValue<bool>(cacheKey, out var cached))
        {
            return cached;
        }

        var provider = await _providerFactory.GetProviderAsync(providerType);
        var isAvailable = provider != null;

        if (_configuration.EnableCache && isAvailable)
        {
            _cache.Set(cacheKey, isAvailable,
                TimeSpan.FromSeconds(_configuration.CacheExpirationSeconds));
        }

        return isAvailable;
    }
}
```

선택기 전략:
- **시나리오 매핑 우선**: 먼저 비즈니스 시나리오에 특정 제공자 매핑이 있는지 확인
- **기본 제공자로 폴백**: 시나리오 매핑이 실패하면 기본 제공자 사용
- **폴백 체인을 최종 안전망으로**: 폴백 체인의 제공자를 순차적으로 시도
- **가용성 캐싱**: 제공자 가용성 확인을 캐시하여 Ping 호출 감소

### 4. Claude Code CLI 제공자 구현

```csharp
public class ClaudeCodeCliProvider : IAIProvider
{
    private readonly ILogger<ClaudeCodeCliProvider> _logger;
    private readonly IClaudeStreamManager _streamManager;
    private readonly ProviderConfiguration _config;

    public string Name => "ClaudeCodeCli";
    public bool SupportsStreaming => true;

    public ProviderCapabilities Capabilities { get; }

    public async Task<AIResponse> ExecuteAsync(AIRequest request, CancellationToken cancellationToken = default)
    {
        _logger.LogInformation("Executing AI request with provider: {Provider}", Name);

        // Claude CLI 프로세스 실행
        var process = new Process
        {
            StartInfo = new ProcessStartInfo
            {
                FileName = "claude",
                Arguments = $"--print \"{request.Prompt}\"",
                RedirectStandardOutput = true,
                RedirectStandardError = true,
                UseShellExecute = false
            }
        };

        process.Start();
        var output = await process.StandardOutput.ReadToEndAsync();
        await process.WaitForExitAsync(cancellationToken);

        return new AIResponse
        {
            Content = output,
            Provider = Name,
            Success = process.ExitCode == 0
        };
    }
}
```

### 5. Codex CLI 제공자 구현

```csharp
public class CodexCliProvider : IAIProvider
{
    private readonly ILogger<CodexCliProvider> _logger;
    private readonly ICodexStreamManager _streamManager;
    private readonly ProviderConfiguration _config;
    private readonly ISessionThreadBindingService _sessionThreadBindingService;

    public string Name => "CodexCli";
    public bool SupportsStreaming => true;

    public async Task<AIResponse> ExecuteAsync(AIRequest request, CancellationToken cancellationToken = default)
    {
        _logger.LogInformation("Executing AI request with provider: {Provider}", Name);

        var threadId = await GetOrCreateThreadAsync(request.SessionId);

        // Codex CLI 프로세스 실행
        var process = new Process
        {
            StartInfo = new ProcessStartInfo
            {
                FileName = _config.ExecutablePath ?? "codex",
                Arguments = $"--thread-id {threadId} \"{request.Prompt}\"",
                RedirectStandardOutput = true,
                RedirectStandardError = true,
                UseShellExecute = false
            }
        };

        process.Start();
        var output = await process.StandardOutput.ReadToEndAsync();
        await process.WaitForExitAsync(cancellationToken);

        return new AIResponse
        {
            Content = output,
            Provider = Name,
            Success = process.ExitCode == 0,
            Metadata = new Dictionary<string, string>
            {
                ["ThreadId"] = threadId
            }
        };
    }

    private async Task<string> GetOrCreateThreadAsync(string sessionId)
    {
        // 세션-스레드 바인딩 확인
        var existingThreadId = await _sessionThreadBindingService.GetThreadIdAsync(sessionId);
        if (existingThreadId != null)
        {
            return existingThreadId;
        }

        // 새 스레드 생성
        var newThreadId = Guid.NewGuid().ToString();
        await _sessionThreadBindingService.BindAsync(sessionId, newThreadId);
        return newThreadId;
    }
}
```

### 사용 예시 - 백엔드 서비스

```csharp
public class AIOrchestrator
{
    private readonly IAIProviderFactory _providerFactory;
    private readonly IAIProviderSelector _providerSelector;
    private readonly ILogger<AIOrchestrator> _logger;

    public AIOrchestrator(
        IAIProviderFactory providerFactory,
        IAIProviderSelector providerSelector,
        ILogger<AIOrchestrator> logger)
    {
        _providerFactory = providerFactory;
        _providerSelector = providerSelector;
        _logger = logger;
    }

    public async Task<AIResponse> ProcessRequestAsync(
        AIRequest request,
        BusinessScenario scenario)
    {
        var provider = await _providerSelector.SelectProviderAsync(scenario);
        if (provider == null)
        {
            throw new InvalidOperationException($"No available provider for scenario: {scenario}");
        }

        _logger.LogInformation("Selected provider {Provider} for scenario {Scenario}", provider.Name, scenario);
        return await provider.ExecuteAsync(request);
    }
}
```

### 사용 예시 - 프론트엔드

```typescript
// AgentCliManager - 데스크톱 측 CLI 관리
export class AgentCliManager {
  private providers: Map<string, AgentCliProvider> = new Map();
  private activeProvider: string = 'claude';

  async switchProvider(providerType: string): Promise<void> {
    const provider = this.providers.get(providerType);
    if (!provider) {
      throw new Error(`Provider not found: ${providerType}`);
    }

    // 활성 제공자 전환
    await this.stopActiveProvider();
    this.activeProvider = providerType;
    await provider.start();

    // 전환 이벤트 발송
    this.emit('providerSwitched', { from: this.activeProvider, to: providerType });
  }

  async executeRequest(request: AIRequest): Promise<AIResponse> {
    const provider = this.providers.get(this.activeProvider);
    if (!provider) {
      throw new Error('No active provider');
    }

    return provider.execute(request);
  }
}
```

### 3. 오류 처리

제공자를 사용할 수 없을 때 폴백 메커니즘이 강건해야 하며, 최소한 하나의 제공자는 사용 가능하도록 보장해야 합니다:

```csharp
public async Task<AIResponse> ExecuteWithFallbackAsync(
    AIRequest request,
    List<AIProviderType> preferredProviders)
{
    Exception? lastException = null;

    foreach (var providerType in preferredProviders)
    {
        try
        {
            var provider = await _providerFactory.GetProviderAsync(providerType);
            if (provider == null) continue;

            // 실행 시도
            return await provider.ExecuteAsync(request);
        }
        catch (Exception ex)
        {
            _logger.LogWarning(ex, "Provider {ProviderType} failed, trying next", providerType);
            lastException = ex;
        }
    }

    // 모든 제공자 실패
    throw new InvalidOperationException(
        "All preferred providers failed. Last error: " + lastException?.Message,
        lastException);
}
```

### 4. 구성 검증

런타임 오류를 방지하기 위해 시작 시 모든 구성된 제공자의 설정을 검증합니다:

```csharp
public void ValidateConfiguration(AIProviderOptions options)
{
    foreach (var (providerType, config) in options.Providers)
    {
        // 실행 파일 경로 검증 (CLI 기반 제공자의 경우)
        if (IsCliBasedProvider(providerType))
        {
            if (string.IsNullOrWhiteSpace(config.ExecutablePath))
            {
                throw new ConfigurationException(
                    $"Provider {providerType} requires ExecutablePath");
            }

            if (!File.Exists(config.ExecutablePath))
            {
                throw new ConfigurationException(
                    $"Executable not found for {providerType}: {config.ExecutablePath}");
            }
        }

        // API 키 검증 (API 기반 제공자의 경우)
        if (IsApiBasedProvider(providerType))
        {
            if (string.IsNullOrWhiteSpace(config.ApiKey))
            {
                throw new ConfigurationException(
                    $"Provider {providerType} requires ApiKey");
            }
        }

        // 모델 이름 검증
        if (string.IsNullOrWhiteSpace(config.Model))
        {
            _logger.LogWarning("No model configured for {ProviderType}, using default", providerType);
        }
    }
}
```

### 5. 캐시 관리

제공자 인스턴스가 캐시되므로 수명 주기 관리와 메모리 사용에 주의해야 합니다:

```csharp
// 비활성 제공자 주기적 정리
public void ClearInactiveProviders(TimeSpan inactiveThreshold)
{
    var now = DateTimeOffset.UtcNow;
    var keysToRemove = new List<AIProviderType>();

    foreach (var (type, instance) in _cache)
    {
        // 마지막 사용 시간 기준으로 비활성 여부 판단
        if (instance.LastUsedTime.HasValue &&
            now - instance.LastUsedTime.Value > inactiveThreshold)
        {
            keysToRemove.Add(type);
        }
    }

    foreach (var key in keysToRemove)
    {
        _cache.TryRemove(key, out _);
        _logger.LogInformation("Cleared inactive provider: {Provider}", key);
    }
}
```

### 7. 스레드 안전성

`ConcurrentDictionary`와 같은 동시성 컬렉션을 사용하여 스레드 안전성을 보장합니다:

```csharp
public class ThreadSafeProviderCache
{
    private readonly ConcurrentDictionary<AIProviderType, IAIProvider> _cache;
    private readonly ReaderWriterLockSlim _lock = new();

    public IAIProvider? GetProvider(AIProviderType type)
    {
        // 읽기 작업은 잠금이 필요하지 않음
        if (_cache.TryGetValue(type, out var provider))
            return provider;

        // 생성에는 쓰기 잠금이 필요
        _lock.EnterWriteLock();
        try
        {
            // 이중 확인
            if (_cache.TryGetValue(type, out provider))
                return provider;

            var newProvider = CreateProvider(type);
            if (newProvider != null)
            {
                _cache[type] = newProvider;
            }
            return newProvider;
        }
        finally
        {
            _lock.ExitWriteLock();
        }
    }
}
```

### 8. 데이터베이스 마이그레이션

세션-스레드 바인딩 데이터베이스 스키마가 변경될 때 데이터 마이그레이션을 고려해야 합니다:

```csharp
public class SessionThreadMigration
{
    public async Task MigrateAsync(string dbPath)
    {
        var version = await GetSchemaVersionAsync(dbPath);

        if (version >= 2) return; // 이미 최신 버전

        using var connection = new SqliteConnection(dbPath);
        connection.Open();

        // v2로 마이그레이션: CreatedAtUtc 열 추가
        if (version < 2)
        {
            _logger.LogInformation("Migrating SessionThreadBindings to v2...");

            using var addColumnCommand = connection.CreateCommand();
            addColumnCommand.CommandText = "ALTER TABLE SessionThreadBindings ADD COLUMN CreatedAtUtc TEXT;";
            addColumnCommand.ExecuteNonQuery();

            using var backfillCommand = connection.CreateCommand();
            backfillCommand.CommandText =
                """
                UPDATE SessionThreadBindings
                SET CreatedAtUtc = COALESCE(NULLIF(UpdatedAtUtc, ''), $nowUtc)
                WHERE CreatedAtUtc IS NULL OR CreatedAtUtc = '';
                """;
            backfillCommand.Parameters.AddWithValue("$nowUtc", DateTimeOffset.UtcNow.ToString("O"));
            backfillCommand.ExecuteNonQuery();
        }

        await UpdateSchemaVersionAsync(dbPath, 2);
        _logger.LogInformation("Migration to v2 completed");
    }
}
```

## 결론

HagiCode는 provider 패턴, factory 패턴, selector 패턴을 결합하여 유연하고 확장 가능한 다중 AI 제공자 아키텍처를 구현합니다:

- **통일된 인터페이스 추상화**: `IAIProvider` 인터페이스가 CLI 간의 차이를 숨깁니다.
- **동적 인스턴스 생성**: `AIProviderFactory`가 런타임에 제공자 인스턴스를 생성합니다.
- **지능적 선택 전략**: `AIProviderSelector`가 시나리오 기반 제공자 선택을 구현합니다.
- **세션 상태 지속성**: 데이터베이스 바인딩이 세션 연속성을 보장합니다.
- **데스크톱 통합**: `AgentCliManager`가 사용자 선택과 구성을 지원합니다.

이 아키텍처의 장점은 다음과 같습니다:
1. **확장성**: 새로운 AI 제공자를 추가하려면 `IAIProvider` 인터페이스를 구현하기만 하면 됩니다.
2. **테스트 가능성**: 제공자를 독립적으로 테스트하고 모킹할 수 있습니다.
3. **유지보수성**: 각 제공자 구현이 분리되어 단일 책임을 갖습니다.
4. **사용자 친화성**: 시나리오 기반 자동 선택과 수동 전환을 모두 지원합니다.

이 설계를 통해 HagiCode는 Claude Code CLI와 Codex CLI 간의 원활한 전환과 상호운용성을 성공적으로 구현하여, 개발자에게 유연하고 강력한 AI 코딩 어시스턴트 경험을 제공합니다.
