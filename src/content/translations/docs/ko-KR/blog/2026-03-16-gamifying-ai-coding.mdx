## 게임화 설계로 AI 프로그래밍을 더 재미있게 만드는 방법

> 사실 전통적인 AI 프로그래밍 도구는 기능이 꽤 강력하지만, 약간의 온기가 부족합니다. 우리는 HagiCode를 만들면서, 어차피 코드를 작성해야 한다면 게임으로 만들면 어떨까 생각했습니다.

## 배경

AI 프로그래밍 어시스턴트를 사용해 본 분들은 아마 이런 경험이 있으실 것입니다. 처음에는 꽤 신선했지만, 쓰다 보면 뭔가 부족한 것이 느껴집니다. 도구 자체의 기능은 매우 강력합니다. 코드 생성, 자동 완성, 버그 수정 모두 가능합니다. 다만...... 온기가 없고, 오래 사용하면 다소 단조롭고 지루해집니다.

이것도 그렇고,毕竟 매일 차가운 도구를 마주하고 싶은 사람은 없겠죠.

게임과 같습니다. 단순히 작업 목록을 완료하는 것만으로는 역할 성장, 성취감 해금, 팀 협력이 없다면 금방 지루해집니다. 아름다운 것은 반드시 소유할 필요가 없습니다. 그녀가 여전히 아름답다면, 그 아름다움을 잘 바라보면 됩니다. 그런데 프로그래밍 도구에는 이런 아름다움조차 없으니, 사람을 낙담하게 만드는 것은 당연합니다.

[HagiCode](https://hagicode.com)를 개발하는 과정에서 우리는 이 문제에 직면했습니다. HagiCode는 다중 AI 어시스턴트 협업 플랫폼으로, 사용자가 장기적으로 사용 열정을 유지해야 합니다. 하지만 현실은 아무리 좋은 도구라도 감정적 연결이 부족하면 사용자가 계속 유지하기 어렵습니다.

이러한 문제를 해결하기 위해 우리는 대담한 결정을 내렸습니다. 프로그래밍을 게임으로 만들자는 것입니다. 단순한 점수 리더보드가 아니라 진정한 롤플레잉식 게임화 경험입니다. 이 결정이 가져온 변화는 상상보다 클 것입니다.

毕竟, 사람은 항상 약간의 의식이 필요합니다.

## HagiCode에 대하여

이 글에서 공유하는 솔루션은 [HagiCode](https://hagicode.com) 프로젝트에서의 실천 경험에서 비롯되었습니다. HagiCode는 Claude Code, Codex, Copilot, OpenCode 등 다양한 AI 어시스턴트가 협업하는 다중 AI 어시스턴트 협업 플랫폼입니다. 다중 AI 협업, 게임화 프로그래밍에 관심이 있으시다면 [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site)를 방문해 주세요.

사실 별것도 아닙니다. 단지 프로그래밍을 한 편의 모험으로 만들었을 뿐입니다.

## 왜 게임화를 선택했는가

게임화의 본질은 "점수판 추가"가 아니라, 사용자가 작업을 수행하는 과정에서 성장감, 성취감, 사회적 인정을 경험할 수 있는 완전한 동기 부여 체계를 구축하는 것입니다.

HagiCode의 게임화 설계는 하나의 핵심 개념을 중심으로 전개됩니다. 각 AI 어시스턴트는 한 명의 "영웅"이며, 사용자는 이 영웅 팀의 대장입니다. 이 영웅들을 이끌고 다양한 "던전"(프로그래밍 작업)을 정복합니다. 이 과정에서 영웅들은 경험을 얻고, 레벨업하여 능력을 해금하며, 당신과 당신의 팀도 다양한 성취를 얻게 됩니다.

이것은 어떤 유행이 아니라, 인간 행동 심리학에 기반한 정교한 설계입니다. 작업에 의미와 진행 피드백이 부여되면 사람의 몰입도와 지속 정도가 현저히 향상됩니다.

옛사람들의 말처럼 "이 정은 추억이 될 수 있으나, 당시에는 이미 몽롱했다"입니다. 우리는 이러한 감정적 경험을 도구에 녹여넣어 프로그래밍이 더 이상 단순한 코드 타이핑이 아니라 추억할 만한 여정이 되도록 했습니다.

## Hero 캐릭터 시스템

Hero는 HagiCode 게임화 시스템의 핵심 개념입니다. 각 Hero는 하나의 AI 어시스턴트를 대표합니다. 예를 들어 Claude Code는 하나의 Hero이고, Codex도 하나의 Hero입니다.

### Hero의 세 가지 슬롯

Hero에는 세 가지 장비 슬롯이 있으며, 이 설계는 꽤 영리합니다:

1. **CLI 슬롯 (주요 직업)**: Hero의 기본 능력을 결정합니다. 예를 들어 Claude Code인지 Codex인지
2. **Model 슬롯 (보조 직업)**: 사용할 모델을 결정합니다. 예를 들어 Claude 4.5인지 Claude 4.6인지
3. **Style 슬롯 (스타일)**: Hero의 행동 스타일을 결정합니다. 예를 들어 "풍락 전략가"인지 "다른 스타일"인지

세 슬롯의 조합으로 고유한 Hero 구성을 만들 수 있습니다. 게임에서 장비를 맞추는 것과 같습니다. 작업의 특징에 따라 적절한 조합을 선택해야 합니다.毕竟 자신에게 맞는 것이 가장 좋습니다. 인생에서 길을 선택하는 것과 비슷합니다. 모든 길이 로마로 통하지만, 어떤 길은 조금 더 걷기 편하고, 어떤 길은 조금 더 우회합니다.

### Hero의 성장 시스템

각 Hero는 자신만의 XP(경험치)와 레벨을 가집니다:

```typescript
type HeroProgressionSnapshot = {
  currentLevel: number;                    // 현재 레벨
  totalExperience: number;                 // 총 경험치
  currentLevelStartExperience: number;     // 현재 레벨 시작 경험치
  nextLevelExperience: number;             // 다음 레벨 필요 경험치
  experienceProgressPercent: number;       // 진행률 퍼센트
  remainingExperienceToNextLevel: number;  // 다음 레벨까지 남은 경험치
  lastExperienceGain: number;              // 마지막으로 획득한 경험치
  lastExperienceGainAtUtc?: string | null; // 경험치 획득 시간
};
```

레벨은 네 단계로 나뉘며, 각 단계의 명명은 꽤 몰입감을 줍니다:

```typescript
export const resolveHeroProgressionStage = (level?: number | null): HeroProgressionStage => {
  const normalizedLevel = Math.max(1, level ?? 1);
  if (normalizedLevel <= 100) return 'rookieSprint';      // 신인 스프린트
  if (normalizedLevel <= 300) return 'growthRun';         // 성장 런
  if (normalizedLevel <= 700) return 'veteranClimb';      // 베테랑 클라임
  return 'legendMarathon';                                // 레전드 마라톤
};
```

"신인"에서 "레전드"까지, 이 성장 경로는 사용자에게 명확한 목표감과 성취감을 줍니다. 인생의 성장과 같이, 항상 무지에서 성숙으로의 과정을 거쳐야 합니다. 여기서는 이 과정을 구체화한 것뿐입니다.

### 커스텀 Hero 생성

Hero를 생성할 때 세 슬롯을 구성해야 합니다:

```typescript
const heroDraft: HeroDraft = {
  name: 'Athena',
  icon: 'hero-avatar:storm-03',
  description: '지략이 뛰어난 전략가',
  executorType: AIProviderType.CLAUDE_CODE_CLI,
  slots: {
    cli: {
      id: 'profession-claude-code',
      parameters: { /* CLI 관련 매개변수 */ }
    },
    model: {
      id: 'secondary-claude-4-sonnet',
      parameters: { /* 모델 관련 매개변수 */ }
    },
    style: {
      id: 'fengluo-strategist',
      parameters: { /* 스타일 관련 매개변수 */ }
    }
  }
};
```

각 Hero는 고유한 아바타, 설명 및 전문 포지션을 가지며, 이는 차가운 AI 어시스턴트에 개성과 온기를 부여합니다.毕竟 성격이 없는 도구와 거래하고 싶은 사람은 없겠죠?

## Dungeon 던전 시스템

"던전"은 게임의 클래식한 개념으로, 팀을 구성하여 공략해야 하는 도전을 나타냅니다. HagiCode에서 각 워크플로우가 하나의 Dungeon입니다.

### Dungeon의 조직 방식

Dungeon은 워크플로우를 다양한 "던전"으로 조직합니다:

- **제안 생성 던전**: 기술 제안 생성 담당
- **제안 실행 던전**: 제안의 작업 실행 담당
- **제안 아카이빙 던전**: 완료된 제안의 정리 및 아카이빙 담당

각 던전에는 자신만의 Captain(대장) Hero가 있으며, 대장은 활성화된 Hero 중 첫 번째에서 자동으로 선택됩니다.

사실 이것은 삶의 분업과 같습니다. 각자 자신의 역할이 있으며, 여기서는 이러한 분업을 게임 메커니즘으로 만든 것뿐입니다.

### 팀 협업 메커니즘

다양한 던전에 다른 Hero 팀을 구성할 수 있습니다:

```typescript
const dungeonRoster: HeroDungeonRoster = {
  scriptKey: 'proposal.generate',
  displayName: '제안 생성',
  members: [
    { heroId: 'hero-1', name: 'Athena', executorType: 'ClaudeCode' },
    { heroId: 'hero-2', name: 'Apollo', executorType: 'Codex' }
  ]
};
```

예를 들어 제안 생성 시 Athena(전략에 능함)를, 코드 실행 시 Apollo(구현에 능함)를 사용하여 각 영웅이 자신의 전문 분야를 발휘할 수 있습니다. 밴드를 구성하는 것과 같습니다. 각자 자신의 악기가 있고, 합쳐야 아름다운 선율을 연주할 수 있습니다.

### Dungeon 프로세스 제어

Dungeon은 고정된 scriptKey로 다양한 프로세스를 식별합니다:

```typescript
// 스크립트 키값은 다양한 워크플로우에 대응
const dungeonScripts = {
  'proposal.generate': '제안 생성',
  'proposal.execute': '제안 실행',
  'proposal.archive': '제안 아카이빙'
};
```

작업 상태 흐름은: queued(대기 중) -> dispatching(배분 중) -> dispatched(배분 완료)입니다. 전체 프로세스가 자동화되며 수동 개입이 필요 없습니다. 이것도 우리의 게으름쟁이 발상입니다.毕竟 누가 이런 일을 수동으로 관리하고 싶겠습니까.

## XP 및 레벨 시스템

XP(경험치)는 게임화 시스템에서 가장 핵심적인 피드백 메커니즘입니다. 사용자는 작업 완료를 통해 XP를 얻고, XP는 영웅을 레벨업시키며, 레벨업은 새로운 능력을 해금하여 순환을 형성합니다.

### XP 획득 방법

HagiCode에서 XP는 다음 활동을 통해 얻을 수 있습니다:

- 코드 실행 완료
- 도구 호출 성공
- 제안 생성
- 세션 관리 작업
- 프로젝트 작업

유효한 작업을 완료할 때마다 해당 Hero가 XP를 얻습니다. 삶의 성장과 같습니다. 모든 단계가 의미 있으며, 여기서는 이 성장을 정량화한 것뿐입니다.

### 실시간 진행률 시각화

XP 및 레벨의 진행률은 실시간으로 시각화됩니다:

```typescript
type HeroDungeonMember = {
  heroId: string;
  name: string;
  icon?: string | null;
  executorType: PCode_Models_AIProviderType;
  currentLevel?: number;                    // 현재 레벨
  totalExperience?: number;                 // 총 경험치
  experienceProgressPercent?: number;       // 진행률 퍼센트
};
```

사용자는 언제든 각 Hero의 레벨과 진행률을 볼 수 있으며, 이러한 즉각적인 피드백은 게임화 설계의 핵심입니다.毕竟 사람은 항상 피드백이 필요합니다. 그렇지 않으면 자신이 발전했는지 어떻게 알겠습니까?

## 성취 시스템

성취는 게임화의 또 다른 중요한 요소로, 장기 목표와 마일스톤식 성취감을 제공합니다.

### 성취 유형

HagiCode는 다양한 성취 유형을 지원합니다:

- **코드 생성 유형**: X줄의 코드 생성, Y개의 파일 생성
- **세션 관리 유형**: Z회의 대화 완료
- **프로젝트 작업 유형**: W개의 프로젝트에서 작업

사실 이러한 성취는 인생의 마일스톤과 같습니다. 여기서는 이것을 게임 메커니즘으로 만든 것뿐입니다.

### 성취 상태

성취에는 세 가지 상태가 있습니다:

```typescript
type AchievementStatus = 'unlocked' | 'in-progress' | 'locked';
```

세 가지 상태는 명확한 시각적 구분이 있습니다:

- **해금됨**: 골드 그라디언트 + 글로우 효과
- **진행 중**: 블루 펄스 애니메이션
- **잠김**: 회색, 해금 조건 표시

각 성취는 트리거 조건을 명확하게 보여주어 사용자가 다음에 무엇을 해야 할지 알 수 있습니다.毕竟 방황할 때 안내가 있으면 좋습니다.

### 해금 시 축하 효과

성취가 해금되면 축하 애니메이션이 트리거됩니다. 이러한 긍정적 강화는 사용자에게 "내가 해냈다"는 만족감을 주어 계속 전진하도록 동기부여합니다. 삶의 작은 보상과 같습니다. 크지 않지만 오래 기분 좋게 할 수 있습니다.

## Battle Report 일일 전투 보고서

Battle Report는 HagiCode의 특징적인 기능으로, 매일 종료 시 전체 화면으로 표시되는 전투 스타일 보고서를 생성합니다.

### 보고서 내용

Battle Report는 다음 정보를 표시합니다:

```typescript
type HeroBattleReport = {
  reportDate: string;
  summary: {
    totalHeroCount: number;        // 총 영웅 수
    activeHeroCount: number;       // 활성 영웅 수
    totalBattleScore: number;      // 총 전투 점수
    mvp: HeroBattleHero;           // 최우수 영웅
  };
  heroes: HeroBattleHero[];        // 모든 영웅의 상세 데이터
};
```

- 팀 총 점수
- 활성 Hero 수
- 도구 호출 횟수
- 총 작업 시간
- MVP(최우수 영웅)
- 각 Hero의 상세 카드

### MVP 하이라이트 표시

MVP는 당일 가장 좋은 성과를 보인 Hero로, 보고서에서 하이라이트로 표시됩니다. 이것은 단순한 데이터 통계가 아니라 명예 인정입니다.毕竟 자신이 인정받기를 원하지 않는 사람이 있겠습니까?

### Hero 상세 카드

각 Hero의 카드에는 다음이 포함됩니다:

- 레벨 진행률
- XP 획득량
- 실행 횟수
- 사용 시간

이러한 데이터는 사용자가 팀의 작업 상태를 명확하게 이해할 수 있게 합니다.毕竟 자신의 노력 성과를 아는 것도 일종의 만족감입니다.

## 기술 구현

HagiCode의 게임화 시스템은 현대적인 기술 스택과 디자인 패턴을 채택했습니다. 사실 별것도 아닙니다. 그냥 편리한 도구를 선택했을 뿐입니다.

### 기술 스택 선택

```typescript
// 프론트엔드는 React + TypeScript 사용
import React from 'react';

// 애니메이션은 Framer Motion 사용
import { AnimatePresence, motion } from 'framer-motion';

// 상태 관리는 Redux Toolkit 사용
import { useAppDispatch, useAppSelector } from '@/store';

// UI 컴포넌트는 shadcn/ui 사용
import { Dialog, DialogContent } from '@/components/ui/dialog';
```

Framer Motion은 모든 애니메이션 효과를 담당하고, shadcn/ui는 기본 UI 컴포넌트를 제공하며, Redux Toolkit은 복잡한 게임화 상태를 관리합니다.毕竟 일을 잘하려면 먼저 도구를 날카롭게 해야 합니다.

### 게임화 UI 디자인 시스템

HagiCode는 Glassmorphism + Tech Dark 디자인 스타일을 채택했습니다:

```css
/* 메인 그라디언트 색상 */
background: linear-gradient(135deg, #22C55E 0%, #25c2a0 50%, #06b6d4 100%);

/* 글래스모피즘 효과 */
backdrop-filter: blur(12px);

/* 글로우 효과 */
background: radial-gradient(circle at center, rgba(34, 197, 94, 0.15) 0%, transparent 70%);
```

그린 계열 그라디언트와 글래스모피즘 효과의 조합으로 테크감과 미래감을 연출합니다.毕竟 시각적 아름다움도 사용자 경험의 일부입니다.

### 애니메이션 효과 구현

Framer Motion을 사용하여 부드러운 진입 애니메이션을 구현합니다:

```typescript
<motion.div
  animate={{ opacity: 1, y: 0 }}
  initial={{ opacity: 0, y: 18 }}
  transition={{ duration: 0.35, ease: 'easeOut', delay: index * 0.08 }}
  className="card"
>
  {/* 카드 내용 */}
</motion.div>
```

각 카드가 순차적으로 진입하며 0.08초의 지연을 가지어 부드러운 시각 효과를 만듭니다.毕竟 부드러운 애니메이션은 경험을 더 좋게 만드는 것은 의심할 여지가 없습니다.

### 데이터 영속화

게임화 데이터는 Grain 저장 시스템을 사용하여 상태 일관성을 보장합니다. Hero의 XP 누적과 같은 세분화된 데이터도 정확하게 영속화할 수 있습니다.毕竟 누구도辛苦하게 쌓은 경험치를 잃고 싶지 않습니다.

## 실천 가이드

### 첫 번째 Hero 생성

첫 번째 Hero를 생성하는 것은 꽤 간단합니다:

1. Hero 관리 페이지로 이동
2. "Hero 생성" 버튼 클릭
3. 세 슬롯 구성 (CLI, Model, Style)
4. Hero에 이름과 설명 부여
5. 저장, 첫 번째 Hero가 탄생합니다

새 친구를 사귀는 것과 같습니다. 이름을 지어주고 특징을 이해한 다음 함께 모험을 떠나면 됩니다.

### Dungeon 팀 구성

팀 구성도 간단합니다:

1. Dungeon 관리 페이지로 이동
2. 구성할 던전 선택 (예: "제안 생성")
3. Hero 목록에서 멤버 선택
4. 시스템이 자동으로 첫 번째 활성화된 Hero를 대장으로 선택
5. 설정 저장

사실 이것은 팀 구성 과정입니다. 삶에서 팀을 구성하는 것과 같습니다. 각자 자신의 역할이 있습니다.

### 일일 보고서 확인

매일 종료 후 당일의 Battle Report를 확인할 수 있습니다:

1. "전투 보고서" 버튼 클릭
2. 전체 화면으로 당일의 작업 성과 표시
3. MVP 및 각 Hero의 상세 데이터 확인
4. 팀원에게 공유 (선택 사항)

이것도 일종의 의식입니다. 오늘 얼마나 노력했는지, 목표에서 얼마나 멀리 있는지 알게 합니다.

## 주의사항 및 베스트 프랙티스

### 성능 최적화

React.memo를 사용하여 불필요한 리렌더링을 방지합니다:

```typescript
const HeroCard = React.memo(({ hero }: { hero: HeroDungeonMember }) => {
  // 컴포넌트 구현
});
```

毕竟 성능도 중요합니다. 끊기는 도구를 사용하고 싶은 사람은 없겠죠?

### 애니메이션 저하 가능

사용자의 모션 환경설정을 감지하여 민감한 사용자에게 단순화된 경험을 제공합니다:

```typescript
const prefersReducedMotion = useReducedMotion();
const duration = prefersReducedMotion ? 0 : 0.35;
```

毕竟 모든 사람이 애니메이션을 좋아하는 것은 아니며, 사용자의 선호를 존중하는 것도 설계의 일부입니다.

### 후방 호환성

legacyIds를 보존하여 이전 버전 마이그레이션을 지원합니다:

```typescript
type HeroDungeonMember = {
  heroId: string;
  legacyIds?: string[];  // 이전 버전 ID 매핑 지원
  // ...
};
```

毕竟 누구도 버전 업그레이드로 인해 데이터를 잃고 싶지 않습니다.

### 국제화 지원

모든 텍스트는 i18n 번역 키를 사용하여 다국어 지원을 용이하게 합니다:

```typescript
const displayName = t(`dungeon.${scriptKey}`, { defaultValue: displayName });
```

毕竟 언어는 사용의 장애가 되어서는 안 됩니다.

## 요약

게임화는 단순한 점수 리더보드가 아니라 완전한 동기 부여 체계입니다. HagiCode는 Hero 시스템, Dungeon 시스템, XP/레벨 시스템, 성취 시스템 및 Battle Report를 통해 프로그래밍 작업을 모험정신이 넘치는 영웅의 여정으로 변환했습니다.

이 시스템의 핵심 가치는 다음과 같습니다:

- **감정적 연결**: 차가운 AI 어시스턴트에 개성 부여
- **긍정적 피드백**: 모든 작업에 즉각적인 피드백 제공
- **장기 목표**: 레벨과 성취로 성장 경로 제공
- **팀 인정**: Dungeon 팀 협업감
- **명예 인정**: Battle Report 및 MVP 표시

게임화 설계는 프로그래밍을 지루하지 않게 만들고, 재미있는 모험으로 만듭니다. 사용자는 코드 작업을 완료하는 동시에 캐릭터 성장, 팀 협업 및 성취 해금의 즐거움을 경험하여 사용 지속성과 활성도를 높입니다.

사실 프로그래밍 자체가 일종의 창조입니다. 우리는 이 창조 과정을 조금 더 재미있게 만들었을 뿐입니다.
