## 타이핑보다 말하기가 빠르고, 말하기보다 스크린샷이 빠릅니다 - AI 코딩 어시스턴트를 위한 멀티모달 입력 실무

> 코드를 작성하는 데에는 타이핑이 아무리 빨라도 속도 한계가 있습니다. 때로는 한 문장으로 말할 수 있는 것을 타이핑하는 데 영원이 걸리고, 때로는 한 장의 스크린샷이 모든 것을 설명하지만 여전히 말로 설명해야 합니다. 이 글에서는 [HagiCode](https://hagicode.com)를 구축하면서 겪은 경험을 이야기합니다. 음성 인식에서 이미지 업로드까지. 결국 우리는 AI 코딩 어시스턴트를 조금 더 사용하기 쉽게 만들고 싶었을 뿐입니다.

## 배경

HagiCode를 구축하면서 우리는 하나의 문제를 발견했습니다. 아니, 사람들이 열심히 사용하기 시작하면서 자연스럽게 표면으로 드러난 문제라고 해야 할 것 같습니다. 타이핑만 의존하면 피곤합니다.

생각해 보십시오. 사용자와 Agent 간의 상호작용은 핵심 시나리오입니다. 하지만 모든 교환마다 키보드에서 쉴 새 없이 타이핑해야 한다면 효율이 그리 좋지 않습니다:

1. **타이핑이 너무 느립니다**: 복잡한 문제의 경우, 오류 메시지나 UI 문제를 타이핑하려면 반나절이 걸릴 수 있지만, 말로 하면 10초면 됩니다. 그 차이는 실제합니다.

2. **이미지가 더 직관적입니다**: 때로는 UI에서 오류를 던지거나, 디자인 초안을 비교하거나, 코드 구조를 보여주고 싶을 때가 있습니다. "백마디 말보다 한 장의 그림"은 오래된 속담이지만, 여전히 유효합니다. AI가 문제를 직접 "보게" 하는 것이 오래 설명하는 것보다 훨씬 명확합니다.

3. **상호작용은 자연스러워야 합니다**: 현대 AI 어시스턴트는 텍스트, 음성, 이미지를 지원해야 합니다. 사용자는 가장 자연스럽게 느껴지는 입력 방법을 선택할 수 있어야 합니다.

그래서 우리는 HagiCode에 음성 인식과 이미지 업로드 지원을 추가하여 Agent 상호작용을 더 편리하게 만들기로 결정했습니다. 사용자가 조금만 덜 타이핑해도, 그것만으로도 성공입니다.

## HagiCode 소개

이 글에서 공유하는 해결책은 [HagiCode](https://hagicode.com) 프로젝트의 실무 작업에서 나온 것입니다. 아니, 더 정확히는 꽤 많은 함정을 겪으면서 배운 교훈에서 나온 것입니다.

HagiCode는 AI를 활용하여 개발 효율을 높이는 것을 목표로 하는 오픈소스 AI 코딩 어시스턴트 프로젝트입니다. 계속 구축하면서 사용자가 멀티모달 입력을 강력하게 원한다는 것을 알게 되었습니다. 때로는 한 문장을 말하는 것이 긴 단락을 타이핑하는 것보다 빠르고, 때로는 스크린샷이 긴 설명보다 훨씬 명확합니다.

그러한 요구가 우리를 앞으로 밀어주었고, 그렇게 음성 인식과 이미지 업로드 기능이 마침내 모양을 갖추게 되었습니다. 사용자는 이제 가장 자연스러운 방식으로 AI와 상호작용할 수 있으며, 그것은 참 좋은 느낌입니다.

## 분석

### 음성 인식의 기술적 과제

음성 인식을 구축하면서 까다로운 문제에 직면했습니다: **브라우저 WebSocket API는 사용자 정의 HTTP 헤더를 지원하지 않습니다.**

우리가 선택한 음성 인식 서비스는 바이트댄스의 더보보 음성 인식 API였습니다. 불행히도 이 API는 `accessToken` 및 `secretKey`와 같은 인증 정보를 HTTP 헤더를 통해 전달해야 합니다. 이로 인해 즉각적인 기술적 충돌이 발생했습니다:

```javascript
// 브라우저 WebSocket API는 이 방식을 지원하지 않습니다
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // 지원되지 않음
  }
});
```

기본적으로 두 가지 옵션이 있었습니다:

1. **URL 쿼리 파라미터 방식**: 인증 정보를 URL에 넣습니다
   - 장점은 구현이 간단합니다
   - 단점은 자격 증명이 프론트엔드에 노출되어 안전하지 않으며, 일부 API는 헤더 기반 인증을 요구합니다

2. **백엔드 프록시 방식**: 백엔드에 WebSocket 프록시를 구현합니다
   - 장점은 자격 증명이 백엔드에 안전하게 저장되고 API 요구사항을 완전히 준수합니다
   - 단점은 구현이 조금 더 복잡합니다

결국 우리는 **백엔드 프록시 방식**을 선택했습니다. 보안은 타협할 수 있는 것이 아닙니다.

### 이미지 업로드의 기능 요구사항

이미지 업로드에 대한 요구사항은 사실 꽤 간단했습니다:

1. **다중 업로드 방법**: 파일 클릭 선택, 드래그 앤 드롭, 클립보드 붙여넣기
2. **파일 검증**: 유형 제한(PNG, JPG, WebP, GIF) 및 크기 제한(5-10MB)은 기본 요구사항입니다
3. **사용자 경험**: 업로드 진행 상황, 미리보기, 오류 메시지로 사용자가 항상 상황을 알 수 있도록 합니다
4. **보안**: 서버 측 검증 및 악의적인 파일 업로드 방지가 필수적입니다

## 해결책

### 음성 인식: WebSocket 프록시 아키텍처

음성 인식을 위해 3계층 아키텍처를 설계하고 작동하는 경로를 찾았습니다:

```
브라우저 WebSocket
       |
       | ws://backend/api/voice/ws
       | (바이너리 오디오)
       v
백엔드 프록시
       |
       | wss://openspeech.bytedance.com/ (인증 헤더 포함)
       v
더보보 API
```

**핵심 컴포넌트 구현**:

1. **프론트엔드 AudioWorklet 프로세서**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // 16kHz로 리샘플링 (더보보 API 요구사항)
    const samples = this.resampleAudio(input, 48000, 16000);

    // 500ms 청크로 누적
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // 16비트 PCM으로 변환 후 전송
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **백엔드 WebSocket 핸들러** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **프론트엔드 `VoiceTextArea` 컴포넌트**:

```tsx
export const VoiceTextArea = forwardRef<HTMLTextAreaElement, VoiceTextAreaProps>(
```

### 이미지 업로드: 다중 방식 업로드 컴포넌트

세 가지 업로드 방법을 모두 지원하는 완전 기능 이미지 업로드 컴포넌트를 구축하여, 사용자가 겪는 가장 일반적인 시나리오를 커버합니다.

**핵심 기능**:

1. **세 가지 업로드 방법**:

```tsx
// 클릭하여 업로드
const handleClick = () => fileInputRef.current?.click();

// 드래그 앤 드롭 업로드
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// 클립보드 붙여넣기
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **프론트엔드 검증**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'PNG, JPG, JPEG, WebP, GIF 이미지만 허용됩니다' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `최대 파일 크기는 ${(maxSize / 1024 / 1024).toFixed(1)}MB입니다` };
  }
  return { valid: true };
};
```

3. **백엔드 업로드 핸들러** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // 검증
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // 파일 저장
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## 실무 가이드

### 음성 인식 사용 방법

1. **음성 인식 서비스 구성**:
   - 음성 인식 설정 페이지 열기
   - 더보보 음성 `AppId` 및 `AccessToken` 구성
   - 도메인 특정 용어의 인식 정확도를 높이기 위해 핫워드(hotword) 구성 (선택사항)

2. **입력 상자에서 사용**:
   - 입력 상자 왼쪽의 마이크 아이콘 클릭
   - 파형 애니메이션이 나타난 후 말하기 시작
   - 아이콘을 다시 클릭하여 녹음 중지
   - 인식된 텍스트가 커서 위치에 자동 삽입됩니다

3. **핫워드 구성 예시**:

```text

React
useState
useEffect
```

### 이미지 업로드 사용 방법

1. **업로드 방법**:
   - 업로드 버튼을 클릭하여 파일 선택
   - 이미지를 업로드 영역으로 직접 드래그
   - `Ctrl+V`로 클립보드에서 스크린샷 붙여넣기

2. **지원 형식**: PNG, JPG, JPEG, WebP, GIF
3. **크기 제한**: 기본 5MB (구성 가능)

### 참고 사항

1. **음성 인식**:
   - 마이크 권한이 필요합니다
   - 가능하면 조용한 환경에서 사용하십시오
   - 최대 녹음 시간은 기본 300초입니다 (구성 가능)

2. **이미지 업로드**:
   - 일반적인 이미지 형식만 지원됩니다
   - 파일 크기 제한에 주의하십시오
   - 업로드된 이미지에 미리보기 URL이 자동으로 부여됩니다

3. **보안 고려사항**:
   - 음성 인식 자격 증명은 백엔드에 저장됩니다
   - 이미지 업로드는 엄격한 서버 측 검증을 거칩니다
   - 프로덕션 환경에서는 HTTPS/WSS를 권장합니다

## 결론

음성 인식과 이미지 업로드를 추가한 후 HagiCode 사용자 경험이 눈에 띄게 향상되었습니다. 사용자는 이제 더 자연스러운 방식으로 AI와 상호작용할 수 있습니다. 말하는 대신 타이핑하고, 모든 것을 수동으로 설명하는 대신 스크린샷을 공유합니다. 마침내 더 편안한 소통 방식을 찾은 것 같은 느낌입니다.

이 기능을 구축하면서 브라우저 WebSocket API가 사용자 정의 헤더를 지원하지 않는다는 문제에 직면했습니다. 결국 백엔드 프록시 방식으로 해결했습니다. 이 해결책은 보안을 보존할 뿐만 아니라, 향후 다른 인증된 WebSocket 서비스를 통합할 기반도 마련했습니다.

이미지 업로드 컴포넌트도 다중 업로드 방법을 지원하여 사용자가 순간에 가장 편리한 방법을 선택할 수 있게 합니다. 클릭, 드래그, 붙여넣기 모두 작동하며, 각 경로는 빠르게 작업을 완료합니다.

"타이핑보다 말하기가 빠르고, 말하기보다 스크린샷이 빠르다"는 이 주제에 잘 맞습니다. 비슷한 AI 어시스턴트 제품을 구축 중이시라면, 이 경험이 조금이라도 도움이 되기를 바랍니다.
