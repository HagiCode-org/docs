## HagiCode Skill 시스템 기술 분석: 확장 가능한 AI 스킬 관리 플랫폼 구축

> 이 글에서는 HagiCode 프로젝트의 Skill 관리 시스템의 아키텍처와 구현을 심층적으로 살펴보며, 네 가지 핵심 기능인 로컬 글로벌 관리, 마켓플레이스 검색, 지능형 추천, 신뢰할 수 있는 제공자 관리의 기술적 세부 사항을 다룹니다.

## 배경

AI 코딩 어시스턴트 분야에서 AI 능력의 경계를 어떻게 확장하는 것은 항상 핵심 질문이었습니다. Claude Code 자체는 이미 코드 지원에 강력하지만, 다른 개발 팀과 다른 기술 스택은 종종 Docker 배포, 데이터베이스 최적화 또는 프론트엔드 컴포넌트 생성과 같은 특정 시나리오에 대한 전문화된 기능이 필요합니다. 바로 이곳에서 Skill 시스템이 특히 중요해집니다.

HagiCode 프로젝트 개발 과정에서 우리는 비슷한 과제에 직면했습니다: Claude Code가 사람처럼 새로운 전문 기술을 "학습"하도록 하면서도, 견고한 사용자 경험과 좋은 엔지니어링 유지보수성을 유지하는 방법은 무엇인가? 이 문제는 그 자체로 어렵기도 하고 간단하기도 합니다. 이 질문을 중심으로 우리는 완전한 Skill 관리 시스템을 설계하고 구현했습니다.

이 글에서는 시스템의 기술 아키텍처와 핵심 구현을 상세히 설명합니다. AI 확장성과 명령줄 도구 통합에 관심 있는 개발자를 대상으로 합니다. 유용할 수도 있고 아닐 수도 있지만, 적어도 이제는 기록되었습니다.

## HagiCode에 대하여

이 글에서 공유하는 접근 방식은 HagiCode 프로젝트의 실무 경험에서 비롯되었습니다. HagiCode는 데스크톱 사용을 지원하는 AI 코딩 어시스턴트입니다. AI 코딩 어시스턴트는 Claude Code CLI를 기반으로 구축되었으며, 팀 협업과 클라우드 확장을 지원합니다.

## 기능 개요

HagiCode의 Skill 시스템은 주로 네 가지 핵심 기능으로 구성됩니다:

1. **로컬 글로벌 관리**: 기기에 설치된 스킬을 보고, 설치하고, 제거할 수 있습니다. 전역 스킬과 프로젝트 로컬 스킬을 모두 지원합니다.
2. **마켓플레이스 검색**: 온라인 API를 통해 커뮤니티 스킬을 검색하고 발견할 수 있습니다. TanStack Query를 사용하여 캐싱과 상태 관리를 처리합니다.
3. **지능형 추천**: 프로젝트 컨텍스트를 기반으로 AI가 적절한 스킬을 추천합니다.
4. **신뢰할 수 있는 제공자 관리**: 사용자가 어떤 스킬 소스를 신뢰할 수 있는지 제어할 수 있습니다.

## 기술 아키텍처

### 프론트엔드 구조

프론트엔드는 TanStack Start 프레임워크를 기반으로 구축되었으며, 주요 모듈은 다음과 같습니다:

- **SkillsDrawer**: 스킬 관리의 메인 진입점으로, 탭 전환과 레이아웃을 담당합니다.
- **SkillGalleryTab**: 마켓플레이스 검색 기능을 제공합니다. 검색 입력, 결과 목록, 상세 정보 패널을 포함합니다.
- **LocalSkillsTab**: 로컬 스킬 목록을 표시하고 설치/제거 작업을 지원합니다.
- **SkillInstallButton**: SSE 스트리밍을 통해 설치 진행 상황을 실시간으로 표시합니다.

### 백엔드 서비스

백엔드는 Orleans 기반의 분산 시스템으로, 주요 Grain은 다음과 같습니다:

- **SkillManagerGrain**: 스킬 설치, 제거, 목록 조회의 핵심 로직을 처리합니다.
- **TrustedSkillProviderGrain**: 신뢰할 수 있는 제공자 구성을 관리합니다.

### 명령어 실행

스킬 설치와 관리는 기본적으로 `npx skills` 명령어를 래핑한 것입니다:

```bash
npx skills add <skill-reference> -g -y   # 글로벌 설치
npx skills remove <skill-name> -g        # 글로벌 제거
npx skills list -g                       # 글로벌 스킬 목록
```

## 핵심 구현 상세

### 로컬 글로벌 관리

로컬 스킬 관리는 비교적 직관적입니다. 시스템은 `npx skills` 명령어를 실행하여 스킬을 설치하고, 프로세스 출력을 파싱하여 결과를 얻습니다.

```csharp
public async Task<SkillListResult> ListSkillsAsync(bool global, CancellationToken cancellationToken = default)
{
    var arguments = global ? "list -g --json" : "list --json";
    var result = await ExecuteCommandAsync("npx", $"skills {arguments}", cancellationToken);

    if (result.ExitCode != 0)
    {
        throw new SkillOperationException($"Failed to list skills: {result.Error}");
    }

    return JsonSerializer.Deserialize<SkillListResult>(result.Output)
        ?? throw new JsonException("Failed to parse skill list output");
}
```

글로벌 설치와 프로젝트 로컬 설치의 차이는 주로 `-g` 플래그에 있습니다. 글로벌 스킬은 모든 프로젝트에서 사용할 수 있고, 로컬 스킬은 특정 프로젝트 내에서만 유효합니다. 이 구분은 사소해 보이지만 실제로는 매우 실용적입니다.

### 마켓플레이스 검색

스킬 검색은 `api.hagicode.com`의 온라인 API를 통해 이루어집니다. 프론트엔드는 TanStack Query를 사용하여 검색 상태를 관리합니다:

```typescript
const searchQuery = useQuery({
  queryKey: ['skill-search', searchTerm],
  queryFn: async () => {
    const response = await fetch(`/api/skills/search?q=${encodeURIComponent(searchTerm)}`);
    return response.json();
  },
  enabled: searchTerm.length >= 2,
  staleTime: 1000 * 60 * 5, // 5분 캐시
});
```

검색 결과에는 스킬 이름, 설명, 설치 수, 신뢰 상태 등의 정보가 포함됩니다. 5분의 캐시 시간은 자주 검색하는 사용자에게 빠른 응답을 제공하면서도 데이터의 신선도를 유지합니다.

### 신뢰할 수 있는 제공자 관리

신뢰할 수 있는 제공자 관리를 통해 사용자는 어떤 스킬 소스를 신뢰할 수 있는지 제어할 수 있습니다. 신뢰는 여전히 사용자가 직접 정의할 수 있어야 하는 것입니다.

#### 매칭 규칙

신뢰할 수 있는 제공자는 두 가지 매칭 규칙을 지원합니다: **정확히 일치**(`exact`)와 **접두사 일치**(`prefix`).

```csharp
public static TrustedSkillProviderResolutionSnapshot Resolve(
    TrustedSkillProviderSnapshot snapshot,
    string source)
{
    var normalizedSource = Normalize(source);

    foreach (var entry in snapshot.Entries.OrderBy(e => e.SortOrder))
    {
        if (!entry.IsEnabled) continue;

        foreach (var rule in entry.MatchRules)
        {
            bool isMatch = rule.MatchType switch
            {
                TrustedSkillProviderMatchRuleType.Exact
                    => string.Equals(normalizedSource, Normalize(rule.Value),
                        StringComparison.OrdinalIgnoreCase),
                TrustedSkillProviderMatchRuleType.Prefix
                    => normalizedSource.StartsWith(Normalize(rule.Value) + "/",
                        StringComparison.OrdinalIgnoreCase),
                _ => false
            };

            if (isMatch)
                return new TrustedSkillProviderResolutionSnapshot
                {
                    IsTrustedSource = true,
                    ProviderId = entry.ProviderId,
                    DisplayName = entry.DisplayName
                };
        }
    }

    return new TrustedSkillProviderResolutionSnapshot { IsTrustedSource = false };
}
```

내장된 신뢰할 수 있는 제공자에는 Vercel, Azure, anthropics, Microsoft, browser-use와 같은 잘 알려진 조직과 프로젝트가 포함됩니다. 사용자 정의 제공자는 구성 파일을 통해 제공자 ID, 표시 이름, 배지 라벨, 매칭 규칙 등을 지정하여 추가할 수 있습니다. 세상은 넓어서 몇 가지 내장 항목만 신뢰하는 것으로는 절대 충분하지 않습니다.

#### 지속성 구현

신뢰 구성은 Orleans Grain을 사용하여 지속됩니다:

```csharp
public class TrustedSkillProviderGrain : Grain<TrustedSkillProviderState>,
    ITrustedSkillProviderGrain
{
    public async Task UpdateConfigurationAsync(TrustedSkillProviderSnapshot snapshot)
    {
        State.Snapshot = snapshot;
        await WriteStateAsync();
    }

    public Task<TrustedSkillProviderSnapshot> GetConfigurationAsync()
    {
        return Task.FromResult(State.Snapshot);
    }
}
```

이 방식의 장점은 구성 변경이 모든 노드에 자동으로 동기화되며, 캐시를 수동으로 새로고침할 필요가 없다는 것입니다. 자동화는 결국 사람의 걱정을 줄여주는 것입니다.

## 핵심 기술 설계

### 명령 실행 어댑터 패턴

Skill 시스템은 다양한 npx 명령어를 실행해야 합니다. 이 로직이 여기저기 흩어져 있으면 코드는 빠르게 유지보수하기 어려워집니다. 그래서 어댑터 인터페이스를 설계했습니다. 디자인 패턴은 결국 코드를 더 쉽게 유지보수하기 위해 존재합니다:

```csharp
public interface ISkillInstallCommandRunner
{
    Task<SkillInstallCommandExecutionResult> ExecuteAsync(
        SkillInstallCommandExecutionRequest request,
        CancellationToken cancellationToken = default);
}
```

다른 명령어는 다른 실행기 구현을 가지지만, 모두 동일한 인터페이스를 구현하여 테스트와 교체를 간단하게 만듭니다.

### SSE 스트리밍 출력

설치 진행 상황은 Server-Sent Events를 통해 프론트엔드에 실시간으로 푸시됩니다:

```csharp
public async Task InstallWithProgressAsync(
    SkillInstallRequestDto request,
    IServerStreamWriter<SkillInstallProgressEventDto> stream,
    CancellationToken cancellationToken)
{
    var process = new Process
    {
        StartInfo = new ProcessStartInfo
        {
            FileName = "npx",
            Arguments = $"skills add {request.FullReference} -g -y",
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false
        }
    };

    process.OutputDataReceived += async (sender, e) =>
    {
        await stream.WriteAsync(new SkillInstallProgressEventDto
        {
            EventType = "output",
            Data = e.Data ?? string.Empty
        });
    };

    process.Start();
    process.BeginOutputReadLine();
    await process.WaitForExitAsync(cancellationToken);
}
```

프론트엔드에서는 사용자가 터미널과 유사한 출력을 실시간으로 볼 수 있어 경험이 매우 직관적입니다. 실시간 피드백은 사람들을 안심시켜 줍니다.

## 실용 가이드

### 커뮤니티 스킬 설치

인기 있는 커뮤니티 스킬인 `pua` 스킬 설치를 예로 들어보겠습니다:

1. Skills 서랍을 열고 `Skill Gallery` 탭으로 전환합니다.
2. 검색 상자에 `pua`를 입력합니다.
3. 검색 결과를 클릭하여 스킬 상세 정보를 확인합니다.
4. `Install` 버튼을 클릭합니다.
5. `Local Skills` 탭으로 전환하여 설치가 성공적으로 완료되었는지 확인합니다.

설치 명령어는 `npx skills add tanweai/pua -g -y`이며, 시스템이 모든 세부 사항을 자동으로 처리합니다. 실제로 하나씩 따라가면 그렇게 많은 단계가 아닙니다.

### 사용자 정의 신뢰 소스 추가

팀에 자체 스킬 저장소가 있는 경우, 이를 신뢰할 수 있는 소스로 추가할 수 있습니다:

```yaml
providerId: "my-team"
displayName: "My Team Skills"
badgeLabel: "MyTeam"
isEnabled: true
sortOrder: 100
matchRules:
  - matchType: "prefix"
    value: "my-team/"
  - matchType: "exact"
    value: "my-team/special-skill"
```

이렇게 하면 팀의 모든 스킬에 신뢰 배지가 표시되어 사용자가 더 편안하게 설치할 수 있습니다. 라벨과 시그널은 사람들의 자신감을 높여주는 데 도움이 됩니다.

### 스킬 개발 기초

사용자 정의 스킬을 생성하려면 다음 구조가 필요합니다:

```
my-skill/
├── SKILL.md          # 스킬 메타데이터 (YAML front matter)
├── index.ts          # 스킬 진입점
├── agents/           # 지원되는 에이전트 구성
└── references/       # 참조 리소스
```

`SKILL.md` 형식 예시:

```yaml
---
name: my-skill
description: 이 스킬이 수행하는 작업에 대한 간단한 설명
---

# My Skill

상세 문서...
```

## 참고 사항

1. **네트워크 요구 사항**: 스킬 검색 및 설치에는 `api.hagicode.com`과 npm 레지스트리에 대한 접근이 필요합니다.
2. **Node.js 버전**: Node.js 18 이상을 권장합니다.
3. **권한 요구 사항**: 전역 npm 설치 권한이 필요합니다.
4. **동시성 제어**: 동일한 스킬에 대해 한 번에 하나의 설치 또는 제거 작업만 실행할 수 있습니다.
5. **타임아웃 설정**: 설치의 기본 타임아웃은 4분이지만, 복잡한 시나리오에서는 조정이 필요할 수 있습니다.

이 참고 사항은 결국 모든 것이 순조롭게 진행되도록 돕기 위해 존재합니다.

## 결론

이 글에서는 HagiCode 프로젝트의 Skill 관리 시스템의 완전한 구현을 소개했습니다. 프론트엔드-백엔드 분리 아키텍처, 어댑터 패턴, Orleans 기반 분산 상태 관리 및 관련 기술을 통해 시스템은 다음을 제공합니다:

- **로컬 글로벌 관리**: `npx skills` 명령어를 래핑하여 구축한 통합 스킬 관리 인터페이스
- **마켓플레이스 검색**: 온라인 API와 캐싱 메커니즘을 통한 커뮤니티 스킬의 빠른 발견
- **지능형 추천**: 프로젝트 컨텍스트 기반 AI 스킬 추천
- **신뢰 관리**: 사용자가 신뢰 경계를 제어할 수 있는 유연한 구성 시스템

이 설계 접근 방식은 Skill 관리뿐만 아니라, 명령줄 도구를 통합하면서 로컬 저장소와 온라인 서비스를 균형 있게 조화시켜야 하는 모든 시나리오에 참고로 유용합니다.
