Hermes를 설치하고 바로 사용하기 시작하려면, 가장 짧은 경로는 실제로 세 단계뿐입니다.

1. 공식 설치 명령어 실행
2. 터미널에서 `hermes`로 CLI 시작
3. Feishu에서 계속 사용하고 싶다면 `hermes gateway setup` 구성

이 글은 Hermes의 모든 기능을 한 번에 설명하려 하지 않습니다. 대신 가장 중요한 초보자 루프를 먼저 완료하도록 돕습니다. 설치하고, 실행하고, 사용하기 시작한 다음, 가장 일반적인 메시징 플랫폼 시나리오 중 하나에 연결합니다.

## Hermes란 무엇이며 누구를 위한 것인가

Hermes Agent는 로컬 터미널이나 메시징 플랫폼 게이트웨이를 통해 사용할 수 있는 AI 에이전트입니다.

대부분의 개발자에게 두 가지 일반적인 진입점이 있습니다.

- **CLI**: 터미널에서 `hermes`를 입력하여 직접 대화형 인터페이스에 진입합니다.
- **메시징 게이트웨이**: `hermes gateway`를 실행한 다음 Feishu, Telegram, Discord, Slack과 같은 플랫폼에서 대화합니다.

현재 목표가 빠르게 시작하는 것이라면, 순서를 바꾸지 마세요. 대신 이 경로로 시작하세요.

- 먼저 Hermes 설치
- 먼저 CLI에서 작동하는지 확인
- 그런 다음 메시징 플랫폼을 연결할지 결정

이렇게 하면 문제 진단이 더 쉬워지며, Hermes를 처음 사용하는 사람에게 더 적합합니다.

## Hermes 설치 전 알아야 할 것

Hermes README에 따르면, 공식 빠른 설치 경로는 다음 환경을 지원합니다.

- Linux
- macOS
- WSL2
- Termux를 통한 Android

### Windows 사용자 참고 사항

Hermes는 현재 네이티브 Windows에서 직접 실행하는 것을 지원하지 않습니다. Windows를 사용 중이라면 WSL2를 먼저 설치한 다음 WSL2 내에서 설치 명령어를 실행하는 것이 좋습니다.

이것은 처음에 명확히 해두는 것이 좋습니다. 많은 설치 실패가 명령어 자체 때문이 아니라 지원되지 않는 런타임 환경을 사용했기 때문이기 때문입니다.

## Hermes 설치 방법

Hermes README에 제공된 빠른 설치 명령어는 다음과 같습니다.

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

이 명령어는 공식 설치 스크립트를 실행하며 플랫폼별 초기화 단계를 처리합니다.

### 설치 후 셸 다시 로드

설치가 완료되면 먼저 셸 환경을 다시 로드하세요. 가장 일반적인 명령어는 다음과 같습니다.

```bash
source ~/.bashrc
```

zsh를 사용한다면 다음을 사용할 수 있습니다.

```bash
source ~/.zshrc
```

### Hermes가 올바르게 설치되었는지 확인하는 방법

가장 직접적인 확인 방법은 다음을 실행하는 것입니다.

```bash
hermes
```

설정과 종속성이 올바르게 작동하는지 추가로 확인하려면 다음도 실행할 수 있습니다.

```bash
hermes doctor
```

`hermes doctor`는 다음 상황에서 특히 유용합니다.

- 설치 후 명령어가 비정상적으로 동작할 때
- 모델 설정이 실패할 때
- 게이트웨이가 시작되지 않을 때
- 환경 종속성이 완전한지 확신이 없을 때

## Hermes를 처음 사용하는 방법

### CLI로 시작하기

Hermes가 작동하는지 최대한 빨리 확인하고 싶다면 가장 간단한 방법은 다음과 같습니다.

```bash
hermes
```

이것이 대화형 Hermes CLI를 시작합니다. 처음 Hermes를 사용하는 사용자에게도 가장 권장되는 시작점입니다. 가장 필수적인 것들을 먼저 확인할 수 있기 때문입니다.

- 명령어가 실제로 사용 가능한지
- 현재 모델 설정이 올바르게 작동하는지
- 터미널 툴체인이 올바르게 작동하는지
- 상호 작용 스타일이 필요에 맞는지

### 첫 번째 설정에 충분한 명령어들

Hermes README에는 여러 고빈도 명령어가 나열되어 있으며, 함께 실용적인 첫 사용 경로를 형성합니다.

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

각각이 무엇을 하는지 확신이 없다면 이렇게 기억하세요.

- `hermes model`: 모델을 선택하거나 전환합니다
- `hermes tools`: 현재 사용 가능한 도구를 보고 구성합니다
- `hermes config set`: 특정 설정 항목을 변경합니다
- `hermes setup`: 전체 초기화 마법사를 한 번 실행합니다
- `hermes update`: Hermes를 업데이트합니다
- `hermes doctor`: 문제를 해결합니다

초보자에게 가장 실용적인 순서는 일반적으로 다음과 같습니다.

1. 먼저 `hermes model`을 실행합니다
2. 모든 일반적인 옵션을 한 번에 구성하고 싶다면 `hermes setup`을 실행합니다

## Hermes를 사용하는 두 가지 가장 일반적인 방법

### 1. 터미널에서 일일 개발 어시스턴트로 Hermes 사용

CLI 모드는 다음 시나리오에 잘 맞습니다.

- 로컬에서 코드를 작성하면서 직접 질문하기
- 프로젝트 검사, 파일 편집, 명령어 실행
- 일회성 디버깅 또는 리뷰 작업
- 현재 작업 디렉토리에서 지속적으로 협업

가장 큰 장점은 가장 짧은 경로라는 것입니다. 추가적인 플랫폼 통합이 필요 없고, 사전에 봇 구성을 처리할 필요가 없으며, 처음 사용 습관을 구축하는 가장 좋은 방법입니다.

### 2. 메시징 플랫폼을 통해 Hermes 사용

Feishu, Telegram, Discord와 같은 플랫폼에서 Hermes와 대화하려면 메시징 게이트웨이를 사용해야 합니다.

가장 일반적인 진입 명령어는 다음과 같습니다.

```bash
hermes gateway setup
hermes gateway
```

구체적으로:

- `hermes gateway setup`은 대화형 플랫폼 구성에 사용됩니다
- `hermes gateway`는 게이트웨이 프로세스를 시작하는 데 사용됩니다

공식 문서에 따르면 게이트웨이는 구성된 플랫폼을 연결하고, 세션을 관리하며, cron 작업과 같은 기능을 처리하는 통합 백그라운드 프로세스입니다.

## Feishu를 예시로: Hermes를 메시징 플랫폼에 연결하는 방법

일상 업무의 대부분이 Feishu에서 이루어진다면, Feishu/Lark를 통한 Hermes 사용은 매우 자연스러운 방법입니다.

### 최소 실행 가능한 통합 경로

공식 문서에서는 Feishu/Lark에 대해 다음 진입 명령어를 권장합니다.

```bash
hermes gateway setup
```

실행 후 마법사에서 **Feishu / Lark**를 선택하기만 하면 됩니다.

Feishu 문서에는 두 가지 연결 모드가 설명되어 있습니다.

- `websocket`: 권장됨
- `webhook`: 선택 사항

Hermes가 노트북, 워크스페이션 또는 개인 서버에서 실행 중이라면, `websocket`을 먼저 사용하는 것이 일반적으로 더 간단합니다. 공개 콜백 URL을 노출할 필요가 없기 때문입니다.

### 수동으로 구성하는 경우 최소한 알아야 할 변수

마법사를 사용하지 않고 수동으로 구성을 작성하는 경우, Feishu 문서에는 다음 핵심 변수가 나열되어 있습니다.

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

그 중 두 가지는 특별한 주의가 필요합니다.

- `FEISHU_ALLOWED_USERS`: 권장됩니다. 봇에 접근할 수 있는 모든 사람이 직접 사용할 수 있는 것은 아니기 때문입니다.
- `FEISHU_HOME_CHANNEL`: cron 결과나 기본 알림을 받을 홈 채팅을 미리 정의할 수 있습니다.

### Hermes가 Feishu 그룹 채팅에서 때때로 응답하지 않는 이유

이 세부 사항은 간과하기 쉽습니다. Feishu 그룹 채팅에서 Hermes는 기본적으로 모든 메시지에 응답하지 않습니다.

공식 문서에 명확하게 설명되어 있습니다.

- 다이렉트 메시지에서 Hermes는 메시지에 응답합니다
- 그룹 채팅에서는 봇을 명시적으로 `@` 멘션해야 메시지를 처리합니다

Feishu 대화를 홈 채널로 설정하려면 채팅에서 다음을 사용할 수도 있습니다.

```text
/set-home
```

또는 미리 구성에서 정의할 수 있습니다.

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## 초보자가 먼저 기억해야 할 Hermes 명령어

CLI에서든 메시징 플랫폼에서든 Hermes를 사용할 때, 다음 명령어만 기억해도 시작하기에 충분합니다.

- `/new` 또는 `/reset`: 새 세션 시작
- `/model`: 모델 보기 또는 전환
- `/retry`: 이전 턴 재시도
- `/undo`: 이전 상호 작용 취소
- `/compress`: 컨텍스트 수동 압축
- `/help`: 도움말 보기

주로 메시징 플랫폼에서 Hermes를 사용한다면 하나 더 기억하세요.

- `/sethome` 또는 `/set-home`: 현재 채팅을 홈 채널로 설정

이러한 명령어는 가장 일반적인 초보자 단계 작업을 포함합니다. 재시작, 조정, 롤백, 확인, 계속하기.

## 자주 묻는 질문

### Windows에 Hermes를 직접 설치할 수 있나요?

아니요. 현재 공식 문서에 네이티브 Windows는 지원되지 않으며 WSL2가 권장된다고 명확하게 나와 있습니다.

### 설치 후 `hermes`를 입력해도 아무 일도 일어나지 않으면 어떻게 해야 하나요?

다음 순서로 문제 해결하는 것이 가장 좋습니다.

1. 먼저 셸을 다시 로드합니다 (예: `source ~/.bashrc`)
2. `hermes`를 다시 실행합니다
3. 여전히 비정상이면 `hermes doctor`를 실행합니다

### Feishu 그룹에서 봇이 응답하지 않으면 어떻게 해야 하나요?

먼저 다음 세 가지를 확인하세요.

- 그룹에서 Hermes를 `@` 멘션했는지
- `FEISHU_ALLOWED_USERS`가 현재 사용자를 제한하는지
- 현재 그룹 채팅 정책이 그룹 메시지 처리를 허용하는지

공식 Feishu 문서에 따르면, 그룹 채팅 시나리오에서는 명시적인 @멘션 사용이 필요합니다.

## 권장 다음 단계

Hermes를 최대한 빨리 사용하기 시작하고 싶다면, 가장 권장되는 순서는 다음과 같습니다.

1. 먼저 설치 명령어 실행
2. 먼저 로컬 CLI에서 `hermes`로 시작
3. `hermes model`과 `hermes setup`으로 기본 설정 완료
4. Feishu에서 계속 사용하고 싶다면 `hermes gateway setup` 구성

이 글이 시리즈의 첫 번째 부분이라면, 가장 좋은 역할은 모든 고급 기능을 한 번에 설명하는 것이 아니라 사용자를 먼저 문턱에 들여오는 것입니다.

다음 주제는 후속 글로 나누는 것이 더 좋습니다.

- 완전한 Hermes Feishu 통합 가이드
- 일반적인 Hermes 슬래시 명령어 가이드
- Hermes 게이트웨이 구성 및 문제 해결 가이드

Hermes 콘텐츠를 계속 만들 계획이라면, 이 글은 이후 글의 시작점으로 활용할 수 있으며, 내부 링크 구조를 점진적으로 구축할 수 있습니다.
