데스크탑 설치 및 최초 실행 후 HagiCode는 자동으로 새로운 초기화 마법사를 엽니다. 현재 버전은 더 이상 이전의 긴 다단계 가이드를 따르지 않습니다. 대신 기본 설정을 완료하고, 하나 이상의 프로젝트를 만들고, 실제 작업 공간에 들어가기 전에 커뮤니티 진입점을 근처에 유지하는 3가지 주요 단계로 구성됩니다.

:::note[이 페이지를 읽는 방법]
이 재작성은 아래의 관리되는 스크린샷 라이브러리를 사용합니다. `repos/docs/src/content/docs/img/screenshots/quick-start/wizard-setup/` 진실의 근원으로서. 현재 관리되는 세트에는 3개의 스크린샷만 포함되어 있습니다. `Foundation settings` 단계이므로 이 페이지에서는 해당 부분을 자세히 설명하고 이후 2단계를 높은 수준으로 유지합니다.
:::

## 마법사 개요

현재 초기화 마법사는 다음과 같은 3가지 주요 단계를 거칩니다.

1. **기본 설정**: 인터페이스 언어를 선택하고, 테마를 선택하고, 종속성을 확인하고, 존재하지 않는 경우 기본 에이전트를 만듭니다.
2. **프로젝트 생성**: 마법사 출력이 실제 작업 디렉터리에 연결되도록 프로젝트를 하나 이상 생성합니다.
3. **커뮤니티 가이드**: 일반 작업 공간에 들어가기 전에 GitHub 및 기타 후속 진입점을 손이 닿는 곳에 두세요.

## 1단계: 기초 설정

왼쪽 탐색은 이미 첫 번째 단계에 다음과 같은 라벨을 붙였습니다. `基础设置` 목적을 명확하게 요약합니다. 인터페이스 언어를 선택하고, 스프링 테마를 유지하고, 확인합니다. `node`, `npm`, `git`, 그리고 `OpenSpec`, 기본값을 생성합니다. `HagiCoder` 아직 영웅이 없다면. 결론: 이 단계는 오래되고 분산된 준비 작업을 하나의 연속적인 설정 표면으로 병합합니다.

### 1.1 인터페이스 언어 및 테마 선택

![초기화 마법사 기초 설정: 언어 및 테마](../../img/screenshots/quick-start/wizard-setup/guide1-language-theme/original.png)

첫 번째 스크린샷은 파운데이션 설정 화면의 상단을 보여줍니다. 그것은 다음으로 시작한다 `选择界面语言`, 도우미 텍스트는 범위를 명시적으로 만듭니다. 이렇게 하면 Hero의 언어 스타일이 아닌 **인터페이스 복사본만** 변경됩니다. 예제에서는 `简体中文` 다음과 같이 표시됩니다. `当前`, 동안 `English` 오른쪽에는 계속 사용 가능합니다.

같은 화면이 즉시 계속됩니다. `选择主题`. 현재 빌드는 spring light 및 spring dark 변형을 노출하며 예시에서는 올바른 테마 카드를 다음과 같이 표시합니다. `当前`. 주요 변경 사항은 구조적입니다. 언어와 테마는 이제 이전 가이드에서 별도의 점프 대신 하나의 연속 섹션으로 처리됩니다.

### 1.2 종속성을 확인하고 근처에 설치 작업을 유지합니다.

![초기화 마법사 기초 설정: 종속성 확인](../../img/screenshots/quick-start/wizard-setup/guide2-dependency-check/original.png)

두 번째 스크린샷은 다음으로 스크롤됩니다. `组件检测`. 이 영역은 자동으로 확인됩니다. `Node.js`, `npm`, `Git`, 그리고 `OpenSpec`, 각 항목을 현재 가용성으로 렌더링합니다. 오른쪽 상단 모서리에는 `重新检测` 행동, 그리고 `OpenSpec` 행은 또한 `安装 OpenSpec` 입장.

논리는 직접적입니다. 마법사는 모든 필수 구성 요소를 수동으로 기억하도록 강요하는 대신 현재 시스템이 준비되었는지 여부를 알려줍니다. 이러한 핵심 구성 요소를 사용할 수 있게 되면 기본 에이전트 생성을 계속한 다음 프로젝트 생성으로 이동할 수 있습니다.

### 1.3 가장 많이 사용되는 CLI와 모델을 선택한 후 기본 에이전트 생성

![초기화 마법사 기초 설정: 기본 에이전트 안내](../../img/screenshots/quick-start/wizard-setup/guide3-default-agent-cli-model/original.png)

세 번째 스크린샷은 다음과 같습니다. `默认Agent引导`. 아직 에이전트가 없으면 마법사는 가장 많이 사용되는 에이전트를 묻습니다. `AgentCLI` 먼저 모델을 지정하고 기본 에이전트를 자동 생성하여 이후 구성 그룹으로 전달합니다. 예시에서 선택된 기본값은 다음과 같습니다. `Claude Code` 그리고 `GLM 5.1`.

이 화면은 기본 ID도 처리합니다. 아래쪽 영역에서는 아바타를 미리 볼 수 있으며 `换一组` 작업을 수행하고 기본 CTA를 오른쪽 하단에 배치합니다. `确认并创建 HagiCoder`. 따라서 현재 마법사는 더 이상 이전 다중 화면 역할 테이블을 먼저 열지 않습니다. 먼저 사용 가능한 기본 실행기가 하나 존재하도록 보장합니다.

## 2단계: 프로젝트 생성

왼쪽 탐색은 두 번째 단계를 다음과 같이 표시합니다. `创建项目`. 페이지 수준 요약은 이미 그 역할을 명시하고 있습니다. 이는 우연한 선택 중지가 아닙니다. 기초 설정에서 생성된 언어, 테마, 종속성 상태 및 기본 에이전트가 실제 작업 디렉터리에 연결할 수 있도록 **최소 하나 이상의 프로젝트**를 만들어야 합니다.

현재 관리되는 스크린샷 세트에는 아직 이 단계에 대한 전용 스크린샷이 포함되어 있지 않으므로 이 페이지에서는 지금은 높은 수준의 설명을 유지합니다. 이 페이지에 도달하면 계속 진행하기 전에 실제 저장소나 작업 폴더를 연결하세요.

## 3단계: 커뮤니티 가이드

왼쪽 탐색의 세 번째 단계는 다음과 같습니다. `社区引导`. 헤더 요약에는 실제 작업 공간에 들어가기 전에 기본 설정을 완료하고, 하나 이상의 프로젝트를 생성하고, **커뮤니티 진입점을 근처에 유지**하는 순서가 명확하게 설명되어 있습니다.

현재 관리되는 스크린샷 세트에는 이 단계에 대한 전용 스크린샷도 부족하기 때문에 문서에서는 여기서 의미하는 흐름만 유지합니다. 즉, 핵심 초기화 결정을 위한 장소가 아니라 후속 진입점을 위한 종료 단계입니다.
