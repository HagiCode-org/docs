Windows 사용자에게는 Microsoft Store가 현재 공개 데스크톱 진입점입니다. 이 문서는 Store 경로만 설명합니다. 일반 Desktop 다운로드, 버전 관리, 문제 해결은 [Desktop Installation](/ko-KR/installation/desktop)을 확인하세요.

Steam 메인 앱 항목은 이미 퇴역했기 때문에 현재 Windows의 주요 배포 채널은 Microsoft Store입니다. 배경은 [Why HagiCode is not available on Steam](/ko-KR/faq/steam-distribution-status/)에서 확인할 수 있습니다.

:::tip[이 경로가 잘 맞는 경우]
- Windows 10 또는 Windows 11을 사용하며 시스템 스토어에서 바로 설치하고 싶다
- 설치 파일을 수동으로 다루기보다 Store 채널로 앱 패키지를 받고 싶다
- 개인 로컬 환경만 필요하고 Docker Compose 배포는 계획하지 않는다
:::

## 시작 전에 확인할 사항

- 장치에서 [HagiCode for Windows의 Microsoft Store 페이지](https://apps.microsoft.com/detail/9N3PM0N3SVDW)를 열 수 있어야 합니다
- 첫 실행 시 추가 런타임 구성요소를 준비할 수 있으므로 네트워크 연결이 가능해야 합니다
- 장치, 지역, 조직 정책에 따라 Microsoft 계정 로그인을 먼저 요구할 수 있습니다
- Store 접근이 막힌 환경이라면 [Desktop Installation](/ko-KR/installation/desktop)으로 전환하세요

## Microsoft Store에서 설치하기

1. [HagiCode for Windows](https://apps.microsoft.com/detail/9N3PM0N3SVDW) 페이지를 엽니다.
2. `받기`, `설치`, 또는 Store가 표시하는 동등한 버튼을 누릅니다.
3. Microsoft Store가 다운로드와 설치를 마칠 때까지 기다립니다.
4. 완료되면 시작 메뉴 또는 Store 페이지에서 HagiCode를 실행합니다.

## 첫 실행 시 일어나는 일

설치가 끝나도 HagiCode는 일반 Desktop 초기화 흐름을 계속 진행합니다.

- 첫 실행은 로컬 실행 환경을 준비해야 해서 일반 앱보다 느리게 보일 수 있습니다
- Microsoft Store / MSIX 채널은 일부 의존성 관리 및 런타임 경로 설정을 의도적으로 잠급니다
- 앱은 자체적으로 독립 업데이트를 수행하지 않으며, 패키지 전달은 설치 채널에 따라 달라집니다. Store 설치의 앱 패키지 전달은 Microsoft Store 채널에 연결됩니다

온보딩을 계속하려면 [Wizard Setup](/ko-KR/quick-start/wizard-setup)을 확인하세요.

## 다른 설치 경로를 써야 하는 경우

- Microsoft Store에 접근할 수 없거나 조직 정책상 Store 설치가 금지된 경우
- `.exe` 또는 `.msix` 파일을 직접 내려받아 설치 출처를 스스로 관리해야 하는 경우
- 서버, NAS, 팀 환경에 배포하려는 경우로, 이때는 [Docker Compose Deployment](/ko-KR/installation/docker-compose)가 더 적합합니다

## 문제 해결

### Microsoft Store 페이지가 열리지 않음

먼저 네트워크, 지역 설정, Microsoft Store 로그인 상태를 확인하세요. 브라우저 링크는 열리지만 Store 클라이언트를 사용할 수 없다면 로컬 시스템 정책이 원인인 경우가 많습니다. 이때는 [Desktop Installation](/ko-KR/installation/desktop)으로 전환하세요.

### 일부 설정이 잠겨 있음

Store / MSIX 설치에서는 일부 의존성 관리 및 런타임 데이터 경로 옵션이 잠깁니다. 이는 설치 실패가 아니라 배포 채널의 제약입니다.

### 첫 실행 때 추가 다운로드나 준비가 계속됨

Store 패키지는 데스크톱 애플리케이션 패키지만 전달합니다. 첫 실행 시 HagiCode는 버전 메타데이터 확인, 로컬 구성요소 준비, 기본 초기화를 계속할 수 있습니다. 네트워크를 유지한 채 현재 단계가 끝날 때까지 기다리세요.
