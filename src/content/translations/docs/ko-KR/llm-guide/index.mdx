## LLM 가이드가 필요한 이유

코드 생성 워크플로에서 모델 성능은 품질, 대기 시간, 비용에 따라 크게 다릅니다. 이 범주는 개발자가 모델을 더 빠르게 선택할 수 있도록 일관된 평가 구조를 제공합니다.

## 권장 읽기 경로

1. 다음으로 시작 [HagiCode의 모델 비교 및 평가](./model-comparison-evaluation/).
2. 우선순위(품질 우선, 비용 우선 또는 속도 우선)에 따라 후보를 선택하세요.
3. 프로덕션 사용에 결론을 적용하기 전에 테스트 시간 메모를 확인하십시오.

## 유지 관리 원칙

- 모델 기능이 크게 변경되면 결론을 새로 고칩니다.
- 한 달에 한 번 이상 테스트 시간 컨텍스트를 재검증합니다.
- 섹션 구조에서 중국어와 영어 페이지를 정렬하세요.
