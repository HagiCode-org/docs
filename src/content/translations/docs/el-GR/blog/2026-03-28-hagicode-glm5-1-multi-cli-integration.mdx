## Οδηγός ενσωμάτωσης Hagicode και GLM-5.1 Multi-CLI

:::note[Ειδοποίηση για το χρονικό πλαίσιο]
Αυτό το άρθρο τεκμηριώνει τη σχεδίαση ολοκλήρωσης πολλαπλών CLI και την αντιστοίχιση μοντέλων όπως υπήρχαν τη στιγμή της δημοσίευσης. Για το τρέχον πάντα υποστηριζόμενο εύρος, ανατρέξτε στο [Εγκατάσταση AI Agent CLI](/el-GR/related-software-installation/ai-agent-cli/) και το [Οδηγός εγκατάστασης](/el-GR/installation); το `IFlowCli` που αναφέρεται εδώ αντιπροσωπεύει μόνο μια ιστορική διαμόρφωση εκείνη τη στιγμή και δεν αποτελεί μέρος του τρέχοντος πεδίου ενεργής υποστήριξης.
:::

## φόντο

Στο έργο Hagicode, οι χρήστες μπορούν να επιλέξουν από πολλά εργαλεία CLI για να οδηγήσουν βοηθούς προγραμματισμού AI, συμπεριλαμβανομένων των Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI και άλλων. Αυτά τα εργαλεία CLI είναι εργαλεία προγραμματισμού τεχνητής νοημοσύνης γενικής χρήσης από μόνα τους, αλλά μέσω του στρώματος αφαίρεσης του Hagicode, μπορούν να συνδεθούν ευέλικτα με διαφορετικούς παρόχους μοντέλων τεχνητής νοημοσύνης.

Το Zhipu AI (ZAI) παρέχει μια διεπαφή συμβατή με το Anthropic Claude API, επιτρέποντας σε αυτά τα εργαλεία CLI να χρησιμοποιούν απευθείας τα εγχώρια μοντέλα της σειράς GLM. Μεταξύ αυτών, το GLM-5.1 είναι η τελευταία έκδοση μοντέλων μεγάλης γλώσσας της Zhipu, με σημαντικές βελτιώσεις σε σχέση με το GLM-5.0.

## Η αρχιτεκτονική αφαίρεσης CLI του Hagicode

Το Hagicode ορίζει 11 τύπους παρόχων CLI μέσω του `AIProviderType` enum, που καλύπτει τα βασικά εργαλεία προγραμματισμού AI CLI:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Κάθε CLI έχει αντίστοιχη διαμόρφωση παραμέτρων μοντέλου και υποστηρίζει το `model` και `reasoning` παράμετροι:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],
        [AIProviderType.CodexCli] = ["model", "reasoning"],
        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],
        [AIProviderType.HermesCli] = ["model", "reasoning"],
        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini does not support the reasoning parameter
        // ...
    };
```

## Σύστημα υποστήριξης μοντέλου GLM

Ο Κατάλογος Secondary Professions της Hagicode ορίζει την πλήρη υποστήριξη για τη σειρά μοντέλων GLM:

| Αναγνωριστικό μοντέλου | Όνομα | Προεπιλεγμένη συλλογιστική | Συμβατές οικογένειες CLI |
|----------|------|-------------------|-------------------------|
| `glm-4.7` | GLM 4.7 | ψηλά | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | ψηλά | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM 5 Turbo | ψηλά | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (παλαιού τύπου) | ψηλά | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | ψηλά | claude, codebuddy, hermes, qoder, kiro |

## Χρήση του GLM-5.1 σε διαφορετικά CLI

### 1. Κωδικός Claude CLI + GLM-5.1

Το Claude Code CLI είναι ένα από τα βασικά CLI της Hagicode και έχει ρυθμιστεί μέσω του συστήματος διαμόρφωσης Hero:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Αντίστοιχη `HeroEquipmentCatalogItem` διαμόρφωση:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

Το OpenCode CLI είναι το πιο ευέλικτο CLI και υποστηρίζει τον καθορισμό οποιουδήποτε μοντέλου στο `provider/model` μορφή:

**Μέθοδος 1: Χρησιμοποιήστε το πρόθεμα παρόχου ZAI**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Μέθοδος 2: Χρησιμοποιήστε απευθείας το αναγνωριστικό μοντέλου**
```json
{
  "model": "glm-5.1"
}
```

**Μέθοδος 3: Διεπαφή χρήστη διαμόρφωσης διεπαφής**

Σε `HeroModelEquipmentForm.tsx`, το OpenCode CLI έχει μια ειδική υπόδειξη για το σύμβολο κράτησης θέσης:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Οι χρήστες μπορούν να εισάγουν:
```
zai/glm-5.1
glm-5.1
```

**Λογική ανάλυσης μοντέλου OpenCode CLI**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Το Codebuddy CLI διαθέτει αποκλειστική λογική χειρισμού παλαιού τύπου:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Σημείωση**: Το Codebuddy διατηρεί ειδικό χειρισμό για το GLM-5.0 και δεν χρησιμοποιεί κανονικοποίηση παλαιού τύπου:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## Πλεονεκτήματα βελτίωσης του GLM-5.1

Σε σύγκριση με το GLM-5.0, το GLM-5.1 φέρνει τις ακόλουθες σημαντικές βελτιώσεις:

### 1. Καλύτερη ικανότητα συλλογισμού

Σύμφωνα με τις επίσημες πληροφορίες έκδοσης του Zhipu, οι βελτιώσεις στο GLM-5.1 περιλαμβάνουν:

- **Ισχυρότερη κατανόηση κώδικα**: Πιο ακριβής ανάλυση σύνθετων δομών κώδικα
- **Μεγαλύτερη κατανόηση περιβάλλοντος**: Υποστηρίζει μεγαλύτερο πλαίσιο συνομιλίας
- **Βελτιωμένη κλήση εργαλείου**: Υψηλότερο ποσοστό επιτυχίας για κλήσεις εργαλείων MCP
- **Σταθερότητα εξόδου**: Μειώνει την τυχαιότητα και τις παραισθήσεις

### 2. Ολοκληρωμένη συμβατότητα πολλαπλών CLI

Το GLM-5.1 καλύπτει όλα τα mainstream CLI που υποστηρίζονται από το Hagicode:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Σημειώσεις

### 1. Διαμόρφωση κλειδιού API

Βεβαιωθείτε ότι το `ANTHROPIC_AUTH_TOKEN` η μεταβλητή περιβάλλοντος έχει ρυθμιστεί σωστά. Είναι το απαιτούμενο διαπιστευτήριο για κάθε CLI για να συνδεθεί στο μοντέλο.

### 2. Διαθεσιμότητα μοντέλου

Το GLM-5.1 πρέπει να ενεργοποιηθεί από τον αντίστοιχο πάροχο μοντέλου:
- Η πλατφόρμα Zhipu AI ZAI το υποστηρίζει από προεπιλογή
- Το Alibaba Cloud DashScope ενδέχεται να απαιτεί ξεχωριστή εφαρμογή

### 3. Μορφή OpenCode CLI

Όταν χρησιμοποιείτε το `provider/model` μορφή, βεβαιωθείτε ότι το αναγνωριστικό παρόχου είναι σωστό:
- Zhipu AI: `zai` ή `zhipuai`
- Alibaba Cloud: `aliyun` ή `dashscope`

### 4. Παράμετρος συλλογισμού

- `high` συνιστάται για τα καλύτερα αποτελέσματα δημιουργίας κώδικα
- Το Gemini CLI δεν υποστηρίζει την παράμετρο συλλογισμού και θα αγνοήσει αυτόματα αυτήν τη διαμόρφωση

## Συμπέρασμα

Μέσω ενός ενοποιημένου στρώματος αφαίρεσης, το Hagicode επιτρέπει την ευέλικτη ενσωμάτωση μεταξύ του GLM-5.1 και πολλαπλών CLI. Οι προγραμματιστές μπορούν να επιλέξουν το εργαλείο CLI που ταιριάζει καλύτερα στις προτιμήσεις και τα σενάρια χρήσης τους και, στη συνέχεια, να χρησιμοποιήσουν το πιο πρόσφατο μοντέλο GLM-5.1 μέσω απλής διαμόρφωσης.

Ως τελευταία έκδοση μοντέλου της Zhipu, το GLM-5.1 προσφέρει σαφείς βελτιώσεις σε σχέση με το GLM-5.0:
- Ένα ανεξάρτητο αναγνωριστικό έκδοσης χωρίς επιβάρυνση παλαιού τύπου
- Ισχυρότερος συλλογισμός και κατανόηση κώδικα
- Ευρεία συμβατότητα πολλαπλών CLI
- Ευέλικτη διαμόρφωση επιπέδου συλλογισμού

Με τις σωστές μεταβλητές περιβάλλοντος και τον εξοπλισμό Hero διαμορφωμένο, οι χρήστες μπορούν να ξεκλειδώσουν πλήρως την ισχύ του GLM-5.1 σε διαφορετικά περιβάλλοντα CLI.
