## Η πληκτρολόγηση είναι πιο αργή από την ομιλία και η ομιλία είναι πιο αργή από ένα στιγμιότυπο οθόνης - Πρακτικές πολυτροπικής εισαγωγής για βοηθούς κωδικοποίησης AI

> Η σύνταξη κώδικα έχει όριο ταχύτητας ανεξάρτητα από το πόσο γρήγορα πληκτρολογείτε. Μερικές φορές κάτι που θα μπορούσατε να πείτε σε μια πρόταση χρειάζεται για πάντα για να πληκτρολογήσετε. Μερικές φορές ένα στιγμιότυπο οθόνης εξηγεί τα πάντα, ωστόσο πρέπει να το περιγράψετε με ένα σωρό κείμενο. Αυτό το άρθρο μιλάει για το τι συναντήσαμε κατά την κατασκευή [HagiCode](https://hagicode.com), από την αναγνώριση ομιλίας έως τη μεταφόρτωση εικόνων. Στο τέλος, θέλαμε απλώς να κάνουμε έναν βοηθό κωδικοποίησης AI λίγο πιο εύκολο στη χρήση.

## φόντο

Κατά τη δημιουργία του HagiCode, παρατηρήσαμε ένα πρόβλημα - ή μάλλον, ένα πρόβλημα που εμφανίστηκε φυσικά όταν οι άνθρωποι άρχισαν να το χρησιμοποιούν σε μεγάλο βαθμό: το να βασίζονται μόνο στην πληκτρολόγηση μπορεί να είναι κουραστικό.

Σκεφτείτε το: η αλληλεπίδραση μεταξύ των χρηστών και του Agent είναι ένα βασικό σενάριο. Αλλά αν κάθε ανταλλαγή απαιτεί συνεχή πληκτρολόγηση στο πληκτρολόγιο, η αποτελεσματικότητα δεν είναι μεγάλη:

1. **Η πληκτρολόγηση είναι πολύ αργή**: Για περίπλοκα ζητήματα, όπως μηνύματα σφάλματος ή προβλήματα διεπαφής χρήστη, η πληκτρολόγηση όλων των στοιχείων μπορεί να διαρκέσει μισό λεπτό, ενώ η εκφώνηση του δυνατού μπορεί να διαρκέσει δέκα δευτερόλεπτα. Αυτό το κενό είναι πραγματικό.

2. **Οι εικόνες είναι πιο άμεσες**: Μερικές φορές η διεπαφή χρήστη παρουσιάζει ένα σφάλμα, μερικές φορές θέλετε να συγκρίνετε ένα προσχέδιο σχεδίασης και μερικές φορές χρειάζεται να εμφανίσετε μια δομή κώδικα. «Μια εικόνα αξίζει όσο χίλιες λέξεις» μπορεί να είναι ένα παλιό ρητό, αλλά εξακολουθεί να αντέχει. Το να αφήνουμε την τεχνητή νοημοσύνη να «δει» άμεσα το πρόβλημα είναι πολύ πιο ξεκάθαρο από το να το περιγράφουμε για αιώνες.

3. **Η αλληλεπίδραση πρέπει να είναι φυσική**: Οι σύγχρονοι βοηθοί τεχνητής νοημοσύνης θα πρέπει να υποστηρίζουν κείμενο, φωνή και εικόνες. Οι χρήστες θα πρέπει να μπορούν να επιλέξουν οποιαδήποτε μέθοδο εισαγωγής θεωρούν πιο φυσική.

Αποφασίσαμε λοιπόν να προσθέσουμε την αναγνώριση ομιλίας και την υποστήριξη μεταφόρτωσης εικόνων στο HagiCode για να κάνουμε πιο βολικές τις αλληλεπιδράσεις Agent. Εάν οι χρήστες μπορούν να πληκτρολογήσουν λίγο λιγότερο, αυτό είναι ήδη μια νίκη.

## Ανάλυση

### Τεχνικές Προκλήσεις στην Αναγνώριση Ομιλίας

Κατά τη δημιουργία της αναγνώρισης ομιλίας, αντιμετωπίσαμε ένα δύσκολο πρόβλημα: **το API WebSocket του προγράμματος περιήγησης δεν υποστηρίζει προσαρμοσμένες κεφαλίδες HTTP**.

Η υπηρεσία αναγνώρισης ομιλίας που επιλέξαμε ήταν το Doubao Speech Recognition API της ByteDance. Δυστυχώς, αυτό το API απαιτεί πληροφορίες ελέγχου ταυτότητας όπως π.χ `accessToken` και `secretKey` να περάσουν μέσω των κεφαλίδων HTTP. Αυτό δημιούργησε μια άμεση τεχνική σύγκρουση:

```javascript
// The browser WebSocket API does not support this approach
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // Not supported
  }
});
```

Βασικά είχαμε δύο επιλογές:

1. **Προσέγγιση παραμέτρων ερωτήματος URL**: τοποθετήστε τις πληροφορίες ελέγχου ταυτότητας στη διεύθυνση URL
   - Το πλεονέκτημα είναι ότι είναι απλό στην εφαρμογή του
   - Το μειονέκτημα είναι ότι τα διαπιστευτήρια εκτίθενται στο frontend, το οποίο είναι ανασφαλές. Ορισμένα API απαιτούν επίσης έλεγχο ταυτότητας βάσει κεφαλίδας

2. **Προσέγγιση διακομιστή μεσολάβησης Backend**: εφαρμόστε έναν διακομιστή μεσολάβησης WebSocket στο backend
   - Το πλεονέκτημα είναι ότι τα διαπιστευτήρια παραμένουν αποθηκευμένα με ασφάλεια στο backend και η λύση είναι πλήρως συμβατή με τις απαιτήσεις API
   - Το μειονέκτημα είναι ότι η υλοποίηση είναι λίγο πιο περίπλοκη

Στο τέλος, επιλέξαμε την **προσέγγιση διακομιστή μεσολάβησης υποστήριξης**. Η ασφάλεια δεν είναι κάτι για το οποίο συμβιβάζεστε.

### Λειτουργικές απαιτήσεις για μεταφορτώσεις εικόνων

Οι απαιτήσεις μας για μεταφορτώσεις εικόνων ήταν στην πραγματικότητα αρκετά απλές:

1. **Πολλαπλές μέθοδοι μεταφόρτωσης**: κάντε κλικ για να επιλέξετε ένα αρχείο, να κάνετε μεταφορά και απόθεση και επικόλληση από το πρόχειρο
2. **Επικύρωση αρχείου**: οι περιορισμοί τύπου (PNG, JPG, WebP, GIF) και τα όρια μεγέθους (5-10 MB) είναι βασικές απαιτήσεις
3. **Εμπειρία χρήστη**: μεταφορτώστε την πρόοδο, τις προεπισκοπήσεις και τα μηνύματα σφάλματος, ώστε οι χρήστες να γνωρίζουν πάντα τι συμβαίνει
4. **Ασφάλεια**: Η επικύρωση από την πλευρά του διακομιστή και η προστασία από κακόβουλες μεταφορτώσεις αρχείων είναι απαραίτητες

## Λύση

### Αναγνώριση ομιλίας: WebSocket Proxy Architecture

Σχεδιάσαμε μια αρχιτεκτονική τριών επιπέδων για την αναγνώριση ομιλίας και βρήκαμε μια διαδρομή που λειτούργησε:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**Εφαρμογές βασικών στοιχείων**:

1. **Επεξεργαστής Frontend AudioWorklet**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Resample to 16 kHz (required by the Doubao API)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Accumulate samples into 500 ms chunks
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Convert to 16-bit PCM and send
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **Χειριστής WebSocket Backend** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **Μπροστά `VoiceTextArea` συστατικό**:

```tsx
export const VoiceTextArea = forwardRef<HTMLTextAreaElement, VoiceTextAreaProps>(
```

### Μεταφορτώσεις εικόνων: Στοιχείο μεταφόρτωσης πολλαπλών μεθόδων

Δημιουργήσαμε ένα στοιχείο μεταφόρτωσης εικόνων με πλήρεις δυνατότητες με υποστήριξη και για τις τρεις μεθόδους μεταφόρτωσης, καλύπτοντας τα πιο συνηθισμένα σενάρια που αντιμετωπίζουν οι χρήστες.

**Βασικά χαρακτηριστικά**:

1. **Τρεις μέθοδοι μεταφόρτωσης**:

```tsx
// Click to upload
const handleClick = () => fileInputRef.current?.click();

// Drag-and-drop upload
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Clipboard paste
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **Επικύρωση διεπαφής**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **Εργαλείο χειρισμού μεταφόρτωσης backend** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Validation
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Save file
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

### Πώς να χρησιμοποιήσετε την Αναγνώριση ομιλίας

1. **Διαμόρφωση της υπηρεσίας αναγνώρισης ομιλίας**:
   - Ανοίξτε τη σελίδα ρυθμίσεων αναγνώρισης ομιλίας
   - Διαμορφώστε την ομιλία Doubao `AppId` και `AccessToken`
   - Προαιρετικά διαμορφώστε τις λέξεις κλειδιά για να βελτιώσετε την ακρίβεια αναγνώρισης για όρους συγκεκριμένους τομέα

2. **Χρησιμοποιήστε το στο πλαίσιο εισαγωγής**:
   - Κάντε κλικ στο εικονίδιο του μικροφώνου στην αριστερή πλευρά του πλαισίου εισόδου
   - Ξεκινήστε να μιλάτε αφού εμφανιστεί το κινούμενο σχέδιο κυματομορφής
   - Κάντε ξανά κλικ στο εικονίδιο για να σταματήσετε την εγγραφή
   - Το αναγνωρισμένο κείμενο εισάγεται αυτόματα στη θέση του δρομέα

3. **Παράδειγμα διαμόρφωσης λέξεων-κλειδιών**:

```text
TypeScript
React
useState
useEffect
```

### Πώς να χρησιμοποιήσετε τις μεταφορτώσεις εικόνων

1. **Μέθοδοι μεταφόρτωσης**:
   - Κάντε κλικ στο κουμπί μεταφόρτωση για να επιλέξετε ένα αρχείο
   - Σύρετε μια εικόνα απευθείας στην περιοχή μεταφόρτωσης
   - Χρήση `Ctrl+V` για να επικολλήσετε ένα στιγμιότυπο οθόνης από το πρόχειρο

2. **Υποστηριζόμενες μορφές**: PNG, JPG, JPEG, WebP, GIF
3. **Όριο μεγέθους**: 5 MB από προεπιλογή (με δυνατότητα διαμόρφωσης)

### Σημειώσεις

1. **Αναγνώριση ομιλίας**:
   - Απαιτείται άδεια μικροφώνου
   - Χρησιμοποιήστε το σε ήσυχο περιβάλλον όταν είναι δυνατόν
   - Η μέγιστη υποστηριζόμενη διάρκεια εγγραφής είναι 300 δευτερόλεπτα από προεπιλογή (με δυνατότητα διαμόρφωσης)

2. **Μεταφορτώσεις εικόνων**:
   - Υποστηρίζονται μόνο κοινές μορφές εικόνας
   - Δώστε προσοχή στα όρια μεγέθους αρχείου
   - Οι μεταφορτωμένες εικόνες λαμβάνουν αυτόματα μια διεύθυνση URL προεπισκόπησης

3. **Ζητήματα ασφαλείας**:
   - Τα διαπιστευτήρια αναγνώρισης ομιλίας αποθηκεύονται στο backend
   - Οι μεταφορτώσεις εικόνων περνούν από αυστηρή επικύρωση από την πλευρά του διακομιστή
   - Το HTTPS/WSS συνιστάται σε περιβάλλοντα παραγωγής

## Συμπέρασμα

Είναι σαν να βρίσκω επιτέλους έναν πιο άνετο τρόπο επικοινωνίας.

Κατά τη δημιουργία αυτής της δυνατότητας, αντιμετωπίσαμε το πρόβλημα ότι τα API WebSocket του προγράμματος περιήγησης δεν υποστηρίζουν προσαρμοσμένες κεφαλίδες. Το στοιχείο μεταφόρτωσης εικόνων επωφελείται επίσης από την υποστήριξη πολλαπλών μεθόδων μεταφόρτωσης, επιτρέποντας στους χρήστες να επιλέξουν ό,τι είναι πιο βολικό αυτήν τη στιγμή. Κάνοντας κλικ, σύροντας ή επικολλώντας όλη την εργασία και κάθε διαδρομή ολοκληρώνει τη δουλειά γρήγορα.
