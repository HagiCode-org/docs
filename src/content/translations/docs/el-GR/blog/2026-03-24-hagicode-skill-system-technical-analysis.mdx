## Τεχνική ανάλυση του συστήματος δεξιοτήτων HagiCode: Δημιουργία μιας κλιμακούμενης πλατφόρμας διαχείρισης δεξιοτήτων AI

> Αυτό το άρθρο εξετάζει σε βάθος την αρχιτεκτονική και την εφαρμογή του συστήματος διαχείρισης δεξιοτήτων στο έργο HagiCode, καλύπτοντας τις τεχνικές λεπτομέρειες πίσω από τέσσερις βασικές δυνατότητες: τοπική παγκόσμια διαχείριση, αναζήτηση αγοράς, έξυπνες προτάσεις και διαχείριση αξιόπιστων παρόχων.

## φόντο

Στον τομέα των βοηθών κωδικοποίησης τεχνητής νοημοσύνης, το πώς να επεκταθούν τα όρια των δυνατοτήτων τεχνητής νοημοσύνης ήταν πάντα ένα βασικό ερώτημα. Ο ίδιος ο Claude Code είναι ήδη ισχυρός στη βοήθεια κώδικα, αλλά διαφορετικές ομάδες ανάπτυξης και διαφορετικές στοίβες τεχνολογίας συχνά χρειάζονται εξειδικευμένες δυνατότητες για συγκεκριμένα σενάρια, όπως ο χειρισμός των αναπτύξεων Docker, η βελτιστοποίηση βάσης δεδομένων ή η δημιουργία στοιχείων frontend. Εκεί ακριβώς είναι που ένα σύστημα Δεξιοτήτων γίνεται ιδιαίτερα σημαντικό.

Κατά την ανάπτυξη του έργου HagiCode, αντιμετωπίσαμε μια παρόμοια πρόκληση: πώς αφήνουμε τον Claude Code να «μάθει» νέες επαγγελματικές δεξιότητες όπως θα έκανε ένας άνθρωπος, διατηρώντας παράλληλα μια σταθερή εμπειρία χρήστη και καλή μηχανική συντήρηση; Αυτό το πρόβλημα είναι και δύσκολο και απλό με τον δικό του τρόπο. Γύρω από αυτό το ερώτημα, σχεδιάσαμε και εφαρμόσαμε ένα πλήρες σύστημα διαχείρισης δεξιοτήτων.

Αυτό το άρθρο περιγράφει λεπτομερώς την τεχνική αρχιτεκτονική και την βασική υλοποίηση του συστήματος. Προορίζεται για προγραμματιστές που ενδιαφέρονται για την επεκτασιμότητα AI και την ενσωμάτωση εργαλείων γραμμής εντολών. Μπορεί να σας φανεί χρήσιμο, μπορεί και όχι, αλλά τουλάχιστον είναι γραμμένο τώρα.

## Επισκόπηση Αρχιτεκτονικής Συστήματος

Το σύστημα Skill χρησιμοποιεί μια αρχιτεκτονική διαχωρισμένη από το frontend-backend. Δεν υπάρχει τίποτα ιδιαίτερα μυστηριώδες σε αυτό.

Το **Frontend** χρησιμοποιεί το TanStack Start + React για τη δημιουργία της διεπαφής χρήστη, με κατάσταση διαχείρισης του Redux Toolkit. Οι τέσσερις κύριες δυνατότητες αντιστοιχίζονται απευθείας σε τέσσερα στοιχεία καρτέλας: Τοπικές δεξιότητες, Συλλογή δεξιοτήτων, Έξυπνες προτάσεις και Αξιόπιστοι πάροχοι. Στο τέλος, ο σχεδιασμός αφορά κυρίως τη βελτίωση της εμπειρίας του χρήστη.

Το **Backend** βασίζεται στο ASP.NET Core + ABP Framework, χρησιμοποιώντας Orleans Grain για διαχείριση κατανεμημένων καταστάσεων. Ο ηλεκτρονικός πελάτης API αναδιπλώνει το `IOnlineApiClient` διεπαφή για επικοινωνία με την υπηρεσία καταλόγου δεξιοτήτων εξ αποστάσεως.

Η συνολική αρχιτεκτονική αρχή είναι ο διαχωρισμός της εκτέλεσης εντολών από την επιχειρηματική λογική. Μέσω του μοτίβου προσαρμογέα, οι λεπτομέρειες υλοποίησης της εκτέλεσης εντολών npm/npx κρύβονται μέσα σε ανεξάρτητες μονάδες. Εξάλλου, κανείς δεν θέλει πραγματικά τις κλήσεις γραμμής εντολών διάσπαρτες σε όλη τη βάση κώδικα.

## Βασική Δυνατότητα 1: Τοπική Παγκόσμια Διαχείριση

Η τοπική παγκόσμια διαχείριση είναι η πιο βασική ενότητα. Είναι υπεύθυνο για την καταχώριση εγκατεστημένων δεξιοτήτων και την υποστήριξη λειτουργιών απεγκατάστασης. Δεν υπάρχει τίποτα υπερβολικά περίπλοκο εδώ. είναι κυρίως να κάνεις καλά τα βασικά.

### Τεχνική Προσέγγιση

Η υλοποίηση ζει `LocalSkillsTab.tsx` και `LocalSkillCommandAdapter.cs`. Η βασική ιδέα είναι να τυλίξετε το `npx skills` εντολή, αναλύστε την έξοδο JSON του και μετατρέψτε το σε εσωτερικές δομές δεδομένων. Ακούγεται απλό, και στην πράξη είναι κυρίως.

```csharp
public async Task<IReadOnlyList<LocalSkillInventoryResponseDto>> GetLocalSkillsAsync(
    CancellationToken cancellationToken = default)
{
    var result = await _commandAdapter.ListGlobalSkillsAsync(cancellationToken);
    return result.Skills.Select(skill => new LocalSkillInventoryResponseDto
    {
        Name = skill.Name,
        Version = skill.Version,
        Source = skill.Source,
        InstalledPath = skill.InstalledPath,
        Description = skill.Description
    }).ToList();
}
```

Η ροή δεδομένων είναι πολύ σαφής: το frontend στέλνει ένα αίτημα -> `SkillGalleryAppService` το λαμβάνει -> `LocalSkillCommandAdapter` εκτελεί την εντολή npx -> το αποτέλεσμα JSON αναλύεται -> επιστρέφεται ένα DTO. Κάθε βήμα ακολουθεί φυσικά από το προηγούμενο.

Η απεγκατάσταση δεξιοτήτων χρησιμοποιεί το `npx skills remove -g <skillName> -y` εντολή και το σύστημα χειρίζεται αυτόματα τις εξαρτήσεις και την εκκαθάριση. Τα μεταδεδομένα εγκατάστασης αποθηκεύονται στο `managed-install.json` μέσα στον κατάλογο δεξιοτήτων, καταγράφοντας πληροφορίες όπως ο χρόνος εγκατάστασης και η αρχική έκδοση για μεταγενέστερες ενημερώσεις και έλεγχος. Μερικά πράγματα απλά αξίζει να τα καταγράψετε.

### Ροή εγκατάστασης αναλυτικά

Η εγκατάσταση δεξιοτήτων απαιτεί πολλά συντονισμένα βήματα. Στην πραγματικότητα, δεν είναι ιδιαίτερα περίπλοκο:

```csharp
public async Task<SkillInstallResultDto> InstallAsync(
    SkillInstallRequestDto request,
    CancellationToken cancellationToken = default)
{
    // 1. Normalize the installation reference
    var normalized = _referenceNormalizer.Normalize(
        request.SkillId,
        request.Source,
        request.SkillSlug,
        request.Version);

    // 2. Check prerequisites
    await _prerequisiteChecker.CheckAsync(cancellationToken);

    // 3. Acquire installation lock
    using var installLock = await _lockProvider.AcquireAsync(normalized.SkillId);

    // 4. Execute installation command
    var result = await _installCommandRunner.ExecuteAsync(
        new SkillInstallCommandExecutionRequest
        {
            Command = $"npx skills add {normalized.FullReference} -g -y",
            Timeout = TimeSpan.FromMinutes(4)
        },
        cancellationToken);

    // 5. Persist installation metadata
    await _metadataStore.WriteAsync(normalized.SkillPath, request);

    return new SkillInstallResultDto { Success = result.Success };
}
```

Εδώ χρησιμοποιούνται πολλά βασικά μοτίβα σχεδιασμού: ο **κανονικοποιητής αναφοράς** μετατρέπει διαφορετικές μορφές εισόδου, όπως π.χ. `tanweai/pua` και `@opencode/docker-skill`, σε μια ενοποιημένη εσωτερική αναπαράσταση. ο **μηχανισμός κλειδώματος εγκατάστασης** διασφαλίζει ότι μόνο μία λειτουργία εγκατάστασης μπορεί να εκτελεστεί για την ίδια ικανότητα κάθε φορά. και **Έξοδος ροής** ωθεί την πρόοδο της εγκατάστασης στο frontend σε πραγματικό χρόνο μέσω των συμβάντων που απεστάλησαν από διακομιστή, ώστε οι χρήστες να μπορούν να παρακολουθούν αρχεία καταγραφής που μοιάζουν με τερματικό καθώς συμβαίνουν.

Τελικά, όλα αυτά τα μοτίβα υπάρχουν για έναν σκοπό: να διατηρήσουν το σύστημα πιο απλό στη χρήση και τη συντήρηση.

## Βασική ικανότητα 2: Αναζήτηση αγοράς

Η αναζήτηση Marketplace επιτρέπει στους χρήστες να ανακαλύψουν και να εγκαταστήσουν δεξιότητες από την κοινότητα. Η ικανότητα ενός ατόμου είναι πάντα περιορισμένη. Η συλλογική γνώση προχωρά πολύ παραπέρα.

### Τεχνική Προσέγγιση

Η δυνατότητα αναζήτησης βασίζεται στο online API `https://api.hagicode.com/v1/skills/search`. Για τη βελτίωση της ταχύτητας απόκρισης, το σύστημα εφαρμόζει προσωρινή αποθήκευση. Η προσωρινή μνήμη μοιάζει λίγο με τη μνήμη: αν διατηρείτε χρήσιμα πράγματα, δεν χρειάζεται να σκεφτείτε τόσο σκληρά την επόμενη φορά.

```csharp
private async Task<IReadOnlyList<SkillGallerySkillDto>> SearchCatalogAsync(
    string query,
    CancellationToken cancellationToken,
    IReadOnlySet<string>? allowedSources = null)
{
    var cacheKey = $"skill_search:{query}:{string.Join(",", allowedSources ?? Array.Empty<string>())}";

    if (_memoryCache.TryGetValue(cacheKey, out var cached))
        return (IReadOnlyList<SkillGallerySkillDto>)cached!;

    var response = await _onlineApiClient.SearchAsync(
        new SearchSkillsRequest
        {
            Query = query,
            Limit = _options.LimitPerQuery,
        },
        cancellationToken);

    var results = response.Skills
        .Where(skill => allowedSources is null || allowedSources.Contains(skill.Source))
        .Select(skill => new SkillGallerySkillDto { ... })
        .ToList();

    _memoryCache.Set(cacheKey, results, TimeSpan.FromMinutes(10));
    return results;
}
```

Τα αποτελέσματα αναζήτησης υποστηρίζουν φιλτράρισμα από αξιόπιστες πηγές, επομένως οι χρήστες βλέπουν μόνο τις πηγές δεξιοτήτων που εμπιστεύονται. Ερωτήματα σποράς όπως `popular` και `recent` χρησιμοποιούνται για την προετοιμασία του καταλόγου, επιτρέποντας στους χρήστες να δουν τις προτεινόμενες δημοφιλείς δεξιότητες την πρώτη φορά που τον ανοίγουν. Οι πρώτες εντυπώσεις έχουν ακόμα σημασία.

## Βασική ικανότητα 3: Έξυπνες συστάσεις

Οι έξυπνες συστάσεις είναι το πιο περίπλοκο μέρος του συστήματος. Μπορούν να προτείνουν αυτόματα τις πιο κατάλληλες δεξιότητες με βάση το τρέχον πλαίσιο του έργου. Όσο σύνθετο κι αν είναι, αξίζει να το χτίσεις.

### Ροή σύστασης

Η πλήρης ροή συστάσεων χωρίζεται σε πέντε στάδια:

```
1. Build project context
   ↓
2. AI generates search queries
   ↓
3. Search the online catalog in parallel
   ↓
4. AI ranks the candidates
   ↓
5. Return the recommendation list
```

Πρώτον, το σύστημα αναλύει χαρακτηριστικά όπως η στοίβα τεχνολογίας του έργου, οι γλώσσες προγραμματισμού και η δομή τομέα για να δημιουργήσει ένα «προφίλ έργου». Αυτό το προφίλ μοιάζει λίγο με ένα βιογραφικό, που καταγράφει τα βασικά χαρακτηριστικά του έργου.

Στη συνέχεια, ένας κόκκος AI χρησιμοποιείται για τη δημιουργία στοχευμένων ερωτημάτων αναζήτησης. Αυτός ο σχεδιασμός είναι πραγματικά πολύ ενδιαφέρον: αντί να ρωτάμε απευθείας την τεχνητή νοημοσύνη, "Ποιες δεξιότητες να προτείνω;", ζητάμε πρώτα να σκεφτεί "Ποιοι όροι αναζήτησης είναι πιθανό να βρουν σχετικές δεξιότητες;" Μερικές φορές ο τρόπος που κάνετε την ερώτηση έχει μεγαλύτερη σημασία από την ίδια την απάντηση:

```csharp
var queryGeneration = await aiGrain.GenerateSkillRecommendationQueriesAsync(
    projectContext,      // Project context
    locale,              // User language preference
    maxQueries,           // Maximum number of queries
    effectiveSearchHero); // AI model selection
```

Στη συνέχεια, αυτά τα ερωτήματα αναζήτησης εκτελούνται παράλληλα για τη συλλογή μιας λίστας δεξιοτήτων υποψηφίων. Η παράλληλη επεξεργασία είναι, στο τέλος της ημέρας, απλώς ένας τρόπος εξοικονόμησης χρόνου.

Τέλος, ένα άλλο AI Grain κατατάσσει τις δεξιότητες των υποψηφίων. Αυτό το βήμα λαμβάνει υπόψη παράγοντες όπως η συνάφεια δεξιοτήτων με το έργο, η κατάσταση εμπιστοσύνης και οι προτιμήσεις ιστορικού χρήστη:

```csharp
var ranking = await aiGrain.RankSkillRecommendationsAsync(
    projectContext,
    candidates,
    installedSkillNames,
    locale,
    maxRecommendations,
    effectiveRankingHero);

response.Items = MergeRecommendations(projectContext, candidates, ranking, maxRecommendations);
```

### Εφεδρικός Μηχανισμός

Τα μοντέλα AI μπορεί να ανταποκρίνονται αργά ή να μην είναι διαθέσιμα προσωρινά. Ακόμα και τα καλύτερα συστήματα σκοντάφτουν μερικές φορές. Για το λόγο αυτό, το σύστημα περιλαμβάνει έναν ντετερμινιστικό εναλλακτικό μηχανισμό: όταν η υπηρεσία τεχνητής νοημοσύνης δεν είναι διαθέσιμη, χρησιμοποιεί έναν ευρετικό αλγόριθμο που βασίζεται σε κανόνες για τη δημιουργία προτάσεων, όπως το συμπέρασμα πιθανών απαιτούμενων δεξιοτήτων από εξαρτήσεις σε `package.json`.

Με απλά λόγια, αυτός ο εναλλακτικός μηχανισμός είναι απλώς ένα εφεδρικό σχέδιο για το σύστημα.

## Βασική ικανότητα 4: Διαχείριση αξιόπιστων παρόχων

Η διαχείριση αξιόπιστων παρόχων επιτρέπει στους χρήστες να ελέγχουν ποιες πηγές δεξιοτήτων θεωρούνται αξιόπιστες. Η εμπιστοσύνη εξακολουθεί να είναι κάτι που οι χρήστες θα πρέπει να μπορούν να ορίσουν οι ίδιοι.

### Κανόνες αντιστοίχισης

Οι αξιόπιστοι πάροχοι υποστηρίζουν δύο κανόνες αντιστοίχισης: **ακριβής αντιστοίχιση** (`exact`) και **ταίριασμα προθέματος** (`prefix`).

```csharp
public static TrustedSkillProviderResolutionSnapshot Resolve(
    TrustedSkillProviderSnapshot snapshot,
    string source)
{
    var normalizedSource = Normalize(source);

    foreach (var entry in snapshot.Entries.OrderBy(e => e.SortOrder))
    {
        if (!entry.IsEnabled) continue;

        foreach (var rule in entry.MatchRules)
        {
            bool isMatch = rule.MatchType switch
            {
                TrustedSkillProviderMatchRuleType.Exact
                    => string.Equals(normalizedSource, Normalize(rule.Value),
                        StringComparison.OrdinalIgnoreCase),
                TrustedSkillProviderMatchRuleType.Prefix
                    => normalizedSource.StartsWith(Normalize(rule.Value) + "/",
                        StringComparison.OrdinalIgnoreCase),
                _ => false
            };

            if (isMatch)
                return new TrustedSkillProviderResolutionSnapshot
                {
                    IsTrustedSource = true,
                    ProviderId = entry.ProviderId,
                    DisplayName = entry.DisplayName
                };
        }
    }

    return new TrustedSkillProviderResolutionSnapshot { IsTrustedSource = false };
}
```

Οι ενσωματωμένοι αξιόπιστοι πάροχοι περιλαμβάνουν γνωστούς οργανισμούς και έργα όπως τα Vercel, Azure, anthropics, Microsoft και χρήση προγράμματος περιήγησης. Οι προσαρμοσμένοι πάροχοι μπορούν να προστεθούν μέσω αρχείων διαμόρφωσης, καθορίζοντας ένα αναγνωριστικό παρόχου, εμφανιζόμενο όνομα, ετικέτα σήματος, κανόνες αντιστοίχισης και άλλα. Ο κόσμος είναι αρκετά μεγάλος που δεν θα ήταν ποτέ αρκετό να εμπιστευτείς μόνο μερικά ενσωματωμένα.

### Εφαρμογή εμμονής

Η αξιόπιστη διαμόρφωση διατηρείται χρησιμοποιώντας ένα Orleans Grain:

```csharp
public class TrustedSkillProviderGrain : Grain<TrustedSkillProviderState>,
    ITrustedSkillProviderGrain
{
    public async Task UpdateConfigurationAsync(TrustedSkillProviderSnapshot snapshot)
    {
        State.Snapshot = snapshot;
        await WriteStateAsync();
    }

    public Task<TrustedSkillProviderSnapshot> GetConfigurationAsync()
    {
        return Task.FromResult(State.Snapshot);
    }
}
```

Το πλεονέκτημα αυτής της προσέγγισης είναι ότι οι αλλαγές διαμόρφωσης συγχρονίζονται αυτόματα σε όλους τους κόμβους, χωρίς καμία ανάγκη μη αυτόματης ανανέωσης της κρυφής μνήμης. Ο αυτοματισμός είναι, τελικά, να αφήνεις τους ανθρώπους να ανησυχούν λιγότερο.

## Βασικός Τεχνικός Σχεδιασμός

### Μοτίβο προσαρμογέα εκτέλεσης εντολών

Το σύστημα Skill χρειάζεται να εκτελέσει διάφορες εντολές npx. Εάν αυτή η λογική ήταν διάσπαρτη παντού, ο κώδικας θα γινόταν γρήγορα δύσκολο να διατηρηθεί. Γι' αυτό σχεδιάσαμε μια διεπαφή προσαρμογέα. Τα μοτίβα σχεδίασης, τελικά, υπάρχουν για να διευκολύνουν τη διατήρηση του κώδικα:

```csharp
public interface ISkillInstallCommandRunner
{
    Task<SkillInstallCommandExecutionResult> ExecuteAsync(
        SkillInstallCommandExecutionRequest request,
        CancellationToken cancellationToken = default);
}
```

Διαφορετικές εντολές έχουν διαφορετικές υλοποιήσεις εκτελεστών, αλλά όλες υλοποιούν την ίδια διεπαφή, καθιστώντας τη δοκιμή και την αντικατάσταση απλές.

### Έξοδος ροής SSE

Η πρόοδος της εγκατάστασης προωθείται στο frontend σε πραγματικό χρόνο μέσω των συμβάντων που απεστάλησαν από τον διακομιστή:

```csharp
public async Task InstallWithProgressAsync(
    SkillInstallRequestDto request,
    IServerStreamWriter<SkillInstallProgressEventDto> stream,
    CancellationToken cancellationToken)
{
    var process = new Process
    {
        StartInfo = new ProcessStartInfo
        {
            FileName = "npx",
            Arguments = $"skills add {request.FullReference} -g -y",
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false
        }
    };

    process.OutputDataReceived += async (sender, e) =>
    {
        await stream.WriteAsync(new SkillInstallProgressEventDto
        {
            EventType = "output",
            Data = e.Data ?? string.Empty
        });
    };

    process.Start();
    process.BeginOutputReadLine();
    await process.WaitForExitAsync(cancellationToken);
}
```

Στο frontend, οι χρήστες μπορούν να δουν έξοδο που μοιάζει με τερματικό σε πραγματικό χρόνο, γεγονός που κάνει την εμπειρία πολύ διαισθητική. Η ανατροφοδότηση σε πραγματικό χρόνο βοηθά τους ανθρώπους να αισθάνονται άνετα.

## Πρακτικός Οδηγός

### Εγκατάσταση κοινοτικής δεξιότητας

Πάρτε την εγκατάσταση του `pua` δεξιότητα ως παράδειγμα (είναι μια δημοφιλής δεξιότητα της κοινότητας):

1. Ανοίξτε το συρτάρι Skills και μεταβείτε στο `Skill Gallery` καρτέλα
2. Εισαγάγετε `pua` στο πλαίσιο αναζήτησης
3. Κάντε κλικ στο αποτέλεσμα αναζήτησης για να δείτε τις λεπτομέρειες δεξιοτήτων
4. Κάντε κλικ στο `Install` κουμπί
5. Εναλλαγή στο `Local Skills` καρτέλα για να επιβεβαιώσετε ότι η εγκατάσταση ολοκληρώθηκε με επιτυχία

Η εντολή εγκατάστασης είναι `npx skills add tanweai/pua -g -y`, και το σύστημα χειρίζεται αυτόματα όλες τις λεπτομέρειες. Δεν υπάρχουν πραγματικά τόσα πολλά βήματα μόλις τα κάνετε ένα προς ένα.

### Προσθήκη προσαρμοσμένης αξιόπιστης πηγής

Εάν η ομάδα σας έχει το δικό της αποθετήριο δεξιοτήτων, μπορείτε να το προσθέσετε ως αξιόπιστη πηγή:

```yaml
providerId: "my-team"
displayName: "My Team Skills"
badgeLabel: "MyTeam"
isEnabled: true
sortOrder: 100
matchRules:
  - matchType: "prefix"
    value: "my-team/"
  - matchType: "exact"
    value: "my-team/special-skill"
```

Με αυτόν τον τρόπο, όλες οι δεξιότητες από την ομάδα σας θα εμφανίζουν ένα σήμα αξιόπιστου, καθιστώντας τους χρήστες πιο άνετους στην εγκατάστασή τους. Οι ετικέτες και τα σήματα βοηθούν τους ανθρώπους να νιώθουν μεγαλύτερη αυτοπεποίθηση.

### Βασικά στοιχεία ανάπτυξης δεξιοτήτων

Η δημιουργία μιας προσαρμοσμένης ικανότητας απαιτεί την ακόλουθη δομή:

```
my-skill/
├── SKILL.md          # Skill metadata (YAML front matter)
├── index.ts          # Skill entry point
├── agents/           # Supported agent configuration
└── references/       # Reference resources
```

Ένα παράδειγμα `SKILL.md` μορφή:

```yaml
---
name: my-skill
description: A brief description of what this skill does
---

# My Skill

Detailed documentation...
```

## Σημειώσεις

1. **Απαιτήσεις δικτύου**: η αναζήτηση δεξιοτήτων και η εγκατάσταση απαιτούν πρόσβαση `api.hagicode.com` και το μητρώο npm
2. **Έκδοση Node.js**: Συνιστάται το Node.js 18 ή νεότερη έκδοση
3. **Απαιτήσεις άδειας**: Απαιτούνται καθολικές άδειες εγκατάστασης npm
4. **Έλεγχος συγχρονισμού**: μόνο μία λειτουργία εγκατάστασης ή απεγκατάστασης μπορεί να εκτελεστεί για την ίδια ικανότητα κάθε φορά
5. **Ρυθμίσεις χρονικού ορίου**: το προεπιλεγμένο χρονικό όριο για την εγκατάσταση είναι 4 λεπτά, αλλά πολύπλοκα σενάρια ενδέχεται να απαιτούν προσαρμογή

Αυτές οι σημειώσεις υπάρχουν, τελικά, για να βοηθήσουν τα πράγματα να πάνε ομαλά.

## Συμπέρασμα

Αυτό το άρθρο εισήγαγε την πλήρη εφαρμογή του συστήματος διαχείρισης δεξιοτήτων στο έργο HagiCode. Μέσω μιας διαχωρισμένης αρχιτεκτονικής frontend-backend, του μοτίβου προσαρμογέα, της διαχείρισης κατανεμημένων καταστάσεων με βάση την Ορλεάνη και σχετικών τεχνικών, το σύστημα παρέχει:

- **Τοπική παγκόσμια διαχείριση**: μια ενοποιημένη διεπαφή διαχείρισης δεξιοτήτων που δημιουργείται με αναδίπλωση `npx skills` εντολές
- **Αναζήτηση αγοράς**: ταχεία ανακάλυψη δεξιοτήτων κοινότητας μέσω του διαδικτυακού API και των μηχανισμών προσωρινής αποθήκευσης
- **Ευφυείς συστάσεις**: Προτάσεις δεξιοτήτων με τεχνητή νοημοσύνη βασισμένες στο πλαίσιο του έργου
- **Διαχείριση εμπιστοσύνης**: ένα ευέλικτο σύστημα διαμόρφωσης που επιτρέπει στους χρήστες να ελέγχουν τα όρια εμπιστοσύνης

Αυτή η προσέγγιση σχεδιασμού δεν εφαρμόζεται μόνο στη διαχείριση δεξιοτήτων. Είναι επίσης χρήσιμο ως αναφορά για οποιοδήποτε σενάριο που χρειάζεται να ενσωματώσει εργαλεία γραμμής εντολών ενώ παράλληλα εξισορροπεί την τοπική αποθήκευση και τις διαδικτυακές υπηρεσίες.
