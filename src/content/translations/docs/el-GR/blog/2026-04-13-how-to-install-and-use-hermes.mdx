Εάν θέλετε να εγκαταστήσετε το Hermes και να αρχίσετε να το χρησιμοποιείτε, η συντομότερη διαδρομή είναι στην πραγματικότητα μόλις τρία βήματα:

1. Εκτελέστε την επίσημη εντολή εγκατάστασης
2. Ξεκινήστε το CLI στο τερματικό σας με `hermes`
3. Εάν θέλετε να συνεχίσετε να το χρησιμοποιείτε στο Feishu, τότε ρυθμίστε τις παραμέτρους `hermes gateway setup`

Αυτό το άρθρο δεν προσπαθεί να εξηγήσει όλες τις δυνατότητες του Hermes ταυτόχρονα. Αντίθετα, σας βοηθά να ολοκληρώσετε πρώτα τον πιο σημαντικό βρόχο αρχαρίων: εγκαταστήστε τον, ξεκινήστε να τον χρησιμοποιείτε και, στη συνέχεια, συνδέστε τον σε ένα από τα πιο κοινά σενάρια πλατφόρμας ανταλλαγής μηνυμάτων.

## Τι είναι ο Ερμής και για ποιον απευθύνεται

Το Hermes Agent είναι ένας πράκτορας AI που μπορείτε να χρησιμοποιήσετε είτε από ένα τοπικό τερματικό είτε μέσω μιας πύλης πλατφόρμας μηνυμάτων.

Για τους περισσότερους προγραμματιστές, έχει δύο κοινά σημεία εισόδου:

- **CLI**: Τύπος `hermes` στο τερματικό σας για να εισέλθετε απευθείας στη διαδραστική διεπαφή.
- **Πύλη μηνυμάτων**: Εκτέλεση `hermes gateway`, στη συνέχεια συνομιλήστε μαζί του από πλατφόρμες όπως Feishu, Telegram, Discord και Slack.

Εάν ο στόχος σας αυτή τη στιγμή είναι απλώς να ξεκινήσετε γρήγορα, μην αντιστρέψετε τη σειρά. Ξεκινήστε με αυτήν τη διαδρομή:

- Εγκαταστήστε πρώτα το Hermes
- Επαληθεύστε ότι λειτουργεί πρώτα από το CLI
- Στη συνέχεια, αποφασίστε εάν θέλετε να συνδέσετε μια πλατφόρμα ανταλλαγής μηνυμάτων

Αυτό διευκολύνει τη διάγνωση των προβλημάτων και είναι πιο κατάλληλο για άτομα που χρησιμοποιούν τον Ερμή για πρώτη φορά.

## Τι πρέπει να γνωρίζετε πριν εγκαταστήσετε το Hermes

Σύμφωνα με το Hermes README, η επίσημη διαδρομή γρήγορης εγκατάστασης υποστηρίζει αυτά τα περιβάλλοντα:

- Linux
- macOS
- WSL2
- Android μέσω Termux

### Σημειώσεις για χρήστες Windows

Ο Hermes δεν υποστηρίζει αυτήν τη στιγμή την απευθείας εκτέλεση σε εγγενή Windows. Εάν χρησιμοποιείτε Windows, η προτεινόμενη προσέγγιση είναι να εγκαταστήσετε πρώτα το WSL2 και μετά να εκτελέσετε την εντολή εγκατάστασης μέσα στο WSL2.

Είναι καλύτερο να το καταστήσετε σαφές στην αρχή, επειδή πολλές αποτυχίες εγκατάστασης δεν προκαλούνται από την ίδια την εντολή, αλλά από τη χρήση ενός μη υποστηριζόμενου περιβάλλοντος χρόνου εκτέλεσης.

## Πώς να εγκαταστήσετε το Hermes

Η εντολή γρήγορης εγκατάστασης που παρέχεται στο Hermes README είναι:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Αυτή η εντολή εκτελεί το επίσημο σενάριο εγκατάστασης και χειρίζεται βήματα προετοιμασίας για συγκεκριμένη πλατφόρμα.

### Επανατοποθετήστε το κέλυφός σας μετά την εγκατάσταση

Μόλις ολοκληρωθεί η εγκατάσταση, φορτώστε ξανά το περιβάλλον του κελύφους σας πρώτα. Η πιο συνηθισμένη εντολή είναι:

```bash
source ~/.bashrc
```

Εάν χρησιμοποιείτε zsh, μπορείτε να χρησιμοποιήσετε:

```bash
source ~/.zshrc
```

### Πώς να επιβεβαιώσετε ότι το Hermes έχει εγκατασταθεί σωστά

Ο πιο άμεσος τρόπος ελέγχου είναι να εκτελέσετε:

```bash
hermes
```

Εάν θέλετε επιπλέον επιβεβαίωση ότι η διαμόρφωση και οι εξαρτήσεις σας λειτουργούν, μπορείτε επίσης να εκτελέσετε:

```bash
hermes doctor
```

`hermes doctor` είναι ιδιαίτερα χρήσιμο σε αυτές τις περιπτώσεις:

- Η εντολή συμπεριφέρεται ασυνήθιστα μετά την εγκατάσταση
- Η διαμόρφωση του μοντέλου αποτυγχάνει
- Η πύλη αποτυγχάνει να ξεκινήσει
- Δεν είστε σίγουροι εάν οι εξαρτήσεις του περιβάλλοντος σας είναι πλήρεις

## Πώς να αρχίσετε να χρησιμοποιείτε το Hermes για πρώτη φορά

### Ξεκινήστε με το CLI

Εάν θέλετε απλώς να επιβεβαιώσετε το συντομότερο δυνατό ότι ο Hermes λειτουργεί, η απλούστερη μέθοδος είναι:

```bash
hermes
```

Αυτό λανσάρει το διαδραστικό Hermes CLI. Για τους πρώτους χρήστες του Hermes, είναι επίσης το πιο συνιστώμενο σημείο εκκίνησης, επειδή μπορείτε να επαληθεύσετε πρώτα τα πιο βασικά πράγματα:

- Εάν η εντολή είναι πραγματικά διαθέσιμη
- Εάν η τρέχουσα διαμόρφωση μοντέλου λειτουργεί σωστά
- Εάν η εργαλειοθήκη τερματικού λειτουργεί σωστά
- Εάν το στυλ αλληλεπίδρασης ταιριάζει με αυτό που χρειάζεστε

### Αυτές οι εντολές είναι αρκετές για τον πρώτο γύρο εγκατάστασης

Το Hermes README παραθέτει πολλές εντολές υψηλής συχνότητας και μαζί αποτελούν μια πρακτική διαδρομή πρώτης χρήσης:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Εάν δεν είστε σίγουροι τι κάνει ο καθένας, θυμηθείτε τα ως εξής:

- `hermes model`: επιλέξτε ή αλλάξτε μοντέλα
- `hermes tools`: προβολή και διαμόρφωση εργαλείων που είναι διαθέσιμα αυτήν τη στιγμή
- `hermes config set`: αλλαγή συγκεκριμένων στοιχείων διαμόρφωσης
- `hermes setup`: εκτελέστε τον οδηγό πλήρους προετοιμασίας μία φορά
- `hermes update`: ενημέρωση Ερμή
- `hermes doctor`: αντιμετώπιση προβλημάτων

Για αρχάριους, η πιο πρακτική παραγγελία είναι συνήθως:

1. Τρέξε `hermes model` πρώτον
2. Εάν θέλετε να διαμορφώσετε όλες τις κοινές επιλογές ταυτόχρονα, τότε εκτελέστε `hermes setup`

## Οι δύο πιο συνηθισμένοι τρόποι χρήσης του Hermes

### 1. Χρησιμοποιήστε το Hermes στο τερματικό ως καθημερινό βοηθό ανάπτυξης

Η λειτουργία CLI είναι κατάλληλη για αυτά τα σενάρια:

- Κάντε ερωτήσεις απευθείας ενώ γράφετε κώδικα τοπικά
- Επιθεωρήστε έργα, επεξεργαστείτε αρχεία και εκτελέστε εντολές
- Εκτελέστε μεμονωμένη εργασία εντοπισμού σφαλμάτων ή αναθεώρησης
- Συνεργαστείτε συνεχώς στον τρέχοντα κατάλογο εργασίας

Το μεγαλύτερο πλεονέκτημά του είναι ότι είναι ο συντομότερος δρόμος: δεν υπάρχει επιπλέον ενσωμάτωση πλατφόρμας, δεν υπάρχει διαμόρφωση bot για χειρισμό εκ των προτέρων και είναι ο καλύτερος τρόπος για να δημιουργήσετε το πρώτο σας σύνολο συνηθειών χρήσης.

### 2. Χρησιμοποιήστε το Hermes μέσω μιας πλατφόρμας ανταλλαγής μηνυμάτων

Εάν θέλετε να συνομιλήσετε με τον Hermes σε πλατφόρμες όπως το Feishu, το Telegram ή το Discord, πρέπει να χρησιμοποιήσετε την πύλη ανταλλαγής μηνυμάτων.

Οι πιο συνηθισμένες εντολές εισαγωγής είναι:

```bash
hermes gateway setup
hermes gateway
```

Συγκεκριμένα:

- `hermes gateway setup` χρησιμοποιείται για διαδραστική διαμόρφωση πλατφόρμας
- `hermes gateway` χρησιμοποιείται για την έναρξη της διαδικασίας πύλης

Σύμφωνα με την επίσημη τεκμηρίωση, η πύλη είναι μια ενοποιημένη διαδικασία παρασκηνίου που συνδέει τις διαμορφωμένες πλατφόρμες σας, διαχειρίζεται συνεδρίες και χειρίζεται λειτουργίες όπως εργασίες cron.

## Χρησιμοποιώντας το Feishu ως παράδειγμα: πώς να συνδέσετε τον Hermes σε μια πλατφόρμα ανταλλαγής μηνυμάτων

Εάν το μεγαλύτερο μέρος της καθημερινής σας εργασίας γίνεται στο Feishu, τότε το Feishu/Lark είναι ένας πολύ φυσικός τρόπος για να χρησιμοποιήσετε τον Hermes.

### Η ελάχιστη βιώσιμη διαδρομή ολοκλήρωσης

Η επίσημη τεκμηρίωση συνιστά αυτήν την εντολή εισαγωγής για Feishu/Lark:

```bash
hermes gateway setup
```

Αφού το εκτελέσετε, απλώς επιλέξτε **Feishu / Lark** στον οδηγό.

Η τεκμηρίωση Feishu περιγράφει δύο τρόπους σύνδεσης:

- `websocket`: συνιστάται
- `webhook`: προαιρετικό

Εάν το Hermes εκτελείται στον φορητό υπολογιστή, στο σταθμό εργασίας ή στον ιδιωτικό σας διακομιστή, χρησιμοποιώντας `websocket` Το πρώτο είναι συνήθως πιο απλό επειδή δεν χρειάζεται να εκθέσετε μια δημόσια διεύθυνση URL επιστροφής κλήσης.

### Εάν το ρυθμίσετε με μη αυτόματο τρόπο, τουλάχιστον γνωρίζετε αυτές τις μεταβλητές

Εάν δεν χρησιμοποιείτε τον οδηγό και γράφετε τη διαμόρφωση με μη αυτόματο τρόπο, η τεκμηρίωση Feishu παραθέτει αυτές τις βασικές μεταβλητές:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Δύο από αυτά αξίζουν ιδιαίτερης προσοχής:

- `FEISHU_ALLOWED_USERS`: συνιστάται, επομένως δεν μπορούν όλοι όσοι μπορούν να φτάσουν στο bot να το χρησιμοποιήσουν απευθείας
- `FEISHU_HOME_CHANNEL`: σας επιτρέπει να προκαθορίσετε μια οικιακή συνομιλία για να λαμβάνετε αποτελέσματα cron ή προεπιλεγμένες ειδοποιήσεις

### Γιατί ο Ερμής μερικές φορές δεν απαντά στις ομαδικές συνομιλίες Feishu

Αυτή η λεπτομέρεια είναι εύκολο να χαθεί: στις ομαδικές συνομιλίες Feishu, ο Hermes δεν απαντά σε κάθε μήνυμα από προεπιλογή.

Η επίσημη τεκμηρίωση αναφέρει ξεκάθαρα:

- Στα άμεσα μηνύματα, ο Ερμής απαντά στα μηνύματα
- Στις ομαδικές συνομιλίες, πρέπει ρητά `@` το bot προτού επεξεργαστεί το μήνυμα

Εάν θέλετε να ορίσετε μια συνομιλία Feishu ως το αρχικό κανάλι, μπορείτε επίσης να το χρησιμοποιήσετε στη συνομιλία:

```text
/set-home
```

Ή ορίστε το στη διαμόρφωση εκ των προτέρων:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Οι εντολές του Ερμή οι αρχάριοι πρέπει να θυμούνται πρώτα

Είτε χρησιμοποιείτε το Hermes στο CLI είτε σε μια πλατφόρμα ανταλλαγής μηνυμάτων, η απομνημόνευση των παρακάτω εντολών είναι ήδη αρκετή για να ξεκινήσετε:

- `/new` ή `/reset`: έναρξη νέας συνεδρίας
- `/model`: προβολή ή αλλαγή μοντέλου
- `/retry`: δοκιμάστε ξανά την προηγούμενη στροφή
- `/undo`: αναίρεση της προηγούμενης αλληλεπίδρασης
- `/compress`: μη αυτόματη συμπίεση του περιβάλλοντος
- `/help`: προβολή βοήθειας

Εάν χρησιμοποιείτε κυρίως τον Hermes σε μια πλατφόρμα ανταλλαγής μηνυμάτων, θυμηθείτε ένα ακόμη:

- `/sethome` ή `/set-home`: ορίστε την τρέχουσα συνομιλία ως το αρχικό κανάλι

Αυτές οι εντολές καλύπτουν τις πιο συνηθισμένες λειτουργίες αρχαρίου: επανεκκίνηση, προσαρμογή, επαναφορά, έλεγχος και συνέχεια.

## Συχνές ερωτήσεις

### Μπορούν τα Windows να εγκαταστήσουν απευθείας τον Hermes;

Όχι. Η τρέχουσα επίσημη τεκμηρίωση δηλώνει ξεκάθαρα ότι τα εγγενή Windows δεν υποστηρίζονται και συνιστάται το WSL2.

### Τι πρέπει να κάνω αν πληκτρολογώ `hermes` κάνει τίποτα μετά την εγκατάσταση;

Είναι καλύτερο να αντιμετωπίσετε το πρόβλημα με αυτή τη σειρά:

1. Φορτώστε ξανά το κέλυφός σας πρώτα, για παράδειγμα με `source ~/.bashrc`
2. Τρέξε `hermes` πάλι
3. Εάν εξακολουθεί να είναι μη φυσιολογικό, τρέξτε `hermes doctor`

### Γιατί το bot δεν απαντά σε μια ομάδα Feishu;

Ελέγξτε πρώτα αυτά τα τρία πράγματα:

- Είτε εσείς `@` ανέφερε τον Ερμή στην ομάδα
- Το αν `FEISHU_ALLOWED_USERS` περιορίζει τον τρέχοντα χρήστη
- Εάν η τρέχουσα πολιτική ομαδικής συνομιλίας επιτρέπει το χειρισμό ομαδικών μηνυμάτων

Σύμφωνα με την επίσημη τεκμηρίωση του Feishu, απαιτείται ρητή χρήση @αναφοράς σε σενάρια ομαδικής συνομιλίας.

## Προτεινόμενα επόμενα βήματα

Εάν θέλετε απλώς να αρχίσετε να χρησιμοποιείτε το Hermes όσο το δυνατόν γρηγορότερα, αυτή είναι η πιο συνιστώμενη παραγγελία:

1. Εκτελέστε πρώτα την εντολή εγκατάστασης
2. Ξεκινήστε με `hermes` στο τοπικό CLI πρώτα
3. Χρήση `hermes model` και `hermes setup` για να ολοκληρώσετε τη βασική διαμόρφωση
4. Εάν θέλετε να συνεχίσετε να το χρησιμοποιείτε στο Feishu, τότε ρυθμίστε τις παραμέτρους `hermes gateway setup`

Εάν αυτό το άρθρο είναι το πρώτο μέρος μιας σειράς, ο καλύτερος ρόλος του δεν είναι να εξηγεί κάθε προηγμένη λειτουργία ταυτόχρονα, αλλά να οδηγεί τους χρήστες πρώτα στην πόρτα.

Τα ακόλουθα θέματα χωρίζονται καλύτερα σε επόμενα άρθρα:

- Ένας πλήρης οδηγός ενσωμάτωσης Hermes Feishu
- Ένας οδηγός για κοινές εντολές κάθετου Hermes
- Ένας οδηγός για τη διαμόρφωση της πύλης Hermes και την αντιμετώπιση προβλημάτων

Εάν σκοπεύετε να συνεχίσετε να δημιουργείτε περιεχόμενο Hermes, αυτό το άρθρο μπορεί να χρησιμεύσει ως το σημείο εκκίνησης για μεταγενέστερες δημοσιεύσεις, ενώ σταδιακά χτίζετε την εσωτερική δομή συνδέσμων.
