### Ορισμοί και επικύρωση τύπου Frontend

Στην υλοποίηση του frontend του HagiCode, ορίσαμε ένα πλήρες σύνολο τύπων διαμόρφωσης λέξεων πρόσβασης και λογική επικύρωσης. Το πρώτο μέρος είναι ο ορισμός του τύπου:

```typescript
export interface HotwordConfig {
  contextText: string;           // Multi-line hotword text
  boostingTableId: string;      // Doubao platform hotword table ID
  combineMode: boolean;          // Whether to use both together
}
```

Αυτή η απλή διεπαφή περιέχει όλα τα στοιχεία διαμόρφωσης για τη λειτουργία hotword. Ανάμεσά τους, `contextText` είναι το τμήμα με το οποίο οι χρήστες αλληλεπιδρούν πιο άμεσα: επιτρέπουμε στους χρήστες να εισάγουν μία φράση λέξης-κλειδί ανά γραμμή, κάτι που είναι πολύ διαισθητικό. Το να ζητάτε από τους χρήστες να εισάγουν έναν όρο ανά γραμμή είναι πολύ πιο εύκολο από το να τους κάνετε να κατανοήσουν μια περίπλοκη μορφή διαμόρφωσης.

Ακολουθεί η συνάρτηση επικύρωσης. Με βάση τους περιορισμούς του Doubao API, ορίσαμε αυστηρούς κανόνες επικύρωσης: το πολύ 100 γραμμές κειμένου hotword, έως 50 χαρακτήρες ανά γραμμή και όχι περισσότερους από 5000 χαρακτήρες συνολικά. `boosting_table_id` μπορεί να έχει το πολύ 200 χαρακτήρες και μπορεί να περιέχει μόνο γράμματα, αριθμούς, κάτω παύλες και παύλες. Αυτά τα όρια δεν είναι αυθαίρετα. προέρχονται απευθείας από την επίσημη τεκμηρίωση του Doubao. Τα όρια API είναι όρια API και πρέπει να τα ακολουθούμε.

```typescript
export function validateContextText(contextText: string): HotwordValidationResult {
  if (!contextText || contextText.trim().length === 0) {
    return { isValid: true, errors: [] };
  }

  const lines = contextText.split('\n').filter(line => line.trim().length > 0);
  const errors: string[] = [];

  if (lines.length > 100) {
    errors.push(`Hotword line count cannot exceed 100 lines; current count is ${lines.length}`);
  }

  const totalChars = contextText.length;
  if (totalChars > 5000) {
    errors.push(`Total hotword character count cannot exceed 5000; current count is ${totalChars}`);
  }

  for (let i = 0; i < lines.length; i++) {
    if (lines[i].length > 50) {
      errors.push(`Hotword on line ${i + 1} exceeds the 50-character limit`);
    }
  }

  return { isValid: errors.length === 0, errors };
}

export function validateBoostingTableId(boostingTableId: string): HotwordValidationResult {
  if (!boostingTableId || boostingTableId.trim().length === 0) {
    return { isValid: true, errors: [] };
  }

  const errors: string[] = [];

  if (boostingTableId.length > 200) {
    errors.push(`boosting_table_id cannot exceed 200 characters; current count is ${boostingTableId.length}`);
  }

  if (!/^[a-zA-Z0-9_-]+$/.test(boostingTableId)) {
    errors.push('boosting_table_id can contain only letters, numbers, underscores, and hyphens');
  }

  return { isValid: errors.length === 0, errors };
}
```

Αυτές οι λειτουργίες επικύρωσης εκτελούνται αμέσως όταν ο χρήστης διαμορφώνει τις λέξεις κλειδιά, διασφαλίζοντας ότι τα προβλήματα εντοπίζονται όσο το δυνατόν νωρίτερα. Από την άποψη της εμπειρίας χρήστη, αυτού του είδους η άμεση ανατροφοδότηση είναι πολύ σημαντική. Είναι πάντα καλύτερο για τους χρήστες να γνωρίζουν τι είναι λάθος ενώ πληκτρολογούν παρά μετά την υποβολή.

### Εμμονή διαμόρφωσης Frontend

Στην υλοποίηση του frontend του HagiCode, επιλέξαμε να χρησιμοποιήσουμε το πρόγραμμα περιήγησης `localStorage` για να αποθηκεύσετε τη διαμόρφωση λέξεων-κλειδιών. Υπήρχαν πολλές σκέψεις πίσω από αυτήν την απόφαση σχεδιασμού. Πρώτον, η διαμόρφωση λέξεων-κλειδιών είναι εξαιρετικά εξατομικευμένη και διαφορετικοί χρήστες μπορεί να έχουν διαφορετικές ανάγκες για συγκεκριμένο τομέα. Δεύτερον, αυτή η προσέγγιση απλοποιεί την υλοποίηση του backend επειδή δεν απαιτεί επιπλέον πίνακες βάσης δεδομένων ή τελικά σημεία API. Τέλος, αφού οι χρήστες το διαμορφώσουν μία φορά στο πρόγραμμα περιήγησης, οι ρυθμίσεις μπορούν να φορτωθούν αυτόματα σε επόμενες χρήσεις, κάτι που είναι πολύ βολικό. Με απλά λόγια, είναι η πιο εύκολη προσέγγιση.

```typescript
const HOTWORD_STORAGE_KEYS = {
  contextText: 'hotword-context-text',
  boostingTableId: 'hotword-boosting-table-id',
  combineMode: 'hotword-combine-mode',
} as const;

export const DEFAULT_HOTWORD_CONFIG: HotwordConfig = {
  contextText: '',
  boostingTableId: '',
  combineMode: false,
};

// Load hotword configuration
export function loadHotwordConfig(): HotwordConfig {
  const contextText = localStorage.getItem(HOTWORD_STORAGE_KEYS.contextText) || '';
  const boostingTableId = localStorage.getItem(HOTWORD_STORAGE_KEYS.boostingTableId) || '';
  const combineMode = localStorage.getItem(HOTWORD_STORAGE_KEYS.combineMode) === 'true';

  return { contextText, boostingTableId, combineMode };
}

// Save hotword configuration
export function saveHotwordConfig(config: HotwordConfig): void {
  localStorage.setItem(HOTWORD_STORAGE_KEYS.contextText, config.contextText);
  localStorage.setItem(HOTWORD_STORAGE_KEYS.boostingTableId, config.boostingTableId);
  localStorage.setItem(HOTWORD_STORAGE_KEYS.combineMode, String(config.combineMode));
}
```

Η λογική σε αυτόν τον κώδικα είναι ξεκάθαρη και ξεκάθαρη. Διαβάζουμε από `localStorage` κατά τη φόρτωση της διαμόρφωσης και γράψτε σε `localStorage` κατά την αποθήκευση. Παρέχουμε επίσης μια προεπιλεγμένη διαμόρφωση, ώστε το σύστημα να μπορεί να λειτουργεί σωστά όταν δεν υπάρχει ακόμα διαμόρφωση. Πρέπει να υπάρχει μια λογική προεπιλογή, τελικά.

### Λογική κατασκευής ωφέλιμου φορτίου

Η κατασκευή ωφέλιμου φορτίου είναι ο πυρήνας ολόκληρης της λειτουργίας hotword. Μόλις έχουμε τη διαμόρφωση της λέξης πρόσβασης, πρέπει να τη μορφοποιήσουμε στη δομή JSON που απαιτείται από το Doubao API. Αυτή η διαδικασία πραγματοποιείται πριν το SDK στείλει το αίτημα:

```csharp
private void AddCorpusToRequest(Dictionary<string, object> request)
{
    var corpus = new Dictionary<string, object>();

    // Add custom hotwords
    if (Config.HotwordContexts != null && Config.HotwordContexts.Count > 0)
    {
        corpus["context"] = new Dictionary<string, object>
        {
            ["context_type"] = "dialog_ctx",
            ["context_data"] = Config.HotwordContexts
                .Select(text => new Dictionary<string, object> { ["text"] = text })
                .ToList()
        };
    }

    // Add platform hotword table ID
    if (!string.IsNullOrEmpty(Config.BoostingTableId))
    {
        corpus["boosting_table_id"] = Config.BoostingTableId;
    }

    // Add corpus to the request only when it is not empty
    if (corpus.Count > 0)
    {
        request["corpus"] = corpus;
    }
}
```

Αυτός ο κώδικας δείχνει πώς να κατασκευάσετε δυναμικά το `corpus` πεδίο με βάση τη διαμόρφωση. Το βασικό σημείο είναι ότι προσθέτουμε το `corpus` πεδίο μόνο όταν υπάρχει στην πραγματικότητα διαμόρφωση λέξεων-κλειδιών. Αυτός ο σχεδιασμός εξασφαλίζει συμβατότητα προς τα πίσω: όταν δεν διαμορφώνονται λέξεις-κλειδιά, η δομή του αιτήματος παραμένει ακριβώς η ίδια με πριν. Η συμβατότητα προς τα πίσω έχει σημασία. Η προσθήκη ενός χαρακτηριστικού δεν πρέπει να διαταράσσει την υπάρχουσα λογική.

### Διαβίβαση παραμέτρων WebSocket

Μεταξύ του frontend και του backend, οι παράμετροι hotword μεταβιβάζονται μέσω μηνυμάτων ελέγχου WebSocket. Το HagiCode έχει σχεδιαστεί έτσι ώστε όταν το frontend ξεκινά την εγγραφή, φορτώνει τη διαμόρφωση της λέξης πρόσβασης από `localStorage` και το στέλνει στο backend μέσω μηνύματος WebSocket.

```typescript
const controlMessage = {
  type: 'control',
  payload: {
    command: 'StartRecognition',
    contextText: '高血压\n糖尿病\n冠心病',
    boosting_table_id: 'medical_table',
    combineMode: false
  }
};
```

Υπάρχει μια λεπτομέρεια που πρέπει να σημειωθεί εδώ: η διεπαφή περνάει κείμενο πολλών γραμμών που χωρίζονται με χαρακτήρες νέας γραμμής και το backend πρέπει να το αναλύσει. Ο χειριστής WebSocket backend αναλύει αυτές τις παραμέτρους και τις μεταβιβάζει στο SDK:

```csharp
private async Task HandleControlMessageAsync(
    string connectionId,
    DoubaoSession session,
    ControlMessage message)
{
    if (message.Payload is SessionControlRequest controlRequest)
    {
        // Parse hotword parameters
        string? contextText = controlRequest.ContextText;
        string? boostingTableId = controlRequest.BoostingTableId;
        bool? combineMode = controlRequest.CombineMode;

        // Parse multi-line text into a hotword list
        if (!string.IsNullOrEmpty(contextText))
        {
            var hotwords = contextText
                .Split('\n', StringSplitOptions.RemoveEmptyEntries)
                .Select(s => s.Trim())
                .Where(s => s.Length > 0)
                .ToList();

            session.HotwordContexts = hotwords;
        }

        session.BoostingTableId = boostingTableId;
    }
}
```

Με αυτόν τον σχεδιασμό, η μετάδοση της διαμόρφωσης λέξεων-κλειδιών από το frontend στο backend γίνεται σαφές και αποτελεσματικό. Δεν υπάρχει τίποτα ιδιαίτερα μυστηριώδες σε αυτό. τα δεδομένα απλώς περνούν από στρώμα προς στρώμα.

## Πρακτικός Οδηγός

### Διαμόρφωση προσαρμοσμένων λέξεων πρόσβασης

Σε πραγματική χρήση, η διαμόρφωση προσαρμοσμένων λέξεων πρόσβασης είναι πολύ απλή. Ανοίξτε τη σελίδα ρυθμίσεων αναγνώρισης ομιλίας στο HagiCode και βρείτε την ενότητα "Διαμόρφωση λέξεων-κλειδιών". Στο πλαίσιο εισαγωγής "Προσαρμοσμένη λέξη-κλειδί", εισαγάγετε μία φράση λέξης κλειδιού ανά γραμμή.

Για παράδειγμα, εάν αναπτύσσετε μια εφαρμογή που σχετίζεται με την ιατρική, μπορείτε να τη διαμορφώσετε ως εξής:

```
高血压
糖尿病
冠心病
心绞痛
心肌梗死
心力衰竭
```

Αφού αποθηκεύσετε τη διαμόρφωση, αυτές οι λέξεις-κλειδιά μεταβιβάζονται αυτόματα στο Doubao API κάθε φορά που ξεκινά η αναγνώριση ομιλίας. Στις δοκιμές μας, μόλις διαμορφώθηκαν οι λέξεις-κλειδιά, η ακρίβεια αναγνώρισης για σχετικούς επαγγελματικούς όρους βελτιώθηκε αισθητά. Η βελτίωση είναι πραγματική, και σαφώς καλύτερη από πριν.

### Διαμόρφωση πίνακα λέξεων-κλειδιών πλατφόρμας

Εάν χρειάζεται να διαχειριστείτε μεγάλο αριθμό λέξεων πρόσβασης ή εάν οι λέξεις-κλειδιά χρειάζονται συχνές ενημερώσεις, η λειτουργία πίνακα λέξεων-κλειδιών πλατφόρμας ταιριάζει καλύτερα. Αρχικά, δημιουργήστε έναν πίνακα λέξεων πρόσβασης στην πλατφόρμα αυτοεκμάθησης Doubao και αποκτήστε το δημιουργημένο `boosting_table_id`, στη συνέχεια εισαγάγετε αυτό το αναγνωριστικό στη σελίδα ρυθμίσεων HagiCode.

Η πλατφόρμα αυτομάθησης Doubao παρέχει δυνατότητες όπως μαζική εισαγωγή και διαχείριση κατηγοριοποιήσεων για hotwords, κάτι που είναι πολύ πρακτικό για ομάδες που χρειάζονται να διαχειριστούν μεγάλα σύνολα εξειδικευμένης ορολογίας. Με τη διαχείριση των λέξεων πρόσβασης στην πλατφόρμα, μπορείτε να τις διατηρήσετε κεντρικά και να διαθέτετε ενημερώσεις με συνέπεια. Μόλις η λίστα λέξεων-κλειδιών γίνει μεγάλη, το να έχετε ένα μόνο μέρος για τη διαχείρισή της είναι πολύ πιο πρακτικό από τη μη αυτόματη καταχώριση κάθε φορά.

### Χρήση της λειτουργίας συνδυασμού

Σε ορισμένα πολύπλοκα σενάρια, μπορεί να χρειαστεί να χρησιμοποιήσετε ταυτόχρονα προσαρμοσμένες λέξεις-κλειδιά και έναν πίνακα λέξεων πρόσβασης πλατφόρμας. Σε αυτήν την περίπτωση, απλώς διαμορφώστε και τα δύο στο HagiCode και ενεργοποιήστε το διακόπτη "Combination Mode".

Στη λειτουργία συνδυασμού, το Doubao API λαμβάνει υπόψη και τις δύο πηγές λέξεων-κλειδιών ταυτόχρονα, επομένως η ακρίβεια αναγνώρισης είναι συνήθως υψηλότερη από τη χρήση μιας από τις δύο πηγές μόνη της. Ωστόσο, αξίζει να σημειωθεί ότι η λειτουργία συνδυασμού αυξάνει την πολυπλοκότητα των αιτημάτων, επομένως είναι καλύτερο να αποφασίσετε εάν θα την ενεργοποιήσετε μετά από πρακτική δοκιμή. Περισσότερη πολυπλοκότητα αξίζει τον κόπο μόνο εάν τα αποτελέσματα του πραγματικού κόσμου το δικαιολογούν.

### Παραδείγματα ενσωμάτωσης κώδικα

Η ενσωμάτωση της λειτουργίας hotword στο έργο HagiCode είναι πολύ απλή. Ακολουθούν ορισμένα αποσπάσματα κώδικα που χρησιμοποιούνται συνήθως:

```typescript
import {
  loadHotwordConfig,
  saveHotwordConfig,
  validateHotwordConfig,
  parseContextText,
  getEffectiveHotwordMode,
  type HotwordConfig
} from '@/types/hotword';

// Load and validate configuration
const config = loadHotwordConfig();
const validation = validateHotwordConfig(config);

if (!validation.isValid) {
  console.error('Hotword configuration validation failed:', validation.errors);
  return;
}

// Parse hotword text
const hotwords = parseContextText(config.contextText);
console.log('Parsed hotwords:', hotwords);

// Get effective hotword mode
const mode = getEffectiveHotwordMode(config);
console.log('Current hotword mode:', mode);
```

Η χρήση του backend είναι εξίσου συνοπτική:

```csharp
var config = new DoubaoVoiceConfig
{
    AppId = "your_app_id",
    AccessToken = "your_access_token",
    ServiceUrl = "wss://openspeech.bytedance.com/api/v3/sauc/bigmodel_async",

    // Configure custom hotwords
    HotwordContexts = new List<string>
    {
        "高血压",
        "糖尿病",
        "冠心病"
    },

    // Configure platform hotword table
    BoostingTableId = "medical_table_v1"
};

var client = new DoubaoVoiceClient(config, logger);
await client.ConnectAsync();
await client.SendFullClientRequest();
```

## Πράγματα που πρέπει να έχετε κατά νου

Υπάρχουν πολλά σημεία που αξίζουν ιδιαίτερης προσοχής κατά την εφαρμογή και τη χρήση της λειτουργίας hotword.

Το πρώτο είναι το όριο χαρακτήρων. Το Doubao API έχει αυστηρούς περιορισμούς στις λέξεις κλειδιά, συμπεριλαμβανομένου του αριθμού γραμμών, των χαρακτήρων ανά γραμμή και του συνολικού αριθμού χαρακτήρων. Εάν ξεπεραστεί οποιοδήποτε όριο, το API επιστρέφει ένα σφάλμα. Στην υλοποίηση του frontend του HagiCode, ελέγχουμε αυτούς τους περιορισμούς κατά την εισαγωγή του χρήστη μέσω των συναρτήσεων επικύρωσης, γεγονός που αποτρέπει την αποστολή μη έγκυρων διαμορφώσεων στο backend. Η έγκαιρη αντιμετώπιση προβλημάτων είναι πάντα καλύτερη από την αναμονή για την αποτυχία του API.

Το δεύτερο είναι η μορφή του `boosting_table_id`. Αυτό το πεδίο επιτρέπει μόνο γράμματα, αριθμούς, κάτω παύλες και παύλες και δεν μπορεί να περιέχει κενά ή άλλους ειδικούς χαρακτήρες. Όταν δημιουργείτε έναν πίνακα λέξεων πρόσβασης στην πλατφόρμα αυτοεκπαίδευσης Doubao, φροντίστε να ακολουθείτε τους κανόνες ονομασίας. Αυτό το είδος αυστηρής επικύρωσης μορφής είναι συνηθισμένο για τα API.

Το τρίτο είναι η συμβατότητα προς τα πίσω. Οι παράμετροι λέξεων-κλειδιών είναι εντελώς προαιρετικές. Εάν δεν έχουν διαμορφωθεί hotwords, το σύστημα συμπεριφέρεται ακριβώς όπως πριν. Αυτός ο σχεδιασμός διασφαλίζει ότι οι υπάρχοντες χρήστες δεν επηρεάζονται με κανέναν τρόπο, ενώ διευκολύνει επίσης τη σταδιακή μετεγκατάσταση και τις αναβαθμίσεις. Η προσθήκη μιας δυνατότητας δεν πρέπει να διαταράσσει την προηγούμενη λογική.

Τέλος, υπάρχει χειρισμός σφαλμάτων. Όταν η διαμόρφωση λέξεων-κλειδιών δεν είναι έγκυρη, το Doubao API επιστρέφει αντίστοιχα μηνύματα σφάλματος. Η εφαρμογή του HagiCode καταγράφει λεπτομερή αρχεία καταγραφής για να βοηθήσει τους προγραμματιστές να αντιμετωπίσουν προβλήματα. Ταυτόχρονα, η διεπαφή εμφανίζει σφάλματα επικύρωσης στη διεπαφή χρήστη για να βοηθήσει τους χρήστες να διορθώσουν τη διαμόρφωση. Ο καλός χειρισμός σφαλμάτων οδηγεί φυσικά σε καλύτερη εμπειρία χρήστη.

## Συμπέρασμα

Μέσω αυτού του άρθρου, παρέχουμε μια λεπτομερή εισαγωγή στην ολοκληρωμένη λύση για την εφαρμογή λέξεων-κλειδιών αναγνώρισης ομιλίας Doubao στο έργο HagiCode. Αυτή η λύση καλύπτει ολόκληρη τη διαδικασία από την ανάλυση απαιτήσεων και την τεχνική επιλογή έως την εφαρμογή κώδικα, δίνοντας στους προγραμματιστές ένα πρακτικό παράδειγμα που μπορούν να χρησιμοποιήσουν ως αναφορά.

Τα βασικά σημεία μπορούν να συνοψιστούν ως εξής. Πρώτον, το Doubao API υποστηρίζει τόσο προσαρμοσμένες λέξεις-κλειδιά όσο και πίνακες λέξεων-κλειδιών πλατφόρμας και μπορούν να χρησιμοποιηθούν ανεξάρτητα ή σε συνδυασμό. Δεύτερον, το frontend χρησιμοποιεί `localStorage` για να αποθηκεύσετε τις ρυθμίσεις παραμέτρων με απλό και αποτελεσματικό τρόπο. Τρίτον, το backend μεταβιβάζει παραμέτρους hotword κατασκευάζοντας δυναμικά το `corpus` πεδίο, διατηρώντας ισχυρή προς τα πίσω συμβατότητα. Τέταρτον, η ολοκληρωμένη λογική επικύρωσης διασφαλίζει την ορθότητα της διαμόρφωσης και αποφεύγει τα μη έγκυρα αιτήματα. Συνολικά, η λύση δεν είναι περίπλοκη. Απλώς ακολουθεί προσεκτικά τις απαιτήσεις του API.

Η εφαρμογή της λειτουργίας hotword ενισχύει περαιτέρω τις δυνατότητες του HagiCode στον τομέα της αναγνώρισης ομιλίας. Με τη διαμόρφωση ευέλικτων επαγγελματικών όρων που σχετίζονται με τις επιχειρήσεις, οι προγραμματιστές μπορούν να βοηθήσουν το σύστημα αναγνώρισης ομιλίας να κατανοήσει καλύτερα το περιεχόμενο από συγκεκριμένους τομείς και επομένως να παρέχει πιο ακριβείς υπηρεσίες. Τελικά, η τεχνολογία θα πρέπει να εξυπηρετεί τις πραγματικές επιχειρηματικές ανάγκες και η επίλυση πρακτικών προβλημάτων είναι αυτό που έχει μεγαλύτερη σημασία.
