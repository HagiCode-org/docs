## .NET Core Dual-Database στην πράξη: Αφήστε τις PostgreSQL και SQLite να συνυπάρχουν ειρηνικά

> Κατά τη δημιουργία σύγχρονων εφαρμογών, αντιμετωπίζουμε συχνά αυτό το συμβιβασμό: τα περιβάλλοντα ανάπτυξης θέλουν κάτι ελαφρύ και βολικό, ενώ τα περιβάλλοντα παραγωγής απαιτούν υψηλή ταυτόχρονη και υψηλή διαθεσιμότητα. Αυτό το άρθρο περιγράφει πώς μπορείτε να υποστηρίξετε κομψά τόσο την PostgreSQL όσο και την SQLite σε ένα έργο .NET Core και να εφαρμόσετε την καλύτερη πρακτική του "SQLite για ανάπτυξη, PostgreSQL για παραγωγή".

Η στοίβα είναι μοντέρνα και αρκετά εκλεπτυσμένη.

Αλλά στην παραγωγή, χρειαζόμασταν να χειριστούμε εγγραφές υψηλής ταυτόχρονης συχνότητας και σύνθετα ερωτήματα JSON, και εκεί ακριβώς το ελαφρύ SQLite αρχίζει να δείχνει τα όριά του.

Πώς μπορούμε να διατηρήσουμε μια ενιαία βάση κώδικα, ενώ επιτρέπουμε στην εφαρμογή να επωφεληθεί από τη φορητότητα του SQLite όπως μια εφαρμογή επιτραπέζιου υπολογιστή και επίσης να αξιοποιήσει την ισχυρή απόδοση της PostgreSQL όπως μια υπηρεσία εταιρικού επιπέδου; Αυτή είναι η βασική ερώτηση που διερευνά αυτό το άρθρο.

## Βασικό Θέμα 1: Αρχιτεκτονικός Σχεδιασμός και Ενοποιημένη Αφαίρεση

Για να υποστηρίξετε δύο βάσεις δεδομένων σε .NET Core, η βασική ιδέα είναι να βασίζεστε σε αφαιρέσεις και όχι σε συγκεκριμένες υλοποιήσεις. Πρέπει να διαχωρίσουμε την επιλογή της βάσης δεδομένων από τον επιχειρηματικό κώδικα και να αφήσουμε το επίπεδο διαμόρφωσης να αποφασίσει.

### Σχεδιαστική Προσέγγιση

1.  **Ενοποιημένη διεπαφή**: Όλη η επιχειρηματική λογική πρέπει να εξαρτάται από το `DbContext` βασική κλάση ή προσαρμοσμένες διεπαφές, αντί για μια συγκεκριμένη `PostgreSqlDbContext`.
2.  **Βάσει διαμόρφωσης**: Χρησιμοποιήστε στοιχεία διαμόρφωσης στο `appsettings.json` για να αποφασίσετε δυναμικά ποιον πάροχο βάσης δεδομένων θα φορτώσει κατά την εκκίνηση της εφαρμογής.
3.  **Απομόνωση λειτουργιών**: Προσθέστε λογική προσαρμογής για ειδικές δυνατότητες PostgreSQL, όπως το JSONB, ώστε η εφαρμογή να μπορεί να υποβαθμίζεται με χάρη στο SQLite.

### Υλοποίηση κώδικα: Δυναμική διαμόρφωση περιβάλλοντος

Σε ASP.NET Core `Program.cs`, δεν πρέπει να κάνουμε σκληρό κώδικα `UseNpgsql` ή `UseSqlite`. Αντίθετα, θα πρέπει να διαβάσουμε τη διαμόρφωση και να αποφασίσουμε δυναμικά.

Πρώτα, ορίστε την κλάση διαμόρφωσης:

```csharp
public class DatabaseSettings
{
    public const string SectionName = "Database";
    
    // Database type: PostgreSQL or SQLite
    public string DbType { get; set; } = "PostgreSQL"; 
    
    // Connection string
    public string ConnectionString { get; set; } = string.Empty;
}
```

Στη συνέχεια, καταχωρήστε την υπηρεσία `Program.cs` με βάση τη διαμόρφωση:

```csharp
// Read configuration
var databaseSettings = builder.Configuration.GetSection(DatabaseSettings.SectionName).Get<DatabaseSettings>();

// Register DbContext
builder.Services.AddDbContext<ApplicationDbContext>(options =>
{
    if (databaseSettings?.DbType?.ToLower() == "sqlite")
    {
        // SQLite configuration
        options.UseSqlite(databaseSettings.ConnectionString);
        
        // Handling SQLite's concurrent write limitations
        // Note: in production, enabling WAL mode is recommended to improve concurrency performance
    }
    else
    {
        // PostgreSQL configuration (default)
        options.UseNpgsql(databaseSettings.ConnectionString, npgsqlOptions =>
        {
            // Enable JSONB support, which is very useful when handling AI conversation records
            npgsqlOptions.UseJsonNet(); 
        });
        
        // Configure connection pool retry policy
        options.EnableRetryOnFailure(3);
    }
});
```

## Βασικό Θέμα 2: Διαχείριση Διαφορών και Στρατηγική Μετανάστευσης

Αν και τα PostgreSQL και SQLite υποστηρίζουν και τα δύο το πρότυπο SQL, διαφέρουν σημαντικά σε συγκεκριμένες δυνατότητες και συμπεριφορά. Εάν αυτές οι διαφορές δεν αντιμετωπιστούν προσεκτικά, μπορείτε εύκολα να καταλήξετε στην άβολη κατάσταση όπου όλα λειτουργούν τοπικά αλλά αποτυγχάνουν μετά την ανάπτυξη.

### 1. Χειρισμός τύπων JSON

Στο HagiCode, πρέπει να αποθηκεύσουμε μεγάλο αριθμό προτροπών και μεταδεδομένων AI, τα οποία συνήθως περιλαμβάνουν στήλες JSON.
- **PostgreSQL**: Έχει εγγενή `JSONB` τύπος με εξαιρετική απόδοση ερωτήματος.
- **SQLite**: Δεν έχει εγγενή τύπο JSON (οι νεότερες εκδόσεις περιλαμβάνουν την επέκταση JSON1, αλλά η αντιστοίχιση αντικειμένων εξακολουθεί να διαφέρει), επομένως τα δεδομένα αποθηκεύονται συνήθως ως TEXT.

**Λύση**:
Στην αντιστοίχιση οντοτήτων EF Core, το διαμορφώνουμε ως μετατρέψιμο τύπο.

```csharp
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    base.OnModelCreating(modelBuilder);

    // Configure entity
    modelBuilder.Entity<PromptTemplate>(entity => 
    {
        entity.Property(e => e.Metadata)
              .HasColumnType("jsonb") // PG uses jsonb
              .HasConversion(
                  v => JsonSerializer.Serialize(v, (JsonSerializerOptions)null),
                  v => JsonSerializer.Deserialize<Dictionary<string, object>>(v, (JsonSerializerOptions)null)
              ); 
    });
}
```

Όταν χρησιμοποιείται το SQLite, `HasColumnType("jsonb")` μπορεί να αγνοηθεί ή να ενεργοποιηθεί μια προειδοποίηση. Ωστόσο, επειδή `HasConversion` έχει ρυθμιστεί, τα δεδομένα εξακολουθούν να είναι σειριοποιημένα και αποσειριοποιημένα σωστά ως συμβολοσειρές που είναι αποθηκευμένες σε ένα πεδίο TEXT, διασφαλίζοντας τη συμβατότητα.

### 2. Διαχωρισμός Στρατηγικών Μετανάστευσης

Ποτέ μην προσπαθήσετε να κάνετε ένα σύνολο σεναρίων μετεγκατάστασης να λειτουργεί ταυτόχρονα και για την PostgreSQL και για το SQLite. Οι διαφορές στις στρατηγικές δημιουργίας πρωτεύοντος κλειδιού, στη σύνταξη ευρετηρίου και σε άλλες λεπτομέρειες της βάσης δεδομένων θα προκαλέσουν αναπόφευκτα αποτυχίες.

**Συνιστώμενη πρακτική**:
Διατηρήστε δύο κλάδους ή έργα μετανάστευσης. Στη ροή εργασιών ανάπτυξης HagiCode, το χειριζόμαστε ως εξής:

1.  **Στάδιο ανάπτυξης**: Εργαστείτε κυρίως με το SQLite. Χρήση `Add-Migration Init_Sqlite -OutputDir Migrations/Sqlite`.
2.  **Στάδιο προσαρμογής**: Αφού αναπτύξετε μια δυνατότητα, αλλάξτε τη συμβολοσειρά σύνδεσης σε PostgreSQL και εκτελέστε `Add-Migration Init_Postgres -OutputDir Migrations/Postgres`.
3.  **Σενάρια αυτοματισμού**: Γράψτε ένα απλό σενάριο PowerShell ή Bash για να εφαρμόσετε αυτόματα τη σωστή μετεγκατάσταση με βάση την τρέχουσα μεταβλητή περιβάλλοντος.

```bash
# Pseudocode for simple deployment logic
if [ "$DATABASE_PROVIDER" = "PostgreSQL" ]; then
    dotnet ef database update --project Migrations.Postgres
else
    dotnet ef database update --project Migrations.Sqlite
fi
```

## Βασικό Θέμα 3: Διδάγματα που αντλήθηκαν από το HagiCode στην παραγωγή

Κατά την ανακατασκευή του **HagiCode** από ένα μοντέλο μιας βάσης δεδομένων σε υποστήριξη διπλής βάσης δεδομένων, αντιμετωπίσαμε μερικές παγίδες και συγκεντρώσαμε μερικά σημαντικά μαθήματα που μπορεί να σας βοηθήσουν να αποφύγετε τα ίδια λάθη.

### 1. Διαφορές σε Συγχρονισμό και Συναλλαγές

Η PostgreSQL χρησιμοποιεί αρχιτεκτονική διακομιστή-πελάτη και υποστηρίζει εγγραφές υψηλής ταυτόχρονης λειτουργίας με ισχυρά επίπεδα απομόνωσης συναλλαγών. Το SQLite χρησιμοποιεί κλείδωμα αρχείων, επομένως οι λειτουργίες εγγραφής κλειδώνουν ολόκληρο το αρχείο της βάσης δεδομένων, εκτός εάν είναι ενεργοποιημένη η λειτουργία WAL.

**Σύσταση**:
Όταν γράφετε επιχειρηματική λογική που περιλαμβάνει συχνές εγγραφές, όπως αποθήκευση σε πραγματικό χρόνο της κατάστασης επεξεργασίας ενός χρήστη, πρέπει να λάβετε υπόψη το μοντέλο κλειδώματος του SQLite. Κατά το σχεδιασμό της ενότητας συνεργασίας OpenSpec στο **HagiCode**, εισαγάγαμε έναν μηχανισμό "συγχώνευσης πριν από την εγγραφή" για να μειώσουμε τη συχνότητα των άμεσων εγγραφών βάσης δεδομένων, επιτρέποντάς μας να διατηρήσουμε καλή απόδοση και στις δύο βάσεις δεδομένων.

### 2. Διαχείριση κύκλου ζωής συμβολοσειρών σύνδεσης

Η δημιουργία μιας σύνδεσης PostgreSQL είναι σχετικά ακριβή και εξαρτάται από τη συγκέντρωση συνδέσεων. Οι συνδέσεις SQLite είναι πολύ ελαφριές, αλλά εάν δεν απελευθερωθούν αμέσως, τα κλείδωμα αρχείων ενδέχεται να προκαλέσουν λήξη χρονικού ορίου για μεταγενέστερες λειτουργίες.

Σε `Program.cs`, μπορούμε να προσαρμόσουμε τη συμπεριφορά για κάθε βάση δεδομένων:

```csharp
if (databaseSettings?.DbType?.ToLower() == "sqlite")
{
    // SQLite: keeping connections open can improve performance, but watch out for file locks
    options.UseSqlite(connectionString, sqliteOptions =>
    {
        // Set command timeout
        sqliteOptions.CommandTimeout(30);
    });
}
else
{
    // PG: make full use of connection pooling
    options.UseNpgsql(connectionString, npgsqlOptions =>
    {
        npgsqlOptions.MaxBatchSize(100);
        npgsqlOptions.CommandTimeout(30);
    });
}
```

### 3. Η σημασία της κάλυψης της δοκιμής

Πολλοί προγραμματιστές, συμπεριλαμβανομένων ορισμένων πρώτων μελών της ομάδας μας, τείνουν να κάνουν ένα κοινό λάθος: να εκτελούν δοκιμές μονάδων μόνο στο περιβάλλον ανάπτυξης, το οποίο είναι συνήθως SQLite.

Στη διοχέτευση CI/CD του HagiCode, επιβάλαμε ένα βήμα GitHub Actions για να βεβαιωθούμε ότι κάθε αίτημα έλξης εκτελεί δοκιμές ενοποίησης PostgreSQL.

```yaml
# Example snippet from .github/workflows/test.yml
- name: Run Integration Tests (PostgreSQL)
  run: |
    docker-compose up -d db_postgres
    dotnet test --filter "Category=Integration"
```

Αυτό μας βοήθησε να εντοπίσουμε αμέτρητα σφάλματα που σχετίζονται με διαφορές σύνταξης SQL και ευαισθησία πεζών-κεφαλαίων.
