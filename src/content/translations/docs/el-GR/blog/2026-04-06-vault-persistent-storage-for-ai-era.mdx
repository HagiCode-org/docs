## Πώς να αναπαράγετε έργα στην εποχή AI: Vault, ένα σύστημα μόνιμης αποθήκευσης μεταξύ έργων

> Στην εποχή της ανάπτυξης με τη βοήθεια AI, πώς μπορούμε να βοηθήσουμε τους βοηθούς AI να κατανοήσουν καλύτερα τους πόρους εκμάθησής μας; Το έργο HagiCode δημιούργησε το σύστημα Vault ως ένα ενιαίο επίπεδο αφαίρεσης αποθήκευσης γνώσης που μπορεί να κατανοήσει η τεχνητή νοημοσύνη, βελτιώνοντας σημαντικά την αποτελεσματικότητα της μάθησης μέσω της αναπαραγωγής έργου.

## φόντο

Στην εποχή της τεχνητής νοημοσύνης, ο τρόπος με τον οποίο οι προγραμματιστές μαθαίνουν νέες τεχνολογίες και αρχιτεκτονικές αλλάζει βαθιά. Η "αναπαραγωγή έργων" - δηλαδή η βαθιά μελέτη και εκμάθηση από τον κώδικα, την αρχιτεκτονική και τα μοτίβα σχεδίασης εξαιρετικών έργων ανοιχτού κώδικα - έχει γίνει ένας αποτελεσματικός τρόπος μάθησης. Σε σύγκριση με τις παραδοσιακές μεθόδους όπως η ανάγνωση βιβλίων ή η παρακολούθηση βίντεο, η απευθείας ανάγνωση και εκτέλεση έργων ανοιχτού κώδικα υψηλής ποιότητας σάς βοηθά να κατανοείτε πολύ πιο γρήγορα τις πραγματικές πρακτικές μηχανικής.

Ωστόσο, αυτή η μέθοδος μάθησης συνοδεύεται από αρκετές προκλήσεις.

Το εκπαιδευτικό υλικό είναι πολύ διάσπαρτο. Οι σημειώσεις σας μπορεί να βρίσκονται στο Obsidian, τα αποθετήρια κώδικα μπορεί να είναι διάσπαρτα σε διαφορετικούς φακέλους και το ιστορικό συνομιλιών του βοηθού τεχνητής νοημοσύνης σας γίνεται ακόμη ένα απομονωμένο νησί δεδομένων. Όταν θέλετε η τεχνητή νοημοσύνη να βοηθήσει στην ανάλυση ενός έργου, πρέπει να αντιγράψετε με μη αυτόματο τρόπο αποσπάσματα κώδικα και να οργανώσετε το πλαίσιο, κάτι που είναι μάλλον κουραστικό.

Αυτό που είναι ακόμη πιο ενοχλητικό είναι το σπασμένο πλαίσιο. Οι βοηθοί τεχνητής νοημοσύνης δεν μπορούν να έχουν άμεση πρόσβαση στους τοπικούς πόρους εκμάθησής σας, επομένως πρέπει να παρέχετε ξανά πληροφορίες ιστορικού σε κάθε συνομιλία. Επιπλέον, τα αποθετήρια αναπαραγόμενου κώδικα ενημερώνονται γρήγορα, ο μη αυτόματος συγχρονισμός είναι επιρρεπής σε σφάλματα και η γνώση είναι δύσκολο να μοιραστεί σε πολλαπλά έργα εκμάθησης.

Στη ρίζα, όλα αυτά τα προβλήματα προέρχονται από «νησιά δεδομένων». Εάν υπήρχε ένα ενοποιημένο επίπεδο αφαίρεσης αποθήκευσης που επέτρεπε στους βοηθούς τεχνητής νοημοσύνης να κατανοούν και να έχουν πρόσβαση σε όλους τους πόρους εκμάθησής σας, το πρόβλημα θα λυνόταν τακτικά.

## Φιλοσοφία σχεδιασμού συστήματος θησαυροφυλακίου

Η βασική ιδέα του συστήματος Vault είναι απλή: δημιουργήστε ένα ενοποιημένο επίπεδο αφαίρεσης αποθήκευσης γνώσης που μπορεί να κατανοήσει η τεχνητή νοημοσύνη. Από την άποψη της υλοποίησης, το σύστημα έχει πολλά βασικά χαρακτηριστικά.

### Υποστήριξη πολλαπλών τύπων

Το σύστημα υποστηρίζει τέσσερις τύπους θησαυροφυλακίου, καθένας από τους οποίους αντιστοιχεί σε διαφορετικό σενάριο χρήσης:

```typescript
// folder: general-purpose folder type
export const DEFAULT_VAULT_TYPE = 'folder';

// coderef: a type specifically for reproduced code projects
export const CODEREF_VAULT_TYPE = 'coderef';

// obsidian: integrated with Obsidian note-taking software
export const OBSIDIAN_VAULT_TYPE = 'obsidian';

// system-managed: vault automatically managed by the system
export const SYSTEM_MANAGED_VAULT_TYPE = 'system-managed';
```

Μεταξύ αυτών, ο τύπος **coderef** είναι ο πιο συχνά χρησιμοποιούμενος στο HagiCode. Έχει σχεδιαστεί ειδικά για έργα αναπαραγόμενου κώδικα, παρέχοντας μια τυποποιημένη δομή καταλόγου και περιγραφές μεταδεδομένων αναγνώσιμες από AI.

### Μόνιμος μηχανισμός αποθήκευσης

Το μητρώο Vault αποθηκεύεται μόνιμα σε μορφή JSON, διασφαλίζοντας ότι η διαμόρφωση παραμένει διαθέσιμη μετά την επανεκκίνηση της εφαρμογής:

```csharp
public class VaultRegistryStore : IVaultRegistryStore
{
    private readonly string _registryFilePath;

    public VaultRegistryStore(IConfiguration configuration, ILogger<VaultRegistryStore> logger)
    {
        var dataDir = configuration["DataDir"] ?? "./data";
        var absoluteDataDir = Path.IsPathRooted(dataDir)
            ? dataDir
            : Path.GetFullPath(Path.Combine(Directory.GetCurrentDirectory(), dataDir));

        _registryFilePath = Path.Combine(absoluteDataDir, "personal-data", "vaults", "registry.json");
    }
}
```

Το πλεονέκτημα αυτού του σχεδίου είναι ότι είναι απλό και αξιόπιστο. Το JSON είναι αναγνώσιμο από τον άνθρωπο, γεγονός που διευκολύνει τον εντοπισμό σφαλμάτων και τη μη αυτόματη επεξεργασία. Η αποθήκευση του συστήματος αρχείων αποφεύγει την πολυπλοκότητα μιας βάσης δεδομένων και μειώνει τις εξαρτήσεις του συστήματος. Εξάλλου, μερικές φορές η πιο απλή επιλογή είναι πραγματικά η καλύτερη.

### Ενσωμάτωση πλαισίου AI

Το πιο σημαντικό, το σύστημα μπορεί να εισάγει αυτόματα πληροφορίες θησαυροφυλακίου στο πλαίσιο των προτάσεων AI:

```typescript
export function buildTargetVaultsText(
  vaults: VaultForText[],
  template: VaultPromptTemplate = DEFAULT_VAULT_PROMPT_TEMPLATE,
): string {
  const readOnlyVaults = vaults.filter((vault) => vault.accessType === 'read');
  const editableVaults = vaults.filter((vault) => vault.accessType === 'write');

  if (readOnlyVaults.length === 0 && editableVaults.length === 0) {
    return '';
  }

  const sections = [
    buildVaultSection(readOnlyVaults, template.reference),
    buildVaultSection(editableVaults, template.editable),
  ].filter(Boolean);

  return `\n\n### ${template.heading}\n\n${sections.join('\n')}`;
}
```

Αυτό επιτρέπει μια σημαντική ικανότητα: οι βοηθοί τεχνητής νοημοσύνης μπορούν αυτόματα να κατανοήσουν τους διαθέσιμους πόρους εκμάθησης χωρίς οι χρήστες να παρέχουν με μη αυτόματο τρόπο το πλαίσιο. Θα μπορούσατε να πείτε ότι αυτό μετράει ως ένα είδος σιωπηρής κατανόησης.

## Η τυποποιημένη δομή του CodeRef Vault

Για τον τύπο coderef του vault, το HagiCode παρέχει μια τυποποιημένη δομή καταλόγου:

```
my-coderef-vault/
├── index.yaml          # vault metadata description
├── AGENTS.md           # operating guide for AI assistants
├── docs/               # stores study notes and documents
└── repos/              # manages reproduced code repositories through Git submodules
```

Όταν δημιουργείτε ένα θησαυροφυλάκιο, το σύστημα αρχικοποιεί αυτόματα αυτήν τη δομή:

```csharp
private async Task EnsureCodeRefStructureAsync(
    string vaultName,
    string physicalPath,
    ICollection<VaultBootstrapDiagnosticDto> diagnostics,
    CancellationToken cancellationToken)
{
    Directory.CreateDirectory(physicalPath);

    var indexPath = Path.Combine(physicalPath, CodeRefIndexFileName);
    var docsPath = Path.Combine(physicalPath, CodeRefDocsDirectoryName);
    var reposPath = Path.Combine(physicalPath, CodeRefReposDirectoryName);

    // Create the standard directory structure
    if (!Directory.Exists(docsPath))
    {
        Directory.CreateDirectory(docsPath);
    }

    if (!Directory.Exists(reposPath))
    {
        Directory.CreateDirectory(reposPath);
    }

    // Create the AGENTS.md guide
    await EnsureCodeRefAgentsDocumentAsync(physicalPath, cancellationToken);

    // Create the index.yaml metadata
    await WriteCodeRefIndexDocumentAsync(indexPath, mergedDocument, cancellationToken);
}
```

Αυτή η δομή είναι επίσης προσεκτικά σχεδιασμένη:

- Το **docs/** αποθηκεύει τις σημειώσεις μελέτης, όπου μπορείτε να καταγράψετε την κατανόησή σας για τον κώδικα, την ανάλυση αρχιτεκτονικής, τα διδάγματα κ.λπ. στο Markdown
- Το **repos/** διαχειρίζεται τα αναπαραγόμενα αποθετήρια μέσω υπομονάδων Git αντί να αντιγράφει απευθείας τον κώδικα, γεγονός που διατηρεί τον κώδικα συγχρονισμένο και εξοικονομεί χώρο
- Το **index.yaml** περιέχει τα μεταδεδομένα του θησαυροφυλακίου, ώστε οι βοηθοί τεχνητής νοημοσύνης να μπορούν γρήγορα να κατανοήσουν τον σκοπό και το περιεχόμενο του θησαυροφυλακίου
- Το **AGENTS.md** είναι ένας οδηγός γραμμένος ειδικά για βοηθούς τεχνητής νοημοσύνης, ο οποίος εξηγεί πώς να χειριστείτε τα περιεχόμενα του θησαυροφυλακίου

Οργανωμένη με αυτόν τον τρόπο, ίσως η τεχνητή νοημοσύνη να καταλάβει τι έχετε στο μυαλό σας λίγο πιο εύκολα.

## Αυτόματη προετοιμασία για θησαυροφυλάκια που διαχειρίζονται το σύστημα

Εκτός από τα θησαυροφυλάκια που δημιουργούνται με μη αυτόματο τρόπο, το HagiCode υποστηρίζει επίσης θησαυροφυλάκια που διαχειρίζονται το σύστημα:

```csharp
public async Task<IReadOnlyList<VaultRegistryEntry>> EnsureAllSystemManagedVaultsAsync(
    CancellationToken cancellationToken = default)
{
    var definitions = GetAllResolvedDefinitions();
    var entries = new List<VaultRegistryEntry>(definitions.Count);

    foreach (var definition in definitions)
    {
        entries.Add(await EnsureResolvedSystemManagedVaultAsync(definition, cancellationToken));
    }

    return entries;
}
```

Το σύστημα δημιουργεί και διαχειρίζεται αυτόματα τα ακόλουθα θησαυροφυλάκια:

- **hagiprojectdata**: αποθήκευση δεδομένων έργου που χρησιμοποιείται για την αποθήκευση της διαμόρφωσης και της κατάστασης του έργου
- **προσωπικά δεδομένα**: αποθήκευση προσωπικών δεδομένων που χρησιμοποιείται για την αποθήκευση των προτιμήσεων των χρηστών
- **hbsprompt**: μια βιβλιοθήκη προτύπων προτροπής που χρησιμοποιείται για τη διαχείριση των προτροπών τεχνητής νοημοσύνης που χρησιμοποιούνται συνήθως

Αυτά τα θησαυροφυλάκια αρχικοποιούνται αυτόματα κατά την εκκίνηση του συστήματος, επομένως οι χρήστες δεν χρειάζεται να τα διαμορφώσουν με μη αυτόματο τρόπο. Κάποια πράγματα απλώς αφήνονται καλύτερα στο σύστημα αντί να ανησυχούν οι άνθρωποι για αυτά.

## Μηχανισμός ελέγχου πρόσβασης

Ένα σημαντικό μέρος του σχεδιασμού είναι ο έλεγχος πρόσβασης. Το σύστημα χωρίζει τα θησαυροφυλάκια σε δύο τύπους πρόσβασης:

```typescript
export interface VaultForText {
  id: string;
  name: string;
  type: string;
  physicalPath: string;
  accessType: 'read' | 'write';  // Key: distinguish read-only from editable
}
```

- **αναφορά (μόνο για ανάγνωση)**: Το AI χρησιμοποιείται μόνο για ανάλυση και κατανόηση και δεν μπορεί να τροποποιήσει το περιεχόμενο. Κατάλληλο για έργα ανοιχτού κώδικα αναφοράς, έγγραφα και παρόμοιο υλικό
- **επεξεργάσιμο (επεξεργάσιμο)**: Το AI μπορεί να τροποποιήσει το περιεχόμενο όπως απαιτείται για την εργασία. Κατάλληλο για τις σημειώσεις, τα προσχέδια και τα παρόμοια υλικά σας

Αυτή η διάκριση έχει σημασία. Λέει στην τεχνητή νοημοσύνη ποιο περιεχόμενο είναι "αναφορά μόνο για ανάγνωση" και ποιο περιεχόμενο είναι "ασφαλές για επεξεργασία", μειώνοντας τον κίνδυνο τυχαίων αλλαγών. Εξάλλου, κανείς δεν θέλει να εξαφανιστεί η σκληρή δουλειά του εξαιτίας μιας ακούσιας επεξεργασίας.

Ακολουθεί ένα πλήρες παράδειγμα κλήσης frontend:

const createCodeRefVault = async () => {

Χρειάζεται μόνο να παρέχετε τις βασικές πληροφορίες και να αφήσετε το σύστημα να χειριστεί τα υπόλοιπα. Είναι ειλικρινά μια αρκετά χωρίς ανησυχία προσέγγιση.

## Βέλτιστες πρακτικές και πράγματα που πρέπει να προσέξετε

Κατά τη χρήση του συστήματος Vault, συνοψίσαμε μερικά διδάγματα.

### Ασφάλεια διαδρομής

Το σύστημα επικυρώνει αυστηρά μονοπάτια για την αποτροπή επιθέσεων διέλευσης μονοπατιών:

```csharp
private static string ResolveFilePath(string vaultRoot, string relativePath)
{
    var rootPath = EnsureTrailingSeparator(Path.GetFullPath(vaultRoot));
    var combinedPath = Path.GetFullPath(Path.Combine(rootPath, relativePath));
    if (!combinedPath.StartsWith(rootPath, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode,
            "Vault file paths must stay inside the registered vault root.");
    }
    return combinedPath;
}
```

Αυτό είναι σημαντικό. Εάν προσαρμόσετε μια διαδρομή αποθήκευσης, βεβαιωθείτε ότι παραμένει εντός του επιτρεπόμενου εύρους, διαφορετικά το σύστημα θα απορρίψει τη λειτουργία. Δεν μπορείτε πραγματικά να δώσετε υπερβολική έμφαση στην ασφάλεια.

### Διαχείριση υπομονάδας Git

Το CodeRef Vault συνιστά υπομονάδες Git αντί για απευθείας αντιγραφή κώδικα:

```csharp
private static string BuildCodeRefAgentsContent()
{
    return """
    # CodeRef Vault Guide

    Repositories under `repos/` should be maintained through Git submodules
    rather than copied directly into the vault root.

    Keep this structure stable so assistants and tools can understand the vault quickly.
    """ + Environment.NewLine;
}
```

Αυτό έχει πολλά πλεονεκτήματα: διατήρηση του κώδικα συγχρονισμένο με το upstream, εξοικονόμηση χώρου στο δίσκο και διευκόλυνση της διαχείρισης πολλαπλών εκδόσεων του κώδικα. Τελικά, ποιος θέλει να κατεβάζει το ίδιο πράγμα ξανά και ξανά;

### Όρια προεπισκόπησης αρχείων

Για την αποφυγή προβλημάτων απόδοσης, το σύστημα περιορίζει το μέγεθος και τον τύπο του αρχείου:

```csharp
private const int FileEnumerationLimit = 500;
private const int PreviewByteLimit = 256 * 1024;  // 256KB
```

Εάν το θησαυροφυλάκιό σας περιέχει μεγάλο αριθμό αρχείων ή πολύ μεγάλα αρχεία, η απόδοση της προεπισκόπησης ενδέχεται να επηρεαστεί. Σε αυτήν την περίπτωση, μπορείτε να εξετάσετε το ενδεχόμενο να επεξεργαστείτε αρχεία κατά παρτίδες ή να χρησιμοποιήσετε εξειδικευμένα εργαλεία αναζήτησης. Μερικές φορές, όταν κάτι γίνεται πολύ μεγάλο, γίνεται πιο δύσκολο να το χειριστείς, όχι πιο εύκολο.

### Διαγνωστικές πληροφορίες

Όταν δημιουργείτε ένα θησαυροφυλάκιο, το σύστημα επιστρέφει διαγνωστικές πληροφορίες για να βοηθήσει με τον εντοπισμό σφαλμάτων:

```csharp
List<VaultBootstrapDiagnosticDto> bootstrapDiagnostics = [];

if (IsCodeRefVaultType(normalizedType))

Εάν η δημιουργία αποτύχει, μπορείτε να επιθεωρήσετε τις διαγνωστικές πληροφορίες για να κατανοήσετε τη συγκεκριμένη αιτία. Όταν κάτι πάει στραβά, ο έλεγχος των διαγνωστικών είναι συχνά ο πιο άμεσος τρόπος για να προχωρήσουμε.

## Περίληψη

Μέσω ενός ενιαίου στρώματος αφαίρεσης αποθήκευσης, το σύστημα Vault επιλύει πολλά βασικά σημεία πόνου της αναπαραγωγής έργων στην εποχή της τεχνητής νοημοσύνης:

- **Κεντρική διαχείριση γνώσης**: όλοι οι πόροι μάθησης συγκεντρώνονται σε ένα μέρος αντί να είναι διάσπαρτοι παντού
- **Αυτόματη ένεση περιβάλλοντος τεχνητής νοημοσύνης**: Οι βοηθοί τεχνητής νοημοσύνης μπορούν να κατανοήσουν αυτόματα τους διαθέσιμους πόρους εκμάθησης χωρίς μη αυτόματη ρύθμιση περιβάλλοντος
- **Επαναχρησιμοποίηση γνώσης μεταξύ έργων**: η γνώση μπορεί να μοιραστεί και να επαναχρησιμοποιηθεί σε πολλαπλά μαθησιακά έργα
- **Τυποποιημένη δομή καταλόγου**: μια συνεπής διάταξη καταλόγου μειώνει την καμπύλη εκμάθησης

Αυτή η λύση έχει ήδη επικυρωθεί στο έργο HagiCode. Εάν κατασκευάζετε επίσης εργαλεία που σχετίζονται με την ανάπτυξη υποβοηθούμενη από AI ή αντιμετωπίζετε παρόμοια προβλήματα διαχείρισης γνώσης, ελπίζω ότι αυτές οι εμπειρίες μπορούν να χρησιμεύσουν ως χρήσιμη αναφορά.

Στην πραγματικότητα, η αξία μιας τεχνικής λύσης δεν έγκειται στο πόσο περίπλοκη είναι, αλλά στο αν επιλύει πραγματικά προβλήματα. Η βασική ιδέα του συστήματος Vault είναι πολύ απλή: δημιουργήστε ένα ενοποιημένο επίπεδο αποθήκευσης γνώσης που να μπορεί να κατανοήσει η τεχνητή νοημοσύνη. Ωστόσο, αυτή ακριβώς η απλή αφαίρεση βελτίωσε αρκετά την αποδοτικότητα ανάπτυξής μας.

Μερικές φορές η απλή προσέγγιση είναι πραγματικά η καλύτερη. Άλλωστε, τα περίπλοκα πράγματα συχνά κρύβουν ακόμα περισσότερες παγίδες...
