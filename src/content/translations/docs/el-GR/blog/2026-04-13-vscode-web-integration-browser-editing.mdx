## VSCode και code-server: Επιλογή λύσης επεξεργασίας κώδικα που βασίζεται σε πρόγραμμα περιήγησης

> Κατά τη δημιουργία δυνατοτήτων επεξεργασίας κώδικα βάσει προγράμματος περιήγησης, οι προγραμματιστές αντιμετωπίζουν μια βασική επιλογή: να χρησιμοποιήσουν το επίσημο πρόγραμμα του VSCode `code serve-web` χαρακτηριστικό ή υιοθετήστε την κοινότητα με γνώμονα `code-server` λύση; Αυτή η απόφαση επηρεάζει όχι μόνο την τεχνική αρχιτεκτονική, αλλά και τη συμμόρφωση με τις άδειες και την ευελιξία ανάπτυξης.

## φόντο

Η τεχνική επιλογή μοιάζει πολύ με την επιλογή ενός μονοπατιού στη ζωή. Μόλις επιλέξετε ένα, συνήθως πρέπει να συνεχίσετε να το περπατάτε και η αλλαγή αργότερα μπορεί να γίνει πολύ ακριβή.

Στην εποχή του προγραμματισμού με τη βοήθεια AI, η επεξεργασία κώδικα με βάση το πρόγραμμα περιήγησης γίνεται όλο και πιο σημαντική. Οι χρήστες αναμένουν ότι αφού ένας βοηθός τεχνητής νοημοσύνης ολοκληρώσει την ανάλυση του κώδικα, θα μπορούν να ανοίξουν αμέσως ένα πρόγραμμα επεξεργασίας στην ίδια περίοδο λειτουργίας προγράμματος περιήγησης και να κάνουν αλλαγές χωρίς να αλλάξουν εφαρμογές. Αυτό το είδος απρόσκοπτης εμπειρίας θα πρέπει απλώς να υπάρχει όταν το χρειάζεστε.

Ωστόσο, κατά την εφαρμογή αυτής της δυνατότητας, οι προγραμματιστές αντιμετωπίζουν μια κρίσιμη τεχνική επιλογή: εάν χρησιμοποιούν το επίσημο VSCode `code serve-web` χαρακτηριστικό ή την κοινότητα `code-server` λύση;

Κάθε επιλογή έχει τα δικά της δυνατά σημεία και συμβιβασμούς, και η κακή επιλογή μπορεί να δημιουργήσει πολλά προβλήματα αργότερα. Η αδειοδότηση είναι ένα παράδειγμα: αν ανακαλύψετε μόνο μετά την κυκλοφορία ότι το προϊόν σας δεν είναι συμβατό με άδεια χρήσης, είναι ήδη πολύ αργά. Η ανάπτυξη είναι ένα άλλο: μια λύση μπορεί να λειτουργήσει τέλεια στην ανάπτυξη και στη συνέχεια να αντιμετωπίσει όλα τα είδη προβλημάτων μόλις μεταφερθεί σε κοντέινερ. Αυτά είναι ακριβώς τα είδη των παγίδων που θέλουν να αποφύγουν οι ομάδες.

## Διαφορές αδειοδότησης (το πιο σημαντικό)

Αυτή είναι η πιο θεμελιώδης διαφορά μεταξύ των δύο λύσεων και ο πρώτος παράγοντας που λάβαμε υπόψη κατά την αξιολόγηση. Όταν κάνετε μια τεχνική επιλογή, είναι σημαντικό να κατανοήσετε εκ των προτέρων τους νομικούς κινδύνους.

**διακομιστής κώδικα**
- Άδεια MIT, πλήρως ανοιχτού κώδικα
- Συντηρείται από την Coder.com με μια ενεργή κοινότητα
- Δωρεάν για εμπορική χρήση, τροποποίηση και διανομή
- Χωρίς περιορισμούς σε σενάρια χρήσης

**Κωδικός VSC `code serve-web`**
- Μέρος του προϊόντος Microsoft VSCode
- Χρησιμοποιεί την άδεια χρήσης της Microsoft (η άδεια VS Code περιλαμβάνει περιορισμούς στην εμπορική χρήση)
- Προορίζεται κυρίως για μεμονωμένους προγραμματιστές
- Η ανάπτυξη της επιχείρησης ενδέχεται να απαιτεί πρόσθετη εμπορική αναθεώρηση αδειοδότησης

Από την άποψη της αδειοδότησης, ο διακομιστής κώδικα είναι πιο φιλικός προς τα εμπορικά έργα. Αυτό είναι κάτι που πρέπει να σκεφτείτε κατά τον προγραμματισμό του προϊόντος, γιατί η μετεγκατάσταση αργότερα μπορεί να είναι πολύ δαπανηρή.

## Διαφορές ανάπτυξης

Μόλις διευθετηθεί η αδειοδότηση, το επόμενο ζήτημα είναι η ανάπτυξη. Αυτό επηρεάζει άμεσα το κόστος λειτουργίας και τον αρχιτεκτονικό σχεδιασμό σας.

**διακομιστής κώδικα**
- Μια αυτόνομη εφαρμογή Node.js που μπορεί να αναπτυχθεί ανεξάρτητα
- Υποστηρίζει πολλαπλές πηγές χρόνου εκτέλεσης:
  - Καθορίζοντας απευθείας την εκτελέσιμη διαδρομή
  - Αναζητώντας το μέσω του συστήματος PATH
  - Αυτόματος εντοπισμός περιβάλλοντος NVM Node.js 22.x
- Δεν χρειάζεται να εγκαταστήσετε την εφαρμογή επιφάνειας εργασίας VSCode στον διακομιστή
- Πιο εύκολο να αναπτυχθεί σε κοντέινερ

**Κωδικός VSC `code serve-web`**
- Πρέπει να εξαρτάται από ένα τοπικά εγκατεστημένο VSCode CLI
- Απαιτεί ένα διαθέσιμο `code` εντολή στον κεντρικό υπολογιστή
- Το σύστημα φιλτράρει τα περιτυλίγματα VS Code Remote CLI
- Σχεδιασμένο κυρίως για σενάρια τοπικής ανάπτυξης

ο διακομιστής κώδικα είναι πιο κατάλληλος για σενάρια ανάπτυξης διακομιστή και κοντέινερ. Εάν το προϊόν σας πρέπει να εκτελεστεί στο Docker ή οι χρήστες σας δεν έχουν εγκαταστήσει το VSCode, ο διακομιστής κώδικα είναι συνήθως η σωστή επιλογή.

## Διαφορές παραμέτρων χαρακτηριστικών

Οι δύο λύσεις διαφέρουν επίσης σε μερικές παραμέτρους χαρακτηριστικών. Οι διαφορές δεν είναι τεράστιες, αλλά μπορούν να δημιουργήσουν τριβή ενοποίησης στην πραγματική χρήση.

| Χαρακτηριστικό | διακομιστής κωδικών | υπηρεσία κώδικα-ιστός |
|------|-------------|----------------|
| Διαδρομή δημόσιας βάσης | `/` (με δυνατότητα διαμόρφωσης) | `/vscode-server` (διορθώθηκε) |
| Έλεγχος ταυτότητας | `--auth` παράμετρος με πολλαπλούς τρόπους λειτουργίας | `--connection-token` / `--without-connection-token` |
| Κατάλογος δεδομένων | `{DataDir}/code-server` | `{DataDir}/vscode-serve-web` |
| Τηλεμετρία | Απενεργοποιημένο από προεπιλογή με `--disable-telemetry` | Εξαρτάται από τις ρυθμίσεις VSCode |
| Ενημερώστε τους ελέγχους | Μπορεί να απενεργοποιηθεί με `--disable-update-check` | Εξαρτάται από τις ρυθμίσεις VSCode |

Αυτές οι διαφορές χρειάζονται ιδιαίτερη προσοχή κατά την ενσωμάτωση. Για παράδειγμα, διαφορετικές διαδρομές URL σημαίνουν ότι ο κώδικας διεπαφής σας χρειάζεται αποκλειστικό χειρισμό.

## Διαφορές ανίχνευσης διαθεσιμότητας

Κατά την εφαρμογή της εναλλαγής επεξεργαστή, η λογική ανίχνευσης διαθεσιμότητας διαφέρει επίσης.

**διακομιστής κώδικα**
- Επιστρέφεται πάντα ως ορατή υλοποίηση
- Εμφανίζεται ακόμα και όταν δεν είναι διαθέσιμο, με ένα `install-required` κατάσταση
- Υποστηρίζει την αυτόματη ανίχνευση ενός περιβάλλοντος NVM Node.js 22.x

**εξυπηρέτηση κώδικα-ιστός**
- Ορατό μόνο όταν είναι τοπικός `code` Ανιχνεύεται CLI
- Εάν δεν είναι διαθέσιμο, το frontend αποκρύπτει αυτόματα αυτήν την επιλογή
- Εξαρτάται από την τοπική κατάσταση εγκατάστασης VSCode

Αυτή η διαφορά επηρεάζει άμεσα την εμπειρία του χρήστη. ο διακομιστής κώδικα είναι πιο διαφανής: οι χρήστες μπορούν να δουν την επιλογή και να κατανοήσουν ότι η εγκατάσταση εξακολουθεί να απαιτείται. κώδικας υπηρεσίας-ιστός είναι πιο κρυφός: οι χρήστες μπορεί να μην συνειδητοποιούν καν ότι υπάρχει η επιλογή. Ποια προσέγγιση είναι καλύτερη εξαρτάται από τη θέση του προϊόντος.

## Αρχιτεκτονική Διπλής Εφαρμογής της HagiCode

Μετά από εις βάθος ανάλυση, το έργο HagiCode υιοθέτησε μια αρχιτεκτονική διπλής υλοποίησης που υποστηρίζει και τις δύο λύσεις σε αρχιτεκτονικό επίπεδο.

### Προεπιλογή σε διακομιστή κώδικα

```csharp
// The default active implementation is code-server
// If an explicit activeImplementation is saved, try that implementation first
// If the requested implementation is unavailable, the resolver tries the other one
// If a fallback occurs, return fallbackReason
```

Από προεπιλογή χρησιμοποιούμε διακομιστή κώδικα κυρίως λόγω της ευελιξίας αδειοδότησης και ανάπτυξης. Ωστόσο, για τους χρήστες που έχουν ήδη ένα τοπικό περιβάλλον VSCode, η υπηρεσία διακομιστή κώδικα είναι επίσης μια σταθερή επιλογή.

### Επιλογέας υλοποίησης

`CodeServerImplementationResolver` είναι υπεύθυνος για την ενοποίηση:
- Επιλογή εφαρμογής κατά την εκκίνηση της προθέρμανσης
- Επιλογή εφαρμογής κατά την ανάγνωση της κατάστασης
- Επιλογή υλοποίησης κατά το άνοιγμα έργων
- Επιλογή υλοποίησης κατά το άνοιγμα Θησαυροθηκών

Αυτός ο σχεδιασμός επιτρέπει στο σύστημα να ανταποκρίνεται με ευελιξία σε διαφορετικά σενάρια και οι χρήστες μπορούν να επιλέξουν την υλοποίηση που ταιριάζει καλύτερα με το περιβάλλον τους.

### Κανόνες προσαρμογής Frontend

```typescript
// When localCodeAvailable=false, do not show code serve-web
// When localCodeAvailable=true, show the code serve-web configuration
```

Το frontend εμφανίζει αυτόματα τις διαθέσιμες επιλογές με βάση το περιβάλλον, έτσι ώστε οι χρήστες να μην μπερδεύονται με τις λειτουργίες που δεν μπορούν να χρησιμοποιήσουν.

## Πρακτικός Οδηγός

Μετά από όλη αυτή τη θεωρία, τι πρέπει να προσέχετε κατά την πραγματική ανάπτυξη; Τελικά, σημασία έχει η εφαρμογή.

### Σύσταση ανάπτυξης Docker

Για την ανάπτυξη με κοντέινερ, ο διακομιστής κώδικα είναι η καλύτερη επιλογή:

```dockerfile
# Use the official code-server image directly
FROM codercom/code-server:latest

# Or install through npm
RUN npm install -g code-server
```

Αυτό λύνει το πρόβλημα σε ένα μόνο επίπεδο χωρίς να απαιτείται πρόσθετη εγκατάσταση VSCode.

### Παραδείγματα διαμόρφωσης

**διαμόρφωση διακομιστή κώδικα**
```json
{
  "vscodeServer": {
    "enabled": true,
    "activeImplementation": "code-server",
    "codeServer": {
      "host": "0.0.0.0",
      "port": 8080,
      "executablePath": "",
      "authMode": "none"
    }
  }
}
```

**διαμόρφωση υπηρεσίας κώδικα-ιστού**
```json
{
  "vscodeServer": {
    "enabled": true,
    "activeImplementation": "serve-web",
    "serveWeb": {
      "host": "0.0.0.0",
      "port": 8080,
      "executablePath": "/usr/local/bin/code"
    }
  }
}
```

Η διαμόρφωση μπορεί να είναι λίγο κουραστική την πρώτη φορά, αλλά μόλις εφαρμοστεί, τα πράγματα γίνονται πολύ πιο εύκολα στη συντήρηση.

### Διαφορές κατασκευής URL

**διακομιστής κώδικα**
```
http://localhost:8080/?folder=/path/to/project&vscode-lang=zh-CN
```

**εξυπηρέτηση κώδικα-ιστός**
```
http://localhost:8080/vscode-server/?folder=/path/to/project&tkn=xxx&vscode-lang=zh-CN
```

Δώστε προσοχή στις διαφορές στα μονοπάτια και τις παραμέτρους. Πρέπει να τα χειριστείτε ξεχωριστά κατά την ενσωμάτωση.

### Εφαρμογές μεταγωγής

Το σύστημα υποστηρίζει την εναλλαγή χρόνου εκτέλεσης και διακόπτει αυτόματα την προηγούμενη υλοποίηση κατά την εναλλαγή:

```csharp
// VsCodeServerManager automatically handles mutual exclusion
// When switching activeImplementation, the old implementation will not keep running in the background
```

Αυτός ο σχεδιασμός επιτρέπει στους χρήστες να δοκιμάσουν διαφορετικές υλοποιήσεις ανά πάσα στιγμή και να βρουν την επιλογή που τους ταιριάζει καλύτερα.

### Παρακολούθηση Κατάστασης

```typescript
const { settings, runtime } = await getVsCodeServerSettings();

// runtime.activeImplementation: "code-server" | "serve-web"
// runtime.fallbackReason: reason for switching
// runtime.status: "running" | "starting" | "stopped" | "unhealthy"
```

Όταν η κατάσταση είναι ορατή, οι χρήστες μπορούν γρήγορα να προσδιορίσουν εάν ένα πρόβλημα προέρχεται από την πλευρά του διακομιστή ή από τη δική τους λειτουργία.

## Συμπέρασμα

| Διάσταση σύγκρισης | διακομιστής κωδικών | υπηρεσία κώδικα-ιστός | Σύσταση |
|---------|-------------|----------------|------|
| Άδεια | MIT (φιλικό προς το εμπόριο) | Microsoft (περιορισμένη) | διακομιστής κωδικών |

| Καταλληλότητα διακομιστή | Σχεδιασμένο για διακομιστές | Κυρίως για την τοπική ανάπτυξη | διακομιστής κωδικών |
