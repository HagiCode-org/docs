## Οδηγός για τη δημιουργία εφέ κινούμενης εικόνας με ελαφρύ σάρωμα περιγράμματος

> Πώς δημιουργείτε αυτό το σημαντικό στοιχείο που παρατηρούν οι χρήστες με μια ματιά χρησιμοποιώντας καθαρό CSS; Στην πραγματικότητα δεν είναι τόσο δύσκολο. Το κόλπο είναι απλώς να ακολουθήσετε μια ελαφρώς κυκλική διαδρομή. Σε αυτό το άρθρο, θα σας καθοδηγήσω στον τρόπο δημιουργίας ενός κινούμενου σχεδίου σάρωσης με φως περιγράμματος από την αρχή, και επίσης θα μοιραστώ μερικές από τις παγίδες που συναντήσαμε κατά τη δημιουργία του HagiCode.

## φόντο

Εάν εργάζεστε στο frontend, πιθανότατα είχατε αυτήν την εμπειρία στο παρελθόν: ένας διαχειριστής προϊόντων μιλάει με αυτήν την έκφραση "αυτό είναι σίγουρα απλό" και λέει: "Μπορούμε να προσθέσουμε κάποιο είδος ειδικού εφέ σε αυτήν την τρέχουσα εργασία, ώστε οι χρήστες να μπορούν να το εντοπίσουν αμέσως;"

Λέτε σίγουρα, μπορούμε απλώς να αλλάξουμε το χρώμα του περιγράμματος. Μετά κουνάνε το κεφάλι τους με αυτό το βλέμμα που λέει, «Δεν το καταλαβαίνεις». Απαντούν, "Αυτό δεν είναι αρκετά προφανές. Θέλω το είδος του εφέ όπου το φως τρέχει γύρω από τα σύνορα, όπως σε μια ταινία επιστημονικής φαντασίας."

Σε εκείνο το σημείο μπορεί να αρχίσετε να αναρωτιέστε πώς να το φτιάξετε. Καμβάς; SVG; Ή μπορεί το CSS να το χειριστεί μόνο του; Σε τελική ανάλυση, κανείς δεν θέλει να παραδεχτεί ότι δεν ξέρει πώς.

Στις σύγχρονες διαδικτυακές εφαρμογές, τα κινούμενα σχέδια σάρωσης φωτός περιγράμματος είναι στην πραγματικότητα πολύ συνηθισμένα. Χρησιμοποιούνται κυρίως σε μερικά σενάρια όπως αυτά:

- **Ενδείξεις κατάστασης**: Επισήμανση εργασιών σε εξέλιξη ή ενεργών στοιχείων
- **Οπτική εστίαση**: Επισήμανση σημαντικών περιοχών περιεχομένου
- **Βελτίωση επωνυμίας**: Δημιουργία ενός κομψού, μοντέρνου οπτικού στυλ με προοπτική τεχνολογίας
- **Εποχιακά θέματα**: Δημιουργία εορταστικής ατμόσφαιρας για ειδικές περιστάσεις

Αντιμετωπίσαμε ακριβώς αυτήν την απαίτηση κατά την κατασκευή [HagiCode](https://hagicode.com). Οι χρήστες έπρεπε να δουν με μια ματιά ποιες περίοδοι λειτουργίας εκτελούνταν και ποιες προτάσεις επεξεργάζονταν αυτήν τη στιγμή. Δοκιμάσαμε πολλές διαφορετικές προσεγγίσεις. Κάποια μονοπάτια ήταν πιο ομαλά, άλλα ήταν λίγο πιο στροφές. Τελικά, καταλήξαμε σε μια αρκετά ώριμη στρατηγική υλοποίησης.

## Βασικές Ιδέες Εφαρμογής

Αφού αναλύσαμε τη βάση κώδικα HagiCode, συνοψίσαμε διάφορα βασικά μοτίβα υλοποίησης παρακάτω. Το καθένα ταιριάζει σε διαφορετικό σενάριο, ή με άλλα λόγια, το καθένα υπάρχει για κάποιο λόγο.

### 1. Περιστρεφόμενη λάμψη με κωνική κλίση (πιο συνηθισμένη)

Αυτός είναι ο κλασικός τρόπος για να εφαρμόσετε ένα εφέ σάρωσης φωτός περιγράμματος. Η βασική ιδέα είναι να χρησιμοποιήσετε CSS `conic-gradient` για να δημιουργήσετε μια κωνική κλίση και, στη συνέχεια, περιστρέψτε την συνεχώς. Σαν ένα φως του δρόμου που γυρίζει τη νύχτα, απλώς κάνει κύκλους.

**Βασικά στοιχεία:**
- Χρησιμοποιήστε το `::before` ψευδοστοιχείο για τη δημιουργία του στρώματος λάμψης
- Χρήση `conic-gradient` για να ορίσετε την κατανομή χρώματος ντεγκραντέ
- Χρησιμοποιήστε το `::after` ψευδοστοιχείο για κάλυψη της κεντρικής περιοχής (προαιρετικό)
- Χρήση `@keyframes` για να εφαρμόσετε το κινούμενο σχέδιο περιστροφής

### 2. Πλαϊνή γραμμή λάμψης

Αυτό λειτουργεί καλά για δείκτες κατάστασης σε στοιχεία λίστας. Δημιουργείτε μια λεπτή λαμπερή γραμμή στη μία πλευρά του στοιχείου αντί να κινείτε ολόκληρο το περίγραμμα. Μερικές φορές αρκεί λίγο φως. Δεν χρειάζεται να φωτίσεις όλο τον κόσμο.

**Βασικά στοιχεία:**
- Ένα λεπτό απολύτως τοποθετημένο στοιχείο γραμμής
- Χρήση `box-shadow` για να δημιουργήσετε το εφέ λάμψης
- Χρήση `scale` και `opacity` για ένα κινούμενο σχέδιο αναπνοής

### 3. Λαμπερό φόντο με κουτί-σκιά

Εάν δεν χρειάζεστε το εφέ πλήρους σάρωσης και θέλετε απλώς μια απαλή λάμψη φόντου, πολλαπλών στρώσεων `box-shadow` οι τιμές είναι αρκετές. Μερικές φορές η απλούστερη επιλογή είναι η καλύτερη.

### 4. Υποστήριξη προσβασιμότητας

Αυτό το μέρος είναι εύκολο να παραβλεφθεί, αλλά είναι εξαιρετικά σημαντικό. Κάθε κινούμενο σχέδιο πρέπει να λαμβάνει υπόψη το `prefers-reduced-motion` ερωτήματα πολυμέσων και παρέχουν μια στατική εναλλακτική λύση για χρήστες που δεν θέλουν κινούμενα σχέδια. Δεν απολαμβάνουν όλοι τη συνεχή κίνηση και ο σεβασμός αυτής της προτίμησης έχει σημασία.

## Επιλογές Υλοποίησης

### Επιλογή 1: Περιστρεφόμενο περίγραμμα κωνικής κλίσης (συνιστάται)

Αυτή είναι η πιο ολοκληρωμένη εφαρμογή του εφέ φωτός σαρωτικού περιγράμματος και είναι επίσης η επιλογή που χρησιμοποιούμε πιο συχνά στο HagiCode. Τελικά, αν κάτι λειτουργεί καλά, γιατί να το αντικαταστήσετε;

```css
/* Parent container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotating glow layer */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Mask layer (optional, for creating a hollow border effect) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Η αρχή πίσω από αυτήν την επιλογή είναι αρκετά απλή: δημιουργήστε ένα ψευδοστοιχείο μεγαλύτερο από το γονικό κοντέινερ, σχεδιάστε μια κωνική κλίση πάνω του και περιστρέψτε το συνεχώς. Το μητρικό δοχείο χρησιμοποιεί `overflow: hidden`, έτσι μόνο το φως που περνά γύρω από το περίγραμμα παραμένει ορατό. Είναι λίγο σαν να παρακολουθείς ένα φως του δρόμου μέσα από ένα παράθυρο. Βλέπεις μόνο τη μικρή φέτα που περνάει.

### Επιλογή 2: Απλοποιημένο περιστρεφόμενο φωτεινό περίγραμμα

Εάν δεν χρειάζεστε το πλήρες εφέ, το HagiCode περιλαμβάνει επίσης μια ελαφρύτερη έκδοση κατηγορίας βοηθητικών προγραμμάτων. Μερικές φορές η απλούστερη προσέγγιση είναι πραγματικά καλύτερη.

```css
/* Rotating light border utility class */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Accessibility support */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Παρατηρήστε το `will-change: transform` εδώ. Λέει στο πρόγραμμα περιήγησης, "Αυτό το στοιχείο θα συνεχίσει να αλλάζει", ώστε το πρόγραμμα περιήγησης να μπορεί να προετοιμάσει ορισμένες βελτιστοποιήσεις εκ των προτέρων και να διατηρήσει την κινούμενη εικόνα πιο ομαλή. Η προετοιμασία εκ των προτέρων είναι συνήθως καλύτερη από το να σκαρφαλώνεις την τελευταία στιγμή.

### Επιλογή 3: Πλαϊνή γραμμή λάμψης

Αυτό είναι ιδιαίτερα κατάλληλο για δείκτες κατάστασης στοιχείων λίστας και είναι ακριβώς αυτό που χρησιμοποιεί η λίστα συνεδριών HagiCode. Μια λεπτή γραμμή μπορεί ακόμα να ξεχωρίζει ανάμεσα σε πολλά αντικείμενα. Αυτό μοιάζει με ένα μάθημα ζωής με τον δικό του τρόπο.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Αυτό χρησιμοποιεί `isolation: isolate` για να δημιουργήσετε ένα νέο πλαίσιο στοίβαξης, στη συνέχεια βασίζεται σε `z-index` για να ελέγξετε τη σειρά εμφάνισης κάθε επιπέδου. `pointer-events: none` είναι επίσης απαραίτητο. Διαφορετικά, το ψευδοστοιχείο θα μπλοκάρει τα κλικ των χρηστών. Κάποια πράγματα μπορεί να φαίνονται ωραία, αλλά και πάλι δεν πρέπει να εμποδίζουν.

### Επιλογή 4: Τυλίξτε το σε ένα στοιχείο React

Εάν το έργο σας χρησιμοποιεί το React, μπορείτε να τυλίξετε αυτήν τη λογική σε ένα στοιχείο, ειδικά στον χειρισμό προσβασιμότητας. Γράψτε το μια φορά, χρησιμοποιήστε το πολλές φορές. Αυτή είναι η όλη ουσία.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Η αντίστοιχη ενότητα CSS:

```css
/* GlowBorder.module.css */

/* Animated version */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Static version (accessibility) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Το `useReducedMotion` γάντζο από `framer-motion` εντοπίζει αυτόματα τις προτιμήσεις συστήματος του χρήστη. Εάν ο χρήστης έχει ενεργοποιήσει τη μειωμένη κίνηση, επιστρέφει `true`, και το στοιχείο εμφανίζει τη στατική έκδοση. Ο σεβασμός στις προτιμήσεις του χρήστη έχει μεγαλύτερη σημασία από το να επιβάλλεις ένα εντυπωσιακό εφέ.

## Πρακτικά Μαθήματα

Αυτά είναι μερικά από τα μαθήματα που μάθαμε κατά τη δημιουργία του HagiCode. Θα μπορούσατε επίσης να τα ονομάσετε σημάδια μάχης. Ας ελπίσουμε ότι θα σας βοηθήσουν να αποφύγετε κάποιες παρακάμψεις.

### 1. Σύστημα μεταβλητής θέματος

Οι μεταβλητές CSS κάνουν την υποστήριξη πολλών θεμάτων ιδιαίτερα βολική. Κανείς δεν θέλει να επεξεργάζεται ένα σωρό κώδικα κάθε φορά που αλλάζει το θέμα.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Usage */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

Με αυτόν τον τρόπο, η εναλλαγή θεμάτων απαιτεί μόνο αλλαγή της κλάσης στο `html` στοιχείο και κάθε χρώμα κινουμένων σχεδίων ενημερώνεται αυτόματα. Μία βάση κώδικα, δύο στυλ. Αυτό ακριβώς θέλουμε.

### 2. Βελτιστοποίηση απόδοσης

**Χρήση `will-change` για να υποδείξετε το πρόγραμμα περιήγησης για βελτιστοποίηση:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Ενημερώστε το πρόγραμμα περιήγησης εκ των προτέρων και θα σας βοηθήσει να βελτιστοποιήσετε. Πολλά πράγματα στη ζωή λειτουργούν καλύτερα με λίγη προετοιμασία.

**Αποφύγετε τη χρήση σύνθετων κουτιών-σκιών σε μεγάλα στοιχεία:**

```css
/* Not ideal - using a blurred shadow on a large element */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Better - use a pseudo-element to limit the glowing area */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Το δοκιμάσαμε στο HagiCode. Η προσθήκη μιας θολής σκιάς απευθείας σε μια μεγάλη κάρτα μείωσε τους ρυθμούς καρέ κύλισης κάτω από 30 fps. Η εναλλαγή σε ένα ψευδοστοιχείο επανέφερε τα πράγματα σε σταθερά 60 fps. Οι χρήστες μπορούν να νιώσουν απολύτως αυτή τη διαφορά.

### 3. Προσβασιμότητα

Πραγματικά δεν πρέπει να το παραλείψετε αυτό. Μερικοί χρήστες βρίσκουν το animation που προκαλεί ζάλη ή αποσπά την προσοχή και ο σεβασμός των προτιμήσεών τους αποτελεί μέρος της δημιουργίας ενός καλού προϊόντος. Τα όμορφα πράγματα δεν χρειάζεται να επιβάλλονται σε όλους.

**Ερώτημα μέσων CSS:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Provide a static fallback */
    opacity: 1;
  }
}
```

**Εντοπισμός προτίμησης χρήστη στο React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Έλεγχος επιπέδου έντασης

Ο δείκτης απόδοσης Token στο HagiCode δείχνει διαφορετικά χρώματα λάμψης με βάση τη διεκπεραίωση σε πραγματικό χρόνο και αυτό υλοποιείται δυναμικά. Οι διαφορετικές καταστάσεις πρέπει να εκφράζονται διαφορετικά.

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

| Πράγματα που πρέπει να προσέξετε | Εξήγηση |
|---------|------|
| **διαχείριση δείκτη z** | Το στρώμα λάμψης θα πρέπει να χρησιμοποιεί ένα κατάλληλο `z-index` έτσι δεν παρεμβαίνει στην αλληλεπίδραση περιεχομένου |
| **δείκτες-γεγονότα** | Το ψευδοστοιχείο λάμψης πρέπει να ρυθμιστεί `pointer-events: none` |
| **Υπερχείλιση ορίων** | Το μητρικό δοχείο χρειάζεται `overflow: hidden`, ή πρέπει να προσαρμόσετε το μέγεθος ψευδοστοιχείων |
| **Επίδραση στην απόδοση** | Τα σύνθετα κινούμενα σχέδια μπορούν να βλάψουν την απόδοση σε κινητές συσκευές, γι' αυτό δοκιμάστε προσεκτικά |
| **Σκοτεινή λειτουργία** | Βεβαιωθείτε ότι το χρώμα λάμψης παραμένει καθαρά ορατό σε σκούρο φόντο |
| **Εναλλαγή θέματος** | Χρησιμοποιήστε μεταβλητές CSS, ώστε τα χρώματα των κινούμενων εικόνων να ενημερώνονται σωστά όταν αλλάζει το θέμα |

### 6. Συμβουλές εντοπισμού σφαλμάτων

Τα ψευδοστοιχεία μπορεί να είναι λίγο δύσκολο να εντοπιστούν στα εργαλεία προγραμματιστών, επομένως μπορείτε να προσθέσετε προσωρινά ένα περίγραμμα για να ελέγξετε τη θέση.

```css
/* Temporarily show pseudo-element boundaries for debugging */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Αφού ολοκληρώσετε την τοποθέτησή του, θυμηθείτε να σχολιάσετε ή να αφαιρέσετε αυτήν τη γραμμή. Διαφορετικά, η παραγωγή μπορεί να γίνει πολύ γρήγορα δύσκολη. Μερικά πράγματα είναι καλύτερα να αφεθούν στην ανάπτυξη.

## Περίληψη

Τα κινούμενα σχέδια σάρωσης φωτός περιγράμματος δεν είναι ούτε ιδιαίτερα δύσκολα ούτε πραγματικά ασήμαντα. Στον πυρήνα, η φόρμουλα είναι `conic-gradient` συν την περιστροφή, αλλά εάν θέλετε καλή απόδοση, δυνατότητα συντήρησης και προσβασιμότητα, υπάρχουν ακόμα πολλές λεπτομέρειες υλοποίησης που πρέπει να χειριστείτε προσεκτικά.
