## Γρήγορη επεξεργασία κώδικα στο πρόγραμμα περιήγησης: Ενσωμάτωση ιστού VSCode στην πράξη

## Αρχιτεκτονικός Σχεδιασμός

Η ενσωμάτωση VSCode Web της HagiCode χρησιμοποιεί μια διαχωρισμένη αρχιτεκτονική από το front-end/back-end.

### Επίπεδο υπηρεσίας Frontend

Το frontend τυλίγει τις αλληλεπιδράσεις με το backend `vscodeServerService.ts`:

```typescript
// Open project
export async function openProjectInCodeServer(
  id: string,
  currentInterfaceLanguage?: string,
): Promise<VsCodeServerLaunchResponseDto>

// Open vault
export async function openVaultInCodeServer(
  id: string,
  path?: string,
  currentInterfaceLanguage?: string,
): Promise<VsCodeServerLaunchResponseDto>
```

Η διαφορά μεταξύ αυτών των δύο μεθόδων είναι απλή: `openProjectInCodeServer` ανοίγει ολόκληρο το έργο, ενώ `openVaultInCodeServer` ανοίγει μια συγκεκριμένη διαδρομή μέσα σε ένα Vault. Για έργα πολλαπλών αποθετηρίων MonoSpecs, το σύστημα δημιουργεί αυτόματα ένα αρχείο χώρου εργασίας. Οι ξεκάθαρες ευθύνες αρκούν συχνά όταν κάθε μέρος κάνει καλά τη δουλειά του.

### Επίπεδο υπηρεσίας Backend

Το backend `VaultAppService.cs` εφαρμόζει την βασική λογική:

```csharp
public async Task<VsCodeServerLaunchResponseDto> OpenInCodeServerAsync(
    string id,
    string? relativePath = null,
    string? currentInterfaceLanguage = null,
    CancellationToken cancellationToken = default)
{
    // 1. Get settings and check whether the feature is enabled
    var settings = await _vsCodeServerSettingsService.GetResolvedSettingsAsync(cancellationToken);
    if (!settings.Enabled) {
        throw new BusinessException(VsCodeServerErrorCodes.Disabled, "VSCode Server is disabled.");
    }

    // 2. Get vault and resolve the launch directory
    var vault = await RequireVaultAsync(id, cancellationToken);
    var launchDirectory = ResolveLaunchDirectory(vault, relativePath);

    // 3. Ensure code-server is running and get runtime info
    var runtime = await _vsCodeServerManager.EnsureStartedAsync(settings, cancellationToken);

    // 4. Resolve language settings
    var language = _vsCodeServerSettingsService.ResolveLaunchLanguage(
        settings.Language,
        currentInterfaceLanguage);

    // 5. Build launch URL
    return new VsCodeServerLaunchResponseDto {
        LaunchUrl = AppendQueryString(runtime.BaseUrl, new Dictionary<string, string?> {
            ["folder"] = launchDirectory,
            ["tkn"] = runtime.ConnectionToken,
            ["vscode-lang"] = language,
        }),
        ConnectionToken = runtime.ConnectionToken,
        OpenMode = "folder",
        Runtime = VsCodeServerSettingsService.MapRuntime(
            await _vsCodeServerManager.GetRuntimeSnapshotAsync(cancellationToken)),
    };
}
```

Αυτή η μέθοδος έχει μια πολύ σαφή ευθύνη: ελέγξτε τις ρυθμίσεις, επιλύστε διαδρομές, ξεκινήστε την υπηρεσία και δημιουργήστε τη διεύθυνση URL. Μεταξύ αυτών, το `ResolveLaunchDirectory` Η μέθοδος εκτελεί ελέγχους ασφαλείας διαδρομής για να αποτρέψει επιθέσεις διέλευσης διαδρομής. Ο κώδικας μπορεί να μοιάζει λίγο με ποίηση όταν κάθε γραμμή έχει έναν σκοπό.

### Αυτόματη διαχείριση χρόνου εκτέλεσης

Το backend διαχειρίζεται τη διαδικασία του διακομιστή κώδικα μέσω `VsCodeServerManager`:

- Ελέγξτε την κατάσταση της διαδικασίας
- Αυτόματη έναρξη των διακοπείσων υπηρεσιών
- Επιστρέψτε στιγμιότυπα χρόνου εκτέλεσης, όπως θύρα, αναγνωριστικό διεργασίας και ώρα έναρξης

Αυτός ο σχεδιασμός επιτρέπει στο σύστημα να χειρίζεται αυτόματα τον κύκλο ζωής του διακομιστή κώδικα, έτσι ώστε οι χρήστες να μην χρειάζεται να διαχειρίζονται τις διαδικασίες υπηρεσίας με μη αυτόματο τρόπο. Η ζωή είναι ήδη αρκετά περίπλοκη. οτιδήποτε μπορεί να αυτοματοποιηθεί θα πρέπει να είναι.

### Συγχρονισμός γλώσσας

Το HagiCode υποστηρίζει μια πολύγλωσση διεπαφή και ο διακομιστής κώδικα πρέπει να ακολουθεί αυτή τη ρύθμιση. Το σύστημα υποστηρίζει τρεις λειτουργίες γλώσσας:

- `follow`: ακολουθήστε την τρέχουσα γλώσσα διεπαφής

## Ενσωμάτωση Frontend

Η διεπαφή HagiCode χρησιμοποιεί React + TypeScript και η ενσωμάτωση του διακομιστή κώδικα δεν είναι ιδιαίτερα περίπλοκη.

### Κουμπί γρήγορης δράσης

Προσθέστε ένα κουμπί Code Server στην κάρτα έργου:

```tsx
// QuickActionsZone.tsx
<Button
  size="sm"
  variant="default"
  onClick={() => onAction({ type: 'open-code-server' })}
>
  <Globe className="h-3 w-3 mr-1" />
  <span className="text-xs">{t('project.openCodeServer')}</span>
</Button>
```

Αυτό το κουμπί ενεργοποιεί την ανοιχτή ενέργεια και καλεί το backend API για να αποκτήσει τη διεύθυνση URL εκκίνησης. Ένα κουμπί, μια δράση, άμεση και απλή.

### Χειρισμός της Ανοιχτής Δράσης

```tsx
const handleAction = async (action: ProjectAction) => {
  if (action.type === 'open-code-server') {
    const response = await openProjectInCodeServer(project.id, i18n.language);
    window.open(response.launchUrl, '_blank', 'noopener,noreferrer');
  }
};
```

Χρήση `window.open` για να ανοίξετε τον διακομιστή κώδικα σε μια νέα καρτέλα. Το `noopener,noreferrer` οι παράμετροι παρέχουν επιπλέον ασφάλεια. Όσον αφορά την ασφάλεια, δεν υπάρχει τέτοιο πράγμα όπως να είσαι πολύ προσεκτικός.

Τα έργα και τα Vaults χρησιμοποιούν τον ίδιο ανοιχτό μηχανισμό, ο οποίος διατηρεί την αλληλεπίδραση σταθερή. Η συνέπεια έχει σημασία σχεδόν όσο και το ίδιο το χαρακτηριστικό.

## Λογική κατασκευής URL

Η μορφή URL για τον διακομιστή κώδικα έχει μερικές λεπτομέρειες που αξίζει να σημειωθούν.

**Λειτουργία φακέλου**:
```
http://{host}:{port}/?folder={path}&tkn={token}&vscode-lang={lang}
```

**Λειτουργία χώρου εργασίας**:
```
http://{host}:{port}/?workspace={workspacePath}&tkn={token}&vscode-lang={lang}
```

Εδώ, `tkn` είναι το διακριτικό σύνδεσης. Δημιουργείται αυτόματα κάθε φορά που ξεκινά ο διακομιστής κώδικα, εξασφαλίζοντας ασφαλή πρόσβαση. Το `vscode-lang` Η παράμετρος ελέγχει τη γλώσσα διεπαφής χρήστη του επεξεργαστή. Κάθε μία από αυτές τις παραμέτρους έχει να παίξει έναν ρόλο.

## Θέματα ασφαλείας

Κατά την εφαρμογή της ενοποίησης κώδικα-διακομιστή, η ασφάλεια αξίζει ιδιαίτερης προσοχής. Εάν η ασφάλεια πάει στραβά, το παρατηρείτε πάντα πολύ αργά.

### Token σύνδεσης

Το `connection-token` δημιουργείται τυχαία και δεν πρέπει να εκτίθεται. Χρησιμοποιείται καλύτερα στο HTTPS για να αποτρέψει την υποκλοπή του διακριτικού από έναν άνθρωπο στη μέση. Αξίζει να προστατεύονται σωστά οι ευαίσθητες πληροφορίες.

### Ασφάλεια διαδρομής αρχείων

Το backend υλοποιεί ελέγχους διέλευσης διαδρομής:

```csharp
private static string ResolveLaunchDirectory(VaultRegistryEntry vault, string? relativePath)
{
    var vaultRoot = EnsureTrailingSeparator(Path.GetFullPath(vault.PhysicalPath));
    var combinedPath = Path.GetFullPath(Path.Combine(vaultRoot, relativePath ?? "."));
    if (!combinedPath.StartsWith(vaultRoot, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode, "Relative path traversal detected.");
    }
    return combinedPath;
}
```

Αυτός ο κωδικός διασφαλίζει ότι οι χρήστες δεν μπορούν να χρησιμοποιήσουν `../` ή παρόμοια μοτίβα για πρόσβαση σε αρχεία εκτός του καταλόγου Vault. Οι έλεγχοι ορίων γίνονται πάντα καλύτερα παρά παραλείπονται.

### Έλεγχος αδειών

Η διαδικασία του διακομιστή κώδικα θα πρέπει να εκτελείται με τα κατάλληλα δικαιώματα χρήστη, έτσι ώστε να μην μπορεί να έχει πρόσβαση σε ευαίσθητα αρχεία συστήματος. Είναι καλύτερο να εκτελείτε την υπηρεσία διακομιστή κώδικα υπό έναν αποκλειστικό χρήστη. Ο έλεγχος αδειών είναι ένα από εκείνα τα βασικά στοιχεία που πρέπει να διατηρείτε πάντα σε ισχύ.

## Βελτιστοποίηση απόδοσης

Ο διακομιστής κώδικα καταναλώνει πόρους διακομιστή, επομένως ακολουθούν μερικές προτάσεις βελτιστοποίησης:

- Παρακολουθήστε τη χρήση της CPU και της μνήμης και προσαρμόστε τα όρια πόρων όταν χρειάζεται
- Τα μεγάλα έργα ενδέχεται να απαιτούν μεγαλύτερα χρονικά διαστήματα
- Εφαρμόστε αυτόματη εκκαθάριση χρονικού ορίου περιόδου σύνδεσης για την απελευθέρωση πόρων
- Εξετάστε την προσωρινή αποθήκευση για να μειώσετε τους επαναλαμβανόμενους υπολογισμούς

Το HagiCode παρέχει ένα API παρακολούθησης κατάστασης χρόνου εκτέλεσης και το frontend μπορεί να καλέσει `getVsCodeServerSettings()` για να ανακτήσετε την τρέχουσα κατάσταση:

```typescript
const { settings, runtime } = await getVsCodeServerSettings();
// runtime.status: 'disabled' | 'stopped' | 'starting' | 'running' | 'unhealthy'
// runtime.baseUrl: "http://localhost:8080"
// runtime.processId: 12345
```

Αυτός ο σχεδιασμός επιτρέπει στους χρήστες να κατανοούν με σαφήνεια την κατάσταση της υγείας του διακομιστή κωδικών και να εντοπίζουν γρήγορα προβλήματα όταν κάτι πάει στραβά. Όταν η κατάσταση είναι ορατή, οι άνθρωποι αισθάνονται ότι έχουν περισσότερο έλεγχο.

## Στοιχεία εμπειρίας χρήστη

Κατά την υλοποίηση, ανακαλύψαμε μερικές λεπτομέρειες που επηρεάζουν αισθητά την εμπειρία του χρήστη και αξίζουν ιδιαίτερης προσοχής.

Το άνοιγμα του διακομιστή κωδικών για πρώτη φορά μπορεί να απαιτεί αναμονή για την εκκίνηση και αυτή η καθυστέρηση μπορεί να κυμαίνεται από μερικά δευτερόλεπτα έως μισό λεπτό. Είναι καλή ιδέα να εμφανίζεται μια κατάσταση φόρτωσης στο frontend, ώστε οι χρήστες να γνωρίζουν ότι το σύστημα εξακολουθεί να λειτουργεί. Η αναμονή είναι ευκολότερη όταν υπάρχει ανατροφοδότηση.

Τα προγράμματα περιήγησης ενδέχεται να αποκλείσουν το αναδυόμενο παράθυρο, επομένως θα πρέπει να ζητηθεί από τους χρήστες να το επιτρέψουν με μη αυτόματο τρόπο. Κατά την πρώτη εκκίνηση, το HagiCode εμφανίζει καθοδήγηση που εξηγεί πώς να εκχωρήσετε τα απαραίτητα δικαιώματα προγράμματος περιήγησης. Η εμπειρία χρήστη συχνά ζει ακριβώς σε αυτές τις μικρές λεπτομέρειες.

Είναι επίσης καλή ιδέα να εμφανίζεται η κατάσταση χρόνου εκτέλεσης, όπως η εκκίνηση, η εκτέλεση ή το σφάλμα, έτσι ώστε όταν προκύπτουν προβλήματα, οι χρήστες να μπορούν γρήγορα να καταλάβουν εάν το πρόβλημα βρίσκεται στην πλευρά του διακομιστή ή στη δική τους λειτουργία. Γνωρίζοντας πού βρίσκεται το πρόβλημα, σας δίνει τουλάχιστον ένα μέρος για να ξεκινήσετε.

## Παράδειγμα διαμόρφωσης

Η διαμόρφωση για τον διακομιστή κώδικα δεν είναι περίπλοκη:

```json
{
  "vscodeServer": {
    "enabled": true,
    "host": "0.0.0.0",
    "port": 8080,
    "language": "follow"
  }
}
```

`enabled` ελέγχει εάν η δυνατότητα είναι ενεργοποιημένη, `host` και `port` ορίστε τη διεύθυνση ακρόασης και `language` ρυθμίζει τη λειτουργία γλώσσας. Αυτές οι ρυθμίσεις μπορούν να τροποποιηθούν μέσω του UI και να τεθούν σε ισχύ αμέσως. Τα απλά πράγματα είναι συχνά τα πιο εύκολα στη χρήση.

## Συμπέρασμα

Η ενσωμάτωση VSCode Web της HagiCode παρέχει μια κομψή λύση: επιτρέπει στον βοηθό AI και την εμπειρία επεξεργασίας κώδικα να συνδέονται απρόσκοπτα. Με την ενσωμάτωση του διακομιστή κώδικα στο πρόγραμμα περιήγησης, οι χρήστες μπορούν να ενεργήσουν γρήγορα στα αποτελέσματα ανάλυσης AI και να ολοκληρώσουν την πλήρη ροή από την ανάλυση έως την επεξεργασία στην ίδια περίοδο λειτουργίας προγράμματος περιήγησης.

Αυτή η λύση φέρνει πολλά βασικά πλεονεκτήματα: μια ενοποιημένη εμπειρία, επειδή τα έργα και τα Vaults χρησιμοποιούν τον ίδιο ανοιχτό μηχανισμό. υποστήριξη πολλαπλών αποθετηρίων, επειδή τα έργα MonoSpecs δημιουργούν αυτόματα χώρους εργασίας. και ελεγχόμενη ασφάλεια, χάρη στην παρακολούθηση της κατάστασης χρόνου εκτέλεσης και στους ελέγχους ασφάλειας διαδρομής.

Η προσέγγιση που κοινοποιείται σε αυτό το άρθρο είναι κάτι που το HagiCode αποστάχθηκε από πραγματική εργασία ανάπτυξης. Εάν βρίσκετε αυτή τη λύση πολύτιμη, αυτό υποδηλώνει ότι η μηχανολογική μας πρακτική κάνει κάτι σωστά και το ίδιο το HagiCode μπορεί να αξίζει μια πιο προσεκτική ματιά. Τα καλά εργαλεία αξίζουν να τα δουν περισσότεροι άνθρωποι.
