## Βαθιά ενοποίηση και πρακτική χρήση του StreamJsonRpc στο HagiCode

> Αυτό το άρθρο εξηγεί λεπτομερώς πώς το έργο HagiCode (πρώην PCode) ενσωμάτωσε με επιτυχία τη βιβλιοθήκη επικοινωνίας StreamJsonRpc της Microsoft για να αντικαταστήσει την αρχική προσαρμοσμένη εφαρμογή JSON-RPC, ενώ επιλύει επίσης τα τεχνικά προβλήματα και τις αρχιτεκτονικές προκλήσεις που αντιμετωπίζονται κατά τη διαδικασία ενσωμάτωσης.

## φόντο

Το StreamJsonRpc είναι η επίσημα συντηρούμενη βιβλιοθήκη επικοινωνίας JSON-RPC της Microsoft για .NET και TypeScript, γνωστή για την ισχυρή ασφάλεια τύπου, την αυτόματη δημιουργία διακομιστή μεσολάβησης και τους μηχανισμούς χειρισμού ώριμων εξαιρέσεων. Στο έργο HagiCode, η ομάδα αποφάσισε να ενσωματώσει το StreamJsonRpc προκειμένου να επικοινωνήσει με εξωτερικά εργαλεία τεχνητής νοημοσύνης, όπως το iflow CLI και το OpenCode CLI μέσω του ACP (Πρωτόκολλο επικοινωνίας με πράκτορα), εξαλείφοντας επίσης το κόστος συντήρησης και τα πιθανά σφάλματα που εισήγαγε η προηγούμενη προσαρμοσμένη εφαρμογή JSON-RPC. Ωστόσο, η διαδικασία ενσωμάτωσης αντιμετώπισε προκλήσεις ειδικά για τη ροή JSON-RPC, ειδικά κατά τον χειρισμό της δέσμευσης στόχου διακομιστή μεσολάβησης και της αναγνώρισης γενικών παραμέτρων.

Για να αντιμετωπίσουμε αυτά τα σημεία πόνου, λάβαμε μια τολμηρή απόφαση: να ξαναχτίσουμε ολόκληρο το σύστημα κατασκευής από την αρχή. Ο αντίκτυπος αυτής της απόφασης μπορεί να είναι ακόμη μεγαλύτερος από ό,τι περιμένετε, και θα το εξηγήσω λεπτομερώς σύντομα.

## Ανάλυση

Το τρέχον έργο βρίσκεται σε ένα κρίσιμο στάδιο ενοποίησης του πρωτοκόλλου ACP και αντιμετωπίζει τα ακόλουθα τεχνικά σημεία πόνου και αρχιτεκτονικές προκλήσεις:

### 1. Περιορισμοί της προσαρμοσμένης υλοποίησης
Η αρχική υλοποίηση JSON-RPC βρίσκεται στο `src/HagiCode.ClaudeHelper/AcpImp/` και περιλαμβάνει εξαρτήματα όπως `JsonRpcEndpoint` και `ClientSideConnection`. Η διατήρηση αυτής της προσαρμοσμένης βάσης κωδικών είναι δαπανηρή και δεν διαθέτει τις προηγμένες δυνατότητες μιας ώριμης βιβλιοθήκης, όπως η αναφορά προόδου και η υποστήριξη ακύρωσης.

### 2. Εμπόδια ενσωμάτωσης StreamJsonRpc
Κατά την προσπάθεια μετεγκατάστασης του υπάρχοντος `CallbackProxyTarget` μοτίβο στο StreamJsonRpc, διαπιστώσαμε ότι το `_rpc.AddLocalRpcTarget(target)` Η μέθοδος δεν μπορούσε να αναγνωρίσει στόχους που δημιουργήθηκαν μέσω του μοτίβου διακομιστή μεσολάβησης. Συγκεκριμένα, το StreamJsonRpc δεν μπορούσε να διαχωρίσει αυτόματα τις ιδιότητες του γενικού τύπου `T` στις παραμέτρους της μεθόδου RPC, με αποτέλεσμα η πλευρά του διακομιστή να αποτύχει κατά την επεξεργασία των κλήσεων μεθόδου που ξεκινούν από τον πελάτη.

### 3. Μπερδεμένη Αρχιτεκτονική Επίστρωση
Το υπάρχον `ClientSideConnection` αναμιγνύει το επίπεδο μεταφοράς (WebSocket/Stdio), το επίπεδο πρωτοκόλλου (JSON-RPC) και το επίπεδο επιχείρησης (διεπαφή ACP Agent), οδηγώντας σε ασαφείς ευθύνες. Υποφέρει επίσης από έλλειψη δεσμεύσεων μεθόδων `AcpAgentCallbackRpcAdapter`.

### 4. Μητρώα που λείπουν
Το επίπεδο μεταφοράς WebSocket δεν διαθέτει καταγραφή ακατέργαστου περιεχομένου JSON, γεγονός που καθιστά δύσκολο τον προσδιορισμό κατά τον εντοπισμό σφαλμάτων RPC εάν ένα πρόβλημα προέρχεται από τη σειριοποίηση ή από το δίκτυο.

## Λύση

Για να αντιμετωπίσουμε τα παραπάνω προβλήματα, υιοθετήσαμε την ακόλουθη συστηματική λύση, βελτιστοποιώντας από τρεις διαστάσεις: αρχιτεκτονική ανακατασκευή, ενοποίηση βιβλιοθήκης και βελτιωμένος εντοπισμός σφαλμάτων.

### 1. Πλήρης μετεγκατάσταση στο StreamJsonRpc

#### Καταργήστε τον κωδικό παλαιού τύπου
Διαγραφή `JsonRpcEndpoint.cs`, `AgentSideConnection.cs`, και σχετικούς μετατροπείς προσαρμοσμένης σειριοποίησης, όπως `JsonRpcMessageJsonConverter`.

#### Ενσωματώστε την Επίσημη Βιβλιοθήκη
Εισαγάγετε το `StreamJsonRpc` Το πακέτο NuGet και χρησιμοποιήστε το `JsonRpc` τάξη για να χειριστεί την βασική λογική επικοινωνίας.

#### Περίληψη του επιπέδου μεταφοράς
Ορίστε το `IAcpTransport` διεπαφή για να χειριστεί και τα δύο `WebSocket` και `Stdio` τρόπους μεταφοράς με ενοποιημένο τρόπο, διασφαλίζοντας ότι το επίπεδο πρωτοκόλλου είναι αποσυνδεδεμένο από το επίπεδο μεταφοράς.

```csharp
// Definition of the IAcpTransport interface
public interface IAcpTransport
{
    Task SendAsync(string message, CancellationToken cancellationToken = default);
    Task<string> ReceiveAsync(CancellationToken cancellationToken = default);
    Task CloseAsync(CancellationToken cancellationToken = default);
}

// WebSocket transport implementation
public class WebSocketTransport : IAcpTransport
{
    private readonly WebSocket _webSocket;

    public WebSocketTransport(WebSocket webSocket)
    {
        _webSocket = webSocket;
    }

    // Implement send and receive methods
    // ...
}

// Stdio transport implementation
public class StdioTransport : IAcpTransport
{
    private readonly StreamReader _reader;
    private readonly StreamWriter _writer;

    public StdioTransport(StreamReader reader, StreamWriter writer)
    {
        _reader = reader;
        _writer = writer;
    }

    // Implement send and receive methods
    // ...
}
```

### 2. Διορθώστε προβλήματα αναγνώρισης στόχου διακομιστή μεσολάβησης

#### Αναλύστε `CallbackProxyTarget`
Επιθεωρήστε την υπάρχουσα λογική δημιουργίας δυναμικού διακομιστή μεσολάβησης και εντοπίστε τη βασική αιτία γιατί το StreamJsonRpc δεν μπορεί να το αναγνωρίσει. Στις περισσότερες περιπτώσεις, αυτό συμβαίνει επειδή το αντικείμενο διακομιστή μεσολάβησης δεν εκθέτει δημόσια τις πραγματικές υπογραφές μεθόδου ή χρησιμοποιεί τύπους παραμέτρων που δεν υποστηρίζονται από το StreamJsonRpc.

#### Διέλευση παραμέτρου Refactor
Διαχωρίστε τις γενικές ιδιότητες σε ρητές παραμέτρους μεθόδου RPC. Αντί να βασίζεστε σε δυναμικές ιδιότητες, ορίστε συγκεκριμένα DTO Αίτησης/Απόκρισης (αντικείμενα μεταφοράς δεδομένων), ώστε το StreamJsonRpc να μπορεί να αναγνωρίσει σωστά τις υπογραφές μεθόδων μέσω της ανάκλασης.

```csharp
// Original generic property approach
public class CallbackProxyTarget<T>
{
    public Func<T, Task> Callback { get; set; }
}

// Refactored concrete method approach
public class ReadTextFileRequest
{
    public string FilePath { get; set; }
}

public class ReadTextFileResponse
{
    public string Content { get; set; }
}

public interface IAcpAgentCallback
{
    Task<ReadTextFileResponse> ReadTextFileAsync(ReadTextFileRequest request);
    // Other methods...
}
```

#### Χρήση `Attach` Αντί για `AddLocalRpcTarget`
Σε ορισμένα πολύπλοκα σενάρια, η μη αυτόματη διαμεσολάβηση του `JsonRpc` αντικείμενο και χειρισμός `RpcConnection` μπορεί να είναι πιο ευέλικτο από την άμεση προσθήκη ενός τοπικού στόχου.

### 3. Εφαρμόστε τη μέθοδο Binding and Stronger Logging

#### Εφαρμογή `AcpAgentCallbackRpcAdapter`
Βεβαιωθείτε ότι αυτό το στοιχείο υλοποιεί ρητά τη διεπαφή διακομιστή μεσολάβησης StreamJsonRpc και αντιστοιχίζει μεθόδους που ορίζονται από το πρωτόκολλο ACP, όπως π.χ. `ReadTextFileAsync`, στους χειριστές επανάκλησης StreamJsonRpc.

#### Ενσωματώστε την καταγραφή
Υποκλέψτε και καταγράψτε το μη επεξεργασμένο κείμενο των αιτημάτων και των απαντήσεων JSON-RPC στη γραμμή επεξεργασίας μηνυμάτων WebSocket ή Stdio. Χρήση `ILogger` για να εξάγετε το ακατέργαστο ωφέλιμο φορτίο πριν από την ανάλυση και μετά τη σειριοποίηση, ώστε τα προβλήματα μορφοποίησης να μπορούν να διαγνωστούν πιο εύκολα.

```csharp
// Transport wrapper with enhanced logging
public class LoggingAcpTransport : IAcpTransport
{
    private readonly IAcpTransport _innerTransport;
    private readonly ILogger<LoggingAcpTransport> _logger;

    public LoggingAcpTransport(IAcpTransport innerTransport, ILogger<LoggingAcpTransport> logger)
    {
        _innerTransport = innerTransport;
        _logger = logger;
    }

    public async Task SendAsync(string message, CancellationToken cancellationToken = default)
    {
        _logger.LogTrace("Sending message: {Message}", message);
        await _innerTransport.SendAsync(message, cancellationToken);
    }

    public async Task<string> ReceiveAsync(CancellationToken cancellationToken = default)
    {
        var message = await _innerTransport.ReceiveAsync(cancellationToken);
        _logger.LogTrace("Received message: {Message}", message);
        return message;
    }

    public async Task CloseAsync(CancellationToken cancellationToken = default)
    {
        _logger.LogDebug("Closing connection");
        await _innerTransport.CloseAsync(cancellationToken);
    }
}
```

### 4. Refactor Architectural Layering

#### Επίπεδο μεταφοράς (`AcpRpcClient`)
Τυλίξτε τη σύνδεση StreamJsonRpc και καταστήστε την υπεύθυνη `InvokeAsync` διαχείριση του κύκλου ζωής κλήσεων και σύνδεσης.

```csharp
public class AcpRpcClient : IDisposable
{
    private readonly JsonRpc _rpc;
    private readonly IAcpTransport _transport;

    public AcpRpcClient(IAcpTransport transport)
    {
        _transport = transport;
        _rpc = new JsonRpc(new StreamRpcTransport(transport));
        _rpc.StartListening();
    }

    public async Task<TResponse> InvokeAsync<TResponse>(string methodName, object parameters)
    {
        return await _rpc.InvokeAsync<TResponse>(methodName, parameters);
    }

    public void Dispose()
    {
        _rpc.Dispose();
        _transport.Dispose();
    }

    // StreamRpcTransport is the StreamJsonRpc adapter for IAcpTransport
    private class StreamRpcTransport : IDuplexPipe
    {
        // Implement the IDuplexPipe interface
        // ...
    }
}
```

#### Επίπεδο πρωτοκόλλου (`IAcpAgentClient` / `IAcpAgentCallback`)
Καθορίστε σαφείς διεπαφές πελάτη προς πράκτορα και πράκτορα σε πελάτη. Αφαιρέστε το κυκλικό εργοστασιακό μοτίβο `Func<IAcpAgent, IAcpClient>` και αντικαταστήστε το με ένεση εξάρτησης ή άμεση εγγραφή επανάκλησης.

## Εξάσκηση

Με βάση τις βέλτιστες πρακτικές του StreamJsonRpc και την εμπειρία του έργου, οι ακόλουθες είναι οι βασικές συστάσεις για εφαρμογή:

### 1. Οι DTO με έντονη πληκτρολόγηση είναι καλύτεροι από τα δυναμικά αντικείμενα
Το βασικό πλεονέκτημα του StreamJsonRpc έγκειται στην ισχυρή πληκτρολόγηση. Μην χρησιμοποιείτε `dynamic` ή `JObject` να περάσει παραμέτρους. Αντίθετα, ορίστε ρητές κλάσεις C# POCO ως παραμέτρους για κάθε μέθοδο RPC. Αυτό όχι μόνο επιλύει το ζήτημα της αναγνώρισης στόχου διακομιστή μεσολάβησης, αλλά εντοπίζει επίσης σφάλματα τύπου κατά τη στιγμή της μεταγλώττισης.

Παράδειγμα: αντικαταστήστε τις γενικές ιδιότητες στο `CallbackProxyTarget` με συγκεκριμένες κατηγορίες όπως `ReadTextFileRequest` και `WriteTextFileRequest`.

### 2. Δηλώστε ρητά τα ονόματα μεθόδων
Χρησιμοποιήστε το `[JsonRpcMethod]` χαρακτηριστικό για να καθορίσετε ρητά τα ονόματα μεθόδων RPC αντί να βασίζεστε στην προεπιλεγμένη αντιστοίχιση ονόματος μεθόδου. Αυτό αποτρέπει τις αποτυχίες επίκλησης που προκαλούνται από διαφορές στο στυλ ονομασίας, όπως το PascalCase έναντι του camelCase.

```csharp
public interface IAcpAgentCallback
{
    [JsonRpcMethod("readTextFile")]
    Task<ReadTextFileResponse> ReadTextFileAsync(ReadTextFileRequest request);
    
    [JsonRpcMethod("writeTextFile")]
    Task WriteTextFileAsync(WriteTextFileRequest request);
}
```

### 3. Εκμεταλλευτείτε τις Επανακλήσεις Κατάστασης Σύνδεσης
Το StreamJsonRpc παρέχει το `JsonRpc.ConnectionLost` εκδήλωση. Θα πρέπει να ακούσετε οπωσδήποτε αυτό το συμβάν για να χειριστείτε απροσδόκητες εξόδους διεργασιών ή αποσυνδέσεις δικτύου, κάτι που είναι πιο επίκαιρο από το να βασίζεστε μόνο στον εντοπισμό αποτυχίας κόκκων Orleans.

```csharp
_rpc.ConnectionLost += (sender, e) =>
{
    _logger.LogError("RPC connection lost: {Reason}", e.ToString());
    // Handle reconnection logic or notify the user
};
```

### 4. Καταγραφή αρχείων καταγραφής ανά επίπεδο
- **Επίπεδο παρακολούθησης**: Καταγράψτε το πλήρες ωφέλιμο φορτίο αιτήματος/απόκρισης JSON.
- **Επίπεδο εντοπισμού σφαλμάτων**: Καταγράψτε στοίβες κλήσεων μεθόδου και περιλήψεις παραμέτρων.
- **Σημείωση**: Βεβαιωθείτε ότι τα αρχεία καταγραφής δεν περιλαμβάνουν ευαίσθητα διακριτικά εξουσιοδότησης ή περιεχόμενο μεγάλων αρχείων με κωδικοποίηση Base64.

### 5. Χειριστείτε την ειδική φύση της μεταφοράς ροής
Το StreamJsonRpc υποστηρίζει εγγενώς `IAsyncEnumerable`. Όταν εφαρμόζετε απαντήσεις εντολών ροής για ACP, χρησιμοποιήστε `IAsyncEnumerable` απευθείας αντί της προσαρμοσμένης λογικής σελιδοποίησης. Αυτό μπορεί να απλοποιήσει σημαντικά την ποσότητα του κώδικα που απαιτείται για την επεξεργασία ροής.

```csharp
public interface IAcpAgentCallback
{
    [JsonRpcMethod("streamText")]
    IAsyncEnumerable<string> StreamTextAsync(StreamTextRequest request);
}
```

### 6. Μοτίβο προσαρμογέα
Κρατήστε `ACPSession` και `ClientSideConnection` χωριστό. `ACPSession` θα πρέπει να επικεντρωθεί στη διαχείριση της πολιτείας της Ορλεάνης και στην επιχειρηματική λογική, όπως η ουρά μηνυμάτων, και θα πρέπει να χρησιμοποιεί το αντικείμενο σύνδεσης StreamJsonRpc μέσω σύνθεσης και όχι κληρονομικότητας.

## Συμπέρασμα

Με την ολοκληρωμένη ενσωμάτωση του StreamJsonRpc, το έργο HagiCode αντιμετώπισε με επιτυχία το υψηλό κόστος συντήρησης, τους λειτουργικούς περιορισμούς και τη σύγχυση αρχιτεκτονικών στρωμάτων της αρχικής προσαρμοσμένης υλοποίησης. Οι βασικές βελτιώσεις περιλαμβάνουν:

1. Αντικατάσταση δυναμικών ιδιοτήτων με ισχυρά πληκτρολογημένους DTO, βελτιώνοντας τη συντηρησιμότητα και την αξιοπιστία
2. Εφαρμογή αφαίρεσης επιπέδου μεταφοράς και διαχωρισμού επιπέδου πρωτοκόλλου, βελτίωση της αρχιτεκτονικής σαφήνειας
3. Ενίσχυση των δυνατοτήτων καταγραφής για τη διευκόλυνση της διάγνωσης προβλημάτων επικοινωνίας
4. Παρουσίαση υποστήριξης ροής για την απλοποίηση της εφαρμογής ροής

Αυτές οι βελτιώσεις παρέχουν στο HagiCode μια πιο σταθερή και πιο αποτελεσματική βάση επικοινωνίας, επιτρέποντάς του να αλληλεπιδρά καλύτερα με εξωτερικά εργαλεία τεχνητής νοημοσύνης και θέτοντας μια σταθερή βάση για μελλοντική επέκταση των χαρακτηριστικών.
