## Αρχιτεκτονικός Σχεδιασμός

### Επισκόπηση Αρχιτεκτονικής σε επίπεδα

Η ενοποίηση μεταξύ C# και CodeBuddy χρησιμοποιεί μια ξεκάθαρη αρχιτεκτονική πολυεπίπεδη. Αυτός ο σχεδιασμός καθιστά σαφείς τις ευθύνες και διευκολύνει τη μακροπρόθεσμη συντήρηση:

```
┌─────────────────────────────────────────────┐
│           Provider Contract Layer           │
│   AIProviderType enum + extension methods   │
├─────────────────────────────────────────────┤
│           Provider Factory Layer            │
│   AIProviderFactory dependency injection factory │
├─────────────────────────────────────────────┤
│           Provider Implementation Layer     │
│   CodebuddyCliProvider concrete implementation │
├─────────────────────────────────────────────┤
│           ACP Infrastructure Layer          │
│  ACPSessionManager / StdioAcpTransport      │
│  AcpRpcClient / AcpAgentClient              │
└─────────────────────────────────────────────┘
```

Ποια είναι τα οφέλη αυτού του layering; Με απλά λόγια, κάθε στρώμα μένει μακριά από το δρόμο των άλλων. Αν αργότερα θέλουμε να αλλάξουμε τον μηχανισμό επικοινωνίας, για παράδειγμα από `stdio` στο WebSocket, χρειάζεται μόνο να τροποποιήσουμε το κάτω επίπεδο και η επιχειρηματική λογική πάνω από αυτό παραμένει ανέγγιχτη. Κανείς δεν θέλει μια αλλαγή επικοινωνίας να κυματίζει ολόκληρη τη βάση κώδικα.

### Ανάλυση βασικών στοιχείων

Το ** Επίπεδο σύμβασης παρόχου** είναι το θεμέλιο ολόκληρης της αρχιτεκτονικής. Ορίζουμε το `AIProviderType` enum, όπου `CodebuddyCli = 3` χρησιμοποιείται ως τιμή enum και εφαρμόζει αμφίδρομη αντιστοίχιση μεταξύ συμβολοσειρών και αριθμών μέσω μεθόδων επέκτασης. Αυτό επιτρέπει στις συμβολοσειρές σε αρχεία διαμόρφωσης να μετατρέπονται εύκολα σε enums και τα enums να μετατρέπονται ξανά σε συμβολοσειρές για έξοδο εντοπισμού σφαλμάτων.

Το **εργοστασιακό επίπεδο παροχέα** είναι υπεύθυνο για τη δημιουργία της αντίστοιχης παρουσίας Παρόχου με βάση τη διαμόρφωση. Χρησιμοποιεί .NET dependency injection μαζί με `ActivatorUtilities.CreateInstance` για δυναμική δημιουργία. Το πλεονέκτημα του εργοστασιακού μοτίβου είναι ότι κατά την προσθήκη ενός νέου Παρόχου, χρειάζεται μόνο να προσθέσετε τη λογική δημιουργίας αντί να τροποποιήσετε τον υπάρχοντα κώδικα.

Το **Επίπεδο υλοποίησης Provider** είναι το σημείο όπου πραγματοποιείται η πραγματική εργασία. `CodebuddyCliProvider` υλοποιεί το `IAIProvider` διεπαφή και παρέχει δύο λειτουργίες επίκλησης: `ExecuteAsync` για κλήσεις χωρίς ροή και `StreamAsync` για κλήσεις ροής.

Το **επίπεδο υποδομής ACP** παρέχει τη βάση επικοινωνίας από κάτω. Αυτό το επίπεδο χειρίζεται όλες τις λεπτομέρειες πρωτοκόλλου, συμπεριλαμβανομένης της διαχείρισης διεργασιών, της σειριοποίησης μηνυμάτων και της ανάλυσης απόκρισης. Είναι το θεμέλιο που κρατά σταθερά τα πάντα πάνω του.

## Μηχανισμός Επικοινωνίας

### Λειτουργία μεταφοράς Stdio

Το CodeBuddy χρησιμοποιεί το **Stdio (τυπική είσοδος/έξοδος)** για επικοινωνία με εξωτερικές διεργασίες. Η εντολή εκκίνησης είναι απλή:

```bash
codebuddy --acp
```

Μετά από αυτό, τα μηνύματα JSON-RPC ανταλλάσσονται μέσω τυπικής εισόδου και εξόδου. Αυτή η προσέγγιση έχει πολλά πλεονεκτήματα:

1. **Γρήγορη εκκίνηση**: Η τοπική επικοινωνία διεργασιών αποφεύγει την καθυστέρηση δικτύου
2. **Απλή διαμόρφωση**: χρειάζεται μόνο να καθορίσετε την εκτελέσιμη διαδρομή
3. **Απομόνωση περιβάλλοντος**: κάθε συνεδρία εκτελείται σε μια ανεξάρτητη διαδικασία, ώστε να μην επηρεάζει η μία την άλλη

Η έγχυση μεταβλητής περιβάλλοντος υποστηρίζεται κατά την επικοινωνία. Τα κοινά παραδείγματα περιλαμβάνουν:

- `CODEBUDDY_API_KEY`: Έλεγχος ταυτότητας κλειδιού API
- `CODEBUDDY_INTERNET_ENVIRONMENT`: διαμόρφωση περιβάλλοντος δικτύου

Όπως και στην επικοινωνία μεταξύ των ανθρώπων, βοηθάει να επιλέξετε πρώτα ένα βολικό κανάλι.

### Πρωτόκολλο μηνυμάτων

Το ACP βασίζεται στο JSON-RPC 2.0. Η μορφή του μηνύματος μοιάζει περίπου με αυτό:

```json
// Request message
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "agent/prompt",
  "params": {
    "prompt": "Help me write a sorting algorithm",
    "sessionId": "session-123"
  }
}

// Response message
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": "Here is the AI response..."
  }
}
```

Στην πραγματική υλοποίηση, ενσωματώνουμε όλες αυτές τις λεπτομέρειες πρωτοκόλλου, έτσι ώστε το ανώτερο επιχειρηματικό επίπεδο να χρειάζεται μόνο να ενδιαφέρεται για την προτροπή και την απόκριση.

### 2. Ενσωμάτωση του Provider Factory

Προσθέστε έναν κλάδο δημιουργίας CodeBuddy στην εργοστασιακή κλάση:

```csharp
// AIProviderFactory.cs
private IAIProvider? CreateProvider(AIProviderType providerType, ProviderConfiguration config)
{
    return providerType switch
    {
        AIProviderType.CodebuddyCli =>
            ActivatorUtilities.CreateInstance<CodebuddyCliProvider>(
                _serviceProvider,
                Options.Create(config)),
        // ... Other providers
        _ => throw new NotSupportedException($"Provider {providerType} not supported")
    };
}
```

Αυτό χρησιμοποιεί ένεση εξάρτησης μέσω `ActivatorUtilities`, το οποίο χειρίζεται αυτόματα την έγχυση παραμέτρων κατασκευαστή και είναι πολύ βολικό.

### 3. Ολοκληρωμένη Εφαρμογή Παρόχου

Παρακάτω είναι η βασική υλοποίηση του `CodebuddyCliProvider`, καλύπτοντας τόσο τις λειτουργίες επίκλησης ροής όσο και μη ροής:

```csharp
public class CodebuddyCliProvider : IAIProvider
{
    private readonly ILogger<CodebuddyCliProvider> _logger;
    private readonly IACPSessionManager _sessionManager;
    private readonly ProviderConfiguration _config;

    public string Name => "CodebuddyCli";
    public bool SupportsStreaming => true;
    public ProviderCapabilities Capabilities { get; }

    public CodebuddyCliProvider(
        ILogger<CodebuddyCliProvider> logger,
        IACPSessionManager sessionManager,
        IOptions<ProviderConfiguration> config)
    {
        _logger = logger;
        _sessionManager = sessionManager;
        _config = config.Value;

        // Define the capabilities of the current Provider
        Capabilities = new ProviderCapabilities
        {
            SupportsStreaming = true,
            SupportsTools = true,
            SupportsSystemMessages = true,
            SupportsArtifacts = false,
            MaxTokens = 8192
        };
    }

    // Non-streaming call: return all results together after completion
    public async Task<AIResponse> ExecuteAsync(
        AIRequest request,
        CancellationToken cancellationToken = default)
    {
        // Create an independent session for the request
        var session = await _sessionManager.CreateSessionAsync(
            "CodebuddyCli",
            request.WorkingDirectory,
            cancellationToken,
            request.SessionId);

        try
        {
            var fullPrompt = BuildPrompt(request);
            await session.SendPromptAsync(fullPrompt, cancellationToken);

            var responseBuilder = new StringBuilder();
            var toolCalls = new List<AIToolCall>();

            // Collect all response chunks
            await foreach (var chunk in StreamFromSession(session, cancellationToken))
            {
                if (!string.IsNullOrEmpty(chunk.Content))
                {
                    responseBuilder.Append(chunk.Content);
                }
                // Handle tool calls...
            }

            return new AIResponse
            {
                Content = AIResultContentSanitizer.SanitizeResultContent(
                    responseBuilder.ToString()),
                ToolCalls = toolCalls,
                Provider = Name,
                Model = string.Empty
            };
        }
        finally
        {
            // Release session resources
            await session.DisposeAsync();
        }
    }

    // Streaming call: return response chunks in real time
    public async IAsyncEnumerable<AIStreamingChunk> StreamAsync(
        AIRequest request,
        [EnumeratorCancellation] CancellationToken cancellationToken = default)
    {
        var session = await _sessionManager.CreateSessionAsync(
            "CodebuddyCli",
            request.WorkingDirectory,
            cancellationToken);

        try
        {
            var fullPrompt = BuildPrompt(request);
            await session.SendPromptAsync(fullPrompt, cancellationToken);

            await foreach (var chunk in StreamFromSession(session, cancellationToken))
            {
                yield return chunk;
            }
        }
        finally
        {
            await session.DisposeAsync();
        }
    }

    private async IAsyncEnumerable<AIStreamingChunk> StreamFromSession(
        IACPSession session,
        [EnumeratorCancellation] CancellationToken cancellationToken)
    {
        // Iterate through all updates in the session
        await foreach (var notification in session.ReceiveUpdatesAsync(cancellationToken))
        {
            switch (notification.Update)
            {
                case AgentMessageChunkSessionUpdate agentMessage:
                    // Handle text content chunks
                    if (agentMessage.Content is AcpImp.TextContentBlock textContent)
                    {
                        yield return new AIStreamingChunk
                        {
                            Content = textContent.Text,
                            Type = StreamingChunkType.ContentDelta,
                            IsComplete = false
                        };
                    }
                    break;

                case ToolCallSessionUpdate toolCall:
                    // Handle tool calls
                    yield return new AIStreamingChunk
                    {
                        Content = string.Empty,
                        Type = StreamingChunkType.ToolCallDelta,
                        ToolCallDelta = new AIToolCallDelta
                        {
                            Id = toolCall.ToolCallId,
                            Name = toolCall.Kind.ToString(),
                            Arguments = toolCall.RawInput?.ToString()
                        }
                    };
                    break;

                case AcpImp.PromptCompletedSessionUpdate:
                    // Response complete
                    yield break;
            }
        }
    }

    // Build the full prompt
    private string BuildPrompt(AIRequest request, string? embeddedCommandPrompt = null)
    {
        var sb = new StringBuilder();

        // Embedded command prompt, if present
        if (!string.IsNullOrEmpty(embeddedCommandPrompt))
        {
            sb.AppendLine(embeddedCommandPrompt);
            sb.AppendLine();
        }

        // System message
        if (!string.IsNullOrEmpty(request.SystemMessage))
        {
            sb.AppendLine(request.SystemMessage);
            sb.AppendLine();
        }

        // User prompt
        sb.Append(request.Prompt);
        return sb.ToString();
    }
}
```

Υπάρχουν πολλά βασικά σημεία σε αυτόν τον κώδικα:

1. **Διαχείριση περιόδου σύνδεσης**: κάθε αίτημα δημιουργεί μια ανεξάρτητη περίοδο λειτουργίας και απελευθερώνει πόρους μετά την ολοκλήρωση του αιτήματος. Αυτό είναι ένα μάθημα που αντλήθηκε από τη δοκιμή και το λάθος. Εάν η επαναχρησιμοποίηση συνεδρίας δεν αντιμετωπιστεί σωστά, η ρύπανση κατάστασης εμφανίζεται εύκολα.

2. **Επεξεργασία ροής**: `IAsyncEnumerable` επιτρέπει την επιστροφή της απάντησης ενώ δημιουργείται ακόμα, αντί να περιμένει να ολοκληρωθεί όλο το περιεχόμενο. Αυτό είναι ιδιαίτερα σημαντικό για σενάρια μεγάλου κειμένου και βελτιώνει σημαντικά την εμπειρία χρήστη.

3. **Κλήσεις εργαλείων**: Το CodeBuddy υποστηρίζει την κλήση εργαλείων (Κλήση λειτουργιών), που γίνεται μέσω `ToolCallSessionUpdate`. Αυτή η δυνατότητα είναι κρίσιμη για πολύπλοκες εργασίες επεξεργασίας κώδικα.

4. **Φιλτράρισμα περιεχομένου**: `AIResultContentSanitizer` χρησιμοποιείται για να φιλτράρει το περιεχόμενο του μπλοκ Think και να διατηρεί την έξοδο καθαρή.

```yaml

  DefaultProvider: "CodebuddyCli"

  Providers:

      WorkingDirectory: "C:/projects/my-app"

    CodebuddyCli:
      ExecutablePath: "C:/tools/codebuddy.cmd"

      StartupTimeoutMs: 5000
      EnvironmentVariables:
        CODEBUDDY_API_KEY: "${CODEBUDDY_API_KEY}"
        CODEBUDDY_INTERNET_ENVIRONMENT: "production"
```

## Πρακτικά Μαθήματα

### Καταγραφή παγίδων

Αντιμετωπίσαμε πολλές τυπικές παγίδες κατά την εφαρμογή και η κοινοποίησή τους εδώ μπορεί να βοηθήσει άλλους να αποφύγουν τις ίδιες παρακάμψεις:

1. **Πρόβλημα διαρροής περιόδου λειτουργίας**: στην αρχή, οι περίοδοι λειτουργίας δεν κυκλοφόρησαν σωστά, γεγονός που εξαντλούσε τους πόρους της διαδικασίας. Η λύση ήταν η χρήση `try-finally` για να διασφαλιστεί ότι αποδεσμεύονται πόροι για κάθε αίτημα.

2. **Διαβίβαση μεταβλητής περιβάλλοντος**: Τα Windows και το Linux χρησιμοποιούν διαφορετική σύνταξη μεταβλητών περιβάλλοντος, επομένως αργότερα τυποποιήσαμε `Dictionary<string, string?>` να το χειριστεί αυτό.

3. **Διαμόρφωση χρονικού ορίου**: Η εκκίνηση του CLI απαιτεί χρόνο, επομένως ορίσαμε ένα χρονικό όριο εκκίνησης 5 δευτερολέπτων για να αποφύγουμε τις γρήγορες αποτυχίες αιτημάτων.

4. **Προβλήματα κωδικοποίησης**: στα Windows, η προεπιλεγμένη κωδικοποίηση μπορεί να προκαλέσει αλλοιώσεις στο κινεζικό κείμενο, επομένως η κωδικοποίηση UTF-8 καθορίζεται ρητά κατά την έναρξη της διαδικασίας.

### Βελτιστοποίηση απόδοσης

1. **Πισίνα περιόδων σύνδεσης**: για συχνά σύντομα αιτήματα, εξετάστε το ενδεχόμενο να εφαρμόσετε μια ομάδα συνεδριών για επαναχρησιμοποίηση διαδικασιών
2. **Cache σύνδεσης**: η εργοστασιακή κλάση υποστηρίζει ήδη την προσωρινή αποθήκευση παρουσιών Παρόχου
3. **Πρώτα ασυγχρονισμός**: χρησιμοποιήστε ασύγχρονο προγραμματισμό για να αποφύγετε τον αποκλεισμό των νημάτων

Η απόδοση αξίζει πάντα τη βελτιστοποίηση. Όσο περισσότερο περιμένουν οι χρήστες, τόσο χειρότερη γίνεται η εμπειρία.

## Συμπέρασμα

Αυτό το άρθρο εισήγαγε μια ολοκληρωμένη λύση για την ενσωμάτωση του CodeBuddy CLI σε ένα backend C#, καλύπτοντας ολόκληρη τη διαδικασία από τον σχεδιασμό της αρχιτεκτονικής έως την υλοποίηση συγκεκριμένης μορφής. Μέσω μιας πολυεπίπεδης αρχιτεκτονικής, διαχωρίζουμε τις λεπτομέρειες του πρωτοκόλλου από την επιχειρηματική λογική, καθιστώντας τον κώδικα σαφέστερο και ευκολότερο στη συντήρηση.

Βασικά φαγητά:

- Χρησιμοποιήστε μια πολυεπίπεδη αρχιτεκτονική με επίπεδο σύμβασης παρόχου, εργοστασιακό επίπεδο, επίπεδο υλοποίησης και επίπεδο υποδομής
- Χρησιμοποιήστε το JSON-RPC μέσω Stdio για επικοινωνία μεταξύ διεργασιών
- Εφαρμόστε ευέλικτη διαμόρφωση και επεκτασιμότητα μέσω της έγχυσης εξάρτησης
- Παρέχετε λειτουργίες επίκλησης ροής και μη ροής

Αυτή η προσέγγιση δεν είναι κατάλληλη μόνο για το CodeBuddy. Η προσθήκη νέων παρόχων AI μπορεί να ακολουθήσει το ίδιο μοτίβο. Εάν δημιουργείτε επίσης μια παρόμοια ενσωμάτωση πολλών AI-Παρόχων, ελπίζω αυτό το άρθρο να σας προσφέρει μια χρήσιμη αναφορά.
