## Design.md: Μια λύση για συνεπή σχεδίαση διεπαφής χρήστη με βάση το AI

> Στην εποχή της ανάπτυξης frontend με τη βοήθεια AI, πώς μπορείτε να διατηρήσετε συνεπείς τις διεπαφές που δημιουργούνται από AI; Αυτό το άρθρο μοιράζεται την πρακτική μας εμπειρία στη δημιουργία ενός ιστότοπου γκαλερί σχεδιασμού με βάση το awesome-design-md, καθώς και πώς να δημιουργήσετε ένα δομημένο `design.md` για να καθοδηγήσει την τεχνητή νοημοσύνη προς την τυποποιημένη σχεδίαση διεπαφής χρήστη.

## φόντο

Όποιος έχει χρησιμοποιήσει την τεχνητή νοημοσύνη για τη σύνταξη κώδικα διεπαφής είχε πιθανώς παρόμοια εμπειρία: ζητήστε από την τεχνητή νοημοσύνη να δημιουργήσει την ίδια σελίδα πολλές φορές και κάθε αποτέλεσμα βγαίνει με διαφορετικό στυλ. Μερικές φορές οι γωνίες είναι στρογγυλεμένες, μερικές φορές είναι αιχμηρές. Μερικές φορές η απόσταση είναι 8 εικονοστοιχεία και άλλες φορές γίνεται 16 εικονοστοιχεία. Ακόμη και το ίδιο κουμπί μπορεί να φαίνεται διαφορετικό σε διαφορετικές συνομιλίες.

Αυτό δεν είναι ένα μεμονωμένο ζήτημα. Καθώς η ανάπτυξη με τη βοήθεια τεχνητής νοημοσύνης γίνεται πιο συνηθισμένη, η **έλλειψη συνέπειας στο διεπαφή χρήστη διεπαφής που δημιουργείται από AI** έχει γίνει ένα ευρέως διαδεδομένο πρόβλημα. Διαφορετικοί βοηθοί τεχνητής νοημοσύνης, διαφορετικές προτροπές, ακόμη και ο ίδιος βοηθός σε διαφορετικές συνομιλίες μπορούν να παράγουν δραματικά διαφορετικά σχέδια διεπαφής. Αυτό δημιουργεί τεράστιο κόστος συντήρησης κατά την επανάληψη του προϊόντος.

Η βασική αιτία είναι στην πραγματικότητα απλή: δεν υπάρχει **έγγραφο αναφοράς επίσημου σχεδιασμού**. Τα παραδοσιακά φύλλα στυλ CSS μπορούν να πουν στους προγραμματιστές "πώς να εφαρμόσουν" κάτι, αλλά δεν μπορούν να επικοινωνήσουν πλήρως "γιατί έχει σχεδιαστεί με αυτόν τον τρόπο" ή "ποιο μοτίβο σχεδίασης πρέπει να χρησιμοποιηθεί σε ποιο σενάριο". Για την τεχνητή νοημοσύνη, μια σαφής και δομημένη περιγραφή είναι ακόμη πιο σημαντική για την κατανόηση των συμβάσεων σχεδιασμού.

Ταυτόχρονα, η κοινότητα ανοιχτού κώδικα προσφέρει ήδη μερικούς εξαιρετικούς πόρους. Το [VoltAgent/wesome-design-md](https://github.com/VoltAgent/awesome-design-md) Το έργο συλλέγει τεκμηρίωση συστημάτων σχεδιασμού από πολλές γνωστές εταιρείες. Κάθε κατάλογος περιέχει ένα `README.md`, α `DESIGN.md`και προεπισκόπηση HTML. Ωστόσο, όλα αυτά είναι διάσπαρτα στο upstream αποθετήριο, καθιστώντας δύσκολη τη γρήγορη περιήγηση και σύγκριση.

Μπορούμε, λοιπόν, να ενοποιήσουμε αυτούς τους πόρους σε μια γκαλερί σχεδίασης εύκολης περιήγησης, ενώ παράλληλα αποστάζουμε μια δομημένη `design.md` για χρήση AI;

Η απάντηση είναι ναι. Στη συνέχεια, επιτρέψτε μου να περιηγηθώ στην προσέγγισή μας.

### 1. Σωλήνας κατάποσης περιεχομένου

Αυτό είναι το θεμέλιο ολόκληρου του συστήματος, το οποίο είναι υπεύθυνο για την εξαγωγή και την κανονικοποίηση περιεχομένου από πόρους ανάντη.

```typescript
// src/lib/content/awesomeDesignCatalog.ts

export async function scanSourceEntries(): Promise<DesignEntry[]> {
  const designDir = 'vendor/awesome-design-md/design-md';
  const entries: DesignEntry[] = [];

  for (const dir of await fs.readdir(designDir)) {
    const entryPath = path.join(designDir, dir);
    if (await isValidDesignEntry(entryPath)) {
      const entry = await normalizeDesignEntry(entryPath);
      entries.push(entry);
    }
  }

  return entries;
}

async function isValidDesignEntry(dir: string): Promise<boolean> {
  const requiredFiles = ['README.md', 'DESIGN.md'];
  for (const file of requiredFiles) {
    if (!(await fileExists(path.join(dir, file)))) {
      return false;
    }
  }
  return true;
}
```

### 3. Λειτουργικότητα αναζήτησης

Η δυνατότητα αναζήτησης βασίζεται στο φιλτράρισμα από την πλευρά του πελάτη, με την κατάσταση να διατηρείται μέσω των παραμέτρων ερωτήματος URL:

```typescript
// src/components/gallery/SearchToolbar.tsx

function SearchToolbar({ entries }: { entries: DesignEntry[] }) {
  const [query, setQuery] = useState('');
  const [results, setResults] = useState(entries);

  useEffect(() => {
    const params = new URLSearchParams(window.location.search);
    const initialQuery = params.get('q') || '';
    setQuery(initialQuery);
    filterEntries(initialQuery);
  }, []);

  const filterEntries = (searchQuery: string) => {
    const filtered = entries.filter(entry =>
      entry.searchText.toLowerCase().includes(searchQuery.toLowerCase())
    );
    setResults(filtered);
  };

  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const value = e.target.value;
    setQuery(value);
    filterEntries(value);

    // Update the URL without triggering a page refresh
    const newUrl = value
      ? `${window.location.pathname}?q=${encodeURIComponent(value)}`
      : window.location.pathname;
    window.history.replaceState({}, '', newUrl);
  };

  return (
    <div className="search-toolbar">
      <input
        type="text"
        value={query}
        onChange={handleChange}
        placeholder="Search design entries..."
      />
      <span className="result-count">{results.length} results</span>
    </div>
  );
}
```

### Σημειώσεις

**Ασφάλεια**: Η απόδοση Markdown απαιτεί φιλτράρισμα μη ασφαλούς HTML. Τα ενσωματωμένα φίλτρα απόδοσης του Astro `script` ετικέτες από προεπιλογή, αλλά εξακολουθείτε να πρέπει να παρακολουθείτε τους κινδύνους XSS.

**Απόδοση**: Ένας μεγάλος αριθμός προεπισκοπήσεων iframe μπορεί να επηρεάσει την απόδοση της πρώτης βαφής. Συνιστάται η χρήση `loading="lazy"` για να φορτώσετε τεμπέλης σε προεπισκόπηση περιεχομένου.

**Δυνατότητα συντήρησης**: `design.md` πρέπει να παραμείνει σε συγχρονισμό με την εφαρμογή του κώδικα. Συνιστάται η προσθήκη ελέγχων CI για να διασφαλιστεί ότι οι μεταβλητές CSS παραμένουν συνεπείς μεταξύ τεκμηρίωσης και κώδικα.

**Προσβασιμότητα**: Βεβαιωθείτε ότι η χρωματική αντίθεση πληροί το πρότυπο WCAG AA (τουλάχιστον 4,5:1).

### Οδηγός χρήσης AI

Μετά τη δημιουργία `design.md`, πώς κάνετε την τεχνητή νοημοσύνη να τη χρησιμοποιήσει πραγματικά; Ακολουθούν μερικές πρακτικές συμβουλές:

Please refer to the design.md file in the project root and use the design conventions defined there to implement the following components:
- Buttons: use the primary color with an 8px border radius
- Cards: use the elevation-2 shadow level

Implement a navigation bar with the following requirements:
- Use --color-bg-primary for the background color
- Use --color-border-subtle for borders
- Use --text-color-primary for text

### Στρατηγική Δοκιμών

**Δοκιμή αγωγών περιεχομένου**:
- Σενάρια αρχείου που λείπει (λείπει `README.md` ή `DESIGN.md`)
- Σενάρια σφάλματος μορφοποίησης (αποτυχία ανάλυσης Markdown)
- Σενάρια κενού καταλόγου

**Δοκιμή λειτουργιών αναζήτησης**:
- Κενός χειρισμός αποτελεσμάτων
- Ειδικοί χαρακτήρες όπως κινέζικα και emoji
- Επαλήθευση συγχρονισμού διεύθυνσης URL

**Δοκιμή στοιχείων διεπαφής χρήστη**:
- Εναλλαγή ανοιχτού/σκούρου θέματος
- Αποκριτική διάταξη
- Προεπισκόπηση καταστάσεων φόρτωσης

### Ροή εργασιών ανάπτυξης

```bash
# 1. Update the submodule to the latest version
git submodule update --remote

# 2. Rebuild the site
npm run build

# 3. Deploy static assets
npm run deploy
```

Συνιστάται να αυτοματοποιούνται οι ενημερώσεις υπομονάδας καθώς και η κατασκευή και η ανάπτυξη, έτσι ώστε το CI να μπορεί να ενεργοποιείται αυτόματα κάθε φορά που ενημερώνεται το αποθετήριο ανάντη.

## Συμπέρασμα

Η ασυνέπεια στις διεπαφές χρήστη που δημιουργήθηκαν από την τεχνητή νοημοσύνη που αντιμετώπισε το HagiCode κατά την ανάπτυξη προκλήθηκε, στον πυρήνα της, από την έλλειψη ενός δομημένου εγγράφου αναφοράς σχεδιασμού. Χτίζοντας ένα site γκαλερί σχεδιασμού και δημιουργώντας ένα τυποποιημένο `design.md`, λύσαμε με επιτυχία αυτό το πρόβλημα.

Η βασική αξία αυτής της λύσης έγκειται στα εξής:
- **Ενοποιημένοι πόροι**: ενοποίηση της τεκμηρίωσης του συστήματος διάσπαρτης σχεδίασης
- **Δομημένα πρότυπα**: έκφραση συμβάσεων σχεδιασμού σε μορφή που μπορεί να κατανοήσει το AI
- **Συνεχής συντήρηση**: ενημέρωση του περιεχομένου μέσω `git submodule`

Εάν χρησιμοποιείτε επίσης AI για ανάπτυξη frontend, αξίζει να δοκιμάσετε αυτήν την προσέγγιση. Δημιουργία δομημένου `design.md` όχι μόνο βελτιώνει τη συνοχή του κώδικα που δημιουργείται από την τεχνητή νοημοσύνη, αλλά βοηθά επίσης την ομάδα σας να διατηρεί ενοποιημένα πρότυπα σχεδίασης εσωτερικά.
