## Πρακτική ολοκλήρωσης πρακτόρων Hermes: Από το Πρωτόκολλο στην Παραγωγή

> Μοιραστείτε την πλήρη εμπειρία του HagiCode από την ενσωμάτωση του Hermes Agent, συμπεριλαμβανομένων βασικών μαθημάτων σχετικά με την προσαρμογή πρωτοκόλλου ACP, τη διαχείριση του χώρου συγκέντρωσης συνεδριών και τον συγχρονισμό συμβολαίων front-end/back-end.

## φόντο

Κατά την κατασκευή του HagiCode, μιας πλατφόρμας κωδικοποίησης με τη βοήθεια AI, η ομάδα μας χρειαζόταν να ενσωματώσει ένα πλαίσιο Agent που θα μπορούσε να εκτελείται τοπικά και επίσης να κλιμακώνεται στο cloud. Μετά από έρευνα, η Hermes Agent από τη Nous Research επιλέχθηκε ως η υποκείμενη μηχανή για τις δυνατότητες του Αντιπροσώπου γενικής χρήσης.

Στην πραγματικότητα, η επιλογή τεχνολογίας δεν είναι ούτε ιδιαίτερα δύσκολη ούτε ιδιαίτερα εύκολη. Υπάρχουν πολλά ισχυρά πλαίσια Agent στην αγορά, αλλά η Hermes ξεχώρισε επειδή το πρωτόκολλο ACP και το σύστημα εργαλείων της ταιριάζουν ιδιαίτερα στις απαιτητικές απαιτήσεις της HagiCode: τοπική ανάπτυξη, συνεργασία ομάδας και επέκταση στο cloud. Ωστόσο, η εισαγωγή του Ερμή σε ένα πραγματικό σύστημα παραγωγής σήμαινε την επίλυση μιας μεγάλης λίστας προβλημάτων μηχανικής. Αυτό το κομμάτι κάθε άλλο παρά τετριμμένο ήταν.

Η στοίβα του HagiCode χρησιμοποιεί την Ορλεάνη για να δημιουργήσει ένα κατανεμημένο σύστημα, ενώ το μπροστινό μέρος είναι κατασκευασμένο με React + TypeScript. Η ενσωμάτωση του Hermes σήμαινε τη διατήρηση της αρχιτεκτονικής συνέπειας, ενώ παράλληλα έκανε τον Hermes έναν εκτελεστή πρώτης κατηγορίας μαζί με τους ClaudeCode και OpenCode. Ακούγεται αρκετά απλό, αλλά η εφαρμογή λέει πάντα την πραγματική ιστορία.

Αυτό το άρθρο μοιράζεται την πρακτική μας εμπειρία με την ενσωμάτωση του Hermes Agent στο HagiCode και ελπίζουμε να προσφέρει χρήσιμο υλικό αναφοράς για ομάδες που αντιμετωπίζουν παρόμοιες ανάγκες. Άλλωστε, από τη στιγμή που πέσατε σε ένα λάκκο, δεν υπάρχει λόγος να αφήσετε κάποιον άλλο να πέσει στο ίδιο.

## Αρχιτεκτονικός Σχεδιασμός

### Πολυεπίπεδη προσέγγιση σχεδιασμού

Η ενσωμάτωση Hermes της HagiCode χρησιμοποιεί μια ξεκάθαρη πολυεπίπεδη αρχιτεκτονική, με κάθε επίπεδο να εστιάζει στις δικές του ευθύνες:

**Πυρήνα στο πίσω μέρος**
- `HermesCliProvider`: υλοποιεί το `IAIProvider` διεπαφή ως το ενοποιημένο σημείο εισόδου του παρόχου AI
- `HermesPlatformConfiguration`: διαχειρίζεται την εκτελέσιμη διαδρομή Hermes, τα ορίσματα, τον έλεγχο ταυτότητας και τις σχετικές ρυθμίσεις
- `ICliProvider<HermesOptions>`: η αφαίρεση CLI χαμηλού επιπέδου που παρέχεται από το HagiCode.Libs για το χειρισμό των κύκλων ζωής υποδιεργασιών

**Στρώμα μεταφοράς**
- `StdioAcpTransport`: επικοινωνεί με την υποδιεργασία ACP Hermes μέσω τυπικής εισόδου και εξόδου
- Μέθοδοι πρωτοκόλλου ACP: `initialize`, `authenticate`, `session/new`, `session/prompt`

**Επίπεδο χρόνου εκτέλεσης**
- `HermesGrain`: Εφαρμογή Orleans Grain που χειρίζεται την κατανεμημένη εκτέλεση συνεδρίας
- `CliAcpSessionPool`: ομάδα συνεδριών που επαναχρησιμοποιεί υποδιεργασίες ACP για την αποφυγή συχνών επιβαρύνσεων εκκίνησης

**Μπροστινό στρώμα**
- `ExecutorAvatar`: Ερμής οπτική ταυτότητα και εικόνα
- `executorTypeAdapter`: Λογική αντιστοίχισης τύπου παρόχου
- SignalR ανταλλαγή μηνυμάτων σε πραγματικό χρόνο: διατηρεί τη συνέπεια της ταυτότητας Hermes σε όλη τη ροή μηνυμάτων

Αυτός ο πολυεπίπεδος σχεδιασμός επιτρέπει σε κάθε στρώμα να εξελίσσεται ανεξάρτητα. Για παράδειγμα, εάν θέλουμε να προσθέσουμε έναν νέο μηχανισμό μεταφοράς στο μέλλον, όπως το WebSocket, χρειάζεται μόνο να τροποποιήσουμε το επίπεδο μεταφοράς. Δεν χρειάζεται να αναποδογυρίσετε ολόκληρο το σύστημα μόνο και μόνο επειδή αλλάζει μία μεταφορά.

### Ενοποιημένη αφαίρεση διεπαφής

Όλοι οι πάροχοι AI εφαρμόζουν το `IAIProvider` διεπαφή, η οποία είναι μία από τις βασικές επιλογές σχεδιασμού στην αρχιτεκτονική του HagiCode:

```csharp
public interface IAIProvider
{
    string Name { get; }
    ProviderCapabilities Capabilities { get; }

    IAsyncEnumerable<AIStreamingChunk> StreamAsync(
        AIRequest request,
        CancellationToken cancellationToken = default);

    Task<AIResponse> ExecuteAsync(
        AIRequest request,
        CancellationToken cancellationToken = default);
}
```

`HermesCliProvider` υλοποιεί αυτή τη διεπαφή και είναι ισότιμη με `ClaudeCodeProvider`, `OpenCodeProvider`, και άλλα. Τα οφέλη αυτού του σχεδιασμού περιλαμβάνουν:

1. **Δυνατότητα αντικατάστασης**: η εναλλαγή παρόχων δεν επηρεάζει την επιχειρηματική λογική του ανώτερου επιπέδου
2. **Δυνατότητα δοκιμής**: Οι πάροχοι μπορούν εύκολα να κοροϊδευτούν για τη δοκιμή μονάδας
3. **Επεκτασιμότητα**: η προσθήκη ενός νέου Παρόχου απαιτεί μόνο την υλοποίηση της διεπαφής

Τελικά, οι διεπαφές μοιάζουν πολύ με κανόνες. Μόλις τεθούν σε ισχύ οι κανόνες, όλοι μπορούν να συνυπάρξουν αρμονικά, να παίξουν με τις δυνάμεις τους και να αποφύγουν να πατήσουν ο ένας τον άλλον. Υπάρχει μια ορισμένη κομψότητα σε αυτό.

## Βασική Υλοποίηση

### Υλοποίηση επιπέδου παρόχου

`HermesCliProvider` είναι ο πυρήνας ολόκληρης της ολοκλήρωσης. Συντονίζει τα διάφορα στοιχεία που απαιτούνται για την ολοκλήρωση μιας μεμονωμένης επίκλησης AI:

```csharp
public sealed class HermesCliProvider : IAIProvider, IVersionedAIProvider
{
    private readonly ICliProvider<LibsHermesOptions> _provider;
    private readonly ConcurrentDictionary<string, string> _sessionBindings;

    public ProviderCapabilities Capabilities { get; } = new()
    {
        SupportsStreaming = true,
        SupportsTools = true,
        SupportsSystemMessages = true,
        SupportsArtifacts = false
    };

    public async IAsyncEnumerable<AIStreamingChunk> StreamAsync(
        AIRequest request,
        [EnumeratorCancellation] CancellationToken cancellationToken = default)
    {
        // 1. Resolve the session binding key
        var bindingKey = ResolveBindingKey(request.CessionId);

        // 2. Get or create a Hermes session through the session pool
        var options = new HermesOptions
        {
            ExecutablePath = _platformConfiguration.ExecutablePath,
            Arguments = _platformConfiguration.Arguments,
            SessionId = _sessionBindings.TryGetValue(bindingKey, out var sessionId) ? sessionId : null,
            WorkingDirectory = request.WorkingDirectory,
            Model = request.Model
        };

        // 3. Execute and collect the streaming response
        await foreach (var message in _provider.ExecuteAsync(options, request.Prompt, cancellationToken))
        {
            // 4. Map ACP messages to AIStreamingChunk
            if (_responseMapper.TryConvertToStreamingChunk(message, out var chunk))
            {
                yield return chunk;
            }
        }
    }
}
```

Πολλά σημεία σχεδιασμού είναι ιδιαίτερα σημαντικά εδώ:

1. **Σύνδεση συνεδρίας**: χρήσεις `CessionId` για σύνδεση πολλαπλών αιτημάτων στην ίδια υποδιεργασία Hermes, διατηρώντας τη συνέχεια του περιβάλλοντος σε συνομιλίες πολλαπλών στροφών
2. **Χαρτογράφηση απόκρισης**: μετατρέπει τη μορφή μηνύματος Hermes ACP σε ενοποιημένη `AIStreamingChunk` μορφή
3. **Υποστήριξη ροής**: χρήσεις `IAsyncEnumerable` για την υποστήριξη πραγματικών απαντήσεων ροής

Το δέσιμο συνεδρίας μοιάζει λίγο με τις ανθρώπινες σχέσεις. Μόλις δημιουργηθεί μια σύνδεση, η μελλοντική επικοινωνία έχει πλαίσιο, επομένως δεν χρειάζεται να ξεκινάτε από το μηδέν κάθε φορά. Φυσικά, αυτή η σχέση πρέπει ακόμη να διατηρηθεί.

### Προσαρμογή πρωτοκόλλου ACP

Η Hermes χρησιμοποιεί ACP (Πρωτόκολλο επικοινωνίας με πράκτορα), το οποίο διαφέρει από ένα παραδοσιακό API HTTP. Το ACP είναι ένα πρωτόκολλο που βασίζεται σε τυπική είσοδο και έξοδο και έχει πολλά χαρακτηριστικά:

1. **Δείκτης εκκίνησης**: μετά την έναρξη της διαδικασίας Hermes, βγάζει το `//ready` δείκτης
2. **Δυναμικός έλεγχος ταυτότητας**: οι μέθοδοι ελέγχου ταυτότητας δεν είναι σταθερές και πρέπει να διαπραγματεύονται μέσω του πρωτοκόλλου
3. **Επαναχρησιμοποίηση περιόδου λειτουργίας**: οι καθιερωμένες περίοδοι σύνδεσης επαναχρησιμοποιούνται μέσω `SessionId`
4. **Κατακερματισμένες απαντήσεις**: μια πλήρης απάντηση μπορεί να χωριστεί σε πολλαπλές `session/update` ειδοποιήσεις

Το HagiCode χειρίζεται αυτά τα χαρακτηριστικά μέσω `StdioAcpTransport`:

```csharp
public class StdioAcpTransport
{
    public async Task InitializeAsync(CancellationToken cancellationToken)
    {
        // Wait for the //ready marker
        var readyLine = await _outputReader.ReadLineAsync(cancellationToken);
        if (readyLine != "//ready")
        {
            throw new InvalidOperationException("Hermes did not send ready signal");
        }

        // Send the initialize request
        await SendRequestAsync(new
        {
            jsonrpc = "2.0",
            id = 1,
            method = "initialize",
            @params = new
            {
                protocolVersion = "2024-11-05",
                capabilities = new { },
                clientInfo = new { name = "HagiCode", version = "1.0.0" }
            }
        }, cancellationToken);
    }
}
```

Τα πρωτόκολλα μοιάζουν λίγο με την αμοιβαία κατανόηση μεταξύ των ανθρώπων. Μόλις υπάρξει αυτή η κατανόηση, η επικοινωνία ρέει πολύ πιο ομαλά. Η κατασκευή του απαιτεί χρόνο.

### Διαχείριση συνεδριακής πισίνας

Η συχνή εκκίνηση υποδιεργασιών Hermes είναι ακριβή, γι' αυτό εφαρμόσαμε έναν μηχανισμό συγκέντρωσης συνεδριών:

```csharp
services.AddSingleton(static _ =>
{
    var registry = new CliProviderPoolConfigurationRegistry();
    registry.Register("hermes", new CliPoolSettings
    {
        MaxActiveSessions = 50,
        IdleTimeout = TimeSpan.FromMinutes(10)
    });
    return registry;
});
```

Βασικές παράμετροι ομάδας συνεδριών:
- `MaxActiveSessions`: ελέγχει το όριο συγχρονισμού για να αποφύγει την εξάντληση πόρων
- `IdleTimeout`: Χρονικό όριο αδράνειας που εξισορροπεί το κόστος εκκίνησης με τη χρήση μνήμης

Στην πράξη διαπιστώσαμε ότι:
1. Εάν το χρονικό όριο αδράνειας είναι πολύ σύντομο, οι περίοδοι λειτουργίας επανεκκινούνται συχνά. εάν είναι πολύ μεγάλο, η μνήμη παραμένει κατειλημμένη
2. Το όριο συγχρονισμού πρέπει να ρυθμιστεί σύμφωνα με το πραγματικό φορτίο, επειδή η ρύθμιση του πολύ ψηλά μπορεί να κάνει το σύστημα υποτονικό
3. Η χρήση της ομάδας συνεδριών χρειάζεται παρακολούθηση, ώστε οι παράμετροι να μπορούν να προσαρμοστούν εγκαίρως

Αυτό μοιάζει πολύ με πολλές επιλογές στη ζωή: το να είσαι πολύ επιθετικός δημιουργεί προβλήματα, ενώ το να είσαι πολύ συντηρητικός χάνεις ευκαιρίες. Ο στόχος είναι απλά να βρεθεί η σωστή ισορροπία.

Το μπροστινό και το πίσω άκρο διατηρούν το συμβόλαιό τους ευθυγραμμισμένο μέσω της δημιουργίας OpenAPI. Το πίσω άκρο ορίζει το `AIProviderType` enum:

public enum AIProviderType

Ένα συμβόλαιο μοιάζει πολύ με υπόσχεση. Μόλις συμφωνηθεί, πρέπει να τιμηθεί, διαφορετικά καταλήγετε σε άβολες καταστάσεις όπως `Unknown`.

## Διαχείριση διαμόρφωσης

Η διαχείριση της διαμόρφωσης Hermes γίνεται μέσω `appsettings.json`:

```json
{
  "Providers": {
    "HermesCli": {
      "ExecutablePath": "hermes",
      "Arguments": "acp",
      "StartupTimeoutMs": 10000,
      "ClientName": "HagiCode",
      "Authentication": {
        "PreferredMethodId": "api-key",
        "MethodInfo": {
          "api-key": "your-api-key-here"
        }
      },
      "SessionDefaults": {
        "Model": "claude-sonnet-4-20250514",
        "ModeId": "default"
      }
    }
  }
}
```

Αυτός ο σχεδιασμός που βασίζεται στη διαμόρφωση φέρνει ευελιξία:
- Τα εκτελέσιμα μονοπάτια μπορούν να παρακαμφθούν, κάτι που είναι βολικό για ανάπτυξη και δοκιμή
- Τα ορίσματα εκκίνησης μπορούν να προσαρμοστούν ώστε να ταιριάζουν με διαφορετικές εκδόσεις Hermes
- Οι πληροφορίες ελέγχου ταυτότητας μπορούν να διαμορφωθούν ώστε να υποστηρίζουν πολλαπλές μεθόδους ελέγχου ταυτότητας

Η διαμόρφωση είναι λίγο σαν ερωτήσεις πολλαπλής επιλογής στη ζωή. Εάν υπάρχουν αρκετές διαθέσιμες επιλογές, συνήθως υπάρχει μία που ταιριάζει. Τούτου λεχθέντος, πάρα πολλές επιλογές μπορούν να δημιουργήσουν από μόνες τους κόπωση αποφάσεων.

## Πρακτική Εμπειρία

### Έλεγχοι υγείας

Η οικοδόμηση ενός αξιόπιστου παρόχου απαιτεί ολοκληρωμένους ελέγχους υγείας:

```csharp
public async Task<ProviderTestResult> PingAsync(CancellationToken cancellationToken = default)
{
    var response = await ExecuteAsync(new AIRequest
    {
        Prompt = "Reply with exactly PONG.",
        CessionId = null,
        AllowedTools = Array.Empty<string>(),
        WorkingDirectory = ResolveWorkingDirectory(null)
    }, cancellationToken);

    var success = string.Equals(response.Content.Trim(), "PONG", StringComparison.OrdinalIgnoreCase);
    return new ProviderTestResult
    {
        ProviderName = Name,
        Success = success,
        ResponseTimeMs = stopwatch.ElapsedMilliseconds,
        ErrorMessage = success ? null : $"Unexpected Hermes ping response: '{response.Content}'."
    };
}
```

Σημεία που πρέπει να προσέξετε στους υγειονομικούς ελέγχους:
1. Χρησιμοποιήστε απλές περιπτώσεις δοκιμής και αποφύγετε τα υπερβολικά πολύπλοκα σενάρια
2. Ορίστε εύλογες τιμές χρονικού ορίου
3. Καταγράψτε το χρόνο απόκρισης για να υποστηρίξετε την ανάλυση απόδοσης

Ακριβώς όπως οι άνθρωποι χρειάζονται φυσικούς ελέγχους, τα συστήματα χρειάζονται επίσης ελέγχους υγείας. Όσο πιο γρήγορα εντοπίζονται προβλήματα, τόσο πιο εύκολα επιλύονται.

### Εργαλείο επικύρωσης

Το HagiCode παρέχει μια αποκλειστική κονσόλα για την επικύρωση της ενοποίησης του Hermes:

```bash
# Basic validation
HagiCode.Libs.Hermes.Console --test-provider

# Full suite (including repository analysis)
HagiCode.Libs.Hermes.Console --test-provider-full --repo .

# Custom executable
HagiCode.Libs.Hermes.Console --test-provider-full --executable /path/to/hermes
```

Αυτό το εργαλείο είναι εξαιρετικά χρήσιμο κατά την ανάπτυξη, επειδή μας επιτρέπει να επαληθεύσουμε γρήγορα εάν η ενσωμάτωση είναι σωστή. Σε τελική ανάλυση, κανείς δεν θέλει να περιμένει μέχρι να εμφανιστεί ένα πρόβλημα προτού θυμηθεί να δοκιμάσει.

### Χειρισμός κοινών θεμάτων

**Αποτυχία ελέγχου ταυτότητας**
- Ελέγξτε αν `Authentication.PreferredMethodId` ταιριάζει με τη μέθοδο ελέγχου ταυτότητας που υποστηρίζει πραγματικά ο Hermes
- Επιβεβαιώστε ότι η μορφή πληροφοριών ελέγχου ταυτότητας είναι σωστή, όπως το κλειδί API ή το Bearer Token

**Λήξη χρόνου συνεδρίας**
- Αυξήστε το `StartupTimeoutMs` αξία
- Ελέγξτε την προσβασιμότητα του διακομιστή MCP
- Ελέγξτε τη χρήση πόρων του συστήματος

**Ημιτελής απάντηση**
- Εξασφαλίστε `session/update` οι ειδοποιήσεις και το τελικό αποτέλεσμα συγκεντρώνονται σωστά
- Ελέγξτε τη λογική ακύρωσης στο χειρισμό ροής
- Βεβαιωθείτε ότι ο χειρισμός σφαλμάτων έχει ολοκληρωθεί

**Μπροστινές οθόνες Άγνωστο**
- Επιβεβαιώστε ότι η δημιουργία OpenAPI περιλαμβάνει ήδη το `HermesCli` Αριθμός τιμής
- Ελέγξτε εάν η αντιστοίχιση τύπων είναι σωστή
- Εκκαθαρίστε την προσωρινή μνήμη του προγράμματος περιήγησης και δημιουργήστε ξανά τύπους

Τα προβλήματα πάντα θα υπάρχουν. Όταν εμφανίζονται, το σημαντικό είναι να μην πανικοβληθείτε. Ανιχνεύστε την αιτία βήμα προς βήμα και στο τέλος, τα περισσότερα από αυτά μπορούν να λυθούν.

### Προτάσεις βελτιστοποίησης απόδοσης

1. **Χρησιμοποιήστε την ομάδα συνεδριών**: επαναχρησιμοποιήστε υποδιεργασίες ACP για να μειώσετε τα έξοδα εκκίνησης
2. **Ρυθμίστε κατάλληλα τα χρονικά όρια**: εξισορροπήστε τη χρήση της μνήμης έναντι του κόστους εκκίνησης
3. **Επαναχρησιμοποίηση αναγνωριστικών περιόδου σύνδεσης**: χρησιμοποιήστε το ίδιο `CessionId` για ομαδικές εργασίες
4. **Διαμόρφωση MCP κατ' απαίτηση**: αποφύγετε τις περιττές κλήσεις εργαλείων

Η απόδοση μοιάζει πολύ με την αποτελεσματικότητα στην καθημερινή ζωή. Όταν το κάνεις σωστά, πετυχαίνεις περισσότερα με λιγότερα. όταν το κάνετε λάθος, η προσπάθεια πολλαπλασιάζεται ενώ τα αποτελέσματα συρρικνώνονται. Η εύρεση αυτού του «σωστού» σημείου απαιτεί εμπειρία και τύχη.

## Συμπέρασμα

Η ενσωμάτωση του Hermes Agent σε ένα σύστημα παραγωγής απαιτεί την εξέταση προβλημάτων σε πολλαπλές διαστάσεις:

1. **Αρχιτεκτονική**: σχεδιάστε μια ενοποιημένη διεπαφή παρόχου και εφαρμόστε μια αντικαταστάσιμη αρχιτεκτονική στοιχείων
2. **Πρωτόκολλο**: χειρίζεται σωστά τη συμπεριφορά του ACP, όπως δείκτες εκκίνησης και δυναμικό έλεγχο ταυτότητας
3. **Απόδοση**: επαναχρησιμοποιήστε πόρους μέσω της ομάδας συνεδριών και εξισορροπήστε το κόστος εκκίνησης με τη χρήση μνήμης
4. **Μπροστινό μέρος**: εξασφαλίστε συγχρονισμό συμβολαίων και παρέχετε μια συνεπή οπτική εμπειρία

Η εμπειρία της HagiCode δείχνει ότι με την καλή πολυεπίπεδη σχεδίαση και την υλοποίηση που βασίζεται στη διαμόρφωση, ένα σύνθετο σύστημα Agent μπορεί να ενσωματωθεί απρόσκοπτα σε μια υπάρχουσα αρχιτεκτονική.

Αυτές οι αρχές ακούγονται απλές όταν περιγράφονται με λόγια, αλλά η πραγματική εφαρμογή πάντα εισάγει πολλά διαφορετικά είδη προβλημάτων. Αυτό είναι καλό. Αν ένα πρόβλημα λυθεί, γίνεται εμπειρία. Αν δεν το κάνει, γίνεται μάθημα. Είτε έτσι είτε αλλιώς, εξακολουθεί να έχει αξία.

Τα όμορφα πράγματα ή οι άνθρωποι δεν χρειάζεται να κατέχονται. Όσο παραμένουν όμορφα, αρκεί να εκτιμάς αθόρυβα αυτή την ομορφιά. Η τεχνολογία είναι περίπου η ίδια. Εάν συμβάλλει στη βελτίωση του συστήματος, τότε το συγκεκριμένο πλαίσιο ή πρωτόκολλο έχει πολύ μικρότερη σημασία από ό,τι πιστεύουν οι άνθρωποι μερικές φορές.
