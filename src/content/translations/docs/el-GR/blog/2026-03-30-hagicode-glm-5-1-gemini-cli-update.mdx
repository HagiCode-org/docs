## Πλήρης υποστήριξη GLM-5.1 και ενσωμάτωση Gemini CLI: Η πορεία της HagiCode για την εξέλιξη πολλαπλών μοντέλων

Μαζί, αυτές οι ενημερώσεις ενισχύουν περαιτέρω τις δυνατότητες πολλαπλών μοντέλων της πλατφόρμας και το οικοσύστημα multi-CLI.

## φόντο

Ο χρόνος κινείται πραγματικά γρήγορα. Η ανάπτυξη μεγάλων γλωσσικών μοντέλων έχει αυξηθεί σαν μπαμπού την άνοιξη. Πριν από λίγο καιρό, εξακολουθούσαμε να ζητωκραυγάζαμε για «μια τεχνητή νοημοσύνη που μπορεί να γράψει κώδικα». Τώρα βρισκόμαστε ήδη σε μια εποχή συνεργασίας πολλαπλών μοντέλων και ενοποίησης πολλαπλών εργαλείων. Είναι αυτό συναρπαστικό; Ισως. Εξάλλου, αυτό που χρειάζονται οι προγραμματιστές δεν ήταν ποτέ μόνο το ίδιο το εργαλείο, αλλά η ευκολία προσαρμογής σε διαφορετικά σενάρια και η ευέλικτη εναλλαγή όταν χρειάζεται.

Ως πλατφόρμα κωδικοποίησης υποβοηθούμενη από AI, η HagiCode καλωσόρισε πρόσφατα δύο σημαντικές εξελίξεις: πρώτον, την πλήρη ενσωμάτωση του μοντέλου GLM-5.1 της Zhipu AI. δεύτερον, η επίσημη προσθήκη του Gemini CLI ως το δέκατο υποστηριζόμενο Agent CLI. Αυτές οι δύο ενημερώσεις μπορεί να μην ακούγονται συγκλονιστικές, αλλά είναι αναμφισβήτητα καλά νέα για τη συνεχή ωρίμανση της πλατφόρμας.

Το GLM-5.1 είναι το πιο πρόσφατο κορυφαίο μοντέλο της Zhipu AI. Σε σύγκριση με το GLM-5.0, προσφέρει ισχυρότερη συλλογιστική, βαθύτερη κατανόηση κώδικα και ομαλότερη κλήση εργαλείων. Το πιο σημαντικό, είναι το πρώτο μοντέλο GLM που υποστηρίζει την εισαγωγή εικόνας. Τι σημαίνει αυτό; Σημαίνει ότι οι χρήστες μπορούν να αφήσουν το AI να κοιτάξει απευθείας σε ένα στιγμιότυπο οθόνης αντί να δυσκολεύονται να περιγράψουν το πρόβλημα με λέξεις. Μόλις χρησιμοποιήσετε αυτήν την ευκολία, καταλαβαίνετε αμέσως την αξία της.

Ταυτόχρονα, μέσω του `HagiCode.Libs.Providers` αρχιτεκτονικής, η HagiCode ενσωμάτωσε με επιτυχία το Gemini CLI στην πλατφόρμα. Αυτό είναι τώρα το δέκατο Agent CLI. Για να είμαι ειλικρινής, το να φτάσουμε σε αυτό το σημείο φέρνει μια μέτρια αίσθηση ολοκλήρωσης.

Αξίζει επίσης να αναφέρουμε ότι η δυνατότητα μεταφόρτωσης εικόνων του HagiCode επιτρέπει στους χρήστες να επικοινωνούν με την τεχνητή νοημοσύνη απευθείας μέσω στιγμιότυπων οθόνης. Ακόμη και όταν εκτελείται το GLM 4.7, η πλατφόρμα εξακολουθεί να λειτουργεί καλά και έχει ήδη βοηθήσει στην ολοκλήρωση πολλών σημαντικών εργασιών κατασκευής. Όσο για το GLM-5.1, φυσικά, πάει ένα βήμα παραπέρα.

## Σχεδιασμός αρχιτεκτονικής Multi-CLI

Ένα από τα βασικά πλεονεκτήματα της HagiCode είναι η υποστήριξη πολλαπλών εργαλείων CLI προγραμματισμού AI μέσω ενός ενοποιημένου στρώματος αφαίρεσης. Το πλεονέκτημα αυτού του σχεδιασμού είναι στην πραγματικότητα πολύ απλό: νέα εργαλεία μπορούν να έρθουν, παλιά εργαλεία μπορούν να παραμείνουν και η βάση κώδικα δεν μετατρέπεται σε χάος. Για να είμαστε δίκαιοι, έτσι θα ήθελε ο καθένας να λειτουργεί η ζωή.

### AIPproviderType enum

Η πλατφόρμα ορίζει υποστηριζόμενους τύπους παρόχων CLI μέσω του `AIProviderType` enum:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI (new)
}
```

Όπως μπορείτε να δείτε, το Gemini CLI εντάσσεται σε αυτήν την οικογένεια ως το δέκατο μέλος. Κάθε CLI έχει τα δικά του ξεχωριστά χαρακτηριστικά και σενάρια χρήσης, ώστε οι χρήστες να μπορούν να επιλέξουν με ευελιξία με βάση τις ανάγκες τους. Τελικά, πολλοί δρόμοι οδηγούν στη Ρώμη. μερικά είναι απλά πιο εύκολα από άλλα.

### Αρχιτεκτονική παρόχου

`HagiCode.Libs.Providers` παρέχει μια ενοποιημένη διεπαφή παρόχου που καθιστά κάθε ενσωμάτωση CLI τυποποιημένη και συνοπτική. Λαμβάνοντας ως παράδειγμα το Gemini CLI:

```csharp
public class GeminiProvider : ICliProvider<GeminiOptions>
{
    private static readonly string[] DefaultExecutableCandidates = ["gemini", "gemini-cli"];
    private const string ManagedBootstrapArgument = "--acp";

    public string Name => "gemini";
    public bool IsAvailable => _executableResolver.ResolveFirstAvailablePath(DefaultExecutableCandidates) is not null;
}
```

Τα οφέλη αυτού του σχεδιασμού είναι:
- Η ενσωμάτωση ενός νέου CLI απαιτεί μόνο την εφαρμογή μιας κλάσης Provider
- Ενοποιημένη διαχείριση κύκλου ζωής και συγκέντρωση συνεδριών
- Αυτοματοποιημένη ανάλυση ψευδωνύμου και εκτελέσιμη ανακάλυψη

Με απλά λόγια, αυτό το σχέδιο μετατρέπει τα περίπλοκα πράγματα σε πιο απλά και κάνει τη ζωή λίγο πιο εύκολη.

### Μητρώο παρόχων

Το Μητρώο Παρόχων χειρίζεται αυτόματα την αντιστοίχιση και την εγγραφή ψευδωνύμων:

```csharp
if (provider is GeminiProvider)
{
    registry.Register(provider.Name, provider, ["gemini-cli"]);
    continue;
}
```

Αυτό σημαίνει ότι οι χρήστες μπορούν να επικαλεστούν το Gemini CLI με οποιοδήποτε από τα δύο `gemini` ή `gemini-cli`, και το σύστημα θα το αναγνωρίσει αυτόματα. Είναι σαν ένας φίλος με επίσημο όνομα και ψευδώνυμο - σε κάθε περίπτωση, οι άνθρωποι ξέρουν ποιον εννοείς.

## Υποστήριξη μοντέλου GLM-5.1

Το GLM-5.1 είναι το πιο πρόσφατο κορυφαίο μοντέλο της Zhipu AI και η HagiCode έχει ολοκληρώσει την πλήρη υποστήριξή του.

### Κατάλογος Δευτεροβάθμιων Επαγγελμάτων

Η HagiCode διαχειρίζεται όλα τα υποστηριζόμενα μοντέλα μέσω του Καταλόγου Δευτεροβάθμιων Επαγγελμάτων. Ακολουθεί η διαμόρφωση για τη σειρά GLM:

| Αναγνωριστικό μοντέλου | Όνομα | SupportsImage | Συμβατές οικογένειες CLI |
|----------|------|----------------|-------------------------|
| `glm-4.7` | GLM 4.7 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM 5 Turbo | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (παλαιού τύπου) | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | **αληθεια** | claude, codebuddy, hermes, qoder, kiro |

Τα βασικά χαρακτηριστικά του GLM-5.1 μπορούν να συνοψιστούν ως εξής:
- Ένα αυτόνομο αναγνωριστικό έκδοσης χωρίς αποσκευές παλαιού τύπου
- Το πρώτο μοντέλο GLM που υποστηρίζει την εισαγωγή εικόνας
- Ισχυρότερος συλλογισμός και κατανόηση κώδικα
- Ευρεία συμβατότητα πολλαπλών CLI

### GLM-5.1 vs GLM-5.0

Σε επίπεδο κώδικα, η βασική διαφορά μεταξύ GLM-5.1 και GLM-5.0 εμφανίζεται εδώ:

```csharp
// GLM-5.0 (Legacy) - contains special retention logic
private const string Glm50CodebuddySecondaryProfessionId = "secondary-glm-5-codebuddy";
private const string Glm50CodebuddyModelValue = "glm-5.0";

// GLM-5.1 - standalone new model identifier
private const string Glm51SecondaryProfessionId = "secondary-glm-5-1";
private const string Glm51ModelValue = "glm-5.1";
```

Το GLM-5.0 φέρει την ετικέτα "Legacy" επειδή είναι ένα παλιό αναγνωριστικό έκδοσης που διατηρείται για συμβατότητα προς τα πίσω. Το GLM-5.1, αντίθετα, είναι μια ολοκαίνουργια αυτόνομη έκδοση χωρίς ιστορικό βάρος. Μερικά πράγματα μένουν στο παρελθόν. άλλοι ταξιδεύουν ελαφρύτερα και κινούνται πιο γρήγορα.

### Διαμόρφωση GLM-5.1

Ακολουθεί ένα παράδειγμα διαμόρφωσης για τη χρήση του GLM-5.1 στο HagiCode:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

## Λειτουργία μεταφόρτωσης εικόνων

Η υποστήριξη εικόνας της HagiCode υλοποιείται μέσω του `SupportsImage` ιδιοκτησία σε `SecondaryProfession`:

```csharp
public class HeroSecondaryProfessionSettingDto
{
    public bool SupportsImage { get; set; }
}
```

Στον Κατάλογο Δευτεροβάθμιων Επαγγελμάτων, η διαμόρφωση του GLM-5.1 μοιάζει με αυτό:

```json
{
  "id": "secondary-glm-5-1",
  "supportsImage": true
}
```

Αυτό σημαίνει ότι οι χρήστες μπορούν να ανεβάζουν στιγμιότυπα οθόνης απευθείας για ανάλυση AI, όπως:
- Στιγμιότυπα οθόνης μηνυμάτων σφάλματος
- Προβλήματα σε μια οθόνη διεπαφής χρήστη
- Διαγράμματα οπτικοποίησης δεδομένων
- Αποτελέσματα εκτέλεσης κώδικα

Δεν χρειάζεται πλέον να περιγράφετε τα πάντα χειροκίνητα - απλώς μεταφορτώστε το στιγμιότυπο οθόνης. Η ευκολία αυτής της δυνατότητας είναι προφανής μόλις τη χρησιμοποιήσετε. Μερικές φορές μια ματιά λέει περισσότερα από μια μεγάλη εξήγηση.

## Ενσωμάτωση Gemini CLI

Ως το δέκατο Agent CLI, το Gemini CLI είναι ενσωματωμένο στο HagiCode μέσω της τυπικής αρχιτεκτονικής Provider.

### Επιλογές διαμόρφωσης

Το Gemini CLI υποστηρίζει ένα πλούσιο σύνολο επιλογών διαμόρφωσης:

```csharp
public class GeminiOptions
{
    public string? ExecutablePath { get; set; }
    public string? WorkingDirectory { get; set; }
    public string? SessionId { get; set; }
    public string? Model { get; set; }
    public string? AuthenticationMethod { get; set; }
    public string? AuthenticationToken { get; set; }
    public Dictionary<string, string?> AuthenticationInfo { get; set; }
    public Dictionary<string, string?> EnvironmentVariables { get; set; }
    public string[] ExtraArguments { get; set; }
    public TimeSpan? StartupTimeout { get; set; }
    public CliPoolSettings? PoolSettings { get; set; }
}
```

Αυτές οι επιλογές καλύπτουν τα πάντα, από τη βασική ρύθμιση έως τις προηγμένες λειτουργίες, δίνοντας στους χρήστες την ευελιξία να διαμορφώσουν το CLI σύμφωνα με τις δικές τους ανάγκες. Η ροή εργασίας του καθενός είναι διαφορετική, επομένως λίγη ευελιξία είναι πάντα ευπρόσδεκτη.

### Πρωτόκολλο επικοινωνίας ACP

Το Gemini CLI υποστηρίζει το ACP (Agent Communication Protocol), το οποίο είναι το ενοποιημένο πρότυπο επικοινωνίας CLI της HagiCode. Μέσω του ACP, διαφορετικά CLI μπορούν να αλληλεπιδρούν με την πλατφόρμα με συνεπή τρόπο, απλοποιώντας σημαντικά τις εργασίες ολοκλήρωσης. Εν ολίγοις, τυποποιεί τα περίπλοκα μέρη, ώστε ο καθένας να μπορεί να εργάζεται πιο εύκολα.

## Η πρακτική κατασκευής της ίδιας της HagiCode

Μιλώντας για πρακτική πρακτική, το καλύτερο παράδειγμα είναι η ροή εργασιών κατασκευής της πλατφόρμας HagiCode. Η διαδικασία ανάπτυξης του HagiCode έχει ήδη κάνει πλήρη χρήση των δυνατοτήτων AI.

### Λειτουργεί καλά ακόμη και με το GLM 4.7

Ο σχεδιασμός της πλατφόρμας του HagiCode είναι καλά βελτιστοποιημένος, επομένως μπορεί να παρέχει μια καλή εμπειρία ανάπτυξης ακόμη και με το GLM 4.7. Η πλατφόρμα έχει ήδη βοηθήσει στην ολοκλήρωση πολλών σημαντικών έργων κατασκευής, όπως:
- Ενσωμάτωση πολλαπλών παρόχων CLI
- Εφαρμογή της δυνατότητας αποστολής εικόνων
- Δημιουργία τεκμηρίωσης και δημοσίευση περιεχομένου

Αυτό είναι πράγματι καλό. Δεν χρειάζονται όλοι το νεότερο πράγμα συνέχεια. Αυτό που σας ταιριάζει περισσότερο είναι συχνά αυτό που έχει μεγαλύτερη σημασία.

### Το GLM-5.1 προσφέρει περισσότερα με λιγότερη προσπάθεια

Μετά την αναβάθμιση σε GLM-5.1, αυτές οι δυνατότητες γίνονται ακόμα πιο δυνατές:
- Ισχυρότερη κατανόηση κώδικα, μειώνοντας την επικοινωνία εμπρός και πίσω
- Πιο ακριβής ανάλυση εξάρτησης, δείχνοντας αμέσως τη σωστή κατεύθυνση
- Πιο αποτελεσματική διάγνωση σφαλμάτων, γρηγορότερος εντοπισμός προβλημάτων
- Υποστήριξη για εισαγωγή εικόνων, επιτάχυνση των περιγραφών προβλημάτων

Είναι σαν να αλλάζεις από ποδήλατο σε αυτοκίνητο. Μπορείτε ακόμα να φτάσετε στον ίδιο προορισμό, αλλά η ταχύτητα και η άνεση δεν είναι τα ίδια.

## Βέλτιστες πρακτικές για ενσωμάτωση πολλαπλών CLI

`HagiCode.Libs.Providers` παρέχει έναν ενοποιημένο μηχανισμό εγγραφής και χρήσης:

```csharp
services.AddHagiCodeLibs();

var gemini = serviceProvider.GetRequiredService<ICliProvider<GeminiOptions>>();
var codebuddy = serviceProvider.GetRequiredService<ICliProvider<CodebuddyOptions>>();
var hermes = serviceProvider.GetRequiredService<ICliProvider<HermesOptions>>();
```

Αυτός ο σχεδιασμός έγχυσης εξάρτησης διατηρεί τη χρήση σε διαφορετικά CLI πολύ συνοπτικά και επίσης κάνει πιο βολικές τις δοκιμές και την κοροϊδία της μονάδας. Ο καθαρός κώδικας είναι ένας τρόπος να είσαι υπεύθυνος απέναντι στον εαυτό σου.

### Σημειώσεις

Υπάρχουν μερικά πράγματα που πρέπει να έχετε υπόψη κατά την πραγματική χρήση:

1. **Διαμόρφωση κλειδιού API**: Βεβαιωθείτε `ANTHROPIC_AUTH_TOKEN` έχει ρυθμιστεί σωστά ή δεν είναι δυνατή η κλήση του μοντέλου
2. **Διαθεσιμότητα μοντέλου**: Το GLM-5.1 πρέπει να ενεργοποιηθεί από τον αντίστοιχο πάροχο μοντέλου
3. **Δυνατότητα εικόνας**: Μόνο μοντέλα με `supportsImage: true` μπορεί να χρησιμοποιήσει τη μεταφόρτωση εικόνων
4. **Εγκατάσταση CLI**: Πριν χρησιμοποιήσετε το Gemini CLI, βεβαιωθείτε `gemini` ή `gemini-cli` βρίσκεται στο σύστημα PATH

Αυτές μπορεί να είναι μικρές λεπτομέρειες, αλλά οι μικρές λεπτομέρειες που δεν αντιμετωπίζονται σωστά μπορεί να μετατραπούν σε μεγάλα προβλήματα, επομένως αξίζει να τους προσέχετε.

## Συμπέρασμα

Με την πλήρη υποστήριξη για το GLM-5.1 και την επιτυχημένη ενσωμάτωση του Gemini CLI, το HagiCode ενισχύει περαιτέρω τις δυνατότητές του ως πλατφόρμα προγραμματισμού AI πολλαπλών μοντέλων, πολλαπλών CLI. Αυτές οι ενημερώσεις όχι μόνο δίνουν στους χρήστες περισσότερες επιλογές, αλλά καταδεικνύουν επίσης την αρχιτεκτονική και την επεκτασιμότητα του HagiCode.

Η υποστήριξη εικόνας του GLM-5.1, σε συνδυασμό με τη δυνατότητα μεταφόρτωσης στιγμιότυπου οθόνης του HagiCode, επιτρέπει στο AI να «καταλαβαίνει από την εικόνα» και μειώνει σημαντικά το κόστος της περιγραφής προβλημάτων. Και με την υποστήριξη δέκα CLI, οι χρήστες μπορούν να επιλέξουν με ευελιξία τον βοηθό προγραμματισμού AI που ταιριάζει καλύτερα στις προτιμήσεις και τα σενάρια τους. Περισσότερες επιλογές είναι σχεδόν πάντα καλό.

Το πιο σημαντικό, η πρακτική κατασκευής της ίδιας της HagiCode αποδεικνύει ότι η πλατφόρμα μπορεί ήδη να εκτελεί καλά και να ολοκληρώσει πολύπλοκες εργασίες ακόμη και με το GLM 4.7, ενώ η αναβάθμιση σε GLM-5.1 μπορεί να βελτιώσει περαιτέρω την αποδοτικότητα ανάπτυξης. Η ζωή είναι συχνά έτσι: δεν χρειάζεσαι πάντα το απόλυτο καλύτερο, μόνο αυτό που σου ταιριάζει. Φυσικά, αν αυτό που σου ταιριάζει μπορεί να γίνει ακόμα καλύτερο, τότε τόσο το καλύτερο.
