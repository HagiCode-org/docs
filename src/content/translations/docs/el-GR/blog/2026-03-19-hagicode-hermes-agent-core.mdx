## Γιατί η HagiCode επέλεξε τον Hermes ως τον ολοκληρωμένο πυρήνα αντιπροσώπου της

> Κατά την κατασκευή μιας πλατφόρμας κωδικοποίησης με τη βοήθεια AI, η επιλογή του σωστού πυρήνα Agent καθορίζει άμεσα το ανώτερο όριο των δυνατοτήτων του συστήματος. Μερικά πράγματα απλά δεν μπορούν να επιβληθούν. επιλέξτε το λάθος πλαίσιο και καμία προσπάθεια δεν θα το κάνει να νιώσει σωστό. Αυτό το άρθρο μοιράζεται τον τρόπο σκέψης πίσω από την τεχνική επιλογή της HagiCode και την πρακτική εμπειρία μας στην ενσωμάτωση του Hermes Agent.

## φόντο

Κατά την κατασκευή ενός προϊόντος κωδικοποίησης με τη βοήθεια AI, ένα από τα πιο δύσκολα μέρη είναι η επιλογή του υποκείμενου πλαισίου Agent. Υπάρχουν στην πραγματικότητα αρκετές επιλογές στην αγορά, αλλά ορισμένες είναι πολύ περιορισμένες σε λειτουργικότητα, μερικές είναι υπερβολικά πολύπλοκες στην ανάπτυξη και άλλες απλώς δεν κλιμακώνονται αρκετά καλά. Αυτό που χρειαζόμασταν ήταν μια λύση που θα μπορούσε να λειτουργεί με VPS $5 ενώ παράλληλα θα μπορούσαμε να συνδεθούμε σε ένα σύμπλεγμα GPU. Αυτή η απαίτηση μπορεί να μην ακούγεται ακραία, αλλά είναι αρκετή για να τρομάξει πολλές ομάδες μακριά.

Στην πράξη, πολλοί λεγόμενοι «όλα σε ένα πράκτορες» είτε εκτελούνται μόνο στο cloud είτε απαιτούν παράλογα υψηλό κόστος τοπικής ανάπτυξης. Αφού περάσαμε δύο εβδομάδες ερευνώντας διαφορετικές προσεγγίσεις, λάβαμε μια τολμηρή απόφαση: να ξαναχτίσουμε ολόκληρο τον πυρήνα του Agent γύρω από τον Hermes ως τον υποκείμενο κινητήρα για τον ολοκληρωμένο Πράκτορά μας.

Όλα όσα ακολούθησαν μπορεί απλώς να ήταν η μοίρα.

## Γιατί το HagiCode χρειάζεται τον Ερμή

Πριν βουτήξουμε στον ίδιο τον Ερμή, βοηθάει να εξηγήσουμε γιατί η HagiCode χρειαζόταν κάτι παρόμοιο εξαρχής. Τα πράγματα σπάνια λειτουργούν ακριβώς όπως θέλετε, επομένως χρειάζεστε έναν πρακτικό λόγο για να δεσμευτείτε σε μια τεχνική κατεύθυνση.

Ως βοηθός κωδικοποίησης AI, το HagiCode πρέπει να υποστηρίζει πολλά σενάρια χρήσης ταυτόχρονα:
- Τοπικά περιβάλλοντα ανάπτυξης: οι προγραμματιστές θέλουν να το εκτελέσουν στα δικά τους μηχανήματα, ώστε τα δεδομένα να μην φεύγουν ποτέ από το τοπικό περιβάλλον. Αυτές τις μέρες, η ασφάλεια των δεδομένων δεν είναι ποτέ μια ασήμαντη ανησυχία.
- Περιβάλλοντα συνεργασίας ομάδας: οι μικρές ομάδες θα πρέπει να μπορούν να μοιράζονται μια ανάπτυξη Agent που εκτελείται σε έναν διακομιστή. Η εξοικονόμηση χρημάτων έχει σημασία και όλοι έχουν όρια.
- Επέκταση ελαστικού νέφους: κατά τον χειρισμό σύνθετων εργασιών, το σύστημα θα πρέπει αυτόματα να κλιμακώνεται σε ένα σύμπλεγμα GPU. Είναι πάντα καλύτερο να είσαι προετοιμασμένος.

Αυτή η απαίτηση «τα θέλουμε όλα μονομιάς» είναι που μας οδήγησε στον Ερμή. Το αν ήταν η τέλεια επιλογή, δεν μπορώ να πω με βεβαιότητα, αλλά εκείνη την εποχή δεν είδαμε καλύτερη επιλογή.

## Τι είναι ο πράκτορας της Ερμής

Το Hermes Agent είναι ένας αυτόνομος πράκτορας AI που δημιουργήθηκε από την **Nous Research**. Ορισμένοι αναγνώστες μπορεί να μην είναι εξοικειωμένοι με το Nous Research. Είναι το εργαστήριο πίσω από μεγάλα μοντέλα ανοιχτού κώδικα όπως το Hermes, το Nomos και το Psyché. Έχουν χτίσει πολλά εξαιρετικά πράγματα, έστω κι αν εξακολουθούν να είναι πιο υποτιμημένα από όσο τους αξίζει.

Σε αντίθεση με τους παραδοσιακούς βοηθούς κωδικοποίησης IDE ή τα απλά περιτυλίγματα συνομιλίας API, ο Hermes έχει ένα καθοριστικό χαρακτηριστικό: όσο περισσότερο εκτελείται, τόσο πιο ικανός γίνεται. Δεν έχει σχεδιαστεί για να ολοκληρώσει μια εργασία μία φορά και να σταματήσει. συνεχίζει να μαθαίνει και να συσσωρεύει εμπειρία κατά τη μακροχρόνια λειτουργία. Υπό αυτή την έννοια, αισθάνεται λίγο σαν άνθρωπος.

### Βασικά Χαρακτηριστικά

Πολλές από τις βασικές δυνατότητες του Hermes τυχαίνει να ευθυγραμμίζονται πολύ στενά με τις ανάγκες του HagiCode.

Αυτό σημαίνει ότι το HagiCode μπορεί να επιλέξει το πιο κατάλληλο μοντέλο ανάπτυξης με βάση το σενάριο κάθε χρήστη: άτομα το εκτελούν τοπικά, οι ομάδες το αναπτύσσουν σε διακομιστές και πολύπλοκες εργασίες χρησιμοποιούν πόρους GPU. Μια βάση κώδικα τα χειρίζεται όλα. Σε έναν κόσμο τόσο απασχολημένο, η εξοικονόμηση ενός επιπέδου πολυπλοκότητας είναι ήδη μια νίκη.

**Πύλη ανταλλαγής μηνυμάτων πολλαπλών πλατφορμών**
Ο Hermes υποστηρίζει εγγενώς Telegram, Discord, Slack, WhatsApp και πολλά άλλα. Για το HagiCode, αυτό σημαίνει ότι μπορούμε να υποστηρίξουμε τους βοηθούς AI σε αυτά τα κανάλια πολύ πιο εύκολα στο μέλλον. Περισσότερα μονοπάτια προς τα εμπρός είναι πάντα ευπρόσδεκτα.

**Πλούσιο σύστημα εργαλείων**
Το Hermes έρχεται με 40+ ενσωματωμένα εργαλεία και υποστηρίζει επεκτάσεις MCP (Model Context Protocol). Αυτό είναι απαραίτητο για έναν βοηθό κωδικοποίησης: η εκτέλεση εντολών φλοιού, η εργασία με το σύστημα αρχείων και η κλήση του Git εξαρτώνται από την υποστήριξη του εργαλείου. Ένας πράκτορας χωρίς εργαλεία είναι σαν ένα πουλί χωρίς φτερά.

**Μνήμη μεταξύ συνεδριών**
Ο Hermes περιλαμβάνει ένα σύστημα επίμονης μνήμης και χρησιμοποιεί αναζήτηση πλήρους κειμένου FTS5 για να ανακαλέσει ιστορικές συνομιλίες. Αυτό επιτρέπει στον Πράκτορα να θυμάται το προηγούμενο πλαίσιο αντί να "χάνει τη μνήμη του" κάθε φορά. Μερικές φορές οι άνθρωποι εύχονται να μπορούσαν να ξεχάσουν πράγματα τόσο εύκολα, αλλά η πραγματικότητα είναι συνήθως λιγότερο γενναιόδωρη.

## Πώς το HagiCode ενσωματώνει τον Ερμή

Τώρα που το «γιατί» είναι ξεκάθαρο, ας δούμε το «πώς». Μόλις κάτι βγάζει νόημα στη θεωρία, το επόμενο βήμα είναι να το χτίσεις.

### Αφαίρεση επιπέδου παρόχου

Στην αρχιτεκτονική του HagiCode, όλοι οι πάροχοι AI εφαρμόζουν ένα ενοποιημένο `IAIProvider` διεπαφή:

```csharp
public sealed class HermesCliProvider : IAIProvider, IVersionedAIProvider
{
    public ProviderCapabilities Capabilities { get; } = new ProviderCapabilities
    {
        SupportsStreaming = true,   // Supports streaming output
        SupportsTools = true,       // Supports tool invocation
        SupportsSystemMessages = true, // Supports system prompts
        SupportsArtifacts = false
    };
}
```

Αυτό το επίπεδο αφαίρεσης επιτρέπει στο HagiCode να εναλλάσσεται απρόσκοπτα μεταξύ διαφορετικών παρόχων AI. Είτε το backend είναι OpenAI, Claude ή Hermes, το μοτίβο κλήσης του ανώτερου επιπέδου παραμένει ακριβώς το ίδιο. Με απλά λόγια, κρατά τα πράγματα απλά.

### Πρωτόκολλο επικοινωνίας ΑΚΕ

Ο Hermes επικοινωνεί μέσω **ACP (Agent Communication Protocol)**. Αυτό το πρωτόκολλο έχει σχεδιαστεί ειδικά για την επικοινωνία Agent και οι κύριες μέθοδοί του περιλαμβάνουν:

| Μέθοδος | Περιγραφή |
|------|------|
| `initialize` | Αρχικοποιήστε τη σύνδεση και αποκτήστε την έκδοση πρωτοκόλλου και τις δυνατότητες πελάτη |
| `authenticate` | Χειριστείτε τον έλεγχο ταυτότητας και υποστηρίζετε πολλαπλές μεθόδους ελέγχου ταυτότητας |
| `session/new` | Δημιουργήστε μια νέα περίοδο λειτουργίας και διαμορφώστε τον κατάλογο εργασίας και τους διακομιστές MCP |
| `session/prompt` | Στείλτε ένα μήνυμα προτροπής και λάβετε απάντηση |

Το HagiCode υλοποιεί το επίπεδο μεταφοράς ACP μέσω `StdioAcpTransport`, εκκινώντας μια υποδιεργασία Hermes και επικοινωνώντας μαζί της μέσω τυπικής εισόδου και εξόδου. Μπορεί να ακούγεται περίπλοκο, αλλά στην πράξη είναι διαχειρίσιμο αρκεί να έχετε αρκετή υπομονή.

### Διαχείριση διαμόρφωσης

Η διαχείριση της διαμόρφωσης γίνεται μέσω του `HermesPlatformConfiguration` τάξη:

```csharp
public sealed class HermesPlatformConfiguration : IAcpPlatformConfiguration
{
    public string ExecutablePath { get; set; } = "hermes";
    public string Arguments { get; set; } = "acp";
    public int StartupTimeoutMs { get; set; } = 5000;
    public string ClientName { get; set; } = "HagiCode";
    public HermesAuthenticationConfiguration Authentication { get; set; }
    public HermesSessionDefaultsConfiguration SessionDefaults { get; set; }
}
```

Ρύθμιση παραμέτρων Hermes in `appsettings.json`:

```json
{
  "Providers": {
    "HermesCli": {
      "ExecutablePath": "hermes",
      "Arguments": "acp",
      "StartupTimeoutMs": 10000,
      "ClientName": "HagiCode",
      "Authentication": {
        "PreferredMethodId": "api-key",
        "MethodInfo": {
          "api-key": "your-api-key-here"
        }
      },
      "SessionDefaults": {
        "Model": "claude-sonnet-4-20250514",
        "ModeId": "default"
      }
    }
  }
}
```

Η διαμόρφωση συχνά φαίνεται απλή στο χαρτί, αλλά η σωστή κάθε λεπτομέρεια απαιτεί πραγματική προσπάθεια.

### Orleans Distributed Architecture

Το HagiCode χρησιμοποιεί την Ορλεάνη για να δημιουργήσει το κατανεμημένο του σύστημα και η ενοποίηση του Hermes υλοποιείται μέσω των ακόλουθων στοιχείων:

- **HermesGrain**: Μια υλοποίηση Orleans Grain που χειρίζεται την εκτέλεση συνεδρίας
- **HermesPlatformConfiguration**: Διαμόρφωση συγκεκριμένης πλατφόρμας
- **HermesAcpSessionAdapter**: Προσαρμογέας περιόδου λειτουργίας ACP
- **HermesConsole**: Μια αποκλειστική κονσόλα επικύρωσης

Το όνομα Ορλεάνη έχει μια ορισμένη γοητεία. Ακόμα κι αν αυτή η Ορλεάνη δεν έχει καμία σχέση με τη θρυλική πόλη, ένα καλό όνομα δεν βλάπτει ποτέ.

### Ροή εκτέλεσης από άκρο σε άκρο

Ακολουθεί η βασική λογική εκτέλεσης του παρόχου Hermes:

```csharp
private async IAsyncEnumerable<AIStreamingChunk> StreamCoreAsync(
    AIRequest request,
    string? embeddedCommandPrompt,
    [EnumeratorCancellation] CancellationToken cancellationToken)
{
    // 1. Create transport layer and launch Hermes subprocess
    await using var transport = new StdioAcpTransport(
        platformConfiguration.GetExecutablePath(),
        platformConfiguration.GetArguments(),
        platformConfiguration.GetEnvironmentVariables(),
        platformConfiguration.GetStartupTimeout(),
        _loggerFactory.CreateLogger<StdioAcpTransport>());
    await transport.ConnectAsync(cancellationToken);

    // 2. Initialize and obtain protocol version and authentication methods
    var initializeResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "initialize",
        BuildInitializeParameters(platformConfiguration), cancellationToken);

    // 3. Handle authentication
    var authMethods = ParseAuthMethods(initializeResult);
    if (!isAuthenticated)
    {
        var methodId = platformConfiguration.Authentication.ResolveMethodId(authMethods);
        await SendHermesRequestAsync(transport, nextRequestId++, "authenticate", ...);
    }

    // 4. Create session
    var newSessionResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "session/new",
        BuildNewSessionParameters(platformConfiguration, workingDirectory, model), cancellationToken);
    var sessionId = ParseSessionId(newSessionResult);

    // 5. Execute prompt and collect streaming responses
    await foreach (var payload in transport.ReceiveMessagesAsync(cancellationToken))
    {
        // Handle session/update notifications and convert them into streaming chunks
        if (TryParseSessionNotification(root, out var notification))
        {
            if (_responseMapper.TryConvertToStreamingChunk(notification, out var chunk))
            {
                yield return chunk;
            }
        }
    }
}
```

Με τον κώδικα, οι λεπτομέρειες γίνονται τελικά εξοικειωμένες. Αυτό που έχει μεγαλύτερη σημασία είναι η συνολική προσέγγιση.

### Έλεγχοι υγείας

Για να διασφαλίσει ότι ο Hermes παραμένει διαθέσιμος, η HagiCode εφαρμόζει έναν μηχανισμό ελέγχου υγείας:

```csharp
public async Task<ProviderTestResult> PingAsync(CancellationToken cancellationToken = default)
{
    var response = await ExecuteAsync(
        new AIRequest
        {
            Prompt = "Reply with exactly PONG.",
            CessionId = null,
            AllowedTools = Array.Empty<string>(),
            WorkingDirectory = ResolveWorkingDirectory(null)
        },
        cancellationToken);

    var success = string.Equals(response.Content.Trim(), "PONG", StringComparison.OrdinalIgnoreCase);
    return new ProviderTestResult
    {
        ProviderName = Name,
        Success = success,
        ResponseTimeMs = stopwatch.ElapsedMilliseconds,
        ErrorMessage = success ? null : $"Unexpected Hermes ping response: '{response.Content}'."
    };
}
```

Αυτό είναι περίπου το πώς φαίνεται εδώ ένας "έλεγχος υγείας". Κατά κάποιο τρόπο, οι άνθρωποι δεν είναι τόσο διαφορετικοί: βοηθάει να κάνουμε check in από καιρό σε καιρό, ακόμα κι αν κανείς δεν μας λέει ακριβώς τι πρέπει να αναζητήσουμε.

## Πρακτικές Θεωρήσεις

Υπάρχουν μερικές παγίδες που αξίζει να κατανοήσετε πριν ενσωματώσετε τον Ερμή. Όλοι αργά ή γρήγορα μπαίνουν σε μερικές παγίδες.

### Διαμόρφωση μεθόδου ελέγχου ταυτότητας

Το Hermes υποστηρίζει πολλαπλές μεθόδους ελέγχου ταυτότητας, συμπεριλαμβανομένων κλειδιών API και διακριτικών, επομένως πρέπει να επιλέξετε με βάση το πραγματικό σενάριο ανάπτυξης. Η εσφαλμένη διαμόρφωση μπορεί να προκαλέσει αποτυχίες σύνδεσης και τα μηνύματα σφάλματος που προκύπτουν δεν είναι πάντα εύχρηστα. Μερικές φορές το αναφερόμενο σφάλμα απέχει πολύ από την πραγματική βασική αιτία, πράγμα που σημαίνει ότι ο αργός και προσεκτικός εντοπισμός σφαλμάτων είναι αναπόφευκτος.

### Διαμόρφωση διακομιστή MCP

Όταν δημιουργείτε μια περίοδο λειτουργίας, μπορείτε να διαμορφώσετε μια λίστα διακομιστών MCP, ώστε η Hermes να μπορεί να καλεί εξωτερικά εργαλεία. Έχετε όμως υπόψη σας τα ακόλουθα σημεία:
- Οι διευθύνσεις διακομιστή MCP πρέπει να είναι προσβάσιμες
- Τα χρονικά όρια πρέπει να διαμορφωθούν εύλογα
- Το σύστημα χρειάζεται χειρισμό υποβάθμισης όταν ένας διακομιστής δεν είναι διαθέσιμος

Στην πράξη, η αμυντική σκέψη έχει μεγαλύτερη σημασία από ό,τι περιμένουν οι άνθρωποι.

### Διαχείριση καταλόγου εργασίας

Κάθε συνεδρία πρέπει να καθορίζει έναν κατάλογο εργασίας, ώστε ο Hermes να έχει τη σωστή πρόσβαση στα αρχεία του έργου. Σε σενάρια πολλών έργων, ο κατάλογος εργασίας πρέπει να εναλλάσσεται δυναμικά. Ακούγεται ξεκάθαρο, αλλά υπάρχουν περισσότερες περιπτώσεις αιχμής από ό, τι νομίζετε.

### Συνάθροιση απαντήσεων

Οι απαντήσεις του Ερμή μπορεί να χωριστούν `session/update` ειδοποιήσεις και το τελικό αποτέλεσμα, επομένως πρέπει να συγχωνευθούν σωστά. Διαφορετικά, το περιεχόμενο μπορεί να χαθεί.

### Στρατηγική χειρισμού σφαλμάτων

Τα σφάλματα χρόνου εκτέλεσης θα πρέπει να επιστρέφονται ρητά αντί να επιστρέφονται σιωπηλά σε άλλον πάροχο. Με αυτόν τον τρόπο, οι χρήστες γνωρίζουν ότι το πρόβλημα προήλθε από την Hermes αντί να αναρωτιούνται γιατί το σύστημα άλλαξε ξαφνικά μοντέλα στα παρασκήνια.

## Συμπέρασμα

Η απόφαση της HagiCode να χρησιμοποιήσει τον Hermes ως τον ενσωματωμένο πυρήνα Agent δεν ήταν μια περιστασιακή παρόρμηση. Ήταν μια προσεκτική επιλογή με βάση τις πρακτικές απαιτήσεις και τα τεχνικά χαρακτηριστικά του πλαισίου. Το αν αποδεικνύεται η τέλεια μακροπρόθεσμη απάντηση είναι ακόμα πολύ νωρίς για να πούμε, αλλά μέχρι στιγμής μας έχει εξυπηρετήσει καλά.

Η Hermes δίνει στην HagiCode την ευελιξία να προσαρμόζεται σε ένα ευρύ φάσμα σεναρίων. Το ισχυρό σύστημα εργαλείων και η υποστήριξη MCP επιτρέπουν στον βοηθό τεχνητής νοημοσύνης να κάνει πραγματική δουλειά, ενώ το πρωτόκολλο ACP και το επίπεδο αφαίρεσης του παρόχου διατηρούν τη διαδικασία ολοκλήρωσης σαφή και ελεγχόμενη.

Εάν επιλέγετε ένα πλαίσιο Agent για το δικό σας έργο AI, ελπίζω αυτό το άρθρο να προσφέρει μια χρήσιμη αναφορά. Η επιλογή της σωστής βασικής αρχιτεκτονικής μπορεί να κάνει οτιδήποτε ακολουθεί πολύ πιο εύκολο.
