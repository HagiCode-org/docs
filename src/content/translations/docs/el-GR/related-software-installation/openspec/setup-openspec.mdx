# Εγκαταστήστε το OpenSpec

Το OpenSpec διαχειρίζεται προτάσεις, προδιαγραφές και εγγραφές αλλαγών. Αυτή η σελίδα διατηρεί έναν στόχο: εξηγήστε τη διαδρομή εγκατάστασης με σαφήνεια, ώστε να μπορείτε να την επισυνάψετε στην υπάρχουσα ροή εργασίας σας.

## Τι είναι το OpenSpec;

Το OpenSpec είναι ένα εργαλείο γραμμής εντολών για τη διαχείριση προτάσεων και προδιαγραφών στην εργασία λογισμικού. Η βασική ροή εργασίας περιλαμβάνει:

- **Διαχείριση προτάσεων**: Δημιουργήστε, ελέγξτε και παρακολουθήστε προτάσεις χαρακτηριστικών
- **Επικύρωση προδιαγραφών**: Βεβαιωθείτε ότι οι αλλαγές κώδικα συμμορφώνονται με προκαθορισμένες προδιαγραφές
- **Παρακολούθηση αλλαγής**: Καταγράψτε όλο το ιστορικό αλλαγών που εξετάστηκε
- **Συνεργασία ομάδας**: Υποστήριξη συνεργασίας για πολλαπλά άτομα

### Εφαρμογή σε HagiCode Projects

Στα έργα HagiCode, το OpenSpec χρησιμοποιείται για:

1. **Διαχείριση ανάπτυξης λειτουργιών**: Όλες οι νέες δυνατότητες πρέπει να περάσουν από τη διαδικασία πρότασης OpenSpec
2. **Επισκόπηση κώδικα**: Βεβαιωθείτε ότι οι αλλαγές συμμορφώνονται με τις προδιαγραφές του έργου
3. **Συγχρονισμός τεκμηρίωσης**: Διατηρήστε τη συνέπεια μεταξύ της τεκμηρίωσης και της εφαρμογής κώδικα
4. **Ενσωμάτωση ελέγχου έκδοσης**: Ομαλή ενοποίηση με ροές εργασιών ελέγχου έκδοσης

## Προαπαιτούμενα

Πριν εγκαταστήσετε το OpenSpec, βεβαιωθείτε ότι το σύστημά σας πληροί τις ακόλουθες απαιτήσεις:

### Node.js και npm

Το OpenSpec διανέμεται μέσω npm και απαιτεί περιβάλλον Node.js. Το OpenSpec απαιτεί **Node.js 18.0 ή νεότερη έκδοση**.

Επαληθεύστε την εγκατάσταση του Node.js:

```bash
node --version
```

Επαλήθευση εγκατάστασης npm:

```bash
npm --version
```

Εάν δεν έχει εγκατασταθεί ή η έκδοση είναι πολύ παλιά, ανατρέξτε στις σχετικές οδηγίες εγκατάστασης λογισμικού στο [Οδηγός εγκατάστασης](/en/installation).

## Εγκατάσταση OpenSpec

:::tip[Απαιτήσεις έκδοσης]
Αυτό το έργο απαιτεί χρήση εύρους εκδόσεων **OpenSpec 1.0.0 ή νεότερης έκδοσης**. Υποστηρίζονται όλες οι εκδόσεις 1.x για να διασφαλίζεται η συμβατότητα με τη ροή εργασίας του έργου.
:::

Το OpenSpec εγκαθίσταται παγκοσμίως μέσω npm, καθιστώντας το διαθέσιμο από οπουδήποτε στο σύστημά σας.

### Καθολική εγκατάσταση με χρήση npm

<Tabs>
  <TabItem value="win" label="Windows">
    Εκτέλεση στο PowerShell ή στη γραμμή εντολών:

```powershell
npm install -g @fission-ai/openspec@1
```

:::note[Ζητήματα αδειών των Windows]
Εάν αντιμετωπίσετε σφάλματα δικαιωμάτων κατά την εγκατάσταση, εκτελέστε το PowerShell ή τη Γραμμή εντολών ως διαχειριστής.
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    Εκτέλεση στο τερματικό:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[Ζητήματα άδειας macOS]
Αν συναντήσετε `EACCES` σφάλματα άδειας, δοκιμάστε:
```bash
sudo npm install -g @fission-ai/openspec@1
```
Ή χρησιμοποιήστε [nvm](https://github.com/nvm-sh/nvm) για να διαχειριστείτε τις εκδόσεις του Node.js και να αποφύγετε προβλήματα αδειών.
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    Εκτέλεση στο τερματικό:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[Ζητήματα άδειας Linux]
Αν συναντήσετε `EACCES` σφάλματα άδειας, μπορείτε:
1. Χρήση `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. Διαμόρφωση npm για χρήση καταλόγου χρήστη:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### Επαλήθευση εγκατάστασης

Μετά την εγκατάσταση, βεβαιωθείτε ότι το OpenSpec έχει εγκατασταθεί σωστά και ότι η έκδοση πληροί τις απαιτήσεις:

```bash
openspec --version
```

Εάν εγκατασταθεί με επιτυχία, θα δείτε την έξοδο του αριθμού έκδοσης, για παράδειγμα:

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::note[Εύρος έκδοσης]
Βεβαιωθείτε ότι η εγκατεστημένη έκδοση OpenSpec είναι 1.0.0 ή νεότερη. Το έργο έχει επαληθευμένη συμβατότητα με την έκδοση 1.1.1. Συνιστάται η χρήση της πιο πρόσφατης συμβατής έκδοσης για να αποκτήσετε νέες δυνατότητες.
:::

## Κοινά ζητήματα και αντιμετώπιση προβλημάτων

### Σφάλματα άδειας

#### Θέμα: `EACCES` Δεν επιτρέπεται η άδεια

Αντιμετώπιση σφαλμάτων άδειας κατά την εγκατάσταση ή την εκτέλεση του OpenSpec.

**Λύσεις**:

<Tabs>
  <TabItem value="win" label="Windows">
    Εκτελέστε το PowerShell ή τη γραμμή εντολών ως διαχειριστής:
    1. Κάντε δεξί κλικ στο PowerShell ή στη Γραμμή εντολών
    2. Επιλέξτε "Εκτέλεση ως διαχειριστής"
    3. Εκτελέστε ξανά την εντολή εγκατάστασης
  </TabItem>
  <TabItem value="mac" label="macOS">
    Εγκαταστήστε χρησιμοποιώντας `sudo`:

```bash
sudo npm install -g @fission-ai/openspec@1
```

Ή χρησιμοποιήστε [nvm](https://github.com/nvm-sh/nvm) για να διαχειριστείτε το Node.js και να αποφύγετε προβλήματα καθολικών αδειών εγκατάστασης.
  </TabItem>
  <TabItem value="linux" label="Linux">
    Λύση 1: Χρησιμοποιήστε sudo
```bash
sudo npm install -g @fission-ai/openspec@1
```

Λύση 2: Διαμόρφωση npm για χρήση του καταλόγου χρήστη
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @fission-ai/openspec@1
```
  </TabItem>
</Tabs>

### Θέματα Δικτύου

#### Πρόβλημα: npm Η εγκατάσταση αποτυγχάνει ή είναι πολύ αργή

Δεν είναι δυνατή η λήψη του OpenSpec από το μητρώο npm για λόγους δικτύου.

**Λύσεις**:

1. **Χρησιμοποιήστε το Domestic npm Mirror**:

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **Επαναφορά επίσημης πηγής** (αν χρειάζεται):

```bash
npm config set registry https://registry.npmjs.org
```

3. **Χρήση cnpm** (προαιρετικό):

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

:::συμβουλή
Συνιστάται στους χρήστες στην ηπειρωτική Κίνα να χρησιμοποιούν τον καθρέφτη Taobao, ο οποίος μπορεί να βελτιώσει σημαντικά την ταχύτητα λήψης.
:::

### Ζητήματα συμβατότητας έκδοσης

#### Πρόβλημα: Οι εντολές OpenSpec εκτελούνται ασυνήθιστα ή αναφέρουν σφάλματα

Η εγκατεστημένη έκδοση OpenSpec δεν είναι συμβατή με το τρέχον περιβάλλον.

**Λύσεις**:

1. **Ελέγξτε την έκδοση Node.js**:

```bash
node --version
```

Βεβαιωθείτε ότι η έκδοση είναι 18.0 ή νεότερη. Εάν η έκδοση είναι πολύ χαμηλή, αναβαθμίστε το Node.js.

2. **Ελέγξτε την έκδοση OpenSpec**:

```bash
openspec --version
```

3. **Επανεγκατάσταση συγκεκριμένης έκδοσης**:

```bash
npm install -g @fission-ai/openspec@1
```

4. **Εκκαθάριση προσωρινής μνήμης npm** (αν το πρόβλημα παραμένει):

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### Η εντολή δεν βρέθηκε

#### Θέμα: Δακτυλογράφηση `openspec` Η εντολή δείχνει "Η εντολή δεν βρέθηκε"

Το OpenSpec είναι εγκατεστημένο αλλά το σύστημα δεν μπορεί να βρει την εντολή.

**Λύσεις**:

<Tabs>
  <TabItem value="win" label="Windows">
    1. Επιβεβαίωση συνολικής διαδρομής εγκατάστασης npm:

```powershell
npm config get prefix
```

2. Προσθήκη διαδρομής στη διαδρομή συστήματος:
   - Ανοίξτε "Ιδιότητες συστήματος" → "Για προχωρημένους" → "Μεταβλητές περιβάλλοντος"
   - Βρείτε `Path` σε "Μεταβλητές συστήματος"
   - Προσθήκη καθολικής διαδρομής npm (συνήθως `C:\Users\<username>\AppData\Roaming\npm`)
   - Επανεκκινήστε το παράθυρο της γραμμής εντολών
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. Επιβεβαίωση συνολικής διαδρομής εγκατάστασης npm:

```bash
npm config get prefix
```

2. Προσθήκη διαδρομής στο PATH (σε `~/.zshrc` ή `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Επαναφόρτωση διαμόρφωσης:

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. Επιβεβαίωση συνολικής διαδρομής εγκατάστασης npm:

```bash
npm config get prefix
```

2. Προσθήκη διαδρομής στο PATH (σε `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Επαναφόρτωση διαμόρφωσης:

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
