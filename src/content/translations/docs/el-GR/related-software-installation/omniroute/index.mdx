Αυτή η σελίδα απευθύνεται σε χρήστες του HagiCode για πρώτη φορά που θέλουν τη συντομότερη διαδρομή εγκατάστασης OmniRoute: ξεκινήστε την τοπική υπηρεσία, επαληθεύστε ότι `http://localhost:20128/v1` λειτουργεί και επιβεβαιώστε ότι τόσο ο Πίνακας ελέγχου όσο και το API είναι διαθέσιμα.

:::tip[Προτεινόμενη διαδρομή]
Αυτή η πρώτη έκδοση διατηρεί πρώτα την πιο σταθερή διαδρομή: `npm install -g omniroute`, `omniroute`, Ταμπλό στο `http://localhost:20128`, και το Βασικό URL συμβατό με OpenAI στο `http://localhost:20128/v1`. Οι λειτουργίες Docker, VPS και πιο προηγμένες δυνατότητες ανάπτυξης παραμένουν στην προαιρετική ενότητα.
:::

## Προαπαιτούμενα

- Το Node.js είναι ήδη εγκατεστημένο. Ολοκληρώθηκε [Εγκατάσταση του Node.js](/en/related-software-installation/nodejs/installation) πρώτα αν χρειαστεί.
- Τοπικό λιμάνι `20128` είναι διαθέσιμο και δεν είναι ήδη κατειλημμένο.
- Έχετε τουλάχιστον έναν λογαριασμό παρόχου μοντέλου ανάντη τον οποίο σκοπεύετε να δρομολογήσετε μέσω του OmniRoute.
- Πριν ενσωματώσετε το HagiCode, ένα CLI ή ένα SDK, διατηρήστε αυτές τις δύο διευθύνσεις ξεχωριστές:
  - Ταμπλό: `http://localhost:20128`
  - Βασική διεύθυνση URL συμβατή με OpenAI: `http://localhost:20128/v1`

## Εγκαταστήστε και εκτελέστε τοπικά

Ξεκινήστε με την προτεινόμενη καθολική διαδρομή npm:

```bash
npm install -g omniroute
omniroute
```

Αφού ξεκινήσει η υπηρεσία, επικυρώστε την με αυτή τη σειρά:

1. Ανοίξτε `http://localhost:20128` σε ένα πρόγραμμα περιήγησης και επιβεβαιώστε ότι ο Πίνακας εργαλείων φορτώνεται.
2. Πηγαίνετε στο `Endpoints` σελίδα στον Πίνακα ελέγχου και δημιουργήστε ή αντιγράψτε ένα κλειδί API.
3. Σημειώστε τη βασική διεύθυνση URL που θα παραδώσετε στα εργαλεία αργότερα: `http://localhost:20128/v1`.

:::note[Γιατί να ξεκινήσετε με τον πίνακα ελέγχου]
Για την πρώτη ρύθμιση, ο Πίνακας ελέγχου είναι ο πιο γρήγορος τρόπος για να επιβεβαιώσετε ότι το OmniRoute εκτελείται, οι πάροχοι σας είναι συνδεδεμένοι και ότι υπάρχει κλειδί API. Εάν μια μεταγενέστερη κλήση πελάτη αποτύχει, ο πρώτος έλεγχος του Πίνακα ελέγχου είναι συνήθως ταχύτερος από τον αμέσως εντοπισμό σφαλμάτων του κώδικα SDK.
:::

## Επικυρώστε την υπηρεσία που εκτελείται

### Επικύρωση 1: επιβεβαιώστε ότι ο κατάλογος του μοντέλου είναι αναγνώσιμος

Βάλτε το κλειδί από το `Endpoints` σελίδα σε μια μεταβλητή περιβάλλοντος:

```bash
export OMNIROUTE_API_KEY="replace-with-the-key-you-copied-from-the-Endpoints-page"
```

Μετά ζητήστε `/v1/models`:

```bash
curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}"
```

Εάν λάβετε πίσω μια λίστα μοντέλων, το τελικό σημείο και η ροή εξουσιοδότησης συμβατή με OpenAI λειτουργούν.

### Επικύρωση 2: στείλτε ένα ελάχιστο αίτημα συνομιλίας

```bash
curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "replace-with-a-model-id-from-v1-models",
    "messages": [
      { "role": "user", "content": "Reply with ok." }
    ]
  }'
```

Ο στόχος εδώ δεν είναι η άμεση ποιότητα. Είναι για να αποδείξετε ότι το μοντέλο που έχετε επιλέξει μπορεί ήδη να δρομολογηθεί με επιτυχία μέσω του OmniRoute.
