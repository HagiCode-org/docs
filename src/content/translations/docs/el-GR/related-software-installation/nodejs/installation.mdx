Αυτός ο οδηγός καλύπτει την εγκατάσταση του Node.js σε Windows, macOS και Linux. Το Node.js είναι μια βασική εξάρτηση για τα τοπικά εργαλεία HagiCode, συμπεριλαμβανομένων των OpenSpec, εργαλείων frontend και πολλών CLI προγραμματιστών.

:::tip[Απαιτήσεις έκδοσης]
Η HagiCode προτείνει το **Node.js 24+** (ελάχιστο υποστηριζόμενο Node.js &gt;= 18.0). Στις περισσότερες περιπτώσεις, εγκαταστήστε την πιο πρόσφατη έκδοση LTS και, στη συνέχεια, βεβαιωθείτε ότι η έκδοσή σας πληροί την απαίτηση.
:::

## Εγκατάσταση του Node.js

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Μέθοδος 1: Επίσημος εγκαταστάτης (Συνιστάται)

1. Επισκεφθείτε το [Επίσημος ιστότοπος Node.js](https://nodejs.org/)
2. Πραγματοποιήστε λήψη της έκδοσης **LTS (Long Term Support)** του προγράμματος εγκατάστασης των Windows (`.msi` αρχείο)
3. Κάντε διπλό κλικ για να εκτελέσετε το πρόγραμμα εγκατάστασης
4. Ακολουθήστε τις οδηγίες του οδηγού εγκατάστασης για να ολοκληρώσετε την εγκατάσταση:
   - Αποδεχτείτε τη συμφωνία άδειας χρήσης
   - Επιλέξτε διαδρομή εγκατάστασης (η προεπιλογή είναι καλή)
   - Βεβαιωθείτε ότι είναι επιλεγμένο το "Αυτόματη εγκατάσταση των απαραίτητων εργαλείων".
   - Κάντε κλικ στο "Εγκατάσταση" για να ξεκινήσει η εγκατάσταση
5. Αφού ολοκληρωθεί η εγκατάσταση, κάντε κλικ στο "Τέλος"

### Μέθοδος δεύτερη: Χρήση winget

Ανοίξτε το PowerShell ή τη γραμμή εντολών και εκτελέστε:

```powershell
winget install OpenJS.NodeJS.LTS
```

Μετά την εγκατάσταση, ανοίξτε ξανά το τερματικό για να τεθούν σε ισχύ οι αλλαγές.

  </TabItem>
  <TabItem value="macos" label="macOS">

### Μέθοδος 1: Επίσημος εγκαταστάτης (Συνιστάται)

1. Επισκεφθείτε το [Επίσημος ιστότοπος Node.js](https://nodejs.org/)
2. Κάντε λήψη της έκδοσης **LTS (Long Term Support)** του προγράμματος εγκατάστασης macOS (`.pkg` αρχείο)
3. Κάντε διπλό κλικ για να εκτελέσετε το πρόγραμμα εγκατάστασης
4. Ακολουθήστε τις οδηγίες του οδηγού εγκατάστασης για να ολοκληρώσετε την εγκατάσταση:
   - Κάντε κλικ στο "Συνέχεια"
   - Αποδεχτείτε τη συμφωνία άδειας χρήσης
   - Επιλέξτε δίσκο εγκατάστασης
   - Κάντε κλικ στο "Εγκατάσταση" και εισαγάγετε τον κωδικό πρόσβασης διαχειριστή
5. Αφού ολοκληρωθεί η εγκατάσταση, κάντε κλικ στο "Κλείσιμο"

### Μέθοδος δεύτερη: Χρήση Homebrew

Εάν έχετε [Homebrew](https://brew.sh/) εγκατεστημένο, μπορείτε να εγκαταστήσετε το Node.js μέσω τερματικού:

```bash
brew install node
```

:::συμβουλή
Το Homebrew θα προσθέσει αυτόματα το Node.js στο PATH σας. Εάν η εντολή δεν είναι διαθέσιμη, επανεκκινήστε το τερματικό ή εκτελέστε:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

  </TabItem>

### Χρήση του αποθετηρίου NodeSource (Συνιστάται)

Αυτός είναι ο καλύτερος τρόπος για να αποκτήσετε την πιο πρόσφατη έκδοση Node.js:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Χρησιμοποιώντας το apt Package Manager

Εάν δεν χρειάζεστε την πιο πρόσφατη έκδοση, μπορείτε να χρησιμοποιήσετε το προεπιλεγμένο αποθετήριο του Ubuntu:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::σημείωση
Η έκδοση Node.js στο προεπιλεγμένο αποθετήριο μπορεί να είναι παλαιότερη. Μετά την εγκατάσταση, ελέγξτε την έκδοση για να βεβαιωθείτε ότι πληροί τις απαιτήσεις Node.js &gt;= 18.0 (συνιστάται το Node.js 24+).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

```bash

```

## Χρήση εργαλείων διαχείρισης εκδόσεων

Εάν χρειάζεται να κάνετε εναλλαγή μεταξύ πολλών εκδόσεων Node.js, μπορείτε να χρησιμοποιήσετε εργαλεία διαχείρισης εκδόσεων.

### nvm (Διαχείριση εκδόσεων κόμβου)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

Μετά την εγκατάσταση, φορτώστε ξανά τη διαμόρφωση του κελύφους σας:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

Στα Windows, συνιστούμε τη χρήση [nvm-windows](https://github.com/coreybutler/nvm-windows):

1. Κατεβάστε το πιο πρόσφατο `nvm-setup.exe` από [εκδόσεις nvm-windows](https://github.com/coreybutler/nvm-windows/releases)
2. Εκτελέστε το πρόγραμμα εγκατάστασης
3. Ανοίξτε ξανά τη γραμμή εντολών ή το PowerShell

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

Μετά την εγκατάσταση του nvm, μπορείτε να το χρησιμοποιήσετε για να εγκαταστήσετε και να διαχειριστείτε τις εκδόσεις του Node.js:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (Fast Node Manager)

Το fnm είναι ένα πιο γρήγορο και απλούστερο εργαλείο διαχείρισης εκδόσεων Node.js που έχει δημιουργηθεί με Rust.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

Χρήση [σέσουλα](https://scoop.sh/) ή [winget](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Επιτάχυνση για χρήστες της ηπειρωτικής Κίνας

Εάν βρίσκεστε στην ηπειρωτική Κίνα, η λήψη των πακέτων Node.js και npm ενδέχεται να είναι αργή. Εδώ είναι μερικές λύσεις επιτάχυνσης.

### Χρήση οικιακών πηγών καθρέφτη

#### npm Taobao Mirror (Συνιστάται)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### Χρησιμοποιώντας το nrm για τη διαχείριση πηγών κατοπτρισμού

Το nrm (npm registry manager) σάς βοηθά να αλλάζετε γρήγορα πηγές κατοπτρισμού npm:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Επιτάχυνση λήψεων του προγράμματος εγκατάστασης Node.js

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Το Taobao παρέχει κατοπτρικές λήψεις για προγράμματα εγκατάστασης Node.js:

- **Διεύθυνση λήψης**: https://npmmirror.com/mirrors/node/

Επισκεφτείτε την παραπάνω διεύθυνση και επιλέξτε την έκδοση που θέλετε να κατεβάσετε.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

Εάν η λήψη του nvm ή άλλων εργαλείων από το GitHub είναι αργή, μπορείτε να χρησιμοποιήσετε τον καθρέφτη FastGit:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::σημείωση
Η υπηρεσία FastGit ενδέχεται να αλλάξει, ανατρέξτε στις επίσημες πληροφορίες.
:::

  </TabItem>
</Tabs>

### Οικιακός καθρέφτης Homebrew

Εάν χρησιμοποιείτε το Homebrew στην ηπειρωτική Κίνα, μπορείτε να διαμορφώσετε τους οικιακούς καθρέφτες για επιτάχυνση:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Ειδικοί διαχειριστές πακέτων ηπειρωτικής Κίνας

#### cnpm

Το cnpm είναι ένας πελάτης npm που παρέχεται από την ομάδα Taobao που χρησιμοποιεί τον καθρέφτη Taobao από προεπιλογή:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::συμβουλή
Το cnpm είναι ως επί το πλείστον συμβατό με εντολές npm και μπορεί να χρησιμοποιηθεί ως άμεση αντικατάσταση.
:::

### Επαλήθευση εφέ επιτάχυνσης καθρέφτη

Αφού ρυθμίσετε τους καθρέφτες, επαληθεύστε από:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Επαλήθευση εγκατάστασης

Μετά την εγκατάσταση, βεβαιωθείτε ότι το Node.js και το npm έχουν εγκατασταθεί σωστά:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::σημείωση
Βεβαιωθείτε ότι η έκδοση Node.js &gt;= 18.0 (συνιστάται το Node.js 24+). Εάν η έκδοση είναι πολύ χαμηλή, εγκαταστήστε ξανά χρησιμοποιώντας τις παραπάνω μεθόδους ή χρησιμοποιήστε εργαλεία διαχείρισης έκδοσης για να εγκαταστήσετε τη σωστή έκδοση.
:::

### Δείγμα εξόδου επαλήθευσης

Η έξοδος επιτυχούς επαλήθευσης θα πρέπει να έχει την εξής μορφή:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Αντιμετώπιση προβλημάτων

### Θέματα Windows

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Αν `node` ή `npm` Οι εντολές δεν είναι διαθέσιμες μετά την εγκατάσταση:

1. Επανεκκινήστε τη γραμμή εντολών ή το PowerShell
2. Εάν εξακολουθεί να μην είναι διαθέσιμο, προσθέστε μη αυτόματα το Node.js στο PATH:
   - Αναζητήστε "Μεταβλητές περιβάλλοντος" και ανοίξτε το "Επεξεργασία μεταβλητών περιβάλλοντος συστήματος"
   - Κάντε κλικ στην επιλογή "Μεταβλητές περιβάλλοντος"
   - Βρείτε "Διαδρομή" στις "Μεταβλητές συστήματος" και κάντε κλικ στο "Επεξεργασία"
   - Προσθήκη διαδρομής εγκατάστασης Node.js (προεπιλογή: `C:\Program Files\nodejs\`)
   - Κάντε κλικ στο "OK" για να αποθηκεύσετε τις αλλαγές

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Εάν αντιμετωπίσετε σφάλματα πολιτικής εκτέλεσης κατά τη χρήση του nvm:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### Ζητήματα macOS

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Εάν αντιμετωπίσετε σφάλματα δικαιωμάτων κατά την εγκατάσταση παγκόσμιων πακέτων npm:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Εάν οι εντολές δεν είναι διαθέσιμες μετά την εγκατάσταση, επανεκκινήστε το τερματικό ή προσθέστε μη αυτόματα στο PATH (συνήθως σε `/usr/local/bin`).

  </TabItem>
</Tabs>

### Θέματα Linux

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Εάν αντιμετωπίσετε σφάλματα αδειών κατά την εγκατάσταση παγκόσμιων πακέτων npm, συνιστάται να χρησιμοποιείτε εργαλεία διαχείρισης εκδόσεων Node.js (όπως nvm ή fnm) αντί για διαχειριστές πακέτων συστήματος.

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Εάν η έκδοση Node.js που έχει εγκατασταθεί μέσω του διαχειριστή πακέτων συστήματος είναι πολύ παλιά:

1. Χρησιμοποιήστε το αποθετήριο NodeSource (συνιστάται, δείτε παραπάνω)
2. Ή χρησιμοποιήστε εργαλεία διαχείρισης εκδόσεων (nvm/fnm)

  </TabItem>
</Tabs>

### Έλεγχος περιβάλλοντος

Εάν η επαλήθευση αποτύχει, ελέγξτε τα ακόλουθα:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
