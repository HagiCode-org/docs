## Επισκόπηση

Το Monospecs είναι η προσέγγιση διαχείρισης πολλαπλών αποθετηρίων της HagiCode. Παρακολουθεί και διαχειρίζεται τα υποαποθετήρια μέσω `monospecs.yaml`, και λειτουργεί για κάθε έργο που χρειάζεται ένα μέρος για τον συντονισμό πολλών ανεξάρτητων αποθετηρίων.

### Βασικές Έννοιες

**Αρχείο διαμόρφωσης**

`monospecs.yaml` αποτελεί τον πυρήνα ολόκληρης της λύσης διαχείρισης, ορίζοντας:
- Λίστα και τοποθεσία των υποαποθετηρίων
- Git URL για κάθε αποθετήριο
- Εμφανιζόμενο όνομα και εικονίδιο
- Συμπεριφορά ενσωμάτωσης με το OpenSpec

**Διαφορές από τα κανονικά αποθετήρια**

| Χαρακτηριστικό | Κανονικό αποθετήριο | Monospecs Managed Project |
|----------|-------------------|---------------------------|
| **Τοποθεσία OpenSpec** | Καταλόγου ρίζας `openspec/` | Κύρια αποθήκη ριζικού καταλόγου `openspec/` |
| **Αλλαγή πεδίου εφαρμογής** | Περιορίζεται στο τρέχον αποθετήριο | Μπορεί να περιλαμβάνει πολλά υποαποθετήρια |
| **Υπο-αποθήκη καθαρότητας** | Προδιαγραφές αναμεμειγμένες με κώδικα | Οι προδιαγραφές που καθορίζονται στο κύριο αποθετήριο, τα υποαποθετήρια είναι πιο καθαρά |
| **Αυτόματη δέσμευση αρχειοθέτησης** | Χρειάζεται χειροκίνητη δέσμευση προδιαγραφών | `commit_when_archive: true` δεσμεύει αυτόματα τις προδιαγραφές στο κύριο αποθετήριο |

### Γιατί να χρησιμοποιήσετε μονοπροδιαγραφές;

- **Ενοποιημένη διαχείριση**: Διαχείριση όλων των υπο-αποθετηρίων σε ένα αρχείο διαμόρφωσης
- **Υποστήριξη Αυτοματισμού**: Παρέχει μεταδεδομένα αποθήκης για σενάρια και εργαλεία
- ** AI Friendliness**: Η δομημένη διαμόρφωση βοηθά τον παράγοντα AI να κατανοήσει τις σχέσεις αποθετηρίου
- **OpenSpec Integration**: Ομαλή ενοποίηση με τη ροή εργασιών διαχείρισης αλλαγών
- **Cleaner Sub-repositories**: Οι προδιαγραφές διατηρούνται στο κύριο αποθετήριο, οι υπο-αποθήκες περιέχουν μόνο τον πραγματικό κωδικό

### Ισχύοντα σενάρια

Χρησιμοποιήστε τα Monospecs πρώτα σε σενάρια όπως αυτά:

**1. Συνεργατική ανάπτυξη πολλαπλών αποθετηρίων**
Όταν ένα έργο χωρίζεται σε πολλαπλά ανεξάρτητα αποθετήρια:
- Η εφαρμογή Frontend, η υπηρεσία υποστήριξης και η εφαρμογή επιφάνειας εργασίας αναπτύχθηκαν ξεχωριστά
- Ανάγκη συντονισμού εκδόσεων και εξαρτήσεων μεταξύ των αποθετηρίων
- Τεκμηρίωση και βοηθητικά έργα χωριστά από τον κύριο κώδικα

**2. Βοηθητική Διαχείριση Έργων**
Χρησιμοποιήστε Monospecs για να διαχειριστείτε:
- Έργο τεκμηρίωσης
- Επίσημος ιστότοπος
- Δημιουργία εργαλείων και σεναρίων
- Κάθε αποθετήριο διατηρεί τον δικό του έλεγχο έκδοσης

**3. Ανάπτυξη λειτουργιών μεταξύ αποθετηρίων**
Μια δυνατότητα που απαιτεί αλλαγές σε πολλαπλά αποθετήρια:
- Ανάγκη παρακολούθησης αλλαγών σε ποια αποθετήρια
- Αυτόματη δέσμευση προδιαγραφών για τη διόρθωση του αποθετηρίου προορισμού
- Συντονίστε ενημερώσεις και εξαρτήσεις έκδοσης

## Γρήγορη εκκίνηση

### Αρχικοποίηση

Ακολουθήστε αυτά τα βήματα για να ολοκληρώσετε την προετοιμασία μονοπροδιαγραφών:

#### 1. Δημιουργήστε Empty git Repository

Δημιουργήστε ένα κενό `git` φάκελο στο κύριο αποθετήριο σας (σημείωση: αυτό είναι `git`, όχι `repos/` - αυτή είναι η σύμβαση των μονοπροδιαγραφών):

```bash
mkdir git
cd git
git init
cd ..
```

#### 2. Διαμορφώστε το αρχείο monospecs

Δημιουργήστε το `monospecs.yaml` αρχείο ρυθμίσεων στον ριζικό κατάλογο του κύριου αποθετηρίου:

```yaml
version: "1.0"
commit_when_archive: true

repositories:
  - path: "repos/frontend"
    url: "https://github.com/your-org/frontend.git"
    displayName: "Frontend App"
    icon: "🌐"

  - path: "repos/backend"
    url: "https://github.com/your-org/backend.git"
    displayName: "Backend Service"
    icon: "⚙️"
```

**Στοιχεία διαμόρφωσης**:
| Στοιχείο διαμόρφωσης | Τύπος | Απαιτείται | Περιγραφή |
|----------------|------|----------|-------------|
| `version` | χορδή | Ναι | Έκδοση αρχείου διαμόρφωσης |
| `commit_when_archive` | boolean | Όχι | Εάν θα γίνει αυτόματη δέσμευση προδιαγραφών κατά την αρχειοθέτηση (μην δεσμεύσετε κωδικό δευτερεύοντος αποθέματος) |
| `repositories` | συστοιχία | Ναι | Λίστα διαμόρφωσης αποθετηρίου |
| `repositories[].path` | χορδή | Ναι | Διαδρομή αποθετηρίου (σε σχέση με τη ρίζα του έργου) |
| `repositories[].url` | χορδή | Ναι | URL αποθετηρίου Git |
| `repositories[].displayName` | χορδή | Όχι | Εμφανιζόμενο όνομα χώρου αποθήκευσης |
| `repositories[].icon` | χορδή | Όχι | Εικονίδιο αποθετηρίου (emoji) |

### Προσθήκη υπο-αποθετηρίων στο κύριο αποθετήριο

Προσθέστε όλα τα υποαποθετήρια στο `repos/` καταλόγου. Αυτός ο κατάλογος πρέπει να **εξαιρείται από τον έλεγχο έκδοσης Git του κύριου αποθετηρίου** για να αποφευχθεί η παρακολούθηση περιεχομένου υπο-αποθήκης:

```plaintext
# Exclude all sub-repositories
repos/
```

#### 3. Προσθήκη γονικού αποθετηρίου στο HagiCode

Εάν χρησιμοποιείτε την εφαρμογή επιφάνειας εργασίας HagiCode, ανοίξτε την εφαρμογή και προσθέστε το κύριο αποθετήριο ως έργο. Κάντε κλικ στο κουμπί "Προσθήκη έργου" ή "+" στην αρχική οθόνη, επιλέξτε τον κύριο φάκελο του αποθετηρίου σας και θα δείτε όλα τα υποαποθετήρια και τις πληροφορίες κατάστασής τους.

#### 4. Κλώνοι Υπο-αποθετήρια

Χρησιμοποιήστε το `monospecs.yaml` ρύθμιση παραμέτρων για κλωνοποίηση όλων των διαμορφωμένων υπο-αποθετηρίων στο `repos/` κατάλογος:

Αυτό θα κλωνοποιήσει όλα τα δευτερεύοντα αποθετήρια με βάση τις διευθύνσεις URL που καθορίζονται στο αρχείο διαμόρφωσης.

## Δομή αρχείου διαμόρφωσης

### Διαμόρφωση επιπέδου ρίζας

```yaml
version: "1.0"
commit_when_archive: true

repositories:
  # ... repository configurations
```

### Διαμόρφωση επιπέδου αποθετηρίου

Κάθε υπο-αποθήκη μπορεί να έχει:
- Φάκελος αλλαγής OpenSpec (`openspec/`)
- Ανεξάρτητος `.git/` καταλόγου
- Προαιρετικό `AGENTS.md` για τη διαμόρφωση AI Agent

## Λειτουργίες διαχείρισης αποθετηρίου

### Προσθήκη νέου αποθετηρίου

Όταν προσθέτετε ένα νέο υπο-αποθήκη στο `monospecs.yaml` `repositories` πίνακας:

```yaml
repositories:
  - path: "repos/new-service"
    url: "https://github.com/HagiCode-org/new-service.git"
    displayName: "New Service"
    icon: "🆕"
```

### Ενημέρωση διαμόρφωσης αποθετηρίου

Όταν αλλάζουν οι διευθύνσεις URL ή τα μεταδεδομένα αποθετηρίου:
1. Επεξεργασία `monospecs.yaml` για να ενημερώσετε τις αντίστοιχες καταχωρήσεις
2. Βεβαιωθείτε ότι η σύνταξη YAML είναι σωστή
3. Εάν αλλάξει ο συγχρονισμός, ενημερώστε μη αυτόματα τη διαμόρφωση τοπικού αποθετηρίου

## Υποστήριξη διαμόρφωσης πράκτορα AI

### ΠΡΑΚΤΟΡΕΣ.md

Το `AGENTS.md` Το αρχείο παρέχει καθοδήγηση ειδικά για το χώρο αποθήκευσης για τον Agent AI:

**Τεχνικές πληροφορίες στοίβας**
- Πλαίσιο και κατασκευή εργαλείων
- Συμβάσεις κώδικα και πρακτικές ονοματοδοσίας
- Διαχείριση έκδοσης και διαμόρφωση ανάπτυξης

**Συγκεκριμένες συμπεριφορές έργου**
- Επεκτάσεις διαμόρφωσης και ειδικές απαιτήσεις
- Μοτίβα ενσωμάτωσης OpenSpec
- Ανάγκες συντονισμού μεταξύ αποθετηρίων

### Οφέλη AI

Χρησιμοποιώντας `AGENTS.md` επιτρέπει στον Πράκτορα AI να:
- Κατανοήστε τις σχέσεις και τις εξαρτήσεις του αποθετηρίου
- Δημιουργήστε κώδικα και refactor σε πολλαπλά αποθετήρια
- Διατηρήστε τη συνέπεια στον έλεγχο έκδοσης και τις διαμορφώσεις

## Ενσωμάτωση OpenSpec

### Πώς λειτουργεί το Monospecs με το OpenSpec

Είτε διαχειρίζεστε ένα έργο με διαχείριση monospecs είτε ένα κανονικό αποθετήριο, μπορείτε να χρησιμοποιήσετε τη ροή εργασίας OpenSpec του HagiCode:

**Κανονικό αποθετήριο:**
- OpenSpec στο root `openspec/` καταλόγου
- Οι αλλαγές επηρεάζουν μόνο το τρέχον αποθετήριο

**Έργο που διαχειρίζεται μονοτεχνικά χαρακτηριστικά:**
- Το OpenSpec διαβάζει `monospecs.yaml` διαμόρφωσης
- Οι αλλαγές παρακολουθούνται από υπο-αποθήκη
- Οι προδιαγραφές δεσμεύονται αυτόματα για τους σωστούς στόχους

### Συμπεριφορά αρχείου

Πότε `commit_when_archive: true`:
- Οι προδιαγραφές δεσμεύονται αυτόματα στο κύριο αποθετήριο
- Ο κωδικός των υποαποθετηρίων δεν έχει δεσμευτεί (παραμένει ξεχωριστός)
- Απλοποιεί τη διαχείριση προδιαγραφών - δεν απαιτούνται μη αυτόματες δεσμεύσεις προδιαγραφών για υποαποθήκες

### Εγχειρίδιο Υπο-Αποθήκη Υποχρεώσεων

Τα υποαποθετήρια διατηρούν ανεξάρτητο έλεγχο έκδοσης:
- Διαπράττουν πραγματικές αλλαγές κώδικα στα δικά τους αποθετήρια
- Οι προδιαγραφές δεσμεύονται χωριστά (από το κύριο αποθετήριο κατά την αρχειοθέτηση)

## Βέλτιστες Πρακτικές

### Συμβάσεις ονομασίας αποθετηρίου

- Χρησιμοποιήστε κεμπάπ-πεζά (πεζά γράμματα και παύλες) για διαδρομές αποθήκευσης
- Χρησιμοποιήστε απλά, περιγραφικά εμφανιζόμενα ονόματα στα κινέζικα
- Χρησιμοποιήστε σχετικά emojis ως εικονίδια

### Οδηγίες εμφανιζόμενων ονομάτων

- Κρατήστε το συνοπτικό και περιγραφικό
- Χρησιμοποιήστε συνεπή ονομασία σε παρόμοια αποθετήρια
- Επιλέξτε εικονίδια που αντιπροσωπεύουν το σκοπό του αποθετηρίου

### Συντήρηση διαμόρφωσης

**Συγχρονισμός διαμόρφωσης με την πραγματική κατάσταση**
- Επαληθεύστε τακτικά `monospecs.yaml` αντικατοπτρίζει την πραγματική δομή του αποθετηρίου
- Ενημερώστε ή αφαιρέστε καταχωρήσεις όταν προστίθενται ή αφαιρούνται αποθετήρια
- Ελέγξτε τις αλλαγές για να διασφαλίσετε την εγκυρότητα της διαμόρφωσης

**Έλεγχος έκδοσης για διαμόρφωση**
- Παρακολουθήστε τις αλλαγές του αρχείου διαμόρφωσης με το git
- Τεκμηριώστε τους λόγους για σημαντικές αλλαγές διαμόρφωσης
- Επισημάνετε τις εκδόσεις του αρχείου διαμόρφωσης για εύκολη επαναφορά

## Αντιμετώπιση προβλημάτων

### Το αρχείο διαμόρφωσης δεν βρέθηκε

Αν το `monospecs.yaml` το αρχείο δεν βρίσκεται στη ρίζα του έργου:
1. Ελέγξτε εάν το αρχείο υπάρχει στη σωστή θέση
2. Βεβαιωθείτε ότι βρίσκεστε στον σωστό κατάλογο εργασίας
3. Ελέγξτε για λάθη στο όνομα του αρχείου διαμόρφωσης

### YAML Συντακτικά σφάλματα

Συνήθη προβλήματα σύνταξης YAML:
- Λανθασμένη εσοχή (χρησιμοποιήστε κενά, όχι καρτέλες)
- Λείπουν ή επιπλέον εισαγωγικά γύρω από τις χορδές
- Μη έγκυροι τύποι δεδομένων (οι συμβολοσειρές χρειάζονται εισαγωγικά, οι αριθμοί όχι)
- Λείπουν υποχρεωτικά πεδία (όπως `path` ή `url`)

### Το αποθετήριο δεν εντοπίστηκε

Εάν δεν εμφανίζεται ένα νέο αποθετήριο:
1. Ελέγξτε το `path` είναι σωστό σε `monospecs.yaml`
2. Βεβαιωθείτε ότι το αποθετήριο κλωνοποιήθηκε με επιτυχία
3. Ελέγξτε εάν το αποθετήριο υπάρχει στην αναμενόμενη τοποθεσία

### Αποτυχίες κλώνων

Εάν η κλωνοποίηση του αποθετηρίου αποτύχει:
1. Ελέγξτε τη σύνδεση δικτύου
2. Βεβαιωθείτε ότι η διεύθυνση URL του αποθετηρίου Git είναι σωστή
3. Ελέγξτε για προβλήματα ελέγχου ταυτότητας
4. Ελέγξτε την κατάσταση του κοντέινερ Docker
5. Ελέγξτε τη διαθεσιμότητα χώρου στο δίσκο
