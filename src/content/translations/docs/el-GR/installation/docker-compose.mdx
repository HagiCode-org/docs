:::tip[Συνιστώμενη μέθοδος]
Το Docker Compose είναι η προεπιλεγμένη διαδρομή ανάπτυξης στις περισσότερες περιπτώσεις επειδή:
- Η απομόνωση περιβάλλοντος μειώνει τις συγκρούσεις εξάρτησης
- η υποστηριζόμενη στοίβα υπηρεσιών Hagicode μπορεί να ξεκινήσει μαζί
- Η διαμόρφωση, οι αναβαθμίσεις και η συντήρηση είναι πιο εύκολο να διατηρηθούν συνεπείς
- λειτουργεί καλά για τοπικές δοκιμές, δοκιμές και ανάπτυξη ομάδας
:::

:::note[Σχετικά με την προεπιλεγμένη αποθήκευση δεδομένων]
Τα υποστηριζόμενα πρότυπα Docker Compose παραμένουν δεδομένα μέσω της προεπιλεγμένης τοπικής διαδρομής SQLite. Στις περισσότερες περιπτώσεις, δεν χρειάζεται να παρέχετε ξεχωριστή υπηρεσία βάσης δεδομένων στον κεντρικό υπολογιστή.
:::

## Προαπαιτούμενα

Πριν ξεκινήσετε, βεβαιωθείτε ότι το σύστημά σας έχει εγκαταστήσει το Docker και το Docker Compose.

### Εγκαταστήστε το Docker

#### Windows

Λήψη και εγκατάσταση [Docker Desktop για Windows](https://www.docker.com/products/docker-desktop/)

Μετά την εγκατάσταση, βεβαιωθείτε ότι το Docker Desktop εκτελείται.

#### macOS

Λήψη και εγκατάσταση [Docker Desktop για Mac](https://www.docker.com/products/docker-desktop/)

Μετά την εγκατάσταση, βεβαιωθείτε ότι το Docker Desktop εκτελείται.

#### Linux

Εγκαταστήστε το Docker χρησιμοποιώντας τον διαχειριστή πακέτων:

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

Μετά την εγκατάσταση, ξεκινήστε την υπηρεσία Docker:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Επαλήθευση εγκατάστασης

Μετά την εγκατάσταση, εκτελέστε τις ακόλουθες εντολές για να επαληθεύσετε ότι το Docker και το Docker Compose έχουν εγκατασταθεί σωστά:

```bash
docker --version
docker compose version
```

## Γρήγορη εκκίνηση

### 1. Δημιουργήστε το αρχείο διαμόρφωσης Docker Compose

:::tip[Χρησιμοποιήστε το Configuration Generator]
**Συστήστε τη χρήση [Docker Compose Builder (αυτόνομος ιστότοπος)](https://builder.hagicode.com/)** για να δημιουργήσετε το αρχείο διαμόρφωσής σας.

Η γεννήτρια υποστηρίζει:
- Πολλαπλές επιλογές παρόχου API (Zhipu AI, Anthropic Official, Custom API)
- Αυτόματη διαμόρφωση διεύθυνσης URL και διακριτικού API
- Προσαρμοσμένες θύρες, ονόματα κοντέινερ και άλλες βασικές ρυθμίσεις
- Διαμορφώστε μόνιμα δεδομένα και διαδρομές καταλόγου εργασίας
- Αυτόματη διαμόρφωση διαφορών πλατφόρμας Windows/Linux
- Χειριστείτε έξυπνα ζητήματα αδειών αρχείων
- **Επιλογή πηγής εικόνας**: Με βάση το περιβάλλον του δικτύου σας, επιλέξτε την κατάλληλη πηγή εικόνας

[🚀 Χρησιμοποιήστε αμέσως τη γεννήτρια →](https://builder.hagicode.com/)
:::

1. Ανοίξτε [Docker Compose Builder (αυτόνομος ιστότοπος)](https://builder.hagicode.com/)
2. Συμπληρώστε τη διαμόρφωση σύμφωνα με τις ανάγκες σας:
   - Επιλέξτε πάροχο API (Zhipu AI, Anthropic Official ή Custom API)
   - Διαμορφώστε τις θύρες, τη βάση δεδομένων, τον κατάλογο εργασίας και άλλες επιλογές
   - **Επιλογή πηγής εικόνας**: Επιλέξτε την κατάλληλη πηγή εικόνας με βάση το περιβάλλον του δικτύου σας
3. Κάντε κλικ στο κουμπί δημιουργίας για να λάβετε το `docker-compose.yml` διαμόρφωσης
4. Αποθηκεύστε τη διαμόρφωση που δημιουργήθηκε ως α `docker-compose.yml` αρχείο
5. Εάν χρειάζεται, δημιουργήστε ένα `.env` αρχείο για τη διαμόρφωση ευαίσθητων πληροφοριών

#### Επιλογή πηγής εικόνας

Η γεννήτρια υποστηρίζει τρεις επιλογές πηγής εικόνας:

**Docker Hub**
- **Διεύθυνση εικόνας**: `newbe36524/hagicode:{tag}`
- **Εφαρμοσμένα σενάρια**: Κατάλληλο για χρήστες με επιτάχυνση καθρέφτη Docker Hub
- **Πλεονεκτήματα**: Επίσημη πηγή εικόνας, έγκαιρες ενημερώσεις, σταθερή πρόσβαση
- **Σημειώσεις**: Η επιτάχυνση καθρέφτη μπορεί να χρειαστεί να διαμορφωθεί σε ορισμένες περιοχές

**Azure Container Registry (ACR)**
- **Διεύθυνση εικόνας**: `hagicode.azurecr.io/hagicode:{tag}`
- **Εφαρμοσμένα σενάρια**: Κατάλληλο για χρήστες που δεν μπορούν να έχουν τοπική πρόσβαση στο Docker Hub
- **Πλεονεκτήματα**: Παρέχει εναλλακτική πηγή εικόνας, επιλύει προβλήματα πρόσβασης στο δίκτυο
- **Σημειώσεις**: Οι εικόνες παραμένουν συγχρονισμένες με το Docker Hub, αλλά ενδέχεται να έχουν σύντομες καθυστερήσεις

**Alibaba Cloud Container Registry (ACR)**
- **Διεύθυνση εικόνας**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **Εφαρμοσμένα σενάρια**: Κατάλληλο για οικιακούς χρήστες, σταθερή και γρήγορη πρόσβαση στο δίκτυο
- **Πλεονεκτήματα**: Εγχώριοι κόμβοι Alibaba Cloud, γρήγορη ταχύτητα πρόσβασης, υψηλή σταθερότητα
- **Σημειώσεις**: Οι εικόνες Alibaba ACR παραμένουν συγχρονισμένες με το Docker Hub, συνήθως εντός 30 λεπτών

:::tip[Προτάσεις επιλογής πηγής εικόνας]
- **Οι εγχώριοι χρήστες προτείνουν το Alibaba Cloud ACR**: Εάν είστε εγχώριος χρήστης, συνιστάται να δώσετε προτεραιότητα στη χρήση της πηγής εικόνας Alibaba Cloud ACR για γρήγορη και σταθερή πρόσβαση
- **Alternative Docker Hub**: Εάν το περιβάλλον του δικτύου σας μπορεί να έχει πρόσβαση στο Docker Hub, μπορείτε επίσης να επιλέξετε Docker Hub ως πηγή εικόνας
- **Εναλλακτικό Azure ACR**: Επιλέξτε μόνο πηγή εικόνας Azure ACR εάν άλλες πηγές εικόνας έχουν δυσκολίες πρόσβασης
- **Συγχρονισμός εικόνας**: Οι εικόνες Alibaba Cloud ACR παραμένουν συγχρονισμένες με το Docker Hub, συνήθως μέσα σε 30 λεπτά
- **Συνέπεια έκδοσης**: Οι ετικέτες εικόνας είναι συνεπείς και στις τρεις πηγές εικόνας, δεν χρειάζεται να ανησυχείτε για προβλήματα έκδοσης
:::

:::note[Λήψη διακριτικού API]
Πρέπει να ρυθμίσετε το Claude API Token για να χρησιμοποιήσετε το Hagicode:

**Zhipu AI (Συνιστάται)**: [Λήψη διακριτικού API →](https://www.bigmodel.cn/glm-coding?ic=14BY54APZA)
- Σταθερή οικιακή πρόσβαση, ταχύτερη απόκριση
- Υψηλή απόδοση κόστους, κατάλληλο για καθημερινή χρήση

**Anthropic Επίσημος**: [Λήψη διακριτικού API →](https://console.anthropic.com/)
- Άμεση χρήση των υπηρεσιών του Anthropic

:::

### 2. Ξεκινήστε τις Υπηρεσίες

Εκτελέστε την ακόλουθη εντολή στον κατάλογο όπου βρίσκεστε `docker-compose.yml` βρίσκεται:

```bash
docker compose up -d
```

Αυτή η εντολή θα:
- Κατεβάστε και ξεκινήστε τα κοντέινερ Hagicode που ορίζονται από το πρότυπο που δημιουργείται
- Δημιουργήστε και διαμορφώστε το δίκτυο
- Αρχικοποιήστε τον προεπιλεγμένο κατάλογο δεδομένων και τις εξαρτήσεις χρόνου εκτέλεσης

### 3. Επαληθεύστε την κατάσταση υπηρεσίας

Ελέγξτε εάν τα κοντέινερ λειτουργούν:

```bash
docker compose ps
```

Θα πρέπει να δείτε τις υπηρεσίες που ορίζονται στο πρότυπο που δημιουργήσατε σε κατάσταση "εκτέλεση".

### Προβολή αρχείων καταγραφής υπηρεσιών

Για να δείτε όλα τα αρχεία καταγραφής υπηρεσιών:

```bash
# View all service logs
docker compose logs

# View Hagicode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## Οδηγίες διαμόρφωσης

Εάν χρειάζεται να τροποποιήσετε τη διαμόρφωση (όπως αλλαγή θυρών, αλλαγή παρόχου API κ.λπ.), μπορείτε:

1. Αναδημιουργήστε νέα διαμόρφωση χρησιμοποιώντας [Docker Compose Builder (αυτόνομος ιστότοπος)](https://builder.hagicode.com/)
2. Μη αυτόματη επεξεργασία `docker-compose.yml` και `.env` αρχεία
3. Επανεκκινήστε τις υπηρεσίες για να τεθεί σε ισχύ η διαμόρφωση:

```bash
docker compose restart
```

Η υποστηριζόμενη διαδρομή Docker Compose χρησιμοποιεί αρχεία δεδομένων SQLite με μόνιμους τόμους από προεπιλογή. Εάν χρειάζεται να επιθεωρήσετε σημεία προσάρτησης ή να σχεδιάσετε αντίγραφα ασφαλείας, ελέγξτε τις αντιστοιχίσεις τόμου και καταλόγου δεδομένων στο δημιουργημένο `docker-compose.yml`.
