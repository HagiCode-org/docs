Το HagiCode Desktop αποστέλλεται με μια αλυσίδα εργαλείων Node που διαχειρίζεται επιφάνεια εργασίας. Δεν αναλαμβάνει την καθολική εγκατάσταση του Node στον υπολογιστή σας. Σκοπός του είναι να δώσει στην επιφάνεια εργασίας εκκίνησης και σε ροές που σχετίζονται με το χρόνο εκτέλεσης μια σταθερή και επαληθεύσιμη βάση χρόνου εκτέλεσης.

Διαβάστε αυτό το FAQ αν δείτε `portable-fixed/toolchain` στα διαγνωστικά, πρέπει να γνωρίζουμε ποια `node` Η επιφάνεια εργασίας χρησιμοποιεί ή αναρωτιέμαι γιατί η έκδοση Node του συστήματός σας δεν καθορίζει άμεσα τη συμπεριφορά χρόνου εκτέλεσης της επιφάνειας εργασίας.

## Γιατί το Desktop δεσμεύει το Node

Η επιφάνεια εργασίας πρέπει να εκτελεί τις ίδιες ροές υποστήριξης σε λειτουργικά συστήματα, κανάλια εγκατάστασης και περιβάλλοντα χρήστη. Εάν εξαρτιόταν μόνο από την υπάρχουσα εγκατάσταση του κόμβου του χρήστη, η υποστήριξη θα έπρεπε να χειρίζεται ταυτόχρονα παλιές εκδόσεις, npm που λείπουν, λανθασμένη σειρά διαδρομής, μολυσμένα παγκόσμια πακέτα και πολιτικές μητρώου για συγκεκριμένο οργανισμό.

Το Desktop περιγράφει αυτό πιο συγκεκριμένα ως έναν απομονωμένο χρόνο εκτέλεσης κόμβου/npm για διαχειριζόμενα CLI αντί για επαναχρησιμοποίηση του περιβάλλοντος σε όλο το σύστημα. Αυτός ο σχεδιασμός έχει τα εξής κύρια πλεονεκτήματα:

- **Συνεπείς εκδόσεις**: Το Desktop χρησιμοποιεί ένα καρφιτσωμένο συμβόλαιο χρόνου εκτέλεσης Node 22 αντί να μαντεύει την τοπική κύρια έκδοση του χρήστη.
- **Ελεγχόμενες ροές εκκίνησης**: Οι εντολές που διαχειρίζονται από επιφάνεια εργασίας μπορούν να προτιμούν το πακέτο `node` και `npm` καταχωρήσεις.
- **Εκκαθάριση ορίων διαμόρφωσης**: Η διαμόρφωση που σχετίζεται με το CLI διατηρείται χωριστά από την καθολική διαμόρφωση σε όλο τον κεντρικό υπολογιστή, επομένως οι ενημερώσεις, η εναλλαγή και η ανάκτηση είναι λιγότερο πιθανό να διαταραχθούν από διαφορές στο περιβάλλον του συστήματος.
- **Μειωμένες διενέξεις**: Οι ομαδοποιημένες διαδρομές ισχύουν για περιβάλλοντα εντολών που δημιουργούνται από την επιφάνεια εργασίας και δεν αντικαθιστούν τον καθολικό κόμβο του χρήστη στο τερματικό του.
- **Μη παρεμβατική συμπεριφορά κεντρικού υπολογιστή**: Αυτή η απομόνωση αποφεύγει την εγγραφή της κατάστασης αλυσίδας εργαλείων στο υπάρχον περιβάλλον Node/npm του χρήστη και η απεγκατάσταση του Desktop δεν διαταράσσει τον κόμβο, το npm ή τη διαμόρφωση που υπάρχει ήδη στο μηχάνημα.

## Πώς το Desktop επιλέγει τη διαδρομή του κόμβου

Όταν εκτελείτε κανονικά το Desktop, το Desktop προτιμά την αλυσίδα εργαλείων Node που είναι εγκατεστημένη με την εφαρμογή αντί να διαβάζει πρώτα την εγκατάσταση του Node από το σύστημά σας. Αυτό διατηρεί την έκδοση Node στο τερματικό σας ξεχωριστά από την έκδοση Node που αναμένει το Desktop κατά την εκτέλεση.

Όταν η ομαδοποιημένη αλυσίδα εργαλείων είναι ενεργοποιημένη, τα περιβάλλοντα εντολών που δημιουργούνται από την επιφάνεια εργασίας τοποθετούν ομαδοποιημένες διαδρομές αλυσίδας εργαλείων στο μπροστινό μέρος του `PATH`. Ως αποτέλεσμα, ακόμα κι αν το σύστημά σας έχει εγκατεστημένη άλλη έκδοση Node, οι ροές που διαχειρίζονται από την επιφάνεια εργασίας επιλύονται πρώτα στον ομαδοποιημένο Node και npm του Desktop.

Αυτό επηρεάζει μόνο περιβάλλοντα χρόνου εκτέλεσης που δημιουργούνται από την επιφάνεια εργασίας. Αν τρέξεις `node`, `npm`, ή άλλες εντολές απευθείας στο τερματικό του συστήματός σας, συνεχίζουν να χρησιμοποιούν το κανονικό περιβάλλον του συστήματος και τη διαμόρφωση του κελύφους.

Σε περιβάλλοντα διαγνωστικών ή θυγατρικών διεργασιών που ξεκινούν από το Desktop, ενδέχεται να δείτε μια μεταβλητή όπως αυτή:

```bash
HAGICODE_PORTABLE_TOOLCHAIN_ROOT=/path/to/portable-fixed/toolchain
```

Αυτή η μεταβλητή σημαίνει ότι το περιβάλλον χρόνου εκτέλεσης που διαχειρίζεται η επιφάνεια εργασίας αναγνώρισε τη συνδυασμένη ρίζα της αλυσίδας εργαλείων. Η εμφάνιση αυτής της διαδρομής δεν σημαίνει ότι η επιφάνεια εργασίας άλλαξε τον κόμβο του συστήματός σας. σημαίνει μόνο ότι το Desktop χρησιμοποιεί το περιβάλλον Node που αποστέλλεται με την εφαρμογή μέσα στο δικό του εύρος χρόνου εκτέλεσης.

## Όπου η επιφάνεια εργασίας εγκαθιστά διαχειριζόμενα πακέτα npm

Το Desktop διατηρεί αρχεία χρόνου εκτέλεσης και μεταβλητά αρχεία πακέτων npm σε διαφορετικές τοποθεσίες. Η ομαδοποιημένη διαδρομή αλυσίδας εργαλείων όπως `portable-fixed/toolchain/node` είναι η αμετάβλητη πηγή για τη διαχείριση της επιφάνειας εργασίας `node` και `npm` εντολές. Τα διαχειριζόμενα καθολικά πακέτα εγκαθίστανται στον κατάλογο δεδομένων χρήστη Electron:

```text
userData/node22/npmGlobal
```

Εάν η επιφάνεια εργασίας μετακινηθεί αργότερα στον Κόμβο 24, χρησιμοποιεί ένα ξεχωριστό δέντρο πακέτων, όπως π.χ `userData/node24/npmGlobal`. Τα πακέτα που έχουν εγκατασταθεί για τον Κόμβο 22 παραμένουν στο δίσκο, αλλά δεν αντιμετωπίζονται ως εγκατεστημένα για τον Κόμβο 24 έως ότου εγκατασταθούν ξανά στο νέο πρόθεμα Node-major.

Η κατάσταση εξάρτησης επιφάνειας εργασίας ελέγχει μόνο το ενεργό `userData/node<major>/npmGlobal` πρόθεμα. Τα αρχεία πακέτων εκτός αυτού του προθέματος αγνοούνται από τη διαχείριση εξαρτήσεων.
