Ce guide couvre l'installation et l'utilisation de HagiCode Desktop. Il s'agit du client de bureau officiel et conserve le chemin de configuration local par défaut ainsi que les contrôles quotidiens en un seul endroit.

:::tip[Méthode recommandée]
Desktop est le chemin local par défaut pour les utilisateurs individuels et les développeurs car :
- le programme d'installation conserve le flux par défaut dans un chemin guidé
- les dépendances et les services sont gérés dans l'application
- l'interface utilisateur facilite l'inspection de l'état et des opérations
- la gestion des versions intégrée facilite le changement
- les données restent locales par défaut
:::

## Qu’est-ce que HagiCode Desktop ?

HagiCode Desktop est l'application de bureau officielle de HagiCode, conçue pour les systèmes Windows, macOS et Linux. Il regroupe les fonctionnalités principales de HagiCode et le programme d'installation conserve le chemin de configuration par défaut dans un flux guidé.

### Principaux avantages

- **Installation en un clic** : effectuez automatiquement toutes les étapes de configuration via l'assistant d'installation.
- **Préparation automatique du runtime** : le programme d'installation par défaut inclut déjà le runtime dont il a besoin, vous n'avez donc pas besoin d'installer un autre runtime. `.NET` l'exécution en premier
- **Gestion des services** : gestion intégrée du serveur HagiCode, démarrage et arrêt faciles des services
- **Changement de version** : prise en charge de la gestion multi-versions, basculez librement entre les différentes versions
- **Opération locale** : toutes les données sont stockées localement, protégeant ainsi votre vie privée.
- **Mises à jour automatiques** : détecte et télécharge automatiquement la dernière version

### Scénarios applicables

L'ordinateur de bureau est un bon choix lorsque :
- Développeurs individuels souhaitant utiliser HagiCode localement
- Utilisateurs qui doivent utiliser HagiCode hors ligne
- Utilisateurs non techniques souhaitant simplifier le processus d'installation
- Développeurs qui doivent fréquemment basculer entre différentes versions

### Autres méthodes de téléchargement

Vous pouvez également obtenir le programme d'installation de HagiCode Desktop à partir des canaux suivants :

1. **Page de sortie officielle** : Visitez [Publications de HagiCode GitHub](https://github.com/HagiCode-org/releases) pour télécharger la dernière version
2. **Site officiel** : Visitez [Site Web HagiCode](https://hagicode.com/) pour obtenir des liens de téléchargement

### Choisissez la bonne version

Sélectionnez le programme d'installation correspondant à votre système d'exploitation :
- **Windows** : Téléchargez le `.exe` installateur
- **macOS** : téléchargez le `.dmg` installateur
- **Linux** : Téléchargez le `.AppImage` ou `.tar.gz` package d'installation

:::note[Configuration système requise]
- **Windows** : Windows 10 ou version supérieure (64 bits)
- **macOS** : macOS 10.15 ou version ultérieure
- **Linux** : distributions Linux grand public (Ubuntu 20.04+, Fedora 33+, etc.)
:::

:::tip[Conditions préalables par défaut]
Pour le dernier flux d'installation local de HagiCode, vous n'avez généralement pas besoin de préparer un fichier séparé. `.NET` exécution. Desktop regroupe déjà le runtime requis pour le flux par défaut et initialise par défaut le chemin SQLite local pris en charge.
:::

## Notes d'installation et de lancement

:::caution[Avis du Gatekeeper macOS]
Si vous avez déjà installé l'application sur le chemin par défaut `/Applications/HagiCode.app` mais macOS affiche "l'application est endommagée", "ne peut pas être ouverte" ou un avertissement Gatekeeper similaire au premier lancement, la cause habituelle est le `com.apple.quarantine` attribut que macOS ajoute aux applications téléchargées sur Internet, et non un échec dans le programme d'installation de HagiCode Desktop lui-même.

Exécutez la commande suivante dans Terminal. Il nécessite des privilèges d'administrateur et supprime de manière récursive l'attribut de quarantaine de l'application dans le chemin d'installation par défaut :

```bash
sudo xattr -dr com.apple.quarantine /Applications/HagiCode.app
```

Une fois la commande terminée, relancez HagiCode Desktop.
:::

## Exécuter le bureau

Après le téléchargement, ouvrez le programme d'installation ou le package d'application et suivez les instructions à l'écran. Desktop prépare le runtime dans le chemin local actuellement pris en charge, gère le serveur HagiCode et affiche la progression et l'état lorsque des téléchargements réseau sont requis.

:::note[Configuration réseau requise]
Assurez-vous que votre connexion réseau est disponible. Desktop peut avoir besoin de récupérer des informations de version ou des packages de composants sur Internet ; si un téléchargement échoue, vérifiez les paramètres de votre réseau, proxy ou pare-feu et réessayez.
:::

## Gestion des versions

HagiCode Desktop prend en charge la gestion multi-versions, vous permettant de basculer librement entre différentes versions.

![Gestion des versions](../../img/screenshots/installation/install-desktop/12-version-management/original.png)

### Changer de version

1. Ouvrez le bureau HagiCode
2. Allez sur la page "Gestion des versions"
3. Sélectionnez la version que vous souhaitez changer
4. Cliquez sur le bouton « Changer de version »
5. Attendez que le changement soit terminé

:::note[Notes sur le changement de version]
- Changer de version redémarrera automatiquement le serveur HagiCode
- Assurez-vous de sauvegarder tout votre travail avant de changer
- Les configurations du projet peuvent différer selon les versions
:::

## Démarrer et arrêter le service

HagiCode Desktop fournit un moyen simple de gérer le service HagiCode Server.

### Démarrer le service

Sur la page d'accueil de l'application de bureau, cliquez sur le bouton « Démarrer le service » pour démarrer HagiCode Server.

![Démarrer le service](../../img/screenshots/installation/install-desktop/13-start-service-homepage/original.png)

### Arrêter le service

Pour arrêter le serveur HagiCode, cliquez sur le bouton « Arrêter le service » sur la page d'accueil.

:::astuce[État du service]
L'application de bureau affiche l'état d'exécution du serveur HagiCode en temps réel, notamment :
- État du service (en cours d'exécution/arrêté)
- Numéro de port de service
- Informations sur la version actuelle
:::

## Dépannage

### Échec de l'installation

Si vous rencontrez des problèmes lors de l'installation :

1. **Vérifiez la connexion réseau** : assurez-vous que vous pouvez accéder à Internet
2. **Vérifiez l'espace disque** : assurez-vous que vous disposez de suffisamment d'espace disque
3. **Exécuter en tant qu'administrateur** : sous Windows, cliquez avec le bouton droit sur le programme d'installation et sélectionnez "Exécuter en tant qu'administrateur".
4. **Afficher les fichiers journaux** : le programme d'installation génère des fichiers journaux détaillés qui peuvent aider à diagnostiquer les problèmes.

### Échec de l'installation des dépendances

Si l'installation des dépendances échoue :

1. **Vérifiez la compatibilité du système** : assurez-vous que votre système d'exploitation répond aux exigences minimales
2. **Vérifiez les paramètres réseau et pare-feu** : assurez-vous que le programme d'installation peut accéder aux téléchargements requis et qu'il n'est pas bloqué par votre pare-feu.
3. **Confirmer les autorisations du programme d'installation** : sous Windows, essayez d'exécuter le programme d'installation en tant qu'administrateur afin qu'il dispose des autorisations suffisantes.
4. **Redémarrez l'application et réessayez** : fermez et rouvrez HagiCode Desktop, puis réessayez l'étape d'installation.
5. **Afficher les journaux** : consultez les journaux d'erreurs détaillés dans l'application de bureau.

### Échec du démarrage du service

Si le serveur HagiCode ne démarre pas :

1. **Vérifiez l'occupation du port** : assurez-vous que le port 45000 n'est pas occupé par d'autres programmes.
2. **Vérifiez les paramètres du pare-feu** : assurez-vous que le pare-feu autorise l'accès au réseau du serveur HagiCode
3. **Redémarrer l'application** : essayez de redémarrer HagiCode Desktop
4. **Afficher les journaux** : affichez les journaux d'erreurs détaillés dans l'application de bureau.
