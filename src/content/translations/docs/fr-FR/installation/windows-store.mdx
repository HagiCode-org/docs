Pour les utilisateurs Windows, Windows Store est actuellement le point d’entrée public du client desktop. Cette page couvre uniquement ce chemin Store. Pour le guide général de téléchargement desktop, de gestion des versions et de dépannage, continuez avec [Desktop Installation](/fr-FR/installation/desktop).

L’entrée principale Steam a déjà été retirée. Windows Store est donc désormais le canal principal côté Windows. Pour le contexte, consultez [Why HagiCode is not available on Steam](/fr-FR/faq/steam-distribution-status/).

:::tip[Ce chemin convient bien si]
- vous utilisez Windows 10 ou Windows 11 et souhaitez installer directement depuis le store système
- vous préférez recevoir le paquet applicatif via le Store plutôt que manipuler des fichiers d’installation manuellement
- vous avez seulement besoin d’un environnement local personnel et ne prévoyez pas de déploiement Docker Compose
:::

## Avant de commencer

- votre appareil doit pouvoir ouvrir la [page Windows Store de HagiCode for Windows](https://apps.microsoft.com/detail/9N3PM0N3SVDW)
- un accès réseau doit être disponible au premier lancement, car l’application peut encore préparer des composants supplémentaires
- selon l’appareil, la région ou la politique d’organisation, une connexion avec un compte Microsoft peut être demandée
- si le Store est bloqué sur votre machine, basculez vers [Desktop Installation](/fr-FR/installation/desktop)

## Installer depuis Windows Store

1. Ouvrez [HagiCode for Windows](https://apps.microsoft.com/detail/9N3PM0N3SVDW).
2. Cliquez sur `Obtenir`, `Installer`, ou l’action équivalente affichée par le Store.
3. Attendez que Windows Store termine le téléchargement et l’installation.
4. Lancez ensuite HagiCode depuis le menu Démarrer ou directement depuis la page Store.

## Ce qui se passe au premier lancement

Après l’installation, HagiCode continue quand même le flux standard d’initialisation Desktop.

- le premier lancement peut sembler plus lent, car l’environnement d’exécution local doit encore être préparé
- le canal Windows Store / MSIX verrouille volontairement certains réglages de gestion des dépendances et de chemins runtime
- l’application ne se met pas à jour seule de manière indépendante ; la livraison du paquet dépend du canal d’installation, et la distribution Store passe par le canal Windows Store

Pour poursuivre l’onboarding, consultez [Wizard Setup](/fr-FR/quick-start/wizard-setup).

## Quand choisir un autre canal

- vous ne pouvez pas accéder à Windows Store, ou votre organisation interdit les installations via le Store
- vous devez télécharger manuellement des fichiers `.exe` ou `.msix` et contrôler vous-même la source de l’installeur
- vous déployez sur serveur, NAS ou environnement d’équipe ; dans ce cas [Docker Compose Deployment](/fr-FR/installation/docker-compose) est plus adapté

## Dépannage

### La page Windows Store ne s’ouvre pas

Vérifiez d’abord le réseau, les paramètres de région et l’état de connexion à Microsoft Store. Si le lien navigateur s’ouvre mais que le client Store reste indisponible, la cause est souvent une politique système locale. Dans ce cas, utilisez [Desktop Installation](/fr-FR/installation/desktop).

### Certains réglages sont verrouillés

Les installations Store / MSIX verrouillent volontairement certaines options de gestion des dépendances et de chemins de données runtime. Ce n’est pas un échec d’installation, mais une contrainte normale de ce canal de distribution.

### Le premier lancement télécharge ou prépare encore du contenu

Le paquet Store ne livre que le paquet de l’application desktop. Au premier lancement, HagiCode peut encore vérifier des métadonnées de version, préparer des composants locaux ou poursuivre l’initialisation par défaut. Gardez le réseau disponible et laissez l’étape en cours se terminer.
