Cette page est l'entrée d'installation CLI locale unifiée de l'agent AI dans `repos/docs`. HagiCode prend actuellement en charge 11 CLI d'agent actives, suivant le backend `PrimaryProfessions` commande : Claude Code, Codex, GitHub Copilot, OpenCode, Hermes, QoderCLI, Kiro, Kimi, Gemini, DeepAgents et Codebuddy.

:::tip[Comment utiliser cette page]
Vérifiez d'abord le tableau pour connaître l'état actuel de la prise en charge et le mode de configuration, puis poursuivez avec les documents d'installation, d'authentification, d'ACP, d'autorisation et de dépannage gérés par le fournisseur. La portée du support et le mode de configuration sont des préoccupations distinctes.
:::

## Glossaire d'État

- **Actuellement pris en charge** signifie que la CLI appartient à la plage de prise en charge active actuelle de HagiCode.
- **Installation en un clic** signifie que HagiCode fournit un point d'entrée d'installation guidée ; certaines CLI nécessitent toujours une connexion du fournisseur ou une étape d'authentification après l'installation.
- **Installation/authentification manuelle** signifie que la CLI est désormais prise en charge, mais vous devez terminer l'installation locale ou le flux d'authentification avant que HagiCode puisse le détecter et l'utiliser.
- **Compatibilité héritée uniquement** signifie une valeur retirée telle que `IFlowCli` existe toujours pour la compatibilité des données historiques, mais il ne fait pas partie de la plage de support actif actuelle.
