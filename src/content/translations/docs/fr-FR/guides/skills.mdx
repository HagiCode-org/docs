Ce guide couvre les `Skills` panneau dans HagiCode.

Ouvrez Skills lorsque vous souhaitez :

- voir quelles compétences sont déjà installées sur cette machine
- recherchez dans la bibliothèque de compétences plus large
- confirmer la source, l'état de confiance et la commande d'installation réelle avant l'installation
- gérer quelles sources sont étiquetées comme fiables

## À quoi servent les compétences

Skills est le panneau de navigation et d'installation de compétences intégré à HagiCode. Il vous offre un seul endroit pour examiner les compétences disponibles localement, rechercher dans la galerie et inspecter les détails d'installation avant d'installer quoi que ce soit.

Utilisez-le quand :

- vous êtes nouveau sur HagiCode et souhaitez voir ce qui est déjà disponible
- vous connaissez le type de travail que vous effectuez et souhaitez des compétences d'aide mieux adaptées
- vous souhaitez installer une nouvelle compétence mais devez d'abord vérifier la source
- vous voulez éviter de réinstaller quelque chose qui est déjà disponible localement

:::astuce[Ordre de lecture recommandé]
Lors d'un premier passage, commencez par `Local Skills`, puis passez à `Gallery`, puis examinez ensuite le panneau de détails avant l'installation.
:::

## 1. Compétences ouvertes

Dans la barre de navigation supérieure de l'espace de travail HagiCode, cliquez sur le bouton `Skills` bouton pour ouvrir le panneau.

![Navigation supérieure HagiCode avec l'entrée Compétences](../../img/screenshots/skills/entry-open-skills-from-top-navigation/original.png)

La flèche rouge dans la capture d'écran met en évidence le point d'entrée.

Après avoir ouvert le panneau, vous verrez trois onglets principaux :

- `Local Skills`
- `Gallery`
- `Trusted Providers`

:::note[Quand ouvrir les compétences]
Vous n'avez pas besoin de quitter votre projet, session ou flux d'intégration actuel. Les compétences s'ouvrent à partir de la barre supérieure dans le même espace de travail.
:::

## 2. Examinez les compétences installées localement

`Local Skills` répond à deux questions courantes : **Qu'est-ce qui est déjà installé ?** et **Laquelle de ces compétences peut être mise à jour dès maintenant ?**

![Onglet Compétences locales avec l'inventaire des compétences installées et les actions par lots](../../img/screenshots/shared/local-skills-view/original.png)

Les domaines les plus importants sur cette page sont :

- **Champ de recherche** : filtrez les compétences locales par nom, ID, chemin ou agent
- **`Refresh inventory`** : analysez à nouveau ce qui est actuellement installé sur cette machine
- **`Update all`** : exécutez des mises à jour par lots pour les compétences prenant en charge les mises à jour gérées.
- **`Batch update summary`** : consultez le dernier résultat du lot et le journal d'état

Chaque ligne affiche généralement :

- le nom de la compétence
- agents compatibles
- le chemin d'installation local
- actions disponibles

Si une ligne avertit que les métadonnées d'installation gérées par le backend sont manquantes, cette compétence ne peut pas utiliser les mises à jour en un clic et doit être vérifiée manuellement.

:::caution[La mise à jour par lots nécessite encore un examen humain]
`Update all` signifie uniquement que le flux de mise à jour est exécuté en masse. Cela ne signifie pas que les compétences répertoriées ont déjà été examinées en termes de sécurité pour votre cas d'utilisation.
:::

## 3. Recherchez et comparez dans la galerie

Si vous savez déjà ce que vous recherchez ou si vous souhaitez comparer des compétences similaires provenant de différentes sources, utilisez `Gallery`.

![Onglet Galerie avec résultats de recherche, filtres sources et nombre d'installations](../../img/screenshots/shared/skills-gallary/original.png)

`Gallery` est le meilleur endroit pour :

- recherche par mot clé
- filtrer d'abord vers des sources fiables
- comparer des compétences similaires provenant de différentes sources
- vérifier le nombre d'installations et l'état de l'installation locale

Commencez avec trois champs :

- **`Source`** : qui fournit la compétence
- **`Installs`** : le nombre d'installations, qui peut servir de signal de support
- **`Installed locally`** : si la compétence est déjà disponible sur votre machine

:::tip[Vérifiez la source avant le nombre d'installations]
La popularité peut aider, mais elle ne doit pas remplacer l’évaluation des sources. Commencez par `Source`, puis utilisez `Installs` comme contexte supplémentaire.
:::

## 4. Vérifiez les détails des compétences avant d'installer

Lorsque vous ouvrez une compétence, HagiCode affiche le panneau de détails. Il s’agit de l’écran le plus important à vérifier avant l’installation.

![Panneau de détails des compétences avec statut de confiance et commande gérée](../../img/screenshots/skills/skill-details-trust-status-and-managed-command/original.png)

Portez une attention particulière à :

- **nom, source et nombre d'installations** : assurez-vous d'avoir ouvert l'élément souhaité
- **`Trust status`** : pourquoi cette source est considérée comme fiable
- **`Managed command`** : la commande actuelle HagiCode est prête à être exécutée
- **`Status`** : indique si l'espace de travail actuel peut installer cette compétence
- **`Overview`** : identifiants pris en charge tels que l'ID de compétence, le slug, la source et le nombre d'installations

`Managed command` est important car il s'agit de la véritable commande d'installation, pas d'un texte décoratif.

:::attention[`trusted` ne veut pas dire "ignorer la révision"]
Même lorsqu'une source est marquée `trusted`, tu devrais quand même lire les deux `Trust status` et `Managed command` avant l'installation.
:::

## 5. Gérer les fournisseurs de confiance

`Trusted Providers` contrôle quelles sources reçoivent le `trusted` étiquette. Ouvrez-le lorsque vous souhaitez savoir pourquoi une source est fiable ou lorsque vous devez ajuster vos règles de confiance.

![Onglet Fournisseurs de confiance avec règles du fournisseur et statut de confiance](../../img/screenshots/skills/trusted-providers-rule-list/original.png)

Sur cette page :

- **`Rule format`** explique comment les règles sont écrites
- **`Add provider`** vous permet d'ajouter un ensemble de règles de fournisseur
- **`Reset defaults`** restaure les préréglages par défaut
- le **`Rules`**, **`Status`**, et **`Source examples`** Les colonnes affichent les règles de correspondance, l'état actuel et des exemples de correspondances

Les styles de règles courants incluent :

- `prefix:owner/` pour la confiance basée sur le préfixe
- `exact:owner/repo` pour une seule source exacte

:::note[Les fournisseurs de confiance ne sont pas une page d'informations d'identification]
Cette page décide uniquement quelles sources sont étiquetées comme fiables. Il ne stocke pas les jetons, mots de passe ou autres secrets.
:::

## Vérifiez ces 4 choses avant d'installer

Avant d'installer une compétence, examinez au moins ces quatre signaux :

| Vérifier | Où chercher | Ce que ça te dit |
| --- | --- | --- |
| Si la source est fiable | `Trusted Providers`, plus `Trust status` dans le panneau de détails | Pourquoi cette source est étiquetée comme fiable |
| Qu'il soit déjà installé | étiquettes d'état dans `Gallery` | Si ça dit `installed locally`, vous n'avez généralement pas besoin de le réinstaller |
| Résultat de la mise à jour par lots | `Batch update summary` dans `Local Skills` | Si la dernière mise à jour groupée a réussi et si quelque chose nécessite une révision manuelle |
| Commande d'installation réelle | `Managed command` dans le panneau de détails | Ce que HagiCode va réellement exécuter |

## Un parcours utilisateur commun

Un flux courant pour la première fois est :

1. Ouvert `Local Skills` et passer en revue ce qui est déjà disponible
2. Utiliser `Gallery` pour rechercher et comparer les sources
3. Ouvrez le panneau de détails pour vérifier l'état de confiance et installer la commande
4. Si nécessaire, examinez ou ajustez les règles de confiance dans `Trusted Providers`
