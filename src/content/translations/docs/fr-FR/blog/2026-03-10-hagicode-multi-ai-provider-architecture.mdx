## Architecture multi-fournisseur IA pratique dans la plateforme HagiCode

> Cet article partage l'approche technique que nous avons utilisée sous l'architecture Orleans Grain pour intégrer deux outils IA, iflow et OpenCode, via une interface `IAIProvider` unifiée, et compare en détail les différences d'implémentation entre la communication WebSocket et HTTP.

## Contexte

Il n'y a rien de particulièrement mystérieux là-dedans. Lors du développement de HagiCode, nous avons rencontré un problème très pratique : les utilisateurs voulaient travailler avec différents outils IA. Ce n'est guère surprenant, puisque chacun a ses propres habitudes. Certains préfèrent Claude Code, d'autres aiment GitHub Copilot, et certaines équipes utilisent des outils qu'elles ont développés eux-mêmes.

Notre solution initiale était simple et directe : écrire du code d'intégration dédié pour chaque outil IA. Mais les inconvénients sont apparus rapidement. La base de code s'est remplie de branches `if-else`, chaque modification nécessitait des tests à plusieurs endroits, et chaque nouvel outil signifiait écrire encore une pile de logique à partir de zéro.

Plus tard, j'ai réalisé qu'il serait préférable de créer une interface `IAIProvider` unifiée et d'abstraire les capacités partagées par tous les fournisseurs IA. De cette façon, quel que soit l'outil utilisé en dessous, le reste du système n'aurait pas besoin de s'en soucier.

## À propos de HagiCode

La solution partagée dans cet article provient de notre expérience pratique dans le projet [HagiCode](https://hagicode.com). HagiCode est un assistant de codage IA open source qui prend en charge plusieurs fournisseurs IA. L'architecture multi-fournisseur présentée ici est l'un de ses piliers fondamentaux.

## Conception de l'interface IAIProvider

### Définition de l'interface

L'interface `IAIProvider` est le cœur de toute l'architecture. Elle définit les capacités communes que tout fournisseur IA doit implémenter :

```csharp
public interface IAIProvider
{
    /// <summary>
    /// Identifiant unique du fournisseur
    /// </summary>
    string ProviderId { get; }
    
    /// <summary>
    /// Nom d'affichage du fournisseur
    /// </summary>
    string DisplayName { get; }
    
    /// <summary>
    /// Vérifier si le fournisseur est disponible
    /// </summary>
    Task<bool> IsAvailableAsync(CancellationToken ct = default);
    
    /// <summary>
    /// Exécuter une tâche IA
    /// </summary>
    Task<AIExecutionResult> ExecuteAsync(
        AIExecutionRequest request,
        IProgress<AIExecutionProgress>? progress = null,
        CancellationToken ct = default);
    
    /// <summary>
    /// Annuler une tâche en cours
    /// </summary>
    Task CancelAsync(string executionId, CancellationToken ct = default);
}
```

### Avantages de cette conception

1. **Extensibilité** : ajouter un nouveau fournisseur ne nécessite que l'implémentation de l'interface, sans modifier le code existant.
2. **Testabilité** : chaque fournisseur peut être testé indépendamment avec un mock de l'interface.
3. **Remplacement à l'exécution** : les utilisateurs peuvent basculer entre les fournisseurs sans redémarrer l'application.

## Intégration Orleans Grain

### Grain de session IA

Dans l'architecture HagiCode, chaque session de codage IA est représentée par un Orleans Grain :

```csharp
public class AISessionGrain : Grain, IAISessionGrain
{
    private IAIProvider? _currentProvider;
    
    public override async Task OnActivateAsync(CancellationToken ct)
    {
        var providerId = this.GetPrimaryKeyString();
        _currentProvider = GrainFactory.GetGrain<IAIProviderRegistryGrain>("registry")
            .GetProvider(providerId);
    }
    
    public async Task<AIExecutionResult> ExecuteAsync(AIExecutionRequest request)
    {
        if (_currentProvider == null)
            throw new InvalidOperationException("No AI provider configured");
            
        return await _currentProvider.ExecuteAsync(request);
    }
}
```

### Registre de fournisseurs

Un Grain de registre gère les fournisseurs disponibles :

```csharp
public class AIProviderRegistryGrain : Grain, IAIProviderRegistryGrain
{
    private readonly Dictionary<string, IAIProvider> _providers = new();
    
    public Task RegisterProvider(IAIProvider provider)
    {
        _providers[provider.ProviderId] = provider;
        return Task.CompletedTask;
    }
    
    public Task<IAIProvider?> GetProvider(string providerId)
    {
        _providers.TryGetValue(providerId, out var provider);
        return Task.FromResult(provider);
    }
    
    public Task<IReadOnlyList<IAIProvider>> GetAllProviders()
    {
        return Task.FromResult<IReadOnlyList<IAIProvider>>(_providers.Values.ToList());
    }
}
```

## Comparaison WebSocket vs HTTP

### Communication WebSocket

Les fournisseurs basés sur WebSocket (comme iflow) maintiennent une connexion persistante :

Avantages :
- Communication bidirectionnelle en temps réel
- Latence plus faible pour les réponses en flux continu
- Moins de surcharge de connexion

Inconvénients :
- Complexité de gestion de la connexion (reconnexion, heartbeat)
- Plus difficile à mettre en cache et à équilibrer

### Communication HTTP

Les fournisseurs basés sur HTTP (comme OpenCode) utilisent des requêtes REST standard :

Avantages :
- Plus simple à implémenter et à déboguer
- Compatible avec les infrastructures existantes (proxy, CDN)
- Facile à mettre en cache

Inconvénients :
- Unidirectionnel (requête-réponse)
- Surcharge de connexion pour chaque requête

### Notre choix

HagiCode supporte les deux modes. Le choix du mode de communication dépend du fournisseur :

```csharp
public class WebSocketAIProvider : IAIProvider
{
    // Implémentation WebSocket
}

public class HttpAIProvider : IAIProvider
{
    // Implémentation HTTP
}
```

## Conclusion

L'architecture multi-fournisseur IA de HagiCode, basée sur l'interface `IAIProvider` et Orleans Grains, offre une solution flexible et extensible pour l'intégration de multiples outils IA. La séparation claire entre l'interface abstraite et les implémentations concrètes permet d'ajouter de nouveaux fournisseurs avec un effort minimal.
