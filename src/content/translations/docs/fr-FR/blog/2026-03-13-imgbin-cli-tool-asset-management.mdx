## Conception de l'outil CLI ImgBin : l'approche de gestion des assets image de HagiCode

> Cet article explique comment construire un pipeline d'assets image automatisable à partir de zéro, couvrant la conception d'outils CLI, une architecture de Provider Adapter et les stratégies de gestion des métadonnées.

## Contexte

Franchement, je ne m'attendais pas à ce que la gestion des assets image nous retienne pendant si longtemps. Dans le projet HagiCode, nous avions besoin d'un moyen de gérer les images de manière cohérente entre le site web, le client de bureau et la documentation. Chaque plateforme a ses propres exigences de taille, de format et d'optimisation, et les gérer manuellement n'était tout simplement pas viable à long terme.

Nous avons donc construit ImgBin, un outil CLI en TypeScript qui automatise l'ensemble du pipeline de traitement des images.

## À propos de HagiCode

L'approche partagée dans cet article provient de notre expérience pratique dans le projet [HagiCode](https://hagicode.com).

## Architecture

ImgBin suit une architecture de Provider Adapter : un cœur de traitement d'images indépendant, avec des adaptateurs pour différents fournisseurs de stockage (local, S3, Azure Blob, etc.).

## Conclusion

ImgBin a considérablement simplifié notre pipeline d'assets image. Si vous avez des besoins similaires en matière de gestion d'images, cette approche vaut la peine d'être explorée.
