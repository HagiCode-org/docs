## Contexte

Dans le projet HagiCode, les utilisateurs peuvent choisir parmi plusieurs outils CLI pour piloter des assistants de programmation IA, notamment Claude Code CLI, GitHub Copilot CLI, OpenCode CLI, et d'autres. Cet article présente la pratique d'intégration multi-CLI d'HagiCode.

## À propos de HagiCode

L'approche partagée dans cet article provient de notre expérience pratique dans le projet [HagiCode](https://hagicode.com). HagiCode est un assistant de codage IA open source qui prend en charge plusieurs outils CLI.

## Intégration multi-CLI

HagiCode fournit une couche d'abstraction unifiée qui permet de basculer entre différents outils CLI. Chaque CLI est intégré via un adaptateur spécifique qui gère les différences d'interface et de format de sortie.

## Support de GLM-5.1

L'intégration du modèle GLM-5.1 étend les capacités multi-modèles d'HagiCode, offrant aux utilisateurs un choix supplémentaire pour leurs tâches de codage.

## Conclusion

L'architecture multi-CLI d'HagiCode offre une flexibilité maximale aux utilisateurs, leur permettant de choisir le meilleur outil pour chaque tâche.
