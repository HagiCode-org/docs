Hvis du vil installere Hermes og begynne å bruke den, er den korteste veien egentlig bare tre trinn:

1. Kjør den offisielle installasjonskommandoen
2. Start CLI i terminalen din med `hermes`
3. Hvis du vil fortsette å bruke den i Feishu, konfigurer `hermes gateway setup`

Denne artikkelen prøver ikke å forklare alle Hermes-evner på en gang. I stedet hjelper den deg med å fullføre den viktigste nybegynnersløyfen først: installer den, få den til å kjøre, begynn å bruke den og koble den deretter til et av de vanligste scenariene for meldingsplattformer.

## Hva Hermes er og hvem det er for

Hermes Agent er en AI-agent som du kan bruke enten fra en lokal terminal eller gjennom en meldingsplattform-gateway.

For de fleste utviklere har den to vanlige inngangspunkter:

- **CLI**: Type `hermes` i terminalen for å gå direkte inn i det interaktive grensesnittet.
- **Meldingsgateway**: Kjør `hermes gateway`, og chat med den fra plattformer som Feishu, Telegram, Discord og Slack.

Hvis målet ditt akkurat nå er å komme raskt i gang, ikke reversere rekkefølgen. Start med denne banen i stedet:

- Installer Hermes først
- Bekreft at det fungerer fra CLI først
- Bestem deretter om du vil koble til en meldingsplattform

Dette gjør problemer lettere å diagnostisere og er mer egnet for personer som bruker Hermes for første gang.

## Hva du bør vite før du installerer Hermes

I følge Hermes README støtter den offisielle hurtiginstallasjonsbanen disse miljøene:

- Linux
- macOS
- WSL2
- Android via Termux

### Merknader for Windows-brukere

Hermes støtter for øyeblikket ikke kjøring direkte på opprinnelig Windows. Hvis du bruker Windows, er den anbefalte fremgangsmåten å installere WSL2 først og deretter kjøre installasjonskommandoen inne i WSL2.

Det er best å gjøre dette klart i begynnelsen, fordi mange installasjonsfeil ikke er forårsaket av kommandoen i seg selv, men ved å bruke et kjøretidsmiljø som ikke støttes.

## Hvordan installere Hermes

Hurtiginstallasjonskommandoen i Hermes README er:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Denne kommandoen kjører det offisielle installasjonsskriptet og håndterer plattformspesifikke initialiseringstrinn.

### Last inn skallet på nytt etter installasjon

Når installasjonen er ferdig, last inn skallmiljøet på nytt først. Den vanligste kommandoen er:

```bash
source ~/.bashrc
```

Hvis du bruker zsh, kan du bruke:

```bash
source ~/.zshrc
```

### Slik bekrefter du at Hermes er riktig installert

Den mest direkte måten å sjekke på er å kjøre:

```bash
hermes
```

Hvis du vil ha ytterligere bekreftelse på at konfigurasjonen og avhengighetene dine fungerer, kan du også kjøre:

```bash
hermes doctor
```

`hermes doctor` er spesielt nyttig i disse situasjonene:

- Kommandoen oppfører seg unormalt etter installasjon
- Modellkonfigurasjonen mislykkes
- Gatewayen starter ikke
- Du er ikke sikker på om miljøavhengighetene dine er fullstendige

## Hvordan begynne å bruke Hermes for første gang

### Start med CLI

Hvis du bare vil bekrefte så raskt som mulig at Hermes fungerer, er den enkleste metoden:

```bash
hermes
```

Dette lanserer den interaktive Hermes CLI. For førstegangsbrukere av Hermes er det også det mest anbefalte utgangspunktet, fordi du kan bekrefte de viktigste tingene først:

- Om kommandoen faktisk er tilgjengelig
- Om den gjeldende modellkonfigurasjonen fungerer som den skal
- Om terminalverktøykjeden fungerer som den skal
- Om interaksjonsstilen stemmer overens med det du trenger

### Disse kommandoene er nok for din første runde med oppsett

Hermes README lister opp flere høyfrekvente kommandoer, og sammen danner de en praktisk førstegangsvei:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Hvis du ikke er sikker på hva hver enkelt gjør, husk dem slik:

- `hermes model`: Velg eller bytt modell
- `hermes tools`: Vis og konfigurer tilgjengelige verktøy
- `hermes config set`: endre spesifikke konfigurasjonselementer
- `hermes setup`: kjør hele initialiseringsveiviseren én gang
- `hermes update`: oppdater Hermes
- `hermes doctor`: feilsøk problemer

For nybegynnere er den mest praktiske rekkefølgen vanligvis:

1. Løp `hermes model` først
2. Hvis du vil konfigurere alle vanlige alternativer samtidig, kjør `hermes setup`

## De to vanligste måtene å bruke Hermes på

### 1. Bruk Hermes i terminalen som daglig utviklingsassistent

CLI-modus passer godt for disse scenariene:

- Still spørsmål direkte mens du skriver kode lokalt
- Inspiser prosjekter, rediger filer og kjør kommandoer
- Gjør engangsfeilsøking eller gjennomgå arbeid
- Samarbeid kontinuerlig i gjeldende arbeidskatalog

Dens største fordel er at det er den korteste veien: ingen ekstra plattformintegrasjon, ingen botkonfigurasjon å håndtere på forhånd, og det er den beste måten å bygge opp ditt første sett med bruksvaner.

### 2. Bruk Hermes gjennom en meldingsplattform

Hvis du vil chatte med Hermes på plattformer som Feishu, Telegram eller Discord, må du bruke meldingsporten.

De vanligste inntastingskommandoene er:

```bash
hermes gateway setup
hermes gateway
```

Nærmere bestemt:

- `hermes gateway setup` brukes til interaktiv plattformkonfigurasjon
- `hermes gateway` brukes til å starte gateway-prosessen

I følge den offisielle dokumentasjonen er gatewayen en enhetlig bakgrunnsprosess som kobler sammen de konfigurerte plattformene dine, administrerer økter og håndterer funksjoner som cron-jobber.

## Bruker Feishu som et eksempel: hvordan koble Hermes til en meldingsplattform

Hvis det meste av det daglige arbeidet ditt skjer i Feishu, er Feishu/Lark en veldig naturlig måte å bruke Hermes på.

### Minimum levedyktig integrasjonsvei

Den offisielle dokumentasjonen anbefaler denne inngangskommandoen for Feishu/Lark:

```bash
hermes gateway setup
```

Etter at du har kjørt den, velg ganske enkelt **Feishu / Lark** i veiviseren.

Feishu-dokumentasjonen beskriver to tilkoblingsmoduser:

- `websocket`: anbefales
- `webhook`: valgfritt

Hvis Hermes kjører på din bærbare datamaskin, arbeidsstasjon eller private server, ved hjelp av `websocket` først er vanligvis enklere fordi du ikke trenger å avsløre en offentlig tilbakeringings-URL.

### Hvis du konfigurerer det manuelt, må du i det minste vite disse variablene

Hvis du ikke bruker veiviseren og skriver konfigurasjonen manuelt, viser Feishu-dokumentasjonen disse kjernevariablene:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

To av dem fortjener spesiell oppmerksomhet:

- `FEISHU_ALLOWED_USERS`: anbefales, så ikke alle som kan nå boten kan bruke den direkte
- `FEISHU_HOME_CHANNEL`: lar deg forhåndsdefinere en hjemmechat for å motta cron-resultater eller standardvarsler

### Hvorfor Hermes noen ganger ikke svarer i Feishu gruppechatter

Denne detaljen er lett å gå glipp av: I Feishu gruppechatter svarer ikke Hermes på hver melding som standard.

Den offisielle dokumentasjonen sier tydelig:

- I direktemeldinger svarer Hermes på meldinger
- I gruppechatter må du eksplisitt `@` boten før den vil behandle meldingen

Hvis du vil angi en Feishu-samtale som hjemmekanal, kan du også bruke dette i chatten:

```text
/set-home
```

Eller definer det i konfigurasjonen på forhånd:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Hermes-kommandoene nybegynnere bør huske først

Enten du bruker Hermes i CLI eller på en meldingsplattform, er det allerede nok å huske følgende kommandoer for å komme i gang:

- `/new` eller `/reset`: start en ny økt
- `/model`: se eller bytt modell
- `/retry`: Prøv forrige sving på nytt
- `/undo`: angre forrige interaksjon
- `/compress`: komprimer konteksten manuelt
- `/help`: se hjelp

Hvis du hovedsakelig bruker Hermes på en meldingsplattform, husk en til:

- `/sethome` eller `/set-home`: angi gjeldende chat som hjemmekanal

Disse kommandoene dekker de vanligste operasjonene på nybegynnerstadiet: omstart, justering, rulle tilbake, sjekke og fortsette.

## Ofte stilte spørsmål

### Kan Windows installere Hermes direkte?

Nei. Den gjeldende offisielle dokumentasjonen sier tydelig at innfødt Windows ikke støttes, og WSL2 anbefales.

### Hva skal jeg gjøre hvis jeg skriver `hermes` gjør ingenting etter installasjonen?

Det er best å feilsøke i denne rekkefølgen:

1. Last inn skallet på nytt først, for eksempel med `source ~/.bashrc`
2. Løp `hermes` igjen
3. Hvis det fortsatt er unormalt, løp `hermes doctor`

### Hvorfor svarer ikke boten i en Feishu-gruppe?

Sjekk disse tre tingene først:

- Enten du `@` nevnte Hermes i gruppen
- Enten `FEISHU_ALLOWED_USERS` begrenser gjeldende bruker
- Hvorvidt gjeldende gruppechat-policy tillater håndtering av gruppemeldinger

I følge den offisielle Feishu-dokumentasjonen kreves det eksplisitt bruk av @omtale i gruppechat-scenarier.

## Anbefalte neste trinn

Hvis du bare vil begynne å bruke Hermes så raskt som mulig, er dette den mest anbefalte bestillingen:

1. Kjør installasjonskommandoen først
2. Start med `hermes` i den lokale CLI først
3. Bruk `hermes model` og `hermes setup` for å fullføre den grunnleggende konfigurasjonen
4. Hvis du vil fortsette å bruke den i Feishu, konfigurer `hermes gateway setup`
