## Veiledning for å lage en border Light Sweep-animasjonseffekt

> Hvordan bygger du det viktige elementet brukere legger merke til med et øyeblikk ved å bruke ren CSS? Det er faktisk ikke så vanskelig. Trikset er bare å ta en litt rundkjøringsvei. I denne artikkelen vil jeg lede deg gjennom hvordan du bygger en border light sweep-animasjon fra bunnen av, og også dele noen av fallgruvene vi møtte mens vi bygde HagiCode.

## Bakgrunn

Hvis du jobber på frontend, har du sannsynligvis hatt denne opplevelsen før: en produktsjef går bort med det "dette er definitivt enkelt" uttrykk og sier: "Kan vi legge til en slags spesialeffekt til denne løpende oppgaven slik at brukere kan oppdage det umiddelbart?"

Du sier sikkert, vi kan bare endre kantfargen. Så rister de på hodet med det blikket som sier: «Du skjønner det ikke». De svarer: "Det er ikke åpenbart nok. Jeg vil ha den typen effekt der lyset løper rundt grensen, som i en sci-fi-film."

På det tidspunktet kan du begynne å lure på hvordan du bygger den. Lerret? SVG? Eller kan CSS håndtere det på egen hånd? Tross alt vil ingen innrømme at de ikke vet hvordan.

I moderne nettapplikasjoner er animasjoner med kantlys faktisk veldig vanlige. De brukes hovedsakelig i noen få scenarier som disse:

- **Statusindikatorer**: Merking av oppgaver som pågår eller aktive elementer
- **Visuelt fokus**: Fremheving av viktige innholdsområder
- **Merkevareforbedring**: Skaper en elegant, moderne visuell stil som er teknologisk
- **Sesongtemaer**: Bygg en feststemning for spesielle anledninger

Vi møtte akkurat dette kravet under byggingen [HagiCode](https://hagicode.com). Brukerne måtte raskt se hvilke økter som ble kjørt og hvilke forslag som for øyeblikket ble behandlet. Vi prøvde flere forskjellige tilnærminger. Noen stier var jevnere, noen var litt mer svingete. Til slutt bestemte vi oss for en ganske moden implementeringsstrategi.

## Kjerneimplementeringsideer

Etter å ha analysert HagiCode-kodebasen, oppsummerte vi flere kjerneimplementeringsmønstre nedenfor. Hvert av dem passer til et annet scenario, eller med andre ord, hvert enkelt eksisterer av en grunn.

### 1. Roterende glød med en konisk gradient (mest vanlig)

Dette er den klassiske måten å implementere en border light sweep-effekt. Kjerneideen er å bruke CSS `conic-gradient` for å lage en konisk gradient, og roter den deretter kontinuerlig. Som en gatelykt som snur om natten, fortsetter den å sirkle.

**Nøkkelelementer:**
- Bruk `::before` pseudo-element for å lage glødelaget
- Bruk `conic-gradient` for å definere gradientfargefordelingen
- Bruk `::after` pseudo-element for å maskere midtområdet (valgfritt)
- Bruk `@keyframes` å implementere rotasjonsanimasjonen

### 2. Sideglødelinje

Dette fungerer bra for statusindikatorer i listeelementer. Du lager en tynn glødende linje på den ene siden av elementet i stedet for å animere hele kanten. Noen ganger er litt lys nok. Du trenger ikke å lyse opp hele verden.

**Nøkkelelementer:**
- Et tynt, absolutt plassert linjeelement
- Bruk `box-shadow` for å skape glødeeffekten
- Bruk `scale` og `opacity` for en pusteanimasjon

### 3. Glødende bakgrunn med boks-skygge

Hvis du ikke trenger full sveipeeffekt og bare vil ha en myk bakgrunnsglød, lag flere lag `box-shadow` verdier er nok. Noen ganger er det enklere alternativet det bedre.

### 4. Tilgjengelighetsstøtte

Denne delen er lett å overse, men den er ekstremt viktig. Hver animasjon bør ta hensyn til `prefers-reduced-motion` mediesøk og gi et statisk alternativ for brukere som ikke ønsker animasjon. Ikke alle liker konstant bevegelse, og det er viktig å respektere den preferansen.

## Implementeringsalternativer

### Alternativ 1: Roterende konisk gradientkant (anbefalt)

Dette er den mest komplette implementeringen av den sveipende kantlyseffekten, og det er også alternativet vi bruker oftest i HagiCode. Tross alt, hvis noe fungerer bra, hvorfor erstatte det?

```css
/* Parent container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotating glow layer */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Mask layer (optional, for creating a hollow border effect) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Prinsippet bak dette alternativet er ganske enkelt: lag et pseudo-element som er større enn den overordnede beholderen, tegn en konisk gradient på den og roter den kontinuerlig. Den overordnede containeren bruker `overflow: hidden`, så bare lyset som passerer rundt grensen forblir synlig. Det er litt som å se en gatelykt gjennom et vindu. Du ser bare den lille skiven som går forbi.

### Alternativ 2: Forenklet roterende lyskant

Hvis du ikke trenger den fulle effekten, inkluderer HagiCode også en lettere versjon i verktøyklassen. Noen ganger er den enklere tilnærmingen virkelig bedre.

```css
/* Rotating light border utility class */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Accessibility support */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Legg merke til `will-change: transform` her. Den forteller nettleseren: "Dette elementet kommer til å fortsette å endre seg," slik at nettleseren kan forberede noen optimaliseringer på forhånd og holde animasjonen jevnere. Å forberede seg på forhånd er vanligvis bedre enn å klatre i siste liten.

### Alternativ 3: Sideglødelinje

Dette er spesielt egnet for statusindikatorer for listeelementer, og det er akkurat det HagiCode-sesjonslisten bruker. En tynn strek kan fortsatt skille seg ut blant mange gjenstander. Det føles som en livsleksjon på sin egen måte.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Denne bruker `isolation: isolate` for å lage en ny stablingskontekst, stoler deretter på `z-index` for å kontrollere visningsrekkefølgen for hvert lag. `pointer-events: none` er også viktig. Ellers ville pseudo-elementet blokkert brukerklikk. Noen ting kan se fine ut, men de bør likevel ikke komme i veien.

### Alternativ 4: Pakk den inn i en React-komponent

Hvis prosjektet ditt bruker React, kan du pakke denne logikken inn i en komponent, spesielt tilgjengelighetshåndteringen. Skriv det en gang, bruk det mange ganger. Det er hele poenget.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Den samsvarende CSS-modulen:

```css
/* GlowBorder.module.css */

/* Animated version */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Static version (accessibility) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Den `useReducedMotion` krok fra `framer-motion` oppdager automatisk brukerens systempreferanse. Hvis brukeren har aktivert redusert bevegelse, kommer den tilbake `true`, og komponenten viser den statiske versjonen i stedet. Å respektere brukerens preferanser betyr mer enn å fremtvinge en prangende effekt.

## Praktiske erfaringer

Dette er noen av leksjonene vi lærte mens vi bygde HagiCode. Du kan også kalle dem kamparr. Forhåpentligvis hjelper de deg med å unngå noen omveier.

### 1. Tema variabelt system

CSS-variabler gjør støtte for flere temaer spesielt praktisk. Ingen ønsker å redigere en haug med kode hver gang temaet endres.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Usage */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

På den måten krever bytte av tema bare å endre klassen på `html` element, og hver animasjonsfarge oppdateres automatisk. En kodebase, to stiler. Det er akkurat det vi ønsker.

### 2. Ytelsesoptimalisering

**Bruk `will-change` for å tipse nettleseren om å optimalisere:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Fortell nettleseren på forhånd, så hjelper det deg med å optimalisere. Mange ting i livet fungerer bedre med litt forberedelser.

**Unngå å bruke komplekse boksskygger på store elementer:**

```css
/* Not ideal - using a blurred shadow on a large element */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Better - use a pseudo-element to limit the glowing area */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Vi testet dette i HagiCode. Ved å legge til en uskarp skygge direkte på et stort kort falt rulleframehastighetene under 30 bilder per sekund. Bytte til et pseudo-element brakte ting tilbake til jevne 60 fps. Brukere kan absolutt føle den forskjellen.

### 3. Tilgjengelighet

Du bør virkelig ikke hoppe over dette. Noen brukere synes animasjon er svimlende eller distraherende, og å respektere deres preferanser er en del av å bygge et godt produkt. Vakre ting trenger ikke å påtvinges alle.

**CSS-mediespørring:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Provide a static fallback */
    opacity: 1;
  }
}
```

**Oppdag brukerpreferanser i React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Kontroll av intensitetsnivå

Token-gjennomstrømningsindikatoren i HagiCode viser forskjellige glødefarger basert på sanntidsgjennomstrømning, og dette implementeres dynamisk. Ulike tilstander bør uttrykkes forskjellig.

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Ting å passe på

Det er fortsatt noen få detaljer verdt å være oppmerksom på, for når du oppdager disse problemene på den harde måten, er det allerede for sent.

| Ting å passe på | Forklaring |
|---------|------|
| **z-indeksstyring** | Glødelaget bør bruke en passende `z-index` så det forstyrrer ikke innholdsinteraksjonen |
| **peker-hendelser** | Glød-pseudo-elementet skal settes `pointer-events: none` |
| **Grenseoverløp** | Den overordnede containeren trenger `overflow: hidden`, eller du må justere størrelsen på pseudoelementet |
| **Ytelsespåvirkning** | Komplekse animasjoner kan skade ytelsen på mobile enheter, så test nøye |
| **Mørk modus** | Sørg for at glødefargen forblir godt synlig på mørk bakgrunn |
| **Temabytte** | Bruk CSS-variabler slik at animasjonsfarger oppdateres riktig når temaet endres |

### 6. Feilsøkingstips

Pseudo-elementer kan være litt vanskelige å finne i utviklerverktøy, så du kan midlertidig legge til en kantlinje for å sjekke posisjonen.

```css
/* Temporarily show pseudo-element boundaries for debugging */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Når du er ferdig med å plassere den, husk å kommentere eller fjerne den linjen. Ellers kan produksjonen bli vanskelig ganske raskt. Noen ting er bedre å la være i utvikling.

## Sammendrag

Border light sweep-animasjoner er verken spesielt vanskelige eller virkelig trivielle. I kjernen er formelen `conic-gradient` pluss rotasjon, men hvis du vil ha god ytelse, vedlikehold og tilgjengelighet, er det fortsatt mange implementeringsdetaljer å håndtere nøye.
