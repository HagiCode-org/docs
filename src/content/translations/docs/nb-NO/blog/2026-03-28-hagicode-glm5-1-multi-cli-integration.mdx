## Hagicode og GLM-5.1 Multi-CLI Integration Guide

:::note[Tidskontekstvarsel]

## Bakgrunn

I Hagicode-prosjektet kan brukere velge mellom flere CLI-verktøy for å drive AI-programmeringsassistenter, inkludert Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI og mer. Disse CLI-verktøyene er generelle AI-programmeringsverktøy alene, men gjennom Hagicodes abstraksjonslag kan de fleksibelt koble seg til forskjellige AI-modellleverandører.

Zhipu AI (ZAI) gir et grensesnitt som er kompatibelt med Anthropic Claude API, slik at disse CLI-verktøyene kan bruke innenlandske GLM-seriemodeller direkte. Blant dem er GLM-5.1 Zhipus siste store språkmodellutgivelse, med betydelige forbedringer i forhold til GLM-5.0.

## Hagicodes CLI abstraksjonsarkitektur

Hagicode definerer 11 CLI-leverandørtyper gjennom `AIProviderType` enum, som dekker mainstream AI-programmerings-CLI-verktøy:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Hver CLI har tilsvarende modellparameterkonfigurasjon og støtter `model` og `reasoning` parametere:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],

        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini does not support the reasoning parameter
        // ...
    };
```

## Bruker GLM-5.1 i forskjellige CLI-er

### 1. Claude-kode CLI + GLM-5.1

Claude Code CLI er en av Hagicodes kjerne-CLI og er konfigurert gjennom Hero-konfigurasjonssystemet:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Tilsvarende `HeroEquipmentCatalogItem` konfigurasjon:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI er den mest fleksible CLI og støtter spesifisering av hvilken som helst modell i `provider/model` format:

**Metode 1: Bruk ZAI-leverandørprefikset**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Metode 2: Bruk modell-ID-en direkte**
```json
{
  "model": "glm-5.1"
}
```

**Metode 3: Grensesnitt for grensesnittkonfigurasjon**

i `HeroModelEquipmentForm.tsx`, OpenCode CLI har et dedikert plassholderhint:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Brukere kan skrive inn:
```
zai/glm-5.1
glm-5.1
```

**OpenCode CLI modell parsing logikk**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI har dedikert eldre håndteringslogikk:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Merk**: Codebuddy beholder spesiell håndtering for GLM-5.0 og bruker ikke eldre normalisering:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## Forbedringsfordeler med GLM-5.1

Sammenlignet med GLM-5.0, gir GLM-5.1 følgende betydelige forbedringer:

### 1. Bedre resonnementevne

I følge Zhipus offisielle utgivelsesinformasjon inkluderer forbedringer i GLM-5.1:

- **Sterkere kodeforståelse**: Mer nøyaktig analyse av komplekse kodestrukturer
- **Lengre kontekstforståelse**: Støtter lengre samtalekontekst
- **Forbedret verktøykall**: Høyere suksessrate for MCP-verktøyanrop
- **Utgangsstabilitet**: Reduserer tilfeldigheter og hallusinasjoner

### 2. Omfattende multi-CLI-kompatibilitet

GLM-5.1 dekker alle vanlige CLI-er som støttes av Hagicode:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Notater

### 1. API-nøkkelkonfigurasjon

Sørg for at `ANTHROPIC_AUTH_TOKEN` miljøvariabelen er satt riktig. Det er den nødvendige legitimasjonen for hver CLI å koble til modellen.

### 2. Modelltilgjengelighet

GLM-5.1 må aktiveres av den tilsvarende modellleverandøren:
- Zhipu AI ZAI-plattformen støtter det som standard
- Alibaba Cloud DashScope kan kreve en separat applikasjon

### 3. OpenCode CLI-format

Når du bruker `provider/model` format, sørg for at leverandør-ID-en er riktig:
- Zhipu AI: `zai` eller `zhipuai`
- Alibaba Cloud: `aliyun` eller `dashscope`

### 4. Begrunnelsesparameter

- `high` anbefales for de beste kodegenereringsresultatene
- Gemini CLI støtter ikke resonneringsparameteren og vil ignorere denne konfigurasjonen automatisk

## Konklusjon

Gjennom et enhetlig abstraksjonslag muliggjør Hagicode fleksibel integrasjon mellom GLM-5.1 og flere CLI-er. Utviklere kan velge CLI-verktøyet som passer best for deres preferanser og bruksscenarier, og deretter bruke den nyeste GLM-5.1-modellen gjennom enkel konfigurasjon.

Som Zhipus siste modellversjon tilbyr GLM-5.1 klare forbedringer i forhold til GLM-5.0:
- En uavhengig versjonsidentifikator uten eldre byrde
- Sterkere resonnement og kodeforståelse
- Bred multi-CLI-kompatibilitet
- Fleksibel konfigurasjon av resonnementnivå

Med de riktige miljøvariablene og Hero-utstyret konfigurert, kan brukere fullt ut låse opp kraften til GLM-5.1 på tvers av forskjellige CLI-miljøer.
