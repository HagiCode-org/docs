## Forutsetninger

Før du starter, sørg for at systemet har Docker og Docker Compose installert.

### Installer Docker

#### Windows

Last ned og installer [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)

Etter installasjonen, sørg for at Docker Desktop kjører.

#### macOS

Last ned og installer [Docker Desktop for Mac](https://www.docker.com/products/docker-desktop/)

Etter installasjonen, sørg for at Docker Desktop kjører.

#### Linux

Installer Docker ved å bruke pakkebehandleren din:

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

Etter installasjon, start Docker-tjenesten:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Bekreft installasjonen

Etter installasjonen, kjør følgende kommandoer for å bekrefte at Docker og Docker Compose er riktig installert:

```bash
docker --version
docker compose version
```

## Rask start

### 1. Generer Docker Compose-konfigurasjonsfil

:::tips[Bruk konfigurasjonsgenerator]
**Anbefaler å bruke [Docker Compose Builder (frittstående nettsted)](https://builder.hagicode.com/)** for å generere konfigurasjonsfilen.

Generatoren støtter:
- Flere API-leverandøralternativer (Zhipu AI, Anthropic Official, Custom API)
- Automatisk konfigurasjon av API URL og Token
- Egendefinerte porter, beholdernavn og annen grunnleggende konfigurasjon
- Konfigurer vedvarende data og arbeidskatalogbaner
- Konfigurer Windows/Linux-plattformforskjeller automatisk
- Håndter filtillatelsesproblemer intelligent
- **Valg av bildekilde**: Velg riktig bildekilde basert på nettverksmiljøet ditt

[🚀 Bruk generatoren umiddelbart →](https://builder.hagicode.com/)
:::

1. Åpne [Docker Compose Builder (frittstående nettsted)](https://builder.hagicode.com/)
2. Fyll inn konfigurasjonen i henhold til dine behov:
   - Velg API-leverandør (Zhipu AI, Anthropic Official eller Custom API)
   - Konfigurer porter, database, arbeidskatalog og andre alternativer
   - **Velg bildekilde**: Velg riktig bildekilde basert på nettverksmiljøet ditt
3. Klikk på generer-knappen for å få `docker-compose.yml` konfigurasjon
4. Lagre den genererte konfigurasjonen som en `docker-compose.yml` fil
5. Opprett om nødvendig en `.env` fil for å konfigurere sensitiv informasjon

#### Valg av bildekilde

Generatoren støtter tre bildekildealternativer:

**Docker Hub**
- **Bildeadresse**: `newbe36524/hagicode:{tag}`
- **Gjeldende scenarier**: Egnet for brukere med Docker Hub-speilakselerasjon
- **Fordeler**: Offisiell bildekilde, rettidige oppdateringer, stabil tilgang
- **Merknader**: Speilakselerasjon må kanskje konfigureres i enkelte regioner

**Azure Container Registry (ACR)**
- **Bildeadresse**: `hagicode.azurecr.io/hagicode:{tag}`
- **Gjeldende scenarier**: Passer for brukere som ikke kan få tilgang til Docker Hub lokalt
- **Fordeler**: Gir alternativ bildekilde, løser problemer med nettverkstilgang
- **Merknader**: Bilder forblir synkronisert med Docker Hub, men kan ha korte forsinkelser

**Alibaba Cloud Container Registry (ACR)**
- **Bildeadresse**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **Gjeldende scenarier**: Egnet for hjemmebrukere, stabil og rask nettverkstilgang
- **Fordeler**: Alibaba Cloud innenlandske noder, rask tilgangshastighet, høy stabilitet
- **Merknader**: Alibaba ACR-bilder forblir synkroniserte med Docker Hub, vanligvis innen 30 minutter

:::tips[Anbefalinger for valg av bildekilde]
- **Innlandske brukere anbefaler Alibaba Cloud ACR**: Hvis du er en innenlandsbruker, anbefales det å prioritere å bruke Alibaba Cloud ACR-bildekilde for rask og stabil tilgang
- **Alternativ Docker Hub**: Hvis nettverksmiljøet ditt har tilgang til Docker Hub, kan du også velge Docker Hub som bildekilde
- **Alternativ Azure ACR**: Velg kun Azure ACR-bildekilde hvis andre bildekilder har tilgangsproblemer
- **Bildesynkronisering**: Alibaba Cloud ACR-bilder forblir synkroniserte med Docker Hub, vanligvis innen 30 minutter
- **Versjonskonsistens**: Bildetagger er konsistente på tvers av alle tre bildekildene, du trenger ikke bekymre deg for versjonsproblemer
:::

### 2. Start Tjenester

Utfør følgende kommando i katalogen der du `docker-compose.yml` ligger:

```bash
docker compose up -d
```

Denne kommandoen vil:
- Last ned og start Hagicode-beholderne definert av den genererte malen
- Opprett og konfigurer nettverket
- Initialiser standard datakatalog og kjøretidsavhengigheter

### 3. Bekreft tjenestestatus

Sjekk om containere kjører:

```bash
docker compose ps
```

Du bør se tjenestene som er definert i den genererte malen din i «kjører»-status.

### Se tjenestelogger

Slik viser du alle tjenesteloggene:

```bash
# View all service logs
docker compose logs

# View Hagicode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## Konfigurasjonsinstruksjoner

Hvis du trenger å endre konfigurasjonen (som å endre porter, bytte API-leverandør osv.), kan du:

1. Regenerer ny konfigurasjon ved å bruke [Docker Compose Builder (frittstående nettsted)](https://builder.hagicode.com/)
2. Rediger manuelt `docker-compose.yml` og `.env` filer
3. Start tjenester på nytt for å få konfigurasjonen til å tre i kraft:

```bash
docker compose restart
```

## Få tilgang til applikasjonen

### Webgrensesnitt

Etter vellykket oppstart av tjenesten, få tilgang via nettleseren din:

```
http://localhost:45000
```

### Datapersistens

Den støttede Docker Compose-banen bruker SQLite-datafiler med vedvarende volumer som standard. Hvis du trenger å inspisere monteringspunkter eller planlegge sikkerhetskopier, gå gjennom volum- og datakatalogtilordningene i den genererte `docker-compose.yml`.
