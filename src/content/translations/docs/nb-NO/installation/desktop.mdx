Denne veiledningen dekker installasjon og bruk av Hagicode Desktop. Det er den offisielle skrivebordsklienten og holder standard lokal oppsettsti pluss daglige kontroller på ett sted.

:::tips[Anbefalt metode]
Desktop er den lokale standardbanen for individuelle brukere og utviklere fordi:
- installasjonsprogrammet holder standardflyten i én veiledet bane
- avhengigheter og tjenester administreres i appen
- brukergrensesnittet gjør tilstand og drift enklere å inspisere
- innebygd versjonsadministrasjon gjør bytte enklere
- data forblir lokale som standard
:::

## Hva er Hagicode Desktop?

Hagicode Desktop er Hagicodes offisielle skrivebordsapplikasjon, designet for Windows-, macOS- og Linux-systemer. Den samler Hagicodes kjernefunksjoner, og installasjonsprogrammet beholder standard oppsettsti i én veiledet flyt.

### Hovedfordeler

- **Installasjon med ett klikk**: Fullfør alle konfigurasjonstrinn automatisk gjennom installasjonsveiviseren
- **Automatisk kjøretidsforberedelse**: Standardinstallasjonsprogrammet inkluderer allerede kjøretiden det trenger, så du trenger ikke å installere en separat `.NET` kjøretid først
- **Service Management**: Innebygd Hagicode Server-administrasjon, enkelt start og stopp av tjenester
- **Versjonsbytte**: Støtte for flerversjonsadministrasjon, bytt fritt mellom ulike versjoner
- **Lokal drift**: Alle data lagres lokalt, og beskytter personvernet ditt
- **Automatiske oppdateringer**: Oppdager og laster ned den nyeste versjonen automatisk

### Gjeldende scenarier

Desktop passer godt når:
- Individuelle utviklere som ønsker å bruke Hagicode lokalt
- Brukere som trenger å bruke Hagicode offline
- Ikke-tekniske brukere som ønsker å forenkle installasjonsprosessen
- Utviklere som ofte trenger å bytte mellom ulike versjoner

### Velg riktig versjon

Velg det tilsvarende installasjonsprogrammet for ditt operativsystem:
- **Windows**: Last ned `.exe` installatør
- **macOS**: Last ned `.dmg` installatør
- **Linux**: Last ned `.AppImage` eller `.tar.gz` installasjonspakken

:::notat[Systemkrav]
- **Windows**: Windows 10 eller nyere versjon (64-bit)
- **macOS**: macOS 10.15 eller høyere
- **Linux**: Vanlige Linux-distribusjoner (Ubuntu 20.04+, Fedora 33+, etc.)
:::

:::tip[Standard forutsetninger]
For den nyeste HagiCode lokale installasjonsflyten trenger du vanligvis ikke utarbeide en separat `.NET` kjøretid. Desktop pakker allerede kjøretiden som kreves for standardflyten og initialiserer den støttede lokale SQLite-banen som standard.
:::

## Installasjons- og lanseringsmerknader

::: forsiktig[macOS Gatekeeper-varsel]
Hvis du allerede har installert appen til standardbanen `/Applications/Hagicode.app` men macOS viser "appen er skadet", "kan ikke åpnes" eller en lignende gatekeeper-advarsel ved første lansering, den vanlige årsaken er `com.apple.quarantine` attributt som macOS legger til apper lastet ned fra internett, ikke en feil i selve Hagicode Desktop-installasjonsprogrammet.

Kjør følgende kommando i Terminal. Det krever administratorrettigheter og fjerner rekursivt karanteneattributtet fra appen i standard installasjonsbane:

```bash
sudo xattr -dr com.apple.quarantine /Applications/Hagicode.app
```

Etter at kommandoen er ferdig, start Hagicode Desktop på nytt.
:::

## Kjør Desktop

Etter nedlasting åpner du installasjons- eller programpakken og følger instruksjonene på skjermen. Desktop forbereder kjøretiden i den gjeldende støttede lokale banen, administrerer Hagicode Server, og viser fremdrift og status når nettverksnedlastinger er nødvendig.

:::notat[Nettverkskrav]
Sørg for at nettverkstilkoblingen din er tilgjengelig. Desktop kan trenge å hente versjonsinformasjon eller komponentpakker fra internett; Hvis en nedlasting mislykkes, sjekk nettverks-, proxy- eller brannmurinnstillingene og prøv på nytt.
:::

## Versjonsbehandling

Hagicode Desktop støtter flerversjonsadministrasjon, slik at du fritt kan bytte mellom forskjellige versjoner.

![Versjonsbehandling](../../img/screenshots/installation/install-desktop/12-version-management/original.png)

### Bytt versjon

1. Åpne Hagicode Desktop
2. Gå til "Versjonsadministrasjon"-siden
3. Velg versjonen du vil bytte
4. Klikk på "Bytt versjon"-knappen
5. Vent til byttet er fullført

:::note[Versjonsbyttenotater]
- Bytting av versjon vil automatisk starte Hagicode Server på nytt
- Sørg for at du lagrer alt arbeid før du bytter
- Prosjektkonfigurasjoner kan variere mellom versjoner
:::

## Start og stopp service

Hagicode Desktop gir en enkel måte å administrere Hagicode Server-tjenesten på.

### Start service

På startsiden til skrivebordsapplikasjonen klikker du på "Start tjeneste"-knappen for å starte Hagicode Server.

![Start service](../../img/screenshots/installation/install-desktop/13-start-service-homepage/original.png)

### Stopp tjenesten

For å stoppe Hagicode Server, klikk på "Stopp tjeneste"-knappen på hjemmesiden.

:::tips[Tjenestestatus]
Desktop-applikasjonen viser Hagicode Servers kjørestatus i sanntid, inkludert:
- Tjenestestatus (kjører/stoppet)
- Serviceportnummer
- Informasjon om gjeldende versjon
:::

## Feilsøking

### Installasjonsfeil

Hvis du støter på problemer under installasjonen:

1. **Sjekk nettverkstilkobling**: Sørg for at du har tilgang til internett
2. **Sjekk diskplass**: Sørg for at du har nok diskplass
3. **Kjør som administrator**: På Windows høyreklikker du på installasjonsprogrammet og velger "Kjør som administrator"
4. **Se loggfiler**: Installasjonsprogrammet genererer detaljerte loggfiler som kan hjelpe med å diagnostisere problemer

### Avhengighetsinstallasjonsfeil

Hvis avhengighetsinstallasjonen mislykkes:

1. **Sjekk systemkompatibilitet**: Sørg for at operativsystemet ditt oppfyller minimumskravene
2. **Sjekk nettverks- og brannmurinnstillinger**: Sørg for at installasjonsprogrammet kan nå nødvendige nedlastinger og ikke er blokkert av brannmuren din
3. **Bekreft installasjonstillatelser**: På Windows, prøv å kjøre installasjonsprogrammet som administrator slik at det har tilstrekkelige tillatelser
4. **Start appen på nytt og prøv på nytt**: Lukk og åpne Hagicode Desktop på nytt, og prøv deretter installasjonstrinnet på nytt
5. **Se logger**: Se gjennom detaljerte feillogger i skrivebordsapplikasjonen

### Feil ved oppstart av tjeneste

Hvis Hagicode Server ikke starter:

1. **Sjekk portbelegg**: Sørg for at port 45000 ikke er okkupert av andre programmer
2. **Sjekk brannmurinnstillinger**: Sørg for at brannmuren tillater Hagicode Server-nettverkstilgang
3. **Start applikasjonen på nytt**: Prøv å starte Hagicode Desktop på nytt
4. **Se logger**: Se detaljerte feillogger i skrivebordsapplikasjonen

### Trenger du mer hjelp?

Hvis du støter på problemer som ikke dekkes her:

1. Sjekk [GitHub-problemer](https://github.com/HagiCode-org/site/issues) for lignende problemer
2. Besøk vår [Fellesskapsforum](https://github.com/HagiCode-org/site/discussions) for hjelp
3. Send inn en ny utgave, og beskriv problemet ditt i detalj
