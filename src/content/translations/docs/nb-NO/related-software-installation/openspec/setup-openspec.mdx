# Installer OpenSpec

OpenSpec administrerer forslag, spesifikasjoner og endringsposter. Denne siden holder seg til ett mål: forklar installasjonsbanen tydelig slik at du kan knytte den til din eksisterende arbeidsflyt.

## Hva er OpenSpec?

OpenSpec er et kommandolinjeverktøy for å administrere forslag og spesifikasjoner i programvarearbeid. Kjernearbeidsflyten inkluderer:

- **Proposal Management**: Lag, gjennomgå og spor funksjonsforslag
- **Spesifikasjonsvalidering**: Sørg for at kodeendringer samsvarer med forhåndsdefinerte spesifikasjoner
- **Endringssporing**: Registrer all gjennomgått endringshistorikk
- **Teamsamarbeid**: Støtt samarbeidsutvikling med flere personer

### Søknad i HagiCode Projects

I HagiCode-prosjekter brukes OpenSpec til:

1. **Funksjonsutviklingsadministrasjon**: Alle nye funksjoner må gå gjennom OpenSpec-forslagsprosessen
2. **Kodegjennomgang**: Sørg for at endringer er i samsvar med prosjektspesifikasjonene
3. **Dokumentasjonssynkronisering**: Oppretthold konsistens mellom dokumentasjon og kodeimplementering
4. **Versjonskontrollintegrasjon**: Sømløs integrasjon med arbeidsflyter for versjonskontroll

## Forutsetninger

Før du installerer OpenSpec, sørg for at systemet ditt oppfyller følgende krav:

### Node.js og npm

OpenSpec distribueres via npm og krever et Node.js-miljø. OpenSpec krever **Node.js 18.0 eller høyere**.

Bekreft Node.js-installasjonen:

```bash
node --version
```

Bekreft npm-installasjon:

```bash
npm --version
```

Hvis den ikke er installert eller versjonen er for gammel, vennligst se instruksjonene for tilhørende programvareinstallasjon i [Installasjonsveiledning](/en/installation).

## Installerer OpenSpec

:::tips[Versjonskrav]
Dette prosjektet krever bruk av **OpenSpec 1.0.0 eller høyere** versjonsområde. Alle 1.x-versjoner støttes for å sikre kompatibilitet med prosjektarbeidsflyten.
:::

OpenSpec installeres globalt via npm, noe som gjør den tilgjengelig fra hvor som helst på systemet ditt.

### Installer globalt ved å bruke npm

<Tabs>
  <TabItem value="win" label="Windows">
    Kjør i PowerShell eller ledetekst:

```powershell
npm install -g @fission-ai/openspec@1
```

:::notat[Windows-tillatelsesproblemer]
Hvis du støter på tillatelsesfeil under installasjonen, kjør PowerShell eller kommandoprompt som administrator.
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    Kjør i terminal:

```bash
npm install -g @fission-ai/openspec@1
```

:::notat[macOS-tillatelsesproblemer]
Hvis du møter `EACCES` tillatelsesfeil, prøv:
```bash
sudo npm install -g @fission-ai/openspec@1
```
Eller bruk [nvm](https://github.com/nvm-sh/nvm) for å administrere Node.js-versjoner og unngå tillatelsesproblemer.
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    Kjør i terminal:

```bash
npm install -g @fission-ai/openspec@1
```

:::notat[Linux-tillatelsesproblemer]
Hvis du møter `EACCES` tillatelsesfeil, kan du:
1. Bruk `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. Konfigurer npm for å bruke brukerkatalog:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### Bekreft installasjonen

Etter installasjonen må du kontrollere at OpenSpec er riktig installert og at versjonen oppfyller kravene:

```bash
openspec --version
```

Hvis installeringen er vellykket, vil du se et versjonsnummer, for eksempel:

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::notat[Versjonsområde]
Sørg for at den installerte OpenSpec-versjonen er 1.0.0 eller høyere. Prosjektet har verifisert kompatibilitet med versjon 1.1.1. Det anbefales å bruke den nyeste kompatible versjonen for å få nye funksjoner.
:::

## Vanlige problemer og feilsøking

### Tillatelsesfeil

#### Problem: `EACCES` Tillatelse nektet

Oppstår tillatelsesfeil når du installerer eller kjører OpenSpec.

**Løsninger**:

<Tabs>
  <TabItem value="win" label="Windows">
    Kjør PowerShell eller kommandoprompt som administrator:
    1. Høyreklikk PowerShell eller ledetekst
    2. Velg "Kjør som administrator"
    3. Kjør installasjonskommandoen på nytt
  </TabItem>
  <TabItem value="mac" label="macOS">
    Installer ved hjelp av `sudo`:

```bash
sudo npm install -g @fission-ai/openspec@1
```

Eller bruk [nvm](https://github.com/nvm-sh/nvm) for å administrere Node.js og unngå problemer med globale installasjonstillatelser.
  </TabItem>
  <TabItem value="linux" label="Linux">
    Løsning 1: Bruk sudo
```bash
sudo npm install -g @fission-ai/openspec@1
```

Løsning 2: Konfigurer npm til å bruke brukerkatalog
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @fission-ai/openspec@1
```
  </TabItem>
</Tabs>

### Nettverksproblemer

#### Problem: npm Installasjonen mislykkes eller er veldig sakte

Kan ikke laste ned OpenSpec fra npm-registeret på grunn av nettverksårsaker.

**Løsninger**:

1. **Bruk innenlands npm-speil**:

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **Gjenopprett offisiell kilde** (hvis nødvendig):

```bash
npm config set registry https://registry.npmjs.org
```

3. **Bruk cnpm** (valgfritt):

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

:::tips
Brukere i fastlands-Kina anbefales å bruke Taobao-speilet, som kan forbedre nedlastingshastigheten betydelig.
:::

### Problemer med versjonskompatibilitet

#### Problem: OpenSpec-kommandoer kjører unormalt eller rapporter feil

Den installerte OpenSpec-versjonen er inkompatibel med gjeldende miljø.

**Løsninger**:

1. **Sjekk Node.js-versjonen**:

```bash
node --version
```

Sørg for at versjonen er 18.0 eller høyere. Hvis versjonen er for lav, vennligst oppgrader Node.js.

2. **Sjekk OpenSpec-versjonen**:

```bash
openspec --version
```

3. **Installer spesifikk versjon på nytt**:

```bash
npm install -g @fission-ai/openspec@1
```

4. **Tøm npm-buffer** (hvis problemet vedvarer):

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### Kommando ikke funnet

#### Problem: Skriving `openspec` Kommandoen viser "Kommando ikke funnet"

OpenSpec er installert, men systemet finner ikke kommandoen.

**Løsninger**:

<Tabs>
  <TabItem value="win" label="Windows">
    1. Bekreft npm global installasjonsbane:

```powershell
npm config get prefix
```

2. Legg til bane til system PATH:
   - Åpne "Systemegenskaper" → "Avansert" → "Miljøvariabler"
   - Finn `Path` i "Systemvariabler"
   - Legg til npm global bane (vanligvis `C:\Users\<username>\AppData\Roaming\npm`)
   - Start kommandolinjevinduet på nytt
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. Bekreft npm global installasjonsbane:

```bash
npm config get prefix
```

2. Legg til bane til PATH (in `~/.zshrc` eller `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Last inn konfigurasjonen på nytt:

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. Bekreft npm global installasjonsbane:

```bash
npm config get prefix
```

2. Legg til bane til PATH (in `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Last inn konfigurasjonen på nytt:

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
