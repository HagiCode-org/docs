Denne veiledningen dekker installasjon av Node.js på Windows, macOS og Linux. Node.js er en nøkkelavhengighet for HagiCode lokalt verktøy, inkludert OpenSpec, frontend-verktøy og flere utvikler-CLI-er.

:::tips[Versjonskrav]
HagiCode anbefaler **Node.js 24+** (minimum støttet Node.js >= 18.0). I de fleste tilfeller, installer den nyeste LTS-versjonen og kontroller deretter at din versjon oppfyller kravet.
:::

## Installerer Node.js

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Metode 1: Offisielt installasjonsprogram (anbefalt)

1. Besøk [Node.js offisielle nettsted](https://nodejs.org/)
2. Last ned **LTS (Long Term Support)** versjon Windows installasjonsprogram (`.msi` fil)
3. Dobbeltklikk for å kjøre installasjonsprogrammet
4. Følg instruksjonene i installasjonsveiviseren for å fullføre installasjonen:
   - Godta lisensavtalen
   - Velg installasjonsbane (standard er greit)
   - Sørg for at "Installer de nødvendige verktøyene automatisk" er merket av
   - Klikk "Installer" for å starte installasjonen
5. Etter at installasjonen er fullført, klikk "Fullfør"

### Metode to: Bruk av vinge

Åpne PowerShell eller kommandoprompt og kjør:

```powershell
winget install OpenJS.NodeJS.LTS
```

Etter installasjonen åpner du terminalen på nytt for at endringer trer i kraft.

  </TabItem>
  <TabItem value="macos" label="macOS">

### Metode 1: Offisielt installasjonsprogram (anbefalt)

1. Besøk [Node.js offisielle nettsted](https://nodejs.org/)
2. Last ned **LTS (Long Term Support)** versjon macOS installasjonsprogrammet (`.pkg` fil)
3. Dobbeltklikk for å kjøre installasjonsprogrammet
4. Følg instruksjonene i installasjonsveiviseren for å fullføre installasjonen:
   - Klikk "Fortsett"
   - Godta lisensavtalen
   - Velg installasjonsdisk
   - Klikk på "Installer" og skriv inn administratorpassord
5. Etter at installasjonen er fullført, klikk "Lukk"

### Metode to: Bruke Homebrew

Hvis du har [Hjemmebrygg](https://brew.sh/) installert, kan du installere Node.js via terminal:

```bash
brew install node
```

:::tips
Homebrew vil automatisk legge til Node.js til din PATH. Hvis kommandoen ikke er tilgjengelig, start terminalen på nytt eller kjør:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

  </TabItem>
  <TabItem value="ubuntu" label="Ubuntu/Debian">

### Bruke NodeSource Repository (anbefalt)

Dette er den beste måten å få den nyeste versjonen av Node.js:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Bruker apt Package Manager

Hvis du ikke trenger den nyeste versjonen, kan du bruke standard Ubuntu-depot:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::merk
Node.js-versjonen i standarddepotet kan være eldre. Etter installasjonen, sjekk versjonen for å sikre at den oppfyller kravene til Node.js >= 18.0 (Node.js 24+ anbefales).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### Bruke NodeSource Repository (anbefalt)

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### Bruker yum Package Manager

```bash
sudo yum install -y nodejs npm
```

  </TabItem>

## Bruke versjonsadministrasjonsverktøy

Hvis du trenger å bytte mellom flere Node.js-versjoner, kan du bruke versjonsadministrasjonsverktøy.

### nvm (Node Version Manager)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

Etter installasjonen, last inn skallkonfigurasjonen på nytt:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

På Windows anbefaler vi å bruke [nvm-vinduer](https://github.com/coreybutler/nvm-windows):

1. Last ned det siste `nvm-setup.exe` fra [nvm-windows utgivelser](https://github.com/coreybutler/nvm-windows/releases)
2. Kjør installasjonsprogrammet
3. Åpne kommandoprompt eller PowerShell på nytt

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

Etter å ha installert nvm, kan du bruke den til å installere og administrere Node.js-versjoner:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (Fast Node Manager)

fnm er et raskere og enklere Node.js versjonsadministrasjonsverktøy bygget med Rust.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

Bruk [scoop](https://scoop.sh/) eller [vinge](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Akselerasjon for brukere av fastlands-Kina

Hvis du er i Kina, kan nedlasting av Node.js- og npm-pakker gå tregt. Her er noen akselerasjonsløsninger.

### Bruke innenlandske speilkilder

#### npm Taobao Mirror (anbefalt)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### Bruke nrm til å administrere speilkilder

nrm (npm registry manager) hjelper deg raskt å bytte npm-speilkilder:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Akselererer nedlastinger av Node.js Installer

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao tilbyr speilnedlastinger for Node.js-installatører:

- **Nedlastingsadresse**: https://npmmirror.com/mirrors/node/

Besøk adressen ovenfor og velg versjonen du vil laste ned.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

Hvis nedlasting av nvm eller andre verktøy fra GitHub går sakte, kan du bruke FastGit-speilet:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::merk
FastGit-tjenesten kan endres, vennligst se offisiell informasjon.
:::

  </TabItem>
</Tabs>

### Hjemmebryggede speil

Hvis du bruker Homebrew i Kina, kan du konfigurere hjemlige speil for akselerasjon:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Spesifikke pakkeansvarlige for fastlands-Kina

#### cnpm

cnpm er en npm-klient levert av Taobao-teamet som bruker Taobao-speilet som standard:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::tips
cnpm er stort sett kompatibel med npm-kommandoer og kan brukes som en direkte erstatning.
:::

### Verifiserer speilakselerasjonseffekt

Etter å ha satt opp speil, verifiser ved å:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Verifisering av installasjon

Etter installasjonen, kontroller at Node.js og npm er riktig installert:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::merk
Sørg for at Node.js versjon >= 18.0 (Node.js 24+ anbefales). Hvis versjonen er for lav, installer på nytt ved å bruke metodene ovenfor eller bruk versjonsadministrasjonsverktøy for å installere riktig versjon.
:::

### Eksempel på verifikasjonsutgang

Vellykket bekreftelsesutgang skal se slik ut:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Feilsøking

### Windows-problemer

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Hvis `node` eller `npm` kommandoer er ikke tilgjengelige etter installasjon:

1. Start kommandoprompt eller PowerShell på nytt
2. Hvis fortsatt ikke tilgjengelig, legg til Node.js manuelt i PATH:
   - Søk etter "Environment Variables" og åpne "Rediger systemmiljøvariabler"
   - Klikk på "Miljøvariabler"
   - Finn "Bi" i "Systemvariabler" og klikk "Rediger"
   - Legg til installasjonsbane for Node.js (standard: `C:\Program Files\nodejs\`)
   - Klikk "OK" for å lagre endringene

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Hvis du støter på utførelsespolicyfeil når du bruker nvm:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### macOS-problemer

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Hvis du støter på tillatelsesfeil når du installerer globale npm-pakker:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Hvis kommandoer ikke er tilgjengelige etter installasjon, start terminalen på nytt eller legg manuelt til PATH (vanligvis i `/usr/local/bin`).

  </TabItem>
</Tabs>

### Linux-problemer

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Hvis du støter på tillatelsesfeil når du installerer globale npm-pakker, anbefales det å bruke Node.js versjonsadministrasjonsverktøy (som nvm eller fnm) i stedet for systempakkeadministratorer.

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Hvis Node.js-versjonen installert via systempakkebehandling er for gammel:

1. Bruk NodeSource-depot (anbefalt, se ovenfor)
2. Eller bruk versjonsadministrasjonsverktøy (nvm/fnm)

  </TabItem>
</Tabs>

### Sjekker miljø

Hvis verifiseringen mislykkes, kontroller følgende:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
