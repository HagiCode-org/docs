Denne siden er for førstegangsbrukere av HagiCode som ønsker den korteste OmniRoute-oppsettstien: start den lokale tjenesten, bekreft at `http://localhost:20128/v1` fungerer, og bekreft at både Dashboard og API er tilgjengelige.

:::tips[Anbefalt bane]
Denne første versjonen holder den mest stabile banen først: `npm install -g omniroute`, `omniroute`, Dashboard kl `http://localhost:20128`, og den OpenAI-kompatible base-URLen på `http://localhost:20128/v1`. Docker, VPS og mer avanserte distribusjonsmoduser forblir i den valgfrie delen.
:::

## Forutsetninger

- Node.js er allerede installert. Komplett [Installerer Node.js](/en/related-software-installation/nodejs/installation) først om nødvendig.
- Lokal havn `20128` er tilgjengelig og ikke allerede opptatt.
- Du har minst én oppstrømsmodellleverandørkonto som du planlegger å rute gjennom OmniRoute.
- Før du integrerer HagiCode, en CLI eller en SDK, hold disse to adressene atskilt:
  - Dashboard: `http://localhost:20128`
  - OpenAI-kompatibel base-URL: `http://localhost:20128/v1`

## Installer og kjør lokalt

Start med den anbefalte globale npm-banen:

```bash
npm install -g omniroute
omniroute
```

Etter at tjenesten starter, valider den i denne rekkefølgen:

1. Åpne `http://localhost:20128` i en nettleser og bekreft at dashbordet lastes inn.
2. Gå til `Endpoints` side i dashbordet og opprett eller kopier en API-nøkkel.
3. Legg merke til basis-URLen du vil gi til verktøy senere: `http://localhost:20128/v1`.

:::notat[Hvorfor starte med dashbordet]
For det første oppsettet er Dashboard den raskeste måten å bekrefte at OmniRoute kjører, leverandørene dine er tilkoblet og en API-nøkkel eksisterer. Hvis et senere klientanrop mislykkes, er det vanligvis raskere å sjekke dashbordet først enn å feilsøke SDK-kode umiddelbart.
:::

## Valider den kjørende tjenesten

### Validering 1: bekreft at modellkatalogen er lesbar

Sett nøkkelen fra `Endpoints` side inn i en miljøvariabel:

```bash
export OMNIROUTE_API_KEY="replace-with-the-key-you-copied-from-the-Endpoints-page"
```

Be så om `/v1/models`:

```bash
curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}"
```

Hvis du får tilbake en modellliste, fungerer det OpenAI-kompatible endepunktet og godkjenningsflyten.

### Validering 2: send en minimal chat-forespørsel

```bash
curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "replace-with-a-model-id-from-v1-models",
    "messages": [
      { "role": "user", "content": "Reply with ok." }
    ]
  }'
```

Målet her er ikke rask kvalitet. Det er for å bevise at din valgte modell allerede kan rutes gjennom OmniRoute.
