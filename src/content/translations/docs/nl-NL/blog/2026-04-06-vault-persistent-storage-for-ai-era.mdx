## Hoe projecten te reproduceren in het AI-tijdperk: Vault, een persistent opslagsysteem voor meerdere projecten

> Hoe kunnen we AI-assistenten helpen onze leermiddelen beter te begrijpen in het tijdperk van door AI ondersteunde ontwikkeling? Het HagiCode-project heeft het Vault-systeem gebouwd als een uniforme abstractielaag voor kennisopslag die AI kan begrijpen, waardoor de efficiëntie van het leren door middel van projectreproductie aanzienlijk wordt verbeterd.

## Achtergrond

In het AI-tijdperk verandert de manier waarop ontwikkelaars nieuwe technologieën en architecturen leren diepgaand. "Het reproduceren van projecten" - dat wil zeggen het diepgaand bestuderen en leren van de code, architectuur en ontwerppatronen van uitstekende open source-projecten - is een efficiënte manier van leren geworden. Vergeleken met traditionele methoden zoals het lezen van boeken of het bekijken van video's, helpt het direct lezen en uitvoeren van open source-projecten van hoge kwaliteit u veel sneller inzicht te krijgen in de praktijk van engineering.

Toch brengt deze leermethode nogal wat uitdagingen met zich mee.

Leermateriaal is te verspreid. Je aantekeningen kunnen in Obsidian staan, codeopslagplaatsen kunnen verspreid zijn over verschillende mappen en de gespreksgeschiedenis van je AI-assistent wordt nog een geïsoleerd gegevenseiland. Als je wilt dat AI helpt bij het analyseren van een project, moet je handmatig codefragmenten kopiëren en de context organiseren, wat nogal vervelend is.

Wat nog lastiger is, is de gebroken context. AI-assistenten kunnen niet direct toegang krijgen tot je lokale leermiddelen, waardoor je bij elk gesprek opnieuw achtergrondinformatie moet geven. Bovendien worden gereproduceerde codeopslagplaatsen snel bijgewerkt, is handmatige synchronisatie foutgevoelig en is kennis moeilijk te delen over meerdere leerprojecten.

In de kern komen al deze problemen voort uit ‘data-eilanden’. Als er een uniforme opslagabstractielaag zou zijn waarmee AI-assistenten al uw leermiddelen kunnen begrijpen en openen, zou het probleem netjes zijn opgelost.

## Over HagiCode

Het Vault-systeem dat in dit artikel wordt gedeeld, is precies de oplossing die we tijdens het bouwen hebben ontwikkeld [HagiCode](https://hagicode.com). HagiCode is een AI-coderingsassistentproject, en in ons dagelijkse ontwikkelingswerk moeten we vaak veel verschillende open source-projecten bestuderen en ernaar verwijzen. Om AI-assistenten deze leermiddelen beter te laten begrijpen, hebben we Vault ontworpen, een persistent opslagsysteem voor meerdere projecten.

Deze oplossing is al gevalideerd in HagiCode in echt gebruik. Als u met soortgelijke uitdagingen op het gebied van kennisbeheer wordt geconfronteerd, hoop ik dat deze ervaringen u wat inspiratie kunnen bieden. Als je eenmaal zelf in een paar kuilen bent gevallen, moet je immers iets achterlaten voor de volgende persoon.

## Ontwerpfilosofie van het kluissysteem

Het kernidee van het Vault-systeem is eenvoudig: creëer een uniforme abstractielaag voor kennisopslag die AI kan begrijpen. Vanuit implementatieperspectief heeft het systeem een ​​aantal belangrijke kenmerken.

### Ondersteuning voor meerdere typen

Het systeem ondersteunt vier kluistypen, die elk overeenkomen met een ander gebruiksscenario:

```typescript
// folder: general-purpose folder type
export const DEFAULT_VAULT_TYPE = 'folder';

// coderef: a type specifically for reproduced code projects
export const CODEREF_VAULT_TYPE = 'coderef';

// obsidian: integrated with Obsidian note-taking software
export const OBSIDIAN_VAULT_TYPE = 'obsidian';

// system-managed: vault automatically managed by the system
export const SYSTEM_MANAGED_VAULT_TYPE = 'system-managed';
```

Onder hen wordt het type **coderef** het meest gebruikt in HagiCode. Het is specifiek ontworpen voor gereproduceerde codeprojecten en biedt een gestandaardiseerde directorystructuur en AI-leesbare metadatabeschrijvingen.

### Persistent opslagmechanisme

Het Vault-register wordt permanent opgeslagen in JSON-indeling, zodat de configuratie beschikbaar blijft nadat de applicatie opnieuw is opgestart:

```csharp
public class VaultRegistryStore : IVaultRegistryStore
{
    private readonly string _registryFilePath;

    public VaultRegistryStore(IConfiguration configuration, ILogger<VaultRegistryStore> logger)
    {
        var dataDir = configuration["DataDir"] ?? "./data";
        var absoluteDataDir = Path.IsPathRooted(dataDir)
            ? dataDir
            : Path.GetFullPath(Path.Combine(Directory.GetCurrentDirectory(), dataDir));

        _registryFilePath = Path.Combine(absoluteDataDir, "personal-data", "vaults", "registry.json");
    }
}
```

Het voordeel van dit ontwerp is dat het eenvoudig en betrouwbaar is. JSON is leesbaar voor mensen, wat foutopsporing en handmatig bewerken eenvoudiger maakt; bestandssysteemopslag vermijdt de complexiteit van een database en vermindert systeemafhankelijkheden. Soms is de eenvoudigste optie tenslotte ook echt de beste.

### AI-contextintegratie

Het belangrijkste is dat het systeem automatisch kluisinformatie kan injecteren in de context van AI-voorstellen:

```typescript
export function buildTargetVaultsText(
  vaults: VaultForText[],
  template: VaultPromptTemplate = DEFAULT_VAULT_PROMPT_TEMPLATE,
): string {
  const readOnlyVaults = vaults.filter((vault) => vault.accessType === 'read');
  const editableVaults = vaults.filter((vault) => vault.accessType === 'write');

  if (readOnlyVaults.length === 0 && editableVaults.length === 0) {
    return '';
  }

  const sections = [
    buildVaultSection(readOnlyVaults, template.reference),
    buildVaultSection(editableVaults, template.editable),
  ].filter(Boolean);

  return `\n\n### ${template.heading}\n\n${sections.join('\n')}`;
}
```

Dit maakt een belangrijke mogelijkheid mogelijk: AI-assistenten kunnen de beschikbare leermiddelen automatisch begrijpen zonder dat gebruikers handmatig context moeten geven. Je zou kunnen zeggen dat dit een soort stilzwijgend begrip is.

## De gestandaardiseerde structuur van CodeRef Vault

Voor het coderef-type kluis biedt HagiCode een gestandaardiseerde mapstructuur:

```
my-coderef-vault/
├── index.yaml          # vault metadata description
├── AGENTS.md           # operating guide for AI assistants
├── docs/               # stores study notes and documents
└── repos/              # manages reproduced code repositories through Git submodules
```

Bij het maken van een kluis initialiseert het systeem automatisch deze structuur:

```csharp
private async Task EnsureCodeRefStructureAsync(
    string vaultName,
    string physicalPath,
    ICollection<VaultBootstrapDiagnosticDto> diagnostics,
    CancellationToken cancellationToken)
{
    Directory.CreateDirectory(physicalPath);

    var indexPath = Path.Combine(physicalPath, CodeRefIndexFileName);
    var docsPath = Path.Combine(physicalPath, CodeRefDocsDirectoryName);
    var reposPath = Path.Combine(physicalPath, CodeRefReposDirectoryName);

    // Create the standard directory structure
    if (!Directory.Exists(docsPath))
    {
        Directory.CreateDirectory(docsPath);
    }

    if (!Directory.Exists(reposPath))
    {
        Directory.CreateDirectory(reposPath);
    }

    // Create the AGENTS.md guide
    await EnsureCodeRefAgentsDocumentAsync(physicalPath, cancellationToken);

    // Create the index.yaml metadata
    await WriteCodeRefIndexDocumentAsync(indexPath, mergedDocument, cancellationToken);
}
```

Ook deze structuur is zorgvuldig ontworpen:

- **docs/** slaat uw studieaantekeningen op, waar u uw begrip van de code, architectuuranalyse, geleerde lessen, enzovoort kunt vastleggen in Markdown
- **repos/** beheert gereproduceerde opslagplaatsen via Git-submodules in plaats van code rechtstreeks te kopiëren, waardoor de code gesynchroniseerd blijft en ruimte wordt bespaard
- **index.yaml** bevat de metagegevens van de kluis, zodat AI-assistenten snel het doel en de inhoud van de kluis kunnen begrijpen
- **AGENTS.md** is een gids die speciaal is geschreven voor AI-assistenten en waarin wordt uitgelegd hoe u met de inhoud van de kluis moet omgaan

Op deze manier georganiseerd, kan AI misschien iets gemakkelijker begrijpen wat u in gedachten heeft.

## Automatische initialisatie voor door het systeem beheerde kluizen

Naast handmatig gemaakte kluizen ondersteunt HagiCode ook door het systeem beheerde kluizen:

```csharp
public async Task<IReadOnlyList<VaultRegistryEntry>> EnsureAllSystemManagedVaultsAsync(
    CancellationToken cancellationToken = default)
{
    var definitions = GetAllResolvedDefinitions();
    var entries = new List<VaultRegistryEntry>(definitions.Count);

    foreach (var definition in definitions)
    {
        entries.Add(await EnsureResolvedSystemManagedVaultAsync(definition, cancellationToken));
    }

    return entries;
}
```

Het systeem maakt en beheert automatisch de volgende kluizen:

- **hagiprojectdata**: opslag van projectgegevens die wordt gebruikt om de projectconfiguratie en -status op te slaan
- **persoonlijke gegevens**: persoonlijke gegevensopslag gebruikt om gebruikersvoorkeuren op te slaan
- **hbsprompt**: een promptsjabloonbibliotheek die wordt gebruikt om veelgebruikte AI-prompts te beheren

Deze kluizen worden automatisch geïnitialiseerd wanneer het systeem opstart, zodat gebruikers ze niet handmatig hoeven te configureren. Sommige dingen kunnen gewoon beter aan het systeem worden overgelaten in plaats van dat mensen zich er zorgen over maken.

## Toegangscontrolemechanisme

Een belangrijk onderdeel van het ontwerp is toegangscontrole. Het systeem verdeelt kluizen in twee soorten toegang:

```typescript
export interface VaultForText {
  id: string;
  name: string;
  type: string;
  physicalPath: string;
  accessType: 'read' | 'write';  // Key: distinguish read-only from editable
}
```

- **referentie (alleen-lezen)**: AI wordt alleen gebruikt voor analyse en begrip en kan de inhoud niet wijzigen. Geschikt voor open source-projecten, documenten en soortgelijke materialen waarnaar wordt verwezen
- **bewerkbaar (bewerkbaar)**: AI kan inhoud aanpassen als dat nodig is voor de taak. Geschikt voor uw aantekeningen, concepten en soortgelijke materialen

Dit onderscheid is van belang. Het vertelt AI welke inhoud een ‘alleen-lezen referentie’ is en welke inhoud ‘veilig kan worden bewerkt’, waardoor het risico op onbedoelde wijzigingen wordt verkleind. Niemand wil immers dat zijn harde werk verloren gaat door een onbedoelde bewerking.

## In de praktijk: Vault maken en gebruiken

Nu we de ideeën hebben besproken, gaan we kijken hoe het in de praktijk werkt.

### Maak een CodeRef-kluis

Hier is een compleet voorbeeld van een frontend-oproep:

```typescript
const createCodeRefVault = async () => {
  const response = await VaultService.postApiVaults({
    requestBody: {
      name: "React Learning Vault",
      type: "coderef",
      physicalPath: "/Users/developer/vaults/react-learning",
      gitUrl: "https://github.com/facebook/react.git"
    }
  });

  // The system will automatically:
  // 1. Clone the React repository into vault/repos/react
  // 2. Create the docs/ directory for notes
  // 3. Generate the index.yaml metadata
  // 4. Create the AGENTS.md guide file

  return response;
};
```

Deze API-aanroep voltooit een reeks acties: het maken van de directorystructuur, het initialiseren van Git-submodules, het genereren van metadatabestanden en meer. U hoeft alleen de basisinformatie op te geven en de rest aan het systeem over te laten. Het is eerlijk gezegd een tamelijk zorgeloze aanpak.

### Gebruik Vault in een AI-voorstel

Nadat u de kluis heeft gemaakt, kunt u ernaar verwijzen in een AI-voorstel:

```typescript
const proposal = composeProposalChiefComplaint({

```

Het systeem injecteert automatisch kluisinformatie in de AI-context, waardoor AI weet welke leermiddelen beschikbaar zijn. Wanneer AI kan begrijpen wat u in gedachten heeft, is dat soort stilzwijgende kennis moeilijk te verkrijgen.

## Best practices en zaken waar u op moet letten

Tijdens het gebruik van het Vault-systeem hebben we een aantal geleerde lessen samengevat.

### Padveiligheid

Het systeem valideert paden strikt om path traversal-aanvallen te voorkomen:

```csharp
private static string ResolveFilePath(string vaultRoot, string relativePath)
{
    var rootPath = EnsureTrailingSeparator(Path.GetFullPath(vaultRoot));
    var combinedPath = Path.GetFullPath(Path.Combine(rootPath, relativePath));
    if (!combinedPath.StartsWith(rootPath, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode,
            "Vault file paths must stay inside the registered vault root.");
    }
    return combinedPath;
}
```

Dit is belangrijk. Als u een kluispad aanpast, zorg er dan voor dat dit binnen het toegestane bereik blijft, anders zal het systeem de bewerking afwijzen. Je kunt de veiligheid echt niet genoeg benadrukken.

### Beheer van Git-submodules

CodeRef Vault raadt Git-submodules aan in plaats van direct code te kopiëren:

```csharp
private static string BuildCodeRefAgentsContent()
{
    return """
    # CodeRef Vault Guide

    Repositories under `repos/` should be maintained through Git submodules
    rather than copied directly into the vault root.

    Keep this structure stable so assistants and tools can understand the vault quickly.
    """ + Environment.NewLine;
}
```

Dit brengt verschillende voordelen met zich mee: code gesynchroniseerd houden met upstream, schijfruimte besparen en het gemakkelijker maken om meerdere versies van de code te beheren. Wie wil tenslotte hetzelfde keer op keer downloaden?

### Limieten voor bestandsvoorbeelden

Om prestatieproblemen te voorkomen, beperkt het systeem de bestandsgrootte en het type:

```csharp
private const int FileEnumerationLimit = 500;
private const int PreviewByteLimit = 256 * 1024;  // 256KB
```

Als uw kluis een groot aantal bestanden of zeer grote bestanden bevat, kunnen de prestaties van de preview worden beïnvloed. In dat geval kunt u overwegen bestanden batchgewijs te verwerken of gespecialiseerde zoekhulpmiddelen te gebruiken. Soms, als iets te groot wordt, wordt het moeilijker om ermee om te gaan, niet gemakkelijker.

### Diagnostische informatie

Bij het maken van een kluis retourneert het systeem diagnostische informatie om te helpen bij het opsporen van fouten:

```csharp
List<VaultBootstrapDiagnosticDto> bootstrapDiagnostics = [];

if (IsCodeRefVaultType(normalizedType))
{
    bootstrapDiagnostics = await EnsureCodeRefBootstrapAsync(
        normalizedName,
        normalizedPhysicalPath,
        normalizedGitUrl,
        cancellationToken);
}
```

Als het maken mislukt, kunt u de diagnostische informatie bekijken om inzicht te krijgen in de specifieke oorzaak. Als er iets misgaat, is het controleren van de diagnose vaak de meest directe oplossing.

## Samenvatting

Via een uniforme opslagabstractielaag lost het Vault-systeem verschillende kernpijnpunten op bij het reproduceren van projecten in het AI-tijdperk:

- **Gecentraliseerd kennisbeheer**: alle leermiddelen worden op één plek verzameld in plaats van overal verspreid
- **Automatische AI-contextinjectie**: AI-assistenten kunnen de beschikbare leermiddelen automatisch begrijpen zonder handmatige contextconfiguratie
- **Kennisoverschrijdend hergebruik**: kennis kan worden gedeeld en hergebruikt in meerdere leerprojecten
- **Gestandaardiseerde directorystructuur**: een consistente directory-indeling verlaagt de leercurve

Deze oplossing is al gevalideerd in het HagiCode-project. Als u ook tools bouwt die verband houden met AI-ondersteunde ontwikkeling, of als u met soortgelijke problemen op het gebied van kennisbeheer wordt geconfronteerd, hoop ik dat deze ervaringen als nuttig referentiepunt kunnen dienen.

In werkelijkheid ligt de waarde van een technische oplossing niet in hoe ingewikkeld ze is, maar in de vraag of ze echte problemen oplost. Het kernidee van het Vault-systeem is heel eenvoudig: bouw een uniforme kennisopslaglaag die AI kan begrijpen. Toch is het juist deze simpele abstractie die onze ontwikkelingsefficiëntie behoorlijk heeft verbeterd.

Soms is de eenvoudige aanpak echt de beste. Achter ingewikkelde zaken schuilen immers vaak nog meer valkuilen...
