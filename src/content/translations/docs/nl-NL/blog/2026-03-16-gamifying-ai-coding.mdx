## Hoe gamificatieontwerp AI-codering leuker maakt

> Traditionele AI-coderingstools zijn eigenlijk behoorlijk krachtig; ze missen gewoon een beetje warmte. Toen we HagiCode bouwden, dachten we: als we toch code gaan schrijven, waarom maken we er dan geen spel van?

## Achtergrond

Iedereen die wel eens een AI-codeerassistent heeft gebruikt, heeft waarschijnlijk deze ervaring gehad: in eerste instantie voelt het fris en spannend, maar na een tijdje begint het te voelen alsof er iets ontbreekt. De tool zelf is krachtig, kan code genereren, automatisch aanvullen en bugfixes uitvoeren, maar... het voelt niet erg warm aan en kan na verloop van tijd eentonig en saai worden.

Dat alleen al is genoeg om je af te vragen wie elke dag naar een koud, onpersoonlijk stuk gereedschap wil staren.

Het lijkt een beetje op het spelen van een spel. Als je alleen maar een takenlijst afwerkt, zonder karaktergroei, zonder prestatie-ontgrendeling en zonder teamcoördinatie, is het al snel niet meer leuk. Mooie dingen en mensen hoeven niet bezeten te zijn om gewaardeerd te worden; hun schoonheid is op zichzelf al voldoende. Programmeertools bieden niet eens dat soort schoonheid, dus het is gemakkelijk om de moed te verliezen.

Tijdens de ontwikkeling kwamen we precies dit probleem tegen [HagiCode](https://hagicode.com). Als samenwerkingsplatform met meerdere AI-assistenten moet HagiCode gebruikers op de lange termijn betrokken houden. Maar in werkelijkheid is zelfs een geweldig hulpmiddel moeilijk vol te houden als het geen enkele emotionele band heeft.

Om dit pijnpunt op te lossen, hebben we een gedurfde beslissing genomen: maak van programmeren een spel. Niet het oppervlakkige soort met een eenvoudig puntenklassement, maar een echte gegamificeerde rollenspelervaring. De impact van die beslissing kan zelfs groter zijn dan je denkt.

Mensen hebben tenslotte een beetje ritueel nodig in hun leven.

## Over HagiCode

De ideeën die in dit artikel worden gedeeld, komen voort uit onze praktijkervaring op de [HagiCode](https://hagicode.com) project. HagiCode is een multi-AI-assistent-samenwerkingsplatform dat de samenwerking van Claude Code, Codex, Copilot, OpenCode en andere AI-assistenten ondersteunt. Als je geïnteresseerd bent in multi-AI-samenwerking of gamified programmeren, bezoek dan [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site) om meer te leren.

Er is niets bijzonder mysterieus aan. We hebben van programmeren eenvoudigweg een avontuur gemaakt.

## Waarom kiezen voor gamificatie

De essentie van gamificatie is niet alleen maar 'het toevoegen van een scorebord'. Het gaat over het bouwen van een compleet beloningssysteem, zodat gebruikers groei, prestatie en sociale erkenning kunnen voelen terwijl ze taken uitvoeren.

Het gamificatie-ontwerp van HagiCode draait om één kernidee: elke AI-assistent is een ‘held’ en de gebruiker is de aanvoerder van dit heldenteam. Jij leidt deze helden om verschillende "Dungeons" (programmeertaken) te veroveren. Onderweg doen helden ervaring op, stijgen ze in niveau, ontgrendelen ze vaardigheden en verdient jouw team ook prestaties.

Dit is geen gimmick. Het is een ontwerp dat gebaseerd is op de menselijke gedragspsychologie. Wanneer taken betekenis krijgen en voortgangsfeedback krijgen, nemen de betrokkenheid en het doorzettingsvermogen van mensen aanzienlijk toe.

Zoals het oude gezegde luidt: 'Dit gevoel kan een herinnering worden, ook al liet het ons destijds verbijsterd achter.' We brengen die emotionele ervaring in de tool, zodat programmeren niet langer alleen maar het typen van code is, maar een reis die de moeite waard is om te onthouden.

## Heldenkaraktersysteem

Hero is het kernconcept in het gamificatiesysteem van HagiCode. Each Hero represents one AI assistant. Claude Code is bijvoorbeeld een held, en Codex is ook een held.

### De drie heldenslots

Een Hero heeft drie uitrustingsslots en het ontwerp is verrassend elegant:

1. **CLI-slot (hoofdklasse)**: bepaalt de basisvaardigheid van de held, bijvoorbeeld of het Claude Code of Codex is
2. **Modelslot (secundaire klasse)**: Bepaalt welk model wordt gebruikt, zoals Claude 4.5 of Claude 4.6
3. **Stijlslot (stijl)**: Bepaalt de gedragsstijl van de held, zoals 'Fengluo Strategist' of een andere stijl

De combinatie van deze drie slots zorgt voor unieke Hero-configuraties. Net zoals apparatuur die in games wordt ingebouwd, kies je de juiste opstelling op basis van de taak. Wat het beste bij u past, is tenslotte het belangrijkste. Het leven is vergelijkbaar: veel wegen leiden naar Rome, maar sommige zijn gladder dan andere.

### Het Hero Growth-systeem

Elke held heeft zijn eigen XP en niveau:

```typescript
type HeroProgressionSnapshot = {
  currentLevel: number;                    // Current level
  totalExperience: number;                 // Total experience
  currentLevelStartExperience: number;     // Experience at the start of the current level
  nextLevelExperience: number;             // Experience required for the next level
  experienceProgressPercent: number;       // Progress percentage
  remainingExperienceToNextLevel: number;  // Experience still needed for the next level
  lastExperienceGain: number;              // Most recent experience gained
  lastExperienceGainAtUtc?: string | null; // Time when experience was gained
};
```

Niveaus zijn onderverdeeld in vier fasen en elke fase heeft een meeslepende naam:

```typescript
export const resolveHeroProgressionStage = (level?: number | null): HeroProgressionStage => {
  const normalizedLevel = Math.max(1, level ?? 1);
  if (normalizedLevel <= 100) return 'rookieSprint';      // Rookie sprint
  if (normalizedLevel <= 300) return 'growthRun';         // Growth run
  if (normalizedLevel <= 700) return 'veteranClimb';      // Veteran climb
  return 'legendMarathon';                                // Legend marathon
};
```

Van 'groentje' tot 'legende': dit groeipad geeft gebruikers een duidelijk gevoel van richting en prestatie. Het weerspiegelt de persoonlijke groei in het leven, van verwarring tot volwassenheid, en wordt hier alleen maar tastbaarder gemaakt.

### Een aangepaste held creëren

Om een held te creëren, moet je drie slots configureren:

```typescript
const heroDraft: HeroDraft = {
  name: 'Athena',
  icon: 'hero-avatar:storm-03',
  description: 'A brilliant strategist',
  executorType: AIProviderType.CLAUDE_CODE_CLI,
  slots: {
    cli: {
      id: 'profession-claude-code',
      parameters: { /* CLI-related parameters */ }
    },
    model: {
      id: 'secondary-claude-4-sonnet',
      parameters: { /* Model-related parameters */ }
    },
    style: {
      id: 'fengluo-strategist',
      parameters: { /* Style-related parameters */ }
    }
  }
};
```

Elke held heeft een unieke avatar, beschrijving en professionele identiteit, waardoor wat anders een koude AI-assistent zou zijn, meer persoonlijkheid en warmte krijgt. Wie wil er immers werken met een tool die geen karakter heeft?

## Dungeon-systeem

Een "Dungeon" is een klassiek spelconcept dat een uitdaging vertegenwoordigt waarvoor een team de oplossing vereist. In HagiCode is elke workflow een kerker.

### Hoe kerkers zijn georganiseerd

Dungeon organiseert workflows in verschillende "Dungeons":

- **Derker voor het genereren van voorstellen**: Verantwoordelijk voor het genereren van technische voorstellen
- **Voorsteluitvoeringskerker**: Verantwoordelijk voor het uitvoeren van taken in voorstellen
- **Voorstelarchiefkerker**: Verantwoordelijk voor het organiseren en archiveren van voltooide voorstellen

Elke kerker heeft zijn eigen Kapitein Held, en de kapitein wordt automatisch gekozen als de eerst ingeschakelde Held.

Dit is eigenlijk gewoon een taakverdeling, zoals in het dagelijks leven, behalve dat het in een spelmechanisme is veranderd.

### Teamsamenwerkingsmechanisme

Je kunt verschillende Hero-squadrons configureren voor verschillende kerkers:

```typescript
const dungeonRoster: HeroDungeonRoster = {
  scriptKey: 'proposal.generate',
  displayName: 'Proposal Generation',
  members: [
    { heroId: 'hero-1', name: 'Athena', executorType: 'ClaudeCode' },
    { heroId: 'hero-2', name: 'Apollo', executorType: 'Codex' }
  ]
};
```

U kunt bijvoorbeeld Athena gebruiken voor het genereren van voorstellen omdat het goed is in strategie, en Apollo voor het implementeren van code omdat het goed is in uitvoering. Op die manier kan elke held zijn sterke punten benutten. Het is net als het vormen van een band: iedereen heeft een instrument en samen creëren ze iets moois.

### Kerkerstroomcontrole

Dungeon gebruikt vast `scriptKey` waarden om verschillende workflows te identificeren:

```typescript
// Script keys map to different workflows
const dungeonScripts = {
  'proposal.generate': 'Proposal Generation',
  'proposal.execute': 'Proposal Execution',
  'proposal.archive': 'Proposal Archive'
};
```

De taakstatusstroom is: `queued` (wachten) -> `dispatching` (wordt toegewezen) -> `dispatched` (toegewezen). Het hele proces is geautomatiseerd en vereist geen handmatige tussenkomst. Dat hoort ook bij onze luie kant, want wie wil dit spul nu met de hand beheren?

## XP en niveausysteem

XP is het belangrijkste feedbackmechanisme in het gamificatiesysteem. Gebruikers verdienen XP door taken te voltooien, XP brengt helden in een hoger niveau en door een hoger niveau te bereiken worden nieuwe vaardigheden ontgrendeld, waardoor een positieve feedbacklus ontstaat.

### Manieren om XP te verdienen

In HagiCode kan XP worden verdiend door de volgende activiteiten:

- Uitvoering van code voltooien
- Hulpmiddelen succesvol aangeroepen
- Het genereren van voorstellen
- Sessiebeheerbewerkingen
- Projectactiviteiten

Elke keer dat een geldige actie wordt voltooid, krijgt de bijbehorende held XP. Net als de groei in het leven telt elke stap, alleen hier wordt die groei gekwantificeerd.

### Realtime voortgangsvisualisatie

XP en niveauvoortgang worden in realtime gevisualiseerd:

```typescript
type HeroDungeonMember = {
  heroId: string;
  name: string;
  icon?: string | null;
  executorType: PCode_Models_AIProviderType;
  currentLevel?: number;                    // Current level
  totalExperience?: number;                 // Total experience
  experienceProgressPercent?: number;       // Progress percentage
};
```

Gebruikers kunnen altijd het niveau en de voortgang van elke held zien, en die onmiddellijke feedback is de sleutel tot gamification-ontwerp. Mensen hebben feedback nodig, hoe zouden ze anders weten dat ze verbeteren?

## Prestatiesysteem

Prestaties zijn een ander belangrijk element in gamification. Ze bieden langetermijndoelen en mijlpaalgedreven tevredenheid.

### Prestatietypen

HagiCode ondersteunt meerdere soorten prestaties:

- **Codegeneratieprestaties**: genereer X regels code, genereer Y-bestanden
- **Sessiebeheerprestaties**: voltooi Z-gesprekken
- **Projectuitvoeringsprestaties**: werk in W-projecten

Deze prestaties lijken eigenlijk op mijlpalen in het leven, behalve dat we er een spelmechanisme van hebben gemaakt.

### Prestatiestaten

Prestaties hebben drie statussen:

```typescript
type AchievementStatus = 'unlocked' | 'in-progress' | 'locked';
```

De drie staten hebben duidelijke visuele verschillen:

- **Ontgrendeld**: Goudverloop met halo-effect
- **In uitvoering**: blauwe pulsanimatie
- **Vergrendeld**: grijs, met ontgrendelingsvoorwaarden weergegeven

Bij elke prestatie wordt duidelijk de triggervoorwaarde weergegeven, zodat gebruikers weten wat ze vervolgens moeten doen. Als mensen zich verloren voelen, helpt een beetje begeleiding altijd.

### Vieringseffect bij ontgrendelen

Wanneer een prestatie wordt ontgrendeld, wordt een feestanimatie geactiveerd. Dat soort positieve bekrachtiging geeft gebruikers het bevredigende gevoel van "Ik heb het gedaan" en motiveert hen om door te gaan. Kleine beloningen in het leven werken op dezelfde manier: ze kunnen klein zijn, maar het geluk kan lang duren.

## Gevechtsrapport Dagelijks gevechtsrapport

Battle Report is een van de kenmerkende functies van HagiCode. Aan het einde van elke dag genereert het een gevechtsrapport op volledig scherm.

### Rapporteer inhoud

Battle Report geeft de volgende informatie weer:

```typescript
type HeroBattleReport = {
  reportDate: string;
  summary: {
    totalHeroCount: number;        // Total number of Heroes
    activeHeroCount: number;       // Number of active Heroes
    totalBattleScore: number;      // Total battle score
    mvp: HeroBattleHero;           // Most valuable Hero
  };
  heroes: HeroBattleHero[];        // Detailed data for all Heroes
};
```

- Totale teamscore
- Aantal actieve helden
- Aantal gereedschapsoproepen
- Totale werktijd
- MVP (meest waardevolle held)
- Gedetailleerde kaart voor elke held

### MVP-hoogtepuntweergave

De MVP is de best presterende Held van de dag en wordt in het rapport uitgelicht. Dit is niet alleen datastatistiek, maar een vorm van eer en erkenning. Wie wil er immers niet erkend worden?

### Gedetailleerde heldenkaarten

Elke heldenkaart bevat:

- Niveau vooruitgang
- XP gewonnen
- Aantal executies
- Gebruikstijd

Deze statistieken helpen gebruikers duidelijk te begrijpen hoe het team presteert. Het zien van de resultaten van je eigen inspanningen is op zichzelf al bevredigend.

## Technische implementatie

HagiCode's gamification system uses a modern technology stack and design patterns. Er is niets bijzonder magisch aan; we hebben gewoon gereedschap gekozen dat bij de klus past.

### Technologiestapelkeuzes

```typescript
// React + TypeScript for the frontend
import React from 'react';

// Framer Motion for animations
import { AnimatePresence, motion } from 'framer-motion';

// Redux Toolkit for state management
import { useAppDispatch, useAppSelector } from '@/store';

// shadcn/ui for UI components
import { Dialog, DialogContent } from '@/components/ui/dialog';
```

Framer Motion verzorgt alle animatie-effecten, shadcn/ui levert de fundamentele UI-componenten en Redux Toolkit beheert de complexe gamificatiestatus. Goed gereedschap maakt goed werk.

### Gegamificeerd UI-ontwerpsysteem

HagiCode uses a Glassmorphism + Tech Dark design style:

```css
/* Primary gradient */
background: linear-gradient(135deg, #22C55E 0%, #25c2a0 50%, #06b6d4 100%);

/* Glass effect */
backdrop-filter: blur(12px);

/* Glow effect */
background: radial-gradient(circle at center, rgba(34, 197, 94, 0.15) 0%, transparent 70%);
```

The green gradient combined with glassmorphism creates a technical, futuristic atmosphere. Visual beauty is part of the user experience too.

### Animatie-effecten

Framer Motion is used to create smooth entrance animations:

```typescript
<motion.div
  animate={{ opacity: 1, y: 0 }}
  initial={{ opacity: 0, y: 18 }}
  transition={{ duration: 0.35, ease: 'easeOut', delay: index * 0.08 }}
  className="card"
>
  {/* Card content */}
</motion.div>
```

Each card enters one after another with a delay of 0.08 seconds, creating a fluid visual effect. Vloeiende animaties verbeteren de ervaring. Dat deel is moeilijk te betwisten.

### Gegevenspersistentie

Gamification data is stored using the Grain storage system to ensure state consistency. Even fine-grained data like accumulated Hero XP can be persisted accurately. No one wants to lose the experience they worked hard to earn.

## Praktische gids

### Creëer je eerste held

Creating your first Hero is actually quite simple:

1. Ga naar de Hero-beheerpagina
2. Klik op de knop 'Held maken'
3. Configureer de drie slots (CLI, Model, Style)
4. Geef de held een naam en beschrijving
5. Bewaar het en je eerste held is geboren

It is like meeting a new friend: you give them a name, learn what makes them special, and then head off on an adventure together.

### Bouw een kerkerteam

Een team samenstellen is ook eenvoudig:

1. Ga naar de Dungeon-beheerpagina
2. Choose the dungeon you want to configure, such as "Proposal Generation"
3. Selecteer leden uit je Heldenlijst
4. Het systeem selecteert automatisch de eerst ingeschakelde held als kapitein
5. Sla de configuratie op

Dit is simpelweg het proces van het vormen van een team, net zoals het bouwen van een team in het echte leven, waarbij iedereen zijn eigen rol heeft.

### Bekijk het dagrapport

Aan het einde van elke dag kun je het gevechtsrapport van de dag bekijken:

1. Klik op de knop 'Gevechtsrapport'
2. Bekijk de werkresultaten van de dag op volledig scherm
3. Bekijk de MVP en de gedetailleerde gegevens voor elke held
4. Deel het met teamleden als je wilt

Dit is ook een soort ritueel, een manier om te zien hoeveel moeite je vandaag hebt gedaan en hoe ver je nog van je doel verwijderd bent.

## Opmerkingen en beste praktijken

### Prestatie-optimalisatie

Gebruik `React.memo` om onnodige herhalingen te voorkomen:

```typescript
const HeroCard = React.memo(({ hero }: { hero: HeroDungeonMember }) => {
  // Component implementation
});
```

Prestaties zijn ook belangrijk. Niemand wil een laggy-tool gebruiken.

### Beweging kan sierlijk degraderen

Detecteer de bewegingsvoorkeurinstellingen van de gebruiker en bied een vereenvoudigde ervaring voor bewegingsgevoelige gebruikers:

```typescript
const prefersReducedMotion = useReducedMotion();
const duration = prefersReducedMotion ? 0 : 0.35;
```

Niet iedereen houdt van animatie, en het respecteren van gebruikersvoorkeuren maakt deel uit van een goed ontwerp.

### Achterwaartse compatibiliteit

Houd `legacyIds` om migratie van oudere versies te ondersteunen:

```typescript
type HeroDungeonMember = {
  heroId: string;
  legacyIds?: string[];  // Supports legacy ID mapping
  // ...
};
```

Niemand wil gegevens verliezen alleen vanwege een versie-upgrade.

### Ondersteuning van internationalisering

Gebruik i18n-vertaalsleutels voor alle tekst om meertalige ondersteuning eenvoudig te maken:

```typescript
const displayName = t(`dungeon.${scriptKey}`, { defaultValue: displayName });
```

Taal mag nooit een belemmering vormen voor het gebruik van het product.

## Samenvatting

Gamification is niet alleen een eenvoudig puntenklassement, maar een compleet beloningssysteem. Via het Hero-systeem, Dungeon-systeem, XP- en levelsysteem, prestatiesysteem en Battle Report transformeert HagiCode programmeerwerk in een heroïsche reis vol avontuur.

De kernwaarde van dit systeem ligt in:

- **Emotionele connectie**: Geef koude AI-assistenten een persoonlijkheid
- **Positieve feedback**: Elke actie levert onmiddellijke feedback op
- **Langetermijndoelen**: Niveaus en prestaties bieden een groeipad
- **Teamidentiteit**: Een gevoel van samenwerking binnen Dungeon-teams
- **Eer en erkenning**: Battle Report en MVP-showcases

Gamification-ontwerp maakt programmeren niet langer saai, maar een interessant avontuur. Tijdens het voltooien van codeertaken ervaren gebruikers ook het plezier van karaktergroei, teamsamenwerking en het ontgrendelen van prestaties, wat de retentie en activiteit verbetert.

In de kern is programmeren al een scheppingsdaad. We hebben het creatieve proces net iets leuker gemaakt.
