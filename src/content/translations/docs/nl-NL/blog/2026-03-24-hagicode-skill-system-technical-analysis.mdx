## Technische analyse van het HagiCode-vaardigheidssysteem: het bouwen van een schaalbaar AI-vaardigheidsbeheerplatform

> Dit artikel gaat diepgaand in op de architectuur en implementatie van het Skill-managementsysteem in het HagiCode-project, waarbij de technische details achter vier kernmogelijkheden worden behandeld: lokaal mondiaal beheer, marktplaatszoeken, intelligente aanbevelingen en vertrouwd providerbeheer.

## Achtergrond

Op het gebied van AI-coderingsassistenten is het altijd een kernvraag geweest hoe de grenzen van AI-mogelijkheden kunnen worden verlegd. Claude Code zelf is al sterk in codeondersteuning, maar verschillende ontwikkelteams en verschillende technologieën hebben vaak gespecialiseerde mogelijkheden nodig voor specifieke scenario's, zoals het afhandelen van Docker-implementaties, database-optimalisatie of het genereren van frontend-componenten. Dat is precies waar een Skill-systeem bijzonder belangrijk wordt.

Tijdens de ontwikkeling van het HagiCode-project kwamen we een soortgelijke uitdaging tegen: hoe laten we Claude Code nieuwe professionele vaardigheden 'leren' zoals een mens dat zou doen, terwijl we toch een solide gebruikerservaring en goede technische onderhoudbaarheid behouden? Dit probleem is op zijn eigen manier zowel moeilijk als eenvoudig. Rond die vraag hebben we een compleet Skill management systeem ontworpen en geïmplementeerd.

In dit artikel worden de technische architectuur en de kernimplementatie van het systeem in detail besproken. Het is bedoeld voor ontwikkelaars die geïnteresseerd zijn in AI-uitbreidbaarheid en integratie van opdrachtregeltools. Het kan nuttig voor je zijn, maar misschien ook niet, maar het staat nu tenminste opgeschreven.

## Over HagiCode

De aanpak die in dit artikel wordt gedeeld, komt voort uit onze praktijkervaring in de [HagiCode](https://hagicode.com) project. HagiCode is een open-source AI-coderingsassistentproject dat is ontworpen om ontwikkelingsteams te helpen de technische efficiëntie te verbeteren. De stack van het project omvat ASP.NET Core, het gedistribueerde Orleans-framework, een TanStack Start + React-frontend en het subsysteem Skill Management dat in dit artikel wordt geïntroduceerd.

De GitHub-repository is [HagiCode-org/site](https://github.com/HagiCode-org/site). Als je de technische benadering in dit artikel waardevol vindt, geef het dan gerust een ster. Meer sterren hebben immers de neiging om de stemming te verbeteren.

## Overzicht systeemarchitectuur

Het Skill-systeem maakt gebruik van een frontend-backend gescheiden architectuur. Daar is niets bijzonder mysterieus aan.

**Frontend** gebruikt TanStack Start + React om de gebruikersinterface te bouwen, waarbij Redux Toolkit de status beheert. De vier belangrijkste mogelijkheden zijn rechtstreeks gekoppeld aan vier tabbladcomponenten: Lokale vaardigheden, Vaardighedengalerij, Intelligente aanbevelingen en Vertrouwde providers. Uiteindelijk gaat het bij het ontwerp vooral om het verbeteren van de gebruikerservaring.

**Backend** is gebaseerd op ASP.NET Core + ABP Framework en gebruikt Orleans Grain voor gedistribueerd statusbeheer. De online API-client verpakt de `IOnlineApiClient` interface om te communiceren met de externe vaardighedencatalogusservice.

Het algemene architectonische principe is om de uitvoering van opdrachten te scheiden van de bedrijfslogica. Via het adapterpatroon worden de implementatiedetails van de uitvoering van de npm/npx-opdracht verborgen in onafhankelijke modules. Niemand wil tenslotte commandoregelaanroepen verspreid over de codebase.

## Kerncapaciteit 1: Lokaal mondiaal management

Lokaal mondiaal management is de meest elementaire module. Het is verantwoordelijk voor het opsommen van geïnstalleerde vaardigheden en het ondersteunen van verwijderingsoperaties. Er is hier niets overdreven ingewikkelds; het gaat er vooral om dat je de basis goed doet.

### Technische aanpak

De implementatie leeft erin `LocalSkillsTab.tsx` en `LocalSkillCommandAdapter.cs`. Het kernidee is om de `npx skills` commando, parseer de JSON-uitvoer en converteer deze naar interne datastructuren. Het klinkt eenvoudig, en dat is het in de praktijk meestal ook.

```csharp
public async Task<IReadOnlyList<LocalSkillInventoryResponseDto>> GetLocalSkillsAsync(
    CancellationToken cancellationToken = default)
{
    var result = await _commandAdapter.ListGlobalSkillsAsync(cancellationToken);
    return result.Skills.Select(skill => new LocalSkillInventoryResponseDto
    {
        Name = skill.Name,
        Version = skill.Version,
        Source = skill.Source,
        InstalledPath = skill.InstalledPath,
        Description = skill.Description
    }).ToList();
}
```

De datastroom is heel duidelijk: de frontend stuurt een verzoek -> `SkillGalleryAppService` ontvangt het -> `LocalSkillCommandAdapter` voert de opdracht npx uit -> het JSON-resultaat wordt geparseerd -> een DTO wordt geretourneerd. Elke stap volgt op natuurlijke wijze uit de vorige.

Bij het verwijderen van vaardigheden wordt gebruik gemaakt van de `npx skills remove -g <skillName> -y` commando, en het systeem handelt automatisch afhankelijkheden en opschoningen af. Metagegevens van de installatie worden opgeslagen in `managed-install.json` in de vaardighedenmap, waarbij informatie wordt vastgelegd zoals installatietijd en bronversie voor latere updates en audits. Sommige dingen zijn gewoon de moeite waard om vast te leggen.

### Installatiestroom in detail

Voor het installeren van vaardigheden zijn verschillende gecoördineerde stappen nodig. In werkelijkheid is het niet bijzonder ingewikkeld:

```csharp
public async Task<SkillInstallResultDto> InstallAsync(
    SkillInstallRequestDto request,
    CancellationToken cancellationToken = default)
{
    // 1. Normalize the installation reference
    var normalized = _referenceNormalizer.Normalize(
        request.SkillId,
        request.Source,
        request.SkillSlug,
        request.Version);

    // 2. Check prerequisites
    await _prerequisiteChecker.CheckAsync(cancellationToken);

    // 3. Acquire installation lock
    using var installLock = await _lockProvider.AcquireAsync(normalized.SkillId);

    // 4. Execute installation command
    var result = await _installCommandRunner.ExecuteAsync(
        new SkillInstallCommandExecutionRequest
        {
            Command = $"npx skills add {normalized.FullReference} -g -y",
            Timeout = TimeSpan.FromMinutes(4)
        },
        cancellationToken);

    // 5. Persist installation metadata
    await _metadataStore.WriteAsync(normalized.SkillPath, request);

    return new SkillInstallResultDto { Success = result.Success };
}
```

Hier worden verschillende belangrijke ontwerppatronen gebruikt: de **referentienormalisator** converteert verschillende invoerformaten, zoals `tanweai/pua` en `@opencode/docker-skill`, in een uniforme interne representatie; het **installatievergrendelingsmechanisme** zorgt ervoor dat slechts één installatiehandeling tegelijk voor dezelfde vaardigheid kan worden uitgevoerd; en **streaming-uitvoer** stuurt de voortgang van de installatie in realtime naar de frontend via door de server verzonden gebeurtenissen, zodat gebruikers terminalachtige logboeken kunnen bekijken terwijl ze plaatsvinden.

Uiteindelijk hebben al deze patronen maar één doel: het systeem eenvoudiger te gebruiken en te onderhouden.

## Kernmogelijkheid 2: Zoeken op de marktplaats

Via Marketplace Search kunnen gebruikers vaardigheden uit de community ontdekken en installeren. Het vermogen van één persoon is altijd beperkt; collectieve kennis gaat veel verder.

### Technische aanpak

De zoekfunctie is afhankelijk van de online API `https://api.hagicode.com/v1/skills/search`. Om de reactiesnelheid te verbeteren, implementeert het systeem caching. Cache lijkt een beetje op geheugen: als je nuttige dingen bij je hebt, hoef je de volgende keer niet zo hard na te denken.

```csharp
private async Task<IReadOnlyList<SkillGallerySkillDto>> SearchCatalogAsync(
    string query,
    CancellationToken cancellationToken,
    IReadOnlySet<string>? allowedSources = null)
{
    var cacheKey = $"skill_search:{query}:{string.Join(",", allowedSources ?? Array.Empty<string>())}";

    if (_memoryCache.TryGetValue(cacheKey, out var cached))
        return (IReadOnlyList<SkillGallerySkillDto>)cached!;

    var response = await _onlineApiClient.SearchAsync(
        new SearchSkillsRequest
        {
            Query = query,
            Limit = _options.LimitPerQuery,
        },
        cancellationToken);

    var results = response.Skills
        .Where(skill => allowedSources is null || allowedSources.Contains(skill.Source))
        .Select(skill => new SkillGallerySkillDto { ... })
        .ToList();

    _memoryCache.Set(cacheKey, results, TimeSpan.FromMinutes(10));
    return results;
}
```

Zoekresultaten ondersteunen filteren op vertrouwde bronnen, zodat gebruikers alleen vaardigheidsbronnen zien die ze vertrouwen. Seed-query's zoals `popular` en `recent` worden gebruikt om de catalogus te initialiseren, zodat gebruikers de aanbevolen populaire vaardigheden kunnen zien wanneer ze deze voor de eerste keer openen. Eerste indrukken zijn nog steeds belangrijk.

## Kerncapaciteit 3: Intelligente aanbevelingen

Intelligente aanbevelingen vormen het meest complexe onderdeel van het systeem. Ze kunnen automatisch de meest geschikte vaardigheden aanbevelen op basis van de huidige projectcontext. Hoe complex het ook is, het is nog steeds de moeite waard om te bouwen.

### Aanbevelingsstroom

De volledige aanbevelingsstroom is verdeeld in vijf fasen:

```
1. Build project context
   ↓
2. AI generates search queries
   ↓
3. Search the online catalog in parallel
   ↓
4. AI ranks the candidates
   ↓
5. Return the recommendation list
```

Eerst analyseert het systeem kenmerken zoals de technologiestapel van het project, de programmeertalen en de domeinstructuur om een 'projectprofiel' op te bouwen. Dat profiel lijkt een beetje op een cv, waarin de belangrijkste kenmerken van het project worden vastgelegd.

Vervolgens wordt een AI Grain gebruikt om gerichte zoekopdrachten te genereren. Dit ontwerp is eigenlijk best interessant: in plaats van de AI rechtstreeks te vragen: "Welke vaardigheden moet ik aanbevelen?", vragen we hem eerst na te denken over "Welke zoektermen zullen waarschijnlijk relevante vaardigheden vinden?" Soms is de manier waarop je de vraag stelt belangrijker dan het antwoord zelf:

```csharp
var queryGeneration = await aiGrain.GenerateSkillRecommendationQueriesAsync(
    projectContext,      // Project context
    locale,              // User language preference
    maxQueries,           // Maximum number of queries
    effectiveSearchHero); // AI model selection
```

Vervolgens worden deze zoekopdrachten parallel uitgevoerd om een lijst met vaardigheden van kandidaten te verzamelen. Parallelle verwerking is uiteindelijk slechts een manier om tijd te besparen.

Ten slotte rangschikt een andere AI Grain de vaardigheden van de kandidaten. Bij deze stap wordt rekening gehouden met factoren zoals de relevantie van vaardigheden voor het project, de vertrouwensstatus en de historische voorkeuren van gebruikers:

```csharp
var ranking = await aiGrain.RankSkillRecommendationsAsync(
    projectContext,
    candidates,
    installedSkillNames,
    locale,
    maxRecommendations,
    effectiveRankingHero);

response.Items = MergeRecommendations(projectContext, candidates, ranking, maxRecommendations);
```

### Terugvalmechanisme

AI-modellen kunnen traag reageren of tijdelijk niet beschikbaar zijn. Zelfs de beste systemen struikelen soms. Om die reden bevat het systeem een deterministisch terugvalmechanisme: wanneer de AI-dienst niet beschikbaar is, gebruikt het een op regels gebaseerd heuristisch algoritme om aanbevelingen te genereren, zoals het afleiden van waarschijnlijk vereiste vaardigheden uit afhankelijkheden in `package.json`.

Simpel gezegd is dit terugvalmechanisme eenvoudigweg een back-upplan voor het systeem.

## Kerncapaciteit 4: Beheer van vertrouwde providers

Met Trusted Provider Management kunnen gebruikers bepalen welke vaardigheidsbronnen als betrouwbaar worden beschouwd. Vertrouwen is nog steeds iets dat gebruikers zelf moeten kunnen definiëren.

### Overeenkomende regels

Vertrouwde providers ondersteunen twee overeenkomstenregels: **exacte overeenkomst** (`exact`) en **voorvoegselovereenkomst** (`prefix`).

```csharp
public static TrustedSkillProviderResolutionSnapshot Resolve(
    TrustedSkillProviderSnapshot snapshot,
    string source)
{
    var normalizedSource = Normalize(source);

    foreach (var entry in snapshot.Entries.OrderBy(e => e.SortOrder))
    {
        if (!entry.IsEnabled) continue;

        foreach (var rule in entry.MatchRules)
        {
            bool isMatch = rule.MatchType switch
            {
                TrustedSkillProviderMatchRuleType.Exact
                    => string.Equals(normalizedSource, Normalize(rule.Value),
                        StringComparison.OrdinalIgnoreCase),
                TrustedSkillProviderMatchRuleType.Prefix
                    => normalizedSource.StartsWith(Normalize(rule.Value) + "/",
                        StringComparison.OrdinalIgnoreCase),
                _ => false
            };

            if (isMatch)
                return new TrustedSkillProviderResolutionSnapshot
                {
                    IsTrustedSource = true,
                    ProviderId = entry.ProviderId,
                    DisplayName = entry.DisplayName
                };
        }
    }

    return new TrustedSkillProviderResolutionSnapshot { IsTrustedSource = false };
}
```

Ingebouwde vertrouwde providers zijn onder meer bekende organisaties en projecten zoals Vercel, Azure, anthropics, Microsoft en browsergebruik. Aangepaste providers kunnen worden toegevoegd via configuratiebestanden door een provider-ID, weergavenaam, badgelabel, matchingregels en meer op te geven. De wereld is zo groot dat alleen het vertrouwen op een paar ingebouwde systemen nooit genoeg zou zijn.

### Volharding Implementatie

De vertrouwde configuratie wordt gehandhaafd met behulp van een Orleans Grain:

```csharp
public class TrustedSkillProviderGrain : Grain<TrustedSkillProviderState>,
    ITrustedSkillProviderGrain
{
    public async Task UpdateConfigurationAsync(TrustedSkillProviderSnapshot snapshot)
    {
        State.Snapshot = snapshot;
        await WriteStateAsync();
    }

    public Task<TrustedSkillProviderSnapshot> GetConfigurationAsync()
    {
        return Task.FromResult(State.Snapshot);
    }
}
```

Het voordeel van deze aanpak is dat configuratiewijzigingen automatisch worden gesynchroniseerd op alle knooppunten, zonder dat de caches handmatig hoeven te worden vernieuwd. Automatisering gaat er uiteindelijk om dat mensen zich minder zorgen hoeven te maken.

## Belangrijk technisch ontwerp

### Adapterpatroon voor opdrachtuitvoering

Het Skill-systeem moet verschillende npx-opdrachten uitvoeren. Als die logica overal verspreid zou zijn, zou de code snel moeilijk te onderhouden worden. Daarom hebben we een adapterinterface ontworpen. Uiteindelijk bestaan er ontwerppatronen om code gemakkelijker te onderhouden:

```csharp
public interface ISkillInstallCommandRunner
{
    Task<SkillInstallCommandExecutionResult> ExecuteAsync(
        SkillInstallCommandExecutionRequest request,
        CancellationToken cancellationToken = default);
}
```

Verschillende commando's hebben verschillende executor-implementaties, maar ze implementeren allemaal dezelfde interface, waardoor testen en vervangen eenvoudig is.

### SSE-streaminguitvoer

De voortgang van de installatie wordt in realtime naar de frontend gepusht via door de server verzonden gebeurtenissen:

```csharp
public async Task InstallWithProgressAsync(
    SkillInstallRequestDto request,
    IServerStreamWriter<SkillInstallProgressEventDto> stream,
    CancellationToken cancellationToken)
{
    var process = new Process
    {
        StartInfo = new ProcessStartInfo
        {
            FileName = "npx",
            Arguments = $"skills add {request.FullReference} -g -y",
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false
        }
    };

    process.OutputDataReceived += async (sender, e) =>
    {
        await stream.WriteAsync(new SkillInstallProgressEventDto
        {
            EventType = "output",
            Data = e.Data ?? string.Empty
        });
    };

    process.Start();
    process.BeginOutputReadLine();
    await process.WaitForExitAsync(cancellationToken);
}
```

Aan de frontend kunnen gebruikers terminalachtige uitvoer in realtime zien, wat de ervaring zeer intuïtief maakt. Realtime feedback zorgt ervoor dat mensen zich op hun gemak voelen.

## Praktische gids

### Een communityvaardigheid installeren

Neem het installeren van de `pua` vaardigheid als voorbeeld (het is een populaire gemeenschapsvaardigheid):

1. Open de vaardighedenlade en ga naar de `Skill Gallery` tabblad
2. Kom binnen `pua` in het zoekvak
3. Klik op het zoekresultaat om de vaardigheidsdetails te bekijken
4. Klik op de `Install` knop
5. Schakel over naar de `Local Skills` tabblad om te bevestigen dat de installatie is geslaagd

Het installatiecommando is `npx skills add tanweai/pua -g -y`, en het systeem verwerkt alle details automatisch. Er zijn niet echt zoveel stappen als je ze één voor één neemt.

### Een aangepaste vertrouwde bron toevoegen

Als uw team een eigen opslagplaats voor vaardigheden heeft, kunt u deze toevoegen als een vertrouwde bron:

```yaml
providerId: "my-team"
displayName: "My Team Skills"
badgeLabel: "MyTeam"
isEnabled: true
sortOrder: 100
matchRules:
  - matchType: "prefix"
    value: "my-team/"
  - matchType: "exact"
    value: "my-team/special-skill"
```

Op deze manier worden alle vaardigheden van uw team weergegeven met een vertrouwde badge, waardoor gebruikers deze gemakkelijker kunnen installeren. Etiketten en signalen zorgen ervoor dat mensen zich zelfverzekerder voelen.

### Basisprincipes van vaardighedenontwikkeling

Voor het maken van een aangepaste vaardigheid is de volgende structuur vereist:

```
my-skill/
├── SKILL.md          # Skill metadata (YAML front matter)
├── index.ts          # Skill entry point
├── agents/           # Supported agent configuration
└── references/       # Reference resources
```

Een voorbeeld `SKILL.md` formaat:

```yaml
---
name: my-skill
description: A brief description of what this skill does
---

# My Skill

Detailed documentation...
```

## Opmerkingen

1. **Netwerkvereisten**: voor het zoeken en installeren van vaardigheden is toegang vereist `api.hagicode.com` en het npm-register
2. **Node.js-versie**: Node.js 18 of hoger wordt aanbevolen
3. **Toestemmingsvereisten**: algemene npm-installatierechten zijn vereist
4. **Gelijktijdigheidscontrole**: er kan slechts één installatie- of verwijderingsbewerking tegelijk voor dezelfde vaardigheid worden uitgevoerd
5. **Time-outinstellingen**: de standaardtime-out voor installatie is 4 minuten, maar complexe scenario's vereisen mogelijk aanpassing

Deze aantekeningen zijn er uiteindelijk om de zaken soepel te laten verlopen.

## Conclusie

Dit artikel introduceerde de volledige implementatie van het Skill-managementsysteem in het HagiCode-project. Via een frontend-backend gescheiden architectuur, het adapterpatroon, op Orleans gebaseerd gedistribueerd statusbeheer en gerelateerde technieken levert het systeem:

- **Lokaal mondiaal beheer**: een uniforme interface voor vaardighedenbeheer, gebouwd door middel van wrappen `npx skills` opdrachten
- **Zoeken op de markt**: snelle ontdekking van gemeenschapsvaardigheden via de online API en caching-mechanismen
- **Intelligente aanbevelingen**: AI-aangedreven vaardigheidsaanbevelingen op basis van de projectcontext
- **Vertrouwensbeheer**: een flexibel configuratiesysteem waarmee gebruikers de vertrouwensgrenzen kunnen controleren

Deze ontwerpbenadering is niet alleen van toepassing op vaardigheidsmanagement. Het is ook nuttig als referentie voor elk scenario waarin opdrachtregelprogramma's moeten worden geïntegreerd, terwijl lokale opslag en online services in evenwicht moeten worden gebracht.
