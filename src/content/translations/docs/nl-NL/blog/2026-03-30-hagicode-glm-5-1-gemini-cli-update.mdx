## Volledige GLM-5.1-ondersteuning en Gemini CLI-integratie: HagiCode's pad van multi-model evolutie

> Dit artikel introduceert twee belangrijke recente updates voor het HagiCode-platform: volledige ondersteuning voor het Zhipu AI GLM-5.1-model en de succesvolle integratie van Gemini CLI als de tiende Agent CLI. Samen versterken deze updates de multi-modelmogelijkheden en het multi-CLI-ecosysteem van het platform verder.

:::note[Tijdcontextmededeling]
In dit artikel wordt de voortgang van de multi-CLI-uitbreiding vastgelegd zoals deze was op het moment van uitgave. Raadpleeg voor het huidige, altijd ondersteunde bereik [AI Agent CLI-installatie](/nl-NL/related-software-installation/ai-agent-cli/) en de [Installatiehandleiding](/nl-NL/installation); de `IFlowCli` hier vermeld vertegenwoordigt slechts een historische configuratie op dat moment en maakt geen deel uit van de huidige actieve ondersteuningsscope.
:::

## Achtergrond

De tijd gaat echt snel. De ontwikkeling van grote taalmodellen is dit voorjaar als een bamboestok in opkomst. Nog niet zo lang geleden juichten we nog steeds voor 'een AI die code kan schrijven'. Nu bevinden we ons al in een tijdperk van samenwerking op meerdere modellen en integratie van meerdere tools. Is dat spannend? Misschien. Wat ontwikkelaars nodig hebben is immers nooit alleen de tool zelf, maar het gemak waarmee ze zich aan verschillende scenario's kunnen aanpassen en flexibel kunnen schakelen wanneer dat nodig is.

Als AI-ondersteund codeerplatform heeft HagiCode onlangs twee belangrijke ontwikkelingen verwelkomd: ten eerste de volledige integratie van Zhipu AI's GLM-5.1-model; ten tweede de officiële toevoeging van Gemini CLI als de tiende ondersteunde Agent CLI. Deze twee updates klinken misschien niet wereldschokkend, maar ze zijn ongetwijfeld goed nieuws voor de voortdurende rijping van het platform.

GLM-5.1 is het nieuwste vlaggenschipmodel van Zhipu AI. Vergeleken met GLM-5.0 biedt het een sterkere redenering, een dieper begrip van de code en een vlottere aanroep van tools. Belangrijker nog is dat dit het eerste GLM-model is dat beeldinvoer ondersteunt. Wat betekent dat? Het betekent dat gebruikers de AI rechtstreeks naar een screenshot kunnen laten kijken in plaats van te worstelen om het probleem in woorden te beschrijven. Als je eenmaal van dat gemak gebruik hebt gemaakt, begrijp je meteen de waarde ervan.

Tegelijkertijd wordt via de `HagiCode.Libs.Providers` architectuur heeft HagiCode Gemini CLI met succes in het platform geïntegreerd. Dit is nu de tiende Agent-CLI. Eerlijk gezegd brengt het bereiken van dit punt een bescheiden gevoel van voldoening met zich mee.

Het is ook vermeldenswaard dat HagiCode's functie voor het uploaden van afbeeldingen gebruikers in staat stelt rechtstreeks met AI te communiceren via schermafbeeldingen. Zelfs met GLM 4.7 werkt het platform nog steeds goed en heeft het al geholpen bij het voltooien van veel belangrijke bouwtaken. Wat GLM-5.1 betreft, het gaat uiteraard nog een stap verder.

## Over HagiCode

De aanpak die in dit artikel wordt gedeeld, komt voort uit onze praktijkervaring in de [HagiCode](https://hagicode.com) project. HagiCode is een open-source AI-ondersteund coderingsplatform dat is ontworpen om ontwikkelaars te voorzien van een flexibele en krachtige AI-programmeerassistent via een multi-model, multi-CLI-architectuur. Projectopslagplaats: [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site)

## Multi-CLI-architectuurontwerp

Een van de sterke punten van HagiCode is de ondersteuning voor meerdere AI-programmeer-CLI-tools via een uniforme abstractielaag. Het voordeel van dit ontwerp is eigenlijk heel simpel: nieuwe tools kunnen binnenkomen, oude tools kunnen blijven en de codebase verandert niet in chaos. Eerlijk gezegd wil iedereen zo graag dat het leven werkt.

### AIProviderType opsomming

Het platform definieert ondersteunde CLI-providertypen via de `AIProviderType` opsomming:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI (new)
}
```

Zoals je ziet voegt Gemini CLI zich als tiende lid bij deze familie. Elke CLI heeft zijn eigen specifieke kenmerken en gebruiksscenario's, zodat gebruikers flexibel kunnen kiezen op basis van hun behoeften. Er leiden immers vele wegen naar Rome; sommige zijn gewoon gemakkelijker dan andere.

### Architectuur van de provider

`HagiCode.Libs.Providers` biedt een uniforme providerinterface die elke CLI-integratie gestandaardiseerd en beknopt maakt. Neem Gemini CLI als voorbeeld:

```csharp
public class GeminiProvider : ICliProvider<GeminiOptions>
{
    private static readonly string[] DefaultExecutableCandidates = ["gemini", "gemini-cli"];
    private const string ManagedBootstrapArgument = "--acp";

    public string Name => "gemini";
    public bool IsAvailable => _executableResolver.ResolveFirstAvailablePath(DefaultExecutableCandidates) is not null;
}
```

De voordelen van dit ontwerp zijn:
- Voor het integreren van een nieuwe CLI is slechts één Provider-klasse nodig
- Uniform levenscyclusbeheer en sessiepooling
- Geautomatiseerde aliasresolutie en uitvoerbare detectie

Simpel gezegd: dit ontwerp maakt van ingewikkelde dingen eenvoudiger en maakt het leven een stuk eenvoudiger.

### Register van aanbieders

Het Provider Register verwerkt automatisch de aliastoewijzing en registratie:

```csharp
if (provider is GeminiProvider)
{
    registry.Register(provider.Name, provider, ["gemini-cli"]);
    continue;
}
```

Dit betekent dat gebruikers met beide Gemini CLI kunnen aanroepen `gemini` of `gemini-cli`en het systeem zal het automatisch herkennen. Het is als een vriend met zowel een formele naam als een bijnaam: hoe dan ook, mensen weten wie je bedoelt.

## Ondersteuning voor GLM-5.1-modellen

GLM-5.1 is het nieuwste vlaggenschipmodel van Zhipu AI en HagiCode heeft er volledige ondersteuning voor voltooid.

### Catalogus secundaire beroepen

HagiCode beheert alle ondersteunde modellen via de Catalogus Bijberoepen. Hier is de configuratie voor de GLM-serie:

| Model-ID | Naam | Ondersteunt afbeelding | Compatibele CLI-families |
|----------|------|----------------|-------------------------|
| `glm-4.7` | GLM 4.7 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GLM 5 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM5 Turbo | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (verouderd) | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | **waar** | claude, codebuddy, hermes, qoder, kiro |

De belangrijkste kenmerken van GLM-5.1 kunnen als volgt worden samengevat:
- Een zelfstandige versie-ID zonder oude bagage
- Het eerste GLM-model dat beeldinvoer ondersteunt
- Sterker redeneren en codebegrip
- Brede multi-CLI-compatibiliteit

### GLM-5.1 versus GLM-5.0

Op codeniveau wordt hier het belangrijkste verschil tussen GLM-5.1 en GLM-5.0 weergegeven:

```csharp
// GLM-5.0 (Legacy) - contains special retention logic
private const string Glm50CodebuddySecondaryProfessionId = "secondary-glm-5-codebuddy";
private const string Glm50CodebuddyModelValue = "glm-5.0";

// GLM-5.1 - standalone new model identifier
private const string Glm51SecondaryProfessionId = "secondary-glm-5-1";
private const string Glm51ModelValue = "glm-5.1";
```

GLM-5.0 draagt het label "Legacy" omdat het een oude versie-ID is die behouden blijft voor achterwaartse compatibiliteit. GLM-5.1 is daarentegen een gloednieuwe, zelfstandige versie zonder historische last. Sommige dingen blijven in het verleden; anderen reizen lichter en bewegen sneller.

### Configureer GLM-5.1

Hier is een configuratievoorbeeld voor het gebruik van GLM-5.1 in HagiCode:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

## Functie voor het uploaden van afbeeldingen

De beeldondersteuning van HagiCode wordt geïmplementeerd via de `SupportsImage` eigendom op `SecondaryProfession`:

```csharp
public class HeroSecondaryProfessionSettingDto
{
    public bool SupportsImage { get; set; }
}
```

In de Catalogus Bijberoepen ziet de GLM-5.1-configuratie er als volgt uit:

```json
{
  "id": "secondary-glm-5-1",
  "supportsImage": true
}
```

Dit betekent dat gebruikers direct screenshots kunnen uploaden voor AI-analyse, zoals:
- Screenshots van foutmeldingen
- Problemen in een UI-scherm
- Gegevensvisualisatiegrafieken
- Resultaten van code-uitvoering

Het is niet langer nodig om alles handmatig te beschrijven; upload gewoon de schermafbeelding. Het gemak van deze functie is duidelijk zodra u er gebruik van maakt. Soms zegt één blik meer dan een lange uitleg.

## Gemini CLI-integratie

Als de tiende Agent CLI is Gemini CLI geïntegreerd in HagiCode via de standaard Provider-architectuur.

### Configuratie-opties

Gemini CLI ondersteunt een uitgebreide reeks configuratieopties:

```csharp
public class GeminiOptions
{
    public string? ExecutablePath { get; set; }
    public string? WorkingDirectory { get; set; }
    public string? SessionId { get; set; }
    public string? Model { get; set; }
    public string? AuthenticationMethod { get; set; }
    public string? AuthenticationToken { get; set; }
    public Dictionary<string, string?> AuthenticationInfo { get; set; }
    public Dictionary<string, string?> EnvironmentVariables { get; set; }
    public string[] ExtraArguments { get; set; }
    public TimeSpan? StartupTimeout { get; set; }
    public CliPoolSettings? PoolSettings { get; set; }
}
```

Deze opties omvatten alles, van basisinstellingen tot geavanceerde functies, waardoor gebruikers de flexibiliteit hebben om de CLI volgens hun eigen behoeften te configureren. De workflow van iedereen is anders, dus een beetje flexibiliteit is altijd welkom.

### ACP-communicatieprotocol

Gemini CLI ondersteunt het ACP (Agent Communication Protocol), de uniforme CLI-communicatiestandaard van HagiCode. Via ACP kunnen verschillende CLI's op een consistente manier met het platform communiceren, waardoor het integratiewerk aanzienlijk wordt vereenvoudigd. Kortom, het standaardiseert de ingewikkelde onderdelen zodat iedereen makkelijker kan werken.

## HagiCode's eigen bouwpraktijk

Over de praktijk gesproken: het beste voorbeeld is de eigen build-workflow van het HagiCode-platform. Het ontwikkelingsproces van HagiCode heeft al volledig gebruik gemaakt van de mogelijkheden van AI.

### Werkt goed, zelfs met GLM 4.7

Het platformontwerp van HagiCode is goed geoptimaliseerd, zodat het zelfs met GLM 4.7 nog steeds een goede ontwikkelervaring kan bieden. Het platform heeft al geholpen bij het voltooien van meerdere belangrijke bouwprojecten, waaronder:
- Integratie van meerdere CLI-providers
- Implementatie van de functie voor het uploaden van afbeeldingen
- Documentatie genereren en inhoud publiceren

Dat is eigenlijk een goede zaak. Niet iedereen heeft altijd het nieuwste nodig. Wat het beste bij je past, is vaak het belangrijkst.

### GLM-5.1 levert meer met minder inspanning

Na het upgraden naar GLM-5.1 worden deze mogelijkheden nog sterker:
- Beter begrip van de code, waardoor heen en weer communicatie wordt verminderd
- Nauwkeurigere afhankelijkheidsanalyse, die onmiddellijk in de goede richting wijst
- Efficiëntere foutdiagnose, waardoor problemen sneller worden gelokaliseerd
- Ondersteuning voor beeldinvoer, waardoor probleembeschrijvingen worden versneld

Het is alsof je overstapt van een fiets naar een auto. Je kunt nog steeds dezelfde bestemming bereiken, maar de snelheid en het comfort zijn niet hetzelfde.

## Best practices voor multi-CLI-integratie

`HagiCode.Libs.Providers` biedt een uniform mechanisme voor registratie en gebruik:

```csharp
services.AddHagiCodeLibs();

var gemini = serviceProvider.GetRequiredService<ICliProvider<GeminiOptions>>();
var codebuddy = serviceProvider.GetRequiredService<ICliProvider<CodebuddyOptions>>();
var hermes = serviceProvider.GetRequiredService<ICliProvider<HermesOptions>>();
```

Dit ontwerp voor afhankelijkheidsinjectie houdt het gebruik tussen verschillende CLI's zeer beknopt en maakt het testen en bespotten van eenheden ook handiger. Schone code is een manier om verantwoordelijk te zijn voor jezelf.

### Opmerkingen

Bij daadwerkelijk gebruik zijn er een aantal zaken waarmee u rekening moet houden:

1. **API-sleutelconfiguratie**: Zorg ervoor dat `ANTHROPIC_AUTH_TOKEN` correct is ingesteld, anders kan het model niet worden opgeroepen
2. **Modelbeschikbaarheid**: GLM-5.1 moet worden ingeschakeld door de overeenkomstige modelaanbieder
3. **Afbeeldingsfunctie**: Alleen modellen met `supportsImage: true` kan gebruik maken van het uploaden van afbeeldingen
4. **CLI-installatie**: Zorg ervoor dat u Gemini CLI gebruikt `gemini` of `gemini-cli` bevindt zich in het systeemPATH

Dit kunnen kleine details zijn, maar kleine details die slecht worden behandeld, kunnen grote problemen veroorzaken, dus het is de moeite waard om er aandacht aan te besteden.

## Conclusie

Met volledige ondersteuning voor GLM-5.1 en de succesvolle integratie van Gemini CLI versterkt HagiCode zijn mogelijkheden als multi-model, multi-CLI AI-programmeerplatform verder. Deze updates geven gebruikers niet alleen meer keuzemogelijkheden, maar demonstreren ook de toekomstgerichte architectuur en schaalbaarheid van HagiCode.

De beeldondersteuning van GLM-5.1, gecombineerd met HagiCode's screenshot-uploadfunctie, maakt het mogelijk om de AI "van het beeld te laten begrijpen" en verlaagt de kosten voor het beschrijven van problemen aanzienlijk. En met ondersteuning voor tien CLI's kunnen gebruikers flexibel de AI-programmeerassistent kiezen die het beste bij hun voorkeuren en scenario's past. Meer keuze is bijna altijd goed.

Het belangrijkste is dat HagiCode's eigen bouwpraktijk bewijst dat het platform zelfs met GLM 4.7 al goed kan draaien en complexe taken kan voltooien, terwijl een upgrade naar GLM-5.1 de ontwikkelingsefficiëntie verder kan verbeteren. Zo is het leven vaak ook: je hebt niet altijd het allerbeste nodig, alleen wat bij je past. Natuurlijk, als wat bij je past nog beter kan worden, des te beter.

Als je geïnteresseerd bent in een multi-model, multi-CLI AI-programmeerplatform, probeer HagiCode dan eens: open source, gratis en nog steeds in ontwikkeling. Het proberen kost niets, en het kan precies zijn wat u nodig heeft.
