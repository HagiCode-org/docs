## Hagicode en GLM-5.1 Multi-CLI-integratiegids

:::note[Tijdcontextmededeling]
Dit artikel documenteert het multi-CLI-integratieontwerp en de modeltoewijzing zoals deze bestonden op het moment van publicatie. Raadpleeg voor het huidige, altijd ondersteunde bereik [AI Agent CLI-installatie](/nl-NL/related-software-installation/ai-agent-cli/) en de [Installatiehandleiding](/nl-NL/installation); de `IFlowCli` hier vermeld vertegenwoordigt slechts een historische configuratie op dat moment en maakt geen deel uit van de huidige actieve ondersteuningsscope.
:::

## Achtergrond

In het Hagicode-project kunnen gebruikers kiezen uit meerdere CLI-tools om AI-programmeerassistenten aan te sturen, waaronder Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI en meer. Deze CLI-tools zijn op zichzelf al algemene AI-programmeertools, maar via de abstractielaag van Hagicode kunnen ze flexibel verbinding maken met verschillende AI-modelaanbieders.

Zhipu AI (ZAI) biedt een interface die compatibel is met de Anthropic Claude API, waardoor deze CLI-tools rechtstreeks gebruik kunnen maken van binnenlandse modellen uit de GLM-serie. Onder hen is GLM-5.1 Zhipu's nieuwste versie van een groottaalmodel, met aanzienlijke verbeteringen ten opzichte van GLM-5.0.

## Hagicode's CLI-abstractiearchitectuur

Hagicode definieert 11 CLI-providertypen via de `AIProviderType` enum, dat betrekking heeft op reguliere AI-programmering CLI-tools:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Elke CLI heeft een corresponderende modelparameterconfiguratie en ondersteunt de `model` en `reasoning` parameters:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],
        [AIProviderType.CodexCli] = ["model", "reasoning"],
        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],
        [AIProviderType.HermesCli] = ["model", "reasoning"],
        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini does not support the reasoning parameter
        // ...
    };
```

## GLM-5.1 gebruiken in verschillende CLI's

### 1. Claude-code CLI + GLM-5.1

Claude Code CLI is een van de belangrijkste CLI's van Hagicode en wordt geconfigureerd via het Hero-configuratiesysteem:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Overeenkomstig `HeroEquipmentCatalogItem` configuratie:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI is de meest flexibele CLI en ondersteunt het specificeren van elk model in de `provider/model` formaat:

**Methode 1: Gebruik het ZAI-providervoorvoegsel**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Methode 2: Gebruik de model-ID rechtstreeks**
```json
{
  "model": "glm-5.1"
}
```

**Methode 3: Frontend-configuratie-UI**

In `HeroModelEquipmentForm.tsx`, OpenCode CLI heeft een speciale tijdelijke aanduiding-hint:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Gebruikers kunnen invoeren:
```
zai/glm-5.1
glm-5.1
```

**OpenCode CLI-model parseerlogica**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI heeft speciale logica voor oudere verwerking:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Opmerking**: Codebuddy behoudt een speciale behandeling voor GLM-5.0 en maakt geen gebruik van verouderde normalisatie:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## Verbetervoordelen van GLM-5.1

Vergeleken met GLM-5.0 brengt GLM-5.1 de volgende belangrijke verbeteringen met zich mee:

### 1. Beter redeneervermogen

Volgens de officiële release-informatie van Zhipu omvatten de verbeteringen in GLM-5.1:

- **Beter begrip van code**: Nauwkeurigere analyse van complexe codestructuren
- **Langer contextbegrip**: Ondersteunt een langere gesprekscontext
- **Verbeterde tooloproepen**: Hoger slagingspercentage voor MCP-tooloproepen
- **Uitvoerstabiliteit**: vermindert willekeur en hallucinaties

### 2. Uitgebreide multi-CLI-compatibiliteit

GLM-5.1 omvat alle reguliere CLI's die door Hagicode worden ondersteund:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Opmerkingen

### 1. API-sleutelconfiguratie

Zorg ervoor dat de `ANTHROPIC_AUTH_TOKEN` omgevingsvariabele is correct ingesteld. Het is de vereiste referentie voor elke CLI om verbinding te maken met het model.

### 2. Beschikbaarheid van modellen

GLM-5.1 moet worden ingeschakeld door de overeenkomstige modelaanbieder:
- Het Zhipu AI ZAI-platform ondersteunt dit standaard
- Voor Alibaba Cloud DashScope is mogelijk een afzonderlijke applicatie vereist

### 3. OpenCode CLI-formaat

Bij gebruik van de `provider/model` formaat, zorg ervoor dat de provider-ID correct is:
- Zhipu-AI: `zai` of `zhipuai`
- Alibaba-wolk: `aliyun` of `dashscope`

### 4. Redeneringsparameter

- `high` wordt aanbevolen voor de beste resultaten voor het genereren van code
- Gemini CLI ondersteunt de redeneringsparameter niet en negeert deze configuratie automatisch

## Conclusie

Via een uniforme abstractielaag maakt Hagicode flexibele integratie mogelijk tussen GLM-5.1 en meerdere CLI's. Ontwikkelaars kunnen de CLI-tool kiezen die het beste bij hun voorkeuren en gebruiksscenario's past, en vervolgens het nieuwste GLM-5.1-model gebruiken via een eenvoudige configuratie.

Als Zhipu's nieuwste modelversie biedt GLM-5.1 duidelijke verbeteringen ten opzichte van GLM-5.0:
- Een onafhankelijke versie-ID zonder verouderde lasten
- Sterker redeneren en codebegrip
- Brede multi-CLI-compatibiliteit
- Flexibele configuratie van redeneerniveaus

Als de juiste omgevingsvariabelen en Hero-apparatuur zijn geconfigureerd, kunnen gebruikers de kracht van GLM-5.1 volledig benutten in verschillende CLI-omgevingen.
