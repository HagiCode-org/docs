## .NET-codebescherming in de praktijk: van verduistering tot bescherming van virtuele machines

> In dit artikel wordt uitgelegd hoe u een meerlaagse codebeschermingsstrategie in .NET-projecten implementeert, waarbij het volledige traject wordt bestreken, van eenvoudige verduistering tot professionele bescherming van virtuele machines.

## Achtergrond

Bij de ontwikkeling van .NET-applicaties is het beschermen van kerncode zoals licentievalidatie, bedrijfslogica en gevoelige configuratie tegen decompilatie en reverse engineering een onderwerp dat je eerlijk gezegd niet kunt vermijden. Naarmate het .NET-ecosysteem volwassener werd, hebben ontwikkelaars toegang gekregen tot een reeks beveiligingsopties, van ingebouwde verduisteringskenmerken tot professionele, op virtualisatie gebaseerde beveiligingstools.

Als complex meertalig monorepo-project omvat HagiCode desktopapplicaties, build-systemen en licentiebeheermogelijkheden. De code bevat onvermijdelijk licentievalidatielogica, gevoelige configuratie zoals API-sleutels en product-ID's, en bedrijfskritische logica. Die delen hebben serieuze bescherming nodig, omdat niemand wil dat hun harde werk zo gemakkelijk aan het licht komt.

Dit artikel deelt de codebeschermingsaanpak die we daadwerkelijk hebben toegepast in het HagiCode-project en vat het volledige traject samen, van de vroege valkuilen tot de latere optimalisatie. Hopelijk levert het je wat bruikbare ideeën op.

## Over HagiCode

[HagiCode](https://hagicode.com) is een open source AI-coderingsassistentproject dat bedoeld is om ontwikkelaars een intelligente programmeerervaring te bieden. Het project maakt gebruik van een monorepo-architectuur en onderhoudt tegelijkertijd een VSCode-extensie, backend AI-services, een platformonafhankelijke desktopclient en meer. Die meertalige, multi-platform complexiteit maakt codebescherming tot een technische uitdaging die we het hoofd moeten bieden.

De aanpak die in dit artikel wordt gedeeld, is het resultaat van echt vallen en opstaan tijdens de ontwikkeling van HagiCode. Als je wilt zien hoe we deze technische problemen hebben opgelost, lees dan verder. Misschien vindt u een paar onverwachte afhaalrestaurants.

## Kerninhoud

### 1. Het ingebouwde verduisteringskenmerk van Microsoft

.NET Framework biedt een ingebouwde `[ObfuscationAttribute]`, de meest elementaire en meest gebruikte codeverduisteringsmarkering. Dit attribuut leeft in de `System.Reflection` naamruimte en stelt u in staat basislijnbescherming op code toe te passen zonder dat u daarvoor tools van derden hoeft te gebruiken.

**Kernfuncties**:
- `Feature` eigenschap: specificeert de verduisteringsfunctie, zoals `"ultra"` (hoge verduistering) of `"all"` (volledige verduistering)
- `Exclude` eigendom: `true` betekent uitsluiten van verduistering, en `false` betekent verduistering toepassen
- Kan worden toegepast op klassen, methoden, eigenschappen en andere typeleden

In het HagiCode-project kun je zien dat het als volgt wordt gebruikt:

```csharp
[Obfuscation(Feature = "ultra", Exclude = false)]
public async Task<LicenseValidationResult?> ValidateLicenseAsync(...)
```

De **voordelen** van deze aanpak liggen vrij voor de hand:
- Geen extra afhankelijkheden; het is kant-en-klaar in .NET Framework ingebouwd
- Kan worden herkend en verwerkt door verduisteringstools van derden
- Vergroot de omvang van de gecompileerde assembly niet significant

Dat gezegd hebbende, het heeft ook beperkingen. Het is slechts een markering en het daadwerkelijke verduisteringsresultaat is afhankelijk van de implementatie van het hulpmiddel. Het kan geen beveiliging op beveiligingsniveau voor virtuele machines bieden.

### 2. VMP (virtuele machinebescherming)

VMP is een professionele codebeschermingstool die bescherming op hoog niveau biedt door code te compileren in instructies voor virtuele machines. In tegenstelling tot eenvoudige naamverwarring transformeert VMP codelogica in een vorm die conventionele decompilers niet kunnen reconstrueren.

**Classificatie van beschermingsniveau**:

| Niveau | Virtualisatie | Mutatie | Anti-foutopsporing | String-codering | Gebruiksscenario's |
|------|--------|------|--------|------------|----------|
| HOOG | vol | hoog | ingeschakeld | ingeschakeld | Licentievalidatie, gelijktijdigheid van sessies, gevoelige constanten |
| GEMIDDELD | gedeeltelijk | middelmatig | ingeschakeld | ingeschakeld | Bedrijfslogica, domeinmodellen |
| LAAG | geen | laag | uitgeschakeld | uitgeschakeld | Nutsklassen, niet-kritieke code |

Het HagiCode-project definieert een declaratief attribuutsysteem voor het markeren van code die bescherming nodig heeft:

```csharp
// High-priority protection
[VmProtect(VmProtectionPriority.High, Reason = "Contains license verification logic")]
public class KeygenClient { ... }

// Exclude from protection
[VmExclude(Reason = "Public API that must remain unchanged")]
public class PublicApi { ... }

// Inherited protection
[VmProtect(Priority.High, ProtectDerived = true)]
public class BaseLicenseValidator { ... }
```

### 3. Build-Time-beschermingsstrategie

VMP-bescherming is niet alleen van belang tijdens runtime. Het moet ook worden geautomatiseerd als onderdeel van de build-pijplijn, omdat het veel te vervelend zou zijn om dit handmatig te doen. Het bouwsysteem van HagiCode ondersteunt verschillende modi:

- **Native Windows-modus**: Roep de VMProtect-tool rechtstreeks op
- **Linux Docker-containermodus**: voer VMP uit in een container om compatibiliteitsproblemen tussen platforms op te lossen
- **Kenmerkscannen**: Ontdek automatisch beveiligingsmarkeringen in code
- **Validatiemechanisme**: Bevestig dat de beveiliging met succes is toegepast

Alles bij elkaar maken deze mogelijkheden het proces veel eenvoudiger te beheren.

## Oplossing

### 1. Het ingebouwde verduisteringskenmerk van Microsoft gebruiken

Toepassen `ObfuscationAttribute` direct in code:

```csharp
using System.Reflection;

[Obfuscation(Feature = "ultra", Exclude = false)]
public class LicenseService
{
    [Obfuscation(Feature = "ultra", Exclude = false)]
    public async Task<bool> ValidateLicenseAsync(string key)
    {
        // License validation logic
    }

    [Obfuscation(Feature = "flow", Exclude = false)]
    private string DecryptToken(string encrypted)
    {
        // Decryption logic
    }
}
```

Soms moet u testassemblages toegang geven tot interne leden terwijl u de productiecode toch veilig houdt:

```csharp
// AssemblyInfo.cs
[assembly: InternalsVisibleTo("HagiCode.Application.Tests")]
[assembly: InternalsVisibleTo("DynamicProxyGenAssembly2")] // for Moq
```

Dit maakt het testen veel handiger, omdat de code nog steeds goed getest moet worden.

### 2. Aangepaste attribuutdefinities voor VMP-beveiliging

Maak aangepaste beveiligingskenmerken om het VMP-gedrag te controleren:

```csharp
using System;

[AttributeUsage(AttributeTargets.Class | AttributeTargets.Method | AttributeTargets.Property)]
public class VmProtectAttribute : Attribute
{
    public VmProtectionPriority Priority { get; set; }
    public string? Reason { get; set; }
    public bool ProtectDerived { get; set; }
}

[AttributeUsage(AttributeTargets.Class | AttributeTargets.Method | AttributeTargets.Property)]
public class VmExcludeAttribute : Attribute
{
    public string? Reason { get; set; }
}

public enum VmProtectionPriority
{
    None = 0,
    Low = 1,
    Medium = 2,
    High = 3
}
```

Aangepaste kenmerken zijn vaak gemakkelijker om mee te werken, omdat ze rechtstreeks uw eigen beveiligingsvereisten weerspiegelen.

### 3. VMP-configuratiebestand

```yaml
# vmp_config.yml
protection:
  priority_mode: "attribute"  # Attribute-based priority
  default_level: "medium"

tools:
  - name: "vmprotect"
    path: "C:\\Program Files\\VMProtect Ultimate\\VMProtect.exe"

protection_levels:
  high:
    virtualization: "full"
    mutation: "high"
    anti_debug: true
    anti_dump: true
    encrypt_strings: true
    encrypt_resources: true

  medium:
    virtualization: "partial"
    mutation: "medium"
    anti_debug: true
    encrypt_strings: true

  low:
    virtualization: "none"
    mutation: "low"
    anti_debug: false
```

Een duidelijkere configuratie maakt het systeem later veel gemakkelijker te onderhouden.

## Praktische gids

### 1. Beschermingspraktijken voor kritieke componenten

Volgens de codebeschermingsspecificatie van HagiCode moeten de volgende componenten bescherming met HOGE prioriteit gebruiken:

```csharp
// Production constants - must be encrypted and protected by VMP
[VmProtect(VmProtectionPriority.High, Reason = "Production constants")]
public static class ProductionConstants
{
    // Encrypted string accessor, protected by VMP
    [VmProtect(VmProtectionPriority.High)]
    public static string GetLicenseServerUrl(IOptions<LicenseOptions> options) => ...;
}

// License validation logic
[VmProtect(VmProtectionPriority.High, Reason = "License verification logic")]
public class KeygenClient : IKeygenClient
{
    [Obfuscation(Feature = "ultra", Exclude = false)]
    public async Task<LicenseValidationResult?> ValidateLicenseAsync(...) { ... }
}

// Machine fingerprint service
[VmProtect(VmProtectionPriority.High)]
public class MachineFingerprintService : IMachineFingerprintService { ... }
```

Kritieke code verdient een sterkere bescherming, omdat het blootleggen van de kernlogica echte problemen zou veroorzaken.

### 2. String-encryptie en runtime-decryptie

Versleutel tekenreeksen tijdens het bouwen en decodeer ze tijdens runtime:

```csharp
public static class StringDecryption
{
    [VmProtect(VmProtectionPriority.High, Reason = "CRITICAL SECURITY")]
    public static string DecryptString(byte[] encryptedData, byte[] key, byte[] iv)
    {
        using var aes = Aes.Create();
        aes.Key = key;
        aes.IV = iv;

        using var decryptor = aes.CreateDecryptor();
        using var ms = new MemoryStream(encryptedData);
        using var cs = new CryptoStream(ms, decryptor, CryptoStreamMode.Read);
        using var reader = new StreamReader(cs);

        return reader.ReadToEnd();
    }
}

// Production constant accessor (lazy loading + caching)
public static class ProductionConstants
{
    private static string? _cachedLicenseServerUrl;

    public static string GetLicenseServerUrl(IOptions<LicenseOptions> options)
    {
        if (_cachedLicenseServerUrl == null)
        {
            var encrypted = GetEncryptedLicenseServerUrl();
#if DEBUG
            _cachedLicenseServerUrl = options.Value.PrimaryServer.Url;
#else
            _cachedLicenseServerUrl = StringDecryption.DecryptString(
                encrypted,
                GetEncryptionKey(),
                GetEncryptionIV());
#endif
        }
        return _cachedLicenseServerUrl;
    }
}
```

Deze stap is belangrijk omdat gevoelige informatie nooit in leesbare tekst mag worden achtergelaten.

### 3. VMP-beveiligingsverificatie

Na de build moet u verifiëren of de beveiliging met succes is toegepast; anders kunt u er niet zeker van zijn dat het daadwerkelijk werkt:

```csharp
// Example verification script
public bool VerifyProtection(string assemblyPath)
{
    // 1. Check the VMP signature
    var bytes = File.ReadAllBytes(assemblyPath);
    var vmpSignature = Encoding.ASCII.GetBytes("VMProtect");
    if (bytes.Any(b => vmpSignature.Contains(b)))
    {
        return true;
    }

    // 2. Check for file size changes (the protected file is usually larger)
    var originalInfo = new FileInfo(assemblyPath.Replace(".dll", ".bak"));
    if (originalInfo.Exists)
    {
        var sizeRatio = (double)new FileInfo(assemblyPath).Length / originalInfo.Length;
        return sizeRatio > 1.1;
    }

    return false;
}
```

Verificatie is altijd de moeite waard, omdat problemen anders onopgemerkt kunnen doorglippen.

### 4. Opmerkingen en kanttekeningen

Er zijn hier verschillende valkuilen die speciale aandacht verdienen:

1. **Verberg niet alle code**: openbare API's, interfacedefinities en DTO-klassen hebben doorgaans geen bescherming nodig. Overmatige verduistering kan de prestaties en foutopsporing negatief beïnvloeden. Het HagiCode-project heeft dit op de harde manier geleerd.

2. **Bescherm sleuteltoegang**: Methoden die encryptiesleutels ophalen moeten hetzelfde of een hoger beschermingsniveau krijgen dan de gecodeerde gegevens zelf; anders verliest de hele opzet zijn waarde.

3. **Balans tussen testen en productie**: DEBUG-builds moeten encryptie overslaan om ontwikkeling en foutopsporing eenvoudiger te maken, terwijl RELEASE-builds volledige bescherming moeten mogelijk maken. Vergeet niet om ze te scheiden met voorwaardelijke compilatie, zoals `#if DEBUG`.

4. **Overweeg de Docker-omgeving**: het uitvoeren van VMP op Linux vereist een containerbenadering om compatibiliteit van tools te garanderen. HagiCode gebruikt een Wine + VMP-containeroplossing om het platformonafhankelijke probleem op te lossen.

5. **Verificatie is verplicht**: nadat de build is voltooid, moet u verifiëren dat de beveiliging met succes is toegepast. Anders kan er nog steeds gevoelige code worden vrijgegeven, en de eerder getoonde verificatiecode bestaat precies voor dit doel.

## Conclusie

Met deze meerlaagse beschermingsstrategie heeft HagiCode een uitgebreid codebeveiligingssysteem gebouwd dat zich uitstrekt van basisverduistering tot bescherming van virtuele machines:

- **Laag 1**: Gebruik `ObfuscationAttribute` voor het markeren van de basislijn en het geven van tips voor tools van derden
- **Laag 2**: Gebruik aangepast `VmProtectAttribute` verklaringen om beschermingsintentie en prioriteit uit te drukken
- **Laag 3**: Gebruik VMP-beveiliging voor virtuele machines om kritieke code om te zetten in onomkeerbare instructies voor virtuele machines
- **Laag 4**: Automatisch scannen en bescherming toepassen tijdens de build, en vervolgens het resultaat verifiëren

Deze aanpak is bestand tegen gewone decompilatietools en is tegelijkertijd beter bestand tegen geavanceerde reverse engineering-aanvallen. Als je een .NET-applicatie bouwt die codebescherming nodig heeft, hoop ik dat dit je op zijn minst een nuttig referentiepunt geeft.
