Als u Hermes wilt installeren en gebruiken, bestaat de kortste weg uit slechts drie stappen:

1. Voer de officiële installatieopdracht uit
2. Start de CLI in uw terminal met `hermes`
3. Als je het in Feishu wilt blijven gebruiken, configureer dan `hermes gateway setup`

Dit artikel probeert niet alle mogelijkheden van Hermes in één keer uit te leggen. In plaats daarvan helpt het u eerst de belangrijkste beginnerslus te voltooien: installeer het, laat het draaien, ga het gebruiken en verbind het vervolgens met een van de meest voorkomende scenario's voor berichtenplatforms.

## Wat Hermes is en voor wie het is

Hermes Agent is een AI-agent die u kunt gebruiken vanaf een lokale terminal of via een gateway voor berichtenplatforms.

Voor de meeste ontwikkelaars heeft het twee gemeenschappelijke toegangspunten:

- **CLI**: Type `hermes` in uw terminal om rechtstreeks naar de interactieve interface te gaan.
- **Berichtengateway**: uitvoeren `hermes gateway`en chat ermee vanaf platforms zoals Feishu, Telegram, Discord en Slack.

Als het uw doel op dit moment is om snel aan de slag te gaan, keer dan de volgorde niet om. Begin in plaats daarvan met dit pad:

- Installeer eerst Hermes
- Controleer eerst of het werkt vanuit de CLI
- Bepaal vervolgens of je een berichtenplatform wilt koppelen

Dit maakt het gemakkelijker om problemen te diagnosticeren en is geschikter voor mensen die Hermes voor het eerst gebruiken.

## Wat u moet weten voordat u Hermes installeert

Volgens de README van Hermes ondersteunt het officiële snelle installatiepad deze omgevingen:

- Linux
- macOS
- WSL2
- Android via Termux

### Opmerkingen voor Windows-gebruikers

Hermes ondersteunt momenteel niet rechtstreeks draaien op native Windows. Als u Windows gebruikt, is de aanbevolen aanpak om eerst WSL2 te installeren en vervolgens de installatieopdracht binnen WSL2 uit te voeren.

Het is het beste om dit in het begin duidelijk te maken, omdat veel installatiefouten niet worden veroorzaakt door de opdracht zelf, maar door het gebruik van een niet-ondersteunde runtime-omgeving.

## Hoe Hermes te installeren

Het snelle installatiecommando in de Hermes README is:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Met deze opdracht wordt het officiële installatiescript uitgevoerd en worden platformspecifieke initialisatiestappen afgehandeld.

### Herlaad uw shell na de installatie

Zodra de installatie is voltooid, laadt u eerst uw shell-omgeving opnieuw. Het meest voorkomende commando is:

```bash
source ~/.bashrc
```

Als u zsh gebruikt, kunt u het volgende gebruiken:

```bash
source ~/.zshrc
```

### Hoe u kunt bevestigen dat Hermes correct is geïnstalleerd

De meest directe manier om dit te controleren is door het volgende uit te voeren:

```bash
hermes
```

Als u extra bevestiging wilt dat uw configuratie en afhankelijkheden werken, kunt u ook het volgende uitvoeren:

```bash
hermes doctor
```

`hermes doctor` is vooral handig in deze situaties:

- De opdracht gedraagt zich abnormaal na de installatie
- Modelconfiguratie mislukt
- De gateway start niet
- U weet niet zeker of de afhankelijkheden van uw omgeving volledig zijn

## Hoe u Hermes voor de eerste keer gebruikt

### Begin met de CLI

Als je zo snel mogelijk wilt bevestigen dat Hermes werkt, is de eenvoudigste methode:

```bash
hermes
```

Hiermee wordt de interactieve Hermes CLI gelanceerd. Voor beginnende Hermes-gebruikers is dit ook het meest aanbevolen startpunt, omdat u de meest essentiële zaken eerst kunt verifiëren:

- Of de opdracht daadwerkelijk beschikbaar is
- Of de huidige modelconfiguratie goed werkt
- Of de terminaltoolchain correct werkt
- Of de interactiestijl aansluit bij wat jij nodig hebt

### Deze opdrachten zijn voldoende voor uw eerste installatieronde

De Hermes README somt verschillende hoogfrequente opdrachten op, en samen vormen ze een praktisch pad voor eerste gebruik:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Als je niet zeker weet wat ze allemaal doen, onthoud ze dan als volgt:

- `hermes model`: modellen kiezen of wisselen
- `hermes tools`: momenteel beschikbare tools bekijken en configureren
- `hermes config set`: specifieke configuratie-items wijzigen
- `hermes setup`: voer de volledige initialisatiewizard één keer uit
- `hermes update`: update Hermes
- `hermes doctor`: problemen oplossen

Voor beginners is de meest praktische volgorde meestal:

1. Rennen `hermes model` eerst
2. Als u alle algemene opties in één keer wilt configureren, voer dan uit `hermes setup`

## De twee meest voorkomende manieren om Hermes te gebruiken

### 1. Gebruik Hermes in de terminal als dagelijkse ontwikkelingsassistent

De CLI-modus past goed bij deze scenario's:

- Stel direct vragen terwijl u lokaal code schrijft
- Inspecteer projecten, bewerk bestanden en voer opdrachten uit
- Voer eenmalig foutopsporings- of beoordelingswerk uit
- Werk continu samen in de huidige werkmap

Het grootste voordeel is dat dit de kortste weg is: geen extra platformintegratie, geen botconfiguratie die je vooraf moet afhandelen, en het is de beste manier om je eerste set gebruiksgewoonten op te bouwen.

### 2. Gebruik Hermes via een berichtenplatform

Als je met Hermes wilt chatten op platforms zoals Feishu, Telegram of Discord, moet je de berichtengateway gebruiken.

De meest voorkomende invoeropdrachten zijn:

```bash
hermes gateway setup
hermes gateway
```

Specifiek:

- `hermes gateway setup` wordt gebruikt voor interactieve platformconfiguratie
- `hermes gateway` wordt gebruikt om het gatewayproces te starten

Volgens de officiële documentatie is de gateway een uniform achtergrondproces dat uw geconfigureerde platforms verbindt, sessies beheert en functies zoals cron-jobs afhandelt.

## Met Feishu als voorbeeld: hoe u Hermes kunt verbinden met een berichtenplatform

Als het grootste deel van je dagelijkse werk in Feishu gebeurt, dan is Feishu/Lark een heel natuurlijke manier om Hermes te gebruiken.

### Het minimaal haalbare integratiepad

De officiële documentatie beveelt dit invoercommando aan voor Feishu/Lark:

```bash
hermes gateway setup
```

Nadat u het hebt uitgevoerd, kiest u eenvoudigweg **Feishu / Lark** in de wizard.

De Feishu-documentatie beschrijft twee verbindingsmodi:

- `websocket`: aanbevolen
- `webhook`: optioneel

Als Hermes op uw laptop, werkstation of privéserver draait, gebruikt u `websocket` de eerste is meestal eenvoudiger omdat u geen openbare callback-URL hoeft vrij te geven.

### Als u het handmatig configureert, kent u in ieder geval deze variabelen

Als u de wizard niet gebruikt en de configuratie handmatig schrijft, vermeldt de Feishu-documentatie deze kernvariabelen:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Twee ervan verdienen speciale aandacht:

- `FEISHU_ALLOWED_USERS`: aanbevolen, dus niet iedereen die de bot kan bereiken, kan deze direct gebruiken
- `FEISHU_HOME_CHANNEL`: hiermee kunt u vooraf een thuischat definiëren om cron-resultaten of standaardmeldingen te ontvangen

### Waarom Hermes soms niet antwoordt in Feishu-groepschats

Dit detail is gemakkelijk over het hoofd te zien: in Feishu-groepschats reageert Hermes niet standaard op elk bericht.

In de officiële documentatie staat duidelijk:

- In directe berichten reageert Hermes op berichten
- In groepschats moet je dit expliciet doen `@` de bot voordat deze het bericht zal verwerken

Als je een Feishu-gesprek als thuiskanaal wilt instellen, kun je dit ook in de chat gebruiken:

```text
/set-home
```

Of definieer het vooraf in de configuratie:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## De Hermes-commando's die beginners als eerste moeten onthouden

Of u Hermes nu in de CLI of op een berichtenplatform gebruikt, het onthouden van de volgende opdrachten is al voldoende om aan de slag te gaan:

- `/new` of `/reset`: start een nieuwe sessie
- `/model`: bekijk of schakel het model
- `/retry`: probeer de vorige beurt opnieuw
- `/undo`: de vorige interactie ongedaan maken
- `/compress`: de context handmatig comprimeren
- `/help`: hulp bekijken

Als u Hermes voornamelijk op een berichtenplatform gebruikt, onthoud er dan nog één:

- `/sethome` of `/set-home`: stel de huidige chat in als thuiskanaal

Deze opdrachten omvatten de meest voorkomende handelingen in de beginnersfase: opnieuw opstarten, aanpassen, terugdraaien, controleren en doorgaan.

## Veelgestelde vragen

### Kan Windows Hermes rechtstreeks installeren?

Nee. In de huidige officiële documentatie staat duidelijk dat native Windows niet wordt ondersteund en dat WSL2 wordt aanbevolen.

### Wat moet ik doen als ik typ `hermes` gebeurt er niets na installatie?

U kunt het probleem het beste in deze volgorde oplossen:

1. Herlaad eerst je shell, bijvoorbeeld met `source ~/.bashrc`
2. Rennen `hermes` opnieuw
3. Als het nog steeds abnormaal is, ga dan rennen `hermes doctor`

### Waarom antwoordt de bot niet in een Feishu-groep?

Controleer eerst deze drie dingen:

- Of jij `@` noemde Hermes in de groep
- Of `FEISHU_ALLOWED_USERS` beperkt de huidige gebruiker
- Of het huidige groepschatbeleid het verwerken van groepsberichten toestaat

Volgens de officiële Feishu-documentatie is het expliciet gebruiken van een @mention vereist in groepschatscenario's.
