## Handleiding voor het creëren van een Border Light Sweep-animatie-effect

> Hoe bouw je met pure CSS dat belangrijke element op dat gebruikers in één oogopslag opmerken? Het is eigenlijk niet zo moeilijk. De truc is om gewoon een klein rond pad te nemen. In dit artikel zal ik je laten zien hoe je een animatie met randlichtbewegingen helemaal opnieuw kunt bouwen, en ook een paar van de valkuilen delen die we tegenkwamen tijdens het bouwen van HagiCode.

## Achtergrond

Als je aan de frontend werkt, heb je deze ervaring waarschijnlijk al eerder gehad: een productmanager komt langs met de uitdrukking "Dit is absoluut eenvoudig" en zegt: "Kunnen we een speciaal effect toevoegen aan deze lopende taak, zodat gebruikers het onmiddellijk kunnen herkennen?"

Je zegt zeker, we kunnen gewoon de randkleur veranderen. Dan schudden ze hun hoofd met die blik die zegt: 'Je begrijpt het niet.' Ze antwoorden: "Dat is niet duidelijk genoeg. Ik wil het soort effect waarbij licht langs de grens stroomt, zoals in een sciencefictionfilm."

Op dat moment begin je je misschien af ​​te vragen hoe je het moet bouwen. Canvas? SVG-bestand? Of kan CSS het zelf aan? Niemand wil immers toegeven dat hij niet weet hoe.

In moderne webapplicaties zijn animaties met randlichtbewegingen eigenlijk heel gebruikelijk. Ze worden voornamelijk gebruikt in een aantal scenario's zoals deze:

- **Statusindicatoren**: Markering van lopende taken of actieve items
- **Visuele focus**: belangrijke inhoudsgebieden benadrukken
- **Merkverbetering**: Het creëren van een strakke, moderne, technisch vooruitstrevende visuele stijl
- **Seizoensthema's**: Een feestelijke sfeer creëren voor speciale gelegenheden

Precies deze eis kwamen we tijdens het bouwen tegen [HagiCode](https://hagicode.com). Gebruikers moesten in één oogopslag zien welke sessies actief waren en welke voorstellen momenteel in behandeling waren. We hebben verschillende benaderingen geprobeerd. Sommige paden waren gladder, sommige waren wat kronkeliger. Uiteindelijk zijn we tot een redelijk volwassen implementatiestrategie gekomen.

## Kernimplementatie-ideeën

Na analyse van de HagiCode-codebase hebben we hieronder een aantal kernimplementatiepatronen samengevat. Ze passen allemaal in een ander scenario, of met andere woorden: ze bestaan ​​allemaal met een reden.

### 1. Roterende gloed met een kegelvormig verloop (meest gebruikelijk)

Dit is de klassieke manier om een lichtveegeffect aan de rand te implementeren. Het kernidee is om CSS te gebruiken `conic-gradient` om een kegelvormig verloop te creëren, en draai het vervolgens continu. Net als een straatlantaarn die 's nachts aangaat, blijft hij maar rondcirkelen.

**Belangrijkste elementen:**
- Gebruik de `::before` pseudo-element om de gloeilaag te maken
- Gebruik `conic-gradient` om de gradiëntkleurverdeling te definiëren
- Gebruik de `::after` pseudo-element om het middengebied te maskeren (optioneel)
- Gebruik `@keyframes` om de rotatieanimatie te implementeren

### 2. Gloeilijn aan de zijkant

Dit werkt goed voor statusindicatoren in lijstitems. U creëert een dunne gloeiende lijn aan één kant van het element in plaats van dat u de hele rand animeert. Soms is een beetje licht al genoeg. Je hoeft niet de hele wereld te verlichten.

**Belangrijkste elementen:**
- Een dun, absoluut gepositioneerd lijnelement
- Gebruik `box-shadow` om het glanseffect te creëren
- Gebruik `scale` en `opacity` voor een ademhalingsanimatie

### 3. Gloeiende achtergrond met doosschaduw

Als je niet het volledige sweep-effect nodig hebt en alleen een zachte achtergrondgloed wilt, kun je meerdere lagen aanbrengen `box-shadow` waarden is genoeg. Soms is de eenvoudigere optie de betere.

### 4. Ondersteuning van toegankelijkheid

Dit onderdeel wordt gemakkelijk over het hoofd gezien, maar is uiterst belangrijk. Elke animatie moet rekening houden met de `prefers-reduced-motion` mediaquery en bieden een statisch alternatief voor gebruikers die geen animatie willen. Niet iedereen houdt van constante beweging, en het respecteren van die voorkeur is belangrijk.

## Implementatieopties

### Optie 1: Roterende rand met kegelvormig verloop (aanbevolen)

Dit is de meest complete implementatie van het vloeiende randlichteffect, en het is ook de optie die we het vaakst gebruiken in HagiCode. Als iets goed werkt, waarom zou je het dan vervangen?

```css
/* Parent container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotating glow layer */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Mask layer (optional, for creating a hollow border effect) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Het principe achter deze optie is vrij eenvoudig: maak een pseudo-element dat groter is dan de bovenliggende container, teken er een kegelvormig verloop op en roteer het continu. De bovenliggende container gebruikt `overflow: hidden`, zodat alleen het licht dat langs de rand passeert zichtbaar blijft. Het is een beetje alsof je door een raam naar een straatlantaarn kijkt. Je ziet alleen het kleine stukje dat voorbij komt.

### Optie 2: Vereenvoudigde roterende lichtrand

Als je niet het volledige effect nodig hebt, bevat HagiCode ook een lichtere versie van nutsklasse. Soms is de eenvoudigere aanpak echt beter.

```css
/* Rotating light border utility class */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Accessibility support */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Let op de `will-change: transform` hier. Het vertelt de browser: "Dit element zal blijven veranderen", zodat de browser van tevoren enkele optimalisaties kan voorbereiden en de animatie vloeiender kan houden. Van tevoren voorbereiden is meestal beter dan op het laatste moment nog even doorbijten.

### Optie 3: Zijgloeilijn

Dit is vooral geschikt voor statusindicatoren van lijstitems, en het is precies wat de HagiCode-sessielijst gebruikt. Tussen veel items kan nog steeds één dunne lijn opvallen. Dat voelt op zijn eigen manier als een levensles.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Dit gebruikt `isolation: isolate` om een nieuwe stapelcontext te creëren, en vertrouwt vervolgens op `z-index` om de weergavevolgorde van elke laag te bepalen. `pointer-events: none` is ook essentieel. Anders zou het pseudo-element klikken van gebruikers blokkeren. Sommige dingen kunnen er leuk uitzien, maar mogen toch niet in de weg zitten.

### Optie 4: Verpak het in een React-component

Als uw project React gebruikt, kunt u deze logica in een component verpakken, vooral de toegankelijkheidsafhandeling. Schrijf het één keer, gebruik het vele malen. Dat is het hele punt.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

De bijpassende CSS-module:

```css
/* GlowBorder.module.css */

/* Animated version */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Static version (accessibility) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

De `useReducedMotion` haak van `framer-motion` detecteert automatisch de systeemvoorkeur van de gebruiker. Als de gebruiker verminderde beweging heeft ingeschakeld, keert deze terug `true`, en de component toont in plaats daarvan de statische versie. Het respecteren van de voorkeur van de gebruiker is belangrijker dan het forceren van een flitsend effect.

## Praktische lessen geleerd

Dit zijn enkele van de lessen die we hebben geleerd tijdens het bouwen van HagiCode. Je zou ze ook gevechtslittekens kunnen noemen. Hopelijk helpen ze je om wat omwegen te vermijden.

### 1. Themavariabelensysteem

CSS-variabelen maken ondersteuning voor meerdere thema's bijzonder handig. Niemand wil elke keer dat het thema verandert een stapel code bewerken.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Usage */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

Op die manier hoeft u voor het wisselen van thema alleen de klasse op de `html` element, en elke animatiekleur wordt automatisch bijgewerkt. Eén codebase, twee stijlen. Dat is precies wat wij willen.

### 2. Prestatieoptimalisatie

**Gebruik `will-change` om de browser een hint te geven om te optimaliseren:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Vertel het de browser van tevoren, en deze zal u helpen optimaliseren. Veel dingen in het leven werken beter met een beetje voorbereiding.

**Vermijd het gebruik van complexe doosschaduwen op grote elementen:**

```css
/* Not ideal - using a blurred shadow on a large element */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Better - use a pseudo-element to limit the glowing area */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Wij hebben dit getest in HagiCode. Door een wazige schaduw rechtstreeks aan een grote kaart toe te voegen, daalde de scroll-framesnelheid onder de 30 fps. Door over te schakelen naar een pseudo-element kwamen de zaken terug naar een stabiele 60 fps. Gebruikers kunnen dat verschil absoluut voelen.

### 3. Toegankelijkheid

Dit mag je echt niet overslaan. Sommige gebruikers vinden animatie duizelingwekkend of afleidend, en het respecteren van hun voorkeuren maakt deel uit van het bouwen van een goed product. Mooie dingen hoeven niet iedereen opgelegd te worden.

**CSS-mediaquery:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Provide a static fallback */
    opacity: 1;
  }
}
```

**Detecteer gebruikersvoorkeur in React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Controle van het intensiteitsniveau

De tokendoorvoerindicator in HagiCode toont verschillende gloedkleuren op basis van realtime doorvoer, en dit wordt dynamisch geïmplementeerd. Verschillende staten moeten verschillend worden uitgedrukt.

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Dingen waar je op moet letten

Er zijn nog een paar details die de moeite waard zijn om op te letten, want tegen de tijd dat je deze problemen op de harde manier ontdekt, is het al te laat.

| Dingen om op te letten | Uitleg |
|---------|------|
| **z-indexbeheer** | De gloeilaag moet een geschikte laag gebruiken `z-index` het interfereert dus niet met de interactie met de inhoud |
| **pointer-gebeurtenissen** | Het gloed-pseudo-element moet worden ingesteld `pointer-events: none` |
| **Grensoverloop** | De bovenliggende container heeft nodig `overflow: hidden`, of u moet de grootte van pseudo-elementen aanpassen |
| **Prestatie-impact** | Complexe animaties kunnen de prestaties op mobiele apparaten negatief beïnvloeden, dus test ze zorgvuldig |
| **Donkere modus** | Zorg ervoor dat de glanskleur duidelijk zichtbaar blijft op donkere achtergronden |
| **Themawisseling** | Gebruik CSS-variabelen zodat animatiekleuren correct worden bijgewerkt wanneer het thema verandert |

### 6. Tips voor foutopsporing

Pseudo-elementen kunnen lastig te vinden zijn in ontwikkelaarstools, dus u kunt tijdelijk een rand toevoegen om de positie te controleren.

```css
/* Temporarily show pseudo-element boundaries for debugging */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Nadat u klaar bent met het positioneren, vergeet dan niet om commentaar te geven of de regel te verwijderen. Anders kan de productie vrij snel lastig worden. Sommige dingen kun je beter in de ontwikkeling laten.
