## Typen is langzamer dan praten, en praten is langzamer dan een screenshot - Multimodale invoerpraktijken voor AI-coderingsassistenten

> Voor het schrijven van code geldt een snelheidslimiet, hoe snel u ook typt. Soms duurt het een eeuwigheid om iets dat je in één zin kunt zeggen uit te typen; soms maakt één screenshot alles duidelijk, maar moet je het toch beschrijven met een stapel tekst. In dit artikel vertellen we waar we tegenaan liepen tijdens het bouwen [HagiCode](https://hagicode.com), van spraakherkenning tot beelduploads. Uiteindelijk wilden we een AI-coderingsassistent een beetje gebruiksvriendelijker maken.

## Achtergrond

Tijdens het bouwen van HagiCode merkten we een probleem op - of beter gezegd, een probleem dat natuurlijk aan de oppervlakte kwam toen mensen het intensief gingen gebruiken: alleen vertrouwen op typen kan vermoeiend zijn.

Denk er eens over na: interactie tussen gebruikers en de Agent is een kernscenario. Maar als elke uitwisseling non-stop typen op het toetsenbord vereist, is de efficiëntie niet groot:

1. **Typen is te langzaam**: bij ingewikkelde problemen, zoals foutmeldingen of UI-problemen, kan het typen van alles een halve minuut duren, terwijl het hardop uitspreken tien seconden kan duren. Die kloof is reëel.

2. **Afbeeldingen zijn directer**: Soms geeft de gebruikersinterface een foutmelding, soms wil je een ontwerpconcept vergelijken en soms moet je een codestructuur laten zien. ‘Een beeld zegt meer dan duizend woorden’ is misschien een oud gezegde, maar het geldt nog steeds. AI het probleem rechtstreeks laten ‘zien’ is veel duidelijker dan het eeuwenlang beschrijven.

3. **Interactie moet natuurlijk aanvoelen**: moderne AI-assistenten moeten tekst, stem en afbeeldingen ondersteunen. Gebruikers moeten de invoermethode kunnen kiezen die het meest natuurlijk aanvoelt.

Daarom hebben we besloten om spraakherkenning en ondersteuning voor het uploaden van afbeeldingen toe te voegen aan HagiCode om interacties met agenten gemakkelijker te maken. Als gebruikers iets minder kunnen typen, is dat al winst.

## Over HagiCode

De oplossingen die in dit artikel worden gedeeld, zijn afkomstig van ons praktische werk in de [HagiCode](https://hagicode.com) project – of, beter gezegd, van de lessen die zijn geleerd terwijl we door een flink aantal valkuilen struikelden.

HagiCode is een open-source AI-coderingsassistentproject met een eenvoudig doel: AI gebruiken om de ontwikkelingsefficiëntie te verbeteren. Terwijl we bleven bouwen, werd het duidelijk dat gebruikers sterk multimodale input wilden. Soms gaat het uitspreken van één zin sneller dan het typen van een lange alinea, en soms is een screenshot veel duidelijker dan een lange uitleg.

Deze behoeften hebben ons vooruit geholpen, en zo kregen functies als spraakherkenning en het uploaden van afbeeldingen uiteindelijk vorm. Gebruikers kunnen nu op de meest natuurlijke manier met AI communiceren, en dat voelt goed.

## Analyse

### Technische uitdagingen bij spraakherkenning

Bij het bouwen van spraakherkenning kwamen we een lastig probleem tegen: **de WebSocket API van de browser ondersteunt geen aangepaste HTTP-headers**.

De spraakherkenningsservice die we kozen was de Doubao Speech Recognition API van ByteDance. Helaas vereist deze API authenticatie-informatie zoals `accessToken` en `secretKey` via HTTP-headers worden doorgegeven. Dat zorgde voor een onmiddellijk technisch conflict:

```javascript
// The browser WebSocket API does not support this approach
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // Not supported
  }
});
```

We hadden eigenlijk twee opties:

1. **URL-queryparameterbenadering**: plaats de authenticatie-informatie in de URL
   - Het voordeel is dat het eenvoudig te implementeren is
   - Het nadeel is dat inloggegevens worden blootgesteld aan de frontend, wat onveilig is; sommige API's vereisen ook op headers gebaseerde authenticatie

2. **Backend proxy-aanpak**: implementeer een WebSocket-proxy op de backend
   - Het voordeel is dat de inloggegevens veilig op de backend blijven opgeslagen en dat de oplossing volledig compatibel is met API-vereisten
   - Het nadeel is dat de implementatie iets complexer is

Uiteindelijk hebben we gekozen voor de **backend proxy-aanpak**. Beveiliging is niet iets waar je compromissen over sluit.

### Functionele vereisten voor het uploaden van afbeeldingen

Onze vereisten voor het uploaden van afbeeldingen waren eigenlijk vrij eenvoudig:

1. **Meerdere uploadmethoden**: klik om een bestand te selecteren, slepen en neerzetten en plakken vanaf het klembord
2. **Bestandsvalidatie**: typebeperkingen (PNG, JPG, WebP, GIF) en groottelimieten (5-10 MB) zijn basisvereisten
3. **Gebruikerservaring**: upload voortgang, voorbeelden en foutmeldingen, zodat gebruikers altijd weten wat er gebeurt
4. **Beveiliging**: validatie aan de serverzijde en bescherming tegen kwaadaardige bestandsuploads zijn essentieel

## Oplossing

### Spraakherkenning: WebSocket Proxy-architectuur

We ontwierpen een drielaagse architectuur voor spraakherkenning en vonden een pad dat werkte:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**Implementaties van kerncomponenten**:

1. **Frontend AudioWorklet-processor**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Resample to 16 kHz (required by the Doubao API)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Accumulate samples into 500 ms chunks
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Convert to 16-bit PCM and send
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **Backend WebSocket-handler** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. ** Frontend `VoiceTextArea` onderdeel**:


### Afbeeldingsuploads: component voor uploaden met meerdere methoden

We hebben een volledig uitgeruste component voor het uploaden van afbeeldingen gebouwd met ondersteuning voor alle drie de uploadmethoden, die de meest voorkomende scenario's dekt die gebruikers tegenkomen.

**Kernfuncties**:

1. **Drie uploadmethoden**:

```tsx
// Click to upload
const handleClick = () => fileInputRef.current?.click();

// Drag-and-drop upload
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Clipboard paste
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **Frontend-validatie**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **Backend-uploadhandler** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Validation
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Save file
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## Praktische gids

### Spraakherkenning gebruiken

1. **Configureer de spraakherkenningsservice**:
   - Open de pagina met instellingen voor spraakherkenning
   - Configureer de Doubao-spraak `AppId` en `AccessToken`
   - Configureer optioneel hotwords om de herkenningsnauwkeurigheid van domeinspecifieke termen te verbeteren

2. **Gebruik het in het invoervak**:
   - Klik op het microfoonpictogram aan de linkerkant van het invoervak
   - Begin te spreken nadat de golfvormanimatie verschijnt
   - Klik nogmaals op het pictogram om de opname te stoppen
   - De herkende tekst wordt automatisch ingevoegd op de cursorpositie

3. **Hotword-configuratievoorbeeld**:


### Hoe u afbeeldingsuploads gebruikt

1. **Uploadmethoden**:
   - Klik op de uploadknop om een bestand te kiezen
   - Sleep een afbeelding rechtstreeks naar het uploadgebied
   - Gebruik `Ctrl+V` om een schermafbeelding vanaf het klembord te plakken

2. **Ondersteunde formaten**: PNG, JPG, JPEG, WebP, GIF
3. **Groottelimiet**: standaard 5 MB (configureerbaar)

### Opmerkingen

1. **Spraakherkenning**:
   - Microfoontoestemming is vereist
   - Gebruik indien mogelijk in een rustige omgeving
   - De maximale ondersteunde opnameduur is standaard 300 seconden (configureerbaar)

2. **Beelduploads**:
   - Alleen gangbare afbeeldingsformaten worden ondersteund
   - Let op de limieten voor de bestandsgrootte
   - Geüploade afbeeldingen krijgen automatisch een voorbeeld-URL

3. **Beveiligingsoverwegingen**:
   - Inloggegevens voor spraakherkenning worden opgeslagen op de backend
   - Afbeeldingsuploads ondergaan een strikte validatie aan de serverzijde
   - HTTPS/WSS wordt aanbevolen in productieomgevingen

## Conclusie

Na het toevoegen van spraakherkenning en het uploaden van afbeeldingen verbeterde de gebruikerservaring van HagiCode merkbaar. Gebruikers kunnen nu op een meer natuurlijke manier met AI communiceren: spreken in plaats van typen, en schermafbeeldingen delen in plaats van alles handmatig te beschrijven. Het voelt alsof ik eindelijk een comfortabelere manier heb gevonden om te communiceren.

Tijdens het bouwen van deze functie kwamen we het probleem tegen dat WebSocket-API's van browsers geen aangepaste headers ondersteunen. Uiteindelijk hebben we het opgelost met een backend proxy-aanpak. Deze oplossing waarborgde niet alleen de veiligheid, maar legde ook de basis voor de latere integratie van andere geauthenticeerde WebSocket-services.

De component voor het uploaden van afbeeldingen profiteert ook van de ondersteuning van meerdere uploadmethoden, zodat gebruikers kunnen kiezen wat op dat moment het handigst is. Als u al het werk klikt, sleept of plakt, wordt de klus snel geklaard met elk pad.
