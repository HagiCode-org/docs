:::

## Vereisten

Zorg ervoor dat Docker en Docker Compose op uw systeem zijn geïnstalleerd voordat u begint.

### Installeer Docker

#### Ramen

Downloaden en installeren [Docker Desktop voor Windows](https://www.docker.com/products/docker-desktop/)

Zorg er na de installatie voor dat Docker Desktop actief is.

#### macOS

Downloaden en installeren [Docker-desktop voor Mac](https://www.docker.com/products/docker-desktop/)

Zorg er na de installatie voor dat Docker Desktop actief is.

#### Linux

Installeer Docker met behulp van uw pakketbeheerder:

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

Na de installatie start u de Docker-service:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Controleer de installatie

Voer na de installatie de volgende opdrachten uit om te controleren of Docker en Docker Compose correct zijn geïnstalleerd:

```bash
docker --version
docker compose version
```

## Snel beginnen

### 1. Genereer het Docker Compose-configuratiebestand

:::tip[Gebruik configuratiegenerator]
**Aanbevolen gebruik [Docker Compose Builder (zelfstandige site)](https://builder.hagicode.com/)** om uw configuratiebestand te genereren.

De generator ondersteunt:
- Meerdere API-provideropties (Zhipu AI, Anthropic Official, Custom API)
- Automatische configuratie van API-URL en token
- Aangepaste poorten, containernamen en andere basisconfiguraties
- Configureer persistente gegevens en werkmappaden
- Configureer automatisch Windows/Linux-platformverschillen
- Handel op intelligente wijze problemen met bestandsrechten af
- **Selectie van beeldbron**: Selecteer op basis van uw netwerkomgeving de juiste beeldbron

[🚀 Gebruik de generator onmiddellijk →](https://builder.hagicode.com/)
:::

1. Openen [Docker Compose Builder (zelfstandige site)](https://builder.hagicode.com/)
2. Vul de configuratie in volgens uw behoeften:
   - Selecteer een API-provider (Zhipu AI, Anthropic Official of Custom API)
   - Configureer poorten, database, werkmap en andere opties
   - **Selecteer afbeeldingsbron**: Kies de juiste afbeeldingsbron op basis van uw netwerkomgeving
3. Klik op de knop genereren om het `docker-compose.yml` configuratie
4. Sla de gegenereerde configuratie op als een `docker-compose.yml` bestand
5. Maak indien nodig een `.env` bestand om gevoelige informatie te configureren

#### Selectie van beeldbron

De generator ondersteunt drie beeldbronopties:

**Docker-hub**
- **Afbeeldingsadres**: `newbe36524/hagicode:{tag}`
- **Toepasselijke scenario's**: Geschikt voor gebruikers met Docker Hub-spiegelversnelling
- **Voordelen**: officiële afbeeldingsbron, tijdige updates, stabiele toegang
- **Opmerkingen**: In sommige regio's moet spiegelversnelling mogelijk worden geconfigureerd

**Azure Container Registry (ACR)**
- **Afbeeldingsadres**: `hagicode.azurecr.io/hagicode:{tag}`
- **Toepasselijke scenario's**: Geschikt voor gebruikers die geen lokale toegang tot Docker Hub hebben
- **Voordelen**: Biedt alternatieve beeldbron, lost netwerktoegangsproblemen op
- **Opmerkingen**: Afbeeldingen blijven gesynchroniseerd met Docker Hub, maar kunnen korte vertragingen hebben

**Alibaba Cloud Container Registry (ACR)**
- **Afbeeldingsadres**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **Toepasselijke scenario's**: Geschikt voor huishoudelijke gebruikers, stabiele en snelle netwerktoegang
- **Voordelen**: binnenlandse knooppunten van Alibaba Cloud, hoge toegangssnelheid, hoge stabiliteit
- **Opmerkingen**: Alibaba ACR-afbeeldingen blijven gesynchroniseerd met Docker Hub, meestal binnen 30 minuten

:::tip[Aanbevelingen voor selectie van beeldbronnen]
- **Binnenlandse gebruikers raden Alibaba Cloud ACR aan**: als u een binnenlandse gebruiker bent, wordt het aanbevolen om prioriteit te geven aan het gebruik van de Alibaba Cloud ACR-beeldbron voor snelle en stabiele toegang
- **Alternatieve Docker Hub**: Als uw netwerkomgeving toegang heeft tot Docker Hub, kunt u ook Docker Hub als afbeeldingsbron kiezen
- **Alternatief Azure ACR**: Kies alleen de Azure ACR-afbeeldingsbron als andere afbeeldingsbronnen toegangsproblemen hebben
- **Beeldsynchronisatie**: Alibaba Cloud ACR-afbeeldingen blijven gesynchroniseerd met Docker Hub, meestal binnen 30 minuten
- **Versieconsistentie**: afbeeldingstags zijn consistent in alle drie de afbeeldingsbronnen, u hoeft zich geen zorgen te maken over versieproblemen
:::

### 2. Start Services

Voer de volgende opdracht uit in de directory waar uw `docker-compose.yml` bevindt zich:

```bash
docker compose up -d
```

Deze opdracht zal:
- Download en start de Hagicode-containers die zijn gedefinieerd door de gegenereerde sjabloon
- Creëer en configureer het netwerk
- Initialiseer de standaardgegevensmap en runtime-afhankelijkheden

### 3. Controleer de servicestatus

Controleer of containers actief zijn:

```bash
docker compose ps
```

U zou de services die in uw gegenereerde sjabloon zijn gedefinieerd, in de status 'actief' moeten zien.

### Servicelogboeken bekijken

Om alle servicelogboeken te bekijken:

```bash
# View all service logs
docker compose logs

# View Hagicode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## Configuratie-instructies

Als u de configuratie moet wijzigen (zoals het wijzigen van poorten, het wisselen van API-provider, enz.), kunt u:

1. Genereer een nieuwe configuratie met behulp van [Docker Compose Builder (zelfstandige site)](https://builder.hagicode.com/)
2. Handmatig bewerken `docker-compose.yml` en `.env` bestanden
3. Start de services opnieuw om de configuratie van kracht te laten worden:

```bash
docker compose restart
```

## Toegang tot applicatie

### Webinterface

Nadat de service succesvol is opgestart, opent u via uw browser:

```
http://localhost:45000
```

### Gegevenspersistentie

Het ondersteunde Docker Compose-pad maakt standaard gebruik van SQLite-gegevensbestanden met persistente volumes. Als u koppelpunten moet inspecteren of back-ups moet plannen, bekijk dan de volume- en gegevensmaptoewijzingen in het gegenereerde bestand `docker-compose.yml`.
