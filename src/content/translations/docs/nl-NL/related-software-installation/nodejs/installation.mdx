Deze handleiding behandelt de installatie van Node.js op Windows, macOS en Linux. Node.js is een belangrijke afhankelijkheid voor lokale HagiCode-tools, waaronder OpenSpec, frontend-tools en verschillende CLI's voor ontwikkelaars.

:::tip[Versievereisten]
HagiCode raadt **Node.js 24+** aan (minimaal ondersteunde Node.js &gt;= 18.0). Installeer in de meeste gevallen de nieuwste LTS-versie en controleer vervolgens of uw versie aan de vereisten voldoet.
:::

## Node.js installeren

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Methode één: officieel installatieprogramma (aanbevolen)

1. Bezoek de [Officiële Node.js-website](https://nodejs.org/)
2. Download de **LTS (Long Term Support)**-versie Windows Installer (`.msi` bestand)
3. Dubbelklik om het installatieprogramma uit te voeren
4. Volg de aanwijzingen van de installatiewizard om de installatie te voltooien:
   - Accepteer de licentieovereenkomst
   - Kies het installatiepad (standaard is prima)
   - Zorg ervoor dat "Automatisch de benodigde tools installeren" is aangevinkt
   - Klik op "Installeren" om de installatie te starten
5. Nadat de installatie is voltooid, klikt u op "Voltooien"

### Methode twee: Winget gebruiken

Open PowerShell of opdrachtprompt en voer het volgende uit:

```powershell
winget install OpenJS.NodeJS.LTS
```

Na de installatie opent u de terminal opnieuw om de wijzigingen door te voeren.

  </TabItem>
  <TabItem value="macos" label="macOS">

### Methode één: officieel installatieprogramma (aanbevolen)

1. Bezoek de [Officiële Node.js-website](https://nodejs.org/)
2. Download het macOS-installatieprogramma van de **LTS (Long Term Support)**-versie (`.pkg` bestand)
3. Dubbelklik om het installatieprogramma uit te voeren
4. Volg de aanwijzingen van de installatiewizard om de installatie te voltooien:
   - Klik op "Doorgaan"
   - Accepteer de licentieovereenkomst
   - Kies installatieschijf
   - Klik op "Installeren" en voer het beheerderswachtwoord in
5. Nadat de installatie is voltooid, klikt u op "Sluiten"

### Methode twee: Homebrew gebruiken

Als je dat hebt [Zelf brouwen](https://brew.sh/) geïnstalleerd, kunt u Node.js via terminal installeren:

```bash
brew install node
```

:::tip
Homebrew voegt Node.js automatisch toe aan uw PATH. Als de opdracht niet beschikbaar is, start u de terminal opnieuw op of voert u het volgende uit:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

  </TabItem>
  <TabItem value="ubuntu" label="Ubuntu/Debian">

### NodeSource Repository gebruiken (aanbevolen)

Dit is de beste manier om de nieuwste Node.js-versie te verkrijgen:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Met behulp van apt Pakketbeheer

Als je de nieuwste versie niet nodig hebt, kun je de standaard Ubuntu-repository gebruiken:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::opmerking
De Node.js-versie in de standaardrepository is mogelijk ouder. Controleer na de installatie de versie om er zeker van te zijn dat deze voldoet aan de vereisten van Node.js &gt;= 18.0 (Node.js 24+ wordt aanbevolen).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### NodeSource Repository gebruiken (aanbevolen)

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### Yum Pakketbeheer gebruiken

```bash
sudo yum install -y nodejs npm
```

  </TabItem>
  <TabItem value="fedora" label="Fedora">

### Dnf-pakketbeheer gebruiken

```bash
sudo dnf install -y nodejs npm
```

:::tip
Fedora's software repository bevat meestal nieuwere Node.js versies en kan direct via dnf worden geïnstalleerd.
:::

  </TabItem>

## Versiebeheertools gebruiken

Als u moet schakelen tussen meerdere Node.js-versies, kunt u versiebeheertools gebruiken.

### nvm (knooppuntversiebeheer)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

Na de installatie laadt u uw shell-configuratie opnieuw:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Ramen

Op Windows raden we aan om [nvm-windows](https://github.com/coreybutler/nvm-windows):

1. Download de nieuwste `nvm-setup.exe` van [nvm-windows-releases](https://github.com/coreybutler/nvm-windows/releases)
2. Voer het installatieprogramma uit
3. Open de opdrachtprompt of PowerShell opnieuw

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

Na het installeren van nvm kunt u het gebruiken om Node.js-versies te installeren en te beheren:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (Fast Node-beheer)

fnm is een snellere en eenvoudigere Node.js-versiebeheertool gebouwd met Rust.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Ramen

Gebruik [schep](https://scoop.sh/) of [vleugeltje](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Versnelling voor gebruikers op het vasteland van China

Als u zich op het vasteland van China bevindt, kan het downloaden van Node.js- en npm-pakketten traag zijn. Hier zijn enkele versnellingsoplossingen.

### Binnenlandse spiegelbronnen gebruiken

#### npm Taobao spiegel (aanbevolen)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### Nrm gebruiken om spiegelbronnen te beheren

nrm (npm registermanager) helpt u snel te wisselen tussen npm mirror-bronnen:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Versnelde downloads van het Node.js-installatieprogramma

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao biedt mirror-downloads voor Node.js-installatieprogramma's:

- **Downloadadres**: https://npmmirror.com/mirrors/node/

Bezoek het bovenstaande adres en selecteer de versie die u wilt downloaden.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

Als het downloaden van nvm of andere tools van GitHub langzaam gaat, kun je de FastGit mirror gebruiken:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::opmerking
De FastGit-service kan veranderen, raadpleeg de officiële informatie.
:::

  </TabItem>
</Tabs>

### Binnenlandse Homebrew-spiegels

Als u Homebrew op het vasteland van China gebruikt, kunt u binnenlandse spiegels configureren voor versnelling:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Specifieke pakketbeheerders voor het vasteland van China

#### cnpm

cnpm is een npm-client geleverd door het Taobao-team dat standaard de Taobao-spiegel gebruikt:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::tip
cnpm is grotendeels compatibel met npm-opdrachten en kan worden gebruikt als directe vervanging.
:::

### Spiegelversnellingseffect verifiëren

Controleer na het instellen van spiegels door:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Installatie verifiëren

Controleer na de installatie of Node.js en npm correct zijn geïnstalleerd:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::opmerking
Zorg ervoor dat Node.js versie &gt;= 18.0 is (Node.js 24+ wordt aanbevolen). Als de versie te laag is, installeer deze dan opnieuw met behulp van de bovenstaande methoden of gebruik versiebeheertools om de juiste versie te installeren.
:::

### Voorbeeldverificatie-uitvoer

Succesvolle verificatie-uitvoer zou er als volgt uit moeten zien:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Problemen oplossen

### Windows-problemen

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Als `node` of `npm` opdrachten zijn niet beschikbaar na installatie:

1. Start de opdrachtprompt of PowerShell opnieuw
2. Indien nog steeds niet beschikbaar, voeg Node.js handmatig toe aan PATH:
   - Zoek naar "Omgevingsvariabelen" en open "Systeemomgevingsvariabelen bewerken"
   - Klik op "Omgevingsvariabelen"
   - Zoek "Pad" in "Systeemvariabelen" en klik op "Bewerken"
   - Voeg het Node.js-installatiepad toe (standaard: `C:\Program Files\nodejs\`)
   - Klik op "OK" om de wijzigingen op te slaan

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Als u uitvoeringsbeleidsfouten tegenkomt bij het gebruik van nvm:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### macOS-problemen

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Als u toestemmingsfouten tegenkomt bij het installeren van globale npm-pakketten:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Als er na de installatie geen opdrachten beschikbaar zijn, start u de terminal opnieuw op of voegt u handmatig iets toe aan PATH (meestal in `/usr/local/bin`).

  </TabItem>
</Tabs>

### Linux-problemen

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Als u toestemmingsfouten tegenkomt bij het installeren van globale npm-pakketten, is het raadzaam om versiebeheertools van Node.js (zoals nvm of fnm) te gebruiken in plaats van systeempakketbeheerders.

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Als de Node.js-versie die via systeempakketbeheer is geïnstalleerd, te oud is:

1. Gebruik NodeSource repository (aanbevolen, zie hierboven)
2. Of gebruik versiebeheertools (nvm/fnm)

  </TabItem>
</Tabs>

### Omgeving controleren

Als de verificatie mislukt, controleer dan het volgende:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
