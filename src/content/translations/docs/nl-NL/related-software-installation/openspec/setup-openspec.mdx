# Installeer OpenSpec

OpenSpec beheert voorstellen, specificaties en wijzigingsrecords. Deze pagina heeft één doel: leg het installatiepad duidelijk uit, zodat u het aan uw bestaande workflow kunt koppelen.

## Wat is OpenSpec?

OpenSpec is een opdrachtregelprogramma voor het beheren van voorstellen en specificaties bij softwarewerk. De kernworkflow omvat:

- **Voorstelbeheer**: functievoorstellen maken, beoordelen en volgen
- **Specificatievalidatie**: Zorg ervoor dat codewijzigingen voldoen aan vooraf gedefinieerde specificaties
- **Wijzigingen bijhouden**: registreer alle beoordeelde wijzigingsgeschiedenis
- **Teamsamenwerking**: Ondersteun de gezamenlijke ontwikkeling van meerdere personen

### Toepassing in HagiCode-projecten

In HagiCode-projecten wordt OpenSpec gebruikt voor:

1. **Functieontwikkelingsbeheer**: alle nieuwe functies moeten het OpenSpec-voorstelproces doorlopen
2. **Code Review**: Zorg ervoor dat wijzigingen voldoen aan de projectspecificaties
3. **Documentatiesynchronisatie**: Behoud consistentie tussen documentatie en code-implementatie
4. **Versiebeheerintegratie**: Naadloze integratie met versiebeheerworkflows

## Vereisten

Voordat u OpenSpec installeert, moet u ervoor zorgen dat uw systeem aan de volgende vereisten voldoet:

### Node.js en npm

OpenSpec wordt gedistribueerd via npm en vereist een Node.js-omgeving. OpenSpec vereist **Node.js 18.0 of hoger**.

Controleer de Node.js-installatie:

```bash
node --version
```

NPM-installatie verifiëren:

```bash
npm --version
```

Als de versie niet is geïnstalleerd of als de versie te oud is, raadpleeg dan de bijbehorende software-installatie-instructies in de [Installatiehandleiding](/en/installation).

## OpenSpec installeren

:::tip[Versievereisten]
Voor dit project is het versiebereik **OpenSpec 1.0.0 of hoger** vereist. Alle 1.x-versies worden ondersteund om compatibiliteit met de projectworkflow te garanderen.
:::

OpenSpec wordt wereldwijd geïnstalleerd via npm, waardoor het overal op uw systeem beschikbaar is.

### Wereldwijd installeren met npm

<Tabs>
  <TabItem value="win" label="Windows">
    Uitvoeren in PowerShell of opdrachtprompt:

```powershell
npm install -g @fission-ai/openspec@1
```

:::note[Problemen met Windows-toestemming]
Als u tijdens de installatie toestemmingsfouten tegenkomt, voert u PowerShell of de opdrachtprompt uit als beheerder.
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    Uitvoeren in Terminal:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[macOS-toestemmingsproblemen]
Als je tegenkomt `EACCES` toestemmingsfouten, probeer het volgende:
```bash
sudo npm install -g @fission-ai/openspec@1
```
Of gebruik [NVM](https://github.com/nvm-sh/nvm) om Node.js-versies te beheren en toestemmingsproblemen te voorkomen.
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    Uitvoeren in Terminal:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[Linux-toestemmingsproblemen]
Als je tegenkomt `EACCES` toestemmingsfouten kunt u:
1. Gebruik `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. Configureer npm om de gebruikersmap te gebruiken:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### Controleer de installatie

Controleer na de installatie of OpenSpec correct is geïnstalleerd en de versie aan de vereisten voldoet:

```bash
openspec --version
```

Als de installatie succesvol is verlopen, ziet u een versienummeruitvoer, bijvoorbeeld:

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::note[Versiebereik]
Zorg ervoor dat de geïnstalleerde OpenSpec-versie 1.0.0 of hoger is. Het project heeft de compatibiliteit met versie 1.1.1 geverifieerd. Het wordt aanbevolen om de nieuwste compatibele versie te gebruiken om nieuwe functies te krijgen.
:::

## Veelvoorkomende problemen en probleemoplossing

### Toestemmingsfouten

#### Probleem: `EACCES` Toestemming geweigerd

Toestemmingsfouten tegenkomen bij het installeren of uitvoeren van OpenSpec.

**Oplossingen**:

<Tabs>
  <TabItem value="win" label="Windows">
    Voer PowerShell of opdrachtprompt uit als beheerder:
    1. Klik met de rechtermuisknop op PowerShell of Opdrachtprompt
    2. Selecteer "Uitvoeren als beheerder"
    3. Voer de installatieopdracht opnieuw uit
  </TabItem>
  <TabItem value="mac" label="macOS">
    Installeer met behulp van `sudo`:

```bash
sudo npm install -g @fission-ai/openspec@1
```

Of gebruik [NVM](https://github.com/nvm-sh/nvm) om Node.js te beheren en algemene problemen met installatierechten te voorkomen.
  </TabItem>
  <TabItem value="linux" label="Linux">
    Oplossing 1: gebruik sudo
```bash
sudo npm install -g @fission-ai/openspec@1
```

Oplossing 2: configureer npm om de gebruikersmap te gebruiken
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @fission-ai/openspec@1
```
  </TabItem>
</Tabs>

### Netwerkproblemen

#### Probleem: npm-installatie mislukt of is erg traag

Kan OpenSpec niet downloaden van het npm-register vanwege netwerkredenen.

**Oplossingen**:

1. **Gebruik binnenlandse npm-spiegel**:

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **Officiële bron herstellen** (indien nodig):

```bash
npm config set registry https://registry.npmjs.org
```

3. **Gebruik cnpm** (optioneel):

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

:::tip
Gebruikers op het vasteland van China wordt aangeraden de Taobao-spiegelserver te gebruiken, die de downloadsnelheid aanzienlijk kan verbeteren.
:::

### Versiecompatibiliteitsproblemen

#### Probleem: OpenSpec-opdrachten worden abnormaal uitgevoerd of rapporteren fouten

De geïnstalleerde OpenSpec-versie is niet compatibel met de huidige omgeving.

**Oplossingen**:

1. **Controleer de Node.js-versie**:

```bash
node --version
```

Zorg ervoor dat de versie 18.0 of hoger is. Als de versie te laag is, upgrade dan Node.js.

2. **Controleer de OpenSpec-versie**:

```bash
openspec --version
```

3. **Specifieke versie opnieuw installeren**:

```bash
npm install -g @fission-ai/openspec@1
```

4. **Npm-cache wissen** (als het probleem zich blijft voordoen):

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### Commando niet gevonden

#### Probleem: typen `openspec` Commando geeft aan: "Commando niet gevonden"

OpenSpec is geïnstalleerd, maar het systeem kan de opdracht niet vinden.

**Oplossingen**:

<Tabs>
  <TabItem value="win" label="Windows">
    1. Bevestig het globale installatiepad van npm:

```powershell
npm config get prefix
```

2. Pad toevoegen aan systeemPATH:
   - Open "Systeemeigenschappen" → "Geavanceerd" → "Omgevingsvariabelen"
   - Vind `Path` in "Systeemvariabelen"
   - Voeg het globale pad npm toe (meestal `C:\Users\<username>\AppData\Roaming\npm`)
   - Start het opdrachtregelvenster opnieuw
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. Bevestig het globale installatiepad van npm:

```bash
npm config get prefix
```

2. Pad toevoegen aan PATH (in `~/.zshrc` of `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Configuratie opnieuw laden:

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. Bevestig het globale installatiepad van npm:

```bash
npm config get prefix
```

2. Pad toevoegen aan PATH (in `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Configuratie opnieuw laden:

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
