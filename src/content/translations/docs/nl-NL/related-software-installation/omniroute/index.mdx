Deze pagina is bedoeld voor nieuwe HagiCode-gebruikers die het kortste OmniRoute-installatiepad willen: start de lokale service, controleer of `http://localhost:20128/v1` werkt en bevestig dat zowel het Dashboard als de API beschikbaar zijn.

:::tip[Aanbevolen pad]
Deze eerste versie houdt het meest stabiele pad eerst: `npm install -g omniroute`, `omniroute`, Dashboard bij `http://localhost:20128`en de OpenAI-compatibele basis-URL op `http://localhost:20128/v1`. Docker, VPS en meer geavanceerde implementatiemodi blijven in het optionele gedeelte.
:::

## Vereisten

- Node.js is al geïnstalleerd. Compleet [Node.js installeren](/en/related-software-installation/nodejs/installation) eerst als dat nodig is.
- Lokale haven `20128` is beschikbaar en nog niet bezet.
- U hebt ten minste één upstream-modelprovideraccount die u via OmniRoute wilt routeren.
- Voordat u HagiCode, een CLI of een SDK integreert, moet u deze twee adressen gescheiden houden:
  - Dashboard: `http://localhost:20128`
  - OpenAI-compatibele basis-URL: `http://localhost:20128/v1`

## Lokaal installeren en uitvoeren

Begin met het aanbevolen globale NPM-pad:

```bash
npm install -g omniroute
omniroute
```

Nadat de service is gestart, valideert u deze in deze volgorde:

1. Openen `http://localhost:20128` in een browser en bevestig dat het Dashboard wordt geladen.
2. Ga naar de `Endpoints` pagina in het Dashboard en maak of kopieer een API-sleutel.
3. Noteer de basis-URL die u later aan tools zult doorgeven: `http://localhost:20128/v1`.

:::note[Waarom beginnen met het Dashboard]
Voor de eerste installatie is het Dashboard de snelste manier om te bevestigen dat OmniRoute actief is, uw providers verbonden zijn en er een API-sleutel bestaat. Als een latere clientaanroep mislukt, is het controleren van het Dashboard meestal sneller dan het onmiddellijk debuggen van de SDK-code.
:::

## Valideer de actieve service

### Validatie 1: bevestig dat de modelcatalogus leesbaar is

Steek de sleutel uit de `Endpoints` pagina naar een omgevingsvariabele:

```bash
export OMNIROUTE_API_KEY="replace-with-the-key-you-copied-from-the-Endpoints-page"
```

Vraag dan aan `/v1/models`:

```bash
curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}"
```

Als u een modellenlijst terugkrijgt, werken het OpenAI-compatibele eindpunt en de authenticatiestroom.

### Validatie 2: stuur een minimaal chatverzoek

```bash
curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "replace-with-a-model-id-from-v1-models",
    "messages": [
      { "role": "user", "content": "Reply with ok." }
    ]
  }'
```

Het doel hier is niet snelle kwaliteit. Het is om te bewijzen dat het door u gekozen model al succesvol door OmniRoute kan worden gerouteerd.
