## Önkoşullar

Başlamadan önce sisteminizde Docker ve Docker Compose'un kurulu olduğundan emin olun.

### Docker'ı yükleyin

#### Windows

İndirin ve yükleyin [Windows için Docker Masaüstü](https://www.docker.com/products/docker-desktop/)

Kurulumdan sonra Docker Desktop'ın çalıştığından emin olun.

#### macOS

İndirin ve yükleyin [Mac için Docker Masaüstü](https://www.docker.com/products/docker-desktop/)

Kurulumdan sonra Docker Desktop'ın çalıştığından emin olun.

#### Linux

Docker'ı paket yöneticinizi kullanarak yükleyin:

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

Kurulumdan sonra Docker hizmetini başlatın:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Kurulumu Doğrulayın

Kurulumdan sonra Docker ve Docker Compose'un doğru şekilde kurulduğunu doğrulamak için aşağıdaki komutları çalıştırın:

```bash
docker --version
docker compose version
```

## Hızlı Başlangıç

### 1. Docker Compose Yapılandırma Dosyasını Oluşturun

:::ipucu[Yapılandırma Oluşturucuyu Kullan]
**Kullanmanızı öneririz [Docker Compose Builder (bağımsız site)](https://builder.hagicode.com/)** yapılandırma dosyanızı oluşturmak için.

Jeneratör şunları destekler:
- Çoklu API sağlayıcı seçenekleri (Zhipu AI, Antropik Resmi, Özel API)

- Kalıcı verileri ve çalışma dizini yollarını yapılandırma
- Windows/Linux platformu farklılıklarını otomatik olarak yapılandırın
- Dosya izin sorunlarını akıllıca ele alın
- **Görüntü Kaynağı Seçimi**: Ağ ortamınıza bağlı olarak uygun görüntü kaynağını seçin

[🚀 Jeneratörü hemen kullanın →](https://builder.hagicode.com/)
:::

1. Açık [Docker Compose Builder (bağımsız site)](https://builder.hagicode.com/)
2. Yapılandırmayı ihtiyaçlarınıza göre doldurun:
   - API sağlayıcısını seçin (Zhipu AI, Antropik Resmi veya Özel API)
   - Bağlantı noktalarını, veritabanını, çalışma dizinini ve diğer seçenekleri yapılandırın
   - **Görüntü Kaynağını Seçin**: Ağ ortamınıza göre uygun görüntü kaynağını seçin
3. almak için oluştur düğmesine tıklayın. `docker-compose.yml` konfigürasyon
4. Oluşturulan konfigürasyonu şu şekilde kaydedin: `docker-compose.yml` dosya
5. Gerekirse bir tane oluşturun `.env` hassas bilgileri yapılandırmak için dosya

#### Görüntü Kaynağı Seçimi

Oluşturucu üç görüntü kaynağı seçeneğini destekler:

**Docker Merkezi**
- **Resim Adresi**: `newbe36524/hagicode:{tag}`
- **Geçerli Senaryolar**: Docker Hub yansıtma hızlandırmasına sahip kullanıcılar için uygundur
- **Avantajları**: Resmi görsel kaynağı, zamanında güncellemeler, istikrarlı erişim
- **Notlar**: Bazı bölgelerde yansıtma hızlandırmanın yapılandırılması gerekebilir

**Azure Container Registry (ACR)**
- **Resim Adresi**: `hagicode.azurecr.io/hagicode:{tag}`
- **Geçerli Senaryolar**: Docker Hub'a yerel olarak erişemeyen kullanıcılar için uygundur
- **Avantajları**: Alternatif görüntü kaynağı sağlar, ağ erişim sorunlarını çözer
- **Notlar**: Görüntüler Docker Hub ile senkronize kalır ancak kısa gecikmeler olabilir

**Alibaba Bulut Konteyner Kaydı (ACR)**
- **Resim Adresi**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **Uygulanabilir Senaryolar**: Ev kullanıcıları için uygun, istikrarlı ve hızlı ağ erişimi
- **Avantajları**: Alibaba Cloud yerel düğümleri, hızlı erişim hızı, yüksek kararlılık
- **Notlar**: Alibaba ACR görüntüleri genellikle 30 dakika içinde Docker Hub ile senkronize kalır

:::tip[Görüntü Kaynağı Seçimi Önerileri]
- **Yerli Kullanıcılar Alibaba Cloud ACR'yi Tavsiye Ediyor**: Yurt içi kullanıcıysanız, hızlı ve istikrarlı erişim için Alibaba Cloud ACR görüntü kaynağının kullanımına öncelik vermeniz önerilir.
- **Alternatif Docker Hub**: Ağ ortamınız Docker Hub'a erişebiliyorsa görüntü kaynağı olarak Docker Hub'ı da seçebilirsiniz.
- **Alternatif Azure ACR**: Yalnızca diğer görüntü kaynaklarında erişim sorunları varsa Azure ACR görüntü kaynağını seçin
- **Görüntü Senkronizasyonu**: Alibaba Cloud ACR görüntüleri genellikle 30 dakika içinde Docker Hub ile senkronize kalır
- **Sürüm Tutarlılığı**: Resim etiketleri üç resim kaynağının tamamında tutarlıdır; sürüm sorunları hakkında endişelenmenize gerek yoktur
:::

### 2. Hizmetleri Başlatın

Bilgisayarınızın bulunduğu dizinde aşağıdaki komutu çalıştırın. `docker-compose.yml` Bulunduğu yer:

```bash
docker compose up -d
```

Bu komut şunları yapacaktır:
- Oluşturulan şablon tarafından tanımlanan Hagicode kapsayıcılarını indirin ve başlatın
- Ağı oluşturma ve yapılandırma
- Varsayılan veri dizinini ve çalışma zamanı bağımlılıklarını başlatın

### 3. Hizmet Durumunu Doğrulayın

Container'ların çalışıp çalışmadığını kontrol edin:

```bash
docker compose ps
```

Oluşturduğunuz şablonda tanımlanan hizmetleri "çalışıyor" durumunda görmelisiniz.

### Hizmet Günlüklerini Görüntüle

Tüm servis günlüklerini görüntülemek için:

```bash
# View all service logs
docker compose logs

# View Hagicode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## Yapılandırma Talimatları

Yapılandırmayı değiştirmeniz gerekiyorsa (bağlantı noktalarını değiştirmek, API sağlayıcısını değiştirmek vb. gibi) şunları yapabilirsiniz:

1. Kullanarak yeni yapılandırmayı yeniden oluşturun [Docker Compose Builder (bağımsız site)](https://builder.hagicode.com/)
2. Manuel olarak düzenle `docker-compose.yml` ve `.env` dosyalar
3. Yapılandırmanın etkili olması için hizmetleri yeniden başlatın:

```bash
docker compose restart
```
