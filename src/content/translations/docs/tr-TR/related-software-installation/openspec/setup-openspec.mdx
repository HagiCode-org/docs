# OpenSpec'i yükleyin

OpenSpec teklifleri, spesifikasyonları ve değişiklik kayıtlarını yönetir. Bu sayfa tek bir hedefe odaklanıyor: mevcut iş akışınıza ekleyebilmeniz için kurulum yolunu açık bir şekilde açıklayın.

## OpenSpec nedir?

OpenSpec, yazılım çalışmalarındaki teklifleri ve spesifikasyonları yönetmek için kullanılan bir komut satırı aracıdır. Temel iş akışı şunları içerir:

- **Teklif Yönetimi**: Özellik teklifleri oluşturun, inceleyin ve izleyin
- **Şartname Doğrulaması**: Kod değişikliklerinin önceden tanımlanmış spesifikasyonlara uygun olduğundan emin olun
- **Değişiklik Takibi**: İncelenen tüm değişiklik geçmişini kaydedin
- **Ekip İşbirliği**: Çok kişili işbirliğine dayalı gelişimi destekleyin

### HagiCode Projelerinde Uygulama

HagiCode projelerinde OpenSpec şunlar için kullanılır:

1. **Özellik Geliştirme Yönetimi**: Tüm yeni özellikler OpenSpec teklif sürecinden geçmelidir
2. **Kod İncelemesi**: Değişikliklerin proje özelliklerine uygun olmasını sağlayın
3. **Belge Senkronizasyonu**: Dokümantasyon ile kod uygulaması arasında tutarlılığı koruyun
4. **Sürüm Kontrolü Entegrasyonu**: Sürüm kontrolü iş akışlarıyla sorunsuz entegrasyon

## Önkoşullar

OpenSpec'i kurmadan önce sisteminizin aşağıdaki gereksinimleri karşıladığından emin olun:

### Node.js ve npm

OpenSpec, npm aracılığıyla dağıtılır ve bir Node.js ortamı gerektirir. OpenSpec **Node.js 18.0 veya üstünü** gerektirir.

Node.js kurulumunu doğrulayın:

```bash
node --version
```

Npm kurulumunu doğrulayın:

```bash
npm --version
```

Kurulu değilse veya sürüm çok eskiyse, lütfen ilgili yazılım yükleme talimatlarına bakın. [Kurulum Kılavuzu](/en/installation).

## OpenSpec'in Kurulumu

:::tip[Sürüm Gereksinimleri]
Bu proje **OpenSpec 1.0.0 veya üzeri** sürüm aralığının kullanılmasını gerektirir. Proje iş akışıyla uyumluluğu sağlamak için tüm 1.x sürümleri desteklenir.
:::

OpenSpec, npm aracılığıyla küresel olarak kurulur ve sisteminizin herhangi bir yerinden kullanılabilir hale gelir.

### Npm Kullanarak Genel Olarak Yükleyin

<Tabs>
  <TabItem value="win" label="Windows">
    PowerShell'de veya Komut İstemi'nde çalıştırın:

```powershell
npm install -g @fission-ai/openspec@1
```

:::note[Windows İzin Sorunları]
Kurulum sırasında izin hatalarıyla karşılaşırsanız PowerShell'i veya Komut İstemi'ni yönetici olarak çalıştırın.
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    Terminalde Çalıştır:

```bash
npm install -g @fission-ai/openspec@1
```

:::not[macOS İzin Sorunları]
Eğer karşılaşırsan `EACCES` izin hataları, şunu deneyin:
```bash
sudo npm install -g @fission-ai/openspec@1
```
Veya kullanın [nvm](https://github.com/nvm-sh/nvm) Node.js sürümlerini yönetmek ve izin sorunlarını önlemek için.
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    Terminalde Çalıştır:

```bash
npm install -g @fission-ai/openspec@1
```

:::not[Linux İzin Sorunları]
Eğer karşılaşırsan `EACCES` izin hataları durumunda şunları yapabilirsiniz:
1. Kullanım `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. Npm'yi kullanıcı dizinini kullanacak şekilde yapılandırın:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### Kurulumu Doğrulayın

Kurulumdan sonra OpenSpec'in doğru şekilde kurulduğunu ve sürümün gereksinimleri karşıladığını doğrulayın:

```bash
openspec --version
```

Başarılı bir şekilde kurulursa, bir sürüm numarası çıktısı göreceksiniz, örneğin:

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::note[Sürüm Aralığı]
Lütfen kurulu OpenSpec sürümünün 1.0.0 veya üzeri olduğundan emin olun. Projenin 1.1.1 sürümüyle uyumluluğu doğrulandı. Yeni özellikler elde etmek için en son uyumlu sürümü kullanmanız önerilir.
:::

## Yaygın Sorunlar ve Sorun Giderme

### İzin Hataları

#### Sayı: `EACCES` İzin Reddedildi

OpenSpec'i yüklerken veya çalıştırırken izin hatalarıyla karşılaşılıyor.

**Çözümler**:

<Tabs>
  <TabItem value="win" label="Windows">
    PowerShell'i veya Komut İstemi'ni yönetici olarak çalıştırın:
    1. PowerShell'e veya Komut İstemi'ne sağ tıklayın
    2. "Yönetici olarak çalıştır"ı seçin
    3. Kurulum komutunu yeniden çalıştırın
  </TabItem>
  <TabItem value="mac" label="macOS">
    Kullanarak yükle `sudo`:

```bash
sudo npm install -g @fission-ai/openspec@1
```

Veya kullanın [nvm](https://github.com/nvm-sh/nvm) Node.js'yi yönetmek ve genel kurulum izni sorunlarından kaçınmak için.
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. Çözüm: Sudo'yu kullanın
```bash
sudo npm install -g @fission-ai/openspec@1
```

2. Çözüm: Npm'yi kullanıcı dizinini kullanacak şekilde yapılandırın
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @fission-ai/openspec@1
```
  </TabItem>
</Tabs>

### Ağ Sorunları

#### Sorun: npm Kurulumu Başarısız veya Çok Yavaş

Ağ nedenlerinden dolayı OpenSpec npm kayıt defterinden indirilemiyor.

**Çözümler**:

1. **Yerli npm Aynasını Kullan**:

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **Resmi Kaynağı Geri Yükle** (gerekirse):

```bash
npm config set registry https://registry.npmjs.org
```

3. **cnpm kullanın** (isteğe bağlı):

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

:::ipucu
Çin ana karasındaki kullanıcıların, indirme hızını önemli ölçüde artırabilen Taobao aynasını kullanmaları önerilir.
:::

### Sürüm Uyumluluk Sorunları

#### Sorun: OpenSpec Komutları Anormal Çalışıyor veya Hataları Bildiriyor

Yüklü OpenSpec sürümü mevcut ortamla uyumlu değil.

**Çözümler**:

1. **Node.js Sürümünü Kontrol Edin**:

```bash
node --version
```

Sürümün 18.0 veya üzeri olduğundan emin olun. Sürüm çok düşükse lütfen Node.js'yi yükseltin.

2. **OpenSpec Sürümünü Kontrol Edin**:

```bash
openspec --version
```

3. **Belirli Sürümü Yeniden Yükleyin**:

```bash
npm install -g @fission-ai/openspec@1
```

4. **npm Önbelleğini temizle** (sorun devam ederse):

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### Komut Bulunamadı

#### Sorun: Yazma `openspec` Komut "Komut Bulunamadı" mesajını gösteriyor

OpenSpec yüklü ancak sistem komutu bulamıyor.

**Çözümler**:

<Tabs>
  <TabItem value="win" label="Windows">
    1. Npm genel kurulum yolunu onaylayın:

```powershell
npm config get prefix
```

2. Sistem PATH'ine yol ekleyin:
   - "Sistem Özellikleri" → "Gelişmiş" → "Ortam Değişkenleri"ni açın
   - Bul `Path` "Sistem değişkenleri" içinde
   - Npm genel yolunu ekleyin (genellikle `C:\Users\<username>\AppData\Roaming\npm`)
   - Komut satırı penceresini yeniden başlat
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. Npm genel kurulum yolunu onaylayın:

```bash
npm config get prefix
```

2. PATH'e yol ekle (içinde `~/.zshrc` veya `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Yapılandırmayı yeniden yükle:

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. Npm genel kurulum yolunu onaylayın:

```bash
npm config get prefix
```

2. PATH'e yol ekle (içinde `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Yapılandırmayı yeniden yükle:

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
