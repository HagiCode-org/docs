Bu sayfa, en kısa OmniRoute kurulum yolunu isteyen ilk kez HagiCode kullanıcıları içindir: yerel hizmeti başlatın, bunu doğrulayın `http://localhost:20128/v1` çalışır ve hem Kontrol Panelinin hem de API'nin kullanılabilir olduğunu doğrulayın.

:::ipucu[Önerilen yol]
Bu ilk sürüm, en kararlı yolu ilk sırada tutar: `npm install -g omniroute`, `omniroute`, Kontrol Paneli `http://localhost:20128`ve OpenAI uyumlu Temel URL'yi şu adreste bulabilirsiniz: `http://localhost:20128/v1`. Docker, VPS ve daha gelişmiş dağıtım modları isteğe bağlı bölümde kalır.
:::

## Önkoşullar

- Node.js zaten yüklü. Tamamlandı [Node.js'yi yükleme](/en/related-software-installation/nodejs/installation) gerekirse ilk önce.
- Yerel liman `20128` mevcut ve halihazırda meşgul değil.
- OmniRoute aracılığıyla yönlendirmeyi planladığınız en az bir yukarı akış model sağlayıcı hesabınız var.
- HagiCode'u, CLI'yi veya SDK'yı entegre etmeden önce bu iki adresi ayrı tutun:
  - Kontrol paneli: `http://localhost:20128`
  - OpenAI uyumlu Temel URL: `http://localhost:20128/v1`

## Yerel olarak yükleyin ve çalıştırın

Önerilen global npm yoluyla başlayın:

```bash
npm install -g omniroute
omniroute
```

Hizmet başladıktan sonra hizmeti şu sırayla doğrulayın:

1. Açık `http://localhost:20128` bir tarayıcıda açın ve Kontrol Panelinin yüklendiğini onaylayın.
2. Şuraya git: `Endpoints` Kontrol Panelindeki sayfaya gidin ve bir API anahtarı oluşturun veya kopyalayın.
3. Daha sonra araçlara vereceğiniz Temel URL'yi not edin: `http://localhost:20128/v1`.

:::note[Neden Kontrol Paneli ile başlamalısınız]
İlk kurulum için Kontrol Paneli, OmniRoute'un çalıştığını, sağlayıcılarınızın bağlı olduğunu ve bir API anahtarının mevcut olduğunu doğrulamanın en hızlı yoludur. Daha sonraki bir istemci çağrısı başarısız olursa, önce Kontrol Panelini kontrol etmek genellikle SDK kodundaki hataları hemen ayıklamaktan daha hızlıdır.
:::

## Çalışan hizmeti doğrulayın

### Doğrulama 1: model kataloğunun okunabilir olduğunu onaylayın

Anahtarı oradan koy `Endpoints` sayfayı bir ortam değişkenine dönüştürün:

```bash
export OMNIROUTE_API_KEY="replace-with-the-key-you-copied-from-the-Endpoints-page"
```

O zaman iste `/v1/models`:

```bash
curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}"
```

Model listesini geri alırsanız OpenAI uyumlu uç nokta ve kimlik doğrulama akışı çalışıyor demektir.

### Doğrulama 2: minimum düzeyde sohbet isteği gönderin

```bash
curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "replace-with-a-model-id-from-v1-models",
    "messages": [
      { "role": "user", "content": "Reply with ok." }
    ]
  }'
```

Burada amaç hızlı kalite değil. Seçtiğiniz modelin OmniRoute üzerinden başarılı bir şekilde yönlendirilebildiğini kanıtlamak içindir.
