Bu kılavuz Windows, macOS ve Linux'ta Node.js kurulumunu kapsar. Node.js, OpenSpec, ön uç araçları ve çeşitli geliştirici CLI'leri de dahil olmak üzere HagiCode yerel araçları için önemli bir bağımlılıktır.

:::tip[Sürüm Gereksinimleri]
HagiCode **Node.js 24+** (desteklenen minimum Node.js >= 18.0) sürümünü önerir. Çoğu durumda en son LTS sürümünü yükleyin ve ardından sürümünüzün gereksinimleri karşıladığını doğrulayın.
:::

## Node.js'yi yükleme

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Birinci Yöntem: Resmi Yükleyici (Önerilen)

1. Ziyaret edin [Node.js Resmi Web Sitesi](https://nodejs.org/)
2. **LTS (Uzun Süreli Destek)** sürümünün Windows yükleyicisini indirin (`.msi` dosya)
3. Yükleyiciyi çalıştırmak için çift tıklayın
4. Kurulumu tamamlamak için kurulum sihirbazının talimatlarını izleyin:
   - Lisans sözleşmesini kabul edin
   - Kurulum yolunu seçin (varsayılan iyidir)
   - "Gerekli araçları otomatik olarak yükle" seçeneğinin işaretli olduğundan emin olun
   - Kurulumu başlatmak için "Yükle"ye tıklayın
5. Kurulum tamamlandıktan sonra "Son"a tıklayın

### İkinci Yöntem: Winget kullanma

PowerShell'i veya Komut İstemi'ni açın ve çalıştırın:

```powershell
winget install OpenJS.NodeJS.LTS
```

Kurulumdan sonra değişikliklerin etkili olması için terminali yeniden açın.

  </TabItem>
  <TabItem value="macos" label="macOS">

### Birinci Yöntem: Resmi Yükleyici (Önerilen)

1. Ziyaret edin [Node.js Resmi Web Sitesi](https://nodejs.org/)
2. **LTS (Uzun Süreli Destek)** sürümü macOS yükleyicisini indirin (`.pkg` dosya)
3. Yükleyiciyi çalıştırmak için çift tıklayın
4. Kurulumu tamamlamak için kurulum sihirbazının talimatlarını izleyin:
   - "Devam Et"i tıklayın
   - Lisans sözleşmesini kabul edin
   - Kurulum diskini seçin
   - "Yükle"ye tıklayın ve yönetici şifresini girin
5. Kurulum tamamlandıktan sonra "Kapat"a tıklayın

### İkinci Yöntem: Homebrew'u Kullanmak

eğer varsa [Ev yapımı bira](https://brew.sh/) Node.js'yi terminal aracılığıyla kurabilirsiniz:

```bash
brew install node
```

:::ipucu
Homebrew, Node.js'yi otomatik olarak PATH'inize ekleyecektir. Komut mevcut değilse terminali yeniden başlatın veya çalıştırın:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

  </TabItem>
  <TabItem value="ubuntu" label="Ubuntu/Debian">

### NodeSource Deposunu Kullanma (Önerilir)

En son Node.js sürümünü edinmenin en iyi yolu budur:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### apt Paket Yöneticisini Kullanma

En son sürüme ihtiyacınız yoksa varsayılan Ubuntu deposunu kullanabilirsiniz:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::not
Varsayılan depodaki Node.js sürümü daha eski olabilir. Kurulumdan sonra, Node.js >= 18.0 gereksinimlerini karşıladığından emin olmak için sürümü kontrol edin (Node.js 24+ önerilir).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### NodeSource Deposunu Kullanma (Önerilir)

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### yum Paket Yöneticisini Kullanma

```bash
sudo yum install -y nodejs npm
```

  </TabItem>
  <TabItem value="fedora" label="Fedora">

### Dnf Paket Yöneticisini Kullanma

```bash
sudo dnf install -y nodejs npm
```

:::ipucu
Fedora'nın yazılım deposu genellikle daha yeni Node.js sürümlerini içerir ve doğrudan dnf aracılığıyla kurulabilir.
:::

  </TabItem>
</Tabs>

## Sürüm Yönetimi Araçlarını Kullanma

Birden fazla Node.js sürümü arasında geçiş yapmanız gerekiyorsa sürüm yönetimi araçlarını kullanabilirsiniz.

### nvm (Düğüm Sürüm Yöneticisi)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

Kurulumdan sonra kabuk yapılandırmanızı yeniden yükleyin:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

Windows'ta kullanmanızı öneririz [nvm-pencereler](https://github.com/coreybutler/nvm-windows):

1. En son sürümü indirin `nvm-setup.exe` itibaren [nvm-windows sürümleri](https://github.com/coreybutler/nvm-windows/releases)
2. Yükleyiciyi çalıştırın
3. Komut İstemini veya PowerShell'i Yeniden Açın

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

Nvm'yi yükledikten sonra Node.js sürümlerini yüklemek ve yönetmek için kullanabilirsiniz:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (Hızlı Düğüm Yöneticisi)

fnm, Rust ile oluşturulmuş daha hızlı ve daha basit bir Node.js sürüm yönetim aracıdır.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

Kullanım [kepçe](https://scoop.sh/) veya [kanatçık](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Çin Anakarası Kullanıcıları için Hızlandırma

Anakara Çin'deyseniz Node.js ve npm paketlerini indirmek yavaş olabilir. İşte bazı hızlandırma çözümleri.

### Yerli Ayna Kaynaklarının Kullanılması

#### npm Taobao Aynası (Önerilen)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### Ayna Kaynaklarını Yönetmek için nrm'yi Kullanma

nrm (npm kayıt defteri yöneticisi), npm ayna kaynaklarını hızlı bir şekilde değiştirmenize yardımcı olur:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Node.js Yükleyici İndirmelerini Hızlandırma

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao, Node.js yükleyicileri için yansıma indirmeleri sağlar:

- **İndirme Adresi**: https://npmmirror.com/mirrors/node/

Yukarıdaki adresi ziyaret edin ve indirmeniz gereken sürümü seçin.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

GitHub'dan nvm veya diğer araçları indirmek yavaşsa FastGit aynasını kullanabilirsiniz:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::not
FastGit hizmeti değişebilir, lütfen resmi bilgilere bakınız.
:::

  </TabItem>
</Tabs>

### Yerli Homebrew Aynaları

Anakara Çin'de Homebrew kullanıyorsanız, hızlanma için yerel aynaları yapılandırabilirsiniz:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Anakara Çin'e Özel Paket Yöneticileri

#### cnpm

cnpm, Taobao ekibi tarafından sağlanan ve varsayılan olarak Taobao aynasını kullanan bir npm istemcisidir:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::ipucu
cnpm çoğunlukla npm komutlarıyla uyumludur ve doğrudan yerine kullanılabilir.
:::

### Ayna Hızlandırma Etkisinin Doğrulanması

Aynaları ayarladıktan sonra aşağıdakileri yaparak doğrulayın:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Kurulumun Doğrulanması

Kurulumdan sonra Node.js ve npm'nin doğru şekilde kurulduğunu doğrulayın:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::not
Node.js sürümünün >= 18.0 olduğundan emin olun (Node.js 24+ önerilir). Sürüm çok düşükse lütfen yukarıdaki yöntemleri kullanarak yeniden yükleyin veya doğru sürümü yüklemek için sürüm yönetimi araçlarını kullanın.
:::

### Örnek Doğrulama Çıktısı

Başarılı doğrulama çıktısı şöyle görünmelidir:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Sorun giderme

### Windows Sorunları

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Eğer `node` veya `npm` komutlar kurulumdan sonra kullanılamaz:

1. Komut İstemi'ni veya PowerShell'i yeniden başlatın
2. Hala kullanılamıyorsa Node.js'yi PATH'e manuel olarak ekleyin:
   - "Ortam Değişkenleri"ni arayın ve "Sistem ortam değişkenlerini düzenle"yi açın
   - "Ortam Değişkenleri"ne tıklayın
   - "Sistem değişkenleri"nde "Yol"u bulun ve "Düzenle"ye tıklayın
   - Node.js kurulum yolunu ekleyin (varsayılan: `C:\Program Files\nodejs\`)
   - Değişiklikleri kaydetmek için "Tamam"ı tıklayın

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Nvm'yi kullanırken yürütme politikası hatalarıyla karşılaşırsanız:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### macOS Sorunları

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Global npm paketlerini kurarken izin hatalarıyla karşılaşırsanız:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Kurulumdan sonra komutlar mevcut değilse, terminali yeniden başlatın veya PATH'e manuel olarak ekleyin (genellikle `/usr/local/bin`).

  </TabItem>
</Tabs>

### Linux Sorunları

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Global npm paketlerini kurarken izin hatalarıyla karşılaşırsanız sistem paket yöneticileri yerine Node.js sürüm yönetim araçlarını (nvm veya fnm gibi) kullanmanız önerilir.

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Sistem paketi yöneticisi aracılığıyla yüklenen Node.js sürümü çok eskiyse:

1. NodeSource deposunu kullanın (önerilir, yukarıya bakın)
2. Veya sürüm yönetimi araçlarını (nvm/fnm) kullanın

  </TabItem>
</Tabs>

### Ortamın Kontrol Edilmesi

Doğrulama başarısız olursa aşağıdakileri kontrol edin:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
