## Design.md: Tutarlı Yapay Zeka Odaklı Ön Uç Kullanıcı Arayüzü Tasarımı için Bir Çözüm

> Yapay zeka destekli ön uç geliştirme çağında, yapay zeka tarafından oluşturulan kullanıcı arayüzlerini nasıl tutarlı tutabilirsiniz? Bu makalede, harika tasarım md'ye dayalı bir tasarım galerisi sitesi oluşturma konusundaki uygulamalı deneyimimizi ve yapılandırılmış bir sitenin nasıl oluşturulacağını paylaşıyoruz. `design.md` Yapay zekayı standartlaştırılmış kullanıcı arayüzü tasarımına yönlendirmek için.

## Arka plan

Ön uç kodu yazmak için yapay zekayı kullanan herkes muhtemelen benzer bir deneyime sahiptir: Yapay zekadan aynı sayfayı birkaç kez oluşturmasını isteyin ve her sonuç farklı bir tarzda ortaya çıksın. Bazen köşeler yuvarlatılmış, bazen de keskindir. Bazen aralık 8 piksel, bazen de 16 piksel olur. Aynı düğme bile farklı görüşmelerde farklı görünebilir.

Bu münferit bir sorun değil. Yapay zeka destekli geliştirme daha yaygın hale geldikçe, **yapay zeka tarafından oluşturulan ön uç kullanıcı arayüzündeki tutarlılık eksikliği** yaygın bir sorun haline geldi. Farklı yapay zeka asistanları, farklı istemler ve hatta aynı asistanın farklı görüşmelerde, önemli ölçüde farklı arayüz tasarımları üretebilmesi mümkündür. Bu, ürünün yinelenmesi sırasında büyük bir bakım maliyeti yaratır.

Temel neden aslında basit: **yetkili tasarım referans belgesi** yok. Geleneksel CSS stil sayfaları, geliştiricilere bir şeyin "nasıl uygulanacağını" söyleyebilir, ancak "neden bu şekilde tasarlandığını" veya "hangi senaryoda hangi tasarım modelinin kullanılması gerektiğini" tam olarak anlatamazlar. Yapay zeka için açık ve yapılandırılmış bir açıklama, tasarım kurallarını anlamak açısından daha da önemlidir.

Aynı zamanda açık kaynak topluluğu zaten bazı mükemmel kaynaklar sunuyor. [VoltAgent/harika-tasarım-md](https://github.com/VoltAgent/awesome-design-md) proje, birçok tanınmış şirketten tasarım sistemi belgelerini toplar. Her dizin bir içerir `README.md`, bir `DESIGN.md`ve HTML'yi önizleyin. Ancak bunların tümü yukarı akış deposuna dağılmış durumda, bu da hızlı bir şekilde göz atmayı ve karşılaştırmayı zorlaştırıyor.

Peki, bu kaynakları göz atması kolay bir tasarım galerisinde birleştirirken aynı zamanda yapılandırılmış bir tasarım galerisini de ayrıştırabilir miyiz? `design.md` AI'nın kullanması için mi?

Cevap evet. Şimdi yaklaşımımızı gözden geçireyim.

## Analiz

Kod yazmadan önce bu sorunun ardındaki teknik zorlukları inceleyelim.

### Kaynak İçerik Yönetimi: Dağınık Tasarım Kaynaklarını Nasıl Birleştirirsiniz?

Yukarı yöndeki harika tasarım md deposu çok sayıda tasarım belgesi içeriyor, ancak bunları kendi projemize getirmenin bir yoluna ihtiyacımız vardı.

**Çözüm: git alt modülünü kullanın**

```
awesome-design-md-site
└── vendor/awesome-design-md          # Upstream resources (git submodule)
```

Bu bize çeşitli faydalar sağlar:
- Sürüm kontrolü: Belirli bir yukarı akış sürümünü sabitleyebiliriz
- Çevrimdışı derlemeler: derleme sırasında harici API istemeye gerek yoktur
- İçerik incelemesi: PR'lerde belirli değişiklikler görülebilir

### Veri Normalleştirme: Farklı Belge Yapılarını Nasıl Standartlaştırırsınız?

Farklı şirketler tasarım belgelerini farklı şekilde yapılandırır. Bazılarında önizleme dosyaları eksik, bazılarında ise tutarsız adlandırma kullanılıyor. Yapım sürecinde bunları normalleştirmemiz gerekiyor.

**Çözüm: Derleme sırasında normalleştirilmiş girişleri tarayın ve oluşturun**

### Statik Site Mimarisi: Tamamen Statik Kalırken Dinamik Aramayı Nasıl Sağlarsınız?

Bu bir tasarım galerisi olduğundan, arama yapılması şarttır. Ancak Astro statik bir site oluşturucudur, peki gerçek zamanlı aramayı nasıl uygularsınız?

**Çözüm: React Island + URL sorgu parametresi senkronizasyonu**


## Çözüm

Yukarıdaki analize dayanarak çözümümüz dört temel modülden oluşmaktadır.

### 1. İçerik Besleme İşlem Hattı

Bu, yukarı akış kaynaklarından içeriğin çıkarılmasından ve normalleştirilmesinden sorumlu olan tüm sistemin temelidir.

```typescript
// src/lib/content/awesomeDesignCatalog.ts

export async function scanSourceEntries(): Promise<DesignEntry[]> {
  const designDir = 'vendor/awesome-design-md/design-md';
  const entries: DesignEntry[] = [];

  for (const dir of await fs.readdir(designDir)) {
    const entryPath = path.join(designDir, dir);
    if (await isValidDesignEntry(entryPath)) {
      const entry = await normalizeDesignEntry(entryPath);
      entries.push(entry);
    }
  }

  return entries;
}

async function isValidDesignEntry(dir: string): Promise<boolean> {
  const requiredFiles = ['README.md', 'DESIGN.md'];
  for (const file of requiredFiles) {
    if (!(await fileExists(path.join(dir, file)))) {
      return false;
    }
  }
  return true;
}
```

### 2. Galeri Tarama Arayüzü

Galeri arayüzü üç ana bölümden oluşur:

**Ana Sayfa**: tüm tasarım girişlerini içeren bir kart tablosunu görüntüler ve her kart şunları içerir:
- Tasarım girişi başlığı ve özeti
- Varsa önizleme resmi
- Hızlı arama vurgulama

**Ayrıntı sayfası**: tek bir tasarım girişi için tüm bilgileri toplar:
- `README` belge
- `DESIGN` belge
- Açık/koyu tema geçişiyle önizleme
- Bitişik girişlere gezinme

**Navigasyon**: galeriye dönmeyi ve bitişik girişlere göz atmayı destekler

Ana sayfa galerisi, yüksek yoğunluklu bir kart düzeni kullanır; `design.md` ekiplerin marka stillerini, düğme desenlerini ve tipografik ritmi hızlı bir şekilde karşılaştırabilmesi için farklı kaynaklardan gelen girişleri birleşik bir görsel çerçeveye sığdırın.

### 3. Arama İşlevselliği

Arama özelliği, URL sorgu parametreleri aracılığıyla durumun korunduğu istemci tarafı filtrelemeyi temel alır:

```typescript
// src/components/gallery/SearchToolbar.tsx

function SearchToolbar({ entries }: { entries: DesignEntry[] }) {
  const [query, setQuery] = useState('');
  const [results, setResults] = useState(entries);

  useEffect(() => {
    const params = new URLSearchParams(window.location.search);
    const initialQuery = params.get('q') || '';
    setQuery(initialQuery);
    filterEntries(initialQuery);
  }, []);

  const filterEntries = (searchQuery: string) => {
    const filtered = entries.filter(entry =>
      entry.searchText.toLowerCase().includes(searchQuery.toLowerCase())
    );
    setResults(filtered);
  };

  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const value = e.target.value;
    setQuery(value);
    filterEntries(value);

    // Update the URL without triggering a page refresh
    const newUrl = value
      ? `${window.location.pathname}?q=${encodeURIComponent(value)}`
      : window.location.pathname;
    window.history.replaceState({}, '', newUrl);
  };

  return (
    <div className="search-toolbar">
      <input
        type="text"
        value={query}
        onChange={handleChange}
        placeholder="Search design entries..."
      />
      <span className="result-count">{results.length} results</span>
    </div>
  );
}
```

## Alıştırma

Artık çözümü ele aldığımıza göre, onu gerçekte nasıl uygularsınız?

### Uygulama Adımları

**1. Adım: Alt modülü başlatın**

```bash
# Add the upstream repository as a submodule
git submodule add https://github.com/VoltAgent/awesome-design-md.git vendor/awesome-design-md

# Initialize and update the submodule
git submodule update --init --recursive
```

**2. Adım: İçerik hattını oluşturun**

Uygula `awesomeDesignCatalog.ts`aşağıdakiler dahil:
- Dosya tarama ve doğrulama mantığı
- Astro'nun yerleşik oluşturucusunu kullanarak işaretleme oluşturma
- Giriş verilerinin çıkarılması

**3. Adım: Galeri kullanıcı arayüzünü oluşturun**

Aşağıdakileri oluşturmak için Astro + React Islands'ı kullanın:
- Ana sayfa galeri düzeni (kart ızgarası)
- Tasarım kartı bileşenleri
- Arama araç çubuğu
- Ayrıntı sayfası düzeni

**4. Adım: Tasarım belgesini yazın**

ClickHouse'un yapısına dayanmaktadır `DESIGN.md`, kendi projenizdeki gerçek tasarım belirteçlerini doldurun. Güncelleme `README.md` ve bir bağlantı ekleyin `design.md`.

### Notlar

**Güvenlik**: İşaretleme oluşturma, güvenli olmayan HTML'nin filtrelenmesini gerektirir. Astro'nun yerleşik oluşturucu filtreleri `script` etiketleri varsayılan olarak kullanabilirsiniz, ancak yine de XSS risklerine dikkat etmeniz gerekir.

**Performans**: Çok sayıda iframe önizlemesi, ilk boyama performansını etkileyebilir. kullanılması tavsiye edilir `loading="lazy"` önizleme içeriğini geç yüklemek için.

**Sürdürülebilirlik**: `design.md` kod uygulamasıyla senkronize kalması gerekir. CSS değişkenlerinin dokümantasyon ve kod arasında tutarlı kalmasını sağlamak için CI kontrollerinin eklenmesi önerilir.

**Erişilebilirlik**: Renk kontrastının WCAG AA standardını karşıladığından emin olun (en az 4,5:1).

### Yapay Zeka Kullanım Kılavuzu

Oluşturduktan sonra `design.md`Yapay zekanın onu gerçekten kullanmasını nasıl sağlarsınız? İşte birkaç pratik ipucu:

**İpucu 1: İstemde açıkça buna referans verin**

```
Please refer to the design.md file in the project root and use the design conventions defined there to implement the following components:
- Buttons: use the primary color with an 8px border radius
- Cards: use the elevation-2 shadow level
```

**İpucu 2: Yapay zekanın belirli CSS değişkenlerine referans vermesini zorunlu kılın**

```
Implement a navigation bar with the following requirements:
- Use --color-bg-primary for the background color
- Use --color-border-subtle for borders
- Use --text-color-primary for text
```

**3. İpucu: Dahil Et `design.md` sistem istemindeki içerik**

Yapay zeka aracınız özel sistem istemlerini destekliyorsa temel içeriğini ekleyebilirsiniz. `design.md` doğrudan ona.

### Test Stratejisi

**İçerik hattı testi**:
- Eksik dosya senaryoları (eksik `README.md` veya `DESIGN.md`)
- Biçim hatası senaryoları (Markdown ayrıştırma hatası)
- Boş dizin senaryoları

**Arama özelliği testi**:
- Boş sonuç işleme
- Çince ve emoji gibi özel karakterler
- URL senkronizasyon doğrulaması

**Kullanıcı arayüzü bileşen testi**:
- Açık/koyu tema geçişi
- Duyarlı düzen
- Yükleme durumlarını önizleyin
