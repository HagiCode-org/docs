## Uygulamada .NET Kod Koruması: Gizlemeden Sanal Makine Korumasına

> Bu makalede, temel gizlemeden profesyonel sanal makine korumasına kadar tüm yolu kapsayan, .NET projelerinde çok katmanlı bir kod koruma stratejisinin nasıl uygulanacağı açıklanmaktadır.

## Arka plan

.NET uygulama geliştirmede lisans doğrulama, iş mantığı ve hassas konfigürasyon gibi temel kodları kaynak koda dönüştürme ve tersine mühendislikten korumak, açıkçası kaçınamayacağınız bir konudur. .NET ekosistemi olgunlaştıkça geliştiriciler, yerleşik gizleme özelliklerinden profesyonel sanallaştırma tabanlı koruma araçlarına kadar çeşitli koruma seçeneklerine erişim kazandı.

Karmaşık, çok dilli bir monorepo projesi olan HagiCode, masaüstü uygulamalarını, derleme sistemlerini ve lisans yönetimi yeteneklerini içerir. Kod kaçınılmaz olarak lisans doğrulama mantığını, API anahtarları ve ürün kimlikleri gibi hassas konfigürasyonları ve iş açısından kritik mantığı içerir. Bu parçaların ciddi bir korumaya ihtiyacı var çünkü kimse emeklerinin bu kadar kolay açığa çıkmasını istemez.

Bu makale, HagiCode projesinde benimsediğimiz kod koruma yaklaşımını paylaşıyor ve ilk tuzaklardan daha sonraki optimizasyona kadar olan yolculuğun tamamını özetliyor. Umarım size bazı yararlı fikirler verir.

## Temel İçerik

### 1. Microsoft'un Yerleşik Gizleme Özelliği

.NET Framework yerleşik bir çözüm sunar `[ObfuscationAttribute]`, en temel ve yaygın olarak kullanılan kod gizleme işaretçisidir. Bu özellik şuralarda yaşıyor: `System.Reflection` ad alanıdır ve üçüncü taraf araçları kullanmadan koda temel koruma uygulamanıza olanak tanır.

**Temel özellikler**:
- `Feature` özellik: Gibi gizleme özelliğini belirtir `"ultra"` (yüksek gizleme) veya `"all"` (tam bir şaşkınlık)
- `Exclude` özellik: `true` gizlemenin dışında kalmak anlamına gelir ve `false` gizleme uygulamak anlamına gelir
- Sınıflara, yöntemlere, özelliklere ve diğer tür üyelerine uygulanabilir

HagiCode projesinde şu şekilde kullanıldığını görebilirsiniz:

```csharp
[Obfuscation(Feature = "ultra", Exclude = false)]
public async Task<LicenseValidationResult?> ValidateLicenseAsync(...)
```

Bu yaklaşımın **avantajları** oldukça açıktır:
- Ekstra bağımlılık yok; kutudan çıktığı gibi .NET Framework'te yerleşik olarak bulunur
- Üçüncü taraf gizleme araçları tarafından tanınabilir ve işlenebilir
- Derlenmiş derlemenin boyutunu önemli ölçüde artırmaz

Bununla birlikte, aynı zamanda sınırlamaları da var. Bu yalnızca bir işaretleyicidir ve gerçek gizleme sonucu, aracın uygulanmasına bağlıdır. Sanal makine koruma düzeyinde güvenlik sağlayamaz.

### 2. VMP (Sanal Makine Koruması)

VMP, kodu sanal makine talimatlarına derleyerek üst düzey koruma sağlayan profesyonel bir kod koruma aracıdır. Basit ad gizlemenin aksine, VMP aslında kod mantığını geleneksel kod çözücülerin yeniden oluşturamayacağı bir biçime dönüştürür.

**Koruma düzeyi sınıflandırması**:

| Seviye | Sanallaştırma | Mutasyon | Hata ayıklamayı önleme | Dize Şifreleme | Kullanım Durumları |
|------|--------|------|--------|------------|----------|
| YÜKSEK | dolu | yüksek | etkin | etkin | Lisans doğrulama, oturum eşzamanlılığı, hassas sabitler |
| ORTA | kısmi | orta | etkin | etkin | İş mantığı, etki alanı modelleri |
| DÜŞÜK | hiçbiri | düşük | devre dışı | devre dışı | Yardımcı sınıflar, kritik olmayan kod |

HagiCode projesi, korunması gereken kodu işaretlemek için bildirime dayalı bir nitelik sistemi tanımlar:

```csharp
// High-priority protection
[VmProtect(VmProtectionPriority.High, Reason = "Contains license verification logic")]
public class KeygenClient { ... }

// Exclude from protection
[VmExclude(Reason = "Public API that must remain unchanged")]
public class PublicApi { ... }

// Inherited protection
[VmProtect(Priority.High, ProtectDerived = true)]
public class BaseLicenseValidator { ... }
```

### 3. Oluşturma Zamanı Koruma Stratejisi

VMP koruması yalnızca çalışma zamanında önemli değildir. Aynı zamanda derleme hattının bir parçası olarak otomatikleştirilmesi de gerekiyor, çünkü bunu manuel olarak yapmak çok sıkıcı olacaktır. HagiCode'un derleme sistemi çeşitli modları destekler:

- **Yerel Windows modu**: VMProtect aracını doğrudan çağırın
- **Linux Docker kapsayıcı modu**: Platformlar arası uyumluluk sorunlarını çözmek için VMP'yi bir kapsayıcı içinde çalıştırın
- **Özellik taraması**: Koddaki koruma işaretlerini otomatik olarak keşfedin
- **Doğrulama mekanizması**: Korumanın başarıyla uygulandığını doğrulayın

Birlikte ele alındığında, bu yetenekler sürecin yönetilmesini çok daha kolay hale getirir.

## Çözüm

### 1. Microsoft'un Yerleşik Gizleme Özelliğini Kullanmak

Uygula `ObfuscationAttribute` doğrudan kodda:

```csharp
using System.Reflection;

[Obfuscation(Feature = "ultra", Exclude = false)]
public class LicenseService
{
    [Obfuscation(Feature = "ultra", Exclude = false)]
    public async Task<bool> ValidateLicenseAsync(string key)
    {
        // License validation logic
    }

    [Obfuscation(Feature = "flow", Exclude = false)]
    private string DecryptToken(string encrypted)
    {
        // Decryption logic
    }
}
```

Bazen üretim kodunu güvende tutarken test derlemelerinin dahili üyelere erişmesine izin vermeniz gerekir:

```csharp
// AssemblyInfo.cs
[assembly: InternalsVisibleTo("HagiCode.Application.Tests")]
[assembly: InternalsVisibleTo("DynamicProxyGenAssembly2")] // for Moq
```

Bu, testi çok daha kolay hale getirir çünkü kodun hala düzgün bir şekilde test edilmesi gerekiyor.

### 2. VMP Koruması için Özel Nitelik Tanımları

VMP davranışını kontrol etmek için özel koruma nitelikleri oluşturun:

```csharp
using System;

[AttributeUsage(AttributeTargets.Class | AttributeTargets.Method | AttributeTargets.Property)]
public class VmProtectAttribute : Attribute
{
    public VmProtectionPriority Priority { get; set; }
    public string? Reason { get; set; }
    public bool ProtectDerived { get; set; }
}

[AttributeUsage(AttributeTargets.Class | AttributeTargets.Method | AttributeTargets.Property)]
public class VmExcludeAttribute : Attribute
{
    public string? Reason { get; set; }
}

public enum VmProtectionPriority
{
    None = 0,
    Low = 1,
    Medium = 2,
    High = 3
}
```

Özel özniteliklerle çalışmak genellikle daha kolaydır çünkü bunlar doğrudan kendi koruma gereksinimlerinizi yansıtır.

## Pratik Kılavuz

### 1. Kritik Bileşenler İçin Koruma Uygulamaları

HagiCode'un kod koruma spesifikasyonuna göre aşağıdaki bileşenlerin YÜKSEK öncelikli koruma kullanması gerekir:

```csharp
// Production constants - must be encrypted and protected by VMP
[VmProtect(VmProtectionPriority.High, Reason = "Production constants")]
public static class ProductionConstants
{
    // Encrypted string accessor, protected by VMP
    [VmProtect(VmProtectionPriority.High)]
    public static string GetLicenseServerUrl(IOptions<LicenseOptions> options) => ...;
}

// License validation logic
[VmProtect(VmProtectionPriority.High, Reason = "License verification logic")]
public class KeygenClient : IKeygenClient
{
    [Obfuscation(Feature = "ultra", Exclude = false)]
    public async Task<LicenseValidationResult?> ValidateLicenseAsync(...) { ... }
}

// Machine fingerprint service
[VmProtect(VmProtectionPriority.High)]
public class MachineFingerprintService : IMachineFingerprintService { ... }
```

Kritik kod daha güçlü bir korumayı hak eder çünkü çekirdek mantığın açığa çıkması gerçek sorunlara neden olur.

### 2. Dize Şifreleme ve Çalışma Zamanı Şifre Çözme

Dizeleri derleme sırasında şifreleyin ve çalışma zamanında şifrelerini çözün:

```csharp
public static class StringDecryption
{
    [VmProtect(VmProtectionPriority.High, Reason = "CRITICAL SECURITY")]
    public static string DecryptString(byte[] encryptedData, byte[] key, byte[] iv)
    {
        using var aes = Aes.Create();
        aes.Key = key;
        aes.IV = iv;

        using var decryptor = aes.CreateDecryptor();
        using var ms = new MemoryStream(encryptedData);
        using var cs = new CryptoStream(ms, decryptor, CryptoStreamMode.Read);
        using var reader = new StreamReader(cs);

        return reader.ReadToEnd();
    }
}

// Production constant accessor (lazy loading + caching)
public static class ProductionConstants
{
    private static string? _cachedLicenseServerUrl;

    public static string GetLicenseServerUrl(IOptions<LicenseOptions> options)
    {
        if (_cachedLicenseServerUrl == null)
        {
            var encrypted = GetEncryptedLicenseServerUrl();
#if DEBUG
            _cachedLicenseServerUrl = options.Value.PrimaryServer.Url;
#else
            _cachedLicenseServerUrl = StringDecryption.DecryptString(
                encrypted,
                GetEncryptionKey(),
                GetEncryptionIV());
#endif
        }
        return _cachedLicenseServerUrl;
    }
}
```

Bu adım önemlidir çünkü hassas bilgiler hiçbir zaman düz metin olarak bırakılmamalıdır.

### 3. VMP Koruma Doğrulaması

Derlemeden sonra korumanın başarıyla uygulanıp uygulanmadığını doğrulamanız gerekir; aksi halde gerçekten çalıştığından emin olamazsınız:

```csharp
// Example verification script
public bool VerifyProtection(string assemblyPath)
{
    // 1. Check the VMP signature
    var bytes = File.ReadAllBytes(assemblyPath);
    var vmpSignature = Encoding.ASCII.GetBytes("VMProtect");
    if (bytes.Any(b => vmpSignature.Contains(b)))
    {
        return true;
    }

    // 2. Check for file size changes (the protected file is usually larger)
    var originalInfo = new FileInfo(assemblyPath.Replace(".dll", ".bak"));
    if (originalInfo.Exists)
    {
        var sizeRatio = (double)new FileInfo(assemblyPath).Length / originalInfo.Length;
        return sizeRatio > 1.1;
    }

    return false;
}
```

Doğrulama her zaman yapılmaya değerdir, çünkü aksi takdirde sorunlar gözden kaçabilir.

### 4. Notlar ve Uyarılar

Burada özel ilgiyi hak eden birkaç tuzak var:

1. **Kodların tamamını gizlemeyin**: Genel API'lerin, arayüz tanımlarının ve DTO sınıflarının genellikle korumaya ihtiyacı yoktur. Aşırı gizleme, performansa ve hata ayıklamaya zarar verebilir. HagiCode projesi bunu zor yoldan öğrendi.

2. **Anahtar erişimcilerini koruyun**: Şifreleme anahtarlarını alan yöntemler, şifrelenmiş verilerle aynı veya daha yüksek bir koruma düzeyine sahip olmalıdır; aksi takdirde tüm kurulum değerini kaybeder.

3. **Denge testi ve üretim**: DEBUG derlemeleri, geliştirme ve hata ayıklamayı kolaylaştırmak için şifrelemeyi atlamalı, RELEASE derlemeleri ise tam koruma sağlamalıdır. Bunları aşağıdaki gibi koşullu derlemeyle ayırmayı unutmayın: `#if DEBUG`.

4. **Docker ortamını göz önünde bulundurun**: VMP'yi Linux'ta çalıştırmak, araç uyumluluğunu sağlamak için kapsayıcılı bir yaklaşım gerektirir. HagiCode, platformlar arası sorunu çözmek için Wine + VMP kapsayıcı çözümünü kullanıyor.

5. **Doğrulama zorunludur**: Derleme tamamlandıktan sonra korumanın başarıyla uygulandığını doğrulamanız gerekir. Aksi takdirde hassas kod yine de açığa çıkabilir ve daha önce gösterilen doğrulama kodu tam olarak bu amaç için mevcuttur.
