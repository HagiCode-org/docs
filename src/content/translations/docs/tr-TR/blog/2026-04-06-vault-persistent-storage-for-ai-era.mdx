## Yapay Zeka Çağında Projeler Nasıl Yeniden Üretilir: Projeler Arası Kalıcı Depolama Sistemi olan Vault

> Yapay zeka destekli geliştirme çağında yapay zeka asistanlarının öğrenme kaynaklarımızı daha iyi anlamalarına nasıl yardımcı olabiliriz? HagiCode projesi, Vault sistemini yapay zekanın anlayabileceği birleşik bir bilgi depolama soyutlama katmanı olarak inşa ederek, proje yeniden üretimi yoluyla öğrenmenin verimliliğini büyük ölçüde artırdı.

## Arka plan

Yapay zeka çağında geliştiricilerin yeni teknolojileri ve mimarileri öğrenme şekli derinden değişiyor. "Projeleri yeniden üretmek", yani mükemmel açık kaynaklı projelerin kodunu, mimarisini ve tasarım kalıplarını derinlemesine incelemek ve bunlardan öğrenmek, öğrenmenin etkili bir yolu haline geldi. Kitap okumak veya video izlemek gibi geleneksel yöntemlerle karşılaştırıldığında, yüksek kaliteli açık kaynaklı projeleri doğrudan okumak ve yürütmek, gerçek dünyadaki mühendislik uygulamalarını çok daha hızlı anlamanıza yardımcı olur.

Yine de bu öğrenme yöntemi birçok zorluğu da beraberinde getiriyor.

Öğrenme materyalleri çok dağınık. Notlarınız Obsidian'da bulunabilir, kod depoları farklı klasörlere dağılmış olabilir ve yapay zeka asistanınızın konuşma geçmişi yine izole edilmiş bir veri adasına dönüşebilir. Yapay zekanın bir projeyi analiz etmeye yardımcı olmasını istediğinizde, kod parçacıklarını manuel olarak kopyalamanız ve bağlamı düzenlemeniz gerekir ki bu oldukça sıkıcıdır.

Daha da sıkıntılı olan ise bozuk bağlamdır. Yapay zeka asistanları yerel öğrenme kaynaklarınıza doğrudan erişemez, dolayısıyla her konuşmada arka plan bilgilerini tekrar vermeniz gerekir. Üstelik çoğaltılan kod depoları hızlı bir şekilde güncellenir, manuel senkronizasyon hataya açıktır ve bilginin birden fazla öğrenme projesi arasında paylaşılması zordur.

Bütün bu sorunların temelinde "veri adaları" yatıyor. Yapay zeka asistanlarının tüm öğrenme kaynaklarınızı anlamasına ve bunlara erişmesine olanak tanıyan birleşik bir depolama soyutlama katmanı olsaydı sorun düzgün bir şekilde çözülürdü.

## Kasa sistemi tasarım felsefesi

Vault sisteminin temel fikri basittir: Yapay zekanın anlayabileceği birleşik bir bilgi depolama soyutlama katmanı oluşturmak. Uygulama perspektifinden bakıldığında sistemin birkaç temel özelliği vardır.

### Çoklu tip desteği

Sistem, her biri farklı bir kullanım senaryosuna karşılık gelen dört kasa tipini destekler:

```typescript
// folder: general-purpose folder type
export const DEFAULT_VAULT_TYPE = 'folder';

// coderef: a type specifically for reproduced code projects
export const CODEREF_VAULT_TYPE = 'coderef';

// obsidian: integrated with Obsidian note-taking software
export const OBSIDIAN_VAULT_TYPE = 'obsidian';

// system-managed: vault automatically managed by the system
export const SYSTEM_MANAGED_VAULT_TYPE = 'system-managed';
```

Bunlar arasında **coderef** türü HagiCode'da en yaygın kullanılanıdır. Standartlaştırılmış bir dizin yapısı ve yapay zeka tarafından okunabilir meta veri açıklamaları sağlayarak çoğaltılmış kod projeleri için özel olarak tasarlanmıştır.

### Kalıcı depolama mekanizması

Vault kayıt defteri kalıcı olarak JSON formatında depolanır ve uygulama yeniden başlatıldıktan sonra yapılandırmanın kullanılabilir kalmasını sağlar:

```csharp
public class VaultRegistryStore : IVaultRegistryStore
{
    private readonly string _registryFilePath;

    public VaultRegistryStore(IConfiguration configuration, ILogger<VaultRegistryStore> logger)
    {
        var dataDir = configuration["DataDir"] ?? "./data";
        var absoluteDataDir = Path.IsPathRooted(dataDir)
            ? dataDir
            : Path.GetFullPath(Path.Combine(Directory.GetCurrentDirectory(), dataDir));

        _registryFilePath = Path.Combine(absoluteDataDir, "personal-data", "vaults", "registry.json");
    }
}
```

Bu tasarımın avantajı basit ve güvenilir olmasıdır. JSON insan tarafından okunabilir, bu da hata ayıklamayı ve manuel düzenlemeyi kolaylaştırır; dosya sistemi depolaması veritabanının karmaşıklığını ortadan kaldırır ve sistem bağımlılıklarını azaltır. Sonuçta bazen en basit seçenek gerçekten en iyisidir.

### Yapay zeka bağlam entegrasyonu

En önemlisi, sistem kasa bilgilerini yapay zeka tekliflerinin bağlamına otomatik olarak enjekte edebilir:

```typescript
export function buildTargetVaultsText(
  vaults: VaultForText[],
  template: VaultPromptTemplate = DEFAULT_VAULT_PROMPT_TEMPLATE,
): string {
  const readOnlyVaults = vaults.filter((vault) => vault.accessType === 'read');
  const editableVaults = vaults.filter((vault) => vault.accessType === 'write');

  if (readOnlyVaults.length === 0 && editableVaults.length === 0) {
    return '';
  }

  const sections = [
    buildVaultSection(readOnlyVaults, template.reference),
    buildVaultSection(editableVaults, template.editable),
  ].filter(Boolean);

  return `\n\n### ${template.heading}\n\n${sections.join('\n')}`;
}
```

Bu önemli bir yeteneği mümkün kılar: Yapay zeka asistanları, kullanıcıların manuel olarak bağlam sağlamasına gerek kalmadan mevcut öğrenme kaynaklarını otomatik olarak anlayabilir. Bunun bir tür örtülü anlayış sayıldığını söyleyebilirsiniz.

## CodeRef Vault'un standartlaştırılmış yapısı

Coderef kasa türü için HagiCode standartlaştırılmış bir dizin yapısı sağlar:

```
my-coderef-vault/
├── index.yaml          # vault metadata description
├── AGENTS.md           # operating guide for AI assistants
├── docs/               # stores study notes and documents
└── repos/              # manages reproduced code repositories through Git submodules
```

Bir kasa oluştururken sistem bu yapıyı otomatik olarak başlatır:

```csharp
private async Task EnsureCodeRefStructureAsync(
    string vaultName,
    string physicalPath,
    ICollection<VaultBootstrapDiagnosticDto> diagnostics,
    CancellationToken cancellationToken)
{
    Directory.CreateDirectory(physicalPath);

    var indexPath = Path.Combine(physicalPath, CodeRefIndexFileName);
    var docsPath = Path.Combine(physicalPath, CodeRefDocsDirectoryName);
    var reposPath = Path.Combine(physicalPath, CodeRefReposDirectoryName);

    // Create the standard directory structure
    if (!Directory.Exists(docsPath))
    {
        Directory.CreateDirectory(docsPath);
    }

    if (!Directory.Exists(reposPath))
    {
        Directory.CreateDirectory(reposPath);
    }

    // Create the AGENTS.md guide
    await EnsureCodeRefAgentsDocumentAsync(physicalPath, cancellationToken);

    // Create the index.yaml metadata
    await WriteCodeRefIndexDocumentAsync(indexPath, mergedDocument, cancellationToken);
}
```

Bu yapı da özenle tasarlanmıştır:

- **docs/** çalışma notlarınızı saklar; burada kodu anladığınızı, mimari analizi, öğrenilen dersleri vb. Markdown'da kaydedebilirsiniz
- **repos/**, kodu doğrudan kopyalamak yerine Git alt modülleri aracılığıyla çoğaltılmış depoları yönetir; bu, kodu senkronize tutar ve yerden tasarruf sağlar
- **index.yaml** kasa meta verilerini içerir, böylece AI asistanları kasanın amacını ve içeriğini hızlı bir şekilde anlayabilir
- **AGENTS.md** özellikle yapay zeka asistanları için yazılmış, kasa içeriğinin nasıl yönetileceğini açıklayan bir kılavuzdur

Bu şekilde organize edildiğinde belki yapay zeka aklınızdan geçenleri biraz daha kolay anlayabilir.

## Sistem tarafından yönetilen kasalar için otomatik başlatma

HagiCode, manuel olarak oluşturulan kasalara ek olarak sistem tarafından yönetilen kasaları da destekler:

```csharp
public async Task<IReadOnlyList<VaultRegistryEntry>> EnsureAllSystemManagedVaultsAsync(
    CancellationToken cancellationToken = default)
{
    var definitions = GetAllResolvedDefinitions();
    var entries = new List<VaultRegistryEntry>(definitions.Count);

    foreach (var definition in definitions)
    {
        entries.Add(await EnsureResolvedSystemManagedVaultAsync(definition, cancellationToken));
    }

    return entries;
}
```

Sistem aşağıdaki kasaları otomatik olarak oluşturur ve yönetir:

- **hagiprojectdata**: proje yapılandırmasını ve durumunu kaydetmek için kullanılan proje veri depolama alanı
- **kişisel veriler**: kullanıcı tercihlerini kaydetmek için kullanılan kişisel veri depolama alanı
- **hbsprompt**: yaygın olarak kullanılan yapay zeka istemlerini yönetmek için kullanılan bir bilgi istemi şablon kitaplığı

Bu kasalar sistem başlatıldığında otomatik olarak başlatılır, böylece kullanıcıların bunları manuel olarak yapılandırmasına gerek kalmaz. İnsanların onlar hakkında endişelenmesi yerine bazı şeylerin sisteme bırakılması daha iyidir.

## Erişim kontrol mekanizması

Tasarımın önemli bir kısmı erişim kontrolüdür. Sistem kasaları iki erişim türüne ayırır:

```typescript
export interface VaultForText {
  id: string;
  name: string;
  type: string;
  physicalPath: string;
  accessType: 'read' | 'write';  // Key: distinguish read-only from editable
}
```

- **referans (salt okunur)**: Yapay zeka yalnızca analiz ve anlama için kullanılır ve içeriği değiştiremez. Başvurulan açık kaynaklı projeler, belgeler ve benzeri materyaller için uygundur
- **düzenlenebilir (düzenlenebilir)**: Yapay zeka, içeriği görevin gerektirdiği şekilde değiştirebilir. Notlarınız, taslaklarınız ve benzeri materyalleriniz için uygundur

Bu ayrım önemlidir. Yapay zekaya hangi içeriğin "salt okunur referans" olduğunu ve hangi içeriğin "düzenlenmesi güvenli" olduğunu söyleyerek kazara değişiklik riskini azaltır. Sonuçta hiç kimse, istenmeyen bir düzenleme nedeniyle emeğinin yok olmasını istemez.

## Uygulamada: Apps Kasası oluşturma ve kullanma

Artık fikirleri ele aldığımıza göre, pratikte nasıl çalıştığına bakalım.

### CodeRef Kasası Oluşturma

İşte tam bir ön uç çağrısı örneği:

```typescript
const createCodeRefVault = async () => {
  const response = await VaultService.postApiVaults({
    requestBody: {
      name: "React Learning Vault",
      type: "coderef",
      physicalPath: "/Users/developer/vaults/react-learning",
      gitUrl: "https://github.com/facebook/react.git"
    }
  });

  // The system will automatically:
  // 1. Clone the React repository into vault/repos/react
  // 2. Create the docs/ directory for notes
  // 3. Generate the index.yaml metadata
  // 4. Create the AGENTS.md guide file

  return response;
};
```

Bu API çağrısı bir dizi eylemi tamamlar: dizin yapısını oluşturmak, Git alt modüllerini başlatmak, meta veri dosyalarını oluşturmak ve daha fazlası. Sadece temel bilgileri sağlamanız ve gerisini sistemin halletmesine izin vermeniz yeterlidir. Dürüst olmak gerekirse oldukça kaygısız bir yaklaşım.

## En iyi uygulamalar ve dikkat edilmesi gerekenler

Vault sistemini kullanırken öğrenilen birkaç dersi özetledik.

### Yol güvenliği

Sistem, yol geçiş saldırılarını önlemek için yolları sıkı bir şekilde doğrular:

```csharp
private static string ResolveFilePath(string vaultRoot, string relativePath)
{
    var rootPath = EnsureTrailingSeparator(Path.GetFullPath(vaultRoot));
    var combinedPath = Path.GetFullPath(Path.Combine(rootPath, relativePath));
    if (!combinedPath.StartsWith(rootPath, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode,
            "Vault file paths must stay inside the registered vault root.");
    }
    return combinedPath;
}
```

Bu önemli. Bir kasa yolunu özelleştirirseniz izin verilen aralıkta kaldığından emin olun, aksi takdirde sistem işlemi reddeder. Güvenliği gerçekten fazla vurgulayamazsınız.

### Git alt modül yönetimi

CodeRef Vault, kodu doğrudan kopyalamak yerine Git alt modüllerini önerir:

```csharp
private static string BuildCodeRefAgentsContent()
{
    return """
    # CodeRef Vault Guide

    Repositories under `repos/` should be maintained through Git submodules
    rather than copied directly into the vault root.

    Keep this structure stable so assistants and tools can understand the vault quickly.
    """ + Environment.NewLine;
}
```

Bu, çeşitli avantajlar sağlar: kodu yukarı akışla senkronize tutmak, disk alanından tasarruf etmek ve kodun birden çok sürümünü yönetmeyi kolaylaştırmak. Sonuçta kim aynı şeyi tekrar tekrar indirmek ister ki?

### Dosya önizleme sınırları

Performans sorunlarını önlemek için sistem dosya boyutunu ve türünü sınırlar:

```csharp
private const int FileEnumerationLimit = 500;
private const int PreviewByteLimit = 256 * 1024;  // 256KB
```

Kasanız çok sayıda dosya veya çok büyük dosyalar içeriyorsa önizleme performansı etkilenebilir. Bu durumda dosyaları toplu olarak işlemeyi veya özel arama araçlarını kullanmayı düşünebilirsiniz. Bazen bir şey çok büyüdüğünde, idare edilmesi kolaylaşmak yerine zorlaşır.

### Teşhis bilgileri

Bir kasa oluştururken sistem, hata ayıklamaya yardımcı olmak için tanılama bilgilerini döndürür:

```csharp
List<VaultBootstrapDiagnosticDto> bootstrapDiagnostics = [];

if (IsCodeRefVaultType(normalizedType))
{
    bootstrapDiagnostics = await EnsureCodeRefBootstrapAsync(
        normalizedName,
        normalizedPhysicalPath,
        normalizedGitUrl,
        cancellationToken);
}
```

Oluşturma işlemi başarısız olursa spesifik nedeni anlamak için tanılama bilgilerini inceleyebilirsiniz. Bir şeyler ters gittiğinde, teşhisleri kontrol etmek genellikle ilerlemenin en doğrudan yoludur.

## Özet

Vault sistemi, birleşik bir depolama soyutlama katmanı aracılığıyla, yapay zeka çağında projelerin çoğaltılmasındaki bazı temel sıkıntı noktalarını çözer:

- **Merkezi bilgi yönetimi**: tüm öğrenme kaynakları her yere dağılmak yerine tek bir yerde toplanır
- **Otomatik AI bağlam ekleme**: AI asistanları, manuel bağlam kurulumuna gerek kalmadan mevcut öğrenme kaynaklarını otomatik olarak anlayabilir
- **Projeler arası bilginin yeniden kullanımı**: Bilgi, birden fazla öğrenim projesi arasında paylaşılabilir ve yeniden kullanılabilir
- **Standartlaştırılmış dizin yapısı**: tutarlı bir dizin düzeni öğrenme eğrisini azaltır

Bu çözüm HagiCode projesinde zaten doğrulanmıştır. Ayrıca yapay zeka destekli geliştirmeyle ilgili araçlar geliştiriyorsanız veya benzer bilgi yönetimi sorunlarıyla karşılaşıyorsanız, bu deneyimlerin faydalı bir referans olabileceğini umuyorum.

Gerçekte, teknik bir çözümün değeri ne kadar karmaşık olduğu değil, gerçek sorunları çözüp çözmediğidir. Vault sisteminin temel fikri çok basit: Yapay zekanın anlayabileceği birleşik bir bilgi depolama katmanı oluşturmak.
