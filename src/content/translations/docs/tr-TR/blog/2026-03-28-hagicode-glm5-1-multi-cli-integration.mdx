## Hagicode ve GLM-5.1 Çoklu CLI Entegrasyon Kılavuzu

:::not[Zaman Bağlamı Bildirimi]
Bu makale, yayınlandığı sırada mevcut olan çoklu CLI entegrasyon tasarımını ve model eşlemesini belgelemektedir. Her zaman desteklenen güncel kapsam için bkz. [AI Aracısı CLI Kurulumu](/tr-TR/related-software-installation/ai-agent-cli/) ve [Kurulum Kılavuzu](/tr-TR/installation); the `IFlowCli` Burada sözü edilenler yalnızca o dönemdeki tarihsel konfigürasyonu temsil eder ve mevcut aktif destek kapsamının bir parçası değildir.

## Arka plan

Hagicode projesinde kullanıcılar, Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI ve daha fazlası dahil olmak üzere yapay zeka programlama asistanlarını yönlendirmek için birden fazla CLI aracı arasından seçim yapabilir. Bu CLI araçları kendi başlarına genel amaçlı yapay zeka programlama araçlarıdır ancak Hagicode'un soyutlama katmanı aracılığıyla farklı yapay zeka modeli sağlayıcılarına esnek bir şekilde bağlanabilirler.

Zhipu AI (ZAI), Anthropic Claude API ile uyumlu bir arayüz sağlayarak bu CLI araçlarının doğrudan yerel GLM serisi modellerini kullanmasına olanak tanır. Bunların arasında GLM-5.1, Zhipu'nun GLM-5.0'a göre önemli iyileştirmeler içeren en son geniş dil modeli sürümüdür.

## Hagicode'un CLI soyutlama mimarisi

Hagicode, 11 CLI sağlayıcı türünü tanımlar. `AIProviderType` ana akım AI programlama CLI araçlarını kapsayan enum:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Her CLI'nin karşılık gelen model parametre konfigürasyonu vardır ve `model` ve `reasoning` parametreler:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
```

## GLM-5.1'i farklı CLI'lerde kullanma

### 1. Claude Kodu CLI + GLM-5.1

Claude Code CLI, Hagicode'un temel CLI'lerinden biridir ve Hero yapılandırma sistemi aracılığıyla yapılandırılır:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

karşılık gelen `HeroEquipmentCatalogItem` yapılandırma:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI en esnek CLI'dir ve herhangi bir modelin belirtilmesini destekler. `provider/model` format:

**Yöntem 1: ZAI sağlayıcı önekini kullanın**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Yöntem 2: Doğrudan model kimliğini kullanın**
```json
{
  "model": "glm-5.1"
}
```

**Yöntem 3: Ön uç yapılandırma kullanıcı arayüzü**

içinde `HeroModelEquipmentForm.tsx`OpenCode CLI'nin özel bir yer tutucu ipucu vardır:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Kullanıcılar şunları girebilir:
```
zai/glm-5.1
glm-5.1
```

**OpenCode CLI modeli ayrıştırma mantığı**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI, eski işleme mantığına sahiptir:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Not**: Codebuddy, GLM-5.0 için özel işlemeyi korur ve eski normalleştirmeyi kullanmaz:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## GLM-5.1'in iyileştirme avantajları

GLM-5.0 ile karşılaştırıldığında GLM-5.1 aşağıdaki önemli iyileştirmeleri sağlar:

### 1. Daha iyi muhakeme yeteneği

Zhipu'nun resmi sürüm bilgilerine göre GLM-5.1'deki iyileştirmeler şunları içeriyor:

- **Daha güçlü kod anlayışı**: Karmaşık kod yapılarının daha doğru analizi
- **Daha uzun bağlam kavrama**: Daha uzun konuşma bağlamını destekler
- **Gelişmiş araç çağırma**: MCP araç çağrılarında daha yüksek başarı oranı
- **Çıktı kararlılığı**: Rastgeleliği ve halüsinasyonları azaltır

### 2. Kapsamlı çoklu CLI uyumluluğu

GLM-5.1, Hagicode tarafından desteklenen tüm ana CLI'leri kapsar:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Notlar

### 1. API anahtarı yapılandırması

Emin olun `ANTHROPIC_AUTH_TOKEN` ortam değişkeni doğru ayarlanmış. Her CLI'nin modele bağlanması için gerekli kimlik bilgisidir.

### 2. Model kullanılabilirliği

GLM-5.1'in ilgili model sağlayıcı tarafından etkinleştirilmesi gerekir:
- Zhipu AI ZAI platformu bunu varsayılan olarak destekler
- Alibaba Cloud DashScope ayrı bir uygulama gerektirebilir

### 3. OpenCode CLI formatı

Kullanırken `provider/model` biçimindeyse sağlayıcı kimliğinin doğru olduğundan emin olun:
- Zhipu AI: `zai` veya `zhipuai`
- Alibaba Bulutu: `aliyun` veya `dashscope`

### 4. Muhakeme parametresi

- `high` En iyi kod oluşturma sonuçları için önerilir
- Gemini CLI, akıl yürütme parametresini desteklemiyor ve bu yapılandırmayı otomatik olarak yok sayacak

## Sonuç

Hagicode, birleşik bir soyutlama katmanı aracılığıyla GLM-5.1 ile birden fazla CLI arasında esnek entegrasyon sağlar. Geliştiriciler, tercihlerine ve kullanım senaryolarına en uygun CLI aracını seçebilir ve ardından basit yapılandırmayla en yeni GLM-5.1 modelini kullanabilir.

Zhipu'nun en son model versiyonu olan GLM-5.1, GLM-5.0'a göre belirgin iyileştirmeler sunuyor:
- Eski yük taşımayan bağımsız bir sürüm tanımlayıcı
- Daha güçlü muhakeme ve kod anlayışı
- Geniş çoklu CLI uyumluluğu
- Esnek akıl yürütme düzeyi yapılandırması

Doğru ortam değişkenleri ve Hero ekipmanı yapılandırıldığında kullanıcılar, farklı CLI ortamlarında GLM-5.1'in gücünden tamamen yararlanabilirler.
