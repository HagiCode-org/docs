## HagiCode Beceri Sisteminin Teknik Analizi: Ölçeklenebilir bir Yapay Zeka Beceri Yönetimi Platformu Oluşturma

> Bu makale, HagiCode projesindeki Beceri yönetimi sisteminin mimarisine ve uygulamasına derinlemesine bir bakış sunarak dört temel özelliğin ardındaki teknik ayrıntıları kapsar: yerel küresel yönetim, pazar araştırması, akıllı öneriler ve güvenilir sağlayıcı yönetimi.

## Arka plan

Yapay zeka kodlama asistanları alanında yapay zeka yeteneklerinin sınırlarının nasıl genişletileceği her zaman temel bir soru olmuştur. Claude Code'un kendisi kod yardımında zaten güçlüdür ancak farklı geliştirme ekipleri ve farklı teknoloji yığınları genellikle Docker dağıtımlarını yönetme, veritabanı optimizasyonu veya ön uç bileşen oluşturma gibi belirli senaryolar için özel yeteneklere ihtiyaç duyar. İşte tam da bu noktada Beceri sistemi özellikle önem kazanıyor.

HagiCode projesinin geliştirilmesi sırasında benzer bir zorlukla karşılaştık: Sağlam bir kullanıcı deneyimini ve iyi mühendislik sürdürülebilirliğini korurken Claude Code'un bir kişinin öğrenebileceği gibi yeni mesleki becerileri "öğrenmesine" nasıl izin verebiliriz? Bu problem kendi açısından hem zor hem de basittir. Bu soru etrafında eksiksiz bir Beceri yönetimi sistemi tasarladık ve uyguladık.

Bu makale, sistemin teknik mimarisini ve temel uygulamasını ayrıntılı olarak ele almaktadır. Yapay zekanın genişletilebilirliği ve komut satırı aracı entegrasyonuyla ilgilenen geliştiricilere yöneliktir. Sizin için yararlı olabilir veya olmayabilir, ama en azından artık yazılmıştır.

## HagiCode Hakkında

Bu makalede paylaşılan yaklaşım, uygulamadaki deneyimlerimizden gelmektedir. [Hagi Kodu](https://hagicode.com) proje. HagiCode, geliştirme ekiplerinin mühendislik verimliliğini artırmasına yardımcı olmak için tasarlanmış açık kaynaklı bir yapay zeka kodlama asistanı projesidir. Projenin yığını ASP.NET Core'u, Orleans dağıtılmış çerçevesini, bir TanStack Start + React ön ucunu ve bu makalede tanıtılan Beceri yönetimi alt sistemini içerir.

GitHub deposu [HagiCode-org/site](https://github.com/HagiCode-org/site). Bu makaledeki teknik yaklaşımı değerli buluyorsanız, ona Yıldız vermekten çekinmeyin. Sonuçta daha fazla Yıldız ruh halini iyileştirme eğilimindedir.

## Sistem Mimarisine Genel Bakış

Skill sistemi ön uç-arka uçtan ayrılmış bir mimari kullanır. Bunda özellikle gizemli bir şey yok.

**Frontend**, Redux Toolkit yönetim durumuyla birlikte kullanıcı arayüzünü oluşturmak için TanStack Start + React'ı kullanır. Dört ana yetenek doğrudan dört Sekme bileşeniyle eşleşir: Yerel Beceriler, Beceri Galerisi, Akıllı Öneriler ve Güvenilir Sağlayıcılar. Sonuçta tasarım çoğunlukla kullanıcı deneyimini daha iyi hale getirmekle ilgilidir.

**Arka uç**, dağıtılmış durum yönetimi için Orleans Grain'i kullanan ASP.NET Core + ABP Çerçevesini temel alır. Çevrimiçi API istemcisi `IOnlineApiClient` Uzaktan beceri kataloğu hizmetiyle iletişim kurmak için arayüz.

Genel mimari prensip, komut yürütmeyi iş mantığından ayırmaktır. Bağdaştırıcı modeli aracılığıyla, npm/npx komut yürütmesinin uygulama ayrıntıları bağımsız modüllerin içinde gizlenir. Sonuçta hiç kimse komut satırı çağrılarının kod tabanının her yerine dağılmasını istemez.

## Temel Yetenek 1: Yerel Küresel Yönetim

Yerel küresel yönetim en temel modüldür. Yüklü becerilerin listelenmesinden ve kaldırma işlemlerinin desteklenmesinden sorumludur. Burada aşırı karmaşık bir şey yok; çoğunlukla temelleri iyi yapmakla ilgilidir.

### Teknik Yaklaşım

Uygulama yaşıyor `LocalSkillsTab.tsx` ve `LocalSkillCommandAdapter.cs`. Ana fikir, ambalajı sarmak. `npx skills` komutunu kullanın, JSON çıktısını ayrıştırın ve dahili veri yapılarına dönüştürün. Kulağa basit geliyor ve pratikte çoğunlukla öyle.

```csharp
public async Task<IReadOnlyList<LocalSkillInventoryResponseDto>> GetLocalSkillsAsync(
    CancellationToken cancellationToken = default)
{
    var result = await _commandAdapter.ListGlobalSkillsAsync(cancellationToken);
    return result.Skills.Select(skill => new LocalSkillInventoryResponseDto
    {
        Name = skill.Name,
        Version = skill.Version,
        Source = skill.Source,
        InstalledPath = skill.InstalledPath,
        Description = skill.Description
    }).ToList();
}
```

Veri akışı çok net: ön uç bir istek gönderir -> `SkillGalleryAppService` alır -> `LocalSkillCommandAdapter` npx komutunu çalıştırır -> JSON sonucu ayrıştırılır -> bir DTO döndürülür. Her adım doğal olarak bir öncekinin devamıdır.

Beceri kaldırma şunları kullanır: `npx skills remove -g <skillName> -y` komut ve sistem bağımlılıkları ve temizliği otomatik olarak yönetir. Kurulum meta verileri şurada saklanır: `managed-install.json` beceri dizini içinde, daha sonraki güncellemeler ve denetim için yükleme zamanı ve kaynak sürümü gibi bilgilerin kaydedilmesi. Bazı şeyler kayda değerdir.

### Detaylı Kurulum Akışı

Beceri kurulumu birkaç koordineli adım gerektirir. Aslında durum pek de karmaşık değil:

```csharp
public async Task<SkillInstallResultDto> InstallAsync(
    SkillInstallRequestDto request,
    CancellationToken cancellationToken = default)
{
    // 1. Normalize the installation reference
    var normalized = _referenceNormalizer.Normalize(
        request.SkillId,
        request.Source,
        request.SkillSlug,
        request.Version);

    // 2. Check prerequisites
    await _prerequisiteChecker.CheckAsync(cancellationToken);

    // 3. Acquire installation lock
    using var installLock = await _lockProvider.AcquireAsync(normalized.SkillId);

    // 4. Execute installation command
    var result = await _installCommandRunner.ExecuteAsync(
        new SkillInstallCommandExecutionRequest
        {
            Command = $"npx skills add {normalized.FullReference} -g -y",
            Timeout = TimeSpan.FromMinutes(4)
        },
        cancellationToken);

    // 5. Persist installation metadata
    await _metadataStore.WriteAsync(normalized.SkillPath, request);

    return new SkillInstallResultDto { Success = result.Success };
}
```

Burada birkaç temel tasarım modeli kullanılmıştır: **referans normalleştirici** aşağıdakiler gibi farklı giriş formatlarını dönüştürür: `tanweai/pua` ve `@opencode/docker-skill`, birleşik bir dahili temsile; **kurulum kilit mekanizması** aynı beceri için aynı anda yalnızca tek bir kurulum işleminin yürütülebilmesini sağlar; ve **akış çıkışı**, Sunucu Tarafından Gönderilen Olaylar aracılığıyla kurulum ilerlemesini gerçek zamanlı olarak ön uca aktarır, böylece kullanıcılar terminal benzeri günlükleri gerçekleşirken izleyebilir.

Sonuçta tüm bu kalıpların tek bir amacı var: Sistemin kullanımını ve bakımını daha basit tutmak.

## Temel Yetenek 2: Pazar Yeri Araması

Pazar yeri araması, kullanıcıların topluluktaki becerileri keşfetmesine ve yüklemesine olanak tanır. Bir kişinin yeteneği her zaman sınırlıdır; kolektif bilgi çok daha ileri gider.

### Teknik Yaklaşım

Arama özelliği çevrimiçi API'ye dayanır `https://api.hagicode.com/v1/skills/search`. Yanıt hızını artırmak için sistem önbelleğe alma işlemini uygular. Önbellek biraz belleğe benzer: eğer yararlı şeyleri etrafınızda tutarsanız, bir dahaki sefere bu kadar çok düşünmenize gerek kalmaz.

```csharp
private async Task<IReadOnlyList<SkillGallerySkillDto>> SearchCatalogAsync(
    string query,
    CancellationToken cancellationToken,
    IReadOnlySet<string>? allowedSources = null)
{
    var cacheKey = $"skill_search:{query}:{string.Join(",", allowedSources ?? Array.Empty<string>())}";

    if (_memoryCache.TryGetValue(cacheKey, out var cached))
        return (IReadOnlyList<SkillGallerySkillDto>)cached!;

    var response = await _onlineApiClient.SearchAsync(
        new SearchSkillsRequest
        {
            Query = query,
            Limit = _options.LimitPerQuery,
        },
        cancellationToken);

    var results = response.Skills
        .Where(skill => allowedSources is null || allowedSources.Contains(skill.Source))
        .Select(skill => new SkillGallerySkillDto { ... })
        .ToList();

    _memoryCache.Set(cacheKey, results, TimeSpan.FromMinutes(10));
    return results;
}
```

Arama sonuçları, güvenilir kaynaklara göre filtrelemeyi destekler; böylece kullanıcılar yalnızca güvendikleri beceri kaynaklarını görür. Tohum sorguları gibi `popular` ve `recent` kataloğu başlatmak için kullanılır ve kullanıcıların önerilen popüler becerileri ilk açtıklarında görmelerine olanak tanır. İlk izlenimler hala önemlidir.

## Temel Yetenek 3: Akıllı Öneriler

Akıllı öneriler sistemin en karmaşık parçasıdır. Mevcut proje bağlamına göre en uygun becerileri otomatik olarak önerebilirler. Ne kadar karmaşık olsa da hâlâ inşa edilmeye değer.

### Öneri Akışı

Tam öneri akışı beş aşamaya ayrılmıştır:

```
1. Build project context
   ↓
2. AI generates search queries
   ↓
3. Search the online catalog in parallel
   ↓
4. AI ranks the candidates
   ↓
5. Return the recommendation list
```

İlk olarak sistem, bir "proje profili" oluşturmak için projenin teknoloji yığını, programlama dilleri ve etki alanı yapısı gibi özellikleri analiz eder. Bu profil, projenin temel özelliklerini kaydeden bir özgeçmişe benziyor.

Daha sonra hedeflenen arama sorguları oluşturmak için bir AI Tahıl kullanılır. Bu tasarım aslında oldukça ilginç: Yapay zekaya doğrudan "Hangi becerileri önermeliyim?" diye sormak yerine, öncelikle "Hangi arama terimlerinin alakalı becerileri bulma olasılığı yüksektir?" sorusunu sormasını istiyoruz. Bazen soruyu sorma şekliniz cevabın kendisinden daha önemlidir:

```csharp
var queryGeneration = await aiGrain.GenerateSkillRecommendationQueriesAsync(
    projectContext,      // Project context
    locale,              // User language preference
    maxQueries,           // Maximum number of queries
    effectiveSearchHero); // AI model selection
```

Daha sonra bu arama sorguları bir aday beceri listesi toplamak için paralel olarak yürütülür. Paralel işleme, sonuçta zamandan tasarruf etmenin bir yoludur.

Son olarak başka bir Yapay Zeka Tahıl, aday becerilerini sıralıyor. Bu adımda, becerinin projeyle ilgisi, güven durumu ve kullanıcı geçmişi tercihleri gibi faktörler dikkate alınır:

```csharp
var ranking = await aiGrain.RankSkillRecommendationsAsync(
    projectContext,
    candidates,
    installedSkillNames,
    locale,
    maxRecommendations,
    effectiveRankingHero);

response.Items = MergeRecommendations(projectContext, candidates, ranking, maxRecommendations);
```

## Temel Yetenek 4: Güvenilir Sağlayıcı Yönetimi

Güvenilir sağlayıcı yönetimi, kullanıcıların hangi beceri kaynaklarının güvenilir olarak kabul edildiğini kontrol etmesine olanak tanır. Güven hâlâ kullanıcıların kendileri için tanımlayabilmeleri gereken bir şeydir.

### Eşleştirme Kuralları

Güvenilir sağlayıcılar iki eşleştirme kuralını destekler: **tam eşleşme** (`exact`) ve **önek eşleşmesi** (`prefix`).

```csharp
public static TrustedSkillProviderResolutionSnapshot Resolve(
    TrustedSkillProviderSnapshot snapshot,
    string source)
{
    var normalizedSource = Normalize(source);

    foreach (var entry in snapshot.Entries.OrderBy(e => e.SortOrder))
    {
        if (!entry.IsEnabled) continue;

        foreach (var rule in entry.MatchRules)
        {
            bool isMatch = rule.MatchType switch
            {
                TrustedSkillProviderMatchRuleType.Exact
                    => string.Equals(normalizedSource, Normalize(rule.Value),
                        StringComparison.OrdinalIgnoreCase),
                TrustedSkillProviderMatchRuleType.Prefix
                    => normalizedSource.StartsWith(Normalize(rule.Value) + "/",
                        StringComparison.OrdinalIgnoreCase),
                _ => false
            };

            if (isMatch)
                return new TrustedSkillProviderResolutionSnapshot
                {
                    IsTrustedSource = true,
                    ProviderId = entry.ProviderId,
                    DisplayName = entry.DisplayName
                };
        }
    }

    return new TrustedSkillProviderResolutionSnapshot { IsTrustedSource = false };
}
```

Yerleşik güvenilir sağlayıcılar arasında Vercel, Azure, anthropics, Microsoft ve tarayıcı kullanımı gibi tanınmış kuruluşlar ve projeler bulunur. Özel sağlayıcılar, bir sağlayıcı kimliği, görünen ad, rozet etiketi, eşleştirme kuralları ve daha fazlasını belirterek yapılandırma dosyaları aracılığıyla eklenebilir. Dünya, yalnızca birkaç yerleşik yapıya güvenmenin asla yeterli olmayacağı kadar büyük.

### Kalıcılık Uygulaması

Güvenilir konfigürasyon Orleans Tahıl kullanılarak sürdürülür:

```csharp
public class TrustedSkillProviderGrain : Grain<TrustedSkillProviderState>,
    ITrustedSkillProviderGrain
{
    public async Task UpdateConfigurationAsync(TrustedSkillProviderSnapshot snapshot)
    {
        State.Snapshot = snapshot;
        await WriteStateAsync();
    }

    public Task<TrustedSkillProviderSnapshot> GetConfigurationAsync()
    {
        return Task.FromResult(State.Snapshot);
    }
}
```

Bu yaklaşımın yararı, önbelleklerin manuel olarak yenilenmesine gerek kalmadan yapılandırma değişikliklerinin tüm düğümlerde otomatik olarak senkronize edilmesidir. Otomasyon, sonuçta insanların daha az endişelenmesine olanak tanımakla ilgilidir.

## Anahtar Teknik Tasarım

### Komut Yürütme Bağdaştırıcı Kalıbı

Skill sisteminin çeşitli npx komutlarını yürütmesi gerekir. Bu mantık her yere dağılmış olsaydı kodun bakımı hızla zorlaşırdı. Bu nedenle bir adaptör arayüzü tasarladık. Sonuçta tasarım desenleri kodun bakımını kolaylaştırmak için mevcuttur:

```csharp
public interface ISkillInstallCommandRunner
{
    Task<SkillInstallCommandExecutionResult> ExecuteAsync(
        SkillInstallCommandExecutionRequest request,
        CancellationToken cancellationToken = default);
}
```

Farklı komutların farklı yürütücü uygulamaları vardır, ancak hepsi aynı arayüzü uygulayarak test etme ve değiştirme işlemlerini kolaylaştırır.

### SSE Akış Çıkışı

Kurulum ilerlemesi, Sunucu Tarafından Gönderilen Etkinlikler aracılığıyla gerçek zamanlı olarak ön uca aktarılır:

```csharp
public async Task InstallWithProgressAsync(
    SkillInstallRequestDto request,
    IServerStreamWriter<SkillInstallProgressEventDto> stream,
    CancellationToken cancellationToken)
{
    var process = new Process
    {
        StartInfo = new ProcessStartInfo
        {
            FileName = "npx",
            Arguments = $"skills add {request.FullReference} -g -y",
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false
        }
    };

    process.OutputDataReceived += async (sender, e) =>
    {
        await stream.WriteAsync(new SkillInstallProgressEventDto
        {
            EventType = "output",
            Data = e.Data ?? string.Empty
        });
    };

    process.Start();
    process.BeginOutputReadLine();
    await process.WaitForExitAsync(cancellationToken);
}
```

Ön uçta kullanıcılar terminal benzeri çıktıları gerçek zamanlı olarak görebilir, bu da deneyimi son derece sezgisel hale getirir. Gerçek zamanlı geri bildirim, insanların kendilerini rahat hissetmelerine yardımcı olur.

## Pratik Kılavuz

### Topluluk Becerisi Yükleme

Kurulumu gerçekleştirin `pua` örnek olarak beceri (popüler bir topluluk becerisidir):

1. Beceriler çekmecesini açın ve `Skill Gallery` sekme
2. Girin `pua` arama kutusunda
3. Beceri ayrıntılarını görüntülemek için arama sonucunu tıklayın
4. tıklayın `Install` düğme
5. Şuna geç: `Local Skills` kurulumun başarılı olduğunu onaylamak için sekme

Kurulum komutu `npx skills add tanweai/pua -g -y`ve sistem tüm ayrıntıları otomatik olarak yönetir. Tek tek attığınızda aslında o kadar fazla adım yok.

### Özel Güvenilir Kaynak Ekleme

Ekibinizin kendi beceri deposu varsa bunu güvenilir bir kaynak olarak ekleyebilirsiniz:

```yaml
providerId: "my-team"
displayName: "My Team Skills"
badgeLabel: "MyTeam"
isEnabled: true
sortOrder: 100
matchRules:
  - matchType: "prefix"
    value: "my-team/"
  - matchType: "exact"
    value: "my-team/special-skill"
```

Bu şekilde ekibinizin tüm becerileri güvenilir bir rozet gösterecek ve kullanıcıların bunları yüklemesini daha rahat hale getirecektir. Etiketler ve sinyaller insanların kendilerini daha güvende hissetmelerine yardımcı olur.

### Beceri Geliştirme Temelleri

Özel bir beceri oluşturmak aşağıdaki yapıyı gerektirir:

```
my-skill/
├── SKILL.md          # Skill metadata (YAML front matter)
├── index.ts          # Skill entry point
├── agents/           # Supported agent configuration
└── references/       # Reference resources
```

Bir örnek `SKILL.md` format:

```yaml
---
name: my-skill
description: A brief description of what this skill does
---

# My Skill

Detailed documentation...
```

## Notlar

1. **Ağ gereksinimleri**: beceri arama ve yükleme için erişim gerekir `api.hagicode.com` ve npm kayıt defteri
2. **Node.js sürümü**: Node.js 18 veya üzeri önerilir
3. **İzin gereksinimleri**: global npm kurulum izinleri gereklidir
4. **Eşzamanlılık kontrolü**: aynı beceri için aynı anda yalnızca bir yükleme veya kaldırma işlemi çalıştırılabilir
5. **Zaman aşımı ayarları**: Kurulum için varsayılan zaman aşımı 4 dakikadır ancak karmaşık senaryolar ayarlama gerektirebilir

Bu notlar sonuçta işlerin sorunsuz ilerlemesine yardımcı olmak için mevcuttur.
