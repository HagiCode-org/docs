## Temel Uygulama

### İki Özel Kelime Modunu Anlamak

Doubao konuşma tanıma API'si, özel kelimeleri yapılandırmak için iki yol sunar ve her birinin kendi ideal kullanım durumları ve avantajları vardır.

**Özel özel kelime modu** özel kelime metnini doğrudan `corpus.context` alan. Bu yaklaşım, özellikle bir ürün adının veya bir kişinin adının geçici olarak tanınması gibi az sayıda özel sözcüğü hızlı bir şekilde yapılandırmanız gereken senaryolar için uygundur. HagiCode'un uygulamasında, kullanıcı tarafından girilen çok satırlı özel kelime metnini bir dize listesine ayrıştırır, ardından bunu `context_data` Doubao API'sinin gerektirdiği dizi. Bu yaklaşım çok doğrudandır: Sisteme hangi kelimelere dikkat etmesi gerektiğini söylersiniz ve sistem de tam olarak bunu yapar.

**Platform özel kelime tablosu modu** şunu kullanır: `corpus.boosting_table_id` Doubao kendi kendine öğrenme platformunda önceden yapılandırılmış bir özel kelime tablosuna başvurmak için alan. Bu yaklaşım, çok sayıda özel sözcüğü yönetmeniz gereken senaryolar için uygundur. Doubao kendi kendine öğrenme platformunda özel kelime tabloları oluşturabilir ve bakımını yapabilir, ardından bunlara kimlikle başvuruda bulunabiliriz. Özel terimlerin sürekli olarak güncellenmesi ve korunması gereken HagiCode gibi bir proje için bu mod çok daha iyi yönetilebilirlik sunar. Özel kelimelerin sayısı arttıkça, bunları yönetmek için merkezi bir yere sahip olmak, bunları her seferinde manuel olarak girmekten çok daha iyidir.

İlginçtir ki bu iki mod birlikte de kullanılabilir. Doubao API, kombinasyon stratejisi tarafından kontrol edilen, aynı isteğe hem özel özel kelimelerin hem de bir platform özel kelime tablosu kimliğinin dahil edilmesini destekler. `combine_mode` parametre. Bu esneklik, HagiCode'un çok çeşitli karmaşık profesyonel terminoloji tanıma ihtiyaçlarını karşılamasına olanak tanır. Bazen birden fazla yaklaşımı birleştirmek daha iyi sonuçlar doğurur.

### Ön Uç Türü Tanımları ve Doğrulaması

HagiCode'un ön uç uygulamasında, eksiksiz bir özel kelime yapılandırma türleri ve doğrulama mantığı kümesi tanımladık. İlk bölüm tip tanımıdır:

```typescript
export interface HotwordConfig {
  contextText: string;           // Multi-line hotword text
  boostingTableId: string;      // Doubao platform hotword table ID
  combineMode: boolean;          // Whether to use both together
}
```

Bu basit arayüz, özel kelime özelliğine ilişkin tüm yapılandırma öğelerini içerir. Aralarında, `contextText` kullanıcıların en doğrudan etkileşime girdiği kısımdır: Kullanıcıların her satıra bir özel kelime öbeği girmesine izin veririz, bu da çok sezgiseldir. Kullanıcılardan her satıra bir terim girmelerini istemek, karmaşık bir yapılandırma biçimini anlamalarını sağlamaktan çok daha kolaydır.

Daha sonra doğrulama işlevi gelir. Doubao API sınırlamalarına dayanarak katı doğrulama kuralları tanımladık: en fazla 100 satır özel kelime metni, satır başına en fazla 50 karakter ve toplamda en fazla 5000 karakter; `boosting_table_id` En fazla 200 karakter olabilir ve yalnızca harf, sayı, alt çizgi ve kısa çizgi içerebilir. Bu sınırlar keyfi değildir; doğrudan resmi Doubao belgelerinden gelirler. API sınırları API sınırlarıdır ve bunlara uymamız gerekir.

```typescript
export function validateContextText(contextText: string): HotwordValidationResult {
  if (!contextText || contextText.trim().length === 0) {
    return { isValid: true, errors: [] };
  }

  const lines = contextText.split('\n').filter(line => line.trim().length > 0);
  const errors: string[] = [];

  if (lines.length > 100) {
    errors.push(`Hotword line count cannot exceed 100 lines; current count is ${lines.length}`);
  }

  const totalChars = contextText.length;
  if (totalChars > 5000) {
    errors.push(`Total hotword character count cannot exceed 5000; current count is ${totalChars}`);
  }

  for (let i = 0; i < lines.length; i++) {
    if (lines[i].length > 50) {
      errors.push(`Hotword on line ${i + 1} exceeds the 50-character limit`);
    }
  }

  return { isValid: errors.length === 0, errors };
}

export function validateBoostingTableId(boostingTableId: string): HotwordValidationResult {
  if (!boostingTableId || boostingTableId.trim().length === 0) {
    return { isValid: true, errors: [] };
  }

  const errors: string[] = [];

  if (boostingTableId.length > 200) {
    errors.push(`boosting_table_id cannot exceed 200 characters; current count is ${boostingTableId.length}`);
  }

  if (!/^[a-zA-Z0-9_-]+$/.test(boostingTableId)) {
    errors.push('boosting_table_id can contain only letters, numbers, underscores, and hyphens');
  }

  return { isValid: errors.length === 0, errors };
}
```

Bu doğrulama işlevleri, kullanıcı özel sözcükleri yapılandırdığında hemen çalıştırılarak sorunların mümkün olduğu kadar erken tespit edilmesini sağlar. Kullanıcı deneyimi açısından bakıldığında bu tür anlık geri bildirimler çok önemlidir. Kullanıcıların, gönderdikten sonra değil, yazarken neyin yanlış olduğunu bilmesi her zaman daha iyidir.

### Ön Uç Yapılandırma Kalıcılığı

HagiCode'un ön uç uygulamasında tarayıcının `localStorage` özel kelime yapılandırmasını saklamak için. Bu tasarım kararının arkasında birkaç düşünce vardı. İlk olarak, özel kelime yapılandırması son derece kişiselleştirilmiştir ve farklı kullanıcıların, alana özgü farklı ihtiyaçları olabilir. İkincisi, bu yaklaşım, ekstra veritabanı tabloları veya API uç noktaları gerektirmediği için arka uç uygulamasını basitleştirir. Son olarak, kullanıcılar bunu tarayıcıda bir kez yapılandırdıktan sonra, ayarlar sonraki kullanımlarda otomatik olarak yüklenebilir ve bu da çok kullanışlıdır. Basitçe söylemek gerekirse, bu en kolay yaklaşımdır.

```typescript
const HOTWORD_STORAGE_KEYS = {
  contextText: 'hotword-context-text',
  boostingTableId: 'hotword-boosting-table-id',
  combineMode: 'hotword-combine-mode',
} as const;

export const DEFAULT_HOTWORD_CONFIG: HotwordConfig = {
  contextText: '',
  boostingTableId: '',
  combineMode: false,
};

// Load hotword configuration
export function loadHotwordConfig(): HotwordConfig {
  const contextText = localStorage.getItem(HOTWORD_STORAGE_KEYS.contextText) || '';
  const boostingTableId = localStorage.getItem(HOTWORD_STORAGE_KEYS.boostingTableId) || '';
  const combineMode = localStorage.getItem(HOTWORD_STORAGE_KEYS.combineMode) === 'true';

  return { contextText, boostingTableId, combineMode };
}

// Save hotword configuration
export function saveHotwordConfig(config: HotwordConfig): void {
  localStorage.setItem(HOTWORD_STORAGE_KEYS.contextText, config.contextText);
  localStorage.setItem(HOTWORD_STORAGE_KEYS.boostingTableId, config.boostingTableId);
  localStorage.setItem(HOTWORD_STORAGE_KEYS.combineMode, String(config.combineMode));
}
```

Bu kodun mantığı açık ve nettir. Bizden okuyoruz `localStorage` Yapılandırmayı yüklerken ve şuraya yazın: `localStorage` kaydederken. Ayrıca, henüz bir yapılandırma olmadığında sistemin düzgün çalışabilmesi için varsayılan bir yapılandırma da sağlıyoruz. Sonuçta makul bir temerrüt olması gerekir.

### Yük Oluşturma Mantığı

Yük taşıma yapısı, tüm özel kelime özelliğinin özüdür. Özel kelime yapılandırmasını elde ettikten sonra, bunu Doubao API'sinin gerektirdiği JSON yapısına göre biçimlendirmemiz gerekir. Bu işlem, SDK isteği göndermeden önce gerçekleşir:

```csharp
private void AddCorpusToRequest(Dictionary<string, object> request)
{
    var corpus = new Dictionary<string, object>();

    // Add custom hotwords
    if (Config.HotwordContexts != null && Config.HotwordContexts.Count > 0)
    {
        corpus["context"] = new Dictionary<string, object>
        {
            ["context_type"] = "dialog_ctx",
            ["context_data"] = Config.HotwordContexts
                .Select(text => new Dictionary<string, object> { ["text"] = text })
                .ToList()
        };
    }

    // Add platform hotword table ID
    if (!string.IsNullOrEmpty(Config.BoostingTableId))
    {
        corpus["boosting_table_id"] = Config.BoostingTableId;
    }

    // Add corpus to the request only when it is not empty
    if (corpus.Count > 0)
    {
        request["corpus"] = corpus;
    }
}
```

Bu kod, dinamik olarak nasıl oluşturulacağını gösterir. `corpus` Yapılandırmaya dayalı alan. Önemli olan şunu eklemektir: `corpus` alanı yalnızca özel kelime yapılandırması gerçekten mevcut olduğunda kullanılabilir. Bu tasarım geriye dönük uyumluluk sağlar: hiçbir özel kelime yapılandırılmadığında istek yapısı öncekiyle tamamen aynı kalır. Geriye dönük uyumluluk önemlidir; özellik eklemek mevcut mantığı bozmamalıdır.

### WebSocket Parametre Geçişi

Ön uç ve arka uç arasında, özel kelime parametreleri WebSocket kontrol mesajları aracılığıyla iletilir. HagiCode, ön uç kayda başladığında özel kelime yapılandırmasını şuradan yükleyecek şekilde tasarlanmıştır: `localStorage` ve bunu bir WebSocket mesajı aracılığıyla arka uca gönderir.

```typescript
const controlMessage = {
  type: 'control',
  payload: {
    command: 'StartRecognition',
    contextText: '高血压\n糖尿病\n冠心病',
    boosting_table_id: 'medical_table',
    combineMode: false
  }
};
```

Burada dikkat edilmesi gereken bir ayrıntı var: ön uç, yeni satır karakterleriyle ayrılmış çok satırlı metni iletir ve arka ucun bunu ayrıştırması gerekir. Arka uç WebSocket işleyicisi bu parametreleri ayrıştırır ve bunları SDK'ya iletir:

```csharp
private async Task HandleControlMessageAsync(
    string connectionId,
    DoubaoSession session,
    ControlMessage message)
{
    if (message.Payload is SessionControlRequest controlRequest)
    {
        // Parse hotword parameters
        string? contextText = controlRequest.ContextText;
        string? boostingTableId = controlRequest.BoostingTableId;
        bool? combineMode = controlRequest.CombineMode;

        // Parse multi-line text into a hotword list
        if (!string.IsNullOrEmpty(contextText))
        {
            var hotwords = contextText
                .Split('\n', StringSplitOptions.RemoveEmptyEntries)
                .Select(s => s.Trim())
                .Where(s => s.Length > 0)
                .ToList();

            session.HotwordContexts = hotwords;
        }

        session.BoostingTableId = boostingTableId;
    }
}
```

Bu tasarımla, özel kelime yapılandırmasının ön uçtan arka uca geçişi net ve verimli hale gelir. Bunda özellikle gizemli bir şey yok; veriler basitçe katman katman aktarılır.

### Platform Özel Kelime Tablosunu Yapılandırma

Çok sayıda özel kelimeyi yönetmeniz gerekiyorsa veya özel kelimelerin sık sık güncellenmesi gerekiyorsa, platformun özel kelime tablosu modu daha uygun olacaktır. Öncelikle Doubao kendi kendine öğrenme platformunda bir özel kelime tablosu oluşturun ve oluşturulan `boosting_table_id`, ardından bu kimliği HagiCode ayarları sayfasına girin.

Doubao kendi kendine öğrenme platformu, özel kelimeler için toplu içe aktarma ve kategorize edilmiş yönetim gibi yetenekler sağlar; bu, çok sayıda özel terminolojiyi yönetmesi gereken ekipler için çok pratiktir. Özel kelimeleri platformda yöneterek bunları merkezi olarak koruyabilir ve güncellemeleri tutarlı bir şekilde yayınlayabilirsiniz. Özel kelime listesi büyüdüğünde, onu yönetmek için tek bir yere sahip olmak, her seferinde manuel giriş yapmaktan çok daha pratiktir.

### Kombinasyon Modunu Kullanma

Bazı karmaşık senaryolarda, hem özel özel kelimeleri hem de bir platform özel kelime tablosunu aynı anda kullanmanız gerekebilir. Bu durumda, her ikisini de HagiCode'da yapılandırın ve "Kombinasyon Modu" anahtarını etkinleştirin.

Kombinasyon modunda, Doubao API her iki özel kelime kaynağını aynı anda dikkate alır, dolayısıyla tanıma doğruluğu genellikle her iki kaynağın tek başına kullanılmasından daha yüksektir. Bununla birlikte, kombinasyon modunun istek karmaşıklığını arttırdığını belirtmekte fayda var; bu nedenle, pratik testlerden sonra etkinleştirilip etkinleştirilmeyeceğine karar vermek en iyisidir. Daha fazla karmaşıklık ancak gerçek dünyadaki sonuçların bunu haklı çıkarması durumunda buna değer.

### Kod Entegrasyon Örnekleri

```typescript
import {
  loadHotwordConfig,
  saveHotwordConfig,
  validateHotwordConfig,
  parseContextText,
  getEffectiveHotwordMode,
  type HotwordConfig
} from '@/types/hotword';

// Load and validate configuration
const config = loadHotwordConfig();
const validation = validateHotwordConfig(config);

if (!validation.isValid) {
  console.error('Hotword configuration validation failed:', validation.errors);
  return;
}

// Parse hotword text
const hotwords = parseContextText(config.contextText);
console.log('Parsed hotwords:', hotwords);

// Get effective hotword mode
const mode = getEffectiveHotwordMode(config);
console.log('Current hotword mode:', mode);
```

## Akılda tutulması gerekenler

Özel kelime özelliğini uygularken ve kullanırken özel dikkat gerektiren birkaç nokta vardır.

Birincisi karakter sınırıdır. Doubao API'nin özel kelimeler üzerinde satır sayısı, satır başına karakter ve toplam karakter sayısı dahil olmak üzere katı kısıtlamaları vardır. Herhangi bir sınır aşılırsa API bir hata döndürür. HagiCode'un ön uç uygulamasında, kullanıcı girişi sırasında doğrulama işlevleri aracılığıyla bu kısıtlamaları kontrol ederiz, bu da geçersiz yapılandırmaların arka uca gönderilmesini engeller. Sorunları erken yakalamak her zaman API'nin başarısız olmasını beklemekten daha iyidir.

İkincisi formattır `boosting_table_id`. Bu alan yalnızca harflere, sayılara, alt çizgilere ve kısa çizgilere izin verir ve boşluk veya diğer özel karakterleri içeremez. Doubao kendi kendine öğrenme platformunda özel kelime tablosu oluştururken adlandırma kurallarına uyduğunuzdan emin olun. Bu tür katı format doğrulama API'ler için yaygındır.

Üçüncüsü geriye dönük uyumluluktur. Özel kelime parametreleri tamamen isteğe bağlıdır. Hiçbir özel kelime yapılandırılmamışsa sistem tam olarak daha önce olduğu gibi davranır. Bu tasarım, mevcut kullanıcıların hiçbir şekilde etkilenmemesini sağlarken, kademeli geçiş ve yükseltmeleri de kolaylaştırıyor. Özellik eklemek önceki mantığı bozmamalı.

Son olarak hata yönetimi var. Özel kelime yapılandırması geçersiz olduğunda Doubao API, ilgili hata mesajlarını döndürür. HagiCode'un uygulaması, geliştiricilerin sorunları gidermesine yardımcı olmak için ayrıntılı günlükler kaydeder. Aynı zamanda ön uç, kullanıcıların yapılandırmayı düzeltmesine yardımcı olmak için kullanıcı arayüzünde doğrulama hatalarını görüntüler. İyi hata yönetimi doğal olarak daha iyi bir kullanıcı deneyimine yol açar.

## Sonuç

Bu makale aracılığıyla, Doubao konuşma tanıma özel kelimelerinin HagiCode projesinde uygulanmasına yönelik eksiksiz çözüme ayrıntılı bir giriş sağladık. Bu çözüm, gereksinim analizi ve teknik seçimden kod uygulamaya kadar tüm süreci kapsayarak geliştiricilere referans olarak kullanabilecekleri pratik bir örnek sunar.

Temel noktaları aşağıdaki gibi özetleyebiliriz. İlk olarak Doubao API, hem özel özel kelimeleri hem de platform özel kelime tablolarını destekler ve bunlar bağımsız olarak veya birlikte kullanılabilir. Üçüncüsü, arka uç, özel kelime parametrelerini dinamik olarak oluşturarak geçirir. `corpus` güçlü geriye dönük uyumluluğu koruyarak alan.
