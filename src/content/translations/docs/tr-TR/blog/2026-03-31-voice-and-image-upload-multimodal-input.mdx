1. **Yazmak çok yavaş**: Hata mesajları veya kullanıcı arayüzü sorunları gibi karmaşık sorunlarda her şeyi yazmak yarım dakika sürebilir, yüksek sesle söylemek ise on saniye sürebilir. Bu boşluk gerçektir.

"Bir resim bin kelimeye bedeldir" eski bir söz olabilir ama hala geçerliliğini koruyor. Yapay zekanın sorunu doğrudan "görmesine" izin vermek, onu yıllarca tanımlamaktan çok daha nettir.

3. **Etkileşim doğal olmalı**: Modern yapay zeka asistanları metin, ses ve görüntüleri desteklemelidir. Kullanıcılar hangi giriş yöntemini en doğal buluyorsa onu seçebilmelidir.

Bu nedenle, Temsilci etkileşimlerini daha rahat hale getirmek için HagiCode'a konuşma tanıma ve görüntü yükleme desteği eklemeye karar verdik. Kullanıcılar biraz daha az yazabilirse bu zaten bir kazançtır.

## Analiz

### Konuşma Tanımadaki Teknik Zorluklar

Konuşma tanımayı geliştirirken zor bir sorunla karşılaştık: **tarayıcı WebSocket API'si özel HTTP başlıklarını desteklemiyor**.

Seçtiğimiz konuşma tanıma hizmeti ByteDance'in Doubao Konuşma Tanıma API'siydi. Ne yazık ki bu API, aşağıdaki gibi kimlik doğrulama bilgileri gerektiriyor: `accessToken` ve `secretKey` HTTP başlıklarından geçirilecek. Bu durum anında teknik bir çatışma yarattı:

```javascript
// The browser WebSocket API does not support this approach
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // Not supported
  }
});
```

Temelde iki seçeneğimiz vardı:

1. **URL sorgu parametresi yaklaşımı**: kimlik doğrulama bilgisini URL'ye yerleştirin
   - Avantajı, uygulanmasının basit olmasıdır
   - Bunun dezavantajı, kimlik bilgilerinin güvensiz olan ön uca açık olmasıdır; bazı API'ler ayrıca başlık tabanlı kimlik doğrulaması gerektirir

2. **Arka uç proxy yaklaşımı**: arka uçta bir WebSocket proxy'si uygulayın
   - Avantajı, kimlik bilgilerinin arka uçta güvenli bir şekilde saklanması ve çözümün API gereklilikleriyle tamamen uyumlu olmasıdır.
   - Dezavantajı ise uygulamanın biraz daha karmaşık olmasıdır

Sonunda **arka uç proxy yaklaşımını** seçtik. Güvenlik taviz vereceğiniz bir şey değildir.

### Resim Yüklemeleri için İşlevsel Gereksinimler

Resim yükleme gereksinimlerimiz aslında oldukça basitti:

1. **Birden fazla yükleme yöntemi**: Bir dosyayı seçmek için tıklayın, sürükleyip bırakın ve panodan yapıştırın
2. **Dosya doğrulama**: tür kısıtlamaları (PNG, JPG, WebP, GIF) ve boyut sınırları (5-10 MB) temel gereksinimlerdir
3. **Kullanıcı deneyimi**: ilerleme durumunu, önizlemeleri ve hata mesajlarını yükleyin, böylece kullanıcılar her zaman ne olduğunu bilir
4. **Güvenlik**: Sunucu tarafı doğrulaması ve kötü amaçlı dosya yüklemelerine karşı koruma önemlidir

## Çözüm

### Konuşma Tanıma: WebSocket Proxy Mimarisi

Konuşma tanıma için üç katmanlı bir mimari tasarladık ve işe yarayan bir yol bulduk:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**Temel bileşen uygulamaları**:

1. **Ön uç AudioWorklet işlemci**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Resample to 16 kHz (required by the Doubao API)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Accumulate samples into 500 ms chunks
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Convert to 16-bit PCM and send
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **Arka uç WebSocket işleyicisi** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **Ön uç `VoiceTextArea` bileşen**:

```tsx
export const VoiceTextArea = forwardRef<HTMLTextAreaElement, VoiceTextAreaProps>(
  ({ value, onChange, onTextRecognized, maxDuration }, ref) => {
    const { isRecording, interimText, volume, duration, startRecording, stopRecording } =
      useVoiceRecording({ onTextRecognized, maxDuration });

    return (

        </button>
        {/* Text input area */}
        <textarea value={displayValue} onChange={handleChange} />
      </div>
    );
  }
);
```

### Görüntü Yüklemeleri: Çok Yöntemli Yükleme Bileşeni

Kullanıcıların karşılaştığı en yaygın senaryoları kapsayan, üç yükleme yönteminin tamamını destekleyen tam özellikli bir görüntü yükleme bileşeni oluşturduk.

**Temel özellikler**:

1. **Üç yükleme yöntemi**:

```tsx
// Click to upload
const handleClick = () => fileInputRef.current?.click();

// Drag-and-drop upload
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Clipboard paste
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **Ön uç doğrulama**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **Arka uç yükleme işleyicisi** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Validation
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Save file
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## Pratik Kılavuz

### Konuşma Tanıma Nasıl Kullanılır

1. **Konuşma tanıma hizmetini yapılandırın**:
   - Konuşma tanıma ayarları sayfasını açın
   - Doubao Konuşmasını Yapılandırma `AppId` ve `AccessToken`
   - Etki alanına özgü terimler için tanıma doğruluğunu artırmak amacıyla isteğe bağlı olarak özel sözcükleri yapılandırın

2. **Giriş kutusunda kullanın**:
   - Giriş kutusunun sol tarafındaki mikrofon simgesine tıklayın
   - Dalga biçimi animasyonu göründükten sonra konuşmaya başlayın
   - Kaydı durdurmak için simgeye tekrar tıklayın
   - Tanınan metin otomatik olarak imleç konumuna eklenir

3. **Özel kelime yapılandırma örneği**:

```text
TypeScript
React
useState
useEffect
```

### Resim Yüklemeleri Nasıl Kullanılır?

1. **Yükleme yöntemleri**:
   - Bir dosya seçmek için yükle düğmesini tıklayın
   - Bir resmi doğrudan yükleme alanına sürükleyin
   - Kullanım `Ctrl+V` panodan bir ekran görüntüsünü yapıştırmak için

2. **Desteklenen formatlar**: PNG, JPG, JPEG, WebP, GIF
3. **Boyut sınırı**: Varsayılan olarak 5 MB (yapılandırılabilir)

### Notlar

1. **Konuşma tanıma**:
   - Mikrofon izni gerekli
   - Mümkün olduğunda sessiz bir ortamda kullanın
   - Desteklenen maksimum kayıt süresi varsayılan olarak 300 saniyedir (yapılandırılabilir)

2. **Resim yüklemeleri**:
   - Yalnızca yaygın resim formatları desteklenir
   - Dosya boyutu sınırlarına dikkat edin
   - Yüklenen resimler otomatik olarak bir önizleme URL'si alır

3. **Güvenlik hususları**:
   - Konuşma tanıma kimlik bilgileri arka uçta saklanır
   - Resim yüklemeleri sunucu tarafı doğrulamasından geçer
   - Üretim ortamlarında HTTPS/WSS önerilir
