## Temel Uygulama Fikirleri

HagiCode kod tabanını analiz ettikten sonra birkaç temel uygulama modelini aşağıda özetledik. Her biri farklı bir senaryoya uyuyor, yani her biri bir sebeple var oluyor.

### 1. Konik bir degradeyle dönen parıltı (en yaygın)

Bu, kenar ışığı süpürme efektini uygulamanın klasik yoludur. Temel fikir CSS kullanmaktır `conic-gradient` konik bir degrade oluşturmak için onu sürekli olarak döndürün. Geceleri yanan bir sokak lambası gibi, dönüp duruyor.

**Temel unsurlar:**
- Şunu kullanın: `::before` parıltı katmanını oluşturmak için sözde eleman
- Kullanım `conic-gradient` degrade renk dağılımını tanımlamak için
- Şunu kullanın: `::after` merkez alanı maskelemek için sözde öğe (isteğe bağlı)
- Kullanım `@keyframes` döndürme animasyonunu uygulamak için

### 2. Yan kızdırma çizgisi

Bu, liste öğelerindeki durum göstergeleri için iyi çalışır. Kenarlığın tamamını hareketlendirmek yerine öğenin bir tarafında ince, parlak bir çizgi oluşturursunuz. Bazen biraz ışık yeterlidir. Tüm dünyayı aydınlatmanıza gerek yok.

**Temel unsurlar:**
- Kesinlikle konumlandırılmış ince bir çizgi öğesi
- Kullanım `box-shadow` Parıltı efekti yaratmak için
- Kullanım `scale` ve `opacity` nefes alma animasyonu için

### 3. Kutu gölgeli parlak arka plan

Tam tarama efektine ihtiyacınız yoksa ve yalnızca yumuşak bir arka plan parıltısı istiyorsanız, birden fazla katmanlama yapın. `box-shadow` değerler yeterlidir. Bazen daha basit olan seçenek daha iyidir.

### 4. Erişilebilirlik desteği

Bu kısmı gözden kaçırmak kolaydır ancak son derece önemlidir. Her animasyon aşağıdakileri hesaba katmalıdır: `prefers-reduced-motion` medya sorgusu ve animasyon istemeyen kullanıcılar için statik bir alternatif sunar. Herkes sürekli hareketten hoşlanmaz ve bu tercihe saygı duymak önemlidir.

## Uygulama Seçenekleri

### 1. Seçenek: Dönen konik degrade kenarlık (önerilir)

Bu, geniş kenarlık ışığı efektinin en eksiksiz uygulamasıdır ve aynı zamanda HagiCode'da en sık kullandığımız seçenektir. Sonuçta, eğer bir şey iyi çalışıyorsa neden değiştirilsin ki?

```css
/* Parent container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotating glow layer */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Mask layer (optional, for creating a hollow border effect) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Bu seçeneğin ardındaki prensip oldukça basittir: ana kaptan daha büyük bir sözde eleman oluşturun, üzerine konik bir degrade çizin ve onu sürekli döndürün. Ana kapsayıcının kullandığı `overflow: hidden`böylece yalnızca sınırın etrafından geçen ışık görünür kalır. Bu biraz pencereden sokak lambasını izlemeye benziyor. Sadece geçip giden küçük dilimi görüyorsunuz.

### Seçenek 2: Basitleştirilmiş dönen ışık kenarlığı

Tam efekte ihtiyacınız yoksa HagiCode ayrıca daha hafif bir yardımcı program sınıfı sürümü de içerir. Bazen daha basit yaklaşım gerçekten daha iyidir.

```css
/* Rotating light border utility class */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Accessibility support */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

dikkat edin `will-change: transform` burada. Tarayıcıya "Bu öğe değişmeye devam edecek" der, böylece tarayıcı bazı optimizasyonları önceden hazırlayabilir ve animasyonu daha düzgün tutabilir. Önceden hazırlanmak genellikle son dakikada çabalamaktan daha iyidir.

### Seçenek 3: Yan kızdırma çizgisi

Bu özellikle liste öğesi durum göstergeleri için uygundur ve tam olarak HagiCode oturum listesinin kullandığı şeydir. Pek çok öğe arasında ince bir çizgi hala öne çıkabilir. Bu başlı başına bir hayat dersi gibi geliyor.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Bu kullanır `isolation: isolate` yeni bir yığınlama bağlamı oluşturmak için `z-index` Her katmanın görüntülenme sırasını kontrol etmek için. `pointer-events: none` aynı zamanda esastır. Aksi takdirde sözde öğe kullanıcı tıklamalarını engeller. Bazı şeyler güzel görünebilir ama yine de engel olmamalıdırlar.

### Seçenek 4: Bunu bir React bileşenine sarın

Projeniz React kullanıyorsa bu mantığı, özellikle erişilebilirlik yönetimini bir bileşene sarabilirsiniz. Bir kere yazın, defalarca kullanın. Bütün mesele bu.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Eşleşen CSS modülü:

```css
/* GlowBorder.module.css */

/* Animated version */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Static version (accessibility) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

 `useReducedMotion` kancadan takmak `framer-motion` kullanıcının sistem tercihini otomatik olarak algılar. Kullanıcı azaltılmış hareketi etkinleştirmişse, `true`ve bileşen bunun yerine statik sürümü gösterir. Kullanıcının tercihine saygı duymak, gösterişli bir etki yaratmaktan daha önemlidir.

## Öğrenilen Pratik Dersler

Bunlar HagiCode'u oluştururken öğrendiğimiz derslerden bazıları. Onlara savaş yaraları da diyebilirsiniz. Umarım bazı sapmalardan kaçınmanıza yardımcı olurlar.

### 1. Tema değişken sistemi

CSS değişkenleri çoklu tema desteğini özellikle kullanışlı hale getirir. Hiç kimse tema her değiştiğinde bir kod yığınını düzenlemek istemez.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Usage */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

Bu şekilde, temaları değiştirmek yalnızca sınıfın değiştirilmesini gerektirir. `html` öğesi ve her animasyon rengi otomatik olarak güncellenir. Bir kod tabanı, iki stil. Biz de tam olarak bunu istiyoruz.

### 2. Performans optimizasyonu

**Kullanım `will-change` tarayıcının optimize edilmesini ipucu olarak vermek için:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Tarayıcıya önceden bildirin, optimizasyon yapmanıza yardımcı olacaktır. Hayattaki pek çok şey biraz hazırlıkla daha iyi sonuç verir.

**Büyük öğeler üzerinde karmaşık kutu gölgeleri kullanmaktan kaçının:**

```css
/* Not ideal - using a blurred shadow on a large element */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Better - use a pseudo-element to limit the glowing area */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Bunu HagiCode'da test ettik. Büyük bir karta doğrudan bulanık bir gölge eklemek, kaydırma kare hızlarını 30 fps'nin altına düşürdü. Sahte öğeye geçiş, işleri sabit 60 fps'ye geri getirdi. Kullanıcılar bu farkı kesinlikle hissedebilirler.

### 3. Erişilebilirlik

Bunu gerçekten atlamamalısınız. Bazı kullanıcılar animasyonun baş döndürücü veya dikkat dağıtıcı olduğunu düşünüyor ve onların tercihlerine saygı duymak iyi bir ürün oluşturmanın bir parçası. Güzel şeylerin herkese empoze edilmesine gerek yoktur.

**CSS medya sorgusu:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Provide a static fallback */
    opacity: 1;
  }
}
```

**React'te kullanıcı tercihini tespit edin:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Yoğunluk seviyesi kontrolü

HagiCode'daki Token çıktı göstergesi, gerçek zamanlı çıktıya dayalı olarak farklı parlak renkler gösterir ve bu, dinamik olarak uygulanır. Farklı durumlar farklı şekilde ifade edilmelidir.

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Dikkat edilmesi gerekenler

Hala dikkat etmeye değer birkaç detay var, çünkü bu sorunları zor yoldan keşfettiğinizde artık çok geç olmuş oluyor.

| Dikkat edilmesi gerekenler | Açıklama |
|---------|------|
| **z-endeksi yönetimi** | Kızdırma katmanı uygun bir şekilde kullanılmalıdır. `z-index` böylece içerik etkileşimini engellemez |
| **işaretçi olayları** | Işıma sözde elemanı ayarlanmalıdır `pointer-events: none` |
| **Sınır taşması** | Ana kapsayıcının ihtiyaçları `overflow: hidden`veya sözde öğe boyutunu ayarlamanız gerekir |
| **Performans etkisi** | Karmaşık animasyonlar mobil cihazlarda performansa zarar verebilir; bu nedenle dikkatli bir şekilde test edin |
| **Karanlık mod** | Parıldayan rengin koyu arka planlarda açıkça görülebildiğinden emin olun |
| **Tema değiştirme** | Tema değiştiğinde animasyon renklerinin doğru şekilde güncellenmesi için CSS değişkenlerini kullanın |

### 6. Hata ayıklama ipuçları

Sözde öğelerin geliştirici araçlarında bulunması biraz zor olabilir, bu nedenle konumu kontrol etmek için geçici olarak bir kenarlık ekleyebilirsiniz.

```css
/* Temporarily show pseudo-element boundaries for debugging */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Konumlandırmayı tamamladıktan sonra, bu satırı açıklamayı veya kaldırmayı unutmayın. Aksi takdirde üretim oldukça hızlı bir şekilde garipleşebilir. Bazı şeylerin geliştirme aşamasında bırakılması daha iyidir.
