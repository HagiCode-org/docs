## CLI Çağrılarından SDK Entegrasyonuna: .NET Projelerinde GitHub Copilot için En İyi Uygulamalar

> Komut satırı çağrısından resmi SDK entegrasyonuna kadar olan yükseltme yolu oldukça uzun bir yolculuktu. Bugün HagiCode'u oluştururken karşılaştığımız tuzakları ve öğrendiklerimizi paylaşmak istiyorum.

## Arka plan

GitHub Copilot SDK'sı 2025'te resmi olarak piyasaya sürüldükten sonra onu yapay zeka yetenek katmanımıza entegre etmeye başladık. Bundan önce, proje esas olarak Copilot CLI komut satırı aracını doğrudan çağırarak GitHub Copilot'u kullanıyordu ancak bu yaklaşımın birkaç belirgin sorunu vardı:

- **Karmaşık süreç yönetimi**: CLI süreç yaşam döngüsünü, başlatma zaman aşımlarını ve süreç temizliğini manuel olarak yönetmek zorunda kaldık. İşlemler uyarı vermeden çökebilir.
- **Eksik olay işleme**: Ham CLI çağrısı, model akıl yürütme ve araç yürütme yoluyla ayrıntılı olayların yakalanmasını zorlaştırır. Düşünme süreci olmadan sadece sonucu görmek gibidir.
- **Zor oturum yönetimi**: Oturumun yeniden kullanılması ve kurtarılması için etkili bir mekanizma yoktu, bu nedenle her etkileşimin baştan başlaması gerekiyordu.
- **Uyumluluk sorunları**: CLI argümanları sık sık değişiyordu; bu da, bu parametreler için uyumluluk mantığını sürekli olarak korumamız gerektiği anlamına geliyordu.

Bu sorunlar, özellikle model mantığını takip etmemiz gerektiğinde, günlük gelişimde giderek daha belirgin hale geldi (`thinking`) ve gerçek zamanlı olarak takım yürütme durumu. Bu noktada daha alt düzeyde ve daha kapsamlı bir entegrasyon yaklaşımına ihtiyacımız olduğu ortaya çıktı.

## Mimari Tasarım

Proje, CLI çağrısının sınırlamalarını gidermek için katmanlı bir mimari kullanıyor:

```
┌─────────────────────────────────────────────────────────┐
│  hagicode-core (Orleans Grains + AI Provider Layer)    │
│  - CopilotAIProvider: Converts AIRequest to CopilotOptions │
│  - GitHubCopilotGrain: Orleans distributed execution interface │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  HagiCode.Libs (Shared Provider Layer)                 │
│  - CopilotProvider: CLI Provider interface implementation │
│  - ICopilotSdkGateway: SDK invocation abstraction      │
│  - GitHubCopilotSdkGateway: SDK session management and event dispatch │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  GitHub Copilot SDK (Official .NET SDK)                │
│  - CopilotClient: SDK client                           │
│  - CopilotSession: Session management                  │
│  - SessionEvent: Event stream                          │
└─────────────────────────────────────────────────────────┘
```

Bu katmanlı tasarım birçok pratik teknik avantaj sağlar:

1. **Endişelerin ayrılması**: Temel iş mantığı, SDK uygulama ayrıntılarından ayrılmıştır.
2. **Test edilebilirlik**: `ICopilotSdkGateway` arayüz birim testini basitleştirir.
3. **Yeniden kullanılabilirlik**: `HagiCode.Libs` birden fazla projeye referans verilebilir.
4. **Bakımlanabilirlik**: SDK yükseltmeleri yalnızca Ağ Geçidi katmanında değişiklik yapılmasını gerektirir, üst katmanlara ise dokunulmaz.

## Temel Uygulama

### Kimlik Doğrulama Akışı

Kimlik doğrulama, SDK entegrasyonunun ilk ve en önemli adımıdır. Kimlik doğrulama başarısız olursa başka hiçbir şeyin önemi yoktur. Birden çok kimlik doğrulama kaynağını destekleyen esnek bir kimlik doğrulama yapılandırması tasarladık:

```csharp
// CopilotProvider.cs - Authentication source configuration
public class CopilotOptions
{
    public bool UseLoggedInUser { get; set; } = true;
    public string? GitHubToken { get; set; }
    public string? CliUrl { get; set; }
}

// Convert to SDK request
return new CopilotSdkRequest(
    GitHubToken: options.AuthSource == CopilotAuthSource.GitHubToken
        ? options.GitHubToken
        : null,
    UseLoggedInUser: options.AuthSource != CopilotAuthSource.GitHubToken
);
```

Bu tasarımın faydaları oldukça açıktır:

- Masaüstü senaryolarına iyi uyum sağlayan, belirteç gerektirmeden oturum açmış kullanıcı modunu destekler.
- Sunucu tarafı dağıtımları ve merkezi yönetim için uygun olan GitHub Token modunu destekler.
- Kurumsal proxy yapılandırmasına yardımcı olan Copilot CLI URL'sinin geçersiz kılınmasını destekler.

Uygulamada bu esnek kimlik doğrulama modeli, farklı dağıtım senaryolarındaki yapılandırmayı büyük ölçüde basitleştirdi. Masaüstü istemcisi her kullanıcının kendi Copilot oturum açma durumunu kullanabilirken, sunucu tarafı kimlik doğrulamayı belirteçler aracılığıyla merkezi olarak yönetebilir.

### Olay Akışını İşleme

SDK'nın en güçlü özelliklerinden biri tam olay akışı yakalamadır. Her türlü SDK olayını gerçek zamanlı olarak işleyebilecek bir olay gönderme sistemi uyguladık:

```csharp
// GitHubCopilotSdkGateway.cs - Core logic for event dispatch
internal static SessionEventDispatchResult DispatchSessionEvent(
    SessionEvent evt, bool sawDelta)
{
    switch (evt)
    {
        case AssistantReasoningEvent reasoningEvent:
            // Capture the model reasoning process
            events.Add(new CopilotSdkStreamEvent(
                CopilotSdkStreamEventType.ReasoningDelta,
                Content: reasoningEvent.Data.Content));
            break;

        case ToolExecutionStartEvent toolStartEvent:
            // Capture the start of a tool invocation
            events.Add(new CopilotSdkStreamEvent(
                CopilotSdkStreamEventType.ToolExecutionStart,
                ToolName: toolStartEvent.Data.ToolName,
                ToolCallId: toolStartEvent.Data.ToolCallId));
            break;

        case ToolExecutionCompleteEvent toolCompleteEvent:
            // Capture tool completion and its result
            events.Add(new CopilotSdkStreamEvent(
                CopilotSdkStreamEventType.ToolExecutionEnd,
                Content: ExtractToolExecutionContent(toolCompleteEvent)));
            break;

        default:
            // Preserve unhandled events as RawEvent
            events.Add(new CopilotSdkStreamEvent(
                CopilotSdkStreamEventType.RawEvent,
                RawEventType: evt.GetType().Name));
            break;
    }
}
```

Bu uygulamanın değeri önemlidir:

- **Model muhakeme sürecinin tam olarak yakalanması** (`thinking`): kullanıcılar yalnızca nihai yanıtı değil, yapay zekanın nasıl mantık yürüttüğünü de görebilir.
- **Takım yürütme durumunun gerçek zamanlı takibi**: Hangi araçların çalıştığını, ne zaman bittiğini ve neye geri döndüğünü biliyoruz.
- **Sıfır olay kaybı**: geri çekilerek `RawEvent`, her olayın kaydedilmesini sağlıyoruz.

HagiCode'da bu ince taneli olaylar, özellikle karmaşık görevlerde hata ayıklarken kullanıcıların yapay zekanın dahili olarak nasıl çalıştığını anlamalarına yardımcı olur.


Bu bize çeşitli faydalar sağlar:

- Çalışma zamanı hatalarını önlemek için uyumsuz CLI bağımsız değişkenlerini otomatik olarak filtreler.
- Geliştiricilerin sorunları hızlı bir şekilde bulmasına yardımcı olmak için net tanılama mesajları üretir.
- SDK'yı kararlı tutar ve CLI parametrelerindeki değişikliklerden korunur.

Yükseltme sırasında bu uyumluluk mekanizması sorunsuz geçiş yapmamıza yardımcı oldu. Mevcut yapılandırma dosyaları hâlâ kullanılabiliyordu ve yalnızca tanılama çıktısına göre kademeli ayarlamalar yapılması gerekiyordu.

### Çalışma Zamanı Havuzu Oluşturma

Copilot SDK oturumları oluşturmak nispeten pahalıdır ve oturumları çok sık oluşturup yok etmek performansa zarar verir. Bunu çözmek için bir oturum havuzu yönetim sistemi uyguladık:

```csharp
// CopilotProvider.cs - Session pool management
await using var lease = await _poolCoordinator.AcquireCopilotRuntimeAsync(
    request,
    async ct => await _gateway.CreateRuntimeAsync(sdkRequest, ct),
    cancellationToken);

if (lease.IsWarmLease)
{
    // Reuse an existing session
    yield return CreateSessionReusedMessage();
}

await foreach (var eventData in lease.Entry.Resource.SendPromptAsync(...))
{
    yield return MapEvent(eventData);
}
```

Oturum havuzu oluşturmanın faydaları şunlardır:

- **Oturumun yeniden kullanımı**: aynı olan istekler `sessionId` mevcut oturumları yeniden kullanabilir ve başlangıç yükünü azaltabilir.
- **Oturum kurtarma desteği**: Bir ağ kesintisinden sonra önceki oturum durumu geri yüklenebilir.
- **Otomatik havuz yönetimi**: Kaynak sızıntılarını önlemek için süresi dolan oturumlar otomatik olarak temizlenir.

HagiCode'da, oturum havuzu oluşturma, özellikle sürekli konuşmalar için yanıt verme hızını gözle görülür biçimde artırdı.

### Orleans Entegrasyonu

HagiCode, dağıtılmış çerçeve olarak Orleans'ı kullanıyor ve Copilot SDK'sını Orleans Grains'e entegre ettik:

```csharp
// GitHubCopilotGrain.cs - Distributed execution
public async IAsyncEnumerable<GitHubCopilotResponse> ExecuteCommandStreamAsync(
    string command,
    CancellationToken token = default)
{
    var provider = await aiProviderFactory.GetProviderAsync(AIProviderType.GitHubCopilot);

    await foreach (var chunk in provider.SendMessageAsync(request, null, token))
    {
        // Map to the unified response format
        yield return BuildChunkResponse(chunk, startedAt);
    }
}
```

Orleans entegrasyonunun avantajları oldukça önemlidir:

- **Birleşik AI Sağlayıcı soyutlaması**: Farklı AI sağlayıcıları arasında geçiş yapmak kolaylaşır.
- **Çok kiracılı izolasyon**: Farklı kullanıcılar için yardımcı pilot oturumları birbirinden izole kalır.
- **Kalıcı oturum durumu**: Oturum durumu, sunucu yeniden başlatıldıktan sonra bile geri yüklenebilir.

Çok sayıda eş zamanlı isteğin karşılanması gereken senaryolar için Orleans güçlü ölçeklenebilirlik sağlar.

## Pratik Kılavuz

### Yapılandırma Örneği

İşte tam bir yapılandırma örneği:

```json
{
  "AI": {
    "Providers": {
      "Providers": {
        "GitHubCopilot": {
          "Enabled": true,
          "ExecutablePath": "copilot",
          "Model": "gpt-5",
          "WorkingDirectory": "/path/to/project",
          "Timeout": 7200,
          "StartupTimeout": 30,
          "UseLoggedInUser": true,
          "NoAskUser": true,
          "Permissions": {
            "AllowAllTools": false,
            "AllowedTools": ["Read", "Bash", "Grep"],
            "DeniedTools": ["Edit"]
          }
        }
      }
    }
  }
}
```

### Kullanım Notları

Gerçek dünya kullanımında dikkat edilmesi gereken birkaç noktayı özetledik:

**Başlatma zaman aşımı yapılandırması**: Copilot CLI'nin ilk başlatılması nispeten uzun sürebilir; bu nedenle, `StartupTimeout` en az 30 saniyeye kadar. Bu ilk giriş ise daha da uzun sürebilir.

**İzin yönetimi**: Üretim ortamlarında izin yönetimi kullanmaktan kaçının `AllowAllTools: true`. Şunu kullanın: `AllowedTools` Hangi araçların kullanılabilir olduğunu kontrol etmek ve izin verilenler listesi `DeniedTools` Tehlikeli işlemleri engellemek için reddetme listesi. Bu, yapay zekanın riskli komutları yürütmesini etkili bir şekilde önler.

**Oturum yönetimi**: Aynı özelliklere sahip istekler `sessionId` oturumları otomatik olarak yeniden kullanın. Oturum durumu şu şekilde sürdürülür: `ProviderSessionId`. İptal işlemi şu şekilde yayılır: `CancellationTokenSource`.

**Teşhis çıktısı**: Uyumsuz CLI bağımsız değişkenleri şu türden mesajlar üretir: `diagnostic`. Ham SDK etkinlikleri şu şekilde korunur: `event.raw`. Hata mesajları, sorun gidermeyi kolaylaştırmak için başlatma zaman aşımı ve bağımsız değişken uyumsuzluğu gibi kategorileri içerir.

### En İyi Uygulamalar

Pratik deneyimlerimize dayanarak aşağıda birkaç en iyi uygulamayı bulabilirsiniz:

**1. Araç izin verilenler listesi kullanın**

```csharp
var request = new AIRequest
{
    Prompt = "Analyze this file",
    AllowedTools = new[] { "Read", "Grep", "Bash(git:*)" }
};
```

İzin verilen araçların bir izin verilenler listesi aracılığıyla açıkça belirtilmesi, beklenmeyen yapay zeka eylemlerinin önlenmesine yardımcı olur. Bu özellikle yazma izinlerine sahip araçlar için önemlidir. `Edit`, ekstra dikkatle ele alınmalıdır.

**2. Makul zaman aşımlarını ayarlayın**

```csharp
options.Timeout = 3600;  // 1 hour
options.StartupTimeout = 60;  // 1 minute
```

Görevin karmaşıklığına göre uygun zaman aşımı değerlerini ayarlayın. Değer çok kısaysa görevler kesintiye uğrayabilir. Çok uzunsa, yanıt vermeyen istekleri bekleyerek kaynaklar israf edilebilir.

**3. Oturumun yeniden kullanımını etkinleştir**

```csharp
options.SessionId = "my-session-123";
```

Aynısını kullanmak `sessionId` ilgili görevler için önceki oturum bağlamını yeniden kullanmanıza ve yanıt hızını artırmanıza olanak tanır.

**4. Akış yanıtlarını yönetin**

```csharp
await foreach (var chunk in provider.StreamAsync(request))
{
    switch (chunk.Type)
    {
        case StreamingChunkType.ThinkingDelta:
            // Handle the reasoning process
            break;
        case StreamingChunkType.ToolCallDelta:
            // Handle tool invocation
            break;
        case StreamingChunkType.ContentDelta:
            // Handle text output
            break;
    }
}
```

Akış yanıtları, yapay zeka işleme sürecini gerçek zamanlı olarak göstermenize olanak tanır ve bu da kullanıcı deneyimini geliştirir. Bu özellikle uzun süren görevler için değerlidir.

**5. Hata işleme ve yeniden denemeler**

```csharp
try
{
    await foreach (var chunk in provider.StreamAsync(request))
    {
        // Handle the response
    }
}
catch (CopilotSessionException ex)
{
    // Handle session exceptions
    logger.LogError(ex, "Copilot session failed");
    // Decide whether to retry based on the exception type
}
```

Doğru hata işleme ve yeniden deneme mantığı, genel sistem kararlılığını artırır.
