## Tarayıcıda Hızlı Kod Düzenleme: Uygulamada VSCode Web Entegrasyonu

> Yapay zeka kodu analiz etmeyi bitirdikten sonra tarayıcıda hemen bir düzenleyiciyi nasıl açıp değişiklik yapmaya başlarsınız? Bu makale, AI asistanı ile kod düzenleme deneyimi arasında kusursuz bir köprü oluşturmak için kod sunucusunu HagiCode projesine entegre etme konusundaki pratik deneyimimizi paylaşıyor.

## Arka plan

Yapay zeka destekli programlama çağında, geliştiricilerin genellikle kodu hızlı bir şekilde incelemesi ve düzenlemesi gerekiyor. Geleneksel iş akışı basittir: projeyi bir masaüstü IDE'de açın, dosyayı bulun, düzenleyin ve kaydedin. Ancak bazı durumlarda bu akış her zaman biraz bozuk geliyor.

Birinci senaryo: uzaktan geliştirme. HagiCode gibi bir yapay zeka asistanı kullanıldığında, arka uç uzak bir sunucuda veya bir konteynerin içinde çalışıyor olabilir ve yerel makineler proje dosyalarına doğrudan erişemez. Kodu incelemeniz veya değiştirmeniz gerektiğinde, SSH veya başka bir yöntemle bağlanmanız gerekir ve deneyim parçalanmış gibi gelir. Bu, birisiyle kalın bir camın ardından tanışmak istemeye benziyor: Onu görebilirsin ama ona ulaşamazsın.

İkinci senaryo: hızlı önizlemeler. Yapay zeka asistanı kodu analiz ettikten sonra kullanıcı yalnızca bir dosyaya hızlı bir şekilde göz atmak veya küçük bir değişiklik yapmak isteyebilir. Tam masaüstü IDE'yi başlatmak ağır hissettirirken, hafif bir tarayıcı içi düzenleyici "hızlı görünüm" ihtiyacını daha iyi karşılar. Sonuçta kim sadece bir göz atmak için tüm takım zincirini harekete geçirmek ister?

Üçüncü senaryo: cihazlar arası işbirliği. Farklı cihazlar arasında çalışırken, tarayıcı tabanlı bir düzenleyici, her makinenin bir geliştirme ortamıyla yapılandırılmasına gerek kalmadan birleşik bir erişim noktası sağlar. Bu tek başına birçok beladan kurtarır. Hayat kısa; neden aynı kurulum çalışmasını defalarca tekrarlayasınız ki?

Bu sıkıntılı noktaları çözmek için VSCode Web'i HagiCode projesine entegre ettik. Bu, AI asistanının ve kod düzenleme deneyiminin sorunsuz bir şekilde bağlantı kurmasını sağlar: AI kodu analiz ettikten sonra kullanıcılar, uygulamaları değiştirmeden hemen bir düzenleyici açabilir ve aynı tarayıcı oturumunda değişiklik yapabilir. Bu, ihtiyacınız olduğunda orada olduğu türden bir deneyimdir.

## Teknik Seçim: Neden kod sunucusu?

Mevcut birçok VSCode Web çözümü arasından kod sunucusunu seçtik. Bu kararın arkasında birkaç somut neden vardı.

Özellik bütünlüğü. code-server, VSCode'un web sürümüdür ve uzantı sistemi, akıllı öneriler, hata ayıklama ve daha fazlası dahil olmak üzere çoğu masaüstü özelliğini destekler. Bu, kullanıcıların tarayıcıda masaüstü sürümüne çok yakın bir düzenleme deneyimi yaşayabileceği anlamına gelir. Sonuçta kim gerçekten işlevsellikten ödün vermek ister ki?

Esnek dağıtım. kod sunucusu bağımsız bir hizmet olarak çalışabilir ve aynı zamanda HagiCode'un mimarisine iyi uyum sağlayan Docker tabanlı dağıtımı da destekler. Arka ucumuz C# ile yazılmıştır, ön uç React'ı kullanır ve ikisi kod sunucusu hizmetiyle REST API'leri aracılığıyla iletişim kurar. Bu, bloklarla inşa etmeye benzer; her parçanın bir yeri vardır.

Güvenli kimlik doğrulama. kod sunucusu yerleşik bir özellik içerir `connection-token` Yetkisiz erişimi engelleyen mekanizma. Her oturumun benzersiz bir belirteci vardır, böylece yalnızca yetkili kullanıcılar düzenleyiciyi açabilir. Güvenlik, ancak sahip olduğunuzda tam anlamıyla takdir edebileceğiniz şeylerden biridir.

## Mimari Tasarım

HagiCode'un VSCode Web entegrasyonu, ön uç/arka uçtan ayrılmış bir mimari kullanır.

### Ön Uç Hizmet Katmanı

Ön uç, arka uçla olan etkileşimleri şu şekilde tamamlar: `vscodeServerService.ts`:

```typescript
// Open project
export async function openProjectInCodeServer(
  id: string,
  currentInterfaceLanguage?: string,
): Promise<VsCodeServerLaunchResponseDto>

// Open vault
export async function openVaultInCodeServer(
  id: string,
  path?: string,
  currentInterfaceLanguage?: string,
): Promise<VsCodeServerLaunchResponseDto>
```

Bu iki yöntem arasındaki fark basittir: `openProjectInCodeServer` tüm projeyi açarken `openVaultInCodeServer` Kasanın içinde belirli bir yol açar. MonoSpecs çoklu depo projeleri için sistem otomatik olarak bir çalışma alanı dosyası oluşturur. Her bir parça kendi işini iyi yaptığında net sorumluluklar genellikle yeterlidir.

### Arka Uç Hizmet Katmanı

Arka uç `VaultAppService.cs` temel mantığı uygular:

```csharp
public async Task<VsCodeServerLaunchResponseDto> OpenInCodeServerAsync(
    string id,
    string? relativePath = null,
    string? currentInterfaceLanguage = null,
    CancellationToken cancellationToken = default)
{
    // 1. Get settings and check whether the feature is enabled
    var settings = await _vsCodeServerSettingsService.GetResolvedSettingsAsync(cancellationToken);
    if (!settings.Enabled) {
        throw new BusinessException(VsCodeServerErrorCodes.Disabled, "VSCode Server is disabled.");
    }

    // 2. Get vault and resolve the launch directory
    var vault = await RequireVaultAsync(id, cancellationToken);
    var launchDirectory = ResolveLaunchDirectory(vault, relativePath);

    // 3. Ensure code-server is running and get runtime info
    var runtime = await _vsCodeServerManager.EnsureStartedAsync(settings, cancellationToken);

    // 4. Resolve language settings
    var language = _vsCodeServerSettingsService.ResolveLaunchLanguage(
        settings.Language,
        currentInterfaceLanguage);

    // 5. Build launch URL
    return new VsCodeServerLaunchResponseDto {
        LaunchUrl = AppendQueryString(runtime.BaseUrl, new Dictionary<string, string?> {
            ["folder"] = launchDirectory,
            ["tkn"] = runtime.ConnectionToken,
            ["vscode-lang"] = language,
        }),
        ConnectionToken = runtime.ConnectionToken,
        OpenMode = "folder",
        Runtime = VsCodeServerSettingsService.MapRuntime(
            await _vsCodeServerManager.GetRuntimeSnapshotAsync(cancellationToken)),
    };
}
```

Bu yöntemin çok açık bir sorumluluğu vardır: ayarları kontrol edin, yolları çözümleyin, hizmeti başlatın ve URL'yi oluşturun. Bunlar arasında, `ResolveLaunchDirectory` yöntem, yol geçiş saldırılarını önlemek için yol güvenliği kontrolleri gerçekleştirir. Her satırın bir amacı olduğunda kod biraz şiir gibi gelebilir.

### Otomatik Çalışma Zamanı Yönetimi

Arka uç kod sunucusu sürecini yönetir. `VsCodeServerManager`:

- İşlem durumunu kontrol edin
- Durdurulan hizmetleri otomatik olarak başlat
- Bağlantı noktası, işlem kimliği ve başlangıç zamanı gibi çalışma zamanı anlık görüntülerini döndürün

Bu tasarım, sistemin kod sunucusu yaşam döngüsünü otomatik olarak yönetmesine olanak tanır, böylece kullanıcıların hizmet süreçlerini manuel olarak yönetmesine gerek kalmaz. Hayat zaten yeterince karmaşık; Otomatikleştirilebilecek her şey olmalıdır.

### Dil Senkronizasyonu

HagiCode çok dilli bir arayüzü destekler ve kod sunucusunun bu ayarı izlemesi gerekir. Sistem üç dil modunu destekler:

- `follow`: mevcut arayüz dilini takip edin
- `zh-CN`: Çinceye sabitlendi
- `en-US`: İngilizceye sabitlendi

Ayar, kod sunucusuna kod sunucusu aracılığıyla iletilir. `vscode-lang` Düzenleyici dilinin HagiCode arayüzüyle tutarlı kalması için URL parametresi. Dil birleştiğinde en iyi hissi verir.

## Ön Uç Entegrasyonu

HagiCode ön ucu React + TypeScript'i kullanır ve kod sunucusunun entegrasyonu özellikle karmaşık değildir.

### Hızlı Eylem Düğmesi

Proje kartına bir Kod Sunucusu düğmesi ekleyin:

```tsx
// QuickActionsZone.tsx
<Button
  size="sm"
  variant="default"
  onClick={() => onAction({ type: 'open-code-server' })}
>
  <Globe className="h-3 w-3 mr-1" />
  <span className="text-xs">{t('project.openCodeServer')}</span>
</Button>
```

Bu düğme açma eylemini tetikler ve başlatma URL'sini almak için arka uç API'sini çağırır. Tek düğme, tek eylem, doğrudan ve basit.

### Açık Eylemi Yönetme

```tsx
const handleAction = async (action: ProjectAction) => {
  if (action.type === 'open-code-server') {
    const response = await openProjectInCodeServer(project.id, i18n.language);
    window.open(response.launchUrl, '_blank', 'noopener,noreferrer');
  }
};
```

Kullanım `window.open` kod sunucusunu yeni bir sekmede açmak için. `noopener,noreferrer` parametreler ekstra güvenlik sağlar. Konu güvenlik olunca çok dikkatli olmak diye bir şey yok.

### Kasa Düzenleme Girişi

Apps Kasası listesine benzer bir düzenleme düğmesi ekleyin:

```tsx
const handleEditVault = async (vault: VaultItemDto) => {
  const response = await openVaultInCodeServer(vault.id);
  window.open(response.launchUrl, '_blank', 'noopener,noreferrer');
};
```

Projeler ve Kasalar, etkileşimi tutarlı tutan aynı açık mekanizmayı kullanır. Tutarlılık neredeyse özelliğin kendisi kadar önemlidir.

## URL Oluşturma Mantığı

Kod sunucusunun URL biçiminde dikkate değer birkaç ayrıntı vardır.

**Klasör modu**:
```
http://{host}:{port}/?folder={path}&tkn={token}&vscode-lang={lang}
```

**Çalışma alanı modu**:
```
http://{host}:{port}/?workspace={workspacePath}&tkn={token}&vscode-lang={lang}
```

Burada, `tkn` bağlantı belirtecidir. Kod sunucusu her başlatıldığında otomatik olarak oluşturulur ve güvenli erişim sağlanır. `vscode-lang` parametresi düzenleyicinin kullanıcı arayüzü dilini kontrol eder. Bu parametrelerin her birinin oynayacağı bir rol vardır.

## Güvenlik Hususları

Kod sunucusu entegrasyonu uygulanırken güvenlik özel ilgiyi hak eder. Güvenlik ters giderse her zaman çok geç fark edersiniz.

### Bağlantı Jetonu

 `connection-token` rastgele oluşturulur ve açığa çıkarılmamalıdır. Tokenın ortadaki adam tarafından ele geçirilmesini önlemek için HTTPS altında kullanılması en iyisidir. Hassas bilgiler uygun şekilde korunmaya değerdir.

### Dosya Yolu Güvenliği

Arka uç yol geçiş kontrollerini uygular:

```csharp
private static string ResolveLaunchDirectory(VaultRegistryEntry vault, string? relativePath)
{
    var vaultRoot = EnsureTrailingSeparator(Path.GetFullPath(vault.PhysicalPath));
    var combinedPath = Path.GetFullPath(Path.Combine(vaultRoot, relativePath ?? "."));
    if (!combinedPath.StartsWith(vaultRoot, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode, "Relative path traversal detected.");
    }
    return combinedPath;
}
```

Bu kod kullanıcıların kullanamamasını sağlar `../` veya benzer kalıpları kullanarak Vault dizini dışındaki dosyalara erişebilirsiniz. Sınır kontrollerinin yapılması, atlanmaktan her zaman daha iyidir.

### İzin Kontrolü

Kod sunucusu işleminin hassas sistem dosyalarına erişememesi için uygun kullanıcı izinleriyle çalışması gerekir. Kod sunucusu hizmetini özel bir kullanıcı altında çalıştırmak en iyisidir. İzin kontrolü her zaman yerinde tutmanız gereken temel unsurlardan biridir.

## Performans Optimizasyonu

code-server sunucu kaynaklarını tükettiğinden burada birkaç optimizasyon önerisi bulacaksınız:

- CPU ve bellek kullanımını izleyin ve gerektiğinde kaynak sınırlarını ayarlayın
- Büyük projeler daha uzun zaman aşımları gerektirebilir
- Kaynakları serbest bırakmak için otomatik oturum zaman aşımı temizliğini uygulayın
- Tekrarlanan hesaplamayı azaltmak için önbelleğe almayı düşünün

HagiCode bir çalışma zamanı durumu izleme API'si sağlar ve ön uç çağrı yapabilir `getVsCodeServerSettings()` mevcut durumu almak için:

```typescript
const { settings, runtime } = await getVsCodeServerSettings();
// runtime.status: 'disabled' | 'stopped' | 'starting' | 'running' | 'unhealthy'
// runtime.baseUrl: "http://localhost:8080"
// runtime.processId: 12345
```

Bu tasarım, kullanıcıların kod sunucusunun sağlık durumunu net bir şekilde anlamalarına ve bir şeyler ters gittiğinde sorunları hızlı bir şekilde tespit etmelerine olanak tanır. Durum görünür olduğunda insanlar kendilerini daha fazla kontrol altında hissederler.

## Yapılandırma Örneği

