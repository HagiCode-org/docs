## Codex SDK Konsol Mesajı Ayrıştırmayla İlgili Eksiksiz Kılavuz

> Bu makalede Codex SDK olay akışı mekanizması, mesaj türü ayrıştırma ve gerçek projelerdeki en iyi uygulamalar anlatılarak geliştiricilerin yapay zeka yürütme hizmetlerinin ardındaki temel becerilerde hızlı bir şekilde uzmanlaşmasına yardımcı olunmaktadır.

## Arka plan

Codex SDK'yı temel alan bir yapay zeka yürütme hizmeti oluştururken kaçınılmaz olarak pratik bir soruyla karşılaşıyoruz: Codex tarafından döndürülen akışlı olay mesajlarını nasıl ele almalıyız? Bu mesajlar yürütme durumu, çıktı içeriği ve hata ayrıntıları gibi önemli bilgiler içerdiğinden dikkatli bir şekilde ele alınmayı hak ederler.

HagiCode projesinin bir parçası olarak yapay zeka kodlama asistanı senaryoları için güvenilir bir uygulayıcıya ihtiyacımız vardı. Codex SDK olay akışı mekanizmasını derinlemesine incelemeye karar vermemizin nedeni tam olarak budur. Sonuçta, yalnızca temeldeki mesajların nasıl çalıştığını anlayarak gerçek anlamda kurumsal düzeyde bir yapay zeka yürütme platformu oluşturabiliriz.

Codex SDK, OpenAI tarafından yayımlanan bir programlama yardımı SDK'sıdır. Bir Olay Akışı aracılığıyla yürütme sonuçlarını döndürür. Geleneksel istek-yanıt modelinin aksine Codex, aşağıdakileri yapabilmek için akışlı olayları kullanır:

- Yürütme ilerlemesini gerçek zamanlı olarak alın
- Hataları hemen ele alın
- Ayrıntılı belirteç kullanım istatistiklerini edinin
- Uzun süren karmaşık görevleri destekleyin

Bu olay türlerini anlamak ve bunları doğru şekilde ayrıştırmak, tam donanımlı bir yapay zeka yürütücüsü uygulamak için çok önemlidir. Sonuçta kimse kara kutuyla çalışmak istemez.

## Olay Akış Mekanizması

### Temel Kavramlar

Codex SDK'sı şunları kullanır: `thread.runStreamed()` eşzamansız bir olay yineleyicisini döndürme yöntemi:

```typescript
import { Codex } from '@openai/codex-sdk';

const client = new Codex({
  apiKey: process.env.CODEX_API_KEY,
  baseUrl: process.env.CODEX_BASE_URL,
});

const thread = client.startThread({
  workingDirectory: '/path/to/project',
  skipGitRepoCheck: false,
});

const { events } = await thread.runStreamed('your prompt here', {
  outputSchema: {
    type: 'object',
    properties: {
      output: { type: 'string' },
      status: { type: 'string', enum: ['ok', 'action_required'] },
    },
    required: ['output', 'status'],
  },
});

for await (const event of events) {
  // Handle each event
}
```

### Etkinlik Türlerinin Ayrıntılı Açıklaması

| Etkinlik Türü | Açıklama | Anahtar Veriler |
|---------|------|----------|
| `thread.started` | Konu başarıyla başlatıldı | `thread_id` |
| `item.updated` | Mesaj içeriği güncellendi | `item.text` |
| `item.completed` | Mesaj tamamlandı | `item.text` |
| `turn.completed` | Yürütme tamamlandı | `usage` (jeton kullanımı) |
| `turn.failed` | Yürütme başarısız oldu | `error.message` |
| `error` | Hata olayı | `message` |

Gerçek projelerde HagiCode'un yürütücü bileşeni bu olay türlerinin üzerine inşa edilmiştir. Sorunsuz bir kullanıcı deneyimi sağlamak için her türlü olayı dikkatli bir şekilde ele almamız gerekiyor. İyi sistemler ayrıntıların ciddiye alınmasıyla oluşturulur.

## Mesaj Ayrıştırma Uygulaması

### Mesaj İçeriğini Çıkarma

Mesaj içeriği bir olay işleyici aracılığıyla çıkarılır:

```typescript
private handleThreadEvent(event: ThreadEvent, onMessage: (content: string) => void): void {
  // Only handle message update and completion events
  if (event.type !== 'item.updated' && event.type !== 'item.completed') {
    return;
  }

  // Only handle agent message content
  if (event.item.type !== 'agent_message') {
    return;
  }

  // Extract text content
  onMessage(event.item.text);
}
```

Anahtar noktalar:
- Yalnızca tutamak `item.updated` ve `item.completed` olaylar
- Yalnızca türün içeriğini işleyin `agent_message`
- Mesaj içeriği şuradadır `event.item.text` alan

### Yapılandırılmış Çıktı Ayrıştırma

Codex, JSON yapılandırılmış çıktısını destekler. Dönüş formatını şu adresten belirleyebilirsiniz: `outputSchema` parametre:

```typescript
const DEFAULT_OUTPUT_SCHEMA = {
  type: 'object',
  properties: {
    output: { type: 'string' },
    status: { type: 'string', enum: ['ok', 'action_required'] },
  },
  required: ['output', 'status'],
  additionalProperties: false,
} as const;
```

Ayrıştırma işlevi JSON'u ayrıştırmaya çalışır ve bu başarısız olursa ham metne geri döner.

```typescript
function toStructuredOutput(raw: string): StructuredOutput {
  try {
    const parsed = JSON.parse(raw) as Partial<StructuredOutput>;
    if (typeof parsed.output === 'string') {
      return {
        output: parsed.output,
        status: parsed.status === 'action_required' ? 'action_required' : 'ok',
      };
    }
  } catch {
    // JSON parsing failed, fall back to the raw text
  }

  return {
    output: raw,
    status: 'ok',
  };
}
```

### Olay İşleme Akışını Tamamlayın

```typescript
private async runWithStreaming(
  thread: Thread,
  input: CodexStageExecutionInput
): Promise<{ output: string; usage: Usage | null }> {
  const abortController = new AbortController();
  const timeoutHandle = setTimeout(() => {
    abortController.abort();
  }, Math.max(1000, input.timeoutMs));

  let latestMessage = '';
  let usage: Usage | null = null;
  let emittedLength = 0;

  try {
    const { events } = await thread.runStreamed(input.prompt, {
      outputSchema: DEFAULT_OUTPUT_SCHEMA,
      signal: abortController.signal,
    });

    for await (const event of events) {
      // Handle message content
      this.handleThreadEvent(event, (nextContent) => {
        const delta = nextContent.slice(emittedLength);
        if (delta.length > 0) {
          emittedLength = nextContent.length;
          input.callbacks?.onChunk?.(delta);  // Streaming callback
        }
        latestMessage = nextContent;
      });

      // Process different data based on the event type
      if (event.type === 'thread.started') {
        this.threadId = event.thread_id;
      } else if (event.type === 'turn.completed') {
        usage = event.usage;
      } else if (event.type === 'turn.failed') {
        throw new CodexExecutorError('gateway_unavailable', event.error.message, true);
      } else if (event.type === 'error') {
        throw new CodexExecutorError('gateway_unavailable', event.message, true);
      }
    }
  } catch (error) {
    if (abortController.signal.aborted) {
      throw new CodexExecutorError(
        'upstream_timeout',
        `Codex stage timed out after ${input.timeoutMs}ms`,
        true
      );
    }
    throw error;
  } finally {
    clearTimeout(timeoutHandle);
  }

  const structured = toStructuredOutput(latestMessage);
  return { output: structured.output, usage };
}
```

## Hata İşleme Stratejisi

### Hata Kodu Eşlemesi

Üst katmanların bunları daha kolay işleyebilmesi için belirli hata modellerini somut hata kodlarıyla eşleştirin:

```typescript
function mapError(error: unknown): CodexExecutorError {
  if (error instanceof CodexExecutorError) {
    return error;
  }

  const message = error instanceof Error ? error.message : String(error);
  const normalized = message.toLowerCase();

  // Authentication errors - not retryable
  if (normalized.includes('401') ||
      normalized.includes('403') ||
      normalized.includes('api key') ||
      normalized.includes('auth')) {
    return new CodexExecutorError('auth_invalid', message, false);
  }

  // Rate limit errors - retryable
  if (normalized.includes('429') || normalized.includes('rate limit')) {
    return new CodexExecutorError('rate_limited', message, true);
  }

  // Timeout errors - retryable
  if (normalized.includes('timeout') || normalized.includes('aborted')) {
    return new CodexExecutorError('upstream_timeout', message, true);
  }

  // Default error
  return new CodexExecutorError('gateway_unavailable', message, true);
}
```

### Hata Türü Tanımları

```typescript
export type CodexErrorCode =
  | 'auth_invalid'      // Authentication failure
  | 'upstream_timeout'  // Upstream timeout
  | 'rate_limited'      // Rate limited
  | 'gateway_unavailable'; // Gateway unavailable

export class CodexExecutorError extends Error {
  readonly code: CodexErrorCode;
  readonly retryable: boolean;

  constructor(code: CodexErrorCode, message: string, retryable: boolean) {
    super(message);
    this.name = 'CodexExecutorError';
    this.code = code;
    this.retryable = retryable;
  }
}
```

## Çalışma Dizini ve Ortam Yapılandırması

### Çalışma Dizini Doğrulaması

Codex SDK, çalışma dizininin geçerli bir Git deposu olmasını gerektirir.

```typescript
export function validateWorkingDirectory(
  workingDirectory: string,
  skipGitRepoCheck: boolean
): void {
  const resolvedWorkingDirectory = path.resolve(workingDirectory);

  if (!existsSync(resolvedWorkingDirectory)) {
    throw new CodexExecutorError(
      'gateway_unavailable',
      'Working directory does not exist.',
      false
    );
  }

  if (!statSync(resolvedWorkingDirectory).isDirectory()) {
    throw new CodexExecutorError(
      'gateway_unavailable',
      'Working directory is not a directory.',
      false
    );
  }

  if (skipGitRepoCheck) {
    return;
  }

  const gitDir = path.join(resolvedWorkingDirectory, '.git');
  if (!existsSync(gitDir)) {
    throw new CodexExecutorError(
      'gateway_unavailable',
      'Working directory is not a git repository.',
      false
    );
  }
}
```

### Ortam Değişkenlerini Yükleme

AI Aracısının sistem komutlarına erişebilmesi için Codex SDK'nın oturum açma kabuğundan ortam değişkenlerini yüklemesi gerekir:

```typescript
function parseEnvironmentOutput(output: Buffer): Record<string, string> {
  const parsed: Record<string, string> = {};

  for (const entry of output.toString('utf8').split('\0')) {
    if (!entry) continue;

    const separatorIndex = entry.indexOf('=');
    if (separatorIndex <= 0) continue;

    const key = entry.slice(0, separatorIndex);
    const value = entry.slice(separatorIndex + 1);
    if (key.length > 0) {
      parsed[key] = value;
    }
  }

  return parsed;
}

function tryLoadEnvironmentFromShell(shellPath: string): Record<string, string> | null {
  const result = spawnSync(shellPath, ['-ilc', 'env -0'], {
    env: process.env,
    stdio: ['ignore', 'pipe', 'pipe'],
    timeout: 5000,
  });

  if (result.error || result.status !== 0) {
    return null;
  }

  return parseEnvironmentOutput(result.stdout);
}

export function createExecutorEnvironment(
  envOverrides: Record<string, string> = {}
): Record<string, string> {
  // Load environment variables from the login shell
  const consoleEnv = loadConsoleEnvironmentFromShell();

  return {
    ...process.env,
    ...consoleEnv,
    ...envOverrides,
  };
}
```

## Tam Kullanım Örneği

### Temel Kullanım

HagiCode projesinde Codex istemcisini başlatmak ve görevleri yürütmek için aşağıdaki yaklaşımı kullanıyoruz:

```typescript
import { Codex } from '@openai/codex-sdk';

async function executeWithCodex(prompt: string, workingDir: string) {
  const client = new Codex({
    apiKey: process.env.CODEX_API_KEY,
    env: { PATH: process.env.PATH },
  });

  const thread = client.startThread({
    workingDirectory: workingDir,
  });

  const { events } = await thread.runStreamed(prompt);

  let result = '';
  for await (const event of events) {
    if (event.type === 'item.updated' && event.item.type === 'agent_message') {
      result = event.item.text;
    }
    if (event.type === 'turn.completed') {
      console.log('Token usage:', event.usage);
    }
  }

  // Try to parse JSON output
  try {
    const parsed = JSON.parse(result);
    return parsed.output;
  } catch {
    return result;
  }
}
```

### Yeniden Denemelerle Uygulamayı Tamamlayın

```typescript
export class CodexSdkExecutor {
  private readonly config: CodexRuntimeConfig;
  private readonly client: Codex;
  private threadId: string | null = null;

  async executeStage(input: CodexStageExecutionInput): Promise<CodexStageExecutionResult> {
    const maxAttempts = Math.max(1, this.config.retryCount + 1);
    let attempt = 0;
    let lastError: CodexExecutorError | null = null;

    while (attempt < maxAttempts) {
      attempt += 1;

      try {
        const thread = this.getThread(input.workingDirectory);
        const { output, usage } = await this.runWithStreaming(thread, input);

        return {
          output,
          usage,
          threadId: this.threadId!,
          attempts: attempt,
          latencyMs: Date.now() - startedAt,
        };
      } catch (error) {
        const mappedError = mapError(error);
        lastError = mappedError;

        // Non-retryable error or max retry attempts reached
        if (!mappedError.retryable || attempt >= maxAttempts) {
          throw mappedError;
        }

        // Wait before retrying
        await new Promise(resolve => setTimeout(resolve, 1000 * attempt));
      }
    }

    throw lastError!;
  }
}
```

## En İyi Uygulamalar

### 1. Çalışma Dizini Gereksinimleri

- Çalışma dizininin geçerli bir Git deposu olduğundan emin olun
- Şunu kullanın: `PROJECT_ROOT` açıkça belirtmek için ortam değişkeni
- Hata ayıklama sırasında şunları ayarlayabilirsiniz: `CODEX_SKIP_GIT_REPO_CHECK=true` kontrolü atlamak

### 2. Ortam Değişkeni Yapılandırması

- Beyaz liste mekanizması aracılığıyla yalnızca gerekli ortam değişkenlerini iletin
- Tüm ortamı yüklemek için oturum açma kabuğunu kullanın
- Hassas bilgileri iletmekten kaçının

### 3. Zaman Aşımı ve Yeniden Deneme Stratejisi

- Görevin karmaşıklığına göre makul zaman aşımları ayarlayın
- Yeniden denenebilir hatalar için üstel geri alma uygulayın
- Yeniden deneme sayılarını ve nedenlerini kaydedin

### 4. Hata İşleme

- Yeniden denenebilen ve yeniden denenemeyen hatalar arasında ayrım yapın
- Açık hata mesajları ve öneriler sağlayın
- Üst katmanların bunları tutarlı bir şekilde işleyebilmesi için birleşik hata kodları kullanın

### 5. Akış Çıkışı

- Kullanıcı deneyimini geliştirmek için artımlı çıktı geri aramaları uygulayın
- Artımlı mesaj güncellemelerini doğru şekilde işleyin
- Maliyet analizi için jeton kullanımını kaydedin
