「冒險團」頁面把英雄編組、負載調優和副本推進放在同一個工作台裡。本頁只講第一次接觸時最該看懂的內容：它負責什麼、如何協作、又該在什麼場景裡用。

## 功能概覽

- 冒險團用來把「誰來執行任務、以什麼配置執行、當前副本推進到哪裡」放到同一個視圖裡統一檢視。
- 目前這組截圖重點覆蓋兩類資訊：英雄編組與負載面板、以及副本卡片與編組編輯器之間的協作關係。
- 適合在提案執行、批次任務推進、或需要快速切換英雄職責時作為總覽工作台使用。
- 這不是戰鬥動畫或純展示頁面；它更像一個圍繞副本目標組織執行者與配置的調度台。

## 成員與編組面板

![冒險團英雄編組與負載面板](../img/screenshots/shared/screenshot-a1e8035a/original.png)

這張截圖最適合用來解釋「成員編組」本身：左側是可瀏覽的英雄 roster，中間是當前選中成員的詳細狀態，右側則是和當前成員直接相關的職業負載與戰備設定。使用者通常會在這裡完成三件事：

1. 先從 roster 中選出本輪任務的主力成員，確認哪些英雄已經準備好進入副本。
2. 再檢視當前成員的狀態詳情，例如戰備標識、角色資訊和當前承擔的職責。
3. 最後微調 profession / loadout 一類的配置，避免把不合適的英雄帶入後續流程。

放到日常團隊協作裡，它更像一個「排班 + 裝備檢查」面板：先確認誰上場，再確認每個人是否以正確配置進入任務。

## 協作流程與副本推進

![冒險團副本管理與編組流程](../img/screenshots/shared/screenshot-11d59f8f/original.png)

這張截圖把「副本目標」與「成員配置」放在同一個工作流裡：左側的 dungeon proposal cards 表示當前可推進或待處理的副本任務，右側的 roster editor 則負責把具體執行者綁定到選中的副本上下文。

可以把它理解為下面這條協作鏈：

1. 先在左側選定本次要推進的副本或任務卡片。
2. 根據副本目標，在右側調整參與成員、狀態標識和已分配英雄。
3. 確認隊伍後，再讓 Proposal、AutoTask 或 Prompt 相關流程繼續推進。

因此，「冒險團」的價值不是單獨展示英雄，而是把任務上下文、成員編組和執行準備串成一條連續的推進路徑。

## 功能邊界與適用場景

### 它解決什麼問題

- 在一個頁面裡同時看到副本上下文和執行 roster，減少來回切換視圖的成本。
- 在進入後續自動執行前，先完成成員檢查與配置校準。
- 為團隊提供一個更容易復盤的「當前任務由誰負責、為什麼這樣分配」的可視化入口。
