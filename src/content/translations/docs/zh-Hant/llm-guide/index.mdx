## 為什麼選擇 LLM 指南

在程式碼生成工作流程中，模型效能在品質、延遲和成本方面存在顯著差異。此類別提供一致的評估結構，幫助開發者更快選擇模型。

## 推薦閱讀路徑

1. 開始於 [HagiCode 中的模型比較和評估](./model-comparison-evaluation/).
2. 根據您的優先順序（品質第一、成本第一或速度第一）選擇候選人。
3. 在將結論應用於生產之前，請檢查測試時間記錄。

## 維護原則

- 當模型能力發生重大變化時刷新結論。
- 每月至少重新驗證一次測試時間上下文。
- 保持中英文頁面的章節架構一致。
