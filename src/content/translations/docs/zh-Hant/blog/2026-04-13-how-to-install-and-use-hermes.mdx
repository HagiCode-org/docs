想安裝並開始使用 Hermes，最短路徑其實只有三步：

1. 執行官方安裝命令
2. 在終端機裏用 `hermes` 啓動 CLI
3. 如果你想在飛書裏繼續用，再設定 `hermes gateway setup`

這篇文章不打算把 Hermes 的所有能力一次講完，而是先幫你完成最關鍵的入門閉環：裝上、跑起來、開始用，然後再接一個最常見的訊息平台場景。

## Hermes 是什麼，適合誰用

Hermes Agent 是一個既可以在本地終端機使用，也可以透過訊息平台網關使用的 AI agent。

對大多數開發人員來說，它有兩個最常見的入口：

- **CLI**：在終端機裏輸入 `hermes`，直接進入互動式界面
- **Messaging Gateway**：執行 `hermes gateway`，再從 Feishu、Telegram、Discord、Slack 等平台和它對話

如果你現在的目標只是快速上手，建議順序不要反過來，先走這條路線：

- 先安裝 Hermes
- 先從 CLI 驗證是否可用
- 再決定要不要接訊息平台

這樣更容易定位問題，也更適合第一次接觸 Hermes 的使用者。

## 安裝 Hermes 之前要知道什麼

根據 Hermes README，官方快速安裝路徑支援這些環境：

- Linux
- macOS
- WSL2
- Android via Termux

### Windows 使用者注意事項

Hermes 當前不支援原生 Windows 直接執行。如果你使用 Windows，推薦先安裝 WSL2，再在 WSL2 裏執行安裝命令。

這點最好在文章開頭就說清楚，因爲很多安裝失敗其實不是命令問題，而是執行環境不符合要求。

## 如何安裝 Hermes

Hermes README 給出的快速安裝命令是：

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

這條命令會執行官方安裝腳本，處理平台相關的初始化流程。

### 安裝完成後重新整理 shell

安裝結束後，先重新載入 shell 環境。最常見的是：

```bash
source ~/.bashrc
```

如果你用的是 zsh，也可以改成：

```bash
source ~/.zshrc
```

### 如何確認 Hermes 已經裝好

最直接的檢查方式就是執行：

```bash
hermes
```

如果你想進一步確認設定和相依是否正常，可以再執行：

```bash
hermes doctor
```

`hermes doctor` 適合在這些時候使用：

- 安裝後命令列爲不正常
- 模型設定失敗
- gateway 啓動失敗
- 不確定環境相依是否完整

## 第一次如何開始使用 Hermes

### 先從 CLI 開始

如果你只是想盡快確認 Hermes 能不能用，最簡單的方法就是：

```bash
hermes
```

這會啓動 Hermes 的互動式 CLI。對於第一次接觸 Hermes 的使用者，這也是最推薦的起點，因爲你可以先驗證最核心的幾件事：

- 命令是否真的可用
- 當前模型設定是否正常
- 終端機工具鏈是否工作正常
- 互動體驗是不是你需要的那種方式

### 這幾個命令足夠你完成第一輪設定

Hermes README 裏列出的幾條高頻命令，基本就構成了第一輪上手路徑：

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

如果你不知道它們各自是做什麼的，可以先這樣記：

- `hermes model`：選模型、切模型
- `hermes tools`：看和配當前可用工具
- `hermes config set`：改具體設定項
- `hermes setup`：跑一次完整初始化嚮導
- `hermes update`：更新 Hermes
- `hermes doctor`：做故障排查

對新手最實用的順序通常是：

1. 先執行 `hermes model`
2. 如果你希望一次把常用項配完整，再執行 `hermes setup`

## Hermes 最常見的兩種使用方式

### 1. 在終端機裏把 Hermes 當成日常開發助手

CLI 模式適合這些場景：

- 本地寫程式碼時直接問問題
- 查專案、改檔案、跑命令
- 做一次性調試或 review
- 在當前工作目錄裏持續協作

它最大的優點就是路徑最短：不用額外接平台，不用先處理機器人設定，也最適合建立第一輪使用習慣。

### 2. 透過訊息平台使用 Hermes

如果你希望在飛書、Telegram、Discord 等平台上和 Hermes 對話，就需要使用 messaging gateway。

最常見的入口命令是：

```bash
hermes gateway setup
hermes gateway
```

其中：

- `hermes gateway setup` 用來做互動式平台設定
- `hermes gateway` 用來啓動網關處理序

根據官方文件，gateway 是一個統一的後臺處理序，用來連接你已設定的平台、管理會話，並處理定時任務等功能。

## 以 Feishu 爲例，如何把 Hermes 接入訊息平台

如果你的日常工作主要在飛書裏，那麼 Feishu/Lark 會是一個很自然的 Hermes 接入方式。

### 最小可行接入方式

官方文件對 Feishu/Lark 的推薦入口是：

```bash
hermes gateway setup
```

執行後，在嚮導中選擇 **Feishu / Lark** 即可。

Feishu 文件給出的兩種連接模式是：

- `websocket`：推薦
- `webhook`：可選

如果 Hermes 跑在你的筆記本、工作站或者私有伺服器上，優先使用 `websocket` 會更簡單，因爲不需要額外暴露公網回呼地址。

### 如果你手動設定，至少要知道這些變數

如果你不是透過嚮導設定，而是手動寫設定，Feishu 文件裏列出的核心變數包括：

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

其中最值得注意的是兩項：

- `FEISHU_ALLOWED_USERS`：建議設定，避免任何能接觸到 bot 的人都可以直接使用它
- `FEISHU_HOME_CHANNEL`：可以預先指定一個 home chat，用來接收 cron 結果或預設通知

### Feishu 羣聊裏爲什麼有時不回覆

這一點很容易被忽略：在 Feishu 羣聊裏，Hermes 預設不是看到每條訊息都回應。

官方文件明確說明：

- 私聊時，Hermes 會回應訊息
- 羣聊裏，必須顯式 `@` bot，它纔會處理訊息

如果你想把某個飛書會話設成 home channel，也可以在聊天裏使用：

```text
/set-home
```

或者提前在設定裏寫：

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## 新手最值得先記住的 Hermes 命令

不管你是在 CLI 裏，還是在訊息平台裏，先記住下面這些命令就已經夠用了：

- `/new` 或 `/reset`：開始新會話
- `/model`：查看或切換模型
- `/retry`：重試上一輪
- `/undo`：撤銷上一輪互動
- `/compress`：手動壓縮上下文
- `/help`：查看說明

如果你主要在訊息平台裏使用，再額外記住一個：

- `/sethome` 或 `/set-home`：把當前聊天設爲 home channel

這些命令覆寫了新手階段最常見的操作：重開、調整、回退、查看和繼續用。

## 常見問題

### Windows 能直接安裝 Hermes 嗎？

不能。當前官方文件明確說明，原生 Windows 不支援，推薦使用 WSL2。

### 安裝之後輸入 `hermes` 沒反應怎麼辦？

建議按下面順序排查：

1. 先重新載入 shell，例如 `source ~/.bashrc`
2. 再重新執行 `hermes`
3. 如果還是例外，執行 `hermes doctor`

### Feishu 羣裏爲什麼 bot 不回覆？

先檢查這三項：

- 你有沒有在羣裏 `@` Hermes
- `FEISHU_ALLOWED_USERS` 是否限制了當前使用者
- 當前羣聊策略是否允許處理羣訊息

根據官方 Feishu 文件，羣聊場景裏，顯式 @mention 是必要條件。

## 下一步建議

如果你只是想盡快開始使用 Hermes，最推薦的順序是：

1. 先執行安裝命令
2. 先用 `hermes` 在本地 CLI 裏開始
3. 再用 `hermes model` 和 `hermes setup` 補齊基礎設定
4. 如果你希望在飛書裏繼續使用，再設定 `hermes gateway setup`
