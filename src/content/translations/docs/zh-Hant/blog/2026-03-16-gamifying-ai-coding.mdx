## 如何用遊戲化設計讓 AI 程式設計變得更好玩

> 其實傳統的 AI 程式設計工具功能挺強大的，就是少了點溫度。我們在做 HagiCode 的時候就想，既然都要寫程式碼，爲什麼不把它變成一場遊戲呢？

## 背景

用過 AI 程式設計助手的朋友應該都有這種體驗：剛開始覺得挺新鮮，用着用着就感覺少了點什麼。工具本身功能很強大，程式碼產生、自動補全、Bug 修復樣樣都能做，只是……沒什麼溫度，用久了會覺得有些單調乏味。

這也罷了，畢竟誰願意每天對着冷冰冰的工具呢。

這就好比打遊戲，如果只是單純地完成任務清單，沒有角色成長、沒有成就感解鎖定、沒有團隊配合，那很快就會覺得沒意思。美的事物或人，不一定要佔有，只是她還是美的，自己好好看着她的美就好了。可程式設計工具連這種美都沒有，難免讓人心灰意冷。

我們在開發 [HagiCode](https://hagicode.com) 的程序中就遇到了這個問題。HagiCode 作爲一個多 AI 助手協作平台，需要讓使用者長期保持使用熱情。但現實是，再好的工具，如果缺乏情感連接，使用者也很難堅持下去。

爲了解決這個痛點，我們做了一個大膽的決定：把程式設計變成一場遊戲。不是那種簡單的積分排行榜，而是真正的角色扮演式的遊戲化體驗。這個決定帶來的變化，可能比你想象的還要大。

畢竟，人嘛，總是需要點儀式感的。

## 關於 HagiCode

本文分享的方案來自我們在 [HagiCode](https://hagicode.com) 專案中的實踐經驗。HagiCode 是一個多 AI 助手協作平台，支援 Claude Code、Codex、Copilot、OpenCode 等多種 AI 助手協同工作。如果你對多 AI 協作、遊戲化程式設計感興趣，可以存取 [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site) 瞭解更多。

其實也沒什麼特別的，只是我們把程式設計變成了一場冒險而已。

## 爲什麼選擇遊戲化

遊戲化的本質不是"加個積分榜"，而是建立一套完整的激勵體系，讓使用者在做任務的程序中體驗到成長感、成就感和社交認同。

HagiCode 的遊戲化設計圍繞一個核心概念展開：每個 AI 助手都是一名"英雄"，使用者就是這支英雄團隊的隊長。你帶領這些英雄去征服各種"地牢"（程式設計任務），在這個程序中，英雄會獲得經驗、升級解鎖定能力，你和你的團隊也會獲得各種成就。

這不是什麼噱頭，而是基於人類行爲心理學的精心設計。當任務被賦予意義和進度反饋時，人的投入度和堅持程度會顯著提升。

就像古人說的，"此情可待成追憶，只是當時已惘然"。我們把這種情感體驗融入到工具中，讓程式設計不再只是敲程式碼，而是一段值得回憶的旅程。

## Hero 角色系統

Hero 是 HagiCode 遊化系統的核心概念。每個 Hero 代表一個 AI 助手，比如 Claude Code 是一個 Hero，Codex 也是一個 Hero。

### Hero 的三大槽位

Hero 有三個裝備槽位，這個設計其實還挺巧妙的：

1. **CLI 槽位（主要職業）**：決定 Hero 的基礎能力，比如是 Claude Code 還是 Codex
2. **Model 槽位（次要職業）**：決定使用的模型，比如 Claude 4.5 還是 Claude 4.6
3. **Style 槽位（風格）**：決定 Hero 的行事風格，比如是"風落策略家"還是"其他風格"

三個槽位的組合可以創造出獨特的 Hero 設定。就像遊戲裏配裝一樣，你需要根據任務特點選擇合適的搭配。畢竟適合自己的纔是最好的，這和生活選路差不多，條條大路通羅馬，只是有的路好走一點，有的路稍微曲折一點罷了。

### Hero 的成長系統

每個 Hero 都有自己的 XP（經驗值）和等級：

```typescript
type HeroProgressionSnapshot = {
  currentLevel: number;                    // 当前等级
  totalExperience: number;                 // 总经验值
  currentLevelStartExperience: number;     // 当前等级起始经验
  nextLevelExperience: number;             // 下一等级所需经验
  experienceProgressPercent: number;       // 进度百分比
  remainingExperienceToNextLevel: number;  // 距离下一级还需要多少经验
  lastExperienceGain: number;              // 最近一次获得的经验
  lastExperienceGainAtUtc?: string | null; // 获得经验的时间
};
```

等級分爲四個階段，每個階段的命名都很有代入感：

```typescript
export const resolveHeroProgressionStage = (level?: number | null): HeroProgressionStage => {
  const normalizedLevel = Math.max(1, level ?? 1);
  if (normalizedLevel <= 100) return 'rookieSprint';      // 新人冲刺
  if (normalizedLevel <= 300) return 'growthRun';         // 成长跑
  if (normalizedLevel <= 700) return 'veteranClimb';      // 老兵攀登
  return 'legendMarathon';                                // 传奇马拉松
};
```

從"新人"到"傳奇"，這個成長路徑讓使用者有明確的目標感和成就感。就像人生的成長，總要經歷從懵懂到成熟的程序，只是這裏把這種程序具象化了而已。

### 建立自定義 Hero

建立 Hero 時需要設定三個槽位：

```typescript
const heroDraft: HeroDraft = {
  name: 'Athena',
  icon: 'hero-avatar:storm-03',
  description: '智谋过人的策略家',
  executorType: AIProviderType.CLAUDE_CODE_CLI,
  slots: {
    cli: {
      id: 'profession-claude-code',
      parameters: { /* CLI 相关参数 */ }
    },
    model: {
      id: 'secondary-claude-4-sonnet',
      parameters: { /* 模型相关参数 */ }
    },
    style: {
      id: 'fengluo-strategist',
      parameters: { /* 风格相关参数 */ }
    }
  }
};
```

每個 Hero 都有獨特的頭像、描述和專業定位，這讓冰冷的 AI 助手變得有個性、有溫度。畢竟誰願意跟沒有性格的工具打交道呢？

## Dungeon 地牢系統

"地牢"是遊戲中的經典概念，代表着需要組隊攻略的挑戰。在 HagiCode 中，每個工作流程就是一個 Dungeon。

### Dungeon 的組織方式

Dungeon 將工作流程組織成不同的"地牢"：

- **提案產生地牢**：負責產生技術提案
- **提案執行地牢**：負責執行提案中的任務
- **提案歸檔地牢**：負責整理和歸檔完成的提案

每個地牢都有自己的 Captain（隊長）Hero，隊長自動從啓用的 Hero 中選擇第一個。

其實這就像生活中的分工，每個人都有自己的角色，只是這裏把這種分工變成了遊戲機制而已。

### 團隊協作機制

你可以爲不同的地牢設定不同的 Hero 小隊：

```typescript
const dungeonRoster: HeroDungeonRoster = {
  scriptKey: 'proposal.generate',
  displayName: '提案生成',
  members: [
    { heroId: 'hero-1', name: 'Athena', executorType: 'ClaudeCode' },
    { heroId: 'hero-2', name: 'Apollo', executorType: 'Codex' }
  ]
};
```

比如產生提案時用 Athena（擅長策略），執行程式碼時用 Apollo（擅長實作），這樣每個英雄都能發揮自己的專長。就像組建一支樂隊，每個人都有自己的樂器，合起來才能奏出動聽的旋律。

### Dungeon 流程控制

Dungeon 使用固定的 scriptKey 來標識不同的流程：

```typescript
// 脚本键值对应不同的工作流程
const dungeonScripts = {
  'proposal.generate': '提案生成',
  'proposal.execute': '提案执行',
  'proposal.archive': '提案归档'
};
```

任務狀態流轉是：queued（排隊中）→ dispatching（分發中）→ dispatched（已分發）。整個程序自動化，不需要手動干預。這也是我們偷懶的小心思，畢竟誰願意手動管這些事呢。

## XP 和等級系統

XP（經驗值）是遊戲化系統中最核心的反饋機制。使用者透過完成任務獲得 XP，XP 讓英雄升級，升級解鎖定新的能力，形成正向循環。

### XP 取得方式

在 HagiCode 中，XP 可以透過以下活動獲得：

- 程式碼執行完成
- 工具呼叫成功
- 提案產生
- 會話管理操作
- 專案操作

每完成一次有效操作，對應的 Hero 就會獲得 XP。就像生活中的成長，每一步都算數，只是這裏把這種成長量化了而已。

### 實時進度可視化

XP 和等級的進度是實時可視化的：

```typescript
type HeroDungeonMember = {
  heroId: string;
  name: string;
  icon?: string | null;
  executorType: PCode_Models_AIProviderType;
  currentLevel?: number;                    // 当前等级
  totalExperience?: number;                 // 总经验值
  experienceProgressPercent?: number;       // 进度百分比
};
```

使用者可以隨時看到每個 Hero 的等級和進度，這種即時反饋是遊戲化設計的關鍵。畢竟人總是需要點反饋，不然怎麼知道自己進步了呢？

## 成就係統

成就是遊戲化中的另一個重要元素，它提供了長期目標和里程碑式的成就感。

### 成就型別

HagiCode 支援多種成就型別：

- **程式碼產生類**：產生 X 行程式碼、產生 Y 個檔案
- **會話管理類**：完成 Z 次對話
- **專案操作類**：在 W 個專案中工作過

其實這些成就就像人生中的里程碑，只是我們把它們變成了一種遊戲機制而已。

### 成就狀態

成就有三種狀態：

```typescript
type AchievementStatus = 'unlocked' | 'in-progress' | 'locked';
```

三種狀態有明顯的視覺區分：

- **已解鎖定**：金色漸變 + 光暈效果
- **進行中**：藍色脈衝動畫
- **未解鎖定**：灰色，顯示解鎖定條件

每個成就都清晰展示觸發條件，讓使用者知道下一步該做什麼。畢竟迷茫的時候，有個指引總是好的。

### 解鎖定時的慶祝效果

當成就解鎖定時，會觸發慶祝動畫。這種正向強化會讓使用者產生"我做到了"的滿足感，激勵他們繼續前進。就像生活中小小的獎勵，雖然不大，卻能讓人開心很久。

## Battle Report 每日戰鬥報告

Battle Report 是 HagiCode 的一個特色功能，每天結束時產生一份全屏展示的戰鬥風格報告。

### 報告內容

Battle Report 顯示以下資訊：

```typescript
type HeroBattleReport = {
  reportDate: string;
  summary: {
    totalHeroCount: number;        // 总英雄数
    activeHeroCount: number;       // 活跃英雄数
    totalBattleScore: number;      // 总战斗分数
    mvp: HeroBattleHero;           // 最有价值英雄
  };
  heroes: HeroBattleHero[];        // 所有英雄的详细数据
};
```

- 隊伍總分數
- 活躍 Hero 數量
- 工具呼叫次數
- 總工作時長
- MVP（最有價值英雄）
- 每個 Hero 的詳細卡片

### MVP 高亮展示

MVP 是當天表現最好的 Hero，會在報告中高亮顯示。這不僅是資料統計，更是一種榮譽認可。畢竟誰不希望自己被認可呢？

### Hero 詳細卡片

每個 Hero 的卡片包含：

- 等級進度
- XP 獲得量
- 執行次數
- 使用時長

這些資料讓使用者清楚地瞭解團隊的工作狀態。畢竟瞭解自己的努力成果，也是一種滿足感。

## 技術實作

HagiCode 的遊戲化系統採用了現代化的技術棧和設計模式。其實也沒什麼特別的，只是選了一些趁手的工具而已。

### 技術棧選擇

```typescript
// 前端使用 React + TypeScript
import React from 'react';

// 动画使用 Framer Motion
import { AnimatePresence, motion } from 'framer-motion';

// 状态管理使用 Redux Toolkit
import { useAppDispatch, useAppSelector } from '@/store';

// UI 组件使用 shadcn/ui
import { Dialog, DialogContent } from '@/components/ui/dialog';
```

Framer Motion 負責所有動畫效果，shadcn/ui 提供基礎的 UI 元件，Redux Toolkit 管理複雜的遊戲化狀態。畢竟工欲善其事，必先利其器。

### 遊戲化 UI 設計系統

HagiCode 採用了 Glassmorphism + Tech Dark 的設計風格：

```css
/* 主渐变色 */
background: linear-gradient(135deg, #22C55E 0%, #25c2a0 50%, #06b6d4 100%);

/* 玻璃态效果 */
backdrop-filter: blur(12px);

/* 光辉效果 */
background: radial-gradient(circle at center, rgba(34, 197, 94, 0.15) 0%, transparent 70%);
```

綠色系的漸變配合玻璃態效果，營造出科技感和未來感。畢竟視覺上的美感，也是使用者體驗的一部分。

### 動畫效果實作

使用 Framer Motion 實作流暢的進場動畫：

```typescript
<motion.div
  animate={{ opacity: 1, y: 0 }}
  initial={{ opacity: 0, y: 18 }}
  transition={{ duration: 0.35, ease: 'easeOut', delay: index * 0.08 }}
  className="card"
>
  {/* 卡片内容 */}
</motion.div>
```

每個卡片依次進場，延遲 0.08 秒，創造出流暢的視覺效果。畢竟流暢的動畫能讓體驗更好，這是毋庸置疑的。

### 資料永續性

遊戲化資料使用 Grain 存儲系統，確保狀態一致性。即使是 Hero 的 XP 累積這種細粒度資料，也能準確永續性。畢竟誰也不想讓辛苦積累的經驗丟失。

## 實踐指南

### 建立第一個 Hero

建立第一個 Hero 其實挺簡單的：

1. 進入 Hero 管理頁面
2. 點擊"建立 Hero"按鈕
3. 設定三個槽位（CLI、Model、Style）
4. 給 Hero 起個名字和描述
5. 保存，你的第一個 Hero 就誕生了

就像認識新朋友一樣，你需要給他一個名字、瞭解他的特點，然後你們就可以一起冒險了。

### 組建 Dungeon 團隊

組建團隊也很簡單：

1. 進入 Dungeon 管理頁面
2. 選擇要設定的地牢（如"提案產生"）
3. 從你的 Hero 清單中選擇成員
4. 系統自動選擇第一個啓用的 Hero 作爲隊長
5. 保存設定

其實這就是一種組隊的程序，就像生活中組建一個團隊，每個人都有自己的角色。

### 查看每日報告

每天結束後，你可以查看當日的 Battle Report：

1. 點擊"戰鬥報告"按鈕
2. 全屏展示當天的工作成果
3. 查看 MVP 和每個 Hero 的詳細資料
4. 分享給團隊成員（可選）

這也是一種儀式感，讓自己知道今天努力了多少，離目標還有多遠。

## 注意事項和最佳實踐

### 效能優化

使用 React.memo 避免不必要的重渲染：

```typescript
const HeroCard = React.memo(({ hero }: { hero: HeroDungeonMember }) => {
  // 组件实现
});
```

畢竟效能也很重要，誰願意用卡頓的工具呢？

### 動畫可降級

檢測使用者的運動偏好設定設定，爲敏感使用者提供簡化體驗：

```typescript
const prefersReducedMotion = useReducedMotion();
const duration = prefersReducedMotion ? 0 : 0.35;
```

畢竟不是所有人都喜歡動畫，尊重使用者的偏好設定也是設計的一部分。

### 向後相容

保留 legacyIds 支援舊版本遷移：

```typescript
type HeroDungeonMember = {
  heroId: string;
  legacyIds?: string[];  // 支持旧版本 ID 映射
  // ...
};
```

畢竟誰也不希望因爲版本升級就丟失資料。

### 國際化支援

所有文字使用 i18n 翻譯鍵，方便多語言支援：

```typescript
const displayName = t(`dungeon.${scriptKey}`, { defaultValue: displayName });
```

畢竟語言不應該成爲使用的障礙。

## 總結

遊戲化不是簡單的積分排行榜，而是一套完整的激勵體系。HagiCode 透過 Hero 系統、Dungeon 系統、XP/等級系統、成就係統和 Battle Report，將程式設計工作轉化爲一場充滿冒險精神的英雄之旅。

這套系統的核心價值在於：

- **情感連接**：讓冰冷的 AI 助手變得有個性
- **正向反饋**：每次操作都有即時反饋
- **長期目標**：等級和成就提供成長路徑
- **團隊認同**：Dungeon 團隊協作感
- **榮譽認可**：Battle Report 和 MVP 展示

遊戲化設計讓程式設計不再枯燥，而是一場有趣的冒險。使用者在完成程式碼任務的同時，體驗到角色成長、團隊協作和成就解鎖定的樂趣，從而提高使用粘性和活躍度。

其實程式設計本身就是一種創造，只是我們把這種創造程序變得更有趣了一點而已。
