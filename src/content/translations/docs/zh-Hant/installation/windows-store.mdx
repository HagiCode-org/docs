對於 Windows 使用者來說，Windows Store 是目前公開的桌面入口。本頁只說明商店安裝路徑；如果你需要更完整的 Desktop 下載、版本管理與故障排除說明，請繼續閱讀 [Desktop 安裝](/zh-Hant/installation/desktop)。

Steam 主應用程式條目已經退役，因此目前 Windows 主要安裝渠道是 Windows Store。相關背景可見 [為什麼 HagiCode 沒有上架 Steam](/zh-Hant/faq/steam-distribution-status/)。

:::tip[適合這條路徑的情況]
- 你正在使用 Windows 10 或 Windows 11，並希望直接從系統商店安裝
- 你更希望由商店渠道交付應用程式，而不是手動處理安裝檔
- 你只需要個人本地安裝，不打算進行 Docker Compose 部署
:::

## 安裝前確認

開始前請先確認：

- 裝置可以正常打開 [HagiCode for Windows 的 Windows Store 頁面](https://apps.microsoft.com/detail/9N3PM0N3SVDW)
- 首次啟動時網路可用，因為程式可能還會繼續準備額外元件
- 某些裝置、地區或組織策略可能會要求你先登入 Microsoft 帳戶
- 如果你的設備禁用了商店，請改用 [Desktop 安裝](/zh-Hant/installation/desktop)

## 從 Windows Store 安裝

1. 打開 [HagiCode for Windows](https://apps.microsoft.com/detail/9N3PM0N3SVDW) 頁面。
2. 點擊 `取得`、`安裝`，或商店頁面中的等效按鈕。
3. 等待 Windows Store 完成下載與安裝。
4. 安裝完成後，從開始功能表或商店頁面直接啟動 HagiCode。

## 首次啟動時會發生什麼

安裝完成後，HagiCode 仍會進入標準 Desktop 初始化流程。

- 首次啟動可能比一般應用程式更慢，因為還要準備預設本地執行環境
- Windows Store / MSIX 渠道會鎖定部分依賴管理與執行時路徑設定，這是此分發方式的預期行為
- 軟體本體不會獨立自動更新，應用程式套件更新取決於安裝渠道；對於 Store 安裝，交付方式由 Windows Store 渠道決定

如果你要繼續完成初始化精靈，請直接閱讀 [初始化精靈設定](/zh-Hant/quick-start/wizard-setup)。

## 什麼時候改用其他安裝方式

以下情況更適合改用其他渠道：

- 你無法存取 Windows Store，或組織設備明確禁止商店安裝
- 你需要手動下載 `.exe` 或 `.msix` 檔案，並自行控制安裝來源
- 你要部署到伺服器、NAS 或團隊共享環境，此時應改看 [Docker Compose 部署](/zh-Hant/installation/docker-compose)

## 常見問題

### 無法打開 Windows Store 頁面

請先確認系統網路、地區設定和 Microsoft Store 登入狀態。如果瀏覽器能打開頁面，但商店用戶端仍不可用，通常代表目前系統策略限制了商店能力。這時請改用 [Desktop 安裝](/zh-Hant/installation/desktop)。

### 為什麼某些設定是灰色的

Store / MSIX 安裝會鎖定部分依賴管理與執行時資料路徑選項。這不是安裝異常，而是目前分發渠道的限制。

### 為什麼第一次啟動還要繼續下載或準備內容

商店安裝只負責交付桌面應用程式套件。首次啟動時，HagiCode 仍可能繼續檢查版本資訊、準備本地元件，或完成預設初始化。請保持網路可用並等待當前步驟結束。
