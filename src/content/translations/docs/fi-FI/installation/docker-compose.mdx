## Edellytykset

Ennen kuin aloitat, varmista, että järjestelmässäsi on Docker ja Docker Compose asennettuna.

### Asenna Docker

#### Windows

Lataa ja asenna [Docker Desktop Windowsille](https://www.docker.com/products/docker-desktop/)

Varmista asennuksen jälkeen, että Docker Desktop on käynnissä.

#### macOS

Lataa ja asenna [Docker Desktop for Mac](https://www.docker.com/products/docker-desktop/)

Varmista asennuksen jälkeen, että Docker Desktop on käynnissä.

#### Linux

Asenna Docker paketinhallinnan avulla:

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

Käynnistä Docker-palvelu asennuksen jälkeen:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Tarkista asennus

Suorita asennuksen jälkeen seuraavat komennot varmistaaksesi, että Docker ja Docker Compose on asennettu oikein:

```bash
docker --version
docker compose version
```

## Pika-aloitus

### 1. Luo Docker Compose -määritystiedosto

:::vinkki[Käytä määritysten luontiohjelmaa]
**Suosittele käyttöä [Docker Compose Builder (erillinen sivusto)](https://builder.hagicode.com/)** luodaksesi määritystiedoston.

Generaattori tukee:
- Useita API-palveluntarjoajan vaihtoehtoja (Zhipu AI, Anthropic Official, Custom API)
- API URL-osoitteen ja tunnuksen automaattinen määritys
- Mukautetut portit, säilöjen nimet ja muut perusasetukset
- Määritä pysyvät tiedot ja työhakemistopolut
- Määritä automaattisesti Windows/Linux-alustan erot
- Käsittele tiedostojen käyttöoikeusongelmat älykkäästi
- **Kuvalähteen valinta**: Valitse sopiva kuvalähde verkkoympäristösi perusteella

[🚀 Käytä generaattoria välittömästi →](https://builder.hagicode.com/)
:::

1. Avaa [Docker Compose Builder (erillinen sivusto)](https://builder.hagicode.com/)
2. Täytä kokoonpano tarpeidesi mukaan:
   - Valitse API-toimittaja (Zhipu AI, Anthropic Official tai Custom API)
   - Määritä portit, tietokanta, työhakemisto ja muut asetukset
   - **Valitse kuvalähde**: Valitse sopiva kuvalähde verkkoympäristösi perusteella
3. Napsauta generointipainiketta saadaksesi `docker-compose.yml` kokoonpano
4. Tallenna luotu kokoonpano nimellä a `docker-compose.yml` tiedosto
5. Luo tarvittaessa a `.env` tiedostoa arkaluonteisten tietojen määrittämiseksi

#### Kuvan lähteen valinta

Generaattori tukee kolmea kuvalähdevaihtoehtoa:

**Docker Hub**
- **Kuvan osoite**: `newbe36524/hagicode:{tag}`
- **Soveltuvat skenaariot**: Soveltuu käyttäjille, joilla on Docker Hubin peilikiihdytys
- **Edut**: Virallinen kuvalähde, oikea-aikaiset päivitykset, vakaa pääsy
- **Huomautuksia**: Peilin kiihtyvyys on ehkä määritettävä joillakin alueilla

**Azure Container Registry (ACR)**
- **Kuvan osoite**: `hagicode.azurecr.io/hagicode:{tag}`
- **Soveltuvat skenaariot**: Sopii käyttäjille, jotka eivät voi käyttää Docker Hubia paikallisesti
- **Edut**: Tarjoaa vaihtoehtoisen kuvalähteen, ratkaisee verkkoon pääsyn ongelmat
- **Huomautuksia**: kuvat pysyvät synkronoituina Docker Hubin kanssa, mutta niissä saattaa esiintyä lyhyitä viiveitä

**Alibaba Cloud Container Registry (ACR)**
- **Kuvan osoite**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **Soveltuvat skenaariot**: Soveltuu kotikäyttäjille, vakaa ja nopea verkkoyhteys
- **Edut**: Alibaba Cloudin kotimaiset solmut, nopea pääsynopeus, korkea vakaus
- **Huomautuksia**: Alibaba ACR -kuvat pysyvät synkronoituina Docker Hubin kanssa, yleensä 30 minuutissa

:::vinkki[Kuvalähteen valintasuositukset]
- **Kotitalouskäyttäjät suosittelevat Alibaba Cloud ACR:ää**: Jos olet kotimainen käyttäjä, on suositeltavaa priorisoida Alibaba Cloud ACR -kuvalähde nopeaa ja vakaata käyttöä varten.
- **Vaihtoehtoinen Docker Hub**: Jos verkkoympäristösi voi käyttää Docker Hubia, voit myös valita Docker Hubin kuvalähteeksi
- **Vaihtoehtoinen Azure ACR**: Valitse Azure ACR -kuvalähde vain, jos muilla kuvalähteillä on pääsyongelmia
- **Kuvien synkronointi**: Alibaba Cloud ACR -kuvat pysyvät synkronoituina Docker Hubin kanssa, yleensä 30 minuutissa
- **Version johdonmukaisuus**: Kuvatunnisteet ovat yhdenmukaiset kaikissa kolmessa kuvalähteessä, ei tarvitse huolehtia versioongelmista
:::

### 2. Käynnistä Palvelut

Suorita seuraava komento hakemistossa, jossa sinun `docker-compose.yml` sijaitsee:

```bash
docker compose up -d
```

Tämä komento:
- Lataa ja käynnistä luodun mallin määrittämät Hagicode-säilöt
- Luo ja määritä verkko
- Alusta oletustietohakemisto ja ajonaikaiset riippuvuudet

### 3. Tarkista palvelun tila

Tarkista, ovatko säiliöt käynnissä:

```bash
docker compose ps
```

Sinun pitäisi nähdä luodussa mallissasi määritetyt palvelut käynnissä-tilassa.

### Näytä palvelulokit

Kaikkien palvelulokien tarkasteleminen:

```bash
# View all service logs
docker compose logs

# View Hagicode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## Asetusohjeet

Jos sinun on muutettava määritystä (kuten porttien vaihtaminen, API-palveluntarjoajan vaihtaminen jne.), voit:

1. Luo uusi kokoonpano uudelleen käyttämällä [Docker Compose Builder (erillinen sivusto)](https://builder.hagicode.com/)
2. Muokkaa manuaalisesti `docker-compose.yml` ja `.env` tiedostot
3. Käynnistä palvelut uudelleen, jotta asetukset tulevat voimaan:

```bash
docker compose restart
```

## Pääsy sovellukseen

### Web-käyttöliittymä

Kun palvelu on käynnistynyt onnistuneesti, käytä selaimesi kautta:

```
http://localhost:45000
```

### Tietojen pysyvyys

Tuettu Docker Compose -polku käyttää oletusarvoisesti pysyviä taltioita sisältäviä SQLite-datatiedostoja. Jos sinun on tarkastettava liitospisteitä tai suunniteltava varmuuskopioita, tarkista volyymi- ja tietohakemistokartoitukset luodussa `docker-compose.yml`.
