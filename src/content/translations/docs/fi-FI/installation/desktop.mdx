Tämä opas kattaa Hagicode Desktopin asennuksen ja käytön. Se on virallinen työpöytäasiakas, ja se pitää oletusarvoisen paikallisen asennuspolun sekä päivittäiset ohjaimet yhdessä paikassa.

:::vinkki[Suositeltu menetelmä]
Työpöytä on oletusarvoinen paikallinen polku yksittäisille käyttäjille ja kehittäjille, koska:
- asennusohjelma säilyttää oletusvirran yhdessä ohjatussa polussa
- riippuvuuksia ja palveluita hallitaan sovelluksessa
- käyttöliittymä helpottaa tilan ja toimintojen tarkastamista
- sisäänrakennettu versionhallinta helpottaa vaihtamista
- tiedot pysyvät oletuksena paikallisina
:::

## Mikä on Hagicode Desktop?

Hagicode Desktop on Hagicoden virallinen työpöytäsovellus, joka on suunniteltu Windows-, macOS- ja Linux-järjestelmille. Se yhdistää Hagicoden ydinominaisuudet, ja asennusohjelma säilyttää oletusasetuspolun yhdessä ohjatussa kulkussa.

### Tärkeimmät edut

- **Asennus yhdellä napsautuksella**: Suorita kaikki määritysvaiheet automaattisesti ohjatun asennustoiminnon kautta
- **Automaattinen suoritusajan valmistelu**: Oletusasennusohjelma sisältää jo sen tarvitseman suoritusajan, joten sinun ei tarvitse asentaa erillistä `.NET` käyttöaika ensin
- **Palvelunhallinta**: Sisäänrakennettu Hagicode-palvelimen hallinta, helppo käynnistää ja pysäyttää palvelut
- **Versionvaihto**: Tuki useiden versioiden hallintaan, voit vaihtaa vapaasti eri versioiden välillä
- **Paikallinen käyttö**: Kaikki tiedot tallennetaan paikallisesti, mikä suojaa yksityisyyttäsi
- **Automaattiset päivitykset**: Tunnistaa ja lataa automaattisesti uusimman version

### Sovellettavat skenaariot

Työpöytä sopii hyvin, kun:
- Yksittäiset kehittäjät, jotka haluavat käyttää Hagicodea paikallisesti
- Käyttäjät, joiden on käytettävä Hagicodea offline-tilassa
- Ei-tekniset käyttäjät, jotka haluavat yksinkertaistaa asennusprosessia
- Kehittäjät, joiden on vaihdettava usein eri versioiden välillä

### Muut lataustavat

Voit myös hankkia Hagicode Desktop -asennusohjelman seuraavista kanavista:

1. **Virallinen julkaisusivu**: Vieraile [Hagicode GitHub -julkaisut](https://github.com/Hagicode-org/releases) ladataksesi uusimman version
2. **Virallinen verkkosivusto**: Vieraile [Hagicode-verkkosivusto](https://hagicode.com/) saadaksesi latauslinkkejä

### Valitse oikea versio

Valitse käyttöjärjestelmääsi vastaava asennusohjelma:
- **Windows**: Lataa `.exe` asentaja
- **macOS**: Lataa `.dmg` asentaja
- **Linux**: Lataa `.AppImage` tai `.tar.gz` asennuspaketti

:::note[Järjestelmävaatimukset]
- **Windows**: Windows 10 tai uudempi versio (64-bittinen)
- **macOS**: macOS 10.15 tai uudempi
- **Linux**: yleiset Linux-jakelut (Ubuntu 20.04+, Fedora 33+ jne.)
:::

:::vinkki[Oletusehdot]
Uusinta HagiCode paikallista asennuskulkua varten sinun ei yleensä tarvitse valmistella erillistä asennusta `.NET` suoritusaika. Desktop niputtaa jo oletusvirran vaatiman ajonajan ja alustaa tuetun paikallisen SQLite-polun oletuksena.
:::

## Asennus- ja käynnistysohjeet

:::varoitus[macOS Gatekeeper -ilmoitus]
Jos olet jo asentanut sovelluksen oletuspolkuun `/Applications/Hagicode.app` mutta macOS näyttää "sovellus on vaurioitunut", "ei voi avata" tai vastaavan portinvartijan varoituksen ensimmäisessä käynnistyksessä, tavallinen syy on `com.apple.quarantine` attribuutti, jonka macOS lisää Internetistä ladattuihin sovelluksiin, ei itse Hagicode Desktop -asennusohjelman vika.

Suorita seuraava komento Terminalissa. Se vaatii järjestelmänvalvojan oikeudet ja poistaa toistuvasti karanteeniattribuutin sovelluksesta oletusasennuspolussa:

```bash
sudo xattr -dr com.apple.quarantine /Applications/Hagicode.app
```

Kun komento on valmis, käynnistä Hagicode Desktop uudelleen.
:::

## Suorita työpöytä

Lataamisen jälkeen avaa asennus- tai sovelluspaketti ja seuraa näytön ohjeita. Desktop valmistelee ajonaikaa tällä hetkellä tuetulle paikalliselle polulle, hallitsee Hagicode-palvelinta ja näyttää edistymisen ja tilan, kun verkkolatauksia tarvitaan.

:::note[Verkkovaatimukset]
Varmista, että verkkoyhteytesi on käytettävissä. Työpöytä saattaa joutua hakemaan versiotiedot tai komponenttipaketit Internetistä; Jos lataus epäonnistuu, tarkista verkko-, välityspalvelin- tai palomuuriasetukset ja yritä uudelleen.
:::

## Versionhallinta

Hagicode Desktop tukee usean version hallintaa, jolloin voit vaihtaa vapaasti eri versioiden välillä.

![Versionhallinta](../../img/screenshots/installation/install-desktop/12-version-management/original.png)

### Vaihda versiota

1. Avaa Hagicode Desktop
2. Siirry "Versionhallinta" -sivulle
3. Valitse versio, jonka haluat vaihtaa
4. Napsauta "Vaihda versiota" -painiketta
5. Odota, että vaihto on valmis

:::note[Versionvaihtohuomautuksia]
- Versioiden vaihtaminen käynnistää Hagicode Serverin automaattisesti uudelleen
- Varmista, että olet tallentanut kaikki työt ennen vaihtamista
- Projektikokoonpanot voivat vaihdella versioittain
:::

## Käynnistä ja lopeta palvelu

Hagicode Desktop tarjoaa yksinkertaisen tavan hallita Hagicode Server -palvelua.

### Aloita palvelu

Napsauta Desktop-sovelluksen kotisivulla "Aloita palvelu" -painiketta käynnistääksesi Hagicode Server.

![Aloita palvelu](../../img/screenshots/installation/install-desktop/13-start-service-homepage/original.png)

### Pysäytä palvelu

Pysäytä Hagicode-palvelin napsauttamalla "Lopeta palvelu" -painiketta kotisivulla.

:::vinkki[Palvelun tila]
Desktop-sovellus näyttää Hagicode Serverin toimintatilan reaaliajassa, mukaan lukien:
- Palvelun tila (käynnissä/pysähdyksissä)
- Palveluportin numero
- Nykyisen version tiedot
:::

## Vianetsintä

### Asennusvirhe

Jos kohtaat ongelmia asennuksen aikana:

1. **Tarkista verkkoyhteys**: Varmista, että pääset Internetiin
2. **Tarkista levytila**: Varmista, että sinulla on riittävästi levytilaa
3. **Suorita järjestelmänvalvojana**: Napsauta Windowsissa asennusohjelmaa hiiren kakkospainikkeella ja valitse "Suorita järjestelmänvalvojana"
4. **Näytä lokitiedostoja**: Asennusohjelma luo yksityiskohtaisia lokitiedostoja, jotka voivat auttaa diagnosoimaan ongelmia

### Riippuvuuden asennusvirhe

Jos riippuvuuden asennus epäonnistuu:

1. **Tarkista järjestelmän yhteensopivuus**: Varmista, että käyttöjärjestelmäsi täyttää vähimmäisvaatimukset
2. **Tarkista verkko- ja palomuuriasetukset**: Varmista, että asennusohjelma saavuttaa vaaditut lataukset ja että palomuurisi ei estä sitä
3. **Varmista asennusohjelman käyttöoikeudet**: Windowsissa, yritä suorittaa asennusohjelma järjestelmänvalvojana, jotta sillä on riittävät käyttöoikeudet
4. **Käynnistä sovellus uudelleen ja yritä uudelleen**: Sulje ja avaa Hagicode Desktop uudelleen ja yritä asennusvaihetta uudelleen
5. **Näytä lokit**: Tarkastele yksityiskohtaisia virhelokeja työpöytäsovelluksessa

### Palvelun käynnistysvirhe

Jos Hagicode-palvelin ei käynnisty:

1. **Tarkista portin käyttöaste**: Varmista, että portti 45000 ei ole muiden ohjelmien käytössä
2. **Tarkista palomuuriasetukset**: Varmista, että palomuuri sallii pääsyn Hagicode Server -verkkoon
3. **Käynnistä sovellus uudelleen**: Yritä käynnistää Hagicode Desktop uudelleen
4. **Näytä lokit**: Tarkastele yksityiskohtaisia virhelokeja työpöytäsovelluksessa

### Tarvitsetko lisää apua?

Jos kohtaat ongelmia, joita ei käsitellä tässä:

1. Tarkista [GitHub-ongelmat](https://github.com/HagiCode-org/site/issues) vastaaviin ongelmiin
2. Vieraile meillä [Yhteisön foorumi](https://github.com/HagiCode-org/site/discussions) apua varten
3. Lähetä uusi ongelma ja kuvaile ongelmasi yksityiskohtaisesti
