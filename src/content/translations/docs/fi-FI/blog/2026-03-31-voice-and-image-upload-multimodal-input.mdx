## Kirjoittaminen on hitaampaa kuin puhuminen ja puhuminen on hitaampaa kuin kuvakaappaus - Multimodaalisia syöttökäytäntöjä tekoälyn koodausavustajille

> Koodin kirjoittamisessa on nopeusrajoitus riippumatta siitä, kuinka nopeasti kirjoitat. Joskus sen kirjoittaminen, jonka voit sanoa yhdellä lauseella, kestää ikuisuuden; joskus yksi kuvakaappaus selittää kaiken, mutta silti sinun täytyy kuvailla sitä kasalla tekstiä. Tässä artikkelissa kerrotaan, mihin törmäsimme rakentamisen aikana [HagiCode](https://hagicode.com)puheentunnistuksesta kuvien lataamiseen. Lopulta halusimme vain tehdä tekoälyn koodausassistentista hieman helpompi käyttää.

## Tausta

HagiCodea rakentaessamme havaitsimme ongelman – tai pikemminkin ongelman, joka ilmeni luonnollisesti, kun ihmiset alkoivat käyttää sitä voimakkaasti: pelkkä kirjoittamiseen luottaminen voi olla väsyttävää.

Ajattele sitä: käyttäjien ja agentin välinen vuorovaikutus on keskeinen skenaario. Mutta jos jokainen vaihto vaatii jatkuvaa kirjoittamista näppäimistöllä, tehokkuus ei ole suuri:

1. **Kirjoittaminen on liian hidasta**: Monimutkaisissa ongelmissa, kuten virheilmoituksissa tai käyttöliittymäongelmissa, kaiken kirjoittaminen voi kestää puoli minuuttia, kun taas ääneen sanominen voi kestää kymmenen sekuntia. Tuo ero on todellinen.

2. **Kuvat ovat suorempia**: Joskus käyttöliittymä antaa virheen, joskus haluat verrata suunnitteluluonnoksia ja joskus sinun on näytettävä koodirakenne. "Kuva on enemmän kuin tuhat sanaa" saattaa olla vanha sanonta, mutta se pitää silti paikkansa. AI:n antaminen suoraan "nähdä" ongelma on paljon selkeämpää kuin sen kuvaileminen aikojen kuluessa.

3. **Vuorovaikutuksen tulee tuntua luonnolliselta**: Nykyaikaisten tekoälyavustajien tulisi tukea tekstiä, ääntä ja kuvia. Käyttäjien tulisi voida valita mikä syöttötapa tuntuu luontevimmalta.

Joten päätimme lisätä puheentunnistuksen ja kuvien lataustuen HagiCodeen tehdäksemme agentin vuorovaikutuksesta mukavampaa. Jos käyttäjät voivat kirjoittaa hieman vähemmän, se on jo voitto.

## Tietoja HagiCodesta

Tässä artikkelissa jaetut ratkaisut ovat peräisin käytännön työstämme [HagiCode](https://hagicode.com) hankkeessa - tai tarkemmin sanottuna oppitunneista, jotka on saatu kompastuessaan useiden sudenkuoppien läpi.

HagiCode on avoimen lähdekoodin tekoälyn koodausassistenttiprojekti, jolla on yksinkertainen tavoite: käytä tekoälyä kehitystehokkuuden parantamiseen. Kun jatkoimme rakentamista, kävi selväksi, että käyttäjät halusivat voimakkaasti multimodaalista panosta. Joskus yhden lauseen puhuminen on nopeampaa kuin pitkän kappaleen kirjoittaminen, ja joskus kuvakaappaus on paljon selkeämpää kuin pitkä selitys.

Nämä tarpeet työnsivät meitä eteenpäin, ja näin ominaisuudet, kuten puheentunnistus ja kuvien lataaminen, lopulta muotoutuivat. Käyttäjät voivat nyt olla vuorovaikutuksessa tekoälyn kanssa luonnollisimmalla käytettävissä olevalla tavalla, ja se tuntuu hyvältä.

## Analyysi

### Tekniset haasteet puheentunnistuksessa

Puheentunnistusta luotaessa törmäsimme hankalaan ongelmaan: **selaimen WebSocket API ei tue mukautettuja HTTP-otsikoita**.

Puheentunnistuspalveluksi valitsimme ByteDancen Doubao Speech Recognition API. Valitettavasti tämä API vaatii todennustietoja, kuten `accessToken` ja `secretKey` lähetetään HTTP-otsikoiden kautta. Tämä aiheutti välittömän teknisen ristiriidan:

```javascript
// The browser WebSocket API does not support this approach
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // Not supported
  }
});
```

Meillä oli periaatteessa kaksi vaihtoehtoa:

1. **URL-kyselyparametrien lähestymistapa**: laita todennustiedot URL-osoitteeseen
   - Etuna on, että se on helppo toteuttaa
   - Huono puoli on se, että käyttöoikeustiedot ovat alttiina käyttöliittymälle, joka on epävarma; jotkin API:t vaativat myös otsikkopohjaisen todennuksen

2. **Taustavälityspalvelintapa**: ota WebSocket-välityspalvelin käyttöön taustajärjestelmässä
   - Etuna on, että tunnistetiedot säilyvät turvallisesti taustalla ja ratkaisu on täysin yhteensopiva API-vaatimusten kanssa.
   - Huono puoli on, että toteutus on hieman monimutkaisempi

Lopulta valitsimme **taustavälityspalvelimen**. Turvallisuus ei ole asia, josta et tee kompromisseja.

### Kuvien lataamisen toiminnalliset vaatimukset

Vaatimuksemme kuvien latauksille olivat itse asiassa melko yksinkertaiset:

1. **Useita lataustapoja**: valitse tiedosto napsauttamalla, vedä ja pudota ja liitä leikepöydältä
2. **Tiedoston validointi**: tyyppirajoitukset (PNG, JPG, WebP, GIF) ja kokorajoitukset (5-10 Mt) ovat perusvaatimuksia
3. **Käyttäjäkokemus**: lataa edistyminen, esikatselut ja virheilmoitukset, jotta käyttäjät tietävät aina, mitä tapahtuu
4. **Turvallisuus**: Palvelinpuolen vahvistus ja suojaus haitallisia tiedostolatauksia vastaan ovat tärkeitä

## Ratkaisu

### Puheentunnistus: WebSocket-välityspalvelinarkkitehtuuri

Suunnittelimme kolmikerroksisen arkkitehtuurin puheentunnistukseen ja löysimme toimivan polun:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**Ydinkomponenttien toteutukset**:

1. **Etuosan AudioWorklet-prosessori**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Resample to 16 kHz (required by the Doubao API)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Accumulate samples into 500 ms chunks
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Convert to 16-bit PCM and send
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **WebSocket-taustakäsittelijä** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **Etu `VoiceTextArea` komponentti**:

```tsx
export const VoiceTextArea = forwardRef<HTMLTextAreaElement, VoiceTextAreaProps>(
  ({ value, onChange, onTextRecognized, maxDuration }, ref) => {
    const { isRecording, interimText, volume, duration, startRecording, stopRecording } =
      useVoiceRecording({ onTextRecognized, maxDuration });

    return (
      <div className="flex gap-2">
        {/* Voice button */}
        <button onClick={handleButtonClick}>
          {isRecording ? <VolumeWaveform volume={volume} /> : <Mic />}
        </button>
        {/* Text input area */}
        <textarea value={displayValue} onChange={handleChange} />
      </div>
    );
  }
);
```

### Kuvien lataukset: Monimenetelmän latauskomponentti

Rakensimme täysin varustetun kuvien latauskomponentin, joka tukee kaikkia kolmea lataustapaa ja joka kattaa yleisimmät tilanteet, joihin käyttäjät törmäävät.

**Ydinominaisuudet**:

1. **Kolme lataustapaa**:

```tsx
// Click to upload
const handleClick = () => fileInputRef.current?.click();

// Drag-and-drop upload
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Clipboard paste
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **Etutarkistus**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **Taustalatauksen käsittelijä** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Validation
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Save file
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## Käytännön opas

### Puheentunnistuksen käyttäminen

1. **Määritä puheentunnistuspalvelu**:
   - Avaa puheentunnistuksen asetussivu
   - Määritä Doubao Speech `AppId` ja `AccessToken`
   - Määritä valinnaisesti toimintosanoja parantaaksesi verkkotunnuskohtaisten termien tunnistustarkkuutta

2. **Käytä sitä syöttökentässä**:
   - Napsauta mikrofonikuvaketta syöttöruudun vasemmalla puolella
   - Aloita puhuminen, kun aaltomuoto-animaatio tulee näkyviin
   - Lopeta tallennus napsauttamalla kuvaketta uudelleen
   - Tunnistettu teksti lisätään automaattisesti kohdistimen kohtaan

3. **Esimerkki toimintosanan määrittämisestä**:

```text
TypeScript
React
useState
useEffect
```

### Kuvalatausten käyttäminen

1. **Lataustavat**:
   - Napsauta latauspainiketta valitaksesi tiedoston
   - Vedä kuva suoraan latausalueelle
   - Käytä `Ctrl+V` liittääksesi kuvakaappauksen leikepöydältä

2. **Tuetut muodot**: PNG, JPG, JPEG, WebP, GIF
3. **Kokorajoitus**: oletuksena 5 Mt (konfiguroitavissa)

### Huomautuksia

1. **Puuntunnistus**:
   - Mikrofonilupa vaaditaan
   - Käytä mahdollisuuksien mukaan rauhallisessa ympäristössä
   - Suurin tuettu tallennusaika on oletuksena 300 sekuntia (konfiguroitavissa)

2. **Kuvien lataukset**:
   - Vain yleisiä kuvamuotoja tuetaan
   - Kiinnitä huomiota tiedostokokorajoituksiin
   - Ladatut kuvat saavat automaattisesti esikatselu-URL-osoitteen

3. **Turvallisuusnäkökohdat**:
   - Puheentunnistuksen tunnistetiedot on tallennettu taustajärjestelmään
   - Kuvien lataukset käyvät läpi tiukan palvelinpuolen vahvistuksen
   - HTTPS/WSS on suositeltavaa tuotantoympäristöissä

## Johtopäätös

Puheentunnistuksen ja kuvien latauksen lisäämisen jälkeen HagiCode-käyttökokemus parani huomattavasti. Käyttäjät voivat nyt olla vuorovaikutuksessa tekoälyn kanssa luonnollisemmalla tavalla – puhua kirjoittamisen sijaan ja jakaa kuvakaappauksia sen sijaan, että kuvaisi kaikkea manuaalisesti. Tuntuu siltä, ​​että vihdoin löydetään mukavampi tapa kommunikoida.

Kun rakennamme tätä ominaisuutta, törmäsimme siihen ongelmaan, että selaimen WebSocket API:t eivät tue mukautettuja otsikoita. Lopulta ratkaisimme sen backend-välityspalvelinlähestymistavalla. Ratkaisu ei ainoastaan ​​säilyttänyt turvallisuutta, vaan loi pohjan muiden todennettujen WebSocket-palvelujen integroimiseksi myöhemmin.

Kuvien latauskomponentti hyötyy myös useiden lataustapojen tukemisesta, jolloin käyttäjät voivat valita, mikä on tällä hetkellä sopivin. Napsauttamalla, vetämällä tai liittämällä kaikki työt, jokainen polku saa työn valmiiksi nopeasti.
