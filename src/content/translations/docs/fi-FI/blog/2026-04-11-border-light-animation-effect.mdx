## Opas Border Light Sweep -animaatiotehosteen luomiseen

> Kuinka rakennat sen tärkeän elementin, jonka käyttäjät huomaavat yhdellä silmäyksellä puhtaalla CSS:llä? Se ei itse asiassa ole niin vaikeaa. Temppu kulkee vain hieman kiertävää polkua. Tässä artikkelissa opastan sinulle, kuinka voit rakentaa reunavalon pyyhkäisyanimaatiota tyhjästä, ja kerron myös muutamia sudenkuoppia, joihin törmäsimme rakentaessamme HagiCodea.

## Keskeiset toteutusideat

Analysoituamme HagiCode-koodikannan, teimme yhteenvedon useista ydintoteutusmalleista alla. Jokainen niistä sopii eri skenaarioon tai toisin sanoen jokainen on olemassa syystä.

### 1. Pyörivä hehku kartiogradientilla (yleisin)

Tämä on klassinen tapa toteuttaa reunavalon pyyhkäisyefekti. Perusideana on käyttää CSS:ää `conic-gradient` luodaksesi kartiomaisen gradientin, pyöritä sitä sitten jatkuvasti. Kuten yöllä kääntyvä katuvalo, se vain kiertää.

**Avainelementit:**
- Käytä `::before` pseudoelementti hehkukerroksen luomiseksi
- Käytä `conic-gradient` määrittääksesi gradientin värijakauman
- Käytä `::after` pseudoelementti keskialueen peittämiseen (valinnainen)
- Käytä `@keyframes` toteuttaa pyöritysanimaatio

### 2. Sivun hehkuviiva

Tämä toimii hyvin luettelokohteiden tilailmaisimissa. Luot ohuen hehkuvan viivan elementin toiselle puolelle koko reunuksen animoinnin sijaan. Joskus pieni valo riittää. Sinun ei tarvitse valaista koko maailmaa.

**Avainelementit:**
- Ohut, täysin sijoitettu viivaelementti
- Käytä `box-shadow` luodaksesi hehkuefektin
- Käytä `scale` ja `opacity` hengittävään animaatioon

### 3. Hehkuva tausta laatikkovarjolla

Jos et tarvitse täydellistä pyyhkäisytehostetta ja haluat vain pehmeän taustahehkun, kerrosta useita `box-shadow` arvot riittää. Joskus yksinkertaisempi vaihtoehto on parempi.

### 4. Esteettömyystuki

Tämä osa on helppo jättää huomiotta, mutta se on erittäin tärkeä. Jokaisen animaation tulee ottaa huomioon `prefers-reduced-motion` mediakyselyä ja tarjota staattinen vaihtoehto käyttäjille, jotka eivät halua animaatioita. Kaikki eivät nauti jatkuvasta liikkeestä, ja tämän mieltymyksen kunnioittaminen on tärkeää.

## Käyttöönottovaihtoehdot

### Vaihtoehto 1: Pyörivä kartiogradienttireunus (suositus)

Tämä on lakaisevan reunuksen valotehosteen täydellisin toteutus, ja se on myös vaihtoehto, jota käytämme useimmin HagiCodessa. Loppujen lopuksi, jos jokin toimii hyvin, miksi se korvataan?

```css
/* Parent container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotating glow layer */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Mask layer (optional, for creating a hollow border effect) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Periaate tämän vaihtoehdon takana on melko yksinkertainen: luo pseudoelementti, joka on suurempi kuin pääsäiliö, piirrä siihen kartiogradientti ja pyöritä sitä jatkuvasti. Pääsäilö käyttää `overflow: hidden`, joten vain reunan ympäri kulkeva valo jää näkyviin. Se on vähän kuin katsoisi katuvaloa ikkunasta. Näet aina vain pienen siivun, joka kulkee ohi.

### Vaihtoehto 2: Yksinkertaistettu pyörivä valoreunus

Jos et tarvitse täydellistä tehostetta, HagiCode sisältää myös kevyemmän hyötyluokan version. Joskus yksinkertaisempi lähestymistapa on todella parempi.

```css
/* Rotating light border utility class */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Accessibility support */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Huomaa, `will-change: transform` täällä. Se kertoo selaimelle: "Tämä elementti muuttuu jatkuvasti", joten selain voi valmistella joitain optimointeja etukäteen ja pitää animaation sujuvana. Ennakkoon valmistautuminen on yleensä parempi kuin viime hetken sekoitus.

### Vaihtoehto 3: Sivuhehkuviiva

Tämä sopii erityisen hyvin listakohteiden tilailmaisimiin, ja juuri sitä HagiCode-istuntoluettelo käyttää. Yksi ohut viiva voi silti erottua monien esineiden joukosta. Se tuntuu elämänopetukselta omalla tavallaan.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Tämä käyttää `isolation: isolate` luodaksesi uuden pinoamiskontekstin, luottaa sitten siihen `z-index` ohjata kunkin kerroksen näyttöjärjestystä. `pointer-events: none` on myös välttämätöntä. Muuten pseudoelementti estäisi käyttäjien napsautukset. Jotkut asiat voivat näyttää hyvältä, mutta niiden ei silti pitäisi olla tiellä.

### Vaihtoehto 4: Kääri se React-komponenttiin

Jos projektisi käyttää Reactia, voit kääriä tämän logiikan komponenttiin, erityisesti esteettömyyskäsittelyyn. Kirjoita se kerran, käytä sitä monta kertaa. Se on koko pointti.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Vastaava CSS-moduuli:

```css
/* GlowBorder.module.css */

/* Animated version */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Static version (accessibility) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

The `useReducedMotion` koukku alkaen `framer-motion` tunnistaa automaattisesti käyttäjän järjestelmäasetukset. Jos käyttäjä on ottanut pienen liikkeen käyttöön, se palaa `true`, ja komponentti näyttää sen sijaan staattisen version. Käyttäjän mieltymysten kunnioittaminen on tärkeämpää kuin räikeän tehosteen pakottaminen.

## Käytännön oppitunteja

Nämä ovat joitain oppitunteja, joita opimme rakentaessamme HagiCodea. Voit myös kutsua niitä taisteluarpeiksi. Toivottavasti ne auttavat sinua välttämään kiertoteitä.

### 1. Teemamuuttujajärjestelmä

CSS-muuttujat tekevät usean teeman tuesta erityisen kätevän. Kukaan ei halua muokata koodipinoa aina, kun teema vaihtuu.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Usage */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

Tällä tavalla teeman vaihtaminen edellyttää vain luokan vaihtamista `html` elementti, ja jokaisen animaation väri päivittyy automaattisesti. Yksi koodikanta, kaksi tyyliä. Juuri sitä me haluamme.

### 2. Suorituskyvyn optimointi

**Käytä `will-change` vihje selaimelle optimoimaan:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Kerro selaimelle etukäteen, niin se auttaa sinua optimoimaan. Monet asiat elämässä toimivat paremmin pienellä valmistautumisella.

**Vältä monimutkaisten laatikkovarjojen käyttöä suurissa elementeissä:**

```css
/* Not ideal - using a blurred shadow on a large element */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Better - use a pseudo-element to limit the glowing area */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Testasimme tämän HagiCodessa. Sumean varjon lisääminen suoraan suurelle kortille pudotti vierityskuvanopeuden alle 30 fps. Vaihtaminen pseudoelementtiin toi asiat takaisin tasaiseen 60 kuvaa sekunnissa. Käyttäjät voivat täysin tuntea tämän eron.

### 3. Helppokäyttöisyys

Tätä ei todellakaan kannata jättää väliin. Joidenkin käyttäjien mielestä animaatio on huimaa tai häiritsevää, ja heidän mieltymystensä kunnioittaminen on osa hyvän tuotteen rakentamista. Kauniita asioita ei tarvitse pakottaa kaikille.

**CSS-mediakysely:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Provide a static fallback */
    opacity: 1;
  }
}
```

**Tunnista käyttäjän mieltymykset Reactissa:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Voimakkuustason säätö

HagiCoden Token-läpäisykyvyn ilmaisin näyttää erilaisia hehkuvärejä reaaliaikaisen suorituskyvyn perusteella, ja tämä toteutetaan dynaamisesti. Eri tilat pitäisi ilmaista eri tavalla.

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Varottavia asioita

Muutamia yksityiskohtia kannattaa vielä kiinnittää huomiota, sillä kun huomaat nämä ongelmat kantapään kautta, on jo liian myöhäistä.

| Varottavia asioita | Selitys |
|---------|------|
| **z-indeksin hallinta** | Hehkukerroksen tulee käyttää sopivaa `z-index` joten se ei häiritse sisällön vuorovaikutusta |
| **osoitintapahtumat** | Hehkun pseudoelementin pitäisi asettua `pointer-events: none` |
| **Rajan ylivuoto** | Pääsäilö tarvitsee `overflow: hidden`, tai sinun on säädettävä pseudoelementin kokoa |
| **Tehokkuusvaikutus** | Monimutkaiset animaatiot voivat heikentää suorituskykyä mobiililaitteilla, joten testaa huolellisesti |
| **Pimeä tila** | Varmista, että hehkuväri pysyy selvästi näkyvissä tummalla taustalla |
| **Teeman vaihto** | Käytä CSS-muuttujia, jotta animaation värit päivittyvät oikein, kun teema vaihtuu |

### 6. Vianetsintävinkkejä

Pseudoelementtejä voi olla hieman vaikea paikantaa kehittäjätyökaluissa, joten voit tilapäisesti lisätä reunuksen sijainnin tarkistamiseksi.

```css
/* Temporarily show pseudo-element boundaries for debugging */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Kun olet sijoittanut sen, muista kommentoida tai poistaa rivi. Muuten tuotanto voi olla hankalaa melko nopeasti. Jotkut asiat on parempi jättää kehitykseen.
