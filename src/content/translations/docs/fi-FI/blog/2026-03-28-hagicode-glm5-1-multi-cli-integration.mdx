## Hagicode- ja GLM-5.1 Multi-CLI -integraatioopas

:::note[Time Context Notice]
Tämä artikkeli dokumentoi usean CLI:n integroinnin suunnittelun ja mallikartoituksen sellaisena kuin ne olivat olemassa julkaisuhetkellä. Katso nykyinen aina tuettu laajuus kohdasta [AI Agent CLI -asennus](/fi-FI/related-software-installation/ai-agent-cli/) ja [Asennusopas](/fi-FI/installation); the `IFlowCli` tässä mainittu edustaa vain historiallista kokoonpanoa tuolloin, eikä se ole osa nykyistä aktiivista tukialuetta.
:::

## Tausta

Hagicode-projektissa käyttäjät voivat valita useista CLI-työkaluista ohjaamaan tekoälyohjelmointiapulaisia, mukaan lukien Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI ja paljon muuta. Nämä CLI-työkalut ovat yksinään yleiskäyttöisiä AI-ohjelmointityökaluja, mutta Hagicoden abstraktiokerroksen kautta ne voivat muodostaa joustavasti yhteyden eri tekoälymallien tarjoajiin.

Zhipu AI (ZAI) tarjoaa Anthropic Claude API:n kanssa yhteensopivan käyttöliittymän, jonka avulla nämä CLI-työkalut voivat käyttää suoraan kotimaisia GLM-sarjan malleja. Niistä GLM-5.1 on Zhipun uusin suuri kielimalliversio, jossa on merkittäviä parannuksia GLM-5.0:aan verrattuna.

## Hagicoden CLI-abstraktioarkkitehtuuri

Hagicode määrittelee 11 CLI-palveluntarjoajatyyppiä `AIProviderType` enum, joka kattaa valtavirran tekoälyohjelmoinnin CLI-työkalut:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Jokaisella CLI:llä on vastaava malliparametrikonfiguraatio ja se tukee `model` ja `reasoning` parametrit:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],
        [AIProviderType.CodexCli] = ["model", "reasoning"],
        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],
        [AIProviderType.HermesCli] = ["model", "reasoning"],
        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini does not support the reasoning parameter
        // ...
    };
```

## GLM-5.1 käyttö eri CLI:issä

### 1. Claude Code CLI + GLM-5.1

Claude Code CLI on yksi Hagicoden keskeisistä CLI:istä ja se konfiguroidaan Hero-määritysjärjestelmän kautta:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Vastaava `HeroEquipmentCatalogItem` kokoonpano:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI on joustavin CLI ja tukee minkä tahansa mallin määrittämistä `provider/model` muoto:

**Tapa 1: Käytä ZAI-palveluntarjoajan etuliitettä**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Tapa 2: Käytä mallitunnusta suoraan**
```json
{
  "model": "glm-5.1"
}
```

**Tapa 3: Käyttöliittymän määrityskäyttöliittymä**

sisään `HeroModelEquipmentForm.tsx`, OpenCode CLI:ssä on oma paikkamerkkivihje:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Käyttäjät voivat syöttää:
```
zai/glm-5.1
glm-5.1
```

**OpenCode CLI -mallin jäsennyslogiikka**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Codebuddy CLI + GLM-5.1

Codebuddy CLI:ssä on oma perinteinen käsittelylogiikka:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Huomaa**: Codebuddy säilyttää erityiskäsittelyn GLM-5.0:lle eikä käytä perinteistä normalisointia:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## GLM-5.1:n parannusedut

GLM-5.0:aan verrattuna GLM-5.1 tuo seuraavat merkittävät parannukset:

### 1. Parempi päättelykyky

Zhipun virallisten julkaisutietojen mukaan GLM-5.1:n parannuksia ovat:

- **Parempi koodin ymmärtäminen**: Tarkempi analyysi monimutkaisista koodirakenteista
- **Pidempi kontekstin ymmärtäminen**: Tukee pidempää keskustelukontekstia
- **Parannettu työkalukutsu**: Parempi onnistumisprosentti MCP-työkalukutsuissa
- **Lähdön vakaus**: Vähentää satunnaisuutta ja hallusinaatioita

### 2. Kattava usean CLI:n yhteensopivuus

GLM-5.1 kattaa kaikki Hagicoden tukemat yleiset CLI:t:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Huomautuksia

### 1. API-avaimen määritys

Varmista, että `ANTHROPIC_AUTH_TOKEN` ympäristömuuttuja on asetettu oikein. Se on jokaisen CLI:n vaadittava tunniste, jotta se voi muodostaa yhteyden malliin.

### 2. Mallin saatavuus

Vastaavan mallintoimittajan on otettava GLM-5.1 käyttöön:
- Zhipu AI ZAI -alusta tukee sitä oletuksena
- Alibaba Cloud DashScope saattaa vaatia erillisen sovelluksen

### 3. OpenCode CLI -muoto

Kun käytät `provider/model` muodossa, varmista, että palveluntarjoajan tunnus on oikea:
- Zhipu AI: `zai` tai `zhipuai`
- Alibaba pilvi: `aliyun` tai `dashscope`

### 4. Päättelyparametri

- `high` suositellaan parhaan koodin luontituloksen saavuttamiseksi
- Gemini CLI ei tue päättelyparametria ja jättää tämän määrityksen huomioimatta automaattisesti

## Johtopäätös

Yhtenäisen abstraktiokerroksen kautta Hagicode mahdollistaa joustavan integraation GLM-5.1:n ja useiden CLI:iden välillä. Kehittäjät voivat valita CLI-työkalun, joka sopii parhaiten heidän mieltymyksiinsä ja käyttöskenaarioihinsa, ja käyttää sitten uusinta GLM-5.1-mallia yksinkertaisen konfiguroinnin avulla.

Zhipun uusimpana malliversiona GLM-5.1 tarjoaa selkeitä parannuksia GLM-5.0:aan verrattuna:
- Itsenäinen version tunniste ilman perintötaakkaa
- Vahvempi päättely ja koodin ymmärtäminen
- Laaja usean CLI:n yhteensopivuus
- Joustava päättelytason konfigurointi

Kun ympäristömuuttujat ja Hero-laitteet on määritetty oikein, käyttäjät voivat vapauttaa GLM-5.1:n tehon täysin eri CLI-ympäristöissä.
