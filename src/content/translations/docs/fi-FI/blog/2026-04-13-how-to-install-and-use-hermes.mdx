Jos haluat asentaa Hermesin ja aloittaa sen käytön, lyhin polku on oikeastaan vain kolme vaihetta:

1. Suorita virallinen asennuskomento
2. Käynnistä CLI päätteessäsi `hermes`
3. Jos haluat jatkaa sen käyttöä Feishussa, määritä `hermes gateway setup`

Tämä artikkeli ei yritä selittää kaikkia Hermesin ominaisuuksia kerralla. Sen sijaan se auttaa sinua suorittamaan ensin tärkeimmän aloittelijasilmukan: asentamaan sen, käynnistämään sen, aloittamaan sen käytön ja yhdistämään sen sitten yhteen yleisimmistä viestintäalustan skenaarioista.

## Mikä Hermes on ja kenelle se on tarkoitettu

Hermes Agent on tekoälyagentti, jota voit käyttää joko paikallisesta päätteestä tai viestinvälitysalustan yhdyskäytävän kautta.

Useimmille kehittäjille sillä on kaksi yhteistä aloituskohtaa:

- **CLI**: Tyyppi `hermes` terminaalissasi päästäksesi suoraan interaktiiviseen käyttöliittymään.
- **Viestiyhdyskäytävä**: Suorita `hermes gateway`, keskustele sen kanssa sellaisilla alustoilla kuin Feishu, Telegram, Discord ja Slack.

Jos tavoitteesi tällä hetkellä on vain aloittaa nopeasti, älä käännä järjestystä. Aloita sen sijaan tästä polusta:

- Asenna ensin Hermes
- Varmista ensin, että se toimii CLI:stä
- Päätä sitten, haluatko yhdistää viestintäalustan

Tämä helpottaa ongelmien diagnosointia ja sopii paremmin ihmisille, jotka käyttävät Hermesiä ensimmäistä kertaa.

## Mitä tietää ennen Hermesin asentamista

Hermes README:n mukaan virallinen pika-asennuspolku tukee näitä ympäristöjä:

- Linux
- macOS
- WSL2
- Android Termuxin kautta

### Huomautuksia Windows-käyttäjille

Hermes ei tällä hetkellä tue käyttöä suoraan alkuperäisessä Windowsissa. Jos käytät Windowsia, suositeltava tapa on asentaa ensin WSL2 ja sitten suorittaa asennuskomento WSL2:n sisällä.

Tämä on parasta tehdä selväksi alussa, koska monet asennusvirheet eivät johdu komennosta itsestään, vaan ajonaikaisen ympäristön käytöstä, jota ei tueta.

## Kuinka asentaa Hermes

Hermes README:n nopea asennuskomento on:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Tämä komento suorittaa virallisen asennuskomentosarjan ja käsittelee alustakohtaisia alustusvaiheita.

### Lataa kuori uudelleen asennuksen jälkeen

Kun asennus on valmis, lataa ensin shell-ympäristö uudelleen. Yleisin komento on:

```bash
source ~/.bashrc
```

Jos käytät zsh:ta, voit käyttää:

```bash
source ~/.zshrc
```

### Kuinka varmistaa, että Hermes on asennettu oikein

Suorin tapa tarkistaa on ajaa:

```bash
hermes
```

Jos haluat lisävahvistuksen siitä, että määritykset ja riippuvuudet toimivat, voit myös suorittaa:

```bash
hermes doctor
```

`hermes doctor` on erityisen hyödyllinen näissä tilanteissa:

- Komento toimii epänormaalisti asennuksen jälkeen
- Mallin määritys epäonnistuu
- Yhdyskäytävä ei käynnisty
- Et ole varma, ovatko ympäristösi riippuvuudet täydellisiä

## Kuinka aloittaa Hermesin käyttö ensimmäistä kertaa

### Aloita CLI:stä

Jos haluat vain varmistaa mahdollisimman nopeasti, että Hermes toimii, yksinkertaisin tapa on:

```bash
hermes
```

Tämä käynnistää interaktiivisen Hermes CLI:n. Ensikertalaisille Hermes-käyttäjille se on myös suositeltavin aloituskohta, koska voit varmistaa tärkeimmät asiat ensin:

- Onko komento todella käytettävissä
- Toimiiko nykyinen mallikokoonpano oikein
- Toimiiko päätetyökaluketju oikein
- Vastaako vuorovaikutustayyli sitä, mitä tarvitset

### Nämä komennot riittävät ensimmäiselle asennuskierrokselle

Hermes README listaa useita korkeataajuisia komentoja, ja yhdessä ne muodostavat käytännöllisen ensikäyttöpolun:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Jos et ole varma, mitä kukin tekee, muista ne seuraavasti:

- `hermes model`: valitse tai vaihda malleja
- `hermes tools`: tarkastella ja määrittää tällä hetkellä käytettävissä olevia työkaluja
- `hermes config set`: muuta tiettyjä kokoonpanokohteita
- `hermes setup`: suorita ohjattu alustustoiminto kerran
- `hermes update`: päivitä Hermes
- `hermes doctor`: ongelmien vianmääritys

Aloittelijoille käytännöllisin järjestys on yleensä:

1. Juokse `hermes model` ensin
2. Jos haluat määrittää kaikki yleiset asetukset kerralla, suorita `hermes setup`

## Kaksi yleisintä tapaa käyttää Hermesiä

### 1. Käytä päätteen Hermesiä päivittäisenä kehitysavustajana

CLI-tila sopii hyvin seuraaviin skenaarioihin:

- Esitä kysymyksiä suoraan kirjoittaessasi koodia paikallisesti
- Tarkista projektit, muokkaa tiedostoja ja suorita komentoja
- Tee kertaluonteinen virheenkorjaus tai tarkistustyö
- Tee jatkuvaa yhteistyötä nykyisessä työhakemistossa

Sen suurin etu on, että se on lyhin polku: ei ylimääräistä alustaintegraatiota, ei bottimäärityksiä etukäteen, ja se on paras tapa rakentaa ensimmäiset käyttötottumuksesi.

### 2. Käytä Hermesiä viestinvälitysalustan kautta

Jos haluat keskustella Hermesin kanssa sellaisilla alustoilla kuin Feishu, Telegram tai Discord, sinun on käytettävä viestintäyhdyskäytävää.

Yleisimmät syöttökomennot ovat:

```bash
hermes gateway setup
hermes gateway
```

Erityisesti:

- `hermes gateway setup` käytetään interaktiiviseen alustan konfigurointiin
- `hermes gateway` käytetään yhdyskäytäväprosessin käynnistämiseen

Virallisen dokumentaation mukaan yhdyskäytävä on yhtenäinen taustaprosessi, joka yhdistää määritetyt alustasi, hallitsee istuntoja ja käsittelee ominaisuuksia, kuten cron-töitä.

## Feishu esimerkkinä: kuinka yhdistää Hermes viestialustaan

Jos suurin osa päivittäisestä työstäsi tapahtuu Feishussa, niin Feishu/Lark on erittäin luonnollinen tapa käyttää Hermesiä.

### Pienin toteuttamiskelpoinen integraatiopolku

Virallinen dokumentaatio suosittelee tätä syöttökomentoa Feishu/Larkille:

```bash
hermes gateway setup
```

Kun olet suorittanut sen, valitse ohjatussa toiminnossa **Feishu / Lark**.

Feishu-dokumentaatiossa kuvataan kaksi yhteystapaa:

- `websocket`: suositeltava
- `webhook`: valinnainen

Jos Hermes toimii kannettavalla tietokoneellasi, työasemallasi tai yksityisellä palvelimellasi, käyttämällä `websocket` ensimmäinen on yleensä yksinkertaisempi, koska sinun ei tarvitse paljastaa julkista takaisinsoitto-URL-osoitetta.

### Jos määrität sen manuaalisesti, tiedä ainakin nämä muuttujat

Jos et käytä ohjattua toimintoa ja kirjoitat määritykset manuaalisesti, Feishu-dokumentaatiossa luetellaan nämä ydinmuuttujat:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Kaksi niistä ansaitsee erityistä huomiota:

- `FEISHU_ALLOWED_USERS`: suositellaan, joten kaikki, jotka voivat tavoittaa botin, eivät voi käyttää sitä suoraan
- `FEISHU_HOME_CHANNEL`: voit määrittää etukäteen kotikeskustelun saadaksesi cron-tuloksia tai oletusilmoituksia

### Miksi Hermes ei toisinaan vastaa Feishu-ryhmäkeskusteluihin

Tämä yksityiskohta on helppo missata: Feishun ryhmäkeskusteluissa Hermes ei vastaa oletusarvoisesti jokaiseen viestiin.

Virallisissa asiakirjoissa sanotaan selvästi:

- Suorissa viesteissä Hermes vastaa viesteihin
- Ryhmäkeskusteluissa sinun on tehtävä erikseen `@` botti ennen kuin se käsittelee viestin

Jos haluat asettaa Feishu-keskustelun kotikanavaksi, voit käyttää tätä myös chatissa:

```text
/set-home
```

Tai määritä se kokoonpanossa etukäteen:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Aloittelijan tulee muistaa Hermes-komennot

Käytätpä sitten Hermesiä CLI:ssä tai viestiympäristössä, seuraavien komentojen muistaminen riittää jo aloittamiseen:

- `/new` tai `/reset`: aloita uusi istunto
- `/model`: näytä tai vaihda mallia
- `/retry`: yritä edellistä käännöstä uudelleen
- `/undo`: kumoa edellinen vuorovaikutus
- `/compress`: pakkaa konteksti manuaalisesti
- `/help`: katso ohje

Jos käytät Hermesiä pääasiassa viestintäalustalla, muista vielä yksi:

- `/sethome` tai `/set-home`: aseta nykyinen chat kotikanavaksi

Nämä komennot kattavat yleisimmät aloittelijavaiheen toiminnot: uudelleenkäynnistys, säätö, peruuttaminen, tarkistaminen ja jatkaminen.

## Usein kysytyt kysymykset

### Voiko Windows asentaa Hermesin suoraan?

Ei. Nykyisessä virallisessa dokumentaatiossa todetaan selvästi, että alkuperäistä Windowsia ei tueta, ja WSL2:ta suositellaan.

### Mitä minun pitäisi tehdä, jos kirjoitan `hermes` ei mitään asennuksen jälkeen?

Vianetsintä on parasta tässä järjestyksessä:

1. Lataa ensin kuori uudelleen, esimerkiksi `source ~/.bashrc`
2. Juokse `hermes` uudelleen
3. Jos se on edelleen epänormaali, juokse `hermes doctor`

### Miksi botti ei vastaa Feishu-ryhmässä?

Tarkista ensin nämä kolme asiaa:

- Olitpa sitten `@` mainitsi Hermeksen ryhmässä
- onko `FEISHU_ALLOWED_USERS` rajoittaa nykyistä käyttäjää
- Salliiko nykyinen ryhmäkeskustelukäytäntö ryhmäviestien käsittelyn

Virallisen Feishu-dokumentaation mukaan ryhmäkeskusteluissa vaaditaan nimenomaan @maininnan käyttäminen.

## Suositellut seuraavat vaiheet

Jos haluat vain aloittaa Hermeksen käytön mahdollisimman nopeasti, tämä on suosituin tilaus:

1. Suorita asennuskomento ensin
2. Aloita `hermes` ensin paikallisessa CLI:ssä
3. Käytä `hermes model` ja `hermes setup` suorittaaksesi peruskokoonpanon loppuun
4. Jos haluat jatkaa sen käyttöä Feishussa, määritä `hermes gateway setup`

Jos tämä artikkeli on sarjan ensimmäinen osa, sen paras tehtävä ei ole selittää kaikkia edistyneitä ominaisuuksia kerralla, vaan saada käyttäjät ovesta ensin.

Seuraavat aiheet on parempi jakaa jatkoartikkeleihin:

- Täydellinen Hermes Feishu -integraatioopas
- Opas yleisiin Hermesin kauttaviivakomentoihin
- Opas Hermes-yhdyskäytävän määritykseen ja vianetsintään

Jos aiot jatkaa Hermes-sisällön luomista, tämä artikkeli voi toimia lähtökohtana myöhemmille viesteille, kun rakennat asteittain sisäistä linkkirakennetta.
