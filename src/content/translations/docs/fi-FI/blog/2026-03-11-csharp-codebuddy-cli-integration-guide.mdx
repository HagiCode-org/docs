## Käytännön opas CodeBuddy CLI:n integroimiseen C#-taustajärjestelmään

> Tässä artikkelissa käydään läpi täydellinen lähestymistapa CodeBuddy CLI:n integroimiseen C#-taustaprojektiin, jotta voit toimittaa AI-koodausavustajan ominaisuuksia päästä päähän.

Katso nykyinen pysyvästi tuettu laajuus kohdasta [AI Agent CLI -asennus](/fi-FI/related-software-installation/ai-agent-cli/) ja [Asennusopas](/fi-FI/installation); `IFlowCli` on nyt säilytetty vain historiallisena yhteensopivuusarvona, eikä se ole osa nykyistä aktiivista tukialuetta.

## Tausta

Nykyaikaisessa tekoälyn koodausavustajakehityksessä yksi AI Provider ei useinkaan pysty tyydyttämään monimutkaisia ja muuttuvia kehitysskenaarioita. HagiCoden on monikäyttöisenä tekoälyn koodausavustajana tuettava useita tekoälyn tarjoajia tarjotakseen paremman käyttökokemuksen. Käyttäjillä tulee olla riittävästi valinnanvapautta. Vuoden 2026 alussa projekti joutui ratkaisemaan keskeisen päätöksen: kuinka palauttaa CodeBuddy ACP (Agent Communication Protocol) -integrointiominaisuudet C#-taustajärjestelmään.

Projektissa oli aiemmin toteutettu CodeBuddy-integraatio, mutta siihen liittyvä koodi poistettiin uudelleenkäsittelyn aikana. Siellä ei ole paljon valittamista; iteratiivisen kehityksen aikana jotain jää aina taakse. Tämän teknisen ratkaisun tavoitteena oli palauttaa tämä ominaisuus täysin ja parantaa arkkitehtuuria, jotta se olisi kestävämpi ja ylläpidettävämpi.

Jos harkitset myös useiden AI-koodausavustajien yhdistämistä omaan projektiisi, alla oleva lähestymistapa saattaa antaa sinulle ideoita. Se kuvastaa oppitunteja, joista teimme yhteenvedon astuttuamme moniin sudenkuoppiin, ja ehkä se voi auttaa sinua välttämään muutamia kiertoteitä.

## Arkkitehtuurisuunnittelu

### Kerroksellisen arkkitehtuurin yleiskatsaus

Integraatio C#:n ja CodeBuddyn välillä käyttää selkeää kerrosrakennetta. Tämä muotoilu tekee vastuista selkeät ja helpottaa pitkäkestoista ylläpitoa:

```
┌─────────────────────────────────────────────┐
│           Provider Contract Layer           │
│   AIProviderType enum + extension methods   │
├─────────────────────────────────────────────┤
│           Provider Factory Layer            │
│   AIProviderFactory dependency injection factory │
├─────────────────────────────────────────────┤
│           Provider Implementation Layer     │
│   CodebuddyCliProvider concrete implementation │
├─────────────────────────────────────────────┤
│           ACP Infrastructure Layer          │
│  ACPSessionManager / StdioAcpTransport      │
│  AcpRpcClient / AcpAgentClient              │
└─────────────────────────────────────────────┘
```

Mitä hyötyä tästä kerrostamisesta on? Yksinkertaisesti sanottuna jokainen kerros pysyy poissa muiden tieltä. Jos haluamme myöhemmin muuttaa viestintämekanismia, esimerkiksi `stdio` WebSocketille, meidän tarvitsee vain muokata alempaa kerrosta, ja sen yläpuolella oleva liiketoimintalogiikka pysyy koskemattomana. Kukaan ei halua viestintämuutoksen leviävän koko koodikannan läpi.

### Ydinkomponenttien erittely

**Provider sopimuskerros** on koko arkkitehtuurin perusta. Me määrittelemme `AIProviderType` enum, missä `CodebuddyCli = 3` käytetään enum-arvona ja toteuta kaksisuuntainen määritys merkkijonojen ja enumien välillä laajennusmenetelmien avulla. Tämän ansiosta määritystiedostojen merkkijonot voidaan muuntaa kätevästi enumiksi ja enumit muuntaa takaisin merkkijonoiksi virheenkorjausta varten.

**Provider-tehdastaso** vastaa vastaavan Provider-esiintymän luomisesta kokoonpanon perusteella. Se käyttää .NET-riippuvuuden lisäystä yhdessä `ActivatorUtilities.CreateInstance` dynaamiseen luomiseen. Tehdasmallin etuna on, että kun lisäät uutta toimittajaa, sinun tarvitsee vain lisätä luontilogiikka olemassa olevan koodin muokkaamisen sijaan.

Varsinainen työ tapahtuu **Provider-toteutustasolla**. `CodebuddyCliProvider` toteuttaa `IAIProvider` käyttöliittymä ja tarjoaa kaksi kutsutilaa: `ExecuteAsync` muille kuin suoratoistopuheluille ja `StreamAsync` puhelujen suoratoistoa varten.

**ACP-infrastruktuurikerros** muodostaa viestintäperustan alla. Tämä kerros käsittelee kaikki protokollan yksityiskohdat, mukaan lukien prosessien hallinta, viestien serialisointi ja vastausten jäsentäminen. Se on perusta, joka pitää kaiken sen yläpuolella vakaana.

## Viestintämekanismi

### Stdio-kuljetustila

CodeBuddy käyttää **Stdioa (vakiotulo/lähtö)** viestiäkseen ulkoisten prosessien kanssa. Käynnistyskomento on yksinkertainen:

```bash
codebuddy --acp
```

Tämän jälkeen JSON-RPC-viestit vaihdetaan vakiotulon ja -lähdön kautta. Tällä lähestymistavalla on useita etuja:

1. **Nopea käynnistys**: paikallinen prosessiviestintä välttää verkon latenssin
2. **Yksinkertainen määritys**: sinun tarvitsee vain määrittää suoritettavan polun
3. **Ympäristön eristäminen**: jokainen istunto suoritetaan itsenäisenä prosessina, joten ne eivät vaikuta toisiinsa

Ympäristömuuttuja-injektiota tuetaan tiedonsiirron aikana. Yleisiä esimerkkejä ovat:

- `CODEBUDDY_API_KEY`: API-avaimen todennus
- `CODEBUDDY_INTERNET_ENVIRONMENT`: verkkoympäristön määritys

Kuten ihmisten välisessä viestinnässä, se auttaa valitsemaan ensin sopiva kanava.

### Viestiprotokolla

ACP perustuu JSON-RPC 2.0:aan. Viestin muoto näyttää suunnilleen tältä:

```json
// Request message
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "agent/prompt",
  "params": {
    "prompt": "Help me write a sorting algorithm",
    "sessionId": "session-123"
  }
}

// Response message
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": "Here is the AI response..."
  }
}
```

Todellisessa toteutuksessa tiivistelemme kaikki nämä protokollan yksityiskohdat, joten ylemmän liiketoimintakerroksen tarvitsee vain välittää kehotuksesta ja vastauksesta.

## Ydintoteutus

### 1. Palauta toimittajasopimus

Palauta ensin CodeBuddy-tyyppi enum-tiedostossa:

```csharp
// PCode.Models/AIProviderType.cs
public enum AIProviderType
{
    ClaudeCodeCli = 0,
    CodexCli = 1,
    GitHubCopilot = 2,
    CodebuddyCli = 3,  // Restore this enum value
    OpenCodeCli = 4,
    IFlowCli = 5,
}
```

Lisää sitten merkkijonokuvaus laajennusmenetelmiin, jotta määritystiedosto voi määrittää palveluntarjoajan merkkijonolla:

```csharp
// AIProviderTypeExtensions.cs
private static readonly Dictionary<string, AIProviderType> _typeMap = new(
    StringComparer.OrdinalIgnoreCase)
{
    ["CodebuddyCli"] = AIProviderType.CodebuddyCli,
    ["Codebuddy"] = AIProviderType.CodebuddyCli,
    ["codebuddy"] = AIProviderType.CodebuddyCli,
    // ... Mappings for other providers
};
```

### 2. Integroi Provider Factory

Lisää CodeBuddyn luontihaara tehdasluokkaan:

```csharp
// AIProviderFactory.cs
private IAIProvider? CreateProvider(AIProviderType providerType, ProviderConfiguration config)
{
    return providerType switch
    {
        AIProviderType.CodebuddyCli =>
            ActivatorUtilities.CreateInstance<CodebuddyCliProvider>(
                _serviceProvider,
                Options.Create(config)),
        // ... Other providers
        _ => throw new NotSupportedException($"Provider {providerType} not supported")
    };
}
```

Tämä käyttää riippuvuusinjektiota `ActivatorUtilities`, joka käsittelee automaattisesti konstruktoriparametrien lisäyksen ja on erittäin kätevä.

### 3. Suorita palveluntarjoajan käyttöönotto loppuun

Alla on ydintoteutus `CodebuddyCliProvider`, joka kattaa sekä suoratoiston että ei-suoratoiston kutsutilat:

```csharp
public class CodebuddyCliProvider : IAIProvider
{
    private readonly ILogger<CodebuddyCliProvider> _logger;
    private readonly IACPSessionManager _sessionManager;
    private readonly ProviderConfiguration _config;

    public string Name => "CodebuddyCli";
    public bool SupportsStreaming => true;
    public ProviderCapabilities Capabilities { get; }

    public CodebuddyCliProvider(
        ILogger<CodebuddyCliProvider> logger,
        IACPSessionManager sessionManager,
        IOptions<ProviderConfiguration> config)
    {
        _logger = logger;
        _sessionManager = sessionManager;
        _config = config.Value;

        // Define the capabilities of the current Provider
        Capabilities = new ProviderCapabilities
        {
            SupportsStreaming = true,
            SupportsTools = true,
            SupportsSystemMessages = true,
            SupportsArtifacts = false,
            MaxTokens = 8192
        };
    }

    // Non-streaming call: return all results together after completion
    public async Task<AIResponse> ExecuteAsync(
        AIRequest request,
        CancellationToken cancellationToken = default)
    {
        // Create an independent session for the request
        var session = await _sessionManager.CreateSessionAsync(
            "CodebuddyCli",
            request.WorkingDirectory,
            cancellationToken,
            request.SessionId);

        try
        {
            var fullPrompt = BuildPrompt(request);
            await session.SendPromptAsync(fullPrompt, cancellationToken);

            var responseBuilder = new StringBuilder();
            var toolCalls = new List<AIToolCall>();

            // Collect all response chunks
            await foreach (var chunk in StreamFromSession(session, cancellationToken))
            {
                if (!string.IsNullOrEmpty(chunk.Content))
                {
                    responseBuilder.Append(chunk.Content);
                }
                // Handle tool calls...
            }

            return new AIResponse
            {
                Content = AIResultContentSanitizer.SanitizeResultContent(
                    responseBuilder.ToString()),
                ToolCalls = toolCalls,
                Provider = Name,
                Model = string.Empty
            };
        }
        finally
        {
            // Release session resources
            await session.DisposeAsync();
        }
    }

    // Streaming call: return response chunks in real time
    public async IAsyncEnumerable<AIStreamingChunk> StreamAsync(
        AIRequest request,
        [EnumeratorCancellation] CancellationToken cancellationToken = default)
    {
        var session = await _sessionManager.CreateSessionAsync(
            "CodebuddyCli",
            request.WorkingDirectory,
            cancellationToken);

        try
        {
            var fullPrompt = BuildPrompt(request);
            await session.SendPromptAsync(fullPrompt, cancellationToken);

            await foreach (var chunk in StreamFromSession(session, cancellationToken))
            {
                yield return chunk;
            }
        }
        finally
        {
            await session.DisposeAsync();
        }
    }

    private async IAsyncEnumerable<AIStreamingChunk> StreamFromSession(
        IACPSession session,
        [EnumeratorCancellation] CancellationToken cancellationToken)
    {
        // Iterate through all updates in the session
        await foreach (var notification in session.ReceiveUpdatesAsync(cancellationToken))
        {
            switch (notification.Update)
            {
                case AgentMessageChunkSessionUpdate agentMessage:
                    // Handle text content chunks
                    if (agentMessage.Content is AcpImp.TextContentBlock textContent)
                    {
                        yield return new AIStreamingChunk
                        {
                            Content = textContent.Text,
                            Type = StreamingChunkType.ContentDelta,
                            IsComplete = false
                        };
                    }
                    break;

                case ToolCallSessionUpdate toolCall:
                    // Handle tool calls
                    yield return new AIStreamingChunk
                    {
                        Content = string.Empty,
                        Type = StreamingChunkType.ToolCallDelta,
                        ToolCallDelta = new AIToolCallDelta
                        {
                            Id = toolCall.ToolCallId,
                            Name = toolCall.Kind.ToString(),
                            Arguments = toolCall.RawInput?.ToString()
                        }
                    };
                    break;

                case AcpImp.PromptCompletedSessionUpdate:
                    // Response complete
                    yield break;
            }
        }
    }

    // Build the full prompt
    private string BuildPrompt(AIRequest request, string? embeddedCommandPrompt = null)
    {
        var sb = new StringBuilder();

        // Embedded command prompt, if present
        if (!string.IsNullOrEmpty(embeddedCommandPrompt))
        {
            sb.AppendLine(embeddedCommandPrompt);
            sb.AppendLine();
        }

        // System message
        if (!string.IsNullOrEmpty(request.SystemMessage))
        {
            sb.AppendLine(request.SystemMessage);
            sb.AppendLine();
        }

        // User prompt
        sb.Append(request.Prompt);
        return sb.ToString();
    }
}
```

Tässä koodissa on useita keskeisiä kohtia:

1. **Istunnon hallinta**: jokainen pyyntö luo itsenäisen istunnon ja vapauttaa resurssit pyynnön valmistumisen jälkeen. Tämä on yrityksen ja erehdyksen kautta opittu opetus. Jos istunnon uudelleenkäyttöä ei käsitellä hyvin, tilan saastuminen ilmenee helposti.

2. **Suoratoiston käsittely**: `IAsyncEnumerable` sallii vastauksen palauttamisen sen luomisen aikana sen sijaan, että odotat kaiken sisällön valmistumista. Tämä on erityisen tärkeää pitkän tekstin skenaarioissa ja parantaa merkittävästi käyttökokemusta.

3. **Työkalukutsut**: CodeBuddy tukee työkalukutsuja (Function Calling), joita käsitellään `ToolCallSessionUpdate`. Tämä ominaisuus on kriittinen monimutkaisissa koodinmuokkaustehtävissä.

4. **Sisällön suodatus**: `AIResultContentSanitizer` käytetään suodattamaan Think-lohkon sisältöä ja pitämään tulosteen puhtaana.

### 4. Dependency Injection Configuration

Lisää asiaan liittyvät palvelut moduulin rekisteröinnin yhteydessä:

```csharp
// PCodeClaudeHelperModule.cs
public void ConfigureModule(IServiceCollection context)
{
    // Register Provider
    context.Services.AddTransient<CodebuddyCliProvider>();

    // Register ACP infrastructure
    context.Services.AddSingleton<IACPSessionManager, ACPSessionManager>();
    context.Services.AddSingleton<IAcpPlatformConfigurationResolver, AcpPlatformConfigurationResolver>();
    context.Services.AddSingleton<IAIRequestToAcpMapper, AIRequestToAcpMapper>();
    context.Services.AddSingleton<IAcpToAIResponseMapper, AcpToAIResponseMapper>();
}
```

## Konfigurointiesimerkki

### Asetustiedosto

Lisää CodeBuddyyn liittyvät asetukset `appsettings.json`:

```yaml
AI:
  # Default Provider to use
  DefaultProvider: "CodebuddyCli"

  # Provider configuration
  Providers:
    CodebuddyCli:
      Type: "CodebuddyCli"
      WorkingDirectory: "C:/projects/my-app"
      ExecutablePath: "C:/tools/codebuddy.cmd"

  # Platform-specific configuration
  PlatformConfigurations:
    CodebuddyCli:
      ExecutablePath: "C:/tools/codebuddy.cmd"
      Arguments: "--acp"
      StartupTimeoutMs: 5000
      EnvironmentVariables:
        CODEBUDDY_API_KEY: "${CODEBUDDY_API_KEY}"
        CODEBUDDY_INTERNET_ENVIRONMENT: "production"
```

### Konfigurointimalli

Vastaava kokoonpanomallin määritelmä:

```csharp
public class CodebuddyPlatformConfiguration : IAcpPlatformConfiguration
{
    public string ProviderName => "CodebuddyCli";
    public AcpTransportType TransportType => AcpTransportType.Stdio;

    public string ExecutablePath { get; set; } = "codebuddy";
    public string Arguments { get; set; } = "--acp";
    public int StartupTimeoutMs { get; set; } = 5000;

    public Dictionary<string, string?>? EnvironmentVariables { get; set; }
}
```

## Käytännön oppitunteja

### Sudenkuoppaloki

Törmäsimme useisiin tyypillisiin sudenkuoppiin käyttöönoton aikana, ja niiden jakaminen täällä voi auttaa muita välttämään samoja kiertoteitä:

1. **Istunnon vuotoongelma**: Aluksi istuntoja ei julkaistu oikein, mikä kulutti prosessiresursseja. Ratkaisu oli käyttää `try-finally` varmistaakseen, että resurssit vapautetaan jokaista pyyntöä varten.

2. **Ympäristömuuttujien välitys**: Windows ja Linux käyttävät eri ympäristömuuttujien syntaksia, joten standardisoimme myöhemmin `Dictionary<string, string?>` hoitamaan tätä.

3. **Aikakatkaisumääritys**: CLI-käynnistys vie aikaa, joten asetamme 5 sekunnin käynnistysaikakatkaisun nopeiden pyyntöjen epäonnistumisen välttämiseksi.

4. **Koodausongelmat**: Windowsissa oletuskoodaus voi aiheuttaa sekavaa kiinalaista tekstiä, joten UTF-8-koodaus määritetään erikseen prosessia käynnistettäessä.

### Suorituskyvyn optimointi

1. **Istuntovarasto**: Jos käytät usein lyhyitä pyyntöjä, harkitse istuntovarannon käyttöönottoa prosessien uudelleenkäyttöä varten
2. **Yhteysvälimuisti**: tehdasluokka tukee jo Provider-esiintymien välimuistia
3. **Asynkronointi ensin**: käytä asynkronista ohjelmointia koko ajan välttääksesi säikeiden tukkeutumisen

Suorituskyky kannattaa aina optimoida. Mitä kauemmin käyttäjät odottavat, sitä huonompi kokemus on.
