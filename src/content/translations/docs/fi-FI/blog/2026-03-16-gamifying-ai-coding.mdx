## Kuinka Gamification Design tekee tekoälykoodauksesta hauskempaa

> Perinteiset AI-koodaustyökalut ovat itse asiassa melko tehokkaita; heiltä puuttuu vain vähän lämpöä. Kun rakensimme HagiCodea, ajattelimme: jos aiomme kirjoittaa koodia joka tapauksessa, miksi emme muuttaisi sitä peliksi?

## Tausta

Jokainen tekoälyn koodausassistenttia käyttänyt on todennäköisesti kokenut tämän: aluksi se tuntuu raikkaalta ja jännittävältä, mutta hetken kuluttua alkaa tuntua siltä, että jotain puuttuu. Työkalu itsessään on tehokas, pystyy luomaan koodia, täydentämään automaattisesti ja korjaamaan virheitä, mutta... se ei tunnu kovin lämpimältä, ja ajan myötä se voi muuttua yksitoikkoiseksi ja tylsäksi.

Pelkästään se riittää ihmettelemään, kuka haluaa tuijottaa kylmää, persoonatonta työkalua joka päivä.

Se on vähän kuin peliä. Jos teet vain tehtäväluettelon loppuun ilman hahmojen kasvua, saavutuksia avautumatta tai tiimikoordinaatiota, se lakkaa nopeasti olemasta hauskaa. Kauniita asioita ja ihmisiä ei tarvitse omistaa tullakseen arvostetuiksi; niiden kauneus riittää yksinään. Ohjelmointityökalut eivät edes tarjoa sellaista kauneutta, joten on helppo menettää sydämensä.

Törmäsimme juuri tähän ongelmaan kehittäessämme [HagiCode](https://hagicode.com). Monen tekoälyn avustajan yhteistyöalustana HagiCoden on pidettävä käyttäjät sitoutuneina pitkällä aikavälillä. Mutta todellisuudessa hienoakaan työkalua on vaikea pitää kiinni, jos siitä puuttuu tunne-elämän yhteys.

Tämän tuskakohdan ratkaisemiseksi teimme rohkean päätöksen: muuta ohjelmointi peliksi. Ei pinnallinen tyyppi, jossa on yksinkertainen pistetaulukko, vaan todellinen roolipeli, pelillinen kokemus. Tuon päätöksen vaikutus voi olla jopa suurempi kuin kuvittelet.

Loppujen lopuksi ihmiset tarvitsevat vähän rituaalia elämäänsä.

## Tietoja HagiCodesta

HagiCode on monen tekoälyn avustajan yhteistyöalusta, joka tukee Claude Codea, Codexia, Copilotia, OpenCodea ja muita tekoälyassistentteja yhdessä. Jos olet kiinnostunut multi-AI-yhteistyöstä tai pelillisestä ohjelmoinnista, vieraile [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site) oppiaksesi lisää.

Siinä ei ole mitään erityisen mystistä. Muutimme ohjelmoinnin yksinkertaisesti seikkailuksi.

## Miksi valita Gamification

Pelillistämisen ydin ei ole vain "tulostaulukon lisääminen". Kyse on täydellisen kannustinjärjestelmän rakentamisesta, jotta käyttäjät voivat tuntea kasvua, saavutuksia ja sosiaalista tunnustusta tehdessään tehtäviä.

HagiCoden pelillistämissuunnittelu pyörii yhden ydinidean ympärillä: jokainen tekoälyassistentti on "sankari" ja käyttäjä on tämän sankaritiimin kapteeni. Ohjaat nämä sankarit valloittamaan erilaisia ​​"Dungeoneja" (ohjelmointitehtäviä). Matkan varrella sankarit saavat kokemusta, nousevat tasoa, avaavat kykyjä, ja myös tiimisi ansaitsee saavutuksia.

Tämä ei ole temppu. Se on ihmisen käyttäytymispsykologiaan perustuva malli. Kun tehtäville annetaan merkitys ja edistymisestä palautetta, ihmisten sitoutuminen ja sinnikkyys lisääntyvät merkittävästi.

Kuten vanha sanonta kuuluu: "Tästä tunteesta voi tulla muisto, vaikka se tuolloin hämmensi meidät." Tuomme tämän tunnekokemuksen työkaluun, joten ohjelmointi ei ole enää vain koodin kirjoittamista, vaan matkaa, joka kannattaa muistaa.

## Sankarihahmojärjestelmä

Hero on HagiCoden pelillistämisjärjestelmän ydinkonsepti. Jokainen sankari edustaa yhtä AI-avustajaa. Esimerkiksi Claude Code on sankari ja Codex on myös sankari.

### Kolme sankari-kolikkopeliä

Herossa on kolme varustepaikkaa, ja muotoilu on yllättävän tyylikäs:

1. **CLI-paikka (pääluokka)**: Määrittää sankarin peruskyvyn, kuten onko kyseessä Claude Code vai Codex
2. **Mallipaikka (toissijainen luokka)**: määrittää käytettävän mallin, kuten Claude 4.5 tai Claude 4.6
3. **Tyylipaikka (tyyli)**: Määrittää sankarin käyttäytymistyylin, kuten "Fengluo Strategist" tai muu tyyli

Näiden kolmen paikan yhdistelmä luo ainutlaatuisia Hero-kokoonpanoja. Aivan kuten peleissä rakennetaan laitteita, valitset oikean asennuksen tehtävän perusteella. Loppujen lopuksi tärkeintä on se, mikä sopii sinulle parhaiten. Elämä on samanlaista: monet tiet johtavat Roomaan, mutta jotkut ovat tasaisempia kuin toiset.

### Sankarin kasvujärjestelmä

Jokaisella sankarilla on oma XP ja taso:

```typescript
type HeroProgressionSnapshot = {
  currentLevel: number;                    // Current level
  totalExperience: number;                 // Total experience
  currentLevelStartExperience: number;     // Experience at the start of the current level
  nextLevelExperience: number;             // Experience required for the next level
  experienceProgressPercent: number;       // Progress percentage
  remainingExperienceToNextLevel: number;  // Experience still needed for the next level
  lastExperienceGain: number;              // Most recent experience gained
  lastExperienceGainAtUtc?: string | null; // Time when experience was gained
};
```

Tasot on jaettu neljään vaiheeseen, ja jokaisella vaiheella on mukaansatempaava nimi:

```typescript
export const resolveHeroProgressionStage = (level?: number | null): HeroProgressionStage => {
  const normalizedLevel = Math.max(1, level ?? 1);
  if (normalizedLevel <= 100) return 'rookieSprint';      // Rookie sprint
  if (normalizedLevel <= 300) return 'growthRun';         // Growth run
  if (normalizedLevel <= 700) return 'veteranClimb';      // Veteran climb
  return 'legendMarathon';                                // Legend marathon
};
```

Tämä kasvupolku "aloittelijasta" "legendaksi" antaa käyttäjille selkeän käsityksen suunnasta ja saavutuksista. Se heijastaa henkilökohtaista kasvua elämässä, hämmennyksestä kypsyyteen, vain konkreettisemmaksi täällä.

### Mukautetun sankarin luominen

Heron luomiseksi sinun on määritettävä kolme paikkaa:

```typescript
const heroDraft: HeroDraft = {
  name: 'Athena',
  icon: 'hero-avatar:storm-03',
  description: 'A brilliant strategist',
  executorType: AIProviderType.CLAUDE_CODE_CLI,
  slots: {
    cli: {
      id: 'profession-claude-code',
      parameters: { /* CLI-related parameters */ }
    },
    model: {
      id: 'secondary-claude-4-sonnet',
      parameters: { /* Model-related parameters */ }
    },
    style: {
      id: 'fengluo-strategist',
      parameters: { /* Style-related parameters */ }
    }
  }
};
```

Jokaisella sankarilla on ainutlaatuinen avatar, kuvaus ja ammatillinen identiteetti, mikä antaa muuten kylmälle tekoälyavustajalle persoonallisuutta ja lämpöä. Loppujen lopuksi, kuka haluaa työskennellä työkalulla, jolla ei ole luonnetta?

## Dungeon System

"Dungeon" on klassinen pelikonsepti, joka edustaa haastetta, joka vaatii joukkueen selvittämistä. HagiCodessa jokainen työnkulku on Dungeon.

### Kuinka Dungeons on järjestetty

Dungeon järjestää työnkulkuja erilaisiin Dungeoneihin:

- **Ehdotusten luontityrmä**: Vastaa teknisten ehdotusten luomisesta
- **Ehdotusten toteuttamistyrmä**: Vastaa ehdotuksissa olevien tehtävien suorittamisesta
- **Ehdotusarkiston vankityrmä**: Vastaa valmiiden ehdotusten järjestämisestä ja arkistointista

Jokaisella vankityrmillä on oma kapteenisankarinsa, ja kapteeni valitaan automaattisesti ensimmäiseksi aktivoiduksi sankariksi.

Tämä on oikeastaan vain työnjakoa, kuten jokapäiväisessä elämässä, paitsi että se on muuttunut pelimekaanikoksi.

### Joukkueen yhteistyömekanismi

Voit määrittää erilaisia sankariryhmiä eri luolaille:

```typescript
const dungeonRoster: HeroDungeonRoster = {
  scriptKey: 'proposal.generate',
  displayName: 'Proposal Generation',
  members: [
    { heroId: 'hero-1', name: 'Athena', executorType: 'ClaudeCode' },
    { heroId: 'hero-2', name: 'Apollo', executorType: 'Codex' }
  ]
};
```

Voit esimerkiksi käyttää Athenaa ehdotusten luomiseen, koska se on hyvä strategiassa, ja Apolloa koodin toteuttamiseen, koska se on hyvä suorituskyvyssä. Näin jokainen sankari voi pelata vahvuuksillaan. Se on kuin muodostaisi bändin: jokaisella on instrumentti, ja yhdessä he luovat jotain kaunista.

### Dungeon Flow Control

Dungeon käyttää kiinteitä `scriptKey` arvot eri työnkulkujen tunnistamiseen:

```typescript
// Script keys map to different workflows
const dungeonScripts = {
  'proposal.generate': 'Proposal Generation',
  'proposal.execute': 'Proposal Execution',
  'proposal.archive': 'Proposal Archive'
};
```

Tehtävän tilan kulku on: `queued` (odottaa) -> `dispatching` (määritetään) -> `dispatched` (määrätty). Koko prosessi on automatisoitu eikä vaadi manuaalista toimenpiteitä. Se on myös osa laiskoja puoliamme, koska kukapa haluaisi hallita näitä asioita käsin?

## XP ja Level System

XP on pelillistämisjärjestelmän ydinpalautemekanismi. Käyttäjät saavat XP:tä suorittamalla tehtäviä, XP:t nostavat sankareita ja tasojen nousu avaa uusia kykyjä muodostaen positiivisen palautesilmukan.

### Tapoja saada XP

HagiCodessa XP:tä voi ansaita seuraavilla toimilla:

- Suoritetaan koodia loppuun
- Työkalujen soittaminen onnistui
- Ehdotusten luominen
- Istuntojen hallintatoiminnot
- Projektitoiminta

Joka kerta kun kelvollinen toiminto on suoritettu, vastaava sankari saa XP. Aivan kuten kasvu elämässä, jokainen askel on tärkeä, vain tässä kasvu mitataan määrällisesti.

### Reaaliaikainen edistymisen visualisointi

XP ja tason edistyminen visualisoidaan reaaliajassa:

```typescript
type HeroDungeonMember = {
  heroId: string;
  name: string;
  icon?: string | null;
  executorType: PCode_Models_AIProviderType;
  currentLevel?: number;                    // Current level
  totalExperience?: number;                 // Total experience
  experienceProgressPercent?: number;       // Progress percentage
};
```

Käyttäjät voivat aina nähdä jokaisen sankarin tason ja edistymisen, ja välitön palaute on pelillistämissuunnittelun avain. Ihmiset tarvitsevat palautetta, muuten mistä he tietäisivät kehittyvänsä?

## Saavutusjärjestelmä

Saavutukset ovat toinen tärkeä elementti pelillistämisessä. Ne tarjoavat pitkän aikavälin tavoitteita ja virstanpylväisiin perustuvaa tyytyväisyyttä.

### Saavutustyypit

HagiCode tukee monenlaisia saavutuksia:

- **Koodin luomisen saavutukset**: Luo X riviä koodia, luo Y-tiedostoa
- **Istunnonhallinnan saavutukset**: Suorita Z-keskusteluja
- **Projektitoiminnan saavutukset**: Työskentele W-projektissa

Nämä saavutukset ovat todella kuin virstanpylväitä elämässä, paitsi että olemme tehneet niistä pelimekaanikkoja.

### Saavutustilat

Saavutuksilla on kolme tilaa:

```typescript
type AchievementStatus = 'unlocked' | 'in-progress' | 'locked';
```

Näillä kolmella osalla on selkeät visuaaliset erot:

- **Unlocked**: Kultagradientti haloefektillä
- **Käynnissä**: Sininen pulssianimaatio
- **Lukittu**: Harmaa, lukituksen avausehdot näkyvät

Jokainen saavutus näyttää selvästi sen laukaisutilanteen, joten käyttäjät tietävät, mitä tehdä seuraavaksi. Kun ihmiset tuntevat itsensä eksyksiksi, pieni ohjaus auttaa aina.

### Juhlavaikutus lukituksen avaamiseen

Kun saavutus avataan, juhlaanimaatio käynnistyy. Tällainen positiivinen vahvistus antaa käyttäjille tyydyttävän tunteen "tein sen" ja motivoi heitä jatkamaan. Pienet palkinnot elämässä toimivat samalla tavalla: ne voivat olla pieniä, mutta onnellisuus voi kestää pitkään.

## Taisteluraportti Päivittäinen taisteluraportti

Battle Report on yksi HagiCoden tunnusominaisuuksista. Jokaisen päivän lopussa se luo koko näytön taistelutyylisen raportin.

### Ilmoita sisällöstä

Taisteluraportti näyttää seuraavat tiedot:

```typescript
type HeroBattleReport = {
  reportDate: string;
  summary: {
    totalHeroCount: number;        // Total number of Heroes
    activeHeroCount: number;       // Number of active Heroes
    totalBattleScore: number;      // Total battle score
    mvp: HeroBattleHero;           // Most valuable Hero
  };
  heroes: HeroBattleHero[];        // Detailed data for all Heroes
};
```

- Joukkueen kokonaispisteet
- Aktiivisten sankarien määrä
- Työkalukutsujen määrä
- Kokonaistyöaika
- MVP (arvokkain sankari)
- Yksityiskohtainen kortti jokaiselle sankarille

### MVP Highlight Display

MVP on päivän parhaiten suoriutunut sankari, ja se on korostettu raportissa. Tämä ei ole vain tilastotietoja, vaan eräänlainen kunnia ja tunnustus. Loppujen lopuksi, kuka ei haluaisi tulla tunnustetuksi?

### Yksityiskohtaiset sankarikortit

Jokainen sankarikortti sisältää:

- Tason edistyminen
- XP saatu
- Teloitusten määrä
- Käyttöaika

Nämä tiedot auttavat käyttäjiä ymmärtämään selkeästi, kuinka tiimi toimii. Omien ponnistelujen tulosten näkeminen on sinänsä tyydytystä.

## Tekninen toteutus

HagiCoden pelillistämisjärjestelmä käyttää nykyaikaista teknologiapinoa ja suunnittelukuvioita. Siinä ei ole mitään erityisen maagista; valitsimme vain työkalut, jotka sopivat työhön.

### Teknologiapinon valinnat

```typescript
// React + TypeScript for the frontend
import React from 'react';

// Framer Motion for animations
import { AnimatePresence, motion } from 'framer-motion';

// Redux Toolkit for state management
import { useAppDispatch, useAppSelector } from '@/store';

// shadcn/ui for UI components
import { Dialog, DialogContent } from '@/components/ui/dialog';
```

Framer Motion hoitaa kaikki animaatiotehosteet, shadcn/ui tarjoaa perustavanlaatuiset käyttöliittymäkomponentit ja Redux Toolkit hallitsee monimutkaisen pelillisen tilan. Hyvät työkalut tekevät hyvää työtä.

### Pelillinen käyttöliittymäsuunnittelujärjestelmä

HagiCode käyttää Glassmorphism + Tech Dark -suunnittelutyyliä:

```css
/* Primary gradient */
background: linear-gradient(135deg, #22C55E 0%, #25c2a0 50%, #06b6d4 100%);

/* Glass effect */
backdrop-filter: blur(12px);

/* Glow effect */
background: radial-gradient(circle at center, rgba(34, 197, 94, 0.15) 0%, transparent 70%);
```

Vihreä gradientti yhdistettynä lasimorfismiin luo teknisen, futuristisen tunnelman. Visuaalinen kauneus on myös osa käyttökokemusta.

### Animaatiotehosteet

Framer Motionilla luodaan sulavia sisääntuloanimaatioita:

```typescript
<motion.div
  animate={{ opacity: 1, y: 0 }}
  initial={{ opacity: 0, y: 18 }}
  transition={{ duration: 0.35, ease: 'easeOut', delay: index * 0.08 }}
  className="card"
>
  {/* Card content */}
</motion.div>
```

Jokainen kortti saapuu peräkkäin 0,08 sekunnin viiveellä, mikä luo sujuvan visuaalisen tehosteen. Tasainen animaatio parantaa kokemusta. Tästä osasta on vaikea väitellä.

### Tietojen pysyvyys

Pelillistämistiedot tallennetaan viljavarastojärjestelmään tilan johdonmukaisuuden varmistamiseksi. Jopa hienojakoiset tiedot, kuten kertynyt Hero XP, voidaan säilyttää tarkasti. Kukaan ei halua menettää kokemusta, jonka ansaitsemiseksi on tehnyt kovasti töitä.

## Käytännön opas

### Luo ensimmäinen sankarisi

Ensimmäisen sankarisi luominen on itse asiassa melko yksinkertaista:

1. Siirry Heron hallintasivulle
2. Napsauta "Luo sankari" -painiketta
3. Määritä kolme paikkaa (CLI, malli, tyyli)
4. Anna sankarille nimi ja kuvaus
5. Tallenna se, ja ensimmäinen sankarisi on syntynyt

Se on kuin uuden ystävän tapaaminen: annat heille nimen, opit, mikä tekee heistä erityisen, ja sitten lähdet seikkailuun yhdessä.

### Rakenna Dungeon-tiimi

Ryhmän rakentaminen on myös helppoa:

1. Siirry Dungeon-hallintasivulle
2. Valitse luomasi, jonka haluat määrittää, kuten "Ehdotuksen luominen"
3. Valitse jäsenet sankariluettelostasi
4. Järjestelmä valitsee automaattisesti ensimmäisen käyttöönotetun sankarin kapteeniksi
5. Tallenna asetukset

Tämä on yksinkertaisesti joukkueen muodostamisprosessi, aivan kuten joukkueen rakentaminen tosielämässä, jossa jokaisella on oma roolinsa.

### Katso päiväraportti

Jokaisen päivän lopussa voit tarkastella päivän taisteluraporttia:

1. Napsauta "Taisteluraportti" -painiketta
2. Tarkastele päivän työtuloksia koko näytön näytöllä
3. Tarkista kunkin sankarin MVP ja yksityiskohtaiset tiedot
4. Jaa se tiimin jäsenten kanssa, jos haluat

Tämä on myös eräänlainen rituaali, tapa nähdä kuinka paljon ponnistelet tänään ja kuinka kaukana olet vielä tavoitteestasi.

## Huomautuksia ja parhaita käytäntöjä

### Suorituskyvyn optimointi

Käytä `React.memo` välttääksesi tarpeettomia toistoja:

```typescript
const HeroCard = React.memo(({ hero }: { hero: HeroDungeonMember }) => {
  // Component implementation
});
```

Myös suorituskyvyllä on väliä. Kukaan ei halua käyttää viivästynyttä työkalua.

### Liike voi heikentyä sulavasti

Tunnista käyttäjän liiketoivomusasetukset ja tarjoa yksinkertaisempi käyttökokemus liikeherkille käyttäjille:

```typescript
const prefersReducedMotion = useReducedMotion();
const duration = prefersReducedMotion ? 0 : 0.35;
```

Kaikki eivät pidä animaatioista, ja käyttäjien mieltymysten kunnioittaminen on osa hyvää suunnittelua.

### Taaksepäin yhteensopivuus

Pidä `legacyIds` tukeaksesi siirtymistä vanhemmista versioista:

```typescript
type HeroDungeonMember = {
  heroId: string;
  legacyIds?: string[];  // Supports legacy ID mapping
  // ...
};
```

Kukaan ei halua menettää tietoja vain versiopäivityksen takia.

### Kansainvälistymisen tuki

Käytä i18n-käännösavaimia kaikkeen tekstiin, jotta monikielinen tuki on helppoa:

```typescript
const displayName = t(`dungeon.${scriptKey}`, { defaultValue: displayName });
```

Kieli ei saa koskaan olla este tuotteen käytölle.

## Yhteenveto

Pelillistäminen ei ole vain yksinkertainen pistetaulukko, vaan täydellinen kannustinjärjestelmä. Hero-järjestelmän, Dungeon-järjestelmän, XP- ja tasojärjestelmän, saavutusjärjestelmän ja Battle Reportin avulla HagiCode muuttaa ohjelmointityön sankarilliseksi matkaksi täynnä seikkailuja.

Tämän järjestelmän ydinarvo on:

- **Emotionaalinen yhteys**: Luo kylmälle tekoälyassistentille persoonallisuutta
- **Positiivinen palaute**: Jokainen toiminta tuottaa välitöntä palautetta
- **Pitkän aikavälin tavoitteet**: Tasot ja saavutukset tarjoavat kasvupolun
- **Tiimin identiteetti**: Yhteistyön tunne Dungeon-tiimeissä
- **Kunnia ja tunnustus**: Battle Report ja MVP-esittelyt

Pelimuotoilu ei tee ohjelmoinnista enää tylsää, vaan mielenkiintoista seikkailua. Samalla kun käyttäjät suorittavat koodaustehtäviä, he kokevat myös hahmojen kasvun, tiimiyhteistyön ja saavutusten avaamisen hauskanpitoa, mikä parantaa pysyvyyttä ja aktiivisuutta.

Ohjelmointi on pohjimmiltaan jo luomistoimi. Teimme vain luovasta prosessista hieman hauskempaa.
