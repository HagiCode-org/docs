Tämä sivu on tarkoitettu ensikertalaisille HagiCode-käyttäjille, jotka haluavat lyhimmän OmniRouten asennuspolun: käynnistä paikallinen palvelu, varmista, että `http://localhost:20128/v1` toimii, ja varmista, että sekä Dashboard että API ovat saatavilla.

:::vinkki[Suositeltu polku]
Tämä ensimmäinen versio pitää vakaimman polun ensin: `npm install -g omniroute`, `omniroute`, Kojelauta osoitteessa `http://localhost:20128`ja OpenAI-yhteensopivan perus-URL-osoitteen osoitteessa `http://localhost:20128/v1`. Docker, VPS ja edistyneemmät käyttöönottotilat pysyvät valinnaisessa osiossa.
:::

## Edellytykset

- Node.js on jo asennettu. Täydellinen [Node.js:n asentaminen](/en/related-software-installation/nodejs/installation) ensin tarvittaessa.
- Paikallinen satama `20128` on saatavilla eikä ole jo varattu.
- Sinulla on vähintään yksi ylävirran mallin tarjoajan tili, jonka aiot reitittää OmniRouten kautta.
- Pidä nämä kaksi osoitetta erillään ennen HagiCoden, CLI:n tai SDK:n integroimista:
  - Kojelauta: `http://localhost:20128`
  - OpenAI-yhteensopiva perus-URL-osoite: `http://localhost:20128/v1`

## Asenna ja suorita paikallisesti

Aloita suositellulla globaalilla npm-polulla:

```bash
npm install -g omniroute
omniroute
```

Kun palvelu on käynnistynyt, vahvista se tässä järjestyksessä:

1. Avaa `http://localhost:20128` selaimessa ja varmista, että hallintapaneeli latautuu.
2. Siirry kohtaan `Endpoints` -sivua hallintapaneelissa ja luo tai kopioi API-avain.
3. Huomaa perus-URL-osoite, jonka annat työkaluille myöhemmin: `http://localhost:20128/v1`.

:::huomautus[Miksi aloittaa hallintapaneelista]
Ensimmäisessä asennuksessa Dashboard on nopein tapa varmistaa, että OmniRoute on käynnissä, palveluntarjoajasi ovat yhteydessä ja API-avain on olemassa. Jos myöhempi asiakaskutsu epäonnistuu, hallintapaneelin tarkistaminen ensin on yleensä nopeampaa kuin SDK-koodin virheenkorjaus välittömästi.
:::

## Vahvista käynnissä oleva palvelu

### Vahvistus 1: varmista, että malliluettelo on luettavissa

Laita avain pois `Endpoints` sivu ympäristömuuttujaksi:

```bash
export OMNIROUTE_API_KEY="replace-with-the-key-you-copied-from-the-Endpoints-page"
```

Pyydä sitten `/v1/models`:

```bash
curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}"
```

Jos saat malliluettelon takaisin, OpenAI-yhteensopiva päätepiste ja todennuskulku toimivat.

### Vahvistus 2: lähetä pieni chat-pyyntö

```bash
curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "replace-with-a-model-id-from-v1-models",
    "messages": [
      { "role": "user", "content": "Reply with ok." }
    ]
  }'
```

Tavoitteena ei ole nopea laatu. Se osoittaa, että valitsemasi malli voidaan jo reitittää OmniRouten kautta onnistuneesti.
