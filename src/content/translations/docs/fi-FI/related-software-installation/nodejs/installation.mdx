Tämä opas kattaa Node.js:n asennuksen Windowsiin, macOS:ään ja Linuxiin. Node.js on avainriippuvuus HagiCode paikallisille työkaluille, mukaan lukien OpenSpec, käyttöliittymätyökalut ja useat kehittäjien CLI:t.

:::vinkki[Versiovaatimukset]
HagiCode suosittelee **Node.js 24+** (vähintään tuettu Node.js >= 18.0). Useimmissa tapauksissa asenna uusin LTS-versio ja varmista sitten, että versiosi täyttää vaatimukset.
:::

## Node.js:n asentaminen

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Tapa yksi: Virallinen asentaja (suositus)

1. Vieraile osoitteessa [Node.js virallinen verkkosivusto](https://nodejs.org/)
2. Lataa **LTS (Long Term Support)** -versio Windows-asennusohjelma (`.msi` tiedosto)
3. Suorita asennusohjelma kaksoisnapsauttamalla
4. Suorita asennus loppuun noudattamalla ohjatun asennustoiminnon ohjeita:
   - Hyväksy lisenssisopimus
   - Valitse asennuspolku (oletus on hyvä)
   - Varmista, että "Asenna tarvittavat työkalut automaattisesti" on valittuna
   - Napsauta "Asenna" aloittaaksesi asennuksen
5. Kun asennus on valmis, napsauta "Valmis"

### Tapa kaksi: Käytä siivekettä

Avaa PowerShell tai komentokehote ja suorita:

```powershell
winget install OpenJS.NodeJS.LTS
```

Asennuksen jälkeen avaa pääte uudelleen, jotta muutokset tulevat voimaan.

  </TabItem>
  <TabItem value="macos" label="macOS">

### Tapa yksi: Virallinen asentaja (suositus)

1. Vieraile osoitteessa [Node.js virallinen verkkosivusto](https://nodejs.org/)
2. Lataa **LTS (Long Term Support)** -versio macOS-asennusohjelma (`.pkg` tiedosto)
3. Suorita asennusohjelma kaksoisnapsauttamalla
4. Suorita asennus loppuun noudattamalla ohjatun asennustoiminnon ohjeita:
   - Napsauta "Jatka"
   - Hyväksy lisenssisopimus
   - Valitse asennuslevy
   - Napsauta "Asenna" ja anna järjestelmänvalvojan salasana
5. Kun asennus on valmis, napsauta "Sulje"

### Tapa kaksi: Homebrew'n käyttäminen

Jos sinulla on [Homebrew](https://brew.sh/) asennettuna, voit asentaa Node.js:n päätteen kautta:

```bash
brew install node
```

:::vinkki
Homebrew lisää automaattisesti Node.js:n polkuun. Jos komento ei ole käytettävissä, käynnistä pääte uudelleen tai suorita:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

  </TabItem>
  <TabItem value="ubuntu" label="Ubuntu/Debian">

### NodeSource-arkiston käyttö (suositus)

Tämä on paras tapa hankkia uusin Node.js-versio:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Apt Package Managerin käyttö

Jos et tarvitse uusinta versiota, voit käyttää Ubuntun oletusarkistoa:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::huomautus
Oletusarkiston Node.js-versio saattaa olla vanhempi. Tarkista asennuksen jälkeen versio varmistaaksesi, että se täyttää Node.js >= 18.0 -vaatimukset (Node.js 24+ suositellaan).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### NodeSource-arkiston käyttö (suositus)

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### Yum Package Managerin käyttö

```bash
sudo yum install -y nodejs npm
```

  </TabItem>
  <TabItem value="fedora" label="Fedora">

### dnf Package Managerin käyttö

```bash
sudo dnf install -y nodejs npm
```

:::vinkki
Fedoran ohjelmistovarasto sisältää yleensä uudempia Node.js-versioita ja ne voidaan asentaa suoraan dnf:n kautta.
:::

  </TabItem>

## Versionhallintatyökalujen käyttö

Jos sinun on vaihdettava useiden Node.js-versioiden välillä, voit käyttää versionhallintatyökaluja.

### nvm (solmun versionhallinta)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

Asennuksen jälkeen lataa shell-kokoonpano uudelleen:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

Windowsissa suosittelemme käyttämään [nvm-windows](https://github.com/coreybutler/nvm-windows):

1. Lataa uusin `nvm-setup.exe` alkaen [nvm-windows -julkaisut](https://github.com/coreybutler/nvm-windows/releases)
2. Suorita asennusohjelma
3. Avaa komentokehote tai PowerShell uudelleen

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

Kun olet asentanut nvm:n, voit käyttää sitä Node.js-versioiden asentamiseen ja hallintaan:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (Fast Node Manager)

fnm on nopeampi ja yksinkertaisempi Node.js-versionhallintatyökalu, joka on rakennettu Rustilla.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Windows

Käytä [kauha](https://scoop.sh/) tai [winget](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Kiihdytys Manner-Kiinan käyttäjille

Jos olet Manner-Kiinassa, Node.js- ja npm-pakettien lataaminen voi olla hidasta. Tässä on joitain kiihdytysratkaisuja.

### Kotimaisten peililähteiden käyttö

#### npm Taobao Mirror (suositus)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### nrm:n käyttö peililähteiden hallintaan

nrm (npm-rekisterinhallinta) auttaa sinua vaihtamaan nopeasti npm-peililähteitä:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Node.js-asennusohjelman latauksia nopeutetaan

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao tarjoaa peililatauksia Node.js-asentajille:

- **Latausosoite**: https://npmmirror.com/mirrors/node/

Mene yllä olevaan osoitteeseen ja valitse ladattava versio.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

Jos nvm:n tai muiden työkalujen lataaminen GitHubista on hidasta, voit käyttää FastGit-peiliä:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::huomautus
FastGit-palvelu saattaa muuttua, katso viralliset tiedot.
:::

  </TabItem>
</Tabs>

### Kotimaiset Homebrew-peilit

Jos käytät Homebrew'ta Manner-Kiinassa, voit määrittää kotimaiset peilit kiihdytystä varten:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Manner-Kiinan erityiset pakettipäälliköt

#### cnpm

cnpm on Taobao-tiimin tarjoama npm-asiakas, joka käyttää oletuksena Taobao-peiliä:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::vinkki
cnpm on enimmäkseen yhteensopiva npm-komentojen kanssa ja sitä voidaan käyttää suorana korvaajana.
:::

### Peilin kiihtyvyysvaikutuksen tarkistaminen

Kun olet asentanut peilit, varmista seuraavasti:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Asennuksen tarkistaminen

Varmista asennuksen jälkeen, että Node.js ja npm on asennettu oikein:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::huomautus
Varmista, että Node.js-versio >= 18.0 (Node.js 24+ on suositeltavaa). Jos versio on liian alhainen, asenna uudelleen käyttämällä yllä olevia menetelmiä tai asenna oikea versio versionhallintatyökalujen avulla.
:::

### Näytevahvistustuloste

Onnistuneen vahvistuksen tulosten pitäisi näyttää tältä:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Vianetsintä

### Windows-ongelmat

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Jos `node` tai `npm` komennot eivät ole käytettävissä asennuksen jälkeen:

1. Käynnistä komentokehote tai PowerShell uudelleen
2. Jos se ei vieläkään ole käytettävissä, lisää Node.js manuaalisesti PATH-tiedostoon:
   - Etsi "Ympäristömuuttujat" ja avaa "Muokkaa järjestelmäympäristömuuttujia"
   - Napsauta "Ympäristömuuttujat"
   - Etsi "Polku" kohdassa "Järjestelmämuuttujat" ja napsauta "Muokkaa"
   - Lisää Node.js-asennuspolku (oletus: `C:\Program Files\nodejs\`)
   - Napsauta "OK" tallentaaksesi muutokset

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Jos kohtaat suorituskäytäntövirheitä nvm:n käytön aikana:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### macOS-ongelmat

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Jos kohtaat käyttöoikeusvirheitä globaaleja npm-paketteja asennettaessa:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Jos komentoja ei ole saatavilla asennuksen jälkeen, käynnistä pääte uudelleen tai lisää manuaalisesti PATH:iin (yleensä in `/usr/local/bin`).

  </TabItem>
</Tabs>

### Linux-ongelmat

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Jos kohtaat käyttöoikeusvirheitä globaaleja npm-paketteja asennettaessa, on suositeltavaa käyttää Node.js-versionhallintatyökaluja (kuten nvm tai fnm) järjestelmäpakettien hallintaohjelmien sijaan.

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Jos järjestelmän paketinhallinnan kautta asennettu Node.js-versio on liian vanha:

1. Käytä NodeSource-arkistoa (suositus, katso yllä)
2. Tai käytä versionhallintatyökaluja (nvm/fnm)

  </TabItem>
</Tabs>

### Ympäristön tarkistaminen

Jos vahvistus epäonnistuu, tarkista seuraavat asiat:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
