# Asenna OpenSpec

OpenSpec hallitsee ehdotuksia, määrityksiä ja muutostietueita. Tällä sivulla on yksi tavoite: selitä asennuspolku selkeästi, jotta voit liittää sen olemassa olevaan työnkulkuusi.

## Mikä on OpenSpec?

OpenSpec on komentorivityökalu ehdotusten ja määrittelyjen hallintaan ohjelmistotyössä. Perustyönkulku sisältää:

- **Ehdotusten hallinta**: Luo, tarkista ja seuraa ominaisuusehdotuksia
- **Specification Validation**: Varmista, että koodimuutokset vastaavat ennalta määritettyjä vaatimuksia
- **Muutosseuranta**: Tallenna kaikki tarkistettu muutoshistoria
- **Tiimiyhteistyö**: Tukee monen henkilön yhteistyökehitystä

### Sovellus HagiCode-projekteissa

HagiCode-projekteissa OpenSpeciä käytetään:

1. **Ominaisuuden kehittämisen hallinta**: Kaikkien uusien ominaisuuksien on läpäistävä OpenSpec-ehdotusprosessi
2. **Koodin tarkistus**: Varmista, että muutokset ovat projektin määritysten mukaisia
3. **Dokumentoinnin synkronointi**: Säilytä johdonmukaisuus dokumentaation ja koodin toteutuksen välillä
4. **Versionhallinnan integrointi**: Saumaton integrointi versionhallintatyönkulkuihin

## Edellytykset

Ennen kuin asennat OpenSpecin, varmista, että järjestelmäsi täyttää seuraavat vaatimukset:

### Node.js ja npm

OpenSpec jaetaan npm:n kautta ja vaatii Node.js-ympäristön. OpenSpec vaatii **Node.js 18.0:n tai uudemman**.

Tarkista Node.js:n asennus:

```bash
node --version
```

Tarkista npm-asennus:

```bash
npm --version
```

Jos sitä ei ole asennettu tai versio on liian vanha, katso ohjelmiston asennusohjeet kohdasta [Asennusopas](/en/installation).

## OpenSpecin asentaminen

:::vinkki[Versiovaatimukset]
Tämä projekti edellyttää **OpenSpec 1.0.0 tai uudemman** versioalueen käyttöä. Kaikkia 1.x-versioita tuetaan yhteensopivuuden varmistamiseksi projektin työnkulun kanssa.
:::

OpenSpec asennetaan maailmanlaajuisesti npm:n kautta, joten se on käytettävissä mistä tahansa järjestelmästäsi.

### Asenna maailmanlaajuisesti npm:n avulla

<Tabs>
  <TabItem value="win" label="Windows">
    Suorita PowerShellissä tai komentokehotteessa:

```powershell

```

:::note[Windowsin käyttöoikeusongelmat]
Jos kohtaat käyttöoikeusvirheitä asennuksen aikana, suorita PowerShell tai Command Prompt järjestelmänvalvojana.
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    Suorita terminaalissa:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[macOS-käyttöoikeusongelmat]
Jos kohtaat `EACCES` lupavirheet, kokeile:
```bash
sudo npm install -g @fission-ai/openspec@1
```
Tai käyttää [nvm](https://github.com/nvm-sh/nvm) hallita Node.js-versioita ja välttää käyttöoikeusongelmia.
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    Suorita terminaalissa:

```bash
npm install -g @fission-ai/openspec@1
```

:::note[Linux-käyttöoikeusongelmat]
Jos kohtaat `EACCES` lupavirheet, voit:
1. Käytä `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. Määritä npm käyttämään käyttäjähakemistoa:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### Tarkista asennus

Varmista asennuksen jälkeen, että OpenSpec on asennettu oikein ja että versio täyttää vaatimukset:

```bash
openspec --version
```

Jos asennus onnistui, näet versionumeron, esimerkiksi:

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::huomautus[versioalue]
Varmista, että asennettu OpenSpec-versio on 1.0.0 tai uudempi. Projekti on varmistanut yhteensopivuuden version 1.1.1 kanssa. On suositeltavaa käyttää uusinta yhteensopivaa versiota uusien ominaisuuksien saamiseksi.
:::

## Yleiset ongelmat ja vianetsintä

### Lupavirheet

#### Ongelma: `EACCES` Lupa evätty

Käyttöoikeusvirheitä asennettaessa tai suoritettaessa OpenSpec.

**Ratkaisut**:

<Tabs>
  <TabItem value="win" label="Windows">
    Suorita PowerShell tai komentokehote järjestelmänvalvojana:
    1. Napsauta hiiren kakkospainikkeella PowerShellia tai komentokehotetta
    2. Valitse "Suorita järjestelmänvalvojana"
    3. Suorita asennuskomento uudelleen
  </TabItem>
  <TabItem value="mac" label="macOS">
    Asenna käyttämällä `sudo`:

```bash
sudo npm install -g @fission-ai/openspec@1
```

Tai käyttää [nvm](https://github.com/nvm-sh/nvm) Node.js:n hallintaan ja yleisten asennuslupaongelmien välttämiseen.
  </TabItem>
  <TabItem value="linux" label="Linux">
    Ratkaisu 1: Käytä sudoa
```bash
sudo npm install -g @fission-ai/openspec@1
```

Ratkaisu 2: Määritä npm käyttämään käyttäjähakemistoa
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @fission-ai/openspec@1
```
  </TabItem>
</Tabs>

### Verkko-ongelmat

#### Ongelma: npm Asennus epäonnistuu tai on erittäin hidas

OpenSpecin lataaminen npm-rekisteristä ei onnistu verkkosyistä.

**Ratkaisut**:

1. **Käytä kotimaista npm-peiliä**:

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **Palauta virallinen lähde** (tarvittaessa):

```bash
npm config set registry https://registry.npmjs.org
```

3. **Käytä cnpm** (valinnainen):

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

:::vinkki
Manner-Kiinan käyttäjiä suositellaan käyttämään Taobao-peiliä, joka voi parantaa merkittävästi latausnopeutta.
:::

### Version yhteensopivuusongelmat

#### Ongelma: OpenSpec-komennot toimivat epänormaalisti tai raportoivat virheistä

Asennettu OpenSpec-versio ei ole yhteensopiva nykyisen ympäristön kanssa.

**Ratkaisut**:

1. **Tarkista Node.js-versio**:

```bash
node --version
```

Varmista, että versio on 18.0 tai uudempi. Jos versio on liian alhainen, päivitä Node.js.

2. **Tarkista OpenSpec-versio**:

```bash
openspec --version
```

3. **Asenna tietty versio uudelleen**:

```bash
npm install -g @fission-ai/openspec@1
```

4. **Tyhjennä npm-välimuisti** (jos ongelma jatkuu):

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### Komentoa ei löydy

#### Ongelma: Kirjoittaminen `openspec` Komento näyttää "Komentoa ei löydy"

OpenSpec on asennettu, mutta järjestelmä ei löydä komentoa.

**Ratkaisut**:

<Tabs>
  <TabItem value="win" label="Windows">
    1. Vahvista npm yleinen asennuspolku:

```powershell
npm config get prefix
```

2. Lisää polku järjestelmään PATH:
   - Avaa "Järjestelmän ominaisuudet" → "Lisäasetukset" → "Ympäristömuuttujat"
   - Etsi `Path` kohdassa "Järjestelmämuuttujat"
   - Lisää npm yleinen polku (yleensä `C:\Users\<username>\AppData\Roaming\npm`)
   - Käynnistä komentorivi-ikkuna uudelleen
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. Vahvista npm yleinen asennuspolku:

```bash
npm config get prefix
```

2. Lisää polku kohteeseen PATH (in `~/.zshrc` tai `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Lataa kokoonpano uudelleen:

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. Vahvista npm yleinen asennuspolku:

```bash
npm config get prefix
```

2. Lisää polku kohteeseen PATH (in `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Lataa kokoonpano uudelleen:

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
