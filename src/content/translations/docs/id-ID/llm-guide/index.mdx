## Mengapa panduan LLM

Dalam alur kerja pembuatan kode, performa model berbeda secara signifikan dalam hal kualitas, latensi, dan biaya. Kategori ini memberikan struktur evaluasi yang konsisten untuk membantu pengembang memilih model dengan lebih cepat.

## Jalur membaca yang direkomendasikan

1. Mulailah dengan [Perbandingan dan Evaluasi Model di HagiCode](./model-comparison-evaluation/).
2. Pilih kandidat berdasarkan prioritas Anda (mengutamakan kualitas, mengutamakan biaya, atau mengutamakan kecepatan).
3. Periksa catatan waktu pengujian sebelum menerapkan kesimpulan pada penggunaan produksi.

## Prinsip pemeliharaan

- Segarkan kesimpulan ketika kemampuan model berubah secara signifikan.
- Validasi ulang konteks waktu pengujian setidaknya sebulan sekali.
- Jaga agar halaman berbahasa Mandarin dan Inggris tetap sejajar dalam struktur bagian.
