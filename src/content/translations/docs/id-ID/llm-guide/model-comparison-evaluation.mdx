## Ruang lingkup dan metode

- **Sasaran**: Memberikan panduan pemilihan model berdasarkan pengalaman integrasi nyata di HagiCode.
- **Jenis tugas**: Implementasi komponen frontend, pemfaktoran ulang API backend, penyelesaian pengujian, dan pembuatan dokumentasi.
- **Sumbu evaluasi**: efektivitas penyampaian (dapatkah menyelesaikan tugas nyata dengan andal) dan efektivitas biaya (biaya + ketersediaan domestik).

## Catatan waktu tes dan skenario

- **Tanggal tes terakhir**: 08-03-2026
- **Periode tes**: 01-03-2026 hingga 08-03-2026
- **Dasar sampel**: Evaluasi subyektif dari alur kerja rekayasa HagiCode yang sebenarnya, bukan angka tolok ukur vendor.
- **Penerapan**: Kesimpulan disesuaikan dengan alur kerja dan batasan proyek saat ini.

## Peringkat efektivitas penyampaian (rekomendasi penulis)

Diberi peringkat berdasarkan kualitas penyelesaian tugas dan kualitas praktik teknik:

1. GPT 5.4
2. Kodeks GPT 5.3
3. GL 5
4. GLM 4.7
5. Minimaks M2.5
6. Qwen (3.5 / Kode Berikutnya)

### Temuan utama

- Kecuali Qwen, semua model lain yang diuji dapat mencapai hasil target kami sampai tingkat tertentu.
- GLM 4.7+ (GLM 4.7 dan GLM 5) secara umum melengkapi persyaratan kami dengan lancar.
- GPT 5.3 Codex dan GPT 5.4 tidak hanya melengkapi persyaratan tetapi juga menghasilkan praktik rekayasa dan kualitas implementasi yang lebih baik.
- Minimax M2.5 memiliki kelemahan berulang: kesalahan penutupan kode (misalnya, penutupan braket/blok tidak lengkap), sehingga diperlukan peninjauan ekstra.

## Peringkat efektivitas biaya (biaya + ketersediaan domestik)

Diurutkan berdasarkan biaya ekonomi dan ketersediaan praktis di Tiongkok:

1. Minimaks M2.5
2. GLM 4.7
3. GL 5
4. Qwen 3.5 / Kode Berikutnya
5. Kodeks GPT 5.3
6. GPT 5.4

> Catatan: peringkat ini sengaja dibuat berbeda dengan peringkat efektivitas penyampaian.

## Panduan seleksi

- **Mengutamakan kualitas**: pilih GPT 5.4 / GPT 5.3 Codex.
- **Strategi seimbang**: pilih GLM 5 / GLM 4.7.
- **Biaya lebih dulu**: pilih Minimax M2.5 (dengan pemeriksaan penutupan kode yang lebih ketat).
- **Perutean praktis**: gunakan model premium untuk tugas penting dan model hemat biaya untuk tugas rutin.

## Pernyataan model yang belum teruji

Untuk model yang tidak tercantum di sini, saat ini kami **tidak memiliki data pengujian** dan **tidak memiliki pengalaman langsung**, jadi kami tidak memberikan evaluasi.

Jika sponsor memberikan akses ke model tambahan, kami akan menjalankan evaluasi berbasis pengalaman dalam alur kerja kami yang sebenarnya dan memperbarui halaman ini.
