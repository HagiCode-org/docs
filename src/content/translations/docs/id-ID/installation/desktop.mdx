Panduan ini mencakup instalasi dan penggunaan Hagicode Desktop. Ini adalah klien desktop resmi dan menyimpan jalur pengaturan lokal default ditambah kontrol harian di satu tempat.

:::tip[Metode yang Direkomendasikan]
Desktop adalah jalur lokal default untuk pengguna individu dan pengembang karena:
- penginstal menyimpan aliran default dalam satu jalur terpandu
- dependensi dan layanan dikelola dalam aplikasi
- UI membuat status dan operasi lebih mudah untuk diperiksa
- manajemen versi bawaan membuat peralihan lebih mudah
- data tetap lokal secara default
:::

## Apa itu Hagicode Desktop?

Hagicode Desktop adalah aplikasi desktop resmi Hagicode, dirancang untuk sistem Windows, macOS, dan Linux. Ini menggabungkan fitur-fitur inti Hagicode, dan penginstal menyimpan jalur pengaturan default dalam satu alur yang dipandu.

### Keuntungan Utama

- **Instalasi Sekali Klik**: Selesaikan semua langkah konfigurasi secara otomatis melalui wizard instalasi
- **Persiapan Runtime Otomatis**: Penginstal default sudah menyertakan runtime yang dibutuhkan, sehingga Anda tidak perlu menginstal secara terpisah `.NET` waktu proses terlebih dahulu
- **Manajemen Layanan**: Manajemen Server Hagicode bawaan, memulai dan menghentikan layanan dengan mudah
- **Peralihan Versi**: Dukungan untuk manajemen multi-versi, bebas beralih di antara versi yang berbeda
- **Operasi Lokal**: Semua data disimpan secara lokal, melindungi privasi Anda
- **Pembaruan Otomatis**: Secara otomatis mendeteksi dan mengunduh versi terbaru

### Skenario yang Berlaku

Desktop cocok jika:
- Pengembang perorangan yang ingin menggunakan Hagicode secara lokal
- Pengguna yang perlu menggunakan Hagicode secara offline
- Pengguna non-teknis yang ingin menyederhanakan proses instalasi
- Pengembang yang perlu sering berpindah antar versi berbeda

1. **Halaman Rilis Resmi**: Kunjungi [Rilis Hagicode GitHub](https://github.com/Hagicode-org/releases) untuk mengunduh versi terbaru
2. **Situs Resmi**: Kunjungi [Situs Hagicode](https://hagicode.com/) untuk mendapatkan tautan unduhan

### Pilih Versi yang Benar

Pilih penginstal yang sesuai untuk sistem operasi Anda:
- **Windows**: Unduh `.exe` pemasang
- **macOS**: Unduh `.dmg` pemasang
- **Linux**: Unduh `.AppImage` atau `.tar.gz` paket pemasang

:::catatan[Persyaratan Sistem]
- **Windows**: Windows 10 atau versi lebih tinggi (64-bit)
- **macOS**: macOS 10.15 atau lebih tinggi
- **Linux**: Distribusi Linux arus utama (Ubuntu 20.04+, Fedora 33+, dll.)
:::

:::tip[Prasyarat Bawaan]
Untuk alur instalasi lokal HagiCode terbaru biasanya tidak perlu menyiapkan tersendiri `.NET` waktu proses. Desktop sudah memaketkan runtime yang diperlukan untuk aliran default dan menginisialisasi jalur SQLite lokal yang didukung secara default.
:::

## Catatan Instalasi dan Peluncuran

:::hati-hati[Pemberitahuan Penjaga Gerbang macOS]
Jika Anda sudah menginstal aplikasi ke jalur default `/Applications/Hagicode.app` tetapi macOS menunjukkan "aplikasi rusak", "tidak dapat dibuka", atau peringatan Penjaga Gerbang serupa pada peluncuran pertama, penyebab umum adalah `com.apple.quarantine` atribut yang ditambahkan macOS ke aplikasi yang diunduh dari internet, bukan kegagalan pada penginstal Hagicode Desktop itu sendiri.

Jalankan perintah berikut di Terminal. Ini memerlukan hak istimewa administrator dan secara rekursif menghapus atribut karantina dari aplikasi di jalur instalasi default:

```bash
sudo xattr -dr com.apple.quarantine /Applications/Hagicode.app
```

Setelah perintah selesai, luncurkan kembali Hagicode Desktop.
:::

## Jalankan Desktop

Setelah mengunduh, buka penginstal atau paket aplikasi dan ikuti petunjuk di layar. Desktop menyiapkan runtime di jalur lokal yang saat ini didukung, mengelola Server Hagicode, dan menunjukkan kemajuan dan status ketika pengunduhan jaringan diperlukan.

:::catatan[Persyaratan Jaringan]
Pastikan koneksi jaringan Anda tersedia. Desktop mungkin perlu mengambil informasi versi atau paket komponen dari internet; jika pengunduhan gagal, periksa pengaturan jaringan, proxy, atau firewall Anda dan coba lagi.
:::

## Manajemen Versi

Hagicode Desktop mendukung manajemen multi-versi, memungkinkan Anda bebas beralih antar versi.

![Manajemen Versi](../../img/screenshots/installation/install-desktop/12-version-management/original.png)

### Ganti Versi

1. Buka Desktop Hagicode
2. Buka halaman "Manajemen Versi".
3. Pilih versi yang ingin Anda alihkan
4. Klik tombol "Ganti Versi".
5. Tunggu hingga peralihan selesai

:::catatan[Catatan Peralihan Versi]
- Beralih versi akan secara otomatis memulai ulang Server Hagicode
- Pastikan Anda menyimpan semua pekerjaan sebelum beralih
- Konfigurasi proyek mungkin berbeda antar versi
:::

## Mulai dan Hentikan Layanan

Hagicode Desktop menyediakan cara sederhana untuk mengelola layanan Hagicode Server.

### Mulai Layanan

Di halaman beranda aplikasi Desktop, klik tombol "Mulai Layanan" untuk memulai Server Hagicode.

![Mulai Layanan](../../img/screenshots/installation/install-desktop/13-start-service-homepage/original.png)

### Hentikan Layanan

Untuk menghentikan Server Hagicode, klik tombol "Hentikan Layanan" di halaman beranda.

:::tip[Status Layanan]
Aplikasi Desktop menampilkan status berjalan Hagicode Server secara real-time, antara lain:
- Status layanan (berjalan/berhenti)
- Nomor port layanan
- Informasi versi saat ini
:::

## Pemecahan masalah

### Kegagalan Instalasi

Jika Anda mengalami masalah selama instalasi:

1. **Periksa Koneksi Jaringan**: Pastikan Anda dapat mengakses internet
2. **Periksa Ruang Disk**: Pastikan Anda memiliki ruang disk yang cukup
3. **Jalankan sebagai Administrator**: Di Windows, klik kanan penginstal dan pilih "Jalankan sebagai administrator"
4. **Lihat File Log**: Penginstal menghasilkan file log terperinci yang dapat membantu mendiagnosis masalah

### Kegagalan Instalasi Ketergantungan

Jika instalasi ketergantungan gagal:

1. **Periksa Kompatibilitas Sistem**: Pastikan sistem operasi Anda memenuhi persyaratan minimum
2. **Periksa Pengaturan Jaringan dan Firewall**: Pastikan penginstal dapat mencapai unduhan yang diperlukan dan tidak diblokir oleh firewall Anda
3. **Konfirmasi Izin Penginstal**: Di Windows, coba jalankan penginstal sebagai administrator agar memiliki izin yang memadai
4. **Restart Aplikasi dan Coba Lagi**: Tutup dan buka kembali Hagicode Desktop, lalu coba lagi langkah instalasi
5. **Lihat Log**: Tinjau log kesalahan mendetail di aplikasi Desktop

### Kegagalan Startup Layanan

Jika Server Hagicode gagal memulai:

1. **Periksa Port Occupancy**: Pastikan port 45000 tidak ditempati oleh program lain
2. **Periksa Pengaturan Firewall**: Pastikan firewall mengizinkan akses jaringan Server Hagicode
3. **Mulai Ulang Aplikasi**: Coba mulai ulang Hagicode Desktop
4. **Lihat Log**: Lihat log kesalahan terperinci di aplikasi Desktop
