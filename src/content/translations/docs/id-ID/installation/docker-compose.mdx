:::tip[Metode yang Direkomendasikan]
Docker Compose adalah jalur penerapan default dalam banyak kasus karena:
- isolasi lingkungan mengurangi konflik ketergantungan
- tumpukan layanan Hagicode yang didukung dapat dimulai bersamaan
- konfigurasi, peningkatan, dan pemeliharaan lebih mudah untuk tetap konsisten
- ini berfungsi dengan baik untuk uji coba lokal, pengujian, dan penerapan tim
:::

:::catatan[Tentang Penyimpanan Data Default]
Templat Docker Compose yang saat ini didukung mempertahankan data melalui jalur SQLite lokal default. Dalam kebanyakan kasus, Anda tidak perlu menyediakan layanan database terpisah pada mesin host.
:::

## Prasyarat

Sebelum memulai, pastikan sistem Anda telah menginstal Docker dan Docker Compose.

### Instal Docker

#### jendela

Unduh dan instal [Docker Desktop untuk Windows](https://www.docker.com/products/docker-desktop/)

Setelah instalasi, pastikan Docker Desktop berjalan.

#### macOS

Unduh dan instal [Docker Desktop untuk Mac](https://www.docker.com/products/docker-desktop/)

Setelah instalasi, pastikan Docker Desktop berjalan.

#### Linux

Instal Docker menggunakan manajer paket Anda:

```bash title="Install Docker on Ubuntu/Debian"
sudo apt-get update
sudo apt-get install docker.io docker-compose-plugin
```

Setelah instalasi, mulai layanan Docker:

```bash
sudo systemctl start docker
sudo systemctl enable docker
```

### Verifikasi Instalasi

Setelah instalasi, jalankan perintah berikut untuk memverifikasi Docker dan Docker Compose diinstal dengan benar:

```bash
docker --version
docker compose version
```

## Mulai Cepat

### 1. Hasilkan File Konfigurasi Penulisan Docker

:::tip[Gunakan Generator Konfigurasi]
**Rekomendasikan penggunaan [Docker Compose Builder (situs mandiri)](https://builder.hagicode.com/)** untuk menghasilkan file konfigurasi Anda.

Generatornya mendukung:
- Beberapa opsi penyedia API (Zhipu AI, Anthropic Official, Custom API)
- Konfigurasi otomatis URL API dan Token
- Port khusus, nama kontainer, dan konfigurasi dasar lainnya
- Konfigurasikan data persisten dan jalur direktori kerja
- Konfigurasikan perbedaan platform Windows/Linux secara otomatis
- Tangani masalah izin file dengan cerdas
- **Pemilihan Sumber Gambar**: Berdasarkan lingkungan jaringan Anda, pilih sumber gambar yang sesuai

[🚀 Segera gunakan generatornya →](https://builder.hagicode.com/)
:::

1. Buka [Docker Compose Builder (situs mandiri)](https://builder.hagicode.com/)
2. Isi konfigurasi sesuai kebutuhan Anda:
   - Pilih penyedia API (Zhipu AI, Anthropic Official, atau Custom API)
   - Konfigurasikan port, database, direktori kerja, dan opsi lainnya
   - **Pilih Sumber Gambar**: Pilih sumber gambar yang sesuai berdasarkan lingkungan jaringan Anda
3. Klik tombol hasilkan untuk mendapatkan `docker-compose.yml` konfigurasi
4. Simpan konfigurasi yang dihasilkan sebagai a `docker-compose.yml` mengajukan
5. Jika diperlukan, buatlah a `.env` file untuk mengonfigurasi informasi sensitif

#### Pemilihan Sumber Gambar

Generator mendukung tiga opsi sumber gambar:

**Pusat Pelabuhan**
- **Alamat Gambar**: `newbe36524/hagicode:{tag}`
- **Skenario yang Berlaku**: Cocok untuk pengguna dengan akselerasi mirror Docker Hub
- **Kelebihan**: Sumber gambar resmi, pembaruan tepat waktu, akses stabil
- **Catatan**: Akselerasi cermin mungkin perlu dikonfigurasi di beberapa wilayah

**Registrasi Kontainer Azure (ACR)**
- **Alamat Gambar**: `hagicode.azurecr.io/hagicode:{tag}`
- **Skenario yang Berlaku**: Cocok untuk pengguna yang tidak dapat mengakses Docker Hub secara lokal
- **Kelebihan**: Menyediakan sumber gambar alternatif, menyelesaikan masalah akses jaringan
- **Catatan**: Gambar tetap tersinkronisasi dengan Docker Hub, namun mungkin mengalami penundaan sebentar

**Registrasi Kontainer Cloud Alibaba (ACR)**
- **Alamat Gambar**: `registry.cn-hangzhou.aliyuncs.com/hagicode/hagicode:{tag}`
- **Skenario yang Berlaku**: Cocok untuk pengguna domestik, akses jaringan stabil dan cepat
- **Kelebihan**: Node domestik Alibaba Cloud, kecepatan akses cepat, stabilitas tinggi
- **Catatan**: Gambar Alibaba ACR tetap tersinkronisasi dengan Docker Hub, biasanya dalam waktu 30 menit

:::tip[Rekomendasi Pemilihan Sumber Gambar]
- **Pengguna Domestik Merekomendasikan Alibaba Cloud ACR**: Jika Anda adalah pengguna domestik, disarankan untuk memprioritaskan penggunaan sumber gambar Alibaba Cloud ACR untuk akses cepat dan stabil
- **Hub Docker Alternatif**: Jika lingkungan jaringan Anda dapat mengakses Docker Hub, Anda juga dapat memilih Docker Hub sebagai sumber gambar
- **Azure ACR Alternatif**: Hanya pilih sumber gambar Azure ACR jika sumber gambar lain mengalami kesulitan akses
- **Sinkronisasi Gambar**: Gambar Alibaba Cloud ACR tetap tersinkronisasi dengan Docker Hub, biasanya dalam waktu 30 menit
- **Konsistensi Versi**: Tag gambar konsisten di ketiga sumber gambar, tidak perlu khawatir tentang masalah versi
:::

### 2. Mulai Layanan

Jalankan perintah berikut di direktori tempat Anda `docker-compose.yml` terletak:

```bash
docker compose up -d
```

Perintah ini akan:
- Unduh dan mulai wadah Hagicode yang ditentukan oleh templat yang dihasilkan
- Membuat dan mengkonfigurasi jaringan
- Inisialisasi direktori data default dan dependensi runtime

### 3. Verifikasi Status Layanan

Periksa apakah container sedang berjalan:

```bash
docker compose ps
```

Anda akan melihat layanan yang ditentukan dalam templat yang Anda buat dalam status "berjalan".

### Lihat Log Layanan

Untuk melihat semua log layanan:

```bash
# View all service logs
docker compose logs

# View Hagicode service logs
docker compose logs hagicode

# Real-time log following
docker compose logs -f hagicode
```

## Petunjuk Konfigurasi

Jika Anda perlu mengubah konfigurasi (seperti mengubah port, mengganti penyedia API, dll.), Anda dapat:

1. Buat ulang konfigurasi baru menggunakan [Docker Compose Builder (situs mandiri)](https://builder.hagicode.com/)
2. Edit secara manual `docker-compose.yml` dan `.env` file
3. Mulai ulang layanan agar konfigurasi diterapkan:

```bash
docker compose restart
```

## Akses Aplikasi

### Antarmuka Web

Setelah startup layanan berhasil, akses melalui browser Anda:

```
http://localhost:45000
```

### Persistensi Data

Jalur Docker Compose yang didukung menggunakan file data SQLite dengan volume persisten secara default. Jika Anda perlu memeriksa titik pemasangan atau merencanakan pencadangan, tinjau pemetaan volume dan direktori data yang dihasilkan `docker-compose.yml`.
