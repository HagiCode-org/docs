# Instal OpenSpec

OpenSpec mengelola proposal, spesifikasi, dan catatan perubahan. Halaman ini memiliki satu tujuan: menjelaskan jalur instalasi dengan jelas sehingga Anda dapat melampirkannya ke alur kerja yang ada.

## Apa itu OpenSpec?

OpenSpec adalah alat baris perintah untuk mengelola proposal dan spesifikasi dalam pekerjaan perangkat lunak. Alur kerja inti meliputi:

- **Manajemen Proposal**: Membuat, meninjau, dan melacak proposal fitur
- **Validasi Spesifikasi**: Pastikan perubahan kode sesuai dengan spesifikasi yang telah ditentukan
- **Pelacakan Perubahan**: Catat semua riwayat perubahan yang ditinjau
- **Kolaborasi Tim**: Mendukung pengembangan kolaboratif multi-orang

### Aplikasi dalam Proyek HagiCode

Dalam proyek HagiCode, OpenSpec digunakan untuk:

1. **Manajemen Pengembangan Fitur**: Semua fitur baru harus melalui proses proposal OpenSpec
2. **Tinjauan Kode**: Pastikan perubahan mematuhi spesifikasi proyek
3. **Sinkronisasi Dokumentasi**: Menjaga konsistensi antara dokumentasi dan implementasi kode
4. **Integrasi Kontrol Versi**: Integrasi yang lancar dengan alur kerja kontrol versi

## Prasyarat

Sebelum menginstal OpenSpec, pastikan sistem Anda memenuhi persyaratan berikut:

### Node.js dan npm

OpenSpec didistribusikan melalui npm dan memerlukan lingkungan Node.js. OpenSpec memerlukan **Node.js 18.0 atau lebih tinggi**.

Verifikasi instalasi Node.js:

```bash
node --version
```

Verifikasi instalasi npm:

```bash
npm --version
```

Jika belum diinstal atau versinya terlalu lama, silakan merujuk ke petunjuk instalasi perangkat lunak terkait di [Panduan Instalasi](/en/installation).

## Menginstal OpenSpec

:::tip[Persyaratan Versi]
Proyek ini memerlukan penggunaan rentang versi **OpenSpec 1.0.0 atau lebih tinggi**. Semua versi 1.x didukung untuk memastikan kompatibilitas dengan alur kerja proyek.
:::

OpenSpec diinstal secara global melalui npm, membuatnya tersedia dari mana saja di sistem Anda.

### Instal Secara Global Menggunakan npm

<Tabs>
  <TabItem value="win" label="Windows">
    Jalankan di PowerShell atau Command Prompt:

```powershell
npm install -g @fission-ai/openspec@1
```

:::catatan[Masalah Izin Windows]
Jika Anda mengalami kesalahan izin selama instalasi, jalankan PowerShell atau Command Prompt sebagai administrator.
:::
  </TabItem>
  <TabItem value="mac" label="macOS">
    Jalankan di Terminal:

```bash
npm install -g @fission-ai/openspec@1
```

:::catatan[Masalah Izin macOS]
Jika Anda menemui `EACCES` kesalahan izin, coba:
```bash
sudo npm install -g @fission-ai/openspec@1
```
Atau gunakan [nvm](https://github.com/nvm-sh/nvm) untuk mengelola versi Node.js dan menghindari masalah izin.
:::
  </TabItem>
  <TabItem value="linux" label="Linux">
    Jalankan di Terminal:

```bash
npm install -g @fission-ai/openspec@1
```

:::catatan[Masalah Izin Linux]
Jika Anda menemui `EACCES` kesalahan izin, Anda dapat:
1. Gunakan `sudo`: `sudo npm install -g @fission-ai/openspec@1`
2. Konfigurasikan npm untuk menggunakan direktori pengguna:
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```
:::
  </TabItem>
</Tabs>

### Verifikasi Instalasi

Setelah instalasi, verifikasi bahwa OpenSpec diinstal dengan benar dan versinya memenuhi persyaratan:

```bash
openspec --version
```

Jika berhasil diinstal, Anda akan melihat keluaran nomor versi, misalnya:

```
openspec/1.1.1 linux-x64 node-v18.17.0
```

:::catatan[Rentang Versi]
Harap pastikan versi OpenSpec yang diinstal adalah 1.0.0 atau lebih tinggi. Proyek ini telah memverifikasi kompatibilitas dengan versi 1.1.1. Disarankan untuk menggunakan versi terbaru yang kompatibel untuk mendapatkan fitur baru.
:::

## Masalah Umum dan Pemecahan Masalah

### Kesalahan Izin

#### Masalah: `EACCES` Izin Ditolak

Mengalami kesalahan izin saat menginstal atau menjalankan OpenSpec.

**Solusi**:

<Tabs>
  <TabItem value="win" label="Windows">
    Jalankan PowerShell atau Command Prompt sebagai administrator:
    1. Klik kanan PowerShell atau Command Prompt
    2. Pilih "Jalankan sebagai administrator"
    3. Jalankan kembali perintah instalasi
  </TabItem>
  <TabItem value="mac" label="macOS">
    Instal menggunakan `sudo`:

```bash
sudo npm install -g @fission-ai/openspec@1
```

Atau gunakan [nvm](https://github.com/nvm-sh/nvm) untuk mengelola Node.js dan menghindari masalah izin instalasi global.
  </TabItem>
  <TabItem value="linux" label="Linux">
    Solusi 1: Gunakan sudo
```bash
sudo npm install -g @fission-ai/openspec@1
```

Solusi 2: Konfigurasikan npm untuk menggunakan direktori pengguna
```bash
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g @fission-ai/openspec@1
```
  </TabItem>
</Tabs>

### Masalah Jaringan

#### Masalah: Instalasi npm Gagal atau Sangat Lambat

Tidak dapat mengunduh OpenSpec dari registri npm karena alasan jaringan.

**Solusi**:

1. **Gunakan Cermin npm Domestik**:

```bash
# Use Taobao mirror
npm config set registry https://registry.npmmirror.com

# Then reinstall
npm install -g @fission-ai/openspec@1
```

2. **Kembalikan Sumber Resmi** (jika diperlukan):

```bash
npm config set registry https://registry.npmjs.org
```

3. **Gunakan cnpm** (opsional):

```bash
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @fission-ai/openspec@1
```

:::tip
Pengguna di Tiongkok daratan disarankan untuk menggunakan mirror Taobao, yang dapat meningkatkan kecepatan pengunduhan secara signifikan.
:::

### Masalah Kompatibilitas Versi

#### Masalah: Perintah OpenSpec Berjalan Tidak Normal atau Laporkan Kesalahan

Versi OpenSpec yang diinstal tidak kompatibel dengan lingkungan saat ini.

**Solusi**:

1. **Periksa Versi Node.js**:

```bash
node --version
```

Pastikan versinya 18.0 atau lebih tinggi. Jika versinya terlalu rendah, harap tingkatkan Node.js.

2. **Periksa Versi OpenSpec**:

```bash
openspec --version
```

3. **Instal Ulang Versi Tertentu**:

```bash
npm install -g @fission-ai/openspec@1
```

4. **Hapus npm Cache** (jika masalah terus berlanjut):

```bash
npm cache clean --force
npm install -g @fission-ai/openspec@1
```

### Perintah Tidak Ditemukan

#### Masalah: Mengetik `openspec` Perintah Menampilkan "Perintah Tidak Ditemukan"

OpenSpec diinstal tetapi sistem tidak dapat menemukan perintahnya.

**Solusi**:

<Tabs>
  <TabItem value="win" label="Windows">
    1. Konfirmasikan jalur instalasi global npm:

```powershell
npm config get prefix
```

2. Tambahkan jalur ke PATH sistem:
   - Buka "System Properties" → "Advanced" → "Variabel Lingkungan"
   - Temukan `Path` dalam "Variabel sistem"
   - Tambahkan jalur global npm (biasanya `C:\Users\<username>\AppData\Roaming\npm`)
   - Mulai ulang jendela baris perintah
  </TabItem>
  <TabItem value="mac" label="macOS">
    1. Konfirmasikan jalur instalasi global npm:

```bash
npm config get prefix
```

2. Tambahkan jalur ke PATH (in `~/.zshrc` atau `~/.bash_profile`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Muat ulang konfigurasi:

```bash
source ~/.zshrc
```
  </TabItem>
  <TabItem value="linux" label="Linux">
    1. Konfirmasikan jalur instalasi global npm:

```bash
npm config get prefix
```

2. Tambahkan jalur ke PATH (in `~/.bashrc`):

```bash
export PATH="$(npm config get prefix)/bin:$PATH"
```

3. Muat ulang konfigurasi:

```bash
source ~/.bashrc
```
  </TabItem>
</Tabs>
