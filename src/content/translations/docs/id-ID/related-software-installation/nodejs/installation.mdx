Panduan ini mencakup instalasi Node.js di Windows, macOS, dan Linux. Node.js adalah ketergantungan utama untuk perkakas lokal HagiCode, termasuk OpenSpec, perkakas frontend, dan beberapa CLI pengembang.

:::tip[Persyaratan Versi]
HagiCode merekomendasikan **Node.js 24+** (minimum Node.js yang didukung >= 18.0). Biasanya, instal versi LTS terbaru, lalu verifikasi bahwa versi Anda memenuhi persyaratan.
:::

## Menginstal Node.js

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Metode Satu: Penginstal Resmi (Disarankan)

1. Kunjungi [Situs Resmi Node.js](https://nodejs.org/)
2. Unduh penginstal Windows versi **LTS (Dukungan Jangka Panjang)** (`.msi` mengajukan)
3. Klik dua kali untuk menjalankan penginstal
4. Ikuti petunjuk wizard instalasi untuk menyelesaikan instalasi:
   - Terima perjanjian lisensi
   - Pilih jalur instalasi (standarnya baik-baik saja)
   - Pastikan "Instal alat yang diperlukan secara otomatis" dicentang
   - Klik "Instal" untuk memulai instalasi
5. Setelah instalasi selesai, klik "Selesai"

### Metode Kedua: Menggunakan winget

Buka PowerShell atau Command Prompt dan jalankan:

```powershell
winget install OpenJS.NodeJS.LTS
```

Setelah instalasi, buka kembali terminal agar perubahan diterapkan.

  </TabItem>
  <TabItem value="macos" label="macOS">

### Metode Satu: Penginstal Resmi (Disarankan)

1. Kunjungi [Situs Resmi Node.js](https://nodejs.org/)
2. Unduh penginstal macOS versi **LTS (Dukungan Jangka Panjang)** (`.pkg` mengajukan)
3. Klik dua kali untuk menjalankan penginstal
4. Ikuti petunjuk wizard instalasi untuk menyelesaikan instalasi:
   - Klik "Lanjutkan"
   - Terima perjanjian lisensi
   - Pilih disk instalasi
   - Klik "Instal" dan masukkan kata sandi administrator
5. Setelah instalasi selesai, klik "Tutup"

### Metode Kedua: Menggunakan Homebrew

Jika Anda punya [minuman rumahan](https://brew.sh/) terinstal, Anda dapat menginstal Node.js melalui terminal:

```bash
brew install node
```

:::tip
Homebrew secara otomatis akan menambahkan Node.js ke PATH Anda. Jika perintah tidak tersedia, restart terminal atau jalankan:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

  </TabItem>
  <TabItem value="ubuntu" label="Ubuntu/Debian">

### Menggunakan Repositori NodeSource (Disarankan)

Ini adalah cara terbaik untuk mendapatkan versi Node.js terbaru:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Menggunakan Manajer Paket yang tepat

Jika Anda tidak memerlukan versi terbaru, Anda dapat menggunakan repositori default Ubuntu:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::catatan
Versi Node.js di repositori default mungkin lebih lama. Setelah instalasi, periksa versi untuk memastikannya memenuhi persyaratan Node.js >= 18.0 (disarankan Node.js 24+).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### Menggunakan Repositori NodeSource (Disarankan)

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### Menggunakan Manajer Paket yum

```bash
sudo yum install -y nodejs npm
```

  </TabItem>
  <TabItem value="fedora" label="Fedora">

### Menggunakan Manajer Paket dnf

```bash
sudo dnf install -y nodejs npm
```

:::tip
Repositori perangkat lunak Fedora biasanya berisi versi Node.js yang lebih baru dan dapat diinstal langsung melalui dnf.
:::

  </TabItem>
</Tabs>

## Menggunakan Alat Manajemen Versi

Jika Anda perlu beralih di antara beberapa versi Node.js, Anda dapat menggunakan alat manajemen versi.

### nvm (Manajer Versi Node)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

Setelah instalasi, muat ulang konfigurasi shell Anda:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### jendela

Di Windows, kami merekomendasikan penggunaan [nvm-windows](https://github.com/coreybutler/nvm-windows):

1. Unduh yang terbaru `nvm-setup.exe` dari [rilis nvm-windows](https://github.com/coreybutler/nvm-windows/releases)
2. Jalankan penginstal
3. Buka kembali Command Prompt atau PowerShell

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

Setelah menginstal nvm, Anda dapat menggunakannya untuk menginstal dan mengelola versi Node.js:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (Manajer Node Cepat)

fnm adalah alat manajemen versi Node.js yang lebih cepat dan sederhana yang dibuat dengan Rust.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### jendela

Gunakan [sendok](https://scoop.sh/) atau [pemain sayap](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Percepatan untuk Pengguna Tiongkok Daratan

Jika Anda berada di Tiongkok Daratan, pengunduhan paket Node.js dan npm mungkin lambat. Berikut beberapa solusi percepatan.

### Menggunakan Sumber Cermin Domestik

#### npm Taobao Mirror (Disarankan)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### Menggunakan nrm untuk Mengelola Sumber Cermin

nrm (npm registry manager) membantu Anda dengan cepat mengganti sumber mirror npm:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Mempercepat Pengunduhan Penginstal Node.js

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao menyediakan unduhan cermin untuk penginstal Node.js:

- **Alamat Unduh**: https://npmmirror.com/mirrors/node/

Kunjungi alamat di atas dan pilih versi yang ingin Anda unduh.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

Jika pengunduhan nvm atau alat lain dari GitHub lambat, Anda dapat menggunakan mirror FastGit:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::catatan
Layanan FastGit dapat berubah, silakan merujuk ke informasi resmi.
:::

  </TabItem>
</Tabs>

### Cermin Homebrew Domestik

Jika Anda menggunakan Homebrew di Tiongkok Daratan, Anda dapat mengonfigurasi mirror domestik untuk akselerasi:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Manajer Paket Khusus Tiongkok Daratan

#### cnpm

cnpm adalah klien npm yang disediakan oleh tim Taobao yang menggunakan mirror Taobao secara default:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::tip
cnpm sebagian besar kompatibel dengan perintah npm dan dapat digunakan sebagai pengganti langsung.
:::

### Memverifikasi Efek Akselerasi Cermin

Setelah menyiapkan cermin, verifikasi dengan:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Memverifikasi Instalasi

Setelah instalasi, verifikasi bahwa Node.js dan npm telah diinstal dengan benar:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::catatan
Pastikan Node.js versi >= 18.0 (disarankan Node.js 24+). Jika versinya terlalu rendah, harap instal ulang menggunakan metode di atas atau gunakan alat manajemen versi untuk menginstal versi yang benar.
:::

### Contoh Hasil Verifikasi

Hasil verifikasi yang berhasil akan terlihat seperti:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Pemecahan masalah

### Masalah Windows

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Jika `node` atau `npm` perintah tidak tersedia setelah instalasi:

1. Mulai ulang Command Prompt atau PowerShell
2. Jika masih tidak tersedia, tambahkan Node.js secara manual ke PATH:
   - Cari "Variabel Lingkungan" dan buka "Edit variabel lingkungan sistem"
   - Klik "Variabel Lingkungan"
   - Temukan "Jalur" di "Variabel sistem" dan klik "Edit"
   - Tambahkan jalur instalasi Node.js (default: `C:\Program Files\nodejs\`)
   - Klik "OK" untuk menyimpan perubahan

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Jika Anda mengalami kesalahan kebijakan eksekusi saat menggunakan nvm:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### Masalah macOS

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Jika Anda mengalami kesalahan izin saat menginstal paket npm global:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Jika perintah tidak tersedia setelah instalasi, restart terminal atau tambahkan secara manual ke PATH (biasanya di `/usr/local/bin`).

  </TabItem>
</Tabs>

### Masalah Linux

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Jika Anda mengalami kesalahan izin saat menginstal paket npm global, disarankan untuk menggunakan alat manajemen versi Node.js (seperti nvm atau fnm) daripada manajer paket sistem.

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Jika versi Node.js yang diinstal melalui manajer paket sistem terlalu lama:

1. Gunakan repositori NodeSource (disarankan, lihat di atas)
2. Atau gunakan alat manajemen versi (nvm/fnm)

  </TabItem>
</Tabs>

### Memeriksa Lingkungan

Jika verifikasi gagal, periksa hal berikut:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
