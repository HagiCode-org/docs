Halaman ini ditujukan untuk pengguna HagiCode pertama kali yang menginginkan jalur penyiapan OmniRoute terpendek: mulai layanan lokal, verifikasi bahwa `http://localhost:20128/v1` berfungsi, dan konfirmasikan bahwa Dasbor dan API tersedia.

:::tip[Jalur yang disarankan]
Versi pertama ini mempertahankan jalur paling stabil terlebih dahulu: `npm install -g omniroute`, `omniroute`, Dasbor di `http://localhost:20128`, dan URL Basis yang kompatibel dengan OpenAI di `http://localhost:20128/v1`. Docker, VPS, dan mode penerapan lanjutan lainnya tetap berada di bagian opsional.
:::

## Prasyarat

- Node.js sudah terinstal. Lengkap [Menginstal Node.js](/en/related-software-installation/nodejs/installation) terlebih dahulu jika diperlukan.
- Pelabuhan lokal `20128` tersedia dan belum ditempati.
- Anda memiliki setidaknya satu akun penyedia model upstream yang Anda rencanakan untuk dirutekan melalui OmniRoute.
- Sebelum mengintegrasikan HagiCode, CLI, atau SDK, pisahkan kedua alamat berikut:
  - Dasbor: `http://localhost:20128`
  - URL Dasar yang kompatibel dengan OpenAI: `http://localhost:20128/v1`

## Instal dan jalankan secara lokal

Mulailah dengan jalur npm global yang direkomendasikan:

```bash
npm install -g omniroute
omniroute
```

Setelah layanan dimulai, validasi dalam urutan ini:

1. Buka `http://localhost:20128` di browser dan konfirmasikan bahwa Dasbor dimuat.
2. Pergi ke `Endpoints` halaman di Dasbor dan buat atau salin kunci API.
3. Catat URL Dasar yang akan Anda berikan ke alat nanti: `http://localhost:20128/v1`.

:::catatan[Mengapa memulai dengan Dasbor]
Untuk penyiapan pertama, Dasbor adalah cara tercepat untuk mengonfirmasi bahwa OmniRoute berjalan, penyedia Anda terhubung, dan kunci API ada. Jika panggilan klien berikutnya gagal, memeriksa Dasbor terlebih dahulu biasanya lebih cepat daripada langsung men-debug kode SDK.
:::

## Validasi layanan yang berjalan

### Validasi 1: konfirmasikan katalog model dapat dibaca

Masukkan kunci dari `Endpoints` halaman ke dalam variabel lingkungan:

```bash
export OMNIROUTE_API_KEY="replace-with-the-key-you-copied-from-the-Endpoints-page"
```

Lalu minta `/v1/models`:

```bash
curl http://localhost:20128/v1/models \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}"
```

Jika Anda mendapatkan kembali daftar model, titik akhir dan alur autentikasi yang kompatibel dengan OpenAI berfungsi.

### Validasi 2: kirim permintaan obrolan minimal

```bash
curl http://localhost:20128/v1/chat/completions \
  -H "Authorization: Bearer ${OMNIROUTE_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "replace-with-a-model-id-from-v1-models",
    "messages": [
      { "role": "user", "content": "Reply with ok." }
    ]
  }'
```

Sasarannya di sini bukanlah kualitas yang cepat. Ini untuk membuktikan bahwa model pilihan Anda sudah berhasil dirutekan melalui OmniRoute.
