## Ikhtisar

Monospecs adalah pendekatan manajemen multi-repositori HagiCode. Ini melacak dan mengelola sub-repositori melalui `monospecs.yaml`, dan ini berfungsi untuk proyek apa pun yang memerlukan satu tempat untuk mengoordinasikan beberapa repositori independen.

### Konsep Inti

**File Konfigurasi**

`monospecs.yaml` adalah inti dari keseluruhan solusi manajemen, yang mendefinisikan:
- Daftar dan lokasi sub-repositori
- URL Git untuk setiap repositori
- Nama tampilan dan ikon
- Perilaku integrasi dengan OpenSpec

**Perbedaan dari Repositori Reguler**

| Fitur | Repositori Reguler | Proyek Terkelola Monospec |
|----------|-------------------|---------------------------|
| **Lokasi Spesifikasi Terbuka** | Direktori root `openspec/` | Direktori root repositori utama `openspec/` |
| **Ubah Cakupan** | Terbatas pada repositori saat ini | Dapat melibatkan banyak sub-repositori |
| **Kemurnian Sub-repositori** | Spesifikasi dicampur dengan kode | Spesifikasi dibuat di repositori utama, sub-repositori lebih bersih |
| **Arsip Komit Otomatis** | Membutuhkan penerapan spesifikasi manual | `commit_when_archive: true` secara otomatis memasukkan spesifikasi ke repositori utama |

### Mengapa Menggunakan Monospec?

- **Manajemen Terpadu**: Kelola semua sub-repositori dalam satu file konfigurasi
- **Dukungan Otomatisasi**: Menyediakan metadata repositori untuk skrip dan alat
- **Keramahan AI**: Konfigurasi terstruktur membantu Agen AI memahami hubungan repositori
- **Integrasi OpenSpec**: Integrasi yang lancar dengan alur kerja manajemen perubahan
- **Sub-repositori yang Lebih Bersih**: Spesifikasi disimpan di repo utama, sub-repositori hanya berisi kode aktual

### Skenario yang Berlaku

Gunakan Monospecs terlebih dahulu dalam skenario seperti ini:

**1. Pengembangan Kolaboratif Multi-Repositori**
Ketika sebuah proyek dipecah menjadi beberapa repositori independen:
- Aplikasi frontend, layanan backend, dan aplikasi desktop dikembangkan secara terpisah
- Perlu mengoordinasikan versi dan dependensi di seluruh repositori
- Dokumentasi dan proyek pembantu terpisah dari kode utama

**2. Manajemen Proyek Tambahan**
Gunakan Monospecs untuk mengelola:
- Proyek dokumentasi
- Situs web resmi
- Bangun alat dan skrip
- Setiap repositori memiliki kontrol versinya sendiri

**3. Pengembangan Fitur Lintas Repositori**
Sebuah fitur yang memerlukan perubahan di beberapa repositori:
- Perlu melacak perubahan di repositori mana
- Secara otomatis menerapkan spesifikasi ke repositori target yang benar
- Mengkoordinasikan pembaruan versi dan dependensi

## Mulai Cepat

### Inisialisasi

Ikuti langkah-langkah berikut untuk menyelesaikan inisialisasi monospec:

#### 1. Buat Repositori git Kosong

Buat yang kosong `git` folder di repositori utama Anda (catatan: ini adalah `git`, tidak `repos/` - ini adalah konvensi monospesifikasi):

```bash
mkdir git
cd git
git init
cd ..
```

#### 2. Konfigurasikan File monospesifikasi

Buat `monospecs.yaml` file konfigurasi di direktori root repositori utama Anda:

```yaml
version: "1.0"
commit_when_archive: true

repositories:
  - path: "repos/frontend"
    url: "https://github.com/your-org/frontend.git"
    displayName: "Frontend App"
    icon: "🌐"

  - path: "repos/backend"
    url: "https://github.com/your-org/backend.git"
    displayName: "Backend Service"
    icon: "⚙️"
```

**Item Konfigurasi**:
| Barang Konfigurasi | Ketik | Diperlukan | Deskripsi |
|----------------|------|----------|-------------|
| `version` | tali | Ya | Versi file konfigurasi |
| `commit_when_archive` | boolean | Tidak | Apakah akan menerapkan spesifikasi secara otomatis saat pengarsipan (jangan menerapkan kode sub-repo) |
| `repositories` | susunan | Ya | Daftar konfigurasi repositori |
| `repositories[].path` | tali | Ya | Jalur repositori (relatif terhadap root proyek) |
| `repositories[].url` | tali | Ya | URL repositori Git |
| `repositories[].displayName` | tali | Tidak | Nama tampilan repositori |
| `repositories[].icon` | tali | Tidak | Ikon repositori (emoji) |

### Tambahkan Sub-Repositori ke Repositori Utama

Tambahkan semua sub-repositori ke `repos/` direktori. Direktori ini harus **dikecualikan dari kontrol versi Git repositori utama** untuk menghindari pelacakan konten sub-repositori:

```plaintext
# Exclude all sub-repositories
repos/
```

#### 3. Tambahkan Repositori Induk ke HagiCode

Jika Anda menggunakan aplikasi desktop HagiCode, buka aplikasi dan tambahkan repositori utama Anda sebagai sebuah proyek. Klik tombol "Tambah Proyek" atau "+" di layar beranda, pilih folder repositori utama Anda, dan Anda akan melihat semua sub-repositori dan informasi statusnya.

#### 4. Sub-Repositori Kloning

Gunakan `monospecs.yaml` konfigurasi untuk mengkloning semua sub-repositori yang dikonfigurasi ke `repos/` direktori:

Ini akan mengkloning semua sub-repositori berdasarkan URL yang ditentukan dalam file konfigurasi.

## Struktur File Konfigurasi

### Konfigurasi Tingkat Root

```yaml
version: "1.0"
commit_when_archive: true

repositories:
  # ... repository configurations
```

### Konfigurasi Tingkat Repositori

Setiap sub-repositori dapat memiliki:
- Folder perubahan OpenSpec (`openspec/`)
- Mandiri `.git/` direktori
- Opsional `AGENTS.md` untuk konfigurasi Agen AI

## Operasi Manajemen Repositori

### Tambahkan Repositori Baru

Saat menambahkan sub-repositori baru ke `monospecs.yaml` `repositories` susunan:

```yaml
repositories:
  - path: "repos/new-service"
    url: "https://github.com/HagiCode-org/new-service.git"
    displayName: "New Service"
    icon: "🆕"
```

### Perbarui Konfigurasi Repositori

Saat URL repositori atau metadata berubah:
1. Sunting `monospecs.yaml` untuk memperbarui entri terkait
2. Verifikasi sintaks YAML sudah benar
3. Jika sinkronisasi berubah, perbarui konfigurasi repositori lokal secara manual

## Dukungan Konfigurasi Agen AI

### AGEN.md

Itu `AGENTS.md` File ini memberikan panduan khusus repositori untuk Agen AI:

**Informasi Tumpukan Teknis**
- Kerangka kerja dan alat pembuatan
- Konvensi kode dan praktik penamaan
- Manajemen versi dan konfigurasi penerapan

**Perilaku Khusus Proyek**
- Ekstensi konfigurasi dan persyaratan khusus
- Pola integrasi OpenSpec
- Kebutuhan koordinasi lintas repositori

### Manfaat AI

Menggunakan `AGENTS.md` memungkinkan Agen AI untuk:
- Memahami hubungan dan ketergantungan repositori
- Hasilkan kode dan refaktorisasi di beberapa repositori
- Pertahankan konsistensi dalam kontrol versi dan konfigurasi

## Integrasi OpenSpec

### Bagaimana Monospec Bekerja dengan OpenSpec

Baik mengelola proyek terkelola monospec atau repositori reguler, Anda dapat menggunakan alur kerja OpenSpec HagiCode:

**Repositori Reguler:**
- OpenSpec di root `openspec/` direktori
- Perubahan hanya mempengaruhi repositori saat ini

**Proyek Terkelola Monospec:**
- OpenSpec berbunyi `monospecs.yaml` konfigurasi
- Perubahan dilacak berdasarkan sub-repositori
- Spesifikasi secara otomatis berkomitmen untuk target yang benar

### Perilaku Arsip

Kapan `commit_when_archive: true`:
- Spesifikasi secara otomatis dikomit ke repositori utama
- Kode sub-repositori tidak dikomit (tetap terpisah)
- Menyederhanakan manajemen spesifikasi - tidak diperlukan penerapan spesifikasi manual untuk sub-repositori

### Komitmen Sub-Repositori Manual

Sub-repositori mempertahankan kontrol versi independen:
- Melakukan perubahan kode aktual ke repositori mereka sendiri
- Spesifikasi dikomit secara terpisah (dari repositori utama saat pengarsipan)

## Praktik Terbaik

### Konvensi Penamaan Repositori

- Gunakan huruf kebab (huruf kecil dan tanda hubung) untuk jalur repositori
- Gunakan nama tampilan yang sederhana dan deskriptif dalam bahasa Mandarin
- Gunakan emoji yang relevan sebagai ikon

### Pedoman Nama Tampilan

- Jaga agar tetap ringkas dan deskriptif
- Gunakan penamaan yang konsisten di seluruh repositori serupa
- Pilih ikon yang mewakili tujuan repositori

### Pemeliharaan Konfigurasi

**Sinkronisasi Konfigurasi dengan Status Aktual**
- Verifikasi secara teratur `monospecs.yaml` mencerminkan struktur repositori sebenarnya
- Perbarui atau hapus entri ketika repositori ditambahkan atau dihapus
- Uji perubahan untuk memastikan validitas konfigurasi

**Kontrol Versi untuk Konfigurasi**
- Lacak perubahan file konfigurasi dengan git
- Dokumentasikan alasan perubahan konfigurasi besar
- Tandai versi file konfigurasi untuk pengembalian yang mudah

## Pemecahan masalah

### File Konfigurasi Tidak Ditemukan

Jika `monospecs.yaml` file tidak ditemukan di root proyek:
1. Periksa apakah file tersebut ada di lokasi yang benar
2. Pastikan Anda berada di direktori kerja yang benar
3. Periksa kesalahan ketik pada nama file konfigurasi

### Kesalahan Sintaks YAML

Masalah umum sintaksis YAML:
- Indentasi salah (gunakan spasi, bukan tab)
- Kutipan hilang atau tambahan di sekitar string
- Tipe data tidak valid (string memerlukan tanda kutip, angka tidak)
- Bidang yang wajib diisi tidak ada (seperti `path` atau `url`)

### Repositori Tidak Terdeteksi

Jika repositori yang baru ditambahkan tidak muncul:
1. Periksa `path` benar di `monospecs.yaml`
2. Verifikasikan repositori berhasil dikloning
3. Periksa apakah repositori ada di lokasi yang diharapkan

### Kegagalan Klon

Jika kloning repositori gagal:
1. Periksa koneksi jaringan
2. Verifikasi URL repositori Git sudah benar
3. Periksa masalah autentikasi
4. Tinjau status kontainer Docker
5. Periksa ketersediaan ruang disk
