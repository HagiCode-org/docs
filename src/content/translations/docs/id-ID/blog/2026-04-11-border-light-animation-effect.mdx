Pada artikel ini, saya akan memandu Anda tentang cara membuat animasi sapuan cahaya batas dari awal, dan juga berbagi beberapa kendala yang kami temui saat membuat HagiCode.

## Ide Implementasi Inti

Setelah menganalisis basis kode HagiCode, kami merangkum beberapa pola implementasi inti di bawah ini. Masing-masing sesuai dengan skenario yang berbeda, atau dengan kata lain, masing-masing ada karena suatu alasan.

### 1. Cahaya berputar dengan gradien berbentuk kerucut (paling umum)

Ini adalah cara klasik untuk menerapkan efek sapuan cahaya batas. Ide intinya adalah menggunakan CSS `conic-gradient` untuk membuat gradien berbentuk kerucut, lalu putar terus menerus. Seperti lampu jalan yang menyala di malam hari, ia terus berputar.

**Elemen kunci:**
- Gunakan `::before` elemen semu untuk membuat lapisan cahaya
- Gunakan `conic-gradient` untuk menentukan distribusi warna gradien
- Gunakan `::after` elemen semu untuk menutupi area tengah (opsional)
- Gunakan `@keyframes` untuk mengimplementasikan animasi rotasi

### 2. Garis cahaya samping

Ini berfungsi dengan baik untuk indikator status dalam item daftar. Anda membuat garis tipis bercahaya di satu sisi elemen alih-alih menganimasikan seluruh batas. Terkadang sedikit cahaya saja sudah cukup. Anda tidak perlu menerangi seluruh dunia.

**Elemen kunci:**
- Elemen garis tipis yang diposisikan secara mutlak
- Gunakan `box-shadow` untuk menciptakan efek cahaya
- Gunakan `scale` dan `opacity` untuk animasi pernapasan

### 3. Latar belakang bersinar dengan bayangan kotak

Jika Anda tidak memerlukan efek sapuan penuh dan hanya menginginkan cahaya latar belakang yang lembut, buatlah beberapa lapisan `box-shadow` nilai sudah cukup. Terkadang pilihan yang lebih sederhana adalah pilihan yang lebih baik.

### 4. Dukungan aksesibilitas

Bagian ini mudah untuk diabaikan, tetapi ini sangat penting. Setiap animasi harus memperhitungkan `prefers-reduced-motion` media query dan memberikan alternatif statis bagi pengguna yang tidak menginginkan animasi. Tidak semua orang menyukai gerakan yang konstan, dan menghargai bahwa preferensi itu penting.

## Opsi Implementasi

### Opsi 1: Memutar batas gradien berbentuk kerucut (disarankan)

Ini adalah implementasi efek cahaya batas yang paling lengkap, dan juga merupakan opsi yang paling sering kami gunakan di HagiCode. Lagi pula, jika sesuatu berfungsi dengan baik, mengapa harus diganti?

```css
/* Parent container */
.glow-border-container {
  position: relative;
  overflow: hidden;
}

/* Rotating glow layer */
.glow-border-container::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    transparent 0deg,
    rgba(59, 130, 246, 0.6) 60deg,
    rgba(59, 130, 246, 0.3) 120deg,
    rgba(59, 130, 246, 0.6) 180deg,
    transparent 240deg
  );
  animation: border-rotate 3s linear infinite;
  z-index: -1;
}

/* Mask layer (optional, for creating a hollow border effect) */
.glow-border-container::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

@keyframes border-rotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Prinsip di balik opsi ini cukup sederhana: buat elemen semu yang lebih besar dari wadah induknya, gambarkan gradien berbentuk kerucut di atasnya, dan putar terus menerus. Wadah induk menggunakan `overflow: hidden`, jadi hanya cahaya yang melewati perbatasan saja yang tetap terlihat. Ini seperti melihat lampu jalan melalui jendela. Anda hanya melihat potongan kecil yang lewat.

### Opsi 2: Batas cahaya berputar yang disederhanakan

Jika Anda tidak memerlukan efek penuh, HagiCode juga menyertakan versi kelas utilitas yang lebih ringan. Terkadang pendekatan yang lebih sederhana memang lebih baik.

```css
/* Rotating light border utility class */
.running-light-border {
  position: absolute;
  inset: -2px;
  background: conic-gradient(
    from 0deg,
    transparent 0deg 270deg,
    var(--theme-running-color) 270deg 360deg
  );
  border-radius: inherit;
  animation: lightRayRotate 3s linear infinite;
  will-change: transform;
  z-index: 0;
}

@keyframes lightRayRotate {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}

/* Accessibility support */
@media (prefers-reduced-motion: reduce) {
  .running-light-border {
    animation: none;
  }
}
```

Perhatikan `will-change: transform` di sini. Ini memberi tahu browser, "Elemen ini akan terus berubah," sehingga browser dapat menyiapkan beberapa pengoptimalan terlebih dahulu dan menjaga animasi tetap lancar. Mempersiapkan terlebih dahulu biasanya lebih baik daripada mengacak-acak di menit-menit terakhir.

### Opsi 3: Garis cahaya samping

Ini sangat cocok untuk indikator status item daftar, dan itulah yang digunakan oleh daftar sesi HagiCode. Satu garis tipis masih bisa menonjol di antara banyak objek. Itu terasa seperti pelajaran hidup dengan caranya sendiri.

```css
.side-glow {
  position: relative;
  isolation: isolate;
}

.side-glow::before {
  content: '';
  position: absolute;
  left: 0;
  top: 14px;
  bottom: 14px;
  width: 1px;
  border-radius: 999px;
  background: var(--theme-running-color);
  box-shadow:
    0 0 16px var(--theme-running-color),
    0 0 28px var(--theme-running-color);
  z-index: 1;
  pointer-events: none;
  animation: sidePulse 2.6s ease-in-out infinite;
}

.side-glow > * {
  position: relative;
  z-index: 2;
}

@keyframes sidePulse {
  0%, 100% {
    opacity: 0.55;
    transform: scaleY(0.96);
  }
  50% {
    opacity: 0.95;
    transform: scaleY(1);
  }
}
```

Ini menggunakan `isolation: isolate` untuk membuat konteks penumpukan baru, lalu mengandalkan `z-index` untuk mengontrol urutan tampilan setiap lapisan. `pointer-events: none` juga penting. Jika tidak, elemen semu akan memblokir klik pengguna. Beberapa hal mungkin terlihat bagus, namun tetap tidak boleh menghalangi.

### Opsi 4: Bungkus dalam komponen React

Jika proyek Anda menggunakan React, Anda dapat menggabungkan logika ini dalam sebuah komponen, terutama penanganan aksesibilitas. Tulis sekali, gunakan berkali-kali. Itulah intinya.

```tsx
import React from 'react';
import { useReducedMotion } from 'framer-motion';
import styles from './GlowBorder.module.css';

interface GlowBorderProps {
  isActive: boolean;
  children: React.ReactNode;
  className?: string;
}

export const GlowBorder = React.memo<GlowBorderProps>(
  ({ isActive, children, className = '' }) => {
    const prefersReducedMotion = useReducedMotion();

    if (!isActive) {
      return <div className={className}>{children}</div>;
    }

    if (prefersReducedMotion) {
      return (
        <div className={`${styles.glowStatic} ${className}`}>
          {children}
        </div>
      );
    }

    return (
      <div className={`${styles.glowAnimated} ${className}`}>
        {children}
      </div>
    );
  }
);
```

Modul CSS yang cocok:

```css
/* GlowBorder.module.css */

/* Animated version */
.glowAnimated {
  position: relative;
  overflow: hidden;
}

.glowAnimated::before {
  content: '';
  position: absolute;
  top: -50%;
  left: -50%;
  width: 200%;
  height: 200%;
  background: conic-gradient(
    from 0deg,
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent,
    rgba(59, 130, 246, 0.6),
    transparent
  );
  animation: rotateGlow 3s linear infinite;
  z-index: -1;
}

.glowAnimated::after {
  content: '';
  position: absolute;
  inset: 2px;
  background: inherit;
  border-radius: inherit;
  z-index: -1;
}

/* Static version (accessibility) */
.glowStatic {
  position: relative;
  border: 1px solid rgba(59, 130, 246, 0.5);
  box-shadow: 0 0 15px rgba(59, 130, 246, 0.3);
}

@keyframes rotateGlow {
  from {
    transform: rotate(0deg);
  }
  to {
    transform: rotate(360deg);
  }
}
```

Itu `useReducedMotion` kait dari `framer-motion` secara otomatis mendeteksi preferensi sistem pengguna. Jika pengguna telah mengaktifkan pengurangan gerakan, ia akan kembali `true`, dan komponen malah menampilkan versi statis. Menghargai preferensi pengguna lebih penting daripada memaksakan efek mencolok.

## Pelajaran Praktis yang Dipetik

Ini adalah beberapa pelajaran yang kami pelajari saat membuat HagiCode. Anda juga bisa menyebutnya bekas luka pertempuran. Semoga mereka membantu Anda menghindari jalan memutar.

### 1. Sistem variabel tema

Variabel CSS membuat dukungan multi-tema menjadi sangat nyaman. Tidak ada seorang pun yang ingin mengedit tumpukan kode setiap kali tema berubah.

```css
:root {
  --glow-color-light: rgb(16, 185, 129);
  --glow-color-dark: rgb(16, 185, 129);
  --theme-glow-color: var(--glow-color-light);
}

html.dark {
  --theme-glow-color: var(--glow-color-dark);
}

/* Usage */
.glow-effect {
  background: var(--theme-glow-color);
  box-shadow: 0 0 20px var(--theme-glow-color);
}
```

Dengan begitu, peralihan tema hanya memerlukan perubahan kelas di dalamnya `html` elemen, dan setiap warna animasi diperbarui secara otomatis. Satu basis kode, dua gaya. Itulah yang kami inginkan.

### 2. Optimalisasi kinerja

**Gunakan `will-change` untuk memberi petunjuk kepada browser untuk mengoptimalkan:**

```css
.animated-glow {
  will-change: transform, opacity;
}
```

Beri tahu browser terlebih dahulu, dan itu akan membantu Anda mengoptimalkan. Banyak hal dalam hidup yang berjalan lebih baik dengan sedikit persiapan.

**Hindari penggunaan bayangan kotak yang rumit pada elemen besar:**

```css
/* Not ideal - using a blurred shadow on a large element */
.large-card {
  box-shadow: 0 0 50px rgba(0, 0, 0, 0.5);
}

/* Better - use a pseudo-element to limit the glowing area */
.large-card::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  box-shadow: 0 0 20px var(--glow-color);
  pointer-events: none;
}
```

Kami mengujinya di HagiCode. Menambahkan bayangan buram langsung ke kartu besar menurunkan kecepatan bingkai gulir di bawah 30fps. Beralih ke elemen semu mengembalikan segalanya ke 60fps stabil. Pengguna benar-benar dapat merasakan perbedaan tersebut.

### 3. Aksesibilitas

Anda sebaiknya tidak melewatkan ini. Beberapa pengguna menganggap animasi memusingkan atau mengganggu, dan menghormati preferensi mereka adalah bagian dari pembuatan produk yang bagus. Hal-hal indah tidak perlu dipaksakan pada semua orang.

**Permintaan media CSS:**

```css
@media (prefers-reduced-motion: reduce) {
  .glow-animation {
    animation: none;
  }

  .glow-animation::before {
    /* Provide a static fallback */
    opacity: 1;
  }
}
```

**Deteksi preferensi pengguna di React:**

```tsx
import { useReducedMotion } from 'framer-motion';

const Component = () => {
  const prefersReducedMotion = useReducedMotion();

  return (
    <div className={prefersReducedMotion ? 'static-glow' : 'animated-glow'}>
      Content
    </div>
  );
};
```

### 4. Kontrol tingkat intensitas

Indikator throughput Token di HagiCode menunjukkan warna cahaya yang berbeda berdasarkan throughput real-time, dan ini diterapkan secara dinamis. Negara bagian yang berbeda harus diungkapkan secara berbeda.

```tsx
const colors = [
  null,       // Level 0 - no color
  '#3b82f6',  // Level 1 - Blue
  '#34d399',  // Level 2 - Emerald
  '#facc15',  // Level 3 - Yellow
  '#fbbf24',  // Level 4 - Amber
  '#f97316',  // Level 5 - Orange
  '#22d3ee',  // Level 6 - Cyan
  '#d946ef',  // Level 7 - Fuchsia
  '#f43f5e',  // Level 8 - Rose
];

const IntensityGlow = ({ intensity }) => {
  const glowColor = colors[Math.min(intensity, colors.length - 1)];

  return (
    <div
      className="glow-effect"
      style={{
        '--glow-color': glowColor,
        opacity: 0.6 + (intensity * 0.08),
      }}
    />
  );
};
```

### 5. Hal-hal yang perlu diperhatikan

Masih ada beberapa detail yang perlu diperhatikan, karena saat Anda menemukan masalah ini dengan susah payah, semuanya sudah terlambat.

| Hal-hal yang harus diperhatikan | Penjelasan |
|---------|------|
| **manajemen indeks-z** | Lapisan cahaya harus menggunakan yang sesuai `z-index` sehingga tidak mengganggu interaksi konten |
| **acara penunjuk** | Elemen semu cahaya harus diatur `pointer-events: none` |
| **Meluapnya batas** | Kebutuhan wadah induk `overflow: hidden`, atau Anda perlu menyesuaikan ukuran elemen semu |
| **Dampak kinerja** | Animasi yang rumit dapat mengganggu kinerja perangkat seluler, jadi ujilah dengan cermat |
| **Mode gelap** | Pastikan warna cahaya tetap terlihat jelas pada latar belakang gelap |
| **Peralihan tema** | Gunakan variabel CSS agar warna animasi diperbarui dengan benar saat tema berubah |

### 6. Tip men-debug

Elemen semu mungkin agak sulit ditemukan di alat pengembang, jadi Anda dapat menambahkan batas sementara untuk memeriksa posisinya.

```css
/* Temporarily show pseudo-element boundaries for debugging */
.glow-effect::before {
  /* debug: border: 1px solid red; */
}
```

Setelah Anda selesai memposisikannya, ingatlah untuk memberi komentar atau menghapus baris tersebut. Jika tidak, produksi akan menjadi canggung dengan cepat. Beberapa hal sebaiknya dibiarkan dalam pengembangan.
