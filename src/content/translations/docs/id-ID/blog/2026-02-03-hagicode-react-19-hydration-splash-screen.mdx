## Merancang 12 Pengalaman Startup Luar Biasa untuk HagiCode: Dari Minimalis hingga Cyberpunk

> Kesenjangan singkat antara mengunduh aplikasi React 19 dan menyelesaikan Hidrasi adalah peluang emas bagi pengguna untuk merasakan kepribadian merek Anda. Pada artikel ini, kami membagikan sistem gaya startup lengkap yang kami buat untuk proyek HagiCode menggunakan HTML/CSS/JS.

## Latar Belakang

Sebagai aplikasi modern yang dibangun dengan ASP.NET Core 10 dan React 19 (Vite), HagiCode menggunakan arsitektur penerapan terpisah frontend-backend. Output build frontend dikemas ke dalam backend `wwwroot/` direktori dan dihosting oleh ASP.NET Core.

Namun, arsitektur ini menimbulkan masalah klasik UX: ketika pengguna mengunjungi halaman, browser terlebih dahulu memuat HTML, kemudian mengunduh bundel JS yang besar, dan akhirnya membiarkan React melakukan Hidrasi. Selama "periode vakum" yang berlangsung dari beberapa ratus milidetik hingga beberapa detik, pengguna melihat layar kosong atau halaman statis tak bernyawa.

Untuk mengisi kesenjangan tersebut dan memasukkan kepribadian merek HagiCode, kami perlu merancang sistem gaya startup yang diterapkan seluruhnya dengan kode inline di dalamnya. `index.html`.

## Tentang HagiCode

Pendekatan desain layar splash yang dibagikan dalam artikel ini berasal dari pengalaman praktis kami di [Kode Hagi](https://github.com/HagiCode-org/site) proyek. Sebagai asisten pengkodean AI, HagiCode tidak hanya peduli pada efisiensi pembuatan kode, tetapi juga pengalaman visual pengembang. Sistem startup ini adalah salah satu hasil dari upaya kami mencapai kinerja frontend terbaik.

## Tantangan Inti dan Desain Arsitektur

Sebelum kami mulai mendesain, pertama-tama kami harus memperjelas batasan teknisnya. Karena semuanya harus dilaksanakan secara inline `index.html`, kami tidak dapat memuat file CSS atau JS eksternal apa pun selain bundel React sendiri.

### Analisis Kendala Teknis

1.  **Prinsip ketergantungan nol**: Semua gaya harus ada di dalam a `<style>` tag, dan semua logika harus ada di dalam a `<script>` menandai.
2.  **CSS Defensif**: Untuk mencegah gaya global mencemari layar splash setelah aplikasi React dipasang, kami memutuskan untuk menggabungkan semua gaya startup dengan awalan ID prioritas tinggi seperti `#boot-screen`.
3.  **Performa pertama**: Animasi harus menggunakan CSS `transform` dan `opacity` sedapat mungkin untuk menghindari perubahan posisi dan memastikan thread utama tetap tidak diblokir.
4.  **Konsistensi visual**: Warna dan font harus tetap selaras dengan konfigurasi Tailwind HagiCode.

### Pola Arsitektur: Shell & Injector

Kami mengadopsi **pola varian**. Logika inti dienkapsulasi di dalam ekspresi fungsi yang segera dipanggil (IIFE), sedangkan logika rendering spesifik dimasukkan melalui konfigurasi. Hal ini memungkinkan kita beralih di antara gaya yang berbeda melalui konfigurasi sederhana alih-alih menulis ulang logika manipulasi DOM berulang kali.

Berikut adalah kode arsitektur inti:

```html
<!-- Inline in index.html -->
<div id="boot-root"></div>

<script>
(function() {
  const BootSequence = {
    config: {
      theme: 'terminal', // Configurable as 'minimal', 'skeleton', 'code-rain', etc.
      color: '#3b82f6'   // Brand color
    },
    
    // Core lifecycle
    init() {
      this.render();
      this.listenForMount();
    },

    // Render the currently selected style
    render() {
      const root = document.getElementById('boot-root');
      if (this.variants[this.config.theme]) {
        root.innerHTML = this.variants[this.config.theme].render();
      }
    },

    // Listen for successful React mount and exit gracefully
    listenForMount() {
      window.addEventListener('hagicode:ready', () => {
        const screen = document.getElementById('boot-root');
        // Fade out first, then remove the DOM to avoid flicker
        screen.style.opacity = '0';
        screen.style.transition = 'opacity 0.3s ease';
        setTimeout(() => screen.remove(), 300);
      });
    },

    // The implementation logic for all 12 styles lives here
    variants: {
      // ...see details below
    }
  };

  BootSequence.init();
})();
</script>
```

## Daftar Periksa 12 Desain Gaya Startup

Kami mengelompokkan 12 gaya ini ke dalam enam kategori utama untuk memenuhi berbagai skenario dan preferensi estetika.

### A. Minimalis

> "Lebih sedikit lebih baik." Untuk skenario yang mengejar kecepatan pemuatan tertinggi, kami menyediakan opsi seringan mungkin.

#### 1. Titik Minimalis
Sebuah titik sederhana berada di tengah layar, dipasangkan dengan animasi pernapasan.
*   **Implementasi**: CSS `@keyframes` mengontrol skala dan opacity.
*   **Terbaik untuk**: Dalam kasus apa pun halaman harus tetap benar-benar bersih.

#### 2. Pengungkapan Merek
Menggunakan SVG `stroke-dasharray` animasi, ini menyimulasikan tampilan garis logo HagiCode yang digambar tangan, diikuti dengan teks yang memudar.
*   **Teknik**: Animasi jalur SVG dengan nuansa yang sangat halus.

### B. Mimikri Layar Kerangka

> “Seni menipu mata.” Dengan mensimulasikan tata letak UI yang sebenarnya, pengguna merasa halamannya sudah setengah dimuat.

#### 3. Kerangka Obrolan Sidebar
Ini mungkin pilihan yang paling praktis. Kami secara manual membuat tata letak dalam HTML yang mencerminkan React `Sidebar` dan `ChatInput` komponen dengan tepat, lalu dilapis dengan animasi kilau abu-abu.
*   **Nilai**: Ketika hidrasi React selesai, kerangka langsung menjadi komponen sebenarnya, dan pengguna hampir tidak dapat merasakan peralihannya.

#### 4. Kerangka Tumpukan Kartu
Ini mensimulasikan gerakan tumpukan kartu proposal saat memuat, menggunakan transformasi 3D untuk membuat kartu melayang secara halus.

### C. Abstraksi dan Seni

> Pamerkan DNA geek HagiCode.

#### 5. Morf Geometris
Bentuk geometris (persegi) ditampilkan di tengah layar, lalu perlahan-lahan berubah menjadi lingkaran, segitiga, dan akhirnya logo.
*   **Teknologi**: Transisi mulus dengan CSS `border-radius`.

#### 6. Kode Hujan
Sebuah penghargaan untuk *The Matrix*. Menggunakan font JetBrains Mono, aliran karakter samar-samar muncul di latar belakang.
*   **Catatan**: Untuk performa, aliran karakter harus berada dalam area yang lebih kecil atau menggunakan kecepatan refresh yang lebih rendah.

#### 7. Pulsa Neon
Cincin bercahaya gaya cyberpunk yang menggunakan banyak `box-shadow` lapisan untuk menciptakan cahaya neon yang kuat.

### D. Branding dan Tema

> Membuat sistem terasa hidup.

#### 8. Tema Musiman
Ini adalah pemuat dinamis. Ia memeriksa tanggal hari libur saat ini seperti Tahun Baru Imlek atau Natal dan memuat animasi SVG yang sesuai.
*   **Contoh**: Selama Tahun Baru Imlek, lentera merah bergoyang lembut di bagian bawah layar.

#### 9. Aliran Gradien
Latar belakang menggunakan gradien cair berdasarkan warna merek HagiCode. Dikombinasikan dengan animasi `background-size` dan `background-position`, ini menciptakan sensasi gerak seperti aurora.

### E. Perasaan Teknis

> Salut untuk pengembang.

#### 10. Booting Terminal
Ini mensimulasikan keluaran konsol. Baris kode bergulir dengan cepat:
```text
> Initializing HagiCode Core...
> Loading models...
> Connecting to neural network...
```
Itu langsung terasa familier bagi setiap pengembang.

#### 11. Bilah Kemajuan
Bilah kemajuan tipis muncul di bagian atas layar, dengan persentase ditampilkan di sebelah kanan. Meskipun kami tidak dapat mengakses kemajuan pengunduhan yang sebenarnya, kami dapat menggunakan pengatur waktu untuk menyimulasikan proses pemuatan yang "dapat dipercaya": cepat untuk 80% pertama, kemudian secara bertahap lebih lambat untuk 20% terakhir.

### F.Kreativitas

#### 12. Perakitan Piksel
Ini adalah ide yang sangat menarik. Kotak-kotak kecil tersebar di seluruh layar, kemudian menyatu ke tengah dan secara bertahap berkumpul menjadi ikon logo HagiCode. Ini melambangkan proses membangun kode.

## Praktik Terbaik dan Kesalahan

Dalam pekerjaan pengembangan HagiCode yang sebenarnya, kami merangkum beberapa detail implementasi yang sangat penting.

### 1. CSS Defensif Adalah Wajib
Jangan pernah malas dan melewatkan awalan. Suatu kali, kami lupa untuk mencakup gaya layar splash dengan ID, dan global `div` gaya setelah React dipasang secara tidak terduga mempengaruhi layar splash, merusak tata letak.
**Pelajaran**: Letakkan setiap pemilih CSS di bawah `#boot-screen`, dan gunakan `!important` untuk meningkatkan prioritas bila diperlukan, tetapi hanya di dalam CSS layar splash.

### 2. Transisi yang Anggun
Setelah React berhasil di mount, jangan langsung `remove()` DOM layar pembuka.
**Pendekatan yang benar**:
1.  Pemicu reaksi `window.dispatchEvent(new Event('hagicode:ready'))`.
2.  Layar splash mendengarkan acara dan set pertama `opacity: 0`.
3.  Tunggu 300 ms, yang sesuai dengan durasi transisi CSS, dan telepon `.remove()` hanya setelah layar benar-benar tidak terlihat.

### 3. Sinkronisasi Variabel Tema
Nilai warna layar splash dikodekan secara keras `index.html`. Jika kami mengubah warna utama Tailwind, kami juga harus memperbarui layar pembuka.
**Optimasi**: Tulis plugin sederhana di skrip build Vite untuk dibaca `tailwind.config.js` dan menyuntikkan variabel warna ke dalam `index.html` variabel templat, menciptakan satu sumber kebenaran.

### 4. Pramuat Font
Layar splash sering kali perlu menggunakan font merek, tetapi jika font dimuat dengan lambat, FOUT (Flash of Unstyled Text) dapat muncul.
**Solusi**: Tambahkan `<link rel="preload" href="/fonts/JetBrainsMono.woff2" as="font" type="font/woff2" crossorigin>` di dalam `<head>`. Ini adalah cara berbiaya rendah dan menghasilkan keuntungan tinggi untuk meningkatkan pengalaman.

### 5. Pemantauan Kinerja
Kami menyuntik `performance.mark('boot-start')` di bagian bawah `index.html`, dan ditandai `boot-end` ketika React berhasil dipasang.
**Mengapa ini penting**: Dengan mengumpulkan data ini melalui Application Insights, kami dapat mengukur secara langsung seberapa banyak splash screen mempersingkat waktu tunggu yang dirasakan. Data menunjukkan bahwa layar kerangka yang sangat baik dapat meningkatkan toleransi pengguna terhadap "jaringan lambat" hingga lebih dari 50%.
