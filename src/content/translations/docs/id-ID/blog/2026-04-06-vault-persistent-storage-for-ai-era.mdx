## Cara Mereproduksi Proyek di Era AI: Vault, Sistem Penyimpanan Persisten Lintas Proyek

> Di era pengembangan yang dibantu AI, bagaimana kita dapat membantu asisten AI lebih memahami sumber daya pembelajaran kita? Proyek HagiCode membangun sistem Vault sebagai lapisan abstraksi penyimpanan pengetahuan terpadu yang dapat dipahami oleh AI, sehingga sangat meningkatkan efisiensi pembelajaran melalui reproduksi proyek.

## Latar Belakang

Di era AI, cara pengembang mempelajari teknologi dan arsitektur baru mengalami perubahan besar. "Mereproduksi proyek" - yaitu mempelajari dan mempelajari secara mendalam kode, arsitektur, dan pola desain proyek sumber terbuka yang sangat baik - telah menjadi cara yang efisien untuk belajar. Dibandingkan dengan metode tradisional seperti membaca buku atau menonton video, membaca langsung dan menjalankan proyek sumber terbuka berkualitas tinggi membantu Anda memahami praktik teknik dunia nyata dengan lebih cepat.

Namun, metode pembelajaran ini memiliki beberapa tantangan.

Materi pembelajaran terlalu tersebar. Catatan Anda mungkin ada di Obsidian, repositori kode mungkin tersebar di berbagai folder, dan riwayat percakapan asisten AI Anda menjadi pulau data terisolasi lainnya. Jika Anda ingin AI membantu menganalisis suatu proyek, Anda harus menyalin cuplikan kode secara manual dan mengatur konteks, yang mana hal ini agak membosankan.

Yang lebih menyusahkan lagi adalah konteks yang rusak. Asisten AI tidak dapat mengakses sumber belajar lokal Anda secara langsung, jadi Anda harus memberikan informasi latar belakang lagi di setiap percakapan. Selain itu, repositori kode yang direproduksi diperbarui dengan cepat, sinkronisasi manual rawan kesalahan, dan pengetahuan sulit dibagikan ke berbagai proyek pembelajaran.

Pada dasarnya, semua masalah ini berasal dari “pulau data”. Jika ada lapisan abstraksi penyimpanan terpadu yang memungkinkan asisten AI memahami dan mengakses semua sumber belajar Anda, masalahnya akan terpecahkan dengan baik.

## Tentang HagiCode

Sistem Vault yang dibagikan dalam artikel ini adalah solusi yang kami kembangkan saat membangun [Kode Hagi](https://hagicode.com). HagiCode adalah proyek asisten pengkodean AI, dan dalam pekerjaan pengembangan sehari-hari, kita sering kali perlu mempelajari dan merujuk ke banyak proyek sumber terbuka yang berbeda. Untuk membantu asisten AI lebih memahami sumber daya pembelajaran ini, kami merancang Vault, sistem penyimpanan persisten lintas proyek.

Solusi ini telah divalidasi di HagiCode dalam penggunaan nyata. Jika Anda menghadapi tantangan manajemen pengetahuan serupa, saya harap pengalaman ini dapat memberikan inspirasi. Lagi pula, begitu Anda sendiri terjatuh ke dalam beberapa lubang, Anda harus meninggalkan sesuatu untuk orang berikutnya.

## Filosofi desain sistem brankas

Ide inti sistem Vault sederhana: membuat lapisan abstraksi penyimpanan pengetahuan terpadu yang dapat dipahami oleh AI. Dari perspektif implementasi, sistem ini memiliki beberapa karakteristik utama.

### Dukungan multi-tipe

Sistem ini mendukung empat jenis brankas, masing-masing sesuai dengan skenario penggunaan berbeda:

```typescript
// folder: general-purpose folder type
export const DEFAULT_VAULT_TYPE = 'folder';

// coderef: a type specifically for reproduced code projects
export const CODEREF_VAULT_TYPE = 'coderef';

// obsidian: integrated with Obsidian note-taking software
export const OBSIDIAN_VAULT_TYPE = 'obsidian';

// system-managed: vault automatically managed by the system
export const SYSTEM_MANAGED_VAULT_TYPE = 'system-managed';
```

Diantaranya, tipe **coderef** adalah yang paling umum digunakan di HagiCode. Ini dirancang khusus untuk proyek kode yang direproduksi, menyediakan struktur direktori standar dan deskripsi metadata yang dapat dibaca AI.

### Mekanisme penyimpanan yang persisten

Registri Vault disimpan secara persisten dalam format JSON, memastikan bahwa konfigurasi tetap tersedia setelah aplikasi dimulai ulang:

```csharp
public class VaultRegistryStore : IVaultRegistryStore
{
    private readonly string _registryFilePath;

    public VaultRegistryStore(IConfiguration configuration, ILogger<VaultRegistryStore> logger)
    {
        var dataDir = configuration["DataDir"] ?? "./data";
        var absoluteDataDir = Path.IsPathRooted(dataDir)
            ? dataDir
            : Path.GetFullPath(Path.Combine(Directory.GetCurrentDirectory(), dataDir));

        _registryFilePath = Path.Combine(absoluteDataDir, "personal-data", "vaults", "registry.json");
    }
}
```

Keuntungan dari desain ini adalah sederhana dan dapat diandalkan. JSON dapat dibaca manusia, sehingga proses debug dan pengeditan manual menjadi lebih mudah; penyimpanan sistem file menghindari kompleksitas database dan mengurangi ketergantungan sistem. Lagi pula, terkadang pilihan yang paling sederhana adalah yang terbaik.

### Integrasi konteks AI

Yang terpenting, sistem dapat secara otomatis memasukkan informasi vault ke dalam konteks proposal AI:

```typescript
export function buildTargetVaultsText(
  vaults: VaultForText[],
  template: VaultPromptTemplate = DEFAULT_VAULT_PROMPT_TEMPLATE,
): string {
  const readOnlyVaults = vaults.filter((vault) => vault.accessType === 'read');
  const editableVaults = vaults.filter((vault) => vault.accessType === 'write');

  if (readOnlyVaults.length === 0 && editableVaults.length === 0) {
    return '';
  }

  const sections = [
    buildVaultSection(readOnlyVaults, template.reference),
    buildVaultSection(editableVaults, template.editable),
  ].filter(Boolean);

  return `\n\n### ${template.heading}\n\n${sections.join('\n')}`;
}
```

Hal ini memungkinkan kemampuan penting: asisten AI dapat secara otomatis memahami sumber belajar yang tersedia tanpa pengguna memberikan konteks secara manual. Bisa dibilang itu dianggap sebagai semacam pemahaman diam-diam.

## Struktur standar CodeRef Vault

Untuk tipe brankas coderef, HagiCode menyediakan struktur direktori standar:

```
my-coderef-vault/
├── index.yaml          # vault metadata description
├── AGENTS.md           # operating guide for AI assistants
├── docs/               # stores study notes and documents
└── repos/              # manages reproduced code repositories through Git submodules
```

Saat membuat brankas, sistem secara otomatis menginisialisasi struktur ini:

```csharp
private async Task EnsureCodeRefStructureAsync(
    string vaultName,
    string physicalPath,
    ICollection<VaultBootstrapDiagnosticDto> diagnostics,
    CancellationToken cancellationToken)
{
    Directory.CreateDirectory(physicalPath);

    var indexPath = Path.Combine(physicalPath, CodeRefIndexFileName);
    var docsPath = Path.Combine(physicalPath, CodeRefDocsDirectoryName);
    var reposPath = Path.Combine(physicalPath, CodeRefReposDirectoryName);

    // Create the standard directory structure
    if (!Directory.Exists(docsPath))
    {
        Directory.CreateDirectory(docsPath);
    }

    if (!Directory.Exists(reposPath))
    {
        Directory.CreateDirectory(reposPath);
    }

    // Create the AGENTS.md guide
    await EnsureCodeRefAgentsDocumentAsync(physicalPath, cancellationToken);

    // Create the index.yaml metadata
    await WriteCodeRefIndexDocumentAsync(indexPath, mergedDocument, cancellationToken);
}
```

Struktur ini juga dirancang dengan cermat:

- **docs/** menyimpan catatan belajar Anda, tempat Anda dapat mencatat pemahaman Anda tentang kode, analisis arsitektur, pembelajaran, dan sebagainya di Markdown
- **repos/** mengelola repositori yang direproduksi melalui submodul Git alih-alih menyalin kode secara langsung, sehingga kode tetap sinkron dan menghemat ruang
- **index.yaml** berisi metadata brankas sehingga asisten AI dapat dengan cepat memahami tujuan dan konten brankas
- **AGENTS.md** adalah panduan yang ditulis khusus untuk asisten AI, menjelaskan cara menangani konten brankas

Diorganisasikan dengan cara ini, mungkin AI dapat memahami apa yang Anda pikirkan dengan lebih mudah.

## Inisialisasi otomatis untuk brankas yang dikelola sistem

Selain brankas yang dibuat secara manual, HagiCode juga mendukung brankas yang dikelola sistem:

```csharp
public async Task<IReadOnlyList<VaultRegistryEntry>> EnsureAllSystemManagedVaultsAsync(
    CancellationToken cancellationToken = default)
{
    var definitions = GetAllResolvedDefinitions();
    var entries = new List<VaultRegistryEntry>(definitions.Count);

    foreach (var definition in definitions)
    {
        entries.Add(await EnsureResolvedSystemManagedVaultAsync(definition, cancellationToken));
    }

    return entries;
}
```

Sistem secara otomatis membuat dan mengelola brankas berikut:

- **hagiprojectdata**: penyimpanan data proyek yang digunakan untuk menyimpan konfigurasi dan status proyek
- **data pribadi**: penyimpanan data pribadi yang digunakan untuk menyimpan preferensi pengguna
- **hbsprompt**: pustaka templat perintah yang digunakan untuk mengelola perintah AI yang umum digunakan

Vault ini diinisialisasi secara otomatis saat sistem dimulai, sehingga pengguna tidak perlu mengonfigurasinya secara manual. Beberapa hal lebih baik diserahkan pada sistem daripada manusia mengkhawatirkannya.

## Mekanisme kontrol akses

Bagian penting dari desain adalah kontrol akses. Sistem membagi brankas menjadi dua jenis akses:

```typescript
export interface VaultForText {
  id: string;
  name: string;
  type: string;
  physicalPath: string;
  accessType: 'read' | 'write';  // Key: distinguish read-only from editable
}
```

- **referensi (hanya baca)**: AI hanya digunakan untuk analisis dan pemahaman dan tidak dapat mengubah konten. Cocok untuk referensi proyek sumber terbuka, dokumen, dan materi serupa
- **dapat diedit (dapat diedit)**: AI dapat mengubah konten sesuai kebutuhan untuk tugas tersebut. Cocok untuk catatan, draf, dan materi serupa Anda

Perbedaan ini penting. Ini memberi tahu AI konten mana yang merupakan "referensi hanya-baca" dan konten mana yang "aman untuk diedit", sehingga mengurangi risiko perubahan yang tidak disengaja. Lagi pula, tidak ada seorang pun yang ingin kerja kerasnya hilang karena pengeditan yang tidak disengaja.

## Dalam praktiknya: membuat dan menggunakan Vault

Sekarang setelah kita membahas ide-idenya, mari kita lihat cara kerjanya dalam praktik.

### Buat Gudang CodeRef

Berikut adalah contoh panggilan frontend lengkap:

```typescript
const createCodeRefVault = async () => {
  const response = await VaultService.postApiVaults({
    requestBody: {
      name: "React Learning Vault",
      type: "coderef",
      physicalPath: "/Users/developer/vaults/react-learning",
      gitUrl: "https://github.com/facebook/react.git"
    }
  });

  // The system will automatically:
  // 1. Clone the React repository into vault/repos/react
  // 2. Create the docs/ directory for notes
  // 3. Generate the index.yaml metadata
  // 4. Create the AGENTS.md guide file

  return response;
};
```

Panggilan API ini menyelesaikan serangkaian tindakan: membuat struktur direktori, menginisialisasi submodul Git, membuat file metadata, dan banyak lagi. Anda hanya perlu memberikan informasi dasar dan membiarkan sistem menangani sisanya. Sejujurnya ini adalah pendekatan yang tidak menimbulkan rasa khawatir.

### Gunakan Vault dalam proposal AI

Setelah membuat brankas, Anda dapat mereferensikannya dalam proposal AI:

```typescript
const proposal = composeProposalChiefComplaint({
  chiefComplaint: "Help me analyze React's concurrent rendering mechanism",
  repositories: [

```

Sistem secara otomatis memasukkan informasi vault ke dalam konteks AI, sehingga AI mengetahui sumber pembelajaran mana yang tersedia. Ketika AI dapat memahami apa yang ada dalam pikiran Anda, pemahaman diam-diam seperti itu sulit didapat.

## Praktik terbaik dan hal-hal yang harus diperhatikan

Saat menggunakan sistem Vault, kami telah merangkum beberapa pembelajaran.

### Keamanan jalur

Sistem secara ketat memvalidasi jalur untuk mencegah serangan traversal jalur:

```csharp
private static string ResolveFilePath(string vaultRoot, string relativePath)
{
    var rootPath = EnsureTrailingSeparator(Path.GetFullPath(vaultRoot));
    var combinedPath = Path.GetFullPath(Path.Combine(rootPath, relativePath));
    if (!combinedPath.StartsWith(rootPath, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode,
            "Vault file paths must stay inside the registered vault root.");
    }
    return combinedPath;
}
```

Ini penting. Jika Anda menyesuaikan jalur vault, pastikan jalur tersebut tetap berada dalam rentang yang diizinkan, jika tidak, sistem akan menolak operasi tersebut. Anda benar-benar tidak bisa terlalu menekankan keamanan.

### Manajemen submodul Git

CodeRef Vault merekomendasikan submodul Git daripada menyalin kode secara langsung:

```csharp
private static string BuildCodeRefAgentsContent()
{
    return """
    # CodeRef Vault Guide

    Repositories under `repos/` should be maintained through Git submodules
    rather than copied directly into the vault root.

    Keep this structure stable so assistants and tools can understand the vault quickly.
    """ + Environment.NewLine;
}
```

Hal ini memberikan beberapa keuntungan: menjaga kode tetap tersinkronisasi dengan upstream, menghemat ruang disk, dan mempermudah pengelolaan beberapa versi kode. Lagi pula, siapa yang ingin mendownload hal yang sama berulang kali?

### Batas pratinjau file

Untuk mencegah masalah kinerja, sistem membatasi ukuran dan jenis file:

```csharp
private const int FileEnumerationLimit = 500;
private const int PreviewByteLimit = 256 * 1024;  // 256KB
```

Jika brankas Anda berisi banyak file atau file yang sangat besar, kinerja pratinjau mungkin terpengaruh. Dalam hal ini, Anda dapat mempertimbangkan untuk memproses file secara batch atau menggunakan alat pencarian khusus. Terkadang ketika sesuatu menjadi terlalu besar, hal itu menjadi lebih sulit untuk ditangani, bukannya lebih mudah.

### Informasi diagnostik

Saat membuat brankas, sistem mengembalikan informasi diagnostik untuk membantu proses debug:

```csharp
List<VaultBootstrapDiagnosticDto> bootstrapDiagnostics = [];

if (IsCodeRefVaultType(normalizedType))
{
    bootstrapDiagnostics = await EnsureCodeRefBootstrapAsync(
        normalizedName,
        normalizedPhysicalPath,
        normalizedGitUrl,
        cancellationToken);
}
```

Jika pembuatan gagal, Anda dapat memeriksa informasi diagnostik untuk memahami penyebab spesifiknya. Ketika terjadi kesalahan, memeriksa diagnosis sering kali merupakan cara paling langsung.

## Ringkasan

Melalui lapisan abstraksi penyimpanan terpadu, sistem Vault memecahkan beberapa masalah inti dalam mereproduksi proyek di era AI:

- **Manajemen pengetahuan terpusat**: semua sumber belajar dikumpulkan di satu tempat, bukan tersebar di mana-mana
- **Injeksi konteks AI otomatis**: Asisten AI dapat secara otomatis memahami sumber belajar yang tersedia tanpa penyiapan konteks manual
- **Penggunaan kembali pengetahuan lintas proyek**: pengetahuan dapat dibagikan dan digunakan kembali di berbagai proyek pembelajaran
- **Struktur direktori terstandar**: tata letak direktori yang konsisten menurunkan kurva pembelajaran

Solusi ini telah divalidasi dalam proyek HagiCode. Jika Anda juga sedang membuat alat terkait pengembangan yang dibantu AI, atau menghadapi masalah manajemen pengetahuan serupa, saya harap pengalaman ini dapat menjadi referensi yang berguna.

Sebenarnya, nilai suatu solusi teknis tidak terletak pada seberapa rumit solusi tersebut, namun pada apakah solusi tersebut dapat menyelesaikan masalah nyata. Ide inti sistem Vault sangat sederhana: membangun lapisan penyimpanan pengetahuan terpadu yang dapat dipahami oleh AI. Namun justru abstraksi sederhana inilah yang cukup meningkatkan efisiensi pengembangan kami.
