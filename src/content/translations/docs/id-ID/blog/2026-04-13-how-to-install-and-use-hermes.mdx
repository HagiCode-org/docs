Jika Anda ingin menginstal Hermes dan mulai menggunakannya, jalur terpendek sebenarnya hanya tiga langkah:

1. Jalankan perintah instalasi resmi
2. Mulai CLI di terminal Anda dengan `hermes`
3. Jika Anda ingin tetap menggunakannya di Feishu, konfigurasikan `hermes gateway setup`

Artikel ini tidak mencoba menjelaskan setiap kemampuan Hermes sekaligus. Sebaliknya, ini membantu Anda menyelesaikan loop pemula yang paling penting terlebih dahulu: menginstalnya, menjalankannya, mulai menggunakannya, lalu menghubungkannya ke salah satu skenario platform perpesanan yang paling umum.

## Apa itu Hermes dan untuk siapa

Agen Hermes adalah agen AI yang dapat Anda gunakan baik dari terminal lokal atau melalui gateway platform perpesanan.

Bagi sebagian besar pengembang, ini memiliki dua titik masuk yang umum:

- **CLI**: Ketik `hermes` di terminal Anda untuk masuk ke antarmuka interaktif secara langsung.
- **Gerbang Pesan**: Jalankan `hermes gateway`, lalu ngobrol dengannya dari platform seperti Feishu, Telegram, Discord, dan Slack.

Jika tujuan Anda saat ini hanyalah memulai dengan cepat, jangan membalikkan urutannya. Mulailah dengan jalur ini sebagai gantinya:

- Instal Hermes terlebih dahulu
- Verifikasikan bahwa ini berfungsi dari CLI terlebih dahulu
- Kemudian putuskan apakah Anda ingin menghubungkan platform perpesanan

Hal ini membuat masalah lebih mudah didiagnosis dan lebih cocok untuk orang yang baru pertama kali menggunakan Hermes.

## Apa yang perlu diketahui sebelum menginstal Hermes

Menurut README Hermes, jalur instalasi cepat resmi mendukung lingkungan berikut:

- Linux
- macOS
- WSL2
- Android melalui Termux

### Catatan untuk pengguna Windows

Hermes saat ini tidak mendukung pengoperasian langsung pada Windows asli. Jika Anda menggunakan Windows, pendekatan yang disarankan adalah menginstal WSL2 terlebih dahulu dan kemudian menjalankan perintah instalasi di dalam WSL2.

Yang terbaik adalah memperjelas hal ini di awal, karena banyak kegagalan instalasi bukan disebabkan oleh perintah itu sendiri, tetapi karena penggunaan lingkungan runtime yang tidak didukung.

## Cara menginstal Hermes

Perintah instalasi cepat yang disediakan di Hermes README adalah:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Perintah ini menjalankan skrip instalasi resmi dan menangani langkah-langkah inisialisasi khusus platform.

### Muat ulang shell Anda setelah instalasi

Setelah instalasi selesai, muat ulang lingkungan shell Anda terlebih dahulu. Perintah yang paling umum adalah:

```bash
source ~/.bashrc
```

Jika Anda menggunakan zsh, Anda dapat menggunakan:

```bash
source ~/.zshrc
```

### Bagaimana cara memastikan Hermes sudah terinstal dengan benar

Cara paling langsung untuk memeriksanya adalah dengan menjalankan:

```bash
hermes
```

Jika Anda ingin konfirmasi tambahan bahwa konfigurasi dan dependensi Anda berfungsi, Anda juga dapat menjalankan:

```bash
hermes doctor
```

`hermes doctor` sangat berguna dalam situasi berikut:

- Perintah berperilaku tidak normal setelah instalasi
- Konfigurasi model gagal
- Gerbang gagal untuk memulai
- Anda tidak yakin apakah dependensi lingkungan Anda sudah selesai

## Bagaimana cara mulai menggunakan Hermes untuk pertama kalinya

### Mulailah dengan CLI

Jika Anda hanya ingin memastikan secepat mungkin bahwa Hermes berfungsi, cara paling sederhana adalah:

```bash
hermes
```

Ini meluncurkan Hermes CLI interaktif. Untuk pengguna Hermes yang baru pertama kali, ini juga merupakan titik awal yang paling direkomendasikan, karena Anda dapat memverifikasi hal yang paling penting terlebih dahulu:

- Apakah perintah tersebut benar-benar tersedia
- Apakah konfigurasi model saat ini berfungsi dengan baik
- Apakah rantai alat terminal berfungsi dengan benar
- Apakah gaya interaksi sesuai dengan kebutuhan Anda

### Perintah-perintah ini cukup untuk pengaturan putaran pertama Anda

Hermes README mencantumkan beberapa perintah frekuensi tinggi, dan bersama-sama membentuk jalur penggunaan pertama yang praktis:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Jika Anda tidak yakin dengan fungsi masing-masingnya, ingatlah seperti ini:

- `hermes model`: memilih atau mengganti model
- `hermes tools`: melihat dan mengonfigurasi alat yang tersedia saat ini
- `hermes config set`: mengubah item konfigurasi tertentu
- `hermes setup`: jalankan wizard inisialisasi penuh satu kali
- `hermes update`: perbarui Hermes
- `hermes doctor`: memecahkan masalah

Bagi pemula, urutan yang paling praktis biasanya adalah:

1. Jalankan `hermes model` pertama
2. Jika Anda ingin mengonfigurasi semua opsi umum sekaligus, jalankan `hermes setup`

## Dua cara paling umum untuk menggunakan Hermes

### 1. Gunakan Hermes di terminal sebagai asisten pengembangan harian

Mode CLI cocok untuk skenario berikut:

- Ajukan pertanyaan secara langsung sambil menulis kode secara lokal
- Periksa proyek, edit file, dan jalankan perintah
- Lakukan pekerjaan debug atau peninjauan satu kali
- Berkolaborasi terus menerus di direktori kerja saat ini

Keuntungan terbesarnya adalah jalur terpendek: tidak ada integrasi platform tambahan, tidak ada konfigurasi bot yang harus ditangani terlebih dahulu, dan ini adalah cara terbaik untuk membangun rangkaian kebiasaan penggunaan pertama Anda.

### 2. Gunakan Hermes melalui platform perpesanan

Jika Anda ingin mengobrol dengan Hermes di platform seperti Feishu, Telegram, atau Discord, Anda perlu menggunakan gateway perpesanan.

Perintah entri yang paling umum adalah:

```bash
hermes gateway setup
hermes gateway
```

Khususnya:

- `hermes gateway setup` digunakan untuk konfigurasi platform interaktif
- `hermes gateway` digunakan untuk memulai proses gateway

Menurut dokumentasi resmi, gateway adalah proses latar belakang terpadu yang menghubungkan platform Anda yang dikonfigurasi, mengelola sesi, dan menangani fitur seperti tugas cron.

## Menggunakan Feishu sebagai contoh: cara menghubungkan Hermes ke platform perpesanan

Jika sebagian besar pekerjaan sehari-hari Anda dilakukan di Feishu, maka Feishu/Lark adalah cara yang sangat alami untuk menggunakan Hermes.

### Jalur integrasi minimum yang layak

Dokumentasi resmi merekomendasikan perintah entri ini untuk Feishu/Lark:

```bash
hermes gateway setup
```

Setelah Anda menjalankannya, cukup pilih **Feishhu / Lark** di wizard.

Dokumentasi Feishu menjelaskan dua mode koneksi:

- `websocket`: direkomendasikan
- `webhook`: opsional

Jika Hermes berjalan di laptop, workstation, atau server pribadi Anda, gunakan `websocket` yang pertama biasanya lebih sederhana karena Anda tidak perlu mengekspos URL panggilan balik publik.

### Jika Anda mengkonfigurasinya secara manual, setidaknya ketahui variabel-variabel ini

Jika Anda tidak menggunakan wizard dan menulis konfigurasi secara manual, dokumentasi Feishu mencantumkan variabel inti berikut:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Dua di antaranya patut mendapat perhatian khusus:

- `FEISHU_ALLOWED_USERS`: direkomendasikan, jadi tidak semua orang yang dapat menjangkau bot tersebut dapat menggunakannya secara langsung
- `FEISHU_HOME_CHANNEL`: memungkinkan Anda menentukan obrolan rumah terlebih dahulu untuk menerima hasil cron atau notifikasi default

### Mengapa Hermes terkadang tidak membalas di obrolan grup Feishu

Detail ini mudah untuk dilewatkan: dalam obrolan grup Feishu, Hermes tidak merespons setiap pesan secara default.

Dokumentasi resmi dengan jelas menyatakan:

- Dalam pesan langsung, Hermes merespons pesan
- Dalam obrolan grup, Anda harus secara eksplisit `@` bot sebelum memproses pesan

Jika Anda ingin menetapkan percakapan Feishu sebagai saluran utama, Anda juga dapat menggunakan ini dalam obrolan:

```text
/set-home
```

Atau tentukan dalam konfigurasi sebelumnya:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Perintah Hermes yang harus diingat oleh pemula terlebih dahulu

Baik Anda menggunakan Hermes di CLI atau pada platform perpesanan, mengingat perintah berikut sudah cukup untuk memulai:

- `/new` atau `/reset`: memulai sesi baru
- `/model`: melihat atau mengganti model
- `/retry`: coba lagi giliran sebelumnya
- `/undo`: membatalkan interaksi sebelumnya
- `/compress`: mengompresi konteks secara manual
- `/help`: lihat bantuan

Jika Anda terutama menggunakan Hermes pada platform perpesanan, ingat satu hal lagi:

- `/sethome` atau `/set-home`: mengatur obrolan saat ini sebagai saluran utama

Perintah ini mencakup operasi tahap pemula yang paling umum: memulai ulang, menyesuaikan, memutar kembali, memeriksa, dan melanjutkan.

## Pertanyaan yang sering diajukan

### Bisakah Windows menginstal Hermes secara langsung?

Tidak. Dokumentasi resmi saat ini dengan jelas menyatakan bahwa Windows asli tidak didukung, dan WSL2 direkomendasikan.

### Apa yang harus saya lakukan jika mengetik `hermes` tidak melakukan apa pun setelah instalasi?

Cara terbaik untuk memecahkan masalah dalam urutan ini:

1. Muat ulang shell Anda terlebih dahulu, misalnya dengan `source ~/.bashrc`
2. Jalankan `hermes` lagi
3. Jika masih tidak normal, jalankan `hermes doctor`

### Mengapa bot tidak membalas di grup Feishu?

Periksa tiga hal ini terlebih dahulu:

- Apakah kamu `@` menyebut Hermes di grup
- Apakah `FEISHU_ALLOWED_USERS` membatasi pengguna saat ini
- Apakah kebijakan obrolan grup saat ini mengizinkan penanganan pesan grup

Menurut dokumentasi resmi Feishu, penggunaan @mention secara eksplisit diperlukan dalam skenario obrolan grup.

## Rekomendasi langkah selanjutnya

3. Gunakan `hermes model` dan `hermes setup` untuk menyelesaikan konfigurasi dasar
4. Jika Anda ingin tetap menggunakannya di Feishu, konfigurasikan `hermes gateway setup`
