## Desain Arsitektur

### Ikhtisar Arsitektur Berlapis

Integrasi antara C# dan CodeBuddy menggunakan arsitektur berlapis yang jelas. Desain ini memperjelas tanggung jawab dan membuat pemeliharaan jangka panjang menjadi lebih mudah:

```
┌─────────────────────────────────────────────┐
│           Provider Contract Layer           │
│   AIProviderType enum + extension methods   │
├─────────────────────────────────────────────┤
│           Provider Factory Layer            │
│   AIProviderFactory dependency injection factory │
├─────────────────────────────────────────────┤
│           Provider Implementation Layer     │
│   CodebuddyCliProvider concrete implementation │
├─────────────────────────────────────────────┤
│           ACP Infrastructure Layer          │
│  ACPSessionManager / StdioAcpTransport      │
│  AcpRpcClient / AcpAgentClient              │
└─────────────────────────────────────────────┘
```

Apa manfaat dari layering ini? Sederhananya, tiap lapisan saling menjauhi lapisan lainnya. Jika nanti kita ingin mengubah mekanisme komunikasinya, misalnya dari `stdio` ke WebSocket, kita hanya perlu memodifikasi lapisan bawah, dan logika bisnis di atasnya tetap tidak tersentuh. Tidak ada seorang pun yang menginginkan perubahan komunikasi terjadi di seluruh basis kode.

### Perincian Komponen Inti

**Lapisan kontrak penyedia** adalah fondasi seluruh arsitektur. Kami mendefinisikan `AIProviderType` enum, di mana `CodebuddyCli = 3` digunakan sebagai nilai enum, dan mengimplementasikan pemetaan dua arah antara string dan enum melalui metode ekstensi. Hal ini memungkinkan string dalam file konfigurasi dikonversi dengan mudah menjadi enum, dan enum dikonversi kembali menjadi string untuk keluaran debugging.

**Lapisan pabrik Penyedia** bertanggung jawab untuk membuat instance Penyedia yang sesuai berdasarkan konfigurasi. Ia menggunakan injeksi ketergantungan .NET bersama-sama `ActivatorUtilities.CreateInstance` untuk penciptaan dinamis. Keuntungan dari pola pabrik adalah ketika menambahkan Penyedia baru, Anda hanya perlu menambahkan logika pembuatan alih-alih mengubah kode yang sudah ada.

**Lapisan implementasi penyedia** adalah tempat terjadinya pekerjaan sebenarnya. `CodebuddyCliProvider` mengimplementasikan `IAIProvider` antarmuka dan menyediakan dua mode pemanggilan: `ExecuteAsync` untuk panggilan non-streaming dan `StreamAsync` untuk panggilan streaming.

**Lapisan infrastruktur ACP** menyediakan fondasi komunikasi di bawahnya. Lapisan ini menangani semua detail protokol, termasuk manajemen proses, serialisasi pesan, dan penguraian respons. Ini adalah fondasi yang menjaga segala sesuatu di atasnya tetap stabil.

## Mekanisme Komunikasi

### Modus Transportasi Stdio

CodeBuddy menggunakan **Stdio (input/output standar)** untuk berkomunikasi dengan proses eksternal. Perintah startupnya sederhana:

```bash
codebuddy --acp
```

Setelah itu, pesan JSON-RPC dipertukarkan melalui input dan output standar. Pendekatan ini memiliki beberapa keuntungan:

1. **Startup cepat**: komunikasi proses lokal menghindari latensi jaringan
2. **Konfigurasi sederhana**: Anda hanya perlu menentukan jalur yang dapat dieksekusi
3. **Isolasi lingkungan**: setiap sesi berjalan dalam proses independen, sehingga tidak mempengaruhi satu sama lain

Injeksi variabel lingkungan didukung selama komunikasi. Contoh umum meliputi:

- `CODEBUDDY_API_KEY`: otentikasi kunci API
- `CODEBUDDY_INTERNET_ENVIRONMENT`: konfigurasi lingkungan jaringan

Seperti halnya komunikasi antar manusia, ada baiknya memilih saluran yang nyaman terlebih dahulu.

### Protokol Pesan

ACP didasarkan pada JSON-RPC 2.0. Format pesannya kira-kira seperti ini:

```json
// Request message
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "agent/prompt",
  "params": {
    "prompt": "Help me write a sorting algorithm",
    "sessionId": "session-123"
  }
}

// Response message
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": "Here is the AI response..."
  }
}
```

Dalam implementasi sebenarnya, kami merangkum semua detail protokol ini sehingga lapisan bisnis atas hanya perlu memperhatikan kecepatan dan respons.

### 2. Integrasikan Pabrik Penyedia

Tambahkan cabang pembuatan CodeBuddy di kelas pabrik:

```csharp
// AIProviderFactory.cs
private IAIProvider? CreateProvider(AIProviderType providerType, ProviderConfiguration config)
{
    return providerType switch
    {
        AIProviderType.CodebuddyCli =>
            ActivatorUtilities.CreateInstance<CodebuddyCliProvider>(
                _serviceProvider,
                Options.Create(config)),
        // ... Other providers
        _ => throw new NotSupportedException($"Provider {providerType} not supported")
    };
}
```

Ini menggunakan injeksi ketergantungan `ActivatorUtilities`, yang secara otomatis menangani injeksi parameter konstruktor dan sangat nyaman.

### 3. Implementasi Penyedia Lengkap

Di bawah ini adalah implementasi inti dari `CodebuddyCliProvider`, mencakup mode pemanggilan streaming dan non-streaming:

```csharp
public class CodebuddyCliProvider : IAIProvider
{
    private readonly ILogger<CodebuddyCliProvider> _logger;
    private readonly IACPSessionManager _sessionManager;
    private readonly ProviderConfiguration _config;

    public string Name => "CodebuddyCli";
    public bool SupportsStreaming => true;
    public ProviderCapabilities Capabilities { get; }

    public CodebuddyCliProvider(
        ILogger<CodebuddyCliProvider> logger,
        IACPSessionManager sessionManager,
        IOptions<ProviderConfiguration> config)
    {
        _logger = logger;
        _sessionManager = sessionManager;
        _config = config.Value;

        // Define the capabilities of the current Provider
        Capabilities = new ProviderCapabilities
        {
            SupportsStreaming = true,
            SupportsTools = true,
            SupportsSystemMessages = true,
            SupportsArtifacts = false,
            MaxTokens = 8192
        };
    }

    // Non-streaming call: return all results together after completion
    public async Task<AIResponse> ExecuteAsync(
        AIRequest request,
        CancellationToken cancellationToken = default)
    {
        // Create an independent session for the request
        var session = await _sessionManager.CreateSessionAsync(
            "CodebuddyCli",
            request.WorkingDirectory,
            cancellationToken,
            request.SessionId);

        try
        {
            var fullPrompt = BuildPrompt(request);
            await session.SendPromptAsync(fullPrompt, cancellationToken);

            var responseBuilder = new StringBuilder();
            var toolCalls = new List<AIToolCall>();

            // Collect all response chunks
            await foreach (var chunk in StreamFromSession(session, cancellationToken))
            {
                if (!string.IsNullOrEmpty(chunk.Content))
                {
                    responseBuilder.Append(chunk.Content);
                }
                // Handle tool calls...
            }

            return new AIResponse
            {
                Content = AIResultContentSanitizer.SanitizeResultContent(
                    responseBuilder.ToString()),
                ToolCalls = toolCalls,
                Provider = Name,
                Model = string.Empty
            };
        }
        finally
        {
            // Release session resources
            await session.DisposeAsync();
        }
    }

    // Streaming call: return response chunks in real time
    public async IAsyncEnumerable<AIStreamingChunk> StreamAsync(
        AIRequest request,
        [EnumeratorCancellation] CancellationToken cancellationToken = default)
    {
        var session = await _sessionManager.CreateSessionAsync(
            "CodebuddyCli",
            request.WorkingDirectory,
            cancellationToken);

        try
        {
            var fullPrompt = BuildPrompt(request);
            await session.SendPromptAsync(fullPrompt, cancellationToken);

            await foreach (var chunk in StreamFromSession(session, cancellationToken))
            {
                yield return chunk;
            }
        }
        finally
        {
            await session.DisposeAsync();
        }
    }

    private async IAsyncEnumerable<AIStreamingChunk> StreamFromSession(
        IACPSession session,
        [EnumeratorCancellation] CancellationToken cancellationToken)
    {
        // Iterate through all updates in the session
        await foreach (var notification in session.ReceiveUpdatesAsync(cancellationToken))
        {
            switch (notification.Update)
            {
                case AgentMessageChunkSessionUpdate agentMessage:
                    // Handle text content chunks
                    if (agentMessage.Content is AcpImp.TextContentBlock textContent)
                    {
                        yield return new AIStreamingChunk
                        {
                            Content = textContent.Text,
                            Type = StreamingChunkType.ContentDelta,
                            IsComplete = false
                        };
                    }
                    break;

                case ToolCallSessionUpdate toolCall:
                    // Handle tool calls
                    yield return new AIStreamingChunk
                    {
                        Content = string.Empty,
                        Type = StreamingChunkType.ToolCallDelta,
                        ToolCallDelta = new AIToolCallDelta
                        {
                            Id = toolCall.ToolCallId,
                            Name = toolCall.Kind.ToString(),
                            Arguments = toolCall.RawInput?.ToString()
                        }
                    };
                    break;

                case AcpImp.PromptCompletedSessionUpdate:
                    // Response complete
                    yield break;
            }
        }
    }

    // Build the full prompt
    private string BuildPrompt(AIRequest request, string? embeddedCommandPrompt = null)
    {
        var sb = new StringBuilder();

        // Embedded command prompt, if present
        if (!string.IsNullOrEmpty(embeddedCommandPrompt))
        {
            sb.AppendLine(embeddedCommandPrompt);
            sb.AppendLine();
        }

        // System message
        if (!string.IsNullOrEmpty(request.SystemMessage))
        {
            sb.AppendLine(request.SystemMessage);
            sb.AppendLine();
        }

        // User prompt
        sb.Append(request.Prompt);
        return sb.ToString();
    }
}
```

Ada beberapa poin penting dalam kode ini:

1. **Manajemen sesi**: setiap permintaan membuat sesi independen dan melepaskan sumber daya setelah permintaan selesai. Ini adalah pelajaran yang didapat melalui trial and error. Jika penggunaan kembali sesi tidak ditangani dengan baik, pencemaran negara akan mudah terjadi.

2. **Pemrosesan streaming**: `IAsyncEnumerable` memungkinkan respons dikembalikan saat masih dibuat, alih-alih menunggu hingga semua konten selesai. Hal ini sangat penting terutama untuk skenario teks panjang dan secara signifikan meningkatkan pengalaman pengguna.

3. **Panggilan alat**: CodeBuddy mendukung pemanggilan alat (Panggilan Fungsi), ditangani melalui `ToolCallSessionUpdate`. Kemampuan ini sangat penting untuk tugas pengeditan kode yang kompleks.

4. **Pemfilteran konten**: `AIResultContentSanitizer` digunakan untuk memfilter konten blok Think dan menjaga output tetap bersih.

### 4. Konfigurasi Injeksi Ketergantungan

Tambahkan layanan terkait selama pendaftaran modul:

```csharp
// PCodeClaudeHelperModule.cs
public void ConfigureModule(IServiceCollection context)
{
    // Register Provider
    context.Services.AddTransient<CodebuddyCliProvider>();

    // Register ACP infrastructure
    context.Services.AddSingleton<IACPSessionManager, ACPSessionManager>();
    context.Services.AddSingleton<IAcpPlatformConfigurationResolver, AcpPlatformConfigurationResolver>();
    context.Services.AddSingleton<IAIRequestToAcpMapper, AIRequestToAcpMapper>();
    context.Services.AddSingleton<IAcpToAIResponseMapper, AcpToAIResponseMapper>();
}
```

## Contoh Konfigurasi

### File Konfigurasi

Tambahkan konfigurasi terkait CodeBuddy ke `appsettings.json`:

```yaml
AI:
  # Default Provider to use
  DefaultProvider: "CodebuddyCli"

  # Provider configuration
  Providers:
    CodebuddyCli:
      Type: "CodebuddyCli"
      WorkingDirectory: "C:/projects/my-app"
      ExecutablePath: "C:/tools/codebuddy.cmd"

  # Platform-specific configuration
  PlatformConfigurations:
    CodebuddyCli:
      ExecutablePath: "C:/tools/codebuddy.cmd"
      Arguments: "--acp"
      StartupTimeoutMs: 5000
      EnvironmentVariables:
        CODEBUDDY_API_KEY: "${CODEBUDDY_API_KEY}"
        CODEBUDDY_INTERNET_ENVIRONMENT: "production"
```

### Model Konfigurasi

Definisi model konfigurasi yang sesuai:

```csharp
public class CodebuddyPlatformConfiguration : IAcpPlatformConfiguration
{
    public string ProviderName => "CodebuddyCli";
    public AcpTransportType TransportType => AcpTransportType.Stdio;

    public string ExecutablePath { get; set; } = "codebuddy";
    public string Arguments { get; set; } = "--acp";
    public int StartupTimeoutMs { get; set; } = 5000;

    public Dictionary<string, string?>? EnvironmentVariables { get; set; }
}
```

## Pelajaran Praktis yang Dipetik

### Catatan Jebakan

Kami menemui beberapa kendala umum selama penerapan, dan membagikannya di sini dapat membantu orang lain menghindari jalan memutar yang sama:

1. **Masalah kebocoran sesi**: pada awalnya, sesi tidak dirilis dengan benar, sehingga menghabiskan sumber daya proses. Solusinya adalah dengan menggunakan `try-finally` untuk memastikan sumber daya dilepaskan untuk setiap permintaan.

2. **Pengiriman variabel lingkungan**: Windows dan Linux menggunakan sintaksis variabel lingkungan yang berbeda, jadi kami kemudian melakukan standarisasi `Dictionary<string, string?>` untuk menangani ini.

3. **Konfigurasi batas waktu**: Startup CLI membutuhkan waktu, jadi kami menetapkan batas waktu startup 5 detik untuk menghindari kegagalan permintaan yang cepat.

4. **Masalah pengodean**: di Windows, pengodean default dapat menyebabkan teks berbahasa Mandarin kacau, sehingga pengodean UTF-8 ditentukan secara eksplisit saat memulai proses.

### Optimasi Kinerja

1. **Kumpulan sesi**: untuk permintaan singkat yang sering terjadi, pertimbangkan untuk menerapkan kumpulan sesi untuk menggunakan kembali proses
2. **Cache koneksi**: kelas pabrik sudah mendukung instance Penyedia cache
3. **Async terlebih dahulu**: gunakan pemrograman asinkron secara menyeluruh untuk menghindari pemblokiran thread

Performa selalu layak untuk dioptimalkan. Semakin lama pengguna menunggu, semakin buruk pengalamannya.

## Kesimpulan

Artikel ini memperkenalkan solusi lengkap untuk mengintegrasikan CodeBuddy CLI ke dalam backend C#, yang mencakup seluruh proses mulai dari desain arsitektur hingga implementasi nyata.
