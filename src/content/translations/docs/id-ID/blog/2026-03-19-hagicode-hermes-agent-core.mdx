## Mengapa HagiCode Memilih Hermes sebagai Agen Inti Terintegrasinya

> Saat membangun platform pengkodean berbantuan AI, pemilihan inti Agen yang tepat secara langsung menentukan batas atas kemampuan sistem. Ada beberapa hal yang tidak bisa dipaksakan; pilih kerangka yang salah, dan upaya apa pun tidak akan membuatnya terasa benar. Artikel ini berbagi pemikiran di balik pemilihan teknis HagiCode dan pengalaman langsung kami dalam mengintegrasikan Agen Hermes.

## Latar Belakang

Saat membuat produk pengkodean berbantuan AI, salah satu bagian tersulit adalah memilih kerangka kerja Agen yang mendasarinya. Sebenarnya terdapat beberapa pilihan di pasaran, namun ada yang fungsinya terlalu terbatas, ada yang terlalu rumit untuk diterapkan, dan ada yang tidak dapat diskalakan dengan cukup baik. Yang kami butuhkan adalah solusi yang dapat berjalan pada VPS seharga $5 dan juga dapat terhubung ke cluster GPU. Persyaratan tersebut mungkin tidak terdengar ekstrem, namun cukup membuat takut banyak tim.

Dalam praktiknya, banyak yang disebut "Agen lengkap" hanya berjalan di cloud atau memerlukan biaya penerapan lokal yang sangat tinggi. Setelah menghabiskan dua minggu untuk meneliti berbagai pendekatan yang berbeda, kami membuat keputusan yang berani: membangun kembali seluruh inti Agen di sekitar Hermes sebagai mesin yang mendasari Agen terintegrasi kami.

Segala sesuatu yang terjadi selanjutnya mungkin hanyalah takdir.

Anda mungkin pernah menggunakan alat serupa sebelumnya dan merasa alat tersebut kehilangan sentuhan terakhirnya; kami memahami perasaan itu dengan baik.

## Mengapa HagiCode Membutuhkan Hermes

Sebelum mendalami Hermes sendiri, ada baiknya menjelaskan mengapa HagiCode membutuhkan sesuatu seperti itu. Jarang sekali segala sesuatunya berjalan sesuai keinginan Anda, jadi Anda memerlukan alasan praktis untuk berkomitmen pada arahan teknis.

Sebagai asisten pengkodean AI, HagiCode perlu mendukung beberapa skenario penggunaan secara bersamaan:
- Lingkungan pengembangan lokal: pengembang ingin menjalankannya di mesin mereka sendiri sehingga data tidak pernah meninggalkan lingkungan lokal. Saat ini, keamanan data bukanlah suatu hal yang sepele.
- Lingkungan kolaborasi tim: tim kecil harus dapat berbagi penerapan Agen yang berjalan di server. Menyimpan uang itu penting, dan setiap orang mempunyai batasan.
- Ekspansi cloud elastis: saat menangani tugas-tugas kompleks, sistem akan secara otomatis memperluas skala ke cluster GPU. Selalu lebih baik untuk bersiap.

Persyaratan "kami menginginkan semuanya sekaligus" inilah yang membawa kami ke Hermes. Apakah ini merupakan pilihan yang sempurna, saya tidak dapat memastikannya, namun pada saat itu kami tidak melihat pilihan yang lebih baik.

## Apa Itu Agen Hermes

Agen Hermes adalah Agen AI otonom yang dibuat oleh **Nous Research**. Beberapa pembaca mungkin belum familiar dengan Nous Research; mereka adalah laboratorium di balik model sumber terbuka besar seperti Hermes, Nomos, dan Psyché. Mereka telah membangun banyak hal yang luar biasa, meskipun hal-hal tersebut masih kurang dihargai daripada yang seharusnya mereka terima.

Tidak seperti asisten pengkodean IDE tradisional atau pembungkus obrolan API sederhana, Hermes memiliki ciri khas: semakin lama dijalankan, semakin mumpuni. Hal ini tidak dirancang untuk menyelesaikan tugas sekali dan berhenti; ia terus belajar dan mengumpulkan pengalaman selama operasi jangka panjang. Dalam hal ini, rasanya seperti seseorang.

### Fitur Inti

Beberapa kemampuan inti Hermes sangat selaras dengan kebutuhan HagiCode.

Ini berarti HagiCode dapat memilih model penerapan yang paling sesuai berdasarkan skenario masing-masing pengguna: individu menjalankannya secara lokal, tim menerapkannya di server, dan tugas kompleks menggunakan sumber daya GPU. Satu basis kode menangani semuanya. Di dunia yang sibuk ini, menyelamatkan satu lapisan kompleksitas sudah merupakan sebuah kemenangan.

**Gerbang perpesanan multi-platform**
Hermes secara asli mendukung Telegram, Discord, Slack, WhatsApp, dan banyak lagi. Untuk HagiCode, ini berarti kami dapat mendukung asisten AI di saluran tersebut dengan lebih mudah di masa mendatang. Lebih banyak jalan ke depan selalu diterima.

**Sistem alat yang kaya **
Hermes hadir dengan 40+ alat bawaan dan mendukung ekstensi MCP (Model Context Protocol). Hal ini penting bagi asisten pengkodean: menjalankan perintah shell, bekerja dengan sistem file, dan memanggil Git semuanya bergantung pada dukungan alat. Agen tanpa alat ibarat burung tanpa sayap.

**Memori lintas sesi**
Hermes menyertakan sistem memori persisten dan menggunakan pencarian teks lengkap FTS5 untuk mengingat percakapan historis. Hal ini memungkinkan Agen mengingat konteks sebelumnya alih-alih "kehilangan ingatannya" setiap saat. Kadang-kadang orang berharap bisa melupakan sesuatu dengan mudah, namun kenyataannya biasanya tidak begitu baik.

## Bagaimana HagiCode Mengintegrasikan Hermes

Sekarang setelah “mengapa” sudah jelas, mari kita lihat “bagaimana”. Ketika sesuatu secara teori masuk akal, langkah selanjutnya adalah membangunnya.

### Abstraksi Lapisan Penyedia

Dalam arsitektur HagiCode, semua Penyedia AI menerapkan sistem terpadu `IAIProvider` antarmuka:

```csharp
public sealed class HermesCliProvider : IAIProvider, IVersionedAIProvider
{
    public ProviderCapabilities Capabilities { get; } = new ProviderCapabilities
    {
        SupportsStreaming = true,   // Supports streaming output
        SupportsTools = true,       // Supports tool invocation
        SupportsSystemMessages = true, // Supports system prompts
        SupportsArtifacts = false
    };
}
```

Lapisan abstraksi ini memungkinkan HagiCode untuk beralih secara mulus antar Penyedia AI yang berbeda. Baik backendnya OpenAI, Claude, atau Hermes, pola panggilan lapisan atas tetap sama. Secara sederhana, ini membuat segalanya tetap sederhana.

### Protokol Komunikasi ACP

Hermes berkomunikasi melalui **ACP (Agent Communication Protocol)**. Protokol ini dirancang khusus untuk komunikasi Agen, dan metode utamanya meliputi:

| Metode | Deskripsi |
|------|------|
| `initialize` | Inisialisasi koneksi dan dapatkan versi protokol dan kemampuan klien |
| `authenticate` | Tangani otentikasi dan dukung beberapa metode otentikasi |
| `session/new` | Buat sesi baru dan konfigurasikan direktori kerja dan server MCP |
| `session/prompt` | Kirim prompt dan terima tanggapan |

HagiCode mengimplementasikan lapisan transport ACP melalui `StdioAcpTransport`, meluncurkan subproses Hermes dan berkomunikasi dengannya melalui input dan output standar. Ini mungkin terdengar rumit, tetapi dalam praktiknya hal ini dapat dilakukan selama Anda memiliki cukup kesabaran.

### Manajemen Konfigurasi

Konfigurasi dikelola melalui `HermesPlatformConfiguration` kelas:

```csharp
public sealed class HermesPlatformConfiguration : IAcpPlatformConfiguration
{
    public string ExecutablePath { get; set; } = "hermes";
    public string Arguments { get; set; } = "acp";
    public int StartupTimeoutMs { get; set; } = 5000;
    public string ClientName { get; set; } = "HagiCode";
    public HermesAuthenticationConfiguration Authentication { get; set; }
    public HermesSessionDefaultsConfiguration SessionDefaults { get; set; }
}
```

Konfigurasikan Hermes di `appsettings.json`:

```json
{
  "Providers": {
    "HermesCli": {
      "ExecutablePath": "hermes",
      "Arguments": "acp",
      "StartupTimeoutMs": 10000,
      "ClientName": "HagiCode",
      "Authentication": {
        "PreferredMethodId": "api-key",
        "MethodInfo": {
          "api-key": "your-api-key-here"
        }
      },
      "SessionDefaults": {
        "Model": "claude-sonnet-4-20250514",
        "ModeId": "default"
      }
    }
  }
}
```

Konfigurasi sering kali terlihat sederhana di atas kertas, namun untuk menyempurnakan setiap detailnya tetap memerlukan upaya yang sungguh-sungguh.

### Arsitektur Terdistribusi Orleans

HagiCode menggunakan Orleans untuk membangun sistem terdistribusinya, dan integrasi Hermes diimplementasikan melalui komponen berikut:

- **HermesGrain**: Implementasi Orleans Grain yang menangani eksekusi sesi
- **HermesPlatformConfiguration**: Konfigurasi khusus platform
- **HermesAcpSessionAdapter**: Adaptor sesi ACP
- **HermesConsole**: Konsol validasi khusus

Nama Orleans memang memiliki daya tarik tersendiri. Meskipun Orleans ini tidak ada hubungannya dengan kota legendaris, nama baik tidak ada salahnya.

### Alur Eksekusi End-to-End

Berikut ini adalah logika eksekusi inti Penyedia Hermes:

```csharp
private async IAsyncEnumerable<AIStreamingChunk> StreamCoreAsync(
    AIRequest request,
    string? embeddedCommandPrompt,
    [EnumeratorCancellation] CancellationToken cancellationToken)
{
    // 1. Create transport layer and launch Hermes subprocess
    await using var transport = new StdioAcpTransport(
        platformConfiguration.GetExecutablePath(),
        platformConfiguration.GetArguments(),
        platformConfiguration.GetEnvironmentVariables(),
        platformConfiguration.GetStartupTimeout(),
        _loggerFactory.CreateLogger<StdioAcpTransport>());
    await transport.ConnectAsync(cancellationToken);

    // 2. Initialize and obtain protocol version and authentication methods
    var initializeResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "initialize",
        BuildInitializeParameters(platformConfiguration), cancellationToken);

    // 3. Handle authentication
    var authMethods = ParseAuthMethods(initializeResult);
    if (!isAuthenticated)
    {
        var methodId = platformConfiguration.Authentication.ResolveMethodId(authMethods);
        await SendHermesRequestAsync(transport, nextRequestId++, "authenticate", ...);
    }

    // 4. Create session
    var newSessionResult = await SendHermesRequestAsync(
        transport, nextRequestId++, "session/new",
        BuildNewSessionParameters(platformConfiguration, workingDirectory, model), cancellationToken);
    var sessionId = ParseSessionId(newSessionResult);

    // 5. Execute prompt and collect streaming responses
    await foreach (var payload in transport.ReceiveMessagesAsync(cancellationToken))
    {
        // Handle session/update notifications and convert them into streaming chunks
        if (TryParseSessionNotification(root, out var notification))
        {
            if (_responseMapper.TryConvertToStreamingChunk(notification, out var chunk))
            {
                yield return chunk;
            }
        }
    }
}
```

Dengan kode, detailnya akhirnya menjadi familiar. Yang paling penting adalah pendekatan keseluruhan.

### Pemeriksaan Kesehatan

Untuk memastikan Hermes tetap tersedia, HagiCode menerapkan mekanisme pemeriksaan kesehatan:

```csharp
public async Task<ProviderTestResult> PingAsync(CancellationToken cancellationToken = default)
{
    var response = await ExecuteAsync(
        new AIRequest
        {
            Prompt = "Reply with exactly PONG.",
            CessionId = null,
            AllowedTools = Array.Empty<string>(),
            WorkingDirectory = ResolveWorkingDirectory(null)
        },
        cancellationToken);

    var success = string.Equals(response.Content.Trim(), "PONG", StringComparison.OrdinalIgnoreCase);
    return new ProviderTestResult
    {
        ProviderName = Name,
        Success = success,
        ResponseTimeMs = stopwatch.ElapsedMilliseconds,
        ErrorMessage = success ? null : $"Unexpected Hermes ping response: '{response.Content}'."
    };
}
```

Kira-kira seperti itulah “pemeriksaan kesehatan” di sini. Dalam beberapa hal, orang-orang tidak begitu berbeda: ada baiknya untuk melapor masuk dari waktu ke waktu, bahkan jika tidak ada yang memberi tahu kita apa yang harus dicari.

## Pertimbangan Praktis

Ada beberapa kendala yang perlu dipahami sebelum mengintegrasikan Hermes. Semua orang cepat atau lambat akan masuk ke dalam beberapa jebakan.

### Konfigurasi Metode Otentikasi

Hermes mendukung beberapa metode autentikasi, termasuk kunci dan token API, jadi Anda harus memilih berdasarkan skenario penerapan sebenarnya. Kesalahan konfigurasi dapat menyebabkan kegagalan koneksi, dan pesan kesalahan yang dihasilkan tidak selalu intuitif. Terkadang kesalahan yang dilaporkan jauh dari penyebab sebenarnya, yang berarti proses debug yang lambat dan hati-hati tidak dapat dihindari.

### Konfigurasi Server MCP

Saat membuat sesi, Anda dapat mengonfigurasi daftar server MCP sehingga Hermes dapat memanggil alat eksternal. Namun ingatlah hal-hal berikut ini:
- Alamat server MCP harus dapat dijangkau
- Batas waktu harus dikonfigurasi secara wajar
- Sistem memerlukan penanganan degradasi ketika server tidak tersedia

Dalam praktiknya, pemikiran defensif lebih penting daripada yang diperkirakan orang.

### Manajemen Direktori Kerja

Setiap sesi harus menentukan direktori kerja sehingga Hermes dapat mengakses file proyek dengan benar. Dalam skenario multi-proyek, direktori kerja perlu berpindah secara dinamis. Kedengarannya mudah, tetapi ada lebih banyak kasus rumit daripada yang Anda kira.

### Agregasi Respons

Tanggapan Hermes mungkin berbeda-beda `session/update` notifikasi dan hasil akhirnya, jadi harus digabungkan dengan benar. Jika tidak, konten mungkin hilang.

### Strategi Penanganan Kesalahan

Kesalahan runtime harus dikembalikan secara eksplisit, bukan secara diam-diam dikembalikan ke Penyedia lain. Dengan begitu, pengguna mengetahui bahwa masalahnya berasal dari Hermes daripada bertanya-tanya mengapa sistem tiba-tiba mengganti model di belakang layar.
