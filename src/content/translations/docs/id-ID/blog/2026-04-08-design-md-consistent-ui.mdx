## Design.md: Solusi untuk Desain UI Frontend Berbasis AI yang Konsisten

> Di era pengembangan frontend yang dibantu AI, bagaimana Anda bisa menjaga konsistensi UI yang dihasilkan AI? Artikel ini berbagi pengalaman langsung kami dalam membangun situs galeri desain berdasarkan awesome-design-md, beserta cara membuat struktur `design.md` untuk memandu AI menuju desain UI standar.

## Latar Belakang

Siapa pun yang pernah menggunakan AI untuk menulis kode frontend mungkin memiliki pengalaman serupa: meminta AI untuk membuat halaman yang sama beberapa kali, dan setiap hasil muncul dalam gaya yang berbeda. Terkadang sudutnya membulat, terkadang tajam. Terkadang spasinya 8px, dan di lain waktu menjadi 16px. Bahkan tombol yang sama dapat terlihat berbeda pada percakapan yang berbeda.

Ini bukanlah persoalan tersendiri. Seiring dengan semakin umumnya pengembangan yang dibantu AI, **kurangnya konsistensi dalam UI frontend yang dihasilkan AI** telah menjadi masalah yang meluas. Asisten AI yang berbeda, perintah yang berbeda, dan bahkan asisten yang sama dalam percakapan yang berbeda dapat menghasilkan desain antarmuka yang sangat berbeda. Hal ini menimbulkan biaya pemeliharaan yang besar selama iterasi produk.

Akar permasalahan sebenarnya sederhana: tidak ada **dokumen referensi desain resmi**. Stylesheet CSS tradisional dapat memberi tahu pengembang "cara mengimplementasikan" sesuatu, namun mereka tidak dapat sepenuhnya mengomunikasikan "mengapa dirancang seperti ini" atau "pola desain mana yang harus digunakan dalam skenario apa." Bagi AI, deskripsi yang jelas dan terstruktur bahkan lebih penting untuk memahami konvensi desain.

Pada saat yang sama, komunitas sumber terbuka telah menawarkan beberapa sumber daya yang sangat baik. Itu [VoltAgent/desain-luar biasa-md](https://github.com/VoltAgent/awesome-design-md) proyek mengumpulkan dokumentasi sistem desain dari banyak perusahaan terkenal. Setiap direktori berisi a `README.md`, sebuah `DESIGN.md`, dan pratinjau HTML. Namun, semua itu tersebar di seluruh repositori upstream, sehingga sulit untuk menelusuri dan membandingkannya dengan cepat.

Jadi, bisakah kita menggabungkan sumber daya tersebut ke dalam galeri desain yang mudah dijelajahi, sekaligus menyaring yang terstruktur `design.md` untuk digunakan AI?

Jawabannya adalah ya. Selanjutnya, izinkan saya menjelaskan pendekatan kita.

Solusi yang dibagikan dalam artikel ini berasal dari pengalaman langsung kami dalam [Kode Hagi](https://hagicode.com) proyek. HagiCode adalah platform pengembangan yang dibantu AI, dan selama pengembangan kami mengalami masalah yang sama yaitu UI yang dihasilkan AI tidak konsisten. Untuk mengatasinya, kami membangun situs galeri desain dan membuat standar `design.md`. Artikel ini adalah ringkasan dari solusi tersebut.

## Analisis

Sebelum menulis kode, mari kita uraikan tantangan teknis di balik masalah ini.

### Manajemen Konten Sumber: Bagaimana Anda Menyatukan Sumber Daya Desain yang Tersebar?

Repositori awesome-design-md hulu berisi sejumlah besar dokumen desain, namun kami memerlukan cara untuk membawanya ke dalam proyek kami sendiri.

**Solusi: gunakan submodul git**

```
awesome-design-md-site
└── vendor/awesome-design-md          # Upstream resources (git submodule)
```

Ini memberi kita beberapa manfaat:
- Kontrol versi: kami dapat menyematkan versi upstream tertentu
- Pembuatan offline: tidak perlu meminta API eksternal selama pembuatan
- Tinjauan konten: perubahan spesifik terlihat di PR

### Normalisasi Data: Bagaimana Anda Menstandarkan Struktur Dokumen yang Berbeda?

Perusahaan yang berbeda menyusun dokumen desainnya secara berbeda. Beberapa file pratinjau tidak ada, dan beberapa menggunakan penamaan yang tidak konsisten. Kita perlu menormalkannya selama proses pembangunan.

**Solusi: pindai dan buat entri yang dinormalisasi pada waktu pembuatan**

3. Mengekstraksi dan merender konten Markdown ke dalam HTML
4. Menghasilkan data entri yang dinormalisasi

### Arsitektur Situs Statis: Bagaimana Anda Menyediakan Pencarian Dinamis Sambil Tetap Statis Sepenuhnya?

Karena ini adalah galeri desain, pencarian adalah hal yang harus dilakukan. Tapi Astro adalah generator situs statis, jadi bagaimana Anda menerapkan pencarian real-time?

**Solusi: Pulau React + sinkronisasi parameter kueri URL**

```typescript
// src/components/gallery/SearchToolbar.tsx

export function SearchToolbar() {
  const [query, setQuery] = useState('');

  // Sync with the URL
  useEffect(() => {
    const params = new URLSearchParams(window.location.search);

```

Keuntungan dari pendekatan ini adalah menjaga kemampuan penerapan situs statis tetap utuh, artinya pendekatan ini dapat diterapkan ke layanan hosting statis apa pun, sambil tetap memberikan pengalaman pemfilteran instan.

### Dokumentasi Desain: Bagaimana Anda Membantu AI Memahami dan Mengikuti Standar Desain?

Ini adalah inti dari keseluruhan solusi. Kita perlu membuat yang terstruktur `design.md` yang dapat dipahami dan diterapkan oleh AI.

**Solusi: pinjam struktur ClickHouse `DESIGN.md`**

[ClickHouse's `DESIGN.md`](https://github.com/ClickHouse/ClickHouse/blob/master/DESIGN.md) adalah referensi yang sangat bagus. Ini termasuk:
- Tema & Suasana Visual
- Palet Warna & Peran
- Aturan Tipografi
- Styling Komponen
- Prinsip Tata Letak
- Kedalaman & Ketinggian
- Anjuran dan Larangan
- Perilaku Responsif
- Panduan Cepat Agen

Pendekatan kami adalah: **gunakan kembali struktur, tulis ulang konten**. Kami menjaga struktur bagian ClickHouse `DESIGN.md`, tetapi ganti kontennya dengan token desain aktual dan konvensi komponen yang digunakan dalam proyek kita sendiri.

## Solusi

Berdasarkan analisis di atas, solusi kami terdiri dari empat modul inti.

### 1. Saluran Penyerapan Konten

Ini adalah fondasi keseluruhan sistem, yang bertanggung jawab untuk mengekstraksi dan menormalisasi konten dari sumber daya hulu.

```typescript
// src/lib/content/awesomeDesignCatalog.ts

export async function scanSourceEntries(): Promise<DesignEntry[]> {
  const designDir = 'vendor/awesome-design-md/design-md';
  const entries: DesignEntry[] = [];

  for (const dir of await fs.readdir(designDir)) {
    const entryPath = path.join(designDir, dir);
    if (await isValidDesignEntry(entryPath)) {
      const entry = await normalizeDesignEntry(entryPath);
      entries.push(entry);
    }
  }

  return entries;
}

async function isValidDesignEntry(dir: string): Promise<boolean> {
  const requiredFiles = ['README.md', 'DESIGN.md'];
  for (const file of requiredFiles) {
    if (!(await fileExists(path.join(dir, file)))) {
      return false;
    }
  }
  return true;
}
```

### 2. Antarmuka Penjelajahan Galeri

Antarmuka galeri mencakup tiga bagian utama:

**Beranda**: menampilkan kisi kartu dari semua entri desain, dan setiap kartu mencakup:
- Judul entri desain dan ringkasan
- Pratinjau gambar, jika tersedia
- Sorotan pencarian cepat

**Halaman detail**: mengumpulkan informasi lengkap untuk satu entri desain:
- `README` dokumen
- `DESIGN` dokumen
- Pratinjau dengan peralihan tema terang/gelap
- Navigasi ke entri yang berdekatan

**Navigasi**: mendukung kembali ke galeri dan menelusuri entri yang berdekatan

Galeri beranda menggunakan tata letak kartu kepadatan tinggi, rata `design.md` entri dari berbagai sumber ke dalam kerangka visual terpadu sehingga tim dapat dengan cepat membandingkan gaya merek, pola tombol, dan ritme tipografi.

Setelah membuka entri tertentu, halaman detail menempatkan ringkasan desain dan pratinjau langsung di halaman yang sama, sehingga mengurangi biaya peralihan antara dokumentasi, pratinjau, dan kode sumber.

### 3. Fungsi Pencarian

Fitur pencarian didasarkan pada pemfilteran sisi klien, dengan status dipertahankan melalui parameter kueri URL:

```typescript
// src/components/gallery/SearchToolbar.tsx

function SearchToolbar({ entries }: { entries: DesignEntry[] }) {
  const [query, setQuery] = useState('');
  const [results, setResults] = useState(entries);

  useEffect(() => {
    const params = new URLSearchParams(window.location.search);
    const initialQuery = params.get('q') || '';
    setQuery(initialQuery);
    filterEntries(initialQuery);
  }, []);

  const filterEntries = (searchQuery: string) => {
    const filtered = entries.filter(entry =>
      entry.searchText.toLowerCase().includes(searchQuery.toLowerCase())
    );
    setResults(filtered);
  };

  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const value = e.target.value;
    setQuery(value);
    filterEntries(value);

    // Update the URL without triggering a page refresh
    const newUrl = value
      ? `${window.location.pathname}?q=${encodeURIComponent(value)}`
      : window.location.pathname;
    window.history.replaceState({}, '', newUrl);
  };

  return (
    <div className="search-toolbar">
      <input
        type="text"
        value={query}
        onChange={handleChange}
        placeholder="Search design entries..."
      />
      <span className="result-count">{results.length} results</span>
    </div>
  );
}
```

## Praktek

Sekarang kita telah membahas solusinya, bagaimana Anda menerapkannya?

### Langkah-Langkah Implementasi

**Langkah 1: Inisialisasi submodul**

```bash
# Add the upstream repository as a submodule
git submodule add https://github.com/VoltAgent/awesome-design-md.git vendor/awesome-design-md

# Initialize and update the submodule
git submodule update --init --recursive
```

**Langkah 2: Buat saluran konten**

Menerapkan `awesomeDesignCatalog.ts`, termasuk:
- Pemindaian file dan logika validasi
- Render penurunan harga menggunakan renderer bawaan Astro
- Ekstraksi data entri

**Langkah 3: Bangun UI galeri**

Gunakan Astro + React Islands untuk membuat:
- Tata letak galeri beranda (kisi kartu)
- Komponen kartu desain
- Bilah alat pencarian
- Tata letak halaman detail

**Langkah 4: Tulis dokumen desain**

Berdasarkan struktur ClickHouse `DESIGN.md`, isi token desain sebenarnya dari proyek Anda sendiri. Pembaruan `README.md` dan tambahkan tautan ke `design.md`.

### Catatan

**Keamanan**: Render penurunan harga memerlukan pemfilteran HTML yang tidak aman. Filter penyaji bawaan Astro `script` tag secara default, tetapi Anda masih perlu memperhatikan risiko XSS.

**Kinerja**: Banyaknya pratinjau iframe dapat memengaruhi performa first-paint. Disarankan untuk digunakan `loading="lazy"` untuk memuat konten pratinjau dengan lambat.

**Kemampuan Pemeliharaan**: `design.md` perlu tetap sinkron dengan implementasi kode. Disarankan untuk menambahkan pemeriksaan CI untuk memastikan bahwa variabel CSS tetap konsisten antara dokumentasi dan kode.

**Aksesibilitas**: Pastikan kontras warna memenuhi standar WCAG AA (setidaknya 4,5:1).

### Panduan Penggunaan AI

Setelah membuat `design.md`, bagaimana caranya agar AI benar-benar menggunakannya? Berikut beberapa tip praktis:

**Kiat 1: Referensikan secara eksplisit di prompt**

```
Please refer to the design.md file in the project root and use the design conventions defined there to implement the following components:
- Buttons: use the primary color with an 8px border radius
- Cards: use the elevation-2 shadow level
```

**Tips 2: Wajibkan AI untuk mereferensikan variabel CSS tertentu**

```
Implement a navigation bar with the following requirements:
- Use --color-bg-primary for the background color
- Use --color-border-subtle for borders
- Use --text-color-primary for text
```

**Kiat 3: Sertakan `design.md` konten di prompt sistem**

Jika alat AI Anda mendukung perintah sistem khusus, Anda dapat menambahkan konten inti `design.md` langsung ke sana.

### Strategi Pengujian

**Pengujian saluran konten**:
- Skenario file hilang (missing `README.md` atau `DESIGN.md`)
- Skenario kesalahan format (kegagalan penguraian penurunan harga)
- Skenario direktori kosong

**Pengujian fitur penelusuran**:
- Penanganan hasil kosong
- Karakter khusus seperti Cina dan emoji
- Verifikasi sinkronisasi URL

**Pengujian komponen UI**:
- Peralihan tema terang/gelap
- Tata letak responsif
- Pratinjau status pemuatan

### Alur Kerja Penerapan

```bash
# 1. Update the submodule to the latest version
git submodule update --remote

# 2. Rebuild the site
npm run build

# 3. Deploy static assets
npm run deploy
```

Disarankan untuk mengotomatiskan pembaruan submodul serta pembangunan dan penerapan, sehingga CI dapat terpicu secara otomatis setiap kali repositori upstream diperbarui.

## Kesimpulan

Inkonsistensi UI yang dihasilkan AI yang ditemui HagiCode selama pengembangan, pada intinya, disebabkan oleh kurangnya dokumen referensi desain terstruktur. Dengan membangun situs galeri desain dan membuat standar `design.md`, kami berhasil memecahkan masalah ini.

Nilai inti dari solusi ini terletak pada:
- **Sumber daya terpadu**: menggabungkan dokumentasi sistem desain yang tersebar
- **Standar terstruktur**: mengekspresikan konvensi desain dalam format yang dapat dipahami AI
- **Pemeliharaan berkelanjutan**: menjaga konten tetap mutakhir `git submodule`

Jika Anda juga menggunakan AI untuk pengembangan frontend, pendekatan ini patut untuk dicoba. Membuat terstruktur `design.md` tidak hanya meningkatkan konsistensi kode yang dihasilkan AI, tetapi juga membantu tim Anda mempertahankan standar desain terpadu secara internal.
