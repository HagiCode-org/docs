## Analisis Teknis Sistem Keterampilan HagiCode: Membangun Platform Manajemen Keterampilan AI yang Dapat Diskalakan

> Artikel ini membahas secara mendalam arsitektur dan implementasi sistem manajemen Keterampilan dalam proyek HagiCode, yang mencakup detail teknis di balik empat kemampuan inti: manajemen global lokal, pencarian pasar, rekomendasi cerdas, dan manajemen penyedia tepercaya.

## Latar Belakang

Di bidang asisten pengkodean AI, bagaimana memperluas batasan kemampuan AI selalu menjadi pertanyaan inti. Claude Code sendiri sudah kuat dalam bantuan kode, namun tim pengembangan yang berbeda dan tumpukan teknologi yang berbeda sering kali memerlukan kemampuan khusus untuk skenario tertentu, seperti menangani penerapan Docker, pengoptimalan database, atau pembuatan komponen frontend. Di sinilah sistem Keterampilan menjadi sangat penting.

Selama pengembangan proyek HagiCode, kami menghadapi tantangan serupa: bagaimana kita membiarkan Claude Code "mempelajari" keterampilan profesional baru seperti yang dilakukan seseorang, sambil tetap mempertahankan pengalaman pengguna yang solid dan pemeliharaan teknis yang baik? Masalah ini sulit dan sederhana dengan caranya sendiri. Sehubungan dengan pertanyaan tersebut, kami merancang dan menerapkan sistem manajemen Keterampilan yang lengkap.

Artikel ini membahas arsitektur teknis dan implementasi inti sistem secara mendetail. Hal ini ditujukan bagi pengembang yang tertarik dengan ekstensibilitas AI dan integrasi alat baris perintah. Ini mungkin berguna bagi Anda, atau mungkin tidak, tapi setidaknya sudah dituliskan sekarang.

## Tentang HagiCode

Pendekatan yang dibagikan dalam artikel ini berasal dari pengalaman praktis kami di [Kode Hagi](https://hagicode.com) proyek. HagiCode adalah proyek asisten pengkodean AI sumber terbuka yang dirancang untuk membantu tim pengembangan meningkatkan efisiensi teknik. Tumpukan proyek mencakup ASP.NET Core, kerangka terdistribusi Orleans, frontend TanStack Start + React, dan subsistem manajemen Keterampilan yang diperkenalkan dalam artikel ini.

Repositori GitHub adalah [HagiCode-org/site](https://github.com/HagiCode-org/site). Jika menurut Anda pendekatan teknis dalam artikel ini berharga, silakan beri Bintang. Lagipula, lebih banyak Bintang cenderung meningkatkan mood.

## Ikhtisar Arsitektur Sistem

Sistem Skill menggunakan arsitektur terpisah frontend-backend. Tidak ada sesuatu pun yang misterius tentang hal itu.

**Frontend** menggunakan TanStack Start + React untuk membangun antarmuka pengguna, dengan status pengelolaan Redux Toolkit. Empat kemampuan utama dipetakan langsung ke empat komponen Tab: Keterampilan Lokal, Galeri Keterampilan, Rekomendasi Cerdas, dan Penyedia Tepercaya. Pada akhirnya, desainnya sebagian besar bertujuan untuk membuat pengalaman pengguna menjadi lebih baik.

**Backend** didasarkan pada ASP.NET Core + ABP Framework, menggunakan Orleans Grain untuk pengelolaan status terdistribusi. Klien API online membungkus `IOnlineApiClient` antarmuka untuk berkomunikasi dengan layanan katalog keterampilan jarak jauh.

Prinsip arsitektur keseluruhannya adalah memisahkan eksekusi perintah dari logika bisnis. Melalui pola adaptor, detail implementasi eksekusi perintah npm/npx disembunyikan di dalam modul independen. Lagi pula, tidak ada seorang pun yang benar-benar menginginkan panggilan baris perintah tersebar di seluruh basis kode.

## Kemampuan Inti 1: Manajemen Lokal Global

Manajemen global lokal adalah modul paling dasar. Ini bertanggung jawab untuk membuat daftar keterampilan yang diinstal dan mendukung operasi penghapusan instalasi. Tidak ada yang terlalu rumit di sini; ini sebagian besar tentang melakukan hal-hal dasar dengan baik.

### Pendekatan Teknis

Implementasinya tetap ada `LocalSkillsTab.tsx` dan `LocalSkillCommandAdapter.cs`. Ide intinya adalah untuk membungkus `npx skills` perintah, parsing keluaran JSON-nya, dan ubah menjadi struktur data internal. Kedengarannya sederhana, dan dalam praktiknya sebagian besar memang demikian.

```csharp
public async Task<IReadOnlyList<LocalSkillInventoryResponseDto>> GetLocalSkillsAsync(
    CancellationToken cancellationToken = default)
{
    var result = await _commandAdapter.ListGlobalSkillsAsync(cancellationToken);
    return result.Skills.Select(skill => new LocalSkillInventoryResponseDto
    {
        Name = skill.Name,
        Version = skill.Version,
        Source = skill.Source,
        InstalledPath = skill.InstalledPath,
        Description = skill.Description
    }).ToList();
}
```

Aliran datanya sangat jelas: frontend mengirimkan permintaan -> `SkillGalleryAppService` menerimanya -> `LocalSkillCommandAdapter` menjalankan perintah npx -> hasil JSON diurai -> DTO dikembalikan. Setiap langkah mengikuti secara alami dari langkah sebelumnya.

Penghapusan keterampilan menggunakan `npx skills remove -g <skillName> -y` perintah, dan sistem secara otomatis menangani dependensi dan pembersihan. Metadata instalasi disimpan di `managed-install.json` di dalam direktori keterampilan, mencatat informasi seperti waktu instalasi dan versi sumber untuk pembaruan dan audit selanjutnya. Beberapa hal memang layak untuk dicatat.

### Alur Instalasi Secara Detail

Pemasangan keterampilan memerlukan beberapa langkah terkoordinasi. Sebenarnya, ini tidak terlalu rumit:

```csharp
public async Task<SkillInstallResultDto> InstallAsync(
    SkillInstallRequestDto request,
    CancellationToken cancellationToken = default)
{
    // 1. Normalize the installation reference
    var normalized = _referenceNormalizer.Normalize(
        request.SkillId,
        request.Source,
        request.SkillSlug,
        request.Version);

    // 2. Check prerequisites
    await _prerequisiteChecker.CheckAsync(cancellationToken);

    // 3. Acquire installation lock
    using var installLock = await _lockProvider.AcquireAsync(normalized.SkillId);

    // 4. Execute installation command
    var result = await _installCommandRunner.ExecuteAsync(
        new SkillInstallCommandExecutionRequest
        {
            Command = $"npx skills add {normalized.FullReference} -g -y",
            Timeout = TimeSpan.FromMinutes(4)
        },
        cancellationToken);

    // 5. Persist installation metadata
    await _metadataStore.WriteAsync(normalized.SkillPath, request);

    return new SkillInstallResultDto { Success = result.Success };
}
```

Beberapa pola desain utama digunakan di sini: **reference normalizer** mengonversi berbagai format input, seperti `tanweai/pua` dan `@opencode/docker-skill`, menjadi representasi internal yang terpadu; **mekanisme kunci instalasi** memastikan hanya satu operasi instalasi yang dapat dijalankan untuk keterampilan yang sama dalam satu waktu; dan **output streaming** mendorong progres instalasi ke frontend secara real-time melalui Peristiwa Terkirim Server, sehingga pengguna dapat melihat log seperti terminal saat terjadi.

Pada akhirnya, semua pola ini ada untuk satu tujuan: menjaga sistem agar lebih mudah digunakan dan dipelihara.

## Kemampuan Inti 2: Pencarian Pasar

Pencarian pasar memungkinkan pengguna menemukan dan menginstal keterampilan dari komunitas. Kemampuan satu orang selalu terbatas; pengetahuan kolektif melangkah lebih jauh.

### Pendekatan Teknis

Fitur pencarian mengandalkan API online `https://api.hagicode.com/v1/skills/search`. Untuk meningkatkan kecepatan respons, sistem menerapkan caching. Cache mirip dengan memori: jika Anda menyimpan hal-hal yang berguna, Anda tidak perlu berpikir keras di lain waktu.

```csharp
private async Task<IReadOnlyList<SkillGallerySkillDto>> SearchCatalogAsync(
    string query,
    CancellationToken cancellationToken,
    IReadOnlySet<string>? allowedSources = null)
{
    var cacheKey = $"skill_search:{query}:{string.Join(",", allowedSources ?? Array.Empty<string>())}";

    if (_memoryCache.TryGetValue(cacheKey, out var cached))
        return (IReadOnlyList<SkillGallerySkillDto>)cached!;

    var response = await _onlineApiClient.SearchAsync(
        new SearchSkillsRequest
        {
            Query = query,
            Limit = _options.LimitPerQuery,
        },
        cancellationToken);

    var results = response.Skills
        .Where(skill => allowedSources is null || allowedSources.Contains(skill.Source))
        .Select(skill => new SkillGallerySkillDto { ... })
        .ToList();

    _memoryCache.Set(cacheKey, results, TimeSpan.FromMinutes(10));
    return results;
}
```

Hasil pencarian mendukung pemfilteran berdasarkan sumber tepercaya, sehingga pengguna hanya melihat sumber keahlian yang mereka percayai. Kueri benih seperti `popular` dan `recent` digunakan untuk menginisialisasi katalog, memungkinkan pengguna melihat keterampilan populer yang direkomendasikan saat pertama kali mereka membukanya. Kesan pertama tetap penting.

## Kemampuan Inti 3: Rekomendasi Cerdas

Rekomendasi cerdas adalah bagian paling kompleks dari sistem. Mereka dapat secara otomatis merekomendasikan keterampilan yang paling sesuai berdasarkan konteks proyek saat ini. Meski rumit, namun masih layak untuk dibangun.

### Alur Rekomendasi

Alur rekomendasi lengkap dibagi menjadi lima tahap:

```
1. Build project context
   ↓
2. AI generates search queries
   ↓
3. Search the online catalog in parallel
   ↓
4. AI ranks the candidates
   ↓
5. Return the recommendation list
```

Pertama, sistem menganalisis karakteristik seperti tumpukan teknologi proyek, bahasa pemrograman, dan struktur domain untuk membangun “profil proyek”. Profil itu seperti resume, yang mencatat ciri-ciri utama proyek.

Kemudian AI Grain digunakan untuk menghasilkan kueri penelusuran yang ditargetkan. Desain ini sebenarnya cukup menarik: daripada bertanya langsung kepada AI, "Keterampilan apa yang harus saya rekomendasikan?", pertama-tama kita memintanya untuk memikirkan tentang "Istilah penelusuran apa yang mungkin menemukan keterampilan yang relevan?" Terkadang cara Anda mengajukan pertanyaan lebih penting daripada jawabannya sendiri:

```csharp
var queryGeneration = await aiGrain.GenerateSkillRecommendationQueriesAsync(
    projectContext,      // Project context
    locale,              // User language preference
    maxQueries,           // Maximum number of queries
    effectiveSearchHero); // AI model selection
```

Selanjutnya, permintaan pencarian tersebut dijalankan secara paralel untuk mengumpulkan daftar keterampilan kandidat. Pemrosesan paralel, pada akhirnya, hanyalah cara untuk menghemat waktu.

Terakhir, AI Grain lainnya memberi peringkat pada keterampilan kandidat. Langkah ini mempertimbangkan faktor-faktor seperti relevansi keterampilan dengan proyek, status kepercayaan, dan preferensi historis pengguna:

```csharp
var ranking = await aiGrain.RankSkillRecommendationsAsync(
    projectContext,
    candidates,
    installedSkillNames,
    locale,
    maxRecommendations,
    effectiveRankingHero);

response.Items = MergeRecommendations(projectContext, candidates, ranking, maxRecommendations);
```

### Mekanisme Penggantian

Model AI dapat merespons dengan lambat atau tidak tersedia untuk sementara. Bahkan sistem terbaik pun terkadang tersandung. Oleh karena itu, sistem ini menyertakan mekanisme fallback deterministik: ketika layanan AI tidak tersedia, sistem ini menggunakan algoritma heuristik berbasis aturan untuk menghasilkan rekomendasi, seperti menyimpulkan kemungkinan keterampilan yang diperlukan dari ketergantungan dalam `package.json`.

Sederhananya, mekanisme fallback ini hanyalah sebuah rencana cadangan untuk sistem.

## Kemampuan Inti 4: Manajemen Penyedia Tepercaya

Manajemen penyedia tepercaya memungkinkan pengguna mengontrol sumber keterampilan mana yang dianggap dapat dipercaya. Kepercayaan masih merupakan sesuatu yang harus dapat didefinisikan sendiri oleh pengguna.

### Aturan Pencocokan

Penyedia tepercaya mendukung dua aturan pencocokan: **pencocokan persis** (`exact`) dan **pencocokan awalan** (`prefix`).

```csharp
public static TrustedSkillProviderResolutionSnapshot Resolve(
    TrustedSkillProviderSnapshot snapshot,
    string source)
{
    var normalizedSource = Normalize(source);

    foreach (var entry in snapshot.Entries.OrderBy(e => e.SortOrder))
    {
        if (!entry.IsEnabled) continue;

        foreach (var rule in entry.MatchRules)
        {
            bool isMatch = rule.MatchType switch
            {
                TrustedSkillProviderMatchRuleType.Exact
                    => string.Equals(normalizedSource, Normalize(rule.Value),
                        StringComparison.OrdinalIgnoreCase),
                TrustedSkillProviderMatchRuleType.Prefix
                    => normalizedSource.StartsWith(Normalize(rule.Value) + "/",
                        StringComparison.OrdinalIgnoreCase),
                _ => false
            };

            if (isMatch)
                return new TrustedSkillProviderResolutionSnapshot
                {
                    IsTrustedSource = true,
                    ProviderId = entry.ProviderId,
                    DisplayName = entry.DisplayName
                };
        }
    }

    return new TrustedSkillProviderResolutionSnapshot { IsTrustedSource = false };
}
```

Penyedia tepercaya bawaan mencakup organisasi dan proyek terkenal seperti Vercel, Azure, anthropics, Microsoft, dan penggunaan browser. Penyedia khusus dapat ditambahkan melalui file konfigurasi dengan menentukan ID penyedia, nama tampilan, label lencana, aturan pencocokan, dan banyak lagi. Dunia ini cukup besar sehingga hanya mempercayai beberapa hal saja tidak akan pernah cukup.

### Implementasi Kegigihan

Konfigurasi tepercaya dipertahankan menggunakan Orleans Grain:

```csharp
public class TrustedSkillProviderGrain : Grain<TrustedSkillProviderState>,
    ITrustedSkillProviderGrain
{
    public async Task UpdateConfigurationAsync(TrustedSkillProviderSnapshot snapshot)
    {
        State.Snapshot = snapshot;
        await WriteStateAsync();
    }

    public Task<TrustedSkillProviderSnapshot> GetConfigurationAsync()
    {
        return Task.FromResult(State.Snapshot);
    }
}
```

Manfaat dari pendekatan ini adalah perubahan konfigurasi secara otomatis disinkronkan di semua node, tanpa perlu menyegarkan cache secara manual. Otomatisasi pada akhirnya adalah untuk mengurangi kekhawatiran orang.

## Desain Teknis Utama

### Pola Adaptor Eksekusi Perintah

Sistem Skill perlu menjalankan berbagai perintah npx. Jika logika tersebut tersebar dimana-mana, kode tersebut akan dengan cepat menjadi sulit untuk dipelihara. Itu sebabnya kami merancang antarmuka adaptor. Pola desain, pada akhirnya, ada untuk membuat kode lebih mudah dipelihara:

```csharp
public interface ISkillInstallCommandRunner
{
    Task<SkillInstallCommandExecutionResult> ExecuteAsync(
        SkillInstallCommandExecutionRequest request,
        CancellationToken cancellationToken = default);
}
```

Perintah yang berbeda memiliki implementasi eksekutor yang berbeda, namun semuanya mengimplementasikan antarmuka yang sama, sehingga pengujian dan penggantian menjadi mudah.

### Keluaran Streaming SSE

Kemajuan instalasi didorong ke frontend secara real time melalui Server-Sent Events:

```csharp
public async Task InstallWithProgressAsync(
    SkillInstallRequestDto request,
    IServerStreamWriter<SkillInstallProgressEventDto> stream,
    CancellationToken cancellationToken)
{
    var process = new Process
    {
        StartInfo = new ProcessStartInfo
        {
            FileName = "npx",
            Arguments = $"skills add {request.FullReference} -g -y",
            RedirectStandardOutput = true,
            RedirectStandardError = true,
            UseShellExecute = false
        }
    };

    process.OutputDataReceived += async (sender, e) =>
    {
        await stream.WriteAsync(new SkillInstallProgressEventDto
        {
            EventType = "output",
            Data = e.Data ?? string.Empty
        });
    };

    process.Start();
    process.BeginOutputReadLine();
    await process.WaitForExitAsync(cancellationToken);
}
```

Di frontend, pengguna dapat melihat keluaran seperti terminal secara real time, yang menjadikan pengalamannya sangat intuitif. Umpan balik real-time membantu orang merasa nyaman.

## Panduan Praktis

### Memasang Keterampilan Komunitas

Ambil instalasi `pua` keterampilan sebagai contoh (ini adalah keterampilan komunitas yang populer):

1. Buka laci Keterampilan dan alihkan ke `Skill Gallery` tab
2. Masuk `pua` di kotak pencarian
3. Klik hasil pencarian untuk melihat detail keterampilan
4. Klik `Install` tombol
5. Beralih ke `Local Skills` tab untuk mengonfirmasi instalasi berhasil

Perintah instalasinya adalah `npx skills add tanweai/pua -g -y`, dan sistem menangani semua detail secara otomatis. Sebenarnya tidak banyak langkah setelah Anda melakukannya satu per satu.

### Menambahkan Sumber Tepercaya Khusus

Jika tim Anda memiliki gudang keahliannya sendiri, Anda dapat menambahkannya sebagai sumber tepercaya:

```yaml
providerId: "my-team"
displayName: "My Team Skills"
badgeLabel: "MyTeam"
isEnabled: true
sortOrder: 100
matchRules:
  - matchType: "prefix"
    value: "my-team/"
  - matchType: "exact"
    value: "my-team/special-skill"
```

Dengan cara ini, semua keterampilan dari tim Anda akan menampilkan lencana tepercaya, sehingga membuat pengguna lebih nyaman memasangnya. Label dan sinyal memang membantu orang merasa lebih percaya diri.

### Dasar-dasar Pengembangan Keterampilan

Membuat keterampilan khusus memerlukan struktur berikut:

```
my-skill/
├── SKILL.md          # Skill metadata (YAML front matter)
├── index.ts          # Skill entry point
├── agents/           # Supported agent configuration
└── references/       # Reference resources
```

Sebuah contoh `SKILL.md` format:

```yaml
---
name: my-skill
description: A brief description of what this skill does
---

# My Skill

Detailed documentation...
```

## Catatan

1. **Persyaratan jaringan**: pencarian dan pemasangan keterampilan memerlukan akses `api.hagicode.com` dan registri npm
2. **Versi Node.js**: Direkomendasikan Node.js 18 atau lebih baru
3. **Persyaratan izin**: izin instalasi npm global diperlukan
4. **Kontrol konkurensi**: hanya satu operasi pemasangan atau pencopotan pemasangan yang dapat dijalankan untuk keterampilan yang sama dalam satu waktu
5. **Pengaturan batas waktu**: batas waktu default untuk instalasi adalah 4 menit, namun skenario yang rumit mungkin memerlukan penyesuaian

Catatan-catatan ini pada akhirnya ada untuk membantu segala sesuatunya berjalan lancar.
