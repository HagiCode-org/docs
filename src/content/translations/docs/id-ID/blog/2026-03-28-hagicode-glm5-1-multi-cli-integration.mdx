## Latar Belakang

Dalam proyek Hagicode, pengguna dapat memilih dari beberapa alat CLI untuk menggerakkan asisten pemrograman AI, termasuk Claude Code CLI, GitHub Copilot, OpenCode CLI, Codebuddy CLI, Hermes CLI, dan banyak lagi. Alat CLI ini merupakan alat pemrograman AI untuk keperluan umum, tetapi melalui lapisan abstraksi Hagicode, alat tersebut dapat secara fleksibel terhubung ke penyedia model AI yang berbeda.

Zhipu AI (ZAI) menyediakan antarmuka yang kompatibel dengan Anthropic Claude API, memungkinkan alat CLI ini untuk langsung menggunakan model seri GLM domestik. Diantaranya, GLM-5.1 adalah rilis model bahasa besar terbaru dari Zhipu, dengan peningkatan signifikan dibandingkan GLM-5.0.

## Arsitektur abstraksi CLI Hagicode

Hagicode mendefinisikan 11 tipe penyedia CLI melalui `AIProviderType` enum, mencakup alat CLI pemrograman AI arus utama:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI
}
```

Setiap CLI memiliki konfigurasi parameter model yang sesuai dan mendukung `model` dan `reasoning` parameter:

```csharp
private static readonly IReadOnlyDictionary<AIProviderType, IReadOnlyList<string>> ManagedModelParameterKeysByProvider =
    new Dictionary<AIProviderType, IReadOnlyList<string>>
    {
        [AIProviderType.ClaudeCodeCli] = ["model", "reasoning"],
        [AIProviderType.CodexCli] = ["model", "reasoning"],
        [AIProviderType.OpenCodeCli] = ["model", "reasoning"],
        [AIProviderType.HermesCli] = ["model", "reasoning"],
        [AIProviderType.CodebuddyCli] = ["model", "reasoning"],
        [AIProviderType.QoderCli] = ["model", "reasoning"],
        [AIProviderType.KiroCli] = ["model", "reasoning"],
        [AIProviderType.GeminiCli] = ["model"],  // Gemini does not support the reasoning parameter
        // ...
    };
```

## Menggunakan GLM-5.1 di CLI yang berbeda

### 1. Kode Claude CLI + GLM-5.1

Claude Code CLI adalah salah satu CLI inti Hagicode dan dikonfigurasi melalui sistem konfigurasi Hero:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

Sesuai `HeroEquipmentCatalogItem` konfigurasi:

```typescript
{
  id: 'secondary-glm-5-1',
  name: 'GLM 5.1',
  family: 'anthropic',
  kind: 'model',
  primaryFamily: 'claude',
  compatiblePrimaryFamilies: ['claude', 'codebuddy', 'hermes', 'qoder', 'kiro'],
  defaultParameters: {
    model: 'glm-5.1',
    reasoning: 'high'
  }
}
```

### 2. OpenCode CLI + GLM-5.1

OpenCode CLI adalah CLI paling fleksibel dan mendukung penentuan model apa pun di `provider/model` format:

**Metode 1: Gunakan awalan penyedia ZAI**
```json
{
  "primaryProfessionId": "profession-opencode",
  "model": "zai/glm-5.1",
  "reasoning": "high"
}
```

**Metode 2: Gunakan ID model secara langsung**
```json
{
  "model": "glm-5.1"
}
```

**Metode 3: UI konfigurasi frontend**

Di `HeroModelEquipmentForm.tsx`, OpenCode CLI memiliki petunjuk placeholder khusus:

```tsx
const OPEN_CODE_MODEL_PLACEHOLDER = 'myprovider/glm-4.7';

const modelPlaceholder = primaryProviderType === PCode_Models_AIProviderType.OPEN_CODE_CLI
  ? OPEN_CODE_MODEL_PLACEHOLDER
  : 'gpt-5.4';
```

Pengguna dapat memasukkan:
```
zai/glm-5.1
glm-5.1
```

**Logika penguraian model OpenCode CLI**:

```csharp
internal OpenCodeModelSelection? ResolveModelSelection(string? rawModel)
{
    var normalized = NormalizeOptionalValue(rawModel);
    if (normalized == null) return null;

    var slashIndex = normalized.IndexOf('/');
    if (slashIndex < 0)
    {
        // No slash: use the model ID directly
        return new OpenCodeModelSelection {
            ProviderId = string.Empty,
            ModelId = normalized,
        };
    }

    // Slash exists: parse the provider/model format
    var providerId = normalized[..slashIndex].Trim();
    var modelId = normalized[(slashIndex + 1)..].Trim();

    return new OpenCodeModelSelection {
        ProviderId = providerId,
        ModelId = modelId,
    };
}
```

### 3. Teman Kode CLI + GLM-5.1

Codebuddy CLI telah mendedikasikan logika penanganan lama:

```json
{
  "primaryProfessionId": "profession-codebuddy",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

**Catatan**: Codebuddy mempertahankan penanganan khusus untuk GLM-5.0 dan tidak menggunakan normalisasi lama:

```csharp
return !string.Equals(providerName, "CodebuddyCli", StringComparison.OrdinalIgnoreCase)
       && string.Equals(normalizedModel, LegacyGlm5TurboModelValue, StringComparison.OrdinalIgnoreCase)
    ? Glm5TurboModelValue
    : normalizedModel;
// For CodebuddyCli, glm-5.0 is not normalized to glm-5-turbo
```

## Keuntungan peningkatan GLM-5.1

Dibandingkan dengan GLM-5.0, GLM-5.1 menghadirkan peningkatan signifikan berikut ini:

### 1. Kemampuan penalaran yang lebih baik

Menurut informasi rilis resmi Zhipu, peningkatan pada GLM-5.1 meliputi:

- **Pemahaman kode yang lebih kuat**: Analisis yang lebih akurat terhadap struktur kode yang kompleks
- **Pemahaman konteks yang lebih panjang**: Mendukung konteks percakapan yang lebih panjang
- **Panggilan alat yang ditingkatkan**: Tingkat keberhasilan yang lebih tinggi untuk panggilan alat MCP
- **Stabilitas keluaran**: Mengurangi keacakan dan halusinasi

### 2. Kompatibilitas multi-CLI yang komprehensif

GLM-5.1 mencakup semua CLI mainstream yang didukung oleh Hagicode:

```
compatiblePrimaryFamilies: [
  "claude",      // Claude Code CLI
  "codebuddy",   // Codebuddy CLI
  "hermes",      // Hermes CLI
  "qoder",       // Qoder CLI
  "kiro"         // Kiro CLI
]
```

## Catatan

### 1. Konfigurasi kunci API

Pastikan `ANTHROPIC_AUTH_TOKEN` variabel lingkungan diatur dengan benar. Ini adalah kredensial yang diperlukan setiap CLI untuk terhubung ke model.

### 2. Ketersediaan model

GLM-5.1 harus diaktifkan oleh penyedia model yang sesuai:
- Platform Zhipu AI ZAI mendukungnya secara default
- Alibaba Cloud DashScope mungkin memerlukan aplikasi terpisah

### 3. Format OpenCode CLI

Saat menggunakan `provider/model` format, pastikan ID penyedia sudah benar:
- Zhipu AI: `zai` atau `zhipuai`
- Alibaba Awan: `aliyun` atau `dashscope`

### 4. Parameter penalaran

- `high` direkomendasikan untuk hasil pembuatan kode terbaik
- Gemini CLI tidak mendukung parameter penalaran dan akan mengabaikan konfigurasi ini secara otomatis

## Kesimpulan

Melalui lapisan abstraksi terpadu, Hagicode memungkinkan integrasi fleksibel antara GLM-5.1 dan beberapa CLI. Pengembang dapat memilih alat CLI yang paling sesuai dengan preferensi dan skenario penggunaan mereka, kemudian menggunakan model GLM-5.1 terbaru melalui konfigurasi sederhana.

Sebagai versi model terbaru Zhipu, GLM-5.1 menawarkan peningkatan yang jelas dibandingkan GLM-5.0:
- Pengidentifikasi versi independen tanpa beban warisan
- Penalaran dan pemahaman kode yang lebih kuat
- Kompatibilitas multi-CLI yang luas
- Konfigurasi tingkat penalaran yang fleksibel

Dengan variabel lingkungan yang benar dan konfigurasi peralatan Hero, pengguna dapat sepenuhnya memanfaatkan kekuatan GLM-5.1 di berbagai lingkungan CLI.

## Lanjutkan Dengan HagiCode

Jika Anda ingin menerapkan GLM-5.1, orkestrasi multi-CLI, dan model konfigurasi HagiCode, berikut adalah titik masuk tercepat:

- Lacak proyek utama dan kemajuan implementasi terbaru di GitHub: [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site)

- Jika Anda lebih menyukai alur kerja desktop lokal, mulailah dengan titik masuk Desktop: [hagicode.com/desktop/](https://hagicode.com/desktop/)

Setelah Anda membandingkan Kimi, Claude Code, OpenCode, dan CLI lainnya di dalam lapisan abstraksi yang sama, pertanyaan tentang peralihan model, pemetaan parameter, dan batasan teknik cenderung menjadi lebih mudah untuk dipikirkan.
