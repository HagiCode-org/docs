## Mengetik Lebih Lambat Dibandingkan Berbicara, dan Berbicara Lebih Lambat Dibandingkan Tangkapan Layar - Praktik Input Multimodal untuk Asisten Pengkodean AI

> Menulis kode memiliki batas kecepatan tidak peduli seberapa cepat Anda mengetik. Terkadang sesuatu yang bisa Anda ucapkan dalam satu kalimat membutuhkan waktu lama untuk diketik; terkadang satu tangkapan layar menjelaskan segalanya, namun Anda tetap harus mendeskripsikannya dengan setumpuk teks. Artikel ini membahas tentang apa yang kami temui saat membangun [Kode Hagi](https://hagicode.com), dari pengenalan suara hingga unggahan gambar. Pada akhirnya, kami hanya ingin membuat asisten pengkodean AI sedikit lebih mudah digunakan.

## Latar Belakang

Saat membangun HagiCode, kami melihat adanya masalah - atau lebih tepatnya, masalah yang muncul secara alami begitu orang mulai sering menggunakannya: mengandalkan mengetik saja bisa melelahkan.

Coba pikirkan: interaksi antara pengguna dan Agen adalah skenario inti. Namun jika setiap pertukaran memerlukan pengetikan tanpa henti di keyboard, efisiensinya tidak bagus:

1. **Mengetik terlalu lambat**: Untuk masalah rumit, seperti pesan error atau masalah UI, mengetik semuanya bisa memakan waktu setengah menit, sedangkan mengucapkannya dengan lantang mungkin butuh waktu sepuluh detik. Kesenjangan itu nyata.

2. **Gambar lebih langsung**: Terkadang UI memunculkan kesalahan, terkadang Anda ingin membandingkan draf desain, dan terkadang Anda perlu menampilkan struktur kode. "Sebuah gambar bernilai ribuan kata" mungkin merupakan pepatah lama, namun masih tetap berlaku. Membiarkan AI secara langsung "melihat" masalahnya jauh lebih jelas daripada menjelaskannya dalam waktu lama.

3. **Interaksi harus terasa alami**: Asisten AI modern harus mendukung teks, suara, dan gambar. Pengguna harus dapat memilih metode masukan mana yang dirasa paling alami.

Jadi kami memutuskan untuk menambahkan dukungan pengenalan suara dan pengunggahan gambar ke HagiCode untuk membuat interaksi Agen lebih nyaman. Jika pengguna bisa mengetik lebih sedikit, itu sudah merupakan sebuah kemenangan.

## Tentang HagiCode

Solusi yang dibagikan dalam artikel ini berasal dari kerja langsung kami di [Kode Hagi](https://hagicode.com) proyek - atau, lebih tepatnya, dari pembelajaran saat tersandung melalui beberapa kendala.

HagiCode adalah proyek asisten pengkodean AI sumber terbuka dengan tujuan sederhana: menggunakan AI untuk meningkatkan efisiensi pengembangan. Seiring kami terus mengembangkannya, menjadi jelas bahwa pengguna sangat menginginkan masukan multimoda. Terkadang mengucapkan satu kalimat lebih cepat daripada mengetik paragraf yang panjang, dan terkadang tangkapan layar jauh lebih jelas daripada penjelasan yang panjang.

Kebutuhan tersebut mendorong kami untuk terus maju, dan dari sinilah fitur-fitur seperti pengenalan suara dan upload gambar akhirnya terbentuk. Pengguna kini dapat berinteraksi dengan AI dengan cara paling alami yang tersedia bagi mereka, dan itu terasa menyenangkan.

## Analisis

### Tantangan Teknis dalam Pengenalan Ucapan

Saat membuat pengenalan ucapan, kami mengalami masalah rumit: **Browser WebSocket API tidak mendukung header HTTP khusus**.

Layanan pengenalan ucapan yang kami pilih adalah Doubao Speech Recognition API dari ByteDance. Sayangnya, API ini memerlukan informasi otentikasi seperti `accessToken` dan `secretKey` untuk dilewatkan melalui header HTTP. Hal ini langsung menimbulkan konflik teknis:

```javascript
// The browser WebSocket API does not support this approach
const ws = new WebSocket('wss://api.com/ws', {
  headers: {
    'Authorization': 'Bearer token'  // Not supported
  }
});
```

Kami pada dasarnya memiliki dua pilihan:

1. **Pendekatan parameter kueri URL**: masukkan info autentikasi di URL
   - Keuntungannya adalah implementasinya mudah
   - Kelemahannya adalah kredensial diekspos ke frontend, yang tidak aman; beberapa API juga memerlukan autentikasi berbasis header

2. **Pendekatan proksi backend**: menerapkan proksi WebSocket di backend
   - Keuntungannya adalah kredensial tetap disimpan dengan aman di backend dan solusinya sepenuhnya kompatibel dengan persyaratan API
   - Kelemahannya adalah implementasinya sedikit lebih rumit

Pada akhirnya, kami memilih **pendekatan proxy backend**. Keamanan bukanlah sesuatu yang Anda kompromikan.

### Persyaratan Fungsional untuk Upload Gambar

Persyaratan kami untuk mengunggah gambar sebenarnya cukup mudah:

1. **Beberapa metode unggah**: klik untuk memilih file, seret dan lepas, lalu tempel dari papan klip
2. **Validasi file**: batasan jenis (PNG, JPG, WebP, GIF) dan batas ukuran (5-10 MB) adalah persyaratan dasar
3. **Pengalaman pengguna**: unggah kemajuan, pratinjau, dan pesan kesalahan sehingga pengguna selalu tahu apa yang terjadi
4. **Keamanan**: validasi sisi server dan perlindungan terhadap unggahan file berbahaya sangat penting

## Solusi

### Pengenalan Ucapan: Arsitektur Proxy WebSocket

Kami merancang arsitektur tiga lapis untuk pengenalan suara dan menemukan jalur yang berhasil:

```
Browser WebSocket
       |
       | ws://backend/api/voice/ws
       | (binary audio)
       v
Backend Proxy
       |
       | wss://openspeech.bytedance.com/ (with auth header)
       v
Doubao API
```

**Implementasi komponen inti**:

1. **Prosesor AudioWorklet Frontend**:

```javascript
class AudioProcessorWorklet extends AudioWorkletProcessor {
  process(inputs, outputs, parameters) {
    const input = inputs[0]?.[0];
    if (!input) return true;

    // Resample to 16 kHz (required by the Doubao API)
    const samples = this.resampleAudio(input, 48000, 16000);

    // Accumulate samples into 500 ms chunks
    this.accumulatedSamples.push(...samples);

    if (this.accumulatedSamples.length >= 8000) {
      // Convert to 16-bit PCM and send
      const pcm = this.floatToPcm16(this.accumulatedSamples);
      this.port.postMessage({ type: 'audioData', data: pcm.buffer }, [pcm.buffer]);
      this.accumulatedSamples = [];
    }
    return true;
  }
}
```

2. **Pengendali WebSocket Backend** (C#):

```csharp
[HttpGet("ws")]
public async Task GetWebSocket()
{
    if (HttpContext.WebSockets.IsWebSocketRequest)
    {
        await _webSocketHandler.HandleAsync(HttpContext);
    }
}
```

3. **Bagian depan `VoiceTextArea` komponen**:

```tsx
export const VoiceTextArea = forwardRef<HTMLTextAreaElement, VoiceTextAreaProps>(
  ({ value, onChange, onTextRecognized, maxDuration }, ref) => {
    const { isRecording, interimText, volume, duration, startRecording, stopRecording } =
      useVoiceRecording({ onTextRecognized, maxDuration });

    return (
      <div className="flex gap-2">
        {/* Voice button */}
        <button onClick={handleButtonClick}>
          {isRecording ? <VolumeWaveform volume={volume} /> : <Mic />}
        </button>
        {/* Text input area */}
        <textarea value={displayValue} onChange={handleChange} />
      </div>
    );
  }
);
```

### Unggahan Gambar: Komponen Unggah Multi-Metode

Kami membuat komponen pengunggahan gambar berfitur lengkap dengan dukungan untuk ketiga metode pengunggahan, yang mencakup skenario paling umum yang dihadapi pengguna.

**Fitur inti**:

1. **Tiga metode pengunggahan**:

```tsx
// Click to upload
const handleClick = () => fileInputRef.current?.click();

// Drag-and-drop upload
const handleDrop = (e: React.DragEvent) => {
  const file = e.dataTransfer.files?.[0];
  if (file) uploadFile(file);
};

// Clipboard paste
const handlePaste = (e: ClipboardEvent) => {
  for (const item of Array.from(e.clipboardData?.items || [])) {
    if (item.type.startsWith('image/')) {
      const file = item.getAsFile();
      if (file) uploadFile(file);
    }
  }
};
```

2. **Validasi tampilan depan**:

```tsx
const validateFile = (file: File): { valid: boolean; error?: string } => {
  if (!acceptedTypes.includes(file.type)) {
    return { valid: false, error: 'Only PNG, JPG, JPEG, WebP, and GIF images are allowed' };
  }
  if (file.size > maxSize) {
    return { valid: false, error: `Maximum file size is ${(maxSize / 1024 / 1024).toFixed(1)}MB` };
  }
  return { valid: true };
};
```

3. **Penanganan unggahan backend** (TypeScript):

```typescript
export const Route = createFileRoute('/api/upload')({
  server: {
    handlers: {
      POST: async ({ request }) => {
        const formData = await request.formData();
        const file = formData.get('file') as File;

        // Validation
        const validation = validateFile(file);
        if (!validation.isValid) {
          return Response.json({ error: validation.error }, { status: 400 });
        }

        // Save file
        const uuid = uuidv4();
        const filePath = join(uploadDir, `${uuid}${extension}`);
        await writeFile(filePath, buffer);

        return Response.json({ url: `/uploaded/${today}/${uuid}${extension}` });
      }
    }
  }
});
```

## Panduan Praktis

### Cara Menggunakan Pengenalan Ucapan

1. **Konfigurasi layanan pengenalan suara**:
   - Buka halaman pengaturan pengenalan suara
   - Konfigurasikan Pidato Doubao `AppId` dan `AccessToken`
   - Konfigurasikan kata cepat secara opsional untuk meningkatkan akurasi pengenalan untuk istilah khusus domain

2. **Gunakan di kotak masukan**:
   - Klik ikon mikrofon di sisi kiri kotak masukan
   - Mulailah berbicara setelah animasi bentuk gelombang muncul
   - Klik ikon itu lagi untuk berhenti merekam
   - Teks yang dikenali secara otomatis disisipkan pada posisi kursor

3. **Contoh konfigurasi kata cepat**:

```text
TypeScript
React
useState
useEffect
```

### Cara Menggunakan Unggahan Gambar

1. **Metode unggah**:
   - Klik tombol unggah untuk memilih file
   - Seret gambar langsung ke area unggah
   - Gunakan `Ctrl+V` untuk menempelkan tangkapan layar dari clipboard

2. **Format yang didukung**: PNG, JPG, JPEG, WebP, GIF
3. **Batas ukuran**: 5 MB secara default (dapat dikonfigurasi)

### Catatan

1. **Pengenalan ucapan**:
   - Izin mikrofon diperlukan
   - Gunakan di lingkungan yang tenang jika memungkinkan
   - Durasi perekaman maksimum yang didukung adalah 300 detik secara default (dapat dikonfigurasi)

2. **Unggahan gambar**:
   - Hanya format gambar umum yang didukung
   - Perhatikan batasan ukuran file
   - Gambar yang diunggah secara otomatis menerima URL pratinjau

3. **Pertimbangan keamanan**:
   - Kredensial pengenalan ucapan disimpan di backend
   - Pengunggahan gambar melalui validasi sisi server yang ketat
   - HTTPS/WSS direkomendasikan di lingkungan produksi

## Kesimpulan

Setelah menambahkan pengenalan suara dan unggahan gambar, pengalaman pengguna HagiCode meningkat secara nyata. Pengguna kini dapat berinteraksi dengan AI dengan cara yang lebih alami - berbicara daripada mengetik, dan berbagi tangkapan layar daripada menjelaskan semuanya secara manual. Rasanya seperti akhirnya menemukan cara yang lebih nyaman untuk berkomunikasi.

Saat membuat fitur ini, kami mengalami masalah karena API WebSocket browser tidak mendukung header khusus. Pada akhirnya, kami menyelesaikannya dengan pendekatan proxy backend. Solusi tersebut tidak hanya menjaga keamanan, namun juga meletakkan dasar untuk mengintegrasikan layanan WebSocket terautentikasi lainnya di kemudian hari.

Komponen pengunggahan gambar juga mendapat manfaat dari dukungan berbagai metode pengunggahan, sehingga pengguna dapat memilih apa pun yang paling nyaman pada saat itu. Mengklik, menyeret, atau menempelkan semua pekerjaan, dan setiap jalur menyelesaikan pekerjaan dengan cepat.

"Mengetik lebih lambat dibandingkan berbicara, dan berbicara lebih lambat dibandingkan mengambil tangkapan layar" sangat sesuai dengan tema di sini. Jika Anda sedang membuat produk asisten AI serupa, saya harap pengalaman ini dapat membantu, meskipun hanya sedikit.
