## Dukungan Penuh GLM-5.1 dan Integrasi Gemini CLI: Jalur Evolusi Multi-Model HagiCode

> Artikel ini memperkenalkan dua pembaruan besar terkini pada platform HagiCode: dukungan penuh untuk model Zhipu AI GLM-5.1 dan keberhasilan integrasi Gemini CLI sebagai Agen CLI kesepuluh. Bersama-sama, pembaruan ini semakin memperkuat kemampuan multi-model platform dan ekosistem multi-CLI.

## Latar Belakang

Waktu benar-benar bergerak cepat. Perkembangan model bahasa besar meningkat seperti bambu di musim semi. Belum lama ini, kami masih mendukung "AI yang dapat menulis kode". Saat ini kita sudah berada di era kolaborasi multi-model dan integrasi multi-alat. Apakah itu menarik? Mungkin. Lagi pula, yang dibutuhkan pengembang bukan sekadar alat itu sendiri, namun kemudahan beradaptasi dengan berbagai skenario dan beralih secara fleksibel bila diperlukan.

Sebagai platform pengkodean yang didukung AI, HagiCode baru-baru ini menyambut baik dua perkembangan penting: pertama, integrasi penuh model GLM-5.1 Zhipu AI; kedua, penambahan resmi Gemini CLI sebagai Agen CLI kesepuluh yang didukung. Kedua pembaruan ini mungkin tidak terdengar menggemparkan, namun tidak diragukan lagi merupakan kabar baik bagi kematangan platform yang berkelanjutan.

GLM-5.1 adalah model andalan terbaru Zhipu AI. Dibandingkan dengan GLM-5.0, ia menawarkan penalaran yang lebih kuat, pemahaman kode yang lebih mendalam, dan pemanggilan alat yang lebih lancar. Yang lebih penting lagi, ini adalah model GLM pertama yang mendukung input gambar. Maksudnya itu apa? Artinya, pengguna dapat membiarkan AI melihat langsung ke tangkapan layar alih-alih kesulitan menjelaskan masalahnya dengan kata-kata. Setelah Anda menggunakan kemudahan itu, Anda segera memahami nilainya.

Pada saat yang sama, melalui `HagiCode.Libs.Providers` arsitektur, HagiCode berhasil mengintegrasikan Gemini CLI ke dalam platform. Ini sekarang menjadi Agen CLI yang kesepuluh. Sejujurnya, mencapai titik ini memang membawa rasa pencapaian yang sederhana.

Perlu juga disebutkan bahwa fitur unggah gambar HagiCode memungkinkan pengguna berkomunikasi dengan AI secara langsung melalui tangkapan layar. Bahkan saat menjalankan GLM 4.7, platform ini masih berfungsi dengan baik dan telah membantu menyelesaikan banyak tugas pembangunan yang penting. Adapun GLM-5.1, tentu saja, selangkah lebih maju.

## Tentang HagiCode

Pendekatan yang dibagikan dalam artikel ini berasal dari pengalaman praktis kami di [Kode Hagi](https://hagicode.com) proyek. HagiCode adalah platform pengkodean berbantuan AI sumber terbuka yang dirancang untuk memberi pengembang asisten pemrograman AI yang fleksibel dan kuat melalui arsitektur multi-model dan multi-CLI. Repositori proyek: [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site)

## Desain arsitektur multi-CLI

Salah satu kekuatan inti HagiCode adalah dukungannya terhadap beberapa alat CLI pemrograman AI melalui lapisan abstraksi terpadu. Keuntungan dari desain ini sebenarnya cukup sederhana: alat-alat baru bisa masuk, alat-alat lama bisa tetap ada, dan basis kode tidak berubah menjadi kekacauan. Agar adil, setiap orang ingin kehidupan berjalan seperti itu.

### enum AIProviderType

Platform ini menentukan jenis penyedia CLI yang didukung melalui `AIProviderType` enum:

```csharp
public enum AIProviderType
{
    ClaudeCodeCli = 0,    // Claude Code CLI
    CodexCli = 1,          // GitHub Copilot Codex
    GitHubCopilot = 2,     // GitHub Copilot
    CodebuddyCli = 3,     // Codebuddy CLI
    OpenCodeCli = 4,      // OpenCode CLI
    IFlowCli = 5,         // IFlow CLI
    HermesCli = 6,        // Hermes CLI
    QoderCli = 7,         // Qoder CLI
    KiroCli = 8,          // Kiro CLI
    KimiCli = 9,          // Kimi CLI
    GeminiCli = 10,       // Gemini CLI (new)
}
```

Seperti yang Anda lihat, Gemini CLI bergabung dengan keluarga ini sebagai anggota kesepuluh. Setiap CLI memiliki karakteristik dan skenario penggunaan yang berbeda, sehingga pengguna dapat memilih secara fleksibel berdasarkan kebutuhannya. Lagi pula, banyak jalan menuju Roma; beberapa lebih mudah dari yang lain.

### Arsitektur penyedia

`HagiCode.Libs.Providers` menyediakan antarmuka Penyedia terpadu yang membuat setiap integrasi CLI terstandarisasi dan ringkas. Mengambil Gemini CLI sebagai contoh:

```csharp
public class GeminiProvider : ICliProvider<GeminiOptions>
{
    private static readonly string[] DefaultExecutableCandidates = ["gemini", "gemini-cli"];
    private const string ManagedBootstrapArgument = "--acp";

    public string Name => "gemini";
    public bool IsAvailable => _executableResolver.ResolveFirstAvailablePath(DefaultExecutableCandidates) is not null;
}
```

Keuntungan dari desain ini adalah:
- Mengintegrasikan CLI baru hanya memerlukan penerapan satu kelas Penyedia
- Manajemen siklus hidup terpadu dan pengumpulan sesi
- Resolusi alias otomatis dan penemuan yang dapat dieksekusi

Sederhananya, desain ini mengubah hal-hal rumit menjadi lebih sederhana dan membuat hidup lebih mudah.

### Registri Penyedia

Registri Penyedia secara otomatis menangani pemetaan dan pendaftaran alias:

```csharp
if (provider is GeminiProvider)
{
    registry.Register(provider.Name, provider, ["gemini-cli"]);
    continue;
}
```

Ini berarti pengguna dapat memanggil Gemini CLI dengan keduanya `gemini` atau `gemini-cli`, dan sistem akan mengenalinya secara otomatis. Ini seperti seorang teman yang memiliki nama formal dan nama panggilan - bagaimanapun juga, orang-orang akan mengetahui siapa yang Anda maksud.

## Dukungan model GLM-5.1

GLM-5.1 adalah model andalan terbaru Zhipu AI, dan HagiCode telah menyelesaikan dukungan penuh untuk model tersebut.

### Katalog Profesi Sekunder

HagiCode mengelola semua model yang didukung melalui Katalog Profesi Sekunder. Berikut konfigurasi untuk seri GLM:

| ID Model | Nama | Mendukung Gambar | Keluarga CLI yang Kompatibel |
|----------|------|----------------|-------------------------|
| `glm-4.7` | GLM 4.7 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5` | GL 5 | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5-turbo` | GLM 5 Turbo | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.0` | GLM 5.0 (Warisan) | - | claude, codebuddy, hermes, qoder, kiro |
| `glm-5.1` | GLM 5.1 | **benar** | claude, codebuddy, hermes, qoder, kiro |

Karakteristik utama GLM-5.1 dapat diringkas sebagai berikut:
- Pengidentifikasi versi mandiri tanpa bagasi lama
- Model GLM pertama yang mendukung input gambar
- Penalaran dan pemahaman kode yang lebih kuat
- Kompatibilitas multi-CLI yang luas

### GLM-5.1 vs GLM-5.0

Pada tingkat kode, perbedaan utama antara GLM-5.1 dan GLM-5.0 ditampilkan di sini:

```csharp
// GLM-5.0 (Legacy) - contains special retention logic
private const string Glm50CodebuddySecondaryProfessionId = "secondary-glm-5-codebuddy";
private const string Glm50CodebuddyModelValue = "glm-5.0";

// GLM-5.1 - standalone new model identifier
private const string Glm51SecondaryProfessionId = "secondary-glm-5-1";
private const string Glm51ModelValue = "glm-5.1";
```

GLM-5.0 diberi label "Legacy" karena merupakan pengenal versi lama yang dipertahankan untuk kompatibilitas ke belakang. Sebaliknya, GLM-5.1 adalah versi baru yang berdiri sendiri tanpa beban historis. Beberapa hal tetap ada di masa lalu; yang lain melakukan perjalanan lebih ringan dan bergerak lebih cepat.

### Konfigurasikan GLM-5.1

Berikut contoh konfigurasi penggunaan GLM-5.1 di HagiCode:

```json
{
  "primaryProfessionId": "profession-claude-code",
  "secondaryProfessionId": "secondary-glm-5-1",
  "model": "glm-5.1",
  "reasoning": "high"
}
```

## Fitur unggah gambar

Dukungan gambar HagiCode diimplementasikan melalui `SupportsImage` properti aktif `SecondaryProfession`:

```csharp
public class HeroSecondaryProfessionSettingDto
{
    public bool SupportsImage { get; set; }
}
```

Pada Katalog Profesi Sekunder, konfigurasi GLM-5.1 terlihat seperti ini:

```json
{
  "id": "secondary-glm-5-1",
  "supportsImage": true
}
```

Artinya pengguna dapat langsung mengunggah tangkapan layar untuk analisis AI, seperti:
- Tangkapan layar pesan kesalahan
- Masalah di layar UI
- Bagan visualisasi data
- Hasil eksekusi kode

Tidak perlu lagi menjelaskan semuanya secara manual - cukup unggah tangkapan layarnya. Kenyamanan fitur ini terlihat jelas setelah Anda menggunakannya. Terkadang satu pandangan mengungkapkan lebih dari sekedar penjelasan panjang.

## Integrasi Gemini CLI

Sebagai Agen CLI kesepuluh, Gemini CLI diintegrasikan ke dalam HagiCode melalui arsitektur Penyedia standar.

### Opsi konfigurasi

Gemini CLI mendukung beragam opsi konfigurasi:

```csharp
public class GeminiOptions
{
    public string? ExecutablePath { get; set; }
    public string? WorkingDirectory { get; set; }
    public string? SessionId { get; set; }
    public string? Model { get; set; }
    public string? AuthenticationMethod { get; set; }
    public string? AuthenticationToken { get; set; }
    public Dictionary<string, string?> AuthenticationInfo { get; set; }
    public Dictionary<string, string?> EnvironmentVariables { get; set; }
    public string[] ExtraArguments { get; set; }
    public TimeSpan? StartupTimeout { get; set; }
    public CliPoolSettings? PoolSettings { get; set; }
}
```

Opsi ini mencakup segalanya mulai dari pengaturan dasar hingga fitur lanjutan, memberikan pengguna fleksibilitas untuk mengonfigurasi CLI sesuai kebutuhan mereka. Alur kerja setiap orang berbeda, jadi sedikit fleksibilitas selalu diterima.

### Protokol komunikasi ACP

Gemini CLI mendukung ACP (Agent Communication Protocol), yang merupakan standar komunikasi CLI terpadu HagiCode. Melalui ACP, CLI yang berbeda dapat berinteraksi dengan platform secara konsisten, sehingga sangat menyederhanakan pekerjaan integrasi. Singkatnya, ini menstandarkan bagian-bagian yang rumit sehingga setiap orang dapat bekerja dengan lebih mudah.

## Praktik pembuatan HagiCode sendiri

Berbicara tentang praktik dunia nyata, contoh terbaiknya adalah alur kerja pembangunan platform HagiCode sendiri. Proses pengembangan HagiCode telah memanfaatkan sepenuhnya kemampuan AI.

### Bekerja dengan baik bahkan dengan GLM 4.7

Desain platform HagiCode dioptimalkan dengan baik, sehingga tetap dapat memberikan pengalaman pengembangan yang baik bahkan dengan GLM 4.7. Platform ini telah membantu menyelesaikan beberapa proyek pembangunan penting, termasuk:
- Integrasi beberapa Penyedia CLI
- Implementasi fitur upload gambar
- Pembuatan dokumentasi dan penerbitan konten

Itu sebenarnya adalah hal yang baik. Tidak semua orang membutuhkan hal terbaru setiap saat. Apa yang paling cocok untuk Anda seringkali merupakan hal yang paling penting.

### GLM-5.1 memberikan lebih banyak dengan sedikit usaha

Setelah ditingkatkan ke GLM-5.1, kemampuan berikut menjadi lebih kuat:
- Pemahaman kode yang lebih kuat, mengurangi komunikasi bolak-balik
- Analisis ketergantungan yang lebih akurat, segera menunjukkan arah yang benar
- Diagnosis kesalahan lebih efisien, menemukan masalah lebih cepat
- Dukungan untuk input gambar, mempercepat deskripsi masalah

Ibarat berpindah dari sepeda ke mobil. Anda tetap bisa mencapai tujuan yang sama, namun kecepatan dan kenyamanannya tidak sama.

## Praktik terbaik untuk integrasi multi-CLI

`HagiCode.Libs.Providers` menyediakan mekanisme terpadu untuk pendaftaran dan penggunaan:

```csharp
services.AddHagiCodeLibs();

var gemini = serviceProvider.GetRequiredService<ICliProvider<GeminiOptions>>();
var codebuddy = serviceProvider.GetRequiredService<ICliProvider<CodebuddyOptions>>();
var hermes = serviceProvider.GetRequiredService<ICliProvider<HermesOptions>>();
```

Desain injeksi ketergantungan ini membuat penggunaan di berbagai CLI menjadi sangat ringkas dan juga membuat pengujian dan tiruan unit menjadi lebih nyaman. Kode bersih adalah cara bertanggung jawab terhadap diri sendiri.

### Catatan

Ada beberapa hal yang perlu diingat dalam penggunaan sebenarnya:

1. **Konfigurasi kunci API**: Pastikan `ANTHROPIC_AUTH_TOKEN` diatur dengan benar, atau model tidak dapat dipanggil
2. **Ketersediaan model**: GLM-5.1 harus diaktifkan oleh penyedia model yang sesuai
3. **Fitur gambar**: Hanya model dengan `supportsImage: true` dapat menggunakan unggahan gambar
4. **Instalasi CLI**: Sebelum menggunakan Gemini CLI, pastikan `gemini` atau `gemini-cli` ada di sistem PATH

Ini mungkin detail kecil, namun detail kecil yang ditangani dengan buruk dapat berubah menjadi masalah besar, sehingga patut diperhatikan.

## Kesimpulan

Dengan dukungan penuh untuk GLM-5.1 dan keberhasilan integrasi Gemini CLI, HagiCode semakin memperkuat kemampuannya sebagai platform pemrograman AI multi-model dan multi-CLI. Pembaruan ini tidak hanya memberi pengguna lebih banyak pilihan, namun juga menunjukkan arsitektur dan skalabilitas HagiCode yang berwawasan ke depan.

Dukungan gambar GLM-5.1, dikombinasikan dengan fitur unggah tangkapan layar HagiCode, memungkinkan AI "memahami dari gambar" dan sangat mengurangi biaya untuk menjelaskan masalah. Dan dengan dukungan sepuluh CLI, pengguna dapat secara fleksibel memilih asisten pemrograman AI yang paling sesuai dengan preferensi dan skenario mereka. Lebih banyak pilihan hampir selalu merupakan hal yang baik.

Yang terpenting, praktik pembangunan HagiCode membuktikan bahwa platform sudah dapat berjalan dengan baik dan menyelesaikan tugas-tugas kompleks bahkan dengan GLM 4.7, sementara peningkatan ke GLM-5.1 dapat lebih meningkatkan efisiensi pengembangan. Hidup juga sering kali seperti itu: Anda tidak selalu membutuhkan yang terbaik, hanya yang cocok untuk Anda. Tentu saja, jika yang sesuai dengan Anda bisa menjadi lebih baik lagi, maka itu akan lebih baik lagi.

Jika Anda tertarik dengan platform pemrograman AI multi-model dan multi-CLI, cobalah HagiCode - sumber terbuka, gratis, dan masih terus berkembang. Mencobanya tidak memerlukan biaya apa pun, dan mungkin itu yang Anda butuhkan.
