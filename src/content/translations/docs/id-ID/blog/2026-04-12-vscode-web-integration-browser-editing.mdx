## Pengeditan Kode Cepat di Browser: Integrasi Web VSCode dalam Praktek

> Setelah AI selesai menganalisis kode, bagaimana cara segera membuka editor di browser dan mulai melakukan perubahan? Artikel ini membagikan pengalaman praktis kami dalam mengintegrasikan server kode dalam proyek HagiCode untuk menciptakan jembatan yang mulus antara asisten AI dan pengalaman pengeditan kode.

## Latar Belakang

Di era pemrograman yang dibantu AI, pengembang sering kali perlu memeriksa dan mengedit kode dengan cepat. Alur kerja tradisionalnya sederhana: buka proyek di IDE desktop, cari file, edit, dan simpan. Namun dalam beberapa situasi, aliran itu selalu terasa agak melenceng.

Skenario satu: pengembangan jarak jauh. Saat menggunakan asisten AI seperti HagiCode, backend mungkin berjalan di server jarak jauh atau di dalam container, dan mesin lokal tidak dapat mengakses file proyek secara langsung. Setiap kali Anda perlu memeriksa atau memodifikasi kode, Anda harus terhubung melalui SSH atau metode lain, dan pengalamannya terasa terfragmentasi. Ini seperti ingin bertemu seseorang melalui kaca tebal: Anda dapat melihatnya, tetapi Anda tidak dapat menjangkaunya.

Skenario dua: pratinjau cepat. Setelah asisten AI menganalisis kode, pengguna mungkin hanya ingin menelusuri file dengan cepat atau membuat perubahan kecil. Meluncurkan IDE desktop lengkap terasa berat, sementara editor dalam browser yang ringan lebih sesuai dengan kebutuhan "tampilan cepat". Lagi pula, siapa yang ingin memobilisasi seluruh rangkaian alat hanya untuk melihatnya sekilas?

Skenario ketiga: kolaborasi lintas perangkat. Saat bekerja di perangkat yang berbeda, editor berbasis browser menyediakan titik akses terpadu tanpa mengharuskan setiap mesin dikonfigurasikan dengan lingkungan pengembangan. Itu saja dapat menyelamatkan banyak masalah. Hidup ini singkat; mengapa mengulangi pekerjaan pengaturan yang sama berulang kali?

Untuk mengatasi masalah ini, kami mengintegrasikan VSCode Web ke dalam proyek HagiCode. Hal ini memungkinkan asisten AI dan pengalaman pengeditan kode terhubung dengan mulus: setelah AI menganalisis kode, pengguna dapat segera membuka editor dan membuat perubahan dalam sesi browser yang sama, tanpa berpindah aplikasi. Ini adalah jenis pengalaman di mana, ketika Anda membutuhkannya, pengalaman itu ada di sana.

## Tentang HagiCode

Solusi yang dibagikan dalam artikel ini berasal dari pengalaman praktis kami di [Kode Hagi](https://hagicode.com) proyek. HagiCode adalah asisten pengkodean berbasis AI yang dirancang untuk meningkatkan efisiensi pengembangan melalui interaksi bahasa alami. Selama pengembangan, kami menemukan bahwa pengguna sering kali perlu beralih dengan cepat antara analisis AI dan pengeditan kode, yang mendorong kami untuk mengeksplorasi cara mengintegrasikan editor langsung ke browser.

Repositori proyek: [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site)

## Pilihan Teknis: Mengapa server kode?

Di antara banyak solusi Web VSCode yang tersedia, kami memilih server kode. Ada beberapa alasan konkrit di balik keputusan itu.

Kelengkapan fitur. code-server adalah versi web VSCode dan mendukung sebagian besar fitur desktop, termasuk sistem ekstensi, saran cerdas, debugging, dan banyak lagi. Artinya pengguna bisa mendapatkan pengalaman mengedit di browser yang sangat mirip dengan versi desktop. Lagi pula, siapa yang benar-benar ingin berkompromi pada fungsionalitas?

Penerapan yang fleksibel. code-server dapat berjalan sebagai layanan independen dan juga mendukung penerapan berbasis Docker, yang cocok dengan arsitektur HagiCode. Backend kami ditulis dalam C#, frontend menggunakan React, dan keduanya berkomunikasi dengan layanan server kode melalui REST API. Ini seperti membangun dengan balok: setiap bagian mempunyai tempatnya.

Otentikasi aman. kode-server termasuk bawaan `connection-token` mekanisme untuk mencegah akses yang tidak sah. Setiap sesi memiliki token unik sehingga hanya pengguna yang berwenang yang dapat membuka editor. Keamanan adalah salah satu hal yang hanya Anda hargai sepenuhnya setelah Anda memilikinya.

## Desain Arsitektur

Integrasi Web VSCode HagiCode menggunakan arsitektur terpisah front-end/back-end.

### Lapisan Layanan Frontend

Frontend membungkus interaksi dengan backend `vscodeServerService.ts`:

```typescript
// Open project
export async function openProjectInCodeServer(
  id: string,
  currentInterfaceLanguage?: string,
): Promise<VsCodeServerLaunchResponseDto>

// Open vault
export async function openVaultInCodeServer(
  id: string,
  path?: string,
  currentInterfaceLanguage?: string,
): Promise<VsCodeServerLaunchResponseDto>
```

Perbedaan antara kedua metode ini sangat jelas: `openProjectInCodeServer` membuka seluruh proyek, sementara `openVaultInCodeServer` membuka jalur tertentu di dalam Vault. Untuk proyek multi-repositori MonoSpecs, sistem secara otomatis membuat file ruang kerja. Tanggung jawab yang jelas sering kali sudah cukup bila masing-masing bagian melakukan tugasnya dengan baik.

### Lapisan Layanan Backend

Bagian belakang `VaultAppService.cs` mengimplementasikan logika inti:

```csharp
public async Task<VsCodeServerLaunchResponseDto> OpenInCodeServerAsync(
    string id,
    string? relativePath = null,
    string? currentInterfaceLanguage = null,
    CancellationToken cancellationToken = default)
{
    // 1. Get settings and check whether the feature is enabled
    var settings = await _vsCodeServerSettingsService.GetResolvedSettingsAsync(cancellationToken);
    if (!settings.Enabled) {
        throw new BusinessException(VsCodeServerErrorCodes.Disabled, "VSCode Server is disabled.");
    }

    // 2. Get vault and resolve the launch directory
    var vault = await RequireVaultAsync(id, cancellationToken);
    var launchDirectory = ResolveLaunchDirectory(vault, relativePath);

    // 3. Ensure code-server is running and get runtime info
    var runtime = await _vsCodeServerManager.EnsureStartedAsync(settings, cancellationToken);

    // 4. Resolve language settings
    var language = _vsCodeServerSettingsService.ResolveLaunchLanguage(
        settings.Language,
        currentInterfaceLanguage);

    // 5. Build launch URL
    return new VsCodeServerLaunchResponseDto {
        LaunchUrl = AppendQueryString(runtime.BaseUrl, new Dictionary<string, string?> {
            ["folder"] = launchDirectory,
            ["tkn"] = runtime.ConnectionToken,
            ["vscode-lang"] = language,
        }),
        ConnectionToken = runtime.ConnectionToken,
        OpenMode = "folder",
        Runtime = VsCodeServerSettingsService.MapRuntime(
            await _vsCodeServerManager.GetRuntimeSnapshotAsync(cancellationToken)),
    };
}
```

Metode ini memiliki tanggung jawab yang sangat jelas: memeriksa pengaturan, menyelesaikan jalur, memulai layanan, dan membuat URL. Diantaranya adalah `ResolveLaunchDirectory` metode melakukan pemeriksaan keamanan jalur untuk mencegah serangan traversal jalur. Kode bisa terasa seperti puisi ketika setiap baris memiliki tujuan.

### Manajemen Waktu Proses Otomatis

Backend mengelola proses server kode melalui `VsCodeServerManager`:

- Periksa status proses
- Secara otomatis memulai layanan yang dihentikan
- Menampilkan snapshot runtime seperti port, ID proses, dan waktu mulai

Desain ini memungkinkan sistem secara otomatis menangani siklus hidup server kode, sehingga pengguna tidak perlu mengelola proses layanan secara manual. Hidup ini sudah cukup rumit; segala sesuatu yang dapat diotomatisasi seharusnya demikian.

### Sinkronisasi Bahasa

HagiCode mendukung antarmuka multibahasa, dan server kode harus mengikuti pengaturan itu. Sistem mendukung tiga mode bahasa:

- `follow`: ikuti bahasa antarmuka saat ini
- `zh-CN`: diperbaiki ke bahasa Cina
- `en-US`: diperbaiki ke bahasa Inggris

Pengaturan diteruskan ke server kode melalui `vscode-lang` Parameter URL agar bahasa editor tetap konsisten dengan antarmuka HagiCode. Bahasa terasa paling baik jika disatukan.

### Ruang Kerja Multi-Repositori MonoSpecs

Untuk proyek MonoSpecs, yang berisi beberapa sub-repositori dalam satu monorepo, sistem secara otomatis membuat a `.code-workspace` mengajukan:

```csharp
private async Task<string> CreateWorkspaceFileAsync(Project project, Guid projectId)
{
    var folders = await ResolveWorkspaceFoldersAsync(project.Path);
    var workspaceDocument = new {
        folders = folders.Select(path => new { path }).ToArray(),
    };
    // Generate workspace file...
}
```

Hal ini memungkinkan untuk mengedit beberapa sub-repositori dalam contoh server kode yang sama, yang sangat praktis untuk proyek monorepo besar. Banyak repositori dalam satu jendela bisa terasa seperti banyak cerita yang dikumpulkan dalam satu buku.

## Integrasi Frontend

Frontend HagiCode menggunakan React + TypeScript, dan mengintegrasikan server kode tidak terlalu rumit.

### Tombol Tindakan Cepat

Tambahkan tombol Server Kode ke kartu proyek:

```tsx
// QuickActionsZone.tsx
<Button
  size="sm"
  variant="default"
  onClick={() => onAction({ type: 'open-code-server' })}
>
  <Globe className="h-3 w-3 mr-1" />
  <span className="text-xs">{t('project.openCodeServer')}</span>
</Button>
```

Tombol ini memicu tindakan terbuka dan memanggil API backend untuk mendapatkan URL peluncuran. Satu tombol, satu tindakan, langsung dan sederhana.

### Menangani Aksi Terbuka

```tsx
const handleAction = async (action: ProjectAction) => {
  if (action.type === 'open-code-server') {
    const response = await openProjectInCodeServer(project.id, i18n.language);
    window.open(response.launchUrl, '_blank', 'noopener,noreferrer');
  }
};
```

Gunakan `window.open` untuk membuka server kode di tab baru. Itu `noopener,noreferrer` parameter memberikan keamanan ekstra. Dalam hal keamanan, tidak ada kata terlalu berhati-hati.

### Entri Pengeditan Vault

Tambahkan tombol edit serupa di daftar Vault:

```tsx
const handleEditVault = async (vault: VaultItemDto) => {
  const response = await openVaultInCodeServer(vault.id);
  window.open(response.launchUrl, '_blank', 'noopener,noreferrer');
};
```

Proyek dan Vault menggunakan mekanisme terbuka yang sama, yang menjaga interaksi tetap konsisten. Konsistensi sama pentingnya dengan fitur itu sendiri.

## Logika Konstruksi URL

Format URL untuk server kode memiliki beberapa detail yang perlu diperhatikan.

**Mode folder**:
```
http://{host}:{port}/?folder={path}&tkn={token}&vscode-lang={lang}
```

**Mode ruang kerja**:
```
http://{host}:{port}/?workspace={workspacePath}&tkn={token}&vscode-lang={lang}
```

Di sini, `tkn` adalah token koneksi. Ini dihasilkan secara otomatis setiap kali server kode dimulai, memastikan akses aman. Itu `vscode-lang` parameter mengontrol bahasa UI editor. Masing-masing parameter ini mempunyai peran masing-masing.

## Skenario Penggunaan

### Skenario Satu: Tinjauan Kode Berbantuan AI

Pengguna berbicara dengan HagiCode, AI menganalisis kode proyek dan menemukan potensi masalah, lalu pengguna mengklik tombol "Buka di Server Kode" untuk membuka editor langsung di browser, memeriksa file yang terpengaruh, memperbaikinya, dan kembali ke HagiCode untuk melanjutkan percakapan. Seluruh aliran terjadi di browser tanpa berpindah aplikasi. Rasanya halus seperti air mengalir yang terasa halus.

### Skenario Dua: Mengedit Materi Pelajaran di Vault

Seorang pengguna membuat Vault untuk mempelajari proyek sumber terbuka dan ingin menambahkan catatan belajar di bawah `docs/` direktori. Dengan server kode, mereka dapat mengedit file Markdown langsung di browser, menyimpannya, dan membiarkan HagiCode segera membaca catatan yang diperbarui. Hal ini sangat berguna untuk membangun basis pengetahuan pribadi. Pengetahuan hanya menjadi lebih berharga jika semakin banyak Anda mengumpulkannya.

### Skenario Tiga: Pengembangan Multi-Repositori MonoSpecs

Proyek MonoSpecs berisi beberapa sub-repositori, dan server kode secara otomatis membuat ruang kerja multi-folder. Di browser, pengguna dapat mengedit kode di beberapa repositori sekaligus dan kemudian melakukan perubahan kembali ke repositori Git masing-masing. Alur kerja ini sangat cocok untuk perubahan yang perlu menjangkau banyak repositori. Mengedit beberapa repositori secara bersamaan membutuhkan sedikit teknik, seperti menangani banyak tugas secara bersamaan.

## Pertimbangan Keamanan

Saat menerapkan integrasi server kode, keamanan perlu mendapat perhatian khusus. Jika keamanan bermasalah, Anda selalu terlambat menyadarinya.

### Token Koneksi

Itu `connection-token` dihasilkan secara acak dan tidak boleh diekspos. Paling baik digunakan di bawah HTTPS untuk mencegah token dicegat oleh orang yang berada di tengah. Informasi sensitif layak dilindungi dengan baik.

### Keamanan Jalur File

Backend mengimplementasikan pemeriksaan traversal jalur:

```csharp
private static string ResolveLaunchDirectory(VaultRegistryEntry vault, string? relativePath)
{
    var vaultRoot = EnsureTrailingSeparator(Path.GetFullPath(vault.PhysicalPath));
    var combinedPath = Path.GetFullPath(Path.Combine(vaultRoot, relativePath ?? "."));
    if (!combinedPath.StartsWith(vaultRoot, StringComparison.OrdinalIgnoreCase))
    {
        throw new BusinessException(VaultRelativePathTraversalCode, "Relative path traversal detected.");
    }
    return combinedPath;
}
```

Kode ini memastikan bahwa pengguna tidak dapat menggunakan `../` atau pola serupa untuk mengakses file di luar direktori Vault. Pemeriksaan batas selalu lebih baik dilakukan daripada dilewati.

### Kontrol Izin

Proses server kode harus dijalankan dengan izin pengguna yang sesuai sehingga tidak dapat mengakses file sistem sensitif. Yang terbaik adalah menjalankan layanan server kode di bawah pengguna khusus. Kontrol izin adalah salah satu hal mendasar yang harus selalu Anda pertahankan.

## Optimasi Kinerja

kode-server menggunakan sumber daya server, jadi berikut adalah beberapa saran pengoptimalan:

- Pantau penggunaan CPU dan memori, dan sesuaikan batas sumber daya bila diperlukan
- Proyek besar mungkin memerlukan waktu tunggu yang lebih lama
- Terapkan pembersihan batas waktu sesi otomatis untuk melepaskan sumber daya
- Pertimbangkan caching untuk mengurangi komputasi berulang

HagiCode menyediakan API pemantauan status runtime, dan frontend dapat menelepon `getVsCodeServerSettings()` untuk mengambil keadaan saat ini:

```typescript
const { settings, runtime } = await getVsCodeServerSettings();
// runtime.status: 'disabled' | 'stopped' | 'starting' | 'running' | 'unhealthy'
// runtime.baseUrl: "http://localhost:8080"
// runtime.processId: 12345
```

Desain ini memungkinkan pengguna untuk memahami dengan jelas status kesehatan server kode dan dengan cepat menemukan masalah ketika terjadi kesalahan. Ketika status terlihat, orang merasa lebih memegang kendali.

## Detail Pengalaman Pengguna

Selama penerapan, kami menemukan beberapa detail yang sangat memengaruhi pengalaman pengguna dan patut mendapat perhatian ekstra.

Membuka server kode untuk pertama kalinya mungkin memerlukan waktu tunggu untuk memulai, dan penundaan tersebut dapat berkisar dari beberapa detik hingga setengah menit. Sebaiknya tunjukkan status pemuatan di frontend sehingga pengguna mengetahui bahwa sistem masih berfungsi. Menunggu lebih mudah bila ada umpan balik.

Browser mungkin memblokir popup, sehingga pengguna harus diminta untuk mengizinkannya secara manual. Pada peluncuran pertama, HagiCode menampilkan panduan yang menjelaskan cara memberikan izin browser yang diperlukan. Pengalaman pengguna sering kali hidup dalam detail kecil ini.

Ada baiknya juga untuk menampilkan status runtime seperti mulai, berjalan, atau error, sehingga ketika terjadi masalah, pengguna dapat dengan cepat mengetahui apakah masalahnya ada di sisi server atau pada operasi mereka sendiri. Mengetahui di mana masalahnya setidaknya memberi Anda titik awal.

## Contoh Konfigurasi

Konfigurasi untuk server kode tidak rumit:

```json
{
  "vscodeServer": {
    "enabled": true,
    "host": "0.0.0.0",
    "port": 8080,
    "language": "follow"
  }
}
```

`enabled` mengontrol apakah fitur tersebut diaktifkan, `host` dan `port` menentukan alamat mendengarkan, dan `language` mengatur mode bahasa. Pengaturan ini dapat diubah melalui UI dan langsung diterapkan. Hal-hal sederhana seringkali paling mudah digunakan.

## Kesimpulan
