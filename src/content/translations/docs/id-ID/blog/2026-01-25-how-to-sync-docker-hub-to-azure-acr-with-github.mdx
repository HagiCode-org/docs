## Mengotomatiskan Sinkronisasi Gambar dari Docker Hub ke Azure ACR

> Artikel ini menjelaskan cara menggunakan GitHub Actions dan alat sinkronisasi gambar untuk mengotomatiskan sinkronisasi gambar dari Docker Hub ke Azure Container Registry, mengatasi lambatnya akses ke Docker Hub di Tiongkok dan beberapa wilayah Azure sekaligus meningkatkan ketersediaan gambar dan efisiensi penerapan di lingkungan Azure.

- Kecepatan penarikan gambar lambat, dengan akses Docker Hub dibatasi di Tiongkok dan beberapa wilayah Azure

## Perbandingan Solusi Teknis

Saat menentukan solusinya, kami membandingkan beberapa opsi teknis:

### 1. sinkronisasi gambar (pilihan akhir)
- Sinkronisasi tambahan: hanya menyinkronkan lapisan gambar yang diubah, sehingga mengurangi transfer jaringan secara signifikan
- Lanjutkan dukungan: sinkronisasi dapat dilanjutkan setelah gangguan jaringan
- Kontrol konkurensi: thread pekerja yang dapat dikonfigurasi meningkatkan efisiensi sinkronisasi gambar yang besar
- Penanganan kesalahan yang kuat: mekanisme percobaan ulang bawaan untuk kegagalan (3 percobaan ulang secara default)
- Penerapan ringan: biner tunggal tanpa ketergantungan
- Dukungan multi-registrasi: kompatibel dengan Docker Hub, Azure ACR, Harbor, dan lainnya

### 2. CLI Docker
- Tidak ada sinkronisasi tambahan: konten gambar lengkap harus ditarik setiap saat
- Efisiensi lebih rendah: volume transfer jaringan besar dan waktu eksekusi lebih lama
- Mudah digunakan: berdasarkan familiar `docker pull` dan `docker push` perintah

### 3. Azure CLI
- Kompleksitas yang lebih tinggi: Autentikasi Azure CLI harus dikonfigurasi
- Fungsionalitas terbatas: `az acr import` cakupannya relatif sempit
- Integrasi asli: terintegrasi dengan baik dengan layanan Azure

- Pantau jumlah dan ukuran gambar yang disinkronkan
- Jalankan sinkronisasi di luar jam sibuk

## Langkah-Langkah Implementasi

### 1. Persiapan
- Buat atau konfirmasikan Azure Container Registry di Portal Azure
- Buat kredensial akses ACR (nama pengguna dan kata sandi)
- Konfirmasikan izin akses untuk repositori gambar Docker Hub

### 2. Konfigurasikan Rahasia GitHub
Tambahkan Rahasia berikut di pengaturan repositori GitHub:
- AZURE_ACR_USERNAME: Nama pengguna Azure ACR
- AZURE_ACR_PASSWORD: Kata sandi Azure ACR

### 3. Buat alur kerja Tindakan GitHub
Konfigurasikan alur kerja di `.github/workflows/sync-docker-acr.yml`:
- Pemicu terjadwal: setiap hari pukul 00:00 UTC
- Pemicu manual: mendukung `workflow_dispatch`
- Pemicu tambahan: berjalan saat ditekan ke `publish` cabang untuk sinkronisasi cepat

### 4. Alur eksekusi alur kerja

| Langkah | Peserta | Tindakan | Deskripsi |
| --- | --- | --- | --- |
| 1 | Tindakan GitHub | Memicu alur kerja | Dipicu oleh jadwal, eksekusi manual, atau dorongan ke `publish` cabang |
| 2 | Tindakan GitHub → sinkronisasi gambar | Unduh dan jalankan alat sinkronisasi | Memasuki fase sinkronisasi sebenarnya |
| 3 | sinkronisasi gambar → Docker Hub | Ambil manifes gambar dan daftar tag | Membaca metadata dari repositori sumber |
| 4 | sinkronisasi gambar → Azure ACR | Ambil data gambar yang ada dari repositori target | Menentukan keadaan sisi target saat ini |
| 5 | sinkronisasi gambar | Bandingkan perbedaan sumber dan target | Mengidentifikasi lapisan gambar yang memerlukan sinkronisasi |
| 6 | sinkronisasi gambar → Docker Hub | Tarik lapisan gambar yang diubah | Hanya mentransfer konten yang perlu diperbarui |
| 7 | sinkronisasi gambar → Azure ACR | Dorong lapisan gambar yang diubah | Menyelesaikan sinkronisasi tambahan |
| 8 | sinkronisasi gambar → Tindakan GitHub | Kembalikan statistik sinkronisasi | Termasuk hasil, perbedaan, dan informasi kesalahan |
| 9 | Tindakan GitHub | Rekam log dan unggah artefak | Mendukung audit dan pemecahan masalah selanjutnya |

## Implementasi Alur Kerja Tindakan GitHub

Di bawah ini adalah konfigurasi alur kerja aktual yang digunakan (`.github/workflows/sync-docker-acr.yml`):

```yaml
name: Sync Docker Image to Azure ACR

on:
  schedule:
    - cron: "0 0 * * *" # Every day at 00:00 UTC
  workflow_dispatch: # Manual trigger
  push:
    branches: [publish]

permissions:
  contents: read

jobs:
  sync:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Download image-syncer
        run: |
          # Download the image-syncer binary
          wget https://github.com/AliyunContainerService/image-syncer/releases/download/v1.5.5/image-syncer-v1.5.5-linux-amd64.tar.gz
          tar -zxvf image-syncer-v1.5.5-linux-amd64.tar.gz
          chmod +x image-syncer

      - name: Create auth config
        run: |
          # Generate the authentication config file (YAML format)
          cat > auth.yaml <<EOF
          hagicode.azurecr.io:
            username: "${{ secrets.AZURE_ACR_USERNAME }}"
            password: "${{ secrets.AZURE_ACR_PASSWORD }}"
          EOF

      - name: Create images config
        run: |
          # Generate the image sync config file (YAML format)
          cat > images.yaml <<EOF
          docker.io/newbe36524/hagicode: hagicode.azurecr.io/hagicode
          EOF

      - name: Run image-syncer
        run: |
          # Run synchronization (using the newer --auth and --images parameters)
          ./image-syncer --auth=./auth.yaml --images=./images.yaml --proc=10 --retries=3

      - name: Upload logs
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: sync-logs
          path: image-syncer-*.log
          retention-days: 7
```

## Detail Konfigurasi

### 1. Kondisi pemicu
- Pemicu terjadwal: `cron: "0 0 * * *"` berjalan setiap hari pada pukul 00:00 UTC
- Pemicu manual: `workflow_dispatch` memungkinkan pengguna untuk menjalankannya secara manual dari UI GitHub
- Pemicu dorong: `push: branches: [publish]` terpicu saat didorong ke cabang rilis untuk sinkronisasi cepat

### 2. Konfigurasi otentikasi (`auth.yaml`)
```yaml
hagicode.azurecr.io:
  username: "${{ secrets.AZURE_ACR_USERNAME }}"
  password: "${{ secrets.AZURE_ACR_PASSWORD }}"
```

### 3. Konfigurasi sinkronisasi gambar
```yaml
docker.io/newbe36524/hagicode: hagicode.azurecr.io/hagicode
```
Konfigurasi ini berarti semua tag dari `docker.io/newbe36524/hagicode` akan disinkronkan ke `hagicode.azurecr.io/hagicode`.

### 4. parameter sinkronisasi gambar
- `--auth=./auth.yaml`: jalur ke file konfigurasi otentikasi
- `--images=./images.yaml`: jalur ke file konfigurasi sinkronisasi gambar
- `--proc=10`: gunakan `10` pekerja bersamaan
- `--retries=3`: coba lagi kegagalan `3` kali

## Catatan dan Praktik Terbaik

### 1. Rekomendasi keamanan
- Memutar kata sandi Azure ACR secara teratur (disarankan setiap 90 hari)
- Gunakan akun layanan ACR khusus dengan izin push-only
- Pantau log akses ACR untuk mendeteksi akses abnormal tepat waktu
- Jangan menampilkan kredensial dalam log
- Jangan memasukkan kredensial ke repositori kode

### 2. Optimalisasi kinerja
- Sesuaikan `--proc` parameter: menyetel konkurensi berdasarkan bandwidth jaringan (disarankan `5-20`)
- Pantau durasi sinkronisasi: jika sinkronisasi memakan waktu terlalu lama, pertimbangkan untuk menurunkan konkurensi
- Bersihkan log secara teratur: tetapkan yang masuk akal `retention-days` nilai (`7` saat ini)

### 3. Pemecahan masalah

#### Masalah 1: Kegagalan otentikasi
```
Error: failed to authenticate to hagicode.azurecr.io
```
Solusi:
1. Periksa apakah Rahasia GitHub dikonfigurasi dengan benar
2. Verifikasi apakah kata sandi Azure ACR telah kedaluwarsa
3. Konfirmasikan bahwa izin akun layanan ACR sudah benar

#### Masalah 2: Batas waktu jaringan habis
```
Error: timeout waiting for response
```
Solusi:
1. Periksa konektivitas jaringan
2. Mengurangi jumlah pekerja bersamaan (the `--proc` parameter)
3. Tunggu hingga jaringan pulih dan jalankan kembali alur kerja

#### Masalah 3: Sinkronisasi gambar tidak lengkap
```
Warning: some tags failed to sync
```
Solusi:
1. Periksa log sinkronisasi untuk mengidentifikasi tag yang gagal
2. Jalankan kembali alur kerja secara manual untuk melakukan sinkronisasi lagi
3. Verifikasi bahwa gambar sumber di Docker Hub sehat

### 4. Pemantauan dan peringatan
- Periksa halaman Tindakan secara rutin untuk mengonfirmasi status alur kerja
- Aktifkan pemberitahuan GitHub sehingga kegagalan alur kerja segera dilaporkan
- Pantau penggunaan penyimpanan Azure ACR
- Verifikasi konsistensi tag di seluruh registri secara berkala

## Pertanyaan dan Solusi yang Sering Diajukan

### Q1: Bagaimana cara menyinkronkan tag tertentu dan bukan semua tag?

Ubah `images.yaml` file konfigurasi:
```yaml
# Sync only the latest and v1.0 tags
docker.io/newbe36524/hagicode:latest: hagicode.azurecr.io/hagicode:latest
docker.io/newbe36524/hagicode:v1.0: hagicode.azurecr.io/hagicode:v1.0
```

### Q2: Bagaimana cara menyinkronkan beberapa repositori gambar?

Tambahkan beberapa baris ke `images.yaml`:
```yaml
docker.io/newbe36524/hagicode: hagicode.azurecr.io/hagicode
docker.io/newbe36524/another-image: hagicode.azurecr.io/another-image
```

### Q3: Bagaimana cara saya mencoba lagi setelah sinkronisasi gagal?
- Coba ulang otomatis: sinkronisasi gambar menyertakan mekanisme coba ulang bawaan (`3` mencoba ulang secara default)
- Coba lagi secara manual: klik `Re-run all jobs` di halaman Tindakan GitHub

### Q4: Bagaimana cara melihat kemajuan sinkronisasi secara detail?
- Lihat log waktu nyata di halaman Tindakan
- Unduh `sync-logs` artefak untuk memeriksa file log lengkap
- File log mencakup status sinkronisasi dan kecepatan transfer untuk setiap tag

### Q5: Berapa lama waktu yang dibutuhkan untuk sinkronisasi?
- Sinkronisasi penuh awal: biasanya `10-30` menit tergantung pada ukuran gambar
- Sinkronisasi tambahan: biasanya `2-5` menit jika perubahan gambar kecil
- Waktunya bergantung pada bandwidth jaringan, ukuran gambar, dan pengaturan konkurensi

## Ekstensi yang Disarankan

### 1. Tambahkan notifikasi sinkronisasi
Tambahkan langkah pemberitahuan ke alur kerja:
```yaml
- name: Notify on success
  if: success()
  run: |
    echo "Docker images synced successfully to Azure ACR"
```

### 2. Menerapkan pemfilteran tag gambar
Tambahkan logika pemfilteran tag ke alur kerja:
```yaml
- name: Filter tags
  run: |
    # Sync only tags prefixed with v
    echo "docker.io/newbe36524/hagicode:v* : hagicode.azurecr.io/hagicode:v*" > images.yaml
```

### 3. Tambahkan laporan statistik sinkronisasi
```yaml
- name: Generate report
  if: always()
  run: |
    echo "## Sync Report" >> $GITHUB_STEP_SUMMARY
    echo "- Total tags: $(grep -c 'synced' image-syncer-*.log)" >> $GITHUB_STEP_SUMMARY
    echo "- Sync time: ${{ steps.sync.outputs.duration }}" >> $GITHUB_STEP_SUMMARY
```
