## Panduan Penerapan Dukungan Kata Cepat untuk Pengenalan Ucapan Doubao

> Artikel ini menjelaskan secara rinci bagaimana menerapkan dukungan kata cepat untuk pengenalan ucapan Doubao dalam proyek HagiCode. Dengan menggunakan kata cepat khusus dan tabel kata cepat platform, Anda dapat meningkatkan akurasi pengenalan untuk kosakata khusus domain secara signifikan.

## Latar Belakang

Teknologi pengenalan suara telah berkembang selama bertahun-tahun, namun ada satu masalah yang selalu mengganggu para pengembang. Model pengenalan suara untuk tujuan umum dapat mencakup bahasa sehari-hari, namun sering kali gagal dalam hal terminologi profesional, nama produk, dan nama pribadi. Coba pikirkan: asisten suara di bidang medis perlu mengenali istilah-istilah seperti "hipertensi", "diabetes", dan "penyakit jantung koroner" secara akurat; sistem hukum perlu secara tepat menangkap istilah-istilah seperti "penyebab tindakan", "pembelaan", dan "beban pembuktian". Dalam skenario ini, model tujuan umum telah berusaha semaksimal mungkin, namun hal tersebut sering kali tidak cukup.

Kami menghadapi tantangan yang sama dalam proyek HagiCode. Sebagai asisten pengkodean AI multifungsi, HagiCode perlu menangani pengenalan suara untuk berbagai terminologi teknis. Namun, API pengenalan suara Doubao, dalam konfigurasi defaultnya, tidak dapat sepenuhnya memenuhi persyaratan akurasi kami untuk istilah khusus. Bukan berarti Doubao tidak cukup baik; sebaliknya, setiap domain memiliki sistem terminologinya sendiri. Setelah melakukan beberapa penelitian dan eksplorasi teknis, kami menemukan bahwa API pengenalan suara Doubao sebenarnya menyediakan dukungan kata cepat. Dengan konfigurasi yang mudah, ini dapat meningkatkan akurasi pengenalan kosakata tertentu secara signifikan. Dalam arti tertentu, setelah Anda memberi tahu kata-kata mana yang harus diperhatikan, ia akan mendengarkannya dengan lebih cermat.

Apa yang dibagikan artikel ini adalah solusi lengkap yang kami gunakan dalam proyek HagiCode untuk mengimplementasikan kata cepat pengenalan suara Doubao. Kedua mode, kata cepat khusus dan tabel kata cepat platform, tersedia, dan keduanya juga dapat digabungkan. Dengan solusi ini, pengembang dapat secara fleksibel mengonfigurasikan kata-kata cepat berdasarkan skenario bisnis sehingga sistem pengenalan suara dapat “mengenali” kosakata profesional, tidak umum, namun penting dengan lebih baik.

## Implementasi Inti

### Memahami Dua Mode Kata Cepat

API pengenalan ucapan Doubao menyediakan dua cara untuk mengonfigurasi kata cepat, dan masing-masing cara memiliki kasus penggunaan dan keunggulan idealnya sendiri.

**Mode kata cepat khusus** memungkinkan kami meneruskan teks kata cepat langsung melalui `corpus.context` bidang. Pendekatan ini sangat cocok untuk skenario ketika Anda perlu mengonfigurasi sejumlah kecil kata cepat dengan cepat, seperti mengenali nama produk atau nama seseorang untuk sementara. Dalam implementasi HagiCode, kami mengurai teks kata cepat multi-baris yang dimasukkan oleh pengguna ke dalam daftar string, lalu memformatnya ke dalam `context_data` array yang dibutuhkan oleh Doubao API. Pendekatan ini sangat langsung: Anda cukup memberi tahu sistem kata mana yang harus diperhatikan, dan sistem akan melakukan hal itu.

**Mode tabel kata cepat platform** menggunakan `corpus.boosting_table_id` bidang untuk mereferensikan tabel kata cepat yang telah dikonfigurasi sebelumnya di platform pembelajaran mandiri Doubao. Pendekatan ini cocok untuk skenario ketika Anda perlu mengelola kata cepat dalam jumlah besar. Kita dapat membuat dan mengelola tabel kata cepat di platform pembelajaran mandiri Doubao, lalu mereferensikannya berdasarkan ID. Untuk proyek seperti HagiCode, dimana istilah khusus perlu terus diperbarui dan dipelihara, mode ini menawarkan pengelolaan yang jauh lebih baik. Ketika jumlah kata cepat bertambah, memiliki tempat terpusat untuk mengelolanya jauh lebih baik daripada memasukkannya secara manual setiap saat.

Menariknya, kedua mode ini juga bisa digunakan bersamaan. Doubao API mendukung penyertaan kata cepat khusus dan ID tabel kata cepat platform dalam permintaan yang sama, dengan strategi kombinasi yang dikontrol oleh `combine_mode` parameter. Fleksibilitas ini memungkinkan HagiCode menangani berbagai kebutuhan pengenalan terminologi profesional yang kompleks. Terkadang, menggabungkan beberapa pendekatan menghasilkan hasil yang lebih baik.

### Definisi dan Validasi Tipe Frontend

Dalam implementasi frontend HagiCode, kami mendefinisikan satu set lengkap tipe konfigurasi kata cepat dan logika validasi. Bagian pertama adalah definisi tipe:

```typescript
export interface HotwordConfig {
  contextText: string;           // Multi-line hotword text
  boostingTableId: string;      // Doubao platform hotword table ID
  combineMode: boolean;          // Whether to use both together
}
```

Antarmuka sederhana ini berisi semua item konfigurasi untuk fitur kata cepat. Diantaranya, `contextText` adalah bagian yang paling banyak berinteraksi secara langsung dengan pengguna: kami mengizinkan pengguna memasukkan satu frasa kata cepat per baris, yang sangat intuitif. Meminta pengguna memasukkan satu istilah per baris jauh lebih mudah daripada membuat mereka memahami format konfigurasi yang rumit.

Berikutnya adalah fungsi validasi. Berdasarkan batasan Doubao API, kami menetapkan aturan validasi yang ketat: maksimal 100 baris teks kata cepat, hingga 50 karakter per baris, dan total tidak lebih dari 5000 karakter; `boosting_table_id` maksimal 200 karakter dan hanya boleh berisi huruf, angka, garis bawah, dan tanda hubung. Batasan ini tidak sembarangan; mereka datang langsung dari dokumentasi resmi Doubao. Batasan API adalah batasan API, dan kami harus mengikutinya.

```typescript
export function validateContextText(contextText: string): HotwordValidationResult {
  if (!contextText || contextText.trim().length === 0) {
    return { isValid: true, errors: [] };
  }

  const lines = contextText.split('\n').filter(line => line.trim().length > 0);
  const errors: string[] = [];

  if (lines.length > 100) {
    errors.push(`Hotword line count cannot exceed 100 lines; current count is ${lines.length}`);
  }

  const totalChars = contextText.length;
  if (totalChars > 5000) {
    errors.push(`Total hotword character count cannot exceed 5000; current count is ${totalChars}`);
  }

  for (let i = 0; i < lines.length; i++) {
    if (lines[i].length > 50) {
      errors.push(`Hotword on line ${i + 1} exceeds the 50-character limit`);
    }
  }

  return { isValid: errors.length === 0, errors };
}

export function validateBoostingTableId(boostingTableId: string): HotwordValidationResult {
  if (!boostingTableId || boostingTableId.trim().length === 0) {
    return { isValid: true, errors: [] };
  }

  const errors: string[] = [];

  if (boostingTableId.length > 200) {
    errors.push(`boosting_table_id cannot exceed 200 characters; current count is ${boostingTableId.length}`);
  }

  if (!/^[a-zA-Z0-9_-]+$/.test(boostingTableId)) {
    errors.push('boosting_table_id can contain only letters, numbers, underscores, and hyphens');
  }

  return { isValid: errors.length === 0, errors };
}
```

Fungsi validasi ini dijalankan segera saat pengguna mengonfigurasi kata cepat, memastikan bahwa masalah dapat diketahui sedini mungkin. Dari perspektif pengalaman pengguna, umpan balik instan semacam ini sangat penting. Itu selalu lebih baik bagi pengguna untuk mengetahui apa yang salah saat mereka mengetik daripada setelah mereka mengirimkan.

### Kegigihan Konfigurasi Frontend

Dalam implementasi frontend HagiCode, kami memilih untuk menggunakan browser `localStorage` untuk menyimpan konfigurasi kata cepat. Ada beberapa pertimbangan di balik keputusan desain ini. Pertama, konfigurasi kata cepat sangat dipersonalisasi, dan pengguna yang berbeda mungkin memiliki kebutuhan spesifik domain yang berbeda. Kedua, pendekatan ini menyederhanakan implementasi backend karena tidak memerlukan tabel database tambahan atau titik akhir API. Terakhir, setelah pengguna mengonfigurasinya sekali di browser, pengaturan dapat dimuat secara otomatis pada penggunaan berikutnya, yang sangat memudahkan. Sederhananya, ini adalah pendekatan termudah.

```typescript
const HOTWORD_STORAGE_KEYS = {
  contextText: 'hotword-context-text',
  boostingTableId: 'hotword-boosting-table-id',
  combineMode: 'hotword-combine-mode',
} as const;

export const DEFAULT_HOTWORD_CONFIG: HotwordConfig = {
  contextText: '',
  boostingTableId: '',
  combineMode: false,
};

// Load hotword configuration
export function loadHotwordConfig(): HotwordConfig {
  const contextText = localStorage.getItem(HOTWORD_STORAGE_KEYS.contextText) || '';
  const boostingTableId = localStorage.getItem(HOTWORD_STORAGE_KEYS.boostingTableId) || '';
  const combineMode = localStorage.getItem(HOTWORD_STORAGE_KEYS.combineMode) === 'true';

  return { contextText, boostingTableId, combineMode };
}

// Save hotword configuration
export function saveHotwordConfig(config: HotwordConfig): void {
  localStorage.setItem(HOTWORD_STORAGE_KEYS.contextText, config.contextText);
  localStorage.setItem(HOTWORD_STORAGE_KEYS.boostingTableId, config.boostingTableId);
  localStorage.setItem(HOTWORD_STORAGE_KEYS.combineMode, String(config.combineMode));
}
```

Logika dalam kode ini lugas dan jelas. Kami membaca dari `localStorage` saat memuat konfigurasi, dan menulis ke `localStorage` saat menyimpannya. Kami juga menyediakan konfigurasi default agar sistem tetap dapat bekerja dengan baik ketika belum ada konfigurasi. Bagaimanapun, harus ada standar yang masuk akal.

### Ekstensi Konfigurasi SDK Backend

Dalam implementasi backend HagiCode, kami perlu menambahkan properti terkait kata cepat ke kelas konfigurasi SDK. Dengan mempertimbangkan karakteristik bahasa C# dan pola penggunaan, kami menggunakan `List<string>` untuk menyimpan konteks kata cepat khusus:

```csharp
public class DoubaoVoiceConfig
{

    /// </summary>
```

Desain kelas konfigurasi ini mengikuti gaya ringkas HagiCode yang biasa. `HotwordContexts` adalah tipe daftar yang dapat dibatalkan, dan `BoostingTableId` adalah string yang dapat dibatalkan, jadi jika tidak ada konfigurasi kata cepat, properti ini tidak berpengaruh sama sekali pada permintaan. Jika Anda tidak menggunakan fitur ini, sebaiknya jangan mengganggu.

### Logika Konstruksi Muatan

Konstruksi muatan adalah inti dari keseluruhan fitur kata cepat. Setelah kita memiliki konfigurasi kata cepat, kita perlu memformatnya ke dalam struktur JSON yang dibutuhkan oleh Doubao API. Proses ini terjadi sebelum SDK mengirimkan permintaan:

```csharp
private void AddCorpusToRequest(Dictionary<string, object> request)
{
    var corpus = new Dictionary<string, object>();

    // Add custom hotwords
    if (Config.HotwordContexts != null && Config.HotwordContexts.Count > 0)
    {
        corpus["context"] = new Dictionary<string, object>
        {
            ["context_type"] = "dialog_ctx",
            ["context_data"] = Config.HotwordContexts
                .Select(text => new Dictionary<string, object> { ["text"] = text })
                .ToList()
        };
    }

    // Add platform hotword table ID
    if (!string.IsNullOrEmpty(Config.BoostingTableId))
    {
        corpus["boosting_table_id"] = Config.BoostingTableId;
    }

    // Add corpus to the request only when it is not empty
    if (corpus.Count > 0)
    {
        request["corpus"] = corpus;
    }
}
```

Kode ini menunjukkan cara membangun secara dinamis `corpus` bidang berdasarkan konfigurasi. Poin kuncinya adalah kita menambahkan `corpus` bidang hanya jika konfigurasi kata cepat benar-benar ada. Desain ini memastikan kompatibilitas ke belakang: ketika tidak ada kata cepat yang dikonfigurasi, struktur permintaan tetap sama seperti sebelumnya. Kompatibilitas mundur itu penting; menambahkan fitur tidak boleh mengganggu logika yang ada.

### Penerus Parameter WebSocket

Antara frontend dan backend, parameter kata cepat diteruskan melalui pesan kontrol WebSocket. HagiCode dirancang sedemikian rupa sehingga ketika frontend mulai merekam, ia memuat konfigurasi kata cepat dari `localStorage` dan mengirimkannya ke backend melalui pesan WebSocket.

```typescript
const controlMessage = {
  type: 'control',
  payload: {
    command: 'StartRecognition',
    contextText: '高血压\n糖尿病\n冠心病',
    boosting_table_id: 'medical_table',
    combineMode: false
  }
};
```

Ada satu detail yang perlu diperhatikan di sini: frontend meneruskan teks multi-baris yang dipisahkan oleh karakter baris baru, dan backend perlu menguraikannya. Pengendali WebSocket backend mem-parsing parameter ini dan meneruskannya ke SDK:

```csharp
private async Task HandleControlMessageAsync(
    string connectionId,
    DoubaoSession session,
    ControlMessage message)
{
    if (message.Payload is SessionControlRequest controlRequest)
    {
        // Parse hotword parameters
        string? contextText = controlRequest.ContextText;
        string? boostingTableId = controlRequest.BoostingTableId;
        bool? combineMode = controlRequest.CombineMode;

        // Parse multi-line text into a hotword list
        if (!string.IsNullOrEmpty(contextText))
        {
            var hotwords = contextText
                .Split('\n', StringSplitOptions.RemoveEmptyEntries)
                .Select(s => s.Trim())
                .Where(s => s.Length > 0)
                .ToList();

            session.HotwordContexts = hotwords;
        }

        session.BoostingTableId = boostingTableId;
    }
}
```

Dengan desain ini, penerusan konfigurasi hotword dari frontend ke backend menjadi jelas dan efisien. Tidak ada sesuatu pun yang misterius tentang hal itu; data hanya dilewatkan lapis demi lapis.

Misalnya, jika Anda mengembangkan aplikasi yang berhubungan dengan medis, Anda dapat mengonfigurasinya seperti ini:

Dalam pengujian kami, setelah kata cepat dikonfigurasi, keakuratan pengenalan istilah profesional terkait meningkat secara signifikan. Peningkatannya nyata dan jelas lebih baik dari sebelumnya.

### Konfigurasikan Tabel Kata Cepat Platform

Jika Anda perlu mengelola kata cepat dalam jumlah besar, atau jika kata cepat perlu sering diperbarui, mode tabel kata cepat platform lebih cocok. Pertama, buat tabel kata cepat pada platform pembelajaran mandiri Doubao dan dapatkan hasilnya `boosting_table_id`, lalu masukkan ID ini di halaman pengaturan HagiCode.

Platform pembelajaran mandiri Doubao menyediakan kemampuan seperti impor massal dan manajemen kategori untuk kata cepat, yang sangat praktis bagi tim yang perlu mengelola sejumlah besar terminologi khusus. Dengan mengelola kata cepat di platform, Anda dapat mengelolanya secara terpusat dan meluncurkan pembaruan secara konsisten. Setelah daftar kata cepat menjadi besar, mempunyai satu tempat untuk mengelolanya jauh lebih praktis daripada entri manual setiap saat.

### Menggunakan Mode Kombinasi

Dalam beberapa skenario yang kompleks, Anda mungkin perlu menggunakan kata cepat khusus dan tabel kata cepat platform secara bersamaan. Dalam hal ini, cukup konfigurasikan keduanya di HagiCode dan aktifkan sakelar "Mode Kombinasi".

Dalam mode kombinasi, Doubao API mempertimbangkan kedua sumber kata cepat secara bersamaan, sehingga akurasi pengenalan biasanya lebih tinggi dibandingkan menggunakan salah satu sumber saja. Namun, perlu dicatat bahwa mode kombinasi meningkatkan kompleksitas permintaan, jadi yang terbaik adalah memutuskan apakah akan mengaktifkannya setelah pengujian praktis. Kompleksitas yang lebih besar hanya akan bermanfaat jika hasil di dunia nyata membenarkan hal tersebut.

### Contoh Integrasi Kode

Mengintegrasikan fitur kata cepat ke dalam proyek HagiCode sangatlah mudah. Berikut beberapa cuplikan kode yang umum digunakan:

```typescript
import {
  loadHotwordConfig,
  saveHotwordConfig,
  validateHotwordConfig,
  parseContextText,
  getEffectiveHotwordMode,
  type HotwordConfig
} from '@/types/hotword';

// Load and validate configuration
const config = loadHotwordConfig();
const validation = validateHotwordConfig(config);

if (!validation.isValid) {
  console.error('Hotword configuration validation failed:', validation.errors);
  return;
}

// Parse hotword text
const hotwords = parseContextText(config.contextText);
console.log('Parsed hotwords:', hotwords);

// Get effective hotword mode
const mode = getEffectiveHotwordMode(config);
console.log('Current hotword mode:', mode);
```

## Hal-hal yang Perlu Diingat

Ada beberapa hal yang patut mendapat perhatian khusus saat mengimplementasikan dan menggunakan fitur hotword.

Pertama adalah batas karakter. Doubao API memiliki batasan ketat pada kata cepat, termasuk jumlah baris, karakter per baris, dan jumlah karakter total. Jika ada batas yang terlampaui, API akan menampilkan kesalahan. Dalam implementasi frontend HagiCode, kami memeriksa batasan ini selama input pengguna melalui fungsi validasi, yang mencegah pengiriman konfigurasi yang tidak valid ke backend. Mengatasi masalah lebih awal selalu lebih baik daripada menunggu API gagal.

Yang kedua adalah formatnya `boosting_table_id`. Bidang ini hanya mengizinkan huruf, angka, garis bawah, dan tanda hubung, dan tidak boleh berisi spasi atau karakter khusus lainnya. Saat membuat tabel kata cepat di platform pembelajaran mandiri Doubao, pastikan untuk mengikuti aturan penamaan. Validasi format ketat seperti itu biasa terjadi pada API.

Ketiga adalah kompatibilitas ke belakang. Parameter kata cepat sepenuhnya opsional. Jika tidak ada kata cepat yang dikonfigurasi, sistem akan berperilaku persis seperti sebelumnya. Desain ini memastikan bahwa pengguna yang sudah ada tidak terpengaruh sama sekali, dan juga membuat migrasi dan peningkatan bertahap menjadi lebih mudah. Menambahkan fitur tidak boleh mengganggu logika sebelumnya.

Terakhir, ada penanganan kesalahan. Ketika konfigurasi kata cepat tidak valid, Doubao API mengembalikan pesan kesalahan yang sesuai. Implementasi HagiCode mencatat log terperinci untuk membantu pengembang memecahkan masalah. Pada saat yang sama, frontend menampilkan kesalahan validasi di UI untuk membantu pengguna memperbaiki konfigurasi. Penanganan kesalahan yang baik secara alami menghasilkan pengalaman pengguna yang lebih baik.
