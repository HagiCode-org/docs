## Bagaimana Desain Gamifikasi Membuat Pengodean AI Lebih Menyenangkan

> Alat pengkodean AI tradisional sebenarnya cukup kuat; mereka hanya kekurangan sedikit kehangatan. Saat kami membuat HagiCode, kami berpikir: jika kami tetap ingin menulis kode, mengapa tidak mengubahnya menjadi sebuah game?

## Latar Belakang

Siapa pun yang pernah menggunakan asisten pengkodean AI mungkin pernah mengalami pengalaman ini: awalnya terasa segar dan mengasyikkan, tetapi setelah beberapa saat mulai terasa ada sesuatu yang hilang. Alat itu sendiri sangat kuat, mampu menghasilkan kode, pelengkapan otomatis, dan perbaikan bug, tapi... tidak terasa terlalu hangat, dan lama kelamaan bisa menjadi monoton dan membosankan.

Itu saja sudah cukup membuat Anda bertanya-tanya siapa yang ingin menatap alat yang dingin dan impersonal setiap hari.

Ini seperti bermain game. Jika yang Anda lakukan hanyalah menyelesaikan daftar tugas, tanpa pertumbuhan karakter, tanpa pencapaian yang dibuka, dan tanpa koordinasi tim, hal itu akan segera berhenti menyenangkan. Hal-hal dan manusia yang indah tidak perlu dimiliki untuk dihargai; kecantikan mereka saja sudah cukup. Alat pemrograman bahkan tidak menawarkan keindahan seperti itu, sehingga mudah untuk berkecil hati.

Kami mengalami masalah ini saat melakukan pengembangan [Kode Hagi](https://hagicode.com). Sebagai platform kolaborasi asisten multi-AI, HagiCode perlu menjaga keterlibatan pengguna dalam jangka panjang. Namun pada kenyataannya, bahkan alat yang hebat pun sulit untuk digunakan jika tidak memiliki hubungan emosional.

Untuk mengatasi permasalahan ini, kami mengambil keputusan berani: mengubah pemrograman menjadi sebuah game. Bukan jenis yang dangkal dengan papan peringkat poin sederhana, tetapi pengalaman bermain peran yang sebenarnya. Dampak dari keputusan tersebut mungkin lebih besar dari yang Anda bayangkan.

Bagaimanapun, manusia membutuhkan sedikit ritual dalam hidupnya.

## Tentang HagiCode

Ide-ide yang dibagikan dalam artikel ini berasal dari pengalaman praktis kami di [Kode Hagi](https://hagicode.com) proyek. HagiCode adalah platform kolaborasi asisten multi-AI yang mendukung Claude Code, Codex, Copilot, OpenCode, dan asisten AI lainnya yang bekerja bersama. Jika Anda tertarik dengan kolaborasi multi-AI atau pemrograman gamified, kunjungi [github.com/HagiCode-org/site](https://github.com/HagiCode-org/site) untuk mempelajari lebih lanjut.

Tidak ada sesuatu pun yang misterius tentang hal itu. Kami hanya mengubah pemrograman menjadi sebuah petualangan.

## Mengapa Memilih Gamifikasi

Inti dari gamifikasi bukan sekadar "menambahkan papan peringkat". Ini tentang membangun sistem insentif yang lengkap sehingga pengguna dapat merasakan pertumbuhan, pencapaian, dan pengakuan sosial saat melakukan tugas.

Desain gamifikasi HagiCode berkisar pada satu ide inti: setiap asisten AI adalah "Pahlawan", dan pengguna adalah kapten tim Pahlawan ini. Anda memimpin Pahlawan ini untuk menaklukkan berbagai "Ruang Bawah Tanah" (tugas pemrograman). Sepanjang perjalanan, Pahlawan mendapatkan pengalaman, naik level, membuka kemampuan, dan tim Anda juga memperoleh pencapaian.

Ini bukan gimmick. Ini adalah desain yang didasarkan pada psikologi perilaku manusia. Ketika tugas diberikan makna dan umpan balik kemajuan, keterlibatan dan ketekunan orang-orang meningkat secara signifikan.

Seperti kata pepatah lama, "Perasaan ini bisa menjadi kenangan, meski pada saat itu membuat kita bingung." Kami menghadirkan pengalaman emosional tersebut ke dalam alat ini, sehingga pemrograman tidak lagi sekadar mengetik kode, namun sebuah perjalanan yang patut diingat.

## Sistem Karakter Pahlawan

Hero adalah konsep inti dalam sistem gamifikasi HagiCode. Setiap Pahlawan mewakili satu asisten AI. Misalnya Claude Code adalah Pahlawan, dan Codex juga merupakan Pahlawan.

### Tiga Slot Pahlawan

Seorang Pahlawan memiliki tiga slot perlengkapan, dan desainnya ternyata sangat elegan:

1. **Slot CLI (kelas utama)**: Menentukan kemampuan dasar Pahlawan, misalnya Claude Code atau Codex
2. **Model slot (kelas menengah)**: Menentukan model mana yang digunakan, misalnya Claude 4.5 atau Claude 4.6
3. **Slot gaya (gaya)**: Menentukan gaya perilaku Pahlawan, seperti "Ahli Strategi Fengluo" atau gaya lainnya

Kombinasi ketiga slot ini menciptakan konfigurasi Hero yang unik. Sama seperti pembuatan peralatan dalam game, Anda memilih pengaturan yang tepat berdasarkan tugas. Bagaimanapun, apa yang paling cocok untuk Anda adalah yang paling penting. Kehidupannya serupa: banyak jalan menuju Roma, namun ada pula yang lebih mulus dibandingkan yang lain.

### Sistem Pertumbuhan Pahlawan

Setiap Pahlawan memiliki XP dan levelnya sendiri:

```typescript
type HeroProgressionSnapshot = {
  currentLevel: number;                    // Current level
  totalExperience: number;                 // Total experience
  currentLevelStartExperience: number;     // Experience at the start of the current level
  nextLevelExperience: number;             // Experience required for the next level
  experienceProgressPercent: number;       // Progress percentage
  remainingExperienceToNextLevel: number;  // Experience still needed for the next level
  lastExperienceGain: number;              // Most recent experience gained
  lastExperienceGainAtUtc?: string | null; // Time when experience was gained
};
```

Level dibagi menjadi empat tahap, dan setiap tahap memiliki nama yang mendalam:

```typescript
export const resolveHeroProgressionStage = (level?: number | null): HeroProgressionStage => {
  const normalizedLevel = Math.max(1, level ?? 1);
  if (normalizedLevel <= 100) return 'rookieSprint';      // Rookie sprint
  if (normalizedLevel <= 300) return 'growthRun';         // Growth run
  if (normalizedLevel <= 700) return 'veteranClimb';      // Veteran climb
  return 'legendMarathon';                                // Legend marathon
};
```

Dari "pemula" hingga "legenda", jalur pertumbuhan ini memberi pengguna gambaran yang jelas tentang arah dan pencapaian. Ini mencerminkan pertumbuhan pribadi dalam hidup, dari kebingungan hingga kedewasaan, hanya menjadi lebih nyata di sini.

### Membuat Pahlawan Kustom

Untuk membuat Pahlawan, Anda perlu mengonfigurasi tiga slot:

```typescript
const heroDraft: HeroDraft = {
  name: 'Athena',
  icon: 'hero-avatar:storm-03',
  description: 'A brilliant strategist',
  executorType: AIProviderType.CLAUDE_CODE_CLI,
  slots: {
    cli: {
      id: 'profession-claude-code',
      parameters: { /* CLI-related parameters */ }
    },
    model: {
      id: 'secondary-claude-4-sonnet',
      parameters: { /* Model-related parameters */ }
    },
    style: {
      id: 'fengluo-strategist',
      parameters: { /* Style-related parameters */ }
    }
  }
};
```

Setiap Pahlawan memiliki avatar, deskripsi, dan identitas profesional yang unik, yang memberikan kepribadian dan kehangatan lebih pada asisten AI yang dingin. Lagi pula, siapa yang mau bekerja dengan alat yang tidak memiliki karakter?

## Sistem Penjara Bawah Tanah

"Penjara Bawah Tanah" adalah konsep permainan klasik yang mewakili tantangan yang harus diselesaikan oleh tim. Di HagiCode, setiap alur kerja adalah Dungeon.

### Bagaimana Ruang Bawah Tanah Diatur

Dungeon mengatur alur kerja ke dalam "Ruang Bawah Tanah" yang berbeda:

- **Ruang bawah tanah pembuatan proposal**: Bertanggung jawab untuk menghasilkan proposal teknis
- **Ruang bawah tanah eksekusi proposal**: Bertanggung jawab untuk melaksanakan tugas dalam proposal
- **Ruang bawah tanah arsip proposal**: Bertanggung jawab untuk mengatur dan mengarsipkan proposal yang sudah selesai

Setiap ruang bawah tanah memiliki Kapten Pahlawannya sendiri, dan kapten secara otomatis dipilih sebagai Pahlawan pertama yang diaktifkan.

Ini sebenarnya hanyalah pembagian kerja, seperti dalam kehidupan sehari-hari, hanya saja berubah menjadi mekanik permainan.

### Mekanisme Kolaborasi Tim

Anda dapat mengonfigurasi regu Pahlawan yang berbeda untuk ruang bawah tanah yang berbeda:

```typescript
const dungeonRoster: HeroDungeonRoster = {
  scriptKey: 'proposal.generate',
  displayName: 'Proposal Generation',
  members: [
    { heroId: 'hero-1', name: 'Athena', executorType: 'ClaudeCode' },
    { heroId: 'hero-2', name: 'Apollo', executorType: 'Codex' }
  ]
};
```

Misalnya, Anda dapat menggunakan Athena untuk menghasilkan proposal karena bagus dalam hal strategi, dan Apollo untuk mengimplementasikan kode karena bagus dalam eksekusi. Dengan begitu, setiap Hero bisa memainkan kelebihannya. Ini seperti membentuk sebuah band: setiap orang mempunyai alat musik, dan bersama-sama mereka menciptakan sesuatu yang indah.

### Kontrol Aliran Bawah Tanah

Penggunaan ruang bawah tanah sudah diperbaiki `scriptKey` nilai untuk mengidentifikasi alur kerja yang berbeda:

```typescript
// Script keys map to different workflows
const dungeonScripts = {
  'proposal.generate': 'Proposal Generation',
  'proposal.execute': 'Proposal Execution',
  'proposal.archive': 'Proposal Archive'
};
```

Alur status tugas adalah: `queued` (menunggu) -> `dispatching` (sedang ditugaskan) -> `dispatched` (ditugaskan). Seluruh proses dilakukan secara otomatis dan tidak memerlukan intervensi manual. Itu juga bagian dari sisi malas kita, karena siapa yang mau mengurus barang ini dengan tangan?

## XP dan Sistem Level

XP adalah mekanisme umpan balik inti dalam sistem gamifikasi. Pengguna mendapatkan XP dengan menyelesaikan tugas, XP menaikkan level Pahlawan, dan naik level akan membuka kemampuan baru, membentuk putaran umpan balik positif.

### Cara Mendapatkan XP

Di HagiCode, XP dapat diperoleh melalui aktivitas berikut:

- Menyelesaikan eksekusi kode
- Berhasil memanggil alat
- Menghasilkan proposal
- Operasi manajemen sesi
- Operasi proyek

Setiap kali tindakan valid diselesaikan, Pahlawan yang bersangkutan memperoleh XP. Sama seperti pertumbuhan dalam kehidupan, setiap langkah berarti, hanya di sini pertumbuhan tersebut diukur.

### Visualisasi Kemajuan Waktu Nyata

XP dan kemajuan level divisualisasikan secara real time:

```typescript
type HeroDungeonMember = {
  heroId: string;
  name: string;
  icon?: string | null;
  executorType: PCode_Models_AIProviderType;
  currentLevel?: number;                    // Current level
  totalExperience?: number;                 // Total experience
  experienceProgressPercent?: number;       // Progress percentage
};
```

Pengguna selalu dapat melihat level dan kemajuan masing-masing Pahlawan, dan umpan balik langsung adalah kunci desain gamifikasi. Masyarakat membutuhkan umpan balik, jika tidak, bagaimana mereka tahu bahwa mereka mengalami kemajuan?

## Sistem Prestasi

Prestasi adalah elemen penting lainnya dalam gamifikasi. Mereka memberikan tujuan jangka panjang dan kepuasan yang didorong oleh pencapaian.

### Jenis Prestasi

HagiCode mendukung berbagai jenis pencapaian:

- **Pencapaian pembuatan kode**: Hasilkan X baris kode, hasilkan file Y
- **Prestasi manajemen sesi**: Selesaikan percakapan Z
- **Pencapaian operasi proyek**: Bekerja di seluruh proyek W

Pencapaian ini benar-benar seperti tonggak sejarah dalam hidup, hanya saja kami mengubahnya menjadi mekanisme permainan.

### Status Prestasi

Pencapaian memiliki tiga status:

```typescript
type AchievementStatus = 'unlocked' | 'in-progress' | 'locked';
```

Ketiga negara bagian tersebut memiliki perbedaan visual yang jelas:

- **Tidak terkunci**: Gradien emas dengan efek halo
- **Sedang berlangsung**: Animasi pulsa biru
- **Terkunci**: Abu-abu, dengan kondisi buka kunci ditampilkan

Setiap pencapaian dengan jelas menampilkan kondisi pemicunya, sehingga pengguna mengetahui apa yang harus dilakukan selanjutnya. Ketika orang merasa tersesat, sedikit bimbingan selalu membantu.

### Efek Perayaan saat Membuka Kunci

Saat pencapaian dibuka, animasi perayaan akan dipicu. Penguatan positif semacam itu memberi pengguna perasaan "Saya berhasil" yang memuaskan dan memotivasi mereka untuk terus maju. Imbalan kecil dalam hidup bekerja dengan cara yang sama: mungkin kecil, tapi kebahagiaan bisa bertahan lama.

## Laporan Pertempuran Laporan Pertempuran Harian

Battle Report adalah salah satu fitur khas HagiCode. Di penghujung hari, ini menghasilkan laporan gaya pertempuran layar penuh.

### Laporkan Konten

Laporan Pertempuran menampilkan informasi berikut:

```typescript
type HeroBattleReport = {
  reportDate: string;
  summary: {
    totalHeroCount: number;        // Total number of Heroes
    activeHeroCount: number;       // Number of active Heroes
    totalBattleScore: number;      // Total battle score
    mvp: HeroBattleHero;           // Most valuable Hero
  };
  heroes: HeroBattleHero[];        // Detailed data for all Heroes
};
```

- Skor total tim
- Jumlah Pahlawan aktif
- Jumlah panggilan alat
- Jumlah waktu kerja
- MVP (Pahlawan Paling Berharga)
- Kartu terperinci untuk setiap Pahlawan

### Tampilan Sorotan MVP

MVP adalah Pahlawan dengan performa terbaik saat ini dan disorot dalam laporan. Ini bukan sekadar data statistik, melainkan bentuk penghormatan dan pengakuan. Lagi pula, siapa yang tidak ingin diakui?

### Kartu Pahlawan Terperinci

Setiap kartu Pahlawan meliputi:

- Kemajuan tingkat
- XP diperoleh
- Jumlah eksekusi
- Waktu penggunaan

Metrik ini membantu pengguna memahami dengan jelas bagaimana kinerja tim. Melihat hasil usaha sendiri adalah sebuah kepuasan tersendiri.

## Teknis Implementasi

Sistem gamifikasi HagiCode menggunakan tumpukan teknologi modern dan pola desain. Tidak ada yang ajaib tentang hal itu; kami tinggal memilih alat yang sesuai dengan pekerjaan.

### Pilihan Tumpukan Teknologi

```typescript
// React + TypeScript for the frontend
import React from 'react';

// Framer Motion for animations
import { AnimatePresence, motion } from 'framer-motion';

// Redux Toolkit for state management
import { useAppDispatch, useAppSelector } from '@/store';

// shadcn/ui for UI components
import { Dialog, DialogContent } from '@/components/ui/dialog';
```

Framer Motion menangani semua efek animasi, shadcn/ui menyediakan komponen UI dasar, dan Redux Toolkit mengelola status gamifikasi yang kompleks. Alat yang bagus menghasilkan pekerjaan yang bagus.

### Sistem Desain UI Gamifikasi

HagiCode menggunakan gaya desain Glassmorphism + Tech Dark:

```css
/* Primary gradient */
background: linear-gradient(135deg, #22C55E 0%, #25c2a0 50%, #06b6d4 100%);

/* Glass effect */
backdrop-filter: blur(12px);

/* Glow effect */
background: radial-gradient(circle at center, rgba(34, 197, 94, 0.15) 0%, transparent 70%);
```

Gradien hijau dipadukan dengan morfisme kaca menciptakan suasana teknis dan futuristik. Keindahan visual juga merupakan bagian dari pengalaman pengguna.

### Efek Animasi

Framer Motion digunakan untuk membuat animasi masuk yang halus:

```typescript
<motion.div
  animate={{ opacity: 1, y: 0 }}
  initial={{ opacity: 0, y: 18 }}
  transition={{ duration: 0.35, ease: 'easeOut', delay: index * 0.08 }}
  className="card"
>
  {/* Card content */}
</motion.div>
```

Setiap kartu masuk satu demi satu dengan penundaan 0,08 detik, menciptakan efek visual yang mengalir. Animasi halus meningkatkan pengalaman. Bagian itu sulit untuk diperdebatkan.

### Persistensi Data

Data gamifikasi disimpan menggunakan sistem penyimpanan Grain untuk memastikan konsistensi status. Bahkan data terperinci seperti akumulasi Hero XP dapat disimpan secara akurat. Tidak ada seorang pun yang ingin kehilangan pengalaman yang telah mereka peroleh dengan kerja keras.

## Panduan Praktis

### Ciptakan Pahlawan Pertama Anda

Membuat Hero pertama Anda sebenarnya cukup sederhana:

1. Buka halaman manajemen Pahlawan
2. Klik tombol "Buat Pahlawan".
3. Konfigurasikan tiga slot (CLI, Model, Style)
4. Beri nama dan deskripsi pada Pahlawan
5. Simpan, dan Pahlawan pertama Anda lahir

Ini seperti bertemu teman baru: Anda memberi mereka nama, mempelajari apa yang membuat mereka istimewa, dan kemudian memulai petualangan bersama.

### Bangun Tim Penjara Bawah Tanah

Membangun tim juga sederhana:

1. Buka halaman manajemen Dungeon
2. Pilih ruang bawah tanah yang ingin Anda konfigurasi, seperti "Pembuatan Proposal"
3. Pilih anggota dari daftar Pahlawan Anda
4. Sistem secara otomatis memilih Pahlawan pertama yang diaktifkan sebagai Kapten
5. Simpan konfigurasi

Ini hanyalah proses pembentukan sebuah tim, seperti membangun sebuah tim dalam kehidupan nyata di mana setiap orang memiliki perannya masing-masing.

### Lihat Laporan Harian

Di penghujung hari, Anda dapat melihat Laporan Pertempuran hari itu:

1. Klik tombol "Laporan Pertempuran".
2. Lihat hasil pekerjaan hari itu dalam tampilan layar penuh
3. Cek MVP dan data detail tiap Hero
4. Bagikan dengan anggota tim jika Anda mau

Ini juga semacam ritual, cara untuk melihat seberapa besar usaha yang Anda lakukan hari ini dan seberapa jauh Anda masih dari tujuan.

## Catatan dan Praktik Terbaik

### Optimasi Kinerja

Gunakan `React.memo` untuk menghindari rendering ulang yang tidak perlu:

```typescript
const HeroCard = React.memo(({ hero }: { hero: HeroDungeonMember }) => {
  // Component implementation
});
```

Kinerja juga penting. Tidak ada yang mau menggunakan alat yang lamban.

### Gerakan Dapat Menurun dengan Baik

Deteksi pengaturan preferensi gerakan pengguna dan berikan pengalaman yang disederhanakan bagi pengguna yang sensitif terhadap gerakan:

```typescript
const prefersReducedMotion = useReducedMotion();
const duration = prefersReducedMotion ? 0 : 0.35;
```

Tidak semua orang menyukai animasi, dan menghormati preferensi pengguna adalah bagian dari desain yang bagus.

### Kompatibilitas Mundur

Simpan `legacyIds` untuk mendukung migrasi dari versi yang lebih lama:

```typescript
type HeroDungeonMember = {
  heroId: string;
  legacyIds?: string[];  // Supports legacy ID mapping
  // ...
};
```

Tidak ada seorang pun yang ingin kehilangan data hanya karena peningkatan versi.

### Dukungan Internasionalisasi

Gunakan kunci terjemahan i18n untuk semua teks guna memudahkan dukungan multi-bahasa:

```typescript
const displayName = t(`dungeon.${scriptKey}`, { defaultValue: displayName });
```

Bahasa tidak boleh menjadi penghalang dalam menggunakan produk.

## Ringkasan

Gamifikasi bukan sekadar papan peringkat poin sederhana, tetapi sistem insentif yang lengkap. Melalui sistem Pahlawan, sistem Dungeon, sistem XP dan level, sistem pencapaian, dan Laporan Pertempuran, HagiCode mengubah pekerjaan pemrograman menjadi perjalanan heroik yang penuh petualangan.

Nilai inti dari sistem ini terletak pada:

- **Hubungan emosional**: Memberikan kepribadian asisten AI yang dingin
- **Umpan balik positif**: Setiap tindakan menghasilkan umpan balik langsung
- **Tujuan jangka panjang**: Level dan pencapaian memberikan jalur pertumbuhan
- **Identitas tim**: Rasa kolaborasi dalam tim Dungeon
- **Kehormatan dan pengakuan**: Laporan Pertempuran dan pertunjukan MVP

Desain gamifikasi membuat pemrograman tidak lagi membosankan, tetapi menjadi petualangan yang menarik. Saat menyelesaikan tugas pengkodean, pengguna juga merasakan kesenangan dari pertumbuhan karakter, kolaborasi tim, dan pembukaan pencapaian, yang meningkatkan retensi dan aktivitas.

Pada intinya, pemrograman sudah merupakan suatu tindakan penciptaan. Kami baru saja membuat proses kreatif sedikit lebih menyenangkan.
