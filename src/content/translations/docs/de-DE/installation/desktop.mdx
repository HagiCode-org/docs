In diesem Handbuch wird die Installation und Verwendung von HagiCode Desktop behandelt. Es ist der offizielle Desktop-Client und behält den standardmäßigen lokalen Setup-Pfad sowie die täglichen Kontrollen an einem Ort.

:::tip[Empfohlene Methode]
Desktop ist der standardmäßige lokale Pfad für einzelne Benutzer und Entwickler, weil:
- Das Installationsprogramm behält den Standardablauf in einem geführten Pfad bei
- Abhängigkeiten und Dienste werden in der App verwaltet
- Die Benutzeroberfläche erleichtert die Überprüfung von Status und Vorgängen
- Die integrierte Versionsverwaltung erleichtert den Wechsel
- Daten bleiben standardmäßig lokal
:::

## Was ist HagiCode Desktop?

HagiCode Desktop ist die offizielle Desktop-Anwendung von HagiCode, die für Windows-, macOS- und Linux-Systeme entwickelt wurde. Es bündelt die Kernfunktionen von HagiCode und das Installationsprogramm behält den Standard-Setup-Pfad in einem geführten Ablauf bei.

### Hauptvorteile

- **Ein-Klick-Installation**: Führen Sie alle Konfigurationsschritte automatisch über den Installationsassistenten aus
- **Automatische Laufzeitvorbereitung**: Das Standardinstallationsprogramm enthält bereits die benötigte Laufzeit, sodass Sie keine separate installieren müssen `.NET` Laufzeit zuerst
- **Dienstverwaltung**: Integrierte HagiCode-Serververwaltung, einfaches Starten und Stoppen von Diensten
- **Versionswechsel**: Unterstützung für die Verwaltung mehrerer Versionen, freier Wechsel zwischen verschiedenen Versionen
- **Lokaler Betrieb**: Alle Daten werden lokal gespeichert, um Ihre Privatsphäre zu schützen
- **Automatische Updates**: Erkennt automatisch die neueste Version und lädt sie herunter

### Anwendbare Szenarien

Desktop passt gut, wenn:
- Einzelne Entwickler, die HagiCode lokal nutzen möchten
- Benutzer, die HagiCode offline verwenden müssen
- Nicht-technische Benutzer, die den Installationsprozess vereinfachen möchten
- Entwickler, die häufig zwischen verschiedenen Versionen wechseln müssen

### Andere Download-Methoden

Sie können das HagiCode Desktop-Installationsprogramm auch über die folgenden Kanäle beziehen:

1. **Offizielle Veröffentlichungsseite**: Besuchen Sie [HagiCode GitHub-Veröffentlichungen](https://github.com/HagiCode-org/releases) um die neueste Version herunterzuladen

### Wählen Sie die richtige Version

Wählen Sie das entsprechende Installationsprogramm für Ihr Betriebssystem:
- **Windows**: Laden Sie die herunter `.exe` Installationsprogramm
- **macOS**: Laden Sie die herunter `.dmg` Installationsprogramm
- **Linux**: Laden Sie die herunter `.AppImage` oder `.tar.gz` Installationspaket

:::Hinweis[Systemanforderungen]
- **Windows**: Windows 10 oder höhere Version (64-Bit)
- **macOS**: macOS 10.15 oder höher
- **Linux**: Mainstream-Linux-Distributionen (Ubuntu 20.04+, Fedora 33+ usw.)
:::

:::tip[Standardvoraussetzungen]
Für den neuesten lokalen HagiCode-Installationsablauf müssen Sie normalerweise keinen separaten vorbereiten `.NET` Laufzeit. Desktop bündelt bereits die für den Standardablauf erforderliche Laufzeit und initialisiert standardmäßig den unterstützten lokalen SQLite-Pfad.
:::

## Installations- und Starthinweise

:::Vorsicht[Hinweis zum macOS Gatekeeper]
Wenn Sie die App bereits im Standardpfad installiert haben `/Applications/HagiCode.app` aber macOS zeigt beim ersten Start „App ist beschädigt“, „kann nicht geöffnet werden“ oder eine ähnliche Gatekeeper-Warnung an, die übliche Ursache ist `com.apple.quarantine` Attribut, das macOS zu aus dem Internet heruntergeladenen Apps hinzufügt, und kein Fehler im HagiCode Desktop-Installationsprogramm selbst.

Führen Sie den folgenden Befehl im Terminal aus. Es erfordert Administratorrechte und entfernt rekursiv das Quarantäneattribut aus der App im Standardinstallationspfad:

```bash
sudo xattr -dr com.apple.quarantine /Applications/HagiCode.app
```

Starten Sie HagiCode Desktop nach Abschluss des Befehls neu.
:::

## Führen Sie Desktop aus

Öffnen Sie nach dem Herunterladen das Installationsprogramm oder Anwendungspaket und befolgen Sie die Anweisungen auf dem Bildschirm. Desktop bereitet die Laufzeit im aktuell unterstützten lokalen Pfad vor, verwaltet den HagiCode-Server und zeigt Fortschritt und Status an, wenn Netzwerk-Downloads erforderlich sind.

:::note[Netzwerkanforderungen]
Stellen Sie sicher, dass Ihre Netzwerkverbindung verfügbar ist. Desktop muss möglicherweise Versionsinformationen oder Komponentenpakete aus dem Internet abrufen. Wenn ein Download fehlschlägt, überprüfen Sie Ihre Netzwerk-, Proxy- oder Firewall-Einstellungen und versuchen Sie es erneut.
:::

## Versionsverwaltung

HagiCode Desktop unterstützt die Verwaltung mehrerer Versionen, sodass Sie frei zwischen verschiedenen Versionen wechseln können.

![Versionsverwaltung](../../img/screenshots/installation/install-desktop/12-version-management/original.png)

### Version wechseln

1. Öffnen Sie den HagiCode-Desktop
2. Gehen Sie zur Seite „Versionsverwaltung“.
3. Wählen Sie die Version aus, die Sie wechseln möchten
4. Klicken Sie auf die Schaltfläche „Version wechseln“.
5. Warten Sie, bis der Wechsel abgeschlossen ist

:::note[Hinweise zum Versionswechsel]
- Beim Wechseln der Versionen wird HagiCode Server automatisch neu gestartet
- Stellen Sie sicher, dass Sie alle Arbeiten speichern, bevor Sie wechseln
- Projektkonfigurationen können je nach Version unterschiedlich sein
:::

## Dienst starten und stoppen

HagiCode Desktop bietet eine einfache Möglichkeit, den HagiCode-Serverdienst zu verwalten.

### Dienst starten

Klicken Sie auf der Startseite der Desktop-Anwendung auf die Schaltfläche „Dienst starten“, um HagiCode Server zu starten.

![Dienst starten](../../img/screenshots/installation/install-desktop/13-start-service-homepage/original.png)

### Stoppen Sie den Dienst

Um den HagiCode-Server zu stoppen, klicken Sie auf der Startseite auf die Schaltfläche „Dienst beenden“.

:::tip[Servicestatus]
Die Desktop-Anwendung zeigt den Betriebsstatus von HagiCode Server in Echtzeit an, einschließlich:
- Dienststatus (läuft/gestoppt)
- Service-Portnummer
- Aktuelle Versionsinformationen
:::

## Fehlerbehebung

### Installationsfehler

Wenn während der Installation Probleme auftreten:

1. **Netzwerkverbindung prüfen**: Stellen Sie sicher, dass Sie auf das Internet zugreifen können
2. **Festplattenspeicher prüfen**: Stellen Sie sicher, dass Sie über ausreichend Speicherplatz verfügen
3. **Als Administrator ausführen**: Klicken Sie unter Windows mit der rechten Maustaste auf das Installationsprogramm und wählen Sie „Als Administrator ausführen“
4. **Protokolldateien anzeigen**: Das Installationsprogramm generiert detaillierte Protokolldateien, die bei der Diagnose von Problemen helfen können

### Fehler bei der Abhängigkeitsinstallation

Wenn die Abhängigkeitsinstallation fehlschlägt:

1. **Systemkompatibilität prüfen**: Stellen Sie sicher, dass Ihr Betriebssystem die Mindestanforderungen erfüllt
2. **Überprüfen Sie die Netzwerk- und Firewall-Einstellungen**: Stellen Sie sicher, dass das Installationsprogramm auf erforderliche Downloads zugreifen kann und nicht von Ihrer Firewall blockiert wird
3. **Installationsberechtigungen bestätigen**: Versuchen Sie unter Windows, das Installationsprogramm als Administrator auszuführen, damit es über ausreichende Berechtigungen verfügt
4. **Starten Sie die App neu und versuchen Sie es erneut**: Schließen Sie HagiCode Desktop, öffnen Sie es erneut und wiederholen Sie dann den Installationsschritt
5. **Protokolle anzeigen**: Überprüfen Sie detaillierte Fehlerprotokolle in der Desktop-Anwendung

### Dienststartfehler

Wenn der HagiCode-Server nicht startet:

1. **Portbelegung prüfen**: Stellen Sie sicher, dass Port 45000 nicht durch andere Programme belegt ist
2. **Überprüfen Sie die Firewall-Einstellungen**: Stellen Sie sicher, dass die Firewall den Netzwerkzugriff auf den HagiCode-Server zulässt
3. **Anwendung neu starten**: Versuchen Sie, HagiCode Desktop neu zu starten
4. **Protokolle anzeigen**: Detaillierte Fehlerprotokolle in der Desktop-Anwendung anzeigen

### Benötigen Sie weitere Hilfe?

Wenn Sie auf Probleme stoßen, die hier nicht behandelt werden:

1. Überprüfen [GitHub-Probleme](https://github.com/HagiCode-org/site/issues) für ähnliche Probleme
2. Besuchen Sie unsere [Community-Forum](https://github.com/HagiCode-org/site/discussions) um Hilfe
3. Reichen Sie ein neues Problem ein und beschreiben Sie Ihr Problem detailliert
