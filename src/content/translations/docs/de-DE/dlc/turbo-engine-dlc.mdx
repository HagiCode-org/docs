Turbo Engine ist das zentrale Erweiterungs-Add-on für HagiCode Desktop, konzipiert für Benutzer, die in die Phase der intensiven Nutzung eingetreten sind. Es löst eine Sache: **HagiCode vom Basismodus in den Hochintensitäts-Arbeitsmodus zu bringen.**

## Kaufmöglichkeit

Turbo Engine ist eine einmalige permanente Lizenz im Microsoft Store (Store-ID: `9NSD809W18Z6`), die nach dem Kauf dauerhaft gültig ist. Sie können die Lizenz direkt im Abonnement-Arbeitsbereich von HagiCode Desktop erwerben und den Status aktualisieren.

## Freigeschaltete Funktionen

Nach dem Kauf von Turbo Engine werden folgende Funktionen dauerhaft freigeschaltet:

| Funktion | Beschreibung |
| --- | --- |
| **Vorschlag-Parallelität 3 → 32** | Generierung, Ausführung und Archivierung von Vorschlägen teilen sich dieses Limit. 32 parallele Vorschläge ermöglichen stabile parallele Bearbeitung mehrerer Aufgabenlinien. |
| **Erweiterte Dokumentthemen** | Wechseln Sie Dokumentthemen in der Abenteurergruppen-Verwaltung und sehen Sie Begriffe und Texte unter verschiedenen Themen in Echtzeit. |
| **Benutzerdefinierte Git-Commit-Signaturen** | Legen Sie ein Standard-`Co-Authored-By`-Muster fest oder überschreiben Sie es temporär für einzelne AI Compose Commit-Durchläufe. |
| **Marken-Anpassung** | Laden Sie ein Header-Logo hoch und legen Sie Anwendungstitel, Browsertitel und Marken-Linkziel fest. |
| **Benutzerdefinierte Avatar-Uploads** | Laden Sie einen benutzerdefinierten Avatar für den aktuellen Helden hoch oder verwenden Sie Einträge aus der gemeinsam genutzten Avatar-Bibliothek in Erstellungs-, Bearbeitungs- und Massenerstellungsprozessen wieder. |
| **DLC-Avatar-Ressourcen** | Turbo Engine Avatar-Pakete sind in die Hero Avatar Gallery und Massenerstellungs-Workflows integriert und nehmen direkt an der Workflow-Auswahl teil. |

## Wer es braucht

- Benutzer, die häufig mehrere Vorschläge oder Aufgaben gleichzeitig bearbeiten.
- Power-User, für die das standardmäßige Limit von 3 parallelen Vorschlägen nicht mehr ausreicht.
- Benutzer, die benutzerdefinierte Git-Signaturen, Dokumentthemen, Avatare und Markenidentität benötigen.
