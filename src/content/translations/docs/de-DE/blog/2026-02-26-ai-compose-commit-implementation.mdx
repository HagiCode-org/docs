## AI Compose Commit: KI-gestelltes Refactoring des Git-Commit-Workflows

## Einleitung

In der Softwareentwicklung ist das Committen von Code eine tägliche Aufgabe. Doch wer kennt nicht das Szenario: Nach einem langen Arbeitstag öffnet man Git und sieht dutzende ungestagte Dateien, ohne zu wissen, wie man sie in sinnvolle Commits organisieren soll.

Der traditionelle Weg besteht darin, Dateien manuell in Gruppen zu stagen, einzeln zu committen und Commit-Nachrichten zu verfassen – ein zeitaufwändiger und fehleranfälliger Prozess.

Im HagiCode-Projekt haben wir eine neue Funktion eingeführt – AI Compose Commit – die diesen Workflow grundlegend verändern soll. Sie analysiert KI-gestützt alle uncommitteten Änderungen im Arbeitsverzeichnis, gruppiert sie automatisch in mehrere logische Commits und führt konforme Commit-Operationen aus. Dieser Artikel beleuchtet die Implementierungsprinzipien, die technische Architektur sowie die Herausforderungen und Lösungen aus unserer Praxis.

## Über HagiCode

## Hintergrund

### Schmerzpunkte des traditionellen Git-Commits

Git bietet als Versionskontrollsystem leistungsstarke Codeverwaltungsfähigkeiten. In der Praxis wird der Commit-Vorgang jedoch oft zum Engpass:
1. **Manuelles Gruppieren zeitaufwändig**: Bei vielen Dateiänderungen muss jeder Entwickler den Inhalt prüfen und semantische Zusammenhänge erkennen.
2. **Inkonsistente Commit-Nachrichten**: Conventional-Commits-konforme Nachrichten erfordern Erfahrung; Anfänger schreiben oft unregelmäßige Nachrichten.
3. **Komplexe Multi-Repository-Verwaltung**: In Monorepos muss zwischen Repositories gewechselt werden.
4. **Unterbrochener Workflow**: Das Committen unterbricht den Entwicklungsfluss.

### Trend der KI-gestützten Entwicklung

KI-Technologie wird in der Softwareentwicklung zunehmend vielfältiger eingesetzt. Bei Git-Workflows bieten einige Tools zwar Commit-Nachrichtengenerierung, aber meist nur für einzelne Commits, ohne intelligente Analyse und Gruppierung des gesamten Arbeitsverzeichnisses.

HagiCodes AI Compose Commit wurde genau entwickelt, um diese Lücke zu schließen – nicht nur Commit-Nachrichten zu generieren, sondern den gesamten Prozess von der Dateianalyse bis zur Ausführung der Commits zu übernehmen.

## Problem

### Technische Herausforderungen

1. **Semantisches Dateiverständnis**: Die KI muss die semantischen Beziehungen von Dateiänderungen verstehen.
2. **Commit-Gruppierungsstrategie**: Nach Funktion, Modul oder Dateityp gruppieren?
3. **Echtzeit-Feedback und asynchrone Verarbeitung**: Git-Operationen können lange dauern.
4. **Multi-Repository-Unterstützung**: Korrektes Routing zwischen Haupt- und Sub-Repositories.
5. **Fehlerbehandlung und Rollback**: Was passiert bei fehlgeschlagenen Commits?
6. **Konsistenz der Commit-Nachrichten**: An den bestehenden Stil des Projekts anpassen.

## Lösung

### Gesamtarchitektur

Wir verwenden eine Schichtenarchitektur:

#### 1. API-Schicht (Web-Schicht)

**GitController** bietet den Endpunkt `POST /api/git/auto-compose-commit`. Im Fire-and-Forget-Modus:
- Der Server antwortet sofort mit HTTP 202 Accepted
- Die eigentliche KI-Verarbeitung läuft asynchron im Hintergrund
- Nach Abschluss wird der Client über SignalR benachrichtigt

#### 2. Anwendungsdienst-Schicht

**GitAppService** übernimmt die Kerngeschäftslogik:
- Repository-Erkennung
- Sperrverwaltung
- Datei-Staging-Koordination
- Fehler-Rollback

#### 3. Verteilte Rechenschicht (Orleans Grains)

**AIGrain** als zentrale Ausführungseinheit implementiert `AutoComposeCommitAsync`:

```csharp
[Alias("AutoComposeCommitAsync")]
[ResponseTimeout("00:20:00")]

    string projectId,
    GitFileStatusDto[] unstagedFiles,
    string? projectPath = null);
```

20 Minuten Timeout für große Changesets.

### Kernimplementierung

#### Dateikontext-Aufbau

Die KI benötigt den Kontext jeder Datei. Wir verwenden `BuildFileChangesXml`:

```csharp
private static string BuildFileChangesXml(GitFileStatusDto[] stagedFiles)
{
    var sb = new StringBuilder();
    sb.AppendLine("<files>");
    foreach (var file in stagedFiles)
    {
        sb.AppendLine("  <file>");
        sb.AppendLine($"    <path>{System.Security.SecurityElement.Escape(file.Path)}</path>");
        sb.AppendLine($"    <status>{System.Security.SecurityElement.Escape(file.Status)}</status>");
        if (!string.IsNullOrEmpty(file.OldPath))
        {
            sb.AppendLine($"    <oldPath>{System.Security.SecurityElement.Escape(file.OldPath)}</oldPath>");
        }
        sb.AppendLine("  </file>");
    }
    sb.AppendLine("</files>");
    return sb.ToString();
}
```

#### KI-Berechtigungsverwaltung


#### SignalR-Echtzeitbenachrichtigung

Nach Abschluss wird über SignalR benachrichtigt:

```csharp
private async Task SendAutoComposeCommitNotificationAsync(
    string projectId, int totalCount, int successCount,
    int failureCount, bool success, string? error)
{
    var notification = new AutoComposeCommitCompletedDto
    {
        ProjectId = projectId,
        TotalCount = totalCount,
        SuccessCount = successCount,
        FailureCount = failureCount,
        Success = success,
        Error = error
    };
    await messageService.SendAutoComposeCommitCompletedAsync(notification);
}
```

### Prompt-Engineering

Der Prompt ist sorgfältig gestaltet mit:
- **Nicht-interaktiver Modus**: Für CI/CD-Umgebungen
- **Branch-Schutz**: Keine Branch-Wechsel erlaubt
- **Intelligenter Gruppierungsentscheidungsbaum**: Nach Konfigurations-, Dokumentations- und Feature-Dateien
- **Historische Formatkonsistenz**: Analyse der letzten 15 Commits zur Stil-Anpassung
- **Co-Authored-By**: Pflicht für jeden Commit

### Fehlerbehandlung

- **Eingabevalidierung**: Prüfung auf leere Dateilisten
- **Fehler-Rollback**: Already-staged Files werden bei Fehlern unstaged
- **Timeout**: 20 Minuten mit automatischer Freigabe der Sperre
- **Sperrmechanismus**: Verhindert konkurrierende Operationen

## Best Practices

### 1. Kluger Einsatzzeitpunkt
- Ideal: Tagesende mit vielen Änderungen
- Weniger geeignet: Einzelne Datei, sensible Inhalte

### 2. AI-Commits prüfen
- Gruppierung überprüfen
- Commit-Nachrichten validieren
- Bei Bedarf mit `git reset --soft HEAD~N` anpassen

### 3. Mit Projektstandards kombinieren
- commitlint für CI/CD-Validierung
- Conventional Commits-Konfiguration

## Fazit

AI Compose Commit repräsentiert den tiefen Einsatz von KI in Entwicklungswerkzeuge. Durch intelligente Analyse, automatische Gruppierung und normierte Commit-Nachrichten wird der Git-Workflow erheblich effizienter.

Wichtige Erkenntnisse:
1. **Benutzerfeedback ist entscheidend**: Fire-and-Forget statt synchroner Wartezeit
2. **Prompt-Qualität bestimmt Output-Qualität**
3. **Sicherheit hat Vorrang**: Strenge Constraints trotz direkter Git-Ausführung
4. **Inkrementelle Verbesserung**: Einfach starten, schrittweise erweitern
