## Vollstaendiger Leitfaden zum Parsen von Codex SDK Konsolennachrichten

> Dieser Artikel erklaert den Ereignisstrom-Mechanismus des Codex SDK, das Parsen von Nachrichtentypen und bewaehrte Praktiken in realen Projekten, um Entwicklern zu helfen, die Kernfaehigkeiten von AI-Ausfuehrungsdiensten schnell zu meistern.

## Hintergrund

Bei der Erstellung eines AI-Ausfuehrungsdienstes auf Basis des Codex SDK stellt sich unausweichlich eine praktische Frage: Wie sollen wir die gestreamten Ereignisnachrichten verarbeiten, die von Codex zurueckgegeben werden? Diese Nachrichten enthalten wichtige Informationen wie Ausfuehrungsstatus, Ausgabeinhalte und Fehlerdetails.

Als Teil des HagiCode-Projekts benoetigten wir einen zuverlaessigen Executor fuer AI-Code-Assistent-Szenarien. Genau deshalb haben wir beschlossen, den Ereignisstrom-Mechanismus des Codex SDK eingehend zu untersuchen. Denn nur wenn man versteht, wie die zugrundeliegenden Nachrichten funktionieren, kann man eine wirklich unternehmensreife AI-Ausfuehrungsplattform aufbauen.

Das Codex SDK ist ein Programmierunterstuetzungs-SDK von OpenAI. Es gibt Ausfuehrungsergebnisse ueber einen Ereignisstrom zurueck. Im Gegensatz zum traditionellen Anfrage-Antwort-Modell verwendet Codex gestreamte Ereignisse, sodass wir:

- Ausfuehrungsfortschritt in Echtzeit erhalten
- Fehler zeitnah behandeln koennen
- Detaillierte Token-Nutzungsstatistiken erhalten
- Lang laufende komplexe Aufgaben unterstuetzen koennen

Das Verstehen dieser Ereignistypen und deren korrektes Parsen ist entscheidend fuer die Implementierung eines voll funktionsfaehigen AI-Executors.

## Ueber HagiCode

Die in diesem Artikel geteilte Loesung stammt aus unserer praktischen Erfahrung im [HagiCode](https://hagicode.com)-Projekt. HagiCode ist ein Open-Source-AI-Code-Assistent-Projekt, das sich der Bereitstellung intelligenter Code-Unterstuetzung fuer Entwickler widmet.

## Ereignisstrom-Mechanismus

### Grundlegende Konzepte

Das Codex SDK verwendet die Methode `thread.runStreamed()`, um einen asynchronen Ereignisiterator zurueckzugeben:

```typescript
import { Codex } from '@openai/codex-sdk';

const client = new Codex({
  apiKey: process.env.CODEX_API_KEY,
  baseUrl: process.env.CODEX_BASE_URL,
});

const thread = client.startThread({
  workingDirectory: '/path/to/project',
  skipGitRepoCheck: false,
});

const { events } = await thread.runStreamed('your prompt here', {
  outputSchema: {
    type: 'object',
    properties: {
      output: { type: 'string' },
      status: { type: 'string', enum: ['ok', 'action_required'] },
    },
    required: ['output', 'status'],
  },
});

for await (const event of events) {
  // Ereignis verarbeiten
}
```

### Ereignistyp-Details

| Ereignistyp | Beschreibung | Schluesseldaten |
|---------|------|----------|
| `thread.started` | Thread erfolgreich gestartet | `thread_id` |
| `item.updated` | Nachrichteninhalt aktualisiert | `item.text` |
| `item.completed` | Nachricht abgeschlossen | `item.text` |
| `turn.completed` | Ausfuehrung abgeschlossen | `usage` |
| `turn.failed` | Ausfuehrung fehlgeschlagen | `error.message` |
| `error` | Fehlerereignis | `message` |

### Nachrichteninhaltsextraktion

```typescript
private handleThreadEvent(event: ThreadEvent, onMessage: (content: string) => void): void {
  if (event.type !== 'item.updated' && event.type !== 'item.completed') {
    return;
  }

  if (event.item.type !== 'agent_message') {
    return;
  }

  onMessage(event.item.text);
}
```

### Strukturierte Ausgabe parsen

```typescript
const DEFAULT_OUTPUT_SCHEMA = {
  type: 'object',
  properties: {
    output: { type: 'string' },
    status: { type: 'string', enum: ['ok', 'action_required'] },
  },
  required: ['output', 'status'],
  additionalProperties: false,
} as const;
```

```typescript
function toStructuredOutput(raw: string): StructuredOutput {
  try {
    const parsed = JSON.parse(raw) as Partial<StructuredOutput>;
    if (typeof parsed.output === 'string') {
      return {
        output: parsed.output,
        status: parsed.status === 'action_required' ? 'action_required' : 'ok',
      };
    }
  } catch {
    // JSON-Parsing fehlgeschlagen, Rueckfall zum Roh-Text
  }

  return {
    output: raw,
    status: 'ok',
  };
}
```

### Vollstaendiger Ereignisbehandlungsablauf

```typescript
private async runWithStreaming(
  thread: Thread,
  input: CodexStageExecutionInput
): Promise<{ output: string; usage: Usage | null }> {
  const abortController = new AbortController();
  const timeoutHandle = setTimeout(() => {
    abortController.abort();
  }, Math.max(1000, input.timeoutMs));

  let latestMessage = '';
  let usage: Usage | null = null;
  let emittedLength = 0;

  try {
    const { events } = await thread.runStreamed(input.prompt, {
      outputSchema: DEFAULT_OUTPUT_SCHEMA,
      signal: abortController.signal,
    });

    for await (const event of events) {
      this.handleThreadEvent(event, (nextContent) => {
        const delta = nextContent.slice(emittedLength);
        if (delta.length > 0) {
          emittedLength = nextContent.length;
          input.callbacks?.onChunk?.(delta);
        }
        latestMessage = nextContent;
      });

      if (event.type === 'thread.started') {
        this.threadId = event.thread_id;
      } else if (event.type === 'turn.completed') {
        usage = event.usage;
      } else if (event.type === 'turn.failed') {
        throw new CodexExecutorError('gateway_unavailable', event.error.message, true);
      } else if (event.type === 'error') {
        throw new CodexExecutorError('gateway_unavailable', event.message, true);
      }
    }
  } catch (error) {
    if (abortController.signal.aborted) {
      throw new CodexExecutorError(
        'upstream_timeout',
        `Codex stage timed out after ${input.timeoutMs}ms`,
        true
      );
    }
    throw error;
  } finally {
    clearTimeout(timeoutHandle);
  }

  const structured = toStructuredOutput(latestMessage);
  return { output: structured.output, usage };
}
```

### Fehlerbehandlungsstrategie

```typescript
function mapError(error: unknown): CodexExecutorError {
  if (error instanceof CodexExecutorError) {
    return error;
  }

  const message = error instanceof Error ? error.message : String(error);
  const normalized = message.toLowerCase();

  if (normalized.includes('401') ||
      normalized.includes('403') ||
      normalized.includes('api key') ||
      normalized.includes('auth')) {
    return new CodexExecutorError('auth_invalid', message, false);
  }

  if (normalized.includes('429') || normalized.includes('rate limit')) {
    return new CodexExecutorError('rate_limited', message, true);
  }

  if (normalized.includes('timeout') || normalized.includes('aborted')) {
    return new CodexExecutorError('upstream_timeout', message, true);
  }

  return new CodexExecutorError('gateway_unavailable', message, true);
}
```

### Fehlertyp-Definition

```typescript
export type CodexErrorCode =
  | 'auth_invalid'
  | 'upstream_timeout'
  | 'rate_limited'
  | 'gateway_unavailable';

export class CodexExecutorError extends Error {
  readonly code: CodexErrorCode;
  readonly retryable: boolean;

  constructor(code: CodexErrorCode, message: string, retryable: boolean) {
    super(message);
    this.name = 'CodexExecutorError';
    this.code = code;
    this.retryable = retryable;
  }
}
```

## Bewaehrte Praktiken

1. **Arbeitsverzeichnis**: Stellen Sie sicher, dass das Arbeitsverzeichnis ein gueltiges Git-Repository ist.
2. **Umgebungsvariablen**: Geben Sie nur erforderliche Umgebungsvariablen weiter.
3. **Timeout und Wiederholung**: Legen Sie angemessene Timeouts fest und implementieren Sie exponentielles Backoff.
4. **Fehlerbehandlung**: Unterscheiden Sie zwischen wiederholbaren und nicht wiederholbaren Fehlern.
5. **Streaming-Ausgabe**: Implementieren Sie inkrementelle Ausgaberueckrufe zur Verbesserung der Benutzererfahrung.
