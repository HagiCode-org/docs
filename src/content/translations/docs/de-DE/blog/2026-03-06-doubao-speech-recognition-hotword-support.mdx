## Leitfaden zur Implementierung der Hotword-Unterstuetzung fuer Doubao-Spracherkennung

> Dieser Artikel erklaert im Detail, wie die Hotword-Unterstuetzung fuer die Doubao-Spracherkennung im HagiCode-Projekt implementiert werden kann. Durch die Verwendung sowohl benutzerdefinierter Hotwords als auch Plattform-Hotword-Tabellen koennen Sie die Erkennungsgenauigkeit fuer domaenenspezifischen Wortschatz erheblich verbessern.

## Hintergrund

Die Spracherkennungstechnologie hat sich ueber viele Jahre hinweg entwickelt, doch ein Problem hat Entwickler stets beschaeftigt. Allgemeine Spracherkennungsmodelle koennen den alltaeglichen Sprachgebrauch abdecken, aber sie stoessen oft an ihre Grenzen, wenn es um Fachbegriffe, Produktnamen und Personennamen geht. Denken Sie daran: Ein sprachgesteuertes Assistenzsystem im medizinischen Bereich muss Begriffe wie ,,Hypertonie``, ,,Diabetes`` und ,,Koronare Herzkrankheit`` korrekt erkennen; ein juristisches System muss Termini wie ,,Klagegrund``, ,,Verteidigung`` und ,,Beweislast`` praezise erfassen. In solchen Szenarien gibt ein allgemeines Modell sein Bestes, aber das reicht oft nicht aus.

Im HagiCode-Projekt standen wir vor der gleichen Herausforderung. Als multifunktionaler AI-Code-Assistent muss HagiCode die Spracherkennung fuer eine breite Palette technischer Fachbegriffe bewaeltigen. Die Doubao-Spracherkennungs-API konnte in ihrer Standardkonfiguration jedoch unsere Genauigkeitsanforderungen fuer Fachbegriffe nicht vollstaendig erfuellen. Es liegt nicht daran, dass Doubao nicht gut genug ist -- vielmehr hat jede Domaene ihr eigenes Begriffssystem. Nach einiger Recherche und technischer Erkundung stellten wir fest, dass die Doubao-Spracherkennungs-API tatsaechlich eine Hotword-Unterstuetzung bietet. Mit einer einfachen Konfiguration kann die Erkennungsgenauigkeit fuer bestimmte Begriffe erheblich gesteigert werden.

Was wir in diesem Artikel vorstellen, ist die vollstaendige Loesung zur Implementierung der Hotword-Funktion fuer die Doubao-Spracherkennung im HagiCode-Projekt. Zwei Modi -- benutzerdefinierte Hotwords und Plattform-Hotword-Tabellen -- koennen sowohl einzeln als auch in Kombination verwendet werden. Mit dieser Loesung koennen Entwickler Hotwords flexibel an ihre Geschaeftsszenarien anpassen und das Spracherkennungssystem dazu bringen, jene fachspezifischen, seltenen, aber entscheidenden Begriffe zu erkennen.

## Kernimplementierung

### Zwei Hotword-Modi

Die Doubao-Spracherkennungs-API bietet zwei Konfigurationsmethoden fuer Hotwords, die jeweils einzigartige Anwendungsszenarien und Vorteile haben.

Der **benutzerdefinierte Hotword-Modus** ermoeglicht die direkte Uebergabe von Hotword-Texten ueber das Feld `corpus.context`. Diese Methode eignet sich hervorragend fuer Szenarien, in denen schnell eine geringe Anzahl von Hotwords konfiguriert werden muss. In der HagiCode-Implementierung parsen wir den mehrzeiligen Hotword-Text des Benutzers in eine Liste von Zeichenketten und formatieren ihn gemaess den Anforderungen der Doubao-API als `context_data`-Array.

Der **Plattform-Hotword-Tabellen-Modus** referenziert ueber das Feld `corpus.boosting_table_id` eine vorab konfigurierte Hotword-Tabelle auf der Doubao-Self-Learning-Plattform. Diese Methode eignet sich fuer Szenarien, in denen eine grosse Anzahl von Hotwords verwaltet werden muss. Fuer Projekte wie HagiCode bietet dieser Modus eine bessere Verwaltbarkeit.

Interessanterweise koennen beide Modi auch kombiniert werden. Die Doubao-API unterstuetzt die gleichzeitige Angabe von benutzerdefinierten Hotwords und einer Plattform-Hotword-Tabellen-ID in derselben Anfrage.

### Frontend-Typdefinition und Validierung

In der Frontend-Implementierung von HagiCode haben wir einen vollstaendigen Satz von Hotword-Konfigurationstypen und Validierungslogiken definiert:

```typescript
export interface HotwordConfig {
  contextText: string;           // Mehrzeiliger Hotword-Text
  boostingTableId: string;      // Doubao-Plattform-Hotword-Tabellen-ID
  combineMode: boolean;          // Ob im Kombinationsmodus verwendet
}
```

Die Validierungsfunktion prueft die Einschraenkungen der Doubao-API: Hoechstens 100 Zeilen, je 50 Zeichen, insgesamt 5000 Zeichen:

```typescript
export function validateContextText(contextText: string): HotwordValidationResult {
  if (!contextText || contextText.trim().length === 0) {
    return { isValid: true, errors: [] };
  }

  const lines = contextText.split('\n').filter(line => line.trim().length > 0);
  const errors: string[] = [];

  if (lines.length > 100) {
    errors.push(`Hotword-Zeilen duerfen 100 nicht ueberschreiten, aktuell ${lines.length} Zeilen`);
  }

  const totalChars = contextText.length;
  if (totalChars > 5000) {
    errors.push(`Gesamtzeichenanzahl der Hotwords darf 5000 nicht ueberschreiten, aktuell ${totalChars}`);
  }

  for (let i = 0; i < lines.length; i++) {
    if (lines[i].length > 50) {
      errors.push(`Zeile ${i + 1} ueberschreitet das Zeichenlimit von 50`);
    }
  }

  return { isValid: errors.length === 0, errors };
}

export function validateBoostingTableId(boostingTableId: string): HotwordValidationResult {
  if (!boostingTableId || boostingTableId.trim().length === 0) {
    return { isValid: true, errors: [] };
  }

  const errors: string[] = [];

  if (boostingTableId.length > 200) {
    errors.push(`boosting_table_id darf 200 Zeichen nicht ueberschreiten, aktuell ${boostingTableId.length}`);
  }

  if (!/^[a-zA-Z0-9_-]+$/.test(boostingTableId)) {
    errors.push('boosting_table_id darf nur Buchstaben, Ziffern, Unterstriche und Bindestriche enthalten');
  }

  return { isValid: errors.length === 0, errors };
}
```

### Frontend-Konfigurationspersistenz

Wir verwenden den localStorage des Browsers zur Speicherung der Hotword-Konfiguration:

```typescript
const HOTWORD_STORAGE_KEYS = {
  contextText: 'hotword-context-text',
  boostingTableId: 'hotword-boosting-table-id',
  combineMode: 'hotword-combine-mode',
} as const;

export const DEFAULT_HOTWORD_CONFIG: HotwordConfig = {
  contextText: '',
  boostingTableId: '',
  combineMode: false,
};

export function loadHotwordConfig(): HotwordConfig {
  const contextText = localStorage.getItem(HOTWORD_STORAGE_KEYS.contextText) || '';
  const boostingTableId = localStorage.getItem(HOTWORD_STORAGE_KEYS.boostingTableId) || '';
  const combineMode = localStorage.getItem(HOTWORD_STORAGE_KEYS.combineMode) === 'true';
  return { contextText, boostingTableId, combineMode };
}

export function saveHotwordConfig(config: HotwordConfig): void {
  localStorage.setItem(HOTWORD_STORAGE_KEYS.contextText, config.contextText);
  localStorage.setItem(HOTWORD_STORAGE_KEYS.boostingTableId, config.boostingTableId);
  localStorage.setItem(HOTWORD_STORAGE_KEYS.combineMode, String(config.combineMode));
}
```

### Payload-Aufbaulogik

```csharp
private void AddCorpusToRequest(Dictionary<string, object> request)
{
    var corpus = new Dictionary<string, object>();

    if (Config.HotwordContexts != null && Config.HotwordContexts.Count > 0)
    {
        corpus["context"] = new Dictionary<string, object>
        {
            ["context_type"] = "dialog_ctx",
            ["context_data"] = Config.HotwordContexts
                .Select(text => new Dictionary<string, object> { ["text"] = text })
                .ToList()
        };
    }

    if (!string.IsNullOrEmpty(Config.BoostingTableId))
    {
        corpus["boosting_table_id"] = Config.BoostingTableId;
    }

    if (corpus.Count > 0)
    {
        request["corpus"] = corpus;
    }
}
```

```csharp
private async Task HandleControlMessageAsync(
    string connectionId,
    DoubaoSession session,
    ControlMessage message)
{
    if (message.Payload is SessionControlRequest controlRequest)
    {
        string? contextText = controlRequest.ContextText;
        string? boostingTableId = controlRequest.BoostingTableId;

        if (!string.IsNullOrEmpty(contextText))
        {
            var hotwords = contextText
                .Split('\n', StringSplitOptions.RemoveEmptyEntries)
                .Select(s => s.Trim())
                .Where(s => s.Length > 0)
                .ToList();
            session.HotwordContexts = hotwords;
        }

        session.BoostingTableId = boostingTableId;
    }
}
```

## Praktischer Leitfaden

### Benutzerdefinierte Hotwords konfigurieren

Oeffnen Sie die Einstellungsseite fuer die Spracherkennung in HagiCode und geben Sie pro Zeile einen Hotword-Begriff ein:

```
高血压
糖尿病
冠心病
```

### Plattform-Hotword-Tabelle konfigurieren

Erstellen Sie auf der Doubao-Self-Learning-Plattform eine Hotword-Tabelle und geben Sie die generierte ID in den HagiCode-Einstellungen ein.

Aktivieren Sie beide Hotword-Arten und schalten Sie den Kombinationsmodus ein.

## Hinweise

- Zeicheneinschraenkungen der Doubao-API muessen beachtet werden
- boosting_table_id erlaubt nur Buchstaben, Ziffern, Unterstriche und Bindestriche
- Hotword-Parameter sind vollstaendig optional (Rueckwaertskompatibilitaet)
- Fehlerbehandlung ist sowohl im Frontend als auch im Backend implementiert

## Zusammenfassung

Die Doubao-API unterstuetzt zwei Modi -- benutzerdefinierte Hotwords und Plattform-Hotword-Tabellen. Das Frontend verwendet localStorage, das Backend baut das corpus-Feld dynamisch auf und wahrt dabei die Rueckwaertskompatibilitaet.
