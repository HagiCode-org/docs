Se você quer instalar o Hermes e começar a usá-lo, o caminho mais curto é realmente só três passos:

1. Execute o comando oficial de instalação
2. Inicie o CLI no seu terminal com `hermes`
3. Se você quiser continuar usando no Feishu, configure `hermes gateway setup`

Este artigo não tenta explicar todas as capacidades do Hermes de uma vez. Em vez disso, ajuda você a completar o ciclo mais importante para iniciantes primeiro: instalar, colocar em execução, começar a usar e, em seguida, conectar a um dos cenários de plataforma de mensagens mais comuns.

## O que é o Hermes e para quem é

O Hermes Agent é um agente de IA que você pode usar tanto a partir de um terminal local quanto através de um gateway de plataforma de mensagens.

Para a maioria dos desenvolvedores, ele tem dois pontos de entrada comuns:

- **CLI**: Digite `hermes` no seu terminal para entrar diretamente na interface interativa.
- **Messaging Gateway**: Execute `hermes gateway` e depois converse com ele a partir de plataformas como Feishu, Telegram, Discord e Slack.

Se o seu objetivo agora é simplesmente começar rapidamente, não inverta a ordem. Comece por este caminho:

- Instale o Hermes primeiro
- Verifique se funciona pelo CLI primeiro
- Depois decida se quer conectar uma plataforma de mensagens

Isso facilita a diagnose de problemas e é mais adequado para pessoas que estão usando o Hermes pela primeira vez.

## O que saber antes de instalar o Hermes

De acordo com o README do Hermes, o caminho oficial de instalação rápida suporta estes ambientes:

- Linux
- macOS
- WSL2
- Android via Termux

### Observações para usuários de Windows

O Hermes atualmente não suporta execução direta no Windows nativo. Se você usa Windows, a abordagem recomendada é instalar o WSL2 primeiro e depois executar o comando de instalação dentro do WSL2.

É melhor deixar isso claro logo no início, porque muitas falhas de instalação não são causadas pelo comando em si, mas pelo uso de um ambiente de execução não suportado.

## Como instalar o Hermes

O comando de instalação rápida fornecido no README do Hermes é:

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

Este comando executa o script de instalação oficial e trata as etapas de inicialização específicas da plataforma.

### Recarregue seu shell após a instalação

Assim que a instalação terminar, recarregue o ambiente do seu shell primeiro. O comando mais comum é:

```bash
source ~/.bashrc
```

Se você usa zsh, pode usar:

```bash
source ~/.zshrc
```

### Como confirmar que o Hermes está instalado corretamente

A maneira mais direta de verificar é executar:

```bash
hermes
```

Se você quer uma confirmação adicional de que sua configuração e dependências estão funcionando, também pode executar:

```bash
hermes doctor
```

`hermes doctor` é especialmente útil nestas situações:

- O comando se comporta de forma anormal após a instalação
- A configuração do modelo falha
- O gateway falha ao iniciar
- Você não tem certeza se suas dependências de ambiente estão completas

## Como começar a usar o Hermes pela primeira vez

### Comece pelo CLI

Se você só quer confirmar o mais rápido possível que o Hermes funciona, o método mais simples é:

```bash
hermes
```

Isso inicia o CLI interativo do Hermes. Para usuários iniciantes do Hermes, também é o ponto de partida mais recomendado, porque você pode verificar as coisas mais essenciais primeiro:

- Se o comando está realmente disponível
- Se a configuração atual do modelo funciona corretamente
- Se a cadeia de ferramentas do terminal está funcionando corretamente
- Se o estilo de interação corresponde ao que você precisa

### Estes comandos são suficientes para sua primeira rodada de configuração

O README do Hermes lista vários comandos de alta frequência e, juntos, eles formam um caminho prático de primeiro uso:

```bash
hermes model
hermes tools
hermes config set
hermes setup
hermes update
hermes doctor
```

Se você não tem certeza do que cada um faz, lembre-se deles assim:

- `hermes model`: escolher ou trocar modelos
- `hermes tools`: ver e configurar as ferramentas disponíveis
- `hermes config set`: alterar itens de configuração específicos
- `hermes setup`: executar o assistente de inicialização completo uma vez
- `hermes update`: atualizar o Hermes
- `hermes doctor`: solucionar problemas

Para iniciantes, a ordem mais prática geralmente é:

1. Execute `hermes model` primeiro
2. Se você quer configurar todas as opções comuns de uma vez, execute `hermes setup`

## As duas formas mais comuns de usar o Hermes

### 1. Usar o Hermes no terminal como assistente diário de desenvolvimento

O modo CLI é adequado para estes cenários:

- Fazer perguntas diretamente enquanto escreve código localmente
- Inspecionar projetos, editar arquivos e executar comandos
- Fazer depuração ou revisão pontual
- Colaborar continuamente no diretório de trabalho atual

Sua maior vantagem é que é o caminho mais curto: sem integração extra de plataforma, sem configuração de bot para lidar antecipadamente, e é a melhor forma de construir seus primeiros hábitos de uso.

### 2. Usar o Hermes através de uma plataforma de mensagens

Se você quer conversar com o Hermes em plataformas como Feishu, Telegram ou Discord, precisa usar o messaging gateway.

Os comandos de entrada mais comuns são:

```bash
hermes gateway setup
hermes gateway
```

Especificamente:

- `hermes gateway setup` é usado para configuração interativa de plataforma
- `hermes gateway` é usado para iniciar o processo do gateway

De acordo com a documentação oficial, o gateway é um processo unificado em segundo plano que conecta suas plataformas configuradas, gerencia sessões e lida com recursos como tarefas agendadas.

## Usando Feishu como exemplo: como conectar o Hermes a uma plataforma de mensagens

Se a maior parte do seu trabalho diário acontece no Feishu, então Feishu/Lark é uma forma muito natural de usar o Hermes.

### O caminho mínimo de integração viável

A documentação oficial recomenda este comando de entrada para Feishu/Lark:

```bash
hermes gateway setup
```

Depois de executar, basta escolher **Feishu / Lark** no assistente.

A documentação do Feishu descreve dois modos de conexão:

- `websocket`: recomendado
- `webhook`: opcional

Se o Hermes roda no seu laptop, estação de trabalho ou servidor privado, usar `websocket` primeiro geralmente é mais simples porque você não precisa expor uma URL pública de callback.

### Se você configurar manualmente, conheça pelo menos estas variáveis

Se você não está usando o assistente e está escrevendo a configuração manualmente, a documentação do Feishu lista estas variáveis principais:

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=***
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
```

Duas delas merecem atenção especial:

- `FEISHU_ALLOWED_USERS`: recomendado, para que nem todos que acessarem o bot possam usá-lo diretamente
- `FEISHU_HOME_CHANNEL`: permite predefinir um chat principal para receber resultados de tarefas agendadas ou notificações padrão

### Por que o Hermes às vezes não responde em chats em grupo do Feishu

Este detalhe é fácil de esquecer: em chats em grupo do Feishu, o Hermes não responde a todas as mensagens por padrão.

A documentação oficial diz claramente:

- Em mensagens diretas, o Hermes responde às mensagens
- Em chats em grupo, você deve fazer `@` no bot explicitamente antes que ele processe a mensagem

Se você quiser definir uma conversa do Feishu como canal principal, também pode usar isto no chat:

```text
/set-home
```

Ou definir na configuração antecipadamente:

```bash
FEISHU_HOME_CHANNEL=oc_xxx
```

## Os comandos do Hermes que iniciantes devem lembrar primeiro

Seja no CLI ou em uma plataforma de mensagens, lembrar dos seguintes comandos já é suficiente para começar:

- `/new` ou `/reset`: iniciar uma nova sessão
- `/model`: ver ou trocar o modelo
- `/retry`: tentar novamente a rodada anterior
- `/undo`: desfazer a interação anterior
- `/compress`: comprimir o contexto manualmente
- `/help`: ver a ajuda

Se você usa o Hermes principalmente em uma plataforma de mensagens, lembre-se de mais um:

- `/sethome` ou `/set-home`: definir o chat atual como canal principal

Estes comandos cobrem as operações mais comuns da fase de iniciante: reiniciar, ajustar, reverter, verificar e continuar.

## Perguntas frequentes

### O Windows pode instalar o Hermes diretamente?

Não. A documentação oficial atual diz claramente que o Windows nativo não é suportado, e o WSL2 é recomendado.

### O que fazer se digitar `hermes` não faz nada após a instalação?

É melhor solucionar nesta ordem:

1. Recarregue seu shell primeiro, por exemplo com `source ~/.bashrc`
2. Execute `hermes` novamente
3. Se ainda estiver anormal, execute `hermes doctor`

### Por que o bot não responde em um grupo do Feishu?

Verifique estas três coisas primeiro:

- Se você fez `@` no Hermes no grupo
- Se `FEISHU_ALLOWED_USERS` restringe o usuário atual
- Se a política atual do chat em grupo permite processar mensagens do grupo

De acordo com a documentação oficial do Feishu, usar uma menção com @ explicitamente é obrigatório em cenários de chat em grupo.
