Este guia cobre a instalação do Node.js no Windows, macOS e Linux. Node.js é uma dependência chave para ferramentas locais HagiCode, incluindo OpenSpec, ferramentas de frontend e vários CLIs de desenvolvedor.

:::dica[Requisitos de versão]
HagiCode recomenda **Node.js 24+** (Node.js mínimo suportado >= 18.0). Na maioria dos casos, instale a versão LTS mais recente e verifique se sua versão atende aos requisitos.
:::

## Instalando Node.js

<Tabs>
  <TabItem value="windows" label="Windows" default>

### Método um: instalador oficial (recomendado)

1. Visite o [Site oficial do Node.js.](https://nodejs.org/)
2. Baixe a versão **LTS (Long Term Support)** do instalador do Windows (`.msi` arquivo)
3. Clique duas vezes para executar o instalador
4. Siga as instruções do assistente de instalação para concluir a instalação:
   - Aceite o contrato de licença
   - Escolha o caminho de instalação (o padrão é bom)
   - Certifique-se de que a opção "Instalar automaticamente as ferramentas necessárias" esteja marcada
   - Clique em "Instalar" para iniciar a instalação
5. Após a conclusão da instalação, clique em "Concluir"

### Método dois: usando winget

Abra o PowerShell ou prompt de comando e execute:

```powershell
winget install OpenJS.NodeJS.LTS
```

Após a instalação, reabra o terminal para que as alterações tenham efeito.

  </TabItem>
  <TabItem value="macos" label="macOS">

### Método um: instalador oficial (recomendado)

1. Visite o [Site oficial do Node.js.](https://nodejs.org/)
2. Baixe o instalador do macOS da versão **LTS (Long Term Support)** (`.pkg` arquivo)
3. Clique duas vezes para executar o instalador
4. Siga as instruções do assistente de instalação para concluir a instalação:
   - Clique em "Continuar"
   - Aceite o contrato de licença
   - Escolha o disco de instalação
   - Clique em "Instalar" e digite a senha do administrador
5. Após a conclusão da instalação, clique em "Fechar"

### Método dois: usando Homebrew

Se você tiver [Cerveja caseira](https://brew.sh/) instalado, você pode instalar o Node.js via terminal:

```bash
brew install node
```

:::dica
O Homebrew adicionará automaticamente o Node.js ao seu PATH. Se o comando não estiver disponível, reinicie o terminal ou execute:
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

  </TabItem>
  <TabItem value="ubuntu" label="Ubuntu/Debian">

### Usando o repositório NodeSource (recomendado)

Esta é a melhor maneira de obter a versão mais recente do Node.js:

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Usando o Gerenciador de Pacotes apt

Se não precisar da versão mais recente, você pode usar o repositório padrão do Ubuntu:

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::nota
A versão do Node.js no repositório padrão pode ser mais antiga. Após a instalação, verifique a versão para garantir que ela atenda aos requisitos do Node.js >= 18.0 (Node.js 24+ é recomendado).
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### Usando o repositório NodeSource (recomendado)

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### Usando o Gerenciador de Pacotes yum

```bash
sudo yum install -y nodejs npm
```

  </TabItem>
  <TabItem value="fedora" label="Fedora">

### Usando o Gerenciador de Pacotes dnf

```bash
sudo dnf install -y nodejs npm
```

:::dica
O repositório de software do Fedora geralmente contém versões mais recentes do Node.js e pode ser instalado diretamente via dnf.
:::

  </TabItem>
</Tabs>

## Usando ferramentas de gerenciamento de versão

Se precisar alternar entre várias versões do Node.js, você pode usar ferramentas de gerenciamento de versão.

### nvm (gerenciador de versão do nó)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

Após a instalação, recarregue a configuração do shell:

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Janelas

No Windows, recomendamos usar [janelas nvm](https://github.com/coreybutler/nvm-windows):

1. Baixe o mais recente `nvm-setup.exe` de [lançamentos nvm-windows](https://github.com/coreybutler/nvm-windows/releases)
2. Execute o instalador
3. Reabra o prompt de comando ou PowerShell

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

Depois de instalar o nvm, você pode usá-lo para instalar e gerenciar versões do Node.js:

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (gerenciador de nó rápido)

fnm é uma ferramenta de gerenciamento de versão Node.js mais rápida e simples construída com Rust.

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### Janelas

Usar [colher](https://scoop.sh/) ou [asa](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## Aceleração para usuários da China Continental

Se você estiver na China continental, o download dos pacotes Node.js e npm pode ser lento. Aqui estão algumas soluções de aceleração.

### Usando fontes de espelho doméstico

#### npm Taobao Mirror (recomendado)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### Usando nrm para gerenciar fontes de espelho

nrm (gerenciador de registro npm) ajuda você a alternar rapidamente as fontes de espelho npm:

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Acelerando downloads do instalador Node.js

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao fornece downloads espelhados para instaladores Node.js:

- **Endereço de download**: https://npmmirror.com/mirrors/node/

Visite o endereço acima e selecione a versão que deseja baixar.

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

Se o download do nvm ou de outras ferramentas do GitHub for lento, você pode usar o espelho FastGit:

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::nota
O serviço FastGit pode mudar, consulte as informações oficiais.
:::

  </TabItem>
</Tabs>

### Espelhos caseiros domésticos

Se você usa o Homebrew na China continental, você pode configurar espelhos domésticos para aceleração:

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### Gerenciadores de pacotes específicos da China continental

#### cnpm

cnpm é um cliente npm fornecido pela equipe Taobao que usa o espelho Taobao por padrão:

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::dica
cnpm é principalmente compatível com comandos npm e pode ser usado como um substituto direto.
:::

### Verificando o efeito de aceleração do espelho

Depois de configurar os espelhos, verifique:

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## Verificando a instalação

Após a instalação, verifique se o Node.js e o npm estão instalados corretamente:

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::nota
Certifique-se de que a versão do Node.js seja >= 18.0 (Node.js 24+ é recomendado). Se a versão for muito baixa, reinstale usando os métodos acima ou use ferramentas de gerenciamento de versão para instalar a versão correta.
:::

### Exemplo de saída de verificação

O resultado da verificação bem-sucedida deve ser semelhante a:

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## Solução de problemas

### Problemas do Windows

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

Se `node` ou `npm` comandos não estão disponíveis após a instalação:

1. Reinicie o prompt de comando ou PowerShell
2. Se ainda não estiver disponível, adicione manualmente o Node.js ao PATH:
   - Procure por "Variáveis de ambiente" e abra "Editar variáveis de ambiente do sistema"
   - Clique em “Variáveis de Ambiente”
   - Encontre "Caminho" em "Variáveis do sistema" e clique em "Editar"
   - Adicione o caminho de instalação do Node.js (padrão: `C:\Program Files\nodejs\`)
   - Clique em "OK" para salvar as alterações

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

Se você encontrar erros de política de execução ao usar o nvm:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### Problemas do macOS

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

Se você encontrar erros de permissão ao instalar pacotes npm globais:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

Se os comandos não estiverem disponíveis após a instalação, reinicie o terminal ou adicione manualmente ao PATH (geralmente em `/usr/local/bin`).

  </TabItem>
</Tabs>

### Problemas do Linux

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

Se você encontrar erros de permissão ao instalar pacotes npm globais, é recomendado usar ferramentas de gerenciamento de versão Node.js (como nvm ou fnm) em vez de gerenciadores de pacotes do sistema.

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

Se a versão do Node.js instalada por meio do gerenciador de pacotes do sistema for muito antiga:

1. Use o repositório NodeSource (recomendado, veja acima)
2. Ou use ferramentas de gerenciamento de versão (nvm/fnm)

  </TabItem>
</Tabs>

### Verificando o ambiente

Se a verificação falhar, verifique o seguinte:

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
