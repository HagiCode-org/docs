## LLM ガイドが必要な理由

コード生成ワークフローでは、モデルのパフォーマンスは品質、レイテンシー、コストによって大きく異なります。このカテゴリは、開発者がモデルをより迅速に選択できるように、一貫した評価構造を提供します。

## 推奨される読み取りパス

1. から始める [ＨａｇｉＣｏｄｅにおけるモデルの比較と評価](./model-comparison-evaluation/).
2. 優先順位 (品質第一、コスト第一、スピード第一) に基づいて候補者を選択します。
3. 結論を運用環境に適用する前に、テスト時のメモを確認してください。

## メンテナンスの原則

- モデルの機能が大幅に変化した場合は、結論を更新します。
- 少なくとも月に 1 回、テスト時のコンテキストを再検証します。
- 中国語と英語のページをセクション構造で揃えてください。
