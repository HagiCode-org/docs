このガイドでは、Windows、macOS、および Linux での Node.js のインストールについて説明します。 Node.js は、OpenSpec、フロントエンド ツール、およびいくつかの開発者 CLI を含む、HagiCode ローカル ツールの重要な依存関係です。

:::tip[必要なバージョン]
HagiCode は **Node.js 24+** (サポートされる最小 Node.js >= 18.0) を推奨します。ほとんどの場合、最新の LTS バージョンをインストールして、バージョンが要件を満たしていることを確認します。
:::

## Node.jsのインストール

<Tabs>
  <TabItem value="windows" label="Windows" default>

### 方法 1: 公式インストーラー (推奨)

1. にアクセスしてください。 [Node.js 公式ウェブサイト](https://nodejs.org/)
2. **LTS (長期サポート)** バージョンの Windows インストーラーをダウンロードします (`.msi` ファイル)
3. ダブルクリックしてインストーラーを実行します
4. インストール ウィザードのプロンプトに従ってインストールを完了します。
   - ライセンス契約に同意する
   - インストールパスを選択します（デフォルトで問題ありません）
   - 「必要なツールを自動的にインストールする」にチェックが入っていることを確認します
   - 「インストール」をクリックしてインストールを開始します
5. インストールが完了したら、「完了」をクリックします

### 方法 2: winget を使用する

PowerShell またはコマンド プロンプトを開いて次を実行します。

```powershell
winget install OpenJS.NodeJS.LTS
```

インストール後、変更を有効にするためにターミナルを再度開きます。

  </TabItem>
  <TabItem value="macos" label="macOS">

### 方法 1: 公式インストーラー (推奨)

1. にアクセスしてください。 [Node.js 公式ウェブサイト](https://nodejs.org/)
2. **LTS (長期サポート)** バージョンの macOS インストーラーをダウンロードします (`.pkg` ファイル)
3. ダブルクリックしてインストーラーを実行します
4. インストール ウィザードのプロンプトに従ってインストールを完了します。
   - 「続行」をクリックします
   - ライセンス契約に同意する
   - インストールディスクの選択
   - 「インストール」をクリックし、管理者パスワードを入力します
5. インストールが完了したら「閉じる」をクリックします

### 方法 2: Homebrew を使用する

持っている場合 [自作](https://brew.sh/) インストールされている場合は、ターミナル経由で Node.js をインストールできます。

```bash
brew install node
```

:::ヒント
Homebrew は自動的に Node.js を PATH に追加します。コマンドが使用できない場合は、ターミナルを再起動するか、次のコマンドを実行します。
```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```
:::

### NodeSource リポジトリの使用 (推奨)

これは、最新の Node.js バージョンを入手する最良の方法です。

```bash
# Use curl to download the installation script
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### apt パッケージ マネージャーの使用

最新バージョンが必要ない場合は、デフォルトの Ubuntu リポジトリを使用できます。

```bash
sudo apt-get update
sudo apt-get install -y nodejs npm
```

:::メモ
デフォルト リポジトリの Node.js バージョンは古い可能性があります。インストール後、バージョンをチェックして、Node.js >= 18.0 の要件を満たしていることを確認します (Node.js 24 以降を推奨します)。
:::

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

### NodeSource リポジトリの使用 (推奨)

```bash
# Use curl to download the installation script
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs
```

### yumパッケージマネージャーの使用

```bash
sudo yum install -y nodejs npm
```

  </TabItem>
  <TabItem value="fedora" label="Fedora">

### dnf パッケージ マネージャーの使用

```bash
sudo dnf install -y nodejs npm
```

:::ヒント
Fedora のソフトウェア リポジトリには通常、新しいバージョンの Node.js が含まれており、dnf 経由で直接インストールできます。

## バージョン管理ツールの使用

複数の Node.js バージョン間で切り替える必要がある場合は、バージョン管理ツールを使用できます。

### nvm (ノード バージョン マネージャー)

<Tabs>
  <TabItem value="nvm-install" label="Installing nvm" default>

#### Linux/macOS

```bash
# Install using curl
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Or install using wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
```

インストール後、シェル構成をリロードします。

```bash
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### 窓

Windows では、次の使用をお勧めします。 [nvm-windows](https://github.com/coreybutler/nvm-windows):

1. 最新のものをダウンロード `nvm-setup.exe` から [nvm-windows リリース](https://github.com/coreybutler/nvm-windows/releases)
2. インストーラーを実行する
3. コマンドプロンプトまたはPowerShellを再度開きます

  </TabItem>
  <TabItem value="nvm-use" label="Using nvm">

nvm をインストールした後、それを使用して Node.js バージョンのインストールと管理を行うことができます。

```bash
# Install the latest LTS version (recommended)
nvm install --lts

# Install Node.js 24 (recommended version)
nvm install 24

# Install a specific Node.js version
nvm install 18.20.0

# List installed versions
nvm list

# Switch to a specified Node.js version
nvm use 24

# Set default version
nvm alias default 24

# Windows nvm-windows syntax is slightly different
nvm install 24
nvm use 24
```

  </TabItem>
</Tabs>

### fnm (高速ノードマネージャー)

fnm は、Rust で構築された、より高速かつシンプルな Node.js バージョン管理ツールです。

<Tabs>
  <TabItem value="fnm-install" label="Installing fnm" default>

#### Linux/macOS

```bash
# Install using curl
curl -fsSL https://fnm.vercel.app/install | bash

# After installation, reload shell configuration
source ~/.bashrc
# Or for Zsh users
source ~/.zshrc
```

#### 窓

使用する [スコップ](https://scoop.sh/) または [ウィンゲット](https://learn.microsoft.com/en-us/windows/package-manager/winget/):

```powershell
# Use scoop
scoop install fnm

# Or use winget
winget install Schniz.fnm
```

  </TabItem>
  <TabItem value="fnm-use" label="Using fnm">

```bash
# Install the latest LTS version (recommended)
fnm install --lts

# Install Node.js 24 (recommended version)
fnm install 24

# Install a specific Node.js version
fnm install 18.20.0

# List installed versions
fnm list

# Switch to a specified Node.js version
fnm use 24

# Set default version
fnm default 24
```

  </TabItem>
</Tabs>

## 中国本土ユーザー向けの高速化

中国本土にお住まいの場合、Node.js および npm パッケージのダウンロードが遅くなる可能性があります。ここでは加速ソリューションをいくつか紹介します。

### 国内ミラーソースの使用

#### npm タオバオ ミラー (推奨)

```bash
# Temporarily use Taobao mirror
npm install --registry=https://registry.npmmirror.com

# Permanently set Taobao mirror
npm config set registry https://registry.npmmirror.com

# Verify mirror is set successfully
npm config get registry
```

#### nrm を使用したミラー ソースの管理

nrm (npm レジストリ マネージャー) は、npm ミラー ソースをすばやく切り替えるのに役立ちます。

```bash
# Install nrm
npm install -g nrm

# List available mirror sources
nrm ls

# Switch to Taobao mirror
nrm use taobao

# Test mirror source speed
nrm test
```

### Node.js インストーラーのダウンロードを高速化する

<Tabs>
  <TabItem value="cnpm-mirror" label="Taobao Node.js Mirror" default>

Taobao では、Node.js インストーラーのミラー ダウンロードを提供しています。

- **ダウンロード アドレス**: https://npmmirror.com/mirrors/node/

上記のアドレスにアクセスし、ダウンロードする必要があるバージョンを選択してください。

  </TabItem>
  <TabItem value="fastgit" label="FastGit Acceleration">

GitHub からの nvm またはその他のツールのダウンロードが遅い場合は、FastGit ミラーを使用できます。

```bash
# Use FastGit to accelerate nvm installation
curl -o- https://fastgit.org/nvm-sh/nvm/raw/master/install.sh | bash
```

:::メモ
FastGit のサービスは変更される場合がありますので、公式情報をご確認ください。
:::

  </TabItem>
</Tabs>

### 国産自作ミラー

中国本土で Homebrew を使用する場合、加速のために国内ミラーを構成できます。

```bash
# Install Homebrew using domestic mirror
/bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

# Or use USTC mirror
export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles
```

### 中国本土固有のパッケージマネージャー

#### CNPM

cnpm は、淘宝網チームによって提供される npm クライアントで、デフォルトで淘宝網ミラーを使用します。

```bash
# Install cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com

# Use cnpm to install packages
cnpm install <package-name>
```

:::ヒント
cnpm は npm コマンドとほぼ互換性があり、直接の置き換えとして使用できます。
:::

### ミラー加速効果の検証

ミラーを設定した後、次の方法で確認します。

```bash
# Test download speed
time npm install express --registry=https://registry.npmmirror.com

# Compare with official source speed
time npm install express --registry=https://registry.npmjs.org
```

## インストールの検証

インストール後、Node.js と npm が正しくインストールされていることを確認します。

```bash
# Check Node.js version
node --version

# Check npm version
npm --version
```

:::メモ
Node.js バージョンが 18.0 以上であることを確認してください (Node.js 24 以降を推奨)。バージョンが低すぎる場合は、上記の方法を使用して再インストールするか、バージョン管理ツールを使用して正しいバージョンをインストールしてください。
:::

### サンプル検証出力

検証が成功すると、出力は次のようになります。

```
$ node --version
v24.3.0

$ npm --version
10.2.3
```

## トラブルシューティング

### Windowsの問題

<Tabs>
  <TabItem value="win-path" label="Command Not Available">

もし `node` または `npm` インストール後はコマンドが使用できなくなります。

1. コマンドプロンプトまたはPowerShellを再起動します
2. それでも利用できない場合は、Node.js を PATH に手動で追加します。
   - 「環境変数」を検索し、「システム環境変数の編集」を開きます。
   - 「環境変数」をクリックします。
   - 「システム環境変数」で「パス」を見つけて「編集」をクリックします。
   - Node.js インストール パスを追加します (デフォルト: `C:\Program Files\nodejs\`)
   - 「OK」をクリックして変更を保存します

  </TabItem>
  <TabItem value="win-policy" label="PowerShell Execution Policy">

nvm の使用時に実行ポリシー エラーが発生した場合:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

  </TabItem>
</Tabs>

### macOS の問題

<Tabs>
  <TabItem value="mac-perm" label="Permission Errors">

グローバル npm パッケージのインストール時に権限エラーが発生した場合:

```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

  </TabItem>
  <TabItem value="mac-path" label="Command Not Available">

インストール後にコマンドが使用できない場合は、ターミナルを再起動するか、手動で PATH (通常は `/usr/local/bin`).

  </TabItem>
</Tabs>

### Linuxの問題

<Tabs>
  <TabItem value="linux-perm" label="Permission Errors">

グローバル npm パッケージをインストールするときに権限エラーが発生した場合は、システム パッケージ マネージャーの代わりに Node.js バージョン管理ツール (nvm や fnm など) を使用することをお勧めします。

  </TabItem>
  <TabItem value="linux-old" label="Version Too Old">

システム パッケージ マネージャー経由でインストールされた Node.js のバージョンが古すぎる場合:

1. NodeSource リポジトリを使用する (推奨、上記を参照)
2. または、バージョン管理ツール (nvm/fnm) を使用します。

  </TabItem>
</Tabs>

### 環境の確認

検証が失敗した場合は、次のことを確認してください。

<Tabs>
  <TabItem value="win-check" label="Windows">

```powershell
# Check if Node.js is in PATH
where node

# Check installation path
Get-Command node
```

  </TabItem>
  <TabItem value="mac-check" label="macOS">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)
```

  </TabItem>
  <TabItem value="linux-check" label="Linux">

```bash
# Check if Node.js is in PATH
which node

# Check installation path
ls -l $(which node)

# Check version details
node -v
```

  </TabItem>
</Tabs>
