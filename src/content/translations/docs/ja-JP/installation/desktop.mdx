このガイドでは、HagiCode Desktop のインストールと使用について説明します。これは公式のデスクトップ クライアントであり、デフォルトのローカル セットアップ パスと日常のコントロールを 1 か所に保持します。

:::tip【おすすめの方法】
デスクトップは、次の理由により、個々のユーザーおよび開発者にとってのデフォルトのローカル パスです。
- インストーラーはデフォルトのフローを 1 つのガイド付きパスに維持します
- 依存関係とサービスはアプリ内で管理されます
- UI により状態と操作を簡単に検査できるようになります
- 組み込みのバージョン管理により切り替えが簡単になります
- データはデフォルトでローカルに留まります
:::

## ハギコードデスクトップとは何ですか?

HagiCode Desktop は、Windows、macOS、および Linux システム用に設計された、HagiCode の公式デスクトップ アプリケーションです。これには、HagiCode のコア機能がバンドルされており、インストーラーは 1 つのガイド付きフロー内でデフォルトのセットアップ パスを保持します。

### 主な利点

- **ワンクリック インストール**: インストール ウィザードを通じてすべての構成手順を自動的に完了します。
- **ランタイムの自動準備**: デフォルトのインストーラーには必要なランタイムがすでに含まれているため、別のランタイムをインストールする必要はありません。 `.NET` ランタイムが最初
- **サービス管理**: 組み込みのHagiCodeサーバー管理により、サービスを簡単に開始および停止できます。
- **バージョンの切り替え**: マルチバージョン管理をサポートし、異なるバージョン間を自由に切り替えることができます。
- **ローカル操作**: すべてのデータはローカルに保存され、プライバシーが保護されます。
- **自動アップデート**: 最新バージョンを自動的に検出してダウンロードします。

### 該当するシナリオ

デスクトップは次の場合に適しています。
- ローカルでHagiCodeを使用したい個人開発者
- オフラインでHagiCodeを使用する必要があるユーザー
- インストールプロセスを簡素化したい非技術ユーザー
- 異なるバージョンを頻繁に切り替える必要がある開発者

### 正しいバージョンを選択してください

オペレーティング システムに対応するインストーラーを選択します。
- **Windows**: ダウンロード `.exe` インストーラ
- **macOS**: ダウンロード `.dmg` インストーラ
- **Linux**: ダウンロード `.AppImage` または `.tar.gz` インストーラーパッケージ

:::note[動作環境]
- **Windows**: Windows 10 以降のバージョン (64 ビット)
- **macOS**: macOS 10.15 以降
- **Linux**: メインストリーム Linux ディストリビューション (Ubuntu 20.04 以降、Fedora 33 以降など)
:::

最新のHagiCodeのローカルインストールフローの場合、通常は別途準備する必要はありません。 `.NET` ランタイム。 Desktop には、デフォルト フローに必要なランタイムがすでにバンドルされており、サポートされているローカル SQLite パスがデフォルトで初期化されています。
:::

## インストールと起動に関する注意事項

:::注意[macOS ゲートキーパーの注意事項]
すでにアプリをデフォルトのパスにインストールしている場合 `/Applications/HagiCode.app` ただし、macOS では、初回起動時に「アプリが破損しています」、「開けません」、または同様のゲートキーパー警告が表示されます。通常の原因は `com.apple.quarantine` これは、macOS がインターネットからダウンロードしたアプリに追加する属性であり、HagiCode Desktop インストーラー自体の障害ではありません。

ターミナルで次のコマンドを実行します。これには管理者権限が必要で、デフォルトのインストール パスにあるアプリから隔離属性を再帰的に削除します。

```bash
sudo xattr -dr com.apple.quarantine /Applications/HagiCode.app
```

コマンドが終了したら、HagiCode Desktop を再起動します。
:::

## デスクトップを実行する

ダウンロード後、インストーラーまたはアプリケーション パッケージを開き、画面上の指示に従います。デスクトップは、現在サポートされているローカル パスでランタイムを準備し、HagiCode サーバーを管理し、ネットワーク ダウンロードが必要な場合は進行状況とステータスを表示します。

:::note[ネットワーク要件]
ネットワーク接続が利用可能であることを確認してください。デスクトップは、バージョン情報またはコンポーネント パッケージをインターネットから取得する必要がある場合があります。ダウンロードが失敗した場合は、ネットワーク、プロキシ、またはファイアウォールの設定を確認して、再試行してください。
:::

## バージョン管理

HagiCode Desktop はマルチバージョン管理をサポートしており、異なるバージョンを自由に切り替えることができます。

![バージョン管理](../../img/screenshots/installation/install-desktop/12-version-management/original.png)

### バージョンの切り替え

1. ハギコードデスクトップを開く
2. 「バージョン管理」ページに移動します
3. 切り替えたいバージョンを選択してください
4. 「バージョンを切り替える」ボタンをクリックします
5. 切り替えが完了するまで待ちます

:::note[バージョン切り替え注意事項]
- バージョンを切り替えると、HagiCode Server が自動的に再起動されます
- 切り替える前にすべての作業を必ず保存してください
- プロジェクト構成はバージョンごとに異なる場合があります
:::

## サービスの開始と停止

HagiCode Desktop は、HagiCode Server サービスを管理する簡単な方法を提供します。

### サービスを開始する

デスクトップアプリケーションのホームページで「サービスの開始」ボタンをクリックしてHagiCode Serverを起動します。

![サービスを開始する](../../img/screenshots/installation/install-desktop/13-start-service-homepage/original.png)

### サービスの停止

ハギコードサーバーを停止するには、ホームページの「サービスの停止」ボタンをクリックしてください。

:::tip[サービス状況]
デスクトップ アプリケーションには、HagiCode Server の実行ステータスがリアルタイムで表示されます。次のものが含まれます。
- サービスのステータス (実行中/停止中)
- サービスポート番号
- 現在のバージョン情報
:::

## トラブルシューティング

### インストールの失敗

インストール中に問題が発生した場合:

1. **ネットワーク接続を確認してください**: インターネットにアクセスできることを確認してください
2. **ディスク容量を確認してください**: 十分なディスク容量があることを確認してください
3. **管理者として実行**: Windows では、インストーラーを右クリックし、[管理者として実行] を選択します。
4. **ログ ファイルの表示**: インストーラーは、問題の診断に役立つ詳細なログ ファイルを生成します。

### 依存関係のインストールの失敗

依存関係のインストールが失敗した場合:

1. **システムの互換性を確認**: オペレーティング システムが最小要件を満たしていることを確認してください。
2. **ネットワークとファイアウォールの設定を確認してください**: インストーラーが必要なダウンロードに到達でき、ファイアウォールによってブロックされていないことを確認してください。
3. **インストーラーの権限を確認します**: Windows では、十分な権限があるように管理者としてインストーラーを実行してみてください。
4. **アプリを再起動して再試行**:HagiCode デスクトップを閉じて再度開き、インストール手順を再試行します。
5. **ログの表示**: デスクトップ アプリケーションで詳細なエラー ログを確認します。

### サービスの起動失敗

ハギコードサーバーの起動に失敗した場合:

1. **ポート占有の確認**: ポート 45000 が他のプログラムによって占有されていないことを確認します。
2. **ファイアウォール設定を確認してください**: ファイアウォールがHagiCodeサーバーのネットワークアクセスを許可していることを確認してください
3. **アプリケーションの再起動**:HagiCode デスクトップを再起動してみてください
4. **ログの表示**: デスクトップ アプリケーションで詳細なエラー ログを表示します。

### さらにサポートが必要ですか?

ここで説明されていない問題が発生した場合は、次の手順を実行してください。

1. チェックする [GitHubの問題](https://github.com/HagiCode-org/site/issues) 同様の問題に対して
2. 訪問してください [コミュニティフォーラム](https://github.com/HagiCode-org/site/discussions) 助けのために
3. 問題を詳しく説明した新しい問題を送信してください
